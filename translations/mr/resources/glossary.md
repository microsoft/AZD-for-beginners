# शब्दांक - Azure आणि AZD संज्ञाशास्त्र

**सर्व अध्यायांसाठी संदर्भ**

- **📖 मूलभूत गोष्टी शिका**: [अध्याय 1: AZD मूलभूत गोष्टी](../docs/getting-started/azd-basics.md)  
- **🤖 AI संज्ञा**: [अध्याय 2: AI-First Development](../docs/microsoft-foundry/microsoft-foundry-integration.md)  

## परिचय

हा सर्वसमावेशक शब्दांक Azure Developer CLI आणि Azure क्लाउड विकासामध्ये वापरल्या जाणाऱ्या संज्ञा, संकल्पना, आणि संक्षेपांसाठी व्याख्या पुरवतो. तांत्रिक दस्तऐवज समजून घेणे, समस्या निवारण करणे, आणि azd प्रकल्प आणि Azure सेवा याबाबत प्रभावी संवाद साधण्यासाठी आवश्यक संदर्भ.

## शिक्षण उद्दिष्टे

हा शब्दांक वापरून आपल्याला:  
- आवश्यक Azure Developer CLI संज्ञा आणि संकल्पना समजतील  
- Azure क्लाउड विकास संबंधित शब्दसंग्रह आणि तांत्रिक संज्ञावलीत प्रभुत्व मिळेल  
- Infrastructure as Code आणि deployment संदर्भातील संज्ञा कार्यक्षमतेने शोधता येतील  
- Azure सेवा नावे, संक्षेप, आणि त्यांचे उद्दीष्ट समजतील  
- समस्या निवारण आणि डिबगिंग संदर्भातील संज्ञा समजतील  
- प्रगत Azure आर्किटेक्चर आणि विकासाच्या संकल्पना शिकता येतील  

## शिकण्याचे परिणाम

या शब्दांकाचा नियमित संदर्भ घेऊन आपण सक्षम व्हाल:  
- योग्य Azure Developer CLI संज्ञा वापरून प्रभावीपणे संवाद साधणे  
- तांत्रिक दस्तऐवज आणि त्रुटी संदेश स्पष्टपणे समजणे  
- Azure सेवा आणि संकल्पना आत्मविश्वासाने नेव्हिगेट करणे  
- तांत्रिक शब्दावली वापरून समस्या निवारण करणे  
- टीम चर्चांमध्ये अचूक तांत्रिक भाषा वापरून योगदान देणे  
- Azure क्लाउड विकास ज्ञान यथावकाश वाढवणे  

## A

**ARM Template**  
Azure Resource Manager टेम्पलेट. JSON-आधारित Infrastructure as Code फॉर्मॅट जो Azure संसाधने स्पष्टपणे व्याख्यीत आणि तैनात करण्यासाठी वापरला जातो.

**App Service**  
Azure ची platform-as-a-service (PaaS) ऑफरिंग जी वेब अनुप्रयोग, REST API, आणि मोबाइल बॅकएंड होस्ट करण्यासाठी असून इन्फ्रास्ट्रक्चर व्यवस्थापनाची गरज नाही.

**Application Insights**  
Azure ची application performance monitoring (APM) सेवा जी अनुप्रयोग कार्यक्षमता, उपलब्धता, आणि वापराबाबत सखोल माहिती देते.

**Azure CLI**  
Azure संसाधने व्यवस्थापित करण्यासाठी कमांड-लाइन इंटरफेस. azd द्वारे प्रमाणीकरण आणि काही ऑपरेशन्ससाठी वापरली जाते.

**Azure Developer CLI (azd)**  
विकसक-केंद्रित कमांड-लाइन टूल जे Azure साठी अनुप्रयोग तयार करण्याची आणि तैनात करण्याची प्रक्रिया जलद करते, टेम्पलेट्स आणि Infrastructure as Code वापरून.

**azure.yaml**  
azd प्रकल्पासाठी मुख्य कॉन्फिगरेशन फाइल जी सेवा, इन्फ्रास्ट्रक्चर, आणि तैनाती हुक्स परिभाषित करते.

**Azure Resource Manager (ARM)**  
Azure ची तैनाती व व्यवस्थापन सेवा जी संसाधने तयार, अद्यतन, आणि हटवण्यासाठी व्यवस्थापन स्तर पुरवते.

## B

**Bicep**  
Microsoft यांनी विकसित केलेली डोमेन-विशिष्ट भाषा (DSL) जी Azure संसाधने तैनात करण्यासाठी वापरली जाते. ARM टेम्पलेटच्या तुलनेत सोपी सिंटॅक्स असून ती ARM मध्ये संकलित होते.

**Build**  
स्रोत कोड संकलित करण्याची, अवलंबन स्थापित करण्याची, आणि अनुप्रयोग तैनातीसाठी तयार करण्याची प्रक्रिया.

**Blue-Green Deployment**  
तैनाती धोरण ज्यात दोन सारखे उत्पादन वातावरण (निळा आणि हिरवा) वापरून downtime आणि जोखीम कमी केली जाते.

## C

**Container Apps**  
Azure ची सर्व्हरलेस कंटेनर सेवा जी जटिल इन्फ्रास्ट्रक्चर न व्यवस्थापित करता कंटेनरयुक्त अनुप्रयोग चालवण्याची परवानगी देते.

**CI/CD**  
Continuous Integration/Continuous Deployment. कोड बदल एकत्र करण्याची आणि अनुप्रयोग तैनात करण्याची स्वयंचलित पद्धती.

**Cosmos DB**  
Azure ची जागतिक स्तरावर वितरित, मल्टी-मॉडेल डेटाबेस सेवा जी थ्रूपुट, लेटेंसी, उपलब्धता, आणि सुसंगततेसाठी व्यापक SLA पुरवते.

**Configuration**  
अनुप्रयोगाचे वर्तन आणि तैनाती पर्याय नियंत्रित करणाऱ्या सेटिंग्ज आणि पॅरामीटर्स.

## D

**Deployment**  
लक्षित इन्फ्रास्ट्रक्चरवर अनुप्रयोग आणि त्याच्या अवलंबनांची स्थापना आणि कॉन्फिगरेशन करण्याची प्रक्रिया.

**Docker**  
कंटेनरायझेशन तंत्रज्ञान वापरून अनुप्रयोग विकसित करणे, गाडी करणे, आणि चालवण्याचा प्लॅटफॉर्म.

**Dockerfile**  
Docker कंटेनर इमेज तयार करण्यासाठी सूचना असणारा टेक्स्ट फाइल.

## E

**Environment**  
तैनातीसाठी ठराविक अनुप्रयोगाची उदाहरणे (उदा. विकास, स्टेजिंग, उत्पादन).

**Environment Variables**  
रनटाइम दरम्यान अनुप्रयोग प्रवेश करू शकणाऱ्या की-वॅल्यू जोडीत संग्रहित केलेले कॉन्फिगरेशन मूल्ये.

**Endpoint**  
URL किंवा नेटवर्क पत्ता जिथे अनुप्रयोग किंवा सेवा उपलब्ध असते.

## F

**Function App**  
Azure ची सर्व्हरलेस कम्प्यूट सेवा जी इव्हेंट-चालित कोड इन्फ्रास्ट्रक्चर न व्यवस्थापित करता चालवते.

## G

**GitHub Actions**  
GitHub रेपॉजिटरीजसह एकत्रित CI/CD प्लॅटफॉर्म जो वर्कफ्लोज स्वयंचलित करतो.

**Git**  
स्रोत कोडमधील बदल ट्रॅक करण्यासाठी वापरलेली वितरित आवृत्ती नियंत्रण प्रणाली.

## H

**Hooks**  
तैनाती जीवनचक्राच्या विशिष्ट टप्प्यांवर चालवले जाणारे सानुकूल स्क्रिप्ट्स किंवा कमांड्स (preprovision, postprovision, predeploy, postdeploy).

**Host**  
तो Azure सेवा प्रकार जिथे अनुप्रयोग तैनात केला जाईल (उदा. appservice, containerapp, function).

## I

**Infrastructure as Code (IaC)**  
संरचना मॅन्युअल प्रक्रियांऐवजी कोडद्वारे परिभाषित व व्यवस्थापित करण्याचा सराव.

**Init**  
साधारणपणे टेम्पलेटवरून नवीन azd प्रकल्प प्रारंभ करण्याची प्रक्रिया.

## J

**JSON**  
JavaScript Object Notation. कॉन्फिगरेशन फाइल्स आणि API प्रतिसादांसाठी सामान्य डेटा आदानप्रदान फॉर्मॅट.

**JWT**  
JSON Web Token. सुरक्षितपणे माहिती पाठविण्यासाठी JSON ऑब्जेक्ट स्वरूपात मानक.

## K

**Key Vault**  
गुपिते, कीज, आणि प्रमाणपत्र सुरक्षितरित्या संग्रहित आणि व्यवस्थापित करण्यासाठी Azure ची सेवा.

**Kusto Query Language (KQL)**  
Azure Monitor, Application Insights, आणि इतर Azure सेवांमध्ये डेटा विश्लेषणासाठी वापरली जाणारी क्वेरी भाषा.

## L

**Load Balancer**  
आगमक नेटवर्क ट्रॅफिक अनेक सर्व्हर किंवा उदाहरणांमध्ये वितरित करणारी सेवा.

**Log Analytics**  
क्लाउड आणि ऑन-प्रिमाइसेस वातावरणातून टेलीमेट्री डेटा गोळा करणे, विश्लेषण करणे, आणि त्यावर कृती करण्यासाठी Azure सेवा.

## M

**Managed Identity**  
इतर Azure सेवा प्रमाणित करण्यासाठी ऑटोमॅटिकली व्यवस्थापित ओळख Azure सेवा प्रदान करते.

**Microservices**  
संरचनात्मक पद्धत जिथे अनुप्रयोग लहान, स्वतंत्र सेवांच्या संग्रहाप्रमाणे बांधले जातात.

**Monitor**  
Azure ची एकात्मिक मॉनिटरिंग सोल्युशन जी अनुप्रयोग आणि इन्फ्रास्ट्रक्चरवर पूर्ण-स्तरीय निरीक्षण देते.

## N

**Node.js**  
Chrome च्या V8 JavaScript इंजिनवर बांधलेले सर्व्हर-साइड अनुप्रयोग तयार करण्यासाठी JavaScript रनटाइम.

**npm**  
Node.js साठी पॅकेज व्यवस्थापक जो अवलंबन्स आणि पॅकेजेस व्यवस्थापित करतो.

## O

**Output**  
इन्फ्रास्ट्रक्चर तैनातीतून परत येणारे मूल्ये जी अनुप्रयोग किंवा इतर संसाधनांनी वापरता येतात.

## P

**Package**  
अॅप्लिकेशन कोड आणि अवलंबने तैनातीसाठी तयार करण्याची प्रक्रिया.

**Parameters**  
तैनाती सानुकूल करण्यासाठी इन्फ्रास्ट्रक्चर टेम्पलेटमध्ये दिली जाणारी इनपुट मूल्ये.

**PostgreSQL**  
Azure मध्ये व्यवस्थापित सेवा म्हणून समर्थित मुक्त स्रोत रिलेशनल डेटाबेस सिस्टीम.

**Provisioning**  
इन्फ्रास्ट्रक्चर टेम्पलेटमध्ये परिभाषित केलेल्या Azure संसाधनांची निर्मिती आणि कॉन्फिगरेशन प्रक्रिया.

## Q

**Quota**  
Azure सदस्यत्व किंवा विभागात तयार होणाऱ्या संसाधनांवरील मर्यादा.

## R

**Resource Group**  
Azure संसाधने ज्यांना समान जीवनचक्र, परवानग्या, आणि धोरणे असतात त्यांचा तार्किक कंटेनर.

**Resource Token**  
azd द्वारे तयार केलेला अद्वितीय स्ट्रिंग जे तैनातींमध्ये संसाधन नावे अद्वितीय ठेवण्यास मदद करतो.

**REST API**  
नेटवर्केड अनुप्रयोगांसाठी HTTP पद्धती वापरून डिझाइन केलेली आर्किटेक्चरल शैली.

**Rollback**  
अॅप्लिकेशन किंवा इन्फ्रास्ट्रक्चर कॉन्फिगरेशनचा मागील आवृत्तीवर परत जाण्याची प्रक्रिया.

## S

**Service**  
तुमच्या अनुप्रयोगाचा एक घटक जो azure.yaml मध्ये परिभाषित केला जातो (उदा. वेब फ्रंटएंड, API बॅकेंड, डेटाबेस).

**SKU**  
Stock Keeping Unit. Azure संसाधनांसाठी वेगवेगळ्या सेवा स्तर किंवा कार्यक्षमतेची पातळी.

**SQL Database**  
Microsoft SQL Server आधारित Azure ची व्यवस्थापित रिलेशनल डेटाबेस सेवा.

**Static Web Apps**  
स्त्रोत कोड रेपॉजिटरीजमधून पूर्ण-संरचना वेब अनुप्रयोग तयार आणि तैनात करण्यासाठी Azure सेवा.

**Storage Account**  
ब्लॉब्स, फाइल्स, क्यूज, आणि टेबल समाविष्ट डेटा वस्तूंसाठी Azure क्लाउड स्टोरेज सेवा.

**Subscription**  
Azure खाते कंटेनर जे संसाधन गट आणि संसाधने धारून ठेवते, बिलिंग आणि प्रवेश व्यवस्थापनासह.

## T

**Template**  
अॅप्लिकेशन कोड, इन्फ्रास्ट्रक्चर व्याख्या, आणि कॉन्फिगरेशनसह सामान्य परिस्थितींसाठी पूर्वनिर्मित प्रकल्प संरचना.

**Terraform**  
एक खुले स्रोत Infrastructure as Code साधन जे Azure समवेत अनेक क्लाउड प्रदात्यांना समर्थन देते.

**Traffic Manager**  
जागतिक Azure विभागांमध्ये ट्रॅफिक वितरणासाठी Azure ची DNS-आधारित ट्रॅफिक लोड बॅलेंसर सेवा.

## U

**URI**  
Uniform Resource Identifier. विशिष्ट संसाधन ओळखण्यासाठी स्ट्रिंग.

**URL**  
Uniform Resource Locator. संसाधन कुठे आहे आणि कसे प्राप्त करायचे याचे वर्णन करणारा URI प्रकार.

## V

**Virtual Network (VNet)**  
Azure मध्ये खाजगी नेटवर्कसाठी मूलभूत बांधकाम घटक, जो पृथक्करण व विभागणी पुरवतो.

**VS Code**  
Visual Studio Code. अत्युत्तम Azure आणि azd एकत्रिकरणासह लोकप्रिय कोड संपादक.

## W

**Webhook**  
विशिष्ट इव्हेंट्सद्वारे सुरू होणारा HTTP कॉलबॅक, मुख्यतः CI/CD पाईपलाइन्समध्ये वापरला जातो.

**What-if**  
Azure वैशिष्ट्य जे तैनातीनंतर कोणते बदल होतील ते दाखवते, पण खरोखर अंमलात आणत नाही.

## Y

**YAML**  
YAML Ain't Markup Language. मानव-पठनीय डेटा सिरीयलायझेशन मानक जे azure.yaml सारख्या कॉन्फिगरेशन फाइल्ससाठी वापरले जाते.

## Z

**Zone**  
Azure विभागातील भौतिकरीत्या वेगवेगळ्या ठिकाणे जी पुनरावृत्ती आणि उच्च उपलब्धता पुरवतात.

## संदर्भानुसार संज्ञा

### विकास संज्ञा  
- **Hot Reload**: विकासादरम्यान अनुप्रयोग पुन्हा सुरू न करता स्वयंचलित अद्ययावत करणे  
- **Build Pipeline**: कोड तयार करण्याची आणि चाचणी करण्याची स्वयंचलित प्रक्रिया  
- **Deployment Slot**: App Service मध्ये स्टेजिंग वातावरण  
- **Environment Parity**: विकास, स्टेजिंग, आणि उत्पादन वातावरणांची समानता राखणे  

### सुरक्षा संज्ञा  
- **Managed Identity**: ऑटोमॅटिक क्रेडेंशियल व्यवस्थापनासाठी Azure वैशिष्ट्य  
- **Key Vault**: गुपिते, कीज, आणि प्रमाणपत्र सुरक्षित साठवण आणि व्यवस्थापन  
- **RBAC**: Azure संसाधनांसाठी भूमिका-आधारित प्रवेश नियंत्रण  
- **Network Security Group**: नेटवर्क ट्रॅफिक नियंत्रित करण्यासाठी व्हर्चुअल फायरवॉल  

### निरीक्षण संज्ञा  
- **Telemetry**: मोजमापे आणि डेटा स्वयंचलितपणे गोळा करणे  
- **Application Performance Monitoring (APM)**: सॉफ्टवेअर कार्यक्षमतेचे निरीक्षण  
- **Log Analytics**: लॉग डेटा गोळा करणे आणि विश्लेषण करण्यासाठी सेवा  
- **Alert Rules**: मेट्रिक्स किंवा स्थितीवर आधारित स्वयंचलित सूचना  

### तैनाती संज्ञा  
- **Blue-Green Deployment**: झिरो-डाउनटाइम तैनाती धोरण  
- **Canary Deployment**: वापरकर्त्यांच्या उपसमूहावर क्रमागत रोलआउट  
- **Rolling Update**: क्रमवार अनुप्रयोग उदाहरणे बदलणे  
- **Rollback**: मागील अनुप्रयोग आवृत्तीवर परत जाणे
