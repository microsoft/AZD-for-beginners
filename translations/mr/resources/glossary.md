# शब्दसंग्रह - Azure आणि AZD शब्दावली

**सर्व अध्यायांसाठी संदर्भ**
- **📚 कोर्स होम**: [AZD For Beginners](../README.md)
- **📖 मूलभूत गोष्टी शिका**: [अध्याय 1: AZD मूलभूत गोष्टी](../docs/getting-started/azd-basics.md)
- **🤖 AI संज्ञा**: [अध्याय 2: AI-First Development](../docs/microsoft-foundry/microsoft-foundry-integration.md)

## प्रस्तावना

हा संपूर्ण शब्दसंग्रह Azure Developer CLI आणि Azure क्लाउड डेव्हलपमेंटमध्ये वापरल्या जाणाऱ्या संज्ञा, संकल्पना आणि संक्षेपांकरिता व्याख्या प्रदान करतो. तांत्रिक दस्तऐवज समजून घेणे, समस्यांचे निराकरण करणे आणि azd प्रकल्प व Azure सेवा याबाबत प्रभावीपणे संवाद साधण्यासाठी आवश्यक संदर्भ आहे.

## शिकण्याचे उद्दिष्ट

हा शब्दसंग्रह वापरून, तुम्ही:
- Azure Developer CLI च्या आवश्यक शब्दावली आणि संकल्पना समजून घेऊ शकता
- Azure क्लाउड डेव्हलपमेंटची शब्दसंग्रह व तांत्रिक संज्ञा आत्मसात करू शकता
- Infrastructure as Code आणि डिप्लॉयमेंटशी संबंधित संज्ञा प्रभावीपणे संदर्भित करू शकता
- Azure सेवा नावे, संक्षेप आणि त्यांचे उद्दिष्ट समजू शकता
- समस्या निवारण आणि डीबगिंगसाठी संज्ञांची व्याख्या मिळवू शकता
- प्रगत Azure आर्किटेक्चर व विकास संकल्पना शिकू शकता

## शिकण्याचे परिणाम

या शब्दसंग्रहाचा नियमित संदर्भ घेऊन, तुम्ही:
- योग्य Azure Developer CLI शब्दावली वापरून प्रभावीपणे संवाद साधू शकता
- तांत्रिक दस्तऐवज आणि त्रुटी संदेश अधिक स्पष्टपणे समजू शकता
- आत्मनिर्भरपणे Azure सेवा व संकल्पना वापरू शकता
- योग्य तांत्रिक शब्दसंग्रह वापरून समस्या निराकरण करू शकता
- अचूक तांत्रिक भाषेत टीम चर्चा मध्ये योगदान देऊ शकता
- Azure क्लाउड डेव्हलपमेंटचे ज्ञान सुसंगतपणे वाढवू शकता

## A

**ARM Template**  
Azure Resource Manager टेम्प्लेट. JSON-आधारित Infrastructure as Code स्वरूप जे Azure संसाधने घोषणात्मकपणे परिभाषित व तैनातीसाठी वापरले जाते.

**App Service**  
Azure चा platform-as-a-service (PaaS) सेवा, जो वेब अनुप्रयोग, REST API, आणि मोबाइल बॅकएंड होस्ट करण्यासाठी इन्फ्रास्ट्रक्चर व्यवस्थापनाशिवाय उपलब्ध आहे.

**Application Insights**  
Azure ची application performance monitoring (APM) सेवा जी अनुप्रयोग कामगिरी, उपलब्धता, आणि वापर यावर खोलवर माहिती देते.

**Azure CLI**  
Azure संसाधने व्यवस्थापित करण्यासाठी कमांड-लाइन इंटरफेस. azd द्वारे प्रमाणीकरण आणि काही ऑपरेशन्ससाठी वापरली जाते.

**Azure Developer CLI (azd)**  
विकसक-केंद्रित कमांड-लाइन टूल जे टेम्प्लेट्स आणि Infrastructure as Code वापरून Azure वर अनुप्रयोग बांधणी आणि तैनात करण्याची प्रक्रिया वेगवान करते.

**azure.yaml**  
azd प्रकल्पासाठी मुख्य कॉन्फिगरेशन फाइल जी सेवा, इन्फ्रास्ट्रक्चर आणि तैनाती हुक्स परिभाषित करते.

**Azure Resource Manager (ARM)**  
Azure चा तैनाती व व्यवस्थापन सेवा जो संसाधने तयार, अद्यतनित आणि हटवण्यासाठी व्यवस्थापन स्तर प्रदान करतो.

## B

**Bicep**  
Microsoft यांनी विकसित केलेली डोमेन-विशिष्ट भाषा (DSL) ज्याचा वापर Azure संसाधने तैनात करण्यासाठी होतो. ARM टेम्प्लेट्सच्या तुलनेत सोपी सिंटॅक्स देते आणि ARM मध्ये कम्पाईल होते.

**Build**  
स्रोत कोड संकलित करणे, अवलंबित्वे इन्स्टॉल करणे, आणि अनुप्रयोग तैनातीसाठी तयार करण्याची प्रक्रिया.

**Blue-Green Deployment**  
तैनाती धोरण जे दोन समान उत्पादन वातावरण (ब्लू आणि ग्रीन) वापरून डाउनटाइम आणि धोका कमी करते.

## C

**Container Apps**  
Azure ची सर्व्हरलेस कंटेनर सेवा जी जटील इन्फ्रास्ट्रक्चर व्यवस्थापनाशिवाय कंटेनरयुक्त अनुप्रयोग चालवण्याची सुविधा देते.

**CI/CD**  
Continuous Integration/Continuous Deployment. कोड बदल एकत्रित करण्यासाठी व अनुप्रयोग तैनात करण्यासाठी स्वयंचलित पद्धती.

**Cosmos DB**  
Azure ची जागतिक पातळीवर वितरित, मल्टी-मॉडेल डेटाबेस सेवा जी थ्रूपुट, लेटन्सी, उपलब्धता आणि सुसंगततेसाठी व्यापक SLA प्रदान करते.

**Configuration**  
अ‍ॅप्लिकेशन वर्तन आणि तैनाती पर्याय नियंत्रित करण्यासाठी सेटिंग्ज व पॅरामीटर्स.

## D

**Deployment**  
लक्ष्य इन्फ्रास्ट्रक्चरवर अनुप्रयोग आणि त्यांचे अवलंबित्व स्थापित व संरचीत करण्याची प्रक्रिया.

**Docker**  
कंटेनरीकरण तंत्रज्ञानाचा वापर करून अनुप्रयोग विकसित, वितरण आणि चालवण्याचे प्लॅटफॉर्म.

**Dockerfile**  
Docker कंटेनर इमेज तयार करण्यासाठी सूचना असलेली मजकूर फाइल.

## E

**Environment**  
तुमच्या अनुप्रयोगाचा विशिष्ट उदाहरण दाखवणारे तैनाती लक्ष्य (उदा., विकास, स्टेजिंग, उत्पादन).

**Environment Variables**  
रनटाइम दरम्यान अनुप्रयोगांना प्रवेश करता येणारी मुख्य-मूल्य जोड्यांच्या स्वरूपातील कॉन्फिगरेशन मूल्ये.

**Endpoint**  
URL किंवा नेटवर्क पत्ता जिथे अनुप्रयोग किंवा सेवा प्रवेशयोग्य आहे.

## F

**Function App**  
Azure ची सर्व्हरलेस कम्प्युट सेवा जी इव्हेंट-चालित कोड इन्फ्रास्ट्रक्चर व्यवस्थापनाशिवाय चालवते.

## G

**GitHub Actions**  
स्वयंचलित वर्कफ्लो साठी GitHub रेपॉझिटरीजमध्ये समाकलित CI/CD प्लॅटफॉर्म.

**Git**  
स्रोत कोडमधील बदल ट्रॅक करण्यासाठी वापरली जाणारी वितरित आवृत्ती नियंत्रण प्रणाली.

## H

**Hooks**  
डिप्लॉयमेंट जीवनचक्रातील विशिष्ट टप्प्यावर चालणारे कस्टम स्क्रिप्ट किंवा आदेश (preprovision, postprovision, predeploy, postdeploy).

**Host**  
अ‍ॅप्लिकेशन तैनात होणारी Azure सेवा प्रकार (उदा., appservice, containerapp, function).

## I

**Infrastructure as Code (IaC)**  
इन्फ्रास्ट्रक्चर मॅन्युअल प्रक्रियांच्या ऐवजी कोडद्वारे व्याख्या आणि व्यवस्थापन करण्याचा सराव.

**Init**  
नवीन azd प्रकल्प प्रारंभ करण्याची प्रक्रिया, सहसा टेम्प्लेटपासून.

## J

**JSON**  
JavaScript Object Notation. कॉन्फिगरेशन फाइल्स आणि API प्रतिसादांसाठी सामान्यपणे वापरली जाणारी डेटा विनिमय फॉर्मॅट.

**JWT**  
JSON Web Token. JSON ऑब्जेक्ट म्हणून सुरक्षित माहिती ट्रान्समिट करण्यासाठीचा मानक.

## K

**Key Vault**  
गुप्त माहिती, कीज, आणि प्रमाणपत्र सुरक्षितपणे साठवण्यासाठी Azure ची सेवा.

**Kusto Query Language (KQL)**  
Azure Monitor, Application Insights, आणि इतर Azure सेवा मध्ये डेटा विश्लेषणासाठी वापरली जाणारी क्वेरी भाषा.

## L

**Load Balancer**  
एकाधिक सर्व्हर किंवा उदाहरणांमध्ये येणारा नेटवर्क ट्रॅफिक विभागणारी सेवा.

**Log Analytics**  
क्लाउड आणि ऑन-प्रिमायसेस वातावरणातून दूरस्थ माहिती संकलित, विश्लेषण आणि कारवाई करण्याची सेवा.

## M

**Managed Identity**  
Azure सेवा दुसऱ्या Azure सेवा प्रमाणित करण्यासाठी स्वयंचलित व्यवस्थापन केलेली ओळख.

**Microservices**  
मुक्त, स्वतंत्र सेवांचा संच बनवून अनुप्रयोग तयार करण्याची आर्किटेक्चरल पद्धत.

**Monitor**  
Azure चे एकत्रित मॉनिटरिंग सोल्यूशन जे अनुप्रयोग आणि इन्फ्रास्ट्रक्चरची पूर्ण स्टॅक निरीक्षणे प्रदान करते.

## N

**Node.js**  
Chrome च्या V8 JavaScript इंजिनवर आधारित JavaScript रनटाइम जो सर्व्हर-साइड अनुप्रयोग तयार करण्यासाठी वापरला जातो.

**npm**  
Node.js साठी पॅकेज व्यवस्थापक जो अवलंबित्वे आणि पॅकेजेस व्यवस्थापित करतो.

## O

**Output**  
इन्फ्रास्ट्रक्चर तैनातीतून परत आलेली मूल्ये जी अनुप्रयोग किंवा इतर संसाधनांनी वापरू शकतात.

## P

**Package**  
अनुप्रयोगाचा कोड आणि अवलंबित्वे तैनातीसाठी तयार करण्याची प्रक्रिया.

**Parameters**  
तैनाती सानुकूल करण्यासाठी इन्फ्रास्ट्रक्चर टेम्प्लेट्सना दिली जाणारी इनपुट मूल्ये.

**PostgreSQL**  
Azure मध्ये व्यवस्थापित सेवा म्हणून समर्थित मुक्त स्रोत रिलेशनल डेटाबेस सिस्टम.

**Provisioning**  
इन्फ्रास्ट्रक्चर टेम्प्लेट्समध्ये परिभाषित Azure संसाधने तयार व संरचीत करण्याची प्रक्रिया.

## Q

**Quota**  
Azure सबस्क्रिप्शन किंवा प्रदेशात तयार करता येणाऱ्या संसाधनांच्या मर्यादा.

## R

**Resource Group**  
Azure संसाधने जे समान जीवनचक्र, अनुमती, आणि धोरणे सामायिक करतात त्यासाठी लॉजिकल कंटेनर.

**Resource Token**  
तैनाती दरम्यान संसाधन नामे अनन्य करतील यासाठी azd द्वारे तयार केलेली अनन्य स्ट्रिंग.

**REST API**  
HTTP पद्धती वापरून नेटवर्क अनुप्रयोग डिझाइन करण्याची आर्किटेक्चरल शैली.

**Rollback**  
अनुप्रयोग किंवा इन्फ्रास्ट्रक्चर कॉन्फिगरेशनच्या आधीच्या आवृत्तीवर परत जाण्याची प्रक्रिया.

## S

**Service**  
azure.yaml मध्ये परिभाषित तुमच्या अनुप्रयोगाचा घटक (उदा., वेब फ्रंटएंड, API बॅकएंड, डेटाबेस).

**SKU**  
Stock Keeping Unit. Azure संसाधनांसाठी वेगवेगळ्या सेवा स्तर किंवा कामगिरी स्तर दर्शवते.

**SQL Database**  
Microsoft SQL Server आधारित Azure ची व्यवस्थापित रिलेशनल डेटाबेस सेवा.

**Static Web Apps**  
स्त्रोत कोड रेपॉझिटरीजमधून पूर्ण-स्टॅक वेब अनुप्रयोग तयार व तैनात करण्यासाठी Azure सेवा.

**Storage Account**  
Azure सेवा जी ब्लॉब्स, फाइल्स, क्यूज, आणि टेबल्स यांसह डेटा वस्तूंसाठी क्लाउड स्टोरेज देते.

**Subscription**  
Azure खाते कंटेनर ज्यामध्ये संसाधन गट आणि संसाधने समाविष्ट असल्याने बिलिंग आणि प्रवेश व्यवस्थापन होते.

## T

**Template**  
सामान्य परिस्थितींसाठी अनुप्रयोग कोड, इन्फ्रास्ट्रक्चर व्याख्या, आणि कॉन्फिगरेशनसह पूर्वनिर्मित प्रकल्प रचना.

**Terraform**  
मुक्त स्रोत Infrastructure as Code साधन जे Azureसह अनेक क्लाउड प्रोव्हायडर्सना समर्थन देते.

**Traffic Manager**  
जागतिक Azure प्रदेशांमध्ये ट्रॅफिक वाटप करण्यासाठी Azure चे DNS-आधारित ट्रॅफिक लोड बॅलेंसर.

## U

**URI**  
Uniform Resource Identifier. विशिष्ट संसाधन ओळखणारी स्ट्रिंग.

**URL**  
Uniform Resource Locator. संसाधन कुठे आहे आणि ते कसे मिळवायचे हे निर्दिष्ट करणाऱ्या URI प्रकार.

## V

**Virtual Network (VNet)**  
Azure मधील खाजगी नेटवर्कसाठी मूलभूत घटक जो वेगळेपणा आणि विभागणी पुरवतो.

**VS Code**  
Visual Studio Code. उत्कृष्ट Azure आणि azd समाकलन असलेला लोकप्रिय कोड संपादनकर्ता.

## W

**Webhook**  
विशिष्ट घटनांवर ट्रिगर होणारा HTTP कॉलबॅक, सामान्यपणे CI/CD पाईपलाईन्समध्ये वापरला जातो.

**What-if**  
Azure वैशिष्ट्य जे वास्तविक अंमलबजावणी शिवाय तैनातीद्वारे होणारे बदल दाखवते.

## Y

**YAML**  
YAML Ain't Markup Language. मुक्तपणे वाचता येणारा डेटा सिरीयलायझेशन मानक जो azure.yaml सारख्या कॉन्फिगरेशन फाइलसाठी वापरला जातो.

## Z

**Zone**  
Azure प्रदेशातील भौतिकदृष्ट्या वेगळ्या ठिकाणे जी पुनरावृत्ती आणि उच्च उपलब्धता प्रदान करतात.

## संदर्भ-विशिष्ट संज्ञा

### विकास संज्ञा
- **Hot Reload**: पुन्हा सुरु न करता विकासादरम्यान अनुप्रयोग स्वयंचलित अद्यतनित करणे
- **Build Pipeline**: कोड बांधणी आणि चाचणीसाठी स्वयंचलित प्रक्रिया
- **Deployment Slot**: App Service मधील स्टेजिंग वातावरण
- **Environment Parity**: विकास, स्टेजिंग, आणि उत्पादन वातावरण समान ठेवणे

### सुरक्षा संज्ञा
- **Managed Identity**: स्वयंचलित क्रेडेन्शियल व्यवस्थापन प्रदान करणारी Azure वैशिष्ट्ये
- **Key Vault**: गुप्त, की आणि प्रमाणपत्रांसाठी सुरक्षित संचय
- **RBAC**: Azure संसाधनांसाठी भूमिका-आधारित प्रवेश नियंत्रण
- **Network Security Group**: नेटवर्क ट्रॅफिक नियंत्रित करण्यासाठी वर्च्युअल फायरवॉल

### मॉनिटरिंग संज्ञा
- **Telemetry**: मोजमाप आणि डेटा स्वयंचलित संकलन
- **Application Performance Monitoring (APM)**: सॉफ्टवेअर कामगिरीचे निरीक्षण
- **Log Analytics**: लॉग डेटा संकलित व विश्लेषित करण्याची सेवा
- **Alert Rules**: मेट्रिक्स किंवा स्थितींवर आधारित स्वयंचलित सूचना

### तैनाती संज्ञा
- **Blue-Green Deployment**: झिरो-डाउनटाइम तैनाती धोरण
- **Canary Deployment**: वापरकर्त्यांच्या संख्येचा हळूहळू विस्तार
- **Rolling Update**: अनुप्रयोग उदाहरणांची सलग बदलती तैनाती
- **Rollback**: पूर्वीच्या अनुप्रयोग आवृत्तीवर परत जाणे
