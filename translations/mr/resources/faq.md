## परिचय

हा व्यापक FAQ Azure Developer CLI (azd) आणि Azure तैनातींबाबत सर्वात सामान्य प्रश्नांची उत्तरे देतो. सामान्य समस्या सोडवण्याचे त्वरित उपाय शोधा, सर्वोत्तम पद्धती समजून घ्या, आणि azd कल्पना व कार्यप्रवाहांवर स्पष्टता मिळवा.

## सुरूवात

### प्रश्न: Azure Developer CLI (azd) काय आहे?
**उत्तर**: Azure Developer CLI (azd) हा एक विकसक-केंद्रित कमांड-लाइन साधन आहे जो आपल्या अॅप्लिकेशनला स्थानिक विकास पर्यावरणातून Azure पर्यंत पोहोचवण्याचा वेळ वेगवान करतो. तो साच्यांद्वारे सर्वोत्तम पद्धती प्रदान करतो आणि संपूर्ण तैनातीच्या जीवनचक्रात मदत करतो.

### प्रश्न: azd आणि Azure CLI यामध्ये काय फरक आहे?
**उत्तर**: 
- **Azure CLI**: Azure स्रोत व्यवस्थापनेसाठी सर्वसाधारण वापराचे साधन
- **azd**: अॅप्लिकेशन तैनातीच्या कार्यप्रवाहांसाठी विकसक-केंद्रित साधन
- azd Azure CLI अंतर्गत वापरते परंतु सामान्य विकास परिस्थितींना उच्च-स्तरीय अमूर्तता देते
- azd मध्ये साचे, वातावरण व्यवस्थापन, आणि तैनाती स्वयंचलन यांचा समावेश आहे

### प्रश्न: azd वापरण्यासाठी मला Azure CLI स्थापित करावा लागतो का?
**उत्तर**: होय, azd साठी प्रमाणीकरण आणि काही कार्यांसाठी Azure CLI आवश्यक आहे. प्रथम Azure CLI स्थापित करा, नंतर azd स्थापित करा.

### प्रश्न: azd कोणत्या प्रोग्रामिंग भाषांना समर्थन देते?
**उत्तर**: azd भाषा-निरपेक्ष आहे. ते खालील भाषांसह कार्य करते:
- Node.js/JavaScript/TypeScript
- Python
- .NET/C#
- Java
- Go
- PHP
- स्थिर वेबसाईट्स
- कंटेनरयुक्त अॅप्लिकेशन्स

### प्रश्न: मी azd विद्यमान प्रकल्पांसह वापरू शकतो का?
**उत्तर**: होय! आपण:
1. `azd init` वापरून विद्यमान प्रकल्पांमध्ये azd कॉन्फिगरेशन जोडू शकता
2. विद्यमान प्रकल्पांना azd साचे संरचनेशी जुळवून घेऊ शकता
3. आपल्या विद्यमान आर्किटेक्चरवर आधारित सानुकूल साचे तयार करू शकता

## प्रमाणीकरण व प्रवेश

### प्रश्न: azd वापरून Azure शी कसे प्रमाणीकरण करावे?
**उत्तर**: `azd auth login` वापरा, जे Azure प्रमाणीकरणासाठी ब्राउझर विंडो उघडेल. CI/CD परिस्थितीसाठी, सेवा प्रमुख किंवा व्यवस्थापित ओळख वापरा.

### प्रश्न: मी azd एकापेक्षा अधिक Azure सदस्यत्त्वांसह वापरू शकतो का?
**उत्तर**: होय. प्रत्येक वातावरणासाठी `azd env set AZURE_SUBSCRIPTION_ID <subscription-id>` वापरून सदस्यत्त्व निवडा.

### प्रश्न: azd सह तैनात करण्यासाठी मला कोणती परवानगी आवश्यक आहे?
**उत्तर**: सामान्यतः आपल्याला हवे:
- स्रोत गट किंवा सदस्यत्त्वावर **Contributor** भूमिका
- भूमिका नियुक्ती आवश्यक असलेल्या संसाधनांसाठी **User Access Administrator**
- विशिष्ट साचे आणि तैनात संसाधनांनुसार परवानग्या वेगळ्या असू शकतात

### प्रश्न: मी azd CI/CD पाइपलाइनमध्ये वापरू शकतो का?
**उत्तर**: नक्कीच! azd CI/CD समाकलनासाठी डिझाइन केले गेले आहे. प्रमाणीकरणासाठी सेवा प्रमुख वापरा आणि कॉन्फिगरेशनसाठी वातावरणीय बदल सेट करा.

### प्रश्न: GitHub Actions मध्ये प्रमाणीकरण कसे हाताळावे?
**उत्तर**: Azure Login क्रिया सेवा प्रमुख प्रमाणपत्रांसह वापरा:
```yaml
- uses: azure/login@v1
  with:
    creds: ${{ secrets.AZURE_CREDENTIALS }}
- run: azd deploy --no-prompt
```

## साचे व प्रकल्प

### प्रश्न: azd साचे कुठे मिळवू शकतो?
**उत्तर**: 
- अधिकृत साचे: [Azure-Samples/awesome-azd](https://github.com/Azure-Samples/awesome-azd)
- समुदाय साचे: GitHub वर "azd-template" शोधा
- उपलब्ध साचे पाहण्यासाठी `azd template list` वापरा

### प्रश्न: कस्टम साचा कसा तयार करावा?
**उत्तर**: 
1. विद्यमान साचा रचनेपासून सुरू करा
2. `azure.yaml`, पायाभूत सुविधा फायली, आणि अनुप्रयोग कोड बदल करा
3. `azd up` ने चांगले चाचणी करा
4. योग्य टॅगसह GitHub वर प्रकाशित करा

### प्रश्न: मी साचा न वापरता azd वापरू शकतो का?
**उत्तर**: होय, विद्यमान प्रकल्पात `azd init` वापरून आवश्यक कॉन्फिगरेशन फायली तयार करा. आपल्याला `azure.yaml` आणि पायाभूत सुविधा फायली मॅन्युअली कॉन्फिगर कराव्या लागतील.

### प्रश्न: अधिकृत आणि समुदाय साच्यांमध्ये काय फरक आहे?
**उत्तर**: 
- **अधिकृत साचे**: Microsoft द्वारे देखरेख, नियमित अद्ययावत, व्यापक दस्तऐवजीकरण
- **समुदाय साचे**: विकसकांनी तयार केलेले, विशिष्ट वापर प्रकरणे, गुणवत्ता व देखभालीत फरक

### प्रश्न: माझ्या प्रकल्पातील साचा कसा अद्यतनीत करावा?
**उत्तर**: साचे स्वयंचलितपणे अद्यतनित होत नाहीत. आपण:
1. स्त्रोत साच्यातील बदल मॅन्युअली तुलना व एकत्र करा
2. नव्या साच्यासह `azd init` वापरून ताजी सुरुवात करा
3. अद्ययावत साच्यांमधून विशिष्ट सुधारणा निवडा

## तैनाती व पायाभूत सुविधा

### प्रश्न: azd कोणत्या Azure सेवा तैनात करू शकतो?
**उत्तर**: azd Bicep/ARM साच्यांद्वारे कोणतीही Azure सेवा तैनात करू शकतो, जसे की:
- App Services, Container Apps, Functions
- डेटाबेस (SQL, PostgreSQL, Cosmos DB)
- स्टोरेज, Key Vault, Application Insights
- नेटवर्किंग, सुरक्षा, व निरीक्षण स्रोत

### प्रश्न: मी एकाहून अधिक प्रदेशांमध्ये तैनात करू शकतो का?
**उत्तर**: होय, आपल्या Bicep साच्यात एकाधिक प्रदेश कॉन्फिगर करा आणि प्रत्येक वातावरणासाठी योग्य location parameter सेट करा.

### प्रश्न: डेटाबेस स्कीमा माइग्रेशन कसे हाताळावे?
**उत्तर**: `azure.yaml` मध्ये तैनाती हुक वापरा:
```yaml
hooks:
  postdeploy:
    posix:
      run: ./scripts/migrate-database.sh
    windows:
      run: ./scripts/migrate-database.ps1
```

### प्रश्न: मी फक्त पायाभूत सुविधा तैनात करू शकतो का अनुप्रयोगांशिवाय?
**उत्तर**: होय, आपल्या साच्यामध्ये परिभाषित फक्त पायाभूत सुविधा घटक तैनात करण्यासाठी `azd provision` वापरा.

### प्रश्न: विद्यमान Azure संसाधनांवर मी कसे तैनात करणार?
**उत्तर**: हे क्लिष्ट असून थेट समर्थित नाही. आपण:
1. विद्यमान संसाधने Bicep साच्यात आयात करा
2. साच्यांमध्ये विद्यमान संसाधन संदर्भ वापरा
3. साचे संसाधने तयार करणे किंवा संदर्भ देत अस्थिर/सशर्त करा

### प्रश्न: Bicep ऐवजी Terraform वापरू शकतो का?
**उत्तर**: सध्या azd मुख्यतः Bicep/ARM साच्यांना समर्थन देतो. Terraform अधिकृतपणे समर्थित नाही, पण समुदायातील काही उपाय असू शकतात.

## कॉन्फिगरेशन व वातावरण

### प्रश्न: वेगवेगळ्या वातावरणांचे (dev, staging, prod) व्यवस्थापन कसे करावे?
**उत्तर**: `azd env new <environment-name>` वापरून वेगळे वातावरण तयार करा आणि प्रत्येकासाठी वेगळ्या सेटिंग्ज कॉन्फिगर करा:
```bash
azd env new development
azd env new staging  
azd env new production
```

### प्रश्न: वातावरण कॉन्फिगरेशन कुठे साठवले जाते?
**उत्तर**: आपल्या प्रोजेक्ट फोल्डरमधील `.azure` फोल्डरमध्ये. प्रत्येक वातावरणाचा स्वतःचा फोल्डर आणि कॉन्फिगरेशन फायली असतात.

### प्रश्न: विशिष्ट वातावरणासाठी कॉन्फिगरेशन कसे सेट करावे?
**उत्तर**: वातावरणीय चल मायक्रोसेट करण्यासाठी `azd env set` वापरा:
```bash
azd env set AZURE_LOCATION eastus
azd env set DATABASE_TIER Basic
```

### प्रश्न: मी टीम सदस्यांमध्ये वातावरण कॉन्फिगरेशन शेअर करू शकतो का?
**उत्तर**: `.azure` फोल्डर संवेदनशील माहिती ठेवतो आणि तो व्हर्जन कंट्रोलमध्ये कमिट करू नका. त्याऐवजी:
1. आवश्यक वातावरण चल दस्तऐवज करा
2. तैनाती स्क्रिप्ट्स वापरून वातावरण सेट करा
3. संवेदनशील कॉन्फिगरेशनसाठी Azure Key Vault वापरा

### प्रश्न: मी साचा डीफॉल्ट्स कसे ओव्हरराईड करू?
**उत्तर**: साचा पॅरामीटर्सशी संबंधित वातावरण चल सेट करा:
```bash
azd env set LOCATION "West US 2"
azd env set SKU_NAME "B1"
```

## समस्या निराकरण

### प्रश्न: `azd up` का अयशस्वी होत आहे?
**उत्तर**: सामान्य कारणे:
1. **प्रमाणीकरण समस्या**: `azd auth login` चालवा
2. **पुरेशी परवानगी नाही**: Azure भूमिका तपासा
3. **संसाधन नावे संघर्ष**: AZURE_ENV_NAME बदला
4. **कोटा/क्षमता मर्यादा**: प्रादेशिक उपलब्धता तपासा
5. **साचा त्रुटी**: Bicep साचा वैधता तपासा

### प्रश्न: तैनाती अपयश कसे डीबग करावेत?
**उत्तर**: 
1. तपशीलवार आउटपुटसाठी `azd deploy --debug` वापरा
2. Azure पोर्टल में तैनाती इतिहास तपासा
3. Azure पोर्टलमधील Activity Log पहा
4. सध्याच्या वातावरण स्थितीला `azd show` वापरून प्रदर्शित करा

### प्रश्न: माझी वातावरणीय चल का काम करत नाहीत?
**उत्तर**: तपासा:
1. चल नावे अचूकपणे साचा पॅरामीटर्सशी जुळतात का?
2. अवकाश असतील तर मूल्ये योग्यरित्या उद्धृत आहेत का?
3. योग्य वातावरण निवडले आहे का: `azd env select <environment>`
4. चल योग्य वातावरणात सेट केले आहेत का?

### प्रश्न: अपयशी तैनाती नंतर साफसफाई कशी करावी?
**उत्तर**: 
```bash
azd down --force --purge
```
हे सर्व संसाधने आणि वातावरण कॉन्फिगरेशन काढून टाकते.

### प्रश्न: तैनाती नंतर माझे अनुप्रयोग का प्रवेशयोग्य नाही?
**उत्तर**: तपासा:
1. तैनाती यशस्वीपणे पूर्ण झाली आहे का
2. अनुप्रयोग चालू आहे (Azure पोर्टलमध्ये लॉग्स तपासा)
3. नेटवर्क सुरक्षा गटांनी ट्रॅफिकसाठी परवानगी दिली आहे का
4. DNS/सानुकूल डोमेन योग्यरित्या कॉन्फिगर केले आहेत का

## खर्च व बिलिंग

### प्रश्न: azd तैनात करण्याचा किती खर्च येईल?
**उत्तर**: खर्चावर अवलंबून:
- तैनात Azure सेवा
- निवडलेले सेवा टियर/ SKU
- प्रादेशिक किमतीतील फरक
- वापराच्या नमुन्यांवर

अंदाजासाठी [Azure Pricing Calculator](https://azure.microsoft.com/pricing/calculator/) वापरा.

### प्रश्न: azd तैनातींमध्ये खर्च कसा नियंत्रित करायचा?
**उत्तर**: 
1. विकास वातावरणासाठी कमी टियर SKU वापरा
2. Azure बजट आणि अलर्ट सेट करा
3. नको असताना संसाधने काढण्यासाठी `azd down` वापरा
4. योग्य प्रदेश निवडा (खर्च प्रदेशानुसार भिन्न असतो)
5. Azure Cost Management टूल वापरा

### प्रश्न: azd साच्यांसाठी मोफत टियर पर्याय आहेत का?
**उत्तर**: अनेक Azure सेवा मोफत टियर प्रदान करतात:
- App Service: मोफत टियर उपलब्ध
- Azure Functions: 1M मोफत अंमलबजावणी/महिना
- Cosmos DB: 400 RU/s सह मोफत टियर
- Application Insights: पहिल्या 5GB/महिना मोफत

साचा मोफत टियर वापरासाठी कॉन्फिगर करा.

### प्रश्न: तैनातीपूर्वी खर्चाचा अंदाज कसा लावायचा?
**उत्तर**: 
1. साच्याचा `main.bicep` तपासा की कोणती संसाधने तयार होणार आहेत
2. विशिष्ट SKU सह Azure Pricing Calculator वापरा
3. प्रथम विकास वातावरणात तैनात करून प्रत्यक्ष खर्च निरीक्षण करा
4. सविस्तर खर्च विश्लेषणासाठी Azure Cost Management वापरा

## सर्वोत्तम पद्धती

### प्रश्न: azd प्रकल्प संरचनेबाबत सर्वोत्तम पद्धती काय आहेत?
**उत्तर**: 
1. अनुप्रयोग कोड पायाभूत सुविधा पासून वेगळे ठेवा
2. `azure.yaml` मध्ये अर्थपूर्ण सेवा नावे वापरा
3. बिल्ड स्क्रिप्टमध्ये योग्य त्रुटी हाताळणी करा
4. वातावरण-विशिष्ट कॉन्फिगरेशन वापरा
5. सविस्तर दस्तऐवजीकरण समाविष्ट करा

### प्रश्न: azd मध्ये अनेक सेवा कशा आयोजित कराव्यात?
**उत्तर**: शिफारस केलेली संरचना वापरा:
```
├── src/
│   ├── web/          # Frontend service
│   ├── api/          # Backend service  
│   └── worker/       # Background service
├── infra/            # Infrastructure templates
├── azure.yaml        # azd configuration
└── README.md         # Project documentation
```

### प्रश्न: `.azure` फोल्डर व्हर्जन कंट्रोलमध्ये कमिट करावा का?
**उत्तर**: **नाही!** `.azure` फोल्डर संवेदनशील माहिती ठेवतो. हे `.gitignore` मध्ये जोडा:
```gitignore
.azure/
```

### प्रश्न: मी रहस्ये आणि संवेदनशील कॉन्फिगरेशन कसे हाताळाव?
**उत्तर**: 
1. रहस्यांसाठी Azure Key Vault वापरा
2. अनुप्रयोग कॉन्फिगरेशनमध्ये Key Vault रहस्यांचा संदर्भ द्या
3. रहस्ये कधीही व्हर्जन कंट्रोलमध्ये कमिट करू नका
4. सेवा-सेवा प्रमाणीकरणासाठी व्यवस्थापित ओळख वापरा

### प्रश्न: azd सह CI/CD साठी शिफारस केलेली पद्धत काय आहे?
**उत्तर**: 
1. प्रत्येक टप्प्यासाठी वेगळे वातावरण वापरा (dev/staging/prod)
2. तैनात करण्यापूर्वी स्वयंचलित चाचणी अंमलात आणा
3. प्रमाणीकरणासाठी सेवा प्रमुख वापरा
4. संवेदनशील कॉन्फिगरेशन पाइपलाइन रहस्ये/चलांमध्ये साठवा
5. उत्पादन तैनातींसाठी मंजुरीचे दारे अंमलात आणा

## प्रगत विषय

### प्रश्न: मी कस्टम कार्यक्षमता वापरून azd कसा वाढवू शकतो?
**उत्तर**: होय, `azure.yaml` मध्ये तैनाती हुक वापरून:
```yaml
hooks:
  predeploy:
    run: ./scripts/custom-setup.sh
  postdeploy:
    run: ./scripts/custom-config.sh
```

### प्रश्न: विद्यमान DevOps प्रक्रियांशी azd कसे एकत्रित करावे?
**उत्तर**: 
1. विद्यमान पाइपलाइन स्क्रिप्टमध्ये azd कमांड वापरा
2. टीममध्ये azd साचे प्रमाणित करा
3. विद्यमान निरीक्षण आणि अलर्टिंगशी एकत्रित करा
4. पाइपलाइन समाकलनासाठी azd चे JSON आउटपुट वापरा

### प्रश्न: मी Azure DevOps सह azd वापरू शकतो का?
**उत्तर**: होय, azd कोणत्याही CI/CD प्रणालीसह काम करते. azd कमांड वापरून Azure DevOps पाइपलाइन तयार करा.

### प्रश्न: मी azd मध्ये कसे योगदान देऊ शकतो किंवा समुदायाचे साचे तयार करू शकतो?
**उत्तर**:
1. **azd टूल**: [Azure/azure-dev](https://github.com/Azure/azure-dev) मध्ये सहकार्य करा  
2. **टेम्पलेट्स**: [टेम्पलेट मार्गदर्शक](https://github.com/Azure-Samples/awesome-azd) अनुसरून टेम्पलेट तयार करा  
3. **दस्तऐवजीकरण**: [MicrosoftDocs/azure-dev-docs](https://github.com/MicrosoftDocs/azure-dev-docs) येथे डॉक्युमेंटेशनमध्ये सहकार्य करा  

### प्रश्न: azd साठी रोडमॅप काय आहे?  
**उ**: नियोजित फिचर्स आणि सुधारणा यासाठी [अधिकृत रोडमॅप](https://github.com/Azure/azure-dev/projects) तपासा.  

### प्रश्न: मी इतर डिप्लॉयमेंट टूल्स वरून azd कडे कसे स्थलांतरित करू?  
**उ**:  
1. सध्याच्या डिप्लॉयमेंट आर्किटेक्चरचे विश्लेषण करा  
2. समतुल्य बायसेप टेम्पलेट्स तयार करा  
3. `azure.yaml` सध्याच्या सेवांशी जुळवा  
4. विकास पर्यावरणात सखोलपणे चाचणी करा  
5. हळूहळू पर्यावरणांचे स्थलांतरण करा
