## 1. बिल्डरचे रूपक

शून्यापासून आधुनिक एंटरप्राइझ-तयार AI अनुप्रयोग तयार करणे धाडसी असू शकते. हे आपल्या नवीन घराचे इमारत स्वतः बांधण्यासारखे आहे, विटा विटा करून. होय, हे करता येते! पण हे अपेक्षित अंतिम परिणाम मिळवण्याचा सर्वात प्रभावी मार्ग नाही!

त्याऐवजी, आपण सहसा एका विद्यमान _डिझाइन ब्लूप्रिंट_ पासून सुरुवात करतो आणि एक आर्किटेक्टसोबत काम करून त्यास आपल्या वैयक्तिक गरजांनुसार सानुकूलित करतो. आणि हुशार अनुप्रयोग तयार करताना हा अचूक दृष्टिकोन आहे. प्रथम, तुमच्या समस्येच्या क्षेत्राशी सुसंगत चांगला डिझाइन आर्किटेक्चर शोधा. मग सोल्यूशन आर्किटेक्टसोबत काम करून तुमच्या विशिष्ट परिस्थितीसाठी सोल्यूशन सानुकूल आणि विकसित करा.

पण हे डिझाइन ब्लूप्रिंट कुठे शोधाले जाऊ शकतात? आणि आपल्याला असे कोणते आर्किटेक्ट सापडेल जे आम्हाला हे ब्लूप्रिंट कसे सानुकूल आणि डिप्लॉय करायचे हे शिकवेल? या कार्यशाळेत, आम्ही तुम्हाला तीन तंत्रज्ञानांशी परिचय करून देऊन या प्रश्नांची उत्तरे देतो:

1. [Azure Developer CLI](https://aka.ms/azd) - एक मुक्त स्रोत साधन जे स्थानिक विकास (बिल्ड) पासून क्लाऊड डिप्लॉयमेंट (शिप) पर्यंतच्या विकासकाच्या मार्गाला वेग देते.
1. [Microsoft Foundry Templates](https://ai.azure.com/templates) - एकसारख्या मुक्त स्रोत रिपॉझिटरीज ज्या AI सोल्यूशन आर्किटेक्चर डिप्लॉय करण्यासाठी नमुना कोड, पायाभूत सुविधा आणि कॉन्फिगरेशन फायली देते.
1. [GitHub Copilot Agent Mode](https://code.visualstudio.com/docs/copilot/chat/chat-agent-mode) - Azure ज्ञानावर आधारित एक कोडिंग एजंट, जो आपल्याला कोडबेस नॅव्हिगेट करण्यास आणि नैसर्गिक भाषेचा वापर करून बदल करण्यास मार्गदर्शन करू शकतो.

या साधनांसह, आपण आता योग्य टेम्प्लेट _शोधू_, त्यास प्रमाणित करण्यासाठी _डिप्लॉय_ करू, आणि आपल्या विशिष्ट परिस्थितींसाठी ते _सानुकूलित_ करू शकतो. चला आत जाऊन शिकूया की हे कसे काम करतात.

## 2. Azure Developer CLI

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) (किंवा `azd`) हा एक मुक्त स्रोत कमांडलाइन टूल आहे जो तुमच्या कोड-टू-क्लाउड प्रवासाला वेगवान बनवू शकतो, ज्यात विकासकांसाठी सुलभ कमांड असतात जे तुमच्या IDE (विकास) आणि CI/CD (डेवऑप्स) वातावरणांमध्ये एकसारखे काम करतात.

`azd` सोबत, तुमचा डिप्लॉयमेंट प्रवास इतका सोपा असू शकतो:

- `azd init` - विद्यमान AZD टेम्प्लेटमधून नवीन AI प्रकल्प आरंभ करतो.
- `azd up` - पायाभूत सुविधा तयार करतो आणि एका टप्प्यात तुमचे अनुप्रयोग डिप्लॉय करतो.
- `azd monitor` - तुमच्या डिप्लॉय केलेल्या अनुप्रयोगासाठी रिअल-टाइम निरीक्षण व निदान प्राप्त करा.
- `azd pipeline config` - Azure वर डिप्लॉयमेंट ऑटोमेट करण्यासाठी CI/CD पाइपलाइन सेटअप करा.

**🎯 | व्यायाम**: <br/>  
आपल्या GitHub Codespaces पर्यावरणात `azd` कमांडलाइन टूल शोधून पहा. काय करू शकते ते पाहण्यासाठी खालील कमांड टाइप करून सुरू करा:

```bash title="" linenums="0"
azd help
```

## 3. AZD टेम्प्लेट

`azd` साठी हे साध्य करण्यासाठी, त्याला कोणती पायाभूत सुविधा तयार करायची आहे, कोणती कॉन्फिगरेशन सेटिंग्ज लागू करायच्या आहेत, आणि कोणता अनुप्रयोग डिप्लॉय करायचा आहे हे माहित असणे आवश्यक आहे. येथे [AZD टेम्प्लेट्स](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/azd-templates?tabs=csharp) मदत करतात.

AZD टेम्प्लेट्स ही मुक्त-स्रोत रिपॉझिटरीज आहेत ज्या सोल्यूशन आर्किटेक्चर डिप्लॉय करण्यासाठी आवश्यक नमुना कोड, पायाभूत सुविधा आणि कॉन्फिगरेशन फाइल्स एकत्र करतात. _Infrastructure-as-Code_ (IaC) दृष्टिकोन वापरून, हे टेम्प्लेट संसाधन व्याख्या आणि कॉन्फिगरेशन सेटिंग्ज आवृत्तीनियंत्रकांत (जरुरी प्रमाणे अनुप्रयोग स्रोत कोड प्रमाणे) ठेवतात - ज्यामुळे त्या प्रकल्पाचा वापर करणाऱ्यांसाठी पुन्हा वापरता येणारे आणि सुसंगत वर्कफ्लो तयार होतात.

तुमच्या परिस्थितीसाठी AZD टेम्प्लेट तयार करताना किंवा पुनरावृत्ती करताना, खालील प्रश्न विचारात घ्या:

1. तुम्ही काय तयार करत आहात? → त्या परिस्थितीसाठी सुरुवातीचा कोड असलेला कोणता टेम्प्लेट आहे?
1. तुमचे सोल्यूशन कसे आर्किटेक्ट केले आहे? → आवश्यक संसाधने असलेला टेम्प्लेट आहे का?
1. तुमचे सोल्यूशन कसे डिप्लॉय केले जाते? → `azd deploy` वापरून पूर्व/नंतर प्रक्रिया हुक्स विचारात घ्या!
1. तुम्ही ते आणखी कसे अनुकूलित करू शकता? → अंगभूत निरीक्षण आणि ऑटोमेशन पाइपलाइन कशी वापरू शकता हे पहा!

**🎯 | व्यायाम**: <br/>  
[Awesome AZD](https://azure.github.io/awesome-azd/) गॅलरीला भेट द्या आणि फिल्टर्स वापरून सध्या उपलब्ध २५०+ टेम्प्लेट्स एक्सप्लोर करा. बघा की तुम्हाला तुमच्या परिस्थितीच्या गरजांसाठी कोणता टेम्प्लेट सापडतो का.

## 4. AI अनुप्रयोग टेम्प्लेट्स

AI-पुरस्कृत अनुप्रयोगांसाठी, Microsoft विशेष टेम्प्लेट्स प्रदान करते ज्यात **Microsoft Foundry** आणि **Foundry Agents** आहेत. हे टेम्प्लेट्स तुमच्या बुद्धिमान, प्रॉडक्शन-तयार अनुप्रयोग तयार करण्याचा मार्ग वेगवान करतात.

### Microsoft Foundry & Foundry Agents टेम्प्लेट्स

खालील टेम्प्लेट निवडा आणि डिप्लॉय करा. प्रत्येक टेम्प्लेट [Awesome AZD](https://azure.github.io/awesome-azd/) वर उपलब्ध आहे आणि एका कमांडसह प्रारंभ केला जाऊ शकतो.

| टेम्प्लेट | वर्णन | डिप्लॉय कमांड |
|----------|-------------|----------------|
| **[AI Chat with RAG](https://azure.github.io/awesome-azd/?tags=ai&tags=rag)** | Microsoft Foundry वापरून Retrieval Augmented Generation सह चॅट अनुप्रयोग | `azd init -t azure-samples/azure-search-openai-demo` |
| **[Foundry Agent Service Starter](https://azure.github.io/awesome-azd/?tags=ai&tags=agents)** | Foundry Agents वापरून स्वयंचलित कार्यासाठी AI एजंट तयार करा | `azd init -t azure-samples/foundry-agent-service-starter` |
| **[Multi-Agent Orchestration](https://azure.github.io/awesome-azd/?tags=ai&tags=agents)** | गुंतागुंतीच्या वर्कफ्लोजसाठी अनेक Foundry एजंट समन्वयित करा | `azd init -t azure-samples/multi-agent-orchestration` |
| **[AI Document Intelligence](https://azure.github.io/awesome-azd/?tags=ai&tags=document)** | Microsoft Foundry मॉडेल्स वापरून कागदपत्रे काढा व विश्लेषण करा | `azd init -t azure-samples/ai-document-processing` |
| **[Conversational AI Bot](https://azure.github.io/awesome-azd/?tags=ai&tags=bot)** | Microsoft Foundry समाकालीन बुद्धिमान चॅटबॉट तयार करा | `azd init -t azure-samples/ai-chat-protocol` |
| **[AI Image Generation](https://azure.github.io/awesome-azd/?tags=ai&tags=dalle)** | Microsoft Foundry द्वारे DALL-E वापरून प्रतिमा तयार करा | `azd init -t azure-samples/ai-image-generation` |
| **[Semantic Kernel Agent](https://azure.github.io/awesome-azd/?tags=ai&tags=semantic-kernel)** | Foundry Agents सह Semantic Kernel वापरून AI एजंट्स | `azd init -t azure-samples/semantic-kernel-agent` |
| **[AutoGen Multi-Agent](https://azure.github.io/awesome-azd/?tags=ai&tags=autogen)** | AutoGen फ्रेमवर्क वापरून बहु-एजंट सिस्टम्स | `azd init -t azure-samples/autogen-multi-agent` |

### झटपट सुरुवात

1. **टेम्प्लेट्स ब्राउझ करा**: [https://azure.github.io/awesome-azd/](https://azure.github.io/awesome-azd/) ला भेट द्या आणि `AI`, `Agents`, किंवा `Microsoft Foundry` नुसार फिल्टर करा
2. **तुमचा टेम्प्लेट निवडा**: तुमच्या वापराच्या प्रकरणाशी सुसंगत असलेला निवडा
3. **प्रारंभ करा**: तुमच्या निवडलेल्या टेम्प्लेटसाठी `azd init` कमांड चालवा
4. **डिप्लॉय करा**: प्राव्हिजन आणि डिप्लॉय करण्यासाठी `azd up` चालवा

**🎯 | व्यायाम**: <br/>
खालील टेम्प्लेट्सपैकी तुमच्या परिस्थितीवर आधारित एक निवडा:

- **चॅटबॉट तयार करत आहात का?** → **AI Chat with RAG** किंवा **Conversational AI Bot** सोबत सुरू करा
- **स्वयंचलित एजंट्स पाहिजे?** → **Foundry Agent Service Starter** किंवा **Multi-Agent Orchestration** वापरून पहा
- **कागदपत्रे प्रक्रिया करत आहात?** → **AI Document Intelligence** वापरा
- **AI कोडिंग मदत पाहिजे?** → **Semantic Kernel Agent** किंवा **AutoGen Multi-Agent** एक्सप्लोर करा

```bash title="Example: Deploy the AI Chat with RAG template" linenums="0"
azd init -t azure-samples/azure-search-openai-demo
azd up
```

    [Awesome AZD Gallery](https://azure.github.io/awesome-azd/) मध्ये २५०+ टेम्प्लेट्स आहेत. भाषांतर, फ्रेमवर्क आणि Azure सेवा यांच्या विशिष्ट गरजांसाठी फिल्टर्स वापरून टेम्प्लेट्स शोधा.
