## 1. परिचय

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) किंवा `azd` हे एक मुक्त-स्रोत कमांडलाइन टूल आहे जे अॅझ्यूरवर अनुप्रयोग तयार करणे आणि डिप्लॉय करताना विकासक कार्यप्रवाह सुलभ करते.

[AZD टेम्पलेट्स](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) हे प्रमाणीकृत रेपॉजिटरीज आहेत ज्यात नमुना अनुप्रयोग कोड, _इन्फ्रास्ट्रक्चर-अ‍ॅज-कोड_ घटक आणि `azd` कॉन्फिगरेशन फाइल्स समाविष्ट असतात जे एकसंध सोल्युशन आर्किटेक्चर तयार करतात. इन्फ्रास्ट्रक्चरची व्यवस्था `azd provision` कमांडइतकी सोपी होते - तर `azd up` वापरल्यास तुम्ही एकाच वेळी इन्फ्रास्ट्रक्चर तयार करू शकता आणि अनुप्रयोग डिप्लॉय करू शकता!

त्यामुळे, तुमच्या अनुप्रयोग विकास प्रक्रियेला सुरूवात करणे जितके सोपे आहे तितके योग्य _AZD स्टार्टर टेम्पलेट_ शोधणे जे तुमच्या अनुप्रयोग आणि इन्फ्रास्ट्रक्चर गरजांना सर्वात जवळचे आहे - आणि नंतर रेपॉजिटरीला तुमच्या वापराच्या गरजेनुसार सानुकूलित करणे.

सुरू करण्यापूर्वी, चला पाहूया तुमच्याकडे Azure Developer CLI स्थापित आहे का.

1. VS Code टर्मिनल उघडा आणि ही कमांड टाइप करा:

      ```bash title="" linenums="0"
      azd version
      ```

1. तुम्हाला यासारखे काहीतरी दिसायला हवे!

      ```bash title="" linenums="0"
      azd version 1.19.0 (commit b3d68cea969b2bfbaa7b7fa289424428edb93e97)
      ```

**आता तुम्ही azd वापरून टेम्पलेट निवडण्यास आणि डिप्लॉय करण्यास तयार आहात**

## 2. टेम्पलेट निवड

Microsoft Foundry प्लॅटफॉर्मसह [सुचवलेल्या AZD टेम्पलेट्सचा संच](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) येतो जो लोकप्रिय सोल्युशन परिस्थिती जसे की _मल्टि-एजंट वर्कफ्लो ऑटोमेशन_ आणि _मल्टि-मोडल कंटेंट प्रक्रिया_ यांना कव्हर करतो. तुम्ही Microsoft Foundry पोर्टलवरून देखील हे टेम्पलेट्स शोधू शकता.

1. भेट द्या [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. विचारल्यास Microsoft Foundry पोर्टलमध्ये लॉगिन करा - तुम्हाला यासारखे काही दिसेल.

![Pick](../../../../../translated_images/mr/01-pick-template.60d2d5fff5ebc374.webp)


**Basic** पर्याय तुमचे स्टार्टर टेम्पलेट्स आहेत:

1. [ ] [AI Chat वापरून सुरुवात करा](https://github.com/Azure-Samples/get-started-with-ai-chat) जे तुमचा डेटा वापरून Azure Container Apps वर एक मूलभूत चॅट अॅप्लिकेशन डिप्लॉय करते. याचा वापर करून बेसिक AI चॅटबॉट परिस्थिती तपासा.
1. [X] [AI Agents वापरून सुरुवात करा](https://github.com/Azure-Samples/get-started-with-ai-agents) जे Microsoft Foundry एजंटसह एक स्टँडर्ड AI एजंट डिप्लॉय करते. याचा वापर एजंटिक AI सोल्युशन्सशी परिचित होण्यासाठी करा ज्यात टूल्स आणि मॉडेल्स वापरलेले असतात.

दुसऱ्या लिंकला नवीन ब्राउझर टॅबमध्ये उघडा (किंवा संबंधित कार्डसाठी `Open in GitHub` क्लिक करा). तुम्हाला या AZD टेम्पलेटचा रेपॉजिटरी दिसेल. README एक्सप्लोर करा. अनुप्रयोग आर्किटेक्चर असे दिसते:

![Arch](../../../../../translated_images/mr/architecture.8cec470ec15c65c7.webp)

## 3. टेम्पलेट सक्रिय करा

चलो, हा टेम्पलेट डिप्लॉय करूया आणि तो वैध आहे याची खात्री करूया. [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) विभागात दिलेल्या मार्गदर्शक सूचनांचे अनुसरण करूया.

1. या [लिंकमध्ये](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) क्लिक करा - डीफॉल्ट क्रिया `Create codespace` पुष्टी करा
1. यामुळे नवीन ब्राउझर टॅब उघडेल - GitHub Codespaces सत्र पूर्ण होईपर्यंत थांबा
1. Codespaces मध्ये VS Code टर्मिनल उघडा - खालील कमांड टाइप करा:

   ```bash title="" linenums="0"
   azd up
   ```

हे वर्कफ्लो पावले पूर्ण करा:

1. Azure मध्ये लॉग इन करण्यासाठी विनंती केली जाईल - प्रमाणीकरणाचे निर्देश पाळा
1. तुम्हाला एक अद्वितीय वातावरण नाव प्रविष्ट करण्यास सांगितले जाईल - उदा., मी वापरले `nitya-mshack-azd`
1. यामुळे `.azure/` फोल्डर तयार होईल - तुम्हाला त्या वातावरण नावाचा एक उपफोल्डर दिसेल
1. तुम्हाला सदस्यत्वाचे नाव निवडण्यास सांगितले जाईल - डीफॉल्ट निवडा
1. स्थान विचारले जाईल - `East US 2` वापरा

आता, तुम्ही प्रोव्हिजनिंग पूर्ण होईपर्यंत प्रतीक्षा करा. **हे 10-15 मिनिटे लागतात**

1. पूर्ण झाल्यावर, तुमच्या कन्सोलमध्ये SUCCESS मेसेज असेल:

      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. तुमच्या Azure पोर्टलमध्ये आता त्या वातावरण नावासह एक प्राव्हिज़न्ड रिसोर्स ग्रुप असेल:

      ![Infra](../../../../../translated_images/mr/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **आता तुम्ही डिप्लॉय केलेल्या इन्फ्रास्ट्रक्चर आणि अनुप्रयोगाचे प्रमाणीकरण करायास तयार आहात**.

## 4. टेम्पलेट प्रमाणीकरण

1. Azure Portal [Resource Groups](https://portal.azure.com/#browse/resourcegroups) पृष्ठाला भेट द्या - विचारल्यास लॉगिन करा
1. तुमच्या वातावरण नावासाठी RG वर क्लिक करा - तुम्हाला वर उल्लेखित पृष्ठ दिसेल

      - Azure Container Apps रिसोर्सवर क्लिक करा
      - _Essentials_ विभागातील (वर उजव्या कोपर्यात) Application Url वर क्लिक करा

1. तुम्हाला होस्टेड अनुप्रयोगाचा फ्रंट-एंड UI याप्रकारे दिसेल:

   ![App](../../../../../translated_images/mr/03-test-application.471910da12c3038e.webp)

1. काही [नमुना प्रश्न](https://github.com/Azure-Samples/get-started-with-ai-agents/blob/main/docs/sample_questions.md) विचारून पाहा

      1. विचारा: ```फ्रान्सची राजधानी काय आहे?``` 
      1. विचारा: ```दोन लोकांसाठी $200 खाली सर्वोत्तम तंबू कोणता आहे, आणि त्यात कोणती वैशिष्ट्ये आहेत?```

1. तुम्हाला खाली दाखवल्याप्रमाणे उत्तरं मिळावीत. _पण हे कसं काम करतं?_

      ![App](../../../../../translated_images/mr/03-test-question.521c1e863cbaddb6.webp)

## 5. एजंट प्रमाणीकरण

Azure Container App एक एन्डपॉइंट डिप्लॉय करतो जो या टेम्पलेटसाठी Microsoft Foundry प्रोजेक्टमध्ये प्रोव्हिजन केलेल्या AI एजंटला जोडतो. चला पाहूया याचा काय अर्थ आहे.

1. Azure Portal मधील तुमच्या रिसोर्स ग्रुपचा _Overview_ पृष्ठावर परत जा

1. त्याच यादीतील `Microsoft Foundry` रिसोर्सवर क्लिक करा

1. तुम्हाला हे दिसेल. `Go to Microsoft Foundry Portal` बटणावर क्लिक करा.
   ![Foundry](../../../../../translated_images/mr/04-view-foundry-project.fb94ca41803f28f3.webp)

1. तुमच्या AI अनुप्रयोगासाठी Foundry प्रोजेक्ट पृष्ठ दिसेल
   ![Project](../../../../../translated_images/mr/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` वर क्लिक करा - तुमच्या प्रोजेक्टमधील डीफॉल्ट एजंट दिसेल
   ![Agents](../../../../../translated_images/mr/06-visit-agents.bccb263f77b00a09.webp)

1. ते निवडा - तुम्हाला एजंटचे तपशील दिसतील. पुढील गोष्टींचा विचार करा:

      - एजंट डीफॉल्टने (नेहमी) फाइल सर्च वापरतो
      - एजंटचा `Knowledge` दर्शवतो की त्याच्याकडे 32 फाइल्स अपलोड केलेल्या आहेत (फाइल सर्चसाठी)
      ![Agents](../../../../../translated_images/mr/07-view-agent-details.0e049f37f61eae62.webp)

1. डाव्या मेनूमधील `Data+indexes` पर्याय शोधा आणि तपशीलांसाठी क्लिक करा.

      - तुम्हाला ज्ञानासाठी अपलोड केलेल्या 32 डेटा फाइल्स दिसतील.
      - हे `src/files` अंतर्गत 12 ग्राहक फाइल्स आणि 20 उत्पादन फाइल्ससह जुळतील.
      ![Data](../../../../../translated_images/mr/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**तुम्ही एजंट कार्यप्रणाली प्रमाणीकरण केली आहे!**

1. एजंटचे प्रतिसाद त्या फाइलमधील ज्ञानावर आधारित आहेत.
1. आता तुम्ही त्या डेटाशी संबंधित प्रश्न विचारू शकता आणि आधारित प्रतिसाद मिळवू शकता.
1. उदाहरण: `customer_info_10.json` मध्ये "Amanda Perez" कडून केलेले 3 खरेदी वर्णन आहेत.

ब्राउझर टॅबमधील Container App एन्डपॉइंटवर परत जा आणि विचारा: `Amanda Perez कोणती उत्पादने मालक आहे?`. तुम्हाला यासारखे काही दिसेल:

![Data](../../../../../translated_images/mr/09-ask-in-aca.4102297fc465a4d5.webp)

## 6. एजंट प्लेग्राउंड

Microsoft Foundry च्या क्षमता समजून घेण्यासाठी, एजंटला Agents Playground मध्ये वापरून पाहूया.

1. Microsoft Foundry मध्ये `Agents` पृष्ठावर परत जा - डीफॉल्ट एजंट निवडा
1. `Try in Playground` पर्यायावर क्लिक करा - तुम्हाला अशा प्रकारचा Playground UI दिसेल
1. तोच प्रश्न विचारा: `Amanda Perez कोणती उत्पादने मालक आहे?`

    ![Data](../../../../../translated_images/mr/09-ask-in-playground.a1b93794f78fa676.webp)

तुम्हाला समान (किंवा साधर्म्य) प्रतिसाद मिळेल - पण तुम्हाला एजंटिक अॅपची गुणवत्ता, खर्च, आणि कार्यक्षमतेला समजण्यासाठी अतिरिक्त माहिती देखील मिळेल. उदाहरण:

1. प्रतिसादासाठी वापरल्या गेलेल्या डेटाफाइल्सची माहिती दिली आहे
1. कोणत्याही फाइल लेबलवर होवर करा - डेटाआणि प्रतिसाद तुमच्या प्रश्नाशी जुळतात का?

तुम्हाला प्रतिसादाखाली एक _stats_ रांग देखील दिसेल.

1. कोणत्याही मेट्रिकवर होवर करा - उदा., Safety. तुम्हाला यासारखे काही दिसतो
1. मूल्यमापन केलेला रेटिंग प्रतिसादाच्या सुरक्षिततेशी तुमच्या मूल्यांकनाशी जुळतो का?

## 7. अंगभूत निरीक्षणशीलता

निरीक्षणशीलता म्हणजे तुमच्या अनुप्रयोगाचे अंतरंगतरीत निरीक्षण करून डेटा तयार करणे जे समजण्यासाठी, डीबग करण्यासाठी, आणि ऑप्टिमाइज करण्यासाठी वापरले जाते. याची झलक पाहण्यासाठी:

1. `View Run Info` बटणावर क्लिक करा - तुम्हाला हा दृश्य दिसेल. हा [एजंट ट्रेसिंग](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) चा एक उदाहरण आहे. _हे दृश्य मूळ मेनूतील Thread Logs क्लिक करूनही मिळवता येते_.

   - रन स्टेप्स आणि एजंटने वापरलेल्या टूल्सची माहिती मिळवा
   - प्रतिसादासाठी टोकन वापर आणि आउटपुट टोकन वापर समजून घ्या
   - लेटन्सी आणि वेळ कुठे खर्च होत आहे हे तपासा

      ![Agent](../../../../../translated_images/mr/10-view-run-info.b20ebd75fef6a1cc.webp)

1. `Metadata` टॅबवर क्लिक करा जेणेकरून रनसाठी अतिरिक्त वैशिष्ट्ये दिसतील, जी भविष्यात डीबग करताना उपयुक्त ठरू शकतात.

      ![Agent](../../../../../translated_images/mr/11-view-run-info-metadata.7966986122c7c2df.webp)


1. `Evaluations` टॅबवर क्लिक करा जेथे एजंट प्रतिसादावर केलेली स्वयंचलित मूल्यमापन दिसतील. यात सुरक्षिततेची तपासणी (उदा., Self-harm) आणि एजंट-विशिष्ट मूल्यमापन (उदा., Intent resolution, Task adherence) यांचा समावेश आहे.

      ![Agent](../../../../../translated_images/mr/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. शेवट आणि महत्त्वाचं म्हणजे, साइडबार मेनूमधील `Monitoring` टॅबवर क्लिक करा.

      - प्रदर्शित पृष्ठात `Resource usage` टॅब निवडा आणि मेट्रिक्स पहा.
      - किंमती (टोकन्स) आणि लोड (रिक्वेस्टी) च्या दृष्टीने अनुप्रयोग वापराचा मागोवा घ्या.
      - इनपुट प्रोसेसिंगसाठी प्रथम बाइट आणि आउटपुटसाठी शेवटचा बाइट पर्यंतचे विलंब ट्रॅक करा.

      ![Agent](../../../../../translated_images/mr/13-monitoring-resources.5148015f7311807f.webp)

## 8. पर्यावरण चल (Environment Variables)

आत्तापर्यंत आपण ब्राउझरमध्ये डिप्लॉयमेंट केले आणि आपला इन्फ्रास्ट्रक्चर प्रोव्हिजनिंग व अनुप्रयोग चालू आहे याची खात्री केली. पण अॅप्लिकेशन _कोड-प्रथम_ पद्धतीने वापरण्यासाठी आपल्याला स्थानिक विकास वातावरण योग्य चलांसोबत कॉन्फिगर करावे लागेल. `azd` वापरणे यास सोपे करते.

1. Azure Developer CLI [पर्यावरण चलांचा वापर करतो](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) जे अॅप्लिकेशन डिप्लॉयमेंटसाठी कॉन्फिगरेशन सेटिंग्ज संचयित करण्यासाठी आणि व्यवस्थापित करण्यासाठी आहे.

1. पर्यावरण चाले `.azure/<env-name>/.env` मध्ये संचयित होतात - हे त्यांना `env-name` मधील डिप्लॉयमेंट पर्यावरणाशी संबंधित ठेवते आणि एकाच रेपॉजिटरीमध्ये वेगवेगळ्या डिप्लॉयमेंट टार्गेट्समधील वातावरण वेगळे ठेवण्यास मदत करते.

1. पर्यावरण चाले `azd` च्या कोणत्याही कमांड (उदा., `azd up`) चालवल्यावर आपोआप लोड होतात. लक्षात घ्या, `azd` ओएस-स्तरीय environment variables (उदा., शेलमध्ये सेट केलेले) वाचत नाही - स्क्रिप्टमध्ये माहिती हस्तांतरित करण्यासाठी `azd set env` आणि `azd get env` वापरा.


चला काही कमांड वापरून पाहूया:

1. या वातावरणातील `azd` साठी सेट केलेल्या सर्व environment variables मिळवा:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      तुम्हाला यासारखे काही दिसेल:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4o-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. विशिष्ट मूल्य मिळवा - उदा., मला माहित करायचे आहे का `AZURE_AI_AGENT_MODEL_NAME` सेट केले आहे का

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      तुम्हाला यासारखे काही दिसेल - हे डीफॉल्टने सेट नव्हते!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd` साठी नवीन environment variable सेट करा. येथे, एजंट मॉडेल नाव अद्यतनित करूया. _टीप: केलेले बदल लगेच `.azure/<env-name>/.env` फाइलमध्ये परावर्तित होतील._

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      आता, आपल्याला मूल्य सेट झाल्याचे दिसेल:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. लक्षात घ्या की काही संसाधने कायमस्वरूपी असतात (उदा., मॉडेल डिप्लॉयमेंट्स) आणि त्यासाठी फक्त `azd up` नाही तर अधिक काही करावे लागते म्हणजे पुन्हा डिप्लॉय करणे शक्य होते. चला मूळ डिप्लॉयमेंट तोडून बदललेल्या env वेरिएबल्ससह पुन्हा डिप्लॉय करूया.

1. **रिफ्रेश करा** जर तुम्ही आधीच azd टेम्पलेट वापरून इन्फ्रास्ट्रक्चर डिप्लॉय केले असेल तर तुम्ही या आज्ञेद्वारे स्थानिक environment variables ची स्थिती Azure डिप्लॉयमेंटच्या वर्तमान स्थितीवरून _रिफ्रेश_ करू शकता:

      ```bash title="" linenums="0"
      azd env refresh
      ```

      हे दोन किंवा अधिक स्थानिक विकास वातावरणांमध्ये (उदाहरणार्थ, अनेक विकसकांसह संघ) पर्यावरण चल भिन्नता _सिंक_ करण्याचा एक शक्तिशाली मार्ग आहे - परिनियोजित पायाभूत सुविधा पर्यावरण चल स्थितीसाठी सत्येक म्हणून कार्य करते. संघ सदस्य फक्त चल भिन्नता _रिफ्रेश_ करतात जेणेकरून ते पुन्हा समक्रमित होतील.
