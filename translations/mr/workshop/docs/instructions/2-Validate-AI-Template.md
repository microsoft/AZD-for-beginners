## 1. प्रस्तावना

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) किंवा `azd` ही एक मुक्त स्रोत कमांडलाइन साधन आहे जे अॅझ्युरमध्ये अनुप्रयोग तयार करताना आणि तैनात करताना विकसकांच्या कामाच्या प्रवाहाला सुलभ करते.

[AZD टेम्पलेट्स](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) ही प्रमाणित रेपॉझिटरीज आहेत ज्यात नमुना अनुप्रयोगाचा कोड, _इन्फ्रास्ट्रक्चर-एज़-कोड_ घटक आणि सुसंगत सोल्यूशन आर्किटेक्चरसाठी `azd` कॉन्फिगरेशन फाइल्स असतात. इन्फ्रास्ट्रक्चर तयार करणे `azd provision` कमांड जितके सोपे आहे - तर `azd up` वापरून तुम्ही एकाच वेळी इन्फ्रास्ट्रक्चर तयार आणि अनुप्रयोग तैनात करू शकता!

त्यामुळे, तुमच्या अनुप्रयोग आणि इन्फ्रास्ट्रक्चर गरजांना सर्वात जवळ असणारा योग्य _AZD स्टार्टर टेम्पलेट_ शोधणे आणि नंतर तुमच्या परिस्थितीच्या गरजांसाठी रेपॉझिटरी सानुकूलित करणे हे तुमच्या अॅप विकास प्रक्रियेला जलद सुरूवात करण्याएवढे सोपे असू शकते.

सुरुवात करण्यापूर्वी, आपण Azure Developer CLI स्थापित केले आहे याची खात्री करूया.

1. VS Code टर्मिनल उघडा आणि हा आदेश टाका:

      ```bash title="" linenums="0"
      azd version
      ```

1. तुम्हाला काहीतरी हेसुद्धा दिसेल!

      ```bash title="" linenums="0"
      azd version 1.27.1 (commit <current-build>)
      ```

**आता तुम्ही azd सोबत टेम्पलेट निवडून तैनात करण्यासाठी तयार आहात**

## 2. टेम्पलेट निवड

Microsoft Foundry प्लॅटफॉर्मसह [शिफारस केलेल्या AZD टेम्पलेट्सचा संच](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) येतो, जे लोकप्रिय सोल्यूशन परिस्थिती जसे की _मल्टी-एजंट वर्कफ्लो ऑटोमेशन_ आणि _मल्टी-मोडल सामग्री प्रक्रिया_ यांचा समावेश करतात. तुम्ही हे टेम्पलेट्स Microsoft Foundry पोर्टलवरून देखील शोधू शकता.

1. भेट द्या [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. जेव्हा विचारले जाईल तेव्हा Microsoft Foundry पोर्टलमध्ये लॉग इन करा - तुम्हाला काहीतरी हेच दिसेल.

![Pick](../../../../../translated_images/mr/01-pick-template.60d2d5fff5ebc374.webp)


**Basic** पर्याय तुमचे स्टार्टर टेम्पलेट्स आहेत:

1. [ ] [AI Chat सह सुरुवात करा](https://github.com/Azure-Samples/get-started-with-ai-chat) जो एक मूलभूत चॅट अनुप्रयोग तुमच्या डेटासह Azure Container Apps मध्ये तैनात करतो. हे मूलभूत AI चॅटबॉट परिस्थिती एक्सप्लोर करण्यासाठी वापरा.
1. [X] [AI Agents सह सुरुवात करा](https://github.com/Azure-Samples/get-started-with-ai-agents) जे एक मानक AI एजंट (Foundry Agents सह) तैनात करते. एजंटसंबंधित AI सोल्यूशन्समध्ये परिचित होण्यासाठी हे वापरा, ज्यामध्ये टूल्स आणि मॉडेल्स असतात.

दुसऱ्या लिंकला नवीन ब्राउझर टॅबमध्ये भेट द्या (किंवा संबंधित कार्डसाठी `Open in GitHub` क्लिक करा). तुम्हाला या AZD टेम्पलेटसाठी रेपॉझिटरी दिसेल. README वाचण्यासाठी एक मिनिट द्या. अनुप्रयोग आर्किटेक्चर असे दिसते:

![Arch](../../../../../translated_images/mr/architecture.8cec470ec15c65c7.webp)

## 3. टेम्पलेट सक्रियता

चला हा टेम्पलेट तैनात करून बघूया आणि त्याचा वैधता तपासूया. आपण [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) विभागातील मार्गदर्शकांचे अनुसरण करू.

1. टेम्पलेट रेपॉझिटरीसाठी कार्यरत पर्यावरण निवडा:

      - **GitHub Codespaces**: [या लिंकवर](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) क्लिक करा आणि `Create codespace` पुष्टी करा
      - **स्थानिक क्लोन किंवा dev कंटेनर**: `Azure-Samples/get-started-with-ai-agents` क्लोन करा आणि VS Code मध्ये उघडा

1. VS Code टर्मिनल तयार होईपर्यंत थांबा, नंतर खालील आदेश टाका:

   ```bash title="" linenums="0"
   azd up
   ```

हा आदेश ट्रिगर करेल अशा वर्कफ्लो टप्प्यांचे पालन करा:

1. तुम्हाला Azure मध्ये लॉग इन करण्यास सांगितले जाईल - प्रमाणीकरणासाठी सूचना पाळा
1. स्वतःसाठी एक अद्वितीय पर्यावरण नाव द्या - उदा., मी `nitya-mshack-azd` वापरले
1. यामुळे `.azure/` फोल्डर तयार होईल - तुम्हाला पर्यावरणाच्या नावासह सबफोल्डर दिसेल
1. तुम्हाला सदस्यत्व नाव निवडण्यास विचारले जाईल - डिफॉल्ट निवडा
1. स्थान विचारले जाईल - `East US 2` वापरा

आता, तुम्ही provisioning पूर्ण होईपर्यंत प्रतीक्षा करा. **हे 10-15 मिनिटे घेतात**

1. पूर्ण झाल्यावर, तुमच्या कन्सोलवर SUCCESS संदेश दिसेल अशा प्रकारे:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. आता तुमच्या Azure Portal मध्ये त्या पर्यावरण नावाने provisioned resource group असेल:

      ![Infra](../../../../../translated_images/mr/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **आता तुम्ही तैनात केलेले इन्फ्रास्ट्रक्चर आणि अनुप्रयोग प्रमाणीकरणासाठी तयार आहात**.

## 4. टेम्पलेट प्रमाणीकरण

1. Azure Portal [Resource Groups](https://portal.azure.com/#browse/resourcegroups) पृष्ठाला भेट द्या - विचारणा केल्यास लॉग इन करा
1. तुमच्या पर्यावरण नावासाठी RG वर क्लिक करा - तुम्हाला वर उल्लेखलेले पृष्ठ दिसेल

      - Azure Container Apps रिसोर्सवर क्लिक करा
      - _Essentials_ विभागातील Application Url वर क्लिक करा (वर उजव्या कोपऱ्यात)

1. तुम्हाला एक होस्ट केलेले अनुप्रयोग फ्रंट-एंड UI असे दिसेल:

   ![App](../../../../../translated_images/mr/03-test-application.471910da12c3038e.webp)

1. काही [नमुना प्रश्न विचारा](https://github.com/Azure-Samples/get-started-with-ai-agents/blob/main/docs/sample_questions.md)

      1. विचारा: ```फ्रान्सची राजधानी काय आहे?```
      1. विचारा: ```$200 खाली दोन लोकांसाठी सर्वोत्तम तंबू कोणता आहे, आणि त्यामध्ये कोणत्या सुविधा आहेत?```

1. तुम्हाला खालीलप्रमाणे उत्तरं मिळालेली पाहिजेत. _पण हे कसे काम करते?_

## 5. एजंट प्रमाणीकरण

Azure Container App एक एंडपॉइंट तैनात करतो जो Microsoft Foundry प्रोजेक्टमध्ये provisioned AI एजंटशी कनेक्ट होतो. चला याचा अर्थ काय होतो ते पाहूया.

1. Azure Portal मध्ये तुमच्या रिसोर्स ग्रुपचे _Overview_ पृष्ठ पुन्हा उघडा

1. त्या यादीत `Microsoft Foundry` रिसोर्सवर क्लिक करा

1. तुम्हाला हे दिसेल. `Go to Microsoft Foundry Portal` बटणावर क्लिक करा.
   ![Foundry](../../../../../translated_images/mr/04-view-foundry-project.fb94ca41803f28f3.webp)

1. तुमच्या AI अनुप्रयोगासाठी Foundry प्रोजेक्ट पृष्ठ दिसेल
   ![Project](../../../../../translated_images/mr/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` वर क्लिक करा - तुमच्या प्रोजेक्टमध्ये डिफॉल्ट एजंट provisioned दिसेल
   ![Agents](../../../../../translated_images/mr/06-visit-agents.bccb263f77b00a09.webp)

1. त्याची निवड करा - तुम्हाला एजंट तपशील दिसतील. पुढील लक्षात ठेवा:

      - एजंट नेहमी फाइल सर्च वापरतो (डिफॉल्ट)
      - एजंटचा `Knowledge` सूचित करतो की त्याच्याकडे 32 फाइल्स अपलोड केल्या आहेत (फाइल सर्चसाठी)
      ![Agents](../../../../../translated_images/mr/07-view-agent-details.0e049f37f61eae62.webp)

1. डाव्या मेनूमध्ये `Data+indexes` पर्याय शोधा आणि तपशीलांसाठी क्लिक करा.

      - तुम्हाला माहितीची 32 फाइल्स अपलोड केल्याचे दिसेल.
      - यामध्ये `src/files` अंतर्गत 12 ग्राहक फाइल्स आणि 20 उत्पादन फाइल्स आहेत.
      ![Data](../../../../../translated_images/mr/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**तुम्ही एजंट ऑपरेशनचे प्रमाणीकरण केले आहे!**

1. एजंट प्रतिक्रिया त्या फाइल्समधील ज्ञानावर आधारित असतात.
1. तुम्ही आता त्या डेटाशी संबंधित प्रश्न विचारू शकता आणि आधारित उत्तरे मिळवू शकता.
1. उदाहरण: `customer_info_10.json` मध्ये "Amanda Perez" ने केलेल्या 3 खरेदींचे वर्णन आहे.

Container App एंडपॉइंट असलेले ब्राउझर टॅब पुन्हा उघडा आणि विचारा: `Amanda Perez कडे कोणती उत्पादने आहेत?`. तुम्हाला असे काही दिसावे:

![Data](../../../../../translated_images/mr/09-ask-in-aca.4102297fc465a4d5.webp)

## 6. एजंट प्लेग्राउंड

1. Microsoft Foundry मध्ये `Agents` पानावर परत जा - डिफॉल्ट एजंट निवडा
1. `Try in Playground` पर्यायावर क्लिक करा - तुम्हाला अशा प्रकारचा Playground UI दिसेल
1. त्याच प्रश्नाचा विचार करा: `Amanda Perez कडे कोणती उत्पादने आहेत?`

1. या फाइल लेबल्सवर हवर करा - डेटा तुमच्या क्वेरी आणि दर्शविलेल्या प्रतिसादाशी जुळतो का?

तुम्हाला प्रतिसादाखाली _stats_ रांग देखील दिसेल.

1. कोणत्याही मेट्रिकवर हवर करा - उदा., Safety. तुम्हाला असे काही दिसेल
1. दिलेल्या रेटिंगमुळे प्रतिसादाच्या सुरक्षिततेची समज जुळते का?

## 7. अंगभूत निरीक्षण क्षमता

निरीक्षण क्षमता म्हणजे तुमच्या अनुप्रयोगात instrumentation करून डेटा तयार करणे जे ऑपरेशन्स समजून घेण्यासाठी, डीबग करण्यासाठी आणि ऑप्टिमाईज करण्यासाठी वापरले जाऊ शकते. याची कल्पना घेण्यासाठी:

1. `View Run Info` बटणावर क्लिक करा - तुम्हाला खालील दृश्य दिसेल. हे [Agent tracing](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) चा एक उदाहरण आहे. _तुम्ही हे दृश्य वरच्या मेनूमधील Thread Logs क्लिक करून देखील पाहू शकता._

   - एजंटने वापरलेल्या स्टेप्स आणि टूल्सचा आढावा घ्या
   - प्राप्त प्रतिसादासाठी टोकनांची एकूण संख्या (आऊटपुट टोकन वापरासोबत तुलना करा) समजून घ्या
   - लेटन्सी आणि अंमलबजावणीत वेळ कुठे खर्च होतो ते समजून घ्या

      ![Agent](../../../../../translated_images/mr/10-view-run-info.b20ebd75fef6a1cc.webp)

1. `Metadata` टॅबवर क्लिक करा जेणेकरून पुढील डीबगिंगसाठी उपयोगी असलेले अतिरिक्त संदर्भ attributes पाहता येतील.

      ![Agent](../../../../../translated_images/mr/11-view-run-info-metadata.7966986122c7c2df.webp)


1. `Evaluations` टॅबवर क्लिक करा जेथे एजंटच्या प्रतिसादावर स्वयंचलित मूल्यमापन (जसे की सुरक्षेचे मूल्यांकन, एजंट-विशिष्ट मूल्यांकन) दिसतील.

      ![Agent](../../../../../translated_images/mr/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. शेवटी, साइडबार मेनूमधील `Monitoring` टॅबवर क्लिक करा.

      - प्रदर्शित पानावर `Resource usage` टॅब निवडा - आणि मेट्रिक्स पाहा.
      - वापराच्या दृष्टीने खर्च (टोकन्स) आणि लोड (विनंत्या) ट्रॅक करा.
      - इनपुट प्रक्रिया (पहिला बाइट) आणि आउटपुट (शेवटचा बाइट) यासाठी अनुप्रयोग लेटन्सी ट्रॅक करा.

      ![Agent](../../../../../translated_images/mr/13-monitoring-resources.5148015f7311807f.webp)

## 8. पर्यावरण बदल

आतपर्यंत आपण ब्राउझरमध्ये तैनात करणे पाहिले आहे - आणि आपल्या इन्फ्रास्ट्रक्चरचे provisioning पूर्ण झाले आहे आणि अनुप्रयोग कार्यान्वित आहे हे प्रमाणीकरण केले आहे. परंतु कोड-प्रथम काम करण्यासाठी, आपल्याला स्थानिक विकास पर्यावरणातील संबंधित बदलांचे कॉन्फिगरेशन आवश्यक आहे. `azd` वापरल्याने हे सोपे होते.

1. Azure Developer CLI [पर्यावरण बदलांचा वापर करतो](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) ज्यामुळे अनुप्रयोग तैनात करण्यासाठी सेटिंग्ज संग्रहित आणि व्यवस्थापित केल्या जातात.

1. पर्यावरण बदल `.azure/<env-name>/.env` मध्ये संग्रहित केले जातात - ज्यामुळे ते त्या `env-name` पर्यावरणाशी मर्यादित राहतात ज्याचा वापर तैनाती दरम्यान झाला आणि तुम्हाला वेगवेगळ्या लक्ष्यांसाठी पर्यावरण वेगळे ठेवण्यास मदत होते.

1. पर्यावरण बदल `azd` कमांडने जेव्हा विशिष्ट आदेश चालवला जातो तेव्हा आपोआप लोड होतात (उदा., `azd up`). लक्षात घ्या की `azd` OS-स्तरीय पर्यावरण बदल (उदा., शेलमध्ये सेट केलेले) आपोआप वाचत नाही - त्यासाठी `azd set env` आणि `azd get env` वापरून स्क्रिप्टमध्ये माहिती हस्तांतरीत करा.


काही आदेश वापरून बघूया:

1. या पर्यावरणातील `azd` साठी सेट केलेले सर्व पर्यावरण बदल मिळवा:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      तुम्हाला असे काही दिसेल:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. विशिष्ट मूल्य मिळवा - उदा., मला `AZURE_AI_AGENT_MODEL_NAME` मूल्य सेट केले आहे का ते माहित करायचे आहे

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      तुम्हाला असे काही दिसेल - डिफॉल्टने सेट केलेले नव्हते!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd` साठी नवीन पर्यावरण बदल सेट करा. येथे, आपण एजंट मॉडेल नाव सुधारित करतो. _टीप: केलेले बदल ताबडतोब `.azure/<env-name>/.env` फाइलमध्ये प्रतिबिंबित होतात._

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      आता, आपण मूल्य सेट आहे हे पाहूया:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. काही संसाधने स्थायी असतात (उदा., मॉडेल तैनाती) आणि कारणास्तव फक्त `azd up` चालवल्याने आग्रहाने पुन्हा तैनात होणार नाहीत. चला मूळ तैनाती कशी तोडायची ते पाहूया आणि बदललेल्या पर्यावरण बदलांसह पुन्हा तैनात करूया.

1. **रिफ्रेश करा** जर तुम्ही पूर्वी azd टेम्पलेट वापरून इन्फ्रास्ट्रक्चर तैनात केले असेल - तर सद्य स्थितीच्या आधारावर स्थानिक पर्यावरण बदलांची स्थिती अपडेट करण्यासाठी खालील कमांड वापरा:

      ```bash title="" linenums="0"
      azd env refresh
      ```


      हे दोन किंवा अधिक स्थानिक विकास पर्यावरणांमध्ये (उदा., एकाधिक विकसकांसह टीम) वातावरणातील चल (environment variables) यांना _सिंक्रोनाइझ_ करण्याचा एक शक्तिशाली मार्ग आहे - ज्यामुळे तैनात केलेली पूर्वाधार परिस्थितीचे चल स्थितीसाठी मूळ सत्य म्हणून काम करते. टीम सदस्य फक्त चलांना _रिफ्रेश_ करतात जेणेकरून ते पुन्हा सिंक होतील.
