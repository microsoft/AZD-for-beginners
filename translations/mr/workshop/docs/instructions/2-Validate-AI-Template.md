## 1. परिचय

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) किंवा `azd` हि एक ओपन-सोर्स कमांडलाइन साधन आहे जी अ‍ॅझ्युअरवर अनुप्रयोग तयार करण्यासाठी आणि डिप्लॉय करण्यासाठी विकसकांच्या वर्कफ्लोला सुलभ करते.

[AZD टेम्प्लेट्स](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) ही प्रमाणित रिपॉझिटरीज आहेत ज्यामध्ये नमुना अनुप्रयोगाचा कोड, _इन्फ्रास्ट्रक्चर-अॅज-कोड_ साधने आणि `azd` कॉन्फिगरेशन फायली समाविष्ट असतात, जे एकसंध सोल्यूशन आर्किटेक्चर तयार करतात. इन्फ्रास्ट्रक्चर पुरवठा करणे हे `azd provision` कमांड इतके सोपे आहे - तर `azd up` वापरल्यास तुम्ही इन्फ्रास्ट्रक्चर पुरवठा करू शकता **आणि** एकाच वेळी अ‍ॅप्लिकेशन देखील डिप्लॉय करू शकता!

यामुळे, तुमच्या अनुप्रयोग विकास प्रक्रियेची सुरुवात करणे इतके सोपे होते की योग्य _AZD स्टार्टर टेम्प्लेट_ शोधणे, जे तुमच्या अनुप्रयोग व इन्फ्रास्ट्रक्चर गरजांशी सर्वात जास्त जुळते - आणि नंतर रिपॉझिटरीला तुमच्या परिस्थितीनुसार सानुकूलित करणे.

आम्ही सुरू करण्याआधी, चला पाहूया की तुमच्याकडे Azure Developer CLI इंस्टॉल आहे का.

1. VS Code टर्मिनल उघडा आणि हा आदेश टाका:

      ```bash title="" linenums="0"
      azd version
      ```

1. तुम्हाला काहीसे असे काही दिसेल!

      ```bash title="" linenums="0"
      azd version 1.19.0 (commit b3d68cea969b2bfbaa7b7fa289424428edb93e97)
      ```

**तुम्ही आता azd वापरून टेम्प्लेट निवडण्यास आणि डिप्लॉय करण्यास तयार आहात**

## 2. टेम्प्लेट निवड

Microsoft Foundry प्लॅटफॉर्ममध्ये [शिफारस केलेल्या AZD टेम्प्लेट्सची एक संच](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) उपलब्ध आहे, जी लोकप्रिय सोल्यूशन परिस्थिती जसे की _मल्टी-एजंट वर्कफ्लो ऑटोमेशन_ आणि _मल्टी-मोडल कंटेंट प्रक्रिया_ या सर्व्हिंग करतात. तुम्ही Microsoft Foundry पोर्टलला भेट देऊन या टेम्प्लेट्सना शोधू शकता.

1. [https://ai.azure.com/templates](https://ai.azure.com/templates) येथे भेट द्या
1. विचारल्यावर Microsoft Foundry पोर्टलमध्ये लॉगिन करा - तुम्हाला काहीसे असे दिसेल.

![Pick](../../../../../translated_images/mr/01-pick-template.60d2d5fff5ebc374.webp)


**Basic** पर्याय हे तुमचे स्टार्टर टेम्प्लेट्स आहेत:

1. [ ] [AI Chat सह सुरुवात करा](https://github.com/Azure-Samples/get-started-with-ai-chat) जे एक मूलभूत चॅट अ‍ॅप्लिकेशन _तुमच्या डेटासह_ Azure Container Apps वर डिप्लॉय करते. मूलभूत AI चॅटबॉट परिस्थिती शोधण्यासाठी याचा वापर करा.
1. [X] [AI Agents सह सुरुवात करा](https://github.com/Azure-Samples/get-started-with-ai-agents) जे Foundry Agents सह एक मानक AI एजंट देखील डिप्लॉय करते. टूल्स आणि मॉडेल्सशी संबंधित एजंटिक AI सोल्यूशन्सशी परिचित होण्यासाठी हे वापरा.

दुसऱ्या लिंकवर नवीन ब्राउझर टॅबमध्ये भेट द्या (किंवा संबंधित कार्डसाठी `Open in GitHub` क्लिक करा). तुम्हाला या AZD टेम्प्लेटसाठी रिपॉझिटरी दिसेल. README एक मिनिटासाठी तपासा. अनुप्रयोग आर्किटेक्चर असे दिसते:

![Arch](../../../../../translated_images/mr/architecture.8cec470ec15c65c7.webp)

## 3. टेम्प्लेट सक्रिय करा

चला हा टेम्प्लेट डिप्लॉय करून पाहू आणि पडताळणी करू की तो वैध आहे का. आपण [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) विभागातील मार्गदर्शकाचे पालन करू.

1. [हा दुवा](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) क्लिक करा - डीफॉल्ट क्रिया `Create codespace` याची पुष्टी करा
1. हे नवीन ब्राउझर टॅब उघडते - GitHub Codespaces सत्र पूर्ण होईपर्यंत प्रतीक्षा करा
1. Codespaces मध्ये VS Code टर्मिनल उघडा - खालील कंमांड टाका:

   ```bash title="" linenums="0"
   azd up
   ```

हे जे वर्कफ्लो स्टेप्स ट्रिगर करेल ते पूर्ण करा:

1. तुम्हाला Azure मध्ये लॉगिन करण्यास सांगितले जाईल - प्रमाणीकरणासाठी सूचनांचे पालन करा
1. तुमच्यासाठी एक अनन्य वातावरणाचे नाव द्या - उदा., मी `nitya-mshack-azd` वापरले
1. यामुळे `.azure/` फोल्डर तयार होईल - तुम्हाला त्यात उपफोल्डर तुमच्या env नावासह दिसेल
1. तुम्हाला एका सबस्क्रिप्शन नावाची निवड करण्यास सांगितले जाईल - डीफॉल्ट निवडा
1. तुम्हाला स्थान विचारले जाईल - `East US 2` वापरा

आता, पुरवठा पूर्ण होईपर्यंत प्रतीक्षा करा. **हे 10-15 मिनिटे लागतात**

1. पूर्ण झाल्यावर, तुमच्या कन्सोलमध्ये SUCCESS संदेश असे दिसेल:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. तुमच्या Azure पोर्टलमध्ये आता त्या env नावासह एक प्राव्हिजन केलेले रिसोर्स ग्रुप असेल:

      ![Infra](../../../../../translated_images/mr/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **तुम्ही आता डिप्लॉय केलेल्या इन्फ्रास्ट्रक्चर आणि अ‍ॅप्लिकेशनची पडताळणी करण्यास तयार आहात**.

## 4. टेम्प्लेट पडताळणी

1. Azure पोर्टल [Resource Groups](https://portal.azure.com/#browse/resourcegroups) पृष्ठाला भेट द्या - विचारल्यास लॉगिन करा
1. तुमच्या वातावरणाच्या नावासाठी RG वर क्लिक करा - तुम्हाला वर दाखवलेले पृष्ठ पहायला मिळेल

      - Azure Container Apps रिसोर्स वर क्लिक करा
      - _Essentials_ विभागात (वरच्या उजव्या बाजूला) Application Url वर क्लिक करा

1. तुम्हाला होस्ट केलेले अ‍ॅप्लिकेशन फ्रंट-एंड UI असे दिसेल:

   ![App](../../../../../translated_images/mr/03-test-application.471910da12c3038e.webp)

1. काही [नमुना प्रश्न](https://github.com/Azure-Samples/get-started-with-ai-agents/blob/main/docs/sample_questions.md) विचारून पहा

      1. विचाराः ```What is the capital of France?```
      1. विचाराः ```What's the best tent under $200 for two people, and what features does it include?```

1. तुम्हाला खाली दाखवल्याप्रमाणे उत्तरे मिळतील. _पण हे कसे कार्य करते?_

## 5. एजंट पडताळणी

Azure Container App एका एंडपॉइंटला डिप्लॉय करते जो या टेम्प्लेटसाठी Microsoft Foundry प्रोजेक्टमधील AI एजंटशी संलग्न असतो. याचा अर्थ काय आहे हे पाहूया.

1. Azure पोर्टलमधील तुमच्या रिसोर्स ग्रुपसाठी _Overview_ पृष्ठावर परत जा

1. त्या यादीत `Microsoft Foundry` रिसोर्सवर क्लिक करा

1. तुम्हाला हे दिसेल. `Go to Microsoft Foundry Portal` बटणावर क्लिक करा.
   ![Foundry](../../../../../translated_images/mr/04-view-foundry-project.fb94ca41803f28f3.webp)

1. तुमच्या AI अ‍ॅप्लिकेशनसाठी Foundry प्रोजेक्ट पृष्ठ दिसेल
   ![Project](../../../../../translated_images/mr/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` वर क्लिक करा - तुमच्या प्रोजेक्टमध्ये डीफॉल्ट एजंट प्राव्हिजन केलेला दिसेल
   ![Agents](../../../../../translated_images/mr/06-visit-agents.bccb263f77b00a09.webp)

1. त्याची निवड करा - एजंट तपशील दिसतील. लक्षात घ्या:

      - एजंट डीफॉल्टने (नेहमी) फाइल सर्च वापरतो
      - एजंटचा `Knowledge` सूचित करतो की त्यात 32 फाइल्स अपलोड केलेल्या आहेत (फाइल सर्चसाठी)
      ![Agents](../../../../../translated_images/mr/07-view-agent-details.0e049f37f61eae62.webp)

1. डाव्या मेनूमध्ये `Data+indexes` पर्याय शोधा आणि तपशिलांसाठी क्लिक करा.

      - तुम्हाला ज्ञानासाठी 32 डेटा फाइल्स अपलोड केल्या असल्याचे दिसेल.
      - हे `src/files` अंतर्गत 12 ग्राहक फाइल्स आणि 20 उत्पादन फाइल्सशी जुळतील
      ![Data](../../../../../translated_images/mr/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**तुम्ही एजंट ऑपरेशन पडताळले आहे!**

1. एजंट प्रतिसाद त्या फाइल्समधील ज्ञानावर आधारित असतात.
1. आता तुम्ही त्या डेटाशी संबंधित प्रश्न विचारू शकता, आणि आधारित प्रतिसाद मिळवू शकता.
1. उदाहरण: `customer_info_10.json` हे "Amanda Perez" ने केलेल्या 3 खरेदी वर्णन करते.

कंटेनर अ‍ॅप एंडपॉइंट असलेल्या ब्राउझर टॅबवर परत जा आणि विचाराः `What products does Amanda Perez own?`. तुम्हाला यासारखे काही दिसेल:

![Data](../../../../../translated_images/mr/09-ask-in-aca.4102297fc465a4d5.webp)

## 6. एजंट प्लेग्राउंड

Microsoft Foundry च्या क्षमतांसाठी आणखी थोडी कल्पना मिळवूया, एजंटला Agents Playground मध्ये वापरून बघूया.

1. Microsoft Foundry मधील `Agents` पृष्ठावर परत जा - डीफॉल्ट एजंट निवडा
1. `Try in Playground` पर्यायावर क्लिक करा - तुम्हाला असा Playground UI दिसेल
1. पुन्हा तोच प्रश्न विचारा: `What products does Amanda Perez own?`

    ![Data](../../../../../translated_images/mr/09-ask-in-playground.a1b93794f78fa676.webp)

तुम्हाला तोच (किंवा सारखाच) प्रतिसाद मिळेल - पण तुम्हाला अतिरिक्त माहिती देखील मिळेल जी तुम्हाला तुमच्या एजंटिक अ‍ॅपच्या गुणवत्तेचे, खर्चाचे आणि कार्यक्षमतेचे समजून घेण्यात मदत करेल. उदाहरणार्थ:

1. लक्षात घ्या की प्रतिसादात वापरलेल्या डेटा फाइल्स "ग्राउंडिंग" साठी दर्शविल्या आहेत
1. या कोणत्याही फाईल लेबलवर हवर करा - डेटा तुमच्या क्वेरीशी आणि प्रदर्शित प्रतिसादाशी जुळतो का?

तुम्हाला प्रतिसादाखाली एक _सांख्यिकी_ रांग देखील दिसेल.

1. कोणत्याही मेट्रिकवर हवर करा - उदा., Safety. तुम्हाला काहीसे असे दिसेल
1. दिलेल्या रेटिंगने प्रतिसाद संरक्षण स्तराशी तुमची अंतर्ज्ञान जुळते का?

## 7. अंगभूत ऑब्झर्वेबिलिटी

ऑब्झर्वेबिलिटी म्हणजे तुमच्या अ‍ॅप्लिकेशनमध्ये अशा सूचना लावणे ज्याद्वारे ऑपरेशन्स समजणे, डिबग करणे आणि ऑप्टिमाइझ करणे शक्य होईल. याचा अनुभव घेण्यासाठी:

1. `View Run Info` बटणावर क्लिक करा - तुम्हाला हा दृश्यमान दिसेल. हा [Agent ट्रेसिंग](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) चा एक उदाहरण आहे. _तुम्ही थ्रेड लॉग्स टॉप-लेव्हल मेनूमध्ये क्लिक करून हा दृष्य देखील मिळवू शकता_.

   - एजंटने केलेले रन स्टेप्स आणि वापरलेले टूल्स समजून घ्या
   - प्रतिसादासाठी टोकन काउंट (आउटपुट टोकन्स वापरापेक्षा) समजून घ्या
   - विलंब समजून घ्या आणि वेळ कोणत्या टप्प्यात खर्चला जातो ते पहा

      ![Agent](../../../../../translated_images/mr/10-view-run-info.b20ebd75fef6a1cc.webp)

1. `Metadata` टॅबवर क्लिक करा, जेव्हा तुम्हाला रन साठी अतिरिक्त गुणधर्म दिसतील, जी नंतर समस्या सोडवताना उपयुक्त असू शकतात.

      ![Agent](../../../../../translated_images/mr/11-view-run-info-metadata.7966986122c7c2df.webp)


1. `Evaluations` टॅबवर क्लिक करा, येथे एजंट प्रतिसादावर स्वयंचलित मूल्यमापन केलेले दिसतील. यात सुरक्षा मूल्यांकन (उदा., Self-harm) आणि एजंट-विशिष्ट मूल्यमापन (उदा., Intent resolution, Task adherence) समाविष्ट आहेत.

      ![Agent](../../../../../translated_images/mr/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. मग, साइडबार मेनूमध्ये `Monitoring` टॅबवर क्लिक करा.

      - दिसणाऱ्या पृष्ठावर `Resource usage` टॅब निवडा आणि मेट्रिक्स पहा.
      - खर्च (टोकन्स) आणि लोड (रिक्वेस्ट) च्या संदर्भातील अ‍ॅप्लिकेशन वापर ट्रॅक करा.
      - इनपुट प्रक्रिया प्रथम बाइटपर्यंत आणि आउटपुट अंतिम बाइटपर्यंत अ‍ॅप्लिकेशन विलंब ट्रॅक करा.

## 8. पर्यावरण चर

आत्तापर्यंत, आपण ब्राउझरमध्ये डिप्लॉयमेंट पार पाडली - आणि पडताळले की आमचे इन्फ्रास्ट्रक्चर पुरवठा झाले आणि अ‍ॅप्लिकेशन कार्यरत आहे. पण अ‍ॅप्लिकेशन _कोड-प्रथम_ पद्धतीने काम करण्यासाठी, आमच्या स्थानिक विकास वातावरणात संबंधित व्हेरिएबल्स कॉन्फिगर करणे आवश्यक आहे जे या संसाधनांसह कार्य करण्यासाठी आवश्यक आहेत. `azd` वापरणे हे सोपे करते.

1. Azure Developer CLI [पर्यावरण व्हेरिएबल्स](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) वापरतो, जे अ‍ॅप्लिकेशन डिप्लॉयमेंट्ससाठी कॉन्फिगरेशन सेटिंग्ज संचयित आणि व्यवस्थापित करतात.

1. पर्यावरण व्हेरिएबल्स `.azure/<env-name>/.env` मध्ये संग्रहित केले जातात - हे त्यांना `env-name` च्या डिप्लॉयमेंट दरम्यान वापरलेल्या वातावरणात स्कोप करते आणि त्याच रिपॉझिटरीतील वेगवेगळ्या डिप्लॉयमेंट टार्गेटमधील वातावरण वेगळे ठेवते.

1. पर्यावरण व्हेरिएबल्स `azd` कमांडला ज्यावेळी विशिष्ट आदेश (उदा., `azd up`) चालवायची असते त्यावेळी आपोआप लोड होतात. लक्षात घ्या की `azd` आपोआप _OS-स्तरीय_ पर्यावरण व्हेरिएबल्स (उदा., शेलमध्ये सेट केलेले) वाचत नाही - त्याऐवजी स्क्रिप्टमध्ये माहिती हस्तांतरीत करण्यासाठी `azd set env` आणि `azd get env` वापरा.

चला काही आदेश वापरून पाहू:

1. `azd` साठी या वातावरणात सर्व पर्यावरण व्हेरिएबल्स मिळवा:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      तुम्हाला काहीसे असे दिसेल:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. एखाद्या विशिष्ट व्हॅल्यू मिळवा - उदा., मला माहित करून घ्यायचे आहे की आपण `AZURE_AI_AGENT_MODEL_NAME` व्हॅल्यू सेट केली आहे का

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      तुम्हाला काहीसे असे दिसेल - डीफॉल्टनुसार सेट नाही!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd` साठी एक नवीन पर्यावरण व्हेरिएबल सेट करा. येथे, आपण एजंट मॉडेल नाव अपडेट करत आहोत. _टीप: केलेले कोणतेही बदल `.azure/<env-name>/.env` फायलीमध्ये त्वरित प्रतिबिंबित होतील._

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      आता, आपण सेट केलेली व्हॅल्यू पाहू शकता:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. लक्षात ठेवा की काही संसाधने कायमस्वरूपी असतात (उदा., मॉडेल डिप्लॉयमेंट्स) आणि `azd up` एवढ्याने पुनःडिप्लॉयमेंटसाठी पुरेसे नसते. चला मूळ डिप्लॉयमेंट तुटवून बदलेल्या env vars सह पुन्हा डिप्लॉय करूया.

1. **रिफ्रेश** जर तुम्ही पूर्वी azd टेम्प्लेट वापरून इन्फ्रास्ट्रक्चर डिप्लॉय केले असेल - तर तुम्ही तुमच्या Azure डिप्लॉयमेंटच्या सद्यस्थितीवर आधारित स्थानिक पर्यावरण व्हेरिएबल्सची स्थिति _रिफ्रेश_ करू शकता खालील आदेश वापरून:

      ```bash title="" linenums="0"
      azd env refresh
      ```

      दोन किंवा अधिक स्थानीय विकास वातावरणांमध्ये (उदाहरणार्थ, अनेक विकसक असलेल्या टीममध्ये) पर्यावरण चलांना _समकालीन_ करण्याचा हा एक शक्तिशाली मार्ग आहे - ज्यामुळे तैनात केलेली पायाभूत सुविधा पर्यावरण चलांच्या स्थितीसाठी मुळ सत्य म्हणून काम करते. टीमचे सदस्य फक्त चालं _रिफ्रेश_ करतात जेणेकरून ते पुन्हा समकालीन होतात.
