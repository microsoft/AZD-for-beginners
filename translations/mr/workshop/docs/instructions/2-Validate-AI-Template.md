## 1. परिचय

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) किंवा `azd` हे एक मुक्त स्रोत कमांडलाइन साधन आहे जे Azure वर अनुप्रयोग तयार करताना आणि वितरित करताना विकसक कार्यप्रवाह सुलभ करते.

[AZD टेम्पलेट्स](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) हे प्रमाणीकृत रिपॉझिटरीज आहेत ज्यात नमुना अनुप्रयोग कोड, _infrastructure-as-code_ साधने, आणि `azd` संरचना फाइल्स समाविष्ट असतात ज्यामुळे सुसंगत सोल्यूशन आर्किटेक्चर तयार होते. इन्फ्रास्ट्रक्चर तयार करणे केवळ `azd provision` कमांडइतके सोपे होते - तर `azd up` वापरून इन्फ्रास्ट्रक्चर तयार करण्याबरोबरच तुमचा अनुप्रयोग एकाच वेळी वितरित केला जातो!

या परिणामी, तुमच्या अनुप्रयोग आणि इन्फ्रास्ट्रक्चरच्या गरजेनुसार सगळ्यात जवळचा _AZD स्टार्टर टेम्पलेट_ शोधून अनुप्रयोग विकास प्रक्रिया सुरू करणे खूप सोपे होते - नंतर रिपॉझिटरी तुमच्या परिस्थितीनुसार सानुकूलित करू शकता.

सुरू करण्यापूर्वी, तुमच्याकडे Azure Developer CLI इन्स्टॉल आहे याची खात्री करूया.

1. VS कोड टर्मिनल उघडा आणि हा आदेश टाका:

      ```bash title="" linenums="0"
      azd version
      ```

1. तुम्हाला अशा प्रकारचे काही स्क्रीन दिसेल!

      ```bash title="" linenums="0"
      azd version 1.25.6 (commit <current-build>)
      ```

**आता तुम्ही azd सह टेम्पलेट निवडून ते वितरित करण्यास तयार आहात**

## 2. टेम्पलेट निवड

Microsoft Foundry प्लॅटफॉर्ममध्ये [सूचित केलेल्या AZD टेम्पलेट्सचा संच](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) आहे जो लोकप्रिय सोल्यूशन परिस्थिती जसे की _मल्टी-एजंट वर्कफ्लो ऑटोमेशन_ आणि _मल्टी-मोडल कंटेंट प्रोसेसिंग_ कव्हर करतो. तुम्ही हे टेम्पलेट्स Microsoft Foundry पोर्टलवर भेट देऊन सुद्धा शोधू शकता.

1. भेट द्या [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. सुचविल्यावर Microsoft Foundry पोर्टलमध्ये लॉग इन करा - तुम्हाला अशा प्रकारचा स्क्रीन दिसेल.

![Pick](../../../../../translated_images/mr/01-pick-template.60d2d5fff5ebc374.webp)

**Basic** पर्याय हे तुमचे स्टार्टर टेम्पलेट्स आहेत:

1. [ ] [AI Chat सह सुरुवात करा](https://github.com/Azure-Samples/get-started-with-ai-chat) जे मूलभूत चॅट अनुप्रयोग _तुमच्या डेटासह_ Azure Container Apps मध्ये डिप्लॉय करते. हे AI चॅटबॉट परिस्थिती शोधण्यासाठी वापरा.
1. [X] [AI Agents सह सुरुवात करा](https://github.com/Azure-Samples/get-started-with-ai-agents) जे Foundry Agents सह एक मानक AI एजंट देखील डिप्लॉय करते. याचा वापर एजंटिक AI सोल्यूशन्स, टूल्स आणि मॉडेल्सशी परिचित होण्यासाठी करा.

दुसऱ्या लिंकला नवीन ब्राउझर टॅबमध्ये भेट द्या (किंवा संबंधित कार्डसाठी `Open in GitHub` क्लिक करा). तुम्हाला या AZD टेम्पलेटसाठी रिपॉझिटरी दिसेल. README कागदपत्र तपासा. अनुप्रयोग आर्किटेक्चर अशी दिसते:

![Arch](../../../../../translated_images/mr/architecture.8cec470ec15c65c7.webp)

## 3. टेम्पलेट सक्रियता

या टेम्पलेटला वितरित करून वैध असल्याची खात्री करूया. आपण [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) विभागातील मार्गदर्शक तत्वांचे पालन करू.

1. टेम्पलेट रिपॉझिटरीसाठी कार्यरत वातावरण निवडा:

      - **GitHub Codespaces**: [हा दुवा](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) क्लिक करा आणि `Create codespace` याची पुष्टी करा
      - **स्थानिक क्लोन किंवा dev कंटेनर**: `Azure-Samples/get-started-with-ai-agents` क्लोन करा आणि VS कोड मध्ये उघडा

1. VS कोड टर्मिनल तयार होईपर्यंत थांबा, नंतर खालील कमांड टाका:

   ```bash title="" linenums="0"
   azd up
   ```

या प्रक्रियेतिल चरण पूर्ण करा:

1. तुम्हाला Azure मध्ये लॉग इन करण्यास सांगितले जाईल - प्रमाणीकरणासाठी सूचना फॉलो करा
1. तुमच्यासाठी एक अद्वितीय वातावरण नाव द्या - उदा., मी `nitya-mshack-azd` वापरले
1. यामुळे `.azure/` फोल्डर तयार होईल - तुम्हाला पर्यावरणाच्या नावाने एक सबफोल्डर दिसेल
1. तुम्हाला सदस्यता नाव निवडण्यास सांगितले जाईल - डिफॉल्ट निवडा
1. तुम्हाला स्थान (location) विचारले जाईल - `East US 2` वापरा

आता, पाहा की प्रोव्हिजनिंग पूर्ण होईपर्यंत थांबा. **यासाठी 10-15 मिनिटे लागतात**

1. पूर्ण झाल्यावर, तुमच्या कन्सोलमध्ये अशी यशस्वीरित्या संदेश दिसेल:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```

1. तुमच्या Azure पोर्टलमध्ये आता त्या पर्यावरण नावाचा एक संसाधन गट तयार झाला असेल:

      ![Infra](../../../../../translated_images/mr/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **आता तुम्ही वितरित केलेल्या इन्फ्रास्ट्रक्चर आणि अनुप्रयोगाची वैधता तपासू शकता**.

## 5. एजंट वैधता

Azure Container App असा एंडपॉइंट तयार करते जो या टेम्पलेटसाठी Microsoft Foundry प्रकल्पात प्रोव्हिजन केलेल्या AI एजंटशी जोडलेले आहे. हे काय आहे ते पाहूया.

1. तुमच्या संसाधन समूहासाठी Azure पोर्टल _Overview_ पृष्ठावर परत जा

1. त्या यादीतील `Microsoft Foundry` रिसोर्सवर क्लिक करा

1. तुम्हाला हे दिसेल. `Go to Microsoft Foundry Portal` बटण क्लिक करा. 
   ![Foundry](../../../../../translated_images/mr/04-view-foundry-project.fb94ca41803f28f3.webp)

1. तुमच्या AI अनुप्रयोगासाठी Foundry प्रकल्प पृष्ठ दिसेल
   ![Project](../../../../../translated_images/mr/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` क्लिक करा - तुमच्या प्रकल्पात डिफॉल्ट एजंट प्रोव्हिजन केला गेला आहे असं दिसेल
   ![Agents](../../../../../translated_images/mr/06-visit-agents.bccb263f77b00a09.webp)

1. त्यावर क्लिक करा - एजंट तपशील दिसतील. पुढील बाबी लक्षात घ्या:

      - एजंट नेहमी फाइल सर्च वापरतो (डिफॉल्टने)
      - एजंट `Knowledge` मध्ये 32 फाइल्स अपलोड केलेल्या आहेत (फाइल सर्चसाठी)
      ![Agents](../../../../../translated_images/mr/07-view-agent-details.0e049f37f61eae62.webp)

1. डाव्या मेनूमध्ये `Data+indexes` पर्याय शोधा आणि तपशीलासाठी क्लिक करा. 

      - तुम्हाला 32 ज्ञानासाठी अपलोड केलेल्या डेटा फाइल्स दिसतील.
      - हे `src/files` अंतर्गत 12 ग्राहक फाइल्स आणि 20 उत्पादन फाइल्सशी संबंधित आहेत 
      ![Data](../../../../../translated_images/mr/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**तुम्ही एजंट ऑपरेशनची वैधता तपासली आहे!** 

1. एजंट रेस्पाँसेस त्या फाइल्समधील ज्ञानावर आधारित आहेत. 
1. तुम्ही आता त्या डेटाशी संबंधित प्रश्न विचारू शकता आणि माहितीवर आधारित उत्तरे मिळवू शकता.
1. उदाहरणार्थ: `customer_info_10.json` "Amanda Perez" यांनी केलेले 3 खरेदी यांचे वर्णन करतो

कंटेनर अॅप एंडपॉइंट असलेल्या ब्राउझर टॅबवर परत जा आणि विचारा: `Amanda Perez कडे कोणती उत्पादने आहेत?`. तुम्हाला काहीसे असे दिसेल:

## 6. एजंट प्लेग्राउंड

1. Microsoft Foundry मध्ये `Agents` पृष्ठावर परत जा - डिफॉल्ट एजंट निवडा
1. `Try in Playground` पर्याय क्लिक करा - तुम्हाला अशी प्लेग्राउंड UI दिसेल
1. तोच प्रश्न विचारा: `Amanda Perez कडे कोणती उत्पादने आहेत?`

1. लक्ष द्या की प्रतिसादात वापरलेल्या डेटापत्रकांचे संदर्भ दिले आहेत जे प्रतिसाद "भूमिका" करतात
1. कोणत्याही फाइल लेबलवर माउस ठेवून पाहा - का डेटा तुमच्या प्रश्नाशी आणि प्रतिसादाशी जुळतो?

1. कोणत्याही मेट्रिकवर माउस ठेवा - उदा., Safety. तुम्हाला असे काही दिसेल
1. दिलेली रेटिंग तुमच्या प्रतिसाद सुरक्षा स्तराच्या अनुभवाशी जुळते का?

## 7. अंगभूत निरीक्षण

निरीक्षण म्हणजे तुमच्या अनुप्रयोगामध्ये उपकरणे लावणे जे डेटा तयार करतात ज्याचा वापर त्याच्या ऑपरेशन्स समजून घेण्यासाठी, डीबग करण्यासाठी, आणि ऑप्टिमाइझ करण्यासाठी केला जातो. याचे उदाहरण पाहूया:

1. `View Run Info` बटण क्लिक करा - याप्रमाणे दृश्य दिसेल. हे [Agent tracing](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) चे उदाहरण आहे. _तुम्ही हे दृश्य टॉप-लेव्हल मेनूतील Thread Logs क्लिक करूनही प्राप्त करू शकता_.

   - एजंटद्वारे चालवलेल्या चरणांचा आणि साधनांचा अनुभव घ्या
   - प्रतिसादासाठी Token संख्या (आउटपुट टोकन्स वापरासह) समजून घ्या
   - विलंब लक्षात घ्या आणि वेळ कुठे खर्च होत आहे ते समजून घ्या

      ![Agent](../../../../../translated_images/mr/10-view-run-info.b20ebd75fef6a1cc.webp)

1. `Metadata` टॅब क्लिक करा जेणे करून रनसाठी अतिरिक्त वैशिष्ट्ये पाहू शकता, जे नंतर डीबगिंगसाठी उपयुक्त ठरतील.   

      ![Agent](../../../../../translated_images/mr/11-view-run-info-metadata.7966986122c7c2df.webp)

1. `Evaluations` टॅब क्लिक करा जेथे एजंट प्रतिसादावर स्वयंचलित मूल्यांकन दर्शवले आहे. यात सुरक्षा मूल्यांकन (उदा., Self-harm) आणि एजंट-विशिष्ट मूल्यांकन (उदा., Intent resolution, Task adherence) समाविष्ट आहेत.

      ![Agent](../../../../../translated_images/mr/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. शेवटी, साइडबार मेनूमधील `Monitoring` टॅब क्लिक करा.

      - दिसणाऱ्या पृष्ठावर `Resource usage` टॅब निवडा - मेट्रिक्स पहा.
      - खर्च (टोकन्स) आणि लोड (विनंत्यां) मधून अनुप्रयोग वापराचे विश्लेषण करा.
      - प्रथम बाइट (इनपुट प्रक्रिया) व अंतिम बाइट (आउटपुट) पर्यंतचा अनुप्रयोग विलंब ट्रॅक करा.

      ![Agent](../../../../../translated_images/mr/13-monitoring-resources.5148015f7311807f.webp)

## 8. पर्यावरण चल (Environment Variables)

आत्तापर्यंत आपण ब्राउझरमध्ये वितरणाचे पाऊल चालवत आलो आहोत - आणि आपल्या इन्फ्रास्ट्रक्चरची पूर्तता व अनुप्रयोग योग्यरित्या चालत असल्याची तपासणी केली आहे. पण _कोड-पहिले_ पद्धतीने काम करण्यासाठी आपल्याला स्थानिक विकास वातावरणाला संबंधित चल (variables) सह कॉन्फिगर करावे लागेल. `azd` वापरल्यामुळे हे सोपे होते.

1. Azure Developer CLI [पर्यावरण चल वापरतो](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) जे अनुप्रयोग वितरणासाठी कॉन्फिगरेशन सेटिंग्ज संचयित आणि व्यवस्थापित करतात.

1. पर्यावरण चल `.azure/<env-name>/.env` मध्ये संग्रहित होतात - यामुळे ते `env-name` पर्यावरणासाठी स्कोप होतात जो वितरणादरम्यान वापरला जातो आणि यामुळे एकाच रिपोजिटरीतील वेगवेगळ्या वितरण हेतूसाठी पर्यावरण वेगळे राहतात.

1. `azd` कमांड जेव्हा कोणतीही विशिष्ट कमांड (उदा., `azd up`) चालवते तेव्हा पर्यावरण चल आपोआप लोड होतात. लक्षात ठेवा, `azd` आपोआप _OS-स्तरीय_ पर्यावरण चल (उदा., शेलमध्ये सेट केलेले) वाचत नाही - त्याऐवजी सिक्रिप्टमध्ये माहिती ट्रान्सफर करण्यासाठी `azd set env` व `azd get env` वापरा.

चला काही कमांड्स वापरून पाहूया:

1. या पर्यावरणात `azd` साठी सेट केलेले सर्व पर्यावरण चल मिळवा:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      तुम्हाला असे काही दिसेल:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. विशिष्ट मूल्य मिळवा - उदा., मला जाणून घ्यायचे आहे की `AZURE_AI_AGENT_MODEL_NAME` मूल्य सेट केले आहे का?

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      तुम्हाला असे दिसेल - हे डिफॉल्टने सेट नव्हते!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd` साठी नवीन पर्यावरण चल सेट करा. येथे आपण एजंट मॉडेल नाव अपडेट करत आहोत. _टीप: कोणतीही बदल लगेच `.azure/<env-name>/.env` फाइलमध्ये प्रतिबिंबित होतात.

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      आता, आपण पाहू की मूल्य सेट झाले आहे:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. लक्षात ठेवा की काही संसाधने स्थायी असतात (उदा., मॉडेल डिप्लॉयमेंट्स) आणि त्यासाठी `azd up` पेक्षा अधिक काही करावे लागेल. चला मूळ वितरण तुडवून, बदललेल्या पर्यावरण चलांसह पुन्हा डिप्लॉय करूया.

1. **रिफ्रेश** जर तुम्ही यापूर्वी azd टेम्पलेट वापरून इन्फ्रास्ट्रक्चर वितरित केले असेल - तर तुम्ही तुमच्या Azure वितरणाच्या वर्तमान स्थितीच्या आधारे स्थानिक पर्यावरण चलांचे स्थिती _रिफ्रेश_ करू शकता या कमांडद्वारे:

      ```bash title="" linenums="0"
      azd env refresh
      ```

      हे दोन किंवा अधिक स्थानिक विकास वातावरणांमध्ये (उदा., अनेक विकासकांसह टीम) पर्यावरणीय चल बदल स्वयंचलितपणे सुसंगत करण्याचा एक शक्तिशाली मार्ग आहे - जेणेकरून तैनात केलेली पायाभूत सुविधा पर्यावरणीय चल बदलांच्या स्थितीसाठी खरा आधार म्हणून कार्य करू शकेल. संघ सदस्य फक्त चल बदलांना _रिफ्रेश_ करतात जेणेकरून ते पुन्हा सुसंगत होतील.
