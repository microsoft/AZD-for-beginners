AZD टेम्पलेट्स आणि Azure Developer CLI (`azd`) सह आपण मानकीकृत रेपॉझिटरीजसह आपल्या AI विकास प्रवासाला जलद प्रारंभ करू शकतो जे नमुना कोड, इन्फ्रास्ट्रक्चर आणि कॉन्फिगरेशन फाइल्स - तैनात करण्यास तयार _स्टार्टर_ प्रकल्पाच्या स्वरूपात प्रदान करतात.

**पण आता, आपल्याला प्रकल्पाची रचना आणि कोडबेस समजून घ्यावी लागेल - आणि AZD टेम्पलेट सानुकूलित करण्यासाठी सक्षम व्हावे लागेल - AZD ची कोणतीही पूर्वानुभव किंवा समज न घेता!**

## 1. GitHub Copilot सक्रिय करा

### 1.1 GitHub Copilot Chat इंस्टॉल करा

आता [GitHub Copilot with Agent Mode](https://code.visualstudio.com/docs/copilot/chat/chat-agent-mode) शोधण्याची वेळ आली आहे. आता आपण नैसर्गिक भाषेचा वापर करून आपल्या कार्याचे वर्णन उच्च पातळीवर करू शकतो, आणि अंमलबजावणीमध्ये सहाय्य मिळवू शकतो. या प्रयोगशाळेसाठी, आपण [Copilot Free plan](https://github.com/github-copilot/signup) वापरणार आहोत ज्याला पूर्णता आणि चॅट संवादांसाठी मासिक मर्यादा आहे.

ही विस्तार मार्केटप्लेसमधून स्थापित केली जाऊ शकते, पण ती आपल्याला आधीच Codespaces वातावरणात उपलब्ध असायला हवी. _Copilot चिन्हाच्या ड्रॉप-डाउन मधून `Open Chat` वर क्लिक करा - आणि `What can you do?` सारखा प्रॉम्प्ट टाइप करा_ - आपल्याला लॉगिन करण्यास सांगितले जाऊ शकते. **GitHub Copilot Chat तयार आहे**.

### 1.2. MCP सर्व्हर्स इंस्टॉल करा

एजंट मोड प्रभावी होण्यासाठी, त्याला योग्य उपकरणे आवश्यक आहेत ज्यामुळे तो ज्ञान पुनर्प्राप्त करू शकतो किंवा क्रिया घेऊ शकतो. येथे MCP सर्व्हर्स मदत करू शकतात. आपण पुढील सर्व्हर्स कॉन्फिगर करू:

1. [Azure MCP Server](../../../../../workshop/docs/instructions)
1. [Microsoft Docs MCP Server](../../../../../workshop/docs/instructions)

हे सक्रिय करण्यासाठी:

1. `.vscode/mcp.json` नावाची फाइल तयार करा जर ती अस्तित्वात नसेल तर
1. खालील कॉपी करा आणि सर्व्हर्स सुरू करा!
   ```json title=".vscode/mcp.json"
   {
      "servers": {
         "Azure MCP Server": {
            "command": "npx",
            "args": [
            "-y",
            "@azure/mcp@latest",
            "server",
            "start"
            ]
         },
         "microsoft.docs.mcp": {
            "type": "http",
            "url": "https://learn.microsoft.com/api/mcp"
         }
      }
   }
   ```

??? warning "आपल्याला `npx` इन्स्टॉल नसल्याचा त्रुटी येऊ शकतो (दुरुस्ती साठी विस्तृत करा)"

      यासाठी, `.devcontainer/devcontainer.json` फाइल उघडा आणि फिचर्स सेक्शनमध्ये खालील ओळ जोडा. नंतर कंटेनर पुन्हा बनवा. आता आपल्याकडे `npx` इन्स्टॉल केलेले असावे.

      ```title="" linenums="0"
         "features": {
            "ghcr.io/devcontainers/features/node:1": {},
            ...
         },
      ```

### 1.3. GitHub Copilot Chat चाचणी करा

**सुरुवातीला VS Code कमांड लाइनवरून Azure सह प्रमाणीकरण करण्यासाठी `az login` वापरा.**

आता आपल्याला आपली Azure सबस्क्रिप्शन स्थिती क्वेरी करू शकता, आणि तैनात केलेल्या संसाधनांबद्दल किंवा कॉन्फिगरेशनबद्दल प्रश्न विचारू शकता. हे प्रॉम्प्ट वापरून पहा:

1. `List my Azure resource groups`
1. `#foundry list my current deployments`

आपण Azure दस्तऐवजीकरणाबद्दल प्रश्नही विचारू शकता आणि Microsoft Docs MCP सर्व्हरवर आधारित प्रतिसाद प्राप्त करू शकता. हे प्रॉम्प्ट वापरून पहा:

1. `#microsoft_docs_search What is Azure Developer CLI?`
1. `#microsoft_docs_search Show me a Python tutorial to chat with deployed model`

किंवा एखाद्या कार्यासाठी कोड स्निपेट्स मागू शकता. हा प्रॉम्प्ट वापरून पहा.

1. `Give me a Python code example that uses AAD for an interactive chat client`

`Ask` मोडमध्ये, यामुळे आपल्याला कोड प्राप्त होईल ज्याला आपण कॉपी-पेस्ट करून वापरू शकता. `Agent` मोडमध्ये, यामुळे एक पाऊल पुढे जाऊन संबंधित संसाधने तयार केली जाऊ शकतात - सेटअप स्क्रिप्ट्स आणि दस्तऐवजीकरणांसह - जेणेकरून आपण ते कार्य अंमलात आणू शकता.

**आता आपण टेम्पलेट रेपॉझिटरी अन्वेषण करण्यास सज्ज आहात**

## 2. आर्किटेक्चरचे विघटन करा

??? prompt "प्रश्न: docs/images/architecture.png मधील अॅप्लिकेशन आर्किटेक्चर एका परिच्छेदात समजावून सांगा"

      हे अॅप्लिकेशन Azure वर आधारित AI-संचालित चॅट अॅप्लिकेशन आहे जे आधुनिक एजंट-आधारित आर्किटेक्चर दाखवते. या सोल्यूशनचा केंद्रबिंदू असा Azure Container App आहे जो मुख्य अॅप्लिकेशन कोड होस्ट करतो, जो वापरकर्त्याच्या इनपुटला प्रक्रिया करतो आणि AI एजंटद्वारे बुद्धिमान प्रतिसाद तयार करतो.
      
      आर्किटेक्चर Microsoft Foundry Project वर आधारित AI क्षमता प्रदान करते, जे Azure AI Services शी कनेक्ट होते ज्यात GPT-4o-mini सारखे भाषा मॉडेल आणि एजंट कार्यक्षमता असते. वापरकर्त्यांचे संवाद React-आधारित फ्रंटएंडमधून FastAPI बॅकएंडकडे जातात जे AI एजंट सेवेशी संवाद साधून संदर्भात्मक प्रतिसाद तयार करतो.
      
      प्रणालीमध्ये ज्ञान पुर्नप्राप्ती क्षमता समाविष्ट आहे, जी फाइल शोध किंवा Azure AI Search सेवेद्वारे एजंटला अपलोड केलेल्या दस्तऐवजांमधील माहिती वापरण्याची आणि सादर करण्याची परवानगी देते.
      
      ऑपरेशनल उत्कृष्टतेसाठी, आर्किटेक्चर Application Insights आणि Log Analytics Workspace द्वारे संपूर्ण निरीक्षण प्रदान करते, ज्यात ट्रेसिंग, लॉगिंग आणि कार्यक्षमता सुधारणा समाविष्ट आहे.
      
      Azure Storage ऍप्लिकेशन डेटा आणि फाइल अपलोडसाठी ब्लॉब स्टोरेज पुरवते, तर Managed Identity Azure संसाधनांदरम्यान सुरक्षित प्रवेश सुनिश्चित करते ज्यामुळे क्रेडेन्शियल्स साठवण्याची आवश्यकता नाही. उद्धृत सोल्यूशन मागणीवर आधारित कंटेनराइज्ड अॅप्लिकेशन आपोआप स्केल करते आणि Azure च्या व्यवस्थापित सेवा इकोसिस्टमद्वारे अंतर्भूत सुरक्षा, निरीक्षण आणि CI/CD क्षमता प्रदान करते.

### 3.1. मुख्य अॅप आर्किटेक्चर

हा टेम्पलेट **पूर्ण-स्टॅक वेब अॅप्लिकेशन** पॅटर्नचा वापर करतो ज्यात:

- **बॅकएंड**: Azure AI एकत्रीकरणासह Python FastAPI
- **फ्रंटएंड**: TypeScript/React सह Vite बिल्ड सिस्टम
- **इन्फ्रास्ट्रक्चर**: क्लाउड संसाधनांसाठी Azure Bicep टेम्पलेट्स
- **कंटेनरायझेशन**: सातत्यपूर्ण तैनातीसाठी Docker

### 3.2 इन्फ्रा एज कोड (bicep)

इन्फ्रास्ट्रक्चर लेयर **Azure Bicep** टेम्पलेट्सचा वापर करते जे मॉड्युलर पद्धतीने संघटित आहेत:

   - **`main.bicep`**: सर्व Azure संसाधने समन्वयित करते
   - **`core/` मॉड्युल्स**: विविध सेवांसाठी पुनर्वापरयोग्य घटक
      - AI सेवा (Azure OpenAI, AI Search)
      - कंटेनर होस्टिंग (Azure Container Apps)
      - मॉनिटरिंग (Application Insights, Log Analytics)
      - सुरक्षा (Key Vault, Managed Identity)

### 3.3 अॅप्लिकेशन स्रोत (`src/`)

**बॅकएंड API (`src/api/`)**:

- FastAPI-आधारित REST API
- Foundry Agents एकत्रीकरण
- ज्ञान पुनर्प्राप्तीसाठी शोध निर्देशांक व्यवस्थापन
- फाइल अपलोड आणि प्रक्रिया क्षमता

**फ्रंटएंड (`src/frontend/`)**:

- आधुनिक React/TypeScript SPA
- वेगवान विकास आणि ऑप्टिमाइज्ड बिल्डसाठी Vite
- एजंट संवादासाठी चॅट इंटरफेस

**ज्ञान आधार (`src/files/`)**:

- ग्राहक आणि उत्पादन डेटा नमुने
- फाइल-आधारित ज्ञान पुनर्प्राप्तीचे प्रदर्शन
- JSON आणि Markdown स्वरूपातील उदाहरणे

### 3.4 देवऑप्स आणि ऑटोमेशन

**स्क्रिप्ट्स (`scripts/`)**:

- क्रॉस-प्लॅटफॉर्म PowerShell आणि Bash स्क्रिप्ट्स
- वातावरण मान्यता आणि सेटअप
- पोस्ट-तैनाती कॉन्फिगरेशन
- मॉडेल कोटा व्यवस्थापन

**Azure Developer CLI इंटिग्रेशन**:

- `azd` वर्कफ्लो साठी `azure.yaml` कॉन्फिगरेशन
