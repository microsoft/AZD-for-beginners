## आढावा

हा अध्याय तुम्हाला Azure Developer CLI सह काम करताना सामान्य समस्या ओळखण्यास आणि निराकरण करण्यास मदत करतो. प्रक्षेपण अयशस्वीतेपासून AI-विशिष्ट समस्या पर्यंत.

> `azd 1.25.6` वरील सत्यापन जून 2026 मध्ये केले गेले.

- AI सेवा कनेक्टिव्हिटी समस्या सोडवाल
- समस्या निवारणासाठी Azure पोर्टल आणि CLI वापराल

## 🚨 त्वरीत निराकरणे

### प्रमाणीकरण समस्या
```bash
# AZD वर्कफ्लो साठी आवश्यक
azd auth login

# जर आपण थेट Azure CLI आज्ञा देखील वापरत असाल तर पर्यायी
az login

azd auth status
```

### प्रोव्हिजनिंग अयशस्वीता
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### संसाधन संघर्ष
```bash
azd down --force --purge
azd env new different-name
azd up
```

### कोटा ओलांडला
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 त्रुटी कोड संदर्भ

| त्रुटी | कारण | उपाय |
|-------|-------|----------|
| `AuthenticationError` | लॉग इन केलेले नाही | `azd auth login` |
| `ResourceNotFound` | संसाधन गहाळ आहे | संसाधन नावे तपासा |
| `QuotaExceeded` | सदस्यता सीमेपेक्षा अधिक | कोटा वाढविण्याची विनंती करा |
| `InvalidTemplate` | बाईसेप सिंटॅक्स त्रुटी | `az bicep build` |
| `Conflict` | संसाधन अस्तित्वात आहे | नवीन नाव वापरा किंवा हटवा |
| `Forbidden` | अपर्याप्त परवानग्या | RBAC भूमिका तपासा |

## 🔄 रीसेट आणि पुनर्प्राप्ती

```bash
# सॉफ्ट रीसेट (संसाधने ठेवा, कोड पुन्हा तैनात करा)
azd deploy --force

# हार्ड रीसेट (सर्व काही हटवा, नवीन सुरुवात करा)
azd down --force --purge
azd up
```
