## आढावा

हा प्रकरण Azure Developer CLI वापरताना सामान्य समस्या निदान आणि निराकरण करण्यात मदत करते. वितरण अपयशांपासून AI-विशिष्ट समस्यांपर्यंत सर्वकाही.

> मार्च 2026 मध्ये `azd 1.23.12` सोबत पडताळले.

## शिकण्याची उद्दिष्टे

या प्रकरणाची पूर्णता केल्यावर, आपण:
- सामान्य AZD डिप्लॉयमेंट अयशस्वीतेचे निदान करू शकता
- प्रमाणीकरण आणि परवानगी समस्या डीबग करू शकता
- AI सेवांच्या कनेक्टिव्हिटी समस्या सोडवू शकता
- समस्या निवारणासाठी Azure Portal आणि CLI वापरू शकता

## 🚨 त्वरित उपाय

### प्रमाणीकरण समस्या
```bash
# AZD वर्कफ्लोसाठी आवश्यक
azd auth login

# जर आपण Azure CLI आदेश थेट वापरत असाल तर हे ऐच्छिक आहे
az login

azd auth status
```

### विनियोजन अयशस्वी
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### संसाधन संघर्ष
```bash
azd down --force --purge
azd env new different-name
azd up
```

### कोटा ओलांडले
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 त्रुटी कोड संदर्भ

| त्रुटी | कारण | उपाय |
|-------|-------|----------|
| `AuthenticationError` | लॉगिन केलेले नाही | `azd auth login` |
| `ResourceNotFound` | संसाधन अनुपस्थित | संसाधनांची नावे तपासा |
| `QuotaExceeded` | सदस्यत्व मर्यादा ओलांडल्या आहेत | कोटा वाढविण्याची विनंती करा |
| `InvalidTemplate` | Bicep सिन्टॅक्स त्रुटी | `az bicep build` |
| `Conflict` | संसाधन आधीच अस्तित्वात आहे | नवीन नाव वापरा किंवा हटवा |
| `Forbidden` | अपर्याप्त परवानग्या | RBAC भूमिका तपासा |

## 🔄 रीसेट आणि पुनर्प्राप्ती

```bash
# सॉफ्ट रीसेट (संसाधने ठेवणे, कोड पुन्हा तैनात करणे)
azd deploy --force

# हार्ड रीसेट (सर्व काही हटवणे, नव्याने सुरुवात करणे)
azd down --force --purge
azd up
```
