## आढावा

हा अध्याय तुम्हाला Azure Developer CLI वापरताना सामान्य समस्या ओळखण्यास आणि सोडविण्यास मदत करतो. तैनातीच्या अपयशापासून AI-विशिष्ट समस्या पर्यंत.

> `azd 1.27.1` मध्ये जुलै 2026 मध्ये तपासले गेले.

## शिक्षण उद्दिष्टे

हा अध्याय पूर्ण केल्यावर तुम्ही:
- सामान्य AZD तैनातीचे अपयश ओळखू शकाल
- प्रमाणीकरण आणि परवानगीच्या समस्या डीबग करू शकाल
- AI सेवांच्या कनेक्टिव्हिटी समस्या सोडवू शकाल
- समस्या निवारणासाठी Azure Portal आणि CLI वापरू शकाल

## 🚨 जलद दुरुस्त्या

### प्रमाणीकरण समस्या
```bash
# AZD वर्कफ्लो साठी आवश्यक
azd auth login

# जर आपण थेट Azure CLI कमांड्स वापरत असाल तर पर्यायी
az login

azd auth status
```

### पूर्तता अयशस्वी
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### संसाधन संघर्ष
```bash
azd down --force --purge
azd env new different-name
azd up
```

### कोटा ओलांडले
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 त्रुटी कोड संदर्भ

| त्रुटी | कारण | उपाय |
|-------|-------|----------|
| `AuthenticationError` | लॉगिन केलेले नाही | `azd auth login` |
| `ResourceNotFound` | संसाधन गायब | संसाधन नावे तपासा |
| `QuotaExceeded` | सदस्यता मर्यादा | कोटा वाढीची विनंती करा |
| `InvalidTemplate` | Bicep वाक्यरचना त्रुटी | `az bicep build` |
| `Conflict` | संसाधन अस्तित्वात आहे | नवीन नाव वापरा किंवा हटवा |
| `Forbidden` | अपुरी परवानग्या | RBAC भूमिका तपासा |

## 🔄 रीसेट आणि पुनर्प्राप्ती

```bash
# सॉफ्ट रीस्ट्रेट (स्रोत ठेवणे, कोड पुनर्संचयित करणे)
azd deploy --force

# हार्ड रीस्ट्रेट (सगळं हटवा, नव्याने सुरुवात करा)
azd down --force --purge
azd up
```
