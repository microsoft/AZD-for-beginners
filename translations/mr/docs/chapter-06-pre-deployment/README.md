**📚 कोर्स**: [AZD नवशिक्यांसाठी](../../README.md) | **⏱️ कालावधी**: 1 तास | **⭐ कठीणपणा**: मध्यम

## अवलोकन

हा अध्याय तुमच्या अनुप्रयोगाची तैनाती करण्यापूर्वी आवश्यक नियोजन आणि पडताळणी पावले कव्हर करतो. योग्य क्षमता नियोजन, SKU निवड आणि प्रीफ्लाइट तपासण्या करून महागड्या चुका टाळायला शिका.

> `azd 1.23.12` च्या आधारावर मार्च 2026 मध्ये पडताळले.

## अभ्यास उद्दिष्टे

हा अध्याय पूर्ण केल्यावर, आपण:
- तैनातीपूर्वी प्रीफ्लाइट तपासण्या चालवाल
- क्षमता नियोजन करा व संसाधन गरजा अंदाजित कराल
- खर्च-समतोलासाठी योग्य SKU निवडाल
- निरीक्षणासाठी Application Insights कॉन्फिगर कराल
- टीम समन्वयाचे नमुने समजून घ्याल

## 🚀 त्वरीत प्रारंभ

```bash
# सदस्यता कोटा तपासा
az vm list-usage --location eastus --output table

# तैनातीचे पूर्वावलोकन (कोणतीही संसाधने तयार केली जात नाहीत)
azd provision --preview

# Bicep वाक्यरचना सत्यापित करा
az bicep build --file infra/main.bicep

# पर्यावरण कॉन्फिगरेशन तपासा
azd env get-values
```

## ☑️ पूर्व-तैनाती तपासणी यादी

### `azd provision` च्या आधी

- [ ] प्रदेशासाठी कोटा पडताळलेला आहे
- [ ] SKU योग्यरित्या निवडलेले आहेत
- [ ] खर्चाचा अंदाज तपासला गेला आहे
- [ ] नावकरणाचे नियम सुसंगत आहेत
- [ ] सुरक्षा/RBAC कॉन्फिगर केलेले आहे

### `azd deploy` च्या आधी

- [ ] पर्यावरण चल सेट केलेले आहेत
- [ ] Key Vault मध्ये गुप्त माहिती आहे
- [ ] कनेक्शन स्ट्रिंग्ज पडताळल्या गेल्या आहेत
- [ ] हेल्थ चेक्स कॉन्फिगर केलेले आहेत
