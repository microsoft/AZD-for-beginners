# अध्याय 6: प्री-डिप्लॉयमेंट नियोजन आणि पडताळणी

**📚 कोर्स**: [शिकणाऱ्यांसाठी AZD](../../README.md) | **⏱️ कालावधी**: 1 तास | **⭐ गुंतागुंत**: मध्यम

---

## आढावा

हा अध्याय तुमच्या अनुप्रयोगाची तैनाती करण्यापूर्वी आवश्यक नियोजन आणि पडताळणी पावले समाविष्ट करतो. योग्य क्षमता नियोजन, SKU निवड आणि प्रीफ्लाइट तपासण्या करून महागडा चुका टाळायला शिका.

> `azd 1.25.6` विरुद्ध जून 2026 मध्ये प्रमाणित.

## शिकण्याच्या उद्दिष्टे

हा अध्याय पूर्ण केल्याने, तुम्ही:
- तैनात करण्यापूर्वी प्रीफ्लाइट तपासण्या चालवण्यास सक्षम व्हाल
- क्षमता नियोजन आणि संसाधन गरजांचे अंदाज लावाल
- खर्च ऑप्टिमायझेशनसाठी योग्य SKU निवडाल
- मॉनिटरिंगसाठी Application Insights कॉन्फिगर कराल
- संघ समन्वय नमुन्य समजून घ्याल

## 🚀 जलद सुरुवात

```bash
# सदस्यता कोटा तपासा
az vm list-usage --location eastus --output table

# पूर्वावलोकन वितरण (कोणतेही संसाधने तयार नाहीत)
azd provision --preview

# Bicep व्याकरणाची पडताळणी करा
az bicep build --file infra/main.bicep

# पर्यावरण संरचना तपासा
azd env get-values
```

## ☑️ प्री-डिप्लॉयमेंट चेकलिस्ट

### `azd provision` करण्यापूर्वी

- [ ] क्षेत्रासाठी कोटा पडताळला
- [ ] SKU योग्यरित्या निवडले आहेत
- [ ] खर्चाचा अंदाज तपासला
- [ ] नावकरण पद्धत सुसंगत आहे
- [ ] सुरक्षा/RBAC कॉन्फिगर केले आहे

### `azd deploy` करण्यापूर्वी

- [ ] एन्व्हायर्नमेंट व्हेरिएबल सेट केले
- [ ] Key Vault मध्ये रहस्ये आहेत
- [ ] कनेक्शन स्ट्रिंग्ज पडताळल्या
- [ ] हेल्थ चेक कॉन्फिगर केले
