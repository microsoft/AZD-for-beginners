# Chapter 6: प्री-डिप्लॉयमेंट योजना आणि प्रमाणीकरण

**📚 Course**: [AZD नवशिक्यांसाठी](../../README.md) | **⏱️ Duration**: 1 तास | **⭐ Complexity**: मध्यम

## विहंगावलोकन

हा अध्याय तुमचे ऍप्लिकेशन डिप्लॉय करण्यापूर्वी आवश्यक नियोजन आणि प्रमाणीकरण टप्पे समाविष्ट करतो. योग्य क्षमता नियोजन, SKU निवड आणि प्री-फ्लाइट तपासण्या करून महागड्या चुका टाळायला शिका.

## शिकण्याची उद्दिष्टे

या अध्यायाची पूर्णत्वापर्यंत, आपण:
- डिप्लॉयमेंटपूर्वी प्री-फ्लाइट तपासण्या चालवा
- क्षमता नियोजन करा आणि संसाधनांची गरज अंदाज करा
- खर्च ऑप्टिमायझेशनसाठी योग्य SKUs निवडा
- मानिटरिंगसाठी Application Insights कॉन्फिगर करा
- टीम समन्वय नमुने समजून घ्या

## 🚀 जलद प्रारंभ

```bash
# सबस्क्रिप्शन कोटा तपासा
az vm list-usage --location eastus --output table

# परिनियोजनाचे पूर्वावलोकन (कोणतीही संसाधने तयार केली जात नाहीत)
azd provision --preview

# Bicep वाक्यरचना सत्यापित करा
az bicep build --file infra/main.bicep

# पर्यावरण कॉन्फिगरेशन तपासा
azd env get-values
```

## ☑️ प्री-डिप्लॉयमेंट चेकलिस्ट

### `azd provision` च्या आधी

- [ ] प्रदेशासाठी कोटा सत्यापित केला आहे
- [ ] SKUs योग्यरित्या निवडले गेले आहेत
- [ ] खर्चाचा अंदाज पुनरावलोकन केलेला आहे
- [ ] नावे देण्याची पद्धत सुसंगत आहे
- [ ] सुरक्षा/RBAC कॉन्फिगर केलेले आहे

### `azd deploy` च्या आधी

- [ ] Environment variables सेट केलेले आहेत
- [ ] Secrets Key Vault मध्ये आहेत
- [ ] कनेक्शन स्ट्रिंग्ज सत्यापित केल्या आहेत
- [ ] हेल्थ चेक कॉन्फिगर केलेले आहेत
