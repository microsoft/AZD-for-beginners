!!! tip "នៅចុងបញ្ចប់នៃមូឌុលនេះ អ្នកនឹងអាច"

    - [ ] **មហោស្រព 2:** ដាក់ចេញ និងផ្ទៀងផ្ទាត់ម៉ូដែល AI Agents

## 1. ការណែនាំ

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) ឬ `azd` គឺជាគ្រឿងចក្របញ្ជាទូរសopen-source ដែលធ្វើឱ្យកម្មវិធីអភិវឌ្ឍរលាយរលោងនៅពេលកសាង និងដាក់បញ្ចូលកម្មវិធីទៅ Azure។

[ម៉ូដែល AZD](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) គឺជាគំរូស្តុកដែលមានកូដកម្មវិធីគំរូ, ក្នុងទ្រព្យសម្បត្តិ _infrastructure-as-code_, និងឯកសារ `azd` សម្រាប់រចនាសម្ព័ន្ធដំណោះស្រាយដែលមានសុវត្ថិភាព។ ការផ្តល់សម្ភារៈគឺយ៉ាងងាយស្រួលតាមបញ្ជា `azd provision` - ខណៈពេលដែលប្រើ `azd up` អនុញ្ញាតឱ្យអ្នកផ្តល់សម្ភារៈ **និង** ដាក់បញ្ចូលកម្មវិធីរបស់អ្នកជាដំណាក់កាលតែមួយ!

ដូច្នេះ ដំណើរការអភិវឌ្ឍកម្មវិធីរបស់អ្នកអាចចាប់ផ្តើមបានយ៉ាងងាយស្រួល ដោយរកម៉ូដែល _AZD Starter_ ដែលសមស្របបំផុតចំពោះតម្រូវការកម្មវិធី និងសម្ភារៈរបស់អ្នក - បន្ទាប់មកប្តូរផ្ទុកតាមតម្រូវការរបស់អ្នក។

មុននឹងចាប់ផ្តើម មកធ្វើឱ្យប្រាកដថាអ្នកបានដំឡើង Azure Developer CLI រួច។

1. បើកផ្ទាំង VS Code terminal ហើយវាយបញ្ជា Command នេះ៖

      ```bash title="" linenums="0"
      azd version
      ```

1. អ្នកគួរតែឃើញអ្វីមួយដូចនេះ!

      ```bash title="" linenums="0"
      azd version 1.27.1 (commit <current-build>)
      ```

**ឥឡូវនេះអ្នករួចរាល់ក្នុងការជ្រើសរើស និងដាក់ចេញម៉ូដែល ជាមួយ azd**

## 2. ការជ្រើសរើសម៉ូដែល

វេទិកា Microsoft Foundry មាន [សំណុំម៉ូដែល AZD ដែលបានផ្តល់អនុសាសន៍](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) ដែលគ្របដណ្តប់លើស្ថានភាពដំណោះស្រាយពេញនិយមដូចជា _ការផ្លាស់ប្តូរដំណើរការជាមួយភ្នាក់ងារច្រើន_ និង _ការបញ្ចូនមាតិកាមួយចំនួនជាច្រើនជាមួយរបៀបផ្សេងៗ_។ អ្នកអាចរកឃើញម៉ូដែលទាំងនេះដោយចូលភ្នែកទៅកាន់វេទិកា Microsoft Foundry ផងដែរ។

1. ចូលទៅ [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. ចូលទៅកាន់វេទិកា Microsoft Foundry នៅពេលដែលបានស្នើ - អ្នកនឹងឃើញអ្វីមួយដូចនេះ។

![Pick](../../../../../translated_images/km/01-pick-template.60d2d5fff5ebc374.webp)


ជម្រើស **មូលដ្ឋាន** គឺជាម៉ូដែលចាប់ផ្តើមរបស់អ្នក៖

1. [ ] [ចាប់ផ្តើមជាមួយ AI Chat](https://github.com/Azure-Samples/get-started-with-ai-chat) ដែលដាក់ចេញកម្មវិធីសន្ទនាមូលដ្ឋាន _ជាមួយទិន្នន័យរបស់អ្នក_ ទៅ Azure Container Apps។ ប្រើប្រាស់នេះដើម្បីស្វែងយល់ពីស្ថានភាព chat bot AI មូលដ្ឋាន។
1. [X] [ចាប់ផ្តើមជាមួយ AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) ដែលក៏ដាក់ចេញ AI Agent ស្ដង់ដារ (ជាមួយ Foundry Agents)។ ប្រើប្រាស់នេះដើម្បីស្គាល់ថ្នាក់ដំណោះស្រាយឯកសារភ្នាក់ងារក្នុងបរិយាកាសដែលពាក់ព័ន្ធនឹងឧបករណ៍ និងគំរូ។

ចូលទៅកាន់តំណទីពីរនៅក្នុងផ្ទាំងប្រោស័រថ្មី (ឬចុច `Open in GitHub` សម្រាប់កាតដែលទាក់ទង)។ អ្នកគួរតែឃើញឃ្លាំងស្តុកសម្រាប់ម៉ូដែល AZD នេះ។ ចំណាយពេលមួយនាទីស្វែងយល់អំពី README។ រចនាសម្ព័ន្ធកម្មវិធីភាគតូចបែបនេះ៖

## 3. ការដំណើរការម៉ូដែល

ចង់សាកល្បងដាក់ចេញម៉ូដែលនេះ ហើយធ្វើឱ្យប្រាកដថាវាត្រឹមត្រូវ។ ធ្វើតាមណែនាំក្នុងផ្នែក [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started)។

1. ជ្រើសបរិយាកាសការងារសម្រាប់ឃ្លាំងម៉ូដែល៖

      - **GitHub Codespaces**: ចុច [តំណនេះ](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) ហើយបញ្ជាក់ `Create codespace`
      - **ក្លូនក្នុងម៉ាស៊ីនឬ dev container**: ក្លូន `Azure-Samples/get-started-with-ai-agents` ហើយបើកវានៅក្នុង VS Code

1. រងចាំរហូតដល់ VS Code terminal ពេញលេញ ហើយវាយបញ្ជារបស់ខាងក្រោម៖

   ```bash title="" linenums="0"
   azd up
   ```

បំពេញជំហានរក្សមុខងារដែលនឹងចាប់ផ្តើម៖

1. អ្នកនឹងត្រូវការ login ចូលទៅ Azure - ចូរតាមការណែនាំដើម្បីបញ្ជាក់ identity
1. បញ្ចូលឈ្មោះបរិយាកាសផ្ទាល់ខ្លួនសម្រាប់អ្នក - ឧទាហរណ៍ ខ្ញុំបានប្រើ `nitya-mshack-azd`
1. នេះនឹងបង្កើតថត `.azure/` - អ្នកនឹងឃើញថតរងមានឈ្មោះបរិយាកាសរបស់អ្នក
1. អ្នកនឹងត្រូវជ្រើសឈ្មោះសំបុត្រជាវសេវា - ជ្រើស ដើម
1. អ្នកនឹងត្រូវជ្រើសទីតាំង - ប្រើ `East US 2`

ឥឡូវនេះ ចាំរង់ចាំការបញ្ចប់ provisioning។ **វាត្រូវការពេល ១០-១៥នាទី**

1. ពេលរួចរាល់ ខ្ទង់បញ្ជារបស់អ្នកនឹងបង្ហាញសារ SUCCESS ដូចនេះ៖
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. ឥឡូវនេះ Azure Portal នឹងមានក្រុមធនធាន provisioned ជាមួយឈ្មោះបរិយាកាសនោះ៖

      ![Infra](../../../../../translated_images/km/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **ឥឡូវនេះអ្នករួចរាល់ក្នុងការផ្ទៀងផ្ទាត់រចនាសម្ព័ន្ធ និងកម្មវិធីដែលបានដាក់ចេញ**។

## 4. ការផ្ទៀងផ្ទាត់ម៉ូដែល

1. ចូលនាយកដ្ឋាន Azure Portal [Resource Groups](https://portal.azure.com/#browse/resourcegroups) - ចូលបញ្ជាក់នៅពេលដែលត្រូវការ
1. ចុចលើ RG សម្រាប់ឈ្មោះបរិយាកាសរបស់អ្នក - អ្នកនឹងឃើញទំព័រដូចខាងលើ

      - ចុចលើធនធាន Azure Container Apps
      - ចុចលើ Application Url ក្នុងផ្នែក _Essentials_ (កំពូលមុខខាងស្តាំ)

1. អ្នកគួរតែឃើញកម្មវិធីផ្ទាល់មុខ UI របស់ម៉ាស៊ីនបម្រើដូចខាងក្រោម៖

   ![App](../../../../../translated_images/km/03-test-application.471910da12c3038e.webp)

1. សាកល្បងសួរបញ្ហាត្រីមុខពីរ [ឧទាហរណ៍ប្រើក្នុងការសាកល្បង](https://github.com/Azure-Samples/get-started-with-ai-agents/blob/main/docs/sample_questions.md)

      1. សួរ: ```រាជធានីនៃប្រទេសបារាំងមានអ្វីខ្លះ?``` 
      1. សួរ: ```តង់ល្អបំផុតក្រោម $200 សម្រាប់មនុស្សពីរនាក់មានអ្វីខ្លះ ហើយវាមានលក្ខណៈពិសេសអ្វីខ្លះ?```

1. អ្នកគួរតែទទួលបានចម្លើយដូចខាងក្រោម។ _តែវាធ្វើការយ៉ាងដូចម្តេច?_

## 5. ការផ្ទៀងផ្ទាត់ភ្នាក់ងារ

Azure Container App បានដាក់ចេញអេនឌ្បូចដែលភ្ជាប់ទៅជាមួយ AI Agent ដែលបាន provision នៅក្នុង Microsoft Foundry សម្រាប់ម៉ូដែលនេះ។ មកមើលអ្វីដែលមានន័យ។

1. ត្រឡប់ទៅទំព័រ Azure Portal _Overview_ សម្រាប់ក្រុមធនធានរបស់អ្នក

1. ចុចលើធនធាន `Microsoft Foundry` នៅក្នុងបញ្ជីនោះ

1. អ្នកគួរតែឃើញនេះ។ ចុចប៊ូតុង `Go to Microsoft Foundry Portal`។
   ![Foundry](../../../../../translated_images/km/04-view-foundry-project.fb94ca41803f28f3.webp)

1. អ្នកគួរតែឃើញទំព័រ Foundry Project សម្រាប់កម្មវិធី AI របស់អ្នក
   ![Project](../../../../../translated_images/km/05-visit-foundry-portal.d734e98135892d7e.webp)

1. ចុចលើ `Agents` - អ្នកឃើញភ្នាក់ងារស្ដង់ដារ provisioned នៅក្នុងគម្រោងរបស់អ្នក
   ![Agents](../../../../../translated_images/km/06-visit-agents.bccb263f77b00a09.webp)

1. ជ្រើសវា - អ្នកឃើញព័ត៌មានលម្អិតភ្នាក់ងារ។ សូមចំណាំ៖

      - ភ្នាក់ងារប្រើ File Search ជាមូលដ្ឋាន (គ្មានបំរែបំរួល)
      - `Knowledge` របស់ភ្នាក់ងារបង្ហាញថាវា មានឯកសារ ៣២ ខ្នាតបានផ្ទុក (សម្រាប់ការស្វែងរកឯកសារ)
      ![Agents](../../../../../translated_images/km/07-view-agent-details.0e049f37f61eae62.webp)

## 8. អថេរបរិយាកាស

ដល់ពេលនេះ យើងបានដំណើរការដាក់ចេញក្នុងប្រោស័រ ហើយផ្ទៀងផ្ទាត់ថា រចនាសម្ព័ន្ធរបស់យើងបាន provision និងកម្មវិធីដំណើរការបាន។ ប៉ុន្តែក្នុងការធ្វើការជាមួយកូដកម្មវិធី _code-first_, យើងត្រូវកំណត់បរិយាកាស local ឲ្យមានអថេរដែលពាក់ព័ន្ធជាមួយសម្ភារៈទាំងនេះ។ ការប្រើ `azd` ធ្វើឲ្យរឿងនេះងាយស្រួល។

1. Azure Developer CLI [ប្រើអថេរបរិយាកាស](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) ដើម្បីរក្សា និងគ្រប់គ្រងការកំណត់រចនាសម្ព័ន្ធសម្រាប់ការដាក់ចេញកម្មវិធី។

1. អថេរបរិយាកាសត្រូវបានរក្សាទុកក្នុង `.azure/<env-name>/.env` - អ្នកចាត់ទុកវាសម្រាប់បរិយាកាស `env-name` ដែលបានប្រើដាក់ចេញ ហើយជួយអ្នកដាច់បរិយាកាសផ្សេងៗនៅក្នុងគំរូតែមួយ។

1. អថេរបរិយាកាសត្រូវបានផ្ទុកដោយស្វ័យប្រវត្តិដោយបញ្ជា `azd` នៅពេលវាចាប់ផ្តើមបញ្ជាពិសេសៗ (ឧ. `azd up`)។ សូមចំណាំថា `azd` មិនអានអថេរបរិយាកាសថ្នាក់ OS-level ដោយស្វ័យប្រវត្តិ (ឧ. កំណត់នៅក្នុង shell) - ជំនួសវាប្រើ `azd set env` និង `azd get env` ដើម្បីផ្ទេរព័ត៌មានក្នុងកូដ។


មកសាកល្បងបញ្ជាចំនួនមួយ៖

1. ទទួលអថេរបរិយាកាសទាំងអស់សម្រាប់ `azd` នៅក្នុងបរិយាកាសនេះ៖

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      អ្នកនឹងឃើញអ្វីមួយដូចជា៖

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. ទទួលបានតម្លៃជាក់លាក់ - ឧ. ខ្ញុំចង់ដឹងថាតើយើងបានកំណត់តម្លៃ `AZURE_AI_AGENT_MODEL_NAME` ទេឬទេ

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      អ្នកនឹងឃើញដូចខាងក្រោម - វាមិនត្រូវបានកំណត់តាមលំនាំដើម!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. កំណត់អថេរបរិយាកាសថ្មីសម្រាប់ `azd`។ នៅទីនេះ យើងធ្វើបច្ចុប្បន្នភាពឈ្មោះម៉ូដែលភ្នាក់ងារ។ _ចំណាំ៖ ការផ្លាស់ប្ដូរណាមួយនឹងត្រូវបានបញ្ចូលភ្លាមៗនៅក្នុងឯកសារ `.azure/<env-name>/.env`។_

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      ឥឡូវនេះ យើងត្រូវស្វែងរកតម្លៃដែលបានកំណត់៖

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. សូមចំណាំថា សមាសភាគខ្លះគឺមានភាពអចិន្រ្តៃយ៍ (ឧ. ការដាក់ម៉ូដែល) ហើយនឹងត្រូវការជាងតែ `azd up` ដើម្បីបង្ខំឲ្យដាក់ចេញឡើងវិញ។ មកសាកល្បងបំបែកការដាក់ចេញដើម ហើយដាក់ចេញឡើងវិញជាមួយអថេរបរិយាកាសដែលបានផ្លាស់ប្តូរ។

1. **សម្រួលថ្មី** ប្រសិនបើអ្នកបានដាក់ចេញរចនាសម្ព័ន្ធដោយប្រើម៉ូដែល azd មុននេះហើយ - អ្នកអាច _សម្រួលថ្មី_ ស្ថានភាពអថេរបរិយាកាសលើផ្ទាល់ដោយផ្អែកលើស្ថានភាពបច្ចុប្បន្ននៃការដាក់ចេញ Azure ដោយប្រើបញ្ជា៖

      ```bash title="" linenums="0"
      azd env refresh
      ```


      នេះគឺជាវិធីមហិច្ឆិតា​មួយសម្រាប់ _សមម៉ង_ អថេរបរិស្ថានឲ្យស្របគ្នាតាមរវាងបរិស្ថានអភិវឌ្ឍន៍មូលដ្ឋានពីរឬច្រើន (ឧ. ក្រុមដែលមានអ្នកអភិវឌ្ឍន៍ច្រើន) - អនុញ្ញាតឲ្យហេដ្ឋារចនាសម្ព័ន្ធដែលបានចែកចាយដើម្បីដឹងប្រភពពិតសម្រាប់ស្ថានភាពអថេរបរិស្ថាន។ សមាជិកក្រុមគ្រាន់តែ _បន្សុត_ អថេរដើម្បីបានស្របគ្នាសារឡើងវិញ។
