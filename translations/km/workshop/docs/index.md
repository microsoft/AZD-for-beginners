# សិក្ខាសាលា AZD សម្រាប់អ្នកអភិវឌ្ឍ AI

> បានផ្ទៀងផ្ទាត់ប្រឆាំងនឹង `azd 1.25.6` នៅខែ មិថុនា 2026។

> [!IMPORTANT]  
> **សិក្ខាសាលានេះត្រូវបានបំពាក់ដោយមេរៀនសិក្ខាសាលាដែលអ្នកអាចមើលមុននៅក្នុងកម្មវិធីរុករកក្នុងកុំព្យូទ័ររបស់អ្នក។ ដើម្បីចាប់ផ្ដើម សូមបើក repo ក្នុង GitHub Codespaces, a dev container, ឬ local clone ដែលបានដំឡើង MkDocs—បន្ទាប់មករង់ចាំរហូតដល់អ្នកឃើញ terminal VS Code ដែលកំពុងដំណើរការ ហើយវាយ៖**  
> `mkdocs serve > /dev/null 2>&1 &`  
> **អ្នកគួរតែឃើញប្រអប់លេចឡើងសម្រាប់បើកទំព័រមើលមុនក្នុងកម្មវិធីរុករក។**

សូមស្វាគមន៍មកកាន់សិក្ខាសាលាអនុវត្តសម្រាប់រៀន Azure Developer CLI (AZD) ដែលផ្តោតលើការដាក់ដំណើរការកម្មវិធី AI។ សិក្ខាសាលានេះជួយអ្នកទទួលបានការយល់ដឹងក្នុងការអនុវត្តពីគំរូ AZD តាម 3 ជំហាន៖

1. **Discovery** - ស្វែងរកគំរូដែលសាកសមសម្រាប់អ្នក។
1. **Deployment** - ដាក់ដំណើរការ និងផ្ទៀងផ្ទាត់ថាវាដំណើរការ
1. **Customization** - កែប្រែតាមតម្រូវការ និងធ្វើម្តងទៀតដើម្បីឲ្យវាជារបស់អ្នក!

ក្នុងដំណើរការសិក្ខាសាលានេះ អ្នកនឹងបានណែនាំអំពីឧបករណ៍ និងសំណុំការងារសម្រាប់អ្នកអភិវឌ្ឍ ដើម្បីជួយកាត់បន្ថយដំណើរការពេញលេញនៃការអភិវឌ្ឍរបស់អ្នក។

## ទិដ្ឋភាពទូទៅនៃសិក្ខាសាលា

**រយៈពេល:** 3-4 ម៉ោង  
**កំរិត:** ចាប់ផ្ដើមទៅមធ្យម  
**លក្ខខណ្ឌមុន:** ស្គាល់មូលដ្ឋានអំពី Azure, គំនិត AI, VS Code និងឧបករណ៍បន្ទាត់ពាក្យបញ្ជា។

## អ្វីដែលអ្នកនឹងរៀន

គិតពីគំរូ AZD ដូចជាតំបន់សិក្សាដើម្បីសាកល្បងសមត្ថភាព និងឧបករណ៍ផ្សេងៗសម្រាប់ការអភិវឌ្ឍពីចុងដល់ចុងលើ Microsoft Foundry។ នៅចុងបញ្ចប់នៃសិក្ខាសាលា អ្នកគួរតែមានការយល់ដឹងយ៉ាងខ្លី និងអនុវត្តបានពីឧបករណ៍ និងគំនិតផ្សេងៗក្នុងបរិបទនេះ។

| គ념ា  | គោលដៅ |
|:---|:---|
| **Azure Developer CLI** | យល់ពីពាក្យបញ្ជារ និងដំណើរការងារ |
| **AZD Templates**| យល់ពីរចនាសម្ព័ន្ធគម្រោង និងការកំណត់ |
| **Azure AI Agent**| រៀបចំ និងដាក់ដំណើរការគម្រោង Microsoft Foundry |
| **Azure AI Search**| អនុញ្ញាតការបង្កើតបរិបទ (context engineering) ជាមួយភ្នាក់ងារ |
| **Observability**| ស្វែងយល់អំពីការតាមដាន (tracing), ការត្រួតពិនិត្យ និងការវាយតម្លៃ |
| **Red Teaming**| ស្វែងយល់អំពីការធ្វើតេស្តប្រឆាំង និងវិធានការទប់ស្កប់ |
| | |
