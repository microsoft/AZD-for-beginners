**យោងសម្រាប់គ្រប់ជំពូក**

- **📖 សិក្សាមូលដ្ឋាន**: [ជំពូកទី 1: មូលដ្ឋាន AZD](../docs/getting-started/azd-basics.md)
- **🤖 ពាក្យដែលទាក់ទងនឹង AI**: [ជំពូកទី 2: ការអភិវឌ្ឍដោយផ្អែកលើ AI](../docs/microsoft-foundry/microsoft-foundry-integration.md)

## ការណែនាំ

បញ្ជីពាក្យទូលំទូលាយនេះផ្តល់នូវនិយមន័យសម្រាប់ពាក្យ គំនិត និងអក្រូនីម ដែលប្រើនៅក្នុង Azure Developer CLI និងការអភិវឌ្ឍនៅលើ Cloud របស់ Azure។ វាជាការយោងសំខាន់សម្រាប់យល់ដឹងឯកសារបច្ចេកទេស ដោះស្រាយបញ្ហា និងការទំនាក់ទំនងយ៉ាងត្រឹមត្រូវអំពីគម្រោង azd និងសេវាកម្ម Azure។

## គោលដៅការសិក្សា

ដោយប្រើបញ្ជីពាក្យនេះ អ្នកនឹង:
- យល់ពីពាក្យ និងគំនិតសំខាន់ៗរបស់ Azure Developer CLI
- ជំនាញវចនานุกรมអភិវឌ្ឍ Cloud របស់ Azure និងពាក្យបច្ចេកទេស
- យោងពាក្យទាក់ទង Infrastructure as Code និងការដាក់បញ្ចូលយ៉ាងមានប្រសិទ្ធិភាព
- យល់នាមសេវាកម្ម Azure អក្រុនីម និងគោលបំណងរបស់ពួកវា
- ចូលដំណើរការនិយមន័យសម្រាប់ការដោះស្រាយបញ្ហា និងការទាញយកព័ត៌មានកំហុស
- រៀនគោលការណ៍ស្ថាបត្យកម្ម Azure និងគំនិតអភិវឌ្ឍទាន់សម័យ

## លទ្ធផលការសិក្សា

ដោយយោងសារវត្តជាប្រចាំទៅកាន់បញ្ជីពាក្យនេះ អ្នកនឹងអាច:
- ទំនាក់ទំនងយ៉ាងមានប្រសិទ្ធភាពដោយប្រើពាក្យត្រឹមត្រូវរបស់ Azure Developer CLI
- យល់ឯកសារបច្ចេកទេស និងសារកំហុសបានច្បាស់លាស់ច្រើនទៀត
- ស្វែងរកសេវាកម្ម និងគំនិតរបស់ Azure ដោយមានទំនុកចិត្ត
- ដោះស្រាយបញ្ហាបានដោយប្រើវចនានុក្រមបច្ចេកទេសសមរម្យ
- រួមចំណែកក្នុងការពិភាក្សាក្រុមដោយភាសាបច្ចេកទេសត្រឹមត្រូវ
- ផ្ដល់ការពង្រឹងសមត្ថភាពជំនាញអភិវឌ្ឍ Cloud របស់ Azure ដោយមានវិធានការកម្ភៃ

## A

**ARM Template**  
Azure Resource Manager template. ទ្រង់ទ្រាយ Infrastructure as Code ដែលមានមូលដ្ឋានលើ JSON ដែលប្រើសម្រាប់កំណត់ និងដាក់បញ្ចូលធនធាន Azure ដោយប្រកាស។

**App Service**  
ផ្លាតហ្វូមជំនួយជាសេវាកម្ម (Platform-as-a-Service - PaaS) របស់ Azure សម្រាប់ភ្ជាប់ម៉ាស៊ីនបម្រើវេប អេភអាយភី REST និងផ្នែកខាងក្រោយសម្រាប់ម៉ាស៊ីនចល័ត ដោយមិនចាំបាច់គ្រប់គ្រងហេដ្ឋារចនាសម្ព័ន្ធ។

**Application Insights**  
សេវាកម្មត្រួតពិនិត្យប្រសិទ្ធភាពកម្មវិធី (APM) របស់ Azure ដែលផ្តល់ការយល់ដឹងជ្រាលជ្រៅអំពីការប្រតិបត្តិការកម្មវិធី ការកាន់តំណែង និងការប្រើប្រាស់។

**Azure CLI**  
ចំណុចបញ្ជារcommand-line សម្រាប់គ្រប់គ្រងធនធាន Azure។ ត្រូវបានប្រើដោយ azd សម្រាប់ការផ្ទៀងផ្ទាត់សម្គាល់ និងប្រតិបត្តិការខ្លះៗ។

**Azure Developer CLI (azd)**  
ឧបករណ៍បន្ទាត់ពាក្យសម្រាប់អ្នកអភិវឌ្ឍដែលផ្តល់ល្បឿនក្នុងដំណើរការកសាង និងដាក់បញ្ចូលកម្មវិធីទៅ Azure ដោយប្រើតំបន់រចនាសម្ព័ន្ធ និង Infrastructure as Code។

**azure.yaml**  
ឯកសារកំណត់រចនាសម្ព័ន្ធចម្បងសម្រាប់គំរោង azd ដែលកំណត់សេវាកម្ម ហេដ្ឋារចនាសម្ព័ន្ធ និងច្រកដាក់បញ្ចូល។

**Azure Resource Manager (ARM)**  
សេវាកម្មដាក់បញ្ចូល និងគ្រប់គ្រងរបស់ Azure ដែលផ្តល់ស្រទាប់គ្រប់គ្រងសម្រាប់បង្កើត បច្ចុប្បន្នភាព និងលុបធនធាន។

## B

**Bicep**  
ភាសាតិសម័យជាក់លាក់ (DSL) ដែល Microsoft បានអភិវឌ្ឍសម្រាប់ដាក់បញ្ចូលធនធាន Azure។ ផ្តល់សំឡីសាងសម្រួលជាង ARM templates ខណៈដែលកំពុងបកចិត្តទៅ ARM។

**Build**  
ដំណើរការបង្កើតកូដប្រភព កំណត់តម្លៃពឹងផ្អែក និងរៀបចំកម្មវិធីសម្រាប់ដាក់បញ្ចូល។

**Blue-Green Deployment**  
យុទ្ធសាស្ត្រដាក់បញ្ចូលដែលប្រើបរិយាកាសផលិតដែលដូចគ្នាពីរជាប្រភេទ (blue និង green) ដើម្បីកាត់បន្ថយការផ្ទេរសេវា និងហានិភ័យ។

## C

**Container Apps**  
សេវាកម្ម containerless របស់ Azure ដែលអនុញ្ញាតឲ្យរត់កម្មវិធីដែលមាន container ដោយមិនចាំបាច់គ្រប់គ្រងហេដ្ឋារចនាសម្ព័ន្ធស្មុគស្មាញ។

**CI/CD**  
Continuous Integration/Continuous Deployment។ វិធីសាស្ត្រដែលស្គាល់ដោយស្វ័យប្រវត្តិសម្រាប់លាយកូដ និងដាក់បញ្ចូលកម្មវិធី។

**Cosmos DB**  
សេវាកម្មទិន្នន័យមូលដ្ឋានម៉ូដែលច្រើនដែលផ្ទុះពិភពលោករបស់ Azure ដែលផ្តល់ SLA លម្អិតសម្រាប់ throughput, latency, availability និង consistency។

**Configuration**  
ការកំណត់ និងប៉ារ៉ាម៉ែត្រដែលគ្រប់គ្រងអាកប្បកិរិយារបស់កម្មវិធី និងជម្រើសការដាក់បញ្ចូល។

## D

**Deployment**  
ដំណើរការដំឡើង និងកំណត់រចនាសម្ព័ន្ធកម្មវិធី និងពឹងផ្អែករបស់ខ្លួនលើហេដ្ឋារចនាសម្ព័ន្ធគោលដៅ។

**Docker**  
ផ្លាតអេនសម្រាប់អភិវឌ្ឍ ផ្ញើ និងរត់កម្មវិធីដោយប្រើបច្ចេកវិទ្យារាវរុក containerization។

**Dockerfile**  
ឯកសារ​អក្សរដែលមានសេចក្ដីណែនាំសម្រាប់បង្កើតរូបភាព container Docker។

## E

**Environment**  
គោលដៅដាក់បញ្ចូលដែលតំណាងឱ្យកំណែជាក់លាក់នៃកម្មវិធីរបស់អ្នក (ឧ. ការអភិវឌ្ឍ, ស្ទាច, ផលិតកម្ម)។

**Environment Variables**  
តម្លៃកំណត់ដែលរក្សាទុកជាគន្លងគន្លង key-value ដែលកម្មវិធីអាចចូលប្រើនៅពេលរត់។

**Endpoint**  
URL ឬអាសយដ្ឋានបណ្ដាញដែលកម្មវិធី ឬសេវាកម្មអាចចូលដំណើរការ។

## F

**Function App**  
សេវាកម្មគណនា serverless របស់ Azure ដែលអនុញ្ញាតឲ្យរត់កូដដោយផ្អែកលើព្រឹត្តិការណ៍ ដោយមិនចាំបាច់គ្រប់គ្រងហេដ្ឋារចនាសម្ព័ន្ធ។

## G

**GitHub Actions**  
វេទិក CI/CD ដែលភ្ជាប់ជាមួយស្តុក GitHub សម្រាប់អូតូម៉ាទិកនៃសកម្មភាពបុព្វកម្ម។

**Git**  
ប្រព័ន្ធគ្រប់គ្រងកំណែចែកចាយដែលប្រើសម្រាប់តាមដានការផ្លាស់ប្តូរក្នុងកូដប្រភព។

## H

**Hooks**  
ស្គ្រីប ឬពាក្យបញ្ជាផ្ទាល់ខ្លួនដែលរត់នៅចំណុចជាក់លាក់ក្នុងរង្វិលជីវិតនៃការដាក់បញ្ចូល (preprovision, postprovision, predeploy, postdeploy)។

**Host**  
ប្រភេទសេវាកម្ម Azure ដែលកម្មវិធីនឹងត្រូវបានដាក់បញ្ចូល (ឧ. appservice, containerapp, function)។

## I

**Infrastructure as Code (IaC)**  
អនុស្សរណៈនៃការកំណត់ និងគ្រប់គ្រងហេដ្ឋារចនាសម្ព័ន្ធតាមរយៈកូដ ជាជំនួសដូចពីដៃ។

**Init**  
ដំណើរការផ្តើមគម្រោង azd ថ្មី ជាច្រើនដដែលពីទំព័រគំរូ។

## J

**JSON**  
JavaScript Object Notation។ ទ្រង់ទ្រាយផ្ទេរទិន្នន័យដែលយើងតែងប្រើសម្រាប់ឯកសារកំណត់ និងការឆ្លើយតប API។

**JWT**  
JSON Web Token។ ស្តង់ដារសម្រាប់ផ្ទេរព័ត៌មានយ៉ាងសុវត្ថិភាពរវាងភាគីជាអ объект JSON។

## K

**Key Vault**  
សេវាកម្មរបស់ Azure សម្រាប់រក្សា និងគ្រប់គ្រងសម្ងាត់ កូនសោ និងវិញ្ញាបនបត្រយ៉ាងសុវត្ថិភាព។

**Kusto Query Language (KQL)**  
ភាសាកំណត់សម្រាប់ស្វែងយល់ទិន្នន័យនៅក្នុង Azure Monitor, Application Insights និងសេវាកម្ម Azure ផ្សេងទៀត។

## L

**Load Balancer**  
សេវាកម្មដែលចែកចាយចរាចរណ៍បណ្ដាញចូលរវាងម៉ាស៊ីនបម្រើឬអាំងស្តង់ខ្លះៗ។

**Log Analytics**  
សេវាកម្ម Azure សម្រាប់ប្រមូល អ្នកវិភាគ និងសកម្មភាពលើទិន្នន័យ telemetry ពីបរិយាកាស cloud និងអនឡាញក្នុងទីកន្លែង។

## M

**Managed Identity**  
មុខងារ Azure ដែលផ្តល់អត្តសញ្ញាណដែលគ្រប់គ្រងដោយស្វ័យប្រវត្តិសម្រាប់សេវាកម្ម Azure ដើម្បីផ្ទៀងផ្ទាត់ទៅសេវាកម្ម Azure ផ្សេងទៀត។

**Microservices**  
វិធីសាស្ត្រស្ថាបត្យកម្មដែលកម្មវិធីត្រូវបានកសាងជាកម្មវិធីតូចៗដាច់ពីគ្នាច្រើនផ្នែក។

**Monitor**  
ដំណោះស្រាយមួយរួមរបស់ Azure សម្រាប់ផ្តល់ទស្សនៈពេញលេញលើកម្មវិធី និងហេដ្ឋារចនាសម្ព័ន្ធ។

## N

**Node.js**  
runtime JavaScript ដែលបង្កើតលើម៉ូទ័ររ V8 របស់ Chrome សម្រាប់កសាងកម្មវិធីផ្នែកខាងម៉ាស៊ីនបម្រើ។

**npm**  
គ្រប់គ្រងកញ្ចប់សម្រាប់ Node.js ដែលគ្រប់គ្រងពឹងផ្អែក និងកញ្ចប់។

## O

**Output**  
តម្លៃដែលត្រូវបានផ្ទេរចេញពីការដាក់បញ្ចូលហេដ្ឋារចនាសម្ព័ន្ធដែលអាចប្រើដោយកម្មវិធីឬធនធានផ្សេងទៀត។

## P

**Package**  
ដំណើរការរៀបចំពាក្យកូដកម្មវិធី និងពឹងផ្អែកសម្រាប់ដាក់បញ្ចូល។

**Parameters**  
តម្លៃបញ្ចូលដែលផ្ញើទៅឯកសារគំរូហេដ្ឋារចនាសម្ព័ន្ធដើម្បីប្ដូរការដាក់បញ្ចូល។

**PostgreSQL**  
ប្រព័ន្ធមូលដ្ឋានទិន្នន័យទំនាក់ទំនងបើកមុខដែលគាំទ្រដោយជាសេវាកម្មដែលគ្រប់គ្រងនៅលើ Azure។

**Provisioning**  
ដំណើរការបង្កើត និងកំណត់រចនាសម្ព័ន្ធធនធាន Azure ដែលបានកំណត់ក្នុងគំរូហេដ្ឋារចនាសម្ព័ន្ធ។

## Q

**Quota**  
ដែនកំណត់លើចំនួនធនធានដែលអាចបង្កើតនៅក្នុង subscription ឬតំបន់មួយនៅលើ Azure។

## R

**Resource Group**  
ធុងតំណាងចិត្តសម្រាប់ធនធាន Azure ដែលមានរយៈពេលជីវិត សិទ្ធិ និងគោលនយោបាយដូចគ្នា។

**Resource Token**  
សម្រង់ខ្សែអក្ខរក្នុងឯកទេសដែល azd បង្កើតឡើងដើម្បីធានាថាឈ្មោះធនធានមានភាពដាច់ដោយឡែកនៅលើការដាក់បញ្ចូល។

**REST API**  
រចនាបទស្ថាបត្យកម្មសម្រាប់រចនាកម្មវិធីបណ្តាញដោយប្រើវិធីសាស្ត្រ HTTP។

**Rollback**  
ដំណើរការបង្វិលទៅកំណែចាស់នៃកម្មវិធីឬកំណត់រចនាសម្ព័ន្ធហេដ្ឋារចនាសម្ព័ន្ធ។

## S

**Service**  
ធាតុផ្សំមួយនៃកម្មវិធីរបស់អ្នកដែលបានកំណត់នៅក្នុង azure.yaml (ឧ. ផ្នែកមុខ web, ផ្នែកក្រោយ API, មូលដ្ឋានទិន្នន័យ)។

**SKU**  
Stock Keeping Unit. តំណាងឱ្យកម្រិតសេវាកម្ម ឬកម្រិតប្រសិទ្ធភាពផ្សេងៗសម្រាប់ធនធាន Azure។

**SQL Database**  
សេវាកម្មមូលដ្ឋានទិន្នន័យទំនាក់ទំនងដែលគ្រប់គ្រងដោយ Azure ស្ថិតលើ Microsoft SQL Server។

**Static Web Apps**  
សេវាកម្ម Azure សម្រាប់កសាង និងដាក់បញ្ចូលកម្មវិធីវេបពេញលេញពីឃ្លាំងកូដប្រភព។

**Storage Account**  
សេវាកម្ម Azure ដែលផ្តល់ផ្ទុកទិន្នន័យ cloud សម្រាប់វត្ថុទិន្នន័យរួមមាន blobs, files, queues និង tables។

**Subscription**  
ធុងគណនី Azure ដែលផ្ទុក resource groups និងធនធាន ដោយមានការគិតលុយ និងការគ្រប់គ្រងចូលប្រើជាប់ទាក់ទង។

## T

**Template**  
រចនាសម្ព័ន្ធគម្រោងដែលបានបង្កើតជាមុន ដែលមានកូដកម្មវិធី កំណត់រចនាសម្ព័ន្ធហេដ្ឋារចនាសម្ព័ន្ធ និងដាក់បញ្ចូលសម្រាប់សេណារីយូមទូទៅ។

**Terraform**  
ឧបករណ៍ Infrastructure as Code បើកប្រភព ដែលគាំទ្រផ្លែផ្គត់ផ្គង់ cloud ច្រើន រួមមាន Azure។

**Traffic Manager**  
ម៉ាស៊ីនបំបែកចរាចរណ៍ដោយផ្អែកលើ DNS របស់ Azure សម្រាប់ចែកចាយចរាចរណ៍នៅតាមតំបន់របស់ Azure នៅលើពិភពលោក។

## U

**URI**  
Uniform Resource Identifier។ ខ្សែសរសេរដែលកំណត់ធនធានមួយជាក់លាក់។

**URL**  
Uniform Resource Locator។ ប្រភេទ URI ដែលបញ្ជាក់ទីតាំងឈ្មោះធនធាន និងរបៀបទាញយកវា។

## V

**Virtual Network (VNet)**  
ប្លុកសាងសង់មូលដ្ឋានសម្រាប់បណ្តាញឯកជននៅលើ Azure ដែលផ្តល់ការបែកផ្នែក និងការផ្តាច់ចេញ។

**VS Code**  
Visual Studio Code។ អ្នកកែសម្រួលកូដដែលពេញនិយម ជាមួយឧបករណ៍ល្អសម្រាប់ Azure និង azd។

## W

**Webhook**  
ការហៅត្រឡប់ HTTP ដែលចាប់ផ្តើមដោយព្រឹត្តិការណ៍ជាក់លាក់ ដែលភាគច្រើនប្រើក្នុងបណ្តាញ CI/CD។

**What-if**  
មុខងាររបស់ Azure ដែលបង្ហាញថាមាត្រាអ្វីខ្លះនឹងត្រូវបានផ្លាស់ប្តូរដោយការដាក់បញ្ចូល ដោយមិនគិតអនុវត្តពិតប្រាកដ។

## Y

**YAML**  
YAML Ain't Markup Language។ ស្តង់ដារ serialization ទិន្នន័យដែលអានបានដោយមនុស្ស ដែលប្រើសម្រាប់ឯកសារកំណត់ដូចជា azure.yaml។

## Z

**Zone**  
ទីតាំងដែលបំបែកជាកន្លែងផ្អែកដោយផាសុកភាពក្នុងតំបន់ Azure ដែលផ្តល់ភាពធន់និងភាពមាននៅដល់ខ្ពស់ក្នុងជួរ availability។

## ពាក្យពាក់ព័ន្ធតាមបរិបទ

### ពាក្យទាក់ទងនឹងការអភិវឌ្ឍ
- **Hot Reload**: ការអាប់ដេតកម្មវិធីដោយស្វ័យប្រវត្តិនៅក្នុងដំណើរការអភិវឌ្ឍដោយមិនចាំបាច់ចាប់ផ្តើមឡើងវិញ
- **Build Pipeline**: ដំណើរការអូតូម៉ាទិកសម្រាប់កសាង និងសាកល្បងកូដ
- **Deployment Slot**: បរិយាកាសស្តាចដែលនៅក្នុង App Service
- **Environment Parity**: ការថែរក្សាបរិយាកាសអភិវឌ្ឍ ស្ទាច និងផលិតកម្មឲ្យស្រដៀងគ្នា

### ពាក្យសុវត្ថិភាព
- **Managed Identity**: មុខងារ Azure ដែលផ្តល់ការគ្រប់គ្រងសម្គាល់ដោយស្វ័យប្រវត្តិ
- **Key Vault**: កន្លែងផ្ទុកសម្ងាត់ សម្រាប់សម្ងាត់ កូនសោ និងវិញ្ញាបនបត្រយ៉ាងមានសុវត្ថិភាព
- **RBAC**: ការគ្រប់គ្រងការចូលដោយផ្អែកលើតួនាទីសម្រាប់ធនធាន Azure
- **Network Security Group**: ភ្លៅភាមួយសម្រាប់ត្រួតពិនិត្យចរាចរណ៍បណ្ដាញ

### ពាក្យត្រួតពិនិត្យ
- **Telemetry**: ការប្រមូលទិន្នន័យ និងវាស់វែងដោយស្វ័យប្រវត្តិ
- **Application Performance Monitoring (APM)**: ការតាមដានប្រសិទ្ធភាពនៃកម្មវិធី
- **Log Analytics**: សេវាកម្មសម្រាប់ប្រមូល និងវិភាគទិន្នន័យកំណត់ហេតុ
- **Alert Rules**: ការជូនដំណឹងដោយស្វ័យប្រវត្តិដែលអាស្រ័យលើម៉េត្រិច ឬលក្ខខណ្ឌ

### ពាក្យទាក់ទងនឹងការដាក់បញ្ចូល
- **Blue-Green Deployment**: យុទ្ធសាស្ត្រដាក់បញ្ចូលដោយគ្មានការកាត់បន្ថយសេវា
- **Canary Deployment**: ការបញ្ចេញយ៉ាងរាបស្មើទៅកាន់ក្រុមអ្នកប្រើប្រាស់តិចៗជាជំហ៊ាន
- **Rolling Update**: ការជំនួសអាំងស្តង់កម្មវិធីជាលំដាប់
- **Rollback**: ការបង្វិលទៅកំណែចាស់នៃកម្មវិធី

---

**Usage Tip**: ប្រើ `Ctrl+F` ដើម្បីស្វែងរកពាក្យជាក់លាក់ទ بسرعة ក្នុងបញ្ជីពាក្យនេះ។ ពាក្យត្រូវបានយោងឆ្លើយគ្នាក្នុងកន្លែងដែលអាចប្រើបាន។
