- **📖 សិក្សាប៊ាសិក**: [ជំពូក 1: មូលដ្ឋាន AZD](../docs/getting-started/azd-basics.md)
- **🤖 ពាក្យពាក់ព័ន្ធ AI**: [ជំពូក 2: ការអភិវឌ្ឍដែលផ្ដល់អាទិភាព AI](../docs/microsoft-foundry/microsoft-foundry-integration.md)

## ការណែនាំ

វចនានុក្រមទាំងមូលនេះផ្តល់និយមន័យសម្រាប់ពាក្យ គំនិត និងអក្រុងីមដែលប្រើក្នុង Azure Developer CLI និងការអភិវឌ្ឍពពក Azure។ វាជាយោងសំខាន់សម្រាប់យល់ពីឯកសារបច្ចេកទេស ដំណោះស្រាយបញ្ហា និងការប្រាស្រ័យទាក់ទងយ៉ាងមានប្រសិទ្ធភាពអំពីគម្រោង azd និងសេវាកម្ម Azure។

## គោលដៅសិក្សា

ដោយប្រើវចនានុក្រមនេះ អ្នកនឹងអាច:
- យល់ពាក្យ និងគំនិតសំខាន់ៗរបស់ Azure Developer CLI
- ជំនាញពាក្យសព្ទផ្នែកអភិវឌ្ឍន៍ពពក Azure និងពាក្យបច្ចេកទេស
- យោងពាក្យដែលពាក់ព័ន្ធ Infrastructure as Code និងការដាក់បញ្ចូល (deployment) ដោយមានប្រសិទ្ធភាព
- យល់ឈ្មោះសេវាកម្ម Azure អក្រុងីម និងគោលបំណងរបស់ពួកវា
- ចូលរួមនឹងនិយមន័យសម្រាប់ការដោះស្រាយបញ្ហា និងការដូសតេស្ត
- រៀនគំនិតស្ថាបត្យកម្ម និងការអភិវឌ្ឍអភិវឌ្ឍន៍ Azure ជម្រៅ

## លទ្ធផលសិក្សា

ដោយយោងជាប្រចាំទៅកាន់វចនានុក្រមនេះ អ្នកនឹងអាច:
- ប្រាស្រ័យទាក់ទងយ៉ាងមានប្រសិទ្ធភាពដោយប្រើពាក្យសព្ទត្រឹមត្រូវរបស់ Azure Developer CLI
- យល់ឯកសារបច្ចេកទេស និងសារ​បញ្ឆាបូមកាន់តែងាយស្រួល
- ឆែកសេវាកម្ម Azure និងគំនិតដោយមានទំនុកចិត្ត
- ដោះស្រាយបញ្ហាដោយប្រើពាក្យបច្ចេកទេសសមរម្យ
- ឈ្នះចូលរួមនៅក្នុងការពិភាក្សាក្រុមដោយភាសាបច្ចេកទេសយ៉ាងត្រឹមត្រូវ
- ពង្រីកចំណេះដឹងអភិវឌ្ឍន៍ពពក Azure របស់អ្នកយ៉ាងមានរបៀប

## A

**ARM Template**  
Azure Resource Manager template. ប្រភារ JSON សម្រាប់ Infrastructure as Code ដែលប្រើដើម្បីកំណត់ និងដាក់បញ្ចូលធនធាន Azure ដោយទ្រង់ទ្រាយពណ៌នា។

**App Service**  
កម្ពុជាជា platform-as-a-service (PaaS) របស់ Azure សម្រាប់ផ្ដល់ដំណឹងអង់ច័រ កម្មវិធីគេហទំព័រ REST APIs និង backend សម្រាប់ទូរស័ព្ទដោយមិនចាំបាច់គ្រប់គ្រងផ្នែកហาร์ដ្វែរនៅក្រោម។

**Application Insights**  
សេវាកម្មមើលសមិទ្ធផលកម្មវិធី (APM) របស់ Azure ដែលផ្តល់ទស្សនៈជ្រៅអំពីប្រសិទ្ធភាព ការចូលប្រើ និងការប្រើប្រាស់កម្មវិធី។

**Azure CLI**  
ចំណុចបញ្ជានៅលើបន្ទាត់សម្រាប់គ្រប់គ្រងធនធាន Azure។ ត្រូវបានប្រើដោយ azd សម្រាប់ការ Authenticate និងប្រតិបត្តិការខ្លះៗ។

**Azure Developer CLI (azd)**  
ឧបករណ៍បញ្ជាប្រុងសម្រាប់អ្នកអភិវឌ្ឍដែលជួយឲ្យដំណើរការការសង់ និងដាក់អនុវត្តកម្មវិធីទៅ Azure លឿនជាងមុន ដោយប្រើទំព័រខ្នង និង Infrastructure as Code។

**azure.yaml**  
ឯកសារកំណត់រចនាសម្ព័ន្ធសំខាន់សម្រាប់គម្រោង azd ដែលកំណត់សេវាកម្ម ស្ថាបត្យកម្ម និងហូកដាក់បញ្ចូល។

**Azure Resource Manager (ARM)**  
សេវាកម្មដាក់បញ្ចូល និងគ្រប់គ្រងរបស់ Azure ដែលផ្តល់ស្រទាប់គ្រប់គ្រងសម្រាប់បង្កើត បច្ចុប្បន្នភាព និងលុបធនធាន។

## B

**Bicep**  
ភាសាជាពិសេសសម្រាប់ដែនកំណត់ដែលបានបង្កើតដោយ Microsoft សម្រាប់ដាក់បញ្ចូលធនធាន Azure។ ផ្តល់ប្រាប្រដៅសន្មត់សម្រាប់សង្ខេបជាង ARM templates ខណៈដែលធ្វើបាន compile ទៅជា ARM។

**Build**  
ដំណើរការបង្កើតកូដប្រភព ការដំឡើងការពឹងផ្អែក និងរៀបចំកម្មវិធីសម្រាប់ដាក់បញ្ចូល។

**Blue-Green Deployment**  
យុទ្ធសាស្រ្តដាក់បញ្ចូលដែលប្រើបរិយាកាសផលិតកម្មពីរដូចគ្នា (ខៀវ និង បៃតង) ដើម្បីបន្ថយពេលពន្យារនិងហានិភ័យ។

## C

**Container Apps**  
សេវាកម្មកុងតឺន័រមិនមានម៉ាស៊ីនបម្រើ (serverless) របស់ Azure ដែលអនុញ្ញាតឲ្យដំណើរការ កម្មវិធីកុងតឺន័រដោយគ្មានការគ្រប់គ្រងផ្នែកហេដ្ឋារចនាសម្ព័ន្ធស្មុះស្មាញ។

**CI/CD**  
Continuous Integration/Continuous Deployment។ ប្រព័ន្ធស្វ័យប្រវត្តសម្រាប់រួមបញ្ចូលការផ្លាស់ប្តូរកូដ និងដាក់បញ្ចូលកម្មវិធី។

**Cosmos DB**  
សេវាកម្មមូលដ្ឋានទិន្នន័យចំរូងពហុម៉ូഡែលដែលចែកចាយជាសកលរបស់ Azure ដែលផ្តល់ SLA ស្រាប់សម្រាប់ throughput, latency, availability និង consistency។

**Configuration**  
ការកំណត់ និងប៉ារ៉ាម៉ែត្រដែលគ្រប់គ្រងអាកប្បកម្មកម្មវិធី និងជម្រើសការដាក់បញ្ចូល។

## D

**Deployment**  
ដំណើរការដំឡើង និងកំណត់រចនាសម្ព័ន្ធកម្មវិធី និងការពឹងផ្អែកលើហេដ្ឋារចនាសម្ព័ន្ធគោលដៅ។

**Docker**  
វេទិកាសម្រាប់អភិវឌ្ឍ ផ្ញើ និងដំណើរការកម្មវិធីដោយប្រើបច្ចេកវិទ្យាកុងតឺន័រមាន។

**Dockerfile**  
ឯកសារអក្សរមានការ៉េបញ្ជាដែលមានណែនាំសម្រាប់សាងសង់រូបភាព Docker container។

## E

**Environment**  
គោលដៅដាក់បញ្ចូលដែលផ្តល់ន័យចំពោះអត្ថន័យជាក់លាក់នៃកម្មវិធីរបស់អ្នក (ឧ. ការអភិវឌ្ឍ, staging, ផលិតកម្ម)។

**Environment Variables**  
តម្លៃកំណត់ដែលរក្សាទុកជា key-value ដែលកម្មវិធីអាចចូលប្រើពេលរត់។

**Endpoint**  
URL ឬអាសយដ្ឋានបណ្ដាញដែលអាចចូលប្រើកម្មវិធី ឬសេវាកម្មបាន។

## F

**Function App**  
សេវាកម្មគណិតវិទ្យាមិនមានម៉ាស៊ីនបម្រើរបស់ Azure ដែលអនុញ្ញាតឲ្យដំណើរការកូដព្រឹត្តិការណ៍ដោយមិនចាំបាច់គ្រប់គ្រងហេដ្ឋារចនាសម្ព័ន្ធ។

## G

**GitHub Actions**  
វេទិកា CI/CD ដែលបញ្ចូលជាមួយឃ្លីប GitHub សម្រាប់ស្វ័យករណ៍ដំណើរការ(workflows)។

**Git**  
ប្រព័ន្ធគ្រប់គ្រងកំណែចែកចាយដែលប្រើសម្រាប់តាមដានការផ្លាស់ប្តូរក្នុងកូដប្រភព។

## H

**Hooks**  
ស្គ្រីប ឬបញ្ជា ប្តូរតាមផ្ទាល់ដែលដំណើរការនៅចំណុចជាក់លាក់ក្នុងរយៈពេលជីវិតការដាក់បញ្ចូល (preprovision, postprovision, predeploy, postdeploy)។

**Host**  
ប្រភេទសេវាកម្ម Azure ដែលកម្មវិធីនឹងត្រូវបានដាក់ (ឧ. appservice, containerapp, function)។

## I

**Infrastructure as Code (IaC)**  
ការអនុវត្តន៍នៃកំណត់ និងគ្រប់គ្រងហេដ្ឋារចនាសម្ព័ន្ធតាមរយៈកូដ ជាជំនួសដល់ដំណើរការដោយដៃ។

**Init**  
ដំណើរការចាប់ផ្តើមគម្រោង azd ថ្មី ម្នាក់ៗជាធម្មតាមកពីទំព័រគំរូ។

## J

**JSON**  
JavaScript Object Notation។ ទ្រង់ទ្រាយប្ដូរទិន្នន័យដែលត្រូវបានប្រើញឹកញាប់សម្រាប់ឯកសារកំណត់រចនាសម្ព័ន្ធ និងចម្លើយ API។

**JWT**  
JSON Web Token។ ស្តង់ដាសម្រាប់ផ្ទេរព័ត៌មានយ៉ាងសុវត្ថិជារូបភាព JSON រវាងភាគី។

## K

**Key Vault**  
សេវាកម្មរបស់ Azure សម្រាប់រក្សា និងគ្រប់គ្រងសម្ងាត់ រក្សាទុកគន្លឹះ និងវិញ្ញាបនបត្រ ដោយមានសុវត្ថិភាព។

**Kusto Query Language (KQL)**  
ភាសាច្បាប់សម្រាប់វិភាគទិន្នន័យនៅក្នុង Azure Monitor, Application Insights និងសេវាកម្ម Azure ផ្សេងទៀត។

## L

**Load Balancer**  
សេវាកម្មដែលចែកចាយចរាចរណ៍បណ្ដាញចូលទៅកាន់ម៉ាស៊ីនបម្រើ ឬ instance ច្រើន។

**Log Analytics**  
សេវាកម្ម Azure សម្រាប់ប្រមូល វិភាគ និងសកម្មសម្រេចលើទិន្នន័យ telemetry ពីបរិយាកាស cloud និង on-premises។

## M

**Managed Identity**  
មុខងាររបស់ Azure ដែលផ្តល់អត្តសញ្ញាណដែលគ្រប់គ្រងដោយស្វ័យប្រវត្តិសម្រាប់សេវាកម្ម Azure ដើម្បី authenticate ទៅសេវាកម្ម Azure ផ្សេងទៀត។

**Microservices**  
វិធីសាស្រ្តស្ថាបត្យកម្មដែលកម្មវិធីត្រូវបានកសាងជាចំណុចសេវាកម្មតូចៗ និងឯករាជ្យ។

**Monitor**  
ដោះស៊ីសង្ស័យណាស់របស់ Azure សម្រាប់ផ្តល់ការចាប់សន្ទស្សន៍ពេញលំហូរ (full-stack observability) ទូទាំងកម្មវិធី និងហេដ្ឋារចនាសម្ព័ន្ធ។

## N

**Node.js**  
 runtime JavaScript ដែលបង្កើតលើម៉ាស៊ីន V8 របស់ Chrome សម្រាប់កសាងកម្មវិធីផ្នែកម៉ាស៊ីនមេ។

**npm**  
កម្មវិធីគ្រប់គ្រងកញ្ចប់សម្រាប់ Node.js ដែលគ្រប់គ្រងការពឹងផ្អែក និងកញ្ចប់ផ្សេងៗ។

## O

**Output**  
តម្លៃដែលត្រូវបានត្រឡប់ពីការដាក់បញ្ចូលហេដ្ឋារចនាសម្ព័ន្ធ ដែលអាចប្រើដោយកម្មវិធី ឬធនធានផ្សេងទៀត។

## P

**Package**  
ដំណើរការរៀបចំកូដកម្មវិធី និងការពឹងផ្អែកសម្រាប់ដាក់បញ្ចូល។

**Parameters**  
តម្លៃបញ្ចូលដែលផ្តល់ទៅកំណត់រចនាសម្ព័ន្ធហេដ្ឋារចនាសម្ព័ន្ធដើម្បីប្តូរ​ការ​ដាក់បញ្ចូល។

**PostgreSQL**  
ប្រព័ន្ធមូលដ្ឋានទិន្នន័យទំនើបប្រភពរូសដែលគាំទ្រជាសេវាកម្មគ្រប់គ្រងនៅលើ Azure។

**Provisioning**  
ដំណើរការបង្កើត និងកំណត់រចនាសម្ព័ន្ធធនធាន Azure ដែលបានកំណត់ក្នុងកំណត់រចនាសម្ព័ន្ធ។

## Q

**Quota**  
កំណត់លើចំនួនធនធានដែលអាចបង្កើតបានក្នុង subscription ឬតំបន់មួយនៅលើ Azure។

## R

**Resource Group**  
ធុងតំណើរយោងផ្លូវគ្រប់គ្រងសម្រាប់ធនធាន Azure ដែលរំលេចពីរយៈពេលជីវិត សិទ្ធិ និងនិតិវិធីដូចគ្នា។

**Resource Token**  
ខ្សែអត្ថបទដាច់ដោយឡែកដែលបានបង្កើតដោយ azd ដើម្បីធានាថាឈ្មោះធនធានមានភាពដាច់ដោយឡែកនៅលើការដាក់បញ្ចូលទាំងអស់។

**REST API**  
ស្ទីលស្ថាបត្យកម្មសម្រាប់រចនាកម្មវិធីបណ្ដាញដោយប្រើវិធី HTTP ។

**Rollback**  
ដំណើរការត្រឡប់ទៅកាន់កំណែចាស់នៃកម្មវិធី ឬកំណត់រចនាសម្ព័ន្ធហេដ្ឋារចនាសម្ព័ន្ធ។

## S

**Service**  
ចំណុចមួយនៃកម្មវិធីរបស់អ្នកដែលបានកំណត់នៅក្នុង azure.yaml (ឧ. frontend កម្មវិធីវែប, backend API, ទិន្នន័យ)។

**SKU**  
Stock Keeping Unit. តំណាងឱ្យថ្នាក់សេវា ឬកម្រិតប្រសិទ្ធភាពផ្សេងៗសម្រាប់ធនធាន Azure។

**SQL Database**  
សេវាកម្មមូលដ្ឋានទិន្នន័យទំនើបដែលគ្រប់គ្រងដោយ Azure ដែលផ្អែកលើ Microsoft SQL Server។

**Static Web Apps**  
សេវាកម្ម Azure សម្រាប់សាងសង់ និងដាក់បញ្ចូលកម្មវិធីវែបពេញលេញពីប្រភពកូដនៅក្នុងឃ្លីប repository។

**Storage Account**  
សេវាកម្ម Azure ដែលផ្តល់ផ្ទុកទិន្នន័យពពកសម្រាប់វត្ថុទិន្នន័យរួមមាន blobs, files, queues និង tables។

**Subscription**  
ធុងគណនី Azure ដែលផ្ទុក resource groups និងធនធាន មានការគ្រប់គ្រងវិក័យប័ត្រ និងការចូលដំណើរការ។

## T

**Template**  
សំណុំរចនាសម្ព័ន្ធគម្រោងដែលមានរួចដែលមានកូដកម្មវិធី កំណត់រចនាសម្ព័ន្ធហេដ្ឋារចនាសម្ព័ន្ធ និងការកំណត់សម្រាប់ស្ថានភាពទូទៅ។

**Terraform**  
ឧបករណ៍ Infrastructure as Code មូលដ្ឋានប្រភពបើកដែលគាំទ្រប្រភពពពកច្រើន រួមមាន Azure។

**Traffic Manager**  
ប្រព័ន្ធ DNS-based traffic load balancer របស់ Azure សម្រាប់ចែកចាយចរាចរណ៍នៅតំបន់ Azure លើកទូទៅ។

## U

**URI**  
Uniform Resource Identifier។ ខ្សែអក្សរ ដែលកំណត់ធនធានជាក់លាក់មួយ។

**URL**  
Uniform Resource Locator។ ប្រភេទនៃ URI ដែលបញ្ជាក់កន្លែងដែលធនធានត្រូវបានរកឃើញ និងវិធីសាស្រ្តទាញយកវា។

## V

**Virtual Network (VNet)**  
ឧបករណ៍មូលដ្ឋានសម្រាប់បណ្តាញឯកជនក្នុង Azure ដែលផ្តល់ការបំបែក និងការតាក់តែង។

**VS Code**  
Visual Studio Code. កម្មវិធីកាត់កូដពេញនិយមដែលមានការរួមបញ្ចូលល្អជាមួយ Azure និង azd។

## W

**Webhook**  
HTTP callback ដែលចាប់ផ្តើមដោយព្រឹត្តិការណ៍ជាក់លាក់ ត្រូវបានប្រើជាញឹកញាប់ក្នុងជង្គង់ CI/CD។

**What-if**  
មុខងាររបស់ Azure ដែលបង្ហាញថាតើមានការផ្លាស់ប្តូរអ្វីខ្លះនឹងត្រូវធ្វើដោយការដាក់បញ្ចូលដោយគ្មានការអនុវត្តពិត។

## Y

**YAML**  
YAML Ain't Markup Language. ស្តង់ដារសរសេរទិន្នន័យដែលអនុញ្ញាតឲ្យមនុស្សអានយាងងាយ ដែលប្រើសម្រាប់ឯកសារកំណត់រចនាសម្ព័ន្ធដូចជា azure.yaml។

## Z

**Zone**  
ទីតាំងដែលបំបែកយ៉ាងដាច់ដោយឡែកនៅក្នុងតំបន់ Azure ដែលផ្តល់ភាពចម្រុះ និងAvailability ខ្ពស់។

## ពាក្យតាមបរិបទជាក់លាក់

### ពាក្យផ្នែកអភិវឌ្ឍន៍
- **Hot Reload**: ការបន្ទាន់ប្តូរកម្មវិធីដោយស្វ័យដោយគ្មានការចាប់ផ្តើមឡើងវិញ
- **Build Pipeline**: ដំណើរការស្វ័យប្រវត្តសម្រាប់សាងសង់ និងតេស្តកូដ
- **Deployment Slot**: បរិយាកាសសាកល្បងនៅក្នុង App Service
- **Environment Parity**: ការរក្សាបរិយាកាសអភិវឌ្ឍ សាកល្បង និងផលិតកម្មឲ្យស្រដៀងគ្នា

### ពាក្យផ្នែកសុវត្ថិភាព
- **Managed Identity**: មុខងារដែលផ្តល់ការគ្រប់គ្រងសម្ងាត់ដោយស្វ័យ
- **Key Vault**: កន្លែងហេដ្ឋារចនាសម្ព័ន្ធសម្រាប់រក្សាសម្ងាត់ គន្លឹះ និងវិញ្ញាបនបត្រ
- **RBAC**: ការគ្រប់គ្រងចូលដោយផ្អែកលើតួនាទីសម្រាប់ធនធាន Azure
- **Network Security Group**: ទ្វារអគ្គិសនីវេចខ្ចប់មួយសម្រាប់គ្រប់គ្រងចរាចរណ៍បណ្ដាញ

### ពាក្យផ្នែកមើលលទ្ធផល
- **Telemetry**: ការប្រមូលទិន្នន័យ និងវិមាត្រដោយស្វ័យ
- **Application Performance Monitoring (APM)**: ការតាមដានប្រសិទ្ធភាពកម្មវិធី
- **Log Analytics**: សេវាកម្មសម្រាប់ប្រមូល និងវិភាគទិន្នន័យកំណត់ហេតុ
- **Alert Rules**: ការជូនដំណឹងស្វ័យប្រវត្តិតាមលទ្ធផលឬលក្ខខណ្ឌ

### ពាក្យផ្នែកដាក់បញ្ចូល
- **Blue-Green Deployment**: យុទ្ធសាស្រ្តដាក់បញ្ចូលដោយគ្មានពេលឈប់សម្រាក
- **Canary Deployment**: ការ​ចេញផ្សាយយ៉ាង تدريجي ទៅកាន់ក្រុមអ្នកប្រើខ្លះៗ
- **Rolling Update**: ការជំនួបជាលំដាប់នៃ instance កម្មវិធី
- **Rollback**: ត្រឡប់ទៅកំណែចាស់នៃកម្មវិធី

---

**ឆ្លើយប្រើប្រាស់**: ប្រើ `Ctrl+F` ដើម្បីស្វែងរកពាក្យជាក់លាក់ក្នុងវចនានុក្រមនេះយ៉ាងរហ័ស។ ពាក្យត្រូវបានភ្ជាប់យោងឆ្លើយគ្នាវាសព្វពេលដែលអាចប្រើបាន។
