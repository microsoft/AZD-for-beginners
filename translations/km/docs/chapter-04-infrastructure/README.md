# អធវទ្ទារ 4៖ វីឡាស្ថាប័នជាកូដ និង ការចេញផ្សាយ

**📚 មុខវិជ្ជា**៖ [AZD សម្រាប់អ្នកចាប់ផ្តើម](../../README.md) | **⏱️ រយៈពេល**៖ 1-1.5 ម៉ោង | **⭐ កម្រិតស្មុគស្មាញ**៖ កម្រិតកណ្ដាល

---

## របាយការណ៍ជាសង្ខេប

ជើងនេះគ្របដណ្តប់លើលំដាប់ IaC (Infrastructure as Code) ជាមួយនឹងគំរូ Bicep, ការផ្គត់ផ្គង់ធនធាន និងយុទ្ធសាស្រ្តចេញផ្សាយ ដោយប្រើ Azure Developer CLI។

> បានផ្ទៀងផ្ទាត់ជាមួយ `azd 1.27.1` នៅខែកក្កដា 2026។

## គោលបំណងសិក្សា

ដោយបញ្ចប់ជើងនេះ អ្នកនឹង:
- យល់ដឹងពីរចនាសម្ព័ន្ធ និងវេយ្យាករណ៍នៃគំរូ Bicep
- ផ្គត់ផ្គង់ធនធាន Azure ដោយប្រើ `azd provision`
- ចេញផ្សាយកម្មវិធី ដោយប្រើ `azd deploy`
- ដំណើរការយុទ្ធសាស្រ្តចេញផ្សាយស្លាបបៃតង និងជ្រាបរាប

## 🚀 ចាប់ផ្តើមយ៉ាងរហ័ស

```bash
# ចាប់ផ្តើមពីគំរូ
azd init --template azure-functions-python-v2-http

# មើលជាមុនថាអ្វីនឹងត្រូវបានបង្កើត
azd provision --preview

# ផ្គត់ផ្គង់ហេដ្ឋារចនាសម្ព័ន្ធតែប៉ុណ្ណោះ
azd provision

# ដាក់ពាក្យកូដតែប៉ុណ្ណោះ
azd deploy

# ឬទាំងពីរជាមួយគ្នា
azd up
```

## 📁 រចនាសម្ព័ន្ធគម្រោង AZD

```
my-project/
├── azure.yaml            # ← Project definition
├── infra/                # ← Infrastructure as Code
│   ├── main.bicep        # Entry point
│   ├── main.parameters.json
│   └── modules/          # Reusable components
│       ├── web.bicep
│       └── database.bicep
└── src/                  # Application code
    └── api/
```

## 🔧 ពាក្យបញ្ជារសំខាន់

| ពាក្យបញ្ជា | ពណ៌នា |
|---------|-------------|
| `azd init` | ចាប់ផ្តើមគម្រោង |
| `azd provision` | បង្កើតធនធាន Azure |
| `azd deploy` | ចេញផ្សាយកូដកម្មវិធី |
| `azd up` | ផ្គត់ផ្គង់ + ចេញផ្សាយ |
| `azd down` | លុបធនធានទាំងអស់ |
