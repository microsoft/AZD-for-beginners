# ជំពូក ៤៖ ការធ្វើអាគារ៉ាសាស្ត្រជា​កូដ និង ការដាក់ចេញ

**📚 코스**: [AZD សម្រាប់អ្នកថ្មី](../../README.md) | **⏱️ រយៈពេល**: ១-១.៥ ម៉ោង | **⭐ ភាពស្មុគស្មាញ**: មធ្យម

## គោលបំណង

ជំពូកនេះគ្របដណ្តប់លើតំរូវការ Infrastructure as Code (IaC) ជាមួយទម្រង់ Bicep, ការផ្គត់ផ្គង់ធនធាន, និងយុទ្ធសាស្ត្រការដាក់ចេញដោយការប្រើ Azure Developer CLI។

> ត្រូវបានផ្ទៀងផ្ទាត់ជាមួយ `azd 1.23.12` ក្នុងខែមីនា ឆ្នាំ ២០២៦។

## គោលបំណងសិក្សា

តាមរយៈការសម្រេចជំពូកនេះ អ្នកនឹង:
- យល់ដឹងអំពីរចនាសម្ព័ន្ធនិងវាគ្មិនទម្រង់ Bicep
- ផ្គត់ផ្គង់ធនធាន Azure ជាមួយ `azd provision`
- ដាក់ពាណិជ្ជកម្មកម្មវិធីដោយ `azd deploy`
- ប្រតិបត្តិយុទ្ធសាស្ត្រដាក់ចេញប្លូ-ហ្គ្រីន និងរោលឡើង

## 🚀 ចាប់ផ្តើមយ៉ាងឆាប់រហ័ស

```bash
# ចាប់ផ្តើមពីគំរូ
azd init --template azure-functions-python-v2-http

# មើលជាមុននូវអ្វីដែលនឹងត្រូវបង្កើត
azd provision --preview

# ផ្ដល់សេវាទំហំហេដ្ឋារចនាសម្ព័ន្ធតែប៉ុណ្ណោះ
azd provision

# ផ្ញើកូដតែប៉ុណ្ណោះ
azd deploy

# ឬទាំងពីរជាមួយគ្នា
azd up
```

## 📁 រចនាសម្ព័ន្ធគម្រោង AZD

```
my-project/
├── azure.yaml            # ← Project definition
├── infra/                # ← Infrastructure as Code
│   ├── main.bicep        # Entry point
│   ├── main.parameters.json
│   └── modules/          # Reusable components
│       ├── web.bicep
│       └── database.bicep
└── src/                  # Application code
    └── api/
```

## 🔧 ពាក្យបញ្ជាចម្បង

| ពាក្យបញ្ជា | ការពិពណ៌នា |
|----------|-------------|
| `azd init` | ចាប់ផ្តើមគម្រោង |
| `azd provision` | បង្កើតធនធាន Azure |
| `azd deploy` | ដាក់កូដកម្មវិធី |
| `azd up` | ផ្គត់ផ្គង់ + ដាក់ចេញ |
| `azd down` | លុបធនធានទាំងអស់ |
