## ការណែនាំ

នៅក្នុង [គម្រោងដំបូងរបស់អ្នក](first-project.md) អ្នកបានដាក់កម្មវិធីដោយចាប់ផ្តើមពីមេរៀនគំរូ។ ប៉ុន្តែភាគច្រើន អ្នកមានកម្មវិធីរួចហើយ — Node.js API, សេវាកម្ម Python Flask, ឬកម្មវិធីវែប .NET — ដែលមានស្ថិតនៅក្នុងថតក្នុងឧបករណ៍របស់អ្នក។ មេរៀននេះបង្ហាញពីរបៀបបន្ថែម azd ទៅកូដដែលមានរួចហើយនេះ ដើម្បីអាចដាក់ឲ្យដំណើរការ​ជាមួយ `azd up` មិនចាំបាច់ប្រើគំរូទេ។

## គោលដៅការសិក្សា

នៅចុងបញ្ចប់នៃមេរៀននេះ អ្នកនឹង:
- យល់ដឹងពីវិធីបីក្នុងការចាប់ផ្តើមគម្រោង azd
- ប្រតិបត្តិ `azd init` នៅក្នុងកូដដែលមានរួចហើយ
- យល់ពីអ្វីដែល `azure.yaml` និងថត `infra/` ធ្វើសម្រាប់កម្មវិធីរបស់អ្នក
- ដឹងពេលណាដែលអនុញ្ញាតឲ្យ azd បង្កើតហេដ្ឋារចនាសម្ព័ន្ធ បើប្រៀបធៀបនឹងសរសេរពាក្យរាល់ខ្លួន
- ដាក់កម្មវិធីដែលមានរួចហើយរបស់អ្នកទៅតំបន់ Azure ជាមួយ `azd up`

## លទ្ធផលការសិក្សា

បន្ទាប់ពីបញ្ចប់មេរៀននេះ អ្នកនឹងអាច:
- ចាប់ផ្តើម azd នៅក្នុងគម្រោងដែលអ្នកមានរួចហើយ
- អាន និងកែសម្រួលឯកសារ `azure.yaml` មូលដ្ឋាន
- បង្កើតហេដ្ឋារចនាសម្ព័ន្ធដំបូងជាមួយ `azd infra generate`
- ជ្រើសរើសម៉ាស៊ីនម៉ាស៊ីនបម្រើ Azure ត្រឹមត្រូវសម្រាប់កម្មវិធីរបស់អ្នក
- ដាក់កម្មវិធី និងសម្អាតកម្មវិធីផ្ទាល់ខ្លួនរបស់អ្នក

## វិធីបីក្នុងការចាប់ផ្តើមគម្រោង azd

| ចំណុចចាប់ផ្តើម | ពាក្យបញ្ជា | ពេលវេលាដើម្បីប្រើ |
|----------------|---------|-------------|
| **ពីគំរូ** | `azd init --template <name>` | សិក្សា ឬចាប់ផ្តើមកម្មវិធីថ្មីពីគំរូដែលពាក់ព័ន្ធ |
| **ពីកូដដែលមានរួចហើយ** | `azd init` (នៅក្នុងថតគម្រោងរបស់អ្នក) | អ្នកមានកម្មវិធីរួចហើយ និងចង់ដាក់ឲ្យដំណើរការ |
| **ពីកន្លែងផ្ទុក Git** | `azd init --from-code` (នៅក្នុង repo ដែលបាន Clone) | យក azd សម្រាប់ repo ដែលមានរួចហើយ |

អ្នកបានហាត់ខ្លួនរួចជាមួយជម្រើសដំបូង។ មេរៀននេះគឺលើវិធីទីពីរ — ជាស្ថានការណ៍ពិតប្រាកដជាច្រើនបំផុត។

## ជំហាន 1: ប្រតិបត្តិ `azd init` ក្នុងគម្រោងរបស់អ្នក

បើកតាមីណាល់ **នៅក្នុងថតគម្រោងដែលមានរួចហើយរបស់អ្នក** និងរត់:

```bash
cd my-existing-app
azd init
```

azd នឹងសួរថាតើអ្នកចង់ចាប់ផ្តើមយ៉ាងដូចម្តេច។ ជ្រើស:

```
? How do you want to initialize your app?
> Use code in the current directory
  Select a template
```

ជ្រើស **"ប្រើកូដនៅក្នុងថតបច្ចុប្បន្ន។"** azd បន្ទាប់មកស្កេនថតរបស់អ្នក ស្វែងរកភាសា និងស៊ុមឡាញវើក ហើយផ្តល់អនុសាសន៍ម៉ាស៊ីនបម្រើ។

### អ្វីដែល azd យល់ឃើញ

azd ស្វែងរកសញ្ញាណដូចជា `package.json`, `requirements.txt`, `pom.xml`, `*.csproj` ឬ `Dockerfile` ហើយផ្តល់អនុសាសន៍ម៉ាស៊ីនបម្រើ Azure ត្រូវបានតំណាងដោយ:

| កម្មវិធីរបស់អ្នក | ម៉ាស៊ីនបម្រើដែលសង្ឃឹមបាន |
|----------|----------------------|
| កម្មវិធី Node.js / Python / .NET web app | Azure App Service ឬ Container Apps |
| កម្មវិធីដែលមាន Container (`Dockerfile`) | Azure Container Apps |
| កម្មវិធី Function | Azure Functions |
| វេបសាយ Static (ផលិតផល React/Vue) | Azure Static Web Apps |

បញ្ជាក់សេវាកម្មដែលយល់ឃើញ ហើយ azd នឹងបង្កើតឯកសារដែលអ្នកត្រូវការដោយស្វ័យប្រវត្តិ។

## ជំហាន 2: យល់អំពីអ្វីដែល azd បង្កើត

បន្ទាប់ពី init អ្នកនឹងមានរឿងថ្មីពីរនៅក្នុងគម្រោងរបស់អ្នក:

```
my-existing-app/
├── azure.yaml          # ← NEW: tells azd what to deploy and where
├── infra/              # ← NEW: Infrastructure as Code (Bicep by default)
│   ├── main.bicep
│   └── main.parameters.json
├── src/ (your code)    # unchanged
└── ...                 # your existing files, untouched
```

### `azure.yaml` — ការកំណត់គម្រោង

នេះគឺជាចំនុចស្នូលនៃគម្រោង azd។ មួយដែលតិចតួចមើលទៅដូច្នេះ៖

```yaml
# azure.yaml
name: my-existing-app
services:
  web:
    project: ./src           # path to your app code
    language: js             # js | python | dotnet | java
    host: appservice         # appservice | containerapp | function | staticwebapp
```

ផ្នែក `services` ជាផ្នែកសំខាន់៖ កំណត់មួយនីមួយសម្របថតកូដរបស់អ្នកទៅម៉ាស៊ីនបម្រើ Azure។ ប្រសិនបើកម្មវិធីរបស់អ្នកមានម៉ាស៊ីនមុខ និង API នោះអ្នកនឹងមានសេវាកម្មពីរប្រភេទ។

### `infra/` — ប្រភពធនធាន Azure របស់អ្នកជាកូដ

ថត `infra/` រួមមានឯកសារ Bicep ដែលកំណត់ធនធាន Azure ដែលកម្មវិធីរបស់អ្នកត្រូវការជាទាំងអស់ (App Service, មូលដ្ឋានទិន្នន័យ, ល។)។ អ្នកមិនចាំបាច់សរសេរជាឡើងដោយដៃទេ — azd បង្កើតចំណុចចាប់ផ្តើមដែលដំណើរការបាន។ អ្នកអាចកែប្រែមកក្រោយ ដើម្បីបន្ថែមធនធាន ឬបង្អែសុវត្ថិភាព (ពិភាក្សាក្នុង [ជំពូក 4](../chapter-04-infrastructure/README.md))។

> **ដំណឹងល្អ៖** ចង់មើល ឬបានកំណត់ប្តូរហេដ្ឋារចនាសម្ព័ន្ធដែលបានបង្កើតមុនដាក់ឲ្យដំណើរការ? រត់ `azd infra generate` (នៅមានជាជម្រើស `azd infra synth`) ដើម្បីសរសេរ IaC ទៅឌីសចក្រពារដូច្នេះអ្នកអាចពិនិត្យ និងគ្រប់គ្រងកំណែបាន។

## ជំហាន 3: កំណត់ការកំណត់ចាំបាច់

ប្រសិនបើកម្មវិធីរបស់អ្នកត្រូវការកំណត់ ឬសម្ងាត់ (ខ្សែការតភ្ជាប់,កូនសោ API) កុំសរសេរជាដៃ។ ប្រើតម្លៃបរិយាកាស៖

```bash
# បង្កើតបរិយាកាស
azd env new dev

# កំណត់តម្លៃមិនមែនជាសម្ងាត់
azd env set API_VERSION 1.0.0
```

សម្រាប់សម្ងាត់ពិត ប្រះត្រូវដាក់បញ្ចូលក្នុង Key Vault ហើយយកយោងពីហេដ្ឋារចនាសម្ព័ន្ធរបស់អ្នក — សូមមើល [ជំពូក 3: ការកំណត់ និងការផ្ទៀងផ្ទាត់](../chapter-03-configuration/authsecurity.md)។

## ជំហាន 4: ដាក់ឲ្យដំណើរការ

ឥឡូវប្រើបែបបទដដែលដែលអ្នកបានស្គាល់រួចហើយ:

```bash
# ចុះឈ្មោះ Authenticate (ត្រូវការសម្រាប់ azd)
azd auth login

# 미리보기 ការបង្កើតធនធាន
azd provision --preview

# ផ្ដល់សោភ័ណរចនាសម្ព័ន្ធ និងផ្សព្វផ្សាយកូដរបស់អ្នក
azd up
```

នៅពេលវាបញ្ចប់ azd នឹងបង្ហាញ URL របស់កម្មវិធីអ្នក។ ចូរត្រួតពិនិត្យវាដូចកម្មវិធី azd ទូទៅ៖

```bash
azd show           # បង្ហាញចំណុចបញ្ចប់
azd monitor --logs # ពិនិត្យកំណត់ហេតុ ប្រសិនបើចាំបាច់
```

- `azd init` → **"ប្រើកូដនៅក្នុងថតបច្ចុប្បន្ន"** បន្ថែម azd ទៅកម្មវិធីដែលអ្នកមានរួចហើយ។
- `azure.yaml` ផ្គូផ្គងថតកូដរបស់អ្នកទៅម៉ាស៊ីនបម្រើ Azure; `infra/` កំណត់ធនធានជាឯកសារ Bicep។
- `azd infra generate` អនុញ្ញាតឲ្យអ្នកពិនិត្យ ឬប្តូរហេដ្ឋារចនាសម្ព័ន្ធដែលបានបង្កើត។
- មួយចប់ការចាប់ផ្តើម គម្រោងរបស់អ្នកដែលមានរួចហើយប្រើបែបបទដូចគ្នានៃ `azd up` / `azd down` ដូចកម្មវិធីដែលផ្អែកលើគំរូ។
