# ជំពូក 6: ការធ្វើផែនការ និងការផ្ទៀងផ្ទាត់ មុនការដាក់ឱ្យដំណើរការ

## សេចក្ដីសង្ខេប

ជំពូកនេះគ្របដណ្តប់ចំពោះជំហានសំខាន់ៗនៃការធ្វើផែនការ និងការផ្ទៀងផ្ទាត់ មុនពេលដាក់ឱ្យដំណើរការ កម្មវិធីរបស់អ្នក ។ រៀនដើម្បីជៀសវាងកំហុសដែលមានប្រសិទ្ធភាពខ្ពស់ ដោយធ្វើផែនការសមត្ថភាព ការជ្រើសរើស SKU និងការត្រួតពិនិត្យមុន (preflight checks) ឲ្យបានត្រឹមត្រូវ។

> បានផ្ទៀងផ្ទាត់ប្រឆាំងនឹង `azd 1.25.6` នៅខែមិថុនា 2026។

## គោលបំណងសិក្សា

ដោយបញ្ចប់ជំពូកនេះ អ្នកនឹងអាច:
- ដំណើរការត្រួតពិនិត្យមុន (preflight checks) មុនការដាក់ឱ្យដំណើរការ
- រៀបចំផែនការសមត្ថភាព និងប៉ាន់ប្រមាណតម្រូវការរបនធន
- ជ្រើសរើស SKU ដែលសមរម្យសម្រាប់បង្កើតប្រសិទ្ធភាពថ្លៃ
- កំណត់រចនាសម្ព័ន្ធ Application Insights សម្រាប់ការត្រួតពិនិត្យ
- យល់អំពីលំនាំសម្របសម្រួលរបស់ក្រុម

## 🚀 ចាប់ផ្តើមយ៉ាងលឿន

```bash
# ពិនិត្យកម្រិតនៃការជាវ
az vm list-usage --location eastus --output table

# មើលមុនការដាក់ចេញ (មិនបានបង្កើតធនធាន)
azd provision --preview

# ផ្ទៀងផ្ទាត់វេយ្យាករណ៍របស់ Bicep
az bicep build --file infra/main.bicep

# ពិនិត្យការកំណត់បរិយាកាស
azd env get-values
```

## ☑️ បញ្ជីត្រួតពិនិត្យ មុនការដាក់ឱ្យដំណើរការ

### មុន `azd provision`

- [ ] Quota បានផ្ទៀងផ្ទាត់សម្រាប់តំបន់
- [ ] បានជ្រើសរើស SKU ឲ្យសមរម្យ
- [ ] បានពិនិត្យការប៉ាន់ប្រមាណថ្លៃ
- [ ] ប្រព័ន្ធកំណត់ឈ្មោះមានភាពត្រួតត្រា និងជាប់គ្នា
- [ ] បានកំណត់សុវត្ថិភាព និង RBAC

### មុន `azd deploy`

- [ ] បានកំណត់អថេរបរិបទ (environment variables)
- [ ] សម្ងាត់ (secrets) បានរក្សាទុកនៅក្នុង Key Vault
- [ ] ខ្សែការតភ្ជាប់ (connection strings) ត្រូវបានផ្ទៀងផ្ទាត់
- [ ] បានកំណត់ការត្រួតពិនិត្យសុខភាព
