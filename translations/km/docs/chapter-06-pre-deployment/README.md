# ឈ្មោះជំពូក 6៖ ការរៀបចំ និងផ្ទៀងផ្ទាត់មុនពេលចាប់ផ្តើមដាក់ពាក្យ

**📚 វគ្គសិក្សា**: [AZD សម្រាប់អ្នកថ្មី](../../README.md) | **⏱️ រយៈពេល**: 1 ម៉ោង | **⭐ ស្ទួនភាព**: មធ្យម

## សេចក្ដីផ្ទាល់ខ្លួន

ជំពូកនេះគ្របដណ្តប់ដល់ជំហានសំខាន់ៗនៃការរៀបចំ និងផ្ទៀងផ្ទាត់ មុនពេលដាក់ពាក្យកម្មវិធីរបស់អ្នក។ រៀនរបៀបការពារកំហុសថ្លៃថ្នូរដោយការរៀបចំកំរិតសមត្ថភាពវាស់វែងបានត្រឹមត្រូវ ការជ្រើសរើស SKU និងការត្រួតពិនិត្យមុនពេលបង្ហាញ។

> បានផ្ទៀងផ្ទាត់ជាមួយ `azd 1.27.1` ក្នុងខែកក្កដាឆ្នាំ 2026។

## គោលបំណងការសិក្សា

ក្រោយពេលបញ្ចប់ជំពូកនេះ អ្នកនឹងអាចៈ
- ប្រតិបត្តិការត្រួតពិនិត្យមុនពេលដាក់ពាក្យ
- រៀបចំសមត្ថភាព និងប៉ាន់ប្រមាណតម្រូវការផលិតកម្ម
- ជ្រើសរើស SKUsដែលសមរម្យដើម្បីអោយថ្លៃត្រូវតាមគម្រោង
- កំណត់ការត្រួតពិនិត្យ Application Insights សម្រាប់ការតាមដាន
- យល់ដឹងពីលំនាំការសម្របសម្រួលក្រុម

## 🚀 ចាប់ផ្តើមរហ័ស

```bash
# ពិនិត្យប្រមាណប័ណ្ណជាវ
az vm list-usage --location eastus --output table

# មើលជាមុនការដាក់ជូន (គ្មានធនធានត្រូវបានបង្កើត)
azd provision --preview

# ផ្ទៀងផ្ទាត់វេយ្យាករណ៍ Bicep
az bicep build --file infra/main.bicep

# ពិនិត្យការកំណត់បរិស្ថាន
azd env get-values
```

## ☑️ បញ្ជីត្រួតពិនិត្យមុនពេលដាក់ពាក្យ

### មុន `azd provision`

- [ ] បានផ្ទៀងផ្ទាត់កាតព្វកិច្ចសម្រាប់តំបន់
- [ ] បានជ្រើសរើស SKU យ៉ាងត្រឹមត្រូវ
- [ ] បានពិនិត្យការប៉ាន់ប្រមាណថ្លៃ
- [ ] អនុវត្តន៍ការណែនាំអំពីឈ្មោះស្ដាងល្អ
- [ ] បានកំណត់សុវត្ថិភាព/សិទ្ធិ RBAC

### មុន `azd deploy`

- [ ] បានកំណត់អថេរបរិស្ថាន
- [ ] រក្សាអាថ៍កំបាំងនៅក្នុង Key Vault
- [ ] បានផ្ទៀងផ្ទាត់ខ្សែផ្សារភ្ជាប់
- [ ] បានកំណត់ការត្រួតពិនិត្យសុខភាព
