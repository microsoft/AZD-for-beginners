# ជំពូក 6: ការធ្វើផែនការ និងផ្ទៀងផ្ទាត់មុនការចែកចាយ

**📚 វគ្គ រៀន**: [AZD សម្រាប់អ្នកចាប់ផ្តើម](../../README.md) | **⏱️ រយៈពេល**: ១ ម៉ោង | **⭐ ជំពូកភាព**: កម្រិតចំរូង

---

## ទិដ្ឋភាពទូទៅ

ជំពូកនេះគ្របដណ្តប់ពីជំហានសំខាន់ៗក្នុងការធ្វើផែនការ និងផ្ទៀងផ្ទាត់មុនចែកចាយកម្មវិធីរបស់អ្នក។ រៀនពីរបៀបជៀសវាងកំហុសឆ្លាតវៃដោយមានការធ្វើផែនកំណត់កម្រិតសមត្ថភាពជាក់លាក់ ការជ្រើសរើស SKU និងការត្រួតពិនិត្យមុនចែកចាយ។

> ផ្ទៀងផ្ទាត់ជាមួយ `azd 1.23.12` នៅខែមីនា ២០២៦។

## គោលដៅការរៀន

ដោយបញ្ចប់ជំពូកនេះ អ្នកនឹងអាច៖
- រត់ត្រួតពិនិត្យមុនចែកចាយ
- ធ្វើផែនការកម្រិតសមត្ថភាព និងប៉ាន់ប្រមាណតម្រូវការរបស់ធនធាន
- ជ្រើសរើស SKU ដែលសមរម្យសម្រាប់បញ្ជាក់ថ្លៃប្រាក់
- កំណត់រចនាសម្ព័ន្ធ Application Insights សម្រាប់ការត្រួតពិនិត្យ
- យល់ដឹងអំពីលំនាំសហការក្រុម

## 🚀 ចាប់ផ្តើមបានឆាប់

```bash
# ពិនិត្យមើលបរិមាណសេវាកម្មជាវ
az vm list-usage --location eastus --output table

# មើលជាមុនការចេញផ្សាយ (មិនបានបង្កើតធនធាន)
azd provision --preview

# បញ្ជាក់វិន័យ Bicep
az bicep build --file infra/main.bicep

# ពិនិត្យការកំណត់បរិស្ថាន
azd env get-values
```

## ☑️ បញ្ជីត្រួតពិនិត្យមុនចែកចាយ

### មុន `azd provision`

- [ ] កំណត់កម្រិតចំនួនត្រូវបានផ្ទៀងផ្ទាត់សម្រាប់តំបន់
- [ ] ជ្រើសរើស SKU ដែលសមរម្យ
- [ ] ពិនិត្យប៉ាន់ប្រមាណថ្លៃ
- [ ] ឯកសារដែលឈ្មោះមានលំនាំស្របគ្នា
- [ ] កំណត់សុវត្ថិភាព/RBAC

### មុន `azd deploy`

- [ ] កំណត់អថេរស្ថានបរិយាកាស
- [ ] សម្ងាត់នៅក្នុង Key Vault
- [ ] ផ្សងពិនិត្យខ្សែការតភ្ជាប់
- [ ] កំណត់ការត្រួតពិនិត្យសុខភាព
