# ជំពូក 3: ការកំណត់រចនាសម្ព័ន្ធ និងការផ្ទៀងផ្ទាត់អត្តសញ្ញាណ

**📚 មុខវិជ្ជា**: [AZD សម្រាប់អ្នកចាប់ផ្តើម](../../README.md) | **⏱️ រយៈពេល**: 45-60 នាទី | **⭐ កម្រិតស្មុគស្មាញ**: មធ្យម

---

## ទិដ្ឋភាពទូទៅ

ជំពូកនេះគ្របដណ្តប់អំពីការកំណត់បរិស្ថាន, លំនាំនៃការផ្ទៀងផ្ទាត់អត្តសញ្ញាណ និងអនុវត្តន៍វិធានការការពារល្អបំផុតសម្រាប់ការដាក់ចេញ Azure Developer CLI។

> បានបញ្ជាក់នឹង `azd 1.27.1` នៅខែកក្កដា 2026។

## គោលបំណងការស្វែងយល់

ក្រោយពេលបញ្ចប់ជំពូកនេះ អ្នកនឹងអាច:
- គ្រប់គ្រងតម្រៀបកំណត់រចនាសម្ព័ន្ធ AZD
- គ្រប់គ្រងបរិស្ថានច្រើន (dev, staging, prod)
- អនុវត្តការផ្ទៀងផ្ទាត់អត្តសញ្ញាណដែលមានសុវត្ថិភាពជាមួយ managed identities
- កំណត់ការកំណត់ពិសេសសម្រាប់បរិស្ថាននីមួយៗ

## 🚀 ចាប់ផ្តើមរហ័ស

```bash
# បង្កើតបរិយាកាសច្រើន
azd env new dev
azd env new staging
azd env new prod

# ប្ដូរបរិយាកាស
azd env select prod

# កំណត់អថេរបរិយាកាស
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# មើលការកំណត់រចនា
azd env get-values
```

## 🔧 តម្រៀបកំណត់រចនាសម្ព័ន្ធ

AZD អនុវត្តការកំណត់ក្នុងលំដាប់នេះ (ដែលបន្ទាប់នឹងដាក់លើការកំណត់មុន):

1. **តម្លៃលំនាំដើម** (ដែលបានបង្កើតក្នុងទម្រង់)
2. **azure.yaml** (កំណត់រចនាសម្ព័ន្ធគម្រោង)
3. **អថេរបរិស្ថាន** (`azd env set`)
4. **ទង់ជួរបញ្ជាលើបញ្ជា** (`--location eastus`)

## 🔐 វិធានការ​សុវត្ថិភាព​ដ៏ល្អបំផុត

```bash
# ប្រើអត្តសញ្ញាណដែលបានគ្រប់គ្រង (ផ្ដល់អនុសាសន៍)
azd env set AZURE_USE_MANAGED_IDENTITY true

# ពិនិត្យស្ថានភាពសម្គាល់ AZD
azd auth status

# ប្រសិនបើចង់៖ សម្រង់បរិបទ Azure CLI ប្រសិនបើអ្នកមានផែនការបើកបរ az ពាក្យបញ្ជា
az account show

# សម្គាល់ឡើងវិញ ប្រសិនបើទាមទារ
azd auth login

# ប្រសិនបើចង់៖ បង្ហាញសម្គាល់ Azure CLI សម្រាប់បញ្ជា az
az login
```
