# Chapter 3: ការកំណត់រចនាសម្ព័ន្ធ និងការផ្ទៀងផ្ទាត់

**📚 코스**: [AZD សម្រាប់អ្នកចាប់ផ្តើម](../../README.md) | **⏱️ រយៈពេល**: ៤៥-៦០ នាទី | **⭐ កម្រិតភាពស្មុគស្មាញ**: មធ្យម

## ទិដ្ឋភាពទូទៅ

ជំពូកនេះគ្របដណ្ដប់ពីការកំណត់បរិវេណ, គំរូការផ្ទៀងផ្ទាត់, និងចំណាំសុវត្ថិភាពល្អបំផុតសម្រាប់ការដាក់ផ្សាយ Azure Developer CLI ។

> បានបញ្ជាក់ជាក់ស្តែងជាមួយ `azd 1.23.12` នៅខែកុម្ភៈ ២០២៦ ។

## គោលបំណងរួមរៀន

ដោយបញ្ចប់ជំពូកនេះ អ្នកនឹង៖
- ជំនាញខាងដំណើរការកំណត់រចនាសម្ព័ន្ធ AZD
- គ្រប់គ្រងបរិវេណច្រើន (dev, staging, prod)
- អនុវត្តការផ្ទៀងផ្ទាត់សុវត្ថិភាពជាមួយសញ្ជាតិបានគ្រប់គ្រង
- កំណត់តម្លៃពិសេសសម្រាប់បរិវេណរៀងរាល់ផ្នែក

## 🚀 ចាប់ផ្តើមយ៉ាងរហ័ស

```bash
# បង្កើតបរិយាកាសជាច្រើន
azd env new dev
azd env new staging
azd env new prod

# ប្តូរបរិយាកាស
azd env select prod

# កំណត់អថេរស័ព្ទបរិយាកាស
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# មើលការកំណត់រចនា
azd env get-values
```

## 🔧 ជំនាន់កំណត់រចនាសម្ព័ន្ធ

AZD អនុវត្តការកំណត់តាមលំដាប់ពីក្រោយទៅមុខ (ក្រោយលើសលែងមុន)៖

១. **តម្លៃលំនាំដើម** (ស្ថិតក្នុងមេគុណ)
២. **azure.yaml** (ការកំណត់គម្រោង)
៣. **អថេរបរិវេណ** (`azd env set`)
៤. **ទង់សញ្ញាបញ្ជាដោយបន្ទាត់ពាក្យបញ្ជា** (`--location eastus`)

## 🔐 ចំណាំសុវត្ថិភាពល្អបំផុត

```bash
# ប្រើអត្តសញ្ញាណដែលបានគ្រប់គ្រង (ជំនើល)
azd env set AZURE_USE_MANAGED_IDENTITY true

# ពិនិត្យស្ថានភាពផ្ទៀងផ្ទាត់ AZD
azd auth status

# មិនចាំបាច់: ពិនិត្យបរិបទ Azure CLI ប្រសិនបើអ្នកមានផែនការប្រើបញ្ជា az
az account show

# ផ្ទៀងផ្ទាត់ឡើងវិញប្រសិនបើត្រូវការជា
azd auth login

# មិនចាំបាច់: ពន្លត់ផ្លូវផ្ទៀងផ្ទាត់ Azure CLI សម្រាប់បញ្ជា az
az login
```
