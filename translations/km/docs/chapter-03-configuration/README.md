# ជំពូក 3: ការកំណត់ និងការផ្ទៀងផ្ទាត់អត្តសញ្ញាណ

**📚 កורס**: [AZD សម្រាប់អ្នកថ្មី](../../README.md) | **⏱️ រយៈពេល**: 45-60 នាទី | **⭐ កម្រិតភាព**: មធ្យម

---

## ទិដ្ឋភាពទូទៅ

ជំពូកនេះគ្របដណ្ដប់ការកំណត់បរិយាកាស, គំរូការផ្ទៀងផ្ទាត់, និងអនុវត្តិល្អបំផុតសុវត្ថិភាពសម្រាប់ការដាក់ប្រើ Azure Developer CLI។

> ត្រូវបានផ្ទៀងផ្ទាត់ប្រឆាំងនឹង `azd 1.25.6` ក្នុងខែមិថុនា 2026។

## គោលបំណងការសិក្សា

ដោយបញ្ចប់ជំពូកនេះ អ្នកនឹង៖
- ចាប់យល់លំដាប់តម្រៀបនៃការកំណត់ AZD
- គ្រប់គ្រងបរិយាកាសច្រើន (dev, staging, prod)
- អនុវត្តការផ្ទៀងផ្ទាត់យ៉ាងសុវត្ថិភាពជាមួយអត្តសញ្ញាណដែលគ្រប់គ្រង (managed identities)
- កំណត់ការកំណត់ជាក់លាក់សម្រាប់បរិយាកាស

## 🚀 ចាប់ផ្ដើមរហ័ស

```bash
# បង្កើតបរិយាកាសច្រើន
azd env new dev
azd env new staging
azd env new prod

# ប្តូរបរិយាកាស
azd env select prod

# កំណត់អថេរបរិយាកាស
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# មើលការកំណត់
azd env get-values
```

---

## 🔧 លំដាប់តម្រៀបនៃការកំណត់

AZD អនុវត្តការកំណត់តាមលំដាប់នេះ (អ្វីដែលនៅក្រោយនឹងជំនួសអ្វីដែលនៅមុន):

1. **តម្លៃលំនាំដើម** (បញ្ចូលក្នុងពុម្ពគំរូ)
2. **azure.yaml** (ការកំណត់គម្រោង)
3. **អថេរបរិយាកាស** (`azd env set`)
4. **ស្លាកបន្ទាត់បញ្ជា** (`--location eastus`)

## 🔐 អនុវិធីល្អបំផុតសម្រាប់សុវត្ថិភាព

```bash
# ប្រើអត្តសញ្ញាណបានគ្រប់គ្រង (ណែនាំ)
azd env set AZURE_USE_MANAGED_IDENTITY true

# ពិនិត្យស្ថានភាពការផ្ទៀងផ្ទាត់ AZD
azd auth status

# ជាជម្រើស: ផ្ទៀងផ្ទាត់បរិបទ Azure CLI ប្រសិនបើអ្នកមានគម្រោងរត់ពាក្យបញ្ជា az
az account show

# ផ្ទៀងផ្ទាត់ឡើងវិញ ប្រសិនបើចាំបាច់
azd auth login

# ជាជម្រើស: ផ្ទៀងផ្ទាត់ Azure CLI ឡើងវិញ សម្រាប់ពាក្យបញ្ជា az
az login
```
