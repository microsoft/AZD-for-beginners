## ទិដ្ឋភាពទូទៅ

មេរៀននេះជួយអ្នកពិនិត្យនិងដោះស្រាយបញ្ហាទូទៅនៅពេលប្រើ Azure Developer CLI។ ចាប់ពីការបរាជ័យក្នុងការចាក់ផ្សាយ ដល់បញ្ហាពិសេសទាក់ទងនឹង AI។

> ត្រូវបានផ្ទៀងផ្ទាត់ជាមួយ `azd 1.23.12` នៅខែកុម្ភៈ ឆ្នាំ ២០២៦។

## គោលបំណងរៀន

ពេលបញ្ចប់មេរៀននេះ អ្នកនឹង​អាចៈ
- ពិនិត្យបញ្ហាបរាជ័យក្នុងការចាក់ផ្សាយ AZD ជារឿយៗ
- កែតម្រូវបញ្ហាការផ្ទៀងផ្ទាត់និងសិทธิ
- ដោះស្រាយបញ្ហាការតភ្ជាប់សេវាកម្ម AI
- ប្រើ Azure Portal និង CLI សម្រាប់ការដោះស្រាយបញ្ហា

## 🚨 ការសម្រួលជាប់លឿន

### បញ្ហាការផ្ទៀងផ្ទាត់
```bash
# ត្រូវការសម្រាប់ដំណើរការ AZD
azd auth login

# ជាជម្រើសបើអ្នកកំពុងប្រើបញ្ជា Azure CLI ដោយផ្ទាល់ផងដែរ
az login

azd auth status
```

### ការបរាជ័យក្នុងការប្រើប្រាស់
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### ប្រថុយធនធាន
```bash
azd down --force --purge
azd env new different-name
azd up
```

### ការលេចធ្លាយកម្រិត
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 ការយោងកូដកំហុស

| កំហុស | មូលហេតុ | ដំណោះស្រាយ |
|-------|-------|----------|
| `AuthenticationError` | មិនបានចូល | `azd auth login` |
| `ResourceNotFound` | ទិន្នន័យធនធានបាត់ | ពិនិត្យឈ្មោះធនធាន |
| `QuotaExceeded` | លក្ខណៈកំណត់ការជាវ | ស្នើឲ្យកើនកម្រិត Quota |
| `InvalidTemplate` | កំហុសសន្ទស្សន៍ Bicep | `az bicep build` |
| `Conflict` | មានធនធានរួចហើយ | ប្រើឈ្មោះថ្មីឬលុប |
| `Forbidden` | មិនមានសិទ្ធិគ្រប់គ្រាន់ | ពិនិត្យតួនាទី RBAC |

## 🔄 ការកំណត់ឡើងវិញ និងការសង្រ្គោះ

```bash
# ការតំឡើងឡើងវិញមុខនន់ (រក្សាទ្រព្យសម្បត្តិ, បញ្ជូនកូដឡើងវិញ)
azd deploy --force

# ការតំឡើងឡើងវិញមុខរឹង (លុបអ្វីៗគ្រប់យ៉ាង, ចាប់ផ្តើមម្តងទៀត)
azd down --force --purge
azd up
```
