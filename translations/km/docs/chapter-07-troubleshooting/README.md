# មេរៀនទី 7៖ ការដោះស្រាយបញ្ហា និងការដោះស្រាយកំហុស

**📚 មេរៀន**: [AZD សម្រាប់អ្នកចាប់ផ្តើម](../../README.md) | **⏱️ រយៈពេល**: 1-1.5 ម៉ោង | **⭐ កម្រិតស្មុគស្មាញ**: មធ្យម

---

## ទិដ្ឋភាពទូទៅ

មេរៀននេះជួយអ្នកកំណត់សម្គាល់ និងដោះស្រាយបញ្ហាទូទៅនៅពេលធ្វើការជាមួយ Azure Developer CLI។ ចាប់ពីការបរាជ័យក្នុងការដាក់លើទៅដល់បញ្ហាជាក់លាក់នៃ AI។

> ត្រូវបានផ្អែកលើ `azd 1.27.1` នៅខែ កក្កដា ឆ្នាំ 2026។

## គោលបំណងរៀន

ក្រោយពេលបញ្ចប់មេរៀននេះ អ្នកនឹងអាច:
- វិញ្ញាណបញ្ហាបរាជ័យ AZD ដាក់លើទូទៅ
- ដោះស្រាយបញ្ហាសម្ងាត់ និងសិទ្ធិ
- ដោះស្រាយបញ្ហាការតភ្ជាប់សេវាកម្ម AI
- ប្រើប្រាស់ Azure Portal និង CLI សំរាប់ការដោះស្រាយបញ្ហា

## 🚨 ការដោះស្រាយរហ័ស

### បញ្ហាសម្ងាត់
```bash
# ត្រូវការសម្រាប់ដំណើរការងារ AZD
azd auth login

# ជាជម្រើសបើអ្នកក៏ប្រើពាក្យបញ្ជា Azure CLI ត្រូវៗផងដែរ
az login

azd auth status
```

### ការបរាជ័យក្នុងការផ្គត់ផ្គង់
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### បញ្ហារវះអ្នកជំនួញធនធាន
```bash
azd down --force --purge
azd env new different-name
azd up
```

### លើសកម្រៃតម្រូវការ
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 អ្នកយោងកូដកំហុស

| កំហុស | មូលហេតុ | ដំណោះស្រាយ |
|-------|-------|----------|
| `AuthenticationError` | មិនបានចូលគណនី | `azd auth login` |
| `ResourceNotFound` | មិនមានធនធាន | ពិនិត្យឈ្មោះធនធាន |
| `QuotaExceeded` | គ្រាប់កំណត់ជាន់ហេតុ | ស្នើរសុំបន្ថែមគ្រាប់កំណត់ |
| `InvalidTemplate` | កំហុសវាយតម្លៃ Bicep | `az bicep build` |
| `Conflict` | មានធនធានរួចហើយ | ប្រើឈ្មោះថ្មីឬលុបចោល |
| `Forbidden` | សិទ្ធិមិនគ្រប់គ្រាន់ | ពិនិត្យតួនាទី RBAC |

## 🔄 ការកំណត់ឡើងវិញ និងការស្ដារឡើងវិញ

```bash
# ការកំណត់ឡើងវិញទន់ (រក្សាទុកធនធាន, តំឡើងកូដឡើងវិញ)
azd deploy --force

# ការកំណត់ឡើងវិញរឹង (លុបអ្វីគ្រប់យ៉ាង, ចាប់ផ្តើមម្តងទៀត)
azd down --force --purge
azd up
```
