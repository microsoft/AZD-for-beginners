Tenk yu for di interest wey you get to contribute to di AZD For Beginners course! Dis document dey give guidelines and information for contributors.

## How You Fit Contribute

### How You Fit Report Issues

If you find bugs, errors, or get suggestions to make am better:

1. **Check Di Issues Wey Don Dey**: Search [di issues wey don dey](https://github.com/microsoft/azd-for-beginners/issues) to avoid duplicates
2. **Create a New Issue**: Use di appropriate issue template
3. **Provide Details**: Give steps wey fit show how to reproduce am, wetin you expect make e do, and screenshots if e apply

### Proposing Changes

1. **Fork the Repository**: Create your own fork of di project
2. **Create a Branch**: Make one feature branch for your changes
3. **Make Changes**: Follow our coding and documentation standards
4. **Test Your Changes**: Make sure say all links dey work and content dey render correct
5. **Submit a Pull Request**: Give clear description of your changes

### Types of Contributions

We dey welcome di following types of contributions:

#### Content Improvements
- Fix typos, grammar, and formatting issues
- Make di existing documentation more clear
- Update outdated information
- Add missing explanations or context

#### New Examples
- Add real-life deployment scenarios
- Create new AZD templates for common use cases
- Provide additional code samples and exercises

#### Translation Support
- Help maintain existing translations
- Improve translation quality
- Add translations for new content

#### Bug Reports
- Report broken links or incorrect code examples
- Identify documentation inconsistencies
- Flag accessibility issues

## Contribution Guidelines

### Documentation Standards

- Use clear language wey beginners fit understand
- Put learning objectives and wetin people go learn
- Give step-by-step instructions and show wetin to expect as output
- Add navigation links to related content
- Use consistent formatting and markdown style

### Code Examples

- Make sure say all code examples don test and dey work
- Include comments wey explain key concepts
- Follow Azure and AZD best practices
- Show expected output where e fit

### Commit Messages

Make your commit messages clear and descriptive:
- `docs: fix typo in installation guide`
- `feat: add new container app example`
- `fix: correct broken link in chapter 2`
