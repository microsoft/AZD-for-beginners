# అధ్యాయం 4: ఇన్‌ఫ్రాస్ట్రక్చర్‌ను కోడ్‌గా & డిప్లాయ్‌మెంట్

**📚 కోర్సు**: [AZD ప్రారంభికుల కోసం](../../README.md) | **⏱️ వ్యవధి**: 1-1.5 గంటలు | **⭐ సంక్లిష్టత**: మధ్యస్థం

---

## అవలోకనం

ఈ అధ్యాయం Bicep టెంప్లేట్స్‌తో Infrastructure as Code (IaC) నమూనాలు, వనరుల ప్రొవిజనింగ్ మరియు Azure Developer CLI ఉపయోగించి డిప్లాయ్‌మెంట్ వ్యూహాలను చర్చిస్తుంది।

> ఈది మార్చి 2026 లో `azd 1.23.12` తో ధృవీకరించబడింది.

## నేర్చుకునే లక్ష్యాలు

ఈ అధ్యాయం పూర్తి చేస్తే, మీరు:
- Bicep టెంప్లేట్ నిర్మాణం మరియు సింటాక్స్‌ను అర్థం చేసుకోవడం
- Azure వనరులను `azd provision` తో ఏర్పాటు చేయడం
- అప్లికేషన్లను `azd deploy` తో డిప్లాయ్ చేయడం
- బ్లూ-గ్రీన్ మరియు రోలింగ్ డిప్లాయ్‌మెంట్ వ్యూహాలను అమలు చేయడం

## 🚀 త్వరిత ప్రారంభం

```bash
# టెంప్లేట్ నుండి ప్రారంభించండి
azd init --template azure-functions-python-v2-http

# ఏమి సృష్టించబడబోతుందో ముందుగా చూడండి
azd provision --preview

# కేవలం ఇన్‌ఫ్రాస్ట్రక్చర్‌ను మాత్రమే ఏర్పాటు చేయండి
azd provision

# కేవలి🏼 కోడ్‌ను మాత్రమే డిప్లాయ్ చేయండి
azd deploy

# లేదా రెండింటినీ ఒకేసారి
azd up
```

## 📁 AZD ప్రాజెక్ట్ నిర్మాణం

```
my-project/
├── azure.yaml            # ← Project definition
├── infra/                # ← Infrastructure as Code
│   ├── main.bicep        # Entry point
│   ├── main.parameters.json
│   └── modules/          # Reusable components
│       ├── web.bicep
│       └── database.bicep
└── src/                  # Application code
    └── api/
```

## 🔧 అవసరమైన కమాండ్లు

| కమాండ్ | వివరణ |
|---------|-------------|
| `azd init` | ప్రాజెక్ట్‌ను ప్రారంభించండి |
| `azd provision` | Azure వనరులను సృష్టించండి |
| `azd deploy` | అప్లికేషన్ కోడ్‌ను డిప్లాయ్ చేయండి |
| `azd up` | ప్రొవిజన్ + డిప్లాయ్ |
| `azd down` | అన్ని వనరులను తొలగించండి |
