# అధ్యాయం 3: కాన్ఫిగరేషన్ & ప్రామాణీకరణ

**📚 కోర్సు**: [AZD ప్రారంభికులకు](../../README.md) | **⏱️ వ్యవధి**: 45-60 నిమిషాలు | **⭐ క్లిష్టత**: మధ్యస్థ

---

## అవలోకనం

ఈ అధ్యాయం Azure Developer CLI ద్వారా నిర్వహించే డిప్లాయ్‌మెంట్స్ కోసం పర్యావరణ కాన్ఫిగరేషన్, ప్రామాణీకరణ నమూనాలు మరియు భద్రత ఉత్తమ పద్ధతులను చర్చిస్తుంది.

## అభ్యాస లక్ష్యాలు

ఈ అధ్యాయాన్ని పూర్తిచేసిన తర్వాత, మీరు:
- AZD కాన్ఫిగరేషన్ హైరార్కీపై నైపుణ్యం పొందండి
- బహుళ పర్యావరణాలను నిర్వహించండి (dev, staging, prod)
- Managed identities ఉపయోగించి భద్రమైన ప్రామాణీకరణను అమలు చేయండి
- పర్యావరణానికి ప్రత్యేక సెట్టింగ్‌లను కాన్ఫిగర్ చేయండి

## 🚀 త్వరిత ప్రారంభం

```bash
# బహుళ ఎన్విరాన్‌మెంట్‌లు సృష్టించండి
azd env new dev
azd env new staging
azd env new prod

# ఎన్విరాన్‌మెంట్‌లను మార్చండి
azd env select prod

# ఎన్విరాన్‌మెంట్ వేరియబుల్స్‌ను సెట్ చేయండి
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# కన్ఫిగరేషన్‌ను వీక్షించండి
azd env get-values
```

---

## 🔧 కాన్ఫిగరేషన్ హైరార్కీ

AZD సెట్టింగులను ఈ క్రమంలో వర్తింప చేస్తుంది (తరువాతిది ముందివాటిని ఓవర్‌రైడ్ చేస్తుంది):

1. **డిఫాల్ట్ విలువలు** (టెంప్లేట్లలో అంతర్గతంగా ఉండేవి)
2. **azure.yaml** (ప్రాజెక్ట్ కాన్ఫిగరేషన్)
3. **పర్యావరణ వేరియబుల్స్** (`azd env set`)
4. **కమాండ్-లైన్ ఫ్లాగ్స్** (`--location eastus`)

## 🔐 భద్రత ఉత్తమ పద్ధతులు

```bash
# నిర్వహిత గుర్తింపును ఉపయోగించండి (సిఫార్సు చేయబడింది)
azd env set AZURE_USE_MANAGED_IDENTITY true

# ప్రామాణీకరణ స్థితిని తనిఖీ చేయండి
azd auth whoami
az account show

# అవసరమైతే మళ్లీ ప్రామాణీకరించండి
azd auth login
az login
```
