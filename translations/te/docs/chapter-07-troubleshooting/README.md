# అధ్యాయం 7: సమస్య పరిష్కారం & డీబగ్గింగ్

**📚 కోర్సు**: [AZD ప్రారంభకులకు](../../README.md) | **⏱️ వ్యవధి**: 1-1.5 గంటలు | **⭐ క్లిష్టత**: మధ్యస్థం

---

## అవలోకనం

ఈ అధ్యాయం Azure Developer CLI తో పని చేస్తున్నప్పుడు సాధారణ సమస్యలను గుర్తించడంలో మరియు పరిష్కరించడంలో సహాయపడుతుంది. పంపిణీ విఫలమయిన చర్యల నుండి AI-సంబంధిత సమస్యల వరకు.

> `azd 1.27.1` లో జూలై 2026 న ధృవీకరించబడింది.

## నేర్చుకునే లక్ష్యాలు

ఈ అధ్యాయం పూర్తి చేయడం ద్వారా, మీరు:
- సాధారణ AZD పంపిణీ విఫలమయిన చర్యలను గుర్తించగలరు
- ధృవీకరణ మరియు అనుమతి సమస్యలను డీబగ్ చేయగలరు
- AI సేవ కనెక్టివిటీ సమస్యలను పరిష్కరించగలరు
- ట్రబుల్‌షూటింగ్ కోసం Azure పోర్టల్ మరియు CLI ఉపయోగించగలరు

## 🚨 తక్షణ పరిష్కారాలు

### ధృవీకరణ సమస్యలు
```bash
# AZD వర్క్‌ఫ్లోల కోసం అవసరం
azd auth login

# మీరు Azure CLI కమాండ్లను నేరుగా కూడా ఉపయోగిస్తుంటే ఐచ్ఛికం
az login

azd auth status
```

### ప్రొవిజనింగ్ విఫలమయ్యే సమస్యలు
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### వనరు ఘర్షణలు
```bash
azd down --force --purge
azd env new different-name
azd up
```

### క్వోటా అధిగమించింది
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 లోపం కోడ్ సూచిక

| లోపం | కారణం | పరిష్కారం |
|-------|-------|----------|
| `AuthenticationError` | లాగిన్ కాలేదు | `azd auth login` |
| `ResourceNotFound` | వనరు లేదు | వనరు పేర్లను తనిఖీ చేయండి |
| `QuotaExceeded` | సబ్‌స్క్రిప్షన్ పరిమితులు | క్వోటా పెంపుని అభ్యర్థించండి |
| `InvalidTemplate` | Bicep వాక్య構త లోపం | `az bicep build` |
| `Conflict` | వనరు ఉన్నది | కొత్త పేరు ఉపయోగించండి లేదా తొలగించండి |
| `Forbidden` | తగిన అనుమతులు లేవు | RBAC పాత్రలను తనిఖీ చేయండి |

## 🔄 రీసెట్ మరియు పునరుద్ధరణ

```bash
# సాఫ్ట్ రీసెట్ (సंसाधనలను ఉంచి, కోడ్ మళ్లీ పంపిణీ చేయండి)
azd deploy --force

# హార్డ్ రీసెట్ (అన్నీ తొలగించి, కొత్తగా ప్రారంభించండి)
azd down --force --purge
azd up
```
