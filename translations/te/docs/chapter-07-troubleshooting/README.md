## అవలోకనం

ఈ అధ్యాయం Azure Developer CLI తో పని చేస్తున్నప్పుడు సాధారణ సమస్యలను గుర్తించడం మరియు పరిష్కరించడం లో మీకు సహాయపడుతుంది. డిప్లాయ్‌మెంట్ వైఫల్యాల నుండి AI-సంబంధిత సమస్యల వరకు.

## అధ్యయన లక్ష్యాలు

ఈ అధ్యాయం పూర్తి చేసిన తర్వాత, మీరు:
- సాధారణ AZD డిప్లాయ్‌మెంట్ వైఫల్యాలను నిర్ధారించగలరై
- ప్రామాణీకరణ మరియు అనుమతి సమస్యలను డీబగ్గ్ చేయగలరై
- AI సర్వీస్ కనెక్టివిటీ సమస్యలను పరిష్కరించగలరై
- త్రౌబుల్‌షూటింగ్ కోసం Azure పోర్టల్ మరియు CLI ఉపయోగించగలరై

## 🚨 తక్షణ పరిష్కారాలు

### ప్రామాణీకరణ సమస్యలు
```bash
azd auth login
az login
azd auth whoami
```

### ప్రొవిజనింగ్ విఫలతలు
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### వనరుల ఘర్షణలు
```bash
azd down --force --purge
azd env new different-name
azd up
```

### క్వోటా మించిపోయింది
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 దోష కోడ్ సూచిక

| దోషం | కారణం | పరిష్కారం |
|-------|-------|----------|
| `AuthenticationError` | లాగిన్ కాలేదు | `azd auth login` |
| `ResourceNotFound` | వనరు దొరకలేదు | వనరు పేర్లను తనిఖీ చేయండి |
| `QuotaExceeded` | సబ్స్క్రిప్షన్ పరిమితులు | క్వోటా పెంపును అభ్యర్థించండి |
| `InvalidTemplate` | Bicep సింటాక్స్ దోషం | `az bicep build` |
| `Conflict` | వనరు ఇప్పటికే ఉంది | కొత్త పేరు ఉపయోగించండి లేదా తొలగించండి |
| `Forbidden` | అనుమతుల లోటు | RBAC పాత్రలను తనిఖీ చేయండి |

## 🔄 రీసెట్ మరియు పునరుద్ధరణ

```bash
# సాఫ్ట్ రీసెట్ (వనరులను అలాగే ఉంచి, కోడ్‌ను మళ్లీ అమలుచేయండి)
azd deploy --force

# హార్డ్ రీసెట్ (అన్నిటినీ తొలగించి, కొత్తగా మొదలు పెట్టండి)
azd down --force --purge
azd up
```
