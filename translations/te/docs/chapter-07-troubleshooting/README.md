## అవలోకనం

ఈ అధ్యాయం Azure Developer CLI తో పనిచేసేటప్పుడు ఆమోదించే సాధారణ సమస్యలను గుర్తించి పరిష్కరించడంలో మీకు సహాయం చేస్తుంది. డిప్లాయ్‌మెంట్ వైఫల్యాల నుండి AI-సంబంధిత సమస్యల వరకు.

> మార్చి 2026లో `azd 1.23.12`తో నిర్ధారించబడింది.

## లక్ష్యాలు

ఈ అధ్యాయాన్ని పూర్తి చేయడం ద్వారా, మీరు:
- సాధారణ AZD డిప్లాయ్‌మెంట్ విఫలతలను గుర్తించడం
- ప్రామాణీకరణ మరియు అనుమతి సమస్యలను డీబగ్ చేయడం
- AI సేవ కనెక్టివిటీ సమస్యలను పరిష్కరించడం
- ట్రబుల్షూటింగ్ కోసం Azure Portal మరియు CLI ఉపయోగించడం

## 🚨 తక్షణ పరిష్కారాలు

### ప్రామాణీకరణ సమస్యలు
```bash
# AZD వర్క్‌ఫ్లోలకు అవసరం
azd auth login

# మీరు Azure CLI కమాండ్లను ప్రత్యక్షంగా కూడా ఉపయోగిస్తున్నట్లయితే ఇది ఐచ్ఛికం
az login

azd auth status
```

### ప్రొవిజనింగ్ విఫలతలు
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### వనరు ఘర్షణలు
```bash
azd down --force --purge
azd env new different-name
azd up
```

### క్వోటా మించిపోయింది
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 లోప కోడ్ సూచిక

| లోపం | కారణం | పరిష్కారం |
|-------|-------|----------|
| `AuthenticationError` | లాగిన్ కాలేదు | `azd auth login` |
| `ResourceNotFound` | వనరు కనబడలేదు | వనరు పేర్లను తనిఖీ చేయండి |
| `QuotaExceeded` | సబ్‌స్క్రిప్షన్ పరిమితులు | క్వోటా పెంపును అభ్యర్థించండి |
| `InvalidTemplate` | Bicep సింటాక్స్ లోపం | `az bicep build` |
| `Conflict` | వనరు ఇప్పటికే ఉంది | కొత్త పేరు ఉపయోగించండి లేదా తొలగించండి |
| `Forbidden` | సముచిత అనుమతులు లేవు | RBAC పాత్రలను తనిఖీ చేయండి |

## 🔄 రీసెట్ మరియు పునరుద్ధరణ

```bash
# సాఫ్ట్ రీసెట్ (వనరులను ఉంచి, కోడ్‌ను తిరిగి డిప్లాయ్ చేయండి)
azd deploy --force

# హార్డ్ రీసెట్ (అన్నింటినీ తొలగించి, కొత్తదాగా ప్రారంభించండి)
azd down --force --purge
azd up
```
