# అధ్యాయం 7: సమస్య పరిష్కరణ & డీబగ్గింగ్

**📚 కోర్సు**: [AZD ప్రారంభकर्तులకు](../../README.md) | **⏱️ వ్యవధి**: 1-1.5 గంటలు | **⭐ క్లిష్టత**: మధ్యస్థ

## అవలోకనం

ఈ అధ్యాయం Azure Developer CLI తో పనిచేసేటప్పుడు సాధారణ సమస్యలను గుర్తించి పరిష్కరించడంలో మీకు సహాయపడుతుంది. డిప్లాయ్‌మెంట్ వైఫల్యాల నుండీ AI-సంబంధిత సమస్యల వరకు.

> Validated against `azd 1.25.6` in June 2026.

## నేర్చుకునే లక్ష్యాలు

ఈ అధ్యాయం పూర్తి చేస్తే, మీరు:
- సాధారణ AZD డిప్లాయ్‌మెంట్ వైఫల్యాలను గుర్తించడం
- ప్రామాణీకరణ మరియు అనుమతి సమస్యలను డీబగ్గింగ్ చేయడం
- AI సర్వీస్ కనెక్టివిటీ సమస్యలను పరిష్కరించడం
- సమస్య పరిష్కరణ కోసం Azure Portal మరియు CLI ఉపయోగించడం

## 🚨 త్వరిత పరిష్కారాలు

### ప్రామాణీకరణ సమస్యలు
```bash
# AZD వర్క్‌ఫ్లోలకు అవసరం
azd auth login

# మీరు Azure CLI కమాండులను నేరుగా కూడా ఉపయోగిస్తున్నట్లయితే ఐచ్ఛికం
az login

azd auth status
```

### ప్రొవిజనింగ్ విఫలతలు
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### వనరు ఘర్షణలు
```bash
azd down --force --purge
azd env new different-name
azd up
```

### కోటా అధిగమించబడింది
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 లోపం కోడ్ సూచిక

| లోపం | కారణం | పరిష్కారం |
|-------|-------|----------|
| `AuthenticationError` | లాగిన్ కాలేదు | `azd auth login` |
| `ResourceNotFound` | వనరు కనబడలేదు | వనరుల పేర్లను తనిఖీ చేయండి |
| `QuotaExceeded` | సబ్‌స్క్రిప్షన్ పరిమితులు | కోటా పెంచడానికి అభ్యర్థించండి |
| `InvalidTemplate` | Bicep సింటాక్స్ లో లోపం | `az bicep build` |
| `Conflict` | వనరు ఇప్పటికే ఉంది | కొత్త పేరు ఉపయోగించండి లేదా తొలగించండి |
| `Forbidden` | అనుమతులు తగినవిగా లేవు | RBAC పాత్రలను తనిఖీ చేయండి |

## 🔄 రీసెట్ మరియు రికవరి

```bash
# సాఫ్ట్ రీసెట్ (రిసోర్సులను ఉంచి, కోడ్‌ను మళ్లీ అమర్చండి)

# హార్డ్ రీసెట్ (అన్నింటినీ తొలగించి, కొత్తగా ప్రారంభించండి)

azd up
```
