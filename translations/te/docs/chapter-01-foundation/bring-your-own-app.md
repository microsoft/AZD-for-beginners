# మీ స్వంత యాప్ తీసుకురండి - ఇప్పటికే ఉన్న ప్రాజెక్ట్‌కు azd జోడించడం

**చాప్టర్ నావిగేషన్:**
- **📚 కోర్సు హోమ్**: [AZD For Beginners](../../README.md)
- **📖 ప్రస్తుత చాప్టర్**: చాప్టర్ 1 - ఫౌండేషన్ & క్విక్ స్టార్ట్
- **⬅️ గతం**: [Your First Project](first-project.md)
- **➡️ తర్వాత**: [Dev Containers & Codespaces](dev-containers.md)

> ఆమోదించబడింది `azd 1.25.6` తో జూన్ 2026లో.

## పరిచయం

[Your First Project](first-project.md)లో మీరు టెంప్లేట్ నుండి ప్రారంభించి ఒక యాప్‌ను డిప్లాయ్ చేసారు. కానీ అధిక భాగం సందర్భాల్లో మీకు ఇప్పటికే ఒక యాప్ ఉంటుంది—ఒక Node.js API, Python Flask సర్వీస్, .NET వెబ్ యాప్—మీ యంత్రంలో ఒక ఫోల్డర్లో నిలువు. ఈ పాఠం ఆ ఉన్న కోడ్‌లో azd ని ఎలా జోడించాలో చూపిస్తుంది, తద్వారా మీరు `azd up` తో డిప్లాయ్ చేయగలిగేరు, టెంప్లేట్కు అవసరం లేదు.

## నేర్చుకునే లక్ష్యాలు

ఈ పాఠం ముగింపునకు మీరు:
- azd ప్రాజెక్ట్‌ను ప్రారంభించడానికి మూడు మార్గాలను అర్ధం చేసుకుంటారు
- ఉన్న కోడ్‌బేస్‌లో `azd init` ఎలా నడుపాలో తెలుసుకుంటారు
- `azure.yaml` మరియు `infra/` ఫోల్డర్ మీ యాప్ కోసం ఏమి చేస్తాయో అర్ధం చేసుకుంటారు
- ఎప్పుడు azd ఇన్‌ఫ్రాస్ట్రక్చర్‌ను ఉత్పత్తి చేయనివ్వడం లేదా మీకే రాయడం ఉత్తమమో తెలుసుకుంటారు
- `azd up` తో మీ ఉన్న యాప్‌ను Azureలో డిప్లాయ్ చేయగలుగుతారు

## నేర్చుకున్న ఫలితాలు

ఈ పాఠం పూర్తి చేసిన తర్వాత, మీరు చేయగలుగుతారు:
- ఇప్పటికే ఉన్న ప్రోజెక్ట్‌లో azd ని ఆరంభించడం
- బేసిక్ `azure.yaml` ఫైల్‌ను చదివి సవరించడం
- `azd infra generate` తో స్టార్టర్ ఇన్‌ఫ్రా ఉత్పత్తి చేయడం
- మీ యాప్‌కు సరిపోయే Azure హోస్ట్‌ని ఎంచుకోవడం
- మీ స్వంత అప్లికేషన్ డిప్లాయ్ చేసి శుభ్రం చేయడం

---

## azd ప్రాజెక్ట్ ప్రారంభించడానికి మూడు మార్గాలు

| ప్రారంభ బిందువు | కమాండ్ | ఎప్పుడు ఉపయోగించాలి |
|----------------|---------|----------------------|
| **టెంప్లేట్ నుండి** | `azd init --template <name>` | నేర్చుకోవడానికి, లేదా నిర్ధిష్ట నమూనా నుంచి కొత్త యప్‌ను ప్రారంభించడానికి |
| **మీ ఉన్న కోడ్ నుండి** | `azd init` (మీ ప్రాజెక్ట్ ఫోల్డర్లో) | మీ వద్ద ఇప్పటికే యాప్ ఉంది మరియు దాన్ని డిప్లాయ్ చేయాలనుకుంటున్నారు |
| **Git రిపో నుంచి** | `azd init --from-code` (క్లోన్ చేసిన రిపోలో) | ఇప్పటికే ఉన్న రిపోజిటరికి azd అప్లై చేయడానికి |

మీరు ఇప్పటికే మొదటి ఎంపికని అభ్యాసించారు. ఈ పాఠం రెండవదానిపై కవర్ చేస్తుంది—అత్యంత సామాన్యమైన వాస్తవ ప్రపంచ సన్నివేశం.

## దశ 1: మీ ప్రాజెక్ట్‌లో `azd init` నడపండి

మీ ఉన్న ప్రాజెక్ట్ ఫోల్డర్లో టెర్మినల్ ఓపెన్ చేసి నడపండి:

```bash
cd my-existing-app
azd init
```

azd మీకు ఎలా ఆరంభించాలనే అడుగుతుంది. ఎంచుకోండి:

```
? How do you want to initialize your app?
> Use code in the current directory
  Select a template
```

పిక్చ్ **"ప్రస్తుత డైరెక్టరీలోని కోడ్ ఉపయోగించండి."** azd తర్వాత మీ ఫోల్డర్‌ను స్కాన్ చేసి, మీ భాష మరియు ఫ్రేమ్‌వర్క్‌ను గుర్తించి, ఒక హోస్ట్ ప్రతిపాదిస్తుంది.

### azd ఏమి గుర్తిస్తుంది

azd `package.json`, `requirements.txt`, `pom.xml`, `*.csproj`, లేదా `Dockerfile` వంటి సంకేతాలను చూస్తుంది, మరియు సరిపోయే Azure హోస్ట్‌ను సూచిస్తుంది:

| మీ యాప్ | సాధ్యమైన గుర్తించిన హోస్ట్ |
|----------|---------------------------|
| Node.js / Python / .NET వెబ్ యాప్ | Azure App Service లేదా Container Apps |
| కంటైనరైజ్డ్ యాప్ (`Dockerfile`) | Azure Container Apps |
| Function యాప్ | Azure Functions |
| స్థిర సైట్ (React/Vue build అవుట్‌పుట్) | Azure Static Web Apps |

గుర్తించిన సర్వీస్(ల)ని నిర్ధారించండి, మరియు azd మీకు అవసరమయ్యే ఫైల్స్‌ను scaffolding చేస్తుంది.

## దశ 2: azd ఏం సృష్టించింది అనేది అర్థం చేసుకోండి

init తర్వాత, మీ ప్రాజెక్ట్‌లో రెండు కొత్త విషయాలు ఉంటాయి:

```
my-existing-app/
├── azure.yaml          # ← NEW: tells azd what to deploy and where
├── infra/              # ← NEW: Infrastructure as Code (Bicep by default)
│   ├── main.bicep
│   └── main.parameters.json
├── src/ (your code)    # unchanged
└── ...                 # your existing files, untouched
```

### `azure.yaml` — ప్రాజెక్ట్ నిర్వచనం

ఇది azd ప్రాజెక్ట్ యొక్క హృదయం. ఒక కనిష్ట ఉదాహరణ ఇలా ఉంటుంది:

```yaml
# azure.yaml
name: my-existing-app
services:
  web:
    project: ./src           # path to your app code
    language: js             # js | python | dotnet | java
    host: appservice         # appservice | containerapp | function | staticwebapp
```

`services` బ్లాక్ ముఖ్యం: ప్రతి ఎంట్రీ మీ కోడ్ ఫోల్డర్ని ఒక Azure హోస్ట్‌కు మ్యాప్ చేస్తుంది. మీ యాప్‌లో ఫ్రంట్-ఎండ్ మరియు API రెండు ఉంటే, మీ వద్ద రెండు సర్వీసులు ఉంటాయి.

### `infra/` — కోడ్ రూపంలో మీ Azure వనరులు

`infra/` ఫోల్డర్ లో Bicep ఫైళ్లుంటాయి అవి మీ యాప్‌కు అవసరమైన Azure వనరులను నిర్వచిస్తాయి (App Service, డేటాబేస్ మొదలైనవి). మీరు ఇవి మానవుల ప్రచేతనతో వ్రాయవలసిన అవసరం లేదు—azd ఒక పని ప్రారంభ బిందువు రూపొందిస్తుంది. తరువాత మీరు వాటిని సవరించవచ్చు వనరులను చేర్చడానికి లేదా సెక్యూరిటీ కట్టుదిట్టం చేయడానికి (దీని గురించి [Chapter 4](../chapter-04-infrastructure/README.md)లో తెలుసుకోండి).

> **టిప్:** డిప్లాయ్ చేసేముందు ఉత్పత్తి చేసిన ఇన్‌ఫ్రాస్ట్రక్చర్‌ని చూడాలనుకుంటే లేదా అనుకూలీకరించాలనుకుంటే, `azd infra generate` (ఇది `azd infra synth` గా కూడా లభ్యం) నడిపి IaC ను డిస్క్‌పై రాయండి తద్వారా మీరు సమీక్షించి వెర్షన్-కంట్రోల్ చేయగలుగుతారు.

## దశ 3: అవసరమైన కాన్ఫిగరేషన్ సెట్ చేయండి

మీ యాప్‌కి సెట్టింగ్స్ లేదా సీక్రెట్స్ (కనెక్షన్ స్ట్రింగ్, API కీ) అవసరమైతే, వాటిని హార్డ్‌కోడ్ చేయవద్దు. ఎన్విరాన్‌మెంట్ విలువలు ఉపయోగించండి:

```bash
# పర్యావరణాన్ని సృష్టించండి
azd env new dev

# రహస్యంగా లేని విలువను సెట్ చేయండి
azd env set API_VERSION 1.0.0
```

నిజమైన సీక్రెట్స్ కోసం, వాటిని Key Vault‌లో నిల్వ చేసి మీ ఇన్‌ఫ్రాస్ట్రక్చర్ నుండి వాటిని రెఫరెన్స్ చేయండి—చూడండి [Chapter 3: Configuration & Authentication](../chapter-03-configuration/authsecurity.md).

## దశ 4: డిప్లాయ్ చేయండి

ఇప్పుడు మీరు ఇప్పటికే తెలుసుకున్న అదే వర్క్‌ఫ్లో ఉపయోగించండి:

```bash
# ప్రామాణీకరించండి (azd కోసం అవసరం)
azd auth login

# సృష్టించబోయే వనరులను ముందుగా చూడండి
azd provision --preview

# మౌలిక సదుపాయాలను ఏర్పాటు చేసి మీ కోడ్‌ను అమలు చేయండి
azd up
```

ముగిసిన తర్వాత, azd మీ యాప్ యొక్క URLను ప్రింట్ చేస్తుంది. దాన్ని సాదారణ azd యాప్ లాంటి రీతిలో నిర్ధారించండి:

```bash
azd show           # ఎండ్‌పాయింట్లు చూపించండి
azd monitor --logs # అవసరమైతే లాగ్‌లను తనిఖీ చేయండి
```

## మొదటికిసారిగా ఎదురయ్యే సామాన్య సమస్యలు

| లక్షణం | సంభావ్య కారణం | పరిష్కారం |
|---------|----------------|-----------|
| azd నా యాప్‌ని గుర్తించలేదు | మేనిఫెస్ట్ లేకపోవడం (ఉదాహరణకి, `package.json`) | మేనిఫెస్ట్ జోడించండి, లేదా `azd init` సమయంలో హోస్ట్‌ను మాన్యువల్‌గా ఎంచుకోండి |
| `azd up` సమయంలో build విఫలమవుతుంది | యాప్‌కు build స్టెప్ అవసరం | `azure.yaml` లో సర్వీస్ క్రింద `buildCommand`/`outputPath` జోడించండి |
| యాప్ స్టార్ట్ అవుతున్నా ఎర్రర్లు తిరిగి అందుతుంది | కాన్ఫిగ్/సీక్రెట్ లేదు | `azd env set` తో విలువలు సెట్ చేయండి లేదా Key Vault ను వైర్ చేయండి |
| తప్పు హోస్ట్ ఎంచుకోబడింది | ఆటో-డిటెక్షన్ అంచనా | `azure.yaml`లో `host:` ని సవరించి `azd up` మళ్ళీ నడపండి |

వివరాలకు, చూడండి [Chapter 7: Troubleshooting](../chapter-07-troubleshooting/README.md).

## మార్గ సంచిక

- `azd init` → **"ప్రస్తుత డైరెక్టరీలోని కోడ్ ఉపయోగించండి"** మీ వద్ద ఇప్పటికే ఉన్న యాప్‌కు azd జోడిస్తుంది.
- `azure.yaml` మీ కోడ్ ఫోల్డర్లను Azure హోస్ట్‌లకు మ్యాప్ చేస్తుంది; `infra/` Bicep రూపంలో వనరులను నిర్వచిస్తుంది.
- `azd infra generate` ఉత్పత్తి చేసిన ఇన్‌ఫ్రాస్ట్రక్చర్‌ను సమీక్షించడానికి లేదా అనుకూలీకరించడానికి అనుమతిస్తుంది.
- ఒకసారి ఆరంభించిన తర్వాత, మీ ఉన్న యాప్ అనేది టెంప్లేట్-ఆధారిత యాప్ వలె అడ్డు అదే `azd up` / `azd down` వర్క్‌ఫ్లోను ఉపయోగిస్తుంది.
