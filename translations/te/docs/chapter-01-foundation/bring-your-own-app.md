## పరిచయం

[మీ మొదటి ప్రాజెక్ట్](first-project.md) లో మీరు ఒక టెంప్లేట్ నుండి ప్రారంభించి యాప్ ను నిబంధించారు. కానీ ఎక్కువగా మీ దగ్గర ఇప్పటికే ఒక యాప్ ఉంటుంది - Node.js API, Python Flask సర్వీస్, .NET వెబ్ యాప్ - మీ మెషీన్ ఫోల్‌డర్‌లో ఉంది. ఈ పాఠం మీ అందుబాటులో ఉన్న కోడ్‌కు azd ఎలా చేర్చాలో చూపిస్తుంది, ఇంతకు ముందు టెంప్లేట్లు అవసరం లేకుండా `azd up` తో డిప్లాయ్ చేయడానికి.

## నేర్చుకునే లక్ష్యాలు

ఈ పాఠం ముగిసేటప్పుడు, మీరు:
- azd ప్రాజెక్ట్ ను ప్రారంభించడానికి మూడు మార్గాలను అర్థం చేసుకోగలుగుతారు
- మీ ఉన్న కోడ్‌బేస్ లో `azd init` ను నడిపించగలుగుతారు
- `azure.yaml` మరియు `infra/` ఫోల్‌డర్ మీ యాప్‌కి ఏమి చేస్తాయో అర్థం చేసుకోగలుగుతారు
- ఎప్పుడైతే azd మిమ్మల్ని బదులుగా ఇన్‌ఫ్రాస్ట్రక్చర్ రాయనివ్వాలో తెలుసుకోగలుగుతారు
- మీ ఉన్న యాప్‌ను Azure లో `azd up` తో డిప్లాయ్ చేయగలుగుతారు

## నేర్చుకునే ఫలితాలు

ఈ పాఠం పూర్తి చేసిన తర్వాత మీరు:
- మీ వద్ద ఇప్పటికే ఉన్న ప్రాజెక్టులో azd ను ప్రారంభించగలుగుతారు
- ప్రాథమిక `azure.yaml` ఫైల్ చదవగలుగుతారు మరియు సవరించగలుగుతారు
- `azd infra generate` తో స్టార్టర్ ఇన్‌ఫ్రాస్ట్రక్చర్‌ను సృష్టించగలుగుతారు
- మీ యాప్‌కి సరైన Azure హోస్ట్ ఎంచుకోవచ్చు
- మీ స్వంత అప్లికేషన్‌ను డిప్లాయ్ చేసి శుభ్రపరచగలుగుతారు

## azd ప్రాజెక్ట్ ప్రారంభించడానికి మూడు మార్గాలు

| ప్రారంభ బిందువు | కమాండ్ | ఎప్పుడు ఉపయోగించాలి |
|----------------|---------|-------------|
| **టెంప్లేట్స్ నుండి** | `azd init --template <name>` | నేర్చుకునే లేదా సాంప్రదాయ నమూనా నుండి కొత్త యాప్ ప్రారంభించడం కోసం |
| **మీ ఉన్న కోడ్ నుండి** | `azd init` (మీ ప్రాజెక్ట్ ఫోల్డర్‌లో) | మీ దగ్గర ఇప్పటికే ఒక యాప్ ఉన్నప్పుడు దాన్ని డిప్లాయ్ చేయాలనుకుంటే |
| **Git రిపో నుండి** | `azd init --from-code` (క్లోన్ చేసిన రిపోలో) | ఉన్న రిపోకు azd అనుసంధానించడం కోసం |

మీరు మొదటి ఎంపికతో ఇప్పటికే అభ్యాసం చేసారు. ఈ పాఠం రెండవది — అత్యంత సాధారణ వాస్తవ ప్రపంచ పరిస్థితిని కవర్ చేస్తుంది.

## దశ 1: మీ ప్రాజెక్ట్‌లో `azd init` నడపండి

మీ **ఉన్న ప్రాజెక్ట్ ఫోల్డర్ లో టెర్మినల్ తెరవండి** మరియు నడపండి:

```bash
cd my-existing-app
azd init
```

azd మీరు ఎలా ఆరంభించాలో అడుగుతుంది. ఎన్నుకోండి:

```
? How do you want to initialize your app?
> Use code in the current directory
  Select a template
```

**"ప్రస్తుతం ఉన్న డైరెక్టరీలోని కోడ్‌ను ఉపయోగించండి."** ను ఎంచుకోండి. azd మీ ఫోల్డర్‌ను స్కాన్ చేస్తుంది, మీ భాష మరియు ఫ్రేమ్‌వర్క్‌ను గుర్తిస్తుంది, మరియు హోస్ట్‌ను సూచిస్తుంది.

### azd ఏమి గుర్తిస్తుంది

azd `package.json`, `requirements.txt`, `pom.xml`, `*.csproj`, లేదా `Dockerfile` లాంటి సంకేతాలను చూస్తుంది, మరియు తగిన Azure హోస్ట్‌ను సూచిస్తుంది:

| మీ యాప్ | సాధారణంగా గుర్తించబడిన హోస్ట్ |
|----------|----------------------|
| Node.js / Python / .NET వెబ్ అప్లికేషన్ | Azure App Service లేదా Container Apps |
| కంటైనరైజ్ చేసిన యాప్ (`Dockerfile`) | Azure Container Apps |
| ఫంక్షన్ యాప్ | Azure Functions |
| స్టాటిక్ సైట్ (React/Vue బిల్డ్ అవుట్‌పుట్) | Azure Static Web Apps |

గుర్తించబడిన సర్వీస్(లు) నొక్కండి, azd మీకు అవసరమైన ఫైళ్లు స్కాఫోల్డ్ చేస్తుంది.

## దశ 2: azd సృష్టించినదాన్ని అర్థం చేసుకోండి

init తర్వాత, మీ ప్రాజెక్టులో ఇద్దరు కొత్త వస్తువులు ఉంటాయి:

```
my-existing-app/
├── azure.yaml          # ← NEW: tells azd what to deploy and where
├── infra/              # ← NEW: Infrastructure as Code (Bicep by default)
│   ├── main.bicep
│   └── main.parameters.json
├── src/ (your code)    # unchanged
└── ...                 # your existing files, untouched
```

### `azure.yaml` — ప్రాజెక్ట్ నిర్వచనం

ఇది azd ప్రాజెక్ట్ హృదయం. ఒక నిమ్నస్థాయి ఇది ఇలా ఉంటుంది:

```yaml
# azure.yaml
name: my-existing-app
services:
  web:
    project: ./src           # path to your app code
    language: js             # js | python | dotnet | java
    host: appservice         # appservice | containerapp | function | staticwebapp
```

`services` బ్లాక్ ముఖ్య భాగం: ప్రతి ఎంట్రీ మీరు ఉన్న కోడ్ ఫోల్డర్ ను Azure హోస్ట్ కు మ్యాప్ చేస్తుంది. మీ యాప్‌కు ఫ్రంట్‌ఎండ్ మరియు API ఉంటే, రెండు సర్వీసులు ఉంటాయి.

### `infra/` — మీ Azure వనరులు కోడ్‌గా

`infra/` ఫోల్డర్ Bicep ఫైళ్లు కలిగి ఉంటుంది, ఇవి Azure వనరులను నిర్వచిస్తాయి (App Service, డేటాబేస్ మొదలైనవి). మీరు ఇవి చేతితో రాయాల్సిన అవసరం లేదు — azd పనిచేసే ప్రారంభ బిందువు సృష్టిస్తుంది. మీరు తరువాత వీటిని సవరించుకుని వనరులను జోడించవచ్చు లేదా భద్రత పెంచవచ్చు ([అధ్యాయం 4](../chapter-04-infrastructure/README.md) లో కవర్ చేయబడింది).

> **సూచన:** డిప్లాయ్ చేయడమునకు ముందు సృష్టించిన ఇన్‌ఫ్రాస్ట్రక్చర్‌ను చూడాలనుకుంటే లేదా అనుకూలీకరించాలనుకుంటే, `azd infra generate` (లేదా `azd infra synth`) నడపండి, దీనితో IaC డిస్క్‌కు వ్రాయబడుతుంది, మీరు సమీక్షించవచ్చు మరియు వర్షన్-కంట్రోల్ చేయవచ్చు.

## దశ 3: అవసరమైన కాన్ఫిగరేషన్‌ను సెట్ చేయండి

మీ యాప్ కు సెట్టింగులు లేదా సీక్రెట్ల (కనెక్షన్ స్ట్రింగ్, API కీ) అవసరమయితే, వాటిని హార్డ్‌కోడ్ చేయకండి. పర్యావరణ విలువలు ఉపయోగించండి:

```bash
# ఒక పరిసరాన్ని సృష్టించండి
azd env new dev

# రహస్యమైన విలువ కాని ఒకదాన్ని సెట్ చేయండి
azd env set API_VERSION 1.0.0
```

నిజమైన సీక్రెట్ల కోసం, దయచేసి వాటి యొక్క సమాచారాన్ని కీ వాల్ట్ లో నిల్వ చేయండి మరియు వాటిని మీ ఇన్‌ఫ్రాస్ట్రక్చర్ నుండి రిఫరెన్స్ చేయండి — చూడండి [అధ్యాయం 3: కాన్ఫిగరేషన్ & ప్రామాణీకరణ](../chapter-03-configuration/authsecurity.md).

## దశ 4: డిప్లాయ్ చేయండి

ఇప్పుడు మీరు ఇంతకు ముందు తెలుసుకున్న workflow ను ఉపయోగించండి:

```bash
# వ్యక్తీకరణ చెయ్యండి (azd కి అవసరం)
azd auth login

# సృష్టించబడే వనరులను ముందస్తుగా చూపించండి
azd provision --preview

# మౌలిక సదుపాయాన్ని ఏర్పాటు చేసి, మీ కోడ్‌ను అమలు చేయండి
azd up
```

పూర్తి అయినప్పుడు, azd మీ యాప్ URL ప్రింట్ చేస్తుంది. దాన్ని ఏ azd యాప్ లా ధృవీకరించండి:

```bash
azd show           # ఎండుపాయింట్లు చూపించు
azd monitor --logs # అవసరమైతే లాగ్స్ తనిఖీ చేయండి
```

## సాధారణ మొదటి సారి సమస్యలు

| లక్షణం | సాధ్యమైన కారణం | పరిష్కారం |
|---------|--------------|-----|
| azd నా యాప్ గుర్తించలేదు | మానిఫెస్ట్ లేమి (ఉదా: `package.json`) | మానిఫెస్ట్ జత చేయండి, లేదా `azd init` సమయంలో హోస్ట్‌ను మాన్యువల్‌గా ఎంచుకోండి |
| `azd up` సమయంలో బిల్డ్ విఫలమవుతుంది | యాప్‌కు బిల్డ్ స్టెప్ అవసరం | `azure.yaml` లో సర్వీస్ అర్థంలో `buildCommand`/`outputPath` జత చేయండి |
| యాప్ స్టార్ట్ అయ్యింది కాని ఎర్రర్స్ వస్తున్నాయి | కాన్ఫిగ్/సీక్రెట్ లేమి | `azd env set` తో విలువలు సెట్ చేయండి లేదా కీ వాల్ట్ ను కనెక్ట్ చేయండి |
| తప్పు హోస్ట్ ఎంచుకున్నాడు | ఆటో-డిటెక్షన్ తప్పుగా అంచనా వేశింది | `azure.yaml` లో `host:` సవరించి `azd up` మళ్ళీ నడపండి |

మరింతకి, చూడండి [అధ్యాయం 7: సమస్య పరిష్కారం](../chapter-07-troubleshooting/README.md).

## సారాంశం

- `azd init` → **"ప్రస్తుతం ఉన్న డైరెక్టరీలోని కోడ్ ను ఉపయోగించండి"** మీ దగ్గర ఇప్పటికే ఉన్న యాప్‌లో azd చేర్చుతుంది.
- `azure.yaml` మీ కోడ్ ఫోల్డర్లను Azure హోస్ట్‌లకు మ్యాప్ చేస్తుంది; `infra/` వనరులను Bicep రూపంలో నిర్వచిస్తుంది.
- `azd infra generate` మీకు సృష్టించిన ఇన్‌ఫ్రాస్ట్రక్చర్‌ను సమీక్షించడానికి లేదా అనుకూలీకరించడానికి అనుమతిస్తుంది.
- సెట్ చేశాక, మీ ఉన్న యాప్ టెంప్లేట్ ఆధారిత యాప్ లాగా అదే `azd up` / `azd down` పని ప్రవాహాన్ని ఉపయోగిస్తుంది.
