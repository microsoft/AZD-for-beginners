# అధ్యాయం 6: డిప్లాయ్‌మెంట్‌కి ముందు ప్లానింగ్ మరియు ధృవీకరణ

**📚 కోర్సు**: [AZD For Beginners](../../README.md) | **⏱️ వ్యవధి**: 1 గంట | **⭐ క్లిష్టత**: మధ్యస్థ

## అవలోకనం

ఈ అధ్యాయం మీ అప్లికేషన్‌ను డిప్లాయ్ చేసే ముందు అవసరమైన ప్లానింగ్ మరియు ధృవీకరణ దశలను కవర్ చేస్తుంది. సరైన కెపాసిటీ ప్లానింగ్, SKU ఎంపిక, మరియు ముందస్తు తనిఖీలతో ఖరీదైన పొరపాట్లను నివారించడానికి నేర్చుకోగలరు.

## నేర్చుకొనే లక్ష్యాలు

ఈ అధ్యాయాన్ని పూర్తి చేయడంతో, మీరు:
- డిప్లాయ్ చేయడానికి ముందుగా ముందస్తు తనిఖీలు నడపగలరు
- కెపాసిటీ షెడ్యూల్ చేయడం మరియు వనరుల అవసరాలను అంచనా వేయడం
- ఖర్చు ఆప్టిమైజేషన్ కోసం సరైన SKUలను ఎంపిక చేయడం
- మానిటరింగ్ కోసం Application Insights కాన్ఫిగర్ చేయడం
- జట్టు సమన్వయ నమూనాలను అర్థం చేసుకోవడం

## 🚀 త్వరిత ప్రారంభం

```bash
# సబ్‌స్క్రిప్షన్ కోటాలను తనిఖీ చేయండి
az vm list-usage --location eastus --output table

# డిప్లాయ్‌మెంట్‌ను ముందు చూపు (ఏ వనరులు సృష్టించబడవు)
azd provision --preview

# Bicep సింటాక్స్‌ను ధృవీకరించండి
az bicep build --file infra/main.bicep

# పరిసరాల అమరికను తనిఖీ చేయండి
azd env get-values
```

## ☑️ డిప్లాయ్‌మెంట్‌కు ముందు చెక్‌లిస్ట్

### Before `azd provision`

- [ ] ప్రాంతానికి కోటా ధృవీకరించబడింది
- [ ] SKUలు సరైనవిగా ఎంపిక చేయబడ్డాయి
- [ ] ఖర్చు అంచనా సమీక్షించబడింది
- [ ] పేరుప్రథా సమంజసంగా ఉంది
- [ ] భద్రత/RBAC కాన్ఫిగర్ చేయబడింది

### Before `azd deploy`

- [ ] ఎన్విరాన్‌మెంట్ వేరియబుల్స్ సెట్ చేయబడ్డాయి
- [ ] రహస్యాలు Key Vaultలో ఉన్నవి
- [ ] కనెక్షన్ స్ట్రింగ్‌లు ధృవీకరించబడ్డాయి
- [ ] హెల్త్ చెక్స్ కాన్ఫిగర్ చేయబడ్డాయి
