# అధ్యాయం 6: డిప్లాయ్‌మెంట్‌కు ముందు ప్రణాళిక & ధృవీకరణ

**📚 కోర్సు**: [AZD ప్రారంభికుల కోసం](../../README.md) | **⏱️ వ్యవధి**: 1 గంట | **⭐ సంక్లిష్టత**: మధ్య స్థాయి

## అవలోకనం

ఈ అధ్యాయం మీ యాప్‌ను డిప్లాయ్ చేయడానికి ముందు అవసరమైన ప్రణాళిక మరియు ధృవీకరణ దశలను కవర్ చేస్తుంది. సరైన సామర్థ్య ప్రణాళిక, SKU ఎంపిక మరియు ప్రీఫ్లైట్ తనిఖీల ద్వారా ఖర్చు ఎక్కువయ్యే పొరపాట్లను ఎలా నివారించాలో తెలుసుకోండి.

## నేర్చుకునే లక్ష్యాలు

ఈ అధ్యాయాన్ని పూర్తి చేయగానే, మీరు:
- డిప్లాయ్‌మెంట్‌కు ముందు ప్రీఫ్లైట్ తనిఖీలు నిర్వహించగలరు
- సామర్థ్యాన్ని ప్లాన్ చేసి వనరుల అవసరాలను అంచనా వేయగలరు
- ఖర్చు ఆప్టిమైజేషన్ కోసం సరైన SKUలను ఎంపిక చేయగలరు
- మానిటరింగ్ కోసం Application Insights‌ను కాన్ఫిగర్ చేయగలరు
- బృంద సమన్వయ నమూనాలను అర్థం చేసుకోవలరు

## 🚀 త్వరిత ప్రారంభం

```bash
# సబ్‌స్క్రిప్షన్ క్వోటాలను తనిఖీ చేయండి
az vm list-usage --location eastus --output table

# డిప్లాయ్‌మెంట్ యొక్క ముందస్తు వీక్షణ (ఏ వనరులు సృష్టించబడవు)
azd provision --preview

# Bicep సింటాక్స్‌ని ధృవీకరించండి
az bicep build --file infra/main.bicep

# పరిసర కాన్ఫిగరేషన్‌ను తనిఖీ చేయండి
azd env get-values
```

## ☑️ డిప్లాయ్‌మెంట్‌కు ముందు చెక్లిస్ట్

### ముందు `azd provision`

- [ ] ప్రాంతానికి క్వోటా నిర్ధారించబడింది
- [ ] SKUలు సరైనవిగా ఎన్నుకోబడ్డాయి
- [ ] ఖర్చు అంచనా సమీక్షించబడింది
- [ ] పేరు నియమావళి సుసంగత ఉంది
- [ ] భద్రత/RBAC కాన్ఫిగర్ చేయబడింది

### ముందు `azd deploy`

- [ ] పర్యావరణ వేరియబుల్స్ సెట్ చేయబడ్డాయి
- [ ] రహస్యాలు Key Vaultలో ఉన్నాయి
- [ ] కనెక్షన్ స్ట్రింగ్‌లు ధృవీకరించబడ్డాయి
- [ ] హెల్త్ చెక్స్ కాన్ఫిగర్ చేయబడ్డాయి
