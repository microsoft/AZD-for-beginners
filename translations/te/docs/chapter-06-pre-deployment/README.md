# అధ్యాయం 6: మునుపటి అమరిక ప్లానింగ్ & ధృవీకరణ

**📚 కోర్సు**: [AZD మొదటిసారిగా](../../README.md) | **⏱️ వ్యవధి**: 1 గంట | **⭐ కష్టం స్థాయి**: మధ్యస్థ

## అవలోకనం

మీ అనువర్తనాన్ని అమలు చేయడానికి ముందు అవసరమైన ప్లానింగ్ మరియు ధృవీకరణ దశలను ఈ అధ్యాయం కవర్ చేస్తుంది. సరైన సామర్థ్యం ప్లానింగ్, SKU ఎంపిక మరియు మునుపటి తనిఖీలు ద్వారా ఖర్చుతో కూడుకున్న తప్పులను నివారించడం నేర్చుకోండి.

> `azd 1.27.1` ని 2026 జూలైలో ధృవీకరించారు.

## అభ్యాస లక్ష్యాలు

ఈ అధ్యాయం పూర్తిచేసిన తర్వాత, మీరు:
- అమలు ముందు మునుపటి తనిఖీలు జరపండి
- సామర్థ్యాన్ని పర్యాలోచించి వనరుల అవసరాలను అంచనా వేయండి
- ఖర్చు ఆదా కోసం సరైన SKU లను ఎంచుకోండి
- Application Insights ని పర్యవేక్షణ కోసం కూర్చండి
- బృంద సమన్వయ నమూనాలను అర్థం చేసుకోండి

## 🚀 త్వరిత ప్రారంభం

```bash
# సబ్‌స్క్రిప్షన్ కోటాలు తనిఖీ చేయండి
az vm list-usage --location eastus --output table

# వినియోగం ముందుచూపు (మరియు వనరులు సృష్టించబడలేదు)
azd provision --preview

# Bicep వ్యాకరణం ధృవీకరించండి
az bicep build --file infra/main.bicep

# పర్యావరణ కాన్ఫిగరేషన్ తనిఖీ చేయండి
azd env get-values
```

## ☑️ మునుపటి అమలు తనిఖీ జాబితా

### `azd provision` ముందుగా

- [ ] ప్రాంతం కోసం కోటా నిర్ధారించబడింది
- [ ] SKU లను సరైన రీతిలో ఎంచుకున్నారు
- [ ] ఖర్చు అంచనా సమీక్షించబడింది
- [ ] పేరుపెట్టే నియమాలు సुसंगతంగా ఉన్నాయి
- [ ] సెక్యూరిటీ/RBAC ఏర్పాటు చేయబడింది

### `azd deploy` ముందుగా

- [ ] ఎన్విరన్మెంట్ వేరియబుల్స్ సెట్ చేయబడ్డాయి
- [ ] రహస్యాలను Key Vault లో ఉంచారు
- [ ] కనెక్షన్ స్ట్రింగ్లు ధృవీకరించబడ్డాయ
- [ ] ఆరోగ్య తనిఖీలు కాన్ఫిగర్ చేయబడ్డాయి
