# అధ్యాయం 6: ప్రీ-డిప్లాయ్‌మెంట్ ప్లానింగ్ & ధృవీకరణ

**📚 కోర్సు**: [AZD ప్రారంభకులకు](../../README.md) | **⏱️ కాలవ్యవధి**: 1 గంట | **⭐ క్లిష్టత**: మధ్యస్థ

## అవలోకనం

ఈ అధ్యాయం మీ అప్లికేషన్ ను డిప్లాయ్ చేసే ముందు అవసరమైన ప్లానింగ్ మరియు ధృవీకరణ దశలను కవరుస్తుంది. సరైన సామర్థ్య ప్రణాళిక, SKU ఎంపిక మరియు ప్రీఫ్లైట్ చెక్లిస్టుతో ఖరీదైన తప్పులను నివారించండి.

> ఒప్పించబడింది `azd 1.25.6` తో జూన్ 2026 లో.

## నేర్చుకోవలసిన లక్ష్యాలు

ఈ అధ్యాయం పూర్తి చేసిన తర్వాత, మీరు:
- డిప్లాయ్‌మెంట్‌కి ముందు ప్రీఫ్లైట్ తనిఖీలు నడిపడం
- సామర్థ్యాన్ని ప్లాన్ చేయడం మరియు రిసోర్స్ అవసరాలను అంచనా వేయడం
- ఖర్చు ఆప్టిమైజేషన్ కోసం తగిన SKUs ఎంపిక చేయడం
- మానిటరింగ్ కోసం Application Insights‌ని కాన్ఫిగర్ చేయడం
- టీమ్ సమన్వయ నమూనాలను అర్థం చేసుకోవడం

## 🚀 త్వరిత ప్రారంభం

```bash
# సబ్‌స్క్రిప్షన్ కోటాలను తనిఖీ చేయండి
az vm list-usage --location eastus --output table

# డిప్లాయ్‌మెంట్‌ను ముందుగా ప్రివ్యూ చేయండి (ఏ వనరులు సృష్టించబడవు)
azd provision --preview

# Bicep సింటాక్స్‌ను ధృవీకరించండి
az bicep build --file infra/main.bicep

# పర్యావరణ కాన్ఫిగరేషన్‌ను తనిఖీ చేయండి
azd env get-values
```

## ☑️ ప్రీ-డిప్లాయ్‌మెంట్ చెక్లిస్ట్

### `azd provision` కి ముందు

- [ ] రీజియన్ కోసం కోటా ధృవీకరించబడినది
- [ ] SKUలు సరిగా ఎంపిక చేయబడ్డాయి
- [ ] ఖర్చు అంచనా సమీక్షించబడింది
- [ ] నామకరణ నియమావళి స్థిరంగా ఉంది
- [ ] సెక్యూరిటీ/RBAC ఆకృతీకరించబడింది

### `azd deploy` కి ముందు

- [ ] Environment variables సెట్ చేయబడ్డాయి
- [ ] Secrets Key Vault లో ఉన్నాయి
- [ ] Connection strings ధృవీకరించబడ్డాయి
- [ ] Health checks కాన్ఫిగర్ చేయబడ్డాయి
