## వర్క్‌షాప్ అవలోకనం

ఈ హ్యాండ్స్-ఆన్ ల్యాబ్ డెవలపర్లకు ఇప్పటికే ఉన్న AI టెంప్లేట్‌ను తీసుకుని Azure Developer CLI (AZD) ఉపయోగించి దాన్ని స్థానంలో ఉంచే ప్రక్రియ ద్వారా మార్గనిర్దేశం చేస్తుంది. Microsoft Foundry సేవల avulla ప్రొడక్షన్ AI స్థాపనలకు అవసరమైన నమూనాలు నేర్చుకుంటారు.

> **వాలిడేషన్ నోట్ (2026-07-13):** ఈ వర్క్‌షాప్ `azd` `1.27.1`తో సమీక్షించబడింది. మీ స్థానిక ఇన్‌స్టాలేషన్ పాతదైతే, కింద చూపిన దశలకు అనుగుణంగా auth, template, మరియు deployment వర్క్‌ఫ్లో సరిపోవడానికి AZD ని అప్‌డేట్ చేయండి.

**దైర్ఘ్యం:** 2-3 గంటలు  
**స్థాయి:** మద్యస్త  
**ఆవశ్యకతలు:** ప్రాథమిక Azure పరిజ్ఞానం, AI/ML కాన్సెప్ట్‌లతో పరిచయం

## మాడ్యూల్ 1: AI అప్లికేషన్ల కోసం AZD నిర్మాణం అర్థం చేసుకోవడం

### AI AZD టెంప్లేట్ శరీరం

AI-రేడీ AZD టెంప్లేట్‌లో ప్రధాన ఫైల్స్‌ను అన్వేషించండి:

```
azure-search-openai-demo/
├── azure.yaml              # AZD configuration
├── infra/                   # Infrastructure as Code
│   ├── main.bicep          # Main infrastructure template
│   ├── main.parameters.json # Environment parameters
│   └── modules/            # Reusable Bicep modules
│       ├── openai.bicep    # Microsoft Foundry Models configuration
│       ├── search.bicep    # Azure AI Search setup
│       └── webapp.bicep    # Web app configuration
├── app/                    # Application code
├── scripts/               # Deployment scripts
└── .azure/               # AZD environment files
```

### **ల్యాబ్ వ్యాయామం 1.1: కాన్ఫిగరేషన్ అన్వేషణ**

1. **azure.yaml ఫైల్ పరిశీలించండి:**
```bash
cat azure.yaml
```

**ఏం చూడాలి:**
- AI భాగాల సేవ నిర్వచనలు
- పర్యావరణ వేరియబుల్ మ్యాపింగ్స్
- హోస్ట్ కాన్ఫిగరేషన్లు

2. **main.bicep ఇన్‌ఫ్రా పరిశీలించండి:**
```bash
cat infra/main.bicep
```

**గুরুతూ AI నమూనాలు:**
- Microsoft Foundry Models సేవ ప్రావిజనింగ్
- Azure AI Search ఇంటిగ్రేషన్
- సెక్యూర్ కీ నిర్వహణ
- నెట్‌వర్క్ సెక్యూరిటీ కాన్ఫిగరేషన్లు

### **చర్చ పాయింట్:** ఈ నమూనాలు AI కి ఎందుకు ముఖ్యం

- **సేవా ఆధారితులు**: AI అప్లికేషన్లు తరచుగా బహుళ సమన్వయ సేవలను అవసరమవుతాయి
- **సెక్యూరిటీ**: API కీలు మరియు ఎండ్పాయింట్లు సురక్షితంగా నిర్వహించాలి
- **స్కేలబిలిటీ**: AI వర్క్లోడ్‌లు ప్రత్యేక స్కేలింగ్ అవసరాలు కలిగి ఉంటాయి
- **ఖర్చు నిర్వహణ**: AI సేవలు సరైన కాన్ఫిగరేషన్ లేకపోతే ఖరీదైనవి కావచ్చు

## మాడ్యూల్ 2: మీ మొదటి AI అప్లికేషన్‌ని డిప్లాయ్ చేయండి

### దశ 2.1: పర్యావరణాన్ని ప్రారంభించండి

1. **కొత్త AZD పర్యావరణం సృష్టించండి:**
```bash
azd env new myai-workshop
```

2. **ఆవశ్యక పరామితులు సెట్ చేయండి:**
```bash
# మీ ఇష్టమైన Azure ప్రాంతాన్ని సెట్ చేయండి
azd env set AZURE_LOCATION eastus

# ఐచ్ఛికం: నిర్ధిష్ట OpenAI మోడల్‌ని సెట్ చేయండి
azd env set AZURE_OPENAI_MODEL gpt-4.1-mini
```

### దశ 2.2: ఇన్‌ఫ్రాస్ట్రక్చర్ మరియు అప్లికేషన్ డిప్లాయ్ చేయండి

1. **AZDతో డిప్లాయ్ చేయండి:**

**`azd up` సమయంలో తప్పక జరుగుతుంది:**
- ✅ Microsoft Foundry Models సేవను ప్రావిజన్ చేస్తుంది
- ✅ Azure AI Search సేవను సృష్టిస్తుంది
- ✅ వెబ్ అప్లికేషన్ కోసం App Service ఏర్పాటు చేస్తుంది
- ✅ నెట్‌వర్కింగ్ మరియు సెక్యూరిటీని కాన్ఫిగర్ చేస్తుంది
- ✅ అప్లికేషన్ కోడ్ ని డిప్లాయ్ చేస్తుంది
- ✅ మానిటరింగ్ మరియు లాగింగ్ ఏర్పాటు చేస్తుంది

2. **డిప్లాయ్‌మెంట్ పురోగతిని పరిశీలించండి** మరియు సృష్టిస్తున్న వనరులను గమనించండి.

2. **డిప్లాయ్ చేసిన అప్లికేషన్ ఓపెన్ చేయండి:**

`azd show` ఔట్‌పుట్ లో చూపించిన వెబ్ ఎండ్పాయింట్ ని తెరవండి.

### **ల్యాబ్ వ్యాయామం 2.1: సమస్య పరిష్కరణ అభ్యాసం**

**సన్నివేశం:** మీ డిప్లాయ్‌మెంట్ విజయవంతమయినప్పటికీ AI స్పందించడం లేదు.

2. **మోడల్ అందుబాటు**: మీ ప్రాంతం మోడల్‌కు మద్దతు ఇస్తుందా?
3. **నెట్‌వర్క్ కనెక్టివిటీ**: సేవలు కమ్యూనికేట్ అవుతున్నాయా?
4. **RBAC అనుమతులు**: యాప్ OpenAIకి యాక్సెస్ చేసుకోవచ్చు కాబట్టి ధృవీకరించండి

```bash
az webapp log tail --name YOUR_APP_NAME --resource-group YOUR_RG
```

## మాడ్యూల్ 3: మీ అవసరాల కోసం AI అప్లికేషన్లని అనుకూలపరచడం

### దశ 3.1: AI కాన్ఫిగరేషన్‌ను మార్చడం

1. **OpenAI మోడల్‌ని అప్డేట్ చేయండి:**
```bash
# వేరే మోడల్‌కు మార్చండి (మీ ప్రాంతంలో అందుబాటులో ఉంటే)
azd env set AZURE_OPENAI_MODEL gpt-4.1

# కొత్త కాన్ఫిగరేషన్‌తో మళ్లీ డిప్లాయ్ చేయండి
azd deploy
```

2. **అదనపు AI సేవలు జోడించండి:**

Document Intelligence ని జోడించడానికి `infra/main.bicep` మార్చండి:

```bicep
// Add to main.bicep
resource documentIntelligence 'Microsoft.CognitiveServices/accounts@2023-05-01' = {
  name: 'doc-intel-${uniqueString(resourceGroup().id)}'
  location: location
  kind: 'FormRecognizer'
  sku: {
    name: 'F0'  // Free tier for workshop
  }
  properties: {
    customSubDomainName: 'doc-intel-${uniqueString(resourceGroup().id)}'
  }
}
```

### దశ 3.2: పర్యావరణ-ప్రత్యేక కాన్ఫిగరేషన్లు

**ఉత్తమ పద్ధతి**: అభివృద్ధి మరియు ప్రొడక్షన్ కొరకు వేర్వేరు కాన్ఫిగరేషన్లు.

1. **ప్రొడక్షన్ పర్యావరణం సృష్టించండి:**
```bash
azd env new myai-production
```

2. **ప్రొడక్షన్-ప్రత్యేక పరామితులు సెట్ చేయండి:**
```bash
# ఉత్పత్తి సాధారణంగా అధిక SKUs ను ఉపయోగిస్తుంది
azd env set AZURE_OPENAI_SKU S0
azd env set AZURE_SEARCH_SKU standard

# అదనపు భద్రతా లక్షణాలను కొనసాగించండి
azd env set ENABLE_PRIVATE_ENDPOINTS true
```

**సవాలు**: కస్టర్ అభివృద్ధికి తక్కువ ఖర్చుతో టెంప్లేట్‌ని కాన్ఫిగర్ చేయండి.

**పనులు:**
1. ఏ SKUs ని ఫ్రీ/బేసిక్ టియర్లుగా సెట్ చేయవచ్చో గుర్తించండి
2. కనిష్ట ఖర్చు కోసం పర్యావరణ వేరియబుల్స్ సెట్ చేయండి
3. ప్రొడక్షన్ కాన్ఫిగరేషన్‌తో ఖర్చులను పోల్చుతూ డిప్లాయ్ చేయండి

**సమాధాన సూచనలు:**
- అవసరమైతే Azure AI Servicesకి F0 (ఫ్రీ) టియర్ ఉపయోగించండి
- అభివృద్ధిలో Search Serviceకి Basic tier వాడండి
- Functions కోసం Consumption ప్లాన్ పరిగణించండి

## మాడ్యూల్ 4: సెక్యూరిటీ మరియు ప్రొడక్షన్ ఉత్తమ పద్ధతులు

**ప్రస్తుత సవాలు**: బహుళ AI యాప్‌లు API కీలు హార్డ్‌కోడ్ చేయడం లేదా అసురక్షిత నిల్వ వాడటం.

**AZD పరిష్కారం**: మేనేజ్డ్ ఐడెంటిటీ + కీ వాల్ట్ ఇంటిగ్రేషన్.

1. **మీ టెంప్లేట్‌లో సెక్యూరిటీ కాన్ఫిగరేషన్ రివ్యూ చేయండి:**
```bash
# కీ వాల్ట్ మరియు మేనేజ్ చేసిన ఐడెంటిటీ కాంఫిగరేషన్ కోసం చూడండి
grep -r "keyVault\|managedIdentity" infra/
```

2. **మేనేజ్డ్ ఐడెంటిటీ పనిచేస్తుందో ధృవీకరించండి:**

```bash
# వెబ్ యాప్ సరి సరైన గుర్తింపు కాన్ఫిగరేషన్ ఉందో లేదో తనిఖీ చేయండి
```

```bicep
  }
}
```

### దశ 5.2: మీ AZD టెంప్లేట్ సృష్టించండి

**ఈ నమూనాను అనుసరించి మీ యాప్‌ని మార్చండి:**

1. **ప్రాథమిక నిర్మాణం సృష్టించండి:**
```bash
mkdir my-ai-app-azd
cd my-ai-app-azd

# AZD టెంప్లెట్‌ను 초기화 చెయండి
azd init --template minimal
```

2. **azure.yaml సృష్టించండి:**
```yaml
# Metadata
name: my-ai-app
metadata:
  template: my-ai-app-template@0.0.1-beta

# Services definition
services:
  api:
    project: ./api
    host: containerapp
  web:
    project: ./web
    host: staticwebapp
    
# Hooks for custom deployment logic  
hooks:
  predeploy:
    shell: sh
    run: echo "Preparing AI models..."
```

3. **ఇన్‌ఫ్రాస్ట్రక్చర్ టెంప్లేట్‌లను సృష్టించండి:**

**infra/main.bicep** - ప్రధాన టెంప్లేట్:
```bicep
@description('Primary location for all resources')
param location string = resourceGroup().location

@description('Name of the OpenAI service')
param openAIServiceName string = 'openai-${uniqueString(resourceGroup().id)}'

// Your AI services here
module openAI 'modules/openai.bicep' = {
  name: 'openai'
  params: {
    name: openAIServiceName
    location: location
  }
}
```

**infra/modules/openai.bicep** - OpenAI మాడ్యూల్:
```bicep
@description('Name of the OpenAI service')
param name string

@description('Location for the OpenAI service')
param location string

resource openAIAccount 'Microsoft.CognitiveServices/accounts@2023-05-01' = {
  name: name
  location: location
  kind: 'OpenAI'
  sku: {
    name: 'S0'
  }
  properties: {
    customSubDomainName: name
  }
}

output endpoint string = openAIAccount.properties.endpoint
output name string = openAIAccount.name
```

### సర్వసాధారణ డిప్లాయ్‌మెంట్ సమస్యలు

#### సమస్య 1: OpenAI సేవ క్వోటా అధిగమించింది

```bash

# క్వోటాను పెంచడానికి అభ్యర్థించండి లేదా వేరే ప్రాంతాన్ని ప్రయత్నించండి
azd env set AZURE_LOCATION westus2
azd up
```

#### సమస్య 2: మోడల్ ప్రాంతంలో అందుబాటులో లేదు
**లక్షణాలు:** AI స్పందనలు విఫలమవుతాయి లేదా మోడల్ డిప్లాయ్‌మెంట్ తప్పిదాలు వస్తున్నాయి
**పరిష్కారాలు:**
```bash
# ప్రాంతం వారీగా మోడల్ అందుబాటును తనిఖీ చేయండి

# అందుబాటులో ఉన్న మోడల్ కు నవీకరణ చేయండి

```

#### సమస్య 3: అనుమతి సమస్యలు

```bash
# పాత్ర కేటాయింపులను తనిఖీ చేయండి

  --scope /subscriptions/YOUR_SUB/resourceGroups/YOUR_RG
```

- ఖర్చుల ట్రాకింగ్

2. **అలెర్ట్స్ సెట్ చేయండి:**
```bash
# అధిక పొరపాటు రేటు కోసం హెచ్చరింపు
az monitor metrics alert create \
  --name "AI-App-High-Error-Rate" \
  --resource-group YOUR_RG \
  --target-resource-id YOUR_APP_ID \
  --condition "avg Http5xx greater than 10" \
  --description "Alert when error rate is high"
```

### **ల్యాబ్ వ్యాయామం 7.1: పనితీరు ఆప్టిమైజేషన్**

**పని**: మీ AI అప్లికేషన్‌ను పనితీరు మరియు ఖర్చు రెండింటికీ ఆప్టిమైజ్ చేయండి.

### సవాలుకు సన్నివేశం

ఈ అవసరాలతో ప్రొడక్షన్-రెడి AI ఆధారిత కస్టమర్ సర్వీస్ చాలెట్‌బాట్‌ సృష్టించాల్సిన బాధ్యత మీకు పెట్టబడింది:

**ఫంక్షనల్ అవసరాలు:**
- కస్టమర్ ఇంటరాక్షన్ కోసం వెబ్ ఇంటర్‌ఫేస్
- ప్రతిస్పందనలకి Microsoft Foundry Models తో ఇంటిగ్రేషన్
- Azure AI Search ఉపయోగించి డాక్యుమెంట్ శోధన సామర్థ్యం
- ఇప్పటికే ఉన్న కస్టమర్ డేటాబేస్‌తో ఇంటిగ్రేషన్
- బహుభాషా మద్దతు

**నాన్-ఫంక్షనల్ అవసరాలు:**
- 1000 సార్వకాలిక వినియోగదారులను నిర్వహించగలదు
- 99.9% అప్టైమ్ SLA
- SOC 2 అనుగుణత
- $500/నెలకు చెరూ ఖర్చు
- బహుళ పర్యావరణాలకు (డెవ్, స్టేజింగ్, ప్రొడ్) డిప్లాయ్ చేయండి
