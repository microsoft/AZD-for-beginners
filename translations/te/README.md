## 🆕 ఈ రోజుతొ azd లో కొత్తదేమిటి

> 📌 ఈ కోర్సు **`azd 1.27.1`** (జూలై 2026) ఆధారంగా ధృవీకరించబడింది. మీ బిల్డ్‌ని తెలుసుకోవడానికి `azd version` నడపండి, తాజా వెర్షన్ పొందడానికి `azd upgrade` ఉపయోగించండి.

Azure Developer CLI సంప్రదాయ వెబ్ యాప్స్ మరియు APIs-కంటే ఎక్కువగా అభివృద్ధి చెందింది. ఈ రోజున, azd అనేది ఏ రకమైన అప్లికేషన్ అయినా Azure లో మోపడానికి ఉపయోగించే ఏకైక టూల్—AI ఆధారిత అప్లికేషన్లు మరియు అవగాహన ఉండే ఏజెంట్లతో సహా.

మీకు దీని అర్థం ఏమిటంటే:

- **AI ఏజెంట్లు ఇప్పుడు మొదటి దశ azd వర్క్‌లోడ్లు.** మీరు ఇప్పటికే తెలుసుకున్న `azd init` → `azd up` వర్క్‌ఫ్లోని అనుసరించి AI ఏజెంట్ ప్రాజెక్టుల ప్రారంభం, మోపడం, నిర్వహణ చేయవచ్చు.
- **CLI ద్వారా పూర్తి ఏజెంట్ జీవనచక్రం.** `azure.ai.agents` ఎక్స్టెన్షన్ ఇప్పుడు మొత్తం ప్రయాణాన్ని కవర్ చేస్తుంది—`azd ai agent init` తో స్కాఫోల్డ్ చేయండి, `azd ai agent invoke` తో పరీక్షించండి (ప్రతిస్పందన సమయం వివరాలతో), `azd ai agent eval generate` మరియు `azd ai agent optimize` తో నాణ్యతను కొలవండి మరియు మెరుగుపరచండి, చివరగా `azd ai agent delete` తో శుభ్రపరచండి.
- **మరిన్ని AI నిర్మాణ بلاక్లు.** కొత్త ప్రివ్యూ ఎక్స్టెన్షన్లు—`azure.ai.skills` మరియు `azure.ai.connections`—మీకు పునర్వినియోగাযোগ్య ఏజెంట్ స్కిల్స్ మరియు ఫౌండ్రీ కనెక్షన్లను నేరుగా azd తో నిర్వహించే అవకాశం ఇస్తాయి.
- **Microsoft Foundry సమగ్రత** మోడల్ మోపడం, ఏజెంట్ హోస్టింగ్, AI సేవల కాన్ఫిగరేషన్‌ను నేరుగా azd టెంప్లేట్ పరిసరాల్లోకి తెస్తుంది.
- **దినచర్యలో సున్నితంగా మార్పులు.** తాజా విడుదలలలో `azd init` మళ్లీ నడపడం సురక్షితంగా (ఇడంపొటెంట్) చేయబడింది, `azd auth login` పాత టోకెన్లను ఆటోమేటిక్‌గా క్లియర్ చేస్తుంది, మరియు ఒక స్నేహపూర్వక `azd tool` మొదటి సారి సెటప్ ప్రాంప్ట్ జోడించబడింది.
- **కోర్ వర్క్‌ఫ్లో మారలేదు.** మీరు టోడో యాప్, మైక్రోసర్వీస్ లేదా బహు-ఏజెంట్ AI పరిష్కారం మోపినా, ఆ కమాండ్లు అదే విధంగా ఉంటాయి.

> **Aspire వాడుతున్నవారికి గమనిక:** Microsoft ఇప్పుడిప్పుడు ఉత్పత్తిని సింపుల్‌గా **Aspire** (ముందుగా ".NET Aspire") అంటుంది. azd యొక్క Aspire మద్దతు తేడాలేదు—కేవలం పేరు మాత్రమే మార్చబడింది.

మీరు ఇప్పటివరకు azd వాడితే, AI మద్దతు సహజంగా విస్తరణ—వేరు టూల్ కాదు లేదా ప్రత్యేక ట్రాక్ కాదు. మీరు కొత్తగా ప్రారంభిస్తే, ఒకే వర్క్‌ఫ్లోతో ప్రతిదీ నేర్చుకుంటారు.

## 🚀 Azure Developer CLI (azd) అంటే ఏమిటి?

**Azure Developer CLI (azd)** అనేది డెవలపర్ కి అనుకూలమైన కమాండ్-లైన్ టూల్, ఇది Azure కి అప్లికేషన్‌లను సులభంగా మోపడానికి సహాయపడుతుంది. వందల Azure వనరులను చేతుచూపుతో సృష్టించడానికి మరియు కనెక్ట్ చేసుకోవడానికి బదులు, ఒకే కమాండ్ తో మొత్తం అప్లికేషన్లను మోపవచ్చు.

### `azd up` యొక్క మాయాజాలం

```bash
# ఈ ఒంటిదే అన్ని పనులను చేస్తుంది:
# ✅ అన్ని Azure వనరులను సృష్టిస్తుంది
# ✅ నెట్‌వర్కింగ్ మరియు భద్రతను అమర్చుతుంది
# ✅ మీ అప్లికేషన్ కోడ్‌ను నిర్మిస్తుంది
# ✅ Azureకి పంపిస్తుంది
# ✅ పని చేసే URLని మీకు ఇస్తుంది
azd up
```

**అది అంతే!** Azure పోర్టల్ పై క్లిక్ చేయాల్సిన అవసరం లేదు, క్లిష్టమైన ARM టెంప్లేట్స్ నేర్చుకోవాల్సిన అవసరం లేదు, చేతితో కాన్ఫిగర్ చేసుకోవాల్సిన అవసరం లేదు - కేవలం Azure పై పనిచేస్తున్న అప్లికేషన్లు.

## ❓ Azure Developer CLI మరియు Azure CLI: తేడా ఏమిటి?

ఇది ప్రారంభీకులు ఎక్కువగా అడిగే ప్రశ్న. ఇక్కడ సరళమైన సమాధానం ఉంది:

| ఫీచర్ | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **ఉద్దేశ్యం** | వ్యక్తిగత Azure వనరులను నిర్వహించు | సంపೂರ್ಣ అప్లికేషన్లను మోపు |
| **మనస్తత్వం** | ఇన్ఫ్రాస్ట్రక్చర్-కేంద్రీకృతం | అప్లికేషన్-కేంద్రీకృతం |
| **ఉదాహరణ** | `az webapp create --name myapp...` | `azd up` |
| **అభ్యాస వక్రీకరణ** | Azure సేవల్ని తెలుసుకోవాలి | మీ అప్లికేషన్ తెలుసుకోవాలి |
| **ఉత్తమం ఎవరిదానికి** | డెవ్‌ఓప్స్, ఇన్ఫ్రాస్ట్రక్చర్ | డెవలపర్లు, ప్రోటోటైపింగ్ |

### సరళమైన ఉపమానం

- **Azure CLI** అనేది ఇలానే, ఇల్లు నిర్మించడానికి గల అన్ని పనిముట్లు—హ్యామర్స్, సంచులు, ద్వారాలు. మీరు ఏదైనా నిర్మించవచ్చు, కానీ నిర్మాణాన్ని తెలుసుకోవాలి.
- **Azure Developer CLI** అనేది కాంట్రాక్టర్‌ను నియమించడం లాంటిది—మీరు ఏమి కావాలో చెప్పండి, వారు నిర్మాణాన్ని చూసుకుంటారు.

### ఎప్పుడు ఏది ఉపయోగించాలి

| పరిస్థితి | ఈ దాన్ని ఉపయోగించండి |
|----------|----------|
| "నేను నా వెబ్ యాప్ ను త్వరగా మోపాలి" | `azd up` |
| "నాకు కేవలం స్టోరేజ్ అకౌంట్ సృష్టించాలి" | `az storage account create` |
| "నేను పూర్తి AI అప్లికేషన్ నిర్మిస్తున్నాను" | `azd init --template azure-search-openai-demo` |
| "నాకు నిర్దిష్ట Azure వనరును డీబగ్గింగ్ చేయాలి" | `az resource show` |
| "నాకు నిమిషాల్లో ప్రొడక్షన్-రెడ్డి మోపడం కావాలి" | `azd up --environment production` |

### ఇవి కలిసి పనిచేస్తాయి!

AZD లో Azure CLI ని లోపలి భాగంగా ఉపయోగిస్తుంది. మీరు రెండింటినీ ఉపయోగించవచ్చు:
```bash
# AZD తో మీ యాప్‌ను నిష్పాదించండి
azd up

# ఆపై Azure CLI తో నిర్దిష్ట వనరులను సరిగ్గా సర్దుబాటు చేయండి
az webapp config set --name myapp --always-on true
```

## 🎯 3 దశల్లో ప్రారంభం

మీరు ప్రారంభించక ముందే, మీరు మోపదలుచుకున్న టెంప్లేట్ కోసం మీ యంత్రాన్ని సిద్దం చేసుకోండి:

**విండోస్:**
```powershell
.\validate-setup.ps1
```

**macOS / Linux:**
```bash
bash ./validate-setup.sh
```

అవసరమైన తనిఖీలు ఏవైనా విఫలమైతే, ముందుగా వాటిని సరిచేసి, తర్వాత త్వరిత ప్రారంభంతో కొనసాగించండి.

### దశ 1: AZD ఇన్‌స్టాల్ చేయండి (2 నిమిషాలు)

**విండోస్:**
```powershell
winget install microsoft.azd
```

**macOS:**
```bash
brew tap azure/azd && brew install azd
```

**Linux:**
```bash
curl -fsSL https://aka.ms/install-azd.sh | bash
```

### దశ 2: AZD కోసం ప్లిహిని పొందండి (Authentication)

```bash
# మీరు ఈ కోర్సులో నేరుగా Azure CLI ఆజ్ఞలను ఉపయోగించాలని ప్లాన్ చేస్తే ఐచ్ఛికం
az login

# AZD వర్క్‌ఫ్లో కోసం అవసరం
azd auth login
```

మీకు ఏదన్నా నిర్ధారించని ఉంటే, [Installation & Setup](docs/chapter-01-foundation/installation.md#authentication-setup) లో పూర్తి సెటప్ ఫ్లో ను అనుసరించండి.

### దశ 3: మీ మొదటి యాప్ మోపండి

```bash
# ఒక టెంప్లేట్నుంచి ప్రారంభించండి
azd init --template todo-nodejs-mongo

# అజూర్‌కి పంపండి (అన్నీ సృష్టిస్తుంది!)
azd up
```

**🎉 ఇది అంతే!** మీ యాప్ ఇప్పుడు Azure పై లైవ్.

### శుభ్రపరిచే దశ (మర్చిపోకండి!)

```bash
# ప్రయోగం పూర్తయినపుడు అన్ని వనరులను తొలగించండి
azd down --force --purge
```

## కోర్సు అవలోకనం

Azure Developer CLI (azd) ను ప్రగతిశీల అభ్యాసానికి రూపొందించిన నిర్మిత అధ్యాయాల ద్వారా నైపుణ్యం సాధించండి. **Microsoft Foundry సమగ్రతతో AI అప్లికేషన్ మోపுதలపై ప్రత్యేక దృష్టి.**


### ఆధునిక డెవలపర్ల కోసం ఈ కోర్సు ఎందుకు అవసరం

Microsoft Foundry Discord కమ్యూనిటీ అవగాహన ఆధారంగా, **45% డెవలపర్లు AI వర్క్‌లోడ్‌ల కోసం AZD ఉపయోగించాలనుకుంటున్నారని** కానీ ఈ సమస్యలతో ఎదురవుతుండటం:
- సంక్లిష్టమైన బహుళ-సేవ AI నిర్మాణాలు
- ఉత్పత్తి AI орнал్దుపరి ఉత్తమ ఆచరణలు  
- Azure AI సేవల సమీకరణ మరియు కాన్ఫిగరేషన్
- AI వర్క్‌లోడ్‌ల కోసం ఖర్చు ఆప్టిమైజేషన్
- AI ప్రత్యేకమైన орнал్దుపరి సమస్యల పరిష్కారం

### నేర్చుకునే లక్ష్యాలు

ఈ నిర్మిత కోర్సు పూర్తి చేసినప్పుడల్లా, మీరు:
- **AZD పునాదులను మాస్టర్ చేయండి**: ముఖ్య భావనలు, ఇన్స్టాలేషన్, మరియు కాన్ఫిగరేషన్
- **AI అప్లికేషన్లను орнал్దుపరచండి**: Microsoft Foundry సేవలతో AZD ఉపయోగించండి
- **ఇన్‌ఫ్రాస్ట్రక్చర్ ఆజ్ కోడ్ అమలు**: Bicep టెంప్లేట్లతో Azure వనరులను నిర్వహించండి
- **ఒర్డర్పరచనలను పరిష్కరించండి**: సాధారణ సమస్యలు పరిష్కరించి బగ్‌లను సరిచూడండి
- **ఉత్పత్తికి ఆప్టిమైజ్ చేయండి**: భద్రత, స్కేలింగ్, మానిటరింగ్, మరియు వ్యయం నిర్వహణ
- **బహుళ-ఏజెంట్ పరిష్కారాలు రూపొందించండి**: సంక్లిష్ట AI నిర్మాణాలను орнал్దుపరచండి

## మీరు ప్రారంభించేముందు: ఖాతాలు, యాక్సెస్, మరియు ఊహాలు

చాప్టర్ 1 ప్రారంభించే ముందు, మీరు క్రింది అంశాలను సిద్ధంగా ఉంచండి. ఈ గైడ్‌లోని ఇన్స్టాలేషన్ దశలు ఈ ప్రాథమికాలను ఇప్పటికే నిర్వహించినట్టు భావిస్తాయి.

- **ఒక Azure సబ్‌స్క్రిప్షన్**: మీరు మీ పని సబ్‌స్క్రిప్షన్ లేదా మీ స్వంత ఖాతా నుండి ఉపయోగించవచ్చు, లేదా ప్రారంభాచేయడానికి [ఉచిత ట్రయల్](https://aka.ms/azurefreetrial) సృష్టించండి.
- **Azure వనరులను సృష్టించడానికి అనుమతి**: ఎక్కువ అధ్యయనాలకు, మీరు లక్ష్య సబ్‌స్క్రిప్షన్ లేదా రిసోర్స్ గ్రూప్‌పై కనీసం **కంట్రిబ్యూటర్** అkses ఉండాలి. కొంత అధ్యాయాలలో రిసోర్స్ గ్రూపులు, మేనేజ్డ్ ఐడెంటిటీలను, మరియు RBAC నియామకాలను మీరు సృష్టించగలరని భావించవచ్చు.
- [**GitHub ఖాతా**](https://github.com): ఈ భాండాగారం ఫార్కింగ్, మీ స్వంత మార్పులను ట్రాక్ చేయడం, మరియు GitHub కోడ్స్‌పేస్‌లను వర్క్‌షాప్ కోసం ఉపయోగించడం కోసం సహాయపడుతుంది.
- **టెంప్లేట్ రన్‌టైమ్ ప్రాథమిక అవసరాలు**: కొన్ని టెంప్లేట్లు స్థానిక సాధనాలు (Node.js, Python, Java, లేదా Docker) అవసరం. ప్రారంభించే ముందు సెటప్ వాలిడేటర్‌ను నడిపి కనబడని సాధనాలను త్వరగా గుర్తించండి.
- **ప్రాథమిక టెర్మినల్ పరిచయం**: మీరు నిపుణులు కావలసిన అవసరం లేదు, కానీ `git clone`, `azd auth login`, మరియు `azd up` వంటి ఆజ్ఞలను నడుపటానికి సౌకర్యంగా ఉండాలి.

> **ఏంటర్‌ప్రైజ్ సబ్‌స్క్రిప్షన్‌లో పని చేస్తున్నారా?**
> మీరు ఉపయోగించబోయే సబ్‌స్క్రిప్షన్ లేదా రిసోర్స్ గ్రూప్‌లో వనరులను орнал్దుపరచగలరని ముందుగా నిర్వాహకుడితో నిర్ధారించుకోండి. కాకపోతే, ప్రారంభించే ముందు సాండ్‌బాక్స్ సబ్‌స్క్రిప్షన్ లేదా కంట్రిబ్యూటర్ యాక్సెస్ అభ్యర్థించండి.

> **Azure కొత్తవారా?**
> మీరు [https://aka.ms/azurefreetrial](https://aka.ms/azurefreetrial) వద్ద స్వంత Azure ట్రయల్ లేదా పే-అస్-యూ-గో సబ్‌స్క్రిప్షన్‌తో ప్రారంభించండి తద్వారా టెనెంట్ స్థాయి ఆమోదాలు కోసం వేచి ఉండకుండా వ్యవాయసాల‌ను పూర్తిచేయగలరు.

```bash
# AI చాట్ ఇంటర్‌ఫేస్‌ను పరీక్షించండి
```

**💰 ఖర్చు అవగాహన:** $80-150/నెల అభివృద్ధి ఖర్చులు, $300-3500/నెల ఉత్పత్తి ఖర్చులు అర్థం చేసుకోండి

#### 💰 AI орнал్దుపరి కోసం ఖర్చు పరిశీలనలు

**అభివృద్ధి వాతావరణం (గణన $80-150/నెల):**
- Microsoft Foundry మోడల్స్ (పే-అస్-యూ-గో): $0-50/నెల (టోకెన్ వినియోగం ఆధారంగా)

- కంటైనర్ ఆప్స్ (ఖర్చు): $0-20/నెల

- AI సెర్చ్ (స్టాండర్డ్ ట్రియర్): $250/నెల
- కంటైనర్ ఆప్స్ (డెడికేటెడ్): $50-100/నెల
- అప్లికేషన్ ఇన్‌సైట్స్: $5-50/నెల
- స్టోరేజ్ (ప్రీమియం): $10-50/నెల

## 📖 లోతైన విశ్లేషణ: AZD సామర్ధ్యాలు

మూల విషయాలను దాటితే, AZD ఉత్పత్తి డిప్లాయ్‌మెంట్‌ల కోసం శక్తివంతమైన లక్షణాలను అందిస్తుంది:

- **ఇన్‌ఫ్రాస్ట్రక్చర్-అజ్-కోడ్** - Bicep లేదా Terraform ఉపయోగించి Azure వనరులను నిర్వహించండి  
- **ఇంటిగ్రేటెడ్ వర్క్‌ఫ్లోలు** - అప్లికేషన్లను సజావుగా ప్రావిజన్, డిప్లాయ్, మరియు విశ్లేషించండి
- **డెవలపర్-ఫ్రెండ్లీ** - డెవలపర్ ఉత్పాదకత మరియు అనుభవం కోసం ఆప్టిమైజ్ చేయబడింది

### **AZD + Microsoft Foundry: AI డిప్లాయ్‌మెంట్‌లకు పరిపూర్ణం**

**ఏదుకు AZD AI పరిష్కారాలకు?** AZD AI డెవలపర్ల ఎదుర్కొనే ప్రధాన సవాళ్లను పరిష్కరించుతుంది:

- **AI-రెడీ టెంప్లేట్లు** - Microsoft Foundry మోడల్స్, Azure AI సేవలు, మరియు ML వర్క్లోడ్స్ కోసం ముందుగా కాన్ఫిగర్ చేసిన టెంప్లేట్లు
- **భద్రతాత్మక AI డిప్లాయ్‌మెంట్‌లు** - AI సేవల, API కీలు, మరియు మోడల్ ఎండ్పాయింట్ల కోసం అద్భుత భద్రత నమూనాలు  
- **ఉత్పత్తి AI నమూనాలు** - స్కేలబుల్, ఖర్చు-సమర్థవంతమైన AI అప్లికేషన్ డిప్లాయ్‌మెంట్‌లకు బెస్ట్ ప్రాక్టీసులు
- **ఎండ్-టు-ఎండ్ AI వర్క్‌ఫ్లోలు** - మోడల్ అభివృద్ధి నుండి ఉత్పత్తి డిప్లాయ్‌మెంట్ వరకు సరైన ట్రాకింగ్‌తో
- **ఖర్చు ఆప్టిమైజేషన్** - AI వర్క్లోడ్స్ కోసం స్మార్ట్ వనరు కేటాయింపు మరియు స్కేలింగ్ వ్యూహాలు
- **Microsoft Foundry ఇంటిగ్రేషన్** - Microsoft Foundry మోడల్ క్యాటలాగ్ మరియు ఎండ్పాయింట్లకు సజావుగా కనెక్షన్

### ఉదాహరణ రకం ఆధారంగా నేర్చుకోండి

> **📌 స్థానిక vs. బాహ్య ఉదాహరణలు:**  
> **స్థానిక ఉదాహరణలు** (ఈ రిపోజిటరీలో) = వెంటనే ఉపయోగించడానికి సిద్ధంగా ఉన్నాయి  
> **బాహ్య ఉదాహరణలు** (Azure నమూనాలు) = లింక్ చేసిన రిపోజిటరీల నుండి క్లోన్ చేయండి

#### స్థానిక ఉదాహరణలు (ఇప్పుడే ఉపయోగించడానికి సిద్ధం)
- [**రిటైల్ బహుళ-ఏజెంట్ పరిష్కారం**](examples/retail-scenario.md) - ARM టెంప్లేట్లతో పూర్తి ఉత్పత్తి-సిద్ధ అమలు
  - బహుళ-ఏజెంట్ వాస్తవికత (కస్టమర్ + ఇన్వెంటరీ ఏజెంట్లు)
  - సమగ్ర మానిటరింగ్ మరియు మూల్యాంకనం
  - ARM టెంప్లేట్ ద్వారా ఒక క్లిక్ డిప్లాయ్‌మెంట్

#### స్థానిక ఉదాహరణలు - కంటेनర్ అప్లికేషన్లు (అధ్యాయాలు 2-5)
**ఈ రిపోజిటరీలో సమగ్ర కంటైనర్ డిప్లాయ్‌మెంట్ ఉదాహరణలు:**

- [**కంటైనర్ యాప్ ఉదాహరణలు**](examples/container-app/README.md) - కంటైనరైజ్డ్ డిప్లాయ్‌మెంట్లకు పూర్తి గైడ్
  - [సింపుల్ ఫ్లాస్క్ API](../../examples/container-app/simple-flask-api) - స్కేల్-టು-జీరోతో ఆధారభూత REST API
  - [మైక్రోసర్వీసెస్ ఆర్కిటెక్చర్](../../examples/container-app/microservices) - ఉత్పత్తి-సిద్ధమైన బహుసేవా డిప్లాయ్‌మెంట్
  - త్వరిత ప్రారంభం, ఉత్పత్తి, మరియు ఆధునిక డిప్లాయ్‌మెంట్ నమూనాలు
  - మానిటరింగ్, భద్రత, మరియు వ్యయ ఆప్టిమైజేషన్ మార్గదర్శకాలు

#### బహిర్గత ఉదాహరణలు - సింపుల్ అప్లికేషన్లు (అధ్యాయాలు 1-2)
**ప్రారంభించడానికి ఈ Azure నమూనాలతో కూడిన రిపోజిటరీలను క్లోన్ చేయండి:**
- [సింపుల్ వెబ్ యాప్ - Node.js + MongoDB](https://github.com/Azure-Samples/todo-nodejs-mongo) - మూలభూత డిప్లాయ్‌మెంట్ నమూనాలు
- [స్థిర వెబ్‌సైట్ - React SPA](https://github.com/Azure-Samples/todo-csharp-sql-swa-func) - స్థిర కంటెంట్ డిప్లాయ్‌మెంట్
- [కంటైనర్ యాప్ - Python Flask](https://github.com/Azure-Samples/container-apps-store-api-microservice) - REST API డిప్లాయ్‌మెంట్

#### బహిర్గత ఉదాహరణలు - డేటాబేస్ ఇంటిగ్రేషన్ (అధ్యాయాలు 3-4)  
- [డేటాబేస్ యాప్ - C# + SQL](https://github.com/Azure-Samples/todo-csharp-sql) - డేటాబేస్ కనెక్టివిటీ నమూనాలు
- [ఫంక్షన్స్ + కోస్మోస్ DB](https://github.com/Azure-Samples/todo-python-mongo-swa-func) - సర్వర్‌లెస్ డేటా వర్క్‌ఫ్లో

#### బహిర్గత ఉదాహరణలు - అధునిక నమూనాలు (అధ్యాయాలు 4-8)
- [జావా మైక్రోసర్వీసెస్](https://github.com/Azure-Samples/java-microservices-aca-lab) - బహుసేవా ఆర్కిటెక్చర్లు

## 🔧 త్వరిత పరిష్కార మార్గదర్శి

**ఇంటర్న్‌లు ఎదుర్కొనే సాధారణ సమస్యలు మరియు తక్షణ పరిష్కారాలు:**

<details>
<summary><strong>❌ "azd: కమాండ్ కనపడలేదు"</strong></summary>

```bash
# AZD మొట్టమొదటి ఇన్‌స్టాల్ చేయండి
# విండోస్ (పవర్‌షెల్):
winget install microsoft.azd

# మాక్ ఓఎస్:
brew tap azure/azd && brew install azd

# లినక్స్:
curl -fsSL https://aka.ms/install-azd.sh | bash

# ఇన్‌స్టాలేషన్‌ని ధృవీకరించండి
azd version
```
</details>

<details>
<summary><strong>❌ "సబ్స్క్రిప్షన్ కనబడలేదు" లేదా "సబ్స్క్రిప్షన్ సెట్ చేయబడలేదు"</strong></summary>

```bash
# అందుబాటులో ఉన్న సబ్‌స్క్రిప్షన్లను జాబితా చేయండి
az account list --output table

# డిఫాల్ట్ సబ్‌స్క్రిప్షన్‌ను సెట్ చేయండి
az account set --subscription "<subscription-id-or-name>"

# AZD పరిసరానికి సెట్ చేయండి
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# ధృవీకరించండి
az account show
```
</details>

<details>
<summary><strong>❌ "అపరిపూర్ణ క్వోటా" లేదా "క్వోటా మించిపోయింది"</strong></summary>

```bash
# వేరే Azure ప్రాంతాన్ని ప్రయత్నించండి
azd env set AZURE_LOCATION "westus2"
azd up

# లేదా అభివృద్ధిలో చిన్న SKUs ఉపయోగించండి
# infra/main.parameters.jsonను సవరించండి:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>

<details>
<summary><strong>❌ "azd up" మధ్యలో విఫలమౌతుంది</strong></summary>

```bash
# ఆప్షన్ 1: శుభ్రపరచి మళ్లీ ప్రయత్నించండి
azd down --force --purge
azd up

# ఆప్షన్ 2: కేవలం ఇన్‌ఫ్రాస్ట్రక్చర్ సరిచూడండి
azd provision

# ఆప్షన్ 3: సమగ్ర స్థితిని తనిఖీ చేయండి
azd show

# ఆప్షన్ 4: Azure మానిటర్‌లో లాగ్స్‌ని తనిఖీ చేయండి
azd monitor --logs
```
</details>

<details>
<summary><strong>❌ "ప్రామాణీకరణ విఫలమైంది" లేదా "టోకెన్ గడువు అయింది"</strong></summary>

```bash
# AZD కోసం మళ్లీ గుర్తింపు చెయ్యండి
azd auth logout
azd auth login

# ఐచ్ఛికం: మీరు az కమాండ్లను నడిపితే Azure CLIని కూడా రిఫ్రెష్ చేయండి
az logout
az login

# గుర్తింపును నిర్ధారించండి
az account show
```
</details>

<details>
<summary><strong>❌ "సాధనం ఇప్పటికే არსებობს" లేదా పేరు బహిర్గత సమస్యలు</strong></summary>

```bash
# AZD ప్రత్యేకమైన పేర్లను సృష్టిస్తుంది, కానీ ప‌రిష్కారం ఉందంటే:
azd down --force --purge

# ఆపై కొత్త వాతావరణంతో మళ్లీ ప్రయత్నించండి
azd env new dev-v2
azd up
```
</details>

<details>
<summary><strong>❌ టెంప్లేట్ డిప్లాయ్‌మెంట్ చాలా కాలం తీసుకుంటుంది</strong></summary>

**సాధారణ వేచి ఉండే సమయాలు:**
- సింపుల్ వెబ్ యాప్: 5-10 నిమిషాలు
- డేటాబేస్ కలిగిన యాప్: 10-15 నిమిషాలు
- AI అప్లికేషన్లు: 15-25 నిమిషాలు (OpenAI ప్రావిజనింగ్ మందగించును)

```bash
# పురోగతిని తనిఖీ చేయండి
azd show

# 30 నిమిషాల కంటే ఎక్కువ సేపు అడ్డుకునే ఉంటే, Azure పోర్టల్‌ను తనిఖీ చేయండి:
azd monitor --overview
# విఫలమైన డిప్లాయ్‌మెంట్‌ల కోసం చూస్తే
```
</details>

<details>
<summary><strong>❌ "అనుమతి నిరాకరించబడింది" లేదా "నిషేధించబడింది"</strong></summary>

```bash

az role assignment list --assignee $(az account show --query user.name -o tsv)

# కనీసం "కాంట్రిబ్యూటర్" పాత్ర అవసరం
# మీ Azure అడ్మిన్‌ను అనుమతించమని అడగండి:
# - కాంట్రిబ్యూటర్ (సంపదల కోసం)
# - యూజర్ 액్సెస్ అడ్మినిస్ట్రేటర్ (పాత్ర కేటాయింపుల కోసం)
```

</details>

<details>
<summary><strong>❌ విస్తరించబడిన అప్లికేషన్ URL కనబడడానికి లేదు</strong></summary>
