## 1. పరిచయం

The [Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) or `azd` ఒక ఓపెన్-సోర్స్ కమాండ్‌లైన్ టూల్, ఇది Azure కు అప్లికేషన్లు నిర్మించడం మరియు డిప్లాయ్ చేయడం సమయంలో డెవలపర్ వర్క్‌ఫ్లోను సులభతరం చేస్తుంది। 

[AZD Templates](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) ప్రామాణికీకృత రెపోసిటరీలు, వీటిలో నమూనా అప్లికేషన్ కోడ్, _infrastructure-as-code_ ఆస్తులు మరియు సమగ్ర సొల్యూషన్ ఆర్కిటెక్చర్ కోసం `azd` కాన్ఫిగరేషన్ ఫైళ్లుంటాయి. ఇన్‌ఫ్రాస్ట్రక్చర్‌ను ప్రొవైజన్ చేయడం `azd provision` ఆజ్ఞలతో చాలా సులభం అవుతుంది — అదే సమయంలో `azd up` ఉపయోగించడం ద్వారా మీరు ఒకే సమయంలో ఇన్‌ఫ్రాస్ట్రక్చర్‌ను ప్రొవైజన్ చేసి మీ అప్లికేషన్‌ను డిప్లాయ్ చేయవచ్చు!

దరఖాస్తుల అభివృద్ధి ప్రక్రియను వేగవంతం చేయడానికి, మీ అప్లికేషన్ మరియు ఇన్‌ఫ్రాస్ట్రక్చర్ అవసరాలకు దగ్గరగా ఉండే సరైన _AZD Starter template_ కనుగొనడం మరియు అప్పుడు ఆ రెపోను మీ సన్నివేశ అవసరాలకు అనుగుణంగా అనుకూలీకరించడం చాల సులభం.

ప్రారంభించేముందు, మీరు Azure Developer CLI ఇన్స్టాల్ చేసుకున్నారా అని నిర్ధారించుకుందాం.

1. VS Code టెర్మినల్‌ను తెరిచి ఈ కమాండ్ టైప్ చేయండి:

      ```bash title="" linenums="0"
      azd version
      ```

1. మీరు ఈలాంటి ఫలితం చూడవచ్చు!

      ```bash title="" linenums="0"
      azd version 1.19.0 (commit b3d68cea969b2bfbaa7b7fa289424428edb93e97)
      ```

**మీరు ఇప్పుడు azd ఉపయోగించి ఒక టెంప్లేట్‌ను ఎంచుకుని డిప్లాయ్ చేయడానికి సిద్ధంగా ఉన్నారు**

## 2. టెంప్లేట్ ఎంపిక

Microsoft Foundry ప్లాట్‌ఫారమ్‌లో [set of recommended AZD templates](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) ఉన్నాయి, అవి _multi-agent workflow atomation_ మరియు _multi-modal content processing_ వంటి ప్రజాప్రియ పరిష్కార సన్నివేశాలను కవర చేస్తాయి. మీరు ఈ టెంప్లేట్స్‌ను Microsoft Foundry పోర్టల్‌లో సందర్శించి కూడా కనుగొనవచ్చు.

1. Visit [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. ప్రాంప్ట్ వచ్చినప్పుడు Microsoft Foundry పోర్టల్‌లో లాగిన్ చేయండి - మీరు ఈలా చూస్తారు.

![ఎంపిక](../../../../../translated_images/te/01-pick-template.60d2d5fff5ebc374.webp)


The **Basic** ఎంపికలు మీ స్టార్టర్ టెంప్లేట్స్:

1. [ ] [Get Started with AI Chat](https://github.com/Azure-Samples/get-started-with-ai-chat) ఇది మీ డేటాతో కూడిన ప్రాథమిక చాట్ అప్లికేషన్‌ని Azure Container Apps కు డిప్లాయ్ చేస్తుంది. ప్రాథమిక AI చాట్ బాట్ సన్నివేశాన్ని అన్వేషించడానికి దీన్ని ఉపయోగించండి.
1. [X] [Get Started with AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) ఇది కూడా Foundry Agents తో ఒక స్టాండర్డ్ AI ఏజెంట్ ను డిప్లాయ్ చేస్తుంది. టూల్స్ మరియు మోడల్స్‌ను ఉపయోగించే ఏజెంటిక్ AI పరిష్కారాలను పరిచయం చేసుకోవడానికి దీన్ని ఉపయోగించండి.

రెండవ లింక్‌ను కొత్త బ్రౌజర్ ట్యాబ్‌లో సందర్శించండి (లేదా సంబంధిత కార్డ్ కోసం `Open in GitHub` క్లిక్ చేయండి). మీరు ఈ AZD టెంప్లేట్ కోసం రెపోసిటరీని చూడగలుగుతారు. README ని ఒక నిమిషం పరిశీలించండి. అప్లికేషన్ ఆర్కిటెక్చర్ ఈ విధంగా ఉంటుంది:

![ఆర్కిటెక్చర్](../../../../../translated_images/te/architecture.8cec470ec15c65c7.webp)

## 3. టెంప్లేట్ యాక్టివేషన్

ఈ టెంప్లేట్‌ను డిప్లాయ్ చేసి అది సరైనదో లేదో నిర్ధారించుకుందాం. మనము [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) విభాగంలో ఉన్న మార్గదర్శకాలను అనుసరిస్తాము.

1. Click [this link](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) - `Create codespace` యొక్క డిఫాల్ట్ చర్యను అనుమతించండి
1. ఇది కొత్త బ్రౌజర్ ట్యాబ్ ఓపెన్ చేస్తుంది - GitHub Codespaces సెషన్ లోడ్ కావడాన్ని వేచి ఉండండి
1. Codespaces లో VS Code టెర్మినల్‌ను తెరవండి - ఈ కింద ఇవ్వబడిన ఆజ్ఞను టైప్ చేయండి:

   ```bash title="" linenums="0"
   azd up
   ```

ఈ ట్రిగ్గర్ చేసేవి వర్క్‌ఫ్లో స్టెప్స్‌ను పూర్తి చేయండి:

1. Azure లో లాగిన్ చేయమని మీకు ప్రాంప్ట్ వస్తుంది - ప్రామాణికరణ కోసం సూచనలను అనుసరించండి
1. మీకు ప్రత్యేకమైన environment పేరు ఇవ్వండి - ఉదాహరణకు నేను `nitya-mshack-azd` వాడాను
1. ఇది `.azure/` ఫోల్డర్‌ను సృష్టిస్తుంది - మీరు ఆ env పేరుతో ఉప ఫోల్డర్‌ను చూడగలుగుతారు
1. సబ్స్క్రిప్షన్ పేరు ఎంచుకోవాలని ప్రాంప్ట్ వస్తుంది - డిఫాల్ట్ ని ఎంచుకోండి
1. ప్రాంతం కోసం మీరు అడిగితే - `East US 2` ఉపయోగించండి

ఇప్పుడు, ప్రొవిజనింగ్ పూర్తి కావటానికి వేచి ఉండండి. **దీకి 10-15 నిమిషాలు పడవచ్చు**

1. పూర్తైనప్పుడు, మీ కన్సోల్‌లో SUCCESS సందేశం ఇలాగా కనిపిస్తుంది:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. మీ Azure పోర్టల్ ఇప్పుడు ఆ env పేరుతో ఒక ప్రొవిజనుడ్ రిసోర్స్ గ్రూప్ కలిగి ఉంటుంది:

      ![ఇన్‌ఫ్రా](../../../../../translated_images/te/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **మీరు ఇప్పుడు డిప్లాయ్ చేసిన ఇన్‌ఫ్రాస్ట్రక్చర్ మరియు అప్లికేషన్‌ను ధృవీకరించడానికి సిద్ధంగా ఉన్నారు**.

## 5.  ఏజెంట్ ధృవీకరణ

Azure Container App ఒక ఎండ్‌పాయింట్‌ను డిప్లాయ్ చేస్తుంది, అది ఈ టెంప్లేట్ కోసం Microsoft Foundry ప్రాజెక్టులో ప్రొవైజన్ చేయబడిన AI ఏజెంట్‌కు కనెక్ట్ అవుతుంది. ఇది ఏమిటీ అనేదాన్ని చూద్దాం.

1. మీ రిసోర్స్ గ్రూప్ కోసం Azure పోర్టల్ _Overview_ పేజీకి తిరిగిరా వెళ్ళండి

1. ఆ జాబితాలోని `Microsoft Foundry` రిసోర్స్‌పై క్లిక్ చేయండి

1. మీరు ఇది చూడగలుగుతారు. `Go to Microsoft Foundry Portal` బటన్‌పై క్లిక్ చేయండి.
   ![Foundry](../../../../../translated_images/te/04-view-foundry-project.fb94ca41803f28f3.webp)

1. మీ AI అప్లికేషన్ కోసం Foundry Project పేజీ కనిపిస్తుంది
   ![Project](../../../../../translated_images/te/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` పై క్లిక్ చేయండి - మీ ప్రాజెక్టులో ప్రొవైజన్ చేసిన డిఫాల్ట్ ఏజెంట్ కనిపిస్తుంది
   ![Agents](../../../../../translated_images/te/06-visit-agents.bccb263f77b00a09.webp)

1. దాన్ని సెలెక్ట్ చేయండి - మీరు ఏజెంట్ వివరాలను చూస్తారు. దయచేసి ఈ విషయాలను గమనించండి:

      - ఏజెంట్ డిఫాల్ట్‌గా File Search ను ఉపయోగిస్తుంది (ఎల్లప్పుడూ)
      - ఏజెంట్ యొక్క `Knowledge` 32 ఫైల్స్ అప్లోడ్ చేయబడ్డాయని సూచిస్తుంది (ఫైల్ సెర్చ్ కోసం)
      ![Agents](../../../../../translated_images/te/07-view-agent-details.0e049f37f61eae62.webp)

1. ఎడమ మెనులో `Data+indexes` ఎంపికను కనిపెడుతూ క్లిక్ చేయండి.

      - మీరు జ్ఞానానికి అప్లోడ్ చేసిన 32 డేటా ఫైళ్లు కనిపిస్తాయని చూడగలుగుతారు.
      - ఇవి `src/files`లోని 12 కస్టమర్ ఫైళ్ల మరియు 20 ఉత్పత్తి ఫైళ్లకు సమానంగా ఉంటాయి
      ![Data](../../../../../translated_images/te/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**మీరు ఏజెంట్ పనితీరు ధృవీకరించారు!**

1. ఏజెంట్ స్పందనలు ఆ ఫైళ్లో ఉన్న జ్ఞానంపై ఆధారపడి ఉంటాయి.
1. మీరు ఇప్పుడు ఆ డేటాతో సంబంధిత ప్రశ్నలు అడిగి, ఆధారమైన స్పందనలు పొందగలరు.
1. ఉదాహరణ: `customer_info_10.json`లో "Amanda Perez" చేసిన 3 కొనుగోళ్లు వివరించబడ్డాయి

Container App ఎండ్‌పాయింట్ ఉన్న బ్రౌజర్ ట్యాబ్‌కు తిరిగి వెళ్లి అడగండి: `What products does Amanda Perez own?`. మీకు ఇలాంటివి కనిపించాలి:

## 6. ఏజెంట్ ప్లేగ్రౌండ్

Microsoft Foundry యొక్క సామర్థ్యాలపై కొంత అవగాహన ఏర్పరచుకోవడానికి, ఏజెంట్‌ను Agents Playground లో పరీక్షిద్దాం.

1. Microsoft Foundry లోని `Agents` పేజీకి తిరిగి వెళ్ళండి - డిఫాల్ట్ ఏజెంట్‌ను సెలెక్ట్ చేయండి
1. `Try in Playground` ఎంపికపై క్లిక్ చేయండి - మీరు ఈ దానికి తాలూకు Playground UI పొందగలుగుతారు
1. అదే ప్రశ్న అడగండి: `What products does Amanda Perez own?`

    ![డేటా](../../../../../translated_images/te/09-ask-in-playground.a1b93794f78fa676.webp)

మీకు అదే (లేదా సమాన) ప్రత్యుత్తరం వస్తుంది - కానీ మీరు మీ ఏజెంటిక్ యాప్ యొక్క నాణ్యత, ఖర్చు మరియు పనితీరును అర్థం చేసుకోవడానికిగాను అదనపు సమాచారం కూడా పొందుతారు. ఉదాహరణకి:

1. ప్రత్యుత్తరం ఏ ఫైల్‌లను "గ్రమించడానికి" ఉపయోగించిందో సూచించబడింది
1. ఈ ఫైల్ లేబుల్స్ పై హోవర్ చేయండి - డేటా మీ ప్రశ్నకు మరియు చూపించిన ప్రత్యుత్తరంతో సరిపోతుందా అని పరిశీలించండి

కూడా ప్రత్యుత్తరానికి క్రింద ఒక _stats_ వరుస కనిపిస్తుంది.

1. ఏ మెట్రిక్‌పై అయితే హోవర్ చేయండి - ఉదాహరణకు, Safety. మీరు ఇలాంటి ఒక చూపును చూస్తారు
1. అంచనా వేయబడిన రేటింగ్ మీ భావం ప్రకారం ప్రత్యుత్తరం సురక్షిత స్థాయికి సరిపోతుందా?

## 7. బిల్ట్-ఇన్ ఆబ్జర్వబిలిటీ

Observability అంటే మీ అప్లికేషన్‌ను ఇన్స్ట్రుమెంట్ చేసి, దాని కార్యకలాపాలను అర్థం చేసుకోవడానికి, డీబగ్ చేయడానికి మరియు ఆప్టిమైజ్ చేయడానికి ఉపయోగపడే డేటా ఉత్పత్తి చేయడం గురించి. దీని యొక్క భావనను పొందడానికి:

1. `View Run Info` బటన్‌పై క్లిక్ చేయండి - మీరు ఈ వీక్షణను చూడగలరు. ఇది [Agent tracing](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) చర్యలో ఒక ఉదాహరణ. _మీరు ఈ వీక్షణను టాప్-లెవల్ మెనูలో Thread Logs క్లిక్ చేయడం ద్వారా కూడా పొందవచ్చు_.

   - ఏజెంట్ నిర్వహించిన రన్ స్టెప్స్ మరియు పాల్గొన్న టూల్స్‌ను గ్రహించండి
   - ప్రత్యుత్తరానికి ఉపయోగించిన మొత్తం Token లెక్క (vs. output tokens usage)ను అర్థం చేసుకోండి
   - లేటెన్సీని మరియు అమలు సమయంలో సమయం ఎక్కడ ఖర్చవుతున్నదో అర్థం చేసుకోండి

      ![ఏజెంట్](../../../../../translated_images/te/10-view-run-info.b20ebd75fef6a1cc.webp)

1. Debugging సమస్యలకు ఉపయోగకరమైన అదనపు సందర్భాన్ని అందించగల రన్‌కు సంబంధించిన గుణల కోసం `Metadata` ట్యాబ్‌పై క్లిక్ చేయండి.   

      ![ఏజెంట్](../../../../../translated_images/te/11-view-run-info-metadata.7966986122c7c2df.webp)


1. ఏజెంట్ ప్రత్యుత్తరంపై స్వయంచాలకంగా చేసిన అంచనాల కోసం `Evaluations` ట్యాబ్‌ను క్లిక్ చేయండి. వీటిలో సేఫ్టీ అంచనాలు (ఉదా., Self-harm) మరియు ఏజెంట్-సంకేత పరమైన అంచనాలు (ఉదా., ఇరుద్దేశం పరిష్కారం, టాస్క్ అనుచితత్వం) ఉన్నాయి.

      ![ఏజెంట్](../../../../../translated_images/te/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. చివరగా, సైడ్బార్ మెనూలోని `Monitoring` ట్యాబ్‌పై క్లిక్ చేయండి.

      - ప్రదర్శిత పేజీలో `Resource usage` ట్యాబ్‌ను ఎంచుకొని మెట్రిక్‌లను వీక్షించండి.
      - ఖర్జీలు (token లు) మరియు లోడ్ (రీక్వెస్టులు) పరంగా అప్లికేషన్ వినియోగాన్ని ట్రాక్ చేయండి.
      - ఫస్ట్ బైట్ (ఇన్పుట్ ప్రాసెసింగ్) మరియు లాస్ట్ బైట్ (ఆుట్పుట్) వరకు అప్లికేషన్ లేటెన్సీని ట్రాక్ చేయండి.

      ![ఏజెంట్](../../../../../translated_images/te/13-monitoring-resources.5148015f7311807f.webp)

## 8. ఎన్విరాన్‌మెంట్ వేరియబుల్స్

ఇప్పటి వరకు, మేము బ్రౌజర్‌లో డిప్లాయ్‌మెంట్ ద్వారా నడిచాము - మరియు మా ఇన్‌ఫ్రాస్ట్రక్చర్ ప్రొవిజన్డ్ అయి అప్లికేషన్ కార్యాచరణలో ఉందో లేదో ధృవీకరించాము. కానీ కోడ్-ఫస్ట్ విధానంతో పని చేయడానికి, ఆ రిసోర్సులతో పని చేయడానికి అవసరమైన సంబంధిత వేరియబుల్స్‌తో మా లోకల్ డెవలప్మెంట్ ఎన్విరాన్‌మెంట్‌ను కాన్ఫిగర్ చేయాలి. `azd` ఉపయోగించడం దీనిని సులభంగా చేస్తుంది.

1. Azure Developer CLI [uses environment variables](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) అప్లికేషన్ డిప్లాయ్‌మెంట్ కోసం కాన్ఫిగరేషన్ సెట్టింగ్స్ నిల్వ చేయడానికి మరియు నిర్వహించడానికి.

1. ఎన్విరాన్‌మెంట్ వేరియబుల్స్ `.azure/<env-name>/.env` లో నిల్వ చేయబడతాయి - ఇవి deployment సమయంలో ఉపయోగించిన `env-name`కు స్కోప్ చేస్తాయి మరియు అదే రెపోలోని వేర్వేరు డిప్లాయ్‌మెంట్ లక్ష్యాల మధ్య ఎన్విరాన్‌మెంట్లను విడుపు చేయడంలో మీకు సహాయం చేస్తాయి.

1. ఎన్విరాన్‌మెంట్ వేరియబుల్స్ `azd` ఆజ్ఞ ప్రతి సారికి ప్రత్యేక ఆజ్ఞ అమలయ్యేటప్పుడు ఆటోమేటిగ్గా లోడ్ అవుతాయి (ఉదా., `azd up`). గమనించండి `azd` ఆటోమేటిగ్గా OS-స్థాయి ఎన్విరాన్‌మెంట్ వేరియబుల్స్ (ఉదా., షెల్‌లో సెట్ చేసినవి)ను చదవదు - బదులుగా స్క్రిప్టుల్లో సమాచారాన్ని బదిలీ చేయడానికి `azd set env` మరియు `azd get env` ఉపయోగించండి.


కొన్ని కమాండులను ప్రయత్నిద్దాం:

1. ఈ ఎన్విరాన్‌మెంట్‌లో `azd` కోసం సెట్ చేసిన అన్ని ఎన్విరాన్‌మెంట్ వేరియబుల్స్ పొందండి:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      మీరు ఇలాభ కనిపించేలా ఉంటుంది:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. ఒక నిర్దిష్ట విలువను పొందండి - ఉదా., నేను `AZURE_AI_AGENT_MODEL_NAME` విలువను సెట్ చేసామో లేదో తెలుసుకోవాలనుకుంటున్నాను

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      మీరు ఇలాగే చూడగలుగుతారు - ఇది డిఫాల్ట్‌గా సెట్ చేయబడలేదు!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd` కోసం కొత్త ఎన్విరాన్‌మెంట్ వేరియబుల్‌ను సెట్ చేయండి. ఇక్కడ, మనం ఏజెంట్ మోడల్ పేరును నవీకరిస్తున్నాము. _గమనిక: చేసిన ఏ మార్పులైనా వెంటనే `.azure/<env-name>/.env` ఫైలులో ప్రతిబింబిస్తాయి._

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      ఇప్పుడు, మనం ఆ విలువ సెట్ అయినట్లు కనుగొంటాము:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. గమనించండి కొన్ని రిసోర్సులు స్థిరంగా ఉంటాయి (ఉదా., మోడల్ డిప్లాయ్‌మెంట్లు) మరియు వాటిని బలవంతంగా పునరుద్ధరించడానికి కేవలం `azd up` కంటే ఎక్కువ అవసరం ఉంటుంది. మొదటి డిప్లాయ్‌మెంట్‌ను తీసివేసి మార్చిన ఎన్విరాన్ వేరియబుల్స్‌తో మళ్లీ డిప్లాయ్ చేయడం ప్రయత్నిద్దాం.

1. **Refresh** మీరు గతంలో azd టెంప్లేట్ ఉపయోగించి ఇన్‌ఫ్రాస్ట్రక్చర్‌ను డిప్లాయ్ చేసుంటే - ఈ కమాండ్ ఉపయోగించి మీ లోకల్ ఎన్విరాన్‌మెంట్ వేరియబుల్స్ యొక్క స్థితిని Azure డిప్లాయ్‌మెంట్ యొక్క ప్రస్తుత స్థితిపై ఆధారపడి రిఫ్రెష్ చేయవచ్చు:

      ```bash title="" linenums="0"
      azd env refresh
      ```

      ఈ విధానం రెండు లేదా అంతకంటే ఎక్కువ స్థానిక అభివృద్ధి పరిసరాల (ఉదాహరణకు, బహుళ డెవలపర్లు ఉన్న టీమ్) మధ్య పర్యావరణ వేరియబుల్స్‌ను _సమకాలీకరించడానికి_ ఒక శక్తివంతమైన మార్గం - అమర్చబడిన ఇన్‌ఫ్రాస్ట్రక్చర్‌ను పర్యావరణ వేరియబుల్స్ స్థితికి ప్రధాన వాస్తవంగా సేవ చేయడానికి అనుమతిస్తుంది. టీమ్ సభ్యులు సులభంగా వేరియబుల్స్‌ను _తాజా చేస్తూ_ తిరిగి సమన్వయానికి చేరుకుంటారు.
