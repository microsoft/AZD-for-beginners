## పరిచయం

ఈ సమగ్ర గ్లోసరీ Azure Developer CLI మరియు Azure క్లౌడ్ డెవలప్‌మెంట్‌లో ఉపయోగించే పదాలు, భావాలు, సంక్షేపాల యొక్క నిర్వచనాలను అందిస్తుంది. టెక్నికల్ డాక్యుమెంటేషన్‌ని అర్థం చేసుకోవడానికి, సమస్యలను పరిష్కరించడానికి మరియు azd ప్రాజెక్ట్స్ మరియు Azure సర్వీసుల గురించి సమర్థవంతంగా కమ్యూనికేట్ చేయడానికి ఇది అవసరమైన సూచిక.

## నేర్చుకునే లక్ష్యాలు

ఈ గ్లోసరీని ఉపయోగించడం ద్వారా మీరు:
- Azure Developer CLI యొక్క ముఖ్య పదజాలం మరియు థింsల్ని అర్థం చేసుకుంటారు
- Azure క్లౌడ్ డెవలప్‌మెంట్ పదాల మరియు సాంకేతిక పదాలను నేర్పుకుంటారు
- Infrastructure as Code మరియు వినియోగ నియమాలకి సంబంధించిన పదజాలాన్ని సమర్ధవంతంగా ప్రాసం చేస్తారు
- Azure సేవల పేర్లు, సంక్షేపాలు మరియు వాటి ఉద్దేశ్యాలను అర్థం చేసుకుంటారు
- ట్రబుల్‌షూటింగ్ మరియు డీబగింగ్ సంబంధిత పదజాలానికి నిర్వచనాలను పొందుతారు
- అధునాతన Azure ఆర్కిటెక్చర్ మరియు డెవలప్‌మెంట్ భావాలపై నేర్చుకుంటారు

## నేర్చుకున్న ఫలితాలు

ఈ గ్లోసరీని తరచుగా సూచించినట్లయితే, మీరు చేయగలుగుతారు:
- సరైన Azure Developer CLI పదజాలాన్ని ఉపయోగించి సమర్థవంతంగా కమ్యూనికేట్ చేయడం
- టెక్నికల్ డాక్యుమెంటేషన్ మరియు ఎర్రర్ సంకేతాలను స్పష్టంగా అర్థం చేసుకోవడం
- Azure సేవలు మరియు భావాలను ఆత్మవిశ్వసంతో నావిగేట్ చేయడం
- తగిన సాంకేతిక పదజాలంతో సమస్యలను పరిష్కరించడం
- జట్టు చర్చలలో ఖచ్చితమైన సాంకేతిక భాషతో భాగస్వామ్యం చేయడం
- మీ Azure క్లౌడ్ డెవలప్‌మెంట్ జ్ఞానాన్ని వ్యవస్థగ్రాహ్యంగా విస్తరించుకోవడం

## A

**ARM Template**  
Azure Resource Manager template. JSON-ఆధారిత Infrastructure as Code ఫార్మాట్, Azure రీసోర్సులను ప్రకటనాత్మకంగా నిర్వచించి నిర్వాహణ చేయడానికి ఉపయోగిస్తారు.

**App Service**  
Azure యొక్క platform-as-a-service (PaaS) ఆಫరింగ్; ఇన్ఫ్రాస్ట్రక్చర్‌ని నిర్వహించకుండా వెబ్ అప్లికేషన్లు, REST APIs మరియు మొబైల్ బ్యాకెండ్స్‌ను హోస్ట్ చేయడానికి ఉపయోగిస్తారు.

**Application Insights**  
అప్లికేషన్ పనితీరుపై లోతైన అవగాహన, అందుబాటు మరియు వినియోగాన్ని అందించే Azure యొక్క అప్లికేషన్ పెర్ఫార్మెన్స్ మానిటరింగ్ (APM) సేవ.

**Azure CLI**  
Azure రీసోర్సులను నిర్వహించడానికి కమాండ్-లైన్ ఇంటర్ఫేస్. పాఠ్యంగా azd ద్వారా authentication మరియు కొన్ని ఆపరేషన్లలో ఉపయోగిస్తారు.

**Azure Developer CLI (azd)**  
డెవలపర్-కేంద్రీకృత కమాండ్-లైన్ టూల్, టెంప్లేట్స్ మరియు Infrastructure as Code ఉపయోగించి Azureకు అప్లికేషన్లు రూపొందించడం మరియు పంపిణీ ప్రక్రియను వేగవంతం చేస్తుంది.

**azure.yaml**  
azd ప్రాజెక్ట్ కోసం మెయిన్ కాన్ఫిగరేషన్ ఫైల్, ఇది సర్వీసులు, ఇన్ఫ్రాస్ట్రక్చర్, మరియు డిప్లాయ్‌మెంట్ హుక్స్‌ను నిర్వచిస్తుంది.

**Azure Resource Manager (ARM)**  
Azure యొక్క డిప్లాయ్‌మెంట్ మరియు మేనేజ్‌మెంట్ సేవ, ఇది రీసోర్సులను సృష్టించడం, నవీకరించడం మరియు తొలగించడం కోసం మేనేజ్మెంట్ లేయర్‌ను అందిస్తుంది.

## B

**Bicep**  
Microsoft అభివృద్ధి చేసిన domain-specific language (DSL) Azure రీసోర్సులను డిప్లాయ్ చేయడానికి. ARM టెంప్లేట్‌ల కంటే సింపుల్ సింటాక్స్‌ను అందిస్తుంది మరియు ARM కి కంపైల్ అవుతుంది.

**Build**  
సోర్స్ కోడ్‌ను కంపైల్ చేయడం, డిపెండెన్సీలను ఇన్‌స్టాల్ చేయడం మరియు అప్లికేషన్లను డిప్లాయ్‌మెంట్ కోసం సిద్ధం చేసే ప్రక్రియ.

**Blue-Green Deployment**  
డౌన్‌టైమ్ మరియు రిస్క్‌ను తగ్గించడానికి రెండు సమాన ప్రొడక్షన్ పరిసరాలను (blue మరియు green) ఉపయోగించే డిప్లాయ్‌మెంట్ వ్యూహం.

## C

**Container Apps**  
సర్వర్‌లెస్ కంటైనర్ సేవ, సంక్లిష్ట ఇన్ఫ్రాస్ట్రక్చర్‌ని నిర్వహించకుండా కంటైనరైజ్డ్ అప్లికేషన్లను నడపడానికి Azure అందించే సేవ.

**CI/CD**  
Continuous Integration/Continuous Deployment. కోడ్ మార్పులను సమీకరించడం మరియు అప్లికేషన్లను ఆటోమేటెడ్‌గా డిప్లాయ్ చేయడానికి ఉపయోగించే ప్రక్రియలు.

**Cosmos DB**  
Azure యొక్క గ్లోబ్లీ పంపిణీ చేయబడిన, మల్టీ-మోడల్ డేటాబేస్ సేవ, throughput, latency, availability మరియు consistency కోసం సమగ్ర SLAలను అందిస్తుంది.

**Configuration**  
అప్లికేషన్ ప్రవర్తన మరియు డిప్లాయ్‌మెంట్ ఆప్షన్లను నియంత్రించే సెట్టింగులు మరియు పారామీటర్లు.

## D

**Deployment**  
లక్ష్యఇన్ఫ్రాస్ట్రక్చర్‌పై అప్లికేషన్లు మరియు వాటి డిపెండెన్సీలను ఇన్‌స్టాల్ చేయడం మరియు కాన్ఫిగర్ చేయడం.

**Docker**  
కంటైనరీజేషన్ టెక్నాలజీ ఉపయోగించి అప్లికేషన్లను డెవలప్ చేయడం, షిప్ చేయడం మరియు రన్ చేయడానికి ఉపయోగించే ప్లాట్‌ఫారమ్.

**Dockerfile**  
Docker కంటైనర్ ఇమేజ్ నిర్మించడానికి సూచనలు కలిగిన టెక్స్ట్ ఫైల్.

## E

**Environment**  
మీ అప్లికేషన్ యొక్క నిర్దిష్ట ఉదాహరణను సూచించే డిప్లాయ్‌మెంట్ టార్గెట్ (ఉదా., development, staging, production).

**Environment Variables**  
రన్‌టైమ్‌లో అప్లికేషన్లు యాక్సెస్ చేయగల కీల-విలువ జంటలుగా నిల్వ చేసిన కాన్ఫిగరేషన్ విలువలు.

**Endpoint**  
అప్లికేషన్ లేదా సేవను యాక్సెస్ చేయగల URL లేదా నెట్‌వర్క్ చిరునామా.

## F

**Function App**  
ఈవెంట్-డ్రివన్ కోడ్ నడపడానికి ఇన్ఫ్రాస్ట్రక్చర్‌ని నిర్వహించకుండా Azure అందించే సర్వర్‌లెస్ కంప్యూట్ సేవ.

## G

**GitHub Actions**  
GitHub రిపోజిటరీలతో ఒకीकృత CI/CD ప్లాట్‌ఫారమ్, వర్క్‌ఫ్లోలను ఆటోమేట్ చేయడానికి ఉపయోగిస్తారు.

**Git**  
సోర్స్ కోడ్‌లో మార్పులను ట్రాక్ చేయడానికి ఉపయోగించే డిస్ట్రిబ్యూటెడ్ వెర్షన్ కంట్రోల్ సిస్టమ్.

## H

**Hooks**  
డిప్లాయ్‌మెంట్ లైఫ్‌సైకల్ నిర్దిష్ట సమయంలో (preprovision, postprovision, predeploy, postdeploy) రన్ అయ్యే కస్టమ్ స్క్రిప్టులు లేదా కమాండ్లు.

**Host**  
పాఠం ఉంది: అప్లికేషన్ ఎక్కడ డిప్లాయ్ చేయబడుతుందో సూచించే Azure సేవ యొక్క రకం (ఉదా., appservice, containerapp, function).

## I

**Infrastructure as Code (IaC)**  
ఇన్ఫ్రాస్ట్రక్చర్‌ను మాన్యువల్ ప్రక్రియల కంటే కోడ్ ద్వారా నిర్వచించి నిర్వహించే అభ్యాసం.

**Init**  
సాధారణంగా టెంప్లేట్ నుండి కొత్త azd ప్రాజెక్ట్‌ను ఇనిషియలైజ్ చేసే ప్రక్రియ.

## J

**JSON**  
JavaScript Object Notation. కాన్ఫిగరేషన్ ఫైళ్ల మరియు API రెస్పాన్సులకు సాధారణంగా ఉపయోగించే డేటా ఇంటర్‌చేంజ్ ఫార్మాట్.

**JWT**  
JSON Web Token. పక్షాల మధ్య సమాచారాన్ని ఒక JSON ఆబ్జెక్ట్‌గా సురక్షితంగా పంపడానికి ఉపయోగించే స్టాండర్డ్.

## K

**Key Vault**  
సీక్రెట్స్, కీలు, సర్టిఫికెట్లను సురక్షితంగా నిల్వ చేయడానికి Azure అందించే సేవ.

**Kusto Query Language (KQL)**  
Azure Monitor, Application Insights మరియు ఇతర Azure సేవలలో డేటాను విశ్లేషించడానికి ఉపయోగించే క్వెరీ భాష.

## L

**Load Balancer**  
వెళ్లే నెట్‌వర్క్ ట్రాఫిక్‌ను బహుళ సర్వర్‌లు లేదా ఇన్స్టాన్సులపై పంపిణీ చేసే సేవ.

**Log Analytics**  
క్లౌడ్ మరియు ఆన్-ప్రేం పరిసరాల నుంచి టెలిమెట్రీ డేటాను సేకరించడం, విశ్లేషించడం మరియు చర్యలు తీసుకోవడం కోసం Azure సేవ.

## M

**Managed Identity**  
ఇతర Azure సేవలకు authenticate కావడానికి Azure సేవలకు స్వయం-నియంత్రిత ఐడెంటిటీని అందించే Azure ఫీచర్.

**Microservices**  
అప్లికేషన్లను చిన్న, స్వతంత్ర సేవల సేకరణగా నిర్మించే ఆర్కిటెక్చర్ విధానం.

**Monitor**  
అప్లికేషన్లు మరియు ఇన్ఫ్రాస్ట్రక్చర్ మీద ఫుల్-స్టాక్ ఆబ్జర్వబిలిటిని అందించే Azure యొక్క యూనిఫైడ్ మానిటరింగ్ సొల్యూషన్.

## N

**Node.js**  
Chrome యొక్క V8 JavaScript ఇంజిన్ అందించిన JavaScript రన్‌టైమ్, సర్వర్-సైడ్ అప్లికేషన్ల కోసం.

**npm**  
Node.js కోసం ప్యాకేజ్ మేనేజర్, డిపెండెన్సీలు మరియు ప్యాకేజీలను నిర్వహిస్తుంది.

## O

**Output**  
ఇన్ఫ్రాస్ట్రక్చర్ డిప్లాయ్‌మెంట్ నుండి తిరిగి అందే విలువలు, ఇవి అప్లికేషన్లు లేదా ఇతర రీసోర్సులు ఉపయోగించగలవు.

## P

**Package**  
అప్లికేషన్ కోడ్ మరియు డిపెండెన్సీలను డిప్లాయ్‌మెంట్ కోసం సిద్ధం చేసే ప్రక్రియ.

**Parameters**  
డిప్లాయ్‌మెంట్‌లను అనుకూలీకరించడానికి ఇన్పుట్ విలువలు, ఇన్ఫ్రాస్ట్రక్చర్ టెంప్లేట్స్‌కు передаются.

**PostgreSQL**  
ఓపెన్-సୋర్స్ రిలేషనల్ డేటాబేస్ సిస్టమ్, Azureలో మేనేజ్‌డ్ సేవగా మద్దతు పొందుతుంది.

**Provisioning**  
ఇన్ఫ్రాస్ట్రక్చర్ టెంప్లేట్‌లలో నిర్వచించబడిన Azure రీసోర్సులను సృష్టించడం మరియు కాన్ఫిగర్ చేయడం.

## Q

**Quota**  
Azure సబ్‌స్క్రిప్షన్ లేదా రీజియన్‌లో సృష్టించగల రీసోర్సుల పరిమితులు.

## R

**Resource Group**  
అనువర్తించే లైఫ్‌సైకిల్, అనుమతులు మరియు పాలసీలను పంచుకోగల Azure రీసోర్సుల లాజికల్ కంటైనర్.

**Resource Token**  
డిప్లాయ్‌మెంట్లలో రీసోర్స్ పేర్లు ప్రత్యేకంగా ఉండేలా చేయడానికి azd రూపొందించే యూనిక్ స్ట్రింగ్.

**REST API**  
HTTP పద్ధతులు ఉపయోగించి నెట్‌వర్క్ చేయబడిన అప్లికేషన్లను డిజైన్ చేసేందుకైన ఆర్కిటెక్చరల్ స్టైల్.

**Rollback**  
అప్లికేషన్ లేదా ఇన్ఫ్రాస్ట్రక్చర్ కాన్ఫిగరేషన్ యొక్క గత వెర్షన్‌కు తిరిగి వెళ్తే ప్రక్రియ.

## S

**Service**  
azure.yamlలో నిర్వచించబడిన మీ అప్లికేషన్ యొక్క ఒక భాగం (ఉదా., web frontend, API backend, database).

**SKU**  
Stock Keeping Unit. Azure రీసోర్సుల వివిధ సేవా స్థాయి లేదా పన సామర్థ్యాలను సూచిస్తుంది.

**SQL Database**  
Microsoft SQL Server ఆధారంగా Azure అందించే మేనేజ్ చేయబడే రిలేషనల్ డేటాబేస్ సేవ.

**Static Web Apps**  
స్టాటిక్ వెబ్ అప్లికేషన్లు సోర్స్ కోడ్ రిపోజిటరీల నుండి నిర్మించి మరియు డిప్లాయ్ చేయడానికి Azure అందించేసేవ.

**Storage Account**  
బ్లాబ్‌లు, ఫైళ్లు, క్యూలు మరియు పట్టికలు వంటి డేటా ఆబ్జెక్టుల కోసం క్లౌడ్ స్టోరేజ్‌ను అందించే Azure సేవ.

**Subscription**  
రిసోర్స్ గ్రూపులు మరియు రీసోర్సులను కలిగిన Azure ఖాతా కంటెయినర్, బిల్లింగ్ మరియు యాక్సెస్ మేనేజ్‌మెంట్‌కు సంబంధించినది.

## T

**Template**  
సాధారణ పరిస్థితుల కోసం అప్లికేషన్ కోడ్, ఇన్ఫ్రాస్ట్రక్చర్ నిర్వచనాలు మరియు కాన్ఫిగరేషన్ కలిగిన ముందే తయారైన ప్రాజెక్ట్ నిర్మాణం.

**Terraform**  
బహుళ క్లౌడ్ ప్రొవైడర్లను సహాయపడే ఓపెన్-సోర్స్ Infrastructure as Code టూల్, Azure మద్దతు చేస్తుంది.

**Traffic Manager**  
గ్లోబల్ Azure రీజియన్ల మధ్య ట్రాఫిక్‌ను పంపిణీ చేయడానికి DNS-ఆధారిత ట్రాఫిక్ లోడ్ బాలాన్సర్.

## U

**URI**  
Uniform Resource Identifier. నిర్దిష్ట రీసో스를 గుర్తించే స్ట్రింగ్.

**URL**  
Uniform Resource Locator. ఒక రీసోర్స్ ఎక్కడ ఉన్నదో మరియు దాన్ని ఎలా பெறాలో పేర్కొనే URI రకం.

## V

**Virtual Network (VNet)**  
Azureలో ప్రైవేట్ నెట్‌వర్క్స్ కోసం మూల భవనం, ఇన్ఫ్రాస్ట్రక్చర్‌కి ఐసొలేషన్ మరియు సెగ్మెంటేషన్‌ను అందిస్తుంది.

**VS Code**  
Visual Studio Code. Azure మరియు azd ఇంటిగ్రేషన్‌తో ప్రసిద్ధ కోడ్ ఎడిటర్.

## W

**Webhook**  
నిర్దిష్ట ఈవెంట్స్ ద్వారా ట్రిగర్ అయ్యే HTTP కాల్‌బ్యాక్, సాధారణంగా CI/CD పైప్లైన్లలో ఉపయోగిస్తారు.

**What-if**  
డిప్లాయ్‌మెంట్ ఇందుకు సృష్టించే మార్పులను ఆచరణలోకి తీసుకురావకుండా చూపించే Azure ఫీచర్.

## Y

**YAML**  
YAML Ain't Markup Language. azure.yaml వంటి కాన్ఫిగరేషన్ ఫైళ్లకు ఉపయోగించే మానవ-రిక్త డేటా సీరియలైజేషన్ స్టాండర్డ్.

## Z

**Zone**  
Azure రీజియన్‌లో భౌతికంగా వేరుచేసిన ప్రదేశాలు, అవి redundancy మరియు yüksək availabilityని అందిస్తాయి.

## Context-Specific Terms

### Development Terms
- **Hot Reload**: అభివృద్ధి సమయంలో రీస్టార్ట్ చేయకుండా ఆటోమేటిక్‌గా అప్లికేషన్లను అప్‌డేట్ చేయడం
- **Build Pipeline**: కోడ్‌ను నిర్మించడం మరియు పరీక్షించడం కోసం ఆటోమేటెడ్ ప్రక్రియ
- **Deployment Slot**: App Service లోని స్టేజింగ్ పరిసరము
- **Environment Parity**: development, staging మరియు production పరిసరాలను సమానంగా ఉంచడం

### Security Terms
- **Managed Identity**: ఆటోమాటిక్ క్రెడెన్షియల్ నిర్వహణని అందించే Azure ఫీచర్
- **Key Vault**: సీక్రెట్స్, కీలు మరియు సర్టిఫికెట్లకు సురక్షిత భద్రతా నిల్వ
- **RBAC**: Azure రీసోర్సుల కోసం పాత్ర ఆధారిత యాక్సెస్ నియంత్రణ
- **Network Security Group**: నెట్‌వర్క్ ట్రాఫిక్‌ను నియంత్రించే వర్చువల్ ఫైర్వాల్

### Monitoring Terms
- **Telemetry**: కొలువులు మరియు డేటా యొక్క ఆటోమేటెడ్ సేకరణ
- **Application Performance Monitoring (APM)**: సాఫ్ట్‌వేర్ పనితీరును మానిటర్ చేయడం
- **Log Analytics**: లాగ్ డేటాను సేకరించి విశ్లేషించడానికి సేవ
- **Alert Rules**: మెట్రిక్‌లు లేదా షరతుల ఆధారంగా ఆటోమేటెడ్ నోటిఫికేషన్లు

### Deployment Terms
- **Blue-Green Deployment**: జీరో-డౌన్‌టైమ్ డిప్లాయ్‌మెంట్ వ్యూహం
- **Canary Deployment**: వినియోగదారుల ఉపసమూహానికి تدريజు rollout చేయడం
- **Rolling Update**: అనువర్తన ఇన్స్టాన్సుల యొక్క వరసగా బదిలీ
- **Rollback**: గత అప్లికేషన్ వెర్షన్‌కు తిరిగిిచ్చే ప్రక్రియ
