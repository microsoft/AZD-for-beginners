#### üí∞ Considerazioni sui costi per i deployment AI

**Ambiente di sviluppo (stimato $80-150/month):**
- Azure OpenAI (Pay-as-you-go): $0-50/month (based on token usage)
- AI Search (Basic tier): $75/month
- Container Apps (Consumption): $0-20/month
- Storage (Standard): $1-5/month

**Ambiente di produzione (stimato $300-3,500+/month):**
- Azure OpenAI (PTU for consistent performance): $3,000+/month OR Pay-as-go with high volume
- AI Search (Standard tier): $250/month
- Container Apps (Dedicated): $50-100/month
- Application Insights: $5-50/month
- Storage (Premium): $10-50/month

**üí° Suggerimenti per l'ottimizzazione dei costi:**
- Usa il **Free Tier** di Azure OpenAI per imparare (50,000 token/mese inclusi)
- Esegui `azd down` per deallocare le risorse quando non stai sviluppando attivamente
- Inizia con fatturazione a consumo, passa a PTU solo per produzione
- Usa `azd provision --preview` per stimare i costi prima del deployment
- Abilita l'auto-scaling: paga solo per l'utilizzo effettivo

**Monitoraggio dei costi:**
```bash
# Controlla i costi mensili stimati

# Monitora i costi effettivi nel Portale di Azure
az consumption budget list --resource-group <your-rg>
```

## üîß Guida rapida alla risoluzione dei problemi

**Problemi comuni che i principianti affrontano e soluzioni immediate:**

### ‚ùå "azd: command not found"

```bash
# Installa prima AZD
# Windows (PowerShell):
winget install microsoft.azd

# macOS:
brew tap azure/azd && brew install azd

# Linux:
curl -fsSL https://aka.ms/install-azd.sh | bash

# Verifica l'installazione
azd version
```

### ‚ùå "No subscription found" or "Subscription not set"

```bash
# Elenca le sottoscrizioni disponibili
az account list --output table

# Imposta la sottoscrizione predefinita
az account set --subscription "<subscription-id-or-name>"

# Imposta per l'ambiente AZD
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# Verifica
az account show
```

### ‚ùå "InsufficientQuota" or "Quota exceeded"

```bash
# Prova una regione Azure diversa
azd env set AZURE_LOCATION "westus2"
azd up

# Oppure usa SKU pi√π piccoli in sviluppo
# Modifica infra/main.parameters.json:
{
  "sku": "B1"  // Instead of "P1V2"
}
```

### ‚ùå "azd up" fails halfway through

```bash
# Opzione 1: Pulisci e riprova
azd down --force --purge
azd up

# Opzione 2: Ripara solo l'infrastruttura
azd provision

# Opzione 3: Controlla i log dettagliati
azd show
azd logs
```

### ‚ùå "Authentication failed" or "Token expired"

```bash
# Riautenticarsi
az logout
az login

azd auth logout
azd auth login

# Verificare l'autenticazione
az account show
```

### ‚ùå "Resource already exists" or naming conflicts

```bash
# AZD genera nomi unici, ma in caso di conflitto:
azd down --force --purge

# Quindi riprova con un ambiente nuovo
azd env new dev-v2
azd up
```

### ‚ùå Template deployment taking too long

**Tempi di attesa normali:**
- Semplice web app: 5-10 minuti
- App con database: 10-15 minuti
- Applicazioni AI: 15-25 minuti (il provisioning di OpenAI √® lento)

```bash
# Controlla lo stato di avanzamento
azd show

# Se bloccato da pi√π di 30 minuti, controlla il Portale di Azure:
azd monitor
# Cerca distribuzioni non riuscite
```

### ‚ùå "Permission denied" or "Forbidden"

```bash
# Controlla il tuo ruolo Azure
az role assignment list --assignee $(az account show --query user.name -o tsv)

# Hai bisogno almeno del ruolo "Contributor"
# Chiedi al tuo amministratore Azure di concedere:
# - Contributor (per le risorse)
# - User Access Administrator (per le assegnazioni di ruolo)
```

### ‚ùå Can't find deployed application URL

```bash
# Mostra tutti gli endpoint di servizio
azd show

# Oppure apri il Portale di Azure
azd monitor

# Controlla un servizio specifico
azd env get-values
# Cerca le variabili *_URL
```
