# Changelog - AZD Para sa Mga Baguhan

## Panimula

Ang changelog na ito ay nagdodokumento ng lahat ng mahahalagang pagbabago, update, at pagpapabuti sa AZD Para sa Mga Baguhan repository. Sinusunod namin ang mga prinsipyo ng semantic versioning at pinapanatili ang log na ito upang matulungan ang mga user na maunawaan ang mga pagbabago sa pagitan ng mga bersyon.

## Mga Layunin sa Pag-aaral

Sa pag-review ng changelog na ito, ikaw ay:
- Mananatiling updated sa mga bagong feature at karagdagang nilalaman
- Mauunawaan ang mga pagpapabuti sa kasalukuyang dokumentasyon
- Masusubaybayan ang mga pag-aayos ng bug at mga koreksyon para sa mas tumpak na impormasyon
- Masusundan ang ebolusyon ng mga materyales sa pag-aaral sa paglipas ng panahon

## Mga Resulta ng Pag-aaral

Pagkatapos i-review ang mga entry sa changelog, magagawa mo:
- Tukuyin ang mga bagong nilalaman at resources para sa pag-aaral
- Mauunawaan kung aling mga seksyon ang na-update o pinahusay
- Magplano ng iyong landas sa pag-aaral batay sa pinakabagong materyales
- Magbigay ng feedback at mga mungkahi para sa mga susunod na pagpapabuti

## Pag-aambag sa Changelog  

### Pag-uulat ng Mga Pagbabago  
Kapag nag-aambag sa repository na ito, tiyaking ang mga entry sa changelog ay naglalaman ng:  

1. **Version Number**: Sumusunod sa semantic versioning (major.minor.patch)  
2. **Date**: Petsa ng release o update sa format na YYYY-MM-DD  
3. **Category**: Added, Changed, Deprecated, Removed, Fixed, Security  
4. **Clear Description**: Maikling paglalarawan ng mga pagbabago  
5. **Impact Assessment**: Paano maaapektuhan ang mga kasalukuyang user  

### Mga Kategorya ng Pagbabago  

#### Added  
- Mga bagong feature, seksyon ng dokumentasyon, o kakayahan  
- Mga bagong halimbawa, template, o learning resources  
- Karagdagang tools, scripts, o utilities  

#### Changed  
- Mga pagbabago sa umiiral na functionality o dokumentasyon  
- Mga update para mapabuti ang kalinawan o katumpakan  
- Pag-aayos ng content o organisasyon  

#### Deprecated  
- Mga feature o approach na unti-unting tinatanggal  
- Mga seksyon ng dokumentasyon na nakatakdang alisin  
- Mga pamamaraan na may mas mahusay na alternatibo  

#### Removed  
- Mga feature, dokumentasyon, o halimbawa na hindi na nauugnay  
- Lipas na impormasyon o deprecated na approach  
- Redundant o pinagsamang content  

#### Fixed  
- Mga pagwawasto sa error sa dokumentasyon o code  
- Pagresolba sa mga naiulat na isyu o problema  
- Mga pagpapabuti sa katumpakan o functionality  

#### Security  
- Mga pagpapabuti o pag-aayos na may kaugnayan sa seguridad  
- Mga update sa security best practices  
- Pagresolba sa mga security vulnerabilities  

### Mga Alituntunin sa Semantic Versioning  

#### Major Version (X.0.0)  
- Mga breaking changes na nangangailangan ng aksyon mula sa user  
- Malaking restructuring ng content o organisasyon  
- Mga pagbabago na nagbabago sa pangunahing approach o methodology  

#### Minor Version (X.Y.0)  
- Mga bagong feature o dagdag na content  
- Mga enhancement na nananatiling backward compatible  
- Karagdagang halimbawa, tools, o resources  

#### Patch Version (X.Y.Z)  
- Mga bug fixes at pagwawasto  
- Minor na pagpapabuti sa umiiral na content  
- Mga paglilinaw at maliliit na enhancement
