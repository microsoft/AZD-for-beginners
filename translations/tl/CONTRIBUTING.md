# Pag-aambag sa AZD For Beginners

Salamat sa iyong interes sa pag-aambag sa kurso ng AZD For Beginners! Ang dokumentong ito ay naglalaman ng mga gabay at impormasyon para sa mga nag-aambag.

## Paano Mag-ambag

### Pag-uulat ng mga Isyu

Kung makakita ka ng mga bug, error, o may mga mungkahi para sa pagpapabuti:

1. **Check Existing Issues**: Hanapin ang [umiiral na mga isyu](https://github.com/microsoft/azd-for-beginners/issues) upang maiwasan ang mga duplikado
2. **Create a New Issue**: Gumamit ng angkop na issue template
3. **Provide Details**: Magbigay ng mga detalye kasama ang mga hakbang para maulit, inaasahang pag-uugali, at mga screenshot kung naaangkop

### Pagmumungkahi ng mga Pagbabago

1. **Fork the Repository**: Gumawa ng iyong sariling fork ng proyekto
2. **Create a Branch**: Gumawa ng isang feature branch para sa iyong mga pagbabago
3. **Make Changes**: Sundin ang aming mga pamantayan sa coding at dokumentasyon
4. **Test Your Changes**: Siguruhing gumagana lahat ng mga link at maayos ang pag-render ng nilalaman
5. **Submit a Pull Request**: Magbigay ng malinaw na paglalarawan ng iyong mga pagbabago

### Mga Uri ng Pag-aambag

Tinatanggap namin ang mga sumusunod na uri ng pag-aambag:

#### Pagpapabuti ng Nilalaman
- Ayusin ang mga typo, gramatika, at mga isyu sa pag-format
- Pahusayin ang kalinawan ng umiiral na dokumentasyon
- I-update ang lipas na impormasyon
- Magdagdag ng nawawalang mga paliwanag o konteksto

#### Mga Bagong Halimbawa
- Magdagdag ng mga aktwal na senaryo ng deployment
- Lumikha ng mga bagong template ng AZD para sa karaniwang mga kaso ng paggamit
- Magbigay ng karagdagang mga halimbawa ng code at mga pagsasanay

#### Suporta sa Pagsasalin
- Tumulong panatilihin ang umiiral na mga pagsasalin
- Pahusayin ang kalidad ng pagsasalin
- Magdagdag ng mga pagsasalin para sa bagong nilalaman

#### Pag-uulat ng mga Bug
- Iulat ang mga sirang link o maling mga halimbawa ng code
- Tukuyin ang mga hindi pagkakatugma sa dokumentasyon
- Iulat ang mga isyu sa accessibility

## Mga Patnubay sa Pag-aambag

### Mga Pamantayan sa Dokumentasyon

- Gumamit ng malinaw na wikang angkop sa mga nagsisimula
- Isama ang mga layunin sa pagkatuto at mga inaasahang kinalabasan
- Magbigay ng sunud-sunod na mga tagubilin na may inaasahang mga output
- Magdagdag ng mga link para sa pag-navigate sa mga kaugnay na nilalaman
- Gumamit ng pare-parehong pag-format at istilo ng markdown

### Mga Halimbawa ng Code

- Tiyakin na ang lahat ng mga halimbawa ng code ay nasubukan at gumagana
- Isama ang mga komento na nagpapaliwanag ng mga pangunahing konsepto
- Sundin ang mga pinakamahusay na kasanayan ng Azure at AZD
- Magbigay ng inaasahang output kung naaangkop

### Mga Mensahe ng Commit

Gumamit ng malinaw at naglalarawang mga mensahe ng commit:
- `docs: fix typo in installation guide`
- `feat: add new container app example`
- `fix: correct broken link in chapter 2`
