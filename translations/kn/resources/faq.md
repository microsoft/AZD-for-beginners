## ಪರಿಚಯ

ಈ ಸಂಪೂರ್ಣ FAQ Azure Developer CLI (azd) ಮತ್ತು Azure ಪರಿನಿಯೋಜನೆಗಳ ಬಗ್ಗೆ ಸಾಮಾನ್ಯವಾಗಿ ಕೇಳಲ್ಪಡುವ ಪ್ರಶ್ನೆಗಳಿಗೆ ಉತ್ತರಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ. ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳಿಗೆ ವೇಗವಾದ ಪರಿಹಾರಗಳನ್ನು ಕಂಡುಹಿಡಿಯಿರಿ, ಉತ್ತಮ ಅಭ್ಯಾಸಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಿ, ಮತ್ತು azd ತತ್ವಗಳು ಮತ್ತು ಕಾರ್ಯಪ್ರವಾಹಗಳ ಕುರಿತು ಸ್ಪಷ್ಟತೆಯನ್ನು ಪಡೆಯಿರಿ.

## Getting Started

### ಪ್ರಶ್ನೆ: Azure Developer CLI (azd) ಎಂದರೇನು?
**ಉತ್ತರ**: Azure Developer CLI (azd)是一 ترقي developer-centric command-line સાધನ ಅದು ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿ ಪರಿಸರದಿಂದ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು Azure ಗೆ ತಲುಪಿಸಲು ಅಗತವಾದ ಸಮಯವನ್ನು ವೇಗಗೊಳಿಸುತ್ತದೆ. ಇದು ಟೆಂಪ್ಲೇಟ್ಗಳ ಮೂಲಕ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ ಮತ್ತು ಪೂರ್ಣ ನಿಯೋಜನೆ ಜೀವನಚರ್ಚೆಯನ್ನು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

### ಪ್ರಶ್ನೆ: azd ಮತ್ತು Azure CLI ನಡುವೆ ಏನು ವ್ಯತ್ಯಾಸವಿದೆ?
**ಉತ್ತರ**: 
- **Azure CLI**: Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿರ್ವಹಿಸುವ ಸಾಮಾನ್ಯ-ಉದ್ದೇಶದ ಸಾಧನ
- **azd**: ಅಪ್ಲಿಕೇಶನ್ ನಿಯೋಜನೆ ಕಾರ್ಯಪ್ರವಾಹಗಳಿಗಾಗಿ ಡೆವೆಲಪರ್-ಕೇಂದ್ರಿತ ಸಾಧನ
- azd ಒಳಗೆ Azure CLI ಅನ್ನು ಬಳಸುತ್ತದೆ ಆದರೆ ಸಾಮಾನ್ಯ ಅಭಿವೃದ್ಧಿ ಸನ್ನಿವೇಶಗಳಿಗಾಗಿ ಉನ್ನತ ಮಟ್ಟದ ಆಬ್ಸ್ಟ್ರ್ಯಾಕ್ಶನ್‌ಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ
- azd ටೆಂಪ್ಲೇಟ್ಗಳು, ಪರಿಸರ ನಿರ್ವಹಣೆ ಮತ್ತು ನಿಯೋಜನೆ ಸ್ವಯಂಚಾಲಿತವಾಗಿಸುವಿಕೆಗಳನ್ನು ಒಳಗೊಂಡಿದೆ

### ಪ್ರಶ್ನೆ: azd ಬಳಸಲು Azure CLI ಸ್ಥಾಪಿತವಾಗಿರಬಹುದೇ?
**ಉತ್ತರ**: ಹೌದು, azd ಗೆ Azure CLI.authenticate ಮತ್ತು ಕೆಲವು ಕಾರ್ಯಾಚರಣೆಗಳಿಗೆ ಅಗತ್ಯವಿದೆ. ಮೊದಲು Azure CLI ಅನ್ನು ಸ್ಥಾಪಿಸಿ, ನಂತರ azd ಅನ್ನು ಸ್ಥಾಪಿಸಿ.

### ಪ್ರಶ್ನೆ: azd ಯಾವ ಪ್ರೋಗ್ರಾಮಿಂಗ್ ಭಾಷೆಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ?
**ಉತ್ತರ**: azd ಭಾಷಾ-ಅಗ್ನೋಸ್ಟಿಕ್ ಆಗಿದೆ. ಇದು ಕೆಳಗಿನೊಂದಿಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ:
- Node.js/JavaScript/TypeScript
- Python
- .NET/C#
- Java
- Go
- PHP
- Static websites
- Containerized applications

### ಪ್ರಶ್ನೆ: ನಾನು ಈಗಾಗಲೇ ಇರುವ ಯೋಜನೆಗಳೊಂದಿಗೆ azd ಬಳಸಬಹುದೇ?
**ಉತ್ತರ**: ಹೌದು! ನೀವು ಅಥವ:
1. ಮುಂದಿನ ಪ್ರಕ್ರಿಯೆಗಳಿಗೆ `azd init` ಬಳಸಿ ಈಗಾಗಲೇ ಇರುವ ಯೋಜನೆಗಳಿಗೆ azd ಸಂರಚನೆಯನ್ನು ಸೇರಿಸಬಹುದು
2. ನಿಮ್ಮ ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ ಯೋಜನೆಗಳನ್ನು azd ಟೆಂಪ್ಲೇಟು ರಚನೆಗೆ ಹೊಂದಿಸಿಕೊಳ್ಳಿ
3. ನಿಮ್ಮ ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ ವಾಸ್ತುಶಿಲ್ಪದ ಆಧಾರದ ಮೇಲೆ ಕಸ್ಟಮ್ ಟೆಂಪ್ಲೇಟುಗಳು ರಚಿಸಿ

## Authentication & Access

### ಪ್ರಶ್ನೆ: azd ಬಳಸಿ Azure ನೊಂದಿಗೆ ನಾನು ಹೇಗೆ ಪ್ರಾಮಾಣೀಕರಿಸಬೇಕು?
**ಉತ್ತರ**: `azd auth login` ಬಳಸಿ, ಇದು Azure ಪ್ರಾಮಾಣೀಕರಣಕ್ಕಾಗಿ ಬ್ರೌಸರ್ ವಿಂಡೋವನ್ನು ತೆರೆಯುತ್ತದೆ. CI/CD ಸನ್ನಿವೇಶಗಳಿಗಾಗಿ, ಸೇವಾ ಪ್ರಿಂಸಿಪಲ್ಗಳು ಅಥವಾ ನಿರ್ವಹಿತ identities ಗಳನ್ನು ಬಳಸಿ.

### ಪ್ರಶ್ನೆ: ನಾನು ಅನೇಕ Azure.subscription ಗಳೊಂದಿಗೆ azd ಬಳಸಬಹುದೇ?
**ಉತ್ತರ**: ಹೌದು. ಪ್ರತಿಯೊಂದು ಪರಿಸರಕ್ಕೆ ಯಾವ subscription ಬಳಸಬೇಕೆಂದು ನಿರ್ದೇಶಿಸಲು `azd env set AZURE_SUBSCRIPTION_ID <subscription-id>` ಅನ್ನು ಬಳಸಿ.

### ಪ್ರಶ್ನೆ: azd ನೊಂದಿಗೆ ನಿಯೋಜಿಸಲು ನನಗೆ ಯಾವ ಅನುಮತಿ ಬೇಕು?
**ಉತ್ತರ**: ಸಾಮಾನ್ಯವಾಗಿ ನಿಮಗೆ:
- ರಿಸೋರ್ಸ್ ಗುಂಪು ಅಥವಾ subscription ಮೇಲೆ **Contributor** ಪಾತ್ರ ಬೇಕಾಗಿರುತ್ತದೆ
- ಪಾತ್ರ ಹಂಚಿಕೆಗಳಿರಬೇಕಾದ ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿಯೋಜಿಸುವಾಗ **User Access Administrator** ಬೇಕಾಗಿರುತ್ತದೆ
- ನಿರ್ದಿಷ್ಟ ಅನುಮತಿಗಳು ಟೆಂಪ್ಲೇಟು ಮತ್ತು ನಿಯೋಜಿಸಲಾಗುತ್ತಿರುವ ಸಂಪನ್ಮೂಲಗಳ ಮೇಲೆ ಅವಲಂಬಿತವಾಗಿರುತ್ತವೆ

### ಪ್ರಶ್ನೆ: ನಾನು CI/CD ಪೈಪ್‌ಲೈನ್ಗಳಲ್ಲಿ azd ಬಳಸಬಹುದೇ?
**ಉತ್ತರ**: ಖಂಡಿತವಾಗಿ! azd CI/CD ಒಗ್ಗೂಡಿಕೆಗೆ ವಿನ್ಯಾಸಗೊಳಿಸಲಾಗಿದೆ. ಪ್ರಾಮಾಣೀಕರಣಕ್ಕಾಗಿ ಸೇವಾ ಪ್ರಿಂಸಿಪಲ್ಗಳನ್ನು ಬಳಸಿ ಮತ್ತು ಸಂರಚನೆಗಾಗಿ ಪರಿಸರ ಚರಗಳನ್ನು ಸೆಟ್ ಮಾಡಿ.

### ಪ್ರಶ್ನೆ: GitHub Actions ನಲ್ಲಿ ನಾನು ಪ್ರಾಮಾಣೀಕರಣವನ್ನು ಹೇಗೆ ನಿರ್ವಹಿಸಬೇಕು?
**ಉತ್ತರ**: ಸೇವಾ ಪ್ರಿಂಸಿಪಲ್ ಕ್ರೆಡೆಂಶಿಯಲ್ಸ್ ಬಳಸಿ Azure Login action ಅನ್ನು ಬಳಸಿ:
```yaml
- uses: azure/login@v1
  with:
    creds: ${{ secrets.AZURE_CREDENTIALS }}
- run: azd deploy --no-prompt
```

## Templates & Projects

### ಪ್ರಶ್ನೆ: azd ಟೆಂಪ್ಲೇಟ್ಗಳನ್ನು ನಾನು ಎಲ್ಲಿಂದ ಹುಡುಕಬಹುದು?
**ಉತ್ತರ**: 
- ಅಧಿಕೃತ ಟೆಂಪ್ಲೇಟ್ಗಳು: [Azure-Samples/awesome-azd](https://github.com/Azure-Samples/awesome-azd)
- ಸಮುದಾಯ ಟೆಂಪ್ಲೇಟ್ಗಳು: GitHub ನಲ್ಲಿ "azd-template" ಅನ್ನು ಹುಡುಕಿ
- ಲಭ್ಯವಿರುವ ಟೆಂಪ್ಲೇಟ್ಗಳನ್ನು ಬ್ರೌಸ್ ಮಾಡಲು `azd template list` ಅನ್ನು ಬಳಸಿ

### ಪ್ರಶ್ನೆ: ನಾನು ಕಸ್ಟಮ್ ಟೆಂಪ್ಲೇಟು ಅನ್ನು ಹೇಗೆ ರಚಿಸಬಹುದು?
**ಉತ್ತರ**: 
1. ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ ಟೆಂಪ್ಲೇಟು ರಚನೆಯೊಂದಿಗೆ ಪ್ರಾರಂಭಿಸಿ
2. `azure.yaml`, ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಫೈಲ್‌ಗಳು ಮತ್ತು ಅಪ್ಲಿಕೇಶನ್ ಕೋಡ್ ಅನ್ನು ಬದಲಾಯಿಸಿ
3. `azd up` ಬಳಸಿ ಚೆನ್ನಾಗಿ ಪರೀಕ್ಷಿಸಿ
4. ಸೂಕ್ತ ಟ್ಯಾಗ್‌ಗಳೊಂದಿಗೆ GitHub ಗೆ ಪ್ರಕಟಿಸಿ

### ಪ್ರಶ್ನೆ: ಟೆಂಪ್ಲೇಟು ಇಲ್ಲದೆ ನಾನು azd ಬಳಸಬಹುದೇ?
**ಉತ್ತರ**: ಹೌದು, `azd init` ಅನ್ನು ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ ಯೋಜನೆದಲ್ಲಿ ಬಳಸಿಕೊಂಡು ಅಗತ್ಯವಿರುವ ಸಂರಚನಾ ಫೈಲ್‌ಗಳನ್ನು ರಚಿಸಬಹುದು. ನಿಮಗೆ `azure.yaml` ಮತ್ತು ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಫೈಲ್‌ಗಳನ್ನು ಕೈಯಿಂದ ಕಾನ್‌ಫಿಗರ್ ಮಾಡಬೇಕಾಗುತ್ತದೆ.

### ಪ್ರಶ್ನೆ: ಅಧಿಕೃತ ಮತ್ತು ಸಮುದಾಯ ಟೆಂಪ್ಲೇಟ್ಗಳ ಮಧ್ಯೆ ವ್ಯತ್ಯಾಸವೇನು?
**ಉತ್ತರ**: 
- **ಅಧಿಕೃತ ಟೆಂಪ್ಲೇಟ್ಗಳು**: Microsoft ಮೂಲಕ ನಿರ್ವಹಿಸಲಾಗುತ್ತವೆ, ನಿಯಮಿತವಾಗಿ ನವೀಕರಿಸಲಾಗಿದೆ, ಸಂಪೂರ್ಣ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಹೊಂದಿವೆ
- ** ಸಮುದಾಯ ಟೆಂಪ್ಲೇಟ್ಗಳು**: ಡೆವೆಲಪರ್‌ಗಳು ರಚಿಸಿದವು, ನಿರ್ದಿಷ್ಟ ಉಪಯೋಗ ಪ್ರಕರಣಗಳಿರಬಹುದು, ಗುಣಮಟ್ಟ ಮತ್ತು ನಿರ್ವಹಣೆ ಬದಲಾಗಬಹುದು

### ಪ್ರಶ್ನೆ: ನನ್ನ ಪ್ರಾಜೆಕ್ಟ್‌ನಲ್ಲಿ ಟೆಂಪ್ಲೇಟನ್ನು ನವೀಕರಿಸಲು ನಾನು ಹೇಗೆ ಅನ್ವಯಿಸಬೇಕು?
**ಉತ್ತರ**: ಟೆಂಪ್ಲೇಟುಗಳು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ನವೀಕರಿಸಲ್ಪಡುವುದಿಲ್ಲ. ನೀವು ಆಗಬಹುದು:
1. ಮೂಲ ಟೆಂಪ್ಲೇಟಿನಿಂದ ಕೈಯಿಂದ ಹೋಲಿಸಿ ಮತ್ತು ಬದಲಾವಣೆಗಳನ್ನು ಮರ್ಜ್ ಮಾಡಿ
2. ನವೀಕರಿಸಿದ ಟೆಂಪ್ಲೇಟು ಬಳಸಿ `azd init` ಮೂಲಕ ಹೊಸದಾಗಿ ಪ್ರಾರಂಭಿಸಿ
3. ನವೀಕರಿಸಿದ ಟೆಂಪ್ಲೇಟಿನಿಂದ ನಿರ್ದಿಷ್ಟ ಸುಧಾರಣೆಗಳನ್ನು ಚೆರಿ-ಪಿಕ್ ಮಾಡಿ

## Deployment & Infrastructure

### ಪ್ರಶ್ನೆ: azd ಮೂಲಕ ಯಾವ Azure ಸೇವೆಗಳನ್ನು ನಿಯೋಜಿಸಬಹುದು?
**ಉತ್ತರ**: azd Bicep/ARM 템್ಪ್ಲೇಟ್‌ಗಳ ಮೂಲಕ ಯಾವುದೇ Azure ಸೇವೆಯನ್ನು ನಿಯೋಜಿಸಬಹುದು, அதರಲ್ಲಿ ಒಳಗೊಂಡಿವೆ:
- App Services, Container Apps, Functions
- Databases (SQL, PostgreSQL, Cosmos DB)
- Storage, Key Vault, Application Insights
- Networking, security, and monitoring resources

### ಪ್ರಶ್ನೆ: ನಾನು ಬಹು ಪ್ರದೇಶಗಳಿಗೆ ನಿಯೋಜಿಸಬಹುದೇ?
**ಉತ್ತರ**: ಹೌದು, ನಿಮ್ಮ Bicep 템್ಪ್ಲೇಟ್‌ಗಳಲ್ಲಿ ಬಹು ಪ್ರದೇಶಗಳನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಿ ಮತ್ತು ಪ್ರತಿ ಪರಿಸರಕ್ಕೆ ಸ್ಥಳ ಪ್ಯಾರಾಮೀಟರ್ ಅನ್ನು ಸರಿಯಾಗಿ ಸೆಟ್ ಮಾಡಿ.

### ಪ್ರಶ್ನೆ: ಡೇಟಾಬೇಸ್ ಸ್ಕೀಮಾ ಮೈಗ್ರೇಶನ್‌ಗಳನ್ನು ನಾನು ಹೇಗೆ ನಿರ್ವಹಿಸ ಬೇಕು?
**ಉತ್ತರ**: `azure.yaml` ನಲ್ಲಿ ಡಿಪ್ಲಾಯ್‌ಮೆಂಟ್ ಹೂಕುಗಳನ್ನು ಬಳಸಿ:
```yaml
hooks:
  postdeploy:
    posix:
      run: ./scripts/migrate-database.sh
    windows:
      run: ./scripts/migrate-database.ps1
```

### ಪ್ರಶ್ನೆ: ನಾನು ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಲ್ಲದೆ ಕೇವಲ ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚಾರ್ ಅನ್ನು ಮಾತ್ರ ನಿಯೋಜಿಸಬಹುದೇ?
**ಉತ್ತರ**: ಹೌದು, ಟೆಂಪ್ಲೇಟುಗಳಲ್ಲಿ ವಿವರಣೆಯಾದ ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಘಟಕಗಳನ್ನು ಮಾತ್ರ ನಿಯೋಜಿಸಲು `azd provision` ಅನ್ನು ಬಳಸಿ.

### ಪ್ರಶ್ನೆ: ನಾನು ಈಗಾಗಲೇ ಇರುವ Azure ಸಂಪನ್ಮೂಲಗಳಿಗೆ ಹೇಗೆ ನಿಯೋಜಿಸಬೇಕು?
**ಉತ್ತರ**: ಇದು ಜಟಿಲವಾಗಿದ್ದು ನೇರವಾಗಿ ಬೆಂಬಲಿತವಾಗಿಲ್ಲ. ನೀವು:
1. ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿಮ್ಮ Bicep ಟೆಂಪ್ಲೇಟ್‌ಗಳಿಗೆ ಇಂಪೋರ್ಟ್ ಮಾಡಬಹುದು
2. ಟೆಂಪ್ಲೇಟುಗಳಲ್ಲಿ ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ ಸಂಪನ್ಮೂಲ ರೆಫರೆನ್ಸ್‌ಗಳನ್ನು ಉಪಯೋಗಿಸಿ
3. ಸಂಪನ್ಮೂಲಗಳನ್ನು ಶರತಾಗಿ ಸೃಷ್ಟಿಸುವ ಅಥವಾ ರೆಫರೆನ್ಸ್ ಮಾಡುವಂತೆ ಟೆಂಪ್ಲೇಟುಗಳನ್ನು ಬದಲಾಯಿಸಿ

### ಪ್ರಶ್ನೆ: ನಾನು Bicep ಬದಲು Terraform ಬಳಸಬಹುದೇ?
**ಉತ್ತರ**: ಪ್ರಸ್ತುತ, azd ಮುಖ್ಯವಾಗಿ Bicep/ARM 템್ಪ್ಲೇಟ್‌ಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ. Terraform ಬೆಂಬಲವು ಅಧಿಕೃತವಾಗಿ ಲಭ್ಯವಿಲ್ಲ, ಆದರೆ ಸಮುದಾಯ ಪರಿಹಾರಗಳು ಇರಬಹುದು.

## Configuration & Environments

### ಪ್ರಶ್ನೆ: ನಾನು ವಿಭಿನ್ನ ಪರಿಸರಗಳನ್ನು (dev, staging, prod) ಹೇಗೆ ನಿರ್ವಹಿಸಬೇಕು?
**ಉತ್ತರ**: ಪ್ರತಿ ಪರಿಸರಕ್ಕೆ ವಿಭಿನ್ನ ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಲು `azd env new <environment-name>` ಬಳಸಿ ವಿವಿಧ ಪರಿಸರಗಳನ್ನು ರಚಿಸಿ:
```bash
azd env new development
azd env new staging  
azd env new production
```

### ಪ್ರಶ್ನೆ: ಪರಿಸರ ಸಂರಚನೆಗಳು ಎಲ್ಲಿದ್ದವೆ?
**ಉತ್ತರ**: ನಿಮ್ಮ ಪ್ರಾಜೆಕ್ಟ್ ಡೈरेक್ಟರಿಯಲ್ಲಿರುವ `.azure` ಫೋಲ್ಡರ್‌ನಲ್ಲಿ ಇವೆ. ಪ್ರತಿ ಪರಿಸರಕ್ಕೆ ಅದರ ಸ್ವಂತ ಫೋಲ್ಡರ್ ಮತ್ತು ಸಂರಚನಾ ಫೈಲ್‌ಗಳಿರುತ್ತವೆ.

### ಪ್ರಶ್ನೆ: ನಾನು ಪರಿಸರ-ನಿರ್ಧಿಷ್ಟ ಸಂರಚನೆಯನ್ನು ಹೇಗೆ ಸೆಟ್ ಮಾಡಬೇಕು?
**ಉತ್ತರ**: ಪರಿಸರ ಚರಗಳನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಲು `azd env set` ಅನ್ನು ಬಳಸಿ:
```bash
azd env set AZURE_LOCATION eastus
azd env set DATABASE_TIER Basic
```

### ಪ್ರಶ್ನೆ: ತಂಡದ ಸದಸ್ಯರೊಂದಿಗೆ ಪರಿಸರ ಸಂರಚನೆಯನ್ನು ಹಂಚಿಕೊಳ್ಳಬಹುದೇ?
**ಉತ್ತರ**: `.azure` ಫೋಲ್ಡರ್ ಸೆನ್ಸಿಟಿವ್ ಮಾಹಿತಿ ಹೊಂದಿರುತ್ತದೆ ಮತ್ತು ಅದನ್ನು ಆವೃತ್ತಿ ನಿಯಂತ್ರಣದಲ್ಲಿ ಕಮಿಟ್ ಮಾಡಬಾರದು. ಬದಲಾಗಿ:
1. ಅಗತ್ಯವಾದ ಪರಿಸರ ಚರಗಳನ್ನು ದಾಖಲಿಸಿ
2. ಪರಿಸರಗಳನ್ನು ಹೊಂದಿಸಲು ನಿಯೋಜನೆ ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳನ್ನು ಬಳಸಿ
3. ಸೆನ್ಸಿಟಿವ್ ಕಾನ್ಫಿಗರೇಶನ್‌ಗಾಗಿ Azure Key Vault ಅನ್ನು ಬಳಸಿ

### ಪ್ರಶ್ನೆ: ನಾನು ಟೆಂಪ್ಲೇಟು ಡೀಫಾಲ್ಟ್‌ಗಳನ್ನು ಹೇಗೆ ಓವರ್‌ರೈಡ್ ಮಾಡುವುದು?
**ಉತ್ತರ**: ಟೆಂಪ್ಲೇಟು ಪ್ಯಾರಾಮೀಟರ್ಗಳಿಗೆ ಅನುಸಾರವಾಗಿರುವ ಪರಿಸರ ಚರಗಳನ್ನು ಸೆಟ್ ಮಾಡಿ:
```bash
azd env set LOCATION "West US 2"
azd env set SKU_NAME "B1"
```

## Troubleshooting

### ಪ್ರಶ್ನೆ: `azd up` ವಿಫಲವಾಗುತ್ತಿರುವುದಕ್ಕೆ ಕಾರಣ ಏನು?
**ಉತ್ತರ**: ಸಾಮಾನ್ಯ ಕಾರಣಗಳು:
1. **ಪ್ರಾಮಾಣೀಕರಣ ಸಮಸ್ಯೆಗಳು**: `azd auth login` ಓಡಿಸಿ
2. **ಅಪ್ರತಿಪಾದಿತ ಅನುಮತಿಗಳು**: ನಿಮ್ಮ Azure ಪಾತ್ರ ಹಂಚಿಕೆಗಳನ್ನು ಪರಿಶೀಲಿಸಿ
3. **ಸಂಪನ್ಮೂಲ ನಾಮಕರಣ ಧ್ವನಿಸಭ್ರಮೆಗಳು**: AZURE_ENV_NAME ಬದಲಾಯಿಸಿ
4. **ಕೊಟಾ/ಕ್ಷಮತಾ ಸಮಸ್ಯೆಗಳು**: ಪ್ರಾದೇಶಿಕ ಲಭ್ಯತೆಯನ್ನು ಪರಿಶೀಲಿಸಿ
5. **ಟೆಂಪ್ಲೇಟು ದೋಷಗಳು**: Bicep ಟೆಂಪ್ಲೇಟುಗಳನ್ನು ಮಾನ್ಯಗೊಳಿಸಿ

### ಪ್ರಶ್ನೆ: ನಿಯೋಜನೆ ವಿಫಲತೆಯನ್ನು ನಾನು ಹೇಗೆ ಡಿಬಗ್ ಮಾಡಬಹುದು?
**ಉತ್ತರ**: 
1. ವಿಶದ ಔಟ್‌ಪುಟ್‌ಗೆ `azd deploy --debug` ಬಳಸಿರಿ
2. Azure ಪೋರ್ಟಲ್‌ನಲ್ಲಿ ನಿಯೋಜನೆ ಇತಿಹಾಸವನ್ನು ಪರಿಶೀಲಿಸಿ
3. Azure ಪೋರ್ಟಲ್‌ನಲ್ಲಿ Activity Log ಅನ್ನು ಪರಿಶೀಲಿಸಿ
4. ಪ್ರಸ್ತುತ ಪರಿಸರ ಸ್ಥಿತಿಯನ್ನು ಪ್ರದರ್ಶಿಸಲು `azd show` ಅನ್ನು ಬಳಸಿ

### ಪ್ರಶ್ನೆ: ನನ್ನ ಪರಿಸರ ಚರಗಳು ಕೆಲಸ ಮಾಡುತ್ತಿಲ್ಲದಿದ್ದರೆ ಏನು ಪರಿಶೀಲಿಸಬೇಕು?
**ಉತ್ತರ**: ಪರಿಶೀಲಿಸಿ:
1. ಚರಗಳ ಹೆಸರುಗಳು ಟೆಂಪ್ಲೇಟು ಪ್ಯಾರಾಮೀಟರ್ಗಳಿಗೆ ನಿಖರವಾಗಿ ಹೊಂದಿಕೆಯಾಗಿವೆ ಎಂದು
2. ಮೌಲ್ಯಗಳು ಸ್ಥಳಗಳನ್ನು ಒಳಗೊಂಡಿದ್ದರೆ ಸರಿಯಾಗಿ ಅಲೆಟ್ ಆಗಿವೆ ಎಂದು
3. ಪರಿಸರ ಆಯ್ಕೆ ಮಾಡಲಾಗಿದೆ: `azd env select <environment>`
4. ಚರಗಳು ಸರಿಯಾದ ಪರಿಸರದಲ್ಲಿ ಸೆಟ್ ಮಾಡబಡೆಯುತ್ತದೆ

### ಪ್ರಶ್ನೆ: ವಿಫಲವಾದ ನಿಯೋಜನೆಗಳನ್ನು ನಾನು ಹೇಗೆ ಕ್ಲೀನ್ ಅಪ್ ಮಾಡಬೇಕು?
**ಉತ್ತರ**: 
```bash
azd down --force --purge
```
ಇದು ಎಲ್ಲ ಸಂಪನ್ಮೂಲಗಳು ಮತ್ತು ಪರಿಸರ ಸಂರಚನೆಯನ್ನು ರದ್ದುಮಾಡುತ್ತದೆ.

### ಪ್ರಶ್ನೆ: ನಿಯೋಜನೆಯ ನಂತರ ನನ್ನ ಅಪ್ಲಿಕೇಶನ್ ಪ್ರವೇಶಿಸಲು ಸಾಧ್ಯವಾಗುತ್ತಿಲ್ಲದಿದ್ದರೆ ಏನು ಕಾರಣ?
**ಉತ್ತರ**: ಪರಿಶೀಲಿಸಿ:
1. ನಿಯೋಜನೆ ಯಶಸ್ವಿಯಾಗಿ ಪೂರ್ಣಗೊಂಡಿದೆಯೇ
2. ಅಪ್ಲಿಕೇಶನ್ ಚಾಲನೆಯಲ್ಲಿ ಇದೆ (Azure ಪೋರ್ಟಲ್‌ನಲ್ಲಿ ಲಾಗ್‌ಗಳನ್ನು ಪರಿಶೀಲಿಸಿ)
3. ನೆಟ್ವರ್ಕ್ ಸೆಕ್ಯುರಿಟಿ ಗುಂಪುಗಳು ಟ್ರಾಫಿಕ್‌ಗೆ ಅವಕಾಶ ನೀಡುತ್ತವೆಯೇ
4. DNS/ಕಸ್ಟಮ್ ಡೊಮೇನ್ಗಳು ಸರಿಯಾಗಿ ಕಾನ್ಫಿಗರ್ ಆಗಿವೆ

5. Azure Cost Management ಸಾಧನಗಳನ್ನು ಬಳಸಿ

### ಪ್ರಶ್ನೆ: azd ಟೆಂಪ್ಲೇಟ್ಗಳಿಗಾಗಿ ಉಚಿತ ಟಿಯರ್ ಆಯ್ಕೆಗಳಿರವೆಯೇ?
**ಉತ್ತರ**: ಅನೇಕ Azure ಸೇವೆಗಳು ಉಚಿತ ಟಿಯರ್‌ಗಳನ್ನು ಒದಗಿಸುತ್ತವೆ:
- App Service: ಉಚಿತ ಟಿಯರ್ ಲಭ್ಯ
- Azure Functions: ಪ್ರತಿ ತಿಂಗಳಿಗೆ 1M ಉಚಿತ ಕಾರ್ಯಾಚರಣೆಗಳು
- Cosmos DB: 400 RU/s ನೊಂದಿಗೆ ಉಚಿತ ಟಿಯರ್
- Application Insights: ಮೊದಲ 5GB/ತಿಂಗಳ ಉಚಿತ

ಲಭ್ಯವಿದ್ದಲ್ಲಿ ಟೆಂಪ್ಲೇಟುಗಳನ್ನು ಉಚಿತ ಟಿಯರ್ ಬಳಕೆಗಾಗಿ ಕಾನ್ಫಿಗರ್ ಮಾಡಿ.

### ಪ್ರಶ್ನೆ: ನಿಯೋಜನೆಗಾಗಿ ಮೊದಲೆಗೆ ವೆಚ್ಚವನ್ನು ನಾನು ಹೇಗೆ ಅಂದಾಜು ಮಾಡಬಹುದು?
**ಉತ್ತರ**: 
1. ರಚನೆಗೊಳ್ಳುವ ಸಂಪನ್ಮೂಲಗಳನ್ನು ನೋಡಲು ಟೆಂಪ್ಲೇಟಿನ `main.bicep` ಪರಿಶೀಲಿಸಿ
2. ನಿರ್ದಿಷ್ಟ SKU ಗಳೊಂದಿಗೆ Azure Pricing Calculator ಬಳಸಿರಿ
3. ಮೊದಲು ಅಭಿವೃದ್ಧಿ ಪರಿಸರವಕ್ಕೆ ನಿಯೋಜಿಸಿ ನಿಜವಾದ ವೆಚ್ಚವನ್ನು ಮೇಲ್ವಿಚಾರಿಸಿರಿ
4. ವಿವರವಾದ ವೆಚ್ಚ ವಿಶ್ಲೇಷಣೆಗೆ Azure Cost Management ಬಳಸಿ

## Best Practices

### ಪ್ರಶ್ನೆ: azd ಪ್ರಾಜೆಕ್ಟ್ ರಚನೆಗಾಗಿ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು ಯಾವುವು?
**ಉತ್ತರ**: 
1. ಅಪ್ಲಿಕೇಶನ್ ಕೋ드를 ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ನಿಂದ ಬೇರ್ಪಡಿಸಿ
2. `azure.yaml` ನಲ್ಲಿ ಅರ್ಥಪೂರ್ಣ ಸೇವಾ ಹೆಸರುಗಳನ್ನು ಬಳಸಿ
3. ಬಿಲ್ಡ್ ಸ್ಕೈಪ್ಟ್‌ಗಳಲ್ಲಿ ಸೂಕ್ತ ದೋಷ ನಿರ್ವಹಣೆಯನ್ನು ಜಾರಿಗೆ ತರುವಿರಿ
4. ಪರಿಸರ-ನಿರ್ಧಿಷ್ಟ ಸಂರಚನೆಯನ್ನು ಬಳಸಿರಿ
5. ಸಮಗ್ರ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಅನ್ನು ಸೇರಿಸಿ

### ಪ್ರಶ್ನೆ: azd ನಲ್ಲಿ ಬಹು ಸೇವೆಗಳನ್ನು ನಾನು ಹೇಗೆ ಸಂಸ್ಥೆಯಲ್ಲಿಡಬೇಕು?
**ಉತ್ತರ**: ಶಿಫಾರಸು ಮಾಡಿದ ರಚನೆಯನ್ನು ಬಳಸಿ:
```
├── src/
│   ├── web/          # Frontend service
│   ├── api/          # Backend service  
│   └── worker/       # Background service
├── infra/            # Infrastructure templates
├── azure.yaml        # azd configuration
└── README.md         # Project documentation
```

### ಪ್ರಶ್ನೆ: `.azure` ಫೋಲ್ಡರ್ ಅನ್ನು ಆವೃತ್ತಿ ನಿಯಂತ್ರಣದಲ್ಲಿ ಕಮಿಟ್ ಮಾಡಬೇಕೇ?
**ಉತ್ತರ**: **ಇಲ್ಲ!** `.azure` ಫೋಲ್ಡರ್ ಸೆನ್ಸಿಟಿವ್ ಮಾಹಿತಿಯನ್ನು ಹೊಂದಿರುತ್ತದೆ. ಅದನ್ನು `.gitignore` ಗೆ ಸೇರಿಸಿ:
```gitignore
.azure/
```

### ಪ್ರಶ್ನೆ: ರಹಸ್ಯಗಳು ಮತ್ತು ಸೆನ್ಸಿಟಿವ್ ಸಂರಚನೆಯನ್ನು ನಾನು ಹೇಗೆ ನಿರ್ವಹಿಸಬೇಕು?
**ಉತ್ತರ**: 
1. ರಹಸ್ಯಗಳಿಗಾಗಿ Azure Key Vault ಬಳಸಿ
2. ಅಪ್ಲಿಕೇಶನ್ ಸಂರಚನೆಯಲ್ಲಿ Key Vault ರಹಸ್ಯಗಳನ್ನು ರೆಫರೆನ್ಸ್ ಮಾಡಿ
3. ರಹಸ್ಯಗಳನ್ನು ಎಂದಿಗೂ ಆವೃತ್ತಿ ನಿಯಂತ್ರಣದಲ್ಲಿ ಕಮಿಟ್ ಮಾಡಬೇಡಿ
4. ಸೇವೆ-ಗೆ-ಸೇವೆಗೆ ಪ್ರಾಮಾಣೀಕರಣಕ್ಕೆ ನಿರ್ವಹಿತ identities ಬಳಸಿ

### ಪ್ರಶ್ನೆ: azd ಮೂಲಕ CI/CD ನ ಶಿಫಾರಸು ಮಾಡಲಾದ ಮಾರ್ಗವೇನು?
**ಉತ್ತರ**: 
1. ಪ್ರತಿ ಹಂತಕ್ಕೆ (dev/staging/prod) ಸೇರ್ಪಡೆಗೆ ವಿಭಿನ್ನ ಪರಿಸರಗಳನ್ನು ಬಳಸಿ
2. ನಿಯೋಜನೆಯ ಮೊದಲು ಸ್ವಯಂಚಾಲಿತ ಪರೀಕ್ಷೆಗಳನ್ನು ಜಾರಿ ಮಾಡಿರಿ
3. ಪ್ರಾಮಾಣೀಕರಣಕ್ಕೆ ಸೇವಾ ಪ್ರಿಂಸಿಪಲ್ಗಳನ್ನು ಬಳಸಿ
4. ಪೈಪ್‌ಲೈನ್ ರಹಸ್ಯ/ಚರಗಳಲ್ಲಿ ಸೆನ್ಸಿಟಿವ್ ಸಂರಚನೆಯನ್ನು ಸಂಗ್ರಹಿಸಿ
5. ಉತ್ಪಾದನೆ ನಿಯೋಜನೆಗಳಿಗಾಗಿ ಅನುಮೋದನೆ ಗೇಟ್ಗಳನ್ನು ಜಾರಿ ಮಾಡಿ

## Advanced Topics

### ಪ್ರಶ್ನೆ: ನಾನು azd ಗೆ ಕಸ್ಟಮ್ ಕಾರ್ಯಕ್ಷಮತೆಯನ್ನು ವಿಸ್ತರಿಸಬಹುದೇ?
**ಉತ್ತರ**: ಹೌದು, `azure.yaml` ನಲ್ಲಿ ಡಿಪ್ಲಾಯ್‌ಮೆಂಟ್ ಹుక్‌ಗಳ ಮೂಲಕ:
```yaml
hooks:
  predeploy:
    run: ./scripts/custom-setup.sh
  postdeploy:
    run: ./scripts/custom-config.sh
```

### ಪ್ರಶ್ನೆ: azd ಅನ್ನು ಇಮ್ಮಡಿನ DevOps ಪ್ರಕ್ರಿಯೆಗಳೊಂದಿಗೆ ನಾನು ಹೇಗೆ ಏಕರೂಪಗೊಳಿಸಬಹುದು?
**ಉತ್ತರ**: 
1. ಇಮ್ಮಡಿನ ಪೈಪ್‌ಲೈನ್ ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳಲ್ಲಿ azd ಆಜ್ಞೆಗಳನ್ನು ಬಳಸಿ
2. ತಂಡಗಳಾದ್ಯಂತ azd ಟೆಂಪ್ಲೇಟುಗಳ ಮೇಲೆ ಸ್ತರೀಕರಣ ಮಾಡಿ
3. ಇಮ್ಮಡಿನ ಮಾನಿಟರಿಂಗ್ ಮತ್ತು ಅಲರ್ಟಿಂಗ್‌ ಜೊತೆಗೆ ಏಕರೂಪಗೊಳಿಸಿ
4. ಪೈಪ್‌ಲೈನ್ ಏಕೆ ಸೇರಿಸಲು azd ನ JSON ಔಟ್‌ಪುಟ್ ಅನ್ನು ಬಳಸಿ

### ಪ್ರಶ್ನೆ: azd ಅನ್ನು Azure DevOps ಜೊತೆಗೆ ಬಳಸಬಹುದೇ?
**ಉತ್ತರ**: ಹೌದು, azd ಯಾವುದೇ CI/CD ವ್ಯವಸ್ಥೆಯೊಂದಿಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ. azd ಆಜ್ಞೆಗಳನ್ನು ಬಳಸಿ Azure DevOps ಪೈಪ್‌ಲೈನ್‌ಗಳನ್ನು ರಚಿಸಿ.

### ಪ್ರಶ್ನೆ: ನಾನು azd ಗೆ ಹೇಗೆ ಕೊಡುಗೆ ನೀಡಬಹುದು ಅಥವಾ ಸಮುದಾಯ ಟೆಂಪ್ಲೇಟುಗಳನ್ನು ರಚಿಸಬಹುದು?

1. **azd ಸಾಧನ**: [Azure/azure-dev](https://github.com/Azure/azure-dev) ಗೆ ಕೊಡುಗೆ ನೀಡಿ
2. **ಟೆಂಪ್ಲೇಟುಗಳು**: [ಟೆಂಪ್ಲೇಟ್ ಮಾರ್ಗಸೂಚಿಗಳನ್ನು](https://github.com/Azure-Samples/awesome-azd) ಅನುಗುಣವಾಗಿ ಟೆಂಪ್ಲೇಟುಗಳನ್ನು ರಚಿಸಿರಿ
3. **ದಾಖಲೆಗಳು**: [MicrosoftDocs/azure-dev-docs](https://github.com/MicrosoftDocs/azure-dev-docs) ನಲ್ಲಿ ಡಾಕ್ಯುಮೆಂಟ್‌ಗಳಿಗೆ ಕೊಡುಗೆ ಮಾಡಿ

### ಪ್ರಶ್ನೆ: azd ಗಾಗಿ ರೋಡ್‌ಮ್ಯಾಪ್ ಏನು?
**ಉತ್ತರ**: ಯೋಜಿಸಲಾದ ವೈಶಿಷ್ಟ್ಯಗಳು ಮತ್ತು ಸುಧಾರಣೆಗಳಿಗಾಗಿ [ಅಧಿಕೃತ ರೋಡ್‌ಮ್ಯಾಪ್](https://github.com/Azure/azure-dev/projects) ಪರಿಶೀಲಿಸಿ.

### ಪ್ರಶ್ನೆ: ಇತರ ನಿಯೋಜನಾ ಸಾಧನಗಳಿಂದ azd ಗೆ ನಾನು ಹೇಗೆ ಸ್ಥಳಾಂತರಿಸಬೇಕು?
**ಉತ್ತರ**: 
1. ಪ್ರಸ್ತುತ ನಿಯೋಜನೆ ಆರ್ಕಿಟೆಕ್ಚರ್ ಅನ್ನು ವಿಶ್ಲೇಷಿಸಿ
2. ಸಮಾನ Bicep ಟೆಂಪ್ಲೇಟುಗಳನ್ನು ರಚಿಸಿ
3. `azure.yaml` ಅನ್ನು ಪ್ರಸ್ತುತ ಸೇವೆಗಳಿಗೆ ಹೊಂದಿಕೊಳ್ಳುವಂತೆ ಸಂರಚಿಸಿ
4. ಅಭಿವೃದ್ಧಿ ಪರಿಸರದಲ್ಲಿ ಸಂಪೂರ್ಣವಾಗಿ ಪರೀಕ್ಷಿಸಿ
5. ಕ್ರಮಾನುಗತವಾಗಿ ಪರಿಸರಗಳನ್ನು ಸ್ಥಳಾಂತರಿಸಿ
