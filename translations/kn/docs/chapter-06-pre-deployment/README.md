# ಅಧ್ಯಾಯ 6: ಪೂರ್ವ-ವಿತರಣಾ ಯೋಜನೆ ಮತ್ತು ಮಾನ್ಯತೆ

**📚 ಕೋರ್ಸ್**: [ಆಜಿಡಿ ಆರಂಭಿಕರಿಗೆ](../../README.md) | **⏱️ ಅವಧಿ**: 1 ಘಂಟೆ | **⭐ ಜಟಿಲತೆ**: ಮಧ್ಯಮ

---

## ಅವಲೋಕನ

ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ವಿತರಿಸುವ ಮೊದಲು ಅಗತ್ಯ ಯೋಜನೆ ಮತ್ತು ಮಾನ್ಯತಾ ಹಂತಗಳನ್ನು ಈ ಅಧ್ಯಾಯದಲ್ಲಿ ಕವರ್ ಮಾಡಲಾಗಿದೆ. ಸರಿಯಾದ ಸಾಮರ್ಥ್ಯ ಯೋಜನೆ, SKU ಆಯ್ಕೆ ಮತ್ತು ಪೂರ್ವ-ಪರೀಕ್ಷಣೆಯೊಂದಿಗೆ ದುಬಾರಿತಾದ ತಪ್ಪುಗಳನ್ನು ತಪ್ಪ گیرಿ ಕಲಿಯಿರಿ.

> ಜುಲೈ 2026 ರಲ್ಲಿ `azd 1.27.1` ಆಧರಿಸಿ ಪರಿಶೀಲಿಸಲಾಗಿದೆ.

## ಕಲಿಕಾ ಗುರಿಗಳು

ಈ ಅಧ್ಯಾಯವನ್ನು ಪೂರ್ಣಗೊಳಿಸಿ, ನೀವು:
- ವಿತರಿಸುವ ಮೊದಲು ಪೂರ್ವ-ಪರೀಕ್ಷಣೆಗಳನ್ನು ನಡೆಸಬೇಕು
- ಸಾಮರ್ಥ್ಯವನ್ನು ಯೋಜಿಸಿ ಮತ್ತು ಸಂಪನ್ಮೂಲ ಅಗತ್ಯಗಳನ್ನು ಅಂದಾಜು ಮಾಡಬೇಕು
- ವೆಚ್ಚ ಸೂಕ್ತತೆಗಾಗಿ ಸೂಕ್ತ SKUಗಳನ್ನು ಆಯ್ಕೆ ಮಾಡಬೇಕು
- ಮೇಲ್ವಿಚಾರಣೆಗೆ ಅಪ್ಲಿಕೇಶನ್ ಇನ್ಸೈಟ್ಸ್ ಅನ್ನು ಸಂರಚಿಸಬೇಕು
- ತಂಡದ ಸಂಯೋಜನೆಯ ಮಾದರಿಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬೇಕು

## 🚀 ಶೀಘ್ರ ಪ್ರಾರಂಭ

```bash
# ಗುರುತುಪತ್ರದ ಹಂಚಿಕೆ ಗಳವನ್ನು ಪರಿಶೀಲಿಸಿ
az vm list-usage --location eastus --output table

# ನಿಯೋಜನೆಯನ್ನು ಪೂರ್ವದೃಶ್ಯಗೊಳಿಸಿ (ಯಾವುದೇ ಸಂಪನ್ಮೂಲಗಳು ಸೃಷ್ಟಿಸಲ್ಪಟ್ಟಿಲ್ಲ)
azd provision --preview

# ಬೈಸೆಪ್ ವಾಕ್ಯರಚನೆಯನ್ನು ಮಾನ್ಯತೆ ನೀಡಿ
az bicep build --file infra/main.bicep

# ಪರಿಸರ ಸಂರಚನೆಯನ್ನು ಪರಿಶೀಲಿಸಿ
azd env get-values
```

## ☑️ ಪೂರ್ವ-ವಿತರಣಾ ಪರಿಶೀಲನಾ ಪಟ್ಟಿ

### `azd provision` ಮೊದಲು

- [ ] ಪ್ರದೇಶಕ್ಕೆ ಕ್ವೋಟಾ ಪರಿಶೀಲಿಸಲಾಗಿದೆ
- [ ] SKUಗಳನ್ನು ಸೂಕ್ತವಾಗಿ ಆಯ್ಕೆ ಮಾಡಲಾಗಿದೆ
- [ ] ವೆಚ್ಚದ ಅಂದಾಜು ವಿಮರ್ಶಿಸಲಾಗಿದೆ
- [ ] ನಾಮಕರಣ ನಿಯಮ ಶಿಸ್ತುಗೊಂಡಿದೆ
- [ ] ಭದ್ರತೆ/RBAC ಸಂರಚಿತವಾಗಿದೆ

### `azd deploy` ಮೊದಲು

- [ ] ಪರಿಸರ ಚರಗಳನ್ನು ಸೆಟ್ ಮಾಡಲಾಗಿದೆ
- [ ] ಕೀ ವಾಲ್ಟ್‌ನಲ್ಲಿ ರಹಸ್ಯಗಳು ಇವೆ
- [ ] ಸಂಪರ್ಕ ಸರಣಿಗಳನ್ನು ಪರಿಶೀಲಿಸಲಾಗಿದೆ
- [ ] ಆರೋಗ್ಯ ಪರಿಶೀಲನೆಗಳು ಸಂರಚಿತವಾಗಿದೆ
