# Chapter 6: ಡಿಪ್ಲಾಯ್ ಮುನ್ನಿನ ಯೋಜನೆ ಮತ್ತು ಮಾನ್ಯತೆ

**📚 Course**: [AZD ಪ್ರಾರಂಭಿಕರಿಗೆ](../../README.md) | **⏱️ Duration**: 1 ಗಂಟೆ | **⭐ Complexity**: ಮಧ್ಯಮ

---

## ಸಾರಾಂಶ

ಈ ಅಧ್ಯಾಯವು ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಡಿಪ್ಲಾಯ್ ಮಾಡುವ ಮೊದಲು ಮಾಡಬೇಕಾದ ಪ್ರಮುಖ ಯೋಜನೆ ಮತ್ತು ಮಾನ್ಯತೆ ಹಂತಗಳನ್ನು ಒಳಗೊಂಡಿದೆ. ಸಮರ್ಪಕ ಸಾಮರ್ಥ್ಯ ಯೋಜನೆ, SKU ಆಯ್ಕೆ ಮತ್ತು ಪ್ರಿಫ್ಲೈಟ್ ಪರಿಶೀಲನೆಗಳೊಂದಿಗೆ ಖರ್ಚುಮಯ ತಪ್ಪುಗಳನ್ನು ತಪ್ಪಿಸಲು ಕಲಿಯಿರಿ.

> 2026 ರ ಜೂನ್ ನಲ್ಲಿ `azd 1.25.6` ವಿರುದ್ಧ ಮಾನ್ಯಗೊಳಿಸಲಾಗಿದೆ.

## ಕಲಿಕೆಯ ಉದ್ದೇಶಗಳು

ಈ ಅಧ್ಯಾಯವನ್ನು ಪೂರ್ಣಗೊಳಿಸಿದ ನಂತರ, ನೀವು:
- ಡಿಪ್ಲಾಯ್ಮೆಂಟ್ ಮುನ್ನ ಪ್ರಿಫ್ಲೈಟ್ ಪರಿಶೀಲನೆಗಳನ್ನು ರನ್ ಮಾಡುವುದು
- ಸಾಮರ್ಥ್ಯವನ್ನು ಯೋಜಿಸಿ ಸಂಪನ್ಮೂಲ ಅಗತ್ಯಗಳನ್ನು ಅಂದಾಜು ಮಾಡುವುದು
- ಖರ್ಚು ಆప్టಿಮೈಜೆಷನ್ ನಿಗೆ ಯೋಗ್ಯ SKU ಗಳನ್ನು ಆಯ್ಕೆ ಮಾಡುವುದು
- ಮಾನಿಟರಿಂಗ್ ಗೆ Application Insights ಅನ್ನು ಸಂರಚಿಸುವುದು
- ತಂಡ ಸಂಯೋಜನೆ ಮಾದರಿಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು

## 🚀 ತ್ವರಿತ ಆರಂಭ

```bash
# ಸಬ್ಸ್ಕ್ರಿಪ್ಷನ್ ಕ್ವೋಟಾಗಳನ್ನು ಪರಿಶೀಲಿಸಿ
az vm list-usage --location eastus --output table

# ನಿಯೋಜನೆಯನ್ನು ಪೂರ್ವಾವಲೋಕನ ಮಾಡಿ (ಯಾವುದೇ ಸಂಪನ್ಮೂಲಗಳು ರಚಿಸಲಾಗುವುದಿಲ್ಲ)
azd provision --preview

# Bicep ವ್ಯಾಕರಣವನ್ನು ಪರಿಶೀಲಿಸಿ
az bicep build --file infra/main.bicep

# ಪರಿಸರ ಸಂರಚನೆಯನ್ನು ಪರಿಶೀಲಿಸಿ
azd env get-values
```

## ☑️ ಪೂರ್ವ-ಡಿಪ್ಲಾಯ್ ಪರಿಶೀಲನಾ ಪಟ್ಟಿ

### `azd provision` ಗಿಂತ ಮೊದಲು

- [ ] ಪ್ರದೇಶಕ್ಕಾಗಿ ಕ್ವೋಟಾ ಪರಿಶೀಲಿಸಲಾಗಿದೆ
- [ ] SKUs ಯನ್ನು ಯೋಗ್ಯವಾಗಿ ಆಯ್ಕೆ ಮಾಡಲಾಗಿದೆ
- [ ] ಖರ್ಚಿನ ಅಂದಾಜು ಪರಿಶೀಲಿಸಲಾಗಿದೆ
- [ ] ಹೆಸರಿಸುವ ನಿಯಮಾವಳಿ ಸुसಂಗತವಾಗಿದೆ
- [ ] ಸುರಕ್ಷತೆ/RBAC ಸಂರಚಿಸಲಾಗಿದೆ

### `azd deploy` ಗಿಂತ ಮೊದಲು

- [ ] ಎನ್ವಿರಾನ್‌ಮೆಂಟ್ ಚರಗಳು ಹೊಂದಿಸಲಾಗಿದೆ
- [ ] Key Vault ನಲ್ಲಿ ರಹಸ್ಯಗಳು (Secrets) ಇದ್ದವೆ
- [ ] ಕನೆಕ್ಷನ್ ಸ್ಟ್ರಿಂಗ್‌ಗಳು ಪರಿಶೀಲಿಸಲ್ಪಟ್ಟಿವೆ
- [ ] ಹೆಲ್ತ್ ಚೆಕ್ಸ್‌ಗಳು ಸಂರಚಿಸಲ್ಪಟ್ಟಿವೆ
