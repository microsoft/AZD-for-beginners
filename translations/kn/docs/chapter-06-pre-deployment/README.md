# Chapter 6: ಡಿಪ್ಲಾಯ್‌ಮೆಂಟ್ ಮುನ್ನ ಯೋಜನೆ ಮತ್ತು ಮಾನ್ಯತೆ

## ಅವಲೋಕನ

ಈ ಅಧ್ಯಾಯವು ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಡಿಪ್ಲಾಯ್ ಮಾಡುವ ಮೊದಲು ಅಗತ್ಯವಾದ ಯೋಜನೆ ಮತ್ತು ಮಾನ್ಯತೆ ಹಂತಗಳನ್ನು ಒಳಗೊಂಡಿದೆ. ಸರಿಯಾದ ಸಾಮರ್ಥ್ಯ ಯೋಜನೆ, SKU ಆಯ್ವೆ ಮತ್ತು ಪ್ರೀಫ್ಲೈಟ್ ಪರಿಶೀಲನೆಗಳ ಮೂಲಕ ખર્ચಸಾಧ್ಯ ತೃಟಿಗಳನ್ನು ತಪ್ಪಿಸುವುದನ್ನು ಕಲಿಯಿರಿ.

## ಅಧ್ಯಯನ ಉದ್ದೇಶಗಳು

By completing this chapter, you will:
- ಡಿಪ್ಲಾಯ್ ಮಾಡುವ ಮೊದಲು ಪ್ರೀಫ್ಲೈಟ್ ಪರೀಕ್ಷೆಗಳು ನಡೆಸುವುದು
- ಸಾಮರ್ಥ್ಯವನ್ನು ಯೋಜಿಸಿ ಮತ್ತು ಸಂಪನ್ಮೂಲ ಅಗತ್ಯಗಳನ್ನು ಅಂದಾಜುಮಾಡುವುದು
- ವೆಚ್ಚ ನಿರ್ವಹಣೆಗೆ ಸೂಕ್ತ SKUs ಆಯ್ಕೆಮಾಡುವುದು
- ನಿರೀಕ್ಷಣೆಗೆ Application Insights ಅನ್ನು ಸಂರಚಿಸುವುದು
- ಟೀಂ ಸಂಯೋಜನಾ ಮಾದರಿಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು

## 🚀 ತ್ವರಿತ ಪ್ರಾರಂಭ

```bash
# ಸಬ್ಸ್ಕ್ರಿಪ್ಶನ್ ಮಿತಿಗಳನ್ನು ಪರಿಶೀಲಿಸಿ
az vm list-usage --location eastus --output table

# ನಿಯೋಜನೆಯನ್ನು ಪೂರ್ವವೀಕ್ಷಿಸಿ (ಯಾವುದೇ ಸಂಪನ್ಮೂಲಗಳನ್ನು ರಚಿಸಲಾಗುವುದಿಲ್ಲ)
azd provision --preview

# Bicep ವ್ಯಾಕರಣವನ್ನು ಪರಿಶೀಲಿಸಿ
az bicep build --file infra/main.bicep

# ಪರಿಸರ ಸಂರಚನೆಯನ್ನು ಪರಿಶೀಲಿಸಿ
azd env get-values
```

## ☑️ ಡಿಪ್ಲಾಯ್‌ಮೆಂಟ್ ಮುನ್ನ ಪರಿಶೀಲನಾ ಪಟ್ಟಿ

### `azd provision` ಮೊದಲು

- [ ] ಪ್ರಾದೇಶಿಕ ಕ್ವೋಟಾ ಪರಿಶೀಲಿಸಲಾಗಿದೆ
- [ ] SKUs ಸೂಕ್ತವಾಗಿ ಆಯ್ಕೆ ಮಾಡಲಾಗಿದೆ
- [ ] ವೆಚ್ಚ ಅಂದಾಜು ಪರಿಶೀಲಿಸಲಾಗಿದೆ
- [ ] ಹೆಸರಿನ ನಿಯಮಾವಳಿ ಸुसಂಗತವಾಗಿದೆ
- [ ] ಭದ್ರತೆ/RBAC ಸಂರಚಿಸಲಾಗಿದೆ

### `azd deploy` ಮೊದಲು

- [ ] Environment variables ಹೊಂದಿಸಲಾಗಿದೆ
- [ ] Key Vault ನಲ್ಲಿ ರಹಸ್ಯಗಳು ಸಂರಕ್ಷಣೆಯಲ್ಲಿವೆ
- [ ] ಸಂಪರ್ಕ ಸ್ಟ್ರಿಂಗ್‌ಗಳು ಪರಿಶೀಲಿಸಲಾಗಿದೆ
- [ ] ಆರೋಗ್ಯ ಪರಿಶೀಲನೆಗಳು ಸಂರಚಿಸಲಾಗಿದೆ
