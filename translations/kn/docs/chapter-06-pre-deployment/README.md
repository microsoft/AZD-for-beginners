# ಅಧ್ಯಾಯ 6: ಪೂರ್ವ-ವಿತರಣೆ ಯೋಜನೆ ಮತ್ತು ಮಾನ್ಯತೆ

**📚 ಕೋರ್ಸ್**: [AZD For Beginners](../../README.md) | **⏱️ ಅವಧಿ**: 1 ಘಂಟೆ | **⭐ ಸಂಕೀರ್ಣತೆ**: ಮಧ್ಯಮ

---

## ಅವಲೋಕನ

ಈ ಅಧ್ಯಾಯವು ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ನಿಯೋಜಿಸುವ ಮೊದಲು ಅಗತ್ಯವಾದ ಯೋಜನೆ ಮತ್ತು ಮಾನ್ಯತಾ ಹಂತಗಳನ್ನು ಒಳಗೊಂಡಿದೆ. ಸೂಕ್ತ ಸಾಮರ್ಥ್ಯ ಯೋಜನೆ, SKU ಆಯ್ಕೆ ಮತ್ತು ಪ್ರಿಫ್ಲೈಟ್ ಪರಿಶೀಲನೆಗಳೊಂದಿಗೆ ದುಬಾರಿಯಾದ ತಪ್ಪುಗಳನ್ನು ತಪ್ಪಿಸುವುದನ್ನು ಕಲಿಯಿರಿ.

## ಕಲಿಕೆಯ ಉದ್ದೇಶಗಳು

ಈ ಅಧ್ಯಾಯವನ್ನು ಪೂರ್ಣಗೊಳಿಸಿದ ಮೂಲಕ, ನೀವು:
- ನಿಯೋಜನೆಯ ಮೊದಲು ಪ್ರಿಫ್ಲೈಟ್ ಪರಿಶೀಲನೆಗಳನ್ನು ನಡೆಸುತ್ತೀರಿ
- ಸಾಮರ್ಥ್ಯವನ್ನು ಯೋಜಿಸಿ ಸಂಪನ್ಮೂಲ ಅವಶ್ಯಕತೆಯನ್ನು ಅಂದಾಜಿಸುತ್ತೀರಿ
- ವೆಚ್ಚದ ತಿರುವಿಗಾಗಿ ಸೂಕ್ತ SKUಗಳನ್ನು ಆಯ್ಕೆಮಾಡುತ್ತೀರಿ
- ಮಾನಿಟರಿಂಗ್‌ಗಾಗಿ Application Insights ಅನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡುತ್ತೀರಿ
- ತಂಡ ಸಂಯೋಜನೆ ಪದ್ಧತಿಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುತ್ತೀರಿ

## 🚀 ತ್ವರಿತ ಪ್ರಾರಂಭ

```bash
# ಸಬ್ಸ್ಕ್ರಿಪ್ಷನ್ ಮಿತಿಗಳನ್ನು ಪರಿಶೀಲಿಸಿ
az vm list-usage --location eastus --output table

# ಡಿಪ್ಲಾಯ್‌ಮೆಂಟ್ ಪೂರ್ವದೃಶ್ಯ (ಯಾವುದೇ ಸಂಪನ್ಮೂಲಗಳು ರಚಿಸಲ್ಪಡುವುದಿಲ್ಲ)
azd provision --preview

# Bicep ಸಿಂಟ್ಯಾಕ್ಸ್ ಮಾನ್ಯಗೊಳಿಸಿ
az bicep build --file infra/main.bicep

# ವಾತಾವರಣ ಸಂರಚನೆಯನ್ನು ಪರಿಶೀಲಿಸಿ
azd env get-values
```

## ☑️ ಪೂರ್ವ-ನಿಯೋಜನೆ ತಪಾಸಣಾ ಪಟ್ಟಿವು

### Before `azd provision`

- [ ] ಪ್ರದೇಶಕ್ಕೆ ಕ್ವೋಟಾ ಪರಿಶೀಲಿಸಲಾಗಿದೆ
- [ ] SKUಗಳು ಸೂಕ್ತವಾಗಿ ಆಯ್ಕೆಮಾಡಲಾಗಿದೆ
- [ ] ವೆಚ್ಚದ ಅಂದಾಜು ಪರಿಶೀಲಿಸಲಾಗಿದೆ
- [ ] ಹೆಸರಿಡುವ ನಿಯಮ ಸ್ಥಿರವಾಗಿದೆ
- [ ] ಭದ್ರತೆ/RBAC ಕಾನ್ಫಿಗರ್ ಆಗಿದೆ

### Before `azd deploy`

- [ ] ಪರಿಸರ ಚರಗಳು ಸೆಟ್ ಆಗಿವೆ
- [ ] ರಹಸ್ಯಗಳು Key Vault ನಲ್ಲಿ ಇವೆ
- [ ] ಸಂಪರ್ಕ ಸ್ಟ್ರಿಂಗ್‌ಗಳನ್ನು ಪರಿಶೀಲಿಸಲಾಗಿದೆ
- [ ] ಆರೋಗ್ಯ ಪರಿಶೀಲನೆಗಳು ಕಾನ್ಫಿಗರ್ ಆಗಿವೆ
