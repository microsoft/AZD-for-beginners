# ಅಧ್ಯಾಯ 6: ಮುನ್ನಡೆಗೊಳ್ಳುವ ಯೋಜನೆ ಮತ್ತು ಪರಿಶೀಲನೆ

**📚 ಕೋರ್ಸ್**: [ಆಜ್ಡ್ ಜನರುಗಳಿಗೆ](../../README.md) | **⏱️ ಅವಧಿ**: 1 ಗಂಟೆ | **⭐ ಸುತ್ತುಮಟ್ಟ**: ಮಧ್ಯಮ

---

## ಅವಲೋಕನ

ನಿಮ್ಮ ಆಪ್ಲಿಕೇಶನ್ ನಿಯೋಜಿಸುವ ಮೊದಲು ಅಗತ್ಯವಿರುವ ಯೋಜನೆ ಮತ್ತು ಪರಿಶೀಲನೆದ ಹೆಜ್ಜೆಗಳನ್ನ ಈ ಅಧ್ಯಾಯದಲ್ಲಿ ಆವರಿಸಲಾಗಿದೆ. ಸರಿಯಾದ ಸಾಮರ್ಥ್ಯ ಯೋಜನೆ, SKU ಆಯ್ಕೆ ಮತ್ತು ಪೂರ್ವಪರೀಕ್ಷೆ ಮೂಲಕ ದುಬಾರಿ ತಪ್ಪುಗಳನ್ನು ತಪ್ಪಿಸುವುದನ್ನು ಕಲಿಯಿರಿ.

> `azd 1.23.12` ನೊಂದಿಗೆ ಮಾರ್ಚ್ 2026 ರಲ್ಲಿ ಧೃಡಪಡಿಸಲಾಗಿದೆ.

## ಕಲಿಕೆ ಗುರಿಗಳು

ಈ ಅಧ್ಯಾಯವನ್ನು ಪೂರ್ಣಗೊಳಿಸುವ ಮೂಲಕ, ನೀವು:
- ನಿಯೋಜನೆಯ ಮೊದಲು ಪೂರ್ವಪರೀಕ್ಷೆಗಳನ್ನು ನಡೆಸಲು
- ಸಾಮರ್ಥ್ಯವನ್ನು ಯೋಜಿಸುವುದು ಮತ್ತು ಸಂಪನ್ಮೂಲ ಅವಶ್ಯಕತೆಗಳನ್ನು ಅಂದಾಜಿಸಲು
- ಖರ್ಚು ಪ್ರಮಾಣೀಕರಣಕ್ಕಾಗಿ ಸೂಕ್ತ SKU ಗಳು ಆಯ್ಕೆ ಮಾಡಲು
- ಮಾನಿಟರಿಂಗ್ ಗಾಗಿ ಅಪ್ಲಿಕೇಶನ್ ಇನ್ಸೈಟ್ಸ್ ಅನ್ನು ಸಂರಚಿಸಲು
- ತಂಡ ಸಹಕಾರ ಮಾದರಿಯನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು

## 🚀 ತ್ವರಿತ ಪ್ರಾರಂಭ

```bash
# ಚಂದಾದಾರಿಕೆ ಕೊಟಾ ಪರಿಶೀಲಿಸಿ
az vm list-usage --location eastus --output table

# ನಿಯೋಜನೆ ಪೂರ್ವಾವಲೋಕನ (ಯಾವುದೇ ಸಂಪನ್ಮೂಲಗಳು ಸೃಷ್ಟಿಸಲಾಗುವುದಿಲ್ಲ)
azd provision --preview

# ಬೈಸೆಪಿನ ವ್ಯಾಕರಣ ಪರಿಶೀಲಿಸಿ
az bicep build --file infra/main.bicep

# ಪರಿಸರ ಸಂರಚನೆಯನ್ನು ಪರಿಶೀಲಿಸಿ
azd env get-values
```

## ☑️ ನಿಯೋಜನೆ ಮುನ್ನ ಪರಿಶೀಲನಾ ಪಟ್ಟಿಗಳು

### `azd provision` ಗೆ ಮುನ್ನ

- [ ] ಪ್ರದೇಶದ ಕುಟಾ ಪರಿಶೀಲಿಸಲಾಗಿದೆ
- [ ] SKU ಗಳನ್ನು ಸರಿಯಾಗಿ ಆಯ್ಕೆಮಾಡಲಾಗಿದೆ
- [ ] ವೆಚ್ಚ ಅಂದಾಜು ಪರಿಶೀಲಿಸಲಾಗಿದೆ
- [ ] ನಾಮಕರಣ ನಿಯಮ一致ತೆ ಇದೆ
- [ ] ಭದ್ರತೆ/RBAC ಸಂರಚಿಸಲಾಗಿದೆ

### `azd deploy` ಗೆ ಮುನ್ನ

- [ ] ಪರಿಸರ ಚರಗಳಿಂದಾದ್ಷಿ ಹೊಂದಿಸಲಾಗಿದೆ
- [ ] ಗೌಪ್ಯ ಮಾಹಿತಿ键ಚಾಬ್ಹ್ ಕವಾಟ್ ನಲ್ಲಿ ಇಟ್ಟುಕೊಂಡಿದೆ
- [ ] ಸಂಪರ್ಕ ಸರಣಿಗಳನ್ನು ಪರಿಶೀಲಿಸಲಾಗಿದೆ
- [ ] ಆರೋಗ್ಯ ತಪಾಸಣೆಗಳನ್ನು ಸಂರಚಿಸಲಾಗಿದೆ
