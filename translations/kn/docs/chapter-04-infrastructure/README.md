# ಅಧ್ಯಾಯ 4: ಮೂಲಸೌಕರ್ಯವನ್ನು ಕೋಡ್ ಆಗಿ ಮತ್ತು ನಿಯೋಜನೆ

**📚 ಕೋರ್ಸ್**: [ಆರಂಭಿಕರಿಗಾಗಿ AZD](../../README.md) | **⏱️ ಅವಧಿ**: 1-1.5 ಗಂಟೆಗಳು | **⭐ ಜಟಿಲತೆ**: ಮಧ್ಯಮ

---

## ಸಾರಾಂಶ

ಈ ಅಧ್ಯಾಯವು Bicep ಟೆಂಪ್ಲೇಟ್ಗಳೊಂದಿಗೆ ಮೂಲಸೌಕರ್ಯವನ್ನು ಕೋಡ್ ಆಗಿ (IaC) ಮಾದರಿಗಳು, ಸಂಪನ್ಮೂಲ ಪೂರೈಕೆ ಮತ್ತು ಆಜೂರ್ ಡೆವಲಪರ್ CLI ಬಳಸಿ ನಿಯೋಜನೆ ತಂತ್ರಗಳನ್ನು ಒಳಗೊಂಡಿದೆ.

> ಮಾರ್ಚ್ 2026ರಲ್ಲಿ `azd 1.23.12` ವಿರುದ್ಧ ಪರಿಶೀಲಿಸಲಾಗಿದೆ.

## ಕೌಶಲ್ಯದ ಗುರಿಗಳು

ಈ ಅಧ್ಯಾಯವನ್ನು ಪೂರ್ಣಗೊಳಿಸುವ ಮೂಲಕ ನೀವು:
- Bicep ಟೆಂಪ್ಲೇಟ್ ರಚನೆ ಮತ್ತು ವ್ಯಾಕರಣವನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬಹುದು
- `azd provision` ಯಿಂದ ಆಜೂರ್ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಪೂರೈಕೆ ಮಾಡಬಹುದು
- `azd deploy` ಯಿಂದ ಅಪ್ಲಿಕೇಶನ್ಗಳನ್ನು ನಿಯೋಜಿಸಬಹುದು
- ನೀಲಿ- ಹಸಿರು ಮತ್ತು ರೋಲಿಂಗ್ ನಿಯೋಜನೆ ತಂತ್ರಗಳನ್ನು ಅನುಷ್ಠಾನ ಮಾಡಬಹುದು

## 🚀 ವೇಗವಾಗಿ ಪ್ರಾರಂಭಿಸಿ

```bash
# ಟೆಂಪ್ಲೇಟಿನಿಂದ ಪ್ರಾರಂಭಿಸಿ
azd init --template azure-functions-python-v2-http

# ಏನು ಸೃಷ್ಟಿಯಾಗುತ್ತದೆ ಎಂದು ಪೂರ್ವದೃಶ್ಯ ಮಾಡಿ
azd provision --preview

# ಮೂಲಸೌಕರ್ಯವನ್ನು ಮಾತ್ರ ಒದಗಿಸಿ
azd provision

# ಕೇಡ್ ಅನ್ನು ಮಾತ್ರ ನಿಯೋಜಿಸಿ
azd deploy

# ಅಥವಾ ಎರಡನ್ನೂ ಒಟ್ಟಿಗೆ
azd up
```

```
my-project/
├── azure.yaml            # ← Project definition
├── infra/                # ← Infrastructure as Code
│   ├── main.bicep        # Entry point
│   ├── main.parameters.json
│   └── modules/          # Reusable components
│       ├── web.bicep
│       └── database.bicep
└── src/                  # Application code
    └── api/
```

## 🔧 ಅಗತ್ಯ ಆದೇಶಗಳು

| ಆದೇಶ | ವಿವರಣೆ |
|---------|-------------|
| `azd init` | ಪ್ರಾಜೆಕ್ಟ್ ಆರಂಭಿಸು |
| `azd provision` | ಆಜೂರ್ ಸಂಪನ್ಮೂಲಗಳನ್ನು ರಚನೆಮಾಡು |
| `azd deploy` | ಅಪ್ಲಿಕೇಶನ್ ಕೋಡ್ ನಿಯೋಜನೆ |
| `azd up` | ಪೂರೈಕೆ + ನಿಯೋಜನೆ |
| `azd down` | ಎಲ್ಲಾ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಅಳಿಸು |
