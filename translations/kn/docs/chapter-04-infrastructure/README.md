## ಅವಲೋಕನ

ಈ ಅಧ್ಯಾಯವು Bicep ಟೆಂಪ್ಲೇಟ್ಗಳೊಂದಿಗೆ Infrastructure as Code (IaC) ಮಾದರಿಗಳು, ಸಂಪನ್ಮೂಲ ಒದಗಿಸುವಿಕೆ ಮತ್ತು Azure Developer CLI ಉಪಯೋಗಿಸಿ ನಿಯೋಜನೆ ತಂತ್ರಗಳನ್ನು ಸ್ಪರ್ಶಿಸುತ್ತದೆ.

> ಜೂನ್ 2026 ರಲ್ಲಿ `azd 1.25.6` ವಿರುದ್ಧ ಪರಿಶೀಲಿಸಲಾಗಿದೆ.

## ಕಲಿಕಾ ಉದ್ದೇಶಗಳು

ಈ ಅಧ್ಯಾಯವನ್ನು ಪೂರ್ಣಗೊಳಿಸಿದರೆ, ನೀವು:
- Bicep ಟೆಂಪ್ಲೇಟಿನ ರಚನೆ ಮತ್ತು ವ್ಯಾಕರಣವನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು
- `azd provision` ಬಳಸಿ Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ಒದಗಿಸುವುದು
- `azd deploy` ಬಳಸಿ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿಯೋಜಿಸುವುದು
- ಬ್ಲೂ-ಗ್ರೀನ್ ಮತ್ತು ರೋಲಿಂಗ್ ನಿಯೋಜನೆ ತಂತ್ರಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸುವುದು

## 🚀 ತ್ವರಿತ ಪ್ರಾರಂಭ

```bash
# ಟೆಂಪ್ಲೇಟಿನಿಂದ ಪ್ರಾರಂಭಿಸಿ
azd init --template azure-functions-python-v2-http

# ಏನು ರಚಿಸಲಾಗುವುದು ಎಂಬುದನ್ನು ಪೂರ್ವವೀಕ್ಷಿಸಿ
azd provision --preview

# ಮೂಲಸೌಕರ್ಯವನ್ನು ಮಾತ್ರ ಒದಗಿಸಿ
azd provision

# ಕೇವಲ ಕೋಡ್ ಅನ್ನು ನಿಯೋಜಿಸಿ
azd deploy

# ಅಥವಾ ಎರಡನ್ನೂ ಒಟ್ಟಿಗೆ
azd up
```

## 📁 AZD ಯೋಜನೆಯ ರಚನೆ

```
my-project/
├── azure.yaml            # ← Project definition
├── infra/                # ← Infrastructure as Code
│   ├── main.bicep        # Entry point
│   ├── main.parameters.json
│   └── modules/          # Reusable components
│       ├── web.bicep
│       └── database.bicep
└── src/                  # Application code
    └── api/
```

## 🔧 ಅತ್ಯಾವಶ್ಯಕ ಕಮಾಂಡ್‌ಗಳು

| Command | Description |
|---------|-------------|
| `azd init` | ಪ್ರಾಜೆಕ್ಟ್ ಆರಂಭಿಸು |
| `azd provision` | Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ರಚಿಸು |
| `azd deploy` | ಅಪ್ಲಿಕೇಶನ್ ಕೋಡ್ ಅನ್ನು ನಿಯೋಜಿಸು |
| `azd up` | ಸಂಪನ್ಮೂಲ ಒದಗಿಸುವಿಕೆ + ನಿಯೋಜನೆ |
| `azd down` | ಎಲ್ಲಾ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಅಳಿಸು |
