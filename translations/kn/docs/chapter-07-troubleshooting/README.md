# ಅಧ್ಯಾಯ 7: ಸಮಸ್ಯೆ ಪರಿಹಾರ ಮತ್ತು ಡಿಬಗಿಂಗ್

**📚 Course**: [AZD For Beginners](../../README.md) | **⏱️ Duration**: 1-1.5 ಗಂಟೆಗಳು | **⭐ Complexity**: ಮಧ್ಯಮ

## ಅವಲೋಕನ

ಈ ಅಧ್ಯಾಯವು Azure Developer CLI ಸಹಿತ ಕೆಲಸ ಮಾಡುವಾಗ ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳನ್ನು ನಿರ್ಧರಿಸಲು ಮತ್ತು ಪರಿಹರಿಸಲು ನಿಮಗೆ ಸಹಾಯ ಮಾಡುತ್ತದೆ. ನಿಯೋಜನೆ ವೈಫಲ್ಯಗಳಾದರೂ AI-ನಿಗಾದ್ರಶ್ಯ ಸಮಸ್ಯೆಗಳವರೆಗೂ ಒಳಗೊಂಡಿದೆ.

> `azd 1.25.6` ವಿರುದ್ಧ ಜೂನ್ 2026 ರಲ್ಲಿ ಪರಿಶೀಲಿಸಲಾಗಿದೆ.

## ಕಲಿಕೆಯ ಉದ್ದೇಶಗಳು

ಈ ಅಧ್ಯಾಯವನ್ನು ಪೂರ್ಣಗೊಳಿಸಿದರೆ, ನೀವು:
- ಸಾಮಾನ್ಯ AZD ನಿಯೋಜನೆ ವೈಫಲ್ಯಗಳನ್ನು ಪತ್ತೆಹಚ್ಚುವುದು
- ಪ್ರಮಾಣೀಕರಣ ಮತ್ತು ಅನುಮತಿ ಸಮಸ್ಯೆಗಳನ್ನು ಡಿಬಗ್ ಮಾಡುವುದು
- AI ಸೇವಾ ಸಂಪರ್ಕ ಸಮಸ್ಯೆಗಳನ್ನು ಪರಿಹರಿಸುವುದು
- ಸಮಸ್ಯೆ ಪರಿಹಾರಕ್ಕೆ Azure Portal ಮತ್ತು CLI ಬಳಸುವುದು

## 🚨 ತ್ವರಿತ ಪರಿಹಾರಗಳು

### ಪ್ರಮಾಣೀಕರಣ ಸಮಸ್ಯೆಗಳು
```bash
# AZD ಕಾರ್ಯಪ್ರವಾಹಗಳಿಗೆ ಅಗತ್ಯವಿದೆ
azd auth login

# ನೀವು ನೇರವಾಗಿ Azure CLI ಕಮಾಂಡ್‌ಗಳನ್ನು ಕೂಡ ಬಳಸುತ್ತಿದ್ದರೆ ಐಚ್ಛಿಕ
az login

azd auth status
```

### ಪ್ರೊವಿಷನಿಂಗ್ ವೈಫಲ್ಯಗಳು
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### ಸಂಪನ್ಮೂಲ ಸಂಘರ್ಷಗಳು
```bash
azd down --force --purge
azd env new different-name
azd up
```

### ಕ್ವೋಟಾ ಮೀರಿದೆ
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 ದೋಷ ಕೋಡ್ ಉಲ್ಲೇಖ

| ದೋಷ | ಕಾರಣ | ಪರಿಹಾರ |
|-------|-------|----------|
| `AuthenticationError` | ಲಾಗಿನ್ ಆಗಿಲ್ಲ | `azd auth login` |
| `ResourceNotFound` | ಸಂಪನ್ಮೂಲ ಕಂಡಿಲ್ಲ | ಸಂಪನ್ಮೂಲದ ಹೆಸರುಗಳನ್ನು ಪರಿಶೀಲಿಸಿ |
| `QuotaExceeded` | ಸಬ್ಸ್ಕ್ರಿಪ್ಶನ್ ಮಿತಿಗಳು | ಕ್ವೋಟಾ ಹೆಚ್ಚಿಸಲು ವಿನಂತಿಸಿ |
| `InvalidTemplate` | Bicep ವಾಕ್ಯರಚನೆ ದೋಷ | `az bicep build` |
| `Conflict` | ಸಂಪನ್ಮೂಲ ಈಗಾಗಲೇ ಇದೆ | ಹೊಸ ಹೆಸರು ಬಳಸಿ ಅಥವಾ ಅಳಿಸಿ |
| `Forbidden` | ಅನುಮತಿ ತಗ್ಗಿದೆ | RBAC ಪಾತ್ರಗಳನ್ನು ಪರಿಶೀಲಿಸಿ |

## 🔄 ರಿಸೆಟ್ ಮತ್ತು ಪುನರುದ್ಧಾರ

```bash
# ಸಾಫ್ಟ್ ರೀಸೆಟ್ (ಸಂಪನ್ಮೂಲಗಳನ್ನು ಉಳಿಸಿ, ಕೋಡ್ ಅನ್ನು ಮರುಪ್ರತಿಷ್ಠಾಪಿಸಿ)
azd deploy --force

# ಹಾರ್ಡ್ ರೀಸೆಟ್ (ಎಲ್ಲವನ್ನೂ ಅಳಿಸಿ, ಹೊಸದಾಗಿ ಪ್ರಾರಂಭಿಸಿ)
azd down --force --purge
azd up
```
