# ಅಧ್ಯಾಯ 7: ತೊಂದರೆ ನಿವಾರಣೆ ಮತ್ತು ನವೀಕರಣ

**📚 ಕೋರ್ಸ್**: [AZD ಆರಂಭಿಕರಿಗಾಗಿ](../../README.md) | **⏱️ ಅವಧಿ**: 1-1.5 ಗಂಟೆಗಳು | **⭐ ಜಟಿಲತೆ**: ಮಧ್ಯಮ

---

## ಅವಲೋಕನ

ಈ ಅಧ್ಯಾಯವು ನೀವು Azure Developer CLI ನೊಂದಿಗೆ ಕೆಲಸ ಮಾಡುವಾಗ ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳನ್ನು ಗುರುತಿಸಲು ಮತ್ತು ಪರಿಹರಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ. ನಿಯೋಜನೆ ವಿಫಲತೆಯಿಂದ AI-ನಿಗಾದ ಸಮಸ್ಯೆಗಳವರೆಗೆ.

> ಜುಲೈ 2026 ರಲ್ಲಿ `azd 1.27.1` ನೊಂದಿಗೆ ಪರಿಶೀಲಿಸಲಾಗಿದೆ.

## ಅಧ್ಯಯನ ಉದ್ದೇಶಗಳು

ಈ ಅಧ್ಯಾಯವನ್ನು ಪೂರ್ಣಗೊಳಿಸುವ ಮೂಲಕ, ನೀವು:
- ಸಾಮಾನ್ಯ AZD ನಿಯೋಜನೆ ವಿಫಲತೆಯನ್ನು ಗುರುತಿಸುವಿರಿ
- ಮಾನ್ಯತಾ ಮತ್ತು ಅನುಮತಿ ಸಮಸ್ಯೆಗಳನ್ನು ನವೀಕರಿಸುವಿರಿ
- AI ಸೇವಾ ಸಂಪರ್ಕ ಸಮಸ್ಯೆಗಳನ್ನು ಪರಿಹರಿಸುವಿರಿ
- ತೊಂದರೆ ನಿವಾರಣೆಗೆ Azure ಪೋರ್ಟಲ್ ಮತ್ತು CLI ಬಳಸುವಿರಿ

## 🚨 ತ್ವರಿತ ಪರಿಹಾರಗಳು

### ಮಾನ್ಯತಾ ಸಮಸ್ಯೆಗಳು
```bash
# AZD ಕಾರ್ಯಪ್ರವಾಹಗಳಿಗೆ ಅಗತ್ಯವಿದೆ
azd auth login

# ನೀವು ನೇರವಾಗಿ Azure CLI ಆಜ್ಞೆಗಳನ್ನೂ ಬಳಸುತ್ತಿರುವರೆ ಬೇಕಾದರೆ ಐಚ್ಛಿಕವಾಗಿದೆ
az login

azd auth status
```

### ನಿಯೋಜನೆ ವಿಫಲತೆಗಳು
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### ಸಂಪನ್ಮೂಲ ತಕರಾರುಗಳು
```bash
azd down --force --purge
azd env new different-name
azd up
```

### ಕ್ವೋಟಾ ಮೀರಿದೆ
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 ದೋಷ ಕೋಡ್ ಉಲ್ಲೇಖ

| ದೋಷ | ಕಾರಣ | ಪರಿಹಾರ |
|-------|-------|----------|
| `AuthenticationError` | ಲಾಗಿನ್ ಆಗಿಲ್ಲ | `azd auth login` |
| `ResourceNotFound` | ಸಂಪನ್ಮೂಲ ಲಭ್ಯವಿಲ್ಲ | ಸಂಪನ್ಮೂಲ ಹೆಸರು ಪರಿಶೀಲಿಸಿ |
| `QuotaExceeded` | ಫಲಾನುಭವಿಗಳ ಮಿತಿಗಳು | ಕ್ವೋಟಾ ಹೆಚ್ಚಿಸಲು ವಿನಂತಿ ಮಾಡಿ |
| `InvalidTemplate` | Bicep ವಾಕ್ಯರಚನೆ ದೋಷ | `az bicep build` |
| `Conflict` | ಸಂಪನ್ಮೂಲ ಈಗಾಗಲೇ ಇದೆ | ಹೊಸ ಹೆಸರು ಬಳಸಿ ಅಥವಾ ಅಳಿಸಿ |
| `Forbidden` | ಅನುಮತಿಗಳು ಇಲ್ಲದಿವೆ | RBAC ಪಾತ್ರಗಳನ್ನು ಪರಿಶೀಲಿಸಿ |

## 🔄 ಮರುಹೊಂದಿಕೆ ಮತ್ತು ಪುನಶ್ಚೇತನ

```bash
# ಸಾಫ್ಟ್ ರೆಸೆಟ್ (ಸಂಪನ್ಮೂಲಗಳನ್ನು ಉಳಿಸಿ, ಕೋಡ್ ಮರುಪ್ರವೇಶಿಸಿ)
azd deploy --force

# ಹಾರ್ಡ್ ರೆಸೆಟ್ (ಎಲ್ಲವನ್ನು ಅಳಿಸಿ, ಹೊಸದಾಗಿ ಪ್ರಾರಂಭಿಸಿ)
azd down --force --purge
azd up
```
