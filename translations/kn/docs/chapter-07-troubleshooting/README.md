# ಅಧ್ಯಾಯ 7: ಸಮಸ್ಯೆ ಪರಿಹಾರ ಮತ್ತು ಡಿಬಗ್

**📚 ಕೋರ್ಸ್**: [AZD ಆರಂಭಿಕರಿಗೆ](../../README.md) | **⏱️ ಅವಧಿ**: 1-1.5 ಗಂಟೆ | **⭐ ಜಟಿಲತೆ**: ಮಧ್ಯಮ

---

## ಅವಲೋಕನ

ಈ ಅಧ್ಯಾಯವು Azure Developer CLI ಜೊತೆ ಕೆಲಸ ಮಾಡುವಾಗ ಸಾಮಾನ್ಯವಾಗಿ ಎದುರಾಗುವ ಸಮಸ್ಯೆಗಳನ್ನು diagnoze ಮಾಡಿ ನಿಭಾಯಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ. ನಿಯೋಜನೆ ವೈಫಲ್ಯಗಳಿಂದ AI-ಸಂಬಂಧಿತ ಸಮಸ್ಯೆಗಳವರೆಗೆ ಒಳಗಾಗಿವೆ.

## ಕಲಿಕಾ ಉದ್ದೇಶಗಳು

ಈ ಅಧ್ಯಾಯವನ್ನು ಪೂರ್ಣಗೊಳಿಸಿದರೆ, ನೀವು:
- AZD ನಿಯೋಜನೆ ವೈಫಲ್ಯಗಳನ್ನು ವಿಶ್ಲೇಷಿಸಬಹುದು
- ಪ್ರಾಮಾಣೀಕರಣ ಮತ್ತು ಅನುಮತಿ ಸಂಬಂಧಿ ಸಮಸ್ಯೆಗಳನ್ನು ಡಿಬಗ್ ಮಾಡಬಹುದು
- AI ಸೇವೆ ಸಂಪರ್ಕ ಸಮಸ್ಯೆಗಳನ್ನು ಪರಿಹರಿಸಬಹುದು
- ಸಮಸ್ಯೆ ಪರಿಹಾರದಿಗಾಗಿ Azure ಪೋರ್ಟಲ್ ಮತ್ತು CLI ಅನ್ನು ಬಳಸಬಹುದು

## 🚨 ದ್ರುತ ಪರಿಹಾರಗಳು

### ಪ್ರಾಮಾಣೀಕರಣ ಸಮಸ್ಯೆಗಳು
```bash
azd auth login
az login
azd auth whoami
```

### ಪ್ರೊವಿಷನಿಂಗ್ ವೈಫಲ್ಯಗಳು
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### ಸಂಪನ್ಮೂಲ ಸಂಘರ್ಷಗಳು
```bash
azd down --force --purge
azd env new different-name
azd up
```

### ಕೋಟಾ ಮೀರಿದೆ
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 ದೋಷ ಕೋಡ್ ಉಲ್ಲೇಖ

| Error | Cause | Solution |
|-------|-------|----------|
| `AuthenticationError` | ಲಾಗಿನ್ ಆಗಿಲ್ಲ | `azd auth login` |
| `ResourceNotFound` | ಸಂಪನ್ಮೂಲ ಕಾಣೆಯಾಗಿಲ್ಲ | ಸಂಪನ್ಮೂಲ ಹೆಸರನ್ನು ಪರಿಶೀಲಿಸಿ |
| `QuotaExceeded` | ಸಬ್ಸ್ಕ್ರಿಪ್ಷನ್ ಮಿತಿಗಳು | ಕೋಟಾ ಹೆಚ್ಚಿಸುವಂತೆ ವಿನಂತಿಸಿ |
| `InvalidTemplate` | Bicep ವಾಕ್ಯರಚನೆ ದೋಷ | `az bicep build` |
| `Conflict` | ಸಂಪನ್ಮೂಲ ಈಗಾಗಲೇ არსებობს | ಹೊಸ ಹೆಸರನ್ನು ಉಪಯೋಗಿಸಿ ಅಥವಾ ಅಳಿಸಿ |
| `Forbidden` | ಅನುಮತಿಗಳು ಅಪರ್ಯಾಪ್ತವಾಗಿದೆ | RBAC ಪಾತ್ರಗಳನ್ನು ಪರಿಶೀಲಿಸಿ |

## 🔄 ರೀಸೆಟ್ ಮತ್ತು ಪುನರುದ್ಧಾರ

```bash
# ಸಾಫ್ಟ್ ರಿಸೆಟ್ (ಸಂಪನ್ಮೂಲಗಳನ್ನು ಉಳಿಸಿ, ಕೋಡ್ ಅನ್ನು ಮರುನಿಯೋಜಿಸು)
azd deploy --force

# ಕಠಿಣ ರಿಸೆಟ್ (ಎಲ್ಲವನ್ನೂ ಅಳಿಸಿ, ಹೊಸದಾಗಿ ಪ್ರಾರಂಭಿಸಿ)
azd down --force --purge
azd up
```
