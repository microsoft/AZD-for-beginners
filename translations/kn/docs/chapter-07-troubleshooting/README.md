# ಅಧ್ಯಾಯ 7: ಸಮಸ್ಯೆ ಪರಿಹಾರ ಮತ್ತು ಡಿಬಗ್ ಮಾಡುವಿಕೆ

**📚 ಕೋರ್ಸ್**: [ಆರಂಭಿಕರಿಗಾಗಿ AZD](../../README.md) | **⏱️ ಅವಧಿ**: 1-1.5 ಗಂಟೆಗಳು | **⭐ ಸಂಕೀರ್ಣತೆ**: ಮಧ್ಯಮ

---

## ಅವಲೋಕನ

ಈ ಅಧ್ಯಾಯವು Azure ಡೆವಲಪರ್ CLI ಬಳಕೆ ಮಾಡುವಾಗ ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳನ್ನು ಗುರುತಿಸಿ ಪರಿಹರಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ. ನಿಯೋಜನೆ ವಿಫಲತೆಗಳಿಂದ AI-ಸಂಬಂಧಿತ ಸಮಸ್ಯೆಗಳವರೆಗೆ.

> ಮಾರಿ 2026 ರಲ್ಲಿ `azd 1.23.12` ವಿರುದ್ಧ ಮಾನ್ಯಗೊಳಿಸಲಾಗಿದೆ.

## ಕಲಿಕಾ ಉದ್ದೇಶಗಳು

ಈ ಅಧ್ಯಾಯವನ್ನು ಪೂರ್ಣಗೊಳಿಸುವ ಮೂಲಕ ನೀವು:
- ಸಾಮಾನ್ಯ AZD ನಿಯೋಜನೆ ವಿಫಲತೆಗಳನ್ನು ಗುರುತಿಸುವಿರಿ
- ಪ್ರಾಮಾಣೀಕರಣ ಮತ್ತು ಪರವಾನಗಿ ಸಮಸ್ಯೆಗಳನ್ನು ಡಿಬಗ್ ಮಾಡುವಿರಿ
- AI ಸೇವೆ ಸಂಪರ್ಕ ಸಮಸ್ಯೆಗಳನ್ನು ಪರಿಹರಿಸುವಿರಿ
- ಸಮಸ್ಯೆ ಪರಿಹಾರದಿಗಾಗಿ Azure ಪೋರ್ಟಲ್ ಮತ್ತು CLI ಬಳಕೆ ಮಾಡುವಿರಿ

## 🚨 ತ್ವರಿತ ಪರಿಹಾರಗಳು

### ಪ್ರಾಮಾಣೀಕರಣ ಸಮಸ್ಯೆಗಳು
```bash
# AZD ವರ್ಕ್‌ಫ್ಲೋಗಳಿಗಾಗಿ ಅಗತ್ಯ
azd auth login

# ನೀವು ನೇರವಾಗಿ Azure CLI ಆಜ್ಞೆಗಳನ್ನು ಬಳಸುತ್ತಿದ್ದರೆ ಐಚ್ಛಿಕವಾಗಿ
az login

azd auth status
```

### ಪ್ರೊವಿಷನಿಂಗ್ ವಿಫಲತೆಗಳು
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### ಸಂಪನ್ಮೂಲ ಸಂಘರ್ಷ
```bash
azd down --force --purge
azd env new different-name
azd up
```

### ಕೊಟಾ ಮೀರಿದ ತಲುಪಿಕೆ
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 ದೋಷ ಕೋಡ್ ಉಲ್ಲೇಖ

| ದೋಷ | ಕಾರಣ | ಪರಿಹಾರ |
|-------|-------|----------|
| `AuthenticationError` | ಲಾಗಿನ್ ಆಗಿಲ್ಲ | `azd auth login` |
| `ResourceNotFound` | ಸಂಪನ್ಮೂಲ ಕಾಣಿಸಿಲ್ಲ | ಸಂಪನ್ಮೂಲ ಹೆಸರುಗಳನ್ನು ಪರಿಶೀಲಿಸಿ |
| `QuotaExceeded` | ಸಬ್ಸ್ಕ್ರಿಪ್ಶನ್ ಮಿತಿಗಳು ಮೀರಿಗೆ | ಕೊಟಾ ಹೆಚ್ಚಿಸುವಂತೆ ಕೇಳಿ |
| `InvalidTemplate` | Bicep ವಾಕ್ಯ ರಚನಾ ತಪ್ಪು | `az bicep build` |
| `Conflict` | ಸಂಪನ್ಮೂಲ ಇರುತ್ತದೆ | ಹೊಸ ಹೆಸರು ಬಳಸಿ ಅಥವಾ ಅಳಿಸಿ |
| `Forbidden` | ಪರವಾನಗಿ ಕಡಿಮೆಯಿದೆ | RBAC ಪಾತ್ರಗಳನ್ನು ಪರಿಶೀಲಿಸಿ |

## 🔄 ಮರುಹೊಂದಿಕೆ ಮತ್ತು ಪುನರ್ ಪಡೆಯುವಿಕೆ

```bash
# ಸಾಫ್ಟ್ ರಿಸೆಟ್ (ಸಂಪನ್ಮೂಲಗಳನ್ನು ಉಳಿಸು, ಕೋಡ್ ಅನ್ನು ಪುನರ್‌ನಿಯೋಜಿಸು)
azd deploy --force

# ಹಾರ್ಡ್ ರಿಸೆಟ್ (ಎಲ್ಲವನ್ನೂ ಅಳಿಸಿ, ಹೊಸದಾಗಿ ಪ್ರಾರಂಭಿಸು)
azd down --force --purge
azd up
```
