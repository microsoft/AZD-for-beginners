## ಪರಿಚಯ

ನಿಮ್ಮ ಮೊದಲ ಆಜೂರ್ ಡೆವೆಲಪರ್ CLI ಪ್ರಾಜೆಕ್ಟಿಗೆ ಸ್ವಾಗತ! ಈ ಸಮಗ್ರ ಹ್ಯಾಂಡ್ಸ್-ಆನ್ ಟ್ಯುಟೋರಿಯಲ್ ಆಜ್ಡ್ನಿಂದ ಆಜೂರ್‌ನಲ್ಲಿ ಪೂರ್ಣಸ್ಟ್ಯಾಕ್ ಅನ್ವಯವನ್ನು ರಚಿಸುವುದು, ವಿತರಿಸುವುದು ಮತ್ತು ನಿರ್ವಹಿಸುವುದರ ಸಂಪೂರ್ಣ ಪ್ರವಾಸವನ್ನು ಒದಗಿಸುತ್ತದೆ. ನೀವು ರಿಯಲ್ ಟೋಡೋ ಅನ್ವಯವನ್ನು ಉಪಯೋಗಿಸುವಿರಿ, ಇದರಲ್ಲಿ ರಿಯಾಕ್ಟ್ ಫ್ರಂಟ್‌ಎಂಡ್, ನೋಡ್.js API ಬ್ಯಾಕೆಂಡ್ ಮತ್ತು MongoDB ಡೇಟಾಬೇಸ್ ಸೇರಿವೆ.

## ಅಧ್ಯಯನ ಗುರಿಗಳು

ಈ ಟ್ಯುಟೋರಿಯಲ್ ಅನ್ನು ಪೂರ್ಣಗೊಳಿಸಿದ ಮೇಲೆ, ನೀವು:
- ಟೆಂಪ್ಲೇಟುಗಳನ್ನು ಉಪಯೋಗಿಸಿ ಆಜಡ್ ಪ್ರಾಜೆಕ್ಟ್ ಪ್ರಾರಂಭ Workflow ನ ಮಾಸ್ಟರ್ ಆಗಿ
- ಆಜೂರ್ ಡೆವೆಲಪರ್ CLI ಪ್ರಾಜೆಕ್ಟ್ ರಚನೆ ಮತ್ತು configuração ಫೈಲ್‌ಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಿ
- ರಚನೆಯೊಂದಿಗೆ ಸಂಪೂರ್ಣ ಅನ್ವಯ ವಿತರಣೆಯನ್ನು ಆಜೂರ್‌ಗೆ ಕಾರ್ಯಗತಗೊಳಿಸಿ
- ಅನ್ವಯದ ಅಪ್ಡೇಟ್‌ಗಳು ಮತ್ತು ಮರುವಿತರಣೆ ಪ್ರಕ್ರಿಯೆಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ
- ಅಭಿವೃದ್ಧಿ ಮತ್ತು ಸ್ಟೇಜಿಂಗ್‌ಗಾಗಿ ಬහු ಪರಿಸರಗಳನ್ನು ನಿರ್ವಹಿಸಿ
- ಸಂಪನ್ಮೂಲ ವಿಸರ್ಜನೆ ಮತ್ತು ವೆಚ್ಚ ನಿರ್ವಹಣಾ ಅಭ್ಯಾಸಗಳನ್ನು ಅನ್ವಯಿಸಿ

## ಅಧ್ಯಯನ ಫಲಿತಾಂಶಗಳು

ಪೂರ್ಣಗೊಳಿಸಿದ ನಂತರ, ನೀವು:
- ಸ್ವತಂತ್ರವಾಗಿ ಟೆಂಪ್ಲೇಟಿನಿಂದ ಆಜಡ್ ಪ್ರಾಜೆಕ್ಟ್‌ಗಳನ್ನು ಪ್ರಾರಂಭಿಸಿ ಮತ್ತು ಸಂರಚಿಸಬಹುದು
- ಆಜಡ್ ಪ್ರಾಜೆಕ್ಟ್ ರಚನೆಗಳನ್ನು ಪರಿಣಾಮಕಾರಿಯಾಗಿ ಸಂಚರಿಸಬಹುದು ಮತ್ತು ಬದಲಾಯಿಸಬಹುದು
- ಸಿಂಗಲ್ ಕಮಾಂಡ್ ಗಳ ಉಪಯೋಗದಿಂದ ಪೂರ್ಣಸ್ಟ್ಯಾಕ್ ಅನ್ವಯಗಳನ್ನು ಆಜೂರ್‌ಗೆ ವಿತರಿಸಬಹುದು
- ಸಾಮಾನ್ಯ ವಿತರಣೆ ಸಮಸ್ಯೆಗಳು ಮತ್ತು ಪ್ರಮಾಣೀಕರಣ ಸಮಸ್ಯೆಗಳನ್ನು ಪರಿಹರಿಸಬಹುದು
- ವಿಭಿನ್ನ ವಿತರಣಾ ಹಂತಗಳಿಗಾಗಿ ಬಹು ಆಜೂರ್ ಪರಿಸರಗಳನ್ನು ನಿರ್ವಹಿಸಬಹುದು
- ಅನ್ವಯದ ನವೀಕರಣಗಳಿಗಾಗಿ ನಿರಂತರ ವಿತರಣಾ ವರ್ಕ್ಫ್ಲೋಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಬಹುದು

## ಹಂತ 1: ಟೆಂಪ್ಲೇಟು ಆಯ್ಕೆಮಾಡಿ ಮತ್ತು ಪ್ರಾರಂಭಿಸಿ

ರಿಯಾಕ್ಟ್ ಫ್ರಂಟ್‌ಎಂಡ್ ಮತ್ತು ನೋಡ್.js API ಬ್ಯಾಕೆಂಡ್ ಒಳಗೊಂಡ ಜನಪ್ರಿಯ ಟೋಡೋ ಅನ್ವಯ ಟೆಂಪ್ಲೇಟಿನಿಂದ ಪ್ರಾರಂಭಿಸೋಣ.

```bash
# ಲಭ್ಯವಿರುವ ಟೆಂಪ್ಲೇಟುಗಳನ್ನು ಬ್ರೌಸ್ ಮಾಡಿ
azd template list

# ಟುಡೂ ಅಪ್ಲಿಕೇಶನ್ ಟೆಂಪ್ಲೇಟನ್ನು ಪ್ರಾರಂಭಿಸಿ
mkdir my-first-azd-app
cd my-first-azd-app
azd init --template todo-nodejs-mongo

# ಸೂಚನೆಗಳನ್ನು ಅನುಸರಿಸಿ:
# - ಒಂದು ಪರಿಸರದ ಹೆಸರು ನಮೂದಿಸಿ: "dev"
# - ಒಂದು ಸಬ್ಸ್ಕ್ರಿಪ್‌ಷನ್ ಆಯ್ಕೆಮಾಡಿ (ನಿಮಗೆ ಬಹುಗಳು ಇದ್ದರೆ)
# - ಒಂದು ಪ್ರಾಂತ್ಯವನ್ನು ಆಯ್ಕೆಮಾಡಿ: "East US 2" (ಅಥವಾ ನಿಮ್ಮ ಇಚ್ಛೆಯ ಪ್ರಾಂತ್ಯ)
```

### ಈಗೇನಾದರು ನಡೆದಿದೆ?
- ಟೆಂಪ್ಲೇಟು ಕೋಡ್ ನಿಮ್ಮ ಸ್ಥಳೀಯ ಡೈರೆಕ್ಟರಿಗೆ ಡೌನ್‌ಲೋಡ್ ಮಾಡಲಾಗಿದೆ
- ಸೇವಾ ವ್ಯಾಖ್ಯಾನಗಳೊಂದಿಗೆ `azure.yaml` ಫೈಲ್ ರಚಿಸಲಾಗಿದೆ
- `infra/` ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ ಮೂಲಸೌಕರ್ಯ ಕೋಡ್ ಸಿದ್ಧಪಡಿಸಲಾಗಿದೆ
- ಪರಿಸರ ಕ್ರಮವಿಧಾನ ಸಿದ್ಧಪಡಿಸಲಾಗಿದೆ

## ಹಂತ 2: ಪ್ರಾಜೆಕ್ಟ್ ರಚನೆಯನ್ನು ಅನ್ವೇಷಿಸಿ

ಅಜಡ್ ನಮಗಾಗಿ ಏನು ರಚಿಸಿದೆ ಎಂದು ನೋಡೋಣ:

```bash
# ಪ್ರಾಜೆಕ್ಟ್ ರಚನೆ ವೀಕ್ಷಿಸಿ
tree /f   # ವಿಂಡೋಸ್
# ಅಥವಾ
find . -type f | head -20   # ಮ್ಯಾಕ್‌ಒಎಸ್/ಲಿನಕ್ಸ
```

ನೀವು ಕಾಣಬಹುದು:
```
my-first-azd-app/

│   ├── main.bicep              # Main infrastructure template
│   ├── main.parameters.json     # Parameters for deployment
│   └── modules/                # Reusable infrastructure modules
├── src/
│   ├── api/                    # Node.js backend API
│   │   ├── src/               # API source code
│   │   ├── package.json       # Node.js dependencies
│   │   └── Dockerfile         # Container configuration
│   └── web/                   # React frontend
│       ├── src/               # React source code
│       ├── package.json       # React dependencies
│       └── Dockerfile         # Container configuration
```

### ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬೇಕಾದ ಮುಖ್ಯ ಫೈಲ್‌ಗಳು

**azure.yaml** - ನಿಮ್ಮ ಆಜಡ್ ಪ್ರಾಜೆಕ್ಟ್‌ನ ಹೃದಯ:

ವಿತರಿಸುವ ಮುಂಚೆ, ನೀವು ಅನ್ವಯವನ್ನು ಕಸ್ಟಮೈಸ್ ಮಾಡಬಹುದು:

### ಫ್ರಂಟ್‌ಎಂಡ್ ಅನ್ನು ಬದಲಾಯಿಸಿ

```bash
# React ಅಪ್ಲಿಕೇಶನ್ ಘಟಕವನ್ನು ತೆರೆಯಿರಿ
```

## ಹಂತ 4: ಆಜೂರ್‌ಗೆ ವಿತರಿಸಿ

ಇವರಿಗೆ ಸಂಪೂರ್ಣವಾದ ಹಂತ - ಎಲ್ಲವನ್ನೂ ಆಜೂರ್‌ಗೆ ವಿತರಿಸೋಣ!

```bash
# ಮೂಲಸೌಕರ್ಯ ಮತ್ತು ಅನ್ವಯಿಕೆಯನ್ನು ನಿಯೋಜಿಸಿ
azd up

# ಈ ಆಜ್ಞೆ ಮಾಡುವುದು:
# 1. ಅಜೂರ್ ಸಂಪನ್ಮೂಲಗಳನ್ನು Provision ಮಾಡುವುದು (ಅ್ಯಪ್ ಸೇವೆ, ಕೋಸ್ಮೋಸ್ ಡಿಬಿ, ಇತ್ಯಾದಿ)
# 2. ನಿಮ್ಮ ಅನ್ವಯಿಕೆಯನ್ನು ನಿರ್ಮಿಸಿ
# 3. Provision ಮಾಡಲಾದ ಸಂಪನ್ಮೂಲಗಳಿಗೆ ನಿಯೋಜಿಸಿ
# 4. ಅನ್ವಯಿಕೆಯ URL ಅನ್ನು ಪ್ರದರ್ಶಿಸಿ
```

### ವಿತರಣೆಯ ಸಮಯದಲ್ಲಿ ಏನು ನಡೆಯುತ್ತದೆ?

`azd up` ಕಮಾಂಡ್ ಈ ಹಂತಗಳನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ:
1. **Provision** (`azd provision`) - ಆಜೂರ್ ಸಂಪನ್ಮೂಲಗಳನ್ನು ರಚಿಸುತ್ತದೆ
2. **Package** - ನಿಮ್ಮ ಅನ್ವಯ ಕೋಡ್ ನಿರ್ಮಿಸುತ್ತದೆ
3. **Deploy** (`azd deploy`) - ಆಜೂರ್ ಸಂಪನ್ಮೂಲಗಳಿಗೆ ಕೋಡ್ ವಿತರಿಸುತ್ತದೆ

## ಹಂತ 5: ನಿಮ್ಮ ಅನ್ವಯವನ್ನು ಪರೀಕ್ಷಿಸಿ

### ನಿಮ್ಮ ಅನ್ವಯಕ್ಕೆ ಪ್ರವೇಶಿಸಿ
ವಿತರಣಾ output ನಲ್ಲಿ ನೀಡಲಾದ URL ಕ್ಲಿಕ್ ಮಾಡಿ ಅಥವಾ ಯಾವುದೇ ಸಮಯದಲ್ಲಿ ಪಡೆಯಿರಿ:
```bash
# ಅನ್ವಯಿಕೆಯ ಎಂಡ್ಪಾಯಿಂಟ್ ಗಳನ್ನು ಪಡೆಯಿರಿ
azd show

# ನಿಮ್ಮ ಬ್ರೌಸರ್ ನಲ್ಲಿ ಅನ್ವಯಿಕೆಯನ್ನು ತೆರೆಯಿರಿ
azd show --output json | jq -r '.services.web.endpoint'
```

### ಟೋಡೋ ಅನ್ವಯವನ್ನು ಪರೀಕ್ಷಿಸಿ
1. **ಟೋಡೋ ಐಟಂ ಸೇರಿಸಿ** - "Add Todo" ಕ್ಲಿಕ್ ಮಾಡಿ ಮತ್ತು ಕಾರ್ಯವನ್ನು ಹಾಕಿ
2. **ಪೂರ್ಣಗೊಂಡಂತೆ ಗುರುತಿಸಿ** - ಪೂರ್ಣಗೊಂಡ ವಸ್ತುಗಳ ಮೇಲೆ ಟಿಕ್ ಮಾಡಿ
3. **ಐಟಂಗಳನ್ನು ಅಳಿಸಿ** - ಅನಗತ್ಯವಾದ ಟೋಡೋಗಳನ್ನು ತೆಗೆಯಿರಿ

### ನಿಮ್ಮ ಅನ್ವಯವನ್ನು ಗಮನಿಸಿ
```bash
# ನಿಮ್ಮ ಸಂಪನ್ಮೂಲಗಳಿಗಾಗಿ ಅಜೂರ್ ಪೋರ್ಟಲ್ ತೆರೆಯಿರಿ
azd monitor

# ಅಪ್ಲಿಕೇಶನ್ ಲಾಗ್‌ಗಳನ್ನು ವೀಕ್ಷಿಸಿ
azd monitor --logs

# ಲೈವ್ ಮೆಟ್ರಿಕ್‌ಗಳನ್ನು ವೀಕ್ಷಿಸಿ
azd monitor --live
```

## ಹಂತ 6: ಬದಲಾವಣೆ ಮಾಡಿ ಮತ್ತು ಮರುವಿತರಿಸಿ

ಬದಲಾವಣೆ ಮಾಡಿ ನೋಡೋಣ, ಹಾಗೂ ಹೇಗೆ ಸುಲಭವಾಗಿ ನವೀಕರಿಸಬಹುದು:

### API ಬದಲಾಯಿಸಿ
```bash
# API ಕೋಡ್ ಸಂಪಾದಿಸಿ
code src/api/src/routes/lists.js
```

ಕಸ್ಟಮ್ ಪ್ರತಿಕ್ರಿಯಾ ಹೆಡರ್ ಸೇರಿಸಿ:
```javascript
// ಒಂದು ಮಾರ್ಗ ಹ್ಯಾಂಡ್ಲರ್ ಅನ್ನು ಕಂಡುಹಿಡಿದು ಸೇರಿಸಿ:
res.header('X-Powered-By', 'Azure Developer CLI');
```

### ಕೇವಲ ಕೋಡ್ ಬದಲಾವಣೆಗಳನ್ನು ವಿತರಿಸಿ
```bash
# ಅಪ್ಲಿಕೇಶನ್ ಕೋಡ್ ಮಾತ್ರ ನಿಯೋಜಿಸಿ (ಭೌತವೌಡಾಪನವನ್ನು ದಾಟಿಸಿ)
azd deploy

# ಮೂಲಸೌಕರ್ಯ ಈಗಾಗಲೇ ಇರುವುದರಿಂದ ಇದು 'azd up'ಕ್ಕಿಂತ ಬಹಳ ವೇಗವಾಗಿದೆ
```

## ಹಂತ 7: ಬಹು ಪರಿಸರಗಳನ್ನು ನಿರ್ವಹಿಸಿ

ಉತ್ಪಾದನ ಮುಂಚೆ ಪರೀಕ್ಷೆಗಾಗಿ ಸ್ಟೇಜಿಂಗ್ ಪರಿಸರವನ್ನು ರಚಿಸಿ:

```bash
# ಹೊಸ ಸ್ಟೇಜಿಂಗ್ ಪರಿಸರವನ್ನು ರಚಿಸಿ
azd env new staging

# ಸ್ಟೇಜಿಂಗ್‌ಗೆ ನಿಯೋಜಿಸಿ
azd up

# ಡೆವ್ ಪರಿಸರಕ್ಕೆ ಹಿಂದಿರುಗಿ
azd env select dev

# ಎಲ್ಲಾ ಪರಿಸರಗಳನ್ನು ಪಟ್ಟಿ ಮಾಡಿ
azd env list
```

### ಪರಿಸರ ಹೋಲಿಕೆ
```bash
# ಡೆವ್ ಪರಿಸರವನ್ನು ವೀಕ್ಷಿಸಿ
azd env select dev
azd show

# ಸ್ತರೀಕರಣ ಪರಿಸರವನ್ನು ವೀಕ್ಷಿಸಿ
azd env select staging
azd show
```

## ಹಂತ 8: ಸಂಪನ್ಮೂಲಗಳನ್ನು ಶುದ್ಧಗೊಳಿಸಿ

ಪ್ರಯೋಗ ಮುಗಿದ ಬಳಿಕ ಮುಂದುವರೆದ ಶುಲ್ಕ ತಪ್ಪಿಸಲು ಶುದ್ಧಗೊಳಿಸಿ:

```bash
# ಪ್ರಸ್ತುತ ವಾತಾವರಣಕ್ಕಾಗಿ ಎಲ್ಲಾ ಅಜೂರ್ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಅಳಿಸಲು
azd down

# ದೃಢೀಕರಣವಿಲ್ಲದೆ ಬಲವಂತದಿಂದ ಅಳಿಸಿ ಮತ್ತು ಸಾಫ್ಟ್-ಅಳಿಸಲಾದ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಶುದ್ಧಮಾಡಿ
azd down --force --purge

# ನಿರ್ದಿಷ್ಟ ವಾತಾವರಣವನ್ನು ಅಳಿಸಿ
azd env select staging
azd down --force --purge
```

## ಪರಂಪರೆಯ ಅಪ್ಲಿಕೇಶನ್ ವಿರುದ್ಧ AI-ಚಾಲಿತ ಅಪ್ಲಿಕೇಶನ್: ಒಂದೇ Workflow

ನೀವು ನಿನ್ನೆಕ್ಲಾಸಿಕ್ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್ ವಿತರಣೆ ಮಾಡಿದ್ದೀರಿ. ಆದರೆ ನೀವು ಏಐ-ಚಾಲಿತ ಅಪ್ಲಿಕೇಶನ್ ವಿತರಿಸಲು ಬಯಸಿದ್ರೆ - ಉದಾಹರಣೆಗೆ, Microsoft Foundry Models ಬೆಂಬಲಿಸಿದ ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್?

ಸುಂದರ ಸುದ್ದಿ: **Workflow ಅದೇ ಇದೆ.**

| ಹಂತ | ಪರಂಪರೆಯ ಟೋಡೋ ಅಪ್ | ಏಐ ಚಾಟ್ ಅಪ್ |
|------|-------------------|-------------|
| ಪ್ರಾರಂಭಿಸಿ | `azd init --template todo-nodejs-mongo` | `azd init --template azure-search-openai-demo` |
| ಪ್ರಮಾಣೀಕರಣ | `azd auth login` | `azd auth login` |
| ವಿತರಿಸಿ | `azd up` | `azd up` |
| ಗಮನಿಸಿ | `azd monitor` | `azd monitor` |
| ಶುದ್ಧಗೊಳಿಸಿ | `azd down --force --purge` | `azd down --force --purge` |

ಒಂದು ಏಕೈಕ ವ್ಯತ್ಯಾಸವೆಂದರೆ **ಟೆಂಪ್ಲೇಟು** ನೀವು ಪ್ರಾರಂಭಿಸುವಂತಹುದು. ಏಐ ಟೆಂಪ್ಲೇಟು ಹೆಚ್ಚುವರಿ ಮೂಲಸೌಕರ್ಯ ಸೇರಿದಂತೆ ಇದೆ (ಉದಾ. Microsoft Foundry Models ಸಂಪನ್ಮೂಲ ಅಥವಾ ಏಐ ಶೋಧ ಸೂಚ್ಯಂಕ), ಆದರೆ ಆಜ್ಡ್ ಅದನ್ನು ನಿಮ್ಮ ಬದ್ಧತೆಯಲ್ಲಿ ನಿರ್ವಹಿಸುತ್ತದೆ. ನೀವು ಹೊಸ ಕಮಾಂಡ್‌ಗಳನ್ನು ಕಲಿಯಬೇಕಾಗಿಲ್ಲ, ಬೇರೆ ಉಪಕರಣವನ್ನು ಉಪಯೋಗಿಸಬೇಕಾಗಿಲ್ಲ, ಅಥವಾ ವಿತರಣೆ ಕುರಿತು ನಿಮ್ಮ ಚಿಂತನೆಯನ್ನು ಬದಲಾಯಿಸುವ ಅಗತ್ಯವಿಲ್ಲ.

ಇದು ಆಜಡ್‌ನ ಮೂಲ ಸಿದ್ಧಾಂತ: **ಒಂದು workflow, ಯಾವುದೇ ಕೆಲಸ.** ನೀವು ಈ ಟ್ಯುಟೋರಿಯಲ್‌ನಲ್ಲಿ ಅಭ್ಯಾಸ ಮಾಡಿದ ಕೌಶಲ್ಯಗಳು — ಪ್ರಾರಂಭ, ವಿತರಣೆಯು, ಗಮನಿಸುವುದು, ಮರು ವಿತರಣೆಯು ಮತ್ತು ಶುದ್ಧಗೊಳಿಸುವುದು — ಏಐ ಅಪ್ಲಿಕೇಶನ್‌ಗಾಗಿಯೂ ಮತ್ತು ಏಜೆಂಟ್‌ಗಳಿಗೆ ಸಹ ಅನ್ವಯಿಸುತ್ತದೆ.

**ಗುರುತಿಸು**: `azd init` ಮತ್ತು ವಿತರಣಾ Workflow ನ‌ನಲ್ಲಿ ಪರಿಣತಿ ಪ್ರದರ್ಶಿಸಲು

```bash
# Python + MongoDB ಸ್ಟಾಕ್ ಪ್ರಯತ್ನಿಸಿ
mkdir todo-python && cd todo-python

azd up

# ನಿಯೋಜನೆಯನ್ನು ಪರಿಶೀಲಿಸಿ
azd show
```

### ವ್ಯಾಯಾಮ 2: ಕಸ್ಟಮೈಸ್ ರಚನೆ (20 ನಿಮಿಷ)
**ಗುರುತಿಸು**: ಪರಿಸರ ಚರಗಳ ಸಂರಚನಾ ಅಭ್ಯಾಸ

```bash
cd my-first-azd-app

# ಕಸ್ಟಮ್ ಪರಿಸರವನ್ನು ರಚಿಸಿ
azd env new custom-config

# ಕಸ್ಟಮ್ ಹರವುಗಳ ಹಂಚಿಕೆ ಮಾಡಿ
azd env set APP_TITLE "My Custom Todo App"

azd env set ENABLE_DEBUG "true"

# ಹರವುಗಳ ಪರಿಶೀಲಿಸಿ
azd env get-values | grep APP_TITLE

# ಕಸ್ಟಮ್ ಸಂರಚನೆಯೊಂದಿಗೆ ನಿಯೋಜಿಸಿ
azd up
```

**ಯಶಸ್ಸಿನ ಮಾನದಂಡಗಳು:**

- [ ] ಪರಿಸರ ಚರಗಳನ್ನು ಹೊಂದಿಸಲಾಗಿದೆ ಮತ್ತು ಪಡೆಯಬಹುದು

### ವ್ಯಾಯಾಮ 3: ಬಹು-ಪರಿಸರ Workflow (25 ನಿಮಿಷ)
**ಗುರುತಿಸು**: ಪರಿಸರ ನಿರ್ವಹಣೆ ಮತ್ತು ವಿತರಣೆ ತಂತ್ರಗಳನ್ನು ಮಾಸ್ಟರ್ ಮಾಡುವುದು

```bash
# ಡೆವ್ ಪರಿಸರವನ್ನು ರಚಿಸಿ
azd env new dev-$(whoami)
azd env set ENVIRONMENT_TYPE dev
azd env set LOG_LEVEL debug
azd up

# ಡೆವ್ URL ಅನ್ನು ಗಮನಿಸಿ
DEV_URL=$(azd show --output json | jq -r '.services.web.endpoint')
echo "Dev: $DEV_URL"

# ಸ್ಟೇಜಿಂಗ್ ಪರಿಸರವನ್ನು ರಚಿಸಿ
azd env new staging-$(whoami)
azd env set ENVIRONMENT_TYPE staging
azd env set LOG_LEVEL info
azd up

# ಸ್ಟೇಜಿಂಗ್ URL ಅನ್ನು ಗಮನಿಸಿ
STAGING_URL=$(azd show --output json | jq -r '.services.web.endpoint')
echo "Staging: $STAGING_URL"

# ಪರಿಸರಗಳನ್ನು ಹೋಲಿಸಿ
azd env list

# ಇಬ್ಬರೂ ಪರಿಸರಗಳನ್ನು ಪರೀಕ್ಷಿಸಿ
curl "$DEV_URL/health"
curl "$STAGING_URL/health"

# ಎರಡನ್ನೂ ಸ್ವಚ್ಛಗೊಳಿಸಿ
azd env select dev-$(whoami) && azd down --force --purge
azd env select staging-$(whoami) && azd down --force --purge
```

**ಯಶಸ್ಸಿನ ಮಾನದಂಡಗಳು:**
- [ ] ವಿಭಿನ್ನ ಸಂರಚನೆಗಳೊಂದಿಗೆ ಎರಡು ಪರಿಸರಗಳನ್ನು ರಚಿಸಲಾಗಿದೆ
- [ ] ಎರಡೂ ಪರಿಸರಗಳು ಯಶಸ್ವಿಯಾಗಿ ವಿತರಿಸಲ್ಪಟ್ಟಿವೆ
- [ ] `azd env select` ಬಳಸಿ ಪರಿಸರಗಳ ನಡುವೆ ಬದಲಾಯಿಸಬಹುದು
- [ ] ಪರಿಸರ ಚರಗಳು ಪರಿಸರಗಳ ನಡುವೆ ಭಿನ್ನವಾಗಿವೆ
- [ ] ಎರಡೂ ಪರಿಸರಗಳನ್ನು ಯಶಸ್ವಿಯಾಗಿ ಶುದ್ಧಗೊಳಿಸಲಾಗಿದೆ

### ಸಂಪನ್ಮೂಲಹೆಸರಿನ ಘರ್ಷಣೆಗಳು
```bash
# ವಿಭಿನ್ನ ಪರಿಸರದ ನಾಮವನ್ನು ಬಳಸಿ
azd env new dev-$(whoami)-$(date +%s)
```

### ಪೋರ್ಟ್/ನೆಟ್ಟ워크 ಸಮಸ್ಯೆಗಳು
```bash
# ಪೋರ್ಟ್‌ಗಳು ಲಭ್ಯವಿದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ
netstat -an | grep :3000
netstat -an | grep :3100
```

```bash
# ವರ್ಗದ ಮೂಲಕ ಟೆಂಪ್ಲೇಟ್ಗಳನ್ನು ಬ್ರೌಸ್ ಮಾಡಿ
```
