## ಪರಿಚಯ

[ನಿಮ್ಮ ಮೊದಲ ಯೋಜನೆ](first-project.md)ಯಲ್ಲಿ ನೀವು ಟೆಂಪ್ಲೇಟ್‌ದಿಂದ ಪ್ರಾರಂಭಿಸಿ ಆಪ್ ಅನ್ನು ವಿನ್ಯಾಸಗೊಳಿಸಿದ್ದೀರಿ. ಆದರೆ ಬಹುತೇಕ ಸಮಯ ನೀವು ಈಗಾಗಲೇ *ಒಂದು* ಆಪ್ ಹೊಂದಿರುತ್ತೀರಿ - ಒಂದು Node.js API, Python Flask ಸೇವೆ, ಒಂದು .NET ವೆಬ್ ಆಪ್ - ನಿಮ್ಮ ಯಂತ್ರದಲ್ಲಿ ಒಂದು ಫೋಲ್ಡರ್‌ಗೆ ಇಟ್ಟಿರುವಂತೆ. ಈ ಪಾಠದಲ್ಲಿ ನಿಮ್ಮ ಆ ಪೂರ್ತಿಯಾದ ಕೋಡ್‌ಗೆ azd ಅನ್ನು ಹೇಗೆ ಸೇರಿಸುವುದು ಎಂಬುದು ತೋರಿಸುತ್ತದೆ, ಆದ್ದರಿಂದ ನೀವು `azd up` ಬಳಸಿ ವಿನ್ಯಾಸಗೊಳಿಸಬಹುದು, ಯಾವುದೇ ಟೆಂಪ್ಲೇಟ್ ಅಗತ್ಯವಿಲ್ಲ.

## ಅಧ್ಯಯನ ಗುರಿಗಳು

ಈ ಪಾಠದ ಕೊನೆಯಲ್ಲಿ, ನೀವು:
- azd ಯೋಜನೆಯನ್ನು ಆರಂಭಿಸುವ ಮೂರು ಮಾರ್ಗಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುತ್ತೀರಿ
- ಆಸ್ತಿತ್ವದಲ್ಲಿದ ಕೋಡ್‌ಬೇಸ್ನಲ್ಲಿ `azd init` ಅನ್ನು ಚಾಲನೆ ಮಾಡುತ್ತೀರಿ
- `azure.yaml` ಮತ್ತು `infra/` ಫೋಲ್ಡರ್ ನಿಮ್ಮ ಆಪ್‌ಗೆ ಏನು ಮಾಡುತ್ತದೆಯೋ ತಿಳಿಯುತ್ತದೆ
- azd ರಚಿಸುವ ಮೂಲಸೌಕರ್ಯವನ್ನು ತೆಗೆದುಕೊಳ್ಳಬೇಕೆಂದಾದರೆ ಅಥವಾ ನಿಮ್ಮದೇ ಬರೆಯಬೇಕೆಂದಾದರೆ ಯಾರು ತಿಳಿದುಕೊಳ್ಳುತ್ತದೆ
- ನಿಮ್ಮ ಆಪ್ತ ಆಪ್ ಅನ್ನು `azd up` ಬಳಸಿ ಅಜೂರ್‌ಗೆ ವಿನ್ಯಾಸಗೊಳಿಸುತ್ತೀರಿ

## ಅಧ್ಯಯನ ಫಲಿತಾಂಶಗಳು

ಈ ಪಾಠವನ್ನು ಪೂರ್ಣಗೊಂಡ ಮೇಲೆ ನೀವು ಮಾಡಬಲ್ಲಿರಿ:
- ನೀವು ಈಗಾಗಲೇ ಹೊಂದಿರುವ ಯೋಜನೆಗೆ azd ಪ್ರಾರಂಭಿಸಿ
- ಮೂಲ `azure.yaml` ಕಡತವನ್ನು ಓದಿ ಮತ್ತು ಸಂಪಾದಿಸಿ
- `azd infra generate` ಬಳಸಿ ಪ್ರಾರಂಭಿಕ ಮೂಲಸೌಲಭ್ಯವನ್ನು ರಚಿಸಿ
- ನಿಮ್ಮ ಆಪ್‌ಗೆ ತಕ್ಕ ಅಜೂರ್ ಹೋಸ್ಟ್ ಆಯ್ಕೆಮಾಡಿ
- ನಿಮ್ಮದೇ ಅನ್ವಯಿಕೆಯನ್ನು ವಿನ್ಯಾಸಗೊಳಿಸಿ ಮತ್ತು ಸ್ವಚ್ಛಗೊಳಿಸಿ

## azd ಯೋಜನೆಯನ್ನು ಪ್ರಾರಂಭಿಸುವ ಮೂರು ಮಾರ್ಗಗಳು

| ಪ್ರಾರಂಭದ ಬಿಂದು | ಕಮಾಂಡ್ | ಯಾವಾಗ ಬಳಸಬೇಕು |
|----------------|---------|-------------|
| **ಟೆಂಪ್ಲೇಟ್‌ನಿಂದ** | `azd init --template <name>` | ಕಲಿಕಾ, ಅಥವಾ ಸಾಬೀತಾದ ಮಾದರಿಯಿಂದ ಹೊಸ ಆಪ್ ಪ್ರಾರಂಭಿಸುವಾಗ |
| **ನೀವು ಹೊಂದಿರುವ ಕೋಡ್‌ನಿಂದ** | `azd init` (ನಿಮ್ಮ ಯೋಜನೆ ಫೋಲ್ಡರ್‌ನಲ್ಲಿ) | ನೀವು ಈಗಾಗಲೇ ಒಂದು ಆಪ್ ಹೊಂದಿದ್ದೀರಿ ಮತ್ತು ಅದನ್ನು ವಿನ್ಯಾಸಗೊಳಿಸಲು ಬಯಸುತ್ತೀರಿ |
| **ಗೂಟ್ ರೆಪೊನಿಂದ** | `azd init --from-code` (ಒಂದು ಕ್ಲೋನ್ ಮಾಡಿದ ರೆಪೊನಲ್ಲಿ) | ಈಗಾಗಲೇ ಇರುವ ರೆಪೊಗೆ azd ಅಳವಡಿಸುವಾಗ |

ನೀವು ಮೊದಲ ಆಯ್ಕೆಯನ್ನು ಈಗಾಗಲೇ ಅಭ್ಯಾಸ ಮಾಡಿದ್ದೀರಿ. ಈ ಪಾಠವು ಎರಡನೇ - ಅತಿ ಸಾಮಾನ್ಯ ನೈಜಜಗತ್ತಿನ ಪರಿಸ್ಥಿತಿಯನ್ನು ನಿರ್ದರಿಸುತ್ತದೆ.

## ಹಂತ 1: ನಿಮ್ಮ ಯೋಜನೆಯಲ್ಲಿ `azd init` ಚಾಲನೆಮಾಡಿ

ನಿಮ್ಮ ಈಗಾಗಲಿರುವ ಯೋಜನೆ ಫೋಲ್ಡರ್ ಒಳಗೆ ಟರ್ಮಿನಲ್ ತೆರೆಯಿರಿ ಮತ್ತು ಚಾಲನೆಮಾಡಿ:

```bash
cd my-existing-app
azd init
```

azd ನಿಮಗೆ ನೀವು ಹೇಗೆ ಪ್ರಾರಂಭಿಸಬೇಕೆಂದು ಕೇಳುತ್ತದೆ. ಆರಿಸಿ:

```
? How do you want to initialize your app?
> Use code in the current directory
  Select a template
```

**"ಪ್ರಸ್ತುತ ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ ಕೋಡ್ ಬಳಸಿ."** ಎಂದು ಆರಿಸಿ. ನಂತರ azd ನಿಮ್ಮ ಫೋಲ್ಡರ್ ಸ್ಕ್ಯಾನ್ ಮಾಡುತ್ತದೆ, ನಿಮ್ಮ ಭಾಷೆ ಮತ್ತು ಫ್ರೇಮ್‌ವರ್ಕ್ ಕಂಡುಹಿಡಿಯುತ್ತದೆ ಮತ್ತು ಒಂದು ಹೋಸ್ಟ್ ಅನ್ನು ಪ್ರಸ್ತಾವಿಸುತ್ತದೆ.

### azd ಏನು ಪತ್ತೆಹಚ್ಚುತ್ತದೆ

azd `package.json`, `requirements.txt`, `pom.xml`, `*.csproj`, ಅಥವಾ `Dockerfile` പോലಿನ ಸಂಕೇತಗಳಿಗಾಗಿ ಹುಡುಕುತ್ತದೆ, ಮತ್ತು ಹೊಂದಿಕೊಂಡಿರುವ ಅజೂರ್ ಹೋಸ್ಟ್ ಅನ್ನು ಸೂಚಿಸುತ್ತದೆ:

| ನಿಮ್ಮ ಆಪ್ | ಸಾಧ್ಯವಿರುವ ಹೋಸ್ಟ್ |
|----------|----------------------|
| Node.js / Python / .NET ವೆಬ್ ಆಪ್ | ಅಜೂರ್ ಆಪ್ ಸರ್ವೀಸ್ ಅಥವಾ ಕಂಟೇನರ್ ಆಪ್ಸ್ |
| ಕಂಟೇನರೈಜ್ ಮಾಡಿದ ಆಪ್ (`Dockerfile`) | ಅಜೂರ್ ಕಂಟೇನರ್ ಆಪ್ಸ್ |
| ಫಂಕ್ಷನ್ ಆಪ್ | ಅಜೂರ್ ಫಂಕ್ಷನ್ಸ್ |
| ಸ್ಥಿರ ಜಾಲ址 (React/Vue ನಿರ್ಮಾಣ output) | ಅಜೂರ್ ಸ್ಥಿರ ವೆಬ್ ಆಪ್ಸ್ |

ಕಂಡುಹಿಡಿದ ಸೇವೆಯನ್ನು ದೃಢೀಕರಿಸಿ, ಮತ್ತು azd ನಿಮಗೆ ಬೇಕಾದ ಕಡತಗಳನ್ನು ನಿರ್ಮಿಸುತ್ತದೆ.

## ಹಂತ 2: azd ರಚಿಸಿದುದನ್ನು ತಿಳಿಯಿರಿ

ಇನಿಟ್ ಮಾಡಿದ ಮೇಲೆ, ನಿಮ್ಮ ಯೋಜನೆಯಲ್ಲಿ ಎರಡು ಹೊಸ ವಸ್ತುಗಳನ್ನು ಹೊಂದಿರುವಿರಿ:

```
my-existing-app/
├── azure.yaml          # ← NEW: tells azd what to deploy and where
├── infra/              # ← NEW: Infrastructure as Code (Bicep by default)
│   ├── main.bicep
│   └── main.parameters.json
├── src/ (your code)    # unchanged
└── ...                 # your existing files, untouched
```

### `azure.yaml` — ಯೋಜನೆಯ ವ್ಯಾಖ್ಯಾನ

ಇದು azd ಯೋಜನೆಯ હ્રદયವಾಗಿದೆ. ಇದರಲ್ಲಿ ಕನಿಷ್ಠವು ಹೀಗಿದೆ:

```yaml
# azure.yaml
name: my-existing-app
services:
  web:
    project: ./src           # path to your app code
    language: js             # js | python | dotnet | java
    host: appservice         # appservice | containerapp | function | staticwebapp
```

`services` ಬ್ಲಾಕ್ ಮುಖ್ಯಭಾಗವಾಗಿದೆ: ಪ್ರತಿ ನಿರ್ದಿಷ್ಟ ಪ್ರವೇಶವು ನಿಮ್ಮ ಕೋಡ್‌ನ ಫೋಲ್ಡરને ಒಂದು ಅಜೂರ್ ಹೋಸ್ಟ್‌ಗೆ ನಕ್ಷೆ ಮಾಡುತ್ತದೆ. ನಿಮ್ಮ ಆಪ್‌ಗೆ ಮೊಬೈಲ್ ಮುಂಭಾಗ ಮತ್ತು API ಇಬ್ಬರೂ ಇದ್ದರೆ, ನೀವು ಎರಡು ಸೇವೆಗಳಿರುತ್ತವೆ.

### `infra/` — ನಿಮ್ಮ ಅಜೂರ್ ಸಂಪನ್ಮೂಲಗಳು ಕೋಡ್ ರೂಪದಲ್ಲಿ

`infra/` ಫೋಲ್ಡರ್ Bicep ಕಡತಗಳನ್ನು ಹೊಂದಿದೆ, ಇದು ನಿಮ್ಮ ಆಪ್‌ಗೆ ಬೇಕಾಗುವ ಅಜೂರ್ ಸಂಪನ್ಮೂಲಗಳನ್ನು (ಆಪ್ ಸರ್ವೀಸ್, ಡೇಟಾಬೇಸ್ ಇತ್ಯಾದಿ) ವ್ಯಾಖ್ಯಾನಿಸುತ್ತದೆ. ನೀವು ಕೈಯಿಂದ ಬರೆಯಬೇಕಾಗಿಲ್ಲ — azd ಕಾರ್ಯನಿರ್ವಹಿಸುವ ಪ್ರಾರಂಭಿಕ ಬಿಂದು ರಚಿಸುತ್ತದೆ. ನೀವು ಅವರನ್ನು ನಂತರ ಸಂಪಾದಿಸಿ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಸೇರಿಸಬಹುದು ಅಥವಾ ಭದ್ರತೆ ಗಟ್ಟಿಗೊಳಿಸಬಹುದು (ಅಧ್ಯಾಯ 4ರಲ್ಲಿ ವಿವರಿಸಲಾಗಿದೆ).

> **ಸೂಚನೆ:** ವಿನ್ಯಾಸಗೊಳಿಸುವುದಕ್ಕೆ ಮುಂಚೆ ರಚಿಸಲಾದ ಮೂಲಸೌಕರ್ಯವನ್ನು ನೋಡಬೇಕೆಂದು ಅಥವಾ ಕಸ್ಟಮೈಸ್ ಮಾಡಬೇಕೆಂದು ಬಯಸಿದರೆ? `azd infra generate` (ಅಥವಾ `azd infra synth`) ಅನ್ನು ಚಾಲನೆ ಮಾಡಿ, ಇದು IaC ಅನ್ನು ಡಿಸ್ಕ್‌ಗೆ ಬರೆಯುತ್ತದೆ, ನೀವು ಪರಿಶೀಲಿಸಿ ಆವರ್ತಿತ ಇತಿಹಾಸವನ್ನು ಇರಿಸಿಕೊಳ್ಳಬಹುದು.

## ಹಂತ 3: ಅಗತ್ಯವಿರುವ ಸಂರಚನೆಯನ್ನು ಹೊಂದಿಸಿ

ನಿಮ್ಮ ಆಪ್‌ಗೆ ಸೆಟ್ಟಿಂಗ್ಸ್ ಅಥವಾ ರಹಸ್ಯಗಳು ಬೇಕಾದರೆ (ಜೋಡಣಾ ಸtring, API ಕೀ), ಅವನ್ನು ಕಠಿಣ ಕ್ರಮದಲ್ಲಿ ಕೊಡಿ ಮುಸಿ. ಪರಿಸರ ಮೌಲ್ಯಗಳನ್ನು ಬಳಸಿ:

```bash
# ಒಂದು ಪರಿಸರವನ್ನು ರಚಿಸಿ
azd env new dev

# ರಹಸ್ಯವಾದ ಮೌಲ್ಯವಿಲ್ಲದ ಮೌಲ್ಯವನ್ನು ಹೊಂದಿಸಿ
azd env set API_VERSION 1.0.0
```

ನಿಜವಾದ ರಹಸ್ಯಗಳಿಗೆ, ಅವುಗಳನ್ನು ಕೀ ವಾಲ್ಟ್ ನಲ್ಲಿ ಸಂಗ್ರಹಿಸಿ ಮತ್ತು ನಿಮ್ಮ ಮೂಲಸೌಲಭ್ಯದಿಂದ ಅವುಗಳನ್ನು ಉಲ್ಲೇಖಿಸಿ - [ಅಧ್ಯಾಯ 3: ಸಂರಚನೆ ಮತ್ತು ಪ್ರಾಮಾಣೀಕರಣ](../chapter-03-configuration/authsecurity.md) ನೋಡಿ.

## ಹಂತ 4: ವಿನ್ಯಾಸಗೊಳಿಸಿ

ಈಗ ನೀವು ಈಗಾಗಲೇ ತಿಳಿದಿರುವ ಅದೇ ಕಾರ್ಯಪ್ರಣಾಳಿಕೆಯನ್ನು ಬಳಸಿ:

```bash
# ಪ್ರಾಮಾಣೀಕರಣ (azd ಗೆ ಅಗತ್ಯವಿದೆ)
azd auth login

# ಸೃಷ್ಟಿಸಲ್ಪಡುವ ಸಂಪನ್ಮೂಲಗಳ ಪೂರ್ವದೃಷ್ಟಿ
azd provision --preview

# ಮೂಲಸೌಕರ್ಯವನ್ನು ಒದಗಿಸಿ ಮತ್ತು ನಿಮ್ಮ ಕೋಡ್ ಅನ್ನು ನಿಯೋಜಿಸಿ
azd up
```

ಮುಗಿಸಿದಾಗ, azd ನಿಮ್ಮ ಆಪ್ URL ಅನ್ನು ಮುದ್ರಿಸುತ್ತದೆ. ಯಾವುದಾದರೂ azd ಆಪ್ ಅಂದರೆ ಹಾಗೆ ಪರಿಶೀಲಿಸಿ:

```bash
azd show           # ಎಂಡ್ಪಾಯಿಂಟ್‌ಗಳನ್ನು ತೋರಿಸಿ
azd monitor --logs # ಅಗತ್ಯವಿದ್ದರೆ ಲಾಗ್‌ಗಳನ್ನು ಪರಿಶೀಲಿಸಿ
```

|---------|--------------|-----|
| azd ನನ್ನ ಆಪ್ ಪತ್ತೆಹಚ್ಚಲಿಲ್ಲ | ಮ್ಯಾನಿಫೆಸ್ಟ್ ಇಲ್ಲ (ಉದಾ., `package.json`) | ಮ್ಯಾನಿಫೆಸ್ಟ್ ಸೇರಿಸಿ ಅಥವಾ `azd init` ಸಮಯದಲ್ಲಿ ಹೋಸ್ಟ್ ಅನ್ನು ಕೈಗೆ ಪಡೆಯಿರಿ |
| `azd up` ಸಮಯದಲ್ಲಿ ನಿರ್ಮಾಣ ವಿಫಲವಾಗುತ್ತದೆ | ಆಪ್‌ಗೆ ನಿರ್ಮಾಣ ಹಂತ ಬೇಕು | `azure.yaml`ಯಲ್ಲಿ ಸೇವೆಯಡಿಯಲ್ಲಿ `buildCommand`/`outputPath` ಸೇರಿಸಿ |
| ಆಪ್ ಪ್ರಾರಂಭವಾಗುತ್ತದೆ ಆದರೆ ದೋಷಗಳನ್ನು ವಾಪಾಸು ಕೊಡುತ್ತದೆ | ಸಂರಚನೆ/ರಹಸ್ಯಗಳ ಕೊರತೆ | `azd env set` ಬಳಸಿ ಮೌಲ್ಯಗಳನ್ನು ಸೆಟ್ ಮಾಡಿ ಅಥವಾ ಕೀ ವಾಲ್ಟ್ ಜೊತೆಗೆ ಸಂಪರ್ಕ ಮಾಡಿ |
| ತಪ್ಪು ಹೋಸ್ಟ್ ಆಯ್ಕೆಮಾಡಲಾಗಿದೆ | ಸ್ವಯಂ-ಪತ್ತೆಹಚ್ಚುವಿಕೆ ತಪ್ಪು | `azure.yaml`ಯಲ್ಲಿ `host:` ಸಂಪಾದಿಸಿ ಮತ್ತು `azd up` ಪುನಃ ಚಾಲನೆ ಮಾಡಿ |

ಇನ್ನಷ್ಟು ಸಮಸ್ಯೆಗಳಿಗಾಗಿ, [ಅಧ್ಯಾಯ 7: ಸಮಸ್ಯೆಗಳ ಸಮಾಧಾನ](../chapter-07-troubleshooting/README.md) ನೋಡಿ.

## ಸಾರಾಂಶ

- `azd init` → **"ಪ್ರಸ್ತುತ ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ ಕೋಡ್ ಬಳಸಿ"** ನಿಮ್ಮ ಈಗಾಗಲೇ ಇರುವ ಆಪ್‌ಗೆ azd ಸೇರಿಸುತ್ತದೆ.
- `azure.yaml` ನಿಮ್ಮ ಕೋಡ್ ಫೋಲ್ಡರ್‌ಗಳನ್ನು ಅಜೂರ್ ಹೋಸ್ಟ್‌ಗಳಿಗೆ ನಕ್ಷೆ ಮಾಡುತ್ತದೆ; `infra/` ಸಂಪನ್ಮೂಲಗಳನ್ನು Bicep ಆಗಿ ವ್ಯಾಖ್ಯಾನಿಸುತ್ತದೆ.
- `azd infra generate` ನಿಮ್ಮಿಗೆ ರಚಿಸಲಾದ ಮೂಲಸೌಕರ್ಯವನ್ನು ಪರಿಶೀಲಿಸಲು ಅಥವಾ ಕಸ್ಟಮೈಸ್ ಮಾಡಲು ಅನುಮತಿಸುತ್ತದೆ.
- ಆರಂಭವಾದ ಮೇಲೆ, ನಿಮ್ಮ ಈಗಾಗಲೇ ಇರುವ ಆಪ್ ಟ್ಯಾಂಪ್ಲೇಟ್ ಆಧಾರಿತ ಆಪ್‌ನಂತೆ ಕೂಡಿದ್ದ `azd up` / `azd down` ಕಾರ್ಯಪ್ರಣಾಳಿಕೆಯನ್ನು ಬಳಕೆ ಮಾಡುತ್ತದೆ.
