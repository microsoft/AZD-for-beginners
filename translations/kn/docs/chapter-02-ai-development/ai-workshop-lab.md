## ಕಾರ್ಯಾಗಾರ ಪರಿಚಯ

ಈ ಕೈಯಿಂದ-ಕೆಲಸ ಪ್ರಯೋಗಶಾಲೆ ಅಭಿವೃದ್ಧಿಕಾರರಿಗೆ ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ AI ಟೆಂಪ್ಲೇಟ್ನ್ನು ತೆಗೆದು ಅದನ್ನು Azure Developer CLI (AZD) ಬಳಸಿ ನಿಯೋಜಿಸುವ ಪ್ರಕ್ರಿಯೆಯ ಮೂಲಕ ಮಾರ್ಗದರ್ಶನ ಮಾಡುತ್ತದೆ. ನೀವು Microsoft Foundry ಸೇವೆಗಳು ಬಳಸಿಕೊಂಡು ಉತ್ಪಾದನಾ_AI ನಿಯೋಜನೆಗಳಿಗೆ ಅಗತ್ಯವಾಗುವ ಮಾದರಿಗಳನ್ನು ತಿಳಿದುಕೊಳ್ಳುತ್ತೀರಿ.

> **ಬಳಕೆ ಮಾನ್ಯತೆ ಟಿಪ್ಪಣಿ (2026-07-13):** ಈ ಕಾರ್ಯಾಗಾರವನ್ನು `azd` `1.27.1` ವಿರುದ್ಧ ಪರಿಶೀಲಿಸಲಾಗಿದೆ. ನಿಮ್ಮ ಸ್ಥಳೀಯ ಸ್ಥಾಪನೆ ಹಳೆಯದಾದರೆ, ಕೆಳಗಿನ ಹಂತಗಳಿಗೆ ಹೊಂದಿಕೊಳ್ಳಲು ಪ್ರಾರಂಭಿಸಲು ಮುಂಚಿತವಾಗಿ AZD ಅಪ್ಡೇಟ್ ಮಾಡಿ.

**ಅವಧಿ:** 2-3 ಘಂಟೆಗಳು  
**ಮಟ್ಟ:** ಮಧ್ಯಂತರ  
**ಪೂರ್ವಾಪೇಕ್ಷೆಗಳು:** ಮೂಲ Azure ಜ್ಞಾನ, AI/ML ತತ್ವಗಳಲ್ಲಿ ಪರಿಚಯ

2. **ಕಾರ್ಯಾಗಾರದ ರೆಪೊ ಕ್ಲೋನ್ ಮಾಡಿ:**


## ಮೋಡ್ಯೂಲ್ 1: AI ಅಪ್ಲಿಕೇಶನ್ಗಳಿಗೆ AZD ರಚನೆಯ ಅರ್ಥ ಮಾಡಿಕೊಳ್ಳುವುದು

### AI ಸಿದ್ಧ AZD ಟೆಂಪ್ಲೇಟಿನ ರಚನೆ

AI-ತಯಾರಾದ AZD ಟೆಂಪ್ಲೇಟಿನ ಪ್ರಮುಖ ಕಡತಗಳನ್ನು ಅನ್ವೇಷಿಸಿ:

```
azure-search-openai-demo/
├── azure.yaml              # AZD configuration
├── infra/                   # Infrastructure as Code
│   ├── main.bicep          # Main infrastructure template
│   ├── main.parameters.json # Environment parameters
│   └── modules/            # Reusable Bicep modules
│       ├── openai.bicep    # Microsoft Foundry Models configuration
│       ├── search.bicep    # Azure AI Search setup
│       └── webapp.bicep    # Web app configuration
├── app/                    # Application code
├── scripts/               # Deployment scripts
└── .azure/               # AZD environment files
```

### **ಪ್ರಯೋಗ 1.1: ಸಂರಚನೆಯನ್ನು ಅನ್ವೇಷಿಸು**

1. **azure.yaml ಕಡತವನ್ನು ಪರಿಶೀಲಿಸಿ:**
```bash
cat azure.yaml
```

**ನೋಡುವುದಕ್ಕೆ:**
- AI घटಕಗಳ ಸೇವೆ ವಿವರಣೆಗಳು
- ಪರಿಸರ ಪರಿವರ್ತಕ ನಕ್ಷೆಗಳು
- ಹೋಸ್ಟ್ ಸಂರಚನೆಗಳು

2. **main.bicep ಮೂಲಮಟ್ಟದ ಸಂರಚನೆಯನ್ನು ಪರಿಶೀಲನೆ:**
```bash
cat infra/main.bicep
```

**ಮನಗೊಳ್ಳಬೇಕಾದ ಪ್ರಮುಖ AI ಮಾದರಿಗಳು:**
- Microsoft Foundry ಮಾದರಿಗಳ ಸೇವೆ ಪೂರೈಕೆ
- Azure AI Search ಏಕೀಕರಣ
- ಸುರಕ್ಷಿತ ಕೀ ನಿರ್ವಹಣೆ
- ಜಾಲತಂತ್ರ ಭದ್ರತೆ ಸಂರಚನೆಗಳು

### **ಚರ್ಚಾ ಬಿಂದುವು:** ಏಕೆ ಈ ಮಾದರಿಗಳು AIಗೆ ಮುಖ್ಯ

- **ಸೇವಾ ಅವಲಂಬನೆಗಳು**: AI ಅಪ್ಲಿಕೇಶನ್ಗಳು ಹಲವಾರು ಸಂಯೋಜಿತ ಸೇವೆಗಳನ್ನು ಬೇಕು
- **ಭದ್ರತೆ**: API ಕೀಯ್ಸ್ ಮತ್ತು ಕಡತಗಳನ್ನು ಸುರಕ್ಷಿತವಾಗಿ ನಿರ್ವಹಿಸಬೇಕು
- **ವಿಸ್ತರಣೆ ಸಾಮರ್ಥ್ಯ**: AI ಕಾರ್ಯಭಾರಗಳಿಗೆ ವಿಶಿಷ್ಟ ವಿಸ್ತರಣೆ ಅವಶ್ಯಕತೆಗಳು ಇವೆ
- **ಖರ್ಚು ನಿರ್ವಹಣೆ**: ಸರಿಯಾದ ರೀತಿ ಸಂರಚನೆ ಮಾಡದೆ AI ಸೇವೆಗಳು ದುಬಾರಿ ಆಗಬಹುದು

## ಮೋಡ್ಯೂಲ್ 2: ನಿಮ್ಮ ಮೊದಲ AI ಅಪ್ಲಿಕೇಶನನ್ನು ನಿಯೋಜಿಸು

### ಹಂತ 2.1: ಪರಿಸರವನ್ನು ಪ್ರಾರಂಭಿಸು

1. **ಹೊಸ AZD ಪರಿಸರ ರಚಿಸು:**
```bash
azd env new myai-workshop
```

2. **ಅಗತ್ಯ ಪ್ರಮಾಣಪತ್ರಗಳನ್ನು ಸೆಟ್ ಮಾಡು:**
```bash
# ನಿಮ್ಮ ಇಚ್ಛಿತ ಅಜ್ಯೂರ್ ಪ್ರದೇಶವನ್ನು ಸೆಟ್ ಮಾಡಿ
azd env set AZURE_LOCATION eastus

# ಐಚ್ಛಿಕ: ನಿರ್ದಿಷ್ಟ OpenAI ಮಾದರಿಯನ್ನು ಸೆಟ್ ಮಾಡಿ
azd env set AZURE_OPENAI_MODEL gpt-4.1-mini
```

### ಹಂತ 2.2: ಮೂಲಸೌಕರ್ಯ ಮತ್ತು ಅಪ್ಲಿಕೇಶನ್ ನಿಯೋಜನೆ

1. **AZD ಬಳಸಿ ನಿಯೋಜಿಸಿ:**
```bash
azd up
```

**`azd up` ಸಮಯದಲ್ಲಿ ಯಾವುದು ಸಂಭವಿಸುತ್ತದೆ:**
- ✅ Microsoft Foundry Models ಸೇವೆಯನ್ನು ಒದಗಿಸುತ್ತದೆ
- ✅ Azure AI Search ಸೇವೆ ರಚಿಸುತ್ತದೆ
- ✅ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಾಗಿ ಅಪ್ ಸರ್ವೀಸ್ ಸಿದ್ಧಪಡಿಸುತ್ತದೆ
- ✅ ಜಾಲತಂತ್ರ ಮತ್ತು ಭದ್ರತೆ ಸಂರಚಿಸುತ್ತದೆ
- ✅ ಅಪ್ಲಿಕೇಶನ್ ಕೋಡ್ ನಿಯೋಜಿಸುತ್ತದೆ
- ✅ ಮೇಲ್ವಿಚಾರಣೆ ಮತ್ತು ಲಾಗಿಂಗ್ ಸೆಟ್ ಮಾಡುತ್ತದೆ

2. **ನಿಯೋಜನೆ ಪ್ರಗತಿಯನ್ನು ವೀಕ್ಷಿಸಿ** ಮತ್ತು ರಚನೆಗೊಂಡ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಗಮನಿಸಿ.

### ಹಂತ 2.3: ನಿಮ್ಮ ನಿಯೋಜನೆ ಪರಿಶೀಲನೆ

1. **ನಿಯೋಜಿತ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಪರಿಶೀಲಿಸಿ:**
```bash
azd show
```

2. **ನಿಯೋಜಿತ ಅಪ್ಲಿಕೇಶನ್ ತೆರೆಯಿರಿ:**
```bash
azd show
```

`azd show` ಔಟ್‌ಪುಟ್‌ನಲ್ಲಿ ತೋರಿಸಲಾದ ವೆಬ್ ಎಂಡ್ಪಾಯಿಂಟ್ ತೆರೆಯಿರಿ.

3. **AI ಕಾರ್ಯಕ್ಷಮತೆಯನ್ನು ಪರೀಕ್ಷಿಸಿ:**
   - ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್ನಿಗೆ ಸಾಗಿರಿ
   - ಮಾದರಿ ಪ್ರಶ್ನೆಗಳನ್ನು ಪ್ರಯತ್ನಿಸಿ
   - AI ಪ್ರತಿಕ್ರಿಯೆಗಳು ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತಿವೆ ಎಂದು ಪರಿಶೀಲಿಸಿ

### **ಪ್ರಯೋಗ 2.1: ಸಮಸ್ಯೆಗಳ ಪರಿಹಾರ ಅಭ್ಯಾಸ**

**ಘಟನೆ**: ನಿಮ್ಮ ನಿಯೋಜನೆ ಯಶಸ್ವಿಯಾಗಿ ನಡೆಯಿತು ಆದರೆ AI ಪ್ರತಿಕ್ರಿಯಿಸುತ್ತಿಲ್ಲ.

**ಪರಿಚಿತ ಸಮಸ್ಯೆಗಳು ಪರಿಶೀಲಿಸುವುದು:**
1. **OpenAI API ಕೀಲಿಗಳು**: ಸರಿಯಾದ ರೀತಿಯಲ್ಲಿ ಹೊಂದಿಸಲಾಗಿದೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
2. **ಮಾದರಿ ಲಭ್ಯತೆ**: ನಿಮ್ಮ ಪ್ರದೇಶದಲ್ಲಿ ಮಾದರಿ ಲಭ್ಯವಿದೆಯೇ ಎಂದು ಪರಿಶೀಲನೆ ಮಾಡಿ
3. **ಜಾಲ ಸಂಪರ್ಕತೆ**: ಸೇವೆಗಳು ಪರಸ್ಪರ ಸಂವಹನ ಮಾಡಬಹುದು ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
4. **RBAC ಅನುಮತಿಗಳು**: ಅಪ್ಲಿಕೇಶನ್ OpenAI ಬಳಕೆ ಮಾಡಬಹುದು ಎಂದು ಪರಿಶೀಲಿಸಿ

**ದೊಡ್ಡಿಸಿ ಆದೇಶಗಳು:**
```bash
# ಪರಿಸರ ಚರಗಳನ್ನು ಪರಿಶೀಲಿಸಿ
azd env get-values

# ನಿಯೋಜನೆ ಲಾಗ್‌ಗಳನ್ನು ವೀಕ್ಷಿಸಿ
az webapp log tail --name YOUR_APP_NAME --resource-group YOUR_RG

# OpenAI ನಿಯೋಜನೆ ಸ್ಥಿತಿಯನ್ನು ಪರಿಶೀಲಿಸಿ
az cognitiveservices account deployment list --name YOUR_OPENAI_NAME --resource-group YOUR_RG
```

## ಮೋಡ್ಯೂಲ್ 3: ನಿಮ್ಮ ಅವಶ್ಯಕತೆಗಳಿಗೆ AI ಅಪ್ಲಿಕೇಶನ್ಗಳನ್ನು ಕಸ್ಟಮೈಸ್ ಮಾಡುವುದು

### ಹಂತ 3.1: AI ಸಂರಚನೆಯನ್ನು ತಿದ್ದುಪಡಿ ಮಾಡು

1. **OpenAI ಮಾದರಿಯನ್ನು ಬದಲಾಯಿಸು:**
```bash
# ನಿಮ್ಮ ಪ್ರದೇಶದಲ್ಲಿ ಲಭ್ಯವಿದ್ದಲ್ಲಿ ಬೇರೆ ಮಾದರಿಯನ್ನು ಬದಲಾಯಿಸಿ
azd env set AZURE_OPENAI_MODEL gpt-4.1

# ಹೊಸ ಸಂರಚನೆಯೊಂದಿಗೆ ಮರು ಸ್ಥಾಪಿಸಿ
azd deploy
```

2. **ಹೆಚ್ಚಿನ AI ಸೇವೆಗಳನ್ನು ಸೇರಿಸು:**

## ಮೋಡ್ಯೂಲ್ 5: ನಿಮ್ಮ ತನ್ನ AI ಅಪ್ಲಿಕೇಶನನ್ನು ಪರಿವರ್ತಿಸುವುದು

### ಹಂತ 5.1: ಮೌಲ್ಯಮಾಪನ ವರ್ಕ್‌ಶೀಟ್

**ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನನ್ನು ಪರಿವರ್ತಿಸಲು ಮುಂಚೆ**, ಈ ಪ್ರಶ್ನೆಗಳಿಗೆ ಉತ್ತರಿಸಿ:

1. **ಅಪ್ಲಿಕೇಶನ್ ವಾಸ್ತುಶಿಲ್ಪ:**
   - ನಿಮ್ಮ ಅಪ್ ಯಾವ AI ಸೇವೆಗಳನ್ನು ಬಳಸುತ್ತದೆ?
   - ಅದರ ಗಣನಾ ಸಂಪನ್ಮೂಲ ಅವಶ್ಯಕತೆಗಳು ಏನು?
   - ಅದಕ್ಕೆ ಡೇಟಾಬೇಸ್ ಬೇಕಾಗುವುದೇ?
   - ಸೇವೆಗಳ ನಡುವೆ ಅವಲಂಬನೆ ಏನಿವೆ?

2. **ಭದ್ರತೆ ಅಗತ್ಯತೆಗಳು:**
   - ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಯಾವ ಸಂವೇದನಾಶೀಲ ಮಾಹಿತಿಯನ್ನು ಡೀಲ್ ಮಾಡುತ್ತದೆ?
   - ನಿಮ್ಮ ಬಳಿ ಯಾವ ಅನುಸರಣಾ ಅಗತ್ಯತೆಗಳಿವೆ?
   - ಖಾಸಗಿ ಜಾಲತಂತ್ರ ಅಗತ್ಯವಿದೆಯೇ?

3. **ವಿಸ್ತರಣೆ ಅಗತ್ಯತೆಗಳು:**

   - ಸ್ವಯಂಚಾಲಿತ ವಿಸ್ತರಣೆ ಬೇಕಿದೆಯೇ?
   - ಪ್ರಾದೇಶಿಕ ಅವಶ್ಯಕತೆಗಳಿವೆಯೇ?

### ಹಂತ 5.2: ನಿಮ್ಮ AZD ಟೆಂಪ್ಲೇಟನ್ನು ರಚಿಸಿ


```yaml
# Metadata
name: my-ai-app
```

```bicep

  }
}
```

**infra/modules/openai.bicep** - OpenAI ಮೋಡ್ಯೂಲ್:

```bicep
  }
```

1. ಕಾರ್ಯಕ್ಷಮತೆ ಮೌಲಕಗಳಿಗಾಗಿ ಅಪ್ಲಿಕೇಶನ್ ಇನ್ಸೈಟ್ಸ್ ಪರಿಶೀಲಿಸಿ
2. Azure ಪೋರ್ಟಲ್‍ನಲ್ಲಿ OpenAI ಸೇವಾ ಮೌಲಕಗಳನ್ನು ಪರಿಶೀಲಿಸಿ
3. ಜಾಲ ಸಂಪರ್ಕತೆ ಮತ್ತು ವಿಳಂಬವನ್ನು ಪರಿಶೀಲಿಸಿ

**ಪರಿಹಾರಗಳು:**
- ಸಾಮಾನ್ಯ ಪ್ರಶ್ನೆಗಳಿಗಾಗಿ ಕ್ಯಾಶಿಂಗ್ ಜಾರಿಗೆ ತಂದಿರಿ
- ನಿಮ್ಮ ಬಳಕೆಗೆ ಸೂಕ್ತ OpenAI ಮಾದರಿಯನ್ನು ಬಳಸಿ
- ಹೆಚ್ಚಿನ ಲೋಡ್ ಸನ್ನಿವೇಶಗಳಿಗೆ ಓದು ಪ್ರತಿ-ನಕಲುಗಳನ್ನು ಪರಿಗಣಿಸಿ

### **ಪ್ರಯೋಗ 6.1: ಡೀಬಗ್ ಸವಾಲು**

**ಘಟನೆ**: ನಿಮ್ಮ ನಿಯೋಜನೆ ಯಶಸ್ವಿಯಾಗಿ ನಡೆಯಿತು, ಆದರೆ ಅಪ್ಲಿಕೇಶನ್ 500 ದೋಷಗಳನ್ನು ನೀಡುತ್ತಿದೆ.

**ಡೀಬಗ್ ಕಾರ್ಯಗಳು:**
1. ಅಪ್ಲಿಕೇಶನ್ ಲಾಗ್‌ಗಳನ್ನು ಪರಿಶೀಲಿಸಿ
2. ಸೇವಾ ಸಂಪರ್ಕತೆ ಪರಿಶೀಲಿಸಿ
3. ಪ್ರಾಮಾಣೀಕರಣ ಪರೀಕ್ಷಿಸಿ
4. ಸಂರಚನೆ ಪರಿಶೀಲಿಸಿ

**ಬಳಸಬೇಕಾದ ಸಾಧನಗಳು:**
- ನಿಯೋಜನೆ ಇನ್ಸೈಟ್‌ಗಾಗಿ `azd show`
- ವಿವರವಾದ ಸೇವಾ ಲಾಗ್ಗಳಿಗಾಗಿ Azure ಪೋರ್ಟಲ್
- ಅಪ್ಲಿಕೇಶನ್ ಟೆಲಿಮೆಟ್ರಿಗಾಗಿ ಅಪ್ಲಿಕೇಶನ್ ಇನ್ಸೈಟ್ಸ್

## ಮೋಡ್ಯೂಲ್ 7: ಮೇಲ್ವಿಚಾರಣೆ ಮತ್ತು ಉನ್ನತಿ

### ಹಂತ 7.1: ಸಮಗ್ರ ಮೇಲ್ವಿಚಾರಣೆ ಸ್ಥಾಪನೆ

1. **ಕಸ್ಟಮ್ ಡ್ಯಾಶ್‌ಬೋರ್ಡ್‌ಗಳನ್ನು ರಚಿಸಿ:**

ನಿಮ್ಮನ್ನು Azure ಪೋರ್ಟಲ್‌ಗೆ ಹೋಗಿ ಈ ಡ್ಯಾಶ್‌ಬೋರ್ಡ್ ಸೃಷ್ಟಿಸಿ:
- OpenAI ವಿನಂತಿಗಳ ಸಂಖ್ಯೆ ಮತ್ತು ವಿಳಂಬ
- ಅಪ್ಲಿಕೇಶನ್ ದೋಷ ದರಗಳು
- ಸಂಪನ್ಮೂಲ ಬಳಕೆ
- ವೆಚ್ಚ ಟ್ರ್ಯಾಕಿಂಗ್

2. **ಅಲರ್ಟ್‌ಗಳನ್ನು ಹೊಂದಿಸಿ:**
```bash
# ಹೆಚ್ಚುವರಿ ದೋಷ ದರಕ್ಕೆ ಎಚ್ಚರಿಕೆ
az monitor metrics alert create \
  --name "AI-App-High-Error-Rate" \
  --resource-group YOUR_RG \
  --target-resource-id YOUR_APP_ID \
  --condition "avg Http5xx greater than 10" \
  --description "Alert when error rate is high"
```

### ಹಂತ 7.2: ವೆಚ್ಚ ಉನ್ನತಿ

1. **ಪ್ರಸ್ತುತ ವೆಚ್ಚವನ್ನ ವಿಶ್ಲೇಷಿಸಿ:**
```bash
# Azure CLI ಅನ್ನು ಬಳಸಿ ವೆಚ್ಚದ ಡೇಟಾವನ್ನು ಪಡೆಯಿರಿ
az consumption usage list --start-date 2024-01-01 --end-date 2024-01-31
```

2. **ವೆಚ್ಚ ನಿಯಂತ್ರಣಗಳನ್ನು ಜಾರಿಮಾಡಿ:**
- ಬಜೆಟ್ ಅಲರ್ಟ್‌ಗಳನ್ನು ಹೊಂದಿಸಿ
- ಸ್ವಯಂಚಾಲಿತ ವಿಸ್ತರಣಾ ಪಾಲಿಸಿಗಳನ್ನು ಬಳಸಿ
- ವಿನಂತಿ ಕ್ಯಾಶಿಂಗ್ ಜಾರಿಗೆ ತರಲು
- OpenAI ಟೋಕನ್ ಬಳಕೆ ಮೇಲ್ವಿಚಾರಣೆ

### **ಪ್ರಯೋಗ 7.1: ಕಾರ್ಯಕ್ಷಮತೆ ಉನ್ನತಿ**

**ಕಾರ್ಯ:** ನಿಮ್ಮ AI ಅಪ್ಲಿಕೇಶನನ್ನು ಕಾರ್ಯಕ್ಷಮತೆ ಮತ್ತು ವೆಚ್ಚ ದೃಷ್ಟಿಯಿಂದ ಉನ್ನತಗೊಳಿಸಿ.

**ಸೂಚ್ಯాంకಗಳನ್ನು ಸುಧಾರಿಸಲು:**
- ಸರಾಸರಿ ಪ್ರತಿಕ್ರಿಯೆ ಕಾಲವನ್ನು 20% ಕಡಿಮೆಯಾಗಿ ಮಾಡು
- ಮಾಸಿಕ ವೆಚ್ಚವನ್ನು 15% ಕಡಿಮೆ ಮಾಡು
- 99.9% ಸಂಚಲನವನ್ನು ಕಾಪಾಡು

**ಪ್ರಯತ್ನಿಸಬಹುದಾದ ತಂತ್ರಗಳು:**
- ಪ್ರತಿಕ್ರಿಯೆ ಕ್ಯಾಶಿಂಗ್ ಜಾರಿಗೊಳಿಸಿ
- ಟೋಕನ್ ಕಾರ್ಯಕ್ಷಮತೆಗೆ ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ಆತ್ಮಸಾತ್ ಮಾಡು
- ಸೂಕ್ತ ಗಣನಾ SKUಗಳನ್ನು ಬಳಸಿ
- ಸರಿಯಾದ ಸ್ವಯಂಚಾಲಿತ ವಿಸ್ತರಣೆ ಹೊಂದಿಸಿ
