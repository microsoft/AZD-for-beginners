# ಅಧ್ಯಾಯ 3: ಸಂರಚನೆ ಮತ್ತು ಪ್ರಾಮಾಣೀಕರಣ

**📚 Course**: [AZD ಆರಂಭಿಕರಿಗೆ](../../README.md) | **⏱️ Duration**: 45-60 ನಿಮಿಷ | **⭐ ಸಂಕೀರ್ಣತೆ**: ಮಧ್ಯಮ

---

## ಅವಲೋಕನ

ಈ ಅಧ್ಯಾಯವು Azure Developer CLI ನಿಯೋಜನೆಗಳಿಗಾಗಿ ಪರಿಸರ ಸಂರಚನೆ, ಪ್ರಾಮಾಣೀಕರಣ ಮಾದರಿಗಳು ಮತ್ತು ಭದ್ರತಾ ಅತ್ಯುತ್ತಮ ಅಭ್ಯಾಸಗಳನ್ನು ಒಳಗೊಂಡಿದೆ.

> 2026 ಜೂನ್‌ನಲ್ಲಿ `azd 1.25.6` ವಿರುದ್ಧ ಪರಿಶೀಲಿಸಲಾಗಿದೆ.

## ಕಲಿಕೆಯ ಉದ್ದೇಶಗಳು

ಈ ಅಧ್ಯಾಯವನ್ನು ಪೂರ್ಣಗೊಳಿಸಿದರೆ, ನೀವು:
- AZD ಸಂರಚನಾ ಹಿಯರಾರ್ಕಿಯನ್ನು ಅರಿತುಕೊಳ್ಳಿ
- ಹಲವು ಪರಿಸರಗಳನ್ನು ನಿರ್ವಹಿಸಿ (dev, staging, prod)
- ನಿರ್ವಹಿತ ಗುರುತುಗಳೊಂದಿಗೆ ಭದ್ರಿತ ಪ್ರಾಮಾಣೀಕರಣವನ್ನು ಜಾರಿಗೊಳ್ಳಿ
- ಪರಿಸರ-ನಿರ್ದಿಷ್ಟ ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು ಸಂರಚಿಸಿ

## 🚀 ತ್ವರಿತ ಪ್ರಾರಂಭ

```bash
# ಬಹು ಪರಿಸರಗಳನ್ನು ರಚಿಸಿ
azd env new dev
azd env new staging
azd env new prod

# ಪರಿಸರಗಳನ್ನು ಬದಲಾಯಿಸಿ
azd env select prod

# ಪರಿಸರ ಚರಗಳನ್ನು ನಿರ್ಧರಿಸಿ
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# ಸಂರಚನೆ ವೀಕ್ಷಿಸಿ
azd env get-values
```

---

## 🔧 ಸಂರಚನಾ ಶ್ರೇಣಿಭಾಗ

AZD ಈ ಕ್ರಮದಲ್ಲಿ ಸೆಟ್ಟಿಂಗ್‍ಗಳನ್ನು ಅನ್ವಯಿಸುತ್ತದೆ (ನಂತರದವು ಮೊದಲಿನದನ್ನು ಮೀರಿಸುತ್ತವೆ):

1. **ಪೂರ್ವನಿಯೋಜಿತ ಮೌಲ್ಯಗಳು** (ಟೆಂಪ್ಲೇಟ್ಗಳಲ್ಲಿ ಅಳವಡಿಸಲಾಗಿದೆ)
2. **azure.yaml** (ಪ್ರಾಜೆಕ್ಟ್ ಸಂರಚನೆ)
3. **ಪರಿಸರ ಚರಗಳು** (`azd env set`)
4. **ಕಮಾಂಡ್-ಲೈನ್ ಫ್ಲಾಗ್‌ಗಳು** (`--location eastus`)

---

## 🔐 ಭದ್ರತಾ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು

```bash
# ನಿರ್ವಹಿತ ಐಡೆಂಟಿಟಿಯನ್ನು ಬಳಸಿ (ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ)
azd env set AZURE_USE_MANAGED_IDENTITY true

# AZD ಪ್ರಮಾಣೀಕರಣ ಸ್ಥಿತಿಯನ್ನು ಪರಿಶೀಲಿಸಿ
azd auth status

# ಐಚ್ಛಿಕ: az ಕಮಾಂಡ್‌ಗಳನ್ನು ಚಲಾಯಿಸಲು ಯೋಜಿಸುತ್ತಿದ್ದರೆ Azure CLI ಸಂದರ್ಭವನ್ನು ಪರಿಶೀಲಿಸಿ
az account show

# ಅವಶ್ಯವಿದ್ದರೆ ಮತ್ತೆ ಪ್ರಮಾಣೀಕರಿಸಿ
azd auth login

# ಐಚ್ಛಿಕ: az ಕಮಾಂಡ್‌ಗಳಿಗಾಗಿ Azure CLI ಪ್ರಮಾಣೀಕರಣವನ್ನು ನವೀಕರಿಸಿ
az login
```
