# ಬದಲಾವಣೆಗಳ ಲಾಗ್ - AZD For Beginners

## ಪರಿಚಯ

ಈ ಚೇಂಜ್‌ಲಾಗ್ AZD For Beginners ರೆಪೊಸಿಟರಿಗೂ ಸಂಬಂಧಿಸಿದ ಎಲ್ಲಾ ಗಮನಾರ್ಹ ಬದಲಾವಣೆಗಳು, ಅಪ್‌ಡೇಟ್‌ಗಳು ಮತ್ತು ಸುಧಾರಣೆಗಳನ್ನು ದಾಖಲೆ ಮಾಡುತ್ತದೆ. ನಾವು ಸೆಮ್ಯಾಂಟಿಕ್ ಆವೃತ್ತಿ ನಿಯಮಾವಳಿಗಳನ್ನು ಅನುಸರಿಸುತ್ತೇವೆ ಮತ್ತು ಬಳಕೆದಾರರು ಆವೃತ್ತಿಗಳ ನಡುವೆ ಏನೇನು ಬದಲಾದುದೆಂದು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಈ ಲಾಗ್ ಅನ್ನು ನಿರ್ವಹಿಸುತ್ತೇವೆ.

## ಕಲಿಕೆಯ ಗುರಿಗಳು

ಈ ಚೇಂಜ್‌ಲಾಗ್ ಪರಿಶೀಲನೆಯ ಮೂಲಕ, ನೀವು:
- ಹೊಸ ವೈಶಿಷ್ಟ್ಯಗಳು ಮತ್ತು ವಿಷಯದ ಸೇರ್ಪಡೆಗಳ ಬಗ್ಗೆ ತಿಳಿದಿರುತ್ತೀರಿ
- ಇತ್ತೀಚಿನ ದಾಖಲೆಗಳಲ್ಲಿ ಮಾಡಲಾದ ಸುಧಾರಣೆಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುತ್ತೀರಿ
- ಪದಗಳ ಸರಿಪಡಿಕೆಗಳು ಮತ್ತು ದೋಷನಿರಾಕರಣೆಗಳನ್ನು ಟ್ರ್ಯಾಕ್ ಮಾಡಿ ನಿಖರತೆ ಖಾತ್ರಿಪಡಿಸುತ್ತೀರಿ
- ಕಾಲಕ್ರಮದಲ್ಲಿ ಕಲಿಕಾ ಸಾಮಗ್ರಿಗಳ ಗಳಿಕೆ ಹೇಗೆ ವಿಕಸಿಸುತ್ತಿದೆ ಎಂದು ಅನುಸರಿಸುತ್ತೀರಿ

## ಕಲಿಕೆಯ ಫಲಿತಾಂಶಗಳು

ಚೇಂಜ್‌ಲಾಗ್ ಪ್ರವೇಶಗಳನ್ನು ಪರಿಶೀಲಿಸಿದ ನಂತರ, ನೀವು ಸಾಮರ್ಥ್ಯ ಹೊಂದಿರುತ್ತೀರಿ:
- ಕಲಿಕೆಗೆ ಲಭ್ಯವಿರುವ ಹೊಸ ವಿಷಯಗಳು ಮತ್ತು ಸಂಪನ್ಮೂಲಗಳನ್ನು ಗುರುತಿಸಲು
- ಯಾವ ವಿಭಾಗಗಳನ್ನು ಅಪ್‌ಡೇಟ್ ಮಾಡಲಾಗಿದೆಯೋ ಅಥವಾ ಸುಧಾರಿಸಲಾಗಿದೆ ಎಂದು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು
- ಅತಿ ಇತ್ತೀಚಿನ ಸಾಮಗ್ರಿಗಳ ಆಧಾರದ ಮೇಲೆ ನಿಮ್ಮ ಕಲಿಕಾ ಮಾರ್ಗವನ್ನು ಯೋಜಿಸಲು
- ಭವಿಷ್ಯದ ಸುಧಾರಣೆಗಾಗಿ ಅಭಿಪ್ರಾಯ ಮತ್ತು ಸಲಹೆಗಳನ್ನು ನೀಡಲು

**ಈ ಆವೃತ್ತಿ ಡಾಕ್ಯುಮೆಂಟೇಶನ್‌ನಲ್ಲಿನ ಅಸ್ತಿತ್ವದಲ್ಲಿ olmayan AZD ಆಜ್ಞೆಗಳನ್ನು ಸರಿ ಮಾಡಿಕೊಂಡು, ಎಲ್ಲಾ ಕೋಡ್ ಉದಾಹರಣೆಗಳು ಮಾನ್ಯ Azure Developer CLI সিন್ಟ್ಯಾಕ್ಸ್ ಬಳಕೆಯಾಗುವಂತೆ ಮಾಡಿದೆ.**

#### ಸರಿಪಡಿಸಲಾಗಿದೆ
- **🔧 ಅಸ್ತಿತ್ವದಲ್ಲಿಲ್ಲದ AZD ಆಜ್ಞೆಗಳು ತೆಗೆದುಹಾಕಲಾಗಿದೆ**: ಅಮಾನ್ಯ ಆಜ್ಞೆಗಳ ಸಮಗ್ರ ಶೋಧನೆ ಮತ್ತು ಸರಿಪಡಣೆ:
  - `azd logs` (ಅಸ್ತಿತ್ವದಲ್ಲಿಲ್ಲ) → `azd monitor --logs` ಅಥವಾ Azure CLI ಪರ್ಯಾಯಗಳಿಗೆ ಬದಲಾಗಿಸಲಾಗಿದೆ
  - `azd service` ಉಪಕಮಾಂಡ್‌ಗಳು (ಅಸ್ತಿತ್ವದಲ್ಲಿಲ್ಲ) → `azd show` ಮತ್ತು Azure CLI ಗೆ ಬದಲಾಗಿಸಲಾಗಿದೆ
  - `azd infra import/export/validate` (ಅಸ್ತಿತ್ವದಲ್ಲಿಲ್ಲ) → ಮಾನ್ಯ ಪರ್ಯಾಯಗಳೊಂದಿಗೆ ತೆಗೆದುಹಾಕಲಾಗಿದೆ ಅಥವಾ ಬದಲಾವಣೆ ಮಾಡಲಾಗಿದೆ
  - `azd deploy --rollback/--incremental/--parallel/--detect-changes` ಫ್ಲಾಗ್‌ಗಳು (ಅಸ್ತಿತ್ವದಲ್ಲಿಲ್ಲ) → ತೆಗೆದುಹಾಕಲಾಗಿದೆ
  - `azd provision --what-if/--rollback` ಫ್ಲಾಗ್‌ಗಳು (ಅಸ್ತಿತ್ವದಲ್ಲಿಲ್ಲ) → `--preview` ಅನ್ನು ಬಳಕೆ ಮಾಡಲು ನವೀಕರಿಸಲಾಗಿದೆ
  - `azd config validate` (ಅಸ್ತಿತ್ವದಲ್ಲಿಲ್ಲ) → `azd config list` ಗೆ ಬದಲಾಗಿಸಲಾಗಿದೆ
  - `azd info`, `azd history`, `azd metrics` (ಅಸ್ತಿತ್ವದಲ್ಲಿಲ್ಲ) → ತೆಗೆದುಹಾಕಲಾಗಿದೆ

- **📚 ಆಜ್ಞೆಗಳ ಸರಿಪಡಣೆಗಳೊಂದಿಗೆ ಫೈಲ್‌ಗಳು ನವೀಕರಿಸಲ್ಪಟ್ಟವು**:
  - `resources/cheat-sheet.md`: ಆಜ್ಞೆಗಳ ಉಲ್ಲೇಖದ ಮಹತ್ವದ ಪುನರ್‍ರಚನೆ
  - `docs/deployment/deployment-guide.md`: ಪರಿಹಾರ ಮತ್ತು ನಿಯೋಜನೆ ರಣನೀತಿಗಳಲ್ಲಿ ಸರಿಪಡಣೆ
  - `docs/troubleshooting/debugging.md`: ಲಾಗ್ ವಿಶ್ಲೇಷಣಾ ವಿಭಾಗಗಳು ಸರಿಪಡಿಸಲಾಗಿದೆ
  - `docs/troubleshooting/common-issues.md`: troubleshooting ಆಜ್ಞೆಗಳು ನವೀಕರಿಸಲ್ಪಟ್ಟವು
  - `docs/troubleshooting/ai-troubleshooting.md`: AZD ಡಿಬੱਗಿಂಗ್ ವಿಭಾಗ ಸರಿಪಡಿಸಲಾಗಿದೆ
  - `docs/getting-started/azd-basics.md`: निगरानी ಆж್ಞೆಗಳನ್ನು ಸರಿಪಡಿಸಲಾಗಿದೆ
  - `docs/getting-started/first-project.md`: ಮಾನಿಟರಿಂಗ್ ಮತ್ತು ಡಿಬಗ್ಗಿಂಗ್ ಉದಾಹರಣೆಗಳು ನವೀಕರಿಸಲ್ಪಟ್ಟವು
  - `docs/getting-started/installation.md`: ಸಹಾಯ ಮತ್ತು ಆವೃತ್ತಿ ಉದಾಹರಣೆಗಳು ಸರಿಪಡಿಸಲ್ಪಟ್ಟವು
  - `docs/pre-deployment/application-insights.md`: ಲಾಗ್ ವೀಕ್ಷಣಾ ಆಜ್ಞೆಗಳು ಸರಿಪಡಿಸಲ್ಪಟ್ಟವು
  - `docs/pre-deployment/coordination-patterns.md`: ಏಜೆಂಟ್ ಡಿಬಗ್ಗಿಂಗ್ ಆಜ್ಞೆಗಳು ಸರಿಪಡಿಸಲ್ಪಟ್ಟವು

- **📝 ಆವೃತ್ತಿ ಉಲ್ಲೇಖವನ್ನು ನವೀಕರಿಸಲಾಗಿದೆ**:
  - `docs/getting-started/installation.md`: ಹಾರ್ಡ್‌ಕೋಡ್ ಮಾಡಿದ `1.5.0` ಆವೃತ್ತಿಯನ್ನು ಸಾಮಾನ್ಯ `1.x.x` ಗೆ ಬದಲಾಯಿಸಲಾಗಿದೆ ಹಾಗೂ ರಿಲೀಸ್‌ಗಳಿಗೆ ಲಿಂಕ್ ಸೇರಿಸಲಾಗಿದೆ

#### ಬದಲಾಯಿಸಲಾಗಿದೆ
- **Rollback ರಣನೀತಿಗಳು**: AZD ಸ್ವಭಾವಿಕ rollback ಇರುವುದಿಲ್ಲವೆಂದು ತಿಳಿದು Git ಆಧಾರದ rollback ಬಳಕೆಗೆ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ನವೀಕರಿಸಲಾಗಿದೆ
- **ಲಾಗ್ ವೀಕ್ಷಣೆ**: `azd logs` ಉಲ್ಲೇಖಗಳನ್ನು `azd monitor --logs`, `azd monitor --live`, ಮತ್ತು Azure CLI ಆಜ್ಞೆಗಳೊಂದಿಗೆ ಬದಲಾಯಿಸಲಾಗಿದೆ
- **ಕಾರ್ಯಕ್ಷಮತೆ ವಿಭಾಗ**: ಅಸ್ತಿತ್ವದಲ್ಲಿಲ್ಲದ parallel/incremental ನಿಯೋಜನಾ ಫ್ಲಾಗ್‌ಗಳನ್ನು ತೆಗೆದುಹಾಕಿ ಮಾನ್ಯ ಪರ್ಯಾಯಗಳನ್ನು ಒದಗಿಸಲಾಗಿದೆ

#### ತಾಂತ್ರಿಕ ವಿವರಗಳು
- **ಮಾನ್ಯ AZD ಆಜ್ಞೆಗಳು**: `init`, `up`, `auth`, `deploy`, `down`, `provision`, `publish`, `completion`, `config`, `env`, `show`, `version`, `monitor`
- **ಮಾನ್ಯ azd monitor ಫ್ಲಾಗ್‌ಗಳು**: `--live`, `--logs`, `--overview`
- **ತೆಗೆದುಹಾಕಲಾದ ವೈಶಿಷ್ಟ್ಯಗಳು**: `azd logs`, `azd service`, `azd infra import/export/validate`, `azd history`, `azd metrics`, `azd info`, `azd config validate`

#### ಗಮನಿಕೆಗಳು

## Contributing to the Changelog

### Reporting Changes
When contributing to this repository, please ensure changelog entries include:

1. **Version Number**: Following semantic versioning (major.minor.patch)
2. **Date**: Release or update date in YYYY-MM-DD format
3. **Category**: Added, Changed, Deprecated, Removed, Fixed, Security
4. **Clear Description**: Concise description of what changed
5. **Impact Assessment**: How changes affect existing users

### Change Categories

#### Added
- New features, documentation sections, or capabilities
- New examples, templates, or learning resources
- Additional tools, scripts, or utilities

#### Changed
- Modifications to existing functionality or documentation
- Updates to improve clarity or accuracy
- Restructuring of content or organization

#### Deprecated
- Features or approaches that are being phased out
- Documentation sections scheduled for removal
- Methods that have better alternatives

#### Removed
- Features, documentation, or examples that are no longer relevant
- Outdated information or deprecated approaches
- Redundant or consolidated content

#### Fixed
- Corrections to errors in documentation or code
- Resolution of reported issues or problems
- Improvements to accuracy or functionality

#### Security
- Security-related improvements or fixes
- Updates to security best practices
- Resolution of security vulnerabilities

### Semantic Versioning Guidelines

#### Major Version (X.0.0)
- Breaking changes that require user action
- Significant restructuring of content or organization
- Changes that alter the fundamental approach or methodology

#### Minor Version (X.Y.0)
- New features or content additions
- Enhancements that maintain backward compatibility
- Additional examples, tools, or resources

#### Patch Version (X.Y.Z)
- Bug fixes and corrections
- Minor improvements to existing content
- Clarifications and small enhancements
