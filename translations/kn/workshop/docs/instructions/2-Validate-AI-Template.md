!!! tip "ಈ ಅಧ್ಯಾಯದ ಕೊನೆಯಲ್ಲಿ ನೀವು ಈ ಕ್ರಮಗಳನ್ನು ಮಾಡಬಲ್ಲಿರಿ"

    - [ ] AZD ನಿಯೋಜನೆ ಕಾರ್ಯಪ್ರವಾಹವನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಿ

## 1. ಪರಿಚಯ

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) ಅಥವಾ `azd` ಎನ್ನುವುದು ವೆಚ್ಚ ಕಡಿತಗೊಳಿಸುವ ಮುಕ್ತ-ಮೂಲದ ಕಮಾಂಡ್ ಲೈನ್ ಸಾಧನವಾಗಿದ್ದು, ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು Azure ಗೆ ನಿರ್ಮಿಸುವಾಗ ಮತ್ತು ನಿಯೋಜಿಸುವಾಗ ಡೆವಲಪರ್ ಕಾರ್ಯಪಾಟಿಯನ್ನು ಸುಗಮಗೊಳಿಸುತ್ತದೆ.

[AZD ಟೆಂಪ್ಲೇಟುಗಳು](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) ಸಾಮಾನ್ಯವಾಗಿ ಮಾದರಿ ಅಪ್ಲಿಕೇಶನ್ ಕೋಡ್, _ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್-ಆಸ್-ಕೋಡ್_ ಆಸ್ತಿ, ಮತ್ತು `azd` ಸಂರಚನಾ ಕಡತಗಳನ್ನು ಒಳಗೊಂಡಿರುವ ನಿಯಮಿತ ಸಂಗ್ರಹಣೆಗಳು. ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಒದಗಿಸುವುದು `azd provision` ಕಮಾಂಡ್ 만큼 ಸರಳವಾಗಿದೆ - ಮತ್ತು `azd up` ಉಪಯೋಗಿಸುವ ಮೂಲಕ ನೀವು ಒಂದೇ ಸಮಯದಲ್ಲಿ ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಒದಗಿಸಿ ಅಪ್ಲಿಕೇಶನ್ ನಿಯೋಜಿಸಬಹುದು!

ಪರಿಣಾಮವಾಗಿ, ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅಭಿವೃದ್ಧಿ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಪ್ರಾರಂಭಿಸುವುದು, ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಮತ್ತು ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಅಗತ್ಯಗಳಿಗೆ ಸೂಕ್ತವಾದ _AZD ಸ್ಟಾರ್ಟರ್ ಟೆಂಪ್ಲೇಟ್_ ಸಿಗುವುದನ್ನು ಕಂಡು, ನಂತರ ಅದರ ಸಂಗ್ರಹಣೆಯನ್ನು ನಿಮ್ಮ ಪರಿಸ್ಥಿತಿಗೆ ಅನುಗುಣವಾಗಿ ಕ್ರಮಾಂತರಗೊಳಿಸುವಷ್ಟು ಸುಲಭ.

ಪ್ರಾರಂಭಕ್ಕೆ, ನಿಮ್ಮ ಬಳಿಯಲ್ಲಿ Azure Developer CLI ಸ್ಥಾಪನೆಯಾಗಿದೆ ಎಂದು ಖಚಿತಪಡಿಸೋಣ.

1. VS ಕೋಡ್ ಟರ್ಮಿನಲ್ ತೆರೆಯಿರಿ ಮತ್ತು ಈ ಕಮಾಂಡ್ ಅನ್ನು টাইಪ್ ಮಾಡಿ:

      ```bash title="" linenums="0"
      azd version
      ```

1. ನೀವು ಈ ರೀತಿಯುದನ್ನೇ ನೋಡಬೇಕು!

      ```bash title="" linenums="0"
      azd version 1.27.1 (commit <current-build>)
      ```

**ನೀವು ಈಗ ಟೆಂಪ್ಲೇಟ್ ಆಯ್ಕೆಮಾಡಿ ಅವನೊಂದಿಗೆ ನಿಯೋಜಿಸಲು ಸಿದ್ಧರಾಗಿದ್ದೀರಿ**

## 2. ಟೆಂಪ್ಲೇಟ್ ಆಯ್ಕೆ

Microsoft Foundry ವೇದಿಕೆ [ಸೂಚಿತ AZD ಟೆಂಪ್ಲೇಟ್‌ಗಳ](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) ಸಮೂಹವನ್ನು ಒದಗಿಸುತ್ತದೆ, ಇದು _ಮಲ್ಟಿ-ಏಜೆಂಟ್ ಕಾರ್ಯಪ್ರವಾಹ ಸ್ವಯಂಚಾಲನೆಯು_ ಮತ್ತು _ಬಹು-ಮಾಧ್ಯಮ ವಿಷಯ ಸಂಸ್ಕರಣೆ_ ಸೇರಿದಂತೆ ಜನಪ್ರಿಯ ಪರಿಹಾರ ಸಂದರ್ಭಗಳನ್ನು ಒಳಗೊಂಡಿದೆ. ನೀವು ಈ ಟೆಂಪ್ಲೇಟ್‌ಗಳನ್ನು Microsoft Foundry ಪೋರ್ಟಲ್ ಗೆ ಭೇಟಿ ನೀಡಿ ಕೂಡಾ ಕಂಡುಕೊಳ್ಳಬಹುದು.

1. ಭೇಟಿ ನೀಡಿ [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. ಸಲ್ಲಿಸಲಾದಾಗ Microsoft Foundry ಪೋರ್ಟಲ್‌ಗೆ ಲಾಗಿನ್ ಆಗಿ - ನೀವು ಇದನ್ನು ನೋಡಬಹುದು.

![Pick](../../../../../translated_images/kn/01-pick-template.60d2d5fff5ebc374.webp)


**ಮೂಲಭೂತ** ಆಯ್ಕೆಗಳು ನಿಮ್ಮ ಸ್ಟಾರ್ಟರ್ ಟೆಂಪ್ಲೇಟ್ ಗಳು:

1. [ ] [AI ಚಾಟ್‍ನೊಂದಿಗೆ ಪ್ರಾರಂಭಿಸಿ](https://github.com/Azure-Samples/get-started-with-ai-chat) ಇದು ನಿಮ್ಮ ಡೇಟಾ ಜೊತೆಗೆ ಅಳವಡಿಸಿರುವ ಸರಳ ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು Azure ಕಂಟೆನೆರ್ ಅಪ್ಸ್ಗೆ ನಿಯೋಜಿಸುತ್ತದೆ. ಇದು ವರ್ಕಿಂಗ್ AI ಚಾಟ್‌ಬಾಟ್ ಸನ್ನಿವೇಶ ತಿಳಿಯಲು ಉಪಯುಕ್ತ.
1. [X] [AI ಏಜೆಂಟ್‍ಗಳೊಂದಿಗೆ ಪ್ರಾರಂಭಿಸಿ](https://github.com/Azure-Samples/get-started-with-ai-agents) ಇದು ಸ್ಟಾಂಡರ್ಡ್ AI ಏಜೆಂಟ್ (Foundry ಏಜೆಂಟ್‌ಗಳೊಂದಿಗೆ) ಅನ್ನು ನಿಯೋಜಿಸುತ್ತದೆ. ಇದನ್ನು ಉಪಕರಣಗಳು ಮತ್ತು ಮಾದರಿಗಳೊಂದಿಗೆ ಏಜೆಂಟಿಕ AI ಪರಿಹಾರಗಳ ಪರಿಚಯ ಪಡೆದುಕೊಳ್ಳಲು ಉಪಯೋಗಿಸಿ.

ಎರಡನೇ ಲಿಂಕ್ ಅನ್ನು ಹೊಸ ಬ್ರೌಸರ್ ಟ್ಯಾಬ್ ನಲ್ಲಿ ತೆರೆಯಿರಿ (ಅಥವಾ ಸಂಬಂಧಿತ ಕಾರ್ಡ್‌ನ "GitHub ನಲ್ಲಿ ತೆರೆಯಿ" ಕ್ಲಿಕ್ ಮಾಡಿ). ನೀವು ಈ AZD ಟೆಂಪ್ಲೇಟ್ ಗಾಗಿ ರೆಪೊಸಿಟರಿಯನ್ನು ನೋಡಬಹುದು. README ಅನ್ವೇಷಿಸಲು ಕೆಲವು ನಿಮಿಷಗಳು ತೆಗೆದುಕೊಳ್ಳಿ. ಅಪ್ಲಿಕೇಶನ್ ವಾಸ್ತುಶಿಲ್ಪ ಇದು ಹೀಗಿದೆ:

![Arch](../../../../../translated_images/kn/architecture.8cec470ec15c65c7.webp)

## 3. ಟೆಂಪ್ಲೇಟ್ ಸಕ್ರಿಯಗೊಳಿಸುವಿಕೆ

ಈ ಟೆಂಪ್ಲೇಟ್ ಅನ್ನು ನಿಯೋಜಿಸಿ ಮತ್ತು ಅದು ಮಾನ್ಯವಾಗಿದೆಯೇ ಎಂಬುದನ್ನು ಪರಿಶೀಲಿಸೋಣ. ನಾವು [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) ವಿಭಾಗದಲ್ಲಿನ ಮಾರ್ಗಸೂಚಿಗಳನ್ನು ಅನುಸರಿಸುವೆವು.

1. ಟೆಂಪ್ಲೇಟ್ ರೆಪೊಸಿಟರಿ ಕಾರ್ಯಸ್ಥಳವನ್ನು ಆರಿಸಿ:

      - **GitHub Codespaces**: [ಈ ಲಿಂಕ್](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) ಕ್ಲಿಕ್ ಮಾಡಿ ಮತ್ತು `Create codespace` ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
      - **ಸ್ಥಳೀಯ ಕ್ಲೋನ್ ಅಥವಾ ಡೆವ್ ಕಂಟೈನರ್**: `Azure-Samples/get-started-with-ai-agents` ಅನ್ನು ಕ್ಲೋನ್ ಮಾಡಿ ಮತ್ತು VS ಕೋಡ್ ನಲ್ಲಿ ತೆರೆಯಿರಿ

1. VS ಕೋಡ್ ಟರ್ಮಿನಲ್ ಸಿದ್ಧವಾಗುವವರೆಗೆ ಕಾಯಿರಿ, ನಂತರ ಕೆಳಗಿನ ಕಮಾಂಡ್ ಅನ್ನು ಟೈಪ್ ಮಾಡಿ:

   ```bash title="" linenums="0"
   azd up
   ```

ಈ ಮೂಲಕ ಚಾಲಿತವಾಗುವ ಕಾರ್ಯಪಟು ಹಂತಗಳನ್ನು ಪೂರ್ಣಗೊಳಿಸಿ:

1. ನೀವು Azure ಗೆ ಲಾಗಿನ್ ಮಾಡಲು ಪ್ರಾಂಪ್ಟ್ ಆಗುತ್ತೀರಿ - ಅನುಮೋದನೆಗೆ ಅನುಸರಿಸಿ
1. ನಿಮ್ಮಿಗೆ ಅನನ್ಯ ಪರಿಸರದ ಹೆಸರು ನಮೂದಿಸಿ - ಉದಾ: ನಾನು `nitya-mshack-azd` ಬಳಸಿದೆ
1. ಇದು `.azure/` ಫೋಲ್ಡರ್ ಅನ್ನು ಸೃಷ್ಟಿಸುತ್ತದೆ - ನೀವು ಆ ಪರಿಸರದ ಹೆಸರಿನ ಉಪ-ಫೋಲ್ಡರ್ ಅನ್ನು ನೋಡುತ್ತೀರಿ
1. ಸಬ್ಸ್ಕ್ರಿಪ್ಷನ್ ಹೆಸರನ್ನು ಆರಿಸಲು ಪ್ರಾಂಪ್ಟ್ ಆಗುತ್ತದೆ - ಡೀಫಾಲ್ಟ್ ಆಯ್ಕೆ ಮಾಡಿ
1. ಸ್ಥಳವನ್ನು ಆರಿಸುವಂತೆ ಕೇಳಲಾಗುತ್ತದೆ - `East US 2` ಬಳಸಿ

ಈಗ, ನೀವು ಪ್ರೊವಿಜನಿಂಗ್ ಪೂರ್ಣಗೊಳ್ಳುವವರೆಗೆ ಕಾಯಬೇಕು. **ಇದು 10-15 ನಿಮಿಷಗಳಾಗಬಹುದು**

1. ಪೂರ್ಣಗೊಂಡಾಗ, ನಿಮ್ಮ ಕಾಂಸೋಲ್ ಈ ರೀತಿಯ SUCCESS ಸಂದೇಶ ತೋರಿಸುತ್ತದೆ:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. ಈಗ ನಿಮ್ಮ Azure ಪೋರ್ಟಲ್‌ನಲ್ಲಿ ಆ ಪರಿಸರ ಹೆಸರಿನೊಂದಿಗೆ ನಿರ್ಮಿಸಲಾದ ಸಾಧನ ಗುಂಪು ಇರುತ್ತದೆ:

      ![Infra](../../../../../translated_images/kn/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **ನೀವು ಈಗ ನಿಯೋಜಿಸಲಾದ ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಮತ್ತು ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಮಾನ್ಯಗೊಳಿಸಲು ಸಿದ್ಧರಾಗಿದ್ದೀರಿ**.

## 4. ಟೆಂಪ್ಲೇಟ್ ಮಾನ್ಯತೆ

1. Azure ಪೋರ್ಟಲ್ [Resource Groups](https://portal.azure.com/#browse/resourcegroups) ಪುಟಕ್ಕೆ ಭೇಟಿ ನೀಡಿ - ಸಲ್ಲಿಸುವಾಗ ಲಾಗಿನ್ ಆಗಿ
1. ನಿಮ್ಮ ಪರಿಸರ ಹೆಸರಿಗಾಗಿ RG ಕ್ಲಿಕ್ ಮಾಡಿ - ಮೇಲಿನ ಪುಟವನ್ನು ನೋಡುತ್ತೀರಿ

      - Azure ಕಂಟೆನರ್ ಅಪ್ಸ್ ಸಂಪನ್ಮೂಲವನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ
      - _Essentials_ ವಿಭಾಗದಲ್ಲಿ ಅಪ್ಲಿಕೇಶನ್ URL ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ (ಎಡಕಡೆಯ ಮೇಲ್ಭಾಗದಲ್ಲಿ)

1. ನೀವು ಹೋಸ್ಟ್ ಮಾಡಲಾದ ಅಪ್ಲಿಕೇಶನ್ ಫ್ರಂಟ್-ಎಂಡ್ UI ಅನ್ನು ಈ ರೀತಿಯಾಗಿ ನೋಡುತ್ತೀರಿ:

   ![App](../../../../../translated_images/kn/03-test-application.471910da12c3038e.webp)

1. ಕೆಲವು [ಮಾದರಿ ಪ್ರಶ್ನೆಗಳು](https://github.com/Azure-Samples/get-started-with-ai-agents/blob/main/docs/sample_questions.md) ಕೇಳಿ ಪ್ರಯತ್ನಿಸಿ

      1. ಕೇಳಿ: ```ಫ್ರಾನ್ಸ್ ರಾಜಧಾನಿ ಯಾವುದು?``` 
      1. ಕೇಳಿ: ```೨ ಜನರಿಗೆ $200 ಕ್ಕಿಂತ ಕಡಿಮೆ ಅತ್ಯುತ್ತಮ ಟೇಂಟ್ ಯಾವುದು ಮತ್ತು ಅದರ ಲಕ್ಷಣಗಳೇನು?```

1. ನೀವು ಕೆಳಗಿನಂತೆಯೇ ಉತ್ತರಗಳನ್ನು ಪಡೆಯಬೇಕು. _ಆದರೆ ಇದು ಹೇಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ?_

      ![App](../../../../../translated_images/kn/03-test-question.521c1e863cbaddb6.webp)

## 5. ಏಜೆಂಟ್ ಮಾನ್ಯತೆ

Azure Container App ಕಳುಹಿಸುವ ಎಂಡ್ಪಾಯಿಂಟ್ Microsoft Foundry ಪ್ರಾಜೆಕ್ಟ್‌ನಲ್ಲಿ ಇರುವ AI ಏಜೆಂಟ್‌ಗೆ ಸಂಪರ್ಕಿಸುತ್ತದೆ. ಇದರ ಅರ್ಥ ಏನೆಂದರೆ ನೋಡೋಣ.

1. ನಿಮ್ಮ ಸಂಪನ್ಮೂಲ ಗುಂಪಿನ Azure ಪೋರ್ಟಲ್ _Overview_ ಪುಟಕ್ಕೆ ಹಿಂತಿರುಗಿ

1. ಆ ಪಟ್ಟಿಯಲ್ಲಿ `Microsoft Foundry` ಸಂಪನ್ಮೂಲವನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ

1. ನೀವು ಇವನ್ನು ನೋಡುತ್ತೀರಿ. `Go to Microsoft Foundry Portal` ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಿ. 
   ![Foundry](../../../../../translated_images/kn/04-view-foundry-project.fb94ca41803f28f3.webp)

1. ನಿಮ್ಮ AI ಅಪ್ಲಿಕೇಶನ್ ಗಾಗಿ Foundry ಪ್ರಾಜೆಕ್ಟ್ ಪುಟವನ್ನು ನೋಡುತ್ತೀರಿ
   ![Project](../../../../../translated_images/kn/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` ಅನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ - ನೀವು ನಿಮ್ಮ ಪ್ರಾಜೆಕ್ಟ್‍ನಲ್ಲಿ ನಿಯೋಜಿಸಲಾದ ಡೀಫಾಲ್ಟ್ ಏಜೆಂಟ್ ನೋಡುತ್ತೀರಿ
   ![Agents](../../../../../translated_images/kn/06-visit-agents.bccb263f77b00a09.webp)

## 6. ಏಜೆಂಟ್ ಪ್ಲೇಗ್ರೌಂಡ್

1. ಅದೇ ಪ್ರಶ್ನೆಯನ್ನು ಕೇಳಿ: `Amanda Perez ಯಾವ ಉತ್ಪನ್ನಗಳನ್ನು ಹೊಂದಿದ್ದಾರೆ?`

1. ಯಾವುದೇ ಮೆಟ್ರಿಕ್ ಮೇಲಿಗೆ ಹೋವರ್ ಮಾಡಿ - ಉದಾ: ಸುರಕ್ಷತೆ. ನೀವು ಈ ರೀತಿಯ ವೀಕ್ಷಣೆಯನ್ನು ಕಾಣುತ್ತೀರಿ

## 7. ನಿರ್ಮಿತ ದರ್ಶನಕ್ಷಮತೆ

ದರ್ಶನಕ್ಷಮತೆ ಎಂದರೆ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಮಾಹಿತಿ ಸಂಗ್ರಹಿಸಿ ಅದನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು, ಡಿಬಗ್ ಮಾಡಲು ಮತ್ತು ಉತ್ತಮಗೊಳಿಸಲು ಉಪಯೋಗಿಸುವುದು. ಇದನ್ನು ಅನುಭವಿಸೋಣ:

1. `View Run Info` ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಿ - ನೀವು ಈ ವೀಕ್ಷಣೆಯನ್ನು ನೋಡುತ್ತೀರಿ. ಇದು [ಏಜೆಂಟ್ ಟ್ರೇಸಿಂಗ್](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) ಉದಾಹರಣೆ ಆಗಿದೆ. _ಈ ವೀಕ್ಷಣೆಯನ್ನು ಮೇಲಿನ ಮೆನುನಲ್ಲಿ Thread Logs ಕ್ಲಿಕ್ ಮಾಡುವ ಮೂಲಕ ಕೂಡ ಪಡೆಯಬಹುದು_.

   - ಏಜೆಂಟ್ ಚಾಲನೆಯ ಹಂತಗಳು ಮತ್ತು ಉಪಕರಣಗಳನ್ನು ತಿಳಿದುಕೊಳ್ಳಿ
   - ಉತ್ತರಕ್ಕಾಗಿ ಒಟ್ಟು ಟೋಕನ್ ಸಂಖ್ಯೆಯನ್ನು (ತಯಾರಿಸಿದ ಟೋಕನ್ ಬಳಕೆಯ ವಿರುದ್ಧ) ತಿಳಿದುಕೊಳ್ಳಿ
   - ವಿಳಂಬ ಮತ್ತು ಕಾರ್ಯಾಚರಣೆಯಲ್ಲಿ ಸಮಯ ಕಳೆಯುವ ಸ್ಥಳವನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಿ

      ![Agent](../../../../../translated_images/kn/10-view-run-info.b20ebd75fef6a1cc.webp)

1. `Metadata` ಟ್ಯಾಬ್ ಕ್ಲಿಕ್ ಮಾಡಿ - ನಂತರದ ಡಿಬಗಿಂಗ್‌ಗೆ ಸಹಾಯವಾಗುವ ನಿರ್ದಿಷ್ಟ ಗುಣಲಕ್ಷಣಗಳನ್ನು ನೋಡಿ.   

      ![Agent](../../../../../translated_images/kn/11-view-run-info-metadata.7966986122c7c2df.webp)


1. `Evaluations` ಟ್ಯಾಬ್ ಕ್ಲಿಕ್ ಮಾಡಿ - ಏಜೆಂಟ್ ಉತ್ತರದ ಸ್ವಯಂಚಾಲಿತ ಮೌಲ್ಯಮಾಪನಗಳನ್ನು ನೋಡಿ. ಇವು ಸೇರಿವೆ: ಸುರಕ್ಷತೆ ಮೌಲ್ಯಮಾಪನಗಳು (ಉದಾ: ಸ್ವಂತಹಾನಿ) ಮತ್ತು ಏಜೆಂಟ್ ನಿರ್ದಿಷ್ಟ ಮೌಲ್ಯಮಾಪನಗಳು (ಉದಾ: ಉದ್ದೇಶ ಪರಿಹಾರ, ಕಾರ್ಯ ಇತರೆ).

## 8. ಪರಿಸರ ಚರಗಳು

ಈವರೆಗೆ, ನಾವು ಬ್ರೌಸರ್ ನಲ್ಲಿ ನಿಯೋಜನೆಯನ್ನು ನಡೆಸಿದ್ದೇವೆ - ಮತ್ತು ನಮ್ಮ ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಒದಗಿಸುವುದೂ ಅಪ್ಲಿಕೇಶನೂ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತಿವೆ ಎಂದು ಪರಿಶೀಲಿಸಿದ್ದೇವೆ. ಆದರೆ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು _ಕೋಡ್-ಮೊದಲ_ ರೀತಿಯಲ್ಲಿ ಕೆಲಸ ಮಾಡಲು, ನಾವು ನಮ್ಮ ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿ ಪರಿಸರವನ್ನು ಈ ಸಂಪನ್ಮೂಲಗಳ ಜೊತೆಗೆ ಕೆಲಸ ಮಾಡಲು ಅಗತ್ಯವಿರುವ чರದ ಅಳವಡಿಕೆಗಳೊಂದಿಗೆ ಹೊಂದಿಸಬೇಕಾಗುತ್ತದೆ. `azd` ಬಳಕೆ ಇದನ್ನು ಸುಲಭಗೊಳಿಸುತ್ತದೆ.

1. Azure Developer CLI [ಪರಿಸರ ಚರಗಳನ್ನು](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) ಅಪ್ಲಿಕೇಶನ್ ನಿಯೋಜನೆಗಳಿಗೆ ಸಂರಚನಾ ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು ಸಂಗ್ರಹಿಸಲು ಮತ್ತು ನಿರ್ವಹಿಸಲು ಬಳಸುತ್ತದೆ.

1. ಪರಿಸರ ಚರುಗಳು `.azure/<env-name>/.env` ನಲ್ಲಿ ಸಂಗ್ರಹಿಸಲ್ಪಟ್ಟಿರುತ್ತವೆ - ಇದು ಅವುಗಳನ್ನು `env-name` ಪ್ರದೇಶದಲ್ಲಿ ಸೀಮಿತಗೊಳಿಸುತ್ತದೆ ಮತ್ತು ಒಂದೇ ರೆಪೊನಲ್ಲಿ ವಿವಿಧ ನಿಯೋಜನೆ ಗಳ್ಳೆಗೆ ಪರಿಸರಗಳನ್ನು ವಿಭಜಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

1. `azd` ಕಮಾಂಡ್ ನಿರ್ದಿಷ್ಟ ಆದೇಶ (ಉದಾ: `azd up`) ಕಾರ್ಯಗತಗೊಳಿಸುವಾಗ, ಪರಿಸರ ಚರನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಲೋಡ್ ಮಾಡುತ್ತದೆ. ಗಮನಿಸಿ, `azd` ಸ್ವಚ್ಛಂದವಾಗಿ _OS ಮಟ್ಟದ_ ಪ್ರವೇಶ ರಹಿತ ಪರಿಸರ ಚರುವನ್ನು ಓದುತ್ತಿಲ್ಲ (ಉದಾ: ಶೆಲ್‌ನಲ್ಲಿ ಸೆಟ್ ಮಾಡಿದವು) - ಬದಲಿಗೆ, ಬರಹಗಳಲ್ಲಿ ಮಾಹಿತಿಯನ್ನು ಪ್ರಸಾರ ಮಾಡಲು `azd set env` ಮತ್ತು `azd get env` ಬಳಸಿ.


ಕೆಲವು ಕಮಾಂಡ್‌ಗಳನ್ನು ಪ್ರಯತ್ನಿಸೋಣ:

1. ಈ ಪರಿಸರದಲ್ಲಿ `azd` ಗಾಗಿ ಎಲ್ಲಾ ಪರಿಸರ ಚರುಗಳನ್ನೂ ಪಡೆಯಿರಿ:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      ನೀವು ಈ ರೀತಿಯುದನ್ನೇ ನೋಡುತ್ತೀರಿ:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. ನಿರ್ದಿಷ್ಟ ಮೌಲ್ಯವನ್ನು ಪಡೆಯಿರಿ - ಉದಾಹರಣೆಗಾಗಿ, `AZURE_AI_AGENT_MODEL_NAME` ಮೌಲ್ಯವನ್ನು ನಾವು ಸೆಟ್ ಮಾಡಿದ್ದೇ ಎಂಬುದನ್ನು ತಿಳಿದುಕೊಳ್ಳಲು

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      ನೀವು ಈ ರೀತಿಯನ್ನು ನೋಡುತ್ತೀರಿ - ಇದು ಡೀಫಾಲ್ಟ್‌ನಲ್ಲಿ ಸೆಟ್ ಆಗಿರಲಿಲ್ಲ!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd` ಗಾಗಿ ಹೊಸ ಪರಿಸರ ಚರವನ್ನು ಸೆಟ್ ಮಾಡಿ. ಇಲ್ಲಿ ನಾವು ಏಜೆಂಟ್ ಮಾದರಿ ಹೆಸರನ್ನು ನವೀಕರಿಸುತ್ತೇವೆ. _ಗಮನಿಸಿ: ಯಾವುದೇ ಬದಲಾವಣೆಗಳು ತಕ್ಷಣ `.azure/<env-name>/.env` ಕಡತದಲ್ಲಿ ಪರಾವರ್ತಿತವಾಗುತ್ತವೆ._

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      ಈಗ, ಮೌಲ್ಯ ಸೆಟ್ ಆಗಿದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಬಹುದು:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. ಕೆಲ ಸಂಪನ್ಮೂಲಗಳು ಸ್ಥಿರವಾಗಿವೆ (ಉದಾ: ಮಾದರಿ ನಿಯೋಜನೆಗಳು) ಮತ್ತು ಭಿನ್ನ `azd up` ಅನ್ನು ಮಾಡಬೇಕು ಮರುನಿಯೋಜನೆಯನ್ನು ಬಲಪಡಿಸಲು. ಪ್ರಾರಂಭದ ನಿಯೋಜನೆಯನ್ನು ನಿವಾರಿಸಿ ಮತ್ತು ಬದಲಾದ ಪರಿಸರ ಚರಗಳೊಂದಿಗೆ ಮರುನಿಯೋಜನೆ ಮಾಡೋಣ.

1. **ರಿಫ್ರೆಶ್:** ನೀವು ಹಿಂದಿನ ಒಮ್ಮೆ azd ಟೆಂಪ್ಲೇಟ್ ಬಳಸಿಕೊಂಡು ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ನಿಯೋಜನೆ ಮಾಡಿದ್ದರೆ - `azd` ಅಭಿವೃದ್ದಿಗೊಳಿಸಲು ನಿಮ್ಮ ಸ್ಥಳೀಯ ಪರಿಸರ ಚರಗಳ ಸ್ಥಿತಿಯನ್ನು ಇತ್ತೀಚಿನ Azure ನಿಯೋಜನೆ ಸ್ಥಿತಿಯ ಆಧಾರದ ಮೇಲೆ _ರಿಫ್ರೆಶ್_ ಮಾಡಬಹುದು ಕೆಳಗಿನ ಕಮಾಂಡ್ ಉಪಯೋಗಿಸಿ:

      ```bash title="" linenums="0"
      azd env refresh
      ```


      ಇದು ಎರಡು ಅಥವಾ ಹೆಚ್ಚು ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿ ಪರಿಸರಗಳ ನಡುವೆ ಪರಿಸರ ವ್ಯತ್ಯಾಸಗಳನ್ನು _sync_ ಮಾಡುವ ಶಕ್ತಿಶಾಲಿ ಮಾರ್ಗ (ಉದಾ., ಹಲವಾರು ಅಭಿವೃದ್ಧಿಗಾರರ ತಂಡ) - ನಿಯೋಜಿತ ಮೂಲಸೌಕರ್ಯವು ಪರಿಸರ ವ್ಯತ್ಯಾಸ ಸ್ಥಿತಿಗಾಗಿ ನೆಲೆನಿಲ್ಲಿಸಿರುವ ಸತ್ಯವಾಗಲು ಅವಕಾಶ ನೀಡುತ್ತದೆ. ತಂಡದ ಸದಸ್ಯರು ಸರಳವಾಗಿ ವ್ಯತ್ಯಾಸಗಳನ್ನು _refresh_ ಮಾಡಿ ಹೋಲಿಕೆಮಾಡಿಕೊಳ್ಳುತ್ತಾರೆ.
