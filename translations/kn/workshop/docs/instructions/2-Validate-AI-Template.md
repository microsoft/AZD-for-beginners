## 1. ಪರಿಚಯ

The [Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) ಅಥವಾ `azd` 는 ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು Azure ಗೆ ನಿರ್ಮಿಸುವಾಗ ಮತ್ತು ನಿಯೋಜಿಸುವಾಗ ಡೆವಲಪರ್ ಕಾರ್ಯಪ್ರವಾಹವನ್ನು ಸರಳಗೊಳಿಸುವ ಓಪನ್-ಸೋರ್ಸ್ ಕಮಾಂಡ್‌ಲೈನ್ ಸಾಧನವಾಗಿದೆ. 

[AZD Templates](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) ಪ್ರಮಾಣಿತ ರೆಪೊಸಿಟರಿಗಳು ಆಗಿವೆ, ಅವುಗಳಲ್ಲಿ ಉದಾಹರಣಾ ಅಪ್ಲಿಕೇಶನ್ ಕೋಡ್, _infrastructure-as-code_ ಸಂಪತ್ತಿಗಳು ಮತ್ತು ಸಮ್ಮಿಲಿತ ಪರಿಹಾರ ವಾಸ್ತುಶಿಲ್ಪಕ್ಕಾಗಿ `azd` ಕಾನ್ಫಿಗರೇಶನ್ ಫೈಲ್‌ಗಳು ಸೇರಿರುತ್ತವೆ. ಮೂಲಸೌಕರ್ಯವನ್ನು ಒಮ್ಮೆಗಳಾಗಿ `azd provision` ಕಮಾಂಡ್ ಮೂಲಕ ಸೃಜಿಸಬಹುದು — ಹೀಗಾಗಿ `azd up` ಮೂಲಕ ನೀವು ಮೂಲಸೌಕರ್ಯವನ್ನು **ಮತ್ತು** ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಒಂದೇ ಆದಿಯಲ್ಲಿ ನಿಯೋಜಿಸಬಹುದು!

ಫಲವಾಗಿ, ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅಭಿವೃದ್ಧಿ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಪ್ರಾರಂಭಿಸುವುದು, ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಹಾಗೂ ಮೂಲಸೌಕರ್ಯ ಅಗತ್ಯಗಳಿಗೆ ಅನುರೂಪವಾಗಿರುವ ಸರಿಯಾದ _AZD Starter template_ ಅನ್ನು ಕಂಡುಬಿಟ್ಟು - ನಂತರ ಆ ರೆಪೊಸಿಟರಿಯನ್ನು ನಿಮ್ಮ ದೃಶ್ಯಾವಳಿ ಅಗತ್ಯಗಳಿಗೆ ಅನುಗುಣವಾಗಿ ಕಸ್ಟಮೈಸ್ ಮಾಡುತ್ತಿರುವಷ್ಟೇ ಸುಲಭವಾಗಿದೆ.

ಆರಂಭಿಸುವ ಮೊದಲು, ನಿಮ್ಮ ಬಳಿ Azure Developer CLI ಸ್ಥಾಪಿತವಾಗಿದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸೋಣ.

1. VS Code ಟರ್ಮಿನಲ್ ತೆರೆಯಿ ಮತ್ತು ಈ ಕಮಾಂಡ್ ಅನ್ನು ಟೈಪ್ ಮಾಡಿ:

      ```bash title="" linenums="0"
      azd version
      ```

1. ನಿಮಗೆ ಹೀಗೇ ಕಾಣುವಂತಿರಬೇಕು!

      ```bash title="" linenums="0"
      azd version 1.25.6 (commit <current-build>)
      ```

**ನೀವು ಈಗ azd ಬಳಸಿ ಟೆಂಪ್ಲೇಟ್ ಆಯ್ಕೆಮಾಡಿ ನಿಯೋಜಿಸಲು ಸಿದ್ಧರಾಗಿದ್ದೀರಿ**

## 2. ಟೆಂಪ್ಲೇಟ್ ಆಯ್ಕೆ

Microsoft Foundry ವೇದಿಕೆ [ನಿರ್ದೇಶಿತ AZD ಟೆಂಪ್ಲೇಟುಗಳ ಒಂದು ಸೆಟ್](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) ಜೊತೆಗೆ ಬರುತ್ತದೆ, ಇವು _ಮಲ್ಟಿ-ಏಜೆಂಟ್ ಕಾರ್ಯಪ್ರವಾಹ ಸ್ವಯಂಚಾಲನೆ_ ಮತ್ತು _ಮಲ್ಟಿ-ಮೋಡಲ್ ವಿಷಯ ಪ್ರಕ್ರಿಯೆ_ದಂತಹ ಜನಪ್ರಿಯ ಪರಿಹಾರ ಸನ್ನಿವೇಶಗಳನ್ನು ಕವರ್ ಮಾಡುತ್ತವೆ. ನೀವು Microsoft Foundry ಪೋರ್ಟಲ್ ಗೆ ಭೇಟಿ ನೀಡಿದ್ದಾರೆವುಂದರೆ ಈ ಟೆಂಪ್ಲೇಟುಗಳನ್ನು ಕೂಡ ಹುಡುಕಬಹುದು.

1. Visit [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. ಪ್ರಾಂಪ್ಟ್ ಬಂದಾಗ Microsoft Foundry ಪೋರ್ಟಲ್‌ನಲ್ಲಿ ಲಾಗ್ ಇನ್ ಮಾಡಿ - ನಿಮಗೆ ಹೀಗೇ ಕಾಣುತ್ತದೆ.

![ಆಯ್ಕೆ](../../../../../translated_images/kn/01-pick-template.60d2d5fff5ebc374.webp)


The **Basic** options are your starter templates:

1. [ ] [Get Started with AI Chat](https://github.com/Azure-Samples/get-started-with-ai-chat) ಇದು ನಿಮ್ಮ ಡೇಟಾ ಜೊತೆಗೆ Azure Container Apps ಗೆ ಮೂಲ ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ನಿಯೋಜಿಸುತ್ತದೆ. ಮೂಲ AI ಚಾಟ್‌ಬಾಟ್ ಸನ್ನಿವೇಶವನ್ನು ಅನ್ವೇಷಿಸಲು ಇದನ್ನು ಬಳಸಿ. _with your data_
1. [X] [Get Started with AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) ಇದು Foundry Agents ಜೊತೆಗೆ ಸ್ಟ್ಯಾಂಡರ್ಡ್ AI ಏಜೆಂಟ್ ಅನ್ನು ನಿಯೋಜಿಸುತ್ತದೆ. ಉಪಕರಣಗಳು ಮತ್ತು ಮಾದರಿಗಳನ್ನು ಒಳಗೊಂಡ ಏಜೆಂಟ್-ಆಧಾರಿತ AI ಪರಿಹಾರಗಳೊಂದಿಗೆ ಪರಿಚಿತರಾಗಲು ಇದನ್ನು ಬಳಸಿ.

ದ್ವಿತೀಯ ಲಿಂಕ್ ಅನ್ನು ಹೊಸ ಬ್ರೌಸರ್ ಟ್ಯಾಬ್‌ನಲ್ಲಿ ತೆರೆಯಿರಿ (ಅಥವಾ ಸಂಬಂಧಿತ ಕಾರ್ಡಿನ `Open in GitHub` ಕ್ಲಿಕ್ ಮಾಡಿ). ನೀವು ಈ AZD ಟೆಂಪ್ಲೇಟ್‌ನ ರೆಪೊಸಿಟರಿ ನೋಡಬಹುದು. README ಅನ್ನು ಪರಿಶೀಲಿಸಲು ನಿಮಗೆ ಕೆಲವು ಕ್ಷಣ ತೆಗೆದುಕೊಳ್ಳಿ. ಅಪ್ಲಿಕೇಶನ್ ವಾಸ್ತುಶಿಲ್ಪ ಹೀಗಿದೆ:

## 3. ಟೆಂಪ್ಲೇಟು ಸಕ್ರಿಯಗೊಳಿಸುವುದು

ಈテンプレート ಅನ್ನು ನಿಯೋಜಿಸಿ ಮತ್ತು ಅದು ಮಾನ್ಯವಾಗಿದೆಯೇ ಎಂದು ಖಚಿತಪಡಿಸೋಣ. ನಾವು [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) ವಿಭಾಗದಲ್ಲಿರುವ ಮಾರ್ಗದರ್ಶಿಗಳನ್ನು ಅನುಸರಿಸುತ್ತೇವೆ.

1. ಟೆಂಪ್ಲೇಟ್ ರೆಪೊಸಿಟರಿ‌ಗೆ ಕಾರ್ಯನಿರ್ವಹಣಾ ವಾತಾವರಣ ಆಯ್ಕೆಮಾಡಿ:

      - **GitHub Codespaces**: [ಈ ಲಿಂಕ್](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) ಕ್ಲಿಕ್ ಮಾಡಿ ಮತ್ತು `Create codespace` ಅನ್ನು ದೃಢೀಕರಿಸಿ
      - **Local clone or dev container**: `Azure-Samples/get-started-with-ai-agents` ಕ್ಲೋನ್ ಮಾಡಿ ಮತ್ತು ಅದನ್ನು VS Code ನಲ್ಲಿ ತೆರೆಯಿರಿ

1. VS Code ಟರ್ಮಿನಲ್ ಸಿದ್ಧವಾಗುವವರೆಗೆ ಕಾಯಿರಿ, ನಂತರ ಕೆಳಗಿನ ಕಮಾಂಡ್ ಅನ್ನು ಟೈಪ್ ಮಾಡಿ:

   ```bash title="" linenums="0"
   azd up
   ```

Complete the workflow steps that this will trigger:

1. ಲಾಗ್ ಇನ್ ಮಾಡಲು ಪ್ರಾಂಪ್ಟ್ ಬರುತ್ತದೆ - ಪ್ರಾಮಾಣೀಕರಿಸಲು ಸೂಚನೆಗಳನ್ನು ಅನುಸರಿಸಿ
1. ನಿಮ್ಮಿಗಾಗಿ ಒಂದು ವಿಭಿನ್ನ ವಾತಾವರಣ ಹೆಸರನ್ನು ನಮೂದಿಸಿ - ಉದಾಹರಣೆಗೆ, ನಾನು `nitya-mshack-azd` ಬಳಸಿದೆ
1. ಇದು  `.azure/` ಫೋಲ್ಡರ್ ಅನ್ನು ರಚಿಸುತ್ತದೆ - ನೀವು env ಹೆಸರಿನೊಂದಿಗೆ ಒಂದು ಉಪಫೋಲ್ಡರ್ ನೋಡುತ್ತೀರಿ
1. ಸಬ್‌ಸ್ಕ್ರಿಪ್ಷನ್ ಹೆಸರನ್ನು ಆಯ್ಕೆಮಾಡಲು ಪ್ರಾಂಪ್ಟ್ ಆಗುತ್ತದೆ - ಡೀಫಾಲ್ಟ್ ಅನ್ನು ಆಯ್ಕೆಮಾಡಿ
1. ಸ್ಥಳ (location) ಗಾಗಿ ಪ್ರಾಂಪ್ಟ್ ಆಗುತ್ತದೆ - `East US 2` ಅನ್ನು ಬಳಸಿ

ಈಗ, provisioning ಪೂರ್ಣಗೊಳ್ಳುವುದನ್ನು ನಿರೀಕ್ಷಿಸಿ. **ಇದರಿಗೂ 10-15 ನಿಮಿಷಗಳು ಬೇಕಾಗಬಹುದು**

1. ಪೂರ್ಣವಾದಾಗ, ನಿಮ್ಮ ಕನ್‍ಸೋಲ್‌ನಲ್ಲಿ SUCCESS ಸಂದೇಶ ಹೀಗಿವೆಂದು ತೋರುತ್ತದೆ:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. ನಿಮ್ಮ Azure Portal ಈಗ ಆ env ಹೆಸರಿನೊಂದಿಗೆ ಒಂದು ಪ್ರೊವೈಶನ್ಡ್ ರಿಸೋರ್ಸ್ ಗುಂಪನ್ನು ಹೊಂದಿರುತ್ತದೆ:

      ![ಇನ್ಫ್ರಾ](../../../../../translated_images/kn/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **ನೀವು ಈಗ ನಿಯೋಜಿಸಲಾದ ಮೂಲಸೌಕರ್ಯ ಮತ್ತು ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಮಾನ್ಯಗೊಳ್ಳಿಸಲು ಸಿದ್ಧರಾಗಿದ್ದೀರಿ**.

## 4. ಟೆಂಪ್ಲೇಟು ಮಾನ್ಯಗೊಳಿಸುವಿಕೆ

1. Azure Portal [ಸಂಪನ್ಮೂಲ ಗುಂಪುಗಳು](https://portal.azure.com/#browse/resourcegroups) ಪುಟಕ್ಕೆ ಭೇಟಿ ನೀಡಿ - ಪ್ರಾಂಪ್ಟ್ ಬಂದಾಗ ಲಾಗಿನ್ ಮಾಡಿ
1. ನಿಮ್ಮ ವಾತಾವರಣ ಹೆಸರಿನ RG ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ - ಮೇಲಿನ ಪುಟವನ್ನು ನೀವು ನೋಡುತ್ತೀರಿ

      - Azure Container Apps ಸಂಪನ್ಮೂಲದ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ
      - _Essentials_ ವಿಭಾಗದಲ್ಲಿ (ಮೇಲ್ಭಾಗದ ಬಲ) Application Url ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ

1. ಹೀಗೆ ಹೋಸ್ಟ್ ಮಾಡಿದ ಅಪ್ಲಿಕೇಶನ್ ಫ್ರಂಟ್-ಎಂಡ್ UI ಅನ್ನು ನೀವು ನೋಡಬೇಕು:

   ![ಅಪ್ಲಿಕೇಶನ್](../../../../../translated_images/kn/03-test-application.471910da12c3038e.webp)

1. ಕೆಲವು [ನಿದರ್ಶನ ಪ್ರಶ್ನೆಗಳನ್ನು](https://github.com/Azure-Samples/get-started-with-ai-agents/blob/main/docs/sample_questions.md) ಕೇಳಿ

      1. ಕೇಳಿ: ```What is the capital of France?``` 
      1. ಕೇಳಿ: ```What's the best tent under $200 for two people, and what features does it include?```

1. ನೀವು ಕೆಳಗಿನಂತೆ ಸಮೀಪವಾದ ಉತ್ತರಗಳನ್ನು ಪಡೆಯಬೇಕು. _ಆದರೆ ಇದು ಹೇಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ?_

      ![ಪ್ರಶ್ನೆ](../../../../../translated_images/kn/03-test-question.521c1e863cbaddb6.webp)

## 5.  ಏಜೆಂಟ್ ಪರಿಶೀಲನೆ

Azure Container App ಒಂದು ಎಂಡ್ಪಾಯಿಂಟ್ ಅನ್ನು ನಿಯೋಜಿಸುತ್ತದೆ ಅದು ಈ ಟೆಂಪ್ಲೇಟ್‌ಗೆ provision ಮಾಡಿದ Microsoft Foundry ಪ್ರಾಜೆಕ್ಟ್‌ನಲ್ಲಿ provision ಆದ AI ಏಜೆಂಟ್ನೊಂದಿಗೆ ಸಂಪರ್ಕಿಸುತ್ತದೆ. ಅದರ ಅರ್ಥ ಏನೆಂದು ನೋಡಿ.

1. ನಿಮ್ಮ ರಿಸೋರ್ಸ್ ಗುಂಪಿನ Azure Portal _Overview_ ಪುಟಕ್ಕೆ ಮರಳಿ ಹೋಗಿ

1. ಆ ಪಟ್ಟಿಯಲ್ಲಿ `Microsoft Foundry` ಸಂಪನ್ಮೂಲದ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ

1. ನೀವು ಹೀಗೆ ನೋಡುತ್ತೀರಿ. `Go to Microsoft Foundry Portal` ಬಟನ್ ಅನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ. 
   ![ಫೌಂಡ್ರಿ](../../../../../translated_images/kn/04-view-foundry-project.fb94ca41803f28f3.webp)

1. ನಿಮ್ಮ AI ಅಪ್ಲಿಕೇಶನ್‌ಗಾಗಿ Foundry Project ಪುಟವನ್ನು ನೀವು ನೋಡಬೇಕು
   ![ಪ್ರಾಜೆಕ್ಟ್](../../../../../translated_images/kn/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ - ನಿಮ್ಮ ಪ್ರಾಜೆಕ್ಟ್‌ನಲ್ಲಿ ಡೀಫಾಲ್ಟ್ ಏಜೆಂಟ್ provision ಆಗಿ ಇದೆ ಎಂದು ನೀವು ನೋಡುತ್ತೀರಿ
   ![ಏಜೆಂಟ್ಸ್](../../../../../translated_images/kn/06-visit-agents.bccb263f77b00a09.webp)

1. ಅದನ್ನು ಆಯ್ಕೆಮಾಡಿ - ಮತ್ತು ನೀವು ಏಜೆಂಟ್ ವಿವರಗಳನ್ನು ನೋಡುತ್ತೀರಿ. ಕೆಳಗಿನ ವಿಷಯಗಳಿಗ ಗಮನ ನೀಡಿ:

      - ಏಜೆಂಟ್ ಡೀಫಾಲ್ಟ್ olaraq File Search ನ್ನು ಬಳಸುತ್ತದೆ (ಯಾವಾಗಲೂ)
      - ಏಜೆಂಟ್‌ನ `Knowledge` ಸೂಚಿಸುತ್ತದೆ ಅದರಲ್ಲಿ 32 ಫೈಲ್‌ಗಳು ಅಪ್‍ಲೋಡ್ ಆಗಿವೆ (ಫೈಲ್ ಶೋಧನಕ್ಕೆ)
      ![ಏಜೆಂಟ್ಸ್](../../../../../translated_images/kn/07-view-agent-details.0e049f37f61eae62.webp)

## 7. ನಿರ್ಮಿತ ಅವಲೋಕನ

ಅವಲೋಕನ ಎಂದರೆ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಇನ್ಸ್ಟ್ರೂಮೆಂಟ್ ಮಾಡಿ ಅಂಶಗಳನ್ನು ಉತ್ಪಾದಿಸುವುದರ ಬಗ್ಗೆ, ಅವುಗಳನ್ನು ಬಳಸಿ ಅದರ ಕಾರ್ಯಾಚರಣೆಯನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು, ದೋಷನಿರ್ವಹಿಸಲು ಮತ್ತು tốiಮೈಸ್ಪಠಿಸಲು ಸಾಧ್ಯವಾಗುತ್ತದೆ. ಇದಕ್ಕಾಗಿ ಕೆಲವು ವಿಷಯಗಳನ್ನು ನೋಡಿ:

1. `View Run Info` ಬಟನ್ ಅನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ - ನಿಮಗೆ ಈ ವೀಕ್ಷಣೆ ಕಾಣುತ್ತದೆ. ಇದು [Agent tracing](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) ನ ಉದಾಹರಣೆ. _ನೀವು ಈ ವೀಕ್ಷಣೆಯನ್ನು Thread Logs ಅನ್ನು ಟಾಪ್-ಲೆವೆಲ್ ಮೆನುಯಲ್ಲಿ ಕ್ಲಿಕ್ ಮಾಡುವ ಮೂಲಕವೂ ಪಡೆಯಬಹುದು_.

   - ರನ್ ಹಂತಗಳು ಮತ್ತು ಏಜೆಂಟ್ ವ್ಯವಹಾರದಲ್ಲಿ ಭಾಗವಹಿಸಿದ ಉಪಕರಣಗಳನ್ನು ತಿಳುಕೊಳ್ಳಿ
   - ಪ್ರತಿಕ್ರಿಯಾದ ಒಟ್ಟು Token ಎಣಿಕೆ (ಮತ್ತು output tokens ಬಳಕೆ) ಅನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಿ
   - ಲ್ಯಾಟೆನ್ಸಿ ಮತ್ತು ನಿರ್ವಹಣೆಯಲ್ಲಿ ಸಮಯ ಎಲ್ಲ ಕಡೆಗೆ ಖರ್ಚಾಗುತ್ತಿದೆ ಎಂಬುದನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಿ

      ![ಎಜೆಂಟ್](../../../../../translated_images/kn/10-view-run-info.b20ebd75fef6a1cc.webp)

1. `Metadata` ಟ್ಯಾಬ್ ಅನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ ರನ್‌ಗೆ ಸಂಬಂಧಿಸಿದ ಹೆಚ್ಚುವರಿ ಗುಣಲಕ್ಷಣಗಳನ್ನು ನೋಡಿ, ಅವು ಭವಿಷ್ಯದಲ್ಲಿ ದೋಷನಿರ್ವಹಣೆಗಾಗಿ ಉಪಯುಕ್ತ ಹಿನ್ನೆಲೆ ನೀಡಬಹುದು.   

      ![ಎಜೆಂಟ್](../../../../../translated_images/kn/11-view-run-info-metadata.7966986122c7c2df.webp)


1. `Evaluations` ಟ್ಯಾಬ್ ಅನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ ಏಜೆಂಟ್ ಪ್ರತಿಕ್ರಿಯೆಯ ಮೇಲಿನ ಸ್ವಯಂ-ಮೊತ್ತದ ಅಂದಾಜುಗಳನ್ನು ನೋಡಿ. ಇದರಲ್ಲಿ ಭದ್ರತಾ ಅಂದಾಜುಗಳು (ಉದಾ., Self-harm) ಮತ್ತು ಏಜೆಂಟ್-ವಿಶಿಷ್ಟ ಅಂದಾಜುಗಳು (ಉದಾ., ಉದ್ದೇಶ ಪರಿಹಾರ, ಕಾರ್ಯಾನುಸರಣ) ಇರಬಹುದು.

      ![ಎಜೆಂಟ್](../../../../../translated_images/kn/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. ಕೊನೆಗೆ, ಸೈಡ್‌ಬಾರ್ ಮೆನುವಿನಲ್ಲಿ `Monitoring` ಟ್ಯಾಬ್ ಅನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ.

      - ಪ್ರದರ್ಶಿತ ಪುಟದಲ್ಲಿ `Resource usage` ಟ್ಯಾಬ್ ಅನ್ನು ಆಯ್ಕೆಮಾಡಿ - ಮತ್ತು ಮೆಟ್ರಿಕ್‌ಗಳನ್ನು ವೀಕ್ಷಿಸಿ.
      - ಖರ್ಚು (ಟೋಕನ್ಗಳು) ಮತ್ತು ಲೋಡ್ (ಕೋರಿಕೆಗಳು) ದೃಷ್ಟಿಯಿಂದ ಅಪ್ಲಿಕೇಶನ್ ಬಳಕೆಯನ್ನು ಟ್ರ್ಯಾಕ್ ಮಾಡಿ.
      - ಫಸ್ಟ್ ಬೈಟ್ (ಇನ್‌ಪುಟ್ ಪ್ರೊಸೆಸಿಂಗ್) ಮತ್ತು ಲಾಸ್ಟ್ ಬೈಟ್ (ಔಟ್‌ಪುಟ್) ಗೆ ಅಪ್ಲಿಕೇಶನ್ ಲ್ಯಾಟೆನ್ಸಿಯನ್ನು ಟ್ರ್ಯಾಕ್ ಮಾಡಿ.

## 8. ಪರಿಸರ ಚರಗಳು

ಇನ್ನೂವರೆಗೆ ನಾವು ಬ್ರೌಸರ್‌ನಲ್ಲಿ ನಿಯೋಜನೆ ಹೇಗಿದೆ ಮತ್ತು ನಮ್ಮ ಮೂಲಸೌಕರ್ಯ provision ಆಗಿದೆ ಮತ್ತು ಅಪ್ಲಿಕೇಶನ್ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತಿದೆ ಎಂಬುದನ್ನು ಪರಿಶೀಲಿಸಿದ್ದೇವೆ. ಆದರೆ ಕೋಡ್-ಪ್ರಥಮವಾಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸಲು, ಈ ಸಂಪನ್ಮೂಲಗಳೊಂದಿಗೆ ಕೆಲಸ ಮಾಡಲು ಸಂಬಂಧಿಸಿದ ಚರಗಳನ್ನು ನಮ್ಮ ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿ ವಾತಾವರಣದಲ್ಲಿ ಕಾನ್ಫಿಗರ್ ಮಾಡಬೇಕಾಗುತ್ತದೆ. `azd` ಬಳಕೆ ಇದನ್ನು ಸುಲಭಗೊಳಿಸುತ್ತದೆ.

1. Azure Developer CLI [uses environment variables](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) ಅಪ್ಲಿಕೇಶನ್ ನಿಯೋಜನೆಗಳಿಗಾಗಿ ಕಾನ್ಫಿಗರೇಶನ್ ಸೆಟ್ಟಿಂಗ್ಸ್ ಸಂಗ್ರಹಿಸಲು ಮತ್ತು ನಿರ್ವಹಿಸಲು.

1. ಪರಿಸರ ಚರಗಳು `.azure/<env-name>/.env` ನಲ್ಲಿ ಸಂಗ್ರಹಿಸಲಾಗುತ್ತವೆ - ಇದು ಅವುಗಳನ್ನು ನಿಯೋಜನೆ ವೇಳೆ ಬಳಸಿದ `env-name` ವಾತಾವರಣಕ್ಕೆ ಸೀಮಿತಗೊಳುತ್ತದೆ ಮತ್ತು ಒಂದೇ ರೆಪೊದಲ್ಲಿ ವಿಭಿನ್ನ ನಿಯೋಜನೆ ಗುರಿಗಳ ನಡುವೆ ವಾತಾವರಣಗಳನ್ನು ವಿಭಜಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

1. `azd` ಕಮಾಂಡ್ ನಿರ್ದಿಷ್ಟ ಕಮಾಂಡ್ ಅನ್ನು ನಿರ್ವಹಿಸುವಾಗ (ಉದಾ., `azd up`) ಪರಿಸರ ಚರಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಲೋಡ್ ಮಾಡುತ್ತದೆ. ಗಮನಿಸಿ, `azd` ಸ್ವಯಂಪ್ರೇರಿತವಾಗಿ _OS-level_ ಪರಿಸರ ಚರಗಳನ್ನು (ಉದಾ., ಶೆಲ್‌ನಲ್ಲಿ ಸೆಟ್ ಮಾಡಿದಂತೆ) ಓದುವುದಿಲ್ಲ — ಬದಲಾಗಿ ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳಲ್ಲಿ ಮಾಹಿತಿ ಹಸ್ತಾಂತರಿಸಲು `azd set env` ಮತ್ತು `azd get env` ಅನ್ನು ಬಳಸಿ.


ಏನು ಮಾಡುತ್ತದೊ ನೋಡಿ ಕೆಲವು ಕಮಾಂಡ್‌ಗಳನ್ನು ಪರೀಕ್ಷಿಸೋಣ:

1. ಈ ವಾತಾವರಣದಲ್ಲಿ `azd` ಗೆ ಸೆಟ್ ಮಾಡಲ್ಪಟ್ಟ ಎಲ್ಲಾ ಪರಿಸರ ಚರಗಳನ್ನು ಪಡೆಯಿರಿ:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      ನೀವು ಹೀಗೆ ಕಾಣಬಹುದು:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. ನಿರ್ದಿಷ್ಟ ಮೌಲ್ಯವನ್ನು ಪಡೆಯಿರಿ - ಉದಾಹರಣೆಗೆ, ನಾವು `AZURE_AI_AGENT_MODEL_NAME` ಮೌಲ್ಯವನ್ನು ಸೆಟ್ ಮಾಡಿದ್ದೇವೆ ಎಂದು ತಿಳಿದುಕೊಳ್ಳಲು ನಾನು ಬಯಸುತ್ತೇನೆ

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      ನೀವು ಹೀಗೇ ಕಾಣಬಲ್ಲಿರಿ - ಡೀಫಾಲ್ಟ್ ಮೂಲಕ ಇದು ಸೆಟ್ ಆಗಿರಲಿಲ್ಲ!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd`ಗಾಗಿ ಹೊಸ ಪರಿಸರ ಚರವನ್ನು ಸೆಟ್ ಮಾಡಿ. ಇಲ್ಲಿ, ನಾವು ಏಜೆಂಟ್ ಮಾದರಿ ಹೆಸರನ್ನು ನವೀಕರಿಸುತ್ತೇವೆ. _ನೋಟ್: ಯಾವುದೇ ಬದಲಾವಣೆಗಳು `.azure/<env-name>/.env` ಫೈಲ್‌ನಲ್ಲಿ ತಕ್ಷಣ ಪ್ರತಿಫಲಿಸುತ್ತವೆ._

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      ಈಗ, ಮೌಲ್ಯ ಸೆಟ್ ಆಗಿದೆಯೆ ಎಂದು ನಾವು ಕಾಣಬೇಕು:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. ಕೆಲವು ಸಂಪನ್ಮೂಲಗಳು ಸ್ಥಿರವಾಗಿರುತ್ತವೆ (ಉದಾ., ಮಾದರಿ ನಿಯೋಜನೆಗಳು) ಮತ್ತು ಅವುಗಳನ್ನು ಪುನರ್‌ನಿಯೋಜಿಸುವಿಗಾಗಿ ಕೇವಲ `azd up` ಕافي ಆಗುವುದಿಲ್ಲ. ಮೂಲ ನಿಯೋಜನೆಯನ್ನು tear down ಮಾಡಿ ಮತ್ತು ಬದಲಿಸಲಾದ env vars ಜೊತೆಗೆ ಮರುನಿಯೋಜಿಸೋಣ ಎಂದು ಪ್ರಯತ್ನಿಸೋಣ.

1. **Refresh** If you had previously deployed infrastructure using an azd template - you can _refresh_ the state of your local environment variables based on the current state of your Azure deployment using this command:

      ```bash title="" linenums="0"
      azd env refresh
      ```

      ಇದು ಎರಡು ಅಥವಾ ಹೆಚ್ಚು ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿ ಪರಿಸರಗಳು (e.g., ಬಹು ಡೆವಲಪರ್‌ಗಳ ತಂಡ) ನಡುವಿನ ಪರಿಸರ ಚರಗಳನ್ನು _ಸಿಂಕ್_ ಮಾಡಲು ಶಕ್ತಿಶಾಲಿ ವಿಧಾನವಾಗಿದೆ - ನಿಯೋಜಿಸಲಾದ ಮೂಲಸೌಕರ್ಯವು ಪರಿಸರ ಚರಗಳ ಸ್ಥಿತಿಗೆ ನಿಜವಾದ ಮೂಲವಾಗಿ ಸೇವೆ ನೀಡಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ. ತಂಡದ ಸದಸ್ಯರು ಮತ್ತೆ ಸಮ್ಮೇಳನಕ್ಕೆ ಬರುವುದಕ್ಕಾಗಿ ಚರಗಳನ್ನು ಸರಳವಾಗಿ _ರಿಫ್ರೆಶ್_ ಮಾಡುತ್ತಾರೆ.
