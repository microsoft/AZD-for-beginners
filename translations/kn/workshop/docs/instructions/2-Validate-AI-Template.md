## 1. ಪರಿಚಯ

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) ಅಥವಾ `azd` ಎಂದರೆ ಆಜೂರ್‌ಗೆ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿರ್ಮಿಸಿ ನಿಯೋಜಿಸುವಾಗ ಅಭಿವೃದ್ಧಿಪೇಯರುಗಳು ಅನುಸರಿಸುವ ಕೆಲಸಪ್ರವಾಹವನ್ನು ಸುಗಮಗೊಳಿಸುವ open-source ಕಮಾಂಡ್‌ಲೇನ್ ಸಾಧನ.

[AZD Templates](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) ಎಂದರೆ ಮಾದರಿ ಅಪ್ಲಿಕೇಶನ್ ಕೋಡ್, _infrastructure-as-code_ ಸಂಪತ್ತುಗಳು, ಮತ್ತು `azd` ಸಂರಚನಾ ಕಡತಗಳನ್ನು ಒಳಗೊಂಡಿರುವ ಮಾನಕಿತ ಸಂಗ್ರಹಗಳು, ಒಂದು ಸಮನ್ವಯಿತ ಪರಿಹಾರ ವಾಸ್ತುಶಿಲ್ಪಕ್ಕಾಗಿ. ಮೂಲಸೌಕರ್ಯವನ್ನು ಒದಗಿಸುವುದು `azd provision` ಕಮಾಂಡ್‌ಷ್ಟು ಸಹಜವಾಗಿದ್ದು - ಮತ್ತು `azd up` ಬಳಸಿದರೆ ನೀವು ಒಮ್ಮೆಗೇ ಮೂಲಸೌಕರ್ಯವನ್ನು ಒದಗಿಸಿ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ನಿಯೋಜಿಸಬಹುದು!

ಅದು ನಿಮಗೆ ಪರಿಹಾರವಸ್ತುಗಳು ಮತ್ತು ಮೂಲಸೌಕರ್ಯ ಅಗತ್ಯಗಳಿಗೆ ಹೆಚ್ಚು ಸಮೀಪ ಇರುವ ಸರಿಯಾದ _AZD Starter template_ ಅನ್ನು ಹುಡುಕಿ ನಂತರ ಸ್ವತಃ ನಿಮ್ಮ სცೇನಾರಿಯೋ ಅಗತ್ಯಗಳಿಗೆ ಅನುಗುಣವಾಗಿ ಸಂಗ್ರಹವನ್ನು ಕಸ್ಟಮೈಜ್ ಮಾಡುವ ಮೂಲಕ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅಭಿವೃದ್ಧಿ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಆರಂಭಿಸುವುದು ಬಹಳ engineered ಆಗಿರುತ್ತದೆ.

ಮುಂದೆಗಡೆ ಹೋಗುವುದಕ್ಕೆ ಮೊದಲಿಗೆ, ನಿಮ್ಮ ಬಳಿ Azure Developer CLI ಸ್ಥಾಪಿತವೋ ಇಲ್ಲವೋ ಅನುಶೀಲಿಸೋಣ.

1. VS Code ಟೆರ್ಮಿನಲ್ ತೆರೆಯಿರಿ ಮತ್ತು ಈ ಕಮಾಂಡ್ ಟೈಪ್ ಮಾಡಿ:

      ```bash title="" linenums="0"
      azd version
      ```

1. ನಿಮಗೆ ಇದರಲ್ಲಿ ಹೀಗೆ ಏನೋ ಕಾಣಿಸಬೇಕು!

      ```bash title="" linenums="0"
      azd version 1.19.0 (commit b3d68cea969b2bfbaa7b7fa289424428edb93e97)
      ```

**ನೀವು ಈಗ azd ಬಳಸಿ ಟೆಂಪ್ಲೇಟನ್ನು ಆಯ್ಕೆಮಾಡಿ ನಿಯೋಜಿಸಲು ಸಿದ್ಧರಾಗಿದ್ದೀರಿ**

## 2. ಟೆಂಪ್ಲೇಟ್ ಆಯ್ಕೆ

Microsoft Foundry ವೇದಿಕೆ [ಸುನಿವೇಶಿತ AZD ಟೆಂಪ್ಲೇಟುಗಳ ಸેટ್](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) ಜೊತೆಗೆ ಬರುತ್ತದೆ, ಇವು ಬಹುಪ್ರಚಲಿತ ಪರಿಹಾರ ದೃಶ್ಯಾವಳಿಗಳನ್ನು ಆವರಿಸುತ್ತದೆ, ಉದಾಹರಣೆಗೆ _multi-agent workflow automation_ ಮತ್ತು _multi-modal content processing_. ನೀವು ಈ ಟೆಂಪ್ಲೇಟುಗಳನ್ನು Microsoft Foundry ಪೋರ್ಟಲ್ ಮೂಲಕಲೂ ಅನ್ವೇಷಿಸಬಹುದು.

1. [https://ai.azure.com/templates](https://ai.azure.com/templates) ಭೇಟಿ ನೀಡಿ
1. ಪ್ರಾಂಪ್ಟ್ ಬಂದಾಗ Microsoft Foundry ಪೋರ್ಟಲಿಗೆ ಲಾಗಿನ್ ಮಾಡಿ - ನಿಮಗೆ ಹೀಗೇ ಏನೋ ಕಾಣಿಸಬಹುದು.

![ಆಯ್ಕೆ](../../../../../translated_images/kn/01-pick-template.60d2d5fff5ebc374.webp)


**Basic** ಆಯ್ಕೆಗಳು ನಿಮ್ಮ ಪ್ರಾರಂಭಿಕ ಟೆಂಪ್ಲೇಟುಗಳು:

1. [ ] [Get Started with AI Chat](https://github.com/Azure-Samples/get-started-with-ai-chat) ಇದು ಒಂದು ಮೂಲಭೂತ ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ನಿಮ್ಮ ಡೇಟಾ ಜೊತೆಗೆ Azure Container Apps ಗೆ ನಿಯೋಜಿಸುತ್ತದೆ. ಮೂಲಭೂತ AI ಚಾಟ್‌ಬಾಟ್ ದೃಶ್ಯಾವಳಿಯನ್ನು ಅನ್ವೇಷಿಸಲು ಇದನ್ನು ಬಳಸಿ.
1. [X] [Get Started with AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) ಇದು Foundry Agents ಜೊತೆಸಹಿತ ಒಂದು ಮಾನಕ AI Agent ಅನ್ನು ನಿಯೋಜಿಸುತ್ತದೆ. ಸಾಧನ ಮತ್ತು ಮಾದರಿಗಳನ್ನು ಒಟ್ಟಿಗೆ ಹೊಂದಿರುವ agentic AI ಪರಿಹಾರಗಳನ್ನು ಪರಿಚಯವಾಗಲು ಇದನ್ನು ಬಳಸಿ.

ರಿಲೇಟೆಡ್ ಕಾರ್ಡ್‌ನಲ್ಲಿ `Open in GitHub` ಕ್ಲಿಕ್ ಮಾಡಿ ಅಥವಾ ಎರಡನೇ ಲಿಂಕ್ ಅನ್ನು ಹೊಸ ಬ್ರೌಸರ್ ಟ್ಯಾಬ್‌ನಲ್ಲಿ ತೆರೆಯಿರಿ. ನಿಮಗೆ ಈ AZD ಟೆಂಪ್ಲೇಟಿಗಾಗಿ ರೆಪೊಸಿಟರಿ ಕಾಣಿಸಬೇಕು. README ಅನ್ನು ಸ್ವಲ್ಪ ಸಮಯ ಕಂಡುಹಿಡಿಯಿರಿ. ಅಪ್ಲಿಕೇಶನ್ ವಾಸ್ತುಶಿಲ್ಪ ಹೀಗೆ ಕಾಣುತ್ತದೆ:

## 3. ಟೆಂಪ್ಲೇಟು ಸಕ್ರಿಯಗೊಳಿಸುವಿಕೆ

ಈ ಟೆಂಪ್ಲೇಟನ್ನು ನಿಯೋಜಿಸಿ ಅದು ಮಾನ್ಯವಾಗಿದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸೋಣ. ನಾವು [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) ವಿಭಾಗದಲ್ಲಿ ನೀಡಿರುವ ಮಾರ್ಗಸೂಚಿಗಳನ್ನು ಅನುಸರಿಸೋಣ.

1. [この ಲಿಂಕ್ ಕ್ಲಿಕ್ ಮಾಡಿ](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) - ಡೀಫಾಲ್ಟ್ ಕ್ರಿಯೆಯನ್ನು `Create codespace` ಎಂದು ದೃಢೀಕರಿಸಿ
1. ಇದು ಹೊಸ браузರ್ ಟ್ಯಾಬ್ ತೆರೆಯುತ್ತದೆ - GitHub Codespaces ಸೆಷನ್ ಲೋಡ್ ಆಗುವವರೆಗೆ ಕಾಯಿರಿ
1. Codespaces ನಲ್ಲಿ VS Code ಟೆರ್ಮಿನಲ್ ತೆರೆಯಿರಿ - ಕೆಳಗಿನ ಕಮಾಂಡ್ ಟೈಪ್ ಮಾಡಿ:

   ```bash title="" linenums="0"
   azd up
   ```

ಈದು 트ಿಗರ್ ಆಗುವ ವರ್ಕ್‌ಫ್ಲೋ ಹಂತಗಳನ್ನು ಪೂರ್ಣಗೊಳಿಸಿ:

1. Azure ಗೆ ಲಾಗಿನ್ ಮಾಡಲು ಪ್ರಾಂಪ್ಟ್ ಬರುತ್ತದೆ - ಪ್ರಮಾಣೀಕರಣದ ಸೂಚನೆಗಳನ್ನು ಅನುಸರಿಸಿ
1. ನಿಮ್ಮಿಗೆ ಅನನ್ಯವಾದ ಎನ್ವಿರಾನ್‌ಮೆಂಟ್ ಹೆಸರು ನಮೂದಿಸಿ - ಉದಾಹರಣೆಗೆ, ನಾನು `nitya-mshack-azd` ಅನ್ನು ಬಳಸಿದೆ
1. ಇದು `.azure/` ಫೋಲ್ಡರ್ ಅನ್ನು ಸೃಷ್ಟಿಸುತ್ತದೆ - ನೀವು env ಹೆಸರಿನ ಉಪ ಫೋಲ್ಡರ್ ನೋಡುತ್ತೀರಿ
1. ಸಬ್‍ಸ್ಕ್ರಿಪ್ಷನ್ ಹೆಸರನ್ನು ಆಯ್ಕೆಮಾಡಲು ಪ್ರಾಂಪ್ಟ್ ಬರಲಿದೆ - ಡೀಫಾಲ್ಟ್ ಆಯ್ಕೆಯನ್ನು ಆರಿಸಿ
1. ಸ್ಥಳ (location) ಕೇಳಾಟಕ್ಕೆ ಪ್ರಾಂಪ್ಟ್ ಬರುತ್ತದೆ - `East US 2` ಬಳಸಿರಿ

ಈಗ, Provisioning ಪೂರ್ಣಗೊಳಿಸುವವರೆಗೆ ಕಾಯಿರಿ. **ಇದು 10-15 ನಿಮಿಷ ಸಮಯ ಸಾಧ್ಯವಾಗುತ್ತದೆ**

1. ಮುಗಿದ ಮೇಲೆ, ನಿಮ್ಮ ಕಾನ್ಸೋಲ್‌ನಲ್ಲಿ SUCCESS ಸಂದೇಶ ಹೀಗೆ ಕಾಣುತ್ತದೆ:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. ನಿಮ್ಮ Azure Portal ನಲ್ಲಿ ಈಗ ಆ env ಹೆಸರಿನಿಂದ ಒಂದು provisioned resource group ಕಾಣಿಸುತ್ತದೆ:

      ![ಇನ್ಫ್ರಾ](../../../../../translated_images/kn/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **ನೀವು ಈಗ ನಿಯೋಜಿಸಲಾದ ಮೂಲಸೌಕರ್ಯ ಮತ್ತು ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಮಾನ್ಯಗೊಳಿಸಲು ಸಿದ್ಧರಾಗಿದ್ದೀರಿ**.

## 4. ಟೆಂಪ್ಲೇಟು ಮಾನ್ಯಗೊಳಿಸುವಿಕೆ

1. Azure Portal [Resource Groups](https://portal.azure.com/#browse/resourcegroups) ಪುಟಕ್ಕೆ ಭೇಟಿ ನೀಡಿ - ಪ್ರಾಂಪ್ಟ್ ಬಂದರೆ ಲಾಗಿನ್ ಮಾಡಿ

      1. ಕೇಳಿ: ```ಬಂಡಲ್ $200 ಕೆಳಗಿನ ಎರಡು ಜನರಿಗೆ ಸೂಕ್ತವಾದ ಅತ್ಯುತ್ತಮ ತೊಟ್ಟು ಹೇಗಿದೆ ಮತ್ತು ಅದರ ವೈಶಿಷ್ಟ್ಯಗಳು ಏನು?```

1. ನಿಮಗೆ ಕೆಳಗಿನಂತೆ ಸಮಾನ ಉತ್ತರಗಳು ಬರಬೇಕು. _ಇದಾಗೆ ಇದು ಹೇಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ?_

## 5. ಏಜೆಂಟ್ ಮಾನ್ಯಗೊಳಿಸುವಿಕೆ

Azure Container App ಒಂದು API końಡು ಒದಗಿಸುತ್ತದೆ ಅದು ಈ ಟೆಂಪ್ಲೇಟ್‌ನಿಗಾಗಿ Microsoft Foundry ಯೋಜನೆಯಲ್ಲಿ provisioning ಆಗಿರುವ AI Agent ಗೆ ಸಂಪರ್ಕ ಮಾಡುತ್ತದೆ. ಇದರ ಅರ್ಥ ಏನೆಂದು ನೋಡೋಣ.

1. ನಿಮ್ಮ resource group ನ Azure Portal _Overview_ ಪುಟಕ್ಕೆ ಹಿಂತಿರುಗಿ

1. ಅವೆಲ್ಲಾದ್ ಪಟ್ಟಿಯಲ್ಲಿ `Microsoft Foundry` ಸಂಪನ್ಮೂಲವನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ

1. ನಿಮಗೆ ಇದು ಕಾಣಿಸಬೇಕು. `Go to Microsoft Foundry Portal` ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಿ. 
   ![Foundry](../../../../../translated_images/kn/04-view-foundry-project.fb94ca41803f28f3.webp)

1. ನಿಮ್ಮ AI ಅಪ್ಲಿಕೇಶನ್‌ಗೆ ಸಂಬಂಧಿಸಿದ Foundry Project ಪುಟ ನೀವು ನೋಡುತ್ತೀರಿ
   ![ಪ್ರಾಜೆಕ್ಟ್](../../../../../translated_images/kn/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ - ನಿಮ್ಮ ಪ್ರಾಜೆಕ್ಟ್‌ನಲ್ಲಿ provision ಆಗಿರುವ ಡೀಫಾಲ್ಟ್ Agent ಕಾಣುತ್ತೆ
   ![ಏಜೆಂಟ್ಸ್](../../../../../translated_images/kn/06-visit-agents.bccb263f77b00a09.webp)

1. ಅದನ್ನು ಆರಿಸಿ - ಹಾಗಾದರೆ Agent ವಿವರಗಳನ್ನೂ ನೋಡಬಹುದು. ಕೆಳಗಿನ ಸಂಗತಿಗಳು ಗಮನಿಸಬೇಕು:

      - ಆಏಜೆಂಟ್ ಡಿಫಾಲ್ಟ್ ಆಗ File Search ಅನ್ನು ಬಳಸುತ್ತದೆ (ಎಂದೆಂದಿಗೂ)
      - Agent ನ `Knowledge` ನಲ್ಲಿ 32 ಫೈಲ್‌ಗಳು ಅಪ್‌ಲೋಡ್ ಆಗಿವೆ ಎಂಬುದನ್ನು ಸೂಚಿಸುತ್ತದೆ (file search ಗಾಗಿ)
      ![ಏಜೆಂಟ್ಸ್](../../../../../translated_images/kn/07-view-agent-details.0e049f37f61eae62.webp)

1. ಎಡ ಮೆನುಯಲ್ಲಿ `Data+indexes` ಆಯ್ಕೆಯನ್ನು ಹುಡುಕಿ ಮತ್ತು ವಿವರಗಳಿಗೆ ಕ್ಲಿಕ್ ಮಾಡಿ. 

      - ಜ್ಞಾನಕ್ಕಾಗಿ ಅಪ್ಲೋಡ್ ಮಾಡಿದ 32 ಡೇಟಾ ಫೈಲ್‌ಗಳನ್ನು ನೀವು ನೋಡಬೇಕು.
      - ಇವು `src/files` ಅಡಿಯಲ್ಲಿ ಇರುವ 12 customer ಫೈಲ್‌ಗಳು ಮತ್ತು 20 product ಫೈಲ್‌ಗಳಿಗೆ ಹೊಂದಿಕೆಯಾಗುತ್ತವೆ
      ![ಡೇಟಾ](../../../../../translated_images/kn/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**ನೀವು ಏಜೆಂಟ್ ಕಾರ್ಯಾಚರಣೆಯನ್ನು ಮಾನ್ಯಗೊಳಿಸಿದ್ದೀರಿ!**

1. ಏಜೆಂಟ್ ಉತ್ತರಗಳು ಆ ಫೈಲ್‌ಗಳಲ್ಲಿರುವ ಜ್ಞಾನದಲ್ಲಿ ನೆಲೆಗೊಂಡಿವೆ.
1. ಈಗ ನೀವು ಆ ಡೇಟಾ ಸಂಬಂಧಿಸಿದ ಪ್ರಶ್ನೆಗಳನ್ನು ಕೇಳಬಹುದು ಮತ್ತು ನೆಲೆಗಾಳಿಸಿದ (grounded) ಉತ್ತರಗಳನ್ನು ಪಡೆಯಬಹುದು.
1. ಉದಾಹರಣೆ: `customer_info_10.json` ನಲ್ಲಿ "Amanda Perez" ಮಾಡಿದ 3 ಖರೀದಿಗಳನ್ನು ವಿವರಿಸಲಾಗಿದೆ

## 6. ಏಜೆಂಟ್ ಪ್ಲೇಗ್ರೌಂಡ್

Microsoft Foundry ನ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಸ್ವಲ್ಪ ಹೆಚ್ಚು ತಿಳಿಯಲು, Agents Playground ನಲ್ಲಿ ಏಜೆಂಟ್ ಅನ್ನು ಪ್ರಯೋಗಿಸೋಣ.

1. Microsoft Foundry ನಲ್ಲಿ `Agents` ಪುಟಕ್ಕೆ ಹಿಂತಿರುಗಿ - ಡೀಫಾಲ್ಟ್ ಏಜೆಂಟ್ ಆಯ್ಕೆಮಾಡಿ
1. `Try in Playground` ಆಯ್ಕೆಮಾಡಿ - ನಿಮಗೆ ಈ ರೀತಿಯ Playground UI ಕಾಣಿಸಬೇಕು
1. ಅದೇ ಪ್ರಶ್ನೆಯನ್ನು ಕೇಳಿ: `Amanda Perez ಯಾವ ಉತ್ಪನ್ನಗಳನ್ನು ಹೊಂದಿದ್ದಾರೆ?`

    ![ಡೇಟಾ](../../../../../translated_images/kn/09-ask-in-playground.a1b93794f78fa676.webp)

1. ಯಾವುದೇ ಈ ಫೈಲ್ ಲೇಬಲ್ ಮೇಲಕ್ಕೆ ಹೇವರ್ (hover) ಮಾಡಿದರೆ - ಡೇಟಾ ನಿಮ್ಮ ಪ್ರಶ್ನೆ ಮತ್ತು ಪ್ರದರ್ಶಿತ ಉತ್ತರಕ್ಕೆ ಹೊಂದಿದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ

ನೀವು ಉತ್ತರದ ಕೆಳಗೆ ಒಂದು _stats_ ಸಾಲನ್ನು ಕೂಡಾ ನೋಡುತ್ತೀರಿ.

1. ಯಾವುದೇ ಮೆಟ್ರಿಕ್ ಮೇಲೆ ಹೇವರ್ ಮಾಡಿ - ಉದಾ., Safety. ನಿಮಗೆ ಹೀಗೆ ಏನೋ ಕಾಣಿಸಬಹುದು

## 7. ನಿರ್ಮಿತ ವೀಕ್ಷಣೀಯತೆ (Built-in Observability)

Observability ಅಂದರೆ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಇಂಸ್ಟ್ರುಮೆಂಟ್ ಮಾಡಿ ನಿರ್ಲಕ್ಷ್ಯ ಮಾಡುವಂತಹ ಡೇಟಾವನ್ನು ಉತ್ಪಾದಿಸುವಂತೆ ಮಾಡಲು ಆಗಿದೆ, ಇದನ್ನು ಬಳಸಿ ಇದರ ಕಾರ್ಯಾಚರಣೆಯನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು, ಡಿಬಗ್ ಮಾಡಲು ಮತ್ತು ಆಪ್ಟಿಮೈಜ್ ಮಾಡಲು ಬಳಸಬಹುದು. ಇದಕ್ಕೆ ಒಮ್ಮೆ ನೋಡಿ:

1. `View Run Info` ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಿ - ನಿಮಗೆ ಈ ದೃಶ್ಯ ಕಾಣಿಸಬೇಕು. ಇದು [Agent tracing](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) ನ ಉದಾಹರಣೆಯಾಗಿದೆ. _ಈ ವೀಕ್ಷಣೆಯನ್ನು ನೀವು ಟೆಾಪ್-ಲೆವೆಲ್ ಮೆನುವಿನ Thread Logs ಕ್ಲಿಕ್ ಮಾಡುವ ಮೂಲಕವೂ ಪಡೆಯಬಹುದು_.

   - ಏಜೆಂಟ್ ಕೈಗೊಂಡ ರನ್ ಹಂತಗಳು ಮತ್ತು ಎಂಜೇಜ್ ಮಾಡಿದ ಟೂಲ್ಗಳನ್ನು ತಿಳಿದುಕೊಳ್ಳಿ
   - ಪ್ರತಿಕ್ರಿಯೆಗಾಗಿ ಒಟ್ಟು Token ಎಣಿಕೆ (vs. output tokens ಬಳಕೆ) ಅನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಿ
   - ವಿಳಂಬ (latency) ಮತ್ತು ಎಕ್ಸಿಕ್ಯೂಶನ್‌ನಲ್ಲಿ ವೇಳೆ ಎಲ್ಲಿ ಕಳೆದಿದೆ ಎಂಬುದನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಿ

      ![ಏಜೆಂಟ್](../../../../../translated_images/kn/10-view-run-info.b20ebd75fef6a1cc.webp)

1. ರನ್‌ಗೆ ಸಂಬಂಧಿಸಿದ ಹೆಚ್ಚುವರಿ ಗುಣಲಕ್ಷಣಗಳನ್ನು ನೋಡಲು `Metadata` ಟ್ಯಾಬ್ ಕ್ಲಿಕ್ ಮಾಡಿ, ಇದು ನಂತರದಲ್ಲಿ ಸಮಸ್ಯೆ ನಿಭಾಯಿಸಲು ಸಹಾಯಕವಾದ ಸಂಧರ್ಭವನ್ನು ಒದಗಿಸಬಹುದು.   

      ![ಏಜೆಂಟ್](../../../../../translated_images/kn/11-view-run-info-metadata.7966986122c7c2df.webp)


1. `Evaluations` ಟ್ಯಾಬ್ ಕ್ಲಿಕ್ ಮಾಡಿ - ಏಜೆಂಟ್ ಉತ್ತರದ ಮೇಲೆ ಮಾಡಿದ ಸ್ವಯಂ-ಅಂಕನಗಳನ್ನು (auto-assessments) ಇಲ್ಲಿ ನೋಡಬಹುದು. ಇದರಲ್ಲಿ ಸುರಕ್ಷತಾ ಅಂಕನಗಳು (ಉದಾ., Self-harm) ಮತ್ತು ಏಜೆಂಟ್-ವಿಶೇಷ ಅಂಕನಗಳು (ಉದಾ., ಉದ್ದೇಶ ಪರಿಹರಿಸುವಿಕೆ, ಕಾರ್ಯನಿಷ್ಠೆ) ಸೇರಿವೆ.

      ![ಏಜೆಂಟ್](../../../../../translated_images/kn/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. ಕೊನೆಗೆ, ಸೈಡ್‍ಬಾರ್ ಮೆನುವಿನಲ್ಲಿ `Monitoring` ಟ್ಯಾಬ್ ಕ್ಲಿಕ್ ಮಾಡಿ.

      - ಪ್ರದರ್ಶಿತ ಪುಟದಲ್ಲಿ `Resource usage` ಟ್ಯಾಬ್ ಆಯ್ಕೆಮಾಡಿ - ಮತ್ತು ಮೆಟ್ರಿಕ್‌ಗಳನ್ನು ವೀಕ್ಷಿಸಿ.
      - ವೆಚ್ಚ (tokens) ಮತ್ತು ಲೋಡ್ (requests) ದೃಷ್ಟಿಯಿಂದ ಅಪ್ಲಿಕೇಶನ್ ಬಳಕೆಯನ್ನು ಟ್ರಾಕ್ ಮಾಡಿ.
      - ಮೊದಲ ಬೈಟ್ (input processing) ಗೆ ಮೊದಲ ನಾಟಕಮಾನದ ವಿಳಂಬ ಮತ್ತು ಕೊನೆಯ ಬೈಟ್ (output) ಗೆ ವಿಳಂಬವನ್ನು ಟ್ರ್ಯಾಕ್ ಮಾಡಿ.

      ![ಏಜೆಂಟ್](../../../../../translated_images/kn/13-monitoring-resources.5148015f7311807f.webp)

## 8. ಪರಿಸರ ಚರ (Environment Variables)

ಈವರೆಗೆ ನಾವು ಬ್ರೌಸರ್‌ನಲ್ಲಿ ನಿಯೋಜನೆಯನ್ನು ಮಾಡಿ — ಮತ್ತು ನಮ್ಮ ಮೂಲಸೌಕರ್ಯ ಒದಗಿಸಲಾಗಿದೆಯೇ ಮತ್ತು ಅಪ್ಲಿಕೇಶನ್ ಕಾರ್ಯಾಚರಿಸುತ್ತಿದೆಯೇ ಎಂದು ಮಾನ್ಯಗೊಳಿಸಿದ್ದೇವೆ. ಆದರೆ ಕೋಡ್-ಫರ್ಸ್ಟ್ ರೀತಿಯಲ್ಲಿ ಅಪ್ಲಿಕೇಶನ್‌ ಜೊತೆಗೆ ಕೆಲಸ ಮಾಡಲು, ನಾವಿನ ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿ ಪರಿಸರವನ್ನು ಈ ಸಂಪನ್ಮೂಲಗಳೊಂದಿಗೆ ಕೆಲಸ ಮಾಡುವುದಕ್ಕೆ ಅಗತ್ಯವಿರುವ ಸಂಬಂಧಿತ ಚರಗಳು (variables) ಜೊತೆಗೆ ಕಾನ್ಫಿಗರ್ ಮಾಡಬೇಕಾಗುತ್ತದೆ. `azd` ಬಳಸಿ ಇದು ಸುಲಭ.

1. Azure Developer CLI [environment variables ಅನ್ನು ಬಳಸುತ್ತದೆ](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) ಅಪ್ಲಿಕೇಶನ್ ನಿಯೋಜನೆಗಳಿಗಾಗಿ ಸಂರಚನಾ ಸೆಟ್ಟಿಂಗ್ಸ್ ಸಂಗ್ರಹಿಸಲು ಮತ್ತು ನಿರ್ವಹಿಸಲು.

1. ಪರಿಸರ ಚರಗಳು `.azure/<env-name>/.env` ನಲ್ಲಿ ಸಂಗ್ರಹಿಸಲಾಗುತ್ತವೆ - ಇದು ಅವುಗಳನ್ನು ನಿಯೋಜನೆಯ ಸಮಯದಲ್ಲಿ ಬಳಿಸಿದ `env-name` ಗೆ ಸೀಮಿತ ಮಾಡುತ್ತದೆ ಮತ್ತು ಇದೇ ರೆಪೊಸಿಟರಿಯಲ್ಲಿ ಬಿನ್ನವಡೆಗಳ ನಡುವೆ ಪರಿಸರಗಳನ್ನು ಬೇರ್ಪಡಿಸಲು ಸಹಾಯಮಾಡುತ್ತದೆ.

1. ಪರಿಸರ ಚರಗಳನ್ನು `azd` ಕಮಾಂಡ್ ಅದೊಂದು ನಿರ್ದಿಷ್ಟ ಕಮಾಂಡ್‌ ಅನ್ನು ಜರುಗಿಸುವಾಗ (ಉದಾ., `azd up`) ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಲೋಡ್ ಮಾಡುತ್ತದೆ. ಗಮನಿಸಿ `azd` OS-ಹಂತದ ಪರಿಸರ ಚರಗಳನ್ನು (ಉದಾ., ಶೆಲ್‌ನಲ್ಲಿ ಸೆಟ್ ಮಾಡಿದವು) ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಓದುತ್ತಿಲ್ಲ - ಬದಲಾಗಿ `azd set env` ಮತ್ತು `azd get env` ಬಳಸಿ ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳಲ್ಲಿ ಮಾಹಿತಿಯನ್ನುfford ಆಚರಣೆಗೆ ಕಳುಹಿಸಿ.

ಸಲಹೆ: ಕೆಲವು ಯಾವ್ ಕಮಾಂಡ್‌ ಉದಾಹರಣೆಗಳನ್ನು ಪ್ರಯತ್ನಿಸೋಣ:

1. ಈ ಎನ್ವಿರಾನ್‌ಮೆಂಟ್‌ನಲ್ಲಿ `azd` ಗೆ ಸೆಟ್ ಮಾಡಿದ ಎಲ್ಲಾ ಪರಿಸರ ಚರಗಳನ್ನು ಪಡೆದುಕೊಳ್ಳಿ:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      ನಿಮಗೆ ಹೀಗೇ ಏನೋ ಕಾಣಿಸತ್ತದೆ:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. ವಿಶೇಷ ಮೌಲ್ಯಗಳನ್ನು ಪಡೆಯಿರಿ - ಉದಾಹರಣೆಗೆ, `AZURE_AI_AGENT_MODEL_NAME` ಮೌಲ್ಯವನ್ನು ಸೆಟ್ ಮಾಡಿದೆಯೇ ಎಂದು ತಿಳಿಯಲು

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      ನಿಮಗೆ ಹೀಗೇ ಏನೋ ಕಾಣಿಸತ್ತದೆ - ಇದು ಡೀಫಾಲ್ಟ್ ಆಗಿ ಸೆಟ್ ಆಗಲಿಲ್ಲ!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd` jaoks ಹೊಸ ಪರಿಸರ ಚರವನ್ನು ಸೆಟ್ ಮಾಡಿ. ಇಲ್ಲಿ, ನಾವು ಏಜೆಂಟ್ ಮಾದರಿಯ ಹೆಸರನ್ನು ಅಪ್ಡೇಟ್ ಮಾಡುತ್ತೇವೆ. _ಗಮನಿಸಿ: ಯಾವುದೇ ಬದಲಾಗಿಸಿದ ವಿಷಯಗಳು ತಕ್ಷಣ `.azure/<env-name>/.env` ಫೈಲ್‌ನಲ್ಲಿ ಪ್ರತಿಬಿಂಬಗೊಳ್ಳುತ್ತವೆ._

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      ಈಗ, ನಾವು ಆ ಮೌಲ್ಯ ಸೆಟ್ ಆಗಿದೆ ಎಂದು ಕಂಡುಹಿಡಿಯಬೇಕು:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. ಕೆಲವು ಸಂಪನ್ಮೂಲಗಳು ಷೇಕಾದಿರುತ್ತವೆ (ಉದಾ., ಮಾದರಿ ನಿಯೋಜನೆಗಳು) ಮತ್ತು ಅವುಗಳನ್ನು ಪುನರ್‍ನಿಯೋಜಿಸಲು ಕೆವಲ `azd up` ಹೆಚ್ಚು ಸಾಕಾಗದಿರಬಹುದು. ಮೂಲ ನಿಯೋಜನೆಯನ್ನು ಕೆಳಗೆ ಸುಳಿದು ಪುನರ್‍ನಿಯೋಜನೆ ಮಾಡಲು ಪ್ರಾರಂಭಿಸೋಣ.

1. **Refresh** ನೀವು ಈಗಾಗಲೇ azd ಟೆಂಪ್ಲೇಟು ಬಳಸಿ ಮೂಲಸೌಕರ್ಯ ನಿಯೋಜನೆ ಮಾಡಿದ್ದರೆ - ಈ ಕಮಾಂಡ್ ಬಳಸಿ ನಿಮ್ಮ ಸ್ಥಳೀಯ ಪರಿಸರ ಚರಗಳ ಸ್ಥಿತಿಯನ್ನು ನಿಮ್ಮ Azure ನಿಯೋಜನೆಯ ಪ್ರಸ್ತುತ ಸ್ಥಿತಿಗೆ ಆಧರಿಸಿ _refresh_ ಮಾಡಬಹುದು:

      ```bash title="" linenums="0"
      azd env refresh
      ```

      ಇದು ಎರಡು ಅಥವಾ ಹೆಚ್ಚು ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿ ವಾತಾವರಣಗಳಲ್ಲಿ ಪರಿಸರ ಚರಗಳನ್ನು _ಸಮಕಾಲೀಕರಿಸಲು_ ಒಂದು ಶಕ್ತಿ ಶಾಲಿ ವಿಧಾನವಾಗಿದೆ (ಉದಾ., ಹಲವಾರು ಡೆವಲಪರ್‌ಗಳಿರುವ ತಂಡ) - ನಿಯುಕ್ತಗೊಳಿಸಲಾದ ಮೂಲಸೌಕರ್ಯವನ್ನು ಪರಿಸರ ಚರಗಳ ಸ್ಥಿತಿಗೆ ನಿಜವಾದ ಮೂಲವಾಗಿ ಸೇವೆ ನೀಡಲು ಅನುಮತಿಸುತ್ತದೆ. ತಂಡದ ಸದಸ್ಯರು ಕೇವಲ ಚರಗಳನ್ನು _ನವೀಕರಿಸಿ_ ಮರುಸಮತೋಲನಕ್ಕೆ ಬರುತ್ತಾರೆ.
