    - [ ] AZD ಬಳಕೆಯ ಕುರಿತು ಸಹಾಯ ಪಡೆಯಲು GitHub Copilot ಅನ್ನು ಬಳಸಿ

## 1. ಪರಿಚಯ

The [Azure ಡೆವೆಲಪರ್ CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) or `azd` ಒಂದು open-source ಕಮಾಂಡ್‌ಲೈನ್ ಸಾಧನವಾಗಿದೆ, ಅದು Azure ಗೆ ಅಪ್ಲಿಕೇಶನ್ ಗಳನ್ನು ನಿರ್ಮಿಸುವಾಗ ಮತ್ತು ನಿಯೋಜಿಸುವಾಗ ಡೆವೆಲಪರ್ ಕಾರ್ಯಪ್ರವಾಹವನ್ನು ಸರಳಗೊಳಿಸುತ್ತದೆ। 

[AZD ಟೆಂಪ್ಲೇಟುಗಳು](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) ಎಂದರೆ ಮಾದರಿಯಾದ ಅಪ್ಲಿಕೇಶನ್ ಕೋಡ್, _infrastructure-as-code_ ಆಸ್ತಿ ಮತ್ತು `azd` ಸಂರಚನಾ ಕಡತಗಳನ್ನು ಒಳಗೊಂಡಿರುವ ಮಾನದಂಡಿತ ರೆಪೊಸಿಟರಿಗಳಾಗಿವೆ, ಅವು ಸುತ್ತಲೂ ಸ್ಥಾಯಿಶೀಲ ಪರಿಹಾರ ವಾಸ್ತುಶಿಲ್ಪವನ್ನು ಒದಗಿಸುತ್ತವೆ. ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಅನ್ನು ಪ್ರೊವೀಶನ್ ಮಾಡುವದು `azd provision` ಕಮಾಂಡ್ ಗಿಂತ ಸುಲಭವಾಗುತ್ತದೆ - ಮತ್ತು `azd up` ಬಳಸಿ ನೀವು ಒಂದು ಸಲದಲ್ಲಿ ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಅನ್ನು ಪ್ರೊವೀಶನ್ ಮಾಡಿ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ನಿಯೋಜಿಸಬಹುದು!

ಫಲವಾಗಿ, ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅಭಿವೃದ್ಧಿ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಆರಂಭಿಸಲು ನೀವು ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಮತ್ತು ಇನ್‌ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಅಗತ್ಯಗಳಿಗೆ ಅತ್ಯಂತ ಸಮೀಪ ಇರುವ _AZD Starter template_ ಅನ್ನು ಹುಡುಕಿ - ನಂತರ ರೆಪೊಸಿಟರಿಯನ್ನು ನಿಮ್ಮ سینಾರಿಯೋ ಅಗತ್ಯಗಳಂತೆ ಕಸ್ಟಮೈಸ್ ಮಾಡಬಹುದು.

ಆರಂಭಿಸುವ ಮೊದಲು, ನಿಮ್ಮ ಬಳಿ Azure Developer CLI ಸ್ಥಾಪಿತವಾಗಿರುವುದನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳೋಣ.

1. VS Code ಟರ್ಮಿನಲ್ ತೆರೆಯಿರಿ ಮತ್ತು ಈ ಕಮಾಂಡ್ ಅನ್ನು ಟೈಪ್ ಮಾಡಿ:

      ```bash title="" linenums="0"
      azd version
      ```

1. ನಿಮಗೆ ಇಂತಹದು ಕಾಣಿಸಬೇಕು!

      ```bash title="" linenums="0"
      azd version 1.19.0 (commit b3d68cea969b2bfbaa7b7fa289424428edb93e97)
      ```

**ನೀವು ಈಗ azd ಬಳಸಿ ಟೆಂಪ್ಲೇಟನ್ನು ಆಯ್ಕೆ ಮಾಡಿ ನಿಯೋಜಿಸಲು ಸಿದ್ಧರಾಗಿದ್ದೀರಿ**

## 2. ಟೆಂಪ್ಲೇಟು ಆಯ್ಕೆ

Microsoft Foundry ಪ್ಲ್ಯಾಟ್‌ಫಾರ್ಮ್ [ಶಿಫಾರಸುಮಾಡಲಾದ AZD ಟೆಂಪ್ಲೇಟ್‌ಗಳ ಒಂದು ಸೆಟ್](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) తో ಬರುತ್ತದೆ, ಅವು ಜನಪ್ರಿಯ ಪರಿಹಾರ ದೃಶ್ಯಾವಳಿಗಳಾದ _ಬಹು-ಏಜೆಂಟ್ ಕಾರ್ಯಪ್ರವಾಹ ಸ್ವಯಂಚಾಲನೆ_ ಮತ್ತು _ಬಹು-ಮೋಡಲ್ ವಿಷಯ ಸಂಸ್ಕರಣೆ_ ಮುಂತಾದವುಗಳನ್ನು ಆವರಿಸುತ್ತದೆ. ನೀವು ಈ ಟೆಂಪ್ಲೇಟುಗಳನ್ನು Microsoft Foundry ಪೋರ್ಟಲ್ ಅನ್ನು ಭೇಟಿಮಾಡಿ ಕೂಡ ಕಂಡುಹಿಡಿಯಬಹುದು.

1. ಭೇಟಿ ನೀಡಿ [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. ಪ್ರಾಂಪ್ಟ್ ಆಗಿದಾಗ Microsoft Foundry ಪೋರ್ಟಲ್‌ಗೆ ಲಾಗಿನ್ ಮಾಡಿ - ನಿಮಗೆ ಹೀಗೆ ಏನಾದರೂ ಕಾಣಿಸಬೇಕು.

![ಆಯ್ಕೆ](../../../../../translated_images/kn/01-pick-template.60d2d5fff5ebc374.webp)


The **Basic** ಆಯ್ಕೆಗಳು ನಿಮ್ಮ ಆರಂಭಿಕ ಟೆಂಪ್ಲೇಟುಗಳು:

1. [ ] [Get Started with AI Chat](https://github.com/Azure-Samples/get-started-with-ai-chat) ಇದು ಒಂದು ಮೂಲಭೂತ ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು _ನಿಮ್ಮ ಡೇಟಾ ಜೊತೆಗೆ_ Azure Container Apps ಗೆ ನಿಯೋಜಿಸುವದು. ಇದನ್ನು ಬಳಸಿಕೊಂಡು ಮೂಲ AI ಚಾಟ್‌ಬಾಟ್ ದೃಶ್ಯಾವಳಿಯನ್ನು ಅನ್ವೇಷಿಸಿ.
1. [X] [Get Started with AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) ಇದು ಕೂಡ Foundry Agents ಸಹಿತ ಮಾನದಂಡವಾದ AI ಏಜೆಂಟ್ ಅನ್ನು ನಿಯೋಜಿಸುತ್ತದೆ. ابزارಗಳು ಮತ್ತು ಮಾದರಿಗಳನ್ನು ಒಳಗೊಂಡಿರುವ ಏಜೆಂಟ್ ಆಧಾರಿತ AI ಪರಿಹಾರಗಳೊಂದಿಗೆ ಪರಿಚಿತರಾಗಲು ಇದನ್ನು ಬಳಸಿ.

ೆಈ ಎರಡನೆಯ ಲಿಂಕನ್ನು ಹೊಸ ಬ್ರೌಸರ್ ಟ್ಯಾಬ್‌ನಲ್ಲಿ ತೆರೆಯಿರಿ (ಅಥವಾ ಸಂಬಂಧಿತ ಕಾರ್ಡ್‌ಗೆ `Open in GitHub` ಕ್ಲಿಕ್ ಮಾಡಿ). ನೀವು ಈ AZD ಟೆಂಪ್ಲೇಟಿನ ರೆಪೊಸಿಟರಿ ನೋಡಬಹುದು. README ಅನ್ನು ಒಂದು ನಿಮಿಷವಾಗಿ ಪರಿಶೀಲಿಸಿ. ಅಪ್ಲಿಕೇಶನ್ ವಾಸ್ತುಶಿಲ್ಪ ಹೀಗೆ ಕಾಣುತ್ತದೆ:

![ವಾಸ್ತುಶಿಲ್ಪ](../../../../../translated_images/kn/architecture.8cec470ec15c65c7.webp)

## 3. ಟೆಂಪ್ಲೇಟು ಸಕ್ರಿಯಗೊಳಿಸು

ಈ ಟೆಂಪ್ಲೇಟನ್ನು ನಿಯೋಜಿಸಲು ಪ್ರಯತ್ನಿಸಿ ಮತ್ತು ಅದು ಮಾನ್ಯವಾಗಿದೆಯೇಂದು ಖಚಿತಪಡಿಸೋಣ. ನಾವು [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) ವಿಭಾಗದ ಮಾರ್ಗಸೂಚಿಗಳನ್ನು ಅನುಸರಿಸುತ್ತೇವೆ.

1. ಈ [ಲಿಂಕ್](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ - ಡಿಫಾಲ್ಟ್ ಕ್ರಿಯೆಯನ್ನು `Create codespace` ಎಂದು ದೃಢೀಕರಿಸಿ
1. ಇದು ಹೊಸ ಬ್ರೌಸರ್ ಟ್ಯಾಬ್ ತೆರೆಯುತ್ತದೆ - GitHub Codespaces ಸೆಷನ್ ಲೋಡ್ ಮುಗಿಯುವವರೆಗೆ ಕಾಯಿರಿ
1. Codespaces ನಲ್ಲಿ VS Code ಟರ್ಮಿನಲ್ ತೆರೆಯಿರಿ - ಕೆಳಗಿನ ಕಮಾಂಡ್ ಅನ್ನು ಟೈಪ್ ಮಾಡಿ:

   ```bash title="" linenums="0"
   azd up
   ```

ಈದು ಟ್ರಿಗರ್ ಮಾಡುವ ಕಾರ್ಯಪ್ರವಾಹದ ಹಂತಗಳನ್ನು ಪೂರ್ಣಗೊಳಿಸಿ:

1. ನಿಮಗೆ Azure ಲಾಗಿನ್ ಮಾಡಲು ಪ್ರಾಂಪ್ಟ್ ತೋರಿಸಲಾಗುತ್ತದೆ - ಪ್ರಾಮಾಣೀಕರಣದ ಸೂಚನೆಗಳನ್ನು ಅನುಸರಿಸಿ
1. ನಿಮ್ಮಿಗಾಗಿ ಒಂದು ಅನನ್ಯ ಎನ್ವಿರನ್‌ಮೆಂಟ್ ಹೆಸರು ಎಂಟರ್ ಮಾಡಿ - ಉದಾಹರಣೆಗೆ, ನಾನು `nitya-mshack-azd` ಅನ್ನು ಬಳೈಸಿದೆ
1. ಇದರಿಂದ `.azure/` ಫೋಲ್ಡರ್ ಸೃಷ್ಟಿಯಾಗುತ್ತದೆ - ನೀವು ಆ ಎನ್ವ್ ಹೆಸರು ಇರುವ ಉಪಫೋಲ್ಡರ್ ನೋಡುತ್ತೀರಿ
1. ನಿಮಗೆ ಸಬ್‌ಸ್ಕ್ರಿಪ್ಶನ್ ಹೆಸರನ್ನು ಆಯ್ಕೆ ಮಾಡಲು ಕೇಳಲಾಗುತ್ತದೆ - ಡಿಫಾಲ್ಟ್ ಅನ್ನು ಆರಿಸಿ
1. ನಿಮಗೆ ಸ್ಥಳವನ್ನು ಕೇಳಲಾಗುತ್ತದೆ - `East US 2` ಅನ್ನು ಬಳಸಿ

ಈಗ, ಪ್ರೊವಿಷನಿಂಗ್ ಪೂರ್ಣಗೊಳ್ಳಲು ನೀವು ಕಾಯಬೇಕು. **ಇದಕ್ಕೆ 10-15 ನಿಮಿಷಗಳು ಬೇಕಾಗಬಹುದು**

1. ಮುಗಿದಾಗ, ನಿಮ್ಮ ಕನ್‌ಸೋಲ್ ಅನ್ನು ಕೆಳಗಿನಂತೆ SUCCESS ಸಂದೇಶ ತೋರಿಸುತ್ತದೆ:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. ನಿಮ್ಮ Azure ಪೋರ್ಟಲ್ ಈಗ ಆ ಎನ್ವ್ ಹೆಸರಿನೊಂದಿಗೆ ಪ್ರೊವಿಷನ್ಡ್ ರಿಸೋರ್ಸ್ ಗ್ರೂಪ್ ಹೊಂದಿರುತ್ತದೆ:

      ![ಇನ್ಫ್ರಾ](../../../../../translated_images/kn/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **ನೀವು ಈಗ ನಿಯೋಜಿಸಲಾದ ಇನ್‌ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಮತ್ತು ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಮಾನ್ಯಗೊಳಿಸಲು ಸಿದ್ಧರಾಗಿದ್ದೀರಿ**.

## 4. ಟೆಂಪ್ಲೇಟು ಮಾನ್ಯಗೊಳಣೆ

1. Azure Portal [Resource Groups](https://portal.azure.com/#browse/resourcegroups) ಪುಟಕ್ಕೆ ಭೇಟಿ ನೀಡಿ - ಪ್ರಾಂಪ್ಟ್ ಬಂದರೆ ಲಾಗಿನ್ ಮಾಡಿ
1. ನಿಮ್ಮ ಎನ್ವಿರಾನ್‌ಮೆಂಟ್ ಹೆಸರಿನ RG ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ - ನೀವು ಮೇಲಿನ ಪುಟವನ್ನು ನೋಡುತ್ತೀರಿ

      - Azure Container Apps ರಿಸೋರ್ಸ್ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ
      - _Essentials_ ವಿಭಾಗದಲ್ಲಿ (ಮೇಲ್ಭಾಗದ ಬಲ) Application Url ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ

1. ನಿಮಗೆ ಹೀಗೆ ಹೋಸ್ಟ್ ಮಾಡಲಾದ ಅಪ್ಲಿಕೇಶನ್ ಫ್ರಂಟ್-ಎಂಡ್ UI ಕಾಣಿಸಬೇಕು:

   ![ಅಪ್ಲಿಕೇಶನ್](../../../../../translated_images/kn/03-test-application.471910da12c3038e.webp)

1. ಕೆಲವು [sample questions](https://github.com/Azure-Samples/get-started-with-ai-agents/blob/main/docs/sample_questions.md) ಕೇಳಿ

      1. ಕೇಳಿ: ```ಫ್ರಾನ್ಸ್‌ನ ರಾಜಧಾನಿ ಯಾವುದು?``` 
      1. ಕೇಳಿ: ```ಎರಡು ಜನರಿಗೆ $200 ಕ್ಕಿಂತ ಕಡಿಮೆ ಬೆಲೆಯ ಅತ್ಯುತ್ತಮ ತೋಂಟು ಯಾವುದು, ಮತ್ತು ಅದು ಯಾವ ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಒಳಗೊಂಡಿದೆ?```

1. ನಿಮಗೆ ಕೆಳಗಿನಂತೆ ಸಮಾನ ಉತ್ತರಗಳು ಬರುತ್ತವೆ. _ಆದರೆ ಇದು ಹೇಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ?_ 

      ![ಅಪ್ಲಿಕೇಶನ್](../../../../../translated_images/kn/03-test-question.521c1e863cbaddb6.webp)

## 5. ಏಜೆಂಟ್ ಮಾನ್ಯಗೊಳಣೆ

Azure Container App ಒಂದು ಎಂಡ್ಪಾಯಿಂಟ್ ಅನ್ನು ನಿಯೋಜಿಸುತ್ತದೆ, ಅದು ಈ ಟೆಂಪ್ಲೇಟಿಗಾಗಿ Microsoft Foundry ಪ್ರಾಜೆಕ್ಟ್‌ನಲ್ಲಿ ಪ್ರೊವಿಜನ್ ಮಾಡಲಾದ AI ಏಜೆಂಟ್‌ಗೆ ಸಂಪರ್ಕ ಹೊಂದುತ್ತದೆ. ಇದನ್ನು ಏನೆಂದು ಅರ್ಥವೋ ನೋಡೋಣ.

1. ನಿಮ್ಮ ರಿಸೋರ್ಸ್ ಗ್ರೂಪ್‌ಗೆ ಸಂಬಂಧಿಸಿದ Azure Portal _Overview_ ಪುಟಕ್ಕೆ ಮರುಹೋಗಿ

1. ಆ ಪಟ್ಟಿಯಿಂದ `Microsoft Foundry` ರಿಸೋರ್ಸ್ನ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ

1. ನಿಮಗೆ ಹೀಗೆ ಕಾಣಿಸಬೇಕು. `Go to Microsoft Foundry Portal` ಬಟನ್ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ. 
   ![Foundry](../../../../../translated_images/kn/04-view-foundry-project.fb94ca41803f28f3.webp)

1. ನಿಮ್ಮ AI ಅಪ್ಲಿಕೇಶನ್‌ಗಾಗಿ Foundry Project ಪುಟವನ್ನು ನೀವು ನೋಡುತ್ತೀರಿ
   ![Project](../../../../../translated_images/kn/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ - ನಿಮ್ಮ ಪ್ರಾಜೆಕ್ಟ್‌ನಲ್ಲಿ ಡಿಫಾಲ್ಟ್ ಏಜೆಂಟ್ ಪ್ರೊವಿಜನ್ ಆಗಿರುವುದು ಕಾಣಿಸುತ್ತದೆ
   ![Agents](../../../../../translated_images/kn/06-visit-agents.bccb263f77b00a09.webp)

      ![Agents](../../../../../translated_images/kn/07-view-agent-details.0e049f37f61eae62.webp)

ನೀವು ಉತ್ತರದ ಕೆಳಭಾಗದಲ್ಲಿ ಒಂದು _stats_ ಸಾಲನ್ನು ಕೂಡ ನೋಡುತ್ತೀರಿ.

## 7. ಸೌಜನ್ಯಗೊಳಿಸಿದ ಅಬ್ಜರ್ವೆಬಿಲಿಟಿ

ಅಬ್ಜರ್ವೆಬಿಲಿಟಿ ಎಂದರೆ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಇನ್ಸ್ಟ್ರುಮೆಂಟ್ ಮಾಡಿ ಡೇಟಾ ಉತ್ಪಾದಿಸುವಂತೆ ಮಾಡುವುದಾಗಿದೆ, ಅದು ಅದರ ಕಾರ್ಯಾಚರಣೆಯನ್ನು ಅರ್ಥಮಾಡಲು, ಡಿಬಗ್ ಮಾಡಲು ಮತ್ತು tốiಮೈಸು ಮಾಡಲು ಉಪಯೋಗಿಸಬಹುದಾದದ್ದು. ಇದರ ಅರಿವು ಪಡೆಯಲು:

1. `View Run Info` ಬಟನ್ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ - ನಿಮಗೆ ಈ ವೀಕ್ಷಣೆಯು ಕಾಣಿಸಬೇಕು. ಇದು [Agent tracing](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) ನ ಉದಾಹರಣೆ ವಾಗಿದೆ. _ನೀವು ಈ ವೀಕ್ಷಣೆಯನ್ನು Thread Logs ಅನ್ನು ಟಾಪ್-ಲೆವಲ್ ಮೆನುದಲ್ಲಿ ಕ್ಲಿಕ್ ಮಾಡಿ ಕೂಡ ಪಡೆಯಬಹುದು_.

   - ಏಜೆಂಟ್ ಚಲಿಸುವ ಹಂತಗಳು ಮತ್ತು ಉಪಕರಣಗಳು ಯಾವುವು ನಿರ್ವಹಿಸಿದವು ಎಂಬುದನ್ನು ತಿಳಿದುಕೊಳ್ಳಿ
   - ಉತ್ತರಕ್ಕಾಗಿ ಒಟ್ಟು Token ಗಣನೆ (vs. output tokens ಬಳಕೆ) ಅರ್ಥಮಾಡಿಕೊಳ್ಳಿ
   - ಲೇಟನ್ಸಿಯನ್ನು ಮತ್ತು ನಿರ್ವಹಣೆ ಎಲ್ಲಿ ಸಮಯ ವ್ಯಯವಾಗುತ್ತಿದೆ ಎಂಬುದನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಿರಿ

      ![Agent](../../../../../translated_images/kn/10-view-run-info.b20ebd75fef6a1cc.webp)

1. `Metadata` ಟ್ಯಾಬ್ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ ಮತ್ತು ರನ್‌ಗೆ ಸಂಬಂಧಿಸಿದ ಹೆಚ್ಚುವರಿ ಗುಣಲಕ್ಷಣಗಳನ್ನು ನೋಡಿ, ಅವು ನಂತರ ಡೀಬಗ್ ಮಾಡುವಾಗ ಉಪಯುಕ್ತ ಹಿನ್ನೆಲೆಯನ್ನು ಒದಗಿಸಬಹುದಾದವು.   

      ![Agent](../../../../../translated_images/kn/11-view-run-info-metadata.7966986122c7c2df.webp)


1. `Evaluations` ಟ್ಯಾಬ್ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ ಏಜೆಂಟ್ ಪ್ರತಿಕ್ರಿಯೆಯ ಮೇಲೆ ಸ್ವಯಂ-ಅಂದಾಜುಗಳನ್ನು ನೋಡಿ. ಇದರಲ್ಲಿ ಸುರಕ್ಷತಾ ಅಂದಾಜುಗಳು (ಉದಾ., Self-harm) ಮತ್ತು ಏಜೆಂಟ್-ವಿಶೇಷ ಅಂದಾಜುಗಳು (ಉದಾ., ಉದ್ದೇಶ ನಿರ್ಧಾರ, ಕಾರ್ಯ ಅನುಸರಣ) ಸೇರಿವೆ.

      ![Agent](../../../../../translated_images/kn/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. ಕೊನೆಗೂ, ಸೈಡ್‌ಬಾರ್ ಮೆನುದಲ್ಲಿ `Monitoring` ಟ್ಯಾಬ್ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ.

      - ಪ್ರದರ್ಶಿತ ಪುಟದಲ್ಲಿ `Resource usage` ಟ್ಯಾಬ್ ಆಯ್ಕೆ ಮಾಡಿ - ಮತ್ತು ಮೆಟ್ರಿಕ್ಸ್‌ಗಳನ್ನು ನೋಡಿ.
      - ಟೋಕನ್ಗಳ ಆಧಾರದ ಮೇಲೆ ವೆಚ್ಚಗಳನ್ನು ಮತ್ತು ಲೋಡ್ (ಕೋರಿಕೆಗಳು) ರೂಪದಲ್ಲಿ ಅಪ್ಲಿಕೇಶನ್ ಬಳಕೆಯನ್ನು ಟ್ರ್ಯಾಕ್ ಮಾಡಿ.
      - ಫಸ್ಟ್ ಬೈಟ್ (ಇನ್‌ಪುಟ್ ಪ್ರೊಸೆಸಿಂಗ್) ಗೆ ಮತ್ತು ಲಾಸ್ಟ್ ಬೈಟ್ (ಔಟ್‌ಪುಟ್) ಗೆ ಅಪ್ಲಿಕೇಶನ್ ಲೇಟನ್ಸಿಯನ್ನು ಟ್ರ್ಯಾಕ್ ಮಾಡಿ.

## 8. ಪರಿಸರ ಚರ (Environment Variables)

ಇದಲ್ಲಾಗಿಯೇ ನಾವು ಬ್ರೌಸರ್‌ನಲ್ಲಿ ನಿಯೋಜನೆಯನ್ನು ನಡೆಸಿದ್ರು - ಮತ್ತು ನಮ್ಮ ಇನ್‌ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಪ್ರೊವಿಜನ್ ಆಗಿದ್ದು ಮತ್ತು ಅಪ್ಲಿಕೇಶನ್ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತಿರುವುದನ್ನು ಮಾನ್ಯಗೊಳಿಸಿದೆವು. ಆದರೆ ಕೋಡ್-ಫರ್ಟ್ ರೀತಿಯಲ್ಲಿ ಅಪ್ಲಿಕೇಶನ್ ಜೊತೆಗೆ ಕೆಲಸ ಮಾಡಲು, ನಾವು ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿ ಪರಿಸರವನ್ನು ಆ ಸಂಪನ್ಮೂಲಗಳೊಂದಿಗೆ ಕೆಲಸ ಮಾಡಲು ಅಗತ್ಯವಿರುವ ಸಂಬಂಧಿತ ಪರಿವರ್ತನೆಗಳೊಂದಿಗೆ ಸಂರಚಿಸಬೇಕಾಗುತ್ತದೆ. `azd` ಬಳಸುವುದರಿಂದ ಇದು ಸುಲಭವಾಗುತ್ತದೆ.

1. Azure Developer CLI [environment variables ಬಳಸಿ](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) ಅಪ್ಲಿಕೇಶನ್ ನಿಯೋಜನೆಗಳಿಗೆ ಕಾನ್ಫಿಗರೇಶನ್ ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು ಸಂಗ್ರಹಿಸಲು ಮತ್ತು ನಿರ್ವಹಿಸಲು ಉಪಯೋಗಿಸುತ್ತದೆ.

1. ಪರಿಸರ ಚರಗಳು `.azure/<env-name>/.env` ನಲ್ಲಿ ಸಂಗ್ರಹಿಸಲಾಗುತ್ತವೆ - ಇದು ಅವುಗಳನ್ನು ನಿಯೋಜನೆ ಸಮಯದಲ್ಲಿ ಉಪಯೋಗಿಸಲಾದ `env-name` ಪರಿಸರಕ್ಕೆ ಪರಿಧಿ ಮಾಡುತ್ತದೆ ಮತ್ತು ಸಮಾನ ರೆಪೊದಲ್ಲಿ ವಿಭಿನ್ನ ನಿಯೋಜನೆ ಗುರಿಗಳಿಗಾಗಿ ಪರಿಸರಗಳನ್ನು ಅಲಗಾಗಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

1. `azd` ಕಮಾಂಡ್ ಯಾವುದೇ ನಿರ್ದಿಷ್ಟ ಕಮಾಂಡ್ (ಉದಾ., `azd up`) ಎಕ್ಸ್‌ಕ್ಯೂಟ್ ಮಾಡುವಾಗ ಪರಿಸರ ಚರಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಲೋಡ್ ಮಾಡುತ್ತದೆ. ಗಮನಿಸಿ `azd` ಸಿಸ್ಟಂ-ಮಟ್ಟದ (ಓಎಸ್-ಮಟ್ಟ) ಪರಿಸರ ಚರಗಳನ್ನು (ಉದಾ., ಶೆಲ್ ನಲ್ಲಿ ಸೆಟ್ ಮಾಡಿದವು) ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಓದುತ್ತಿಲ್ಲ - ಬದಲಿ `azd set env` ಮತ್ತು `azd get env` ಅನ್ನು ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳ ನಡುವೆ ಮಾಹಿತಿ ವರ್ಗಾಯಿಸಲು ಬಳಸಿರಿ.


ನಾವು ಕೆಲವು ಕಮಾಂಡ್‌ಗಳನ್ನು ಪ್ರಯತ್ನಿಸೋಣ:

1. ಈ ಪರಿಸರದಲ್ಲಿ `azd`ಗಾಗಿ ಸೆಟ್ ಮಾಡಿದ ಎಲ್ಲಾ ಪರಿಸರ ಚರಗಳನ್ನು ಪಡೆಯಿರಿ:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      ನಿಮಗೆ ಇಂತಹದು ಕಾಣಿಸಬಹುದು:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4o-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. ವಿಶೇಷ ಮೌಲ್ಯವನ್ನು ಪಡೆಯಿರಿ - ಉದಾ., ನಾನು `AZURE_AI_AGENT_MODEL_NAME` ಮೌಲ್ಯವನ್ನು ಸೆಟ್ ಮಾಡarlಾ ಎಂಬುದನ್ನು ತಿಳಿದುಕೊಳಲು

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      ನಿಮಗೆ ಹೀಗೆ ಕಾಣಿಸಬಹುದು - ಇದು ಡಿಫಾಲ್ಟ್ ಮೂಲಕ ಸೆಟ್ ಆಗಿರಲಿಲ್ಲ!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd` ಗಾಗಿ ಹೊಸ ಪರಿಸರ ಚರವನ್ನು ಸೆಟ್ ಮಾಡಿ. ಇಲ್ಲಿ, ನಾವು ಏಜೆಂಟ್ ಮಾದರಿಯ ಹೆಸರನ್ನು ಅಪ್ಡೇಟ್ ಮಾಡುತ್ತೇವೆ. _ನೋಟು: ಯಾವುದೇ ಬದಲಾವಣೆಗಳು ತಕ್ಷಣ `.azure/<env-name>/.env` ಫೈಲ್‌ನಲ್ಲಿ ಪ್ರತಿಬಿಂಬಿತವಾಗುತ್ತವೆ._

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      ಈಗ, ನಾವು ಅಲ್ಲಿ ಮೌಲ್ಯ ಸેટ್ ಆಗಿದೆ ಎಂದು ಕಂಡುಕೊಳ್ಳಬೇಕು:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. ಕೆಲವೊಂದು ಸಂಪನ್ಮೂಲಗಳು ಸ್ಥಿರವಾಗಿವೆ (ಉದಾ., ಮಾದರಿ ನಿಯೋಜನೆಗಳು) ಮತ್ತು ಅವುಗಳನ್ನು ಪುನರ್‌ನಿಯೋಜಿಸಲು ಕೇವಲ `azd up` ಯನ್ನು ಬಿಡಲು ಸಾಕಾಗುವುದಿಲ್ಲ. ಮೂಲ ನಿಯೋಜನೆಯನ್ನು ತೆಗೆಯುವ ಮತ್ತು ಬದಲಿಸಿದ ಎನ್‌ವಿ.vars ಸಹಿತ ಮರುನಿಯೋಜಿಸುವ ಪ್ರಯತ್ನ ಮಾಡೋಣ.

1. **Refresh** If you had previously deployed infrastructure using an azd template - you can _refresh_ the state of your local environment variables based on the current state of your Azure deployment using this command:

      ```bash title="" linenums="0"
      azd env refresh
      ```

      ಇದು ಎರಡು ಅಥವಾ ಹೆಚ್ಚಿನ ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿ ಪರಿಸರಗಳಲ್ಲಿ (ಉದಾ., ಅನೇಕ ಡೆವಲಪರ್‌ಗಳೊಂದಿಗೆ ತಂಡ) ಪರಿಸರ ಚರಗಳನ್ನು _ಸಮನ್ವಯಗೊಳಿಸುವ_ ಶಕ್ತಿಶಾಲಿ ವಿಧಾನವಾಗಿದೆ - ನಿಯೋಜಿಸಲಾದ ಇನ್‌ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್‌ನ್ನು ಪರಿಸರ ಚರಗಳ ಸ್ಥಿತಿಗೆ ಯಥಾರ್ಥ ಮೂಲವಾಗಿರಲು ಅನುಮತಿಸುತ್ತದೆ. ತಂಡದ ಸದಸ್ಯರು ಕೇವಲ ಚರಗಳನ್ನು _ನವೀಕರಿಸಿ_ ಮರುಸಮನ್ವಯಕ್ಕೆ ಬರಬಹುದು.
