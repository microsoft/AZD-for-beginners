# 3. ಸ್ವರೂಪವನ್ನು ವಿಭಜಿಸಿ

!!! tip "ಈಮಾಡ್ಯೂಲ್‌ನ ಕೊನೆಯಲ್ಲಿ ನೀವು ಸಾಧ್ಯವಾಗುವುದು"

    - [ ] Azure ಸಹಾಯಕ್ಕಾಗಿ MCP ಸರ್ವರ್‌ಗಳೊಂದಿಗೆ GitHub Copilot ಅನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿ
    - [ ] AZD ಟೆಂಪ್ಲೇಟ್ ಫೋಲ್ಡರ್ ರಚನೆ ಮತ್ತು ಘಟಕಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಿ
    - [ ] ಮೂಲಸೌಕರ್ಯ-ಆಕ್ಸ ಕೋಡ್ (Bicep) ಸಂಘಟನೆಯ ಪ್ಯಾಟರ್ನ್‌ಗಳನ್ನು ಅನ್ವೇಷಿಸಿ
    - [ ] **ಲ್ಯಾಬ್ 3:** GitHub Copilot ಬಳಸಿ ರೆಪೋಸಿಟರಿ ಆರ್ಕಿಟೆಕ್ಚರ್ ಅನ್ನು ಅನ್ವೇಷಿಸಿ ಮತ್ತು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಿ

---

AZD ಟೆಂಪ್ಲೇಟುಗಳು ಮತ್ತು Azure ಡೆವಲಪರ್ CLI (`azd`) ಜೊತೆ ನಾವು ಮಾದರಿ ಕೋಡ್, ಮೂಲಸೌಕರ್ಯ ಮತ್ತು ಸಂರಚನಾ ಫೈಲ್‌ಗಳನ್ನು ಒದಗಿಸುವ ಮಾನಕೃತ ರೆಪೋಸಿಟರಿಗಳನ್ನು ಉಪಯೋಗಿಸಿಕೊണ്ട് ನಮ್ಮ AI ಅಭಿವೃದ್ಧಿ ಪ್ರಯಾಣವನ್ನು ವೇಗವಾಗಿ ಪ್ರಾರಂಭಿಸಬಹುದು - ತಯಾರಿನೊಂದಿಗೆ ನಿಯೋಜಿಸಲು ಸಿದ್ಧವಾದ _ಸ್ಟಾರ್ಟರ್_ ಪ್ರಾಜೆಕ್ಟ್ ರೂಪದಲ್ಲಿ.

**ಆದರೆ ಈಗ, ನಾವು ಪ್ರಾಜೆಕ್ಟ್ ರಚನೆ ಮತ್ತು ಕೋಡ್‌ಬೇಸ್ ಅನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬೇಕಾಗಿದೆ - ಮತ್ತು ಯಾವುದೇ ಪೂರ್ವಾನುಭವವಿಲ್ಲದೆ AZD ಟೆಂಪ್ಲೆಟ್ನನ್ನು ಕಸ್ಟಮೈಸ್ ಮಾಡಲು ಸಾಧ್ಯವಾಗಬೇಕು!**

## 1. GitHub Copilot ಸಕ್ರಿಯಗೊಳಿಸಿ

### 1.1 GitHub Copilot ಚಾಟ್ ಅನ್ನು ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಿ

[GitHub Copilot Agent Mode](https://code.visualstudio.com/docs/copilot/chat/chat-agent-mode) ಅನ್ನು ಅನ್ವೇಷಿಸಲು ಸಮಯವಾಗಿದೆ. ಈಗ, ನಾವು ಸ್ವಾಭಾವಿಕ ಭಾಷೆಯನ್ನು ಉಪಯೋಗಿಸಿ ನಮ್ಮ ಕಾರ್ಯವನ್ನು ಉನ್ನತ ಮಟ್ಟದಲ್ಲಿ ವಿವರಿಸಲು ಮತ್ತು ನೆರವನ್ನು ಪಡೆಯಲು ಸಾಧ್ಯ. ಈ ಲ್ಯಾಬ್‌ಗೆ, ನಾವು ಮಾಸಿಕ ಪೂರ್ಣಗೊಳಿಸುವಿಕೆ ಮತ್ತು ಚಾಟ್ ಸಂವಾದಗಳ ಮಿತಿ ಹೊಂದಿರುವ [Copilot Free ಯೋಜನೆಯನ್ನು](https://github.com/github-copilot/signup) ಬಳಸುತ್ತೇವೆ.

ವಿಸ್ತರಣೆ ಮಾರುಕಟ್ಟೆಯಿಂದ ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಬಹುದು ಮತ್ತು ಇದೊಂದೇ Codespaces ಅಥವಾ dev container ಪರಿಸರಗಳಲ್ಲಿ ಅಗತ್ಯವಿದ್ದರೆ ಈಗಾಗಲೇ ಲಭ್ಯವಿರಬಹುದು. _Copilot ಐಕಾನ್ ಡ್ರಾಪ್-ಡೌನ್ ನಿಂದ `Open Chat` ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ - ಮತ್ತು `What can you do?` ಎಂಬ ಪ್ರಾಂಪ್ಟ್ ಟೈಪ್ ಮಾಡಿ_ - ನೀವು ಲಾಗಿನ್ ಆಗಲು ಕೇಳಬಹುದು. **GitHub Copilot Chat ಸಿದ್ಧವಾಗಿದೆ**.

### 1.2. MCP ಸರ್ವರ್‌ಗಳನ್ನು ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಿ

Agent ಮೋಡ್ ಪರಿಣಾಮಕಾರಿ ಆಗಿರಲು ಸರಿಯಾದ ಸಾಧನಗಳಿಗೆ ಪ್ರವೇಶ ಬೇಕು, ಅದು ಜ್ಞಾನವನ್ನು ಪಡೆಯುವುದಕ್ಕೆ ಅಥವಾ ಕಾರ್ಯಗಳನ್ನು ಕೈಗೊಳ್ಳಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ. ಇಲ್ಲಿ MCP ಸರ್ವರ್‌ಗಳು ಸಹಾಯ ಮಾಡುತ್ತವೆ. ನಾವು ಕೆಳಗಿನ ಸರ್ವರ್‌ಗಳನ್ನು ಸಂರಚಿಸುವೆವು:

1. [Azure MCP Server](../../../../../workshop/docs/instructions)
1. [Microsoft Docs MCP Server](../../../../../workshop/docs/instructions)

ಇವುಗಳನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಲು:

1. `.vscode/mcp.json` ಎಂಬ ಫೈಲ್ ಅನ್ನು ಇಲ್ಲದಿದ್ದರೆ ರಚಿಸಿ
1. ಈ ಕೆಳಗಿನ ವಿಷಯವನ್ನು ಆ ಫೈಲಿಗೆ ನಕಲಿಸಿ - ಮತ್ತು ಸರ್ವರ್‌ಗಳನ್ನು ಪ್ರಾರಂಭಿಸಿ!
   ```json title=".vscode/mcp.json"
   {
      "servers": {
         "Azure MCP Server": {
            "command": "npx",
            "args": [
            "-y",
            "@azure/mcp@latest",
            "server",
            "start"
            ]
         },
         "microsoft.docs.mcp": {
            "type": "http",
            "url": "https://learn.microsoft.com/api/mcp"
         }
      }
   }
   ```

??? warning "ನೀವು `npx` ಇನ್‌ಸ್ಟಾಲ್ ಆಗಿಲ್ಲ ಎಂದು ದೋಷವೊಂದು ಎದುರಾಗಬಹುದು (ತಿದ್ದುಪಡಿ ನೋಡಲು ಕ್ಲಿಕ್ ಮಾಡಿ)"

      ಈ ಸಮಸ್ಯೆಯನ್ನು ಸರಿಪಡಿಸಲು, `.devcontainer/devcontainer.json` ಫೈಲ್ ಅನ್ನು ತೆರೆಯಿರಿ ಮತ್ತು features ವಿಭಾಗದಲ್ಲಿ ಈ ಸಾಲನ್ನು ಸೇರಿಸಿ. ನಂತರ ಕಂಟೇನರ್ ಅನ್ನು ಪುನಃ ನಿರ್ಮಿಸಿ. ಇದರಿಂದ ನೀವು `npx` ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಿಕೊಳ್ಳಬಹುದು.

      ```title="" linenums="0"
         "features": {
            "ghcr.io/devcontainers/features/node:1": {},
            ...
         },
      ```

### 1.3. GitHub Copilot Chat ಅನ್ನು ಪರೀಕ್ಷಿಸಿ

**ಪ್ರಥಮವಾಗಿ VS Code ಕಮಾಂಡ್ ಲೈನ್‌ನಿಂದ Azureಗೆ ಪ್ರಾಮಾಣೀಕರಿಸಲು `azd auth login` ಬಳಸಿ. ನೇರವಾಗಿ Azure CLI ಕಮಾಂಡ್‌ಗಳನ್ನು ನಡೆಯಿಸಲು `az login` ಅನ್ನು ಸಹ ಬಳಸಿ.**

ನೀವು ಈಗ ನಿಮ್ಮ Azure ಸಬ್ಸ್ಕ್ರಿಪ್ಷನ್ ಸ್ಥಿತಿಯನ್ನು ಪ್ರಶ್ನಿಸಿ, ನಿಯೋಜಿತ ಸಂಪನ್ಮೂಲಗಳು ಅಥವಾ ಸಂರಚನೆಯ ಬಗ್ಗೆ ಪ್ರಶ್ನೆಗಳೆಂದು ಕೇಳಬಹುದು. ಈ ಕೆಳಗಿನ ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ಪ್ರಯತ್ನಿಸಿ:

1. `List my Azure resource groups`
1. `#foundry list my current deployments`

ನೀವು Azure ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಬಗ್ಗೆ ಪ್ರಶ್ನೆಗಳು ಕೇಳಬಹುದು ಮತ್ತು Microsoft Docs MCP ಸರ್ವರ್‌ನ ಆಧಾರಿತ ಉತ್ತರಗಳನ್ನು ಪಡೆಯಬಹುದು. ಈ ಕೆಳಗಿನವು ಪ್ರಯತ್ನಿಸಿ:

1. `#microsoft_docs_search What is Azure Developer CLI?`
1. `#microsoft_docs_search Show me a Python tutorial to chat with deployed model`

ಅಥವಾ ಕಾರ್ಯ ಪೂರ್ಣಗೊಳಿಸಲು ಕೋಡ್ ಸ್ನಿಪೆಟ್‌ಗಳನ್ನು ಕೇಳಬಹುದು. ಈ ಪ್ರಾಂಪ್ಟ್ ಅನ್ನು ಪ್ರಯತ್ನಿಸಿ.

1. `Give me a Python code example that uses AAD for an interactive chat client`

`Ask` ಮೋಡ್‌ನಲ್ಲಿ, ನೀವು ಈ ಕೋಡ್ ನಯವಾಗಿ ನಕಲಿಸಿ ಪ್ರಯತ್ನಿಸಬಹುದು. `Agent` ಮೋಡ್‌ನಲ್ಲಿ, ಇದು ಮತ್ತಷ್ಟು ಮುಂದುವರಿದು ಸಂಬಂಧಿತ ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿಮ್ಮ ಪರ ನಿರ್ಮಿಸುವುದು - ಸೆಟ್ ಅಪ್ ಸ್ಕ್ರಿಪ್ಟ್ ಮತ್ತು ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಸೇರಿದಂತೆ - ಆ ಕಾರ್ಯವನ್ನು ನಡಿಸುವಲ್ಲಿ ಸಹಾಯ ಮಾಡಲು.

**ನೀವು ಈಗ ಟೆಂಪ್ಲೇಟ್ ರೆಪೋಸಿಟರಿ ಅನ್ವೇಷಿಸಲು ಸಜ್ಜಾಗಿದ್ದೀರಿ**

## 2. ವಾಸ್ತುಶಿಲ್ಪವನ್ನು ವಿಭಜಿಸಿ

??? prompt "ಅನುರೋಧ: docs/images/architecture.png ನಲ್ಲಿ ಅಪ್ಲಿಕೇಷನ್ ವಾಸ್ತುಶಿಲ್ಪವನ್ನು 1 ಪ್ಯಾರಾಗ್ರಾಫ್ನಲ್ಲಿ ವಿವರಿಸಿ"

      ಈ ಅಪ್ಲಿಕೇಷನ್ ಒಂದು Azure ಮೇಲೆ ನಿರ್ಮಿತ ಆಧುನಿಕ ಏಜೆಂಟ್ ಆಧಾರಿತ ವಾಸ್ತುಶಿಲ್ಪವನ್ನು ಪ್ರತಿಪಾದಿಸುವ AI ಚಾಲಿತ ಚಾಟ್ ಅಪ್ಲಿಕೇಷನ್ ಆಗಿದೆ. ಈ ಪರಿಹಾರದ ಕೇಂದ್ರವಾಗಿದೆ Azure ಕಂಟೇನರ್ ಅಪ್ಲಿಕೇಷನ್, ಅದು ಮುಖ್ಯ ಅಪ್ಲಿಕೇಷನ್ ಕೋಡ್ ಅನ್ನು ಹೊಂದಿದ್ದು, ಬಳಕೆದಾರರ ಇನ್‌ಪುಟ್ ಅನ್ನು ಪ್ರಕ್ರಿಯೆ ಮಾಡಿ AI ಏಜೆಂಟ್ ಮುಖಾಂತರ ಬುದ್ಧಿವಂತ ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು ಉತ್ಪಾದಿಸುತ್ತದೆ.
      
      ವಾಸ್ತುಶಿಲ್ಪವು Microsoft Foundry ಪ್ರಾಜೆಕ್ಟ್ ಅನ್ನು AI ಸಾಮರ್ಥ್ಯಗಳಿಗಾಗಿನ ಮೂಲಭೂತವಾಗಿ ನಿರ್ವಹಿಸುತ್ತದೆ, ಅದು ಮೂಲ ಭಾಷಾ ಮಾದರಿಗಳನ್ನು (gpt-4.1-mini ಮುಂತಾದವುಗಳು) ಮತ್ತು ಏಜೆಂಟ್ ಕಾರ್ಯಾಚರಣೆಗಳು ದೊರಕುವಂತೆ Azure AI ಸೇವೆಗಳೊಂದಿಗೆ ಸಂಪರ್ಕ ಹೊಂದಿದೆ. ಬಳಕೆದಾರರ ಸಂವಾದಗಳು React ಆಧಾರಿತ ಫ್ರಂಟೆಂಡ್ ಮೂಲಕ FastAPI ಬ್ಯಾಕ್‌ಎಂಡ್‌ಗೆ ಹೋಗುತ್ತವೆ, ಅದು AI ಏಜೆಂಟ್ ಸೇವೆಯನ್ನು ಸಂಪರ್ಕಿಸಿ ಸಾಂದರ್ಭಿಕ ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು ಉತ್ಪಾದಿಸುತ್ತದೆ.
      
      ವ್ಯವಸ್ಥೆಯಲ್ಲಿ ಜ್ಞಾನ ಪಡೆದಿರುವ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಕಡತ ಶೋಧನೆಯಿಂದ ಅಥವಾ Azure AI Search ಸೇವೆಯಿಂದ ಬಳಸಲಾಗುತ್ತದೆ, ಏಜೆಂಟ್ ಅಪ್ಲೋಡ್ ಮಾಡಿದ ಡಾಕ್ಯುಮೆಂಟ್‌ಗಳಿಂದ ಮಾಹಿತಿ ಪ್ರವೇಶಿಸಿ ಉಲ್ಲೇಖಿಸಲು ಸಾಧ್ಯವಾಗಿದೆ.
      
      ಕಾರ್ಯಚಟುವಟಿಕೆಯ ಶ್ರೇಷ್ಠತೆಯನ್ನು ಸಾಧಿಸಲು ಅಪ್ಲಿಕೇಷನ್ ಇನ್ಸೈಟ್ಸ್ ಮತ್ತು ಲಾಗ್ ಅನಾಲಿಟಿಕ್ಸ್ ವರ್ಕ್‌ಶೇಸ್ ಮೂಲಕ ವಿಸ್ತೃತ ಮониಟರಿಂಗ್ ಸೌಲಭ್ಯಗಳನ್ನು ಒಳಗೊಂಡಿದೆ.
      
      ಅಪ್ಲಿಕೇಷನ್ ಡೇಟಾ ಮತ್ತು ಫೈಲ್ ಅಪ್ಲೋಡ್ ಗಳಿಗಾಗಿ Azure ಸ್ಟೋರೇಜ್ ಬ್ಲಾಬ್ ಸ್ಟೋರೇಜ್ ಒದಗಿಸುತ್ತದೆ, ಮತ್ತು ಮೇಲ್ವಿಚಾರಿತ ಗುರುತಿನೈರಾಶ್ಯ (Managed Identity) ಮೂಲಕ ಕ್ರೆಡೆನ್ಷಿಯಲ್‌ಗಳನ್ನು ಸಂಗ್ರಹಿಸದೆ ಸುರಕ್ಷಿತ ಪ್ರವೇಶವನ್ನು ಖಾತ್ರಿ ಪಡಿಸುತ್ತದೆ. ಈ ಸಂಪೂರ್ಣ ಪರಿಹಾರವು ಸ್ಕೇಲಬಿಲಿಟಿ ಮತ್ತು ನಿರ್ವಹಣಾಶೀಲತೆಗೆ ವಿನ್ಯಾಸಗೊಳಿಸಲಾಗಿದೆ, ಡಿಮ್ಯಾಂಡ್ ಆಧರಿಸಿ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಕಂಟೇನರ್ ಅಪ್ಲಿಕೇಷನ್ ವಿಸ್ತಾರಗೊಳ್ಳುತ್ತದೆ ಮತ್ತು Azureನ ನಿರ್ವಹಿತ ಸೇವೆಗಳ ಮೂಲಕ ಅಳವಡಿಕೆಯಲ್ಲಿ ಸುರಕ್ಷತೆ, ನಿಗಾವಣೆ ಮತ್ತು ಸಿಐ/ಸಿಡಿ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ.

## 3. ರೆಪೋಸಿಟರಿ ರಚನೆ

!!! prompt "ಅನುರೋಧ: ಟೆಂಪ್ಲೇಟಿನ ಫೋಲ್ಡರ್ ರಚನೆಯನ್ನು ವಿವರಿಸಿ. ದೃಶ್ಯಮಟ್ಟದ ಸೋರಣೆಸಿದ ಹೈರಾರ್ಕಿಕಲ್ ಡಯಾಗ್ರಾಮ್‌ನಿಂದ ಪ್ರಾರಂಭಿಸಿ."

??? info "ಉತ್ತರ: ದೃಶ್ಯಮಟ್ಟದ ಹೈರಾರ್ಕಿಕಲ್ ಡಯಾಗ್ರಾಮ್"

      ```bash title="" 
      get-started-with-ai-agents/
      ├── 📋 ಸಂರಚನೆ & ಸೆಟ್ ಅಪ್
      ```

### 3.1. ಕೋರ್ ಅಪ್ಲಿಕೇಷನ್ ವಾಸ್ತುಶಿಲ್ಪ

ಈ ಟೆಂಪ್ಲೇಟು **ಫುಲ್-ಸ್ಟಾಕ್ ವೆಬ್ ಅಪ್ಲಿಕೇಷನ್** ಮಾದರಿಯನ್ನು ಅನುಸರಿಸುತ್ತದೆ:

- **ಬ್ಯಾಕ್‌ಎಂಡ್**: Python FastAPI ಜೊತೆ Azure AI ಸಂಯೋಜನೆ
- **ಫ್ರಂಟೆಂಡ್**: TypeScript/React ಜೊತೆ Vite ನಿರ್ಮಾಣ ವ್ಯವಸ್ಥೆ
- **ಮೂಲಸೌಕರ್ಯ**: ಕ್ಲೌಡ್ ಸಂಪನ್ಮೂಲಗಳಿಗೆ Azure Bicep ಟೆಂಪ್ಲೇಟ್‌ಗಳು
- **ಕಂಟೇನರೈಜೆಶನ್**: ಒಂದೇ ಸಮರ್ಪಕ ನಿಯೋಜನೆಗಾಗಿ Docker

### 3.2 ಮೂಲಸೌಕರ್ಯ ಆಕ್ಸ್ ಕೋಡ್ (bicep)

ಮೂಲಸೌಕರ್ಯದ ಹೊಂಗೆ **Azure Bicep** ಟೆಂಪ್ಲೇಟ್‌ಗಳನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ ಮತ್ತು ಉಪಘಟಕಗಳಾಗಿ ಸಂಘಟಿಸಲಾಯಿತು:

   - **`main.bicep`**: ಎಲ್ಲಾ Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿಯಂತ್ರಿಸುತ್ತದೆ
   - **`core/` ಘಟಕಗಳು**: ವಿವಿಧ ಸೇವೆಗಳಿಗೆ ಪುನಃಬಳಕೆಯಾದ ಘಟಕಗಳು
      - AI ಸೇವೆಗಳು (Microsoft Foundry Models, AI Search)
      - ಕಂಟೇನರ್ ಆತಿಥ್ಯ (Azure Container Apps)
      - ನಿಗಾವಣೆ (Application Insights, Log Analytics)
      - ಭದ್ರತೆ (Key Vault, Managed Identity)

### 3.3 ಅಪ್ಲಿಕೇಷನ್ ಮೂಲ (`src/`)

**ಬ್ಯಾಕ್‌ಎಂಡ್ API (`src/api/`)**:

- FastAPI ಆಧಾರಿತ REST API
- Foundry ಏಜೆಂಟ್ಸ್ ಸಂಯೋಜನೆ
- ಜ್ಞಾನ ಪಡೆದಿರುವ ಶೋಧ ಸೂಚ್ಯಂಕ ನಿರ್ವಹಣೆ
- ಫೈಲ್ ಅಪ್‌ಲೋಡ್ ಮತ್ತು ಪ್ರಕ್ರಿಯೆಯನ್ನು ಮಾಡುವುದು

**ಫ್ರಂಟೆಂಡ್ (`src/frontend/`)**:

- ಆಧುನಿಕ React/TypeScript SPA
- ವೇಗದ ಅಭಿವೃದ್ಧಿ ಮತ್ತು ಸುಧಾರಿತ ನಿರ್ಮಾಣಕ್ಕಾಗಿ Vite
- ಏಜೆಂಟ್ ಸಂವಾದ ಇಂಟರ್‌ಫೇಸ್

**ಜ್ಞಾನ ಆಧಾರ (`src/files/`)**:

- ಮಾದರಿ ಗ್ರಾಹಕ ಮತ್ತು ಉತ್ಪನ್ನ ಡೇಟಾ
- ಫೈಲ್ ಆಧಾರಿತ ಜ್ಞಾನ ಪಡೆಯುವುದನ್ನು ಪ್ರತಿಪಾದಿಸುತ್ತದೆ
- JSON ಮತ್ತು Markdown ರೂಪದಲ್ಲಿ ಉದಾಹರಣೆಗಳು

### 3.4 ಡೆವ್‌ಆಪ್ಸ್ ಮತ್ತು ಸ್ವಯಂಚಾಲನೆ

**ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳು (`scripts/`)**:

- ಕ್ರಾಸ್-ಪ್ಲಾಟ್‌ಫಾರ್ಮ್ PowerShell ಮತ್ತು Bash ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳು
- ಪರಿಸರ ಮೌಲ್ಯಗಳ ಮಾನ್ಯತೆ ಮತ್ತು ಸೆಟ್ ಅಪ್
- ನಿಯೋಜನೆ ನಂತರ ಸಂರಚನೆ
- ಮಾದರಿ ಕೋಟಾ ನಿರ್ವಹಣೆ

**Azure ಡೆವಲಪರ್ CLI ಏಕೀಕರಣ**:

- `azure.yaml` ಫೈಲ್‌ನಲ್ಲಿ `azd` ವರ್ಕ್ಫ್ಲೋ ಸಂರಚನೆ
- ಸ್ವಯಂಚಾಲಿತ ಜಾಗೃತಿ ಮತ್ತು ನಿಯೋಜನೆ
- ಪರಿಸರ ಮೌಲ್ಯ ನಿರ್ವಹಣೆ

### 3.5 ಪರೀಕ್ಷಣೆ ಮತ್ತು ಗುಣಮಟ್ಟ ಖಾತ್ರಿ

**ಮೌಲ್ಯಮಾಪನ ಚೌಕಟ್ಟಿ (`evals/`)**:

- ಏಜೆಂಟ್ ಕಾರ್ಯಕ್ಷಮತೆ ಮೌಲ್ಯಮಾಪನ
- ಪ್ರಶ್ನೆ-ಪ್ರತಿಕ್ರಿಯಾ ಗುಣಮಟ್ಟ ಪರೀಕ್ಷೆ
- ಸ್ವಯಂಚಾಲಿತ ಅಂದಾಜು ಹಾದಿ

**AI ಸುರಕ್ಷತೆ (`airedteaming/`)**:

- AI ಸುರಕ್ಷತೆಗೆ ರೆಡ್ ಟೀಮ್ ಪರೀಕ್ಷೆ
- ಭದ್ರತಾ ದೌರ್ಬಲ್ಯ ಸ್ಕ್ಯಾನಿಂಗ್
- ಜವಾಬ್ದಾರಿಯುತ AI ಆಯಾಮಗಳು
