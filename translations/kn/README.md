## 🆕 ಇಂದು azd ನಲ್ಲಿ ಹೊಸದು ಏನು

> 📌 ಈ ಕೋರ್ಸ್ ಅನ್ನು **`azd 1.27.1`** (ಜುಲೈ 2026) ಆಧರಿಸಿ ಪರಿಶೀಲಿಸಲಾಗಿದೆ. ನಿಮ್ಮ ಬಿಳ್ಡ್ ಪರಿಶೀಲಿಸಲು `azd version` ಅನ್ನು ರನ್ ಮಾಡಿ, ಮತ್ತು ಇತ್ತೀಚಿನದನ್ನು ಪಡೆಯಲು `azd upgrade` ರನ್ ಮಾಡಿ.

ಅಜೂರ್ ಡೆವಲಪರ್ CLI ಸಾಂಪ್ರದಾಯಿಕ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್ಗಳು ಮತ್ತು API ಗಳನ್ನೇ ಮೀರಿಸಿ ಬೆಳೆಯಿತು. ಇಂದಿನ ದಿನದಲ್ಲಿ, azd ಯಾವುದೇ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಅಜೂರ್ ಗೆ ನಿಯೋಜಿಸಲು ಏಕೈಕ ಸಾಧನವಾಗಿದೆ— AI ಚಾಲಿತ ಅಪ್ಲಿಕೇಶನ್ಗಳಿಗೂ ಒಳಗೊಂಡಂತೆ.

ಇದೇ ನಿಮ್ಮಿಗಾಗಿಯೇ ಅರ್ಥವಾಗುವುದು:

- **AI ಏಜೆಂಟ್‌ಗಳು ಈಗ ಮೊದಲ-ಶ್ರೇಣಿಯ azd ಕೆಲಸಗಳಾಗಿವೆ.** ನೀವು ಈಗಾಗಲೇ ಪರಿಚಿತರಾಗಿರುವ `azd init` → `azd up` ಕಾರ್ಯಪ್ರವಾಹವನ್ನು ಬಳಸಿ AI ಏಜೆಂಟ್ ಯೋಜನೆಗಳನ್ನು ಆರಂಭಿಸಲು, ನಿಯೋಜಿಸಲು ಮತ್ತು ನಿರ್ವಹಿಸಲು ಸಾಧ್ಯ.
- **CLI ಮೂಲಕ ಸಂಪೂರ್ಣ ಏಜೆಂಟ್ ಜೀವನಚಕ್ರ.** `azure.ai.agents` ಎಕ್ಸ್‌ಟೆನ್ಷನ್ ಈಗ ಸಂಪೂರ್ಣ ಪಯಣವನ್ನು ಒಳಗೊಂಡಿದೆ—`azd ai agent init` ನಲ್ಲಿ ರೂಪುರೇಷೆ ಮಾಡಲು, `azd ai agent invoke` ಮೂಲಕ ಪರೀಕ್ಷಿಸಲು (ಪ್ರತಿಕ್ರಿಯಾ ಸಮಯ ಚಿತ್ರೀಕರಣದೊಂದಿಗೆ), `azd ai agent eval generate` ಮತ್ತು `azd ai agent optimize` ಮೂಲಕ ಗುಣಮಟ್ಟವನ್ನು ಅಳೆಯಲು ಮತ್ತು ಸುಧಾರಿಸಲು, ಮತ್ತು `azd ai agent delete` ಮೂಲಕ ಶುದ್ಧೀಕರಿಸಲು.
- **ಹೆಚ್ಚಿನ AI ನಿರ್ಮಾಣ ಬ್ಲಾಕ್ಗಳು.** ಹೊಸ ಪೂರ್ವವೀಕ್ಷಣೆ ಎಕ್ಸ್‌ಟೆನ್ಷನ್‌ಗಳು—`azure.ai.skills` ಮತ್ತು `azure.ai.connections`—ನನಗೆ ಮರುಬಳಕೆ ಮಾಡಬಹುದಾದ ಏಜೆಂಟ್ ಕೌಶಲ್ಯಗಳನ್ನು ಮತ್ತು ಫೌಂಡ್ರಿ ಸಂಪರ್ಕಗಳನ್ನು ನೇರವಾಗಿ azd ಮೂಲಕ ನಿರ್ವಹಿಸಲು ಅವಕಾಶ ನೀಡುತ್ತವೆ.
- **Microsoft Foundry ಒಕ್ಕೂಟ** ಮಾದರಿ ನಿಯೋಜನೆ, ಏಜೆಂಟ್ ಆತಿಥ್ಯ ಮತ್ತು AI ಸೇವಾ ಕನ್ಫಿಗರೇಶನನ್ನು ನೇರವಾಗಿ azd ಟೆಂಪ್ಲೇಟ್ ಪರಿಸಿ ವ್ಯವಸ್ಥೆಗೆ ತರಲಿದೆ.
- **ದಿನನಿತ್ಯದ ಮೂಲಭೂತ ಕಾರ್ಯಗಳಲ್ಲಿ ಸುಗಮತೆ.** ಇತ್ತೀಚಿನ ಬಿಡುಗಡೆಗಳು `azd init` ಅನ್ನು ಪುನರಾವರ್ತಿಸಲು ಸುರಕ್ಷಿತ (idempotent) ಮಾಡಿವೆ, `azd auth login` ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಹಳೆಯ ಟೋಕನ್‌ಗಳನ್ನು ತೆರವುಗೊಳಿಸುತ್ತದೆ, ಮತ್ತು `azd tool` ಪ್ರಥಮ-ರನ್ ಸೆಟಪ್ ಪ್ರಾಂಪ್ಟ್ ಅನ್ನು ಸೇರಿಸಿದೆ.
- **ಮೂಲ ಕಾರ್ಯಪ್ರವಾಹದಲ್ಲಿ ಯಾವುದೇ ಬದಲಾವಣೆ ಇಲ್ಲ.** ನೀವು ಟೋಡೋ ಅಪ್ಲಿಕೇಶನ್, ಮೈಕ್ರೋಸರ್ವೀಸ್ ಅಥವಾ ಬಹು-ಏಜೆಂಟ್ AI ಪರಿಹಾರವನ್ನು ನಿಯೋಜಿಸುತ್ತಿದ್ದರೂ, ಆಜ್ಞೆಗಳು ಒಂದೇ ರೀತಿ ಇವೆ.

> **Aspire ಬಳಕೆದಾರರಿಗಾಗಿ ಸೂಚನೆ:** ಮೈಕ್ರೋಸಾಫ್ಟ್ ಈಗ ಉತ್ಪನ್ನವನ್ನು ಸರಳವಾಗಿ **Aspire** (ಹಿಂದಿನ ".NET Aspire") ಎಂದು ಕರೆಯುತ್ತದೆ. azd ರಲ್ಲಿ Aspire ಬೆಂಬಲದಲ್ಲಿ ಯಾವುದೇ ಬದಲಾವಣೆ ಇಲ್ಲ—ಕೆवल ಹೆಸರು ನವೀಕರಿಸಲಾಗಿದೆ.

ನೀವು ಹಿಂದೆ azd ಬಳಸಿದ್ದರೆ, AI ಬೆಂಬಲವು ಒಂದು ಸಹಜ ವಿಸ್ತರಣೆ—ವೆಂಪತ್ಯಾದ ಸOxA ಮಾಡಲಾರದ ಸಾಧನ ಅಥವಾ ಏಕ.Advanced ಪಾಠವಲ್ಲ. ನೀವು ಹೊಸದಾಗಿ ಹೊಳೆಹೊಗೆದರೆ, ನೀವು ಎಲ್ಲಕ್ಕೂ ಪರಿಣತಿಯಿಂದ ಕೆಲಸ ಮಾಡುವ ಒಂದು ಕಾರ್ಯಪ್ರವಾಹ ಕಲಿತೀರಿ.

## 🚀 ಅಜೂರ್ ಡೆವಲಪರ್ CLI (azd) ಎಂದರೆ ಏನು?

**ಅಜೂರ್ ಡೆವಲಪರ್ CLI (azd)** ಎಂಬುದು ಬಯಲಿನ-ಅನುಕೂಲಕರ ಕಮಾಂಡ್-ಲೈನ್ ಸಾಧನವಾಗಿದ್ದು, ಅಜೂರ್ ಗೆ ಅಪ್ಲಿಕೇಶನ್ಗಳನ್ನು ಸುಲಭವಾಗಿ ನಿಯೋಜಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ. ಅನೇಕ ಅಜೂರ್ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಕೈಯಿಂದ ಸೃಷ್ಟಿಸುವ ಬದಲು, ನೀವು ಒಂದೊಂದು ಆಜ್ಞೆಯಿಂದ ಸಂಪೂರ್ಣ ಅಪ್ಲಿಕೇಶನ್ಗಳನ್ನು ನಿಯೋಜಿಸಬಹುದು.

### `azd up` ಮಂತ್ರ

```bash
# ಈ ಏಕೈಕคำನ್ಡು ಎಲ್ಲಾ ಕೆಲಸ ಮಾಡುತ್ತದೆ:
# ✅ ಎಲ್ಲಾ ಅಜೂರ್ ಸಂಪನ್ಮೂಲಗಳನ್ನು ರಚಿಸುತ್ತದೆ
# ✅ ನೆಟ್ವರ್ಕಿಂಗ್ ಮತ್ತು ಭದ್ರತೆಯನ್ನು ಸಂರಚಿಸುತ್ತದೆ
# ✅ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಕೋಡ್ ನಿರ್ಮಿಸುತ್ತದೆ
# ✅ ಅಜೂರ್‌ಗೆ ನಿಯೋಜಿಸುತ್ತದೆ
# ✅ ನಿಮಗೆ ಕೆಲಸ ಮಾಡುವ URL ಅನ್ನು ನೀಡುತ್ತದೆ
azd up
```

**ಅಷ್ಟು!** ಅಜೂರ್ ಪೋರ್ಟಲ್ ಕ್ಲಿಕ್ ಮಾಡಬೇಕಿಲ್ಲ, ಜಟಿಲ ARM ಟೆಂಪ್ಲೇಟುಗಳನ್ನು ಮೊದಲಿಗೆ ಕಲಿಯಬೇಕಾಗಿಲ್ಲ, ಕೈಚಾಲಿತ ಸಂರಚನೆ ಬೇಡ—ಕೇವಲ ಫಂಕ್ಷನಲ್ ಅಪ್ಲಿಕೇಶನ್ಗಳು ಅಜೂರ್ ನಲ್ಲಿ.

## ❓ ಅಜೂರ್ ಡೆವಲಪರ್ CLI ಮತ್ತು ಅಜೂರ್ CLI: ವ್ಯತ್ಯಾಸವೇನು?

ಇದು ಹೊಸಬರಿಗೆ ಸಾಮಾನ್ಯವಾಗಿ ಕೇಳುವ ಪ್ರಶ್ನೆ. ಸರಳ ಉತ್ತರ ಇಲ್ಲಿದೆ:

| ವೈಶಿಷ್ಟ್ಯ | **ಅಜೂರ್ CLI (`az`)** | **ಅಜೂರ್ ಡೆವಲಪರ್ CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **ಉದ್ದೇಶ** | ವೈಯಕ್ತಿಕ ಅಜೂರ್ ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿರ್ವಹಿಸುವುದು | ಸಂಪೂರ್ಣ ಅಪ್ಲಿಕೇಶನ್ಗಳನ್ನು ನಿಯೋಜಿಸುವುದು |
| **ಮನೋಭಾವ** | ಮೂಲಸೌಕರ್ಯ-ಕೇಂದ್ರೀಕೃತ | ಅಪ್ಲಿಕೇಶನ್-ಕೇಂದ್ರೀಕೃತ |
| **ಉದಾಹರಣೆ** | `az webapp create --name myapp...` | `azd up` |
| **ಕಲಿಕೆ ತಿರುವು** | ಅಜೂರ್ ಸೇವೆಗಳನ್ನು ತಿಳಿದುಕೊಳ್ಳಬೇಕು | ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಮಾತ್ರ ತಿಳಿದುಕೊಳ್ಳಬೇಕು |
| **ಅತ್ಯುತ್ತಮ** | ಡೆವ್ಓಪ್ಸ್, ಮೂಲಸೌಕರ್ಯ | ಡೆವಲಪರ್ಸ್, ಪ್ರೋಟೋಟೈಪಿಂಗ್ |

### ಸರಳ ಉಪಮೆ

- **ಅಜೂರ್ CLI** ಎಲ್ಲಾ ಮನೆ ನಿರ್ಮಾಣ ಸಾಧನಗಳನ್ನು ಹೊಂದಿರುವಂತೆ—ಹ್ಯಾಮರ್‌ಗಳು, ಕತ್ತರಿ, ನೇಲ-ಕುಶಿಲ. ನೀವು ಏನನ್ನಾದರೂ ನಿರ್ಮಿಸಬಹುದು, ಆದರೆ ನೌಕರಿಕೆಯನ್ನು ತಿಳಿದುಕೊಳ್ಳಬೇಕು.
- **ಅಜೂರ್ ಡೆವಲಪರ್ CLI** ಒಬ್ಬ ಗೃಹನಿರ್ಮಾಪಕನನ್ನು ನೇಮಿಸುವಂತೆ—ನೀವು ಬೇಕಾದುದನ್ನು ವಿವರಿಸುತ್ತೀರಿ, ಅವರು ಕಟ್ಟಡವನ್ನು ನಿರ್ವಹಿಸುತ್ತಾರೆ.

### ಅವುಗಳು ಒಟ್ಟಿಗೆ ಕೆಲಸ ಮಾಡುತ್ತವೆ!

ಮೈಕ್ರೋಸಾಫ್ಟ್ ಫೌಂಡರಿ ಡಿಸ್‌ಕಾರ್ಡ್ ಸಮುದಾಯದ ಒಳನೋಟಗಳ ಆಧಾರದ ಮೇಲೆ, **45% ಡೆವಲಪರ್‌ಗಳು AI ಕೆಲಸಗಳಿಗಾಗಿ AZD ಬಳಸಲು ಬಯಸುತ್ತಾರೆ** ಆದರೆ ಈ ಸಮಸ್ಯೆಗಳನ್ನು ಎದುರಿಸುತ್ತಾರೆ:
- ಸಂಕೀರ್ಣ ಬಹು-ಸೇವಾ AI ವಾಸ್ತುಶಿಲ್ಪಗಳು
- ಉತ್ಪಾದನಾ AI ನಿಯೋಜನೆ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು  
- ಅಜೂರ್ AI ಸೇವಾ ಏಕೀಕರಣ ಮತ್ತು ಸಂರಚನೆ
- AI ಕೆಲಸಗಳಗಾಗಿ ವೆಚ್ಚದ ಆಪ್ಟಿಮೈಸು ಮಾಡುವುದು
- AI-ನಿರ್ಧಿಷ್ಟ ನಿಯೋಜನೆ ಸಮಸ್ಯೆಗಳ નિರાકರಣೆ

- **AZD ಮೂಲಭೂತಗಳನ್ನು ಮಾಸ್ಟರ್ ಮಾಡಿ**: ಕೋರಿನ概念ಗಳು, ಸ್ಥಾಪನೆ, ಮತ್ತು ಸಂರಚನೆ
- **AI ಅನ್ವಯಗಳನ್ನು ನಿಯೋಜಿಸಿ**: ಮೈಕ್ರೋಸಾಫ್ಟ್ ಫೌಂಡರಿ ಸೇವೆಗಳಿಗೆ AZD ಬಳಸಿ

- **ನಿಯೋಜನೆಗಳ ಸಮಸ್ಯೆಗಳನ್ನು ನಿವಾರಣೆ ಮಾಡಿ**: ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳನ್ನು ಪರಿಹರಿಸಿ ಮತ್ತು ದೋಷಗಳನ್ನು ನಿಖರಗೊಳಿಸಿ
- **ಉತ್ಪಾದನೆಗೆ ಆಪ್ಟಿಮೈಸು ಮಾಡಿ**: ಭದ್ರತೆ, ವಿವರಣೆ, ಮೇಲ್ವಿಚಾರಣೆ, ಮತ್ತು ವೆಚ್ಚ ನಿರ್ವಹಣೆ

- **ಮೂಲಭೂತ ಟರ್ಮಿನಲ್ ಪರಿಚಯ**: ನೀವು ತಜ್ಞರಾಗಿರಬೇಕಾಗಿಲ್ಲ, ಆದರೆ `git clone`, `azd auth login`, ಮತ್ತು `azd up` ನಂತಹ ಆಜ್ಞೆಗಳನ್ನು ಆಯಾ ಮಾಡಬಲ್ಲಿರಿ.

> **ಎಂಟರ್ಪ್ರೈಸ್ ಸಬ್ಸ್ಕ್ರಿಪ್ಶನ್ಗಳಲ್ಲಿ ಕೆಲಸ ಮಾಡುತ್ತಿರಾ?**
> ನಿಮ್ಮ ಅಜೂರ್ ಪರಿಸರವನ್ನು ಆಡಳಿತದವರು ನಿರ್ವಹಿಸುತ್ತಿದ್ದರೆ, ನೀವು ಬಳಕೆಯ ಸಬ್ಸ್ಕ್ರಿಪ್ಶನ್ ಅಥವಾ ಸಂಪನ್ಮೂಲ ಗುಂಪಿನಲ್ಲಿ ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿಯೋಜಿಸಬಹುದೇ ಎಂದು ಮುಂಚಿತವಾಗಿ ದೃಢಿಮಾಡಿ. ಇಲ್ಲದಿದ್ದರೆ, ಸ್ಯಾಂಡ್‌ಬಾಕ್ಸ್ ಸಬ್ಸ್ಕ್ರಿಪ್ಶನ್ ಅಥವಾ ಕಂಟ್ರಿಬ್ಯೂಟರ್ ಪ್ರವೇಶವನ್ನು ಆರಂಭಿಸುವ ಮುನ್ನ ಕೇಳಿ.

> **ಅಜೂರ್ ಹೊಸದಾಗಿ ಕಲಿಯುತ್ತಿರುವಿರಾ?**
> ನೀವು ನಿಮ್ಮ ಸ್ವಂತ ಅಜೂರ್ ಟ್ರಯಲ್ ಅಥವಾ ಪೇ-ಏಸ್-ಯು-ಗೋ ಸಬ್ಸ್ಕ್ರಿಪ್ಶನ್ ನಿಂದ ಪ್ರಾರಂಭಿಸಿ, https://aka.ms/azurefreetrial ನಲ್ಲಿ ಅನುಸರಿಸಿ, tenant-ಮಟ್ಟದ ಅನುಮतಿಗಳನ್ನು ಕಾಯದೆ ವ್ಯಾಯಾಮಗಳನ್ನು ಹಿಂದೆ ಮುಗಿಸಲು.

# ಪ್ರಶ್ನೆಗಳನ್ನು ಕೇಳಿ ಮತ್ತು ಮೂಲಗಳೊಂದಿಗೆ AI-ಚಾಲಿತ ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು ಪಡೆಯಿರಿ
# ಹುಡುಕಾಟ ಏಕೀಕರಣ ಕೆಲಸ ಮಾಡುತ್ತಿದೆ ಎಂಬುದನ್ನು ಪರಿಶೀಲಿಸಿ
azd monitor  # ಆಪ್ಲಿಕೇಶನ್ ಇನ್ಸೈಟ್ಸ್ ಟೆಲಿಮೆಟ್ರಿಯನ್ನು ತೋರಿಸುತ್ತದೆ ಎಂಬುದನ್ನು ಪರಿಶೀಲಿಸಿ
azd down --force --purge

**📊 ಸಮಯ ಹೂಡಿಕೆ:** 1-2 ಗಂಟೆಗಳು  
**📈 ಕೌಶಲ್ಯ ಮಟ್ಟ ನಂತರ:** ಉತ್ಪಾದನಾ ಸಿದ್ಧ AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿಯೋಜಿಸಿ ಮತ್ತು ಸಂರಚಿಸಬಹುದು  
**💰 ವೆಚ್ಚ ಅರಿವು:** $80-150/ತಿಂಗಳ ಅಭಿವೃದ್ಧಿ ವೆಚ್ಚಗಳು, $300-3500/ತಿಂಗಳ ಉತ್ಪಾದನಾ ವೆಚ್ಚಗಳು

#### 💰 AI ನಿಯೋಜನೆಗಾಗಿ ವೆಚ್ಚ ಪರಿಗಣನೆಗಳು

**ಅಭಿವೃದ್ಧಿ ಪರಿಸರ (ಅಂದಾಜು $80-150/ತಿಂಗಳು):**
- ಮೈಕ್ರೋಸಾಫ್ಟ್ ಫೌಂಡರಿ ಮಾದರಿಗಳು (ಪೇ-ಏಸ್-ಯು-ಗೋ): ಟೋಕನ್ ಬಳಕೆಯ ಆಧಾರದ ಮೇಲೆ $0-50/ತಿಂಗಳು
- AI ಹುಡುಕಿ (ಮೂಲಭೂತ ಹಂತ): $75/ತಿಂಗಳು
- ಕಂಟೈನರ್ ಅಪ್ಲಿಕೇಶನ್ಗಳು (ಪ್ರಮಾಣಿತ ಬಳಕೆ): $0-20/ತಿಂಗಳು
- ಸಂಗ್ರಹಣೆ (ಸಾಮಾನ್ಯ): $1-5/ತಿಂಗಳು

**ಉತ್ಪಾದನಾ ಪರಿಸರ (ಅಂದಾಜು $300-3,500+/ತಿಂಗಳು):**
- ಮೈಕ್ರೋಸಾಫ್ಟ್ ಫೌಂಡರಿ ಮಾದರಿಗಳು (ನಿರಂತರ ಪ್ರದರ್ಶನನಿಗಾಗಿ PTU): $3,000+/ತಿಂಗಳು ಅಥವಾ ಹೆಚ್ಚಿನ ಪ್ರಮಾಣದೊಂದಿಗೆ ಪೇ-ಏಸ್-ಯು-ಗೋ
- AI ಹುಡುಕಿ (ಸ್ಟ್ಯಾಂಡರ್ಡ್ ಹಂತ): $250/ತಿಂಗಳು
- ಕಂಟೈನರ್ ಅಪ್ಲಿಕೇಶನ್ಗಳು (ಪ್ರತ್ಯೇಕ): $50-100/ತಿಂಗಳು
- ಅಪ್ಲಿಕೇಶನ್ ಇನ್‌ಸೈಟ್ಸ್: $5-50/ತಿಂಗಳು
- ಸಂಗ್ರಹಣೆ (ಪ್ರೀಮಿಯಂ): $10-50/ತಿಂಗಳು

**💡 ವೆಚ್ಚ ಆಪ್ಟಿಮೈಸೇಶನ್ ಸಲಹೆಗಳು:**
- ಕಲಿಕೆಯಿಗಾಗಿ **ಫ್ರೀ ಟಿಯರ್** ಮೈಕ್ರೋಸಾಫ್ಟ್ ಫೌಂಡರಿ ಮಾದರಿಗಳನ್ನು ಬಳಸಿ (ಅಜೂರ್ ಓಪನ್AI 50,000 ಟೋಕನ್/ತಿಂಗಳ ಒಳಗೊಂಡಿದೆ)
- ಕ್ರಿಯಾಶೀಲವಾಗಿ ಅಭಿವೃದ್ಧಿ ಮಾಡುವಾಗ ಮಾತ್ರ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಕ್ಕೆ `azd down` ಆಜ್ಞೆಯನ್ನು ಓಡಿಸಿ
- ಬಳಕೆಯ ಆಧಾರಿತ ಬಿಲ್ಲಿಂಗ್ ಮೂಲಕ ಪ್ರಾರಂಭಿಸಿ, ಉತ್ಪಾದನೆಗಾಗಿ ಮಾತ್ರ PTU ಗೆ ಕ್ರಮವರ್ಧಿಸಿ
- ನಿಯೋಜನೆಕ್ಕೂ ಮುನ್ನ ವೆಚ್ಚಗಳನ್ನು ಅಂದಾಜಿಸಲು `azd provision --preview` ಬಳಸಿ
- ಸ್ವಯಂ-ಸ್ಕೇಲಿಂಗ್ ಇನೇಬಲ್ ಮಾಡಿ: ನಿಜವಾದ ಬಳಕೆಗೆ ಮಾತ್ರ ಪಾವತಿಸಿ

**ವೆಚ್ಚ ಮೇಲ್ವಿಚಾರಣೆ:**
```bash

azd provision --preview

# ಆಸುರ್ ಪೋರ್ಟಲ್‌ನಲ್ಲಿ ವಾಸ್ತಿ ವೆಚ್ಚಗಳನ್ನು ಮುನ್ನೋಟವಿರಹಿಸಿ
az consumption budget list --resource-group <your-rg>
```

### ಸಂವಹನಕಾರಿ ಕಾರ್ಯಾಗಾರ ಸಾಮಗ್ರಿಗಳು
**ಬ್ರೌಸರ್ ಆಧಾರಿತ ಸಾಧನಗಳು ಮತ್ತು ಮಾರ್ಗದರ್ಶಿತ ವ್ಯಾಯಾಮಗಳೊಂದಿಗೆ ಸಂಪೂರ್ಣ ಕೈಗೆಟ್ ಕಲಿಕೆ**

ನಮ್ಮ ಕಾರ್ಯಾಗಾರ ಸಾಮಗ್ರಿಗಳು ಮೇಲಿನ ಅಧ್ಯಾಯ-ಆಧಾರಿತ ಪಠ್ಯಕ್ರಮವನ್ನು ಪೂರ್ಣಗೊಳಿಸುವ ರಚಿತ, ಸಂವಹನಕಾರಿ ಕಲಿಕೆಯ ಅನುಭವವನ್ನು ಒದಗಿಸುತ್ತವೆ. ಕಾರ್ಯಾಗಾರವು ಸ್ವಯಂ-ಮಟ್ಟದ ಕಲಿಕೆ ಮತ್ತು ಉಪನ್ಯಾಸಕರ ನೇತೃತ್ವದ ಸೆಷನ್‌ಗಳಿಗೆ ವಿನ್ಯಾಸಗೊಳಿಸಲಾಗಿದೆ.

#### 🛠️ ಕಾರ್ಯಾಗಾರದ ವೈಶಿಷ್ಟ್ಯಗಳು

- **ರಚಿತ ಕಲಿಕೆ ಮಾರ್ಗ**: 8 ಘಟಕಗಳ ಮಾರ್ಗದರ್ಶಿತ ವ್ಯಾಯಾಮಗಳು (ಒಟ್ಟಿಗೆ 3-4 ಗಂಟೆಗಳು)

- **ಟೆಂಪ್ಲೇಟು-ಆಧಾರಿತ ನಿಯೋಜನೆಗಳು** - ಸಾಮಾನ್ಯ ಅಪ್ಲಿಕೇಶನ್ ಮಾದರಿಗಳು үчүн ಪೂರ್ವ ನಿರ್ಮಿತ ಟೆಂಪ್ಲೇಟುಗಳನ್ನು ಬಳಸಿ
- **ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಟ್ಚರ್ ಆಗ್ ಕೋಡ್** - Bicep ಅಥವಾ Terraform ಬಳಸಿ Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿರ್ವಹಿಸಿ  
- **ಒಗ್ಗೂಡಿಸಿದ ವರ್ಕ್‌ಫ್ಲೋಗಳು** - ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಸೌಕರ್ಯಪೂರ್ವಕವಾಗಿ ಪ್ರಾವೀಣ್ಯಗೊಳಿಸಿ, ನಿಯೋಜಿಸಿ ಮತ್ತು ಮೇಲ್ವಿಚಾರಣೆ ಮಾಡಿ
- **ಡೆವಲಪರ್-ಸ್ನೇಹಿ** - ಡೆವಲಪರ್ ಉತ್ಪಾದಕತೆ ಮತ್ತು ಅನುಭವಕ್ಕಾಗಿ ಆಪ್ಟಿಮೈಸ್ ಆಗಿದೆ

### **AZD + Microsoft Foundry: AI ನಿಯೋಜನೆಗಳಿಗೆ ಪರಿಪೂರ್ಣ**

**AI ಪರಿಹಾರಗಳಿಗಾಗಿ ಏಕೆ AZD?** AZD AI ಡೆವಲಪರ್‌ಗಳು ಎದುರಿಸುವ ಪ್ರಮುಖ ಸವಾಲುಗಳನ್ನು ಪರಿಹರಿಸುತ್ತದೆ:

- **AI-ತಯಾರ ಟೆಂಪ್ಲೇಟುಗಳು** - Microsoft Foundry ಮಾದರಿಗಳು, Azure AI ಸೇವೆಗಳು ಮತ್ತು ML ಕಾರ್ಯಭಾರಗಳಿಗಾಗಿ ಪೂರ್ವ-ಸಂರಚಿತ ಟೆಂಪ್ಲೇಟುಗಳು
- **ಭದ್ರತಾ AI ನಿಯೋಜನೆಗಳು** - AI ಸೇವೆಗಳು, API ಕೀಗಳು ಮತ್ತು ಮಾದರಿ ಅಂತಿಮ ಬಿಂದುಗಳಿಗಾಗಿ ನವೀನ ಭದ್ರತಾ ಮಾದರಿಗಳು  
- **ಉತ್ಪಾದನಾ AI ಮಾದರಿಗಳು** - ಅಳವಡಿಕೆ ಮಾಡಲು ಸೂಕ್ತವಾದ, ವೆಚ್ಚ-ಪ್ರಭಾವಿ AI ಅಪ್ಲಿಕೇಶನ್ ನಿಯೋಜನೆಗಳನ್ನು ಅನುಸರಿಸುವ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು
- **ಅಂತ್ಯ-ದಿಂದ-ಅಂತ್ಯ AI ವರ್ಕ್ಫ್ಲೋಗಳು** - ಮಾದರಿ ಅಭಿವೃದ್ಧಿಯಿಂದ ಉತ್ಪಾದನಾ ನಿಯೋಜನೆಗೆ ನಿಯಮಿತ ಮೇಲ್ವಿಚಾರಣೆಯೊಂದಿಗೆ
- **ವೆಚ್ಚದ ಆಪ್ಟಿಮೈಸೇಶನ್** - AI ಕಾರ್ಯಭಾರಗಳಿಗಾಗಿ ನುಸುಳಿದ ಸಂಪನ್ಮೂಲ ಹಂಚಿಕೆ ಮತ್ತು ಅಳವಡಿಕೆ ತಂತ್ರಗಳು
- **Microsoft Foundry ಏಕೀಕರಣ** - Microsoft Foundry ಮಾದರಿ ಕ್ಯಾಟಲಾಗ್ ಮತ್ತು ಅಂತಿಮ ಬಿಂದುಗಳಿಗೆ ನಿಹಿತ ಸಂಪರ್ಕ

## 🔧 ದ್ರುತ ದೋಷ ಪರಿಹಾರ ಮಾರ್ಗದರ್ಶಿ

**ಶುರುವಾಗುವವರಿಗೆ ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳಾಗುವ ಮತ್ತು ತಕ್ಷಣದ ಪರಿಹಾರಗಳು:**

<details>
<summary><strong>❌ "azd: ಕಮಾಂಡ್ ಕಂಡುಬರಲಿಲ್ಲ"</strong></summary>

```bash
# ಮೊದಲು AZD ಅನ್ನು ಸ್ಥಾಪಿಸಿ
# ವಿಂಡೋಸ್ (ಪವರ್‌ಶೆಲ್):
winget install microsoft.azd

# ಮ್ಯಾಕ್‌ಓಎಸ್:
brew tap azure/azd && brew install azd

# ಲಿನಕ್ಸ್:
curl -fsSL https://aka.ms/install-azd.sh | bash

# ಸ್ಥಾಪನೆಯನ್ನು ಪರಿಶೀಲಿಸಿ
azd version
```
</details>

<details>
<summary><strong>❌ "ಯಾವುದೇ ಸಬ್ಸ್ಕ್ರಿಪ್ಷನ್ ಕಂಡುಬಂದಿಲ್ಲ" ಅಥವಾ "ಸಬ್ಸ್ಕ್ರಿಪ್ಷನ್ ಸಿಟ್ ಆಗಿಲ್ಲ"</strong></summary>

```bash
# ಲಭ್ಯವಿರುವ ಚಂದಾಗಳನ್ನು ಪಟ್ಟಿ ಮಾಡಿ
az account list --output table

# ಡೀಫಾಲ್ಟ್ ಚಂದಾ ಹೊಂದಿಸಿ
az account set --subscription "<subscription-id-or-name>"

# AZD ಪರಿಸರಕ್ಕಾಗಿ ಹೊಂದಿಸಿ
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# ಪರಿಶೀಲಿಸಿ
az account show
```
</details>

<details>
<summary><strong>❌ "ಅಪರ್ಯಾಪ್ತಕೀಟ್ಟು" ಅಥವಾ "ಕೀಟ್ಟು ಮೀರಿದೆ"</strong></summary>

```bash
# ವಿಭಿನ್ನ ಆಜ್ಯೂರ್ ಪ್ರದೇಶವನ್ನು ಪ್ರಯತ್ನಿಸಿ
azd env set AZURE_LOCATION "westus2"
azd up

# ಅಥವಾ ಅಭಿವೃದ್ಧಿಯಲ್ಲಿ ಸಣ್ಣ SKUs ಅನ್ನು ಬಳಸಿಕೊಳ್ಳಿ
# infra/main.parameters.jsonನ್ನು ಸಂಪಾದಿಸಿ:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>

<details>
<summary><strong>❌ "azd up" ಮಧ್ಯಂತರದಲ್ಲಿ ವಿಫಲವಾಗಿದೆ</strong></summary>

```bash
# ಆಯ್ಕೆ 1: ಸ್ವಚ್ಛಗೊಳಿಸಿ ಮತ್ತು ಮರುಪ್ರಯತ್ನಿಸಿ
azd down --force --purge
azd up

# ಆಯ್ಕೆ 2: ಕೇವಲ ಮೂಲಸೌಕರ್ಯವನ್ನು ಸರಿಪಡಿಸಿ
azd provision

# ಆಯ್ಕೆ 3: ವಿವರವಾದ ಸ್ಥಿತಿಯನ್ನು ಪರಿಶೀಲಿಸಿ
azd show

# ಆಯ್ಕೆ 4: ಅಝೂರ್ ಮಾನಿಟರ್‌ನಲ್ಲಿ ಲಾಗ್‌ಗಳನ್ನು ಪರಿಶೀಲಿಸಿ
azd monitor --logs
```
</details>

<details>
<summary><strong>❌ "ಪ್ರಾಮಾಣೀಕರಣ ವಿಫಲವಾಗಿದೆ" ಅಥವಾ "ಟೋಕನ್ ಅವಧಿ ಮುಗಿದಿದೆ"</strong></summary>

```bash
# AZDಗಾಗಿ ಮತ್ತೊಮ್ಮೆ ಪ್ರಮಾಣೀಕರಿಸಿ
azd auth logout
azd auth login

# ಐಚ್ಛಿಕ: ನೀವು az ಕಮಾಂಡ್‌ಗಳನ್ನು ಬಳಸುತ್ತಿದ್ದರೆ Azure CLI ಅನ್ನು ಕೂಡ ರಿಫ್ರೆಶ್ ಮಾಡಿ
az logout
az login

# ಪ್ರಮಾಣೀಕರಣವನ್ನು ಪರಿಶೀಲಿಸಿ
az account show
```
</details>

<details>
<summary><strong>❌ "ಸಂಪನ್ಮೂಲವು ಈಗಾಗಲೇ ಇದೆ" ಅಥವಾ ಹೆಸರಿನಲ್ಲಿ ಸಂಧರ್ಭಗಳು</strong></summary>

```bash
# AZD ವಿಶಿಷ್ಟವಾದ ಹೆಸರುಗಳನ್ನು ರಚಿಸುತ್ತದೆ, ಆದರೆ ಸಂಘರ್ಷವಿದ್ದರೆ:
azd down --force --purge

# ನಂತರ ಹೊಸ ಪರಿಸರದೊಂದಿಗೆ ಮರುಪ್ರಯತ್ನಿಸಿ
azd env new dev-v2
azd up
```
</details>

<details>
<summary><strong>❌ ಟೆಂಪ್ಲೇಟು ನಿಯೋಜನೆ ಹೆಚ್ಚು ಸಮಯ ತೆಗೆದುಕೊಳ್ಳುತ್ತಿದೆ</strong></summary>

**ಸಾಮಾನ್ಯ ನಿರೀಕ್ಷಣಾ ಸಮಯಗಳು:**
- ಸರಳ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್: 5-10 ನಿಮಿಷಗಳು
- ಡೇಟಾಬೇಸ್ ಇರುವ ಅಪ್ಲಿಕೇಶನ್: 10-15 ನಿಮಿಷಗಳು
- AI ಅಪ್ಲಿಕೇಶನುಗಳು: 15-25 ನಿಮಿಷಗಳು (OpenAI ಪ್ರೊವಿಷನಿಂಗ್ ನಿಧಾನ)

```bash
# ಪ್ರಗತಿಯ ಪರಿಶೀಲನೆ ಮಾಡಿ
azd show

# 30 ನಿಮಿಷಕ್ಕೂ ಹೆಚ್ಚು ತಡೆಗಟ್ಟಿದರೆ, ಅಜೂರ್ ಪೋರ್ಟಲ್ ಪರಿಶೀಲಿಸಿ:
azd monitor --overview
# ವಿಫಲಗೊಂಡ ನಿಯೋಜನೆಗಳನ್ನು ನೋಡಿರಿ
```
</details>

<details>
<summary><strong>❌ "ಅನುಮತಿ ನಿರಾಕರಿಸಲಾಗಿದೆ" ಅಥವಾ "ನಿಷಿದ್ಧ"</strong></summary>

```bash
# ನಿಮ್ಮ ಅಜೂರು ಪಾತ್ರವನ್ನು ಪರಿಶೀಲಿಸಿ
az role assignment list --assignee $(az account show --query user.name -o tsv)

# ನಿಮಗೆ ಕನಿಷ್ಟ "ಕಾಂಟ್ರಿಬ್ಯೂಟರ್" ಪಾತ್ರ ಬೇಕಾಗಿರುತ್ತದೆ
# ನಿಮ್ಮ ಅಜೂರು ನಿರ್ವಾಹಕನನ್ನು ಕೇಳಿ ಅನುಮತಿಸಲು:
# - ಕಾಂಟ್ರಿಬ್ಯೂಟರ್ (ಸಂಪನ್ಮೂಲಗಳಿಗೆ)
# - ಬಳಕೆದಾರ ಪ್ರವೇಶ ನಿರ್ವಾಹಕ (ಪಾತ್ರ ನಿಯೋಜನೆಗಳಿಗೆ)
```
</details>

<details>
<summary><strong>❌ ನಿಯೋಜಿಸಲಾದ ಅಪ್ಲಿಕೇಶನ್ URL ಕಂಡುಬಾರದೆ</strong></summary>

```bash
# ಎಲ್ಲಾ ಸೇವಾ ಎಂಡ್‌ಪಾಯಿಂಟ್‌ಗಳನ್ನು ತೋರಿಸಿ
azd show

# ಅಥವಾ Azure ಪೋರ್ಟಲ್ ತೆರೆಯಿರಿ
azd monitor

# ನಿರ್ದಿಷ್ಟ ಸೇವೆಯನ್ನು ಪರಿಶೀಲಿಸಿ
azd env get-values
# *_URL ವ್ಯಾರೀಯಬಲ್ಗಳನ್ನು ಹುಡುಕಿ
```
</details>

ಪ್ರತಿ ಅಧ್ಯಾಯವನ್ನು ಪೂರ್ಣಗೊಳಿಸಿದ ನಂತರ, ನಿಮ್ಮ ಜ್ಞಾನವನ್ನು ಪರಿಶೀಲಿಸಿ:

2. **ಜ್ಞಾನ ಪರಿಶೀಲನೆ**: ನಿಮ್ಮ ಅಧ್ಯಾಯದ FAQ ವಿಭಾಗವನ್ನು ಪರಿಶೀಲಿಸಿ
