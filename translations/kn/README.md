## 🆕 azd ನಲ್ಲಿ ಇಂದಿನ ಹೊಸದು

Azure Developer CLI ಪರಂಪರೆಯಾತ್ಮಕ ವೆಬ್ ಅಪ್‌ಗಳು ಮತ್ತು API ಗಳನ್ನೆ ಮೀರಿಸಿ ಬೆಳೆಯುತ್ತದೆ. ಇಂದಿನ ದಿನದಲ್ಲೇ, azd ಯಾವದರಿಗೂ—ಎರೆಗಾದರೆ AI-ಸಂಪನ್ನ ಅಪ್ಲಿಕೇಶನ್‌ಗಳು ಮತ್ತು ಬುದ್ಧಿವಂತ ಯಂತ್ರಗಳು ಸೇರಿದಂತೆ—Azure ಗೆ ನಿಯೋಜಿಸುವ ಏಕಮಾತ್ರ ಸಾಧನವಾಗಿದೆ.

ಇದರಿಂದ ನಿಮಗೆ ಏನಾಗುತ್ತದೆ:

- **AI ಏಜೆಂಟ್ಗಳು ಈಗ ಪ್ರಥಮ-ಶ್ರೇಣಿಯ azd ಕೆಲಸಧೋರ್ಯಗಳಾಗಿವೆ.** ನೀವು ಈಗಾಗಲೇ ತಿಳಿದಿರುವ `azd init` → `azd up` ಕಾರ್ಯಪ್ರವಾಹದ ಮೂಲಕ AI ಏಜೆಂಟ್ ಪ್ರಾಜೆಕ್ಟ್‌ಗಳನ್ನು ಪ್ರಾರಂಭಿಸಿ, ನಿಯೋಜಿಸಿ ಮತ್ತು ನಿರ್ವಹಿಸಬಹುದು.
- **Microsoft Foundry ಏಕೀಕರಣ** ಮಾದರಿ ನಿಯೋಜನೆ, ಏಜೆಂಟ್ ಹೋಸ್ಟಿಂಗ್ ಮತ್ತು AI ಸೇವೆಗಳ ಸಂರಚನೆಯನ್ನು ನೇರವಾಗಿ azd ಟೆಂಪ್ಲೇಟ್ ಪರಿಸರಕ್ಕೆ ತರುತ್ತದೆ.
- **ಮೂಲ ಕಾರ್ಯಪ್ರವಾಹ ಬದಲಾಯಿಲ್ಲ.** ನೀವು todo ಆಪ್, ಮೈಕ್ರೋಸರ್ವೀಸ್ ಅಥವಾ ಬಹು-ಏಜೆಂಟ್ AI ಪರಿಹಾರವನ್ನು ನಿಯೋಜಿಸುತ್ತಿದ್ದೀರಾ, ಆದೇಶಗಳು ಒಂದೇ ರೀತಿ ಇರುತ್ತವೆ.

ನೀವು ಹಿಂದಿನಾಗಲೀ azd ಬಳಸಿರಿದ್ದರೆ, AI ಬೆಂಬಲವು ಸಹಜ ವಿಸ್ತರಣೆ — ಪ್ರತ್ಯೇಕ ಸಾಧನ ಅಥವಾ ಉನ್ನತ ಪಥವಲ್ಲ. ಹೊಸದಾಗಿ ಪ್ರಾರಂಭಿಸುತ್ತಿದ್ದರೆ, ನೀವು ಎಲ್ಲದರಿಗೂ ಕೆಲಸ ಮಾಡುವ ಒಬ್ಬ ಕಾರ್ಯಪ್ರವಾಹವನ್ನು ಕಲಿಯುತ್ತೀರಿ.

## 🚀 Azure Developer CLI (azd) ಅಂದರೆ ಏನು?

**Azure Developer CLI (azd)** ಬೇಸಿಗೆ ಅಭಿವಿಕಸಕರಿಗೆ ಅನುಕೂಲಕರವಾದ ಕಮಾಂಡ್-ಲೈನ್ ಸಾಧನ, ಇದು Azure ಗೆ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿಯೋಜಿಸಲು ಸರಳಗೊಳಿಸುತ್ತದೆ. ಅನೇಕ Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ಕೈಯಿಂದ ಸೃಷ್ಟಿಸಿ ಮತ್ತು ಸಂಪರ್ಕಿಸುವ ಬದಲು, ನೀವು ಒಂದೇ ಆಜ್ಞೆಯಿಂದ ಸಂಪೂರ್ಣ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿಯೋಜಿಸಬಹುದು.

### `azd up` ಯ ಮಾಯಾಜಾಲ

```bash
# ಈ ಒಕ್ಕೂಟ ಆದೇಶವು ಎಲ್ಲವನ್ನೂ ಮಾಡುತ್ತದೆ:
# ✅ ಎಲ್ಲಾ ಅಝುರ್ ಸಂಪನ್ಮೂಲಗಳನ್ನು ರಚಿಸುತ್ತದೆ
# ✅ ನೆಟ್ವರ್ಕಿಂಗ್ ಮತ್ತು ಭದ್ರತೆಯನ್ನು ಸಂರಚಿಸುತ್ತದೆ
# ✅ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಕೋಡ್ ನಿರ್ಮಿಸುತ್ತದೆ
# ✅ ಅಝುರ್‌ಗೆ ನಿಯೋಜಿಸುತ್ತದೆ
# ✅ ನಿಮಗೆ ಕಾರ್ಯನಿರ್ವಹಿಸುವ URL ಅನ್ನು ನೀಡುತ್ತದೆ
azd up
```

**ಇದೇ ಆಗಿದೆ!** ಯಾವುದೇ Azure Portal ಕ್ಲಿಕ್‌ಗಳು ಇಲ್ಲ, ಜಟಿಲ ARM ಟೆಂಪ್ಲೇಟುಗಳನ್ನು ಮೊದಲು ಕಲಿಯಬೇಕಾಗಿಲ್ಲ, ಯಾವುದೇ ಮನುಯಲ್ ಸಂರಚನೆ ಇಲ್ಲ - ಕೇವಲ Azure ಮೇಲೆ ಕಾರ್ಯನಿರ್ವಹಿಸುವ ಅಪ್ಲಿಕೇಶನ್‌ಗಳು.

## ❓ Azure Developer CLI ಮತ್ತು Azure CLI: ವ್ಯತ್ಯಾಸ ಏನು?

ಇದು ಹೊಸಬರು ಬಹುಮಟ್ಟಿಗೆ ಕೇಳುವ ಸಾಮಾನ್ಯ ಪ್ರಶ್ನೆ. ಸರಳ ಉತ್ತರ ಇಲ್ಲಿದೆ:

| ವೈಶಿಷ್ಟ್ಯ | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **ಉದ್ದೇಶ** | ವೈಯಕ್ತಿಕ Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿರ್ವಹಿಸುವುದು | ಸಂಪೂರ್ಣ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿಯೋಜಿಸುವುದು |
| **ಮನೋಭಾವ** | ಮೂಲಸೌಕರ್ಯ-ಕೇಂದ್ರೀಕೃತ | ಅಪ್ಲಿಕೇಶನ್-ಕೇಂದ್ರೀಕೃತ |
| **ಉದಾಹರಣೆ** | `az webapp create --name myapp...` | `azd up` |
| **ಕಲಿಕೆಯ ವಕ್ರತೆ** | Azure ಸೇವೆಗಳನ್ನ ತಿಳಿದಿರಬೇಕು | ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್‌ನ್ನು ಮಾತ್ರ ತಿಳಿದಿರಲಿ |
| **ಯಾವರಿಗೆ ಉತ್ತಮ** | DevOps, ಮೂಲಸೌಕರ್ಯ | ಡೆವಲಪರ್‌ಗಳು, ಪ್ರೋಟೋಟೈಪಿಂಗ್ |

### ಸರಳ ಉಪಮಾನ

- **Azure CLI** ಅನ್ನು ಮನೆಯೊಂದನ್ನು ಕಟ್ಟಲು ಎಲ್ಲಾ સાધನಗಳಿರೋದು ಹಾಗೆ ಊಹಿಸಬಹುದು - ಹೇಮರ್‌ಗಳು, ಸಾಲುಗಳು, ನೆಲಕಲ್ಲುಗಳು. ನೀವು ಏನು ಬೇಕಾದರೂ ಕಟ್ಟಬಹುದು, ಆದರೆ ನಿರ್ಮಾಣದ ಬಗ್ಗೆ ತಿಳಿದುಕೊಳ್ಳಬೇಕು.
- **Azure Developer CLI** ಅನ್ನು ಒಬ್ಬ ಗೃಹಸಂರಚಕಿಯನ್ನು ಭಾಡಿಗೆಕೆಟ್ಟಿದಂತೆ ಊಹಿಸಬಹುದು - ನೀವು ಏನು ಬೇಕೆಂದು ವಿವರಿಸಿರಿ, ಅವರು ಕಟ್ಟುವ ಕೆಲಸವನ್ನು ನಿಭಾಯಿಸುತ್ತಾರೆ.

### ಯಾವಾಗ ಯಾವದನ್ನು ಬಳಸಬೇಕು

| ಸನ್ನಿವೇಶ | ಇದನ್ನು ಬಳಸಿ |
|----------|----------|
| "I want to deploy my web app quickly" | `azd up` |
| "I need to create just a storage account" | `az storage account create` |
| "I'm building a full AI application" | `azd init --template azure-search-openai-demo` |
| "I need to debug a specific Azure resource" | `az resource show` |
| "I want production-ready deployment in minutes" | `azd up --environment production` |

### ಅವುಗಳು ಜೊತೆಯಾಗಿ ಕೆಲಸ ಮಾಡುತ್ತವೆ!

AZD ಹಿನ್ನೆಲೆಗಳಲ್ಲಿ Azure CLI ಅನ್ನು ಬಳಸುತ್ತದೆ. ನೀವು ಎರಡನ್ನೂ ಬಳಸಬಹುದು:
```bash
# ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು AZD ಜೊತೆಗೆ ನಿಯೋಜಿಸಿ
azd up

# ನಂತರ Azure CLI ಬಳಸಿ ನಿರ್ದಿಷ್ಟ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಸೂಕ್ಷ್ಮಗೊಳಿಸಿ
az webapp config set --name myapp --always-on true
```

## 🎯 3 ಹಂತಗಳಲ್ಲಿ ಪ್ರಾರಂಭಿಸುವುದು

ಪ್ರಾರಂಭಿಸಲು ಮೊದಲು, ನೀವು ನಿಯೋಜಿಸಲು ಬಯಸುವ ಟೆಂಪ್ಲೇಟಿಗೆ ನಿಮ್ಮ ಯಂತ್ರ ಸಿದ್ಧವಾಗಿದೆಯೇ ಎಂದು ಖಚಿತಪಡಿಸಿ:

**Windows:**
```powershell
.\validate-setup.ps1
```

**macOS / Linux:**
```bash
bash ./validate-setup.sh
```

ಏನಾದರೂ ಅಗತ್ಯತೆಯಿರುವ ಪರಿಶೀಲನೆ ವಿಫಲವಾದರೆ, ಮೊದಲು ಅದನ್ನು ಸರಿಪಡಿಸಿ ಮತ್ತು ನಂತರ ತ್ವರಿತ ಪ್ರಾರಂಭವನ್ನು ಮುಂದುವರೆಸಿರಿ.

### ಹಂತ 1: AZD ಅನ್ನು ಸ್ಥಾಪಿಸಿ (2 ನಿಮಿಷ)

**Windows:**
```powershell
winget install microsoft.azd
```

**macOS:**
```bash
brew tap azure/azd && brew install azd
```

**Linux:**
```bash
curl -fsSL https://aka.ms/install-azd.sh | bash
```

### ಹಂತ 2: AZD ಗಾಗಿ ಪ್ರಾಮಾಣೀಕರಣ

```bash
# ನೀವು ಈ ಕೋರ್ಸಿನಲ್ಲಿ ನೇರವಾಗಿ ಅಜ್ಯೂರ್ CLI ಕಮಾಂಡ್‌ಗಳನ್ನು ಬಳಸಲು ಯೋಜಿಸುತ್ತಿದ್ದರೆ ಐಚ್ಛಿಕ
az login

# AZD ಕಾರ್ಯಪ್ರವಾಹಗಳಿಗಾಗಿ ಅಗತ್ಯವಿದೆ
azd auth login
```

ನೀವು ಯಾವದು ಬೇಕೆಂದಿದ್ದೀರೋ ಅಂತ ಖಚಿತವಾಗದಿದ್ದರೆ, [Installation & Setup](docs/chapter-01-foundation/installation.md#authentication-setup) ನಲ್ಲಿ ಪೂರ್ಣ ಸೆಟ್‌ಅಪ್ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಅನುಸರಿಸಿ.

### ಹಂತ 3: ನಿಮ್ಮ ಮೊದಲ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ನಿಯೋಜಿಸಿ

```bash
# ಟೆಂಪ್ಲೇಟಿನಿಂದ ಪ್ರಾರಂಭಿಸಿ
azd init --template todo-nodejs-mongo

# ಅಜೂರ್‌ಗೆ ನಿಯೋಜಿಸಿ (ಎಲ್ಲವನ್ನೂ ರಚಿಸುತ್ತದೆ!)
azd up
```

**🎉 ಇದಾದೇಂತೂ!** ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಈಗ Azure ಮೇಲೆ ಲೈವ್ ಆಗಿದೆ.

### ಸ್ವಚ್ಛತೆ (ಮರೆತಬೇಡಿ!)

```bash
# Remove all resources when done experimenting
azd down --force --purge
```

Microsoft Foundry Discord ಸಮುದಾಯದ洞察ಗಳ ಆಧಾರದಲ್ಲಿ, **ವಿಕಸಕರಲ್ಲಿ 45%ರು AI ಕೆಲಸಭಾರಗಳಿಗೆ AZD ಬಳಸಲು ಇಚ್ಛಿಸುತ್ತಿದ್ದಾರೆ** ಆದರೆ ಕೆಳಗಿನ ಸವಾಲುಗಳನ್ನು ಎದುರಿಸುತ್ತಾರೆ:
- ಸಂಕೀರ್ಣ ಬಹು-ಸೇವಾ AI ವಾಸ್ತುಶಿಲ್ಪಗಳು
- ಉತ್ಪಾದನಾ AI ನಿಯೋಜನೆಯ ಉತ್ತಮ ಪ್ರಕ್ರಿಯೆಗಳು  
- Azure AI ಸೇವೆಗಳ ಏಕೀಕರಣ ಮತ್ತು ಸಂರಚನೆ
- AI ಕೆಲಸಭಾರಗಳಿಗಾಗಿ ವೆಚ್ಚದ оптимೈಜೆಶನ್
- AI-ನಿರ್ದಿಷ್ಟ ನಿಯೋಜನೆ ಸಮಸ್ಯೆಗಳನ್ನು ಪರಿಹರಿಸುವಲ್ಲಿ ತೊಂದರೆಗಳು

ಈ ಸಂರಚಿತ ಕೋರ್ಸ್ ಅನ್ನು ಪೂರ್ಣಗೊಳಿಸುವ ಮೂಲಕ, ನೀವು:
- **AZD ಮೂಲಭೂತಗಳನ್ನು ನೀಪುಣಗೊಳ್ಳಿ**: ಮೂಲ ತತ್ವಗಳು, ಸ್ಥಾಪನೆ ಮತ್ತು ಸಂರಚನೆ
- **AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿಯೋಜಿಸಿ**: AZD ಅನ್ನು Microsoft Foundry ಸೇವೆಗಳೊಂದಿಗೆ ಬಳಸಿ
- **Infrastructure as Code ಅನ್ನು ಜಾರಿಗೆ ತರುವುದು**: Bicep ಟೆಂಪ್ಲೇಟುಗಳೊಂದಿಗೆ Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿರ್ವಹಿಸಿ
- **ನಿಯೋಜನೆಗಳ ದೋಷನಿರ್ಣಯ**: ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳನ್ನು ಪರಿಹರಿಸಿ ಮತ್ತು ಡಿಬಗಿನ ಮಾಡಿ
- **ಉತ್ಪಾದನೆಗೆ ಸುಧಾರಿಸಿ**: ಭದ್ರತಾ, ಸ್ಕೇಲಿಂಗ್, ಮಾನಿಟರಿಂಗ್ ಮತ್ತು ವೆಚ್ಚ ನಿರ್ವಹಣೆ
- **ಬಹು-ಏಜೆಂಟ್ ಪರಿಹಾರಗಳ ನಿರ್ಮಾಣ**: ಸಂಕೀರ್ಣ AI ವಾಸ್ತುಶಿಲ್ಪಗಳನ್ನು ನಿಯೋಜಿಸಿ

## ಪ್ರಾರಂಭಿಸಲು ಮೊದಲು: ಖಾತೆಗಳು, ಪ್ರವೇಶ, ಮತ್ತು ಊಹೆಗಳು

ಅಧ್ಯಾಯ 1 ಪ್ರಾರಂಭಿಸುವ ಮುನ್ನ, ಕೆಳಗಿನವುಗಳು ನಿಮಗೆ ಸಿದ್ಧವಾಗಿರಬೇಕು. ಈ ಮಾರ್ಗದರ್ಶನದ ನಂತರದ ಸ್ಥಾಪನೆಯ ಹಂತಗಳು ಈ ಮೂಲಭೂತಗಳನ್ನು ಈಗಾಗಲೇ ಕೈಗೊಳ್ಳಲಾಗಿದೆ ಎಂದು فرضಿಸುತ್ತವೆ.
- **ಒಂದು Azure ಸಬ್ಸ್ಕ್ರಿಪ್ಶನ್**: You can use an existing subscription from work or your own account, or create a [ಉಚಿತ ಪ್ರಯೋಗ](https://aka.ms/azurefreetrial) to get started.
- **Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ರಚಿಸುವ ಅನುಮತಿ**: ಹೆಚ್ಚಿನ ವ್ಯಾಯಾಮಗಳಿಗೆ, ಗುರಿ ಸಬ್ಸ್ಕ್ರಿಪ್ಶನ್ ಅಥವಾ ರಿಸೋರ್ಸ್ ಗುಂಪಿನಲ್ಲಿ ಕನಿಷ್ಠ **Contributor** ಪ್ರವೇಶ ಇರುವುದಾಗಿ ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ. ಕೆಲವು ಅಧ್ಯಾಯಗಳು ರಿಸೋರ್ಸ್ ಗುಂಪುಗಳನ್ನು, managed identities, ಮತ್ತು RBAC ನಿಯೋಜನೆಗಳನ್ನು ರಚಿಸಲು ನಿಮ್ಮ ಬಳಿ ಸಾಮರ್ಥ್ಯವಿದೆ ಎಂದು ধরেಕೊಳ್ಳಬಹುದು.
- [**GitHub ಖಾತೆ**](https://github.com): ರೆಪೊವನ್ನು fork ಮಾಡುವುದು, ನಿಮ್ಮ ವೈಯಕ್ತಿಕ ಬದಲಾವಣೆಗಳನ್ನು ಟ್ರ್ಯಾಕ್ ಮಾಡುವುದು ಮತ್ತು ವರ್ಕ್‌ಶಾಪ್‌ಗಾಗಿ GitHub Codespaces ಬಳಸುವುದು ಇದರಿಂದ ಸವಲತ್ತು ಸಿಗುತ್ತದೆ.
- **ಟೆಂಪ್ಲೇಟ್_RUNTIME_ಅಗತ್ಯತೆಗಳು**: ಕೆಲವು ಟೆಂಪ್ಲೇಟ್ಗಳು ಸ್ಥಳೀಯ ಸಾಧನಗಳನ್ನು ಅಗತ್ಯವಿರುತ್ತವೆ, ಉದಾಹರಣೆಗೆ Node.js, Python, Java, ಅಥವಾ Docker. ಪ್ರಾರಂಭಿಸುವ ಮೊದಲು ಸೆಟ್‌ಅಪ್ ಮಾನ್ಯತಾಪತ್ರವನ್ನು ಚಾಲನೆ ಮಾಡಿ zodat you catch missing tools early.
- **ಮೂಲಭೂತ ಟರ್ಮಿನಲ್ ಪರಿಚಯ**: ನಿಮಗೆ ವೃತ್ತಿಪರ ಮಾಸ್ಟರಿ ಬೇಕಾಗಿಲ್ಲ, ಆದರೆ `git clone`, `azd auth login`, ಮತ್ತು `azd up` ಇಂತಹ ಕಮಾಂಡ್‌ಗಳನ್ನು ಓಡಿಸಲು ಆರಾಮವಾಗಿದೆ ಇರಬೇಕೆಂದುಕೊಳ್ಳಿ.

> **ಸಂಸ್ಥೆಯ ಸಬ್ಸ್ಕ್ರಿಪ್ಶನ್‌ನಲ್ಲಿ ಕೆಲಸ ಮಾಡುತ್ತಿದ್ದೀರಾ?**
> ನಿಮ್ಮ Azure ವಾತಾವರಣವು ನಿರ್ವಾಹಕರಿಂದ ನಿರ್ವಹಿಸಲ್ಪಟ್ಟಿದ್ದರೆ, ನೀವು ಉಪಯೋಗಿಸಲು ಯೋಜಿಸಿರುವ ಸಬ்ஸ್ಕ್ರಿಪ್ಷನ್ ಅಥವಾ ರಿಸೋರ್ಸ್ ಗುಂಪಿನಲ್ಲಿ ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿಯೋಜಿಸಲು ಸಾಧ್ಯವಿದೆಯೇ ಎಂದು ಮುಂಚಿತವಾಗಿ ದೃঢ়ಪಡಿಸಿಕೊಳ್ಳಿ. ಸಾಧ್ಯವಾಗದಿದ್ದರೆ, ಪ್ರಾರಂಭಿಸುವ ಮೊದಲು ಸ್ಯಾಂಡ್‌ಬಾಕ್ಸ್ ಸಬ್ಸ್ಕ್ರಿಪ್ಶನ್ ಅಥವಾ Contributor ಪ್ರವೇಶವನ್ನು ವಿನಂತಿಸಿ.

> **Azure ಹೊಸದೋ?**
> ಮುಕ್ತವಾಗಿಯೂ ಪ್ರಾರಂಭಿಸಲು ನಿಮ್ಮ ಸ್ವಂತ Azure ಟ್ರಯಲ್ ಅಥವಾ pay-as-you-go subscription ಅನ್ನು https://aka.ms/azurefreetrial ನಲ್ಲಿ ಪ್ರಾರಂಭಿಸಿ, zodat you can complete the exercises end to end without waiting on tenant-level approvals.

- ನಿಮ್ಮ ವೇದಿಕೆಯಲ್ಲಿ AZD ಅನ್ನು ಸ್ಥಾಪಿಸುವುದು

#### ಕಲಿಕೆಯ ಸಂಪನ್ಮಳುಗಳು
- **🎯 ಇಲ್ಲಿ ಪ್ರಾರಂಭಿಸಿ**: [Azure Developer CLI ಎಂದರೆ ಏನು?](#what-is-azure-developer-cli)
- **📖 ಸಿದ್ಧಾಂತ**: [AZD ಮೂಲಾಂಶಗಳು](docs/chapter-01-foundation/azd-basics.md) - ಕೋರ್ ತತ್ವಗಳು ಮತ್ತು ಪದಾವಳಿಗಳು
- **⚙️ ಸೆಟ್‌ಅಪ್**: [ಸ್ಥಾಪನೆ & ಸેટಪ್](docs/chapter-01-foundation/installation.md) - ವೇದಿಕಾ-ನಿರ್ದಿಷ್ಟ ಮಾರ್ಗದರ್ಶಿಗಳು
- **🛠️ ಹ್ಯಾಂಡ್ಸ್-ಆನ್**: [ನಿಮ್ಮ ಮೊದಲ ಪ್ರಾಜೆಕ್ಟ್](docs/chapter-01-foundation/first-project.md) - ಹಂತರಿದ್ದಂತೆ ತರಬೇತಿ
- **📋 ಕ್ವಿಕ್ ರೆಫರೆನ್ಸ್**: [ಕಮಾಂಡ್ ಚೀಟ್ ಶೀಟ್](resources/cheat-sheet.md)

#### ಪ್ರಾಯೋಗಿಕ ವ್ಯಾಯಾಮಗಳು
```bash
# ತ್ವರಿತ ಸ್ಥಾಪನೆ ಪರಿಶೀಲನೆ

# ನಿಮ್ಮ ಮೊದಲاپ್ಲಿಕೇಶನ್ ಅನ್ನು ನಿಯೋಜಿಸಿ
azd init --template todo-nodejs-mongo
azd up
```

**💡 ಅಧ್ಯಾಯ ಫಲಿತಾಂಶ**: AZD ಉಪಯೋಗಿಸಿ ಒಂದು ಸರಳ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು Azure ಗೆ ಯಶಸ್ವಿಯಾಗಿ ನಿಯೋಜಿಸಲಾಗುವುದು

**✅ ಯಶಸ್ಸಿನ ಮಾನ್ಯತೆ:**
```bash
# ಅಧ್ಯಾಯ 1 ಅನ್ನು ಪೂರ್ಣಗೊಳಿಸಿದ ನಂತರ, ನೀವು ಇವುಗಳನ್ನು ಮಾಡಬಲ್ಲಿರಿ:
azd version              # ಸ್ಥಾಪಿಸಲಾದ ಆವ.UNKNOWN_TOKEN?VERSION? ಯನ್ನು ತೋರಿಸುತ್ತದೆ
azd init --template todo-nodejs-mongo  # ಪ್ರಾಜೆಕ್ಟ್ ಅನ್ನು ಪ್ರಾರಂಭಿಸುತ್ತದೆ
azd up                  # Azure ಗೆ ನಿಯೋಜಿಸುತ್ತದೆ
azd show                # ಓಡುತ್ತಿರುವ ಅಪ್ಲಿಕೇಶನಿನ URL ಅನ್ನು ತೋರಿಸುತ್ತದೆ
# ಅಪ್ಲಿಕೇಶನ್ ಬ್ರೌಸರ್‌ನಲ್ಲಿ ತೆರೆಯುತ್ತದೆ ಮತ್ತು ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ
azd down --force --purge  # ಸಂಪನ್ಮೂಲಗಳನ್ನು ತೆರವುಗೊಳಿಸುತ್ತದೆ
```

**📊 ಸಮಯ ಬಂಡವಾಳ:** 30-45 ನಿಮಿಷಗಳು  
**📈 ವರ್ಗ ಕೌಶಲ್ಯ ನಂತರ:** ಸ್ವತಂತ್ರವಾಗಿ ಮೂಲ ಪರೀಕ್ಷೆಗಳ ಅಪ್ಲಿಕೇಶನ್ಗಳನ್ನು ನಿಯೋಜಿಸಬಹುದು  
**📈 ವರ್ಗ ಕೌಶಲ್ಯ ನಂತರ:** ಸ್ವತಂತ್ರವಾಗಿ ಮೂಲ ಪರೀಕ್ಷೆಗಳ ಅಪ್ಲಿಕೇಶನ್ಗಳನ್ನು ನಿಯೋಜಿಸಬಹುದು

```bash
# ಅಧ್ಯಾಯ 2 ನಂತರ, ನೀವು ಕೆಳಗಿನವುಗಳನ್ನು ಮಾಡಬಲ್ಲಿರಿ:
azd init --template azure-search-openai-demo
azd up
# AI ಚಾಟ್ ಇಂಟರ್ಫೇಸ್ ಅನ್ನು ಪರೀಕ್ಷಿಸಲು
```

#### 💰 AI ನಿಯೋಜನೆಗಳಿಗಾಗಿ ವೆಚ್ಚ ಪರಿಗಣನೆಗಳು

**ವಿಕಾಸ ಪರಿಸರ (ಅಂದಾಜು $80-150/ತಿಂಗಳಿಗೆ):**
- Microsoft Foundry Models (Pay-as-you-go): $0-50/month (based on token usage)
- AI Search (Basic tier): $75/month
- Container Apps (Consumption): $0-20/month
- Storage (Standard): $1-5/month

**ಉತ್ಪಾದನಾ ಪರಿಸರ (ಅಂದಾಜು $300-3,500+/ತಿಂಗಳಿಗೆ):**
- Microsoft Foundry Models (PTU for consistent performance): $3,000+/month OR Pay-as-go with high volume
- AI Search (Standard tier): $250/month
- Container Apps (Dedicated): $50-100/month
- Application Insights: $5-50/month
- Storage (Premium): $10-50/month

**💡 ವೆಚ್ಚ tốiಮೀಕರಣ ಸಲಹೆಗಳು:**
- ಕಲಿಕೆಯಿಗಾಗಿ Microsoft Foundry Models ನ **Free Tier** ಅನ್ನು ಬಳಸಿ (Azure OpenAI 50,000 tokens/ತಿಂಗಳಿಗೆ ಒಳಗೊಂಡಿದೆ)
- ಸಕ್ರಿಯವಾಗಿ ಅಭಿವೃದ್ಧಿ ಮಾಡದಾಗ `azd down` ಓಡಿಸಿ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಡೀ-ಅಲ್ಲಿೋಕೆಟ್ ಮಾಡಿ
- ಉತ್ಪಾದನೆಗೆ ಮಾತ್ರ PTU ಗೆ ಅಪ್‌ಗ್ರೇಡ್ ಮಾಡಿ; ಆರಂಭದಲ್ಲಿ consumption-ಆಧಾರಿತ ಬಿಲ್ಲಿಂಗ್ ಬಳಸಿ
- ನಿಯೋಜನೆ ಮುಂಚೆ ವೆಚ್ಚ ಅಂದಾಜು ಮಾಡಲು `azd provision --preview` ನ್ನು ರನ್ ಮಾಡಿ
- ಸ್ವಯಂ-ಸ್ಕೇಲಿಂಗ್ ಅನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿ: ಕೇವಲiven ಬಳಕೆಯಿಗಾಗಿ ಪಾವತಿಸಿ

## 📖 ಆಳವಾದ ಅಧ್ಯಯನ: AZD ಸಾಮರ್ಥ್ಯಗಳು

ಮೂಲಭೂತ ವಿಷಯಗಳ ಹೊರತಾಗಿ, AZD ಉತ್ಪಾದನಾ ನಿಯೋಜನೆಗಳಿಗೆ ಶಕ್ತಿಶಾಲಿ ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ:

- **ಟೆಂಪ್ಲೇಟ್ ಆಧಾರಿತ ನಿಯೋಜನೆಗಳು** - ಸಾಮಾನ್ಯ ಅಪ್ಲಿಕೇಶನ್ ಮಾದರಿಗಾಗಿ ಪೂರ್ವ-ನಿರ್ಮಿತ ಟೆಂಪ್ಲೇಟ್ಗಳ ಬಳಕೆ  
- **ಕೋಡ್ ರೂಪದಲ್ಲಿ ಮೂಲಸೌಕರ್ಯ** - Bicep ಅಥವಾ Terraform ಬಳಸಿ Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿರ್ವಹಣೆ  
- **ಒಗ್ಗಟ್ಟಾದ ವರ್ಕ್‌ಫ್ಲೋಗಳು** - ಅನಿವಾರ್ಯವಾಗಿ ನಿಯೋಜನೆ, ನಿಗಮಿಸುವಿಕೆ ಮತ್ತು ಮೇಲ್ವಿಚಾರಣೆ  
- **ಡೆವಲಪರ್ ಸ್ನೇಹಿ** - ಡೆವಲಪರ್ ಉತ್ಪಾದಕತೆ ಮತ್ತು ಅನುಭವಕ್ಕಾಗಿ ಸುಧಾರಣೆ

### **AZD + Microsoft Foundry: AI ನಿಯೋಜನೆಗಳಿಗೆ ಪರಿಪೂರ್ಣವಾಗಿದೆ**

**AI ಪರಿಹಾರಗಳಿಗೆ ಯಾಕೆ AZD?** AZD AI ಅಭಿವೃದ್ಧಿಪಡಿಸಿದವರು ಎದುರಿಸುವ ಪ್ರಮುಖ ಸವಾಲುಗಳನ್ನು ಪರಿಹರಿಸುತ್ತದೆ:

- **AI-ಸಿದ್ಧ ಟೆಂಪ್ಲೇಟುಗಳು** - Microsoft Foundry ಮಾದರಿಗಳು, ಜ್ಞಾನ ಸೇವೆಗಳು, ಮತ್ತು ML ಕಾರ್ಯಭಾರಗಳಿಗೆ ಪೂರ್ವ-ಸಂರಚಿತ ಟೆಂಪ್ಲೇಟುಗಳು  
- **ಭದ್ರತೆಯ AI ನಿಯೋಜನೆಗಳು** - AI ಸೇವೆಗಳು, API ಕೀಲಿಗಳು ಮತ್ತು ಮಾದರಿ ಅಂತಿಮ ಬಿಂದುಗಳಿಗೆ ನಿರ್ಮಿತ ಭದ್ರತಾ ಮಾದರಿಗಳು  
- **ಉತ್ಪಾದನಾ AI ಮಾದರಿಗಳು** - ವಿಸ್ತಾರಗೊಳ್ಳುವ, ವೆಚ್ಚ-ಪ್ರಭಾವಿ AI ಅಪ್ಲಿಕೇಶನ್ ನಿಯೋಜನೆಗಳ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು  
- **ಮುಗಿದಂತೆ AI ವರ್ಕ್‌ಫ್ಲೋಗಳು** - ಮಾದರಿ ಅಭಿವೃದ್ಧಿಯಿಂದ ಉತ್ಪಾದನಾ ನಿಯೋಜನೆ ಮತ್ತು ಸರಿಯಾದ ಮೇಲ್ವಿಚಾರಣೆಯವರೆಗೆ  
- **ವೆಚ್ಚ ಪರಿಷ್ಕರಣೆ** - ಬುದ್ಧಿವಂತ ಸಂಪನ್ಮೂಲ ವಿನ್ಯಾಸ ಮತ್ತು ವಿಸ್ತರಣಾ ತಂತ್ರಗಳು AI ಕಾರ್ಯಭಾರಗಳಿಗೆ  
- **Microsoft Foundry ಸಮ್ಮಿಲನ** - Microsoft Foundry ಮಾದರಿ ಕ್ಯಾಟಲಾಗ್ ಮತ್ತು ಅಂತಿಮ ಬಿಂದುಗಳ ನಡವಳಿಕೆ

### ಕಲಿಕೆಯ ಖಚಿತತೆ
ಪ್ರತಿ ಅಧ್ಯಾಯವನ್ನು ಪೂರ್ಣಗೊಳಿಸಿದ ನಂತರ ನಿಮ್ಮ ಜ್ಞಾನವನ್ನು ಪರಿಶೀಲಿಸಿ:
1. **ಪ್ರಾಯೋಗಿಕ ವ್ಯಾಯಾಮ**: ಅಧ್ಯಾಯದ ಅನುಭವಕ್ಕೆ ನೇರವಾಗಿ ನಿಯೋಜನೆ ಮಾಡಿ
2. **ಜ್ಞಾನ ಪರಿಶೀಲನೆ**: ನಿಮ್ಮ ಅಧ್ಯಾಯದ ಪ್ರಶ್ನೋತ್ತರ ವಿಭಾಗವನ್ನು ಪರಿಶೀಲಿಸಿ
3. **ಸಮುದಾಯ ಚರ್ಚೆ**: ನಿಮ್ಮ ಅನುಭವವನ್ನು ಏಝೂರ್ ಡಿಸ್ಕೋರ್ಡ್‌ನಲ್ಲಿ ಹಂಚಿಕೊಳ್ಳಿ
4. **ಮುಂದಿನ ಅಧ್ಯಾಯ**: ಮುಂದಿನ ಕಠಿಣನ್ನು ಮಟ್ಟವನ್ನು ಪ್ರಾರಂಭಿಸಿ

### ಕೋರ್ಸ್ ಪೂರ್ಣಗೊಳಿಸುವ ಲಾಭಗಳು
ಎಲ್ಲಾ ಅಧ್ಯಾಯಗಳನ್ನು ಪೂರ್ಣಗೊಳಿಸಿದಾಗ, ನೀವು ಹೊಂದಿರುವಿರಿ:
- **ಉತ್ಪಾದನಾ ಅನುಭವ**: ನೈಜ AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಏಝೂರ್‌ಗೆ ನಿಯೋಜಿಸಲಾಗಿದೆ
- **ವೃತ್ತಿಪರ ಕೌಶಲ್ಯಗಳು**: ಎಂಟರ್ಪ್ರೈಸ್ ಸಿದ್ಧ ನಿಯೋಜನೆ ಸಾಮರ್ಥ್ಯಗಳು  
- **ಸಮುದಾಯ ಗೌರವ**: ಏಝೂರ್ ಡೆವಲಪರ್ ಸಮುದಾಯದ ಸಕ್ರಿಯ ಸದಸ್ಯ
- **ವೃತ್ತಿ ಪ್ರಗತಿ**: ಬೇಡಿಕೆಯಿರುವ AZD ಮತ್ತು AI ನಿಯೋಜನಾ ಪರಿಣತಿ

### ಮೈಕ್ರೋಸಾಫ್ಟ್ ಫೌಂಡ್ರಿ ಡಿಸ್ಕೋರ್ಡ್‌ನಿಂದ ಸಮುದಾಯ洞察ಗಳು

**ನಮ್ಮ ಸಮುದಾಯದಲ್ಲಿ ಸೇರಿ:**
- ನಿಮ್ಮ AZD + AI ಅನುಭವಗಳನ್ನು ಹಂಚಿಕೊಳ್ಳಿ ಮತ್ತು ಸಹಾಯವನ್ನು ಪಡೆಯಿರಿ
- ಹೊಸ AI ಟೆಂಪ್ಲೇಟ್‌ಗಳ ಪೂರ್ವದೃಷ್ಟಿ ಪಡೆಯಿರಿ
- AI ನಿಯೋಜನೆ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳಿಗೆ ಕೊಡುಗೆ ನೀಡಿ
- ಮುಂದಿನ AI + AZD ವೈಶಿಷ್ಟ್ಯಗಳ ಅಭಿವೃದ್ಧಿಯನ್ನು ಪ್ರಭಾವಿತ ಮಾಡಿಕೊಳ್ಳಿ
