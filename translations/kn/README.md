## 🆕 azd ನಲ್ಲಿ ಇವತ್ತಿನ ನವೀಕರಣಗಳು

Azure Developer CLI ಈಗ ಪರಂಪರাগত ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳು ಮತ್ತು API ಗಳಿಗಿಂತ ಮುಂದೆ ಬೆಳೆಯಿದೆ. ಇಂದು, azd ಎಂದರೆ Azure ಗೆ ಯಾವುದೇ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ನಿಯೋಜಿಸಲು ಒಂದು ಏಕೈಕ ಸಾಧನ — ಇದರಲ್ಲಿ AI-ಚಾಲಿತ ಅಪ್ಲಿಕೇಶನ್‌ಗಳು ಮತ್ತು ಬುದ್ಧಿವಂತ ಎಜೆಂಟ್‌ಗಳು ಸಹ ಸೇರಿವೆ.

ಇದರಿಂದ ನಿಮಗೆ ಏನಾಗುತ್ತದೆ:

- **AI ಏಜೆಂಟ್ಗಳು ಈಗ ಪ್ರಥಮ-ಶ್ರೇಣಿಯ azd ಕಾರ್ಯಭಾರಗಳಾಗಿದೆ.** ನೀವು ಈಗಿರುವ `azd init` → `azd up` ವರ್ಕ್‌ಫ್ಲೋ ಬಳಸಿ AI ಏಜೆಂಟ್ ಪ್ರಾಜೆಕ್ಟ್‌ಗಳನ್ನು ಪ್ರಾರಂಭಿಸಲು, ನಿಯೋಜಿಸಲು ಮತ್ತು ನಿರ್ವಹಿಸಲು ಸಾಧ್ಯ.
- **Microsoft Foundry ಇಂಟégrೇಶನ್** ಮಾದರಿ ನಿಯೋಜನೆ, ಏಜೆಂಟ್ ಹೋಸ್ಟಿಂಗ್ ಮತ್ತು AI ಸೇವೆ ಸಂರಚನೆಯನ್ನು ನೇರವಾಗಿ azd ಟೆಂಪ್ಲೇಟ್ ಇಕೋಸಿಸ್ಟಂ içine ತರುತ್ತದೆ.
- **ಮೂಲ ವರ್ಕ್‌ಫ್ಲೋ ಬದಲಾಗಿಲ್ಲ.** ನೀವು todo ಅಪ್ಲಿಕೇಶನ್, ಮೈಕ್ರೋಸರ್ವಿಸ್ ಅಥವಾ ಬಹು-ಏಜೆಂಟ್ AI ಪರಿಹಾರ ನಿಯೋಜಿಸುತ್ತಿದ್ದರೂ, ಆಜ್ಞೆಗಳು ಒಂದೇ ರೀತಿಯನ್ನೇ ಇವೆ.

ನೀವು ಹಿಂದೆ azd ಬಳಸಿದ್ದರೆ, AI ಬೆಂಬಲವು ಪ್ರಾಕೃತಿಕ ವಿಸ್ತರಣೆ — ಯಾವುದೇ ಬೇರೆ ಸಾಧನ ಅಥವಾ ವಿಶೇಷ ಮಾರ್ಗವಲ್ಲ. ಹೊಸದಾಗಿ ಪ್ರಾರಂಭಿಸುತ್ತಿದ್ದರೆ, ಒಂದೇ ವರ್ಕ್‌ಫ್ಲೋ ಎಲ್ಲದಕ್ಕೂ ಕೆಲಸಮಾಡುತ್ತದೆ.

## 🚀 Azure Developer CLI (azd) ಎಂಬುದು ಏನು?

**Azure Developer CLI (azd)** ಒಂದು ಡೆವಲಪರ್-ಅನುಕೂಲವಾದ ಕಮಾಂಡ್-ಲೈನ್ ಸಾಧನವಾಗಿದ್ದು, Azure ಗೆ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿಯೋಜಿಸಲು ಸರಳವಾಗಿಸುತ್ತದೆ. ಅನೇಕ Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ಕೈಯಿಂದ ರಚಿಸುವ ಮತ್ತು ಸಂಪರ್ಕಿಸುವ ಬದಲಿ, ನೀವು ಒಂದು ಕಮಾಂಡ್‌ನಿಂದ ಸಂಪೂರ್ಣ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿಯೋಜಿಸಬಹುದು.

### `azd up` ಅವರ ಮಾಯಾಜಾಲ

```bash
# ಈ ಒಂದೇ ಆದೇಶ ಎಲ್ಲವನ್ನೂ ಮಾಡಲು ಸಾಧ್ಯ:
# ✅ ಎಲ್ಲಾ ಆಜೂರ್ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಸೃಷ್ಟಿಸುತ್ತದೆ
# ✅ ನೆಟ್ವರ್ಕಿಂಗ್ ಮತ್ತು ಭದ್ರತೆ ಕನ್ಫಿಗರ್ ಮಾಡುತ್ತದೆ
# ✅ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಕೋಡ್ ಅನ್ನು ರಚಿಸುತ್ತದೆ
# ✅ ಆಜೂರ್‌ಗೆ ಅಪ್ಲೋಡ್ ಮಾಡುತ್ತದೆ
# ✅ ಕಾರ್ಯಾಚರಣೆ ಮಾಡಬಹುದಾದ URL ಅನ್ನು ನೀಡುತ್ತದೆ
azd up
```

**ಅಲ್ಲೇ!** Azure Portal ಕ್ಲಿಕ್‌ಮಾಡುವ ಅವಶ್ಯಕತೆ ಇಲ್ಲ, ಮೊದಲು ARM ಟೆಂಪ್ಲೇಟುಗಳನ್ನು ಜಟಿಲವಾಗಿ ಕಲಿಯಬೇಕಾದ ಅಗತ್ಯ ಇಲ್ಲ, ಕೈಯಿಂದ ಸಂರಚನೆ ಅಗತ್ಯವಿಲ್ಲ — ಕೇವಲ ಕೆಲಸ ಮಾಡುವ ಅಪ್ಲಿಕೇಶನ್‌ಗಳು Azure ಮೇಲೆ ಸಜ್ಜು.

## ❓ Azure Developer CLI vs Azure CLI: ವ್ಯತ್ಯಾಸವೇನು?

ಇದು ಪ್ರಾರಂಭಿಕರುಳ್ಳಲ್ಲಿ ಅತ್ಯಂತ ಸಾಮಾನ್ಯವಾದ ಪ್ರಶ್ನೆ. ಇಲ್ಲಿದೆ ಸರಳ ಉತ್ತರ:

| ವೈಶಿಷ್ಟ್ಯ | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **ಉದ್ದೇಶ** | ವೈಯಕ್ತಿಕ Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿರ್ವಹಿಸುವುದು | ಸಂಪೂರ್ಣ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿಯೋಜಿಸುವುದು |
| **ಮನೋಭಾವ** | ಇನ್‌ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್-ಕೇಂದ್ರೀಯ | ಅಪ್ಲಿಕೇಶನ್-ಕೇಂದ್ರೀಯ |
| **ಉದಾಹರಣೆ** | `az webapp create --name myapp...` | `azd up` |
| **ಕಲಿಕೆಯ ವಕ್ರತೆ** | Azure ಸೇವೆಗಳ ಬಗ್ಗೆ ತಿಳಿದಿರಬೇಕು | ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಮಾತ್ರ ತಿಳಿದುಕೊಳ್ಳಿ |
| **ಯಾರಿಗೆ ಉತ್ತಮ** | DevOps, ಇನ್‌ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ | ಡೆವಲಪರ್‌ಗಳು, ಪ್ರೋಟೋಟೈಪಿಂಗ್ |

### ಸರಳ ರೂಪಕ

- **Azure CLI** ಮನೆ ಕಟ್ಟುವ ಎಲ್ಲಾ ಸಾಧನಗಳನ್ನು ಹೊಂದಿರುವಂತೆ — ಹ್ಯಾಮರ್‌ಗಳು, ಕತ್ತಿಗಳು, ಹಿಮ್ಮುಖಗಳು. ನೀವು ಏನನ್ನಾದರೂ ಕಟ್ಟಬಹುದು, ಆದರೆ ನಿರ್ಮಾಣ ತಿಳಿದಿರಬೇಕು.
- **Azure Developer CLI** ಗೃಹವ್ಯಾಸಿಯನ್ನು ನೇಮಿಸುವಂತೆ — ನೀವು ಏನು ಬೇಕೊ ಹೇಳುತ್ತೀರಿ, ಅವರು ಕಟ್ಟುವ ಕೆಲಸವನ್ನು ತೆಗೆದುಕೊಳ್ಳುತ್ತಾರೆ.

### ಯಾವಾಗ ಯಾವುದು ಬಳಸಬೇಕು

| ಸ್ಥಿತಿ | ಇದನ್ನು ಬಳಸಿ |
|----------|----------|
| "ನಾನು ನನ್ನ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಬೇಗನೇ ನಿಯೋಜಿಸಲು ಬಯಸುತ್ತೇನೆ" | `azd up` |
| "ನಾನು ಕೇವಲ ಒಂದು ಸ್ಟೋರೇಜ್ ಖಾತೆಯನ್ನು ರಚಿಸಬೇಕಿದೆ" | `az storage account create` |
| "ನಾನು ಪೂರ್ಣ AI ಅಪ್ಲಿಕೇಶನ್ ನಿರ್ಮಿಸುತ್ತಿದ್ದೇನೆ" | `azd init --template azure-search-openai-demo` |
| "ನಾನು विशಿಷ್ಟ Azure ಸಂಪನ್ಮೂಲವನ್ನು ಡಿಬಗ್ ಮಾಡಬೇಕಿದೆ" | `az resource show` |
| "ನಾನು ನிமಿಷಗಳಲ್ಲಿ ಪ್ರೊಡಕ್ಷನ್-ಯೋಗ್ಯ ನಿಯೋಜನೆಯನ್ನು ಬಯಸುತ್ತೇನೆ" | `azd up --environment production` |

### ಅವು ಒಟ್ಟಿಗೆ ಕೆಲಸ ಮಾಡುತ್ತವೆ!

AZD ಒಳಗೆ Azure CLI ಅನ್ನು ಬಳಸುತ್ತದೆ. ನೀವು ಎರಡನ್ನೂ ಬಳಸಿಬಹುದು:
```bash
# ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನನ್ನು AZD ಬಳಸಿ ನಿಯೋಜಿಸಿ
azd up

# ನಂತರ Azure CLI ಯೊಂದಿಗೆ ನಿರ್ದಿಷ್ಟ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಸುಧಾರಿಸಿ
az webapp config set --name myapp --always-on true
```

**Windows:**
```powershell

```

**macOS:**
```bash
brew tap azure/azd && brew install azd
```

**Linux:**
```bash
curl -fsSL https://aka.ms/install-azd.sh | bash
```

### ಹಂತ 2: Azure ಗೆ ಲಾಗಿನ್ ಮಾಡಿ

```bash
azd auth login
```

### ಹಂತ 3: ನಿಮ್ಮ ಮೊದಲ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ನಿಯೋಜಿಸಿ

```bash
# ಟೆಂಪ್ಲೇಟಿನಿಂದ ಪ್ರಾರಂಭಿಸಿ

azd up
```

```bash
# Remove all resources when done experimenting
azd down --force --purge
```

- AI ವರ್ಕ್‌ಲೋಡ್‌ಗಳ ವೆಚ್ಚ 최적화

## 📖 ಆಳವಾದ ಅಧ್ಯಯನ: AZD ಸಾಮರ್ಥ್ಯಗಳು

ಮೂಲಭೂತಾಂಶಗಳನ್ನೊಳ್ಳದೆ, AZD ಉತ್ಪಾದನಾ ನಿಯೋಜನೆಗಳಿಗೆ ಶಕ್ತಿಶಾಲಿ ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ:

- **ಟೆಂಪ್ಲೇಟ್ ಆಧಾರಿತ ನಿಯೋಜನೆಗಳು** - ಸಾಮಾನ್ಯ ಅಪ್ಲಿಕೇಶನ್ ಮಾದರಿಗಳಿಗೆ ಪೂರ್ವ-ನಿರ್ಮಿತ ಟೆಂಪ್ಲೇಟುಗಳನ್ನು ಬಳಸಿ
- **Infrastructure as Code** - Bicep ಅಥವಾ Terraform ಬಳಸಿ Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿರ್ವಹಿಸಿ  
- **ಏಕೀಕೃತ ವರ್ಕ್‌ಫ್ಲೋಗಳು** - ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿರ್ವಹಿಸಲು, ನಿಯೋಜಿಸಲು ಮತ್ತು ಮೇಲ್ವಿಚಾರಣೆಗೆ ಸೀಮೆರಹಿತ ಪ್ರಕ್ರಿಯೆ
- **ಡೆವೆಲಪರ್-ಸ್ನೇಹಿ** - ಡೆವೆಲಪರ್ ಉತ್ಪಾದಕತೆ ಮತ್ತು ಅನುಭವಕ್ಕಾಗಿ tốiಮೈಜ್ಡ್

### **AZD + Microsoft Foundry: AI ನಿಯೋಜನೆಗಳಿಗಾಗಿ ಸೂಕ್ತ**

**AI ಪರಿಹಾರಗಳಿಗೆ AZD ಯಾಕೆ?** AZD AI ಅಭಿವೃದ್ಧಕರ ಎದುರಿಸುವ ಪ್ರಮುಖ ಸವಾಲುಗಳನ್ನು ಪರಿಹರಿಸುತ್ತದೆ:

- **AI-ಸಿದ್ಧ ಟೆಂಪ್ಲೇಟುಗಳು** - Microsoft Foundry ಮಾದರಿ, Cognitive Services, ಮತ್ತು ML ಕಾರ್ಯಪ್ರವೃತ್ತಿಗಳಿಗಾಗಿ ಪೂರ್ವ-ಕಾನ್ಫಿಗರ್ಡ್ ಟೆಂಪ್ಲೇಟುಗಳು
- **ಭದ್ರತಾ AI ನಿಯೋಜನೆಗಳು** - AI ಸೇವೆಗಳು, API ಕీలు ಮತ್ತು ಮಾದರಿ ಎಂಡ್ಪಾಯಿಂಟ್‌ಗಳಿಗೆ ಒಳಗೊಂಡ ಭದ್ರತಾ ಮಾದರಿಗಳು  
- **ಉತ್ಪಾದನಾ AI ಮಾದರಿಗಳು** - ಪ್ರಮಾಣಾರ್ಹ, ಖರ್ಚು-ಆಶಕ್ತ AI ಅಪ್ಲಿಕೇಶನ್ ನಿಯೋಜನೆಗಳ ಅತ್ಯುತ್ತಮ ಅಭ್ಯಾಸಗಳು
- **ಎಂಡ್-ಟು-ಎಂಡ್ AI ವರ್ಕ್‌ಫ್ಲೋಗಳು** - ಮಾದರಿ ಅಭಿವೃದ್ಧಿಯಿಂದ ಉತ್ಪಾದನಾ ನಿಯೋಜನೆಯವರೆಗೆ ಸೂಕ್ತ ಮೇಲ್ವಿಚಾರಣೆಯೊಂದಿಗೆ
- **ಖರ್ಚು tốiಮೈಜೇಶನ್** - AI ಕಾರ್ಯಪ್ರವೃತ್ತಿಗಳಿಗಾಗಿ ಸ್ಮಾರ್ಟ್ ಸಂಪನ್ಮೂಲ ಹಂಚಿಕೆ ಮತ್ತು ಸ್ಕೇಲಿಂಗ್ ತಂತ್ರಗಳು
- **Microsoft Foundry ಏಕೀಕರಣ** - Microsoft Foundry ಮಾದರಿ ಕ್ಯಾಟಲಾಗ್ ಮತ್ತು ಎಂಡಪ್‌ಾಯಿಂಟ್‌ಗಳಿಗೆ ಸ್ಲೀಸ್ ಸಂಪರ್ಕ

## 🔧 ತ್ವರಿತ ಸಮಸ್ಯೆ ಪರಿಹಾರ ಮಾರ್ಗದರ್ಶಿ

**ಆರಂಭಿಕರು ಎದುರಿಸುವ ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳು ಮತ್ತು ತಕ್ಷಣದ ಪರಿಹಾರಗಳು:**

<details>
<summary><strong>❌ "azd: command not found"</strong></summary>

```bash
# البداية AZD ಅನ್ನು ಸ್ಥಾಪಿಸಿ
# ವಿಂಡೋಸ್ (ಪವರ್‌ಶೆಲ್):
winget install microsoft.azd

# ಮ್ಯಾಕ್‌ಒಎಸ್:
brew tap azure/azd && brew install azd

# ಲಿನಕ್ಸ್:
curl -fsSL https://aka.ms/install-azd.sh | bash

# ಸ್ಥಾಪನೆಯನ್ನು ಪರಿಶೀಲಿಸಿ
azd version
```
</details>

<details>
<summary><strong>❌ "No subscription found" or "Subscription not set"</strong></summary>

```bash
# ಲಭ್ಯವಿರುವ ಸಬ್ಸ್ಕ್ರಿಪ್ಷನ್‌ಗಳನ್ನು ಪಟ್ಟಿ ಮಾಡಿ
az account list --output table

# ಪೂರ್ವನಿರ್ಧಿಷ್ಟ ಸಬ್ಸ್ಕ್ರಿಪ್ಷನ್ ಅನ್ನು ಹೊಂದಿಸಿ
az account set --subscription "<subscription-id-or-name>"

# AZD ಪರಿಸರಕ್ಕಾಗಿ ಹೊಂದಿಸಿ
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# ಸತ್ಯಾಪಿಸಿ
az account show
```
</details>

<details>
<summary><strong>❌ "InsufficientQuota" or "Quota exceeded"</strong></summary>

```bash
# ವಿಭಿನ್ನ Azure ಪ್ರದೇಶವನ್ನು ಪ್ರಯತ್ನಿಸಿ
azd env set AZURE_LOCATION "westus2"
azd up

# ಅಥವಾ ಅಭಿವೃದ್ಧಿ ಪರಿಸರದಲ್ಲಿ ಸಣ್ಣ SKUsಗಳನ್ನು ಬಳಸಿ
# infra/main.parameters.json ಅನ್ನು ಸಂಪಾದಿಸಿ:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>

<details>
<summary><strong>❌ "azd up" fails halfway through</strong></summary>

```bash
# ಆಯ್ಕೆ 1: ಸ್ವಚ್ಛಗೊಳಿಸಿ ಮತ್ತು ಮರುಪ್ರಯತ್ನಿಸಿ
azd down --force --purge
azd up

# ಆಯ್ಕೆ 2: ಮೂಲಸೌಕರ್ಯವನ್ನು ಮಾತ್ರ ಸರಿಪಡಿಸಿ
azd provision

# ಆಯ್ಕೆ 3: ವಿಸ್ತೃತ ಸ್ಥಿತಿಯನ್ನು ಪರಿಶೀಲಿಸಿ
azd show

# ಆಯ್ಕೆ 4: Azure Monitor ನಲ್ಲಿ ಲಾಗ್‌ಗಳನ್ನು ಪರಿಶೀಲಿಸಿ
azd monitor --logs
```
</details>

<details>
<summary><strong>❌ "Authentication failed" or "Token expired"</strong></summary>

```bash
# ಮರುಪ್ರಾಮಾಣೀಕರಿಸಿ
az logout
az login

azd auth logout
azd auth login

# ಪ್ರಾಮಾಣೀಕರಣವನ್ನು ಪರಿಶೀಲಿಸಿ
az account show
```
</details>

<details>
<summary><strong>❌ "Resource already exists" or naming conflicts</strong></summary>

```bash
# AZD ಅನನ್ಯ ಹೆಸರುಗಳನ್ನು ರಚಿಸುತ್ತದೆ, ಆದರೆ ಘರ್ಷಣೆ ಇದ್ದರೆ:
azd down --force --purge

# ನಂತರ ಹೊಸ ಪರಿಸರದೊಂದಿಗೆ ಮರುಪ್ರಯತ್ನಿಸಿ
azd env new dev-v2
azd up
```
</details>

<details>
<summary><strong>❌ ಟೆಂಪ್ಲೇಟ್ ನಿಯೋಜನೆ ಬಹಳ ಸಮಯ ತೆಗೆದುಕೊಳ್ಳುತ್ತಿದೆ</strong></summary>

**ಸಾಮಾನ್ಯ ನಿರೀಕ್ಷೆ ಸಮಯಗಳು:**
- ಸರಳ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್: 5-10 ನಿಮಿಷ
- ಡೇಟಾಬೇಸ್ ಹೊಂದಿದ ಅಪ್ಲಿಕೇಶನ್: 10-15 ನಿಮಿಷ
- AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳು: 15-25 ನಿಮಿಷ (OpenAI ಪ್ರೊವಿಷನಿಂಗ್ ನಿಧಾನವಾಗಿದೆ)

```bash
# ಪ್ರಗತಿಯನ್ನು ಪರಿಶೀಲಿಸಿ
azd show

# 30 ನಿಮಿಷಕ್ಕಿಂತ ಹೆಚ್ಚು ಅಡಕವಾಗಿದ್ದರೆ, Azure ಪೋರ್ಟಲ್ ಪರಿಶೀಲಿಸಿ:
azd monitor
# ವಿಫಲವಾದ ಡಿಪ್ಲಾಯ್ಮೆಂಟ್‌ಗಳನ್ನು ಹುಡುಕಿ
```
</details>

<details>
<summary><strong>❌ "Permission denied" or "Forbidden"</strong></summary>

```bash
# ನಿಮ್ಮ Azure ಪಾತ್ರವನ್ನು ಪರಿಶೀಲಿಸಿ
az role assignment list --assignee $(az account show --query user.name -o tsv)

# ನಿಮಗೆ ಕನಿಷ್ಠ "Contributor" کردار ಬೇಕು
# ನಿಮ್ಮ Azure ಅಡ್ಮಿನ್‌ಗೆ ನೀಡಲು ಕೇಳಿ:
# - Contributor (ಸಂಪನ್ಮೂಲಗಳಿಗಾಗಿ)
# - User Access Administrator (ಭೂಮಿಕಾ ನಿಯೋಜನೆಗಳಿಗೆ)
```
</details>

<details>
<summary><strong>❌ Can't find deployed application URL</strong></summary>

```bash
# ಎಲ್ಲಾ ಸೇವಾ ಎಂಡ್ಪಾಯಿಂಟ್‌ಗಳನ್ನು ತೋರಿಸಿ
azd show

# ಅಥವಾ Azure ಪೋರ್ಟಲ್ ತೆರೆಯಿರಿ
azd monitor

# ವಿಶಿಷ್ಟ ಸೇವೆಯನ್ನು ಪರಿಶೀಲಿಸಿ
azd env get-values
# *_URL ವೇರಿಯಬಲ್‌ಗಳನ್ನು ಹುಡುಕಿ
```
</details>
