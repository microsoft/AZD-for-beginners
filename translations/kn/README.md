## 🚀 Azure Developer CLI (azd) ಎಂದರೆ ಏನು?

**Azure Developer CLI (azd)** ಡೆವಲಪರ್ ಸ್ನೇಹಿ ಕಮಾಂಡ್-ಲೈನ್ ಉಪಕರಣವಾಗಿದೆ, ಅದು ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು Azure ಗೆ ಸರಳವಾಗಿ ಡಿಪ್ಲಾಯ್ ಮಾಡಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ. ಅನೇಕ Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ಕೈಯಿಂದ ಸೃಷ್ಟಿಸಿ ಸಂಪರ್ಕಿಸುವದಕ್ಕೆ ಬದಲು, ನೀವು ಒಂದು ಕಮಾಂಡ್‌ನಲ್ಲಿ ಸಂಪೂರ್ಣ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಡಿಪ್ಲಾಯ್ ಮಾಡಬಹುದು.

### `azd up` ನ ಮಾಯಾಜಾಲ

```bash
# ಈ ಏಕೈಕ ಆಜ್ಞೆ ಎಲ್ಲವನ್ನೂ ಮಾಡುತ್ತದೆ:
# ✅ ಎಲ್ಲಾ ಅಜ್ಯೂರ್ ಸಂಪನ್ಮೂಲಗಳನ್ನು ರಚಿಸುತ್ತದೆ
# ✅ ನೆಟ್ವರ್ಕಿಂಗ್ ಮತ್ತು ಭದ್ರತೆಯನ್ನು ಸಂರಚಿಸುತ್ತದೆ
# ✅ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಕೋಡ್ ಅನ್ನು ನಿರ್ಮಿಸುತ್ತದೆ
# ✅ ಅಜ್ಯೂರ್ ಗೆ ನಿಯೋಜಿಸುತ್ತದೆ
# ✅ ಕಾರ್ಯನಿರ್ವಹಿಸುವ URL ಅನ್ನು ಒದಗಿಸುತ್ತದೆ
azd up
```

**ಅಷ್ಟೇ!** Azure ಪೋರ್ಟಲ್‌ನಲ್ಲಿ ಕ್ಲಿಕ್‌ಮಾಡುವ ಅವಶ್ಯಕತೆ ಇಲ್ಲ, ಮೊದಲಿಗೆ ಕಲಿಯಲು ಸಂಕೀರ್ಣ ARM ಟೆಂಪ್ಲೇಟ್ಗಳ ಅಗತ್ಯವಿಲ್ಲ, ಕೈಯಿಂದ ಕಾನ್ಫಿಗರ್‌ಮಾಡುವ ಅವಶ್ಯಕತೆ ಇಲ್ಲ — ಕೇವಲ Azure ಮೇಲೆ ಕಾರ್ಯನಿರತ ಅಪ್ಲಿಕೇಶನ್‌ಗಳು.

## ❓ Azure Developer CLI ಮತ್ತು Azure CLI: ವ್ಯತ್ಯಾಸ ಏನು?

ಇದು ಆರಂಭಿಕರು ಸಾಮಾನ್ಯವಾಗಿ ಕೇಳುವ ಪ್ರಶ್ನೆಯಾಗಿದೆ. ಸಾದಾ ಉತ್ತರ ಇಲ್ಲಿದೆ:

| ವೈಶಿಷ್ಟ್ಯ | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **ಉದ್ದೇಶ** | ವೈಯಕ್ತಿಕ Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿರ್ವಹಿಸಲು | ಸಂಪೂರ್ಣ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಡಿಪ್ಲಾಯ್ ಮಾಡಲು |
| **ಮನೋಭಾವ** | ಅಡಿಪಾಯ-ಕೇಂದ್ರೀಕೃತ | ಅಪ್ಲಿಕೇಶನ್-ಕೇಂದ್ರೀಕೃತ |
| **ಉದಾಹರಣೆ** | `az webapp create --name myapp...` | `azd up` |
| **ಕಲಿಕೆಯ ವಕ್ರತೆ** | Azure ಸೇವೆಗಳನ್ನು ತಿಳಿದುಕೊಂಡಿರಬೇಕು | ಕೇವಲ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಅರಿತುಕೊಳ್ಳುವುದು ಸಾಕು |
| **ಉತ್ತಮವಾಗಿದೆ** | DevOps, ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ | ಡೆವಲಪರ್‌ಗಳು, ಪ್ರೋಟೋಟೈಪಿಂಗ್ |

### ಸರಳ ಹೋಲಿಕೆ

- **Azure CLI** ಅನ್ನು ಮನೆ ಕಟ್ಟಲು ಬೇಕಾದ ಎಲ್ಲಾ ಸಾಧನಗಳನ್ನು ಹೊಂದಿರುವಂತೆ - ಹ್ಯಾಮರ್‌ಗಳು, ಸಾ, ನೋಲು. ನೀವು ಏಕೆ ಬೇಕಾದರೂ ಕಟ್ಟಬಹುದು, ಆದರೆ ನಿರ್ಮಾಣವನ್ನು ತಿಳಿಯಬೇಕು.
- **Azure Developer CLI** ಅನ್ನು ಒಬ್ಬ ಗುತ್ತಿಗೆಯಿಂದ ಕೆಲಸಕ್ಕೆ ನೀಡಿಕೊಂಡಿರುವಂತೆ - ನೀವು ಏನು ಬೇಕು ಎಂದು ವಿವರಿಸುತ್ತೀರಿ, ಅವರು ಕಟ್ಟುವ ಕೆಲಸವನ್ನು ನಿರ್ವಹಿಸುತ್ತಾರೆ.

### ಪ್ರತಿ ಒಂದು ಯಾವಾಗ ಬಳಸಬೇಕು

| ಪರಿಸ್ಥಿತಿ | ಇದನ್ನು ಬಳಸಿ |
|----------|----------|
| "I want to deploy my web app quickly" | `azd up` |
| "I need to create just a storage account" | `az storage account create` |
| "I'm building a full AI application" | `azd init --template azure-search-openai-demo` |
| "I need to debug a specific Azure resource" | `az resource show` |
| "I want production-ready deployment in minutes" | `azd up --environment production` |

### ಅವುಗಳು ಜೊತೆಯಾಗಿ ಕೆಲಸ ಮಾಡುತ್ತವೆ!

AZD ಒಳಗಿನಿಂದ Azure CLI ಅನ್ನು ಬಳಸುತ್ತದೆ. ನೀವು دونوں ಅನ್ನು ಬಳಸಬಹುದು:
```bash
# ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು AZD ನಲ್ಲಿ ನಿಯೋಜಿಸಿ
azd up

# ನಂತರ Azure CLI ಬಳಸಿ ನಿರ್ದಿಷ್ಟ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಸೂಕ್ಷ್ಮವಾಗಿ ಸರಿಹೊಂದಿಸಿ
az webapp config set --name myapp --always-on true
```

## 🎯 3 ಹಂತಗಳಲ್ಲಿ ಪ್ರಾರಂಭಿಸೋದು

### ಹಂತ 1: AZD ಅನ್ನು ಸ್ಥಾಪಿಸಿ (2 ನಿಮಿಷ)

**Windows:**
```powershell
winget install microsoft.azd
```

**macOS:**
```bash
brew tap azure/azd && brew install azd
```

**Linux:**
```bash
curl -fsSL https://aka.ms/install-azd.sh | bash
```

### ಹಂತ 2: Azure ಗೆ ಲಾಗಿನ್ ಆಗಿ

```bash
azd auth login
```

### ಹಂತ 3: ನಿಮ್ಮ ಮೊದಲ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಡಿಪ್ಲಾಯ್ ಮಾಡಿ

```bash
# ಟೆಂಪ್ಲೇಟಿನಿಂದ ಪ್ರಾರಂಭಿಸಿ
azd init --template todo-nodejs-mongo

# ಅಜೂರ್‌ಗೆ ನಿಯೋಜಿಸಿ (ಎಲ್ಲವನ್ನು ರಚಿಸುತ್ತದೆ!)
azd up
```

**🎉 ಅಷ್ಟೇ!** ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಈಗ Azure ನಲ್ಲಿ ಲೈವ್ ಆಗಿದೆ.

### ಕ್ಲೀನ್ ಅಪ್ (ಮರೆತಿಕೊಂಡಿರುವುದಿಲ್ಲ!)

```bash
# Remove all resources when done experimenting
azd down --force --purge
```

## ಕೋರ್ಸ್ ಅವಲೋಕನ

ಸಂರಚಿತ ಅಧ್ಯಾಯಗಳ ಮೂಲಕ Azure Developer CLI (azd) ನ್ನು овಗಲೆ ಮಾಡಿ, ಪ್ರಗತಿಯುತ ಕಲಿಕೆಗೆ ತಯಾರಾಗಿರುವ ವಿನ್ಯಾಸ. **Microsoft Foundry ಏಕೀಕರಣದೊಂದಿಗೆ AI ಅಪ್ಲಿಕೇಶನ್ ನಿಯೋಜನೆ ಮೇಲೆ ವಿಶೇಷ ಗುರಿ.**

### ಆಧುನಿಕ ಡೆವಲಪರ್‌ಗಳಿಗಾಗಿ ಈ ಕೋರ್ಸ್ ಅಗತ್ಯವಿರುವ ಕಾರಣ

Microsoft Foundry ಡಿಸ್ಕಾರ್ಡ್ ಸಮುದಾಯದ洞察ಗಳ ಆಧಾರದಲ್ಲಿ, **ಡೆವಲಪರ್‌ಗಳ 45% ಗೊಳ್ಳುವುದು AZD ಅನ್ನು AI ಕೆಲಸಭಾರಗಳಿಗೆ ಉಪಯೋಗಿಸಲು ಬಯಸುತ್ತಾರೆ** ಆದರೆ ಇವರು ಕೆಳಗಿನ ಸವಾಲುಗಳನ್ನು ಎದುರಿಸುತ್ತಾರೆ:
- ಸಂಕೀರ್ಣ ಬಹು-ಸೇವಾ AI ವಾಸ್ತುಶಿಲ್ಪಗಳು
- ಉತ್ಪಾದನಾ AI ನಿಯೋಜನೆ ಉತ್ತಮ ಕಾರ್ಯವಿಧಾನಗಳು  
- Azure AI ಸೇವೆಗಳ ಏಕೀಕರಣ ಮತ್ತು ಕಾನ್ಫಿಗರೇಶನ್
- AI ಕೆಲಸಭಾರಗಳಗಾಗಿ ವೆಚ್ಚದ 최적화
- AI-ನಿರ್ದಿಷ್ಟ ನಿಯೋಜನೆ ಸಮಸ್ಯೆಗಳನ್ನು ಡಿಬಗ್ ಮಾಡುವುದು

### ಕಲಿಕಾ ಉದ್ದೇಶಗಳು

ಈ ಸಂರಚಿತ ಕೋರ್ಸ್ ಅನ್ನು ಪೂರ್ಣಗೊಳಿಸುವ ಮೂಲಕ, ನೀವು:
- **AZD ಮೂಲಭೂತಗಳನ್ನು ಆಳವಾಗಿ ತಿಳಿದುಕೊಳ್ಳಿ**: ಮೂಲ ಸಿದ್ಧಾಂತಗಳು, ಸ್ಥಾಪನೆ ಮತ್ತು ಕಾನ್ಫಿಗರೇಶನ್
- **AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಡಿಪ್ಲಾಯ್ ಮಾಡಿ**: Microsoft Foundry ಸೇವೆಗಳೊಂದಿಗೆ AZD ಬಳಸಿ
- **Infrastructure as Code ಅನ್ನು ಜಾರಿಗೆ ಮಾಡಿ**: Bicep ಟೆಂಪ್ಲೇಟ್ಗಳಿಂದ Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿರ್ವಹಿಸಿ
- **ನಿಯೋಜನೆಗಳನ್ನು ಡಿಬಗ್ ಮಾಡಿ**: ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳನ್ನು ಪರಿಹರಿಸಿ ಮತ್ತು ದೋಷ ನಿಗದಿಪಡಿಸಿ
- **ಉತ್ಪಾದನಕ್ಕಾಗಿ 최적化 ಮಾಡಿ**: ಭದ್ರತೆ, ಸ್ಕೇಲಿಂಗ್, ಮಾನಿಟರಿಂಗ್ ಮತ್ತು ವೆಚ್ಚ ನಿರ್ವಹಣೆ
- **ಬಹು-ಏಜೆಂಟ್ ಪರಿಹಾರಗಳನ್ನು ರಚಿಸಿ**: ಸಂಕೀರ್ಣ AI ವಾಸ್ತುಶಿಲ್ಪಗಳನ್ನು ಡಿಪ್ಲಾಯ್ ಮಾಡಿ

## 📖 ಆಳವಾದ ಅವಲೋಕನ: AZD ಸಾಮರ್ಥ್ಯಗಳು

ಮೂಲಭೂತಗಳ ಹೊರತಾಗಿ, AZD ಉತ್ಪಾದನಾ ನಿಯೋಜನೆಗಳಿಗೆ ಶಕ್ತಿಶಾಲಿ ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ:

- **ಟೆಂಪ್ಲೇಟಿನ ಆಧಾರದ ನಿಯೋಜನೆಗಳು** - ಸಾಮಾನ್ಯ ಆಪ್ ಮಾದರಿಗಳಿಗೆ ಪೂರ್ವ-ನಿರ್ಮಿತ ಟೆಂಪ್ಲೇಟುಗಳನ್ನು ಬಳಸಿ
- **Infrastructure as Code** - Bicep ಅಥವಾ Terraform ಬಳಸಿ Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿರ್ವಹಿಸಿ  
- **ಎಂಟಿಗ್ರೇಟೆಡ್ ವರ್ಕ್‌ಫ್ಲೋಗಳು** - ಅನಾಯಾಸವಾಗಿ ಅಳವಡಿಕೆ, ನಿಯೋಜನೆ ಮತ್ತು ಮನಿಟರಿಂಗ್ ಮಾಡಿ
- **ಡೆವೆಲಪರ್-ಅನುಕೂಲ** - ಡೆವೆಲಪರ್ ಉತ್ಪಾದಕತೆ ಮತ್ತು ಅನುಭವಕ್ಕೆ tốiimiz ಮಾಡಿದಿದೆ

### **AZD + Microsoft Foundry: AI ನಿಯೋಜನೆಗಳಿಗೆ ಪರಿಪೂರ್ಣ**

**AI ಪರಿಹಾರಗಳಿಗೆ AZD ಯಾಕೆ?** AZD AI ಅಭಿವೃದ್ಧಿಪಕರ ಎದುರಿಸುವ ಪ್ರಮುಖ ಸವಾಲುಗಳಿಗೆ ಉತ್ತರ ನೀಡುತ್ತದೆ:

- **AI-ಸಿದ್ಧ ಟೆಂಪ್ಲೇಟುಗಳು** - Azure OpenAI, Cognitive Services, ಮತ್ತು ML ವರ್ಕ್‌ಲೋಸ್ಟುಗಳಿಗಾಗಿ ಪೂರ್ವ-ಕಾನ್ಫಿಗರ್ ಮಾಡಿದ ಟೆಂಪ್ಲೇಟುಗಳು
- **ಭದ್ರ AI ನಿಯೋಜನೆಗಳು** - AI ಸೇವೆಗಳು, API ಕೀಗಳು ಮತ್ತು ಮಾದರಿ ಎಂಡ್ಪಾಯಿಂಟ್‌ಗಳಿಗಾಗಿ ಒಳಗೆ ನಿರ್ಮಿತ ಭದ್ರತಾ ಮಾದರಿಗಳು  
- **ಉತ್ಪಾದನಾ AI ಮಾದರಿಗಳು** - ವಿಸ್ತಾರಗೊಳ್ಳುವ, ವೆಚ್ಚ-ಪ್ರಭಾವಿ AI ಅಪ್ಲಿಕೇಶನ್ ನಿಯೋಜನೆಗಳ ಅತ್ಯುತ್ತಮ ಅಭ್ಯಾಸಗಳು
- **ಎಂಡ್-ಟು-ಎಂಡ್ AI ವರ್ಕ್ಫ್ಲೋಗಳು** - ಮಾದರಿ ಅಭಿವೃದ್ಧಿಯಿಂದ ಉತ್ಪಾದನಾ ನಿಯೋಜನೆವರೆಗೆ ಸೂಕ್ತ ಮಾನಿಟರಿಂಗ್ ಸಹಿತ
- **ವೆಚ್ಚ ಆಪ್ಟಿಮೈಸೇಶನ್** - AI ವರ್ಕ್‌ಲೋಗಳಿಗೆ ಬುದ್ಧಿವಂತರಿಯ ಸಂಪನ್ಮೂಲ ಹಂಚಿಕೆ ಮತ್ತು ಸ್ಕೇಲಿಂಗ್ ತಂತ್ರಗಳು
- **Microsoft Foundry ಏಕೀಕರಣ** - Microsoft Foundry ಮಾದರಿ ಕ್ಯಾಟಲಾಗ್ ಮತ್ತು ಎಂಡ್ಪಾಯಿಂಟ್‌ಗಳೊಂದಿಗೆ ನಿರ್ವಿಚ್ಚಿರ ಸಂಪರ್ಕ
