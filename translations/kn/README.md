## 🆕 azd ನಲ್ಲಿ ಇಂದಿನ ಹೊಸದು

> 📌 ಈ ಕೋರ್ಸ್ **`azd 1.25.6`** (ಜೂನ್ 2026) ವಿರುದ್ಧ ಮಾನ್ಯಗೊಂಡಿದೆ. ನಿಮ್ಮ ಬಿಲ್ಡ್ ಪರಿಶೀಲಿಸಲು `azd version` ಓಡಿಸಿ, ಮತ್ತು తాజಾವನ್ನಾಗಿಸಲು `azd upgrade` ಅನ್ನು ಬಳಸಿ.

Azure Developer CLI ಈಗ ಪರಂಪರাগত ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳು ಮತ್ತು API ಗಳಿಗಿಂತ ಮೀರಿ ವಿಸ್ತಾರಗೊಂಡಿದೆ. ಇಂದಿನ ದಿನದಲ್ಲಿ, azd ಯಾವ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಆದರೂ Azure ಗೆ ನಿಯೋಜಿಸಲು ಏಕೈಕ ಸಾಧನವಾಗಿದೆ — AI-ಸಹಾಯಿತ ಅಪ್ಲಿಕೇಶನ್‌ಗಳು ಮತ್ತು ಬುದ್ಧಿವಂತ ಎಜೆಂಟ್ಗಳನ್ನು ಸಹ ಸೇರಿಸಲಾಗಿದೆ.

ಇದಾಗಿರುವುದರಿಂದ ನಿಮಗೆ ಏನು ಅರ್ಥ:

- **AI ಎಜೆಂಟ್ಗಳು ಈಗ ಪ್ರಥಮ-ಶ್ರೇಣಿಯ azd ವರ್ಕ್‌ಲೋಡ್ ಗಳು.** ನೀವು ಈಗಾಗಲೇ ತಿಳಿದಿರುವ `azd init` → `azd up` ವರ್ಕ್‌ಫ್ಲೋವನ್ನು ಬಳಸಿ AI ಎಜೆಂಟ್ ಪ್ರಾಜೆಕ್ಟ್‌ಗಳನ್ನು ಪ್ರವೇಶಿಸಬಹುದು, ನಿಯೋಜಿಸಬಹುದು ಮತ್ತು ನಿರ್ವಹಿಸಬಹುದು.
- **CLI ಮೂಲಕ ಸಂಪೂರ್ಣ ಎಜೆಂಟ್ ಲೈಫ್‌ಸೈಕಲ್.** `azure.ai.agents` ಎಕ್ಸ್ಟೆನ್ಶನ್ ಈಗ ಸಂಪೂರ್ಣ ಪ್ರಯಾಣವನ್ನು ಒಳಗೊಳ್ಳುತ್ತದೆ—ಸ್ಕಾಫೋಲ್ಡ್ ಮಾಡಲು `azd ai agent init`, ಪರೀಕ್ಷಿಸಲು `azd ai agent invoke` (ಪ್ರತಿಕ್ರಿಯಾ-ಟೈಮಿಂಗ್ ಔಟ್ಪುಟ್ ಒದಗಿಸುತ್ತದೆ), ಗುಣಮಟ್ಟವನ್ನು ಅಳತೆಮಾಡಲು `azd ai agent eval generate` ಮತ್ತು ಸುಧಾರಿಸಲು `azd ai agent optimize`, ಮತ್ತು ಸ್ವಚ್ಛಗೊಳಿಸಲು `azd ai agent delete`.
- **ಹೆಚ್ಚು AI ನಿರ್ಮಾಣ ಗೊಂದಲದ ಘಟಕಗಳು.** ಹೊಸ ಪೂರ್ವವೀಕ್ಷಣೆ ಎಕ್ಸ್ಟೆನ್ಶನ್‌ಗಳು—`azure.ai.skills` ಮತ್ತು `azure.ai.connections`—ನೀವು ಪುನಃಬಳಕೆಗೆ ತಗಲುವ ಎಜೆಂಟ್ ಸ್ಕಿಲ್ಸ್ ಮತ್ತು Foundry ಸಂಪರ್ಕಗಳನ್ನು ನೇರವಾಗಿ azd ಮೂಲಕ ನಿರ್ವಹಿಸಲು ಅನುಮತಿಸುತ್ತವೆ.
- **Microsoft Foundry ಏಕರೂಪಿಕೆ** ಮಾದರಿ ನಿಯೋಜನೆ, ಏಜೆಂಟ್ ಹೋಸ್ಟಿಂಗ್ ಮತ್ತು AI ಸೇವಾ ಸಂರಚನೆಯನ್ನು ನೇರವಾಗಿ azd ಟೆಂಪ್ಲೇಟ್ ಪರಿಸರಕ್ಕೆ ತರುತ್ತದೆ.
- **ದಿನನಿತ್ಯದ ಮೂಲಭೂತ ಕೆಲಸಗಳು ಮೃದುವಾಗಿವೆ.** ಇತ್ತೀಚಿನ ಬಿಡುಗಡೆಯುಗಳು `azd init` ಅನ್ನು ಐಡಂಪೊಟೆಂಟ್ (ಪುನಃ ಓಡಿಸುವುದು ಸುರಕ್ಷಿತ)ಮಾಡಿರುವದು, `azd auth login` ಹಳೆಯ ಟೋಕನ್‌ಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಕ್ಲಿಯರ್ ಮಾಡುತ್ತದೆ, ಮತ್ತು ಸ್ನೇಹಪರ `azd tool` ಪ್ರಥಮ-ಓದಲು ಸೆಟ್‌ಅಪ್ ಪ್ರಾಂಪ್ಟ್ ಸೇರಿಸಲಾಗಿದೆ.
- **ಮೂಲ ವರ್ಕ್‌ಫ್ಲೋ ಬದಲಾಗಿಲ್ಲ.** ನೀವು todo ಆಪ್, ಮೈಕ್ರೋಸರ್ವಿಸ್, ಅಥವಾ ಬಹು-ಎಜೆಂಟ್ AI ಪರಿಹಾರವನ್ನು ನಿಯೋಜಿಸುತ್ತಿದ್ದೀರಾ ಎಂದು ಅವಶ್ಯಕತೆ ಇಲ್ಲ—ಕಮ್ಯಾಂಡ್ ಗಳು ಒಂದೇ ಇದ್ದೇ ಇವೆ.

> **Aspire ಬಳಕೆದಾರರಿಗೆ ಟಿಪ್ಪಣಿ:** Microsoft ಈಗ ಉತ್ಪನ್ನವನ್ನು ಸರಳವಾಗಿ **Aspire** (ಹಿಂದಿನ ".NET Aspire") ಎಂದು ಉಲ್ಲೇಖಿಸುತ್ತದೆ. azd ನ Aspire ಬೆಂಬಲ ಬದಲಾವಣೆಗೊಳ್ಳಿಲ್ಲ—ಕೇವಲ ಹೆಸರು ನವೀಕರಿಸಲಾಯಿತು.

ನೀವು ಈಗಾಗಲೇ azd ಬಳಸಿದಿದ್ದರೆ, AI ಬೆಂಬಲವು ಒಂದು ಸಹಜ ವಿಸ್ತಾರ—ಕೇಲುವೊಂದು ವೈಯಕ್ತಿಕ ಸಾಧನ ಅಥವಾ ಅಭಿವೃದ್ಧಿಯ ಉನ್ನತ ಪಥವಲ್ಲ. ಹೊಸದಾಗಿ ಪ್ರಾರಂಭಿಸಿದರೆ, ಎಲ್ಲಾವುದಕ್ಕೂ ಕೆಲಸ ಮಾಡೋ ಒಂದು ವರ್ಕ್‌ಫ್ಲೋ ಅನ್ನು ನೀವು ಕಲಿಯುತ್ತೀರಿ.

## 🚀 Azure Developer CLI (azd) ಎಂದರೆ ಏನು?

**Azure Developer CLI (azd)** ಎಂಬುದು ಡೆವಲಪರ್-ಸ್ನೇಹಿ ಕಮಾಂಡ್-ಲೈನ್ ತಂತ್ರಾಂಶವಾಗಿದ್ದು, ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು Azure ಗೆ ನಿಯೋಜಿಸಲು ಸರಳಗೊಳಿಸುತ್ತದೆ. ಸುತ್ತಮುತ್ತಲಿನ ಅನೇಕ Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ಕೈಯಲ್ಲಿ ರಚಿಸಿ ಸಂಪರ್ಕಿಸುವ ಬದಲು, ನೀವು ಒಂದೇ ಕಮಾಂಡ್‌ನೊಂದಿಗೆ ಸಂಪೂರ್ಣ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿಯೋಜಿಸಬಹುದು.

### `azd up` ರ ಮಾಯಾಜಾಲ

```bash
# ಈ ಏಕಕಾಮಾಂಡ್ ಎಲ್ಲವನ್ನೂ ಮಾಡುತ್ತದೆ:
# ✅ ಎಲ್ಲಾ ಅಜೂರ್ ಸಂಪನ್ಮೂಲಗಳನ್ನು ರಚಿಸುತ್ತದೆ
# ✅ ನೆಟ್‌ವರ್ಕಿಂಗ್ ಮತ್ತು ಭದ್ರತೆಯನ್ನು ಸಂರಚಿಸುತ್ತದೆ
# ✅ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಕೋಡ್ ಅನ್ನು ನಿರ್ಮಿಸುತ್ತದೆ
# ✅ ಅಜೂರ್‌ಗೆ ನಿಯೋಜಿಸುತ್ತದೆ
# ✅ ನಿಮಗೆ ಕಾರ್ಯನಿರ್ವಹಿಸುವ URL ಅನ್ನು ನೀಡುತ್ತದೆ
azd up
```

**ಅದುಷ್ಟೇ!** Azure ಪೋರ್ಟಲ್‌ನಲ್ಲಿ ಕ್ಲಿಕ್ ಮಾಡಬೇಕಿಲ್ಲ, ಮೊದಲಿಗೆ ಗೊಂದಲಪ್ರಿಯ ARM ಟೆಂಪ್ಲೇಟ್ಗಳನ್ನು ಕಲಿಯಬೇಕಾಗಿಲ್ಲ, ಹಾಗೂ ಕೈಯಿಂದ ಸಂರಚನೆ ಮಾಡಬೇಕಾಗಿಲ್ಲ — ಕೇವಲ Azure ನಲ್ಲಿ ಕಾರ್ಯನಿರ್ವಹಿಸುವ ಅಪ್ಲಿಕೇಶನ್‌ಗಳು.

## ❓ Azure Developer CLI vs Azure CLI: ವ್ಯತ್ಯಾಸ ಏನು?

ಇದು ಆರಂಭಿಕರು ಹೆಚ್ಚಿನವೇಳೆ ಕೇಳುವ ಪ್ರಶ್ನೆ. ಸರಳ ಉತ್ತರ ಇಲ್ಲಿದೆ:

| ವೈಶಿಷ್ಟ್ಯ | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **ಉದ್ದೇಶ** | ವೈಯಕ್ತಿಕ Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿರ್ವಹಿಸು | ಸಂಪೂರ್ಣ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿಯೋಜಿಸು |
| **ಮನೋಭಾವ** | ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್-ಕೇಂದ್ರಿತ | ಅಪ್ಲಿಕೇಶನ್-ಕೇಂದ್ರಿತ |
| **ಉದಾಹರಣೆ** | `az webapp create --name myapp...` | `azd up` |
| **ಕಲಿಕೆಯ ವಕ್ರತೆ** | Azure ಸೇವೆಗಳನ್ನು ತಿಳಿದುಕೊಳ್ಳಬೇಕಾಗುತ್ತದೆ | ನಿಮ್ಮ ಆಪ್‌ನ್ನು ಮಾತ್ರ ತಿಳಿದುಕೊಳ್ಳಿ |
| **ಉತ್ತಮ ಉದ್ದೇಶ** | DevOps, ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ | ಅಭಿವೃದ್ಧಿಕಾರರು, ಪ್ರೋಟೋಟೈಪಿಂಗ್ |

### ಸರಳ ಉಪಮೆ

- **Azure CLI** ನಿಮಗೆ ಮನೆಯ ನಿರ್ಮಾಣका ಎಲ್ಲಾ ಕಾರ್ಯೋಪಕರಣಗಳನ್ನು ಕೊಡುತ್ತದೆ — ಹ್ಯಾಮರ್, ಹಚ್ಚು, ನೈಲ್‌ಗಳು. ನೀವು ಯಾವುದೇವನ್ನೂ ನಿರ್ಮಿಸಬಹುದು, ಆದರೆ ನಿರ್ಮಾಣದ ಜ್ಞಾನ ಬೇಕಾಗುತ್ತದೆ.
- **Azure Developer CLI** ಎಂದರೆ ನೀವು ಒಂದು ಕಂಟ್ರಾಕ್ಟರ್ ನೇಮಿಸಿದಂತೆ — ನೀವು ಬೇಕೆನೋ ವಿವರಿಸಿ, ಅವರು ನಿರ್ಮಾಣವನ್ನು ನಿಮ್ಮ ಪರ ಕಾರ್ತಾರ.

### ಯಾವಾಗ ಯಾವದನ್ನು ಬಳಸಬೇಕು

| ದೃಶ್ಯ | ಇದನ್ನು ಬಳಸಿ |
|----------|----------|

| "ನನಗೆ ಕೇವಲ ಒಂದು ಸ್ಟೋರೇಜ್ ಖಾತೆಯನ್ನು ರಚಿಸುವ ಅಗತ್ಯವಿದೆ" | `az storage account create` |
| "ನಾನು ಸಂಪೂರ್ಣ AI ಅಪ್ಲಿಕೇಶನ್ ನಿರ್ಮಿಸುತ್ತಿದ್ದೇನೆ" | `azd init --template azure-search-openai-demo` |

## 🎯 3 ಹಂತಗಳಲ್ಲಿ ಪ್ರಾರಂಭಿಸಲಾಗುವುದು

ನೀವು ಪ್ರಾರಂಭಿಸುವ ಮೊದಲು, ನೀವು ನಿಯೋಜಿಸಲು ಉದ್ದೇಶಿಸುವ ಟೆಂಪ್ಲೇಟ್‌ಗೆ ನಿಮ್ಮ ಯಂತ್ರ ಸಿದ್ಧವಾಗಿರುವುದನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ:

ಯಾವುದೇ ಅಗತ್ಯ ಪರಿಶೀಲನೆ ವಿಫಲವಾದರೆ, ಮೊದಲು ಅದನ್ನು ಸರಿಪಡಿಸಿ ಮತ್ತು ನಂತರ ದ್ರುತ ಪ್ರಾರಂಭವನ್ನು ಮುಂದುವರಿಸಿ.

**Windows:**
```powershell
winget install microsoft.azd
```

**macOS:**
```bash
brew tap azure/azd && brew install azd
```

**Linux:**
```bash
curl -fsSL https://aka.ms/install-azd.sh | bash
```

### ಹಂತ 2: AZD ಗೆ પ્રમાણೀಕರಣ

```bash
# ಈ ಕೋರ್ಸ್‌ನಲ್ಲಿ ನೇರವಾಗಿ Azure CLI ಕಮಾಂಡ್‌ಗಳನ್ನು ಬಳಸಲು ಯೋಜಿಸಿರುವಿದ್ದರೆ ಐಚ್ಛಿಕ
az login
```

ನೀವು ಯಾವದನ್ನು ಬೇಕಾಗುತ್ತದೆ ಎಂದು ಖಚಿತವಾಗಿದ್ದರೆ, [Installation & Setup](docs/chapter-01-foundation/installation.md#authentication-setup) ಎಂಬ ಸಂಪೂರ್ಣ ಸೆಟ್‌ಅಪ್ ಫ್ಲೋವನ್ನು ಅನುಸರಿಸಿ.


**🎉 ಅದಷ್ಟೇ!** ನಿಮ್ಮ ಆಪ್ ಈಗ Azure ಮೇಲೆ ಸಕ್ರಿಯವಾಗಿದೆ.

ಅಧ್ಯಾಯ 1 ಪ್ರಾರಂಭಿಸುವ ಮೊದಲು, ಕೆಳಗಿನವುಗಳನ್ನು ನೀವು ಹೊಂದಿರಬೇಕು ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ. ಈ ಮಾರ್ಗದರ್ಶನದ ನಂತರದ ಸ್ಥಾಪನೆ ಹಂತಗಳು ಈ ಮೂಲಭೂತ ಸಾಮಗ್ರಿಗಳನ್ನು ಈಗಾಗಲೇ ನಿಭಾಯಿಸಲಾಗಿದೆಯೆಂದು ಊಹಿಸುತ್ತವೆ.

- **ಒಂದು Azure ಚಂದಾದಾರಿಕೆ**: ನೀವು ಕೆಲಸದಿಂದ ಇರುವ ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ ಚಂದಾದಾರಿಕೆಯನ್ನು ಅಥವಾ ನಿಮ್ಮ ಸ್ವಂತ ಖಾತೆಯನ್ನು ಬಳಸಬಹುದು, ಅಥವಾ ಪ್ರಾರಂಭಿಸಲು [ಉಚಿತ ಪ್ರಯೋಗ](https://aka.ms/azurefreetrial) ರಚಿಸಬಹುದು.
- **Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ರಚಿಸುವ ಪರವಾನಗಿ**: ಹೆಚ್ಚಿನ ಅಭ್ಯಾಸಗಳಿಗಾಗಿ, ಗುರಿ ಚಂದಾದಾರಿಕೆ ಅಥವಾ ಸಂಪನ್ಮೂಲ ಗುಂಪಿನ ಮೇಲೆ ಕನಿಷ್ಟ **Contributor** ಪ್ರವೇಶ ಇರಬೇಕು. ಕೆಲವು ಅಧ್ಯಾಯಗಳು ಸಂಪನ್ಮೂಲ ಗುಂಪುಗಳು, ನಿರ್ವಹಿತ ಐಡೆಂಟಿಟಿಗಳು ಮತ್ತು RBAC ನಿಯೋಜನೆಗಳನ್ನು ರಚಿಸುವುದಕ್ಕೆ ಕೂಡ ಅವಕಾಶವಿರಬೇಕು ಎಂದು ಊಹಿಸಬಹುದು.
- [**GitHub ಖಾತೆ**](https://github.com): ರೆಪೊ ಫೋರ್ಕ್ ಮಾಡುವುದು, ನಿಮ್ಮ ಬದಲಾವಣೆಗಳನ್ನು ಟ್ರ್ಯಾಕ್ ಮಾಡುವುದು ಮತ್ತು ಕಾರ್ಯಾಗಾರಕ್ಕಾಗಿ GitHub Codespaces ಬಳಸುವುದು ಇತ್ಯಾದಿಗಳಿಗೆ ಇದು ಉಪಯುಕ್ತ.
- **ಟೆಂಪ್ಲೇಟ್ ರನ್‌ಟೈಮ್ ಪೂರ್ವಾಪೇಕ್ಷಿತಗಳು**: ಕೆಲವು ಟೆಂಪ್ಲೇಟುಗಳಿಗೆ Node.js, Python, Java, ಅಥವಾ Docker 같은 ಸ್ಥಳೀಯ ಉಪಕರಣಗಳು ಅಗತ್ಯವಿರಬಹುದು. ಪ್ರಾರಂಭಿಸುವ ಮೊದಲು ಸೆಟ್‌ಅಪ್ ವಾಲಿಡೇಟರ್ ಅನ್ನು ಓಡಿಸಿ zodat ನೀವು ಅಂಗವೈಕಲ್ಯಗಳನ್ನು ಮುಂಚಿತವಾಗಿ ಹಿಡಿಯಬಹುದು.
- **ಮೂಲಭೂತ ಟರ್ಮಿನಲ್ ಪರಿಚಯ**: ನೀವು ತಜ್ಞರಾಗಿರಬೇಕಾಗಿಲ್ಲ, ಆದರೆ `git clone`, `azd auth login`, ಮತ್ತು `azd up` போன்ற ಆಜ್ಞೆಗಳು ಚಾಲನೆ ಮಾಡಲು ಆರಾಮವಾಗಿ ಇರಬೇಕು.

> **ಉದ್ಯಮ ಚಂದಾದಾರಿಕೆಯಲ್ಲಿ ಕೆಲಸ ಮಾಡುತ್ತೀರಾ?**
> ನಿಮ್ಮ Azure ಪರಿಸರವನ್ನು ನಿರ್ವಾಹಕನೊಬ್ಬ ವ್ಯವಸ್ಥಾಪನೆ ಮಾಡುತ್ತಿದ್ದರೆ, ನೀವು ಬಳಸದ ಚಂದಾದಾರಿಕೆ ಅಥವಾ ಸಂಪನ್ಮೂಲ ಗುಂಪಿನಲ್ಲಿ ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿಯೋಜಿಸಲು ಸಾಧ್ಯವಿದೆಯೇ ಎಂದು ಮುಂಚಿತವಾಗಿ ದೃಢಪಡಿಸಿ. ಸಾಧ್ಯವಿಲ್ಲದಿದ್ದರೆ, ಪ್ರಾರಂಭಿಸುವ ಮೊದಲು ಸ್ಯಾಂಡ್‌ಬಾಕ್ಸ್ ಚಂದಾದಾರಿಕೆ ಅಥವಾ Contributor ಪ್ರವೇಶ ಕೇಳಿ.

> **Azure ಗೆ ಹೊಸವೀರಾ?**
> ನಿಮ್ಮ ಸ್ವಂತ Azure ಟ್ರಯಲ್ ಅಥವಾ ಪೇ-ಏಸ್-ಯು-ಗೋ ಚಂದಾದಾರಿಕೆಯಿಂದ (https://aka.ms/azurefreetrial) ಪ್ರಾರಂಭಿಸಿ zodat ನೀವು ತರಬೇತಿ ವ್ಯಾಯಾಮಗಳನ್ನು ಮುಕ್ತವಾಗಿ ಮುಕ್ತಾಯಿಸಬಹುದು ಮತ್ತು ಟೆನಂಟ್-ಮಟ್ಟದ ಅನುಮೋದನೆಗಳನ್ನು ಕಾದಿರಲು ಅಗತ್ಯವಿಲ್ಲ.

#### 💰 AI ನಿಯೋಜನೆಗಳಿಗಾಗಿ ವೆಚ್ಚ ಪರಿಗಣನೆಗಳು

**ಅಭಿವೃದ್ಧಿ ಪರಿಸರ (ಅಂದಾಜು $80-150/ತಿಂಗಳು):**
- Microsoft Foundry Models (Pay-as-you-go): $0-50/ತಿಂಗಳು (ಟೋಕನ್ ಬಳಕೆಯ ಆಧಾರದ ಮೇಲೆ)
- AI Search (Basic tier): $75/ತಿಂಗಳು
- Container Apps (Consumption): $0-20/ತಿಂಗಳು
- Storage (Standard): $1-5/ತಿಂಗಳು

**ಉತ್ಪಾದನಾ ಪರಿಸರ (ಅಂದಾಜು $300-3,500+/ತಿಂಗಳು):**
- Microsoft Foundry Models (PTU for consistent performance): $3,000+/ತಿಂಗಳು ಅಥವಾ ಹೆಚ್ಚಿನ ಬಳಕೆಯಿದ್ದರೆ Pay-as-you-go
- AI Search (Standard tier): $250/ತಿಂಗಳು
- Container Apps (Dedicated): $50-100/ತಿಂಗಳು
- Application Insights: $5-50/ತಿಂಗಳು
- Storage (Premium): $10-50/ತಿಂಗಳು

**💡 ವೆಚ್ಚ ಅನುಕೂಲೀಕರಣ ಸಲಹೆಗಳು:**
- ಕಲಿಕೆಗೆ Microsoft Foundry Models ನ **Free Tier** ಅನ್ನು ಬಳಸಿರಿ (Azure OpenAI 50,000 tokens/ತಿಂಗಳು ಸೇರಿದೆ)
- ಸಕ್ರಿಯವಾಗಿ ಅಭಿವೃದ್ಧಿ ನಡೆಸದಾಗ `azd down` ನಡೆಸಿ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಡೀಅಲೊಕೇಟ್ ಮಾಡಿ
- ಉತ್ಪಾದನೆಗೆ ಮಾತ್ರ PTU ಗೆ ಅಪ್‌ಗ್ರೇಡ್ ಮಾಡಿಕೊಳ್ಳಿ; ಪ್ರಾರಂಭದಲ್ಲಿ ಖರ್ಚು ಆಧಾರಿತ ಪಾವತಿ (consumption-based) ನ್ನೇ ಬಳಸಿರಿ
- ನಿಯೋಜನೆಯ ಮುನ್ನ ವೆಚ್ಚ ಅಂದಾಜು ಕಂಡುಹಿಡಿಯಲು `azd provision --preview` ಅನ್ನು ಬಳಸಿ
- ಆಟೋ-ಸ್ಕೇಲಿಂಗ್ ಸಕ್ರಿಯಗೊಳಿಸಿ: πραγματικά ಬಳಕೆಯಷ್ಟರಿಗಾಗಿ ಮಾತ್ರ ಪಾವತಿಸಿ

**ವೆಚ್ಚದ ಮೇಲ್ವಿಚಾರಣೆ:**

```bash
# Azure ಪೋರ್ಟಲ್‌ನಲ್ಲಿ ವಾಸ್ತವಿಕ ವೆಚ್ಚಗಳನ್ನು ನಿಗಾ ವಹಿಸಿ
```

## 📖 ಆಳವಾದ ವಿಶ್ಲೇಷಣೆ: AZD ಸಾಮರ್ಥ್ಯಗಳು

ಮೂಲಭೂತ ಮಾಹಿತಿಯನ್ನೇ ಮೀರಿ, AZD ಉತ್ಪಾದನೆ ನಿಯೋಜನೆಗಳಿಗಾಗಿ ಶಕ್ತಿಯುತ ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ:

- **ಟೆಂಪ್ಲೇಟು ಆಧಾರಿತ ನಿಯೋಜನೆಗಳು** - ಸಾಮಾನ್ಯ ಅಪ್ಲಿಕೇಶನ್ ಮಾದರಿಗಳಿಗೆ ಪೂರ್ವ-ನಿರ್ಮಿತ ಟೆಂಪ್ಲೇಟುಗಳನ್ನು ಬಳಸಿ
- **ಕೋಡ್ ಮೂಲಕ ಮೂಲಸೌಕರ್ಯ** - Bicep ಅಥವಾ Terraform ಬಳಸಿ Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿರ್ವಹಿಸಿ  
- **ಒಕ್ಕೂಟಿತ ವರ್ಕ್‌ಫ್ಲೋಗಳು** - ಅನುಗಢವಾಗಿ ಪ್ರೊವಿಷನ್, ನಿಯೋಜನೆ ಮತ್ತು ನಿಗಾ ವಹಿಸುವಿಕೆ ಮಾಡಿ
- **ವಿಕಾಸಕ-ಸೌಹಾರ್ದಯುತ** - ಅಭಿವೃದ್ಧಿಪಡಕರ ಉತ್ಪಾದಕತೆ ಮತ್ತು ಅನುಭವಕ್ಕಾಗಿ ಅತ್ಯಂತ ಸೂಕ್ತವಾಗಿಸಿದ್ದಾಗಿದೆ

### **AZD + Microsoft Foundry: AI ನಿಯೋಜನೆಗಳಿಗೆ ಸೂಕ್ತ**

**AI ಪರಿಹಾರಗಳಿಗೆ AZD ಯಾಕೆ?** AZD AI ಅಭಿವೃದ್ಧಿಪಡಕರು ಎದುರಿಸುವ ಪ್ರಮುಖ ಸವಾಲುಗಳನ್ನು ಪರಿಹರಿಸುತ್ತದೆ:

- **AI-ಸಿದ್ಧ ಟೆಂಪ್ಲೇಟ್‌ಗಳು** - Microsoft Foundry Models, Azure AI Services ಮತ್ತು ML ಕೆಲಸಭಾರಗಳಿಗಾಗಿ ಪೂರ್ವ-ಕಾನ್ಫಿಗರ್ ಮಾಡಿತ್ತ ಟೆಂಪ್ಲೇಟ್‌ಗಳು
- **ಭದ್ರತಾ AI ನಿಯೋಜನೆಗಳು** - AI ಸೇವೆಗಳು, API ಕೀಗಳು ಮತ್ತು ಮಾದರಿ ಎಂಡ್ಪಾಯಿಂಟ್‌ಗಳಿಗಾಗಿ ಒಳಗೊಳಿಸಿದ ಭದ್ರತಾ ರೂಪರೇಖೆಗಳು  
- **ಉತ್ಪಾದನಾ AI ರೂಪರೇಖೆಗಳು** - ವ್ಯಾಪಕ, ವೆಚ್ಚದೃಷ್ಟಿಯಿಂದ ಪರಿಣಾಮಕಾರಿಯಾದ AI ಅಪ್ಲಿಕೇಶನ್ ನಿಯೋಜನೆಗಳಿಗೆ ಉತ್ತಮ ಪದ್ಧತಿಗಳು
- **ಎಂಡ್-ಟು-ಎಂಡ್ AI ವರ್ಕ್‌ಫ್ಲೋಗಳು** - ಮಾದರಿ ಅಭಿವೃದ್ಧಿಯಿಂದ ಉತ್ಪಾದನೆ ನಿಯೋಜನೆ ವರೆಗೆ ಸಮಗ್ರ ನಿಗಾ ವ್ಯವಸ್ಥೆಯೊಂದಿಗೆ
- **ವೆಚ್ಚದ ಸೂಕ್ಷ್ಮೀಕರಣ** - AI ಕೆಲಸಭಾರಗಳಿಗೆ ಶ್ರೇಷ್ಠ ಸಂಪನ್ಮೂಲ ವಿತರಣೆ ಮತ್ತು ಸ್ಕೇಲಿಂಗ್ ತಂತ್ರಗಳು
- **Microsoft Foundry ಏಕೀಕರಣ** - Microsoft Foundry ಮಾದರಿ ಕ್ಯಾಟಲಾಗ್ ಮತ್ತು ಎಂಡ್ಪಾಯಿಂಟ್‌ಗಳಿಗೆ ಸರಳ ಸಂಪರ್ಕ

## 🔧 ತ್ವರಿತ ಸಮಸ್ಯೆ ಪರಿಹಾರದ ಮಾರ್ಗದರ್ಶಿ

**ಹುಡುಗಾಟ ಮಾಡುವವರು ಎದುರಿಸುವ ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳು ಮತ್ತು ತಕ್ಷಣದ ಪರಿಹಾರಗಳು:**

<details>
<summary><strong>❌ "azd: command not found"</strong></summary>

```bash
# ಮೊದಲೇ AZD ಅನ್ನು ಇನ್ಸ್ಟಾಲ್ ಮಾಡಿ
# ವಿಂಡೋಸ್ (PowerShell):
winget install microsoft.azd

# ಮ್ಯಾಕ್‌ಒಎಸ್:
brew tap azure/azd && brew install azd

# ಲಿನಕ್ಸ:
curl -fsSL https://aka.ms/install-azd.sh | bash

# ಸ್ಥಾಪನೆಯನ್ನು ಪರಿಶೀಲಿಸಿ
azd version
```
</details>

<details>
<summary><strong>❌ "No subscription found" or "Subscription not set"</strong></summary>

```bash
# ಲಭ್ಯವಿರುವ ಚಂದಾಗಳನ್ನು ಪಟ್ಟಿ ಮಾಡಿ
az account list --output table

# ಡೀಫಾಲ್ಟ್ ಚಂದೆಯನ್ನು ಆಯ್ಕೆಮಾಡಿ
az account set --subscription "<subscription-id-or-name>"

# AZD ಪರಿಸರಕ್ಕೆ ಹೊಂದಿಸಿ
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# ಪರಿಶೀಲಿಸಿ
az account show
```
</details>

<details>
<summary><strong>❌ "InsufficientQuota" or "Quota exceeded"</strong></summary>

```bash
# ಬೇರೆ Azure ಪ್ರದೇಶವನ್ನು ಪ್ರಯತ್ನಿಸಿ
azd env set AZURE_LOCATION "westus2"
azd up

# ಅಥವಾ ಅಭಿವೃದ್ಧಿಯಲ್ಲಿ ಸಣ್ಣ SKUಗಳನ್ನು ಬಳಸಿ
# infra/main.parameters.json ಅನ್ನು ಸಂಪಾದಿಸಿ:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>

<details>
<summary><strong>❌ "azd up" fails halfway through</strong></summary>

```bash
# ಆಯ್ಕೆ 1: ಶುಚಿಗೊಳಿಸಿ ಮತ್ತು ಮರುಪ್ರಯತ್ನಿಸಿ
azd down --force --purge
azd up

# ಆಯ್ಕೆ 2: ಕೇವಲ ಮೂಲಸೌಕರ್ಯವನ್ನು ಸರಿಪಡಿಸಿ
azd provision

# ಆಯ್ಕೆ 3: ವಿವರವಾದ ಸ್ಥಿತಿಯನ್ನು ಪರಿಶೀಲಿಸಿ
azd show

# ಆಯ್ಕೆ 4: Azure Monitor ನಲ್ಲಿ ಲಾಗ್‌ಗಳನ್ನು ಪರಿಶೀಲಿಸಿ
azd monitor --logs
```
</details>

<details>
<summary><strong>❌ "Authentication failed" or "Token expired"</strong></summary>

```bash
# AZDಗಾಗಿ ಮರುಪ್ರಮಾಣೀಕರಿಸಿ
azd auth logout
azd auth login

# ಐಚ್ಛಿಕ: ನೀವು az ಕಮಾಂಡ್‌ಗಳನ್ನು ನಡೆಸುತ್ತಿದ್ದರೆ Azure CLI-ನನ್ನೂ ತಾಜಾಗೊಳಿಸಿ
az logout
az login

# ಪ್ರಮಾಣೀಕರಣವನ್ನು ಪರಿಶೀಲಿಸಿ
az account show
```
</details>

<details>
<summary><strong>❌ "Resource already exists" or naming conflicts</strong></summary>

```bash
# AZD ಅನನ್ಯ ಹೆಸರುಗಳನ್ನು ರಚಿಸುತ್ತದೆ, ಆದರೆ ಸಂಘರ್ಷವಿದ್ದರೆ:
azd down --force --purge

# ಆಮೇಲೆ ಹೊಸ ಪರಿಸರದೊಂದಿಗೆ ಮರುಪ್ರಯತ್ನಿಸಿ
azd env new dev-v2
azd up
```
</details>

<details>
<summary><strong>❌ Template deployment taking too long</strong></summary>

**ಸಾಮಾನ್ಯ ಕಾಯುವ ಸಮಯಗಳು:**
- ಸರಳ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್: 5-10 ನಿಮಿಷ
- ಡೇಟಾಬೇಸ್ ಹೊಂದಿರುವ ಅಪ್ಲಿಕೇಶನ್: 10-15 ನಿಮಿಷ
- AI ಅಪ್ಲಿಕೇಶನ್ಗಳು: 15-25 ನಿಮಿಷಗಳು (OpenAI ಒದಗಿಸಿರುವುದು ನಿಧಾನವಾಗಬಹುದು)

```bash
# ಪ್ರಗತಿಯನ್ನು ಪರಿಶೀಲಿಸಿ
azd show

# ನಿಲುಕಿದ್ದು >30 ನಿಮಿಷಗಳು ಆಗಿದ್ದರೆ, Azure ಪೋರ್ಟಲ್ ಪರಿಶೀಲಿಸಿ:
azd monitor --overview
# ವಿಫಲವಾದ ನಿಯೋಜನೆಗಳನ್ನು ಹುಡುಕಿ
```
</details>

<details>
<summary><strong>❌ "Permission denied" or "Forbidden"</strong></summary>

```bash
# ನಿಮ್ಮ Azure ಪಾತ್ರವನ್ನು ಪರಿಶೀಲಿಸಿ
az role assignment list --assignee $(az account show --query user.name -o tsv)

# ನಿಮಗೆ ಕನಿಷ್ಠ "Contributor" ಪಾತ್ರ ಅಗತ್ಯವಿದೆ
# ನಿಮ್ಮ Azure ನಿರ್ವಾಹಕನಿಗೆ ಕೆಳಗಿನವುಗಳನ್ನು ಮಂಜೂರು ಮಾಡಲು ಕೇಳಿ:
# - Contributor (ಸಂಪನ್ಮೂಲಗಳಿಗಾಗಿ)
# - User Access Administrator (ಪಾತ್ರ ನಿಯೋಜನೆಗಳಿಗೆ)
```
</details>

<details>
<summary><strong>❌ Can't find deployed application URL</strong></summary>

```bash
# ಎಲ್ಲಾ ಸೇವಾ ಎಂಡ್‌ಪಾಯಿಂಟ್‌ಗಳನ್ನು ತೋರಿಸಿ
azd show

# ಅಥವಾ Azure ಪೋರ್ಟಲ್ ಅನ್ನು ತೆರೆಯಿರಿ
azd monitor

# ನಿರ್ದಿಷ್ಟ ಸೇವೆಯನ್ನು ಪರಿಶೀಲಿಸಿ
azd env get-values
# *_URL ಚರಗಳನ್ನು ಹುಡುಕಿ
```
</details>
