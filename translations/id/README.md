Kuasi Azure Developer CLI (azd) melalui bab terstruktur yang dirancang untuk pembelajaran progresif. **Fokus khusus pada penerapan aplikasi AI dengan integrasi Microsoft Foundry.**

- Arsitektur AI multi-layanan yang kompleks

- **Mengoptimalkan untuk Produksi**: Keamanan, penskalaan, pemantauan, dan pengelolaan biaya

## ğŸ“– Apa itu Azure Developer CLI?

Azure Developer CLI (azd) adalah antarmuka baris perintah yang berfokus pada pengembang untuk mempercepat proses membangun dan menempatkan aplikasi ke Azure. Ini menyediakan:

- **Penempatan berbasis template** - Gunakan template yang sudah dibuat untuk pola aplikasi umum
- **Infrastruktur sebagai Kode** - Kelola sumber daya Azure menggunakan Bicep atau Terraform  
- **Alur kerja terintegrasi** - Menyediakan provisioning, penempatan, dan pemantauan aplikasi dengan mulus
- **Ramah pengembang** - Dioptimalkan untuk produktivitas dan pengalaman pengembang

### **AZD + Microsoft Foundry: Sempurna untuk Penempatan AI**

**Mengapa AZD untuk Solusi AI?** AZD mengatasi tantangan utama yang dihadapi pengembang AI:

- **Template AI-Siap** - Template konfigurasi awal untuk Azure OpenAI, Cognitive Services, dan beban kerja ML
- **Penempatan AI Aman** - Pola keamanan bawaan untuk layanan AI, kunci API, dan endpoint model  
- **Pola AI Produksi** - Praktik terbaik untuk penempatan aplikasi AI yang skalabel dan hemat biaya
- **Alur Kerja AI End-to-End** - Dari pengembangan model hingga penempatan produksi dengan pemantauan yang tepat
- **Optimasi Biaya** - Alokasi sumber daya dan strategi penskalaan yang cerdas untuk beban kerja AI
- **Integrasi Microsoft Foundry** - Koneksi mulus dengan katalog model dan endpoint Microsoft Foundry

## ğŸ”§ Panduan Pemecahan Masalah Cepat

**Masalah umum yang dihadapi pemula dan solusi segera:**

### âŒ "azd: command not found"

```bash
# Instal AZD terlebih dahulu
# Windows (PowerShell):
winget install microsoft.azd

# macOS:
brew tap azure/azd && brew install azd

# Linux:
curl -fsSL https://aka.ms/install-azd.sh | bash

# Verifikasi instalasi
azd version
```

### âŒ "No subscription found" atau "Subscription not set"

```bash
# Daftar langganan yang tersedia
az account list --output table

# Atur langganan default
az account set --subscription "<subscription-id-or-name>"

# Tetapkan untuk lingkungan AZD
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# Verifikasi
az account show
```

### âŒ "InsufficientQuota" atau "Quota exceeded"

```bash
# Coba wilayah Azure yang berbeda
azd env set AZURE_LOCATION "westus2"
azd up

# Atau gunakan SKU yang lebih kecil dalam pengembangan
# Edit infra/main.parameters.json:
{
  "sku": "B1"  // Instead of "P1V2"
}
```

### âŒ "azd up" gagal di tengah jalan

```bash
# Opsi 1: Bersihkan dan coba lagi
azd down --force --purge
azd up

# Opsi 2: Hanya perbaiki infrastruktur
azd provision

# Opsi 3: Periksa log detil
azd show
azd logs
```

### âŒ "Authentication failed" atau "Token expired"

```bash
# Autentikasi ulang
az logout
az login

azd auth logout
azd auth login

# Verifikasi autentikasi
az account show
```

### âŒ "Resource already exists" atau konflik penamaan

```bash
# AZD menghasilkan nama unik, tetapi jika terjadi konflik:
azd down --force --purge

# Maka coba ulang dengan lingkungan baru
azd env new dev-v2
azd up
```

### âŒ Template deployment terlalu lama

**Waktu tunggu normal:**
- Aplikasi web sederhana: 5-10 menit
- Aplikasi dengan database: 10-15 menit
- Aplikasi AI: 15-25 menit (provisi OpenAI lambat)

```bash
# Periksa kemajuan
azd show

# Jika terhenti >30 menit, periksa Portal Azure:
azd monitor
# Cari penyebaran yang gagal
```

### âŒ "Permission denied" atau "Forbidden"

```bash
# Periksa peran Azure Anda
az role assignment list --assignee $(az account show --query user.name -o tsv)

# Anda memerlukan peran minimal "Kontributor"
# Minta admin Azure Anda untuk memberikan:
# - Kontributor (untuk sumber daya)
# - Administrator Akses Pengguna (untuk penugasan peran)
```

### âŒ Tidak dapat menemukan URL aplikasi yang sudah dideploy

```bash
# Tampilkan semua titik akhir layanan
azd show

# Atau buka Azure Portal
azd monitor

# Periksa layanan tertentu
azd env get-values
# Cari variabel *_URL
```
