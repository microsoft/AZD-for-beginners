# AZD ഉപയോഗിച്ച് ഒരു Microsoft SQL ഡാറ്റാബേസ് மற்றும் വെബ് ആപ്പ് ഡിപ്ലോയ് ചെയ്യുന്നത്

⏱️ **അനുമാനിച്ച സമയം**: 20-30 മിനിറ്റ് | 💰 **അനുമാനിച്ച ചെലവ്**: ~$15-25/മാസം | ⭐ **സങ്കീർണ്ണത**: ഇടവയpauമ

ഈ **സമ്പൂർണവും പ്രവർത്തനക്ഷമവുമായ ഉദാഹരണം** [Azure Developer CLI (azd)](https://learn.microsoft.com/azure/developer/azure-developer-cli/) ഉപയോഗിച്ച് Python Flask വെബ് അപ്ലിക്കേഷൻ ഒരു Microsoft SQL ഡാറ്റാബേസ് സഹിതം Azure-ലേക്ക് എങ്ങനെ ഡിപ്ലോയ് ചെയ്യാമെന്ന് വ്യക്തമാക്കുന്നു. എല്ലാ കോഡുകളും ഉൾപ്പെടുത്തിയതും പരിശോദിക്കപ്പെട്ടതുമാണ്—പുറം ആശ്രിതങ്ങൾ ആവശ്യമാണ്.

## നിങ്ങൾ പഠിക്കാനിരിക്കുന്നതെന്തെല്ലാം

ഈ ഉദാഹരണം പൂർത്തിയാക്കുന്നതിലൂടെ, നിങ്ങൾ:
- ഇൻഫ്രാസ്ട്രക്ചർ ആസ് കോഡ് ഉപയോഗിച്ച് ബഹു-ടിയർ അപ്ലിക്കേഷൻ (വെബ് ആപ്പ് + ഡാറ്റാബേസ്) ഡിപ്ലോയ് ചെയ്യുന്നു
- രഹസ്യങ്ങൾ ഹാർഡ്‌കോഡ് ചെയ്യാതെ സുരക്ഷിത ഡാറ്റാബേസ് കണക്ഷനുകൾ ക്രമീകരിക്കുന്നു
- Application Insights ഉപയോഗിച്ച് ആപ്ലിക്കേഷൻ ആരോഗ്യ നില നിരീക്ഷണം നടത്തുന്നു
- AZD CLI ഉപയോഗിച്ച് Azure റിസോർസുകൾ കാര്യക്ഷമമായി മാനേജ് ചെയ്യുന്നു
- സുരക്ഷ, ചെലവ് മെച്ചപ്പെടുത്തൽ, നിരീക്ഷണ സംബന്ധിച്ച Azure മികച്ച അഭ്യാസങ്ങൾ പാലിക്കുന്നു

## ദുരന്താവസ്ഥാ അവലോകനം
- **വെബ് ആപ്പ്**: ഡാറ്റാബേസ് കണക്ടിവിറ്റിയുള്ള Python Flask REST API
- **ഡാറ്റാബേസ്**: സാമ്പിൾ ഡാറ്റയോടെയുള്ള Azure SQL ഡാറ്റാബേസ്
- **ഇൻഫ്രാസ്ട്രക്ചർ**: Bicep ഉപയോഗിച്ച് പ്രൊവിഷൻ ചെയ്‌തത് (മോഡുലാർ, പുനരുപയോഗയോഗ്യ ടെംപ്ലേറ്റുകൾ)
- **ഡിപ്ലോയ്മെന്റ്**: `azd` കമാൻഡുകൾ ഉപയോഗിച്ച് പൂർണ ഓട്ടോമേറ്റഡ്
- **നിരീക്ഷണം**: ലോഗുകളും ടെലിമെട്രിയുമുള്ള Application Insights

## മുൻ‌അവസരങ്ങൾ

### ആവശ്യമായ ഉപകരണങ്ങൾ

തുടങ്ങുന്നതിന് മുമ്പ്, താഴെ പറയുന്ന ഉപകരണങ്ങൾ ഇൻസ്റ്റാൾ ചെയ്തിട്ടുണ്ടോ എന്ന് പരിശോധിക്കുക:

1. **[Azure CLI](https://learn.microsoft.com/cli/azure/install-azure-cli)** (വർഷൻ 2.50.0 അല്ലെങ്കിൽ അതിലും മേൽ)
   ```sh
   az --version
   # പ്രതീക്ഷിക്കുന്ന ഔട്ട്പുട്ട്: azure-cli 2.50.0 അല്ലെങ്കിൽ അതിനും മുകളിൽ
   ```

2. **[Azure Developer CLI (azd)](https://learn.microsoft.com/azure/developer/azure-developer-cli/install-azd)** (വർഷൻ 1.0.0 അല്ലെങ്കിൽ അതിലും മേൽ)
   ```sh
   azd version
   # പ്രതീക്ഷിക്കുന്ന പ്രവർത്തനഫലം: azd പതിപ്പ് 1.0.0 അല്ലെങ്കിൽ അതിൽ ഉയർന്നത്
   ```

3. **[Python 3.8+](https://www.python.org/downloads/)** (ലോകൽ വികസനത്തിന്)
   ```sh
   python --version
   # പ്രതീക്ഷിച്ച ഔട്ട്പുട്ട്: Python 3.8 അല്ലെങ്കിൽ അതിന് മുകളിൽ
   ```

4. **[Docker](https://www.docker.com/get-started)** (ഐച്ഛികം, ലോകൽ കണ്ടെയ്‌നറൈസ്ഡ് വികസനത്തിന്)
   ```sh
   docker --version
   # പ്രതീക്ഷിച്ച ഔട്ട്പുട്ട്: Docker പതിപ്പ് 20.10 അല്ലെങ്കിൽ അതിൽ മേൽ
   ```

## ആർക്കിടെക്ചർ

ഈ ഉദാഹരണം ഒരു രണ്ട്-ടിയർ ആർക്കിടെക്ചർ ഡിപ്ലോയ് ചെയ്യുന്നു, വെബ് അപ്ലിക്കേഷനും SQL ഡാറ്റാബേസും:

```
┌─────────────────┐        ┌──────────────────────┐
│  User Browser   │◄──────►│   Azure Web App      │
└─────────────────┘        │   (Flask API)        │
                           │   - /health          │
                           │   - /products        │
                           └──────────┬───────────┘
                                      │
                                      │ Secure Connection
                                      │ (Encrypted)
                                      │
                           ┌──────────▼───────────┐
                           │ Azure SQL Database   │
                           │   - Products table   │
                           │   - Sample data      │
                           └──────────────────────┘
```

**റിസോർസ് ഡിപ്ലോയ്മെന്റ്:**
- **റിസോർസ് ഗ്രൂപ്പ്**: എല്ലാ റിസോഴ്സുകളുടെയും കണ്ടെയ്‌നർ
- **ആപ്പ് സർവീസ് പ്ലാൻ**: ലിനക്സ് അടിസ്ഥാന ഹോസ്റ്റിംഗ് (ചെലവ് കുറയ്ക്കാൻ B1 ടിയർ)
- **വെബ് ആപ്പ്**: Python 3.11 റൺടൈം ഫ്ലാസ്ക് അപ്ലിക്കേഷൻ
- **SQL സർവർ**: TLS 1.2 ಕನಿಷ್ಠസഹിത നിയന്ത്രിത ഡാറ്റാബേസ് സർവർ
- **SQL ഡാറ്റാബേസ്**: അടിസ്ഥാന ടിയർ (2GB, വികസന/ടെസ്റ്റിങ്ങിന് അനുയോജ്യം)
- **അപ്ലിക്കേഷൻ ഇന്‍സൈറ്റ്സ്**: നിരീക്ഷണവും ലോഗിംഗും
- **ലോഗ് അനലിറ്റിക്സ് വർക്ക്‌സ്‌പെയ്‌സ്**: ലോഗുകൾ ശേഖരിക്കുന്ന കേന്ദ്രം

**ഉദാഹരണം**: ഇതിനെക്കാൾ നമുക്ക് ഒരു ഭക്ഷണശാല (വെബ് ആപ്പ്) ഉണ്ട്, അതിൽ ഒരു നടത്തിപ്പുണ്ടായുള്ള ടാൾക്ക് ഇൻ ഫ്രീസർ (ഡാറ്റാബേസ്). ഉപഭോക്താക്കൾ മെനുവിൽ നിന്ന് ഓർഡർ ചെയ്യുന്നു (API എൻഡ്പോയിന്റുകൾ), അടുക്കളം (ഫ്ലാസ്ക് ആപ്പ്) ഫ്രീസർ നിന്ന് ഘടകങ്ങൾ (ഡാറ്റ) തിരയുന്നു. റസ്റ്റോറന്റ് മാനേജർ (അപ്ലിക്കേഷൻ ഇന്‍സൈറ്റ്‌സ്) എല്ലാത്തും നിരീക്ഷിക്കുന്നു.

## ഫോൾഡർ ഘടന

ഈ ഉദാഹരണത്തിലുള്ള എല്ലാ ഫയലുകളും ഉൾപ്പെടുത്തിയിരിക്കുന്നു—പുറത്തുള്ള ആശ്രിതങ്ങൾ വേണ്ട:

```
examples/database-app/
│
├── README.md                    # This file
├── azure.yaml                   # AZD configuration file
├── .env.sample                  # Sample environment variables
├── .gitignore                   # Git ignore patterns
│
├── infra/                       # Infrastructure as Code (Bicep)
│   ├── main.bicep              # Main orchestration template
│   ├── abbreviations.json      # Azure naming conventions
│   └── resources/              # Modular resource templates
│       ├── sql-server.bicep    # SQL Server configuration
│       ├── sql-database.bicep  # Database configuration
│       ├── app-service-plan.bicep  # Hosting plan
│       ├── app-insights.bicep  # Monitoring setup
│       └── web-app.bicep       # Web application
│
└── src/
    └── web/                    # Application source code
        ├── app.py              # Flask REST API
        ├── requirements.txt    # Python dependencies
        └── Dockerfile          # Container definition
```

**ഓരോ ഫയലിന്റെയും ജോലി:**
- **azure.yaml**: AZD-യ്ക്ക് എന്ത് ഡിപ്ലോയ് ചെയ്യണം, എവിടെ എന്നാണ് बतിക്കുന്നത്
- **infra/main.bicep**: എല്ലാ Azure റിസോഴ്സുകളും ഏകോപിപ്പിക്കുന്നു
- **infra/resources/*.bicep**: വ്യത്യസ്ത റിസോഴ്സുകളുടെ നിർവചനങ്ങൾ (പുനരുപയോഗം സുതാര്യമായി)
- **src/web/app.py**: ഡാറ്റാബേസ് ലജിക് ഉള്ള ഫ്ലാസ്ക് അപ്ലിക്കേഷൻ
- **requirements.txt**: Python പാക്കേജ് ആശ്രിതങ്ങൾ
- **Dockerfile**: ഡിപ്ലോയ്മെന്റിനുള്ള കണ്ടെയ്‌നറൈസേഷൻ നിർദ്ദേശങ്ങൾ

## ക്വിക്ക്‌സ്റ്റാർട് (ഘട്ടംതുടർന്ന്)

### ഘട്ടം 1: ക്ലോൺ ചെയ്ത് നാവിഗേറ്റ് ചെയ്യുക

```sh
git clone https://github.com/microsoft/AZD-for-beginners.git
cd AZD-for-beginners/examples/database-app
```

**✓ വിജയം പരിശോധിക്കുക**: `azure.yaml` ഒപ്പം `infra/` ഫോൾഡർ കാണണം:
```sh
ls
# പ്രതീക്ഷിക്കുന്നത്: README.md, azure.yaml, infra/, src/
```

### ഘട്ടം 2: Azure-ൽ അംഗീകരിക്കുക

```sh
azd auth login
```

ഇത് നിങ്ങളുടെ ബ്രൗസർ തുറന്നു Azure അംഗീകാരം ആവശ്യപ്പെടും. നിങ്ങളുടെ Azure ക്രെഡൻഷ്യൽ ഉപയോഗിച്ച് സൈൻ ഇൻ ചെയ്യുക.

**✓ വിജയം പരിശോധിക്കുക**: താഴെ കാണണം:
```
Logged in to Azure.
```

### ഘട്ടം 3: പരിസ്ഥിതി ആരംഭിക്കുക

```sh
azd init
```

**ഏത് കാര്യങ്ങളാണ് നടക്കുന്നത്**: AZD നിങ്ങളുടെ ഡിപ്ലോയ്മെന്റിനായി ഒരു ലോക്കൽ കോൺഫിഗറേഷൻ സൃഷ്ടിക്കുന്നു.

**നിങ്ങൾ കാണാൻ പോകുന്ന പ്രോംപ്റ്റുകൾ:**
- **പരിസ്ഥിതി പേര്**: ചെറിയ പേര് നൽകുക (ഉദാ: `dev`, `myapp`)
- **Azure സബ്സ്ക്രിപ്‌ഷൻ**: ലിസ്റ്റിൽ നിന്ന് നിങ്ങളുടെ സബ്സ്ക്രിപ്‌ഷൻ തിരഞ്ഞെടുക്കുക
- **Azure സ്ഥാനം**: ഒരു റീജിയൻ തിരഞ്ഞെടുക്കുക (ഉദാ: `eastus`, `westeurope`)

**✓ വിജയ പരിശോധന**: അതുപോലെ കാണണം:
```
SUCCESS: New project initialized!
```

### ഘട്ടം 4: Azure റിസോഴ്സുകൾ പ്രൊവിഷൻ ചെയ്യുക

```sh
azd provision
```

**ഏത് കാര്യങ്ങളാണ് നടക്കുന്നത്**: AZD എല്ലാ ഇൻഫ്രാസ്ട്രക്ചറും ഡിപ്ലോയ് ചെയ്യുന്നു (5-8 മിനിറ്റ് വേണം):
1. റിസോർസ് ഗ്രൂപ്പ് സൃഷ്ടിക്കുന്നു
2. SQL സർവർ, ഡാറ്റാബേസ് സൃഷ്ടിക്കുന്നു
3. ആപ്പ് സർവീസ് പ്ലാൻ സൃഷ്ടിക്കുന്നു
4. വെബ് ആപ്പ് സൃഷ്ടിക്കുന്നു
5. അപ്ലിക്കേഷൻ ഇൻസൈറ്റ്‌സ് സൃഷ്ടിക്കുന്നു
6. നെറ്റ്‌വർക്ക്, സുരക്ഷ ക്രമീകരിക്കുന്നു

## കോൺഫിഗറേഷൻ രൂപകൽപ്പന

### പരിസ്ഥിതി വ്യത്യാസങ്ങൾ

രഹസ്യങ്ങൾ സുരക്ഷിതമായി Azure App Service കോൺഫിഗറേഷൻ വഴി മാനേജുചെയ്യുന്നു—**സോഴ്‌സ് കോഡിൽ കഠിനമായി ഹാർഡ്‌കോഡ് ചെയ്യരുത്**.

**AZD സ്വയം ക്രമീകരിക്കുന്നവ**:
- `SQL_CONNECTION_STRING`: എൻക്രിപ്റ്റ് ചെയ്ത ക്രെഡൻഷ്യലുകൾ അടങ്ങിയ ഡാറ്റാബേസ് കണക്ഷൻ
- `APPLICATIONINSIGHTS_CONNECTION_STRING`: നിരീക്ഷണ ടെലിമെറ്റ്രി എൻഡ്പോയിന്റ്
- `SCM_DO_BUILD_DURING_DEPLOYMENT`: സ്വയം ആശ്രിതങ്ങൾ ഇൻസ്റ്റാൾ ചെയ്യാൻ അനുവദിക്കുന്നു

**രഹസ്യങ്ങൾ എവിടെ സംഭരിക്കുന്നു**:
1. `azd provision` ഘട്ടത്തിൽ നിങ്ങൾ SQL ക്രെഡൻഷ്യലുകൾ നൽകുന്നു സുരക്ഷിത പ്രോംപ്റ്റുകൾ വഴി
2. AZD ഇവ നിങ്ങളുടെ ലോക്കൽ `.azure/<env-name>/.env` ഫയലിൽ സംരക്ഷിക്കുന്നു (git-യിൽ നിന്ന് ഒഴിവാക്കുന്നു)
3. AZD ഇവ Azure App Service കോൺഫിഗറേഷനിലേക്ക് ചേർക്കുന്നു (റസ്റ്റ്‌ ൽ എൻക്രിപ് ചെയ്ത)
4. ആപ്ലിക്കേഷൻ റൺടൈമിൽ `os.getenv()` വഴി ഇവ വായിക്കുന്നു

### ലോക്കൽ വികസനം

ലോകൽ ടെസ്റ്റിംഗിനായി ഒരു സാമ്പിൾ `.env` ഫയൽ സൃഷ്ടിക്കുക:

```sh
cp .env.sample .env
# നിങ്ങളുടെ പ്രാദേശിക ഡാറ്റാബേസാ കണക്ഷനുകൾ ഉപയോഗിച്ച് .env എഡിറ്റ് ചെയ്യുക
```

**ലോകൽ വികസന വർക്‌ഫ്ലോ**:
```sh
# ആശ്രിതങ്ങൾ ഇൻസ്റ്റാൾ ചെയ്യുക
cd src/web
pip install -r requirements.txt

# പരിസ്ഥിതി ചാരങ്ങളുടെ ക്രമീകരണം
export SQL_CONNECTION_STRING="your-local-connection-string"

# അപേക്ഷ പ്രവർത്തിപ്പിക്കുക
python app.py
```

**ലോകലായിൽ ടെസ്റ്റ് ചെയ്യുക**:
```sh
curl http://localhost:8000/health
# പ്രതീക്ഷിച്ചത്: {"സ്ഥിതി": "ആരോഗ്യവാനായി", "ഡാറ്റാബേസ്": "കണക്ട് ചെയ്തത്"}
```

1. **പഠനത്തിനായി സൗജന്യ ടിയർ ഉപയോഗിക്കുക**:
   - ആപ്പ് സർവീസ്: F1 ടിയർ (സൗജന്യം, പരിമിത മണിക്കൂറുകൾ)
   - SQL ഡാറ്റാബേസ്: Azure SQL ഡാറ്റാബേസ് സെർവർലെസ് ഉപയോഗിക്കുക
   - അപ്ലിക്കേഷൻ ഇൻസൈറ്റ്‌സ്: 5GB/മാസം സൗജന്യ ഇൻജെക്ഷൻ

2. **ഉപയോഗം ഇല്ലാതിരിക്കുമ്പോൾ റിസോഴ്‌സുകൾ നിർത്തുക**:
   ```sh
   # വെബ് ആപ്പ് അവസാനിപ്പിക്കുക (ഡേറ്റാബേസ് ചിലവുകൾ തുടരും)
   az webapp stop --name <app-name> --resource-group <rg-name>
   
   # ആവശ്യത്തിൽ പ്പോൾ പുനः ആരംഭിക്കുക
   az webapp start --name <app-name> --resource-group <rg-name>
   ```

3. **ടെസ്റ്റിങ്ങ് കഴിഞ്ഞാൽ എല്ലാം ഡിലീറ്റ് ചെയ്യുക**:
   ```sh
   azd down
   ```
   ഇത് എല്ലാ റിസോഴ്സുകളും നീക്കം ചെയ്ത് ചെലവുകൾ നിർത്തും.

4. **വികസനവും പ്രൊഡക്ഷനും സ്പെസിഫിക് SKUs**:
   - **വികസനം**: ഈ ഉദാഹരണത്തിൽ ഉപയോഗിച്ചബേസി ക ടിയർ
   - **പ്രൊഡക്ഷൻ**: സ്റ്റാൻഡേർഡ്/പ്രീമിയം ടിയർ കൂടിയ റിഡണ്ടൻസിയോടെ

**ചെലവ് നിരീക്ഷണം**:
- [Azure Cost Management](https://portal.azure.com/#view/Microsoft_Azure_CostManagement) ൽ ചെലവുകൾ കാണുക
- അനിഷ്ടാവസ്ഥ ഒഴിവാക്കാൻ ചെലവ് അലർട്ടുകൾ സജ്ജമാക്കുക
- ട്രാക്കിംഗിന് എല്ലാ റിസോഴ്‌സുകളെയും `azd-env-name` ടാഗ് ചെയ്യുക

**ഫ്രീ ടിയർ ബദൽ**:
പഠനാർത്ഥം, `infra/resources/app-service-plan.bicep` മാറ്റാം:
```bicep
sku: {
  name: 'F1'  // Free tier
  tier: 'Free'
}
```
**ശ്രദ്ധിക്കുക**: സൗജന്യ ടിയറിന് കുറഞ്ഞ സേവനപരിധികളുണ്ട് (പ്ര dnev CPU 60 മിനിറ്റ്, എപ്പോഴും ഓണല്ല).

## നിരീക്ഷണവും കാണത്തെയും

### അപ്ലിക്കേഷൻ ഇൻസൈറ്റ്‌സ് ഇന്റിഗ്രേഷൻ

ഈ ഉദാഹരണം സമഗ്രമായ നിരീക്ഷണത്തിന് **അപ്ലിക്കേഷൻ ഇൻസൈറ്റ്‌സ്** ഉൾപ്പെടുത്തുന്നു:

**നിരീക്ഷിക്കുന്നവ**:
- ✅ HTTP അഭ്യർത്ഥനകൾ (വിലാസം, നില കോഡുകൾ, എൻഡ്‌പോയിന്റുകൾ)
- ✅ അപ്ലിക്കേഷൻ പിശകുകളും പ്രത്യക്ഷ പാടങ്ങളും
- ✅ ഫ്ലാസ്ക് അപ്ലിക്കേഷനിൽ നിന്ന് കസ്റ്റം ലോഗിങ്ങ്
- ✅ ഡാറ്റാബേസ് കണക്ഷൻ ആരോഗ്യ നില
- ✅ പ്രകടന മീറ്ററുകൾ (CPU, മെമ്മറി)

**അപ്ലിക്കേഷൻ ഇൻസൈറ്റ്‌സ് ആക്‌സസ് ചെയ്യുക**:
1. [Azure പോർട്ടൽ](https://portal.azure.com) തുറക്കുക
2. നിങ്ങളുടെ റിസോഴ്‌സ് ഗ്രൂപ്പിലേക്ക് (`rg-<env-name>`) നാവിഗേറ്റ് ചെയ്യുക
3. അപ്ലിക്കേഷൻ ഇൻസൈറ്റ്‌സ് റിസോഴ്‌സ് (`appi-<unique-id>`) ക്ലിക്ക് ചെയ്യുക

**ഉപയോഗപ്രദമായ ക്വെറികൾ** (അപ്ലിക്കേഷൻ ഇൻസൈറ്റ്‌സ് → ലോഗുകൾ):

**എല്ലാ അഭ്യർത്ഥനകളും കാണുക**:
```kusto
requests
| where timestamp > ago(1h)
| order by timestamp desc
| project timestamp, name, url, resultCode, duration
```

**പിശകുകൾ കണ്ടെത്തുക**:
```kusto
exceptions
| where timestamp > ago(24h)
| order by timestamp desc
| project timestamp, type, outerMessage, operation_Name
```

**ഹെൽത്ത് എൻഡ്‌പോയിന്റ് പരിശോധിക്കുക**:
```kusto
requests
| where name contains "health"
| summarize count() by resultCode, bin(timestamp, 1h)
```

### SQL ഡാറ്റാബേസ് ഓഡിറ്റിങ്

**SQL ഡാറ്റാബേസ് ഓഡിറ്റിങ് സജീവമാണ്** ദിവസേന അതുകളിൽ:
- ഡാറ്റാബേസ് ആക്സസ് പാറ്റേണുകൾ
- പരാജയപ്പെട്ട ലോഗിൻ ശ്രമങ്ങൾ
- സ്കീമ മാറ്റങ്ങൾ
- ഡാറ്റ ആക്സസ് (അനുസരണത്തിന്)

**ഓഡിറ്റ് ലോഗുകൾ ആക്‌സസ് ചെയ്യുക**:
1. Azure പോർട്ടൽ → SQL ഡാറ്റാബേസ് → ഓഡിറ്റിങ്
2. ലോഗ് അനലിറ്റിക്സ് വർക്ക്‌സ്‌പെയ്‌സിൽ ലോഗുകൾ കാണുക

### റിയൽ ടൈം നിരീക്ഷണം

**ലൈവ് മെട്രിക്‌സ് കാണുക**:
1. അപ്ലിക്കേഷൻ ഇൻസൈറ്റ്‌സ് → ലൈവ് മെട്രിക്സ്
2. അഭ്യർത്ഥനകൾ, പരാജയങ്ങൾ, പ്രകടനം റിയൽ-ടൈമിൽ കാണുക

**അലർട്ടുകൾ സജ്ജമാക്കുക**:
ഗുരുതരമായ സംഭവങ്ങൾക്ക് അലർട്ടുകൾ:
- HTTP 500 പിശകുകൾ 5 മിനിറ്റിൽ 5 തവണയ്ക്ക് മുകളിൽ
- ഡാറ്റാബേസ് കണക്ഷൻ പരാജയങ്ങൾ
- ഉയർന്ന പ്രതികരണ സമയം (>2 സെക്കൻഡ്)

**ഉദാഹരണ അലർട്ട് സൃഷ്ടി**:
```sh
az monitor metrics alert create \
  --name "High-Response-Time" \
  --resource-group <rg-name> \
  --scopes <app-insights-resource-id> \
  --condition "avg requests/duration > 2000" \
  --description "Alert when response time exceeds 2 seconds"
```

### സാധാരണ പ്രശ്നങ്ങളും പരിഹാരങ്ങളും

#### 1. `azd provision` "Location not available" പ്രസ്താവനയുമായ് പരാജയപ്പെടുന്നു

**ലക്ഷണം**:
```
Error: The subscription is not registered for the resource type 'components' in the location 'centralus'.
```

**പരിഹാരം**:
മറ്റൊരു Azure മേഖല തിരഞ്ഞെടുക്കുക അല്ലെങ്കിൽ റിസോഴ്‌സ് പ്രൊവൈഡർ രജിസ്റ്റർ ചെയ്യുക:
```sh
az provider register --namespace Microsoft.Insights
```

#### 2. ഡിപ്ലോയ്മെന്റ് സമയത്ത് SQL ബന്ധം പരാജയപ്പെടുന്നു

**ലക്ഷണം**:
```
pyodbc.OperationalError: ('08001', '[08001] [Microsoft][ODBC Driver 18 for SQL Server]TCP Provider...')
```

**പരിഹാരം**:
- SQL സെർവർ ഫയർവാൾ Azure സര്‍വീസുകൾക്ക് അനുമതി നൽകുന്നുവെന്ന് പരിശോധിക്കുക (സ്വയം ക്രമീകരിക്കുന്നു)
- `azd provision` സമയത്ത് SQL അഡ്മിൻ പാസ്‌വേഡ് ശരിയായി നൽകിയിട്ടുണ്ടോ എന്ന് പരിശോധിക്കുക
- SQL സെർവർ പൂർണ്ണമായും പ്രൊവിഷൻ ചെയ്തിട്ടുണ്ടെന്ന് ഉറപ്പാക്കുക (2-3 മിനിറ്റ് സമയം വെക്കാം)

**ബന്ധം പരിശോധിക്കുക**:
```sh
# Azure പോർട്ടലിൽ നിന്നും, SQL ഡാറ്റാബേസ് → ക്വയറി എഡിറ്ററിലേക്ക് പോകുക
# നിങ്ങളുടെ വിവരങ്ങൾ ഉപയോഗിച്ച് കണക്റ്റ് ചെയ്യാൻ ശ്രമിക്കുക
```

#### 3. വെബ് ആപ്പിൽ "Application Error" കാണിക്കുന്നു

**ലക്ഷണം**:
ബ്രൗസർ പൊതുവായ പിശക് പേജു കാണിക്കുന്നു.

**പരിഹാരം**:
ആപ്ലിക്കേഷൻ ലോഗുകൾ പരിശോധിക്കുക:
```sh
# അടുത്തിടെ നടന്ന ലോഗുകൾ കാണുക
az webapp log tail --name <app-name> --resource-group <rg-name>
```

**സാധാരണ കാരണം**:
- സ്വയം പരിതസ്ഥിതിയുടെ പരിവർത്തനങ്ങൾ കാണാനില്ല (App Service → Configuration പരിശോധിക്കുക)
- Python പാക്കേജ് ഇൻസ്റ്റാളേഷൻ_FAIL_ (ഡിപ്ലോയ്മെന്റ് ലോഗുകൾ പരിശോധിക്കുക)
- ഡാറ്റാബേസ് ആരംഭിക്കൽ പിശക് (SQL കണക്ടിവിറ്റി പരിശോധിക്കുക)

#### 4. `azd deploy` "Build Error" എന്നാണ് പരാജയം

**ലക്ഷണം**:
```
Error: Failed to build project
```

**പരിഹാരം**:
- `requirements.txt` യിലെ സംവಿಧാന പിശകുകൾ ഇല്ലെന്ന് ഉറക്കുക
- `infra/resources/web-app.bicep` ൽ Python 3.11 വ്യക്തമാക്കിയിട്ടുണ്ടോ എന്നും പരിശോധിക്കുക
- Dockerfile-ൽ ശരിയായ ബേസ്സ്ഇമേജ് ഉപയോഗിച്ചിട്ടുണ്ടോ എന്നും പരിശോധിക്കുക

**സ്ഥാനാന്തരത്തിൽ ഡിബഗ് ചെയ്യുക**:
```sh
cd src/web
docker build -t test-app .
docker run -p 8000:8000 test-app
```

#### 5. AZD കമാൻഡുകൾ പ്രവർത്തിപ്പിക്കുമ്പോൾ "Unauthorized"

**ലക്ഷണം**:
```
ERROR: (Unauthorized) The client '<id>' with object id '<id>' does not have authorization
```

**പരിഹാരം**:

#### 6. ഉയർന്ന ഡാറ്റാബേസ് ചെലവുകൾ

**ലക്ഷണം**:
അപ്രതീക്ഷിത Azure ബിൽ.

**പരിഹാരം**:
- ടെസ്റ്റിംഗ് കഴിഞ്ഞ് `azd down` ഓടിക്കുവാൻ മറന്നിട്ടില്ലെന്ന് പരിശോധിക്കുക

### അഭ്യാസം 1: നിങ്ങളുടെ ഡിപ്ലോയ്മെന്റ് സ്ഥിരീകരിക്കുക (ആരംഭक)

**ലക്ഷ്യം**: എല്ലാ റിസോഴ്‌സുകളും ഡിപ്ലോയ് ചെയ്‌തിട്ടുണ്ട് എന്ന് ഉറപ്പാക്കുകയും ആപ്പ്ലിക്കേഷൻ പ്രവർത്തിക്കുന്നുണ്ടെന്നും പരിശോധിക്കുക.

   ```sh
   az resource list --resource-group rg-<env-name> --output table
   ```
   **പ്രതീക്ഷിതം**: 6-7 റിസോഴ്‌സുകൾ (Web App, SQL Server, SQL Database, App Service Plan, Application Insights, Log Analytics)

2. എല്ലാ API എൻഡ്‌പോയിന്റുകളും പരീക്ഷിക്കുക:
   ```sh
   curl https://app-<your-id>.azurewebsites.net/
   curl https://app-<your-id>.azurewebsites.net/health
   curl https://app-<your-id>.azurewebsites.net/products
   curl https://app-<your-id>.azurewebsites.net/products/1
   ```
   **പ്രതീക്ഷിതം**: എല്ലാ എൻഡ്‌പോയിന്റുകളും വാലിഡ് JSON തിരിച്ചു നൽകണം, പിശകുകൾ ഇല്ലാതെ

3. ആപ്ലിക്കേഷൻ ഇൻസൈറ്റ്സുകൾ പരിശോധിക്കുക:
   - Azure പോർട്ടലിൽ Application Insights-ലേക്ക് പോകുക
   - "Live Metrics" തിരഞ്ഞെടുക്കുക
   - വെബ് ആപ്പിൽ ബ്രൗസർ റിഫ്രഷ് ചെയ്യുക
   **പ്രതീക്ഷിതം**: റിയൽ ടൈമിൽ ആവശ്യങ്ങൾ കാണണം

**വിജയ വ്യവസ്ഥകൾ**: എല്ലാ 6-7 റിസോഴ്‌സുകളും ഉണ്ട്, എല്ലാ എൻഡ്‌പോയിന്റുകളും ഡാറ്റ നൽകുന്നു, Live Metrics സജീവമാണ്.

### അഭ്യാസം 2: പുതിയ API എൻഡ്‌പോയിന്റ് ചേർക്കുക (മധ്യനില)

**ലക്ഷ്യം**: Flask ആപ്പിൽ ഒരു പുതിയ എൻഡ്‌പോയിന്റ് ചേർക്കുക.

**ആരംഭ കോഡ്**: നിലവിലുള്ള എൻഡ്‌പോയിന്റുകൾ `src/web/app.py` ൽ

**പടികൾ**:
1. `src/web/app.py` എഡിറ്റ് ചെയ്ത് `get_product()` ഫങ്ഷനും പിന്നാലെ പുതിയ എൻഡ്‌പോയിന്റ് ചേർക്കുക:
   ```python
   @app.route('/products/search/<keyword>')
   def search_products(keyword):
       """Search products by name or description."""
       try:
           conn = get_db_connection()
           cursor = conn.cursor()
           cursor.execute(
               "SELECT id, name, description, price, created_at FROM products WHERE name LIKE ? OR description LIKE ?",
               (f'%{keyword}%', f'%{keyword}%')
           )
           
           products = []
           for row in cursor.fetchall():
               products.append({
                   'id': row[0],
                   'name': row[1],
                   'description': row[2],
                   'price': float(row[3]) if row[3] else None,
                   'created_at': row[4].isoformat() if row[4] else None
               })
           
           cursor.close()
           conn.close()
           
           logger.info(f"Search for '{keyword}' returned {len(products)} results")
           return jsonify(products), 200
           
       except Exception as e:
           logger.error(f"Error searching products: {str(e)}")
           return jsonify({'error': str(e)}), 500
   ```

2. അപ്ഡേറ്റ് ചെയ്ത ആപ്പ്ലിക്കേഷൻ ഡിപ്ലോയ് ചെയ്യുക:
   ```sh
   azd deploy
   ```

3. പുതിയ എൻഡ്‌പോയിന്റ് പരീക്ഷിക്കുക:
   ```sh
   curl https://app-<your-id>.azurewebsites.net/products/search/laptop
   ```
   **പ്രതീക്ഷിതം**: "laptop" അനുയോജ്യമായ ഉൽപ്പന്നങ്ങൾ തിരികെ നൽകണം

**വിജയ വ്യവസ്ഥകൾ**: പുതിയ എൻഡ്‌പോയിന്റ് പ്രവർത്തിക്കുന്നു, ഫിൽറ്റർ ചെയ്ത ഫലങ്ങൾ തിരികെ നൽകുന്നു, Application Insights ലോഗുകളിൽ കാണപ്പെടുന്നു.

### അഭ്യാസം 3: നിരീക്ഷണവും അലർട്ടുകളും ചേർക്കുക (ഊന്നൽ)

**ലക്ഷ്യം**: ആഗ്രഹിക്കപ്പെട്ട നിരീക്ഷണവും അലർട്ടുമുകൾ സജ്ജീകരിക്കുക.

**പടികൾ**:
1. HTTP 500 പിശക് അലർട്ട് സൃഷ്ടിക്കുക:
   ```sh
   # അപേക്ഷ ഇൻസൈറ്റ്‌സ് റിസോഴ്‌സ് ഐഡി നേടുക
   AI_ID=$(az monitor app-insights component show \
     --app appi-<your-id> \
     --resource-group rg-<env-name> \
     --query id -o tsv)
   
   # അലർട്ട് സൃഷ്‌ടിക്കുക
   az monitor metrics alert create \
     --name "High-Error-Rate" \
     --resource-group rg-<env-name> \
     --scopes $AI_ID \
     --condition "count requests/failed > 5" \
     --window-size 5m \
     --evaluation-frequency 1m \
     --description "Alert when >5 failed requests in 5 minutes"
   ```

2. പിശകുകൾ ഉണ്ടാക്കിയാണ് അലർട്ട് സജീവമാക്കുക:
   ```sh
   # നിലവിൽ ഇല്ലാത്ത ഒരു ഉൽപ്പന്നം അഭ്യർത്ഥിക്കുക
   for i in {1..10}; do curl https://app-<your-id>.azurewebsites.net/products/999; done
   ```

3. അലർട്ട് ഫയർ ചെയ്തിട്ടുണ്ടോ എന്ന് പരിശോധിക്കുക:
   - Azure പോർട്ടൽ → അലർട്ടുകൾ → അലർട്ട് റൂൾസ്
   - നിങ്ങളുടെ ഇമെയിൽ പരിശോധിക്കുക (കോൺഫിഗർ ചെയ്തിട്ടുണ്ടെങ്കിൽ)

**വിജയ വ്യവസ്ഥകൾ**: അലർട്ട് റൂൾ സൃഷ്ടിച്ചിരിക്കുന്നു, പിശകുകളിൽ ട്രിഗർ ചെയ്യുന്നു, അറിയിപ്പുകൾ ലഭിക്കുന്നു.

### അഭ്യാസം 4: ഡാറ്റാബേസ് സ്കീമ മാറ്റങ്ങൾ (ഊന്നൽ)

**ലക്ഷ്യം**: പുതിയ ഒരു പട്ടിക ചേർക്കുകയും ആപ്ലിക്കേഷൻ അതിനെ ഉപയോഗിക്കാൻ അപ്ഡേറ്റ് ചെയ്യുകയും ചെയ്യുക.

**പടികൾ**:
1. Azure പോർട്ടലിലെ Query എഡിറ്റർ വഴി SQL ഡാറ്റാബേസുമായി সংയോജിക്കുക

2. പുതിയ `categories` പട്ടിക സൃഷ്ടിക്കുക:
   ```sql
   CREATE TABLE categories (
       id INT PRIMARY KEY IDENTITY(1,1),
       name NVARCHAR(50) NOT NULL,
       description NVARCHAR(200)
   );
   
   INSERT INTO categories (name, description) VALUES
   ('Electronics', 'Electronic devices and accessories'),
   ('Office Supplies', 'Office equipment and supplies');
   
   -- Add category to products table
   ALTER TABLE products ADD category_id INT;
   UPDATE products SET category_id = 1; -- Set all to Electronics
   ```

3. `src/web/app.py` അപ്ഡേറ്റ് ചെയ്യുക, ഓരോ പ്രതികരണത്തിലും category വിവരങ്ങൾ ഉൾപ്പെടുത്തുക

4. ഡിപ്ലോയ്കർ കൂടി പരീക്ഷിക്കുക

**വിജയ വ്യവസ്ഥകൾ**: പുതിയ പട്ടിക ഉണ്ടാകും, ഉൽപ്പന്നങ്ങൾ category വിവരങ്ങളോടൊപ്പം കാണിക്കും, ആപ്പ് ഇപ്പോഴും പ്രവർത്തിക്കും.

### അഭ്യാസം 5: കാഷിംഗ് നടപ്പിലാക്കുക (തൊഴിൽനിപുണൻ)

**ലക്ഷ്യം**: പ്രകടനം മെച്ചപ്പെടുത്താൻ Azure Redis Cache ചേർക്കുക.

**പടികൾ**:
1. Redis Cache `infra/main.bicep` ൽ ചേർക്കുക
2. `src/web/app.py` അപ്ഡേറ്റ് ചെയ്ത് ഉൽപ്പന്ന ക്വെറികൾ കാഷ് ചെയ്യുക
3. Application Insights ഉപയോഗിച്ച് പ്രകടനം മെച്ചപ്പെട്ടതായി പരിശോധിക്കുക
4. കാഷിംഗ് മുൻപ് / ശേഷമുള്ള പ്രതികരണ സമയങ്ങൾ താരതമ്യം ചെയ്യുക

**വിജയ വ്യവസ്ഥകൾ**: Redis ഡിപ്ലോയ് ചെയ്തു, കാഷിംഗ് പ്രവർത്തിക്കുന്നു, പ്രതികരണ സമയം >50% മെച്ചപ്പെട്ടു.

**സൂചന**: ആരംഭിക്കുക [Azure Cache for Redis documentation](https://learn.microsoft.com/azure/azure-cache-for-redis/).

### മറ്റ് സമീപനങ്ങളുമായി താരതമ്യം

**AZD vs. ARM Templates**:
- ✅ AZD: ഉയർന്ന തലത്തിന്റെ സർവീസുകൾ, ലളിതമായ കമാൻഡുകൾ
- ⚠️ ARM: കൂടുതൽ വിശദമായ, സൂക്ഷ്മ നിയന്ത്രണം

**AZD vs. Terraform**:
- ✅ AZD: Azure-നാട്യവും Azure സേവനങ്ങളുമായി സംയോജിതവും
- ⚠️ Terraform: ബഹുമേഖല പിന്തുണ, വലിയ സമ്പ്രദായം

**AZD vs. Azure Portal**:
- ✅ AZD: ആവർത്തിക്കാൻ കഴിയുന്ന, പതിപ്പു നിയന്ത്രിതം, ഓട്ടോമേറ്റ് ചെയ്യാവുന്ന
- ⚠️ Portal: മാനുവൽ ക്ലിക്കുകൾ, പുനരാവർത്തനം പ്രയാസം

**AZD-യെ ചിന്തിക്കുക**: Docker Compose നെ പോലെ Azure-ക്കായി - സങ്കീര്‍ണ്ണമായ ഡിപ്ലോയ്മെന്റുകൾക്ക് ലളിതമായ കോൺഫിഗറേഷൻ.

## സാധാരണ ചോദിച്ചത് ചോദ്യങ്ങൾ

**Q: ഞാൻ വേറെ പ്രോഗ്രാമിംഗ് ഭാഷ ഉപയോഗിക്കാമോ?**  
A: തീർച്ചയായും! `src/web/` മാറ്റി Node.js, C#, Go, അല്ലെങ്കിൽ വേറെ ഭാഷ ഉപയോഗിക്കാം. `azure.yaml` മാറിച്ച് Bicep അപ്ഡേറ്റ് ചെയ്യുക.

**Q: കൂടുതൽ ഡാറ്റാബേസുകൾ ചേർക്കാൻ ആഗ്രഹം ഉണ്ടെങ്കിൽ?**  
A: `infra/main.bicep` ൽ മറ്റൊരു SQL Database മോഡ്യൂൾ ചേർക്കുക അല്ലെങ്കിൽ Azure ഡാറ്റാബേസ് സർവീസുകളിൽ നിന്നും PostgreSQL/MySQL ഉപയോഗിക്കുക.

**Q: പ്രൊഡക്ഷനായി ഇത് ഉപയോഗിക്കാമോ?**  
A: ഇത് ഒരു ആരംഭവിന്ദു മാത്രമാണ്. പ്രൊഡക്ഷനായി: Managed identity, Private endpoints, Redundancy, ബാക്കപ്പ് തന്ത്രം, WAF, മെച്ചപ്പെട്ട നിരീക്ഷണം എന്നിവ ചേർക്കുക.

**Q: കോഡ് ഡിപ്ലോയ്മെന്റ് ഒഴിവാക്കി കണ്ടെയ്നറുകൾ ഉപയോഗിക്കാമോ?**  
A: Docker കണ്ടെയ്നറുകൾ മുഴുവൻ ഉപയോഗിക്കുന്ന [Container Apps Example](../../../../examples/container-app) നോക്കുക.

**Q: എന്റെ ലോക്കൽ മെഷീനിൽ നിന്നു ഡാറ്റാബേസുമായി കണക്ട് ചെയ്യാൻ എങ്ങനെ?**  
A: നിങ്ങളുടെ IP SQL സെർവർ ഫയർവാൾ-ലേക്ക് ചേർക്കുക:
```sh
az sql server firewall-rule create \
  --resource-group rg-<env-name> \
  --server sql-<unique-id> \
  --name AllowMyIP \
  --start-ip-address <your-ip> \
  --end-ip-address <your-ip>
```

**Q: പുതിയ ഡാറ്റാബേസ് പരസ്യം ചെയ്യാതെ നിലവിലുള്ള ഡാറ്റാബേസ് ഉപയോഗിക്കാമോ?**  
A: തീർച്ചയായും, `infra/main.bicep` പ്രവർത്തിപ്പിച്ച് നിലവിലുള്ള SQL Server നെ സൂചിപ്പിച്ച് കണക്ഷൻ സ്ട്രിംഗ് പാരാമീറ്ററുകൾ അപ്ഡേറ്റ് ചെയ്യുക.
