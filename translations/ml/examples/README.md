# ഉദാഹരണങ്ങൾ - അഭ്യാസപരമായ AZD ടെംപ്ലേറ്റുകളും കോൺഫിഗറേഷനുകളും

**ഉദാഹരണത്തിലൂടെ പഠിക്കുക - അധ്യായമുകൾ അനുസരിച്ച് ക്രമീകരിച്ചത്**
- **📚 കോഴ്‌സ് ഹോം**: [AZD തുടങ്ങുന്നവർക്ക്](../README.md)
- **📖 അധ്യായ മാപ്പിംഗ്**: പഠന സങ്കീർണ്ണത പ്രകാരം ക്രമീകരിച്ച ഉദാഹരണങ്ങൾ
- **🚀 ലൊക്കൽ ഉദാഹരണം**: [റീട്ടെയിൽ മൾട്ടി-ഏജന്റ് സൊല്യൂഷൻ](retail-scenario.md)
- **🤖 ബാഹ്യ AI ഉദാഹരണങ്ങൾ**: Azure സാമ്പിൾ റപ്പോസിറ്ററികൾക്ക് ലിങ്കുകൾ

> **📍 പ്രധാന്യം: ലൊക്കൽ vs ബാഹ്യ ഉദാഹരണങ്ങൾ**  
> ഈ റപ്പോസിറ്ററിയിൽ **4 പൂർത്തിയായ ലൊക്കൽ ഉദാഹരണങ്ങൾ** പൂർണ്ണമായി ഉൾപ്പെടുത്തിയിട്ടുണ്ട്:  
> - **Microsoft Foundry Models Chat** (gpt-4.1 ഡീപ്ലോയ്മെന്റ് ചാറ്റ് ഇന്റർഫേസ് ഉൾപ്പെടുത്തി)  
> - **Container Apps** (സിംപ്ല് ഫ്ലാസ്ക് API + മൈക്രോസർവിസസുകൾ)  
> - **Database App** (വെബ് + SQL ഡാറ്റാബേസ്)  
> - **Retail Multi-Agent** (എന്റർപ്രൈസ് AI സൊല്യൂഷൻ)  
>  
> അധിക ഉദാഹരണങ്ങൾ **ബാഹ്യ റഫറൻസുകൾ** ആക്‌ഷൂർ-സാമ്പിൾസ് റപ്പോസിറ്ററികളിലേക്ക് ക്ലോൺ ചെയ്യാവുന്നതാണ്.

## പരിചയം

ഈ ഡയറക്ടറി പ്രായോഗിക ഉദാഹരണങ്ങളും റഫറൻസുകളും നൽകുന്നു, Azure Developer CLI കൈകാര്യം ചെയ്യുന്നത് കൈകൊണ്ട് പരിശീലിപ്പിക്കാൻ. Retail Multi-Agent സിനാരിയോ ഈ റപ്പോസിറ്ററിയിൽ ഉൾപ്പെടുത്തിയ പ്രൊഡക്ഷൻ റെഡിയായ പൂർണ്ണമായ ഇംപ്ലിമെന്റേഷൻ ആണ്. അധിക ഉദാഹരണങ്ങൾ ഔദ്യോഗിക Azure സാമ്പിൾസുകൾക്ക് റഫറൻസ് നൽകുന്നു, വ്യത്യസ്ത AZD പാറ്റേണുകൾ കാണിക്കുന്നതിനായി.

### സങ്കീർണ്ണത റേറ്റിങ് ലെജൻഡ്

- ⭐ **ആരംഭക** - അടിസ്ഥാന ആശയങ്ങൾ, ഒറ്റ സർവീസ്, 15-30 മിനിറ്റ്
- ⭐⭐ **മദ്ധ്യസ്ഥ** - നിരവധി സർവിസുകൾ, ഡാറ്റാബേസ് ഇന്റഗ്രേഷൻ, 30-60 മിനിറ്റ്
- ⭐⭐⭐ **ഉന്നത** - സങ്കീർണ്ണമായ ആർക്കിടെക്‌ചർ, AI ഇന്റഗ്രേഷൻ, 1-2 മണിക്കൂർ
- ⭐⭐⭐⭐ **നിപുണൻ** - പ്രൊഡക്ഷൻ റെഡിയായ, എന്റർപ്രൈസ് പാറ്റേണുകൾ, 2+ മണിക്കൂർ

## 🎯 ഈ റപ്പോസിറ്ററിയിൽ യഥാർത്ഥത്തിൽ എന്താണ് ഉള്ളത്

### ✅ ലൊക്കൽ ഇംപ്ലിമെന്റേഷൻ (ഉപയോഗിക്കാൻ തയ്യാറാണ്)

#### [Microsoft Foundry Models Chat അപ്ലിക്കേഷൻ](azure-openai-chat/README.md) 🆕
**ഈ റപ്പോസിറ്ററിയിൽ ഉള്‍പ്പെടുത്തിയ പൂർത്തിയായ gpt-4.1 ഡീപ്ലോയ്മെന്റ് ചാറ്റ് ഇന്റർഫേസ് സഹിതം**

- **സ്ഥലം:** `examples/azure-openai-chat/`
- **സങ്കീർണ്ണത:** ⭐⭐ (മദ്ധ്യസ്ഥ)
- **ഉൾപ്പെടുത്തിയതെല്ലാം:**
  - മുഴുവൻ Microsoft Foundry Models ഡീപ്ലോയ്മെന്റ് (gpt-4.1)
  - Python കമാൻഡ്-ലൈൻ ചാറ്റ് ഇന്റർഫേസ്
  - സുരക്ഷിത API കീകൾക്ക് കി വേൾട്ട് ഇന്റഗ്രേഷൻ
  - Bicep ഇൻഫ്രാസ്ട്രക്‌ചർ ടെംപ്ലേറ്റുകൾ
  - ടോക്കൺ ഉപയോഗവും ചിലവ് ട്രാക്കിംഗ്
  - റേറ്റ് ലിമിറ്റിംഗ്, എറർ ഹാൻഡ്ലിംഗ്

**ക്വിക് സ്റ്റാർട്ട്:**
```bash
# എക്സാംപിളിലേക്ക് നാവിഗേറ്റ് ചെയ്യുക
cd examples/azure-openai-chat

# എല്ലാം ഡിപ്പ്ലോയ് ചെയ്യുക
azd up

# ആശ്രിതങ്ങൾ ഇൻസ്റ്റാൾ ചെയ്ത് ചാറ്റ് ആരംഭിക്കുക
pip install -r src/requirements.txt
python src/chat.py
```

**സാങ്കേതികവിദ്യകൾ:** Microsoft Foundry Models, gpt-4.1, കി വേൾട്ട്, Python, Bicep

#### [Container App ഉദാഹരണങ്ങൾ](container-app/README.md) 🆕
**ഈ റപ്പോസിറ്ററിയിൽ ഉൾപ്പെടുത്തിയ സമഗ്രമായ കണ്ടെയ്‌നർ ഡീപ്ലോയ്മെന്റ് ഉദാഹരണങ്ങൾ**

- **സ്ഥലം:** `examples/container-app/`
- **സങ്കീർണ്ണത:** ⭐-⭐⭐⭐⭐ (ആരംഭക മുതൽ ഉന്നത വരെ)
- **ഉൾപ്പെടുത്തിയതെല്ലാം:**
  - [മാസ്റ്റർ ഗൈഡ്](container-app/README.md) - കണ്ടെയ്‌നർ ഡീപ്ലോയ്മെന്റുകളുടെയും അവലോകനം
  - [സിംപ്ല് ഫ്ലാസ്ക് API](../../../examples/container-app/simple-flask-api) - അടിസ്ഥാന REST API ഉദാഹരണം
  - [മൈക്രോസർവിസസ് ആർക്കിടെക്ചർ](../../../examples/container-app/microservices) - പ്രൊഡക്ഷൻ റെഡിയായ മൾട്ടി-സർവീസ് ഡീപ്ലോയ്മെന്റ്
  - ക്വിക് സ്റ്റാർട്ട്, പ്രൊഡക്ഷൻ, ഉന്നത പാറ്റേണുകൾ
  - মনിറ്ററിംഗ്, സുരക്ഷ, ചിലവു റെഡക്ഷൻ

**ക്വിക് സ്റ്റാർട്ട്:**
```bash
# മാസ്റ്റർ ഗൈഡ് കാണുക
cd examples/container-app

# ലളിതമായ Flask API വിന്യസിക്കുക
cd simple-flask-api
azd up

# മൈക്രോസർവീസുകളുടെ ഉദാഹരണം വിന്യസിക്കുക
cd ../microservices
azd up
```

**സാങ്കേതികവിദ്യകൾ:** Azure Container Apps, Docker, Python Flask, Node.js, C#, Go, Application Insights

#### [Retail Multi-Agent Solution](retail-scenario.md) 🆕

### ഉദാഹരണങ്ങൾ ലോക്കലായി പ്രവർത്തിപ്പിക്കൽ

1. **ഉദാഹരണം ക്ലോൺ ചെയ്യുക അല്ലെങ്കിൽ കോപ്പി ചെയ്യുക**  
   ```bash
   # ആഗ്രഹിക്കുന്ന ഉദാഹരണത്തിലേക്ക് പോകുക
   cd examples/simple-web-app
   ```
  
2. **AZD പരിസ്ഥിതി തുടക്കം**  
   ```bash
   # നിലവിലെ ടെംപ്ലറ്റുമായി ആരംഭിക്കുക
   azd init
   
   # അല്ലെങ്കിൽ പുതിയ പരിസ്ഥിതി സൃഷ്‌ടിക്കുക
   azd env new my-environment
   ```
  
3. **പരിസ്ഥിതി ക്രമീകരണം**  
   ```bash
   # ആവശ്യമായ വ്യത്യാസങ്ങൾ സജ്ജമാക്കുക
   azd env set AZURE_LOCATION eastus
   azd env set AZURE_SUBSCRIPTION_ID your-subscription-id
   ```
  
4. **വിന്യാസം നടത്തുക**  
   ```bash
   # ഇൻഫ്രാസ്ട്രക്ചറും അപ്ലിക്കേഷനും വിന്യസിക്കുക
   azd up
   ```
  
5. **വിന്യാസം സ്ഥിരീകരിക്കുക**  
   ```bash
   # സേവന എൻഡ്പോയിന്റുകൾ നേടുക
   azd env get-values
   
   # എൻഡ്പോയിന്റ് പരിശോധിക്കുക (ഉദാഹരണം)
   curl https://your-app-url.azurecontainer.io/health
   ```
   
   **പ്രതീക്ഷിക്കാവുന്ന വിജയ സൂചനകൾ:**  
   - ✅ `azd up` പിഴവുകൾ കൂടാതെ പൂർത്തീകരിക്കുക  
   - ✅ സർവീസ് എണ്ഡ്‌പോയിന്റിൽ HTTP 200 ലഭ്യമാകുക  
   - ✅ Azure പോർട്ടൽ "Running" നിലയിൽ കാണിക്കുക  
   - ✅ Application Insights ടെലിമെട്രി സ്വീകരിക്കുന്നത്  

> **⚠️ പ്രശ്‌നങ്ങളുണ്ടോ?** വിന്യാസ പ്രശ്‌നപരിഹാരത്തിനായി [Common Issues](../docs/chapter-07-troubleshooting/common-issues.md) കാണുക

   - Azure സർവീസ് ആവശ്യങ്ങൾ പരിശോധിക്കുക  
   - സബ്സ്ക്രിപ്ഷൻ പരിധികൾ സ്ഥിരീകരിക്കുക  
   - ചെലവ് അനാലിസിസ് മനസിലാക്കുക  

2. **കോൺഫിഗറേഷൻ മാറ്റം വരുത്തുക**  
   - `azure.yaml` സർവീസ് നിർണ്ണയങ്ങൾ പുതുക്കുക  
   - ബൈസപ് ടെംപ്ലേറ്റുകൾ ഇഷ്ടാനുസൃതമാക്കുക  
   - എൻവയോൺമെന്റ് വരിയബിളുകൾ ക്രമീകരിക്കുക  

3. **വിശദമായ പരിശോധന നടത്തുക**  
   - ആദ്യം ഡെവലപ്പ്മെന്റ് പരിസ്ഥിതിയിൽ വിന്യാസം നടത്തി  
   - ഫംഗ്ഷനാലിറ്റി പരിശോധിക്കുക  
   - സ്കെയ്ലിംഗും പ്രകടനവും പരീക്ഷിക്കുക  

4. **സുരക്ഷാ അവലോകനം**  
   - ആക്‌സസ് നിയന്ത്രണങ്ങൾ പരിശോധിക്കുക  
   - രഹസ്യ മാനേജ്മെന്റ് നടപ്പിലാക്കുക  
   - മേൽനോട്ടവും അലേർറ്റുകളും പ്രവർത്തനം സുഗമമാക്കുക

1. **പൂർണ نوവൻ?** → [Flask API](../../../examples/container-app/simple-flask-api) (⭐, 20 മിനിറ്റ്)കൊണ്ടുള്ള ആരംഭം
2. **അടിസ്ഥാന AZD അറിവുണ്ടോ?** → [Microservices](../../../examples/container-app/microservices) (⭐⭐⭐⭐, 60 മിനിറ്റ്) ശ്രമിക്കുക
3. **AI ആപ്പുകൾ നിർമ്മിക്കുന്നുണ്ടോ?** → [Microsoft Foundry Models Chat](../../../examples/azure-openai-chat) (⭐⭐, 35 മിനിറ്റ്) ഉപയോഗിച്ച് തുടങ്ങുക അല്ലെങ്കിൽ [Retail Multi-Agent](retail-scenario.md) (⭐⭐⭐⭐, 2+ മണിക്കൂർ) പരീക്ഷിക്കുക
4. **കുറഞ്ഞ പ്രത്യേക ടെക് സ്റ്റാക്ക് ആവശ്യമായോ?** → മുകളിൽ [Finding the Right Example](#-finding-the-right-example) വിഭാഗം ഉപയോഗിക്കുക
