# മൈക്രോസർവീസസ് ആർക്കിടെക്ചർ - കണ്ടെയ്‌നർ ആപ്പ് ഉദാഹരണം

⏱️ **അനുമാനිත സമയം**: 25-35 മിനിറ്റ് | 💰 **അനുമാനിത ചെലവ്**: ~$50-100/മാസം | ⭐ **സങ്കീർണത**: പ്രഗത്ഭത

AZD CLI ഉപയോഗിച്ച് Azure Container Apps-ലേക്ക് വിന്യസിച്ച ഒരു **സരളമാക്കിയിട്ടും പ്രവർത്തനക്ഷമമായ** മൈക്രോസർവീസസ് ആർക്കിടെക്ചർ. ഈ ഉദാഹരണം സേവനങ്ങളിലിടയിലെ ബന്ധം, കണ്ടെയ്‌നർ ഓർക്കസ്ട്രേഷൻ, മേൽനോട്ടം എന്നിവ പ്രായോഗികമായ 2-സെർവീസ് സെറ്റപ്പിനൊപ്പം കാണിക്കുന്നു.

> **📚 പഠന സമീപനം**: ഈ ഉദാഹരണം ഒരു മിനിമൽ 2-സെർവീസ് ആർക്കിടെക്ചർ (API ഗേറ്റ്വേ + ബാക്ക്‌എൻഡ് സേവനം)-ന്‍റെ ശൈലിയിലാകും തുടങ്ങിയിരിക്കുന്നത്, നിങ്ങള്ക്ക് ഇത് വിന്യസിക്കാനും പഠിക്കാനും സാധിക്കും. ഈ അടിസ്ഥാനമറിവ് കൈവരിച്ചതിനു ശേഷം, ഒരു പൂർണ്ണ മൈക്രോസർവീസസ് ഇക്കോസിസ്റ്റത്തിലേക്ക് വിപുലീകരിക്കാനുള്ള മാർഗനിർദ്ദേശം നാം നൽകുന്നു.

## നിങ്ങൾ പഠിക്കാനിരിക്കുന്നതു

ഈ ഉദാഹരണം പൂർത്തിയാക്കുന്നതിലൂടെ, നിങ്ങൾ:
- Azure Container Apps-ലേക്ക് ബഹുസ്ഥായിയായ കണ്ടെയ്‌നറുകൾ വിന്യസിക്കാൻ കഴിയും
- സൌകര്യമുള്ള ആന്തരിക നെറ്റ്‌വർകിംഗ് ഉപയോഗിച്ച് സേവനം-തുടർന്ന്-സേവനം (service-to-service) ബന്ധം നടപ്പാക്കാം
- പരിസ്ഥിതി അടിസ്ഥാന scaling, ഹെൽത്ത് ചെക്കുകൾ ക്രമീകരിക്കും
- Application Insights ഉപയോഗിച്ച് വിതരണം ചെയ്ത ആപ്ലിക്കേഷനുകൾ മേൽനോട്ടം വീക്ഷിക്കും
- മൈക്രോസർവീസസ് വിന്യാസ മാതൃകകളും മികച്ച രീതികളും മനസിലാക്കും
- ലളിതമായ ആർക്കിടെക്ചറുകളിൽ നിന്ന് സങ്കീർണ്ണത്തിലേക്കുള്ള പുരോഗമന വിപുലീകരണം പഠിക്കും

## ആർക്കിടെക്ചർ

### ഘട്ടം 1: ഞങ്ങൾ നിർമ്മിക്കുന്നതു (ഈ ഉദാഹരണത്തിൻ്റെ ഉൾക്കാഴ്‌ച്ചയിലുണ്ട്)

```

                    └──────────────┬──────────────┘
                                   │

                    ┌──────────────▼──────────────┐

                    │   - Routes requests         │
                    │   - Health checks           │
```

**സരളമാക്കാൻ എന്തു കാരണം?**
- ✅ വേഗത്തിൽ വിന്യസിക്കുകയും മനസ്സിലാക്കുകയും ചെയ്യുക (25-35 മിനിട്ട്)
- ✅ മൈക്രോസർവീസസ് പ്രാഥമിക ഗുണങ്ങൾ സങ്കീർണത കൂടാതെ പഠിക്കുക
- ✅ പ്രവർത്തിക്കുന്ന കോഡ് നിങ്ങൾക്ക് തിരുത്താനും പരീക്ഷിക്കാനുമുള്ളത്
- ✅ പഠനത്തിനുള്ള കുറഞ്ഞ ചെലവ് (~$50-100/മാസം യുഗമായ $300-1400/മാസത്തിനു പകരം)
- ✅ ഡാറ്റാബേസുകളും മെസേജ് ക്യൂയുകളും ചേർക്കുമ്പോൾ ആത്മവിശ്വാസം ഉണ്ടാക്കുക

**ഉദാഹരണം**: ഡ്രൈവിംഗ് പഠിക്കുന്ന രീതിയിലാണ് ഇത്. നിങ്ങൾ ഇരുട്ടുള്ള ഒരു പാർക്കിംഗ് ലോട്ട് (2 സേവനങ്ങൾ) ഉപയോഗിച്ച് തുടങ്ങുന്നു, അടിസ്ഥാനങ്ങൾ കൈവരിക്കുന്നു, പിന്നീട് നഗര ട്രാഫിക് (5+ സേവനങ്ങൾ ഡാറ്റാബേസുകളോടുകൂടെ) പഠിക്കും.

### ഘട്ടം 2: ഭാവി വിപുലീകരണം (റഫറൻസ് ആർക്കിടെക്ചർ)

അവസാനത്തെ "വിപുലീകരണ മാർഗ്‌ദർശകൻ" വിഭാഗം സ്ര്‍ദ്ദേശാനുസരിച്ച് നോക്കുക.

## ഉൾപ്പെട്ട സവിശേഷതകൾ

✅ **സേവനി വിദ്യാഭ്യാസം**: കണ്ടെയ്‌നറുകൾക്കിടയിലെ സ്വയം-സംഘടിത DNS അടിസ്ഥാനത്തിലുള്ള കണ്ടൽ  
✅ **ലോഡ് ബാലൻസിംഗ്**: റിപ്ലിക്കകൾക്കിടയിലെ നിർമാണ ഭാരം  
✅ **സ്വയം സ്കെയിലിംഗ്**: ഓരോ സേവനത്തിനും HTTP അഭ്യർത്ഥനകളുടെ അടിസ്ഥാനത്തിൽ സ്വതന്ത്യം സ്കെയിൽ ചെയ്യുക  
✅ **ഹെൽത്ത് മേൽനോട്ടം**: ഇരു സേവനങ്ങൾക്കും ജലദാരിത്വവും റെഡിനസ്സും പ്രോബുകൾ  
✅ **വിതരിച്ച ലോഗിംഗ്**: Application Insights ഉപയോഗിച്ച് കേന്ദ്രീകരിച്ച ലോഗിംഗ്  
✅ **ആന്തരിക നെറ്റ്‌വർകിംഗ്**: സുരക്ഷിതമായ സേവന-സേവനം സംവാദം  
✅ **കണ്ടെയ്‌നർ ഓർക്കസ്ട്രേഷൻ**: സ്വയം വിന്യസിക്കൽയും സ്കെയിലിംഗ്  
✅ **സീറോ-ഡൗൺടൈം അപ്‌ഡേറ്റുകൾ**: റോളിംഗ് അപ്‌ഡേറ്റുകൾ റിവിഷൻ മാനേജ്മെന്റോടെ

2. **[Azure CLI](https://learn.microsoft.com/cli/azure/install-azure-cli)** (പതിപ്പ് 2.50.0 അല്ലെങ്കിൽ അതിനും മുകളിൽ)
   ```bash
   az --version
   # പ്രതീക്ഷിക്കപ്പെട്ട ഫലം: azure-cli 2.50.0 അല്ലെങ്കിൽ അതിനുമുകളി
   ```

3. **[Docker](https://www.docker.com/get-started)** (പ്രാദേശിക വികസന / ടെസ്റ്റ് - ഐച്ഛികം)

   ```bash
   docker --version
   # പ്രതീക്ഷിക്കുന്ന ഔട്ട്‌പുട്ട്: Docker പതിപ്പ് 20.10 അല്ലെങ്കിൽ അതിൽ മുകളിൽ
   ```

## ദ്രുതാരംഭം (പടിയായി)

### പടി 1: ക്ലോൺ ചെയ്ത് തിരുകുക

```bash
git clone https://github.com/microsoft/AZD-for-beginners.git
cd AZD-for-beginners/examples/container-app/microservices
```

**✓ വിജയ പരിശോധന**: `azure.yaml` കാണുന്നുണ്ടെന്ന് ഉറപ്പാക്കുക:
```bash
ls
# പ്രതീക്ഷിക്കുന്നത്: README.md, azure.yaml, infra/, src/
```

### പടി 2: Azure-യിൽ അഥന്റിക്കേറ്റ് ചെയുക

```bash
azd auth login
```

നിങ്ങളുടെ ബ്രൗസർ തുറന്ന് Azure അഥന്റിക്കേഷൻ നടക്കും. നിങ്ങളുടെ ക്രെഡൻഷ്യലുകൾ ഉപയോഗിച്ച് സൈൻ ഇൻ ചെയ്യുക.

**✓ വിജയ പരിശോധന**: താഴെ കാണും:
```
Logged in to Azure.
```

### പടി 3: പരിസ്ഥിതി ഇൻഷ്യലൈസ് ചെയ്യുക

```bash
azd init
```

**നിങ്ങൾ കാണുന്ന പ്രോമ്പ്റ്റുകൾ**:
- **പരിസ്ഥിതി നാമം**: ഒരു ചെറു പേര് നൽകുക (ഉദാ: `microservices-dev`)
- **Azure സബ്സ്ക്രിപ്ഷൻ**: നിങ്ങളുടെ സബ്സ്ക്രിപ്ഷൻ തിരഞ്ഞെടുക്കുക
- **Azure സ്ഥലം**: ഒരു റീജിയൻ തിരഞ്ഞെടുക്കുക (ഉദാ: `eastus`, `westeurope`)

**✓ വിജയ പരിശോധന**: താഴെ കാണും:
```
SUCCESS: New project initialized!
```

### പടി 4: അടിസ്ഥാന സൗകര്യവും സേവനങ്ങളും വിന്യാസം

```bash
azd up
```

**എന്താണ് നടക്കുന്നത്** (8-12 മിനിറ്റ് വരെ):
1. കണ്ടെയ്‌നർ ആപ്പ്സ് പരിസ്ഥിതി സൃഷ്ടിക്കുന്നു
2. Application Insights മേൽനോട്ടത്തിനായി സജ്ജമാക്കുന്നു
3. API ഗേറ്റ്‌വേ കണ്ടെയ്‌നർ നിർമ്മിക്കുന്നു (Node.js)
4. പ്രോഡക്റ്റ് സേവൻ കണ്ടെയ്‌നർ നിർമ്മിക്കുന്നു (Python)
5. ಎರಡുപേരെയും Azure-ലേക്കും വിന്യസിക്കുന്നു
6. നെറ്റ്‌വർകിംഗ്, ഹെൽത്ത് ചെക്കുകൾ ക്രമീകരിക്കുന്നു
7. മേൽനോട്ടം, ലോഗിംഗ് സജ്ജീകരിക്കുന്നു

**✓ വിജയ പരിശോധന**: കാണുക:
```
SUCCESS: Your application was deployed to Azure in X minutes Y seconds.
Endpoint: https://api-gateway-<unique-id>.azurecontainerapps.io
```

**⏱️ സമയം**: 8-12 മിനിറ്റ്

## സേവന ബന്ധം മനസിലാക്കുക

### സേവനങ്ങൾ തമ്മിൽ എങ്ങനെ സംസാരിക്കുന്നു

ഈ ഉദാഹരണത്തിൽ API ഗേറ്റ്വേ പ്രോഡക്റ്റ് സേവനത്തോട് **ആന്തരിക HTTP വിളി** ഉപയോഗിച്ച് ബന്ധപ്പെടുന്നു:

```javascript
// എപി ഐ ഗേറ്റ്‌വേ (src/api-gateway/app.js)
const PRODUCT_SERVICE_URL = process.env.PRODUCT_SERVICE_URL;

// അന്തർഗത HTTP അഭ്യർത്ഥന നടത്തുക
const response = await axios.get(`${PRODUCT_SERVICE_URL}/products`);
```

**പ്രധാന ബിന്ദുക്കൾ**:

1. **DNS അടിസ്ഥാനത്തിലുള്ള കണ്ടെത്തൽ**: Container Apps സ്വയം ആന്തരിക സേവനങ്ങൾക്ക് DNS വീതികളുടെ സേവനം നൽകുന്നു
   - പ്രോഡക്റ്റ് സേവൻ FQDN: `product-service.internal.<environment>.azurecontainerapps.io`
   - ലളിതമാക്കി: `http://product-service` (Container Apps നിർവ്വചിക്കുന്നു)

2. **പബ്ലിക്ക് എക്സ്പോഷർ ഇല്ല**: പ്രോഡക്റ്റ് സേവനത്തിൻ്റെ Bicep-ൽ `external: false`
   - കണ്ടെയ്‌നർ ആപ്പ്സ് പരിസരത്തിനുള്ളിൽ മാത്രം ലഭ്യമാണ്
   - ഇന്റർനെറ്റിൽ നിന്ന് ആക്‌സസ് കഴിയില്ല

3. **പരിസ്ഥിതി വ്യത്യാസങ്ങൾ**: സെർവീസ് URLs വിന്യാസ സമയത്തു ഇൻജെക്ട് ചെയ്യുന്നു
   - Bicep ഗേറ്റ്വേയ്ക്കായി FQDN നൽകുന്നു
   - അപ്ലിക്കേഷൻ കോഡിൽ ഹാർഡ്‌കോഡുചെയ്‌തിരിക്കുക ഇല്ല

**ഉദാഹരണം**: ഓഫീസ് മുറികളായി ചിന്തിക്കുക. API ഗേറ്റ്വേ സ്വീകരണ ഡെസ്‌ക് (പബ്ലിക്), പ്രോഡക്റ്റ് സേവനം ഓഫീസ് മുറി (ആന്തരികം) ആണ്. സന്ദർശകർ ഓഫീസ് മുറികളിലേക്കു പോകാൻ സ്വീകരണമുറി വഴി പോകണം.

- കാരണം: ലഘുവായ ഇൻ-മെമ്മറി പ്രവർത്തനങ്ങൾ

### ഹെൽത്ത് ചെക്കുകൾ

ഇരു സേവനങ്ങളും ലൈവ്‌നെസ്സ്, റെഡിനസ് പ്രോബുകളിലുണ്ട്:

```bicep
probes: [
  {
    type: 'Liveness'
    httpGet: {
      path: '/health'
      port: 8080
    }
    initialDelaySeconds: 10
    periodSeconds: 30
  }
  {
    type: 'Readiness'
    httpGet: {
      path: '/health'
      port: 8080
    }
    initialDelaySeconds: 5
    periodSeconds: 10
  }
]
```

**ഇത് എന്താണ് അർത്ഥം**:
- **ലൈവ്‌നെസ്സ്**: ഹെൽത്ത് ചെക്ക് പരാജയപ്പെട്ടാൽ Container Apps കണ്ടെയ്‌നർ പുനരാരംഭിക്കും
- **റെഡിനസ്**: റെഡി അല്ലെങ്കിൽ ഇല്ലെങ്കിൽ Container Apps ആ റിപ്ലിക്കയ്ക്ക് ട്രാഫിക് റൂട്ടിംഗ് നിർത്തും

### Application Insights ചോദനകൾ

Azure പോർട്ടലിൽ Application Insights പ്രവേശിച്ച് താഴെ പറയുന്ന ചോദനകൾ ഓടിക്കുക:

**മന്ദഗതിയിലുള്ള അഭ്യർത്ഥനകൾ കണ്ടെത്തുക**:
```kusto
requests
| where timestamp > ago(1h)
| where duration > 1000  // Requests taking >1 second
| summarize count() by name, cloud_RoleName
| order by count_ desc
```

**സേവനം-സേവനം വിളികൾ ട്രാക്ക് ചെയ്യുക**:
```kusto
dependencies
| where timestamp > ago(1h)
| where type == "Http"
| project timestamp, name, target, duration, success
| order by timestamp desc
```

**സേവൻ അടിസ്ഥാന പിശക് നിരക്ക്**:
```kusto
exceptions
| where timestamp > ago(24h)
| summarize errorCount = count() by cloud_RoleName, type
| order by errorCount desc
```

**അഭ്യർത്ഥന വാള്യം സമയം അനുസരിച്ച്**:
```kusto
requests
| where timestamp > ago(1h)
| summarize requestCount = count() by bin(timestamp, 5m), cloud_RoleName
| render timechart
```

### മേൽനോട്ട ഡാഷ്ബോർഡ് പ്രവേശിക്കുക

```bash
  --query "appId" -o tsv
```

### ലൈവ് മെറ്റ്രിക്‌സ്

1. Azure പോർട്ടലിൽ Application Insights നാവിഗേറ്റ് ചെയ്യുക

3. റിയൽ-ടൈം അഭ്യർത്ഥനകൾ, പരാജയങ്ങൾ, പ്രകടനം കാണുക  
4. ഈ കമാൻഡ് റൺ ചെയ്ത് ശ്രമിക്കുക: `curl $(azd env get-values | grep API_GATEWAY_URL | cut -d '=' -f2 | tr -d '"')/api/products`

## വിപുലീകരണ മാർഗ്ഗരേഖ: 2 സർവിസിൽ നിന്ന് 5+ സർവിസുകളിലേക്ക്

ഈ 2-സർവീസ് ആർക്കിടെക്ചർ നന്നായി മനസിലാക്കിയ ശേഷം, വിപുലീകരിക്കാൻ പറ്റുന്നത്:

### ഘട്ടം 1: ഡാറ്റാബേസ് പേഴ്സിസ്റ്റൻസ് കൂട്ടിച്ചേർക്കുക (അടുത്ത പടി)

**പ്രോഡക്റ്റ് സർവീസിനായി Cosmos DB ചേർക്കുക**:

1. `infra/core/cosmos.bicep` സൃഷ്‌ടിക്കുക:
   ```bicep
   resource cosmosAccount 'Microsoft.DocumentDB/databaseAccounts@2023-04-15' = {
     name: name
     location: location
     kind: 'GlobalDocumentDB'
     properties: {
       databaseAccountOfferType: 'Standard'
       locations: [{ locationName: location, failoverPriority: 0 }]
     }
   }
   ```

2. പ്രോഡക്റ്റ് സർവീസ് in-memory ഡാറ്റയുടെ പകരം Cosmos DB ഉപയോഗിക്കുന്നുവെന്ന് അപ്ഡേറ്റ് ചെയ്യുക

3. കണക്കാക്കിയ അധിക ചെലവ്: ഏകദേശം $25/മാസം (serverless)

### ഘട്ടം 2: മൂന്നാം സർവീസ് (ഓർഡർ മാനേജ്മെന്റ്) ചേർക്കുക

**ഓർഡർ സർവീസ് സൃഷ്‌ടിക്കുക**:

1. പുതിയ ഫോൾഡർ: `src/order-service/` (Python/Node.js/C#)
2. പുതിയ Bicep: `infra/app/order-service.bicep`
3. API ഗേറ്റ്വേ `/api/orders` റൂട്ടിലേക്ക് അപ്ഡേറ്റ് ചെയ്യുക
4. ഓർഡർ പേഴ്സിസ്റ്റൻസിനായി Azure SQL Database ചേർക്കുക

**ആർക്കിടെക്ചർ ഇങ്ങനെ മാറും**:
```
API Gateway → Product Service (Cosmos DB)
           → Order Service (Azure SQL)
```

### ഘട്ടം 3: അസിങ്ക്രേണസ് സംവാദം ചേർക്കുക (Service Bus)

**ഇവന്റ്-ഡ്രിവൻ ആർക്കിടെക്ചർ നടപ്പിലാക്കുക**:

1. Azure Service Bus ചേർക്കുക: `infra/core/servicebus.bicep`
2. പ്രോഡക്റ്റ് സർവീസ് "ProductCreated" ഇവന്റ് പ്രസിദ്ധീകരിക്കുന്നത്
3. ഓർഡർ സർവീസ് പ്രോഡക്റ്റ് ഇവന്റുകൾക്ക് സസ്‌ക്രൈബ് ചെയ്യുക
4. ഇവന്റ് പ്രോസസ്സ് ചെയ്യാനായി Notification Service ചേർക്കുക

**പാറ്റേൺ**: Request/Response (HTTP) + Event-Driven (Service Bus)

### ഘട്ടം 4: ഉപയോക്തൃ അംഗീകാരം ചേർക്കുക

**യൂസർ സർവീസ് നടപ്പിലാക്കുക**:

1. `src/user-service/` സൃഷ്‌ടിക്കുക (Go/Node.js)
2. Azure AD B2C അല്ലെങ്കിൽ കസ്റ്റം JWT അംഗീകാര സംവിധാനം ചേർക്കുക
3. API ഗേറ്റ്വേ ടോക്കണുകൾ പരിശോധന നടത്തും
4. സർവീസുകൾ ഉപയോക്തൃ അനുമതികൾ പരിശോധിക്കും

### ഘട്ടം 5: പ്രൊഡക്ഷൻ റെഡിയ്നസ്

**ഇവ ചേർക്കുക**:
- Azure Front Door (ഗ്ലോബൽ ലോഡ്ബാലൻസിംഗ്)
- Azure Key Vault (സീക്രട്ട് മാനേജ്മെന്റ്)
- Azure Monitor Workbooks (കസ്റ്റം ഡാഷ്ബോർഡുകൾ)
- CI/CD പൈപ്പ്ലൈൻ (GitHub Actions)
- ബ്ലൂ-ഗ്രീൻ ഡിപ്ലോയ്മെന്റ്സ്
- എല്ലാം സർവിസുകൾക്കും Managed Identity

**പൂർണ്ണ പ്രൊഡക്ഷൻ ആർക്കിടെക്ചർ ചെലവ്**: ഏകദേശം $300-1,400/മാസം

- **ചെലവ്**: ഏകദേശം $60-250/മാസം

**കുബർണേറ്റിസ് (AKS)**:
- ✅ പരമാവധി നിയന്ത്രണവും ഫ്ലെക്സിബിലിറ്റിയും
- ✅ മൾട്ടി-ക്ലൗഡ് പോർട്ടബിൾ
- ✅ അഭിവൃദ്ധ vínculo നെറ്റ്വർക്കിംഗ്
- ❌ കുബർണേറ്റിസ് വിദഗ്ധത ആവശ്യമാണ്
- **ചെലവ്**: കുറഞ്ഞത് $150-500/മാസം

**ശുപാർശ**: Container Apps മുതൽ തുടക്കം വെക്കുക; കുബർണേറ്റിസ്-നിഷ്ഠ ഫീച്ചറുകൾ ആവശ്യമായപ്പോഴേ AKS ഉപയോഗിക്കുക.

## συνηθισμένες ερωτήσεις

**പ്ര:** 2 സർവീസ് മാത്രം, 5+ ആയി അല്ലാതിരിക്കാൻ എന്ത് കാരണം?  
ഉ: വിദ്യാഭ്യാസ പടിമുറി. സർവീസ് കമ്മ്യൂണിക്കേഷൻ, മോണിറ്ററിംഗ്, സ്കെയിലിംഗ് എന്നിങ്ങനെ അടിസ്ഥാന കാര്യങ്ങൾ ലളിതമായ ഉദാഹരണത്തോടെ ശ്രദ്ധിക്കുക. ഇവിടെ പഠിക്കുന്ന പാറ്റേണുകൾ 100 സര്‍വീസ് ആർക്കിടെക്ചറുകൾക്കും ബാധകമാണ്.

**പ്ര:** കൂടുതൽ സര്‍വിസുകൾ സ്വയം ചേർക്കാമോ?  
ഉ: തീർച്ചയായും! മുകളിലുള്ള വിപുലീകരണ മാർഗ്ഗരേഖ പിന്തുടരുക. ഓരോ പുതിയ സർവീസും src ഫോൾഡർ, Bicep ഫയൽ സൃഷ്‌ടികരിക്കുകയും azure.yaml അപ്ഡേറ്റ് ചെയ്യുകയും ഡിപ്ലോയ് ചെയ്യുക.

**പ്ര:** ഇത് പ്രൊഡക്ഷൻ റെഡി ആണോ?  
ഉ: ഹൃസ്വമായി പറഞ്ഞാൽ, നല്ല അടിസ്ഥാനമാണ്. പ്രൊഡക്ഷൻ-നായി managed identity, Key Vault, സ്ഥിരതയുള്ള ഡാറ്റാബേസുകൾ, CI/CD പൈപ്പ്ലൈൻ, മോണിറ്ററിംഗ് അലേർട്ടുകൾ, ബ്ഹ്യാക്കപ്പ് സ്റ്റ്രാറ്റജി ചേർക്കുക.

**പ്ര:** Dapr അല്ലെങ്കിൽ മറ്റൊരു സർവീസ് മെഷ് ഉപയോഗിക്കേണ്ടതില്ലേ?  
ഉ: പഠനത്തിനായി ലളിതമാക്കി വെയ്ക്കുക. നേറ്റീവ് Container Apps നെറ്റ്വർക്കിംഗ് മനസ്സിലാക്കിയശേഷം Dapr ഉൾപ്പെടുത്താം.

**പ്ര:** ലൊക്കലായി എങ്ങനെ ഡീബഗ് ചെയ്യാം?  
ഉ: Docker ഉപയോഗിച്ച് സർവിസുകൾ ലൊക്കലായി ഓടിക്കുക:
```bash
cd src/api-gateway
docker build -t local-gateway .
docker run -p 8080:8080 -e PRODUCT_SERVICE_URL=http://localhost:8000 local-gateway
```

**പ്ര:** വ്യത്യസ്ത പ്രോഗ്രാമിംഗ് ഭാഷകൾ ഉപയോഗിക്കാമോ?  
ഉ: തീർച്ച. ഈ ഉദാഹരണം Node.js (ഗേറ്റ്വേ) + Python (പ്രോഡക്റ്റ് സർവീസ്) ആണ്. കോൺറ്റെയ്നറിൽ ഓടുന്ന ഏതൊരു ഭാഷയും മിക്സും ചെയ്യാം.

**പ്ര:** Azure ക്രെഡിറ്റുകൾ ഇല്ലെങ്കിൽ?  
ഉ: Azure ഫ്രീ ടിയർ (പുതിയ അക്കൗണ്ടുകൾക്ക് ആദ്യ 30 ദിവസങ്ങൾ) ഉപയോഗിക്കുക അല്ലെങ്കിൽ ചെറിയ ടെസ്റ്റിംഗ് സമയത്ത് ഡിപ്ലോയ് ചെയ്ത് ഉടൻ ഡിലീറ്റ് ചെയ്യുക.
