## അവലോകനം

Azure Developer CLI ഉപയോഗിക്കുന്നപ്പോൾ നേരിടുന്ന സാധാരണ പ്രശ்னങ്ങൾ ഓർത്തു കണ്ടെത്താനും പരിഹരിക്കാനും ഈ അധ്യായം സഹായിക്കും. ഡിപ്ലോയ്മെന്റ് ഫെയില്യറുകളിൽ നിന്ന് AI-നു വശ്യമായ പ്രയാസങ്ങൾ വരെ.

> `azd 1.25.6` ന് വിരുദ്ധമായി 2026 ജൂണിൽ സ്ഥിരീകരിച്ചു.

## പഠനലക്ഷ്യങ്ങൾ

ഈ അധ്യായം പൂർത്തിയാക്കി, നിങ്ങൾ ചെയ്യും:
- സാധാരണ AZD ഡിപ്ലോയ്മെന്റ് പരാജയങ്ങൾ കണ്ടെത്തുക
- അത്നീക്റ്റികേഷൻ, അനുവാദ പ്രശ്നങ്ങൾ ഡീബഗ് ചെയ്യുക
- AI സർവീസ് കണക്ഷൻ പ്രശ്നങ്ങൾ പരിഹരിക്കുക
- പ്രശ്നപരിഹാരത്തിന് Azure പോർട്ടൽ, CLI ഉപയോഗിക്കുക

## 🚨 ത്വരിത പരിഹാരങ്ങൾ

### അത്നീക്റ്റികേഷൻ പ്രശ്നങ്ങൾ
```bash
# AZD വർക്‌ഫ്ലോകൾക്കായി ആവശ്യമാണ്
azd auth login

# നിങ്ങൾ നേരിട്ട് Azure CLI കമാൻഡുകളും ഉപയോഗിക്കുന്നുവെങ്കിൽ ഐച്ഛികമാണ്
az login

azd auth status
```

### പ്രൊവിഷനിംഗ് പരാജയങ്ങൾ
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### റിസോഴ്‌സ് മേധാവ്യത്വം
```bash
azd down --force --purge
azd env new different-name
azd up
```

### ക്വോട്ടാ മത്യസ്ഥം
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 പിഴവുകോഡ് സൂചിക

| പിഴവ് | കാരണം | പരിഹാരം |
|-------|-------|----------|
| `AuthenticationError` | പ്രവേശനം സജ്ജമല്ല | `azd auth login` |
| `ResourceNotFound` | റിസോഴ്‌സ് ഇല്ല | റിസോഴ്‌സ് നാമങ്ങൾ പരിശോധിക്കുക |
| `QuotaExceeded` | സബ്സ്ക്രിപ്ഷൻ ലിമിറ്റുകൾ | ക്വോട്ടാ വർധിപ്പിക്കാൻ അഭ്യർത്ഥിക്കുക |
| `InvalidTemplate` | Bicep സിന്‍ടാക്‌സ് പിഴവ് | `az bicep build` |
| `Conflict` | റിസോഴ്‌സ് നിലവിൽ ഉണ്ട് | പുതിയ പേരു ഉപയോഗിക്കുക അല്ലെങ്കിൽ ഇല്ലാതാക്കുക |
| `Forbidden` | അനിവാര്യമായ അനുമതികൾ ഇല്ല | RBAC റോളുകൾ പരിശോധിക്കുക |

## 🔄 റീസറ്റ് & വീഴ്ച്ച

```bash
# സോഫ്റ്റ് റീസെറ്റ് (സ്രോതസ്സുകൾ നിലനിർത്തുക, കോഡ് പുനഃപ്രവർത്തിപ്പിക്കുക)
azd deploy --force

# ഹാർഡ് റീസെറ്റ് (എല്ലാം അകറ്റുക, പുതുതായി ആരംഭിക്കുക)
azd down --force --purge
azd up
```
