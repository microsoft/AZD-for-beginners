# അദ്ധ്യായം 7: പ്രശ്നപരിഹാരം & ഡീബഗിംഗ്

**📚 കോഴ്‌സ്**: [AZD ആരംഭക്കാർക്ക്](../../README.md) | **⏱️ കാലാവധി**: 1-1.5 മണിക്കൂർ | **⭐ ബുദ്ധിമുട്ട്**: ഇടത്തരം

## അവലോകനം

ഈ അദ്ധ്യായം Azure Developer CLI ഉപയോഗിക്കുമ്പോഴുണ്ടാകുന്ന പൊതുവായ പ്രശ്നങ്ങൾ കണ്ടെത്തുകയും പരിഹരിക്കുകയും ചെയ്യാൻ സഹായിക്കും. ഡിപ്ലോയ്മെന്റ് പരാജയങ്ങളിൽ നിന്ന് AI-സംബന്ധമായ പ്രശ്നങ്ങളിലേക്കാണ് ഉൾപ്പെടുത്തലുകൾ.

## പഠന ലക്ഷ്യങ്ങൾ

ഈ അദ്ധ്യായം പൂർത്തിയാക്കിയാൽ, നിങ്ങൾക്ക് താഴെ ചെയ്യാനാകും:
- AZD ഡിപ്ലോയ്മെന്റ് പരാജയങ്ങൾ നിശ്ചയിക്കുക
- പ്രാമാണീകരണവും അനുവാദ പ്രശ്നങ്ങളും ഡീബഗ് ചെയ്യുക
- AI സേവന കണക്ടിവിറ്റി പ്രശ്നങ്ങൾ പരിഹരിക്കുക
- ട്രബിൾഷൂട്ടിംഗിനായി Azure പോർട്ടലും CLIയും ഉപയോഗിക്കുക

## 🚨 ത്വരിത പരിഹാരങ്ങൾ

### പ്രാമാണീകരണ പ്രശ്നങ്ങൾ
```bash
azd auth login
az login
azd auth whoami
```

### പ്രൊവിഷനിംഗ് പരാജയങ്ങൾ
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### റിസോഴ്‌സ് സംഘട്ടനങ്ങൾ
```bash
azd down --force --purge
azd env new different-name
azd up
```

### ക്വോട്ടാ മീതെ
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 പിശക് കോഡ് സൂചന

| പിശക് | കാരണം | പരിഹാരം |
|-------|-------|----------|
| `AuthenticationError` | ലോഗിൻ ചെയ്തിട്ടില്ല | `azd auth login` |
| `ResourceNotFound` | റിസോഴ്‌സ് കാണില്ല | റിസോഴ്‌സ് പേരുകൾ പരിശോധിക്കുക |
| `QuotaExceeded` | സബ്സ്ക്രിപ്ഷൻ പരിധികൾ | ക്വോട്ടാ വർദ്ധന ആവശ്യപ്പെടുക |
| `InvalidTemplate` | Bicep സിന്റാക്‌സ് പിഴവ് | `az bicep build` |
| `Conflict` | റിസോഴ്‌സ് നിലവിൽ ഉണ്ട് | പുതിയ പേര് ഉപയോഗിക്കുക അല്ലെങ്കിൽ നീക്കം ചെയ്യുക |
| `Forbidden` | അപര്യാപ്തമായ അനുവാദങ്ങൾ | RBAC റോളുകൾ പരിശോധിക്കുക |

## 🔄 റീസെറ്റ് & പുനരുദ്ധാനം

```bash
# സോഫ്റ്റ് റീസെറ്റ് (റിസോഴ്സുകൾ നിലനിർത്തുക, കോഡ് വീണ്ടും വിന്യസിക്കുക)
azd deploy --force

# ഹാർഡ് റീസെറ്റ് (എല്ലാം നീക്കം చేసి, പുതുതായി ആരംഭിക്കുക)
azd down --force --purge
azd up
```
