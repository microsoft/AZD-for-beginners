# അധ്യായം 7: പ്രശ്നപരിഹാരം & ഡീബഗിങ്

**📚 കോഴ്‌സ്**: [AZD For Beginners](../../README.md) | **⏱️ ദൈർഘ്യം**: 1-1.5 മണിക്കൂർ | **⭐ സങ്കീർണത**: ഇടത്തരം

## അവലോകനം

ഈ അധ്യായം Azure Developer CLI ഉപയോഗിക്കുമ്പോൾ ഒട്ടനവധി പ്രശ്നങ്ങൾ കണ്ടെത്താനും പരിഹരിക്കാനുമുള്ള സഹായം നൽകുന്നു. വിന്യസനം പരാജയങ്ങൾ മുതൽ AI-നിർദ്ദിഷ്ട പ്രശ്നങ്ങൾ വരെ.

> മാർച്ച് 2026-ൽ `azd 1.23.12` ശാസ്ത്രീയമായി പരിശോധന നടത്തി.

## പഠന ലക്ഷ്യങ്ങൾ

ഈ അധ്യായം പൂർത്തീകരിച്ച് നിങ്ങൾക്ക്:
- സാധാരണ AZD വിന്യസനം പരാജയങ്ങൾ കണ്ടെത്താൻ
- പ്രമാണപത്ര പരിശോധനയും അധികാര പ്രശ്നങ്ങളും ഡീബഗ് ചെയ്യാൻ
- AI സേവന കണക്ടിവിറ്റി പ്രശ്നങ്ങൾ പരിഹരിക്കാൻ
- പ്രശ്നപരിഹാരത്തിനായി Azure പോർട്ടലും CLIയും ഉപയോഗിക്കാൻ കഴിയും

## 🚨 ദ്രുത പരിഹാരങ്ങൾ

### അധികാര പരിശോധന പ്രശ്നങ്ങൾ
```bash
# AZD പ്രവൃത്തി പ്രവാഹങ്ങൾക്ക് ആവശ്യമാണ്
azd auth login

# നിങ്ങൾ നേരിട്ട് Azure CLI കമാൻഡുകളും ഉപയോഗിക്കുന്നുവെങ്കിൽ ഓപ്ഷണൽ ആണ്
az login

azd auth status
```

### വിന്യസന പരാജയങ്ങൾ
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### റിസോഴ്‌സ് സംഘർഷങ്ങൾ
```bash
azd down --force --purge
azd env new different-name
azd up
```

### ക്വോണ്ട അતીക്രമണം
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 പിശക് കോഡ് റഫറൻസ്

| പിശക് | കാരണം | പരിഹാരം |
|-------|-------|----------|
| `AuthenticationError` | ലോഗിൻ ചെയ്തിട്ടില്ല | `azd auth login` |
| `ResourceNotFound` | റിസോഴ്‌സ് ലഭ്യമല്ല | റിസോഴ്‌സ് നാമങ്ങൾ പരിശോധിക്കുക |
| `QuotaExceeded` | സബ്സ്ക്രിപ്ഷൻ പരിധികൾ തീർന്നിരിക്കുന്നു | ക്വോട്ട വർദ്ധിപ്പിക്കാൻ അപേക്ഷിക്കുക |
| `InvalidTemplate` | ബിസ്പ് സിന്താക്സ് പിഴവ് | `az bicep build` |
| `Conflict` | റിസോഴ്‌സ് നിലവിലുണ്ട് | പുതിയ പേരോ ഇല്ലാതാക്കലോ ഉപയോഗിക്കുക |
| `Forbidden` | മതിയായ അനുമതികൾ ഇല്ല | RBAC ചുമതലകൾ പരിശോധിക്കുക |

## 🔄 റീസെറ്റ് ആൻഡ് റിക്കവറി

```bash
# സോഫ്റ്റ് റീസെറ്റ് (സാധനങ്ങൾ നിലനിർത്തുക, കോഡ് പുനഃപ്രവർത്തിപ്പിക്കുക)
azd deploy --force

# ഹാർഡ് റീസെറ്റ് (എല്ലാം മായ്ക്കുക, പുതിയത് തുടങ്ങി)
azd down --force --purge
azd up
```
