## അവലോകനം

ഈ അധ്യായം Azure Developer CLI ഡിപ്ലോയ്മെന്റുകൾക്കുള്ള പരിസ്ഥിതി ക്രമീകരണം, പ്രാമാണീകരണ മാതൃകകൾ, ಮತ್ತು സുരക്ഷാ മികച്ച രീതികൾ ഉൾക്കൊള്ളുന്നു.

## പഠന ലക്ഷ്യങ്ങൾ

By completing this chapter, you will:
- AZD ക്രമീകരണ ഹൈറാർക്കിയെ ആഴത്തിൽ മനസ്സിലാക്കുക
- Manage multiple environments (dev, staging, prod)
- മാനേജ്ഡ് ഐഡന്റിറ്റികളോടെ സുരക്ഷിത പ്രാമാണീകരണം നടപ്പാക്കുക
- പരിസ്ഥിതി-നിര്‍ദിഷ്ട ക്രമീകരണങ്ങൾ ക്രമീകരിക്കുക

## 🚀 ക്വിക്ക് സ്റ്റാർട്ട്

```bash
# അനേകം പരിസ്ഥിതികൾ സൃഷ്ടിക്കുക
azd env new dev
azd env new staging
azd env new prod

# പരിസ്ഥിതികൾ മാറ്റുക
azd env select prod

# പരിസ്ഥിതി വേരിയബിൾസ് സജ്ജമാക്കുക
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# വിന്യാസം കാണുക
azd env get-values
```

## 🔧 ക്രമീകരണ അനുക്രമം

AZD ക്രമീകരണങ്ങൾ ഈ അനുക്രമത്തിൽ പ്രയോഗിക്കുന്നു (പിന്നിലുള്ളത് മുൻവന്നയെ മറികടക്കും):

1. **ഡീഫോൾട്ട് മൂല്യങ്ങൾ** (ടെംപ്ലേറ്റുകളിൽ നിർമ്മിച്ചിരിക്കുന്നു)
2. **azure.yaml** (പ്രോജക്റ്റ് ക്രമീകരണം)
3. **പരിസ്ഥിതി വേരിയബിളുകൾ** (`azd env set`)
4. **കമാൻഡ്-ലൈൻ ഫ്ലാഗുകൾ** (`--location eastus`)

---

## 🔐 സുരക്ഷാ മികച്ച രീതികൾ

```bash
# മാനേജ്ഡ് ഐഡന്റിറ്റി ഉപയോഗിക്കുക (ശുപാർശ ചെയ്യപ്പെടുന്നു)
azd env set AZURE_USE_MANAGED_IDENTITY true

# പ്രാമാണീകരണ നില പരിശോധിക്കുക
azd auth whoami
az account show

# ആവശ്യമെങ്കിൽ വീണ്ടും പ്രാമാണീകരിക്കുക
azd auth login
az login
```
