## അവലോകനം

ഈ അദ്ധ്യായം Azure Developer CLI വിനിയോഗങ്ങൾക്ക് പരിസര കോൺഫിഗറേഷൻ, ഓതന്റിക്കേഷൻ മാതൃകകൾ, സുരക്ഷാ മികച്ച രീതികൾ എന്നിവ നമ്മുടെ മുന്നയിലും വെക്കുന്നു.

> `azd 1.23.12`-നെ 2026 മാർച്ചിൽ സ്ഥിരീകരിച്ചിട്ടുണ്ട്.

## പഠന ലക്ഷ്യങ്ങൾ

ഈ അദ്ധ്യായം പൂർത്തിയാക്കിയാൽ, നിങ്ങളെക്കുട്ടി എത്തും:
- AZD കോൺഫിഗറേഷൻ ഹയർാർക്കിയ നിയന്ത്രണം അഭ്യസിക്കുക
- നിരവധി പരിസരങ്ങൾ (dev, staging, prod) കൈകാര്യം ചെയ്യുക
- മാനേജുചെയ്യുന്ന ഐഡന്റിറ്റികളുമായി സുരക്ഷിത ഓതന്റിക്കേഷൻ നടപ്പിലാക്കുക
- പരിസരനിര്ദ്ദിഷ്ട ക്രമീകരണങ്ങൾ കോൺഫിഗർ ചെയ്യുക

## 🚀 ക്വിക്ക് സ്റ്റാർട്ട്

```bash
# ബഹു പരിസരങ്ങൾ സൃഷ്ടിക്കുക
azd env new dev
azd env new staging
azd env new prod

# പരിസരങ്ങൾ മാറ്റുക
azd env select prod

# പരിസര પરિવർത്തനങ്ങൾ സജ്ജമാക്കുക
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# കോൺഫിഗറേഷൻ കാണുക
azd env get-values
```

## 🔧 കോൺഫിഗറേഷൻ ഹയർാർക്കി

AZD നിയമങ്ങൾ ഈ ക്രമത്തിൽ പ്രയോഗിക്കുന്നു (ശേഷമുള്ളതുകൊണ്ട് മുൻപുള്ളത് മറയ്ക്കും):

1. **ഡിഫോൾട്ട് മൂല്യങ്ങൾ** (ടെംപ്ലേറ്റുകളിൽ നിർമിച്ചതാണ്)
2. **azure.yaml** (പ്രോജക്ട് കോൺഫിഗറേഷൻ)
3. **പരിസര വ്യത്യാസങ്ങൾ** (`azd env set`)
4. **കമാൻഡ്-ലൈൻ ഫ്ലാഗുകൾ** (`--location eastus`)

## 🔐 സുരക്ഷé മികവ് പൂർത്തിവ്യവസ്ഥകൾ

```bash
# മാനേജ്ഡ് ഐഡന്റിറ്റി ഉപയോഗിക്കുക (ശുപാർശ ചെയ്യുന്നു)
azd env set AZURE_USE_MANAGED_IDENTITY true

# AZD പ്രാമാണീകരണ സ്ഥിതി പരിശോധന നടത്തുക
azd auth status

# ഐച്ഛികം: az കമാൻഡുകൾ പ്രവർത്തിപ്പിക്കാൻ പദ്ധതിയിടുകയാണെങ്കിൽ Azure CLI പാർശ്വം പരിശോധിക്കുക
az account show

# ആവശ്യമുണ്ടെങ്കിൽ മടങ്ങി പ്രാമാണീകരണം നടത്തുക
azd auth login

# ഐച്ഛികം: az കമാൻഡുകൾക്ക് Azure CLI പ്രാമാണീകരണം പുതുക്കുക
az login
```
