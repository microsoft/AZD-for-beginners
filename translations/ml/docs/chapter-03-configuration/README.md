# Chapter 3: ക്രമീകരണവും സ്ഥിരീകരണവും

**📚 കോഴ്സ്**: [AZD For Beginners](../../README.md) | **⏱️ ദൈർഘ്യം**: 45-60 മിനിറ്റ് | **⭐ സങ്കീർണ്ണത**: ഇന്റർമീഡിയറ്റ്

---

## അവലോകനം

ഈ അധ്യായം Azure Developer CLI ഡിപ്പ്ലോയ്മെന്റിനായുള്ള പരിസ്ഥിതി ക്രമീകരണം, സ്ഥിരീകരണ മാതൃകകൾ, സുരക്ഷാ മികച്ച പ്രവർത്തനരീതി എന്നിവ ഉൾക്കൊള്ളുന്നു.

> `azd 1.25.6` ജൂൺ 2026-ൽ പരിശോധന നടത്തി സ്ഥിരീകരിച്ചത്.

## പഠന ലക്ഷ്യങ്ങൾ

ഈ അധ്യായം പൂർത്തിയാക്കിയാൽ നിങ്ങൾക്ക്:
- AZD ക്രമീകരണ ഘടനയിൽ പ്രാവീണ്യം നേടാം
- നിരവധി പരിസ്ഥിതികളെ (ഡെവ്, സ്റ്റേജിംഗ്, പ്രോഡ്) നിയന്ത്രിക്കാൻ കഴിയും
- മാനേജ് ചെയ്ത ഐഡന്റിറ്റികളോടുള്ള സുരക്ഷിത സ്ഥിരീകരണം നടപ്പിലാക്കാം
- പരിസ്ഥിതി-സ്വഭാവമുള്ള ക്രമീകരണങ്ങൾ സജ്ജമാക്കാം

## 🚀 വേഗത്തിലുള്ള പ്രാരംഭം

```bash
# പല പരിതസ്ഥിതികളും സൃഷ്ടിക്കുക
azd env new dev
azd env new staging
azd env new prod

# പരിതസ്ഥിതികൾ മാറ്റുക
azd env select prod

# പരികുളം ചാരങ്ങൾ ക്രമീകരിക്കുക
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# കോൺഫിഗറേഷൻ കാണുക
azd env get-values
```

## 🔧 ക്രമീകരണ ഘടന

AZD സജ്ജീകരണങ്ങൾ താഴെ പറയുന്ന ക്രമത്തിൽ പ്രയോഗിക്കുന്നു (പിന്നീട് വരുന്നവ മുമ്പെത്തന്നെയെ മാറ്റിമറിക്കും):

1. **ഡീഫോൾട്ട് മൂല്യങ്ങൾ** (ടെംപ്ലേറ്റുകളിൽ നിർമ്മിച്ചിരിക്കുന്നു)
2. **azure.yaml** (പദ്ധതി ക്രമീകരണം)
3. **പരിസ്ഥിതി ചര മാസ്സേജ് വേരിയബിളുകൾ** (`azd env set`)
4. ** കമാൻഡ്-ലൈൻ ഫ്ലാഗുകൾ** (`--location eastus`)

## 🔐 സുരക്ഷാ മികച്ച പ്രവർത്തനരീതി

```bash
# നിയന്ത്രിക്കപ്പെട്ട ഐഡന്റിറ്റി ഉപയോഗിക്കുക (ശുപാർശ ചെയ്യും)
azd env set AZURE_USE_MANAGED_IDENTITY true

# AZD प्रमाणीकरण നില പരിശോധിക്കുക
azd auth status

# വൈകല്യം: az കമാൻഡുകൾ പ്രവർത്തിപ്പിക്കാൻ ആഗ്രഹിക്കുന്നുവെങ്കിൽ Azure CLI സാന്ദർഭ്യം പരിശോധിക്കുക
az account show

# ആവശ്യമായാൽ വീണ്ടും പ്രാമാണീകരണം നടത്തി
azd auth login

# വൈകല്യം: az കമാൻഡുകൾക്കായി Azure CLI प्रमाणीकरण പുതുക്കുക
az login
```
