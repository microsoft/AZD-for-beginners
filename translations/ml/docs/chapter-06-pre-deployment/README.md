# Chapter 6: മുൻ-ഡിപ്ലോയ്‌മെന്റ് പദ്ധതിയുണ്ടാക്കലും സ്ഥിരീകരണവും

**📚 കോഴ്സ്**: [AZD For Beginners](../../README.md) | **⏱️ ദൈർഘ്യം**: 1 മണിക്കൂർ | **⭐ സങ്കീർണ്ണത**: മധ്യസ്ഥത

---

## അവലോകനം

ഈ അദ്ധ്യായം നിങ്ങളുടെ ആപ്ലിക്കേഷൻ ഡിപ്ലോയ്മെന്റിന് മുമ്പ് നിർബന്ധമായ പദ്ധതീകരണവും സ്ഥിരീകരണവും ഉൾക്കൊള്ളുന്നു. ശരിയായ ക്യാപാസിറ്റി പ്ലാനിംഗ്, SKU തിരഞ്ഞെടുപ്പ്, പ്രിഫ്ലൈറ്റ് ചെക്കുകൾ എന്നിവയിലൂടെ ചെലവേറിയ പിശകുകൾ ഒഴിവാക്കുന്നത് പഠിക്കുക.

## പഠന ലക്ഷ്യങ്ങൾ

ഈ അദ്ധ്യായം പൂർത്തിയാക്കുന്നതിലൂടെ, നിങ്ങൾക്ക് താഴെ ചെയ്യാൻ കഴിയും:
- ഡിപ്ലോയ്മെന്റിന് മുമ്പ് പ്രിഫ്ലൈറ്റ് ചെക്കുകൾ നടത്തുക
- ക്യാപാസിറ്റി പദ്ധതിയിടുക һәм റിസോഴ്‌സ് ആവശ്യകതകൾ അനുമാനിക്കുക
- ചെലവ് ഒപ്റ്റിമൈസേഷനRoutes ന് അനുയോജ്യമായ SKUs തിരഞ്ഞെടുക്കുക
- മോണിറ്ററിംഗിനായി Application Insights ക്രമീകരിക്കുക
- ടീമിന്റെ കോഓർഡിനേഷൻ മാതൃകകൾ മനസിലാക്കുക

## 🚀 ശീഘ്രാരംഭം

```bash
# സബ്സ്ക്രിപ്ഷൻ ക്വോട്ടുകൾ പരിശോധിക്കുക
az vm list-usage --location eastus --output table

# ഡിപ്ലോയ്മെന്റ് മുൻദർശനം (റിസോഴ്സുകൾ സൃഷ്ടിക്കപ്പെടില്ല)
azd provision --preview

# Bicep സിന്ടാക്സ് പരിശോധിക്കുക
az bicep build --file infra/main.bicep

# പരിസ്ഥിതി ക്രമീകരണം പരിശോധിക്കുക
azd env get-values
```

## ☑️ മുൻ-ഡിപ്ലോയ്മെന്റ് ചെക്ക്ലിസ്റ്റ്

### `azd provision` ന് മുമ്പ്

- [ ] പ്രദേശത്തിന് ക്വോട്ട പരിശോധിച്ചു
- [ ] SKUs യോജ്യമായി തിരഞ്ഞെടുത്തു
- [ ] ചെലവ് അനുമാനം പരിശോധിച്ചു
- [ ] നാമകരണം പൊരുത്തപ്പെടുന്നു
- [ ] സുരക്ഷ/RBAC ക്രമീകരിച്ചു

### `azd deploy` ന് മുമ്പ്

- [ ] പരിസ്ഥിതി വേരിയബിളുകൾ സെറ്റ് ചെയ്തു
- [ ] രഹസ്യങ്ങൾ Key Vault-ൽ സംഭരിച്ചു
- [ ] കണക്ഷൻ സ്ട്രിംഗുകൾ പരിശോധിച്ചു
- [ ] ഹെൽത്ത് ചെക്കുകൾ ക്രമീകരിച്ചു
