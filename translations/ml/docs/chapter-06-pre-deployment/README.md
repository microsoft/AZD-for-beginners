# അദ്ധ്യായം 6: പ്രീ-ഡിപ്ലോയ്മെന്റ് പദ്ധതി & പരിശോധന

**📚 കോഴ്‌സ്**: [AZD For Beginners](../../README.md) | **⏱️ ദൈർഘ്യം**: 1 മണിക്കൂർ | **⭐ സങ്കീർണത**: മധ്യസ്ഥാനം

---

## അവലോകനം

നിങ്ങളുടെ ആപ്ലിക്കേഷൻ ഡിപ്ലോയ്മെന്റിന് മുമ്പ് അനിവാര്യമായ പദ്ധതി രൂപീകരണവും പരിശോധനയും ഈ അദ്ധ്യായം ഉൾക്കൊള്ളുന്നു. ശരിയായ ശേഷി പദ്ധതി, SKU തിരഞ്ഞെടുപ്പ്, പ്രീഫ്ലൈറ്റ് ചെക്കുകൾ എന്നിവയിലൂടെ ചെലവേറിയ പിഴവുകൾ ഒഴിവാക്കി പഠിക്കുക.

> `azd 1.25.6` ജൂൺ 2026-ൽ പരിശോധന ചെയ്തു.

## പഠന ലക്ഷ്യങ്ങൾ

ഈ അദ്ധ്യായം പൂർത്തിയാക്കുമ്പോൾ നിങ്ങൾക്ക് സാധിക്കും:
- ഡിപ്ലോയ്മെന്റിന് മുമ്പ് പ്രീഫ്ലൈറ്റ് ചെക്കുകൾ നടത്തുക
- ശേഷി പദ്ധതി രൂപീകരിച്ച് വിഭവങ്ങളുടെ ആവശ്യങ്ങൾ കണക്കുകൂട്ടുക
- ചെലവ് ലാഭത്തിനായി അനുയോജ്യമായ SKUകൾ തിരഞ്ഞെടുക്കുക
- സംരക്ഷണത്തിനായി അപ്ലിക്കേഷൻ ഇൻസൈറ്റ്‌സ് കോൺഫിഗർ ചെയ്യുക
- ടീമിന്റെ ഏകോപന മാതൃകകൾ മനസ്സിലാക്കുക

## 🚀 ക്വിക്ക് സ്റ്റാർട്ട്

```bash
# സബ്സ്ക്രിപ്ഷൻ കോറ്റാസ് പരിശോധിക്കുക
az vm list-usage --location eastus --output table

# ഡിപ്ലോയ്മെന്റ് മുൻദൃഷ്ടി (എന്തെങ്കിലും റിസോഴ്‌സുകൾ സൃഷ്ടിക്കില്ല)
azd provision --preview

# ബൈസെപ് സിന്തക്‌സ് സ്ഥിരീകരിക്കുക
az bicep build --file infra/main.bicep

# പരിസര ക്രമീകരണം പരിശോധിക്കുക
azd env get-values
```

## ☑️ പ്രീ-ഡിപ്ലോയ്മെന്റ് ചെക്ക്ലിസ്റ്റ്

### `azd provision` മുൻപ്

- [ ] പ്രദേശത്തിനുള്ള ക്വോട്ട സ്ഥിരീകരിച്ചു
- [ ] SKUകൾ അനുയോജ്യമായി തിരഞ്ഞെടുക്കപ്പെട്ടു
- [ ] ചെലവ് കണക്കുകൂട്ടൽ അവലോകനം ചെയ്തു
- [ ] നാമകരണം സാരവസ്തുവായി പാലിച്ചു
- [ ] സുരക്ഷ / RBAC കോൺഫിഗർ ചെയ്തു

### `azd deploy` മുൻപ്

- [ ] പരിസ്ഥിതി വ്യത്യാസങ്ങൾ സജ്ജമാക്കി
- [ ] കീ വോൾട്ടിൽ രഹസ്യങ്ങൾ ചേർത്തു
- [ ] കണക്ഷൻ സ്ട്രിംഗുകൾ സ്ഥിരീകരിച്ചു
- [ ] ഹെൽത്ത് ചെക്കുകൾ കോൺഫിഗർ ചെയ്തു
