# Chapter 6: മുൻ-പ്രവർത്തനം പദ്ധതി & സ്ഥിരീകരണം

**📚 കോഴ്സ്**: [AZD For Beginners](../../README.md) | **⏱️ കാലാവധി**: 1 മണിക്കൂർ | **⭐ സങ്കീർണത**: മധ്യస్థാനം

---

## അവലോകനം

ഈ അധ്യായം നിങ്ങളുടെ അപ്ലിക്കേഷൻ ഡെപ്ലോയ്മെന്റ് നടത്തുന്നതിനുമുൻപ് ആവശ്യമുള്ള പ്രധാന പദ്ധതി തയ്യാറെടുപ്പും സ്ഥിരീകരണ നടപടികളും ഉൾക്കൊള്ളുന്നു. ശരിയായ ശേഷി പദ്ധതി, SKU തിരഞ്ഞെടുപ്പ്, മുൻപരിശോധന എന്നിവയിലൂടെ ചെലവേറിയ പിഴവുകൾ ഒഴിവാക്കുക.

## പഠന ലക്ഷ്യങ്ങൾ

ഈ അധ്യായം പൂർത്തിയാക്കിയാൽ, നിങ്ങൾക്ക് സാധിക്കും:
- ഡെപ്ലോയ്മെന്റിന് മുൻപ് മുൻപരിശോധന നടത്തുക
- ശേഷി പദ്ധതി രൂപപ്പെടുത്തുകയും ആവശ്യമായ സ്രോതസ്സ് കണക്കുകൂട്ടുകയും ചെയ്യുക
- ചെലവ് പരമാവധി കുറയ്ക്കാൻ അനുയോജ്യമായ SKUകൾ തിരഞ്ഞെടുക്കുക
- മോനിറ്ററിംഗിനായി അപ്ലിക്കേഷൻ ഇൻസൈറ്റ്സ് കോൺഫിഗർ ചെയ്യുക
- ടീമിന്റെ നിരക്കുള്ള സമന്വയ മാതൃകകൾ മനസ്സിലാക്കുക

## 🚀 ദ്രുതാരംഭം

```bash
# സബ്സ്ക്രിപ്ഷൻ ക്വോട്ടകൾ പരിശോധിക്കുക
az vm list-usage --location eastus --output table

# ഡിപ്ലോയ്മെന്റ് പ്രിവ്യൂ (സ്രോതസ്സ് സൃഷ്ടിക്കപ്പെട്ടിട്ടില്ല)
azd provision --preview

# ബൈസപ് സിന്റാക്സ് പരിശോധന
az bicep build --file infra/main.bicep

# പരിസ്ഥിതി ക്രമീകരണം പരിശോധിക്കുക
azd env get-values
```

## ☑️ മുൻ-ഡെപ്ലോയ്മെന്റ് ചെക്ക്ലിസ്റ്റ്

### `azd provision` നടത്തുന്നതിനുമുമ്പ്

- [ ] മേഖലക്ക് ക്വോട്ട സ്ഥിരീകരിച്ചു
- [ ] SKUകൾ ശരിയായി തിരഞ്ഞെടുക്കിയത്
- [ ] ചെലവ് കണക്കുകൂട്ടൽ പരിശോധിച്ചു
- [ ] നാമവ്യവസ്ഥാ ഏകീകൃതം
- [ ] സുരക്ഷ/റോൾ അധിഷ്ഠിത ആക്‌സസ് നിയന്ത്രണം കോൺഫിഗർ ചെയ്തു

### `azd deploy` നടത്തുന്നതിനുമുമ്പ്

- [ ] പരിസ്ഥിതി വേരിയബിൾസുകൾ സജ്ജമാക്കി
- [ ] സീക്രെറ്റ്സ് കീ വോൾട്ടിൽ സൂക്ഷിച്ചു
- [ ] കണക്ഷൻ സ്ട്രിംഗുകൾ സ്ഥിരീകരിച്ചു
- [ ] ഹെൽത്ത് ചെക്കുകൾ കോൺഫിഗർ ചെയ്തു
