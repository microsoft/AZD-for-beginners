# അധ്യായം 6: മുന്‍നിര്‍ത്തിയുള്ള പ്ലാനിംഗ് & വെരിഫിക്കേഷന്‍

**📚 കോഴ്‍സ്**: [AZD For Beginners](../../README.md) | **⏱️ ദൈര്‍ഘ്യം**: 1 മണിക്കൂര്‍ | **⭐ സങ്കീര്‍ണത**: മധ്യേ

---

## അവലോകനം

നിങ്ങളുടെ ആപ്പ്ലിക്കേഷന്‍ പ്രക്ഷേപണത്തിന് മുമ്പ് ആവശ്യമുള്ള പ്രധാന പ്ലാനിംഗ്, വെരിഫിക്കേഷന്‍ ഘട്ടങ്ങളെക്കുറിച്ചാണ് ഈ അധ്യായം. ശരിയായ ശേഷി പ്ലാനിംഗും SKU തിരഞ്ഞെടുപ്പും പ്രീഫ്ലൈറ്റ് ചെക്കുകളും വഴികാട്ടിയാകുന്നതിലൂടെ ചെലവേറിയ പിശകുകള്‍ ഒഴിവാക്കുക.

> `azd 1.27.1` യുമായി ജൂലൈ 2026ിൽ സാധൂകരിച്ചു.

## പഠന ലക്ഷ്യങ്ങള്‍

ഈ അധ്യായം പൂര്‍ത്തിയാക്കിയാൽ നിങ്ങൾക്ക്:
- പ്രക്ഷേപണത്തിന് മുമ്പ് പ്രീഫ്ലൈറ്റ് ചെക്കുകള്‍ നടത്താന്‍ കഴിയും
- ശേഷി പദ്ധതി തയ്യാറാക്കാനും വിഭവ ആവശ്യമെന്നും കണക്കുകൂട്ടാനും കഴിയും
- ചെലവ് മെച്ചപ്പെടുത്താനുള്ള അനുയോജ്യമായ SKUകള്‍ തിരഞ്ഞെടുക്കാം
- മോണിറ്ററിംഗിനായി ആപ്ലിക്കേഷൻ ഇൻസൈറ്റ്സ് ക്രമീകരിക്കാം
- ടീം കോര്‍ഡിനേഷൻ പാറ്റേണുകള്‍ മനസ്സിലാക്കാം

## 🚀 വേഗത്തിലുള്ള ആരംഭം

```bash
# സബ്സ്ക്രിപ്ഷൻ ക്വോട്ടകൾ പരിശോദിക്കുക
az vm list-usage --location eastus --output table

# വിന്യസനം മുൻദൃശ്യം (വനക്കങ്ങൾ സൃഷ്ടിക്കപ്പെടുന്നില്ല)
azd provision --preview

# ബൈസിപ്പ് സിന್ಟാക്സുകൾ സാധൂകരിക്കുക
az bicep build --file infra/main.bicep

# പരിസര ക്രമീകരണം പരിശോദിക്കുക
azd env get-values
```

## ☑️ പ്രക്ഷേപണത്തിന് മുന്‍പുള്ള ചെക്ക് ലിസ്റ്റ്

### `azd provision` മുന്നോട്ട് പോകുന്നതിന് മുമ്പ്

- [ ] മേഖലയുടെ ക്വോട്ട പരിശോധിച്ചു
- [ ] SKUകള്‍ ശരിയായി തിരഞ്ഞെടുക്കപ്പെട്ടത്
- [ ] ചെലവ് കണക്കുകൂട്ടൽ അവലോകനം ചെയ്‌തു
- [ ] നാമകരണം ചട്ടം പൊരുത്തപ്പെടുന്നു
- [ ] സുരക്ഷ / RBAC ക്രമീകരിച്ചു

### `azd deploy` മുമ്പ്

- [ ] പരിസ്ഥിതി വേരിയബിളുകള്‍ ക്രമീകരിച്ചു
- [ ] കീ വാൾട്ടിൽ രഹസ്യങ്ങൾ സൂക്ഷിച്ചു
- [ ] കണക്ഷന്‍ സ്ട്രിങ്‌സ് പരിശോധന ചെയ്തു
- [ ] ആരോഗ്യ പരിശോധന ക്രമീകരിച്ചത്
