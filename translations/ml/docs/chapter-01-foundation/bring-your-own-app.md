## പരിചയം

[നിങ്ങളുടെ ആദ്യ പ്രോജക്റ്റ്](first-project.md) ൽ നിങ്ങൾ ടെംപ്ലേറ്റിൽ നിന്നാരംഭിച്ച് ഒരു ആപ്പ് വിന്യസിച്ചിട്ടുണ്ട്. പക്ഷേ çoതും സമയത്ത് നിങ്ങൾക്ക് ഇതിനകം തന്നെ *ഒരു* ആപ്പുണ്ട്—ഒരു Node.js API, Python Flask സർവീസ്, .NET വെബ് ആപ്പ്—മഷീനിലെ ഒരു ഫോള്ഡറിൽ ഇരിക്കുന്നു. ഈ പാഠം azd ആ ആസ്വദിച്ച കോഡിൽ ചേർക്കുന്നതാണ്, അതിലൂടെ നിങ്ങൾക്ക് `azd up` ഉപയോഗിച്ച് വിന്യസിക്കാം, ടെംപ്ലേറ്റിന്റെ ആവശ്യമില്ല.

## പഠന ലക്ഷ്യങ്ങൾ

ഈ പാഠത്തിന്റെ അവസാനകാലത്ത്, നിങ്ങൾക്ക്:
- azd പ്രോജക്റ്റ് തുടങ്ങാനുള്ള മൂന്ന് മാർഗങ്ങൾ മനസിലാക്കാം
- നിലവിലുള്ള കോഡിന്റെ ഉള്ളിൽ `azd init` ശ്രമിക്കാം
- `azure.yaml` എങ്ങനെ പ്രവർത്തിക്കുന്നു, `infra/` ഫോള്ഡർ ആപ്പ്‌ക്ക് എന്താണ് ചെയ്യുന്നത് എന്നറിയാം
- വിലാസസഹായം azd സൃഷ്ടിക്കുന്നതും നിങ്ങൾ തന്നെ എഴുതേണ്ടതുമായ ഒഴിവു തിരിച്ചറിയാം
- നിങ്ങളുടെ നിലവിലുള്ള ആപ്പ് Azure പ്രദാനം ചെയ്യാം `azd up` ഉപയോഗിച്ച്

## പഠന ഫലങ്ങൾ

ഈ പാഠം പൂർത്തിയാക്കിയതിന് ശേഷം, നിങ്ങൾക്ക് കഴിയും:
- നിങ്ങൾക്കുണ്ടായ ഒരു പ്രോജക്റ്റിൽ azd തുടക്കം കുറിക്കുക
- ഒരു അടിസ്ഥാന `azure.yaml` ഫയൽ വായിക്കുകയും തിരുത്തുകയും ചെയ്യുക
- `azd infra generate` ഉപയോഗിച്ച് സ്റ്റാർട്ടർ ഇൻഫ്രാസ്ട്രക്ചർ സൃഷ്ടിക്കുക
- ആപ്പിന് അനുയോജ്യമായ Azure ഹോസ്റ്റ് തിരഞ്ഞെടുക്കുക
- നിങ്ങളുടെ ആപ്പ് വിന്യസിക്കുകയും ശുചിയാക്കുകയും ചെയ്യുക

## azd പ്രോജക്റ്റ് തുടങ്ങാനുള്ള മൂന്ന് മാർഗങ്ങൾ

| തുടങ്ങുന്ന സ്ഥലം | കമാൻഡ് | ഉപയോഗിക്കേണ്ട സമയത്തു |
|----------------|---------|-------------|
| **ഒരു ടെംപ്ലേറ്റിൽ നിന്ന്** | `azd init --template <name>` | പഠനത്തിനും, പുതിയ ആപ്പ് ആരംഭിക്കുവാനും ഒരു ഉറപ്പുള്ള സാമ്പിൾ ഉപയോഗിക്കൽ |
| **നിലവിലുള്ള നിങ്ങളുടെ കോഡിൽ നിന്ന്** | `azd init` (നിങ്ങൾക്കുള്ള പ്രോജക്റ്റ് ഫോൾഡറിൽ) | നിങ്ങൾക്കതികൂടി ഒരു ആപ്പ് ഉണ്ട്, അതിനെ വിന്യസിക്കാൻ ആഗ്രഹിക്കുന്നുവെങ്കിൽ |
| **Git റിപോസിറ്ററിയിൽ നിന്ന്** | `azd init --from-code` (പ്രതിലിപി എടുത്തിരിക്കുന്ന റിപോസിറ്ററി) | നിലവിലുള്ള റിപോസിറ്ററിയിൽ azd സ്വീകരിക്കുന്നത് |

നിങ്ങൾ ആദ്യ ഓപ്‌ഷൻ പ്രായോഗികമായി പരീക്ഷിച്ചിട്ടുണ്ട്. ഈ പാഠം രണ്ടാമത്തേതാണ് - ഏറ്റവും സാധാരണമായ യഥാർത്ഥ ലോക സാന്നിധ്യം.

## ഘട്ടം 1: നിങ്ങളുടെ പ്രോജക്റ്റിൽ `azd init` പ്രവർത്തിപ്പിക്കുക

നിങ്ങളുടെ നിലവിലുള്ള പ്രോജക്റ്റ് ഫോൾഡറിലുള്ള ഒരു ടേർമിനൽ തുറക്കുക, തുടർന്ന് ഇത് പ്രവർത്തിപ്പിക്കുക:

```bash
cd my-existing-app
azd init
```

azd നിങ്ങളെ എങ്ങനെ ആരംഭിക്കണമെന്ന് ചോദിക്കും. തിരഞ്ഞെടുക്കുക:

```
? How do you want to initialize your app?
> Use code in the current directory
  Select a template
```

**"നിലവിലെ ഡയറക്ടറിയിലുള്ള കോഡ് ഉപയോഗിക്കുക."** azd നിങ്ങളുടെ ഫോൾഡർ സ്കാൻ ചെയ്ത്, നിങ്ങളുടെ ഭാഷയും ഫ്രെയിംവർക്കും കണ്ടെത്തുകയും, ഹോസ്റ്റ് നിർദ്ദേശിക്കുകയും ചെയ്യും.

### azd കണ്ടെത്തുന്നത് എന്തൊക്കെയാണ്

azd `package.json`, `requirements.txt`, `pom.xml`, `*.csproj`, അല്ലെങ്കിൽ `Dockerfile` പോലുള്ള സിഗ്ണലുകൾ അന്വേഷിച്ച്, അനുയോജ്യമായ Azure ഹോസ്റ്റ് നിർദ്ദേശിക്കുന്നു:

| നിങ്ങളുടെ ആപ്പ് | സാധ്യതയുള്ള കണ്ടെത്തിയ ഹോസ്റ്റ് |
|----------|----------------------|
| Node.js / Python / .NET വെബ് ആപ്പ് | Azure App Service അല്ലെങ്കിൽ Container Apps |
| കണ്ടൈനറൈസ്ഡ് ആപ്പ് (`Dockerfile`) | Azure Container Apps |
| ഫംഗ്ഷൻ ആപ്പ് | Azure Functions |
| സ്റ്റാറ്റിക് സൈറ്റ് (React/Vue ബിൽഡ് ഔട്ട്പുട്ട്) | Azure Static Web Apps |

കണ്ടെത്തിയ സർവീസുകൾ സ്ഥിരീകരിക്കുക, പിന്നീട് azd നിങ്ങൾക്ക് ആവശ്യമായ ഫയലുകൾ scaffolding ചെയ്യും.

## ഘട്ടം 2: azd ക്രിയേറ്റ് ചെയ്തതെന്തെന്ന് മനസിലാക്കുക

തുടക്കം കഴിഞ്ഞ്, നിങ്ങളുടെ പ്രോജക്റ്റിൽ രണ്ട് പുതിയ വസ്തുക്കളുണ്ടാകും:

```
my-existing-app/
├── azure.yaml          # ← NEW: tells azd what to deploy and where
├── infra/              # ← NEW: Infrastructure as Code (Bicep by default)
│   ├── main.bicep
│   └── main.parameters.json
├── src/ (your code)    # unchanged
└── ...                 # your existing files, untouched
```

### `azure.yaml` — പ്രോജക്റ്റ് നിർവചനം

ഇത് azd പ്രോജക്റ്റിന്റെ ഹൃദയമാണ്. ഒരു ഏറ്റവും ലഘുവായ മാതൃക ഇങ്ങനെ കാണാം:

```yaml
# azure.yaml
name: my-existing-app
services:
  web:
    project: ./src           # path to your app code
    language: js             # js | python | dotnet | java
    host: appservice         # appservice | containerapp | function | staticwebapp
```

`services` ബ്ലോക്ക് முக்கியമാണ്: ഓരോ എൻട്രിയും നിങ്ങളുടെ കോഡിന്റെ ഒരു ഫോൾഡർ Azure ഹോസ്റ്റിലേക്ക് മാപ്പ് ചെയ്യുന്നു. നിങ്ങളുടെ ആപ്പിന് ഫ্রണ്ട്‌എൻഡ് കൂടാതെ API ഉണ്ടെങ്കിൽ, രണ്ട് സർവീസുകൾ ഉണ്ടാകും.

### `infra/` — കോഡ് രൂപത്തിൽ നിങ്ങളുടെ Azure വിഭവങ്ങൾ

`infra/` ഫോൾഡറിൽ നിങ്ങളുടെ ആപ്പിന് ആവശ്യമായ Azure വിഭവങ്ങൾ Bicep ഫയലുകളായി കണ്ടെയ്ന് ചെയ്യുന്നു (App Service, ഡാറ്റാബേസ് തുടങ്ങിയവ). നിങ്ങൾക്കിവയെ എഴുതേണ്ടതില്ല—azd പ്രവർത്തനക്ഷമമായ ഒരു ആരംഭ ബിന്ദു സൃഷ്ടിക്കുന്നു. പിന്നീട് നിങ്ങൾ അവ തിരുത്തുകയും വിഭവങ്ങൾ കൂട്ടിച്ചേർക്കുകയും സുരക്ഷ ശക്തമാക്കുകയും ചെയ്യാം ([അധ്യായം 4](../chapter-04-infrastructure/README.md) ൽ വിവരണം).

> **ടിപ്പ്:** വിന്യസിക്കുന്നതിന് മുൻപ് സൃഷ്ടിച്ച ഇൻഫ്രാസ്ട്രക്ചർ കാണാൻ അല്ലെങ്കിൽ അതിനെ ഇഷ്ടാനുസൃതമാക്കാൻ ആഗ്രഹിക്കുന്നുവേ? `azd infra generate` (അഥവാ `azd infra synth`) പ്രവർത്തിപ്പിച്ച് IaC ഡിസ്കിലേക്ക് എഴുതുക, അതിലൂടെ നിങ്ങൾ അവ പരിശോധിച്ച് വേർഷൻ-നിയന്ത്രിക്കാം.

## ഘട്ടം 3: ആവശ്യമായ കോൺഫിഗറേഷൻ സെറ്റ് ചെയ്യുക

നിങ്ങളുടെ ആപ്പിന് സജ്ജീകരണങ്ങളോ രഹസ്യങ്ങളോ (കണക്ഷൻ സ്ട്രിംഗ്, API കീ) ആവശ്യമുണ്ടെങ്കിൽ, അവ ഹാർഡ്കോഡ് ചെയ്യാതെ, പരിസ്ഥിതി മൂല്യങ്ങൾ ഉപയോഗിക്കുക:

```bash
# ഒരു പരിസരം സൃഷ്ടിക്കുക
azd env new dev

# രഹസ്യമല്ലാത്ത ഒരു മൂല്യം സജ്ജീകരിക്കുക
azd env set API_VERSION 1.0.0
```

യഥാർത്ഥ രഹസ്യങ്ങൾക്കായി, അവ കീ വാൾട്ടിൽ സൂക്ഷിക്കുക, പിന്നീട് നിങ്ങളുടെ ഇൻഫ്രാസ്ട്രക്ചറിൽ ഉപയോഗിക്കുക — [അധ്യായം 3: കോൺഫിഗറേഷൻ & സാക്ഷ്യപ്പെടുത്തൽ](../chapter-03-configuration/authsecurity.md) കാണുക.

## ഘട്ടം 4: വിന്യസിക്കുക

ഇപ്പോൾ നിങ്ങൾ മുൻപ് പഠിച്ച Workflow പിന്തുടരുക:

```bash
# പ്രാമാണീകരിക്കുക (azd നു വേണ്ടി ആവശ്യമാണ്)
azd auth login

# സൃഷ്ടിക്കപ്പെടുന്ന വിഭവങ്ങൾ മുൻകാഴ്ചയാക്കുക
azd provision --preview

# അടിസ്ഥാന സൗകര്യം ഒരുക്കുകയും നിങ്ങളുടെ കോഡ് വിന്യാസം ചെയ്യുകയും ചെയ്യുക
azd up
```

ശേഷം, azd നിങ്ങളുടെ ആപ്പിന്റെ URL പ്രിന്റ് ചെയ്യും. സാധാരണ azd ആപ്പുപോലെ പരിശോധിക്കുക:

```bash
azd show           # അടിയന്തരമറിയിപ്പുകൾ കാണിക്കുക
azd monitor --logs # ആവശ്യമായാൽ ലോഗുകൾ പരിശോധിക്കുക
```

| azd എന്റെ ആപ്പ് കണ്ടെത്തിയില്ല | മാനിഫെസ്റ്റ് ഇല്ല (ഉദാ. `package.json`) | മാനിഫെസ്റ്റ് ചേർക്കുക, അല്ലെങ്കിൽ `azd init` ൽ ഹോസ്റ്റ് മാനുവലായി തിരഞ്ഞെടുക്കുക |
| `azd up` വേളയിൽ ബിൽഡ് പരാജയപ്പെടുന്നു | ആപ്പിന് ബിൽഡ് ഘട്ടം ആവശ്യമാണ് | `azure.yaml` ൽ സർവീസ് കീഴിൽ `buildCommand`/`outputPath` ചേർക്കുക |
| ആപ്പ് ആരംഭിക്കുന്നെങ്കിലും പിശകുകൾ കാണിക്കുന്നു | കോൺഫിഗ്/രഹസ്യം ഇല്ലാതിരിക്കുന്നു | `azd env set` ഉപയോഗിച്ച് മൂല്യങ്ങൾ സെറ്റ് ചെയ്യുക അല്ലെങ്കിൽ കീ വാൾട്ട്ൽ വെബ്ബ് ബന്ധിപ്പിക്കുക |
| തെറ്റായ ഹോസ്റ്റ് തിരഞ്ഞെടുക്കപ്പെട്ടു | സ്വയം കണ്ടെത്തൽ തെറ്റായി ഭാവിച്ചു | `azure.yaml` ൽ `host:` തിരുത്തുക, ശേഷം `azd up` വീണ്ടും പ്രവർത്തിപ്പിക്കുക |

കൂടുതൽ വിവരങ്ങൾക്ക്, [അധ്യായം 7: പ്രശ്ന പരിഹാരം](../chapter-07-troubleshooting/README.md) കാണുക.

## സംഗ്രഹം

- `azd init` → **"നിലവിലെ ഡയറക്ടറിയിലുള്ള കോഡ് ഉപയോഗിക്കുക"** നിങ്ങള്‍ക്കുള്ള ആപ്പിൽ azd ചേർക്കുന്നു.
- `azure.yaml` നിങ്ങളുടെ കോഡ് ഫോൾഡറുകളെ Azure ഹോസ്റ്റുകളിലേക്ക് മാപ്പ് ചെയ്യുന്നു; `infra/` Bicep ആയി വിഭവങ്ങൾ നിർവചിക്കുന്നു.
- `azd infra generate` നിങ്ങൾക്ക് സൃഷ്ടിച്ച ഇൻഫ്രാസ്ട്രക്ചർ പരിശോധിക്കാൻ അല്ലെങ്കിൽ ഇഷ്ടാനുസൃതമാക്കാൻ കഴിയും.
- തുടക്കം കഴിഞ്ഞാൽ, നിങ്ങളുടെ നിലവിലുള്ള ആപ്പ് ഒരു ടെംപ്ലേറ്റ്സാഹിതമായ ആപ്പ് പോലെ തന്നെ `azd up` / `azd down` workflow ഉപയോഗിക്കുന്നു.
