## പരിചയം

നിങ്ങളുടെ ആദ്യ Azure ഡെവലപ്പർ CLI പ്രോജക്റ്റിലേക്ക് സ്വാഗതം! azd ഉപയോഗിച്ച് Azure-ൽ സമ്പൂർണമായ ഫുൾ-സ്റ്റാക്ക് ആപ്ലിക്കേഷൻ സൃഷ്ടിക്കാനും, വിനിയോഗിക്കാനും, നിയന്ത്രിക്കാനും ഒരു സമഗ്രമായ ഹാൻഡ്‌സ്-ഓൺ ട്യൂട്ടോറിയൽ ഇത് നൽകുന്നു. ഒരു React ഫ്രണ്ട്‌എൻഡ്, Node.js API ബാക്ക്‌എൻഡ്, MongoDB ഡാറ്റാബേസ് ഉൾക്കൊള്ളുന്ന യാഥാർത്ഥ്യ ടുഡോ ആപ്ലിക്കേഷനോടെ നിങ്ങൾ പ്രവർത്തിക്കും.

## പഠന ലക്ഷ്യങ്ങൾ

ഈ ട്യൂട്ടോറിയൽ പൂർത്തിയാക്കുമ്പോൾ, നിങ്ങൾക്ക് കഴിയുന്നതു:
- ടെമ്പ്ലേറ്റുകൾ ഉപയോഗിച്ചു azd പ്രോജക്റ്റ് ഇൻഷുലൈസേഷൻ വർക്ക്‌ഫ്ലോ മാസ്റ്റർ ചെയ്യുക
- Azure ഡെവലപ്പർ CLI പ്രോജക്റ്റ് ഘടനയും കൺഫിഗറേഷൻ ഫയലുകളും മനസിലാക്കുക
- ഇന്‍ഫ്രാസ്ട്രക്ചറു പ്രൊവിഷനിംഗ് നടത്തിക്കൊണ്ടു Azure-ലേക്ക് ആപ്ലിക്കേഷൻ പൂർണ്ണമായി വിനിയോഗിക്കുക
- ആപ്ലിക്കേഷൻ അപ്‌ഡേറ്റുകളും പുനരുള്ളചിത്രീകരണ തന്ത്രങ്ങളും നടപ്പിലാക്കുക
- ഡെവലപ്പ്മെന്റ്, സ്റ്റേജിംഗ് എന്നിവയ്ക്കായി വിവിധ പരിസ്ഥിതികൾ പരിപാലിക്കുക
- റിസോഴ്‌സ് ക്ലീൻഅപ്പ്, ചെലവ് മാനേജ്‌മെന്റ് പ്രാക്ടീസുകൾ പ്രയോഗിക്കുക

## പഠന ഫലങ്ങൾ

പൂർത്തിയാക്കലിനു ശേഷം, നിങ്ങൾക്ക് കഴിയും:
- ടെമ്പ്ലേറ്റുകളിൽ നിന്നു സ്വതന്ത്രമായി azd പ്രോജക്റ്റുകൾ ഇൻഷുലൈസ് ചെയ്യാനും കൺഫിഗർ ചെയ്യാനും
- azd പ്രോജക്റ്റ് ഘടനകൾ പ്രാവർത്തികമായി നയിക്കാനും മാറ്റങ്ങളുണ്ടാക്കാനുമുള്ള കഴിവ്
- ഏകകമാന്റുകൾ ഉപയോഗിച്ച് Azure-ലേക്ക് ഫുൾ-സ്റ്റാക്ക് ആപ്ലിക്കേഷൻ വിനിയോഗിക്കുക
- സാധാരണ വിനിയോഗ പ്രശ്‌നങ്ങളെയും ആധികാരികതാ പ്രശ്‌നങ്ങളെയും പരിഹരിക്കുക
- deployment സ്റ്റെേജുകൾ വ്യത്യസ്തമായ നിരവധി Azure പരിസ്ഥിതികൾ പരിപാലിക്കുക
- ആപ്ലിക്കേഷൻ അപ്ഡേറ്റുകൾക്കായി തുടർച്ചയായ വിനിയോഗ വർക്ക്ഫ്ലോകൾ നടപ്പിലാക്കുക

## പടി 1: ഒരു ടെമ്പ്ലേറ്റ് തിരഞ്ഞെടുക്കുകയും ഇൻഷുലൈസ് ചെയ്യുകയും ചെയ്യുക

React ഫ്രണ്ട്‌എൻഡ്, Node.js API ബാക്ക്‌എൻഡ് ഉൾപ്പെടുന്ന പോപ്പുലർ ടുഡോ ആപ്ലിക്കേഷൻ ടെമ്പ്ലേറ്റ് ഉപയോഗിച്ച് തുടങ്ങിയാൽ സുഗമമാണ്.

```bash
# ലഭ്യമായ ടെംപ്ലേറ്റുകൾ ബ്രൗസ് ചെയ്യുക
azd template list

# ടു-ഡു ആപ്പ് ടെംപ്ലേറ്റ് പ്രാരംഭീകരിക്കുക
mkdir my-first-azd-app
cd my-first-azd-app
azd init --template todo-nodejs-mongo

# പ്രോംപ്റ്റുകൾ പിന്തുടരുക:
# - ഒരു എൻവയോൺമെന്റ് പേര് നൽകുക: "dev"
# - ഒരു സബ്സ്ക്രിപ്ഷൻ തിരഞ്ഞെടുക്കുക (നിങ്ങൾക്കหลายאַם ഉണ്ടെങ്കിൽ)
# - ഒരു മേഖല തിരഞ്ഞെടുക്കുക: "East US 2" (അല്ലെങ്കിൽ നിങ്ങളുടെ പ്രീഫർഡ് മേഖല)
```

### എന്തു സംഭവിച്ചു?
- ടെമ്പ്ലേറ്റ് കോഡ് നിങ്ങളുടെ ലോക്കൽ ഡയറക്ടറിയിലേക്ക് ഡൌൺലോഡ് ചെയ്തു
- സേവന നിർവചനങ്ങളോടെ `azure.yaml` ഫയൽ സൃഷ്ടിച്ചു
- `infra/` ഡയറക്ടറിയിൽ ഇന്‍ഫ്രാസ്ട്രക്ചർ കോഡ് ഒരുക്കി
- ഒരു പരിസ്ഥിതി കൺഫിഗറേഷൻ സൃഷ്ടിച്ചു

## പടി 2: പ്രോജക്റ്റ് ഘടന അവലോകനം ചെയ്യുക

azd ഞങ്ങൾക്ക് സൃഷ്ടിച്ചത്തിന്റെ അവലോകനം ചെയ്യാം:

```bash
# പ്രോജക്ട് ഘടന കാണുക
tree /f   # വിൻഡോസ്
# അല്ലെങ്കിൽ
find . -type f | head -20   # macOS/Linux
```

നിങ്ങൾ കാണേണ്ടത്:
```
my-first-azd-app/
├── .azd/
│   └── config.json              # Project configuration
├── .azure/
│   └── dev/                     # Environment-specific files
├── .devcontainer/               # Development container config
├── .github/workflows/           # GitHub Actions CI/CD
├── .vscode/                     # VS Code settings
├── infra/                       # Infrastructure as code (Bicep)
│   ├── main.bicep              # Main infrastructure template
│   ├── main.parameters.json     # Parameters for deployment
│   └── modules/                # Reusable infrastructure modules
├── src/
│   ├── api/                    # Node.js backend API
│   │   ├── src/               # API source code
│   │   ├── package.json       # Node.js dependencies
│   │   └── Dockerfile         # Container configuration
│   └── web/                   # React frontend
│       ├── src/               # React source code
│       ├── package.json       # React dependencies
│       └── Dockerfile         # Container configuration
├── azure.yaml                  # azd project configuration
└── README.md                   # Project documentation
```

### മനസ്സിലാക്കേണ്ട പ്രധാന ഫയലുകൾ

**azure.yaml** - നിങ്ങളുടെ azd പ്രോജക്റ്റിന്റെ ഹൃദയം:
```bash
# പ്രോജക്ട് ക്രമീകരണം കാണുക
cat azure.yaml
```

**infra/main.bicep** - ഇന്‍ഫ്രാസ്ട്രക്ചർ നിർവചനങ്ങൾ:
```bash
# ഇൻഫ്രാസ്ട്രക്ചർ കോഡ് കാണുക
head -30 infra/main.bicep
```

## പടി 3: നിങ്ങളുടെ പ്രോജക്റ്റ് ഇഷ്ടാനുസൃതമാക്കുക (ഐച്ഛികം)

വിനിയോഗത്തിന് മുമ്പ്, നിങ്ങൾ ആപ്ലിക്കേഷൻ ഇഷ്ടാനുസൃതമാക്കാം:

### ഫ്രണ്ട്‌എൻഡ് മാറ്റം വരുത്തുക
```bash
# React ആപ്പ് കോമ്പോണന്റ് തുറക്കുക
code src/web/src/App.tsx
```

ഒരു ലളിതമായ മാറ്റം ചെയ്യുക:
```typescript
// തലക്കെട്ട് കണ്ടെത്തി അത് മാറ്റുക
<h1>My Awesome Todo App</h1>
```

### പരിസ്ഥിതി വ്യത്യാസങ്ങൾ ക്രമീകരിക്കുക
```bash
# ഇഷ്‌ടാനുസൃത പരിസ്ഥിതി മാറ്റം സജ്ജമാക്കുക
azd env set WEBSITE_TITLE "My First AZD App"
azd env set API_VERSION "v1.18"
# എല്ലാ പരിസ്ഥിതി മാറ്റങ്ങളും കാണുക
azd env get-values
```

## പടി 4: Azure-ലേക്ക് വിനിയോഗിക്കുക

ഇപ്പോൾ രസകരമായ ഭാഗം – മുഴുവൻ കാര്യങ്ങളും Azure-ലേക്ക് വിനിയോഗിക്കുക!

```bash
# അടിസ്ഥാന ഘടനയും അപേക്ഷയും വിന്യസിക്കുക
azd up

# ഈ കമാൻഡ് ഇത് ചെയ്യുക:
# 1. ആസ്യൂർ ഉറവിടങ്ങൾ (ആപ്പ് സർവീസ്, കോസ്മോസ് ഡിബി, മുതലായവ) ഒരുക്കുക
# 2. നിങ്ങളുടെ അപേക്ഷ നിർമ്മിക്കുക
# 3. ഒരുക്കിയ ഉറവിടങ്ങളിലേക്ക് വിന്യസിക്കുക
# 4. അപേക്ഷയുടെ URL പ്രദർശിപ്പിക്കുക
```

### വിനിയോഗം നടക്കുമ്പോൾ എന്ത് സംഭവിക്കുന്നു?

`azd up` കമാൻഡ് ഈ ഘട്ടങ്ങൾ നടപ്പിലാക്കുന്നു:
1. **Provision** (`azd provision`) - Azure റിസോഴ്‌സുകൾ സൃഷ്ടിക്കുന്നു
2. **Package** - നിങ്ങളുടെ ആപ്ലിക്കേഷൻ കോഡ് ബിൽഡ് ചെയ്യുന്നു
3. **Deploy** (`azd deploy`) - കോഡ് Azure റിസോഴ്‌സുകളിൽ വിനിയോഗിക്കുന്നു

### പ്രതീക്ഷിക്കാവുന്ന ഫലം
```
Packaging services (azd package)

SUCCESS: Your up workflow to provision and deploy to Azure completed in 4 minutes 32 seconds.

You can view the resources created under the resource group rg-my-first-azd-app-dev in the Azure portal:
https://portal.azure.com/#@/resource/subscriptions/{subscription-id}/resourceGroups/rg-my-first-azd-app-dev

Navigate to the Todo app at:
https://app-web-abc123def.azurewebsites.net
```

## പടി 5: നിങ്ങളുടെ ആപ്ലിക്കേഷൻ പരീക്ഷിക്കുക

### ആപ്ലിക്കേഷനിലേക്ക് ആക്സസ് ചെയ്യുക
വിനിയോഗ ഫലത്തിൽ ലഭിക്കുന്ന URL-ൽ ക്ലിക്ക് ചെയ്യുക, അല്ലെങ്കിൽ എപ്പോൾ വേണമെങ്കിലും വാങ്ങുക:
```bash
# ആപ്ലിക്കേഷൻ എന്റ്പോയിന്റുകൾ നേടുക
azd show

# ആപ്ലിക്കേഷൻ നിങ്ങളുടെ ബ്രൗസറിൽ തുറക്കുക
azd show --output json | jq -r '.services.web.endpoint'
```

### ടുഡോ ആപ്ലിക്കേഷൻ പരീക്ഷിക്കുക
1. **ടുഡോ ഇനം ചേർക്കുക** - "Add Todo" അന്വയം ചെയ്യുക, ഒരു ടാസ്‌ക് നൽകുക
2. **പൂർത്തിയായി അടയാളപ്പെടുത്തുക** - പൂർത്തിയാക്കിയ ഇനങ്ങൾ പരിശോധിക്കുക
3. **ഐറ്റങ്ങൾ നീക്കംചെയ്യുക** - ആവശ്യമില്ലാത്ത ടുഡോകൾ ഒഴിവാക്കുക

### നിങ്ങളുടെ ആപ്ലിക്കേഷൻ നിരീക്ഷിക്കുക
```bash
# നിങ്ങളുടെ സ്രോതസുകൾക്കായി Azure പോർട്ടൽ തുറക്കൂ
azd monitor

# ആപ്ലിക്കേഷൻ ലോഗുകൾ കാണുക
azd monitor --logs

# ലൈവ് മെട്രിക്സ് കാണുക
azd monitor --live
```

### ✅ നിങ്ങളുടെ വിനിയോഗം സ്ഥിരീകരിക്കുക

തുടരുന്നതിനു മുൻപ് ഈ ചുരുങ്ങിയ പരിശോധന ലിസ്റ്റ് ഉപയോഗിച്ച് എല്ലാം ശരിയാകുന്നതായി ഉറപ്പാക്കുക — "deploy succeeded" എന്ന് സൂചിപ്പിക്കുന്നത് "ആപ്പ് പ്രവർത്തിക്കുന്നു" എന്നില്ല:

```bash
# 1. എതിരെ പോയിന്റ് ഉണ്ടെന്ന് ഉറപ്പുവരുത്തുകയും എത്താൻ സാധിക്കുകയുമാണ്
azd show

# 2. എതിരെ പോയിന്റ് സ്മോക്ക്-ടെസ്റ്റ് ചെയ്യുക (HTTP 200 പ്രതീക്ഷിക്കുന്നു)
curl -I "$(azd show --output json | jq -r '.services.web.endpoint')"

# 3. നിങ്ങളുടെ ആപ്പ് ഒരു ഹെൽത്ത് എതിരെ പോയിന്റ് പ്രദർശിപ്പിക്കുന്നുണ്ടെങ്കിൽ അത് പരിശോധിക്കുക
curl "$(azd show --output json | jq -r '.services.web.endpoint')/health"
```

**വിനിയോഗം സ്ഥിരീകരിക്കപ്പെടുന്നു, ഈ സാഹചര്യങ്ങളിൽ:**
- ✅ `azd show` പോർട്ട് രേഖപ്പെടുത്തുന്ന ഒരു URL ലിസ്റ്റ് ചെയ്യുന്നു
- ✅ ആ URL നിങ്ങളുടെ ബ്രൗസറിന്റെ പിഴവുകളൊന്നും കൂടാതെ തുറക്കുന്നു
- ✅ പ്രധാന ഫീച്ചറുകൾ പ്രവർത്തിക്കുന്നു (ടുഡോ ചേർക്കുക/പൂർത്തിയാക്കുക/നീക്കംചെയ്യുക)
- ✅ `azd monitor --logs` അസാധാരണ പിഴവുകളില്ലാത്ത അഭ്യർത്ഥനകൾ നിരീക്ഷിക്കുന്നു

ഏതെങ്കിലും പരിശോധന fail ആണെങ്കിൽ, [അദ്ധ്യായം 7: പ്രശ്‌നപരിഹാരങ്ങൾ](../chapter-07-troubleshooting/README.md) കാണുക.

## പടി 6: മാറ്റങ്ങൾ വരുത്തുകയും പുനർവിനിയോഗം നടത്തുകയും ചെയ്യുക

ഒരു മാറ്റം വരുത്തി എളുപ്പത്തിൽ അപ്‌ഡേറ്റ് എങ്ങനെ ചെയ്യാമെന്ന് നോക്കാം:

### API മാറ്റം വരുത്തുക
```bash
# API കോഡ് തിരുത്തുക
code src/api/src/routes/lists.js
```

ഒരു കസ്റ്റം റെസ്പോൺസ് ഹെഡർ ചേർക്കുക:
```javascript
// ഒരു റൂട്ട് ഹാൻഡളർ കണ്ടെത്തി ചേർക്കുക:
res.header('X-Powered-By', 'Azure Developer CLI');
```

### കോഡ് മാറ്റങ്ങൾ മാത്രം വിനിയോഗിക്കുക
```bash
# പ്രയോഗ കോഡ് മാത്രം ഡിപ്പ്ലോയ് ചെയ്യുക (ഇൻഫ്രാസ്ട്രക്ചർ ഒഴിവാക്കുക)
azd deploy

# ഇൻഫ്രാസ്ട്രക്ചർ ഇതിനകം നിലനില്ക്കുന്നതിനാൽ ഇത് 'azd up' എന്നതിനേക്കാൾ വേഗത്തിലാണ്
```

## പടി 7: നിരവധി പരിസ്ഥിതികൾ പരിപാലിക്കുക

പ്രൊഡക്ഷൻ മുമ്പ് പരീക്ഷണങ്ങൾക്ക് സ്റ്റേജിംഗ് പരിസ്ഥിതി സൃഷ്ടിക്കുക:

```bash
# ഒരു പുതിയ സ്റ്റേജിംഗ് പരിസരം സൃഷ്ടിക്കുക
azd env new staging

# സ്റ്റേജിംഗിലേക്ക് വിന്യസിക്കുക
azd up

# ഡെവ് പരിസരത്തിലേക്ക് മടങ്ങുക
azd env select dev

# എല്ലാ പരിസരങ്ങളും പട്ടികപ്പെടുത്തുക
azd env list
```

### പരിസ്ഥിതി ഒപ്പം താരതമ്യം
```bash
# ഡെവ് പരിസ്ഥിതി കാണുക
azd env select dev
azd show

# സ്റ്റേജിങ് പരിസ്ഥിതി കാണുക
azd env select staging
azd show
```

## പടി 8: റിസോഴ്‌സുകൾ ക്ലീൻഅപ്പ് ചെയ്യുക

പരീക്ഷണങ്ങൾ അവസാനിപ്പിച്ചതിനു ശേഷം, നിലനിൽക്കുന്ന ചെലവുകൾ ഒഴിവാക്കാൻ ക്ലീൻഅപ്പ് ചെയ്യുക:

```bash
# നിലവിലെ പരിസ്ഥിതിക്ക് എല്ലാ Azure റിപ്പോഴ്സുകളും നീക്കം ചെയ്യുക
azd down

# സ്ഥിരീകരണമില്ലാതെ ബലവിധേയമായി നീക്കം ചെയ്യുക, മൃദുവായി നീക്കം ചെയ്ത റിപ്പോഴ്സുകൾ ശുദ്ധീകരിക്കുക
azd down --force --purge

# നിർദിഷ്‌ട പരിസ്ഥിതിക്ക് നീക്കം ചെയ്യുക
azd env select staging
azd down --force --purge
```

## ക്ലാസിക് ആപ്പ് vs AI-പവർഡ് ആപ്പ്: സമാന വർക്ക്‌ഫ്ലോ

നിങ്ങൾ വെബ് ആപ്ലിക്കേഷൻ പരമ്പരാഗതവും വിജയകരമായി വിനിയോഗിച്ചു. എന്നാൽ നിങ്ങൾക്ക് AI-പവർഡ് ഒരു ആപ്പ് വിനിയോഗിക്കാനുണ്ടെങ്കിൽ — ഉദാഹരണത്തിന്, Microsoft Foundry Models പിന്തുണയോടെ ഒരു ചാറ്റ് ആപ്ലിക്കേഷൻ?

സന്തോഷകരമായ വാർത്ത: **വർക്ക്ഫ്ലോ സമാനമാണ്.**

| പടി | ക്ലാസിക് ടുഡോ ആപ്പ് | AI ചാറ്റ് ആപ്പ് |
|------|-----------------|-------------|
| ഇൻഷുലൈസ് ചെയ്യുക | `azd init --template todo-nodejs-mongo` | `azd init --template azure-search-openai-demo` |
| ഓതന്റിക്കേറ്റ് ചെയ്യുക | `azd auth login` | `azd auth login` |
| വിനിയോഗിക്കുക | `azd up` | `azd up` |
| മോണിറ്റർ ചെയ്യുക | `azd monitor` | `azd monitor` |
| ക്ലീൻഅപ്പ് | `azd down --force --purge` | `azd down --force --purge` |

മാത്രം ഭേദം നിങ്ങൾ ആരംഭിക്കുന്നത് ടെമ്പ്ലേറ്റാണ്. AI ടെമ്പ്ലേറ്റ് അധിക ഇൻഫ്രാസ്ട്രക്ചർ ഉൾക്കൊള്ളുന്നു (മൈക്രോസോഫ്‌ട് ഫൗണ്ട്രി മോഡൽസ് റിസോഴ്‌സ് അല്ലെങ്കിൽ AI സെർച്ച് ഇൻഡെക്‌സ് പോലെയുള്ളത്), എന്നാൽ azd അതെല്ലാം കൈകാര്യം ചെയ്യുന്നു. നിങ്ങൾക്ക് പുതിയ കമാൻഡുകൾ പഠിക്കേണ്ടതില്ല, പുതിയ ടൂൾ സ്വീകരിക്കേണ്ടതില്ല, അല്ലെങ്കിൽ വിനിയോഗത്തിന്റെ രീതിയിൽ മാറ്റം വരുത്തേണ്ടതില്ല.

azd-യുടെ മൂല സിദ്ധാന്തം: **ഒരു വർക്ക്‌ഫ്ലോ, ഏതൊരു ജോലിയും.** ഈ ട്യൂട്ടോറിയലിൽ ഉപയോഗിച്ചുകഴിച്ച കഴിവുകൾ — ഇൻഷുലൈസ് ചെയ്യുക, വിനിയോഗിക്കുക, മോണിറ്റർ ചെയ്യുക, പുനർവിനിയോഗിക്കുക, ക്ലീൻഅപ്പ് — AI ആപ്ലിക്കേഷനുകൾക്കും ഏജന്റുകൾക്കുമായി സദൃശ്യമായി പ്രയോഗിക്കാവുന്നതാണ്.

## 🎯 നൈപുണ്യ പരിശോധനാ വ്യായാമങ്ങൾ

### വ്യായാമം 1: വേറിട്ട ടെമ്പ്ലേറ്റ് വിനിയോഗിക്കുക (15 മിനിറ്റ്)
**ലക്ഷ്യം**: azd init, വിനിയോഗം വർക്ക്‌ഫ്ലോയിൽ പ്രാവീണ്യമുണർത്തുക

```bash
# Python + MongoDB സ്റ്റാക്ക് ശ്രമിക്കുക
mkdir todo-python && cd todo-python
azd init --template todo-python-mongo
azd up

# ഡിപ്ലോയ്മെന്റ് പരിശോധിക്കുക
azd show
curl $(azd show --output json | jq -r '.services.web.endpoint')

# ശുചിയാക്കി അറ്റുക
azd down --force --purge
```

**വിജയത്തിന്റെ മാനദണ്ഡങ്ങൾ:**
- [ ] പിഴവുകളടങ്ങാതെ ആപ്ലിക്കേഷൻ വിനിയോഗിക്കുക
- [ ] ആപ്ലിക്കേഷൻ URL ബ്രൗസറിൽ തുറക്കാൻ കഴിയുക
- [ ] ആപ്ലിക്കേഷൻ ശരിയായി പ്രവർത്തിക്കണം (ടുഡോ ചേർക്കുക/നീക്കംചെയ്യുക)
- [ ] എല്ലാ റിസോഴ്‌സുകളും വിജയകരമായി ക്ലീൻഅപ്പ് ചെയ്യുക

### വ്യായാമം 2: കൺഫിഗറേഷൻ ഇഷ്ടാനുസൃതമാക്കുക (20 മിനിറ്റ്)
**ലക്ഷ്യം**: പരിസ്ഥിതി വ്യത്യാസ ക്രമീകരണം അഭ്യാസമാക്കുക

```bash
cd my-first-azd-app

# കസ്റ്റം എൻവയോൺമെന്റ് സൃഷ്ടിക്കുക
azd env new custom-config

# കസ്റ്റം വെറിയബിൾസുകൾ സെറ്റ് ചെയ്യുക
azd env set APP_TITLE "My Custom Todo App"
azd env set API_VERSION "2.0.0"
azd env set ENABLE_DEBUG "true"

# വെറിയബിൾസുകൾ സ്ഥിരീകരിക്കുക
azd env get-values | grep APP_TITLE

# കസ്റ്റം കോൺഫിഗർന്ന് ഉപയോഗിച്ച് ഡിപ്ലോയ് ചെയ്യുക
azd up
```

**വിജയത്തിന്റെ മാനദണ്ഡങ്ങൾ:**
- [ ] ഇഷ്ടാനുസൃത പരിസ്ഥിതി വിജയകരമായി സൃഷ്ടിക്കുക
- [ ] പരിസ്ഥിതി വ്യത്യാസങ്ങൾ ക്രമീകരിച്ച് അവ തിരികെ ലഭ്യമാക്കുക
- [ ] ഇഷ്ടാനുസൃത കൺഫിഗറേഷനോടുകൂടി ആപ്ലിക്കേഷൻ വിനിയോഗിക്കുക
- [ ] വിനിയോഗിച്ച ആപ്പിൽ ഇഷ്ടാനുസൃത ക്രമീകരണങ്ങൾ സ്ഥിരീകരിക്കുക

### വ്യായാമം 3: ബഹു-പരിസ്ഥിതി വർക്ക്‌ഫ്ലോ (25 മിനിറ്റ്)
**ലക്ഷ്യം**: പരിസ്ഥിതി മാനേജ്മെന്റും വിനിയോഗ തന്ത്രങ്ങളും കാര്യക്ഷമമാക്കുക

```bash
# ഡെവ് പരിസ്ഥിതിയെ സൃഷ്ടിക്കുക
azd env new dev-$(whoami)
azd env set ENVIRONMENT_TYPE dev
azd env set LOG_LEVEL debug
azd up

# ഡെവ് URL ശ്രദ്ധിക്കുക
DEV_URL=$(azd show --output json | jq -r '.services.web.endpoint')
echo "Dev: $DEV_URL"

# സ്റ്റേജിംഗ് പരിസ്ഥിതിയെ സൃഷ്ടിക്കുക
azd env new staging-$(whoami)
azd env set ENVIRONMENT_TYPE staging
azd env set LOG_LEVEL info
azd up

# സ്റ്റേജിംഗ് URL ശ്രദ്ധിക്കുക
STAGING_URL=$(azd show --output json | jq -r '.services.web.endpoint')
echo "Staging: $STAGING_URL"

# പരിസ്ഥിതികൾ തുലനചെയ്യുക
azd env list

# രണ്ട് പരിസ്ഥിതികളും ടെസ്റ്റ് ചെയ്യുക
curl "$DEV_URL/health"
curl "$STAGING_URL/health"

# രണ്ടും ശുചിയാക്കുക
azd env select dev-$(whoami) && azd down --force --purge
azd env select staging-$(whoami) && azd down --force --purge
```

**വിജയ മാനദണ്ഡങ്ങൾ:**
- [ ] വ്യത്യസ്ത കൺഫിഗറേഷനുകളോടെ രണ്ട് പരിസ്ഥിതികൾ സൃഷ്ടിക്കുക
- [ ] രണ്ടും విజయകരമായി വിനിയോഗിക്കുക
- [ ] `azd env select` ഉപയോഗിച്ച് പരിസ്ഥിതികൾ തമ്മിൽ മാറാൻ കഴിവുള്ളത്
- [ ] പരിസ്ഥിതി വ്യത്യാസങ്ങൾ തമ്മിൽ വ്യത്യസ്തമാണോ എന്ന് തിരിച്ചറിയുക
- [ ] എന്തും> വിജയകരമായി രണ്ടു പരിസ്ഥിതികളും ക്ലീൻഅപ്പ് ചെയ്യാം
