### ഘട്ടം 1: പരിസ്ഥിതി ഒരുക്കല്‍

1. **ഉപകരണ ഇൻസ്റ്റലേഷൻ പരിശോധന:**
```bash
# AZD ഇൻസ്റ്റലേഷൻ പരിശോധിക്കുക
azd version

# Azure CLI പരിശോധിക്കുക
az --version

# AZD വർക്ക്‌ഫ്ലോമാർക്ക് Azure-ലോഗിൻ ചെയ്യുക
azd auth login

# നിർണയ സമയത്ത് az കമാൻഡുകൾ നടത്താൻ ഉദ്ദേശിക്കുന്നെങ്കിൽകൂടി മാത്രമേ Azure CLI ലോഗിൻ ചെയ്യേണ്ടതുള്ളൂ
az login
```

നിങ്ങൾ വിവിധ ടെന്നന്റുകളിൽ ജോലി ചെയ്താൽ അല്ലെങ്കിൽ സബ്സ്ക്രിപ്ഷന്‍ സ്വയം കണ്ടെത്താനാവുകയില്ല എങ്കിൽ, `azd auth login --tenant-id <tenant-id>` ഉപയോഗിച്ച് വീണ്ടും ശ്രമിക്കുക.

2. **വർക്ഷോപ്പ് റിപോസിറ്ററി ക്ലോൺ ചെയ്യുക:**
```bash
git clone https://github.com/Azure-Samples/azure-search-openai-demo
cd azure-search-openai-demo
```

## മോഡ്യൂള്‍ 1: AI അപ്ലിക്കേഷനിൽ AZD ഘടന മനസിലാക്കുക

### AI AZD ടെംപ്ലേറ്റിന്റെ ഘടന

AI-റെഡി AZD ടെംപ്ലേറ്റിലെ പ്രധാന ഫയലുകൾ പരിശോധിക്കുക:

```
azure-search-openai-demo/
├── azure.yaml              # AZD configuration
├── infra/                   # Infrastructure as Code
│   ├── main.bicep          # Main infrastructure template
│   ├── main.parameters.json # Environment parameters
│   └── modules/            # Reusable Bicep modules
│       ├── openai.bicep    # Microsoft Foundry Models configuration
│       ├── search.bicep    # Cognitive Search setup
│       └── webapp.bicep    # Web app configuration
├── app/                    # Application code
├── scripts/               # Deployment scripts
└── .azure/               # AZD environment files
```

### **ലാബ് വ്യായാമം 1.1: കോൺഫിഗറേഷൻ പരിശോധിക്കുക**

1. **azure.yaml ഫയൽ പരിശോധിക്കുക:**
```bash
cat azure.yaml
```

**പരിശോധിക്കാനുള്ള കാര്യങ്ങൾ:**
- AI ഘടകങ്ങളിലേക്കുള്ള സർവീസ് നിർവചനങ്ങൾ
- എന്‍വയോൺമെന്റ് വാരിയബിൾ മാപ്പിംഗുകൾ
- ഹോസ്റ്റ് കോൺഫിഗറേഷനുകൾ

2. **main.bicep ഇൻഫ്രാസ്ട്രക്ചർ അവലോകനം:**
```bash
cat infra/main.bicep
```

**സൂചനകൾ തിരിച്ചറിയുക:**
- Microsoft Foundry Models സേവന പ്രൊവിഷനിംഗ്
- Cognitive Search ഇന്റഗ്രേഷൻ
- സുരക്ഷിത കീ മാനേജ്‌മെന്റ്
- നെറ്റ്‌വർക്ക് സുരക്ഷാ ക്രമീകരണങ്ങൾ

### **ചർച്ചാ വിഷയം:** AI-ക്കായി ഈ പാറ്റേണുകൾ എന്തുകൊണ്ട് പ്രധാനമാണ്

- **സർവീസ് ആശ്രിതത്വങ്ങൾ**: AI ആപ്ലിക്കേഷനുകൾ സാധാരണയായി പല സേവനങ്ങളും ഒരുമിച്ച് ആവശ്യമാണ്
- **സുരക്ഷ**: API കീകളും എൻഡ്പോയിന്റുകളും സുരക്ഷിതമായി കൈകാര്യം ചെയ്യേണ്ടത്
- **സ്കേലബിലിറ്റി**: AI വർക്ക്ലോഡുകൾക്ക് പ്രത്യേക സ്കേലിംഗുകൾ ആവശ്യമാണ്
- **ചെലവ് നിയന്ത്രണം**: AI സേവനങ്ങൾ ശരിയായി കോൺഫിഗർ ചെയ്തില്ലെങ്കിൽ ചെലവ് ഉയരും

## മോഡ്യൂള്‍ 2: നിങ്ങളുടെ ആദ്യ AI അപ്ലിക്കേഷൻ ഡിപ്പ്ലോയ് ചെയ്യുക

### ഘട്ടം 2.1: പരിസ്ഥിതി ആരംഭിക്കുക

1. **പുതിയ AZD പരിസ്ഥിതി സൃഷ്ടിക്കുക:**
```bash
azd env new myai-workshop
```

2. **ആവശ്യമായ പാരാമീറ്ററുകൾ സജ്ജമാക്കുക:**
```bash
# നിങ്ങളുടെ പ്രിയപ്പെട്ട Azure പ്രദേശം സജ്ജമാക്കുക
azd env set AZURE_LOCATION eastus

# ഐച്ഛികം: നിശ്ചിത OpenAI മോഡൽ സജ്ജമാക്കുക
azd env set AZURE_OPENAI_MODEL gpt-4.1-mini
```

### ഘട്ടം 2.2: ഇൻഫ്രാസ്ട്രക്ചറും അപ്ലിക്കേഷനും ഡിപ്പ്ലോയ്മെന്റ് ചെയ്യുക

1. **AZD ഉപയോഗിച്ച് ഡിപ്പ്ലോയ്മെന്റ്:**
```bash
azd up
```

**`azd up` നടക്കുമ്പോൾ происходит:**
- ✅ Microsoft Foundry Models സേവനം പ്രൊവിഷൻ ചെയ്യുന്നു
- ✅ Cognitive Search സേവനം സൃഷ്ടിക്കുന്നു
- ✅ വെബ് അപ്ലിക്കേഷനുവേണ്ടി ആപ്പ് സർവീസ് സെറ്റ് ചെയ്യുന്നു
- ✅ നെറ്റ്‌വർക്ക് സുരക്ഷയും ക്രമീകരിക്കുന്നു
- ✅ അപ്ലിക്കേഷൻ കോഡ് ഡിപ്പ്ലോയു ചെയ്യുന്നു
- ✅ നിരീക്ഷണവും ലോഗിംഗും സജ്ജീകരിക്കുന്നു

2. **ഡിപ്പ്ലോയ്മെന്റ് പുരോഗതി നിരീക്ഷിക്കുക** ഒപ്പം സൃഷ്ടിക്കപ്പെടുന്ന റിസോഴ്‌സുകൾ കുറക്കുക.

### ഘട്ടം 2.3: നിങ്ങളുടെ ഡിപ്പ്ലോയ്മെന്റ് പരിശോദിക്കുക

1. **ഡിപ്പ്ലോയ്മെന്റ് റിസോഴ്‌സുകൾ പരിശോധിക്കുക:**
```bash
azd show
```

2. **ഡിപ്പ്ലോയ്മെന്റ് അപ്ലിക്കേഷൻ തുറക്കുക:**
```bash
azd show
```

`azd show` ഔട്ട്‌പുട്ടിൽ കാണിക്കുന്ന വെബ് എൻഡ്പോയിന്റ് തുറക്കുക.

3. **AI പ്രവർത്തനം പരിശോദിക്കുക:**
   - വെബ് അപ്ലിക്കേഷനിലേക്ക് പോകുക
   - സാമ്പിൾ ക്വെറികൾ പരീക്ഷിക്കുക
   - AI പ്രതികരണങ്ങൾ പ്രവർത്തിക്കുന്നുണ്ടെന്ന് ഉറപ്പാക്കുക

### **ലാബ് വ്യായാമം 2.1: പ്രശ്നപരിഹാരം അഭ്യാസം**

**സാന്റാരിയോ**: നിങ്ങളുടെ ഡിപ്പ്ലോയ്മെന്റ് വിജയിച്ചെങ്കിലും AI പ്രതികരിക്കുന്നില്ല.

**സാധാരണ പരിശോധിക്കേണ്ട പ്രശ്നങ്ങൾ:**
1. **OpenAI API കീസുകൾ**: ശരിയായി ക്രമീകരിച്ചിട്ടുണ്ടോ എന്ന് പരിശോധിക്കുക
2. **മോഡൽ ലഭ്യത**: നിങ്ങളുടെ റീജിയനിൽ മോഡൽ ലഭ്യമാണോ എന്നത് പരിശോധിക്കുക
3. **നെറ്റ്‌വർക്ക് കണക്ടിവിറ്റി**: സേവനങ്ങൾ തമ്മിൽ ആശയവിനിമയം സാധ്യമാണോ എന്ന് ഉറപ്പാക്കുക
4. **RBAC അനുമതികൾ**: ആപ്പ് OpenAI ആക്‌സസ് ചെയ്യുന്നതിന് അനുമതിയുണ്ട് എങ്കിൽ പരിശോധിക്കുക

**ഡീബഗിംഗ് കമാൻഡുകൾ:**
```bash
# പരിസ്ഥിതി ചഞ്ചലങ്ങൾ പരിശോധിക്കുക
azd env get-values

# വിന്യസന ലോഗുകൾ കാണുക
az webapp log tail --name YOUR_APP_NAME --resource-group YOUR_RG

# OpenAI വിന്യസന നില പരിശോധിക്കുക
az cognitiveservices account deployment list --name YOUR_OPENAI_NAME --resource-group YOUR_RG
```

### ഘട്ടം 3.1: AI കോൺഫിഗറേഷൻ മാറ്റുക

1. **OpenAI മോഡൽ അപ്‌ഡേറ്റ് ചെയ്യുക:**
```bash
# വ്യത്യസ്ത മോഡലിലേക്ക് മാറുക (നിങ്ങളുടെ പ്രദേശത്ത് ലഭ്യമെങ്കിൽ)

#### പ്രശ്നം 3: അനുമതി പ്രശ്നങ്ങൾ

  --scope /subscriptions/YOUR_SUB/resourceGroups/YOUR_RG
```

### പ്രകടന പ്രശ്നങ്ങൾ

#### പ്രശ്നം 4: ക slows AI പ്രതികരണങ്ങൾ
**ഗവേഷണഘട്ടങ്ങൾ:**
1. Application Insights പ്രകടന മീട്രിക്സ് പരിശോധിക്കുക
2. Azure പോർട്ടലിൽ OpenAI സേവന മീട്രിക്സ് അവലോകനം ചെയ്യുക
3. നെറ്റ്‌വർക്ക് കണക്ടിവിറ്റി, ലാറ്റൻസി പരിശോധിക്കുക

**പരിഹാരങ്ങൾ:**
- സാധാരണ ക്വെറികൾക്ക് കാഷിംഗ് നടപ്പാക്കുക
- നിങ്ങളുടെ ഉപയോഗത്തിനനുസരിച്ച് അനുയോജ്യമായ OpenAI മോഡൽ ഉപയോഗിക്കുക
- ഉയർന്ന ലോഡിനായി റീഡ് റിപ്പ്ലിക്കാസുകൾ പരിഗണിക്കുക

### **ലാബ് വ്യായാമം 6.1: ഡീബഗിംഗ് വെല്ലുവിളി**

**സാന്റാരിയോ**: നിങ്ങളുടെ ഡിപ്പ്ലോയ്മെന്റ് വിജയിച്ചെങ്കിലും അപ്ലിക്കേഷൻ 500 പിശകുകൾ നൽകുന്നു.

**ഡീബഗിങ്ങ് കർമങ്ങൾ:**
1. അപ്ലിക്കേഷൻ ലോഗുകൾ പരിശോധിക്കുക
2. സേവന ബന്ധം സ്ഥിരീകരിക്കുക
3. Authentication പരിശോധന നടത്തുക
4. കോൺഫിഗറേഷൻ അവലോകനം ചെയ്യുക

**ഉപകരണങ്ങൾ:**
- ഡിപ്പ്ലോയ്മെന്റ് അവലോകനത്തിന് `azd show`
- സേവന ലോഗുകൾക്കായി Azure പോർട്ടൽ
- അപ്ലിക്കേഷൻ ടెలിമെറ്റ്രിക്ക് പരിശോധിക്കാൻ Application Insights

## മോഡ്യൂള്‍ 7: നിരീക്ഷണവും മെച്ചപ്പെടുത്തലും

### ഘട്ടം 7.1: സമഗ്ര നിരീക്ഷണം സജ്ജമാക്കുക

1. **കസ്റ്റം ഡാഷ്ബോർഡുകൾ സൃഷ്ടിക്കുക:**

Azure പോർട്ടലിലേക്ക് നാവിഗേറ്റ് ചെയ്ത് ഡാഷ്ബോർഡ് സൃഷ്ടിക്കുക, ഇവിടെ ഉൾപ്പെടുന്നവ:
- OpenAI അഭ്യർത്ഥനകൾ എണ്ണം, ലാറ്റൻസി
- അപ്ലിക്കേഷൻ പിശകുകൾ
- വിഭവ ഉപയോഗം
- ചെലവ് ട്രാക്കിംഗ്

2. **അലേർട്ടുകൾ സജ്ജീകരിക്കുക:**
```bash
# ഉയർന്ന പിശക് നിരക്കിനുള്ള ജാഗ്രത
az monitor metrics alert create \
  --name "AI-App-High-Error-Rate" \
  --resource-group YOUR_RG \
  --target-resource-id YOUR_APP_ID \
  --condition "avg Http5xx greater than 10" \
  --description "Alert when error rate is high"
```

### ഘട്ടം 7.2: ചെലവ് മെച്ചപ്പെടുത്തല്‍

1. **നിലവിലെ ചെലവുകൾ വിശകലനം ചെയ്യുക:**
```bash
# ചെലവ് ഡേറ്റ നേടാൻ Azure CLI ഉപയോഗിക്കുക
az consumption usage list --start-date 2024-01-01 --end-date 2024-01-31
```

2. **ചെലവ് നിയന്ത്രണങ്ങൾ നടപ്പിലാക്കുക:**
- ബഡ്ജറ്റ് അലേർട്ടുകൾ സജ്ജമാക്കുക
- ഓട്ടോസ്കെയ്‌ലിംഗ് നയങ്ങൾ പ്രയോഗിക്കുക
- അഭ്യർത്ഥന കാഷിംഗ് നടപ്പാക്കുക
- OpenAI ടോക്കൺ ഉപയോഗം നിരീക്ഷിക്കുക

### **ലാബ് വ്യായാമം 7.1: പ്രകടന മെച്ചപ്പെടുത്തലുകൾ**

**കർമ്മം**: നിങ്ങളുടെ AI അപ്ലിക്കേഷൻ പ്രകടനത്തിനും ചെലവിനും അനുയോജ്യമായി മെച്ചപ്പെടുത്തുക.

**മീറ്റ്രിക്സ് മെച്ചപ്പെടുത്തുക:**
- ശരാശരി പ്രതികരണ സമയം 20% കുറഞ്ഞത്
- മാസവ്യാപക ചെലവ് 15% കുറഞ്ഞത്
- 99.9% അപ്ഡൈം നിലനിർത്തുക

**പ്രയാസങ്ങൾ പരീക്ഷിക്കുക:**
- പ്രതികരണ കാഷിംഗ് നടപ്പാക്കുക
- ടോക്കൺ കാര്യക്ഷമതക്കായി പ്രോംപ്റ്റുകൾ 최적화 ചെയ്യുക
- അനുയോജ്യമായ കംപ്യൂട്ട് SKUs ഉപയോഗിക്കുക
- ശരിയായ ഓട്ടോസ്കെയ്‌ലിംഗ് സജ്ജീകരിക്കുക

## അന്തിമ വെല്ലുവിളി: എന്റ്-ടു-എന്റ് നടപ്പാക്കല്‍
