## വർക്ക്‌ഷോപ്പ് അവലോകനം

ഈ ഹാൻഡ്‌സ്-ഓൺ ലാബ് വികസിപ്പകർക്ക് നിലവിലുള്ള AI ടെംപ്ലേറ്റ് എടുത്ത് Azure Developer CLI (AZD) ഉപയോഗിച്ച് ഡിപ്ലോയുചെയ്യുന്നതിനുള്ള പ്രക്രിയയിലൂടെ മാർഗ്ഗനിർദ്ദേശം നൽകുന്നു. Microsoft Foundry സേവനങ്ങൾ ഉപയോഗിച്ച് പ്രൊഡക്ഷൻ AI ഡിപ്ലോയ്മെന്റുകൾക്കുള്ള അനിവാര്യമായ മാതൃകകൾ നിങ്ങൾക്ക് പഠിക്കാം.

- [Azure CLI](https://docs.microsoft.com/cli/azure/install-azure-cli) ഇൻസ്റ്റാൾ ചെയ്തിരിക്കുന്നത്

- കമാൻഡ്-ലൈൻ ഇന്റർഫേസുകളോട് പരിചയം
- അടിസ്ഥാന AI/ML ആശയങ്ങൾ (APIകൾ, മോഡലുകൾ, പ്രോംപ്റ്റുകൾ)

## ലാബ് സജ്ജീകരണം

### ഘട്ടം 1: പരിസ്ഥിതി തയ്യാറാകൽ

1. **ഉപകരണ ഇൻസ്റ്റാളേഷനുകൾ സ്ഥിരീകരിക്കുക:**
```bash
# AZD ഇൻസ്റ്റാളേഷൻ പരിശോധിക്കുക
azd version

# Azure CLI പരിശോധിക്കുക
az --version

# AZD വർക്ഫ്ലോകൾക്കായി Azure ലോഗിൻ ചെയ്തു
azd auth login

# ഡയഗ്നോസ്റ്റിക് സമയത്ത് az കമാൻഡുകൾ പ്രവർത്തിപ്പിക്കാൻ ഉദ്ദേശിക്കുന്ന പക്ഷം മാത്രമേ Azure CLI ലോഗിൻ ചെയ്യാവു്
az login
```

നിങ്ങൾ പല ടെനന്റുകളിലും പ്രവർത്തിക്കുന്നുവെങ്കിൽ അല്ലെങ്കിൽ നിങ്ങളുടെ സബ്സ്ക്രിപ്ഷൻ സ്വയം കണ്ടെത്തിച്ചില്ലെങ്കിൽ, `azd auth login --tenant-id <tenant-id>` ഉപയോഗിച്ച് വീണ്ടും ശ്രമിക്കുക.

2. **വർക്ക്‌ഷോപ്പ് റിപോസിറ്ററി ക്ലോൺ ചെയ്യുക:**
```bash
git clone https://github.com/Azure-Samples/azure-search-openai-demo
cd azure-search-openai-demo
```

## മോഡ്യൂൾ 1: AI അപ്ലിക്കേഷനുകൾക്കായുള്ള AZD ഘടന മനസ്സിലാക്കൽ

### ഒരു AI AZD ടെംപ്ലറ്റിന്റെ ഘടന

AI-സജ്ജമായ AZD ടെംപ്ലേറ്റിലെ പ്രധാന ഫയലുകൾ പരിശോധിക്കുക:

```
azure-search-openai-demo/
├── azure.yaml              # AZD configuration
├── infra/                   # Infrastructure as Code
│   ├── main.bicep          # Main infrastructure template
│   ├── main.parameters.json # Environment parameters
│   └── modules/            # Reusable Bicep modules
│       ├── openai.bicep    # Microsoft Foundry Models configuration
│       ├── search.bicep    # Azure AI Search setup
│       └── webapp.bicep    # Web app configuration
├── app/                    # Application code
├── scripts/               # Deployment scripts
└── .azure/               # AZD environment files
```

### **ലാബ് വ്യായാമം 1.1: കോൺഫിഗറേഷൻ പരിശോധിക്കുക**

1. **azure.yaml ഫയൽ പരിശോധിക്കുക:**
```bash
cat azure.yaml
```

**എന്തെഴുതണമെന്ന് നോക്കേണ്ടത്:**
- AI ഘടകങ്ങളുടെ സേവന നിർവചനങ്ങൾ
- അന്തരീക്ഷ വ്യത്യാസങ്ങളുടെ മാപ്പിംഗ്
- ഹോസ്റ്റ് കോൺഫിഗറേഷനുകൾ

2. **main.bicep ഇൻഫ്രാസ്ട്രക്ചർ അവലോകനം ചെയ്യുക:**
```bash
cat infra/main.bicep
```

**പറയേണ്ട പ്രധാന AI മാതൃകകൾ:**
- Microsoft Foundry മോഡൽ സർവീസ് പ്രൊവിഷനിംഗ്
- Azure AI Search ഇന്റഗ്രേഷൻ
- സുരക്ഷിത കീ മാനേജ്‌മെന്റ്
- നെറ്റ്‌വർക്ക് സുരക്ഷാ കോൺഫിഗറേഷനുകൾ

### **ചർച്ചാ വിഷയങ്ങൾ:** AI-ക്കായുള്ള ഈ മാതൃകകൾ എന്തുകൊണ്ട് പ്രധാനമാണ്

- **സേവന ആശ്രിതത്വങ്ങൾ**: AI ആപ്ലിക്കേഷനുകൾക്ക് പല കോ-ഓർഡിനേറ്റഡ് സേവനങ്ങൾ ആവശ്യമാണ്
- **സുരക്ഷ**: API കീകൾക്കും എന്റ്പോയിന്റുകൾക്കും സുരക്ഷിത മാനേജ്‌മെന്റ് ആവശ്യമാണ്
- **സ്കെയിലബിലിറ്റി**: AI വർക്ക്‌ലോഡുകൾക്കു പ്രത്യേക സ്‌കെയിലിങ് ആവശ്യമുണ്ട്
- **ചെലവ് മാനേജ്‌മെന്റ്**: ശരിയായി കോൺഫിഗർ ചെയ്യാതിരുന്നാൽ AI സേവനങ്ങൾ ചെലവേറിയതാണ്

## മോഡ്യൂൾ 2: നിങ്ങളുടെ ആദ്യ AI അപ്ലിക്കേഷൻ ഡിപ്ലോയ് ചെയ്യുക

### ഘട്ടം 2.1: പരിസ്ഥിതി ആരംഭിപ്പിക്കുക

1. **പുതിയ AZD പരിസ്ഥിതി സൃഷ്ടിക്കുക:**
```bash
azd env new myai-workshop
```

2. **അവശ്യ പാരാമീറ്ററുകൾ സജ്ജമാക്കുക:**
```bash
# നിങ്ങളുടെ ഇഷ്ടമുള്ള അജ്യൂർ പ്രദേശം സെറ്റ് ചെയ്യുക
azd env set AZURE_LOCATION eastus

# ഐച്ഛികം: പ്രത്യേക OpenAI മോഡൽ സെറ്റ് ചെയ്യുക
azd env set AZURE_OPENAI_MODEL gpt-4.1-mini
```

### ഘട്ടം 2.2: ഇൻഫ്രാസ്ട്രക്ചർ, അപ്ലിക്കേഷൻ ഡിപ്ലോയ് ചെയ്യുക

1. **AZD ഉപയോഗിച്ച് ഡിപ്ലോയ് ചെയ്യുക:**
```bash
azd up
```

**`azd up` നടക്കുമ്പോൾ സംഭവിക്കുന്നത്:**
- ✅ Microsoft Foundry മോഡൽ സർവീസ് പ്രൊവിഷൻ ചെയ്യുന്നു
- ✅ Azure AI Search സർവീസ് സൃഷ്ടിക്കുന്നു
- ✅ വെബ് അപ്ലിക്കേഷനായി ആപ്പ് സർവീസ് സജ്ജമാക്കുന്നു
- ✅ നെറ്റ്‌വർക്ക്, സുരക്ഷ സജ്ജീകരിക്കുന്നു
- ✅ അപ്ലിക്കേഷൻ കോഡ് ഡിപ്ലോയ് ചെയ്യുന്നു
- ✅ മോണിറ്ററിംഗ്, ലോഗിംഗ് സജ്ജമാക്കുന്നു

2. **ഡിപ്ലോയ്മെന്റ് പുരോഗതി നിരീക്ഷിക്കുക** ഡിപ്ലോയ് ചെയ്യപ്പെടുന്ന വൻപെട്ടികൾ ശ്രദ്ധിക്കുക.

### ഘട്ടം 2.3: ഡിപ്ലോയ്മെന്റ് പരിശോധിക്കുക

1. **ഡിപ്ലോയ്ചെയ്ത വൻപെട്ടികൾ പരിശോധിക്കുക:**
```bash
azd show
```

2. **ഡിപ്ലോയ്ചെയ്ത അപ്ലിക്കേഷൻ തുറക്കുക:**
```bash
azd show
```

`azd show` ഔട്ട്പുട്ടിൽ കാണിക്കുന്ന വെബ് എണ്ട്പോയിന്റ് തുറക്കുക.

3. **AI പ്രവർത്തനം പരിശോധിക്കുക:**
   - വെബ് അപ്ലിക്കേഷൻത്തിലേക്ക് പ്രവേശിക്കുക
   - സാമ്പിൾ ക്വെറിയുകൾ ശ്രമിക്കുക
   - AI പ്രതികരണങ്ങൾ ശരിയായി പ്രവർത്തിക്കുന്നുണ്ടോ പരിശോധിക്കുക

### **ലാബ് വ്യായാമം 2.1: പ്രശ്നനിവാരണ പരിശീലനം**

**സാഹചര്യം**: നിങ്ങളുടെ ഡിപ്ലോയ്മെന്റ് വിജയിച്ചു, പക്ഷേ AI പ്രതികരിക്കുന്നില്ല.

**പരിശോധിക്കേണ്ട പൊതുവായ പ്രശ്നങ്ങൾ:**
1. **OpenAI API കീകൾ**: ശരിയായി സജ്ജമാണോ എന്ന് പരിശോധിക്കുക
2. **മോഡൽ ലഭ്യത**: നിങ്ങളുടെ പ്രദേശം മോഡലിന് പിന്തുണ നൽകുമോ പരിശോധിക്കുക
3. **നെറ്റ്‌വർക്ക് ബന്ധം**: സേവനങ്ങൾ തമ്മിൽ സംവദിക്കാനാകുമോ പരിശോധിക്കുക
4. **RBAC അനുവാദങ്ങൾ**: ആപ്പ് OpenAI-യിലേക്ക് ആക്‌സസ് പ്രവർത്തിക്കുന്നുണ്ടോ പരിശോധിക്കുക

**ഡി ബഗിംഗ് കമാൻഡുകൾ:**
```bash
# പരിസ്ഥിതി വ്യത്യാസങ്ങൾ പരിശോധിക്കുക
azd env get-values

# ടെൻറെ ലോക്കുകൾ കാണുക

### ഘട്ടം 5.2: നിങ്ങളുടെ AZD ടെംപ്ലേറ്റ് സൃഷ്ടിക്കുക

# റോളിന്റെ നിയോഗങ്ങൾ പരിശോധിക്കുക
az role assignment list --scope /subscriptions/YOUR_SUB/resourceGroups/YOUR_RG

# നഷ്ടപ്പെട്ട റോളുകൾ ചേർക്കുക
az role assignment create \
  --assignee YOUR_PRINCIPAL_ID \
  --role "Cognitive Services OpenAI User" \
  --scope /subscriptions/YOUR_SUB/resourceGroups/YOUR_RG
```

### **ലാബ് വ്യായാമം 6.1: ഡീബഗ്ഗിംഗ് വെല്ലുവിളി**

**സാഹചര്യം**: നിങ്ങളുടെ ഡിപ്ലോയ്മെന്റ് വിജയിച്ചു, എന്നാൽ അപ്ലിക്കേഷൻ 500 errors നൽകുന്നു.

**ഡി-ബഗ്ഗിംഗ് ടാസ്കുകൾ:**

2. സേവന കണക്ടിവിറ്റി പരിശോധിക്കുക
3. ഓതന്റിക്കേഷൻ പരിശോദിക്കുക
4. കോൺഫിഗറേഷൻ അവലോകനം ചെയ്യുക

**ഉപകരണങ്ങൾ:**
- ഡിപ്ലോയ്മെന്റ് അവലോകനത്തിന് `azd show`
- സർവീസ് ലൈവ് ലോഗുകൾക്കായി Azure പോർട്ടൽ
- അപ്ലിക്കേഷൻ ടെലിമെട്രിക്ക്‌സിനായി Application Insights

## മോഡ്യൂൾ 7: മോണിറ്ററിംഗും ഓപ്റ്റിമൈസേഷനും

### ഘട്ടം 7.1: സമഗ്രമായ മോണിറ്ററിംഗ് സ്ഥാപിക്കുക

1. **കസ്റ്റം ഡാഷ്ബോർഡുകൾ സൃഷ്ടിക്കുക:**

Azure പോർട്ടലിലോട്ട് പോകുകയും താഴെപ്പറയുന്നവ ഉൾപ്പെടുന്ന ഡാഷ്ബോർഡ് സൃഷ്ടിക്കുകയും ചെയ്യുക:
- OpenAI അഭ്യർത്ഥനകൾ എണ്ണവും ലേട്ടൻസിയും
- അപ്ലിക്കേഷൻ പിശക് നിരക്കുകൾ
- വൻപെട്ടികളിൽ ഉപയോഗം
- ചെലവ് ട്രാക്കിംഗ്

2. **അലർട്ടുകൾ സജ്ജമാക്കുക:**
```bash
# ഉന്നത പിശക് നിരക്കിനുള്ള മുന്നറിയിപ്പ്
az monitor metrics alert create \
  --name "AI-App-High-Error-Rate" \
  --resource-group YOUR_RG \
  --target-resource-id YOUR_APP_ID \
  --condition "avg Http5xx greater than 10" \
  --description "Alert when error rate is high"
```

### **ലാബ് വ്യായാമം 7.1: പ്രകടന ഓപ്റ്റിമൈസേഷൻ**

**ടാസ്ക്**: നിങ്ങളുടെ AI അപ്ലിക്കേഷൻ പ്രകടനത്തിലും ചെലവിലും Оп്റ്റിമൈസ് ചെയ്യുക.
