### ഘട്ടം 1: പരിസ്ഥിതി തയ്യാറാക്കൽ

1. **ടൂൾസ് ഇൻസ്റ്റലേഷൻ പരിശോധിക്കുക:**
```bash
# AZD ഇൻസ്റ്റലേഷൻ പരിശോധിക്കുക
azd version

# Azure CLI പരിശോധിക്കുക
az --version

# AZD പ്രവർത്തനങ്ങൾക്ക് Azure ലോഗിൻ ചെയ്യുക
azd auth login

# ഡയഗ്നോസ്റ്റിക് സമയത്ത് az കമാൻഡുകൾ പ്രവർത്തിക്കാനാണ് ഉദ്ദേശിച്ചത് എങ്കിൽ മാത്രമേ Azure CLI ലോഗിൻ ചെയ്യപ്പെടേണ്ടത്
az login
```

നീങ്ങൾ നിരവധി ടെനന്റുകളിൽ ജോലി ചെയ്യുന്നുവെങ്കിൽ അല്ലെങ്കിൽ നിങ്ങളുടെ സബ്‌സ്‌ക്രിപ്ഷൻ സ്വയം കണ്ടിട്ടില്ലെങ്കിൽ, `azd auth login --tenant-id <tenant-id>` ഉപയോഗിച്ച് വീണ്ടും ശ്രമിക്കുക.

2. **വർക്ക്‌ഷോപ്പ് റിപ്പോ ക്ലോൺ ചെയ്യുക:**
```bash
git clone https://github.com/Azure-Samples/azure-search-openai-demo
cd azure-search-openai-demo
```

## മോഡ്യൂൾ 1: AI ആപ്ലിക്കേഷനുകൾക്കുള്ള AZD ഘടന അറിയുക

### AI AZD ടെംപ്ലേറ്റിന്റെ ഘടന

AI-സന്ദർശനത്തിന് അനുയോജ്യമായ AZD ടെംപ്ലേറ്റിലെ മുഖ്യ ഫയലുകൾ പരിശോധിക്കുക:

```
azure-search-openai-demo/
├── azure.yaml              # AZD configuration
├── infra/                   # Infrastructure as Code
│   ├── main.bicep          # Main infrastructure template
│   ├── main.parameters.json # Environment parameters
│   └── modules/            # Reusable Bicep modules
│       ├── openai.bicep    # Microsoft Foundry Models configuration
│       ├── search.bicep    # Azure AI Search setup
│       └── webapp.bicep    # Web app configuration
├── app/                    # Application code
├── scripts/               # Deployment scripts
└── .azure/               # AZD environment files
```

### **ലാബ് വ്യായാമം 1.1: കോൺഫിഗർ പരിശോധിക്കുക**

1. **azure.yaml ഫയൽ പരിശോധിക്കുക:**
```bash
cat azure.yaml
```

**കണ്ടെത്തേണ്ടത്:**
- AI ഘടകങ്ങളുടേയും സേവന നിർവചനങ്ങൾ
- പരിസ്ഥിതി ചറയ്ട്ടപ്പറിവ് മാപ്പിൽക്കൾ
- ഹോസ്റ്റ് കോൺഫിഗറേഷൻസ്

2. **main.bicep ഇൻഫ്രاس്ട്രക്ചർ നിരീക്ഷിക്കുക:**
```bash
cat infra/main.bicep
```

**അവിടത്തുള്ള പ്രധാന AI പാറ്റേണുകൾ:**
- Microsoft Foundry Models സേവന പ്ര Provisioning
- Azure AI Search സംയോജനം
- സുരക്ഷിത കീ മാനേജ്മെന്റ്
- നെറ്റ്‌വർക്ക് സുരക്ഷ കോൺഫിഗറേഷൻ

### **സംവാദം:** ഈ പാറ്റേണുകൾ AI-ക്കു എന്ത سببമാണ്?

- **സേവന ആശ്രിതത്വങ്ങൾ**: AI ആപ്ലിക്കേഷനുകൾ പലസ് സർവീസുകളുടെ ഏകോപനം ആവശ്യമാണ്
- **സുരക്ഷ**: API കീകൾക്കും എൻഡ്പോയിന്റുകൾക്കും സുരക്ഷിത മാനേജ്മെന്റ് ആവശ്യമുണ്ട്
- **സ്കേലബിലിറ്റി**: AI വർക്ക്ലോഡുകൾക്കായി വ്യത്യസ്ത സ്കെയിലിംഗ് ആവശ്യകതകൾ ഉണ്ട്
- **ചെലവ് നിയന്ത്രണം**: AI സേവനങ്ങൾ ശരിയായി കോൺഫിഗർ ചെയ്തില്ലെങ്കിൽ ചെലവുകൾ കൂടുതലാകും

## മോഡ്യൂൾ 2: നിങ്ങളുടെ ആദ്യ AI ആപ്ലിക്കേഷൻ ഡെപ്ലോയ്ചെയ്യുക

### ഘട്ടം 2.1: പരിസ്ഥിതി ആരംഭിക്കുക

1. **പുതിയ AZD പരിസ്ഥിതി സൃഷ്ടിക്കുക:**
```bash
azd env new myai-workshop
```

2. **ആവശ്യമായ പരിഗണനകൾ സജ്ജീകരിക്കുക:**
```bash
# നിങ്ങളുടെ ഇഷ്ടമുള്ള Azure പ്രദേശം സജ്ജീകരിക്കുക
azd env set AZURE_LOCATION eastus

# ഐച്ഛികം: പ്രത്യേക OpenAI മോഡൽ സജ്ജീകരിക്കുക
azd env set AZURE_OPENAI_MODEL gpt-4.1-mini
```

### ഘട്ടം 2.2: ഇൻഫ്രാസ്ട്രക്ചറും ആപ്ലിക്കേഷനും ഡെപ്ലോയ് ചെയ്യുക

1. **AZD ഉപയോഗിച്ച് ഡെപ്ലോയ് ചെയ്യുക:**
```bash
azd up
```

`azd up` നടത്തുമ്പോൾ സംഭവിക്കുന്നതിൽ:
- ✅ Microsoft Foundry Models സേവനം പ്രവിഷൻ ചെയ്യുന്നു
- ✅ Azure AI Search സേവനം സൃഷ്ടിക്കുന്നു
- ✅ വെബ് ആപ്ലിക്കേഷനുള്ള ആപ്പ് സർവീസ് സജ്ജമാക്കുന്നു
- ✅ നെറ്റ്‌വർക്ക്, സുരക്ഷാ ക്രമീകരണങ്ങൾ നടപ്പിലാക്കുന്നു
- ✅ ആപ്ലിക്കേഷൻ കോഡ് ഡെപ്ലോയ്ചെയ്യുന്നു
- ✅ മോണിറ്ററിംഗ്, ലോഗിംഗ് സജ്ജീകരിക്കുന്നു

2. **ഡെപ്ലോയ്മെന്റ് പുരോഗതി നിരീക്ഷിക്കുക** കൂടാതെ സൃഷ്‌ടിച്ചിരുന്ന റിസോഴ്‌സുകൾ ശ്രദ്ധിക്കുക.

### ഘട്ടം 2.3: ഡെപ്ലോയ്മെന്റ് സ്ഥിരീകരിക്കുക

1. **ഡെപ്ലോയുചെയ്‌ത റിസോഴ്‌സുകൾ പരിശോധിക്കുക:**
```bash
azd show
```

2. **ഡെപ്ലോയുചെയ്‌ത ആപ്ലിക്കേഷൻ തുറക്കുക:**
```bash
azd show
```

`azd show` ഔട്ട്‌പുട്ടിൽ കാണിച്ച വെബ് എൻഡ്പോയിന്റ് തുറക്കുക.

3. **AI പ്രവർത്തനം പരിശോധിക്കുക:**
   - വെബ് ആപ്ലിക്കേഷനിൽ നാവിഗേറ്റ് ചെയ്യുക
   - സാമ്പിൾ ചോദ്യങ്ങൾ പരീക്ഷിക്കുക
   - AI പ്രതികരണങ്ങൾ പ്രവർത്തിക്കുന്നുവെന്ന് ഉറപ്പുവരുത്തുക

### **ലാബ് വ്യായാമം 2.1: പ്രശ്‌നപരിഹാര പരീക്ഷണം**

**സംഭവം**: ഡെപ്ലോയ്മെന്റ് വിജയമായി, എങ്കിലും AI പ്രതികരിക്കുന്നില്ല.

**സാധാരണ പരിശോധിക്കേണ്ട കാര്യങ്ങൾ:**
1. **OpenAI API കീകൾ**: ശരിയായി സജ്ജമാക്കിയിട്ടുണ്ടോ പരിശോധിക്കുക
2. **മോഡൽ ലഭ്യത**: നിങ്ങളുടെ മേഖല മോഡൽ പിന്തുണയുണ്ടോ പരിശോധിക്കുക
3. **നെറ്റ്‌വർക്ക് കണക്ടിവിറ്റി**: സേവനങ്ങൾ തമ്മിൽ ബന്ധപ്പെടാൻ കഴിയും എന്നതായി ഉറപ്പ് വരുത്തുക
4. **RBAC അനുമതികൾ**: ആപ്പ് OpenAI ആക്‌സസ് ചെയ്യാനാകുന്നുണ്ടോ പരിശോധിക്കുക

**ഡീബഗിംഗ് കമാൻഡുകൾ:**
```bash
# ماحولیات വേരിയബിളുകൾ പരിശോധിക്കുക
azd env get-values

# വിന്യസിപ്പിക്കൽ ലോഗുകൾ കാണുക
az webapp log tail --name YOUR_APP_NAME --resource-group YOUR_RG

# OpenAI വിന്യസിക്കൽ നില പരിശോധിക്കുക
az cognitiveservices account deployment list --name YOUR_OPENAI_NAME --resource-group YOUR_RG
```

## മോഡ്യൂൾ 3: നിങ്ങളുടെ ആവശ്യങ്ങൾക്ക് AI ആപ്ലിക്കേഷൻ കസ്റ്റമൈസ് ചെയ്യുക

### ഘട്ടം 3.1: AI കോൺഫിഗറേഷൻ മാറ്റുക

1. **OpenAI മോഡൽ അപ്ഡേറ്റ് ചെയ്യുക:**

### ഘട്ടം 5.2: നിങ്ങളുടെ AZD ടെംപ്ലേറ്റ് സൃഷ്ടിക്കുക

**ഈ പാറ്റേൺ പിന്തുടർന്ന് നിങ്ങളുടെ ആപ്പ് മാറ്റുക:**

1. **അടിസ്ഥാന ഘടന സൃഷ്ടിക്കുക:**
```bash
mkdir my-ai-app-azd
cd my-ai-app-azd

# AZD ടെംപ്ലേറ്റ് ആരംഭിക്കുക
azd init --template minimal
```

2. **azure.yaml സൃഷ്ടിക്കുക:**
```yaml
# Metadata
name: my-ai-app
metadata:
  template: my-ai-app-template@0.0.1-beta

# Services definition
services:
  api:
    project: ./api
    host: containerapp
  web:
    project: ./web
    host: staticwebapp
    
# Hooks for custom deployment logic  
hooks:
  predeploy:
    shell: sh
    run: echo "Preparing AI models..."
```

3. **ഇൻഫ്രാസ്ട്രക്ചർ ടെംപ്ലേറ്റുകൾ സൃഷ്ടിക്കുക:**

**infra/main.bicep** - മെയിൻ ടെംപ്ലേറ്റ്:
```bicep
@description('Primary location for all resources')
param location string = resourceGroup().location

@description('Name of the OpenAI service')
param openAIServiceName string = 'openai-${uniqueString(resourceGroup().id)}'

// Your AI services here
module openAI 'modules/openai.bicep' = {
  name: 'openai'
  params: {
    name: openAIServiceName
    location: location
  }
}
```

**infra/modules/openai.bicep** - OpenAI മോഡ്യൂൾ:
```bicep
@description('Name of the OpenAI service')
param name string

@description('Location for the OpenAI service')
param location string

resource openAIAccount 'Microsoft.CognitiveServices/accounts@2023-05-01' = {
  name: name
  location: location
  kind: 'OpenAI'
  sku: {
    name: 'S0'
  }
  properties: {
    customSubDomainName: name
  }
}

output endpoint string = openAIAccount.properties.endpoint
output name string = openAIAccount.name
```

#### പ്രശ്നം 1: OpenAI സെർവീസ് ക്വോട്ടാ മുകളിൽ പോയി


#### പ്രശ്നം 2: മോഡൽ നിങ്ങളുടെ മേഖലയിൽ ലഭ്യമല്ല

```bash
# പ്രദേശം അനുസരിച്ചുള്ള മോഡൽ ലഭ്യത പരിശോധിക്കുക
az cognitiveservices model list --location eastus

# ലഭ്യമായ മോഡലിലേക്ക് അപ്ഡേറ്റ് ചെയ്യുക
azd env set AZURE_OPENAI_MODEL gpt-4.1-mini
```

#### പ്രശ്നം 3: അനുമതി സംബന്ധിച്ച പ്രശ്നങ്ങൾ

```bash
  --scope /subscriptions/YOUR_SUB/resourceGroups/YOUR_RG
```

### പ്രവർത്തന കാര്യക്ഷമത പ്രശ്നങ്ങൾ

#### പ്രശ്നം 4: AI പ്രതികരണങ്ങൾ മന്ദഗതിയിലാണെന്ന്  
**പരിശോധന ഘട്ടങ്ങൾ:**
1. ആപ്ലിക്കേഷൻ ഇൻസൈറ്റ്സിൽ പ്രകടന മെട്രിക്സ് പരിശോധിക്കുക
2. Azure പോർട്ടലിൽ OpenAI സേവന മെട്രിക്സ് പരിശോധിക്കുക
3. നെറ്റ്‌വർക്ക് കണക്ടിവിറ്റിയും ലാറ്റൻസിയും പരിശോധിക്കുക

**പരിഹാരങ്ങൾ:**
- സാധാരണ ചോദ്യങ്ങൾക്ക് കാഷിങ്ങ് നടപ്പിലാക്കുക
- നിങ്ങളുടെ ഉപയോഗത്തിന് അനുയോജ്യമായ OpenAI മോഡൽ ഉപയോഗിക്കുക
- ഭാരമുള്ള സാഹചര്യം ഉള്ളപ്പോൾ റീഡ് റിപ്പ്ലിക്കകൾ പരിഗണിക്കുക

### **ലാബ് വ്യായാമം 6.1: ഡീബഗിംഗ് ചിട്ടി**

**സംഭവം:** നിങ്ങളുടെ ഡെപ്ലോയ്മെന്റ് വിജയിച്ചു, എങ്കിലും ആപ്ലിക്കേഷൻ 500 പിശകുകൾ നൽകുന്നു.

**ഡീബഗിംഗ് ടാസ്കുകൾ:**
1. ആപ്ലിക്കേഷൻ ലോഗുകൾ പരിശോധിക്കുക
2. സർവീസ് കണക്ടിവിറ്റി ഉറപ്പാക്കുക
3. സ്ഥിരീകരണം പരീക്ഷിക്കുക
4. കോൺഫിഗറേഷൻ പരിശോധിക്കുക

**ഉപകരണം:**
- `azd show` ഡെപ്ലോയ്മെന്റ് അവലോകനത്തിന്
- സർവീസ് ലോഗുകൾക്കായി Azure പോർട്ടൽ
- ആപ്ലിക്കേഷൻ ടെലിമെട്രിക്കും ആപ്ലിക്കേഷൻ ഇൻസൈറ്റ്സ്

## മോഡ്യൂൾ 7: മോണിറ്ററിംഗ്, ഓപ്റ്റിമൈസേഷൻ

### ഘട്ടം 7.1: സമഗ്ര മോണിറ്ററിംഗ് സജ്ജീകരിക്കുക

1. **കസ്റ്റം ഡാഷ്‌ബോർഡ് സൃഷ്ടിക്കുക:**

Azure പോർട്ടലിലേക്കു പ്രവേശിച്ച് താഴെ ഫീച്ചറുകൾ ചേർന്ന് ഡാഷ്ബോർഡ് ഉണ്ടാക്കുക:
- OpenAI അഭ്യർത്ഥനാ എണ്ണം, നിരക്കുകൾ
- ആപ്ലിക്കേഷൻ പിശക് നിരക്കുകൾ
- റിസോഴ്‌സ് ഉപയോഗം
- ചെലവ് ട്രാക്കിംഗ്

2. **അലേർട്ടുകൾ സജ്ജമാക്കുക:**
```bash
# ഉയർന്ന പിശക് നിരക്കിനുള്ള മുന്നറിയിപ്പ്
az monitor metrics alert create \
  --name "AI-App-High-Error-Rate" \
  --resource-group YOUR_RG \
  --target-resource-id YOUR_APP_ID \
  --condition "avg Http5xx greater than 10" \
  --description "Alert when error rate is high"
```

### ഘട്ടം 7.2: ചെലവ് ഓപ്റ്റിമൈസേഷൻ

1. **നിലവിലെ ചെലവുകൾ വിശകലനം ചെയ്യുക:**
```bash
# ചെലവ് ഡാറ്റ ലഭിക്കാൻ Azure CLI ഉപയോഗിക്കുക
az consumption usage list --start-date 2024-01-01 --end-date 2024-01-31
```

2. **ചെലവ് നിയന്ത്രണങ്ങൾ നടപ്പിലാക്കുക:**
- ബഡ്ജറ്റ് അലേർട്ട് സജ്ജമാക്കുക
- ഓട്ടോസ്കെയിലിംഗ് നയങ്ങൾ ഉപയോഗിക്കുക
- അഭ്യർത്ഥന കാഷിങ്ങ് നടപ്പിലാക്കുക
- OpenAI ടോക്കൺ ഉപയോഗം നിരീക്ഷിക്കുക

### **ലാബ് വ്യായാമം 7.1: കർമക്ഷമത ഓപ്റ്റിമൈസേഷൻ**

**ടാസ്‌ക്:** നിങ്ങളുടെ AI ആപ്ലിക്കേഷൻ സർവകലാശാലാ, ചെലവ് ഉയർത്താതെ ഓപ്റ്റിമൈസ് ചെയ്യുക.

**സംഭവം മെച്ചപ്പെടുത്താൻ:**
- ശരാശരി പ്രതികരണ സമയം 20% കുറക്കുക
- മാസേന ചെലവ് 15% കുറക്കുക
- 99.9% അപ്രാപ്യത കൂട്ടുക

**സൺവിധികൾ പരീക്ഷിക്കുക:**
- പ്രതികരണം കാഷിങ്ങ് നടപ്പിലാക്കുക
- ടോക്കൺ കാര്യക്ഷമതയ്ക്ക് പ്രോംപ്റ്റുകൾ ഒപ്റ്റിമൈസ് ചെയ്യുക
- അനുയോജ്യമായ കംപ്യൂട്ട് SKUs ഉപയോഗിക്കുക
- ശരിയായ ഓട്ടോസ്കെയിലിംഗ് സജ്ജമാക്കുക
