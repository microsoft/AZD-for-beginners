## 1. പരിചയം

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) അല്ലെങ്കിൽ `azd` എന്നത് വളരെ വിനിയോഗപ്രദമായ ഒരു ഓപ്പൺ-സോഴ്‌സ് കമാൻഡ്-ലൈൻ ടൂൾ ആണ്, Azure-യിലേക്ക് ആപ്ലിക്കേഷനുകൾ നിർമ്മിക്കുമ്പോഴും ഡിപ്ലോയ് ചെയ്യുമ്പോഴും ഡവലപ്പർ പ്രവൃത്തി പ്രക്രിയയെ സുഗമമാക്കുന്നു.

[AZD ടെംപ്ലേറ്റുകൾ](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) നിലവിലുള്ള സ്റ്റാൻഡേർഡൈസ് ചെയ്ത റപ്പോസിറ്ററികൾ ആണ്, അവയിൽ സാമ്പിൾ ആപ്ലിക്കേഷൻ കോഡ്, _ഇൻഫ്രാസ്ട്രക്ചർ-ആസ്-കോഡ്_ അസെറ്റുകളും `azd` കോൺഫിഗറേഷൻ ഫയലുകളും ഉൾപ്പെടുന്നു, ഇതിന്റെ സഹായത്തോടെ ഏകീകൃത സൊല്യൂഷൻ ആർക്കിടെക്ചർ ലഭിക്കുന്നു. ഇൻഫ്രാസ്ട്രക്ചർ പ്രൊവിഷനിംഗ് `azd provision` എന്ന കമാൻഡുപയോഗിച്ചുകൊണ്ടെയും, `azd up` ഉപയോഗിച്ച് ഇൻഫ്രാസ്ട്രക്ചർ പ്രൊവിഷനും ആപ്ലിക്കേഷൻ ഡിപ്ലോയ്മെന്റും ഒരേസമയം നടത്താം!

അതിനാൽ നിങ്ങളുടെ ആപ്ലിക്കേഷൻ വികസന പ്രക്രിയ തുടങ്ങാൻ ഏറ്റവും അനുയോജ്യമായ _AZD സ്റ്റാർട്ടർ ടെംപ്ലേറ്റ്_ കണ്ടെത്തുക അതിന്റെ അനുയായത്തിലുള്ള റിപ്പോസിറ്ററി നിങ്ങളുടെ ആവശ്യങ്ങൾക്കനുസരിച്ച് ഇഷ്ടാനുസൃതമാക്കുക എന്നതാണ് താഴെ പറയുന്നത്.

ആരംഭിക്കാൻ മുൻപ്, Azure Developer CLI ഇൻസ്റ്റാൾ ചെയ്തിട്ടുണ്ടെന്ന് ഉറപ്പാക്കാം.

1. VS കോഡ് ടെർമിനൽ തുറക്കുക, താഴെ കമാൻഡ് ടൈപ്പ് ചെയ്യുക:

      ```bash title="" linenums="0"
      azd version
      ```

1. ഇതുപോലൊരു എളുപ്പത്തിലുള്ള ഔട്ട്പുട്ട് കാണണം!

      ```bash title="" linenums="0"
      azd version 1.25.6 (commit <current-build>)
      ```

**ഇപ്പോൾ നിങ്ങൾ azd ഉപയോഗിച്ച് ടെംപ്ലേറ്റ് തിരഞ്ഞെടുത്ത് ഡിപ്ലോയ് ചെയ്യാൻ തയ്യാറാണ്**

## 2. ടെംപ്ലേറ്റ് തിരഞ്ഞെടുപ്പ്

Microsoft Foundry പ്ലാറ്റ്ഫോംവിൽ [ശൂന്യമായ AZD ടെംപ്ലേറ്റുകളുടെ ഒരു സെറ്റ്](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) ഉണ്ട്, ഇത് _മൾട്ടി-ഏജന്റ് വർക്‌ഫ്ലോ ഓട്ടോമേഷൻ_ പോലെയുള്ള പ്രശസ്തമായ സൊല്യൂഷൻ സീനാറിയോകളും _മൾട്ടി-മോഡൽ ഉള്ളടക്കം പ്രോസസിംഗ്_ ഉൾപ്പെടാണ്. ഇവ കണ്ടെത്താൻ Microsoft Foundry പോർട്ടലിലും സന്ദർശിക്കാം.

1. സന്ദർശിക്കുക [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. ആവശ്യമായപ്പോൾ Microsoft Foundry പോർട്ടലിൽ ലോഗിൻ ചെയ്യുക - ഇങ്ങനെ കാണും.

![Pick](../../../../../translated_images/ml/01-pick-template.60d2d5fff5ebc374.webp)


**Basic** ഓപ്ഷനുകൾ നിങ്ങളുടെ സ്റ്റാർട്ടർ ടെംപ്ലേറ്റുകളാണ്:

1. [ ] [AI ചാറ്റുമായി ആരംഭിക്കുക](https://github.com/Azure-Samples/get-started-with-ai-chat) ഇത് നിങ്ങളുടെ ഡാറ്റ ഉപയോഗിച്ചു Azure Container Apps-ൽ ഒരു മൂളധാരണ ചാറ്റ് ആപ്ലിക്കേഷൻ ഡിപ്ലോയ് ചെയ്യും. ഒരു അടിസ്ഥാന AI ചാറ്റ്ബോട്ട് സീനാരിയോ വിശദീകരിക്കാൻ ഉപയോഗിക്കുക.
1. [X] [AI ഏജന്റുകളുമായി ആരംഭിക്കുക](https://github.com/Azure-Samples/get-started-with-ai-agents) ഇത് സ്റ്റാൻഡേർഡ് AI ഏജന്റ്(Foundry ഏജന്റുകൾ ഉപയോഗിച്ചു) ഡിപ്ലോയ് ചെയ്യുന്നു. ടൂളുകളും മോഡലുകളും ഉൾപ്പെടുന്ന ഏജന്റിക് AI സൊല്യൂഷനുകളിൽ പരിചയപ്പെടാൻ ഉപയോഗിക്കുക.

രണ്ടാമത്തെ ലിങ്ക് പുതിയ ബ്രൗസർ ടാബിൽ തുറക്കുക (അല്ലെങ്കിൽ ബന്ധപ്പെട്ട കാർഡിൽ `Open in GitHub` ക്ലിക്ക് ചെയ്യുക). ഈ AZD ടെംപ്ലേറ്റിന്റെ റിപ്പോസിറ്ററി കാണാം. ഒരു മിനിറ്റ് ഈ README പരിശോധിക്കുക. ആപ്ലിക്കേഷൻ ആർക്കിടെക്ചർ ഇങ്ങനെ കാണാം:

## 3. ടെംപ്ലേറ്റ് ആക്ടിവേഷൻ

ഈ ടെംപ്ലേറ്റ് ഡിപ്ലോയ് ചെയ്ത് അത് സാധുവാണെന്ന് ഉറപ്പാക്കാം. [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) വിഭാഗത്തിൽ നൽകിയ മാർഗ്ഗനിർദ്ദേശങ്ങൾ അനുസരിക്കാം.

1. ടെംപ്ലേറ്റ് റിപ്പോസിറ്ററിയുടെ പ്രവർത്തനപരിസരം തിരഞ്ഞെടുക്കുക:

      - **GitHub Codespaces**: ഈ ലിങ്ക് [https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) ക്ലിക്ക് ചെയ്ത് `Create codespace` സ്ഥിരീകരിക്കുക
      - **ലോക്കൽ ക്ലോൺ അല്ലെങ്കിൽ ഡെവ് കണ്ടൈനർ**: `Azure-Samples/get-started-with-ai-agents` ക്ലോൺ ചെയ്ത് VS കോഡിൽ തുറക്കുക

1. VS കോഡ് ടെർമിനൽ തയ്യാറാകുന്നതുവരെ കാത്തിരിക്കുക, പിന്നീട് താഴെ കമാൻഡ് ടൈപ്പ് ചെയ്യുക:

   ```bash title="" linenums="0"
   azd up
   ```

ഈ കമാൻഡ് പ്രവർത്തനകാര്യങ്ങൾ പൂർത്തിയാക്കുക:

1. Azure ലോഗിൻ സാന്ദർശിക്കുക - പ്രോമ്പ്റ്റ് അനുസരിച്ച് ഓത് ചെയ്യുക
1. നിങ്ങൾക്കായി ഒരു വ്യത്യസ്തമായ പരിസരനാമം നൽകുക - ഉദാ: ഞാൻ ഉപയോഗിച്ചു `nitya-mshack-azd`
1. ഇത് ഒരു `.azure/` ഫോൾഡർ ഉണ്ടാക്കും - ഇവിടെ പരിസരനാമത്തിലുള്ള സബ്-ഫോൾഡർ കാണും
1. ഒരു ചുമതല സബ്സ്ക്രിപ്ഷൻ തിരഞ്ഞെടുക്കാൻ പ്രോംപ്റ്റ് വരും - ഡിഫോൾട്ട് തിരഞ്ഞെടുക്കുക
1. ഒരു ലൊക്കേഷൻ തിരഞ്ഞെടുക്കാൻ പ്രോംപ്റ്റ് വരും - `East US 2` തിരഞ്ഞെടുക്കുക

ഇപ്പോൾ, പ്രൊവിഷനിംഗ് പൂർത്തിയാകുന്നതിനായി കാത്തിരിക്കുക. **ഇതിനുള്ള സമയം 10-15 മിനിറ്റ്**

1. പ്രവർത്തനം പൂർത്തിയായപ്പോൾ, നിങ്ങളുടെ കൺസോളിൽ SUCCESS സന്ദേശം ഇങ്ങനെ കാണും:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. അപ്പോൾ Azure പോർട്ടലിൽ ആ പരിസരനാമം ഉള്ള പ്രൊവിഷൻ ചെയ്ത റിസോഴ്സ് ഗ്രൂപ്പ് കാണാം:

      ![Infra](../../../../../translated_images/ml/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **ഇപ്പോൾ നിങ്ങൾ ഡിപ്ലോയ്ഡ് ഇൻഫ്രാസ്ട്രക്ചറും ആപ്ലിക്കേഷൻ സ്ഥിരീകരിക്കാൻ തയ്യാറാണ്**.

## 4. ടെംപ്ലേറ്റ് നിലവാര പരിശോധന

1. Azure പോർട്ടൽ [Resource Groups](https://portal.azure.com/#browse/resourcegroups) പേജ് സന്ദർശിക്കുക - ആവശ്യമായെങ്കിൽ ലോഗിൻ ചെയ്യുക
1. നിങ്ങളുടെ പരിസരനാമം പങ്കിട്ട RG ക്ലിക്ക് ചെയ്യുക - മുകളിൽ കാണിച്ച പേജ് കാണും

      - Azure Container Apps റിസോഴ്സ് ക്ലിക്ക് ചെയ്യുക
      - _Essentials_ സെക്ഷനിലെ Application Url (മുകളിൽ വലതു) ക്ലിക് ചെയ്യുക

1. ഇത്തരം ഹോസ്റ്റുചെയ്‌ത ആപ്ലിക്കേഷൻ ഫ്രണ്ട്-എൻഡ് UI കാണണം:

   ![App](../../../../../translated_images/ml/03-test-application.471910da12c3038e.webp)

1. കുറച്ച് [സാമ്പിൾ ചോദ്യങ്ങൾ](https://github.com/Azure-Samples/get-started-with-ai-agents/blob/main/docs/sample_questions.md) ചോദിക്കാം:

      1. ചോദിക്കുക: ```What is the capital of France?``` 
      1. ചോദിക്കുക: ```What's the best tent under $200 for two people, and what features does it include?```

1. താഴെ കാണുന്നതുപോലെയുള്ള ഉത്തരങ്ങൾ ലഭിക്കണം. _ഇത് എങ്ങനെ പ്രവർത്തിക്കുന്നു?_

      ![App](../../../../../translated_images/ml/03-test-question.521c1e863cbaddb6.webp)

## 5. ഏജന്റ് സ്ഥിരീകരണം

Azure Container App ഒരു എൻഡ്‌പോയിന്റ് ഡിപ്ലോയ് ചെയ്യുന്നു, ഇത് ഈ ടെംപ്ലേറ്റിനുള്ള Microsoft Foundry പ്രോജക്റ്റിൽ പ്രൊവിഷൻ ചെയ്ത AI ഏജന്റുമായി കണക്ടുചെയ്യുന്നു. അതിന്റെ അർത്ഥം നോക്കാം.

1. നിങ്ങളുടെ റിസോഴ്സ് ഗ്രൂപ്പിന്റെ Azure പോർട്ടലിലെ _Overview_ പേജിലേക്ക് തിരികെ പോവുക

1. ആ ലിസ്റ്റിലുള്ള `Microsoft Foundry` റിസോഴ്സ് ക്ലിക്ക് ചെയ്യുക

1. ഇത് കാണാം. `Go to Microsoft Foundry Portal` ബട്ടൺ ക്ലിക്ക് ചെയ്യുക.
   ![Foundry](../../../../../translated_images/ml/04-view-foundry-project.fb94ca41803f28f3.webp)

1. നിങ്ങളുടെ AI ആപ്ലിക്കേഷനുള്ള Foundry പ്രോജക്റ്റ് പേജ് കാണാം
   ![Project](../../../../../translated_images/ml/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` ക്ലിക്ക് ചെയ്യുക - പ്രോജക്റ്റിൽ പ്രൊവിഷൻ ചെയ്ത ഡിഫോൾട്ട് ഏജന്റ് കാണാം
   ![Agents](../../../../../translated_images/ml/06-visit-agents.bccb263f77b00a09.webp)

1. അതിനെ തിരഞ്ഞെടുക്കുക - ഏജന്റ് വിശദാംശങ്ങൾ കാണാം. ശ്രദ്ധിക്കുക:

      - ഏജന്റ് ഡിഫോൾട്ടായി ഫയൽ സർച്ച് ഉപയോഗിക്കുന്നു (എപ്പോഴും)
      - ഏജന്റിന്റെ `Knowledge` 32 ഫയലുകൾ അപ്‌ലോഡ് ചെയ്തതായി കാണിക്കുന്നു (ഫയൽ സർച്ചിനായി)

## 7. ഇൻബിൽറ്റ് ഒബ്സർവബിലിറ്റി

ഒബ്സർവബിലിറ്റി നിങ്ങളുടെ ആപ്ലിക്കേഷനിൽ ഡാറ്റ ജനറേറ്റ് ചെയ്‌തു അതിന്റെ പ്രവർത്തനം മനസിലാക്കാനും, ദോഷനിവാരണത്തിനും, മെച്ചപ്പെടുത്തലിനും ഉപകരിക്കുന്നതും ആണ്. ഇതു മനസ്സിലാക്കാൻ:

1. `View Run Info` ബട്ടൺ ക്ലിക്ക് ചെയ്യുക - ഇങ്ങനെ ഒരു കാഴ്ച നോക്കാം. ഇത് [Agent tracing](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground)യുടെ ഉദാഹരണമാണ്. _ഈ കാഴ്ച ലഭിക്കുന്നതിന് Thread Logs എന്നിവ ടോപ്പ് ലെവൽ മെനുവിൽ ക്ലിക്ക് ചെയ്യുന്നതിലൂടെ സാധിക്കും_.

   - ഏജന്റ് ഉപയോഗിച്ച പ്രവർത്തനചുവടുകൾ കാണുക
   - മൊത്തം ടോക്കൺ എണ്ണം (ഔട്ട്പുട്ട് ടോക്കൺ ഉപയോഗത്തോട് താരതമ്യം) മനസ്സിലാക്കുക
   - എവിടെ എത്രവേലിൻ സമയമാണ് ചെലവായത് അത് മനസ്സിലാക്കുക

      ![Agent](../../../../../translated_images/ml/10-view-run-info.b20ebd75fef6a1cc.webp)

1. `Metadata` ടാബ് ക്ലിക്ക് ചെയ്ത് റൺ ഓപ്പറേഷന്റെ കൂടുതൽ പ്രത്യേകതകൾ കാണുക, ഇത് പ്രശ്നപരിഹാരത്തിനായി കൈകാര്യം ചെയ്യുന്ന സഹായകമായ ആശയക്കുഴപ്പം നൽകും.   

      ![Agent](../../../../../translated_images/ml/11-view-run-info-metadata.7966986122c7c2df.webp)


1. `Evaluations` ടാബ് ക്ലിക്ക് ചെയ്ത് ഏജന്റ് മറുപടിയിൽ സ്വയം വിലയിരുത്തലുകൾ (ഉദാ: സ്വയം ഹാനി) ഉൾപ്പെടെ കാണുക. അതുപോലെയുള്ള ഏജന്റ്-സവിശേഷ മൂല്യനിർണയങ്ങളും (ഉദാ: ഉദ്ദേശം നിശ്ചയം, ടാസ്‌ക് പാലന) കാണാം.

      ![Agent](../../../../../translated_images/ml/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. ഒടുവിൽ, സൈഡ്ബാർ മെനുവിൽ `Monitoring` ടാബ് ക്ലിക്ക് ചെയ്യുക.

## 8. പരിസരവരി മാറ്റികൾ

ഇത് വരെ, ബ്രൗസർ വഴി ഡിപ്ലോയ്മെന്റ് നടത്തിയിട്ടുണ്ട് - ഇൻഫ്രാസ്ട്രക്ചർ പ്രൊവിഷൻ ചെയ്തിട്ടുണ്ട്, ആപ്ലിക്കേഷൻ പ്രവർത്തനक्षमമാണ് എന്നും സ്ഥിരീകരിച്ചു. എന്നാൽ ആപ്ലിക്കേഷൻ _കോഡ്-പ്രഥമമായി_ ഉപയോഗിക്കാൻ, ഈ റിസോഴ്സുകളുമായി പ്രവർത്തിക്കുന്നതിന് ആവശ്യമായ ബന്ധപ്പെട്ട പരിസരവരി മാറ്റികൾ പ്രാദേശിക ഡെവലപ്‌മെന്റ് പരിസരത്തിൽ കോൺഫിഗർ ചെയ്യേണ്ടി വരും. `azd` ഉപയോഗിച്ച് ഇത് എളുപ്പമാണ്.

1. Azure Developer CLI [പരിസരവരി മാറ്റികൾ](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) ഉപയോഗിച്ച് ആപ്ലിക്കേഷൻ ഡിപ്ലോയ്മെന്റിന് ആവശ്യമായ കോൺഫിഗറേഷൻ സജ്ജീകരണങ്ങൾ സൂക്ഷിക്കുകയും നിയന്ത്രിക്കുകയും ചെയ്യുന്നു.

1. പരിസരവരി മാറ്റികൾ `.azure/<env-name>/.env` ൽ സൂക്ഷിക്കുന്നു - ഇത് ഡിപ്ലോയ്മെന്റ് സമയത്ത് ഉപയോഗിച്ചിരിക്കുന്ന പരിസരനാമം-സ്പെസിഫിക് ആണ്, അതുവഴി ഒരേ റിപ്പോസിറ്ററിയിലെ വ്യത്യസ്ത ഡിപ്ലോയ്മെന്റ് ലക്ഷ്യങ്ങൾക്കിടയിലെ പരിസരങ്ങളെ വേർതിരിക്കാമായിരിക്കും.

1. പരിസരവരി മാറ്റികൾ `azd` കമാൻഡ് ഏത് പ്രത്യേക കമാൻഡും (ഉദാ., `azd up`) പ്രവർത്തിപ്പിക്കുന്നത് സമയത്ത് സ്വയം ലോഡുചെയ്യുന്നു. Linux ഷെൽ പോലുള്ള ഒഎസ് ലെവൽ പരിസരവരി മാറ്റികൾ സ്വയം വായിക്കാറില്ല - അതിനു പകരം സ്ക്രിപ്റ്റുകളിലെ വിവരങ്ങളിലേക്കു കൈമാറ്റം നടത്ത `azd set env` മറ്റ് `azd get env` ഉപയോഗിക്കുക.

കേടാക്കാതെ കുറച്ച് കമാൻഡുകൾ പരീക്ഷിക്കാം:

1. ഈ പരിസരത്തിനായി `azd` രൂപീകരിച്ചിട്ടുള്ള എല്ലാ പരിസരവരി മാറ്റികളും നേടുക:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      ഇങ്ങനെ ഒന്നുപോലെ സന്ദേശം കാണാം:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. ഒരു നിശ്ചിത മൂല്യം നേടുക - ഉദാ: ഞാൻ `AZURE_AI_AGENT_MODEL_NAME` മൂല്യം സജ്ജമാണോ എന്ന് അറിയണം

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      ഇങ്ങനെ കാണാം - അത് ഡിഫോൾട്ടായി സജ്ജമാക്കിയിരുന്നില്ല!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd` നായി പുതിയ ഒരു പരിസരവരി മാറ്റി സജ്ജീകരിക്കുക. ഇവിടെ ഏജന്റ് മോഡൽ നാമം അപ്ഡേറ്റ് ചെയ്യുന്നു. _മറുപടി: നിലവിലുള്ള മാറ്റങ്ങൾ `.azure/<env-name>/.env` ഫയലിൽ ഉടനെ പ്രതിഫലിക്കും._

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      ഇപ്പോൾ മൂല്യം സജ്ജമാണെന്ന് കണ്ടുപിടിക്കുക:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. ചില റിസോഴ്സുകൾ സ്ഥിരതയുള്ളവയാണ് (ഉദാ: മോഡൽ ഡിപ്ലോയ്മെന്റുകൾ) അതിനാൽ വെറും `azd up` കൊണ്ട് പുന:ഡിപ്പ്ലോയ്മെന്റ് സാധ്യമല്ല. അതിനാൽ ആദ്യം ഉള്ള ഡിപ്ലോയ്മെന്റ് ഡിസ്‌മിസ്സുചെയ്‌തു എത്രയും വേഗം പുന:ഡിപ്ലോയ് ചെയ്യാം.

1. **പുതുക്കുക** മുമ്പ് azd ടെംപ്ലേറ്റ് ഉപയോഗിച്ച് ഡിപ്ലോയ്മെന്റ് ചെയ്തിട്ടുണ്ടെങ്കിൽ - നിങ്ങളുടെ പ്രാദേശിക പരിസരവരി മാറ്റികളുടെ നില ഇപ്പോഴത്തെ Azure ഡിപ്ലോയ്മെന്റിന്റെ അവസ്ഥ പ്രകാരം പുതുക്കാൻ ഈ കമാൻഡ് ഉപയോഗിക്കുക:

      ```bash title="" linenums="0"
      azd env refresh
      ```

      ഇത് രണ്ട് അല്ലെങ്കിൽ അതിലധികം ലൊക്കൽ ഡെവലപ്പ്മെന്റ് പരിസ്ഥിതികളിൽ (ഉദാഹരണത്തിന്, നിരവധി ഡെവലപ്പർമാർ ഉള്ള ടീം) പരിസ്ഥിതി ചേരുവകൾ _സിങ്ക്_ ചെയ്യുന്നതിനുള്ള ശക്തമായ ഒരു മാർഗമാണ് - ഡിപ്ലോയുചെയ്ത ഐൻഫ്രാസ്ട്രക്ചർ പരിസ്ഥിതി ചേരുവകളുടെ അവസ്ഥയ്ക്ക് അടിസ്ഥാനമായ സത്യം ആയി പ്രവർത്തിക്കുന്നു. സംഘ അംഗങ്ങൾ ലളിതമായി ചേരുവകൾ _റിഫ്രഷ്_ ചെയ്തു സിങ്ക് ആവാന് കഴിയുന്നു.
