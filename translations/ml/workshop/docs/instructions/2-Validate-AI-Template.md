## 1. പരിചയം

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) അല്ലെങ്കിൽ `azd` ഒരു ഓപ്പൺ-സോഴ്‌സ് കമാൻഡ് ലൈനുള്ള ടൂൾ ആണ്, ഇത് Azure-മേറ്റുള്ള ആപ്ലിക്കേഷനുകൾ നിർമിക്കാനും വിന്യസിക്കാനും ഡെവലപ്പർ പ്രവൃത്തി സൌകര്യമാർഗ്ഗം സുതാര്യമാക്കുന്നു.

[AZD ടെംപ്ലേറ്റുകൾ](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) സാംപിള്‍ ആപ്ലിക്കേഷൻ കോഡ്, _ഇൻഫ്രാസ്ട്രക്ചർ-ആസ്-കോഡ്_ അസറ്റുകളും `azd` കോൺഫിഗറേഷൻ ഫയലുകളും ഉൾക്കൊള്ളുന്ന സ്റ്റാൻഡർഡൈസ്ഡ് റീപ്പോസിറ്ററികളാണ്, ഒരു ഏകീകൃത പരിഹാര ശില്പശാലയ്ക്ക് അനുയോജ്യം. ഇൻഫ്രാസ്ട്രക്ചർ പ്രൊവിഷനിംഗ് എല്ലാം കുറിച്ച് `azd provision` കമാൻഡ് സമയം എളുപ്പമാണ് - കൂടാതെ `azd up` ഉപയോഗിച്ച് നിങ്ങൾക്ക് ഒരേസമയം ഇൻഫ്രാസ്ട്രക്ചർ പ്രൊവിഷൻ ചെയ്യാനും ആപ്ലിക്കേഷൻ വിന്യസിക്കാനും കഴിയും!

ഫലമായി, നിങ്ങളുടെ ആപ്ലിക്കേഷൻ വികസന പ്രക്രിയ തുടങ്ങി എളുപ്പമുള്ളതാകുന്നത് ഏറ്റവും അനുയോജ്യമായ _AZD സ്റ്റാർട്ടർ ടെംപ്ലേറ്റ്_ കണ്ടെത്തുക തന്നെയാകാം, ഇത് നിങ്ങളുടെ ആപ്ലിക്കേഷനും ഇൻഫ്രാസ്ട്രക്ചർ ആവശ്യകതകൾക്കുമടുത്തു കൂട്ടിച്ചേർക്കാൻ റീപ്പോസിറ്ററി കസ്റ്റമൈസ് ചെയ്യുക.

തുടങ്ങാൻ മുമ്പ്, Azure Developer CLI ഇൻസ്റ്റാൾ ചെയ്തിട്ടുണ്ടെന്ന് ഉറപ്പാക്കാം.

1. VS കോഡ് ടർമിനൽ തുറന്ന് ഈ കമാൻഡ് ടൈപ്പ് ചെയ്യുക:

      ```bash title="" linenums="0"
      azd version
      ```

1. നിങ്ങൾക്ക് ഇത്തരമൊരു പ്രദർശനം കാണാം!

      ```bash title="" linenums="0"
      azd version 1.19.0 (commit b3d68cea969b2bfbaa7b7fa289424428edb93e97)
      ```

**ഇപ്പോൾ നിങ്ങൾ അജ്ഡിനെ ഉപയോഗിച്ച് ടെംപ്ലേറ്റ് തിരഞ്ഞെടുക്കാനും വിന്യാസപ്പെടുകയും ചെയ്യാൻ തയ്യാറാണ്**

## 2. ടെംപ്ലേറ്റ് തിരഞ്ഞെടുപ്പ്

Microsoft Foundry പ്ലാറ്റ്ഫോം ഒരു [ഏഴി ശുപാർശ ചെയ്യപ്പെട്ട AZD ടെംപ്ലേറ്റുകളുടെ സെറ്റ്](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) കൊണ്ടാണ് വരുന്നത്, ഈ ടെംപ്ലേറ്റുകൾ ജനപ്രിയ പരിഹാര სცീനാറികളായ _മൾട്ടി- ഏജൻറ് വർക്ക്‌ഫ്ലോ ആറ്റൊമേഷൻ_ , _മൾട്ടി-മോഡൽ ഉള്ളടക്ക പ്രോസസ്സിംഗ്_ എന്നിവ ഉൾക്കൊള്ളുന്നു. അഡീഷണലി, നിങ്ങൾക്ക് ഈ ടെംപ്ലേറ്റുകൾ മൈക്രോസോഫ്റ്റ് ഫൗണ്ടറി പോർട്ടൽ സന്ദർശിച്ച് കണ്ടെത്താം.

1. സന്ദർശിക്കുക [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. പ്രൊഫൈൽ-ഓതെന്റിക്കേഷൻ ആവശ്യപ്പെട്ടാൽ Microsoft Foundry പോർട്ടലിൽ ലോഗിൻ ചെയ്യുക - നിങ്ങൾക്ക് ഇത്തരമൊരു സ്ക്രീൻ കാണാം.

![Pick](../../../../../translated_images/ml/01-pick-template.60d2d5fff5ebc374.webp)


**ബേസിക്** ഓപ്ഷനുകൾ നിങ്ങളുടെ സ്റ്റാർട്ടർ ടെംപ്ലേറ്റുകളാണ്:

1. [ ] [AI ചാറ്റ് തുടങ്ങുക](https://github.com/Azure-Samples/get-started-with-ai-chat) - ഒരു അടിസ്ഥാന ചാറ്റ് ആപ്ലിക്കേഷൻ നിങ്ങളുടെ ഡാറ്റ ഉപയോഗിച്ച് Azure Container Apps-ൽ വിന്യസിക്കുന്നു. ഇതുപയോഗിച്ച് ഒരു അടിസ്ഥാന AI ചാറ്റ്ബോട്ട് സ്‌ക്കീനാർ പരിശോധിക്കാം.
1. [X] [AI ഏജന്റുകൾ ആരംഭിക്കുക](https://github.com/Azure-Samples/get-started-with-ai-agents) - സ്റ്റാൻഡേർഡ് AI ഏജന്റ് (Foundry ഏജന്റുകൾ ഉൾക്കൊള്ളുന്നു) വിന്യസിക്കുന്നു. ടൂൾസ്, മോഡലുകൾ എന്നിവ ഉൾപ്പെട്ട ഏജന്റിക് AI പരിഹാരങ്ങൾ അറിയാനായി ഇതുപയോഗിക്കാം.

രണ്ടാമത്തെ ലിങ്ക് പുതിയ ബ്രൗസർ ടാബിൽ തുറക്കുക (അല്ലെങ്കിൽ ബന്ധപ്പെട്ട കാർഡിൽ `Open in GitHub` ക്ലിക്ക് ചെയ്യുക). AZD ടെംപ്ലേറ്റിനുള്ള റീപ്പോസിറ്ററി നിങ്ങൾ കാണും. README പരീക്ഷിക്കാൻ ഒരു മിനിറ്റ് എടുത്തു നോക്കൂ. ആപ്ലിക്കേഷൻ ശില്പശാല ഇങ്ങനെ കാണാം:

## 3. ടേംപ്ലേറ്റ് ആക്ടിവേഷൻ

ഈ ടെംപ്ലേറ്റ് വിന്യസിക്കാനും ഇത് സാധുവാണെന്ന് ഉറപ്പാക്കാനും ശ്രമിച്ചോളാം. [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) വിഭാഗത്തിലെ മാർഗനിർദ്ദേശങ്ങൾ പാലിക്കാം.

1. [ഈ ലിങ്ക്](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) ക്ലിക്ക് ചെയ്യുക - ഡിഫോൾട്ട് അക്ഷൻ `Create codespace` സ്ഥിരീകരിക്കുക
1. പുതിയ ബ്രൗസർ ടാബ് തുറക്കുന്നു - GitHub Codespaces സെഷൻ പൂർണ്ണമായി ലോഡ് ആവുന്നത് കാത്തിരിക്കുക
1. Codespaces-ൽ VS കോഡ് ടർമിനൽ തുറക്കുക - താഴെ കൊടുത്ത കമാൻഡ് ടൈപ്പ് ചെയ്യുക:

   ```bash title="" linenums="0"
   azd up
   ```

ഈ പ്രവൃത്തി ഘട്ടങ്ങൾ പൂർത്തിയാക്കൂ:

1. Azure-ലോഗിൻ ആവശ്യപ്പെടാൻ നിർദ്ദേശിക്കുന്നു - ഓത്തന്റിക്കേഷൻ നിർദ്ദേശങ്ങൾ പാലിക്കുക
1. നിങ്ങളുടെ தனിവിധമായ എൻവയോൺമെന്റ് പേര് നൽകുക - ഉദാഹരണത്തിന്, ഞാൻ `nitya-mshack-azd` ഉപയോഗിച്ചു
1. ഇതിൽ `.azure/` ഫോൾഡർ ഉണ്ടാകും - എൻവൈറൺമെന്റ് പേരുള്ള ഒരു സബ്ഫോൾഡർ നിങ്ങൾക്ക് കാണാവുന്നതായി
1. സബ്സ്ക്രിപ്ഷൻ പേര് തിരഞ്ഞെടുക്കാൻ നിർദ്ദേശിക്കപ്പെടും - ഡിഫോൾട്ട് തിരഞ്ഞെടുക്കുക
1. ഒരു സ്ഥലം തിരഞ്ഞെടുക്കാൻ ചോദിക്കും - `East US 2` ഉപയോഗിക്കുക

ഇപ്പോൾ, പ്രൊവിഷനിംഗ് പൂർത്തിയാകുന്നതിന് കാത്തിരിക്കുക. **ഇതിന് 10-15 മിനിറ്റ് വേണ്ടി വരും**

1. പൂർത്തിയായപ്പോൾ, നിങ്ങളുടെ കോൺസോൾ SUCCESS സന്ദേശം കാണിക്കും:

      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. നിങ്ങളുടെ Azure പോർട്ടലിൽ ആ എൻവയോൺമെന്റ് പേരുള്ള ഒരു പ്രൊവിഷൻ ചെയ്ത റിസോഴ്സ് ഗ്രൂപ്പുണ്ടാകും:

      ![Infra](../../../../../translated_images/ml/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **ഇപ്പോൾ നിങ്ങൾ വിന്യസിച്ച ഇൻഫ്രാസ്ട്രക്ചർ-ആൻഡ് ആപ്ലിക്കേഷൻ പരിശോധിക്കാൻ തയ്യാറാണ്**.

## 4. ടെംപ്ലേറ്റ് പരിശോധന

1. Azure പോർട്ടൽ [Resource Groups](https://portal.azure.com/#browse/resourcegroups) പേജ് സന്ദർശിക്കുക - ലോഗിൻ ആവശ്യമുണ്ടെങ്കിൽ ലോഗിൻ ചെയ്യുക
1. നിങ്ങളുടെ എൻവൈറൺമെന്റ് പേരുള്ള RG ക്ലിക്ക് ചെയ്യുക - മേലിൽ കാണിച്ച പേജ് കാണാം

      - Azure Container Apps റിസോഴ്സ് ക്ലിക്ക് ചെയ്യുക
      - _Essentials_ വിഭാഗത്തിൽ ഉള്ള Application Url ക്ലിക്ക് ചെയ്യുക (മേലൊരു വലമായി)

1. ഇങ്ങനെ ഒരു ഹോസ്റ്റ് ചെയ്ത ആപ്ലിക്കേഷൻ ഫ്രണ്ട്-എൻഡ് UI കാണാം:

   ![App](../../../../../translated_images/ml/03-test-application.471910da12c3038e.webp)

1. ഒരു കിടിലൻ [സാംപിള്‍ ചോദ്യങ്ങൾ](https://github.com/Azure-Samples/get-started-with-ai-agents/blob/main/docs/sample_questions.md) ചോദിക്കാം

      1. ചോദിക്കൂ: ```What is the capital of France?``` 
      1. ചോദിക്കൂ: ```What's the best tent under $200 for two people, and what features does it include?```

1. താഴെ കാണിച്ചിരിക്കുന്നതിനുപോലെയായി നിങ്ങൾ ഉത്തരങ്ങൾ ലഭിക്കും. _എന്തിനാണ് ഇത് പ്രവർത്തിക്കുന്നത്?_

## 5. ഏജന്റ് പരിശോധന

Azure Container App ഈ ടെംപ്ലേറ്റിനായി Microsoft Foundry പ്രോജക്ടിൽ പ്രൊവിഷൻ ചെയ്ത AI ഏജന്റുമായി കണക്ട് ചെയ്യുന്ന ഒരു എൻഡ്‌പോയിന്റ് വിന്യസിക്കുന്നു. ഇത് എന്താണെന്ന് നോക്കാം.

1. നിങ്ങളുടെ റിസോഴ്‌സ് ഗ്രൂപ്പിന്റെ Azure പോർട്ടൽ _Overview_ പേജിലേക്ക് മടങ്ങുക

1. ലിസ്റ്റിലുള്ള `Microsoft Foundry` റിസോഴ്സ് ക്ലിക്ക് ചെയ്യുക

1. ദൃശ്യം ഇങ്ങനെ കാണും. `Go to Microsoft Foundry Portal` ബട്ടൺ ക്ലിക്ക് ചെയ്യുക.

   ![Foundry](../../../../../translated_images/ml/04-view-foundry-project.fb94ca41803f28f3.webp)

1. നിങ്ങളുടെ AI ആപ്ലിക്കേഷൻക്ക് അനുയോജ്യമായ Foundry പ്രോജക്ട് പേജ് കാണാം
   ![Project](../../../../../translated_images/ml/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` ക്ലിക്ക് ചെയ്യുക - നിങ്ങളുടെ പ്രോജക്ടിൽ പ്രൊവിഷൻ ചെയ്ത ഡിഫോൾട്ട് ഏജന്റ് കാണും

## 8. എൻവയോൺമെന്റ് വേരിയബിളുകൾ

ഇതിന് മുമ്പ്, നാം ബ്രൗസറിൽ വിന്യാസത്തിൽ നിന്നും നടത്തിയപ്പോള്‍ - ഇൻഫ്രാസ്ട്രക്ചർ പ്രൊവിഷൻ ചെയ്തതും ആപ്ലിക്കേഷൻ ഓപ്പറേഷണലായതും സ്ഥിരീകരിച്ചു. എന്നാൽ ആപ്ലിക്കേഷൻ _കോഡ്-ഫസ്റ്റ്_ രീതിയിലേക്ക് പ്രവർത്തിപ്പിക്കാൻ, ഈ റിസോഴ്സുകൾ ഉപയോഗിക്കാൻ ആവശ്യമായ പ്രാദേശിക ഡെവലപ്പ്മെന്റ് എൻവയോൺമെന്റ് വേരിയബിളുകൾ കൺഫിഗർ ചെയ്യേണ്ടതാണ്. `azd` ഉപയോഗിച്ച് ഇത് എളുപ്പമാണ്.

1. Azure Developer CLI ആപ്ലിക്കേഷൻ വിന്യാസത്തിനായി കോൺഫിഗറേഷൻ സജ്ജീകരണങ്ങൾ സംഭരിക്കുന്നതിനും നിയന്ത്രിക്കുന്നതിനും [എൻവയോൺമെന്റ് വേരിയബിളുകൾ](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) ഉപയോഗിക്കുന്നു.

1. എൻവയോൺമെന്റ് വേരിയബിളുകൾ `.azure/<env-name>/.env` ഫയലിൽ സൂക്ഷിക്കുന്നു - ഇത് `env-name` എൻവയോൺമെന്റ് ഉപയോഗിച്ച് വിന്യാസം നടത്തിയതിനോടനുബന്ധിച്ച് അവയെ സ്കോപ്പ് ചെയ്യുകയും, ഒരേ റീപ്പോയിൽ വ്യത്യസ്ത വിന്യാസ ലക്ഷ്യങ്ങൾക്കിടയിലെ എൻവയോൺമെന്റുകൾ വേർതിരിക്കാൻ സഹായിക്കുന്നു.

1. `azd` കമാൻഡ് ഏതെങ്കിലും കമാൻഡ് പ്രവർത്തിപ്പുമ്പോൾ (ഉദാ: `azd up`) ഈ വേരിയബിളുകൾ സ്വയമേ ലോഡ് ചെയ്യപ്പെടുന്നു. `azd` ഓപ്പറേറ്റിംഗ് സിസ്റ്റം-തല എൻവയോൺമെന്റ് വേരിയബിളുകൾ (ഉദാ: ഷെല്ലിൽ സെറ്റ് ചെയ്തവ) സ്വയം വായിക്കാറില്ല - സ്ട്രിപ്റ്റിനുള്ളിൽ വിവരങ്ങൾ കൈമാറാൻ `azd set env`  , `azd get env`  ഉപയോഗിക്കാം.

നിന്നെ ചില കമാൻഡുകൾ പരീക്ഷിക്കാം:

1. ഈ എൻവയോൺമെൻറിൽ `azd`ക്ക് സജ്ജീകരിച്ചിരിക്കുന്ന എല്ലാ എൻവയോൺമെന്റ് വേരിയബിളുകളും കണ്ടെത്തുക:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      നിങ്ങൾ ഇങ്ങനെ കാണും:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4o-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. പ്രത്യേക ഒരു മൂല്യം കണ്ടെത്തുക - ഉദാഹരണം: `AZURE_AI_AGENT_MODEL_NAME` മൂല്യം നാം സജ്ജീകരിച്ചിട്ടുണ്ടോ എന്ന് അറിയാൻ

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      ഇങ്ങനെ ചിലപ്പോഴായി കാണാം - ഡിഫോൾട്ടായി സജ്ജീകരിച്ചിട്ടില്ല!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd`യ്ക്ക് പുതിയ എൻവയോൺമെന്റ് വേരിയബിൾ സജ്ജീകരിക്കുക. ഇവിടെ, ഏജന്റ് മോഡൽ പേര് പുതുക്കുന്നു. _ഗൗരവമായി : മാറ്റങ്ങൾ ഉടൻ `.azure/<env-name>/.env` ഫയലിൽ പ്രതിഫലിക്കും._

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      ഇപ്പോൾ, മൂല്യം സജ്ജീകരിച്ചിരിക്കുന്നതായി കാണാം:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. ചില റിസോഴ്‌സുകൾ സ്ഥിരമേയും (ഉദാ: മോഡൽ വിന്യാസങ്ങൾ) ഉളളവയാണ്. അവ വീണ്ടും വിന്യസിക്കാൻ കൂടുതൽ പ്രവർത്തനങ്ങൾ ആവാം `azd up` മാത്രം പോരാതിരിക്കും. ഓരൊരിക്കൽ ഒറിജിനൽ വിന്യാസം നീക്കം ചെയ്ത് പുതിയ എൻവയോൺമെന്റ് വേരിയബിളുകളുമായി വീണ്ടും വിന്യസിക്കാം.

1. **രിഫ്രഷ്** മുൻപ് azd ടെംപ്ലേറ്റ് ഉപയോഗിച്ച് എൻഫ്രാസ്ട്രക്ചർ വിന്യസിച്ചിട്ടുണ്ടെങ്കിൽ - നിങ്ങളുടെ ലോക്കൽ എൻവയോൺമെന്റ് വേരിയബിളുകളുടെ നില നിലവിലെ Azure വിന്യാസത്തിന്റെ അടിസ്ഥാനത്തിൽ _രിഫ്രഷ്_ ചെയ്യാം ഈ കമാൻഡ് ഉപയോഗിച്ച്:

      ```bash title="" linenums="0"
      azd env refresh
      ```

      ഇത് രണ്ട് അല്ലെങ്കിൽ കൂടുതൽ ലൊക്കൽ ഡെവലപ്പ്മെന്റ് പരിസ്ഥിതികളിൽ (ഉദാ., നിരവധി ഡെവലപ്പർമാരുള്ള ടീം) പരിസ്ഥിതി ചരങ്ങളായുള്ള _sync_ ചെയ്യാനുള്ള ശക്തമായ മാർഗമാണ് - വിന്യസിച്ച ഇൻഫ്രാസ്ട്രക്ചർ പരിസ്ഥിതി ചരങ്ങളുടെ അവസ്ഥയുടെ അടിസ്ഥാന സത്യം ആയിരിക്കും. ടീം അംഗങ്ങൾക്ക് സ്വതന്ത്രമായി ചരങ്ങൾ _refresh_ ചെയ്ത് വേറിട്ടുവരാം.
