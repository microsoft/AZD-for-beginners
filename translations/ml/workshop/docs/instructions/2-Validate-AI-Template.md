## 1. പരിചയം

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) അല്ലെങ്കിൽ `azd` ആജ=open-source കമാൻഡ് ലൈൻ ടൂളാണ്, അത് ആസ്യൂറെ ആപ്ലിക്കേഷനുകൾ നിർമ്മിക്കുകയും ഡിപ്ലോയ് ചെയ്യുകയും ചെയ്യുന്ന ഡെവലപ്പർ പ്രവൃത്തി പ്രവാഹം ലളിതമാക്കുന്നു.

[AZD ടംപ്ലേറ്റുകൾ](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) ഒത്തിണക്കി കോഡ്, _infrastructure-as-code_ ആസ്തികൾ, കൂടാതെ `azd` കോൺഫിഗറേഷൻ ഫയലുകൾ എന്നിവയടങ്ങിയ ഏകീകൃത സൊല്യൂഷൻ ആർക്കിടെക്ചർ ഉള്ള സ്റ്റാൻഡേർഡൈസ്ഡ് റീപ്പോസിറ്ററികളാണ്. ഇൻഫ്രാസ്ട്രക്ചർ പ്രൊവിഷനിംഗ് ഒരേ `azd provision` കമാൻഡ് പോലെ ലളിതമാണ് - അതേസമയം `azd up` ഉപയോഗിച്ച് ഇൻഫ്രാസ്ട്രക്ചർ പ്രൊവിഷനിംഗും ആപ്ലിക്കേഷൻ ഡിപ്ലോയ്മെന്റും ഒരിക്കൽ നടത്താം!

ഫലമായി, നിങ്ങളുടെ ആപ്ലിക്കേഷൻ വികസന പ്രക്രിയ തുടക്കം കുറിക്കാൻ ഏറ്റവും അനുയോജ്യമായ _AZD Starter template_ കണ്ടെത്തി നിങ്ങളുടെ ആപ്ലിക്കേഷനും ഇൻഫ്രാസ്ട്രക്ചർ ആവശ്യങ്ങൾക്കനുസരിച്ച് റെപ്പോസിറ്ററി ക്രമീകരിക്കുക ഇതു പോലെ ലളിതമാണ്.

ആരംഭിക്കാൻ മുൻപ്, നിങ്ങൾക്ക് Azure Developer CLI ഇൻസ്റ്റാൾ ചെയ്‌തുണ്ടെന്ന് ഉറപ്പാക്കാം.

1. ഒരു VS കോഡ് ടെർമിനൽ തുറന്ന് ഈ കമാൻഡ് ടൈപ്പുചെയ്യുക:

      ```bash title="" linenums="0"
      azd version
      ```

1. നിങ്ങൾക്ക് ഇങ്ങനെ ഒന്നിനെ കാണണം!

      ```bash title="" linenums="0"
      azd version 1.23.12 (commit <current-build>)
      ```

**ഇപ്പോൾ azd ഉപയോഗിച്ച് ഒരു ടംപ്ലേറ്റ് തിരഞ്ഞെടുക്കാനും ഡിപ്ലോയ്ചെയ്യാനും തയ്യാറാണ്**

## 2. ടംപ്ലേറ്റ് തിരഞ്ഞെടുക്കൽ

Microsoft Foundry പ്ലാറ്റ്ഫോം [ശുപാർശ ചെയ്യുന്ന AZD ടംപ്ലേറ്റുകളുടെ ഒരു സെറ്റ്](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) ഉണ്ട്, അവ മൾട്ടി-ഏജന്റ് വർക്‌ഫ്ലോ ഓട്ടോമേഷൻ, മൾട്ടി-മോഡൽ ഉള്ളടക്കം പ്രോസസ്സിംഗ് തുടങ്ങിയ ജനപ്രിയ സൊല്യൂഷൻ സീനാരിയോകളെ കवर്ചെയ്യുന്നു. നിങ്ങൾക്ക് ഈ ടംപ്ലേറ്റുകളും Microsoft Foundry പോർട്ടൽ സന്ദർശിച്ച് കണ്ടെത്താം.

1. [https://ai.azure.com/templates](https://ai.azure.com/templates) സന്ദർശിക്കുക
1. ആവശ്യമായപ്പോൾ Microsoft Foundry പോർട്ടൽ ലോഗിൻ ചെയ്യുക - നിങ്ങൾക്ക് ഇങ്ങനെ ഒന്നിനെ കാണാം.

![Pick](../../../../../translated_images/ml/01-pick-template.60d2d5fff5ebc374.webp)


**Basic** ഓപ്ഷനുകൾ നിങ്ങളുടെ സ്റ്റാർട്ടർ ടംപ്ലേറ്റുകളാണ്:

1. [ ] [AI ചാറ്റ് ഉപയോഗിച്ച് തുടക്കം കുറിക്കുക](https://github.com/Azure-Samples/get-started-with-ai-chat), ഇത് നിങ്ങളുടെ ഡാറ്റയോടെ ഒരു അടിസ്ഥാന ചാറ്റ് ആപ്ലിക്കേഷൻ Azure Container Apps-ലേക്ക് ഡിപ്ലോയ്ചെയ്യുന്നു. അടിസ്ഥാന AI ചാറ്റ്ബോട്ട് സീനാരി പരീക്ഷിക്കാൻ ഇത് ഉപയോഗിക്കുക.
1. [X] [AI ഏജന്റ്സുമായി തുടക്കം കുറിക്കുക](https://github.com/Azure-Samples/get-started-with-ai-agents), ഇത് Foundry ഏജന്റുമാരോടുകൂടിയ സ്റ്റാൻഡേർഡായ AI ഏജന്റ് ഡിപ്ലോയ്ചെയ്യുന്നു. ടൂളുകളും മോഡലുകളുമുള്ള ഏജന്റിക് AI സൊല്യൂഷനുകളുമായി പരിചിതരാകാൻ ഇത് ഉപയോഗിക്കുക.

രണ്ടാം ലിങ്ക് പുതിയ ബ്രൗസർ ടാബിൽ തുറക്കുക (അല്ലെങ്കിൽ ബന്ധപ്പെട്ട കാർഡിൽ 'Open in GitHub' ക്ലിക്കുചെയ്യുക). നിങ്ങൾക്ക് ഈ AZD ടംപ്ലേറ്റിന്റെ റെപ്പോസിറ്ററി കാണാം. README പരിശോധിക്കാൻ ഒരു മിനിട്ട് എടുത്തു നോക്കൂ. ആപ്ലിക്കേഷൻ ആർക്കിടെക്ചർ ഇങ്ങനെ കാണുന്നു:

## 3. ടംപ്ലേറ്റ് സജീവമാക്കൽ

ഈ ടംപ്ലേറ്റ് ഡിപ്ലോയ്ചെയ്യാം, സാധുത പരിശോധിക്കാം. [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) വിഭാഗത്തിലെ മാർഗനിർദ്ദേശങ്ങൾ പാലിക്കാം.

1. ടംപ്ലേറ്റ് റെപ്പോസിറ്ററിയ്ക്ക് ഒരു പ്രവർത്തന പരിസ്ഥിതി തിരഞ്ഞെടുക്കുക:

      - **GitHub Codespaces**: [ഈ ലിങ്ക്](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) ക്ലിക്കുചെയ്യുക, പിന്നെ `Create codespace` സ്ഥിരീകരിക്കുക
      - **ലോകൽ ക്ലോൺ അല്ലെങ്കിൽ ഡെവ്കണ്ടെയ്‌നർ**: `Azure-Samples/get-started-with-ai-agents` ക്ലോൺ ചെയ്ത് VS കോഡിൽ തുറക്കുക

1. VS കോഡ് ടർമിനൽ സജ്ജമാകുന്നതുവരെ കാത്തിരിക്കുക, പിന്നെ താഴെയുള്ള കമാൻഡ് ടൈപ്പുചെയ്യുക:

   ```bash title="" linenums="0"
   azd up
   ```

കുറ്റം ചെയ്യപ്പെടുന്ന പ്രവർത്തനക്രമപടികൾ പൂർത്തിയാക്കുക:

1. നിങ്ങൾക്ക് Azure ലോഗിൻ ചെയ്യാൻ അനുവാദം ചോദിക്കും - അതിന്റെ നിർദ്ദേശങ്ങൾ പിന്തുടരുക
1. നിങ്ങൾക്കായൊരു വ്യത്യസ്ത പരിസ്ഥിതി നാമം നൽകുക - ഉദാ., ഞാനിത് ഉപയോഗിച്ചു `nitya-mshack-azd`
1. ഇത് `.azure/` ഫോൾഡർ സൃഷ്ടിക്കും - അതിന്റെ കീഴിൽ പരിസ്ഥിതി നാമം അടങ്ങിയ സബ്ഫോൾഡർ കാണും
1. സബ്സ്ക്രിപ്ഷൻ നാമം തിരഞ്ഞെടുക്കാൻ പറയും - ഡിഫോൾട്ട് തിരഞ്ഞെടുക്കുക
1. ലൊക്കേഷൻ ചോദിക്കും - `East US 2` ഉപയോഗിക്കുക

ഇപ്പോൾ, പ്രൊവിഷനിംഗ് പൂർത്തിയാകുന്നത് കാത്തിരിക്കുക. **ഇത് 10-15 മിനിറ്റ് എടുക്കും**

1. പൂർത്തിയായപ്പോൾ നിങ്ങളുടെ കോൺസോളിൽ SUCCESS സന്ദേശം ഇങ്ങനെ കാണും:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. ഇപ്പോൾ നിങ്ങളുടെ Azure പോർട്ടലിൽ ആ പരിസ്ഥിതി നാമം ഉള്ള ഒരു പ്രൊവിഷനുചെയ്‌ത റിസോഴ്‌സ് ഗ്രൂപ്പ് ഉണ്ടാകും:

      ![Infra](../../../../../translated_images/ml/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **ഇപ്പോൾ നിങ്ങൾക്ക് ഡിപ്ലോയ്ചെയ്‌ത ഇൻഫ്രാസ്ട്രക്ചറും ആപ്ലിക്കേഷനും സാധുതാപരീശോധിക്കാനുള്ള ഒരുങ്ങിയിരിക്കുന്നു**.

## 4. ടംപ്ലേറ്റ് സാധുതാപരീശോധനം

1. Azure Portal [Resource Groups](https://portal.azure.com/#browse/resourcegroups) പേജ് സന്ദർശിക്കുക - ആവശ്യപ്പെട്ടാൽ ലോഗിൻ ചെയ്യുക
1. നിങ്ങളുടെ പരിസ്ഥിതി നാമമുള്ള RG ക്ലിക്കുചെയ്യേണ്ടതാണ് - മുകളിൽ തന്ന പേജ് കാണാം

      - Azure Container Apps റിസോഴ്‌സ് ക്ലിക്കുചെയ്യൂ
      - _Essentials_ വിഭാഗത്തിൽ (മുകളിൽ വലത്) Application Url ക്ലിക്കുചെയ്യൂ

1. നിങ്ങൾക്ക് ഇങ്ങനെ ഹോസ്റ്റ് ചെയ്ത ആപ്ലിക്കേഷൻ ഫ്രണ്ട്-എൻഡ് UI കാണാം:

   ![App](../../../../../translated_images/ml/03-test-application.471910da12c3038e.webp)

1. കുറച്ച് [സാമ്പിൾ ചോദ്യങ്ങൾ](https://github.com/Azure-Samples/get-started-with-ai-agents/blob/main/docs/sample_questions.md) ചോദിക്കാൻ ശ്രമിക്കുക

      1. ചോദിക്കുക: ```What is the capital of France?``` 
      1. ചോദ്യമാക്കുക: ```What's the best tent under $200 for two people, and what features does it include?```

1. താഴെ കാണുന്ന സമാനമായ ഉത്തരങ്ങൾ നിങ്ങൾക്ക് ലഭിക്കുക. _എങ്കിലും ഇത് എങ്ങനെ പ്രവർത്തിക്കുന്നു?_

## 5. ഏജന്റ് സാധുതാപരീശോധനം

Azure Container App ഈ ടംപ്ലേറ്റിനായി Microsoft Foundry പ്രോജക്റ്റിൽ പ്രൊവിഷൻ ചെയ്‌ത AI ഏജന്റുമായി ബന്ധിപ്പിച്ച ഒരു എണ്ട്പോയിന്റ് ഡിപ്ലോയ്ചെയ്യുന്നു. ഇതിന്റെഅർത്ഥം എന്താണെന്ന് നോക്കാം.

1. നിങ്ങളുടെ റിസോഴ്‌സ് ഗ്രൂപ്പിന്റെ Azure Portal _Overview_ പേജിലേക്ക് തിരികെ പോവുക

1. ആ ലിസ്റ്റിൽ `Microsoft Foundry` റിസോഴ്‌സ് ക്ലിക്കുചെയ്യുക

1. നിങ്ങൾക്ക് ഇത് കാണാം. `Go to Microsoft Foundry Portal` ബട്ടൺ ക്ലിക്കുചെയ്യുക.
   ![Foundry](../../../../../translated_images/ml/04-view-foundry-project.fb94ca41803f28f3.webp)

1. നിങ്ങളുടെ AI ആപ്ലിക്കേഷനായി Foundry Project പേജ് കാണും
   ![Project](../../../../../translated_images/ml/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` ക്ലിക്കുചെയ്യൂ - നിങ്ങളുടെ പ്രോജക്റ്റിൽ ഡിഫോൾട്ട് ഏജന്റ് പ്രൊവിഷൻ ചെയ്തുകാണാം
   ![Agents](../../../../../translated_images/ml/06-visit-agents.bccb263f77b00a09.webp)

1. അതിനെ തിരഞ്ഞെടുക്കുക - ഏജന്റ് വിശദാംശങ്ങൾ കാണാം. ശ്രദ്ധിക്കേണ്ട കാര്യങ്ങൾ:

      - ഏജന്റ് ഫയൽ സെർച് ഡിഫോൾട്ടായും (എപ്പോഴും) ഉപയോഗിക്കുന്നു
      - ഏജന്റ് `Knowledge` 32 ഫയലുകൾ അപ്‌ലോഡ് ചെയ്തതായി കാണിക്കുന്നു (ഫയൽ സർച്ച് വേണ്ടി)

## 8. പരിസ്ഥിതി വേരിയബിളുകൾ

ഇതുവരെയുള്ളത്രയോളം, നാം ബ്രൗസറിൽ ഡിപ്ലോയ്മെന്റ് വഴ ഘട്ടങ്ങൾ പൂർത്തിയാക്കി - ഇൻഫ്രാസ്ട്രക്ചർ പ്രൊവിഷൻ ചെയ്‌തിരിക്കുന്നതും ആപ്ലിക്കേഷൻ പ്രവർത്തനക്ഷമമായിട്ടുള്ളതും സാധുതാപരീശോധിച്ചു. പക്ഷേ ആപ്ലിക്കേഷൻ _കോഡ്-ഫസ്റ്റ്_ ആയി ഉപയോഗിക്കാൻ, ഈ റിസോഴ്‌സുകൾ ഉപയോഗിക്കാൻ ആവശ്യമായ ബന്ധപ്പെട്ട വേരിയബിളുകൾ ഉപയോഗിച്ച് നമ്മുടെ ലോക്കൽ ഡെവലപ്പ്‌മെന്റ് എൻവയോൺമെന്റ് ക്രമീകരിക്കേണ്ടതാണ്. `azd` ഉപയോഗിക്കുന്നത് ഇതെളുപ്പമാക്കുന്നു.

1. Azure Developer CLI [പരിസ്ഥിതി വേരിയബിളുകൾ ഉപയോഗിക്കുന്നു](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) ആപ്ലിക്കേഷൻ ഡിപ്ലോയ്മെന്റുകളുടെ കോൺഫിഗറേഷൻ സജ്ജീകരണങ്ങൾ സൂക്ഷിക്കാൻ നിയന്ത്രിക്കാൻ.

1. പരിസ്ഥിതി വേരിയബിളുകൾ `.azure/<env-name>/.env` ൽ സൂക്ഷിച്ചിരിക്കുന്നു - ഇത് അവ `env-name` എന്ന deployment ൽ ഉപയോഗിച്ച പരിസ്ഥിതിയിലേക്കു സിംഗിൾസ്‌കോപ്പുചെയ്യുന്നു എന്നതും അതേ റീപ്പോയിലെ വ്യത്യസ്ത ഡിപ്ലോയ്മെന്റ് ലക്ഷ്യങ്ങൾക്കുള്ള വേരിയബിളുകൾ തമ്മിൽ വേർതിരിയുന്നതിനും സഹായിക്കുന്നു.

1. `azd` കമാൻഡ് പ്രത്യേക കമാൻഡ് (ഉദാ., `azd up`) റൺ ചെയ്യുമ്പോൾ പരിസ്ഥിതി വേരിയബിളുകൾ സ്വയമേവ ലോഡ് ചെയ്യും. പക്ഷേ `azd` ഓപ്പറേറ്റിങ് സിസ്റ്റത്തിന്റെ ലെവൽ പായലാറ്റ് വേരിയബിളുകൾ (ഉദാ., ഷെല്ലിൽ ക്രമീകരിച്ചവ) സ്വയം വായിക്കാറില്ല - അതിനർഥം `azd set env` ഉം `azd get env` ഉം സ്ക്രിപ്റ്റുകളിൽ വിവരങ്ങൾ കൈമാറാൻ ഉപയോഗിക്കുക.

ചില കമാൻഡുകൾ പരീക്ഷിയ്ക്കാം:

1. ഈ പരിസ്ഥിതിയിലുള്ള `azd` സജ്ജീകരിച്ച എല്ലാ പരിസ്ഥിതി വേരിയബിളുകളും ലഭിക്കുക:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      നിങ്ങൾക്ക് ഇങ്ങനെ ഏകദേശം കാണാം:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. ഒരു പ്രത്യേക മൂല്യം കണ്ടെത്തുക - ഉദാ., `AZURE_AI_AGENT_MODEL_NAME` മൂല്യം സജ്ജമാക്കിയിട്ടുണ്ടോ എന്ന് അറിയാൻ

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      നിങ്ങൾക്ക് ഇങ്ങനെ ഒന്നിനെ കാണാം - ഇത് ഡീഫോൾട്ടായി സജ്ജമാക്കിയിട്ടില്ല!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd`ക്കായി പുതിയ ഒരു പരിസ്ഥിതി വേരിയബിൾ സജ്ജമാക്കുക. ഇവിടെ, ഞങ്ങൾ ഏജന്റ് മോഡൽ നാമം അപ്ഡേറ്റ് ചെയ്യുന്നു. _ഗమనിക്കുക: ഏത് മാറ്റവും ഉടൻ `.azure/<env-name>/.env` ഫയലിൽ പ്രതിഫലിക്കും.

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      ഇപ്പോൾ, മൂല്യം സജ്ജീകരിച്ചിട്ടുണ്ടോ എന്ന് പരിശോധിയ്ക്കാം:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. കുറച്ച് റിസോഴ്‌സുകൾ സ്ഥിരതയുള്ളതാണ് (ഉദാ., മോഡൽ ഡിപ്ലോയ്മെന്റുകൾ) അതിനാൽ `azd up` മാത്രം ചെയ്താൽ അത് പുതുക്കാനാകില്ല. നിങ്ങളുടെ ആരംഭ ഡിപ്ലോയ്മെന്റ് നീക്കം ചെയ്ത്, മാറ്റങ്ങൾ വരുത്തിയ പരിസ്ഥിതി വേരിയബിളുകളോടെ വീണ്ടും ഡിപ്ലോയ് ചെയ്യാൻ ശ്രമിയ്ക്കുക.

1. **Refresh** മുമ്പ് azd ടെംപ്ലേറ്റ് ഉപയോഗിച്ച് ഇൻഫ്രാസ്ട്രക്ചർ ഡിപ്ലോയ്ചെയ്‌തിട്ടുണ്ടെങ്കിൽ - നിലവിലെ Azure ഡിപ്ലോയ്മെന്റിന്റെ അവസ്ഥ അനുസരിച്ച് നിങ്ങളുടെ ലോക്കൽ പരിസ്ഥിതി വേരിയബിളുകളുടെ നില പുതുക്കാൻ ഈ കമാൻഡ് ഉപയോഗിക്കുക:

      ```bash title="" linenums="0"
      azd env refresh
      ```

      രണ്ട് അല്ലെങ്കിൽ കൂടുതൽ പ്രാദേശിക ഡിവലപ്‌മെന്റ് പരിസ്ഥിതികളിലേക്ക് (ഉദാഹരണത്തിന്, നിരവധി ഡെവലപ്പർമാരുള്ള ടീം) പരിസ്ഥിതി ചാരത്തിന്റെ ഘടകങ്ങൾ _sync_ ചെയ്യാനുള്ള ശക്തമായ ഒരു മാർഗ്ഗമാണ് ഇത് - അനുധാനിച്ച ഇൻഫ്രാസ്ട്രക്ചർ പരിസ്ഥിതി ഘടക നിലയ്ക്കായുള്ള ഗ്രൗണ്ട് ത്രൂത്തിന്റെ റോളിലും പ്രവർത്തിക്കുന്നു. ടീം അംഗങ്ങൾ വെരിയബിൾസ് _refresh_ ചെയ്ത് പുനഃസംയോജിതമാകും.
