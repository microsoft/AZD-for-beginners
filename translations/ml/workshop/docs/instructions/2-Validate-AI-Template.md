> ജൂലൈ 2026-ൽ `azd 1.27.1` പ്രകാരം സാധുതയുള്ളത്.

!!! tip "ഈ മാഡ്യൂളിന്റെ അവസാനം നിങ്ങൾക്ക് കഴിയും"

    - [ ] AI സൊല്യൂഷൻ ആർക്കിടെക്ചർ വിശകലനം ചെയ്യുക
    - [ ] AZD ഡിപ്ലോയ്മെന്റ് വർക്ക്ഫ്ലോ മനസിലാക്കുക
    - [ ] AZD ഉപയോഗത്തിൽ സഹായത്തിനായി GitHub Copilot ഉപയോഗിക്കുക
    - [ ] **ലാബ് 2:** AI ഏജന്റുകൾ ടെമ്പ്ലേറ്റ് ഡിപ്ലോയ് ചെയ്യുകയും സാധുത പരിശോധിക്കുകയും ചെയ്യുക

## 1. പരിചയം

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) അല്ലെങ്കിൽ `azd` ഒരു ഓപ്പൺ-സോഴ്സ് കമാൻഡ് ലൈൻ സ്‌ട്രിംലൈൻ ചെയ്യുന്ന ഉപകരണം ആണ്, അത് ആസ്യൂർക്ക് ആപ്ലിക്കേഷൻ നിർമ്മിക്കാനും ഡിപ്ലോയ് ചെയ്യാനും സഹായിക്കുന്നു.

[AZD ടെമ്പ്ലേറ്റുകൾ](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) മാതൃക ആപ്ലിക്കേഷൻ കോഡ്, _ഇൻഫ്രാസ്ട്രക്‌ചർ-ആസ്-കೋഡ്_ ആസ്തികൾ, നിറഞ്ഞു കിടക്കുന്ന റീപോസിറ്ററികളാണ്, കൂടാതെ `azd` കോൺഫിഗറേഷൻ ഫയലുകൾ ഉൾക്കൊള്ളുന്നു. ആസ്യൂർക്ക് ഇൻഫ്രാസ്ട്രക്‌ചർ അനുവദിക്കുന്നതിന് `azd provision` കമാൻഡ് വളരെ എളുപ്പമാണ് - `azd up` ഉപയോഗിച്ച് നിങ്ങൾക്ക് ഇൻഫ്രാസ്ട്രക്‌ചർ അണിയറയും ആപ്ലിക്കേഷൻ ഒരേ സമയം ഡിപ്പ്ലോയ് ചെയ്യാം!

ഇതിന്റെ ഫലമായി, നിങ്ങളുടെ ആപ്ലിക്കേഷൻ വികസന പ്രക്രിയ ആരംഭിക്കാനായി ഏറ്റവും അടുത്തുള്ള _AZD സ്റ്റാർട്ടർ ടെമ്പ്ലേറ്റ്_ കണ്ടെത്തി, അതിനനുസരിച്ച് റീപോസിറ്ററി നിങ്ങളുടെ ആവശ്യങ്ങൾക്ക് അനുയോജ്യമായി മാറ്റിയെടുക്കുക എന്നതാണ് ഏറ്റവും ലളിതം.

തുടക്കമാക്കാൻ മുമ്പ്, നമുക്ക് ഉറപ്പാക്കാം Azure Developer CLI ഇൻസ്റ്റാൾ ചെയ്തിട്ടുണ്ടോയെന്ന്.

1. VS കോഡ് ടെർമിനൽ തുറന്ന് ഈ കമാൻഡ് ടൈപ്പ് ചെയ്യുക:

      ```bash title="" linenums="0"
      azd version
      ```

1. ഇങ്ങനെ എന്തെങ്കിലും കാണണം!

      ```bash title="" linenums="0"
      azd version 1.27.1 (commit <current-build>)
      ```

**ഇപ്പോൾ നിങ്ങൾ തയ്യാറാണ് azd ഉപയോഗിച്ച് ടെമ്പ്ലേറ്റ് തിരഞ്ഞെടുക്കാനും ഡിപ്ലോയ് ചെയ്യാനും**

## 2. ടെമ്പ്ലേറ്റ് തിരഞ്ഞെടുപ്പ്

Microsoft Foundry പ്ലാറ്റ്ഫോം [സുപാരി ലോൺ-ഇ സെറ്റ് ഓഫ് ആർടിസൺ AZD ടെമ്പ്ലേറ്റുകൾ](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) നൽകുന്നു, ജോലിസ്ഥലങ്ങളായ _മൾട്ടി-ഏജന്റ് വർക്ക്ഫ്ലോ ഓട്ടോമേഷൻ_ ഉം _മൾട്ടി-മോഡൽ ഉള്ളടക്കം പ്രോസസ്സിങ്_ എന്നിവ ഉൾപ്പെടുന്നു. ഈ ടെമ്പ്ലേറ്റുകൾ Microsoft Foundry പോർട്ടൽ സന്ദർശിച്ചും കണ്ടെത്താം.

1. സന്ദർശിക്കുക [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. ആവശ്യപ്പെട്ടാൽ Microsoft Foundry പോർട്ടലിൽ ലോഗിൻ ചെയ്യുക - ഇങ്ങനെ ഒരു പേജ് കാണാം.

![Pick](../../../../../translated_images/ml/01-pick-template.60d2d5fff5ebc374.webp)


**ബേസിക്** ഓപ്ഷനുകൾ നിങ്ങളുടെ സ്റ്റാർട്ടർ ടെമ്പ്ലേറ്റുകളാണ്:

1. [ ] [AI ചാറ്റിൽ തുടങ്ങുക](https://github.com/Azure-Samples/get-started-with-ai-chat) - ഇത് ഒരു അടിസ്ഥാന ചാറ്റ് ആപ്ലിക്കേഷൻ ആസ്യൂർ കണ്ടെയ്‌നർ ആപ്സിൽ ഡിപ്ലോയ് ചെയ്യുന്നു. ഇത് ഒരു അടിസ്ഥാന AI ചാറ്റ്ബോട്ട് സീനാരിയോ പരീക്ഷിക്കാനുദ്ദേശിക്കുന്നു.
1. [X] [AI ഏജന്റുകളുമായി തുടങ്ങുക](https://github.com/Azure-Samples/get-started-with-ai-agents) - ഇത് സ്റ്റാൻഡേർഡ് AI ഏജന്റ് (Foundry ഏജന്റുകളോടെ) ഡിപ്ലോയ് ചെയ്യുന്നു. ടൂളുകളും മോഡലുകളും ഉൾപ്പെടുത്തിയ ഏജന്റിക് AI സൊല്യൂഷനുകൾക്കായി പരിചയപ്പെടാൻ ഇത് ഉപയോഗിക്കുക.

രണ്ടാം ലിങ്ക് പുതിയ ബ്രൗസർ ടാബിൽ തുറക്കുക (അല്ലെങ്കിൽ സംബന്ധിച്ച കാർഡിൽ `Open in GitHub` ക്ലിക്ക് ചെയ്യുക). ഈ AZD ടെമ്പ്ലേറ്റിന്റെ റീപോസിറ്ററി കാണണം. README പരിശോധിക്കാൻ ഒരു നിമിഷം എടുത്തു നോക്കൂ. ആപ്ലിക്കേഷൻ ആർക്കിടെക്ചർ ഇങ്ങനെ കാണപ്പെടുന്നു:

## 3. ടെമ്പ്ലേറ്റ് സജീവമാക്കൽ

ഈ ടെമ്പ്ലേറ്റ് ഡിപ്ലോയ് ചെയ്തു അത് സാധുവാണെന്ന് ഉറപ്പാക്കാം. [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) വിഭാഗത്തിലെ മാർഗനിർദേശങ്ങൾ പാലിക്കും.

1. ടെമ്പ്ലേറ്റ് റീപോസിറ്ററിയ്ക്ക് ഒരു പ്രവർത്തനപരിസരം തിരഞ്ഞെടുക്കുക:

      - **GitHub Codespaces**: [ഇത്](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) ലിങ്കിൽ ക്ലിക്ക് ചെയ്ത് `Create codespace` സ്ഥിരീകരിക്കുക
      - **ലോകൽ ക്ലോൺ അല്ലെങ്കിൽ ഡെവ് കണ്ടെയ്നർ**: `Azure-Samples/get-started-with-ai-agents` ക്ലോൺ ചെയ്ത് VS കോഡിൽ തുറക്കുക

1. VS കോഡ് ടെർമിനൽ ശ്രമിച്ചു തുടങ്ങുന്നത് വരെ കാത്തിരിക്കുക, പിന്നീട് താഴെയുള്ള കമാൻഡ് ടൈപ്പ് ചെയ്യുക:

   ```bash title="" linenums="0"
   azd up
   ```

ഇത് ട്രിഗർ ചെയ്യുന്ന വർക്ക്ഫ്ലോース്റ്റെപുകൾ പൂർത്തിയാക്കുക:

1. ആസ്യൂറിൽ ലോഗിൻ ചെയ്യാൻ ആവശ്യപ്പെടും - സ്ഥിരീകരണത്തിന്റെ നിർദ്ദേശങ്ങൾ പാലിക്കുക
1. നിങ്ങൾക്കായിട്ടുള്ള വ്യത്യസ്ത പ്രവർത്തനപരിസരം നാമം നൽകുക - ഉദാഹരണത്തിന്: `nitya-mshack-azd`
1. `.azure/` ഫോൾഡർ സൃഷ്ടിക്കും - അതിന്റെ കീഴിലെ പ്രവർത്തനപരിസരം സൂചിപ്പിക്കുന്ന ഉപ-ഫോൾഡർ കാണാം
1. സബ്‌സ്ക്രിപ്ഷൻ നാമം തിരഞ്ഞെടുക്കാൻ ആവശ്യപ്പെടും - ഡിഫോൾട്ട് തെരഞ്ഞടുക്കുക
1. സ്ഥലം (ലൊക്കേഷൻ) തിരഞ്ഞെടുക്കാൻ ആവശ്യപ്പെടും - `East US 2` ഉപയോഗിക്കുക

ഇപ്പോൾ, പ്രൊവിഷനിംഗ് പൂർത്തിയാക്കാൻ കാത്തിരിക്കുക. **ഇത് 10-15 മിനിറ്റ് സമയമെടുക്കും**

1. പൂർത്തിയാകുമ്പോൾ, നിങ്ങളുടെ കൺസോൾ SUCCESS സന്ദേശം കാണിക്കും:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. ആസ്യൂർ പോർട്ടലിൽ നിങ്ങളുടെ പ്രവർത്തനപരിസരം ഉപയോഗിച്ച് പ്രൊവിഷൻ ചെയ്ത റിസോഴ്‌സ് ഗ്രൂപ്പ് കാണാൻ കഴിയും:

      ![Infra](../../../../../translated_images/ml/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **ഇപ്പോൾ നിങ്ങൾ തയ്യാറാണ് ഡിപ്ലോയ്മെന്റ് ചെയ്ത ഇൻഫ്രാസ്ട്രക്ടർവും ആപ്ലിക്കേഷനും സാധുത പരിശോധിക്കാൻ**.

## 4. ടെമ്പ്ലേറ്റ് സാധുത പരിശോധന

1. ആസ്യൂർ പോർട്ടൽ [Resource Groups](https://portal.azure.com/#browse/resourcegroups) പേജ് സന്ദർശിക്കുക - ആവശ്യപ്പെട്ടാൽ ലോഗിൻ ചെയ്യുക
1. നിങ്ങളുടെ പ്രവർത്തനപരിസര നാമത്തിലെ RG ക്ലിക്ക് ചെയ്യുക - മുകളിൽ തെളിയിച്ച പേജ് കാണും

      - ആസ്യൂർ കണ്ടെയ്നർ ആപ്സിലെ റെസോഴ്‌സ് ക്ലിക്ക് ചെയ്യുക
      - _Essentials_ സെക്ഷനിലെ ആപ്ലിക്കേഷൻ URL (മേൽക്കൂര വലത്) ക്ലിക്ക് ചെയ്യുക

1. ഇങ്ങനെ ഹോസ്റ്റ് ചെയ്ത ആപ്ലിക്കേഷൻ ഫ്രണ്ട് -എൻഡ് UI കാണണം:

   ![App](../../../../../translated_images/ml/03-test-application.471910da12c3038e.webp)

1. [ആദർശ ചോദ്യങ്ങൾ](https://github.com/Azure-Samples/get-started-with-ai-agents/blob/main/docs/sample_questions.md) ചോദ്യങ്ങൾ ചോദിച്ച് നോക്കൂ

      1. ചോദിക്കുക: ```ഫ്രാൻസിന്റെ തലസ്ഥാനം എന്താണ്?``` 
      1. ചോദിക്കുക: ```രണ്ട് ആളുകൾക്ക് $200 അകത്തെയുള്ള ഏറ്റവും നല്ല ടന്റ് ഏതാണ്, അതിന്റെ പ്രത്യേകതകൾ എന്തെല്ലാം?```

1. താഴെ കാണുന്നതുപോലെ നിങ്ങൾക്ക് ഉത്തറുകൾ ലഭിക്കാൻ സാധ്യതയുണ്ട്. _പക്ഷേ ഇത് എങ്ങനെ പ്രവർത്തിക്കുന്നു?_

      ![App](../../../../../translated_images/ml/03-test-question.521c1e863cbaddb6.webp)

## 5. ഏജന്റ് സാധുത പരിശോധന

ഈ ആസ്യൂർ കണ്ടെയ്നർ ആപ്പ് Microsoft Foundry പ്രൊജക്ടിൽ പ്രൊവിഷൻ ചെയ്ത AI ഏജന്റുമായി കണക്റ്റ് ചെയ്യുന്ന എന്റ്പോയിന്റ് ഡിപ്ലോയ് ചെയ്യുന്നു. ഇതിന്റെ അർത്ഥം എന്തെന്ന് നോക്കാം.

1. നിങ്ങളുടെ റെസോഴ്‌സ് ഗ്രൂപ്പിന്റെ ആസ്യൂർ പോർട്ടൽ _Overview_ പേജിലേക്ക് മടങ്ങുക

1. ആ ലിസ്റ്റിൽ നിന്നു `Microsoft Foundry` റെസോഴ്‌സ് ക്ലിക്ക് ചെയ്യുക

1. ഇതെങ്ങനെ കാണണം. `Go to Microsoft Foundry Portal` ബട്ടൺ क्लिक ചെയ്യുക. 
   ![Foundry](../../../../../translated_images/ml/04-view-foundry-project.fb94ca41803f28f3.webp)

1. നിങ്ങളുടെ AI ആപ്ലിക്കേഷനുള്ള Foundry പ്രൊജക്ട് പേജ് കാണണം
   ![Project](../../../../../translated_images/ml/05-visit-foundry-portal.d734e98135892d7e.webp)

## 8. പരിസരം വ്യത്യാസങ്ങൾ (Environment Variables)

ഇതുവരെയായി, ബ്രൗസർ വഴി ഡിപ്ലോയ് ചെയ്തും ഇൻഫ്രാസ്ട്രക്ടർ പ്രൊവിഷൻ ചെയ്തും ആപ്ലിക്കേഷൻ പ്രവർത്തനക്ഷമമാണെന്ന് പരിശോധിച്ചും കഴിഞ്ഞു. എന്നാൽ ആപ്ലിക്കേഷൻ _കോഡ്-ഫസ്റ്റ്_ ആയി ഉപയോഗിക്കാൻ, ഈ റിസോഴ്‌സുകൾക്ക് വേണ്ട ആവശ്യമായ പരിസരം വ്യത്യാസങ്ങൾ സജ്ജമാക്കേണ്ടത് അനിവാര്യമാണ്. ഇതിനായി `azd` ഉപയോഗിക്കുന്നത് എളുപ്പമാണ്.

1. ആസ്യൂർ ഡെവലപ്പർ CLI [പരിസരം വ്യത്യാസങ്ങൾ ഉപയോഗിക്കുന്നു](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) ആപ്ലിക്കേഷൻ ഡിപ്ലോയ്മെന്റുമുറ്റം കോൺഫിഗറേഷൻ ക്രമീകരണങ്ങൾ സൂക്ഷിക്കാൻ.

1. പരിസരം വ്യത്യാസങ്ങൾ `.azure/<env-name>/.env` ഫയലിൽ സേവ് ചെയ്യും - ഇത് `env-name` എന്ന പ്രവർത്തനപരിസരം പരിധിയിൽ മാത്രമേ ഇവ തുടരുമെന്നും ഒരേ റീപോസിറ്ററിയിലുള്ള വ്യത്യസ്ത ഡിപ്ലോയ്മെന്റ് ലക്ഷ്യങ്ങൾക്ക് പരിസരം വേർതിരിക്കുന്നു.

1. `azd` കമാൻഡ് ഒരു പ്രത്യേക കമാൻഡ് (ഉദാ., `azd up`) നടപ്പിലാക്കുമ്പോൾ പരിസരം വ്യത്യാസങ്ങൾ സ്വയം ലോഡ് ചെയ്യുന്നുണ്ട്. ഗണനാത്മകമായി, `azd` ഓപ്പറേറ്റിംഗ് സിസ്റ്റം ലെവലിലെ പരിസരം വ്യത്യാസങ്ങൾ (ഉദാ., ഷെല്ലിൽ സജ്ജീകരിച്ച) വായിക്കില്ല - ഇതിനായി `azd set env` , `azd get env` ഉപയോഗിക്കുക.


ചില കമാൻഡുകൾ പരീക്ഷിക്കാം:

1. ഈ പ്രവർത്തനപരിസരത്തിനായി `azd` ഇൻസ്റ്റാളാക്കിയ എല്ലാ പരിസരം വ്യത്യാസങ്ങളും ലഭിക്കുക:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      ഇങ്ങനെ ഒന്നും കാണും:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. ഒരു പ്രത്യേക മൂല്യം ലഭിക്കുക - ഞാൻ കാണുക ist `AZURE_AI_AGENT_MODEL_NAME` സജ്ജമാക്കിയിട്ടുണ്ടോ എന്ന്

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      ഇതുപോലെ കാണും - ഇത് ഡിഫോള്ട് ആയി സജ്ജമിച്ചിട്ടില്ല!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd`-ക്ക് പുതിയ പരിസരം വ്യത്യാസം സജ്ജീകരിക്കുക. ഇവിടെ ഏജന്റ് മോഡൽ നാമം അപ്ഡേറ്റ് ചെയ്യുന്നു. _കുറിപ്പ്: മാറ്റങ്ങൾ `.azure/<env-name>/.env` ഫയലിൽ ഉടനെ പ്രതിഫലിക്കും.

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      ഇപ്പോൾ മൂല്യം സജ്ജമാക്കിയതായി കാണാം:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. ചില റിസോഴ്‌സുകൾ സ്ഥിരമായതാണ് (ഉദാ., മോഡൽ ഡിപ്ലോയ്മെന്റ്) അതിനാൽ `azd up` മാത്രം കൊണ്ട് പുതിയ ഡിപ്ലോയ്മെന്റ് ബഹുമാനിക്കില്ല. ആദ്യം ഡിപ്ലോയ്മെന്റ് നീക്കം ചെയ്ത് പരിസരം വ്യത്യാസങ്ങൾ മാറ്റിയ ശേഷം വീണ്ടും ഡിപ്ലോയ് ചെയ്യാം.

1. **റിഫ്രഷ്** നിങ്ങൾ മുമ്പ് azd ടെമ്പ്ലേറ്റ് ഉപയോഗിച്ച് ഇൻഫ്രാസ്ട്രക്ടർ ഡിപ്ലോയ് ചെയ്തിട്ടുണ്ടെങ്കിൽ - നിലവിലുള്ള Azure ഡിപ്ലോയ്മെന്റ് നില അടിസ്ഥാനമാക്കി നിങ്ങളുടെ ലോക്കൽ പരിസരം വ്യത്യാസം അപ്ഡേറ്റ് ചെയ്യാനായി ഈ കമാൻഡ് ഉപയോഗിക്കുക:

      ```bash title="" linenums="0"
      azd env refresh
      ```


      ഇത് രണ്ട് അല്ലെങ്കിൽ അതിലധികം പ്രാദേശിക വികസന പരിസ്ഥിതികളിലേയ്ക്ക് (ഉദാ., നിരവധി വികസനക്കാരുള്ള ടീം) പരിസ്ഥിതിവ്യത്യാസങ്ങൾ _സിങ്ക്_ ചെയ്യുന്നതിനുള്ള ശക്തമായ മാർഗമാണ് - നടപ്പിലാക്കിയ ഘടനയെ പരിസ്ഥിതിവ്യത്യാസങ്ങളുടെ നിലവാരത്തിന്റെ യാഥാർത്ഥ്യമായി ഉപയോഗിക്കാൻ അനുവദിക്കുന്നു. ടീം അംഗങ്ങൾ უბრალოდ വ്യത്യാസങ്ങൾ ફરીമുഴുവനാക്കും (refresh) സിങ്ക് ആക്കുന്നതിനായി.
