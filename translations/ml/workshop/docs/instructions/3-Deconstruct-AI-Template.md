# 3. ഒരു ടെംപ്ലേറ്റ് ഡികൺസ്ട്രക്ട് ചെയ്യുക

!!! tip "ഈ മൊഡ്യൂൾ അവസാനിക്കുന്നതിനും മുമ്പ് നിങ്ങൾക്ക് സാധിക്കണം"

    - [ ] Azure സഹായത്തിനായി MCP സെർവറുകളുമായി GitHub Copilot സജീവമാക്കുക
    - [ ] AZD ടെംപ്ലേറ്റ് ഫോൾഡർ ഘടനയും ഘടകങ്ങളും മനസിലാക്കുക
    - [ ] ഇൻഫ്രാസ്റ്റ്രക്ചർ-ആസ്-കോഡ് (Bicep) സംഘടന നമുനകൾ എക്സ്പ്ലോർ ചെയ്യുക
    - [ ] **ലാബ് 3:** GitHub Copilot ഉപയോഗിച്ച് റിപോസിറ്ററി ആർക്കിടെക്ചർ എക്സ്പ്ലോർ ചെയ്ത് മനസിലാക്കുക

---

AZD ടെംപ്ലേറ്റുകളുമായി കൂടെ Azure Developer CLI (`azd`) നാം സ്റ്റാൻഡേഡൈസ്ഡ് റിപോസിറ്ററികളിലൂടെ മാതൃകാ കോഡ്, ഇൻഫ്രാസ്റ്റ്രക്ചർ, കോൺഫിഗറേഷൻ ഫയലുകൾ - രെഡിയായി ഡിപ്പ്ലോയ് ചെയ്യുന്നതിനുള്ള _സ്റ്റാർട്ടർ_ പ്രോജക്റ്റിന്റെ രൂപത്തിൽ - വേഗത്തിൽ ആരംഭിക്കാം.

**എന്നാൽ ഇപ്പോൾ, നമുക്ക് പ്രോജക്റ്റ് ഘടനയും കോഡ്ബേസ് ഉം മനസിലാക്കേണ്ടതുണ്ട് - കൂടാതെ AZD ടെംപ്ലേറ്റ് നിയന്ത്രിക്കാൻ കഴിയും - AZD സംബന്ധിച്ച മുൻ പരിചയമോ ധാരണയോ ഇല്ലാതെ!**

## 1. GitHub Copilot സജീവമാക്കുക

### 1.1 GitHub Copilot Chat ഇൻസ്റ്റാൾ ചെയ്യുക

ഇപ്പോൾ [GitHub Copilot with Agent Mode](https://code.visualstudio.com/docs/copilot/chat/chat-agent-mode) എക്സ്പ്ലോർ ചെയ്യാനുള്ള സമയം. നാം طبیعتഭാഷ ഉപയോഗിച്ച് നമ്മുടെ ചുമതല ഉയർന്ന തലത്തിൽ വിശദീകരിക്കുകയും, തുടർന്ന് സഹായം ലഭിക്കുകയും ചെയ്യാം. ഈ ലാബിനായി, [Copilot Free plan](https://github.com/github-copilot/signup) ഉപയോഗിക്കും, അതിൽ പൂർത്തീകരണങ്ങൾക്കും ചാറ്റ് ഇന്ററാക്ഷനുകൾക്കും മാസതല നിയന്ത്രണമുണ്ട്.

ഈ എക്സ്റ്റൻഷൻ മാർക്കറ്റ് പ്ലേസിൽ നിന്ന് ഇൻസ്റ്റാൾ ചെയ്യാം, പക്ഷേ നിങ്ങളുടെ Codespaces പരിസരത്ത് ഇത് ഇതിനകം ലഭ്യമായിരിക്കണം. _Copilot ഐക്കൺ ഡ്രോപ്-ഡൗൺ-ൽ നിന്ന് `Open Chat` ക്ലിക്ക് ചെയ്യുക - പിന്നീട് `What can you do?` പോലുള്ള പ്രോംപ്‌ട് ടൈപ്പ് ചെയ്യുക_ - ലോഗിൻ ചെയ്യാൻ പ്രോംപ്‌ട്ട് വന്നേക്കാം. **GitHub Copilot Chat സജ്ജമാണ്**.

### 1.2. MCP സെർവറുകൾ ഇൻസ്റ്റാൾ ചെയ്യുക

Agent മോഡ് ഫലപ്രദമാകാൻ, ശരിയായ ടൂളുകളിലേക്ക് ആക്‌സസ് വേണം അത് ജ്ഞാനം ഫെച് ചെയ്യാനും പ്രവർത്തനങ്ങൾ നടത്താനും. ഇതിനു MCP സെർവറുകൾ സഹായിക്കും. നാം താഴെയുള്ള സെർവറുകൾ ക്രമീകരിക്കും:

1. [Azure MCP Server](../../../../../workshop/docs/instructions)
1. [Microsoft Docs MCP Server](../../../../../workshop/docs/instructions)

അതു സജീവമാക്കാൻ:

1. `.vscode/mcp.json` എന്ന ഫയൽ ഉണ്ടാകാത്ത പക്ഷം സൃഷ്ടിക്കുക
1. താഴെ കൊടുത്തിരിക്കുന്ന കോഡ് ആ ഫയലിൽ പകർത്തി സെർവറുകൾ ആരംഭിക്കുക!
   ```json title=".vscode/mcp.json"
   {
      "servers": {
         "Azure MCP Server": {
            "command": "npx",
            "args": [
            "-y",
            "@azure/mcp@latest",
            "server",
            "start"
            ]
         },
         "microsoft.docs.mcp": {
            "type": "http",
            "url": "https://learn.microsoft.com/api/mcp"
         }
      }
   }
   ```

??? warning "നിങ്ങൾക്ക് `npx` ഇൻസ്റ്റാൾ ചെയ്യുന്നില്ലെന്ന് ഒരു പിഴവ് കാണാമെന്ന് (പരിഹാരത്തിനായി പടിയെടുത്തു)"

      ഇത് പരിഹരിക്കാൻ, `.devcontainer/devcontainer.json` ഫയൽ തുറന്ന് features സെക്ഷനിൽ ഈ ലൈൻ ചേർക്കുക. അപ്പോൾ കണ്ടെയ്നർ പുനർനിർമ്മിക്കണം. ഇനി `npx` ഇൻസ്റ്റാൾ ചെയ്തതായി കാണും.

      ```title="" linenums="0"
         "features": {
            "ghcr.io/devcontainers/features/node:1": {},
            ...
         },
      ```

### 1.3. GitHub Copilot Chat പരിശോധിക്കുക

**ആദ്യം `az login` ഉപയോഗിച്ച് VS Code കമാൻഡ് ലൈനിൽ നിന്ന് Azure-യുമായി ഓട്ടൻറിക്കേറ്റ് ചെയ്യുക.**

ഇപ്പോൾ നിങ്ങളുടെ Azure സബ്സ്ക്രിപ്ഷൻ സ്റ്റാറ്റസ് ചോദിക്കാനും, ഡിപ്പ്ലോയ്മെന്റ് ചെയ്ത റിസോഴ്‌സുകളോ കോൺഫിഗറേഷനോ സംബന്ധിച്ച ചോദ്യങ്ങൾ ചോദിക്കാനും കഴിയും. ഈ പ്രോംപ്‌റുകൾ പരീക്ഷിക്കുക:

1. `List my Azure resource groups`
1. `#foundry list my current deployments`

നീകളും Azure ഡോക്യുമെന്റേഷൻ സംബന്ധിച്ച ചോദ്യങ്ങൾ ചോദിച്ച് Microsoft Docs MCP സെർവറിലുള്ള അടിസ്ഥാനത്തിൽ മറുപടി നേടാം. ഈ പ്രോംപ്‌റുകൾ പരീക്ഷിക്കുക:

1. `#microsoft_docs_search What is Azure Developer CLI?`
1. `#microsoft_docs_search Show me a Python tutorial to chat with deployed model`

അല്ലെങ്കിൽ ഒരു ടാസ്ക് പൂർത്തിയാക്കാൻ കോഡ് സ്നിപ്പെറ്റുകൾ ചോദിക്കാം. ഈ പ്രോംപ്‌റ്റ് പരീക്ഷിക്കുക.

1. `Give me a Python code example that uses AAD for an interactive chat client`

`Ask` മോഡിൽ, ഇത് നിങ്ങൾക്ക് കോഡ് കോപ്പി ചെയ്തു പരീക്ഷിക്കാൻ നൽകും. `Agent` മോഡിൽ ഇത് ഒരു പടി മുന്നോട്ട് പോയി, നിർബന്ധമായ റിസോഴ്‌സുകളും സെറ്റപ്പ് സ്ക്രിപ്റ്റുകളും ഡോക്യുമെന്റേഷനും സൃഷ്ടിച്ചേക്കാം, നിങ്ങളെ ടാസ്ക് നടപ്പിലാക്കാൻ സഹായിക്കാനായി.

**ഇപ്പോൾ നിങ്ങൾക്ക് ടെംപ്ലേറ്റ് റിപോസിറ്ററി എക്സ്പ്ലോർ ചെയ്യാൻ യോഗ്യം ആകും**

## 2. ആർക്കിടെക്ചർ ഡികൺസ്ട്രക്ട് ചെയ്യുക

??? prompt "ചോദ്യം: docs/images/architecture.png യിലെ അപ്ലിക്കേഷൻ ആർക്കിടെക്ചർ 1 പാരഗ്രാഫിൽ വിശദീകരിക്കുക"

      ഈ അപ്ലിക്കേഷൻ Azure-ൽ നിർമ്മിച്ച AI- ചാലിത ചാറ്റ് അപ്ലിക്കേഷനാണ്, ഇത് ആധുനിക ഏജന്റ്-അടിസ്ഥാനത്തിലുള്ള ആർക്കിടെക്ചർ കാണിക്കുന്നു. പരിഹാരം Azure Container App-യെ കേന്ദ്രീകരിച്ചാണ്, ഇത് പ്രധാന അപ്ലിക്കേഷൻ കോഡ് ഹോസ്റ്റ് ചെയ്യുന്നു, ഉപയോക്തൃ ഇൻപുട്ട് പ്രോസസ്സ് ചെയ്ത് AI ഏജന്റിന്റെ മുഖാന്തിരം ബുദ്ധിമുട്ടുള്ള പ്രതികരണങ്ങൾ സൃഷ്ടിക്കുന്നു.
      
      ഈ ആർക്കിടെക്ചർ Microsoft Foundry Project-നെ AI കഴിവുകളുടെ അടിസ്ഥാനമായി ഉപയോഗിക്കുന്നു, Azure AI Services-നുമായി ബന്ധപ്പെടുന്നു, അവിടെ അടിസ്ഥാന ഭാഷാ മോഡലുകൾ (ഉദാ: GPT-4o-mini)യും ഏജന്റ് പ്രവർത്തനങ്ങളും നൽകുന്നു. ഉപയോക്തൃ ഇടപെടലുകൾ React അടിസ്ഥാനത്തിലുള്ള ഫ്രണ്ട്‌എൻഡിലൂടെ FastAPI ബാക്ക്എൻഡിലേക്ക് ഒഴുകുന്നു, അത് AI ഏജന്റ് സർവിസുമായി ചേർന്ന് പ്രസംഗം ബന്ധമായ മറുപടികൾ സൃഷ്ടിക്കുന്നു.
      
      സിസ്റ്റം ഫയൽ തിരയൽ അല്ലെങ്കിൽ Azure AI Search സർവീസ് വഴി ജ്ഞാനം വീണ്ടെടുക്കൽ കഴിവുകൾ ഉൾക്കൊള്ളുന്നു, ഏജന്റ് അപ്ലോഡ് ചെയ്ത ഡോക്യുമെന്റുകളിൽ നിന്ന് വിവരം ആക്സസ് ചെയ്യുകയും ഉദ്ധരിച്ചും ചെയ്യാൻ കഴിയും. പ്രവർത്തന കാര്യക്ഷമതക്ക് ആർക്കിടെക്ചറിൽ Application Insights, Log Analytics Workspace വഴുകം ട്രേസിംഗ്, ലോഗിംഗ്, പ്രകടനം മെച്ചപ്പെടുത്തൽ എന്നിവ ഉൾപ്പെടുത്തിയിട്ടുണ്ട്.
      
      Azure Storage ആപ്ലിക്കേഷൻ ഡാറ്റക്കും ഫയൽ അപ്ലോഡിനും ബ്ലോബ്സ്ടോറേജ് നൽകുന്നു, Managed Identity ക്രെഡൻഷ്യലുകൾ സംഭരിച്ചു കൂടാതെ Azure റിസോഴ്‌സുകൾക്ക് സുരക്ഷിത ആക്‌സസ് ഉറപ്പാക്കുന്നു. ഈ മുഴുവൻ പരിഹാരം സ്കെയിലബിൾ, മനേജബിള്‍, കണ്ടെയ്നറൈസഡ് അപ്ലിക്കേഷൻ ആവശ്യകത അനുസരിച്ച് സ്വയം സ്കെയിൽ ചെയ്യുകയും, Azure മാനേജുചെയ്യുന്ന സേവനങ്ങളുടെ സുരക്ഷാ, നിരീക്ഷണ, CI/CD കഴിവുകൾ നൽകുകയും ചെയ്യുന്ന രൂപത്തിലാണ്.

## 3. റിപോസിറ്ററി ഘടന

!!! prompt "ചോദ്യം: ടെംപ്ലേറ്റ് ഫോൾഡർ ഘടന വ്യക്തമാക്കുക. ദൃശ്യമയ hierarchical ഡിസൈൻകൊണ്ട് ആരംഭിക്കുക."

??? info "ഉത്തരം: ദൃശ്യമ hierarchical ഡിസൈൻ"

      ```bash title="" 
      get-started-with-ai-agents/
      ├── 📋 കോൺഫിഗറേഷൻ & സെറ്റ്‌അപ്പ്
      │   ├── azure.yaml                    # Azure Developer CLI കോൺഫിഗറേഷൻ
      │   ├── docker-compose.yaml           # ലോക്കൽ ഡവലപ്പ്മെന്റ് കണ്ടെയ്‌നറുകൾ
      │   ├── pyproject.toml                # Python പ്രോജക്റ്റ് കോൺഫിഗറേഷൻ
      │   ├── requirements-dev.txt          # ഡവലപ്പ്മെന്റ് ഡിപ്പൻഡൻസികൾ
      │   └── .devcontainer/                # VS Code ഡെവ് കണ്ടെയ്‌നർ സെറ്റ്‌അപ്പ്
      │
      ├── 🏗️ ഇൻഫ്രാസ്റ്റ്രക്ചർ (infra/)
      │   ├── main.bicep                    # പ്രധാന ഇൻഫ്രാസ്റ്റ്രക്ചർ ടെംപ്ലേറ്റ്
      │   ├── api.bicep                     # API-സ്പെസിഫിക് റിസോഴ്‌സുകൾ
      │   ├── main.parameters.json          # ഇൻഫ്രാസ്റ്റ്രക്ചർ പാരാമീറ്ററുകൾ
      │   └── core/                         # മോഡുലാർ ഇൻഫ്രാസ്ട്രക്ചർ ഘടകങ്ങൾ
      │       ├── ai/                       # AI സേവന കോൺഫിഗറേഷനുകൾ
      │       ├── host/                     # ഹോസ്റ്റിംഗ് ഇൻഫ്രാസ്റ്റ്രക്ചർ
      │       ├── monitor/                  # നിരീക്ഷണവും ലോഗിംഗും
      │       ├── search/                   # Azure AI Search സജ്ജീകരണം
      │       ├── security/                 # സുരക്ഷയും ഐഡന്റിറ്റിയും
      ```

### 3.1. കോർ ആപ്പ് ആർക്കിടെക്ചർ

ഈ ടെംപ്ലേറ്റ് ഒരു **ഫുള്‍-സ്റ്റാക്ക് വെബ് ആപ്ലിക്കേഷൻ** മാതൃക പിന്തുടരുന്നു:

- **ബാക്ക്‌എൻഡ്**: Python FastAPI Azure AI സംയോജിതം
- **ഫ്രണ്ട്‌എൻഡ്**: TypeScript/React Vite ബിൽഡ് സിസ്റ്റം
- **ഇൻഫ്രാസ്റ്റ്രക്ചർ**: Azure Bicep ടെംപ്ലേറ്റുകൾ ക്ളൗഡ് റിസോഴ്‌സുകൾക്ക്
- **കണ്ടെയിനറൈസേഷൻ**: Docker സ്ഥിരതയുള്ള ഡിപ്പ്ലോയ്മെന്റിന്

### 3.2. ഇൻഫ്രാ അസ്സ് കോഡ് (bicep)

ഇൻഫ്രാസ്‌ട്രക്ചർ ലെയർ **Azure Bicep** ടെംപ്ലേറ്റുകൾ ഉപയോഗിച്ച് മോഡുലാർ ആയി ക്രമീകരിച്ചിരിക്കുന്നു:

   - **`main.bicep`**: എല്ലാ Azure റിസോഴ്‌സുകളും ഓർക്കസ്ട്രേറ്റ് ചെയ്യുന്നു
   - **`core/` മോഡ്യൂളുകൾ**: വിവിധ സേവനങ്ങൾക്ക് പുനരുപയോഗയോഗ്യ ഘടകങ്ങൾ
      - AI സേവനങ്ങൾ (Azure OpenAI, AI Search)
      - കണ്ടെയ്‌നർ ഹോസ്റ്റിംഗ് (Azure Container Apps)
      - നിരീക്ഷണം (Application Insights, Log Analytics)
      - സുരക്ഷ (Key Vault, Managed Identity)

### 3.3 ആപ്ലിക്കേഷൻ സ്രോതസ് (`src/`)

**ബാക്ക്‌എൻഡ് API (`src/api/`)**:

- FastAPI അടിസ്ഥാനമാക്കിയ REST API
- Foundry Agents സംയോജനം
- ജ്ഞാനം വീണ്ടെടുക്കലിനുള്ള സെർച്ച് ഇൻഡക്സ് മാനേജ്മെന്റ്
- ഫയൽ അപ്ലോഡ്, പ്രോസസ്സിംഗ് കഴിവുകൾ

**ഫ്രണ്ട്‌എൻഡ് (`src/frontend/`)**:

- ആധുനിക React/TypeScript SPA
- വേഗത്തിലുള്ള ഡവലപ്പ്മെന്റിനും അപ്റ്റിമൈസ്ഡ് ബിൽഡിനും Vite
- ഏജന്റ് ഇടപെടലുകൾക്കുള്ള ചാറ്റ് ഇന്റർഫേസ്

**ജ്ഞാനമേഖല (`src/files/`)**:

- സാമ്പിൾ കസ്റ്റമർ, പ്രോഡക്ട് ഡാറ്റ
- ഫയൽ അടിസ്ഥാനത്തിലുള്ള ജ്ഞാനം വീണ്ടെടുക്കൽ കാണിക്കുന്നു
- JSON, Markdown ഫോർമാറ്റ് ഉദാഹരണങ്ങൾ

### 3.4 ഡെവ്ഓപ്‌സ് & ഓട്ടോമേഷൻ

**സ്ക്രിപ്റ്റുകൾ (`scripts/`)**:

- മോഡൽ ക്വോട്ടാ മാനേജ്മെന്റ്

**Azure Developer CLI ഇന്റഗ്രേഷൻ**:

- `azure.yaml` `azd` വർക്‌ഫ്ളോക്സിനുള്ള കോൺഫിഗറേഷൻ
- ഓട്ടോമേറ്റഡ് പ്രൊവിഷനിംഗ്, ഡിപ്ലോയ്‌മെന്റ്
- എൻവയിരോൺമെന്റ് വെറിൿബ്ൾ മാനേജ്മെന്റ്

### 3.5 ടെസ്റ്റിങ് & ഗുണനിലവാരം ഉറപ്പാക്കൽ
