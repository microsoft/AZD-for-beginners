# 3. ഒരു ടെംപ്ലേറ്റ് വിഭജിക്കുക

!!! tip "ഈ മഡ്യൂളിന്റെ അന്ത്യത്തിൽ നിങ്ങൾക്ക് കഴിയും"

    - [ ] Azure സഹായത്തിനായി MCP സർവറുകൾ ഉപയോഗിച്ച് GitHub Copilot സജീവമാക്കുക
    - [ ] AZD ടെംപ്ലേറ്റ് ഫോൾഡർ ഘടനയും ഘടകങ്ങളും മനസ്സിലാക്കുക
    - [ ] ഇൻഫ്രാസ്ട്രക്ചർ-എസ്-കോഡ് (Bicep) സംഘാടകമുറികൾ പരിശോധിക്കുക
    - [ ] **ലാബ് 3:** GitHub Copilot ഉപയോഗിച്ച് റിപ്പോസിറ്ററി പാശ്ചാത്തലത്തെ അന്വേഷിച്ച് മനസ്സിലാക്കുക

---


AZD ടെംപ്ലേറ്റുകളും Azure Developer CLI (`azd`) ഉം ഉപയോഗിച്ച് നമുക്ക് സാധ്യമാണ് സാമ്പിൾ കോഡ്, ഇൻഫ്രാസ്ട്രക്ചർ, കോൺഫിഗറേഷൻ ഫയലുകൾ എന്നിവ നൽകുന്ന സാധാരണ രൂപകല്പനയുള്ള റിപ്പോസിറ്ററികൾ ഉപയോഗിച്ച് എഐ ഡെവലപ്മെന്റ് യാത്ര അടിയന്തിരമായി തുടങ്ങുക - ഇത് തയ്യാറായിട്ടുള്ള _സ്റ്റാർട്ടർ_ പ്രോജക്റ്റ് രൂപത്തിൽ കാണപ്പെടുന്നു.

**എങ്കിലും ഇപ്പോൾ, നമുക്ക് പ്രോജക്റ്റ് ഘടനയും കോഡ്‌ബേസും മനസ്സിലാക്കേണ്ടതുണ്ട് - കൂടാതെ AZD ടെംപ്ലേറ്റ് ഇഷ്ടാനുസൃതമാക്കാൻ കഴിവ് നേടേണ്ടതാണ് - AZD നെ കുറിച്ചുള്ള മുൻപരിചയമോ ബോധമോ ഇല്ലാതെ!**

## 1. GitHub Copilot സജീവമാക്കുക

### 1.1 GitHub Copilot Chat ഇൻസ്റ്റാൾ ചെയ്യുക

ഇപ്പോൾ സമയം വന്നു [GitHub Copilot with Agent Mode](https://code.visualstudio.com/docs/copilot/chat/chat-agent-mode) നിരീക്ഷിക്കാൻ. ഇനി നാം നാച്ചുറൽ ഭാഷ ഉപയോഗിച്ച് ഞങ്ങളുടെ ടാസ്ക് ഉയർന്ന തലത്തിൽ വിവരണമാക്കുകയും പ്രവർത്തനത്തിൽ സഹായം ലഭിക്കുകയും ചെയ്യാം. ഈ ലാബ്‌ക്കായി, നാം [Copilot Free plan](https://github.com/github-copilot/signup) ഉപയോഗിക്കും, ഇത് മാസശേഷം പൂർത്തീകരണങ്ങൾക്കും ചാറ്റ് ഇടപെടലുകൾക്കും പരിധിയുണ്ട്.

ഈ എക്സ്റ്റെൻഷൻ മാർക്കറ്റ്പ്ലേസിൽ നിന്ന് ഇൻസ്റ്റാൾ ചെയ്യാവുന്നതാണ്, പക്ഷേ ഇത് നിങ്ങളുടെ Codespaces പരിസ്ഥിതിയിൽ ഇതിനകം ലഭ്യമായിരിക്കണം. _Copilot ഐക്കൺ ഡ്രോപ്പ്ഡൗണിൽ നിന്ന് `Open Chat` ക്ലിക്ക് ചെയ്ത് `What can you do?` പോലെയുള്ള പ്രോംപ്റ്റ് ടൈപ്പ് ചെയ്യുക_ - നിങ്ങളെ ലോഗിൻ ചെയ്യാനായി പ്രോംപ്റ്റ് ചെയ്യാം. **GitHub Copilot Chat സജ്ജമാണ്**.

### 1.2. MCP സർവറുകൾ ഇൻസ്റ്റാൾ ചെയ്യുക

Agent മോഡ് ഫലപ്രദമാക്കാൻ, അതിന് ശരിയായ ടൂളുകൾക്കുള്ള പ്രവേശനം വേണം, ഇത് ജ്ഞാനം തിരയുന്നതിൽ സഹായിക്കുകയും പ്രവർത്തികൾ കൈകാര്യം ചെയ്യുകയും ചെയ്യുന്നു. ഇതിൽ MCP സർവറുകൾ സഹായിക്കും. നാം ഈ സർവറുകൾ ക്രമീകരിക്കും:

1. [Azure MCP Server](../../../../../workshop/docs/instructions)
1. [Microsoft Docs MCP Server](../../../../../workshop/docs/instructions)

ഇവ സജീവമാക്കാൻ:

1. `.vscode/mcp.json` എന്നൊരു ഫയൽ ഉണ്ടാകാത്ത പക്ഷം സൃഷ്ടിക്കുക
1. താഴെ കൊടുത്തിരിക്കുന്നത് ആ ഫയലിലേക്ക് പകർത്തുക - തുടർന്ന് സർവറുകൾ ആരംഭിക്കുക!
   ```json title=".vscode/mcp.json"
   {
      "servers": {
         "Azure MCP Server": {
            "command": "npx",
            "args": [
            "-y",
            "@azure/mcp@latest",
            "server",
            "start"
            ]
         },
         "microsoft.docs.mcp": {
            "type": "http",
            "url": "https://learn.microsoft.com/api/mcp"
         }
      }
   }
   ```

??? warning "നിങ്ങൾക്ക് `npx` ഇൻസ്റ്റാൾ ചെയ്തിട്ടില്ലെന്നൊരു പിശക് കിട്ടാൻ ഇടയുണ്ട് (പരിഹാരത്തിനായി ക്ലിക്ക് ചെയ്യുക)"

      പരിഹരിക്കാൻ, `.devcontainer/devcontainer.json` ഫയൽ തുറന്ന് ഫീച്ചേഴ്‌സ് സെക്ഷൻിൽ താഴെയുള്ള വരി ചേർക്കുക. തുടർന്ന് കണ്ടെയ്‌നർ പുനഃനിർമിക്കുക. ഇനി നിങ്ങൾക്ക് `npx` ഇൻസ്റ്റാൾ ചെയ്തതായി കാണാം.

      ```title="" linenums="0"
         "features": {
            "ghcr.io/devcontainers/features/node:1": {},
            ...
         },
      ```

### 1.3. GitHub Copilot Chat ടെസ്റ്റ് ചെയ്യുക

**ആദ്യമായി VS Code കമാൻഡ് ലൈൻ ഉപയോഗിച്ച് `az login` ചെയ്യുക, Azure ൽ അംഗീകാരം നേടുന്നതിന്.**

ഇപ്പോൾ നിങ്ങൾക്ക് നിങ്ങളുടെ Azure സബ്‌സ്‌ക്രിപ്ഷൻ നില പരിശോധിക്കാനും, നിർവഹിച്ചിട്ടുള്ള റിസോഴ്‌സുകൾ അല്ലെങ്കിൽ കോൺഫിഗറേഷൻ സംബന്ധിച്ച ചോദ്യങ്ങൾ ചോദിക്കാനും കഴിയും. ഈ പ്രോംപ്റ്റുകൾ പരീക്ഷിക്കുക:

1. `List my Azure resource groups`
1. `#foundry list my current deployments`

നീങ്ങളും Azure ഡോക്യുമെന്റേഷൻ സംബന്ധിച്ച ചോദ്യങ്ങൾ ചോദിച്ച് Microsoft Docs MCP സർവർ ആധാരമാക്കിയ മറുപടികൾ ലഭിക്കാൻ കഴിയും. ഈ പ്രോംപ്റ്റുകൾ ശ്രമിക്കുക:

1. `#microsoft_docs_search What is Azure Developer CLI?`
1. `#microsoft_docs_search Show me a Python tutorial to chat with deployed model`

അഥവാ ഒരു ടാസ്‌ക് പൂർത്തിയാക്കാൻ കോഡ് സ്നിപ്പറ്റുകൾ ചോദിക്കാം. ഈ പ്രോംപ്റ്റ് പരീക്ഷിക്കുക.

1. `Give me a Python code example that uses AAD for an interactive chat client`

`Ask` മോഡിൽ, ഇത് നിങ്ങൾക്ക് കോഡ് കോപ്പി-പേച് ചെയ്ത് പരീക്ഷിക്കാമായിരിക്കും. `Agent` മോഡിൽ, ഇത് ഒരു മുകളിൽ പോയി ബന്ധപ്പെട്ട റിസോഴ്‌സുകൾ സൃഷ്ടിക്കുകയും - സജ്ജമാക്കൽ സ്ക്രിപ്റ്റുകളും ഡോക്യുമെന്റേഷനും ഉൾപെടുത്തി - നിങ്ങളെന്ന ടാസ്‌ക് നടപ്പിലാക്കാൻ സഹായിക്കുകയും ചെയ്യും.

**ഇപ്പോൾ നിങ്ങൾക്ക് ടെംപ്ലേറ്റ് റിപ്പോസിറ്ററി പഠനം ആരംഭിക്കാൻ സമർപ്പിതരാണ്**

## 2. ആർക്കിടെക്ചർ വേർതിരിക്കുക

??? prompt "ചോദ്യം: docs/images/architecture.png എന്ന ചിത്രത്തിലെ അപ്ലിക്കേഷൻ ആർക്കിടെക്ചർ 1 പാരഗ്രാഫിൽ വിശദീകരിക്കുക"

      ഈ അപ്ലിക്കേഷൻ ഒരു എഐചാലിത ചാറ്റ് അപ്ലിക്കേഷൻ ആണ്, അത് Azure ൽ നിർമ്മിച്ചിരിക്കുന്നപ്പോൾ ഒരു ആധുനിക ഏജന്റ് അധിഷ്ഠിത ആർക്കിടെക്ചർ കാണിക്കുന്നു. ഈ സൊല്യൂഷൻ Azure Container App ഉപയോഗിച്ച് പ്രധാന അപ്ലിക്കേഷൻ കോഡ് ഹോസ്റ്റ് ചെയ്യുന്നു, ഇത് ഉപയോക്തൃ ഇൻപുട്ടും എഐ ഏജന്റാണ് വഴി ബുദ്ധിമുട്ടുള്ള മറുപടികൾ സൃഷ്ടിക്കുന്നു. 
      
      ആർക്കിടെക്ചർ Microsoft Foundry Project നെ ആധാരമാക്കി AI കഴിവുകൾക്ക് അടിസ്ഥാനസഹായം നൽകുന്നു, Azure AI Services ലായിയാംഗിക മോഡലുകൾ (ഉദാഹരണത്തിന് gpt-4.1-mini) എന്നിവ കണക്റ്റ് ചെയ്യുന്നു. ഉപയോക്തൃ ഇടപെടലുകൾ React അടിസ്ഥാനമാക്കിയ ഫ്രണ്ട്‌എൻഡിൽ നിന്നു FastAPI ബാക്ക്‌എൻഡിലേക്കു പാഞ്ഞ്, എഐ ഏജന്റ് സർവീസുമായി ബന്ധപ്പെടുന്നു, പ്രസക്തമായ മറുപടികൾ സൃഷ്ടിക്കാൻ. 
      
      സിസ്റ്റം ഫയൽ തിരച്ചിലോ Azure AI Search സർവീസോ വഴി ജ്ഞാന ഉദ്ധരണമുള്ള കഴിവുകൾ ഉൾപ്പെടുന്നു, അതിലൂടെ ഏജന്റ് അപ്‌ലോഡ് ചെയ്ത ഡോക്യമെന്റുകളിൽ നിന്നുള്ള വിവരങ്ങൾ ആക്സസ് ചെയ്ത് ചൂണ്ടിക്കാണിക്കുന്നു. പ്രവര്‍ത്തനത്തിന് ഉത്തമത്വം നൽകാൻ, ആർക്കിടെക്ചറിൽ അപ്ലിക്കേഷൻ ഇൻസൈറ്റ്‌സ്, ലോഡ് അനലിറ്റിക്സ് വർക്ക്സ്പേസ് എന്നിവയിലൂടെ ട്രേസിംഗ്, ലോഗിംഗ്, പ്രകടന മെച്ചപ്പെടുത്തൽ എന്നിവ കാണാൻ കഴിയും. 
      
      Azure സ്റ്റോറേജ് blob സ്റ്റോറേജ് നൽകുന്നു അപ്ലിക്കേഷൻ ഡാറ്റക്കും ഫയൽ അപ്‌ലോഡ്സിനും, Managed Identity വഴി ക്രെഡൻഷ്യലുകൾ സൂക്ഷിക്കാതെ സുരക്ഷിത ആക്സസ് ഉറപ്പാക്കുന്നു. മുഴുവൻ സൊല്യൂഷനും സ്കെയിലബിലിട്ടിയുടെയും പരിപാലനക്ഷമതയുടെയും ആവശ്യങ്ങളുണ്ടായി രൂപകൽപ്പന ചെയ്തിട്ടുണ്ട്, ആവശ്യാനുസരണം കണ്ടെയ്‌നർ ആപ്പിനോട് ചേർന്ന് സ്വയം സ്കെയിൽ ചെയ്യുക, Azure മാനേജുചെയ്യുന്ന സർവീസുകളിലൂടെ ഇൻബിൽറ്റ് സുരക്ഷ, നിരീക്ഷണവും CI/CD കഴിവുകൾ നൽകുന്നു.

### 3.1. കോർ ആപ്പ് ആർക്കിടെക്ചർ

ഈ ടെംപ്ലേറ്റ് ഒരു **ഫുൾ-സ്റ്റാക്ക് വെബ് അപ്ലിക്കേഷൻ** പാറ്റേൺ പിന്തുടരുന്നു:

- **ബാക്ക്‌എൻഡ്**: Python FastAPI Azure AI സമന്വയത്തോടെ
- **ഫ്രണ്ട്‌എൻഡ്**: TypeScript/React Vite ബിൽഡ് സിസ്റ്റവും ഉപയോഗിച്ച്
- **ഇൻഫ്രാസ്ട്രക്ചർ**: ക്ലൗഡ് റിസോഴ്‌സുകൾക്കായി Azure Bicep ടെംപ്ലേറ്റുകൾ
- **കണ്ടെയ്‌നറൈസേഷൻ**: സ്ഥിരതയുള്ള ഡിപ്പ്ലോയ്മെന്‌റ്Docker ഉപയോഗിച്ച്

### 3.2 ഇൻഫ്രാ അസ്സ് കോഡ് (bicep)

ഇൻഫ്രാസ്ട്രക്ചർ ലെയർ **Azure Bicep** ടെംപ്ലേറ്റുകൾ ഉപയോഗിച്ച് മൊഡ്യൂളർ ആയി സംഘാടകമുറികളുമായി കിടക്കുന്നു:

   - **`main.bicep`**: എല്ലാ Azure റിസോഴ്‌സുകളും കോർഡിനേറ്റു ചെയ്യുന്നു
   - **`core/` മൊഡ്യൂളുകൾ**: വ്യത്യസ്ത സർവീസുകൾക്ക് പുനരുപയോഗ യോഗ്യ ഘടകങ്ങൾ
      - AI സർവീസുകൾ (Microsoft Foundry Models, AI Search)
      - കണ്ടെയ്‌നർ ഹോസ്റ്റിംഗ് (Azure Container Apps)
      - നിരീക്ഷണം (Application Insights, Log Analytics)
      - സുരക്ഷ (Key Vault, Managed Identity)

### 3.3 അപ്ലിക്കേഷൻ സോഴ്‌സ് (`src/`)

**ബാക്ക്‌എൻഡ് API (`src/api/`)**:

- FastAPI അടിസ്ഥാനമാക്കിയ REST API
- Foundry Agents സമന്വയം
- ജ്ഞാനം തിരയലിനുള്ള സെർച്ച് ഇൻഡക്സ് മാനേജ്മെന്റ്
- ഫയൽ അപ്‌ലോഡ്, പ്രോസസ്സിംഗ് കഴിവുകൾ

**ഫ്രണ്ട്‌എൻഡ് (`src/frontend/`)**:

- ആധുനിക React/TypeScript SPA
- ഫാസ്റ്റ് ഡെവലപ്മെന്റിനായി Vite ഉപയോഗിക്കുന്നു
- ഏജന്റ് ഇടപെടലുകൾക്കുള്ള ചാറ്റ് ഇന്റർഫേസ്

**ജ്ഞാന അധഃസ്ഥിതം (`src/files/`)**:

- സാമ്പിൾ കസ്റ്റമർ, ഉൽപ്പന്ന ഡാറ്റ
- ഫയൽ അടിസ്ഥാനത്തിലുള്ള ജ്ഞാന ഉദ്ധരണമെത്തിക്കുന്നത് പ്രദർശിപ്പിക്കുന്നു
- JSON, Markdown ഫോർമാറ്റ് ഉദാഹരണങ്ങൾ
