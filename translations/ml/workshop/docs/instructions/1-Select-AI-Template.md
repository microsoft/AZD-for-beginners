## 1. ഒരു ബില്‍ഡര്‍ ഉപമ

ആധുനിക എന്റര്‍പ്രൈസ്-സജ്ജമായ AI അപ്ലിക്കേഷന്‍ _പുതിയതായി_ നിര്‍മിക്കുന്നത് ഭയങ്കരമായ കാര്യമായിരിക്കാം. ഇത് നിങ്ങളുടെ പുതിയ വീടു തനിച്ച് തൊടു മുതൽ ഇട്ടു കൂട്ടിവിടുന്നതുപോലെയാണ്. അതെ, അത് സാധ്യമാണ്! പക്ഷേ ആഗ്രഹിക്കുന്ന അവസാന ഫലം നേടാനുള്ള ഏറ്റവും ഫലപ്രദമായ മാര്‍ഗം അത് അല്ല!

അതിനുപകരം, നമ്മള്‍ സാധാരണയായി നിലവിലുള്ള ഒരു _ഡിസൈൻ ബ്ലൂപ്രിന്റ്_ ഉപയോഗിച്ച് തുടങ്ങുകയും അത് നമ്മുടെ വ്യക്തിഗത ആവശ്യങ്ങള്‍ക്ക് ദിച്ചെടുക്കാന്‍ ആർക്കിടെക്ടുമായി ജോലി ചെയ്യുകയും ചെയ്യുന്നു. ബുദ്ധിമുട്ടുള്ള അപ്ലിക്കേഷനുകള്‍ നിര്‍മ്മിക്കുമ്പോള്‍ ഇതാണ് ശരിയായ സമീപനം. ആദ്യം, നിങ്ങളുടെ പ്രശ്നപരിധിയുമായി പൊരുത്തപ്പെടുന്ന നല്ല ഒരു ഡിസൈൻ ആർക്കിടെക്ചര്‍ കണ്ടെത്തുക. ശേഷം ഒരു സൊല്യൂഷന്‍ ആർക്കിടെക്ടുമായി ചേർന്ന് അത് മാറ്റി നിങ്ങളുടെ സാഹചര്യമനുസരിച്ച് വികസിപ്പിക്കുക.

എവിടെ ഈ ഡിസൈന്‍ ബ്ലൂപ്രിന്റുകള്‍ കണ്ടെത്താം? ഇവ തായാറാക്കിയെടുക്കാനും, സ്വയം ക്രമീകരിച്ച് വിന്യസിപ്പിക്കാനും തയ്യാറുള്ള ഒരു ആർക്കിടെക്ട് എങ്ങനെ കണ്ടെത്താം? ഈ വർക്ക്ഷോപ്പിൽ, ഈ ചോദ്യങ്ങള്‍ക്ക് മറുപടി നല്‍കുകയാണ് ഞങ്ങൾ. ഇതിനായി നിങ്ങൾക്ക് പരിചയപ്പെടുത്തുന്നത് മൂന്ന് സാങ്കേതികവിദ്യകളാണ്:

1. [Azure Developer CLI](https://aka.ms/azd) - ഒരു ഓപ്പൺ സോഴ്‌സ് ഉപകരണം, ഇത് ഡെവലപ്പറെ ലോക്കൽ ഡെവലപ്മെന്റിൽ നിന്നും ക്ലൗഡിലേക്ക് (ബിൽഡ് മുതൽ ഡെപ്ലോയ്മെന്റ് വരെ) പോകാൻ സഹായിക്കുന്നു.
1. [Microsoft Foundry Templates](https://ai.azure.com/templates) - AI പരിഹാര ആർക്കിടെക്ടർ വിന്യസിക്കാൻ സാമ്പിൾ കോഡ്, ഇന്ഫ്രാസ്ട്രക്ചർ, കോൺഫിഗറേഷൻ ഫയലുകൾ ഉൾക്കൊള്ളുന്ന സ്റ്റാന്‍ഡേർഡ് ഓപ്പൺ സോഴ്സ് റിപ്പോസിറ്ററികൾ.
1. [GitHub Copilot Agent Mode](https://code.visualstudio.com/docs/copilot/chat/chat-agent-mode) - Azure അറിവിന്മേൽ ആസ്പദമാക്കി സാങ്കേതികമായി സഹായിക്കുന്ന കോഡിംഗ് ഏജന്റ്, ഇത് നാച്ചുറൽ ലക്കംജിൽ കേട്ടുകേട്ട് കോഡ്ബേസ് എങ്ങനെ സഞ്ചരിക്കാനോ മാറ്റം വരുത്താനോ സഹായിക്കും.

ഈ ഉപകരണംകൾ ഉപയോഗിച്ച്, ശരിയായ ടെംപ്ലേറ്റ് കണ്ടെത്തുക, അത് വിന്യസിച്ച് പ്രവർത്തനസാധ്യത പരിശോധിക്കുക, പിന്നെ നിങ്ങളുടെ പ്രത്യേക സാഹചര്യങ്ങൾക്ക് അനുയോജ്യമായി അതിനെ ക്രമീകരിക്കുക എന്നതിൽ കഴിവ് നേടാം. ഇനി നാം പഠിക്കാം ഇവ എങ്ങനെ പ്രവർത്തിക്കുന്നു.

## 2. Azure Developer CLI

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) (അഥവാ `azd`) ഒരു ഓപ്പൺ സോഴ്‌സ് കമാൻഡ്-ലൈനർ ഉപകരണം ആണ്, ഇത് നിങ്ങളുടെ കോഡ്-ഇരുട്ടം (code-to-cloud) യാത്രയെ വേഗത്തിലാക്കുന്നു, IDE (ഡെവലപ്മെന്റ്) ഒപ്പം CI/CD (ഡെവ്‌ഓപ്സ്) പരിസ്ഥിതികളിൽ സ്ഥിരമായി പ്രവർത്തിക്കുന്ന ഡെവലപ്പർ സൗഹൃദ കമാന്റുകളുമായി.

`azd` ഉപയോഗിക്കുമ്പോൾ, നിങ്ങളുടെ വിന്യസന യാത്ര ഇങ്ങനെ എളുപ്പം ആയിരിക്കും:

- `azd init` - നിലവിലുള്ള AZD ടെംപ്ലേറ്റ് ഉപയോഗിച്ച് ഒരു പുതിയ AI പ്രോജക്ട് ആരംഭിക്കും.
- `azd up` - ഒരു നിര്‍മാണ കാഴ്ചപ്പാടുള്ള ഇന്ഫ്രാസ്ട്രക്ചറും നിങ്ങളുടെ അപ്ലിക്കേഷനും ഒരുമിച്ച് വിന്യസിക്കും.
- `azd monitor` - നിങ്ങളുടെ വിന്യസിച്ച അപ്ലിക്കേഷനു വേണ്ടി യഥാര്‍ഥ സമയ നിരീക്ഷണം, ഡയഗ്നോസ്റ്റിക്‌സ് ലഭിക്കും.
- `azd pipeline config` - Azure-യിലേക്ക് വിന്യസനം സ്വയം സജ്ജമാക്കാൻ CI/CD പൈപ്പ്‌ലൈനുകൾ സജ്ജമാക്കും.

**🎯 | അഭ്യസനം**: <br/> GitHub Codespaces പരിസ്ഥിതിയിൽ `azd` കമാൻഡ്-ലൈൻ ഉപകരണം പരീക്ഷിക്കുക. ഇത് എന്ത് ചെയ്യുമെന്ന് കാണാനായി താഴെയുള്ള കമാൻഡ് ടൈപ്പ് ചെയ്യുക:

```bash title="" linenums="0"
azd help
```

## 3. AZD ടെമ്പ്ലേറ്റ്

`azd` ഈ കാര്യങ്ങൾ നേടാൻ, അത് ഏത് ഇന്ഫ്രാസ്ട്രക്ചർ പ്രൊവിഷൻ ചെയ്യേണ്ടതാണെന്ന്, ഏത് കോൺഫിഗറേഷൻ ക്രമീകരണങ്ങൾ നടപ്പിലാക്കണം, പിന്നീട് ഏത് അപ്ലിക്കേഷൻ വിന്യസിക്കണം എന്നൊക്കെ അറിയേണ്ടിവരും. അതിനാ [AZD ടെംപ്ലേറ്റുകൾ](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/azd-templates?tabs=csharp) ഉപയോഗിക്കുന്നത്.

AZD ടെംപ്ലേറ്റുകൾ സാമ്പിൾ കോഡ്, ഇൻഫ്രാസ്ട്രക്ചർ, കോൺഫിഗറേഷൻ ഫയലുകൾ എന്നിവ ചേർത്ത് സൃഷ്ടിച്ച(open-source) റിപ്പോസിറ്ററികൾ ആണ്, ഒരു സൊല്യൂഷൻ ആർക്കിടെക്ചർ വിന്യസിക്കാൻ ആവശ്യമുള്ളവ.

*Infrastructure-as-Code* (IaC) സമീപനം ഉപയോഗിച്ച്, ഇവ ടെംപ്ലേറ്റ് റിസോഴ്സ് നിർവ്വചനങ്ങളും കോൺഫിഗറേഷൻ ക്രമീകരണങ്ങളും (ആപ്പ് സോഴ്സ് കോഡ് പോലെ) പതിപ്പായി നിയന്ത്രിക്കാമാക്കി - അതുകൊണ്ട് ആ പ്രോജക്ട് ഉപയോക്താക്കൾക്കിടയിൽ പുനരുപയോഗയോഗ്യമായ ഒപ്പം സ്ഥിരതയുള്ള പ്രവൃത്തിസിദ്ധാന്തങ്ങൾ സൃഷ്ടിക്കുന്നു.

നിങ്ങളുടെ സാഹചര്യത്തിന് AZD ടെംപ്ലേറ്റ് സൃഷ്ടിക്കുമ്പോഴും പുനഃുപയോഗിക്കുമ്പോഴുമുള്ള ചോദ്യങ്ങൾ:

1. നിങ്ങൾ എന്ത് നിർമ്മിക്കുന്നു? → ആ സാഹചര്യത്തിന് സ്റ്റാർട്ടർ കോഡ് ഉള്ള ടെംപ്ലേറ്റ് ഉണ്ടോ?
1. നിങ്ങളുടെ സൊല്യൂഷൻ എങ്ങനെ ആർക്കിടെക്ചർ ചെയ്തിട്ടുണ്ട്? → ആവശ്യമായ റിസോഴ്സുകൾ ഉള്ള ടെംപ്ലേറ്റ് ഉണ്ടോ?
1. നിങ്ങളുടെ സൊല്യൂഷൻ എങ്ങനെ വിന്യസിക്കപ്പെടുന്നു? → കണക്കാക്കുക `azd deploy` മുൻ/പശ്ചാത്തല പ്രോസസ്സിംഗ് ഹൂക്കുകൾ!
1. എങ്ങനെ ആധികമായി മെച്ചപ്പെടുത്താം? → ഓട്ടോമേഷൻ പൈപ്പ്‌ലൈനുകളും മോണിറ്ററിങ് സംവിധാനങ്ങളും ഉൾപ്പെടുത്തുക!

**🎯 | അഭ്യസനം**: <br/> [Awesome AZD](https://azure.github.io/awesome-azd/) ഗാലറി സന്ദർശിച്ച് 250+ ടെംപ്ലേറ്റുകൾ ഫിൽറ്റർ ഉപയോഗിച്ച് പരീക്ഷിക്കുക. നിങ്ങളുടെ ആവശ്യകതകൾക്കനുസൃതമായി ഒന്ന് കണ്ടെത്താനാകുമോ നോക്കുക.

## 4. AI അപ്ലിക്കേഷൻ ടെംപ്ലേറ്റുകൾ

AI ആധാരിത അപ്ലിക്കേഷനുകൾക്ക്, Microsoft പ്രത്യേക ടെംപ്ലേറ്റുകൾ നൽകിയിട്ടുണ്ട്, ഇവ **Microsoft Foundry** ഉം **Foundry Agents** ഉം ഉൾക്കൊള്ളുന്നു. ഈ ടെംപ്ലേറ്റുകൾ ബുദ്ധിമുട്ടില്ലാതെ പ്രൊഡക്ഷന്‍-സജ്ജമായ സ്മാർട്ട് അപ്ലിക്കേഷനുകൾ നിർമ്മിക്കാൻ സഹായിക്കുന്നു.

### Microsoft Foundry & Foundry Agents Templates

താഴെ കാണുന്ന ടെംപ്ലേറ്റ് ഏതൊന്നും തിരഞ്ഞെടുക്കാം. ഓരോ ടെംപ്ലേറ്റും [Awesome AZD](https://azure.github.io/awesome-azd/) ല്‍ ലഭ്യമായതാണ്, ഒരു കമാൻഡ് ഉപയോഗിച്ച് തൽക്ഷണം ആരംഭിക്കാം.

| ടെംപ്ലേറ്റ് | വിവരണം | വിന്യാസ കമാൻഡ് |
|----------|-------------|----------------|
| **[AI Chat with RAG](https://azure.github.io/awesome-azd/?tags=ai&tags=rag)** | Microsoft Foundry ഉപയോഗിച്ച് Retrieval Augmented Generation വഹിക്കുന്ന ചാറ്റ് അപ്ലിക്കേഷൻ | `azd init -t azure-samples/azure-search-openai-demo` |

1. **ടെംപ്ലേറ്റുകൾ ബ്രൗസ് ചെയ്യുക**: [https://azure.github.io/awesome-azd/](https://azure.github.io/awesome-azd/) സന്ദർശിച്ച് `AI`, `Agents`, അല്ലെങ്കിൽ `Microsoft Foundry`-ന് ഫിൽറ്റർ ചെയ്യുക
