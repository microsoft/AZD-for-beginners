## 🆕 ഇന്നു azd യില്‍ പുതിയതെന്ത്

Azure Developer CLI പരമ്പരാഗത വെബ് ആപ്പുകളും API കളും കടന്നുപോയി. ഇന്ന്, azd ആണ് Azure ലേക്ക് **ഏതൊരു** ആപ്ലിക്കേഷനും ഡിപ്ലോയ് ചെയ്യാനുള്ള ഒറ്റ ഉപകരണം—AI-ശക്തമായ ആപ്ലിക്കേഷനും ബുദ്ധിമാനായ ഏജന്റുമടങ്ങിയവ അടങ്ങും.

നിങ്ങൾക്കായി ഇതിന്റെ അർത്ഥം:

- **AI ഏജന്റുകൾ ഇപ്പോൾ ആദ്യ-തരം azd വേൽക്കളാണ്.** നിങ്ങൾക്ക് നിലവിലുള്ള `azd init` → `azd up` പ്രവൃത്തി പ്രവാഹം ഉപയോഗിച്ച് AI ഏജന്റ് പ്രോജക്റ്റുകൾ തുടങ്ങിയ്, ഡിപ്ലോയ് ചെയ്ത് മാനേജു ചെയ്യാം.
- **Microsoft Foundry സംയോജനത്തോടെ** മോഡൽ ഡിപ്ലോയ്മെന്റ്, ഏജന്റ് ഹോസ്റ്റിംഗ്, AI സേവന കോൺഫിഗറേഷൻ azd ടെംപ്ലേറ്റ് സമ്പ്രദായത്തിൽ നേരിട്ട് പോകുന്നു.
- **പ്രധാന പ്രവൃത്തി പ്രവാഹം മാറിയിട്ടില്ല.** നിങ്ങള്‍ todo ആപ്പ്, മൈക്രോസർവീസ്, അല്ലെങ്കിൽ മൾട്ടി-ഏജന്റ് AI സൊല്യൂഷൻ ഡിപ്ലോയ് ചെയ്താലും, കമാൻഡുകൾ ഒന്നே.

നിങ്ങൾ മുമ്പ് azd ഉപയോഗിച്ചിട്ടുണ്ടെങ്കിൽ, AI പിന്തുണ ഒരു സ്വാഭാവിക വിപുലീകരണമാണെന്നും—വയറ്റിൽ പുത്തൻ ടൂൾ അല്ല, അല്ലെങ്കിൽ ഒരു വിദഗ്ധ ട്രാക്ക് അല്ല. പുതിയതായി തുടങ്ങിയാൽ, എല്ലാം ഒരേ പ്രവൃത്തി പ്രവാഹം കൊണ്ട് പഠിക്കും.

## 🚀 Azure Developer CLI (azd) എന്താണ്?

**Azure Developer CLI (azd)** ഒരു ഡെവലപ്പർ സൗഹൃദ കമാൻഡ്-ലൈൻ ഉപകരണം ആണ്, ഇത് Azure ലേക്ക് ആപ്ലിക്കേഷനുകൾ ഡിപ്ലോയ് ചെയ്യുന്നത് എളുപ്പമാക്കുന്നു. നിരവധി Azure റിസോഴ്സുകൾ കൈയോടെ സൃഷ്ടിക്കുകയും ബന്ധിപ്പിക്കുകയും ചെയ്യാനുള്ള പകരം, ഒന്നാമത്തെ കമാൻഡ് ഉപയോഗിച്ച് മുഴുവൻ ആപ്ലിക്കേഷൻ ഡിപ്ലോയ് ചെയ്യാം.

### `azd up` - അതിന്റെ മാജിക്

```bash
# ഈ ഒറ്റ കമാന്‍‍ഡ് എല്ലാം ചെയ്യുന്നു:
# ✅ എല്ലാ Azure റിസോഴ്‌സുകളും സൃഷ്ടിക്കുന്നു
# ✅ നെറ്റ്‌വർക്കിംഗ്‌യും സുരക്ഷയും ക്രമീകരിക്കുന്നു
# ✅ നിങ്ങളുടെ ആപ്ലിക്കേഷൻ കോഡ് നിര്‍മ്മിക്കുന്നു
# ✅ Azure-ലേക്ക് വിന്യസിക്കുന്നു
# ✅ പ്രവർത്തനക്ഷമമായ ഒരു URL നിങ്ങൾക്ക് നൽകുന്നു
azd up
```

**ഇതായിരുന്നു!** Azure പോർട്ടൽ ക്ലിക്ക് ചെയ്യേണ്ട, സങ്കീർണ ARM ടെംപ്ലേറ്റുകൾ പഠിക്കേണ്ട, മാനുവൽ കോൺഫിഗറേഷൻ ഒന്നും വേണ്ട—സ്വല്പം തൊഴിൽ ചെയ്യുന്ന ആപ്ലിക്കേഷൻ Azure-ൽ.

## ❓ Azure Developer CLI vs Azure CLI: വ്യത്യാസം എന്ത്?

ഇതാണ് തുടക്കക്കാർ കൂടുതലായി ചോദിക്കുന്നത്. എളുപ്പത്തിൽ മറുപടി:

| ഫീച്ചർ | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **ഉദ്ദേശ്യം** | വ്യക്തിഗത Azure റിസോഴ്സ് മാനേജ് ചെയ്യുക | പൂര്‍ണ്ണമായ ആപ്ലിക്കേഷനുകള്‍ ഡിപ്ലോയ് ചെയ്യുക |
| **ചിന്താഗതിയും** | അടിസ്ഥാന ഘടനാമൂലം | ആപ്ലിക്കേഷൻ കേന്ദ്രീകൃതം |
| **ഉദാഹരണം** | `az webapp create --name myapp...` | `azd up` |
| **പഠന വക്രം** | Azure സേവനങ്ങൾ അറിയണം | നിങ്ങളുടെ ആപ്പ് മാത്രമേ അറിയണം |
| **ശ്രേഷ്ഠം** | ഡെവ്‌ഓപ്സ്, സ്ഥിതി | ഡെവലപ്പർമാർ, പ്രോട്ടോടൈപ്പിംഗ് |

### ലളിതമായ ഉപമ

- **Azure CLI** ഒരു വീടു പണിയാൻ ആവശ്യമായ എല്ലാ ഉപകരണങ്ങളുമായി സമാനമാണ്—തണ്ട്, സംഘടിപ്പങ്ങൾ, വെളിപാടുകൾ. നിങ്ങൾക്ക് എല്ലാം പണിയാം, പക്ഷെ നിർമ്മാണം അറിയണം.
- **Azure Developer CLI** ഒരു കോൺട്രാക്ടർ നിയമിക്കാൻ തുല്യമാണ്—നിങ്ങൾ എന്ത് വേണമെന്ന് പറയുക, അവർ പണികൾ കൈകാര്യം ചെയ്യും.

### എപ്പോൾ ഏത് ഉപയോഗിക്കും

| സാഹചര്യീകൽ | ഇത് ഉപയോഗിക്കുക |
|----------|----------------|
| "എനിക്ക് എന്റെ വെബ് ആപ്പ് വേഗത്തിൽ ഡിപ്ലോയ് ചെയ്യാൻ ആഗ്രഹം" | `azd up` |
| "ഞാൻ ഒറ്റ ഒരു സ്റ്റോറേജ് അക്കൗണ്ട് സൃഷ്ടിക്കണം" | `az storage account create` |
| "ഞാൻ ഒരു പൂർണ്ണ AI ആപ്ലിക്കേഷൻ പണിയുകയാണ്" | `azd init --template azure-search-openai-demo` |
| "ഞാൻ പ്രത്യേക Azure റിസോഴ്‌സിന്റെ ഡീബഗ്ഗ് ആവശ്യം ഉണ്ട്" | `az resource show` |
| "നമുക്ക് പ്രൊഡക്ഷന്‍ റെഡി ഡിപ്ലോയ്മെന്റ് ലക്ഷ്യമിടാം" | `azd up --environment production` |

### അവ ചേർന്ന് പ്രവർത്തിക്കുന്നു!

AZD നെ Azure CLI അടിയിൽ ഉപയോഗിക്കുന്നു. നിങ്ങൾ രണ്ടും ഉപയോഗിക്കാം:
```bash
# AZD ഉപയോഗിച്ച് നിങ്ങളുടെ ആപ്പ് വിന്യസിക്കുക
azd up

# തുടർന്ന് Azure CLI ഉപയോഗിച്ച് പ്രത്യേക റിസോഴ്‌സുകൾ സൂക്ഷ്മമായി ക്രമീകരിക്കുക
az webapp config set --name myapp --always-on true
```

### പടി 2: AZD-ക്കായിAuthenticate ചെയ്യുക

```bash
# ഈ കോഴ്സിൽ നിങ്ങൾ നേരിട്ട് Azure CLI കമാൻഡുകൾ ഉപയോഗിക്കാൻ പദ്ധതിയിടുകയാണെങ്കിൽ ഐച്ഛികമാണ്
az login

# AZD വർക്ക്‌ഫ്ലോകുകൾക്ക് ആവശ്യമാണ്
azd auth login
```

നിങ്ങൾക്ക് ആവശ്യമുണ്ടെന്നുറപ്പാകാത്ത പക്ഷം, [Installation & Setup](docs/chapter-01-foundation/installation.md#authentication-setup) ൽ പൂർണ്ണ സെറ്റപ്പ് ഫ്ലോ പിന്തുടരുക.

### പടി 3: നിങ്ങളുടെ ആദ്യ ആപ്പ് ഡിപ്ലോയ് ചെയ്യുക

```bash
# ടെംപ്ലേറ്റിൽ നിന്ന് ആരംഭിക്കുക
azd init --template todo-nodejs-mongo

# Azure-ലേക്ക് വിന്യസിക്കുക (എല്ലാം സൃഷ്ടിക്കുന്നു!)
azd up
```

**🎉 അതായ്ക!** നിങ്ങളുടെ ആപ്പ് ഇപ്പോൾ Azure-ൽ ലൈവ്.

### ക്ലീൻ അപ്പ് (മറക്കരുത്!)

```bash
# Remove all resources when done experimenting
azd down --force --purge
```

ഈ ഘടനാപരമായ കോഴ്സ് പൂര്‍ത്തിയാക്കിയാൽ, നിങ്ങൾ:

## തുടങ്ങുന്നതിനു മുന്നിൽ: അക്കൗണ്ടുകൾ, ആക്‌സസ്, ആശങ്കകൾ

Chapter 1 തുടങ്ങുന്നതിന് മുൻപ് താഴെപ്പറയുന്നവ ഒരുക്കിയിട്ടുണ്ടെന്ന് ഉറപ്പാക്കുക. ഈ ഗൈഡിലെ ഇൻസ്റ്റലേഷൻ ഘട്ടങ്ങൾ ഇവയ്ക്ക് അടിസ്ഥാനമാക്കിയുള്ളതാണ്.
- **ഒരു Azure സബ്സ്ക്രിപ്ഷൻ**: ജോലി പ്രവർത്തന സാധ്യതയുള്ള സബ്സ്ക്രിപ്ഷനോ നിങ്ങളുടെ സ്വന്തം അക്കൗണ്ടിലൂടെയോ ഉപയോഗിക്കാം, അല്ലെങ്കിൽ തുടങ്ങിയതിന് ഒരു [സൗജന്യ ട്രയൽ](https://aka.ms/azurefreetrial) സൃഷ്ടിക്കാം.
- **Azure വിഭവങ്ങൾ സൃഷ്ടിക്കാനുള്ള അധികാരം**: കൂടുതൽ പല പരിശീലനങ്ങൾക്കും, ലക്ഷ്യമിടുന്ന സബ്സ്ക്രിപ്ഷനിൽ അല്ലെങ്കിൽ റിസോഴ്‌സ് ഗ്രൂപ്പിൽ കുറഞ്ഞതും **Contributor** അധികാരം ഉണ്ടായിരിക്കണം. ചില അധ്യായങ്ങൾ റിസോഴ്‌സ് ഗ്രൂപ്പ്, മാനേജുചെയ്ത ഐഡന്റിറ്റികൾ, RBAC നിയോഗങ്ങൾ സൃഷ്ടിക്കാൻ കഴിവുണ്ടെന്ന് കരുതും.
- [**GitHub അക്കൗണ്ട്**](https://github.com): റിപ്പോസിറ്ററി ഫോർക്ക് ചെയ്യാനും, സ്വന്തം മാറ്റങ്ങൾ പിന്തുടരാനും, GitHub Codespaces വർക്ക്‌ഷോപ്പിനായി ഉപയോഗിക്കാനും ഇത് സഹായിക്കും.
- **ടെംപ്ലേറ്റ് റൺടൈം മുൻകൂട്ടിയുള്ള ആവശ്യകതകൾ**: ചില ടെംപ്ലേറ്റുകൾക്ക് Node.js, Python, Java, Docker പോലുള്ള പ്രാദേശിക ഉപകരണങ്ങൾ ആവശ്യമാണ്. മിസ്സിംഗ് ഉപകരണങ്ങൾ മുന്‍കൂട്ടി കാണാൻ ആരംഭിക്കുന്നതിന് മുമ്പ് സെറ്റപ്പ് വാലിഡേറ്റർ പ്രവർത്തിപ്പിക്കുക.
- **അടിസ്ഥാന ടെർമിനൽ പരിചയം**: വിദഗ്ധർ ആകേണ്ടതില്ല, എന്നാൽ `git clone`, `azd auth login`, `azd up` പോലുള്ള കമാൻഡുകൾ നടത്താൻ സുഖമായിരിക്കണം.

> **ഇന്റർപ്രൈസ് സബ്സ്ക്രിപ്ഷനിൽ ജോലി ചെയ്യുകയാണോ?**  
> നിങ്ങളുടെ Azure പരിസരം ഒരു അഡ്മിനിസ്ട്രേറ്റർ പാരാമർശം ചെയ്യുകയാണെങ്കിൽ, നിങ്ങൾ ഉപയോഗിക്കാൻ പോകുന്ന സബ്സ്ക്രിപ്ഷനിലോ റിസോഴ്‌സ് ഗ്രൂപ്പിലോ വിഭവങ്ങൾ വിന്യസിക്കാൻ കഴിയുമെന്ന് മുൻകൂട്ടി സ്ഥിരീകരിച്ച് നോക്കി വാങ്ങുക. കഴിയാതെപോയാൽ, വർക്ക്‌ഷോപ്പ് ആരംഭിക്കുന്നതിന് മുൻപ് സാൻഡ്ബോക്‌സ് സബ്സ്ക്രിപ്ഷനോ Contributor അധികാരമോ അഭ്യർത്ഥിക്കുക.

> **Azure തുടക്കക്കാരനാണോ?**  
> നിങ്ങളുടെ സ്വന്തം Azure ട്രയൽ അല്ലെങ്കിൽ പേ-ആസ്-യൂ-ഗോ സബ്സ്ക്രിപ്ഷൻ https://aka.ms/azurefreetrial-യിൽ ആരംഭിക്കുക, ടെനന്റ് തസ്തിക അംഗീകാരങ്ങൾക്കായി കാത്തിരിയ്ക്കാതെ പരിശീലനങ്ങൾ മുഴുവൻ നന്നായി പൂർത്തിയാക്കാൻ.

# ചോപ്റ്റർ 1 പൂർത്തിയാക്കിയശേഷം, നിങ്ങൾക്ക് കഴിയും:
azd version              # ഇൻസ്റ്റാൾ ചെയ്ത പതിപ്പ് കാണിക്കുന്നു
azd init --template todo-nodejs-mongo  # പ്രോജക്ട് ആരംഭിക്കുന്നു
azd up                  # Azure-ലേക്ക് വിന്യസിക്കുന്നു
azd show                # ഫലപ്രദമായി പ്രവർത്തിക്കുന്ന ആപ്പ് URL കാണിക്കുന്നു
# ആപ്പ് ബ്രൗസറിൽ തുറന്നു പ്രവർത്തിക്കുന്നു
azd down --force --purge  # വിഭവങ്ങൾ ശുചിയാക്കുന്നു

**ചെലവ് നിരീക്ഷണം:**  
```bash

```

- **കോഡ് രൂപത്തിലുള്ള ഇൻഫ്രასტ്രക്ചർ** - ബൈസിപ് അല്ലെങ്കിൽ ടെറഫോം ഉപയോഗിച്ച് Azure വിഭവങ്ങൾ നിയന്ത്രിക്കുക  
- **ഇന്റഗ്രേറ്റഡ് വർക്‌ഫ്ലോകൾ** - ആപ്ലിക്കേഷനുകൾ തുടർച്ചയായി പ്രൊവിഷൻ ചെയ്യുകയും ഡിപ്ലോയ് ചെയ്യുകയും നിരീക്ഷിക്കുകയും ചെയ്യുക
- **ഡെവലപ്പർ സൗഹൃദം** - ഡെവലപ്പർ ഉൽപ്പാദകത്വവും അനുഭവവും മെച്ചപ്പെടുത്താൻ അനുയോജ്യമായത്

### **AZD + Microsoft Foundry: AI ഡിപ്ലോയ്മെന്റുകൾക്കായി ഉത്തമം**

**എന്തുകൊണ്ട് AI പരിഹാരങ്ങൾക്കായി AZD?** AZD AI ഡെവലപ്പർമാരുടെ മുഖ്യ വെല്ലുവിളികളെ നേരിടുന്നു:

- **AI-സജ്ജമായ ടെംപ്ലേറ്റുകൾ** - Microsoft Foundry മോഡലുകൾ, കോഗ്നിറ്റീവ് സർവീസസ്, ML വർക്ക്‌ലോഡുകൾക്കായി മുൻകൂർ ക്രമീകരിച്ച ടെംപ്ലേറ്റുകൾ
- **സുരക്ഷിത AI ഡിപ്ലോയ്മെന്റുകൾ** - AI സേവനങ്ങൾ, API കീസുകൾ, മോഡൽ എൻഡ്‌പോയിന്റുകൾക്കുള്ള ഉൾക്കൊള്ളിച്ച സുരക്ഷാ പതാക്കൾ  
- **പ്രൊഡക്ഷൻ AI പാറ്റേണുകൾ** - സ്കെയിലബിൾ, ചെലവിൽ ഭദ്രമായ AI ആപ്ലിക്കേഷൻ ഡിപ്ലോയ്മെന്റ് മികച്ച രീതികൾ
- **എന്തിന്നുമെടുത്ത AI വർക്‌ഫ്ലോകൾ** - മോഡൽ വികസനത്തിൽ നിന്ന് പ്രൊഡക്ഷൻ വരെ ശരിയായ നിരീക്ഷണത്തോടൊപ്പം
- **ചെലവ് മെച്ചപ്പെടുത്തൽ** - AI വർക്ക്‌ലോഡുകൾക്കായി സ്മാർട്ട് വിഭവൻ വകുപ്പും സ്കെയിലും
- **Microsoft Foundry ഇന്റഗ്രേഷൻ** - Microsoft Foundry മോഡൽ കാറ്റലോഗും എൻഡ്‌പോയിന്റുകളും സൗകര്യമുള്ള കണക്ഷൻ
