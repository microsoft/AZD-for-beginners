## 🆕 azd-യിൽ ഇന്ന്有什么新功能

> 📌 ഈ കോഴ്സ് **`azd 1.27.1`** (ജൂലൈ 2026) അനുസരിച്ചാണ് സാക്ഷ്യപ്പെടുത്തിയിരിക്കുന്നത്. നിങ്ങളുടെ ബിൽഡുള്ളത് പരിശോധിക്കാൻ `azd version` പ്രവർത്തിപ്പിക്കൂ, ഏറ്റവും പുതിയത് ലഭിക്കാൻ `azd upgrade` പ്രവർത്തിപ്പിക്കൂ.

Azure Developer CLI റിവാസം വെബ് ആപ്പുകൾക്കും API-കൾക്കും മീതെ വളർന്നു. ഇന്നത്തെ azd ആച്ച് **ഏതെങ്കിലും** ആപ്പ് Azure-ലേക്ക് വിന്യസിക്കാൻ ഉപയോ​ഗിക്കുന്ന ഏക ഉപകരണമാണ്—AI ശക്തിക伍മായ ആപളിക്കേഷനുകളും ബുദ്ധിമുട്ടുള്ള ഏജന്റുകളും ഉൾപ്പെടെ.

ഇത് നിങ്ങള്ക്ക് എന്ത് അർത്ഥം പറയുന്നു:

- **AI ഏജന്റുകൾ ഇനി azd-യിലെ പ്രധാന വർക്ക്ലോഡുകളായി.** നിങ്ങൾക്ക് ഇതിനകം അറിയുന്ന `azd init` → `azd up` പ്രവൃത്തപ്രവാഹം ഉപയോഗിച്ച് AI ഏജന്റ് പ്രോജക്റ്റുകൾ ആരംഭിക്കാനും വിന്യസിക്കാനും മാനേജുചെയ്യാനും കഴിയും.
- **CLI-ൽ നിന്ന് ഒരു മുഴുവൻ ഏജന്റ് ജീവിതചക്രം.** `azure.ai.agents` എക്സ്റ്റന്‌ഷൻ ഇപ്പോൾ മുഴുവൻ യാത്ര പരിഗണിക്കുന്നു—`azd ai agent init` നിർമാണിക്കാൻ, `azd ai agent invoke` പരിശോധനയ്ക്കു (ഉത്തരം സമയം കാണിച്ച്), `azd ai agent eval generate`യും `azd ai agent optimize`യും ഗുണനിലവാരം അളക്കാനും മെച്ചപ്പെടുത്താനും, `azd ai agent delete` ശുചീകരണത്തിനു.
- **കൂടുതൽ AI നിർമ്മാണ ഘടകങ്ങൾ.** പുതിയ പ്രിവ്യൂ എക്സ്റ്റന്‌ഷനുകൾ—`azure.ai.skills`യും `azure.ai.connections`യും—azd ഉപയോഗിച്ച് പുനരുപയോഗിക്കാവുന്ന ഏജന്റ് കഴിവുകളും Foundry ബന്ധങ്ങളും നേരിട്ട് മാനേജുചെയ്യാൻ അനുവദിക്കുന്നു.
- **Microsoft Foundry സംയോജനം** മാതൃക വിന്യസനം, ഏജന്റ് ഹോസ്റ്റിംഗ്, AI സേവന ക്രമീകരണം azd ടെംപ്ലേറ്റ് പ്രക്രിയയിലേക്ക് നേരിട്ട് കൊണ്ടുവരുന്നു.
- **ദൈനന്ദിനങ്ങളുടെ സുഗമവൽക്കരണം.** അടുത്തിടെ പുറത്തിറങ്ങിയ പതിപ്പുകൾ `azd init` പുനരവർത്തിക്കാനും സുരക്ഷിതമാക്കാനും, `azd auth login` പഴയ ടോക്കണുകൾ സ്വയം നീക്കാനും, `azd tool` ആദ്യ ഘട്ട ക്രമീകരണ പ്രോംപ്റ്റ് ഉൾപ്പെടുത്തിയിട്ടുണ്ട്.
- **പ്രധാന പ്രവൃത്തി രീതിയിൽ മാറ്റം ഇല്ല.** നിങ്ങൾ ടൂഡോ ആപ്പ്, മൈക്രോസർവീസ്, അല്ലെങ്കിൽ മൾട്ടി-ഏജന്റ് AI പരിഹാരം വിന്യസിക്കുമ്പോൾ, കമാൻഡുകൾ ഒരുപോലെയാണ്.

> **Aspire ഉപയോക്താക്കൾക്ക് പരിശോധന:** Microsoft ഇപ്പോൾ ഉൽപ്പന്നത്തെ ചെറുതായി **Aspire** (മുൻപ് ".NET Aspire") എന്ന് വിളിക്കുന്നു. azd-യുടെ Aspire പിന്തുണയിൽ മാറ്റം വരാത്തതാണ്—പേര് മാത്രമാണ് പുതുക്കിയത്.

നിങ്ങൾ മുൻപ് azd ഉപയോഗിച്ചിട്ടുണ്ടെങ്കിൽ, AI പിന്തുണ സ്വാഭാവികമായ ഒരു വിപുലീകരണമാണ്—പിന്നുഞ്ഞൻ ഉപകരണമോ അതുല്യ ട്രാക്കോ അല്ല. നിങ്ങൾ പുതിയതായി തുടങ്ങുന്നുവെങ്കിൽ, എല്ലാമായി പ്രവർത്തിക്കുന്ന ഒരു പ്രവൃത്തി രീതിയാണ് നിങ്ങൾ പഠിക്കുന്നത്.

## 🚀 എന്താണ് Azure Developer CLI (azd)?

**Azure Developer CLI (azd)** വെബ് ആപ്പുകൾ Azure-ലേക്ക് വിന്യസിക്കാൻ സൗകര്യപ്രദമായ ഒരു ഡെവലപ്പർ സൗഹൃദ കമാൻഡ് ലൈനായ ഉപകരണമാണ്. മറ്റു എത്രമാത്രം Azure സ്രോതസ്സ് സൃഷ്ടിക്കാമെന്നു കരുതാതെ, ഒരു കമാൻഡുമായ് പൂർണ ആപ്പുകൾ വിന്യസിക്കാം.

### `azd up` എന്ന തലശ്ശേരി

```bash
# ഈ ഒറ്റ കമാൻഡ് എല്ലാം ചെയ്യുന്നു:
# ✅ എല്ലാ Azure റിസൊഴ്സ്‌സുകളും സൃഷ്ടിക്കുന്നു
# ✅ നെറ്റ്വർക്ക് സുരക്ഷയും ക്രമീകരിക്കുന്നു
# ✅ നിങ്ങളുടെ അപ്ലിക്കേഷൻ കോഡ് നിർമ്മിക്കുന്നു
# ✅ Azure-ൽ ഡിപ്ലോയ് ചെയ്യുന്നു
# ✅ പ്രവർത്തിക്കുന്ന ഒരു URL നൽകുന്നു
azd up
```

**അതുതന്നെയാണ്!** Azure പോർട്ടലിൽ ക്ളിക്ക് ചെയ്യാൻ ആവശ്യമില്ല, സങ്കീർണമായ ARM ടെംപ്ലേറ്റുകൾ പഠിക്കേണ്ടതില്ല, മാനുവൽ കോൺഫിഗറേഷൻ വേണ്ട - വെറും പ്രവർത്തിക്കുന്ന ആപ്പുകൾ Azure-ൽ.

## ❓ Azure Developer CLI-യും Azure CLI-യും: വ്യത്യാസങ്ങൾ എന്തെല്ലാം?

ഇത് തുടങ്ങുന്നവർ asking ചെയ്തത് ഏറ്റവും സാധാരണപ്പെട്ട ചോദ്യം ആണ്. ഇതാ ലളിതമായ ഉത്തരം:

| സവിശേഷത | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **ഉദ്ദേശ്യം** | വ്യക്തിഗത Azure സ്രോതസ്സ് മാനേജ് ചെയ്യുക | പൂർണമായ ആപ്പുകൾ വിന്യസിക്കുക |
| **കരുതൽ** | ഇൻഫ്രാസ്ട്രക്ചർ-ഉദ്ദേശമായ | ആപ്പ്-ഉദ്ദേശമായ |
| **ഉദാഹരണം** | `az webapp create --name myapp...` | `azd up` |
| **പഠന വക്രം** | Azure സേവനങ്ങൾ അറിയണം | നിങ്ങളുടെ ആപ്പ് അറിയുക മതി |
| **ഉത്തമം** | DevOps, ഇൻഫ്രാസ്ട്രക്ചർ | ഡെവലപ്പർമാർ, പ്രോട്ടോടൈപ്പിംഗ് |

### ലളിതമായ സാമ്യമില്ല

- **Azure CLI** ഒരു വീടു പണിയാനുള്ള എല്ലാ ഉപകരണങ്ങളും കൈവശം വെച്ചിരിക്കുന്നത് പോലെയാണ് - ഹാമറുകൾ, കരടികൾ, നഖങ്ങൾ. ഏതെന്തു വേണമെങ്കിലും പണിയാം, പക്ഷെ കൺസ്ട്രക്ഷൻ അറിയണം.
- **Azure Developer CLI** ഒരു കോൺട്രാക്ടർ നിയോഗിച്ചതുപോലെയാണ് - നിങ്ങൾ പറയുന്നത് അവർ നിർമ്മിക്കുന്നു.

### ഉപയോ​ഗിക്കേണ്ട സമയങ്ങൾ

| സാഹചര്യം | ഇതുപയോഗിക്കുക |
|----------|----------|
| "എന്റെ വെബ് ആപ്പ് വേഗം വിന്യസിക്കാൻ" | `azd up` |
| "ഞാൻ ഒരു സ്റ്റോറേജ് അക്കൗണ്ട് മാത്രം സൃഷ്ടിക്കണം" | `az storage account create` |
| "ഞാൻ ഒരു AI ആപ്ലിക്കേഷൻ പണിയുന്നു" | `azd init --template azure-search-openai-demo` |
| "ഒരു പ്രത്യേക Azure സ്രോതസ്സ് ഡീബഗ് ചെയ്യണം" | `az resource show` |
| "നേരം കുറഞ്ഞിടത്ത് പ്രൊഡക്ഷൻ റെഡി വിന്യസനം വേണം" | `azd up --environment production` |

### അവ തമ്മിൽ ചേർന്ന് പ്രവർത്തിക്കുന്നു!

AZD അടിസ്ഥാനം Azure CLI ഉപയോഗിക്കുന്നു. രണ്ട് ഉപകരണങ്ങളും ഉപയോഗിക്കാം:
```bash
# AZD ഉപയോഗിച്ച് നിങ്ങളുടെ അപ്ലിക്കേഷൻ വിന്യസിക്കുക
azd up

# പിന്നീട് Azure CLI ഉപയോഗിച്ച് പ്രത്യേക സ്രോതസ്സുകൾ നന്നാക്കുക
az webapp config set --name myapp --always-on true
```

```bash
azd init --template get-started-with-ai-agents
```

## 🎯 3 ഘട്ടങ്ങളിൽ തുടങ്ങി തുടങ്ങുക

തുടങ്ങുന്നതിന് മുമ്പ്, നിങ്ങള് വിന്യസിക്കാൻ പോകുന്ന ടെംപ്ലേറ്റിന് അനുസൃതമായി നിങ്ങളുടെ യന്ത്രം തയ്യാറാണെന്ന് ഉറപ്പാക്കുക:

**Windows:**
```powershell
.\validate-setup.ps1
```

**macOS / Linux:**
```bash
bash ./validate-setup.sh
```

ആവശ്യമായ പരിശോധനയിൽ ഏതെങ്കിലും പരാജയം ഉണ്ടെങ്കിൽ, ആദ്യം അതു പരിഹരിച്ച് തുടർന്ന് ക്വിക്ക് സ്റ്റാർട്ടുമായി മുന്നോട്ട് പോവുക.

### ഘട്ടം 1: AZD ഇൻസ്റ്റാൾ ചെയ്യുക (2 മിനിറ്റ്)

**Windows:**
```powershell
winget install microsoft.azd
```

**macOS:**
```bash
brew tap azure/azd && brew install azd
```

**Linux:**
```bash
curl -fsSL https://aka.ms/install-azd.sh | bash
```

### ഘട്ടം 2: AZD-യ്ക്ക് സാക്ഷ്യപ്പെടുത്തുക

```bash
# ഈ കോഴ്‌സിൽ നേരിട്ട് Azure CLI കമാൻഡുകൾ ഉപയോഗിക്കാൻ പദ്ധതിയുണ്ടെങ്കിൽ ഐച്ഛികം
az login

# AZD വർക്‌ഫ്ലോകൾക്കായി ആവശ്യമാണ്
azd auth login
```

നിങ്ങൾക്ക് ഏത് ആവശ്യമാണ് എന്നു അറിയുകയില്ലെങ്കിൽ, [Installation & Setup](docs/chapter-01-foundation/installation.md#authentication-setup) ൽ സമ്പൂർണ്ണ ക്രമീകരണ പ്രവൃത്തി പിന്തുടരുക.

### ഘട്ടം 3: നിങ്ങളുടെ ആദ്യ ആപ്പ് വിന്യസിക്കുക

```bash
# ഒരു നിർദ്ദേശത്തിൽ നിന്നാണ് ആരംഭിക്കുക
azd init --template todo-nodejs-mongo

# Azure-ൽ വിന്യസിക്കുക (എല്ലാം ഉണ്ടാക്കുന്നു!)
azd up
```

**🎉 ഇതു തന്നെയാണ്!** നിങ്ങളുടെ ആപ്പ് ഇപ്പോൾ Azure-ൽ ചിട്ടയായി പ്രവർത്തിക്കുന്നു.

### ശുചിയാക്കൽ (മൂല്യമൊപ്പം മറക്കാതിരിക്കുക!)

```bash
# പരീക്ഷണം കഴിഞ്ഞാൽ എല്ലാ വിഭവങ്ങളും നീക്കം ചെയ്യുക
azd down --force --purge
```

## ആരംഭിക്കുന്നതിന് മുമ്പ്: അക്കൗണ്ട്, ആക്‌സസ്, ആദ്ധാരങ്ങൾ

ചാപ്റ്റർ 1 തുടങ്ങുന്നതിന് മുൻപ്, ചുവടെയുള്ള കാര്യങ്ങൾ ഉറപ്പാക്കുക. ഈ ഗൈഡിലെ ഇൻസ്റ്റാളേഷൻ ഘട്ടങ്ങൾ ഈ അടിസ്ഥാന കാര്യങ്ങൾ കൈകാര്യം ചെയ്യുന്നുവെന്നാണ് പൂർവ്വാനുമാനം.

- **ഒരു Azure സബ്സ്ക്രിപ്ഷൻ**: ജോലി അക്കൗണ്ടിൽ നിന്നോ അല്ലെങ്കിൽ നിങ്ങളുടെ സ്വന്തം അക്കൗണ്ടിൽ നിന്നോ ഈ സബ്സ്ക്രിപ്ഷൻ ഉപയോഗിക്കാം, അല്ലെങ്കിൽ ആരംഭിക്കാൻ [ഫ്രീ ട്രയൽ](https://aka.ms/azurefreetrial) സൃഷ്ടിക്കാം.
- **Azure റിസോഴ്‌സുകൾ സൃഷ്ടിക്കാൻ അനുവാദം**: പല വ്യായാമങ്ങൾക്കായി, ലക്ഷ്യ സബ്സ്ക്രിപ്ഷനിൽ അല്ലെങ്കിൽ റിസോഴ്‌സ് ഗ്രൂപ്പിൽ കുറഞ്ഞത് **Contributor** ആക്‌സസ് ഉണ്ടായിരിക്കണം. ചില ചാപ്റ്ററുകൾ നിങ്ങളെ റിസോഴ്‌സ് ഗ്രൂപ്പുകൾ, മാനേജ്ഡ്ഐഡന്റിറ്റികൾ, RBAC അസൈൻമെന്റുകൾ സൃഷ്ടിക്കാൻ കഴിയും എന്ന് സമീപിക്കുന്നതായി കരുതാം.
- [**GitHub അക്കൗണ്ട്**](https://github.com): ഈ അക്കൗണ്ട് റിപോസിറ്ററി ഫോർക്ക് ചെയ്യുന്നതിന്, നിങ്ങളുടെ മാറ്റങ്ങൾ ട്രാക്ക് ചെയ്യുന്നതിനും, GitHub Codespaces പ്രയോജനപ്പെടുത്തുമ്പോൾ രാജ്യസന്ദർശനത്തിനും ഉപയോഗിക്കുന്നു.

```bash
# ചോദ്യങ്ങൾ ചോദിച്ച് സ്രോതസുകളിൽ നിന്ന് ലഭ്യമായ AI-ശക്തിയുള്ള മറുപടികൾ നേടുക
# സെർച്ചിന്റെ സംയോജനം പ്രവർത്തിക്കുന്നുണ്ടോ എന്ന് സ്ഥിരീകരിക്കുക
azd monitor  # ആപ്ലിക്കേഷൻ ഇൻസൈറ്റുകൾ ടെലിമെട്രി കാണിക്കുന്നുണ്ടോ എന്ന് പരിശോധിക്കുക
azd down --force --purge
```

**📊 സമയ നിക്ഷേപം:** 1-2 മണിക്കൂർ  
**📈 പഠന ശേഷമുള്ള കരുത്ത്:** പ്രൊഡക്ഷൻ-സജ്ജമായ AI ആപ്ലിക്കേഷനുകൾ ഡിപ്ലോയ് ചെയ്ത് കോൺഫിഗർ ചെയ്യാൻ കഴിയും  
**💰 ചെലവ് ബോധവൽക്കരണം:** $80-150/മാസം ഡെവ് ചെലവുകൾ, $300-3500/മാസം പ്രൊഡക്ഷൻ ചെലവുകൾ മനസിലാക്കുക

#### 💰 AI ഡിപ്ലോയ്മെന്റുകൾക്കുള്ള ചെലവ് പരിഗണനകൾ

**വികസന പരിസ്ഥിതി (അനുമാനിച്ചത് $80-150/മാസം):**
- Microsoft Foundry മോഡലുകൾ (pay-as-you-go): ടോക്കൺ ഉപയോഗത്തെ ആശ്രയിച്ച് $0-50/മാസം
- AI സെർച്ച് (ബേസിക് ടയറു): $75/മാസം
- കണ്ടെയ്നർ ആപ്പുകൾ (കൺസംപ്ഷൻ): $0-20/മാസം
- സ്‌റ്റോറേജ് (സ്റ്റാൻഡേർഡ്): $1-5/മാസം

**പ്രൊഡക്ഷൻ പരിസ്ഥിതി (അനുമാനിച്ചത് $300-3500+/മാസം):**
- Microsoft Foundry മോഡലുകൾ (PTU സ്ഥിരമായ പ്രകടനത്തിനായി): $3,000+/മാസം അല്ലെങ്കിൽ ഉയർന്ന വാള്യം ഉള്ള pay-as-you-go
- AI സെർച്ച് (സ്റ്റാൻഡേർഡ് ടയറു): $250/മാസം
- കണ്ടെയ്നർ ആപ്പുകൾ (ഡെഡിക്കേറ്റഡ്): $50-100/മാസം
- ആപ്ലിക്കേഷൻ ഇൻസൈറ്റ്സ്: $5-50/മാസം
- സ്റ്റോറേജ് (പ്രീമിയം): $10-50/മാസം

**💡 ചെലവ് ഓപ്റ്റിമൈസേഷൻ ടിപ്സ്:**
- പഠനത്തിനായി **ഫ്രീ ടയറിലെ** Microsoft Foundry മോഡലുകൾ ഉപയോഗിക്കുക (Azure OpenAI 50,000 ടോക്കൺ/മാസം ഉൾപ്പെടുന്നു)
- സജീവമായി വികസനം നടത്താത്തപ്പോൾ `azd down` ഓടിച്ച് റിസോഴ്‌സുകൾ ഡിഎലോക്കേറ്റ് ചെയ്യുക
- കൺസംപ്ഷൻ-ബേസഡ് ബില്ലിംഗിൽ തുടക്കം കുറിച്ച്, പ്രൊഡക്ഷനായി എന്നാൽ PTU-യിലേക്ക് അപ്ഗ്രേഡ് ചെയ്യുക
- ഡിപ്ലോയ്മെന്റ് മുന്നിൽ ചെലവുകൾ കണക്കാക്കാൻ `azd provision --preview` ഉപയോഗിക്കുക
- ഓട്ടോ-സ്കെയ്ലിംഗ് സജ്ജമാക്കുക: യഥാർത്ഥ ഉപയോഗത്തിനായി മാത്രമേ പണമടക്കേണ്ടതുള്ളൂ

**ചെലവ് നിരീക്ഷണം:**
```bash
# അంచനചെയ്ത മാസത്തിൽ പ്രതീക്ഷിക്കുന്ന ചെലവുകൾ പരിശോധിക്കുക
azd provision --preview

# ആസ്യൂർ പോർട്ടലിൽ യഥാർത്ഥ ചെലവുകൾ നിരീക്ഷിക്കുക
az consumption budget list --resource-group <your-rg>
```

### **AZD + Microsoft Foundry: AI വിന്യാസങ്ങൾക്ക് അനുയോജ്യം**

- **സുരക്ഷിത AI വിന്യാസങ്ങൾ** - AI സേവനങ്ങൾ, API കീകൾ, മോഡൽ എക്സ്പോർട്ട് പോയിന്റുകൾക്കുള്ള സുരക്ഷാ സാമ്പ്രദായങ്ങൾ ഉൾപ്പെടുത്തിയിരിക്കുന്നു  
- **പ്രൊഡക്ഷൻ AI സാമ്പ്രദായങ്ങൾ** - സ്കെയ്ലുബിളും ചെലവു-പ്രഭാവമുള്ളവുമായ AI ആപ്ലിക്കേഷൻ വിന്യാസത്തിന് മികച്ച ആചാരങ്ങൾ
- **ആരംഭം മുതൽ പ്രൊഡക്ഷൻ വരെ AI വർക്ക്‌ഫ്ളോ** - മോഡൽ വികസനത്തിൽ നിന്നുള്ള പ്രൊഡക്ഷൻ വിന്യാസവും ശരിയായ മേൽനോട്ടവും
- **ചെലവ് മെച്ചപ്പെടുത്തൽ** - AI പ്രവർത്തനത്തിനായി സ്മാർട്ടായ വിഭവ വിന്യസനവും സ്കെയ്ലിംഗും
- **Microsoft Foundry ഇന്റഗ്രേഷൻ** - Microsoft Foundry മോഡൽ കാറ്റലോഗിനും എക്സ്പോർട്ടിനും മോടിച്ച ബന്ധം

## 🔧 വേഗത്തിലുള്ള പ്രശ്നപരിഹാര ഗൈഡ്

**ആരംഭകർക്കുള്ള പൊതുവായ പ്രശ്നങ്ങളും ഉടൻ പരിഹാരങ്ങളും:**

<details>
<summary><strong>❌ "azd: കമാൻഡ് കണ്ടെത്തിയില്ല"</strong></summary>

```bash
# ആദ്യം AZD ഇൻസ്റ്റാൾ ചെയ്യുക
# വിൻഡോസ് (പവർഷെൽ):
winget install microsoft.azd

# മാക്‌ഓഎസ്:
brew tap azure/azd && brew install azd

# ലിനക്സ്സ്:
curl -fsSL https://aka.ms/install-azd.sh | bash

# ഇൻസ്റ്റാളേഷൻ സ്ഥിരീകരിക്കുക
azd version
```
</details>

<details>
<summary><strong>❌ "സബ്സ്ക്രിപ്ഷൻ കണ്ടെത്താനായില്ല" അല്ലെങ്കിൽ "സബ്സ്ക്രിപ്ഷൻ സെറ്റ് ചെയ്തിട്ടില്ല"</strong></summary>

```bash
# ലഭ്യമായ സബ്സ്ക്രിപ്ഷനുകൾ പട്ടികപ്പെടുത്തുക
az account list --output table

# ഡിഫാൾട്ട് സബ്സ്ക്രിപ്ഷൻ സജ്ജമാക്കുക
az account set --subscription "<subscription-id-or-name>"

# AZD പരിസ്ഥിതിമായായി സജ്ജമാക്കുക
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# സ്ഥിരീകരിക്കുക
az account show
```
</details>

<details>
<summary><strong>❌ "InsufficientQuota" അല്ലെങ്കിൽ "Quota exceeded"</strong></summary>

```bash
# വ്യത്യസ്തമായ Azure പ്രദേശം പരീക്ഷിക്കുക
azd env set AZURE_LOCATION "westus2"
azd up

# അല്ലെങ്കിൽ വികസനത്തിൽ ചെറുതായുള്ള SKUs ഉപയോഗിക്കുക
# infra/main.parameters.json എന്ന ഫയൽ തിരുത്തുക:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>

<details>
<summary><strong>❌ "azd up" എ半മായി പരാജയപ്പെടുന്നു</strong></summary>

```bash
# ഓപ്ഷൻ 1: ശുദ്ധീകരിച്ച് വീണ്ടും ശ്രമിക്കുക
azd down --force --purge
azd up

# ഓപ്ഷൻ 2: ലളിതമായി അടിസ്ഥാനസൗകര്യം ശരിയാക്കുക
azd provision

# ഓപ്ഷൻ 3: വിശദമായ അവസ്ഥ പരിശോധിക്കുക
azd show

# ഓപ്ഷൻ 4: Azure Monitor-ലുള്ള ലോഗുകൾ പരിശോധിക്കുക
azd monitor --logs
```
</details>

<details>
<summary><strong>❌ "ആശയവിനിമയം പരാജയപ്പെട്ടു" അല്ലെങ്കിൽ "ടോക്കൺ കാലഹരണപ്പെട്ടു"</strong></summary>

```bash
# AZD വേണ്ടി വീണ്ടും ആധികാരികരിക്കുക
azd auth logout
azd auth login

# ഐച്ഛികം: നിങ്ങൾ az കമാൻഡുകൾ പ്രവർത്തിപ്പിച്ചുകൊണ്ടിരിക്കുന്നുവെങ്കിൽ Azure CLIയും പുതുക്കുക
az logout
az login

# ആധികാരികത സ്ഥിരീകരിക്കുക
az account show
```
</details>

<details>
<summary><strong>❌ "സ്രോതസ്സ് ইতിമധ്യമാണ" അല്ലെങ്കിൽ "പേര്മാറ്റം സംഘർഷങ്ങൾ"</strong></summary>

```bash
# AZD അപരിചിതമായ പേരുകൾ സൃഷ്ടിക്കുന്നു, പക്ഷേ സംഘർഷങ്ങൾ ഉണ്ടെങ്കിൽ:
azd down --force --purge

# തുടർന്ന് പുതിയത് അന്തരീക്ഷത്തോടെ വീണ്ടും ശ്രമിക്കുക
azd env new dev-v2
azd up
```
</details>

<details>
<summary><strong>❌ ടെംപ്‌ലെറ്റ് ഡിപ്ലോയ്മെന്റ് വളരെ വൈകുകയാണ്</strong></summary>

**സാധാരണ കാത്തിരിച്ചു സമയങ്ങൾ:**
- ലളിതമായ വെബ് ആപ്പ്: 5-10 മിനിട്ടുകൾ
- ഡാറ്റാബേസ് ഒപ്പം ആപ്പ്: 10-15 മിനിട്ടുകൾ
- AI ആപ്ലിക്കേഷനുകൾ: 15-25 മിനിട്ടുകൾ (OpenAI പ്രൊവിഷനിംഗ് മന്ദമാണ്)

```bash
# പുരോഗതി പരിശോധിക്കുക
azd show

# 30 മിനിറ്റിന് മുകളിൽ തടസം ഉണ്ടായാൽ, Azure പോർട്ടൽ പരിശോധിക്കുക:
azd monitor --overview
# പരാജയപ്പെട്ട വിന്യസ്സങ്ങൾ അന്വേഷിക്കുക
```
</details>

<details>
<summary><strong>❌ "അനുമതി നിരസിച്ചു" അല്ലെങ്കിൽ "നിഷിദ്ധം"</strong></summary>

```bash
# നിങ്ങളുടെ അസ്യൂര്‍ റോള്‍ പരിശോധിക്കുക
az role assignment list --assignee $(az account show --query user.name -o tsv)

# കുറഞ്ഞത് "Contributor" റോള്‍ വേണ്ടതാണ്
# നിങ്ങളുടെ അസ്യൂര്‍ അഡ്മിനോട് അനുവദിക്കുക ആവശ്യപ്പെടുക:
# - Contributor (സ്രോതസ്സുകള്‍ക്കായി)
# - User Access Administrator (റോള്‍ നിയുക്തികള്‍ക്കായി)
```
</details>

<details>
<summary><strong>❌ ഡിപ്ലോയ്ഡ് ആപ്പ് URL കണ്ടെത്താനായില്ല</strong></summary>

```bash
# എല്ലാ സർവീസ് എൻഡ്‌പോയിന്റുകളും കാണിക്കുക
azd show

# അല്ലെങ്കിൽ Azure പോർട്ടൽ തുറക്കുക
azd monitor

# പ്രത്യേക സർവീസ് പരിശോധിക്കുക
azd env get-values
# *_URL വേരിയബിളുകൾ തിരയുക
```
</details>
