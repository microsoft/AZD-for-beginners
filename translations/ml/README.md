## 🆕 azd ഇന്നത്തെ പുതിയതെല്ലാം

> 📌 ഈ കോഴ്‌സ് **`azd 1.25.6`** (ജൂൺ 2026) അനുസരിച്ച് സാധൂകരിച്ചിരിക്കുന്നു. നിങ്ങളുടെ ബിൽഡ് പരിശോധിക്കാൻ `azd version` ഓടിക്കുക, ഏറ്റവും പുതിയത് ലഭിക്കാൻ `azd upgrade` ഉപയോഗിക്കുക.

Azure Developer CLI പരമ്പരാഗത വെബ് ആപ്പ്‌സും API-സും അതിർത്തി മറികടന്നിരിക്കുന്നു. ഇന്ന്, azd ഏത് അപ്ലിക്കേഷനെയും Azure-ൽ വിന്യസിക്കാൻ ഏകോപിത ഉപകരണമായിരിക്കുന്നു — AI-ചാലിത അപ്ലിക്കേഷനുകളും ബുദ്ധിമുട്ടുള്ള ഏജന്റുമാരും ഉൾപ്പെടുന്നു.

നിങ്ങൾക്ക് ഇതിനർത്ഥമുള്ളത്:

- **AI ഏജന്റുമാർ ഇപ്പോൾ പ്രഥമ ഘട്ട azd വേലകളാണ്.** നിങ്ങൾ അറിയുന്ന `azd init` → `azd up` പ്രവാഹം ഉപയോഗിച്ച് AI ഏജന്റ് പ്രോജക്റ്റുകൾ ആരംഭിക്കുകയും വിന്യാസം ചെയ്യുകയും നിയന്ത്രിക്കുകയും ചെയ്യാം.
- **CLI-യിൽ നിന്നുള്ള ഒരു സമ്പൂർണ ഏജന്റ് ജീവിതചക്രം.** `azure.ai.agents` വ്യാപ്തി ഇനി മുഴുവൻ യാത്രയും ഉൾക്കൊള്ളുന്നു — `azd ai agent init` സ്‌കാഫോൾഡ് ചെയ്യാൻ, `azd ai agent invoke` പരീക്ഷിക്കാൻ (പ്രതികരണ സമയ ഔട്ട്‌പുട്ടോടെ), `azd ai agent eval generate` ഉം `azd ai agent optimize` ഉം ഗുണന്മായത നിശ്ചയിക്കുകയും മെച്ചപ്പെടുത്തുകയും ചെയ്യാൻ, `azd ai agent delete` നാനാക്കി.
- **കൂടുതൽ AI നിർമ്മാണഘടകങ്ങൾ.** പുതിയ പ്രിവ്യൂ വിപുലീകരണങ്ങൾ—`azure.ai.skills` ഉം `azure.ai.connections` ഉം—മாற்றി ഉപയോഗിക്കാവുന്ന ഏജന്റ് കഴിവുകളും ഫൗണ്ടറി കണക്ഷനുകളും നേരിട്ട് azd ന്ടയിലേക്ക് നിയന്ത്രിക്കാം.
- **Microsoft Foundry ഇന്റഗ്രേഷൻ** മോഡൽ വിന്യാസം, ഏജന്റ് ഹോസ്റ്റിങ്, AI സേവന ക്രമീകരണം azd ടെംപ്ലേറ്റ് ഇക്കോസിസ്റ്റത്തിലേക്ക് നേരിട്ട് എത്തിക്കുന്നു.
- **ദിവസേനയുടെ അടിസ്ഥാന കാര്യങ്ങൾ കൂടുതൽ സുലഭമായി.** അടുത്തിടെ റിലീസുകൾ `azd init` മുൻപേ ഉപയോഗിച്ചപ്പോൾ സുരക്ഷിതമാക്കി (പുനഃസഞ്ചയിക്കുക സൗകര്യപ്രദം), `azd auth login` പഴയ ടോക്കണുകൾ സ്വയം നീക്കംചെയ്യുന്നു, സൗഹൃദപരമായ `azd tool` ആദ്യ ഓട്ടോമേറ്റഡ് സജ്ജീകരണ പ്രോമ്പ്റ്റ് ചേർത്തു.
- **മൂല പ്രക്രിയ പരിവർത്തനമില്ല.** വിസ്താരപ്പെട്ട ആപ്പ്, മൈക്രോസർവീസ് അല്ലെങ്കിൽ ബഹുഏജന്റ് AI പരിഹാരം വിന്യാസം ചെയ്യുമ്പോൾ കമാൻഡുകൾ ഒരുപോലെയാണ്.

> **Aspire ഉപയോക്താക്കൾക്ക് കുറിപ്പ്:** Microsoft ഇപ്പോൾ ഉൽപ്പന്നത്തെ **Aspire** (മുന്ന് ".NET Aspire") എന്നറിയിക്കുന്നു. azd ലെ Aspire പിന്തുണ മാറ്റമില്ല—പേര് മാത്രം പുതുക്കി.

നിങ്ങൾക്ക് azd മുമ്പ് ഉപയോഗിച്ചിട്ടുണ്ടെങ്കിൽ, AI പിന്തുണ സ്വാഭാവികമായ ഉയർച്ചയാണ് — വേർതിരിച്ചിട്ടുള്ള ഉപകരണം അല്ല അല്ലെങ്കിൽ മുന്നേറ്റ പാത. ഇളഞ്ഞു തുടങ്ങുന്നവർക്ക്, എല്ലാംക്കുമായി ഒരേ പ്രവാഹം പഠിക്കും.

## 🚀 Azure Developer CLI (azd) എളുപ്പത്തിൽ

**Azure Developer CLI (azd)** ഒരു ഡെവലപ്പർ സുഹൃദ് കമാൻഡ് ലൈൻ ഉപകരണമാണ്, ഇത് Azure-യിൽ അപ്ലിക്കേഷനുകൾ വിന്യസിക്കുന്നത് സരളമാക്കുന്നു. നിരവധി Azure വിഭവങ്ങൾ കൈമാറി സൃഷ്ടിക്കുകയും ബന്ധിപ്പിക്കുകയും ചെയ്യുന്ന പാതിവിധി ഒഴിവാക്കി, ഒരു കമാൻഡ് ഉപയോഗിച്ച് മുഴുവൻ ആപ്പുകൾ വിന്യസിക്കാം.

### `azd up` അത്ഭുതം

```bash
# ഈ ഒറ്റ കമാൻഡ് എല്ലാം ചെയ്യുന്നു:
# ✅ എല്ലാ Azure റിസോർസുകളും സൃഷ്ടിക്കുന്നു
# ✅ നെറ്റ്‌വർക്കിംഗ് மற்றும் സുരക്ഷ ക്രമീകരിക്കുന്നു
# ✅ നിങ്ങളുടെ ആപ്ലിക്കേഷൻ കോഡ് നിർമ്മിക്കുന്നു
# ✅ Azure-ലേക്ക് ഡിപ്ലോയ് ചെയ്യുന്നു
# ✅ നിങ്ങൾക്ക് പ്രവർത്തനക്ഷമമായ ഒരു URL നൽകുന്നു
azd up
```

**അതുകൂടാ!** Azure പോർട്ടൽ ഉപയോഗിക്കേണ്ടതില്ല, സങ്കീർണ ARM ടെംപ്ലേറ്റുകൾ പഠിക്കേണ്ടതില്ല, ഹസ്തപരിഷ്കരണം വേണ്ടതില്ല — Azure-ൽ പ്രവർത്തിക്കുന്ന ആപ്പുകൾ فقط.

## ❓ Azure Developer CLI vs Azure CLI: എന്താണ് വ്യത്യാസം?

പുതിയവർക്ക് ഏറ്റവും സാധാരണമായി ചോദിക്കുന്ന ചോദ്യം. തരത്തിലുള്ള ഉത്തരം:

| ഫീച്ചർ | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **ഉദ്ദേശ്യം** | വ്യക്തിഗത Azure വിഭവങ്ങൾ നിയന്ത്രിക്കുക | സമ്പൂർണ ആപ്പുകൾ വിന്യസിക്കുക |
| **വിസ്താരം** | മറ്റു ഘടനകൾ പരിപാലനം | അപ്ലിക്കേഷൻ കേന്ദ്രീകൃതം |
| **ഉദാഹരണം** | `az webapp create --name myapp...` | `azd up` |
| **പഠനവലുപ്പം** | Azure സേവനങ്ങൾ അറിയണം | നിങ്ങളുടെ ആപ്പ് മാത്രം അറിയണം |
| **ആവശ്യക്കാർക്ക് ഉത്തമം** | DevOps, ഇൻഫ്രാസ്ട്രക്ചർ | ഡെവലപ്പർമാർ, പ്രോട്ടോടൈപ്പിംഗ് |

### ലളിതമായ ഉപമ

- **Azure CLI** ഹാമർ, കൊണ്ട, നീലുകൾ ഉള്ള ഗൃഹനിർമ്മാണ ഉപകരണങ്ങൾ ഉള്ളവനെ പോലെയാണ്. നിങ്ങൾക്ക് എന്തെന്തും നിർമ്മിക്കാം, പക്ഷെ നിർമ്മാണം അറിയണം.
- **Azure Developer CLI** ഒരു കരാരുകാരനെ നിയമിക്കുന്നതുപോലെ ആണ് — നിങ്ങൾ ഒരു വിവരമറക്കുന്നു, അവർ ബിൽഡിങും പരിചരിക്കുമെന്ന്.

### എപ്പോഴാണ് ഓരോത് ഉപയോഗിക്കേണ്ടത്

| സ്ഥിതിവിവരം | ഇതിന്റെ ഉപയോഗം |
|----------|----------|
| "എന്റെ വെബ് ആപ്പ് വേഗം വിന്യാസം ചെയ്യണം" | `azd up` |
| "ഫലമായി ഒരു സ്റ്റോറേജ് അക്കൗണ്ട് സൃഷ്ടിക്കണം" | `az storage account create` |
| "പൂർണ്ണ AI അപ്ലിക്കേഷൻ നിർമ്മിക്കുന്നു" | `azd init --template azure-search-openai-demo` |
| "ഒരു Azure വിഭവം ഡീബഗ് ചെയ്യണം" | `az resource show` |
| "നിമിഷങ്ങളിൽ പ്രൊഡക്ഷൻ-തയ്യാറായ വിന്യാസം വേണം" | `azd up --environment production` |

### ഇവ കൂടെ പ്രവർത്തിക്കുന്നു!

AZD Azure CLI നെ അടിവരയാക്കിയിട്ടുണ്ട്. നിങ്ങൾ ഇരുവിധവും ഉപയോഗിക്കാം:
```bash
# AZD ഉപയോഗിച്ച് നിങ്ങളുടെ ആപ്പ് ഡിപ്ലോയ് ചെയ്യുക
azd up

# ശേഷം Azure CLI ഉപയോഗിച്ച് പ്രത്യേക റിസോഴ്സുകൾ സൂക്ഷ്മമായി ക്രമീകരിക്കുക
az webapp config set --name myapp --always-on true
```

## 🎯 3 ഘട്ടങ്ങളിൽ ആരംഭിക്കൽ

നിങ്ങൾ ആരംഭിക്കുന്നതിന് മുമ്പ്, വിന്യാസം ചെയ്യാൻ ഉദ്ദേശിക്കുന്ന ടെംപ്ലേറ്റിനുള്ള യന്ത്രം തയ്യാറായിട്ടുണ്ടെന്നും ഉറപ്പു വരുത്തുക:

**Windows:**
```powershell
.\validate-setup.ps1
```

**macOS / Linux:**
```bash
bash ./validate-setup.sh
```

ഏതുടെയെങ്കിലും പരിശോധന പരാജയപ്പെട്ടാൽ, ആദ്യം അത് പരിഹരിച്ച് തുടർന്ന് തുടർന്നു.

### ചുവടും 1: AZD ഇൻസ്റ്റാൾ ചെയ്യുക (2 മിനിറ്റ്)

**Windows:**
```powershell
winget install microsoft.azd
```

**macOS:**
```bash
brew tap azure/azd && brew install azd
```

**Linux:**
```bash
curl -fsSL https://aka.ms/install-azd.sh | bash
```

### ചുവടും 2: AZD-ക്കായി പ്രാമാണീകരണം

```bash
# ഈ കോഴ്സിൽ നിങ്ങൾ നേരിട്ട് Azure CLI കമാൻഡുകൾ ഉപയോഗിക്കാൻ പദ്ധതിയുണ്ടെങ്കിൽ ഇത് ഐച്ഛികമാണ്.
az login

# AZD പ്രവാഹങ്ങൾക്ക് ഇത് ആവശ്യമാണ്.
azd auth login
```

നിങ്ങൾക്ക് ഏതാണ് വേണ്ടത് ധരിപ്പിക്കാത്താൽ, മുഴുവൻ ക്രമീകരണ പ്രവാഹം [Installation & Setup](docs/chapter-01-foundation/installation.md#authentication-setup) ൽ പിന്തുടരുക.

### ചുവടും 3: നിങ്ങളുടെ ആദ്യ ആപ്പ് വിന്യാസം ചെയ്യുക

```bash
# ടെംപ്ലേറ്റിൽ നിന്ന് ആരംഭിക്കുക
azd init --template todo-nodejs-mongo

# Azure-ലേക്ക് വിന്യസിക്കുക (എല്ലാം സൃഷ്ടിക്കുന്നു!)
azd up
```

**🎉 ഇതാണ്!** നിങ്ങളുടെ ആപ്പ് ഇപ്പോൾ Azure-ൽ ലൈവിലാണ്.

### ശുചിത്വം (മടക്കാതെ!)

```bash
# Remove all resources when done experimenting
azd down --force --purge
```

## ആരംഭിക്കുമ്ബോൾ: അക്കൗണ്ടുകൾ, ആക്‌സസ്, ധാരണകൾ

ചാപ്റ്റർ 1 ആരംഭിക്കുന്നതിന് മുന്നേ, താഴെയുള്ള കാര്യങ്ങൾ ഉറപ്പുവരുത്തുക. ഗൈഡിലെ ഇൻസ്റ്റലേഷൻ ഘട്ടങ്ങൾ ഈ അടിസ്ഥാനങ്ങൾ കൈകാര്യമാക്കാറാണെന്ന് തഴച്ചുക.

- **ഒരു Azure സബ്സ്ക്രിപ്ഷൻ**: ജോലി സബ്സ്ക്രിപ്ഷൻ അല്ലെങ്കിൽ നിങ്ങളുടെ തന്നെ അക്കൗണ്ട് ഉപയോഗിക്കാമെന്നും, തുടക്കംക്കായി [https://aka.ms/azurefreetrial](https://aka.ms/azurefreetrial) വഴി സൗജന്യ ട്രയൽ സൃഷ്ടിക്കാമെന്നും.
- **Azure റിസോഴ്‌സുകൾ സൃഷ്ടിക്കാൻ അനുമതി**: നിങ്ങളുടെ ടാർഗറ്റ് സബ്സ്ക്രിപ്ഷനിൽ അല്ലെങ്കിൽ റിസോഴ്‌സ് ഗ്രൂപ്പിൽ കുറഞ്ഞത് **Contributor** ആക്‌സസ് ഉണ്ടായിരിക്കണം. ചില ചാപ്റ്ററുകളിൽ റിസോഴ്‌സ് ഗ്രൂപ്പുകൾ, മാനേജഡ് ഐഡന്റിറ്റീസ്, RBAC അസൈൻമെന്റുകൾ സൃഷ്ടിക്കാൻ കഴിയും എന്നാണ് തന്റെ ധാരണ.
- [**GitHub അക്കൗണ്ട്**](https://github.com): റെപ്പോ ഫോർക്ക് ചെയ്യാനും, നിങ്ങളുടെ മാറ്റങ്ങൾ ട്രാക്ക് ചെയ്യാനും, കൂടാതെ GitHub Codespaces ഉപയോഗിച്ച് വർക്ഷോപ്പിൽ പ്രവർത്തിക്കാൻ ഇത് സഹായിക്കും.
- **ടെംപ്ലേറ്റ് റൺടൈം മുൻ‌‌വഴിപ്പാട്ട്**: ചില ടെംപ്ലേറ്റുകൾക്ക് Node.js, Python, Java, Docker പോലുള്ള ലോക്കൽ ടൂളുകൾ ആവശ്യമാണ്. കുറവുള്ള ടൂളുകൾ നേരത്തേ കണ്ടെത്താൻ സെറ്റ് അപ് വാലിഡേറ്റർ പ്രവർത്തിപ്പിക്കുക.
- **ടേംിനൽ അടിസ്ഥാന പരിചയപരിധി**: ഗിറ്റ് ക്ലോൺ, azd auth login, azd up പോലുള്ള കമാൻഡുകൾ നടത്താൻ സുഖവുമുള്ളവനാകുക, വിദഗ്ധനാകേണ്ടതില്ല.

> **ഉയർന്ന സ്ഥാപന സബ്സ്ക്രിപ്ഷനിൽ പ്രവർത്തിക്കുന്നുവോ?**

### **AZD + Microsoft Foundry: എ.ഐ വിന്യാസങ്ങൾക്ക് അനുയോജ്യമാണ്**

**എന്തുകൊണ്ട് AZD എ.ഐ പരിഹാരങ്ങൾക്ക്?** AZD എ.ഐ ഡെവലപ്പർമാർ നേരിടുന്ന പ്രധാന വെല്ലുപ്പുകൾ പരിഹരിക്കുന്നു:

- **എ.ഐ-റേഡി ടെംപ്ലേറ്റുകൾ** - Microsoft Foundry മോഡലുകൾ, Azure AI സേവനങ്ങൾ, ML വർക്ക്‌ലോഡുകൾക്കായി സജ്ജീകരിച്ച ടെംപ്ലേറ്റുകൾ  
- **സുരക്ഷിത എ.ഐ ഡിപ്ലോയ്മെന്റുകൾ** - എ.ഐ സേവനങ്ങൾ, API കീകൾ, മോഡൽ എന്റ്പോയിന്റുകൾക്കായി ഇൻ ബിൽറ്റ് സുരക്ഷാ പാറ്റേണുകൾ  
- **പ്രൊഡക്ഷൻ എ.ഐ പാറ്റേണുകൾ** - സ്കെയിലബിൾ, ചെലവ്-പ്രവണമായ എ.ഐ ആപ്ലിക്കേഷൻ ഡിപ്ലോയ്മെന്റുകൾ  
- **എൻഡ് ടു എಂಡ್ എ.ഐ വർക്ക്‌ഫ്ലോകൾ** - മോഡൽ വികസനത്തിലേര്‍ന്നുപോലും പ്രൊഡക്ഷൻ ഡിപ്ലോയ്മെന്റിലും ഗൈഡൻസ് ഉൾപ്പെടെ  
- **ചെലവ് ഒപ്റ്റിമൈസേഷൻ** - എ.ഐ വർക്ക്‌ലോഡുകൾക്കുള്ള സ്മാർട്ട് റിസോഴ്സ് നിയോഗവും സ്കെയിലിംഗ് തന്ത്രങ്ങളും  
- **Microsoft Foundry ഇന്റഗ്രേഷൻ** - Microsoft Foundry മോഡൽ കാറ്റലോഗിലേക്കും എന്റ്പോയിന്റുകളിലേക്കും പരിപാലനം

## 🔧 ദ്രുത പരിഹാര നിർദേശങ്ങൾ  

**ആരംഭക്കാർ നേരിടുന്ന സാധാരണ പ്രശ്‌നങ്ങളും ഉടൻ പരിഹാരങ്ങളും:**  

<details>  
<summary><strong>❌ "azd: command not found"</strong></summary>  

```bash
# ആദ്യം AZD ഇൻസ്റ്റാൾ ചെയ്യുക
# Windows (PowerShell):
winget install microsoft.azd

# macOS:
brew tap azure/azd && brew install azd

# Linux:
curl -fsSL https://aka.ms/install-azd.sh | bash

# ഇൻസ്റ്റലേഷൻ സ്ഥിരീകരിക്കുക
azd version
```
</details>  

<details>  
<summary><strong>❌ "No subscription found" or "Subscription not set"</strong></summary>  

```bash
# ലഭ്യമായ സബ്സ്ക്രിപ്ഷനുകൾ പട്ടികവദ്‌വാക്കുക
az account list --output table

# ഡിഫോൾട്ട് സബ്സ്ക്രിപ്ഷൻ സെറ്റ് ചെയ്യുക
az account set --subscription "<subscription-id-or-name>"

# AZD പരിസ്ഥിതിക്ക് സെറ്റ് ചെയ്യുക
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# പരിശോധിക്കുക
az account show
```
</details>  

<details>  
<summary><strong>❌ "InsufficientQuota" or "Quota exceeded"</strong></summary>  

```bash
# വ്യത്യസ്തമായ Azure പ്രദേശം പരീക്ഷിക്കൂ
azd env set AZURE_LOCATION "westus2"
azd up

# അല്ലെങ്കിൽ വികസനത്തിൽ ചെറുതായ SKUs ഉപയോഗിക്കുക
# infra/main.parameters.json തിരുത്തുക:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>  

<details>  
<summary><strong>❌ "azd up" fails halfway through</strong></summary>  

```bash
# ഓപ്ഷൻ 1: ശുദ്ധീകരിച്ച് വീണ്ടും ശ്രമിക്കുക
azd down --force --purge
azd up

# ഓപ്ഷൻ 2: ഇൻഫ്രാസ്ട്രക്ചർ മാത്രം ശരിയാക്കുക
azd provision

# ഓപ്ഷൻ 3: വിശദമായ സ്ഥലം പരിശോധിക്കുക
azd show

# ഓപ്ഷൻ 4: ആസ്യൂർ മോണിറ്ററിൽ ലോഗുകൾ പരിശോധിക്കുക
azd monitor --logs
```
</details>  

<details>  
<summary><strong>❌ "Authentication failed" or "Token expired"</strong></summary>  

```bash
# AZD വേണ്ടി വീണ്ടും ഓതന്റിക്കേറ്റ് ചെയ്യുക
azd auth logout
azd auth login

# ঐচ্ছികം: നിങ്ങളുടെ az കമാൻഡുകൾ പ്രവർത്തിപ്പിക്കുന്നുവെങ്കിൽ Azure CLI യും പുതുക്കുക
az logout
az login

# ഓതന്റിക്കേഷൻ പരിശോധിക്കുക
az account show
```
</details>  

<details>  
<summary><strong>❌ "Resource already exists" or naming conflicts</strong></summary>  

```bash
# AZD വ്യത്യസ്തമായ നാമങ്ങൾ സൃഷ്ടിക്കുന്നു, പക്ഷേ ശ്രദ്ധാഭ്രംശം ഉണ്ടാകുമെങ്കിൽ:
azd down --force --purge

# തുടർന്ന് പുതിയ അന്തരീക്ഷത്തോടെ വീണ്ടും ശ്രമിക്കുക
azd env new dev-v2
azd up
```
</details>  

<details>  
<summary><strong>❌ Template deployment taking too long</strong></summary>  

**സാധാരണ കാത്തിരിപ്പു സമയങ്ങൾ:**  
- ലളിതമായ വെബ് ആപ്പ്: 5-10 മിനിറ്റ്  
- ഡാറ്റാബേസ് ഉള്ള ആപ്പ്: 10-15 മിനിറ്റ്  
- AI അപ്ലിക്കേഷനുകൾ: 15-25 മിനിറ്റ് (OpenAI പ്രൊവിഷനിംഗ് സ്ലോ ആണ്)  

```bash
# പുരോഗതി പരിശോധിക്കുക
azd show

# 30 മിനിറ്റുകൾക്കധികം കഴിയുമ്പോഴും തടസ്സം വന്നാൽ, Azure പോർട്ടൽ പരിശോധിക്കുക:
azd monitor --overview
# പരാജയപ്പെട്ട വിന്യസനങ്ങൾ നോക്കുക
```
</details>  

<details>  
<summary><strong>❌ "Permission denied" or "Forbidden"</strong></summary>  

```bash
# നിങ്ങളുടെ Azure റോളിനെ പരിശോധിക്കുക
az role assignment list --assignee $(az account show --query user.name -o tsv)

# ഏറ്റവും കുറഞ്ഞ രീതിയിൽ "Contributor" റോളത് ആവശ്യമാണ്
# നിങ്ങളുടെ Azure അഡ്മിനിനോട് ആവശ്യപ്പെട്ടുകൊൾളുക:
# - Contributor (സമ്പന്ത്രങ്ങൾക്ക്)
# - User Access Administrator (റോൾ നിയുക്തികൾക്കായി)
```
</details>  

<details>  
<summary><strong>❌ Can't find deployed application URL</strong></summary>  

```bash
# എല്ലാ സേവന എൻഡ്‌പോയിന്റുകളും കാണിക്കുക
azd show

# അല്ലെങ്കിൽ അസ്യൂർ പോർട്ടൽ തുറക്കുക
azd monitor

# പ്രത്യേക സേവനം പരിശോധിക്കുക
azd env get-values
```
