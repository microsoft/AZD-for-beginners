## 🆕 ഇന്ന് azd-ല有什么更新

Azure Developer CLI പരമ്പരാഗത വെബ് ആപ്പുകളും APIs-ഉം വിട്ട് വളരെ വലിയ കാര്യങ്ങൾ ചെയ്യാൻ വളർന്നിട്ടുണ്ട്. ഇന്ന്, azd ഏതേ ഡിപ്പ്ലോയ് ചെയ്യുന്നതിനും ഏകോപിതമായ ഉപകരണം ആണ് - AI-ശക്തി പ്രയോഗങ്ങളും ബുദ്ധിമുട്ടുള്ള ഏജന്റുമാർക്കും ഉൾപ്പെടെ.

ഇത് നിങ്ങൾക്ക് എന്തുതന്നെയാണ് സമർത്ഥ്യമാക്കുന്നത്:

- **AI ഏജന്റുമാർ ഇപ്പോൾ azd-യുടെ മുഖ്യ ജോലികൾ ആകുന്നു.** നിങ്ങൾക്ക് `azd init` → `azd up` പ്രവൃത്തിതുറവിനെ ഉപയോഗിച്ച് AI ഏജന്റ് പ്രൊജക്ടുകൾ ആരംഭിക്കുകയും, ഉദ്ധരിക്കുകയും, കൈകാര്യം ചെയ്യുകയും ചെയ്യാം.
- **Microsoft Foundry സംയോജനം** മോഡൽ ഡിപ്പ്ലോയ്‌മെന്റ്, ഏജന്റ് ഹോസ്റ്റിംഗ്, AI സേവന കോൺഫിഗറേഷൻ azd ടെംപ്ലേറ്റ് οικοσύστημαയ്ക്ക് നേരിട്ട് കൊണ്ടുവരുന്നു.
- **പ്രധാന പ്രവൃത്തി ഒഴുക്കിൽ മാറ്റമില്ല.** ഒരു ടൊഡോ ആപ്പ്, മൈക്രോസർവീസ് അല്ലെങ്കിൽ മൾട്ടി ഏജന്റ് AI പരിഹാരമെങ്കിലും, കമാൻഡുകൾ ഒരേതന്നെയാണ്.

നിങ്ങൾ azd മുമ്പേ ഉപയോഗിച്ചിട്ടില്ലെങ്കിൽ, AI പിന്തുണ സ്വാഭാവികമായ ഒരു എക്സ്റ്റെൻഷനാണ് - വേറെ ഉപകരണം അല്ലെങ്കിൽ ഒരു പ്രഗത്ഭമായ ട്രാക്ക് അല്ല. നിങ്ങൾ പുതുതായി ആരംഭിക്കുന്നുവെങ്കിൽ, എല്ലാം കവര്‍ക്കുന്ന ഏക പ്രവൃത്തി രീതിയാണ് നിങ്ങൾ പഠിക്കുന്നത്.

## 🚀 Azure Developer CLI (azd) എന്നത് എന്താണ്?

**Azure Developer CLI (azd)** ഡവലപ്പർമാർക്ക് സൗഹൃദപരമായ കമാൻഡ്-ലൈൻ ഉപകരണം ആണ്, ഇത് Azure-ലേക്ക് അപ്ലിക്കേഷനുകൾ വിന്യസിക്കാൻ എളുപ്പമാക്കുന്നു. പല ഡോ‌സന്റുമടങ്ങിയ Azure വിഭവങ്ങൾ യാന്ത്രിതമായി സൃഷ്ടിക്കുകയും ബന്ധിപ്പിക്കുകയും ചെയ്യുന്നതിനുപകരം, നിങ്ങൾക്ക് ഒരൊറ്റ കമാൻഡിൽ മുഴുവൻ അപ്ലിക്കേഷനങ്ങളും വിന്യസിക്കാം.

### `azd up` എന്ന മാജിക്

```bash
# ഈ ഒറ്റ കമാൻഡ് എല്ലാം ചെയ്യുന്നു:
# ✅ എല്ലാ അസ്യൂർ റിസോഴ്‌സുകളും സൃഷ്ടിക്കുന്നു
# ✅ നെറ്റ്വർക്കിംഗ്‌യും സുരക്ഷയും ക്രമീകരിക്കുന്നു
# ✅ നിങ്ങളുടെ ആപ്ലിക്കേഷൻ കോഡ് നിർമ്മിക്കുന്നു
# ✅ അസ്യൂറിലേക്ക് ഡിപ്ലോയ് ചെയ്യുന്നു
# ✅ നിങ്ങൾക്ക് പ്രവർത്തനക്ഷമമായ ഒരു URL നൽകുന്നു
azd up
```

**ഇതായിരിക്കും!** Azure Portal ക്ലിക്ക് ചെയ്യേണ്ടതില്ല, ബുദ്ധിമുട്ടുള്ള ARM ടെംപ്ലേറ്റുകൾ പഠിക്കേണ്ടതില്ല, മാൻവൽ കോൺഫിഗറേഷൻ വേണ്ട - വെറും Azure-ൽ പ്രവർത്തിക്കുന്ന അപ്ലിക്കേഷനുകൾ.

## ❓ Azure Developer CLIേയും Azure CLI-യും തമ്മിലുള്ള വ്യത്യാസം?

ഈ ചോദ്യം തുടങ്ങുന്നവർക്ക് ഏറ്റവും പൊതുതാണ്. ലളിതമായ ഉത്തരം ഇതാ:

| പ്രത്യേകത | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **ഉദ്ദേശ്യം** | വ്യക്തിഗത Azure വിഭവങ്ങൾ മാനേജ് ചെയ്യുക | പൂര്‍ണമായ അപ്ലിക്കേഷനുകൾ വിന്യസിക്കുക |
| **ചിന്താഗതിയിത്** | ഇൻഫ്രാസ്ട്രക്ചർ കേന്ദ്രീകരിച്ചത് | അപ്ലിക്കേഷൻ കേന്ദ്രീകരിച്ചത് |
| **ഉദാഹണം** | `az webapp create --name myapp...` | `azd up` |
| **പഠനവേദി** | Azure സർവീസുകൾ അറിയണം | നിങ്ങളുടെ അപ്ലിക്കേഷൻ മാത്രമേ അറിയൂ |
| **ഏറ്റവും അനുയോജ്യം** | DevOps, ഇൻഫ്രാസ്ട്രക്ചർ | ഡവലപ്പർമാർ, പ്രോട്ടോടൈപ്പിംഗ് |

### ലളിതമായ ഉദാഹരണം

- **Azure CLI** ഹാമറുകൾ, കുരുത്തിമുറികൾ, നഖങ്ങൾ ഉള്ള ഒരു വീട്ടു നിർമ്മാണ സമാഗ്രി പോലെയാണ്. നിങ്ങൾക്ക് ഏതെങ്കിലും തരം വീട് നിർമ്മിക്കാം, പക്ഷേ നിർമ്മാണം അറിയണം.
- **Azure Developer CLI** ഒരു കൺട്രാക്ടറെ നിയമിക്കുന്നതു പോലെയാണ് - നിങ്ങൾ എന്ത് വേണമെന്ന് പറയുന്നു, അവൻ നിർമ്മാണം കൈകാര്യം ചെയ്യുന്നു.

### ഏത് അവസരത്തിൽ ഏത് ഉപയോഗിക്കും

| സ്ഥിതി | ഇത് ഉപയോഗിക്കുക |
|----------|----------|
| "എന്റെ വെബ് ആപ്പ് വേഗത്തിൽ വിന്യസിക്കണം" | `azd up` |
| "ഞാൻ ഒരു സ്റ്റോറേജ് അക്കൗണ്ട് മാത്രം സൃഷ്ടിക്കണം" | `az storage account create` |
| "ഞാൻ ഒരു പൂർണ്ണ AI അപ്ലിക്കേഷൻ നിർമ്മിക്കുന്നു" | `azd init --template azure-search-openai-demo` |
| "ഒരു പ്രത്യേക Azure വിഭവം ഡീബഗ് ചെയ്യണം" | `az resource show` |
| "നിമിഷങ്ങളിൽ പ്രൊഡക്ഷൻ റെഡി ഡിപ്പ്ലോയ്‌മെന്റ് വേണം" | `azd up --environment production` |

### അവർ ചേർന്ന് പ്രവർത്തിക്കുന്നു!

AZD പിന്നിലെത്തുമ്പോൾ Azure CLI ഉപയോഗിക്കുന്നു. നിങ്ങൾ ഇരുവരും ഉപയോഗിക്കാം:
```bash
# AZD ഉപയോഗിച്ച് നിങ്ങളുടെ ആപ്പ് ഡിപ്ലോയ് ചെയ്യുക
azd up

# തുടർന്ന് Azure CLI ഉപയോഗിച്ച് നിശ്ചിത റിസോഴ്‌സുകൾ സൂക്ഷ്മമായി ക്രമീകരിക്കുക
az webapp config set --name myapp --always-on true
```

## 🔧 വേഗത്തിൽ പ്രശ്നപരിഹാര ഗൈഡ്

**പ്രായോഗിക പ്രശ്നങ്ങളും ഉടൻ പരിഹാരവും:**

<details>
<summary><strong>❌ "azd: കമണ്ട് കണ്ടെത്താനായില്ല"</strong></summary>

```bash
# ആദ്യം AZD ഇൻസ്റ്റാൾ ചെയ്യുക
# വിൻഡോസ് (പവർഷെൽ):
winget install microsoft.azd

# മാക്‌ഓഎസ്:
brew tap azure/azd && brew install azd

# ലിനക്‌സ്:
curl -fsSL https://aka.ms/install-azd.sh | bash

# ഇൻസ്റ്റലേഷൻ പരിശോധന നടത്തുക
azd version
```
</details>

<details>
<summary><strong>❌ "സബ്സ്ക്രിപ്ഷൻ കണ്ടെത്താനായില്ല" അല്ലെങ്കിൽ "സബ്സ്ക്രിപ്ഷൻ സെറ്റ് ചെയ്തിട്ടില്ല"</strong></summary>

```bash
# ലഭ്യമായ സബ്‌സ്‌ക്രിപ്ഷനുകൾ പട്ടികയാക്കുക
az account list --output table

# ഡീഫാൾട്ട്ഡ് സബ്‌സ്‌ക്രിപ്ഷൻ സജ്ജമാക്കുക
az account set --subscription "<subscription-id-or-name>"

# AZD പരിസ്ഥിതിക്ക് സജ്ജമാക്കുക
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# ശരിയെന്ന് ഉറപ്പാക്കുക
az account show
```
</details>

<details>
<summary><strong>❌ "InsufficientQuota" അല്ലെങ്കിൽ "കോറ്റ വെട്ടിപ്പോയി"</strong></summary>

```bash
# വ്യത്യസ്തമായ അസ്യൂർ റീജിയൻ പരീക്ഷിക്കുക
azd env set AZURE_LOCATION "westus2"
azd up

# വികസനത്തിൽ ചെറിയ SKUs ഉപയോഗിക്കുക
# infra/main.parameters.json എഡിറ്റ് ചെയ്യുക:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>

<details>
<summary><strong>❌ "azd up" മദ്ധ്യത്തിലുണ്ടായ പിഴവ്</strong></summary>

```bash
# ഓപ്ഷൻ 1: ശുദ്ധമാക്കിയ്‌ച്ച് വീണ്ടും ശ്രമിക്കുക
azd down --force --purge
azd up

# ഓപ്ഷൻ 2: ഇൻഫ്രാസ്ട്രക്ചർ മാത്രം പരിഹരിക്കുക
azd provision

# ഓപ്ഷന് 3: വിശദമായ നില പരിശോധിക്കുക
azd show

# ഓപ്ഷൻ 4: ആസ്യൂർ മോണിറ്ററിൽ ലോഗുകൾ പരിശോധിക്കുക
azd monitor --logs
```
</details>

<details>
<summary><strong>❌ "പ്രാമാണീകരണം പരാജയപ്പെട്ടു" അല്ലെങ്കിൽ "ടോക്കൺ കാലഹരണപ്പെട്ടു"</strong></summary>

```bash
# വീണ്ടും പ്രവേശനം സ്ഥിരീകരിക്കുക
az logout
az login

azd auth logout
azd auth login

# പ്രാമാണീകരണം സ്ഥിരീകരിക്കുക
az account show
```
</details>

<details>
<summary><strong>❌ "റിസോഴ്‌സ് ഇതിനകം നിലവിലുണ്ട്" അല്ലെങ്കിൽ നാമകരണം-related പ്രശ്നങ്ങൾ</strong></summary>

```bash
# AZD ഏകാന്തമായ പേരുകൾ സൃഷ്ടിക്കുന്നു, പക്ഷേ സങ്കടങ്ങൾ ഉണ്ടെങ്കിൽ:
azd down --force --purge

# പിന്നീട് പുതിയ പരിസരത്തോടെ വീണ്ടും ശ്രമിക്കുക
azd env new dev-v2
azd up
```
</details>

<details>
<summary><strong>❌ ടെംപ്ലേറ്റ് ഡെപ്ലോയ്മെന്റ് വളരെ സമയം എടുക്കുന്നു</strong></summary>

**സാധാരണ കാത്തിരിപ്പുകാലം:**
- ലളിതമായ വെബ് ആപ്പ്: 5-10 മിനിറ്റ്
- ഡാറ്റാബേസ് ഉൾപ്പെടുന്ന ആപ്പ്: 10-15 മിനിറ്റ്
- AI അപ്ലിക്കേഷനുകൾ: 15-25 മിനിറ്റ് (OpenAI പ്രൊവിഷനിംഗ് സ്ലോ ആണ്)

```bash
# പുരോഗതി പരിശോധിക്കുക
azd show

# 30 മിനിറ്റിൽ കൂടുതൽ തടഞ്ഞാൽ, Azure പോർട്ടൽ പരിശോധിക്കുക:
azd monitor
# പരാജയപ്പെട്ട വിന്യസനങ്ങൾ തിരയുക
```
</details>

<details>
<summary><strong>❌ "അനുമതി നിഷേധിച്ചു" അല്ലെങ്കിൽ "നിഷിദ്ധം"</strong></summary>

```bash
# നിങ്ങളുടെ അസ്യൂർ റോളുകൾ പരിശോധിക്കുക
az role assignment list --assignee $(az account show --query user.name -o tsv)

# നിങ്ങൾക്ക് കുറഞ്ഞത് "Contributor" റോളുണ്ടാകണം
# നിങ്ങളുടെ അസ്യൂർ അഡ്മിനിന്റെ അഭ്യർത്ഥിക്കൂ നൽകാൻ:
# - Contributor (സ്രോതസുകൾക്കായി)
# - User Access Administrator (റോൾ നിയുക്തികൾക്കായി)
```
</details>

<details>
<summary><strong>❌ നെറ്റ്-ഡെപ്ലോയ്ഡ് ആപ്പ് URL കണ്ടെത്താനായില്ല</strong></summary>

```bash
# എല്ലാ സേവന എൻഡ്പോയിന്റുകളും കാണിക്കുക
azd show

# അല്ലെങ്കിൽ Azure പോർട്ടൽ തുറക്കുക
azd monitor

# പ്രത്യേക സേവനം പരിശോധിക്കുക
azd env get-values
```

ഓരോ അധ്യായവും വഴി നിങ്ങളുടെ പഠന പുരോഗതി ട്രാക്ക് ചെയ്യുക:

- [ ] **അധ്യായം 1**: അടിസ്ഥാനവും വേഗതയുള്ള ആരംഭവും ✅
- [ ] **അധ്യായം 2**: AI-ഫസ്റ്റ് ഡെവലപ്പുമെന്റ് ✅  
- [ ] **അധ്യായം 3**: കോൺഫിഗറേഷൻ & പ്രാമാണീകരണം ✅
- [ ] **അധ്യായം 4**: ഇൻഫ്രാസ്ട്രക്ചർ ആസ് കോഡ് & ഡെപ്ലോയ്മെന്റ് ✅
- [ ] **അധ്യായം 5**: മൾട്ടി-ഏജന്റ് AI പരിഹാരങ്ങൾ ✅
- [ ] **അധ്യായം 6**: പ്രീ-ഡെപ്ലോയ്മെന്റ് ചെക്ക് & പ്ലാനിങ് ✅
- [ ] **അധ്യായം 7**: പ്രശ്നപരിഹാരവും ഡീബഗിംഗും ✅
- [ ] **അധ്യായം 8**: പ്രൊഡക്ഷൻ & എന്റർപ്രൈസ് മാതൃകകൾ ✅

### പഠന സ്ഥിരീകരണം
ഓരോ അധ്യായവും പൂർത്തിയാക്കിയ ശേഷം നിങ്ങളുടെ അറിവ് പരിശോധിക്കുക:
1. **പ്രായോഗിക വ്യായാമം**: അധ്യായത്തിലെ ഹാൻഡ്‌സൺ ഡെപ്ലോയ് മെന്റ് പൂർത്തിയാക്കുക
2. **അറിവ് പരിശോധന**: നിങ്ങളുടെ അധ്യായത്തിനുള്ള FAQ വിഭാഗം പരിശോധിക്കുക
3. **സമൂഹ ചർച്ച**: Azure Discord-ലിൽ നിങ്ങളുടെ അനുഭവം പങ്കുവെക്കുക
4. **അടുത്ത അധ്യായം**: അടുത്ത സങ്കീർണ്ണത നിലവാരത്തിലേക്കു പോയി പഠനം തുടരുക

### കോഴ്‌സ് പൂർത്തീകരണ ലാഭങ്ങൾ
എല്ലാ അധ്യായങ്ങളും പൂർത്തിയാക്കിയ ശേഷം നിങ്ങൾക്ക്:
- **ഉത്പാദനാനുഭവം**: യഥാർത്ഥ AI അപ്ലിക്കേഷനുകൾ Azure-ൽ ഡെപ്ലോയ്ചെയ്തിട്ടുണ്ട്
- **പ്രൊഫഷണൽ കഴിവുകൾ**: എന്റർപ്രൈസ്-തയ്യാറായ ഡെപ്ലോയ്മെന്റ് കഴിവുകൾ  
- **സമൂഹ അംഗീകാരം**: ആക്ടിവ് Azure ഡെവലപ്പർ കമ്മ്യൂണിറ്റി അംഗം
- **കാര്യർ പുരോഗതി**: അഭിരുചിയുള്ള AZD & AI ഡെപ്ലോയ്മെന്റ് വിദഗ്ധത

- പുതിയ AI ടെംപ്ലേറ്റുകളുടെ പ്രാഥമിക പ്രിവ്യൂസ് ആക്സസ് ചെയ്യാനും
- AI ഡെപ്ലോയ്മെന്റ് മികച്ച അഭ്യാസങ്ങളിൽ സഹായിക്കാനും
- ഭാവിയിലെ AI + AZD ഫീച്ചർ വികസനത്തെ സ്വാധീനിക്കാനും
