# AZD For Beginners에 기여하기

AZD For Beginners 과정에 기여해 주셔서 감사합니다! 이 문서는 기여자를 위한 지침과 정보를 제공합니다.

## 기여 방법

### 이슈 보고하기

버그, 오류를 발견했거나 개선에 대한 제안이 있는 경우:

1. **기존 이슈 확인**: 중복을 피하기 위해 [기존 이슈](https://github.com/microsoft/azd-for-beginners/issues)를 검색하세요
2. **새 이슈 만들기**: 적절한 이슈 템플릿을 사용하세요
3. **세부 정보 제공**: 재현 단계, 예상 동작 및 해당되는 경우 스크린샷을 포함하세요

### 변경 제안하기

1. **레포지토리 포크**: 프로젝트를 자신의 포크로 만드세요
2. **브랜치 생성**: 변경 사항을 위한 기능 브랜치를 만드세요
3. **변경 작업**: 당사의 코딩 및 문서화 기준을 따르세요
4. **변경 사항 테스트**: 모든 링크가 작동하고 콘텐츠가 올바르게 렌더링되는지 확인하세요
5. **풀 리퀘스트 제출**: 변경 사항에 대한 명확한 설명을 제공하세요

### 기여 유형

다음과 같은 유형의 기여를 환영합니다:

#### 콘텐츠 개선
- 오타, 문법 및 형식 문제 수정
- 기존 문서의 명확성 개선
- 오래된 정보 업데이트
- 누락된 설명 또는 맥락 추가

#### 새로운 예제
- 실제 배포 시나리오 추가
- 일반적인 사용 사례를 위한 새로운 AZD 템플릿 생성
- 추가 코드 샘플 및 연습 문제 제공

#### 번역 지원
- 기존 번역 유지 보수 지원
- 번역 품질 향상
- 새로운 콘텐츠에 대한 번역 추가

#### 버그 보고
- 끊어진 링크 또는 잘못된 코드 예제 보고
- 문서 불일치 식별
- 접근성 문제 신고

## 기여 지침

### 문서 기준

- 명확하고 초보자 친화적인 언어 사용
- 학습 목표 및 결과 포함
- 예상 출력이 포함된 단계별 지침 제공
- 관련 콘텐츠로의 네비게이션 링크 추가
- 일관된 형식 및 마크다운 스타일 사용

### 코드 예제

- 모든 코드 예제가 테스트되어 작동하는지 확인
- 핵심 개념을 설명하는 주석 포함
- Azure 및 AZD 권장 사례 준수
- 해당되는 경우 예상 출력 제공

### 커밋 메시지

명확하고 설명적인 커밋 메시지를 사용하세요:
- `docs: 설치 가이드의 오타 수정`
- `feat: 새 컨테이너 앱 예제 추가`
- `fix: 2장에서 끊어진 링크 수정`
