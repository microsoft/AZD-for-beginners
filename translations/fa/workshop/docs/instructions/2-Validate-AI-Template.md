## ۱. مقدمه

[رابط خط فرمان توسعه‌دهنده آزور](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) یا `azd` یک ابزار خط فرمان متن‌باز است که جریان کاری توسعه‌دهنده را هنگام ساخت و استقرار برنامه‌ها به آزور ساده‌سازی می‌کند.

[قالب‌های AZD](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) مخازن استانداردی هستند که شامل کد نمونه برنامه، دارایی‌های زیرساخت به‌صورت کد و فایل‌های پیکربندی `azd` برای یک معماری راه‌حل منسجم می‌باشند. فراهم‌سازی زیرساخت به سادگی فرمان `azd provision` است در حالی که استفاده از `azd up` به شما اجازه می‌دهد زیرساخت را فراهم کنید **و** برنامه خود را در یک مرحله مستقر کنید!

در نتیجه، شروع سریع فرآیند توسعه برنامه شما می‌تواند به سادگی یافتن قالب شروع‌کننده مناسب AZD باشد که نزدیک‌ترین نیازهای برنامه و زیرساخت شما را دارد - سپس مخزن را برای تطبیق با نیازهای سناریوی خود سفارشی کنید.

قبل از شروع، بیایید مطمئن شویم که رابط خط فرمان توسعه‌دهنده آزور را نصب کرده‌اید.

۱. ترمینال VS Code را باز کنید و این فرمان را تایپ کنید:

      ```bash title="" linenums="0"
      azd version
      ```

۱. باید چیزی شبیه به این ببینید!

      ```bash title="" linenums="0"
      azd version 1.27.1 (commit <current-build>)
      ```

**حال آماده‌اید تا یک قالب را با azd انتخاب و مستقر کنید**

۱. به آدرس [https://ai.azure.com/templates](https://ai.azure.com/templates) مراجعه کنید

## ۳. فعال‌سازی قالب

بیایید تلاش کنیم این قالب را مستقر کنیم و مطمئن شویم که معتبر است. دستورالعمل‌های بخش [شروع کار](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) را دنبال خواهیم کرد.

۱. یک محیط کاری برای مخزن قالب انتخاب کنید:

      - **GitHub Codespaces**: روی [این لینک](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) کلیک کنید و تأیید کنید `Create codespace`
      - **کلون محلی یا کانتینر توسعه**: مخزن `Azure-Samples/get-started-with-ai-agents` را کلون کنید و در VS Code باز کنید

۱. تا آماده شدن ترمینال VS Code صبر کنید، سپس فرمان زیر را تایپ کنید:

   ```bash title="" linenums="0"
   azd up
   ```

مراحل گردش کاری که این فرمان آغاز می‌کند را کامل کنید:

۱. از شما خواسته می‌شود وارد آزور شوید - دستورالعمل‌ها را برای احراز هویت دنبال کنید
۱. یک نام محیط منحصربه‌فرد برای خود وارد کنید - به عنوان مثال، من `nitya-mshack-azd` را استفاده کردم
۱. این کار پوشه‌ای به نام `.azure/` ایجاد می‌کند - شما یک زیرشاخه با نام محیط خود خواهید دید
۱. از شما خواسته می‌شود نام یک اشتراک را انتخاب کنید - پیش‌فرض را انتخاب کنید
۱. از شما محل را درخواست می‌کند - از `East US 2` استفاده کنید

اکنون منتظر بمانید تا فراهم‌سازی کامل شود. **این کار ۱۰-۱۵ دقیقه طول می‌کشد**

۱. پس از پایان، کنسول شما پیام موفقیتی مانند این نشان خواهد داد:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. در پرتال آزور شما اکنون یک گروه منابع تهیه شده با نام همان محیط دارید:

      ![Infra](../../../../../translated_images/fa/02-provisioned-infra.46c706b14f56e0bf.webp)

۱. **اکنون آماده‌اید زیرساخت و برنامه مستقر شده را اعتبارسنجی کنید**.

## ۷. قابلیت مشاهده ساخته شده

قابلیت مشاهده به معنای ابزارسازی برنامه شما برای تولید داده‌هایی است که می‌توان از آن‌ها برای درک، رفع اشکال و بهینه‌سازی عملیات برنامه استفاده کرد. برای آشنایی با این موضوع:

۱. دکمه `View Run Info` را کلیک کنید - باید این نمای را ببینید. این نمونه‌ای از [ردیابی عامل](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) در عمل است. _همچنین می‌توانید این نما را با کلیک بر Thread Logs در منوی سطح بالا مشاهده کنید_.

   - با مراحل اجرا و ابزارهای استفاده‌شده توسط عامل آشنا شوید
   - کل تعداد توکن‌ها (در مقابل توکن‌های خروجی) برای پاسخ را درک کنید
   - میزان تأخیر و محل صرف زمان در اجرای عملیات را بفهمید

      ![Agent](../../../../../translated_images/fa/10-view-run-info.b20ebd75fef6a1cc.webp)

۱. تب `Metadata` را کلیک کنید تا ویژگی‌های اضافی اجرای عملیات را که ممکن است برای عیب‌یابی بعدی مفید باشد ببینید.

      ![Agent](../../../../../translated_images/fa/11-view-run-info-metadata.7966986122c7c2df.webp)


۱. تب `Evaluations` را کلیک کنید تا ارزیابی‌های خودکار انجام‌شده بر پاسخ عامل را ببینید. این‌ها شامل ارزیابی‌های ایمنی (مثلاً خودآسیب‌رسانی) و ارزیابی‌های خاص عامل (مثلاً حل قصد، پیروی از وظیفه) هستند.

      ![Agent](../../../../../translated_images/fa/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

۱. در نهایت، تب `Monitoring` را در منوی کناری کلیک کنید.

      - تب `Resource usage` را در صفحه نمایش داده شده انتخاب کنید - و معیارها را مشاهده کنید.
      - مصرف منابع برنامه از نظر هزینه (توکن‌ها) و بار (درخواست‌ها) را پیگیری کنید.
      - تأخیر برنامه تا دریافت اولین بایت (پردازش ورودی) و آخرین بایت (خروجی) را پیگیری کنید.

      ![Agent](../../../../../translated_images/fa/13-monitoring-resources.5148015f7311807f.webp)

## ۸. متغیرهای محیطی

تاکنون، ما از استقرار در مرورگر عبور کرده‌ایم و تأیید کرده‌ایم که زیرساخت فراهم شده و برنامه عملیاتی است. اما برای کار با _کدمحور_ برنامه، نیاز است محیط توسعه محلی خود را با متغیرهای مرتبط پیکربندی کنیم تا با این منابع کار کنیم. استفاده از `azd` این کار را آسان می‌کند.

۱. رابط خط فرمان توسعه‌دهنده آزور [از متغیرهای محیطی استفاده می‌کند](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) تا تنظیمات پیکربندی برای استقرار برنامه‌ها را ذخیره و مدیریت کند.

۱. متغیرهای محیطی در `.azure/<env-name>/.env` ذخیره می‌شوند - این آن‌ها را به محیط `env-name` که در هنگام استقرار استفاده شده مقید می‌کند و به شما کمک می‌کند محیط‌ها را بین اهداف استقرار مختلف در یک مخزن جدا کنید.

۱. متغیرهای محیطی به طور خودکار توسط فرمان `azd` هنگام اجرای فرمان خاصی (مثلاً `azd up`) بارگذاری می‌شوند. توجه کنید که `azd` به طور خودکار متغیرهای محیطی سطح سیستم‌عامل (مثلاً تنظیم شده در پوسته) را نمی‌خواند - به جای آن از `azd set env` و `azd get env` برای انتقال اطلاعات درون اسکریپت‌ها استفاده کنید.


بیایید چند فرمان را امتحان کنیم:

۱. همه متغیرهای محیطی تنظیم شده برای `azd` در این محیط را دریافت کنید:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      چیزی شبیه به این خواهید دید:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

۱. یک مقدار خاص را دریافت کنید - مثلاً می‌خواهم بدانم آیا مقدار `AZURE_AI_AGENT_MODEL_NAME` تنظیم شده است

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      چیزی شبیه به این خواهید دید - مقدار پیش‌فرض نبود!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

۱. یک متغیر محیطی جدید برای `azd` تنظیم کنید. اینجا نام مدل عامل را به‌روزرسانی می‌کنیم. _توجه: هرگونه تغییر فوراً در فایل `.azure/<env-name>/.env` منعکس خواهد شد.

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      حالا باید مقدار را پیدا کنیم که تنظیم شده است:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

۱. توجه کنید که برخی منابع پایدار هستند (مثلاً استقرار مدل‌ها) و نیاز به کاری بیش از `azd up` برای اجبار به استقرار مجدد دارند. بیایید ابتدا استقرار اولیه را پاک کنیم و سپس با متغیرهای محیطی تغییر یافته مجدداً استقرار دهیم.

۱. **تازه‌سازی** اگر قبلاً زیرساختی با استفاده از قالب azd مستقر کرده‌اید - می‌توانید وضعیت متغیرهای محیطی محلی خود را بر اساس وضعیت فعلی استقرار آزور خود با این فرمان _تازه‌سازی_ کنید:

      ```bash title="" linenums="0"
      azd env refresh
      ```


      این یک روش قدرتمند برای _همگام‌سازی_ متغیرهای محیطی در دو یا چند محیط توسعه محلی (مثلاً تیمی با چند توسعه‌دهنده) است - که اجازه می‌دهد زیرساخت استقرار یافته به عنوان حقیقت مبنا برای وضعیت متغیرهای محیطی عمل کند. اعضای تیم به سادگی متغیرها را _تازه‌سازی_ می‌کنند تا دوباره همگام شوند.
