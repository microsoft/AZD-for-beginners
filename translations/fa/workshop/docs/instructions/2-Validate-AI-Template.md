## 1. مقدمه

[ابزار خط فرمان توسعه‌دهنده Azure](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) یا `azd` یک ابزار خط فرمان متن‌باز است که روند کاری توسعه‌دهنده را هنگام ساخت و استقرار برنامه‌ها به Azure ساده می‌کند. 

[قالب‌های AZD](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) مخازنی استاندارد شده هستند که شامل کد نمونه برنامه، دارایی‌های _زیرساخت-به‌عنوان-کد_ و فایل‌های پیکربندی `azd` برای یک معماری راه‌حل یکپارچه می‌باشند. فراهم‌سازی زیرساخت به اندازهٔ یک فرمان `azd provision` ساده می‌شود - در حالی که استفاده از `azd up` به شما امکان می‌دهد تا زیرساخت را فراهم کنید **و** برنامهٔ خود را به‌صورت یکجا مستقر کنید!

در نتیجه، شروع سریع فرآیند توسعهٔ برنامهٔ شما می‌تواند به سادگی یافتن قالب شروع _AZD Starter_ مناسب که بیشترین نزدیکی را به نیازهای برنامه و زیرساخت شما دارد باشد - سپس مخزن را برای تطبیق با نیازهای سناریوی شما سفارشی کنید.

قبل از شروع، بیایید مطمئن شویم که Azure Developer CLI را نصب کرده‌اید.

1. ترمینال VS Code را باز کرده و این دستور را تایپ کنید:

      ```bash title="" linenums="0"
      azd version
      ```

1. چیزی شبیه به این باید ببینید!

      ```bash title="" linenums="0"
      azd version 1.25.6 (commit <current-build>)
      ```

**شما اکنون آماده‌اید تا یک قالب را با azd انتخاب و مستقر کنید**

## 2. انتخاب قالب

پلتفرم Microsoft Foundry همراه با یک [مجموعه‌ای از قالب‌های AZD پیشنهادی](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) ارائه می‌شود که سناریوهای محبوبی مانند _خودکارسازی جریان‌کار چندعامله_ و _پردازش محتوای چندمودال_ را پوشش می‌دهند. شما همچنین می‌توانید این قالب‌ها را با مراجعه به پرتال Microsoft Foundry کشف کنید.

1. به [https://ai.azure.com/templates](https://ai.azure.com/templates) مراجعه کنید
1. هنگام درخواست وارد پرتال Microsoft Foundry شوید - چیزی شبیه به این را خواهید دید.

![انتخاب](../../../../../translated_images/fa/01-pick-template.60d2d5fff5ebc374.webp)


گزینه‌های **Basic** قالب‌های شروع شما هستند:

1. [ ] [شروع کار با AI Chat](https://github.com/Azure-Samples/get-started-with-ai-chat) که یک برنامهٔ چت پایه را _با داده‌های شما_ به Azure Container Apps مستقر می‌کند. از این برای کاوش سناریوی پایهٔ چت‌بات هوش مصنوعی استفاده کنید.

## 3. فعال‌سازی قالب

بیایید این قالب را مستقر کنیم و مطمئن شویم که معتبر است. ما از دستورالعمل‌های بخش [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) پیروی خواهیم کرد.

1. یک محیط کاری برای مخزن قالب انتخاب کنید:

      - **GitHub Codespaces**: روی [this link](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) کلیک کنید و `Create codespace` را تأیید کنید
      - **کلون محلی یا دِو کانتینر**: `Azure-Samples/get-started-with-ai-agents` را کلون کرده و آن را در VS Code باز کنید

1. تا آماده شدن ترمینال VS Code صبر کنید، سپس دستور زیر را تایپ کنید:

   ```bash title="" linenums="0"
   azd up
   ```

روند کاری‌هایی را که این دستور فعال می‌کند تکمیل کنید:

1. از شما خواسته می‌شود وارد Azure شوید - دستورالعمل‌ها را برای احراز هویت دنبال کنید
1. یک نام محیط یکتا برای خود وارد کنید - به‌عنوان مثال، من از `nitya-mshack-azd` استفاده کردم
1. این یک پوشهٔ `.azure/` ایجاد خواهد کرد - شما یک زیردایرکتوری با نام محیط را مشاهده خواهید کرد
1. از شما خواسته می‌شود نام یک اشتراک را انتخاب کنید - پیش‌فرض را انتخاب کنید
1. از شما برای انتخاب یک مکان پرسیده می‌شود - از `East US 2` استفاده کنید

اکنون منتظر بمانید تا فراهم‌سازی کامل شود. **این کار 10-15 دقیقه طول می‌کشد**

1. وقتی کار تمام شد، کنسول شما یک پیام SUCCESS مانند این را نشان می‌دهد:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. اکنون پرتال Azure شما یک گروه منابع فراهم‌شده با آن نام محیط خواهد داشت:

      ![زیرساخت](../../../../../translated_images/fa/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **شما اکنون آماده‌اید تا زیرساخت و برنامهٔ مستقر شده را اعتبارسنجی کنید**.

## 4. اعتبارسنجی قالب

1. به صفحهٔ Azure Portal [Resource Groups](https://portal.azure.com/#browse/resourcegroups) مراجعه کنید - هنگام درخواست وارد شوید

1. باید یک رابط کاربری فرانت‌اند میزبانی‌شدهٔ برنامه مانند این را ببینید:

## 5.  اعتبارسنجی عامل

Azure Container App یک endpoint مستقر می‌کند که به Agent هوش مصنوعی فراهم‌شده در پروژهٔ Microsoft Foundry برای این قالب متصل می‌شود. بیایید ببینیم این به چه معناست.

1. به صفحهٔ Overview گروه منابع خود در Azure Portal بازگردید

1. روی منبع `Microsoft Foundry` در آن فهرست کلیک کنید

1. باید این را ببینید. روی دکمهٔ `Go to Microsoft Foundry Portal` کلیک کنید. 
   ![Foundry](../../../../../translated_images/fa/04-view-foundry-project.fb94ca41803f28f3.webp)

1. شما باید صفحهٔ پروژهٔ Foundry برای برنامهٔ هوش مصنوعی خود را ببینید
   ![پروژه](../../../../../translated_images/fa/05-visit-foundry-portal.d734e98135892d7e.webp)

1. روی `Agents` کلیک کنید - شما Agent پیش‌فرض فراهم‌شده در پروژهٔ خود را می‌بینید
   ![Agents](../../../../../translated_images/fa/06-visit-agents.bccb263f77b00a09.webp)

1. آن را انتخاب کنید - و جزئیات Agent را می‌بینید. به نکات زیر توجه کنید:

      - عامل به‌طور پیش‌فرض از File Search استفاده می‌کند (همیشه)
      - قسمت `Knowledge` عامل نشان می‌دهد که 32 فایل بارگذاری شده دارد (برای جستجوی فایل)
      ![Agents](../../../../../translated_images/fa/07-view-agent-details.0e049f37f61eae62.webp)

1. گزینهٔ `Data+indexes` را در منوی سمت چپ پیدا کرده و برای جزئیات کلیک کنید. 

      - باید 32 فایل داده بارگذاری‌شده برای دانش را مشاهده کنید.
      - این‌ها متناظر با 12 فایل مشتری و 20 فایل محصول در زیر `src/files` خواهند بود 
      ![داده](../../../../../translated_images/fa/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**شما عملیات Agent را اعتبارسنجی کردید!** 

1. پاسخ‌های عامل بر اساس دانش موجود در آن فایل‌ها پایه‌گذاری شده‌اند. 
1. شما اکنون می‌توانید سوالاتی مرتبط با آن داده بپرسید و پاسخ‌های مبتنی بر داده دریافت کنید.
1. مثال: `customer_info_10.json` سه خرید انجام‌شده توسط "Amanda Perez" را توصیف می‌کند

تب مرورگری که endpoint Container App را دارد دوباره باز کنید و بپرسید: `What products does Amanda Perez own?`. شما باید چیزی شبیه به این ببینید:

![داده](../../../../../translated_images/fa/09-ask-in-aca.4102297fc465a4d5.webp)

## 6. محیط بازی (Agent Playground)

بیایید با آوردن Agent به محیط بازی Agents Playground، کمی شهود بیشتری نسبت به قابلیت‌های Microsoft Foundry پیدا کنیم. 

1. به صفحهٔ `Agents` در Microsoft Foundry بازگردید - عامل پیش‌فرض را انتخاب کنید
1. گزینهٔ `Try in Playground` را کلیک کنید - باید یک رابط Playground مانند این دریافت کنید
1. همان سوال را بپرسید: `What products does Amanda Perez own?`

    ![داده](../../../../../translated_images/fa/09-ask-in-playground.a1b93794f78fa676.webp)

شما همان (یا پاسخ مشابه) را دریافت می‌کنید - اما علاوه بر آن اطلاعات اضافی‌ای نیز دریافت می‌کنید که می‌توانید از آن برای درک کیفیت، هزینه و عملکرد برنامهٔ عامل‌محور خود استفاده کنید. برای مثال:

1. توجه کنید که پاسخ به فایل‌های داده‌ای که برای "پایه‌گذاری" پاسخ استفاده شده‌اند اشاره می‌کند
1. روی هر یک از این برچسب‌های فایل حرکت کنید - آیا داده‌ها با پرسش شما و پاسخ نمایش‌داده‌شده مطابقت دارند؟

شما همچنین یک ردیف آمار (stats) را زیر پاسخ می‌بینید. 

1. روی هر معیار قرار بگیرید - مثلاً Safety. شما چیزی شبیه به این خواهید دید
1. آیا رتبه‌بندی ارزیابی‌شده با شهود شما دربارهٔ سطح ایمنی پاسخ مطابقت دارد؟

      ![داده](../../../../../translated_images/fa/10-view-run-info-meter.6cdb89a0eea5531f.webp)

## 7. قابلیت مشاهده داخلی

قابلیت مشاهده دربارهٔ ابزارسازی برنامهٔ شما برای تولید داده‌هایی است که می‌توان از آن‌ها برای درک، اشکال‌زدایی و بهینه‌سازی عملیات آن استفاده کرد. برای به‌دست آوردن درک از این موضوع:

1. روی دکمهٔ `View Run Info` کلیک کنید - باید این نما را ببینید. این نمونه‌ای از [Agent tracing](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) در عمل است. _شما همچنین می‌توانید با کلیک روی Thread Logs در منوی سطح بالا به این نما دسترسی پیدا کنید_.

   - نگاهی به مراحل اجرا و ابزارهای درگیر توسط عامل بیندازید
   - مجموع شمار توکن‌ها را درک کنید (در مقابل استفادهٔ توکن‌های خروجی) برای پاسخ
   - تأخیر و محل صرف زمان در اجرای فرایند را درک کنید

      ![Agent](../../../../../translated_images/fa/10-view-run-info.b20ebd75fef6a1cc.webp)

1. روی زبانهٔ `Metadata` کلیک کنید تا ویژگی‌های اضافی اجرا را ببینید، که ممکن است برای اشکال‌زدایی مسائل بعدی زمینهٔ مفیدی فراهم کنند.   

      ![Agent](../../../../../translated_images/fa/11-view-run-info-metadata.7966986122c7c2df.webp)


1. روی زبانهٔ `Evaluations` کلیک کنید تا ارزیابی‌های خودکار انجام‌شده روی پاسخ عامل را ببینید. این‌ها شامل ارزیابی‌های ایمنی (مثلاً خودآسیب‌رسانی) و ارزیابی‌های خاص عامل (مثلاً حل نیت، پایبندی به وظیفه) می‌شوند.

      ![Agent](../../../../../translated_images/fa/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. در نهایت، روی زبانهٔ `Monitoring` در منوی کناری کلیک کنید.

      - در صفحهٔ نمایش‌داده‌شده زبانهٔ `Resource usage` را انتخاب کنید - و معیارها را مشاهده کنید.
      - مصرف برنامه را از نظر هزینه‌ها (توکن‌ها) و بار (درخواست‌ها) پیگیری کنید.
      - زمان پاسخ برنامه تا اولین بایت (پردازش ورودی) و آخرین بایت (خروجی) را پیگیری کنید.

      ![Agent](../../../../../translated_images/fa/13-monitoring-resources.5148015f7311807f.webp)

## 8. متغیرهای محیطی

تا کنون ما استقرار را در مرورگر مرور کردیم - و تأیید کردیم که زیرساخت ما فراهم شده و برنامه عملیاتی است. اما برای کار با برنامه به‌صورت _کد-اول_، باید محیط توسعهٔ محلی خود را با متغیرهای مرتبط مورد نیاز برای کار با این منابع پیکربندی کنیم. استفاده از `azd` این کار را آسان می‌کند.

1. Azure Developer CLI از [متغیرهای محیطی](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) برای ذخیره و مدیریت تنظیمات پیکربندی برای استقرارهای برنامه استفاده می‌کند.

1. متغیرهای محیطی در `.azure/<env-name>/.env` ذخیره می‌شوند - این آن‌ها را به محیط `env-name` که در زمان استقرار استفاده شده، محدود می‌کند و به شما کمک می‌کند محیط‌ها را بین هدف‌های استقرار مختلف در همان مخزن ایزوله کنید.

1. متغیرهای محیطی به‌صورت خودکار توسط فرمان `azd` هر زمان که فرمان خاصی اجرا می‌شود (مثلاً `azd up`) بارگذاری می‌شوند. توجه داشته باشید که `azd` به‌طور خودکار متغیرهای محیطی سطح OS (مثلاً تنظیم‌شده در شل) را نمی‌خواند - در عوض از `azd set env` و `azd get env` برای انتقال اطلاعات درون اسکریپت‌ها استفاده کنید.


بیایید چند فرمان را امتحان کنیم:

1. تمام متغیرهای محیطی تنظیم‌شده برای `azd` در این محیط را دریافت کنید:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      شما چیزی شبیه به این می‌بینید:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. یک مقدار خاص را دریافت کنید - مثلاً من می‌خواهم بدانم آیا مقدار `AZURE_AI_AGENT_MODEL_NAME` تنظیم شده است یا خیر

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      شما چیزی شبیه به این می‌بینید - به طور پیش‌فرض تنظیم نشده بود!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. یک متغیر محیطی جدید برای `azd` تنظیم کنید. در اینجا، نام مدل عامل را به‌روزرسانی می‌کنیم. _توجه: هر تغییر انجام‌شده فوراً در فایل `.azure/<env-name>/.env` منعکس خواهد شد._

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      اکنون باید مقدار تنظیم‌شده را پیدا کنیم:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. توجه داشته باشید که برخی منابع پایدار هستند (مثلاً استقرار مدل‌ها) و نیاز به بیش از تنها یک `azd up` برای مجبور کردن استقرار مجدد خواهند داشت. بیایید سعی کنیم استقرار اصلی را قطع کنیم و با متغیرهای محیطی تغییر یافته دوباره مستقر کنیم.

1. **تازه‌سازی** اگر قبلاً زیرساخت را با استفاده از یک قالب azd مستقر کرده‌اید - می‌توانید وضعیت متغیرهای محیطی محلی خود را بر اساس وضعیت فعلی استقرار Azure خود با استفاده از این فرمان _تازه‌سازی_ کنید:

      ```bash title="" linenums="0"
      azd env refresh
      ```

      این یک روش قدرتمند برای _همگام‌سازی_ متغیرهای محیطی بین دو یا چند محیط توسعه محلی است (برای مثال، تیمی با چند توسعه‌دهنده) - اجازه می‌دهد که زیرساخت مستقر به‌عنوان مرجع نهایی وضعیت متغیرهای محیطی عمل کند. اعضای تیم به‌سادگی متغیرها را _تازه‌سازی_ می‌کنند تا دوباره همگام شوند.
