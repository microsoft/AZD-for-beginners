# کارگاه AZD برای توسعه‌دهندگان هوش مصنوعی

به کارگاه عملی خوش آمدید برای یادگیری Azure Developer CLI (AZD) با تمرکز بر استقرار برنامه‌های هوش مصنوعی. این کارگاه به شما کمک می‌کند تا فهم کاربردی از قالب‌های AZD را در 3 مرحله به دست آورید:

1. **کشف** - قالب مناسب خود را پیدا کنید.
1. **استقرار** - استقرار دهید و تأیید کنید که کار می‌کند
1. **سفارشی‌سازی** - ویرایش و تکرار کنید تا آن را مال خود کنید!

در طول این کارگاه، با ابزارها و جریان‌های کاری اصلی توسعه‌دهنده نیز آشنا خواهید شد تا به شما در ساده‌سازی مسیر توسعه انتها-به-انتهای‌تان کمک کند.

## نمای کلی کارگاه

**مدت:** 3-4 hours  
**سطح:** مبتدی تا متوسط  
**پیش‌نیازها:** آشنایی با Azure، مفاهیم هوش مصنوعی، VS Code و ابزارهای خط فرمان.

این یک کارگاه عملی است که در آن با عمل کردن یاد می‌گیرید. پس از تکمیل تمرین‌ها، توصیه می‌کنیم برای ادامه مسیر یادگیری خود به سمت بهترین شیوه‌های امنیت و بهره‌وری، برنامه درسی AZD For Beginners را مرور کنید.

| 30 mins | [اعتبارسنجی قالب AI](docs/instructions/2-Validate-AI-Template.md) | راه‌حل پیش‌فرض را در Azure مستقر کنید |

## آنچه یاد می‌گیرید

قالب AZD را به‌عنوان یک محیط شبیه‌ساز یادگیری در نظر بگیرید تا قابلیت‌ها و ابزارهای مختلف برای توسعه انتها-به-انتها روی Microsoft Foundry را کاوش کنید. تا پایان این کارگاه، باید درکی شهودی از ابزارها و مفاهیم مختلف در این زمینه داشته باشید.

| Concept  | Objective |
|:---|:---|
| **Azure Developer CLI** | درک دستورات ابزار و جریان‌های کاری |
| **AZD Templates**| درک ساختار پروژه و پیکربندی |
| **Azure AI Agent**| تهیه و استقرار پروژه Microsoft Foundry |
| **Azure AI Search**| فعال‌سازی مهندسی زمینه با عامل‌ها |
| **Observability**| بررسی ردیابی، پایش و ارزیابی‌ها |
| **Red Teaming**| بررسی تست‌های خصمانه و راهکارهای مقابله |

## ساختار کارگاه

این کارگاه به‌گونه‌ای ساختاربندی شده است که شما را از کشف قالب تا استقرار، بررسی ساختار و سفارشی‌سازی همراهی کند - با استفاده از قالب شروع رسمی [شروع کار با عوامل هوش مصنوعی](https://github.com/Azure-Samples/get-started-with-ai-agents) به‌عنوان پایه.

### [ماژول ۱: انتخاب قالب هوش مصنوعی](docs/instructions/1-Select-AI-Template.md) (30 mins)

- قالب‌های AI چیستند؟
- قالب‌های AI را از کجا می‌توان پیدا کرد؟
- چگونه می‌توانم شروع به ساختن AI Agents کنم؟
- **تمرین**: راه‌اندازی سریع در Codespaces یا یک کانتینر توسعه

### [ماژول ۲: اعتبارسنجی قالب AI](docs/instructions/2-Validate-AI-Template.md) (30 mins)

- معماری قالب AI چیست؟
- جریان کاری توسعه AZD چیست؟
- چگونه می‌توانم در توسعه با AZD کمک بگیرم؟
- **تمرین**: استقرار و اعتبارسنجی قالب AI Agents

### [ماژول ۳: بررسی ساختار قالب AI](docs/instructions/3-Deconstruct-AI-Template.md) (30 mins)

- مرور محیط خود در `.azure/` 
- مرور تنظیمات منابع خود در `infra/` 
- پیکربندی AZD خود را در `azure.yaml`s بررسی کنید
- **تمرین**: تغییر متغیرهای محیطی و استقرار مجدد

### [ماژول ۴: پیکربندی قالب AI](docs/instructions/4-Configure-AI-Template.md) (30 mins)
- بررسی: Retrieval Augmented Generation
- بررسی: ارزیابی عامل و Red Teaming
- بررسی: ردیابی و پایش
- **تمرین**: بررسی عامل AI و قابلیت مشاهده 

### [ماژول ۵: سفارشی‌سازی قالب AI](docs/instructions/5-Customize-AI-Template.md) (30 mins)
- تعریف: PRD همراه با نیازمندی‌های سناریو
- پیکربندی: متغیرهای محیطی برای AZD
- پیاده‌سازی: Lifecycle Hooks برای وظایف اضافی
- **تمرین**: سفارشی‌سازی قالب برای سناریوی من

### [ماژول ۶: پاک‌سازی زیرساخت](docs/instructions/6-Teardown-Infrastructure.md) (30 mins)
- مرور: قالب‌های AZD چیستند؟
- مرور: چرا از Azure Developer CLI استفاده کنیم؟
- گام‌های بعدی: یک قالب متفاوت را امتحان کنید!
- **تمرین**: حذف تخصیص زیرساخت و پاکسازی

## چالش کارگاه

می‌خواهید خودتان را بیشتر به چالش بکشید؟ در اینجا چند پیشنهاد پروژه وجود دارد - یا ایده‌های خود را با ما به اشتراک بگذارید!!

| Project | Description |
|:---|:---|
|1. **بررسی ساختار یک قالب پیچیده هوش مصنوعی** | از جریان کاری و ابزارهایی که توضیح دادیم استفاده کنید و ببینید آیا می‌توانید یک قالب راه‌حل متفاوت AI را مستقر، اعتبارسنجی و سفارشی کنید. _چه آموختید؟_|
|2. **سفارشی‌سازی بر اساس سناریوی خود**  | سعی کنید یک PRD (سند نیازمندی‌های محصول) برای یک سناریوی متفاوت بنویسید. سپس از GitHub Copilot در مخزن قالب خود در Agent Model استفاده کنید - و از آن بخواهید یک جریان کاری سفارشی‌سازی برای شما تولید کند. _چه آموختید؟ چگونه می‌توانید این پیشنهادها را بهبود بخشید؟_|
| | |
