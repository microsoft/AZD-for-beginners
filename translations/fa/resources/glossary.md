## مقدمه

این فرهنگ‌لغت جامع، تعاریف اصطلاحات، مفاهیم و مخفف‌هایی را که در Azure Developer CLI و توسعهٔ ابری Azure استفاده می‌شوند ارائه می‌دهد. مرجع ضروری برای درک مستندات فنی، عیب‌یابی مشکلات و ارتباط موثر در مورد پروژه‌های azd و سرویس‌های Azure.

## اهداف یادگیری

با استفاده از این فرهنگ‌لغت، شما:
- با اصطلاحات و مفاهیم اساسی Azure Developer CLI آشنا می‌شوید
- واژگان توسعهٔ ابری Azure و اصطلاحات فنی را فرا می‌گیرید
- به‌طور مؤثر به اصطلاحات زیرساخت به‌عنوان کد و استقرار مراجعه می‌کنید
- نام‌ها، مخفف‌ها و کاربرد سرویس‌های Azure را درک می‌کنید
- تعاریف مرتبط با عیب‌یابی و دیباگ را دسترسی خواهید داشت
- مفاهیم پیشرفتهٔ معماری و توسعهٔ Azure را می‌آموزید

## نتایج یادگیری

با مراجعهٔ منظم به این فرهنگ‌لغت، قادر خواهید بود:
- با استفاده از اصطلاحات صحیح Azure Developer CLI به‌طور مؤثر ارتباط برقرار کنید
- مستندات فنی و پیام‌های خطا را واضح‌تر درک کنید
- با اعتماد به نفس در سرویس‌ها و مفاهیم Azure گردش کنید
- مسائل را با واژگان فنی مناسب عیب‌یابی کنید
- در بحث‌های تیمی با زبان فنی دقیق مشارکت کنید
- دانش توسعهٔ ابری Azure خود را به‌صورت سیستماتیک گسترش دهید

## A

**ARM Template**  
قالب Azure Resource Manager. فرمت Infrastructure as Code مبتنی بر JSON که برای تعریف و استقرار منابع Azure به‌صورت اعلامی استفاده می‌شود.

**App Service**  
پلتفرم به‌عنوان سرویس (PaaS) Azure برای میزبانی برنامه‌های وب، REST APIها و بک‌اندهای موبایل بدون مدیریت زیرساخت.

**Application Insights**  
خدمت پایش عملکرد برنامه (APM) در Azure که بینش عمیقی دربارهٔ عملکرد، قابلیت‌دسترس‌پذیری و استفادهٔ برنامه‌ها فراهم می‌کند.

**Azure CLI**  
رابط خط فرمان برای مدیریت منابع Azure. توسط azd برای احراز هویت و برخی عملیات استفاده می‌شود.

**Azure Developer CLI (azd)**  
ابزار خط فرمان متمرکز بر توسعه‌دهنده که فرآیند ساخت و استقرار برنامه‌ها در Azure را با استفاده از الگوها و Infrastructure as Code تسریع می‌کند.

**azure.yaml**  
فایل پیکربندی اصلی برای یک پروژهٔ azd که خدمات، زیرساخت و هوک‌های استقرار را تعریف می‌کند.

**Azure Resource Manager (ARM)**  
سرویس استقرار و مدیریت Azure که لایهٔ مدیریتی برای ایجاد، به‌روزرسانی و حذف منابع فراهم می‌کند.

## B

**Bicep**  
زبان دامنه‌محور (DSL) توسعه‌یافته توسط مایکروسافت برای استقرار منابع Azure. نحوی ساده‌تر از قالب‌های ARM ارائه می‌دهد و به ARM کامپایل می‌شود.

**Build**  
فرآیند کامپایل کد منبع، نصب وابستگی‌ها و آماده‌سازی برنامه‌ها برای استقرار.

**Blue-Green Deployment**  
استراتژی استقرار که از دو محیط تولیدی یکسان (آبی و سبز) برای به حداقل رساندن زمان خاموشی و ریسک استفاده می‌کند.

## C

**Container Apps**  
خدمت کانتینری بدون سرور Azure که اجرای برنامه‌های کانتینری‌شده را بدون مدیریت زیرساخت پیچیده ممکن می‌سازد.

**CI/CD**  
تلفیق پیوسته/استقرار پیوسته. روش‌های خودکار برای ادغام تغییرات کد و استقرار برنامه‌ها.

**Cosmos DB**  
خدمت پایگاه‌دادهٔ چندمدلی و توزیع‌شدهٔ جهانی Azure که SLAهای جامع برای توان، تأخیر، قابلیت‌دسترس‌پذیری و سازگاری ارائه می‌دهد.

**Configuration**  
تنظیمات و پارامترهایی که رفتار برنامه و گزینه‌های استقرار را کنترل می‌کنند.

## D

**Deployment**  
فرآیند نصب و پیکربندی برنامه‌ها و وابستگی‌های آن‌ها بر روی زیرساخت هدف.

**Docker**  
پلتفرمی برای توسعه، ارسال و اجرای برنامه‌ها با استفاده از فناوری کانتینریزه‌سازی.

**Dockerfile**  
فایل متنی حاوی دستورالعمل‌هایی برای ساخت یک تصویر کانتینر Docker.

## E

**Environment**  
هدف استقرار که نمایانگر یک نمونهٔ مشخص از برنامهٔ شما است (مثلاً توسعه، مرحله‌بندی، تولید).

**Environment Variables**  
مقادیر پیکربندی ذخیره شده به‌صورت زوج‌های کلید-مقدار که برنامه‌ها می‌توانند در زمان اجرا به آن‌ها دسترسی داشته باشند.

**Endpoint**  
URL یا آدرس شبکه‌ای که برنامه یا سرویس در آن قابل دسترسی است.

## F

**Function App**  
خدمت محاسباتی بدون سرور Azure که اجرای کد رویدادمحور را بدون مدیریت زیرساخت ممکن می‌سازد.

## G

**GitHub Actions**  
پلتفرم CI/CD یکپارچه با مخازن GitHub برای خودکارسازی گردش‌کارها.

**Git**  
سیستم کنترل نسخه توزیع‌شده که برای ردیابی تغییرات در کد منبع استفاده می‌شود.

## H

**Hooks**  
اسکریپت‌ها یا دستورات سفارشی که در نقاط مشخصی از چرخهٔ عمر استقرار اجرا می‌شوند (preprovision, postprovision, predeploy, postdeploy).

**Host**  
نوع سرویس Azure که برنامه در آن مستقر خواهد شد (مثلاً appservice, containerapp, function).

## I

**Infrastructure as Code (IaC)**  
عملی که زیرساخت را از طریق کد به‌جای فرایندهای دستی تعریف و مدیریت می‌کند.

**Init**  
فرآیند مقداردهی اولیهٔ یک پروژهٔ جدید azd، معمولاً از یک قالب.

## J

**JSON**  
JavaScript Object Notation. فرمت تبادل داده که معمولاً برای فایل‌های پیکربندی و پاسخ‌های API استفاده می‌شود.

**JWT**  
JSON Web Token. استانداردی برای انتقال امن اطلاعات بین طرف‌ها به‌صورت یک شیء JSON.

## K

**Key Vault**  
سرویس Azure برای ذخیره و مدیریت امن اسرار، کلیدها و گواهی‌ها.

**Kusto Query Language (KQL)**  
زبان پرس‌وجو که برای تحلیل داده‌ها در Azure Monitor، Application Insights و سایر سرویس‌های Azure استفاده می‌شود.

## L

**Load Balancer**  
سرویسی که ترافیک ورودی شبکه را بین سرورها یا نمونه‌های متعدد توزیع می‌کند.

**Log Analytics**  
سرویسی در Azure برای جمع‌آوری، تحلیل و واکنش به داده‌های تله‌متری از محیط‌های ابری و درون‌محیطی.

## M

**Managed Identity**  
قابلیت Azure که به سرویس‌های Azure هویت مدیریت‌شدهٔ خودکار برای احراز هویت به سرویس‌های دیگر می‌دهد.

**Microservices**  
رویکرد معماری که در آن برنامه‌ها به‌صورت مجموعه‌ای از سرویس‌های کوچک و مستقل ساخته می‌شوند.

**Monitor**  
راهکار یکپارچهٔ پایش Azure که قابلیت مشاهدهٔ کامل از برنامه‌ها و زیرساخت را فراهم می‌کند.

## N

**Node.js**  
محیط اجرای جاوااسکریپت مبتنی بر موتور V8 کروم برای ساخت برنامه‌های سمت سرور.

**npm**  
مدیر بسته برای Node.js که وابستگی‌ها و بسته‌ها را مدیریت می‌کند.

## O

**Output**  
مقادیر بازگردانده‌شده از استقرار زیرساخت که می‌توانند توسط برنامه‌ها یا سایر منابع استفاده شوند.

## P

**Package**  
فرآیند آماده‌سازی کد برنامه و وابستگی‌ها برای استقرار.

**Parameters**  
مقادیر ورودی که به قالب‌های زیرساختی برای سفارشی‌سازی استقرارها ارسال می‌شوند.

**PostgreSQL**  
سیستم پایگاه‌دادهٔ رابطه‌ای متن‌باز که به‌صورت سرویس مدیریتی در Azure پشتیبانی می‌شود.

**Provisioning**  
فرآیند ایجاد و پیکربندی منابع Azure که در قالب‌های زیرساخت تعریف شده‌اند.

## Q

**Quota**  
محدودیت‌هایی بر مقدار منابعی که می‌توان در یک اشتراک یا منطقهٔ Azure ایجاد کرد.

## R

**Resource Group**  
محیط منطقی برای منابع Azure که دورهٔ عمر، مجوزها و سیاست‌های مشترک دارند.

**Resource Token**  
رشتهٔ یکتایی که توسط azd تولید می‌شود تا اطمینان حاصل کند نام منابع در استقرارها منحصر به‌فرد است.

**REST API**  
سبک معماری برای طراحی برنامه‌های شبکه‌ای که از روش‌های HTTP استفاده می‌کند.

**Rollback**  
فرآیند بازگرداندن به نسخهٔ قبلی یک برنامه یا پیکربندی زیرساخت.

## S

**Service**  
یک مولفه از برنامهٔ شما که در azure.yaml تعریف شده است (مثلاً رابط کاربری وب، بک‌اند API، پایگاه‌داده).

**SKU**  
واحد نگهداری موجودی. نمایانگر سطوح سرویس یا سطح عملکرد متفاوت برای منابع Azure.

**SQL Database**  
خدمت مدیریت‌شدهٔ پایگاه‌دادهٔ رابطه‌ای Azure مبتنی بر Microsoft SQL Server.

**Static Web Apps**  
سرویس Azure برای ساخت و استقرار برنامه‌های وب تمام‌پشته از مخازن کد منبع.

**Storage Account**  
سرویسی در Azure که ذخیره‌سازی ابری برای اشیاء داده از جمله بلاگ‌ها، فایل‌ها، صف‌ها و جدول‌ها فراهم می‌کند.

**Subscription**  
کانتینر حساب Azure که گروه‌های منبع و منابع را نگه می‌دارد و دارای صورتحساب و مدیریت دسترسی مرتبط است.

## T

**Template**  
ساختار پروژه از پیش‌ساخته‌شده که شامل کد برنامه، تعریف‌های زیرساخت و پیکربندی برای سناریوهای رایج است.

**Terraform**  
ابزار متن‌باز Infrastructure as Code که از چندین ارائه‌دهندهٔ ابری از جمله Azure پشتیبانی می‌کند.

**Traffic Manager**  
متعادل‌کنندهٔ بار مبتنی بر DNS در Azure برای توزیع ترافیک در مناطق جهانی Azure.

## U

**URI**  
شناسهٔ منبع یکنواخت. رشته‌ای که یک منبع خاص را شناسایی می‌کند.

**URL**  
مسیر یکنواخت منبع. نوعی از URI که مشخص می‌کند یک منبع کجا قرار دارد و چگونه بازیابی شود.

## V

**Virtual Network (VNet)**  
بلوک ساختمانی پایه برای شبکه‌های خصوصی در Azure که جداسازی و بخش‌بندی را فراهم می‌کند.

**VS Code**  
Visual Studio Code. ویرایشگر کد محبوب با ادغام عالی با Azure و azd.

## W

**Webhook**  
بازخوان HTTP که توسط رویدادهای خاص فعال می‌شود و معمولاً در خطوط کاری CI/CD استفاده می‌شود.

**What-if**  
قابلیت Azure که نشان می‌دهد یک استقرار چه تغییراتی ایجاد خواهد کرد بدون اینکه عملاً آن را اجرا کند.

## Y

**YAML**  
YAML Ain't Markup Language. استاندارد سریال‌سازی داده‌ای خوانا برای انسان که برای فایل‌های پیکربندی مانند azure.yaml استفاده می‌شود.

## Z

**Zone**  
مکان‌های فیزیکی جداگانه در یک منطقهٔ Azure که افزونگی و دسترسی‌پذیری بالا را فراهم می‌کنند.

## اصطلاحات خاص با زمینه

### اصطلاحات توسعه
- **Hot Reload**: به‌روزرسانی خودکار برنامه‌ها در طول توسعه بدون راه‌اندازی مجدد
- **Build Pipeline**: فرآیند خودکار برای ساخت و تست کد
- **Deployment Slot**: محیط مرحله‌بندی درون یک App Service
- **Environment Parity**: حفظ شباهت بین محیط‌های توسعه، مرحله‌بندی و تولید

### اصطلاحات امنیتی
- **Managed Identity**: قابلیت Azure که مدیریت خودکار اعتبارنامه‌ها را فراهم می‌کند
- **Key Vault**: ذخیرهٔ امن اسرار، کلیدها و گواهی‌ها
- **RBAC**: کنترل دسترسی مبتنی بر نقش برای منابع Azure
- **Network Security Group**: فایروال مجازی برای کنترل ترافیک شبکه

### اصطلاحات پایش
- **Telemetry**: جمع‌آوری خودکار سنجه‌ها و داده‌ها
- **Application Performance Monitoring (APM)**: پایش عملکرد نرم‌افزار
- **Log Analytics**: سرویسی برای جمع‌آوری و تحلیل داده‌های لاگ
- **Alert Rules**: اعلان‌های خودکار مبتنی بر سنجه‌ها یا شرایط

### اصطلاحات استقرار
- **Blue-Green Deployment**: استراتژی استقرار بدون قطعی سرویس
- **Canary Deployment**: عرضهٔ تدریجی به زیرمجموعه‌ای از کاربران
- **Rolling Update**: جایگزینی متوالی نمونه‌های برنامه
- **Rollback**: بازگرداندن به نسخهٔ قبلی برنامه
