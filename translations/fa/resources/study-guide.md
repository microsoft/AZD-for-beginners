## معرفی

این راهنمای مطالعهٔ جامع اهداف یادگیری ساختاریافته، مفاهیم کلیدی، تمرین‌های عملی و مواد ارزیابی را برای کمک به تسلط شما بر Azure Developer CLI (azd) فراهم می‌کند. از این راهنما برای پیگیری پیشرفت خود و اطمینان از پوشش همهٔ موضوعات ضروری استفاده کنید.

## اهداف یادگیری

با کامل کردن این راهنمای مطالعه، شما خواهید:
- بر تمام مفاهیم پایه‌ای و پیشرفتهٔ Azure Developer CLI تسلط پیدا کرد
- مهارت‌های عملی در استقرار و مدیریت برنامه‌های Azure توسعه دهید
- در عیب‌یابی و بهینه‌سازی استقرارها اعتماد به نفس کسب کنید
- با شیوه‌های آمادهٔ تولید برای استقرار و ملاحظات امنیتی آشنا شوید

## نتایج یادگیری

پس از تکمیل همه بخش‌های این راهنمای مطالعه، شما قادر خواهید بود:
- طراحی، استقرار و مدیریت معماری‌های کامل برنامه با استفاده از azd
- پیاده‌سازی مانیتورینگ جامع، امنیت و استراتژی‌های بهینه‌سازی هزینه
- به‌صورت مستقل مسائل پیچیدهٔ استقرار را عیب‌یابی کنید
- قالب‌های سفارشی ایجاد کرده و به جامعهٔ azd کمک کنید

## ساختار یادگیری 8 فصلی

### فصل 1: مبانی و شروع سریع (هفته 1)
**مدت زمان**: 30-45 دقیقه | **پیچیدگی**: ⭐

#### اهداف یادگیری
- درک مفاهیم اصلی و واژگان Azure Developer CLI
- نصب و پیکربندی موفق AZD در پلتفرم توسعهٔ خود
- استقرار اولین برنامهٔ خود با استفاده از یک قالب موجود
- پیمایش مؤثر رابط خط فرمان AZD

#### مفاهیم کلیدی برای تسلط
- ساختار و اجزای پروژهٔ AZD (azure.yaml, infra/, src/)
- جریان‌های کاری استقرار مبتنی بر قالب
- اصول پیکربندی محیط
- مدیریت گروه منابع و اشتراک

#### تمرین‌های عملی
1. **تأیید نصب**: نصب AZD و تأیید با `azd version`
2. **اولین استقرار**: با موفقیت قالب todo-nodejs-mongo را مستقر کنید
3. **تنظیم محیط**: اولین متغیرهای محیطی خود را پیکربندی کنید
4. **کاوش منابع**: منابع مستقر شده را در پرتال Azure مرور کنید

#### سوالات ارزیابی
- اجزای اصلی یک پروژهٔ AZD چه هستند؟
- چگونه یک پروژهٔ جدید را از قالبی مقداردهی اولیه می‌کنید؟
- تفاوت بین `azd up` و `azd deploy` چیست؟
- چگونه با AZD محیط‌های متعدد را مدیریت می‌کنید؟

### فصل 2: توسعهٔ مبتنی بر هوش مصنوعی (هفته 2)
**مدت زمان**: 1-2 ساعت | **پیچیدگی**: ⭐⭐

#### اهداف یادگیری
- یکپارچه‌سازی خدمات Microsoft Foundry با جریان‌های کاری AZD
- استقرار و پیکربندی برنامه‌های مجهز به هوش مصنوعی
- درک الگوهای پیاده‌سازی RAG (Retrieval-Augmented Generation)
- مدیریت استقرار و مقیاس‌بندی مدل‌های هوش مصنوعی

#### مفاهیم کلیدی برای تسلط
- یکپارچه‌سازی سرویس Models در Microsoft Foundry و مدیریت API
- پیکربندی AI Search و ایندکسینگ برداری
- استراتژی‌های استقرار مدل و برنامه‌ریزی ظرفیت
- مانیتورینگ برنامه‌های هوش مصنوعی و بهینه‌سازی عملکرد

#### تمرین‌های عملی
1. **استقرار چت هوش مصنوعی**: قالب azure-search-openai-demo را مستقر کنید
2. **پیاده‌سازی RAG**: ایندکس‌گذاری اسناد و بازیابی را پیکربندی کنید
3. **پیکربندی مدل**: چند مدل هوش مصنوعی با اهداف متفاوت راه‌اندازی کنید
4. **مانیتورینگ هوش مصنوعی**: Application Insights را برای بارهای کاری هوش مصنوعی پیاده‌سازی کنید

#### سوالات ارزیابی
- چگونه سرویس‌های Microsoft Foundry Models را در یک قالب AZD پیکربندی می‌کنید؟
- اجزای کلیدی یک معماری RAG چه هستند؟
- چگونه ظرفیت و مقیاس مدل‌های هوش مصنوعی را مدیریت می‌کنید؟
- کدام معیارهای مانیتورینگ برای برنامه‌های هوش مصنوعی اهمیت دارند؟

### فصل 3: پیکربندی و احراز هویت (هفته 3)
**مدت زمان**: 45-60 دقیقه | **پیچیدگی**: ⭐⭐

#### اهداف یادگیری
- تسلط بر استراتژی‌های پیکربندی و مدیریت محیط‌ها
- پیاده‌سازی الگوهای احراز هویت ایمن و managed identity
- سازماندهی منابع با قراردادهای نام‌گذاری مناسب
- پیکربندی استقرارهای چندمحیطی (dev, staging, prod)

#### مفاهیم کلیدی برای تسلط
- سلسله‌مراتب محیط و تقدم پیکربندی
- managed identity و احراز هویت service principal
- یکپارچه‌سازی Key Vault برای مدیریت اسرار
- مدیریت پارامترهای مخصوص محیط

#### تمرین‌های عملی
1. **تنظیم چندمحیطی**: محیط‌های dev، staging و prod را پیکربندی کنید
2. **پیکربندی امنیت**: احراز هویت managed identity را پیاده‌سازی کنید
3. **مدیریت اسرار**: Azure Key Vault را برای داده‌های حساس یکپارچه کنید
4. **مدیریت پارامترها**: پیکربندی‌های مخصوص هر محیط ایجاد کنید

#### سوالات ارزیابی
- چگونه محیط‌های مختلف را با AZD پیکربندی می‌کنید؟
- مزایای استفاده از managed identity نسبت به service principal چیست؟
- چگونه اسرار برنامه را به‌صورت ایمن مدیریت می‌کنید؟
- سلسله‌مراتب پیکربندی در AZD چگونه است؟

- درک استراتژی‌های تخصیص منابع
- طراحی معماری‌های مقیاس‌پذیر چندخدمتی

- استقرار برنامه‌های کانتینری شده با Azure Container Apps و AZD


#### مفاهیم کلیدی برای تسلط
- ساختار قالب Bicep و بهترین شیوه‌ها
- وابستگی‌های منابع و ترتیب استقرار
- فایل‌های پارامتر و مدولار بودن قالب‌ها
- هوک‌های سفارشی و اتوماسیون استقرار
- الگوهای استقرار برنامهٔ کانتینر (شروع سریع، تولید، میکروسرویس‌ها)

#### تمرین‌های عملی
1. **ایجاد قالب سفارشی**: یک قالب برنامهٔ چندخدمتی بسازید
2. **تسلط بر Bicep**: کامپوننت‌های زیرساختی مدولار و قابل بازاستفاده ایجاد کنید
3. **اتوماسیون استقرار**: هوک‌های قبل/بعد از استقرار را پیاده‌سازی کنید
4. **طراحی معماری**: معماری میکروسرویس پیچیده‌ای استقرار دهید
5. **استقرار برنامهٔ کانتینری**: مثال [نمونه‌ی سادهٔ Flask API](../../../examples/container-app/simple-flask-api) و [معماری میکروسرویس‌ها](../../../examples/container-app/microservices) را با استفاده از AZD مستقر کنید

#### سوالات ارزیابی
- چگونه قالب‌های Bicep سفارشی برای AZD ایجاد می‌کنید؟
- بهترین شیوه‌ها برای سازماندهی کد زیرساختی چیست؟
- چگونه وابستگی‌های منابع را در قالب‌ها مدیریت می‌کنید؟

- درک تخصص‌یابی عامل‌ها و الگوهای گردش کار
- یکپارچه‌سازی میکروسرویس‌های کانتینری به‌عنوان بخشی از راهکارهای چندعامل

#### مفاهیم کلیدی برای تسلط
- الگوها و اصول طراحی معماری چندعامل
- پروتکل‌های ارتباطی عامل‌ها و جریان داده

5. **یکپارچه‌سازی میکروسرویس‌ها**: مثال [معماری میکروسرویس‌ها](../../../examples/container-app/microservices) را برای پشتیبانی از گردش‌کارهای مبتنی بر عامل گسترش دهید

#### سوالات ارزیابی

- چگونه سیستم‌های هوش مصنوعی چندعامل را مانیتور و اشکال‌زدایی می‌کنید؟

- اعمال بهترین شیوه‌ها برای استقرارهای تولیدی برنامهٔ کانتینری (امنیت، مانیتورینگ، هزینه، CI/CD)

#### مفاهیم کلیدی برای تسلط
- الزامات امنیتی و انطباق سازمانی
- چارچوب‌های حکمرانی و پیاده‌سازی سیاست
- مانیتورینگ پیشرفته و مدیریت هزینه

- کدام الگوهای حکمرانی تضمین‌کنندهٔ انطباق و کنترل هزینه هستند؟
- چگونه مانیتورینگ مقیاس‌پذیر برای سیستم‌های تولید طراحی می‌کنید؟
- کدام الگوهای CI/CD بهترین کارایی را با جریان‌های کاری AZD دارند؟

#### اهداف یادگیری
- درک اصول و مفاهیم اصلی Azure Developer CLI
- نصب و پیکربندی موفق azd در محیط توسعهٔ خود
- تکمیل اولین استقرار خود با استفاده از یک قالب موجود
- پیمایش ساختار پروژهٔ azd و درک اجزای کلیدی

#### مفاهیم کلیدی برای تسلط
- قالب‌ها، محیط‌ها و سرویس‌ها
- ساختار پیکربندی azure.yaml
- دستورات پایهٔ azd (init, up, down, deploy)
- اصول Infrastructure as Code
