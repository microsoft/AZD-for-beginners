## نمای کلی

این فصل مراحل اساسی برنامه‌ریزی و اعتبارسنجی قبل از استقرار برنامه شما را پوشش می‌دهد. بیاموزید چگونه با برنامه‌ریزی ظرفیت مناسب، انتخاب SKU و چک‌های پیش‌پروازی از اشتباهات پرهزینه جلوگیری کنید.

## اهداف یادگیری

با تکمیل این فصل، شما خواهید توانست:
- اجرای چک‌های پیش‌از استقرار قبل از استقرار
- برنامه‌ریزی ظرفیت و برآورد نیازهای منابع
- انتخاب SKUهای مناسب برای بهینه‌سازی هزینه
- پیکربندی Application Insights برای نظارت
- درک الگوهای هماهنگی تیمی

## 🚀 شروع سریع

```bash
# بررسی سهمیه‌های اشتراک
az vm list-usage --location eastus --output table

# پیش‌نمایش استقرار (بدون ایجاد منابع)
azd provision --preview

# اعتبارسنجی نحو Bicep
az bicep build --file infra/main.bicep

# بررسی پیکربندی محیط
azd env get-values
```

---

## ☑️ چک‌لیست پیش از استقرار

### قبل از `azd provision`

- [ ] سهمیه برای منطقه تأیید شده
- [ ] SKUها به‌درستی انتخاب شده‌اند
- [ ] برآورد هزینه بررسی شده
- [ ] قواعد نام‌گذاری سازگار است
- [ ] امنیت/RBAC پیکربندی شده‌اند

### قبل از `azd deploy`

- [ ] متغیرهای محیطی تنظیم شده‌اند
- [ ] محرمانه‌ها در Key Vault قرار دارند
- [ ] رشته‌های اتصال تأیید شده‌اند
- [ ] بررسی‌های سلامت پیکربندی شده‌اند
