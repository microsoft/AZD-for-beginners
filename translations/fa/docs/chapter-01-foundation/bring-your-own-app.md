# آور برنامه خود - افزودن azd به یک پروژه موجود

**ناوبری فصل:**
- **📚 صفحهٔ دوره**: [AZD برای مبتدیان](../../README.md)
- **📖 فصل جاری**: فصل 1 - پایه و شروع سریع
- **⬅️ قبلی**: [پروژهٔ اول شما](first-project.md)
- **➡️ بعدی**: [Dev Containers و Codespaces](dev-containers.md)

> تأیید شده با `azd 1.25.6` در ژوئن 2026.

## مقدمه

در [پروژهٔ اول شما](first-project.md) شما یک برنامه را با شروع از یک قالب مستقر کردید. اما در بیشتر مواقع شما از قبل یک برنامه *دارید* — یک API Node.js، یک سرویس Python Flask، یا یک وب‌اپ .NET — که در پوشه‌ای روی ماشینتان قرار دارد. این درس نشان می‌دهد چگونه azd را به آن کد موجود اضافه کنید تا بتوانید آن را با `azd up` مستقر کنید، بدون نیاز به الگو.

## اهداف یادگیری

تا پایان این درس شما:
- درک سه راه برای شروع یک پروژه azd
- اجرای `azd init` درون یک کدبیس موجود
- درک اینکه `azure.yaml` و پوشهٔ `infra/` چه کاری برای برنامهٔ شما انجام می‌دهند
- بدانید چه زمانی اجازه دهید azd زیرساخت را تولید کند و چه زمانی خودتان بنویسید
- استقرار برنامهٔ موجود خود در Azure با `azd up`

## نتایج یادگیری

پس از تکمیل این درس، شما قادر خواهید بود:
- راه‌اندازی azd در پروژه‌ای که قبلاً دارید
- خواندن و ویرایش یک فایل پایه `azure.yaml`
- تولید زیرساخت شروعی با `azd infra generate`
- انتخاب میزبان مناسب Azure برای برنامه‌تان
- استقرار و پاک‌سازی برنامهٔ خود

---

## سه راه برای شروع یک پروژه azd

| نقطهٔ شروع | فرمان | چه زمانی استفاده شود |
|----------------|---------|-------------|
| **از یک قالب** | `azd init --template <name>` | یادگیری یا شروع یک برنامهٔ جدید از یک نمونهٔ آزموده |
| **از کد موجود خود** | `azd init` (در پوشهٔ پروژهٔ شما) | شما از قبل یک برنامه دارید و می‌خواهید آن را مستقر کنید |
| **از یک مخزن Git** | `azd init --from-code` (در یک مخزن کلون شده) | پذیرش azd برای یک مخزن موجود |

شما قبلاً گزینهٔ اول را تمرین کردید. این درس گزینهٔ دوم — رایج‌ترین سناریوی دنیای واقعی — را پوشش می‌دهد.

---

## مرحلهٔ 1: اجرای `azd init` در پروژهٔ شما

یک ترمینال **درون پوشهٔ پروژهٔ موجود خود** باز کنید و اجرا کنید:

```bash
cd my-existing-app
azd init
```

azd از شما خواهد پرسید چگونه می‌خواهید راه‌اندازی کنید. انتخاب کنید:

```
? How do you want to initialize your app?
> Use code in the current directory
  Select a template
```

گزینهٔ **"Use code in the current directory."** را انتخاب کنید. azd سپس پوشهٔ شما را اسکن می‌کند، زبان و فریم‌ورک را تشخیص می‌دهد و یک میزبان پیشنهاد می‌دهد.

### آنچه azd تشخیص می‌دهد

azd به دنبال نشانه‌هایی مانند `package.json`، `requirements.txt`، `pom.xml`، `*.csproj` یا یک `Dockerfile` می‌گردد و یک میزبان Azure متناظر پیشنهاد می‌دهد:

| برنامهٔ شما | میزبان محتمل |
|----------|----------------------|
| Node.js / Python / برنامهٔ وب .NET | Azure App Service یا Container Apps |
| برنامهٔ کانتینری‌شده (`Dockerfile`) | Azure Container Apps |
| برنامهٔ Function | Azure Functions |
| سایت ایستا (خروجی ساخت React/Vue) | Azure Static Web Apps |

سرویس(های) تشخیص‌داده‌شده را تأیید کنید؛ azd فایل‌های مورد نیاز را ایجاد می‌کند.

---

## مرحلهٔ 2: درک آنچه azd ایجاد کرد

بعد از init، دو مورد جدید در پروژهٔ شما ایجاد خواهند شد:

```
my-existing-app/
├── azure.yaml          # ← NEW: tells azd what to deploy and where
├── infra/              # ← NEW: Infrastructure as Code (Bicep by default)
│   ├── main.bicep
│   └── main.parameters.json
├── src/ (your code)    # unchanged
└── ...                 # your existing files, untouched
```

### `azure.yaml` — تعریف پروژه

این قلب یک پروژهٔ azd است. یک مثال حداقلی به این شکل است:

```yaml
# azure.yaml
name: my-existing-app
services:
  web:
    project: ./src           # path to your app code
    language: js             # js | python | dotnet | java
    host: appservice         # appservice | containerapp | function | staticwebapp
```

بخش `services` قسمت کلیدی است: هر ورودی یک پوشه از کد شما را به یک میزبان Azure نگاشت می‌کند. اگر برنامهٔ شما هم فرانت‌اند و هم API داشته باشد، دو سرویس خواهید داشت.

### `infra/` — منابع Azure شما به‌صورت کد

پوشهٔ `infra/` شامل فایل‌های Bicep است که منابع Azure مورد نیاز برنامهٔ شما را تعریف می‌کنند (مثلاً App Service، پایگاه داده، و غیره). لازم نیست این‌ها را دستی بنویسید — azd یک نقطهٔ شروع کاری تولید می‌کند. شما *می‌توانید* بعداً آن‌ها را ویرایش کنید تا منابع بیشتری اضافه کنید یا امنیت را تقویت کنید (در [فصل 4](../chapter-04-infrastructure/README.md) پوشش داده شده است).

> **نکته:** آیا می‌خواهید زیرساخت تولیدشده را قبل از استقرار ببینید یا سفارشی‌سازی کنید؟ دستور `azd infra generate` را اجرا کنید (که به‌صورت `azd infra synth` نیز در دسترس است) تا IaC روی دیسک نوشته شود تا بتوانید آن را بازبینی و کنترل نسخه کنید.

---

## مرحلهٔ 3: تنظیم پیکربندی مورد نیاز

اگر برنامهٔ شما به تنظیمات یا اسرار نیاز دارد (مثلاً یک connection string یا API key)، آن‌ها را سخت‌کد نکنید. از مقادیر محیطی استفاده کنید:

```bash
# یک محیط ایجاد کنید
azd env new dev

# یک مقدار غیر محرمانه تنظیم کنید
azd env set API_VERSION 1.0.0
```

برای اسرار واقعی، آن‌ها را در Key Vault ذخیره کنید و از زیرساخت خود به آن‌ها ارجاع دهید — ببینید [فصل 3: پیکربندی و احراز هویت](../chapter-03-configuration/authsecurity.md).

---

## مرحلهٔ 4: استقرار

حال از همان گردش کاری که قبلاً می‌دانید استفاده کنید:

```bash
# احراز هویت (الزامی برای azd)
azd auth login

# پیش‌نمایش منابعی که ایجاد خواهند شد
azd provision --preview

# زیرساخت را فراهم کرده و کد شما را مستقر می‌کند
azd up
```

وقتی کار تمام شد، azd آدرس URL برنامهٔ شما را چاپ می‌کند. آن را به همان روشی که برای هر برنامهٔ azd انجام می‌دهید بررسی کنید:

```bash
azd show           # نمایش نقاط انتهایی
azd monitor --logs # در صورت نیاز، لاگ‌ها را بررسی کنید.
```

## مشکلات رایج برای اولین اجرا

| نشانه | علت محتمل | رفع |
|---------|--------------|-----|
| azd برنامهٔ من را شناسایی نکرد | عدم وجود manifest (مثلاً `package.json`) | manifest را اضافه کنید، یا هنگام اجرای `azd init` به‌صورت دستی میزبان را انتخاب کنید |
| ساخت در طول `azd up` شکست می‌خورد | برنامه به یک مرحلهٔ ساخت نیاز دارد | مقادیر `buildCommand`/`outputPath` را تحت سرویس در `azure.yaml` اضافه کنید |
| برنامه اجرا می‌شود اما خطا برمی‌گرداند | پیکربندی/راز مفقود است | مقادیر را با `azd env set` تنظیم کنید یا Key Vault را راه‌اندازی کنید |
| میزبان نادرست انتخاب شده | تشخیص خودکار اشتباه حدس زده | مقدار `host:` را در `azure.yaml` ویرایش کنید و دوباره `azd up` را اجرا کنید |

برای اطلاعات بیشتر، ببینید [فصل 7: رفع اشکال](../chapter-07-troubleshooting/README.md).

## پاک‌سازی

```bash
azd down --force --purge
```

---

## خلاصه

- `azd init` → **"Use code in the current directory"** باعث می‌شود azd به برنامه‌ای که قبلاً دارید اضافه شود.
- `azure.yaml` پوشه‌های کد شما را به میزبان‌های Azure نگاشت می‌کند؛ `infra/` منابع را به‌صورت Bicep تعریف می‌کند.
- `azd infra generate` به شما اجازه می‌دهد زیرساخت تولیدشده را بازبینی یا سفارشی‌سازی کنید.
- پس از راه‌اندازی، برنامهٔ موجود شما از همان گردش کاری `azd up` / `azd down` دقیقاً مانند یک برنامهٔ مبتنی بر قالب استفاده می‌کند.
