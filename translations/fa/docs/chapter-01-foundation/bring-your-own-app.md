# برنامه خود را بیاورید - افزودن azd به یک پروژه موجود

**هدایت فصل:**
- **📚 خانه دوره**: [AZD برای مبتدیان](../../README.md)
- **📖 فصل جاری**: فصل 1 - بنیاد و شروع سریع
- **⬅️ قبلی**: [اولین پروژه شما](first-project.md)
- **➡️ بعدی**: [کانتینرهای توسعه و کداسپیس‌ها](dev-containers.md)

> اعتبارسنجی شده در برابر `azd 1.27.1` در جولای 2026.

## معرفی

در [اولین پروژه شما](first-project.md) شما یک برنامه را با شروع از یک قالب مستقر کردید. اما اغلب اوقات شما قبلاً *یک* برنامه دارید—یک API نود.جی‌اس، یک سرویس Flask پایتون، یک برنامه وب .NET—که در یک پوشه روی دستگاه شما نشسته است. این درس نشان می‌دهد چگونه azd را به آن کد موجود اضافه کنید تا بتوانید با `azd up` آن را مستقر کنید، بدون نیاز به قالب.

## اهداف یادگیری

تا پایان این درس، شما:
- سه روش شروع یک پروژه azd را درک خواهید کرد
- اجرای `azd init` درون یک کدبیس موجود
- می‌فهمید `azure.yaml` و پوشه `infra/` برای برنامه شما چه کاری انجام می‌دهند
- می‌دانید چه زمانی بگذارید azd زیرساخت را تولید کند و چه زمانی خودتان بنویسید
- برنامه موجود خود را با `azd up` در Azure مستقر کنید

## نتایج یادگیری

پس از اتمام این درس، شما قادر خواهید بود:
- azd را در پروژه‌ای که قبلاً دارید مقداردهی اولیه کنید
- یک فایل پایه `azure.yaml` را بخوانید و ویرایش کنید
- زیرساخت شروع‌کننده را با `azd infra generate` تولید کنید
- یک میزبان مناسب Azure برای برنامه خود انتخاب کنید
- برنامه خود را مستقر و پاک‌سازی کنید

---

## سه روش برای شروع یک پروژه azd

| نقطه شروع | فرمان | زمان استفاده |
|----------------|---------|-------------|
| **از یک قالب** | `azd init --template <name>` | یادگیری یا شروع برنامه جدید از نمونه معتبر |
| **از کد موجود شما** | `azd init` (در پوشه پروژه شما) | شما قبلاً یک برنامه دارید و می‌خواهید آن را مستقر کنید |
| **از یک مخزن گیت** | `azd init --from-code` (در مخزن کلون شده) | پذیرش azd برای یک مخزن موجود |

شما گزینه اول را قبلاً تمرین کرده‌اید. این درس مورد دوم را پوشش می‌دهد—رایج‌ترین سناریوی دنیای واقعی.

---

## مرحله 1: اجرای `azd init` در پروژه شما

یک ترمینال **درون پوشه پروژه موجود خود** باز کنید و اجرا کنید:

```bash
cd my-existing-app
azd init
```

azd از شما می‌پرسد چگونه می‌خواهید مقداردهی اولیه کنید. انتخاب کنید:

```
? How do you want to initialize your app?
> Use code in the current directory
  Select a template
```

گزینه **"استفاده از کد در دایرکتوری فعلی."** را انتخاب کنید. azd سپس پوشه شما را اسکن می‌کند، زبان و چارچوب شما را تشخیص می‌دهد و یک میزبان پیشنهاد می‌دهد.

### آنچه azd تشخیص می‌دهد

azd به دنبال سیگنال‌هایی مانند `package.json`، `requirements.txt`، `pom.xml`، `*.csproj`، یا یک `Dockerfile` می‌گردد و یک میزبان Azure مناسب پیشنهاد می‌دهد:

| برنامه شما | میزبان احتمالی تشخیص داده شده |
|----------|----------------------|
| برنامه Node.js / Python / .NET وب | Azure App Service یا Container Apps |
| برنامه کانتینری (با `Dockerfile`) | Azure Container Apps |
| برنامه تابع | Azure Functions |
| سایت ثابت (خروجی ساخت React/Vue) | Azure Static Web Apps |

سرویس(های) تشخیص داده شده را تایید کنید و azd فایل‌های مورد نیاز شما را ایجاد می‌کند.

---

## مرحله 2: درک آنچه azd ایجاد کرده است

پس از init، دو چیز جدید در پروژه شما خواهید داشت:

```
my-existing-app/
├── azure.yaml          # ← NEW: tells azd what to deploy and where
├── infra/              # ← NEW: Infrastructure as Code (Bicep by default)
│   ├── main.bicep
│   └── main.parameters.json
├── src/ (your code)    # unchanged
└── ...                 # your existing files, untouched
```

### `azure.yaml` – تعریف پروژه

این قلب یک پروژه azd است. یک نمونه حداقلی به این شکل است:

```yaml
# azure.yaml
name: my-existing-app
services:
  web:
    project: ./src           # path to your app code
    language: js             # js | python | dotnet | java
    host: appservice         # appservice | containerapp | function | staticwebapp
```

بلوک `services` بخش کلیدی است: هر ورودی یک پوشه از کد شما را به یک میزبان Azure نگاشت می‌کند. اگر برنامه شما هر دو بخش frontend و API دارد، دو سرویس خواهید داشت.

### `infra/` – منابع Azure شما به عنوان کد

پوشه `infra/` فایل‌های Bicep را نگه می‌دارد که منابع Azure مورد نیاز برنامه شما را تعریف می‌کنند (اپ سرویس، پایگاه داده، و غیره). شما لازم نیست آنها را دستی بنویسید — azd یک نقطه شروع کارآمد تولید می‌کند. شما *می‌توانید* بعداً آنها را ویرایش کنید تا منابع اضافه یا امنیت را تقویت کنید (در [فصل 4](../chapter-04-infrastructure/README.md) پوشش داده شده است).

> **نکته:** می‌خواهید زیرساخت ایجاد شده را قبل از استقرار مشاهده یا سفارشی کنید؟ دستور `azd infra generate` (که به صورت `azd infra synth` هم موجود است) را اجرا کنید تا IaC روی دیسک نوشته شود و بتوانید آن را بازبینی و نسخه‌بندی کنید.

---

## مرحله 3: تنظیم پیکربندی مورد نیاز

اگر برنامه شما نیاز به تنظیمات یا اسرار دارد (مثلاً رشته اتصال، کلید API)، آنها را سخت‌کد نکنید. از مقادیر محیطی استفاده کنید:

```bash
# ایجاد یک محیط
azd env new dev

# تنظیم یک مقدار غیر مخفی
azd env set API_VERSION 1.0.0
```

برای اسرار واقعی، آنها را در Key Vault ذخیره کرده و از زیرساخت خود به آنها ارجاع دهید—ببینید [فصل 3: پیکربندی و احراز هویت](../chapter-03-configuration/authsecurity.md).

---

## مرحله 4: استقرار

اکنون همان جریان کاری را که می‌دانید استفاده کنید:

```bash
# احراز هویت (الزامی برای azd)
azd auth login

# پیش‌نمایش منابعی که ایجاد خواهند شد
azd provision --preview

# فراهم‌سازی زیرساخت و استقرار کد شما
azd up
```

وقتی تمام شد، azd آدرس URL برنامه شما را چاپ می‌کند. همانند هر برنامه azd آن را بررسی کنید:

```bash
azd show           # نمایش نقاط پایانی
azd monitor --logs # در صورت نیاز به بررسی لاگ‌ها
```

## مشکلات رایج بار اول

| نشانه | علت احتمالی | راه حل |
|---------|--------------|-----|
| azd برنامه من را تشخیص نداد | کمبودmanifest (مثلاً `package.json`) | manifest را اضافه کنید یا در زمان `azd init` میزبان را دستی انتخاب کنید |
| ساخت در هنگام `azd up` شکست می‌خورد | برنامه نیاز به مرحله ساخت دارد | `buildCommand`/`outputPath` را در قسمت سرویس در `azure.yaml` اضافه کنید |
| برنامه شروع می‌شود ولی خطا برمی‌گرداند | کمبود تنظیم/راز | مقادیر را با `azd env set` تنظیم کنید یا Key Vault را وصل کنید |
| میزبان اشتباه انتخاب شده | حدس خودکار | `host:` را در `azure.yaml` ویرایش کنید و دوباره `azd up` را اجرا کنید |

برای موارد بیشتر، ببینید [فصل 7: رفع مشکل](../chapter-07-troubleshooting/README.md).

## پاک‌سازی

```bash
azd down --force --purge
```

---

## خلاصه

- `azd init` → **"استفاده از کد در دایرکتوری فعلی"** azd را به برنامه‌ای که قبلاً دارید اضافه می‌کند.
- `azure.yaml` پوشه‌های کد شما را به میزبان‌های Azure نگاشت می‌کند؛ `infra/` منابع را به صورت Bicep تعریف می‌کند.
- `azd infra generate` اجازه می‌دهد زیرساخت تولید شده را مرور یا سفارشی کنید.
- پس از مقداردهی اولیه، برنامه موجود شما با همان جریان کاری `azd up` / `azd down` همانند برنامه مبتنی بر قالب استفاده می‌کند.
