## مروری

این فصل به شما کمک می‌کند که هنگام کار با Azure Developer CLI مشکلات رایج را تشخیص داده و برطرف کنید. از شکست‌های استقرار تا مشکلات خاص سرویس‌های هوش مصنوعی.

> اعتبارسنجی شده در برابر `azd 1.23.12` در مارس ۲۰۲۶.

## اهداف یادگیری

با کامل کردن این فصل، شما خواهید:
- تشخیص شکست‌های رایج استقرار AZD
- رفع اشکال مشکلات احراز هویت و مجوزها
- رفع مشکلات اتصال سرویس‌های هوش مصنوعی
- استفاده از پورتال Azure و CLI برای عیب‌یابی

## 🚨 راه‌حل‌های سریع

### مشکلات احراز هویت
```bash
# برای گردش‌کارهای AZD لازم است
azd auth login

# اگر همچنین مستقیماً از دستورات Azure CLI استفاده می‌کنید، اختیاری است
az login

azd auth status
```

### خطاهای فراهم‌سازی
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### تداخل منابع
```bash
azd down --force --purge
azd env new different-name
azd up
```

### تجاوز از سهمیه
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 مرجع کدهای خطا

| خطا | علت | راه‌حل |
|-------|-------|----------|
| `AuthenticationError` | وارد نشده‌اید | `azd auth login` |
| `ResourceNotFound` | منبع پیدا نشد | نام منابع را بررسی کنید |
| `QuotaExceeded` | محدودیت‌های اشتراک | درخواست افزایش سهمیه دهید |
| `InvalidTemplate` | خطای نحو Bicep | `az bicep build` |
| `Conflict` | منبع وجود دارد | از نام جدید استفاده کنید یا حذف کنید |
| `Forbidden` | مجوزهای ناکافی | نقش‌های RBAC را بررسی کنید |

## 🔄 بازنشانی و بازیابی

```bash
# بازنشانی نرم (حفظ منابع، استقرار مجدد کد)
azd deploy --force

# بازنشانی سخت (حذف همه‌چیز، شروع از نو)
azd down --force --purge
azd up
```
