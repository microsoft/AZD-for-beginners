> تایید شده با نسخه `azd 1.27.1` در جولای ۲۰۲۶.

## اهداف یادگیری

با کامل کردن این فصل، شما قادر خواهید بود:
- تشخیص خطاهای رایج استقرار AZD

- حل مشکلات اتصال سرویس AI
- استفاده از پرتال Azure و CLI برای عیب‌یابی

## 🚨 رفع سریع

### مشکلات احراز هویت
```bash
# مورد نیاز برای گردش‌های کاری AZD
azd auth login

# اختیاری است اگر شما همچنین به طور مستقیم از دستورات Azure CLI استفاده می‌کنید
az login

azd auth status
```

### خطاهای پروویژنینگ
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### تعارض منابع
```bash
azd down --force --purge
azd env new different-name
azd up
```

### تجاوز از سهمیه
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 مرجع کد خطا

| خطا | علت | راه‌حل |
|-------|-------|----------|
| `AuthenticationError` | ورود انجام نشده است | اجرای `azd auth login` |
| `ResourceNotFound` | منبع موجود نیست | نام منابع را بررسی کنید |
| `QuotaExceeded` | محدودیت‌های اشتراک | درخواست افزایش سهمیه دهید |
| `InvalidTemplate` | خطای نحوی بایسک | اجرای `az bicep build` |
| `Conflict` | منبع وجود دارد | از نام جدید استفاده کنید یا حذف کنید |
| `Forbidden` | دسترسی ناکافی | نقش‌های RBAC را بررسی کنید |

## 🔄 بازنشانی و بازیابی

```bash
# بازنشانی نرم (حفظ منابع، بازاستقرار کد)
azd deploy --force

# بازنشانی سخت (حذف همه چیز، شروع از نو)
azd down --force --purge
azd up
```
