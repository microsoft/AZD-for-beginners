**📚 دوره**: [AZD برای مبتدیان](../../README.md) | **⏱️ مدت**: ۴۵–۶۰ دقیقه | **⭐ پیچیدگی**: متوسط

## نمای کلی

این فصل پیکربندی محیط، الگوهای احراز هویت، و بهترین شیوه‌های امنیتی برای استقرارها با Azure Developer CLI را پوشش می‌دهد.

> تأیید شده با `azd 1.23.12` در مارس ۲۰۲۶.

## اهداف یادگیری

با تکمیل این فصل، شما:
- بر سلسله‌مراتب پیکربندی AZD تسلط پیدا خواهید کرد
- مدیریت چندین محیط (dev، staging، prod)
- پیاده‌سازی احراز هویت امن با هویت‌های مدیریت‌شده
- پیکربندی تنظیمات مخصوص هر محیط

## 🚀 شروع سریع

```bash
# ایجاد محیط‌های متعدد
azd env new dev
azd env new staging
azd env new prod

# تغییر بین محیط‌ها
azd env select prod

# تنظیم متغیرهای محیطی
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# مشاهده پیکربندی
azd env get-values
```

---

## 🔧 سلسله‌مراتب پیکربندی

AZD تنظیمات را به این ترتیب اعمال می‌کند (موارد بعدی موارد قبلی را بازنویسی می‌کنند):

1. **مقدارهای پیش‌فرض** (ساخته‌شده در قالب‌ها)
2. **azure.yaml** (پیکربندی پروژه)
3. **متغیرهای محیطی** (`azd env set`)
4. **پرچم‌های خط فرمان** (`--location eastus`)

---

## 🔐 بهترین شیوه‌های امنیتی

```bash
# از شناسهٔ مدیریت‌شده استفاده کنید (توصیه‌شده)
azd env set AZURE_USE_MANAGED_IDENTITY true

# وضعیت احراز هویت AZD را بررسی کنید
azd auth status

# اختیاری: در صورتی که قصد اجرای دستورات az را دارید، زمینهٔ Azure CLI را بررسی کنید
az account show

# در صورت نیاز دوباره احراز هویت کنید
azd auth login

# اختیاری: احراز هویت Azure CLI را برای دستورات az تازه کنید
az login
```
