**📚 دوره**: [AZD برای مبتدیان](../../README.md) | **⏱️ مدت زمان**: ۴۵-۶۰ دقیقه | **⭐ سطح دشواری**: متوسط

## مرور کلی

این فصل شامل پیکربندی محیط، الگوهای احراز هویت و بهترین شیوه‌های امنیتی برای استقرارهای خط فرمان توسعه‌دهنده Azure است.

> اعتبارسنجی شده با `azd 1.27.1` در جولای ۲۰۲۶.

## اهداف یادگیری

با اتمام این فصل، شما قادر خواهید بود:
- تسلط بر سلسله مراتب پیکربندی AZD
- مدیریت چندین محیط (توسعه، پیش‌تولید، تولید)
- پیاده‌سازی احراز هویت ایمن با هویت‌های مدیریت‌شده
- پیکربندی تنظیمات خاص محیط

## 🚀 شروع سریع

```bash
# ایجاد چندین محیط
azd env new dev
azd env new staging
azd env new prod

# سوئیچ محیط‌ها
azd env select prod

# تنظیم متغیرهای محیطی
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# مشاهده پیکربندی
azd env get-values
```

---

## 🔧 سلسله مراتب پیکربندی

AZD تنظیمات را به این ترتیب اعمال می‌کند (تنظیمات بعدی جایگزین قبلی می‌شوند):

۱. **مقادیر پیش‌فرض** (در قالب‌ها تعبیه شده)
۲. **azure.yaml** (پیکربندی پروژه)
۳. **متغیرهای محیطی** (`azd env set`)
۴. **پرچم‌های خط فرمان** (`--location eastus`)

---

## 🔐 بهترین شیوه‌های امنیتی

```bash
# از هویت مدیریت شده استفاده کنید (توصیه شده)
azd env set AZURE_USE_MANAGED_IDENTITY true

# وضعیت احراز هویت AZD را بررسی کنید
azd auth status

# اختیاری: اگر قصد اجرای دستورات az را دارید، زمینه Azure CLI را تأیید کنید
az account show

# در صورت نیاز دوباره احراز هویت کنید
azd auth login

# اختیاری: احراز هویت Azure CLI را برای دستورات az تازه کنید
az login
```
