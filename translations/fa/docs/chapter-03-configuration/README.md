## بررسی اجمالی

این فصل پیکربندی محیط، الگوهای احراز هویت و بهترین شیوه‌های امنیتی برای استقرار با Azure Developer CLI را پوشش می‌دهد.

> اعتبارسنجی شده با `azd 1.25.6` در ژوئن 2026.

## اهداف یادگیری

با تکمیل این فصل، شما خواهید توانست:
- تسلط بر سلسله مراتب پیکربندی AZD
- مدیریت محیط‌های متعدد (dev, staging, prod)
- پیاده‌سازی احراز هویت ایمن با شناسه‌های مدیریت‌شده
- پیکربندی تنظیمات مخصوص هر محیط

## 🚀 شروع سریع

```bash
# ایجاد چندین محیط
azd env new dev
azd env new staging
azd env new prod

# تغییر بین محیط‌ها
azd env select prod

# تنظیم متغیرهای محیطی
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# مشاهده پیکربندی
azd env get-values
```

---

## 🔧 سلسله مراتب پیکربندی

AZD تنظیمات را به این ترتیب اعمال می‌کند (موارد بعدی بر موارد قبلی اولویت دارند):

1. **مقادیر پیش‌فرض** (ساخته‌شده در قالب‌ها)
2. **azure.yaml** (پیکربندی پروژه)
3. **متغیرهای محیطی** (`azd env set`)
4. **گزینه‌های خط فرمان** (`--location eastus`)

---

## 🔐 بهترین شیوه‌های امنیتی

```bash
# از هویت مدیریت‌شده استفاده کنید (توصیه‌شده)
azd env set AZURE_USE_MANAGED_IDENTITY true

# وضعیت احراز هویت AZD را بررسی کنید
azd auth status

# اختیاری: در صورت قصد اجرای دستورات az، زمینهٔ Azure CLI را تأیید کنید
az account show

# در صورت نیاز، دوباره احراز هویت کنید
azd auth login

# اختیاری: احراز هویت Azure CLI را برای دستورات az تازه‌سازی کنید
az login
```
