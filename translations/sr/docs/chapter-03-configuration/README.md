Након завршетка овог поглавља, ви ћете:
- Савладати AZD хијерархију конфигурације
- Управљати више окружења (dev, staging, prod)
- Имплементирати сигурну аутентификацију са управљаним идентитетима
- Конфигурисати подешавања специфична за окружење

## 🚀 Брзи почетак

```bash
# Креирајте више окружења
azd env new dev
azd env new staging
azd env new prod

# Пређите између окружења
azd env select prod

# Поставите променљиве окружења
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# Прикажите конфигурацију
azd env get-values
```

---

## 🔧 Хијерархија конфигурације

AZD примењује подешавања у овом редоследу (каснија преовлађују над ранијим):

1. **Подразумеване вредности** (уграђене у шаблоне)
2. **azure.yaml** (конфигурација пројекта)
3. **Променљиве окружења** (`azd env set`)
4. **Опције командне линије** (`--location eastus`)

---

## 🔐 Најбоље праксе безбедности

```bash
# Користите управљани идентитет (препоручено)
azd env set AZURE_USE_MANAGED_IDENTITY true

# Проверите статус аутентификације
azd auth whoami
az account show

# Поново се аутентификујте ако је потребно
azd auth login
az login
```
