## Преглед

Ово поглавље обухвата конфигурацију окружења, шаблоне аутентификације и најбоље праксе безбедности за имплементације помоћу Azure Developer CLI.

> Верификовано против `azd 1.25.6` у јуну 2026.

## Циљеви учења

Завршетком овог поглавља, ви ћете:
- Савладати хијерархију конфигурације AZD
- Управљати више окружења (dev, staging, prod)
- Имплементирати безбедну аутентификацију помоћу управљених идентитета
- Конфигурисати подешавања специфична за окружење

## 🚀 Брзи почетак

```bash
# Креирај више окружења
azd env new dev
azd env new staging
azd env new prod

# Пређи између окружења
azd env select prod

# Постави променљиве окружења
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# Прикажи конфигурацију
azd env get-values
```

---

## 🔧 Хијерархија конфигурације

AZD примењује подешавања по овом редоследу (каснија превага над ранијима):

1. **Подразумеване вредности** (уграђене у шаблоне)
2. **azure.yaml** (конфигурација пројекта)
3. **Променљиве окружења** (`azd env set`)
4. **Опције командне линије** (`--location eastus`)

---

## 🔐 Најбоље праксе безбедности

```bash
# Користите управљани идентитет (препоручено)
azd env set AZURE_USE_MANAGED_IDENTITY true

# Проверите статус аутентификације AZD
azd auth status

# Опционо: проверите контекст Azure CLI ако планирате да покрећете az команде
az account show

# Поново се аутентификујте ако је потребно
azd auth login

# Опционо: обновите аутентификацију Azure CLI за az команде
az login
```
