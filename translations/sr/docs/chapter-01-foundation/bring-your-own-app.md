> Проверавано са `azd 1.27.1` у јулу 2026.

## Увод

У [Ваш први пројекат](first-project.md) сте распоредили апликацију полазећи од шаблона. Али већином у напретку већ *имате* апликацију — Node.js API, Python Flask сервис, .NET веб апликацију — која се налази у фасцикли на вашем рачунару. Ова лекција приказује како да додате азд у тај постојећи код како бисте је могли распоредити помоћу `azd up`, без потребе за шаблоном.

## Циљеви учења

На крају ове лекције ћете:
- Разумети три начина за покретање азд пројекта
- Покренути `azd init` унутар постојеће кодне базе
- Разумети шта раде `azure.yaml` и фасцикла `infra/` за вашу апликацију
- Знати када дозволити азд-у да генерише инфраструктуру, а када сами писати
- Распоредити вашу постојећу апликацију на Azure са `azd up`

## Резултати учења

Након завршетка ове лекције, моћи ћете да:
- Иницијализујете азд у пројекту који већ имате
- Читате и уређујете основни `azure.yaml` фајл
- Генеришете почетну инфраструктуру са `azd infra generate`
- Изаберете одговарајући Azure хост за вашу апликацију
- Распоредите и очистите вашу апликацију

---

## Три начина за покретање азд пројекта

| Почетна тачка | Команда | Када користити |
|----------------|---------|-------------|
| **Из шаблона** | `azd init --template <name>` | За учење или почетак нове апликације из провереног примерка |
| **Из постојећег кода** | `azd init` (у фасцикли вашег пројекта) | Кад већ имате апликацију коју желите да распоредите |
| **Из Git репозиторијума** | `azd init --from-code` (у клонованом репозиторијуму) | Примена азд на постојећи репозиторијум |

Већ сте вежбали прву опцију. Ова лекција покрива другу — најчешћи сценарио у стварном свету.

## Корак 1: Покрените `azd init` у свом пројекту

Отворите терминал **унутар фасцикле вашег постојећег пројекта** и покрените:

```bash
cd my-existing-app
azd init
```

азд ће питати како желите да иницијализујете. Изаберите:

```
? How do you want to initialize your app?
> Use code in the current directory
  Select a template
```

Изаберите **"Use code in the current directory."** азд ће онда скенирати ваш фолдер, открити језик и оквир и предложити хост.

### Шта азд открива

азд тражи сигнале као `package.json`, `requirements.txt`, `pom.xml`, `*.csproj` или `Dockerfile`, и предлаже одговарајући Azure хост:

| Ваша апликација | Веројатно откривени хост |
|----------|----------------------|
| Node.js / Python / .NET веб апликација | Azure App Service или Container Apps |
| Апликација у контејнеру (`Dockerfile`) | Azure Container Apps |
| Функцијска апликација | Azure Functions |
| Статички сајт (React/Vue build резултат) | Azure Static Web Apps |

Потврдите детектовану услугу/услуге, и азд ће направити потребне фајлове.

---

## Корак 2: Разумите шта је азд креирао

Након иницијализације бићете у пројекту са два нова дела:

```
my-existing-app/
├── azure.yaml          # ← NEW: tells azd what to deploy and where
├── infra/              # ← NEW: Infrastructure as Code (Bicep by default)
│   ├── main.bicep
│   └── main.parameters.json
├── src/ (your code)    # unchanged
└── ...                 # your existing files, untouched
```

### `azure.yaml` — дефиниција пројекта

Ово је срж азд пројекта. Минимални пример изгледа овако:

```yaml
# azure.yaml
name: my-existing-app
services:
  web:
    project: ./src           # path to your app code
    language: js             # js | python | dotnet | java
    host: appservice         # appservice | containerapp | function | staticwebapp
```

Блок `services` је кључни део: сваки унос упућује на фолдер вашег кода за одређени Azure хост. Ако ваша апликација има и фронтенд и API, имаћете две услуге.

### `infra/` — ваши Azure ресурси као код

Фасцикла `infra/` садржи Bicep фајлове који дефинишу Azure ресурсе потребне вашој апликацији (App Service, базу података итд.). Не морате то ручно писати — азд генерише радну почетну тачку. Можете касније да их уређујете да додате ресурсе или пооштрите безбедност (обрађено у [Поглављу 4](../chapter-04-infrastructure/README.md)).

> **Савет:** Желите да видите или прилагодите генерисану инфраструктуру пре распореда? Покрените `azd infra generate` (такође доступно као `azd infra synth`) да бисте написали IaC на диск ради прегледа и контроле верзија.

## Корак 3: Поставите потребну конфигурацију

Ако вашој апликацији требају подешавања или тајне (на пример конекциони низ, API кључ), немојте их тврдописати. Користите вредности окружења:

```bash
# Направи окружење
azd env new dev

# Постави јавну вредност
azd env set API_VERSION 1.0.0
```

За праве тајне, чувајте их у Key Vault и позивајте их из инфраструктуре — видети [Поглавље 3: Конфигурација и аутентификација](../chapter-03-configuration/authsecurity.md).

---

## Корак 4: Распоредите

Сада користите исти ток рада који већ знате:

```bash
# Аутентификујте се (потребно за azd)
azd auth login

# Прегледајте ресурсе који ће бити креирани
azd provision --preview

# Обезбедите инфраструктуру и поставите ваш код
azd up
```

Када заврши, азд приказује URL ваше апликације. Потврдите је на исти начин као и било коју азд апликацију:

```bash
azd show           # прикажи крајње тачке
azd monitor --logs # проверите записнике ако је потребно
```

## Чести проблеми при првом покретању

| Симптом | Веројатни узрок | Решавање |
|---------|--------------|-----|
| азд није открио моју апликацију | Недостаје манифест (нпр. `package.json`) | Додајте манифест или изаберите хост ручно током `azd init` |
| Грешка при изградњи током `azd up` | Апликацији треба корак изградње | Додајте `buildCommand`/`outputPath` у сервис у `azure.yaml` |
| Апликација почиње, али враћа грешке | Недостаје конфигурација/тајна | Поставите вредности помоћу `azd env set` или повежите Key Vault |
| Погрешно одабран хост | Погрешна аутоматска детекција | Уредите `host:` у `azure.yaml` и поново покрените `azd up` |

За више информација, видети [Поглавље 7: Решавање проблема](../chapter-07-troubleshooting/README.md).

---

## Чишћење

```bash
azd down --force --purge
```

---

## Резиме

- `azd init` → **"Use code in the current directory"** додаје азд у постојећу апликацију.
- `azure.yaml` мапира фасцикле вашег кода на Azure хостове; `infra/` дефинише ресурсе као Bicep.
- `azd infra generate` вам омогућава да прегледате или прилагодите генерисану инфраструктуру.
- Након иницијализације, ваша постојећа апликација користи исти ток рада `azd up` / `azd down` као апликација базирана на шаблону.
