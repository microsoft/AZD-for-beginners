# Користите сопствену апликацију - Додајте azd у постојећи пројекат

**Навигација поглавља:**
- **📚 Почетна странице курса**: [AZD за почетнике](../../README.md)
- **📖 Тренутно поглавље**: Поглавље 1 - Основе и брзи почетак
- **⬅️ Претходно**: [Ваш први пројекат](first-project.md)
- **➡️ Следеће**: [Dev Containers & Codespaces](dev-containers.md)

> Проверено против `azd 1.25.6` у јуну 2026.

## Увод

У [Ваш први пројекат](first-project.md) разврнули сте апликацију почевши од шаблона. Али већином случајева већ имате апликацију—Node.js API, Python Flask сервис, .NET веб апликацију—која седи у фолдеру на вашем рачунару. Ова лекција показује како да додате azd у тај постојећи код тако да можете да је деплујујете помоћу `azd up`, без потребе за шаблоном.

## Циљеви учења

До краја ове лекције, видећете како да:
- Разумете три начина за покретање azd пројекта
- Покренете `azd init` унутар постојећег кода
- Разумете шта раде `azure.yaml` и фолдер `infra/` за вашу апликацију
- Знате када дозволити azd-у да генерише инфраструктуру уместо да напишете своју
- Деплујете вашу постојећу апликацију на Azure помоћу `azd up`

## Исходи учења

Након завршетка ове лекције, моћи ћете да:
- Иницијализујете azd у пројекту који већ имате
- Читате и уређујете основни фајл `azure.yaml`
- Генеришете почетну инфраструктуру помоћу `azd infra generate`
- Изаберете одговарајући Azure хост за вашу апликацију
- Деплујете и очистите сопствену апликацију

---

## Три начина за покретање azd пројекта

| Почетна тачка | Команда | Када користити |
|----------------|---------|-------------|
| **Из шаблона** | `azd init --template <name>` | За учење, или покретање нове апликације на основу провереног примера |
| **Из вашег постојећег кода** | `azd init` (у фолдеру вашег пројекта) | Већ имате апликацију и желите да је деплујете |
| **Из Git репозиторијума** | `azd init --from-code` (у клонованом репозиторијуму) | Усвајање azd-а за постојећи репозиторијум |

Вежбали сте прву опцију. Ова лекција покрива другу—најчешћи сценарио у реалном свету.

## Корак 1: Покрените `azd init` у вашем пројекту

Отворите терминал **унутар фолдера вашег постојећег пројекта** и покрените:

```bash
cd my-existing-app
azd init
```

azd ће питати како желите да иницијализујете. Изаберите:

```
? How do you want to initialize your app?
> Use code in the current directory
  Select a template
```

Одаберите **"Користи код у тренутном директоријуму."** azd затим скенира ваш фолдер, детектује ваш језик и фрејмворк, и предлаже хост.

### Шта azd детектује

azd тражи сигнале попут `package.json`, `requirements.txt`, `pom.xml`, `*.csproj`, или `Dockerfile`, и предлаже одговарајући Azure хост:

| Ваша апликација | Вероватно откривени хост |
|----------|----------------------|
| Node.js / Python / .NET веб апликација | Azure App Service или Container Apps |
| Контейнеризована апликација (`Dockerfile`) | Azure Container Apps |
| Function апликација | Azure Functions |
| Статички сајт (излаз build-а React/Vue) | Azure Static Web Apps |

Потврдите детектоване сервисе, и azd ће скелетовати фајлове који су вам потребни.

## Корак 2: Разумевање онога што je azd креирао

Након `azd init`, имаћете две нове ствари у вашем пројекту:

```
my-existing-app/
├── azure.yaml          # ← NEW: tells azd what to deploy and where
├── infra/              # ← NEW: Infrastructure as Code (Bicep by default)
│   ├── main.bicep
│   └── main.parameters.json
├── src/ (your code)    # unchanged
└── ...                 # your existing files, untouched
```

### `azure.yaml` — дефиниција пројекта

Ово је срж azd пројекта. Минимални пример изгледа овако:

```yaml
# azure.yaml
name: my-existing-app
services:
  web:
    project: ./src           # path to your app code
    language: js             # js | python | dotnet | java
    host: appservice         # appservice | containerapp | function | staticwebapp
```

Блок `services` је кључни део: сваки унос мапира фолдер вашег кода на Azure хост. Ако ваша апликација има и фронтенд и API, имаћете два сервиса.

### `infra/` — ваше Azure ресурсе као код

Фолдер `infra/` садржи Bicep фајлове који дефинишу Azure ресурсе које ваша апликација захтева (App Service, базу података итд.). Не морате да их пишете ручно—azd генерише радни почетни образац. Можете их касније уредити да додате ресурсе или пооштритe безбедност (покривено у [Поглавље 4](../chapter-04-infrastructure/README.md)).

> **Савет:** Желите да видите или прилагодите генерисану инфраструктуру пре деплоја? Покрените `azd infra generate` (такође доступно као `azd infra synth`) да бисте уписали IaC на диск тако да га можете прегледати и верзионисати.

## Корак 3: Поставите потребну конфигурацију

Ако вашој апликацији требају подешавања или тајне (connection string, API key), немојте их хардкодирати. Користите вредности из окружења:

```bash
# Креирај окружење
azd env new dev

# Постави вредност која није тајна
azd env set API_VERSION 1.0.0
```

За стварне тајне, чувате их у Key Vault-у и реферишите их из ваше инфраструктуре—видети [Поглавље 3: Конфигурација и аутентикација](../chapter-03-configuration/authsecurity.md).

---

## Корак 4: Деплој

Сада користите исти радни ток који већ познајете:

```bash
# Аутентификујте се (обавезно за azd)
azd auth login

# Прегледајте ресурсе који ће бити креирани
azd provision --preview

# Обезбедите инфраструктуру и разместите свој код
azd up
```

Када заврши, azd исписује URL ваше апликације. Веруфикујте га на исти начин као код било које azd апликације:

```bash
azd show           # прикажи крајње тачке
azd monitor --logs # провери логове ако је потребно
```

## Уобичајени проблеми при првом коришћењу

| Симптом | Вероватан узрок | Решење |
|---------|--------------|-----|
| azd није открио моју апликацију | Недостаје манифест (нпр. `package.json`) | Додајте манифест, или изаберите хост ручно током `azd init` |
| Изградња не успева током `azd up` | Апликацији је потребан корак изградње | Додајте `buildCommand`/`outputPath` под сервисом у `azure.yaml` |
| Апликација се покреће али враћа грешке | Недостајућа конфигурација/тајна | Поставите вредности помоћу `azd env set` или повежите Key Vault |
| Изабран је погрешан хост | Аутоматско откривање погрешило | Уредите `host:` у `azure.yaml` и поново покрените `azd up` |

За више, видети [Поглавље 7: Решавање проблема](../chapter-07-troubleshooting/README.md).

---

## Чишћење

```bash
azd down --force --purge
```

---

## Резиме

- `azd init` → **"Користи код у тренутном директоријуму"** додаје azd апликацији коју већ имате.
- `azure.yaml` мапира фолдере вашег кода на Azure хостове; `infra/` дефинише ресурсе као Bicep.
- `azd infra generate` вам омогућава да прегледате или прилагодите генерисану инфраструктуру.
- Једном иницијализована, ваша постојећа апликација користи исти `azd up` / `azd down` радни ток као апликација базирана на шаблону.
