## Преглед

Ово поглавље обухвата суштинске кораке планирања и валидације пре распоређивања ваше апликације. Научите како да избегнете скупе грешке уз правилно планирање капацитета, избор SKU-а и извршавање претходних провера.

## Циљеви учења

По завршетку овог поглавља, моћи ћете:
- Извршити претходне провере пре распоређивања
- Планирати капацитет и проценити захтеве за ресурсима
- Одабрати одговарајуће SKU-ове за оптимизацију трошкова
- Конфигурисати Application Insights за праћење
- Разумети шаблоне координације тима

## 🚀 Брзи почетак

```bash
# Проверите квоте претплате
az vm list-usage --location eastus --output table

# Прегледајте распоређивање (ресурси се не креирају)
azd provision --preview

# Валидирајте синтаксу Bicep-а
az bicep build --file infra/main.bicep

# Проверите конфигурацију окружења
azd env get-values
```

---

## ☑️ Контролна листа пре распоређивања

### Пре `azd provision`

- [ ] Квота проверена за регион
- [ ] SKU-ови адекватно одабрани
- [ ] Прегледана процена трошкова
- [ ] Конвенција именовања усклађена
- [ ] Безбедност/RBAC конфигурисана

### Пре `azd deploy`

- [ ] Постављене променљиве окружења
- [ ] Тајне у Key Vault-у
- [ ] Проверени connection strings
- [ ] Конфигурисане провере статуса (health checks)

## 💰 Водич за избор SKU-ова

| Радно оптерећење | Развој | Производња |
|----------|-------------|------------|
| Container апликације | Consumption | Dedicated D4 |
| App Service | B1/B2 | P1v3+ |
| Microsoft Foundry модели | Standard | Standard + PTU |
| AI Search | Basic | Standard S2+ |
