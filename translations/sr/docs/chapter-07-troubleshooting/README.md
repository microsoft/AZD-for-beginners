## Преглед

Ово поглавље вам помаже да дијагностикујете и решите уобичајене проблеме приликом рада са Azure Developer CLI. Од неуспеха при распоређивању до проблема специфичних за вештачку интелигенцију.

## Циљеви учења

Проучавањем овог поглавља, моћи ћете да:
- Дијагностикујете уобичајене неуспехе при распоређивању AZD
- Отстраните проблеме са аутентификацијом и дозволама
- Решите проблеме са повезивањем AI услуга
- Користите Azure портал и CLI за решавање проблема

## 🚨 Брза решења

### Проблеми са аутентификацијом
```bash
azd auth login
az login
azd auth whoami
```

### Неуспеси при провизији
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### Конфликти ресурса
```bash
azd down --force --purge
azd env new different-name
azd up
```

### Прекорачење квоте
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 Референца кодова грешака

| Грешка | Узрок | Решење |
|-------|-------|----------|
| `AuthenticationError` | Нисте пријављени | `azd auth login` |
| `ResourceNotFound` | Недостаје ресурс | Проверите имена ресурса |
| `QuotaExceeded` | Ограничења претплате | Затражите повећање квоте |
| `InvalidTemplate` | Синтаксна грешка у Bicep | `az bicep build` |
| `Conflict` | Ресурс већ постоји | Користите ново име или избришите |
| `Forbidden` | Недовољне дозволе | Проверите RBAC улоге |

## 🔄 Ресетовање и опоравак

```bash
# Меки ресет (сачува ресурсе, поново распоређује код)
azd deploy --force

# Тврди ресет (избриши све, почни изнова)
azd down --force --purge
azd up
```
