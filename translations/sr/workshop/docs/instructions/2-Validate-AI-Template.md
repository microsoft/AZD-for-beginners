## 1. Увод

The [Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) или `azd` је алат отвореног кода за командну линију који поједностављује радни ток развојних инжењера при изградњи и распоредању апликација на Azure.

[AZD Templates](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) су стандардизовани репозиторијуми који укључују пример кода апликације, _инфраструктуру као код_ ресурсе и `azd` конфигурационе датотеке за кохерентну архитектуру решења. Провиђирање инфраструктуре постаје једноставно као команда `azd provision` - док `azd up` омогућава да провиђирате инфраструктуру **и** распоредите вашу апликацију у једном кораку!

Као резултат, покретање развоја апликације може бити једноставно као проналажење правог _AZD Starter шаблона_ који највише одговара вашим потребама апликације и инфраструктуре - а затим прилагођавање репозиторијума да одговара вашим захтевима сценарија.

Пре него што почнемо, хајде да проверимо да ли имате инсталиран Azure Developer CLI.

1. Отворите терминал у VS Code и унесите ову команду:

      ```bash title="" linenums="0"
      azd version
      ```

1. Требало би да видите нешто овако!

      ```bash title="" linenums="0"
      azd version 1.25.6 (commit <current-build>)
      ```

**Сада сте спремни да изаберете и распоредите шаблон са azd**

Такође можете открити ове шаблоне посетом Microsoft Foundry порталу.

## 3. Активирање шаблона

Хајде да покушамо да распоредимо овај шаблон и уверимо се да је валидан. Практиковаћемо смернице из секције [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started).

1. Изаберите радно окружење за репозиторијум шаблона:

      - **GitHub Codespaces**: Кликните [this link](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) и потврдите `Create codespace`
      - **Локални клон или dev container**: Клонирајте `Azure-Samples/get-started-with-ai-agents` и отворите га у VS Code

1. Сачекајте док терминал у VS Code не буде спреман, затим унесите следећу команду:

   ```bash title="" linenums="0"
   azd up
   ```

Попуните кораке радног тока које ће ово покренути:

1. Бићете затражени да се пријавите у Azure - пратите упутства за аутентикацију
1. Унесите јединствено име окружења за себе - на пример, ја сам користио `nitya-mshack-azd`
1. Ово ће направити фасциклу `.azure/` - видећете подпапку са именом окружења
1. Бићете упитани да одаберете име претплате - изаберите подразумевану
1. Бићете упитани за локацију - користите `East US 2`

Сада, сачекајте да провиђирање заврши. **Ово траје 10-15 минута**

1. Када се заврши, ваш конзол ће показати SUCCESS поруку као ова:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. Ваш Azure портал ће сада имати групу ресурса са тим именом окружења:

      ![Инфраструктура](../../../../../translated_images/sr/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **Сада сте спремни да верификујете распоређену инфраструктуру и апликацију**.

## 5. Валидација агента

Azure Container App распоређује ендпоинт који се повезује са AI агентом провајдованим у Microsoft Foundry пројекту за овај шаблон. Хајде да погледамо шта то значи.

1. Вратите се на Azure Portal страницу _Overview_ за вашу групу ресурса

1. Кликните на `Microsoft Foundry` ресурс у тој листи

1. Требало би да видите ово. Кликните дугме `Go to Microsoft Foundry Portal`. 
   ![Foundry](../../../../../translated_images/sr/04-view-foundry-project.fb94ca41803f28f3.webp)

1. Требало би да видите страницу Foundry пројекта за вашу AI апликацију
   ![Пројекат](../../../../../translated_images/sr/05-visit-foundry-portal.d734e98135892d7e.webp)

1. Кликните на `Agents` - видећете подразумеваног агента провиђованог у вашем пројекту
   ![Агенти](../../../../../translated_images/sr/06-visit-agents.bccb263f77b00a09.webp)

1. Изаберите га - и видећете детаље агента. Обратите пажњу на следеће:

      - агент подразумевано користи File Search (увек)
      - означено је у `Knowledge` да има отпремљено 32 датотеке (за претрагу фајлова)
      ![Агенти детаљи](../../../../../translated_images/sr/07-view-agent-details.0e049f37f61eae62.webp)

1. Потражите опцију `Data+indexes` у левом менију и кликните за детаље. 

      - Требало би да видите 32 отпремљене датотеке података за знање.
      - Оне ће одговарати 12 датотека о купцима и 20 датотека о производима у `src/files`
      ![Подaци](../../../../../translated_images/sr/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**Верификовали сте рад агента!**

1. Одговори агента се заснивају на знању из тих датотека.
1. Сада можете постављати питања везана за те податке и добијати одговоре са основом у тим подацима.
1. Примера ради: `customer_info_10.json` описује 3 куповине које је направила "Amanda Perez"

Вратите се на картицу прегледача са ендпоинтом Container App-а и поставите: `What products does Amanda Perez own?`. Требало би да видите нешто овако:

![Подаци у ACA](../../../../../translated_images/sr/09-ask-in-aca.4102297fc465a4d5.webp)

## 7. Уграђена опсервабилност

Опсервабилност се односи на инструментисање апликације да генерише податке који се могу користити за разумевање, дебаговање и оптимизацију њеног рада. Да бисте стекли утисак о томе:

1. Кликните дугме `View Run Info` - требало би да видите овај приказ. Ово је пример [Agent tracing](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) у пракси. _Овај приказ такође можете добити кликом на Thread Logs у горњем менију_.

   - Стекнете увид у кораке извршавања и алате које је агент покренуо
   - Разумете укупан број Token-а (у односу на коришћене токене за излаз) за одговор
   - Разумете латенцију и где се време троши у извршењу

      ![Агент покрет](../../../../../translated_images/sr/10-view-run-info.b20ebd75fef6a1cc.webp)

1. Кликните на картицу `Metadata` да видите додатне атрибуте извршавања, који могу пружити користан контекст за касније решавање проблема.   

      ![Агент метаподаци](../../../../../translated_images/sr/11-view-run-info-metadata.7966986122c7c2df.webp)


1. Кликните на картицу `Evaluations` да видите аутоматске процене направљене за одговор агента. Ово укључује процене безбедности (нпр. Self-harm) и специфичне оцене агента (нпр. разрешавање намера, придржавање задатка).

      ![Агент процене](../../../../../translated_images/sr/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. На крају, кликните на картицу `Monitoring` у бочном менију.

      - Изаберите картицу `Resource usage` на приказаној страници - и погледајте метрике.
      - Пратите употребу апликације у смислу трошкова (token-и) и оптерећења (захтева).
      - Пратите латенцију апликације до првог бајта (обрада улаза) и последњег бајта (излаз).

      ![Мониторинг ресурси](../../../../../translated_images/sr/13-monitoring-resources.5148015f7311807f.webp)

## 8. Променљиве окружења

До сада смо прошли постављање у прегледачу - и потврдили да је наша инфраструктура провиђована и да апликација ради. Али да бисмо радили са апликацијом _прво из кода_, потребно је да конфигуришемо наше локално развојно окружење са релевантним променљивим које су потребне за рад са овим ресурсима. Коришћење `azd` то чини једноставним.

1. Azure Developer CLI [користи променљиве окружења](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) за чување и управљање конфигурационим подешавањима за развоје апликације.

1. Променљиве окружења се чувају у `.azure/<env-name>/.env` - ово их ограничава на окружење `env-name` коришћено током распоредања и помаже да изолујете окружења између различитих циљева распореда у истом репозиторијуму.

1. Променљиве окружења се аутоматски учитавају када команда `azd` изврши одређену наредбу (нпр. `azd up`). Имајте у виду да `azd` аутоматски не чита _OS-ниво_ променљивих окружења (нпр. подешених у shell-у) - уместо тога користите `azd set env` и `azd get env` да пренесете информације унутар скрипти.

Хајде да испробамо неколико команди:

1. Добијте све променљиве окружења постављене за `azd` у овом окружењу:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      Видећете нешто овако:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. Добијте специфичну вредност - нпр. желим да знам да ли је подешена вредност `AZURE_AI_AGENT_MODEL_NAME`

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      Видећете нешто овако - није била подешена подразумевано!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. Поставите нову променљиву окружења за `azd`. Овде ажурирамо име модела агента. _Напомена: било које измене ће одмах бити видљиве у `.azure/<env-name>/.env` датотеци._

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      Сада би требало да нађемо вредност подешену:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. Имајте на уму да су неки ресурси перзистентни (нпр. распореди модела) и захтеваће више од само `azd up` да би се принудно поново распоредили. Хајде да покушамо да скинемо оригинални распоред и поново распоредимо са измењеним променљивим окружења.

1. **Освежите** Ако сте раније распореди ли инфраструктуру помоћу azd шаблона - можете _освежити_ стање локалних променљивих окружења на основу тренутног стања вашег Azure распореда помоћу следеће команде:

      ```bash title="" linenums="0"
      azd env refresh
      ```

      Ово је моћан начин да _синхронизујете_ променљиве окружења између два или више локалних развојних окружења (нпр., тим са више програмера) - омогућавајући да распоређена инфраструктура служи као коначна истина за стање променљивих окружења. Чланови тима једноставно _освежавају_ променљиве да би се поново ускладили.
