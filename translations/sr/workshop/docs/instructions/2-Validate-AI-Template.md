## 1. Увод

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) или `azd` је алат отвореног кода за командну линију који поједностављује ток рада програмера приликом прављења и деплојовања апликација на Azure.

[AZD Templates](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) су стандардизовани репозиторијуми који садрже пример кода апликација, _инфраструктуру као код_ ресурсе и конфигурационе датотеке за `azd`, за кохерентну архитектуру решења. Поред тога, обезбеђивање инфраструктуре је једноставно као команда `azd provision` - док `azd up` омогућава да једним покретом обезбедите инфраструктуру **и** деплојтујете апликацију!

Као резултат, покретање процеса развоја апликације може бити једноставно као проналажење правог _AZD Starter шаблона_ који највише одговара вашим потребама апликације и инфраструктуре - а затим прилагођавање репозиторијума за ваше сценарије.

Пре него што почнемо, уверимо се да имате инсталиран Azure Developer CLI.

1. Отворите терминал у VS Code и укуцајте ову команду:

      ```bash title="" linenums="0"
      azd version
      ```

1. Требало би да видите нешто овако!

      ```bash title="" linenums="0"
      azd version 1.27.1 (commit <current-build>)
      ```

**Сада сте спремни да изаберете и деплојтујете шаблон помоћу azd**

## 2. Избор шаблона

Microsoft Foundry платформа доноси [скид препоручених AZD шаблона](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) који покривају популарне сценарије решења као што су _мулти-агент аутоматизација тока рада_ и _обрада мулти-модалног садржаја_. Ове шаблоне такође можете пронаћи посетом Microsoft Foundry порталу.

1. Посетите [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. Пријавите се на Microsoft Foundry портал када вас систем затражи - видећете нешто овако.

![Pick](../../../../../translated_images/sr/01-pick-template.60d2d5fff5ebc374.webp)


Опције **Basic** су ваши почетни шаблони:

1. [X] [Почните са AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) који такође деплојтује стандардног AI агента (са Foundry Agents). Користите овај да се упознате са агенцијским AI решењима која укључују алате и моделе.

Посетите други линк у новом прозору прегледача (или кликните на `Open in GitHub` за повезану картицу). Требало би да видите репозиторијум за овај AZD шаблон. Погледајте README. Архитектура апликације изгледа овако:

## 3. Активирање шаблона

Покушаћемо да деплојтујемо овај шаблон и проверимо да ли је валидан. Пратићемо смернице у [Уводном делу](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started).

1. Изаберите радно окружење за репозиторијум шаблона:

      - **GitHub Codespaces**: Кликните [овде](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) и потврдите `Create codespace`
      - **Локална копија или дев контејнер**: Клонирајте `Azure-Samples/get-started-with-ai-agents` и отвори га у VS Code

1. Сачекајте да терминал у VS Code буде спреман, затим укуцајте следећу команду:

   ```bash title="" linenums="0"
   azd up
   ```

Завршите кораке у току који ће ова команда покренути:

1. Бићете упитани да се пријавите на Azure - пратите упутства за аутентификацију
1. Унесите јединствено име окружења за себе - нпр. ја сам користио `nitya-mshack-azd`
1. Ово ће креирати фасциклу `.azure/` - видећете подфасциклу са именом окружења
1. Бићете упитани да одаберете име претплате - изаберите подразумевано
1. Бићете упитани за локацију - користите `East US 2`

Сада сачекајте да се обезбеђивање заврши. **Ово траје 10-15 минута**

1. Када заврши, ваша конзола ће приказати ПОРУКУ О УСПЕХУ овако:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. Ваш Azure портал ће сада имати обезбеђену ресурсну групу са тим именом окружења:

      ![Infra](../../../../../translated_images/sr/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **Сада сте спремни да проверите деплојовану инфраструктуру и апликацију**.

## 5. Проверa агента

Azure Container App деплојтује крајњу тачку која се повезује на AI агента обезбеђеног у Microsoft Foundry пројекту за овај шаблон. Погледајмо шта то значи.

1. Вратите се на Azure портал страницу _Overview_ за вашу ресурсну групу

1. Кликните на ресурс `Microsoft Foundry` у тој листи

1. Требало би да видите ово. Кликните на дугме `Go to Microsoft Foundry Portal`.
   ![Foundry](../../../../../translated_images/sr/04-view-foundry-project.fb94ca41803f28f3.webp)

1. Требало би да видите страницу на Foundry порталу за вашу AI апликацију
   ![Project](../../../../../translated_images/sr/05-visit-foundry-portal.d734e98135892d7e.webp)

1. Кликните на `Agents` - видећете подразумеваног агента обезбеђеног у вашем пројекту
   ![Agents](../../../../../translated_images/sr/06-visit-agents.bccb263f77b00a09.webp)

1. Изаберите га - видећете детаље о агенту. Обратите пажњу на следеће:

      - Агенат по подразумеваној вредности користи претрагу фајлова (увек)
      - Подаци у „Knowledge“ показују да има 32 отпремљена фајла (за претрагу фајлова)
      ![Agents](../../../../../translated_images/sr/07-view-agent-details.0e049f37f61eae62.webp)

1. Потражите опцију `Data+indexes` у левом менију и кликните да бисте видели детаље.

      - Требало би да видите 32 отпремљена податка за знање.
      - Ово одговара 12 фајлова клијената и 20 фајлова производа у `src/files`
      ![Data](../../../../../translated_images/sr/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**Ви сте верификовали рад агента!**

1. Одговори агента се заснивају на знању из тих фајлова.
1. Сада можете постављати питања везана за те податке и добијати изворно засноване одговоре.
1. Пример: `customer_info_10.json` описује три куповине које је направила "Amanda Perez".

Вратите се на картицу претраживача са крајњом тачком Container App-а и питајте: `Које производе поседује Amanda Perez?`. Требало би да видите нешто овако:

## 7. Уграђена посматрачност

Посматрачност значи инструментализацију апликације да генерише податке који се могу користити за разумевање, исправку и оптимизацију њеног рада. Да бисте то осетили:

1. Кликните на дугме `View Run Info` - требало би да видите овај приказ. Ово је пример [траговања агента](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) у акцији. _Овај приказ можете добити и кликом на Thread Logs у горњем менију_.

   - Схватите кораке извођења и алате које агент користи
   - Разумете укупан број токена (у поређењу са бројем излазних токена) за одговор
   - Разумете кашњење и где се време троши у извршењу

      ![Agent](../../../../../translated_images/sr/10-view-run-info.b20ebd75fef6a1cc.webp)

1. Кликните на картицу `Metadata` да видите додатне атрибуте извођења, који могу пружити користан контекст за касније исправке грешака.

      ![Agent](../../../../../translated_images/sr/11-view-run-info-metadata.7966986122c7c2df.webp)


1. Кликните на картицу `Evaluations` да видите аутоматске процене агента о одговору. Ово укључује евалуације безбедности (нпр. Самооштрење) и специфичне процене агента (нпр. Решавање намере, придржавање задатака).

      ![Agent](../../../../../translated_images/sr/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. На крају, али не и најмање важно, кликните на картицу `Monitoring` у бочном менију.

      - Изаберите картицу `Resource usage` на приказаној страници - и прегледајте метрике.
      - Пратите коришћење апликације у смислу трошкова (токени) и оптерећења (захтеви).
      - Пратите кашњење апликације до првог бајта (обрада уноса) и последњег бајта (излаз).

      ![Agent](../../../../../translated_images/sr/13-monitoring-resources.5148015f7311807f.webp)

## 8. Променљиве окружења

До сада смо шетали кроз деплојмент у прегледачу - и проверили да је наша инфраструктура обезбеђена и апликација оперативна. Али да бисмо радили са апликацијом _прво из кода_, треба да конфигуришемо наше локално развојно окружење са релевантним променљивим које су потребне за рад са тим ресурсима. Kоришћење `azd` то олакшава.

1. Azure Developer CLI [користи променљиве окружења](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) за чување и управљање конфигурационим подешавањима за деплојмент апликација.

1. Променљиве окружења се чувају у `.azure/<env-name>/.env` - ово их ограничава на окружење `env-name` коришћено током деплојмента и помаже да изолујете окружења између различитих места деплоја у истом репозиторијуму.

1. Променљиве окружења аутоматски учитава команда `azd` кад год се изврши специфична команда (нпр. `azd up`). Имајте у виду да `azd` аутоматски не чита _системске_ променљиве окружења (нпр. оне подешене у шкољци) - уместо тога користите `azd set env` и `azd get env` за пренос информација у скриптама.


Хајде да испробамо неколико команди:

1. Добијте све променљиве окружења подешене за `azd` у овом окружењу:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      Видећете нешто овако:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. Добијте специфичну вредност - нпр. желим да знам да ли је постављена вредност `AZURE_AI_AGENT_MODEL_NAME`

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      Видећете нешто овако - није подешено подразумевано!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. Поставите нову променљиву окружења за `azd`. Овде ажурирамо име модела агента. _Напомена: све промене ће одмах бити приказане у фајлу `.azure/<env-name>/.env`.

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      Сада би требало да видимо да је вредност подешена:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. Имајте у виду да су неки ресурси трајни (нпр. деплојменти модела) и захтеваће више од једног `azd up` да се поново деплојтују. Покушајмо да скинемо првобитни деплојмент и поново га поставимо са промењеним вредностима окружења.

1. **Освежите** Уколико сте раније постављали инфраструктуру користећи azd шаблон - можете _освежити_ стање својих локалних променљивих окружења на основу тренутног стања Azure деплојмента коришћењем ове команде:

      ```bash title="" linenums="0"
      azd env refresh
      ```


      Ово је моћан начин да _синхронизујете_ променљиве окружења између два или више локалних развојних окружења (нпр. тим са више програмера) - омогућавајући да распоређена инфраструктура служи као поуздани извор стања променљивих окружења. Чланови тима једноставно _освежавају_ променљиве да би поново били синхронизовани.
