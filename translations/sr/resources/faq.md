## Почетак рада

### Q: Шта је Azure Developer CLI (azd)?
**A**: Azure Developer CLI (azd) је командно-линијски алат усмерен на програмере који убрзава преношење ваше апликације из локалног окружења за развој у Azure. Пружа најбоље праксе кроз шаблоне и помаже током целог живота распореда.

### Q: По чему се azd разликује од Azure CLI?
**A**: 
- **Azure CLI**: Алат општег назначења за управљање Azure ресурсима
- **azd**: Алат фокусиран на програмере за токове рада распоређивања апликација
- azd користи Azure CLI унутар себе, али пружа апстракције вишег нивоа за уобичајене сценарије развоја
- azd укључује шаблоне, управљање окружењима и аутоматизацију распореда

### Q: Да ли ми треба Azure CLI инсталиран да бих користио azd?
**A**: Да, azd захтева Azure CLI за аутентификацију и неке операције. Прво инсталирајте Azure CLI, а затим инсталирајте azd.

### Q: Које програмске језике azd подржава?
**A**: azd је независан од језика. Ради са:
- Node.js/JavaScript/TypeScript
- Python
- .NET/C#
- Java
- Go
- PHP
- Статички веб сајтови
- Контейнеризоване апликације

### Q: Могу ли да користим azd са постојећим пројектима?
**A**: Да! Можете или:
1. Користити `azd init` да додате azd конфигурацију у постојеће пројекте
2. Прилагодити постојеће пројекте да одговарају структури azd шаблона
3. Креирати прилагођене шаблоне на основу ваше постојеће архитектуре

## Аутентификација и приступ

### Q: Како се аутентификујем у Azure користећи azd?
**A**: Користите `azd auth login` који ће отворити прозор прегледача за Azure аутентификацију. За CI/CD сценарије, користите service principals или managed identities.

### Q: Могу ли да користим azd са више Azure претплата?
**A**: Да. Користите `azd env set AZURE_SUBSCRIPTION_ID <subscription-id>` да одредите коју претплату да користите за свако окружење.

### Q: Која дозволе су ми потребне да распорежем уз azd?
**A**: Типично вам треба:
- **Contributor** улога на групи ресурса или претплати
- **User Access Administrator** ако распоређујете ресурсе који захтевају доделе улога
- Конкретне дозволе варирају по шаблону и ресурсима који се распоређују

### Q: Могу ли да користим azd у CI/CD пипелинима?
**A**: Апсолутно! azd је дизајниран за интеграцију у CI/CD. Користите service principals за аутентификацију и подесите environment променљиве за конфигурацију.

### Q: Како да решим аутентификацију у GitHub Actions?
**A**: Користите Azure Login action са сервисним службеним акредитивима:
```yaml
- uses: azure/login@v1
  with:
    creds: ${{ secrets.AZURE_CREDENTIALS }}
- run: azd deploy --no-prompt
```

## Шаблони и пројекти

### Q: Где могу да пронађем azd шаблоне?
**A**: 
- Званични шаблони: [Azure-Samples/awesome-azd](https://github.com/Azure-Samples/awesome-azd)
- Заједнички шаблони: Претражите GitHub за "azd-template"
- Користите `azd template list` да прегледате доступне шаблоне

### Q: Како да креирам прилагођени шаблон?
**A**: 
1. Почните са постојећом структуром шаблона
2. Измените `azure.yaml`, инфраструктурне датотеке и код апликације
3. Temeljno testirajte са `azd up`
4. Објавите на GitHub уз одговарајуће тагове

### Q: Могу ли да користим azd без шаблона?
**A**: Да, користите `azd init` у постојећем пројекту да креирате потребне конфигурационе датотеке. Биће потребно да ручно конфигуришете `azure.yaml` и инфраструктурне датотеке.

### Q: Која је разлика између званичних и заједничких шаблона?
**A**: 
- **Званични шаблони**: Одржава Microsoft, редовно ажурирани, свеобухватна документација
- **Заједнички шаблони**: Креирају их програмери, могу имати специфичне случајеве употребе, квалитет и одржавање варирају

### Q: Како да ажурирам шаблон у мом пројекту?
**A**: Шаблони се не ажурирају аутоматски. Можете:
1. Ручно упоредити и спојити измене са изворног шаблона
2. Почети испочетка са `azd init` користећи ажуриран шаблон
3. Изабрати конкретна побољшања из ажурираног шаблона

## Распоређивање и инфраструктура

### Q: Које Azure услуге azd може да распореди?
**A**: azd може да распореди било коју Azure услугу преко Bicep/ARM шаблона, укључујући:
- App Services, Container Apps, Functions
- Базе података (SQL, PostgreSQL, Cosmos DB)
- Сториџ, Key Vault, Application Insights
- Мрежне, безбедносне и мониторинг ресурсе

### Q: Могу ли да распоређујем у више регија?
**A**: Да, конфигуришите више региона у вашим Bicep шаблонским и подесите параметар location адекватно за свако окружење.

### Q: Како да управљам миграцијама шеме базе података?
**A**: Користите deployment hooks у `azure.yaml`:
```yaml
hooks:
  postdeploy:
    posix:
      run: ./scripts/migrate-database.sh
    windows:
      run: ./scripts/migrate-database.ps1
```

### Q: Могу ли да распоредим само инфраструктуру без апликација?
**A**: Да, користите `azd provision` да распоредите само инфраструктурне компоненте дефинисане у вашим шаблонима.

### Q: Како да распоредим на постојеће Azure ресурсе?
**A**: Ово је сложено и није директно подржано. Можете:
1. Импортовати постојеће ресурсе у ваше Bicep шаблоне
2. Користити референце на постојеће ресурсе у шаблонима
3. Изменити шаблоне да условно креирају или референцирају ресурсе

### Q: Могу ли да користим Terraform уместо Bicep?
**A**: Тренутно, azd углавном подржава Bicep/ARM шаблоне. Подршка за Terraform није званично доступна, иако могу постојати решења заједнице.

## Конфигурација и окружења

### Q: Како да управљам различитим окружењима (dev, staging, prod)?
**A**: Креирајте одвојена окружења са `azd env new <environment-name>` и конфигуришите различита подешавања за свако:
```bash
azd env new development
azd env new staging  
azd env new production
```

### Q: Где су сачуване конфигурације окружења?
**A**: У фолдеру `.azure` унутар коренског директоријума вашег пројекта. Свако окружење има свој фолдер са конфигурационим датотекама.

### Q: Како да подесим конфигурацију специфичну за окружење?
**A**: Користите `azd env set` да конфигуришете environment променљиве:
```bash
azd env set AZURE_LOCATION eastus
azd env set DATABASE_TIER Basic
```

### Q: Могу ли да делим конфигурације окружења међу члановима тима?
**A**: Фолдер `.azure` садржи осетљиве информације и не би требао бити додат у контролу верзија. Уместо тога:
1. Документујте потребне environment променљиве
2. Користите скрипте за распоређивање да подесите окружења
3. Користите Azure Key Vault за осетљиву конфигурацију

### Q: Како да прећи параметре подразумеваних вредности шаблона?
**A**: Подесите environment променљиве које одговарају параметрима шаблона:
```bash
azd env set LOCATION "West US 2"
azd env set SKU_NAME "B1"
```

## Отстрањивање проблема

### Q: Зашто `azd up` не ради?
**A**: Уобичајени узроци:
1. **Проблеми са аутентификацијом**: Покрените `azd auth login`
2. **Недовољне дозволе**: Проверите своје Azure доделе улога
3. **Сукоб имена ресурса**: Промените AZURE_ENV_NAME
4. **Проблеми са квотом/капацитетом**: Проверите доступност у региону
5. **Грешке у шаблону**: Валидирајте Bicep шаблоне

### Q: Како да дебагујем неуспехе распореда?
**A**: 
1. Користите `azd deploy --debug` за детаљан излаз
2. Проверите историју распореда у Azure порталу
3. Прегледајте Activity Log у Azure порталу
4. Користите `azd show` да прикажете тренутно стање окружења

### Q: Зашто моје environment променљиве не раде?
**A**: Проверите:
1. Да имена променљивих тачно одговарају параметрима шаблона
2. Да су вредности правилно цитиране ако садрже размаке
3. Да је окружење изабрано: `azd env select <environment>`
4. Да су променљиве подешене у правилном окружењу

### Q: Како да очистим неуспеле распоређивања?
**A**: 
```bash
azd down --force --purge
```
Ово уклања све ресурсе и конфигурацију окружења.

### Q: Зашто моја апликација није доступна након распореда?
**A**: Проверите:
1. Да ли је распоређивање успешно завршено
2. Да ли апликација ради (проверите логове у Azure порталу)
3. Да ли мрежни security групе дозвољавају саобраћај
4. Да ли су DNS/прилагођени домени исправно конфигурисани

## Трошкови и наплата

### Q: Колико ће коштати распоређивања са azd?
**A**: Трошкови зависе од:
- Azure услуга које се распореде
- Изабраних service tiers/SKUs
- Разлика у ценама по регионима
- Обрасци коришћења

Користите [Azure Pricing Calculator](https://azure.microsoft.com/pricing/calculator/) за процене.

### Q: Како контролисати трошкове у azd распореде?
**A**: 
1. Користите нижи tier/sku за развојна окружења
2. Подесите Azure бюджете и аларме
3. Користите `azd down` да уклоните ресурсе када нису потребни
4. Изаберите одговарајуће регионе (трошкови варирају по локацији)
5. Користите Azure Cost Management алате

### Q: Постоје ли опције бесплатног нивоа за azd шаблоне?
**A**: Многе Azure услуге нуде бесплатне нивое:
- App Service: доступан бесплатни ниво
- Azure Functions: 1M бесплатних извршавања/месечно
- Cosmos DB: бесплатни ниво са 400 RU/s
- Application Insights: првих 5GB/месечно бесплатно

Конфигуришите шаблоне да користе бесплатне нивое где је то доступно.

### Q: Како да проценим трошкове пре распореда?
**A**: 
1. Прегледајте `main.bicep` шаблон да видите које ресурсе креира
2. Користите Azure Pricing Calculator са специфичним SKU-овима
3. Распоредите у развојно окружење прво да бисте пратили стварне трошкове
4. Користите Azure Cost Management за детаљну анализу трошкова

## Најбоље праксе

### Q: Које су најбоље праксе за структуру azd пројекта?
**A**: 
1. Држите код апликације одвојено од инфраструктуре
2. Користите смислене називе сервиса у `azure.yaml`
3. Имплементирајте правилно руковање грешкама у build скриптама
4. Користите конфигурацију специфичну за окружење
5. Укључите свеобухватну документацију

### Q: Како да организујем више сервиса у azd?
**A**: Користите препоручену структуру:
```
├── src/
│   ├── web/          # Frontend service
│   ├── api/          # Backend service  
│   └── worker/       # Background service
├── infra/            # Infrastructure templates
├── azure.yaml        # azd configuration
└── README.md         # Project documentation
```

### Q: Да ли треба да комитујем фолдер `.azure` у контролу верзија?
**A**: **Не!** Фолдер `.azure` садржи осетљиве информације. Додајте га у `.gitignore`:
```gitignore
.azure/
```

### Q: Како да управљам тајнама и осетљивом конфигурацијом?
**A**: 
1. Користите Azure Key Vault за тајне
2. Референтите тајне из Key Vault-а у конфигурацији апликације
3. Никада не комитујте тајне у контролу верзија
4. Користите managed identities за аутентификацију сервис-до-сервис

### Q: Који је препоручени приступ за CI/CD са azd?
**A**: 
1. Користите одвојена окружења за сваки стадјум (dev/staging/prod)
2. Имплементирајте аутоматизовано тестирање пре распореда
3. Користите service principals за аутентификацију
4. Чувајте осетљиву конфигурацију у тајнама/варцијаблама пипелина
5. Имплементирајте одобрења за продукцијска распореде

## Напредне теме

### Q: Могу ли да проширим azd прилагођеном функционалношћу?
**A**: Да, кроз deployment hooks у `azure.yaml`:
```yaml
hooks:
  predeploy:
    run: ./scripts/custom-setup.sh
  postdeploy:
    run: ./scripts/custom-config.sh
```

### Q: Како да интегришем azd у постојеће DevOps процесе?
**A**: 
1. Користите azd команде у постојећим скриптама пипелина
2. Стандардузујте azd шаблоне међу тимовима
3. Интегришите са постојећим мониторингом и алармима
4. Користите JSON излаз azd-а за интеграцију у пипелине

### Q: Могу ли да користим azd са Azure DevOps?
**A**: Да, azd ради са било којим CI/CD системом. Креирајте Azure DevOps pipeline-ове који користе azd команде.

### Q: Како да допринесем azd-у или креирам заједничке шаблоне?
**A**:
1. **azd tool**: Допринесите [Azure/azure-dev](https://github.com/Azure/azure-dev)
2. **Шаблони**: Направите предлошке у складу са [смерницама за предлошке](https://github.com/Azure-Samples/awesome-azd)
3. **Документација**: Допринесите документацији на [MicrosoftDocs/azure-dev-docs](https://github.com/MicrosoftDocs/azure-dev-docs)

### Q: Који је план развоја за azd?
**A**: Погледајте [службени план развоја](https://github.com/Azure/azure-dev/projects) за планиране функционалности и побољшања.

### Q: Како да мигрирам са других алата за распоређивање (deployment) на azd?
**A**: 
1. Анализирајте тренутну архитектуру распоређивања
2. Направите одговарајеће Bicep предлошке
3. Подесите `azure.yaml` да одговара тренутним сервисима
4. Темељно тестирајте у развојном окружењу
5. Постепено мигрирајте окружења
