## Увод

Овај обимни речник пружа дефиниције термина, концепата и скраћеница које се користе у Azure Developer CLI и развоју у Azure облаку. Неопходна референца за разumeвање техничке документације, решавање проблема и ефикасну комуникацију о azd пројектима и Azure сервисима.

## Цели учења

Коришћењем овог речника ћете:
- Разумети основну терминологију и концепте Azure Developer CLI
- Усвојити вокабулар и техничке термине за развој у Azure облаку
- Ефикасно користити терминологију везану за Infrastructure as Code и деплојмент
- Разумети имена Azure сервиса, скраћенице и њихове сврхе
- Приступити дефиницијама за решавање проблема и дебаговање
- Научити напредне концепте архитектуре и развоја у Azure-у

## Резултати учења

Редовним коришћењем овог речника, бићете у стању да:
- Ефикасно комуницирате користећи правилну терминологију Azure Developer CLI
- Јасније разумете техничку документацију и поруке о грешкама
- Са поверењем навигирате Azure сервисима и концептима
- Решавате проблеме користећи одговарајући технички вокабулар
- Доприносите тимским дискусијама прецизним техничким језиком
- Систематски проширујете своје знање о развоју у Azure облаку

## A

**ARM Template**  
Azure Resource Manager template. Формат Infrastructure as Code заснован на JSON-у који се користи за декларативно дефинисање и распоређивање Azure ресурса.

**App Service**  
Azure-ова платформа као услуга (PaaS) за хостовање веб апликација, REST API-ја и мобилних бекенда без управљања инфраструктуром.

**Application Insights**  
Azure сервис за мониторинг перформанси апликација (APM) који пружа дубоке увиде у перформансе апликације, доступност и коришћење.

**Azure CLI**  
Интерфејс командне линије за управљање Azure ресурсима. Користи се од стране azd за аутентификацију и неке операције.

**Azure Developer CLI (azd)**  
Командни алат оријентисан на развојне инжењере који убрзава процес изградње и распоређивања апликација у Azure користећи шаблоне и Infrastructure as Code.

**azure.yaml**  
Главни конфигурациони фајл за azd пројекат који дефинише сервисе, инфраструктуру и хукс за деплојмент.

**Azure Resource Manager (ARM)**  
Azure сервис за распоређивање и управљање који пружа управљачки слој за креирање, ажурирање и брисање ресурса.

## B

**Bicep**  
Домент-специфичан језик (DSL) који је развио Microsoft за распоређивање Azure ресурса. Пружа поједностављен синтакс у односу на ARM шаблоне док се компајлира у ARM.

**Build**  
Процес компајлирања изворног кода, инсталирања зависности и припреме апликација за распоређивање.

**Blue-Green Deployment**  
Стратегија распоређивања која користи два идентична продукциона окружења (плаво и зелено) да би се минимализовало престанак рада и ризик.

## C

**Container Apps**  
Azure-ов серверлес сервис за контејнере који омогућава покретање контејнеризованих апликација без управљања комплексном инфраструктуром.

**CI/CD**  
Continuous Integration/Continuous Deployment. Аутоматизоване праксе за интеграцију промена у коду и распоређивање апликација.

**Cosmos DB**  
Глобално дистрибуиран, мулти-моделни базни сервис у Azure-у који пружа свеобухватне SLA-ове за пропусност, латенцију, доступност и конзистентност.

**Configuration**  
Подешавања и параметри који контролишу понашање апликације и опције распоређивања.

## D

**Deployment**  
Процес инсталирања и конфигурисања апликација и њихових зависности на циљној инфраструктури.

**Docker**  
Платформа за развој, испоруку и покретање апликација користећи технологију контејнеризације.

**Dockerfile**  
Текстуални фајл који садржи упутства за изградњу Docker контејнер слике.

## E

**Environment**  
Циљно окружење за распоређивање које представља конкретну инстанцу ваше апликације (нпр. development, staging, production).

**Environment Variables**  
Вредности конфигурације складиштене као парови кључ-вредност које апликације могу да приступе у време извршавања.

**Endpoint**  
URL или мрежна адреса на којој се апликација или сервис може приступити.

## F

**Function App**  
Azure-ов серверлес рачунски сервис који омогућава покретање догађајима вођеног кода без управљања инфраструктуром.

## G

**GitHub Actions**  
CI/CD платформа интегрисана са GitHub репозиторијумима за аутоматизацију радних токова.

**Git**  
Дистрибуирани систем контроле верзија који се користи за праћење измена у изворном коду.

## H

**Hooks**  
Прилагођени скриптови или команде које се покрећу у одређеним тачкама током животног циклуса деплоја (preprovision, postprovision, predeploy, postdeploy).

**Host**  
Тип Azure сервиса на који ће апликација бити деплојована (нпр. appservice, containerapp, function).

## I

**Infrastructure as Code (IaC)**  
Пракса дефинисања и управљања инфраструктуром путем кода уместо ручних процеса.

**Init**  
Процес иницијализације новог azd пројекта, обично из шаблона.

## J

**JSON**  
JavaScript Object Notation. Формат за размeну података који се често користи за конфигурационе фајлове и одговоре API-ја.

**JWT**  
JSON Web Token. Стандард за безбедан пренос информација између страна као JSON објекат.

## K

**Key Vault**  
Azure-ов сервис за безбедно чување и управљање тајнама, кључевима и сертификатима.

**Kusto Query Language (KQL)**  
Језик упита који се користи за анализу података у Azure Monitor-у, Application Insights-у и другим Azure сервисима.

## L

**Load Balancer**  
Сервис који распоређује долазни мрежни саобраћај међу више сервера или инстанци.

**Log Analytics**  
Azure сервис за прикупљање, анализу и деловање на телеметријским подацима из облака и on-premise окружења.

## M

**Managed Identity**  
Функција у Azure-у која обезбеђује сервисима аутоматски управљан идентитет за аутентификацију према другим Azure сервисима.

**Microservices**  
Архитектонски приступ где се апликације граде као скуп малих, независних сервиса.

**Monitor**  
Унифицирано решење за мониторинг у Azure-у које пружа опсежну опсервабилност кроз апликације и инфраструктуру.

## N

**Node.js**  
Време извршавања JavaScript-а изграђено на V8 JavaScript мотору за изградњу сервер-сајд апликација.

**npm**  
Менаџер пакета за Node.js који управља зависностима и пакетима.

## O

**Output**  
Вредности које се враћају из распоређивања инфраструктуре и које апликације или други ресурси могу користити.

## P

**Package**  
Процес припреме кода апликације и зависности за распоређивање.

**Parameters**  
Улазне вредности прослеђене инфраструктурним шаблонима за прилагођавање деплоја.

**PostgreSQL**  
Отворено-кодни релациони систем за управљање базама података који је подржан као управљани сервис у Azure-у.

**Provisioning**  
Процес креирања и конфигурисања Azure ресурса дефинисаних у инфраструктурним шаблонима.

## Q

**Quota**  
Ограничења у вези са количином ресурса која се може креирати у Azure претплати или региону.

## R

**Resource Group**  
Логички контејнер за Azure ресурсе који деле исти животни циклус, дозволе и политике.

**Resource Token**  
Јединствени низ генерисан од стране azd-а да обезбеди да имена ресурса буду јединствена преко деплоя.

**REST API**  
Архитектонски стил за дизајн мрежних апликација коришћењем HTTP метода.

**Rollback**  
Процес враћања на претходну верзију апликације или конфигурације инфраструктуре.

## S

**Service**  
Компонента ваше апликације дефинисана у azure.yaml (нпр. web frontend, API backend, база података).

**SKU**  
Stock Keeping Unit. Представља различите нивое сервиса или перформанси за Azure ресурсе.

**SQL Database**  
Управљани релациони базни сервис у Azure-у заснован на Microsoft SQL Server-у.

**Static Web Apps**  
Azure сервис за изградњу и распоређивање full-stack веб апликација из репозиторијума извора.

**Storage Account**  
Azure сервис који пружа облачно складиште за податке укључујући blobs, files, queues и tables.

**Subscription**  
Контејнер Azure налога који држи resource групе и ресурсе, са повезаним обрачуном и управљањем приступом.

## T

**Template**  
Унапред направљена структура пројекта која садржи код апликације, дефиниције инфраструктуре и конфигурацију за уобичајене сценарије.

**Terraform**  
Open-source алат Infrastructure as Code који подржава више cloud провајдера укључујући Azure.

**Traffic Manager**  
Azure-ов DNS-базирани балансер саобраћаја за дистрибуцију саобраћаја преко глобалних Azure региона.

## U

**URI**  
Uniform Resource Identifier. Низ који идентификује одређени ресурс.

**URL**  
Uniform Resource Locator. Тип URI-а који спецификује где се ресурс налази и како га дохватити.

## V

**Virtual Network (VNet)**  
Основни градивни елемент за приватне мреже у Azure-у, пружа изолацију и сегментацију.

**VS Code**  
Visual Studio Code. Популаран уређивач кода са одличном интеграцијом за Azure и azd.

## W

**Webhook**  
HTTP повратни позив који се покреће одређеним догађајима, често коришћен у CI/CD пајплајновима.

**What-if**  
Azure функција која показује које би промене биле направљене распоређивањем без њиховог стварног извршавања.

## Y

**YAML**  
YAML Ain't Markup Language. Стандард за људски читљиву серијализацију података који се користи за конфигурационе фајлове као што је azure.yaml.

## Z

**Zone**  
Физички одвојене локације унутар Azure региона које пружају редунданцију и високу доступност.

## Термини специфични за контекст

### Development Terms
- **Hot Reload**: Аутоматско ажурирање апликација током развоја без поновног покретања
- **Build Pipeline**: Аутоматизовани процес изградње и тестирања кода
- **Deployment Slot**: Стагирано окружење у оквиру App Service-а
- **Environment Parity**: Одржавање подударности развојног, стејџинг и продукционог окружења

### Security Terms
- **Managed Identity**: Azure функција која обезбеђује аутоматско управљање акредитивима
- **Key Vault**: Безбедно складиште за тајне, кључеве и сертификате
- **RBAC**: Контрола приступа заснована на улогама за Azure ресурсе
- **Network Security Group**: Виртуелни заштитни зид за контролу мрежног саобраћаја

### Monitoring Terms
- **Telemetry**: Аутоматско прикупљање мера и података
- **Application Performance Monitoring (APM)**: Надгледање перформанси софтвера
- **Log Analytics**: Сервис за прикупљање и анализу лог података
- **Alert Rules**: Аутоматизована обавештења заснована на метрикама или условима

### Deployment Terms
- **Blue-Green Deployment**: Стратегија распоређивања без прекида
- **Canary Deployment**: Поступно пуштање малом делу корисника
- **Rolling Update**: Секвенцијална замена инстанци апликације
- **Rollback**: Враћање на претходну верзију апликације
