# 為 AZD For Beginners 做出貢獻

感謝你有意向 AZD For Beginners 課程做出貢獻！本文件提供貢獻者的指引和資訊。

## 如何貢獻

### 回報問題

如果你發現錯誤、問題，或有改進建議：

1. **Check Existing Issues**: Search [現有議題](https://github.com/microsoft/azd-for-beginners/issues) to avoid duplicates
2. **Create a New Issue**: Use the appropriate 議題範本
3. **Provide Details**: Include steps to reproduce, expected behavior, and screenshots if applicable

### 提議更改

1. **Fork the Repository**: Create your own fork of the project
2. **Create a Branch**: Create a feature branch for your changes
3. **Make Changes**: Follow our coding and documentation standards
4. **Test Your Changes**: Verify all links work and content renders correctly
5. **Submit a Pull Request**: Provide a clear description of your changes

### 貢獻類型

我們歡迎以下類型的貢獻：

#### 內容改進
- 修正拼字錯誤、語法及格式問題
- 改善現有文件的清晰度
- 更新過時的資訊
- 補充遺漏的說明或背景

#### 新範例
- 新增實際部署情境
- 為常見使用情境建立新的 AZD 範本
- 提供額外的程式碼範例與練習

#### 翻譯支援
- 協助維護現有翻譯
- 提升翻譯品質
- 為新內容新增翻譯

#### 錯誤回報
- 回報斷開的連結或不正確的程式碼範例
- 找出文件內容不一致之處
- 標記可及性問題

## 貢獻指南

### 文件標準

- 使用清楚、適合初學者的語言
- 包含學習目標與成果
- 提供逐步指示並標示預期輸出
- 新增導覽連結至相關內容
- 使用一致的格式與 markdown 風格

### 程式碼範例

- 確保所有程式碼範例皆已測試並可運行
- 加入說明關鍵概念的註解
- 遵循 Azure 與 AZD 的最佳實務
- 在適用處提供預期輸出

### 提交訊息

Use clear, descriptive commit messages:
- `docs: fix typo in installation guide`
- `feat: add new container app example`
- `fix: correct broken link in chapter 2`
