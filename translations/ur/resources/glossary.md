# اصطلاحات - ایزور اور AZD کی اصطلاحات

**تمام ابواب کے لیے حوالہ**
- **📚 کورس ہوم**: [نئی شروعات کے لیے AZD](../README.md)
- **📖 بنیادی معلومات سیکھیں**: [باب 1: AZD کی بنیادیں](../docs/getting-started/azd-basics.md)
- **🤖 AI اصطلاحات**: [باب 2: AI-فرسٹ ڈیولپمنٹ](../docs/microsoft-foundry/microsoft-foundry-integration.md)

## تعارف

یہ جامع اصطلاحات کی فہرست Azure Developer CLI اور Azure کلاؤڈ ڈیولپمنٹ میں استعمال ہونے والی اصطلاحات، تصورات، اور مخففات کی تعریفات فراہم کرتی ہے۔ تکنیکی دستاویزات کو سمجھنے، مسائل کی تشخیص کرنے، اور azd پراجیکٹس اور Azure خدمات کے بارے میں مؤثر طریقے سے بات چیت کرنے کے لیے ایک ضروری حوالہ۔

## سیکھنے کے مقاصد

اس اصطلاحات کی فہرست کا استعمال کرتے ہوئے، آپ:
- Azure Developer CLI کی ضروری اصطلاحات اور تصورات کو سمجھیں گے
- Azure کلاؤڈ ڈیولپمنٹ کی زبان اور تکنیکی اصطلاحات میں مہارت حاصل کریں گے
- انفراسٹرکچر ایز کوڈ اور تعیناتی کی اصطلاحات کو مؤثر طریقے سے حوالہ دیں گے
- Azure سروسز کے نام، مخففات، اور ان کے مقاصد کو سمجھیں گے
- مسائل کی تشخیص اور ڈیبگنگ کی اصطلاحات کے تعارف تک رسائی حاصل کریں گے
- اعلیٰ درجے کے Azure آرکیٹیکچر اور ڈیولپمنٹ تصورات سیکھیں گے

## سیکھنے کے نتائج

اس اصطلاحات کی فہرست کا باقاعدہ حوالہ دے کر، آپ قادِر ہوں گے:
- Azure Developer CLI کی درست اصطلاحات استعمال کرتے ہوئے مؤثر بات چیت کریں
- تکنیکی دستاویزات اور خرابی کے پیغامات کو بہتر طریقے سے سمجھیں
- Azure خدمات اور تصورات کو اعتماد کے ساتھ نیویگیٹ کریں
- مناسب تکنیکی زبان کا استعمال کرتے ہوئے مسائل تلاش اور حل کریں
- ٹیم کی گفتگو میں صحیح تکنیکی زبان سے حصہ لیں
- منظم طریقے سے Azure کلاؤڈ ڈیولپمنٹ کا علم بڑھائیں

## A

**ARM ٹیمپلیٹ**  
Azure Resource Manager ٹیمپلیٹ۔ JSON پر مبنی Infrastructure as Code فارمیٹ جو Azure وسائل کی وضاحت اور تعیناتی کے لیے استعمال ہوتا ہے۔

**App Service**  
Azure کا پلیٹ فارم بطور سروس (PaaS) جس سے ویب ایپلیکیشنز، REST APIs، اور موبائل بیک اینڈز کو بغیر انفراسٹرکچر مینج کیے ہوسٹ کیا جا سکتا ہے۔

**Application Insights**  
Azure کی ایپلیکیشن پرفارمنس مانیٹرنگ (APM) سروس جو ایپلیکیشن کی کارکردگی، دستیابی، اور استعمال کی گہری معلومات فراہم کرتی ہے۔

**Azure CLI**  
Azure وسائل کو مینیج کرنے کے لیے کمانڈ لائن انٹرفیس۔ azd کے ذریعے توثیق اور کچھ آپریشنز کے لیے استعمال ہوتا ہے۔

**Azure Developer CLI (azd)**  
ڈویلپرمرکوز کمانڈ لائن ٹول جو ٹیمپلیٹس اور Infrastructure as Code کا استعمال کرتے ہوئے ایپلیکیشنز کو Azure پر جلدی بنانے اور تعینات کرنے کا عمل تیز کرتا ہے۔

**azure.yaml**  
azd پراجیکٹ کا مرکزی کنفیگریشن فائل جو خدمات، انفراسٹرکچر، اور تعیناتی ہکس کی وضاحت کرتا ہے۔

**Azure Resource Manager (ARM)**  
Azure کی تعیناتی اور انتظامی سروس جو وسائل کو بنانے، اپ ڈیٹ کرنے، اور حذف کرنے کے لیے مینجمنٹ لئیر فراہم کرتی ہے۔

## B

**Bicep**  
Microsoft کی جانب سے تیار کردہ ڈومین مخصوص زبان (DSL) جو Azure وسائل کی تعیناتی کے لیے استعمال ہوتی ہے۔ یہ ARM ٹیمپلیٹس سے آسان تر نحو فراہم کرتی ہے اور ARM میں کمپائل ہوتی ہے۔

**Build**  
سورس کوڈ کو مرتب کرنے، انحصار کو انسٹال کرنے، اور ایپلیکیشنز کو تعیناتی کے لیے تیار کرنے کا عمل۔

**Blue-Green Deployment**  
تعیناتی کی حکمت عملی جو دو یکساں پروڈکشن ماحول (نیلا اور سبز) استعمال کرتی ہے تاکہ ڈاؤن ٹائم اور خطرے کو کم کیا جا سکے۔

## C

**Container Apps**  
Azure کی سرور لیس کنٹینر سروس جو پیچیدہ انفراسٹرکچر کے بغیر کنٹینر کی گئی ایپلیکیشنز چلانے کی سہولت دیتی ہے۔

**CI/CD**  
مسلسل انضمام/مسلسل تعیناتی۔ کوڈ کی تبدیلیوں کو ضم کرنے اور ایپلیکیشنز کو تعینات کرنے کے خودکار طریقے۔

**Cosmos DB**  
Azure کی عالمی طور پر تقسیم شدہ، کثیر ماڈل ڈیٹا بیس سروس جو تھروپٹ، تاخیر، دستیابی، اور مستقل مزاجی کے لیے جامع SLA فراہم کرتی ہے۔

**Configuration**  
ایپلیکیشن کے رویے اور تعیناتی کے اختیارات کو کنٹرول کرنے والی ترتیبات اور پیرامیٹرز۔

## D

**Deployment**  
ایپلیکیشنز اور ان کے انحصار کو ہدف انفراسٹرکچر پر انسٹال اور ترتیب دینے کا عمل۔

**Docker**  
کنٹینرائزیشن ٹیکنالوجی کا استعمال کرتے ہوئے ایپلیکیشنز کی ترقی، شپنگ، اور چلانے کا پلیٹ فارم۔

**Dockerfile**  
متن پر مشتمل فائل جو Docker کنٹینر امیج بنانے کی ہدایات پر مشتمل ہوتی ہے۔

## E

**Environment**  
تعیناتی کا ہدف جو آپ کی ایپلیکیشن کا ایک مخصوص انسٹانس ظاہر کرتا ہے (مثلاً، ڈیولپمنٹ، اسٹینگ، پروڈکشن)۔

**Environment Variables**  
تشکیلی اقدار جو کلید-قدر جوڑوں کی صورت میں محفوظ ہوتی ہیں جنہیں ایپلیکیشنز رن ٹائم پر رسائی کر سکتی ہیں۔

**Endpoint**  
URL یا نیٹ ورک ایڈریس جہاں ایپلیکیشن یا سروس تک رسائی حاصل کی جا سکتی ہے۔

## F

**Function App**  
Azure کی سرور لیس کمپیوٹنگ سروس جو بغیر انفراسٹرکچر کے ایونٹ ڈرائیون کوڈ چلانے کی سہولت دیتی ہے۔

## G

**GitHub Actions**  
GitHub ریپوزٹریز کے ساتھ مربوط CI/CD پلیٹ فارم برائے خودکار ورک فلو۔

**Git**  
تقسیم شدہ ورژن کنٹرول سسٹم جو سورس کوڈ میں تبدیلیوں کو ٹریک کرنے کے لیے استعمال ہوتا ہے۔

## H

**Hooks**  
حسب ضرورت اسکرپٹ یا کمانڈز جو تعیناتی کے مخصوص مراحل (preprovision، postprovision، predeploy، postdeploy) میں چلائے جاتے ہیں۔

**Host**  
Azure سروس کی قسم جہاں ایپلیکیشن تعینات کی جائے گی (مثلاً، appservice، containerapp، function)۔

## I

**Infrastructure as Code (IaC)**  
انفراسٹرکچر کو دستی عمل کی بجائے کوڈ کے ذریعے بیان کرنے اور مینیج کرنے کا عمل۔

**Init**  
نئے azd پراجیکٹ کو ٹیمپلیٹ سے شروع کرنے کا عمل۔

## J

**JSON**  
JavaScript Object Notation۔ کنفیگریشن فائلوں اور API جوابات کے لیے عام ڈیٹا تبادلے کا فارمیٹ۔

**JWT**  
JSON Web Token۔ فریقین کے درمیان معلومات کی محفوظ ترسیل کے لیے ایک معیاری فارمیٹ۔

## K

**Key Vault**  
Azure کی سروس جو خفیہ معلومات، چابیاں، اور سرٹیفیکیٹس کو محفوظ طریقے سے ذخیرہ اور منظم کرتی ہے۔

**Kusto Query Language (KQL)**  
Azure Monitor، Application Insights، اور دیگر Azure خدمات میں ڈیٹا کا تجزیہ کرنے والی زبان۔

## L

**Load Balancer**  
سروس جو متعدد سرورز یا انسٹانسز میں آنے والے نیٹ ورک ٹریفک کو تقسیم کرتی ہے۔

**Log Analytics**  
Azure کی سروس جو کلاؤڈ اور آن-پرمیسس ماحول سے ٹیلیمیٹری ڈیٹا جمع کرنے، تجزیہ کرنے، اور اس پر عمل کرنے کے لیے استعمال ہوتی ہے۔

## M

**Managed Identity**  
Azure فیچر جو Azure خدمات کو خودکار طور پر منظم شناخت فراہم کرتا ہے تاکہ دیگر Azure خدمات کی تصدیق کر سکیں۔

**Microservices**  
معماری کا طریقہ جہاں ایپلیکیشنز کو چھوٹے، خودمختار خدمات کے مجموعے کے طور پر بنایا جاتا ہے۔

**Monitor**  
Azure کا یونیفائیڈ مانیٹرنگ حل جو ایپلیکیشنز اور انفراسٹرکچر کی مکمل نگرانی فراہم کرتا ہے۔

## N

**Node.js**  
Chrome کے V8 JavaScript انجن پر مبنی JavaScript رن ٹائم جو سرور سائڈ ایپلیکیشنز بنانے کے لیے استعمال ہوتا ہے۔

**npm**  
Node.js کے لیے پیکیج مینیجر جو انحصار اور پیکیجز کو مینیج کرتا ہے۔

## O

**Output**  
انفراسٹرکچر تعیناتی سے واپس آنے والی قدریں جو ایپلیکیشنز یا دیگر وسائل استعمال کر سکتے ہیں۔

## P

**Package**  
ایپلیکیشن کوڈ اور انحصار کو تعیناتی کے لیے تیار کرنے کا عمل۔

**Parameters**  
انفراسٹرکچر ٹیمپلیٹس کو دیے جانے والے ان پٹ ویلیوز تاکہ تعیناتیاں حسبِ ضرورت کی جا سکیں۔

**PostgreSQL**  
اوپن سورس رشنل ڈیٹا بیس سسٹم جو Azure میں مینیجڈ سروس کے طور پر دستیاب ہے۔

**Provisioning**  
Azure وسائل کو بنانے اور ترتیب دینے کا عمل جیسا کہ انفراسٹرکچر ٹیمپلیٹس میں متعین ہے۔

## Q

**Quota**  
Azure سبسکریپشن یا خطے میں پیدا کیے جانے والے وسائل کی مقدار پر پابندیاں۔

## R

**Resource Group**  
Azure وسائل کا منطقی کنٹینر جو ایک ہی لائف سائیکل، اجازتیں، اور پالیسیز شئیر کرتا ہے۔

**Resource Token**  
azd کی طرف سے تیار کردہ منفرد سٹرنگ جو تعیناتیوں میں وسائل کے ناموں کی انفرادیت کو یقینی بناتی ہے۔

**REST API**  
HTTP طریقوں کا استعمال کرتے ہوئے نیٹ ورک ایپلیکیشنز کے ڈیزائن کے لیے ایک آرکیٹیکچرل اسٹائل۔

**Rollback**  
ایپلیکیشن یا انفراسٹرکچر کی پچھلی ورژن پر واپس جانے کا عمل۔

## S

**Service**  
آپ کی ایپلیکیشن کا وہ جزو جو azure.yaml میں متعین ہوتا ہے (مثلاً، ویب فرنٹ اینڈ، API بیک اینڈ، ڈیٹابیس)۔

**SKU**  
Stock Keeping Unit۔ Azure وسائل کے مختلف سروس ٹیئرز یا کارکردگی کی سطح کی نمائندگی کرتا ہے۔

**SQL Database**  
Azure کی منظم رشنل ڈیٹا بیس سروس جو Microsoft SQL Server پر مبنی ہے۔

**Static Web Apps**  
سورس کوڈ ریپوزٹریز سے مکمل اسٹیک ویب ایپلیکیشنز بنانے اور تعینات کرنے کی Azure سروس۔

**Storage Account**  
Azure کی سروس جو بلیبز، فائلز، قطاروں، اور ٹیبلز سمیت ڈیٹا اشیاء کے لیے کلاؤڈ ذخیرہ فراہم کرتی ہے۔

**Subscription**  
Azure اکاؤنٹ کنٹینر جو ریسورس گروپس اور وسائل کو رکھتا ہے، ساتھ ہی بلنگ اور رسائی مینجمنٹ بھی شامل ہے۔

## T

**Template**  
پری بلٹ پراجیکٹ ڈھانچہ جس میں ایپلیکیشن کوڈ، انفراسٹرکچر کی تعریفیں، اور عام منظرناموں کے لیے کنفیگریشن شامل ہوتی ہے۔

**Terraform**  
اوپن سورس Infrastructure as Code ٹول جو Azure سمیت متعدد کلاؤڈ فراہم کنندگان کی حمایت کرتا ہے۔

**Traffic Manager**  
Azure کا DNS پر مبنی ٹریفک لوڈ بیلینسر جو عالمی Azure علاقوں میں ٹریفک تقسیم کرتا ہے۔

## U

**URI**  
Uniform Resource Identifier۔ ایک سٹرنگ جو کسی مخصوص وسیلہ کی شناخت کرتی ہے۔

**URL**  
Uniform Resource Locator۔ URI کی قسم جو بتاتی ہے کہ کسی وسیلہ کو کہاں اور کیسے حاصل کیا جائے۔

## V

**Virtual Network (VNet)**  
Azure میں نجی نیٹ ورکس کے لیے بنیادی بلاک جو علیحدگی اور تقسیم فراہم کرتا ہے۔

**VS Code**  
Visual Studio Code۔ مقبول کوڈ ایڈیٹر جو Azure اور azd انٹیگریشن کے ساتھ آتا ہے۔

## W

**Webhook**  
HTTP کال بیک جو مخصوص ایونٹس پر ٹرگر ہوتا ہے، عام طور پر CI/CD پائپ لائنز میں استعمال ہوتا ہے۔

**What-if**  
Azure فیچر جو دکھاتا ہے کہ تعیناتی کے نتیجے میں کیا تبدیلیاں ہوں گی بغیر انہیں حقیقی طور پر لاگو کیے۔

## Y

**YAML**  
YAML Ain't Markup Language۔ انسانی پڑھنے کے قابل ڈیٹا سیریلائزیشن اسٹینڈرڈ جو azure.yaml جیسی کنفیگریشن فائلوں میں استعمال ہوتا ہے۔

## Z

**Zone**  
Azure خطے کے اندر جسمانی طور پر الگ الگ مقامات جو ریڈنڈنسی اور اعلیٰ دستیابی فراہم کرتے ہیں۔

## مخصوص سیاق و سباق کی اصطلاحات

### ترقیاتی اصطلاحات
- **Hot Reload**: ایپلیکیشنز کو دوبارہ شروع کیے بغیر خودکار طور پر اپ ڈیٹ کرنا
- **Build Pipeline**: کوڈ بنانے اور جانچنے کا خودکار عمل
- **Deployment Slot**: App Service کے اندر اسٹینگ ماحول
- **Environment Parity**: ترقی، اسٹینگ، اور پروڈکشن ماحول کو مماثل رکھنا

### سیکورٹی اصطلاحات
- **Managed Identity**: Azure فیچر جو خودکار اسناد مینجمنٹ فراہم کرتا ہے
- **Key Vault**: خفیہ معلومات، چابیوں، اور سرٹیفیکیٹس کے محفوظ ذخیرے کی سروس
- **RBAC**: Azure وسائل کے لیے کردار پر مبنی رسائی کا کنٹرول
- **Network Security Group**: نیٹ ورک ٹریفک کو کنٹرول کرنے والا ورچوئل فائر وال

### مانیٹرنگ اصطلاحات
- **Telemetry**: پیمائش اور ڈیٹا کا خودکار مجموعہ
- **Application Performance Monitoring (APM)**: سافٹ ویئر کی کارکردگی کی نگرانی
- **Log Analytics**: لوگ ڈیٹا جمع کرنے اور تجزیہ کرنے کی سروس
- **Alert Rules**: میٹرکس یا شرائط کی بنیاد پر خودکار اطلاعات

### تعیناتی کی اصطلاحات
- **Blue-Green Deployment**: زیرو ڈاؤن ٹائم تعیناتی کی حکمت عملی
- **Canary Deployment**: آہستہ آہستہ صارفین کے ایک حصے کو تعینات کرنا
- **Rolling Update**: ایپلیکیشن کے انسٹانس کی ترتیب وار تبدیلی
- **Rollback**: ایپلیکیشن کے پچھلے ورژن پر واپس جانا

---

**استعمال کی ترغیب**: مخصوص اصطلاحات کو جلدی تلاش کرنے کے لیے `Ctrl+F` استعمال کریں۔ اصطلاحات جہاں ممکن ہوں ایک دوسرے سے منسلک کی گئی ہیں۔
