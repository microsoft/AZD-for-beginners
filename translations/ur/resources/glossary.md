# گلاسری - Azure Developer CLI کی اصطلاحات

## تعارف

یہ جامع گلاسری Azure Developer CLI اور Azure کلاؤڈ ڈیولپمنٹ میں استعمال ہونے والی اصطلاحات، تصورات، اور مخففات کی تعریفیں فراہم کرتی ہے۔ تکنیکی دستاویزات کو سمجھنے، مسائل کو حل کرنے، اور azd پروجیکٹس اور Azure سروسز کے بارے میں مؤثر طریقے سے بات چیت کرنے کے لیے ایک ضروری حوالہ۔

## سیکھنے کے اہداف

اس گلاسری کے استعمال سے آپ:
- Azure Developer CLI کی بنیادی اصطلاحات اور تصورات کو سمجھ سکیں گے
- Azure کلاؤڈ ڈیولپمنٹ کی لغت اور تکنیکی اصطلاحات میں مہارت حاصل کریں گے
- انفراسٹرکچر کوڈ اور ڈیپلائمنٹ کی اصطلاحات کو مؤثر طریقے سے حوالہ دیں گے
- Azure سروسز کے نام، مخففات، اور ان کے مقاصد کو سمجھیں گے
- مسائل کو حل کرنے اور ڈیبگنگ کی اصطلاحات کی تعریفیں حاصل کریں گے
- Azure کی جدید آرکیٹیکچر اور ڈیولپمنٹ کے تصورات سیکھیں گے

## سیکھنے کے نتائج

اس گلاسری کا باقاعدہ حوالہ دینے سے آپ:
- Azure Developer CLI کی صحیح اصطلاحات کا استعمال کرتے ہوئے مؤثر طریقے سے بات چیت کر سکیں گے
- تکنیکی دستاویزات اور ایرر میسیجز کو زیادہ واضح طور پر سمجھ سکیں گے
- Azure سروسز اور تصورات کو اعتماد کے ساتھ نیویگیٹ کریں گے
- مسائل کو مناسب تکنیکی لغت کا استعمال کرتے ہوئے حل کریں گے
- ٹیم کی گفتگو میں درست تکنیکی زبان کے ساتھ حصہ لیں گے
- Azure کلاؤڈ ڈیولپمنٹ کے علم کو منظم طریقے سے بڑھائیں گے

## A

**ARM Template**  
Azure Resource Manager ٹیمپلیٹ۔ JSON پر مبنی انفراسٹرکچر کوڈ فارمیٹ جو Azure وسائل کو وضاحتی طور پر ڈیفائن اور ڈیپلائے کرنے کے لیے استعمال ہوتا ہے۔

**App Service**  
Azure کا پلیٹ فارم-ایز-اے-سروس (PaaS) پیشکش جو ویب ایپلیکیشنز، REST APIs، اور موبائل بیک اینڈز کو بغیر انفراسٹرکچر مینجمنٹ کے ہوسٹ کرنے کے لیے استعمال ہوتا ہے۔

**Application Insights**  
Azure کی ایپلیکیشن پرفارمنس مانیٹرنگ (APM) سروس جو ایپلیکیشن کی کارکردگی، دستیابی، اور استعمال کے بارے میں گہرے تجزیے فراہم کرتی ہے۔

**Azure CLI**  
Azure وسائل کو مینج کرنے کے لیے کمانڈ لائن انٹرفیس۔ azd کے ذریعے تصدیق اور کچھ آپریشنز کے لیے استعمال ہوتا ہے۔

**Azure Developer CLI (azd)**  
ڈیولپر مرکوز کمانڈ لائن ٹول جو ٹیمپلیٹس اور انفراسٹرکچر کوڈ کا استعمال کرتے ہوئے Azure پر ایپلیکیشنز بنانے اور ڈیپلائے کرنے کے عمل کو تیز کرتا ہے۔

**azure.yaml**  
azd پروجیکٹ کے لیے مرکزی کنفیگریشن فائل جو سروسز، انفراسٹرکچر، اور ڈیپلائمنٹ ہکس کو ڈیفائن کرتی ہے۔

**Azure Resource Manager (ARM)**  
Azure کی ڈیپلائمنٹ اور مینجمنٹ سروس جو وسائل کو بنانے، اپڈیٹ کرنے، اور حذف کرنے کے لیے مینجمنٹ لیئر فراہم کرتی ہے۔

## B

**Bicep**  
Microsoft کی طرف سے تیار کردہ ڈومین-اسپیسفک لینگویج (DSL) جو Azure وسائل کو ڈیپلائے کرنے کے لیے استعمال ہوتی ہے۔ ARM ٹیمپلیٹس کے مقابلے میں آسان سینٹیکس فراہم کرتی ہے جبکہ ARM میں کمپائل ہوتی ہے۔

**Build**  
سورس کوڈ کو کمپائل کرنے، ڈیپینڈنسیز انسٹال کرنے، اور ایپلیکیشنز کو ڈیپلائمنٹ کے لیے تیار کرنے کا عمل۔

**Blue-Green Deployment**  
ڈیپلائمنٹ کی حکمت عملی جو دو ایک جیسے پروڈکشن ماحول (بلیو اور گرین) کا استعمال کرتی ہے تاکہ ڈاؤن ٹائم اور خطرے کو کم کیا جا سکے۔

## C

**Container Apps**  
Azure کی سرور لیس کنٹینر سروس جو کنٹینرائزڈ ایپلیکیشنز کو بغیر پیچیدہ انفراسٹرکچر مینجمنٹ کے چلانے کی اجازت دیتی ہے۔

**CI/CD**  
کنٹینیوس انٹیگریشن/کنٹینیوس ڈیپلائمنٹ۔ کوڈ تبدیلیوں کو انٹیگریٹ کرنے اور ایپلیکیشنز کو ڈیپلائے کرنے کے لیے خودکار طریقے۔

**Cosmos DB**  
Azure کی عالمی سطح پر تقسیم شدہ، ملٹی ماڈل ڈیٹا بیس سروس جو تھروپٹ، لیٹینسی، دستیابی، اور مستقل مزاجی کے لیے جامع SLA فراہم کرتی ہے۔

**Configuration**  
سیٹنگز اور پیرامیٹرز جو ایپلیکیشن کے رویے اور ڈیپلائمنٹ کے اختیارات کو کنٹرول کرتے ہیں۔

## D

**Deployment**  
ایپلیکیشنز اور ان کے ڈیپینڈنسیز کو ہدف انفراسٹرکچر پر انسٹال اور کنفیگر کرنے کا عمل۔

**Docker**  
کنٹینرائزیشن ٹیکنالوجی کا استعمال کرتے ہوئے ایپلیکیشنز کو ڈیولپ، شپ، اور چلانے کے لیے پلیٹ فارم۔

**Dockerfile**  
ٹیکسٹ فائل جو Docker کنٹینر امیج بنانے کے لیے ہدایات پر مشتمل ہوتی ہے۔

## E

**Environment**  
ڈیپلائمنٹ کا ہدف جو آپ کی ایپلیکیشن کا ایک مخصوص انسٹینس ظاہر کرتا ہے (جیسے، ڈیولپمنٹ، اسٹیجنگ، پروڈکشن)۔

**Environment Variables**  
کنفیگریشن ویلیوز جو کی-ویلیو پیرز کے طور پر اسٹور کی جاتی ہیں اور ایپلیکیشنز رن ٹائم پر ان تک رسائی حاصل کر سکتی ہیں۔

**Endpoint**  
URL یا نیٹ ورک ایڈریس جہاں ایپلیکیشن یا سروس تک رسائی حاصل کی جا سکتی ہے۔

## F

**Function App**  
Azure کی سرور لیس کمپیوٹ سروس جو ایونٹ-ڈرائیون کوڈ کو بغیر انفراسٹرکچر مینجمنٹ کے چلانے کی اجازت دیتی ہے۔

## G

**GitHub Actions**  
GitHub ریپوزیٹریز کے ساتھ انٹیگریٹڈ CI/CD پلیٹ فارم جو ورک فلو کو خودکار بناتا ہے۔

**Git**  
ڈسٹریبیوٹڈ ورژن کنٹرول سسٹم جو سورس کوڈ میں تبدیلیوں کو ٹریک کرنے کے لیے استعمال ہوتا ہے۔

## H

**Hooks**  
کسٹم اسکرپٹس یا کمانڈز جو ڈیپلائمنٹ لائف سائیکل کے مخصوص پوائنٹس پر چلتے ہیں (پری پروویژن، پوسٹ پروویژن، پری ڈیپلائے، پوسٹ ڈیپلائے)۔

**Host**  
Azure سروس کی قسم جہاں ایپلیکیشن ڈیپلائے کی جائے گی (جیسے، appservice، containerapp، function)۔

## I

**Infrastructure as Code (IaC)**  
انفراسٹرکچر کو کوڈ کے ذریعے ڈیفائن اور مینج کرنے کا عمل بجائے دستی طریقوں کے۔

**Init**  
عام طور پر ٹیمپلیٹ سے ایک نیا azd پروجیکٹ شروع کرنے کا عمل۔

## J

**JSON**  
JavaScript Object Notation۔ ڈیٹا انٹرچینج فارمیٹ جو عام طور پر کنفیگریشن فائلز اور API رسپانسز کے لیے استعمال ہوتا ہے۔

**JWT**  
JSON Web Token۔ JSON آبجیکٹ کے طور پر معلومات کو محفوظ طریقے سے منتقل کرنے کا معیار۔

## K

**Key Vault**  
Azure کی سروس جو راز، چابیاں، اور سرٹیفکیٹس کو محفوظ طریقے سے اسٹور اور مینج کرتی ہے۔

**Kusto Query Language (KQL)**  
Azure Monitor، Application Insights، اور دیگر Azure سروسز میں ڈیٹا کا تجزیہ کرنے کے لیے استعمال ہونے والی کوئری لینگویج۔

## L

**Load Balancer**  
سروس جو آنے والے نیٹ ورک ٹریفک کو متعدد سرورز یا انسٹینسز میں تقسیم کرتی ہے۔

**Log Analytics**  
Azure کی سروس جو کلاؤڈ اور آن-پریمیس ماحول سے ٹیلیمیٹری ڈیٹا کو جمع، تجزیہ، اور اس پر عمل کرتی ہے۔

## M

**Managed Identity**  
Azure کی خصوصیت جو Azure سروسز کو دوسرے Azure سروسز کے ساتھ تصدیق کے لیے خودکار طور پر مینج کردہ شناخت فراہم کرتی ہے۔

**Microservices**  
آرکیٹیکچرل اپروچ جہاں ایپلیکیشنز کو چھوٹے، آزاد سروسز کے مجموعے کے طور پر بنایا جاتا ہے۔

**Monitor**  
Azure کا متحد مانیٹرنگ حل جو ایپلیکیشنز اور انفراسٹرکچر کے لیے مکمل اسٹیک آبزرویبلٹی فراہم کرتا ہے۔

## N

**Node.js**  
Chrome کے V8 JavaScript انجن پر مبنی JavaScript رن ٹائم جو سرور-سائیڈ ایپلیکیشنز بنانے کے لیے استعمال ہوتا ہے۔

**npm**  
Node.js کے لیے پیکیج مینیجر جو ڈیپینڈنسیز اور پیکیجز کو مینج کرتا ہے۔

## O

**Output**  
انفراسٹرکچر ڈیپلائمنٹ سے واپس آنے والی ویلیوز جو ایپلیکیشنز یا دیگر وسائل کے ذریعے استعمال کی جا سکتی ہیں۔

## P

**Package**  
ایپلیکیشن کوڈ اور ڈیپینڈنسیز کو ڈیپلائمنٹ کے لیے تیار کرنے کا عمل۔

**Parameters**  
انفراسٹرکچر ٹیمپلیٹس کو ڈیپلائمنٹ کو حسب ضرورت بنانے کے لیے دی جانے والی ان پٹ ویلیوز۔

**PostgreSQL**  
اوپن سورس ریلیشنل ڈیٹا بیس سسٹم جو Azure میں مینجڈ سروس کے طور پر سپورٹ کیا جاتا ہے۔

**Provisioning**  
انفراسٹرکچر ٹیمپلیٹس میں ڈیفائن کردہ Azure وسائل کو بنانے اور کنفیگر کرنے کا عمل۔

## Q

**Quota**  
Azure سبسکرپشن یا ریجن میں بنائے جانے والے وسائل کی مقدار پر حدود۔

## R

**Resource Group**  
Azure وسائل کے لیے منطقی کنٹینر جو ایک ہی لائف سائیکل، اجازتوں، اور پالیسیز کو شیئر کرتے ہیں۔

**Resource Token**  
azd کے ذریعے جنریٹ کردہ منفرد اسٹرنگ جو وسائل کے ناموں کو ڈیپلائمنٹس کے درمیان منفرد بناتی ہے۔

**REST API**  
ایپلیکیشنز کو HTTP طریقوں کا استعمال کرتے ہوئے ڈیزائن کرنے کے لیے آرکیٹیکچرل اسٹائل۔

**Rollback**  
ایپلیکیشن یا انفراسٹرکچر کنفیگریشن کے پچھلے ورژن پر واپس جانے کا عمل۔

## S

**Service**  
آپ کی ایپلیکیشن کا ایک جزو جو azure.yaml میں ڈیفائن کیا گیا ہے (جیسے، ویب فرنٹ اینڈ، API بیک اینڈ، ڈیٹا بیس)۔

**SKU**  
اسٹاک کیپنگ یونٹ۔ Azure وسائل کے لیے مختلف سروس ٹائرز یا پرفارمنس لیولز کی نمائندگی کرتا ہے۔

**SQL Database**  
Azure کی مینجڈ ریلیشنل ڈیٹا بیس سروس جو Microsoft SQL Server پر مبنی ہے۔

**Static Web Apps**  
Azure کی سروس جو سورس کوڈ ریپوزیٹریز سے مکمل اسٹیک ویب ایپلیکیشنز بنانے اور ڈیپلائے کرنے کے لیے استعمال ہوتی ہے۔

**Storage Account**  
Azure کی سروس جو ڈیٹا آبجیکٹس جیسے blobs، files، queues، اور tables کے لیے کلاؤڈ اسٹوریج فراہم کرتی ہے۔

**Subscription**  
Azure اکاؤنٹ کنٹینر جو ریسورس گروپس اور وسائل کو رکھتا ہے، اور اس کے ساتھ بلنگ اور رسائی مینجمنٹ۔

## T

**Template**  
پری بلٹ پروجیکٹ اسٹرکچر جو ایپلیکیشن کوڈ، انفراسٹرکچر ڈیفینیشنز، اور عام منظرناموں کے لیے کنفیگریشن پر مشتمل ہوتا ہے۔

**Terraform**  
اوپن سورس انفراسٹرکچر کوڈ ٹول جو Azure سمیت متعدد کلاؤڈ پرووائیڈرز کو سپورٹ کرتا ہے۔

**Traffic Manager**  
Azure کا DNS پر مبنی ٹریفک لوڈ بیلینسر جو عالمی Azure ریجنز میں ٹریفک کو تقسیم کرتا ہے۔

## U

**URI**  
یونیفارم ریسورس آئیڈینٹیفائر۔ اسٹرنگ جو کسی خاص ریسورس کی شناخت کرتی ہے۔

**URL**  
یونیفارم ریسورس لوکیٹر۔ URI کی قسم جو ریسورس کے مقام اور اسے حاصل کرنے کا طریقہ بتاتی ہے۔

## V

**Virtual Network (VNet)**  
Azure میں پرائیویٹ نیٹ ورکس کے لیے بنیادی بلڈنگ بلاک، جو آئسولیشن اور سیگمنٹیشن فراہم کرتا ہے۔

**VS Code**  
ویژول اسٹوڈیو کوڈ۔ مقبول کوڈ ایڈیٹر جو Azure اور azd کے ساتھ بہترین انٹیگریشن فراہم کرتا ہے۔

## W

**Webhook**  
HTTP کال بیک جو مخصوص ایونٹس کے ذریعے ٹرگر ہوتا ہے، عام طور پر CI/CD پائپ لائنز میں استعمال ہوتا ہے۔

**What-if**  
Azure کی خصوصیت جو دکھاتی ہے کہ ڈیپلائمنٹ کے ذریعے کیا تبدیلیاں کی جائیں گی بغیر اصل میں اسے انجام دیے۔

## Y

**YAML**  
YAML Ain't Markup Language۔ انسان کے لیے قابل مطالعہ ڈیٹا سیریلائزیشن اسٹینڈرڈ جو کنفیگریشن فائلز جیسے azure.yaml کے لیے استعمال ہوتا ہے۔

## Z

**Zone**  
Azure ریجن کے اندر جسمانی طور پر الگ الگ مقامات جو ریڈنڈنسی اور ہائی ایویلیبیلیٹی فراہم کرتے ہیں۔

## سیاق و سباق کے لحاظ سے اصطلاحات

### ڈیولپمنٹ کی اصطلاحات
- **Hot Reload**: ڈیولپمنٹ کے دوران ایپلیکیشنز کو بغیر ری اسٹارٹ کے خودکار طور پر اپڈیٹ کرنا
- **Build Pipeline**: کوڈ کو بنانے اور ٹیسٹ کرنے کے لیے خودکار عمل
- **Deployment Slot**: App Service کے اندر اسٹیجنگ ماحول
- **Environment Parity**: ڈیولپمنٹ، اسٹیجنگ، اور پروڈکشن ماحول کو ایک جیسا رکھنا

### سیکیورٹی کی اصطلاحات
- **Managed Identity**: Azure کی خصوصیت جو خودکار کریڈینشل مینجمنٹ فراہم کرتی ہے
- **Key Vault**: راز، چابیاں، اور سرٹیفکیٹس کے لیے محفوظ اسٹوریج
- **RBAC**: Azure وسائل کے لیے رول پر مبنی رسائی کنٹرول
- **Network Security Group**: نیٹ ورک ٹریفک کو کنٹرول کرنے کے لیے ورچوئل فائر وال

### مانیٹرنگ کی اصطلاحات
- **Telemetry**: پیمائش اور ڈیٹا کو خودکار طور پر جمع کرنا
- **Application Performance Monitoring (APM)**: سافٹ ویئر کی کارکردگی کی نگرانی
- **Log Analytics**: لاگ ڈیٹا کو جمع اور تجزیہ کرنے
