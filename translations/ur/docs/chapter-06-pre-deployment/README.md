# باب 6: پری ڈپلائمنٹ منصوبہ بندی اور توثیق

**📚 کورس**: [نئے صارفین کے لیے AZD](../../README.md) | **⏱️ دورانیہ**: 1 گھنٹہ | **⭐ پیچیدگی**: متوسط

---

## جائزہ

یہ باب آپ کی ایپلیکیشن کی تعیناتی سے قبل اہم منصوبہ بندی اور توثیقی اقدامات کا احاطہ کرتا ہے۔ مناسب صلاحیت کی منصوبہ بندی، SKU کا انتخاب، اور پری فلائٹ چیکس کے ذریعے مہنگی غلطیوں سے بچنا سیکھیں۔

> `azd 1.25.6` کے خلاف جون 2026 میں تصدیق شدہ۔

## سیکھنے کے مقاصد

اس باب کو مکمل کرنے کے بعد، آپ:
- تعیناتی سے پہلے پری فلائٹ چیکس چلائیں گے
- صلاحیت کی منصوبہ بندی کریں گے اور وسائل کی ضروریات کا اندازہ لگائیں گے
- لاگت کی بچت کے لیے مناسب SKUs کا انتخاب کریں گے
- نگرانی کے لیے Application Insights کو ترتیب دیں گے
- ٹیم کے اشتراک کے نمونے سمجھیں گے

## 🚀 جلدی آغاز

```bash
# رکنیت کوٹے چیک کریں
az vm list-usage --location eastus --output table

# تعیناتی کا پیش نظارہ (کوئی وسائل تخلیق نہیں کیے گئے)
azd provision --preview

# بائسک نحو کی توثیق کریں
az bicep build --file infra/main.bicep

# ماحول کی ترتیب چیک کریں
azd env get-values
```

## ☑️ پری ڈپلائمنٹ چیک لسٹ

### `azd provision` سے پہلے

- [ ] خطے کے لیے کوٹہ کی تصدیق کی گئی
- [ ] SKUs مناسب طریقے سے منتخب کیے گئے
- [ ] لاگت کا تخمینہ جائزہ لیا گیا
- [ ] نام رکھنے کا طریقہ مستقل ہے
- [ ] سیکیورٹی/RBAC ترتیب دیا گیا

### `azd deploy` سے پہلے

- [ ] ماحول کے متغیرات سیٹ کیے گئے
- [ ] خفیہ معلومات کی وراثت میں محفوظ کی گئی
- [ ] کنکشن سٹرنگز کی تصدیق کی گئی
- [ ] صحت کے چیکز ترتیب دیے گئے
