# باب 6: پری-ڈپلائمنٹ کی منصوبہ بندی اور تصدیق

**📚 کورس**: [ابتدائیوں کے لیے AZD](../../README.md) | **⏱️ دورانیہ**: 1 گھنٹہ | **⭐ پیچیدگی**: درمیانی

---

## جائزہ

یہ باب آپ کی ایپلیکیشن کی ڈپلائمنٹ سے پہلے ضروری منصوبہ بندی اور تصدیقی مراحل کا احاطہ کرتا ہے۔ مناسب صلاحیت کی منصوبہ بندی، SKU کے انتخاب، اور پری فلائٹ چیکس کے ذریعے مہنگی غلطیوں سے بچنا سیکھیں۔

> `azd 1.23.12` کے خلاف مارچ 2026 میں تصدیق شدہ۔

## سیکھنے کے مقاصد

اس باب کو مکمل کرنے پر، آپ:
- ڈپلائمنٹ سے پہلے پری فلائٹ چیکس چلائیں گے
- صلاحیت کی منصوبہ بندی کریں گے اور وسائل کی ضروریات کا تخمینہ لگائیں گے
- لاگت کی بہترنی کے لیے مناسب SKUs کا انتخاب کریں گے
- مانیٹرنگ کے لیے Application Insights کو ترتیب دیں گے
- ٹیم کی ہم آہنگی کے نمونے سمجھیں گے

## 🚀 جلد آغاز

```bash
# رکنیت کوٹہ چیک کریں
az vm list-usage --location eastus --output table

# تعیناتی کا پیش نظارہ (کوئی وسائل تخلیق نہیں کیے گئے)
azd provision --preview

# بائسِپ نحو کی جانچ کریں
az bicep build --file infra/main.bicep

# ماحول کی تشکیلات کو چیک کریں
azd env get-values
```

## ☑️ پری-ڈپلائمنٹ چیک لسٹ

### `azd provision` سے پہلے

- [ ] علاقے کے لیے کوٹہ کی تصدیق کی گئی
- [ ] SKUs مناسب طریقے سے منتخب کیے گئے
- [ ] لاگت کا تخمینہ جائزہ لیا گیا
- [ ] نامکاری کے طریقہ کار میں یکسانیت
- [ ] سیکیورٹی/RBAC ترتیب دی گئی

### `azd deploy` سے پہلے

- [ ] ماحول کے متغیرات سیٹ کیے گئے
- [ ] Key Vault میں سیکرٹس
- [ ] کنکشن سٹرنگز کی تصدیق کی گئی
- [ ] صحت کی جانچ کی ترتیب
