## تعارف

یہ سبق آپ کو Azure Developer CLI (azd) سے روشناس کراتا ہے، جو ایک طاقتور کمانڈ لائن ٹول ہے جو آپ کے لوکل ڈیولپمنٹ سے Azure پر تعیناتی کے سفر کو تیز کرتا ہے۔ آپ بنیادی تصورات، اہم خصوصیات سیکھیں گے اور سمجھیں گے کہ azd کس طرح کلاؤڈ نیٹیو ایپلیکیشن تعیناتی کو آسان بناتا ہے۔

## سیکھنے کے مقاصد

اس سبق کے آخر تک، آپ:
- سمجھ سکیں گے کہ Azure Developer CLI کیا ہے اور اس کا بنیادی مقصد کیا ہے
- ٹیمپلیٹس، ماحولیات، اور خدمات کے بنیادی تصورات سیکھیں گے
- اہم خصوصیات جیسے ٹیمپلیٹ سے چلنے والی ترقی اور انفراسٹرکچر از کوڈ کو جانچیں گے
- azd پروجیکٹ کے ڈھانچے اور ورک فلو کو سمجھیں گے
- azd کو آپ کے ترقیاتی ماحول کے لیے انسٹال اور کنفیگر کرنے کے لیے تیار ہوں گے

## سیکھنے کے نتائج

اس سبق کو مکمل کرنے کے بعد، آپ قابل ہوں گے:
- جدید کلاؤڈ ڈیولپمنٹ ورک فلو میں azd کے کردار کی وضاحت کرنا
- azd پروجیکٹ کے ساخت کے اجزاء کی شناخت کرنا
- وضاحت کرنا کہ ٹیمپلیٹس، ماحولیات، اور خدمات کس طرح ایک ساتھ کام کرتی ہیں
- azd کے ذریعے انفراسٹرکچر از کوڈ کے فوائد کو سمجھنا
- مختلف azd کمانڈز اور ان کے مقاصد کو پہچاننا

## Azure Developer CLI (azd) کیا ہے؟

Azure Developer CLI (azd) ایک کمانڈ لائن ٹول ہے جو آپ کے لوکل ڈیولپمنٹ سے Azure پر تعیناتی کے سفر کو تیز کرنے کے لیے ڈیزائن کیا گیا ہے۔ یہ Azure پر کلاؤڈ نیٹیو ایپلیکیشنز کی تعمیر، تعیناتی، اور انتظام کے عمل کو آسان بناتا ہے۔

### آپ azd کے ساتھ کیا تعینات کر سکتے ہیں؟

azd بہت سے کام کے بوجھ کی حمایت کرتا ہے—اور فہرست بڑھتی رہتی ہے۔ آج آپ azd کا استعمال کر کے تعینات کر سکتے ہیں:

| کام کا بوجھ کی قسم | مثالیں | ایک جیسا ورک فلو؟ |
|-------------------|---------|------------------|
| **روایتی ایپلیکیشنز** | ویب ایپس، REST APIs، جامد سائٹس | ✅ `azd up` |
| **خدمات اور مائیکرو سروسز** | کنٹینر ایپس، فنکشن ایپس، کثیرالخدمتی بیک اینڈز | ✅ `azd up` |
| **AI سے چلنے والی ایپلیکیشنز** | Microsoft Foundry ماڈلز کے ساتھ چیٹ ایپس، AI سرچ کے ساتھ RAG حل | ✅ `azd up` |
| **ذہین ایجنٹس** | Foundry ہوسٹ کردہ ایجنٹس، کثیر ایجنٹ آرکیسٹریشنز | ✅ `azd up` |

اہم بات یہ ہے کہ **azd لائف سائیکل اس سے قطع نظر ایک جیسا رہتا ہے کہ آپ کیا تعینات کر رہے ہیں**۔ آپ پروجیکٹ کو انیشئیلائز کرتے ہیں، انفراسٹرکچر پروویژن کرتے ہیں، اپنا کوڈ تعینات کرتے ہیں، ایپ کی نگرانی کرتے ہیں، اور صفائی کرتے ہیں—چاہے یہ ایک سادہ ویب سائٹ ہو یا ایک پیچیدہ AI ایجنٹ۔

یہ تسلسل ارادے کے مطابق ہے۔ azd AI صلاحیتوں کو آپ کی ایپلیکیشن کے استعمال کے لیے ایک اور قسم کی سروس کے طور پر دیکھتا ہے، نہ کہ کچھ بنیادی طور پر مختلف۔ Microsoft Foundry Models سے سپورٹ شدہ چیٹ اینڈپوائنٹ، azd کے نقطہ نظر سے، صرف ایک اور سروس ہے جسے کنفیگر اور تعینات کیا جانا ہے۔

### 🎯 کیوں AZD استعمال کریں؟ ایک عملی موازنہ

آئیں ایک سادہ ویب ایپ کے ڈیٹا بیس کے ساتھ تعیناتی کا موازنہ کریں:

#### ❌ بغیر AZD: دستی Azure تعیناتی (30+ منٹ)

```bash
# مرحلہ 1: ریسورس گروپ بنائیں
az group create --name myapp-rg --location eastus

# مرحلہ 2: ایپ سروس پلان بنائیں
az appservice plan create --name myapp-plan \
  --resource-group myapp-rg \
  --sku B1 --is-linux

# مرحلہ 3: ویب ایپ بنائیں
az webapp create --name myapp-web-unique123 \
  --resource-group myapp-rg \
  --plan myapp-plan \
  --runtime "NODE:18-lts"

# مرحلہ 4: کاسموس ڈی بی اکاؤنٹ بنائیں (10-15 منٹ)
az cosmosdb create --name myapp-cosmos-unique123 \
  --resource-group myapp-rg \
  --kind MongoDB

# مرحلہ 5: ڈیٹا بیس بنائیں
az cosmosdb mongodb database create \
  --account-name myapp-cosmos-unique123 \
  --resource-group myapp-rg \
  --name tododb

# مرحلہ 6: کلیکشن بنائیں
az cosmosdb mongodb collection create \
  --account-name myapp-cosmos-unique123 \
  --resource-group myapp-rg \
  --database-name tododb \
  --name todos

# مرحلہ 7: کنکشن اسٹرنگ حاصل کریں
CONN_STR=$(az cosmosdb keys list \
  --name myapp-cosmos-unique123 \
  --resource-group myapp-rg \
  --type connection-strings \
  --query "connectionStrings[0].connectionString" -o tsv)

# مرحلہ 8: ایپ سیٹنگز ترتیب دیں
az webapp config appsettings set \
  --name myapp-web-unique123 \
  --resource-group myapp-rg \
  --settings MONGODB_URI="$CONN_STR"

# مرحلہ 9: لاگنگ فعال کریں
az webapp log config --name myapp-web-unique123 \
  --resource-group myapp-rg \
  --application-logging filesystem \
  --detailed-error-messages true

# مرحلہ 10: اپلیکیشن انسائٹس سیٹ اپ کریں
az monitor app-insights component create \
  --app myapp-insights \
  --location eastus \
  --resource-group myapp-rg

# مرحلہ 11: ایپ انسائٹس کو ویب ایپ سے منسلک کریں
INSTRUMENTATION_KEY=$(az monitor app-insights component show \
  --app myapp-insights \
  --resource-group myapp-rg \
  --query "instrumentationKey" -o tsv)

az webapp config appsettings set \
  --name myapp-web-unique123 \
  --resource-group myapp-rg \
  --settings APPINSIGHTS_INSTRUMENTATIONKEY="$INSTRUMENTATION_KEY"

# مرحلہ 12: لوکل طور پر ایپلیکیشن تیار کریں
npm install
npm run build

# مرحلہ 13: ڈپلائمنٹ پیکیج بنائیں
zip -r app.zip . -x "*.git*" "node_modules/*"

# مرحلہ 14: ایپلیکیشن تعینات کریں
az webapp deployment source config-zip \
  --resource-group myapp-rg \
  --name myapp-web-unique123 \
  --src app.zip

# مرحلہ 15: انتظار کریں اور دعا کریں کہ یہ کام کرے 🙏
# (کوئی خودکار تصدیق نہیں، دستی جانچ کی ضرورت ہے)
```

**مسائل:**
- ❌ یاد رکھنے اور ترتیب میں چلانے کے لیے 15+ کمانڈز
- ❌ 30-45 منٹ کا دستی کام
- ❌ غلطیاں کرنے میں آسانی (ٹائپوز، غلط پیرامیٹرز)
- ❌ کنکشن سٹرنگز ٹرمینل ہسٹری میں ظاہر ہوتی ہیں
- ❌ ناکامی کی صورت میں خودکار رول بیک نہیں
- ❌ ٹیم کے دیگر اراکین کے لیے نقل کرنا مشکل
- ❌ ہر بار مختلف (نقل نہیں کیا جا سکتا)

#### ✅ AZD کے ساتھ: خودکار تعیناتی (5 کمانڈز، 10-15 منٹ)

```bash
# مرحلہ 1: سانچہ سے آغاز کریں
azd init --template todo-nodejs-mongo

# مرحلہ 2: تصدیق کریں
azd auth login

# مرحلہ 3: ماحول بنائیں
azd env new dev

# مرحلہ 4: تبدیلیوں کا جائزہ لیں (اختیاری لیکن تجویز کردہ)
azd provision --preview

# مرحلہ 5: سب کچھ تعینات کریں
azd up

# ✨ مکمل! سب کچھ تعینات، تشکیل شدہ، اور نگرانی کی گئی ہے
```

**فوائد:**
- ✅ **5 کمانڈز** بمقابلہ 15+ دستی مراحل
- ✅ **10-15 منٹ** کل وقت (زیادہ تر Azure کے انتظار میں)
- ✅ **صفر غلطیاں** - خودکار اور جانچی گئی
- ✅ **راز محفوظ طریقے سے منظم** شدہ کلید والٹ کے ذریعے
- ✅ **ناکامیوں پر خودکار رول بیک**
- ✅ **مکمل طور پر قابل باز پیدا** - ہر بار ایک جیسا نتیجہ
- ✅ **ٹیم کے لیے تیار** - کوئی بھی ایک جیسے کمانڈز سے تعینات کر سکتا ہے
- ✅ **انفراسٹرکچر از کوڈ** - ورژن کنٹرولڈ بائسک ٹیمپلیٹس
- ✅ **بلٹ ان مانیٹرنگ** - ایپلیکیشن انسائٹس خودکار طریقے سے کنفیگر

## بنیادی تصورات

### ٹیمپلیٹس
ٹیمپلیٹس azd کی بنیاد ہیں۔ ان میں شامل ہیں:
- **ایپلیکیشن کوڈ** - آپ کا سورس کوڈ اور انحصار
- **انفراسٹرکچر کی تعریفیں** - Azure ریسورسز جو بائسک یا Terraform میں متعین ہیں
- **کنفیگریشن فائلز** - ترتیبات اور ماحولاتی متغیرات
- **تعیناتی اسکرپٹس** - خودکار تعیناتی ورک فلو

### ماحولیات
ماحولیات مختلف تعیناتی کے ٹارگٹس کی نمائندگی کرتے ہیں:
- **ڈیولپمنٹ** - جانچ اور ترقی کے لیے
- **اسٹیجنگ** - پری پروڈکشن ماحول
- **پروڈکشن** - زندہ پروڈکشن ماحول

ہر ماحولیاتی کا اپنا ہوتا ہے:
- Azure ریسورس گروپ
- کنفیگریشن کی ترتیبات
- تعیناتی کی حالت

### خدمات
خدمات آپ کی ایپلیکیشن کے بنیادی اجزاء ہیں:
- **فرنٹ اینڈ** - ویب ایپلیکیشنز، SPAs
- **بیک اینڈ** - APIs، مائیکرو سروسز
- **ڈیٹا بیس** - ڈیٹا اسٹوریج حل
- **اسٹوریج** - فائل اور بلاک اسٹوریج

## اہم خصوصیات

### 1. ٹیمپلیٹ سے چلنے والی ترقی
```bash
# دستیاب ٹیمپلیٹس کو براؤز کریں
azd template list

# ایک ٹیمپلیٹ سے شروع کریں
azd init --template <template-name>
```

### 2. انفراسٹرکچر از کوڈ
- **بائسک** - Azure کا مخصوص زبان
- **Terraform** - ملٹی کلاؤڈ انفراسٹرکچر ٹول
- **ARM ٹیمپلیٹس** - Azure ریسورس مینیجر ٹیمپلیٹس

### 3. مربوط ورک فلو
```bash
# مکمل تعیناتی کا ورک فلو
azd up            # پروویژن + تعیناتی، پہلی بار سیٹ اپ کے لیے بغیر کسی ہاتھ کے

# 🧪 نیا: تعیناتی سے پہلے انفراسٹرکچر کی تبدیلیوں کا پیش نظارہ (محفوظ)
azd provision --preview    # تبدیلی کیے بغیر انفراسٹرکچر کی تعیناتی کی نقل کریں

azd provision     # اگر آپ انفراسٹرکچر کو اپ ڈیٹ کریں تو Azure وسائل بنائیں
azd deploy        # ایپلیکیشن کوڈ کو تعینات کریں یا اپ ڈیٹ کے بعد دوبارہ تعینات کریں
azd down          # وسائل کی صفائی کریں
```

#### 🛡️ سیف انفراسٹرکچر پلاننگ پریویو کے ساتھ
`azd provision --preview` کمانڈ محفوظ تعیناتیوں کے لیے ایک کھیل بدلنے والا ہے:
- **ڈرائی رن تجزیہ** - دکھاتا ہے کہ کیا بنایا جائے گا، ترمیم کی جائے گی، یا حذف کیا جائے گا
- **زیرو رسک** - آپ کے Azure ماحول میں کوئی اصل تبدیلی نہیں ہوتی
- **ٹیم تعاون** - تعیناتی سے پہلے پریویو نتائج شیئر کریں
- **لاگت کا اندازہ** - التزام سے پہلے وسائل کی لاگت سمجھیں

```bash
# پیش نظارہ کام کا بہاؤ کی مثال
azd provision --preview           # دیکھیں کیا بدلے گا
# نتیجہ کا جائزہ لیں، ٹیم سے بات کریں
azd provision                     # تبدیلیاں اعتماد کے ساتھ لاگو کریں
```

**ورک فلو کی وضاحت:**
1. **Init** - ٹیمپلیٹ یا نیا پروجیکٹ شروع کریں
2. **Auth** - Azure کے ساتھ تصدیق کریں
3. **Environment** - علیحدہ تعیناتی ماحولیاتی تخلیق کریں
4. **Preview** - 🆕 انفراسٹرکچر تبدیلیوں کا ہمیشہ پہلے جائزہ لیں (محفوظ عمل)
5. **Provision** - Azure وسائل بنائیں/اپ ڈیٹ کریں
6. **Deploy** - اپنی ایپلیکیشن کوڈ کو پش کریں
7. **Monitor** - ایپلیکیشن کی کارکردگی کا مشاہدہ کریں
8. **Iterate** - تبدیلیاں کریں اور کوڈ دوبارہ تعینات کریں
9. **Cleanup** - استعمال کے بعد وسائل کو ہٹائیں

### 4. ماحولیات کا انتظام
```bash
# ماحول تخلیق کریں اور ان کا انتظام کریں
azd env new <environment-name>
azd env select <environment-name>
azd env list
```

### 5. ایکسٹینشنز اور AI کمانڈز

azd ایکسٹینشن سسٹم استعمال کرتا ہے تاکہ بنیادی CLI سے آگے صلاحیتیں شامل کی جا سکیں۔ یہ خاص طور پر AI کام کے بوجھ کے لیے مفید ہے:

```bash
# دستیاب ایکسٹینشنز کی فہرست بنائیں
azd extension list

# فاؤنڈری ایجنٹس ایکسٹینشن انسٹال کریں
azd extension install azure.ai.agents

# ایک مینیفیسٹ سے AI ایجنٹ پراجیکٹ شروع کریں
azd ai agent init -m agent-manifest.yaml

# AI معاونت یافتہ ترقی کے لیے MCP سرور شروع کریں (الفا)
azd mcp start
```

**استعمال کریں جب:**
- تعیناتی درمیان میں ناکام ہو گئی ہو

- نیا آغاز چاہیے

## 🎪 اصل ورک فلو ریفرنس

### نیا پروجیکٹ شروع کرنا
```bash
# طریقہ 1: موجودہ ٹیمپلیٹ استعمال کریں
azd init --template todo-nodejs-mongo

# طریقہ 2: صفر سے شروع کریں
azd init

# طریقہ 3: موجودہ ڈائریکٹری استعمال کریں
azd init .
```

### ترقیاتی سائیکل
```bash
# ترقیاتی ماحول قائم کریں
azd auth login
azd env new dev
azd env select dev

# سب کچھ تعینات کریں
azd up

# تبدیلیاں کریں اور دوبارہ تعینات کریں
azd deploy

# کام مکمل ہونے پر صفائی کریں
azd down --force --purge # Azure Developer CLI میں کمانڈ آپ کے ماحول کے لیے **سخت ری سیٹ** ہے—خاص طور پر تب مفید جب آپ ناکام تعیناتیاں ٹھیک کر رہے ہوں، چھوڑ دیے گئے وسائل کی صفائی کر رہے ہوں، یا دوبارہ تعیناتی کے لیے تیار ہو رہے ہوں۔
```

## `azd down --force --purge` کی سمجھ

`azd down --force --purge` کمانڈ آپ کے azd ماحول اور اس سے منسلک تمام وسائل کو مکمل طور پر ختم کرنے کا طاقتور طریقہ ہے۔ ہر فلیگ کیا کرتا ہے اس کی تفصیل درج ذیل ہے:

```
--force
```
- تصدیقی پیغامات کو نظر انداز کرتا ہے۔  
- خودکاری یا اسکرپٹنگ کے لیے مفید جہاں دستی اندراج ممکن نہ ہو۔  
- یقینی بناتا ہے کہ اگر CLI تضادات پہچانے بھی تو رکاوٹ کے بغیر ختم کرے۔

```
--purge
```
تمام متعلقہ میٹا ڈیٹا حذف کر دیتا ہے، بشمول:  
ماحولیاتی حالت  
مقامی `.azure` فولڈر  
کیچ شدہ تعیناتی معلومات  
azd کو پچھلی تعیناتیاں "یاد رکھنے" سے روکتا ہے، جو وسائل گروپس کے عدم مطابقت یا پرانے رجسٹری حوالہ جات جیسے مسائل پیدا کر سکتی ہیں۔

### دونوں کیوں استعمال کریں؟

جب `azd up` کے ساتھ ناپسندیدہ حالت یا جزوی تعیناتیوں کی وجہ سے رکاوٹ پیش آئے، تو یہ امتزاج ایک **صاف صفحہ** کو یقینی بناتا ہے۔

یہ خاص طور پر Azure پورٹل میں دستی وسائل کو حذف کرنے کے بعد یا ٹیمپلیٹس، ماحولیات، یا ریسورس گروپ ناموں کی تبدیلی کے وقت مفید ہے۔

### متعدد ماحولیات کا انتظام
```bash

azd env new staging
azd env select staging
azd up

# دوبارہ ڈیولپمنٹ میں جائیں
```

## 🔐 تصدیق کاری اور اسناد

تصدیق کی سمجھ azd تعیناتیوں میں کامیابی کے لیے بہت ضروری ہے۔ Azure متعدد تصدیقی طریقے استعمال کرتا ہے، اور azd وہی اسناد چین استعمال کرتا ہے جو دوسرے Azure ٹولز استعمال کرتے ہیں۔

### Azure CLI تصدیق (`az login`)

azd استعمال کرنے سے پہلے، آپ کو Azure کے ساتھ تصدیق کرنی ہوگی۔ سب سے عام طریقہ Azure CLI کا استعمال ہے:

```bash
# انٹرایکٹو لاگ ان (براؤزر کھولتا ہے)
az login

# مخصوص کرایہ دار کے ساتھ لاگ ان کریں
az login --tenant <tenant-id>

# سروس پرنسپل کے ساتھ لاگ ان کریں
az login --service-principal -u <app-id> -p <password> --tenant <tenant-id>

# موجودہ لاگ ان کی حالت چیک کریں
az account show

# دستیاب سبسکرپشنز کی فہرست بنائیں
az account list --output table

# ڈیفالٹ سبسکرپشن سیٹ کریں
az account set --subscription <subscription-id>
```

### تصدیقی فلو
1. **انٹریکٹو لاگ ان**: آپ کے ڈیفالٹ براؤزر کو کھول کر تصدیق
2. **ڈیوائس کوڈ فلو**: ان ماحول کے لیے جن میں براؤزر تک رسائی نہیں
3. **سروس پرنسپل**: خودکاری اور CI/CD میں استعمال کے لیے
4. **مینجڈ آئیڈینٹٹی**: Azure پر میزبانی شدہ ایپلیکیشنز کے لیے

### DefaultAzureCredential چین

`DefaultAzureCredential` ایک ایسا قسم کا کرڈینشل ہے جو ایک سادہ تصدیقی تجربہ فراہم کرتا ہے اور خود بخود متعدد کرڈینشل ذرائع کو مخصوص ترتیب میں آزما جاتا ہے:

#### کرڈینشل چین کی ترتیب
```mermaid
graph TD
    A[DefaultAzureCredential] --> B[محولیاتی متغیرات]
    B --> C[ورک لوڈ شناخت]
    C --> D[منظم شناخت]
    D --> E[وزوئل اسٹوڈیو]
    E --> F[وزوئل اسٹوڈیو کوڈ]
    F --> G[ایزور CLI]
    G --> H[ایزور پاورشیل]
    H --> I[انٹرایکٹو براؤزر]
```
#### 1. ماحول کے متغیرات
```bash
# سروس پرنسپل کے لیے ماحول کے متغیرات سیٹ کریں
export AZURE_CLIENT_ID="<app-id>"
export AZURE_CLIENT_SECRET="<password>"
export AZURE_TENANT_ID="<tenant-id>"
```

#### 2. ورکلوڈ آئیڈینٹٹی (Kubernetes/GitHub Actions)
خودکار طریقے سے استعمال ہوتا ہے:  
- Azure Kubernetes Service (AKS) میں ورکلوڈ آئیڈینٹٹی کے ساتھ  
- GitHub Actions میں OIDC فیڈریشن کے ساتھ  
- دیگر وفاقی شناخت کے مناظر

#### 3. مینجد آئیڈینٹٹی
Azure وسائل کے لیے جیسے:  
- ورچوئل مشینز  
- ایپ سروس  
- Azure فنکشنز  
- کنٹینر انسٹینسز

```bash
# چیک کریں کہ آیا Azure وسائل پر منیجڈ شناخت کے ساتھ چل رہا ہے
az account show --query "user.type" --output tsv
# واپس کرتا ہے: "servicePrincipal" اگر منیجڈ شناخت استعمال کی جا رہی ہو
```

#### 4. ڈیولپر ٹولز انٹیگریشن
- **Visual Studio**: خودکار طریقے سے سائن ان اکاؤنٹ استعمال کرتا ہے  
- **VS Code**: Azure اکاؤنٹ ایکسٹینشن کی اسناد استعمال کرتا ہے  
- **Azure CLI**: `az login` اسناد استعمال کرتا ہے (لوکل ترقی کے لیے سب سے عام)

### AZD تصدیقی سیٹ اپ

```bash
# طریقہ 1: Azure CLI استعمال کریں (ترقی کے لیے سفارش کی جاتی ہے)
az login

# طریقہ 2: براہ راست azd توثیق
azd auth login --use-device-code  # ہیڈ لیس ماحول کے لیے

# طریقہ 3: توثیق کی حیثیت چیک کریں
azd auth login --check-status

# طریقہ 4: لاگ آؤٹ کریں اور دوبارہ توثیق کریں
azd auth logout
azd auth login
```

### تصدیق کے بہترین طریقے

#### لوکل ڈیولپمنٹ کے لیے
```bash

# 2. صحیح سبسکرپشن کی تصدیق کریں

# 3. موجودہ اسناد کے ساتھ azd استعمال کریں

```

#### CI/CD پائپ لائنز کے لیے
```yaml

- name: Azure Login
  uses: azure/login@v1
  with:
    creds: ${{ secrets.AZURE_CREDENTIALS }}

- name: Deploy with azd
```

#### پروڈکشن ماحول کے لیے
- **مینجڈ آئیڈینٹٹی** استعمال کریں جب Azure وسائل پر چل رہے ہوں  
- خودکاری کے مناظر کے لیے **سروس پرنسپل** استعمال کریں  
- کوڈ یا کنفیگریشن فائلوں میں اسناد محفوظ نہ کریں  
- حساس کنفیگریشن کے لیے **Azure Key Vault** استعمال کریں

- مقامی مشین سے براہِ راست پروڈکشن میں تعیناتی کبھی نہ کریں  
- پروڈکشن کی تعیناتی کے لیے CI/CD پائپ لائنز استعمال کریں

### 4. کنفیگریشن مینجمنٹ  
- حساس ڈیٹا کے لیے ماحول کے متغیرات استعمال کریں  
- کنفیگریشن کو ورژن کنٹرول میں رکھیں  
- ماحول مخصوص ترتیبات کی دستاویزات بنائیں

## سیکھنے کی پیش رفت

### مبتدی (ہفتہ 1-2)  
1. azd انسٹال کریں اور توثیق کریں  
2. ایک سادہ ٹیمپلیٹ تعینات کریں  
3. پروجیکٹ کے ڈھانچے کو سمجھیں  
4. بنیادی کمانڈز سیکھیں (up, down, deploy)

### درمیانہ سطح (ہفتہ 3-4)  
1. ٹیمپلیٹس کو حسبِ ضرورت بنائیں  
2. متعدد ماحول کا انتظام کریں  
3. انفراسٹرکچر کوڈ سمجھیں  
4. CI/CD پائپ لائنز ترتیب دیں

### اعلیٰ سطح (ہفتہ 5+)  
1. حسبِ ضرورت ٹیمپلیٹس بنائیں  
2. اعلیٰ درجے کے انفراسٹرکچر پیٹرنز  
3. ملٹی ریجن کی تعیناتیاں  
4. انٹرپرائز سطح کی کنفیگریشنز

## 🙋 اکثر پوچھے جانے والے سوالات

### عمومی سوالات

**س: AZD اور Azure CLI میں کیا فرق ہے؟**

ج: Azure CLI (`az`) انفرادی Azure وسائل کو مینج کرنے کے لیے ہے۔ AZD (`azd`) پورے ایپلیکیشنز کو مینج کرنے کے لیے ہے:  

```bash
# آزور CLI - کم سطح کا وسائل کا انتظام
az webapp create --name myapp --resource-group rg
az sql server create --name myserver --resource-group rg
# ...مزید بہت سے احکامات کی ضرورت ہے

# AZD - ایپلیکیشن سطح کا انتظام
azd up  # تمام وسائل کے ساتھ پوری ایپ کو تعینات کرتا ہے
```
  
**اسے یوں سوچیں:**  
- `az` = انفرادی لیگو بلاکس پر کام کرنا  
- `azd` = مکمل لیگو سیٹ کے ساتھ کام کرنا

---

**س: کیا AZD استعمال کرنے کے لیے مجھے Bicep یا Terraform آنا چاہیے؟**

ج: نہیں! ٹیمپلیٹس سے شروع کریں:  
```bash
# موجودہ ٹیمپلیٹ استعمال کریں - IaC کا کوئی علم درکار نہیں
azd init --template todo-nodejs-mongo
azd up
```
  
آپ بعد میں Bicep سیکھ کر انفراسٹرکچر کو اپنی مرضی کے مطابق بنا سکتے ہیں۔ ٹیمپلیٹس کام کرنے والے نمونے فراہم کرتے ہیں جن سے سیکھا جا سکتا ہے۔

---

**س: AZD ٹیمپلیٹس چلانے کی کتنی لاگت آتی ہے؟**

ج: لاگت ٹیمپلیٹ کے مطابق مختلف ہوتی ہے۔ زیادہ تر ترقی کے ٹیمپلیٹس $50-150/ماه خرچ آتے ہیں:  

```bash
# تعیناتی سے پہلے اخراجات کا پیش نظارہ کریں
azd provision --preview

# ہمیشہ استعمال نہ کرنے پر صفائی کریں
azd down --force --purge  # تمام وسائل کو ہٹا دیتا ہے
```
  
**پرو ٹپ:** جہاں ممکن ہو مفت درجات استعمال کریں:  
- ایپ سروس: F1 (مفت) ٹئیر  
- Microsoft Foundry ماڈلز: Azure OpenAI 50,000 ٹوکنز/ماه مفت  
- Cosmos DB: 1000 RU/s مفت ٹئیر

---

**س: کیا میں AZD کو موجودہ Azure وسائل کے ساتھ استعمال کر سکتا ہوں؟**

ج: ہاں، لیکن تازہ شروع کرنا زیادہ آسان ہے۔ AZD بہترین کام کرتا ہے جب وہ پوری لائف سائیکل کو مینج کرے۔ موجودہ وسائل کے لیے:  

```bash
# اختیار 1: موجودہ وسائل درآمد کریں (اعلی درجے کا)
azd init
# پھر infra/ کو موجودہ وسائل کی طرف حوالہ دینے کے لیے تبدیل کریں

# اختیار 2: نیا آغاز کریں (تجویز کردہ)
azd init --template matching-your-stack
azd up  # نیا ماحول بناتا ہے
```
  
---

**س: میں اپنا پروجیکٹ ساتھیوں کے ساتھ کیسے شیئر کروں؟**

ج: AZD پروجیکٹ کو Git میں کمیٹ کریں (لیکن `.azure` فولڈر نہیں):  

```bash
# پہلے سے ہی .gitignore میں بطور ڈیفالٹ شامل ہے
.azure/        # راز اور ماحول کے ڈیٹا پر مشتمل ہے
*.env          # ماحول کے متغیرات

# ٹیم کے ارکان پھر:
git clone <your-repo>
azd auth login
azd env new <their-name>-dev
azd up
```
  
ہر کوئی ایک ہی انفراسٹرکچر حاصل کرتا ہے جو ایک جیسے ٹیمپلیٹس سے بنتا ہے۔

---

### مسئلہ حل کرنے کے سوالات

**س: "azd up" آدھے راستے میں ناکام ہو گیا، میں کیا کروں؟**

ج: ایرر دیکھیں، اسے درست کریں، پھر دوبارہ کوشش کریں:  

```bash
# تفصیلی لاگز دیکھیں
azd show

# عام اصلاحات:

# 1. اگر کوٹا ختم ہو گیا ہو:
azd env set AZURE_LOCATION "westus2"  # مختلف خطہ آزماؤ

# 2. اگر وسائل کے نام میں تصادم ہو:
azd down --force --purge  # صاف آغاز
azd up  # دوبارہ کوشش کریں

# 3. اگر توثیق کی میعاد ختم ہو گئی ہو:
az login
azd auth login
azd up
```
  
**سب سے عام مسئلہ:** غلط Azure سبسکرپشن منتخب ہونا  
```bash
az account list --output table
az account set --subscription "<correct-subscription>"
```
  
---

**س: کیا میں کوڈ کی تبدیلیاں صرف deploy کر سکتا ہوں بغیر infrastructure کو دوبارہ بنانے کے؟**

ج: `azd up` کی بجائے `azd deploy` استعمال کریں:  

```bash
azd up          # پہلی بار: فراہمی + تعیناتی (سست)

# کوڈ میں تبدیلیاں کریں...

azd deploy      # بعد کے اوقات: صرف تعیناتی (تیز)
```
  
رفتار کا موازنہ:  
- `azd up`: 10-15 منٹ (انفراسٹرکچر تیار کرتا ہے)  
- `azd deploy`: 2-5 منٹ (صرف کوڈ)

---

**س: کیا میں انفراسٹرکچر ٹیمپلیٹس کو حسبِ ضرورت بنا سکتا ہوں؟**

ج: ہاں! `infra/` میں Bicep فائلوں کو ایڈٹ کریں:  

```bash
# ازد انِٹ کے بعد
cd infra/
code main.bicep  # وی ایس کوڈ میں ترمیم کریں

# تبدیلیوں کو پیش نظارہ کریں
azd provision --preview

# تبدیلیاں نافذ کریں
azd provision
```
  
**ٹپ:** چھوٹے چھوٹے شروع کریں - پہلے SKUs تبدیل کریں:  
```bicep
// infra/main.bicep
sku: {
  name: 'B1'  // Change to 'P1V2' for production
}
```
  
---

**س: میں AZD کے ذریعے بنائے گئے تمام وسائل کیسے حذف کروں؟**

ج: ایک کمانڈ سے تمام وسائل حذف ہو جاتے ہیں:  

```bash
azd down --force --purge

# یہ حذف کرتا ہے:
# - تمام آژور وسائل
# - وسائل کا گروپ
# - مقامی ماحولیاتی حالت
# - کیشڈ تعیناتی کا ڈیٹا
```
  
**ہمیشہ چلائیں جب:**  
- ٹیمپلیٹ کی جانچ مکمل ہو جائے  
- مختلف پروجیکٹ پر شفٹ ہونا ہو  
- نیا آغاز کرنا ہو

**لاگت کی بچت:** غیر ضروری وسائل کو حذف کرنے سے $0 چارجز لگتے ہیں

---

**س: اگر میں نے غلطی سے Azure پورٹل میں وسائل حذف کر دیے تو کیا کروں؟**

ج: AZD کی حالت ہم آہنگی سے باہر ہو سکتی ہے۔ صاف آغاز کا طریقہ:  

```bash
# 1. مقامی حالت کو ہٹائیں
azd down --force --purge

# 2. نیا آغاز کریں
azd up

# متبادل: AZD کو شناخت اور درستگی کرنے دیں
azd provision  # غائب وسائل تخلیق کرے گا
```
  
---

### اعلیٰ سطح کے سوالات

**س: کیا میں AZD کو CI/CD پائپ لائنز میں استعمال کر سکتا ہوں؟**

ج: ہاں! GitHub Actions کی مثال:  

```yaml
# .github/workflows/deploy.yml
name: Deploy with AZD

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      
      - name: Install azd
        run: curl -fsSL https://aka.ms/install-azd.sh | bash
      
      - name: Azure Login
        run: |
          azd auth login \
            --client-id ${{ secrets.AZURE_CLIENT_ID }} \
            --client-secret ${{ secrets.AZURE_CLIENT_SECRET }} \
            --tenant-id ${{ secrets.AZURE_TENANT_ID }}
      
      - name: Deploy
        run: azd up --no-prompt
```
  
---

**س: میں سیکریٹس اور حساس ڈیٹا کو کیسے ہینڈل کروں؟**

ج: AZD خود بخود Azure Key Vault کے ساتھ انضمام کرتا ہے:  

```bash
# راز کوڈ میں نہیں بلکہ کی والٹ میں محفوظ کیے جاتے ہیں
azd env set DATABASE_PASSWORD "$(openssl rand -base64 32)"

# AZD خودکار طور پر:
# 1. کی والٹ بناتا ہے
# 2. راز محفوظ کرتا ہے
# 3. ایپ کو مینیجڈ آئیڈینٹیٹی کے ذریعے رسائی دیتا ہے
# 4. رن ٹائم پر انجیکٹ کرتا ہے
```
  
**کبھی کمیٹ نہ کریں:**  
- `.azure/` فولڈر (ماحول کا ڈیٹا)  
- `.env` فائلیں (مقامی سیکریٹس)  
- کنکشن سٹرنگز

---

**س: کیا میں متعدد خطوں میں تعینات کر سکتا ہوں؟**

ج: ہاں، ہر خطے کے لیے الگ ماحول بنائیں:  

```bash
# مشرقی امریکہ کا ماحول
azd env new prod-eastus
azd env set AZURE_LOCATION eastus
azd up

# مغربی یورپ کا ماحول
azd env new prod-westeurope
azd env set AZURE_LOCATION westeurope
azd up

# ہر ماحول خودمختار ہے
azd env list
```
  
حقیقی ملٹی ریجن ایپس کے لیے، Bicep ٹیمپلیٹس کو ایک ساتھ کئی خطوں میں تعینات کرنے کے لیے حسبِ ضرورت بنائیں۔
