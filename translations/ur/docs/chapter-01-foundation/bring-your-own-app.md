# اپنی موجودہ ایپ لائیں - ایک موجودہ پروجیکٹ میں azd شامل کریں

**باب کی نیویگیشن:**
- **📚 Course Home**: [AZD برائے مبتدی](../../README.md)
- **📖 موجودہ باب**: باب 1 - بنیاد اور فوری شروعات
- **⬅️ پچھلا**: [آپ کا پہلا پروجیکٹ](first-project.md)
- **➡️ اگلا**: [Dev کنٹینرز اور Codespaces](dev-containers.md)

> جون 2026 میں `azd 1.25.6` کے ساتھ تصدیق شدہ۔

## تعارف

In [آپ کا پہلا پروجیکٹ](first-project.md) you deployed an app by starting from a template. لیکن اکثر اوقات آپ کے پاس پہلے ہی ایک ایپ موجود ہوتی ہے — ایک Node.js API، ایک Python Flask سروس، ایک .NET ویب ایپ — جو آپ کی مشین میں کسی فولڈر میں پڑی ہوتی ہے۔ یہ سبق دکھاتا ہے کہ اس موجودہ کوڈ میں azd کیسے شامل کریں تاکہ آپ اسے `azd up` کے ذریعے ڈپلائے کر سکیں، کسی ٹیمپلیٹ کی ضرورت نہیں۔

## سیکھنے کے مقاصد

سبق کے اختتام تک آپ:
- azd پروجیکٹ شروع کرنے کے تین طریقوں کو سمجھیں گے
- موجودہ کوڈ بیس میں `azd init` چلائیں گے
- سمجھیں گے کہ `azure.yaml` اور `infra/` فولڈر آپ کی ایپ کے لیے کیا کرتے ہیں
- جانیں گے کب azd انفراسٹرکچر جنریٹ کرے اور کب آپ خود لکھیں
- اپنی موجودہ ایپ کو Azure پر `azd up` کے ساتھ ڈپلائے کریں گے

## سیکھنے کے نتائج

یہ سبق مکمل کرنے کے بعد آپ قادر ہوں گے کہ:
- azd کو ایسے پروجیکٹ میں initialize کریں جو آپ کے پاس پہلے سے موجود ہو
- ایک بنیادی `azure.yaml` فائل کو پڑھیں اور ایڈٹ کریں
- `azd infra generate` کے ساتھ اسٹارٹر انفراسٹرکچر جنریٹ کریں
- اپنی ایپ کے لیے مناسب Azure ہوسٹ کا انتخاب کریں
- اپنی درخواست کو ڈپلائے اور کلین اپ کریں

---

## azd پروجیکٹ شروع کرنے کے تین طریقے

| آغاز نقطہ | کمانڈ | استعمال کرنے کا وقت |
|----------------|---------|-------------|
| **ایک ٹیمپلیٹ سے** | `azd init --template <name>` | سیکھنے کے لیے، یا ایک ثابت شدہ نمونے سے نیا ایپ شروع کرنے کے لیے |
| **اپنے موجودہ کوڈ سے** | `azd init` (آپ کے پروجیکٹ فولڈر میں) | آپ کے پاس پہلے سے ایک ایپ موجود ہے اور آپ اسے ڈپلائے کرنا چاہتے ہیں |
| **Git ریپو سے** | `azd init --from-code` (کلون کیے گئے ریپو میں) | موجودہ ریپوزیٹری میں azd کو اپنانا |

آپ نے پہلے آپشن کی مشق کی تھی۔ یہ سبق دوسرا کور کرتا ہے — جو کہ حقیقی دنیا کا سب سے عام منظرنامہ ہے۔

## مرحلہ 1: اپنے پروجیکٹ میں `azd init` چلائیں

Open a terminal **اپنے موجودہ پروجیکٹ فولڈر کے اندر** اور چلائیں:

```bash
cd my-existing-app
azd init
```

azd آپ سے پوچھے گا کہ آپ کس طرح initialize کرنا چاہتے ہیں۔ منتخب کریں:

```
? How do you want to initialize your app?
> Use code in the current directory
  Select a template
```

منتخب کریں **"موجودہ ڈائریکٹری میں موجود کوڈ استعمال کریں۔"** azd پھر آپ کے فولڈر کو اسکین کرتا ہے، آپ کی زبان اور فریم ورک کا پتہ لگاتا ہے، اور ایک ہوسٹ تجویز کرتا ہے۔

### azd کیا پتہ لگاتا ہے

azd ایسے اشارے تلاش کرتا ہے جیسے `package.json`، `requirements.txt`، `pom.xml`، `*.csproj`، یا `Dockerfile`، اور ایک میچ کرنے والا Azure ہوسٹ تجویز کرتا ہے:

| آپ کی ایپ | ممکنہ طور پر پتہ لگایا گیا ہوسٹ |
|----------|----------------------|
| Node.js / Python / .NET ویب ایپ | Azure App Service یا Container Apps |
| کنٹینر شدہ ایپ (`Dockerfile`) | Azure Container Apps |
| Function ایپ | Azure Functions |
| Static سائٹ (React/Vue کے build نتائج) | Azure Static Web Apps |

تجویز کردہ سروس(ز) کی تصدیق کریں، اور azd وہ فائلیں scaffold کر دے گا جن کی آپ کو ضرورت ہے۔

## مرحلہ 2: سمجھیں کہ azd نے کیا بنایا

init کے بعد، آپ کے پروجیکٹ میں دو نئی چیزیں ہوں گی:

```
my-existing-app/
├── azure.yaml          # ← NEW: tells azd what to deploy and where
├── infra/              # ← NEW: Infrastructure as Code (Bicep by default)
│   ├── main.bicep
│   └── main.parameters.json
├── src/ (your code)    # unchanged
└── ...                 # your existing files, untouched
```

### `azure.yaml` — پروجیکٹ کی تعریف

یہ azd پروجیکٹ کا دل ہے۔ ایک کم از کم فائل ایسی دکھتی ہے:

```yaml
# azure.yaml
name: my-existing-app
services:
  web:
    project: ./src           # path to your app code
    language: js             # js | python | dotnet | java
    host: appservice         # appservice | containerapp | function | staticwebapp
```

`services` بلاک اہم حصہ ہے: ہر اندراج آپ کے کوڈ کے ایک فولڈر کو Azure ہوسٹ سے میپ کرتا ہے۔ اگر آپ کی ایپ میں فرنٹ اینڈ اور API دونوں ہیں، تو آپ کے پاس دو سروسز ہوں گی۔

### `infra/` — آپ کے Azure وسائل بطور کوڈ

`infra/` فولڈر میں وہ Bicep فائلیں ہوتی ہیں جو آپ کی ایپ کو درکار Azure وسائل (App Service، ڈیٹا بیس وغیرہ) کی تعریف کرتی ہیں۔ آپ کو انہیں خود لکھنے کی ضرورت نہیں — azd ایک کام کرنے والا آغاز نقطہ جنریٹ کرتا ہے۔ آپ انہیں بعد میں ایڈٹ کر کے وسائل شامل کر سکتے ہیں یا سیکیورٹی سخت کر سکتے ہیں (یہ [باب 4](../chapter-04-infrastructure/README.md) میں زیرِ بحث ہے)۔

> **Tip:** کیا آپ جنریٹ شدہ انفراسٹرکچر کو ڈپلائے کرنے سے پہلے دیکھنا یا حسبِ ضرورت بنانا چاہتے ہیں؟ `azd infra generate` چلائیں (جو `azd infra synth` کے طور پر بھی دستیاب ہے) تاکہ IaC ڈسک پر لکھا جائے اور آپ اسے ریویو اور ورژن کنٹرول کرسکیں۔

## مرحلہ 3: ضروری کنفیگریشن سیٹ کریں

اگر آپ کی ایپ کو سیٹنگز یا سیکریٹس درکار ہیں (مثلاً کنکشن سٹرنگ، API کلید)، تو انہیں ہارڈ کوڈ نہ کریں۔ ماحول کے ویلیوز استعمال کریں:

```bash
# ماحول بنائیں
azd env new dev

# ایک غیر خفیہ قدر مقرر کریں
azd env set API_VERSION 1.0.0
```

حقیقی سیکریٹس کے لیے، انہیں Key Vault میں اسٹور کریں اور اپنے انفراسٹرکچر سے ریفرنس کریں — دیکھیں [باب 3: کنفیگریشن اور توثیق](../chapter-03-configuration/authsecurity.md)۔

---

## مرحلہ 4: ڈپلائے کریں

اب وہی ورک فلو استعمال کریں جو آپ پہلے سے جانتے ہیں:

```bash
# توثیق کریں (azd کے لیے ضروری)
azd auth login

# ان وسائل کا پیش نظارہ کریں جو بنائے جائیں گے
azd provision --preview

# انفراسٹرکچر فراہم کریں اور اپنا کوڈ تعینات کریں
azd up
```

جب یہ مکمل ہو جائے گا، azd آپ کی ایپ کا URL پرنٹ کرے گا۔ اسے وہی طریقے سے ویریفائی کریں جیسا کہ کسی بھی azd ایپ کے لیے کرتے ہیں:

```bash
azd show           # اینڈ پوائنٹس دکھائیں
azd monitor --logs # ضرورت پڑنے پر لاگز چیک کریں
```

## عام ابتدائی مسائل

| علامت | ممکنہ وجہ | حل |
|---------|--------------|-----|
| azd نے میری ایپ detect نہیں کی | مینفیسٹ غائب ہے (مثلاً `package.json`) | مینفیسٹ شامل کریں، یا `azd init` کے دوران ہوسٹ دستی طور پر منتخب کریں |
| `azd up` کے دوران build ناکام ہو جاتا ہے | ایپ کو build قدم کی ضرورت ہے | سروس کے تحت `azure.yaml` میں `buildCommand`/`outputPath` شامل کریں |
| ایپ شروع تو ہوتی ہے مگر ایررز واپس کرتی ہے | کنفیگ/سیکریٹ غائب ہے | `azd env set` کے ساتھ ویلیوز سیٹ کریں یا Key Vault جوڑیں |
| غلط ہوسٹ منتخب ہوا | آٹو-ڈٹیکشن نے اندازہ لگایا | `azure.yaml` میں `host:` ایڈٹ کریں اور دوبارہ `azd up` چلائیں |

مزید کے لیے دیکھیں [باب 7: مسائل کا حل](../chapter-07-troubleshooting/README.md)۔

```bash
azd down --force --purge
```

## خلاصہ

- `azd init` → **"موجودہ ڈائریکٹری میں موجود کوڈ استعمال کریں"** آپ کی پہلے سے موجود ایپ میں azd شامل کرتا ہے۔
- `azure.yaml` آپ کے کوڈ فولڈرز کو Azure ہوسٹس سے میپ کرتا ہے؛ `infra/` وسائل کو Bicep کے طور پر بیان کرتا ہے۔
- `azd infra generate` آپ کو جنریٹ شدہ انفراسٹرکچر کا جائزہ لینے یا حسبِ ضرورت بنانے دیتا ہے۔
- ایک بار initialize ہونے کے بعد، آپ کی موجودہ ایپ وہی `azd up` / `azd down` ورک فلو استعمال کرتی ہے جو ٹیمپلیٹ بیسڈ ایپ استعمال کرتی ہے۔
