# اپنا ایپ خود لائیں - ایک موجودہ پروجیکٹ میں azd شامل کریں

**باب کی نیویگیشن:**
- **📚 کورس ہوم**: [ابتدائیوں کے لیے AZD](../../README.md)
- **📖 موجودہ باب**: باب 1 - بنیاد اور فوری آغاز
- **⬅️ پچھلا**: [آپ کا پہلا پروجیکٹ](first-project.md)
- **➡️ اگلا**: [ڈیو کنٹینرز اور کوڈ اسپیسز](dev-containers.md)

> جولائی 2026 میں `azd 1.27.1` کے خلاف تصدیق شدہ۔

## تعارف

[آپ کا پہلا پروجیکٹ](first-project.md) میں آپ نے ایک ٹیمپلیٹ سے شروع کرکے ایک ایپ کو تعینات کیا تھا۔ لیکن زیادہ تر وقت آپ کے پاس پہلے سے ہی ایک ایپ ہوتی ہے — Node.js API، Python Flask سروس، .NET ویب ایپ — جو آپ کی مشین کے کسی فولڈر میں موجود ہوتی ہے۔ یہ سبق دکھاتا ہے کہ کیسے موجودہ کوڈ میں azd شامل کیا جائے تاکہ آپ اسے `azd up` کے ذریعے تعینات کر سکیں، بغیر کسی ٹیمپلیٹ کی ضرورت کے۔

## سیکھنے کے مقاصد

اس سبق کے آخر تک، آپ:
- azd پروجیکٹ شروع کرنے کے تین طریقے سمجھیں گے
- موجودہ کوڈ بیس میں `azd init` چلائیں گے
- سمجھیں گے کہ `azure.yaml` اور `infra/` فولڈر آپ کی ایپ کے لیے کیا کرتے ہیں
- جانیں گے کہ کب azd انفراسٹرکچر بنائے اور کب خود لکھیں
- اپنی موجودہ ایپ کو `azd up` کے ذریعے Azure پر تعینات کریں گے

## سیکھنے کے نتائج

اس سبق کو مکمل کرنے کے بعد، آپ قابل ہوں گے:
- azd کو اپنے موجودہ پروجیکٹ میں شروع کرنا
- ایک بنیادی `azure.yaml` فائل پڑھنا اور ترمیم کرنا
- `azd infra generate` کے ذریعے ابتدائی انفراسٹرکچر بنانا
- اپنی ایپ کے لیے مناسب Azure ہوسٹ منتخب کرنا
- اپنی ایپلیکیشن کو تعینات کرنا اور صفائی کرنا

---

## azd پروجیکٹ شروع کرنے کے تین طریقے

| شروع کرنے کی جگہ | کمانڈ | استعمال کا وقت |
|----------------|---------|-------------|
| **ٹیمپلیٹ سے** | `azd init --template <name>` | سیکھنے کے لیے، یا ایک نئے ایپ کو ثابت شدہ نمونے سے شروع کرنے کے لیے |
| **اپنے موجودہ کوڈ سے** | `azd init` (اپنے پروجیکٹ فولڈر میں) | آپ کے پاس پہلے سے ایپ ہے اور آپ اسے تعینات کرنا چاہتے ہیں |
| **Git ریپو سے** | `azd init --from-code` (کلون کیے ہوئے ریپو میں) | موجودہ ریپوزیٹری کے لیے azd اپنانا |

آپ نے پہلے آپشن کی مشق کی ہے۔ یہ سبق دوسرا گھیرے گا — سب سے عام حقیقی دنیا کا منظر نامہ۔

---

## قدم 1: اپنے پروجیکٹ میں `azd init` چلائیں

ایک ٹرمینل کھولیں **اپنے موجودہ پروجیکٹ فولڈر کے اندر** اور چلائیں:

```bash
cd my-existing-app
azd init
```

azd پوچھے گا کہ آپ کیسے شروع کرنا چاہتے ہیں۔ منتخب کریں:

```
? How do you want to initialize your app?
> Use code in the current directory
  Select a template
```

**"Use code in the current directory."** منتخب کریں۔ azd پھر آپ کے فولڈر کو اسکین کرتا ہے، آپ کی زبان اور فریم ورک کا پتہ لگاتا ہے، اور ایک ہوسٹ تجویز کرتا ہے۔

### azd کیا دریافت کرتا ہے

azd ایسے اشارے دیکھتا ہے جیسے `package.json`، `requirements.txt`، `pom.xml`، `*.csproj`، یا `Dockerfile`، اور ایک مناسب Azure ہوسٹ تجویز کرتا ہے:

| آپ کی ایپ | ممکنہ دریافت شدہ ہوسٹ |
|----------|----------------------|
| Node.js / Python / .NET ویب ایپ | Azure App Service یا Container Apps |
| کنٹینرائزڈ ایپ (`Dockerfile`) | Azure Container Apps |
| فنکشن ایپ | Azure Functions |
| سٹیٹک سائٹ (React/Vue بلڈ آؤٹ پٹ) | Azure Static Web Apps |

دریافت شدہ سروس(ز) کی تصدیق کریں، اور azd آپ کو درکار فائلیں تیار کرتا ہے۔

## قدم 2: سمجھیں کہ azd نے کیا تخلیق کیا ہے

init کے بعد، آپ کے پروجیکٹ میں دو نئی چیزیں ہوں گی:

```
my-existing-app/
├── azure.yaml          # ← NEW: tells azd what to deploy and where
├── infra/              # ← NEW: Infrastructure as Code (Bicep by default)
│   ├── main.bicep
│   └── main.parameters.json
├── src/ (your code)    # unchanged
└── ...                 # your existing files, untouched
```

### `azure.yaml` — پروجیکٹ کی تعریف

یہ azd پروجیکٹ کا دل ہے۔ ایک کم از کم والا اس طرح دکھتا ہے:

```yaml
# azure.yaml
name: my-existing-app
services:
  web:
    project: ./src           # path to your app code
    language: js             # js | python | dotnet | java
    host: appservice         # appservice | containerapp | function | staticwebapp
```

`services` بلاک کلیدی حصہ ہے: ہر اندراج آپ کے کوڈ کے ایک فولڈر کو Azure ہوسٹ سے میپ کرتا ہے۔ اگر آپ کی ایپ میں فرنٹ اینڈ اور API دونوں ہیں، تو آپ کے پاس دو خدمات ہوں گی۔

### `infra/` — آپ کے Azure وسائل کوڈ کی صورت میں

`infra/` فولڈر میں Bicep فائلیں ہوتی ہیں جو Azure کے وسائل کی تعریف کرتی ہیں جن کی آپ کی ایپ کو ضرورت ہے (App Service، ڈیٹا بیس، وغیرہ)۔ آپ کو یہ ہاتھ سے لکھنے کی ضرورت نہیں — azd ایک کام کرنے والا ابتدائی نقطہ پیدا کرتا ہے۔ آپ بعد میں ان میں ترمیم کر کے وسائل شامل کر سکتے ہیں یا سیکیورٹی سخت کر سکتے ہیں (یہ [باب 4](../chapter-04-infrastructure/README.md) میں شامل ہے)۔

> **ٹپ:** کیا آپ تعیناتی سے پہلے پیدا کردہ انفراسٹرکچر دیکھنا یا حسب ضرورت بنانا چاہتے ہیں؟ `azd infra generate` چلائیں (جو `azd infra synth` کے طور پر بھی دستیاب ہے) تاکہ IaC ڈسک پر لکھا جائے تاکہ آپ اس کا جائزہ لے سکیں اور ورژن کنٹرول کر سکیں۔

## قدم 3: مطلوبہ کنفیگریشن سیٹ کریں

اگر آپ کی ایپ کو سیٹنگز یا سیکریٹس کی ضرورت ہے (مثلاً کنکشن اسٹرنگ، API کی)، تو انہیں ہارڈ کوڈ نہ کریں۔ ماحول کے متغیرات استعمال کریں:

```bash
# ایک ماحول بنائیں
azd env new dev

# ایک غیر خفیہ قدر مقرر کریں
azd env set API_VERSION 1.0.0
```

اصلی سیکریٹس کے لیے، انہیں Key Vault میں محفوظ کریں اور انہیں اپنے انفراسٹرکچر سے رجوع کریں — دیکھیں [باب 3: کنفیگریشن اور توثیق](../chapter-03-configuration/authsecurity.md)۔

---

## قدم 4: تعینات کریں

اب وہی ورک فلو استعمال کریں جسے آپ پہلے سے جانتے ہیں:

```bash
# توثیق کریں (azd کیلئے ضروری)
azd auth login

# ان وسائل کا پیش نظارہ کریں جو بنائے جائیں گے
azd provision --preview

# بنیادی ڈھانچہ مہیا کریں اور اپنا کوڈ تعینات کریں
azd up
```

مکمل ہونے پر، azd آپ کی ایپ کا URL دکھائے گا۔ اسے اسی طرح تصدیق کریں جیسا کہ کسی بھی azd ایپ کو کرتے ہیں:

```bash
azd show           # اختتامی نکات دکھائیں
azd monitor --logs # ضرورت ہو تو لاگز چیک کریں
```

## عام پہلی بار مسائل

| علامت | ممکنہ وجہ | حل |
|---------|--------------|-----|
| azd نے میری ایپ کا پتہ نہیں لگایا | مینیفیسٹ غائب ہے (مثلاً `package.json`) | مینیفیسٹ شامل کریں، یا `azd init` کے دوران ہوسٹ دستی طور پر منتخب کریں |
| `azd up` کے دوران بلڈ ناکام ہو جاتا ہے | ایپ کو بلڈ مرحلے کی ضرورت ہے | `azure.yaml` میں سروس کے تحت `buildCommand`/`outputPath` شامل کریں |
| ایپ شروع ہوتی ہے لیکن غلطیاں دیتی ہے | کنفیگریشن یا سیکریٹ غائب ہے | `azd env set` کا استعمال کرتے ہوئے ویلیوز سیٹ کریں یا Key Vault جوڑیں |
| غلط ہوسٹ منتخب کیا گیا | خودکار پتہ لگانا غلط تھا | `azure.yaml` میں `host:` ایڈیٹ کریں اور `azd up` دوبارہ چلائیں |

مزید معلومات کے لیے دیکھیں [باب 7: مسئلہ حل](../chapter-07-troubleshooting/README.md)۔

## صفائی

```bash
azd down --force --purge
```

## خلاصہ

- `azd init` → **"Use code in the current directory"** آپ کی موجودہ ایپ میں azd شامل کرتا ہے۔
- `azure.yaml` آپ کے کوڈ فولڈرز کو Azure ہوسٹس کے ساتھ میپ کرتا ہے؛ `infra/` وسائل کو Bicep کے طور پر تعریف کرتا ہے۔
- `azd infra generate` آپ کو پیدا کردہ انفراسٹرکچر کا جائزہ لینے یا حسب ضرورت بنانے دیتا ہے۔
- ایک بار initialize ہو جانے کے بعد، آپ کی موجودہ ایپ بالکل اسی `azd up` / `azd down` ورک فلو کو استعمال کرتی ہے جو ٹیمپلیٹ پر مبنی ایپ استعمال کرتی ہے۔
