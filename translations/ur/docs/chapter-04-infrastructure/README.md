# باب 4: انفراسٹرکچر بطور کوڈ اور تعیناتی

**📚 کورس**: [ابتدائی افراد کے لیے AZD](../../README.md) | **⏱️ دورانیہ**: 1-1.5 گھنٹے | **⭐ پیچیدگی**: درمیانہ

---

## جائزہ

یہ باب انفراسٹرکچر بطور کوڈ (IaC) کے نمونے، بائسک ٹیمپلیٹس، وسائل کی فراہمی، اور ازور ڈویلپر CLI کے ذریعے تعیناتی کی حکمتِ عملیوں کا احاطہ کرتا ہے۔

> جون 2026 میں `azd 1.25.6` کے خلاف تصدیق شدہ۔

## تعلیمی مقاصد

اس باب کو مکمل کرکے، آپ:
- بائسک ٹیمپلیٹ کی ساخت اور نحو کو سمجھیں گے
- `azd provision` کے ذریعے ازور وسائل کی فراہمی کریں گے
- `azd deploy` کے ذریعے ایپلیکیشنز کی تعیناتی کریں گے
- نیلا-سبز اور رولنگ تعیناتی کی حکمتِ عملیوں کو نافذ کریں گے

## 🚀 فوری آغاز

```bash
# ٹیمپلیٹ سے شروعات کریں
azd init --template azure-functions-python-v2-http

# جو کچھ بنایا جائے گا اسے پیش نظارہ کریں
azd provision --preview

# صرف انفراسٹرکچر مہیا کریں
azd provision

# صرف کوڈ تعینات کریں
azd deploy

# یا دونوں ساتھ میں
azd up
```

## 📁 AZD پروجیکٹ کی ساخت

```
my-project/
├── azure.yaml            # ← Project definition
├── infra/                # ← Infrastructure as Code
│   ├── main.bicep        # Entry point
│   ├── main.parameters.json
│   └── modules/          # Reusable components
│       ├── web.bicep
│       └── database.bicep
└── src/                  # Application code
    └── api/
```

---

## 🔧 اہم کمانڈز

| کمانڈ | وضاحت |
|---------|-------------|
| `azd init` | پروجیکٹ کا آغاز کریں |
| `azd provision` | ازور وسائل بنائیں |
| `azd deploy` | ایپلیکیشن کوڈ کی تعیناتی کریں |
| `azd up` | فراہمی + تعیناتی |
| `azd down` | تمام وسائل حذف کریں |
