# AI ڈویلپرز کے لیے AZD ورکشاپ

Azure Developer CLI (AZD) کو سیکھنے کے لیے عملی ورکشاپ میں خوش آمدید، جس کا مرکز AI ایپلیکیشن کی تعیناتی ہے۔ یہ ورکشاپ آپ کو AZD ٹیمپلیٹس کی عملی سمجھ 3 مراحل میں فراہم کرتی ہے:

1. **دریافت** - اپنے لیے درست ٹیمپلیٹ تلاش کریں۔
1. **تعیناتی** - تعینات کریں اور یہ تصدیق کریں کہ یہ کام کرتا ہے۔
1. **حسب ضرورت بنانا** - تبدیلی کریں اور اسے اپنا بنائیں!

اس ورکشاپ کے دوران، آپ کو بنیادی ڈویلپر ٹولز اور ورک فلو سے بھی متعارف کرایا جائے گا تاکہ آپ کی مکمل ترقیاتی سفر کو آسان بنایا جا سکے۔

<br/>

## براؤزر پر مبنی گائیڈ

ورکشاپ کے اسباق مارک ڈاؤن میں ہیں۔ آپ انہیں براہ راست GitHub پر نیویگیٹ کر سکتے ہیں - یا نیچے دیے گئے اسکرین شاٹ کی طرح براؤزر پر پیش نظارہ شروع کر سکتے ہیں۔

![Workshop](../../../translated_images/ur/workshop.75906f133e6f8ba0.webp)

اس آپشن کو استعمال کرنے کے لیے - رپوزیٹری کو اپنے پروفائل پر فورک کریں، اور GitHub Codespaces شروع کریں۔ جیسے ہی VS Code ٹرمینل فعال ہوتا ہے، یہ کمانڈ ٹائپ کریں:

یہ براؤزر پر پیش نظارہ GitHub Codespaces، dev containers، یا Python اور MkDocs کے ساتھ مقامی کلون میں کام کرتا ہے۔

```bash title="" linenums="0"
mkdocs serve > /dev/null 2>&1 &
```
  
کچھ سیکنڈز میں، آپ کو ایک پاپ اپ ڈائیلاگ نظر آئے گا۔ `Open in browser` کا آپشن منتخب کریں۔ ویب پر مبنی گائیڈ اب ایک نئے براؤزر ٹیب میں کھل جائے گی۔ اس پیش نظارہ کے کچھ فوائد:

1. **بلٹ ان سرچ** - کی ورڈز یا اسباق تیزی سے تلاش کریں۔
1. **نقل کرنے کا آئیکن** - کوڈ بلاکس پر ہوور کریں تاکہ یہ آپشن دکھائی دے۔
1. **تھیم ٹوگل** - ڈارک اور لائٹ تھیم کے درمیان سوئچ کریں۔
1. **مدد حاصل کریں** - فوٹر میں Discord آئیکن پر کلک کر کے شامل ہوں!

<br/>

## آپ کیا سیکھیں گے

AZD ٹیمپلیٹ کو ایک لرننگ سینڈباکس کے طور پر لیں جہاں آپ مائیکروسافٹ فاؤنڈری پر مکمل ترقی کے لیے مختلف صلاحیتوں اور ٹولز کو دریافت کر سکیں۔ اس ورکشاپ کے اختتام تک، آپ کو اس سیاق و سباق میں مختلف ٹولز اور تصورات کا ایک فطری ادراک حاصل ہو جائے گا۔

| تصور  | ہدف |
|:---|:---|
| **Azure Developer CLI** | ٹول کے کمانڈز اور ورک فلو کو سمجھیں |
| **AZD ٹیمپلیٹس** | پروجیکٹ کی ساخت اور ترتیب کو سمجھیں |
| **Azure AI Agent** | مائیکروسافٹ فاؤنڈری پروجیکٹ کی پروویژن اور تعیناتی کریں |
| **Azure AI Search** | ایجنٹس کے ساتھ مواد کی انجینئرنگ فعال کریں |
| **Observability** | ٹریسنگ، نگرانی اور تشخیصات کو دریافت کریں |
| **Red Teaming** | دشمنانہ ٹیسٹنگ اور تخفیف کے طریقے دریافت کریں |

<br/>

## ورکشاپ کی ساخت

ورکشاپ آپ کو ٹیمپلیٹ کی دریافت سے لے کر تعیناتی، تحلیل، اور تخصیص تک سفر پر لے جانے کے لیے ترتیب دی گئی ہے - جس کی بنیاد [Getting Started with AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) کا سرکاری ابتدائی ٹیمپلیٹ ہے۔

### [ماڈیول 1: AI ٹیمپلیٹ منتخب کریں](docs/instructions/1-Select-AI-Template.md) (30 منٹ)

- AI ٹیمپلیٹس کیا ہوتے ہیں؟
- AI ٹیمپلیٹس کہاں ملتے ہیں؟
- میں AI ایجنٹس کی تعمیر کیسے شروع کر سکتا ہوں؟
- **لیب**: Codespaces یا dev container میں فوری آغاز

### [ماڈیول 2: AI ٹیمپلیٹ کی تصدیق کریں](docs/instructions/2-Validate-AI-Template.md) (30 منٹ)

- AI ٹیمپلیٹ کی ساخت کیا ہے؟
- AZD ڈویلپمنٹ ورک فلو کیا ہے؟
- AZD ڈویلپمنٹ میں مدد کیسے حاصل کریں؟
- **لیب**: AI ایجنٹس ٹیمپلیٹ تعینات اور تصدیق کریں

### [ماڈیول 3: AI ٹیمپلیٹ کو تحلیل کریں](docs/instructions/3-Deconstruct-AI-Template.md) (30 منٹ)

- اپنے ماحول کو `.azure/` میں دریافت کریں
- اپنے وسائل کی ترتیب کو `infra/` میں دریافت کریں
- اپنے AZD کی ترتیب `azure.yaml`s میں دریافت کریں
- **لیب**: ماحولیات کے متغیرات میں ترمیم کریں اور دوبارہ تعینات کریں

### [ماڈیول 4: AI ٹیمپلیٹ کو ترتیب دیں](docs/instructions/4-Configure-AI-Template.md) (30 منٹ)
- دریافت کریں: Retrieval Augmented Generation
- دریافت کریں: Agent Evaluation & Red Teaming
- دریافت کریں: ٹریسنگ اور نگرانی
- **لیب**: AI ایجنٹ + مشاہدہ دریافت کریں

### [ماڈیول 5: AI ٹیمپلیٹ کو حسب ضرورت بنائیں](docs/instructions/5-Customize-AI-Template.md) (30 منٹ)
- تعین کریں: PRD کے ساتھ منظرنامہ کی ضروریات
- ترتیب دیں: AZD کے لیے ماحولیات کے متغیرات
- نافذ کریں: اضافی کاموں کے لیے لائف سائیکل ہکس
- **لیب**: اپنے منظرنامہ کے لیے ٹیمپلیٹ کو حسب ضرورت بنائیں

### [ماڈیول 6: انفراسٹرکچر کو ختم کریں](docs/instructions/6-Teardown-Infrastructure.md) (30 منٹ)
- جائزہ: AZD ٹیمپلیٹس کیا ہیں؟
- جائزہ: Azure Developer CLI کیوں استعمال کریں؟
- اگلے اقدامات: کوئی مختلف ٹیمپلیٹ آزمائیں!
- **لیب**: انفراسٹرکچر کو ختم کریں اور صفائی کریں

## ورکشاپ چیلنج

مزید کرنے کا چیلنج قبول کرنا چاہتے ہیں؟ یہاں کچھ پروجیکٹ تجاویز ہیں - یا اپنے آئیڈیاز ہمارے ساتھ شیئر کریں!!

| پروجیکٹ | وضاحت |
|:---|:---|
|1. **ایک پیچیدہ AI ٹیمپلیٹ کو تحلیل کریں** | ہم نے جو ورک فلو اور ٹولز بیان کیے ہیں ان کا استعمال کریں اور دیکھیں کہ کیا آپ کسی دوسرے AI حل کے ٹیمپلیٹ کو تعینات، تصدیق، اور حسب ضرورت بنا سکتے ہیں۔ _آپ نے کیا سیکھا؟_|
|2. **اپنے منظرنامہ کے ساتھ حسب ضرورت بنائیں**  | ایک مختلف منظرنامے کے لیے PRD (پروڈکٹ ریکوائرمنٹس ڈاکیومنٹ) لکھنے کی کوشش کریں۔ پھر GitHub Copilot کو اپنے ٹیمپلیٹ ریپو میں Agent Model میں استعمال کریں - اور اسے اپنے لیے تخصیص ورک فلو بنانے کو کہیں۔ _آپ نے کیا سیکھا؟ آپ ان تجاویز کو کیسے بہتر بنا سکتے ہیں؟_|
