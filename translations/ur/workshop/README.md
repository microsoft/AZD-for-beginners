# AZD برائے AI ڈویلپرز ورکشاپ

Azure Developer CLI (AZD) کے ساتھ AI ایپلیکیشن ڈپلائمنٹ سیکھنے کے لیے عملی ورکشاپ میں خوش آمدید۔ یہ ورکشاپ آپ کو AZD ٹیمپلیٹس کو تین مراحل میں سمجھنے میں مدد دیتی ہے:

1. **دریافت** - وہ ٹیمپلیٹ تلاش کریں جو آپ کے لیے موزوں ہو۔
1. **ڈپلائمنٹ** - ڈپلائے کریں اور تصدیق کریں کہ یہ کام کرتا ہے۔
1. **حسب ضرورت** - اپنی ضروریات کے مطابق اسے تبدیل کریں اور بہتر بنائیں!

اس ورکشاپ کے دوران، آپ کو بنیادی ڈویلپر ٹولز اور ورک فلو سے بھی متعارف کرایا جائے گا تاکہ آپ کی مکمل ترقیاتی سفر کو آسان بنایا جا سکے۔

<br/>

## براؤزر پر مبنی گائیڈ

ورکشاپ کے اسباق مارک ڈاؤن میں ہیں۔ آپ انہیں براہ راست GitHub میں نیویگیٹ کر سکتے ہیں - یا نیچے دی گئی تصویر کے مطابق براؤزر پر مبنی پیش نظارہ لانچ کر سکتے ہیں۔

![ورکشاپ](../../../translated_images/workshop.75906f133e6f8ba07ab0302ce17f67ff90f357513f3d4c4bbafa5978b10f058b.ur.png)

اس آپشن کو استعمال کرنے کے لیے - ریپوزیٹری کو اپنے پروفائل میں فورک کریں، اور GitHub Codespaces لانچ کریں۔ جب VS Code ٹرمینل فعال ہو جائے، تو یہ کمانڈ ٹائپ کریں:

```bash title="" linenums="0"
mkdocs serve > /dev/null 2>&1 &
```

چند سیکنڈ میں، آپ کو ایک پاپ اپ ڈائیلاگ نظر آئے گا۔ `براؤزر میں کھولیں` کا آپشن منتخب کریں۔ ویب پر مبنی گائیڈ اب ایک نئے براؤزر ٹیب میں کھل جائے گا۔ اس پیش نظارہ کے کچھ فوائد:

1. **بلٹ ان سرچ** - مطلوبہ الفاظ یا اسباق کو جلدی تلاش کریں۔
1. **کاپی آئیکن** - کوڈ بلاکس پر ہوور کریں تاکہ یہ آپشن دیکھ سکیں۔
1. **تھیم ٹوگل** - ڈارک اور لائٹ تھیمز کے درمیان سوئچ کریں۔
1. **مدد حاصل کریں** - فوٹر میں Discord آئیکن پر کلک کریں اور شامل ہوں!

| وقت| ماڈیول  | مقصد |

## آپ کیا سیکھیں گے

AZD ٹیمپلیٹ کو ایک سیکھنے کے سینڈ باکس کے طور پر سوچیں تاکہ Azure AI Foundry پر مکمل ترقی کے لیے مختلف صلاحیتوں اور ٹولز کو دریافت کیا جا سکے۔ ورکشاپ کے اختتام تک، آپ کو اس سیاق و سباق میں مختلف ٹولز اور تصورات کے بارے میں ایک بدیہی احساس ہونا چاہیے۔

| تصور  | مقصد |
|:---|:---|
| **Azure Developer CLI** | ٹول کمانڈز اور ورک فلو کو سمجھیں|
| **AZD ٹیمپلیٹس**| پروجیکٹ کی ساخت اور کنفیگریشن کو سمجھیں|
| **Azure AI ایجنٹ**| Azure AI Foundry پروجیکٹ کو پروویژن اور ڈپلائے کریں  |
| **Azure AI سرچ**| ایجنٹس کے ساتھ سیاق و سباق انجینئرنگ کو فعال کریں |
| **مشاہدہ پذیری**| ٹریسنگ، مانیٹرنگ اور ایویلیوایشنز کو دریافت کریں |
| **ریڈ ٹیمنگ**| ایڈورسریل ٹیسٹنگ اور تخفیف کو دریافت کریں |

<br/>

## ورکشاپ کی ساخت

ورکشاپ کو اس طرح ترتیب دیا گیا ہے کہ آپ کو ٹیمپلیٹ کی دریافت، ڈپلائمنٹ، ڈی کنسٹرکشن، اور حسب ضرورت کے سفر پر لے جائے - سرکاری [Getting Started with AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) اسٹارٹر ٹیمپلیٹ کو بنیاد کے طور پر استعمال کرتے ہوئے۔

### [ماڈیول 1: AI ٹیمپلیٹ منتخب کریں](docs/instructions/1-Select-AI-Template.md) (30 منٹ)

- AI ٹیمپلیٹس کیا ہیں؟
- AI ٹیمپلیٹس کہاں مل سکتے ہیں؟
- میں AI ایجنٹس بنانا کیسے شروع کر سکتا ہوں؟
- **لیب**: GitHub Codespaces کے ساتھ کوئیک اسٹارٹ

### [ماڈیول 2: AI ٹیمپلیٹ کی تصدیق کریں](docs/instructions/2-Validate-AI-Template.md) (30 منٹ)

- AI ٹیمپلیٹ آرکیٹیکچر کیا ہے؟
- AZD ترقیاتی ورک فلو کیا ہے؟
- AZD ترقی کے ساتھ مدد کیسے حاصل کی جا سکتی ہے؟
- **لیب**: AI ایجنٹس ٹیمپلیٹ کو ڈپلائے اور تصدیق کریں

### [ماڈیول 3: AI ٹیمپلیٹ کو ڈی کنسٹرکٹ کریں](docs/instructions/3-Deconstruct-AI-Template.md) (30 منٹ)

- اپنے ماحول کو `.azure/` میں دریافت کریں 
- اپنے وسائل کی سیٹ اپ کو `infra/` میں دریافت کریں 
- اپنے AZD کنفیگریشن کو `azure.yaml`s میں دریافت کریں
- **لیب**: ماحول کے متغیرات کو تبدیل کریں اور دوبارہ ڈپلائے کریں

### [ماڈیول 4: AI ٹیمپلیٹ کو کنفیگر کریں](docs/instructions/4-Configure-AI-Template.md) (30 منٹ)
- دریافت کریں: ریٹریول اگمینٹڈ جنریشن
- دریافت کریں: ایجنٹ ایویلیوایشن اور ریڈ ٹیمنگ
- دریافت کریں: ٹریسنگ اور مانیٹرنگ
- **لیب**: AI ایجنٹ + مشاہدہ پذیری کو دریافت کریں 

### [ماڈیول 5: AI ٹیمپلیٹ کو حسب ضرورت بنائیں](docs/instructions/5-Customize-AI-Template.md) (30 منٹ)
- PRD کو منظر نامے کی ضروریات کے ساتھ بیان کریں
- AZD کے لیے ماحول کے متغیرات کو کنفیگر کریں
- اضافی کاموں کے لیے لائف سائیکل ہکس کو نافذ کریں
- **لیب**: اپنے منظر نامے کے لیے ٹیمپلیٹ کو حسب ضرورت بنائیں

### [ماڈیول 6: انفراسٹرکچر کو ختم کریں](docs/instructions/6-Teardown-Infrastructure.md) (30 منٹ)
- خلاصہ: AZD ٹیمپلیٹس کیا ہیں؟
- خلاصہ: Azure Developer CLI کیوں استعمال کریں؟
- اگلے مراحل: ایک مختلف ٹیمپلیٹ آزمائیں!
- **لیب**: انفراسٹرکچر کو ڈی پروویژن کریں اور صاف کریں

## ورکشاپ چیلنج

کیا آپ خود کو مزید کرنے کے لیے چیلنج کرنا چاہتے ہیں؟ یہاں کچھ پروجیکٹ تجاویز ہیں - یا اپنے خیالات ہمارے ساتھ شیئر کریں!!

| پروجیکٹ | تفصیل |
|:---|:---|
|1. **ایک پیچیدہ AI ٹیمپلیٹ کو ڈی کنسٹرکٹ کریں** | وہ ورک فلو اور ٹولز استعمال کریں جو ہم نے بیان کیے ہیں اور دیکھیں کہ کیا آپ ایک مختلف AI حل ٹیمپلیٹ کو ڈپلائے، تصدیق، اور حسب ضرورت بنا سکتے ہیں۔ _آپ نے کیا سیکھا؟_|
|2. **اپنے منظر نامے کے ساتھ حسب ضرورت بنائیں**  | ایک مختلف منظر نامے کے لیے PRD (پروڈکٹ ریکوائرمنٹس ڈاکیومنٹ) لکھنے کی کوشش کریں۔ پھر اپنے ٹیمپلیٹ ریپو میں GitHub Copilot کو ایجنٹ ماڈل میں استعمال کریں - اور اس سے حسب ضرورت ورک فلو پیدا کرنے کے لیے کہیں۔ _آپ نے کیا سیکھا؟ آپ ان تجاویز کو کیسے بہتر بنا سکتے ہیں؟_|
| | |
