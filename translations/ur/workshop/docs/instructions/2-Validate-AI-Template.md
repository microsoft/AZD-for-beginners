## 1. تعارف

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) یا `azd` ایک اوپن سورس کمانڈ لائن ٹول ہے جو Azure پر ایپلیکیشنز بنانے اور ڈپلائے کرنے والے ڈویلپر کے ورک فلو کو سہل بناتا ہے۔ 

[AZD ٹیمپلیٹس](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) معیاری ریپوزیٹریز ہیں جن میں سیمپل ایپلیکیشن کوڈ، _Infrastructure-as-code_ اثاثے، اور `azd` کنفیگریشن فائلیں شامل ہوتی ہیں جو مربوط سلوشن آرکیٹیکچر کے لیے ہوتی ہیں۔ انفراسٹرکچر کی پروویژننگ ایک `azd provision` کمانڈ جتنی آسان ہے – جبکہ `azd up` کمانڈ آپ کو انفراسٹرکچر پروویژن اور ایپلیکیشن کو ایک ہی بار میں ڈپلائے کرنے دیتی ہے!

اس کا نتیجہ یہ ہے کہ آپ کی ایپلیکیشن ڈیویلپمنٹ کا آغاز اتنا آسان ہو سکتا ہے جتنا کہ اپنی ایپلیکیشن اور انفراسٹرکچر کی ضروریات کے قریب ترین _AZD Starter template_ تلاش کرنا اور پھر اپنے سیناریو کی مطابق ریپوزیٹری کو حسب ضرورت بنانا۔

شروع کرنے سے پہلے، آیئے یقینی بنائیں کہ آپ کے پاس Azure Developer CLI انسٹال ہے۔

1. VS Code ٹرمینل کھولیں اور یہ کمانڈ ٹائپ کریں:

      ```bash title="" linenums="0"
      azd version
      ```

1. آپ کو کچھ اس طرح نظر آنا چاہئے!

      ```bash title="" linenums="0"
      azd version 1.25.6 (commit <current-build>)
      ```

**آپ اب azd کے ساتھ ٹیمپلیٹ منتخب اور ڈپلائے کرنے کے لئے تیار ہیں**

## 2. ٹیمپلیٹ کا انتخاب

Microsoft Foundry پلیٹ فارم کے ساتھ [تجویز کردہ AZD ٹیمپلیٹس کا ایک مجموعہ](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) آتا ہے جو مشہور سلوشن سیناریوز کا احاطہ کرتا ہے جیسے _ملٹی ایجنٹ ورک فلو آٹومیشن_ اور _ملٹی-موڈل مواد کی پروسیسنگ_۔ آپ Microsoft Foundry پورٹل پر جا کر بھی ان ٹیمپلیٹس کو دریافت کر سکتے ہیں۔

1. [https://ai.azure.com/templates](https://ai.azure.com/templates) ملاحظہ کریں
1. جب پرامپٹ آئے تو Microsoft Foundry پورٹل میں لاگ ان کریں – آپ کو کچھ اس طرح نظر آئے گا۔

![Pick](../../../../../translated_images/ur/01-pick-template.60d2d5fff5ebc374.webp)


**Basic** اختیارات آپ کے اسٹارٹر ٹیمپلیٹس ہیں:

1. [ ] [AI Chat کے ساتھ آغاز کریں](https://github.com/Azure-Samples/get-started-with-ai-chat) جو آپ کے ڈیٹا کے ساتھ ایک بنیادی چیٹ ایپلیکیشن Azure Container Apps پر ڈپلائے کرتا ہے۔ اس سے ایک بنیادی AI چیٹ بوٹ سیناریو کو دریافت کریں۔
1. [X] [AI Agents کے ساتھ آغاز کریں](https://github.com/Azure-Samples/get-started-with-ai-agents) جو معیاری AI ایجنٹ (Foundry Agents کے ساتھ) بھی ڈپلائے کرتا ہے۔ اس سے وہ ایجنٹک AI سلوشنز سمجھیں جن میں ٹولز اور ماڈلز شامل ہیں۔

دوسرے لنک کو نئے براؤزر ٹیب میں کھولیں (یا متعلقہ کارڈ پر `Open in GitHub` پر کلک کریں)۔ آپ کو اس AZD ٹیمپلیٹ کے ریپوزیٹری کا صفحہ نظر آئے گا۔ README کو ایک منٹ کے لیے دیکھیں۔ ایپلیکیشن آرکیٹیکچر کچھ اس طرح دکھائی دیتا ہے:

![Arch](../../../../../translated_images/ur/architecture.8cec470ec15c65c7.webp)

## 3. ٹیمپلیٹ کی فعالیت

آئیے اس ٹیمپلیٹ کو ڈپلائے کرنے کی کوشش کریں اور یقینی بنائیں کہ یہ جائز ہے۔ ہم [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) سیکشن کی ہدایات پر عمل کریں گے۔

1. ٹیمپلیٹ ریپوزیٹری کے لیے کام کرنے والا ماحول منتخب کریں:

      - **GitHub Codespaces**: اس [لنک](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) پر کلک کریں اور `Create codespace` کی تصدیق کریں
      - **مقامی کلون یا ڈیولپمنٹ کنٹینر**: `Azure-Samples/get-started-with-ai-agents` کو کلون کریں اور اسے VS Code میں کھولیں

1. جب تک VS Code ٹرمینل تیار نہ ہو جائے، انتظار کریں، پھر مندرجہ ذیل کمانڈ ٹائپ کریں:

   ```bash title="" linenums="0"
   azd up
   ```

اس سے جو ورک فلو خطوات چلیں گے، وہ مکمل کریں:

1. آپ کو Azure میں لاگ ان کرنے کے لیے کہا جائے گا – ہدایات کے مطابق تصدیق کریں
1. اپنے لیے منفرد ماحول کا نام درج کریں – مثال کے طور پر میں نے `nitya-mshack-azd` استعمال کیا
1. اس سے ایک `.azure/` فولڈر بن جائے گا – آپ کو اس ماحول کے نام والا سب فولڈر نظر آئے گا
1. آپ کو سبسکرپشن نام منتخب کرنے کے لیے کہا جائے گا – ڈیفالٹ منتخب کریں
1. آپ کو مقام (location) منتخب کرنے کے لیے کہا جائے گا – `East US 2` استعمال کریں

اب، آپ پروویژننگ کے مکمل ہونے کا انتظار کریں۔ **یہ 10-15 منٹ لیتا ہے**

1. جب مکمل ہو جائے، آپ کے کنسول میں کامیابی کا پیغام نظر آئے گا جیسے یہ:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. آپ کے Azure پورٹل میں اب اس ماحول کے نام کا ایک پروویژندہ ریسورس گروپ ہوگا:

      ![Infra](../../../../../translated_images/ur/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **آپ اب ڈپلائے شدہ انفراسٹرکچر اور ایپلیکیشن کو تصدیق کرنے کے لئے تیار ہیں۔**

## 4. ٹیمپلیٹ کی تصدیق

1. Azure پورٹل [Resource Groups](https://portal.azure.com/#browse/resourcegroups) صفحہ ملاحظہ کریں – لاگ ان کریں جب پرامپٹ آئے
1. اپنے ماحول کے نام والے RG پر کلک کریں – آپ کو اوپر والا صفحہ نظر آئے گا

      - Azure Container Apps وسعت نمائندہ پر کلک کریں
      - _Essentials_ سیکشن (اوپر دائیں) میں Application Url پر کلک کریں

1. آپ کو میزبانی شدہ ایپلیکیشن فرنٹ اینڈ UI اس طرح نظر آنا چاہئے:

   ![App](../../../../../translated_images/ur/03-test-application.471910da12c3038e.webp)

1. کچھ [نمونہ سوالات](https://github.com/Azure-Samples/get-started-with-ai-agents/blob/main/docs/sample_questions.md) پوچھنے کی کوشش کریں

      1. پوچھیں: ```فرانس کا دارالحکومت کیا ہے؟``` 
      1. پوچھیں: ```دو افراد کے لیے $200 کے تحت بہترین خیمہ کون سا ہے، اور اس میں کون سے خصوصیات شامل ہیں؟```

1. آپ کو نیچے دکھائے گئے جیسے جوابات ملنے چاہئیں۔ _لیکن یہ کیسے کام کرتا ہے؟_

      ![App](../../../../../translated_images/ur/03-test-question.521c1e863cbaddb6.webp)

## 5. ایجنٹ کی تصدیق

Azure Container App ایک اینڈپوائنٹ ڈپلائے کرتا ہے جو اس ٹیمپلیٹ کے Microsoft Foundry پروجیکٹ میں پروویژندہ AI ایجنٹ سے جڑتا ہے۔ آئیے دیکھتے ہیں کہ اس کا کیا مطلب ہے۔

1. Azure پورٹل میں اپنے ریسورس گروپ کے _Overview_ صفحے پر واپس جائیں

1. اس فہرست میں `Microsoft Foundry` ریسورس پر کلک کریں

1. آپ کو یہ نظر آئے گا۔ `Go to Microsoft Foundry Portal` بٹن پر کلک کریں۔ 
   ![Foundry](../../../../../translated_images/ur/04-view-foundry-project.fb94ca41803f28f3.webp)

1. آپ کو آپ کی AI ایپلیکیشن کے لیے Foundry پروجیکٹ صفحہ نظر آئے گا
   ![Project](../../../../../translated_images/ur/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` پر کلک کریں - آپ کو اپنے پروجیکٹ میں پروویژندہ ڈیفالٹ ایجنٹ نظر آئے گا
   ![Agents](../../../../../translated_images/ur/06-visit-agents.bccb263f77b00a09.webp)

1. اسے منتخب کریں - آپ کو ایجنٹ کی تفصیلات نظر آئیں گی۔ درج ذیل نوٹ کریں:

      - ایجنٹ عام طور پر فائل سرچ کا استعمال کرتا ہے (ہمیشہ)
      - ایجنٹ کی `Knowledge` ظاہر کرتی ہے کہ اس میں 32 فائلیں اپ لوڈ کی گئی ہیں (فائل سرچ کے لئے)
      ![Agents](../../../../../translated_images/ur/07-view-agent-details.0e049f37f61eae62.webp)

1. بائیں مینو میں `Data+indexes` آپشن تلاش کریں اور تفصیلات دیکھنے کے لیے کلک کریں۔ 

      - آپ کو 32 ڈیٹا فائلیں اپ لوڈ کی گئی نظر آئیں گی جنہیں علم کے لیے استعمال کیا جاتا ہے۔
      - یہ `src/files` کے تحت 12 کسٹمر فائلوں اور 20 پروڈکٹ فائلوں سے میل کھائیں گی۔
      ![Data](../../../../../translated_images/ur/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**آپ نے ایجنٹ کے آپریشن کی تصدیق کر لی!** 

1. ایجنٹ کے جوابات ان فائلوں میں علم کی بنیاد پر ہوتے ہیں۔
1. آپ اب ان ڈیٹا سے متعلق سوالات پوچھ سکتے ہیں، اور مربوط جوابات حاصل کر سکتے ہیں۔
1. مثال: `customer_info_10.json` "Amanda Perez" کی کی گئی 3 خریداریوں کی وضاحت کرتا ہے۔

براؤزر ٹیب جہاں Container App اینڈپوائنٹ ہے وہاں واپس جائیں اور پوچھیں: `Amanda Perez کے پاس کون سے مصنوعات ہیں؟`. آپ کو کچھ ایسا کچھ نظر آئے گا:

![Data](../../../../../translated_images/ur/09-ask-in-aca.4102297fc465a4d5.webp)

## 6. ایجنٹ پلے گراؤنڈ

Microsoft Foundry کی صلاحیتوں کو بہتر طور پر سمجھنے کے لیے، ایجنٹ کو Agents Playground میں لے کر چلاتے ہیں۔

1. Microsoft Foundry میں `Agents` صفحہ پر واپس جائیں - ڈیفالٹ ایجنٹ منتخب کریں
1. `Try in Playground` آپشن پر کلک کریں - آپ کو کچھ ایسا Playground UI نظر آنا چاہیے
1. وہی سوال پوچھیں: `Amanda Perez کے پاس کون سے مصنوعات ہیں؟`

    ![Data](../../../../../translated_images/ur/09-ask-in-playground.a1b93794f78fa676.webp)

آپ کو وہی (یا ملتا جلتا) جواب ملے گا - لیکن آپ کو اضافی معلومات بھی ملتی ہے جو آپ کو اپنے ایجنٹک ایپ کی کوالٹی، لاگت اور کارکردگی سمجھنے میں مدد دیتی ہے۔ مثال کے طور پر:

1. نوٹ کریں کہ جواب میں وہ ڈیٹا فائلز شامل ہیں جو جواب کو "گراؤنڈ" کرتی ہیں
1. ان فائل لیبلز پر ہوور کریں - کیا ڈیٹا آپ کے سوال اور دکھائی گئی جواب سے میل کھاتا ہے؟

آپ کو جواب کے نیچے ایک _stats_ صف بھی نظر آئے گی۔

1. کسی بھی میٹرک پر ہوور کریں - مثلاً Safety۔ آپ کو کچھ اس طرح کا نظارہ ملتا ہے
1. کیا اس جائزہ شدہ ریٹنگ کا آپ کی سیفٹی کے بارے میں سوچ سے میل کھاتا ہے؟

      ![Data](../../../../../translated_images/ur/10-view-run-info-meter.6cdb89a0eea5531f.webp)

## 7. بلٹ ان آبزرویبیلٹی

Observability اس بات کے بارے میں ہے کہ آپ کی ایپلیکیشن کو آلات لگانے کا طریقہ جو ڈیٹا پیدا کرتا ہے جسے آپ آپریشنز کو سمجھنے، ڈیبگ کرنے، اور بہتر بنانے کے لیے استعمال کر سکتے ہیں۔ اس کا اندازہ لگانے کے لیے:

1. `View Run Info` بٹن پر کلک کریں - آپ کو یہ منظر نظر آئے گا۔ یہ [Agent tracing](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) کی ایک مثال ہے۔ _آپ ٹاپ لیول مینو میں Thread Logs پر کلک کر کے بھی یہ منظر حاصل کر سکتے ہیں_۔

   - رن کے مراحل اور ایجنٹ کے استعمال کردہ ٹولز کا اندازہ لگائیں
   - کل ٹوکن کی تعداد سمجھیں (خروجی ٹوکنز کے استعمال کے مقابلے میں) برائے جواب
   - تاخیر اور عملدرآمد میں وقت کہاں صرف ہو رہا ہے سمجھیں

      ![Agent](../../../../../translated_images/ur/10-view-run-info.b20ebd75fef6a1cc.webp)

1. `Metadata` ٹیب پر کلک کریں تاکہ رن کے مزید صفات دیکھ سکیں جو بعد میں مسائل کی خرابی کے لیے مفید ہو سکتی ہیں۔

      ![Agent](../../../../../translated_images/ur/11-view-run-info-metadata.7966986122c7c2df.webp)


1. `Evaluations` ٹیب پر کلک کریں تاکہ ایجنٹ کے جواب پر خودکار تشخیصات دیکھ سکیں۔ ان میں سیفٹی کے جائزے شامل ہیں (مثلاً Self-harm) اور ایجنٹ-مخصوص جائزے (مثلاً Intent resolution، Task adherence)۔

      ![Agent](../../../../../translated_images/ur/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. آخری لیکن اہم بات، سائڈبار مینو میں `Monitoring` ٹیب پر کلک کریں۔

      - ظاہر کردہ صفحے میں `Resource usage` ٹیب منتخب کریں - اور میٹرکس دیکھیں۔
      - ایپلیکیشن کے استعمال کو لاگت (ٹوکنز) اور لوڈ (درخواستوں) کے حوالے سے ٹریک کریں۔
      - ایپلیکیشن کی تاخیر کو پہلے بائٹ (ان پٹ پروسیسنگ) اور آخری بائٹ (آؤٹ پٹ) تک ٹریک کریں۔

      ![Agent](../../../../../translated_images/ur/13-monitoring-resources.5148015f7311807f.webp)

## 8. ماحول کے متغیرات

اب تک، ہم نے براؤزر میں ڈپلائمنٹ کی ٹریکنگ کی ہے - اور اس بات کی تصدیق کی ہے کہ ہمارا انفراسٹرکچر پروویژندہ ہے اور ایپلیکیشن فعال ہے۔ لیکن ایپلیکیشن کے _کوڈ-فرسٹ_ طور پر کام کرنے کے لیے، ہمیں اپنے مقامی ڈیویلپمنٹ ماحول کو متعلقہ متغیرات سے ترتیب دینا ہوگا جو ان ریسورسز کے ساتھ کام کرنے کے لیے درکار ہیں۔ `azd` اس کو آسان بناتا ہے۔

1. Azure Developer CLI [ماحول کے متغیرات](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) کو استعمال کرتا ہے تاکہ ایپلیکیشن ڈپلائمنٹس کی کنفیگریشن سیٹنگز کو ذخیرہ اور منظم کیا جا سکے۔

1. ماحول کے متغیرات `.azure/<env-name>/.env` میں محفوظ ہوتے ہیں - یہ انھیں اس `env-name` ماحول کے دائرے میں رکھتا ہے جو ڈپلائمنٹ کے دوران استعمال کیا گیا اور مختلف ڈپلائمنٹ ٹارگٹس کے درمیان ماحول کو الگ کرنے میں مدد دیتا ہے۔

1. جب بھی `azd` کمانڈ مخصوص کمانڈ چلائے (جیسے `azd up`)، ماحول کے متغیرات خود بخود لوڈ ہو جاتے ہیں۔ نوٹ کریں کہ `azd` خود بخود _OS-لیول_ ماحول کے متغیرات (جیسے شیل میں سیٹ شدہ) کو نہیں پڑھتا - اس کے بجائے `azd set env` اور `azd get env` کمانڈز اس معلومات کو اسکرپٹس میں منتقل کرنے کے لیے استعمال کریں۔

آئیے چند کمانڈز آزما کر دیکھتے ہیں:

1. اس ماحول میں `azd` کے لیے سیٹ کیے گئے تمام ماحول کے متغیرات حاصل کریں:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      آپ کو کچھ ایسا نظر آئے گا:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. ایک مخصوص قدر حاصل کریں – جیسے میں جاننا چاہتا ہوں کہ کیا `AZURE_AI_AGENT_MODEL_NAME` کی ویلیو سیٹ کی گئی ہے

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      آپ کو ایسا کچھ نظر آئے گا - یہ ڈیفالٹ کے طور پر سیٹ نہیں تھی!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd` کے لیے نیا ماحول متغیر سیٹ کریں۔ یہاں، ہم ایجنٹ ماڈل کا نام اپ ڈیٹ کرتے ہیں۔ _نوٹ کریں: کی گئی کوئی بھی تبدیلی فوری طور پر `.azure/<env-name>/.env` فائل میں نظر آئے گی۔

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      اب، آپ کو یہ ویلیو سیٹ شدہ نظر آئے گی:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. نوٹ کریں کہ کچھ ریسورسز مستقل ہوتے ہیں (جیسے ماڈل ڈپلائمنٹس) اور ان کو دوبارہ ڈپلائے کرنے کے لیے صرف `azd up` کافی نہیں ہوگا۔ آیئے اصل ڈپلائمنٹ کو ختم کرکے تبدیلی شدہ env vars کے ساتھ دوبارہ ڈپلائے کرنے کی کوشش کریں۔

1. **ریفریش** اگر آپ نے پہلے کہیں azd ٹیمپلیٹ کے ساتھ انفراسٹرکچر ڈپلائے کیا تھا - آپ اپنے مقامی ماحول کے متغیرات کی حالت کو Azure ڈپلائمنٹ کی موجودہ حالت کی بنیاد پر _ریفریش_ کر سکتے ہیں اس کمانڈ کے ذریعے:

      ```bash title="" linenums="0"
      azd env refresh
      ```

یہ ایک طاقتور طریقہ ہے کہ دو یا زیادہ مقامی ترقیاتی ماحولیات (مثلاً، متعدد ڈویلپرز کی ٹیم) کے درمیان ماحول کے متغیرات کو _sync_ کیا جائے - جس سے تعینات شدہ ڈھانچہ ماحول کے متغیرات کی حالت کے لیے حقیقی بنیاد کے طور پر کام کرتا ہے۔ ٹیم کے ارکان بس متغیرات کو _refresh_ کرتے ہیں تاکہ وہ دوبارہ ہم آہنگ ہو جائیں۔
