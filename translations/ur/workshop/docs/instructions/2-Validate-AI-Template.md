## 1. تعارف

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) یا `azd` ایک اوپن سورس کمانڈ لائن ٹول ہے جو Azure پر ایپلیکیشنز کی تعمیر اور تعیناتی کے وقت ڈویلپر کے ورک فلو کو آسان بناتا ہے۔

[AZD ٹیمپلیٹس](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) معیاری ذخائر ہیں جن میں نمونہ ایپلیکیشن کوڈ، _انفراسٹرکچر از کوڈ_ اثاثے، اور `azd` کنفیگریشن فائلیں شامل ہوتی ہیں تاکہ ایک مربوط حل کے فن تعمیر کو یقینی بنایا جائے۔ انفراسٹرکچر کی تیاری `azd provision` کمانڈ جتنا آسان ہو جاتا ہے - جبکہ `azd up` استعمال کرتے ہوئے آپ ایک ہی وقت میں انفراسٹرکچر تیار اور اپنی ایپلیکیشن تعینات کر سکتے ہیں!

نتیجتاً، آپ کی ایپلیکیشن کی ترقی کا آغاز صحیح _AZD Starter template_ تلاش کرنے جتنا آسان ہے جو آپ کی ایپلیکیشن اور انفراسٹرکچر کی ضروریات کے قریب ہو - پھر ذخیرے کو اپنی صورت حال کے مطابق حسبِ ضرورت ڈھالیں۔

شروع کرنے سے پہلے، آئیے یقین کر لیتے ہیں کہ آپ کے پاس Azure Developer CLI نصب ہے۔

1. VS Code کا ٹرمینل کھولیں اور یہ کمانڈ لکھیں:

      ```bash title="" linenums="0"
      azd version
      ```

1. آپ کو کچھ ایسا ہی نظر آنا چاہیے!

      ```bash title="" linenums="0"
      azd version 1.23.12 (commit <current-build>)
      ```

**آپ اب azd کے ساتھ ٹیمپلیٹ منتخب اور تعینات کرنے کے لیے تیار ہیں**

## 2. ٹیمپلیٹ کا انتخاب

Microsoft Foundry پلیٹ فارم کے ساتھ [مقر شدہ AZD ٹیمپلیٹس کا ایک سیٹ](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) آتا ہے جو مقبول حل کے منظرنامے، جیسے _کئی ایجنٹ ورک فلو آٹومیشن_ اور _کئی قسم کے مواد کی پروسیسنگ_ کو کور کرتا ہے۔ آپ Microsoft Foundry پورٹل پر جا کر بھی یہ ٹیمپلیٹس دریافت کر سکتے ہیں۔

1. وزٹ کریں: [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. جب لاگ ان کرنے کو کہا جائے تو Microsoft Foundry پورٹل میں لاگ ان کریں - آپ کو کچھ ایسا نظر آئے گا۔

![Pick](../../../../../translated_images/ur/01-pick-template.60d2d5fff5ebc374.webp)


**Basic** آپشنز آپ کے اسٹارٹر ٹیمپلیٹس ہیں:

1. [ ] [AI چیٹ کے ساتھ شروعات کریں](https://github.com/Azure-Samples/get-started-with-ai-chat) جو ایک بنیادی چیٹ ایپلیکیشن کو _آپ کے ڈیٹا کے ساتھ_ Azure Container Apps پر تعینات کرتا ہے۔ اس کا استعمال بنیادی AI چیٹ بوٹ منظرنامے کو دریافت کرنے کے لیے کریں۔
1. [X] [AI Agents کے ساتھ شروعات کریں](https://github.com/Azure-Samples/get-started-with-ai-agents) جو اجازت دیتا ہے کہ معیاری AI Agent (Foundry Agents کے ساتھ) تعینات کریں۔ اس کا استعمال ان آلہ اور ماڈلز پر مبنی agentic AI حلوں سے واقف ہونے کے لیے کریں۔

دوسری لنک کو نئے براؤزر ٹیب میں کھولیں (یا متعلقہ کارڈ کے لیے `Open in GitHub` پر کلک کریں)۔ آپ کو اس AZD ٹیمپلیٹ کے ریپوزٹری کا صفحہ نظر آئے گا۔ README کو ایک منٹ دیکھیں۔ ایپلیکیشن کے فن تعمیر کچھ اس طرح ہے:

![Arch](../../../../../translated_images/ur/architecture.8cec470ec15c65c7.webp)

## 3. ٹیمپلیٹ کی سرگرمی

آئیے اس ٹیمپلیٹ کو تعینات کرنے کی کوشش کرتے ہیں اور یقینی بناتے ہیں کہ یہ درست ہے۔ ہم [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) سیکشن کی ہدایات پر عمل کریں گے۔

1. ٹیمپلیٹ ریپوزٹری کے لیے ورکنگ ماحول منتخب کریں:

      - **GitHub Codespaces**: [اس لنک](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) پر کلک کریں اور `Create codespace` کی تصدیق کریں
      - **مقامی کلون یا ڈویلپمنٹ کنٹینر**: `Azure-Samples/get-started-with-ai-agents` کو کلون کریں اور اسے VS Code میں کھولیں

1. جب VS Code کا ٹرمینل تیار ہو جائے، تو درج ذیل کمانڈ لکھیں:

   ```bash title="" linenums="0"
   azd up
   ```

اس سے متعلقہ ورک فلو کے مراحل مکمل کریں:

1. آپ کو Azure میں لاگ ان کرنے کے لیے کہا جائے گا - ہدایت کے مطابق توثیق کریں
1. اپنے لیے ایک منفرد ماحول کا نام درج کریں - مثلاً، میں نے `nitya-mshack-azd` استعمال کیا
1. یہ `.azure/` فولڈر بنائے گا - آپ کو اس کے اندر بطور ذیلی فولڈر ماحول کا نام نظر آئے گا
1. آپ کو ایک سبسکرپشن نام منتخب کرنے کے لیے کہا جائے گا - ڈیفالٹ منتخب کریں
1. آپ کو ایک مقام منتخب کرنے کے لیے کہا جائے گا - `East US 2` استعمال کریں

اب، provisioning کے مکمل ہونے کا انتظار کریں۔ **یہ 10-15 منٹ لیتا ہے**

1. جب مکمل ہو جائے، آپ کے کنسول میں کامیابی کا پیغام کچھ اس طرح نظر آئے گا:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. آپ کے Azure پورٹل میں اب اس ماحول کے نام کے ساتھ ایک provision شدہ resource group ہوگا:

      ![Infra](../../../../../translated_images/ur/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **آپ اب تعینات شدہ انفراسٹرکچر اور ایپلیکیشن کی توثیق کے لیے تیار ہیں**۔

## 4. ٹیمپلیٹ کی توثیق

1. Azure پورٹل [Resource Groups](https://portal.azure.com/#browse/resourcegroups) صفحہ پر جائیں - لاگ ان کریں جب کہا جائے
1. اپنے ماحول کے نام کے لیے RG پر کلک کریں - اوپر دیا گیا صفحہ نظر آئے گا

      - Azure Container Apps resource پر کلک کریں
      - _Essentials_ سیکشن میں Application Url پر کلک کریں (اوپر دائیں)

1. آپ کو ایک ہوسٹ کی ہوئی ایپلیکیشن فرنٹ اینڈ UI اس طرح نظر آئے گا:

   ![App](../../../../../translated_images/ur/03-test-application.471910da12c3038e.webp)

1. چند [نمونہ سوالات](https://github.com/Azure-Samples/get-started-with-ai-agents/blob/main/docs/sample_questions.md) پوچھ کر آزمائیں

      1. پوچھیں: ```فرانس کا دارالحکومت کیا ہے؟```
      1. پوچھیں: ```دو افراد کے لیے $200 کے نیچے بہترین خیمہ کون سا ہے، اور اس میں کیا خصوصیات شامل ہیں؟```

1. آپ کو نیچے دکھائی گئی جیسی جوابات ملیں گے۔ _لیکن یہ کیسے کام کرتا ہے؟_

      ![App](../../../../../translated_images/ur/03-test-question.521c1e863cbaddb6.webp)

## 5. ایجنٹ کی توثیق

Azure Container App ایک اینڈپوائنٹ تعینات کرتا ہے جو Microsoft Foundry پروجیکٹ میں فراہم کردہ AI Agent سے منسلک ہوتا ہے۔ آئیے دیکھتے ہیں کہ اس کا مطلب کیا ہے۔

1. اپنے resource group کے Azure پورٹل کے _Overview_ صفحہ پر واپس جائیں

1. اس فہرست میں `Microsoft Foundry` resource پر کلک کریں

1. آپ کو یہ نظر آنا چاہیے۔ `Go to Microsoft Foundry Portal` بٹن پر کلک کریں۔
   ![Foundry](../../../../../translated_images/ur/04-view-foundry-project.fb94ca41803f28f3.webp)

1. آپ کو اپنے AI اپلیکیشن کے لیے Foundry پروجیکٹ صفحہ نظر آئے گا
   ![Project](../../../../../translated_images/ur/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` پر کلک کریں - آپ کو آپ کے پروجیکٹ میں ڈیفالٹ ایجنٹ نظر آئے گا
   ![Agents](../../../../../translated_images/ur/06-visit-agents.bccb263f77b00a09.webp)

1. اسے منتخب کریں - اور آپ کو ایجنٹ کی تفصیلات نظر آئیں گی۔ درج ذیل باتوں پر غور کریں:

      - ایجنٹ ڈیفالٹ طور پر فائل سرچ استعمال کرتا ہے (ہمیشہ)
      - ایجنٹ کا `Knowledge` ظاہر کرتا ہے کہ اس کے پاس 32 فائلیں اپ لوڈ کی گئی ہیں (فائل سرچ کے لیے)
      ![Agents](../../../../../translated_images/ur/07-view-agent-details.0e049f37f61eae62.webp)

1. بائیں مینو میں `Data+indexes` آپشن تلاش کریں اور تفصیلات کے لیے کلک کریں۔

      - آپ کو علم کے لیے اپ لوڈ کی گئی 32 ڈیٹا فائلیں نظر آئیں گی۔
      - یہ `src/files` کے تحت 12 کسٹمر فائلیں اور 20 پراڈکٹ فائلوں کے مطابق ہوں گی۔
      ![Data](../../../../../translated_images/ur/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**آپ نے ایجنٹ کی فعالیت کی تصدیق کر لی!**

1. ایجنٹ کے جوابات ان فائلوں میں موجود معلومات پر مبنی ہوتے ہیں۔
1. آپ اب ان ڈیٹا سے متعلق سوالات پوچھ سکتے ہیں، اور بنیاد پرست جوابات حاصل کر سکتے ہیں۔
1. مثال کے طور پر: `customer_info_10.json` "Amanda Perez" کی کی گئی 3 خریداریوں کی تفصیل دیتا ہے۔

براؤزر ٹیب جہاں Container App endpoint موجود ہے اس پر واپس جائیں اور پوچھیں: `Amanda Perez کے پاس کون سے محصولات ہیں؟` آپ کو کچھ اس طرح نظر آنا چاہیے:

![Data](../../../../../translated_images/ur/09-ask-in-aca.4102297fc465a4d5.webp)

## 6. ایجنٹ پلے گراؤنڈ

آئیے Microsoft Foundry کی صلاحیتوں کے بارے میں مزید فہم پیدا کریں، ایجنٹ کو Agents Playground میں آزما کر۔

1. Microsoft Foundry میں `Agents` صفحہ پر واپس جائیں - ڈیفالٹ ایجنٹ منتخب کریں
1. `Try in Playground` آپشن پر کلک کریں - آپ کو اس طرح کا Playground UI نظر آئے گا
1. وہی سوال پوچھیں: `Amanda Perez کے پاس کون سے محصولات ہیں؟`

    ![Data](../../../../../translated_images/ur/09-ask-in-playground.a1b93794f78fa676.webp)

آپ کو وہی (یا ملتا جلتا) جواب ملے گا - لیکن آپ کو اضافی معلومات بھی ملتی ہے جو آپ کو اپنے agentic ایپ کی معیار، لاگت، اور کارکردگی سمجھنے میں مدد دیتی ہے۔ مثال کے طور پر:

1. نوٹ کریں کہ جواب میں ان ڈیٹا فائلوں کے حوالے شامل ہیں جو جواب کو "زمین پر" لاتی ہیں
1. ان فائلوں کے لیبل پر کرسر لے جائیں - کیا ڈیٹا آپ کی تلاش اور دکھائے گئے جواب کے مطابق ہے؟

آپ کو جواب کے نیچے ایک _اسٹٹس_ لائن بھی نظر آئے گی۔

1. کسی بھی میٹرک پر کرسر لے جائیں - مثلاً Safety۔ آپ کو کچھ ایسا ہی نظر آتا ہے
1. کیا جائزہ شدہ درجہ بندی آپ کی جواب کی حفاظت کی سطح کے لیے آپ کے تخیل سے میل کھاتی ہے؟

      ![Data](../../../../../translated_images/ur/10-view-run-info-meter.6cdb89a0eea5531f.webp)

## 7. بلٹ ان آبزرویبلٹی

آبزرویبلٹی کا مطلب ہے آپ کی ایپلیکیشن کو اس طرح ترتیب دینا کہ وہ ایسے ڈیٹا تیار کرے جو آپریشنز کو سمجھنے، ڈیبگ کرنے، اور بہتر بنانے کے لیے استعمال ہو سکے۔ اس کا احساس حاصل کرنے کے لیے:

1. `View Run Info` بٹن پر کلک کریں - آپ کو یہ منظر نظر آئے گا۔ یہ [Agent tracing](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) کی ایک مثال ہے۔ _آپ اس منظر کو ٹاپ لیول مینو میں Thread Logs پر کلک کر کے بھی حاصل کر سکتے ہیں_۔

   - ایجنٹ کے چلائے گئے مراحل اور استعمال شدہ ٹولز کا جائزہ لیں
   - جوابی ردعمل کے لیے کل ٹوکن کی تعداد (آؤٹ پٹ ٹوکنز کے مقابلے میں) کا ادراک کریں
   - تاخیر کو سمجھیں اور دیکھیں عمل درآمد میں وقت کہاں صرف ہو رہا ہے

      ![Agent](../../../../../translated_images/ur/10-view-run-info.b20ebd75fef6a1cc.webp)

1. `Metadata` ٹیب پر کلک کریں تاکہ چلانے کے لیے اضافی خصوصیات دیکھ سکیں، جو بعد میں مسائل کی تشخیص کے لیے مددگار ہو سکتی ہیں۔  

      ![Agent](../../../../../translated_images/ur/11-view-run-info-metadata.7966986122c7c2df.webp)


1. `Evaluations` ٹیب پر کلک کریں تاکہ ایجنٹ کے جواب پر کی گئی خودکار تشخیصات دیکھ سکیں۔ ان میں حفاظت کی تشخیصات (مثلاً Self-harm) اور ایجنٹ خاص تشخیصات (مثلاً Intent resolution، Task adherence) شامل ہیں۔

      ![Agent](../../../../../translated_images/ur/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. آخر میں، سائڈبار مینو میں `Monitoring` ٹیب پر کلک کریں۔

      - دکھائی گئی صفحے میں `Resource usage` ٹیب منتخب کریں - اور میٹرکس دیکھیں۔
      - لاگت (ٹوکنز) اور بوجھ (درخواستیں) کی صورت میں ایپلیکیشن کے استعمال کو ٹریک کریں۔
      - ایپلیکیشن کی تاخیر کو پہلے بائٹ (ان پٹ پروسیسنگ) اور آخری بائٹ (آؤٹ پٹ) تک ٹریک کریں۔

      ![Agent](../../../../../translated_images/ur/13-monitoring-resources.5148015f7311807f.webp)

## 8. ماحول کے متغیرات

اب تک ہم نے براؤزر میں تعیناتی کا جائزہ لیا ہے - اور توثیق کی ہے کہ ہمارا انفراسٹرکچر فراہم ہوگیا ہے اور ایپلیکیشن چل رہی ہے۔ لیکن ایپلیکیشن کے ساتھ _کوڈ فرسٹ_ کام کرنے کے لیے، ہمیں اپنے مقامی ترقیاتی ماحول کو متعلقہ متغیرات کے ساتھ ترتیب دینا ہوگا جو ان وسائل کے ساتھ کام کرنے کے لیے درکار ہوتے ہیں۔ `azd` کا استعمال اس کو آسان بناتا ہے۔

1. Azure Developer CLI [ماحول کے متغیرات](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) استعمال کرتا ہے تاکہ ایپلیکیشن کی تعیناتی کے لیے کنفیگریشن سیٹنگز محفوظ اور منظم کی جا سکیں۔

1. ماحول کے متغیرات `.azure/<env-name>/.env` میں محفوظ کیے جاتے ہیں - یہ انہیں تعیناتی کے دوران استعمال ہونے والے `env-name` ماحول کے دائرہ کار میں رکھتے ہیں اور آپ کو ایک ہی ریپو میں مختلف تعیناتی اہداف کے درمیان ماحول کو الگ رکھنے میں مدد دیتے ہیں۔

1. ماحول کے متغیرات خود بخود `azd` کمانڈ کے چلنے پر لوڈ ہوتے ہیں (مثلاً `azd up`)۔ دھیان دیں کہ `azd` خود بخود _OS سطح کے_ ماحول کے متغیرات (مثلاً شیل میں سیٹ کیے گئے) کو نہیں پڑھتا - اس کے بجائے اندرونی سکرپٹس میں معلومات منتقل کرنے کے لیے `azd set env` اور `azd get env` استعمال کریں۔

آئیے چند کمانڈز آزماتے ہیں:

1. اس ماحول میں `azd` کے لیے سیٹ کیے گئے تمام ماحول کے متغیرات حاصل کریں:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      آپ کو کچھ ایسا نظر آئے گا:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. ایک مخصوص قدر حاصل کریں - مثلاً، میں جاننا چاہتا ہوں کہ کیا `AZURE_AI_AGENT_MODEL_NAME` کی قدر سیٹ ہوئی ہے

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      آپ کو کچھ ایسا نظر آئے گا - یہ ڈیفالٹ طور پر سیٹ نہیں تھی!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd` کے لیے ایک نیا ماحول کا متغیر سیٹ کریں۔ یہاں، ہم ایجنٹ ماڈل کا نام اپ ڈیٹ کر رہے ہیں۔ _نوٹ: جو بھی تبدیلیاں کی جائیں گی وہ فوری طور پر `.azure/<env-name>/.env` فائل میں ظاہر ہوں گی۔_

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      اب، ہمیں وہ قدر سیٹ ہوتی نظر آنی چاہیے:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. نوٹ کریں کہ کچھ وسائل مستقل ہوتے ہیں (مثلاً، ماڈل کی تعیناتیاں) اور ان کی دوبارہ تعیناتی کے لیے صرف `azd up` کافی نہیں ہوتا۔ آئیے موجودہ تعیناتی کو ختم کرکے تبدیل شدہ ماحول کے متغیرات کے ساتھ دوبارہ تعینات کریں۔

1. **ریفریش** اگر آپ نے پہلے کسی azd ٹیمپلیٹ کے استعمال سے انفراسٹرکچر تعینات کیا ہے - تو آپ اپنی مقامی ماحول کے متغیرات کی حالت کو اپنے Azure تعیناتی کی موجودہ حالت کی بنیاد پر _ریفریش_ کر سکتے ہیں اس کمانڈ کے ذریعے:

      ```bash title="" linenums="0"
      azd env refresh
      ```

یہ ایک طاقتور طریقہ ہے کہ آپ دو یا زیادہ مقامی ترقیاتی ماحول (مثلاً کئی ڈویلپرز کی ٹیم) میں ماحول کے متغیرات کو ہم آہنگ کریں - جس سے تعین شدہ انفراسٹرکچر کو ماحول متغیر کی حالت کے لیے حقیقی حقیقت کے طور پر استعمال کیا جا سکے۔ ٹیم کے ارکان صرف متغیرات کو _ریفرش_ کرتے ہیں تاکہ دوبارہ ہم آہنگ ہو سکیں۔
