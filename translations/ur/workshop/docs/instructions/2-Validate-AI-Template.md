!!! tip "اس ماڈیول کے اختتام تک آپ یہ کر سکیں گے"

    - [ ] AI حل کے فن تعمیر کا تجزیہ کریں
    - [ ] AZD ڈپلائمنٹ کے ورک فلو کو سمجھیں
    - [ ] AZD استعمال کرنے میں مدد کے لیے GitHub Copilot کا استعمال کریں

## 1. تعارف

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) یا `azd` ایک اوپن سورس کمانڈ لائن ٹول ہے جو ایپلیکیشنز کو Azure پر بنانے اور تعینات کرنے کے دوران ڈیولپر کے ورک فلو کو آسان بناتا ہے۔

[AZD ٹیمپلیٹس](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) معیاری ریپوزٹریز ہیں جن میں نمونہ ایپلیکیشن کوڈ، _infrastructure-as-code_ اثاثے، اور `azd` کنفیگریشن فائلز شامل ہوتی ہیں تاکہ ایک مربوط حل کا فن تعمیر فراہم کیا جا سکے۔ بنیادی ڈھانچے کی فراہمی ایک `azd provision` کمانڈ جتنا آسان ہو جاتا ہے — جبکہ `azd up` سے آپ بنیادی ڈھانچہ فراہم کرنے کے ساتھ ہی اپنی ایپلیکیشن کو بھی ایک ہی وقت میں تعینات کر سکتے ہیں!

نتیجتاً، اپنی ایپلیکیشن کی ترقی کا عمل شروع کرنا اسی _AZD Starter ٹیمپلیٹ_ کو تلاش کرنے جتنا آسان ہو سکتا ہے جو آپ کی ایپلیکیشن اور بنیادی ڈھانچے کی ضروریات کے قریب ترین ہو — پھر ریپوزٹری کو اپنے منظرنامے کے مطابق حسب ضرورت بنانا۔

شروع کرنے سے پہلے، آئیے یقینی بنائیں کہ آپ نے Azure Developer CLI انسٹال کر رکھا ہے۔

1. VS Code ٹرمینل کھولیں اور یہ کمانڈ ٹائپ کریں:

      ```bash title="" linenums="0"
      azd version
      ```

1. آپ کو کچھ اس طرح دکھائی دے گا!

      ```bash title="" linenums="0"
      azd version 1.27.1 (commit <current-build>)
      ```

**آپ اب azd کے ساتھ ٹیمپلیٹ منتخب کرنے اور تعینات کرنے کے لیے تیار ہیں**

## 2. ٹیمپلیٹ کا انتخاب

Microsoft Foundry پلیٹ فارم میں [تجویز کردہ AZD ٹیمپلیٹس کا ایک مجموعہ](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) شامل ہے جو مقبول حل کے منظرنامے جیسے _کئی ایجنٹوں کے ورک فلو کی خود کاری_ اور _کئی طریقوں سے مواد کی پراسیسنگ_ کو کور کرتے ہیں۔ آپ Microsoft Foundry پورٹل پر جا کر بھی یہ ٹیمپلیٹس دریافت کر سکتے ہیں۔

1. وزٹ کریں [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. جب کہا جائے تو Microsoft Foundry پورٹل میں لاگ ان کریں - آپ کو کچھ اس طرح نظر آئے گا۔

![Pick](../../../../../translated_images/ur/01-pick-template.60d2d5fff5ebc374.webp)


**Basic** آپشنز آپ کے اسٹارٹر ٹیمپلیٹس ہیں:

1. [ ] [Get Started with AI Chat](https://github.com/Azure-Samples/get-started-with-ai-chat) جو Azure Container Apps پر آپ کے ڈیٹا کے ساتھ ایک بنیادی چیٹ ایپلیکیشن تعینات کرتا ہے۔ اسے ایک بنیادی AI چیٹ بوٹ منظرنامے کو دریافت کرنے کے لیے استعمال کریں۔
1. [X] [Get Started with AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) جو ایک معیاری AI ایجنٹ (Foundry Agents کے ساتھ) بھی تعینات کرتا ہے۔ اسے ٹولز اور ماڈلز کے ساتھ ایجنٹک AI حل کو سمجھنے کے لیے استعمال کریں۔

دوسرے لنک کو نئے براؤزر ٹیب میں کھولیں (یا متعلقہ کارڈ کے لیے `Open in GitHub` پر کلک کریں)۔ آپ کو اس AZD ٹیمپلیٹ کا ریپوزٹری دیکھنا چاہیے۔ README کو دریافت کرنے میں ایک منٹ گزاریں۔ ایپلیکیشن کا فن تعمیر یوں دکھائی دیتا ہے:

![Arch](../../../../../translated_images/ur/architecture.8cec470ec15c65c7.webp)

## 3. ٹیمپلیٹ کی فعالیت

آئیے اس ٹیمپلیٹ کو تعینات کرنے کی کوشش کریں اور اس کی درستگی یقینی بنائیں۔ ہم [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) سیکشن کی ہدایات پر عمل کریں گے۔

1. ٹیمپلیٹ ریپوزٹری کے لیے ایک کام کرنے والا ماحول منتخب کریں:

      - **GitHub Codespaces**: اس لنک [https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) پر کلک کریں اور `Create codespace` کی تصدیق کریں۔
      - **لوکل کلون یا ڈیولپمنٹ کنٹینر**: `Azure-Samples/get-started-with-ai-agents` کو کلون کریں اور VS Code میں کھولیں۔

1. جب VS Code ٹرمینل تیار ہو جائے، تو درج ذیل کمانڈ ٹائپ کریں:

   ```bash title="" linenums="0"
   azd up
   ```

اس سے شروع ہونے والے ورک فلو اقدامات مکمل کریں:

1. آپ سے Azure میں لاگ ان کرنے کو کہا جائے گا - ہدایات پر عمل کر کے توثیق کریں۔
1. اپنے لیے ایک منفرد ماحول کا نام درج کریں - مثال کے طور پر، میں نے `nitya-mshack-azd` استعمال کیا۔
1. یہ `.azure/` فولڈر بنائے گا - آپ کو ماحول کے نام کے ساتھ ایک ذیلی فولڈر نظر آئے گا۔
1. آپ سے سبسکرپشن کا نام منتخب کرنے کو کہا جائے گا - ڈیفالٹ منتخب کریں۔
1. آپ سے لوکیشن پوچھا جائے گا - `East US 2` استعمال کریں۔

اب، پروویژننگ مکمل ہونے کا انتظار کریں۔ **اس میں 10-15 منٹ لگتے ہیں**

1. مکمل ہونے پر، آپ کے کنسول میں مندرجہ ذیل کامیابی کا پیغام دکھائی دے گا:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. آپ کے Azure پورٹل میں اب ایک پروویژنڈ ریسورس گروپ ہوگا جس کا نام وہی ماحول کا نام ہوگا:

      ![Infra](../../../../../translated_images/ur/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **آپ اب تعینات کردہ بنیادی ڈھانچے اور ایپلیکیشن کی تصدیق کے لیے تیار ہیں۔**

## 4. ٹیمپلیٹ کی تصدیق

1. Azure پورٹل [Resource Groups](https://portal.azure.com/#browse/resourcegroups) صفحہ پر جائیں - جب کہا جائے لاگ ان کریں۔
1. اپنے ماحول کے نام والے RG پر کلک کریں - آپ کو اوپر والا صفحہ نظر آئے گا۔

      - Azure Container Apps ریسورس پر کلک کریں۔
      - _Essentials_ سیکشن میں، ایپلیکیشن یو آر ایل پر کلک کریں (اوپر دائیں)۔

1. آپ کو اس طرح کی ہوسٹ کی گئی ایپلیکیشن فرنٹ اینڈ UI نظر آئے گی:

   ![App](../../../../../translated_images/ur/03-test-application.471910da12c3038e.webp)

1. چند [نمونہ سوالات](https://github.com/Azure-Samples/get-started-with-ai-agents/blob/main/docs/sample_questions.md) پوچھنے کی کوشش کریں۔

      1. پوچھیں: ```فرانس کا دارالحکومت کیا ہے؟``` 
      1. پوچھیں: ```دو افراد کے لیے 200 ڈالر سے کم میں بہترین خیمہ کون سا ہے، اور اس میں کون سی خصوصیات شامل ہیں؟```

1. آپ کو نیچے دکھائے گئے جوابوں جیسے جوابات ملنے چاہئیں۔ _لیکن یہ کیسے کام کرتا ہے؟_ 

      ![App](../../../../../translated_images/ur/03-test-question.521c1e863cbaddb6.webp)

## 5. ایجنٹ کی تصدیق

Azure Container App ایک اینڈپوائنٹ تعینات کرتا ہے جو اس ٹیمپلیٹ کے لیے Microsoft Foundry پروجیکٹ میں پروویژن کیے گئے AI ایجنٹ سے جڑتا ہے۔ آئیے دیکھتے ہیں کہ اس کا کیا مطلب ہے۔

1. Azure پورٹل میں اپنے ریسورس گروپ کے _Overview_ صفحہ پر واپس جائیں۔

1. اس فہرست میں `Microsoft Foundry` ریسورس پر کلک کریں۔

1. آپ کو یہ نظر آنا چاہیے۔ `Go to Microsoft Foundry Portal` بٹن پر کلک کریں۔
   ![Foundry](../../../../../translated_images/ur/04-view-foundry-project.fb94ca41803f28f3.webp)

1. آپ کو اپنے AI ایپلیکیشن کے لیے Foundry پروجیکٹ صفحہ نظر آئے گا۔
   ![Project](../../../../../translated_images/ur/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` پر کلک کریں - آپ کو اپنے پروجیکٹ میں ڈیفالٹ ایجنٹ نظر آئے گا۔
   ![Agents](../../../../../translated_images/ur/06-visit-agents.bccb263f77b00a09.webp)

1. اسے منتخب کریں - اور آپ کو ایجنٹ کی تفصیلات نظر آئیں گی۔ درج ذیل باتوں پر توجہ دیں:

      - ایجنٹ ڈیفالٹ طور پر فائل سرچ استعمال کرتا ہے (ہمیشہ)
      - ایجنٹ کی `Knowledge` ظاہر کرتی ہے کہ اس میں 32 فائلیں اپ لوڈ کی گئی ہیں (فائل سرچ کے لیے)
      ![Agents](../../../../../translated_images/ur/07-view-agent-details.0e049f37f61eae62.webp)

1. بائیں مینو میں `Data+indexes` آپشن تلاش کریں اور تفصیلات دیکھنے کے لیے کلک کریں۔

      - آپ کو علم کے لیے 32 ڈیٹا فائلیں اپ لوڈ کی ہوئی نظر آئیں گی۔
      - یہ `src/files` کے تحت 12 کسٹمر فائلیں اور 20 پروڈکٹ فائلوں سے میل کھائیں گی۔
      ![Data](../../../../../translated_images/ur/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**آپ نے ایجنٹ کے آپریشن کی تصدیق کر لی ہے!** 

1. ایجنٹ کے جوابات ان فائلوں میں موجود علم پر مبنی ہوتے ہیں۔
1. آپ اب اس ڈیٹا سے متعلق سوالات پوچھ سکتے ہیں، اور معروضی جوابات حاصل کر سکتے ہیں۔
1. مثال کے طور پر: `customer_info_10.json` میں "Amanda Perez" کی کی گئی 3 خریداریوں کی تفصیل ہے۔

براؤزر کے اس ٹیب پر واپس جائیں جہاں Container App اینڈپوائنٹ کھلا ہے اور پوچھیں: `Amanda Perez کے کون سے پروڈکٹس ہیں؟`. آپ کو کچھ اس طرح نظر آئے گا:

![Data](../../../../../translated_images/ur/09-ask-in-aca.4102297fc465a4d5.webp)

## 6. ایجنٹ کا پلے گراؤنڈ

Microsoft Foundry کی صلاحیتوں کے بارے میں مزید سمجھ بوجھ حاصل کرنے کے لیے، ایجنٹ کو Agents Playground میں آزمائیں۔

1. Microsoft Foundry میں `Agents` صفحہ پر واپس جائیں - ڈیفالٹ ایجنٹ منتخب کریں۔
1. `Try in Playground` آپشن پر کلک کریں - آپ کو اس طرح کا Playground UI ملے گا۔
1. وہی سوال پوچھیں: `Amanda Perez کے کون سے پروڈکٹس ہیں؟`

    ![Data](../../../../../translated_images/ur/09-ask-in-playground.a1b93794f78fa676.webp)

آپ کو وہی (یا ملتا جلتا) جواب ملے گا - لیکن آپ کو اضافی معلومات بھی ملے گی جو آپ کو آپ کے ایجنٹک ایپ کی معیار، لاگت، اور کارکردگی کو سمجھنے میں مدد دے گی۔ مثال کے طور پر:

1. نوٹ کریں کہ جواب میں اس ڈیٹا فائل کا حوالہ دیا گیا ہے جس سے جواب کو "گراؤنڈ" کیا گیا ہے۔
1. ان فائل لیبلز پر ہوور کریں - کیا ڈیٹا آپ کی استفسار اور دکھائے گئے جواب سے میل کھاتا ہے؟

آپ کو جواب کے نیچے ایک _stats_ صف بھی نظر آئے گی۔

1. کسی بھی میٹرک پر ہوور کریں - مثلاً Safety۔ آپ کو کچھ اس طرح نظر آئے گا۔
1. کیا آپ کی سوچ کے مطابق، اس جائزے میں جواب کی حفاظتی سطح کا اندازہ درست ہے؟

      ![Data](../../../../../translated_images/ur/10-view-run-info-meter.6cdb89a0eea5531f.webp)

## 7. بلٹ-ان آبزرویبیلیٹی

آبزرویبیلیٹی کا مطلب ہے اپنی ایپلیکیشن کو اس طرح آلہ بند کرنا کہ وہ آپریشنز سے متعلق ڈیٹا پیدا کرے جسے سمجھنے، ڈی بگ کرنے، اور بہتر بنانے کے لیے استعمال کیا جا سکے۔ اس کا اندازہ لگانے کے لیے:

1. `View Run Info` بٹن پر کلک کریں - آپ کو یہ منظر نظر آنا چاہیے۔ یہ [Agent tracing](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) کی مثال ہے۔ _آپ اسے Thread Logs پر کلک کر کے بھی دیکھ سکتے ہیں۔_

   - ایجنٹ کے استعمال کردہ قدم اور ٹولز کا اندازہ لگائیں
   - جواب کے لیے کل ٹوکنز کی گنتی سمجھیں (آؤٹ پٹ ٹوکنز کے مقابلے میں)
   - تاخیر کو سمجھیں اور دیکھیں کہ عملدرآمد میں کس جگہ وقت صرف ہو رہا ہے

      ![Agent](../../../../../translated_images/ur/10-view-run-info.b20ebd75fef6a1cc.webp)

1. `Metadata` ٹیب پر کلک کریں تاکہ اضافی خصوصیات دیکھ سکیں جو بعد میں مسائل کی درستگی کے لیے مددگار ہو سکتی ہیں۔

      ![Agent](../../../../../translated_images/ur/11-view-run-info-metadata.7966986122c7c2df.webp)


1. `Evaluations` ٹیب پر کلک کریں تاکہ ایجنٹ کے جواب پر خودکار جائزے دیکھ سکیں۔ ان میں حفاظتی جائزے شامل ہیں (جیسے Self-harm) اور ایجنٹ مخصوص جائزے (جیسے Intent resolution, Task adherence).

      ![Agent](../../../../../translated_images/ur/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. آخر میں، سائڈبار مینو میں `Monitoring` ٹیب پر کلک کریں۔

      - ظاہر شدہ صفحے میں `Resource usage` ٹیب منتخب کریں اور میٹرکس دیکھیں۔
      - اپلیکیشن کے استعمال کو لاگت (ٹوکنز) اور بوجھ (درخواستیں) کے لحاظ سے ٹریک کریں۔
      - اپلیکیشن کی تاخیر کو پہلے بائٹ (ان پٹ پروسیسنگ) اور آخری بائٹ (آؤٹ پٹ) کے لحاظ سے ٹریک کریں۔

      ![Agent](../../../../../translated_images/ur/13-monitoring-resources.5148015f7311807f.webp)

## 8. ماحول کے متغیرات

اب تک ہم نے براؤزر میں تعیناتی کی اور تصدیق کی کہ ہمارا بنیادی ڈھانچہ فراہم کیا گیا ہے اور ایپلیکیشن فعال ہے۔ لیکن ایپلیکیشن کے _کوڈ-فرسٹ_ کام کرنے کے لیے، ہمیں اپنے مقامی ترقیاتی ماحول کو متعلقہ متغیرات سے ترتیب دینا ہوگا جو ان وسائل کے ساتھ کام کرنے کے لیے ضروری ہیں۔ `azd` کے استعمال سے یہ آسان ہو جاتا ہے۔

1. Azure Developer CLI [ماحول کے متغیرات](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) استعمال کرتا ہے تاکہ ایپلیکیشن کی تعیناتی کے لیے کنفیگریشن سیٹنگز کو ذخیرہ اور منظم کیا جا سکے۔

1. ماحول کے متغیرات `.azure/<env-name>/.env` میں ذخیرہ کیے جاتے ہیں — یہ انہیں تعیناتی کے دوران استعمال ہونے والے `env-name` ماحول کے لیے مخصوص کرتا ہے اور آپ کو ایک ہی ریپو میں مختلف تعیناتی اہداف کے ماحول کو الگ کرنے میں مدد دیتا ہے۔

1. ماحول کے متغیرات کو `azd` کمانڈ خود بخود لوڈ کرتا ہے جب بھی وہ کسی مخصوص کمانڈ کو چلائے (مثلاً `azd up`)۔ نوٹ کریں کہ `azd` خود بخود _OS سطح کے_ ماحول کے متغیرات (مثلاً شیل میں سیٹ کیے گئے) کو نہیں پڑھتا — اس کے بجائے، اسکرپٹس میں معلومات منتقل کرنے کے لیے `azd set env` اور `azd get env` استعمال کریں۔


آئیے کچھ کمانڈز آزمائیں:

1. اس ماحول میں `azd` کے لیے سیٹ کیے گئے تمام ماحول کے متغیرات حاصل کریں:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      آپ کو کچھ اس طرح نظر آئے گا:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. ایک مخصوص قدر حاصل کریں — مثلاً، میں جاننا چاہتا ہوں کہ ہم نے `AZURE_AI_AGENT_MODEL_NAME` کی قیمت سیٹ کی ہے یا نہیں۔

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      آپ کو کچھ اس طرح نظر آئے گا — یہ ڈیفالٹ کے طور پر سیٹ نہیں تھی!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd` کے لیے نیا ماحول کا متغیر سیٹ کریں۔ یہاں، ہم ایجنٹ ماڈل کا نام اپڈیٹ کر رہے ہیں۔ _نوٹ: جو بھی تبدیلی کی جائے گی وہ فوری طور پر `.azure/<env-name>/.env` فائل میں نظر آئے گی۔_

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      اب، ہمیں معلوم ہوگا کہ قدر سیٹ ہو گئی ہے:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. نوٹ کریں کہ کچھ وسائل مستقل ہوتے ہیں (مثلاً ماڈل کی تعیناتیاں) اور انہیں صرف `azd up` سے دوبارہ تعینات کرنے کی ضرورت ہوگی۔ آئیے اصل تعیناتی کو ختم کر کے تبدیل شدہ ماحولیاتی متغیرات کے ساتھ دوبارہ تعیناتی کریں۔

1. **تازہ کاری** اگر آپ نے پہلے کسی azd ٹیمپلیٹ کا استعمال کر کے بنیادی ڈھانچہ تعینات کیا تھا ـ تو آپ اپنے مقامی ماحول کے متغیرات کو Azure کی موجودہ تعیناتی کی حالت کے مطابق تازہ کاری کر سکتے ہیں اس کمانڈ کے ذریعے:

      ```bash title="" linenums="0"
      azd env refresh
      ```


یہ دو یا زیادہ مقامی ترقیاتی ماحول (مثلاً، متعدد ڈویلپرز کی ٹیم) میں ماحول کی متغیرات کو ہم آہنگ کرنے کا ایک طاقتور طریقہ ہے - جو تعینات شدہ انفراسٹرکچر کو ماحول کی متغیر حالت کے لیے حقیقی بنیاد کے طور پر کام کرنے دیتا ہے۔ ٹیم کے ارکان بس متغیرات کو تازہ کرتے ہیں تاکہ دوبارہ ہم آہنگ ہو جائیں۔
