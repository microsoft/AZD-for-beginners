# AZD Pour les développeurs d'IA

!!! info "PUBLIC POUR CET ATELIER"
   
    Cet atelier s'adresse à toute personne souhaitant créer des applications Agentic IA sur Microsoft Foundry en tirant parti des outils et workflows de premier ordre. L'atelier convient aussi bien aux débutants qu'aux développeurs expérimentés ayant une certaine familiarité avec Azure, les concepts d'IA et le code Python. Veuillez vérifier les prérequis avant de commencer.

    **Durée :** 3-4 heures  
    **Niveau :** Débutant à intermédiaire  

---


## Prérequis

- Un compte GitHub personnel → [Créez-en un gratuitement](https://github.com/signup)
- Un compte Azure actif → [Créez-en un gratuitement](https://aka.ms/free)
- Familiarité avec les concepts d'IA, le développement Python, les outils en ligne de commande
- Expérience avec VS Code et les outils et workflows GitHub.

---

## Objectifs d'apprentissage

À la fin de cet atelier, vous saurez comment :

- [X] **Travailler avec Azure Developer CLI** → pour provisionner, déployer et gérer vos ressources Azure.
- [X] **Utiliser les modèles d'applications IA** → pour démarrer rapidement des projets AI Agent sur Microsoft Foundry.
- [X] **Configurer les modèles** → pour des fonctionnalités comme Red Teaming, Évaluations, Traçage & Surveillance.
- [X] **Déconstruire et Explorer** → les modèles et workflows AZD avec GitHub Copilot.
- [X] **Personnaliser et redéployer** → le modèle de démarrage AI Agent, avec vos exigences personnalisées.

---

## Plan de l'atelier

L'atelier est structuré en modules, chacun étant centré sur un exercice pratique en laboratoire :

1. **Introduction** - Comprendre ce que sont les modèles AZD et leur fonctionnement
1. **Sélection** - Découvrir le bon modèle "starter" pour votre prochain projet IA
1. **Validation** - Déployer et explorer le modèle pour vérifier qu'il fonctionne comme annoncé
1. **Déconstruction** - Décomposer le modèle pour comprendre ses composants et sa configuration
1. **Configuration** - Apprendre à activer et utiliser différentes fonctionnalités de Microsoft Foundry
1. **Personnalisation** - Reconstruire l’application avec vos données et vos besoins en tête
1. **Démontage** - Apprendre à désapprovisionner l’infrastructure et récupérer les ressources supprimées en douceur
