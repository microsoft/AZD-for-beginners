# 為 AZD For Beginners 做出貢獻

感謝你有興趣為 AZD For Beginners 課程做出貢獻！本文件提供給貢獻者的指引與資訊。

## 如何貢獻

### 回報問題

如果你發現錯誤、問題或有改進建議：

1. **檢查現有議題**：搜尋 [現有議題](https://github.com/microsoft/azd-for-beginners/issues) 以避免重複
2. **建立新議題**：使用適當的議題範本
3. **提供詳情**：包含重現步驟、預期行為，以及如適用的截圖

### 提議變更

1. **Fork the Repository**：建立你自己的專案 fork
2. **Create a Branch**：為你的變更建立一個功能分支
3. **Make Changes**：遵循我們的程式碼與文件標準
4. **Test Your Changes**：確認所有連結可用且內容正確呈現
5. **Submit a Pull Request**：提供清楚的變更描述

### 貢獻類型

我們歡迎以下類型的貢獻：

#### 內容改進
- 修正錯字、文法與格式問題
- 提升現有文件的清晰度
- 更新過時資訊
- 補充缺少的說明或背景

#### 新增範例
- 新增真實世界的部署情境
- 為常見使用情境建立新的 AZD 範本
- 提供更多程式碼範例與練習題

#### 翻譯支援
- 協助維護現有翻譯
- 提升翻譯品質
- 為新內容新增翻譯

#### 錯誤回報
- 回報斷掉的連結或不正確的程式範例
- 指認文件不一致處
- 標記可及性（accessibility）問題

## 貢獻準則

### 文件標準

- 使用清楚、適合初學者的語言
- 包含學習目標與學習成果
- 提供逐步教學並附上預期輸出
- 新增導覽連結至相關內容
- 使用一致的格式與 markdown 風格

### 程式範例

- 確保所有程式範例已測試並可運作
- 加入說明關鍵概念的註解
- 遵循 Azure 與 AZD 的最佳實務
- 在適用情況下提供預期輸出

### 提交訊息

使用清楚且具描述性的提交訊息：
- `docs: fix typo in installation guide`
- `feat: add new container app example`
- `fix: correct broken link in chapter 2`

## 行為守則

請閱讀並遵守我們的 [行為守則](CODE_OF_CONDUCT.md)，以維持一個友善且包容的社群。

## 尋求協助

- **問題**：開啟一個 [GitHub 討論](https://github.com/microsoft/azd-for-beginners/discussions)
- **社群**：加入 [Azure Discord](https://discord.gg/microsoft-azure)
- **文件**：檢視 [Azure Developer CLI 文件](https://learn.microsoft.com/azure/developer/azure-developer-cli/)
