# Contribuir para o AZD For Beginners

Obrigado pelo seu interesse em contribuir para o curso AZD For Beginners! Este documento fornece diretrizes e informações para colaboradores.

## Como Contribuir

### Reportar Problemas

Se encontrar bugs, erros ou tiver sugestões de melhorias:

1. **Verificar Issues Existentes**: Pesquise [issues existentes](https://github.com/microsoft/azd-for-beginners/issues) para evitar duplicados
2. **Criar um Novo Issue**: Use o template apropriado para issues
3. **Fornecer Detalhes**: Inclua passos para reproduzir, comportamento esperado e capturas de ecrã, se aplicável

### Propor Alterações

1. **Fazer um Fork do Repositório**: Crie o seu próprio fork do projeto
2. **Criar uma Branch**: Crie uma branch de funcionalidade para as suas alterações
3. **Fazer Alterações**: Siga os nossos padrões de codificação e de documentação
4. **Testar as Suas Alterações**: Verifique se todos os links funcionam e se o conteúdo é apresentado corretamente
5. **Submeter um Pull Request**: Forneça uma descrição clara das suas alterações

### Tipos de Contribuições

Apreciamos os seguintes tipos de contribuições:

#### Melhorias de Conteúdo
- Corrigir erros tipográficos, gramaticais e de formatação
- Melhorar a clareza da documentação existente
- Atualizar informações desatualizadas
- Adicionar explicações ou contexto em falta

#### Novos Exemplos
- Adicionar cenários de implementação no mundo real
- Criar novos templates AZD para casos de uso comuns
- Fornecer exemplos de código adicionais e exercícios

#### Apoio à Tradução
- Ajudar a manter traduções existentes
- Melhorar a qualidade das traduções
- Adicionar traduções para novos conteúdos

#### Relatórios de Bugs
- Reportar links quebrados ou exemplos de código incorretos
- Identificar inconsistências na documentação
- Assinalar questões de acessibilidade

## Diretrizes de Contribuição

### Padrões de Documentação

- Utilize uma linguagem clara e adequada para iniciantes
- Inclua objetivos de aprendizagem e resultados
- Forneça instruções passo a passo com as saídas esperadas
- Adicione links de navegação para conteúdos relacionados
- Use formatação consistente e estilo markdown

### Exemplos de Código

- Assegure que todos os exemplos de código sejam testados e funcionem
- Inclua comentários a explicar os conceitos-chave
- Siga as melhores práticas da Azure e do AZD
- Forneça a saída esperada quando aplicável

### Mensagens de Commit

Use mensagens de commit claras e descritivas:
- `docs: fix typo in installation guide`
- `feat: add new container app example`
- `fix: correct broken link in chapter 2`

## Código de Conduta

Por favor, leia e siga o nosso [Código de Conduta](CODE_OF_CONDUCT.md) para manter uma comunidade acolhedora e inclusiva.

## Obter Ajuda

- **Perguntas**: Abra uma [Discussão no GitHub](https://github.com/microsoft/azd-for-beginners/discussions)
- **Comunidade**: Junte-se ao [Discord da Azure](https://discord.gg/microsoft-azure)
- **Documentação**: Consulte a [documentação do Azure Developer CLI](https://learn.microsoft.com/azure/developer/azure-developer-cli/)
