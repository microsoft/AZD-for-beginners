## 1. المقدمة

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) أو `azd` هو أداة سطر أوامر مفتوحة المصدر تبسط سير عمل المطور عند بناء ونشر التطبيقات إلى Azure.

[قوالب AZD](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) هي مستودعات معيارية تشمل رمز تطبيق نموذجي، وأصول _البنية التحتية كرمز_، وملفات تكوين `azd` لهندسة حل متماسكة. يصبح توفير البنية التحتية بسيطًا مثل أمر `azd provision` - بينما يتيح لك استخدام `azd up` توفير البنية التحتية **ونشر** تطبيقك دفعة واحدة!

نتيجة لذلك، يمكن أن تكون بداية عملية تطوير تطبيقك بسيطة مثل إيجاد _قالب AZD المبدئي_ الأقرب لاحتياجات تطبيقك والبنية التحتية - ثم تخصيص المستودع ليتناسب مع متطلبات سيناريو حالتك.

قبل أن نبدأ، دعنا نتحقق من تثبيت Azure Developer CLI.

1. افتح محطة أوامر في VS Code واكتب هذا الأمر:

      ```bash title="" linenums="0"
      azd version
      ```

1. يجب أن ترى شيئًا مثل هذا!

      ```bash title="" linenums="0"
      azd version 1.27.1 (commit <current-build>)
      ```

**أنت الآن جاهز لاختيار ونشر قالب باستخدام azd**

## 2. اختيار القالب

منصة Microsoft Foundry تأتي مع [مجموعة من قوالب AZD الموصى بها](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) التي تغطي سيناريوهات حلول شائعة مثل _أتمتة سير عمل متعدد الوكلاء_ و _معالجة محتوى متعددة الوسائط_. يمكنك أيضًا اكتشاف هذه القوالب بزيارة بوابة Microsoft Foundry.

1. زر [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. سجّل الدخول إلى بوابة Microsoft Foundry عند الطلب - سترى شيئًا مثل هذا.

![اختر](../../../../../translated_images/ar/01-pick-template.60d2d5fff5ebc374.webp)


خيارات **Basic** هي قوالب البداية الخاصة بك:

1. [ ] [ابدأ مع دردشة الذكاء الاصطناعي](https://github.com/Azure-Samples/get-started-with-ai-chat) التي تنشر تطبيق دردشة أساسي _مع بياناتك_ إلى Azure Container Apps. استخدم هذا لاستكشاف سيناريو روبوت دردشة AI بسيط.
1. [X] [ابدأ مع وكلاء الذكاء الاصطناعي](https://github.com/Azure-Samples/get-started-with-ai-agents) التي تنشر أيضًا وكيل AI قياسي (مع وكلاء Foundry). استخدم هذا للتعرف على حلول AI الوكيلة التي تتضمن أدوات ونماذج.

زر الرابط الثاني في علامة تبويب جديدة بالمتصفح (أو انقر على `Open in GitHub` للبطاقة ذات الصلة). يجب أن ترى المستودع لهذا القالب AZD. خذ دقيقة لاستكشاف ملف README. تبدو هندسة التطبيق كما يلي:

![الهندسة](../../../../../translated_images/ar/architecture.8cec470ec15c65c7.webp)

## 3. تفعيل القالب

دعنا نحاول نشر هذا القالب والتأكد من صحته. سنتبع الإرشادات في قسم [البدء](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started).

1. اختر بيئة عمل لمستودع القالب:

      - **GitHub Codespaces**: انقر [على هذا الرابط](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) وقم بتأكيد `Create codespace`
      - **نسخة محلية أو حاوية تطوير**: استنسخ `Azure-Samples/get-started-with-ai-agents` وافتحه في VS Code

1. انتظر حتى تصبح محطة VS Code جاهزة، ثم اكتب الأمر التالي:

   ```bash title="" linenums="0"
   azd up
   ```

أكمل خطوات سير العمل التي ستحفزها هذه الخطوة:

1. سيُطلب منك تسجيل الدخول إلى Azure - اتبع التعليمات للمصادقة
1. أدخل اسم بيئة فريد خاص بك - على سبيل المثال، استخدمت `nitya-mshack-azd`
1. هذا سينشئ مجلد `.azure/` - سترى مجلدًا فرعيًا يحمل اسم البيئة
1. سيُطلب منك اختيار اسم اشتراك - اختر الافتراضي
1. سيُطلب منك اختيار موقع - استخدم `East US 2`

الآن، انتظر حتى يكتمل التوفير. **هذا يستغرق 10-15 دقيقة**

1. عند الانتهاء، سيظهر في وحدة التحكم لديك رسالة نجاح مثل هذه:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. سيكون لديك في بوابة Azure مجموعة موارد مزودة بهذا الاسم:

      ![البنية التحتية](../../../../../translated_images/ar/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **أنت الآن جاهز للتحقق من صحة البنية التحتية والتطبيق المنشور**.

## 5. التحقق من الوكيل

يقوم تطبيق Azure Container App بنشر نقطة نهاية تتصل بوكيل AI المزود في مشروع Microsoft Foundry لهذا القالب. دعنا نلقي نظرة على ما يعنيه ذلك.

1. عد إلى صفحة _نظرة عامة_ في بوابة Azure لمجموعة الموارد الخاصة بك

1. انقر على مورد `Microsoft Foundry` في تلك القائمة

1. يجب أن ترى هذا. انقر على زر `Go to Microsoft Foundry Portal`.
   ![Foundry](../../../../../translated_images/ar/04-view-foundry-project.fb94ca41803f28f3.webp)

1. سترى صفحة مشروع Foundry لتطبيق الذكاء الاصطناعي الخاص بك
   ![المشروع](../../../../../translated_images/ar/05-visit-foundry-portal.d734e98135892d7e.webp)

1. انقر على `Agents` - سترى الوكيل الافتراضي المزود في مشروعك
   ![الوكلاء](../../../../../translated_images/ar/06-visit-agents.bccb263f77b00a09.webp)

1. حدده - وسترى تفاصيل الوكيل. لاحظ ما يلي:

      - الوكيل يستخدم بحث الملفات كإعداد افتراضي (دائمًا)
      - يشير `المعرفة` في الوكيل إلى أنه يحتوي على 32 ملفًا محملاً (لبحث الملفات)
      ![الوكلاء](../../../../../translated_images/ar/07-view-agent-details.0e049f37f61eae62.webp)

1. ابحث عن خيار `Data+indexes` في القائمة اليسرى وانقر لعرض التفاصيل.

      - سترى 32 ملف بيانات تم تحميلها للمعرفة.
      - هذه ستتوافق مع 12 ملف للعميل و 20 ملف للمنتج تحت `src/files`
      ![البيانات](../../../../../translated_images/ar/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**لقد تحققت من تشغيل الوكيل!**

1. ردود الوكيل تستند إلى المعرفة في تلك الملفات.
1. يمكنك الآن طرح أسئلة متعلقة بتلك البيانات، والحصول على ردود مؤسسة عليها.
1. مثال: يصف `customer_info_10.json` عمليات الشراء الثلاثة التي قامت بها "Amanda Perez"

عد إلى علامة تبويب المتصفح التي تحتوي على نقطة نهاية تطبيق الحاوية واطرح السؤال: `ما هي المنتجات التي تمتلكها Amanda Perez؟`. يجب أن ترى شيئًا مثل هذا:

![البيانات](../../../../../translated_images/ar/09-ask-in-aca.4102297fc465a4d5.webp)

## 6. ملعب الوكيل

لنبني فهمًا أعمق لقدرات Microsoft Foundry، من خلال تجربة الوكيل في ملعب الوكلاء.

1. عد إلى صفحة `Agents` في Microsoft Foundry - اختر الوكيل الافتراضي
1. انقر على خيار `Try in Playground` - يجب أن تحصل على واجهة ملعب مثل هذا
1. اسأل نفس السؤال: `ما هي المنتجات التي تمتلكها Amanda Perez؟`

    ![البيانات](../../../../../translated_images/ar/09-ask-in-playground.a1b93794f78fa676.webp)

ستحصل على نفس الرد (أو مشابه) - ولكنك ستحصل أيضًا على معلومات إضافية يمكنك استخدامها لفهم جودة، وتكلفة، وأداء تطبيق الوكيل الخاص بك. على سبيل المثال:

1. لاحظ أن الرد يستشهد بملفات بيانات استخدمت "لأساس" الرد
1. مرر فوق أي من هذه الملصقات الخاصة بالملفات - هل تتطابق البيانات مع استفسارك والرد المعروض؟

سترى أيضًا صف _الإحصائيات_ تحت الرد.

1. مرر فوق أي مقياس - مثلًا السلامة. سترى شيئًا مثل هذا
1. هل يتوافق التقييم المقيم مع حدسك لمستوى أمان الرد؟

      ![البيانات](../../../../../translated_images/ar/10-view-run-info-meter.6cdb89a0eea5531f.webp)

## 7. الرصد المدمج

الرصد يتعلق بأدوات تطبيقك لتوليد بيانات يمكن استخدامها لفهم، وتصحيح، وتحسين عمليه. للحصول على فكرة عن ذلك:

1. انقر زر `View Run Info` - يجب أن ترى هذا العرض. هذا مثال على [تتبع الوكيل](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) أثناء العمل. _يمكنك أيضًا الحصول على هذا العرض بالنقر على Thread Logs في القائمة العلوية_.

   - اطلع على خطوات التشغيل والأدوات التي شارك بها الوكيل
   - فهم العدد الإجمالي للرموز (مقابل استخدام رموز الإخراج) للرد
   - فهم زمن الاستجابة وأين يقضي الوقت في التنفيذ

      ![الوكيل](../../../../../translated_images/ar/10-view-run-info.b20ebd75fef6a1cc.webp)

1. انقر تبويب `Metadata` لرؤية سمات إضافية للتشغيل، قد توفر سياقًا مفيدًا لتصحيح المشاكل لاحقًا.

      ![الوكيل](../../../../../translated_images/ar/11-view-run-info-metadata.7966986122c7c2df.webp)


1. انقر تبويب `Evaluations` لرؤية التقييمات الذاتية للردود التي أجراها الوكيل. هذه تشمل تقييمات السلامة (مثل إيذاء النفس) وتقييمات خاصة بالوكيل (مثل حل النية، الالتزام بالمهمة).

      ![الوكيل](../../../../../translated_images/ar/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. وأخيرًا، انقر تبويب `Monitoring` في قائمة الشريط الجانبي.

      - اختر تبويب `Resource usage` في الصفحة المعروضة - وشاهد المقاييس.
      - تتبع استخدام التطبيق من حيث التكلفة (الرموز) والتحميل (الطلبات).
      - تتبع زمن استجابة التطبيق لأول بايت (معالجة الإدخال) وآخر بايت (الإخراج).

      ![الوكيل](../../../../../translated_images/ar/13-monitoring-resources.5148015f7311807f.webp)

## 8. متغيرات البيئة

حتى الآن، مررنا بنشر التطبيق في المتصفح - وتحققنا من توفير البنية التحتية وتشغيل التطبيق. ولكن للعمل مع _كود التطبيق أولاً_، نحتاج إلى تكوين بيئة التطوير المحلية مع المتغيرات ذات الصلة المطلوبة للعمل مع هذه الموارد. استخدام `azd` يجعل الأمر سهلاً.

1. Azure Developer CLI [يستخدم متغيرات البيئة](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) لتخزين وإدارة إعدادات التكوين لنشر التطبيقات.

1. متغيرات البيئة مخزنة في `.azure/<env-name>/.env` - وهذا يحدد نطاقها للبيئة `env-name` المستخدمة أثناء النشر ويساعدك على عزل البيئات بين أهداف النشر المختلفة في نفس المستودع.

1. يتم تحميل متغيرات البيئة تلقائيًا بواسطة أمر `azd` كلما نفذ أمرًا محددًا (مثل `azd up`). لاحظ أن `azd` لا يقرأ تلقائيًا متغيرات البيئة على مستوى النظام (مثل تلك المعينة في الصدفة) - بل استخدم `azd set env` و `azd get env` لنقل المعلومات ضمن السكربتات.


دعنا نجرب بعض الأوامر:

1. احصل على جميع متغيرات البيئة المعينة لـ `azd` في هذه البيئة:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      سترى شيئًا مثل:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. احصل على قيمة معينة - على سبيل المثال، أريد أن أعرف إذا قمنا بتعيين قيمة `AZURE_AI_AGENT_MODEL_NAME`

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      سترى شيئًا مثل هذا - لم يتم تعيينها افتراضيًا!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. قم بتعيين متغير بيئة جديد لـ `azd`. هنا، نقوم بتحديث اسم نموذج الوكيل. _ملاحظة: أي تغييرات يتم إجراؤها ستنعكس فورًا في ملف `.azure/<env-name>/.env`.

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      الآن، يجب أن نجد أن القيمة تم تعيينها:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. لاحظ أن بعض الموارد دائمة (مثل نشرات النماذج) وستتطلب أكثر من مجرد `azd up` لإجبار إعادة النشر. دعنا نجرب تفكيك النشر الأصلي وإعادة النشر مع متغيرات البيئة المعدلة.

1. **تحديث** إذا قمت سابقًا بنشر البنية التحتية باستخدام قالب azd - يمكنك _تحديث_ حالة متغيرات البيئة المحلية استنادًا إلى الحالة الحالية لنشر Azure باستخدام هذا الأمر:

      ```bash title="" linenums="0"
      azd env refresh
      ```


      هذه طريقة قوية لـ _مزامنة_ متغيرات البيئة عبر بيئتين أو أكثر من بيئات التطوير المحلية (على سبيل المثال، فريق مع عدة مطورين) - مما يسمح للبنية التحتية المنشورة بأن تكون المصدر الحقيقي لحالة متغيرات البيئة. يقوم أعضاء الفريق ببساطة بـ _تحديث_ المتغيرات للعودة للمزامنة.
