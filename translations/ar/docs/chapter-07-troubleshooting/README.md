# الفصل 7: استكشاف الأخطاء وإصلاحها وتصحيحها

**📚 الدورة**: [AZD للمبتدئين](../../README.md) | **⏱️ المدة**: من 1 إلى 1.5 ساعة | **⭐ الصعوبة**: متوسط

---

## نظرة عامة

يساعدك هذا الفصل في تشخيص وحل المشاكل الشائعة عند العمل مع Azure Developer CLI. من فشل عمليات النشر إلى المشاكل الخاصة بالذكاء الاصطناعي.

> تم التحقق من صحة المعلومات باستخدام `azd 1.27.1` في يوليو 2026.

## أهداف التعلم

عند إكمال هذا الفصل، ستتمكن من:
- تشخيص فشل النشر الشائع في AZD
- تصحيح مشكلات المصادقة والأذونات
- حل مشاكل الاتصال بخدمات الذكاء الاصطناعي
- استخدام بوابة Azure وCLI لاستكشاف الأخطاء وإصلاحها

## 🚨 إصلاحات سريعة

### مشكلات المصادقة
```bash
# مطلوب لتدفقات عمل AZD
azd auth login

# اختياري إذا كنت تستخدم أوامر Azure CLI مباشرة أيضًا
az login

azd auth status
```

### فشل التهيئة
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### تعارضات الموارد
```bash
azd down --force --purge
azd env new different-name
azd up
```

### تجاوز الحصة
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 قائمة رموز الأخطاء

| الخطأ | السبب | الحل |
|-------|-------|----------|
| `AuthenticationError` | غير مسجل الدخول | `azd auth login` |
| `ResourceNotFound` | المورد مفقود | تحقق من أسماء الموارد |
| `QuotaExceeded` | تجاوز حدود الاشتراك | طلب زيادة الحصة |
| `InvalidTemplate` | خطأ في بناء جملة Bicep | `az bicep build` |
| `Conflict` | المورد موجود | استخدم اسمًا جديدًا أو احذفه |
| `Forbidden` | أذونات غير كافية | تحقق من أدوار RBAC |

## 🔄 إعادة التعيين والاسترداد

```bash
# إعادة ضبط ناعمة (الاحتفاظ بالموارد، إعادة نشر الكود)
azd deploy --force

# إعادة ضبط صعبة (حذف كل شيء، البدء من جديد)
azd down --force --purge
azd up
```
