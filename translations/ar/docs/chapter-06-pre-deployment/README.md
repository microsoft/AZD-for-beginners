# الفصل 6: التخطيط والتحقق قبل النشر

**📚 الدورة**: [AZD للمبتدئين](../../README.md) | **⏱️ المدة**: 1 ساعة | **⭐ التعقيد**: متوسط

---

## نظرة عامة

يغطي هذا الفصل خطوات التخطيط والتحقق الأساسية قبل نشر تطبيقك. تعلّم كيفية تجنب الأخطاء المكلفة من خلال التخطيط السليم للسعة، اختيار SKU المناسب، وفحوصات ما قبل الطيران.

## أهداف التعلم

عند إكمال هذا الفصل، ستتمكن من:
- تشغيل فحوصات ما قبل الطيران قبل النشر
- تخطيط السعة وتقدير متطلبات الموارد
- اختيار SKU المناسبة لتحسين التكلفة
- تكوين Application Insights للمراقبة
- فهم أنماط تنسيق الفريق

## 🚀 بدء سريع

```bash
# التحقق من حصص الاشتراك
az vm list-usage --location eastus --output table

# معاينة النشر (لن يتم إنشاء أي موارد)
azd provision --preview

# التحقق من صحة بناء جملة Bicep
az bicep build --file infra/main.bicep

# التحقق من تكوين البيئة
azd env get-values
```

---

## ☑️ قائمة التحقق قبل النشر

### قبل `azd provision`

- [ ] التحقق من الحصة للمنطقة
- [ ] اختيار SKU المناسبة
- [ ] مراجعة تقدير التكلفة
- [ ] تناسق قواعد التسمية
- [ ] تكوين الأمان/RBAC

### قبل `azd deploy`

- [ ] تعيين متغيرات البيئة
- [ ] الأسرار في Key Vault
- [ ] التحقق من سلاسل الاتصال
- [ ] تكوين فحوصات الصحة

| بحث الذكاء الاصطناعي | Basic | Standard S2+ |
