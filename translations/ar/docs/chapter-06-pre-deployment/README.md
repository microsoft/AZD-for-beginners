# الفصل 6: التخطيط والمصادقة قبل النشر

**📚 الدورة**: [AZD للمبتدئين](../../README.md) | **⏱️ المدة**: ساعة واحدة | **⭐ التعقيد**: متوسط

---

## النظرة العامة

يغطي هذا الفصل الخطوات الأساسية للتخطيط والمصادقة قبل نشر تطبيقك. تعلم كيفية تجنب الأخطاء المكلفة من خلال التخطيط السليم للسعة، واختيار SKU، وفحوصات ما قبل الطيران.

> تم التحقق بواسطة `azd 1.27.1` في يوليو 2026.

## أهداف التعلم

بعد إكمال هذا الفصل، ستتمكن من:
- تشغيل فحوصات ما قبل الطيران قبل النشر
- تخطيط السعة وتقدير متطلبات الموارد
- اختيار SKU المناسبة لتحسين التكلفة
- تكوين Application Insights للمراقبة
- فهم أنماط تنسيق الفريق

## 🚀 بدء سريع

```bash
# تحقق من حصص الاشتراك
az vm list-usage --location eastus --output table

# معاينة النشر (لا يتم إنشاء موارد)
azd provision --preview

# التحقق من صحة بناء جملة Bicep
az bicep build --file infra/main.bicep

# التحقق من تكوين البيئة
azd env get-values
```

---

## ☑️ قائمة التحقق قبل النشر

### قبل `azd provision`

- [ ] تم التحقق من الحصة للمنطقة
- [ ] تم اختيار SKU بشكل مناسب
- [ ] تمت مراجعة تقدير التكلفة
- [ ] اتساق قواعد التسمية
- [ ] تم تكوين الأمان/إدارة الوصول بناءً على الأدوار

### قبل `azd deploy`

- [ ] تم تعيين المتغيرات البيئية
- [ ] الأسرار موجودة في Key Vault
- [ ] تم التحقق من سلاسل الاتصال
- [ ] تم تكوين فحوصات الصحة
