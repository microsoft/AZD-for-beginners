# الفصل 6: التخطيط والتحقق قبل النشر

**📚 الدورة**: [AZD للمبتدئين](../../README.md) | **⏱️ المدة**: ساعة واحدة | **⭐ التعقيد**: متوسط

---

## نظرة عامة

يغطي هذا الفصل خطوات التخطيط والتحقق الأساسية قبل نشر التطبيق الخاص بك. تعلّم كيف تتجنب الأخطاء المكلفة من خلال التخطيط الصحيح للسعة، واختيار SKU، وفحوص ما قبل التشغيل.

> تم التحقق مقابل `azd 1.23.12` في مارس 2026.

## أهداف التعلم

باستكمال هذا الفصل، سوف:
- إجراء فحوصات ما قبل النشر
- تخطيط السعة وتقدير متطلبات الموارد
- اختيار SKU المناسبة لتحسين التكلفة
- تهيئة Application Insights للمراقبة
- فهم أنماط تنسيق الفريق

## 🚀 بدء سريع

```bash
# التحقق من حصص الاشتراك
az vm list-usage --location eastus --output table

# معاينة النشر (لا يتم إنشاء موارد)
azd provision --preview

# التحقق من صحة بناء جملة Bicep
az bicep build --file infra/main.bicep

# التحقق من تكوين البيئة
azd env get-values
```

---

## ☑️ قائمة التحقق قبل النشر

### قبل `azd provision`

- [ ] التحقق من الحصة المتاحة في المنطقة
- [ ] اختيار SKU المناسبة
- [ ] مراجعة تقدير التكلفة
- [ ] التحقق من اتساق اتفاقية التسمية
- [ ] تكوين الأمان / RBAC

### قبل `azd deploy`

- [ ] ضبط متغيرات البيئة
- [ ] تخزين الأسرار في Key Vault
- [ ] التحقق من سلاسل الاتصال
- [ ] تكوين فحوصات الصحة

|----------|-------------|------------|
