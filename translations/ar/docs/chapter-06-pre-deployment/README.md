# الفصل 6: التخطيط والتحقق قبل النشر

**📚 الدورة**: [AZD للمبتدئين](../../README.md) | **⏱️ المدة**: ساعة واحدة | **⭐ التعقيد**: متوسط

---

## نظرة عامة

يغطي هذا الفصل خطوات التخطيط والتحقق الأساسية قبل نشر تطبيقك. تعلّم كيفية تجنّب الأخطاء المكلفة من خلال التخطيط السليم للسعة، اختيار SKU المناسب، وفحوصات ما قبل الإقلاع.

> تم التحقق مقابل `azd 1.25.6` في يونيو 2026.

## أهداف التعلم

بإكمال هذا الفصل، ستتمكن من:
- تشغيل فحوصات ما قبل الإقلاع قبل النشر
- تخطيط السعة وتقدير متطلبات الموارد
- اختيار SKUs المناسبة لتحسين التكلفة
- تهيئة Application Insights للمراقبة
- فهم أنماط تنسيق الفريق

## 🚀 بدء سريع

```bash
# تحقق من حصص الاشتراك
az vm list-usage --location eastus --output table

# معاينة النشر (لن يتم إنشاء موارد)
azd provision --preview

# التحقق من صحة بناء جملة Bicep
az bicep build --file infra/main.bicep

# تحقق من تكوين البيئة
azd env get-values
```

---

## ☑️ قائمة التحقق قبل النشر

### قبل `azd provision`

- [ ] التحقق من الحصة للمنطقة
- [ ] اختيار SKUs بشكل مناسب
- [ ] مراجعة تقدير التكلفة
- [ ] اتساق قواعد التسمية
- [ ] تكوين الأمان/RBAC

### قبل `azd deploy`

- [ ] تعيين متغيرات البيئة
- [ ] الأسرار في Key Vault
- [ ] التحقق من سلاسل الاتصال
- [ ] تهيئة فحوصات الصحة

## 💰 دليل اختيار SKU

| عبء العمل | التطوير | الإنتاج |
|----------|-------------|------------|
| تطبيقات الحاويات | Consumption | Dedicated D4 |
| App Service | B1/B2 | P1v3+ |
| نماذج Microsoft Foundry | Standard | Standard + PTU |
| بحث الذكاء الاصطناعي | Basic | Standard S2+ |
