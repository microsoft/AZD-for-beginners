# الفصل 3: التكوين والمصادقة

**📚 الدورة**: [AZD للمبتدئين](../../README.md) | **⏱️ المدة**: 45-60 دقيقة | **⭐ التعقيد**: متوسط

---

## نظرة عامة

يغطي هذا الفصل تكوين البيئة، أنماط المصادقة، وأفضل ممارسات الأمان لنشر Azure Developer CLI.

> تم التحقق من الصحة باستخدام `azd 1.27.1` في يوليو 2026.

## أهداف التعلم

من خلال إتمام هذا الفصل، ستتمكن من:
- إتقان تسلسل التكوين في AZD
- إدارة بيئات متعددة (تطوير، اختبار، إنتاج)
- تنفيذ مصادقة آمنة باستخدام الهويات المدارَة
- تكوين الإعدادات الخاصة بكل بيئة

## 🚀 البداية السريعة

```bash
# إنشاء بيئات متعددة
azd env new dev
azd env new staging
azd env new prod

# تبديل بين البيئات
azd env select prod

# تعيين متغيرات البيئة
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# عرض التكوين
azd env get-values
```

---

## 🔧 تسلسل التكوين

يطبق AZD الإعدادات بهذا الترتيب (الأخير يبطل السابق):

1. **القيم الافتراضية** (المضمنة في القوالب)
2. **azure.yaml** (تكوين المشروع)
3. **متغيرات البيئة** (`azd env set`)
4. **معلمات سطر الأوامر** (`--location eastus`)

---

## 🔐 أفضل ممارسات الأمان

```bash
# استخدم الهوية المُدارة (موصى بها)
azd env set AZURE_USE_MANAGED_IDENTITY true

# تحقق من حالة المصادقة في AZD
azd auth status

# اختياري: تحقق من سياق Azure CLI إذا كنت تخطط لتشغيل أوامر az
az account show

# أعد المصادقة إذا لزم الأمر
azd auth login

# اختياري: حدث مصادقة Azure CLI لأوامر az
az login
```
