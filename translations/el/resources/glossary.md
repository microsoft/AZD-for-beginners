# Γλωσσάριο - Ορολογία Azure Developer CLI

## Εισαγωγή

Αυτό το εκτενές γλωσσάριο παρέχει ορισμούς για όρους, έννοιες και ακρωνύμια που χρησιμοποιούνται στο Azure Developer CLI και την ανάπτυξη στο Azure cloud. Απαραίτητη αναφορά για την κατανόηση τεχνικής τεκμηρίωσης, την επίλυση προβλημάτων και την αποτελεσματική επικοινωνία σχετικά με έργα azd και υπηρεσίες Azure.

## Στόχοι Μάθησης

Χρησιμοποιώντας αυτό το γλωσσάριο, θα:
- Κατανοήσετε βασική ορολογία και έννοιες του Azure Developer CLI
- Εξοικειωθείτε με το λεξιλόγιο και τους τεχνικούς όρους της ανάπτυξης στο Azure cloud
- Ανατρέχετε αποτελεσματικά σε ορολογία Infrastructure as Code και ανάπτυξης
- Κατανοείτε ονόματα υπηρεσιών Azure, ακρωνύμια και τους σκοπούς τους
- Έχετε πρόσβαση σε ορισμούς για την επίλυση προβλημάτων και την αποσφαλμάτωση
- Μάθετε προχωρημένες έννοιες αρχιτεκτονικής και ανάπτυξης στο Azure

## Αποτελέσματα Μάθησης

Με τακτική αναφορά σε αυτό το γλωσσάριο, θα μπορείτε να:
- Επικοινωνείτε αποτελεσματικά χρησιμοποιώντας σωστή ορολογία του Azure Developer CLI
- Κατανοείτε πιο καθαρά τεχνική τεκμηρίωση και μηνύματα σφαλμάτων
- Πλοηγείστε στις υπηρεσίες και τις έννοιες του Azure με αυτοπεποίθηση
- Εντοπίζετε προβλήματα χρησιμοποιώντας κατάλληλη τεχνική ορολογία
- Συμμετέχετε σε συζητήσεις ομάδας με ακριβή τεχνική γλώσσα
- Επεκτείνετε συστηματικά τις γνώσεις σας για την ανάπτυξη στο Azure cloud

## Α

**ARM Template**  
Πρότυπο Azure Resource Manager. Μορφή JSON για Infrastructure as Code που χρησιμοποιείται για τον καθορισμό και την ανάπτυξη πόρων Azure δηλωτικά.

**App Service**  
Η πλατφόρμα ως υπηρεσία (PaaS) του Azure για φιλοξενία εφαρμογών ιστού, REST APIs και mobile backends χωρίς διαχείριση υποδομής.

**Application Insights**  
Η υπηρεσία παρακολούθησης απόδοσης εφαρμογών (APM) του Azure που παρέχει βαθιά ανάλυση της απόδοσης, διαθεσιμότητας και χρήσης εφαρμογών.

**Azure CLI**  
Διεπαφή γραμμής εντολών για τη διαχείριση πόρων Azure. Χρησιμοποιείται από το azd για αυθεντικοποίηση και ορισμένες λειτουργίες.

**Azure Developer CLI (azd)**  
Εργαλείο γραμμής εντολών προσανατολισμένο στους προγραμματιστές που επιταχύνει τη διαδικασία δημιουργίας και ανάπτυξης εφαρμογών στο Azure χρησιμοποιώντας πρότυπα και Infrastructure as Code.

**azure.yaml**  
Το κύριο αρχείο ρυθμίσεων για ένα έργο azd που καθορίζει υπηρεσίες, υποδομή και hooks ανάπτυξης.

**Azure Resource Manager (ARM)**  
Η υπηρεσία ανάπτυξης και διαχείρισης του Azure που παρέχει ένα επίπεδο διαχείρισης για τη δημιουργία, ενημέρωση και διαγραφή πόρων.

## Β

**Bicep**  
Γλώσσα ειδικού τομέα (DSL) που αναπτύχθηκε από τη Microsoft για την ανάπτυξη πόρων Azure. Παρέχει απλούστερη σύνταξη από τα ARM templates ενώ μεταγλωττίζεται σε ARM.

**Build**  
Η διαδικασία μεταγλώττισης πηγαίου κώδικα, εγκατάστασης εξαρτήσεων και προετοιμασίας εφαρμογών για ανάπτυξη.

**Blue-Green Deployment**  
Στρατηγική ανάπτυξης που χρησιμοποιεί δύο πανομοιότυπα περιβάλλοντα παραγωγής (μπλε και πράσινο) για ελαχιστοποίηση του χρόνου διακοπής και του κινδύνου.

## Γ

**Container Apps**  
Η serverless υπηρεσία container του Azure που επιτρέπει την εκτέλεση containerized εφαρμογών χωρίς διαχείριση σύνθετης υποδομής.

**CI/CD**  
Συνεχής Ενσωμάτωση/Συνεχής Ανάπτυξη. Αυτοματοποιημένες πρακτικές για την ενσωμάτωση αλλαγών κώδικα και την ανάπτυξη εφαρμογών.

**Cosmos DB**  
Η παγκοσμίως κατανεμημένη, πολυ-μοντέλου υπηρεσία βάσης δεδομένων του Azure που παρέχει ολοκληρωμένα SLA για throughput, latency, διαθεσιμότητα και συνέπεια.

**Configuration**  
Ρυθμίσεις και παράμετροι που ελέγχουν τη συμπεριφορά εφαρμογών και επιλογές ανάπτυξης.

## Δ

**Deployment**  
Η διαδικασία εγκατάστασης και ρύθμισης εφαρμογών και των εξαρτήσεών τους στην υποδομή στόχου.

**Docker**  
Πλατφόρμα για την ανάπτυξη, αποστολή και εκτέλεση εφαρμογών χρησιμοποιώντας τεχνολογία containerization.

**Dockerfile**  
Αρχείο κειμένου που περιέχει οδηγίες για τη δημιουργία εικόνας container Docker.

## Ε

**Environment**  
Στόχος ανάπτυξης που αντιπροσωπεύει μια συγκεκριμένη περίπτωση της εφαρμογής σας (π.χ., ανάπτυξη, staging, παραγωγή).

**Environment Variables**  
Τιμές ρυθμίσεων αποθηκευμένες ως ζεύγη κλειδιού-τιμής που οι εφαρμογές μπορούν να προσπελάσουν κατά την εκτέλεση.

**Endpoint**  
URL ή διεύθυνση δικτύου όπου μπορεί να προσπελαστεί μια εφαρμογή ή υπηρεσία.

## Φ

**Function App**  
Η serverless υπηρεσία υπολογισμού του Azure που επιτρέπει την εκτέλεση κώδικα βάσει γεγονότων χωρίς διαχείριση υποδομής.

## Γ

**GitHub Actions**  
Πλατφόρμα CI/CD ενσωματωμένη με αποθετήρια GitHub για αυτοματοποίηση ροών εργασίας.

**Git**  
Κατανεμημένο σύστημα ελέγχου εκδόσεων που χρησιμοποιείται για την παρακολούθηση αλλαγών στον πηγαίο κώδικα.

## Χ

**Hooks**  
Προσαρμοσμένα scripts ή εντολές που εκτελούνται σε συγκεκριμένα σημεία κατά τη διάρκεια του κύκλου ζωής ανάπτυξης (preprovision, postprovision, predeploy, postdeploy).

**Host**  
Ο τύπος υπηρεσίας Azure όπου θα αναπτυχθεί μια εφαρμογή (π.χ., appservice, containerapp, function).

## Υ

**Infrastructure as Code (IaC)**  
Πρακτική καθορισμού και διαχείρισης υποδομής μέσω κώδικα αντί για χειροκίνητες διαδικασίες.

**Init**  
Η διαδικασία αρχικοποίησης ενός νέου έργου azd, συνήθως από ένα πρότυπο.

## Ι

**JSON**  
JavaScript Object Notation. Μορφή ανταλλαγής δεδομένων που χρησιμοποιείται συχνά για αρχεία ρυθμίσεων και αποκρίσεις API.

**JWT**  
JSON Web Token. Πρότυπο για ασφαλή μετάδοση πληροφοριών μεταξύ μερών ως αντικείμενο JSON.

## Κ

**Key Vault**  
Η υπηρεσία του Azure για ασφαλή αποθήκευση και διαχείριση μυστικών, κλειδιών και πιστοποιητικών.

**Kusto Query Language (KQL)**  
Γλώσσα ερωτημάτων που χρησιμοποιείται για την ανάλυση δεδομένων στο Azure Monitor, Application Insights και άλλες υπηρεσίες Azure.

## Λ

**Load Balancer**  
Υπηρεσία που διανέμει εισερχόμενη κίνηση δικτύου σε πολλαπλούς διακομιστές ή instances.

**Log Analytics**  
Υπηρεσία του Azure για τη συλλογή, ανάλυση και δράση σε δεδομένα τηλεμετρίας από περιβάλλοντα cloud και on-premises.

## Μ

**Managed Identity**  
Χαρακτηριστικό του Azure που παρέχει στις υπηρεσίες του Azure μια αυτόματα διαχειριζόμενη ταυτότητα για αυθεντικοποίηση σε άλλες υπηρεσίες Azure.

**Microservices**  
Αρχιτεκτονική προσέγγιση όπου οι εφαρμογές κατασκευάζονται ως συλλογή μικρών, ανεξάρτητων υπηρεσιών.

**Monitor**  
Η ενοποιημένη λύση παρακολούθησης του Azure που παρέχει πλήρη παρατηρησιμότητα σε εφαρμογές και υποδομή.

## Ν

**Node.js**  
Runtime JavaScript που βασίζεται στη μηχανή V8 της Chrome για την κατασκευή εφαρμογών server-side.

**npm**  
Διαχειριστής πακέτων για το Node.js που διαχειρίζεται εξαρτήσεις και πακέτα.

## Ο

**Output**  
Τιμές που επιστρέφονται από την ανάπτυξη υποδομής και μπορούν να χρησιμοποιηθούν από εφαρμογές ή άλλους πόρους.

## Π

**Package**  
Η διαδικασία προετοιμασίας κώδικα εφαρμογής και εξαρτήσεων για ανάπτυξη.

**Parameters**  
Τιμές εισόδου που περνούν σε πρότυπα υποδομής για την προσαρμογή των αναπτύξεων.

**PostgreSQL**  
Σύστημα σχεσιακής βάσης δεδομένων ανοιχτού κώδικα που υποστηρίζεται ως διαχειριζόμενη υπηρεσία στο Azure.

**Provisioning**  
Η διαδικασία δημιουργίας και ρύθμισης πόρων Azure που καθορίζονται σε πρότυπα υποδομής.

## Q

**Quota**  
Περιορισμοί στον αριθμό πόρων που μπορούν να δημιουργηθούν σε μια συνδρομή ή περιοχή του Azure.

## R

**Resource Group**  
Λογικό δοχείο για πόρους Azure που μοιράζονται τον ίδιο κύκλο ζωής, δικαιώματα και πολιτικές.

**Resource Token**  
Μοναδική συμβολοσειρά που δημιουργείται από το azd για να διασφαλίσει ότι τα ονόματα πόρων είναι μοναδικά σε όλες τις αναπτύξεις.

**REST API**  
Αρχιτεκτονικό στυλ για το σχεδιασμό δικτυακών εφαρμογών χρησιμοποιώντας μεθόδους HTTP.

**Rollback**  
Διαδικασία επαναφοράς σε προηγούμενη έκδοση μιας εφαρμογής ή ρύθμισης υποδομής.

## Σ

**Service**  
Ένα συστατικό της εφαρμογής σας που ορίζεται στο azure.yaml (π.χ., web frontend, API backend, βάση δεδομένων).

**SKU**  
Μονάδα διατήρησης αποθέματος. Αντιπροσωπεύει διαφορετικά επίπεδα υπηρεσιών ή επίπεδα απόδοσης για πόρους Azure.

**SQL Database**  
Η διαχειριζόμενη υπηρεσία σχεσιακής βάσης δεδομένων του Azure που βασίζεται στον Microsoft SQL Server.

**Static Web Apps**  
Υπηρεσία του Azure για την κατασκευή και ανάπτυξη εφαρμογών ιστού πλήρους stack από αποθετήρια πηγαίου κώδικα.

**Storage Account**  
Υπηρεσία του Azure που παρέχει αποθήκευση cloud για αντικείμενα δεδομένων, συμπεριλαμβανομένων blobs, αρχείων, ουρών και πινάκων.

**Subscription**  
Δοχείο λογαριασμού Azure που περιέχει ομάδες πόρων και πόρους, με συσχετισμένη χρέωση και διαχείριση πρόσβασης.

## Τ

**Template**  
Προκατασκευασμένη δομή έργου που περιέχει κώδικα εφαρμογής, ορισμούς υποδομής και ρυθμίσεις για κοινά σενάρια.

**Terraform**  
Εργαλείο Infrastructure as Code ανοιχτού κώδικα που υποστηρίζει πολλούς παρόχους cloud, συμπεριλαμβανομένου του Azure.

**Traffic Manager**  
Η υπηρεσία DNS-based load balancer του Azure για τη διανομή κίνησης σε παγκόσμιες περιοχές του Azure.

## Υ

**URI**  
Uniform Resource Identifier. Συμβολοσειρά που προσδιορίζει έναν συγκεκριμένο πόρο.

**URL**  
Uniform Resource Locator. Τύπος URI που καθορίζει πού βρίσκεται ένας πόρος και πώς να τον ανακτήσετε.

## V

**Virtual Network (VNet)**  
Θεμελιώδες δομικό στοιχείο για ιδιωτικά δίκτυα στο Azure, παρέχοντας απομόνωση και τμηματοποίηση.

**VS Code**  
Visual Studio Code. Δημοφιλής επεξεργαστής κώδικα με εξαιρετική ενσωμάτωση Azure και azd.

## W

**Webhook**  
HTTP callback που ενεργοποιείται από συγκεκριμένα γεγονότα, συνήθως χρησιμοποιείται σε pipelines CI/CD.

**What-if**  
Χαρακτηριστικό του Azure που δείχνει ποιες αλλαγές θα γίνουν από μια ανάπτυξη χωρίς να την εκτελέσει πραγματικά.

## Y

**YAML**  
YAML Ain't Markup Language. Πρότυπο σειριοποίησης δεδομένων που είναι ευανάγνωστο από ανθρώπους και χρησιμοποιείται για αρχεία ρυθμίσεων όπως το azure.yaml.

## Z

**Zone**  
Φυσικά ξεχωριστές τοποθεσίες μέσα σε μια περιοχή του Azure που παρέχουν πλεονασμό και υψηλή διαθεσιμότητα.

## Όροι Ειδικοί για το Πλαίσιο

### Όροι Ανάπτυξης
- **Hot Reload**: Αυτόματη ενημέρωση εφαρμογών κατά την ανάπτυξη χωρίς επανεκκίνηση
- **Build Pipeline**: Αυτοματοποιημένη διαδικασία για τη δημιουργία και δοκιμή κώδικα
- **Deployment Slot**: Περιβάλλον staging μέσα σε ένα App Service
- **Environment Parity**: Διατήρηση ομοιότητας μεταξύ περιβαλλόντων ανάπτυξης, staging και παραγωγής

### Όροι Ασφαλείας
- **Managed Identity**: Χαρακτηριστικό του Azure που παρέχει αυτόματη διαχείριση διαπιστευτηρίων
- **Key Vault**: Ασφαλής αποθήκευση για μυστικά, κλειδιά και πιστοποιητικά
- **RBAC**: Έλεγχος πρόσβασης βάσει ρόλων για πόρους Azure
- **Network Security Group**: Εικονικό firewall για τον έλεγχο της κίνησης δικτύου
