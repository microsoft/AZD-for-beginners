Με τα πρότυπα AZD και το Azure Developer CLI (`azd`) μπορούμε γρήγορα να ξεκινήσουμε το ταξίδι ανάπτυξης AI με τυποποιημένα αποθετήρια που παρέχουν δείγματα κώδικα, υποδομή και αρχεία ρύθμισης - με τη μορφή ενός έτοιμου για ανάπτυξη _starter_ έργου.

**Αλλά τώρα, πρέπει να κατανοήσουμε τη δομή του έργου και τη βάση κώδικα - και να μπορούμε να προσαρμόσουμε το πρότυπο AZD - χωρίς καμία προηγούμενη εμπειρία ή κατανόηση του AZD!**

## 1. Ενεργοποιήστε το GitHub Copilot

### 1.1 Εγκαταστήστε το GitHub Copilot Chat

Ήρθε η ώρα να εξερευνήσετε το [GitHub Copilot with Agent Mode](https://code.visualstudio.com/docs/copilot/chat/chat-agent-mode). Τώρα, μπορούμε να χρησιμοποιήσουμε φυσική γλώσσα για να περιγράψουμε το έργο μας σε υψηλό επίπεδο και να λάβουμε βοήθεια στην εκτέλεση. Για αυτό το εργαστήριο, θα χρησιμοποιήσουμε το [Copilot Free plan](https://github.com/github-copilot/signup) που έχει μηνιαίο όριο για ολοκληρώσεις και συνομιλίες.

Η επέκταση μπορεί να εγκατασταθεί από το marketplace, και συχνά είναι ήδη διαθέσιμη σε Codespaces ή dev container περιβάλλοντα. _Κάντε κλικ στο `Open Chat` από το μενού του εικονιδίου Copilot - και πληκτρολογήστε ένα prompt όπως `What can you do?`_ - μπορεί να σας ζητηθεί να συνδεθείτε. **Το GitHub Copilot Chat είναι έτοιμο**.

### 1.2. Εγκαταστήστε τους διακομιστές MCP

Για να είναι αποτελεσματική η λειτουργία Agent mode, χρειάζεται πρόσβαση στα σωστά εργαλεία για να βοηθήσει στην ανάκτηση γνώσης ή στη λήψη ενεργειών. Εδώ μπορούν να βοηθήσουν οι διακομιστές MCP. Θα ρυθμίσουμε τους ακόλουθους διακομιστές:

1. [Azure MCP Server](../../../../../workshop/docs/instructions)
1. [Microsoft Docs MCP Server](../../../../../workshop/docs/instructions)

Για να τους ενεργοποιήσετε:

1. Δημιουργήστε ένα αρχείο με όνομα `.vscode/mcp.json` αν δεν υπάρχει
1. Αντιγράψτε τα παρακάτω σε αυτό το αρχείο - και ξεκινήστε τους διακομιστές!
   ```json title=".vscode/mcp.json"
   {
      "servers": {
         "Azure MCP Server": {
            "command": "npx",
            "args": [
            "-y",
            "@azure/mcp@latest",
            "server",
            "start"
            ]
         },
         "microsoft.docs.mcp": {
            "type": "http",
            "url": "https://learn.microsoft.com/api/mcp"
         }
      }
   }
   ```

??? warning "Μπορεί να εμφανιστεί σφάλμα ότι το `npx` δεν είναι εγκατεστημένο (κάντε κλικ για να επεκτείνετε για διόρθωση)"

      Για να το διορθώσετε, ανοίξτε το αρχείο `.devcontainer/devcontainer.json` και προσθέστε αυτή τη γραμμή στην ενότητα features. Στη συνέχεια ξαναχτίστε το container. Πλέον θα έχετε εγκατεστημένο το `npx`.

      ```title="" linenums="0"
         "features": {
            "ghcr.io/devcontainers/features/node:1": {},
            ...
         },
      ```

### 1.3. Δοκιμάστε το GitHub Copilot Chat

**Πρώτα χρησιμοποιήστε `azd auth login` για να πιστοποιηθείτε στο Azure από τη γραμμή εντολών του VS Code. Χρησιμοποιήστε επίσης `az login` μόνο αν σκοπεύετε να εκτελέσετε εντολές Azure CLI απευθείας.**

Πλέον θα πρέπει να μπορείτε να ελέγξετε την κατάσταση της συνδρομής Azure σας και να ζητήσετε πληροφορίες σχετικά με τους αναπτυγμένους πόρους ή τη ρύθμιση. Δοκιμάστε αυτά τα prompts:

1. `Λίστα των ομάδων πόρων Azure μου`
1. `#foundry list my current deployments`

Μπορείτε επίσης να κάνετε ερωτήσεις σχετικά με την τεκμηρίωση του Azure και να λάβετε απαντήσεις βασισμένες στον Microsoft Docs MCP server. Δοκιμάστε αυτά τα prompts:

1. `#microsoft_docs_search What is Azure Developer CLI?`
1. `#microsoft_docs_search Show me a Python tutorial to chat with deployed model`

Ή μπορείτε να ζητήσετε αποσπάσματα κώδικα για να ολοκληρώσετε μια εργασία. Δοκιμάστε αυτό το prompt.

1. `Δώστε μου ένα παράδειγμα κώδικα Python που χρησιμοποιεί AAD για έναν διαδραστικό πελάτη συνομιλίας`

Σε λειτουργία `Ask`, αυτό θα παράσχει κώδικα που μπορείτε να αντιγράψετε και να δοκιμάσετε. Σε λειτουργία `Agent`, αυτό μπορεί να πάει ένα βήμα παραπέρα και να δημιουργήσει τους σχετικούς πόρους για εσάς - συμπεριλαμβανομένων σεναρίων ρύθμισης και τεκμηρίωσης - για να σας βοηθήσει να εκτελέσετε αυτήν την εργασία.

**Τώρα είστε εξοπλισμένοι για να αρχίσετε να εξερευνάτε το αποθετήριο προτύπου**

## 2. Αποδόμηση της Αρχιτεκτονικής

??? prompt "ΖΗΤΗΣΤΕ: Εξηγήστε την αρχιτεκτονική της εφαρμογής στο docs/images/architecture.png σε 1 παράγραφο"

      Αυτή η εφαρμογή είναι μια συνομιλιακή εφαρμογή με δυνατότητες AI που βασίζεται στο Azure και παρουσιάζει μια σύγχρονη αρχιτεκτονική με βάση agents. Η λύση επικεντρώνεται γύρω από ένα Azure Container App που φιλοξενεί τον κύριο κώδικα της εφαρμογής, ο οποίος επεξεργάζεται τις εισροές των χρηστών και δημιουργεί ευφυείς απαντήσεις μέσω ενός AI agent.
      
      Η αρχιτεκτονική αξιοποιεί το Microsoft Foundry Project ως θεμέλιο για τις δυνατότητες AI, συνδεόμενη με τις Azure AI Services που παρέχουν τα υποκείμενα γλωσσικά μοντέλα (όπως το gpt-4.1-mini) και τη λειτουργικότητα agent. Οι αλληλεπιδράσεις των χρηστών ρέουν μέσω ενός frontend βασισμένου σε React σε ένα backend FastAPI που επικοινωνεί με την υπηρεσία AI agent για την παραγωγή συμφραζόμενων απαντήσεων.
      
      Το σύστημα ενσωματώνει δυνατότητες ανάκτησης γνώσης μέσω είτε αναζήτησης αρχείων είτε της υπηρεσίας Azure AI Search, επιτρέποντας στον agent να έχει πρόσβαση και να αναφέρει πληροφορίες από ανεβασμένα έγγραφα. Για επιχειρησιακή αριστεία, η αρχιτεκτονική περιλαμβάνει ολοκληρωμένη παρακολούθηση μέσω του Application Insights και του Log Analytics Workspace για ιχνηλάτηση, καταγραφή και βελτιστοποίηση απόδοσης.
      
      Το Azure Storage παρέχει blob storage για δεδομένα εφαρμογής και μεταφορτώσεις αρχείων, ενώ η Managed Identity εξασφαλίζει ασφαλή πρόσβαση μεταξύ πόρων Azure χωρίς αποθήκευση διαπιστευτηρίων. Ολόκληρη η λύση έχει σχεδιαστεί για κλιμάκωση και διατηρησιμότητα, με την κοντεϊνεροποιημένη εφαρμογή να κλιμακώνεται αυτόματα βάσει ζήτησης ενώ προσφέρονται ενσωματωμένες δυνατότητες ασφαλείας, παρακολούθησης και CI/CD μέσω του οικοσυστήματος διαχειριζόμενων υπηρεσιών του Azure.

### 3.1. Κύρια Αρχιτεκτονική της Εφαρμογής

Αυτό το πρότυπο ακολουθεί ένα **εφαρμογή web πλήρους στοίβας** μοτίβο με:

- **Backend**: Python FastAPI με ενσωμάτωση Azure AI
- **Frontend**: TypeScript/React με σύστημα build Vite
- **Infrastructure**: Πρότυπα Azure Bicep για πόρους cloud
- **Containerization**: Docker για συνεπή ανάπτυξη

### 3.2 Υποδομή ως Κώδικας (bicep)

Το επίπεδο υποδομής χρησιμοποιεί **Azure Bicep** πρότυπα οργανωμένα σε μονάδες:

   - **`main.bicep`**: Συντονίζει όλους τους πόρους Azure
   - **`core/` modules**: Επαναχρησιμοποιήσιμα συστατικά για διάφορες υπηρεσίες
      - Υπηρεσίες AI (Microsoft Foundry Models, AI Search)
      - Φιλοξενία κοντέινερ (Azure Container Apps)
      - Παρακολούθηση (Application Insights, Log Analytics)
      - Ασφάλεια (Key Vault, Managed Identity)

### 3.3 Πηγαίος Κώδικας Εφαρμογής (`src/`)

**Backend API (`src/api/`)**:

- REST API βασισμένο σε FastAPI
- Ενσωμάτωση Foundry Agents
- Διαχείριση ευρετηρίου αναζήτησης για ανάκτηση γνώσης
- Δυνατότητες μεταφόρτωσης και επεξεργασίας αρχείων

**Frontend (`src/frontend/`)**:

- Σύγχρονη SPA με React/TypeScript
- Vite για γρήγορη ανάπτυξη και βελτιστοποιημένα builds
- Διεπαφή συνομιλίας για αλληλεπιδράσεις με τον agent

**Βάση Γνώσης (`src/files/`)**:

- Δείγματα δεδομένων πελατών και προϊόντων
- Δείχνει ανάκτηση γνώσης από αρχεία
- Παραδείγματα σε μορφή JSON και Markdown


### 3.4 DevOps & Αυτοματοποίηση

**Scripts (`scripts/`)**:

- Διασταυρούμενης πλατφόρμας σενάρια PowerShell και Bash
- Επικύρωση και ρύθμιση περιβάλλοντος
- Διαμόρφωση μετά την ανάπτυξη
- Διαχείριση ορίων μοντέλων

**Ενσωμάτωση Azure Developer CLI**:

- `azure.yaml` διαμόρφωση για ροές εργασίας `azd`
- Αυτοματοποιημένη προμήθεια και ανάπτυξη
- Διαχείριση μεταβλητών περιβάλλοντος

### 3.5 Δοκιμές & Διασφάλιση Ποιότητας

**Πλαίσιο Αξιολόγησης (`evals/`)**:

- Αξιολόγηση απόδοσης agents
- Δοκιμή ποιότητας απάντησης σε ερωτήματα
- Αυτοματοποιημένο pipeline αξιολόγησης

**Ασφάλεια AI (`airedteaming/`)**:

- Red team testing για ασφάλεια AI
- Σάρωση για ευπάθειες ασφαλείας
- Πρακτικές υπεύθυνου AI
