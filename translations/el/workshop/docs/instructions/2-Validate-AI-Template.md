## 1. Εισαγωγή

Το [Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) ή `azd` είναι ένα εργαλείο γραμμής εντολών ανοιχτού κώδικα που απλοποιεί τη ροή εργασίας του προγραμματιστή κατά την κατασκευή και ανάπτυξη εφαρμογών στο Azure.

Τα [Πρότυπα AZD](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) είναι τυποποιημένα αποθετήρια που περιλαμβάνουν δείγματα κώδικα εφαρμογών, περιουσιακά στοιχεία _infrastructure-as-code_, και αρχεία ρύθμισης `azd` για μια συνεκτική αρχιτεκτονική λύσης. Η προμήθεια της υποδομής γίνεται τόσο απλή όσο η εντολή `azd provision` - ενώ η χρήση του `azd up` σας επιτρέπει να προμηθεύσετε υποδομή **και** να αναπτύξετε την εφαρμογή με μία κίνηση!

Ως αποτέλεσμα, η εκκίνηση της ανάπτυξης της εφαρμογής σας μπορεί να είναι τόσο απλή όσο η εύρεση του κατάλληλου _AZD Starter template_ που πλησιάζει τις ανάγκες εφαρμογής και υποδομής σας - και στη συνέχεια η προσαρμογή του αποθετηρίου ώστε να ταιριάζει στις απαιτήσεις του σεναρίου σας.

Πριν ξεκινήσουμε, ας βεβαιωθούμε ότι έχετε εγκαταστήσει το Azure Developer CLI.

1. Ανοίξτε ένα τερματικό VS Code και πληκτρολογήστε αυτήν την εντολή:

      ```bash title="" linenums="0"
      azd version
      ```

1. Θα πρέπει να δείτε κάτι σαν αυτό!

      ```bash title="" linenums="0"
      azd version 1.27.1 (commit <current-build>)
      ```

**Τώρα είστε έτοιμοι να επιλέξετε και να αναπτύξετε ένα πρότυπο με το azd**

## 2. Επιλογή Προτύπου

Η πλατφόρμα Microsoft Foundry συνοδεύεται από ένα [σετ προτεινόμενων προτύπων AZD](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) που καλύπτουν δημοφιλή σενάρια λύσεων όπως _αυτοματοποίηση ροής εργασιών πολλών πρακτόρων_ και _πολυμορφική επεξεργασία περιεχομένου_. Μπορείτε επίσης να ανακαλύψετε αυτά τα πρότυπα επισκεπτόμενοι το portal Microsoft Foundry.

1. Επισκεφτείτε το [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. Συνδεθείτε στο portal Microsoft Foundry όταν σας ζητηθεί - θα δείτε κάτι σαν αυτό.

![Pick](../../../../../translated_images/el/01-pick-template.60d2d5fff5ebc374.webp)


Οι επιλογές **Basic** είναι τα αρχικά σας πρότυπα:

1. [ ] [Ξεκινήστε με AI Chat](https://github.com/Azure-Samples/get-started-with-ai-chat) που αναπτύσσει μια βασική εφαρμογή συνομιλίας _με τα δεδομένα σας_ στο Azure Container Apps. Χρησιμοποιήστε το για να εξερευνήσετε ένα βασικό σενάριο chat bot AI.
1. [X] [Ξεκινήστε με AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) που επίσης αναπτύσσει έναν τυπικό AI Agent (με τους Foundry Agents). Χρησιμοποιήστε το για να εξοικειωθείτε με λύσεις agentic AI που περιλαμβάνουν εργαλεία και μοντέλα.

Επισκεφτείτε τον δεύτερο σύνδεσμο σε νέα καρτέλα προγράμματος περιήγησης (ή κάντε κλικ στο `Open in GitHub` για την αντίστοιχη κάρτα). Θα πρέπει να δείτε το αποθετήριο για αυτό το AZD Template. Αφιερώστε λίγο χρόνο να εξετάσετε το README. Η αρχιτεκτονική της εφαρμογής φαίνεται έτσι:

![Arch](../../../../../translated_images/el/architecture.8cec470ec15c65c7.webp)

## 3. Ενεργοποίηση Προτύπου

Ας προσπαθήσουμε να αναπτύξουμε αυτό το πρότυπο και να βεβαιωθούμε ότι είναι έγκυρο. Θα ακολουθήσουμε τις οδηγίες στην ενότητα [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started).

1. Επιλέξτε ένα περιβάλλον εργασίας για το αποθετήριο του προτύπου:

      - **GitHub Codespaces**: Κάντε κλικ [σε αυτόν τον σύνδεσμο](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) και επιβεβαιώστε `Create codespace`
      - **Τοπικό κλωνοποίηση ή dev container**: Κλωνοποιήστε το `Azure-Samples/get-started-with-ai-agents` και ανοίξτε το στο VS Code

1. Περιμένετε μέχρι το τερματικό του VS Code να είναι έτοιμο, στη συνέχεια πληκτρολογήστε την ακόλουθη εντολή:

   ```bash title="" linenums="0"
   azd up
   ```

Ολοκληρώστε τα βήματα ροής εργασίας που θα ενεργοποιηθούν:

1. Θα σας ζητηθεί να συνδεθείτε στο Azure - ακολουθήστε τις οδηγίες για την αυθεντικοποίηση
1. Εισάγετε ένα μοναδικό όνομα περιβάλλοντος για εσάς - π.χ., χρησιμοποίησα `nitya-mshack-azd`
1. Αυτό θα δημιουργήσει έναν φάκελο `.azure/` - θα δείτε έναν υποφάκελο με το όνομα του περιβάλλοντος
1. Θα σας ζητηθεί να επιλέξετε ένα όνομα συνδρομής - επιλέξτε την προεπιλογή
1. Θα σας ζητηθεί μια τοποθεσία - χρησιμοποιήστε `East US 2`

Τώρα, περιμένετε να ολοκληρωθεί η προμήθεια. **Αυτό διαρκεί 10-15 λεπτά**

1. Όταν ολοκληρωθεί, η κονσόλα σας θα εμφανίσει ένα μήνυμα ΕΠΙΤΥΧΙΑΣ όπως το παρακάτω:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. Το Azure Portal σας θα έχει τώρα μια προμηθευμένη ομάδα πόρων με το όνομα του περιβάλλοντος:

      ![Infra](../../../../../translated_images/el/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **Τώρα είστε έτοιμοι να επικυρώσετε την αναπτυγμένη υποδομή και εφαρμογή**.

## 5. Επικύρωση Αντιπροσώπου

Η εφαρμογή Azure Container App αναπτύσσει ένα endpoint που συνδέεται με τον AI Agent που έχει προμηθευτεί στο έργο Microsoft Foundry για αυτό το πρότυπο. Ας ρίξουμε μια ματιά σε τι σημαίνει αυτό.

1. Επιστρέψτε στη σελίδα _Overview_ του Azure Portal για την ομάδα πόρων σας

1. Κάντε κλικ στον πόρο `Microsoft Foundry` στη λίστα

1. Θα δείτε αυτό. Κάντε κλικ στο κουμπί `Go to Microsoft Foundry Portal`.
   ![Foundry](../../../../../translated_images/el/04-view-foundry-project.fb94ca41803f28f3.webp)

1. Θα δείτε τη σελίδα του έργου Foundry για την εφαρμογή AI σας
   ![Project](../../../../../translated_images/el/05-visit-foundry-portal.d734e98135892d7e.webp)

1. Κάντε κλικ στο `Agents` - θα δείτε τον προεπιλεγμένο Agent που προμηθεύτηκε στο έργο σας
   ![Agents](../../../../../translated_images/el/06-visit-agents.bccb263f77b00a09.webp)

1. Επιλέξτε τον - και θα δείτε τις λεπτομέρειες του Agent. Σημειώστε τα παρακάτω:

      - Η `Knowledge` του agent υποδεικνύει ότι έχει ανεβεί 32 αρχεία (για την αναζήτηση αρχείων)

**Επικυρώσατε τη λειτουργία του Agent!**

## 6. Playground του Agent

Ας αποκτήσουμε λίγο περισσότερη διαίσθηση για τις δυνατότητες του Microsoft Foundry, παίρνοντας τον Agent για μια δοκιμή στο Agents Playground.

1. Επιστρέψτε στη σελίδα `Agents` στο Microsoft Foundry - επιλέξτε τον προεπιλεγμένο agent
1. Κάντε κλικ στην επιλογή `Try in Playground` - θα πρέπει να δείτε μια διεπαφή Playground όπως αυτή
1. Ρωτήστε την ίδια ερώτηση: `Ποια προϊόντα έχει η Amanda Perez;`

    ![Data](../../../../../translated_images/el/09-ask-in-playground.a1b93794f78fa676.webp)

Λαμβάνετε την ίδια (ή παρόμοια) απάντηση - αλλά επίσης λαμβάνετε επιπλέον πληροφορίες για να κατανοήσετε την ποιότητα, το κόστος και την απόδοση της εφαρμογής agentic σας. Για παράδειγμα:

1. Σημειώστε ότι η απάντηση αναφέρει τα αρχεία δεδομένων που χρησιμοποιήθηκαν για να "τεκμηριώσουν" την απάντηση
1. Κάντε hovering πάνω από οποιαδήποτε από αυτές τις ετικέτες αρχείων - ταιριάζουν τα δεδομένα με την ερώτηση και την απάντηση που εμφανίζεται;

Επίσης, βλέπετε μια γραμμή _στατιστικών_ κάτω από την απάντηση.

1. Κάντε hovering πάνω από οποιοδήποτε μέτρο - π.χ., Ασφάλεια. Θα δείτε κάτι σαν αυτό
1. Συμφωνεί η εκτιμημένη βαθμολογία με την αντίληψή σας για το επίπεδο ασφάλειας της απάντησης;

      ![Data](../../../../../translated_images/el/10-view-run-info-meter.6cdb89a0eea5531f.webp)

## 7. Ενσωματωμένη Παρατηρησιμότητα

Η παρατηρησιμότητα αφορά την οργάνωση της εφαρμογής σας ώστε να δημιουργεί δεδομένα που μπορούν να χρησιμοποιηθούν για κατανόηση, αποσφαλμάτωση και βελτιστοποίηση των λειτουργιών της. Για να αποκτήσετε μια αίσθηση αυτού:

1. Κάντε κλικ στο κουμπί `View Run Info` - θα δείτε αυτήν την προβολή. Αυτό είναι ένα παράδειγμα [Agent tracing](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) σε δράση. _Μπορείτε επίσης να αποκτήσετε αυτήν την προβολή κάνοντας κλικ στα Thread Logs στο κορυφαίο μενού_.

   - Αποκτήστε μια αίσθηση για τα βήματα εκτέλεσης και τα εργαλεία που χρησιμοποίησε ο agent
   - Κατανοήστε τον συνολικό αριθμό Token (σε σύγκριση με τη χρήση tokens του output) για την απάντηση
   - Κατανοήστε την καθυστέρηση και που ξοδεύεται ο χρόνος στην εκτέλεση

      ![Agent](../../../../../translated_images/el/10-view-run-info.b20ebd75fef6a1cc.webp)

1. Κάντε κλικ στην καρτέλα `Metadata` για να δείτε επιπρόσθετα χαρακτηριστικά της εκτέλεσης, που μπορεί να παρέχουν χρήσιμο πλαίσιο για αποσφαλμάτωση αργότερα.

      ![Agent](../../../../../translated_images/el/11-view-run-info-metadata.7966986122c7c2df.webp)


1. Κάντε κλικ στην καρτέλα `Evaluations` για να δείτε τις αυτο-αξιολογήσεις που έγιναν στην απάντηση του agent. Αυτές περιλαμβάνουν αξιολογήσεις ασφάλειας (π.χ., αυτό-βλάβη) και αξιολογήσεις ειδικές για τον agent (π.χ., επίλυση προθέσεων, τήρηση έργου).

      ![Agent](../../../../../translated_images/el/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. Τέλος, κάντε κλικ στην καρτέλα `Monitoring` στο πλάι του μενού.

      - Επιλέξτε την καρτέλα `Resource usage` στη σελίδα που εμφανίζεται - και δείτε τα μετρικά.
      - Παρακολουθήστε τη χρήση εφαρμογής σε όρους κόστους (tokens) και φορτίου (αιτήσεις).
      - Παρακολουθήστε την καθυστέρηση εφαρμογής μέχρι το πρώτο byte (επεξεργασία εισόδου) και το τελευταίο byte (έξοδο).

      ![Agent](../../../../../translated_images/el/13-monitoring-resources.5148015f7311807f.webp)

## 8. Μεταβλητές Περιβάλλοντος

Έως τώρα, περάσαμε από την ανάπτυξη στο πρόγραμμα περιήγησης - και επικυρώσαμε ότι η υποδομή μας έχει προμηθευτεί και η εφαρμογή λειτουργεί. Όμως για να δουλέψουμε με τον κώδικα εφαρμογής _πρώτα_, πρέπει να ρυθμίσουμε το τοπικό μας περιβάλλον ανάπτυξης με τις σχετικές μεταβλητές που απαιτούνται για την εργασία με αυτούς τους πόρους. Η χρήση του `azd` το καθιστά εύκολο.

1. Το Azure Developer CLI [χρησιμοποιεί μεταβλητές περιβάλλοντος](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) για να αποθηκεύει και να διαχειρίζεται τις ρυθμίσεις διαμόρφωσης για τις αναπτύξεις εφαρμογών.

1. Οι μεταβλητές περιβάλλοντος αποθηκεύονται στο `.azure/<env-name>/.env` - αυτό τις περιορίζει στο περιβάλλον `env-name` που χρησιμοποιείται κατά την ανάπτυξη και σας βοηθά να απομονώσετε περιβάλλοντα μεταξύ διαφορετικών στόχων ανάπτυξης στο ίδιο αποθετήριο.

1. Οι μεταβλητές περιβάλλοντος φορτώνονται αυτόματα από την εντολή `azd` κάθε φορά που εκτελεί μια συγκεκριμένη εντολή (π.χ., `azd up`). Σημειώστε ότι το `azd` δεν διαβάζει αυτόματα μεταβλητές περιβάλλοντος σε επίπεδο λειτουργικού συστήματος (π.χ., ορισμένες στο κέλυφος) - αντίθετα χρησιμοποιήστε `azd set env` και `azd get env` για τη μεταφορά πληροφοριών εντός scripts.


Ας δοκιμάσουμε μερικές εντολές:

1. Λάβετε όλες τις μεταβλητές περιβάλλοντος που ορίζονται για το `azd` σε αυτό το περιβάλλον:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      Θα δείτε κάτι σαν:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. Λάβετε μια συγκεκριμένη τιμή - π.χ., θέλω να μάθω αν ορίσαμε την τιμή `AZURE_AI_AGENT_MODEL_NAME`

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      Θα δείτε κάτι σαν αυτό - δεν είχε οριστεί από προεπιλογή!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. Ορίστε μια νέα μεταβλητή περιβάλλοντος για το `azd`. Εδώ, ενημερώνουμε το όνομα μοντέλου agent. _Σημείωση: οποιεσδήποτε αλλαγές γίνουν θα αντικατοπτρίζονται αμέσως στο αρχείο `.azure/<env-name>/.env`.

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      Τώρα, θα πρέπει να βρούμε ότι η τιμή έχει οριστεί:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. Σημειώστε ότι κάποιοι πόροι είναι επίμονοι (π.χ., αναπτύξεις μοντέλων) και θα χρειαστούν περισσότερο από ένα `azd up` για να υποχρεώσετε την επανανάπτυξη. Ας δοκιμάσουμε να καταστρέψουμε την αρχική ανάπτυξη και να την αναπτύξουμε εκ νέου με τις αλλαγμένες μεταβλητές περιβάλλοντος.

1. **Ανανέωση** Αν είχατε αναπτύξει προηγουμένως υποδομή με χρήση προτύπου azd - μπορείτε να _ανανεώσετε_ την κατάσταση των τοπικών μεταβλητών περιβάλλοντος με βάση την τρέχουσα κατάσταση της ανάπτυξης Azure χρησιμοποιώντας αυτή την εντολή:

      ```bash title="" linenums="0"
      azd env refresh
      ```


      Αυτός είναι ένας ισχυρός τρόπος να _συγχρονίσετε_ μεταβλητές περιβάλλοντος σε δύο ή περισσότερα τοπικά περιβάλλοντα ανάπτυξης (π.χ., ομάδα με πολλούς προγραμματιστές) - επιτρέποντας στην αναπτυγμένη υποδομή να λειτουργεί ως η βασική αλήθεια για την κατάσταση των μεταβλητών env. Τα μέλη της ομάδας απλά _ανανεώνουν_ τις μεταβλητές για να επανέλθουν σε συγχρονισμό.
