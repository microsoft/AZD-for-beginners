## 1. Αναλογία με έναν οικοδόμο

Η δημιουργία μιας σύγχρονης, έτοιμης για επιχειρήσεις εφαρμογής AI _από το μηδέν_ μπορεί να είναι αποθαρρυντική. Είναι λίγο σαν να χτίζετε το νέο σας σπίτι μόνοι σας, τούβλο το τούβλο. Ναι, μπορεί να γίνει! Αλλά δεν είναι ο πιο αποτελεσματικός τρόπος για να επιτύχετε το επιθυμητό αποτέλεσμα! 

Αντίθετα, συχνά ξεκινάμε από ένα υπάρχον _σχέδιο αρχιτεκτονικής_, και συνεργαζόμαστε με έναν αρχιτέκτονα για να το προσαρμόσουμε στις προσωπικές μας απαιτήσεις. Και αυτή είναι ακριβώς η προσέγγιση που πρέπει να ακολουθήσουμε όταν κατασκευάζουμε έξυπνες εφαρμογές. Πρώτα, βρείτε μια καλή αρχιτεκτονική σχεδίασης που ταιριάζει στο πρόβλημα σας. Στη συνέχεια συνεργαστείτε με έναν αρχιτέκτονα λύσεων για να προσαρμόσετε και να αναπτύξετε τη λύση για το συγκεκριμένο σενάριό σας.

Αλλά πού μπορούμε να βρούμε αυτά τα σχέδια αρχιτεκτονικής; Και πώς βρίσκουμε έναν αρχιτέκτονα που είναι πρόθυμος να μας διδάξει πώς να τα προσαρμόζουμε και να τα αναπτύσσουμε μόνοι μας; Σε αυτό το εργαστήριο, απαντάμε σε αυτές τις ερωτήσεις παρουσιάζοντάς σας τρεις τεχνολογίες:

1. [Azure Developer CLI](https://aka.ms/azd) - ένα εργαλείο ανοικτού κώδικα που επιταχύνει την πορεία του προγραμματιστή από την τοπική ανάπτυξη (build) στη διάθεση στο cloud (ship).
1. [Microsoft Foundry Templates](https://ai.azure.com/templates) - τυποποιημένα αποθετήρια ανοικτού κώδικα που περιέχουν δείγματα κώδικα, αρχεία υποδομής και αρχεία διαμόρφωσης για την ανάπτυξη μιας αρχιτεκτονικής λύσης AI.
1. [GitHub Copilot Agent Mode](https://code.visualstudio.com/docs/copilot/chat/chat-agent-mode) - ένας πράκτορας κωδικοποίησης βασισμένος στη γνώση του Azure, που μπορεί να μας καθοδηγήσει στην περιήγηση του κώδικα και στη διενέργεια αλλαγών - χρησιμοποιώντας φυσική γλώσσα.

Με αυτά τα εργαλεία στα χέρια μας, μπορούμε τώρα να _εντοπίσουμε_ το σωστό πρότυπο, να το _αναπτύξουμε_ για να επικυρώσουμε ότι λειτουργεί, και να το _προσαρμόσουμε_ ώστε να ταιριάζει στα δικά μας σενάρια. Ας βουτήξουμε και ας μάθουμε πώς λειτουργούν όλα αυτά.

## 2. Azure Developer CLI

Το [Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) (ή `azd`) είναι ένα εργαλείο γραμμής εντολών ανοικτού κώδικα που μπορεί να επιταχύνει το ταξίδι σας από τον κώδικα στο cloud με ένα σύνολο εντολών φιλικών προς τον προγραμματιστή που λειτουργούν με συνέπεια στο IDE (development) και στα περιβάλλοντα CI/CD (devops).

Με το `azd`, το ταξίδι ανάπτυξής σας μπορεί να είναι τόσο απλό όσο:

- `azd init` - Αρχικοποιεί ένα νέο έργο AI από ένα υπάρχον πρότυπο AZD.
- `azd up` - Δημιουργεί την υποδομή και αναπτύσσει την εφαρμογή σας σε ένα βήμα.
- `azd monitor` - Παρέχει παρακολούθηση σε πραγματικό χρόνο και διαγνωστικά για την εφαρμογή σας.
- `azd pipeline config` - Διαμορφώνει CI/CD pipelines για αυτοματοποίηση της ανάπτυξης στο Azure.

**🎯 | ΑΣΚΗΣΗ**: <br/> Εξερευνήστε το εργαλείο γραμμής εντολών `azd` στο τρέχον περιβάλλον εργαστηρίου σας τώρα. Αυτό μπορεί να είναι GitHub Codespaces, ένα dev container, ή ένα τοπικό clone με τις απαραίτητες προϋποθέσεις εγκατεστημένες. Ξεκινήστε πληκτρολογώντας αυτήν την εντολή για να δείτε τι μπορεί να κάνει το εργαλείο:

```bash title="" linenums="0"
azd help
```

## 3. Το Πρότυπο AZD

Για να το πετύχει αυτό, το `azd` χρειάζεται να γνωρίζει την υποδομή που θα δημιουργήσει, τις ρυθμίσεις διαμόρφωσης που πρέπει να εφαρμόσει και την εφαρμογή που θα αναπτυχθεί. Εδώ μπαίνουν τα [AZD templates](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/azd-templates?tabs=csharp). 

Τα πρότυπα AZD είναι αποθετήρια ανοικτού κώδικα που συνδυάζουν δείγματα κώδικα με αρχεία υποδομής και διαμόρφωσης που απαιτούνται για την ανάπτυξη της αρχιτεκτονικής λύσης. Χρησιμοποιώντας μια προσέγγιση _Infrastructure-as-Code_ (IaC), επιτρέπουν τον έλεγχο έκδοσης των ορισμών πόρων και των ρυθμίσεων διαμόρφωσης (όπως και του πηγαίου κώδικα της εφαρμογής) - δημιουργώντας επαναχρησιμοποιήσιμα και συνεπή workflows για τους χρήστες του έργου.

Όταν δημιουργείτε ή επαναχρησιμοποιείτε ένα πρότυπο AZD για το _δικό σας_ σενάριο, σκεφτείτε τις εξής ερωτήσεις:

1. Τι κατασκευάζετε; → Υπάρχει κάποιο πρότυπο που έχει αρχικό κώδικα για αυτό το σενάριο;
1. Πώς είναι σχεδιασμένη η λύση σας; → Υπάρχει κάποιο πρότυπο που περιλαμβάνει τους απαραίτητους πόρους;
1. Πώς αναπτύσσεται η λύση σας; → Σκεφτείτε `azd deploy` με hooks πριν/μετά την επεξεργασία!
1. Πώς μπορείτε να το βελτιστοποιήσετε περαιτέρω; → Σκεφτείτε ενσωματωμένη παρακολούθηση και αυτοματοποιημένα pipelines!

## 4. Πρότυπα Εφαρμογών AI

Για εφαρμογές με δυνατότητες AI, η Microsoft παρέχει εξειδικευμένα πρότυπα που περιλαμβάνουν **Microsoft Foundry** και **Foundry Agents**. Αυτά τα πρότυπα επιταχύνουν την πορεία σας προς την κατασκευή έξυπνων, παραγωγικών εφαρμογών.

### Microsoft Foundry & Foundry Agents Templates

Επιλέξτε ένα πρότυπο παρακάτω για ανάπτυξη. Κάθε πρότυπο είναι διαθέσιμο στο [Awesome AZD](https://azure.github.io/awesome-azd/) και μπορεί να αρχικοποιηθεί με μία εντολή.

| Πρότυπο | Περιγραφή | Εντολή ανάπτυξης |
|----------|-------------|----------------|
| **[AI Chat with RAG](https://azure.github.io/awesome-azd/?tags=ai&tags=rag)** | Εφαρμογή συνομιλίας με Retrieval Augmented Generation (RAG) χρησιμοποιώντας Microsoft Foundry | `azd init -t azure-samples/azure-search-openai-demo` |
| **[Foundry Agent Service Starter](https://azure.github.io/awesome-azd/?tags=ai&tags=agents)** | Δημιουργήστε πράκτορες AI με τα Foundry Agents για αυτόνομη εκτέλεση εργασιών | `azd init -t azure-samples/foundry-agent-service-starter` |
| **[Multi-Agent Orchestration](https://azure.github.io/awesome-azd/?tags=ai&tags=agents)** | Συντονίστε πολλούς Foundry Agents για πολύπλοκες ροές εργασίας | `azd init -t azure-samples/multi-agent-orchestration` |
| **[AI Document Intelligence](https://azure.github.io/awesome-azd/?tags=ai&tags=document)** | Εξαγωγή και ανάλυση εγγράφων με μοντέλα Microsoft Foundry | `azd init -t azure-samples/ai-document-processing` |
| **[Conversational AI Bot](https://azure.github.io/awesome-azd/?tags=ai&tags=bot)** | Κατασκευάστε έξυπνα chatbots με ενσωμάτωση Microsoft Foundry | `azd init -t azure-samples/ai-chat-protocol` |
| **[AI Image Generation](https://azure.github.io/awesome-azd/?tags=ai&tags=dalle)** | Δημιουργία εικόνων με DALL-E μέσω Microsoft Foundry | `azd init -t azure-samples/ai-image-generation` |
| **[Semantic Kernel Agent](https://azure.github.io/awesome-azd/?tags=ai&tags=semantic-kernel)** | Πράκτορες AI που χρησιμοποιούν Semantic Kernel με Foundry Agents | `azd init -t azure-samples/semantic-kernel-agent` |
| **[AutoGen Multi-Agent](https://azure.github.io/awesome-azd/?tags=ai&tags=autogen)** | Συστήματα πολλαπλών πρακτόρων χρησιμοποιώντας το πλαίσιο AutoGen | `azd init -t azure-samples/autogen-multi-agent` |

### Γρήγορη Εκκίνηση

1. **Περιηγηθείτε στα πρότυπα**: Επισκεφθείτε [https://azure.github.io/awesome-azd/](https://azure.github.io/awesome-azd/) και φιλτράρετε κατά `AI`, `Agents`, ή `Microsoft Foundry`
2. **Επιλέξτε το πρότυπό σας**: Διαλέξτε ένα που ταιριάζει στην περίπτωσή σας
3. **Αρχικοποίηση**: Εκτελέστε την εντολή `azd init` για το επιλεγμένο πρότυπό σας
4. **Ανάπτυξη**: Εκτελέστε `azd up` για να δημιουργήσετε υποδομή και να αναπτύξετε

**🎯 | ΑΣΚΗΣΗ**: <br/>
Επιλέξτε ένα από τα παραπάνω πρότυπα με βάση το σενάριό σας:

- **Κατασκευάζετε chatbot;** → Ξεκινήστε με **AI Chat with RAG** ή **Conversational AI Bot**
- **Χρειάζεστε αυτόνομους πράκτορες;** → Δοκιμάστε **Foundry Agent Service Starter** ή **Multi-Agent Orchestration**
- **Επεξεργάζεστε έγγραφα;** → Χρησιμοποιήστε **AI Document Intelligence**
- **Θέλετε βοήθεια στην κωδικοποίηση με AI;** → Εξερευνήστε **Semantic Kernel Agent** ή **AutoGen Multi-Agent**

```bash title="Example: Deploy the AI Chat with RAG template" linenums="0"
azd init -t azure-samples/azure-search-openai-demo
azd up
```
