## 🆕 Τι Νέο στο azd Σήμερα

> 📌 Αυτό το μάθημα είναι επικυρωμένο με την έκδοση **`azd 1.27.1`** (Ιούλιος 2026). Εκτελέστε `azd version` για να ελέγξετε την έκδοσή σας και `azd upgrade` για να λάβετε την τελευταία.

Το Azure Developer CLI έχει ξεφύγει από τις παραδοσιακές web εφαρμογές και API. Σήμερα, το azd είναι το μοναδικό εργαλείο για την ανάπτυξη **οποιασδήποτε** εφαρμογής στο Azure—συμπεριλαμβανομένων εφαρμογών με τεχνητή νοημοσύνη και ευφυών πρακτόρων.

Τι σημαίνει αυτό για εσάς:

- **Οι πράκτορες AI είναι πλέον πρώτης τάξης φορτία εργασίας azd.** Μπορείτε να ξεκινήσετε, να αναπτύξετε και να διαχειριστείτε έργα πρακτόρων AI χρησιμοποιώντας την ίδια ροή εργασίας `azd init` → `azd up` που ήδη γνωρίζετε.
- **Ολόκληρος κύκλος ζωής πρακτόρα από το CLI.** Η επέκταση `azure.ai.agents` καλύπτει πλέον όλο το ταξίδι—`azd ai agent init` για δημιουργία, `azd ai agent invoke` για δοκιμή (με έξοδο χρονισμού απόκρισης), `azd ai agent eval generate` και `azd ai agent optimize` για μέτρηση και βελτίωση ποιότητας, και `azd ai agent delete` για καθαρισμό.
- **Περισσότερα δομικά στοιχεία AI.** Νέες προκαταρκτικές επεκτάσεις—`azure.ai.skills` και `azure.ai.connections`—σας επιτρέπουν να διαχειρίζεστε επαναχρησιμοποιούμενες δεξιότητες πρακτόρων και συνδέσεις Foundry απευθείας με το azd.
- **Ενσωμάτωση Microsoft Foundry** φέρνει την ανάπτυξη μοντέλων, τη φιλοξενία πρακτόρων και τη διαμόρφωση υπηρεσιών AI απευθείας στο οικοσύστημα προτύπων azd.
- **Ομαλότερα καθημερινά βασικά.** Πρόσφατες εκδόσεις καθιστούν το `azd init` αμετάβλητο (ασφαλές για επανεκτέλεση), καθαρίζουν αυτόματα παρωχημένα εικονικά διακριτικά με `azd auth login`, και προσθέτουν έναν φιλικό οδηγό πρώτης εκτέλεσης `azd tool`.
- **Η βασική ροή εργασιών δεν έχει αλλάξει.** Είτε αναπτύσσετε μια εφαρμογή todo, μια μικροϋπηρεσία ή μια λύση πολλαπλών πρακτόρων AI, οι εντολές είναι οι ίδιες.

> **Σημείωση για τους χρήστες Aspire:** Η Microsoft αναφέρεται τώρα στο προϊόν απλά ως **Aspire** (προηγουμένως ".NET Aspire"). Η υποστήριξη Aspire στο azd δεν έχει αλλάξει—μόνο το όνομα ενημερώθηκε.

Αν έχετε χρησιμοποιήσει το azd πριν, η υποστήριξη AI είναι μια φυσική επέκταση—όχι ξεχωριστό εργαλείο ή προχωρημένη διαδρομή. Αν ξεκινάτε τώρα, θα μάθετε μια ροή εργασίας που λειτουργεί για τα πάντα.

## 🚀 Τι είναι το Azure Developer CLI (azd);

**Το Azure Developer CLI (azd)** είναι ένα εργαλείο γραμμής εντολών φιλικό προς τον προγραμματιστή που κάνει απλή την ανάπτυξη εφαρμογών στο Azure. Αντί να δημιουργείτε και να συνδέετε χειροκίνητα δεκάδες πόρους Azure, μπορείτε να αναπτύξετε ολόκληρες εφαρμογές με μια εντολή.

### Η Μαγεία του `azd up`

```bash
# Αυτή η εντολή κάνει τα πάντα:
# ✅ Δημιουργεί όλους τους πόρους Azure
# ✅ Διαμορφώνει το δίκτυο και την ασφάλεια
# ✅ Δημιουργεί τον κώδικα της εφαρμογής σας
# ✅ Αναπτύσσει στην Azure
# ✅ Σας δίνει ένα λειτουργικό URL
azd up
```

**Αυτό είναι!** Χωρίς κλικ στο Azure Portal, χωρίς να χρειάζεται να μάθετε πολύπλοκα ARM templates, χωρίς χειροκίνητες ρυθμίσεις — απλώς λειτουργικές εφαρμογές στο Azure.

## ❓ Azure Developer CLI vs Azure CLI: Ποια η διαφορά;

Αυτή είναι η πιο συνηθισμένη ερώτηση που κάνουν οι αρχάριοι. Εδώ είναι η απλή απάντηση:

| Χαρακτηριστικό | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **Σκοπός** | Διαχείριση μεμονωμένων πόρων Azure | Ανάπτυξη ολοκληρωμένων εφαρμογών |
| **Νοοτροπία** | Εστίαση στην υποδομή | Εστίαση στην εφαρμογή |
| **Παράδειγμα** | `az webapp create --name myapp...` | `azd up` |
| **Καμπύλη μάθησης** | Πρέπει να ξέρετε υπηρεσίες Azure | Αρκεί να ξέρετε την εφαρμογή σας |
| **Καλύτερο για** | DevOps, Υποδομή | Προγραμματιστές, Πρωτότυπα |

### Απλή Αναλογία

- **Το Azure CLI** είναι σαν να έχετε όλα τα εργαλεία για να χτίσετε ένα σπίτι - σφυριά, πριόνια, καρφιά. Μπορείτε να φτιάξετε οτιδήποτε, αλλά πρέπει να ξέρετε οικοδομική.
- **Το Azure Developer CLI** είναι σαν να προσλαμβάνετε μια εργοληπτική εταιρεία - περιγράφετε τι θέλετε, και αυτοί αναλαμβάνουν την κατασκευή.

### Πότε να χρησιμοποιήσετε το καθένα

| Σενάριο | Χρησιμοποιήστε Αυτό |
|----------|----------|
| "Θέλω να αναπτύξω γρήγορα την web εφαρμογή μου" | `azd up` |
| "Χρειάζομαι να δημιουργήσω μόνο έναν λογαριασμό αποθήκευσης" | `az storage account create` |
| "Κατασκευάζω μια πλήρη εφαρμογή AI" | `azd init --template azure-search-openai-demo` |
| "Πρέπει να αποσφαλματώσω έναν συγκεκριμένο πόρο Azure" | `az resource show` |
| "Θέλω ανάπτυξη παραγωγής σε λίγα λεπτά" | `azd up --environment production` |

### Συνεργάζονται!

Το AZD χρησιμοποιεί το Azure CLI εσωτερικά. Μπορείτε να χρησιμοποιήσετε και τα δύο:
```bash
# Αναπτύξτε την εφαρμογή σας με το AZD
azd up

# Στη συνέχεια, βελτιώστε συγκεκριμένους πόρους με το Azure CLI
az webapp config set --name myapp --always-on true
```

## 🎯 Ξεκινώντας σε 3 Βήματα

Πριν ξεκινήσετε, βεβαιωθείτε ότι το μηχάνημά σας είναι έτοιμο για το πρότυπο που θέλετε να αναπτύξετε:

**Windows:**
```powershell
.\validate-setup.ps1
```

**macOS / Linux:**
```bash
bash ./validate-setup.sh
```

Αν κάποιος απαιτούμενος έλεγχος αποτύχει, διορθώστε τον πρώτα και συνεχίστε με τον γρήγορο οδηγό.

### Βήμα 1: Εγκαταστήστε το AZD (2 λεπτά)

**Windows:**
```powershell
winget install microsoft.azd
```

**macOS:**
```bash
brew tap azure/azd && brew install azd
```

**Linux:**
```bash
curl -fsSL https://aka.ms/install-azd.sh | bash
```

### Βήμα 2: Πιστοποιηθείτε για το AZD

```bash
# Προαιρετικό αν σκοπεύετε να χρησιμοποιήσετε εντολές Azure CLI απευθείας σε αυτό το μάθημα
az login

# Απαραίτητο για ροές εργασίας AZD
azd auth login
```

Αν δεν είστε σίγουροι ποιο χρειάζεστε, ακολουθήστε ολόκληρη τη ροή ρύθμισης στο [Installation & Setup](docs/chapter-01-foundation/installation.md#authentication-setup).

### Βήμα 3: Αναπτύξτε την Πρώτη σας Εφαρμογή

```bash
# Αρχικοποίηση από ένα πρότυπο
azd init --template todo-nodejs-mongo

# Ανάπτυξη στο Azure (δημιουργεί τα πάντα!)
azd up
```

**🎉 Αυτό είναι!** Η εφαρμογή σας είναι πλέον ζωντανή στο Azure.

### Καθαρισμός (Μην το Ξεχάσετε!)

```bash
# Αφαιρέστε όλους τους πόρους όταν ολοκληρώσετε τα πειράματα
azd down --force --purge
```

## Πριν Ξεκινήσετε: Λογαριασμοί, Πρόσβαση και Προϋποθέσεις

- **Προαπαιτούμενα για το runtime προτύπων**: Κάποια πρότυπα χρειάζονται τοπικά εργαλεία όπως Node.js, Python, Java ή Docker. Εκτελέστε τον έλεγχο εγκατάστασης πριν ξεκινήσετε για να εντοπίσετε έγκαιρα τυχόν ελλείψεις εργαλείων.
- **Βασική εξοικείωση με το τερματικό**: Δεν χρειάζεται να είστε ειδικός, αλλά θα πρέπει να νιώθετε άνετα να εκτελείτε εντολές όπως `git clone`, `azd auth login` και `azd up`.

azd init --template azure-search-openai-demo

# Δοκιμάστε επιπλέον προσαρμογές τεχνητής νοημοσύνης
azd init --template openai-chat-app-quickstart

**✅ Επιβεβαίωση Επιτυχίας:**
```bash
# Μετά το Κεφάλαιο 2, θα πρέπει να μπορείτε να:
azd init --template azure-search-openai-demo
azd up
# Δοκιμάσετε τη διασύνδεση συνομιλίας AI
# Κάνετε ερωτήσεις και λαμβάνετε απαντήσεις με τεχνητή νοημοσύνη με πηγές
# Επαληθεύσετε ότι η ενσωμάτωση αναζήτησης λειτουργεί
azd monitor  # Ελέγξετε ότι το Application Insights εμφανίζει τηλεμετρία
azd down --force --purge
```

**📊 Επένδυση Χρόνου:** 1-2 ώρες  
**📈 Επίπεδο Δεξιοτήτων Μετά:** Μπορεί να αναπτύσσει και ρυθμίζει εφαρμογές AI έτοιμες για παραγωγή  
**💰 Ενημέρωση Κόστους:** Κατανόηση κόστους ανάπτυξης $80-150/μήνα, κόστους παραγωγής $300-3500/μήνα

#### 💰 Εκτιμήσεις Κόστους για AI Αναπτύξεις

**Περιβάλλον Ανάπτυξης (Εκτιμώμενο $80-150/μήνα):**
- Μοντέλα Microsoft Foundry (Πληρωμή με χρήση): $0-50/μήνα (ανάλογα με χρήση tokens)
- AI Search (Βασικό επίπεδο): $75/μήνα
- Container Apps (Κατανάλωση): $0-20/μήνα
- Αποθηκευτικός χώρος (Standard): $1-5/μήνα

**Περιβάλλον Παραγωγής (Εκτιμώμενο $300-3,500+/μήνα):**
- Μοντέλα Microsoft Foundry (PTU για συνεπή απόδοση): $3,000+/μήνα Ή Πληρωμή με χρήση και υψηλό όγκο
- AI Search (Κανονικό επίπεδο): $250/μήνα
- Container Apps (Αφιερωμένα): $50-100/μήνα
- Application Insights: $5-50/μήνα
- Αποθηκευτικός χώρος (Premium): $10-50/μήνα

**💡 Συμβουλές Βελτιστοποίησης Κόστους:**
- Χρησιμοποιήστε **Δωρεάν Επίπεδα** Μοντέλων Microsoft Foundry για μάθηση (περιλαμβάνονται 50.000 tokens/μήνα Azure OpenAI)
- Εκτελέστε `azd down` για να αποδεσμεύσετε πόρους όταν δεν αναπτύσσετε ενεργά
- Ξεκινήστε με χρέωση κατανάλωσης, αναβαθμίστε σε PTU μόνο για παραγωγή
- Χρησιμοποιήστε `azd provision --preview` για να εκτιμήσετε το κόστος πριν την ανάπτυξη
- Ενεργοποιήστε την αυτόματη κλιμάκωση: πληρώστε μόνο για την πραγματική χρήση

**Παρακολούθηση Κόστους:**
```bash
# Ελέγξτε τα εκτιμώμενα μηνιαία κόστη
azd provision --preview

# Παρακολουθήστε τα πραγματικά κόστη στο Azure Portal
az consumption budget list --resource-group <your-rg>
```

Πέρα από τα βασικά, το AZD παρέχει ισχυρά χαρακτηριστικά για παραγωγικές αναπτύξεις:

**Γιατί AZD για λύσεις AI;** Το AZD αντιμετωπίζει τις κορυφαίες προκλήσεις που έχουν οι προγραμματιστές AI:

- **Πρότυπα έτοιμα για AI** - Προδιαμορφωμένα πρότυπα για Microsoft Foundry Models, Azure AI Services και ML workloads
- **Ασφαλείς αναπτύξεις AI** - Ενσωματωμένα πρότυπα ασφάλειας για υπηρεσίες AI, κλειδιά API και τελικά σημεία μοντέλων  
- **Παραγωγικά πρότυπα AI** - Καλύτερες πρακτικές για κλιμακούμενες, οικονομικές αναπτύξεις εφαρμογών AI
- **Ολοκληρωμένες ροές εργασίας AI** - Από την ανάπτυξη μοντέλου μέχρι την παραγωγική ανάπτυξη με σωστή παρακολούθηση
- **Βελτιστοποίηση κόστους** - Έξυπνες στρατηγικές κατανομής και κλιμάκωσης πόρων για workloads AI
- **Ενσωμάτωση Microsoft Foundry** - Απρόσκοπτη σύνδεση με τον κατάλογο μοντέλων και τελικά σημεία του Microsoft Foundry

</details>

<details>
<summary><strong>❌ "azd up" αποτυγχάνει στη μέση της διαδικασίας</strong></summary>

```bash
# Επιλογή 1: Καθαρίστε και δοκιμάστε ξανά
azd down --force --purge
azd up

# Επιλογή 2: Απλώς διορθώστε τη δομή υποδομής
azd provision

# Επιλογή 3: Ελέγξτε την αναλυτική κατάσταση
azd show

# Επιλογή 4: Ελέγξτε τα αρχεία καταγραφής στο Azure Monitor
azd monitor --logs
```
</details>

<details>
<summary><strong>❌ "Authentication failed" ή "Token expired"</strong></summary>

```bash
# Επαναπιστοποίηση για το AZD
azd auth logout
azd auth login

# Προαιρετικό: ανανεώστε επίσης το Azure CLI αν εκτελείτε εντολές az
az logout
az login

# Επαληθεύστε την αυθεντικοποίηση
az account show
```
</details>

<details>
<summary><strong>❌ "Resource already exists" ή συγκρούσεις ονομάτων</strong></summary>

```bash
# Το AZD δημιουργεί μοναδικά ονόματα, αλλά σε περίπτωση σύγκρουσης:
azd down --force --purge

# Στη συνέχεια, δοκιμάστε ξανά με νέο περιβάλλον
azd env new dev-v2
azd up
```
</details>

<details>
<summary><strong>❌ Η ανάπτυξη προτύπου διαρκεί πολύ</strong></summary>

**Φυσιολογικοί χρόνοι αναμονής:**  
- Απλή web εφαρμογή: 5-10 λεπτά  
- Εφαρμογή με βάση δεδομένων: 10-15 λεπτά  
- Εφαρμογές AI: 15-25 λεπτά (η προμήθεια OpenAI είναι αργή)  

```bash
# Έλεγχος προόδου
azd show

# Αν κολλήσει >30 λεπτά, ελέγξτε το Azure Portal:
azd monitor --overview
# Αναζητήστε αποτυχημένες αναπτύξεις
```
</details>

<details>
<summary><strong>❌ "Permission denied" ή "Forbidden"</strong></summary>

```bash
# Ελέγξτε τον ρόλο σας στο Azure
az role assignment list --assignee $(az account show --query user.name -o tsv)

# Χρειάζεστε τουλάχιστον τον ρόλο "Συνεισφέροντα"
# Ζητήστε από τον διαχειριστή Azure να παραχωρήσει:
# - Συνεισφέροντα (για πόρους)
# - Διαχειριστή Πρόσβασης Χρηστών (για αναθέσεις ρόλων)
```
</details>

<details>
<summary><strong>❌ Δεν μπορεί να βρεθεί το URL της αναπτυγμένης εφαρμογής</strong></summary>

```bash
# Εμφάνιση όλων των τελικών σημείων υπηρεσίας
azd show

# Ή άνοιγμα του Azure Portal
azd monitor

# Έλεγχος συγκεκριμένης υπηρεσίας
azd env get-values
# Αναζήτηση μεταβλητών *_URL
```
</details>
