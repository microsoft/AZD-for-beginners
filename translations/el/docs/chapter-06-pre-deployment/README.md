# Κεφάλαιο 6: Σχεδιασμός και Επαλήθευση πριν την Ανάπτυξη

**📚 Μάθημα**: [AZD Για Αρχάριους](../../README.md) | **⏱️ Διάρκεια**: 1 ώρα | **⭐ Πολυπλοκότητα**: Ενδιάμεσο

---

## Επισκόπηση

Αυτό το κεφάλαιο καλύπτει βασικά βήματα σχεδιασμού και επαλήθευσης πριν την ανάπτυξη της εφαρμογής σας. Μάθετε πώς να αποφεύγετε δαπανηρά λάθη με σωστό σχεδιασμό χωρητικότητας, επιλογή SKU και ελέγχους προετοιμασίας.

> Επαληθεύτηκε με `azd 1.23.12` τον Μάρτιο 2026.

## Μαθησιακοί Στόχοι

By completing this chapter, you will:
- Εκτελέστε ελέγχους προετοιμασίας πριν την ανάπτυξη
- Σχεδιάστε τη χωρητικότητα και εκτιμήστε τις απαιτήσεις πόρων
- Επιλέξτε κατάλληλα SKU για βελτιστοποίηση κόστους
- Διαμορφώστε το Application Insights για παρακολούθηση
- Κατανοήστε τα πρότυπα συντονισμού ομάδας

## 🚀 Γρήγορη Εκκίνηση

```bash
# Έλεγχος ορίων συνδρομής
az vm list-usage --location eastus --output table

# Προεπισκόπηση ανάπτυξης (δεν δημιουργούνται πόροι)
azd provision --preview

# Επικύρωση σύνταξης Bicep
az bicep build --file infra/main.bicep

# Έλεγχος διαμόρφωσης περιβάλλοντος
azd env get-values
```

## ☑️ Λίστα Ελέγχου πριν την Ανάπτυξη

### Πριν από το `azd provision`

- [ ] Επαληθεύτηκε το όριο για την περιοχή
- [ ] Επιλέχθηκαν τα SKU κατάλληλα
- [ ] Ελέγχθηκε η εκτίμηση κόστους
- [ ] Συνεπής σύμβαση ονοματοδοσίας
- [ ] Ρυθμίστηκε η ασφάλεια/RBAC

### Πριν από το `azd deploy`

- [ ] Ορίστηκαν οι μεταβλητές περιβάλλοντος
- [ ] Τα μυστικά στο Key Vault
- [ ] Επαληθεύτηκαν οι συμβολοσειρές σύνδεσης
- [ ] Ρυθμίστηκαν οι έλεγχοι υγείας
