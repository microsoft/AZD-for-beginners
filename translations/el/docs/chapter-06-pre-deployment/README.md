**📚 Μάθημα**: [AZD For Beginners](../../README.md) | **⏱️ Διάρκεια**: 1 ώρα | **⭐ Πολυπλοκότητα**: Ενδιάμεση

## Επισκόπηση

Αυτό το κεφάλαιο καλύπτει τα απαραίτητα βήματα σχεδιασμού και επικύρωσης πριν την ανάπτυξη της εφαρμογής σας. Μάθετε πώς να αποφεύγετε δαπανηρά λάθη με σωστό σχεδιασμό χωρητικότητας, επιλογή SKU και προελέγχους.

> Επικυρώθηκε με `azd 1.25.6` τον Ιούνιο 2026.

## Στόχοι Μάθησης

Με την ολοκλήρωση αυτού του κεφαλαίου, θα:
- Εκτελείτε προελέγχους πριν την ανάπτυξη
- Σχεδιάζετε τη χωρητικότητα και εκτιμάτε τις απαιτήσεις πόρων
- Επιλέγετε κατάλληλα SKUs για βελτιστοποίηση κόστους
- Ρυθμίζετε το Application Insights για παρακολούθηση
- Κατανοείτε τα πρότυπα συντονισμού ομάδων

## 🚀 Γρήγορη Εκκίνηση

```bash
# Ελέγξτε τα όρια συνδρομής
az vm list-usage --location eastus --output table

# Προεπισκόπηση ανάπτυξης (δεν δημιουργούνται πόροι)
azd provision --preview

# Επαληθεύστε τη σύνταξη Bicep
az bicep build --file infra/main.bicep

# Ελέγξτε τη διαμόρφωση του περιβάλλοντος
azd env get-values
```

## ☑️ Λίστα Ελέγχου πριν την Ανάπτυξη

### Πριν το `azd provision`

- [ ] Επαληθεύτηκε το όριο για την περιοχή
- [ ] Τα SKUs επιλέχθηκαν κατάλληλα
- [ ] Ελέγχθηκε η εκτίμηση κόστους
- [ ] Συνεπής σύμβαση ονοματοδοσίας
- [ ] Ρυθμίστηκε η ασφάλεια/RBAC

### Πριν το `azd deploy`

- [ ] Ορίστηκαν οι μεταβλητές περιβάλλοντος
- [ ] Μυστικά στο Key Vault
- [ ] Επαληθεύτηκαν οι συμβολοσειρές σύνδεσης
- [ ] Ρυθμίστηκαν οι έλεγχοι υγείας
