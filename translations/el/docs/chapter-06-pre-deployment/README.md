# Κεφάλαιο 6: Σχεδιασμός και Επαλήθευση πριν την Ανάπτυξη

**📚 Μάθημα**: [AZD για Αρχάριους](../../README.md) | **⏱️ Διάρκεια**: 1 ώρα | **⭐ Πολυπλοκότητα**: Ενδιάμεσο

---

## Επισκόπηση

Αυτό το κεφάλαιο καλύπτει βασικά βήματα σχεδιασμού και επαλήθευσης πριν την ανάπτυξη της εφαρμογής σας. Μάθετε να αποφεύγετε δαπανηρά λάθη με σωστό προγραμματισμό χωρητικότητας, επιλογή SKU και προελέγχους.

## Στόχοι Μάθησης

By completing this chapter, you will:
- Εκτελέστε προελέγχους πριν την ανάπτυξη
- Σχεδιάστε τη χωρητικότητα και εκτιμήστε τις απαιτήσεις πόρων
- Επιλέξτε κατάλληλα SKUs για βελτιστοποίηση κόστους
- Διαμορφώστε το Application Insights για παρακολούθηση
- Κατανοήστε τα μοτίβα συνεργασίας της ομάδας

## 🚀 Γρήγορη Εκκίνηση

```bash
# Ελέγξτε τα όρια της συνδρομής
az vm list-usage --location eastus --output table

# Προεπισκόπηση ανάπτυξης (δεν δημιουργούνται πόροι)
azd provision --preview

# Επικύρωση σύνταξης Bicep
az bicep build --file infra/main.bicep

# Ελέγξτε τη διαμόρφωση του περιβάλλοντος
azd env get-values
```

## ☑️ Λίστα Ελέγχου πριν την Ανάπτυξη

### Πριν το `azd provision`

- [ ] Το όριο επαληθεύτηκε για την περιοχή
- [ ] Τα SKUs επιλέχθηκαν κατάλληλα
- [ ] Η εκτίμηση κόστους ελέγχθηκε
- [ ] Η σύμβαση ονοματοδοσίας είναι συνεπής
- [ ] Η ασφάλεια/RBAC ρυθμίστηκε

### Πριν το `azd deploy`

- [ ] Οι μεταβλητές περιβάλλοντος ορίστηκαν
- [ ] Μυστικά στο Key Vault
- [ ] Οι συμβολοσειρές σύνδεσης επαληθεύτηκαν
- [ ] Οι έλεγχοι υγείας ρυθμίστηκαν
