# Κεφάλαιο 6: Προγραμματισμός & Επικύρωση Πριν την Ανάπτυξη

**📚 Μάθημα**: [AZD Για Αρχάριους](../../README.md) | **⏱️ Διάρκεια**: 1 ώρα | **⭐ Πολυπλοκότητα**: Μεσαίο επίπεδο

---

## Επισκόπηση

Αυτό το κεφάλαιο καλύπτει βασικά βήματα προγραμματισμού και επικύρωσης πριν την ανάπτυξη της εφαρμογής σας. Μάθετε πώς να αποφεύγετε δαπανηρά λάθη με σωστό σχεδιασμό χωρητικότητας, επιλογή SKU και ελέγχους προ-ανάπτυξης.

> Επικυρώθηκε με το `azd 1.27.1` τον Ιούλιο του 2026.

## Στόχοι Μάθησης

Ολοκληρώνοντας αυτό το κεφάλαιο, θα:
- Εκτελείτε ελέγχους προ-ανάπτυξης
- Σχεδιάζετε χωρητικότητα και εκτιμάτε τις απαιτήσεις πόρων
- Επιλέγετε κατάλληλα SKUs για βελτιστοποίηση κόστους
- Ρυθμίζετε το Application Insights για παρακολούθηση
- Κατανοείτε πρότυπα συντονισμού ομάδας

## 🚀 Γρήγορη Εκκίνηση

```bash
# Ελέγξτε τα όρια συνδρομής
az vm list-usage --location eastus --output table

# Προεπισκόπηση ανάπτυξης (χωρίς δημιουργία πόρων)
azd provision --preview

# Επικυρώστε τη σύνταξη Bicep
az bicep build --file infra/main.bicep

# Ελέγξτε τη διαμόρφωση περιβάλλοντος
azd env get-values
```

## ☑️ Λίστα Ελέγχου Πριν την Ανάπτυξη

### Πριν το `azd provision`

- [ ] Επαλήθευση ποσόστωσης για την περιοχή
- [ ] Επιλογή SKU κατάλληλα
- [ ] Ανασκόπηση εκτίμησης κόστους
- [ ] Συνεπής ονομασία
- [ ] Ρύθμιση ασφάλειας/RBAC

### Πριν το `azd deploy`

- [ ] Ορισμός μεταβλητών περιβάλλοντος
- [ ] Μυστικά στο Key Vault
- [ ] Επαλήθευση συμβολοσειρών σύνδεσης
- [ ] Ρύθμιση ελέγχων υγείας
