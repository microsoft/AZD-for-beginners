# Κεφάλαιο 6: Σχεδιασμός και Επαλήθευση πριν την Ανάπτυξη

**📚 Μάθημα**: [AZD Για Αρχάριους](../../README.md) | **⏱️ Διάρκεια**: 1 ώρα | **⭐ Δυσκολία**: Μέτρια

---

## Επισκόπηση

Αυτό το κεφάλαιο καλύπτει βασικά βήματα σχεδιασμού και επαλήθευσης πριν την ανάπτυξη της εφαρμογής σας. Μάθετε πώς να αποφύγετε δαπανηρά λάθη με σωστό προγραμματισμό χωρητικότητας, επιλογή SKU και προ-έλεγχους.

## Στόχοι Μάθησης

Με την ολοκλήρωση αυτού του κεφαλαίου, θα:
- Εκτελείτε προ-ελέγχους πριν την ανάπτυξη
- Σχεδιάζετε χωρητικότητα και εκτιμάτε τις απαιτήσεις πόρων
- Επιλέγετε κατάλληλα SKUs για βελτιστοποίηση κόστους
- Διαμορφώνετε το Application Insights για παρακολούθηση
- Κατανοείτε πρότυπα συντονισμού ομάδας

## 🚀 Γρήγορη Εκκίνηση

```bash
# Έλεγχος ορίων συνδρομής
az vm list-usage --location eastus --output table

# Προεπισκόπηση ανάπτυξης (δεν δημιουργούνται πόροι)
azd provision --preview

# Επαλήθευση σύνταξης Bicep
az bicep build --file infra/main.bicep

# Έλεγχος διαμόρφωσης περιβάλλοντος
azd env get-values
```

## ☑️ Λίστα Ελέγχου Πριν την Ανάπτυξη

### Πριν το `azd provision`

- [ ] Επαληθεύτηκε το όριο (quota) για την περιοχή
- [ ] Επιλέχθηκαν κατάλληλα SKUs
- [ ] Επανεξέταση εκτίμησης κόστους
- [ ] Συνεπής σύμβαση ονοματοδοσίας
- [ ] Διαμορφώθηκε ασφάλεια/RBAC

### Πριν το `azd deploy`

- [ ] Ορίστηκαν μεταβλητές περιβάλλοντος
- [ ] Μυστικά στο Key Vault
- [ ] Επαληθεύτηκαν τα connection strings
- [ ] Διαμορφώθηκαν health checks
