## Επισκόπηση

Αυτό το κεφάλαιο καλύπτει προηγμένα πρότυπα αρχιτεκτονικής πολλαπλών πρακτόρων, ορχήστρωση πρακτόρων και αναπτύξεις AI έτοιμες για παραγωγή για σύνθετα σενάρια.

> Επικυρώθηκε με `azd 1.23.12` τον Μάρτιο του 2026.

## Μαθησιακοί Στόχοι

Ολοκληρώνοντας αυτό το κεφάλαιο, θα:
- Κατανοήσετε τα πρότυπα αρχιτεκτονικής πολλαπλών πρακτόρων
- Αναπτύξετε συντονισμένα συστήματα πρακτόρων AI
- Υλοποιήσετε επικοινωνία πράκτορα-προς-πράκτορα
- Δημιουργήσετε λύσεις πολλαπλών πρακτόρων έτοιμες για παραγωγή

## 🚀 Γρήγορη Εκκίνηση

```bash
# Επιλογή 1: Ανάπτυξη από πρότυπο
azd init --template agent-openai-python-prompty
azd up

# Επιλογή 2: Ανάπτυξη από manifest πράκτορα (απαιτεί την επέκταση azure.ai.agents)
azd extension install azure.ai.agents
azd ai agent init -m agent-manifest.yaml
azd up
```

> **Ποια προσέγγιση;** Χρησιμοποιήστε `azd init --template` για να ξεκινήσετε από ένα λειτουργικό δείγμα. Χρησιμοποιήστε `azd ai agent init` όταν έχετε το δικό σας manifest πράκτορα. Δείτε την [Αναφορά CLI του AZD AI](../chapter-08-production/production-ai-practices.md#azd-ai-cli-commands-and-extensions) για πλήρεις λεπτομέρειες.

## 🎯 Επιλεγμένη Λύση: Λιανική Πολλαπλών Πρακτόρων

Η [Λύση Πολλαπλών Πρακτόρων Λιανικής](../../examples/retail-scenario.md) παρουσιάζει:

- **Πράκτορας Πελάτη**: Διαχειρίζεται τις αλληλεπιδράσεις με τον χρήστη και τις προτιμήσεις
- **Πράκτορας Αποθεμάτων**: Διαχειρίζεται το απόθεμα και την επεξεργασία παραγγελιών
- **Ορχηστρωτής**: Συντονίζει μεταξύ των πρακτόρων
- **Κοινή Μνήμη**: Διαχείριση συμφραζομένων μεταξύ πρακτόρων

| Cosmos DB | Κατάσταση πράκτορα και μνήμη |
| Container Apps | Φιλοξενία πρακτόρων |
