## Επισκόπηση

Αυτό το κεφάλαιο σας βοηθά να διαγνώσετε και να επιλύσετε κοινά προβλήματα όταν εργάζεστε με το Azure Developer CLI. Από αποτυχίες ανάπτυξης έως ζητήματα ειδικά για AI.

> Επικυρώθηκε με `azd 1.23.12` τον Μάρτιο του 2026.

## Στόχοι μάθησης

Με την ολοκλήρωση αυτού του κεφαλαίου, θα:
- Διαγνώσετε κοινές αποτυχίες ανάπτυξης AZD
- Εντοπίσετε σφάλματα αυθεντικοποίησης και δικαιωμάτων
- Επιλύσετε προβλήματα σύνδεσης υπηρεσιών AI
- Χρησιμοποιήσετε το Azure Portal και το CLI για αντιμετώπιση προβλημάτων

## 🚨 Γρήγορες Διορθώσεις

### Προβλήματα αυθεντικοποίησης
```bash
# Απαραίτητο για ροές εργασίας AZD
azd auth login

# Προαιρετικό αν χρησιμοποιείτε επίσης απευθείας εντολές του Azure CLI
az login

azd auth status
```

### Αποτυχίες διάθεσης
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### Συγκρούσεις πόρων
```bash
azd down --force --purge
azd env new different-name
azd up
```

### Υπέρβαση ορίου
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 Αναφορά Κωδικών Σφάλματος

| Σφάλμα | Αίτιο | Λύση |
|-------|-------|----------|
| `AuthenticationError` | Δεν έχει γίνει σύνδεση | `azd auth login` |
| `ResourceNotFound` | Απουσία πόρου | Ελέγξτε τα ονόματα πόρων |
| `QuotaExceeded` | Όρια συνδρομής | Ζητήστε αύξηση ορίου |
| `InvalidTemplate` | Σφάλμα σύνταξης Bicep | `az bicep build` |
| `Conflict` | Ο πόρος υπάρχει | Χρησιμοποιήστε νέο όνομα ή διαγράψτε |
| `Forbidden` | Μη επαρκή δικαιώματα | Ελέγξτε ρόλους RBAC |

## 🔄 Επαναφορά και Ανάκτηση

```bash
# Απαλή επαναφορά (διατήρηση πόρων, επανανάπτυξη του κώδικα)
azd deploy --force

# Σκληρή επαναφορά (διαγραφή όλων, ξεκίνημα από την αρχή)
azd down --force --purge
azd up
```
