# Συνεισφορά στο AZD For Beginners

Σας ευχαριστούμε για το ενδιαφέρον σας να συνεισφέρετε στο μάθημα AZD For Beginners! Αυτό το έγγραφο παρέχει οδηγίες και πληροφορίες για τους συνεισφέροντες.

## Πώς να Συνεχίσετε

### Αναφορά Προβλημάτων

Εάν βρείτε σφάλματα, λάθη ή έχετε προτάσεις για βελτιώσεις:

1. **Ελέγξτε τα υπάρχοντα ζητήματα**: Αναζητήστε [υπάρχοντα ζητήματα](https://github.com/microsoft/azd-for-beginners/issues) για να αποφύγετε διπλότυπα
2. **Δημιουργήστε ένα νέο ζήτημα**: Χρησιμοποιήστε το κατάλληλο πρότυπο ζητήματος
3. **Παρέχετε λεπτομέρειες**: Συμπεριλάβετε βήματα για την αναπαραγωγή, την αναμενόμενη συμπεριφορά και στιγμιότυπα οθόνης εάν ισχύει

### Πρόταση Αλλαγών

1. **Fork του αποθετηρίου**: Δημιουργήστε το δικό σας fork του έργου
2. **Δημιουργήστε ένα branch**: Δημιουργήστε ένα feature branch για τις αλλαγές σας
3. **Κάντε αλλαγές**: Ακολουθήστε τα πρότυπα κώδικα και τεκμηρίωσης μας
4. **Δοκιμάστε τις αλλαγές σας**: Επιβεβαιώστε ότι όλοι οι σύνδεσμοι λειτουργούν και το περιεχόμενο αποδίδεται σωστά
5. **Υποβάλετε ένα Pull Request**: Παρέχετε μια σαφή περιγραφή των αλλαγών σας

### Τύποι Συνεισφορών

Καλωσορίζουμε τους παρακάτω τύπους συνεισφορών:

#### Βελτιώσεις Περιεχομένου
- Διορθώστε τυπογραφικά, γραμματικά και ζητήματα μορφοποίησης
- Βελτιώστε την σαφήνεια της υπάρχουσας τεκμηρίωσης
- Ενημερώστε ξεπερασμένες πληροφορίες
- Προσθέστε ελλείπουσες εξηγήσεις ή πλαίσιο

#### Νέα Παραδείγματα
- Προσθέστε σενάρια ανάπτυξης από πραγματικές περιπτώσεις χρήσης
- Δημιουργήστε νέα πρότυπα AZD για κοινές χρήσεις
- Παρέχετε επιπλέον δείγματα κώδικα και ασκήσεις

#### Υποστήριξη Μετάφρασης
- Βοηθήστε στη διατήρηση των υπαρχουσών μεταφράσεων
- Βελτιώστε την ποιότητα των μεταφράσεων
- Προσθέστε μεταφράσεις για νέο περιεχόμενο

#### Αναφορές Σφαλμάτων
- Αναφέρετε σπασμένους συνδέσμους ή εσφαλμένα παραδείγματα κώδικα
- Εντοπίστε ασυνέπειες στην τεκμηρίωση
- Επισημάνετε ζητήματα προσβασιμότητας

## Οδηγίες Συνεισφοράς

### Πρότυπα Τεκμηρίωσης

- Χρησιμοποιήστε σαφή, φιλική προς αρχάριους γλώσσα
- Συμπεριλάβετε μαθησιακούς στόχους και αποτελέσματα
- Παρέχετε βήμα-βήμα οδηγίες με αναμενόμενα αποτελέσματα
- Προσθέστε συνδέσμους πλοήγησης σε σχετικό περιεχόμενο
- Χρησιμοποιήστε συνεπή μορφοποίηση και στυλ markdown

### Παραδείγματα Κώδικα

- Βεβαιωθείτε ότι όλα τα παραδείγματα κώδικα έχουν δοκιμαστεί και λειτουργούν
- Συμπεριλάβετε σχόλια που εξηγούν βασικές έννοιες
- Ακολουθήστε τις βέλτιστες πρακτικές του Azure και του AZD
- Παρέχετε το αναμενόμενο αποτέλεσμα όπου εφαρμόζεται

### Μηνύματα Commit

Χρησιμοποιήστε σαφή, περιγραφικά μηνύματα commit:
- `docs: fix typo in installation guide`
- `feat: add new container app example`
- `fix: correct broken link in chapter 2`
