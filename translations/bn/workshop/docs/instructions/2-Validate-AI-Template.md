## ১. পরিচিতি

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) বা `azd` একটি ওপেন-সোর্স কমান্ডলাইন টুল যা Azure-এ অ্যাপ্লিকেশন তৈরি এবং ডিপ্লয় করার সময় ডেভেলপার ওয়ার্কফ্লোকে সহজ করে তোলে।

[AZD টেমপ্লেটগুলি](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) একটি মানকৃত রিপোজিটরি যা নমুনা অ্যাপ্লিকেশন কোড, _ইনফ্রাস্ট্রাকচার-অ্যাস-কোড_ সম্পদ এবং `azd` কনফিগারেশন ফাইল অন্তর্ভুক্ত করে একটি সমন্বিত সলিউশন আর্কিটেকচারের জন্য। ইনফ্রাস্ট্রাকচার প্রোভিশন করা একটি `azd provision` কমান্ডের মতো সহজ - আর `azd up` ব্যবহার করলে একবারেই ইনফ্রাস্ট্রাকচার **এবং** আপনার অ্যাপ্লিকেশন ডিপ্লয় করা যায়!

ফলে, আপনার অ্যাপ্লিকেশন উন্নয়ন প্রক্রিয়া শুরু করা হতে পারে ঠিক আপনার অ্যাপ্লিকেশন এবং ইনফ্রাস্ট্রাকচার প্রয়োজনের কাছাকাছি _AZD স্টার্টার টেমপ্লেট_ খুঁজে পাওয়া এবং রিপোজিটরি কাস্টমাইজ করার মতো সহজ।

আমরা শুরু করার আগে, নিশ্চিত করুন যে আপনার Azure Developer CLI ইনস্টল করা আছে।

১. একটি VS কোড টার্মিনাল খুলুন এবং এই কমান্ডটি টাইপ করুন:

      ```bash title="" linenums="0"
      azd version
      ```

১. আপনি এইরকম কিছু দেখতে পাবেন!

      ```bash title="" linenums="0"
      azd version 1.27.1 (commit <current-build>)
      ```

**এখন আপনি azd দিয়ে একটি টেমপ্লেট নির্বাচন এবং ডিপ্লয় করার জন্য প্রস্তুত**

## ২. টেমপ্লেট নির্বাচন

Microsoft Foundry প্ল্যাটফর্মের সঙ্গে একটি [নির্দেশিত AZD টেমপ্লেট সেট](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) আছে যা জনপ্রিয় সলিউশন সিনারিও যেমন _মাল্টি-এজেন্ট ওয়ার্কফ্লো অটোমেশন_ এবং _মাল্টি-মোডাল কনটেন্ট প্রসেসিং_ কভার করে। আপনি Microsoft Foundry পোর্টালে গিয়ে এই টেমপ্লেটগুলি আবিষ্কার করতে পারেন।

১. ভিজিট করুন [https://ai.azure.com/templates](https://ai.azure.com/templates)
১. যখন প্রম্পট আসবে তখন Microsoft Foundry পোর্টালে লগইন করুন - আপনি এরকম কিছু দেখতে পাবেন।

![Pick](../../../../../translated_images/bn/01-pick-template.60d2d5fff5ebc374.webp)


**বেসিক** অপশনগুলো আপনার স্টার্টার টেমপ্লেট:

১. [ ] [AI চ্যাট দিয়ে শুরু করুন](https://github.com/Azure-Samples/get-started-with-ai-chat) যা একটি বেসিক চ্যাট অ্যাপ্লিকেশন _আপনার ডেটাসহ_ Azure Container Apps-এ ডিপ্লয় করে। এটি একটি সাধারণ AI চ্যাটবট সিনারিও অন্বেষণ করতে ব্যবহার করুন।
১. [X] [AI এজেন্ট দিয়ে শুরু করুন](https://github.com/Azure-Samples/get-started-with-ai-agents) যা Foundry এজেন্ট সহ একটি স্ট্যান্ডার্ড AI এজেন্ট ডিপ্লয় করে। এটি টুল এবং মডেল সংবলিত এজেন্টিক AI সলিউশন সম্পর্কে পরিচিত হতে ব্যবহার করুন।

দ্বিতীয় লিংকটি নতুন ব্রাউজার ট্যাবে খুলুন (অথবা সম্পর্কিত কার্ডে `Open in GitHub` এ ক্লিক করুন)। আপনি এই AZD টেমপ্লেটটির রিপোজিটরি দেখতে পাবেন। README দ্রুত দেখুন। অ্যাপ্লিকেশন আর্কিটেকচার এমন দেখায়:

## ৩. টেমপ্লেট সক্রিয়করণ

এই টেমপ্লেটটি ডিপ্লয় করার চেষ্টা করি এবং যাচাই করি এটি সঠিক কিনা। আমরা [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) সেকশনের নির্দেশনা অনুসরণ করব।

১. টেমপ্লেট রিপোজিটরির জন্য একটি কাজের পরিবেশ নির্বাচন করুন:

      - **GitHub Codespaces**: এই [লিঙ্কে ক্লিক করুন](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) এবং `Create codespace` নিশ্চিত করুন
      - **লোকাল ক্লোন বা ডেভ কন্টেইনার**: `Azure-Samples/get-started-with-ai-agents` ক্লোন করুন এবং এটি VS কোডে খুলুন

১. VS কোড টার্মিনাল যখন প্রস্তুত হবে, তখন নিম্নলিখিত কমান্ডটি টাইপ করুন:

   ```bash title="" linenums="0"
   azd up
   ```

এই কমান্ডটি যা ট্রিগার করবে সে প্রক্রিয়ার ধাপগুলো সম্পূর্ণ করুন:

১. লগইন করার জন্য আপনাকে Azure-তে প্রম্পট করা হবে - নির্দেশনা অনুসরণ করুন
১. আপনার জন্য একটি অনন্য পরিবেশ নাম দিন - যেমন আমি ব্যবহার করেছি `nitya-mshack-azd`
১. এটি একটি `.azure/` ফোল্ডার তৈরি করবে - আপনি পরিবেশ নাম সহ একটি সাবফোল্ডার দেখতে পাবেন
১. সাবস্ক্রিপশন নাম নির্বাচন করতে আপনাকে প্রম্পট করা হবে - ডিফল্ট নির্বাচন করুন
১. একটি লোকেশন নির্বাচন করতে বলা হবে - `East US 2` ব্যবহার করুন

এবার, প্রোভিশনিং শেষ হওয়া পর্যন্ত অপেক্ষা করুন। **এটি ১০-১৫ মিনিট সময় নেয়**

১. শেষ হলে, আপনার কনসোল SUCCESS মেসেজ দেখাবে এভাবে:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. আপনার Azure পোর্টালে ওই পরিবেশ নাম সহ একটি প্রোভিশনকৃত রিসোর্স গ্রুপ থাকবে:

      ![Infra](../../../../../translated_images/bn/02-provisioned-infra.46c706b14f56e0bf.webp)

১. **এখন আপনি ডিপ্লয় করা ইনফ্রাস্ট্রাকচার এবং অ্যাপ্লিকেশন যাচাই করার জন্য প্রস্তুত**।

## ৫. এজেন্ট যাচাই

Azure Container App একটি এন্ডপয়েন্ট ডিপ্লয় করে যা এই টেমপ্লেটের Microsoft Foundry প্রজেক্টে প্রোভিশনকৃত AI এজেন্টের সাথে সংযুক্ত। আসুন দেখি এর অর্থ কী।

১. আপনার রিসোর্স গ্রুপের Azure পোর্টালের _Overview_ পেজে ফিরুন

১. ঐ তালিকায় `Microsoft Foundry` রিসোর্সে ক্লিক করুন

১. আপনি এটা দেখতে পাবেন। `Go to Microsoft Foundry Portal` বোতামে ক্লিক করুন। 
   ![Foundry](../../../../../translated_images/bn/04-view-foundry-project.fb94ca41803f28f3.webp)

১. আপনার AI অ্যাপ্লিকেশনের Foundry প্রজেক্ট পেজ দেখতে পাবেন
   ![Project](../../../../../translated_images/bn/05-visit-foundry-portal.d734e98135892d7e.webp)

১. `Agents` এ ক্লিক করুন - আপনার প্রজেক্টে ডিফল্ট এজেন্ট প্রোভিশন করা আছে দেখতে পাবেন
   ![Agents](../../../../../translated_images/bn/06-visit-agents.bccb263f77b00a09.webp)

১. সেটি নির্বাচন করুন - আপনি এজেন্টের বিস্তারিত তথ্য দেখতে পাবেন। নোট করুন:

      - এজেন্ট ডিফল্টরূপে ফাইল সার্চ ব্যবহার করে (সর্বদা)
      - এজেন্টের `Knowledge` নির্দেশ করে ৩২ টি ফাইল আপলোড করা হয়েছে (ফাইল সার্চের জন্য)
      ![Agents](../../../../../translated_images/bn/07-view-agent-details.0e049f37f61eae62.webp)

১. বাম মেনুতে `Data+indexes` অপশনটি দেখুন এবং বিস্তারিত জানার জন্য ক্লিক করুন। 

      - আপনি ৩২ টি ডেটা ফাইল knowledge এর জন্য আপলোড করা দেখবেন।
      - এগুলি `src/files` এর অধীনে ১২ টি কাস্টমার ফাইল এবং ২০ টি প্রোডাক্ট ফাইলের সাথে মিল থাকবে
      ![Data](../../../../../translated_images/bn/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**আপনি এজেন্ট অপারেশন যাচাই করেছেন!** 

১. এজেন্টের উত্তরের ভিত্তি ওই ফাইলগুলোর জ্ঞানে।
১. আপনি এখন ওই ডেটা সম্পর্কিত প্রশ্ন করতে পারেন এবং ভিত্তিপ্রাপ্ত উত্তর পেতে পারেন।
১. উদাহরণ: `customer_info_10.json` "Amanda Perez" এর ৩ টি কেনাকাটার বর্ণনা দেয়

ব্রাউজার ট্যাবে Container App এন্ডপয়েন্টে ফিরে যান এবং জিজ্ঞাসা করুন: `Amanda Perez কোন কোন প্রোডাক্টের মালিক?`. আপনি এরকম কিছু দেখতে পাবেন:

১. এই ফাইল লেবেলগুলোর উপরে কারসর রাখুন - ডেটা আপনার অনুসন্ধান এবং প্রদর্শিত উত্তরের সাথে মিলে যায় কি না পরীক্ষা করুন

উত্তরের নিচে একটি _স্ট্যাটস_ সারি দেখতে পাবেন। 

১. যেকোনো ম্যাট্রিক্সে কারসর রাখুন - যেমন Safety. আপনি এরকম কিছু দেখতে পাবেন
১. মূল্যায়িত রেটিং আপনার উত্তরের নিরাপত্তা স্তরের ধারণার সাথে মিলছে কি?

## ৭. বিল্ট-ইন অবজারভেবিলিটি

অবজারভেবিলিটি হল আপনার অ্যাপ্লিকেশনকে ডিবাগ, অপ্টিমাইজ এবং অপারেশন বুঝতে সাহায্য করার জন্য তথ্য তৈরি করার উপায়। এর ধারণা পেতে:

১. `View Run Info` বোতামে ক্লিক করুন - আপনি এই ভিউটি দেখতে পাবেন। এটি [এজেন্ট ট্রেসিং](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) এর একটি উদাহরণ। _আপনি উপরের মেনু থেকে Thread Logs ক্লিক করেও এই ভিউ পেতে পারেন_।

   - এজেন্ট দ্বারা সম্পন্ন ধাপ এবং ব্যবহৃত টুলগুলি বুঝুন
   - প্রতিক্রিয়ার জন্য মোট Token গণনা (আউটপুট টোকেন ব্যবহার থেকে পার্থক্যসহ) জানুন
   - লেটেন্সি ও কার্যনির্বাহে সময় কোথায় লাগছে বুঝুন

১. চালনার অতিরিক্ত প্রাসঙ্গিক তথ্য দেখতে `Metadata` ট্যাবে ক্লিক করুন, যা ভবিষ্যতে ডিবাগ করতে সাহায্য করবে।   

      ![Agent](../../../../../translated_images/bn/11-view-run-info-metadata.7966986122c7c2df.webp)


১. `Evaluations` ট্যাবে ক্লিক করুন, যেখানে এজেন্টের প্রতিক্রিয়ার উপর অটো-মূল্যায়ন রয়েছে। যেমন নিরাপত্তা মূল্যায়ন (Self-harm ইত্যাদি) এবং এজেন্ট-নির্দিষ্ট মূল্যায়ন (Intent resolution, Task adherence ইত্যাদি)।

      - প্রদর্শিত পেজে `Resource usage` ট্যাব নির্বাচন করুন এবং মেট্রিক্স দেখুন।
      - খরচ (টোকেন) এবং লোড (রিকোয়েস্ট) অনুযায়ী অ্যাপ্লিকেশন ব্যবহার ট্র্যাক করুন।

## ৮. এনভায়রনমেন্ট ভ্যারিয়েবলস্‌

এখন পর্যন্ত, আমরা ব্রাউজারে ডিপ্লয়মেন্ট করেছি এবং যাচাই করেছি যে ইনফ্রাস্ট্রাকচার প্রোভিশন হয়েছে এবং অ্যাপ্লিকেশন চালু আছে। কিন্তু যখন আমরা _কোড-প্রথম_ পদ্ধতিতে কাজ করব, তখন আমাদের স্থানীয় ডেভেলপমেন্ট পরিবেশে সংশ্লিষ্ট ভ্যারিয়েবলস্‌ কনফিগার করতে হবে যেগুলো এই রিসোর্সগুলোর সঙ্গে কাজ করার জন্য দরকার। `azd` এর মাধ্যমে এটি সহজ হয়।

১. Azure Developer CLI [পরিবেশ ভ্যারিয়েবলস্‌ ব্যবহার করে](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) অ্যাপ্লিকেশন ডিপ্লয়মেন্টের কনফিগারেশন সেটিংস সংরক্ষণ ও পরিচালনা করতে।

১. পরিবেশ ভ্যারিয়েবলস্‌ `.azure/<env-name>/.env` ফাইলে সংরক্ষিত থাকে - যা ডিপ্লয়মেন্টের সময় ব্যবহৃত `env-name` পরিবেশে সীমাবদ্ধ করে এবং একই রিপোজিটরির বিভিন্ন ডিপ্লয়মেন্ট টার্গেটের মধ্যে পরিবেশ আলাদা রাখতে সহায়তা করে।

১. পরিবেশ ভ্যারিয়েবলস্‌ `azd` কমান্ড দ্বারা স্বয়ংক্রিয়ভাবে লোড হয় যখন এটি একটি নির্দিষ্ট কমান্ড (যেমন `azd up`) চালায়। লক্ষণীয় যে `azd` স্বয়ংক্রিয়ভাবে _OS-লেভেলের_ পরিবেশ ভ্যারিয়েবলস্‌ পড়ে না (যেমন শেল-এ সেট করা) - এর বদলে `azd set env` ও `azd get env` ব্যবহার করে তথ্য স্ক্রিপ্টের মধ্যে স্থানান্তর করুন।


কিছু কমান্ড পরীক্ষা করি:

১. এই পরিবেশে `azd`-এর জন্য সেট করা সব পরিবেশ ভ্যারিয়েবলস্‌ পান:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      আপনি এরকম কিছু দেখতে পাবেন:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

১. একটি নির্দিষ্ট মান পান - যেমন, আমি জানতে চাই `AZURE_AI_AGENT_MODEL_NAME` সেট করা হয়েছে কিনা

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      আপনি এরকম কিছু দেখতে পাবেন - এটি ডিফল্টে সেট ছিল না!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

১. `azd` এর জন্য নতুন একটি পরিবেশ ভ্যারিয়েবল সেট করুন। এখানে, আমরা এজেন্ট মডেল নাম আপডেট করছি। _দ্রষ্টব্য: সকল পরিবর্তন সঙ্গে সঙ্গেই `.azure/<env-name>/.env` ফাইলে প্রতিফলিত হবে।

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      এখন, আমরা দেখতে পারি মান সেট হয়েছে:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

১. লক্ষ্য করুন যে কিছু রিসোর্স স্থায়ী (যেমন মডেল ডিপ্লয়মেন্ট) এবং সেগুলোর পুনঃডিপ্লয় করার জন্য শুধু `azd up` যথেষ্ট নয়। আসুন মূল ডিপ্লয়মেন্টটি গুছিয়ে পরিবর্তিত env vars নিয়ে পুনরায় ডিপ্লয় করি।

১. **রিফ্রেশ** যদি আপনি পূর্বে কোনো azd টেমপ্লেট ব্যবহার করে ইনফ্রাস্ট্রাকচার ডিপ্লয় করে থাকেন - তাহলে আপনি বর্তমান Azure ডিপ্লয়মেন্টের অবস্থা অনুসারে স্থানীয় এনভায়রনমেন্ট ভ্যারিয়েবলস্‌ের অবস্থা _রিফ্রেশ_ করতে পারেন এই কমান্ড দিয়ে:

      ```bash title="" linenums="0"
      azd env refresh
      ```


      এটি দুটি বা তার বেশি স্থানীয় ডেভেলপমেন্ট পরিবেশের (যেমন, একাধিক ডেভেলপার সহ দল) মধ্যে পরিবেশ ভেরিয়েবল _sync_ করার একটি শক্তিশালী উপায় - যা মোতায়েনকৃত অবকাঠামোকে পরিবেশ ভেরিয়েবল অবস্থার জন্য ভিত্তি সত্য হিসেবে কাজ করতে দেয়। দলীয় সদস্যরা সহজেই _refresh_ করে ভেরিয়েবলগুলি আবার সিঙ্ক করতে পারে।
