## 1. একটি নির্মাতা উপমা

সম্পূর্ণ নতুন থেকে একটি আধুনিক এন্টারপ্রাইজ-সজ্জিত AI অ্যাপ্লিকেশন তৈরি করা ভয়ংকর হতে পারে। এটি একটু এমন যেন নিজে নিজের বাড়ি ইটে ইটে তৈরি করা। হ্যাঁ, এটা করা সম্ভব! কিন্তু এটি কাঙ্ক্ষিত চূড়ান্ত ফলাফল পাওয়ার সবচেয়ে কার্যকর উপায় নয়!

পরিবর্তে, আমরা প্রায়শই একটি বিদ্যমান _ডিজাইন ব্লুপ্রিন্ট_ দিয়ে শুরু করি, এবং একটি আর্কিটেক্টের সাথে কাজ করি সেটিকে আমাদের ব্যক্তিগত প্রয়োজন অনুযায়ী কাস্টমাইজ করার জন্য। এবং বুদ্ধিমান অ্যাপ্লিকেশন তৈরি করার সময় এটাই সঠিক পদ্ধতি। প্রথমে এমন একটি ভাল ডিজাইন আর্কিটেকচার খুঁজুন যা আপনার সমস্যা ক্ষেত্রের সাথে মানানসই। তারপর একটি সলিউশন আর্কিটেক্টের সাথে কাজ করুন যাতে আপনার নির্দিষ্ট পরিস্থিতির জন্য সলিউশনটি কাস্টমাইজ এবং উন্নত করা যায়।

কিন্তু আমরা কোথা থেকে এই ডিজাইন ব্লুপ্রিন্টগুলি খুঁজে পাব? এবং আমরা কীভাবে এমন একজন আর্কিটেক্ট পাব যিনি আমাদের শেখাবেন কীভাবে আমরা এই ব্লুপ্রিন্টগুলি কাস্টমাইজ এবং নিজে নিজে ডিপ্লয় করব? এই কর্মশালায়, আমরা সেই প্রশ্নগুলোর উত্তর দেব তিনটি প্রযুক্তি পরিচয় করিয়ে দিয়ে:

1. [Azure Developer CLI](https://aka.ms/azd) - একটি ওপেন-সোর্স টুল যা ডেভেলপারদের স্থানীয় উন্নয়ন (বিল্ড) থেকে ক্লাউড ডিপ্লয়মেন্ট (শিপ) পর্যন্ত যাত্রাকে দ্রুততর করে।
1. [Microsoft Foundry Templates](https://ai.azure.com/templates) - AI সলিউশন আর্কিটেকচার ডিপ্লয়মেন্টের জন্য নমুনা কোড, অবকাঠামো এবং কনফিগারেশন ফাইল সহ মানকৃত ওপেন-সোর্স রিপোজিটোরি।
1. [GitHub Copilot Agent Mode](https://code.visualstudio.com/docs/copilot/chat/chat-agent-mode) - একটি কোডিং এজেন্ট যা Azure জ্ঞানের উপর ভিত্তি করে তৈরি, যা আমাদের কোডবেস নেভিগেট করতে এবং পরিবর্তন করতে সাহায্য করে - প্রাকৃতিক ভাষা ব্যবহার করে।

এই টুলগুলি হাতে পেয়ে, আমরা এখন _উপযুক্ত টেমপ্লেট আবিষ্কার_ করতে পারি, সেটি যাচাই করার জন্য _ডিপ্লয়_ করতে পারি, এবং আমাদের নির্দিষ্ট পরিস্থিতির জন্য _কাস্টমাইজ_ করতে পারি। চলুন শুরু করি এবং শিখি এগুলো কীভাবে কাজ করে।

## 2. Azure Developer CLI

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) (বা `azd`) একটি ওপেন-সোর্স কমান্ডলাইন টুল যা ডেভেলপার-অনুকূল কমান্ডসের একটি সেট দিয়ে আপনার কোড-থেকে-ক্লাউড যাত্রাকে গতি দেয় যা আপনার IDE (ডেভেলপমেন্ট) এবং CI/CD (ডেভঅপ্স) পরিবেশগুলোর মধ্যে ধারাবাহিকভাবে কাজ করে।

`azd` দিয়ে, আপনার ডিপ্লয়মেন্ট যাত্রা হতে পারে খুবই সহজ:

- `azd init` - বিদ্যমান AZD টেমপ্লেট থেকে একটি নতুন AI প্রকল্প ইনিশিয়ালাইজ করে।
- `azd up` - এক ধাপে অবকাঠামো সরবরাহ এবং আপনার অ্যাপ্লিকেশন ডিপ্লয় করে।
- `azd monitor` - আপনার ডিপ্লয়কৃত অ্যাপ্লিকেশনের জন্য রিয়েল-টাইম মনিটরিং এবং ডায়গনস্টিক্স পান।
- `azd pipeline config` - Azure এ স্বয়ংক্রিয় ডিপ্লয়মেন্টের জন্য CI/CD পাইপলাইন সেটআপ করুন।

**🎯 | অনুশীলন**: <br/> আপনার বর্তমান কর্মশালা পরিবেশে এখনই `azd` কমান্ডলাইন টুলটি পরীক্ষা করুন। এটি GitHub Codespaces, একটি ডেভ কন্টেইনার, অথবা প্রয়োজনীয়তাসহ একটি লোকাল ক্লোন হতে পারে। কী টুলটি করতে পারে তা দেখার জন্য এই কমান্ডটি দিয়ে শুরু করুন:

```bash title="" linenums="0"
azd help
```

## 3. AZD টেমপ্লেট

`azd` এর এই সক্ষমতা অর্জনের জন্য, এটি জানতে হবে কোন অবকাঠামো সরবরাহ করতে হবে, কোন কনফিগারেশন সেটিংস প্রয়োগ করতে হবে, এবং কোন অ্যাপ্লিকেশনটি ডিপ্লয় করতে হবে। এখানেই [AZD টেমপ্লেটগুলি](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/azd-templates?tabs=csharp) আসে।

AZD টেমপ্লেটগুলি ওপেন-সোর্স রিপোজিটোরি যা নমুনা কোডকে অবকাঠামো এবং কনফিগারেশন ফাইলের সাথে যুক্ত করে যা সলিউশন আর্কিটেকচার ডিপ্লয় করার জন্য প্রয়োজনীয়।
_ইনফ্রাস্ট্রাকচার-এ-এ- কোড_ (IaC) পদ্ধতি ব্যবহার করে, তারা টেমপ্লেট রিসোর্স ডেফিনেশন এবং কনফিগারেশন সেটিংসকে ভার্সন-কন্ট্রোলযোগ্য করে তোলে (ঠিক যেমন অ্যাপ সোর্স কোড) - যাতে প্রোজেক্ট ব্যবহারকারীদের মধ্যে পুনর্ব্যবহারযোগ্য এবং ধারাবাহিক ওয়ার্কফ্লো তৈরি করা যায়।

আপনার পরিস্থিতির জন্য একটি AZD টেমপ্লেট তৈরি বা পুনঃব্যবহার করার সময়, এই প্রশ্নগুলি বিবেচনা করুন:

1. আপনি কী তৈরি করছেন? → এই পরিস্থিতির জন্য কি কোনও টেমপ্লেটে স্টার্টার কোড আছে?
1. আপনার সলিউশন কিভাবে আর্কিটেক্ট হয়েছে? → কি দরকারি রিসোর্স নিয়ে একটি টেমপ্লেট আছে?
1. আপনার সলিউশন কিভাবে ডিপ্লয় করা হয়? → `azd deploy` এর পূর্ব/পরবর্তী প্রসেসিং হুক চিন্তা করুন!
1. আরও কীভাবে অপ্টিমাইজ করা যায়? → ইন-বিল্ট মনিটরিং এবং অটোমেশন পাইপলাইন চিন্তা করুন!

**🎯 | অনুশীলন**: <br/> 
[Awesome AZD](https://azure.github.io/awesome-azd/) গ্যালারিতে যান এবং বর্তমানে উপলব্ধ ২৫০+ টেমপ্লেটগুলো ছাঁকনি ব্যবহার করে অনুসন্ধান করুন। দেখুন কোনটি আপনার পরিস্থিতির প্রয়োজনীয়তার সাথে মানানসই।

## 4. AI অ্যাপ টেমপ্লেট

AI-সম্পৃক্ত অ্যাপ্লিকেশনের জন্য, Microsoft বিশেষায়িত টেমপ্লেট প্রদান করে যা **Microsoft Foundry** এবং **Foundry Agents** অন্তর্ভুক্ত করে। এই টেমপ্লেটগুলি আপনার বুদ্ধিমান, প্রোডাকশন-সজ্জিত অ্যাপ্লিকেশন তৈরির পথকে দ্রুততর করে।

### Microsoft Foundry & Foundry Agents টেমপ্লেটসমূহ

নিচের টেমপ্লেট থেকে একটি নির্বাচন করুন এবং ডিপ্লয় করুন। প্রতিটি টেমপ্লেট [Awesome AZD](https://azure.github.io/awesome-azd/) এ পাওয়া যায় এবং একটি একক কমান্ড দিয়ে ইনিশিয়ালাইজ করা যায়।

| টেমপ্লেট | বর্ণনা | ডিপ্লয় কমান্ড |
|----------|-------------|----------------|
| **[RAG সহ AI চ্যাট](https://azure.github.io/awesome-azd/?tags=ai&tags=rag)** | Microsoft Foundry ব্যবহার করে রিট্রিভাল অগমেন্টেড জেনারেশনের সঙ্গে চ্যাট অ্যাপ্লিকেশন | `azd init -t azure-samples/azure-search-openai-demo` |
| **[Foundry Agent সার্ভিস স্টার্টার](https://azure.github.io/awesome-azd/?tags=ai&tags=agents)** | Foundry Agents সহ স্বয়ংক্রিয় টাস্ক সম্পাদনার জন্য AI এজেন্ট তৈরি করুন | `azd init -t azure-samples/foundry-agent-service-starter` |
| **[মাল্টি-এজেন্ট অর্কেস্ট্রেশন](https://azure.github.io/awesome-azd/?tags=ai&tags=agents)** | জটিল ওয়ার্কফ্লোর জন্য একাধিক Foundry Agents সমন্বয় করুন | `azd init -t azure-samples/multi-agent-orchestration` |
| **[AI ডকুমেন্ট ইন্টেলিজেন্স](https://azure.github.io/awesome-azd/?tags=ai&tags=document)** | Microsoft Foundry মডেল ব্যবহার করে নথিপত্র উত্তোলন এবং বিশ্লেষণ করুন | `azd init -t azure-samples/ai-document-processing` |
| **[কনভারসেশনাল AI বট](https://azure.github.io/awesome-azd/?tags=ai&tags=bot)** | Microsoft Foundry ইন্টিগ্রেশনের মাধ্যমে বুদ্ধিমান চ্যাটবট তৈরি করুন | `azd init -t azure-samples/ai-chat-protocol` |
| **[AI ইমেজ জেনারেশন](https://azure.github.io/awesome-azd/?tags=ai&tags=dalle)** | Microsoft Foundry এর মাধ্যমে DALL-E ব্যবহার করে ছবি তৈরি করুন | `azd init -t azure-samples/ai-image-generation` |
| **[সেম্যান্টিক কার্নেল এজেন্ট](https://azure.github.io/awesome-azd/?tags=ai&tags=semantic-kernel)** | Foundry Agents সহ Semantic Kernel ব্যবহার করে AI এজেন্ট | `azd init -t azure-samples/semantic-kernel-agent` |
| **[AutoGen মাল্টি-এজেন্ট](https://azure.github.io/awesome-azd/?tags=ai&tags=autogen)** | AutoGen ফ্রেমওয়ার্ক ব্যবহার করে মাল্টি-এজেন্ট সিস্টেম | `azd init -t azure-samples/autogen-multi-agent` |

### দ্রুত শুরু

1. **টেমপ্লেট ব্রাউজ করুন**: যান [https://azure.github.io/awesome-azd/](https://azure.github.io/awesome-azd/) এবং `AI`, `Agents`, অথবা `Microsoft Foundry` দ্বারা ফিল্টার করুন
2. **টেমপ্লেট নির্বাচন করুন**: আপনার ব্যবহারিক ক্ষেত্রে উপযুক্ত একটি বেছে নিন
3. **ইনিশিয়ালাইজ করুন**: আপনার নির্বাচিত টেমপ্লেটের জন্য `azd init` কমান্ড চালান
4. **ডিপ্লয় করুন**: `azd up` চালিয়ে অবকাঠামো সরবরাহ এবং ডিপ্লয় করুন

**🎯 | অনুশীলন**: <br/>
উপরে উল্লেখিত টেমপ্লেটগুলোর মধ্যে আপনার পরিস্থিতি অনুযায়ী একটি নির্বাচন করুন:

- **কোনও চ্যাটবট বানাচ্ছেন?** → **RAG সহ AI চ্যাট** অথবা **কনভারসেশনাল AI বট** দিয়ে শুরু করুন
- **স্বায়ত্তশাসিত এজেন্ট চাই?** → **Foundry Agent সার্ভিস স্টার্টার** বা **মাল্টি-এজেন্ট অর্কেস্ট্রেশন** চেষ্টা করুন
- **ডকুমেন্ট প্রসেসিং করছেন?** → **AI ডকুমেন্ট ইন্টেলিজেন্স** ব্যবহার করুন
