# AZD For Beginners-এ অবদান রাখা

Thank you for your interest in contributing to the AZD For Beginners course! This document provides guidelines and information for contributors.

## অবদান কীভাবে করবেন

### সমস্যা রিপোর্ট করা

If you find bugs, errors, or have suggestions for improvements:

1. **বিদ্যমান ইস্যু যাচাই করুন**: ডুপ্লিকেট এড়াতে [বিদ্যমান ইস্যুসমূহ](https://github.com/microsoft/azd-for-beginners/issues) অনুসন্ধান করুন
2. **নতুন ইস্যু তৈরি করুন**: উপযুক্ত ইস্যু টেমপ্লেট ব্যবহার করুন
3. **বিস্তারিত দিন**: পুনরুত্পাদনের ধাপ, প্রত্যাশিত আচরণ, এবং প্রযোজ্য ক্ষেত্রে স্ক্রিনশট অন্তর্ভুক্ত করুন

### পরিবর্তন প্রস্তাব করা

1. **রিপোজিটরি ফর্ক করুন**: প্রকল্পের নিজের একটি ফর্ক তৈরি করুন
2. **একটি ব্রাঞ্চ তৈরি করুন**: আপনার পরিবর্তনগুলোর জন্য একটি ফিচার ব্রাঞ্চ তৈরি করুন
3. **পরিবর্তন করুন**: আমাদের কোডিং ও ডকুমেন্টেশন মান অনুসরণ করুন
4. **আপনার পরিবর্তনগুলি পরীক্ষা করুন**: নিশ্চিত করুন সব লিংক কাজ করে এবং কনটেন্ট সঠিকভাবে রেন্ডার হয়
5. **পুল রিকোয়েস্ট সাবমিট করুন**: আপনার পরিবর্তনের একটি স্পষ্ট বর্ণনা দিন

### অবদানের ধরনসমূহ

We welcome the following types of contributions:

#### বিষয়বস্তুর উন্নতি
- টাইপো, ব্যাকরণ এবং ফরম্যাটিং সমস্যা সংশোধন করুন
- বিদ্যমান ডকুমেন্টেশনের স্পষ্টতা উন্নত করুন
- পুরনো তথ্য হালনাগাদ করুন
- অনুপস্থিত ব্যাখ্যা বা প্রেক্ষাপট যোগ করুন

#### নতুন উদাহরণ
- বাস্তব জীবনের ডেপ্লয়মেন্ট পরিস্থিতি যোগ করুন
- সাধারণ ব্যবহার ক্ষেত্রে জন্য নতুন AZD টেমপলেট তৈরি করুন
- অতিরিক্ত কোড স্যাম্পল এবং অনুশীলন যোগ করুন

#### অনুবাদ সমর্থন
- বিদ্যমান অনুবাদ বজায় রাখতে সহায়তা করুন
- অনুবাদের গুণগত মান উন্নত করুন
- নতুন কনটেন্টের জন্য অনুবাদ যোগ করুন

#### বাগ রিপোর্ট
- ভাঙা লিংক বা ভুল কোড উদাহরণ রিপোর্ট করুন
- ডকুমেন্টেশনের অসঙ্গতি শনাক্ত করুন
- অ্যাক্সেসিবিলিটি সমস্যা চিহ্নিত করুন

## অবদান নির্দেশিকা

### ডকুমেন্টেশন মানদণ্ড

- পরিষ্কার, নবাগত-বান্ধব ভাষা ব্যবহার করুন
- শিক্ষা লক্ষ্য এবং ফলাফল অন্তর্ভুক্ত করুন
- প্রত্যাশিত আউটপুটসহ ধাপে ধাপে নির্দেশনা প্রদান করুন
- সম্পর্কিত কন্টেন্টের জন্য নেভিগেশন লিঙ্ক যোগ করুন
- সামঞ্জস্যপূর্ণ ফরম্যাটিং এবং মার্কডাউন স্টাইল ব্যবহার করুন

### কোড উদাহরণ

- নিশ্চিত করুন সব কোড উদাহরণ পরীক্ষিত এবং কাজ করছে
- মূল ধারণাগুলো ব্যাখ্যা করে মন্তব্য যোগ করুন
- Azure এবং AZD-এর সেরা অভ্যাস অনুসরণ করুন
- প্রযোজ্য ক্ষেত্রে প্রত্যাশিত আউটপুট প্রদান করুন

### কমিট মেসেজ

Use clear, descriptive commit messages:
- `docs: fix typo in installation guide`
- `feat: add new container app example`
- `fix: correct broken link in chapter 2`

## আচরণবিধি

দয়া করে একটি বন্ধুত্বপূর্ণ ও অন্তর্ভুক্তিমূলক কমিউনিটি বজায় রাখতে আমাদের [আচরণবিধি](CODE_OF_CONDUCT.md) পড়ুন এবং অনুসরণ করুন।

## সাহায্য পেতে

- **প্রশ্নসমূহ**: একটি [GitHub আলোচনা](https://github.com/microsoft/azd-for-beginners/discussions) খুলুন
- **কমিউনিটি**: [Azure Discord](https://discord.gg/microsoft-azure)-এ যোগ দিন
- **ডকুমেন্টেশন**: [Azure Developer CLI docs](https://learn.microsoft.com/azure/developer/azure-developer-cli/) পর্যালোচনা করুন
