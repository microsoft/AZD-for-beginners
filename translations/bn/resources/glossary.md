- **📖 বেসিক শিখুন**: [Chapter 1: AZD Basics](../docs/getting-started/azd-basics.md)
- **🤖 AI টার্মস**: [Chapter 2: AI-First Development](../docs/microsoft-foundry/microsoft-foundry-integration.md)

## পরিচিতি

এই বিস্তৃত গ্লসারি Azure Developer CLI এবং Azure ক্লাউড ডেভেলপমেন্টে ব্যবহৃত টার্ম, ধারণা এবং সংক্ষিপ্তরূপের সংজ্ঞা প্রদান করে। azd প্রকল্প এবং Azure সার্ভিস সম্পর্কে প্রযুক্তিগত ডকুমেন্টেশন বোঝার, সমস্যার সমাধান করার এবং কার্যকর যোগাযোগের জন্য এটি অপরিহার্য রেফারেন্স।

## শেখার লক্ষ্য

এই গ্লসারি ব্যবহার করে আপনি:
- Azure Developer CLI-এর নির্দিষ্ট টার্মিনোলজি এবং ধারণা বুঝতে পারবেন
- Azure ক্লাউড ডেভেলপমেন্ট শব্দভাণ্ডার ও প্রযুক্তিগত শব্দসমূহ আয়ত্ত করতে পারবেন
- Infrastructure as Code এবং ডিপ্লয়মেন্ট টার্মিনোলজি দ্রুত রেফারেন্স করতে পারবেন
- Azure সার্ভিসের নাম, সংক্ষিপ্তরূপ এবং তাদের উদ্দেশ্য সম্পর্কে ধারণা পাবেন
- ট্রাবলশুটিং এবং ডিবাগিং টার্মিনোলজির সংজ্ঞা অ্যাক্সেস করতে পারবেন
- উন্নত Azure আর্কিটেকচার ও ডেভেলপমেন্ট ধারণা শিখতে পারবেন

## শেখার ফলাফল

নিয়মিত এই গ্লসারি রেফারেন্স করলে আপনি সক্ষম হবেন:
- সঠিক Azure Developer CLI টার্মিনোলজি ব্যবহার করে কার্যকরভাবে যোগাযোগ করতে
- প্রযুক্তিগত ডকুমেন্টেশন এবং এরর মেসেজগুলি আরও স্পষ্টভাবে বুঝতে
- আত্মবিশ্বাসের সঙ্গে Azure সার্ভিস এবং ধারণাগুলির মধ্যে ন্যাভিগেট করতে
- উপযুক্ত প্রযুক্তিগত শব্দভাণ্ডার ব্যবহার করে সমস্যা সমাধান করতে
- সঠিক প্রযুক্তিগত ভাষায় টিম আলাপচারিতায় অংশগ্রহণ করতে
- সিস্টেম্যাটিকভাবে আপনার Azure ক্লাউড ডেভেলপমেন্ট জ্ঞান বাড়াতে

## A

**ARM Template**  
Azure Resource Manager টেমপ্লেট। JSON-ভিত্তিক Infrastructure as Code ফরম্যাট যা ঘোষণা-ভিত্তিকভাবে Azure রিসোর্সগুলি সংজ্ঞায়িত এবং ডিপ্লয় করতে ব্যবহার করা হয়।

**App Service**  
ওয়েব অ্যাপ্লিকেশন, REST API এবং মোবাইল ব্যাকএন্ড হোস্ট করার জন্য Azure-এর platform-as-a-service (PaaS) সেবা, যেখানে ইনফ্রাস্ট্রাকচার ম্যানেজ করতে হয় না।

**Application Insights**  
Azure-এর অ্যাপ্লিকেশন পারফরম্যান্স মনিটরিং (APM) সার্ভিস যা অ্যাপের পারফরম্যান্স, উপলব্ধতা এবং ব্যবহার সম্পর্কে গভীর অন্তর্দৃষ্টি প্রদান করে।

**Azure CLI**  
Azure রিসোর্স ম্যানেজ করার জন্য কমান্ড-লাইন ইন্টারফেস। azd দ্বারা অথেনটিকেশন এবং কিছু অপারেশনের জন্য ব্যবহৃত হয়।

**Azure Developer CLI (azd)**  
ডেভেলপার-কেন্দ্রিক কমান্ড-লাইন টুল যা টেমপ্লেট এবং Infrastructure as Code ব্যবহার করে অ্যাপ্লিকেশন Azure-এ বিল্ড এবং ডিপ্লয় করার প্রক্রিয়াকে ত্বরান্বিত করে।

**azure.yaml**  
একটি azd প্রকল্পের প্রধান কনফিগারেশন ফাইল যা সার্ভিস, ইনফ্রাস্ট্রাকচার এবং ডিপ্লয়মেন্ট হুকগুলি সংজ্ঞায়িত করে।

**Azure Resource Manager (ARM)**  
Azure-এর ডিপ্লয়মেন্ট এবং ম্যানেজমেন্ট সার্ভিস যা রিসোর্স তৈরি, আপডেট এবং ডিলিট করার জন্য একটি ম্যানেজমেন্ট লেয়ার প্রদান করে।

## B

**Bicep**  
Microsoft দ্বারা ডেভেলপ করা ডোমেইন-স্পেসিফিক ল্যাঙ্গুয়েজ (DSL) যা Azure রিসোর্স ডিপ্লয় করার জন্য। ARM টেমপ্লেটের চেয়ে সহজ সিনট্যাক্স প্রদান করে এবং ARM-এ কম্পাইল হয়।

**Build**  
সোর্স কোড কম্পাইল করা, ডিপেন্ডেন্সি ইনস্টল করা এবং অ্যাপ্লিকেশনকে ডিপ্লয়মেন্টের জন্য প্রস্তুত করার প্রক্রিয়া।

**Blue-Green Deployment**  
ডিপ্লয়মেন্ট কৌশল যা দুটি অভিন্ন প্রোডাকশন পরিবেশ (blue এবং green) ব্যবহার করে ডাউনটাইম এবং ঝুঁকি কমায়।

## C

**Container Apps**  
কন্টেইনারাইজড অ্যাপ্লিকেশন রান করার জন্য Azure-এর serverless কন্টেইনার সার্ভিস, যেখানে জটিল ইনফ্রাস্ট্রাকচার ম্যানেজ করতে হয় না।

**CI/CD**  
Continuous Integration/Continuous Deployment। কোড পরিবর্তন একীভূতকরণ এবং অ্যাপ্লিকেশন ডিপ্লয় করার স্বয়ংক্রিয় অনুশীলন।

**Cosmos DB**  
Azure-এর গ্লোবালি ডিসট্রিবিউটেড, মাল্টি-মডেল ডাটাবেস সার্ভিস যা থ্রুপুট, লেটেন্সি, উপলব্ধতা এবং কনসিস্টেন্সির জন্য ব্যাপক SLA প্রদান করে।

**Configuration**  
সেটিংস এবং প্যারামিটার যা অ্যাপ্লিকেশনের আচরণ এবং ডিপ্লয়মেন্ট অপশনগুলি নিয়ন্ত্রণ করে।

## D

**Deployment**  
অ্যাপ্লিকেশন এবং তাদের ডিপেন্ডেন্সিগুলি লক্ষ্য ইনফ্রাস্ট্রাকচারে ইনস্টল এবং কনফিগার করার প্রক্রিয়া।

**Docker**  
কন্টেইনারাইজেশন প্রযুক্তি ব্যবহার করে অ্যাপ্লিকেশন ডেভেলপ, শিপ এবং রান করার প্ল্যাটফর্ম।

**Dockerfile**  
Docker কন্টেইনার ইমেজ বিল্ড করার নির্দেশাবলী ধারণকারী টেক্সট ফাইল।

## E

**Environment**  
একটি ডিপ্লয়মেন্ট টার্গেট যা আপনার অ্যাপ্লিকেশনের একটি নির্দিষ্ট ইনস্ট্যান্সকে প্রতিনিধিত্ব করে (যেমন, ডেভেলপমেন্ট, স্টেজিং, প্রোডাকশন)।

**Environment Variables**  
কী-ভ্যালু জোড়া হিসেবে সংরক্ষিত কনফিগারেশন মান যেগুলো অ্যাপ্লিকেশন রানটাইমে অ্যাক্সেস করতে পারে।

**Endpoint**  
URL বা নেটওয়ার্ক ঠিকানা যেখানে একটি অ্যাপ্লিকেশন বা সার্ভিসকে অ্যাক্সেস করা যায়।

## F

**Function App**  
ইভেন্ট-চালিত কোড চালানোর জন্য Azure-এর serverless compute সার্ভিস, যেখানে ইনফ্রাস্ট্রাকচার ম্যানেজ করতে হয় না।

## G

**GitHub Actions**  
GitHub রিপোজিটরিগোর সাথে ইন্টিগ্রেটেড CI/CD প্ল্যাটফর্ম যা ওয়ার্কফ্লো স্বয়ংক্রিয় করে।

**Git**  
সোর্স কোডে পরিবর্তন ট্র্যাক করার জন্য ব্যবহৃত ডিস্ট্রিবিউটেড ভার্সন কনট্রোল সিস্টেম।

## H

**Hooks**  
ডিপ্লয়মেন্ট লাইফসাইকেলের নির্দিষ্ট পয়েন্টে চলা কাস্টম স্ক্রিপ্ট বা কমান্ড (preprovision, postprovision, predeploy, postdeploy)।

**Host**  
Azure সার্ভিস টাইপ যেখানে একটি অ্যাপ্লিকেশন ডিপ্লয় করা হবে (উদাহরণস্বরূপ, appservice, containerapp, function)।

## I

**Infrastructure as Code (IaC)**  
ম্যানুয়াল প্রক্রিয়ার পরিবর্তে কোডের মাধ্যমে ইনফ্রাস্ট্রাকচার সংজ্ঞায়িত এবং ম্যানেজ করার অনুশীলন।

**Init**  
সাধারণত একটি টেমপ্লেট থেকে নতুন azd প্রকল্প ইনিশিয়ালাইজ করার প্রক্রিয়া।

## J

**JSON**  
JavaScript Object Notation। কনফিগারেশন ফাইল এবং API রেসপন্সের জন্য সাধারণত ব্যবহৃত ডেটা বিনিময় ফরম্যাট।

**JWT**  
JSON Web Token। পক্ষগুলির মধ্যে নিরাপদভাবে তথ্য প্রেরণের জন্য একটি স্ট্যান্ডার্ড JSON অবজেক্ট ফর্ম্যাট।

## K

**Key Vault**  
সিক্রেট, কী এবং সার্টিফিকেট নিরাপদভাবে সংরক্ষণ এবং ম্যানেজ করার জন্য Azure সার্ভিস।

**Kusto Query Language (KQL)**  
Azure Monitor, Application Insights এবং অন্যান্য Azure সার্ভিসে ডেটা বিশ্লেষণের জন্য ব্যবহৃত কুয়েরি ভাষা।

## L

**Load Balancer**  
ইনকামিং নেটওয়ার্ক ট্র্যাফিককে একাধিক সার্ভার বা ইনস্ট্যান্সে বিতরণ করার সার্ভিস।

**Log Analytics**  
ক্লাউড এবং অন-প্রিমাইসেস পরিবেশ থেকে টেলিমেট্রি ডেটা সংগ্রহ, বিশ্লেষণ এবং কার্যকর করার জন্য Azure সার্ভিস।

## M

**Managed Identity**  
অন্য Azure সার্ভিসে অথেনটিকেট করার জন্য Azure সার্ভিসগুলিকে স্বয়ংক্রিয়ভাবে ম্যানেজ করা পরিচয় প্রদান করা একটি Azure ফিচার।

**Microservices**  
আর্কিটেকচারের একটি উপায় যেখানে অ্যাপ্লিকেশনগুলো ছোট, স্বাধীন সার্ভিসগুলোর সংগ্রহ হিসাবে নির্মিত হয়।

**Monitor**  
অ্যাপ্লিকেশন এবং ইনফ্রাস্ট্রাকচারের উপর ফুল-স্ট্যাক অবজার্ভেবিলিটি প্রদান করে এমন Azure-এর ইউনিফাইড মনিটরিং সলিউশন।

## N

**Node.js**  
Chrome-এর V8 JavaScript ইঞ্জিনের উপর নির্মিত সার্ভার-সাইড অ্যাপ্লিকেশন তৈরির জন্য JavaScript রানটাইম।

**npm**  
Node.js-এর প্যাকেজ ম্যানেজার যা ডিপেন্ডেন্সি এবং প্যাকেজ ম্যানেজ করে।

## O

**Output**  
ইনফ্রাস্ট্রাকচার ডিপ্লয়মেন্ট থেকে ফেরত প্রস্তুত মানগুলো যা অ্যাপ্লিকেশন বা অন্যান্য রিসোর্স দ্বারা ব্যবহার করা যায়।

## P

**Package**  
অ্যাপ্লিকেশন কোড এবং ডিপেন্ডেন্সিগুলো ডিপ্লয়মেন্টের জন্য প্রস্তুত করার প্রক্রিয়া।

**Parameters**  
ডিপ্লয়মেন্ট কাস্টমাইজ করার জন্য ইনফ্রাস্ট্রাকচার টেমপ্লেটে পাস করা ইনপুট মান।

**PostgreSQL**  
ওপেন-সোর্স রিলেশনাল ডাটাবেস সিস্টেম যা Azure-এ একটি ম্যানেজড সার্ভিস হিসেবে সমর্থিত।

**Provisioning**  
ইনফ্রাস্ট্রাকচার টেমপ্লেটে সংজ্ঞায়িত Azure রিসোর্স তৈরি এবং কনফিগার করার প্রক্রিয়া।

## Q

**Quota**  
একটি Azure সাবস্ক্রিপশন বা রিজিয়নে তৈরি করা যাবে এমন রিসোর্সের পরিমাণে সীমা।

## R

**Resource Group**  
একই লাইফসাইকেল, পারমিশন এবং নীতিমালা শেয়ার করা Azure রিসোর্সগুলোর জন্য একটি লজিক্যাল কনটেইনার।

**Resource Token**  
azd দ্বারা জেনারেট করা ইউনিক স্ট্রিং যা ডিপ্লয়মেন্ট জুড়ে রিসোর্স নামগুলো ইউনিক রয়েছে তা নিশ্চিত করে।

**REST API**  
HTTP পদ্ধতি ব্যবহার করে নেটওয়ার্কেড অ্যাপ্লিকেশন ডিজাইন করার আর্কিটেকচারের স্টাইল।

**Rollback**  
অ্যাপ্লিকেশন বা ইনফ্রাস্ট্রাকচার কনফিগারেশনের আগের ভার্সনে ফিরে যাওয়ার প্রক্রিয়া।

## S

**Service**  
আপনার অ্যাপ্লিকেশনের একটি উপাদান যা azure.yaml-এ সংজ্ঞায়িত (যেমন, web frontend, API backend, database)।

**SKU**  
Stock Keeping Unit। Azure রিসোর্সের বিভিন্ন সার্ভিস টিয়ার বা পারফরম্যান্স লেভেল উপস্থাপন করে।

**SQL Database**  
Microsoft SQL Server ভিত্তিক Azure-এর ম্যানেজড রিলেশনাল ডাটাবেস সার্ভিস।

**Static Web Apps**  
সোর্স কোড রিপোজিটরি থেকে ফুল-স্ট্যাক ওয়েব অ্যাপ্লিকেশন বিল্ড এবং ডিপ্লয় করার জন্য Azure সার্ভিস।

**Storage Account**  
ব্লব, ফাইল, কিউ এবং টেবিলসহ ডেটা অবজেক্টগুলির জন্য ক্লাউড স্টোরেজ প্রদানকারী Azure সার্ভিস।

**Subscription**  
রিসোর্স গ্রুপ এবং রিসোর্স ধারণকারী Azure অ্যাকাউন্ট কনটেইনার, সাথে বিলিং এবং অ্যাক্সেস ম্যানেজমেন্ট সংযুক্ত।

## T

**Template**  
প্রি-বিল্ট প্রজেক্ট স্ট্রাকচার যা অ্যাপ্লিকেশন কোড, ইনফ্রাস্ট্রাকচার সংজ্ঞা এবং সাধারণ সিনারিওগুলির কনফিগারেশন ধারণ করে।

**Terraform**  
একটি ওপেন-সোর্স Infrastructure as Code টুল যা Azure সহ একাধিক ক্লাউড প্রদানকারীকে সাপোর্ট করে।

**Traffic Manager**  
গ্লোবাল Azure রিজিয়ন জুড়ে ট্র্যাফিক বিতরণ করার জন্য Azure-এর DNS-ভিত্তিক ট্র্যাফিক লোড ব্যালান্সার।

## U

**URI**  
Uniform Resource Identifier। একটি নির্দিষ্ট রিসোর্সকে শনাক্ত করার স্ট্রিং।

**URL**  
Uniform Resource Locator। একটি রিসোর্স কোথায় অবস্থিত এবং কীভাবে পুনরুদ্ধার করা যায় তা নির্দিষ্ট করে এমন URI-এর ধরনের স্ট্রিং।

## V

**Virtual Network (VNet)**  
Azure-এ প্রাইভেট নেটওয়ার্কের একটি মৌলিক বিল্ডিং ব্লক, যা আইসোলেশন এবং সেগমেন্টেশন প্রদান করে।

**VS Code**  
Visual Studio Code। চমৎকার Azure এবং azd ইন্টিগ্রেশনের সাথে একটি জনপ্রিয় কোড এডিটর।

## W

**Webhook**  
নির্দিষ্ট ইভেন্ট দ্বারা ট্রিগার করা HTTP কলব্যাক, সাধারণত CI/CD পাইপলাইনে ব্যবহৃত হয়।

**What-if**  
Azure ফিচার যা একটি ডিপ্লয়মেন্ট দ্বারা কী পরিবর্তন করা হবে তা বাস্তবে চালানোর আগে দেখায়।

## Y

**YAML**  
YAML Ain't Markup Language। মানব-পাঠযোগ্য ডেটা সিরিয়ালাইজেশন স্ট্যান্ডার্ড যা azure.yaml-এর মতো কনফিগারেশন ফাইলের জন্য ব্যবহৃত হয়।

## Z

**Zone**  
একটি Azure রিজিয়নের ভৌতভাবে আলাদা অবস্থান যা রিডানড্যান্স এবং উচ্চ উপলব্ধতা প্রদান করে।

## প্রসঙ্গ-নির্দিষ্ট টার্মস

### ডেভেলপমেন্ট টার্মস
- **Hot Reload**: ডেভেলপমেন্টের সময় অ্যাপ্লিকেশনগুলো পুনরায় স্টার্ট না করে স্বয়ংক্রিয়ভাবে আপডেট করা
- **Build Pipeline**: কোড বিল্ড এবং টেস্ট করার স্বয়ংক্রিয় প্রক্রিয়া
- **Deployment Slot**: App Service-এর মধ্যে একটি স্টেজিং পরিবেশ
- **Environment Parity**: ডেভেলপমেন্ট, স্টেজিং এবং প্রোডাকশন পরিবেশগুলোকে সমান রাখা

### সিকিউরিটি টার্মস
- **Managed Identity**: স্বয়ংক্রিয় ক্রেডেনশিয়াল ম্যানেজমেন্ট প্রদানকারী Azure ফিচার
- **Key Vault**: সিক্রেট, কী এবং সার্টিফিকেটের জন্য নিরাপদ স্টোরেজ
- **RBAC**: Azure রিসোর্সগুলির জন্য রো-ভিত্তিক অ্যাক্সেস কন্ট্রোল
- **Network Security Group**: নেটওয়ার্ক ট্র্যাফিক নিয়ন্ত্রণ করার জন্য ভার্চুয়াল ফায়ারওয়াল

### মনিটরিং টার্মস
- **Telemetry**: পরিমাপ এবং ডেটার স্বয়ংক্রিয় সংগ্রহ
- **Application Performance Monitoring (APM)**: সফটওয়্যার পারফরম্যান্স মনিটরিং
- **Log Analytics**: লগ ডেটা সংগ্রহ এবং বিশ্লেষণের সার্ভিস
- **Alert Rules**: মেট্রিক্স বা শর্তের উপর ভিত্তি করে স্বয়ংক্রিয় নোটিফিকেশন

### ডিপ্লয়মেন্ট টার্মস
- **Blue-Green Deployment**: শূন্য-ডাউনটাইম ডিপ্লয়মেন্ট কৌশল
- **Canary Deployment**: ব্যবহারকারীর উপসেটের উপর ধাপে ধাপে রোলআউট
- **Rolling Update**: অ্যাপ্লিকেশন ইনস্ট্যান্সগুলোর ধারাবাহিক প্রতিস্থাপন
- **Rollback**: পূর্ববর্তী অ্যাপ্লিকেশন ভার্সনে ফিরিয়ে আনা
