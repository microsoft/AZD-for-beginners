- **🤖 AI টার্মস**: [অধ্যায় 2: AI-প্রথম উন্নয়ন](../docs/microsoft-foundry/microsoft-foundry-integration.md)

## ভূমিকা

এই বিস্তৃত শব্দকোষটি Azure Developer CLI এবং Azure ক্লাউড ডেভেলপমেন্টে ব্যবহৃত পদ, ধারণা এবং সংক্ষিপ্ত রূপগুলির সংজ্ঞা প্রদান করে। প্রযুক্তিগত ডকুমেন্টেশন বোঝা, ত্রুটি নির্ণয় করা এবং azd প্রকল্প ও Azure পরিষেবাগুলি সম্পর্কে কার্যকরভাবে যোগাযোগ করার জন্য এটি একটি অপরিহার্য রেফারেন্স।

## শেখার লক্ষ্য

এই শব্দকোষ ব্যবহার করে আপনি:
- Azure Developer CLI-এর মৌলিক পরিভাষা ও ধারণা বুঝতে পারবেন
- Azure ক্লাউড ডেভেলপমেন্টের শব্দভাণ্ডার ও প্রযুক্তিগত শব্দগুলো আয়ত্ত করতে পারবেন
- Infrastructure as Code এবং ডিপ্লয়মেন্ট পরিভাষা কার্যকরভাবে রেফারেন্স করতে পারবেন
- Azure সার্ভিস নাম, সংক্ষিপ্ত রূপ এবং তাদের উদ্দেশ্য বুঝতে পারবেন
- ট্রাবলশুটিং ও ডিবাগিং পরিভাষার সংজ্ঞা অ্যাক্সেস করতে পারবেন
- উন্নত Azure আর্কিটেকচার ও ডেভেলপমেন্ট ধারণা শিখতে পারবেন

## শেখার ফলাফল

নিয়মিত এই শব্দকোষ রেফারেন্স করলে আপনি সক্ষম হবেন:
- সঠিক Azure Developer CLI পরিভাষা ব্যবহার করে কার্যকরভাবে যোগাযোগ করতে
- প্রযুক্তিগত ডকুমেন্টেশন ও ত্রুটি বার্তাগুলো আরও পরিষ্কারভাবে বুঝতে
- আত্মবিশ্বাসের সাথে Azure সার্ভিস ও ধারণাগুলো নেভিগেট করতে
- উপযুক্ত প্রযুক্তিগত শব্দভাণ্ডার ব্যবহার করে সমস্যাগুলো ট্রাবলশুট করতে
- সঠিক প্রযুক্তিগত ভাষা নিয়ে টিম আলোচনায় অংশগ্রহণ করতে
- সুসংহতভাবে আপনার Azure ক্লাউড ডেভেলপমেন্ট জ্ঞান বৃদ্ধি করতে

## A

**ARM টেমপ্লেট**  
Azure Resource Manager template। JSON-ভিত্তিক Infrastructure as Code ফরম্যাট যা Azure রিসোর্সগুলোকে ঘোষণামূলকভাবে সংজ্ঞায়িত এবং ডিপ্লয় করার জন্য ব্যবহৃত হয়।

**App Service**  
ইনফ্রাস্ট্রাকচার ম্যানেজ না করেই ওয়েব অ্যাপ্লিকেশন, REST API এবং মোবাইল বেকএন্ড হোস্ট করার জন্য Azure-এর platform-as-a-service (PaaS) অফারিং।

**Application Insights**  
অ্যাপ্লিকেশন পারফরম্যান্স, উপলভ্যতা এবং ব্যবহার সম্পর্কে গভীর অন্তর্দৃষ্টি প্রদানকারী Azure-এর application performance monitoring (APM) সেবা।

**Azure CLI**  
Azure রিসোর্স ম্যানেজ করার জন্য কমান্ড-লাইন ইন্টারফেস। প্রমাণীকরণ এবং কিছু অপারেশনের জন্য azd দ্বারা ব্যবহৃত হয়।

**Azure Developer CLI (azd)**  
ডেভেলপার-কেন্দ্রিক কমান্ড-লাইন টুল যা টেমপলেট এবং Infrastructure as Code ব্যবহার করে Azure-এ অ্যাপ্লিকেশন তৈরী ও ডিপ্লয় করার প্রক্রিয়া ত্বরান্বিত করে।

**azure.yaml**  
কোনো azd প্রকল্পের প্রধান কনফিগারেশন ফাইল যা সার্ভিস, ইনফ্রাস্ট্রাকচার এবং ডিপ্লয়মেন্ট হুকগুলো সংজ্ঞায়িত করে।

**Azure Resource Manager (ARM)**  
Azure-এর ডিপ্লয়মেন্ট এবং ম্যানেজমেন্ট সার্ভিস যা রিসোর্স তৈরি, আপডেট এবং মুছার জন্য একটি ম্যানেজমেন্ট লেয়ার প্রদান করে।

## B

**Bicep**  
Microsoft কর্তৃক বিকশিত ডোমেইন-নির্দিষ্ট ভাষা (DSL) যা Azure রিসোর্স ডিপ্লয়মেন্টের জন্য ব্যবহৃত হয়। ARM টেমপ্লেটগুলোর তুলনায় সহজ সিনট্যাক্স প্রদান করে এবং ARM-এ কম্পাইল হয়।

**Build**  
সোর্স কোড কম্পাইল করা, ডিপেন্ডেন্সি ইনস্টল করা এবং অ্যাপ্লিকেশনগুলোকে ডিপ্লয়মেন্টের জন্য প্রস্তুত করার প্রক্রিয়া।

**Blue-Green Deployment**  
দুইটি অভিন্ন প্রোডাকশন পরিবেশ (blue এবং green) ব্যবহার করে ডাউনটাইম এবং ঝুঁকি কমানোর ডিপ্লয়মেন্ট কৌশল।

## C

**Container Apps**  
জটিল ইনফ্রাস্ট্রাকচার ম্যানেজ না করেই কন্টেইনারাইজড অ্যাপ্লিকেশন চালানোর জন্য Azure-এর serverless container সার্ভিস।

**CI/CD**  
Continuous Integration/Continuous Deployment। কোড পরিবর্তন একত্রিত করা এবং অ্যাপ্লিকেশন ডিপ্লয় করার স্বয়ংক্রিয় পদ্ধতি।

**Cosmos DB**  
Azure-এর গ্লোবালি বিতরণকৃত, মাল্টি-মডেল ডাটাবেস সার্ভিস যা থ্রুপুট, লেটেন্সি, উপলভ্যতা এবং কনসিস্টেন্সির জন্য বিস্তৃত SLA প্রদান করে।

**Configuration**  
অ্যাপ্লিকেশন আচরণ এবং ডিপ্লয়মেন্ট অপশন নিয়ন্ত্রণ করার সেটিংস এবং প্যারামিটারসমূহ।

## D

**Deployment**  
টার্গেট ইনফ্রাস্ট্রাকচারে অ্যাপ্লিকেশন এবং তাদের ডিপেন্ডেন্সিগুলো ইনস্টল ও কনফিগার করার প্রক্রিয়া।

**Docker**  
কন্টেইনারাইজেশন প্রযুক্তি ব্যবহার করে অ্যাপ্লিকেশন ডেভেলপ, শিপ এবং রান করার প্ল্যাটফর্ম।

**Dockerfile**  
একটি টেক্সট ফাইল যা Docker কন্টেইনার ইমেজ বিল্ড করার জন্য নির্দেশাবলী ধারণ করে।

## E

**Environment**  
ডিপ্লয়মেন্ট টার্গেট যা আপনার অ্যাপ্লিকেশনের একটি নির্দিষ্ট ইনস্ট্যান্সকে প্রতিনিধিত্ব করে (যেমন, development, staging, production)।

**Environment Variables**  
কী-ভ্যালু জোড়া হিসেবে সংরক্ষিত কনফিগারেশন মান যা অ্যাপ্লিকেশন রUNTIME-এ অ্যাক্সেস করতে পারে।

**Endpoint**  
URL বা নেটওয়ার্ক ঠিকানা যেখানে কোনো অ্যাপ্লিকেশন বা সার্ভিস অ্যাক্সেস করা যায়।

## F

**Function App**  
ইনফ্রাস্ট্রাকচার ম্যানেজ না করেই ইভেন্ট-চালিত কোড চালানোর জন্য Azure-এর serverless compute সার্ভিস।

## G

**GitHub Actions**  
GitHub রেপোজিটরির সাথে ইন্টিগ্রেটেড CI/CD প্ল্যাটফর্ম যা ওয়ার্কফ্লো অটোমেট করতে ব্যবহৃত হয়।

**Git**  
সোর্স কোডে পরিবর্তন ট্র্যাক করার জন্য ব্যবহৃত বিতরণকৃত ভার্সন কন্ট্রোল সিস্টেম।

## H

**Hooks**  
ডিপ্লয়মেন্ট লাইফসাইকেলের নির্দিষ্ট পয়েন্টে চলমান কাস্টম স্ক্রিপ্ট বা কমান্ড (preprovision, postprovision, predeploy, postdeploy)।

**Host**  
প্রকারভেদে Azure সার্ভিস যেখানে একটি অ্যাপ্লিকেশন ডিপ্লয় করা হবে (উদাহরণস্বরূপ, appservice, containerapp, function)।

## I

**Infrastructure as Code (IaC)**  
ইনফ্রাস্ট্রাকচার ম্যানেজ করার প্র্যাকটিস যেখানে ম্যানুয়াল প্রক্রিয়ার বদলে কোডের মাধ্যমে ইনফ্রাস্ট্রাকচার সংজ্ঞায়িত ও পরিচালিত হয়।

**Init**  
সাধারণত একটি টেমপ্লেট থেকে নতুন azd প্রকল্প আরম্ভ করার প্রক্রিয়া।

## J

**JSON**  
JavaScript Object Notation। কনফিগারেশন ফাইল এবং API রেসপন্সের জন্য সাধারণত ব্যবহৃত ডেটা বিনিময় ফরম্যাট।

**JWT**  
JSON Web Token। পক্ষগুলোর মধ্যে নিরাপদভাবে তথ্য প্রেরণের জন্য ব্যবহৃত স্ট্যান্ডার্ড যা একটি JSON অবজেক্ট হিসেবে থাকে।

## K

**Key Vault**  
সিক্রেট, কী এবং সার্টিফিকেটগুলি নিরাপদভাবে সংরক্ষণ ও পরিচালনা করার জন্য Azure-এর সার্ভিস।

**Kusto Query Language (KQL)**  
Azure Monitor, Application Insights এবং অন্যান্য Azure সার্ভিসে ডেটা বিশ্লেষণের জন্য ব্যবহৃত কুয়েরি ভাষা।

## L

**Load Balancer**  
একাধিক সার্ভার বা ইনস্ট্যান্সের মধ্যে ইনকামিং নেটওয়ার্ক ট্রাফিক বিতরণ করার সার্ভিস।

**Log Analytics**  
ক্লাউড এবং অন-প্রিমাইস পরিবেশ থেকে টেলিমেট্রি ডেটা সংগ্রহ, বিশ্লেষণ এবং কার্যকর করার জন্য Azure সার্ভিস।

## M

**Managed Identity**  
Azure সার্ভিসগুলিকে অন্যান্য Azure সেবায় প্রমাণীকরণ করার জন্য স্বয়ংক্রিয়ভাবে পরিচালিত একটি পরিচয় প্রদান করার বৈশিষ্ট্য।

**Microservices**  
ধারণাগত পদ্ধতি যেখানে অ্যাপ্লিকেশনগুলো ছোট, স্বতন্ত্র সার্ভিসের সংগ্রহ হিসেবে নির্মিত হয়।

**Monitor**  
অ্যাপ্লিকেশন ও ইনফ্রাস্ট্রাকচার জুড়ে ফুল-স্ট্যাক অবজারভেবিলিটি প্রদান করে এমন Azure-এর ইউনিফায়েড মনিটরিং সলিউশন।

## N

**Node.js**  
Chrome-এর V8 JavaScript ইঞ্জিনের উপর নির্মিত সার্ভার-সাইড অ্যাপ্লিকেশন তৈরির জন্য JavaScript runtime।

**npm**  
Node.js-এর জন্য প্যাকেজ ম্যানেজার যা ডিপেন্ডেন্সি ও প্যাকেজগুলো পরিচালনা করে।

## O

**Output**  
ইনফ্রাস্ট্রাকচার ডিপ্লয়মেন্ট থেকে ফেরত দেওয়া মান যা অ্যাপ্লিকেশন বা অন্য রিসোর্সগুলো দ্বারা ব্যবহার করা যেতে পারে।

## P

**Package**  
অ্যাপ্লিকেশন কোড ও ডিপেন্ডেন্সিগুলো ডিপ্লয়মেন্টের জন্য প্রস্তুত করার প্রক্রিয়া।

**Parameters**  
ডিপ্লয়মেন্ট কাস্টমাইজ করার জন্য ইনফ্রাস্ট্রাকচার টেমপ্লেটে পাস করা ইনপুট মানগুলো।

**PostgreSQL**  
ওপেন-সোর্স রিলেশনাল ডাটাবেস সিস্টেম যা Azure-এ ম্যানেজড সার্ভিস হিসেবে সমর্থিত।

**Provisioning**  
ইনফ্রাস্ট্রাকচার টেমপ্লেটে সংজ্ঞায়িত Azure রিসোর্সগুলো তৈরি এবং কনফিগার করার প্রক্রিয়া।

## Q

**Quota**  
কোনো Azure সাবস্ক্রিপশন বা অঞ্চলে তৈরি করা যেতে পারা রিসোর্সের পরিমাণের সীমা।

## R

**Resource Group**  
একই লাইফসাইকেল, পারমিশন এবং নীতিমালার শেয়ার করা Azure রিসোর্সগুলোর জন্য লজিক্যাল কনটেইনার।

**Resource Token**  
ডেপ্লয়মেন্ট জুড়ে রিসোর্স নামগুলিকে ইউনিক রাখতে azd দ্বারা জেনারেট করা ইউনিক স্ট্রিং।

**REST API**  
HTTP পদ্ধতি ব্যবহার করে নেটওয়ার্কেড অ্যাপ্লিকেশন ডিজাইনের আর্কিটেকচারাল স্টাইল।

**Rollback**  
একটি অ্যাপ্লিকেশন বা ইনফ্রাস্ট্রাকচার কনফিগারেশনের পূর্ববর্তী ভার্সনে ফিরে যাওয়ার প্রক্রিয়া।

## S

**Service**  
azure.yaml-এ সংজ্ঞায়িত আপনার অ্যাপ্লিকেশনের একটি উপাদান (উদাহরণ: ওয়েব ফ্রন্টএন্ড, API ব্যাকএন্ড, ডাটাবেস)।

**SKU**  
Stock Keeping Unit। Azure রিসোর্সগুলোর বিভিন্ন সার্ভিস স্তর বা পারফরম্যান্স লেভেল উপস্থাপন করে।

**SQL Database**  
Microsoft SQL Server ভিত্তিক Azure-এর ম্যানেজড রিলেশনাল ডাটাবেস সার্ভিস।

**Static Web Apps**  
সোর্স কোড রেপোজিটরি থেকে ফুল-স্ট্যাক ওয়েব অ্যাপ্লিকেশন বিল্ড ও ডিপ্লয় করার জন্য Azure সার্ভিস।

**Storage Account**  
ব্লব, ফাইল, কিউ এবং টেবিলসহ ডেটা অবজেক্টের জন্য ক্লাউড স্টোরেজ প্রদানকারী Azure সার্ভিস।

**Subscription**  
রিসোর্স গ্রুপ এবং রিসোর্সগুলো ধারণকারী Azure অ্যাকাউন্ট কনটেইনার, যা বিলিং এবং অ্যাক্সেস ম্যানেজমেন্টের সঙ্গে যুক্ত।

## T

**Template**  
প্রি-বিল্ট প্রকল্প স্ট্রাকচার যা সাধারণ পরিস্থিতির জন্য অ্যাপ্লিকেশন কোড, ইনফ্রাস্ট্রাকচার সংজ্ঞা এবং কনফিগারেশন ধারণ করে।

**Terraform**  
ওপেন-সোর্স Infrastructure as Code টুল যা Azure সহ একাধিক ক্লাউড প্রোভাইডার সমর্থন করে।

**Traffic Manager**  
গ্লোবাল Azure রিজিয়নগুলোর মধ্যে ট্রাফিক বিতরণ করার জন্য Azure-এর DNS-ভিত্তিক ট্রাফিক লোড ব্যালান্সার।

## U

**URI**  
Uniform Resource Identifier। কোনো নির্দিষ্ট রিসোর্সকে চিহ্নিত করার স্ট্রিং।

**URL**  
Uniform Resource Locator। URI-এর একটি ধরন যা নির্দেশ করে কোথায় একটি রিসোর্স অবস্থিত এবং কিভাবে তা পুনরুদ্ধার করা যাবে।

## V

**Virtual Network (VNet)**  
Azure-এ প্রাইভেট নেটওয়ার্কগুলোর জন্য মৌলিক বিল্ডিং ব্লক, যা আইসোলেশন এবং সেগমেন্টেশন প্রদান করে।

**VS Code**  
Visual Studio Code। চমৎকার Azure এবং azd ইন্টিগ্রেশনে সমৃদ্ধ জনপ্রিয় কোড এডিটর।

## W

**Webhook**  
নির্দিষ্ট ঘটনা ঘটলে ট্রিগার হওয়া HTTP কলব্যাক, যা সাধারণত CI/CD পাইপলাইনে ব্যবহৃত হয়।

**What-if**  
Azure-এর একটি বৈশিষ্ট্য যা ডিপ্লয়মেন্ট বাস্তবায়ন না করেই দেখায় কি পরিবর্তনগুলোর সম্ভাবনা আছে।

## Y

**YAML**  
YAML Ain't Markup Language। azure.yaml-এর মতো কনফিগারেশন ফাইলের জন্য ব্যবহৃত মানব-পঠনযোগ্য ডেটা সিরিয়ালাইজেশন স্ট্যান্ডার্ড।

## Z

**Zone**  
Azure অঞ্চলের ভেতরে শারীরিকভাবে পৃথক অবস্থান যা রিডান্ডেন্সি এবং উচ্চ উপলভ্যতা প্রদান করে।

## প্রসঙ্গ-নির্দিষ্ট টার্মসমূহ

### ডেভেলপমেন্ট টার্মস
- **হট রিলোড**: ডেভেলপমেন্টের সময় পুনরায় শুরু না করে স্বয়ংক্রিয়ভাবে অ্যাপ্লিকেশন আপডেট করা
- **বিল্ড পাইপলাইন**: কোড বিল্ড এবং টেস্ট করার জন্য অটোমেটেড প্রক্রিয়া
- **ডিপ্লয়মেন্ট স্লট**: App Service-এর ভিতরে একটি স্টেজিং পরিবেশ
- **Environment Parity**: ডেভেলপমেন্ট, স্টেজিং এবং প্রোডাকশন পরিবেশকে সমজাতীয় রাখা

### সিকিউরিটি টার্মস
- **Managed Identity**: স্বয়ংক্রিয় ক্রেডেনশিয়াল ম্যানেজমেন্ট প্রদানকারী Azure বৈশিষ্ট্য
- **Key Vault**: সিক্রেট, কী এবং সার্টিফিকেটের জন্য নিরাপদ স্টোরেজ
- **RBAC**: Azure রিসোর্সগুলোর জন্য রোল-ভিত্তিক অ্যাক্সেস কন্ট্রোল
- **Network Security Group**: নেটওয়ার্ক ট্রাফিক নিয়ন্ত্রণের জন্য ভার্চুয়াল ফায়ারওয়াল

### মনিটরিং টার্মস
- **Telemetry**: মাপ এবং ডেটা স্বয়ংক্রিয় সংগ্রহ
- **Application Performance Monitoring (APM)**: সফটওয়্যারের পারফরম্যান্স মনিটরিং
- **Log Analytics**: লগ ডেটা সংগ্রহ ও বিশ্লেষণের সার্ভিস
- **Alert Rules**: মেট্রিক্স বা শর্তের ভিত্তিতে স্বয়ংক্রিয় নোটিফিকেশন

### ডিপ্লয়মেন্ট টার্মস
- **Blue-Green Deployment**: জিরো-ডাউনটাইম ডিপ্লয়মেন্ট কৌশল
- **Canary Deployment**: ব্যবহারকারীর একটি উপসেটের কাছে ধাপে ধাপে রোলআউট
- **Rolling Update**: অ্যাপ্লিকেশন ইনস্ট্যান্সগুলো ক্ৰমশঃ পরিবর্তন করা
- **Rollback**: পূর্বের অ্যাপ্লিকেশন ভার্সনে ফিরে যাওয়া
