# অধ্যায় ৬: প্রি-ডিপ্লয়মেন্ট পরিকল্পনা ও যাচাইকরণ

**📚 কোর্স**: [AZD শুরুকারীদের জন্য](../../README.md) | **⏱️ সময়কাল**: ১ ঘন্টা | **⭐ জটিলতা**: মধ্যম

---

## ওভারভিউ

এই অধ্যায়টি আপনার অ্যাপ্লিকেশন ডিপ্লয় করার আগে প্রয়োজনীয় পরিকল্পনা ও যাচাইকরণ ধাপগুলোকে কভার করে। সঠিক সক্ষমতা পরিকল্পনা, SKU নির্বাচন এবং প্রিফ্লাইট চেকের মাধ্যমে ব্যয়বহুল ভুল এড়ানো শিখুন।

> `azd 1.25.6`-এর বিরুদ্ধে যাচাই করা হয়েছে, জুন ২০২৬ এ।

## শেখার উদ্দেশ্য

এই অধ্যায় সম্পন্ন করার মাধ্যমে আপনি:
- ডিপ্লয়মেন্টের আগে প্রিফ্লাইট চেক চালাবেন
- সক্ষমতা পরিকল্পনা করবেন এবং রিসোর্সের চাহিদা অনুমান করবেন
- খরচ অপ্টিমাইজেশনের জন্য উপযুক্ত SKU নির্বাচন করবেন
- মনিটরিংয়ের জন্য Application Insights কনফিগার করবেন
- টিম সমন্বয় প্যাটার্নগুলো বুঝে উঠবেন

## 🚀 দ্রুত শুরু

```bash
# সাবস্ক্রিপশন কোটা পরীক্ষা করুন
az vm list-usage --location eastus --output table

# ডিপ্লয়মেন্টের পূর্বরূপ দেখুন (কোনো রিসোর্স তৈরি হবে না)
azd provision --preview

# Bicep সিনট্যাক্স যাচাই করুন
az bicep build --file infra/main.bicep

# পরিবেশ কনফিগারেশন পরীক্ষা করুন
azd env get-values
```

## ☑️ প্রি-ডিপ্লয়মেন্ট চেকলিস্ট

### `azd provision` এর আগে

- [ ] রিজিয়নভিত্তিক কোটা যাচাই করা আছে
- [ ] SKUs উপযুক্তভাবে নির্বাচন করা হয়েছে
- [ ] খরচের অনুমান পর্যালোচনা করা হয়েছে
- [ ] নামকরণের কনভেনশন সামঞ্জস্যপূর্ণ
- [ ] নিরাপত্তা/RBAC কনফিগার করা হয়েছে

### `azd deploy` এর আগে

- [ ] পরিবেশ ভেরিয়েবল সেট করা হয়েছে
- [ ] Key Vault-এ সিক্রেট আছে
- [ ] কানেকশন স্ট্রিংগুলি যাচাই করা হয়েছে
- [ ] হেলথ চেক কনফিগার করা হয়েছে
