# অধ্যায় 6: পূর্ব-মোতায়েন পরিকল্পনা ও যাচাই

**📚 কোর্স**: [AZD নবীনদের জন্য](../../README.md) | **⏱️ সময়কাল**: 1 ঘণ্টা | **⭐ জটিলতা**: মধ্যম

---

## সংক্ষিপ্ত বিবরণ

এই অধ্যায়টি আপনার অ্যাপ্লিকেশন মোতায়েন করার আগে প্রয়োজনীয় পরিকল্পনা এবং যাচাই ধাপগুলি কভার করে। সঠিক ক্ষমতা পরিকল্পনা, SKU নির্বাচন এবং প্রিফ্লাইট চেকের মাধ্যমে ব্যয়বহুল ভুলগুলো এড়াতে শিখুন।

## শিখনের উদ্দেশ্য

এই অধ্যায় সম্পন্ন করলে আপনি:
- মোতায়েনের আগে প্রিফ্লাইট চেক চালানো শিখবেন
- ক্ষমতা পরিকল্পনা এবং রিসোর্সের প্রয়োজন অনুমান করতে পারবেন
- ব্যয় অপ্টিমাইজেশনের জন্য উপযুক্ত SKU নির্বাচন করবেন
- মনিটরিং জন্য Application Insights কনফিগার করবেন
- দলের সমন্বয় প্যাটার্নগুলো বুঝতে পারবেন

## 🚀 দ্রুত শুরু

```bash
# সাবস্ক্রিপশন কোটা পরীক্ষা করুন
az vm list-usage --location eastus --output table

# ডিপ্লয়মেন্ট পূর্বদর্শন (কোনও রিসোর্স তৈরি হবে না)
azd provision --preview

# Bicep সিনট্যাক্স যাচাই করুন
az bicep build --file infra/main.bicep

# পরিবেশ কনফিগারেশন পরীক্ষা করুন
azd env get-values
```

## ☑️ পূর্ব-মোতায়েন চেকলিস্ট

### `azd provision` এর আগে

- [ ] রিজিয়নের কোটা যাচাই করা হয়েছে
- [ ] উপযুক্তভাবে SKUs নির্বাচন করা হয়েছে
- [ ] খরচের আনুমানিকতা পর্যালোচনা করা হয়েছে
- [ ] নামকরণ কনভেনশন সঙ্গতিপূর্ণ
- [ ] Security/RBAC কনফিগার করা হয়েছে

### `azd deploy` এর আগে

- [ ] পরিবেশ ভেরিয়েবল সেট করা হয়েছে
- [ ] Key Vault-এ সিক্রেট রয়েছে
- [ ] কানেকশন স্ট্রিং যাচাই করা হয়েছে
- [ ] হেলথ চেক কনফিগার করা হয়েছে
