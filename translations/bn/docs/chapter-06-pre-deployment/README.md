# অধ্যায় ৬: প্রি-ডিপ্লয়মেন্ট পরিকল্পনা এবং যাচাই

**📚 কোর্স**: [AZD For Beginners](../../README.md) | **⏱️ সময়কাল**: ১ ঘণ্টা | **⭐ জটিলতা**: মধ্যবর্তী

---

## ওভারভিউ

এই অধ্যায়ে আপনার অ্যাপ্লিকেশন ডিপ্লয়মেন্টের আগে অপরিহার্য পরিকল্পনা এবং যাচাই ধাপগুলি আলোচনা করা হয়েছে। সঠিক ক্ষমতা পরিকল্পনা, SKU নির্বাচনের মাধ্যমে এবং প্রিফ্লাইট চেক চালিয়ে ব্যয়বহুল ভুল এড়ানো শিখুন।

> মার্চ ২০২৬-এ `azd 1.23.12` এর বিরুদ্ধে যাচাইকৃত।

## শেখার উদ্দেশ্য

এই অধ্যায় সম্পন্ন করার মাধ্যমে আপনি পারবেন:
- ডিপ্লয়মেন্টের আগে প্রিফ্লাইট চেক চালানো
- ক্ষমতা পরিকল্পনা করা এবং সম্পদ প্রয়োজনীয়তা অনুমান করা
- খরচ অপ্টিমাইজেশনের জন্য উপযুক্ত SKU নির্বাচন করা
- মনিটরিংয়ের জন্য অ্যাপ্লিকেশন ইনসাইটস কনফিগারেশন করা
- দলের সমন্বয় প্যাটার্নগুলো বুঝতে

## 🚀 দ্রুত শুরু

```bash
# সাবস্ক্রিপশন কোটা পরীক্ষা করুন
az vm list-usage --location eastus --output table

# ডিপ্লয়মেন্ট প্রিভিউ (কোনো রিসোর্স তৈরি হয়নি)
azd provision --preview

# Bicep সিনট্যাক্স যাচাই করুন
az bicep build --file infra/main.bicep

# পরিবেশ কনফিগারেশন যাচাই করুন
azd env get-values
```

## ☑️ প্রি-ডিপ্লয়মেন্ট চেকলিস্ট

### `azd provision` এর আগে

- [ ] অঞ্চলের জন্য কোটা যাচাই করা হয়েছে
- [ ] SKU যথাযথভাবে নির্বাচন করা হয়েছে
- [ ] খরচ অনুমান পর্যালোচনা করা হয়েছে
- [ ] নামকরণ রীতিনীতির সাথে সামঞ্জস্যপূর্ণ
- [ ] নিরাপত্তা/RBAC কনফিগার করা হয়েছে

### `azd deploy` এর আগে

- [ ] পরিবেশ ভেরিয়েবল সেট করা হয়েছে
- [ ] কী ভল্টে সিক্রেটস রাখা হয়েছে
- [ ] সংযোগ স্ট্রিং যাচাই করা হয়েছে
- [ ] স্বাস্থ্য পরীক্ষা কনফিগার করা হয়েছে
