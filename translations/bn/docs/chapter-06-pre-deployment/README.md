# চ্যাপ্টার ৬: প্রি-ডিপ্লয়মেন্ট পরিকল্পনা ও যাচাইকরণ

**📚 কোর্স**: [AZD ফর বিগিনারস](../../README.md) | **⏱️ সময়কাল**: ১ ঘণ্টা | **⭐ জটিলতা**: মাঝারি

---

## ওভারভিউ

এই চ্যাপ্টারটি আপনার অ্যাপ্লিকেশন ডিপ্লয়মেন্টের আগে আবশ্যক পরিকল্পনা ও যাচাইকরণের ধাপগুলি কভার করে। সঠিক ক্যাপাসিটি পরিকল্পনা, SKU নির্বাচন এবং প্রিফ্লাইট চেকের মাধ্যমে ব্যয়বহুল ভুল এড়ানো শিখুন।

> `azd 1.27.1` অনুসারে জুলাই ২০২৬ এ যাচাই করা হয়েছে।

## শেখার লক্ষ্যসমূহ

এই চ্যাপ্টার সম্পূর্ণ করার মাধ্যমে আপনি করবেন:
- ডিপ্লয়মেন্টের আগে প্রিফ্লাইট চেক চালানো
- ক্যাপাসিটি পরিকল্পনা করা এবং রিসোর্স প্রয়োজনীয়তা নির্ধারণ
- খরচ অপ্টিমাইজেশনের জন্য উপযুক্ত SKU নির্বাচন করা
- মনিটরিংয়ের জন্য অ্যাপ্লিকেশন ইনসাইটস কনফিগার করা
- দল সমন্বয় প্যাটার্ন বোঝা

## 🚀 দ্রুত শুরু

```bash
# সাবস্ক্রিপশন কোটা পরীক্ষা করুন
az vm list-usage --location eastus --output table

# ডিপ্লয়মেন্ট পূর্বরূপ দেখুন (কোনো রিসোর্স তৈরি হবে না)
azd provision --preview

# Bicep সিনট্যাক্স যাচাই করুন
az bicep build --file infra/main.bicep

# পরিবেশ কনফিগারেশন পরীক্ষা করুন
azd env get-values
```

## ☑️ প্রি-ডিপ্লয়মেন্ট চেকলিস্ট

### `azd provision` এর আগে

- [ ] কোটা অঞ্চলের জন্য নিশ্চিত করা হয়েছে
- [ ] সঠিক SKU নির্বাচন করা হয়েছে
- [ ] খরচ অনুমান পর্যালোচনা করা হয়েছে
- [ ] নামে নিয়মাবলী সামঞ্জস্যপূর্ণ
- [ ] নিরাপত্তা/RBAC কনফিগার করা হয়েছে

### `azd deploy` এর আগে

- [ ] পরিবেশ ভেরিয়েবল সেট করা হয়েছে
- [ ] সিক্রেটস কী ভল্টে সংরক্ষিত
- [ ] সংযোগ স্ট্রিং যাচাই করা হয়েছে
- [ ] হেলথ চেক কনফিগার করা হয়েছে
