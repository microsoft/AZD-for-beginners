# অধ্যায় 6: প্রি-ডিপ্লয়মেন্ট পরিকল্পনা ও যাচাইকরণ

**📚 কোর্স**: [AZD For Beginners](../../README.md) | **⏱️ সময়কাল**: 1 ঘন্টা | **⭐ জটিলতা**: মধ্যবর্তী

---

## সংক্ষিপ্ত বিবরণ

এই অধ্যায়টি আপনার অ্যাপ্লিকেশন ডিপ্লয় করার আগে প্রয়োজনীয় পরিকল্পনা এবং যাচাইকরণ ধাপগুলি কভার করে। সঠিক ক্যাপাসিটি পরিকল্পনা, SKU নির্বাচনের এবং প্রিফ্লাইট চেকগুলির মাধ্যমে খরচসাপেক্ষ ভুল এড়ানো শিখুন।

## শিখনের লক্ষ্য

এই অধ্যায় সম্পন্ন করলে, আপনি:
- ডিপ্লয়মেন্টের আগে প্রিফ্লাইট চেক চালাবেন
- ক্যাপাসিটি পরিকল্পনা করবেন এবং রিসোর্স চাহিদা অনুমান করবেন
- খরচ অপ্টিমাইজেশনের জন্য উপযুক্ত SKU নির্বাচন করবেন
- মনিটরিংয়ের জন্য Application Insights কনফিগার করবেন
- টিম সমন্বয় প্যাটার্নগুলো বোঝবেন

## 🚀 দ্রুত শুরু

```bash
# সাবস্ক্রিপশন কোটা পরীক্ষা করুন
az vm list-usage --location eastus --output table

# ডেপ্লয়মেন্ট প্রিভিউ (কোনো রিসোর্স তৈরি হবে না)
azd provision --preview

# Bicep সিনট্যাক্স যাচাই করুন
az bicep build --file infra/main.bicep

# পরিবেশ কনফিগারেশন পরীক্ষা করুন
azd env get-values
```

## ☑️ প্রি-ডিপ্লয়মেন্ট চেকলিস্ট

### `azd provision`-এর আগে

- [ ] অঞ্চলের জন্য কোটা যাচাই করা হয়েছে
- [ ] উপযুক্ত SKU নির্বাচন করা হয়েছে
- [ ] খরচের অনুমান পর্যালোচনা করা হয়েছে
- [ ] নামকরণ কনভেনশন সামঞ্জস্যপূর্ণ
- [ ] Security/RBAC কনফিগার করা হয়েছে

### `azd deploy`-এর আগে

- [ ] পরিবেশ ভেরিয়েবল সেট করা হয়েছে
- [ ] Key Vault-এ সিক্রেট রাখা হয়েছে
- [ ] কানেকশন স্ট্রিং যাচাই করা হয়েছে
- [ ] হেলথ চেক কনফিগার করা হয়েছে
