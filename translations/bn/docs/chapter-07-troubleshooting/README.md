# অধ্যায় ৭: সমস্যা সমাধান ও ডিবাগিং

**📚 কোর্স**: [AZD ফর বিগিনার্স](../../README.md) | **⏱️ সময়কাল**: ১-১.৫ ঘন্টা | **⭐ জটিলতা**: মধ্যম

---

## পর্যালোচনা

এই অধ্যায়টি আপনাকে Azure Developer CLI ব্যবহার করার সময় সাধারণ সমস্যাগুলো নির্ণয় এবং সমাধান করতে সাহায্য করবে। ডিপ্লয়মেন্ট ব্যর্থতা থেকে AI-সংক্রান্ত সমস্যা পর্যন্ত।

> `azd 1.27.1` এর বিরুদ্ধে জুলাই ২০২৬-এ যাচাই করা হয়েছে।

## শেখার লক্ষ্যসমূহ

এই অধ্যায়টি সম্পন্ন করে, আপনি:
- সাধারণ AZD ডিপ্লয়মেন্ট ব্যর্থতা নির্ণয় করতে পারবেন
- প্রমাণীকরণ এবং অনুমতি সমস্যাগুলো ডিবাগ করতে পারবেন
- AI সেবা সংযোগ সমস্যাগুলো সমাধান করতে পারবেন
- সমস্যা সমাধানের জন্য Azure Portal এবং CLI ব্যবহার করতে পারবেন

## 🚨 দ্রুত সমাধান

### প্রমাণীকরণ সমস্যা
```bash
# AZD কার্যপ্রবাহের জন্য প্রয়োজনীয়
azd auth login

# যদি আপনি সরাসরি Azure CLI কমান্ডও ব্যবহার করেন তবে এটি ঐচ্ছিক
az login

azd auth status
```

### প্রোভিশনিং ব্যর্থতা
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### রিসোর্স সংঘর্ষ
```bash
azd down --force --purge
azd env new different-name
azd up
```

### কোটার অতিরিক্ত ব্যবহার
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 ত্রুটি কোড রেফারেন্স

| ত্রুটি | কারণ | সমাধান |
|-------|-------|----------|
| `AuthenticationError` | লগইন করা হয়নি | `azd auth login` |
| `ResourceNotFound` | রিসোর্স অনুপস্থিত | রিসোর্স নামগুলো পরীক্ষা করুন |
| `QuotaExceeded` | সাবস্ক্রিপশন সীমা অতিরিক্ত | কোটা বৃদ্ধি অনুরোধ করুন |
| `InvalidTemplate` | Bicep সিনট্যাক্স ত্রুটি | `az bicep build` |
| `Conflict` | রিসোর্স বিদ্যমান | নতুন নাম ব্যবহার করুন বা মুছে দিন |
| `Forbidden` | অপর্যাপ্ত অনুমতি | RBAC ভূমিকা পরীক্ষা করুন |

## 🔄 রিসেট এবং পুনরুদ্ধার

```bash
# সফট রিসেট (সম্পদগুলি রাখুন, কোড পুনঃপ্রয়োগ করুন)

# হার্ড রিসেট (সবকিছু মুছে দিন, নতুন করে শুরু করুন)
azd down --force --purge
azd up
```
