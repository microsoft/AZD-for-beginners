# অধ্যায় ৭: সমস্যা সমাধান ও ডিবাগিং

**📚 কোর্স**: [AZD ফর বিগিনার্স](../../README.md) | **⏱️ সময়কাল**: ১-১.৫ ঘণ্টা | **⭐ জটিলতা**: মধ্যবর্তী

---

## ওভারভিউ

এই অধ্যায়টি আপনাকে Azure Developer CLI-তে কাজ করার সময় সাধারণ সমস্যা নির্ণয় এবং সমাধান করতে সাহায্য করবে। ডিপ্লয়মেন্ট ব্যর্থতা থেকে AI-সংক্রান্ত সমস্যাগুলো পর্যন্ত।

> মার্চ ২০২৬-এ `azd 1.23.12` এর বিপরীতে যাচাই করা হয়েছে।

## শেখার লক্ষ্যসমূহ

এই অধ্যায় সম্পন্ন করে, আপনি পারবেন:
- সাধারণ AZD ডিপ্লয়মেন্ট ব্যর্থতা নির্ণয় করতে
- প্রমাণীকরণ এবং অনুমতির সমস্যা ডিবাগ করতে
- AI সার্ভিস সংযোগ সমস্যাগুলো সমাধান করতে
- সমস্যা সমাধানের জন্য Azure পোর্টাল এবং CLI ব্যবহার করতে

## 🚨 দ্রুত সমাধান

### প্রমাণীকরণের সমস্যা
```bash
# AZD ওয়ার্কফ্লো জন্য আবশ্যক
azd auth login

# ঐচ্ছিক যদি আপনি সরাসরি Azure CLI কমান্ডও ব্যবহার করেন
az login

azd auth status
```

### প্রভিশনিং ব্যর্থতা
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### রিসোর্স সংঘর্ষ
```bash
azd down --force --purge
azd env new different-name
azd up
```

### কোটা অতিক্রম
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 এরর কোড রেফারেন্স

| এরর | কারণ | সমাধান |
|-------|-------|----------|
| `AuthenticationError` | লগ ইন করা নেই | `azd auth login` |
| `ResourceNotFound` | রিসোর্স অনুপস্থিত | রিসোর্স নাম যাচাই করুন |
| `QuotaExceeded` | সাবস্ক্রিপশন সীমা অতীত | কোটা বৃদ্ধির অনুরোধ দিন |
| `InvalidTemplate` | বাইসপ সিনট্যাক্স ত্রুটি | `az bicep build` |
| `Conflict` | রিসোর্স বিদ্যমান | নতুন নাম ব্যবহার করুন বা মুছুন |
| `Forbidden` | যথেষ্ট অনুমতি নেই | RBAC ভূমিকা যাচাই করুন |

## 🔄 রিসেট এবং পুনরুদ্ধার

```bash
# সফ্ট রিসেট (সম্পদ রাখা, কোড পুনরায় স্থাপন)
azd deploy --force

# হার্ড রিসেট (সব কিছু মুছে ফেলা, নতুনভাবে শুরু)
azd down --force --purge
azd up
```
