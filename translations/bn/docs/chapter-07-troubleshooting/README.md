# Chapter 7: ত্রুটিনির্ণয় ও ডিবাগিং

**📚 কোর্স**: [AZD নবীনদের জন্য](../../README.md) | **⏱️ সময়**: 1-1.5 ঘণ্টা | **⭐ কঠিনতা**: মধ্যম

---

## সংক্ষিপ্ত বিবরণ

এই অধ্যায়টি আপনাকে Azure Developer CLI-এর সাথে কাজ করার সময় সাধারণ সমস্যাগুলো নির্ণয় এবং সমাধান করতে সাহায্য করবে। ডিপ্লয়মেন্ট ব্যর্থতা থেকে এআই-নির্দিষ্ট সমস্যাগুলি পর্যন্ত।

## শিখন লক্ষ্য

এই অধ্যায়টি সম্পন্ন করার মাধ্যমে, আপনি:
- সাধারণ AZD ডিপ্লয়মেন্ট ব্যর্থতা নির্ণয় করতে পারবেন
- প্রমাণীকরণ ও অনুমতি সংক্রান্ত সমস্যা ডিবাগ করতে পারবেন
- AI সার্ভিস সংযোগজনিত সমস্যাগুলি সমাধান করতে পারবেন
- সমস্যা সমাধানের জন্য Azure Portal ও CLI ব্যবহার করতে পারবেন

## 🚨 দ্রুত সমাধান

### প্রমাণীকরণ সমস্যা
```bash
azd auth login
az login
azd auth whoami
```

### প্রোভিশনিং ব্যর্থতা
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### রিসোর্স সংঘাত
```bash
azd down --force --purge
azd env new different-name
azd up
```

### কোটা অতিক্রম
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 ত্রুটি কোড রেফারেন্স

| ত্রুটি | কারণ | সমাধান |
|-------|-------|----------|
| `AuthenticationError` | লগইন করা নেই | `azd auth login` |
| `ResourceNotFound` | রিসোর্স অনুপস্থিত | রিসোর্স নামগুলো পরীক্ষা করুন |
| `QuotaExceeded` | সাবস্ক্রিপশন সীমা | কোটা বৃদ্ধির অনুরোধ করুন |
| `InvalidTemplate` | Bicep সিনট্যাক্স ত্রুটি | `az bicep build` |
| `Conflict` | রিসোর্স আছে | নতুন নাম ব্যবহার করুন অথবা মুছুন |
| `Forbidden` | পর্যাপ্ত অনুমতি নেই | RBAC রোলগুলো পরীক্ষা করুন |

## 🔄 রিসেট এবং পুনরুদ্ধার

```bash
# সফট রিসেট (রিসোর্সগুলো রাখা হবে, কোড পুনরায় ডিপ্লয় করা হবে)
azd deploy --force

# হার্ড রিসেট (সবকিছু মুছে ফেলা হবে, নতুনভাবে শুরু করা হবে)
azd down --force --purge
azd up
```
