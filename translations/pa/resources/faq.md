## Getting Started

### Q: What is Azure Developer CLI (azd)?
**A**: Azure Developer CLI (azd) ਇੱਕ ਡਿਵੈਲਪਰ-ਕੇਂਦਰਿਤ ਕਮਾਂਡ-ਲਾਈਨ ਟੂਲ ਹੈ ਜੋ ਤੁਹਾਡੇ ਐਪਲੀਕੇਸ਼ਨ ਨੂੰ ਲੋਕਲ ਡਿਵੈਲਪਮੈਂਟ ਵਾਤਾਵਰਣ ਤੋਂ Azure ਤੱਕ ਲਿਜਾਣ ਦਾ ਸਮਾਂ ਤੇਜ਼ ਕਰਦਾ ਹੈ। ਇਹ ਟੈਂਪਲੇਟਾਂ ਰਾਹੀਂ ਸਰਵੋਤਮ ਅਭਿਆਸ ਮੁਹੱਈਆ ਕਰਦਾ ਹੈ ਅਤੇ ਪੂਰੇ ਡਿਪਲੋਇਮੈਂਟ ਲਾਈਫਸਾਈਕਲ ਵਿੱਚ ਸਹਾਇਤਾ ਕਰਦਾ ਹੈ।

### Q: How is azd different from Azure CLI?
**A**: 
- **Azure CLI**: Azure ਰਿਸੋਰਸ ਪ੍ਰਬੰਧਨ ਲਈ ਜਨਰਲ-ਪਪਰਪਜ਼ ਟੂਲ
- **azd**: ਐਪਲੀਕੇਸ਼ਨ ਡਿਪਲੋਇਮੈਂਟ ਵਰਕਫਲੋਜ਼ ਲਈ ਡਿਵੈਲਪਰ-ਕੇਂਦਰਿਤ ਟੂਲ
- azd ਅੰਦਰੂਨੀ ਤੌਰ 'ਤੇ Azure CLI ਦੀ ਵਰਤੋਂ ਕਰਦਾ ਹੈ ਪਰ ਆਮ ਡਿਵੈਲਪਮੈਂਟ ਸਨੈਰੀਓ ਲਈ ਉੱਚ-ਸਤਰ ਦੇ ਐਬਸਟ੍ਰੈਕਸ਼ਨਾਂ ਪ੍ਰਦਾਨ ਕਰਦਾ ਹੈ
- azd ਵਿੱਚ ਟੈਂਪਲੇਟ, ਵਾਤਾਵਰਣ ਪ੍ਰਬੰਧਨ, ਅਤੇ ਡਿਪਲੋਇਮੈਂਟ ਆਟੋਮੇਸ਼ਨ ਸ਼ਾਮਲ ਹਨ

### Q: Do I need Azure CLI installed to use azd?
**A**: ਹਾਂ, azd ਲਈ ਪ੍ਰਮਾਣੀਕਰਨ ਅਤੇ ਕੁਝ ਓਪਰੇਸ਼ਨਾਂ ਲਈ Azure CLI ਦੀ ਲੋੜ ਹੁੰਦੀ ਹੈ। ਪਹਿਲਾਂ Azure CLI ਇੰਸਟਾਲ ਕਰੋ, ਫਿਰ azd ਇੰਸਟਾਲ ਕਰੋ।

### Q: What programming languages does azd support?
**A**: azd ਭਾਸ਼ਾ-ਐਗਨੋਸਟਿਕ ਹੈ। ਇਹ ਨਾਲ ਕੰਮ ਕਰਦਾ ਹੈ:
- Node.js/JavaScript/TypeScript
- Python
- .NET/C#
- Java
- Go
- PHP
- Static websites
- Containerized applications

### Q: Can I use azd with existing projects?
**A**: ਹਾਂ! ਤੁਸੀਂ ਜਾਂ ਤਾਂ:
1. ਮੌਜੂਦਾ ਪ੍ਰੋਜੈਕਟਾਂ ਵਿੱਚ azd ਕੰਫਿਗਰੇਸ਼ਨ ਸ਼ਾਮਲ ਕਰਨ ਲਈ `azd init` ਦੀ ਵਰਤੋਂ ਕਰੋ
2. ਮੌਜੂਦਾ ਪ੍ਰੋਜੈਕਟਾਂ ਨੂੰ azd ਟੈਂਪਲੇਟ ਸਟਰੱਕਚਰ ਨਾਲ ਮੇਲ ਖਾਂਦੇ ਹੋਏ ਅਨੁਕੂਲ ਬਣਾਓ
3. ਆਪਣੇ ਮੌਜੂਦਾ ਆਰਕੀਟੈਕਚਰ ਦੇ ਆਧਾਰ 'ਤੇ ਕਸਟਮ ਟੈਂਪਲੇਟ ਬਣਾਓ

## Authentication & Access

### Q: How do I authenticate with Azure using azd?
**A**: `azd auth login` ਦੀ ਵਰਤੋਂ ਕਰੋ ਜੋ Azure ਪ੍ਰਮਾਣੀਕਰਨ ਲਈ ਬ੍ਰਾਉਜ਼ਰ ਵਿੰਡੋ ਖੋਲੇਗੀ। CI/CD ਸਨੈਰੀਓਜ਼ ਲਈ, ਸਰਵਿਸ ਪ੍ਰਿੰਸੀਪਲ ਜਾਂ ਮੈਨੇਜਡ ਆਈਡੈਂਟਿਟੀ ਦੀ ਵਰਤੋਂ ਕਰੋ।

### Q: Can I use azd with multiple Azure subscriptions?
**A**: ਹਾਂ। ਹਰ ਵਾਤਾਵਰਣ ਲਈ ਕਿਸ ਸਬਸਕ੍ਰਿਪਸ਼ਨ ਨੂੰ ਵਰਤਨਾ ਹੈ ਇਹ ਨਿਰਧਾਰਿਤ ਕਰਨ ਲਈ `azd env set AZURE_SUBSCRIPTION_ID <subscription-id>` ਦੀ ਵਰਤੋਂ ਕਰੋ।

### Q: What permissions do I need to deploy with azd?
**A**: ਆਮ ਤੌਰ 'ਤੇ ਤੁਹਾਨੂੰ ਲੋੜ ਹੁੰਦੀ ਹੈ:
- ਰਿਸੋਰਸ ਗਰੁੱਪ ਜਾਂ ਸਬਸਕ੍ਰਿਪਸ਼ਨ 'ਤੇ **Contributor** ਭੂਮਿਕਾ
- ਜੇ ਕਿਸੇ ਰਿਸੋਰਸ ਲਈ ਰੋਲ ਅਸਾਈਨਮੈਂਟ ਦੀ ਲੋੜ ਹੈ ਤਾਂ **User Access Administrator**
- ਵਿਸ਼ੇਸ਼ ਅਨੁਮਤੀਆਂ ਟੈਂਪਲੇਟ ਅਤੇ ਡਿਪਲੋਇਮੈਂਟ ਕੀਤੇ ਜਾ ਰਹੇ ਰਿਸੋਰਸਾਂ ਅਨੁਸਾਰ ਵੱਖ-ਵੱਖ ਹੋ ਸਕਦੀਆਂ ਹਨ

### Q: Can I use azd in CI/CD pipelines?
**A**: ਬਿਲਕੁਲ! azd CI/CD ਐਕਸਪੋਜ਼ ਕਰਨ ਲਈ ਡਿਜ਼ਾਇਨ ਕੀਤਾ ਗਿਆ ਹੈ। ਪ੍ਰਮਾਣੀਕਰਨ ਲਈ ਸਰਵਿਸ ਪ੍ਰਿੰਸੀਪਲ ਦੀ ਵਰਤੋਂ ਕਰੋ ਅਤੇ ਕੰਫਿਗਰੇਸ਼ਨ ਲਈ ਵਾਤਾਵਰਣ ਚਰਾਂ (environment variables) ਸੈੱਟ ਕਰੋ।

### Q: How do I handle authentication in GitHub Actions?
**A**: ਸਰਵਿਸ ਪ੍ਰਿੰਸੀਪਲ ਕ੍ਰੈਡੈਂਸ਼ਲਸ ਨਾਲ Azure Login action ਦੀ ਵਰਤੋਂ ਕਰੋ:
```yaml
- uses: azure/login@v1
  with:
    creds: ${{ secrets.AZURE_CREDENTIALS }}
- run: azd deploy --no-prompt
```

## Templates & Projects

### Q: Where can I find azd templates?
**A**: 
- ਅਧਿਕਾਰਿਕ ਟੈਂਪਲੇਟ: [Azure-Samples/awesome-azd](https://github.com/Azure-Samples/awesome-azd)
- ਕਮਿਊਨਿਟੀ ਟੈਂਪਲੇਟ: GitHub 'ਤੇ "azd-template" ਲਈ ਖੋਜ ਕਰੋ
- ਉਪਲਬਧ ਟੈਂਪਲੇਟ ਵੇਖਣ ਲਈ `azd template list` ਦੀ ਵਰਤੋਂ ਕਰੋ

### Q: How do I create a custom template?
**A**: 
1. ਮੌਜੂਦਾ ਟੈਂਪਲੇਟ ਸਟਰੱਕਚਰ ਨਾਲ ਸ਼ੁਰੂ ਕਰੋ
2. `azure.yaml`, infrastructure ਫਾਈਲਾਂ, ਅਤੇ ਐਪਲੀਕੇਸ਼ਨ ਕੋਡ ਵਿੱਚ ਸੋਧ ਕਰੋ
3. `azd up` ਨਾਲ ਪੂਰੀ ਤਰ੍ਹਾਂ ਟੈਸਟ ਕਰੋ
4. ਉਚਿਤ ਟੈਗਾਂ ਨਾਲ GitHub 'ਤੇ ਪ੍ਰਕਾਸ਼ਿਤ ਕਰੋ

### Q: Can I use azd without a template?
**A**: ਹਾਂ, ਮੌਜੂਦਾ ਪ੍ਰੋਜੈਕਟ ਵਿੱਚ ਲੋੜੀਂਦੇ ਕੰਫਿਗਰੇਸ਼ਨ ਫਾਈਲਾਂ ਬਣਾਉਣ ਲਈ `azd init` ਦੀ ਵਰਤੋਂ ਕਰੋ। ਤੁਹਾਨੂੰ `azure.yaml` ਅਤੇ infrastructure ਫਾਈਲਾਂ ਨੂੰ ਹੱਥੋਂ ਕਨਫਿਗਰ کرنا ਪਵੇਗਾ।

### Q: What's the difference between official and community templates?
**A**: 
- **ਅਧਿਕਾਰਿਕ ਟੈਂਪਲੇਟ**: Microsoft ਵੱਲੋਂ ਰੱਖ-ਰਖਾਅ ਕੀਤੇ ਜਾਂਦੇ ਹਨ, ਨਿਯਮਿਤ ਅਪਡੇਟ ਮਿਲਦੇ ਹਨ, ਵਿਸਤ੍ਰਿਤ ਦਸਤਾਵੇਜ਼ਕਰਨ
- **ਕਮਿਊਨਿਟੀ ਟੈਂਪਲੇਟ**: ਡਿਵੈਲਪਰਾਂ ਵੱਲੋਂ ਬਣਾਏ ਜਾਂਦੇ ਹਨ, ਵਿਸ਼ੇਸ਼ ਪ੍ਰਯੋਗ ਕੇਸ ਹੋ ਸਕਦੇ ਹਨ, ਗੁਣਵੱਤਾ ਅਤੇ ਮਰੰਟੇਨੈਂਸ ਵਿੱਚ ਫਰਕ ਹੋ ਸਕਦਾ ਹੈ

### Q: How do I update a template in my project?
**A**: ਟੈਂਪਲੇਟ ਆਪਣੇ ਆਪ ਅਪਡੇਟ ਨਹੀਂ ਹੁੰਦੇ। ਤੁਸੀਂ:
1. ਸੋਰਸ ਟੈਂਪਲੇਟ ਤੋਂ ਬਦਲਾਅ ਨੂੰ ਮੈਨੁਅਲੀ ਤੌਰ 'ਤੇ ਤੁਲਨਾ ਅਤੇ ਮਰਜ ਕਰੋ
2. ਅਪਡੇਟ ਕੀਤੇ ਟੈਂਪਲੇਟ ਨਾਲ `azd init` ਕਰਕੇ ਨਵਾਂ ਪ੍ਰਾਜੈਕਟ ਸ਼ੁਰੂ ਕਰੋ
3. ਅਪਡੇਟ ਕੀਤੇ ਟੈਂਪਲੇਟ ਤੋਂ ਵਿਸ਼ੇਸ਼ ਸੁਧਾਰ ਚੈਰੀ-ਪਿਕ ਕਰੋ

## Deployment & Infrastructure

### Q: What Azure services can azd deploy?
**A**: azd Bicep/ARM ਟੈਂਪਲੇਟਾਂ ਰਾਹੀਂ ਕਿਸੇ ਵੀ Azure ਸਰਵਿਸ ਨੂੰ ਡਿਪਲੋਇ ਕਰ ਸਕਦਾ ਹੈ, ਜਿਵੇਂ ਕਿ:
- App Services, Container Apps, Functions
- ਡੇਟਾਬੇਸ (SQL, PostgreSQL, Cosmos DB)
- Storage, Key Vault, Application Insights
- Networking, security, ਅਤੇ ਮਾਨੀਟਰਿੰਗ ਰਿਸੋਰਸ

### Q: Can I deploy to multiple regions?
**A**: ਹਾਂ, ਆਪਣੀਆਂ Bicep ਟੈਂਪਲੇਟਾਂ ਵਿੱਚ ਕਈ ਰੀਜਨ ਕਨਫਿਗਰ ਕਰੋ ਅਤੇ ਹਰ ਵਾਤਾਵਰਣ ਲਈ location ਪੈਰਾਮੀਟਰ ਨੂੰ ਢੰਗ ਨਾਲ ਸੈੱਟ ਕਰੋ।

### Q: How do I handle database schema migrations?
**A**: `azure.yaml` ਵਿੱਚ ਡਿਪਲੋਇਮੈਂਟ ਹੂਕਸ ਦੀ ਵਰਤੋਂ ਕਰੋ:
```yaml
hooks:
  postdeploy:
    posix:
      run: ./scripts/migrate-database.sh
    windows:
      run: ./scripts/migrate-database.ps1
```

### Q: Can I deploy only infrastructure without applications?
**A**: ਹਾਂ, ਸਿਰਫ infrastructure ਕੰਪੋਨੈਂਟਾਂ ਨੂੰ ਡਿਪਲੋਇ ਕਰਨ ਲਈ `azd provision` ਦੀ ਵਰਤੋਂ ਕਰੋ ਜੋ ਤੁਹਾਡੇ ਟੈਂਪਲੇਟਾਂ ਵਿੱਚ ਪਰਿਭਾਸ਼ਿਤ ਹਨ।

### Q: How do I deploy to existing Azure resources?
**A**: ਇਹ ਜਟਿਲ ਹੈ ਅਤੇ ਸਿੱਧਾ ਸਮਰਥਿਤ ਨਹੀਂ ਹੈ। ਤੁਸੀਂ:
1. ਮੌਜੂਦਾ ਰਿਸੋਰਸਾਂ ਨੂੰ ਆਪਣੇ Bicep ਟੈਂਪਲੇਟਾਂ ਵਿੱਚ ਇੰਪੋਰਟ ਕਰੋ
2. ਟੈਂਪਲੇਟਾਂ ਵਿੱਚ ਮੌਜੂਦਾ ਰਿਸੋਰਸ ਰੈਫਰੰਸਾਂ ਦੀ ਵਰਤੋਂ ਕਰੋ
3. ਰਿਸੋਰਸ ਬਣਾਉਣ ਜਾਂ ਉਨ੍ਹਾਂ ਦਾ ਰੈਫਰੰਸ ਕਰਨ ਲਈ ਟੈਂਪਲੇਟਾਂ ਨੂੰ ਸ਼ਰਤੀ (conditional) ਤਰੀਕੇ ਨਾਲ ਸੋਧੋ

### Q: Can I use Terraform instead of Bicep?
**A**: ਫਿਲਹਾਲ, azd ਮੁੱਖ ਤੌਰ 'ਤੇ Bicep/ARM ਟੈਂਪਲੇਟਾਂ ਨੂੰ ਸਹਿਯੋਗ ਦਿੰਦਾ ਹੈ। Terraform ਲਈ ਅਧਿਕਾਰਿਕ ਸਪੋਰਟ ਮੌਜੂਦ ਨਹੀਂ ਹੈ, ਹਾਲਾਂਕਿ ਕਮਿਊਨਿਟੀ ਹੱਲ ਹੋ ਸਕਦੇ ਹਨ।

## Configuration & Environments

### Q: How do I manage different environments (dev, staging, prod)?
**A**: ਵੱਖ-ਵੱਖ ਵਾਤਾਵਰਣ ਬਣਾਉਣ ਲਈ `azd env new <environment-name>` ਦੀ ਵਰਤੋਂ ਕਰੋ ਅਤੇ ਹਰ ਇੱਕ ਲਈ ਵੱਖ-ਵੱਖ ਸੈਟਿੰਗਾਂ ਕਨਫਿਗਰ ਕਰੋ:
```bash
azd env new development
azd env new staging  
azd env new production
```

### Q: Where are environment configurations stored?
**A**: ਤੁਹਾਡੇ ਪ੍ਰੋਜੈਕਟ ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ `.azure` ਫੋਲਡਰ ਵਿੱਚ। ਹਰ ਵਾਤਾਵਰਣ ਦੀ ਆਪਣੀ ਫੋਲਡਰ ਹੁੰਦੀ ਹੈ ਜਿਸ ਵਿੱਚ ਕਨਫਿਗਰੇਸ਼ਨ ਫਾਈਲਾਂ ਹੁੰਦੀਆਂ ਹਨ।

### Q: How do I set environment-specific configuration?
**A**: ਵਾਤਾਵਰਣ ਚਰਾਂ ਸੈੱਟ ਕਰਨ ਲਈ `azd env set` ਦੀ ਵਰਤੋਂ ਕਰੋ:
```bash
azd env set AZURE_LOCATION eastus
azd env set DATABASE_TIER Basic
```

### Q: Can I share environment configurations across team members?
**A**: `.azure` ਫੋਲਡਰ ਵਿੱਚ ਸੰਵੇਦਨਸ਼ੀਲ ਜਾਣਕਾਰੀ ਹੁੰਦੀ ਹੈ ਅਤੇ ਇਸਨੂੰ ਵਰਜਨ ਕੰਟਰੋਲ ਵਿੱਚ ਕਮੀਟ ਨਹੀਂ ਕੀਤਾ ਜਾਣਾ ਚਾਹੀਦਾ। ਇਸ ਦੀ ਥਾਂ:
1. ਲੋੜੀਂਦੇ ਵਾਤਾਵਰਣ ਚਰਾਂ ਦਾ ਦਸਤਾਵੇਜ਼ ਬਣਾਓ
2. ਵਾਤਾਵਰਣ ਸੈੱਟਅਪ ਕਰਨ ਲਈ ਡਿਪਲੋਇਮੈਂਟ ਸਕ੍ਰਿਪਟ ਵਰਤੋਂ
3. ਸੰਵੇਦਨਸ਼ੀਲ ਕਨਫਿਗਰੇਸ਼ਨ ਲਈ Azure Key Vault ਵਰਤੋਂ

### Q: How do I override template defaults?
**A**: ਟੈਂਪਲੇਟ ਪੈਰਾਮੀਟਰਾਂ ਨੂੰ ਅਨੁਕੂਲ ਕਰਨ ਵਾਲੇ ਵਾਤਾਵਰਣ ਚਰਾਂ ਸੈੱਟ ਕਰੋ:
```bash
azd env set LOCATION "West US 2"
azd env set SKU_NAME "B1"
```

## Troubleshooting

### Q: Why is `azd up` failing?
**A**: ਆਮ ਕਾਰਨ:
1. **ਪ੍ਰਮਾਣੀਕਰਨ ਸਮੱਸਿਆਵਾਂ**: `azd auth login` ਚਲਾਓ
2. **ਅਪਰਯਾਪਤ ਅਨੁਮਤੀਆਂ**: ਆਪਣੇ Azure ਰੋਲ ਅਸਾਈਨਮੈਂਟ ਚੈੱਕ ਕਰੋ
3. **ਰਿਸੋਰਸ ਨਾਮਕ ਟਕਰਾਅ**: AZURE_ENV_NAME ਬਦਲੋ
4. **ਕੋਟਾ/ਸਮਰੱਥਾ ਸਮੱਸਿਆਵਾਂ**: ਖੇਤਰੀ ਉਪਲਬਧਤਾ ਚੈੱਕ ਕਰੋ
5. **ਟੈਂਪਲੇਟ ਤਰੁੱਟੀਆਂ**: Bicep ਟੈਂਪਲੇਟਾਂ ਨੂੰ ਵੈਧ ਕਰੋ

### Q: How do I debug deployment failures?
**A**: 
1. ਵਿਸਥਾਰਪੂਰਕ ਆਉਟਪੁੱਟ ਲਈ `azd deploy --debug` ਵਰਤੋ
2. Azure ਪੋਰਟਲ ਡਿਪਲੋਇਮੈਂਟ ਇਤਿਹਾਸ ਚੈੱਕ ਕਰੋ
3. Azure ਪੋਰਟਲ ਵਿੱਚ Activity Log ਰਿਵਿਊ ਕਰੋ
4. ਮੌਜੂਦਾ ਵਾਤਾਵਰਣ ਦੀ ਸਥਿਤੀ ਵਿਖਾਉਣ ਲਈ `azd show` ਵਰਤੋਂ

### Q: Why are my environment variables not working?
**A**: ਚੈੱਕ ਕਰੋ:
1. ਵੈਰੀਏਬਲ ਨਾਮ ਟੈਂਪਲੇਟ ਪੈਰਾਮੀਟਰਾਂ ਨਾਲ ਬਿਲਕੁਲ ਮੇਲ ਖਾਂਦੇ ਹਨ
2. ਜੇ ਉਹਨਾਂ ਵਿੱਚ ਖਾਲੀ ਥਾਵਾਂ ਹਨ ਤਾਂ ਮੁੱਲ ਸਹੀ ਤਰੀਕੇ ਨਾਲ ਕੋਟ ਕੀਤੇ ਹੋਏ ਹਨ
3. ਵਾਤਾਵਰਣ ਚੁਣਿਆ ਗਿਆ ਹੈ: `azd env select <environment>`
4. ਵੈਰੀਏਬਲ ਸਹੀ ਵਾਤਾਵਰਣ ਵਿੱਚ ਸੈੱਟ ਕੀਤੇ ਗਏ ਹਨ

### Q: How do I clean up failed deployments?
**A**: 
```bash
azd down --force --purge
```
This removes all resources and environment configuration.

### Q: Why is my application not accessible after deployment?
**A**: ਚੈੱਕ ਕਰੋ:
1. ਡਿਪਲੋਇਮੈਂਟ ਸਫਲਤਾਪੂਰਵਕ ਪੂਰਾ ਹੋਇਆ ਹੈ
2. ਐਪਲੀਕੇਸ਼ਨ ਚੱਲ ਰਹੀ ਹੈ (Azure ਪੋਰਟਲ ਵਿੱਚ ਲੌਗ ਚੈੱਕ ਕਰੋ)
3. ਨੈਟਵਰਕ ਸਿਕਯੂਰਟੀ ਗਰੁੱਪ ਟ੍ਰੈਫਿਕ ਦੀ ਆਗਿਆ ਦਿੰਦੇ ਹਨ
4. DNS/ਕਸਟਮ ਡੋਮੇਨ ਸਹੀ ਤਰੀਕੇ ਨਾਲ ਕਨਫਿਗਰ ਕੀਤੇ ਗਏ ਹਨ

## Cost & Billing

### Q: How much will azd deployments cost?
**A**: ਖਰਚਾ ਇਸ 'ਤੇ ਨਿਰਭਰ ਕਰਦਾ ਹੈ:
- ਡਿਪਲੋਇ ਕੀਤੀਆਂ Azure ਸਰਵਿਸਿਜ਼
- ਚੁਣੀ ਗਈ ਸੇਵਾ ਟੀਅਰ/SKU
- ਖੇਤਰੀ ਕੀਮਤ ਭਿੰਨਤਾਵਾਂ
- ਵਰਤੋਂ ਦੇ ਪੈਟਰਨ

ਅੰਦਾਜ਼ਾ ਲੱਗਾਉਣ ਲਈ [Azure Pricing Calculator](https://azure.microsoft.com/pricing/calculator/) ਦੀ ਵਰਤੋਂ ਕਰੋ।

### Q: How do I control costs in azd deployments?
**A**: 
1. ਡਿਵੈਲਪਮੈਂਟ ਵਾਤਾਵਰਣਾਂ ਲਈ ਘੱਟ-ਟੀਅਰ SKUs ਦੀ ਵਰਤੋਂ ਕਰੋ

4. موزੂਨ ਖੇਤਰ ਚੁਣੋ (ਕਿੰਮਤਾਂ ਸਥਾਨ ਅਨੁਸਾਰ ਵੱਖ-ਵੱਖ ਹੁੰਦੀਆਂ ਹਨ)
5. Azure Cost Management ਟੂਲ ਵਰਤੋਂ

ਜਿੱਥੇ ਉਪਲਬਧ ਹੋਵੇ, ਟੈਂਪਲੇਟਾਂ ਨੂੰ ਮੁਫ਼ਤ ਟੀਅਰ ਵਰਤਣ ਲਈ ਕਨਫਿਗਰ ਕਰੋ।

### Q: How do I estimate costs before deployment?
**A**: 
1. ਦੇਖੋ ਕਿ ਟੈਂਪਲੇਟ ਦਾ `main.bicep` ਕਿਹੜੇ ਰਿਸੋਰਸ ਬਣਾਉਂਦਾ ਹੈ
2. ਨਿਰਧਾਰਿਤ SKUs ਨਾਲ Azure Pricing Calculator ਵਰਤੋਂ
3. ਪਹਿਲਾਂ ਇੱਕ ਡਿਵੈਲਪਮੈਂਟ ਵਾਤਾਵਰਣ 'ਤੇ ਡਿਪਲੋਇ ਕਰੋ ਅਤੇ ਅਸਲ ਖ਼ਰਚਿਆਂ ਦੀ ਨਿਗਰਾਨੀ ਕਰੋ
4. ਵਿਸਤ੍ਰਿਤ ਖ਼ਰਚ ਵਿਸ਼ਲੇਸ਼ਣ ਲਈ Azure Cost Management ਵਰਤੋਂ

## Best Practices

### Q: What are the best practices for azd project structure?
**A**: 
1. ਇੰਫਰਾਸਟ੍ਰਕਚਰ ਤੋਂ ਐਪਲੀਕੇਸ਼ਨ ਕੋਡ ਨੂੰ ਵੱਖਰਾ ਰੱਖੋ
2. `azure.yaml` ਵਿੱਚ ਅਰਥਪੂਰਨ ਸਰਵਿਸ ਨਾਮ ਵਰਤੋਂ
3. ਬਿਲਡ ਸਕ੍ਰਿਪਟਾਂ ਵਿੱਚ ਢੁੰਗਦਾਰ ਐਰਰ ਹੈਂਡਲਿੰਗ ਲਾਗੂ ਕਰੋ
4. ਵਾਤਾਵਰਣ-ਨਿਰਧਾਰਿਤ ਕੰਫਿਗਰੇਸ਼ਨ ਵਰਤੋਂ
5. ਵਿਸਥਾਰਪੂਰਕ ਦਸਤਾਵੇਜ਼ ਸ਼ਾਮਲ ਕਰੋ

### Q: How should I organize multiple services in azd?
**A**: ਸਿਫਾਰਸ਼ੀ ਢਾਂਚਾ ਵਰਤੋਂ:
```
├── src/
│   ├── web/          # Frontend service
│   ├── api/          # Backend service  
│   └── worker/       # Background service
├── infra/            # Infrastructure templates
├── azure.yaml        # azd configuration
└── README.md         # Project documentation
```

### Q: Should I commit the `.azure` folder to version control?
**A**: **ਨਹੀਂ!** `.azure` ਫੋਲਡਰ ਸੰਵੇਦਨਸ਼ੀਲ ਜਾਣਕਾਰੀ ਰੱਖਦਾ ਹੈ। ਇਸਨੂੰ `.gitignore` ਵਿੱਚ ਸ਼امل ਕਰੋ:
```gitignore
.azure/
```

### Q: How do I handle secrets and sensitive configuration?
**A**: 
1. ਗੁਪਤ ਜਾਣਕਾਰੀ ਲਈ Azure Key Vault ਵਰਤੋਂ
2. ਐਪਲੀਕੇਸ਼ਨ ਕੰਫਿਗਰੇਸ਼ਨ ਵਿੱਚ Key Vault ਗੁਪਤਾਂ ਦਾ ਰੈਫਰੰਸ ਕਰੋ
3. ਗੁਪਤਾਂ ਨੂੰ ਵਰਜਨ ਕੰਟਰੋਲ ਵਿੱਚ ਕਦੇ ਵੀ ਕਮੀਟ ਨਾ ਕਰੋ
4. ਸਰਵਿਸ-ਟੂ-ਸਰਵਿਸ ਪ੍ਰਮਾਣੀਕਰਨ ਲਈ ਮੈਨੇਜਡ ਆਈਡੈਂਟਿਟੀ ਦੀ ਵਰਤੋਂ ਕਰੋ

### Q: What's the recommended approach for CI/CD with azd?
**A**: 
1. ਹਰ ਸਟੇਜ (dev/staging/prod) ਲਈ ਵੱਖ-ਵੱਖ ਵਾਤਾਵਰਣ ਵਰਤੋਂ
2. ਡਿਪਲੋਇਮੈਂਟ ਤੋਂ ਪਹਿਲਾਂ ਆਟੋਮੇਟਿਡ ਟੈਸਟਿੰਗ ਲਾਗੂ ਕਰੋ
3. ਪ੍ਰਮਾਣੀਕਰਨ ਲਈ ਸਰਵਿਸ ਪ੍ਰਿੰਸੀਪਲ ਦੀ ਵਰਤੋਂ ਕਰੋ
4. ਸੰਵੇਦਨਸ਼ੀਲ ਕੰਫਿਗਰੇਸ਼ਨ ਨੂੰ ਪਾਈਪਲਾਈਨ ਸੀਕ੍ਰੇਟਸ/ਵੈਰੀਏਬਲਸ ਵਿੱਚ ਸਟੋਰ ਕਰੋ
5. ਪ੍ਰੋਡਕਸ਼ਨ ਡਿਪਲੋਇਮੈਂਟ ਲਈ ਅਨੁਮੋਦਨ ਗੇਟ ਲਗਾਓ

## Advanced Topics

### Q: Can I extend azd with custom functionality?
**A**: ਹਾਂ, `azure.yaml` ਵਿੱਚ ਡਿਪਲੋਇਮੈਂਟ ਹੂਕਸ ਰਾਹੀਂ:
```yaml
hooks:
  predeploy:
    run: ./scripts/custom-setup.sh
  postdeploy:
    run: ./scripts/custom-config.sh
```

### Q: How do I integrate azd with existing DevOps processes?
**A**: 
1. ਮੌਜੂਦਾ ਪਾਈਪਲਾਈਨ ਸਕ੍ਰਿਪਟਾਂ ਵਿੱਚ azd ਕਮਾਂਡਾਂ ਦੀ ਵਰਤੋਂ ਕਰੋ
2. ਟੀਮਾਂ ਵਿੱਚ azd ਟੈਂਪਲੇਟਾਂ 'ਤੇ ਸਟੈਂਡਰਡਾਈਜ਼ ਕਰੋ
3. ਮੌਜੂਦਾ ਮਾਨੀਟਰਿੰਗ ਅਤੇ ਅਲਰਟਿੰਗ ਨਾਲ ਇੰਟੀਗਰੇਟ ਕਰੋ
4. ਪਾਈਪਲਾਈਨ ਇੰਟੀਗਰੇਸ਼ਨ ਲਈ azd ਦੇ JSON ਆਉਟਪੁੱਟ ਦੀ ਵਰਤੋਂ ਕਰੋ

### Q: Can I use azd with Azure DevOps?
**A**: ਹਾਂ, azd ਕਿਸੇ ਵੀ CI/CD ਸਿਸਟਮ ਨਾਲ ਕੰਮ ਕਰਦਾ ਹੈ। azd ਕਮਾਂਡਾਂ ਦੀ ਵਰਤੋਂ ਕਰਨ ਵਾਲੀਆਂ Azure DevOps ਪਾਈਪਲਾਈਨਾਂ ਬਣਾਓ।

### Q: azd ਲਈ ਰੋਡਮੈਪ ਕੀ ਹੈ?
**A**: ਯੋਜਿਤ ਫੀਚਰਾਂ ਅਤੇ ਸੁਧਾਰਾਂ ਲਈ [ਆਧਿਕਾਰਕ ਰੋਡਮੈਪ](https://github.com/Azure/azure-dev/projects) ਨੂੰ ਚੈੱਕ ਕਰੋ।

### Q: ਮੈਂ ਹੋਰ ਡਿਪਲੌਇਮੈਂਟ ਟੂਲਾਂ ਤੋਂ azd 'ਤੇ ਕਿਵੇਂ ਮਾਈਗ੍ਰੇਟ ਕਰਾਂ?

4. ਡਿਵੈਲਪਮੈਂਟ ਵਾਤਾਵਰਨ ਵਿੱਚ ਪੂਰੀ ਤਰ੍ਹਾਂ ਟੈਸਟ ਕਰੋ
