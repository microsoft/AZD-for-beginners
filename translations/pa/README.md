## 🆕 azd ਵਿੱਚ ਅੱਜ ਨਵਾਂ ਕੀ ਹੈ

Azure Developer CLI ਹੁਣ ਰਵਾਇਤੀ ਵੈੱਬ ਐਪਾਂ ਅਤੇ APIs ਤੋਂ ਅੱਗੇ ਵੱਧ ਚੁੱਕੀ ਹੈ। ਅੱਜ, azd ਇੱਕੋ ਟੂਲ ਹੈ ਜੋ Azure 'ਤੇ ਕਿਸੇ ਵੀ ਐਪਲੀਕੇਸ਼ਨ ਨੂੰ ਡਿਪਲੋਏ ਕਰਨ ਲਈ ਵਰਤੀ ਜਾ ਸਕਦੀ ਹੈ — ਜਿਸ ਵਿੱਚ AI-ਸਮਰਥਿਤ ਐਪਲੀਕੇਸ਼ਨ ਅਤੇ ਬੁੱਧੀਮਾਨ ਏਜੰਟ ਵੀ ਸ਼ਾਮਿਲ ਹਨ।

ਇਸਦਾ ਤੁਹਾਡੇ ਲਈ ਮਤਲਬ ਇਹ ਹੈ:

- **AI ਏਜੰਟ ਹੁਣ azd ਦੇ ਪ੍ਰਮੁੱਖ ਵਰਕਲੋਡ ਹਨ।** ਤੁਸੀਂ ਉਹੀ `azd init` → `azd up` ਵਰਕਫਲੋਵ ਵਰਤਕੇ AI ਏਜੰਟ ਪ੍ਰੋਜੈਕਟ ਸ਼ੁਰੂ, ਡਿਪਲੋਏ ਅਤੇ ਪ੍ਰਬੰਧਿਤ ਕਰ ਸਕਦੇ ਹੋ।
- **Microsoft Foundry ਏਕੀਕਰਨ** ਮਾਡਲ ਡਿਪਲੋਏਮੈਂਟ, ਏਜੰਟ ਹੋਸਟਿੰਗ, ਅਤੇ AI ਸੇਵਾ ਸੰਰਚਨਾ ਨੂੰ ਸਿੱਧਾ azd ਟੈਮਪਲੇਟ ਇਕੋਸਿਸਟਮ ਵਿੱਚ ਲਿਆਉਂਦਾ ਹੈ।
- **ਮੂਲ ਵਰਕਫਲੋ ਵਧੇ ਨਹੀ ਹੈ।** ਚਾਹੇ ਤੁਸੀਂ ਇੱਕ todo ਐਪ, ਮਾਈਕ੍ਰੋਸਰਵਿਸ, ਜਾਂ ਇੱਕ ਬਹੁ-ਏਜੰਟ AI ਹੱਲ ਡਿਪਲੋਏ ਕਰ ਰਹੇ ਹੋ, ਕਮਾਂਡਸ ਇੱਕੋ ਜਿਹੇ ਹਨ।

ਜੇ ਤੁਸੀਂ ਪਹਿਲਾਂ azd ਵਰਤ ਚੁੱਕੇ ਹੋ, ਤਾਂ AI ਸਮਰਥਨ ਇੱਕ ਪ੍ਰਾਕ੍ਰਿਤਿਕ ਵਾਧਾ ਹੈ — ਕੋਈ ਵੱਖਰਾ ਟੂਲ ਜਾਂ ਉੱਚ ਪੱਧਰੀ ਟ੍ਰੈਕ ਨਹੀਂ। ਜੇ ਤੁਸੀਂ ਨਵੇਂ ਹੋ, ਤਾਂ ਤੁਸੀਂ ਇੱਕ ਐਸਾ ਵਰਕਫਲੋ ਸਿੱਖੋਗੇ ਜੋ ਹਰ ਚੀਜ਼ ਲਈ ਕੰਮ ਕਰਦਾ ਹੈ।

## 🚀 Azure Developer CLI (azd) ਕੀ ਹੈ?

**Azure Developer CLI (azd)** ਇੱਕ ਡਿਵੈਲਪਰ-ਮਿੱਤਰ ਕਮਾਂਡ-ਲਾਈਨ ਟੂਲ ਹੈ ਜੋ Azure 'ਤੇ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਡਿਪਲੋਏ ਕਰਨਾ ਸੌਖਾ ਬਣਾਉਂਦਾ ਹੈ। ਸੈਂਕੜਿਆਂ Azure ਰਿਸੋਰਸ ਹੱਥੋਂ-ਹੱਥ ਬਣਾਉਣ ਅਤੇ ਜੁੜਨ ਦੀ ਥਾਂ, ਤੁਸੀਂ ਇੱਕ ਹੀ ਕਮਾਂਡ ਨਾਲ ਪੂਰੀਆਂ ਐਪਲੀਕੇਸ਼ਨਾਂ ਡਿਪਲੋਏ ਕਰ ਸਕਦੇ ਹੋ।

### `azd up` ਦਾ ਜਾਦੂ

```bash
# ਇਹ ਇਕੱਲਾ ਕਮਾਂਡ ਸਾਰਾ ਕੰਮ ਕਰਦਾ ਹੈ:
# ✅ ਸਾਰੇ ਅਜ਼ੁਰ ਸਰੋਤ ਬਣਾਉਂਦਾ ਹੈ
# ✅ ਨੈੱਟਵਰਕਿੰਗ ਅਤੇ ਸੁਰੱਖਿਆ ਨੂੰ ਸੈੱਟ ਕਰਦਾ ਹੈ
# ✅ ਤੁਹਾਡਾ ਐਪਲੀਕੇਸ਼ਨ ਕੋਡ ਤਿਆਰ ਕਰਦਾ ਹੈ
# ✅ ਅਜ਼ੁਰ 'ਤੇ ਡਿਪਲੋਇ ਕਰਦਾ ਹੈ
# ✅ ਤੁਹਾਨੂੰ ਇੱਕ ਕਾਮਯਾਬ URL ਦਿੰਦਾ ਹੈ
azd up
```

**ਬੱਸ ਇੰਨਾ ਹੀ!** ਕੋਈ Azure ਪੋਰਟਲ 'ਤੇ ਕਲਿੱਕਿੰਗ, ਕੋਈ ਜਟਿਲ ARM ਟੈਮਪਲੇਟ ਪਹਿਲਾਂ ਸਿੱਖਣ ਦੀ ਲੋੜ ਨਹੀਂ, ਕੋਈ ਹੱਥੋਂ-ਹੱਥ ਸੰਰਚਨਾ ਨਹੀਂ — ਸਿਰਫ਼ Azure 'ਤੇ ਚੱਲ ਰਹੀਆਂ ਕਾਰਜਸ਼ੀਲ ਐਪਲੀਕੇਸ਼ਨ।

## ❓ Azure Developer CLI ਅਤੇ Azure CLI: ਫਰਕ ਕੀ ਹੈ?

ਇਹ ਉਹ ਸਭ ਤੋਂ ਆਮ ਸਵਾਲ ਹੈ ਜੋ ਸ਼ੁਰੂਆਤੀ ਪੁੱਛਦੇ ਹਨ। ਇੱਥੇ ਸਧਾਰਨ ਜਵਾਬ ਹੈ:

| ਖਾਸੀਅਤ | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **ਮਕਸਦ** | ਵਿਅਕਤੀਗਤ Azure ਰਿਸੋਰਸਾਂ ਨੂੰ ਪ੍ਰਬੰਧਿਤ ਕਰੋ | ਪੂਰੀਆਂ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਡਿਪਲੋਏ ਕਰੋ |
| **ਦ੍ਰਿਸ਼ਟੀਕੋਣ** | ਇੰਫਰਾਸਟ੍ਰਕਚਰ-ਕੇਂਦਰਿਤ | ਐਪਲੀਕੇਸ਼ਨ-ਕੇਂਦਰਿਤ |
| **ਉਦਾਹਰਨ** | `az webapp create --name myapp...` | `azd up` |
| **ਸਿੱਖਣ ਦੀ ਲੁੜੀ** | Azure ਸੇਵਾਵਾਂ ਬਾਰੇ ਜਾਣਨਾ ਲਾਜ਼ਮੀ | ਸਿਰਫ਼ ਆਪਣੀ ਐਪ ਜਾਣੋ |
| **ਸਭ ਤੋਂ ਵਧੀਆ ਲਈ** | DevOps, Infrastructure | ਡਿਵੈਲਪર્સ, ਪ੍ਰੋਟੋਟਾਈਪਿੰਗ |

### ਸਧਾਰਨ ਤੌਰ ਤੇ ਮਿਸਾਲ

- **Azure CLI** ਉਹਨਾਂ ਸਾਰਿਆਂ ਯੰਤਰਾਂ ਵਰਗੀ ਹੈ ਜੋ ਇੱਕ ਘਰ ਤਿਆਰ ਕਰਨ ਲਈ ਲੋੜੀਂਦੇ ਹਨ - ਹਥੌੜੇ, ਦਰਾਜ, ਕੇਲ। ਤੁਸੀਂ ਕੁਝ ਵੀ ਬਣਾ ਸਕਦੇ ਹੋ, ਪਰ ਤੁਹਾਨੂੰ ਨਿਰਮਾਣ ਦਾ ਗਿਆਨ ਲੋੜੀਂਦਾ ਹੈ।
- **Azure Developer CLI** ਇੱਕ ਠੇਕੇਦਾਰ ਹਾਇਰ ਕਰਨ ਵਰਗਾ ਹੈ - ਤੁਸੀਂ ਦੱਸੋ ਕਿ ਤੁਸੀਂ ਕੀ ਚਾਹੁੰਦੇ ਹੋ, ਅਤੇ ਉਹ ਬਨਾਉਂਦੇ ਹਨ।

### ਕਦੋਂ ਕਿਸਦੀ ਵਰਤੋਂ ਕਰੋ

| ਪਰਿਸਥਿਤੀ | ਇਹ ਵਰਤੋ |
|----------|----------|
| "ਮੈਂ ਆਪਣੀ ਵੇਬ ਐਪ ਜਲਦੀ ਡਿਪਲੋਏ ਕਰਨੀ ਚਾਹੁੰਦਾ/ਚਾਹੁੰਦੀ ਹਾਂ" | `azd up` |
| "ਮੈਨੂੰ ਸਿਰਫ਼ ਇੱਕ storage account ਬਣਾਉਣਾ ਹੈ" | `az storage account create` |
| "ਮੈਂ ਇੱਕ ਪੂਰਾ AI ਐਪ ਤਿਆਰ ਕਰ ਰਿਹਾ/ਰਿਹੀ ਹਾਂ" | `azd init --template azure-search-openai-demo` |
| "ਮੈਨੂੰ ਕਿਸੇ ਖਾਸ Azure ਰਿਸੋਰਸ ਨੂੰ ਡਿਬੱਗ ਕਰਨਾ ਹੈ" | `az resource show` |
| "ਮੈਂ ਮਿੰਟਾਂ ਵਿੱਚ ਪ੍ਰੋਡਕਸ਼ਨ-ਤਿਅਰ ਡਿਪਲੋਏਮੈਂਟ ਚਾਹੁੰਦਾ/ਚਾਹੁੰਦੀ ਹਾਂ" | `azd up --environment production` |

### ਇਹ ਇਕੱਠੇ ਕੰਮ ਕਰਦੇ ਹਨ!

AZD ਅੰਦਰੋਂ Azure CLI ਦੀ ਵਰਤੋਂ ਕਰਦਾ ਹੈ। ਤੁਸੀਂ ਦੋਹਾਂ ਵਰਤ ਸਕਦੇ ਹੋ:
```bash
# ਆਪਣੇ ਐਪ ਨੂੰ AZD ਨਾਲ ਤਾਇਨਾਤ ਕਰੋ
azd up

# ਫਿਰ Azure CLI ਨਾਲ ਖ਼ਾਸ ਸਰੋਤਾਂ ਨੂੰ ਬਿਹਤਰ ਬਣਾਓ
az webapp config set --name myapp --always-on true
```

## 🎯 3 ਕਦਮਾਂ ਵਿੱਚ ਸ਼ੁਰੂਆਤ

ਸ਼ੁਰੂ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ, ਯਕੀਨੀ ਬਣਾਓ ਕਿ ਤੁਹਾਡੀ ਮਸ਼ੀਨ ਉਸ ਟੈਮਪਲੇਟ ਲਈ ਤਿਆਰ ਹੈ ਜੋ ਤੁਸੀਂ ਡਿਪਲੋਏ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ:

**Windows:**
```powershell
.\validate-setup.ps1
```

**macOS / Linux:**
```bash
bash ./validate-setup.sh
```

ਜੇ ਕੋਈ ਲੋੜੀਂਦੀ ਜਾਂਚ ਅਸਫਲ ਹੁੰਦੀ ਹੈ, ਤਾਂ ਪਹਿਲਾਂ ਉਸ ਨੂੰ ਠੀਕ ਕਰੋ ਅਤੇ ਫਿਰ ਕਵਿਕ ਸਟਾਰਟ ਨਾਲ ਜਾਰੀ ਰੱਖੋ।

### ਕਦਮ 1: AZD ਇੰਸਟਾਲ ਕਰੋ (2 ਮਿੰਟ)

**Windows:**
```powershell
winget install microsoft.azd
```

**macOS:**
```bash
brew tap azure/azd && brew install azd
```

**Linux:**
```bash
curl -fsSL https://aka.ms/install-azd.sh | bash
```

### ਕਦਮ 2: AZD ਲਈ ਪ੍ਰਮਾਣਿਕਤਾ

```bash
# ਜੇ ਤੁਸੀਂ ਇਸ ਕੋਰਸ ਵਿੱਚ ਸੀਧਾ Azure CLI ਹੁਕਮ ਵਰਤਣ ਦਾ ਯੋਜਨਾ ਬਣਾਈ ਹੈ ਤਾਂ ਵੈਕਲਪਿਕ
az login

# AZD ਵਰਕਫਲੋਜ਼ ਲਈ ਲਾਜ਼ਮੀ
azd auth login
```

ਜੇ ਤੁਹਾਨੂੰ ਪਤਾ ਨਹੀਂ ਕਿ ਤੁਹਾਨੂੰ ਕਿਹੜੀ ਲੋੜ ਹੈ, ਤਾਂ ਪੂਰੇ ਸੈਟਅਪ ਫਲੋ ਲਈ [ਇੰਸਟਾਲੇਸ਼ਨ ਅਤੇ ਸੈਟਅਪ](docs/chapter-01-foundation/installation.md#authentication-setup) ਦੇ ਨਿਰਦੇਸ਼ਾਂ ਦੀ ਪਾਲਣਾ ਕਰੋ।

### ਕਦਮ 3: ਆਪਣੀ ਪਹਿਲੀ ਐਪ ਡਿਪਲੋਏ ਕਰੋ

```bash
# ਟੈਮਪਲੇਟ ਤੋਂ ਸ਼ੁਰੂਆਤ ਕਰੋ
azd init --template todo-nodejs-mongo

# ਅਜ਼ੂਰ 'ਤੇ ਰੋਲਆਉਟ ਕਰੋ (ਸਭ ਕੁਝ ਬਣਾਉਂਦਾ ਹੈ!)
azd up
```

**🎉 ਹੋ ਗਿਆ!** ਤੁਹਾਡੀ ਐਪ ਹੁਣ Azure 'ਤੇ ਲਾਈਵ ਹੈ।

### ਸਾਫ਼-ਸਫਾਈ (ਭੁੱਲਣਾ ਨਾ!)

```bash
# Remove all resources when done experimenting
azd down --force --purge
```

ਸੰਰਚਿਤ ਅਧਿਆਇਆਂ ਰਾਹੀਂ Azure Developer CLI (azd) 'ਤੇ ਮਾਹਿਰ ਬਣੋ, ਜੋ ਪ੍ਰਗਟਾਵਾ ਸਿੱਖਣ ਲਈ ਡਿਜ਼ਾਇਨ ਕੀਤੇ ਗਏ ਹਨ। **Microsoft Foundry ਏਕੀਕਰਨ ਨਾਲ AI ਐਪਲੀਕੇਸ਼ਨ ਡਿਪਲੋਏਮੈਂਟ 'ਤੇ ਖ਼ਾਸ ਜ਼ੋਰ।**

- ਪ੍ਰੋਡਕਸ਼ਨ AI ਡਿਪਲੋਏਮੈਂਟ ਦੀਆਂ ਸਭ ਤੋਂ ਵਧੀਆ ਪ੍ਰਥਾਵਾਂ

- **Infrastructure as Code ਲਾਗੂ ਕਰੋ**: Bicep ਟੈਮਪਲੇਟ ਨਾਲ Azure ਰਿਸੋਰਸਾਂ ਦਾ ਪ੍ਰਬੰਧ

## ਸ਼ੁਰੂ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ: ਅਕਾਊਂਟ, ਐਕਸੈਸ, ਅਤੇ ਧਾਰਣਾਵਾਂ

ਅਧਿਆਇ 1 ਸ਼ੁਰੂ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ, ਇਹ ਯਕੀਨੀ ਬਣਾਓ ਕਿ ਤੁਹਾਡੇ ਕੋਲ ਹੇਠ ਲਿਖੀਆਂ ਚੀਜ਼ਾਂ ਮੌਜੂਦ ਹਨ। ਇਸ ਗਾਈਡ ਵਿੱਚ ਅਗਲੇ ਇੰਸਟਾਲੇਸ਼ਨ ਕਦਮ ਇਹ ਮੂਲ ਧਾਰਣਾਵਾਂ ਮੰਨ ਕੇ ਚੱਲਦੇ ਹਨ।
- **ਇੱਕ Azure ਸਬਸਕ੍ਰਿਪਸ਼ਨ**: ਤੁਸੀਂ ਕੰਮ ਤੋਂ ਮੌਜੂਦਾ ਸਬਸਕ੍ਰਿਪਸ਼ਨ ਜਾਂ ਆਪਣੇ ਖਾਤੇ ਵਾਲਾ ਸਬਸਕ੍ਰਿਪਸ਼ਨ ਵਰਤ ਸਕਦੇ ਹੋ, ਜਾਂ ਸ਼ੁਰੂ ਕਰਨ ਲਈ [free trial](https://aka.ms/azurefreetrial) ਬਣਾਉ।
- **Azure ਰਿਸੋਰਸ ਬਣਾਉਣ ਦੀ ਆਗਿਆ**: ਜ਼ਿਆਦਾਤਰ ਅਭਿਆਸਾਂ ਲਈ, ਤੁਹਾਡੇ ਕੋਲ ਟਾਰਗੇਟ ਸਬਸਕ੍ਰਿਪਸ਼ਨ ਜਾਂ ਰਿਸੋਰਸ ਗਰੁੱਪ 'ਤੇ ਘੱਟੋ-ਘੱਟ **Contributor** ਪਰਵਾਨਗੀ ਹੋਣੀ ਚਾਹੀਦੀ ਹੈ। ਕੁਝ ਅਧਿਆਇ ਇਹ ਵੀ ਮੰਨ ਸਕਦੇ ਹਨ ਕਿ ਤੁਸੀਂ ਰਿਸੋਰਸ ਗਰੁੱਪ, managed identities, ਅਤੇ RBAC ਅਸਾਈਨਮੈਂਟ ਬਣਾਉ ਸਕਦੇ ਹੋ।
- [**ਇੱਕ GitHub ਖਾਤਾ**](https://github.com): ਇਹ ਰਿਪੋਜ਼ਟਰੀ ਨੂੰ fork ਕਰਨ, ਆਪਣੀਆਂ ਸੇਟਾਂ ਟਰੈਕ ਕਰਨ, ਅਤੇ ਵਰਕਸ਼ਾਪ ਲਈ GitHub Codespaces ਵਰਤਣ ਲਈ ਫਾਇਦੇਮੰਦ ਹੈ।
- **ਟੈਂਪਲੇਟ ਰਨਟਾਈਮ ਦੀਆਂ ਲੋੜਾਂ**: ਕੁਝ ਟੈਂਪਲੇਟਸ ਨੂੰ ਲੋਕਲ ਟੂਲਾਂ ਜਿਵੇਂ Node.js, Python, Java, ਜਾਂ Docker ਦੀ ਲੋੜ ਹੁੰਦੀ ਹੈ। ਸ਼ੁਰੂ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ setup validator ਚਲਾਓ ਤਾਂ ਜੋ ਗੁੰਮ ਟੂਲ ਸਵੇਰੇ ਪਤਾ ਲੱਗ ਜਾਣ।
- **ਬੁਨਿਆਦੀ ਟਰਮੀਨਲ ਜਾਣੂਤਾ**: ਤੁਹਾਨੂੰ ਵਿਸ਼ੇਸ਼ਜ੍্ঞ ਹੋਣ ਦੀ ਲੋੜ ਨਹੀਂ, ਪਰ ਤੁਸੀਂ `git clone`, `azd auth login`, ਅਤੇ `azd up` ਵਰਗੀਆਂ ਕਮਾਂਡਾਂ ਚਲਾਉਣ ਵਿੱਚ ਆਰਾਮਦায়ਕ ਹੋਣਾ ਚਾਹੀਦਾ ਹੈ।

> **ਕਿਸੇ ਏਂਟਰਪ੍ਰਾਈਜ਼ ਸਬਸਕ੍ਰਿਪਸ਼ਨ 'ਚ ਕੰਮ ਕਰ ਰਹੇ ਹੋ?**
> ਜੇ ਤੁਹਾਡਾ Azure ਵਾਤਾਵਰਣ ਕਿਸੇ ਐਡਮਿਨ ਦੁਆਰਾ ਮੈਨੇਜ ਕੀਤਾ ਜਾਂਦਾ ਹੈ, ਤਾਂ ਪਹਿਲਾਂ ਹੀ ਪੁਸ਼ਟੀ ਕਰੋ ਕਿ ਤੁਸੀਂ ਜਿਸ ਸਬਸਕ੍ਰਿਪਸ਼ਨ ਜਾਂ ਰਿਸੋਰਸ ਗਰੁੱਪ ਨੂੰ ਵਰਤਣ ਦੀ ਯੋਜਨਾ ਬਣਾ ਰਹੇ ਹੋ ਉਸ ਵਿੱਚ ਰਿਸੋਰਸ ਡਿਪਲੌਇ ਕਰ ਸਕਦੇ ਹੋ। ਜੇ ਨਹੀਂ, ਤਾਂ ਸ਼ੁਰੂ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ sandbox ਸਬਸਕ੍ਰਿਪਸ਼ਨ ਜਾਂ Contributor ਐਕਸੈਸ ਦੀ ਮੰਗ ਕਰੋ।

## 🔧 ਤੁਰੰਤ ਟ੍ਰਬਲਸ਼ੂਟਿੰਗ ਗਾਈਡ

**ਆਮ ਮੁੱਦੇ ਜੋ ਸ਼ੁਰੂਆਤੀ ਅਕਸਰ ਸਾਹਮਣੇ ਆਉਂਦੇ ਹਨ ਅਤੇ ਤੁਰੰਤ ਹੱਲ:**

<details>
<summary><strong>❌ "azd: command not found"</strong></summary>

```bash
# ਸਭ ਤੋਂ ਪਹਿਲਾਂ AZD ਇੰਸਟਾਲ ਕਰੋ
# ਵਿਂਡੋਜ਼ (PowerShell):
winget install microsoft.azd

# ਮੈਕਓਐਸ:
brew tap azure/azd && brew install azd

# ਲਿਨਕਸ:
curl -fsSL https://aka.ms/install-azd.sh | bash

# ਇੰਸਟਾਲੇਸ਼ਨ ਦੀ ਪੁਸ਼ਟੀ ਕਰੋ
azd version
```
</details>

<details>
<summary><strong>❌ "No subscription found" or "Subscription not set"</strong></summary>

```bash
# ਉਪਲਬਧ ਸਬਸਕ੍ਰਿਪਸ਼ਨਾਂ ਦੀ ਸੂਚੀ
az account list --output table

# ਡਿਫਾਲਟ ਸਬਸਕ੍ਰਿਪਸ਼ਨ ਸੈੱਟ ਕਰੋ
az account set --subscription "<subscription-id-or-name>"

# AZD ਵਾਤਾਵਰਣ ਲਈ ਸੈੱਟ ਕਰੋ
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# ਪੁਸ਼ਟੀ ਕਰੋ
az account show
```
</details>

<details>
<summary><strong>❌ "InsufficientQuota" or "Quota exceeded"</strong></summary>

```bash
# ਵੱਖਰੇ Azure ਰੀਜਨ ਨੂੰ ਆਜ਼ਮਾਓ
azd env set AZURE_LOCATION "westus2"
azd up

# ਜਾਂ ਡਿਵੈਲਪਮੈਂਟ ਵਿੱਚ ਛੋਟੇ SKUs ਵਰਤੋ
# infra/main.parameters.json ਨੂੰ ਸੰਪਾਦਿਤ ਕਰੋ:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>

<details>
<summary><strong>❌ "azd up" fails halfway through</strong></summary>

```bash
# ਵਿਕਲਪ 1: ਸਾਫ਼ ਕਰੋ ਅਤੇ ਮੁੜ ਕੋਸ਼ਿਸ਼ ਕਰੋ
azd down --force --purge
azd up

# ਵਿਕਲਪ 2: ਸਿਰਫ਼ ਬੁਨਿਆਦੀ ਢਾਂਚਾ ਠੀਕ ਕਰੋ
azd provision

# ਵਿਕਲਪ 3: ਵਿਸਥਾਰਤ ਸਥਿਤੀ ਦੀ ਜਾਂਚ ਕਰੋ
azd show

# ਵਿਕਲਪ 4: Azure Monitor ਵਿੱਚ ਲਾਗਾਂ ਦੀ ਜਾਂਚ ਕਰੋ
azd monitor --logs
```
</details>

<details>
<summary><strong>❌ "Authentication failed" or "Token expired"</strong></summary>

```bash
# AZD ਲਈ ਦੁਬਾਰਾ ਪ੍ਰਮਾਣੀਕਰਨ ਕਰੋ
azd auth logout
azd auth login

# ਵਿਕਲਪਿਕ: ਜੇ ਤੁਸੀਂ az ਕਮਾਂਡਾਂ ਚਲਾ ਰਹੇ ਹੋ ਤਾਂ Azure CLI ਨੂੰ ਵੀ ਰੀਫ੍ਰੈਸ਼ ਕਰੋ
az logout
az login

# ਪ੍ਰਮਾਣੀਕਰਨ ਦੀ ਜਾਂਚ ਕਰੋ
az account show
```
</details>

<details>
<summary><strong>❌ "Resource already exists" or naming conflicts</strong></summary>

```bash
# AZD ਵਿਲੱਖਣ ਨਾਮ ਬਣਾਉਂਦਾ ਹੈ, ਪਰ ਜੇ ਟਕਰਾਅ ਹੋਵੇ:
azd down --force --purge

# ਫਿਰ ਨਵੇਂ ਮਾਹੌਲ ਨਾਲ ਦੁਬਾਰਾ ਕੋਸ਼ਿਸ਼ ਕਰੋ
azd env new dev-v2
azd up
```
</details>

<details>
<summary><strong>❌ Template deployment taking too long</strong></summary>

**ਸਧਾਰਣ ਪ੍ਰਤੀਕਸ਼ਾ ਸਮਾਂ:**
- ਸਧਾਰਣ ਵੈੱਬ ਐਪ: 5-10 ਮਿੰਟ
- ਡੇਟਾਬੇਸ ਵਾਲਾ ਐਪ: 10-15 ਮਿੰਟ
- AI ਐਪਲੀਕੇਸ਼ਨ: 15-25 ਮਿੰਟ (OpenAI ਪ੍ਰੋਵਿਜ਼ਨਿੰਗ ਸਲੋ ਹੈ)

```bash
# ਤਰੱਕੀ ਦੀ ਜਾਂਚ ਕਰੋ
azd show

# ਜੇ 30 ਮਿੰਟ ਤੋਂ ਵੱਧ ਫਸੇ ਹੋ ਤਾਂ, Azure ਪੋਰਟਲ ਜਾਂਚੋ:
azd monitor --overview
# ਅਸਫਲ ਡਿਪਲੋਇਮੈਂਟਾਂ ਦੀ ਖੋਜ ਕਰੋ
```
</details>

<details>
<summary><strong>❌ "Permission denied" or "Forbidden"</strong></summary>

```bash
# ਆਪਣੇ Azure ਰੋਲ ਦੀ ਜਾਂਚ ਕਰੋ
az role assignment list --assignee $(az account show --query user.name -o tsv)

# ਤੁਹਾਨੂੰ ਘੱਟੋ-ਘੱਟ "Contributor" ਰੋਲ ਦੀ ਲੋੜ ਹੈ
# ਆਪਣੇ Azure ਐਡਮਿਨ ਨੂੰ ਬੇਨਤੀ ਕਰੋ ਕਿ ਇਹ ਦਿਓ:
# - Contributor (ਰਿਸੋਰਸਾਂ ਲਈ)
# - User Access Administrator (ਰੋਲ ਅਸਾਈਨਮੈਂਟਸ ਲਈ)
```
</details>

<details>
<summary><strong>❌ Can't find deployed application URL</strong></summary>

```bash
# ਸਾਰੇ ਸਰਵਿਸ ਐਂਡਪੋਇੰਟ ਦਿਖਾਓ
azd show

# ਜਾਂ Azure ਪੋਰਟਲ ਖੋਲੋ
azd monitor

# ਕਿਸੇ ਖਾਸ ਸੇਵਾ ਦੀ ਜਾਂਚ ਕਰੋ
azd env get-values
# *_URL ਵੈਰੀਏਬਲਾਂ ਲਈ ਵੇਖੋ
```
</details>
