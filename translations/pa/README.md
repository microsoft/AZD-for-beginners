## 🆕 ਅੱਜ azd ਵਿੱਚ ਕੀ ਨਵਾਂ ਹੈ

Azure Developer CLI ਹੁਣ ਰਵਾਇਤੀ ਵੈੱਬ ਐਪਸ ਅਤੇ APIs ਤੋਂ ਬਹੁਤ آگੇ ਵਧ ਚੁੱਕੀ ਹੈ। ਅੱਜ, azd ਇੱਕਲਾ ਟੂਲ ਹੈ ਜੋ Azure 'ਤੇ ਕਿਸੇ ਵੀ ਐਪਲੀਕੇਸ਼ਨ ਨੂੰ ਤਾਇਨਾਤ ਕਰਨ ਲਈ ਵਰਤਿਆ ਜਾ ਸਕਦਾ ਹੈ — ਜਿਸ ਵਿੱਚ AI-ਸਮਰੱਥ ਐਪਲੀਕੇਸ਼ਨਾਂ ਅਤੇ ਇੰਟੈਲੀਜੈਂਟ ਏਜੰਟ ਵੀ ਸ਼ਾਮਲ ਹਨ।

ਇਸਦਾ ਤੁਹਾਡੇ ਲਈ ਕੀ ਮਤਲਬ ਹੈ:

- **AI ਏਜੰਟ ਹੁਣ ਪਹਿਲੇ ਦਰਜੇ ਦੇ azd ਵਰਕਲੋਡ ਹਨ।** ਤੁਸੀਂ ਇੱਕੋ `azd init` → `azd up` ਵਰਕਫਲੋਵ ਦੀ ਵਰਤੋਂ ਕਰਕੇ AI ਏਜੰਟ ਪ੍ਰਾਜੈਕਟ ਸ਼ੁਰੂ, ਤਾਇਨਾਤ ਅਤੇ ਪਰਬੰਧਿਤ ਕਰ ਸਕਦੇ ਹੋ।
- **Microsoft Foundry ਇੰਟਿਗਰੇਸ਼ਨ** ਮਾਡਲ ਤਾਇਨਾਤੀ, ਏਜੰਟ ਹੋਸਟਿੰਗ ਅਤੇ AI ਸੇਵਾ ਸੰਰਚਨਾ ਨੂੰ ਸਿੱਧਾ azd ਟੈਂਪਲੇਟ ਇਕੋਸਿਸਟਮ ਵਿੱਚ ਲਿਆਉਂਦਾ ਹੈ।
- **ਮੁੱਖ ਵਰਕਫਲੋਵ ਬਦਲਿਆ ਨਹੀਂ ਹੈ।** ਚਾਹੇ ਤੁਸੀਂ ਇੱਕ todo ਐਪ, ਇੱਕ ਮਾਈਕ੍ਰੋਸਰਵਿਸ ਜਾਂ ਇੱਕ ਬਹੁ-ਏਜੰਟ AI ਹੱਲ ਤਾਇਨਾਤ ਕਰ ਰਹੇ ਹੋ, ਕਮਾਂਡ ਉਹੀ ਰਹਿੰਦੀਆਂ ਹਨ।

ਜੇ ਤੁਸੀਂ ਪਹਿਲਾਂ azd ਵਰਤ ਚੁੱਕੇ ਹੋ, ਤਾਂ AI ਸਹਾਇਤਾ ਇੱਕ ਕੁਦਰਤੀ ਵਿਰੋਧੀ ਵਾਧਾ ਹੈ — ਕੋਈ ਵੱਖਰਾ ਟੂਲ ਜਾਂ ਪ੍ਰगतਿ ਸਕੀਮ ਨਹੀਂ। ਜੇ ਤੁਸੀਂ ਨਵੇਂ ਹੋ, ਤਾਂ ਤੁਸੀਂ ਇੱਕ ਐਸਾ ਵਰਕਫਲੋ ਸਿੱਖੋਗੇ ਜੋ ਹਰ ਚੀਜ਼ ਲਈ ਕੰਮ ਕਰਦਾ ਹੈ।

## 🚀 Azure Developer CLI (azd) ਕੀ ਹੈ?

**Azure Developer CLI (azd)** ਇੱਕ ਵਿਕਾਸਕ-ਮਿੱਤਰ ਕਮਾਂਡ-ਲਾਈਨ ਟੂਲ ਹੈ ਜੋ Azure 'ਤੇ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਤਾਇਨਾਤ ਕਰਨਾ ਸੌਖਾ ਬਣਾਉਂਦਾ ਹੈ। ਤਕਨੀਕੀ ਤੌਰ 'ਤੇ ਕਈ Azure ਰਿਸੋਰਸਾਂ ਨੂੰ ਹੱਥੋਂ-ਹੱਥ ਬਣਾਉਣ ਅਤੇ ਜੁੜਨ ਦੀ ਥਾਂ, ਤੁਸੀਂ ਇੱਕ ਹੀ ਕਮਾਂਡ ਨਾਲ ਪੂਰੀਆਂ ਐਪਲੀਕੇਸ਼ਨਾਂ ਤਾਇਨਾਤ ਕਰ ਸਕਦੇ ਹੋ।

### `azd up` ਦੀ ਜਾਦੂਈ ਸਮਰੱਥਾ

```bash
# ਇਹ ਇੱਕੋ ਹੁਕਮ ਸਾਰਾ ਕੰਮ ਕਰਦਾ ਹੈ:
# ✅ ਸਾਰੇ Azure ਸਰੋਤ ਬਣਾਉਂਦਾ ਹੈ
# ✅ ਨੈਟਵਰਕਿੰਗ ਅਤੇ ਸੁਰੱਖਿਆ ਨੂੰ ਸੰਰਚਿਤ ਕਰਦਾ ਹੈ
# ✅ ਤੁਹਾਡੇ ਐਪਲੀਕੇਸ਼ਨ ਕੋਡ ਨੂੰ ਬਣਾਉਂਦਾ ਹੈ
# ✅ Azure 'ਤੇ ਤੈਨਾਤ ਕਰਦਾ ਹੈ
# ✅ ਤੁਹਾਨੂੰ ਇੱਕ ਕੰਮ ਕਰਦਾ URL ਦਿੰਦਾ ਹੈ
azd up
```

**ਬਸ ਇਹੀ!** Azure ਪੋਰਟਲ 'ਤੇ ਕਲਿੱਕ ਕਰਨ ਦੀ ਲੋੜ ਨਹੀਂ, ਕੋਈ ਜਟਿਲ ARM ਟੈਮਪਲੇਟ ਪਹਿਲਾਂ ਸਿੱਖਣ ਦੀ ਲੋੜ ਨਹੀਂ, ਕੋਈ ਮੈਨੁਅਲ ਸੰਰਚਨਾ ਨਹੀਂ - ਸਿਰਫ਼ Azure 'ਤੇ ਕੰਮ ਕਰ ਰਹੀਆਂ ਐਪਸ।

## ❓ Azure Developer CLI ਅਤੇ Azure CLI: ਫ਼ਰਕ ਕੀ ਹੈ?

ਇਹ ਨਵਿਆਂ ਵੱਲੋਂ ਪੁੱਛਿਆ ਜਾਣ ਵਾਲਾ ਸਭ ਤੋਂ ਆਮ ਸਵਾਲ ਹੈ। ਇੱਥੇ ਸਾਦਾ ਜਵਾਬ ਹੈ:

| ਵਿਸ਼ੇਸ਼ਤਾ | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **ਮਕਸਦ** | ਵਿਅਕਤੀਗਤ Azure ਸੰਸਾਧਨਾਂ ਦਾ ਪ੍ਰਬੰਧ | ਪੂਰੇ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਤਾਇਨਾਤ ਕਰਨਾ |
| **ਮਨੋਭਾਵ** | ਇੰਫਰਾਸਟਰੱਕਚਰ-ਕੇਂਦਰਿਤ | ਐਪਲੀਕੇਸ਼ਨ-ਕੇਂਦਰਿਤ |
| **ਉਦਾਹਰਨ** | `az webapp create --name myapp...` | `azd up` |
| **ਸਿੱਖਣ ਵਕਰਾ** | Azure ਸੇਵਾਵਾਂ ਜਾਣਣੀਆਂ ਹੋਣੀਆਂ ਚਾਹੀਦੀਆਂ ਹਨ | ਸਿਰਫ਼ ਆਪਣੀ ਐਪ ਜਾਣੋ |
| **ਸਭ ਤੋਂ ਵਧੀਆ ਲਈ** | DevOps, ਇੰਫਰਾਸਟਰੱਕਚਰ | ਡਿਵੈਲਪਰ, ਪ੍ਰੋਟੋਟਾਈਪਿੰਗ |

### ਸਧਾਰਨ ਤੁਲਨਾ

- **Azure CLI** ਉਹਨਾਂ ਸਾਰਿਆਂ ਟੂਲਾਂ ਵਾਂਗ ਹੈ ਜੋ ਇੱਕ ਘਰ ਬਣਾਉਣ ਲਈ ਚਾਹੀਦੇ ਹਨ - ਹਮਰ, ਆਰਾ, ਨੱਲ। ਤੁਸੀਂ ਕੁਝ ਵੀ ਬਣਾ ਸਕਦੇ ਹੋ, ਪਰ ਤੁਹਾਨੂੰ ਨਿਰਮਾਣ ਦੀ ਜਾਣਕਾਰੀ ਹੋਣੀ ਚਾਹੀਦੀ ਹੈ।
- **Azure Developer CLI** ਇੱਕ ਕੰਟ੍ਰੈਕਟਰ ਰੱਖਣ ਵਰਗੀ ਹੈ - ਤੁਸੀਂ ਦੱਸਦੇ ਹੋ ਕਿ ਤੁਸੀਂ ਕੀ ਚਾਹੁੰਦੇ ਹੋ, ਅਤੇ ਉਹ ਬਨਾਉਣ ਦੀ ਜ਼ਿੰਮੇਵਾਰੀ ਲਈ ਲੈਂਦਾ ਹੈ।

### ਕਦੋਂ ਕਿਹੜਾ ਵਰਤਣਾ ਹੈ

| ਪਰਿਸਥਿਤੀ | ਇਹ ਵਰਤੋ |
|----------|----------|
| "ਮੈਂ ਆਪਣੀ ਵੈੱਬ ਐਪ ਤੇਜ਼ੀ ਨਾਲ ਤਾਇਨਾਤ ਕਰਨਾ ਚਾਹੁੰਦਾ/ਚਾਹੁੰਦੀ ਹਾਂ" | `azd up` |
| "ਮੈਨੂੰ ਸਿਰਫ਼ ਇਕ ਸਟੋਰੇਜ ਅਕਾਉਂਟ ਬਣਾਉਣੀ ਹੈ" | `az storage account create` |
| "ਮੈਂ ਇੱਕ ਪੂਰਾ AI ਐਪ ਬਣਾ ਰਿਹਾ/ਰਿਹੀ ਹਾਂ" | `azd init --template azure-search-openai-demo` |
| "ਮੈਨੂੰ ਕਿਸੇ ਨਿਰਦਿਸ਼ਟ Azure ਰਿਸੋਰਸ ਨੂੰ ਡੀਬੱਗ ਕਰਨ ਦੀ ਲੋੜ ਹੈ" | `az resource show` |
| "ਮੈਂ ਮਿੰਟਾਂ ਵਿੱਚ ਪ੍ਰੋਡਕਸ਼ਨ-ਤਿਆਰ ਡਿਪਲੋਇ ਚਾਹੁੰਦਾ/ਚਾਹੁੰਦੀ ਹਾਂ" | `azd up --environment production` |

### ਇਹ ਇਕੱਠੇ ਕੰਮ ਕਰਦੇ ਹਨ!

AZD ਅੰਦਰੋਂ Azure CLI ਦੀ ਵਰਤੋਂ ਕਰਦਾ ਹੈ। ਤੁਸੀਂ ਦੋਹਾਂ ਨੂੰ ਵਰਤ ਸਕਦੇ ਹੋ:
```bash
# ਆਪਣੇ ਐਪ ਨੂੰ AZD ਨਾਲ ਤਾਇਨਾਤ ਕਰੋ
azd up

# ਫਿਰ Azure CLI ਨਾਲ ਵਿਸ਼ੇਸ਼ ਸਰੋਤਾਂ ਨੂੰ ਬਿਹਤਰ ਬਣਾਓ
az webapp config set --name myapp --always-on true
```


- AI Search (Basic ਟੀਅਰ): $75/ਮਹੀਨਾ

- Application Insights: $5-50/ਮਹੀਨਾ

**💡 ਲਾਗਤ ਘਟਾਉਣ ਦੇ ਸੁਝਾਅ:**

### **AZD + Microsoft Foundry: AI ਡਿਪਲੌਇਮੈਂਟਸ ਲਈ ਉਤਮ**

**AZD ਕਿਉਂ AI ਹੱਲਾਂ ਲਈ?** AZD ਉਹ ਸਭ ਤੋਂ ਵੱਡੀਆਂ ਚੁਣੌਤੀਆਂਾਂ ਦਾ ਹੱਲ ਕਰਦਾ ਹੈ ਜਿਹੜੀਆਂ AI ਡਿਵੈਲਪਰਾਂ ਸਾਹਮਣੇ ਹੁੰਦੀਆਂ ਹਨ:

<details>
<summary><strong>❌ "azd: command not found"</strong></summary>

```bash
# ਪਹਿਲਾਂ AZD ਇੰਸਟਾਲ ਕਰੋ
# ਵਿਂਡੋਜ਼ (ਪਾਵਰਸ਼ੈੱਲ):
winget install microsoft.azd

# ਮੈਕਓਐਸ:
brew tap azure/azd && brew install azd

# ਲਿਨਕਸ:
curl -fsSL https://aka.ms/install-azd.sh | bash

# ਇੰਸਟਾਲੇਸ਼ਨ ਦੀ ਪੁਸ਼ਟੀ ਕਰੋ
azd version
```
</details>

<details>
<summary><strong>❌ "No subscription found" or "Subscription not set"</strong></summary>

```bash
# ਉਪਲਬਧ ਸਬਸਕ੍ਰਿਪਸ਼ਨਾਂ ਦੀ ਸੂਚੀ
az account list --output table

# ਡਿਫੌਲਟ ਸਬਸਕ੍ਰਿਪਸ਼ਨ ਸੈੱਟ ਕਰੋ
az account set --subscription "<subscription-id-or-name>"

# AZD ਮਾਹੌਲ ਲਈ ਸੈੱਟ ਕਰੋ
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# ਪੁਸ਼ਟੀ ਕਰੋ
az account show
```
</details>

<details>
<summary><strong>❌ "InsufficientQuota" or "Quota exceeded"</strong></summary>

```bash
# ਕਿਸੇ ਹੋਰ Azure ਰੀਜਨ ਦੀ ਕੋਸ਼ਿਸ਼ ਕਰੋ
azd env set AZURE_LOCATION "westus2"
azd up

# ਜਾਂ ਵਿਕਾਸ ਵਿੱਚ ਛੋਟੇ SKUs ਦੀ ਵਰਤੋਂ ਕਰੋ
# infra/main.parameters.json ਨੂੰ ਸੋਧੋ:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>

<details>
<summary><strong>❌ "azd up" fails halfway through</strong></summary>

```bash
# ਵਿਕਲਪ 1: ਸਾਫ਼ ਕਰੋ ਅਤੇ ਮੁੜ ਕੋਸ਼ਿਸ਼ ਕਰੋ
azd down --force --purge
azd up

# ਵਿਕਲਪ 2: ਸਿਰਫ਼ ਢਾਂਚਾ ਠੀਕ ਕਰੋ
azd provision

# ਵਿਕਲਪ 3: ਵਿਸਤ੍ਰਿਤ ਸਥਿਤੀ ਜਾਂਚੋ
azd show

# ਵਿਕਲਪ 4: Azure Monitor ਵਿੱਚ ਲੌਗਾਂ ਦੀ ਜਾਂਚ ਕਰੋ
azd monitor --logs
```
</details>

<summary><strong>❌ "Authentication failed" or "Token expired"</strong></summary>

```bash
az login
```

```bash
azd down --force --purge

# ਫਿਰ ਤਾਜ਼ਾ ਵਾਤਾਵਰਣ ਨਾਲ ਦੁਬਾਰਾ ਕੋਸ਼ਿਸ਼ ਕਰੋ
azd env new dev-v2
```
