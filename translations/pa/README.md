## 🚀 Azure Developer CLI (azd) ਕੀ ਹੈ?

**Azure Developer CLI (azd)** ਇੱਕ ਡਿਵੈਲਪਰ-ਮਿੱਤਰ ਕਮਾਂਡ-ਲਾਈਨ ਟੂਲ ਹੈ ਜੋ ਐਜ਼ੁਰ 'ਤੇ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਡਿਪਲੋਏ ਕਰਨਾ ਆਸਾਨ ਬਣਾਉਂਦਾ ਹੈ। ਦਰਜਨ ਭਰ Azure ਸਰੋਤਾਂ ਨੂੰ ਹੱਥੋਂ-ਹੱਥ ਬਣਾਉਣ ਅਤੇ ਜੋੜਨ ਦੀ ਥਾਂ, ਤੁਸੀਂ ਇੱਕ ਹੀ ਕਮਾਂਡ ਨਾਲ ਪੂਰੇ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਡਿਪਲੋਏ ਕਰ ਸਕਦੇ ਹੋ।

### `azd up` ਦਾ ਜਾਦੂ

```bash
# ਇਹ ਇਕੋ ਇੱਕ ਕਮਾਂਡ ਸਭ ਕੁਝ ਕਰਦੀ ਹੈ:
# ✅ ਸਾਰੇ Azure ਸਰੋਤ ਬਣਾਉਂਦਾ ਹੈ
# ✅ ਨੈੱਟਵਰਕਿੰਗ ਅਤੇ ਸੁਰੱਖਿਆ ਸੰਰਚਿਤ ਕਰਦਾ ਹੈ
# ✅ ਤੁਹਾਡਾ ਐਪਲੀਕੇਸ਼ਨ ਕੋਡ ਬਣਾਉਂਦਾ ਹੈ
# ✅ Azure 'ਤੇ ਤਾਇਨਾਤ ਕਰਦਾ ਹੈ
# ✅ ਤੁਹਾਨੂੰ ਇੱਕ ਕੰਮ ਕਰ ਰਹੀ URL ਦਿੰਦਾ ਹੈ
azd up
```

**ਇਨੀ ਹੀ ਗੱਲ ਹੈ!** ਕੋਈ Azure ਪੋਰਟਲ 'ਤੇ ਕਲਿੱਕ ਕਰਨ ਦੀ ਲੋੜ ਨਹੀਂ, ਕੋਈ ਜਟਿਲ ARM ਟੈਂਪਲੇਟ ਪਹਿਲਾਂ ਸਿੱਖਣ ਦੀ ਲੋੜ ਨਹੀਂ, ਕੋਈ ਹੱਥੋਂ-ਹੱਥ ਕਨਫਿਗਰੇਸ਼ਨ ਨਹੀਂ - ਸਿਰਫ਼ ਕੰਮ ਕਰਨ ਵਾਲੀਆਂ ਐਪਸ Azure 'ਤੇ।

## ❓ Azure Developer CLI ਅਤੇ Azure CLI: ਕੀ ਫਰਕ ਹੈ?

ਇਹ ਸ਼ੁਰੂਆਤੀਆਂ ਵੱਲੋਂ ਸਭ ਤੋਂ ਆਮ ਸਵਾਲ ਹੈ। ਇਹ ਰਿਹਾ ਸਧਾਰਣ ਜਵਾਬ:

| Feature | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **Purpose** | ਵੱਖ-ਵੱਖ Azure ਸਰੋਤਾਂ ਦਾ ਪ੍ਰਬੰਧ | ਪੂਰੇ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਡਿਪਲੋਏ ਕਰੋ |
| **Mindset** | ਢਾਂਚਾ-ਕੇਂਦਰਿਤ | ਐਪਲੀਕੇਸ਼ਨ-ਕੇਂਦਰਿਤ |
| **Example** | `az webapp create --name myapp...` | `azd up` |
| **Learning Curve** | Azure ਸੇਵਾਵਾਂ ਬਾਰੇ ਜਾਣਕਾਰੀ ਲਾਜ਼ਮੀ | ਸਿਰਫ਼ ਆਪਣੀ ਐਪ ਜਾਣੋ |
| **Best For** | DevOps, ਢਾਂਚਾ | ਡਿਵੈਲਪਰ, ਪ੍ਰੋਟੋਟਾਈਪਿੰਗ |

### ਸਧਾਰਣ ਤੁਲਨਾ

- **Azure CLI** ਇਸ ਤਰ੍ਹਾਂ ਹੈ ਜਿਵੇਂ ਤੁਹਾਡੇ ਕੋਲ ਘਰ ਬਣਾਉਣ ਲਈ ਸਾਰੇ ਟੂਲ ਹੋਣ - ਹਥੌੜੇ, ਆਰੀਆਂ, ਕੀਲਾਂ। ਤੁਸੀਂ ਕੁਝ ਵੀ ਬਣਾ ਸਕਦੇ ਹੋ, ਪਰ ਤੁਹਾਨੂੰ ਨਿਰਮਾਣ ਦੇ ਗਿਆਨ ਦੀ ਲੋੜ ਹੈ।
- **Azure Developer CLI** ਇਸ ਤਰ੍ਹਾਂ ਹੈ ਜਿਵੇਂ ਤੁਸੀਂ ਇੱਕ ठੇਕੇਦਾਰ ਨੂੰ ਰੱਖਦੇ ਹੋ - ਤੁਸੀਂ ਦੱਸਦੇ ਹੋ ਕਿ ਤੁਸੀਂ ਕੀ ਚਾਹੁੰਦੇ ਹੋ, ਅਤੇ ਉਹ ਨਿਰਮਾਣ ਸੰਭਾਲ ਲੈਂਦਾ ਹੈ।

### ਕਦੋਂ ਕਿਹੜਾ ਵਰਤਣਾ ਹੈ

| Scenario | Use This |
|----------|----------|
| "ਮੈਂ ਆਪਣੀ ਵੈਬ ਐਪ ਤੇਜ਼ੀ ਨਾਲ ਡਿਪਲੋਏ ਕਰਨੀ ਚਾਹੁੰਦਾ/ਚਾਹੁੰਦੀ ਹਾਂ" | `azd up` |
| "ਮੈਨੂੰ صرف ਇੱਕ storage account ਬਣਾਉਣਾ ਹੈ" | `az storage account create` |
| "ਮੈਂ ਇੱਕ ਪੂਰੀ AI ਐਪ ਬਣਾ ਰਿਹਾ/ਰਹੀ ਹਾਂ" | `azd init --template azure-search-openai-demo` |
| "ਮੈਨੂੰ ਕਿਸੇ ਖਾਸ Azure ਸਰੋਤ ਨੂੰ ਡੀਬੱਗ ਕਰਨ ਦੀ ਲੋੜ ਹੈ" | `az resource show` |
| "ਮੈਂ ਮਿੰਟਾਂ ਵਿੱਚ ਪ੍ਰੋਡਕਸ਼ਨ-ਰੇਡੀ ਡਿਪਲੋਯਮੈਂਟ ਚਾਹੁੰਦਾ/ਚਾਹੁੰਦੀ ਹਾਂ" | `azd up --environment production` |

### ਇਹ ਇਕੱਠੇ ਕੰਮ ਕਰਦੇ ਹਨ!

AZD ਹੇਠਲਾ ਕੰਮ ਕਰਨ ਵਾਸਤੇ Azure CLI ਦਾ ਉਪਯੋਗ ਕਰਦਾ ਹੈ। ਤੁਸੀਂ ਦੋਹਾਂ ਵਰਤ ਸਕਦੇ ਹੋ:
```bash
# ਆਪਣੇ ਐਪ ਨੂੰ AZD ਨਾਲ ਤਾਇਨਾਤ ਕਰੋ
azd up

# ਫਿਰ Azure CLI ਨਾਲ ਖਾਸ ਸਰੋਤਾਂ ਨੂੰ ਸੁਧਾਰੋ
az webapp config set --name myapp --always-on true
```

### ਕਦਮ 1: AZD ਇੰਸਟਾਲ ਕਰੋ (2 ਮਿੰਟ)

**Windows:**
```powershell
winget install microsoft.azd
```

**macOS:**
```bash
brew tap azure/azd && brew install azd
```

**Linux:**
```bash
curl -fsSL https://aka.ms/install-azd.sh | bash
```

### ਕਦਮ 2: Azure ਵਿੱਚ ਲੌਗਿਨ ਕਰੋ

```bash
azd auth login
```

### ਕਦਮ 3: ਆਪਣੀ ਪਹਿਲੀ ਐਪ ਡਿਪਲੋਏ ਕਰੋ

```bash
# ਇੱਕ ਟੈਂਪਲੇਟ ਤੋਂ ਸ਼ੁਰੂ ਕਰੋ
azd init --template todo-nodejs-mongo

# ਅਜ਼ੂਰ 'ਤੇ ਤਾਇਨਾਤ ਕਰੋ (ਸਭ ਕੁਝ ਬਣਾਉਂਦਾ ਹੈ!)
```

## 🔧 ਤੁਰੰਤ ਸਮੱਸਿਆ-ਹੱਲ ਗਾਈਡ

**ਨਵੇਂ ਸ਼ੁਰੂਆਤੀਅਾਂ ਵੱਲੋਂ ਆਮ ਸਮੱਸਿਆਵਾਂ ਅਤੇ ਤੁਰੰਤ ਹੱਲ:**

<details>
<summary><strong>❌ "azd: ਕਮਾਂਡ ਨਹੀਂ ਮਿਲੀ"</strong></summary>

```bash
# ਸਭ ਤੋਂ ਪਹਿਲਾਂ AZD ਇੰਸਟਾਲ ਕਰੋ
# ਵਿੰਡੋਜ਼ (ਪਾਵਰਸ਼ੈੱਲ):
winget install microsoft.azd

# ਮੈਕਓਐਸ:
brew tap azure/azd && brew install azd

# ਲਿਨਕਸ:
curl -fsSL https://aka.ms/install-azd.sh | bash

# ਇੰਸਟਾਲੇਸ਼ਨ ਦੀ ਜਾਂਚ ਕਰੋ
azd version
```
</details>

<details>
<summary><strong>❌ "ਕੋਈ ਸਬਸਕ੍ਰਿਪਸ਼ਨ ਨਹੀਂ ਮਿਲੀ" ਜਾਂ "ਸਬਸਕ੍ਰਿਪਸ਼ਨ ਸੈੱਟ ਨਹੀਂ ਹੈ"</strong></summary>

```bash
# ਉਪਲਬਧ ਸਬਸਕ੍ਰਿਪਸ਼ਨਾਂ ਦੀ ਸੂਚੀ
az account list --output table

# ਡਿਫੌਲਟ ਸਬਸਕ੍ਰਿਪਸ਼ਨ ਸੈੱਟ ਕਰੋ
az account set --subscription "<subscription-id-or-name>"

# AZD ਵਾਤਾਵਰਣ ਲਈ ਸੈੱਟ ਕਰੋ
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# ਤਸਦੀਕ ਕਰੋ
az account show
```
</details>
<details>
<summary><strong>❌ "InsufficientQuota" ਜਾਂ "Quota exceeded"</strong></summary>

```bash
# ਵੱਖ-ਵੱਖ Azure ਰੀਜਨ ਅਜ਼ਮਾਓ
azd env set AZURE_LOCATION "westus2"
azd up

# ਜਾਂ ਵਿਕਾਸ ਵਿੱਚ ਛੋਟੇ SKUs ਵਰਤੋ
# infra/main.parameters.json ਨੂੰ ਸੋਧੋ:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>

<details>
<summary><strong>❌ "azd up" ਅੱਧੇ ਰਾਹ 'ਤੇ ਫੇਲ ਹੁੰਦਾ ਹੈ</strong></summary>

```bash
# ਵਿਕਲਪ 1: ਸਾਫ਼ ਕਰੋ ਅਤੇ ਮੁੜ ਕੋਸ਼ਿਸ਼ ਕਰੋ
azd down --force --purge
azd up

# ਵਿਕਲਪ 2: ਸਿਰਫ਼ ਇਨਫਰਾਸਟ੍ਰਕਚਰ ਠੀਕ ਕਰੋ
azd provision

# ਵਿਕਲਪ 3: ਵਿਸਤ੍ਰਤ ਸਥਿਤੀ ਦੀ ਜਾਂਚ ਕਰੋ
azd show

# ਵਿਕਲਪ 4: ਐਜ਼ੂਰ ਮਾਨੀਟਰ ਵਿੱਚ ਲੌਗਾਂ ਦੀ ਜਾਂچ ਕਰੋ
azd monitor --logs
```
</details>

<details>
<summary><strong>❌ "Authentication failed" ਜਾਂ "Token expired"</strong></summary>

```bash
# ਦੁਬਾਰਾ ਪ੍ਰਮਾਣਿਤ ਕਰੋ
az logout
az login

azd auth logout
azd auth login

# ਪ੍ਰਮਾਣਿਕਤਾ ਦੀ ਜਾਂਚ ਕਰੋ
az account show
```
</details>

<details>
<summary><strong>❌ "Resource already exists" ਜਾਂ ਨਾਮ ਰੁਕਾਵਟਾਂ</strong></summary>

```bash
# AZD ਵਿਲੱਖਣ ਨਾਮ ਪੈਦਾ ਕਰਦਾ ਹੈ, ਪਰ ਜੇ ਟਕਰਾਅ ਹੋਵੇ:
azd down --force --purge

# ਫਿਰ ਨਵੇਂ ਵਾਤਾਵਰਣ ਨਾਲ ਦੁਬਾਰਾ ਕੋਸ਼ਿਸ਼ ਕਰੋ
azd env new dev-v2
azd up
```
</details>

<details>
<summary><strong>❌ ਟੈਮਪਲੇਟ ਡਿਪਲੋਇਮੈਂਟ ਬਹੁਤ ਸਮਾਂ ਲੈ ਰਿਹਾ ਹੈ</strong></summary>

**ਆਮ ਉਡੀਕ ਸਮੇਂ:**
- ਸਧਾਰਨ ਵੈੱਬ ਐਪ: 5-10 ਮਿੰਟ
- ਡੇਟਾਬੇਸ ਵਾਲਾ ਐਪ: 10-15 ਮਿੰਟ
- AI ਐਪਲੀਕੇਸ਼ਨ: 15-25 ਮਿੰਟ (OpenAI ਪ੍ਰੋਵੀਜ਼ਨਿੰਗ ਹੌਲੀ ਹੁੰਦੀ ਹੈ)

```bash
# ਪ੍ਰਗਤੀ ਦੀ ਜਾਂਚ ਕਰੋ
azd show

# ਜੇ 30 ਮਿੰਟ ਤੋਂ ਵੱਧ ਫਸਿਆ ਹੋਵੇ, Azure ਪੋਰਟਲ ਚੈੱਕ ਕਰੋ:
azd monitor

```
</details>

<details>
<summary><strong>❌ "Permission denied" ਜਾਂ "Forbidden"</strong></summary>

```bash
az role assignment list --assignee $(az account show --query user.name -o tsv)

# ਤੁਹਾਨੂੰ ਘੱਟੋ-ਘੱਟ "Contributor" ਰੋਲ ਦੀ ਲੋੜ ਹੈ
```
