## 🆕 ਅੱਜ azd ਵਿੱਚ ਕੀ ਨਵਾਂ ਹੈ

Azure Developer CLI ਰਵਾਇਤੀ ਵੈੱਬ ਐਪਸ ਅਤੇ APIs ਤੋਂ ਅੱਗੇ ਵਧ ਗਿਆ ਹੈ। ਅੱਜ, azd ਇੱਕ ਇਕੱਲਾ ਟੂਲ ਹੈ ਜਿਸ ਨਾਲ ਤੁਸੀਂ Azure 'ਤੇ ਕਿਸੇ ਵੀ ਐਪਲੀਕੇਸ਼ਨ ਨੂੰ ਡਿਪਲੌਏ ਕਰ ਸਕਦੇ ਹੋ—ਇਸ ਵਿੱਚ AI-ਚਲਿਤ ਐਪਲੀਕੇਸ਼ਨ ਅਤੇ ਬੁੱਧੀਮਾਨ ਏਜੰਟ ਵੀ ਸ਼ਾਮਲ ਹਨ।

ਇਸਦਾ ਤੁਹਾਡੇ ਲਈ ਕੀ ਮਤਲਬ ਹੈ:

- **AI ਏਜੰਟ ਹੁਣ azd ਦੇ ਮੁੱਖ-ਪੱਧਰ ਵਾਲੇ ਵਰਕਲੋਡ ਹਨ।** ਤੁਸੀਂ ਉਹੀ `azd init` → `azd up` ਵਰਕਫਲੋਅ ਵਰਤ ਕੇ AI ਏਜੰਟ ਪ੍ਰੋਜੈਕਟ ਸ਼ੁਰੂ, ਡਿਪਲੌਏ, ਅਤੇ ਪ੍ਰਬੰਧਿਤ ਕਰ ਸਕਦੇ ਹੋ ਜਿਸ ਨੂੰ ਤੁਸੀਂ ਪਹਿਲਾਂ ਜਾਣਦੇ ਹੋ।
- **Microsoft Foundry ਇਨਟੀਗਰੇਸ਼ਨ** ਮਾਡਲ ਡਿਪਲੌਇਮੈਂਟ, ਏਜੰਟ ਹੋਸਟਿੰਗ, ਅਤੇ AI ਸਰਵਿਸ ਕੰਫ਼ਿਗਰੇਸ਼ਨ ਨੂੰ ਸਿੱਧਾ azd ਟੈੰਪਲੇਟ ਇਕੋਸਿਸਟਮ ਵਿੱਚ ਲਿਆਉਂਦਾ ਹੈ।
- **ਮੁੱਖ ਵਰਕਫਲੋਅ ਬਦਲਿਆ ਨਹੀਂ ਗਿਆ।** ਚਾਹੇ ਤੁਸੀਂ ਇੱਕ todo ਐਪ, ਇੱਕ ਮਾਈਕ੍ਰੋਸਰਵਿਸ, ਜਾਂ ਇੱਕ ਮਲਟੀ-ਏਜੰਟ AI ਹੱਲ ਡਿਪਲੌਏ ਕਰ ਰਹੇ ਹੋ, ਕਮਾਂਡ ਇੱਕੋ ਜਿਹੀਆਂ ਹੀ ਹਨ।

ਜੇ ਤੁਸੀਂ ਪਹਿਲਾਂ azd ਵਰਤ ਚੁੱਕੇ ਹੋ, ਤਾਂ AI ਸਹਿਯੋਗ ਇੱਕ ਤਰਕਸੀਕ ਤੌਰ ਤੇ ਫੈਲਾਵ ਹੈ—ਕੋਈ ਵੱਖਰਾ ਟੂਲ ਜਾਂ ਉन्नਤ ਟਰੈਕ ਨਹੀਂ। ਜੇ ਤੁਸੀਂ ਨਵੇਂ ਹੋ, ਤਾਂ ਤੁਸੀਂ ਇੱਕ ਹੀ ਵਰਕਫਲੋਅ ਸਿੱਖੋਗੇ ਜੋ ਹਰ ਚੀਜ਼ ਲਈ ਕੰਮ ਕਰਦਾ ਹੈ।

## 🚀 Azure Developer CLI (azd) ਕੀ ਹੈ?

**Azure Developer CLI (azd)** ਇੱਕ ਡਿਵੈਲਪਰ-ਅਨੁਕੂਲ ਕਮਾਂਡ-ਲਾਈਨ ਟੂਲ ਹੈ ਜੋ ਐਜ਼ੂਰ 'ਤੇ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਡਿਪਲੌਏ ਕਰਨਾ ਆਸਾਨ ਬਣਾਉਂਦਾ ਹੈ। ਦਹਾਕਿਆਂ Azure ਰਿਸੋਰਸਜ਼ ਨੂੰ ਹੱਥੋਂ-ਹੱਥ ਬਣਾਏ ਬਿਨਾਂ, ਤੁਸੀਂ ਇੱਕ ਹੀ ਕਮਾਂਡ ਨਾਲ ਪੂਰੀਆਂ ਐਪਲੀਕੇਸ਼ਨਾਂ ਡਿਪਲੌਏ ਕਰ ਸਕਦੇ ਹੋ।

### `azd up` ਦਾ ਜਾਦੂ

```bash
# ਇਹ ਇਕ ਹੀ ਕਮਾਂਡ ਸਾਰਾ ਕੁਝ ਕਰਦੀ ਹੈ:
# ✅ ਸਾਰੇ Azure ਸਰੋਤ ਬਣਾਉਂਦੀ ਹੈ
# ✅ ਨੈੱਟਵਰਕਿੰਗ ਅਤੇ ਸੁਰੱਖਿਆ ਨੂੰ ਸੰਰਚਿਤ ਕਰਦੀ ਹੈ
# ✅ ਤੁਹਾਡੇ ਐਪਲੀਕੇਸ਼ਨ ਕੋਡ ਨੂੰ ਬਣਾਉਂਦੀ ਹੈ
# ✅ Azure 'ਤੇ ਡਿਪਲੋਈ ਕਰਦੀ ਹੈ
# ✅ ਤੁਹਾਨੂੰ ਕੰਮ ਕਰਨ ਵਾਲਾ URL ਦਿੰਦੀ ਹੈ
azd up
```

**ਇਹੀ ਹੈ!** ਕੋਈ Azure ਪੋਰਟਲ ਤੇ ਕਲਿਕ ਕਰਨਾ ਨਹੀਂ, ਕੋਈ ਜਟਿਲ ARM ਟੈੰਪਲੇਟ ਪਹਿਲਾਂ ਸਿੱਖਣ ਦੀ ਲੋੜ ਨਹੀਂ, ਕੋਈ ਹੱਥੋਂ-ਹੱਥ ਕਨਫ਼ਿਗਰੇਸ਼ਨ ਨਹੀਂ - ਸਿਰਫ਼ Azure 'ਤੇ ਕੰਮ ਕਰਨ ਵਾਲੀਆਂ ਐਪਲੀਕੇਸ਼ਨਜ਼।

## ❓ Azure Developer CLI ਅਤੇ Azure CLI: ਫਰਕ ਕੀ ਹੈ?

ਇਹ ਸਭ ਤੋਂ ਆਮ ਸਵਾਲ ਹੈ ਜੋ ਸ਼ੁਰੂਆਤੀ ਪੁੱਛਦੇ ਹਨ। ਇਥੇ ਸਧਾਰਨ ਜਵਾਬ ਹੈ:

| ਫੀਚਰ | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **ਮਕਸਦ** | ਅਲੱਗ-ਅਲੱਗ Azure ਰਿਸੋਰਸز ਦਾ ਪ੍ਰਬੰਧ | ਪੂਰੀਆਂ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਡਿਪਲੌਏ ਕਰਨਾ |
| **ਮਨੋਵ੍ਰਿਤੀ** | ਇਨਫਰਾਸਟ੍ਰਕਚਰ-ਕੇਂਦਰਤ | ਐਪਲੀਕੇਸ਼ਨ-ਕੇਂਦਰਤ |
| **ਉਦਾਹরণ** | `az webapp create --name myapp...` | `azd up` |
| **ਸਿੱਖਣ ਦਾ ਮਾਰਗ** | Azure ਸਰਵਿਸਜ਼ ਦਾ ਗਿਆਨ ਲਾਜ਼ਮੀ | ਸਿਰਫ਼ ਆਪਣੀ ਐਪ ਨੂੰ ਜਾਣੋ |
| **ਸਭ ਤੋਂ ਵਧੀਆ** | DevOps, ਇਨਫਰਾਸਟ੍ਰਕਚਰ | ਡਿਵੈਲਪਰ, ਪ੍ਰੋਟੋਟਾਈਪਿੰਗ |

### ਸਧਾਰਨ ਤੁਲਨਾ

- **Azure CLI** ਉਸੇ ਤਰ੍ਹਾਂ ਹੈ ਜਿਵੇਂ ਘਰ ਬਣਾਉਣ ਲਈ ਸਾਰੇ ਔਜ਼ਾਰ ਹੋਣ—ਹਥੋੜੇ, ਆਰੀਆਂ, ਕੀਲਾਂ। ਤੁਸੀਂ ਕੁਝ ਵੀ ਬਣਾ ਸਕਦੇ ਹੋ, ਪਰ ਤੁਹਾਨੂੰ ਨਿਰਮਾਣ ਦਾ ਗਿਆਨ ਹੋਣਾ ਚਾਹੀਦਾ ਹੈ।
- **Azure Developer CLI** ਉਸੇ ਤਰ੍ਹਾਂ ਹੈ ਜਿਵੇਂ ਇੱਕ ठੇਕੈਦਾਰ ਨੂੰ ਕਿਰਾਏ 'ਤੇ ਲੈ ਲੈਣਾ—ਤੁਸੀਂ ਦੱਸਦੇ ਹੋ ਕਿ ਤੁਸੀਂ ਕੀ ਚਾਹੁੰਦੇ ਹੋ, ਅਤੇ ਉਹ ਨਿਰਮਾਣ ਦਾ ਕੰਮ Sambhal ਲੈਂਦਾ ਹੈ।

### ਕਿਸ ਸਮੇਂ ਕਿਸ ਦਾ ਇਸਤੇਮਾਲ ਕਰਨਾ ਚਾਹੀਦਾ ਹੈ

| ਪਰਿਦ੍ਰਿਸ਼ | ਇਸਨੂੰ ਵਰਤੋ |
|----------|----------|
| "ਮੈਂ ਆਪਣੀ ਵੈੱਬ ਐਪ ਜਲਦੀ ਡਿਪਲੌਏ ਕਰਨਾ ਚਾਹੁੰਦਾ ਹਾਂ" | `azd up` |
| "ਮੈਨੂੰ ਸਿਰਫ਼ ਇੱਕ storage account ਬਣਾਉਣ ਦੀ ਲੋੜ ਹੈ" | `az storage account create` |
| "ਮੈਂ ਇੱਕ ਪੂਰੀ AI ਐਪ ਬਣਾਉ ਰਿਹਾ ਹਾਂ" | `azd init --template azure-search-openai-demo` |
| "ਮੈਨੂੰ ਕਿਸੇ ਖਾਸ Azure ਰਿਸੋਰਸ ਨੂੰ ਡੀਬੱਗ ਕਰਨਾ ਹੈ" | `az resource show` |
| "ਮੈਂ ਚਾਹੁੰਦਾ ਹਾਂ ਕਿ ਮਿੰਟਾਂ ਵਿੱਚ ਪ੍ਰੋਡਕਸ਼ਨ-ਤਿਆਰ ਡਿਪਲੌਏ ਹੋਵੇ" | `azd up --environment production` |

### ਇਹ ਇਕੱਠੇ ਵੀ ਕੰਮ ਕਰਦੇ ਹਨ!

AZD ਅੰਦਰੋਂ Azure CLI ਨੂੰ ਵਰਤਦਾ ਹੈ। ਤੁਸੀਂ ਦੋਹਾਂ ਵਰਤ ਸਕਦੇ ਹੋ:
```bash
# ਆਪਣੇ ਐਪ ਨੂੰ AZD ਨਾਲ ਤਾਇਨਾਤ ਕਰੋ
azd up

# ਫਿਰ Azure CLI ਨਾਲ ਵਿਸ਼ੇਸ਼ ਸਰੋਤਾਂ ਨੂੰ ਬਰੀਕੀ ਨਾਲ ਸੁਧਾਰੋ
az webapp config set --name myapp --always-on true
```

## 🔧 Quick Troubleshooting Guide

**ਨਵੀਨਕਾਰਾਂ ਨੂੰ ਆਮ ਸਮੱਸਿਆਵਾਂ ਅਤੇ ਤੁਰੰਤ ਹੱਲ:**

<details>
<summary><strong>❌ "azd: command not found"</strong></summary>

```bash
# ਪਹਿਲਾਂ AZD ਇੰਸਟਾਲ ਕਰੋ
# ਵਿਂਡੋਜ਼ (ਪਾਵਰਸ਼ੈੱਲ):
winget install microsoft.azd

# ਮੈਕਓਐਸ:
brew tap azure/azd && brew install azd

# ਲਿਨਕਸ:
curl -fsSL https://aka.ms/install-azd.sh | bash

# ਇੰਸਟਾਲੇਸ਼ਨ ਦੀ ਪੁਸ਼ਟੀ ਕਰੋ
azd version
```
</details>

<details>
<summary><strong>❌ "No subscription found" or "Subscription not set"</strong></summary>

```bash
# ਉਪਲਬਧ ਸਬਸਕ੍ਰਿਪਸ਼ਨਾਂ ਦੀ ਸੂਚੀ
az account list --output table

# ਡਿਫਾਲਟ ਸਬਸਕ੍ਰਿਪਸ਼ਨ ਸੈੱਟ ਕਰੋ
az account set --subscription "<subscription-id-or-name>"

# AZD ਵਾਤਾਵਰਨ ਲਈ ਸੈੱਟ ਕਰੋ
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# ਪੁਸ਼ਟੀ ਕਰੋ
az account show
```
</details>

<details>
<summary><strong>❌ "InsufficientQuota" or "Quota exceeded"</strong></summary>

```bash
# ਕੋਈ ਵੱਖਰਾ Azure ਰੀਜਨ ਆਜ਼ਮਾਓ
azd env set AZURE_LOCATION "westus2"
azd up

# ਜਾਂ ਡਿਵੈਲਪਮੈਂਟ ਵਿੱਚ ਛੋਟੇ SKUs ਵਰਤੋ
# infra/main.parameters.json ਨੂੰ ਸੋਧੋ:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>

<details>
<summary><strong>❌ "azd up" fails halfway through</strong></summary>

```bash
# ਵਿਕਲਪ 1: ਸਾਫ਼ ਕਰੋ ਅਤੇ ਫਿਰ ਕੋਸ਼ਿਸ਼ ਕਰੋ
azd down --force --purge
azd up

# ਵਿਕਲਪ 2: ਸਿਰਫ ਬੁਨਿਆਦੀ ਢਾਂਚਾ ਠੀਕ ਕਰੋ
azd provision

# ਵਿਕਲਪ 3: ਵਿਸਤ੍ਰਿਤ ਸਥਿਤੀ ਦੀ ਜਾਂਚ ਕਰੋ
azd show

# ਵਿਕਲਪ 4: Azure Monitor ਵਿੱਚ ਲੌਗਾਂ ਦੀ ਜਾਂਚ ਕਰੋ
azd monitor --logs
```
</details>

<details>
<summary><strong>❌ "Authentication failed" or "Token expired"</strong></summary>

```bash
# ਮੁੜ ਪ੍ਰਮਾਣਿਕਤਾ ਕਰੋ
az logout
az login

azd auth logout
azd auth login

# ਪ੍ਰਮਾਣਿਕਤਾ ਦੀ ਪੁਸ਼ਟੀ ਕਰੋ
az account show
```
</details>

<details>
<summary><strong>❌ "Resource already exists" or naming conflicts</strong></summary>

```bash
# AZD ਵਿਲੱਖਣ ਨਾਮ ਬਣਾਉਂਦਾ ਹੈ, ਪਰ ਜੇ ਟਕਰਾਅ ਹੋਵੇ:
azd down --force --purge

# ਫਿਰ ਤਾਜ਼ਾ ਮਾਹੌਲ ਨਾਲ ਦੁਬਾਰਾ ਕੋਸ਼ਿਸ਼ ਕਰੋ
azd env new dev-v2
azd up
```
</details>

<details>
<summary><strong>❌ ਟੈਂਪਲੇਟ ਡਿਪਲੌਇਮੈਂਟ ਬਹੁਤ ਦੇਰ ਲੈ ਰਿਹਾ ਹੈ</strong></summary>

**ਆਮ ਉਡੀਕ ਸਮਾਂ:**
- ਸਧਾਰਨ ਵੈੱਬ ਐਪ: 5-10 ਮਿੰਟ
- ਡੇਟਾਬੇਸ ਵਾਲੀ ਐਪ: 10-15 ਮਿੰਟ
- AI ਐਪਲੀਕੇਸ਼ਨ: 15-25 ਮਿੰਟ (OpenAI provisioning ਧੀਮਾ ਹੁੰਦਾ ਹੈ)

```bash
# ਤਰੱਕੀ ਦੀ ਜਾਂਚ ਕਰੋ
azd show

# ਜੇ 30 ਮਿੰਟ ਤੋਂ ਵੱਧ ਫਸਿਆ ਰਹੋ, ਤਾਂ Azure ਪੋਰਟਲ ਚੈੱਕ ਕਰੋ:
azd monitor
# ਫੇਲ ਹੋਈਆਂ ਡਿਪਲੌਇਮੈਂਟਾਂ ਵੇਖੋ
```
</details>

<details>
<summary><strong>❌ "Permission denied" or "Forbidden"</strong></summary>

```bash
# ਆਪਣੇ Azure ਰੋਲ ਦੀ ਜਾਂਚ ਕਰੋ
az role assignment list --assignee $(az account show --query user.name -o tsv)

# ਤੁਹਾਨੂੰ ਘੱਟੋ ਘੱਟ "Contributor" ਰੋਲ ਦੀ ਲੋੜ ਹੈ
# ਆਪਣੇ Azure ਐਡਮਿਨ ਨੂੰ ਇਹ ਅਧਿਕਾਰ ਦੇਣ ਲਈ ਕਹੋ:
# - Contributor (ਸਰੋਤਾਂ ਲਈ)
# - User Access Administrator (ਭੂਮਿਕਾ ਸੌਂਪਣ ਲਈ)
```
</details>

<details>
<summary><strong>❌ Can't find deployed application URL</strong></summary>

```bash
# ਸਾਰੇ ਸੇਵਾ ਐਂਡਪਾਇੰਟ ਦਿਖਾਓ
azd show

# ਜਾਂ Azure ਪੋਰਟਲ ਖੋਲ੍ਹੋ
azd monitor

# ਕਿਸੇ ਵਿਸ਼ੇਸ਼ ਸੇਵਾ ਦੀ ਜਾਂਚ ਕਰੋ
azd env get-values
# *_URL ਵੈਰੀਏਬਲ ਲੱਭੋ
```
</details>
