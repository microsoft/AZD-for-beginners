## 🆕 ਅੱਜ azd ਵਿੱਚ ਨਵਾਂ ਕੀ ਹੈ

> 📌 ਇਹ ਕੋਰਸ **`azd 1.25.6`** (ਜੂਨ 2026) ਦੇ ਵਿਰੁੱਧ ਸਬੂਤ ਕੀਤਾ ਗਿਆ ਹੈ। ਆਪਣੀ ਬਣਾਉਟ ਚੈੱਕ ਕਰਨ ਲਈ `azd version` ਚਲਾਓ, ਅਤੇ ਨਵਾਂ ਪਾਉਣ ਲਈ `azd upgrade` ਚਲਾਓ।

Azure Developer CLI ਹੁਣ ਰਵਾਇਤੀ ਵੈਬ ਐਪਸ ਅਤੇ API ਤੋਂ ਬਹੁਤ ਅੱਗੇ ਵਧ ਚੁੱਕਾ ਹੈ। ਅੱਜ, azd Azure ਤੇ ਕਿਸੇ ਵੀ ਐਪਲੀਕੇਸ਼ਨ — ਜਿਸ ਵਿੱਚ AI-ਚਲਿਤ ਐਪਲੀਕੇਸ਼ਨ ਅਤੇ ਬੁੱਧੀਮਾਨ ਏਜੰਟ ਵੀ ਸ਼ਾਮਲ ਹਨ — ਨੂੰ ਡਿਪਲੋਏ ਕਰਨ ਲਈ ਇੱਕ ਹੀ ਟੂਲ ਹੈ।

ਇਸਦਾ ਤੁਹਾਡੇ ਲਈ ਕੀ ਮਤਲਬ ਹੈ:

- **AI ਏਜੰਟ ਹੁਣ ਪਹਿਲੀ ਕਲਾਸ azd ਵਕਰੀਆਂ ਹਨ।** ਤੁਸੀਂ ਜਾਣ ਪਛਾਣ ਵਾਲੇ `azd init` → `azd up` ਵਰਕਫਲੋ ਨਾਲ AI ਏਜੰਟ ਪ੍ਰੋਜੈਕਟਾਂ ਨੂੰ ਸ਼ੁਰੂ, ਡਿਪਲੋਏ ਅਤੇ ਮੈਨੇਜ ਕਰ ਸਕਦੇ ਹੋ।
- **CLI ਤੋਂ ਪੂਰਾ ਏਜੰਟ ਲਾਈਫਸਾਇਕਲ।** `azure.ai.agents` ਐਕਸਟੇਨਸ਼ਨ ਹੁਣ ਸਾਰੀ ਯਾਤਰਾ ਕਵਰ ਕਰਦਾ ਹੈ—ਸਕੈਫੋਲਡ ਕਰਨ ਲਈ `azd ai agent init`, ਟੈਸਟ ਕਰਨ ਲਈ `azd ai agent invoke` (ਜਿੱਦ ਵਿੱਚ ਰਿਸਪਾਂਸ-ਟਾਈਮਿੰਗ ਆਊਟਪੁੱਟ), ਗੁਣਵੱਤਾ ਮਾਪਣ ਅਤੇ ਸੁਧਾਰਨ ਲਈ `azd ai agent eval generate` ਅਤੇ `azd ai agent optimize`, ਅਤੇ ਸਾਫ਼ ਕਰਨ ਲਈ `azd ai agent delete`।
- **ਹੋਰ AI ਬਿਲਡਿੰਗ ਬਲਾਕ।** ਨਵੇਂ ਪ੍ਰੀਵਿਊ ਐਕਸਟੇਨਸ਼ਨ—`azure.ai.skills` ਅਤੇ `azure.ai.connections`—ਤੁਹਾਨੂੰ azd ਨਾਲ ਸਿੱਧਾ ਦੁਹਰਾਏ ਜਾ ਸਕਣ ਵਾਲੇ ਏਜੰਟ ਸ.killਸ ਅਤੇ Foundry ਕਨੈਕਸ਼ਨਾਂ ਨੂੰ ਮੈਨੇਜ ਕਰਨ ਦਿੰਦੇ ਹਨ।
- **Microsoft Foundry ਇਨਟੀਗ੍ਰੇਸ਼ਨ** ਮਾਡਲ ਡਿਪਲੋਇੰਟ, ਏਜੰਟ ਹੋਸਟਿੰਗ, ਅਤੇ AI ਸਰਵਿਸ ਕਨਫ਼ਿਗਰੇਸ਼ਨ ਨੂੰ azd ਟੈਮਪਲੇਟ ਏਕੋਸਿਸਟਮ ਵਿੱਚ ਸਿੱਧਾ ਲਿਆਉਂਦਾ ਹੈ।
- **ਰੋਜ਼ਾਨਾ ਦੀਆਂ ਬੁਨਿਆਦੀ ਗੱਲਾਂ ਹੋਰ ਸੁਚੱਜੀਆਂ।** ਹਾਲੀਆ ਰਿਲੀਜਾਂ ਨੇ `azd init` ਨੂੰ ਆਇਡੈਂਪੋਟੇਂਟ (ਮੁੜ ਚਲਾਉਣ ਲਈ ਸੁਰੱਖਿਅਤ) ਬਣਾਇਆ, `azd auth login` ਨੂੰ ਹੋਰ ਸਪਸ਼ਟ ਤਰੀਕੇ ਨਾਲ ਬੁਜ਼ੁਰਗ ਟੋਕਨਾਂ ਨੂੰ ਆਟੋਮੈਟਿਕਲੀ ਸਾਫ਼ ਕਰਨ ਜੋੜਿਆ, ਅਤੇ ਇੱਕ ਮਿੱਤਰ-ਭਾਵ ਵਾਲਾ `azd tool` ਪਹਿਲੀ ਵਾਰੀ ਸੈਟਅਪ ਪ੍ਰੰਪਟ ਸ਼ਾਮਲ ਕੀਤਾ।
- **ਕੋਰ ਵਰਕਫਲੋ ਬਦਲਿਆ ਨਹੀਂ।** ਚਾਹੇ ਤੁਸੀਂ ਇੱਕ todo ਐਪ, ਇੱਕ ਮਾਈਕ੍ਰੋਸਰਵਿਸ, ਜਾਂ ਇੱਕ ਬਹੁ-ਏਜੰਟ AI ਹੱਲ ਡਿਪਲੋਈ ਕਰ ਰਹੇ ਹੋ, ਕਮਾਂਡ ਉਹੀ ਰਹਿੰਦੀਆਂ ਹਨ।

> **Aspire ਉਪਭੋਗਤਿਆਂ ਲਈ ਨੋਟ:** Microsoft ਹੁਣ ਉਤਪਾਦ ਲਈ ਸਿਰਫ **Aspire** (ਪਹਿਲਾਂ ".NET Aspire") ਨਾਮ ਵਰਤਦਾ ਹੈ। azd ਦੀ Aspire ਸਹਾਇਤਾ ਬਦਲੀ ਨਹੀਂ—ਸਿਰਫ ਨਾਮ ਅਪਡੇਟ ਕੀਤਾ ਗਿਆ ਸੀ।

ਜੇ ਤੁਸੀਂ ਪਹਿਲਾਂ azd ਵਰਤਿਆ ਹੈ, ਤਾਂ AI ਸਹਾਇਤਾ ਇੱਕ ਕੁਦਰਤੀ ਵਾਧਾ ਹੈ—ਇਹ ਕੋਈ ਵੱਖਰਾ ਟੂਲ ਜਾਂ ਉੱਚ ਦਰਜੇ ਦਾ ਟ੍ਰੈਕ ਨਹੀਂ ਹੈ। ਜੇ ਤੁਸੀਂ ਨਵੇਂ ਹੋ, ਤਾਂ ਤੁਸੀਂ ਇੱਕ ਐਸਾ ਵਰਕਫਲੋ ਸਿੱਖੋਗੇ ਜੋ ਹਰ ਚੀਜ਼ ਲਈ ਕੰਮ ਕਰਦਾ ਹੈ।

## 🚀 Azure Developer CLI (azd) ਕੀ ਹੈ?

**Azure Developer CLI (azd)** ਇੱਕ ਡਿਵੈਲਪਰ-ਮੈਤ੍ਰੀ ਕਮਾਂਡ-ਲਾਈਨ ਟੂਲ ਹੈ ਜੋ Azure ਤੇ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਡਿਪਲੋਏ ਕਰਨਾ ਸੌਖਾ ਬਣਾਉਂਦਾ ਹੈ। ਦਸਾਂ-ਦਸਾਂ Azure ਰਿਸੋਰਸ ਨੂੰ ਮੈਨੂਅਲੀ ਤਰੀਕੇ ਨਾਲ ਬਣਾਉਣ ਅਤੇ ਕਨੈਕਟ ਕਰਨ ਬਦਲੇ, ਤੁਸੀਂ ਇੱਕ ਹੀ ਕਮਾਂਡ ਨਾਲ ਪੂਰੀਆਂ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਡਿਪਲੋਏ ਕਰ ਸਕਦੇ ਹੋ।

### `azd up` ਦਾ ਜਾਦੂ

```bash
# ਇਹ ਇੱਕ ਕਮਾਂਡ ਸਭ ਕੁਝ ਕਰਦਾ ਹੈ:
# ✅ ਸਾਰੇ ਏਜ਼ੂਰ ਸਰੋਤ ਬਣਾਉਂਦਾ ਹੈ
# ✅ ਨੈੱਟਵਰਕਿੰਗ ਅਤੇ ਸੁਰੱਖਿਆ ਸੈਟ ਕਰਦਾ ਹੈ
# ✅ ਤੁਹਾਡੇ ਐਪਲੀਕੇਸ਼ਨ ਕੋਡ ਨੂੰ ਬਣਾਉਂਦਾ ਹੈ
# ✅ ਏਜ਼ੂਰ 'ਤੇ ਡਿਪਲੋਇ ਕਰਦਾ ਹੈ
# ✅ ਤੁਹਾਨੂੰ ਕਾਰਜ ਕਰਨ ਵਾਲਾ ਯੂਆਰਐਲ ਦਿੰਦਾ ਹੈ
azd up
```

**ਬਸ ਹੋ ਗਿਆ!** ਕੋਈ Azure Portal 'ਤੇ ਕਲਿਕ ਕਰਨ ਦੀ ਲੋੜ ਨਹੀਂ, ਕੋਈ ਔਖੇ ARM ਟੈਮਪਲੇਟ ਪਹਿਲਾਂ ਸਿੱਖਣ ਦੀ ਲੋੜ ਨਹੀਂ, ਕੋਈ ਮੈਨੂਅਲ ਕੰਫਿਗਰੇਸ਼ਨ ਨਹੀਂ - ਸਿਰਫ਼ Azure ਤੇ ਕੰਮ ਕਰ ਰਹੀਆਂ ਐਪਲੀਕੇਸ਼ਨਾਂ।

## ❓ Azure Developer CLI ਬਨਾਮ Azure CLI: ਫਰਕ ਕੀ ਹੈ?

ਇਹ ਸ਼ੁਰੂਆਤੀ ਲੋਕਾਂ ਤੋਂ ਸਭ ਤੋਂ ਆਮ ਸਵਾਲ ਹੈ। ਇਹ ਰਿਹਾ ਸਧਾਰਨ ਜਵਾਬ:

| ਫੀਚਰ | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **ਮਕਸਦ** | ਵਿਅਕਤੀਗਤ Azure ਰਿਸੋਰਸਾਂ ਦਾ ਪ੍ਰਬੰਧਨ | ਪੂਰੀਆਂ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਡਿਪਲੋਏ ਕਰਨਾ |
| **ਮਨੋਭਾਵ** | ਇੰਫਰਾਸਟਰਕਚਰ-ਕੇਂਦ੍ਰਿਤ | ਐਪਲੀਕੇਸ਼ਨ-ਕੇਂਦ੍ਰਿਤ |
| **ਉਦਾਹਰਨ** | `az webapp create --name myapp...` | `azd up` |
| **ਸਿੱਖਣ ਵਕਲ਼** | Azure ਸਰਵਿਸਾਂ ਦੇ ਗਿਆਨ ਦੀ ਲੋੜ | ਸਿਰਫ ਆਪਣੀ ਐਪ ਜਾਣੋ |
| **ਕਿਸ ਲਈ ਵਧੀਆ** | DevOps, ਇੰਫਰਾਸਟਰਕਚਰ | ਡਿਵੈਲਪਰ, ਪ੍ਰੋਟੋਟਾਈਪਿੰਗ |

### ਸਧਾਰਨ ਉਦਾਹਰਨ

- **Azure CLI** ਇਕ ਘਰ ਬਣਾਉਣ ਲਈ ਸਾਰੇ ਟੂਲਾਂ ਵਰਗਾ ਹੈ - ਹੈਮਰ, ਕਾਸ਼, ਨੈਲ। ਤੁਸੀਂ ਕੁਝ ਵੀ ਬਣਾ ਸਕਦੇ ਹੋ, ਪਰ ਤੁਹਾਨੂੰ ਨਿਰਮਾਣ ਦਾ ਗਿਆਨ ਹੋਣਾ ਚਾਹੀਦਾ ਹੈ।
- **Azure Developer CLI** ਇੱਕ ਠੇਕੇਦਾਰ ਰੱਖਣ ਵਾਲਾ ਹੈ - ਤੁਸੀਂ ਦੱਸਦੇ ਹੋ ਕਿ ਤੁਸੀਂ ਕੀ ਚਾਹੁੰਦੇ ਹੋ, ਅਤੇ ਉਹ ਨਿਰਮਾਣ ਦਾ ਕੰਮ ਸੰਭਾਲ ਲੈਂਦਾ ਹੈ।

### ਕਦੋਂ ਕਿਸਦਾ ਉਪਯੋਗ ਕਰੀਏ

## ਸ਼ੁਰੂ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ: ਖਾਤੇ, ਐਕਸੇਸ, ਅਤੇ ਧਾਰਨਾਵਾਂ

ਅਧਿਆਇ 1 ਸ਼ੁਰੂ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ, ਇਹ ਹਨ ਚੀਜ਼ਾਂ ਜੋ ਤੁਹਾਡੇ ਕੋਲ ਹੋਣੀਆਂ ਚਾਹੀਦੀਆਂ ਹਨ। ਇਸ ਗਾਈਡ ਵਿੱਚ ਬਾਅਦ ਵਿੱਚ ਦਿੱਤੇ ਇੰਸਟਾਲੇਸ਼ਨ ਕਦਮ ਇਹ ਮੂਲ ਗੱਲਾਂ ਮੰਨ ਕੇ ਚੱਲਦੇ ਹਨ ਕਿ ਇਹ ਪਹਿਲਾਂ ਹੀ ਸੰਭਾਲੀਆਂ ਗਈਆਂ ਹਨ।

- **Azure ਰਿਸੋਰਸ ਬਣਾਉਣ ਦੀ ਅਨੁਮਤੀ**: ਜ਼ਿਆਦਾਤਰ ਐਕਸਰਸਾਇਜ਼ ਲਈ, ਤੁਹਾਡੇ ਕੋਲ ਟарਗੇਟ ਸਬਸਕ੍ਰਿਪਸ਼ਨ ਜਾਂ ਰਿਸੋਰਸ ਗਰੂਪ 'ਤੇ ਘੱਟੋ-ਘੱਟ **Contributor** ਐਕਸੇਸ ਹੋਣਾ ਚਾਹੀਦਾ ਹੈ। ਕੁਝ ਅਧਿਆਇ ਇਹ ਵੀ ਮੰਨ ਸਕਦੇ ਹਨ ਕਿ ਤੁਸੀਂ ਰਿਸੋਰਸ ਗਰੂਪ, managed identities, ਅਤੇ RBAC ਅਸਾਈਨਮੈਂਟ ਬਣਾਉ ਸਕਦੇ ਹੋ।
- [**GitHub ਖਾਤਾ**](https://github.com): ਰਿਪੋਜ਼ਟਰੀ ਫੋਰਕ ਕਰਨ, ਆਪਣੇ ਬਦਲਾਅ ਟਰੈਕ ਕਰਨ, ਅਤੇ ਵਰਕਸ਼ਾਪ ਲਈ GitHub Codespaces ਵਰਤਣ ਵਿੱਚ ਇਹ ਲਾਭਦਾਇਕ ਹੈ।

- **ਬੁਨਿਆਦੀ ਟਰਮੀਨਲ ਜਾਣੂਗਰੀ**: ਤੁਹਾਨੂੰ ਮਹਿਰਤ ਦੀ ਲੋੜ ਨਹੀਂ, ਪਰ ਤੁਸੀਂ `git clone`, `azd auth login`, ਅਤੇ `azd up` ਵਰਗੀਆਂ ਕਮਾਂਡਾਂ ਚਲਾਉਣ ਵਿੱਚ ਸਹੂਲਤ ਮਹਿਸੂਸ ਕਰਨੀ ਚਾਹੀਦੀ ਹੈ।

> **ਇਕ ਐਨਟਰਪ੍ਰਾਈਜ਼ ਸਬਸਕ੍ਰਿਪਸ਼ਨ ਵਿੱਚ ਕੰਮ ਕਰ ਰਹੇ ਹੋ?**

- **ਐਂਡ-ਟੂ-ਐਂਡ ਏਆਈ ਵਰਕਫਲੋਜ਼** - ਮਾਡਲ ਵਿਕਾਸ ਤੋਂ ਲੈ ਕੇ ਪ੍ਰੋਡਕਸ਼ਨ ਡਿਪਲੋਇਮੈਂਟ ਤੱਕ ਸਹੀ ਨਿਗਰਾਨੀ ਦੇ ਨਾਲ
- **ਲਾਗਤ ਅਪਟੀਮਾਈਜ਼ੇਸ਼ਨ** - ਏਆਈ ਵਰਕਲੋਡ ਲਈ ਸਮਝਦਾਰ ਸਰੋਤ ਵੰਡ ਅਤੇ ਸਕੇਲਿੰਗ ਰਣਨੀਤੀਆਂ

<details>
<summary><strong>❌ "azd: command not found"</strong></summary>

```bash
# ਸਭ ਤੋਂ ਪਹਿਲਾਂ AZD ਇੰਸਟਾਲ ਕਰੋ
# ਵਿੰਡੋਜ਼ (ਪਾਵਰਸ਼ੈੱਲ):
winget install microsoft.azd

# ਮੈਕਓਐਸ:
brew tap azure/azd && brew install azd

# ਲਿਨਕਸ:
curl -fsSL https://aka.ms/install-azd.sh | bash

# ਇੰਸਟਾਲੇਸ਼ਨ ਦੀ ਜਾਂਚ ਕਰੋ
azd version
```
</details>

<details>
<summary><strong>❌ "No subscription found" ਜਾਂ "Subscription not set"</strong></summary>

```bash
# ਉਪਲਬਧ ਸਬਸਕ੍ਰਿਪਸ਼ਨਾਂ ਦੀ ਸੂਚੀ
az account list --output table

# ਡਿਫਾਲਟ ਸਬਸਕ੍ਰਿਪਸ਼ਨ ਸੈੱਟ ਕਰੋ
az account set --subscription "<subscription-id-or-name>"

# AZD ਵਾਤਾਵਰਣ ਲਈ ਸੈੱਟ ਕਰੋ
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# ਤਸਦੀਕ ਕਰੋ
az account show
```
</details>

<details>
<summary><strong>❌ "InsufficientQuota" ਜਾਂ "Quota exceeded"</strong></summary>

```bash
# ਹੋਰ Azure ਰੀਜਨ ਆਜ਼ਮਾਓ
azd env set AZURE_LOCATION "westus2"
azd up

# ਜਾਂ ਡਿਵੈਲਪਮੈਂਟ ਵਿੱਚ ਛੋਟੇ SKUs ਦੀ ਵਰਤੋਂ ਕਰੋ
# infra/main.parameters.json ਨੂੰ ਸੋਧੋ:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>

<details>
<summary><strong>❌ "azd up" ਅੱਧੇ ਰਾਹ 'ਤੇ ਨਾਕਾਮ ਹੋ ਜਾਂਦਾ ਹੈ</strong></summary>

```bash

azd up

# ਵਿਕਲਪ 2: ਸਿਰਫ਼ ਬੁਨਿਆਦੀ ਢਾਂਚਾ ਠੀਕ ਕਰੋ

azd show

# ਵਿਕਲਪ 4: Azure Monitor ਵਿੱਚ ਲੌਗਾਂ ਦੀ ਜਾਂਚ ਕਰੋ
azd monitor --logs
```
</details>

<details>
<summary><strong>❌ "Authentication failed" ਜਾਂ "Token expired"</strong></summary>

azd down --force --purge
