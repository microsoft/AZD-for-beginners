## 1. ਪ੍ਰਸਤਾਵਨਾ

The [Azure ਡਿਵੈਲਪਰ CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) or `azd` ਇੱਕ ਓਪਨ-ਸੋਰਸ ਕਮਾਂਡ ਲਾਈਨ ਟੂਲ ਹੈ ਜੋ Azure ਤੇ ਐਪਲੀਕੇਸ਼ਨ ਬਣਾਉਣ ਅਤੇ ਡਿਪਲੋਇ ਕਰਨ ਵੇਲੇ ਡਿਵੈਲਪਰ ਵਾਲੇ ਵਰਕਫਲੋ ਨੂੰ ਸੁਗਮ ਬਨਾਉਂਦਾ ਹੈ। 

[AZD ਟੈਂਪਲੇਟਸ](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) ਮਿਆਰੀਕ੍ਰਿਤ ਰਿਪੋਜ਼ਟਰੀਆਂ ਹਨ ਜੋ ਸੈਂਪਲ ਐਪਲੀਕੇਸ਼ਨ ਕੋਡ, _ਇੰਫ੍ਰਾਸਟਰਕਚਰ-ਐਜ਼-ਕੋਡ_ ਐਸੈਟਸ, ਅਤੇ `azd` ਕੰਫਿਗਰੇਸ਼ਨ ਫਾਈਲਾਂ ਨੂੰ ਇੱਕਠੇ ਇੱਕ ਸੰਯੁਕਤ ਹੱਲ ਆਰਕੀਟੈਕਚਰ ਲਈ ਸ਼ਾਮਲ ਕਰਦੀਆਂ ਹਨ। ਇੰਫ੍ਰਾਸਟਰਕਚਰ ਪ੍ਰੋਵਿਜ਼ਨ ਕਰਨਾ ਇਕ `azd provision` ਕਮਾਂਡ ਵਰਗਾ ਸੌਖਾ ਹੋ ਜਾਂਦਾ ਹੈ - ਜਦਕਿ `azd up` ਦੀ ਵਰਤੋਂ ਤੁਹਾਨੂੰ ਇਕੋ ਵਾਰ ਇੰਫ੍ਰਾਸਟਰਕਚਰ ਪ੍ਰੋਵਿਜ਼ਨ ਕਰਨ ਅਤੇ ਆਪਣੀ ਐਪਲੀਕੇਸ਼ਨ ਡਿਪਲੋਇ ਕਰਨ ਦੀ ਆਗਿਆ ਦਿੰਦੀ ਹੈ!

ਇਸ ਦੇ ਨਤੀਜੇ ਵਜੋਂ, ਆਪਣੀ ਐਪਲੀਕੇਸ਼ਨ ਡਿਵੈਲਪਮੈਂਟ ਪ੍ਰਕਿਰਿਆ ਨੂੰ ਸ਼ੁਰੂ ਕਰਨ ਲਈ ਸਹੀ _AZD Starter template_ ਲੱਭਣਾ ਜੋ ਤੁਹਾਡੇ ਐਪਲੀਕੇਸ਼ਨ ਅਤੇ ਇੰਫ੍ਰਾਸਟਰਕਚਰ ਦੀਆਂ ਜ਼ਰੂਰਤਾਂ ਦੇ ਨੇੜੇ ਹੁੰਦਾ ਹੈ — ਫਿਰ ਰਿਪੋਜ਼ਟਰੀ ਨੂੰ ਆਪਣੀ ਸਥਿਤੀ ਮੁਤਾਬਕ ਅਨੁਕੂਲਿਤ ਕਰਨਾ ਬਹੁਤ ਸੌਖਾ ਹੈ।

ਸ਼ੁਰੂ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ, ਆਓ ਯਕੀਨੀ ਬਣਾਈਏ ਕਿ ਤੁਹਾਡੇ ਕੋਲ Azure ਡਿਵੈਲਪਰ CLI ਇੰਸਟਾਲ ਹੈ।

1. ਇੱਕ VS Code ਟਰਮੀਨਲ ਖੋਲ੍ਹੋ ਅਤੇ ਇਹ ਕਮਾਂਡ ਟਾਈਪ ਕਰੋ:

      ```bash title="" linenums="0"
      azd version
      ```

1. ਤੁਹਾਨੂੰ ਇਸ ਤਰ੍ਹਾਂ ਕੁਝ ਦੇਖਣਾ ਚਾਹੀਦਾ ਹੈ!

      ```bash title="" linenums="0"
      azd version 1.19.0 (commit b3d68cea969b2bfbaa7b7fa289424428edb93e97)
      ```

**ਤੁਸੀਂ ਹੁਣ azd ਨਾਲ ਇੱਕ ਟੈਂਪਲੇਟ ਚੁਣ ਕੇ ਡਿਪਲੋਇ ਕਰਨ ਲਈ ਤਿਆਰ ਹੋ**

## 2. ਟੈਂਪਲੇਟ ਚੋਣ

Microsoft Foundry ਪਲੇਟਫਾਰਮ ਨਾਲ [ਸਿਫ਼ਾਰਸ਼ੀ AZD ਟੈਂਪਲੇਟਸ](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) ਦਾ ਇੱਕ ਸੈੱਟ ਆਉਂਦਾ ਹੈ ਜੋ ਲੋਕਪ੍ਰਿਯ ਹੱਲ ਪਰਿਦ੍ਰਿਸ਼ਿਆਂ ਨੂੰ ਕਵਰ ਕਰਦਾ ਹੈ ਜਿਵੇਂ ਕਿ _multi-agent workflow atomation_ ਅਤੇ _multi-modal content processing_. ਤੁਸੀਂ ਇਹਨਾਂ ਟੈਂਪਲੇਟਸ ਨੂੰ Microsoft Foundry ਪੋਰਟਲ 'ਤੇ ਜਾ ਕੇ ਵੀ ਖੋਜ ਸਕਦੇ ਹੋ।

1. [https://ai.azure.com/templates](https://ai.azure.com/templates) 'ਤੇ ਜਾਓ
1. ਜਦੋਂ ਪ੍ਰਾਂਪਟ ਆਵੇ ਤਾਂ Microsoft Foundry ਪੋਰਟਲ ਵਿੱਚ ਲੌਗਇਨ ਕਰੋ - ਤੁਹਾਨੂੰ ਇਸ ਤਰ੍ਹਾਂ ਕੁਝ ਵੇਖਣ ਨੂੰ ਮਿਲੇਗਾ।

![ਚੁਣੋ](../../../../../translated_images/pa/01-pick-template.60d2d5fff5ebc374.webp)


The **Basic** ਵਿਕਲਪ ਤੁਹਾਡੇ ਸ਼ੁਰੂਆਤੀ ਟੈਂਪਲੇਟ ਹਨ:

1. [ ] [AI Chat ਨਾਲ ਸ਼ੁਰੂ ਕਰੋ](https://github.com/Azure-Samples/get-started-with-ai-chat) ਜੋ ਕਿ ਇੱਕ ਮੁੱਖ ਚੈਟ ਐਪਲੀਕੇਸ਼ਨ _ਤੁਾਡੇ ਡੇਟਾ ਨਾਲ_ Azure Container Apps 'ਤੇ ਡਿਪਲੋਇ ਕਰਦਾ ਹੈ। ਇਸ ਨੂੰ ਬੁਨਿਆਦੀ AI ਚੈਟਬੌਟ ਪਰਿਦ੍ਰਿਸ਼ ਦੇ ਖੋਜ ਲਈ ਵਰਤੋਂ ਕਰੋ।
1. [X] [AI Agents ਨਾਲ ਸ਼ੁਰੂ ਕਰੋ](https://github.com/Azure-Samples/get-started-with-ai-agents) ਜੋ ਕਿ ਇੱਕ ਸਟੈਂਡਰਡ AI Agent (Foundry Agents ਨਾਲ) ਵੀ ਡਿਪਲੋਇ ਕਰਦਾ ਹੈ। ਟੂਲਾਂ ਅਤੇ ਮਾਡਲਾਂ ਸ਼ਾਮਲ ਕਰਨ ਵਾਲੇ ਏਜੰਟ-ਅਧਾਰਿਤ AI ਹੱਲਾਂ ਨਾਲ ਜਾਣੂ ਹੋਣ ਲਈ ਇਸ ਦਾ ਉਪਯੋਗ ਕਰੋ।

ਦੂਜੇ ਲਿੰਕ ਨੂੰ ਨਵੇਂ ਬ੍ਰਾਊਜ਼ਰ ਟੈਬ ਵਿੱਚ ਖੋਲ੍ਹੋ (ਜਾਂ ਸੰਬੰਧਤ ਕਾਰਡ ਲਈ `Open in GitHub` 'ਤੇ ਕਲਿਕ ਕਰੋ)। ਤੁਹਾਨੂੰ ਇਸ AZD ਟੈਂਪਲੇਟ ਲਈ ਰਿਪੋਜ਼ਟਰੀ ਵੇਖਣ ਨੂੰ ਮਿਲੇਗੀ। README ਦੀ ਜਾਂਚ ਕਰਨ ਲਈ ਇੱਕ ਮਿੰਟ ਲਓ। ਐਪਲੀਕੇਸ਼ਨ ਆਰਕੀਟੈਕਚਰ ਇਹੋ ਜਿਹਾ ਹੈ:

![ਆਰਕੀਟੈਕਚਰ](../../../../../translated_images/pa/architecture.8cec470ec15c65c7.webp)

## 3. ਟੈਂਪਲੇਟ ਐਕਟੀਵੇਸ਼ਨ

ਆਓ ਇਸ ਟੈਂਪਲੇਟ ਨੂੰ ਡਿਪਲੋਇ ਕਰਨ ਦੀ ਕੋਸ਼ਿਸ਼ ਕਰੀਏ ਅਤੇ ਯਕੀਨੀ ਬਣਾਈਏ ਕਿ ਇਹ ਵੈਧ ਹੈ। ਅਸੀਂ [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) ਸੈਕਸ਼ਨ ਵਿੱਚ ਦਿੱਤੀਆਂ ਹਦਾਇਤਾਂ ਦੀ ਪਾਲਣਾ ਕਰਾਂਗੇ।

1. ਇਸ ਲਿੰਕ 'ਤੇ ਕਲਿਕ ਕਰੋ: [https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) - `Create codespace` ਦੀ ਡਿਫਾਲਟ ਐਕਸ਼ਨ ਦੀ ਪੁਸ਼ਟੀ ਕਰੋ
1. ਇਹ ਨਵੇਂ ਬ੍ਰਾਊਜ਼ਰ ਟੈਬ ਨੂੰ ਖੋਲ੍ਹਦਾ ਹੈ - GitHub Codespaces ਸੈਸ਼ਨ ਦੇ ਲੋਡ ਹੋਣ ਦੀ ਉਡੀਕ ਕਰੋ
1. Codespaces ਵਿੱਚ VS Code ਟਰਮੀਨਲ ਖੋਲ੍ਹੋ - ਹੇਠ ਲਿਖੀ ਕਮਾਂਡ ਟਾਈਪ ਕਰੋ:

   ```bash title="" linenums="0"
   azd up
   ```

Complete the workflow steps that this will trigger:

1. ਤੁਹਾਨੂੰ Azure ਵਿੱਚ ਲੌਗਿਨ ਕਰਨ ਲਈ ਪੁੱਛਿਆ ਜਾਵੇਗਾ - ਪ੍ਰਮਾਣਿਕਤਾ ਲਈ ਨਿਰਦੇਸ਼ਾਂ ਦੀ ਪਾਲਣਾ ਕਰੋ
1. ਆਪਣੇ ਲਈ ਇੱਕ ਵਿਲੱਖਣ Environment ਨਾਮ ਦਿਓ - ਉਦਾਹਰਨ ਵਜੋਂ, ਮੈਂ `nitya-mshack-azd` ਵਰਤਿਆ
1. ਇਹ `.azure/` ਫੋਲਡਰ ਬਣਾਏਗਾ - ਤੁਸੀਂ env ਨਾਮ ਵਾਲਾ ਇੱਕ ਸਬਫੋਲਡਰ ਦੇਖੋਗੇ
1. ਤੁਹਾਨੂੰ ਇੱਕ subscription ਨਾਮ ਚੁਣਨ ਲਈ ਪੁੱਛਿਆ ਜਾਵੇਗਾ - ਡਿਫਾਲਟ ਚੁਣੋ
1. ਤੁਹਾਨੂੰ ਇੱਕ location ਲਈ ਪੁੱਛਿਆ ਜਾਵੇਗਾ - `East US 2` ਵਰਤੋ

ਹੁਣ, ਤੁਸੀਂ ਪ੍ਰੋਵਿਜ਼ਨਿੰਗ ਦੇ ਪੂਰਾ ਹੋਣ ਦੀ ਉਡੀਕ ਕਰੋ। **ਇਸ ਵਿੱਚ 10-15 ਮਿੰਟ ਲੱਗਦੇ ਹਨ**

1. ਪੂਰਾ ਹੋਣ 'ਤੇ, ਤੁਹਾਡੇ ਕਨਸੋਲ ਵਿੱਚ SUCCESS ਸੁਨੇਹਾ ਇਸ ਤਰ੍ਹਾਂ ਦਿਖਾਈ ਦੇਵੇਗਾ:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. ਤੁਹਾਡੇ Azure Portal ਵਿੱਚ ਹੁਣ ਉਸ env ਨਾਮ ਨਾਲ ਇੱਕ ਪ੍ਰੋਵਿਜ਼ਨਡ ਰਿਸੋਰਸ ਗਰੁੱਪ ਹੋਵੇਗਾ:

      ![ਇੰਫ੍ਰਾ](../../../../../translated_images/pa/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **ਤੁਸੀਂ ਹੁਣ ਡਿਪਲੋਇ ਕੀਤੀ ਗਈ ਇੰਫ੍ਰਾਸਟਰਕਚਰ ਅਤੇ ਐਪਲੀਕੇਸ਼ਨ ਦੀ ਜਾਂਚ ਕਰਨ ਲਈ ਤਿਆਰ ਹੋ।**

## 4. ਟੈਂਪਲੇਟ ਵੈਰੀਫਿਕੇਸ਼ਨ

1. Azure Portal ਦੇ [ਰਿਸੋਰਸ ਗਰੁੱਪ](https://portal.azure.com/#browse/resourcegroups) ਪੰਨੇ 'ਤੇ ਜਾਓ - ਜਦੋਂ ਪ੍ਰਾਂਪਟ ਹੋਵੇ ਤਾਂ ਲੌਗਿਨ ਕਰੋ
1. ਆਪਣੇ Environment ਨਾਮ ਲਈ RG 'ਤੇ ਕਲਿਕ ਕਰੋ - ਤੁਸੀਂ ਉੱਪਰ ਦਿੱਤਾ ਪੰਨਾ ਵੇਖੋਗੇ

      - Azure Container Apps ਰਿਸੋਰਸ 'ਤੇ ਕਲਿਕ ਕਰੋ
      - _Essentials_ ਸੈਕਸ਼ਨ (ਉੱਪਰ ਸੱਜੇ) ਵਿੱਚ Application Url 'ਤੇ ਕਲਿਕ ਕਰੋ

1. ਤੁਹਾਨੂੰ ਇਸ ਤਰ੍ਹਾਂ ਇੱਕ ਹੋਸਟ ਕੀਤੀ ਗਈ ਐਪਲੀਕੇਸ਼ਨ ਫਰੰਟ-ਐਂਡ UI ਦੇਖਣ ਨੂੰ ਮਿਲੇਗੀ:

   ![ਐਪ](../../../../../translated_images/pa/03-test-application.471910da12c3038e.webp)

1. ਕੁਝ [ਨਮੂਨਾ ਪ੍ਰਸ਼ਨਾਂ](https://github.com/Azure-Samples/get-started-with-ai-agents/blob/main/docs/sample_questions.md) ਪੁੱਛਣ ਦੀ ਕੋਸ਼ਿਸ਼ ਕਰੋ

      1. ਪ੍ਰਸ਼ਨ ਕਰੋ: ```ਫ਼ਰਾਂਸ ਦੀ ਰਾਜਧਾਨੀ ਕਿਹੜੀ ਹੈ?``` 
      1. ਪ੍ਰਸ਼ਨ ਕਰੋ: ```ਦੋ ਲੋਕਾਂ ਲਈ $200 ਤੋਂ ਘੱਟ ਸਭ ਤੋਂ ਵਧੀਆ ਟੈਂਟ ਕਿਹੜਾ ਹੈ, ਅਤੇ ਇਸ ਵਿੱਚ ਕਿਹੜੀਆਂ ਖਾਸੀਅਤਾਂ ਹਨ?```

1. ਤੁਹਾਨੂੰ ਹੇਠਾਂ ਦਿਖਾਈ ਗਈਆਂ ਜਿਹੀਆਂ ਜਵਾਬਾਂ ਮਿਲਣੀਆਂ ਚਾਹੀਦੀਆਂ ਹਨ। _ਪਰ ਇਹ ਕਿਵੇਂ ਕੰਮ ਕਰਦਾ ਹੈ?_ 

      ![ਐਪ](../../../../../translated_images/pa/03-test-question.521c1e863cbaddb6.webp)

## 5.  ਏਜੰਟ ਵੈਰੀਫਿਕੇਸ਼ਨ

Azure Container App ਇੱਕ ਐਸਾ ਏਂਡਪੁਇੰਟ ਡਿਪਲੋਇ ਕਰਦਾ ਹੈ ਜੋ ਇਸ ਟੈਂਪਲੇਟ ਲਈ Microsoft Foundry ਪ੍ਰੋਜੈਕਟ ਵਿੱਚ ਪ੍ਰੋਵਿਜ਼ਨਡ AI Agent ਨਾਲ ਜੁੜਦਾ ਹੈ। ਆਓ ਵੇਖੀਏ ਕਿ ਇਸ ਦਾ ਕੀ ਮਤਲਬ ਹੈ।

1. ਆਪਣੇ ਰਿਸੋਰਸ ਗਰੁੱਪ ਦੇ Azure Portal _Overview_ ਪੰਨੇ 'ਤੇ ਵਾਪਸ ਜਾਓ

1. ਉਹਨਾਂ ਸੂਚੀ ਵਿੱਚ `Microsoft Foundry` ਰਿਸੋਰਸ 'ਤੇ ਕਲਿਕ ਕਰੋ

1. ਤੁਹਾਨੂੰ ਇਹ ਵੇਖਣਾ ਚਾਹੀਦਾ ਹੈ। `Go to Microsoft Foundry Portal` ਬਟਨ 'ਤੇ ਕਲਿਕ ਕਰੋ। 
   ![Foundry](../../../../../translated_images/pa/04-view-foundry-project.fb94ca41803f28f3.webp)

1. ਤੁਹਾਨੂੰ ਆਪਣੇ AI ਐਪਲੀਕੇਸ਼ਨ ਲਈ Foundry ਪ੍ਰੋਜੈਕਟ ਪੰਨਾ ਵੇਖਣ ਨੂੰ ਮਿਲੇਗਾ
   ![Project](../../../../../translated_images/pa/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` 'ਤੇ ਕਲਿਕ ਕਰੋ - ਤੁਸੀਂ ਆਪਣੇ ਪ੍ਰੋਜੈਕਟ ਵਿੱਚ ਡਿਫਾਲਟ Agent ਦੇ ਪ੍ਰੋਵਿਜ਼ਨ ਨੂੰ ਵੇਖਦੇ ਹੋ
   ![Agents](../../../../../translated_images/pa/06-visit-agents.bccb263f77b00a09.webp)

1. ਇਸ ਨੂੰ ਚੁਣੋ - ਅਤੇ ਤੁਸੀਂ ਏਜੰਟ ਵੇਰਵੇ ਵੇਖੋਗੇ। ਹੇਠ ਲਿਖੀਆਂ ਗੱਲਾਂ ਨੂੰ ਨੋਟ ਕਰੋ:

      - ਏਜੰਟ ਮੂਲ ਰੂਪ ਵਿੱਚ File Search ਦੀ ਵਰਤੋਂ ਕਰਦਾ ਹੈ (ਹਮੇਸ਼ਾਂ)
      - ਏਜੰਟ ਦੀ `Knowledge` ਦਰਸਾਉਂਦੀ ਹੈ ਕਿ ਇਸ ਵਿੱਚ 32 ਫਾਈਲਾਂ ਅੱਪਲੋਡ ਕੀਤੀਆਂ ਗਈਆਂ ਹਨ (ਫਾਇਲ ਖੋਜ ਲਈ)
      ![Agents](../../../../../translated_images/pa/07-view-agent-details.0e049f37f61eae62.webp)

1. ਵੱਛੇ ਮੀਨੂ ਵਿੱਚ `Data+indexes` ਵਿਵਿੱਕਲਪ ਦੀ ਖੋਜ ਕਰੋ ਅਤੇ ਵੇਰਵੇ ਲਈ ਕਲਿਕ ਕਰੋ।

## 8. Environment Variables

ਹੁਣ ਤੱਕ, ਅਸੀਂ ਬ੍ਰਾਊਜ਼ਰ ਵਿੱਚ ਡਿਪਲੋਇਮੈਂਟ ਦੇ ਰਾਹੀਂ ਚੱਲਕੇ ਵੇਖਿਆ ਹੈ - ਅਤੇ ਯਕੀਨੀ ਬਣਾਇਆ ਹੈ ਕਿ ਸਾਡੀ ਇੰਫ੍ਰਾਸਟਰਕਚਰ ਪ੍ਰੋਵਿਜ਼ਨਡ ਹੈ ਅਤੇ ਐਪਲੀਕੇਸ਼ਨ ਕਾਰਜਕਸ਼ਮ ਹੈ। ਪਰ ਕੋਡ-ਪਹਿਲਾਂ ਢੰਗ ਨਾਲ ਐਪਲੀਕੇਸ਼ਨ 'ਤੇ ਕੰਮ ਕਰਨ ਲਈ, ਸਾਨੂੰ ਆਪਣੇ ਲੋਕਲ ਡਿਵੈਲਪਮੈਂਟ ਵਾਤਾਵਰਣ ਨੂੰ ਉਹਨਾਂ ਮਾਹਰ ਜਾਣਚਾਂ ਨਾਲ ਸੰਰਚਿਤ ਕਰਨਾ ਪੈਂਦਾ ਹੈ ਜੋ ਇਨ੍ਹਾਂ ਰਿਸੋਰਸਾਂ ਨਾਲ ਕੰਮ ਕਰਨ ਲਈ ਲੋੜੀਂਦੇ ਹਨ। `azd` ਦੀ ਵਰਤੋਂ ਇਹ ਸੌਖਾ ਕਰ ਦਿੰਦੀ ਹੈ।

1. Azure Developer CLI [environment variables ਦੀ ਵਰਤੋਂ ਕਰਦਾ ਹੈ](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) ਤਾਂ ਜੋ ਐਪਲੀਕੇਸ਼ਨ ਡਿਪਲੋਇਮੈਂਟ ਲਈ ਕੰਫਿਗਰੇਸ਼ਨ ਸੈਟਿੰਗਜ਼ ਸਟੋਰ ਅਤੇ ਪ੍ਰਬੰਧ ਕੀਤੀਆਂ ਜਾ ਸਕਣ।

1. Environment variables `.azure/<env-name>/.env` ਵਿੱਚ ਸਟੋਰ ਕੀਤੀਆਂ ਜਾਂਦੀਆਂ ਹਨ - ਇਹਨਾਂ ਨੂੰ ਉਸ `env-name` ਵਾਤਾਵਰਣ ਤੱਕ ਸੀਮਿਤ ਕਰਦਾ ਹੈ ਜੋ ਡਿਪਲੋਇਮੈਂਟ ਦੌਰਾਨ ਵਰਤਿਆ ਗਿਆ ਸੀ ਅਤੇ ਤੁਹਾਨੂੰ ਇੱਕੋ ਰਿਪੋ ਵਿੱਚ ਵੱਖ-ਵੱਖ ਡਿਪਲੋਇਮੈਂਟ ਟਾਰਗਟਾਂ ਵਿਚਕਾਰ ਵਾਤਾਵਰਣਾਂ ਨੂੰ ਅਲੱਗ ਕਰਨ ਵਿੱਚ ਸਹਾਇਤਾ ਕਰਦਾ ਹੈ।

1. Environment variables ਨੂੰ `azd` ਕਮਾਂਡ ਦੁਆਰਾ ਆਟੋਮੈਟਿਕ ਤੌਰ 'ਤੇ ਲੋਡ ਕੀਤਾ ਜਾਂਦਾ ਹੈ ਜਦੋਂ ਵੀ ਇਹ ਕਿਸੇ ਖਾਸ ਕਮਾਂਡ ਨੂੰ ਚਲਾਉਂਦਾ ਹੈ (ਉਦਾਹਰਨ ਵਜੋਂ, `azd up`). ਧਿਆਨ ਰੱਖੋ ਕਿ `azd` ਆਪਣੇ-ਆਪ ਨੂੰ ਓਐਸ-ਸਤਰ ਦੇ environment variables (ਉਦਾਹਰਨ ਲਈ ਸ਼ੈੱਲ ਵਿੱਚ ਸੈਟ ਕੀਤੀਆਂ ਗਈਆਂ) ਨੂੰ ਆਟੋਮੈਟਿਕ ਤੌਰ 'ਤੇ ਨਹੀਂ ਪੜ੍ਹਦਾ - ਇਸਦੇ ਬਦਲੇ `azd set env` ਅਤੇ `azd get env` ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਸਕ੍ਰਿਪਟਾਂ ਵਿੱਚ ਜਾਣਕਾਰੀ ਟ੍ਰਾਂਸਫਰ ਕਰੋ।

ਆਓ ਕੁਝ ਕਮਾਂਡਜ਼ ਅਜ਼ਮਾਈਏ:

1. ਇਸ ਵਾਤਾਵਰਣ ਵਿੱਚ `azd` ਲਈ ਸੈਟ ਕੀਤੀਆਂ ਸਾਰੀਆਂ environment variables ਪ੍ਰਾਪਤ ਕਰੋ:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      ਤੁਹਾਨੂੰ ਇਸ ਤਰ੍ਹਾਂ ਕੁਝ ਵੇਖਣ ਨੂੰ ਮਿਲੇਗਾ:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4o-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. ਇੱਕ ਖਾਸ ਕੀਮਤ ਪ੍ਰਾਪਤ ਕਰੋ - ਉਦਾਹਰਨ ਵਜੋਂ, ਮੈਂ ਦੇਖਣਾ ਚਾਹੁੰਦਾ ਹਾਂ ਕਿ ਕੀ ਅਸੀਂ `AZURE_AI_AGENT_MODEL_NAME` ਮੁੱਲ ਸੈਟ ਕੀਤਾ ਸੀ

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      ਤੁਹਾਨੂੰ ਇਸ ਤਰ੍ਹਾਂ ਕੁਝ ਵੇਖਣ ਨੂੰ ਮਿਲਦਾ ਹੈ - ਇਹ ਡਿਫਾਲਟ ਤੌਰ 'ਤੇ ਸੈਟ ਨਹੀਂ ਸੀ!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd` ਲਈ ਇੱਕ ਨਵਾਂ environment variable ਸੈਟ ਕਰੋ। ਇੱਥੇ, ਅਸੀਂ ਏਜੰਟ ਮਾਡਲ ਨਾਮ ਨੂੰ ਅੱਪਡੇਟ ਕਰਦੇ ਹਾਂ। _ਨੋਟ: ਕੀਤੇ ਗਏ ਕੋਈ ਵੀ ਬਦਲਾਅ ਤੁਰੰਤ `.azure/<env-name>/.env` ਫਾਈਲ ਵਿੱਚ ਪ੍ਰਤੀਬਿੰਬਤ ਹੋ ਜਾਣ گے।_

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      ਹੁਣ, ਅਸੀਂ ਵੇਖਾਂਗੇ ਕਿ ਕੀਮਤ ਸੈਟ ਹੋ ਚੁੱਕੀ ਹੈ:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. ਧਿਆਨ ਦਿਓ ਕਿ ਕੁਝ ਰਿਸੋਰਸਸ ਸਥਾਇਤਵਸ਼ੀਲ ਹੁੰਦੀਆਂ ਹਨ (ਉਦਾਹਰਨ ਵਜੋਂ, ਮਾਡਲ ਡਿਪਲੋਇਮੈਂਟਜ਼) ਅਤੇ ਉਹਨਾਂ ਨੂੰ ਫੋਰਸ ਰੀਡਿਪਲੋਇਮੈਂਟ ਲਈ ਸਿਰਫ਼ `azd up` ਤੋਂ ਵੱਧ ਦੀ ਜ਼ਰੂਰਤ ਹੋ ਸਕਦੀ ਹੈ। ਆਓ ਅਸਲ ਡਿਪਲੋਇਮੈਂਟ ਨੂੰ ਤੋੜ ਕੇ ਨਵੇਂ env vars ਨਾਲ ਦੁਬਾਰਾ ਡਿਪਲੋਇ ਕਰਨ ਦੀ ਕੋਸ਼ਿਸ਼ ਕਰੀਏ।

1. **Refresh** ਜੇ ਤੁਸੀਂ ਪਿਛਲੇ ਸਮੇਂ ਵਿੱਚ azd ਟੈਂਪਲੇਟ ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਇੰਫ੍ਰਾਸਟਰਕਚਰ ਡਿਪਲੋਇ ਕੀਤਾ ਸੀ - ਤਾਂ ਤੁਸੀਂ ਆਪਣੇ ਲੋਕਲ environment variables ਦੀ ਸਥਿਤੀ ਨੂੰ ਆਪਣੇ Azure ਡਿਪਲੋਇਮੈਂਟ ਦੀ ਮੌਜੂਦਾ ਸਥਿਤੀ ਅਨੁਸਾਰ _refresh_ ਕਰ ਸਕਦੇ ਹੋ ਇਹ ਕਮਾਂਡ ਵਰਤ ਕੇ:

      ```bash title="" linenums="0"
      azd env refresh
      ```

      ਇਹ ਦੋ ਜਾਂ ਉਸ ਤੋਂ ਵੱਧ ਸਥਾਨਕ ਡਿਵੈਲਪਮੈਂਟ ਵਾਤਾਵਰਣਾਂ (ਉਦਾਹਰਨ ਵਜੋਂ, ਕਈ ਡਿਵੈਲਪਰਾਂ ਵਾਲੀ ਟੀਮ) ਵਿੱਚ ਇਨਵਾਇਰਨਮੈਂਟ ਵੈਰੀਏਬਲਾਂ ਨੂੰ _ਸਮਕਾਲੀਕ_ ਕਰਨ ਦਾ ਇੱਕ ਸ਼ਕਤੀਸ਼ਾਲੀ ਤਰੀਕਾ ਹੈ - ਜਿਸ ਨਾਲ ਡਿਪਲੋਈਡ ਇਨਫਰਾਸਟ੍ਰੱਕਚਰ ਇਨਵਾਇਰਨਮੈਂਟ ਵੈਰੀਏਬਲਾਂ ਦੀ ਸਥਿਤੀ ਲਈ ਮੂਲ ਸੱਚਾਈ ਵਜੋਂ ਕੰਮ ਕਰਦੀ ਹੈ। ਟੀਮ ਦੇ ਮੈਂਬਰ ਸਿਰਫ਼ ਵੇਰੀਏਬਲਾਂ ਨੂੰ _ਰੀਫਰੈਸ਼_ ਕਰਦੇ ਹਨ ਤਾਂ ਜੋ ਮੁੜ ਸਿੰਕ ਹੋ ਸਕਣ।
