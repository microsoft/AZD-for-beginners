**ਅਧਿਆਇ ਨੈਵੀਗੇਸ਼ਨ:**
- **📚 ਕੋਰਸ ਹੋਮ**: [AZD For Beginners](../../README.md)
- **📖 ਮੌਜੂਦਾ ਅਧਿਆਇ**: ਅਧਿਆਇ 1 - ਫਾਊਂਡੇਸ਼ਨ ਅਤੇ ਕਵਿਕ ਸਟਾਰਟ
- **⬅️ ਪਹਿਲਾ**: [Your First Project](first-project.md)
- **➡️ ਅਗਲਾ**: [Dev Containers & Codespaces](dev-containers.md)

> `azd 1.25.6` ਦੇ ਖ਼ਿਲਾਫ ਜੂਨ 2026 ਵਿੱਚ ਮਨਜ਼ੂਰ ਕੀਤਾ ਗਿਆ।

## ਪਰਚੇ

[Your First Project](first-project.md) ਵਿੱਚ ਤੁਸੀਂ ਟੈਂਪਲੇਟੋਂ ਸ਼ੁਰੂ ਕਰਕੇ ਇੱਕ ਐਪ ਡਿਪਲੌਇ ਕੀਤਾ ਸੀ। ਪਰ ਜ਼ਿਆਦਾਤਰ ਵਾਰ ਤੁਹਾਡੇ ਕੋਲ ਪਹਿਲਾਂ ਹੀ ਇੱਕ ਐਪ ਹੁੰਦੀ ਹੈ—ਇੱਕ Node.js API, ਇੱਕ Python Flask ਸੇਵਾ, ਇੱਕ .NET ਵੈੱਬ ਐਪ—ਜੋ ਤੁਹਾਡੇ ਮਸ਼ੀਨ ਦੇ ਫੋਲਡਰ ਵਿੱਚ ਬੈਠੀ ਹੋਈ ਹੈ। ਇਹ ਪਾਠ ਦਿਖਾਉਂਦਾ ਹੈ ਕਿ ਮੌਜੂਦਾ ਕੋਡ ਵਿੱਚ azd ਕਿਵੇਂ ਜੋੜਨਾ ਹੈ ਤਾਂ ਕਿ ਤੁਸੀਂ `azd up` ਨਾਲ ਇਸਨੂੰ ਡਿਪਲੌਇ ਕਰ ਸਕੋ, ਕਿਸੇ ਟੈਂਪਲੇਟ ਦੀ ਲੋੜ ਨਹੀਂ।

## ਸਿੱਖਣ ਦੇ ਲਕੜ੍ਹੇ

ਇਸ ਪਾਠ ਦੇ ਅੰਤ ਤੱਕ, ਤੁਸੀਂ:
- azd ਪ੍ਰਾਜੈਕਟ ਸ਼ੁਰੂ ਕਰਨ ਦੇ ਤਿੰਨ ਤਰੀਕੇ ਸਮਝ ਲਵੋਗੇ
- ਮੌਜੂਦਾ ਕੋਡਬੇਸ ਵਿੱਚ `azd init` ਚਲਾਉਣਾ ਸਿੱਖ ਲਵੋਗੇ
- ਸਮਝ ਪਾਓਗੇ ਕਿ `azure.yaml` ਅਤੇ `infra/` ਫੋਲਡਰ ਤੁਹਾਡੇ ਐਪ ਲਈ ਕੀ ਕਰਦੇ ਹਨ
- ਜਾਣੋਗੇ ਕਿ ਕਦੋਂ azd ਇੰਫ੍ਰਾਸ਼ਟ੍ਰੱਕਚਰ ਜਨਰੇਟ ਕਰਨ ਦਿਓ ਅਤੇ ਕਦੋਂ ਖੁਦ ਲਿਖੋ
- `azd up` ਨਾਲ ਆਪਣੇ ਮੌਜੂਦਾ ਐਪ ਨੂੰ Azure 'ਤੇ ਡਿਪਲੌਇ ਕਰਨ ਦੇ ਯੋਗ ਹੋ ਜਾਓਗੇ

## ਸਿੱਖਣ ਦੇ ਨਤੀਜੇ

ਇਸ ਪਾਠ ਨੂੰ ਮੁਕੰਮਲ ਕਰਨ ਤੋਂ ਬਾਅਦ, ਤੁਸੀਂ ਸਮਰੱਥ ਹੋਵੋਗੇ:
- ਕਿਸੇ ਮੌਜੂਦਾ ਪ੍ਰਾਜੈਕਟ ਵਿੱਚ azd ਨੂੰ ਇਨੀਸ਼ੀਅਲਾਈਜ਼ ਕਰਨਾ
- ਇੱਕ ਬੁਨਿਆਦੀ `azure.yaml` ਫਾਇਲ ਨੂੰ ਪੜ੍ਹਨਾ ਅਤੇ ਸੋਧਣਾ
- `azd infra generate` ਨਾਲ ਸਟਾਰਟਰ ਇੰਫ੍ਰਾਸ਼ਟ੍ਰੱਕਚਰ ਜਨਰੇਟ ਕਰਨਾ
- ਆਪਣੇ ਐਪ ਲਈ ਇੱਕ ਉਚਿਤ Azure ਹੋਸਟ ਚੁਣਨਾ
- ਆਪਣਾ ਐਪ ਡਿਪਲੌਇ ਅਤੇ ਸਾਫ਼ ਕਰਨਾ

## azd ਪ੍ਰਾਜੈਕਟ ਸ਼ੁਰੂ ਕਰਨ ਦੇ ਤਿੰਨ ਤਰੀਕੇ

| ਸ਼ੁਰੂਆਤੀ ਬਿੰਦੂ | ਕਮਾਂਡ | ਕਦੋਂ ਵਰਤਣਾ ਹੈ |
|----------------|---------|-------------|
| **ਟੈਂਪਲੇਟ ਤੋਂ** | `azd init --template <name>` | ਸਿੱਖਣ ਲਈ, ਜਾਂ ਇੱਕ ਪ੍ਰਮਾਣਿਤ ਨਮੂਨੇ ਤੋਂ ਨਵੀਂ ਐਪ ਸ਼ੁਰੂ ਕਰਨ ਲਈ |
| **ਤੁਹਾਡੇ ਮੌਜੂਦਾ ਕੋਡ ਤੋਂ** | `azd init` (ਤੁਹਾਡੇ ਪ੍ਰਾਜੈਕਟ ਫੋਲਡਰ ਵਿੱਚ) | ਤੁਹਾਡੇ ਕੋਲ ਪਹਿਲਾਂ ਹੀ ਇੱਕ ਐਪ ਹੈ ਅਤੇ ਤੁਸੀਂ ਇਸਨੂੰ ਡਿਪਲੌਇ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ |
| **Git ਰਿਪੋ ਤੋਂ** | `azd init --from-code` (ਇੱਕ ਕਲੋਨ ਕੀਤੇ ਹੋਏ ਰਿਪੋ ਵਿੱਚ) | ਮੌਜੂਦਾ ਰਿਪੋਜ਼ਿਟਰੀ ਲਈ azd ਅਪਣਾਉਣਾ |

ਤੁਸੀਂ ਪਹਿਲੀ ਵਿਕਲਪ ਦਾ ਅਭਿਆਸ ਪਹਿਲਾਂ ਹੀ ਕੀਤਾ ਹੈ। ਇਹ ਪਾਠ ਦੂਜੇ ਵਿਕਲਪ ਨੂੰ ਕਵਰ ਕਰਦਾ ਹੈ—ਸਭ ਤੋਂ ਆਮ ਅਸਲੀ ਦੁਨੀਆ ਦਾ ਮਾਮਲਾ।

## ਕਦਮ 1: ਆਪਣੇ ਪ੍ਰਾਜੈਕਟ ਵਿੱਚ `azd init` ਚਲਾਓ

ਆਪਣੇ ਮੌਜੂਦਾ ਪ੍ਰਾਜੈਕਟ ਫੋਲਡਰ ਦੇ ਅੰਦਰ ਟਰਮਿਨਲ ਖੋਲ੍ਹੋ ਅਤੇ ਚਲਾਓ:

```bash
cd my-existing-app
azd init
```

azd ਤੁਹਾਡੀ ਇਨੀਸ਼ੀਅਲਾਈਜ਼ੇਸ਼ਨ ਕਿਸ ਤਰ੍ਹਾਂ करनी ਹੈ, ਬਾਰੇ ਪੁੱਛੇਗਾ। ਚੁਣੋ:

```
? How do you want to initialize your app?
> Use code in the current directory
  Select a template
```

ਚੁਣੋ **"Use code in the current directory."** azd ਫਿਰ ਤੁਹਾਡੇ ਫੋਲਡਰ ਨੂੰ ਸਕੈਨ ਕਰਦਾ ਹੈ, ਤੁਹਾਡੀ ਭਾਸ਼ਾ ਅਤੇ ਫਰੇਮਵਰਕ ਦਾ ਪਤਾ ਲਗਾਉਂਦਾ ਹੈ, ਅਤੇ ਇੱਕ ਹੋਸਟ ਸੁਝਾਅ ਦਿੰਦਾ ਹੈ।

### azd ਕੀ ਪਤਾ ਲਗਾਉਂਦਾ ਹੈ

azd ਐਸੇ ਸੰਕੇਤ ਲੱਭਦਾ ਹੈ ਜਿਵੇਂ `package.json`, `requirements.txt`, `pom.xml`, `*.csproj`, ਜਾਂ `Dockerfile`, ਅਤੇ ਇੱਕ ਮਿਲਦਾ-ਜੁਲਦਾ Azure ਹੋਸਟ ਸੁਝਾਉਂਦਾ ਹੈ:

| ਤੁਹਾਡਾ ਐਪ | ਸੰਭਾਵਿਤ ਪਤਾ ਲੱਗਿਆ ਹੋਇਆ ਹੋਸਟ |
|----------|----------------------|
| Node.js / Python / .NET web app | Azure App Service or Container Apps |
| ਕੰਟੇਨਰਾਈਜ਼ਡ ਐਪ (`Dockerfile`) | Azure Container Apps |
| Function app | Azure Functions |
| Static site (React/Vue build output) | Azure Static Web Apps |

ਪਤਾ ਲੱਗੀਆਂ ਸੇਵਾਵਾਂ ਦੀ ਪੁਸ਼ਟੀ ਕਰੋ, ਅਤੇ azd ਤੁਹਾਨੂੰ ਲੋੜੀਂਦੇ ਫਾਇਲਾਂ ਨੂੰ ਸਕੈਫੋਲਡ ਕਰੇਗਾ।

## ਕਦਮ 2: ਸਮਝੋ ਕਿ azd ਨੇ ਕੀ ਬਣਾਇਆ

init ਦੇ ਬਾਅਦ, ਤੁਹਾਡੇ ਪ੍ਰਾਜੈਕਟ ਵਿੱਚ ਦੋ ਨਵੀਆਂ ਚੀਜ਼ਾਂ ਹੋਣਗੀਆਂ:

```
my-existing-app/
├── azure.yaml          # ← NEW: tells azd what to deploy and where
├── infra/              # ← NEW: Infrastructure as Code (Bicep by default)
│   ├── main.bicep
│   └── main.parameters.json
├── src/ (your code)    # unchanged
└── ...                 # your existing files, untouched
```

### `azure.yaml` — ਪ੍ਰਾਜੈਕਟ ਪਰਿਭਾਸ਼ਾ

ਇਹ azd ਪ੍ਰਾਜੈਕਟ ਦਾ ਦਿਲ ਹੈ। ਇੱਕ ਨਿਊਨਤਮ ਫਾਇਲ ਇਸ ਤਰ੍ਹਾਂ ਦਿਖਦੀ ਹੈ:

```yaml
# azure.yaml
name: my-existing-app
services:
  web:
    project: ./src           # path to your app code
    language: js             # js | python | dotnet | java
    host: appservice         # appservice | containerapp | function | staticwebapp
```

`services` ਬਲਾਕ ਮੁੱਖ ਹਿੱਸਾ ਹੈ: ਹਰ ਐਂਟਰੀ ਤੁਹਾਡੇ ਕੋਡ ਦੇ ਇੱਕ ਫੋਲਡਰ ਨੂੰ ਇੱਕ Azure ਹੋਸਟ ਨਾਲ ਨਕਸ਼ਾ ਕਰਦੀ ਹੈ। ਜੇ ਤੁਹਾਡੇ ਐਪ ਵਿੱਚ ਫਰੰਟਐਂਡ ਅਤੇ API ਦੋਵੇਂ ਹਨ, ਤਾਂ ਤੁਹਾਡੇ ਕੋਲ ਦੋ ਸੇਵਾਵਾਂ ਹੋਣਗੀਆਂ।

### `infra/` — ਤੁਹਾਡੇ Azure ਰਿਸੋਰਸਜ਼ ਕੋਡ ਵਜੋਂ

`infra/` ਫੋਲਡਰ ਵਿੱਚ Bicep ਫਾਇਲਾਂ ਹੁੰਦੀਆਂ ਹਨ ਜੋ ਉਹ Azure ਰਿਸੋਰਸਜ਼ ਪਰਿਭਾਸ਼ਿਤ ਕਰਦੀਆਂ ਹਨ ਜਿਨ੍ਹਾਂ ਦੀ ਤੁਹਾਡੇ ਐਪ ਨੂੰ ਲੋੜ ਹੈ (App Service, ਡੇਟਾਬੇਸ ਆਦਿ)। ਤੁਹਾਨੂੰ ਇਹ ਹੱਥੋਂ ਲਿਖਣ ਦੀ ਲੋੜ ਨਹੀਂ—azd ਇਕ ਕਾਰਗਰ ਸ਼ੁਰੂਆਤ ਨੁਕਤਾ ਜਨਰੇਟ ਕਰਦਾ ਹੈ। ਤੁਸੀਂ ਬਾਅਦ ਵਿੱਚ ਇਹਨਾਂ ਨੂੰ ਸੋਧ ਸਕਦੇ ਹੋ ਤਾਂ ਕਿ ਰਿਸੋਰਸਜ਼ ਸ਼ਾਮਲ ਕਰੋ ਜਾਂ ਸੁਰੱਖਿਆ ਕਸੋਬਨ ਕਰੋ (ਜੋ [ਅਧਿਆਇ 4](../chapter-04-infrastructure/README.md) ਵਿੱਚ ਕਵਰ ਕੀਤਾ ਗਿਆ ਹੈ)।

> **ਸੁਝਾਅ:** ਡਿਪਲੌਇ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ ਜਨਰੇਟ ਕੀਤੀ ਇੰਫ੍ਰਾਸ਼ਟ੍ਰੱਕਚਰ ਨੂੰ ਵੇਖਣਾ ਜਾਂ ਕਸਟਮਾਈਜ਼ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ? `azd infra generate` (ਜੋ `azd infra synth` ਦੇ ਤੌਰ ਤੇ ਵੀ ਉਪਲਬਧ ਹੈ) ਚਲਾਓ, ਤਾਂ ਜੋ IaC ਨੂੰ ਡਿਸਕ ਤੇ ਲਿਖਿਆ ਜਾਵੇ ਤਾਂ ਜੋ ਤੁਸੀਂ ਇਸਨੂੰ ਸਮੀਖਿਆ ਅਤੇ ਵਰਜ਼ਨ-ਕੰਟਰੋਲ ਕਰ ਸਕੋ।

## ਕਦਮ 3: ਜਰੂਰੀ ਸੰਰਚਨਾ ਸੈਟ ਕਰੋ

ਜੇ ਤੁਹਾਡੇ ਐਪ ਨੂੰ ਸੈਟਿੰਗਜ਼ ਜਾਂ ਸਿਕ੍ਰੇਟ ਦੀ ਲੋੜ ਹੈ (ਜਿਵੇਂ ਕਨੇਕਸ਼ਨ ਸਟਰਿੰਗ, API ਕੀ), ਤਾਂ ਇਨ੍ਹਾਂ ਨੂੰ ਹਾਰਡਕੋਡ ਨਾ ਕਰੋ। ਇਨਵਾਇਰਨਮੈਂਟ ਵੈਲਿਊਜ਼ ਵਰਤੋਂ:

```bash
# ਇੱਕ ਵਾਤਾਵਰਣ ਬਣਾਓ
azd env new dev

# ਇੱਕ ਗੈਰ-ਰਹੱਸਮਈ ਮੁੱਲ ਸੈੱਟ ਕਰੋ
azd env set API_VERSION 1.0.0
```

ਅਸਲੀ ਸਿਕ੍ਰੇਟਾਂ ਲਈ, ਉਨ੍ਹਾਂ ਨੂੰ Key Vault ਵਿੱਚ ਸਟੋਰ ਕਰੋ ਅਤੇ ਆਪਣੀ ਇੰਫ੍ਰਾਸ਼ਟ੍ਰੱਕਚਰ ਤੋਂ ਉਨ੍ਹਾਂ ਨੂੰ ਰੈਫਰੈਂਸ ਕਰੋ—ਦੇਖੋ [Chapter 3: Configuration & Authentication](../chapter-03-configuration/authsecurity.md).

## ਕਦਮ 4: ਡਿਪਲੌਇ

ਹੁਣ ਉਹੀ ਵਰਕਫਲੋ ਵਰਤੋਂ ਜਿਸਨੂੰ ਤੁਸੀਂ ਪਹਿਲਾਂ ਜਾਣਦੇ ਹੋ:

```bash
# ਪ੍ਰਮਾਣਿਕਤਾ (azd ਲਈ ਲਾਜ਼ਮੀ)
azd auth login

# ਉਹ ਸਰੋਤ ਵੇਖੋ ਜੋ ਬਣਾਏ ਜਾਣਗੇ
azd provision --preview

# ਬੁਨਿਆਦੀ ਢਾਂਚਾ ਮੁਹੱਈਆ ਕਰੋ ਅਤੇ ਆਪਣਾ ਕੋਡ ਡਿਪਲੋਇ ਕਰੋ
azd up
```

ਜਦੋਂ ਇਹ ਮੁਕੰਮਲ ਹੋ ਜਾਵੇ, azd ਤੁਹਾਡੇ ਐਪ ਦਾ URL ਪ੍ਰਿੰਟ ਕਰਦਾ ਹੈ। ਇਸਨੂੰ ਕਿਸੇ ਵੀ azd ਐਪ ਵਾਂਗ ਹੀ ਵੈਰੀਫਾਈ ਕਰੋ:

```bash
azd show           # ਐਂਡਪੌਇੰਟਸ ਦਿਖਾਓ
azd monitor --logs # ਲੌਗ ਚੈੱਕ ਕਰੋ ਜੇ ਲੋੜ ਹੋਵੇ
```

| ਲਕਸ਼ਣ | ਸੰਭਾਵਿਤ ਕਾਰਨ | ਠੀਕ ਕਰਨਾ |
|---------|--------------|-----|
| azd ਨੇ ਮੇਰਾ ਐਪ ਡਿਟੈਕਟ ਨਹੀਂ ਕੀਤਾ | ਮੈਨਿਫੈਸਟ ਦੀ ਘਾਟ (ਉਦਾਹਰਣ ਲਈ `package.json`) | ਮੈਨਿਫੈਸਟ ਜੋੜੋ, ਜਾਂ `azd init` ਦੌਰਾਨ ਹੋਸਟ ਨੂੰ ਮੈਨੂਅਲੀ ਚੁਣੋ |
| `azd up` ਦੌਰਾਨ ਬਿਲਡ ਫੇਲ ਹੁੰਦੀ ਹੈ | ਐਪ ਨੂੰ ਬਿਲਡ ਸਟੈਪ ਦੀ ਲੋੜ ਹੈ | `azure.yaml` ਵਿੱਚ ਸੇਵਾ ਹੇਠਾਂ `buildCommand`/`outputPath` ਜੋੜੋ |
| ਐਪ ਸਟਾਰਟ ਹੁੰਦਾ ਹੈ ਪਰ ਐਰਰਸ ਵਾਪਸ ਕਰਦਾ ਹੈ | ਕਾਨਫਿਗ/ਸਿਕ੍ਰੇਟ ਗੁੰਮ ਹੈ | `azd env set` ਨਾਲ ਵੈਲਿਊਜ਼ ਸੈੱਟ ਕਰੋ ਜਾਂ Key Vault ਨੂੰ ਵਾਇਰ ਕਰੋ |
| ਗਲਤ ਹੋਸਟ ਚੁਣਿਆ ਗਿਆ | ਆਟੋ-ਡਿਟੈਕਸ਼ਨ ਨੇ ਅਨੁਮਾਨ ਲਾਇਆ | `azure.yaml` ਵਿੱਚ `host:` ਐਡੀਟ ਕਰੋ ਅਤੇ ਫਿਰ `azd up` ਚਲਾਓ |

ਵਧੇਰੇ ਲਈ ਵੇਖੋ [Chapter 7: Troubleshooting](../chapter-07-troubleshooting/README.md).

- `azd init` → **"Use code in the current directory"** ਤੁਹਾਡੇ ਕੋਲ ਪਹਿਲਾਂ ਤੋਂ ਮੌਜੂਦਾ ਐਪ ਵਿੱਚ azd ਜੋੜਦਾ ਹੈ।
- `azure.yaml` ਤੁਹਾਡੇ ਕੋਡ ਫੋਲਡਰਾਂ ਨੂੰ Azure ਹੋਸਟਾਂ ਨਾਲ ਨਕਸ਼ਾ ਕਰਦਾ ਹੈ; `infra/` ਰਿਸੋਰਸਜ਼ ਨੂੰ Bicep ਵਜੋਂ ਪਰਿਭਾਸ਼ਿਤ ਕਰਦਾ ਹੈ।
- `azd infra generate` ਤੁਹਾਨੂੰ ਜਨਰੇਟ ਕੀਤੀ ਇੰਫ੍ਰਾਸ਼ਟ੍ਰੱਕਚਰ ਨੂੰ ਸਮੀਖਿਆ ਜਾਂ ਕਸਟਮਾਈਜ਼ ਕਰਨ ਦੀ ਆਜ਼ਾਦੀ ਦਿੰਦਾ ਹੈ।
- ਇੱਕ ਵਾਰੀ ਇਨੀਸ਼ੀਅਲਾਈਜ਼ ਹੋਣ ਤੋਂ ਬਾਅਦ, ਤੁਹਾਡਾ ਮੌਜੂਦਾ ਐਪ ਬਿਲਕੁਲ ਉਹੀ `azd up` / `azd down` ਵਰਕਫਲੋ ਵਰਤੇਗਾ ਜੋ ਟੈਂਪਲੇਟ-ਅਧਾਰਿਤ ਐਪ ਵਰਤਦਾ ਹੈ।
