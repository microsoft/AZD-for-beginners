## ਪਰਿਚਯ

[ਤੁਹਾਡਾ ਪਹਿਲਾ ਪ੍ਰਾਜੈਕਟ](first-project.md) ਵਿੱਚ ਤੁਸੀਂ ਟੈਮਪਲੇਟ ਤੋਂ ਸ਼ੁਰੂ ਕਰਕੇ ਇੱਕ ਐਪ ਤਿਆਰ ਕੀਤਾ। ਪਰ ਅਕਸਰ ਤੁਹਾਡੇ ਕੋਲ ਪਹਿਲਾਂ ਹੀ ਇੱਕ ਐਪ ਹੁੰਦੀ ਹੈ—ਜਿਵੇਂ ਕਿ Node.js API, Python Flask ਸੇਵਾ, .NET ਵੈੱਬ ਐਪ—ਜੋ ਤੁਹਾਡੇ ਕੰਪਿਊਟਰ ਦੀ ਫੋਲਡਰ ਵਿਚ ਬੈਠੀ ਹੁੰਦੀ ਹੈ। ਇਹ ਪਾਠ ਦਿਖਾਉਂਦਾ ਹੈ ਕਿ ਮੌਜੂਦਾ ਕੋਡ ਵਿੱਚ ਕਿਵੇਂ azd ਸ਼ਾਮਲ ਕੀਤਾ ਜਾਵੇ ਤਾਂ ਜੋ ਤੁਸੀਂ ਇਹ `azd up` ਨਾਲ ਡਿਪਲੋਇ ਕਰ ਸਕੋ, ਕਿਸੇ ਟੈਮਪਲੇਟ ਦੀ ਲੋੜ ਨਹੀਂ।

## ਸਿੱਖਣ ਦੇ ਲਕੜੀ

ਇਸ ਪਾਠ ਦੇ ਅਖੀਰ ਤੱਕ, ਤੁਸੀਂ:
- azd ਪ੍ਰਾਜੈਕਟ ਸ਼ੁਰੂ ਕਰਨ ਦੇ ਤਿੰਨ ਤਰੀਕਿਆਂ ਨੂੰ ਸਮਝੋਗੇ
- ਮੌਜੂਦਾ ਕੋਡਬੇਸ ਵਿੱਚ `azd init` ਚਲਾਉਣਗੇ
- ਜਾਣੋਗੇ ਕਿ `azure.yaml` ਅਤੇ `infra/` ਫੋਲਡਰ ਤੁਹਾਡੇ ਐਪ ਲਈ ਕੀ ਕਰਦੇ ਹਨ
- ਕਦੋਂ azd ਇੰਫ੍ਰਾਸਟਰੱਕਚਰ ਬਣਾਉਂਦਾ ਹੈ ਤੇ ਕਦੋਂ ਤੁਸੀਂ ਖੁਦ ਲਿਖਦੇ ਹੋ, ਇਹ ਸਮਝੋਗੇ
- `azd up` ਨਾਲ ਆਪਣੀ ਮੌਜੂਦਾ ਐਪ ਨੂੰ Azure 'ਤੇ ਡਿਪਲੋਇ ਕਰੋ

## ਸਿੱਖਣ ਦੇ ਨਤੀਜੇ

ਇਸ ਪਾਠ ਨੂੰ ਪੂਰਾ ਕਰਨ ਦੇ ਬਾਅਦ, ਤੁਸੀਂ ਸਮਰੱਥ ਹੋਵੋਗੇ:
- azd ਨੂੰ ਆਪਣੇ ਮੌਜੂਦਾ ਪ੍ਰਾਜੈਕਟ ਵਿੱਚ ਸ਼ੁਰੂ ਕਰੋ
- ਇੱਕ ਮੂਲ `azure.yaml` ਫਾਇਲ ਪੜ੍ਹਨ ਅਤੇ ਸੋਧਣ ਦੇ ਯੋਗ ਹੋਵੋ
- `azd infra generate` ਨਾਲ ਸਟਾਰਟਰ ਇਨਫ੍ਰਾਸਟਰੱਕਚਰ ਤਿਆਰ ਕਰੋ
- ਆਪਣੇ ਐਪ ਲਈ ਯੋਗ Azure ਹੋਸਟ ਚੁਣੋ
- ਆਪਣੀ ਐਪ ਨੂੰ ਡਿਪਲੋਇ ਅਤੇ ਸਾਫ਼ ਕਰੋ

## azd ਪ੍ਰਾਜੈਕਟ ਸ਼ੁਰੂ ਕਰਨ ਦੇ ਤਿੰਨੇ ਤਰੀਕੇ

| ਸ਼ੁਰੂਆਤ ਦਾ ਮੂਕਾਮ | ਕਮਾਣਡ | ਕਦੋਂ ਵਰਤਣਾ ਹੈ |
|----------------|---------|-------------|
| **ਇੱਕ ਟੈਮਪਲੇਟ ਤੋਂ** | `azd init --template <name>` | ਸਿੱਖਣ ਲਈ, ਜਾਂ ਇੱਕ ਨਵੀਂ ਐਪ ਸ਼ੁਰੂ ਕਰਨ ਲਈ ਜੋ ਸਾਬਤ ਨਮੂਨਾ ਤੋਂ ਹੈ |
| **ਤੁਹਾਡੇ ਮੌਜੂਦਾ ਕੋਡ ਤੋਂ** | `azd init` (ਤੁਹਾਡੇ ਪ੍ਰਾਜੈਕਟ ਫੋਲਡਰ ਵਿੱਚ) | ਤੁਹਾਡੇ ਕੋਲ ਪਹਿਲਾਂ ਹੀ ਇੱਕ ਐਪ ਹੈ ਅਤੇ ਤੁਸੀਂ ਇਸ ਨੂੰ ਡਿਪਲੋਇ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ |
| **Git ਰਿਪੋ ਤੋਂ** | `azd init --from-code` (ਕਲੋਨ ਕੀਆ ਰਿਪੋ ਵਿੱਚ) | ਮੌਜੂਦਾ ਰਿਪੋ ਲਈ azd ਅਪਣਾਉਣਾ |

ਤੁਸੀਂ ਪਹਿਲੀ ਵਿਕਲਪ ਦਾ ਅਭਿਆਸ ਕੀਤਾ ਹੈ। ਇਹ ਪਾਠ ਦੂਸਰੇ ਖਾਸ-ਕਿਸਮ ਸਥਿਤੀ ਨੂੰ ਕਵਰ ਕਰਦਾ ਹੈ ਜੋ ਕਿ ਸਭ ਤੋਂ ਆਮ ਹੈ।

## ਕਦਮ 1: ਆਪਣੇ ਪ੍ਰਾਜੈਕਟ ਵਿੱਚ `azd init` ਚਲਾਓ

ਆਪਣੇ ਮੌਜੂਦਾ ਪ੍ਰਾਜੈਕਟ ਫੋਲਡਰ ਦੇ ਅੰਦਰ ਇੱਕ ਟਰਮੀਨਲ ਖੋਲ੍ਹੋ ਅਤੇ ਚਲਾਓ:

```bash
cd my-existing-app
azd init
```

azd ਪੁੱਛੇਗਾ ਕਿ ਤੁਸੀਂ ਕਿਵੇਂ ਸ਼ੁਰੂਆਤ ਕਰਨੀ ਹੈ। ਚੁਣੋ:

```
? How do you want to initialize your app?
> Use code in the current directory
  Select a template
```

***"ਮੌਜੂਦਾ ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਕੋਡ ਦੀ ਵਰਤੋਂ ਕਰੋ."*** ਚੁਣੋ। ਫਿਰ azd ਤੁਹਾਡੇ ਫੋਲਡਰ ਨੂੰ ਸਕੈਨ ਕਰਦਾ ਹੈ, ਭਾਸ਼ਾ ਅਤੇ ਫਰੇਮਵਰਕ ਪਤਾ ਲਗਾਉਂਦਾ ਹੈ ਅਤੇ ਹੋਸਟ ਸੁਝਾਅ ਦਿੰਦਾ ਹੈ।

### azd ਕੀ ਪਤਾ ਲਗਾਉਂਦਾ ਹੈ

azd `package.json`, `requirements.txt`, `pom.xml`, `*.csproj` ਜਾਂ `Dockerfile` ਵਰਗੇ ਸੰਕੇਤ ਖੋਜਦਾ ਹੈ ਅਤੇ ਇੱਕ ਮਿਲਦੇ-ਜੁਲਦੇ Azure ਹੋਸਟ ਦੀ ਸਿਫਾਰਿਸ਼ ਕਰਦਾ ਹੈ:

| ਤੁਹਾਡੀ ਐਪ | ਮੁਮਕਿਨ ਹੈ ਜੋ ਜਾਚੀ ਗਈ ਹੋਸਟ ਹੈ |
|----------|-----------------------------|
| Node.js / Python / .NET ਵੈੱਬ ਐਪ | Azure App Service ਜਾਂ Container Apps |
| ਕੰਟੇਨਰਾਈਜ਼ਡ ਐਪ (`Dockerfile`) | Azure Container Apps |
| ਫੰਕਸ਼ਨ ਐਪ | Azure Functions |
| ਸਟੈਟਿਕ ਸਾਈਟ (React/Vue ਬਿਲਡ ਆਉਟਪੁੱਟ) | Azure Static Web Apps |

ਪਤਾ ਲਗਾਏ ਗਏ ਸਰਵਿਸ(ਜ਼) ਦੀ ਪੁਸ਼ਟੀ ਕਰੋ, ਅਤੇ azd ਤੁਹਾਨੂੰ ਲੋੜੀਂਦੇ ਫਾਇਲਾਂ ਦਾ ਸੰਰਚਨਾ ਕਰਦਾ ਹੈ।

## ਕਦਮ 2: ਸਮਝੋ ਕਿ azd ਨੇ ਕੀ ਬਣਾਇਆ

init ਤੋਂ ਬਾਅਦ, ਤੁਹਾਡੇ ਪ੍ਰਾਜੈਕਟ ਵਿੱਚ ਦੋ ਨਵੀਆਂ ਚੀਜ਼ਾਂ ਹੋਣਗੀਆਂ:

```
my-existing-app/
├── azure.yaml          # ← NEW: tells azd what to deploy and where
├── infra/              # ← NEW: Infrastructure as Code (Bicep by default)
│   ├── main.bicep
│   └── main.parameters.json
├── src/ (your code)    # unchanged
└── ...                 # your existing files, untouched
```

### `azure.yaml` — ਪ੍ਰਾਜੈਕਟ ਦੀ ਪਰਿਭਾਸ਼ਾ

ਇਹ azd ਪ੍ਰਾਜੈਕਟ ਦਾ ਦਿਲ ਹੈ। ਇੱਕ ਸਧਾਰਣ ਵਰਜਨ ਇਸ ਤਰ੍ਹਾਂ ਹੁੰਦਾ ਹੈ:

```yaml
# azure.yaml
name: my-existing-app
services:
  web:
    project: ./src           # path to your app code
    language: js             # js | python | dotnet | java
    host: appservice         # appservice | containerapp | function | staticwebapp
```

`services` ਹਿੱਸਾ ਮੁੱਖ ਹੈ: ਹਰ ਐਂਟਰੀ ਤੁਹਾਡੇ ਕੋਡ ਦੇ ਇੱਕ ਫੋਲਡਰ ਨੂੰ Azure ਹੋਸਟ ਨਾਲ ਜੋੜਦੀ ਹੈ। ਜੇ ਤੁਹਾਡੇ ਐਪ ਦਾ ਫਰੰਟਐਂਡ ਅਤੇ API ਦੋਵੇਂ ਹਨ, ਤਾਂ ਤੁਹਾਡੇ ਕੋਲ ਦੋ ਸਰਵਿਸ ਫੋਲਡਰ ਹੋਣਗੇ।

### `infra/` — ਤੁਹਾਡੇ Azure ਸਰੋਤ ਕੋਡ ਵਜੋਂ

`infra/` ਫੋਲਡਰ ਵਿੱਚ ਉਹ Bicep ਫਾਇਲਾਂ ਹਨ ਜੋ ਤੁਹਾਡੇ ਐਪ ਲਈ Azure ਸਰੋਤ (ਜਿਵੇਂ ਕਿ App Service, ਡੈਟਾਬੇਸ ਆਦਿ) ਦੀ ਪਰਿਭਾਸ਼ਾ ਕਰਦੀਆਂ ਹਨ। ਤੁਹਾਨੂੰ ਇਨ੍ਹਾਂ ਨੂੰ ਹੱਥ ਨਾਲ ਲਿਖਣ ਦੀ ਲੋੜ ਨਹੀਂ—azd ਇੱਕ ਕਾਮਯਾਬ ਸ਼ੁਰੂਆਤੀ ਨੁਕਤਾ ਤਿਆਰ ਕਰਦਾ ਹੈ। ਬਾਅਦ ਵਿੱਚ ਤੁਸੀਂ ਇਨ੍ਹਾਂ ਨੂੰ ਸੋਧ ਕੇ ਸਰੋਤ ਜੋੜ ਸਕਦੇ ਹੋ ਜਾਂ ਸੁਰੱਖਿਆ ਕੜੀ ਕਰ ਸਕਦੇ ਹੋ ([ਅਧਿਆਇ 4](../chapter-04-infrastructure/README.md) ਵਿੱਚ ਵਰਣਨ ਕੀਤਾ ਗਿਆ ਹੈ)।

> **ਟਿਪ:** ਜੇ ਤੁਸੀਂ ਡਿਪਲੋਇ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ ਬਣਾਇਆ ਹੋਇਆ ਇਨਫ੍ਰਾਸਟਰੱਕਚਰ ਦੇਖਣਾ ਜਾਂ ਕাস্টਮਾਈਜ਼ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ ਤਾਂ `azd infra generate` (ਜੋ `azd infra synth` ਵਜੋਂ ਵੀ ਉਪਲੱਬਧ ਹੈ) ਚਲਾਓ, ਇਸ ਨਾਲ IaC ਤੁਹਾਡੇ ਡਿਸਕ 'ਤੇ ਲਿਖ ਜਾਂਦਾ ਹੈ ਅਤੇ ਤੁਸੀਂ ਇਸ ਦੀ ਸਮੀਖਿਆ ਅਤੇ ਵਰਜਨ-ਕੰਟਰੋਲ ਕਰ ਸਕਦੇ ਹੋ।

## ਕਦਮ 3: ਜਰੂਰੀ ਸੰਰਚਨਾ ਸੈਟ ਕਰੋ

ਜੇ ਤੁਹਾਡੇ ਐਪ ਨੂੰ ਸੈਟਿੰਗਜ਼ ਜਾਂ ਸਟੀਕੜੇ (ਜਿਵੇਂ ਕਨੇਕਸ਼ਨ ਸਟਰਿੰਗ, API ਕੀ) ਦੀ ਲੋੜ ਹੈ ਤਾਂ ਉਹਨਾਂ ਨੂੰ ਹਾਰਡਕੋਡ ਨਾ ਕਰੋ। ਵਾਤਾਵਰਣ ਮੁੱਲਾਂ ਦੀ ਵਰਤੋਂ ਕਰੋ:

```bash
# ਇੱਕ ਵਾਤਾਵਰਣ ਬਣਾਓ
azd env new dev

# ਇਕ ਗੁਪਤ-ਭਰੇ ਹੋਏ ਨਾਲ ਨਾ ਹੋਈ ਮੁੱਲ ਸੈੱਟ ਕਰੋ
azd env set API_VERSION 1.0.0
```

ਅਸਲੀ ਸਟੀਕੜੇ Key Vault ਵਿੱਚ ਸਟੋਰ ਕਰੋ ਅਤੇ ਉਹਨਾਂ ਨੂੰ ਆਪਣੇ ਇਨਫ੍ਰਾਸਟਰੱਕਚਰ ਵਿੱਚ ਰੈਫਰੰਸ ਕਰੋ—ਵੇਖੋ [ਅਧਿਆਇ 3: ਸੰਰਚਨਾ ਅਤੇ ਪ੍ਰਮਾਣਿਕਤਾ](../chapter-03-configuration/authsecurity.md)।

## ਕਦਮ 4: ਡਿਪਲੋਇ ਕਰੋ

ਹੁਣ ਉਹੀ ਵਰਕਫਲੋ ਵਰਤੋਂ ਜੋ ਤੁਸੀਂ ਪਹਿਲਾਂ ਜਾਣਦੇ ਹੋ:

```bash
# ਪ੍ਰਮਾਣਿਕਤਾ (azd ਲਈ ਲਾਜ਼ਮੀ)
azd auth login

# ਉਹ ਸਰੋਤ ਦੇਖੋ ਜੋ ਬਣਾਏ ਜਾਣਗੇ
azd provision --preview

# ਢਾਂਚਾ ਮੁਹਈਆ ਕਰਵਾਉ ਅਤੇ ਆਪਣਾ ਕੋਡ ਤਾਇਨਾਤ ਕਰੋ
azd up
```

ਮੁਕੰਮਲ ਹੋਣ ਤੇ, azd ਤੁਹਾਡੇ ਐਪ ਦਾ URL ਪ੍ਰਿੰਟ ਕਰਦਾ ਹੈ। ਇਸਨੂੰ ਕਿਸੇ ਵੀ azd ਐਪ ਵਾਂਗ ਜाँचੋ:

```bash
azd show           # ਐਂਡਪੋਇੰਟ ਦਿਖਾਓ
azd monitor --logs # ਜ਼ਰੂਰਤ ਹੋਵੇ ਤਾਂ ਲੌਗ ਜਾਂਚੋ
```

## ਆਮ ਪਹਿਲੀ ਵਾਰੀ ਸਮੱਸਿਆਵਾਂ

| ਲੱਛਣ | ਸੰਭਾਵਿਤ ਕਾਰਨ | ਸਹੀ ਕਰੋ |
|---------|--------------|-----|
| azd ਨੇ ਮੇਰੀ ਐਪ ਨਹੀਂ ਪਹਿਚਾਨੀ | ਮੈਨਿਫੈਸਟ ਨਹੀਂ (ਜਿਵੇਂ ਕਿ `package.json`) | ਮੈਨਿਫੈਸਟ ਸ਼ਾਮਲ ਕਰੋ, ਜਾਂ `azd init` ਚਲਾਉਂਦਿਆਂ ਹੋਸਟ ਹੱਥੋਂ ਚੁਣੋ |
| `azd up` ਦੌਰਾਨ ਬਿਲਡ ਫੇਲ ਹੁੰਦਾ ਹੈ | ਐਪ ਨੂੰ ਬਿਲਡ ਕਦਮ ਦੀ ਜ਼ਰੂਰਤ ਹੈ | `azure.yaml` ਵਿੱਚ ਸਰਵਿਸ ਹੇਠਾਂ `buildCommand`/`outputPath` ਜੋੜੋ |
| ਐਪ ਚੱਲਦਾ ਹੈ ਪਰ ਗਲਤੀਆਂ ਵਾਪਸ ਕਰਦਾ ਹੈ | ਸੰਰਚਨਾ/ਸਟੀਕੜਾ ਗੁੰਮ ਹੈ | `azd env set` ਨਾਲ ਮੁੱਲ ਸੈਟ ਕਰੋ ਜਾਂ Key Vault ਨਾਲ ਜੋੜੋ |
| ਗਲਤ ਹੋਸਟ ਚੁਣਿਆ ਗਿਆ | ਸਵੈ-ਪਹਿਚਾਣੀ ਅੰਦਾਜ਼ਾ ਹੈ | `azure.yaml` ਵਿੱਚ `host:` ਸੋਧੋ ਅਤੇ `azd up` ਦੁਬਾਰਾ ਚਲਾਓ |

ਵਧੇਰੇ ਜਾਣਕਾਰੀ ਲਈ, ਵੇਖੋ [ਅਧਿਆਇ 7: ਸਮੱਸਿਆ ਨਿਵਾਰਨ](../chapter-07-troubleshooting/README.md)।

## ਸੰਖੇਪ

- `azd init` → **"ਮੌਜੂਦਾ ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਕੋਡ ਦੀ ਵਰਤੋਂ ਕਰੋ"** ਤੁਹਾਡੇ ਕੋਲ ਪਹਿਲਾਂ ਹੀ ਮੌਜੂਦਾ ਐਪ ਵਿੱਚ azd ਸ਼ਾਮਲ ਕਰਦਾ ਹੈ।
- `azure.yaml` ਤੁਹਾਡੇ ਕੋਡ ਫੋਲਡਰਾਂ ਨੂੰ Azure ਹੋਸਟ ਨਾਲ ਜੋੜਦਾ ਹੈ; `infra/` ਸਰੋਤਾਂ ਨੂੰ Bicep ਵਜੋਂ ਪਰਿਭਾਸ਼ਿਤ ਕਰਦਾ ਹੈ।
- `azd infra generate` ਤੁਹਾਨੂੰ ਬਣਾਈ ਹੋਈ ਇਨਫ੍ਰਾਸਟਰੱਕਚਰ ਦੀ ਸਮੀਖਿਆ ਜਾਂ ਕਸਟਮਾਈਜ਼ ਕਰਨ ਦਿੰਦਾ ਹੈ।
- ਇੱਕ ਵਾਰੀ ਸ਼ੁਰੂ ਹੋਣ ਤੋਂ ਬਾਅਦ, ਤੁਹਾਡੀ ਮੌਜੂਦਾ ਐਪ ਟੈਮਪਲੇਟ-ਅਧਾਰਿਤ ਐਪ ਵਾਂਗ ਹੀ `azd up` / `azd down` ਵਰਕਫਲੋ ਦੀ ਵਰਤੋਂ ਕਰਦੀ ਹੈ।
