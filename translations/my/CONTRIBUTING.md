# AZD For Beginners သို့ ပါဝင်ဆောင်ရွက်ခြင်း

AZD For Beginners သင်တန်းတွင် ပါဝင်ရန် စိတ်ပါဝင်စားမှုအတွက် ကျေးဇူးတင်ပါသည်။ ဤစာတမ်းသည် ပံ့ပိုးသူများအတွက် လမ်းညွှန်ချက်များနှင့် အချက်အလက်များကို ပေးထားပါသည်။

## ပါဝင်ဆောင်ရွက်နည်းများ

### ပြဿနာများ တင်ပြခြင်း

If you find bugs, errors, or have suggestions for improvements:

1. **ရှိပြီးသား ပြဿနာများကို စစ်ဆေးပါ**: တူထပ်မှုများ မဖြစ်စေရန် [ရှိပြီးသား ပြဿနာများ](https://github.com/microsoft/azd-for-beginners/issues) ကို ရှာဖွေပါ
2. **ပြဿနာအသစ်တစ်ခု ဖန်တီးပါ**: သင့်အတွက် သင့်လျော်သော issue ဖောင်ကို အသုံးပြုပါ
3. **အသေးစိတ် အချက်အလက် ပေးပါ**: ပြန်ဖန်တီးရန် လုပ်ဆောင်ရမည့် အဆင့်များ၊ မျှော်လင့်ထားသော လုပ်ဆောင်မှုနှင့် လိုလျှင် screenshot များကို ပါဝင်စေရန် ထည့်ပါ

### ပြောင်းလဲချက်များ အကြံပြုခြင်း

1. **Repository ကို Fork လုပ်ပါ**: ပရောဂျက်၏ ကိုယ့်ကိုယ်ပိုင် fork ကို ဖန်တီးပါ
2. **Branch တစ်ခု ဖန်တီးပါ**: သင့်ပြောင်းလဲမှုများအတွက် feature branch တစ်ခု ဖန်တီးပါ
3. **ပြောင်းလဲချက်များ ပြုလုပ်ပါ**: ကျွန်တော်တို့၏ ကုတ်ရေးနည်းနှင့် စာတမ်း စံသတ်မှတ်ချက်များကို လိုက်နာပါ
4. **သင့်ပြောင်းလဲချက်များကို စမ်းသပ်ပါ**: လင့်ခ်များအားလုံး လုပ်ဆောင်နိုင်ကြောင်းနှင့် အကြောင်းအရာများ မှန်ကန်စွာ ပြသနေကြောင်း အတည်ပြုပါ
5. **Pull Request တင်ပါ**: သင့်ပြောင်းလဲချက်များကို ဖော်ပြချက် ထပ်၍ ရှင်းလင်းစွာ ပေးပို့ပါ

### ပံ့ပိုးမှု အမျိုးအစားများ

ကျွန်တော်တို့သည် အောက်ပါ ပံ့ပိုးမှုအမျိုးအစားများကို ကြိုဆိုပါသည်။

#### အကြောင်းအရာ တိုးတက်စေခြင်း
- စာလုံးပေါင်း အမှား၊ သဒ္ဒါနှင့် ဖော်မတ်တင် ပြဿနာများကို ပြင်ဆင်ပါ
- ရှင်းလင်းမှုရှိစေရန် ရှိပြီးသား စာတမ်းများကို တိုးတက်စေပါ
- ဟောင်းနေသော အချက်အလက်များကို အပ်ဒိတ်လုပ်ပေးပါ
- မပါရှိသေးသော ရှင်းပြချက်များ သို့မဟုတ် ပတ်ဝန်းကျင် အချက်အလက်များ ထည့်သွင်းပေးပါ

#### ဥပမာ အသစ်များ
- လက်တွေ့ deployment အခြေအနေများကို ထည့်သွင်းပါ
- ပုံမှန် အသုံးပြုမှုများအတွက် AZD templates အသစ်များ ဖန်တီးပါ
- နောက်ထပ် ကုတ်နမူနာများနှင့် လေ့ကျင့်ခန်းများ ပံ့ပိုးပါ

#### ဘာသာပြန် အထောက်အပံ့
- ရှိပြီးသား ဘာသာပြန်များကို ထိန်းသိမ်းရန် ကူညီပါ
- ဘာသာပြန်အရည်အသွေးကို တိုးတက်စေပါ
- အကြောင်းအရာ အသစ်များအတွက် ဘာသာပြန်များ ထည့်သွင်းပါ

#### အမှား ရှာဖွေတင်ပြချက်များ
- ဖျက်စီးနေသော လင့်ခ်များ သို့မဟုတ် မှားနေသော ကုတ်နမူနာများကို တင်ပြပါ
- စာတမ်းများတွင် မညီမျှနေမှုများကို ဖော်ထုတ်ပါ
- အသုံးပြုနိုင်မှုပြဿနာများ (accessibility issues) ကို သတိပေးပါ

## ပံ့ပိုးမှု လမ်းညွှန်ချက်များ

### စာတမ်း စံနှုန်းများ

- ရှင်းလင်း၍ စတင်သင်ယူသူများအတွက် သင့်လျော်သော ဘာသာစကားကို အသုံးပြုပါ
- သင်ယူရမည့် ရည်မှန်းချက်များနှင့် ရလဒ်များကို ထည့်ပါ
- အဆင့်လိုက် လမ်းညွှန်ချက်များနှင့် မျှော်လင့်ထားသော ထွက်ရှိမှုများကို ဖော်ပြပါ
- ဆက်စပ် အကြောင်းအရာများသို့ သွားရန် လင့်ခ်များ ထည့်ပါ
- ဖော်မတ်တင်းနှင့် markdown စတိုင်ကို တစ်ပြိုင်နက် ထိန်းသိမ်းထားပါ

### ကုတ် နမူနာများ

- ကုတ်နမူနာများအားလုံးကို စမ်းသပ်ပြီး အလုပ်လုပ်နိုင်ကြောင်း အတည်ပြုပါ
- အဓိက အယူအဆများကို ရှင်းပြသည့် comment များ ထည့်ပါ
- Azure နှင့် AZD ၏ အကောင်းဆုံး လေ့ကျင့်မှုများကို လိုက်နာပါ
- လိုအပ်သလောက် မျှော်လင့်ထားသော ထွက်ရှိမှုများကို ဖော်ပြပါ

### Commit မက်ဆေ့ချ်များ

Use clear, descriptive commit messages:
- `docs: fix typo in installation guide`
- `feat: add new container app example`
- `fix: correct broken link in chapter 2`
