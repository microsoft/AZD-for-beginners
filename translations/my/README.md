## သင်တန်း အနှစ်ချုပ်

Azure Developer CLI (azd) ကို အဆင့်လိုက် သင်ကြားသည့် ပုဒ်မများမှ တက်ကြွပြီး ကျွမ်းကျင်ရန် လေ့ကျင့်ပါ။ **Microsoft Foundry အုံတွဲနဲ့ AI application deploy များအပေါ် အထူးအာရုံစိုက်ထားသည်။**

### မူရင်းဘာကြောင့် ဒီသင်တန်းသည် ခေတ်မီ ဖွံ့ဖြိုးသူများအတွက် အရေးကြီးသနည်း

Microsoft Foundry Discord ကွန်ယူနတီ အချက်အလက်အရ၊ **ဖွံ့ဖြိုးသူ ၄၅% မှ AZD ကို AI အလုပ်စနစ်များအတွက် အသုံးပြုလိုစေ** သို့သော် အောက်ပါ စိန်ခေါ်မှုများနှင့် ကြုံတွေ့ကြသည် -
- မျိုးစုံ multi-service AI အင်ဖရန်စင့်ချာများ၏ ရှုပ်ထွေးမှု
- ထုတ်လုပ်မှု AI deploy လုပ်ရာတွင် အကောင်းဆုံးလက်တွေ့များ
- Azure AI service များ၏ ပေါင်းစည်းမှုနှင့် ဖန်တီးမှုဆိုင်ရာ ဆက်တင်များ
- AI workloads များအတွက် ကုန်ကျစရိတ် ပြOptimမည့်နည်းလမ်းများ
- AI အထူး deployment ပြဿနာများကို ဖြေရှင်းခြင်း

### သင်ယူရေး ရည်ရွယ်ချက်များ

ဒီစနစ်တကျ သင်တန်းကို အပြီးသတ်သွားလျှင်၊ သင်သည် -
- **AZD အခြေခံ ကောင်းစွာ ကျွမ်းကျင်နိုင်မည်**: အခြေခံအတွေးအခေါ်များ၊ တပ်ဆင်ခြင်း၊ ဖန်တီးမှုဆိုင်ရာ ဆက်တင်များ
- **AI Applications များ Deploy လုပ်နိုင်မည်**: AZD ကို Microsoft Foundry ဝန်ဆောင်မှုများနှင့် တွဲပြီး အသုံးပြုခြင်း
- **Infrastructure as Code ကို အကောင်အထည်ဖော်နိုင်မည်**: Bicep templates ဖြင့် Azure resources ကို စီမံခန့်ခွဲခြင်း
- **Deployments များကို Troubleshoot လုပ်နိုင်မည်**: ပုံမှန် အခက်အခဲများ ဖြေရှင်းခြင်းနှင့် debugging
- **ထုတ်လုပ်မှုအတွက် အဆင်ပြေစေခြင်း**: လုံခြုံရေး၊ scaling၊ နေကြတက်စောင့်ကြည့်ခြင်း၊ ကုန်ကျစရိတ်စီမံခြင်း
- **Multi-Agent ဖြေရှင်းချက်များ တည်ဆောက်နိုင်မည်**: ရှုပ်ထွေးသော AI ဖန်တီးပုံစံများကို Deploy လုပ်နိုင်မှု

## 📚 သင်ယူရေး အခန်းများ

*သင့်စွမ်းရည်နှင့် ရည်မှန်းချက်အရ သင်ယူရေး လမ်းကြောင်း ရွေးချယ်ပါ*

### 🚀 အခန်း ၁: အခြေခံနှင့် အမြန်စတาร์့
**လိုအပ်ချက်များ**: Azure subscription, အခြေခံ command line အသိပညာ  
**ကာလ**: 30-45 မိနစ်  
**ရှုပ်ထွေးမှု**: ⭐

#### သင်ယူမည့် အရာများ
- Azure Developer CLI အခြေခံကို နားလည်ခြင်း
- သင့်ပလက်ဖောင်းပေါ် AZD တပ်ဆင်ခြင်း
- သင့် ပထမဆုံး အောင်မြင်သော deployment

#### သင်ယူရေး အရင်းအမြစ်များ
- **🎯 စတင်ရန် ဒီနေရာမှ**: [Azure Developer CLI ဆိုတာဘာလဲ?](../..)
- **📖 သီအိုရီ**: [AZD အခြေခံများ](docs/getting-started/azd-basics.md) - အဓိက အတွေးအခေါ်များနှင့် သတ်မှတ်ချက်များ
- **⚙️ စက်တင်ခြင်း**: [Installation & Setup](docs/getting-started/installation.md) - ပလက်ဖောင်းအလိုက် ညွှန်ကြားချက်များ
- **🛠️ လက်တွေ့ လက်ဖက်**: [သင့် ပထမဆုံး စီမံကိန်း](docs/getting-started/first-project.md) - ဆင့်ဆင့် လင့်ကဒ်
- **📋 အမြန်အရည်အချင်း ကိုင်တွယ်ရန်**: [Command Cheat Sheet](resources/cheat-sheet.md)

#### လက်တွေ့ လေ့ကျင့်ခန်းများ
```bash
# တပ်ဆင်မှုကို အမြန်စစ်ဆေးခြင်း
azd version

# သင့် ပထမဆုံး အက်ပလီကေးရှင်းကို တင်ပို့ပါ
azd init --template todo-nodejs-mongo
azd up
```

**💡 အခန်းရလဒ်**: AZD ကို အသုံးပြုပြီး ရိုးရှင်းသော ဝက်ဘ် အက်ပလီကေးရှင်းတစ်ခုကို Azure တွင် အောင်မြင်စွာ deploy လုပ်ခြင်း

**✅ အောင်မြင်မှု စစ်ဆေးခြင်း:**
```bash
# အခန်း ၁ ကို ပြီးမြောက်နိုင်သောအခါ သင်သည် အောက်ပါအရာများကို လုပ်ဆောင်နိုင်မည်။
azd version              # တပ်ဆင်ထားသော ဗားရှင်းကို ပြသသည်
azd init --template todo-nodejs-mongo  # ပရောဂျက်ကို အစပြုသည်
azd up                  # Azure သို့ တပ်ဆင်သည်
azd show                # လည်ပတ်နေသော အက်ပ်၏ URL ကို ပြသသည်
# အက်ပလီကေးရှင်းကို ဘရောဇာတွင် ဖွင့်၍ အလုပ်လုပ်သည်
azd down --force --purge  # အရင်းအမြစ်များကို ရှင်းလင်းသည်
```

**📊 စုစုပေါင်း အချိန်မြှုပ်နှံမှု:** 30-45 မိနစ်  
**📈 သင်ယူပြီးနောက် ကျွမ်းကျင်မှု အဆင့်:** မူရင်းအခြေခံ အက်ပလီကေးရှင်းများကို ကိုယ်ပိုင်အတိုင်း deploy လုပ်နိုင်သည်

**✅ အောင်မြင်မှု စစ်ဆေးခြင်း:**
```bash
# အခန်း ၁ ကို ပြီးမြောက်သည့်အခါ သင်သည် အောက်ပါအရာများကို ပြုလုပ်နိုင်ရမည်။
azd version              # တပ်ဆင်ထားသော ဗားရှင်းကို ပြသည်
azd init --template todo-nodejs-mongo  # ပရောဂျက်ကို စတင်တည်ဆောက်သည်
azd up                  # Azure သို့ တင်ပို့သည်
azd show                # လည်ပတ်နေသော အက်ပ်၏ URL ကို ပြသည်
# အက်ပ်ကို ဘရောက်ဇာတွင် ဖွင့်၍ မှန်ကန်စွာ လုပ်ဆောင်သည်
azd down --force --purge  # ရင်းမြစ်များကို ရှင်းလင်းသည်
```

**📊 စုစုပေါင်း အချိန်မြှုပ်နှံမှု:** 30-45 မိနစ်  
**📈 သင်ယူပြီးနောက် ကျွမ်းကျင်မှု အဆင့်:** မူရင်းအခြေခံ အက်ပလီကေးရှင်းများကို ကိုယ်ပိုင်အတိုင်း deploy လုပ်နိုင်သည်

#### 💰 AI Deployments များအတွက် ကုန်ကျစရိတ် စဉ်းစားချက်များ

**ဖွံ့ဖြိုးရေး ပတ်ဝန်းကျင် (အကန့်အသတ် $80-150/လ အနား):**
- Azure OpenAI (Pay-as-you-go): $0-50/လ (token အသုံးအချိန်ပေါ်မူတည်၍)
- AI Search (အခြေခံ အဆင့်): $75/လ
- Container Apps (Consumption): $0-20/လ
- Storage (စံနှုန်း): $1-5/လ

**ထုတ်လုပ်မှု ပတ်ဝန်းကျင် (အကန့်အသတ် $300-3,500+/လ):**
- Azure OpenAI (PTU အတွက် တည်ငြိမ်သော စွမ်းဆောင်မှု): $3,000+/လ သို့မဟုတ် အမြင့်အသုံးအဆင့်ရှိ Pay-as-you-go

- Container Apps (ရ اختصاص): $50-100/လ
- Application Insights: $5-50/လ
- Storage (ပရီမြီယံ): $10-50/လ

**💡 ကုန်ကျစရိတ် အမြှုပ်ကောင်းခြင်း အကြံပေးချက်များ:**
- သင်ယူရန် Azure OpenAI **Free Tier** ကိုအသုံးပြုပါ (တစ်လလျှင် 50,000 tokens အပါရှိ)
- တိုးတက်စီမံနေစဉ် မဟုတ်စဉ်တွင် အရင်းအမြစ်များကို လွှတ်နေစေဖို့ `azd down` လုပ်ပါ
- စမችမှုအတွက် consumption-based billing ဖြင့် စတင်ပါ၊ ထုတ်လုပ်မှုအတွက်သာ PTU သို့ ညှိပါ
- Deploy မလုပ်ခင် `azd provision --preview` ဖြင့် ကုန်ကျစရိတ် ခန့်မှန်းပါ
- auto-scaling ကို အသုံးပြုပါ: အသုံးပြုမှုသာ အတွက်ပဲ ပေးချေပါ

```bash
azd provision --preview

# Azure Portal တွင် လက်ရှိကုန်ကျစရိတ်များကို စောင့်ကြည့်ပါ
```

## 📖 Azure Developer CLI (azd) ဆိုတာ ဘာလဲ?

Azure Developer CLI (azd) သည် အထူးသဖြင့် ဖွံ့ဖြိုးရေးသူများအတွက် တည်ဆောက်ထားသော command-line အင်တာဖေ့စ်တစ်ခုဖြစ်ပြီး Azure သို့ အပလီကေးရှင်းများကို တည်ဆောက်ခြင်းနှင့် တပ်ဆင်ခြင်းလုပ်ငန်းစဉ်များကို အရှိန်မြှင့်ပေးသည်။ ၎င်းသည် အောက်ပါအရာများကို ပေးဆောင်ပါသည်။

- **Template-based deployments** - ပုံစံပြုထားသော template များကို အသုံးပြု၍ ပုံမှန် အပလီကေးရှင်းပုံစံများကို တပ်ဆင်နိုင်သည်
- **Infrastructure as Code** - Bicep သို့မဟုတ် Terraform အသုံးပြု၍ Azure အရင်းအမြစ်များကို စီမံနိုင်သည်  
- **Integrated workflows** - ပရိုဗင်းရှင်း၊ တပ်ဆင်မှုနှင့် ကြည့်ရှုမှုများကို တစ်ပြိုင်နက် ဆက်တိုက်ပြုလုပ်နိုင်သည်
- **Developer-friendly** - ဖွံ့ဖြိုးရေးသူများ၏ ထုတ်လုပ်မှုနှင့် အသုံးပြုမှု အတွေ့အကြုံကို အာရုံစိုက် အကောင်းမြင့်စေသည်

**AZD ကို AI ဖြေရှင်းချက်များအတွက် ရွေးချယ်ရန် အကြောင်းရင်းများ** AZD သည် AI ဖွံ့ဖြိုးရေးသူများ မျက်နှာမူသော အဓိက စိန်ခေါ်မှုများကို ကိုင်တွယ်ပေးသည်။

- **လုံခြုံစိတ်ချရသော AI တင်သွင်းမှုများ** - AI ဝန်ဆောင်မှုများ၊ API key များနှင့် မော်ဒယ် endpoint များအတွက် အပေါင်းသင်းလုံခြုံရေး မော်ဒယ်များပါဝင်သည်  
- **ထုတ်လုပ်မှု AI ပုံစံများ** - အရွယ်အစားတိုးနိုင်သည့်၊ ကုန်ကျစရိတ်ထိရောက်သော AI အပလီကေးရှင်း တင်သွင်းခြင်းအတွက် အကောင်းဆုံး လေ့လာချက်များ
- **အစမှ အဆုံး အစီအစဉ် AI Workflow များ** - မော်ဒယ်ဖွံ့ဖြိုးရေးမှ ထုတ်လုပ်မှုတင်သွင်းမှုအထိ ကြည့်ရှုမှုနှင့်တကွ
- **ကုန်ကျစရိတ် ထိရောက်မှု** - AI အလုပ်တင်များအတွက် အရင်းအမြစ် ချမှတ်ခြင်းနှင့် အရွယ်အစားချိန်ညှိမူ မဟာဗျူဟာများ
- **Microsoft Foundry ပေါင်းစည်းမှု** - Microsoft Foundry မော်ဒယ် ကဒလော့ဂ်နှင့် endpoint များနှင့် အလွယ်တကူ ချိတ်ဆက်နိုင်ခြင်း

## 🔧 Quick Troubleshooting Guide

**စတင်သူများ မကြုံရသော ပြဿနာများနှင့် ချက်ချင်း ဖြေရှင်းနည်းများ။**

### ❌ "azd: command not found"

```bash
# အရင်ဆုံး AZD ကို တပ်ဆင်ပါ
# Windows (PowerShell):
winget install microsoft.azd

# macOS:
brew tap azure/azd && brew install azd

# Linux:
curl -fsSL https://aka.ms/install-azd.sh | bash

# တပ်ဆင်မှုကို စစ်ဆေးပါ
azd version
```

### ❌ "No subscription found" or "Subscription not set"

```bash
# ရရှိနိုင်သည့် subscription များကို စာရင်းပြပါ
az account list --output table

# ပုံမှန် subscription ကို သတ်မှတ်ပါ
az account set --subscription "<subscription-id-or-name>"

# AZD ပတ်ဝန်းကျင်အတွက် သတ်မှတ်ပါ
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# အတည်ပြုပါ
az account show
```

### ❌ "InsufficientQuota" or "Quota exceeded"

```bash
# ခြားသော Azure ဒေသကို စမ်းကြည့်ပါ
azd env set AZURE_LOCATION "westus2"
azd up

# သို့မဟုတ် ဖွံ့ဖြိုးရေးတွင် ပိုသေးငယ်သော SKU များကို အသုံးပြုပါ
# infra/main.parameters.json ကို တည်းဖြတ်ပါ:
{
  "sku": "B1"  // Instead of "P1V2"
}
```

### ❌ "azd up" fails halfway through

```bash
# ရွေးချယ်မှု ၁: သန့်ရှင်းပြီး ထပ်စမ်းကြည့်ပါ
azd down --force --purge
azd up

# ရွေးချယ်မှု ၂: အခြေခံအဆောက်အအုံကိုသာ ပြင်ဆင်ပါ
azd provision

# ရွေးချယ်မှု ၃: အသေးစိတ် မှတ်တမ်းများကို စစ်ဆေးပါ
azd show
azd logs
```

### ❌ "Authentication failed" or "Token expired"

```bash
# ထပ်မံ အတည်ပြုပါ
az logout
az login

azd auth logout
azd auth login

# အတည်ပြုမှုကို စစ်ဆေးပါ
az account show
```

### ❌ "Resource already exists" or naming conflicts

```bash
# AZD သည် ထူးခြားသော အမည်များ ဖန်တီးပေးသည်၊ သို့သော် အမည်တူ ဖြစ်လာပါက:
azd down --force --purge

# ထပ်မံ လုပ်ဆောင်ရန် သန့်ရှင်းသော ပတ်ဝန်းကျင် အသစ်ဖြင့် ပြန်ကြိုးစားပါ
azd env new dev-v2
azd up
```

### ❌ Template deployment taking too long

**ပုံမှန် စောင့်ရမည့် အချိန်များ:**
- ရိုးရှင်းသော ဝက်ဘ် အက်ပ်: 5-10 မိနစ်
- ဒေတာဘေ့(စ်) ပါရှိသော အက်ပ်: 10-15 မိနစ်
- AI လျှောက်လွှာများ: 15-25 မိနစ် (OpenAI ပစ္စည်းထည့်သွင်းခြင်း များသည် နှေးကွေးသည်)

```bash
# တိုးတက်မှုကို စစ်ဆေးပါ
azd show
```

## 🔧 Quick Troubleshooting Guide

**စတင်သူများ မကြုံရသော ပြဿနာများနှင့် ချက်ချင်း ဖြေရှင်းနည်းများ။**

<details>
<summary><strong>❌ "azd: command not found"</strong></summary>

```bash
# ပထမဦးစွာ AZD ကို တပ်ဆင်ပါ
# Windows (PowerShell):
winget install microsoft.azd

# macOS:
brew tap azure/azd && brew install azd

# Linux:
curl -fsSL https://aka.ms/install-azd.sh | bash

# တပ်ဆင်မှုကို အတည်ပြုပါ
azd version
```
</details>

<details>
<summary><strong>❌ "No subscription found" or "Subscription not set"</strong></summary>

```bash
# ရရှိနိုင်သည့် စာရင်းသွင်းမှုများကို ပြပါ
az account list --output table

# ပုံမှန် စာရင်းသွင်းမှုကို သတ်မှတ်ပါ
az account set --subscription "<subscription-id-or-name>"

# AZD ပတ်ဝန်းကျင်အတွက် သတ်မှတ်ပါ
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# အတည်ပြုပါ
az account show
```
</details>

<details>
<summary><strong>❌ "InsufficientQuota" or "Quota exceeded"</strong></summary>

```bash
# အခြား Azure ဒေသကို စမ်းကြည့်ပါ
azd env set AZURE_LOCATION "westus2"
azd up

# သို့မဟုတ် ဖွံ့ဖြိုးရေးတွင် ပိုသေးငယ်သော SKU များကို အသုံးပြုပါ
# infra/main.parameters.json ကို ပြင်ဆင်ပါ:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>

<details>
<summary><strong>❌ "azd up" fails halfway through</strong></summary>

```bash
# ရွေးချယ်မှု ၁: သန့်ရှင်းပြီး ထပ်မံကြိုးစားပါ
azd down --force --purge
azd up

# ရွေးချယ်မှု ၂: အောက်ခံအဆောက်အအုံကိုသာ ပြင်ဆင်ပါ
azd provision

# ရွေးချယ်မှု ၃: အသေးစိတ် မှတ်တမ်းများကို စစ်ဆေးပါ
azd show
azd logs
```
</details>

<details>
<summary><strong>❌ "Authentication failed" or "Token expired"</strong></summary>

```bash
# ပြန်လည်အတည်ပြု
az logout
az login

azd auth logout
azd auth login

# အတည်ပြုမှုကို စစ်ဆေး
az account show
```
</details>

<details>
<summary><strong>❌ "Resource already exists" or naming conflicts</strong></summary>

```bash
# AZD သည် ထူးခြားသော နာမည်များ ဖန်တီးပေးသည်၊ သို့သော် နာမည်တူမှု ဖြစ်ပါက:
azd down --force --purge

# ထိုနောက် အသစ်သော ပတ်ဝန်းကျင်ဖြင့် ထပ်မံကြိုးစားပါ
azd env new dev-v2
azd up
```
</details>

<details>
<summary><strong>❌ Template deployment taking too long</strong></summary>

**ပုံမှန် စောင့်ရမည့် အချိန်များ:**
- ရိုးရှင်းသော ဝက်ဘ် အက်ပ်: 5-10 မိနစ်
- ဒေတာဘေ့(စ်) ပါရှိသော အက်ပ်: 10-15 မိနစ်
- AI လျှောက်လွှာများ: 15-25 မိနစ် (OpenAI ပစ္စည်းထည့်သွင်းခြင်း များသည် နှေးကွေးသည်)

```bash
# တိုးတက်မှုကို စစ်ဆေးပါ
azd show

# ၃၀ မိနစ်ထက်ပို အတားအဆီးရှိနေပါက Azure Portal ကို စစ်ဆေးပါ:
azd monitor
# မအောင်မြင်ခဲ့သည့် တပ်ဆင်မှုများကို ရှာဖွေပါ
```
</details>
