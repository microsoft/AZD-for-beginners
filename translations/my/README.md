## သင်တန်းအနှစ်ချုပ်

အဆင့်ချင်းစီ များဖြင့် တိကျစွာ စီစဉ်ထားသော အခန်းများဖြင့် Azure Developer CLI (azd) ကို ကျွမ်းကျင်ပါ။ **Microsoft Foundry ပေါင်းစည်းမှုဖြင့် AI အက်ပလီကေးရှင်း ထုတ်ပေးမှုအပေါ် အထူးအာရုံစိုက်ထားသည်။**

### ယနေ့ခေတ် ဖွံ့ဖြိုးရေးသူများအတွက် ဤသင်တန်း အရေးကြီးသည့် အကြောင်းရင်း

Microsoft Foundry Discord အသိုင်းအဝိုင်းမှ ရရှိသည့် မွေ့လျော်ချက်များအရ၊ **လူကြီးမင်းတို့ထဲမှ 45% ရှိသော ဖွံ့ဖြိုးရေးသူများသည် AZD ကို AI အလုပ်ခွင်များအတွက် အသုံးပြုလိုကြသည်** သို့သော် အောက်ပါ စိန်ခေါ်မှုများနှင့် တွေ့ကြုံနေသည် -
- အဆင့်များစုံ multi-service AI ဖွဲ့စည်းမှုများ၏ ကျယ်ပြန့်စွာရှိခြင်း
- ထုတ်လုပ်မှု AI ထောက်ပံ့မှု စနစ်များအတွက် အကောင်းဆုံးလေ့လာနည်းလမ်းများ
- Azure AI ဝန်ဆောင်မှုများနှင့် ပေါင်းစည်းမှုနှင့် ဖွင့်ချိန်းချက်များ
- AI အလုပ်ခွင်များအတွက် ကုန်ကျစရိတ် အလိုက်အမြန် ဂရုစိုက်ခြင်း
- AI အထူး deployment ပြဿနာများကို ဖြေရှင်းခြင်း

### သင်ယူရမည့် ရည်မှန်းချက်များ

ဤစနစ်တကျ သင်တန်းကို ပြီးမြောက်စေပြီးနောက်၊ သင်သည် အောက်ပါများကို လုပ်နိုင်မည်။
- **AZD အခြေခံများ ကို ကျွမ်းကျင်ပါမည်**: အဓိက အယူအဆများ၊ 설치ခြင်းနှင့် ဖော်ပြချက်များ
- **AI အက်ပလီကေးရှင်းများ ထုတ်ပေးပါမည်**: Microsoft Foundry ဝန်ဆောင်မှုများနှင့် AZD ကို အသုံးပြု၍
- **Infrastructure as Code ကို အသုံးပြုပါမည်**: Bicep templates ဖြင့် Azure အရင်းအမြစ်များကို စီမံခန့်ခွဲခြင်း
- **Deployment များကို စစ်ဆေးဖြေရှင်းပါမည်**: ပုံမှန် ပြဿနာများကို ဖြေရှင်းခြင်းနှင့် ဒေဘတ်ဂ်လုပ်ခြင်း
- **ထုတ်လုပ်မှုအတွက် အကောင်းဆုံးအခြေအနေပြုလုပ်ပါမည်**: လုံခြုံရေး၊ အတိုင်းအတာချိန်ညှိခြင်း၊ မော်နစ်တာနှင့် ကုန်ကျစရိတ် စီမံခန့်ခွဲခြင်း
- **Multi-Agent ဖြေရှင်းချက်များ တည်ဆောက်ပါမည်**: ရှုပ်ထွေးသော AI ဖွဲ့စည်းမှုများ ထုတ်ပေးခြင်း

## 📚 သင်ယူရန် အခန်းများ

*သင့် အတွေ့အကြုံနှင့် ရည်မှန်းချက်များအရ သင်ယူလမ်းကြောင်းကို ရွေးချယ်ပါ*

### 🚀 အခန်း ၁: အခြေခံနှင့် အလျင်အမြန် စတင်ခြင်း
**လိုအပ်ချက်များ**: Azure စာရင်းသုံးမှု၊ ကွန်မန်းဒ်လိုင်း အခြေခံ ဗဟုသုတ  
**ကြာချိန်**: 30-45 မိနစ်  
**ရှုပ်ထွေးမှု**: ⭐

#### သင်ယူမည့် အရာများ
- Azure Developer CLI အခြေခံ အယူအဆများကို နားလည်ခြင်း
- သင့် ပလက်ဖောင်းပေါ်တွင် AZD ကို 설치ခြင်း
- ပထမဆုံး အောင်မြင်သော deployment ကို ဆောင်ရွက်ခြင်း

#### သင်ယူရန် အရင်းအမြစ်များ
- **🎯 ဒီနေရာကနေ စတင်ပါ**: [Azure Developer CLI ဆိုတာ ဘာလဲ?](../..)
- **📖 သီအိုရီ**: [AZD အခြေခံများ](docs/getting-started/azd-basics.md) - အဓိက အယူအဆများနှင့် ဝေါဟာရများ
- **⚙️ အဆင်သင့်ပြင်ဆင်မှု**: [Installation & Setup](docs/getting-started/installation.md) - ပလက်ဖောင်းအဆင်သင့်လမ်းညွှန်များ
- **🛠️ လက်တွေ့ လက်ခံလုပ်ဆောင်ရေး**: [သင့် ပထမဆုံး ပရောဂျက်](docs/getting-started/first-project.md) - တစ်ဆင့်ချင်း ဆောင်ရွက်နည်းလမ်းညွှန်
- **📋 အလျင်အမြန် ကိုးကားစာရွက်**: [Command Cheat Sheet](resources/cheat-sheet.md)

#### လက်တွေ့လေ့ကျင့်ခန်း
```bash
# မြန်စွာ တပ်ဆင်မှု စစ်ဆေးခြင်း
azd version

# ပထမဆုံး အက်ပလီကေးရှင်းကို ဖြန့်ချိပါ
azd init --template todo-nodejs-mongo
azd up
```

**💡 အခန်းရလဒ်**: AZD ကို အသုံးပြု၍ Azure သို့ ရိုးရှင်းသော ဝဘ် အက်ပလီကေးရှင်း တစ်ခုကို အောင်မြင်စွာ ထုတ်ပေးနိုင်ခြင်း

**✅ အောင်မြင်မှု စစ်ဆေးခြင်း:**
```bash
# အခန်း ၁ ကို ပြီးမြောက်ပြီးနောက်၊ သင်သည် အောက်ပါလုပ်ဆောင်ချက်များကို ပြုလုပ်နိုင်သင့်သည်။
azd version              # တပ်ဆင်ထားသော ဗားရှင်းကို ပြသသည်
azd init --template todo-nodejs-mongo  # ပရောဂျက်ကို စတင်ပြုလုပ်သည်
azd up                  # Azure သို့ ဖြန့်ချိသည်
azd show                # ပြေးနေသည့် အက်ပ်၏ URL ကို ပြသသည်
# အက်ပ်ကို ဘရောက်ဇာတွင် ဖွင့်၍ လည်ပတ်သည်
azd down --force --purge  # အရင်းအမြစ်များကို ရှင်းလင်းသည်
```

**📊 အချိန်မိနစ်ထည့်သွင်းမှု:** 30-45 မိနစ်  
**📈 သင်ယူပြီးနောက် ကျွမ်းကျင်မှုပမာဏ:** အခြေခံ အက်ပလီကေးရှင်းများကို ကိုယ်တိုင် ထုတ်ပေးနိုင်ပါသည်

**✅ အောင်မြင်မှု စစ်ဆေးခြင်း:**
```bash
# အခန်း ၁ ကို ပြီးမြောက်လျင် သင် အောက်ပါအရာများကို လုပ်ဆောင်နိုင်ရမည်
azd version              # တပ်ဆင်ထားသော ဗားရှင်းကို ပြသသည်
azd init --template todo-nodejs-mongo  # ပရောဂျက်ကို စတင်ပြင်ဆင်သည်
azd up                  # Azure သို့ တင်ပို့သည်
azd show                # လည်ပတ်နေသည့် အက်ပ်၏ URL ကို ပြသသည်
# အပလီကေးရှင်းကို ဘရောက်ဇာတွင် ဖွင့်လှစ်၍ မှန်ကန်စွာ လည်ပတ်သည်
azd down --force --purge  # အရင်းအမြစ်များကို ရှင်းလင်းသည်
```

**📊 အချိန်မိနစ်ထည့်သွင်းမှု:** 30-45 မိနစ်  
**📈 သင်ယူပြီးနောက် ကျွမ်းကျင်မှုပမာဏ:** အခြေခံ အက်ပလီကေးရှင်းများကို ကိုယ်တိုင် ထုတ်ပေးနိုင်ပါသည်

**📈 သင်ယူပြီးနောက် ကျွမ်းကျင်မှုပမာဏ:** ထုတ်လုပ်မှု-အသင့် AI အက်ပလီကေးရှင်းများကို ထုတ်ပေး၍ ဖော်ပြနိုင်ပါသည်  
**💰 ကုန်ကျစရိတ် အသိပညာ:** တိုးတက်ရေးအတွက် $80-150/လ၊ ထုတ်လုပ်မှုအတွက် $300-3500/လ အလားအလာကို နားလည်ပါ

#### 💰 AI ထုတ်ပေးမှုများအတွက် ကုန်ကျစရိတ် စဉ်းစားရန်

**ဖွံ့ဖြိုးရေး ပတ်ဝန်းကျင် (ခန့်မှန်း $80-150/လ):**
- Azure OpenAI (Pay-as-you-go): $0-50/လ (token အသုံးပြုမှုအပေါ် မူတည်)
- AI Search (Basic tier): $75/လ
- Container Apps (Consumption): $0-20/လ
- Storage (Standard): $1-5/လ

**ထုတ်လုပ်မှု ပတ်ဝန်းကျင် (ခန့်မှန်း $300-3,500+/လ):**
- Azure OpenAI (PTU အတွက် ဆက်လက်တည်ငြိမ်သော အဖွဲ့အစည်း): $3,000+/လ OR အများရှိသော အသုံးအများကြောင့် Pay-as-go
- AI Search (Standard tier): $250/လ
- Container Apps (Dedicated): $50-100/လ
- Application Insights: $5-50/လ
- Storage (Premium): $10-50/လ

**💡 ကုန်ကျစရိတ် အကောင်းမြှင့်သတ်မှတ်ချက်များ:**
- သင်ယူရန် Azure OpenAI **Free Tier** ကို အသုံးပြုပါ (လစဉ် 50,000 tokens ပါဝင်သည်)
- ဖွံ့ဖြိုးရေးလုပ်နေစဉ် မဟုတ်သောအချိန်တွင် `azd down` ကို 실행၍ အရင်းအမြစ်များကို ပိတ်ထားပါ
- စတင်သည့်အချိန်တွင် consumption-based billing ဖြင့်စတင်ပြီး ထုတ်လုပ်မှုအတွက်သာ PTU သို့ မြှင့်တင်ပါ
- deployment ပြုလုပ်မည့် အခါ `azd provision --preview` ဖြင့် ကုန်ကျစရိတ် ခန့်မှန်းချက် ရယူပါ
- အလိုအလျောက် auto-scaling ကို ဖွင့်ထားပါ: အသုံးအနှုန်းအရသာသာ ပေးဆောင်ပါ

**ကုန်ကျစရိတ် ကြည့်ရှုခြင်း:**

# ခန့်မှန်းထားသော လစဉ်ကုန်ကျစရိတ်များကို စစ်ဆေးပါ
azd provision --preview

# Azure Portal တွင် လက်တွေ့ကုန်ကျစရိတ်များကို စောင့်ကြည့်ပါ

## 📖 What is Azure Developer CLI?

Azure Developer CLI (azd) သည် Developer များအခြေစိုက် အတိအကျ command-line interface တစ်ခု ဖြစ်ပြီး Azure သို့ အပလီကေးရှင်းများ တည်ဆောက်ခြင်းနှင့် တပ်ဆင်ခြင်း လုပ်ငန်းစဉ်ကို မြန်ဆန်စေသည်။ ၎င်းသည် အောက်ပါအရာများကို ပေးသည် -

- **Template-based deployments** - မကြာခဏ အသုံးပြုသော အပလီကေးရှင်း ပုံစံများအတွက် ကြိုတင်ပြင်ဆင်ထားသော နမူနာများကို အသုံးပြုပါ
- **Infrastructure as Code** - Bicep သို့မဟုတ် Terraform ကို အသုံးပြု၍ Azure အရင်းအမြစ်များကို ကုဒ်ဖြင့် စီမံနိုင်သည်  
- **Integrated workflows** - မျဉ်းစဉ်တိုင်း ပေါင်းစည်း၍ provisioning, deployment, နှင့် monitoring ကို ဆက်လက်ထိန်းချုပ်နိုင်သည်
- **Developer-friendly** - Developer များ၏ ထုတ်လုပ်နိုင်မှုနှင့် အသုံးပြုသူ အ अनुभव ကို တိုးတက်စေသည်

### **AZD + Microsoft Foundry: Perfect for AI Deployments**

**Why AZD for AI Solutions?** AZD သည် AI developer များ ကြုံတွေ့ရသော အဓိက စိန်ခေါ်မှုများကို ဖြေရှင်းပေးသည် -

- **Secure AI Deployments** - AI ဝန်ဆောင်မှုများ၊ API keys နှင့် model endpoints များအတွက် သွေးစည်း အလုံခြုံရေး ပုံစံများ ပါဝင်သည်

- **End-to-End AI Workflows** - မော်ဒယ် ဖွံ့ဖြိုးမှုမှ ထုတ်လုပ်မှု အထိ အပြီးအစီး အလုပ်စဉ်များနှင့် သင့်တော်သော စောင့်ကြည့်မှုများ
- **Cost Optimization** - AI အလုပ်များအတွက် အရင်းအမြစ် ချမှတ်ခြင်းနှင့် အလိုအလျောက်ချဲ့ထွင်မှု မဟာဗျူဟာများ

## 🔧 အမြန် ဖြေရှင်းနည်းလမ်းညွှန်

**စတင်သင်ယူသူများ ကြုံတွေ့ရသည့် ပုံမှန် ပြဿနာများနှင့် מיד အဖြေများ:**

### ❌ "azd: command not found"

```bash
# အရင်ဆုံး AZD ကို ထည့်သွင်းပါ
# Windows (PowerShell):
winget install microsoft.azd

# macOS:
brew tap azure/azd && brew install azd

# Linux:
curl -fsSL https://aka.ms/install-azd.sh | bash

# ထည့်သွင်းမှုကို စစ်ဆေးပါ
azd version
```

### ❌ "No subscription found" or "Subscription not set"

```bash
# ရရှိနိုင်သော subscription များကို စာရင်းပြပါ
az account list --output table

# ပုံမှန် subscription ကို သတ်မှတ်ပါ
az account set --subscription "<subscription-id-or-name>"

# AZD ပတ်ဝန်းကျင် အတွက် သတ်မှတ်ပါ
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# အတည်ပြုပါ
az account show
```

### ❌ "InsufficientQuota" or "Quota exceeded"

```bash
# အခြား Azure ဒေသကို စမ်းကြည့်ပါ
azd env set AZURE_LOCATION "westus2"
azd up

# သို့မဟုတ် ဖွံ့ဖြိုးရေးတွင် သေးငယ်သော SKU များကို အသုံးပြုပါ
# infra/main.parameters.json ကို တည်းဖြတ်ပါ:
{
  "sku": "B1"  // Instead of "P1V2"
}
```

### ❌ "azd up" fails halfway through

```bash
# ရွေးချယ်မှု 1: ရှင်းလင်းပြီး ထပ်မံကြိုးစားပါ
azd down --force --purge
azd up

# ရွေးချယ်မှု 2: အခြေခံအဆောက်အအုံကိုသာ ပြုပြင်ပါ
azd provision

# ရွေးချယ်မှု 3: အသေးစိတ်မှတ်တမ်းများကို စစ်ဆေးပါ
azd show
azd logs
```

### ❌ "Authentication failed" or "Token expired"

```bash
# ပြန်လည်အတည်ပြုပါ
az logout
az login

azd auth logout
azd auth login

# အတည်ပြုမှုကို စစ်ဆေးပါ
az account show
```

### ❌ "Resource already exists" or naming conflicts

```bash
# AZD သည် ထူးခြားသော အမည်များ ဖန်တီးပေးသော်လည်း ပဋိပက္ခ ဖြစ်ပါက:
azd down --force --purge

# ထို့နောက် အသစ်သော ပတ်ဝန်းကျင်ဖြင့် ထပ်မံကြိုးစားပါ
azd env new dev-v2
azd up
```

### ❌ Template deployment taking too long

**ပုံမှန် စောင့်ဆိုင်းချိန်များ:**
- ရိုးရှင်းသော ဝက်ဘ်အက်ပ်: ၅-၁၀ မိနစ်
- ဒေတာဘေ့စ်ပါ အက်ပ်: ၁၀-၁၅ မိနစ်
- AI အက်ပ်များ: ၁၅-၂၅ မိနစ် (OpenAI provisioning သည် နှာရှည်တတ်သည်)

```bash
# တိုးတက်မှုကို စစ်ဆေးပါ
azd show

# ၃၀ မိနစ်ထက်ပို ရပ်နေပါက၊ Azure Portal ကို စစ်ဆေးပါ:
azd monitor
# မအောင်မြင်သော တပ်ဆင်မှုများကို ရှာဖွေပါ
```

### ❌ "Permission denied" or "Forbidden"

```bash
# သင့် Azure အခန်းကဏ္ဍကို စစ်ဆေးပါ
az role assignment list --assignee $(az account show --query user.name -o tsv)

# အနည်းဆုံး "Contributor" အခန်းကဏ္ဍ လိုအပ်သည်
# သင့် Azure အက်မင်ထံ အောက်ပါ ခွင့်ပြုချက်များ ပေးရန် တောင်းဆိုပါ:
# - Contributor (ရင်းမြစ်များအတွက်)
# - User Access Administrator (အခန်းကဏ္ဍ ခန့်အပ်ရေးအတွက်)
```

### ❌ Can't find deployed application URL

```bash
# ဆာဗစ် endpoints အားလုံးကို ပြပါ
azd show

# သို့မဟုတ် Azure Portal ကို ဖွင့်ပါ
azd monitor

# တိကျသော ဆာဗစ်ကို စစ်ဆေးပါ
azd env get-values
# *_URL ပြောင်းလဲနိုင်သော တန်ဖိုးများကို ရှာပါ
```
