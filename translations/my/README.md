## 🆕 azd တွင် ယနေ့ ဘာအသစ်တွေရှိနေလဲ

Azure Developer CLI သည် ရိုးရိုး web apps နှင့် APIs ထက် အကျယ်ပြန့်၍ ဖွံ့ဖြိုးလာပါပြီ။ ယခုအချိန်တွင် azd သည် Azure သို့ အတိုင်းအတာမရွေး ချိန်ပေးရန် အသုံးပြုနိုင်သည့် တစ်ခုတည်းသော ကိရိယာဖြစ်ပါသည် — AI ခံနိုင်ရည်ရှိ အက်ပ်များနှင့် ဉာဏ်ရည် agent များကိုပါ ထည့်သွင်းပါသည်။

ဒါက သင့်အတွက် အဓိပ္ပာယ်ရသည်မှာ -

- **AI agent များသည် ယခု azd တွင် အဓိက အလုပ်အမျိုးအစားတစ်ခုဖြစ်လာပါသည်။** သင်သည် ရိုးရိုး `azd init` → `azd up` workflow ကို အသုံးပြု၍ AI agent project များကို initialize, deploy နှင့် မန်နေဂျ်နိုင်သည်။
- **Microsoft Foundry ပေါင်းစည်းခြင်း** သည် မော်ဒယ် တင်သွင်းခြင်း၊ agent ကို ဟိုစ့် ပြုလုပ်ခြင်းနှင့် AI ဝန်ဆောင်မှု ဖော်ပြချက်များကို azd template ပတ်ဝန်းကျင်ထဲသို့ တိုက်ရိုက် ဖွဲ့စည်းပေးသည်။
- **အဓိက workflow က မပြောင်းဘူး။** todo app တစ်ခု၊ microservice တစ်ခု သို့မဟုတ် multi-agent AI ဖြေရှင်းချက် တစ်ခုကို တင်သွင်းနေစဉ်၌ command များသည် အတိုင်းတူပဲ ဖြစ်ပါသည်။

မူလက azd ကို အသုံးပြုဖူးသူများအတွက် AI ပံ့ပိုးမှုသည် သဘာဝကျသော တိုးချဲ့မှုတစ်ခုပဲဖြစ်ပါသည် — အခြားကိရိယာသစ်တစ်ခု သို့မဟုတ် ကြီးကျယ်သော လမ်းကြောင်းတစ်ခုမဟုတ်ပါ။ အသစ်စပြောမည်ဆိုလျှင်၊ အရာရာအတွက် အလုပ်လုပ်နိုင်သော workflow တစ်ခုကို သင်လေ့လာမည်ဖြစ်သည်။

## 🚀 Azure Developer CLI (azd) ဆိုတာဘာလဲ?

**Azure Developer CLI (azd)** သည် developer များအတွက် မိတ်ဖက် CLI ကိရိယာတစ်ခုဖြစ်ပြီး Azure သို့ အက်ပ်များကို တင်သွင်းခြင်းကို လွယ်ကူစေသည်။ Azure အရင်းအမြစ်များစွာကို လက်ဖြင့် ဖန်တီးချိတ်ဆက်ရန်မလိုဘဲ၊ သင်သည် တစ်ခုတည်းသော command ဖြင့် အက်ပ်တစ်ခုလုံးကို တင်သွင်းနိုင်သည်။

### `azd up` ၏ စွမ်းဆောင်ရည်

```bash
# ဒီတစ်ခုတည်းသော command က အားလုံးကိုလုပ်ဆောင်ပေးပါတယ်။
# ✅ Azure resource များအားလုံးဖန်တီးသည်
# ✅ ကွန်ရက်နှင့် လုံခြုံရေးကို ဖန်တီးသည်
# ✅ သင့်အက်ပ်လီကေးရှင်းကုဒ်ကို တည်ဆောက်သည်
# ✅ Azure သို့ချထားသည်
# ✅ လုပ်ဆောင်နေသော URL ကို ပေးသည်
azd up
```

**အဲ့တာပဲ!** Azure Portal တွင် မနှိပ်ရ၊ ရိုးရှင်းမဲ့ ARM template များကို အရင်လေ့လာရန် မလို၊ လက်ဖြင့် ဆက်တင်တစ်ခုစီ မလုပ်နိုင်ပါ— အကျိုးသက်ရောက်သော အက်ပ်များ သာ Azure ပေါ်တွင် လက်တွေ့အသုံးပြုနိုင်ပါတယ်။

## ❓ Azure Developer CLI နှင့် Azure CLI: ကွာခြားချက်ကဘာလဲ?

ဒီဟာက စတင်သင်ယူသူများ မေးမြန်းလေ့ရှိတဲ့ ထိပ်တန်းမေးခွန်းပါ။ ရိုးရှင်းတဲ့ ဖြေကြားချက်က ေအာ:

| အင်္ဂါရပ် | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **ရည်ရွယ်ချက်** | Azure ရင်းမြစ်များကို တစ်ခုချင်း စီ စီမံရန် | အပြည့်အစုံ အက်ပ်များကို တင်သွင်းရန် |
| **စဉ်းစားမှုပုံစံ** | အဆောက်အအုံကို ဦးတည်သည် | အက်ပ်ကို ဦးတည်သည် |
| **ဥပမာ** | `az webapp create --name myapp...` | `azd up` |
| **သင်ယူခြင်းအဆင့်** | Azure ဝန်ဆောင်မှုများကို သိရှိထားရမည် | သင့်အက်ပ်ကိုသာ သိထားရမည် |
| **သင့်စာလုံး** | DevOps, အင်ဖရာ | ဖန်တီးသူများ, ပရိုတိုတိုက်ပညာရှင်များ |

### ရိုးရှင်းသော နှိုင်းယှဉ်ချက်

- **Azure CLI** သည် အိမ်တစ်လုံးကို ဆောက်ရန် လိုအပ်သော ကိရိယာအားလုံးကို ပိုင်ဆိုင်ထားသလို ဖြစ်သည် — စက်ကိုက်များ၊ သားသီးများ၊ နက်များ။ မည်သည့်အရာမဆို ဆောက်နိုင်သော်လည်း ဆောက်လုပ်ပညာကို သိရှိရမည်။
- **Azure Developer CLI** သည် တာဝန်ရောက်ဆောင်သူတစ်ဦးကို ငှားထားသလို ဖြစ်သည် — သင်လိုချင်သည့်အရာကို ဖော်ပြပေးမယ်၊ ဆောက်လုပ်ရေးကို သူတို့ ကျွမ်းကျင်စွာ ကိုင်တွယ်ပေးမည်။

### ဘယ်အချိန်တွင် ဘယ်တစ်ခုကို အသုံးပြုမလဲ

| ပြသနာအခြေအနေ | သုံးရန် |
|----------|----------|
| "ကျွန်တော့် web app ကို အမြန်တင်ချင်ပါတယ်" | `azd up` |
| "ကျွန်တော် storage account တစ်ခုပဲ ဖန်တီးချင်ပါတယ်" | `az storage account create` |
| "ကျွန်တော် AI အက်ပ် တစ်ခု ဆောက်နေပါတယ်" | `azd init --template azure-search-openai-demo` |
| "ကျွန်တော် Azure ရင်းမြစ်တစ်ခုကို debug လုပ်ချင်ပါတယ်" | `az resource show` |
| "မိနစ်ပိုင်းအတွင်း production-ready deployment လိုချင်ပါတယ်" | `azd up --environment production` |

### အတူတကွ သုံးနိုင်သည်!

AZD သည် အောက်ခံတွင် Azure CLI ကို အသုံးပြုသည်။ သင်နှစ်ခုလုံးကို အတူတကွ အသုံးပြုနိုင်သည်:
```bash
# သင်၏အက်ပ်ကို AZD ဖြင့်တပ်ဆင်ပါ
azd up

# ထို့နောက် Azure CLI ဖြင့် သတ်မှတ်ထားသောအရင်းအမြစ်များကို အတိအသွားချိန်ညှိပါ
az webapp config set --name myapp --always-on true
```

## 🎯 3 အဆင့်ဖြင့် စတင်လိုက်ပါ

### အဆင့် 1: AZD တပ်ဆင်ခြင်း (2 မိနစ်)

**Windows:**
```powershell
winget install microsoft.azd
```

**macOS:**
```bash
brew tap azure/azd && brew install azd
```

**Linux:**
```bash
curl -fsSL https://aka.ms/install-azd.sh | bash
```

### အဆင့် 2: Azure သို့ လော့ဂ်အင် ဝင်ရန်

```bash
azd auth login
```

### အဆင့် 3: သင်၏ ပထမဆုံး အက်ပ်ကို တင်ရန်

```bash
# ပုံစံမှ စတင်ဖန်တီးပါ
azd init --template todo-nodejs-mongo

# Azure သို့ ထည့်သွင်းပါ (အားလုံးကို ဖန်တီးပါ!)
azd up
```

**🎉 အဲတာပဲ!** သင်၏ အက်ပ်သည် ယခု Azure ပေါ်တွင် အွန်လိုင်းဖြစ်နေပါပြီ။

### သန့်ရှင်းရေး (မမေ့ပါနဲ့!)

```bash
# Remove all resources when done experimenting
azd down --force --purge
```

## သင်တန်း အကျဉ်းချုံး

Azure Developer CLI (azd) ကို အဆင့်လိုက် အခန်းပိုင်းများမှတဆင့် လေ့လာပါ။ **Microsoft Foundry ပေါင်းစည်းမှုနှင့် AI အက်ပ် တင်သွင်းခြင်း အပေါ် အထူးအာရုံစိုက်ထားပါသည်။**

- Azure AI ဝန်ဆောင်မှု ပေါင်းစည်းခြင်းနှင့် ဖော်ပြချက်များ

### သင်ယူရမည့် ရည်ရွယ်ချက်များ

- **AZD အခြေခံများကို ကျွမ်းကျင်နိုင်မည်**: အခြေခံ အကြောင်းအရာများ၊ တပ်ဆင်ခြင်းနှင့် ဖွဲ့စည်းခြင်း
- **AI အက်ပ်များ တင်သွင်းနိုင်မည်**: AZD နှင့် Microsoft Foundry ဝန်ဆောင်မှုများကို အသုံးပြုနိုင်ခြင်း
- **Infrastructure as Code ကို အသုံးပြုတည်ဆောက်နိုင်မည်**: Bicep template များဖြင့် Azure ရင်းမြစ်များကို မန်နေဂျ်နိုင်ခြင်း
- **Deployment များကို ပြဿနာရှာဖွေ ဖြေရှင်းနိုင်မည်**: ပုံမှန် ပြဿနာများအား ဖြေရှင်းနည်းများနှင့် debug နည်းလမ်းများ
- **Production အတွက် ဂရုစိုက်နိုင်မည်**: လုံခြုံရေး၊ ရှုပ်ထိန်းမှု၊ မွန်နီတာ၊ ကုန်ကျစရိတ် စီမံခန့်ခွဲမှု
- **Multi-Agent ဖြေရှင်းချက်များ ဆောက်နိုင်မည်**: ရှုပ်ထွေးသော AI ဖွဲ့စည်းမှုများ ကို တင်သွင်းနိုင်ခြင်း

**ကြာချိန်**: 30-45 မိနစ်  
**ရှုပ်ထွေးမှု**: ⭐

#### သင်ယူရန် အရာများ

#### သင်ယူရန် အရင်းအမြစ်များ
- **🎯 စတင်ရန်**: [What is Azure Developer CLI?](#what-is-azure-developer-cli)
- **📖 သီအိုရီ**: [AZD Basics](docs/chapter-01-foundation/azd-basics.md) - အဓိက အယူအဆများနှင့် အသုံးအနှုန်းများ
- **⚙️ ပြင်ဆင်မှု**: [Installation & Setup](docs/chapter-01-foundation/installation.md) - ပလက်ဖောင်းအလိုက် လမ်းညွှန်ချက်များ
- **🛠️ လက်တွေ့**: [Your First Project](docs/chapter-01-foundation/first-project.md) - အဆင့်လိုက် လမ်းညွှန်ချက်
- **📋 အလျင်အမြန် ကိုးကားစာရွက်**: [Command Cheat Sheet](resources/cheat-sheet.md)

#### လက်တွေ့ လေ့ကျင့်ခန်းများ
```bash
# တပ်ဆင်မှုကို အမြန်စစ်ဆေးခြင်း
azd version

# သင့်ပထမဆုံး အက်ပလီကေးရှင်းကို ဖြန့်ချိပါ
azd init --template todo-nodejs-mongo
azd up
```

**💡 အခန်း ရလဒ်**: AZD ကို အသုံးပြုပြီး Azure သို့ ရိုးရှင်းတဲ့ web application တစ်ခုကို အောင်မြင်စွာ တပ်ဆင်နိုင်ခြင်း

**✅ အောင်မြင်မှု စစ်ဆေးချက်:**
```bash
# အခန်း ၁ ကို ပြီးဆုံးပါက သင်သည် အောက်ပါအရာများကို ပြုလုပ်နိုင်သင့်သည်
azd version              # တပ်ဆင်ထားသော ဗားရှင်းကို ပြသည်
azd init --template todo-nodejs-mongo  # ပရောဂျက်ကို အစပြုသည်
azd up                  # Azure သို့ ဖြန့်ချိသည်
azd show                # လုပ်ဆောင်နေသော အက်ပ်၏ URL ကို ပြသည်
# အက်ပ်ကို ဘရောက်ဇာတွင် ဖွင့်၍ ပုံမှန်အားဖြင့် အလုပ်လုပ်သည်
```

**ဖွံ့ဖြိုးရေး ပတ်ဝန်းကျင် (ခန့်မှန်း $80-150/လ):**

## 📖 နက်ရှိုင်းစွာ ခွဲခြမ်းစိတ်ဖြာခြင်း: AZD ၏ အင်အားများ

အခြေခံသဘောတရားများကျော်လွန်သွားပြီးနောက် AZD သည် ထုတ်လုပ်မှု တင်ပို့မှုများအတွက် အင်အားရှိသည့် အင်္ဂါရပ်များကို ပေးစွမ်းသည်။

- **Template-based deployments** - စီးပွားရေးပုံစံများအတွက် ကြိုတင်တည်ဆောက်ထားသော templates များကို အသုံးပြုပါ
- **Infrastructure as Code** - Azure အရင်းအမြစ်များကို Bicep သို့မဟုတ် Terraform အသုံးပြု၍ စီမံပါ  
- **Integrated workflows** - အလွယ်တကူ provision, deploy နှင့် monitor လုပ်ဆောင်နိုင်သည်
- **Developer-friendly** - ဖွံ့ဖြိုးရေးသူ ပေါ်တင်လုပ်ဆောင်မှုနှင့် အသုံးပြုမှုအတွေ့အကြုံအတွက် အထူးကောင်းမွန်စေသည်

### **AZD + Microsoft Foundry: AI တင်ပို့မှုများအတွက် သင့်တော်သည်**

**AZD ကို AI ဖြေရှင်းချက်များအတွက် ယေဘူယျ အကြောင်းမည့်?** AZD သည် AI ဖွံ့ဖြိုးရေးသူများ ကြုံတွေ့ရသည့် အဓိက စိန်ခေါ်မှုများကို ဖြေရှင်းပေးသည် -

- **AI-စီစဉ်ထားသော Templates** - Microsoft Foundry Models, Cognitive Services, နှင့် ML အလုပ်များအတွက် ကြိုတင်စီစဉ်ထားသော templates များ
- **လုံခြုံသော AI တင်ပို့မှုများ** - AI ဝန်ဆောင်မှုများ၊ API keys နှင့် မော်ဒယ် endpoints များအတွက် တည်ဆောက်ထားသော လုံခြုံရေး ပုံစံများ  
- **ထုတ်လုပ်မှု AI ပုံစံများ** - တိုးချဲ့နိုင်ပြီး ကုန်ကျစရိတ်ထိရောက်သော AI အက်ပလီကေးရှင်းများ ဖန်တီးရန် အကောင်းဆုံး လမ်းလျှောက်များ
- **အစမှအဆုံး AI လုပ်ငန်းစဉ်များ** - မော်ဒယ် ဖွံ့ဖြိုးမှုမှ ထုတ်လုပ်မှု တင်ပို့မှုအထိ မှန်ကန်သော စောင့်ကြည့်မှုဖြင့်
- **ကုန်ကျစရိတ် အ tối ưu화** - AI အလုပ်များအတွက် ရင်းနှီးမြှပ်နှံမှုနှင့် အကစားပြုနည်းများကို အသုံးချခြင်း
- **Microsoft Foundry အပြန်အလှန်ချိတ်ဆက်မှု** - Microsoft Foundry မော်ဒယ် ကတ်တလော့ နှင့် endpoint များနှင့် တမူထူးခြားစွာ ချိတ်ဆက်နိုင်ခြင်း
