## သင်တန်း အကျဉ်းချုပ်

Azure Developer CLI (azd) ကို အဆင့်လိုက် သင်ယူမှုအတွက် ဖွဲ့စည်းထားသည့် အခန်းများမှတဆင့် ကျွမ်းကျင်ပါ။ **Microsoft Foundry ပေါင်းစည်းမှုဖြင့် AI အပလီကေးရှင်း တပ်ဆင်ခြင်းကို အထူးအာရုံစိုက်ပါ။**

### ဒီသင်တန်းသည် ခေတ်မီ ဖွံ့ဖြိုးသူများအတွက် အရေးကြီးသော အကြောင်းရင်း

Microsoft Foundry Discord အသိုင်းအဝိုင်း၏ အမြင်များအရ၊ **45% အထိ ဖွံ့ဖြိုးသူများသည် AZD ကို AI လုပ်ငန်းများအတွက် အသုံးပြုလိုကြသည်** သော်လည်း အောက်ပါအချက်များကြောင့် ခက်ခဲချက်များကြုံတွေ့နေကြသည်။
- ဝန်ဆောင်မှုများစုံသော AI ဖွဲ့စည်းပုံများ
- ထုတ်လုပ်ရေး AI တပ်ဆင်မှု အကောင်းဆုံး လုပ်နည်းများ  
- Azure AI ဝန်ဆောင်မှုပေါင်းစည်းခြင်းနှင့် ဖွဲ့စည်းတပ်ဆင်ခြင်း
- AI လုပ်ငန်းအတွက် စရိတ် အချွတ်ချုတ်ဆောင်ခြင်း
- AI သက်ဆိုင်ရာ တပ်ဆင်မှု ပြဿနာများ ဖြေရှင်းခြင်း

### သင်ယူရမည့် ရည်မှန်းချက်များ

ဤအစီအစဉ်ကို ပြီးမြောက်စေခြင်းဖြင့် သင်သည်-
- **AZD အခြေခံများကို ကျွမ်းကျင်ရမည်**: အဓိက အယူအဆများ၊ တပ်ဆင်ခြင်းနှင့် ဖွဲ့စည်းတပ်ဆင်ခြင်း
- **AI အပလီကေးရှင်းများ တပ်ဆင်နိုင်ရန်**: AZD ကို Microsoft Foundry ဝန်ဆောင်မှုများနှင့် အသုံးပြုနိုင်ရန်
- **Infrastructure as Code ကို အကောင်အထည်ဖော်ရန်**: Bicep နမူနာများဖြင့် Azure ရင်းမြစ်များကို စီမံခန့်ခွဲနိုင်ရန်
- **တပ်ဆင်မှုများကို ပြင်ဆင်ရန်**: ပုံမှန် ပြဿနာများ ဖြေရှင်းခြင်းနှင့် ဒိဘတ်လုပ်ခြင်း
- **ထုတ်လုပ်ရေးအတွက် အကောင်းဆုံး ပြုလုပ်ရန်**: လုံခြုံရေး၊ အရွယ်ချဲ့ခြင်း၊ ကြည့်ရှုခြင်းနှင့် စရိတ် စီမံခန့်ခွဲမှု
- **Multi-Agent ဖြေရှင်းချက်များ တည်ဆောက်ရန်**: ရှုပ်ထွေးသော AI ဖွဲ့စည်းပုံများ တပ်ဆင်နိုင်ရန်

## 📚 သင်ယူရန် အခန်းများ

*သင့်အတွေ့အကြုံနှင့် ရည်မှန်းချက်များအပေါ် မူတည်၍ သင်ယူမည့် လမ်းကြောင်းကို ရွေးချယ်ပါ*

### 🚀 အခန်း 1: အခြေခံနှင့် အလျင်အမြန် စတင်ခြင်း
**လိုအပ်ချက်များ**: Azure subscription, အခြေခံ command line နည်းပညာ  
**ကာလ**: 30-45 မိနစ်  
**ရှုပ်ထွေးမှု**: ⭐

#### သင်သင်ယူမည့် အရာများ
- Azure Developer CLI အခြေခံများ နားလည်မှု
- သင့်ပလက်ဖောင်းပေါ် AZD တပ်ဆင်ခြင်း
- သင့်ရဲ့ ပထမဆုံး အောင်မြင်သော တပ်ဆင်မှု

#### သင်ယူရေး အရင်းအမြစ်များ
- **🎯 စတင်ရန်**: [Azure Developer CLI ဆိုတာဘာလဲ?](../..)
- **📖 အယူအဆ**: [AZD အခြေခံ](docs/getting-started/azd-basics.md) - အဓိက အကြောင်းအရာများနှင့် အသုံးအနှုန်းများ
- **⚙️ တပ်ဆင်ခြင်းနှင့် ပြင်ဆင်ခြင်း**: [Installation & Setup](docs/getting-started/installation.md) - ပလက်ဖောင်းအလိုက် လမ်းညွှန်ချက်များ
- **🛠️ လက်တွေ့ လေ့ကျင့်ခန်း**: [သင့်ရဲ့ ပထမဆုံး ပရောဂျက်](docs/getting-started/first-project.md) - ခြေလှမ်းချင်းစီ လေ့ကျင့်နိုင်ရေး
- **📋 အမြန်အချက်အလက်**: [Command လမ်းညွှန် စာရွက်](resources/cheat-sheet.md)

#### လက်တွေ့ လေ့ကျင့်ခန်းများ
```bash
# တပ်ဆင်ခြင်းကို အမြန် စစ်ဆေးခြင်း
azd version

# သင့် ပထမဆုံး အက်ပလီကေးရှင်းကို ဖြန့်ချိပါ
azd init --template todo-nodejs-mongo
azd up
```

**💡 အခန်းရလဒ်**: AZD ကို အသုံးပြု၍ ရိုးရှင်းသော ဝက်ဘ်အက်ပလီကေးရှင်းတစ်ခုကို Azure သို့ အောင်မြင်စွာ တပ်ဆင်နိုင်ရန်

**✅ အောင်မြင်မှု အတည်ပြုချက်:**
```bash
# အခန်း ၁ ကို ပြီးမြောက်ပါက သင်သည် အောက်ပါအရာများကို လုပ်နိုင်ရမည်။
azd version              # တပ်ဆင်ထားသော ဗားရှင်းကို ပြသည်
azd init --template todo-nodejs-mongo  # ပရောဂျက်ကို စတင်ပြင်ဆင်သည်
azd up                  # Azure သို့ တင်ပို့သည်
azd show                # လည်ပတ်နေသည့် အက်ပ်၏ URL ကို ပြသည်
# အက်ပလီကေးရှင်းကို ဘရောက်ဇာတွင် ဖွင့်ပြီး အလုပ်လုပ်သည်
azd down --force --purge  # ရင်းမြစ်များကို ရှင်းလင်းသည်
```

**📊 ကြာချိန် အားထုတ်မှု:** 30-45 မိနစ်  
**📈 အတတ်ပညာအဆင့် နောက်ထပ်:** အခြေခံ အက်ပလီကေးရှင်းများကို ကိုယ့်တိုင် တပ်ဆင်နိုင်သည်

**✅ အောင်မြင်မှု အတည်ပြုချက်:**
```bash
# အခန်း 1 ကို ပြီးစီးသည့်အခါ သင်သည် အောက်ပါအရာများကို ပြုလုပ်နိုင်လိမ့်မည်:
azd version              # တပ်ဆင်ထားသော ဗားရှင်းကို ပြသည်
azd init --template todo-nodejs-mongo  # ပရောဂျက်ကို စတင်ဖန်တီးသည်
azd up                  # Azure သို့ ဖြန့်ချိသည်
azd show                # လည်ပတ်နေသော အက်ပ်၏ URL ကို ပြသည်
# အက်ပ်ကို ဘရောက်ဇာတွင် ဖွင့်ပြီး ပုံမှန် လည်ပတ်သည်
azd down --force --purge  # အရင်းအမြစ်များကို ရှင်းလင်းသည်
```

**📊 ကြာချိန် အားထုတ်မှု:** 30-45 မိနစ်  
**📈 အတတ်ပညာအဆင့် နောက်ထပ်:** အခြေခံ အက်ပလီကေးရှင်းများကို ကိုယ့်တိုင် တပ်ဆင်နိုင်သည်

#### လက်တွေ့ လေ့ကျင့်ခန်းများ
```bash
# သင့်ပထမဆုံး AI အက်ပလီကေးရှင်းကို ဖြန့်ချိပါ

# နောက်ထပ် AI နမူနာများကို စမ်းသပ်ပါ
```

**💡 အခန်းရလဒ်**: RAG အရည်အချင်းပါရှိသော AI စကားပြော အက်ပလီကေးရှင်းကို တပ်ဆင်၍ ဖွဲ့စည်းတပ်ဆင်နိုင်ရန်

**✅ အောင်မြင်မှု အတည်ပြုချက်:**
```bash
# အခန်း ၂ ပြီးနောက် သင်သည် အောက်ပါအချက်များကို လုပ်နိုင်လိမ့်မည်။
azd init --template azure-search-openai-demo
azd up
# AI စကားပြော အင်တာဖေ့စ်ကို စမ်းသပ်နိုင်ရန်
# မေးခွန်းများ မေးပြီး အရင်းအမြစ်များပါသော AI ဖြေကြားချက်များ ရယူနိုင်ရန်
# ရှာဖွေရေး ပေါင်းစပ်မှု အလုပ်လုပ်သည်ကို အတည်ပြုနိုင်ရန်
azd monitor  # Application Insights တွင် telemetry ကို ပြသထားသည်ကို စစ်ဆေးရန်
azd down --force --purge
```

**📊 ကြာချိန် အားထုတ်မှု:** 1-2 နာရီ  
**📈 အတတ်ပညာအဆင့် နောက်ထပ်:** ထုတ်လုပ်ရေးအဆင့် ပြင်ဆင်ထားသော AI အက်ပလီကေးရှင်းများကို တပ်ဆင်၍ ဖွဲ့စည်းနိုင်သည်  
**💰 စရိတ် အသိပညာ:** ဖွံ့ဖြိုးရေး စရိတ် $80-150/လ ကန့်သတ်၊ ထုတ်လုပ်ရေး စရိတ် $300-3500/လ

#### 💰 AI တပ်ဆင်မှုများအတွက် စရိတ်စဉ်းစားရန် အချက်များ

**ဖွံ့ဖြိုးရေး ပတ်ဝန်းကျင် (ခန့်မှန်း $80-150/လ):**
- Azure OpenAI (Pay-as-you-go): $0-50/လ (token အသုံးအရ)
- AI Search (Basic ထပ်): $75/လ
- Container Apps (သုံးစွဲအရ): $0-20/လ
- Storage (Standard): $1-5/လ

**ထုတ်လုပ်ရေး ပတ်ဝန်းကျင် (ခန့်မှန်း $300-3,500+/လ):**
- Azure OpenAI (တည်ကြည်သော စွမ်းဆောင်ရည်အတွက် PTU): $3,000+/လ သို့မဟုတ် အမြင့်သုံးစွဲမှုအတွက် Pay-as-you-go
- AI Search (Standard ထပ်): $250/လ
- Container Apps (ပေးအပ်ထားသော): $50-100/လ
- Application Insights: $5-50/လ
- Storage (Premium): $10-50/လ

**💡 စရိတ် ထိထိရောက်ရောက် လျှော့ချနည်းများ:**
- သင်ယူရန် **Free Tier** Azure OpenAI ကို အသုံးပြုပါ (လစဉ် 50,000 tokens ပါဝင်သည်)
- တတ်နိုင်ပါက ဖွံ့ဖြိုးရေး မလုပ်နေချိန်တွင် `azd down` ကို လ chạy၍ ရင်းမြစ်များကို ဖယ်ရှားပါ
- consumption-အခြေခံ ဘီလ်မှုန့်ဖြင့် စတင်ပြီး ထုတ်လုပ်ရေးအတွက်သာ PTU သို့ အဆင့်တင်ပါ
- တပ်ဆင်မှုပြုမီ စရိတ် ခန့်မှန်းရန် `azd provision --preview` ကို အသုံးပြုပါ
- auto-scaling ကို ဖွင့်ပါ: အမှန်တကယ် အသုံးပြုသည့်အတွက်သာ ငွေပေးချေရပါ

စရိတ် ကြည့်ရှုခြင်း:
```bash
# ခန့်မှန်းထားသော လစဉ် ကုန်ကျစရိတ်များကို စစ်ဆေးပါ
azd provision --preview

# Azure Portal တွင် အမှန်တကယ် ကျသင့်သော ကုန်ကျစရိတ်များကို စောင့်ကြည့်ပါ
az consumption budget list --resource-group <your-rg>
```

### အပြန်အလှန် အလုပ်ရုံ သင်တန်း ပစ္စည်းများ

#### 📚 အလုပ်ရုံ သင်တန်း ဖွဲ့စည်းပုံ
အလုပ်ရုံ သင်တန်းသည် **ရှာဖွေမှု → တင်သွင်းမှု → ကိုယ်ပိုင်ပြင်ဆင်မှု** နည်းလမ်းကို ကျင့်သုံးသည်-

1. **ရှာဖွေရေး အဆင့်** (45 mins)
   - Microsoft Foundry တမ်းပလိတ်များနှင့် ဝန်ဆောင်မှုများကို ရှာဖွေပါ
   - multi-agent ဖွဲ့စည်းပုံ ပုံစံများကို နားလည်ပါ
   - တင်သွင်းရန် လိုအပ်ချက်များနှင့် prerequisite များကို ပြန်လည်စိစစ်ပါ

2. **တင်သွင်းရေး အဆင့်** (2 hours)
   - AZD ဖြင့် AI အပလီကေးရှင်းများကို လက်တွေ့ တင်သွင်းပါ
   - Azure AI ဝန်ဆောင်မှုများနှင့် endpoints များကို ဖွဲ့စည်းတပ်ဆင်ပါ
   - လုံခြုံရေးနှင့် အတည်ပြုခြင်း ပုံစံများကို အကောင်အထည်ဖော်ပါ

3. **ကိုယ်ပိုင် ပြင်ဆင်ရေး အဆင့်** (45 mins)
   - သတ်မှတ်အသုံးချမှုများအတွက် အက်ပလီကေးရှင်းများကို ပြင်ဆင်ပါ
   - ထုတ်လုပ်မှုတင်သွင်းမှုအတွက် အကောင်းဆုံးအခြေအနေသို့ စီမံပြင်ဆင်ပါ
   - စောင့်ကြည့်မှုနှင့် ကုန်ကျစရိတ် စီမံခန့်ခွဲမှုကို ဖော်ဆောင်ပါ

## 📖 Azure Developer CLI ဆိုတာဘာလဲ?

Azure Developer CLI (azd) သည် ဖွံ့ဖြိုးရေးသူအခြေပြု command-line အင်တာဖေ့စ်တစ်ခုဖြစ်ပြီး Azure တွင် အက်ပလီကေးရှင်းများကို တည်ဆောက်ခြင်းနှင့် တင်သွင်းခြင်း လုပ်ငန်းစဉ်ကို အားမြှင့်ပေးသည်။ ၎င်းသည် ပေးစွမ်းသည်-

- **တမ်းပလိတ်အခြေပြု တင်သွင်းမှုများ** - ယေဘူယျ အက်ပလီကေးရှင်း ပုံစံများအတွက် ကြိုတင်တည်ဆောက်ထားသော တမ်းပလိတ်များကို အသုံးပြုပါ
- **Infrastructure as Code** - Bicep သို့မဟုတ် Terraform ကို အသုံးပြုကာ Azure အရင်းအမြစ်များကို စီမံပါ  
- **ပေါင်းစည်းထားသည့် လုပ်ငန်းစဉ်များ** - အက်ပလီကေးရှင်းများကို အလွယ်တကူ provision၊ တင်သွင်းနှင့် စောင့်ကြည့်နိုင်စေရန်
- **ဖွံ့ဖြိုးရေးသူအတွက် အဆင်ပြေသည်** - ဖွံ့ဖြိုးရေးသူ၏ ထုတ်လုပ်နိုင်စွမ်းနှင့် အတွေ့အကြုံကို အဓိကထား ထိရောက်အောင် ပြုလုပ်ထားသည်

### **AZD + Microsoft Foundry: AI တင်သွင်းမှုများအတွက် အကောင်းဆုံး**

**AZD ကို AI ဖြေရှင်းချက်များအတွက် အသုံးချရသည့် အကြောင်းရင်း** AZD သည် AI ဖွံ့ဖြိုးရေးသူများ ကြုံတွေ့ရသော အဓိက စိန်ခေါ်မှုများကို ဖြေရှင်းပေးသည်-

- **AI-သင့်တမ်းပလိတ်များ** - Azure OpenAI, Cognitive Services, နှင့် ML အလုပ်ပမာဏများအတွက် ကြိုတင် ဖွဲ့စည်းထားသည့် တမ်းပလိတ်များ
- **လုံခြုံစိတ်ချရသော AI တင်သွင်းမှုများ** - AI ဝန်ဆောင်မှုများ၊ API key များနှင့် မော်ဒယ် endpoint များအတွက် အတွင်းရေး လုံခြုံရေး ပုံစံများ အပါရှိ
- **ထုတ်လုပ်မှု အတွက် AI ပုံစံများ** - ချဲ့ထွင်နိုင်ပြီး ကုန်ကျစရိတ် သက်သာစေရန် အကောင်းဆုံးလက်တွေ့များ
- **ကုမ္ပဏီအတွင်း အပြီးအစီး AI လုပ်ငန်းစဉ်များ** - မော်ဒယ် ဖွံ့ဖြိုးမှ ထုတ်လုပ်မှုပြီး စောင့်ကြည့်မှုအထိ
- **ကုန်ကျစရိတ် တိုးတက်မှု** - AI အလုပ်ပမာဏများအတွက် စမတ်resource ချိန်ညှိခြင်းနှင့် ချဲ့ထွင်နည်းဗျူဟာများ
- **Microsoft Foundry ပေါင်းစည်းမှု** - Microsoft Foundry မော်ဒယ် ဇယားနှင့် endpoint များကို ချိတ်ဆက်ပေးနိုင်ခြင်း


### ❌ "No subscription found" or "Subscription not set"

```bash
# ရရှိနိုင်သော subscription များကို ပြပါ
az account list --output table

# ပုံမှန် subscription ကို သတ်မှတ်ပါ
az account set --subscription "<subscription-id-or-name>"

# AZD ပတ်ဝန်းကျင်အတွက် သတ်မှတ်ပါ
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# အတည်ပြုပါ
az account show
```

### ❌ "InsufficientQuota" or "Quota exceeded"

```bash
# အခြား Azure ဒေသကို စမ်းကြည့်ပါ
azd env set AZURE_LOCATION "westus2"
azd up

# သို့မဟုတ် ဖွံ့ဖြိုးရေးတွင် ပိုသေးငယ်သော SKU များကို အသုံးပြုပါ
# infra/main.parameters.json ကို တည်းဖြတ်ပါ:
{
  "sku": "B1"  // Instead of "P1V2"
}
```

### ❌ "azd up" fails halfway through

```bash
# ရွေးချယ်စရာ 1: ရှင်းပြီး ထပ်မံစမ်းကြည့်ပါ
azd down --force --purge
azd up

# ရွေးချယ်စရာ 2: အခြေခံအဆောက်အအုံကိုသာ ပြင်ပါ
azd provision

# ရွေးချယ်စရာ 3: အသေးစိတ် မှတ်တမ်းများကို စစ်ဆေးပါ
azd show
azd logs
```

### ❌ "Authentication failed" or "Token expired"

```bash
# ပြန်လည် အတည်ပြုရန်
az logout
az login

azd auth logout
azd auth login

# အတည်ပြုမှုကို စစ်ဆေးရန်
az account show
```

### ❌ "Resource already exists" or naming conflicts

```bash
# AZD သည် ထူးခြားသော အမည်များ ဖန်တီးပေးသော်လည်း၊ အမည်ထပ်နေပါက:
azd down --force --purge

# ထိုအခါ သန့်ရှင်းပြီး အသစ်သော ပတ်ဝန်းကျင်ဖြင့် ထပ်မံ လုပ်ကြည့်ပါ။
azd env new dev-v2
azd up
```

### ❌ Template deployment taking too long

**ပုံမှန် စောင့်ဆိုင်းချိန်များ:**
- ရိုးရှင်းသော ဝဘ်အက်ပ်: 5-10 မိနစ်
- ဒေတာဘေ့စ်ပါ အက်ပ်: 10-15 မိနစ်
- AI အက်ပလီကေးရှင်းများ: 15-25 မိနစ် (OpenAI ရဲ့ provisioning သည် နည်းနှေးသည်)

```bash
# တိုးတက်မှုကိုစစ်ဆေးပါ
azd show

# အကယ်၍ ၃၀ မိနစ်ထက်ပို ရပ်တန့်နေပါက Azure Portal ကို စစ်ဆေးပါ:
azd monitor
# မအောင်မြင်သော ဖြန့်ချိမှုများကို ရှာဖွေပါ
```

### ❌ "Permission denied" or "Forbidden"

```bash
# သင့် Azure ရာထူးကို စစ်ဆေးပါ
az role assignment list --assignee $(az account show --query user.name -o tsv)

# အနည်းဆုံး "Contributor" ရာထူးလိုအပ်သည်
# သင့် Azure အက်မင်ထံ အောက်ပါအရာများပေးရန် တောင်းဆိုပါ:
# - Contributor (အရင်းအမြစ်များအတွက်)
# - User Access Administrator (ရာထူး ခန့်အပ်ပေးမှုများအတွက်)
```

### ❌ Can't find deployed application URL

```bash
# ဝန်ဆောင်မှု endpoint များအားလုံးကို ပြပါ
azd show

# သို့မဟုတ် Azure Portal ကို ဖွင့်ပါ
azd monitor

# တိကျသော ဝန်ဆောင်မှုကို စစ်ဆေးပါ
azd env get-values
# *_URL ဟု အမည်ပေးထားသော အပြောင်းအလဲများကို ရှာပါ
```

- **45%** က တည်ဆောက်သူများသည် AI လုပ်ငန်းများအတွက် AZD ကို အသုံးပြုချင်ကြသည်
- **အဓိက စိန်ခေါ်မှုများ**: Multi-service deployments၊ credential management၊ production readiness  
- **အများအားဖြင့် တောင်းဆိုသည့်အရာများ**: AI အထူးပုံစံများ၊ ပြဿနာဖြေရှင်းလမ်းညွှန်များ၊ အကောင်းဆုံး လေ့လာနည်းများ

**ကျွန်ုပ်တို့၏ ကော်မျူနီတီတွင် ပါဝင်၍:**
- သင်၏ AZD + AI အတွေ့အကြုံများကို မျှဝေပြီး အကူအညီရယူနိုင်သည်
- AI ပုံစံအသစ်များ၏ မျှPreview များကို အကြို ဝင်ရောက်ကြည့်ရှုနိုင်သည်
- AI တင်သွင်းမှု အကောင်းဆုံး လေ့လာနည်းများထဲတွင် လက်က်ကျော် ပါဆောင်နိုင်သည်
- အနာဂတ် AI + AZD လုပ်ဆောင်ချက် ဖွံ့ဖြိုးတိုးတက်မှုကို ထိခိုက်သက်ရောက်စေနိုင်သည်

### သင်တန်းတွင် ဝင်ပါက ဆောင်ရွက်ရန်
ကျွန်ုပ်တို့ သင့်ထံ အပေါင်းအလှပေါင်းကို ကြိုဆိုပါသည်! အသေးစိတ်အချက်အလက်များအတွက် [Contributing Guide](CONTRIBUTING.md) ကို ဖတ်ပေးပါ:
- **အကြောင်းအရာ တိုးတက်မှုများ**: ရှိပြီးသား အခန်းများနှင့် ကိုးကားနမူနာများကို တိုးတက်စေရန်
- **နမူနာသစ်များ**: အမှန်တကယ် ဖြစ်ပေါ်နိုင်သော အခြေအနေများနှင့် ပုံစံများ ထည့်သွင်းရန်  
- **ဘာသာပြန်ခြင်း**: ဘာသာစကား များစွာ အထောက်အပံ့ကို ထိန်းသိမ်းရန် အကူအညီ
- **အမှားအစီရင်ခံစာများ**: တိကျမှုနှင့် စပပ်ပြတ်မှုများကို ချဲ့ထွင်ဖြစ်စေခြင်း
- **အဖွဲ့အစည်း စံနှုန်းများ**: ကျွန်ုပ်တို့၏ ဖလှယ်ကင်းသော ကော်မျူနီတီ ညွှန်ကြားချက်များကို လိုက်နာပါ
