## 🆕 azd တွင် ယနေ့ အသစ်ထည့်သွင်းချက်များ

Azure Developer CLI သည် ရိုးရာ web app များနှင့် API များမှ ကျော်လွန်ကာ တိုးတက်မြှင့်တင်လာပါသည်။ ယနေ့တွင် azd သည် Azure သို့ မည်သည့် application မဆို ထည့်သွင်းဖို့ အသုံးပြုနိုင်သော တစ်ခုတည်းသော အ工具 ဖြစ်ပါသည်—AI အခြေပြု application များနှင့် သတိပေးစနစ် agent များကိုပါ အပါအဝင်။

ဒါက သင့်အတွက် အဓိပ္ပါယ်ရတဲ့ အချက်တွေကတော့ -

- **AI agent များသည် ယခု azd ၏ ထူးခြားသော workload များ အဖြစ် လက်ခံခံထားသွားပါပြီ။** `azd init` → `azd up` ဆိုတဲ့ အလွယ်ကူသိပြီးသား workflow ကိုအသုံးပြုပြီး AI agent project များကို initialize, deploy, နှင့် စီမံခန့်ခွဲနိုင်ပါတယ်။
- **Microsoft Foundry ပေါင်းစည်းမှု** က model deployment, agent hosting, နှင့် AI service configuration များကို azd template ecosystem ထဲသို့ တိုက်ရိုက် ယူဆောင်လာပါတယ်။
- **အခြေခံ workflow သည် မပြောင်းလဲသေးပါ။** todo app, microservice, သို့မဟုတ် multi-agent AI ဖြေရှင်းချက် တက်ကြွစွာ တာဝန်ကျေသဖြင့် deploy လုပ်နေသော်လည်း, command များသည် ယေဘူယျ တူညီနေဆဲဖြစ်သည်။

သင့်မှာ azd ကို အရင်ကအသုံးပြုခဲ့ဖူးပါက AI စနစ်များအတွက် ထောက်ခံမှုသည် သဘာဝတရားအဆက်တစ်ခုဖြစ်သည်—တခြားသံစဉ်ကတစ်ခု သို့မဟုတ် တိုးတက်မြှင့်တင်သော ကိရိယာမဟုတ်ပါ။ သင် အသစ်စတင်ပါကလည်း၊ တစ်ခုတည်းသော workflow ကို သင်ယူမည်ဖြစ်ပြီး အရာအားလုံးအတွက် အသုံးဝင်ပါသည်။

## 🚀 Azure Developer CLI (azd) ဆိုတာ ဘာလဲ?

**Azure Developer CLI (azd)** သည် developer များအတွက် ရည်ရွယ်ထားသော command-line ကိရိယာဖြစ်ပြီး Azure သို့ application များ တင်သွင်းရန် လွယ်ကူစွာ ပြုလုပ်ပေးသည်။ Azure resource အများအပြားကို လက်ဖြင့် တစ်ခုချင်းစီ ဖန်တီးချိတ်ဆက်စရာမလိုဘဲ၊ တစ်ချက် command ဖြင့် အပလီကေးရှင်းတစ်ခုလုံးကို deploy လုပ်နိုင်စေသည်။

### `azd up` ၏ မာဂျစ်

```bash
# ဒီတစ်ခုတည်းသောcommandသည် အားလုံးကိုလုပ်ဆောင်သည်။
# ✅ Azure ရင်းမြစ်အားလုံးကို ဖန်တီးသည်
# ✅ ကွန်ရက်နှင့်လုံခြုံရေးကို ပြင်ဆင်သည်
# ✅ သင့် application ကုဒ်ကို ဆောက်လုပ်သည်
# ✅ Azure သို့ deploy လုပ်သည်
# ✅ လုပ်ကိုင်နိုင်သော URL ကိုပေးသည်
azd up
```

**ရလဒ်က ရပါတယ်!** Azure Portal တွင် မထိထိချုပ်ချင်း မနှိပ်ရန်၊ ရိုးရှင်းမလှသော ARM templates များကို အရင်လေ့လာရန် မလိုအပ်တော့ပါ၊ လက်ဖြင့် configure ပြုလုပ်ရန် မလိုတော့ပါ—Azure ပေါ်တွင် လက်ရှိ အလုပ်လုပ်သော application များသာ ရှိပါသည်။

## ❓ Azure Developer CLI နှင့် Azure CLI: ဘာတွေကွာသလဲ?

ဒီဟာက စတင်သူများ အလွန် မေးခွန်းများ မေးလေ့ရှိတဲ့ အကြောင်းအရာဖြစ်ပါတယ်။ ရိုးရှင်းတဲ့ ပြตอบကတော့ -

| Feature | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **ရည်ရွယ်ချက်** | တစ်ခုချင်း Azure resource များကို စီမံခန့်ခွဲရန် | အပလီကေးရှင်း ပြည့်ဝစွာ ကို deploy လုပ်ရန် |
| **စိတ်ထား** | အုတ်မြစ်အခြေခံ (Infrastructure-focused) | အပလီကေးရှင်း အခြေခံ (Application-focused) |
| **ဥပမာ** | `az webapp create --name myapp...` | `azd up` |
| **လေ့လာရန် လှမ်းလျှောက်မှု** | Azure services များကို သိထားဖို့ လိုအပ်သည် | သင့် app ကို သိထားခြင်းသာ လုံလောက်သည် |
| **အကောင်းဆုံး အသုံးပြုရန်** | DevOps, Infrastructure | Developers, Prototyping |

### ရိုးရှင်းသော ဆင်တူခြင်း

- **Azure CLI** က အိမ်တစ်လုံး တည်ဆောက်ဖို့ လက်စွဲကိရိယာများ (ဖျားမာ, saw, nails) စုစည်းထားသလို ဖြစ်ပါတယ် - အသုံးပြုသူသည် ဘာမှမဆိုတည်ဆောက်နိုင်သော်လည်း ဆောက်လုပ်ရေးဗဟုသုတ လိုအပ်သည်။
- **Azure Developer CLI** က ဆောက်လုပ်ပေးမည့် စီမံခန့်ခွဲသူကို ငှားထားခြင်းကဲ့သို့ ဖြစ်သည် - သင်လိုချင်သည်ကို ဖော်ပြလိုက်ပါ၊ မျက်နှာချင်းဆိုင် အလုပ်ကို သူတို့လုပ်ပေးပါလိမ့်မယ်။

### ဘယ်အချိန်မှာ မည်ကို အသုံးပြုမလဲ

| အခြေအနေ | သုံးရန် |
|----------|----------|
| "ကျွန်တော့် web app ကို လျင်မြန်စွာ deploy လုပ်ချင်တယ်" | `azd up` |
| "storage account အတစ်ခု ဖန်တီးဖို့ လိုပါတယ်" | `az storage account create` |
| "AI application တစ်ခု တည်ဆောက်နေပါတယ်" | `azd init --template azure-search-openai-demo` |
| "တစ်ခုချင်း Azure resource ကို debug လုပ်ဖို့ လိုပါတယ်" | `az resource show` |
| "မိနစ်ပိုင်းအတွင်း production အသင့် deployment လုပ်ချင်ပါတယ်" | `azd up --environment production` |

### သူတို့ သုံးပြိုင်နိုင်ကြသည်!

AZD သည် အောက်ခံတွင် Azure CLI ကို အသုံးပြုထားသည်။ သင်နှစ်ခုလုံးကို အသုံးပြုနိုင်ပါသည်:
```bash
# AZD ဖြင့် သင့်အက်ပ်ကို ထည့်သွင်းပါ
azd up

# ထို့နောက် Azure CLI ဖြင့် သတ်မှတ်ထားသော အရင်းအမြစ်များကို စိစစ်ပြင်ဆင်ပါ
az webapp config set --name myapp --always-on true
```

### အဆင့် 3: သင့် ပထမဆုံး App ကို Deploy လုပ်ပါ


## သင်တန်း အကျဥ်းချုပ်

Azure Developer CLI (azd) ကို အဆင့်လိုက်အခန်းများမှတဆင့် ကျွမ်းကျင်ပါစေ။ **Microsoft Foundry ပေါင်းစည်းမှုနှင့် အတူ AI application deployment အပေါ် အထူးအာရုံစိုက်ထားပါသည်။**

### ဒီ သင်တန်းသည် ခေတ်သစ် developer များအတွက် မည်ကြောင့် အရေးကြီးသလဲ

Microsoft Foundry Discord community ၏ အတွေ့အကြုံများအရ၊ **developer 45% က AZD ကို AI workloads များအတွက် အသုံးချချင်ကြောင်း** ဖော်ပြခဲ့ကြပြီး အောက်ပါအခက်အခဲများနှင့် တွေ့ကြုံနေကြသည် -
- ဆင်းကျဉ်းသော multi-service AI architecture များ
- Production AI deployment အတွက် အကောင်းဆုံး လက်ထောက်အတွေ့အကြုံများ  
- Azure AI service ပေါင်းစည်းမှု နှင့် configuration
- AI workloads များအတွက် ကုန်ကျစရိတ် တိုးချဲ့မှုကို လျော့နည်းစေရန် ဗျူဟာများ
- AI-ဆိုင်ရာ deployment ပြဿနာများ ကို ကောက်နုတ်ရှာဖွေခြင်း

### သင်ယူ ရမည့် ရလဒ်များ

ဒီ ဖွဲ့စည်းထားသော သင်တန်းကို အပြီးသတ်လျှင်၊ သင်သည် -
- **AZD အခြေခံကျွမ်းကျင်မှု**: အဓိက သဘောတရားများ၊ installation နှင့် configuration များကို ကျွမ်းကျင်ပါမည်
- **AI Applications များ Deploy လုပ်နည်း**: AZD ကို Microsoft Foundry services များနှင့် အသုံးပြုပါ
- **Infrastructure as Code ကို အကောင်အထည်ဖော်ခြင်း**: Bicep templates ဖြင့် Azure resources များကို စီမံပါ
- **Deployment များ Troubleshoot လုပ်နည်း**: ပုံမှန်ပြဿနာများကို ဖြေရှင်း၍ debug ပြုလုပ်နိုင်ပါမည်
- **Production အတွက် အဖိုးတန် စနစ်များ Optimize လုပ်ခြင်း**: လုံခြုံရေး၊ scaling, monitoring, နှင့် ကုန်ကျစရိတ် စီမံခန့်ခွဲမှု
- **Multi-Agent ဖြေရှင်းချက်များ တည်ဆောက်ခြင်း**: ပိုမိုရှုပ်ထွေးသော AI architecture များကို deploy လုပ်နိုင်ခြင်း

#### သင်ယူရန် အရင်းအမြစ်များ

- **⚙️ အင်စတောလေးရှင်း**: [Installation & Setup](docs/chapter-01-foundation/installation.md) - ပလက်ဖောင်း သီးသန့် ဗီဒီယိုများ
- **🛠️ လက်တွေ့**: [Your First Project](docs/chapter-01-foundation/first-project.md) - အဆင့်လိုက် လမ်းညွှန်ချက်

#### လက်တွေ့ လေ့ကျင့်ခန်းများ
```bash
# တပ်ဆင်မှု အမြန်စစ်ဆေးချက်
azd version

# သင့်ပထမဆုံး အက်ပလီကေးရှင်းကို ဖြန့်ချိပါ
azd init --template todo-nodejs-mongo
azd up
```

**💡 အခန်းရလဒ်**: AZD ကို အသုံးပြု၍ Azure သို့ ရိုးရှင်းသော ဝဘ်အက်ပလီကေးရှင်း တစ်ခုကို အောင်မြင်စွာ တင်ပို့နိုင်ပါသည်

**✅ အောင်မြင်မှု အတည်ပြုချက်:**
```bash
# အခန်း ၁ ကိုပြီးမြောက်ပါက သင်သည် အောက်ပါအရာများကို လုပ်နိုင်သင့်သည်:
azd version              # တပ်ဆင်ထားသော ဗားရှင်းကို ပြသည်
azd init --template todo-nodejs-mongo  # ပရောဂျက်ကို စတင်တည်ထောင်သည်
azd up                  # Azure သို့ တင်ပို့သည်
azd show                # လည်ပတ်နေသော အက်ပ်၏ URL ကို ပြသည်
# အက်ပလီကေးရှင်းသည် ဘရောက်ဇာတွင် ဖွင့်ပြီး မှန်ကန်စွာ လည်ပတ်သည်
azd down --force --purge  # အရင်းအမြစ်များကို ရှင်းလင်းသည်
```

**📊 လိုအပ်သည့် အချိန်:** 30-45 မိနစ်  
**📈 ပို့ချပြီးနောက် ကျွမ်းကျင်မှုအဆင့်:** မူလ အက်ပလီကေးရှင်းများကို ကိုယ်တိုင် တင်ပို့နိုင်သည်

### အပြန်အလှန် လက်တွေ့ အလုပ်ရုံ ပစ္စည်းများ

ကျွန်တော်တို့၏ အလုပ်ရုံပစ္စည်းများသည် အခန်းအလိုက် သင်ကြားမှုကို ပြည့်စုံစေပြီး ကိုယ်တိုင်တက်ကြွလေ့လာနိုင်စေရန်နှင့် သင်တန်းပေးသူဦးဆောင်သော အစီအစဉ်များနှစ်မျိုးစလုံးအတွက် သင့်တော်စေရန် ဒီဇိုင်းရေးဆွဲထားပါသည်။

#### 🛠️ အလုပ်ရုံ အင်္ဂါရပ်များ
- **Browser-Based Interface**: MkDocs အား အခြေခံထားသော browser အပြင်အဆင်ဖြင့် ရှာဖွေမည့်၊ ကူးယူမည့်နှင့် သာမန်အရောင်အပြောင်းများပါ
- **GitHub Codespaces Integration**: တစ်ချက်နှိပ်၍ ဖွင့်နိုင်သော ဖွံ့ဖြိုးရေး ပတ်ဝန်းကျင်
- **Structured Learning Path**: ၈ မော်ဂျူး ဦးတည် လမ်းညွှန် လေ့ကျင့်ခန်းများ (စုစုပေါင်း ၃-၄ နာရီ)
- **Progressive Methodology**: နိဒါန်း → ရွေးချယ်ခြင်း → အတည်ပြုခြင်း → ချိုးဖျက်၍ နားလည်ခြင်း → ဖော်ပြချက် → စိတ်ကြိုက်ပြင်ဆင်ခြင်း → ဖျက်ပစ်ခြင်း → နိဂုံးချုပ်
- **Interactive DevContainer Environment**: ကြိုတင် တပ်ဆင်ထားသော ကိရိယာများနှင့် အားကိုးနိုင်သော မိုဒူးများ

#### 📚 အလုပ်ရုံ မော်ဂျူး အဖွဲ့စည်းပုံ

| Module | Topic | What You'll Do | Duration |
|--------|-------|----------------|----------|
| **0. Introduction** | အလုပ်ရုံ အနှစ်ချုပ် | သင်ကြားမှု ရည်မှန်းချက်များ၊ မရှိမဖြစ်လိုအပ်ချက်များနှင့် အလုပ်ရုံ ဖွဲ့စည်းပုံကို နားလည်ပါ | 15 မိနစ် |
| **1. Selection** | ပုံစံ ရှာဖွေမှု | AZD ပုံစံများကို စူးစမ်း၍ သင့်အခြေအနေအတွက် သင့်တော်သော AI ပုံစံကို ရွေးချယ်ပါ | 20 မိနစ် |
| **2. Validation** | တပ်ဆင်ပြီး အတည်ပြုခြင်း | ပုံစံကို `azd up` ဖြင့် တပ်ဆင်ပြီး အမြင့်မြင့် အင်ဖရာစထ်ာခ်ချာ အလုပ်ဖြစ်မှုကို အတည်ပြုပါ | 30 မိနစ် |
| **3. Deconstruction** | ဖွဲ့စည်းပုံ ကို နားလည်ခြင်း | GitHub Copilot ကို အသုံးပြု၍ ပုံစံ အင်္ဂါရပ်များ၊ Bicep ဖိုင်များနှင့် ကုဒ် စီမံခန့်ခွဲမှုကို စူးစမ်းပါ | 30 မိနစ် |
| **4. Configuration** | azure.yaml အတွင်း အနက်ရှိုင်း သေချာ လေ့လာမှု | `azure.yaml` ဖော်ပြချက်များ၊ lifecycle hooks နှင့် ပတ်ဝန်းကျင် မော်ကွန်းများကို ကျွမ်းကျင်စေပါ | 30 မိနစ် |
| **5. Customization** | ကိုယ့်အတွက် ပြောင်းလဲခြင်း | AI Search, tracing, evaluation ကို ဖွင့်စေပြီး သင့်အစီအစဉ်အတွက် စိတ်တိုင်းမိအောင် ပြုပြင်ပါ | 45 မိနစ် |
| **6. Teardown** | ရှင်းလင်းပစ်ခြင်း | `azd down --purge` ဖြင့် အရင်းအမြစ်များကို လုံခြုံစွာ ဖျက်ပစ်ပါ | 15 မိနစ် |
| **7. Wrap-up** | နောက်တိုးလမ်းညွှန် | အောင်မြင်ချက်များ၊ အဓိက အတွေးအခေါ်များကို ပြန်လည်ဆန်းစစ်၍ ဆက်လက် လေ့လာနိုင်ရန် လမ်းကြောင်းပေးပါ | 15 မိနစ် |

## 📖 အနက်ရှိုင်း လေ့လာမှု: AZD အင်္ဂါရပ်များ

အခြေခံအရာများကို ကျော်လွန်၍ AZD သည် ထုတ်လုပ်ရေးအတွက် အားကောင်းသော လုပ်ဆောင်ချက်များကို ပေးစွမ်းပါသည်။

- **Template-based deployments** - ယေဘူယျ အက်ပလိကေးရှင်း ပုံစံများအတွက် ကြိုတင်ပြင်ဆင်ထားသော ပုံစံများကို အသုံးပြုပါ
- **Infrastructure as Code** - Bicep သို့မဟုတ် Terraform အသုံးပြု၍ Azure အရင်းအမြစ်များကို စီမံခန့်ခွဲပါ  
- **Integrated workflows** - အက်ပလိကေးရှင်းများကို ချက်ချင်း ပြင်ဆင်တပ်ဆင်၊ တင်ပို့နှင့် စောင့်ကြည့်နိုင်ရန် ဆက်လက်ပေါင်းစည်းထားသည်
- **Developer-friendly** - ဖွံ့ဖြိုးရေးသူ ထုတ်လုပ်ရေးစေသော အတွေ့အကြုံနှင့် ထိရောက်မှုအတွက် အထူးပြုပြင်ထားသည်

### **AZD + Microsoft Foundry: AI တင်ပို့မှုအတွက် သင့်တော်သည်**

**AZD သည် AI ဖြေရှင်းချက်များအတွက် မည်သို့အကျိုးရှိသနည်း?** AZD သည် AI ဖွံ့ဖြိုးရေးသူများ ကြုံတွေ့ရသော အဓိက စိန်ခေါ်ချက်များကို ဖြေရှင်းပေးပါသည်။

- **AI-Ready Templates** - Microsoft Foundry Models, Cognitive Services နှင့် ML လုပ်ငန်းများအတွက် ကြိုတင်ပြင်ဆင်ထားသော ပုံစံများ
- **Secure AI Deployments** - AI ဝန်ဆောင်မှုများ၊ API key များနှင့် မော်ဒယ် endpoints များအတွက် အတွင်းပိုင်း လုံခြုံရေး ပုံစံများ ပါဝင်သည်  
- **Production AI Patterns** - တိုးမြင့်နိုင်ပြီး စျေးနှုန်းထိရောက်သော AI အက်ပလိကေးရှင်း တပ်ဆင်မှုများအတွက် အကောင်းဆုံး လမ်းစဉ်များ
- **End-to-End AI Workflows** - မော်ဒယ် ဖွံ့ဖြိုးမှုမှ စ၍ ထုတ်လုပ်မှုသို့ တင်ပို့ခြင်းအထိ၊ သေချာစမ်းသပ်မှုနှင့် စောင့်ကြည့်မှုပါရှိသည်
- **Cost Optimization** - AI လုပ်ငန်းများအတွက် အရင်းအမြစ် ခွဲဝေပေးမှုနှင့် အရွယ်ချိန်ညှိနှိုင်းမှုနည်းဗျူဟာများ
- **Microsoft Foundry Integration** - Microsoft Foundry မော်ဒယ် ကတ်တလော့နှင့် endpoint များနှင့် တိုက်ရိုက် ချိတ်ဆက်နိုင်စေသည်

<details>
<summary><strong>❌ "azd: command not found"</strong></summary>

```bash
# AZD ကို ပထမဦးစွာ တပ်ဆင်ပါ
```

</details>

<details>
<summary><strong>❌ "No subscription found" or "Subscription not set"</strong></summary>

```bash
# ရရှိနိုင်သော စာရင်းသွင်းမှုများကို စာရင်းပြပါ
az account list --output table

# ပုံမှန် စာရင်းသွင်းမှုကို သတ်မှတ်ပါ
az account set --subscription "<subscription-id-or-name>"

# AZD ပတ်ဝန်းကျင်အတွက် သတ်မှတ်ပါ
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# အတည်ပြုပါ
az account show
```
</details>

<details>
<summary><strong>❌ "InsufficientQuota" or "Quota exceeded"</strong></summary>
