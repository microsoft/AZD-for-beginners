## သင်တန်းအကျဉ်းချုပ်

AZD ကို အဆင့်ဆင့် သင်ကြားနိုင်ရန် စနစ်တကျ အခန်းပိုင်းများဖြင့် Azure Developer CLI (azd) ကို ကျွမ်းကျင်ပါ။ **Microsoft Foundry ပေါင်းစည်းမှုဖြင့် AI အသုံးပြုမှုများ စတင် ထုတ်လုပ်ခြင်းအပေါ် အထူးအာရုံစိုက်ထားပါသည်။**

### ဒီသင်တန်းသည် ခေတ်မီ Developer များအတွက် အရေးပါသည်

Microsoft Foundry Discord အသိုင်းအဝိုင်းမှ အကြံပြုချက်များအရ၊ **developer ၄၅% သည် AZD ကို AI အတွက် အသုံးပြုလိုသည်**၊ သို့သော် အခက်အခဲများ ကြုံတွေ့နေသည်-
- မျိုးစုံသော ဝန်ဆောင်မှုများစနစ်ရှုပ်ထွေးမှု
- ထုတ်လုပ်မှုအဆင့် AI အတည်ပြု deployment နည်းလမ်းများ
- Azure AI ဝန်ဆောင်မှု ပေါင်းစည်းမှုနှင့် ကွန်ဖစ်ချ်ရေးရှင်းများ
- AI အလုပ်အတွက် ကုန်ကျစရိတ် Optimize လုပ်ခြင်း
- AI-specific မျိုးသော deployment ပြဿနာများကို ဖြေရှင်းခြင်း

### သင်ယူရမည့် ရည်မှန်းချက်များ

ဤ စနစ်တကျ သင်တန်းကို ပြီးမြောက်ခြင်းဖြင့်-
- **AZD အခြေခံများ ကျွမ်းကျင်ပါမည်**: အခြေခံသဘောတရားများ၊ 설치ခြင်းနှင့် ပြင်ဆင်ခြင်း
- **AI အက်ပလီကေးရှင်းများ ထုတ်လုပ်ပါမည်**: Microsoft Foundry ဝန်ဆောင်မှုများနှင့် AZD ကို အသုံးပြု၍
- **Infrastructure as Code ကို မေးမြန်းပါမည်**: Bicep ကမ်းလှမ်းမှုဖြင့် Azure အရင်းအမြစ်များ စီမံခန့်ခွဲခြင်း
- **Deployment ပြဿနာများ အဖြေရှာပါမည်**: ပုံမှန် ဖြစ်ပေါ်သည့် ပြဿနာများ ကြိုးစားပြီး ပြင်ဆင်ခြင်း
- **ထုတ်လုပ်မှုအထောက်အပံ့များ အတွက် Optimize လုပ်ပါမည်**: လုံခြုံရေး၊ လမ်းရွေ့၊ ကြည့်ရှုမှုနှင့် ကုန်ကျစရိတ် စီမံခန့်ခွဲမှု
- **Multi-Agent ဖြေရှင်းချက်များ ဖန်တီးပါမည်**: စနစ်ရှုပ်ထွေးသော AI စနစ်များ ထုတ်လုပ်ခြင်း

#### သင်ယူဖို့ ဗဟုသုတရရှိစေမည့် ကုဒ်များ

- **🛠️ လက်တွေ့ လေ့ကျင့်မှု**: [ပထမဆုံး ပရောဂျက်](docs/getting-started/first-project.md) - လမ်းညွှန်ထောက်လှမ်းချက်
- **📋 အကြမ်းဖျင်း ညွှန်ကြားချက်**: [Command Cheat Sheet](resources/cheat-sheet.md)

#### လက်တွေ့ လေ့ကျင့်မှုများ
```bash
# အမြန်တပ်ဆင်မှု စစ်ဆေးမှု
azd version

# သင့်ပထမဆုံး အပလီကေးရှင်းကို တင်သွင်းပါ။
azd init --template todo-nodejs-mongo
azd up
```

**💡 အခန်းရလဒ်**: AZD အသုံးပြု၍ အခြေခံ web application တစ်ခုကို Azure သို့ အောင်မြင်စွာ ထုတ်လွှင့်ပါ

**✅ အောင်မြင်မှု အတည်ပြုချက်**:
```bash
# အခန်း ၁ ပြီးဆုံးပြီးနောက်၊ သင်မှာ အောက်ပါအရာများကို ကျွမ်းကျင်နိုင်သင့်သည်။
azd version              # ထည့်သွင်းပြီး ဖြစ်နေသော ဗားရှင်းကို ပြသည်
azd init --template todo-nodejs-mongo  # ပရောဂျက်ကို စတင်တည်ဆောက်သည်
azd up                  # Azure သို့ တင်ပို့သည်
azd show                # လည်ပတ်နေသော အက်ပ်၏ URL ကို ပြသည်
# အက်ပ်ကို ဘရောင်းဇာတွင် ဖွင့်ပြီး အလုပ်လုပ်သည်
azd down --force --purge  # အရင်းအမြစ်များကို စင်ကြယ်အောင် ပြုလုပ်သည်
```

**📊 အချိန်လိုအပ်မှု**: ၃၀-၄၅ မိနစ်  
**📈 အတတ်ပညာ အဆင့်ဖြစ်သည်**: မိမိတိုင် အခြေခံ အက်ပလီကေးရှင်းများ ထုတ်လွှင့်နိုင်ခြင်း

**✅ အောင်မြင်မှု အတည်ပြုချက်**:
```bash
# အပိုင်း ၁ ကိုပြီးစီးပါက၊ သင်သည် အောက်ပါအကြောင်းများကို ပြုလုပ်နိုင်သင့်သည်။
azd version              # ထည့်သွင်းထားသောဗားရှင်းကိုပြသည်
azd init --template todo-nodejs-mongo  # ပရောဂျက်ကိုစတင်တည်ဆောက်ပါ
azd up                  # Azure သို့တင်သွင်းသည်
azd show                # အသက်သွင်းထားသောအက်ပ်၏ URL ကိုပြသည်
# အက်ပ်သည် ဘရောင်ဇာတွင်ဖွင့်ပြီး အလုပ်လုပ်သည်
azd down --force --purge  # အရင်းအမြစ်များကို သန့်စင်သည်
```

**📊 အချိန်လိုအပ်မှု**: ၃၀-၄၅ မိနစ်  
**📈 အတတ်ပညာ အဆင့်ဖြစ်သည်**: မိမိတိုင် အခြေခံ အက်ပလီကေးရှင်းများ ထုတ်လွှင့်နိုင်ခြင်း

## 📖 Azure Developer CLI ဆိုတာဘာလဲ?

Azure Developer CLI (azd) သည် developer များအတွက် အလျင်အမြန် အက်ပ်များကို Azure သို့ တည်ဆောက် ထည့်သွင်းရာတွင် အထောက်အကူပြုစေသော command-line interface ဖြစ်သည်။ ၎င်းသည်-

- **ပုံစံအခြေပြု ထည့်သွင်းမှုများ** - ပုံမှန် အက်ပ်ပေါ်တွင် အသုံးပြုရန် ကြိုတင် ဖွဲ့စည်းထားသည့် ပုံစံများ
- **Infrastructure as Code** - Bicep သို့မဟုတ် Terraform ဖြင့် Azure အရင်းအမြစ်များစီမံခန့်ခွဲခြင်း  
- **ပေါင်းစပ်လုပ်ဆောင်မှုများ** - အက်ပ်များကို အဆက်မပြတ် ထည့်သွင်း၊ စောင့်ကြည့်နှင့် စနစ်တကျ ကိုင်တွယ်နိုင်ခြင်း
- **Developer အကြိုက်ဆုံး ရည်ရွယ်ချက်** - developer အလုပ်ထိရောက်မှုနှင့် ခံစားမှု အတွက် ထိရောက်စွာ တင်ဆက်ထားခြင်း

### **AZD + Microsoft Foundry: AI ထည့်သွင်းမှုအတွက် ပြည့်စုံသော နည်းလမ်း**

**AI ဖြေရှင်းချက်များအတွက် AZD ကို ဘာကြောင့်ထုံးစံသုံးသလဲ?** AZD သည် AI developer များ ကြုံတွေ့ရသည့် ထိပ်တန်း စိန်ခေါ်မှုများ ပေါ် မူတည်၍ ဖြေရှင်းပေးသည်-

## 🔧 အမြန် ပြဿနာဖြေရှင်းလမ်းညွှန်

**စတင်လေ့လာသူများ တွေ့ကြုံရသော အပြည့်အဝပြဿနာများနှင့် ချက်ချင်း ဖြေရှင်းနည်းများ:**

### ❌ "azd: command not found"

```bash
# ပထမဆုံး AZD ကို ထည့်သွင်းပါ
# Windows (PowerShell):
winget install microsoft.azd

# macOS:
brew tap azure/azd && brew install azd

# Linux:
curl -fsSL https://aka.ms/install-azd.sh | bash

# ထည့်သွင်းမှုကို စစ်ဆေးပါ
azd version
```

### ❌ "No subscription found" or "Subscription not set"

```bash
# ရနိုင်သော စာရင်းသွင်းမှုများကို စာရင်းပြုစုပါ
az account list --output table

# ပုံမှန်စာရင်းသွင်းမှု စတင်သတ်မှတ်ပါ
az account set --subscription "<subscription-id-or-name>"

# AZD ပတ်ဝန်းကျင်အတွက် သတ်မှတ်ပါ
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# စစ်ဆေးပါ
az account show
```

### ❌ "InsufficientQuota" or "Quota exceeded"

```bash
# အမျိုးမျိုးသော Azure ဒေသများကို စမ်းသပ်ကြည့်ပါ
azd env set AZURE_LOCATION "westus2"
azd up

# သို့မဟုတ် ဖွံ့ဖြိုးရေးတွင် အသေးစား SKUs များကို အသုံးပြုပါ
# infra/main.parameters.json ကို တည်းဖြတ်ပါ:
{
  "sku": "B1"  // Instead of "P1V2"
}
```

### ❌ "azd up" fails halfway through

```bash
# ရွေးချယ်စရာ ၁: သန့်ရှင်းပြီး ထပ်မံကြိုးစားပါ
azd down --force --purge
azd up

# ရွေးချယ်စရာ ၂: အောက်ခံအဆောက်အအုံကိုသာ ပြင်ဆင်ပါ
azd provision

# ရွေးချယ်စရာ ၃: အသေးစိတ် မှတ်တမ်းများကို စစ်ဆေးပါ
azd show
azd logs
```

### ❌ "Authentication failed" or "Token expired"

```bash
# ပြန်လည်အတည်ပြုခြင်း
az logout
az login

azd auth logout
azd auth login

# အတည်ပြုမှုကို စစ်ဆေးပါ
az account show
```

### ❌ "Resource already exists" or naming conflicts

```bash
# AZD သည် ထူးခြားသောနာမည်များ ထုတ်လုပ်ပေးသော်လည်း၊ ပဋိပက္ခရှိပါက။
azd down --force --purge

# ထိုအခါ မျက်နှာပြင်အသစ်ဖြင့် ထပ်မံကြိုးစားပါ။
azd env new dev-v2
azd up
```

### ❌ Template deployment taking too long

**ပုံမှန် စောင့်ဆိုင်းချိန်များ:**
- လွယ်ကူသရုပ် ပြု လုပ်ငန်း: ၅-၁၀ မိနစ်  
- ဒေတာဘေ့စ် ပါဝင်သော အက်ပ်: ၁၀-၁၅ မိနစ်

ပူးပေါင်းဆောင်ရွက်မှုများကို ကြိုဆိုပါသည်။ အသေးစိတ်အတွက် ကျေးဇူးပြု၍ [ပူးပေါင်းဆောင်ရွက်ခြင်း လမ်းညွှန်](CONTRIBUTING.md) ကို ဖတ်ပါ၍:

- **ဥပမာအသစ်များ ထည့်သွင်းခြင်း**: အမှန်တကယ် အသုံးပြုနိုင်သော အခြေအနေများနှင့် နမူနာများ  
- **ဘာသာပြန်မှု**: မျိုးစုံဘာသာအထောက်အပံ့ ထိန်းသိမ်းရန်  
- **အမှားအစီရင်ခံမှု**: မှန်ကန်မှုနှင့် ရှင်းလင်းမှု တိုးတက်အောင်  
- **အသိုင်းအဝိုင်း စံသတ်မှတ်ချက်များ**: ကျွန်ုပ်တို့၏ ထည့်သွင်းသဘောထား အသိုင်းအဝိုင်း လမ်းညွှန်ချက်များကို လိုက်နာခြင်း
