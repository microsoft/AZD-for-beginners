# အခန်း ၃: ဆက်တင်များနှင့် အတည်ပြုခြင်း

**📚 သင်တန်း**: [AZD စတင်သူများအတွက်](../../README.md) | **⏱️ ကြာချိန်**: 45-60 မိနစ် | **⭐ ခက်ခဲမှု**: အလယ်အလတ်

---

## အတိုချုံး

ဤအခန်းတွင် Azure Developer CLI ဖြင့် တပ်ဆင်ခြင်းများအတွက် ပတ်ဝန်းကျင် ဆက်တင်များ၊ အတည်ပြုခြင်း မျိုးစနစ်များနှင့် လုံခြုံရေး အကောင်းဆုံးကျင့်ဝတ်များကို ဖော်ပြထားသည်။

> သက်ဆိုင်စစ်ဆေးပြီး `azd 1.25.6` နှင့် June 2026 တွင် အတည်ပြုထားသည်။

## သင်ယူရမည့် ရည်မှန်းချက်များ

ဤအခန်းကို ပြီးမြောက်လျှင် သင်သည်
- AZD ဆက်တင်အဆင့်တန်းကို ကျွမ်းကျင်နိုင်ပါမည်
- ပတ်ဝန်းကျင်များစွာ (dev, staging, prod) ကို စီမံခန့်ခွဲနိုင်ပါမည်
- managed identities ဖြင့် လုံခြုံသော အတည်ပြုမှုကို အကောင်အထည်ဖော်နိုင်ပါမည်
- ပတ်ဝန်းကျင်အလိုက် ဆက်တင်များကို ပြင်ဆင်နိုင်ပါမည်

## 🚀 လျင်မြန် စတင်ခြင်း

```bash
# ပတ်ဝန်းကျင်များစွာ ဖန်တီးပါ
azd env new dev
azd env new staging
azd env new prod

# ပတ်ဝန်းကျင်ကို ပြောင်းပါ
azd env select prod

# ပတ်ဝန်းကျင် တန်ဖိုးများ သတ်မှတ်ပါ
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# ဖွဲ့စည်းပုံကို ကြည့်ရှုပါ
azd env get-values
```

---

## 🔧 ဆက်တင် အဆင့်အစီအစဉ်

AZD သည် အောက်ပါ အဆင့်စဉ်အတိုင်း ဆက်တင်များကို အသုံးပြုသည် (နောက်ပိုင်းတွင် သတ်မှတ်ထားသော တန်ဖိုးသည် ယခင်တန်ဖိုးကို အစားထိုးပါသည်)။

1. **ပုံမှန်တန်ဖိုးများ** (template များထဲတွင် ထည့်သွင်းထားသည်)
2. **azure.yaml** (ပရောဂျက် ဆက်တင်)
3. **ပတ်ဝန်းကျင်ပြောင်းလဲမှုများ** (`azd env set`)
4. **အမိန့်လိုင်း အရွေးများ** (`--location eastus`)

---

## 🔐 လုံခြုံရေး အကောင်းဆုံး လမ်းညွှန်ချက်များ

```bash
# Managed Identity ကို အသုံးပြုပါ (အကြံပြုသည်)
azd env set AZURE_USE_MANAGED_IDENTITY true

# AZD အတည်ပြုမှု အခြေအနေကို စစ်ဆေးပါ
azd auth status

# ရွေးချယ်စရာ: az ကော်မန်ဒ်များကို လုပ်ဆောင်ရန် ရည်ရွယ်ပါက Azure CLI context ကို အတည်ပြုပါ
az account show

# လိုအပ်ပါက ပြန်လည်အတည်ပြုပါ
azd auth login

# ရွေးချယ်စရာ: az ကော်မန်ဒ်များအတွက် Azure CLI အတည်ပြုမှုကို ပြန်လည်အသစ်ပြုလုပ်ပါ
az login
```
