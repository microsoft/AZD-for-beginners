# အခန်း ၃: ဖွဲ့စည်းမှုနှင့် အတည်ပြုမှု

**📚 သင်တန်း**: [AZD For Beginners](../../README.md) | **⏱️ ကြာချိန်**: 45-60 မိနစ် | **⭐ ရှုပ်ထွေးမှု**: အလယ်အလတ်

---

## အနှစ်ချုပ်

ဤအခန်းတွင် Azure Developer CLI ဖြင့် ဖြန့်ချိမှုများအတွက် ပတ်ဝန်းကျင် ဆက်တင်များ၊ အတည်ပြုမှု ပုံစံများနှင့် လုံခြုံရေးအတွက် အကောင်းဆုံး လက်တွေ့နည်းလမ်းများကို ဖေါ်ပြပါသည်။

> ဒီစာတမ်းကို `azd 1.23.12` ဖြင့် ၂၀၂၆ မတ်လတွင် စစ်ဆေးထောက်ခံထားသည်။

## သင်ယူရမည့် ရည်ရွယ်ချက်များ

ဤအခန်းကို ပြီးမြောက်ပါက သင်အောက်ပါများကို ရရှိမည်။

- AZD ဆက်တင် အဆင့်စဉ်ကို ကျွမ်းကျင်စွာ အသုံးပြုနိုင်ရန်
- အမျိုးမျိုးသော ပတ်ဝန်းကျင်များ (dev, staging, prod) ကို စီမံခန့်ခွဲနိုင်ရန်
- managed identities ဖြင့် လုံခြုံသော အတည်ပြုမှုကို တပ်ဆင်နိုင်ရန်
- ပတ်ဝန်းကျင်အလိုက် ဆက်တင်များကို ဖော်ပြုနိုင်ရန်

## 🚀 အမြန်စတင်

```bash
# အမျိုးမျိုးသော ပတ်ဝန်းကျင်များ ဖန်တီးပါ
azd env new dev
azd env new staging
azd env new prod

# ပတ်ဝန်းကျင်များကို ပြောင်းပါ
azd env select prod

# ပတ်ဝန်းကျင်ဆက်တင်များကို သတ်မှတ်ပါ
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# ဖွဲ့စည်းမှုကို ကြည့်ပါ
azd env get-values
```

---

## 🔧 ဆက်တင် အဆင့်စဉ်

AZD သည် ဆက်တင်များကို အောက်ပါ အစီအစဉ်အတိုင်း အသုံးပြုသည် (နောက်ပိုင်း သတ်မှတ်ချက်များသည် ယခင် အချက်များကို အစားထိုးသည်):

1. **ပုံမှန်တန်ဖိုးများ** (နမူနာဖိုင်များတွင် ထည့်သွင်းထားသည်)
2. **azure.yaml** (ပရောဂျက် ဖွဲ့စည်းမှု)
3. **ပတ်ဝန်းကျင် တန်ဖိုးများ** (`azd env set`)
4. **Command-line ရွေးချယ်မှုများ** (`--location eastus`)

---

## 🔐 လုံခြုံရေးအတွက် အကောင်းဆုံး လက်တွေ့နည်းလမ်းများ

```bash
# Managed identity ကို အသုံးပြုပါ (အကြံပြု)
azd env set AZURE_USE_MANAGED_IDENTITY true

# AZD အတည်ပြုမှု အခြေအနေကို စစ်ဆေးပါ
azd auth status

# ရွေးချယ်စရာ: az command များကို လုပ်ဆောင်ရန် ရည်ရွယ်ပါက Azure CLI context ကို စစ်ဆေးပါ
az account show

# လိုအပ်ပါက ပြန်လည် အတည်ပြုပါ
azd auth login

# ရွေးချယ်စရာ: az command များအတွက် Azure CLI အတည်ပြုချက်ကို ပြန်လည်အသစ်ပြုလုပ်ပါ
az login
```
