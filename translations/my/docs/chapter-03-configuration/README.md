# အခန်း ၃: ဖွဲ့စည်းမှုနှင့် အတည်ပြုခြင်း

**📚 သင်တန်း**: [AZD အစပြုသူများ](../../README.md) | **⏱️ ကြာချိန်**: 45-60 မိနစ် | **⭐ ความซับซ้อน**: အလယ်အလတ်

---

## အကျဉ်းချုပ်

ဤအခန်းတွင် Azure Developer CLI ဖြင့် ထုတ်လုပ်မှုဆိုင်ရာ deployments များအတွက် ပတ်ဝန်းကျင် ဖွဲ့စည်းခြင်း၊ အတည်ပြုမှု ပုံစံများနှင့် လုံခြုံရေး အကောင်းဆုံး လုပ်ထုံးလုပ်နည်းများကို ဖော်ပြပါသည်။

## သင်ယူရမည့် ရည်ရွယ်ချက်များ

ဤအခန်းကို ပြီးမြောက်ပါက သင်သည် -
- AZD ဖွဲ့စည်းမှု အဆင့်စဉ်ကို ကျွမ်းကျင်စွာ အသိပညာ ရရှိမည်
- ဘာသာရပ်ပတ်ဝန်းကျင်များစွာ (dev, staging, prod) ကို စီမံခန့်ခွဲနိုင်မည်
- managed identities ဖြင့် လုံခြုံစွာ အတည်ပြုနိုင်မည်
- ပတ်ဝန်းကျင်ဆိုင်ရာ သတ်မှတ်ချက်များကို ဖွဲ့စည်းနိုင်မည်

## 🚀 အမြန် စတင်ရန်

```bash
# ပတ်ဝန်းကျင်များ အများအပြား ဖန်တီးပါ
azd env new dev
azd env new staging
azd env new prod

# ပတ်ဝန်းကျင်များအကြား ပြောင်းရွှေ့ပါ
azd env select prod

# ပတ်ဝန်းကျင်ပြောင်းလဲတတ်သော တန်ဖိုးများကို သတ်မှတ်ပါ
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# ဖွဲ့စည်းမှုကို ကြည့်ပါ
azd env get-values
```

---

## 🔧 ဖွဲ့စည်းမှု အဆင့်စဉ်

AZD သည် ဤအောက်ပါ အစီအစဉ်အတိုင်း ဆက်တင်များကို အသုံးပြုပါသည် (နောက်ပိုင်းရှိတန်ဖိုးများသည် ရှေ့ပိုင်းရှိတန်ဖိုးများကို override လုပ်မည်)။

1. **မူရင်းတန်ဖိုးများ** (templates တွင် ထည့်သွင်းထားသည်)
2. **azure.yaml** (ပရိုဂျက် ဖော်ပြမှု)
3. **ပတ်ဝန်းကျင် အပြောင်းအလဲများ** (`azd env set`)
4. **ကွန်မန့်လိုင်း ဖလက်များ** (`--location eastus`)

---

## 🔐 လုံခြုံရေး အကောင်းဆုံး လုပ်ထုံးလုပ်နည်းများ

```bash
# managed identity ကို အသုံးပြုပါ (အကြံပြု)
azd env set AZURE_USE_MANAGED_IDENTITY true

# အတည်ပြုခြင်း အခြေအနေကို စစ်ဆေးပါ
azd auth whoami
az account show

# လိုအပ်ပါက ပြန်လည် အတည်ပြုပါ
azd auth login
az login
```
