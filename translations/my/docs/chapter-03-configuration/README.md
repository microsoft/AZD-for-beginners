# အခန်း ၃: စီမံခန့်ခွဲမှုနှင့် အတည်ပြုခြင်း

**📚 သင်တန်း**: [AZD အစောပိုင်းများအတွက်](../../README.md) | **⏱️ ကြာမြင့်ချိန်**: ၄၅-၆၀ မိနစ် | **⭐ အဆင့်ခက်ခဲမှု**: အလယ်အလတ်

## အမြင်အထွေထွေ

ဤအခန်းတွင် Azure Developer CLI အသုံးပြုမှုများအတွက် ပတ်ဝန်းကျင် စီမံခန့်ခွဲမှု၊ အတည်ပြုမှု များနှင့် ဘေးကင်းလုံခြုံမှု အကောင်းဆုံးလေ့လာမှုများကို ဖော်ပြထားသည်။

> `azd 1.27.1` ဖြင့် ၂၀၂၆ ခုနှစ် ဇူလိုင်လတွင် စစ်ဆေးပြီး။

## သင်ယူရန် ရည်ရွယ်ချက်များ

ဤအခန်းကို ပြီးမြောက်စဉ်၊ သင်သည်-
- AZD စီမံခန့်ခွဲမှု အဆင့်လိုက်ဖွဲ့စည်းမှုကို ကျွမ်းကျင်မည်
- ပတ်ဝန်းကျင် အမျိုးမျိုး (dev, staging, prod) ကို စီမံခန့်ခွဲမည်
- စီမံခန့်ခွဲထားသော အတည်ပြုမှုဖြင့် လုံခြုံမှုကို အကောင်အထည်ဖော်မည်
- ပတ်ဝန်းကျင် အထူးပြု ဆက်တင်များကို ဖွင့်တပ်မည်

## 🚀 အမြန်စတင်ရန်

```bash
# ပတ်ဝန်းကျင်များ အများအပြား ဖန်တီးပါ
azd env new dev
azd env new staging
azd env new prod

# ပတ်ဝန်းကျင်များ ပြောင်းပါ
azd env select prod

# ပတ်ဝန်းကျင် အပြောင်းအလဲများ သတ်မှတ်ပါ
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# ဖွဲ့စည်းမှုကို ကြည့်ပါ
azd env get-values
```

---

## 🔧 စီမံခန့်ခွဲမှု အဆင့်လိုက် ဖွဲ့စည်းမှု

AZD သည် အောက်ပါ အမှာစာအတိုင်း ဆက်တင်များကို သတ်မှတ်သည် (နောက်ဆုံး သတ်မှတ်ချက်သည် ပထမဆုံးကိုမက်ပြီး):

၁။ **ပုံမှန်တန်ဖိုးများ** (တမ်းပလိတ်များ၌ ပေါင်းထည့်ထားသည်)
၂။ **azure.yaml** (ပရောဂျက် စီမံခန့်ခွဲမှု)
၃။ **ပတ်ဝန်းကျင် အပြောင်းအလဲများ** (`azd env set`)
၄။ **command-line အလံများ** (`--location eastus`)

---

## 🔐 လုံခြုံရေး အကောင်းဆုံးလေ့လာမှုများ

```bash
# စီမံခန့်ခွဲထားသော မှတ်ပုံတင်ကို အသုံးပြုပါ (အကြံပြု)
azd env set AZURE_USE_MANAGED_IDENTITY true

# AZD အတည်ပြုမှု အခြေအနေကို စစ်ဆေးပါ
azd auth status

# ရွေးချယ်စရာ- az command များကို လည်ပတ်ရန် ရည်ရွယ်ပါက Azure CLI context ကို သေချာစစ်ဆေးပါ
az account show

# လိုအပ်ပါက ပြန်လည်အတည်ပြုပါ
azd auth login

# ရွေးချယ်စရာ- az command များအတွက် Azure CLI အတည်ပြုမှုကို ပြန်လည်ပြောင်းလဲပါ
az login
```
