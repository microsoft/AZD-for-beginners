- **📖 လက်ရှိအခန်း**: အခန်း ၁ - အခြေခံ နှင့် အမြန်စတင်ခြင်း
- **⬅️ ယခင်**: [သင်၏ ပထမဆုံးပရောဂျက်](first-project.md)

> `azd 1.27.1` နဲ့ ဇူလိုင် ၂၀၂၆ တွင် စစ်ဆေးပြီး ဖြစ်ပါသည်။

## နိဒါန်း

[သင်၏ ပထမဆုံးပရောဂျက်](first-project.md) မှာ template တစ်ခုမှ စတင်ပြီး အက်ပ်တစ်ခုကို ဖြန့်ချိခြင်း ပြုလုပ်ခဲ့သည်။ ဒါပေမယ့် မကြာခဏ သင်မှာ Node.js API, Python Flask ဝန်ဆောင်မှု, .NET ဝဘ်အက်ပ် တို့ကဲ့သို့ ရှိပြီးသားအက်ပ် တစ်ခုရှိပြီး၊ သင်၏ ကွန်ပျူတာအတွင်း ဖိုင်ဖိုဒါတွင် ရှိနေပါသည်။ ဒီစာသင်ခန်းမှာ azd ကို အဲဒီ ရှိပြီးသားကုဒ်ထဲ ထည့်သွင်းပြီး `azd up` နဲ့ ဖြန့်ချိနိုင်ဖို့ ပြသပေးပါမယ်၊ template မလိုအပ်ပါ။

## သင်ယူလိုသည့် ရည်မှန်းချက်များ

ဒီသင်ခန်းပြီးဆုံးချိန်မှာ သင်သည်
- azd ပရောဂျက်စတင်နိုင်တဲ့နည်းလမ်း သုံးခုကို နားလည်သွားမည်
- ရှိပြီးသားကုဒ်အတွင်း `azd init` ကို run လုပ်နိုင်မည်
- `azure.yaml` နှင့် `infra/` ဖိုဒါက သင့်အက်ပ်အတွက် ဘာတွေ လုပ်ဆောင်ပေးသလဲ နားလည်မည်
- azd က infrastructure ကို ဘယ်အချိန်မှာ generate ပြုလုပ်သင့်ပြီး ဘယ္အချိန်မှာ ကိုယ့်အတိုင်ရေးသင့် သဘောနားလည်မည်
- ရှိပြီးသားအက်ပ်ကို Azure ပေါ်သို့ `azd up` ဖြင့် ဖြန့်ချိနိုင်မည်

## သင်ယူပြီးရလဒ်များ

ဒီသင်ခန်းပြီးဆုံးမှသာ သင်သည်:
- ရှိပြီးသား ပရောဂျက်ထဲ azd ကို မိတ်ဆက်နိုင်မည်
- မူလ `azure.yaml` ဖိုင်ကို ဖတ်ရှု ပြင်ဆင်နိုင်မည်
- `azd infra generate` ဖြင့် စတင်အသုံးပြုရန် infrastructure ကို generate ပြုလုပ်နိုင်မည်
- သင့်အက်ပ်အတွက် သင့်လျော်သော Azure host ကို ရွေးချယ်နိုင်မည်
- ကိုယ်ပိုင်အက်ပ်ကို ဖြန့်ချိပြီး သန့်ရှင်းရေး ပြုလုပ်နိုင်မည်

## azd ပရောဂျက် စတင်နိုင်သည့် နည်းလမ်း သုံခု

| စတင်ရာနေရာ | ကောင်မန်း | အသုံးပြုရမည့်အချိန် |
|----------------|---------|-------------|
| **Template မှစ၍** | `azd init --template <name>` | သင်ယူခြင်း သို့မဟုတ် ပ probado အသုံးပြုရခြင်း အက်ပ်အသစ် တစ်ခုစတင်ခြင်း |
| **ရှိပြီးသားကုဒ်မှ** | `azd init` (သင့်ပရောဂျက်ဖိုဒါတွင်) | သင့်ထဲမှာ အက်ပ်ရှိပြီး ဖြန့်ချိလိုသည့်အခါ |
| **Git repo မှ** | `azd init --from-code` (clone များ repo တွင်) | ရှိပြီးသား repository တစ်ခုအတွက် azd ကို အသုံးပြုမည် |

သင်သည် ပထမဆုံးရွေးချယ်မှုကို လေ့ကျင့်ပြီး ဖြစ်သည်။ ဒီသင်ခန်းမှာ ဒုတိယနည်းကို ဖော်ပြပါမည် - လူများအသုံးပြုရလွယ်ကူဆုံး နောက်ခံဖြစ်သောအခြေအနေဖြစ်သည်။

## အဆင့် ၁: သင့်ပရောဂျက်တွင်း `azd init` ကို run လုပ်ပါ

သင့် ရှိပြီးသားပရောဂျက်ဖိုဒါအတွင်း ရောက်ဖို့ terminal ကိုဖွင့်ပြီး အောက်က ကောင်မန်းကို run လုပ်ပါ။

```bash
cd my-existing-app
azd init
```

azd သင့်ကို မိတ်ဆက်နည်းရွေးချယ်ရန် မေးမြန်းပါလိမ့်မယ်။ ရွေးချယ်ပါ။

```
? How do you want to initialize your app?
> Use code in the current directory
  Select a template
```

**"Use code in the current directory."** ကိုရွေးချယ်ပါ။ azd က သင့်ဖိုဒါကို scan လုပ်ပြီး သင့်ဘာသာစကားနှင့် framework ကို တွေ့ရှိပီး host တစ်ခုကို အကြံပြုပါလိမ့်မယ်။

### azd ရှာဖွေသော အကြောင်းအရာများ

azd က `package.json`, `requirements.txt`, `pom.xml`, `*.csproj`, သို့မဟုတ် `Dockerfile` ကဲ့သို့ အချက်အလက်များကို ရှာဖွေပြီး ထို နှိုင်းယှဉ်တိုက်ဆိုင်သော Azure host ကို အကြံပြုသည်။

| သင့်အက်ပ် | ထင်ရှားစွာ တွေ့ရှိသည့် host |
|----------|----------------------|
| Node.js / Python / .NET ဝဘ်အက်ပ် | Azure App Service သို့မဟုတ် Container Apps |
| Containerized အက်ပ် (`Dockerfile`) | Azure Container Apps |
| Function app | Azure Functions |
| Static site (React/Vue build output) | Azure Static Web Apps |

ဆောက်ရွက်မှုတွေကို အတည်ပြုပြီး azd က သင့်လိုအပ်သောဖိုင်များကို စုခြင်းပြုလုပ်ပါလိမ့်မယ်။

## အဆင့် ၂: azd မှ ဖန်တီးထားသည့်အရာများကို နားလည်ပါ

init ပြီးနောက် သင့်ပရောဂျက်၌ အရာများနှစ်ခုအသစ် ရရှိမည်ဖြစ်သည်။

```
my-existing-app/
├── azure.yaml          # ← NEW: tells azd what to deploy and where
├── infra/              # ← NEW: Infrastructure as Code (Bicep by default)
│   ├── main.bicep
│   └── main.parameters.json
├── src/ (your code)    # unchanged
└── ...                 # your existing files, untouched
```

### `azure.yaml` — ပရောဂျက်အဓိပ္ပါယ်ဖိုင်

အဲဒါက azd ပရောဂျက်ရဲ့ ဦးနှောက်ပါ။ အနည်းဆုံး ရှိသင့်သည့်ပုံစံက ဒီလို ကြည့်ရပါမယ်။

```yaml
# azure.yaml
name: my-existing-app
services:
  web:
    project: ./src           # path to your app code
    language: js             # js | python | dotnet | java
    host: appservice         # appservice | containerapp | function | staticwebapp
```

`services` block က အဓိကပါဝင်ပစ္စည်းဖြစ်သည်။ အဲဒီမှာ တစ်ခုချင်းစီက သင့်ကုဒ်ဖိုဒါတစ်ခုကို Azure host တစ်ခုနဲ့ ချိတ်ဆက်ထားပါတယ်။ သင့်အက်ပ်မှာ frontend နဲ့ API နှစ်ခုလုံးပါက service နှစ်ခုရှိမယ်။

### `infra/` — သင့်အက်ပ်အတွက် Azure စက်ရုပ်တွေ အနေနဲ့ကုဒ်ရေးသားမှု

`infra/` ဖိုဒါထဲမှာ သင့်အက်ပ်လိုအပ်သော Azure အရင်းအမြစ်များ (App Service, database စသည်) ရည်ညွှန်းပေးသည့် Bicep ဖိုင်များကို ထည့်ထားသည်။ သင့်ကိုယ်တိုင် လက်ဖြင့်ရေးဖို့မလိုပါ - azd က အလုပ်လုပ်တဲ့ စတင်ချက်ကို auto generate လုပ်ပေးသည်။ ဆက်လက်ပြီး resource တွေ ထပ်ထည့်ရန် သို့မဟုတ် လုံခြုံရေး ကောင်းမွန်မှု တိုးတက်စေရန် ပြင်ဆင်နိုင်သည် ([အခန်း ၄](../chapter-04-infrastructure/README.md) တွင် ဖော်ပြထားသည်)။

> **အကြံပြုချက်:** ဖြန့်ချိမပြုမီ ဖန်တီးထားသော infrastructure ကို ကြည့်ရှု သို့မဟုတ် ျပင်ဆင်လိုပါက `azd infra generate` (သို့မဟုတ် `azd infra synth`) ကို run လိုက်ပါ။ IaC ကို disk ပြင်ပသို့ ရေးသားပြီး စစ်ဆေး၊ version control ပြုလုပ်နိုင်အောင် ပြုလုပ်ပါ။

## အဆင့် ၃: လိုအပ်သော ဆက်တင်များ သတ်မှတ်ပါ

သင့်အက်ပ်တွင် ဆက်တင်များ သို့မဟုတ် လျှို့ဝှက်ချက်များ (connection string, API key) လိုအပ်ပါက တိုက်ရိုက်ကုဒ်တွင် ထည့်သွင်းမထားပါနဲ့။ Environment value များကို အသုံးပြုပါ။

```bash
# ပတ်ဝန်းကျင်တစ်ခုဖန်တီးပါ
azd env new dev

# ဝှက်ထားမှုမရှိသောတန်ဖိုးတစ်ခုကိုသတ်မှတ်ပါ
azd env set API_VERSION 1.0.0
```

လျှို့ဝှက်ချက်များအတွက် Key Vault တွင် သိမ်းဆည်းပြီး သင့် infrastructure မှ ရည်ညွှန်းပါ - [အခန်း ၃: ဆက်တင်ညှိနှိုင်းခြင်းနှင့် အတည်ပြုခြင်း](../chapter-03-configuration/authsecurity.md) ကို ကြည့်ရှုနိုင်သည်။

## အဆင့် ၄: ဖြန့်ချိခြင်း

ယခု သင်သည် ရှိပြီးသား အသုံးပြုနေပြီး workflow အတူတူကို အသုံးပြုနိုင်ပြီဖြစ်သည်။

```bash
# အတည်ပြုခြင်း (azd အတွက် လိုအပ်သည်)
azd auth login

# ဖန်တီးမည့် အရင်းအမြစ်များကို ကြိုတင်ကြည့်ရှုပါ
azd provision --preview

# အခြေခံအဆောက်အအုံ ထားရှိပြီး သင်၏ကုဒ်ကို တပ်ဆင်ပါ
azd up
```

ပြီးဆုံးချိန်တွင် azd က သင့်အက်ပ်၏ URL ကို ပြမည်။ အဲဒီ URL ကို azd အက်ပ်အားလုံးကဲ့သို့စမ်းသပ်လေ့လာပါ။

```bash
azd show           # endpoint များပြပါ
azd monitor --logs # လိုအပ်ပါက မှတ်တမ်းများစစ်ဆေးပါ
```

## ပထမဆုံးကြုံတွေ့ရသည့် ပြဿနာများ

| ရောဂါ လက္ခဏာ | ဖြစ်ပေါ်ခြင်းဖြစ်စေသော အကြောင်း | ဖြေရှင်းနည်း |
|---------|--------------|-----|
| azd က သင့်အက်ပ်ကို မတွေ့ရှိနိုင်ပါ | manifest မရှိခြင်း (ဥပမာ-`package.json`) | manifest ထည့်ရန်၊ သို့မဟုတ် `azd init` မှ Host ကို ကိုယ်တိုင်ရွေးပါ |
| `azd up` ကြာရှည် build မအောင်မြင်ခြင်း | အက်ပ်မှာ build အဆင့် လိုအပ်ခြင်း | `azure.yaml` မှာ service အောက် `buildCommand`/`outputPath` ကို ထည့်ပါ |
| အက်ပ် စတင်ပြီးနောက် error ပြန်တာ | config/secret မရှိခြင်း | `azd env set` ဖြင့် တန်ဖိုးထားပါ သို့မဟုတ် Key Vault ကို ချိတ်ဆက်ပါ |
| မမှန်ကန်သော host ရွေးချယ်ခံရခြင်း | auto-detection မှားယွင်းခြင်း | `azure.yaml` အတွင်း `host:` ကိုပြင်ပြီး `azd up` ကို ပြန် run ပါ |

ပိုမိုသိရှိလိုပါက [အခန်း ၇: ပြဿနာဖြေရှင်းခြင်း](../chapter-07-troubleshooting/README.md) ကို ကြည့်ပါ။

## အနှစ်ချုပ်

- `azd init` → **"Use code in the current directory"** က သင့်မှာ ရှိပြီးသား app ထဲ azd ထည့်သွင်းသည်။
- `azure.yaml` က သင့်ကုဒ်ဖိုဒါများကို Azure hosts နဲ့ ချိတ်ဆက်သည်။ `infra/` မှာ resource များကို Bicep ဖြင့် သတ်မှတ်ထားသည်။
- `azd infra generate` သည် ဖန်တီးထားသော infrastructure ကို ပြန်လည်ပြင်ဆင်ရန် သို့မဟုတ် စစ်ဆေးရန် ခွင့်ပြုသည်။
- မိတ်ဆက်ပြီးနောက် သင့် ရှိပြီးသားအက်ပ်သည် template-based app တစ်ခုလို `azd up` / `azd down` workflow ကို သရုပ်ဆောင်သည်။
