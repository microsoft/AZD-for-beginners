# အခန်း 6: ဖြန့်ချိမတိုင်ခင် စီမံခြင်းနှင့် အတည်ပြုခြင်း

## အကျဉ်းချုပ်

ဤအခန်းတွင် သင့် အက်ပလီကေးရှင်းကို ဖြန့်ချိမတိုင်ခင် လုပ်ဆောင်သင့်သည့် အရေးပါသော စီမံရေးစဉ်များနှင့် အတည်ပြုခြင်း အဆင့်များကို ဖော်ပြထားသည်။ တိုက်ရိုက်စွာ ကျသင့်စရာများမဖြစ်စေရန် စွမ်းရည် စီမံခြင်း၊ SKU ရွေးချယ်မှုနှင့် preflight စစ်ဆေးမှုများကို မှန်ကန်စွာ လုပ်ဆောင်ခြင်းဖြင့် အမှားကြီးများကို ရှောင်ကြဉ်နိုင်မည်ဖြစ်သည်။

## သင်ယူရမည့် ရည်မှန်းချက်များ

ဤအခန်းကို ပြီးမြောက်လျှင် သင်သည်၊
- ဖြန့်ချိမတိုင်ခင် preflight စစ်ဆေးမှုများ ပြုလုပ်နိုင်မည်
- စွမ်းရည်ကို စီမံပြီး အရင်းအမြစ် လိုအပ်ချက်များ ခန့်မှန်းနိုင်မည်
- ကုန်ကျစရိတ် ထိရောက်စေရန် သင့်တော်သော SKU များ ရွေးချယ်နိုင်မည်
- စောင့်ကြည့်မှုအတွက် Application Insights ကို ပြင်ဆင်နိုင်မည်
- အဖွဲ့ ပူးပေါင်းလုပ်ဆောင်မှု ပုံစံများကို နားလည်နိုင်မည်

## 🚀 အမြန်စတင်ခြင်း

```bash
# Subscription အကန့်အသတ်များကို စစ်ဆေးပါ
az vm list-usage --location eastus --output table

# တပ်ဆင်မှုကို ကြိုကြည့်ရှုခြင်း (ရင်းမြစ်များ မဖန်တီးပါ)
azd provision --preview

# Bicep စာလုံးပုံစံကို စစ်ဆေးပါ
az bicep build --file infra/main.bicep

# ပတ်ဝန်းကျင် ဖွဲ့စည်းမှုကို စစ်ဆေးပါ
azd env get-values
```

## ☑️ ဖြန့်ချိမတိုင်ခင် စစ်ဆေးစာရင်း

### `azd provision` မပြုမီ

- [ ] ဒေသအတွက် Quota ကို စစ်ဆေးပြီး
- [ ] SKU များကို သင့်တော်စွာ ရွေးချယ်ပြီး
- [ ] ကုန်ကျစရိတ် ခန့်မှန်းချက်ကို ပြန်လည်စိစစ်ပြီး
- [ ] နာမည်ပေးစနစ်ကို တစ်ထပ်တည်း လိုက်နာထားသည်
- [ ] လုံခြုံရေး/RBAC ကို ဖွဲ့စည်းထားသည်

### `azd deploy` မပြုမီ

- [ ] Environment variables များ သတ်မှတ်ပြီး
- [ ] လျှို့ဝှက်တန်ဖိုးများကို Key Vault ထဲတွင် သိမ်းဆည်းထားပြီး
- [ ] Connection strings များကို အတည်ပြုပြီး
- [ ] Health checks များကို ဖွဲ့စည်းထားပြီး
