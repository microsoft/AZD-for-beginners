# ဒုတိယပိုဒ် ၆: မတိုင်မီ အစီအစဉ်ရေးဆွဲခြင်းနှင့် စစ်ဆေးခြင်း

## အနှစ်ချုပ်

ဤပိုဒ်တွင် သင့်လျော်သော အစီအစဉ်ရေးဆွဲခြင်းနှင့် စစ်ဆေးမှု အဆင့်များကို မတိုင်မှီ သိရှိနိုင်အောင် ဖော်ပြထားသည်။ သင့်လျော်သော စွမ်းရည်ရေးဆွဲခြင်း၊ SKU ရွေးချယ်ခြင်းနှင့် မတိုင်မီ စစ်ဆေးမှုများဖြင့် ငွေကြေးအသုံးစရိတ်မြင့်မားသည့် အမှားများမှ ကာကွယ်သင့်သည်။

> `azd 1.27.1` နှင့် ၂၀၂၆ ခုနှစ် ဇူလိုင်လတွင် အတည်ပြုပြီး။

## သင်ယူရမည့် အချက်များ

ဤပိုဒ်ကို ပြီးစီးခြင်းဖြင့် သင်မှာ-
- တပ်ဆင်မှုမတိုင်မီ ကြိုတင် စစ်ဆေးမှုများ ပြုလုပ်နိုင်မည်
- စွမ်းအားရေးဆွဲခြင်းနှင့် အရင်းအမြစ် လိုအပ်ချက်များ ခန့်မှန်းနိုင်မည်
- ကျသင့်စရိတ် အလိုက် သင့်တော်သော SKU များ ရွေးချယ်နိုင်မည်
- ကြည့်ရှုမှု အတွက် Application Insights ကို သတ်မှတ်နိုင်မည်
- အဖွဲ့ပူးပေါင်းမှု စနစ်များကို နားလည်နိုင်မည်

## 🚀 အမြန် စတင်နည်း

```bash
# စာရင်းသွင်းမှု အရေအတွက်များကို စစ်ဆေးပါ
az vm list-usage --location eastus --output table

# အကောက်ခွန် ပြင်ဆင်မှုကို ကြိုတင်ကြည့်ရှူပါ (အရင်းအမြစ် မဖြစ်မနေမဖန်တီးပါ)
azd provision --preview

# Bicep စာလုံးပေါင်းစံအတည်ပြုပါ
az bicep build --file infra/main.bicep

# ပတ်ဝန်းကျင် ဖွဲ့စည်းမှုကို စစ်ဆေးပါ
azd env get-values
```

## ☑️ မတိုင်မီ စစ်ဆေးရမည့် စာရင်း

### `azd provision` မတိုင်မီ

- [ ] ဒေသအလိုက် ဂဏန်းစစ်ဆေးပြီး
- [ ] SKU များ သင့်တော်စွာ ရွေးချယ်ပြီး
- [ ] ကုန်ကျစရိတ် ခန့်မှန်းချက် ပြန်လည်သုံးသပ်ပြီး
- [ ] အမည်ပုံစံ တိကျမှန်ကန်ပြီး
- [ ] လုံခြုံရေး/RBAC စီမံထားပြီး

### `azd deploy` မတိုင်မီ

- [ ] ပတ်ဝန်းကျင် အပြောင်းအလဲများ သတ်မှတ်ပြီး
- [ ] Key Vault တွင် လျှို့ဝှက်ချက်များထားရှိပြီး
- [ ] ချိတ်ဆက်မှု စာကြောင်းများ စစ်ဆေးပြီး
- [ ] ကျန်းမာရေး စစ်ဆေးမှုများ သတ်မှတ်ပြီး
