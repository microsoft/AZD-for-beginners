# အခန်း 6: တင်မတင်မီ စီစဉ်ခြင်းနှင့် အတည်ပြုခြင်း

**📚 သင်တန်း**: [AZD For Beginners](../../README.md) | **⏱️ ကြာချိန်**: 1 နာရီ | **⭐ ရှုပ်ထွေးမှု**: အလယ်အလတ်

## မိတ်ဆက်

ဤအခန်းတွင် သင်၏ အက်ပလီကေးရှင်းကို တင်မလုပ်ခင် လိုအပ်သော စီစဉ်မှုများနှင့် အတည်ပြုရေးဆိုင်ရာ အဆင့်များကို လေ့လာပါ။ စွမ်းဆောင်ရည် စီမံချက်၊ SKU ရွေးချယ်မှုနှင့် မတင်ခင် စစ်ဆေးမှုများဖြင့် ကျသင့်ရက်စွဲများကို ရှောင်ရှားနိုင်ပါသည်။

> 2026 ခုနှစ် ဇွန်လတွင် `azd 1.25.6` နှင့်အတည်ပြုထားသည်။

## သင်ယူရမည့် ရည်မှန်းချက်များ

ဤအခန်းကို ပြီးမြောက်စေခြင်းဖြင့် သင်သည် -
- တင်မှု မလုပ်မီ မတင်ခင် စစ်ဆေးမှုများကို chạy လုပ်နိုင်မည်
- စွမ်းဆောင်ရည်ကို စီမံကိန်းချပြီး ရင်းမြစ်လိုအပ်ချက်များကို ခန့်မှန်းနိုင်မည်
- ကုန်ကျစရိတ် အျကော့မပေးရန် သင့်တော်သော SKU များကို ရွေးချယ်နိုင်မည်
- ကြိုတင်ကြည့်ရှုမှုအတွက် Application Insights ကို ဖွဲ့စည်းနိုင်မည်
- အဖွဲ့ ပူးပေါင်းဆောင်ရွက်မှု ပုံစံများကို နားလည်နိုင်မည်

## 🚀 အမြန်စတင်ချက်

```bash
# စာရင်းသွင်းမှု ကန့်သတ်ချက်များကို စစ်ဆေးပါ
az vm list-usage --location eastus --output table

# တပ်ဆင်မှုကို ကြိုတင်ကြည့်ရှုပါ (အရင်းအမြစ်များ မတည်ဆောက်ပါ)
azd provision --preview

# Bicep ၏ သဒ္ဒယကို အတည်ပြုပါ
az bicep build --file infra/main.bicep

# ပတ်ဝန်းကျင် ဖွဲ့စည်းမှုကို စစ်ဆေးပါ
azd env get-values
```

---

## ☑️ တင်မတင်မီ စစ်ဆေးစာရင်း

### Before `azd provision`

- [ ] တည်နေရာအတွက် ကွဘို (Quota) ပြန်စစ်ပြီးပြီ
- [ ] SKU များ သင့်တော်စွာ ရွေးချယ်ပြီးပြီ
- [ ] ကုန်ကျစရိတ် ခန့်မှန်းချက်ကို ပြန်လည်ဆန်းစစ်ပြီးပြီ
- [ ] အမည်ပေးစနစ် တစ်ခုတည်း သတ်မှတ်ထားသည်
- [ ] လုံခြုံရေး/RBAC ကို ဖွဲ့စည်းထားသည်

### Before `azd deploy`

- [ ] ပတ်ဝန်းကျင် များအတွက် ကွယ်လ်အဖွဲ့ စာမျက်နှာများ (Environment variables) သတ်မှတ်ပြီးပြီ
- [ ] Secrets များကို Key Vault ထဲသို့ ထည့်ပြီးပြီ
- [ ] Connection strings များကို အတည်ပြုပြီးပြီ
- [ ] Health checks များကို ဖွဲ့စည်းထားသည်
