# အခန်း 7: ပြဿနာဖြေရှင်းနှင့် အမှားရှာဖွေရေး

**📚 သင်တန်း**: [AZD အစပြုသူများအတွက်](../../README.md) | **⏱️ ကြာချိန်**: 1-1.5 နာရီ | **⭐ ခက်ခဲမှု**: အလယ်အလတ်

---

## အနှစ်ချုပ်

ဤအခန်းသည် Azure Developer CLI ကို အသုံးပြုရာတွင် တွေ့ကြုံရသော ပုံမှန် ပြဿနာများကို ရှာဖွေစစ်ဆေးခြင်းနှင့် ဖြေရှင်းရန် ကူညီပေးပါသည်။ တပ်ဆင်မှု မအောင်မြင်ခြင်းများမှ AI သီးသန့် ပြဿနာများအထိ ပါဝင်သည်။

> Validated against `azd 1.25.6` in June 2026.

## သင်ယူရမည့်ရည်မှန်းချက်များ

ဤအခန်းကို ပြီးမြောက်ပါက သင်သည်:
- AZD တပ်ဆင်မှု မအောင်မြင်မှုများကို ရှာဖွေခြင်း
- အတည်ပြုရေးနှင့် ခွင့်ပြုချက် ပြဿနာများကို အမှားရှာဖွေခြင်း
- AI ဝန်ဆောင်မှု ဆက်သွယ်မှု ပြဿနာများကို ဖြေရှင်းခြင်း
- ပြဿနာရှာဖွေရေးအတွက် Azure Portal နှင့် CLI ကို အသုံးပြုခြင်း

## 🚨 အမြန် ဖြေရှင်းနည်းများ

### အတည်ပြုရေး ပြဿနာများ
```bash
# AZD အလုပ်စဉ်များအတွက် လိုအပ်သည်
azd auth login

# သင် Azure CLI အမိန့်များကို တိုက်ရိုက်လည်း အသုံးပြုနေပါက ရွေးချယ်နိုင်သည်
az login

azd auth status
```

### ပေးပို့ခြင်း မအောင်မြင်မှုများ
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### ရင်းမြစ် တိုက်မှုများ
```bash
azd down --force --purge
azd env new different-name
azd up
```

### Quota ကျော်လွန်ခြင်း
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 အမှားကုဒ် ကိုးကားချက်

| အမှား | အကြောင်းရင်း | ဖြေရှင်းနည်း |
|-------|-------|----------|
| `AuthenticationError` | လော့ဂ်အင် မရှိပါ | `azd auth login` |
| `ResourceNotFound` | ရင်းမြစ် မတွေ့ပါ | ရင်းမြစ်အမည်များကို စစ်ဆေးပါ |
| `QuotaExceeded` | subscription ကန့်သတ်ချက်များ | ကွိုတာ တိုးပေးရန် တောင်းဆိုပါ |
| `InvalidTemplate` | Bicep သဒ္ဓါန် အမှား | `az bicep build` |
| `Conflict` | ရင်းမြစ် ရှိပြီးသား | အမည်အသစ် သုံးရန် သို့မဟုတ် ဖျက်ပါ |
| `Forbidden` | ခွင့်ပြုချက် မလုံလောက် | RBAC အခန်းကဏ္ဍများကို စစ်ဆေးပါ |

---

## 🔄 ပြန်စတင်ခြင်းနှင့် ကယ်ဆယ်ရေး

```bash
# ဆော့ဖ် ရီဆက် (အရင်းအမြစ်များကို ထိန်းသိမ်းထားပြီး ကုဒ်ကို ပြန်တင်ခြင်း)
azd deploy --force

# ဟတ်ဒ် ရီဆက် (အရာအားလုံးကို ဖျက်ပြီး အသစ်စတင်ခြင်း)
azd down --force --purge
azd up
```
