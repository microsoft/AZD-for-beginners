# အခန်း 7: ပြဿနာရှာဖွေခြင်းနှင့် အမှားရှာဖွေခြင်း

## အကျဉ်းချုပ်

ဤအခန်းသည် Azure Developer CLI ဖြင့် အလုပ်လုပ်စဉ်တွင် တွေ့ကြုံရနိုင်သည့် ပုံမှန်ပြဿနာများကို ရှာဖွေခြင်းနှင့် ဖြေရှင်းပေးနိုင်ရန် အထောက်အကူဖြစ်စေသည်။ တပ်ဆင်မှု မအောင်မြင်မှုများမှ AI အထူးပြဿနာများအထိ။

## လေ့လာရန်ရည်ရွယ်ချက်များ

ဒီအခန်းကို ပြီးမြောက်ခြင်းဖြင့် သင်သည် -
- AZD တပ်ဆင်မှု မအောင်မြင်မှုများကို ရှာဖွေနိုင်မည်
- အတည်ပြုခြင်းနှင့် ခွင့်ပြုချက်ဆိုင်ရာ ပြဿနာများကို ဒေဘတ်လုပ်နိုင်မည်
- AI ဝန်ဆောင်မှုဆိုင်ရာ ချိတ်ဆက်မှုပြဿနာများကို ဖြေရှင်းနိုင်မည်
- ပြဿနာရှာဖွေရေးအတွက် Azure Portal နှင့် CLI ကို အသုံးပြုနိုင်မည်

## 🚨 အမြန် ဖြေရှင်းနည်းများ

### အတည်ပြုခြင်းဆိုင်ရာ ပြဿနာများ
```bash
azd auth login
az login
azd auth whoami
```

### ပရိုဗိရှင်း မအောင်မြင်ခြင်းများ
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### အရင်းအမြစ် ဆန့်ကျင်မှုများ
```bash
azd down --force --purge
azd env new different-name
azd up
```

### ကိုတာ ကျော်လွန်ခြင်း
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 အမှား ကုဒ် ရည်ညွှန်းချက်

| အမှား | အကြောင်းရင်း | ဖြေရှင်းနည်း |
|-------|-------|----------|
| `AuthenticationError` | လော့ဂ်အင် မရှိခြင်း | `azd auth login` |
| `ResourceNotFound` | အရင်းအမြစ် ပျောက်ဆုံးခြင်း | အရင်းအမြစ် အမည်များကို စစ်ဆေးပါ |
| `QuotaExceeded` | စာရင်းသုံးစွဲမှု အကန့်အသတ် | ကိုတာ တိုးမြှင့်ရန် တောင်းဆိုပါ |
| `InvalidTemplate` | Bicep စာရိုက်နည်း မှားယွင်းမှု | `az bicep build` |
| `Conflict` | အရင်းအမြစ် ရှိပြီးသား | အသစ်အမည် သို့မဟုတ် ဖျက်ပစ်ပါ |
| `Forbidden` | ခွင့်ပြုချက် မလုံလောက်ခြင်း | RBAC အခန်းကဏ္ဍများကို စစ်ဆေးပါ |

## 🔄 ပြန်လည်စတင်ခြင်းနှင့် ပြန်လည်ရယူခြင်း

```bash
# နူးညံ့ပြန်စတင်ခြင်း (ရင်းမြစ်များကို ထိန်းသိမ်းထား၍ ကုဒ်ကို ပြန်တပ်ဆင်ခြင်း)
azd deploy --force

# တင်းကြပ်ပြန်စတင်ခြင်း (အားလုံးကို ဖျက်ပစ်၍ အသစ်ကဲ့သို့ ပြန်စတင်ခြင်း)
azd down --force --purge
azd up
```
