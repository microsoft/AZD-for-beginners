# AZD für KI-Entwickler

!!! info "ZIELGRUPPE FÜR DIESEN WORKSHOP"
   
    Dieser Workshop richtet sich an alle, die AI Agentic-Anwendungen auf Microsoft Foundry entwickeln möchten und dabei erstklassige Tools und Workflows nutzen wollen. Der Workshop eignet sich sowohl für Einsteiger als auch für erfahrene Entwickler, die mit Azure, AI-Konzepten und Python-Code vertraut sind. Bitte prüfen Sie die Voraussetzungen, bevor Sie beginnen.

    **Dauer:** 3-4 Stunden  
    **Niveau:** Anfänger bis Mittelstufe  

---


## Voraussetzungen

- A personal GitHub account → [Kostenlos erstellen](https://github.com/signup)
- An active Azure account → [Kostenlos erstellen](https://aka.ms/free)
- Vertrautheit mit AI-Konzepten, Python-Entwicklung, Kommandozeilen-Tools
- Erfahrung mit VS Code sowie GitHub-Tools und -Workflows.

---

## Lernziele

Am Ende dieses Workshops wissen Sie, wie Sie:

- [X] **Work with Azure Developer CLI** → Azure-Ressourcen bereitzustellen, bereitzustellen und zu verwalten.
- [X] **Use AI App Templates** → AI Agent-Projekte auf Microsoft Foundry schnell zu starten.
- [X] **Configure Templates** → Funktionen wie Red Teaming, Evaluations, Tracing & Monitoring zu konfigurieren.
- [X] **Deconstruct and Explore** → AZD-Vorlagen und Workflows mit GitHub Copilot zu zerlegen und zu erkunden.
- [X] **Customize and Redeploy** → die AI Agent Starter-Vorlage mit Ihren individuellen Anforderungen anzupassen und neu bereitzustellen.

---

## Workshop-Gliederung

Der Workshop ist in Module gegliedert, die jeweils durch eine praktische Übung gestützt werden:

1. **Introduction** - Erhalten Sie ein Verständnis dafür, was AZD-Vorlagen sind und wie sie funktionieren
1. **Selection** - Finden Sie die richtige "Starter"-Vorlage für Ihr nächstes AI-Projekt
1. **Validation** - Stellen Sie die Vorlage bereit und erkunden Sie sie, um zu überprüfen, dass sie wie beworben funktioniert
1. **Deconstruction** - Zerlegen Sie die Vorlage, um ihre Komponenten und Konfiguration zu verstehen
1. **Configuration** - Lernen Sie, verschiedene Microsoft Foundry-Funktionen zu aktivieren und zu nutzen
1. **Customization** - Rekonstruieren Sie die Anwendung unter Berücksichtigung Ihrer Daten und Anforderungen
1. **Teardown** - Lernen Sie, Infrastruktur zu deprovisionieren und soft-gelöschte Ressourcen wiederherzustellen
