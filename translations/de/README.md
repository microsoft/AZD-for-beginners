## Einführung

Willkommen bei der umfassenden Anleitung für Azure Developer CLI (azd). Dieses Repository wurde entwickelt, um Entwicklern auf allen Ebenen – von Studenten bis hin zu professionellen Entwicklern – zu helfen, Azure Developer CLI zu lernen und zu meistern, um effiziente Cloud-Bereitstellungen durchzuführen, **mit besonderem Fokus auf die Bereitstellung von KI-Anwendungen mit Azure AI Foundry**. Diese strukturierte Lernressource bietet praktische Erfahrungen mit Azure-Cloud-Bereitstellungen, der Behebung häufiger Probleme und der Implementierung von Best Practices für erfolgreiche AZD-Template-Bereitstellungen.

### **Warum diese Anleitung für KI-Entwickler wichtig ist**
Basierend auf unserer jüngsten Umfrage in der Azure AI Foundry Discord-Community sind **45 % der Entwickler daran interessiert, AZD für KI-Workloads zu nutzen**, stehen jedoch vor Herausforderungen wie:
- Komplexe Multi-Service-KI-Architekturen
- Best Practices für die Bereitstellung von KI in der Produktion
- Integration und Konfiguration von Azure AI-Diensten
- Kostenoptimierung für KI-Workloads
- Behebung von KI-spezifischen Bereitstellungsproblemen

## Lernziele

Durch die Arbeit mit diesem Repository werden Sie:
- Die Grundlagen und Kernkonzepte von Azure Developer CLI meistern
- Lernen, Azure-Ressourcen mit Infrastructure as Code bereitzustellen und zu verwalten
- Fähigkeiten zur Fehlerbehebung bei häufigen AZD-Bereitstellungsproblemen entwickeln
- Vorbereitungsvalidierung und Kapazitätsplanung verstehen
- Sicherheits-Best-Practices und Strategien zur Kostenoptimierung implementieren
- Selbstbewusstsein beim Bereitstellen von produktionsreifen Anwendungen in Azure gewinnen

## Lernergebnisse

Nach Abschluss dieses Kurses werden Sie in der Lage sein:
- Azure Developer CLI erfolgreich zu installieren, zu konfigurieren und zu nutzen
- Anwendungen mit AZD-Templates zu erstellen und bereitzustellen
- Authentifizierungs-, Infrastruktur- und Bereitstellungsprobleme zu beheben

## Was ist Azure Developer CLI?

Azure Developer CLI (azd) ist eine entwicklerzentrierte Befehlszeilenschnittstelle, die den Prozess des Erstellens und Bereitstellens von Anwendungen in Azure beschleunigt. Sie bietet:

- **Template-basierte Bereitstellungen** - Verwenden Sie vorgefertigte Templates für gängige Anwendungsmuster
- **Infrastructure as Code** - Verwalten Sie Azure-Ressourcen mit Bicep oder Terraform
- **Integrierte Workflows** - Anwendungen nahtlos bereitstellen, provisionieren und überwachen
- **Entwicklerfreundlich** - Optimiert für Produktivität und Benutzererfahrung von Entwicklern

### **AZD + Azure AI Foundry: Perfekt für KI-Bereitstellungen**

**Warum AZD für KI-Lösungen?** AZD adressiert die größten Herausforderungen, denen KI-Entwickler gegenüberstehen:

- **KI-fähige Templates** - Vorab konfigurierte Templates für Azure OpenAI, Cognitive Services und ML-Workloads
- **Sichere KI-Bereitstellungen** - Eingebaute Sicherheitsmuster für KI-Dienste, API-Schlüssel und Modellendpunkte
- **Produktions-KI-Muster** - Best Practices für skalierbare, kosteneffiziente KI-Anwendungsbereitstellungen
- **End-to-End-KI-Workflows** - Von der Modellentwicklung bis zur Produktionsbereitstellung mit ordnungsgemäßer Überwachung
- **Kostenoptimierung** - Intelligente Ressourcenzuweisung und Skalierungsstrategien für KI-Workloads
- **Azure AI Foundry-Integration** - Nahtlose Verbindung zum Modellkatalog und den Endpunkten von AI Foundry

## Schnellstart

### Voraussetzungen
- Azure-Abonnement
- Installierte Azure CLI
- Git (zum Klonen von Templates)

### Installation
```bash
# Windows (PowerShell)
powershell -ex AllSigned -c "Invoke-RestMethod 'https://aka.ms/install-azd.ps1' | Invoke-Expression"

# macOS/Linux
curl -fsSL https://aka.ms/install-azd.sh | bash
```

### Ihre erste Bereitstellung
```bash
# Initialize a new project
azd init --template todo-nodejs-mongo

# Provision Azure resources and deploy
azd up
```

### Ihre erste KI-Bereitstellung
```bash
# Initialize an AI-powered chat application with Azure OpenAI
azd init --template azure-search-openai-demo

# Configure AI services and deploy
azd up

# Or try other AI templates:
azd init --template openai-chat-app-quickstart
azd init --template ai-document-processing
azd init --template contoso-chat
```
