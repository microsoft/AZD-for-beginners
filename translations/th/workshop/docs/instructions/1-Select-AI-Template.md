## 1. อุปมาของช่างก่อสร้าง

การสร้างแอปพลิเคชัน AI สำหรับองค์กรที่ทันสมัย _ตั้งแต่เริ่มต้น_ อาจเป็นเรื่องน่ากลัว มันเหมือนกับการสร้างบ้านใหม่ด้วยตัวเอง ทีละก้อนอิฐ ใช่ มันสามารถทำได้! แต่ไม่ใช่วิธีที่มีประสิทธิภาพที่สุดเพื่อให้ได้ผลลัพธ์ที่ต้องการ!

แทนที่จะเริ่มต้นใหม่ เรามักเริ่มจาก _แบบแปลนการออกแบบ_ ที่มีอยู่ และทำงานร่วมกับสถาปนิกในการปรับแต่งให้ตรงกับความต้องการส่วนบุคคล และนี่คือวิธีที่ควรทำเมื่อสร้างแอปพลิเคชันอัจฉริยะ ก่อนอื่น ให้หาสถาปัตยกรรมการออกแบบที่ดีที่เหมาะกับปัญหาของคุณ จากนั้นทำงานร่วมกับสถาปนิกโซลูชันเพื่อปรับแต่งและพัฒนาโซลูชันให้เหมาะกับสถานการณ์เฉพาะของคุณ

แต่เราจะหาต้นแบบการออกแบบเหล่านี้จากที่ไหน? และจะหาสถาปนิกที่ยินดีสอนวิธีปรับแต่งและใช้งานต้นแบบเหล่านี้ด้วยตัวเองได้อย่างไร? ในเวิร์กช็อปนี้ เราจะตอบคำถามเหล่านั้นโดยแนะนำคุณรู้จักกับเทคโนโลยีสามอย่าง:

1. [Azure Developer CLI](https://aka.ms/azd) - เครื่องมือโอเพนซอร์สที่เร่งเส้นทางของนักพัฒนา ตั้งแต่การพัฒนาท้องถิ่น (สร้าง) ไปจนถึงการติดตั้งในคลาวด์ (ส่ง)
1. [Microsoft Foundry Templates](https://ai.azure.com/templates) - ที่เก็บซอร์สโค้ดพร้อมตัวอย่าง รหัสพื้นฐาน โครงสร้างพื้นฐาน และไฟล์การกำหนดค่าที่เป็นมาตรฐานสำหรับการติดตั้งโซลูชัน AI
1. [GitHub Copilot Agent Mode](https://code.visualstudio.com/docs/copilot/chat/chat-agent-mode) - เอเจนต์เขียนโค้ดที่มีความรู้พื้นฐานจาก Azure ที่สามารถช่วยแนะนำวิธีนำทางและแก้ไขโค้ดโดยใช้ภาษาธรรมชาติ

ด้วยเครื่องมือเหล่านี้ในมือ เราสามารถ _ค้นหา_ แม่แบบที่ถูกต้อง _ติดตั้ง_ เพื่อตรวจสอบให้แน่ใจว่าสามารถทำงานได้ และ _ปรับแต่ง_ ให้เหมาะกับสถานการณ์เฉพาะของเราได้ ตอนนี้ลองเรียนรู้วิธีการทำงานของเครื่องมือเหล่านี้กัน

## 2. Azure Developer CLI

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) (หรือ `azd`) เป็นเครื่องมือบรรทัดคำสั่งโอเพนซอร์สที่ช่วยเร่งเส้นทางจากรหัสไปสู่คลาวด์ด้วยชุดของคำสั่งที่เป็นมิตรกับนักพัฒนาซอฟต์แวร์ สามารถใช้งานได้อย่างสม่ำเสมอทั้งใน IDE (สำหรับการพัฒนา) และสภาพแวดล้อม CI/CD (สำหรับ devops)

ด้วย `azd` เส้นทางการติดตั้งของคุณจะง่ายเพียง:

- `azd init` - เริ่มต้นโปรเจกต์ AI ใหม่จากแม่แบบ AZD ที่มีอยู่
- `azd up` - สร้างโครงสร้างพื้นฐานและติดตั้งแอปพลิเคชันของคุณในขั้นตอนเดียว
- `azd monitor` - รับการเฝ้าระวังและการวินิจฉัยแบบเรียลไทม์สำหรับแอปที่ติดตั้งแล้ว
- `azd pipeline config` - ตั้งค่า pipeline CI/CD เพื่อทำให้การติดตั้งไปยัง Azure เป็นอัตโนมัติ

**🎯 | แบบฝึกหัด**: <br/> ลองสำรวจเครื่องมือบรรทัดคำสั่ง `azd` ในสภาพแวดล้อม GitHub Codespaces ของคุณทันที เริ่มต้นโดยพิมพ์คำสั่งนี้เพื่อดูว่าเครื่องมือนี้ทำอะไรได้บ้าง:

```bash title="" linenums="0"
azd help
```

## 3. แม่แบบ AZD

เพื่อให้ `azd` ทำงานได้ มันจำเป็นต้องรู้โครงสร้างพื้นฐานที่ต้องจัดเตรียม การตั้งค่ากำหนดค่าที่ต้องใช้บังคับ และแอปพลิเคชันที่ต้องติดตั้ง ซึ่งก็คือหน้าที่ของ [แม่แบบ AZD](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/azd-templates?tabs=csharp)

แม่แบบ AZD คือที่เก็บซอร์สโค้ดโอเพนซอร์สที่รวมโค้ดตัวอย่างกับไฟล์โครงสร้างพื้นฐานและไฟล์การกำหนดค่าที่จำเป็นสำหรับการติดตั้งสถาปัตยกรรมโซลูชัน
ด้วยแนวทาง _โครงสร้างพื้นฐานในรูปแบบโค้ด_ (IaC) แม่แบบเหล่านี้อนุญาตให้กำหนดทรัพยากรและการตั้งค่าที่เกี่ยวข้องสามารถจัดการโดยระบบควบคุมเวอร์ชัน (เช่นเดียวกับซอร์สโค้ดของแอป) ซึ่งสร้างกระบวนงานที่นำกลับมาใช้ใหม่ได้และมีความสม่ำเสมอสำหรับผู้ใช้โปรเจกต์นั้นๆ

เมื่อสร้างหรือใช้แม่แบบ AZD สำหรับ _สถานการณ์ของคุณ_ ให้พิจารณาคำถามเหล่านี้:

1. คุณกำลังสร้างอะไร? → มีแม่แบบที่มีโค้ดเริ่มต้นสำหรับสถานการณ์นั้นหรือไม่?
1. โซลูชันของคุณถูกออกแบบอย่างไร? → มีแม่แบบที่มีทรัพยากรจำเป็นหรือไม่?
1. โซลูชันของคุณติดตั้งอย่างไร? → คิดถึงคำสั่ง `azd deploy` พร้อมฮุกก่อนและหลังประมวลผล!
1. คุณสามารถปรับปรุงให้ดีขึ้นได้อย่างไร? → คิดถึงการเฝ้าระวังในตัวและ pipeline อัตโนมัติ!

**🎯 | แบบฝึกหัด**: <br/> 
เยี่ยมชมแกลเลอรี [Awesome AZD](https://azure.github.io/awesome-azd/) และใช้ตัวกรองเพื่อสำรวจแม่แบบมากกว่า 250 แบบที่มีอยู่ในปัจจุบัน ดูว่าคุณสามารถหาแม่แบบที่ตรงกับความต้องการ _สถานการณ์ของคุณ_ ได้หรือไม่

## 4. แม่แบบแอป AI

สำหรับแอปพลิเคชันที่ขับเคลื่อนด้วย AI, Microsoft มีแม่แบบเฉพาะที่ใช้ **Microsoft Foundry** และ **Foundry Agents** แม่แบบเหล่านี้ช่วยเร่งเส้นทางสู่การสร้างแอปพลิเคชันอัจฉริยะที่พร้อมใช้งานในสภาพแวดล้อมการผลิต

### แม่แบบ Microsoft Foundry & Foundry Agents

เลือกแม่แบบด้านล่างเพื่อติดตั้ง แต่ละแม่แบบมีอยู่ใน [Awesome AZD](https://azure.github.io/awesome-azd/) และสามารถเริ่มต้นใช้งานด้วยคำสั่งเดียว

| แม่แบบ | คำอธิบาย | คำสั่งติดตั้ง |
|----------|-------------|----------------|
| **[AI Chat with RAG](https://azure.github.io/awesome-azd/?tags=ai&tags=rag)** | แอปพลิเคชันแชทแบบ Retrieval Augmented Generation โดยใช้ Microsoft Foundry | `azd init -t azure-samples/azure-search-openai-demo` |
| **[Foundry Agent Service Starter](https://azure.github.io/awesome-azd/?tags=ai&tags=agents)** | สร้างเอเจนต์ AI ด้วย Foundry Agents เพื่อดำเนินงานอัตโนมัติ | `azd init -t azure-samples/foundry-agent-service-starter` |
| **[Multi-Agent Orchestration](https://azure.github.io/awesome-azd/?tags=ai&tags=agents)** | ประสานงานหลาย Foundry Agents สำหรับเวิร์กโฟลว์ที่ซับซ้อน | `azd init -t azure-samples/multi-agent-orchestration` |
| **[AI Document Intelligence](https://azure.github.io/awesome-azd/?tags=ai&tags=document)** | ดึงข้อมูลและวิเคราะห์เอกสารด้วยโมเดล Microsoft Foundry | `azd init -t azure-samples/ai-document-processing` |
| **[Conversational AI Bot](https://azure.github.io/awesome-azd/?tags=ai&tags=bot)** | สร้างแชทบอทอัจฉริยะด้วยการผสานรวม Microsoft Foundry | `azd init -t azure-samples/ai-chat-protocol` |
| **[AI Image Generation](https://azure.github.io/awesome-azd/?tags=ai&tags=dalle)** | สร้างภาพโดยใช้ DALL-E ผ่าน Microsoft Foundry | `azd init -t azure-samples/ai-image-generation` |
| **[Semantic Kernel Agent](https://azure.github.io/awesome-azd/?tags=ai&tags=semantic-kernel)** | เอเจนต์ AI โดยใช้ Semantic Kernel กับ Foundry Agents | `azd init -t azure-samples/semantic-kernel-agent` |
| **[AutoGen Multi-Agent](https://azure.github.io/awesome-azd/?tags=ai&tags=autogen)** | ระบบหลายเอเจนต์โดยใช้เฟรมเวิร์ก AutoGen | `azd init -t azure-samples/autogen-multi-agent` |

### เริ่มต้นอย่างรวดเร็ว

1. **เรียกดูแม่แบบ**: เยี่ยมชม [https://azure.github.io/awesome-azd/](https://azure.github.io/awesome-azd/) และกรองด้วย `AI`, `Agents` หรือ `Microsoft Foundry`
2. **เลือกแม่แบบของคุณ**: เลือกแม่แบบที่เหมาะกับกรณีการใช้งานของคุณ
3. **เริ่มต้นใช้งาน**: รันคำสั่ง `azd init` สำหรับแม่แบบที่เลือก
4. **ติดตั้ง**: รัน `azd up` เพื่อจัดเตรียมและติดตั้ง

**🎯 | แบบฝึกหัด**: <br/>
เลือกแม่แบบหนึ่งรายการด้านบนตามสถานการณ์ของคุณ:

- **กำลังสร้างแชทบอท?** → เริ่มต้นด้วย **AI Chat with RAG** หรือ **Conversational AI Bot**
- **ต้องการเอเจนต์อัตโนมัติ?** → ลองใช้ **Foundry Agent Service Starter** หรือ **Multi-Agent Orchestration**
- **กำลังประมวลผลเอกสาร?** → ใช้ **AI Document Intelligence**
- **ต้องการความช่วยเหลือเขียนโค้ด AI?** → สำรวจ **Semantic Kernel Agent** หรือ **AutoGen Multi-Agent**

```bash title="Example: Deploy the AI Chat with RAG template" linenums="0"
azd init -t azure-samples/azure-search-openai-demo
azd up
```

!!! info "สำรวจแม่แบบเพิ่มเติม"
    แกลเลอรี [Awesome AZD](https://azure.github.io/awesome-azd/) มีแม่แบบมากกว่า 250 แบบ ใช้ตัวกรองเพื่อค้นหาแม่แบบที่ตรงกับความต้องการเฉพาะของคุณในด้านภาษา เฟรมเวิร์ก และบริการ Azure ได้อย่างง่ายดาย
