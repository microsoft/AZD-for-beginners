# 3. การแยกส่วนเทมเพลต

!!! tip "เมื่อจบบทนี้คุณจะสามารถ"

    - [ ] เปิดใช้งาน GitHub Copilot กับเซิร์ฟเวอร์ MCP สำหรับช่วยเหลือ Azure
    - [ ] เข้าใจโครงสร้างโฟลเดอร์และส่วนประกอบของเทมเพลต AZD
    - [ ] สำรวจรูปแบบการจัดการโครงสร้างพื้นฐานด้วยโค้ด (Bicep)
    - [ ] **แล็บ 3:** ใช้ GitHub Copilot เพื่อสำรวจและเข้าใจสถาปัตยกรรมของรีโพซิทอรี

---

ด้วยเทมเพลต AZD และ Azure Developer CLI (`azd`) เราสามารถเริ่มต้นการพัฒนา AI ได้อย่างรวดเร็วด้วยรีโพซิทอรีมาตรฐานที่ให้ตัวอย่างโค้ด โครงสร้างพื้นฐาน และไฟล์กำหนดค่า — ในรูปแบบของโปรเจ็กต์ _starter_ ที่พร้อมใช้งาน

**แต่ตอนนี้ เราต้องเข้าใจโครงสร้างโปรเจ็กต์และฐานโค้ด — และสามารถปรับแต่งเทมเพลต AZD ได้ — โดยไม่จำเป็นต้องมีประสบการณ์หรือความเข้าใจใน AZD มาก่อน!**

## 1. เปิดใช้งาน GitHub Copilot

### 1.1 ติดตั้ง GitHub Copilot Chat

ถึงเวลาสำรวจ [GitHub Copilot กับโหมดเอเย่นต์](https://code.visualstudio.com/docs/copilot/chat/chat-agent-mode) แล้ว ตอนนี้เราสามารถใช้ภาษาธรรมชาติในการอธิบายงานระดับสูงและรับความช่วยเหลือในการดำเนินการ สำหรับแล็บนี้ เราจะใช้ [แผนฟรีของ Copilot](https://github.com/github-copilot/signup) ซึ่งมีขีดจำกัดรายเดือนสำหรับการตอบกลับและการสนทนา

ส่วนขยายสามารถติดตั้งได้จาก marketplace แต่ควรพร้อมใช้งานแล้วในสภาพแวดล้อม Codespaces ของคุณ _คลิกที่ `Open Chat` จากไอคอน Copilot แล้วพิมพ์คำถามเช่น `What can you do?`_ — คุณอาจจะถูกขอให้ล็อกอิน **GitHub Copilot Chat พร้อมใช้งานแล้ว**

### 1.2 ติดตั้ง MCP Servers

สำหรับโหมดเอเย่นต์เพื่อให้ได้ผล ต้องเข้าถึงเครื่องมือที่ถูกต้องเพื่อช่วยดึงข้อมูลหรือทำการกระทำ นี่คือที่เซิร์ฟเวอร์ MCP เข้ามาช่วย เราจะตั้งค่าเซิร์ฟเวอร์ต่อไปนี้:

1. [Azure MCP Server](../../../../../workshop/docs/instructions)
1. [Microsoft Docs MCP Server](../../../../../workshop/docs/instructions)

เพื่อเปิดใช้งาน:

1. สร้างไฟล์ชื่อ `.vscode/mcp.json` หากยังไม่มี
1. คัดลอกข้อมูลต่อไปนี้ลงในไฟล์นี้ — แล้วเริ่มเซิร์ฟเวอร์!

   ```json title=".vscode/mcp.json"
   {
      "servers": {
         "Azure MCP Server": {
            "command": "npx",
            "args": [
            "-y",
            "@azure/mcp@latest",
            "server",
            "start"
            ]
         },
         "microsoft.docs.mcp": {
            "type": "http",
            "url": "https://learn.microsoft.com/api/mcp"
         }
      }
   }
   ```

??? warning "คุณอาจเจอข้อผิดพลาดว่า `npx` ยังไม่ได้ติดตั้ง (คลิกขยายเพื่อแก้ไข)"

      วิธีแก้ไข ให้เปิดไฟล์ `.devcontainer/devcontainer.json` แล้วเพิ่มบรรทัดนี้ในส่วน features จากนั้นสร้างคอนเทนเนอร์ใหม่ คุณจะมี `npx` ติดตั้งแล้ว

      ```title="" linenums="0"
         "features": {
            "ghcr.io/devcontainers/features/node:1": {},
            ...
         },
      ```

### 1.3 ทดสอบ GitHub Copilot Chat

**ใช้คำสั่ง `az login` เพื่อเข้าสู่ระบบ Azure จากบรรทัดคำสั่ง VS Code ก่อน**

ตอนนี้คุณจะสามารถสอบถามสถานะการสมัครใช้งาน Azure ของคุณ และถามคำถามเกี่ยวกับทรัพยากรหรือการตั้งค่าที่ถูก deploy แล้วได้ ลองใช้งานคำสั่งเหล่านี้:

1. `List my Azure resource groups`
1. `#foundry list my current deployments`

คุณยังสามารถถามคำถามเกี่ยวกับเอกสารของ Azure และรับคำตอบที่อิงจากเซิร์ฟเวอร์ Microsoft Docs MCP ลองใช้คำสั่งเหล่านี้:

1. `#microsoft_docs_search What is Azure Developer CLI?`
1. `#microsoft_docs_search Show me a Python tutorial to chat with deployed model`

หรือขอโค้ดตัวอย่างเพื่อทำงานเฉพาะ ลองใช้คำสั่งนี้

1. `Give me a Python code example that uses AAD for an interactive chat client`

ในโหมด `Ask` จะให้โค้ดที่คุณสามารถคัดลอกและทดลองได้ ในโหมด `Agent` อาจทำขั้นตอนต่อไปด้วยการสร้างทรัพยากรที่จำเป็น รวมถึงสคริปต์ตั้งค่าและเอกสาร เพื่อช่วยคุณดำเนินงานนั้น

**คุณพร้อมแล้วที่จะเริ่มสำรวจเทมเพลตรีโพซิทอรี**

## 2. แยกสถาปัตยกรรม

??? prompt "ถาม: อธิบายสถาปัตยกรรมแอปพลิเคชันใน docs/images/architecture.png ใน 1 ย่อหน้า"

      แอปพลิเคชันนี้เป็นแอปแชตขับเคลื่อนด้วย AI ที่สร้างบน Azure ซึ่งแสดงถึงสถาปัตยกรรมแบบเอเย่นต์สมัยใหม่ โซลูชันนี้มุ่งเน้นไปที่ Azure Container App ที่โฮสต์โค้ดหลักของแอปพลิเคชัน ซึ่งประมวลผลข้อมูลที่ผู้ใช้ป้อนและสร้างคำตอบที่ฉลาดผ่านเอเย่นต์ AI
      
      สถาปัตยกรรมนี้ใช้ Microsoft Foundry Project เป็นฐานสำหรับความสามารถ AI โดยเชื่อมต่อกับ Azure AI Services ที่ให้โมเดลภาษาพื้นฐาน (เช่น GPT-4o-mini) และฟังก์ชันเอเย่นต์ การมีปฏิสัมพันธ์ของผู้ใช้จะไหลผ่าน frontend ที่พัฒนาด้วย React ไปยัง backend FastAPI ที่สื่อสารกับบริการเอเย่นต์ AI เพื่อสร้างคำตอบตามบริบท
      
      ระบบนี้รวมความสามารถดึงข้อมูลความรู้ผ่านการค้นหาไฟล์หรือ Azure AI Search service ทำให้เอเย่นต์เข้าถึงและอ้างอิงข้อมูลจากเอกสารที่อัปโหลด สำหรับความเป็นเลิศในการดำเนินงาน สถาปัตยกรรมรวมการตรวจสอบผ่าน Application Insights และ Log Analytics Workspace สำหรับการติดตาม บันทึก และการเพิ่มประสิทธิภาพ
      
      Azure Storage ให้บริการ blob storage สำหรับข้อมูลแอปพลิเคชันและการอัปโหลดไฟล์ ในขณะที่ Managed Identity ช่วยรักษาความปลอดภัยเข้าถึงระหว่างทรัพยากร Azure โดยไม่ต้องเก็บข้อมูลประจำตัว โซลูชันทั้งหมดออกแบบเพื่อรองรับการปรับขนาดและบำรุงรักษาง่าย โดยแอปพลิเคชันแบบคอนเทนเนอร์จะปรับขนาดโดยอัตโนมัติตามความต้องการ และมีความปลอดภัย การตรวจสอบ และความสามารถ CI/CD ผ่านระบบนิเวศบริการจัดการของ Azure

### 3.1 สถาปัตยกรรมแอปหลัก

เทมเพลตนี้ใช้รูปแบบ **เว็บแอปแบบฟูลสแตก** โดยมี:

- **Backend**: Python FastAPI พร้อมการผสาน Azure AI
- **Frontend**: TypeScript/React พร้อมระบบ build ด้วย Vite
- **โครงสร้างพื้นฐาน**: เทมเพลต Azure Bicep สำหรับทรัพยากรคลาวด์
- **คอนเทนเนอร์**: Docker สำหรับการ deploy ที่เสถียร

### 3.2 โครงสร้างพื้นฐานเป็นโค้ด (bicep)

เลเยอร์โครงสร้างพื้นฐานใช้เทมเพลต **Azure Bicep** ที่จัดแบบโมดูลาร์:

   - **`main.bicep`**: ควบคุมทรัพยากร Azure ทั้งหมด
   - **โมดูล `core/`**: ส่วนประกอบนำกลับมาใช้ซ้ำกับบริการต่าง ๆ
      - บริการ AI (Azure OpenAI, AI Search)
      - โฮสต์คอนเทนเนอร์ (Azure Container Apps)
      - การตรวจสอบ (Application Insights, Log Analytics)
      - ความปลอดภัย (Key Vault, Managed Identity)

### 3.3 ซอร์สแอปพลิเคชัน (`src/`)

**Backend API (`src/api/`)**:

- REST API บน FastAPI
- ผสานเอเย่นต์ Foundry
- การจัดการดัชนีค้นหาสำหรับดึงความรู้
- ความสามารถอัปโหลดและประมวลผลไฟล์

**Frontend (`src/frontend/`)**:

- SPA สมัยใหม่ React/TypeScript
- Vite สำหรับพัฒนาเร็วและการ build ที่ปรับแต่ง
- อินเทอร์เฟซแชตสำหรับการโต้ตอบกับเอเย่นต์

**ฐานความรู้ (`src/files/`)**:

- ตัวอย่างข้อมูลลูกค้าและผลิตภัณฑ์
- แสดงการดึงความรู้จากไฟล์
- ตัวอย่างไฟล์ JSON และ Markdown


### 3.4 DevOps และการอัตโนมัติ

**สคริปต์ (`scripts/`)**:

- สคริปต์ PowerShell และ Bash ข้ามแพลตฟอร์ม
- ตรวจสอบและตั้งค่าสภาพแวดล้อม
- กำหนดค่าหลัง deployment
- จัดการโควต้ารุ่น

**ผสาน Azure Developer CLI**:

- การกำหนดค่า `azure.yaml` สำหรับเวิร์กโฟลว์ `azd`
- การจัดสรรและการ deploy อัตโนมัติ
- การจัดการตัวแปรสภาพแวดล้อม

### 3.5 การทดสอบและการประกันคุณภาพ

**เฟรมเวิร์กประเมินผล (`evals/`)**:

- การประเมินประสิทธิภาพเอเย่นต์
- การทดสอบคุณภาพการตอบกลับคำถาม
- ระบบประเมินอัตโนมัติ

**ความปลอดภัย AI (`airedteaming/`)**:

- การทดสอบทีมแดงเพื่อความปลอดภัย AI
- การสแกนหาช่องโหว่ด้านความปลอดภัย
- แนวปฏิบัติ AI ที่รับผิดชอบ

## 4. ขอแสดงความยินดี 🏆

- [X] เปิดใช้งาน GitHub Copilot สำหรับ Azure
- [X] เข้าใจสถาปัตยกรรมแอปพลิเคชัน
