## 1. บทนำ

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) หรือ `azd` คือเครื่องมือคำสั่งแบบ open-source ที่ช่วยให้งานนักพัฒนาราบรื่นขึ้นเมื่อสร้างและปรับใช้แอปพลิเคชันไปยัง Azure

[AZD Templates](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) คือที่เก็บโค้ดตัวอย่างมาตรฐานที่รวมโค้ดแอปพลิเคชัน, ทรัพย์สิน _infrastructure-as-code_ และไฟล์กำหนดค่า `azd` สำหรับสถาปัตยกรรมโซลูชันที่สอดคล้องกัน การออกโครงสร้างพื้นฐานง่ายเหมือนใช้คำสั่ง `azd provision` - ขณะที่ `azd up` ช่วยคุณ provision โครงสร้างพื้นฐาน **และ** ปรับใช้แอปพลิเคชันครั้งเดียว!

ด้วยเหตุนี้ การเริ่มพัฒนาแอปพลิเคชันของคุณจึงง่ายเพียงแค่หาต้นแบบ _AZD Starter template_ ที่เหมาะสมกับแอปพลิเคชันและโครงสร้างพื้นฐานของคุณ แล้วปรับแต่งที่เก็บโค้ดให้ตรงตามความต้องการของสถานการณ์

ก่อนเริ่ม ให้แน่ใจว่าคุณติดตั้ง Azure Developer CLI เรียบร้อยแล้ว

1. เปิดเทอร์มินัล VS Code และพิมพ์คำสั่งนี้:

      ```bash title="" linenums="0"
      azd version
      ```

1. คุณจะเห็นบางอย่างแบบนี้!

      ```bash title="" linenums="0"
      azd version 1.27.1 (commit <current-build>)
      ```

**ตอนนี้คุณพร้อมที่จะเลือกและปรับใช้เทมเพลตด้วย azd แล้ว**

## 2. การเลือกเทมเพลต

แพลตฟอร์ม Microsoft Foundry มาพร้อมกับ [ชุดเทมเพลต AZD ที่แนะนำ](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) ซึ่งครอบคลุมสถานการณ์โซลูชันยอดนิยมเช่น _การทำงานอัตโนมัติแบบ multi-agent_ และ _การประมวลผลเนื้อหาแบบ multi-modal_ คุณยังสามารถค้นพบเทมเพลตเหล่านี้ได้โดยเข้าเยี่ยมชมพอร์ทัล Microsoft Foundry

1. ไปที่ [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. ลงชื่อเข้าใช้พอร์ทัล Microsoft Foundry เมื่อได้รับแจ้ง - คุณจะเห็นบางอย่างแบบนี้

![Pick](../../../../../translated_images/th/01-pick-template.60d2d5fff5ebc374.webp)


ตัวเลือก **Basic** คือเทมเพลตเริ่มต้นของคุณ:

1. [ ] [เริ่มต้นด้วย AI Chat](https://github.com/Azure-Samples/get-started-with-ai-chat) ซึ่งปรับใช้แอปแชทพื้นฐาน _พร้อมข้อมูลของคุณ_ ไปยัง Azure Container Apps ใช้สิ่งนี้เพื่อสำรวจสถานการณ์ chatbot AI พื้นฐาน
1. [X] [เริ่มต้นด้วย AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) ที่ปรับใช้ AI Agent มาตรฐาน (พร้อม Foundry Agents) ใช้สิ่งนี้เพื่อทำความคุ้นเคยกับโซลูชัน agentic AI ที่เกี่ยวข้องกับเครื่องมือและโมเดล

เปิดลิงก์ที่สองในแท็บเบราว์เซอร์ใหม่ (หรือคลิก `Open in GitHub` สำหรับการ์ดที่เกี่ยวข้อง) คุณควรเห็นที่เก็บเทมเพลต AZD นี้ ใช้เวลาสักครู่เพื่อสำรวจ README สถาปัตยกรรมแอปพลิเคชันเป็นดังนี้:

![Arch](../../../../../translated_images/th/architecture.8cec470ec15c65c7.webp)

## 3. การเปิดใช้งานเทมเพลต

ลองปรับใช้เทมเพลตนี้และตรวจสอบว่าเป็นเทมเพลตที่ถูกต้อง เราจะปฏิบัติตามคำแนะนำในส่วน [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started)

1. เลือกสภาพแวดล้อมทำงานสำหรับที่เก็บเทมเพลต:

      - **GitHub Codespaces**: คลิก [ลิงก์นี้](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) และยืนยัน `Create codespace`
      - **โคลนในเครื่องหรือ dev container**: โคลน `Azure-Samples/get-started-with-ai-agents` และเปิดใน VS Code

1. รอจนกว่าเทอร์มินัล VS Code พร้อม จากนั้นพิมพ์คำสั่งนี้:

   ```bash title="" linenums="0"
   azd up
   ```

ดำเนินการตามขั้นตอนใน workflow ที่จะเริ่มทำงาน:

1. คุณจะได้รับแจ้งให้เข้าสู่ระบบ Azure - ทำตามคำแนะนำเพื่อลงชื่อเข้าใช้
1. ป้อนชื่อสภาพแวดล้อมที่ไม่ซ้ำกันสำหรับคุณ - เช่น ฉันใช้ `nitya-mshack-azd`
1. สิ่งนี้จะสร้างโฟลเดอร์ `.azure/` - คุณจะเห็นโฟลเดอร์ย่อยที่มีชื่อตามสภาพแวดล้อม
1. คุณจะได้รับแจ้งให้เลือกชื่อบัญชีการสมัครสมาชิก - เลือกค่าเริ่มต้น
1. คุณจะถูกขอให้เลือกตำแหน่งที่ตั้ง - ใช้ `East US 2`

ตอนนี้ รอให้การ provisioning เสร็จสิ้น **ซึ่งใช้เวลาประมาณ 10-15 นาที**

1. เมื่อเสร็จแล้ว คอนโซลของคุณจะแสดงข้อความ SUCCESS เช่นนี้:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. ตอนนี้พอร์ทัล Azure ของคุณจะมี resource group ที่ provision แล้วพร้อมชื่อตามสภาพแวดล้อมนั้น:

      ![Infra](../../../../../translated_images/th/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **ตอนนี้คุณพร้อมที่จะตรวจสอบโครงสร้างพื้นฐานและแอปพลิเคชันที่ปรับใช้แล้ว**

## 4. การตรวจสอบเทมเพลต

1. ไปที่หน้า Azure Portal [Resource Groups](https://portal.azure.com/#browse/resourcegroups) - ลงชื่อเข้าใช้เมื่อได้รับแจ้ง
1. คลิกที่ RG สำหรับชื่อสภาพแวดล้อมของคุณ - คุณจะเห็นหน้าดังที่กล่าวข้างต้น

      - คลิกที่ทรัพยากร Azure Container Apps
      - คลิกที่ URL แอปพลิเคชันในส่วน _Essentials_ (ด้านบนขวา)

1. คุณควรเห็น UI หน้าแอปที่โฮสต์ไว้ดังนี้:

   ![App](../../../../../translated_images/th/03-test-application.471910da12c3038e.webp)

1. ลองถาม [คำถามตัวอย่าง](https://github.com/Azure-Samples/get-started-with-ai-agents/blob/main/docs/sample_questions.md) สองสามข้อ

      1. ถาม: ```เมืองหลวงของฝรั่งเศสคืออะไร?```
      1. ถาม: ```เต็นท์ที่ดีที่สุดราคาต่ำกว่า $200 สำหรับสองคนคืออะไร และมีฟีเจอร์อะไรบ้าง?```

1. คุณควรได้รับคำตอบที่คล้ายกับที่แสดงด้านล่าง _แต่มันทำงานอย่างไร?_

## 5. การตรวจสอบ Agent

Azure Container App ปรับใช้อินเทอร์เฟซที่เชื่อมต่อกับ AI Agent ที่ถูก provision ในโปรเจกต์ Microsoft Foundry สำหรับเทมเพลตนี้ ลองดูความหมายของสิ่งนั้น

1. กลับไปที่หน้า _Overview_ ของ resource group ใน Azure Portal

1. คลิกที่ทรัพยากร `Microsoft Foundry` ในรายการนั้น

1. คุณควรเห็นสิ่งนี้ คลิกที่ปุ่ม `Go to Microsoft Foundry Portal`
   ![Foundry](../../../../../translated_images/th/04-view-foundry-project.fb94ca41803f28f3.webp)

1. คุณจะเห็นหน้า Foundry Project สำหรับแอป AI ของคุณ
   ![Project](../../../../../translated_images/th/05-visit-foundry-portal.d734e98135892d7e.webp)

1. คลิกที่ `Agents` - คุณจะเห็น Agent เริ่มต้นที่ provision ในโปรเจกต์ของคุณ
   ![Agents](../../../../../translated_images/th/06-visit-agents.bccb263f77b00a09.webp)

1. เลือกมัน - และคุณจะเห็นรายละเอียดของ Agent โปรดสังเกต:

      - Agent ใช้ File Search เป็นค่าเริ่มต้น (เสมอ)
      - `Knowledge` ของ Agent แสดงว่ามีไฟล์ 32 ไฟล์ที่อัปโหลดแล้ว (สำหรับการค้นหาไฟล์)
      ![Agents](../../../../../translated_images/th/07-view-agent-details.0e049f37f61eae62.webp)

1. มองหาตัวเลือก `Data+indexes` ในเมนูด้านซ้ายแล้วคลิกดูรายละเอียด

      - คุณควรเห็นไฟล์ข้อมูล 32 ไฟล์ที่อัปโหลดสำหรับความรู้
      - ไฟล์เหล่านี้ตรงกับไฟล์ลูกค้า 12 ไฟล์ และไฟล์สินค้า 20 ไฟล์ในโฟลเดอร์ `src/files`
      ![Data](../../../../../translated_images/th/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**คุณตรวจสอบการทำงานของ Agent สำเร็จแล้ว!**

1. การตอบกลับของ Agent มีพื้นฐานมาจากความรู้ที่อยู่ในไฟล์เหล่านั้น
1. คุณสามารถถามคำถามที่เกี่ยวข้องกับข้อมูลนั้นและได้รับคำตอบที่มีพื้นฐาน
1. ตัวอย่าง: `customer_info_10.json` อธิบายการซื้อ 3 รายการโดย "Amanda Perez"

กลับไปที่แท็บเบราว์เซอร์ที่มี Container App endpoint และถามว่า: `Amanda Perez เป็นเจ้าของสินค้าประเภทใดบ้าง?` คุณควรเห็นแบบนี้:

![Data](../../../../../translated_images/th/09-ask-in-aca.4102297fc465a4d5.webp)

## 6. Agent Playground

มาสร้างความเข้าใจเพิ่มเติมเกี่ยวกับความสามารถของ Microsoft Foundry โดยนำ Agent ไปลองใน Agents Playground

1. กลับไปที่หน้า `Agents` ใน Microsoft Foundry - เลือก Agent เริ่มต้น
1. คลิกตัวเลือก `Try in Playground` - คุณจะเห็น UI Playground แบบนี้
1. ถามคำถามเดียวกัน: `Amanda Perez เป็นเจ้าของสินค้าประเภทใดบ้าง?`

    ![Data](../../../../../translated_images/th/09-ask-in-playground.a1b93794f78fa676.webp)

คุณจะได้รับคำตอบเดียวกัน (หรือคล้ายกัน) - แต่นอกจากนี้ยังได้รับข้อมูลเพิ่มเติมที่ช่วยให้เข้าใจคุณภาพ, ค่าใช้จ่าย และประสิทธิภาพของแอป agentic ตัวอย่างเช่น:

1. สังเกตว่าคำตอบอ้างถึงไฟล์ข้อมูลที่ใช้เพื่อ "ยืนยัน" คำตอบ
1. เลื่อนเมาส์ชี้ที่ป้ายไฟล์เหล่านี้ - ข้อมูลตรงกับคำถามและคำตอบที่แสดงหรือไม่?

คุณยังจะเห็นแถว _สถิติ_ อยู่ใต้คำตอบ

1. เลื่อนเมาส์ไปที่เมตริก เช่น Safety คุณจะเห็นแบบนี้
1. การประเมินนี้ตรงกับความรู้สึกของคุณเกี่ยวกับระดับความปลอดภัยของคำตอบหรือไม่?

      ![Data](../../../../../translated_images/th/10-view-run-info-meter.6cdb89a0eea5531f.webp)

## 7. การสังเกตการณ์ในตัว

การสังเกตการณ์หมายถึงการติดตั้งเครื่องมือในแอปของคุณเพื่อสร้างข้อมูลที่ใช้ในการเข้าใจ, ดีบัก และปรับปรุงการทำงานของแอป เพื่อให้คุณเข้าใจสิ่งนี้:

1. คลิกปุ่ม `View Run Info` - คุณควรเห็นมุมมองนี้ นี่คือตัวอย่างของ [Agent tracing](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) ในการปฏิบัติ _คุณยังสามารถดูมุมมองนี้โดยคลิก Thread Logs ในเมนูระดับบน_

   - เข้าใจขั้นตอนและเครื่องมือที่ Agent ใช้งาน
   - เข้าใจจำนวน Token ทั้งหมด (เทียบกับ Token ที่ใช้ในการตอบกลับ)
   - เข้าใจความล่าช้าและเวลาที่ใช้ในการประมวลผล

      ![Agent](../../../../../translated_images/th/10-view-run-info.b20ebd75fef6a1cc.webp)

1. คลิกแท็บ `Metadata` เพื่อดูคุณลักษณะเพิ่มของการรันงาน ซึ่งอาจมีประโยชน์ในการดีบักปัญหาภายหลัง

      ![Agent](../../../../../translated_images/th/11-view-run-info-metadata.7966986122c7c2df.webp)


1. คลิกแท็บ `Evaluations` เพื่อดูการประเมินอัตโนมัติของการตอบกลับ Agent ซึ่งรวมถึงการประเมินความปลอดภัย (เช่น การทำร้ายตนเอง) และการประเมินเฉพาะ Agent (เช่น การแก้ไขเจตนา, การติดตามงาน)

      ![Agent](../../../../../translated_images/th/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. สุดท้าย คลิกแท็บ `Monitoring` ในเมนูด้านข้าง

      - เลือกแท็บ `Resource usage` ในหน้าที่แสดง - แล้วดูข้อมูลเมตริก
      - ติดตามการใช้งานแอปในแง่ของค่าใช้จ่าย (tokens) และภาระงาน (requests)
      - ติดตามความล่าช้าของแอปจากไบต์แรก (การประมวลผลอินพุต) ถึงไบต์สุดท้าย (ผลลัพธ์)

      ![Agent](../../../../../translated_images/th/13-monitoring-resources.5148015f7311807f.webp)

## 8. ตัวแปรสภาพแวดล้อม

จนถึงตอนนี้ เราได้ผ่านขั้นตอนการปรับใช้ในเบราว์เซอร์ และตรวจสอบว่าโครงสร้างพื้นฐานได้รับการ provision แล้วและแอปพลิเคชันทำงานได้ แต่เพื่อทำงานกับ _โค้ดก่อน_ เราจำเป็นต้องตั้งค่าตัวแปรที่เกี่ยวข้องในสภาพแวดล้อมพัฒนาในเครื่องของเรา การใช้ `azd` ทำให้ง่าย

1. Azure Developer CLI [ใช้ตัวแปรสภาพแวดล้อม](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) เพื่อจัดเก็บและจัดการการตั้งค่ากำหนดค่าสำหรับการปรับใช้แอป

1. ตัวแปรสภาพแวดล้อมถูกเก็บไว้ใน `.azure/<env-name>/.env` - ซึ่งจำกัดเฉพาะสภาพแวดล้อม `env-name` ที่ใช้ในระหว่างการปรับใช้ และช่วยให้แยกแยะสภาพแวดล้อมระหว่างเป้าหมายการปรับใช้ที่ต่างกันในที่เก็บเดียวกัน

1. ตัวแปรสภาพแวดล้อมจะถูกโหลดโดยอัตโนมัติเมื่อใช้คำสั่ง `azd` ทุกครั้งที่รันคำสั่งเฉพาะ (เช่น `azd up`) โปรดทราบว่า `azd` จะไม่อ่านตัวแปรสภาพแวดล้อมระดับระบบปฏิบัติการ (เช่น ตั้งในเชลล์) โดยอัตโนมัติ - ให้ใช้คำสั่ง `azd set env` และ `azd get env` เพื่อถ่ายโอนข้อมูลภายในสคริปต์


มาลองใช้คำสั่งบางคำสั่งกัน:

1. ดึงตัวแปรสภาพแวดล้อมทั้งหมดที่ตั้งค่าไว้สำหรับ `azd` ในสภาพแวดล้อมนี้:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      คุณจะเห็นบางอย่างแบบนี้:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. ดึงค่าที่เฉพาะเจาะจง - เช่น ฉันอยากรู้ว่าได้ตั้งค่า `AZURE_AI_AGENT_MODEL_NAME` หรือไม่

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      คุณจะเห็นสิ่งนี้ - ซึ่งไม่ได้ตั้งค่าโดยค่าเริ่มต้น!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. ตั้งค่าตัวแปรสภาพแวดล้อมใหม่สำหรับ `azd` ที่นี่อัปเดตชื่อโมเดล Agent _หมายเหตุ: การเปลี่ยนแปลงใด ๆ จะถูกบันทึกทันทีในไฟล์ `.azure/<env-name>/.env`

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      ตอนนี้ลองดูค่าว่าตั้งค่าแล้ว:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. โปรดทราบว่าทรัพยากรบางตัวเป็นแบบถาวร (เช่น การปรับใช้โมเดล) และอาจต้องมากกว่าการสั่ง `azd up` เพื่อบังคับให้ปรับใช้ใหม่ ลองทำการลบการปรับใช้เดิมแล้วปรับใช้ใหม่พร้อมกับตัวแปรสภาพแวดล้อมที่เปลี่ยนแปลงดู

1. **รีเฟรช** หากคุณเคยปรับใช้โครงสร้างพื้นฐานด้วยเทมเพลต azd มาก่อน - คุณสามารถ _รีเฟรช_ สถานะตัวแปรสภาพแวดล้อมในเครื่องของคุณให้ตรงกับสถานะปัจจุบันของการปรับใช้ Azure โดยใช้คำสั่งนี้:

      ```bash title="" linenums="0"
      azd env refresh
      ```


      นี่เป็นวิธีที่ทรงพลังในการ _ซิงค์_ ตัวแปรสภาพแวดล้อมข้ามสภาพแวดล้อมการพัฒนาท้องถิ่นสองแห่งขึ้นไป (เช่น ทีมที่มีนักพัฒนาหลายคน) - ช่วยให้โครงสร้างพื้นฐานที่ใช้งานจริงเป็นข้อมูลอ้างอิงที่ถูกต้องสำหรับสถานะตัวแปร env สมาชิกทีมเพียงแค่ _รีเฟรช_ ตัวแปรเพื่อให้กลับมาอยู่ในซิงค์กัน
