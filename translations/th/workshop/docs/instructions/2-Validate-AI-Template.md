## 1. บทนำ

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) หรือ `azd` เป็นเครื่องมือบรรทัดคำสั่งโอเพนซอร์สที่ช่วยให้กระบวนการทำงานของนักพัฒนาซอฟต์แวร์เมื่อสร้างและปรับใช้แอปพลิเคชันไปยัง Azure รวดเร็วขึ้น

[แม่แบบ AZD](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) เป็นที่เก็บมาตรฐานที่รวมโค้ดตัวอย่างแอปพลิเคชัน, ทรัพยากร _infrastructure-as-code_ และไฟล์กำหนดค่าของ `azd` สำหรับสถาปัตยกรรมโซลูชันที่สอดคล้องกัน การจัดเตรียมโครงสร้างพื้นฐานจึงง่ายเพียงคำสั่ง `azd provision` — ในขณะเดียวกัน การใช้ `azd up` ช่วยให้คุณจัดเตรียมโครงสร้างพื้นฐาน **และ** ปรับใช้แอปพลิเคชันของคุณได้ในครั้งเดียว!

ดังนั้น การเริ่มต้นพัฒนาแอปพลิเคชันของคุณจึงง่ายเหมือนการค้นหาแม่แบบ _AZD Starter_ ที่ใกล้เคียงกับแอปพลิเคชันและความต้องการโครงสร้างพื้นฐานของคุณที่สุด — แล้วปรับแต่งที่เก็บโค้ดให้เหมาะกับความต้องการของสถานการณ์คุณ

ก่อนจะเริ่มกัน มาแน่ใจว่าคุณติดตั้ง Azure Developer CLI เรียบร้อยแล้ว

1. เปิดเทอร์มินัลใน VS Code และพิมพ์คำสั่งนี้:

      ```bash title="" linenums="0"
      azd version
      ```

1. คุณจะเห็นข้อความประมาณนี้!

      ```bash title="" linenums="0"
      azd version 1.19.0 (commit b3d68cea969b2bfbaa7b7fa289424428edb93e97)
      ```

**ตอนนี้คุณพร้อมที่จะเลือกและปรับใช้แม่แบบด้วย azd แล้ว**

## 2. การเลือกแม่แบบ

แพลตฟอร์ม Microsoft Foundry มาพร้อมกับ [ชุดแม่แบบ AZD ที่แนะนำ](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) ที่ครอบคลุมสถานการณ์โซลูชันยอดนิยม เช่น _การทำงานอัตโนมัติแบบหลายเอเย่นต์_ และ _การประมวลผลเนื้อหาหลายโหมด_ คุณยังสามารถค้นพบแม่แบบเหล่านี้ได้โดยเข้าไปที่พอร์ทัล Microsoft Foundry

1. เข้าไปที่ [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. เข้าสู่ระบบพอร์ทัล Microsoft Foundry เมื่อมีการแจ้ง — คุณจะเห็นหน้าตาแบบนี้

![Pick](../../../../../translated_images/th/01-pick-template.60d2d5fff5ebc374.webp)


ตัวเลือก **Basic** เป็นแม่แบบเริ่มต้นของคุณ:

1. [ ] [เริ่มต้นกับ AI Chat](https://github.com/Azure-Samples/get-started-with-ai-chat) ที่ปรับใช้แอปพลิเคชันแชทพื้นฐาน _พร้อมข้อมูลของคุณ_ ไปยัง Azure Container Apps ใช้เพื่อสำรวจสถานการณ์ chatbot AI พื้นฐาน
1. [X] [เริ่มต้นกับ AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) ที่ปรับใช้เอเย่นต์ AI มาตรฐาน (พร้อม Foundry Agents) ใช้เพื่อทำความคุ้นเคยกับโซลูชัน AI แบบ agent ที่ใช้เครื่องมือและโมเดล

เยี่ยมชมลิงก์ที่สองในแท็บเบราว์เซอร์ใหม่ (หรือคลิก `Open in GitHub` บนการ์ดที่เกี่ยวข้อง) คุณจะเห็นที่เก็บสำหรับแม่แบบ AZD นี้ ใช้เวลาสักครู่เพื่อสำรวจ README สถาปัตยกรรมแอปพลิเคชันมีหน้าตาแบบนี้:

![Arch](../../../../../translated_images/th/architecture.8cec470ec15c65c7.webp)

## 3. การเปิดใช้งานแม่แบบ

มาลองปรับใช้แม่แบบนี้และตรวจสอบว่าใช้งานได้ เราจะทำตามแนวทางในส่วน [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started)

1. คลิก [ลิงก์นี้](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) — ยืนยันการดำเนินการเริ่มต้น `Create codespace`
1. หน้านี้จะเปิดในแท็บเบราว์เซอร์ใหม่ — รอให้เซสชัน GitHub Codespaces โหลดเสร็จ
1. เปิดเทอร์มินัล VS Code ใน Codespaces — พิมพ์คำสั่งต่อไปนี้:

   ```bash title="" linenums="0"
   azd up
   ```

ดำเนินขั้นตอนตามเวิร์กโฟลวที่เกิดขึ้น:

1. ระบบจะขอให้คุณเข้าสู่ระบบ Azure — ทำตามคำแนะนำเพื่อลงชื่อเข้าใช้
1. ป้อนชื่อสภาพแวดล้อมที่ไม่ซ้ำกันสำหรับคุณ — เช่น ฉันใช้ `nitya-mshack-azd`
1. จะสร้างโฟลเดอร์ `.azure/` — คุณจะเห็นโฟลเดอร์ย่อยที่มีชื่อสภาพแวดล้อม
1. จะมีการขอให้คุณเลือกชื่อการสมัครใช้งาน — เลือกค่าเริ่มต้น
1. จะขอให้คุณเลือกตำแหน่ง — ใช้ `East US 2`

ตอนนี้ รอการจัดเตรียมให้เสร็จสมบูรณ์ **ใช้เวลาประมาณ 10-15 นาที**

1. เมื่อเสร็จแล้ว คอนโซลของคุณจะแสดงข้อความ SUCCESS แบบนี้:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. ตอนนี้ใน Azure Portal ของคุณจะมี resource group ที่จัดเตรียมไว้แล้วชื่อสภาพแวดล้อมนั้น:

      ![Infra](../../../../../translated_images/th/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **ตอนนี้คุณพร้อมที่จะตรวจสอบโครงสร้างพื้นฐานและแอปพลิเคชันที่ปรับใช้แล้ว**

## 4. การตรวจสอบแม่แบบ

      - คลิกที่ทรัพยากร Azure Container Apps
      - คลิกที่ URL ของแอปพลิเคชันในส่วน _Essentials_ (มุมบนขวา)

1. คุณควรเห็น UI หน้าแอปพลิเคชันโฮสต์แบบนี้:

   ![App](../../../../../translated_images/th/03-test-application.471910da12c3038e.webp)

      1. ถาม: ```What is the capital of France?```  
      1. ถาม: ```What's the best tent under $200 for two people, and what features does it include?```

## 5. การตรวจสอบ Agent

Azure Container App ปรับใช้ endpoint ที่เชื่อมต่อกับ AI Agent ที่จัดเตรียมไว้ในโปรเจกต์ Microsoft Foundry สำหรับแม่แบบนี้ มาดูกันว่าหมายความว่าอย่างไร

1. กลับไปที่หน้า _Overview_ ของ resource group ใน Azure Portal

1. คลิกที่ทรัพยากร `Microsoft Foundry` ในรายการนั้น

1. คุณจะเห็นหน้านี้ คลิกปุ่ม `Go to Microsoft Foundry Portal`  
   ![Foundry](../../../../../translated_images/th/04-view-foundry-project.fb94ca41803f28f3.webp)

1. คุณจะเห็นหน้า Foundry Project สำหรับแอปพลิเคชัน AI ของคุณ  
   ![Project](../../../../../translated_images/th/05-visit-foundry-portal.d734e98135892d7e.webp)

1. คลิกที่ `Agents` — คุณจะเห็นเอเย่นต์เริ่มต้นที่จัดเตรียมไว้ในโปรเจกต์  
   ![Agents](../../../../../translated_images/th/06-visit-agents.bccb263f77b00a09.webp)

1. เลือกเอเย่นต์นั้น — คุณจะเห็นรายละเอียดเอเย่นต์ โปรดสังเกต:

      - เอเย่นต์ใช้ File Search ตามค่าเริ่มต้น (เสมอ)
      - ตัว `Knowledge` ของเอเย่นต์แสดงว่ามีไฟล์อัปโหลด 32 ไฟล์ (สำหรับการค้นหาไฟล์)
      ![Agents](../../../../../translated_images/th/07-view-agent-details.0e049f37f61eae62.webp)

1. มองหาตัวเลือก `Data+indexes` ในเมนูด้านซ้ายและคลิกดูรายละเอียด

      - คุณจะเห็นไฟล์ข้อมูล 32 ไฟล์ที่อัปโหลดเพื่อความรู้
      - ไฟล์เหล่านี้ตรงกับ 12 ไฟล์ลูกค้าและ 20 ไฟล์ผลิตภัณฑ์ภายใต้ `src/files`
      ![Data](../../../../../translated_images/th/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**คุณตรวจสอบการทำงานของ Agent เรียบร้อยแล้ว!**

1. คำตอบของเอเย่นต์ยึดอยู่บนความรู้ที่มีในไฟล์เหล่านั้น
1. คุณสามารถถามคำถามที่เกี่ยวข้องกับข้อมูลนั้นและรับคำตอบที่ยึดความรู้ได้
1. ตัวอย่าง: `customer_info_10.json` อธิบายการซื้อ 3 รายการโดย "Amanda Perez"

กลับไปที่แท็บเบราว์เซอร์ที่มี endpoint ของ Container App และถามว่า: `What products does Amanda Perez own?` คุณจะเห็นประมาณนี้:

![Data](../../../../../translated_images/th/09-ask-in-aca.4102297fc465a4d5.webp)

## 6. สนามเล่น Agent

มาเสริมความเข้าใจความสามารถของ Microsoft Foundry โดยการทดลอง Agent ใน Agents Playground

1. กลับไปที่หน้า `Agents` ใน Microsoft Foundry — เลือกเอเย่นต์เริ่มต้น
1. คลิกตัวเลือก `Try in Playground` — คุณจะเห็น UI Playground แบบนี้
1. ถามคำถามเดียวกัน: `What products does Amanda Perez own?`

    ![Data](../../../../../translated_images/th/09-ask-in-playground.a1b93794f78fa676.webp)

คุณจะได้รับคำตอบเหมือนกัน (หรือคล้ายกัน) — แต่คุณจะได้รับข้อมูลเพิ่มเติมที่ช่วยให้เข้าใจคุณภาพ, ค่าใช้จ่าย และประสิทธิภาพของแอปที่มีลักษณะ agent เช่นนี้ เช่น:

1. สังเกตว่าคำตอบอ้างอิงไฟล์ข้อมูลที่ใช้เป็นฐานสำหรับคำตอบ
1. เลื่อนเมาส์ไปวางบนป้ายไฟล์เหล่านี้ — ข้อมูลตรงกับคำถามและคำตอบที่แสดงหรือไม่?

คุณยังเห็นแถว _stats_ ด้านล่างคำตอบ

1. เลื่อนเมาส์ไปบนเมตริกใด ๆ — เช่น Safety คุณจะเห็นแบบนี้
1. การประเมินนี้ตรงกับความรู้สึกของคุณเกี่ยวกับระดับความปลอดภัยของคำตอบหรือไม่?

      ![Data](../../../../../translated_images/th/10-view-run-info-meter.6cdb89a0eea5531f.webp)

## 7. การสังเกตการณ์ในตัว

การสังเกตการณ์หมายถึงการติดตั้งเครื่องมือในแอปพลิเคชันของคุณเพื่อสร้างข้อมูลที่ใช้ในการเข้าใจ, แก้ไขข้อผิดพลาด, และเพิ่มประสิทธิภาพการทำงาน มาทดลองดูกัน:

1. คลิกปุ่ม `View Run Info` — คุณจะเห็นมุมมองนี้ นี่คือตัวอย่างการ [ติดตาม Agent](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) ในการทำงาน _คุณยังสามารถดูมุมมองนี้ได้โดยคลิก Thread Logs ในเมนูระดับบน_  

   - รับรู้ขั้นตอนการทำงานและเครื่องมือที่เอเย่นต์ใช้
   - เข้าใจจำนวน Token รวม (เทียบกับ token ที่ถูกใช้ในผลลัพธ์)
   - เข้าใจความหน่วงเวลาและเวลาที่ใช้ในแต่ละขั้นตอน

      ![Agent](../../../../../translated_images/th/10-view-run-info.b20ebd75fef6a1cc.webp)

1. คลิกแท็บ `Metadata` เพื่อดูคุณลักษณะเพิ่มเติมของการทำงาน ซึ่งอาจมีประโยชน์สำหรับการแก้ไขข้อผิดพลาดในภายหลัง

      ![Agent](../../../../../translated_images/th/11-view-run-info-metadata.7966986122c7c2df.webp)

1. คลิกแท็บ `Evaluations` เพื่อดูการประเมินอัตโนมัติที่ทำกับคำตอบของเอเย่นต์ รวมถึงการประเมินความปลอดภัย (เช่น การทำร้ายตัวเอง) และการประเมินเฉพาะเอเย่นต์ (เช่น การระบุเจตนา, การปฏิบัติงานตามภารกิจ)

      ![Agent](../../../../../translated_images/th/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. สุดท้าย คลิกแท็บ `Monitoring` ในเมนูด้านข้าง

      - เลือกแท็บ `Resource usage` ในหน้าที่แสดง — ดูเมตริกต่าง ๆ
      - ติดตามการใช้งานแอปพลิเคชันในแง่ต้นทุน (tokens) และการโหลด (คำร้องขอ)
      - ติดตามความหน่วงเวลาของแอปพลิเคชันตั้งแต่ไบต์แรก (การประมวลผลอินพุต) ถึงไบต์สุดท้าย (ผลลัพธ์)

      ![Agent](../../../../../translated_images/th/13-monitoring-resources.5148015f7311807f.webp)

## 8. ตัวแปรสภาพแวดล้อม

จนถึงตอนนี้ เราได้ดำเนินการปรับใช้ในเบราว์เซอร์และตรวจสอบว่าโครงสร้างพื้นฐานพร้อมใช้งานและแอปพลิเคชันทำงานได้ แต่หากต้องการทำงานกับ _โค้ดแอปพลิเคชัน_ ก่อน ต้องตั้งค่าตัวแปรสภาพแวดล้อมของเครื่องท้องถิ่นที่จำเป็นสำหรับการทำงานกับทรัพยากรเหล่านี้ การใช้ `azd` ทำให้ง่ายขึ้น

1. Azure Developer CLI [ใช้ตัวแปรสภาพแวดล้อม](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) เพื่อเก็บและจัดการการตั้งค่ากำหนดค่าของการปรับใช้แอปพลิเคชัน

1. ตัวแปรสภาพแวดล้อมถูกเก็บไว้ใน `.azure/<env-name>/.env` — โดยกำหนดขอบเขตให้กับสภาพแวดล้อมชื่อ `env-name` ที่ใช้ระหว่างการปรับใช้ และช่วยแยกสภาพแวดล้อมระหว่างเป้าหมายการปรับใช้ในที่เก็บเดียวกัน

1. ตัวแปรสภาพแวดล้อมจะถูกโหลดโดยอัตโนมัติด้วยคำสั่ง `azd` ทุกครั้งที่รันคำสั่ง เช่น `azd up` โปรดทราบว่า `azd` ไม่ได้อ่านตัวแปรสภาพแวดล้อมในระดับระบบปฏิบัติการ (เช่น ที่ตั้งไว้ในเชลล์) โดยตรง — แต่ให้ใช้คำสั่ง `azd set env` และ `azd get env` เพื่อถ่ายโอนข้อมูลภายในสคริปต์

ลองใช้คำสั่งสักสองสามคำสั่งดู:

1. ดูตัวแปรสภาพแวดล้อมทั้งหมดที่ตั้งไว้สำหรับ `azd` ในสภาพแวดล้อมนี้:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      คุณจะเห็นบางอย่างเช่นนี้:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. ดูค่าตัวแปรเฉพาะ — เช่น ฉันอยากรู้ว่าเราได้ตั้งค่า `AZURE_AI_AGENT_MODEL_NAME` หรือไม่

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      คุณจะเห็นแบบนี้ — ค่าดังกล่าวไม่ได้ถูกตั้งค่าโดยค่าเริ่มต้น!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. ตั้งค่าตัวแปรสภาพแวดล้อมใหม่สำหรับ `azd` ตัวอย่างนี้คือการอัปเดตชื่อโมเดลของเอเย่นต์ _หมายเหตุ: การเปลี่ยนแปลงใด ๆ จะถูกสะท้อนทันทีในไฟล์ `.azure/<env-name>/.env`

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      ตอนนี้ ควรจะพบว่าค่าถูกตั้งค่าเรียบร้อย:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. โปรดทราบว่าบางทรัพยากรมีสถานะคงที่ (เช่น การปรับใช้โมเดล) และอาจต้องทำมากกว่าคำสั่ง `azd up` ธรรมดาเพื่อบังคับการปรับใช้ซ้ำ ลองลบการปรับใช้เดิมแล้วปรับใช้ใหม่พร้อมกับตัวแปรสภาพแวดล้อมที่เปลี่ยนไป

1. **รีเฟรช** หากคุณเคยปรับใช้โครงสร้างพื้นฐานโดยใช้แม่แบบ azd มาก่อน — คุณสามารถ _รีเฟรช_ สถานะตัวแปรสภาพแวดล้อมภายในเครื่องของคุณจากสถานะปัจจุบันของการปรับใช้ Azure ด้วยคำสั่งนี้:

      ```bash title="" linenums="0"
      azd env refresh
      ```

นี่เป็นวิธีที่ทรงพลังในการ _ซิงค์_ ตัวแปรสภาพแวดล้อมข้ามสองหรือมากกว่าของสภาพแวดล้อมการพัฒนาท้องถิ่น (เช่น ทีมที่มีนักพัฒนาหลายคน) - ช่วยให้โครงสร้างพื้นฐานที่ปรับใช้อยู่ทำหน้าที่เป็นความจริงพื้นฐานสำหรับสถานะของตัวแปร env สมาชิกในทีมเพียงแค่ _รีเฟรช_ ตัวแปรเพื่อให้กลับมาเป็นซิงค์
