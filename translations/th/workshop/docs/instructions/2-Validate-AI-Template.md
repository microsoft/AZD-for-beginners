## 1. บทนำ

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) หรือ `azd` เป็นเครื่องมือบรรทัดคำสั่งแบบโอเพ่นซอร์สที่ช่วยเพิ่มประสิทธิภาพกระบวนการทำงานของนักพัฒนาเมื่อสร้างและปรับใช้แอปพลิเคชันไปยัง Azure

[AZD Templates](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) เป็นที่เก็บมาตรฐานที่รวมโค้ดตัวอย่างแอปพลิเคชัน, สินทรัพย์ _infrastructure-as-code_ และไฟล์การกำหนดค่า `azd` สำหรับสถาปัตยกรรมโซลูชันที่สอดคล้องกัน การจัดเตรียมโครงสร้างพื้นฐานกลายเป็นเรื่องง่ายเพียงแค่ใช้คำสั่ง `azd provision` - ในขณะที่การใช้ `azd up` ช่วยให้คุณจัดเตรียมโครงสร้างพื้นฐาน **และ** ปรับใช้แอปพลิเคชันของคุณในครั้งเดียว!

ดังนั้น การเริ่มต้นกระบวนการพัฒนาแอปพลิเคชันของคุณสามารถทำได้ง่ายเพียงแค่ค้นหา _AZD Starter template_ ที่เหมาะสมกับความต้องการแอปพลิเคชันและโครงสร้างพื้นฐานของคุณมากที่สุด - แล้วปรับแต่งที่เก็บข้อมูลให้ตรงตามความต้องการของสถานการณ์ของคุณ

ก่อนที่เราจะเริ่ม ให้ตรวจสอบว่าคุณได้ติดตั้ง Azure Developer CLI แล้ว

1. เปิดเทอร์มินัล VS Code และพิมพ์คำสั่งนี้:

      ```bash title="" linenums="0"
      azd version
      ```

1. คุณควรเห็นอะไรบางอย่างเช่นนี้!

      ```bash title="" linenums="0"
      azd version 1.19.0 (commit b3d68cea969b2bfbaa7b7fa289424428edb93e97)
      ```

**คุณพร้อมแล้วที่จะเลือกและปรับใช้เทมเพลตด้วย azd**

## 2. เลือกเทมเพลต

แพลตฟอร์ม Microsoft Foundry มาพร้อมกับ [ชุดเทมเพลต AZD ที่แนะนำ](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) ซึ่งครอบคลุมสถานการณ์โซลูชันยอดนิยมเช่น _การดำเนินงานแบบหลายตัวแทน_ และ _การประมวลผลเนื้อหาหลายโหมด_ คุณยังสามารถค้นพบเทมเพลตเหล่านี้ได้โดยการเยี่ยมชมพอร์ทัล Microsoft Foundry

1. เยี่ยมชม [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. เข้าสู่ระบบพอร์ทัล Microsoft Foundry เมื่อมีการแจ้ง - คุณจะเห็นอะไรบางอย่างเช่นนี้

![Pick](../../../../../translated_images/th/01-pick-template.60d2d5fff5ebc374.webp)


ตัวเลือก **Basic** คือเทมเพลตเริ่มต้นของคุณ:

1. [ ] [เริ่มต้นกับ AI Chat](https://github.com/Azure-Samples/get-started-with-ai-chat) ที่ปรับใช้แอปแชทพื้นฐาน _พร้อมข้อมูลของคุณ_ บน Azure Container Apps ใช้เทมเพลตนี้เพื่อสำรวจสถานการณ์แชทบอท AI ขั้นพื้นฐาน
1. [X] [เริ่มต้นกับ AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) ที่ปรับใช้ AI Agent มาตรฐาน (พร้อม Foundry Agents) ใช้เทมเพลตนี้เพื่อทำความคุ้นเคยกับโซลูชัน AI แบบตัวแทนที่เกี่ยวข้องกับเครื่องมือและโมเดล

เปิดลิงก์ที่สองในแท็บเบราว์เซอร์ใหม่ (หรือคลิก `Open in GitHub` สำหรับการ์ดที่เกี่ยวข้อง) คุณควรเห็นที่เก็บสำหรับ AZD Template นี้ ใช้เวลาสักครู่เพื่อสำรวจ README สถาปัตยกรรมแอปพลิเคชันมีลักษณะดังนี้:

![Arch](../../../../../translated_images/th/architecture.8cec470ec15c65c7.webp)

## 3. การเปิดใช้งานเทมเพลต

ลองปรับใช้เทมเพลตนี้และตรวจสอบให้แน่ใจว่าถูกต้องตามข้อกำหนด เราจะปฏิบัติตามแนวทางในส่วน [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started)

1. คลิก [ลิงก์นี้](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) - ยืนยันการดำเนินการเริ่มต้น `Create codespace`
1. เบราว์เซอร์จะเปิดแท็บใหม่ - รอให้เซสชัน GitHub Codespaces โหลดเสร็จสิ้น
1. เปิดเทอร์มินัล VS Code ใน Codespaces - พิมพ์คำสั่งต่อไปนี้:

   ```bash title="" linenums="0"
   azd up
   ```

ดำเนินการขั้นตอนการทำงานที่คำสั่งนี้จะเปิดใช้งาน:

1. คุณจะได้รับแจ้งให้เข้าสู่ระบบ Azure - ทำตามคำแนะนำเพื่อยืนยันตัวตน
1. ป้อนชื่อสภาพแวดล้อมที่ไม่ซ้ำสำหรับคุณ - เช่น ฉันใช้ `nitya-mshack-azd`
1. จะสร้างโฟลเดอร์ `.azure/` - คุณจะเห็นโฟลเดอร์ย่อยที่มีชื่อสภาพแวดล้อม
1. คุณจะได้รับแจ้งให้เลือกชื่อสมัครสมาชิก - เลือกค่าดีฟอลต์
1. คุณจะถูกถามตำแหน่ง - ใช้ `East US 2`

ตอนนี้ รอให้การจัดเตรียมเสร็จสมบูรณ์ **ใช้เวลาประมาณ 10-15 นาที**

1. เมื่อเสร็จสิ้น คอนโซลจะแสดงข้อความ SUCCESS ดังนี้:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. ตอนนี้ใน Azure Portal ของคุณจะมี resource group ที่ถูกสร้างขึ้นโดยใช้ชื่อสภาพแวดล้อมนั้น:

      ![Infra](../../../../../translated_images/th/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **คุณพร้อมที่จะตรวจสอบโครงสร้างพื้นฐานและแอปพลิเคชันที่ปรับใช้แล้ว**

## 4. การตรวจสอบเทมเพลต

1. เยี่ยมชมหน้า [Resource Groups](https://portal.azure.com/#browse/resourcegroups) ใน Azure Portal - เข้าสู่ระบบเมื่อถูกขอ
1. คลิกที่ RG สำหรับชื่อสภาพแวดล้อมของคุณ - คุณจะเห็นหน้าดังรูปข้างบน

      - คลิกที่ทรัพยากร Azure Container Apps
      - คลิกที่ Application Url ในส่วน _Essentials_ (มุมขวาบน)

1. คุณควรเห็นส่วนติดต่อหน้าของแอปพลิเคชันโฮสต์แบบนี้:

   ![App](../../../../../translated_images/th/03-test-application.471910da12c3038e.webp)

1. ลองถาม [คำถามตัวอย่าง](https://github.com/Azure-Samples/get-started-with-ai-agents/blob/main/docs/sample_questions.md) สองสามคำถาม

      1. ถามว่า: ```เมืองหลวงของประเทศฝรั่งเศสคืออะไร?``` 
      1. ถามว่า: ```เต็นท์ที่ดีที่สุดที่มีราคาต่ำกว่า 200 ดอลลาร์สำหรับสองคนคืออะไร และประกอบด้วยคุณสมบัติอะไรบ้าง?```

1. คุณควรได้รับคำตอบที่คล้ายกับที่แสดงด้านล่างนี้ _แต่สิ่งนี้ทำงานอย่างไร?_

      ![App](../../../../../translated_images/th/03-test-question.521c1e863cbaddb6.webp)

## 5. การตรวจสอบ Agent

Azure Container App จะปรับใช้ endpoint ที่เชื่อมต่อกับ AI Agent ที่ถูกจัดเตรียมไว้ในโครงการ Microsoft Foundry สำหรับเทมเพลตนี้ มาดูกันว่าหมายความว่าอย่างไร

1. กลับไปที่หน้า _Overview_ ของกลุ่มทรัพยากรใน Azure Portal

1. คลิกที่ทรัพยากร `Microsoft Foundry` ในรายการนั้น

1. คุณควรเห็นแบบนี้ คลิกปุ่ม `Go to Microsoft Foundry Portal` 
   ![Foundry](../../../../../translated_images/th/04-view-foundry-project.fb94ca41803f28f3.webp)

1. คุณจะเห็นหน้าโครงการ Foundry สำหรับแอปพลิเคชัน AI ของคุณ
   ![Project](../../../../../translated_images/th/05-visit-foundry-portal.d734e98135892d7e.webp)

1. คลิกที่ `Agents` - คุณจะเห็น Agent เริ่มต้นที่ถูกจัดเตรียมในโครงการของคุณ
   ![Agents](../../../../../translated_images/th/06-visit-agents.bccb263f77b00a09.webp)

1. เลือกมัน - และคุณจะเห็นรายละเอียด Agent โปรดสังเกตสิ่งต่อไปนี้:

      - เอเจนต์ใช้ File Search เป็นค่ามาตรฐานเสมอ
      - `Knowledge` ของเอเจนต์บ่งชี้ว่ามีไฟล์ 32 ไฟล์ที่อัปโหลดไว้ (สำหรับการค้นหาไฟล์)
      ![Agents](../../../../../translated_images/th/07-view-agent-details.0e049f37f61eae62.webp)

1. มองหาเมนู `Data+indexes` ที่อยู่ฝั่งซ้ายและคลิกดูรายละเอียด

      - คุณจะเห็นไฟล์ข้อมูล 32 ไฟล์ที่อัปโหลดเพื่อใช้ใน knowledge
      - ไฟล์เหล่านี้จะสอดคล้องกับไฟล์ลูกค้า 12 ไฟล์และไฟล์ผลิตภัณฑ์ 20 ไฟล์ในโฟลเดอร์ `src/files`
      ![Data](../../../../../translated_images/th/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**คุณได้ตรวจสอบการทำงานของ Agent สำเร็จแล้ว!**

1. การตอบกลับของเอเจนต์มีฐานข้อมูลจากไฟล์เหล่านั้น
1. คุณสามารถถามคำถามที่เกี่ยวข้องกับข้อมูลนั้นและรับคำตอบที่มีพื้นฐานได้
1. ตัวอย่าง: `customer_info_10.json` อธิบายการซื้อ 3 รายการที่ทำโดย "Amanda Perez"

กลับไปที่แท็บเบราว์เซอร์ที่มี endpoint ของ Container App และถาม: `Amanda Perez มีผลิตภัณฑ์อะไรบ้าง?` คุณควรเห็นอะไรบางอย่างเช่นนี้:

![Data](../../../../../translated_images/th/09-ask-in-aca.4102297fc465a4d5.webp)

## 6. สนามเล่น Agent

ลองสร้างความเข้าใจเพิ่มเติมเกี่ยวกับความสามารถของ Microsoft Foundry โดยการทดลอง Agent ใน Agents Playground

1. กลับไปที่หน้าของ `Agents` ใน Microsoft Foundry - เลือกเอเจนต์เริ่มต้น
1. คลิกตัวเลือก `Try in Playground` - คุณจะเห็น UI สนามเล่นเช่นนี้
1. ถามคำถามเดิม: `Amanda Perez มีผลิตภัณฑ์อะไรบ้าง?`

    ![Data](../../../../../translated_images/th/09-ask-in-playground.a1b93794f78fa676.webp)

คุณจะได้รับคำตอบเดียวกัน (หรือคล้ายกัน) - แต่คุณจะได้รับข้อมูลเพิ่มเติมที่ช่วยให้คุณเข้าใจคุณภาพ, ค่าใช้จ่าย และประสิทธิภาพของแอปที่ใช้ agentic เช่น

1. สังเกตว่าการตอบกลับมีการอ้างอิงไฟล์ข้อมูลที่ใช้ "รองรับ" การตอบคำถาม
1. เลื่อนเมาส์ไปวางบนป้ายชื่อไฟล์เหล่านี้ - ข้อมูลตรงกับคำถามและคำตอบที่แสดงหรือไม่?

คุณยังจะเห็นแถว _stats_ อยู่ใต้คำตอบ

1. เลื่อนเมาส์ไปวางบนเมตริกใด ๆ - เช่น ความปลอดภัย คุณจะเห็นอะไรคล้ายๆ นี้
1. การประเมินตรงกับความรู้สึกของคุณเกี่ยวกับระดับความปลอดภัยของคำตอบหรือไม่?

      ![Data](../../../../../translated_images/th/10-view-run-info-meter.6cdb89a0eea5531f.webp)

## 7. การสังเกตการณ์ในตัว

การสังเกตการณ์หมายถึงการติดตั้งเครื่องมือในแอปพลิเคชันของคุณเพื่อสร้างข้อมูลที่สามารถใช้ในการทำความเข้าใจ, ตรวจสอบข้อผิดพลาด, และเพิ่มประสิทธิภาพการทำงาน เพื่อให้เข้าใจมากขึ้น:

1. คลิกปุ่ม `View Run Info` - คุณจะเห็นมุมมองนี้ นี่คือตัวอย่างของ [Agent tracing](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) ในการใช้งาน _คุณยังสามารถดูมุมมองนี้ได้โดยคลิก Thread Logs ในเมนูระดับบนสุด_

   - ทำความเข้าใจขั้นตอนการทำงานและเครื่องมือที่ agent ใช้
   - เข้าใจจำนวน Token รวม (เทียบกับการใช้งาน token ของผลลัพธ์) สำหรับการตอบกลับ
   - เข้าใจความหน่วงและเวลาที่ใช้ในการทำงานในแต่ละขั้นตอน

      ![Agent](../../../../../translated_images/th/10-view-run-info.b20ebd75fef6a1cc.webp)

1. คลิกแท็บ `Metadata` เพื่อดูแอตทริบิวต์เพิ่มเติมสำหรับการรันที่อาจเป็นประโยชน์ในการดีบักปัญหาในภายหลัง

      ![Agent](../../../../../translated_images/th/11-view-run-info-metadata.7966986122c7c2df.webp)


1. คลิกแท็บ `Evaluations` เพื่อดูการประเมินอัตโนมัติที่ทำกับคำตอบของ agent ซึ่งรวมถึงการประเมินความปลอดภัย (เช่น การทำร้ายตนเอง) และการประเมินเฉพาะของ agent (เช่น การระบุเจตนา, การปฏิบัติตามงาน)

      ![Agent](../../../../../translated_images/th/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. สุดท้าย คลิกแท็บ `Monitoring` ในเมนูด้านข้าง

      - เลือกแท็บ `Resource usage` ในหน้าที่แสดงขึ้น - และดูเมตริก
      - ติดตามการใช้งานแอปในแง่ของค่าใช้จ่าย (token) และปริมาณโหลด (คำขอ)
      - ติดตามความหน่วงของแอปตั้งแต่ไบต์แรก (การประมวลผลข้อมูลเข้า) ถึงไบต์สุดท้าย (ผลลัพธ์)

      ![Agent](../../../../../translated_images/th/13-monitoring-resources.5148015f7311807f.webp)

## 8. ตัวแปรสภาพแวดล้อม

จนถึงตอนนี้ เราได้เดินผ่านการปรับใช้ในเบราว์เซอร์และตรวจสอบว่าโครงสร้างพื้นฐานของเราได้รับการจัดเตรียมและแอปพลิเคชันทำงานได้แล้ว แต่เพื่อทำงานกับโค้ดแอปพลิเคชัน _แบบโค้ด-เฟิร์ส_ เราจำเป็นต้องกำหนดค่าสภาพแวดล้อมการพัฒนาท้องถิ่นของเราด้วยตัวแปรที่เกี่ยวข้องซึ่งจำเป็นสำหรับการทำงานร่วมกับทรัพยากรเหล่านี้ การใช้ `azd` ช่วยให้เรื่องนี้ง่ายขึ้น

1. Azure Developer CLI [ใช้ตัวแปรสภาพแวดล้อม](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) เพื่อเก็บและจัดการการตั้งค่าการกำหนดค่าสำหรับการปรับใช้แอปพลิเคชัน

1. ตัวแปรสภาพแวดล้อมจะถูกเก็บไว้ใน `.azure/<env-name>/.env` - ซึ่งจำกัดขอบเขตไว้กับสภาพแวดล้อมที่ใช้ระหว่างการปรับใช้และช่วยให้คุณแยกแยะสภาพแวดล้อมระหว่างเป้าหมายการปรับใช้ต่าง ๆ ใน repo เดียวกัน

1. ตัวแปรสภาพแวดล้อมจะถูกโหลดโดยอัตโนมัติโดยคำสั่ง `azd` ทุกครั้งที่มันเรียกใช้คำสั่งเฉพาะ (เช่น `azd up`) โปรดทราบว่า `azd` จะไม่อ่านตัวแปรสภาพแวดล้อมระดับระบบปฏิบัติการ (เช่น กำหนดในเชลล์) โดยอัตโนมัติ - แต่ใช้คำสั่ง `azd set env` และ `azd get env` เพื่อถ่ายโอนข้อมูลภายในสคริปต์แทน

ลองใช้คำสั่งบางอย่างกัน:

1. รับตัวแปรสภาพแวดล้อมทั้งหมดที่ตั้งค่าให้กับ `azd` ในสภาพแวดล้อมนี้:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      คุณจะเห็นอะไรบางอย่างเช่น:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4o-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. รับค่าตัวแปรเฉพาะ - เช่น ฉันต้องการทราบว่าได้ตั้งค่า `AZURE_AI_AGENT_MODEL_NAME` หรือไม่

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      คุณจะเห็นอะไรอย่างนี้ - ซึ่งไม่ได้ตั้งค่าโดยค่าเริ่มต้น!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. ตั้งค่าตัวแปรสภาพแวดล้อมใหม่สำหรับ `azd` ที่นี่ เราอัปเดตชื่อโมเดล agent _หมายเหตุ: การเปลี่ยนแปลงใด ๆ จะสะท้อนในไฟล์ `.azure/<env-name>/.env` ทันที_

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      ตอนนี้ เราควรจะเห็นค่าว่าตั้งค่าแล้ว:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. โปรดทราบว่าทรัพยากรบางอย่างเป็นแบบถาวร (เช่น การปรับใช้โมเดล) และจะต้องใช้มากกว่าการรัน `azd up` เพียงอย่างเดียวเพื่อบังคับให้ปรับใช้ใหม่ ลองยกเลิกการปรับใช้เดิมและปรับใช้ใหม่โดยใช้ตัวแปรสภาพแวดล้อมที่เปลี่ยนแปลงแล้ว

1. **รีเฟรช** หากคุณเคยปรับใช้โครงสร้างพื้นฐานโดยใช้เทมเพลต azd มาก่อน - คุณสามารถ _รีเฟรช_ สถานะของตัวแปรสภาพแวดล้อมท้องถิ่นของคุณตามสถานะปัจจุบันของการปรับใช้ Azure ของคุณโดยใช้คำสั่งนี้:

      ```bash title="" linenums="0"
      azd env refresh
      ```

      นี่เป็นวิธีที่ทรงพลังในการ _ซิงค์_ ตัวแปรสภาพแวดล้อมข้ามสองหรือมากกว่านั้นของสภาพแวดล้อมการพัฒนาท้องถิ่น (เช่น ทีมที่มีนักพัฒนาหลายคน) - ช่วยให้โครงสร้างพื้นฐานที่ปรับใช้งานทำหน้าที่เป็นข้อเท็จจริงพื้นฐานสำหรับสถานะของตัวแปร env สมาชิกทีมเพียงแค่ _รีเฟรช_ ตัวแปรเพื่อให้กลับมาซิงค์กัน
