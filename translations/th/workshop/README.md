# เวิร์กช็อป AZD สำหรับนักพัฒนาปัญญาประดิษฐ์

ยินดีต้อนรับสู่เวิร์กช็อปปฏิบัติการสำหรับการเรียนรู้ Azure Developer CLI (AZD) โดยเน้นไปที่การปรับใช้แอป AI เวิร์กช็อปนี้ช่วยให้คุณเข้าใจการใช้งานเทมเพลต AZD ใน 3 ขั้นตอน:

1. **ค้นหา** - หาคุณสมบัติเทมเพลตที่เหมาะกับคุณ
1. **ปรับใช้** - ปรับใช้และตรวจสอบว่าใช้ได้จริง
1. **ปรับแต่ง** - แก้ไขและทำซ้ำจนเป็นของคุณ!

ในระหว่างเวิร์กช็อปนี้ คุณจะได้รู้จักเครื่องมือและเวิร์กโฟลว์ของนักพัฒนาหลักเพื่อช่วยให้การพัฒนาตั้งแต่ต้นจนจบของคุณเป็นไปอย่างราบรื่น

<br/>

## คู่มือบนเบราว์เซอร์

บทเรียนเวิร์กช็อปอยู่ในรูปแบบ Markdown คุณสามารถเข้าใช้งานได้โดยตรงใน GitHub หรือเปิดตัวอย่างการดูผ่านเบราว์เซอร์ตามภาพหน้าจอด้านล่าง

![Workshop](../../../translated_images/th/workshop.75906f133e6f8ba0.webp)

เพื่อใช้ตัวเลือกนี้ - ให้ทำการ fork รีโพซิทอรีไปยังโปรไฟล์ของคุณ และเปิดใช้งาน GitHub Codespaces เมื่อตัวเทอร์มินัลของ VS Code พร้อมใช้งาน ให้พิมพ์คำสั่งนี้:

```bash title="" linenums="0"
mkdocs serve > /dev/null 2>&1 &
```

ในไม่กี่วินาที คุณจะเห็นกล่องโต้ตอบป๊อปอัพ เลือกตัวเลือก `Open in browser` คู่มือบนเว็บจะเปิดในแท็บเบราว์เซอร์ใหม่ ประโยชน์ของการดูตัวอย่างนี้:

1. **การค้นหาในตัว** - ค้นหาคำหรือบทเรียนได้อย่างรวดเร็ว
1. **ไอคอนคัดลอก** - วางเมาส์เหนือโค้ดบล็อกเพื่อดูตัวเลือกนี้
1. **สลับธีม** - เปลี่ยนระหว่างธีมมืดและธีมสว่าง
1. **ขอความช่วยเหลือ** - คลิกไอคอน Discord ที่ส่วนท้ายหน้าเพื่อติดต่อ!

## ภาพรวมเวิร์กช็อป

**ระยะเวลา:** 3-4 ชั่วโมง  
**ระดับ:** ผู้เริ่มต้นถึงระดับกลาง

นี่คือเวิร์กช็อปที่เน้นการเรียนรู้ด้วยการลงมือทำ หลังจากทำแบบฝึกหัดครบถ้วน เราแนะนำให้ทบทวนหลักสูตร AZD For Beginners เพื่อเรียนรู้เพิ่มเติมเกี่ยวกับความปลอดภัยและแนวทางปฏิบัติที่ดีที่สุดด้านประสิทธิภาพ

## สิ่งที่คุณจะได้เรียนรู้

ให้คิดว่าเทมเพลต AZD เป็น sandbox การเรียนรู้เพื่อสำรวจความสามารถและเครื่องมือต่าง ๆ สำหรับการพัฒนาตั้งแต่ต้นจนจบบน Microsoft Foundry เมื่อจบเวิร์กช็อปนี้ คุณจะมีความเข้าใจโดยสัญชาตญาณเกี่ยวกับเครื่องมือและแนวคิดต่าง ๆ ในบริบทนี้

| แนวคิด  | วัตถุประสงค์ |
|:---|:---|
| **Azure Developer CLI** | เข้าใจคำสั่งเครื่องมือและเวิร์กโฟลว์|
| **เทมเพลต AZD**| เข้าใจโครงสร้างโปรเจกต์และการตั้งค่า|
| **Azure AI Agent**| provisioning & ปรับใช้โปรเจกต์ Microsoft Foundry  |
| **Azure AI Search**| เปิดใช้งานการออกแบบบริบทด้วยแอเจนต์ |
| **Observability**| สำรวจการติดตาม, การตรวจสอบ และการประเมินผล |
| **Red Teaming**| สำรวจการทดสอบเชิงฝ่ายตรงข้ามและการบรรเทาผลกระทบ |

## โครงสร้างเวิร์กช็อป

เวิร์กช็อปนี้จะพาคุณเดินทางจากการค้นหาเทมเพลต ไปจนถึงการปรับใช้ การแยกวิเคราะห์ และการปรับแต่ง - โดยใช้เทมเพลตเริ่มต้นทางการอย่างเป็นทางการ [Getting Started with AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) เป็นฐาน

### [โมดูล 1: เลือกเทมเพลต AI](docs/instructions/1-Select-AI-Template.md) (30 นาที)

- เทมเพลต AI คืออะไร?
- ฉันจะหามันได้จากที่ไหน?
- ฉันจะเริ่มต้นสร้าง AI Agents ได้อย่างไร?
- **ปฏิบัติ:** เริ่มใช้งานอย่างรวดเร็วกับ GitHub Codespaces

### [โมดูล 2: ตรวจสอบเทมเพลต AI](docs/instructions/2-Validate-AI-Template.md) (30 นาที)

- สถาปัตยกรรมเทมเพลต AI คืออะไร?
- เวิร์กโฟลว์การพัฒนา AZD คืออะไร?
- ฉันจะขอความช่วยเหลือกับการพัฒนา AZD ได้อย่างไร?
- **ปฏิบัติ:** ปรับใช้และตรวจสอบเทมเพลต AI Agents

### [โมดูล 3: แยกส่วนเทมเพลต AI](docs/instructions/3-Deconstruct-AI-Template.md) (30 นาที)

- สำรวจสภาพแวดล้อมของคุณใน `.azure/`
- สำรวจการตั้งค่าทรัพยากรใน `infra/`
- สำรวจการตั้งค่า AZD ใน `azure.yaml`s
- **ปฏิบัติ:** แก้ไขตัวแปรสภาพแวดล้อมและปรับใช้ใหม่

### [โมดูล 4: กำหนดค่าเทมเพลต AI](docs/instructions/4-Configure-AI-Template.md) (30 นาที)
- สำรวจ: Retrieval Augmented Generation
- สำรวจ: การประเมินตัวแทน & Red Teaming
- สำรวจ: การติดตาม & การตรวจสอบ
- **ปฏิบัติ:** สำรวจ AI Agent + Observability 

### [โมดูล 5: ปรับแต่งเทมเพลต AI](docs/instructions/5-Customize-AI-Template.md) (30 นาที)
- กำหนด: PRD พร้อมความต้องการตัวอย่าง
- กำหนดค่า: ตัวแปรสภาพแวดล้อมสำหรับ AZD
- นำไปใช้: Lifecycle Hooks สำหรับงานเพิ่มเติม
- **ปฏิบัติ:** ปรับแต่งเทมเพลตให้เหมาะกับสถานการณ์ของฉัน

### [โมดูล 6: รื้อถอนโครงสร้างพื้นฐาน](docs/instructions/6-Teardown-Infrastructure.md) (30 นาที)
- ทบทวน: เทมเพลต AZD คืออะไร?
- ทบทวน: ทำไมต้องใช้ Azure Developer CLI?
- ขั้นตอนถัดไป: ลองใช้เทมเพลตอื่น!
- **ปฏิบัติ:** รื้อถอนโครงสร้างพื้นฐานและทำความสะอาด

## ความท้าทายในเวิร์กช็อป

อยากท้าทายตัวเองให้มากขึ้นหรือ? นี่คือข้อเสนอโปรเจกต์บางส่วน หรือแบ่งปันไอเดียกับเรา!!

| โปรเจกต์ | คำอธิบาย |
|:---|:---|
|1. **แยกส่วนเทมเพลต AI ที่ซับซ้อน** | ใช้เวิร์กโฟลว์และเครื่องมือที่เราให้ไว้ดูว่าคุณจะสามารถปรับใช้ ตรวจสอบ และปรับแต่งเทมเพลตโซลูชัน AI ที่แตกต่างได้ไหม _คุณได้เรียนรู้อะไร?_|
|2. **ปรับแต่งด้วยสถานการณ์ของคุณ**  | ลองเขียน PRD (เอกสารข้อกำหนดผลิตภัณฑ์) สำหรับสถานการณ์ที่แตกต่าง จากนั้นใช้ GitHub Copilot ในรีโปเทมเพลตของคุณในโหมด Agent Model และขอให้มันสร้างเวิร์กโฟลว์การปรับแต่งให้ _คุณได้เรียนรู้อะไร? คุณจะปรับปรุงคำแนะนำเหล่านี้ได้อย่างไร?_|
