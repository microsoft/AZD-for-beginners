# เวิร์กช็อป AZD สำหรับนักพัฒนา AI

ยินดีต้อนรับสู่เวิร์กช็อปแบบลงมือทำสำหรับการเรียนรู้ Azure Developer CLI (AZD) โดยเน้นการปรับใช้แอปพลิเคชัน AI เวิร์กช็อปนี้จะช่วยให้คุณเข้าใจการใช้งานเทมเพลต AZD ใน 3 ขั้นตอน:

1. **ค้นหา** - ค้นหาเทมเพลตที่เหมาะสมกับคุณ
1. **ปรับใช้** - ปรับใช้และตรวจสอบว่าใช้งานได้จริง
1. **ปรับแต่ง** - แก้ไขและปรับปรุงให้เป็นของคุณเอง!

ตลอดระยะเวลาเวิร์กช็อปนี้ คุณจะได้รู้จักกับเครื่องมือและเวิร์กโฟลว์หลักของนักพัฒนา เพื่อช่วยให้คุณทำงานพัฒนาแบบครบวงจรได้อย่างราบรื่น

<br/>

## คู่มือบนเบราว์เซอร์

บทเรียนในเวิร์กช็อปเป็นแบบ Markdown คุณสามารถเข้าถึงบทเรียนเหล่านี้ได้โดยตรงบน GitHub หรือเปิดดูตัวอย่างบนเบราว์เซอร์ได้ตามภาพตัวอย่างด้านล่าง

![Workshop](../../../translated_images/th/workshop.75906f133e6f8ba0.webp)

หากต้องการใช้ตัวเลือกนี้ ให้ทำการฟอร์กโค้ดรีโพซิทอรีไปยังโปรไฟล์ของคุณ แล้วเปิดใช้งาน GitHub Codespaces เมื่อเปิดเทอร์มินัล VS Code แล้ว ให้พิมพ์คำสั่งนี้:

การดูตัวอย่างบนเบราว์เซอร์นี้ใช้งานได้ทั้งใน GitHub Codespaces, dev containers หรือโคลนในเครื่องที่ติดตั้ง Python และ MkDocs แล้ว

```bash title="" linenums="0"
mkdocs serve > /dev/null 2>&1 &
```


ในไม่กี่วินาที คุณจะเห็นหน้าต่างป๊อปอัพ ให้เลือกตัวเลือก `Open in browser` คู่มือแบบเว็บก็จะเปิดในแท็บเบราว์เซอร์ใหม่ คุณสมบัติบางอย่างของการดูตัวอย่างนี้คือ:

1. **ค้นหาในตัว** - ค้นหาคำหรือบทเรียนได้รวดเร็ว
1. **ไอคอนคัดลอก** - นำเมาส์ไปวางบนโค้ดบล็อกเพื่อเห็นตัวเลือกนี้
1. **สลับธีม** - สลับระหว่างธีมมืดและสว่าง
1. **รับความช่วยเหลือ** - คลิกไอคอน Discord ที่ส่วนท้ายเพื่อเข้าร่วม!

## สิ่งที่คุณจะได้เรียนรู้

ให้มองเทมเพลต AZD เป็นแซนด์บ็อกซ์สำหรับเรียนรู้เพื่อสำรวจความสามารถและเครื่องมือต่างๆ สำหรับการพัฒนาแบบครบวงจรบน Microsoft Foundry เมื่อจบเวิร์กช็อปนี้ คุณจะมีความเข้าใจสัญชาตญาณเกี่ยวกับเครื่องมือและแนวคิดต่างๆ ในบริบทนี้

| แนวคิด  | วัตถุประสงค์ |
|:---|:---|
| **Azure Developer CLI** | เข้าใจคำสั่งเครื่องมือและเวิร์กโฟลว์ |
| **AZD Templates**| เข้าใจโครงสร้างโปรเจ็กต์และการตั้งค่า |
| **Azure AI Agent**| จัดเตรียมและปรับใช้โปรเจ็กต์ Microsoft Foundry |
| **Azure AI Search**| เปิดใช้งานวิศวกรรมบริบทด้วยเอเจนต์ |
| **Observability**| สำรวจการติดตาม, การตรวจสอบ และการประเมินผล |
| **Red Teaming**| สำรวจการทดสอบเชิงปฏิบัติการและการป้องกัน |

## โครงสร้างเวิร์กช็อป

เวิร์กช็อปนี้ถูกออกแบบให้คุณเดินทางจากการค้นหาเทมเพลตไปสู่การปรับใช้ การแยกส่วน และการปรับแต่ง โดยใช้เทมเพลตเริ่มต้นอย่างเป็นทางการ [Getting Started with AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) เป็นฐาน

### [โมดูล 1: เลือกเทมเพลต AI](docs/instructions/1-Select-AI-Template.md) (30 นาที)

- เทมเพลต AI คืออะไร?
- จะหาเทมเพลต AI ได้จากที่ไหน?
- เริ่มต้นสร้าง AI Agents อย่างไร?
- **แลป**: เริ่มต้นใช้งานอย่างรวดเร็วใน Codespaces หรือ dev container

### [โมดูล 2: ตรวจสอบเทมเพลต AI](docs/instructions/2-Validate-AI-Template.md) (30 นาที)

- สถาปัตยกรรมเทมเพลต AI คืออะไร?
- เวิร์กโฟลว์พัฒนา AZD คืออะไร?
- วิธีขอความช่วยเหลือในการพัฒนา AZD
- **แลป**: ปรับใช้และตรวจสอบเทมเพลต AI Agents

### [โมดูล 3: แยกส่วนเทมเพลต AI](docs/instructions/3-Deconstruct-AI-Template.md) (30 นาที)

- สำรวจสภาพแวดล้อมที่ `.azure/`
- สำรวจการตั้งค่าใน `infra/`
- สำรวจการตั้งค่า AZD ในไฟล์ `azure.yaml`
- **แลป**: แก้ไข Environment Variables และปรับใช้ใหม่

### [โมดูล 4: กำหนดค่าเทมเพลต AI](docs/instructions/4-Configure-AI-Template.md) (30 นาที)
- สำรวจ: การสร้างแบบเสริม Retrieval Augmented Generation
- สำรวจ: การประเมิน Agent & Red Teaming
- สำรวจ: การติดตามและตรวจสอบ
- **แลป**: สำรวจ AI Agent + Observability

### [โมดูล 5: ปรับแต่งเทมเพลต AI](docs/instructions/5-Customize-AI-Template.md) (30 นาที)
- กำหนด: PRD กับข้อกำหนดของสถานการณ์
- กำหนดค่า: Environment Variables สำหรับ AZD
- นำไปใช้: Lifecycle Hooks สำหรับงานเพิ่มเติม
- **แลป**: ปรับแต่งเทมเพลตให้เหมาะกับสถานการณ์ของฉัน

### [โมดูล 6: จัดเก็บโครงสร้างพื้นฐาน](docs/instructions/6-Teardown-Infrastructure.md) (30 นาที)
- ทบทวน: เทมเพลต AZD คืออะไร?
- ทบทวน: ทำไมต้องใช้ Azure Developer CLI?
- ก้าวต่อไป: ลองใช้เทมเพลตอื่น!
- **แลป**: ยกเลิกการจัดสรรโครงสร้างพื้นฐานและทำความสะอาด

| โครงการ | คำอธิบาย |
|:---|:---|
