# AZD สำหรับผู้เริ่มต้น: โครงร่างหลักสูตรและกรอบการเรียนรู้

## ภาพรวมหลักสูตร

เรียนรู้ Azure Developer CLI (azd) ผ่านบทเรียนที่จัดเป็นระเบียบเพื่อการเรียนรู้แบบก้าวหน้า โดยมุ่งเน้นเป็นพิเศษที่การปรับใช้แอปพลิเคชัน AI ด้วยการรวม Microsoft Foundry

### ทำไมหลักสูตรนี้สำคัญสำหรับนักพัฒนายุคใหม่

อ้างอิงจากข้อมูลเชิงลึกจากชุมชน Microsoft Foundry บน Discord, 45% ของนักพัฒนาต้องการใช้ AZD สำหรับงาน AI แต่ประสบปัญหาเกี่ยวกับ:
- สถาปัตยกรรม AI ที่ประกอบด้วยหลายบริการซับซ้อน
- แนวปฏิบัติที่ดีที่สุดสำหรับการปรับใช้ AI ในสภาพแวดล้อมการผลิต  
- การรวมและการกำหนดค่าบริการ Azure AI
- การเพิ่มประสิทธิภาพค่าใช้จ่ายสำหรับงาน AI
- การแก้ไขปัญหาเฉพาะด้านการปรับใช้ AI

### วัตถุประสงค์การเรียนรู้หลัก

เมื่อสำเร็จหลักสูตรที่มีโครงสร้างนี้ คุณจะ:
- เชี่ยวชาญพื้นฐาน AZD: แนวคิดหลัก การติดตั้ง และการกำหนดค่า
- ปรับใช้แอปพลิเคชัน AI: ใช้ AZD ร่วมกับบริการ Microsoft Foundry
- นำ Infrastructure as Code มาใช้: จัดการทรัพยากร Azure ด้วยเทมเพลต Bicep
- แก้ไขปัญหาการปรับใช้: แก้ไขปัญหาทั่วไปและดีบักปัญหา
- ปรับให้เหมาะกับการผลิต: ความปลอดภัย การปรับขนาด การตรวจสอบ และการจัดการค่าใช้จ่าย
- สร้างโซลูชัน Multi-Agent: ปรับใช้สถาปัตยกรรม AI ที่ซับซ้อน

## 🎓 ประสบการณ์การเรียนรู้เชิงปฏิบัติการ

### ตัวเลือกการส่งมอบการเรียนรู้ที่ยืดหยุ่น
หลักสูตรนี้ออกแบบเพื่อสนับสนุนทั้งการเรียนรู้ด้วยตนเองแบบยืดหยุ่นและการฝึกอบรมที่มีผู้นำการสอน ช่วยให้ผู้เรียนได้ฝึกปฏิบัติ AZD และพัฒนาทักษะเชิงปฏิบัติผ่านแบบฝึกหัดเชิงโต้ตอบ

#### 🚀 โหมดการเรียนรู้ด้วยตนเอง
เหมาะสำหรับนักพัฒนารายบุคคลและการเรียนรู้อย่างต่อเนื่อง

คุณสมบัติ:
- ส่วนต่อประสานในเบราว์เซอร์: เวิร์กช็อปที่ขับเคลื่อนด้วย MkDocs เข้าถึงได้ผ่านเว็บเบราว์เซอร์ใดก็ได้
- การรวมกับ GitHub Codespaces: สภาพแวดล้อมการพัฒนาด้วยคลิกเดียวพร้อมเครื่องมือที่ตั้งค่าล่วงหน้า
- สภาพแวดล้อม DevContainer แบบโต้ตอบ: ไม่จำเป็นต้องตั้งค่าท้องถิ่น - เริ่มเขียนโค้ดได้ทันที
- การติดตามความคืบหน้า: จุดตรวจและแบบฝึกหัดการตรวจสอบในตัว
- การสนับสนุนจากชุมชน: เข้าถึงช่องทาง Discord ของ Azure สำหรับคำถามและความร่วมมือ

โครงสร้างการเรียนรู้:
- เวลาที่ยืดหยุ่น: ทำบทต่างๆ ให้เสร็จตามจังหวะของคุณเองภายในหลายวันหรือหลายสัปดาห์
- ระบบจุดตรวจ: ยืนยันการเรียนรู้ก่อนก้าวสู่หัวข้อที่ซับซ้อน
- ห้องสมุดทรัพยากร: เอกสาร ตัวอย่าง และคำแนะนำการแก้ปัญหาอย่างครอบคลุม
- การพัฒนาแฟ้มผลงาน: สร้างโครงการที่สามารถปรับใช้ได้สำหรับแฟ้มผลงานมืออาชีพ

เริ่มต้น (เรียนด้วยตนเอง):
```bash
# ตัวเลือกที่ 1: GitHub Codespaces (แนะนำ)
# ไปที่ repository แล้วคลิก "Code" → "Create codespace on main"

# ตัวเลือกที่ 2: การพัฒนาในเครื่อง
git clone https://github.com/microsoft/azd-for-beginners.git
cd azd-for-beginners/workshop
# ทำตามคำแนะนำการตั้งค่าใน workshop/README.md
```

#### 🏛️ การอบรมเชิงปฏิบัติการแบบมีผู้สอน
เหมาะสำหรับการฝึกอบรมองค์กร บูทแคมป์ และสถาบันการศึกษา

รูปแบบการฝึกอบรม:

📚 การบูรณาการหลักสูตรทางวิชาการ (8-12 สัปดาห์)
- โปรแกรมมหาวิทยาลัย: หลักสูตรตลอดเทอม พร้อมชั้นเรียนสัปดาห์ละ 2 ชั่วโมง
- รูปแบบบูทแคมป์: โปรแกรมเข้มข้น 3-5 วัน พร้อมชั้นเรียนวันละ 6-8 ชั่วโมง
- การฝึกอบรมองค์กร: เซสชันทีมรายเดือนพร้อมการนำโครงการจริงไปปฏิบัติ
- กรอบการประเมิน: การมอบหมายงานที่ให้คะแนน การประเมินโดยเพื่อน และโครงการสุดท้าย

🚀 เวิร์กช็อปเข้มข้น (1-3 วัน)
- Day 1: พื้นฐาน + การพัฒนา AI (Chapters 1-2) - 6 ชั่วโมง
- Day 2: การกำหนดค่า + โครงสร้างพื้นฐาน (Chapters 3-4) - 6 ชั่วโมง  
- Day 3: รูปแบบขั้นสูง + การผลิต (Chapters 5-8) - 8 ชั่วโมง
- การติดตามผล: ให้คำปรึกษาแบบไม่บังคับเป็นเวลา 2 สัปดาห์สำหรับการเสร็จสิ้นโครงการ

⚡ บรรยายสรุประดับผู้บริหาร (4-6 ชั่วโมง)
- ภาพรวมเชิงกลยุทธ์: ข้อเสนอคุณค่า AZD และผลกระทบทางธุรกิจ (1 ชั่วโมง)
- การสาธิตแบบลงมือปฏิบัติ: ปรับใช้แอปพลิเคชัน AI ตั้งแต่ต้นจนจบ (2 ชั่วโมง)
- ทบทวนสถาปัตยกรรม: รูปแบบองค์กรและการกำกับดูแล (1 ชั่วโมง)
- การวางแผนการดำเนินการ: ยุทธศาสตร์การนำไปใช้ในองค์กร (1-2 ชั่วโมง)

#### 🛠️ วิธีการเรียนรู้เชิงปฏิบัติการ
แนวทาง Discovery → Deployment → Customization สำหรับการพัฒนาทักษะเชิงปฏิบัติ

เฟส 1: การค้นพบ (45 นาที)
- การสำรวจเทมเพลต: ประเมินเทมเพลตและบริการของ Microsoft Foundry
- การวิเคราะห์สถาปัตยกรรม: เข้าใจรูปแบบ multi-agent และกลยุทธ์การปรับใช้
- การประเมินข้อกำหนด: ระบุความต้องการและข้อจำกัดขององค์กร
- การตั้งค่าสภาพแวดล้อม: กำหนดค่าสภาพแวดล้อมการพัฒนาและทรัพยากร Azure

เฟส 2: การปรับใช้ (2 ชั่วโมง)
- การดำเนินการแบบมีคำแนะนำ: ขั้นตอนการปรับใช้แอปพลิเคชัน AI ด้วย AZD
- การกำหนดค่าบริการ: กำหนดค่าบริการ Azure AI, endpoints และการพิสูจน์ตัวตน
- การนำความปลอดภัยมาใช้: ปรับใช้รูปแบบความปลอดภัยระดับองค์กรมและการควบคุมการเข้าถึง
- การทดสอบการยืนยัน: ยืนยันการปรับใช้และแก้ไขปัญหาทั่วไป

เฟส 3: การปรับแต่ง (45 นาที)
- การแก้ไขแอปพลิเคชัน: ปรับเทมเพลตให้เหมาะกับกรณีการใช้งานและข้อกำหนดเฉพาะ
- การปรับให้เหมาะกับการผลิต: นำการตรวจสอบ การจัดการค่าใช้จ่าย และกลยุทธ์การปรับขนาดมาใช้
- รูปแบบขั้นสูง: สำรวจการประสานงานของ multi-agent และสถาปัตยกรรมที่ซับซ้อน
- การวางแผนขั้นต่อไป: กำหนดเส้นทางการเรียนรู้สำหรับการพัฒนาทักษะต่อเนื่อง

#### 🎯 ผลลัพธ์การเรียนรู้จากเวิร์กช็อป
ทักษะที่วัดผลได้จากการฝึกปฏิบัติ

ความสามารถทางเทคนิค:
- ปรับใช้แอปพลิเคชัน AI ในสภาพแวดล้อมการผลิต: ปรับใช้และกำหนดค่าโซลูชันที่ขับเคลื่อนด้วย AI สำเร็จ
- เชี่ยวชาญ Infrastructure as Code: สร้างและจัดการเทมเพลต Bicep แบบกำหนดเอง
- สถาปัตยกรรม Multi-Agent: นำโซลูชันตัวแทน AI ที่ประสานงานกันมาใช้
- ความพร้อมสำหรับการผลิต: นำรูปแบบความปลอดภัย การตรวจสอบ และการกำกับดูแลมาใช้
- ความเชี่ยวชาญด้านการแก้ปัญหา: แก้ไขปัญหาการปรับใช้และการกำหนดค่าได้อย่างอิสระ

ทักษะเชิงวิชาชีพ:
- ภาวะผู้นำโครงการ: นำทีมทางเทคนิคในโครงการปรับใช้บนคลาวด์
- การออกแบบสถาปัตยกรรม: ออกแบบโซลูชัน Azure ที่สามารถปรับขนาดได้และคุ้มค่า
- การถ่ายทอดความรู้: ฝึกอบรมและให้คำปรึกษาเพื่อนร่วมงานในแนวปฏิบัติที่ดีที่สุดของ AZD
- การวางแผนเชิงกลยุทธ์: มีอิทธิพลต่อยุทธศาสตร์การนำคลาวด์ไปใช้ในองค์กร

#### 🏢 การนำเวิร์กช็อปไปใช้งานในองค์กร
กลยุทธ์การปรับใช้และการฝึกอบรมในระดับองค์กร

โปรแกรมการฝึกอบรมขององค์กร:
- การปฐมนิเทศนักพัฒนา: การแนะนำพนักงานใหม่ด้วยพื้นฐาน AZD (2-4 สัปดาห์)
- การยกระดับทักษะทีม: เวิร์กช็อปรายไตรมาสสำหรับทีมพัฒนาที่มีอยู่ (1-2 วัน)
- การทบทวนสถาปัตยกรรม: เซสชันรายเดือนสำหรับวิศวกรอาวุโสและสถาปนิก (4 ชั่วโมง)
- การบรรยายสำหรับผู้นำ: เวิร์กช็อปสำหรับผู้ตัดสินใจทางเทคนิค (ครึ่งวัน)

การสนับสนุนการดำเนินการ:
- ออกแบบเวิร์กช็อปที่กำหนดเอง: เนื้อหาที่ปรับให้ตรงกับความต้องการขององค์กร
- การจัดการโครงการนำร่อง: การเปิดตัวเชิงโครงสร้างพร้อมตัวชี้วัดความสำเร็จและช่องทางรับข้อเสนอแนะ  
- การให้คำปรึกษาต่อเนื่อง: การสนับสนุนหลังเวิร์กช็อปสำหรับการนำโครงการไปปฏิบัติ
- การสร้างชุมชน: ชุมชนนักพัฒนา Azure AI ภายในองค์กรและการแบ่งปันความรู้

ตัวชี้วัดความสำเร็จ:
- การได้มาซึ่งทักษะ: การประเมินก่อน/หลังที่วัดการเติบโตด้านความสามารถทางเทคนิค
- ความสำเร็จในการปรับใช้: เปอร์เซ็นต์ของผู้เข้าร่วมที่ปรับใช้แอปพลิเคชันในสภาพแวดล้อมการผลิตได้สำเร็จ
- เวลาสู่การผลิต: ลดเวลาในการปฐมนิเทศสำหรับโครงการ Azure AI ใหม่
- การรักษาความรู้: การประเมินติดตามผล 3-6 เดือนหลังการอบรม

## 🎓 ผลลัพธ์การเรียนรู้อย่างครบถ้วน

### ระดับพื้นฐาน (บทที่ 1-2)
เมื่อสำเร็จบทพื้นฐาน ผู้เรียนจะสามารถแสดงให้เห็น:

ความสามารถทางเทคนิค:
- ปรับใช้เว็บแอปง่าย ๆ ไปยัง Azure โดยใช้คำสั่ง AZD
- กำหนดค่าและปรับใช้แอปแชทที่ขับเคลื่อนด้วย AI พร้อมความสามารถ RAG
- เข้าใจแนวคิดหลักของ AZD: เทมเพลต, สภาพแวดล้อม, เวิร์กโฟลว์การจัดหา
- รวมบริการ Microsoft Foundry กับการปรับใช้ AZD
- นำทางการกำหนดค่าบริการ Azure AI และจุดเชื่อมต่อ API

ทักษะเชิงอาชีพ:
- ปฏิบัติตามเวิร์กโฟลว์การปรับใช้ที่มีโครงสร้างเพื่อผลลัพธ์ที่สม่ำเสมอ
- แก้ไขปัญหาการปรับใช้พื้นฐานโดยใช้บันทึกและเอกสาร
- สื่อสารอย่างมีประสิทธิภาพเกี่ยวกับกระบวนการปรับใช้บนคลาวด์
- นำแนวทางปฏิบัติที่ดีที่สุดสำหรับการรวมบริการ AI อย่างปลอดภัยไปใช้

การยืนยันการเรียนรู้:
- ✅ Successfully deploy `todo-nodejs-mongo` template
- ✅ Deploy and configure `azure-search-openai-demo` with RAG
- ✅ Complete interactive workshop exercises (Discovery phase)
- ✅ Participate in Azure Discord community discussions

### ระดับกลาง (บทที่ 3-4)
เมื่อสำเร็จบทระดับกลาง ผู้เรียนจะสามารถแสดงให้เห็น:

ความสามารถทางเทคนิค:
- จัดการการปรับใช้หลายสภาพแวดล้อม (dev, staging, production)
- สร้างเทมเพลต Bicep ที่กำหนดเองสำหรับโครงสร้างพื้นฐานเป็นโค้ด
- นำรูปแบบการพิสูจน์ตัวตนด้วย managed identity ไปใช้
- ปรับใช้แอปหลายบริการที่ซับซ้อนด้วยการกำหนดค่าที่กำหนดเอง
- ปรับแต่งกลยุทธ์การจัดหาแหล่งทรัพยากรเพื่อประสิทธิภาพและต้นทุน

ทักษะเชิงอาชีพ:
- ออกแบบสถาปัตยกรรมโครงสร้างพื้นฐานที่ปรับขนาดได้
- นำแนวปฏิบัติด้านความปลอดภัยที่ดีที่สุดสำหรับการปรับใช้บนคลาวด์ไปใช้
- จัดทำเอกสารรูปแบบโครงสร้างพื้นฐานสำหรับการทำงานร่วมกันของทีม
- ประเมินและเลือกบริการ Azure ที่เหมาะสมกับความต้องการ

การยืนยันการเรียนรู้:
- ✅ Configure separate environments with environment-specific settings
- ✅ Create and deploy custom Bicep template for multi-service application
- ✅ Implement managed identity authentication for secure access
- ✅ Complete configuration management exercises with real scenarios

- นำสถาปัตยกรรมตัวแทน Customer และ Inventory ไปใช้สำหรับสถานการณ์ค้าปลีก
- ดำเนินการวางแผนความจุและการตรวจสอบทรัพยากรอย่างครบถ้วน
- ดำเนินการตรวจสอบก่อนปรับใช้และการปรับแต่งแบบอัตโนมัติ
- ออกแบบการเลือก SKU ที่คุ้มค่าสำหรับความต้องการงาน

ทักษะเชิงอาชีพ:
- ออกแบบสถาปัตยกรรม AI ที่ซับซ้อนสำหรับสภาพแวดล้อมการผลิต
- นำการอภิปรายเชิงเทคนิคเกี่ยวกับกลยุทธ์การปรับใช้ AI
- ให้คำปรึกษาแก่ผู้พัฒนาระดับจูเนียร์ในแนวปฏิบัติ AZD และการปรับใช้ AI
- ประเมินและแนะนำรูปแบบสถาปัตยกรรม AI สำหรับความต้องการทางธุรกิจ

การยืนยันการเรียนรู้:

- ✅ Demonstrate agent coordination and workflow orchestration
- ✅ Complete capacity planning exercises with real resource constraints
- ✅ Validate deployment readiness through automated pre-flight checks
