# บทที่ 5: โซลูชัน AI หลายเอเจนต์

**📚 หลักสูตร**: [AZD สำหรับผู้เริ่มต้น](../../README.md) | **⏱️ ระยะเวลา**: 2-3 ชั่วโมง | **⭐ ความซับซ้อน**: ขั้นสูง

---

## ภาพรวม

บทนี้ครอบคลุมรูปแบบสถาปัตยกรรมหลายเอเจนต์ขั้นสูง การประสานงานของเอเจนต์ และการปรับใช้ AI ที่พร้อมใช้งานจริงสำหรับสถานการณ์ที่ซับซ้อน

## วัตถุประสงค์การเรียนรู้

เมื่อเรียนบทนี้จบ คุณจะสามารถ:
- เข้าใจรูปแบบสถาปัตยกรรมหลายเอเจนต์
- ปรับใช้ระบบเอเจนต์ AI ที่ประสานงานกัน
- นำการสื่อสารระหว่างเอเจนต์มาใช้
- สร้างโซลูชันหลายเอเจนต์ที่พร้อมใช้งานในภาคการผลิต

## 🚀 เริ่มต้นอย่างรวดเร็ว

```bash
# ปรับใช้โซลูชันหลายตัวแทนสำหรับค้าปลีก
cd examples/retail-multiagent-arm-template
./deploy.sh

# หรือใช้แม่แบบโดยตรง
azd init --template agent-openai-python-prompty
azd up
```

## 🎯 โซลูชันเด่น: โซลูชันค้าปลีกหลายเอเจนต์

The [โซลูชันค้าปลีกหลายเอเจนต์](../../examples/retail-scenario.md) แสดงให้เห็น:

- **เอเจนต์ลูกค้า**: จัดการการโต้ตอบกับผู้ใช้และความชอบ
- **เอเจนต์คลังสินค้า**: จัดการสต็อกและการประมวลผลคำสั่งซื้อ
- **ตัวประสานงาน**: ประสานงานระหว่างเอเจนต์
- **หน่วยความจำที่ใช้ร่วมกัน**: การจัดการบริบทข้ามเอเจนต์

### บริการที่ใช้

| บริการ | วัตถุประสงค์ |
|---------|---------|
| Azure OpenAI | การเข้าใจภาษา |
| Azure AI Search | แค็ตตาล็อกสินค้า |
| Cosmos DB | สถานะและหน่วยความจำของเอเจนต์ |
| Container Apps | โฮสต์เอเจนต์ |
| Application Insights | การตรวจสอบ |
