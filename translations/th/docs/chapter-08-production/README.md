# บทที่ 8: รูปแบบการผลิตและองค์กร

**📚 หลักสูตร**: [AZD สำหรับผู้เริ่มต้น](../../README.md) | **⏱️ ระยะเวลา**: 2-3 ชั่วโมง | **⭐ ความซับซ้อน**: ขั้นสูง

---

## ภาพรวม

บทนี้ครอบคลุมรูปแบบการปรับใช้ที่พร้อมใช้งานในองค์กร การเสริมความปลอดภัย การตรวจสอบ และการปรับให้เหมาะสมด้านค่าใช้จ่ายสำหรับงาน AI ในการผลิต

## วัตถุประสงค์การเรียนรู้

By completing this chapter, you will:
- ปรับใช้แอปพลิเคชันที่ทนทานแบบหลายภูมิภาค
- นำรูปแบบความปลอดภัยระดับองค์กรไปใช้
- กำหนดค่าการตรวจสอบอย่างครอบคลุม
- ปรับค่าใช้จ่ายให้เหมาะสมในระดับใหญ่
- ตั้งค่า pipeline CI/CD ด้วย AZD

## 🚀 รายการตรวจสอบสำหรับการผลิต

- [ ] ปรับใช้หลายภูมิภาคเพื่อความทนทาน
- [ ] ใช้ Managed Identity สำหรับการยืนยันตัวตน (ไม่ใช้คีย์)
- [ ] ใช้ Application Insights สำหรับการตรวจสอบ
- [ ] กำหนดงบประมาณค่าใช้จ่ายและการแจ้งเตือน
- [ ] เปิดใช้งานการสแกนความปลอดภัย
- [ ] การเชื่อมต่อกับ CI/CD pipeline
- [ ] แผนการกู้คืนจากภัยพิบัติ

## 🔐 แนวปฏิบัติด้านความปลอดภัยที่ดีที่สุด

```bicep
// Use managed identity
identity: {
  type: 'SystemAssigned'
}

// Private endpoints for AI services
properties: {
  publicNetworkAccess: 'Disabled'
  networkAcls: {
    defaultAction: 'Deny'
  }
}
```

---

## 💰 การปรับให้เหมาะสมด้านค่าใช้จ่าย

| กลยุทธ์ | การประหยัด |
|----------|---------|
| ปรับขนาดเป็นศูนย์ (Container Apps) | 60-80% |
| ใช้ระดับการใช้งานสำหรับการพัฒนา | 50-70% |
| การปรับขนาดตามตารางเวลา | 30-50% |
| ความจุที่จองไว้ | 20-40% |

```bash
# ตั้งค่าแจ้งเตือนงบประมาณ
az consumption budget create \
  --budget-name "AI-Budget" \
  --amount 500 \
  --category Cost \
  --time-grain Monthly
```

## 📊 การตั้งค่าการตรวจสอบ

```bash
# สตรีมบันทึก
azd monitor --logs

# ตรวจสอบ Application Insights
azd monitor

# ดูตัวชี้วัด
az monitor metrics list --resource <resource-id>
```
