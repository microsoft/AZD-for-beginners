**📚 คอร์ส**: [AZD สำหรับผู้เริ่มต้น](../../README.md) | **⏱️ ระยะเวลา**: 1 ชั่วโมง | **⭐ ความซับซ้อน**: ขั้นกลาง

## ภาพรวม

บทนี้ครอบคลุมขั้นตอนสำคัญในการวางแผนและตรวจสอบก่อนการปรับใช้แอปพลิเคชันของคุณ เรียนรู้วิธีหลีกเลี่ยงความผิดพลาดที่มีค่าใช้จ่ายสูงด้วยการวางแผนความจุ การเลือก SKU ที่เหมาะสม และการตรวจสอบเบื้องต้น

## วัตถุประสงค์การเรียนรู้

เมื่อจบบทนี้ คุณจะสามารถ:
- รันการตรวจสอบเบื้องต้นก่อนการปรับใช้
- วางแผนความจุและประเมินความต้องการทรัพยากร
- เลือก SKU ที่เหมาะสมเพื่อการประหยัดค่าใช้จ่าย
- กำหนดค่า Application Insights สำหรับการตรวจสอบ
- เข้าใจรูปแบบการประสานงานของทีม

## 🚀 เริ่มต้นอย่างรวดเร็ว

```bash
# ตรวจสอบโควตาการสมัครสมาชิก
az vm list-usage --location eastus --output table

# ดูตัวอย่างการปรับใช้ (ไม่มีการสร้างทรัพยากร)
azd provision --preview

# ตรวจสอบไวยากรณ์ Bicep
az bicep build --file infra/main.bicep

# ตรวจสอบการกำหนดค่าสภาพแวดล้อม
azd env get-values
```

## ☑️ รายการตรวจสอบก่อนการปรับใช้

### ก่อน `azd provision`

- [ ] ตรวจสอบโควต้าในภูมิภาค
- [ ] เลือก SKU อย่างเหมาะสม
- [ ] ตรวจสอบการประเมินค่าใช้จ่าย
- [ ] ชุดการตั้งชื่อเป็นไปในแนวทางเดียวกัน
- [ ] กำหนดค่าความปลอดภัย/RBAC

### ก่อน `azd deploy`

- [ ] ตั้งค่าตัวแปรสภาพแวดล้อม
- [ ] ความลับใน Key Vault
- [ ] ตรวจสอบสตริงการเชื่อมต่อ
- [ ] กำหนดค่าการตรวจสอบสุขภาพ
