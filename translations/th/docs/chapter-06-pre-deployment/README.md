# บทที่ 6: การวางแผนและการตรวจสอบก่อนการปรับใช้

**📚 หลักสูตร**: [AZD สำหรับผู้เริ่มต้น](../../README.md) | **⏱️ ระยะเวลา**: 1 ชั่วโมง | **⭐ ความซับซ้อน**: ระดับกลาง

---

## ภาพรวม

บทนี้ครอบคลุมขั้นตอนการวางแผนและการตรวจสอบที่จำเป็นก่อนการปรับใช้แอปพลิเคชันของคุณ เรียนรู้วิธีหลีกเลี่ยงความผิดพลาดที่มีค่าใช้จ่ายสูงด้วยการวางแผนความจุที่เหมาะสม การเลือก SKU และการตรวจสอบก่อนการบิน

> ได้รับการตรวจสอบความถูกต้องกับ `azd 1.27.1` ในเดือนกรกฎาคม 2026

## วัตถุประสงค์การเรียนรู้

เมื่อคุณทำบทนี้สำเร็จ คุณจะสามารถ:
- ดำเนินการตรวจสอบก่อนการปรับใช้
- วางแผนความจุและประมาณการความต้องการทรัพยากร
- เลือก SKU ที่เหมาะสมเพื่อเพิ่มประสิทธิภาพด้านค่าใช้จ่าย
- กำหนดค่า Application Insights สำหรับการเฝ้าติดตาม
- เข้าใจรูปแบบการประสานงานของทีม

## 🚀 เริ่มต้นอย่างรวดเร็ว

```bash
# ตรวจสอบโควต้าการสมัครสมาชิก
az vm list-usage --location eastus --output table

# ดูตัวอย่างการปรับใช้ (ไม่สร้างทรัพยากร)
azd provision --preview

# ตรวจสอบไวยากรณ์ Bicep
az bicep build --file infra/main.bicep

# ตรวจสอบการกำหนดค่าสภาพแวดล้อม
azd env get-values
```

## ☑️ รายการตรวจสอบก่อนการปรับใช้

### ก่อน `azd provision`

- [ ] ตรวจสอบโควต้าในภูมิภาค
- [ ] เลือก SKU อย่างเหมาะสม
- [ ] ตรวจสอบประมาณค่าใช้จ่าย
- [ ] ใช้มาตรฐานการตั้งชื่อที่สอดคล้องกัน
- [ ] กำหนดค่าความปลอดภัย/RBAC

### ก่อน `azd deploy`

- [ ] ตั้งค่าตัวแปรสภาพแวดล้อม
- [ ] เก็บความลับใน Key Vault
- [ ] ตรวจสอบสายการเชื่อมต่อ
- [ ] กำหนดค่าการตรวจสอบสุขภาพ
