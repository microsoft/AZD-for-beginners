# บทที่ 6: การวางแผนและการตรวจสอบก่อนการปรับใช้

**📚 หลักสูตร**: [AZD For Beginners](../../README.md) | **⏱️ ระยะเวลา**: 1 ชั่วโมง | **⭐ ความซับซ้อน**: ระดับกลาง

---

## ภาพรวม

บทนี้ครอบคลุมขั้นตอนการวางแผนและการตรวจสอบที่จำเป็นก่อนการปรับใช้แอปของคุณ เรียนรู้วิธีหลีกเลี่ยงความผิดพลาดที่มีค่าใช้จ่ายสูงด้วยการวางแผนความจุที่เหมาะสม การเลือก SKU และการตรวจสอบก่อนการบิน

## วัตถุประสงค์การเรียนรู้

เมื่อเรียนจบบทนี้ คุณจะสามารถ:
- รันการตรวจสอบก่อนการปรับใช้ (preflight checks)
- วางแผนความจุและประมาณการทรัพยากรที่ต้องการ
- เลือก SKU ที่เหมาะสมเพื่อปรับแต่งต้นทุน
- กำหนดค่า Application Insights สำหรับการตรวจสอบ
- เข้าใจรูปแบบการประสานงานของทีม

## 🚀 เริ่มต้นอย่างรวดเร็ว

```bash
# ตรวจสอบโควต้าของการสมัครใช้งาน
az vm list-usage --location eastus --output table

# แสดงตัวอย่างการปรับใช้ (ไม่สร้างทรัพยากรใดๆ)
azd provision --preview

# ตรวจสอบไวยากรณ์ของ Bicep
az bicep build --file infra/main.bicep

# ตรวจสอบการกำหนดค่าของสภาพแวดล้อม
azd env get-values
```

## ☑️ รายการตรวจสอบก่อนการปรับใช้

### ก่อน `azd provision`

- [ ] ตรวจสอบโควต้าสำหรับภูมิภาคแล้ว
- [ ] เลือก SKUs ให้เหมาะสม
- [ ] ตรวจสอบประมาณการค่าใช้จ่ายแล้ว
- [ ] รูปแบบการตั้งชื่อสอดคล้องกัน
- [ ] ตั้งค่า Security/RBAC แล้ว

### ก่อน `azd deploy`

- [ ] ตั้งค่าตัวแปรสภาพแวดล้อมแล้ว
- [ ] เก็บความลับไว้ใน Key Vault แล้ว
- [ ] ตรวจสอบสตริงการเชื่อมต่อแล้ว
- [ ] ตั้งค่าการตรวจสอบสถานะ (health checks) แล้ว
