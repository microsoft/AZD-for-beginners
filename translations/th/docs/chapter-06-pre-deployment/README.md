# Chapter 6: การวางแผนและการตรวจสอบก่อนการนำขึ้นใช้งาน

**📚 คอร์ส**: [AZD สำหรับผู้เริ่มต้น](../../README.md) | **⏱️ ระยะเวลา**: 1 ชั่วโมง | **⭐ ความซับซ้อน**: ปานกลาง

---

## ภาพรวม

บทนี้ครอบคลุมขั้นตอนการวางแผนและการตรวจสอบที่สำคัญก่อนการนำแอปพลิเคชันของคุณขึ้นใช้งาน เรียนรู้วิธีหลีกเลี่ยงข้อผิดพลาดที่มีค่าใช้จ่ายสูงด้วยการวางแผนความจุอย่างเหมาะสม การเลือก SKU และการตรวจสอบก่อนการนำขึ้นใช้งาน

> ตรวจสอบความถูกต้องกับ `azd 1.25.6` ในเดือนมิถุนายน 2026

## วัตถุประสงค์การเรียนรู้

เมื่อคุณทำบทนี้เสร็จสมบูรณ์ คุณจะสามารถ:
- รันการตรวจสอบก่อนการนำขึ้นใช้งาน
- วางแผนความจุและประเมินความต้องการทรัพยากร
- เลือก SKU ที่เหมาะสมเพื่อเพิ่มประสิทธิภาพต้นทุน
- กำหนดค่า Application Insights เพื่อการตรวจสอบ
- เข้าใจรูปแบบการประสานงานของทีม

## 🚀 เริ่มต้นอย่างรวดเร็ว

```bash
# ตรวจสอบโควต้าการสมัครสมาชิก
az vm list-usage --location eastus --output table

# ดูตัวอย่างการปรับใช้ (ไม่มีการสร้างทรัพยากร)
azd provision --preview

# ตรวจสอบไวยากรณ์ Bicep
az bicep build --file infra/main.bicep

# ตรวจสอบการกำหนดค่าสภาพแวดล้อม
azd env get-values
```

## ☑️ รายการตรวจสอบก่อนนำขึ้นใช้งาน

### ก่อน `azd provision`

- [ ] ตรวจสอบโควต้าในภูมิภาค
- [ ] เลือก SKU อย่างเหมาะสม
- [ ] ทบทวนการประเมินต้นทุน
- [ ] ตั้งชื่อตามรูปแบบที่สอดคล้องกัน
- [ ] การตั้งค่าความปลอดภัย/RBAC

### ก่อน `azd deploy`

- [ ] ตั้งค่าตัวแปรสภาพแวดล้อม
- [ ] เก็บความลับใน Key Vault
- [ ] ตรวจสอบสายการเชื่อมต่อ
- [ ] กำหนดค่าการตรวจสอบสุขภาพ
