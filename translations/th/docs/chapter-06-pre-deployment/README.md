## ภาพรวม

บทนี้ครอบคลุมขั้นตอนสำคัญในการวางแผนและการตรวจสอบก่อนปรับใช้แอปพลิเคชันของคุณ เรียนรู้วิธีหลีกเลี่ยงข้อผิดพลาดที่มีค่าใช้จ่ายสูงด้วยการวางแผนความจุที่เหมาะสม การเลือก SKU และการเช็คก่อนบิน

> ตรวจสอบกับ `azd 1.23.12` ในเดือนมีนาคม 2026

## วัตถุประสงค์การเรียนรู้

หลังจากทำบทนี้เสร็จ คุณจะสามารถ:
- รันการเช็คก่อนบินก่อนการปรับใช้
- วางแผนความจุและประมาณการทรัพยากรที่จำเป็น
- เลือก SKU ที่เหมาะสมเพื่อเพิ่มประสิทธิภาพค่าใช้จ่าย
- กำหนดค่า Application Insights สำหรับการตรวจสอบ
- เข้าใจรูปแบบการประสานงานของทีม

## 🚀 เริ่มต้นอย่างรวดเร็ว

```bash
# ตรวจสอบโควต้าการสมัครสมาชิก
az vm list-usage --location eastus --output table

# ตัวอย่างการปรับใช้ (ไม่มีการสร้างทรัพยากร)
azd provision --preview

# ตรวจสอบไวยากรณ์ Bicep
az bicep build --file infra/main.bicep

# ตรวจสอบการกำหนดค่าสภาพแวดล้อม
azd env get-values
```

## ☑️ รายการตรวจสอบก่อนการปรับใช้

### ก่อน `azd provision`

- [ ] ตรวจสอบโควต้าในภูมิภาค
- [ ] เลือก SKU อย่างเหมาะสม
- [ ] ทบทวนการประเมินค่าใช้จ่าย
- [ ] ใช้ระบบการตั้งชื่อที่สอดคล้อง
- [ ] กำหนดค่าความปลอดภัย/RBAC

### ก่อน `azd deploy`

- [ ] ตั้งค่าตัวแปรสภาพแวดล้อม
- [ ] เก็บความลับใน Key Vault
- [ ] ตรวจสอบสายการเชื่อมต่อ
- [ ] กำหนดค่าการตรวจสุขภาพ
