# บทที่ 3: การกำหนดค่า & การตรวจสอบสิทธิ์

**📚 คอร์ส**: [AZD สำหรับผู้เริ่มต้น](../../README.md) | **⏱️ ระยะเวลา**: 45-60 นาที | **⭐ ความซับซ้อน**: ระดับกลาง

---

## ภาพรวม

บทนี้ครอบคลุมการกำหนดค่าสภาพแวดล้อม รูปแบบการตรวจสอบสิทธิ์ และแนวทางปฏิบัติที่ดีที่สุดด้านความปลอดภัยสำหรับการปรับใช้งาน Azure Developer CLI

> ตรวจสอบความถูกต้องกับ `azd 1.27.1` ในเดือนกรกฎาคม 2026

## วัตถุประสงค์การเรียนรู้

เมื่อจบบทนี้ คุณจะสามารถ:
- เป็นผู้เชี่ยวชาญในลำดับชั้นการกำหนดค่า AZD
- จัดการหลายสภาพแวดล้อม (dev, staging, prod)
- ใช้งานการตรวจสอบสิทธิ์อย่างปลอดภัยด้วย managed identities
- กำหนดค่าการตั้งค่าสำหรับแต่ละสภาพแวดล้อม

## 🚀 เริ่มต้นอย่างรวดเร็ว

```bash
# สร้างหลายสภาพแวดล้อม
azd env new dev
azd env new staging
azd env new prod

# สลับสภาพแวดล้อม
azd env select prod

# ตั้งค่าตัวแปรสภาพแวดล้อม
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# ดูการตั้งค่า
azd env get-values
```

---

## 🔧 ลำดับชั้นการกำหนดค่า

AZD ใช้การตั้งค่าในลำดับนี้ (การตั้งค่าที่หลังจะเขียนทับการตั้งค่าก่อนหน้า):

1. **ค่าเริ่มต้น** (ฝังอยู่ในเทมเพลต)
2. **azure.yaml** (การกำหนดค่าโครงการ)
3. **ตัวแปรสภาพแวดล้อม** (`azd env set`)
4. **แฟล็กคำสั่งบรรทัดคำสั่ง** (`--location eastus`)

## 🔐 แนวทางปฏิบัติที่ดีที่สุดด้านความปลอดภัย

```bash
# ใช้ตัวตนที่จัดการ (แนะนำ)
azd env set AZURE_USE_MANAGED_IDENTITY true

# ตรวจสอบสถานะการตรวจสอบสิทธิ์ AZD
azd auth status

# ตัวเลือก: ตรวจสอบบริบท Azure CLI หากคุณวางแผนจะรันคำสั่ง az
az account show

# ตรวจสอบสิทธิ์ใหม่หากจำเป็น
azd auth login

# ตัวเลือก: รีเฟรชการตรวจสอบสิทธิ์ Azure CLI สำหรับคำสั่ง az
az login
```
