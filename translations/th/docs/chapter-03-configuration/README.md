# Chapter 3: การกำหนดค่า & การรับรองความถูกต้อง

**📚 หลักสูตร**: [AZD สำหรับผู้เริ่มต้น](../../README.md) | **⏱️ ระยะเวลา**: 45-60 นาที | **⭐ ความซับซ้อน**: ระดับกลาง

---

## ภาพรวม

บทนี้ครอบคลุมการกำหนดค่าสภาพแวดล้อม รูปแบบการรับรองความถูกต้อง และแนวทางปฏิบัติที่ดีที่สุดด้านความปลอดภัยสำหรับการปรับใช้ Azure Developer CLI

> ตรวจสอบความถูกต้องกับ `azd 1.23.12` ในเดือนมีนาคม 2026

## จุดประสงค์การเรียนรู้

โดยการทำบทนี้ให้เสร็จสมบูรณ์ คุณจะ:
- เชี่ยวชาญในการจัดลำดับชั้นการกำหนดค่า AZD
- จัดการสภาพแวดล้อมหลายตัว (dev, staging, prod)
- นำระบบการรับรองความถูกต้องที่ปลอดภัยด้วย managed identities ไปใช้
- กำหนดค่าการตั้งค่าสำหรับแต่ละสภาพแวดล้อมเฉพาะ

## 🚀 เริ่มต้นอย่างรวดเร็ว

```bash
# สร้างสภาพแวดล้อมหลายตัว
azd env new dev
azd env new staging
azd env new prod

# สลับสภาพแวดล้อม
azd env select prod

# ตั้งค่าตัวแปรสภาพแวดล้อม
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# ดูการตั้งค่า
azd env get-values
```

---

## 🔧 ลำดับชั้นการกำหนดค่า

AZD นำการตั้งค่าไปใช้ตามลำดับนี้ (ค่าหลังจะ override ค่าก่อนหน้า):

1. **ค่าปริยาย** (ที่ฝังอยู่ในเท็มเพลต)
2. **azure.yaml** (การกำหนดค่าโปรเจกต์)
3. **ตัวแปรสภาพแวดล้อม** (`azd env set`)
4. **คำสั่งบรรทัดคำสั่ง** (`--location eastus`)

## 🔐 แนวทางปฏิบัติที่ดีที่สุดด้านความปลอดภัย

```bash
# ใช้ managed identity (แนะนำ)
azd env set AZURE_USE_MANAGED_IDENTITY true

# ตรวจสอบสถานะการยืนยันตัวตนของ AZD
azd auth status

# ตัวเลือก: ตรวจสอบบริบทของ Azure CLI หากคุณวางแผนที่จะรันคำสั่ง az
az account show

# ลงชื่อเข้าใช้อีกครั้งหากจำเป็น
azd auth login

# ตัวเลือก: รีเฟรชการยืนยันตัวตน Azure CLI สำหรับคำสั่ง az
az login
```
