# บทที่ 3: การตั้งค่า & การตรวจสอบสิทธิ์

**📚 คอร์ส**: [AZD สำหรับผู้เริ่มต้น](../../README.md) | **⏱️ ระยะเวลา**: 45-60 นาที | **⭐ ระดับความยาก**: ปานกลาง

---

## ภาพรวม

บทนี้ครอบคลุมการตั้งค่าสภาพแวดล้อม รูปแบบการตรวจสอบสิทธิ์ และแนวปฏิบัติด้านความปลอดภัยที่ดีที่สุดสำหรับการปรับใช้ Azure Developer CLI

> ได้รับการทดสอบกับ `azd 1.25.6` ในเดือนมิถุนายน 2026

## วัตถุประสงค์การเรียนรู้

เมื่อคุณจบบทนี้ คุณจะสามารถ:
- เชี่ยวชาญลำดับชั้นการตั้งค่า AZD
- จัดการสภาพแวดล้อมหลายแบบ (dev, staging, prod)
- ใช้การตรวจสอบสิทธิ์ที่ปลอดภัยด้วย managed identities
- กำหนดค่าการตั้งค่าที่เฉพาะเจาะจงสำหรับแต่ละสภาพแวดล้อม

## 🚀 เริ่มต้นอย่างรวดเร็ว

```bash
# สร้างสภาพแวดล้อมหลายรายการ
azd env new dev
azd env new staging
azd env new prod

# สลับสภาพแวดล้อม
azd env select prod

# ตั้งค่าตัวแปรสภาพแวดล้อม
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# ดูการกำหนดค่า
azd env get-values
```

---

## 🔧 ลำดับชั้นการตั้งค่า

AZD ใช้การตั้งค่าในลำดับนี้ (การตั้งค่าภายหลังแทนที่ก่อนหน้า):

1. **ค่าเริ่มต้น** (ฝังในเทมเพลต)
2. **azure.yaml** (การตั้งค่าโครงการ)
3. **ตัวแปรสภาพแวดล้อม** (`azd env set`)
4. **คำสั่งบรรทัดคำสั่ง** (`--location eastus`)

## 🔐 แนวปฏิบัติที่ดีที่สุดด้านความปลอดภัย

```bash
# ใช้ตัวตนที่มีการจัดการ (แนะนำ)
azd env set AZURE_USE_MANAGED_IDENTITY true

# ตรวจสอบสถานะการรับรองความถูกต้องของ AZD
azd auth status

# เป็นทางเลือก: ยืนยันบริบท Azure CLI หากคุณวางแผนจะรันคำสั่ง az
az account show

# เข้าสู่ระบบใหม่หากจำเป็น
azd auth login

# เป็นทางเลือก: รีเฟรชการรับรองความถูกต้องของ Azure CLI สำหรับคำสั่ง az
az login
```
