# บทที่ 7: การแก้ไขปัญหาและดีบัก

**📚 คอร์ส**: [AZD สำหรับผู้เริ่มต้น](../../README.md) | **⏱️ ระยะเวลา**: 1-1.5 ชั่วโมง | **⭐ ความซับซ้อน**: ระดับกลาง

---

## ภาพรวม

บทนี้ช่วยคุณวิเคราะห์และแก้ไขปัญหาทั่วไปเมื่อใช้งาน Azure Developer CLI ตั้งแต่ความล้มเหลวในการปรับใช้งานจนถึงปัญหาเฉพาะทาง AI

> ตรวจสอบความถูกต้องกับ `azd 1.27.1` ในเดือนกรกฎาคม 2026

## วัตถุประสงค์การเรียนรู้

โดยการเรียนบทนี้ คุณจะสามารถ:
- วิเคราะห์ความล้มเหลวทั่วไปของการปรับใช้งาน AZD
- ดีบักปัญหาการตรวจสอบสิทธิ์และสิทธิ์การใช้งาน
- แก้ไขปัญหาการเชื่อมต่อบริการ AI
- ใช้ Azure Portal และ CLI เพื่อการแก้ไขปัญหา

## 🚨 การแก้ไขด่วน

### ปัญหาการตรวจสอบสิทธิ์
```bash
# จำเป็นสำหรับกระบวนการทำงาน AZD
azd auth login

# เป็นทางเลือกถ้าคุณใช้คำสั่ง Azure CLI โดยตรงด้วย
az login

azd auth status
```

### ความล้มเหลวในการจัดเตรียม
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### ความขัดแย้งของทรัพยากร
```bash
azd down --force --purge
azd env new different-name
azd up
```

### เกินโควต้า
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 อ้างอิงรหัสข้อผิดพลาด

| ข้อผิดพลาด | สาเหตุ | วิธีแก้ไข |
|-------|-------|----------|
| `AuthenticationError` | ยังไม่ได้เข้าสู่ระบบ | `azd auth login` |
| `ResourceNotFound` | ไม่มีทรัพยากร | ตรวจสอบชื่อทรัพยากร |
| `QuotaExceeded` | ขีดจำกัดการสมัครใช้งาน | ขอเพิ่มโควต้า |
| `InvalidTemplate` | ไวยากรณ์ Bicep ผิดพลาด | `az bicep build` |
| `Conflict` | ทรัพยากรมีอยู่แล้ว | ใช้ชื่อใหม่หรือลบออก |
| `Forbidden` | สิทธิ์ไม่เพียงพอ | ตรวจสอบบทบาท RBAC |

## 🔄 การรีเซ็ตและการกู้คืน

```bash
# รีเซ็ตแบบนุ่มนวล (เก็บทรัพยากรไว้, ปล่อยโค้ดใหม่)
azd deploy --force

# รีเซ็ตแบบเต็มรูปแบบ (ลบทุกอย่าง, เริ่มใหม่)
azd down --force --purge
azd up
```
