# Chapter 7: การแก้ไขปัญหาและดีบัก

**📚 หลักสูตร**: [AZD สำหรับผู้เริ่มต้น](../../README.md) | **⏱️ ระยะเวลา**: 1-1.5 ชั่วโมง | **⭐ ความซับซ้อน**: ระดับกลาง

---

## ภาพรวม

บทนี้ช่วยคุณวินิจฉัยและแก้ไขปัญหาทั่วไปเมื่อทำงานกับ Azure Developer CLI ตั้งแต่ความล้มเหลวในการดีพลอยไปจนถึงปัญหาเฉพาะของ AI

> ได้รับการตรวจสอบกับ `azd 1.23.12` ในเดือนมีนาคม 2026

## วัตถุประสงค์การเรียนรู้

เมื่อทำบทนี้จบ คุณจะสามารถ:
- วินิจฉัยความล้มเหลวทั่วไปในการดีพลอย AZD
- ดีบักปัญหาการพิสูจน์ตัวตนและสิทธิ์
- แก้ไขปัญหาการเชื่อมต่อบริการ AI
- ใช้ Azure Portal และ CLI ในการแก้ไขปัญหา

## 🚨 การแก้ไขด่วน

### ปัญหาการพิสูจน์ตัวตน
```bash
# จำเป็นสำหรับเวิร์กโฟลว์ AZD
azd auth login

# เป็นทางเลือกถ้าคุณใช้คำสั่ง Azure CLI โดยตรงด้วย
az login

azd auth status
```

### ความล้มเหลวในการจัดเตรียม
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### ขัดแย้งของทรัพยากร
```bash
azd down --force --purge
azd env new different-name
azd up
```

### เกินโควต้า
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 อ้างอิงรหัสข้อผิดพลาด

| ข้อผิดพลาด | สาเหตุ | วิธีแก้ไข |
|-------|-------|----------|
| `AuthenticationError` | ยังไม่ได้เข้าสู่ระบบ | `azd auth login` |
| `ResourceNotFound` | ไม่มีทรัพยากรดังกล่าว | ตรวจสอบชื่อทรัพยากร |
| `QuotaExceeded` | ข้อจำกัดการสมัครสมาชิก | ขอเพิ่มโควต้า |
| `InvalidTemplate` | ข้อผิดพลาดไวยากรณ์ Bicep | `az bicep build` |
| `Conflict` | ทรัพยากรมีอยู่แล้ว | ใช้ชื่อใหม่หรือ ลบทิ้ง |
| `Forbidden` | สิทธิ์ไม่เพียงพอ | ตรวจสอบบทบาท RBAC |

## 🔄 รีเซ็ตและกู้คืน

```bash
# รีเซ็ตแบบนุ่มนวล (เก็บทรัพยากร, ติดตั้งโค้ดใหม่)
azd deploy --force

# รีเซ็ตแบบหนัก (ลบทุกอย่าง, เริ่มใหม่)
azd down --force --purge
azd up
```
