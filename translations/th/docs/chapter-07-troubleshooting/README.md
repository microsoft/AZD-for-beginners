# บทที่ 7: การแก้ไขปัญหาและการดีบัก

**📚 หลักสูตร**: [AZD สำหรับผู้เริ่มต้น](../../README.md) | **⏱️ ระยะเวลา**: 1-1.5 ชั่วโมง | **⭐ ความซับซ้อน**: ปานกลาง

---

## ภาพรวม

บทนี้ช่วยให้คุณวินิจฉัยและแก้ไขปัญหาที่พบบ่อยเมื่อทำงานกับ Azure Developer CLI ตั้งแต่ความล้มเหลวในการปรับใช้ไปจนถึงปัญหาเฉพาะของ AI

## วัตถุประสงค์การเรียนรู้

เมื่อสำเร็จบทนี้ คุณจะสามารถ:
- วินิจฉัยข้อผิดพลาดการปรับใช้ AZD ที่พบบ่อย
- ดีบักปัญหาการพิสูจน์ตัวตนและสิทธิ์
- แก้ไขปัญหาการเชื่อมต่อบริการ AI
- ใช้ Azure Portal และ CLI สำหรับการแก้ไขปัญหา

## 🚨 การแก้ไขด่วน

### ปัญหาการพิสูจน์ตัวตน
```bash
azd auth login
az login
azd auth whoami
```

### ข้อผิดพลาดในการจัดเตรียม
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### ความขัดแย้งของทรัพยากร
```bash
azd down --force --purge
azd env new different-name
azd up
```

### เกินโควต้า
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 รายการรหัสข้อผิดพลาด

| ข้อผิดพลาด | สาเหตุ | วิธีแก้ไข |
|-------|-------|----------|
| `AuthenticationError` | ยังไม่ได้เข้าสู่ระบบ | `azd auth login` |
| `ResourceNotFound` | ไม่พบทรัพยากร | ตรวจสอบชื่อทรัพยากร |
| `QuotaExceeded` | ขีดจำกัดของการสมัครใช้งาน | ขอเพิ่มโควต้า |
| `InvalidTemplate` | ข้อผิดพลาดไวยากรณ์ Bicep | `az bicep build` |
| `Conflict` | ทรัพยากรมีอยู่แล้ว | ใช้ชื่อใหม่หรือทำการลบ |
| `Forbidden` | สิทธิ์ไม่เพียงพอ | ตรวจสอบบทบาท RBAC |

## 🔄 รีเซ็ตและกู้คืน

```bash
# รีเซ็ตแบบนุ่มนวล (เก็บทรัพยากรไว้ ปรับใช้โค้ดใหม่)
azd deploy --force

# รีเซ็ตแบบหนัก (ลบทุกอย่าง เริ่มใหม่จากศูนย์)
azd down --force --purge
azd up
```
