# Chapter 7: การแก้ไขปัญหาและดีบัก

**📚 คอร์ส**: [AZD สำหรับผู้เริ่มต้น](../../README.md) | **⏱️ ระยะเวลา**: 1-1.5 ชั่วโมง | **⭐ ระดับความซับซ้อน**: ระดับกลาง

---

## ภาพรวม

บทนี้จะช่วยให้คุณวินิจฉัยและแก้ไขปัญหาทั่วไปเมื่อทำงานกับ Azure Developer CLI ตั้งแต่การดีพลอยล้มเหลวจนถึงปัญหาเฉพาะด้าน AI

> ทดสอบกับ `azd 1.25.6` ในเดือนมิถุนายน ค.ศ. 2026

## วัตถุประสงค์การเรียนรู้

เมื่อทำบทนี้จบ คุณจะสามารถ:
- วินิจฉัยปัญหาการดีพลอย AZD ที่พบบ่อย
- ดีบักปัญหาการยืนยันตัวตนและสิทธิ์
- แก้ไขปัญหาการเชื่อมต่อบริการ AI
- ใช้งาน Azure Portal และ CLI ในการแก้ไขปัญหา

## 🚨 แก้ไขด่วน

### ปัญหาการยืนยันตัวตน
```bash
# จำเป็นสำหรับเวิร์กโฟลว์ AZD
azd auth login

# เป็นทางเลือกหากคุณใช้คำสั่ง Azure CLI โดยตรงด้วย
az login

azd auth status
```

### การดีพลอยล้มเหลว
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### ความขัดแย้งของทรัพยากร
```bash
azd down --force --purge
azd env new different-name
azd up
```

### โควต้าเกินขีดจำกัด
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 อ้างอิงรหัสข้อผิดพลาด

| ข้อผิดพลาด | สาเหตุ | วิธีแก้ไข |
|-------|-------|----------|
| `AuthenticationError` | ไม่ได้เข้าสู่ระบบ | `azd auth login` |
| `ResourceNotFound` | ไม่มีทรัพยากร | ตรวจสอบชื่อทรัพยากร |
| `QuotaExceeded` | เกินขีดจำกัดการสมัครใช้งาน | ขอเพิ่มโควต้า |
| `InvalidTemplate` | ไวยากรณ์ Bicep ผิดพลาด | `az bicep build` |
| `Conflict` | ทรัพยากรมีอยู่แล้ว | ใช้ชื่อใหม่หรือทำการลบ |
| `Forbidden` | สิทธิ์ไม่เพียงพอ | ตรวจสอบบทบาท RBAC |

## 🔄 รีเซ็ตและกู้คืน

```bash
# รีเซ็ตแบบนุ่มนวล (เก็บทรัพยากร, ปล่อยรหัสใหม่)
azd deploy --force

# รีเซ็ตแบบแข็ง (ลบทุกอย่าง, เริ่มต้นใหม่)
azd down --force --purge
azd up
```
