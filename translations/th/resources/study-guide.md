# Study Guide - วัตถุประสงค์การเรียนรู้ที่ครอบคลุม

**การนำทางเส้นทางการเรียนรู้**
- **📚 หน้าแรกของหลักสูตร**: [AZD For Beginners](../README.md)
- **📖 เริ่มต้นการเรียนรู้**: [บทที่ 1: พื้นฐาน & เริ่มต้นด่วน](../README.md#-chapter-1-foundation--quick-start)
- **🎯 การติดตามความก้าวหน้า**: [การจบหลักสูตร](../README.md#-course-completion--certification)

## บทนำ

คู่มือการศึกษาครอบคลุมนี้มอบวัตถุประสงค์การเรียนรู้ที่มีโครงสร้าง, แนวคิดสำคัญ, แบบฝึกหัดปฏิบัติ, และวัสดุประเมินผลเพื่อช่วยให้คุณเชี่ยวชาญ Azure Developer CLI (azd) ใช้คู่มือนี้เพื่อติดตามความก้าวหน้าของคุณและมั่นใจว่าคุณได้ครอบคลุมหัวข้อที่สำคัญทั้งหมดแล้ว

## เป้าหมายการเรียนรู้

เมื่อทำคู่มือนี้จนครบถ้วน คุณจะสามารถ:
- เชี่ยวชาญแนวคิดพื้นฐานและขั้นสูงทั้งหมดของ Azure Developer CLI
- พัฒนาทักษะเชิงปฏิบัติในการปรับใช้และจัดการแอป Azure
- สร้างความมั่นใจในการแก้ปัญหาและเพิ่มประสิทธิภาพการปรับใช้
- เข้าใจแนวทางปฏิบัติการปรับใช้ที่พร้อมใช้งานในระบบจริงและข้อพึงระวังด้านความปลอดภัย

## ผลลัพธ์การเรียนรู้

หลังจากทำแต่ละส่วนของคู่มือนี้เสร็จสิ้น คุณจะสามารถ:
- ออกแบบ, ปรับใช้, และจัดการสถาปัตยกรรมแอปทั้งระบบโดยใช้ azd
- ดำเนินกลยุทธ์การตรวจสอบ, ความปลอดภัย, และการเพิ่มประสิทธิภาพค่าใช้จ่ายที่ครอบคลุม
- แก้ไขปัญหาการปรับใช้ที่ซับซ้อนได้อย่างอิสระ
- สร้างแม่แบบแบบกำหนดเองและมีส่วนร่วมกับชุมชน azd

## โครงสร้างการเรียนรู้ 8 บท

### บทที่ 1: พื้นฐาน & เริ่มต้นด่วน (สัปดาห์ที่ 1)
**ระยะเวลา**: 30-45 นาที | **ระดับความยาก**: ⭐

#### วัตถุประสงค์การเรียนรู้
- เข้าใจแนวคิดหลักและคำศัพท์ของ Azure Developer CLI
- ติดตั้งและกำหนดค่า AZD บนแพลตฟอร์มการพัฒนาของคุณสำเร็จ
- ปรับใช้แอปแรกของคุณโดยใช้แม่แบบที่มีอยู่
- ใช้งานอินเทอร์เฟซบรรทัดคำสั่ง AZD อย่างมีประสิทธิภาพ

#### แนวคิดสำคัญที่ต้องเชี่ยวชาญ
- โครงสร้างโครงการ AZD และส่วนประกอบ (azure.yaml, infra/, src/)
- กระบวนการปรับใช้โดยใช้แม่แบบ
- พื้นฐานการกำหนดค่าสภาพแวดล้อม
- การจัดการกลุ่มทรัพยากรและการสมัครสมาชิก

#### แบบฝึกหัดปฏิบัติ
1. **การตรวจสอบการติดตั้ง**: ติดตั้ง AZD และตรวจสอบด้วยคำสั่ง `azd version`
2. **การปรับใช้ครั้งแรก**: ปรับใช้แม่แบบ todo-nodejs-mongo ให้สำเร็จ
3. **การตั้งค่าสภาพแวดล้อม**: กำหนดค่าตัวแปรสภาพแวดล้อมชุดแรกของคุณ
4. **การสำรวจทรัพยากร**: นำทางทรัพยากรที่ปรับใช้ใน Azure Portal

#### คำถามเพื่อประเมินผล
- ส่วนประกอบหลักของโครงการ AZD มีอะไรบ้าง?
- คุณเริ่มโครงการใหม่จากแม่แบบได้อย่างไร?
- ความแตกต่างระหว่าง `azd up` กับ `azd deploy` คืออะไร?
- คุณจัดการหลายสภาพแวดล้อมด้วย AZD อย่างไร?

### บทที่ 2: การพัฒนา AI เป็นอันดับแรก (สัปดาห์ที่ 2)
**ระยะเวลา**: 1-2 ชั่วโมง | **ระดับความยาก**: ⭐⭐

#### วัตถุประสงค์การเรียนรู้
- ผสานบริการ Microsoft Foundry เข้ากับเวิร์กโฟลว์ AZD
- ปรับใช้และกำหนดค่าแอปที่ใช้ AI
- เข้าใจรูปแบบการใช้งาน RAG (Retrieval-Augmented Generation)
- จัดการปรับใช้และการขยายขนาดโมเดล AI

#### แนวคิดสำคัญที่ต้องเชี่ยวชาญ
- การผสานบริการ Microsoft Foundry Models และการจัดการ API
- การกำหนดค่า AI Search และการจัดทำดัชนีเวกเตอร์
- กลยุทธ์การปรับใช้โมเดลและการวางแผนความจุ
- การตรวจสอบแอป AI และการเพิ่มประสิทธิภาพการทำงาน

#### แบบฝึกหัดปฏิบัติ
1. **ปรับใช้แชท AI**: ปรับใช้แม่แบบ azure-search-openai-demo
2. **การใช้งาน RAG**: กำหนดค่าการจัดทำดัชนีและการดึงเอกสาร
3. **การตั้งค่าโมเดล**: ตั้งค่าโมเดล AI หลายแบบตามวัตถุประสงค์ต่าง ๆ
4. **การตรวจสอบ AI**: ใช้ Application Insights เพื่อตรวจสอบเวิร์กโหลด AI

#### คำถามเพื่อประเมินผล
- คุณกำหนดค่าบริการ Microsoft Foundry Models ในแม่แบบ AZD อย่างไร?
- ส่วนประกอบหลักของสถาปัตยกรรม RAG คืออะไร?
- คุณจัดการความจุและการขยายขนาดโมเดล AI อย่างไร?
- ตัวชี้วัดใดสำคัญสำหรับการตรวจสอบแอป AI?

### บทที่ 3: การตั้งค่าและการยืนยันตัวตน (สัปดาห์ที่ 3)
**ระยะเวลา**: 45-60 นาที | **ระดับความยาก**: ⭐⭐

#### วัตถุประสงค์การเรียนรู้
- เชี่ยวชาญกลยุทธ์การตั้งค่าและจัดการสภาพแวดล้อม
- ดำเนินรูปแบบการยืนยันตัวตนอย่างปลอดภัยและ Managed Identity
- จัดระเบียบทรัพยากรด้วยรูปแบบการตั้งชื่อที่เหมาะสม
- กำหนดค่าการปรับใช้หลายสภาพแวดล้อม (dev, staging, prod)

#### แนวคิดสำคัญที่ต้องเชี่ยวชาญ
- ลำดับขั้นและลำดับความสำคัญของการตั้งค่าสภาพแวดล้อม
- การยืนยันตัวตน Managed Identity และ Service Principal
- การผสาน Azure Key Vault สำหรับการจัดการความลับ
- การจัดการพารามิเตอร์เฉพาะสภาพแวดล้อม

#### แบบฝึกหัดปฏิบัติ
1. **การตั้งค่าสภาพแวดล้อมหลายตัว**: กำหนดค่า dev, staging, และ prod
2. **การกำหนดค่าความปลอดภัย**: ดำเนินการยืนยันตัวตนด้วย Managed Identity
3. **การจัดการความลับ**: ผสาน Azure Key Vault สำหรับข้อมูลที่ละเอียดอ่อน
4. **การจัดการพารามิเตอร์**: สร้างการตั้งค่าเฉพาะสภาพแวดล้อม

#### คำถามเพื่อประเมินผล
- คุณกำหนดค่าสภาพแวดล้อมที่แตกต่างกันด้วย AZD อย่างไร?
- ประโยชน์ของการใช้ Managed Identity แทน Service Principals คืออะไร?
- คุณจัดการความลับของแอปได้อย่างปลอดภัยอย่างไร?
- ลำดับขั้นการกำหนดค่าใน AZD เป็นอย่างไร?

### บทที่ 4: Infrastructure as Code & การปรับใช้ (สัปดาห์ที่ 4-5)
**ระยะเวลา**: 1-1.5 ชั่วโมง | **ระดับความยาก**: ⭐⭐⭐

#### วัตถุประสงค์การเรียนรู้
- สร้างและปรับแต่งแม่แบบโครงสร้างพื้นฐานด้วย Bicep
- ดำเนินรูปแบบและเวิร์กโฟลว์การปรับใช้ขั้นสูง
- เข้าใจกลยุทธ์การจัดเตรียมทรัพยากร
- ออกแบบสถาปัตยกรรมขยายขนาดหลายบริการ

- ปรับใช้แอปที่ใช้คอนเทนเนอร์โดยใช้ Azure Container Apps และ AZD


#### แนวคิดสำคัญที่ต้องเชี่ยวชาญ
- โครงสร้างแม่แบบ Bicep และแนวทางปฏิบัติที่ดีที่สุด
- การขึ้นต่อกันของทรัพยากรและลำดับการปรับใช้
- ไฟล์พารามิเตอร์และการแยกโมดูลแม่แบบ
- ตัว钩กำหนดเองและการอัตโนมัติการปรับใช้
- รูปแบบการปรับใช้แอป Container (เริ่มต้นด่วน, ผลิต, ไมโครเซอร์วิส)

#### แบบฝึกหัดปฏิบัติ
1. **การสร้างแม่แบบแบบกำหนดเอง**: สร้างแม่แบบแอปหลายบริการ
2. **ฝึกเชี่ยวชาญ Bicep**: สร้างส่วนประกอบโครงสร้างพื้นฐานแบบโมดูลและนำกลับมาใช้ได้
3. **อัตโนมัติการปรับใช้**: ดำเนิน钩ก่อน/หลังการปรับใช้
4. **ออกแบบสถาปัตยกรรม**: ปรับใช้สถาปัตยกรรมไมโครเซอร์วิสที่ซับซ้อน
5. **ปรับใช้แอป Container**: ปรับใช้ตัวอย่าง [Simple Flask API](../../../examples/container-app/simple-flask-api) และ [Microservices Architecture](../../../examples/container-app/microservices) โดยใช้ AZD

#### คำถามเพื่อประเมินผล
- คุณสร้างแม่แบบ Bicep แบบกำหนดเองสำหรับ AZD อย่างไร?
- แนวทางปฏิบัติที่ดีที่สุดสำหรับการจัดระเบียบโค้ดโครงสร้างพื้นฐานคืออะไร?
- คุณจัดการความขึ้นต่อกันของทรัพยากรในแม่แบบอย่างไร?
- รูปแบบการปรับใช้ใดที่สนับสนุนการอัปเดตโดยไม่หยุดชะงัก?

### บทที่ 5: โซลูชัน AI หลายตัวแทน (สัปดาห์ที่ 6-7)
**ระยะเวลา**: 2-3 ชั่วโมง | **ระดับความยาก**: ⭐⭐⭐⭐

#### วัตถุประสงค์การเรียนรู้
- ออกแบบและดำเนินสถาปัตยกรรม AI หลายตัวแทน
- ประสานงานและสื่อสารระหว่างตัวแทน
- ปรับใช้โซลูชัน AI พร้อมใช้งานในระบบจริงพร้อมการตรวจสอบ
- เข้าใจความชำนาญเฉพาะตัวของตัวแทนและรูปแบบเวิร์กโฟลว์
- ผสานไมโครเซอร์วิสที่ใช้คอนเทนเนอร์เป็นส่วนหนึ่งของโซลูชันหลายตัวแทน

#### แนวคิดสำคัญที่ต้องเชี่ยวชาญ
- รูปแบบสถาปัตยกรรมหลายตัวแทนและหลักการออกแบบ
- โปรโตคอลการสื่อสารของตัวแทนและการไหลของข้อมูล
- กลยุทธ์การปรับสมดุลโหลดและขยายขนาดสำหรับตัวแทน AI
- การตรวจสอบระบบหลายตัวแทนในสภาพแวดล้อมการผลิต
- การสื่อสารระหว่างบริการในสภาพแวดล้อมที่ใช้คอนเทนเนอร์

#### แบบฝึกหัดปฏิบัติ
1. **ปรับใช้โซลูชันค้าปลีก**: ปรับใช้สถานการณ์ค้าปลีกหลายตัวแทนครบวงจร
2. **ปรับแต่งตัวแทน**: แก้ไขพฤติกรรมตัวแทน Customer และ Inventory
3. **ขยายสถาปัตยกรรม**: ดำเนินการปรับสมดุลโหลดและการขยายอัตโนมัติ
4. **ตรวจสอบระบบในสภาพแวดล้อมจริง**: ตั้งค่าการตรวจสอบและแจ้งเตือนครอบคลุม
5. **ผสานไมโครเซอร์วิส**: ขยายตัวอย่าง [Microservices Architecture](../../../examples/container-app/microservices) เพื่อสนับสนุนเวิร์กโฟลว์แบบตัวแทน

#### คำถามเพื่อประเมินผล
- คุณออกแบบรูปแบบการสื่อสารหลายตัวแทนอย่างไรให้มีประสิทธิภาพ?
- ข้อพิจารณาหลักสำหรับการขยายงานโหลดของตัวแทน AI คืออะไร?
- คุณตรวจสอบและแก้ไขปัญหาระบบ AI หลายตัวแทนอย่างไร?
- รูปแบบการผลิตใดที่ช่วยให้ AI Agent มีความน่าเชื่อถือ?

### บทที่ 6: การตรวจสอบและวางแผนก่อนปรับใช้ (สัปดาห์ที่ 8)
**ระยะเวลา**: 1 ชั่วโมง | **ระดับความยาก**: ⭐⭐

#### วัตถุประสงค์การเรียนรู้
- ปฏิบัติการวางแผนความจุและตรวจสอบทรัพยากรอย่างครบถ้วน
- เลือก SKU Azure ที่เหมาะสมสำหรับประสิทธิภาพค่าใช้จ่าย
- ดำเนินการตรวจสอบล่วงหน้าที่อัตโนมัติและการยืนยันผล
- วางแผนการปรับใช้ด้วยกลยุทธ์การเพิ่มประสิทธิภาพค่าใช้จ่าย

#### แนวคิดสำคัญที่ต้องเชี่ยวชาญ
- โควต้าทรัพยากร Azure และข้อจำกัดความจุ
- เกณฑ์การเลือก SKU และการเพิ่มประสิทธิภาพค่าใช้จ่าย
- สคริปต์การตรวจสอบอัตโนมัติและการทดสอบ
- การวางแผนการปรับใช้และการประเมินความเสี่ยง

#### แบบฝึกหัดปฏิบัติ
1. **การวิเคราะห์ความจุ**: วิเคราะห์ข้อกำหนดทรัพยากรสำหรับแอปของคุณ
2. **การเพิ่มประสิทธิภาพ SKU**: เปรียบเทียบและเลือกแผนบริการที่คุ้มค่า
3. **อัตโนมัติสำหรับการตรวจสอบ**: ดำเนินสคริปต์ตรวจสอบก่อนปรับใช้
4. **การวางแผนค่าใช้จ่าย**: สร้างประมาณการและงบประมาณค่าใช้จ่ายปรับใช้

#### คำถามเพื่อประเมินผล

## โครงการปฏิบัติ

### โครงการ 1: เว็บไซต์แสดงผลงานส่วนบุคคล
**ความซับซ้อน**: ระดับเริ่มต้น  
**ระยะเวลา**: 1-2 สัปดาห์

สร้างและปรับใช้เว็บไซต์แสดงผลงานส่วนบุคคลโดยใช้:
- การโฮสต์เว็บไซต์แบบคงที่บน Azure Storage
- การกำหนดค่าโดเมนแบบกำหนดเอง
- การบูรณาการ CDN เพื่อประสิทธิภาพทั่วโลก
- ท่อการปรับใช้อัตโนมัติ

**ผลลัพธ์ที่ส่งมอบ**:
- เว็บไซต์ทำงานจริงบน Azure
- ต้นแบบ azd แบบกำหนดเองสำหรับการปรับใช้ผลงาน
- เอกสารกระบวนการปรับใช้
- การวิเคราะห์ค่าใช้จ่ายและคำแนะนำการเพิ่มประสิทธิภาพ

### โครงการ 2: แอปพลิเคชันจัดการงาน
**ความซับซ้อน**: ระดับกลาง  
**ระยะเวลา**: 2-3 สัปดาห์

สร้างแอปพลิเคชันจัดการงานแบบฟูลสแตกด้วย:
- ฟรอนต์เอนด์ React ปรับใช้บน App Service
- แบ็กเอนด์ API Node.js พร้อมการตรวจสอบสิทธิ์
- ฐานข้อมูล PostgreSQL พร้อมสคริปต์มิเกรชัน
- การตรวจสอบด้วย Application Insights

**ผลลัพธ์ที่ส่งมอบ**:
- แอปพลิเคชันสมบูรณ์พร้อมการตรวจสอบสิทธิ์ผู้ใช้
- สคีมาฐานข้อมูลและสคริปต์มิเกรชัน
- แดชบอร์ดตรวจสอบและกฎแจ้งเตือน
- การกำหนดค่าการปรับใช้หลายสภาพแวดล้อม

### โครงการ 3: แพลตฟอร์มอีคอมเมิร์ซแบบไมโครเซอร์วิส
**ความซับซ้อน**: ขั้นสูง  
**ระยะเวลา**: 4-6 สัปดาห์

ออกแบบและดำเนินการแพลตฟอร์มอีคอมเมิร์ซแบบไมโครเซอร์วิส:
- บริการ API หลายรายการ (แคตตาล็อก, คำสั่งซื้อ, การชำระเงิน, ผู้ใช้)
- การบูรณาการคิวข้อความกับ Service Bus
- Redis cache เพื่อเพิ่มประสิทธิภาพ
- การบันทึกและตรวจสอบที่ครอบคลุม

**ตัวอย่างอ้างอิง**: ดู [สถาปัตยกรรมไมโครเซอร์วิส](../../../examples/container-app/microservices) สำหรับต้นแบบพร้อมใช้งานจริงและคำแนะนำการปรับใช้

**ผลลัพธ์ที่ส่งมอบ**:
- สถาปัตยกรรมไมโครเซอร์วิสครบถ้วน
- รูปแบบการสื่อสารระหว่างบริการ
- การทดสอบประสิทธิภาพและการเพิ่มประสิทธิภาพ
- การติดตั้งความปลอดภัยพร้อมใช้งานจริง

## การประเมินและการรับรอง

### การตรวจสอบความรู้

ทำการประเมินเหล่านี้หลังจากแต่ละโมดูล:

**การประเมินโมดูล 1**: แนวคิดพื้นฐานและการติดตั้ง
- คำถามแบบเลือกตอบเกี่ยวกับแนวคิดหลัก
- งานปฏิบัติการติดตั้งและกำหนดค่า
- แบบฝึกหัดปรับใช้พื้นฐาน

**การประเมินโมดูล 2**: การกำหนดค่าและสภาพแวดล้อม
- กรณีจัดการสภาพแวดล้อม
- แบบฝึกหัดแก้ไขปัญหาการกำหนดค่า
- การดำเนินการกำหนดค่าความปลอดภัย

**การประเมินโมดูล 3**: การปรับใช้และจัดหา
- ความท้าทายด้านการออกแบบโครงสร้างพื้นฐาน
- กรณีการปรับใช้หลายบริการ
- แบบฝึกหัดเพิ่มประสิทธิภาพ

**การประเมินโมดูล 4**: การตรวจสอบก่อนปรับใช้
- กรณีศึกษาการวางแผนความสามารถ
- กรณีการเพิ่มประสิทธิภาพค่าใช้จ่าย
- การดำเนินการท่อการตรวจสอบ

**การประเมินโมดูล 5**: การแก้ไขปัญหาและดีบัก
- แบบฝึกหัดวินิจฉัยปัญหา
- งานการติดตั้งระบบตรวจสอบ
- การจำลองการตอบสนองเหตุการณ์

**การประเมินโมดูล 6**: หัวข้อขั้นสูง
- การออกแบบท่อ CI/CD
- การพัฒนาต้นแบบแบบกำหนดเอง
- กรณีสถาปัตยกรรมองค์กร

### โครงการสรุปขั้นสุดท้าย

ออกแบบและดำเนินการโซลูชันสมบูรณ์ที่แสดงถึงความเชี่ยวชาญในทุกแนวคิด:

**ข้อกำหนด**:
- สถาปัตยกรรมแอปหลายชั้น
- การปรับใช้หลายสภาพแวดล้อม
- การตรวจสอบและแจ้งเตือนแบบครบวงจร
- การดำเนินการด้านความปลอดภัยและการปฏิบัติตาม
- การเพิ่มประสิทธิภาพค่าใช้จ่ายและการปรับปรุงประสิทธิภาพ
- เอกสารและหนังสือคำสั่งการใช้งานครบถ้วน

**เกณฑ์การประเมิน**:
- คุณภาพการดำเนินการทางเทคนิค
- ความสมบูรณ์ของเอกสาร
- การปฏิบัติตามความปลอดภัยและแนวทางปฏิบัติที่ดีที่สุด
- การเพิ่มประสิทธิภาพประสิทธิภาพและค่าใช้จ่าย
- ประสิทธิผลในการแก้ไขปัญหาและการตรวจสอบ

- [ ] **ปรับแต่งเอเจนต์**: แก้ไขพฤติกรรมเอเจนต์ลูกค้าและสินค้าคงคลัง
- [ ] **ปรับขนาดสถาปัตยกรรม**: ดำเนินการบาลานซ์โหลดและสเกลอัตโนมัติ
- [ ] **ตรวจสอบการผลิต**: ตั้งค่าการตรวจสอบและแจ้งเตือนแบบครบวงจร
- [ ] **ปรับแต่งประสิทธิภาพ**: เพิ่มประสิทธิภาพระบบหลายเอเจนต์

#### 🔍 บทที่ 6: การตรวจสอบและวางแผนก่อนปรับใช้
- [ ] **วิเคราะห์ความสามารถ**: วิเคราะห์ความต้องการทรัพยากรสำหรับแอปพลิเคชัน
- [ ] **เพิ่มประสิทธิภาพ SKU**: เลือกระดับบริการที่คุ้มค่า
- [ ] **อัตโนมัติการตรวจสอบ**: ดำเนินการสคริปต์ตรวจสอบก่อนปรับใช้
- [ ] **วางแผนค่าใช้จ่าย**: สร้างประมาณการและงบประมาณค่าใช้จ่าย
- [ ] **ประเมินความเสี่ยง**: ระบุและลดความเสี่ยงการปรับใช้

#### 🚨 บทที่ 7: การแก้ไขปัญหาและดีบัก
- [ ] **ทักษะวินิจฉัย**: ดีบักการปรับใช้ที่ถูกทำให้เสียอย่างตั้งใจได้สำเร็จ
- [ ] **วิเคราะห์บันทึก**: ใช้ Azure Monitor และ Application Insights อย่างมีประสิทธิภาพ
- [ ] **ปรับแต่งประสิทธิภาพ**: ปรับปรุงแอปที่ทำงานช้า
- [ ] **ขั้นตอนกู้คืน**: ดำเนินการสำรองข้อมูลและกู้คืนภัยพิบัติ
- [ ] **ติดตั้งการตรวจสอบ**: สร้างระบบตรวจสอบและแจ้งเตือนเชิงรุก

#### 🏢 บทที่ 8: การผลิตและรูปแบบองค์กร
- [ ] **ความปลอดภัยองค์กร**: ดำเนินการรูปแบบความปลอดภัยครบวงจร
- [ ] **กรอบกำกับดูแล**: ตั้งค่า Azure Policy และการจัดการทรัพยากร
- [ ] **การตรวจสอบขั้นสูง**: สร้างแดชบอร์ดและแจ้งเตือนอัตโนมัติ
- [ ] **รวม CI/CD**: สร้างท่อการปรับใช้อัตโนมัติ
- [ ] **ดำเนินการตามข้อกำหนด**: ปฏิบัติตามข้อกำหนดขององค์กร

### กำหนดเวลาการเรียนรู้และเป้าหมายสำคัญ

#### สัปดาห์ที่ 1-2: สร้างพื้นฐาน
- **เป้าหมาย**: ปรับใช้แอป AI แรกโดยใช้ AZD
- **การตรวจสอบ**: แอปทำงานเข้าถึงได้ผ่าน URL สาธารณะ
- **ทักษะ**: เวิร์กโฟลว์ AZD พื้นฐานและการบูรณาการบริการ AI

#### สัปดาห์ที่ 3-4: เชี่ยวชาญการกำหนดค่า
- **เป้าหมาย**: ปรับใช้หลายสภาพแวดล้อมพร้อมระบบพิสูจน์ตัวตนที่ปลอดภัย
- **การตรวจสอบ**: แอปเดียวกันปรับใช้ใน dev/staging/prod
- **ทักษะ**: การจัดการสภาพแวดล้อมและการติดตั้งความปลอดภัย

#### สัปดาห์ที่ 5-6: ความเชี่ยวชาญโครงสร้างพื้นฐาน
- **เป้าหมาย**: สร้างต้นแบบกำหนดเองสำหรับแอปหลายบริการซับซ้อน
- **การตรวจสอบ**: ต้นแบบนำกลับมาใช้ใหม่และปรับใช้โดยสมาชิกทีมอื่น
- **ทักษะ**: เชี่ยวชาญ Bicep และอัตโนมัติโครงสร้างพื้นฐาน

#### สัปดาห์ที่ 7-8: การใช้งาน AI ขั้นสูง
- **เป้าหมาย**: โซลูชัน AI หลายเอเจนต์พร้อมผลิต
- **การตรวจสอบ**: ระบบรองรับโหลดจริงพร้อมระบบตรวจสอบ
- **ทักษะ**: การจัดการหลายเอเจนต์และการเพิ่มประสิทธิภาพประสิทธิภาพ

#### สัปดาห์ที่ 9-10: พร้อมสู่การใช้งานจริง
- **เป้าหมาย**: ปรับใช้ระดับองค์กรพร้อมการปฏิบัติตามครบถ้วน
- **การตรวจสอบ**: ผ่านการตรวจสอบความปลอดภัยและการตรวจสอบค่าใช้จ่าย
- **ทักษะ**: การกำกับดูแล การตรวจสอบ และการบูรณาการ CI/CD

### การประเมินและการรับรอง

#### วิธีการตรวจสอบความรู้
1. **การปรับใช้จริง**: แอปที่ใช้งานได้สำหรับแต่ละบท
2. **การตรวจสอบโค้ด**: ประเมินคุณภาพต้นแบบและการกำหนดค่า
3. **การแก้ปัญหา**: กรณีศึกษาการแก้ไขปัญหาและโซลูชัน
4. **การสอนเพื่อน**: อธิบายแนวคิดให้ผู้อื่นฟัง
5. **การมีส่วนร่วมกับชุมชน**: แชร์ต้นแบบหรือการปรับปรุง
- **โครงการพอร์ตโฟลิโอ**: การปรับใช้ที่พร้อมใช้งานจริง 8 โครงการ  
- **ทักษะทางเทคนิค**: ความเชี่ยวชาญด้าน AZD และการปรับใช้ AI ตามมาตรฐานอุตสาหกรรม  
- **ความสามารถในการแก้ปัญหา**: การแก้ไขปัญหาและปรับแต่งอย่างอิสระ  
- **การรับรองจากชุมชน**: การมีส่วนร่วมอย่างแข็งขันในชุมชนนักพัฒนา Azure  
- **ความก้าวหน้าในอาชีพ**: ทักษะที่นำไปใช้ได้โดยตรงในบทบาทคลาวด์และ AI  

#### ตัวชี้วัดความสำเร็จ  
- **อัตราความสำเร็จในการปรับใช้**: ปรับใช้สำเร็จ >95%  
- **เวลาการแก้ไขปัญหา**: น้อยกว่า 30 นาทีสำหรับปัญหาทั่วไป  
- **การปรับปรุงประสิทธิภาพ**: การแสดงให้เห็นถึงการปรับปรุงต้นทุนและประสิทธิภาพ  
- **การปฏิบัติตามมาตรฐานความปลอดภัย**: การปรับใช้ทั้งหมดเป็นไปตามมาตรฐานความปลอดภัยขององค์กร  
- **การถ่ายทอดความรู้**: ความสามารถในการให้คำปรึกษานักพัฒนาคนอื่น  

### การเรียนรู้อย่างต่อเนื่องและการมีส่วนร่วมในชุมชน  

#### ยังคงทันสมัย  
- **ข่าวสาร Azure**: ติดตามบันทึกการอัปเดต Azure Developer CLI  
- **กิจกรรมชุมชน**: เข้าร่วมกิจกรรมนักพัฒนา Azure และ AI  
- **เอกสารประกอบ**: มีส่วนร่วมในการจัดทำเอกสารและตัวอย่างของชุมชน  
- **วงจรข้อเสนอแนะ**: ให้ข้อเสนอแนะเกี่ยวกับเนื้อหาหลักสูตรและบริการของ Azure  

#### การพัฒนาอาชีพ  
- **เครือข่ายมืออาชีพ**: ติดต่อกับผู้เชี่ยวชาญด้าน Azure และ AI  
- **โอกาสการพูด**: นำเสนอความรู้ในการประชุมหรืองานพบปะ  
- **การมีส่วนร่วมในโอเพนซอร์ซ**: มีส่วนร่วมในแม่แบบและเครื่องมือ AZD  
- **การให้คำปรึกษา**: ช่วยแนะนำผู้พัฒนาอื่นในเส้นทางการเรียนรู้ AZD
