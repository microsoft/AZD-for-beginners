## บทนำ

พจนานุกรมคำศัพท์ฉบับสมบูรณ์นี้ให้คำจำกัดความของคำศัพท์ แนวคิด และตัวย่อที่ใช้ใน Azure Developer CLI และการพัฒนา Azure cloud เป็นเอกสารอ้างอิงสำคัญสำหรับการทำความเข้าใจเอกสารทางเทคนิค การแก้ไขปัญหา และการสื่อสารอย่างมีประสิทธิภาพเกี่ยวกับโปรเจค azd และบริการ Azure

## เป้าหมายการเรียนรู้

โดยการใช้พจนานุกรมคำศัพท์นี้ คุณจะสามารถ:
- เข้าใจคำศัพท์และแนวคิดพื้นฐานของ Azure Developer CLI
- เชี่ยวชาญคำศัพท์และคำเทคนิคในการพัฒนา Azure cloud
- อ้างอิงคำศัพท์โครงสร้างพื้นฐานเป็นโค้ดและการตั้งค่าใช้งานอย่างมีประสิทธิภาพ
- เข้าใจชื่อบริการ Azure, ตัวย่อ และวัตถุประสงค์ของพวกมัน
- เข้าถึงคำจำกัดความสำหรับการแก้ไขปัญหาและคำศัพท์การดีบัก
- เรียนรู้แนวคิดด้านสถาปัตยกรรมและการพัฒนา Azure ขั้นสูง

## ผลลัพธ์การเรียนรู้

โดยการอ้างอิงพจนานุกรมคำศัพท์นี้เป็นประจำ คุณจะสามารถ:
- สื่อสารอย่างมีประสิทธิภาพโดยใช้คำศัพท์ที่ถูกต้องของ Azure Developer CLI
- เข้าใจเอกสารทางเทคนิคและข้อความข้อผิดพลาดได้ชัดเจนยิ่งขึ้น
- นำทางบริการและแนวคิดของ Azure ด้วยความมั่นใจ
- แก้ไขปัญหาโดยใช้คำศัพท์ทางเทคนิคที่เหมาะสม
- มีส่วนร่วมในการอภิปรายทีมด้วยภาษาเทคนิคที่ถูกต้องแม่นยำ
- ขยายความรู้ด้านการพัฒนา Azure cloud อย่างเป็นระบบ

## A

**ARM Template**  
เทมเพลต Azure Resource Manager รูปแบบ Infrastructure as Code ที่ใช้ JSON สำหรับกำหนดและปรับใช้ทรัพยากร Azure อย่างระบุเงื่อนไข

**App Service**  
แพลตฟอร์มแบบบริการ (PaaS) ของ Azure สำหรับโฮสต์เว็บแอปพลิเคชัน, REST API และ backend มือถือโดยไม่ต้องจัดการโครงสร้างพื้นฐาน

**Application Insights**  
บริการตรวจสอบประสิทธิภาพแอปของ Azure (APM) ที่ให้ข้อมูลเชิงลึกลึกเกี่ยวกับประสิทธิภาพ, ความพร้อมใช้งาน และการใช้งานของแอป

**Azure CLI**  
อินเทอร์เฟซบรรทัดคำสั่งสำหรับจัดการทรัพยากร Azure ใช้โดย azd สำหรับการยืนยันตัวตนและการดำเนินการบางอย่าง

**Azure Developer CLI (azd)**  
เครื่องมือบรรทัดคำสั่งที่มุ่งเน้นนักพัฒนา ช่วยเร่งกระบวนการสร้างและปรับใช้แอปพลิเคชันไปยัง Azure โดยใช้เทมเพลตและ Infrastructure as Code

**azure.yaml**  
ไฟล์คอนฟิกหลักสำหรับโปรเจค azd ที่กำหนดบริการ โครงสร้างพื้นฐาน และ hooks การปรับใช้

**Azure Resource Manager (ARM)**  
บริการปรับใช้และจัดการของ Azure ที่ให้เลเยอร์การจัดการสำหรับการสร้าง อัปเดต และลบทรัพยากร

## B

**Bicep**  
ภาษาเฉพาะทาง (DSL) ที่พัฒนาโดย Microsoft สำหรับการปรับใช้ทรัพยากร Azure มีไวยากรณ์ที่ง่ายกว่า ARM templates ในขณะที่แปลงเป็น ARM

**Build**  
กระบวนการคอมไพล์โค้ดต้นฉบับ ติดตั้งการพึ่งพา และเตรียมแอปพลิเคชันสำหรับการปรับใช้

**Blue-Green Deployment**  
กลยุทธ์การปรับใช้ที่ใช้สองสภาพแวดล้อมผลิตภัณฑ์ที่เหมือนกัน (สีน้ำเงินและสีเขียว) เพื่อลดเวลาหยุดทำงานและความเสี่ยง

## C

**Container Apps**  
บริการคอนเทนเนอร์แบบไร้เซิร์ฟเวอร์ของ Azure ที่ช่วยให้รันแอปพลิเคชันที่บรรจุในคอนเทนเนอร์โดยไม่ต้องจัดการโครงสร้างพื้นฐานที่ซับซ้อน

**CI/CD**  
การบูรณาการอย่างต่อเนื่อง/การปรับใช้อย่างต่อเนื่อง แนวทางอัตโนมัติสำหรับการรวมการเปลี่ยนแปลงโค้ดและปรับใช้แอปพลิเคชัน

**Cosmos DB**  
บริการฐานข้อมูลหลายรูปแบบทั่วโลกของ Azure ที่ให้ SLA ครอบคลุมสำหรับอัตราการทำงาน, เวลาแฝง, ความพร้อมใช้งาน และความสอดคล้อง

**Configuration**  
การตั้งค่าและพารามิเตอร์ที่ควบคุมพฤติกรรมแอปพลิเคชันและตัวเลือกการปรับใช้

## D

**Deployment**  
กระบวนการติดตั้งและกำหนดค่าแอปพลิเคชันและการพึ่งพาของพวกมันบนโครงสร้างพื้นฐานเป้าหมาย

**Docker**  
แพลตฟอร์มสำหรับพัฒนา ส่ง และรันแอปพลิเคชันโดยใช้เทคโนโลยีคอนเทนเนอร์

**Dockerfile**  
ไฟล์ข้อความที่มีคำสั่งสำหรับสร้างภาพคอนเทนเนอร์ Docker

## E

**Environment**  
เป้าหมายการปรับใช้ที่แสดงถึงอินสแตนซ์เฉพาะของแอปพลิเคชันของคุณ (เช่น การพัฒนา, สเตจ, การผลิต)

**Environment Variables**  
ค่าการตั้งค่าที่เก็บในรูปแบบคู่กุญแจ-ค่า ซึ่งแอปพลิเคชันสามารถเข้าถึงได้ในขณะรันไทม์

**Endpoint**  
URL หรือที่อยู่เครือข่ายที่สามารถเข้าถึงแอปพลิเคชันหรือบริการได้

## F

**Function App**  
บริการคำนวณแบบไร้เซิร์ฟเวอร์ของ Azure ที่ช่วยให้รันโค้ดขับเคลื่อนเหตุการณ์โดยไม่ต้องจัดการโครงสร้างพื้นฐาน

## G

**GitHub Actions**  
แพลตฟอร์ม CI/CD ที่ผสานรวมกับที่เก็บข้อมูล GitHub สำหรับอัตโนมัติ workflow

**Git**  
ระบบควบคุมเวอร์ชันแบบกระจายที่ใช้ติดตามการเปลี่ยนแปลงในซอร์สโค้ด

## H

**Hooks**  
สคริปต์หรือคำสั่งที่กำหนดเองซึ่งรันในจุดเวลาที่กำหนดในช่วงวงจรการปรับใช้ (preprovision, postprovision, predeploy, postdeploy)

**Host**  
ประเภทบริการ Azure ที่แอปจะถูกปรับใช้ (เช่น appservice, containerapp, function)

## I

**Infrastructure as Code (IaC)**  
แนวปฏิบัติในการกำหนดและจัดการโครงสร้างพื้นฐานผ่านโค้ดแทนกระบวนการด้วยมือ

**Init**  
กระบวนการเริ่มต้นโปรเจค azd ใหม่ มักเริ่มจากเทมเพลต

## J

**JSON**  
JavaScript Object Notation รูปแบบการแลกเปลี่ยนข้อมูลที่ใช้กันทั่วไปสำหรับไฟล์คอนฟิกและการตอบสนอง API

**JWT**  
JSON Web Token มาตรฐานสำหรับการส่งข้อมูลอย่างปลอดภัยระหว่างฝ่ายต่าง ๆ ในรูปแบบ JSON

## K

**Key Vault**  
บริการของ Azure สำหรับจัดเก็บและจัดการความลับ, คีย์, และใบรับรองอย่างปลอดภัย

**Kusto Query Language (KQL)**  
ภาษาคิวรีที่ใช้วิเคราะห์ข้อมูลใน Azure Monitor, Application Insights และบริการ Azure อื่น ๆ

## L

**Load Balancer**  
บริการที่กระจายทราฟฟิกเครือข่ายขาเข้าข้ามเซิร์ฟเวอร์หรืออินสแตนซ์หลายตัว

**Log Analytics**  
บริการของ Azure สำหรับรวบรวม วิเคราะห์ และดำเนินการบนข้อมูลเทเลเมทรีจากสภาพแวดล้อมคลาวด์และบนที่ตั้ง

## M

**Managed Identity**  
ฟีเจอร์ของ Azure ที่จัดการตัวตนอัตโนมัติให้บริการ Azure สำหรับการยืนยันตัวตนไปยังบริการ Azure อื่น ๆ

**Microservices**  
แนวทางสถาปัตยกรรมที่สร้างแอปพลิเคชันเป็นชุดของบริการขนาดเล็กและอิสระ

**Monitor**  
โซลูชันการตรวจสอบแบบรวมของ Azure ที่มอบการสังเกตการณ์เต็มรูปแบบทั้งในแอปและโครงสร้างพื้นฐาน

## N

**Node.js**  
รันไทม์ JavaScript ที่สร้างบนเครื่องมือ V8 ของ Chrome สำหรับสร้างแอปเซิร์ฟเวอร์

**npm**  
ตัวจัดการแพ็กเกจสำหรับ Node.js ที่จัดการการพึ่งพาและแพ็กเกจต่าง ๆ

## O

**Output**  
ค่าที่ส่งกลับจากการปรับใช้โครงสร้างพื้นฐานซึ่งสามารถใช้โดยแอปพลิเคชันหรือทรัพยากรอื่น ๆ

## P

**Package**  
กระบวนการเตรียมโค้ดแอปและการพึ่งพาสำหรับการปรับใช้

**Parameters**  
ค่าป้อนข้อมูลที่ส่งให้กับเทมเพลตโครงสร้างพื้นฐานเพื่อปรับแต่งการปรับใช้

**PostgreSQL**  
ระบบจัดการฐานข้อมูลเชิงสัมพันธ์แบบโอเพนซอร์สที่สนับสนุนในรูปแบบบริการจัดการบน Azure

**Provisioning**  
กระบวนการสร้างและกำหนดค่าทรัพยากร Azure ที่กำหนดไว้ในเทมเพลตโครงสร้างพื้นฐาน

## Q

**Quota**  
ข้อจำกัดเกี่ยวกับปริมาณทรัพยากรที่สามารถสร้างได้ในบัญชี Azure หรือตามภูมิภาค

## R

**Resource Group**  
ภาชนะตรรกะสำหรับทรัพยากร Azure ที่มีวงจรชีวิต สิทธิ์ และนโยบายร่วมกัน

**Resource Token**  
สตริงเฉพาะที่ azd สร้างขึ้นเพื่อให้แน่ใจว่าชื่อทรัพยากรไม่ซ้ำกันในแต่ละครั้งที่ปรับใช้

**REST API**  
รูปแบบสถาปัตยกรรมสำหรับการออกแบบแอปพลิเคชันเครือข่ายโดยใช้เมธอด HTTP

**Rollback**  
กระบวนการย้อนกลับเป็นเวอร์ชันก่อนหน้าของแอปพลิเคชันหรือการตั้งค่าโครงสร้างพื้นฐาน

## S

**Service**  
องค์ประกอบของแอปพลิเคชันในไฟล์ azure.yaml (เช่น เว็บฟรอนต์เอนด์, API แบ็กเอนด์, ฐานข้อมูล)

**SKU**  
หน่วยเก็บสินค้า แทนระดับบริการหรืองานประสิทธิภาพที่แตกต่างกันของทรัพยากร Azure

**SQL Database**  
บริการฐานข้อมูลเชิงสัมพันธ์ที่จัดการโดย Azure อิงตาม Microsoft SQL Server

**Static Web Apps**  
บริการของ Azure สำหรับสร้างและปรับใช้เว็บแอปแบบเต็มสแตกจากที่เก็บซอร์สโค้ด

**Storage Account**  
บริการ Azure ที่ให้พื้นที่เก็บข้อมูลคลาวด์สำหรับวัตถุข้อมูล เช่น บล็อบ, ไฟล์, คิว, และตาราง

**Subscription**  
ภาชนะบัญชี Azure ที่เก็บ Resource Group และทรัพยากร พร้อมการจัดการบิลและสิทธิ์เข้าถึง

## T

**Template**  
โครงสร้างโปรเจคที่สร้างไว้ล่วงหน้าซึ่งประกอบด้วยโค้ดแอป นิยามโครงสร้างพื้นฐาน และการตั้งค่าสำหรับสถานการณ์ทั่วไป

**Terraform**  
เครื่องมือ Infrastructure as Code แบบโอเพนซอร์สที่รองรับหลายผู้ให้บริการคลาวด์รวมถึง Azure

**Traffic Manager**  
ตัวจัดการโหลดทราฟฟิกแบบ DNS ของ Azure สำหรับกระจายทราฟฟิกในหลายภูมิภาค Azure ทั่วโลก

## U

**URI**  
Uniform Resource Identifier สตริงที่ระบุทรัพยากรเฉพาะเจาะจง

**URL**  
Uniform Resource Locator ประเภทของ URI ที่ระบุที่ตั้งของทรัพยากรและวิธีเข้าถึง

## V

**Virtual Network (VNet)**  
บล็อคพื้นฐานสำหรับเครือข่ายส่วนตัวใน Azure ที่มอบการแยกและแบ่งส่วน

**VS Code**  
Visual Studio Code ตัวแก้ไขโค้ดยอดนิยมที่มีการผสานรวม Azure และ azd อย่างยอดเยี่ยม

## W

**Webhook**  
HTTP callback ที่กระตุ้นโดยเหตุการณ์เฉพาะ ใช้กันทั่วไปใน pipeline CI/CD

**What-if**  
ฟีเจอร์ของ Azure ที่แสดงว่าการเปลี่ยนแปลงใดจะเกิดขึ้นจากการปรับใช้โดยไม่ต้องดำเนินการจริง

## Y

**YAML**  
YAML Ain't Markup Language มาตรฐานการซีเรียลไลซ์ข้อมูลที่อ่านได้โดยมนุษย์ ใช้สำหรับไฟล์คอนฟิกเช่น azure.yaml

## Z

**Zone**  
ที่ตั้งแยกทางกายภาพภายในภูมิภาค Azure ที่ให้ความพร้อมใช้งานสูงและระบบสำรองข้อมูล

---

## ตัวย่อที่ใช้บ่อย

| ตัวย่อ | คำเต็ม | คำอธิบาย |
|---------|-----------|-------------|
| AAD | Azure Active Directory (ปัจจุบัน Microsoft Entra ID) | บริการจัดการตัวตนและการเข้าถึง |
| ACR | Azure Container Registry | บริการทะเบียนภาพคอนเทนเนอร์ |
| AKS | Azure Kubernetes Service | บริการ Kubernetes ที่จัดการแล้ว |
| API | Application Programming Interface | ชุดโปรโตคอลสำหรับสร้างซอฟต์แวร์ |

## คำศัพท์เฉพาะบริบท

### คำศัพท์ด้านการพัฒนา
- **Hot Reload**: อัปเดตแอปโดยอัตโนมัติในระหว่างการพัฒนาโดยไม่ต้องรีสตาร์ท
- **Build Pipeline**: กระบวนการอัตโนมัติสำหรับสร้างและทดสอบโค้ด
- **Deployment Slot**: สภาพแวดล้อมสเตจใน App Service
- **Environment Parity**: การรักษาสภาพแวดล้อมพัฒนา, สเตจ และผลิตให้เหมือนกัน

### คำศัพท์ด้านความปลอดภัย
- **Managed Identity**: ฟีเจอร์ Azure ที่จัดการข้อมูลประจำตัวโดยอัตโนมัติ
- **Key Vault**: พื้นที่จัดเก็บความลับ, คีย์ และใบรับรองแบบปลอดภัย
- **RBAC**: การควบคุมการเข้าถึงตามบทบาทสำหรับทรัพยากร Azure
- **Network Security Group**: ไฟร์วอลล์เสมือนสำหรับควบคุมทราฟฟิกเครือข่าย

### คำศัพท์ด้านการตรวจสอบ
- **Telemetry**: การเก็บรวบรวมการวัดและข้อมูลโดยอัตโนมัติ
- **Application Performance Monitoring (APM)**: การตรวจสอบประสิทธิภาพซอฟต์แวร์
- **Log Analytics**: บริการสำหรับรวบรวมและวิเคราะห์ข้อมูลบันทึก
- **Alert Rules**: การแจ้งเตือนอัตโนมัติอิงตามเมตริกหรือเงื่อนไข

### คำศัพท์ด้านการปรับใช้
- **Blue-Green Deployment**: กลยุทธ์ปรับใช้แบบไม่มีเวลาหยุดทำงาน
- **Canary Deployment**: การเปิดตัวแบบค่อยเป็นค่อยไปกับกลุ่มผู้ใช้บางส่วน
- **Rolling Update**: การแทนที่อินสแตนซ์แอปทีละตัวแบบเรียงลำดับ
- **Rollback**: การย้อนกลับเป็นเวอร์ชันก่อนหน้าของแอปพลิเคชัน

---
