## บทนำ

คำศัพท์ฉบับสมบูรณ์นี้ให้คำจำกัดความสำหรับคำศัพท์ แนวคิด และตัวย่อที่ใช้ใน Azure Developer CLI และการพัฒนา Azure cloud เป็นคู่มืออ้างอิงสำคัญสำหรับการเข้าใจเอกสารทางเทคนิค การแก้ไขปัญหา และการสื่อสารอย่างมีประสิทธิภาพเกี่ยวกับโครงการ azd และบริการ Azure

## เป้าหมายการเรียนรู้

โดยใช้คำศัพท์นี้ คุณจะสามารถ:
- เข้าใจคำศัพท์และแนวคิดพื้นฐานของ Azure Developer CLI
- เชี่ยวชาญคำศัพท์และคำทางเทคนิคของการพัฒนา Azure cloud
- อ้างอิงคำศัพท์เกี่ยวกับ Infrastructure as Code และการปรับใช้ได้อย่างมีประสิทธิภาพ
- เข้าใจชื่อบริการ Azure ตัวย่อ และวัตถุประสงค์ของแต่ละบริการ
- เข้าถึงคำจำกัดความสำหรับการแก้ไขปัญหาและการดีบัก
- เรียนรู้แนวคิดชั้นสูงด้านสถาปัตยกรรมและการพัฒนา Azure

## ผลลัพธ์การเรียนรู้

ด้วยการอ้างอิงคำศัพท์นี้เป็นประจำ คุณจะสามารถ:
- สื่อสารอย่างมีประสิทธิภาพโดยใช้คำศัพท์ Azure Developer CLI ที่ถูกต้อง
- เข้าใจเอกสารทางเทคนิคและข้อความแสดงข้อผิดพลาดได้อย่างชัดเจนยิ่งขึ้น
- นำทางบริการและแนวคิดของ Azure ได้อย่างมั่นใจ
- แก้ไขปัญหาโดยใช้คำศัพท์ทางเทคนิคที่เหมาะสม
- มีส่วนร่วมในการอภิปรายของทีมด้วยภาษาทางเทคนิคที่ถูกต้อง
- ขยายความรู้ด้านการพัฒนา Azure cloud อย่างเป็นระบบ

## A

**ARM Template**  
เทมเพลต Azure Resource Manager เป็นรูปแบบ Infrastructure as Code บนพื้นฐาน JSON ที่ใช้กำหนดและปรับใช้ทรัพยากร Azure อย่างเป็นทางการ

**App Service**  
แพลตฟอร์มแบบบริการ (PaaS) ของ Azure สำหรับโฮสต์เว็บแอปพลิเคชัน, REST APIs และแบ็กเอนด์มือถือโดยไม่ต้องจัดการโครงสร้างพื้นฐาน

**Application Insights**  
บริการการตรวจสอบประสิทธิภาพแอปพลิเคชัน (APM) ของ Azure ที่ให้ข้อมูลเชิงลึกลึกซึ้งเกี่ยวกับประสิทธิภาพ, ความพร้อมใช้งาน และการใช้งานแอปพลิเคชัน

**Azure CLI**  
อินเทอร์เฟซบรรทัดคำสั่งสำหรับจัดการทรัพยากร Azure ใช้โดย azd สำหรับการรับรองความถูกต้องและบางการดำเนินการ

**Azure Developer CLI (azd)**  
เครื่องมือบรรทัดคำสั่งสำหรับนักพัฒนาที่เร่งกระบวนการสร้างและปรับใช้แอปพลิเคชันสู่ Azure โดยใช้เทมเพลตและ Infrastructure as Code

**azure.yaml**  
ไฟล์กำหนดค่าหลักสำหรับโครงการ azd ที่กำหนดบริการ, โครงสร้างพื้นฐาน และการตั้งค่าการปรับใช้

**Azure Resource Manager (ARM)**  
บริการปรับใช้และจัดการของ Azure ที่มีชั้นการจัดการสำหรับการสร้าง, อัปเดต และลบทรัพยากร

## B

**Bicep**  
ภาษาเฉพาะโดเมน (DSL) ที่พัฒนาโดย Microsoft สำหรับการปรับใช้ทรัพยากร Azure มีไวยากรณ์ที่เรียบง่ายกว่าเทมเพลต ARM ขณะคอมไพล์เป็น ARM

**Build**  
กระบวนการคอมไพล์ซอร์สโค้ด, ติดตั้ง dependencies และเตรียมแอปพลิเคชันสำหรับการปรับใช้

**Blue-Green Deployment**  
กลยุทธ์การปรับใช้ที่ใช้สภาพแวดล้อมการผลิตสองชุดที่เหมือนกัน (สีน้ำเงินและสีเขียว) เพื่อลดเวลาหยุดทำงานและความเสี่ยง

## C

**Container Apps**  
บริการคอนเทนเนอร์แบบไร้เซิร์ฟเวอร์ของ Azure ที่ช่วยให้รันแอปพลิเคชันที่บรรจุในคอนเทนเนอร์โดยไม่ต้องจัดการโครงสร้างพื้นฐานที่ซับซ้อน

**CI/CD**  
Continuous Integration/Continuous Deployment แนวปฏิบัติอัตโนมัติสำหรับการผนวกการเปลี่ยนแปลงโค้ดและปรับใช้แอปพลิเคชัน

**Cosmos DB**  
บริการฐานข้อมูลหลายรุ่นที่กระจายทั่วโลกของ Azure พร้อม SLA ครอบคลุมด้าน throughput, latency, ความพร้อมใช้งาน และความสอดคล้อง

**Configuration**  
การตั้งค่าและพารามิเตอร์ที่ควบคุมพฤติกรรมของแอปพลิเคชันและตัวเลือกการปรับใช้

## D

**Deployment**  
กระบวนการติดตั้งและกำหนดค่าแอปพลิเคชันและ dependencies บนโครงสร้างพื้นฐานเป้าหมาย

**Docker**  
แพลตฟอร์มสำหรับพัฒนา, ส่งมอบ และรันแอปพลิเคชันโดยใช้เทคโนโลยีคอนเทนเนอร์

**Dockerfile**  
ไฟล์ข้อความที่บรรจุคำสั่งสำหรับสร้างอิมเมจคอนเทนเนอร์ Docker

## E

**Environment**  
เป้าหมายการปรับใช้ที่แสดงถึงอินสแตนซ์เฉพาะของแอปพลิเคชันของคุณ (เช่น การพัฒนา, สเตจจิง, การผลิต)

**Environment Variables**  
ค่าการกำหนดค่าที่เก็บเป็นคู่คีย์-ค่า ที่แอปพลิเคชันสามารถเข้าถึงได้ตอนรันไทม์

**Endpoint**  
URL หรือที่อยู่เครือข่ายที่สามารถเข้าถึงแอปพลิเคชันหรือบริการได้

## F

**Function App**  
บริการประมวลผลแบบไร้เซิร์ฟเวอร์ของ Azure ที่ช่วยให้รันโค้ดแบบ event-driven โดยไม่ต้องจัดการโครงสร้างพื้นฐาน

## G

**GitHub Actions**  
แพลตฟอร์ม CI/CD ที่ผนวกรวมกับที่เก็บ GitHub สำหรับอัตโนมัติการทำงาน

**Git**  
ระบบควบคุมเวอร์ชันแบบกระจายที่ใช้ติดตามการเปลี่ยนแปลงในซอร์สโค้ด

## H

**Hooks**  
สคริปต์หรือคำสั่งกำหนดเองที่รันในจุดต่างๆ ในวงจรชีวิตของการปรับใช้ (ก่อนจัดเตรียม, หลังจัดเตรียม, ก่อนปรับใช้, หลังปรับใช้)

**Host**  
ประเภทบริการ Azure ที่แอปพลิเคชันจะปรับใช้ (เช่น appservice, containerapp, function)

## I

**Infrastructure as Code (IaC)**  
แนวปฏิบัติในการกำหนดและจัดการโครงสร้างพื้นฐานผ่านโค้ดแทนกระบวนการด้วยมือ

**Init**  
กระบวนการเริ่มต้นโครงการ azd ใหม่ โดยทั่วไปจากเทมเพลต

## J

**JSON**  
JavaScript Object Notation รูปแบบแลกเปลี่ยนข้อมูลที่ใช้บ่อยสำหรับไฟล์กำหนดค่าและการตอบสนอง API

**JWT**  
JSON Web Token มาตรฐานสำหรับการส่งข้อมูลอย่างปลอดภัยระหว่างฝ่ายต่างๆ ในรูปแบบอ็อบเจ็กต์ JSON

## K

**Key Vault**  
บริการของ Azure สำหรับจัดเก็บและจัดการความลับ, กุญแจ และใบรับรองอย่างปลอดภัย

**Kusto Query Language (KQL)**  
ภาษาคิวรีที่ใช้วิเคราะห์ข้อมูลใน Azure Monitor, Application Insights และบริการ Azure อื่นๆ

## L

**Load Balancer**  
บริการที่กระจายทราฟฟิกเครือข่ายที่เข้ามาไปยังเซิร์ฟเวอร์หรืออินสแตนซ์หลายๆ ตัว

**Log Analytics**  
บริการของ Azure สำหรับรวบรวม วิเคราะห์ และดำเนินการข้อมูลเทเลเมทรีจากสภาพแวดล้อมคลาวด์และภายในองค์กร

## M

**Managed Identity**  
ฟีเจอร์ของ Azure ที่ให้บริการ Azure มีตัวตนที่จัดการโดยอัตโนมัติสำหรับการรับรองความถูกต้องกับบริการ Azure อื่นๆ

**Microservices**  
แนวทางสถาปัตยกรรมที่สร้างแอปพลิเคชันเป็นชุดของบริการขนาดเล็กและอิสระ

**Monitor**  
โซลูชันตรวจสอบแบบครบวงจรของ Azure ที่ให้การสังเกตการณ์เต็มรูปแบบทั้งแอปพลิเคชันและโครงสร้างพื้นฐาน

## N

**Node.js**  
รันไทม์ JavaScript สร้างบนเครื่องยนต์ V8 ของ Chrome สำหรับสร้างแอปพลิเคชันฝั่งเซิร์ฟเวอร์

**npm**  
ตัวจัดการแพ็กเกจสำหรับ Node.js ที่จัดการ dependencies และแพ็กเกจต่างๆ

## O

**Output**  
ค่าที่คืนมาจากการปรับใช้โครงสร้างพื้นฐานซึ่งสามารถใช้โดยแอปพลิเคชันหรือทรัพยากรอื่นๆ

## P

**Package**  
กระบวนการเตรียมโค้ดแอปพลิเคชันและ dependencies สำหรับการปรับใช้

**Parameters**  
ค่าป้อนเข้าที่ส่งไปยังเทมเพลตโครงสร้างพื้นฐานเพื่อปรับแต่งการปรับใช้

**PostgreSQL**  
ระบบฐานข้อมูลเชิงสัมพันธ์โอเพนซอร์สที่ให้บริการแบบจัดการภายใต้ Azure

**Provisioning**  
กระบวนการสร้างและกำหนดค่าทรัพยากร Azure ที่กำหนดในเทมเพลตโครงสร้างพื้นฐาน

## Q

**Quota**  
ข้อจำกัดของจำนวนทรัพยากรที่สามารถสร้างได้ในบัญชีสมัครใช้ Azure หรือภูมิภาค

## R

**Resource Group**  
ภาชนะตรรกะสำหรับทรัพยากร Azure ที่มีวงจรชีวิต, สิทธิ์ และนโยบายร่วมกัน

**Resource Token**  
สตริงเฉพาะที่สร้างโดย azd เพื่อให้แน่ใจว่าชื่อทรัพยากรไม่ซ้ำกันข้ามการปรับใช้

**REST API**  
รูปแบบสถาปัตยกรรมสำหรับออกแบบแอปพลิเคชันที่เชื่อมต่อเครือข่ายโดยใช้วิธี HTTP

**Rollback**  
กระบวนการย้อนกลับไปยังเวอร์ชันก่อนหน้าของแอปพลิเคชันหรือการกำหนดค่าโครงสร้างพื้นฐาน

## S

**Service**  
ส่วนประกอบของแอปพลิเคชันที่กำหนดใน azure.yaml (เช่น เว็บ frontend, API backend, ฐานข้อมูล)

**SKU**  
หน่วยเก็บสต็อกที่แสดงระดับชั้นของบริการหรือประสิทธิภาพสำหรับทรัพยากร Azure

**SQL Database**  
บริการฐานข้อมูลเชิงสัมพันธ์ที่จัดการโดย Azure บนพื้นฐาน Microsoft SQL Server

**Static Web Apps**  
บริการ Azure สำหรับสร้างและปรับใช้เว็บแอปพลิเคชันแบบเต็มรูปแบบจากที่เก็บซอร์สโค้ด

**Storage Account**  
บริการ Azure ที่ให้บริการจัดเก็บข้อมูลบนคลาวด์สำหรับข้อมูลประเภทบล็อบ, ไฟล์, คิว และตาราง

**Subscription**  
ภาชนะบัญชี Azure ที่เก็บกลุ่มทรัพยากรและทรัพยากรพร้อมการจัดการบิลและการเข้าถึง

## T

**Template**  
โครงสร้างโครงการที่สร้างไว้ล่วงหน้าซึ่งบรรจุโค้ดแอปพลิเคชัน, นิยามโครงสร้างพื้นฐาน และการกำหนดค่าสำหรับสถานการณ์ทั่วไป

**Terraform**  
เครื่องมือ Infrastructure as Code แบบโอเพนซอร์สที่รองรับผู้ให้บริการคลาวด์หลากหลายรวมถึง Azure

**Traffic Manager**  
ตัวจัดการทราฟฟิกแบบ DNS ของ Azure สำหรับกระจายการรับส่งข้อมูลข้ามภูมิภาค Azure ทั่วโลก

## U

**URI**  
ตัวระบุทรัพยากรแบบยูนีฟอร์ม สตริงที่ระบุทรัพยากรเฉพาะ

**URL**  
ตัวระบุแหล่งที่อยู่แบบยูนีฟอร์ม ประเภทของ URI ที่ระบุที่อยู่และวิธีการเข้าถึงทรัพยากร

## V

**Virtual Network (VNet)**  
องค์ประกอบพื้นฐานสำหรับเครือข่ายส่วนตัวใน Azure ที่ให้การแยกและแบ่งส่วน

**VS Code**  
Visual Studio Code โปรแกรมแก้ไขโค้ดยอดนิยมพร้อมการผนวกรวม Azure และ azd อย่างยอดเยี่ยม

## W

**Webhook**  
คอลแบ็ค HTTP ที่ถูกทริกเกอร์โดยเหตุการณ์เฉพาะ โดยทั่วไปใช้ในท่อ CI/CD

**What-if**  
ฟีเจอร์ Azure ที่แสดงว่าการปรับใช้จะเปลี่ยนแปลงอะไรโดยไม่ต้องดำเนินการจริง

## Y

**YAML**  
YAML Ain't Markup Language มาตรฐานการจัดเรียงข้อมูลที่อ่านง่ายสำหรับไฟล์กำหนดค่าเช่น azure.yaml

## Z

**Zone**  
ตำแหน่งแยกกันจริงภายในภูมิภาค Azure ที่ให้ความซ้ำซ้อนและความพร้อมใช้งานสูง

## ตัวย่อทั่วไป

| ตัวย่อ | รูปแบบเต็ม | คำอธิบาย |
|---------|-----------|-------------|
| AAD | Azure Active Directory | บริการการจัดการตัวตนและสิทธิ์เข้าถึง |
| ACR | Azure Container Registry | บริการรีจิสทรีอิมเมจคอนเทนเนอร์ |
| AKS | Azure Kubernetes Service | บริการ Kubernetes แบบจัดการ |
| API | Application Programming Interface | ชุดโปรโตคอลสำหรับสร้างซอฟต์แวร์ |

## คำศัพท์ตามบริบทเฉพาะ

### คำศัพท์ด้านการพัฒนา
- **Hot Reload**: การอัปเดตแอปพลิเคชันโดยอัตโนมัติในระหว่างการพัฒนาโดยไม่ต้องรีสตาร์ท
- **Build Pipeline**: กระบวนการอัตโนมัติสำหรับการสร้างและทดสอบโค้ด
- **Deployment Slot**: สภาพแวดล้อมสเตจจิงภายใน App Service
- **Environment Parity**: การรักษาความเหมือนกันระหว่างสภาพแวดล้อมพัฒนา, สเตจจิง และการผลิต

### คำศัพท์ด้านความปลอดภัย
- **Managed Identity**: ฟีเจอร์ Azure ที่ให้การจัดการข้อมูลรับรองอัตโนมัติ
- **Key Vault**: ที่เก็บความลับ, กุญแจ, และใบรับรองอย่างปลอดภัย
- **RBAC**: การควบคุมการเข้าถึงตามบทบาทสำหรับทรัพยากร Azure
- **Network Security Group**: ไฟร์วอลล์เสมือนสำหรับควบคุมทราฟฟิกเครือข่าย

### คำศัพท์ด้านการตรวจสอบ
- **Telemetry**: การรวบรวมข้อมูลและการวัดโดยอัตโนมัติ
- **Application Performance Monitoring (APM)**: การติดตามประสิทธิภาพซอฟต์แวร์
- **Log Analytics**: บริการสำหรับรวบรวมและวิเคราะห์ข้อมูลบันทึก
- **Alert Rules**: การแจ้งเตือนโดยอัตโนมัติบนพื้นฐานเมตริกหรือเงื่อนไข

### คำศัพท์ด้านการปรับใช้
- **Blue-Green Deployment**: กลยุทธ์การปรับใช้ที่ไม่มีเวลาหยุดทำงาน
- **Canary Deployment**: การเปิดตัวแบบค่อยเป็นค่อยไปไปยังกลุ่มผู้ใช้ย่อย
- **Rolling Update**: การทดแทนอินสแตนซ์แอปพลิเคชันอย่างต่อเนื่อง
- **Rollback**: การย้อนกลับไปยังเวอร์ชันแอปพลิเคชันก่อนหน้า
