# การมีส่วนร่วมใน AZD For Beginners

ขอขอบคุณที่สนใจมีส่วนร่วมในหลักสูตร AZD For Beginners! เอกสารนี้ให้แนวทางและข้อมูลสำหรับผู้ที่ต้องการมีส่วนร่วม

## วิธีการมีส่วนร่วม

### การรายงานปัญหา

หากคุณพบข้อบั๊ก ข้อผิดพลาด หรือมีข้อเสนอแนะสำหรับการปรับปรุง:

1. **ตรวจสอบปัญหาที่มีอยู่**: ค้นหา [ปัญหาที่มีอยู่](https://github.com/microsoft/azd-for-beginners/issues) เพื่อหลีกเลี่ยงการซ้ำซ้อน
2. **สร้าง Issue ใหม่**: ใช้เทมเพลต Issue ที่เหมาะสม
3. **ระบุรายละเอียด**: รวมขั้นตอนที่ใช้ทำให้เกิดปัญหา พฤติกรรมที่คาดหวัง และภาพหน้าจอถ้ามี

### การเสนอการเปลี่ยนแปลง

1. **Fork รีโพซิทอรี**: สร้าง fork ของโปรเจกต์ในบัญชีของคุณ
2. **สร้าง Branch**: สร้างสาขาฟีเจอร์สำหรับการเปลี่ยนแปลงของคุณ
3. **ทำการเปลี่ยนแปลง**: ปฏิบัติตามมาตรฐานการเขียนโค้ดและเอกสารของเรา
4. **ทดสอบการเปลี่ยนแปลงของคุณ**: ตรวจสอบว่าลิงก์ทั้งหมดทำงานและเนื้อหาถูกแสดงอย่างถูกต้อง
5. **ส่ง Pull Request**: ให้คำอธิบายที่ชัดเจนเกี่ยวกับการเปลี่ยนแปลงของคุณ

### ประเภทของการมีส่วนร่วม

เรายินดีต้อนรับการมีส่วนร่วมในรูปแบบต่อไปนี้:

#### การปรับปรุงเนื้อหา
- แก้ไขการพิมพ์ผิด ไวยากรณ์ และปัญหาการจัดรูปแบบ
- ปรับปรุงความชัดเจนของเอกสารที่มีอยู่
- อัปเดตข้อมูลที่ล้าสมัย
- เพิ่มคำอธิบายหรือบริบทที่ขาดหายไป

#### ตัวอย่างใหม่
- เพิ่มสถานการณ์การปรับใช้งานในโลกจริง
- สร้างเทมเพลต AZD ใหม่สำหรับกรณีการใช้งานที่พบบ่อย
- จัดเตรียมตัวอย่างโค้ดและแบบฝึกหัดเพิ่มเติม

#### การสนับสนุนการแปล
- ช่วยบำรุงรักษาการแปลที่มีอยู่
- ปรับปรุงคุณภาพการแปล
- เพิ่มการแปลสำหรับเนื้อหาใหม่

#### รายงานบั๊ก
- รายงานลิงก์ที่เสียหรือ ตัวอย่างโค้ดที่ไม่ถูกต้อง
- ระบุความไม่สอดคล้องของเอกสาร
- แจ้งปัญหาการเข้าถึง

## แนวทางการมีส่วนร่วม

### มาตรฐานเอกสาร

- ใช้ภาษาที่ชัดเจนและเป็นมิตรกับผู้เริ่มต้น
- ระบุวัตถุประสงค์การเรียนรู้และผลลัพธ์
- ให้คำแนะนำทีละขั้นตอนพร้อมผลลัพธ์ที่คาดหวัง
- เพิ่มลิงก์นำทางไปยังเนื้อหาที่เกี่ยวข้อง
- ใช้รูปแบบและสไตล์ Markdown ที่สอดคล้องกัน

### ตัวอย่างโค้ด

- ตรวจให้แน่ใจว่าตัวอย่างโค้ดทั้งหมดถูกทดสอบและใช้งานได้
- รวมคอมเมนต์ที่อธิบายแนวคิดสำคัญ
- ปฏิบัติตามแนวปฏิบัติที่ดีที่สุดของ Azure และ AZD
- ให้ผลลัพธ์ที่คาดหวังเมื่อมีความเหมาะสม

### ข้อความ Commit

ใช้ข้อความ commit ที่ชัดเจนและบรรยายได้ดี:
- `docs: fix typo in installation guide`
- `feat: add new container app example`
- `fix: correct broken link in chapter 2`
