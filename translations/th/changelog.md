# บันทึกการเปลี่ยนแปลง - AZD สำหรับผู้เริ่มต้น

## บทนำ

บันทึกการเปลี่ยนแปลงนี้บันทึกการเปลี่ยนแปลงที่สำคัญทั้งหมด การอัปเดต และการปรับปรุงในที่เก็บ AZD สำหรับผู้เริ่มต้น เราปฏิบัติตามหลักการการกำหนดเวอร์ชันแบบมีความหมายและดูแลบันทึกนี้เพื่อช่วยให้ผู้ใช้เข้าใจว่ามีการเปลี่ยนแปลงอะไรบ้างระหว่างเวอร์ชัน

## เป้าหมายการเรียนรู้

โดยการทบทวนบันทึกการเปลี่ยนแปลงนี้ คุณจะสามารถ:
- ติดตามข่าวสารเกี่ยวกับฟีเจอร์ใหม่และการเพิ่มเนื้อหา
- เข้าใจการปรับปรุงที่ทำกับเอกสารที่มีอยู่
- ติดตามการแก้ไขข้อผิดพลาดและการแก้ไขเพื่อความถูกต้อง
- ติดตามวิวัฒนาการของสื่อการเรียนรู้ตลอดเวลา

## ผลลัพธ์การเรียนรู้

หลังจากทบทวนรายการบันทึกการเปลี่ยนแปลง คุณจะสามารถ:
- ระบุเนื้อหาและแหล่งข้อมูลใหม่สำหรับการเรียนรู้
- เข้าใจส่วนที่ได้รับการอัปเดตหรือปรับปรุง
- วางแผนเส้นทางการเรียนรู้ตามสื่อที่เป็นปัจจุบันที่สุด
- มีส่วนร่วมกับความคิดเห็นและข้อเสนอแนะเพื่อการปรับปรุงในอนาคต

### [v3.10.0] - 2026-02-05

#### การแก้ไขความถูกต้องของคำสั่ง Azure Developer CLI
**เวอร์ชันนี้แก้ไขคำสั่ง AZD ที่ไม่มีอยู่ในเอกสารทั้งหมด เพื่อให้ตัวอย่างโค้ดใช้ไวยากรณ์ Azure Developer CLI ที่ถูกต้อง**

#### แก้ไข
- **🔧 ลบคำสั่ง AZD ที่ไม่มีอยู่**: ตรวจสอบและแก้ไขคำสั่งที่ไม่ถูกต้องอย่างครอบคลุม:
  - `azd logs` (ไม่มีอยู่จริง) → แทนที่ด้วย `azd monitor --logs` หรือทางเลือก Azure CLI
  - คำสั่งย่อย `azd service` (ไม่มีอยู่จริง) → แทนที่ด้วย `azd show` และ Azure CLI
  - `azd infra import/export/validate` (ไม่มีอยู่จริง) → ลบหรือแทนที่ด้วยตัวเลือกที่ถูกต้อง
  - ออปชัน `azd deploy --rollback/--incremental/--parallel/--detect-changes` (ไม่มีอยู่จริง) → ลบ
  - ออปชัน `azd provision --what-if/--rollback` (ไม่มีอยู่จริง) → อัปเดตเป็น `--preview`
  - `azd config validate` (ไม่มีอยู่จริง) → แทนที่ด้วย `azd config list`
  - `azd info`, `azd history`, `azd metrics` (ไม่มีอยู่จริง) → ลบออก

  - `docs/troubleshooting/ai-troubleshooting.md`: แก้ไขส่วนดีบัก AZD

### [v3.6.0] - 2025-11-19

#### อัปเดตใหญ่: ตัวอย่างการปรับใช้ Container App
**เวอร์ชันนี้แนะนำตัวอย่างการปรับใช้แอปพลิเคชันคอนเทนเนอร์สมบูรณ์แบบพร้อมใช้งานจริงโดยใช้ Azure Developer CLI (AZD) พร้อมเอกสารครบถ้วนและการผนวกในเส้นทางการเรียนรู้**

#### เพิ่มเติม
- **🚀 ตัวอย่างแอปคอนเทนเนอร์**: ตัวอย่างท้องถิ่นใหม่ที่ `examples/container-app/`:
  - [คู่มือหลัก](examples/container-app/README.md): ภาพรวมครบถ้วนของการปรับใช้แบบคอนเทนเนอร์, เริ่มต้นเร็ว, การผลิต, และรูปแบบขั้นสูง
  - [Simple Flask API](../../examples/container-app/simple-flask-api): REST API สำหรับผู้เริ่มต้นพร้อมฟีเจอร์ scale-to-zero, ตรวจสอบสุขภาพ, การตรวจสอบ, และการแก้ไขปัญหา
  - [สถาปัตยกรรมไมโครเซอร์วิส](../../examples/container-app/microservices): การปรับใช้หลายบริการพร้อมใช้งานจริง (API Gateway, Product, Order, User, Notification), การสื่อสารแบบอะซิงค์, Service Bus, Cosmos DB, Azure SQL, การติดตามแบบกระจาย, การปรับใช้แบบ blue-green/canary
- **แนวปฏิบัติที่ดีที่สุด**: ความปลอดภัย, การตรวจสอบ, การปรับแต่งค่าใช้จ่าย และแนวทาง CI/CD สำหรับภาระงานคอนเทนเนอร์
- **ตัวอย่างโค้ด**: ไฟล์ `azure.yaml` ครบถ้วน, เทมเพลต Bicep และการใช้งานบริการหลายภาษา (Python, Node.js, C#, Go)
- **การทดสอบ & แก้ไขปัญหา**: กรณีทดสอบครบวงจร, คำสั่งตรวจสอบ, คำแนะนำแก้ไขปัญหา

#### เปลี่ยนแปลง

#### ได้รับการยืนยัน

- **ศัพท์หลายเอเย่นต์**: เปลี่ยนชื่อ "เอเย่นต์ Cora" เป็น "เอเย่นต์ลูกค้า" ตลอดโซลูชันค้าปลีกหลายเอเย่นต์เพื่อความเข้าใจชัดเจนขึ้น
- **สถาปัตยกรรมเอเย่นต์**: อัปเดตเอกสาร, เทมเพลต ARM, และตัวอย่างโค้ดทั้งหมดให้ใช้ชื่อ "เอเย่นต์ลูกค้า" สอดคล้องกัน
- **ตัวอย่างการกำหนดค่า**: ปรับรูปแบบการกำหนดค่าเอเย่นต์ให้ทันสมัยด้วยรูปแบบการตั้งชื่อใหม่
- **ความสอดคล้องเอกสาร**: ตรวจสอบให้แน่ใจว่าอ้างอิงทั้งหมดใช้ชื่อเอเย่นต์ที่มืออาชีพและชัดเจน

#### ที่ปรับปรุง
- **แพ็กเกจเทมเพลต ARM**: อัปเดต retail-multiagent-arm-template ด้วยการอ้างอิงเอเย่นต์ลูกค้า
- **แผนภาพสถาปัตยกรรม**: ปรับปรุงแผนภาพ Mermaid ด้วยชื่อเอเย่นต์ใหม่
- **ตัวอย่างโค้ด**: ใช้คลาส Python และตัวอย่างการใช้งานเป็น CustomerAgent
- **ตัวแปรสิ่งแวดล้อม**: อัปเดตสคริปต์ปรับใช้ทั้งหมดให้ใช้รูปแบบ CUSTOMER_AGENT_NAME
- **ประสบการณ์นักพัฒนา**: กำหนดบทบาทและความรับผิดชอบของเอเจนต์ให้ชัดเจนในเอกสาร
- **ความพร้อมใช้งานในระบบผลิต**: ปรับให้สอดคล้องกับหลักเกณฑ์การตั้งชื่อขององค์กร
- **สื่อการเรียนรู้**: การตั้งชื่อเอเจนต์ที่เข้าใจง่ายขึ้นสำหรับการศึกษา
- **ความสะดวกในการใช้เทมเพลต**: ทำความเข้าใจฟังก์ชันเอเจนต์และรูปแบบการปรับใช้ให้เรียบง่ายขึ้น

#### รายละเอียดทางเทคนิค
- อัปเดตแผนภาพสถาปัตยกรรม Mermaid โดยมีการอ้างอิง CustomerAgent
- แทนที่ชื่อคลาส CoraAgent ด้วย CustomerAgent ในตัวอย่างภาษา Python
- แก้ไขการตั้งค่า JSON ของ ARM template เพื่อใช้ประเภทเอเจนต์ "customer"
- อัปเดตตัวแปรสภาพแวดล้อมจาก CORA_AGENT_* เป็น CUSTOMER_AGENT_*
- รีเฟรชคำสั่งปรับใช้และการกำหนดคอนเทนเนอร์ทั้งหมด

- **ส่วนขยาย VS Code**: ประสบการณ์พัฒนา AZD + AI Foundry แบบบูรณาการ

- **บทเรียนแบบโต้ตอบ**: แบบฝึกหัดเขียนโค้ดด้วยการตรวจสอบอัตโนมัติสำหรับสถานการณ์ AI
- **เนื้อหาวิดีโอ**: วิดีโอสอนเสริมสำหรับผู้เรียนสายสายตาที่มุ่งเน้นการปรับใช้ AI

- **AI แบบไฮบริดคลาวด์**: รูปแบบการปรับใช้หลายคลาวด์และไฮบริดสำหรับงาน AI

- **การเพิ่มประสิทธิภาพประสิทธิผล**: การปรับจูนและการขยายขนาดขั้นสูงสำหรับแอป AI ที่มีความต้องการสูง
- **การกระจายทั่วโลก**: รูปแบบการแจกจ่ายเนื้อหาและการแคชชิ่งที่ edge สำหรับแอป AI

## การมีส่วนร่วมในบันทึกการเปลี่ยนแปลง

### การรายงานการเปลี่ยนแปลง
เมื่อมีส่วนร่วมในที่เก็บนี้ กรุณาระบุรายการบันทึกการเปลี่ยนแปลงให้ประกอบด้วย:

1. **หมายเลขเวอร์ชัน**: ตาม semantic versioning (major.minor.patch)
2. **วันที่**: วันที่ปล่อยหรืออัปเดตในรูปแบบ YYYY-MM-DD
3. **หมวดหมู่**: Added, Changed, Deprecated, Removed, Fixed, Security
4. **คำอธิบายชัดเจน**: สรุปสั้น ๆ ว่าสิ่งใดเปลี่ยนแปลง
5. **การประเมินผลกระทบ**: ผลกระทบต่อผู้ใช้เดิมอย่างไร

### หมวดหมู่การเปลี่ยนแปลง

#### Added
- ฟีเจอร์ใหม่, ส่วนของเอกสาร หรือความสามารถใหม่
- ตัวอย่างใหม่, เทมเพลต หรือสื่อการเรียนรู้
- เครื่องมือ, สคริปต์ หรือตัวช่วยใหม่

#### Changed
- การแก้ไขฟังก์ชันหรือเอกสารที่มีอยู่
- อัปเดตเพื่อเพิ่มความชัดเจนหรือความแม่นยำ
- การปรับโครงสร้างเนื้อหาหรือการจัดองค์กร

#### Deprecated
- ฟีเจอร์หรือแนวทางที่ยกเลิกใช้งาน
- ส่วนของเอกสารที่กำหนดจะนำออก
- วิธีการที่มีทางเลือกที่ดีกว่า

#### Removed
- ฟีเจอร์, เอกสาร หรือ ตัวอย่างที่หมดความเกี่ยวข้อง
- ข้อมูลล้าสมัยหรือแนวทางที่เลิกใช้แล้ว
- เนื้อหาซ้ำซ้อนหรือลบรวม

#### Fixed
- การแก้ไขข้อผิดพลาดในเอกสารหรือโค้ด
- การแก้ไขปัญหาที่รายงาน
- ปรับปรุงความแม่นยำหรือฟังก์ชัน

#### Security
- การปรับปรุงหรือแก้ไขเรื่องความปลอดภัย
- อัปเดตแนวทางรักษาความปลอดภัยที่ดีที่สุด
- การแก้ไขช่องโหว่ด้านความปลอดภัย

### แนวทางการใช้ Semantic Versioning

#### เวอร์ชันหลัก (X.0.0)
- การเปลี่ยนแปลงที่ทำให้ไม่เข้ากันกับเวอร์ชันก่อนหน้า ต้องให้ผู้ใช้เปลี่ยนแปลงตาม
- การปรับโครงสร้างเนื้อหาหรือองค์กรอย่างมีนัยสำคัญ
- การเปลี่ยนแปลงที่ส่งผลต่อแนวทางหรือวิธีการพื้นฐาน

#### เวอร์ชันรอง (X.Y.0)
- เพิ่มฟีเจอร์หรือเนื้อหาใหม่
- ปรับปรุงที่ยังคงเข้ากันได้ย้อนหลัง
- ตัวอย่าง เครื่องมือ หรือสื่อเพิ่มเติม

#### เวอร์ชันย่อย (X.Y.Z)
- แก้ไขบั๊กและข้อผิดพลาด
- ปรับปรุงเล็กน้อยในเนื้อหาที่มีอยู่
- ข้อชี้แจงและการปรับปรุงเล็กน้อย

## ข้อเสนอแนะและคำติชมจากชุมชน

เราสนับสนุนให้ชุมชนแสดงความคิดเห็นเพื่อปรับปรุงสื่อการเรียนรู้นี้:

### วิธีส่งคำติชม
- **ปัญหาใน GitHub**: รายงานปัญหาหรือแนะนำปรับปรุง (ยินดีรับปัญหาเฉพาะ AI)
- **การพูดคุยใน Discord**: แบ่งปันความคิดและร่วมกับชุมชน Microsoft Foundry
- **Pull Requests**: ร่วมแก้ไขและปรับปรุงเนื้อหาโดยตรง โดยเฉพาะเทมเพลตและคู่มือ AI
- **Microsoft Foundry Discord**: เข้าร่วมในช่อง #Azure สำหรับการสนทนา AZD + AI
- **ฟอรัมชุมชน**: ร่วมสนทนาในฟอรัมผู้พัฒนา Azure กว้างขึ้น

### หมวดหมู่คำติชม
- **ความถูกต้องเนื้อหา AI**: การแก้ไขข้อมูลการรวมบริการและการปรับใช้ AI
- **ประสบการณ์การเรียนรู้**: ข้อเสนอแนะเพื่อเพิ่มประสิทธิผลการเรียนรู้ของนักพัฒนา AI
- **เนื้อหา AI ที่ขาดหาย**: ขอเทมเพลต รูปแบบ หรือ ตัวอย่าง AI เพิ่มเติม
- **การเข้าถึง**: ปรับปรุงให้เหมาะกับความต้องการการเรียนรู้ที่หลากหลาย
- **การรวมเครื่องมือ AI**: ข้อเสนอแนะการรวมเวิร์กโฟลว์พัฒนา AI ที่ดีกว่า
- **รูปแบบปรับใช้ AI ในระบบผลิต**: คำขอรูปแบบการปรับใช้ AI ระดับองค์กร

### การตอบกลับข้อเสนอแนะ
- **ตอบปัญหา**: ภายใน 48 ชั่วโมงสำหรับปัญหาที่รายงาน
- **คำขอฟีเจอร์**: ประเมินภายในหนึ่งสัปดาห์
- **การมีส่วนร่วมของชุมชน**: ตรวจสอบภายในหนึ่งสัปดาห์
- **ปัญหาด้านความปลอดภัย**: ให้ความสำคัญทันทีพร้อมการตอบกลับเร่งด่วน
