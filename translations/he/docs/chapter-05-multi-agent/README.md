## סקירה כללית

פרק זה מכסה דפוסי ארכיטקטורה מתקדמים למספר סוכנים, תזמור סוכנים, והטמעות AI מוכנות לפרודקשן בתרחישים מורכבים.

> נבדק עם `azd 1.27.1` ביולי 2026.

## יעדי למידה

בסיום פרק זה תוכל:
- להבין דפוסי ארכיטקטורה מרובי סוכנים
- לפרוס מערכות סוכנים מתואמות
- ליישם תקשורת סוכן-לסוכן
- לבנות פתרונות מרובי סוכנים מוכנים לפרודקשן

## 🚀 התחלה מהירה

```bash
# אפשרות 1: פריסה מתבנית
azd init --template agent-openai-python-prompty
azd up

# אפשרות 2: פריסה ממניפסט סוכן (דורש את ההרחבה azure.ai.agents)
azd extension install azure.ai.agents
azd ai agent init -m agent-manifest.yaml
azd up
```

> **איזה גישה?** השתמש ב־`azd init --template` כדי להתחיל מדוגמה עובדת. השתמש ב־`azd ai agent init` כשיש לך מיפרט סוכן משלך. ראה את [מדריך שורת הפקודה של AZD AI](../chapter-08-production/production-ai-practices.md#azd-ai-cli-commands-and-extensions) לפרטים מלאים.
