## סקירה

פרק זה מכסה תצורת סביבה, דפוסי אימות, והנחיות בטיחות לשימוש ב-Azure Developer CLI.

> אומת עם `azd 1.23.12` במרץ 2026.

## יעדי למידה

בסיום פרק זה, תלמדו:
- לשלוט בהיררכיית התצורה של AZD
- לנהל מספר סביבות (פיתוח, בדיקה, הפקה)
- להטמיע אימות מאובטח עם זהויות מנוהלות
- להגדיר הגדרות ספציפיות לסביבה

## 🚀 התחלה מהירה

```bash
# צור סביבות מרובות
azd env new dev
azd env new staging
azd env new prod

# החלף סביבות
azd env select prod

# הגדר משתני סביבה
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# הצג תצורה
azd env get-values
```

---

## 🔧 היררכיית תצורה

AZD מיישם את ההגדרות לפי הסדר הבא (הגדרות מאוחרות מבטלות מוקדמות):

1. **ערכי ברירת מחדל** (מובנים בתבניות)
2. **azure.yaml** (תצורת הפרויקט)
3. **משתני סביבה** (`azd env set`)
4. **דגלי שורת הפקודה** (`--location eastus`)

---

## 🔐 הנחיות בטיחות מיטביות

```bash
# השתמש בזיהוי מנוהל (מומלץ)
azd env set AZURE_USE_MANAGED_IDENTITY true

# בדוק את מצב האימות של AZD
azd auth status

# אופציונלי: אמת את הקשר של Azure CLI אם אתה מתכנן להריץ פקודות az
az account show

# אמת מחדש אם יש צורך
azd auth login

# אופציונלי: רענן את האימות של Azure CLI עבור פקודות az
az login
```
