## סקירה כללית

פרק זה מכסה תצורת סביבה, דפוסי אימות, ושיטות עבודה מומלצות לאבטחה עבור פריסות של Azure Developer CLI.

## מטרות למידה

בסיום פרק זה תוכלו:
- לשלוט בהיררכיית התצורה של AZD
- לנהל סביבות מרובות (dev, staging, prod)
- ליישם אימות מאובטח באמצעות זהויות מנוהלות
- להגדיר הגדרות ספציפיות לסביבה

## 🚀 התחלה מהירה

```bash
# צור מספר סביבות
azd env new dev
azd env new staging
azd env new prod

# החלף בין סביבות
azd env select prod

# הגדר משתני סביבה
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# הצג תצורה
azd env get-values
```

---

## 🔧 היררכיית תצורה

AZD מיישם הגדרות בסדר הבא (הגדרות מאוחרות יותר מחליפות את הקודמות):

1. **ערכי ברירת מחדל** (מובנים בתבניות)
2. **azure.yaml** (תצורת הפרויקט)
3. **משתני סביבה** (`azd env set`)
4. **דגלי שורת הפקודה** (`--location eastus`)

---

## 🔐 שיטות עבודה מומלצות לאבטחה

```bash
# השתמש בזהות מנוהלת (מומלץ)
azd env set AZURE_USE_MANAGED_IDENTITY true

# בדוק מצב אימות
azd auth whoami
az account show

# בצע אימות מחדש אם נדרש
azd auth login
az login
```
