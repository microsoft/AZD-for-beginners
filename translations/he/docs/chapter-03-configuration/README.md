## סקירה כללית

פרק זה מכסה תצורת סביבה, דפוסי אימות, ושיטות אבטחה מיטביות לפריסות Azure Developer CLI.

> אומת נגד `azd 1.25.6` ביוני 2026.

## מטרות הלמידה

בסיום פרק זה, תדע:
- לשלוט בהיררכיית התצורה של AZD
- לנהל סביבות מרובות (פיתוח, בדיקה, הפקה)
- ליישם אימות מאובטח עם זהויות מנוהלות
- לקנפג הגדרות ייחודיות לכל סביבה

## 🚀 התחלה מהירה

```bash
# יצירת סביבות מרובות
azd env new dev
azd env new staging
azd env new prod

# החלפת סביבות
azd env select prod

# הגדרת משתני סביבה
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# צפייה בקונפיגורציה
azd env get-values
```

---

## 🔧 היררכיית תצורה

AZD מיישם הגדרות לפי הסדר הבא (לאחר מבטל קודמות):

1. **ערכי ברירת מחדל** (מובנים בתבניות)
2. **azure.yaml** (תצורת הפרויקט)
3. **משתני סביבה** (`azd env set`)
4. **דגלי שורת הפקודה** (`--location eastus`)

---

## 🔐 שיטות אבטחה מיטביות

```bash
# השתמש בזהות מנוהלת (מומלץ)
azd env set AZURE_USE_MANAGED_IDENTITY true

# בדוק את מצב האימות של AZD
azd auth status

# אופציונלי: אמת את הקשר של Azure CLI אם אתה מתכנן להריץ פקודות az
az account show

# אמת מחדש את האימות במידת הצורך
azd auth login

# אופציונלי: רענן את האימות של Azure CLI עבור פקודות az
az login
```
