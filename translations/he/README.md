## סקירת הקורס

למדו את Azure Developer CLI (azd) באמצעות פרקים מובנים המיועדים ללמידה מתקדמת. **התמקדות מיוחדת בפריסת יישומי AI עם אינטגרציה של Microsoft Foundry.**

### למה הקורס הזה חיוני למפתחים מודרניים

בהתבסס על תובנות מקהילת Microsoft Foundry ב-Discord, **45% מהמפתחים רוצים להשתמש ב-AZD לעבודה עם AI** אך נתקלים באתגרים כמו:
- ארכיטקטורות AI מרובות שירותים ומורכבות
- שיטות עבודה מומלצות לפריסת AI בסביבת ייצור  
- אינטגרציה וקונפיגורציה של שירותי Azure AI
- אופטימיזציה של עלויות לעומסי עבודה של AI
- פתרון בעיות בפריסות AI

### מטרות למידה

בסיום הקורס המובנה הזה, תלמדו:
- **שליטה ביסודות AZD**: מושגים מרכזיים, התקנה וקונפיגורציה
- **פריסת יישומי AI**: שימוש ב-AZD עם שירותי Microsoft Foundry
- **יישום Infrastructure as Code**: ניהול משאבי Azure עם תבניות Bicep
- **פתרון בעיות בפריסות**: זיהוי ותיקון בעיות נפוצות
- **אופטימיזציה לסביבת ייצור**: אבטחה, סקיילינג, ניטור וניהול עלויות
- **בניית פתרונות מרובי סוכנים**: פריסת ארכיטקטורות AI מורכבות

### 🚀 פרק 1: יסודות והתחלה מהירה
**דרישות מוקדמות**: מנוי Azure, ידע בסיסי בשורת הפקודה  
**משך זמן**: 30-45 דקות  
**רמת קושי**: ⭐

#### מה תלמדו
- הבנת יסודות Azure Developer CLI
- התקנת AZD על הפלטפורמה שלכם
- הפריסה הראשונה שלכם בהצלחה

#### משאבי למידה
- **🎯 התחילו כאן**: [מהו Azure Developer CLI?](../..)
- **📖 תיאוריה**: [יסודות AZD](docs/getting-started/azd-basics.md) - מושגים ומונחים מרכזיים
- **⚙️ התקנה**: [מדריך התקנה וקונפיגורציה](docs/getting-started/installation.md) - מדריכים לפי פלטפורמה
- **🛠️ תרגול מעשי**: [הפרויקט הראשון שלכם](docs/getting-started/first-project.md) - מדריך שלב אחר שלב
- **📋 עזר מהיר**: [דף פקודות](resources/cheat-sheet.md)

#### תרגילים מעשיים
```bash
# בדיקת התקנה מהירה
azd version

# פריסת היישום הראשון שלך
azd init --template todo-nodejs-mongo
azd up
```

**💡 תוצאה מהפרק**: פריסה מוצלחת של יישום אינטרנט פשוט ל-Azure באמצעות AZD

**✅ אימות הצלחה:**
```bash
# לאחר סיום פרק 1, עליך להיות מסוגל:
azd version              # מציג גרסה מותקנת
azd init --template todo-nodejs-mongo  # מאתחל פרויקט
azd up                  # מפרסם ל-Azure
azd show                # מציג כתובת URL של אפליקציה פועלת
# האפליקציה נפתחת בדפדפן ועובדת
azd down --force --purge  # מנקה משאבים
```

**📊 השקעת זמן:** 30-45 דקות  
**📈 רמת מיומנות לאחר מכן:** יכולת לפרוס יישומים בסיסיים באופן עצמאי

### 🤖 פרק 2: פיתוח ממוקד AI (מומלץ למפתחי AI)
**דרישות מוקדמות**: סיום פרק 1  
**משך זמן**: 1-2 שעות  
**רמת קושי**: ⭐⭐

#### מה תלמדו
- אינטגרציה של Microsoft Foundry עם AZD
- פריסת יישומים מבוססי AI
- הבנת קונפיגורציות של שירותי AI

#### משאבי למידה
- **🎯 התחילו כאן**: [אינטגרציית Microsoft Foundry](docs/microsoft-foundry/microsoft-foundry-integration.md)
- **📖 תבניות**: [פריסת מודל AI](docs/microsoft-foundry/ai-model-deployment.md) - פריסה וניהול של מודלים
- **🛠️ סדנה**: [מעבדת סדנת AI](docs/microsoft-foundry/ai-workshop-lab.md) - הכנת פתרונות AI ל-AZD
- **🎥 מדריך אינטראקטיבי**: [חומרי סדנה](workshop/README.md) - למידה מבוססת דפדפן עם MkDocs * DevContainer Environment
- **📋 תבניות**: [תבניות Microsoft Foundry](../..)
- **📝 דוגמאות**: [דוגמאות לפריסת AZD](examples/README.md)

#### תרגילים מעשיים
```bash
# פרס את יישום הבינה המלאכותית הראשון שלך
azd init --template azure-search-openai-demo
azd up

# נסה תבניות בינה מלאכותית נוספות
azd init --template openai-chat-app-quickstart
azd init --template agent-openai-python-prompty
```

**💡 תוצאה מהפרק**: פריסה וקונפיגורציה של יישום צ'אט מבוסס AI עם יכולות RAG

**✅ אימות הצלחה:**
```bash
# לאחר פרק 2, עליך להיות מסוגל:
azd init --template azure-search-openai-demo
azd up
# לבדוק את ממשק הצ'אט של הבינה המלאכותית
# לשאול שאלות ולקבל תשובות מבוססות בינה מלאכותית עם מקורות
# לוודא שהאינטגרציה של החיפוש פועלת
azd monitor  # לבדוק ש-Application Insights מציג טלמטריה
azd down --force --purge
```

**📊 השקעת זמן:** 1-2 שעות  
**📈 רמת מיומנות לאחר מכן:** יכולת לפרוס ולנהל יישומי AI מוכנים לייצור  
**💰 מודעות לעלויות:** הבנת עלויות פיתוח של $80-150 לחודש, ועלויות ייצור של $300-3500 לחודש

#### 💰 שיקולי עלות לפריסות AI

**סביבת פיתוח (הערכה $80-150 לחודש):**
- Azure OpenAI (תשלום לפי שימוש): $0-50 לחודש (בהתאם לשימוש בטוקנים)
- AI Search (רמת בסיס): $75 לחודש
- Container Apps (תשלום לפי צריכה): $0-20 לחודש
- אחסון (סטנדרטי): $1-5 לחודש

**סביבת ייצור (הערכה $300-3,500+ לחודש):**
- Azure OpenAI (PTU לביצועים עקביים): $3,000+ לחודש או תשלום לפי שימוש בנפח גבוה
- AI Search (רמת סטנדרט): $250 לחודש
- Container Apps (ייעודי): $50-100 לחודש
- Application Insights: $5-50 לחודש
- אחסון (פרימיום): $10-50 לחודש

**💡 טיפים לאופטימיזציית עלויות:**
- השתמשו ב-**Free Tier** של Azure OpenAI ללמידה (כולל 50,000 טוקנים לחודש)
- הריצו `azd down` כדי לשחרר משאבים כשאינכם מפתחים באופן פעיל
- התחילו עם תשלום לפי צריכה, ושדרגו ל-PTU רק לייצור
- השתמשו ב-`azd provision --preview` להערכת עלויות לפני פריסה
- הפעילו אוטו-סקיילינג: שלמו רק על שימוש בפועל

**מעקב עלויות:**
```bash
# בדוק עלויות חודשיות משוערות
azd provision --preview

# עקוב אחר עלויות בפועל בפורטל Azure
az consumption budget list --resource-group <your-rg>
```

## 📖 מהו Azure Developer CLI?  

Azure Developer CLI (azd) הוא ממשק שורת פקודה ממוקד מפתחים שמאיץ את תהליך בניית ופריסת יישומים ל-Azure. הוא מספק:  

- **פריסות מבוססות תבניות** - שימוש בתבניות מוכנות לדפוסי יישומים נפוצים  
- **תשתית כקוד** - ניהול משאבי Azure באמצעות Bicep או Terraform  
- **תהליכי עבודה משולבים** - אספקה, פריסה וניטור יישומים בצורה חלקה  
- **ידידותי למפתחים** - מותאם לפרודוקטיביות וחוויית מפתחים  

### **AZD + Microsoft Foundry: מושלם לפריסות AI**  

**למה AZD לפתרונות AI?** AZD מתמודד עם האתגרים המרכזיים שמפתחים בתחום AI נתקלים בהם:  

- **תבניות מוכנות ל-AI** - תבניות מוגדרות מראש ל-Azure OpenAI, Cognitive Services ועומסי עבודה של ML  
- **פריסות AI מאובטחות** - דפוסי אבטחה מובנים לשירותי AI, מפתחות API ונקודות קצה של מודלים  
- **דפוסי AI לייצור** - שיטות מיטביות ליישומי AI בקנה מידה חסכוני  
- **תהליכי עבודה מקצה לקצה ב-AI** - מפיתוח מודלים ועד פריסה בייצור עם ניטור מתאים  
- **אופטימיזציה של עלויות** - אסטרטגיות הקצאת משאבים חכמות וסקלביליות לעומסי עבודה של AI  
- **אינטגרציה עם Microsoft Foundry** - חיבור חלק לקטלוג המודלים ונקודות הקצה של Microsoft Foundry

## 🔧 מדריך פתרון בעיות מהיר  

**בעיות נפוצות למתחילים ופתרונות מיידיים:**  

### ❌ "azd: command not found"  

```bash
# התקן את AZD תחילה
# Windows (PowerShell):
winget install microsoft.azd

# macOS:
brew tap azure/azd && brew install azd

# Linux:
curl -fsSL https://aka.ms/install-azd.sh | bash

# אמת את ההתקנה
azd version
```
  
### ❌ "No subscription found" או "Subscription not set"  

```bash
# רשימת מנויים זמינים
az account list --output table

# הגדר מנוי ברירת מחדל
az account set --subscription "<subscription-id-or-name>"

# הגדר עבור סביבת AZD
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# אמת
az account show
```
  
### ❌ "InsufficientQuota" או "Quota exceeded"  

```bash
# נסה אזור Azure שונה
azd env set AZURE_LOCATION "westus2"
azd up

# או השתמש ב-SKUs קטנים יותר בפיתוח
# ערוך infra/main.parameters.json:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
  
### ❌ "azd up" נכשל באמצע  

```bash
# אפשרות 1: לנקות ולנסות שוב
azd down --force --purge
azd up

# אפשרות 2: לתקן רק את התשתית
azd provision

# אפשרות 3: לבדוק יומנים מפורטים
azd show
azd logs
```
  
### ❌ "Authentication failed" או "Token expired"  

```bash
# אימות מחדש
az logout
az login

azd auth logout
azd auth login

# אימות זהות
az account show
```
  
### ❌ "Resource already exists" או קונפליקטים בשמות  

```bash
# AZD יוצר שמות ייחודיים, אך אם יש התנגשות:
azd down --force --purge

# אז נסה שוב עם סביבה חדשה
azd env new dev-v2
azd up
```
  
### ❌ פריסת תבנית לוקחת יותר מדי זמן  

**זמני המתנה רגילים:**  
- אפליקציית ווב פשוטה: 5-10 דקות  
- אפליקציה עם מסד נתונים: 10-15 דקות  
- אפליקציות AI: 15-25 דקות (הקצאת OpenAI איטית)  

```bash
# בדוק התקדמות
azd show

# אם תקוע >30 דקות, בדוק את פורטל Azure:
azd monitor
# חפש פריסות שנכשלו
```
  
### ❌ "Permission denied" או "Forbidden"  

```bash
# בדוק את התפקיד שלך ב-Azure
az role assignment list --assignee $(az account show --query user.name -o tsv)

# אתה צריך לפחות תפקיד "Contributor"
# בקש ממנהל ה-Azure שלך להעניק:
# - Contributor (עבור משאבים)
# - User Access Administrator (עבור הקצאות תפקידים)
```
  
### ❌ לא ניתן למצוא את כתובת ה-URL של האפליקציה שפורסה  

```bash
# הצג את כל נקודות הקצה של השירות
azd show

# או פתח את פורטל Azure
azd monitor

# בדוק שירות ספציפי
azd env get-values
# חפש משתני *_URL
```

## 🔧 מדריך פתרון בעיות מהיר  

**בעיות נפוצות למתחילים ופתרונות מיידיים:**  

<details>  
<summary><strong>❌ "azd: command not found"</strong></summary>  

```bash
# התקן את AZD תחילה
# Windows (PowerShell):
winget install microsoft.azd

# macOS:
brew tap azure/azd && brew install azd

# Linux:
curl -fsSL https://aka.ms/install-azd.sh | bash

# אמת את ההתקנה
azd version
```
</details>  

<details>  
<summary><strong>❌ "No subscription found" או "Subscription not set"</strong></summary>  

```bash
# רשימת מנויים זמינים
az account list --output table

# הגדר מנוי ברירת מחדל
az account set --subscription "<subscription-id-or-name>"

# הגדר עבור סביבת AZD
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# אימות
az account show
```
</details>  

<details>  
<summary><strong>❌ "InsufficientQuota" או "Quota exceeded"</strong></summary>  

```bash
# נסה אזור Azure שונה
azd env set AZURE_LOCATION "westus2"
azd up

# או השתמש ב-SKUs קטנים יותר בפיתוח
# ערוך infra/main.parameters.json:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>  

<details>  
<summary><strong>❌ "azd up" נכשל באמצע</strong></summary>  

```bash
# אפשרות 1: לנקות ולנסות שוב
azd down --force --purge
azd up

# אפשרות 2: לתקן רק את התשתית
azd provision

# אפשרות 3: לבדוק יומנים מפורטים
azd show
azd logs
```
</details>  

<details>  
<summary><strong>❌ "Authentication failed" או "Token expired"</strong></summary>  

```bash
# אימות מחדש
az logout
az login

azd auth logout
azd auth login

# אימות זהות
az account show
```
</details>  

<details>  
<summary><strong>❌ "Resource already exists" או קונפליקטים בשמות</strong></summary>  

```bash
# AZD יוצר שמות ייחודיים, אך אם יש התנגשות:
azd down --force --purge

# אז נסה שוב עם סביבה חדשה
azd env new dev-v2
azd up
```
</details>  

<details>  
<summary><strong>❌ פריסת תבנית לוקחת יותר מדי זמן</strong></summary>  

**זמני המתנה רגילים:**  
- אפליקציית ווב פשוטה: 5-10 דקות  
- אפליקציה עם מסד נתונים: 10-15 דקות  
- אפליקציות AI: 15-25 דקות (הקצאת OpenAI איטית)  

```bash
# בדוק התקדמות
azd show

# אם תקוע >30 דקות, בדוק את פורטל Azure:
azd monitor
# חפש פריסות שנכשלו
```
</details>  

<details>  
<summary><strong>❌ "Permission denied" או "Forbidden"</strong></summary>  

```bash
# בדוק את התפקיד שלך ב-Azure
az role assignment list --assignee $(az account show --query user.name -o tsv)

# אתה צריך לפחות תפקיד "Contributor"
# בקש ממנהל ה-Azure שלך להעניק:
# - Contributor (עבור משאבים)
# - User Access Administrator (עבור הקצאות תפקידים)
```
</details>  

<details>  
<summary><strong>❌ לא ניתן למצוא את כתובת ה-URL של האפליקציה שפורסה</strong></summary>  

```bash
# הצג את כל נקודות הקצה של השירות
azd show

# או פתח את פורטל Azure
azd monitor

# בדוק שירות ספציפי
azd env get-values
# חפש משתנים *_URL
```
</details>
