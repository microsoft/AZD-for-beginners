## סקירת הקורס

שלוט ב-Azure Developer CLI (azd) באמצעות פרקים מובנים המיועדים ללמידה הדרגתית. **הדגשה מיוחדת על פריסת יישומי בינה מלאכותית עם אינטגרציה ל-Microsoft Foundry.**

### למה הקורס הזה חיוני למפתחים מודרניים

בהתבסס על תובנות מקהילת Discord של Microsoft Foundry, **45% מהמפתחים רוצים להשתמש ב-AZD לעומסי עבודה של בינה מלאכותית** אך נתקלים באתגרים ב:
- ארכיטקטורות AI מורכבות הכוללות שירותים מרובים
- שיטות עבודה מומלצות לפריסת AI בסביבת ייצור  
- אינטגרציה וקונפיגורציה של שירותי Azure AI
- אופטימיזציה של עלויות לעומסי עבודה של AI
- איתור ופתרון בעיות בפריסות ייחודיות ל-AI

### מטרות הלמידה

על ידי השלמת הקורס המובנה הזה, תשיגו:
- **שליטה ביסודות AZD**: עקרונות מרכזיים, התקנה וקונפיגורציה
- **פריסת יישומי AI**: שימוש ב-AZD עם שירותי Microsoft Foundry
- **הטמעת תשתית כקוד**: ניהול משאבי Azure עם תבניות Bicep
- **איתור תקלות בפריסות**: פתרון בעיות נפוצות וניפוי שגיאות
- **אופטימיזציה לייצור**: אבטחה, קנה מידה, ניטור וניהול עלויות
- **בניית פתרונות רב-סוכנים**: פריסת ארכיטקטורות AI מורכבות

#### משאבי למידה
- **🎯 התחל כאן**: [מהו Azure Developer CLI?](../..)
- **📖 תיאוריה**: [יסודות AZD](docs/getting-started/azd-basics.md) - עקרונות מרכזיים ומונחים
- **⚙️ התקנה**: [התקנה והגדרות](docs/getting-started/installation.md) - מדריכים ספציפיים לפלטפורמה

#### תרגילים מעשיים
```bash
# בדיקת התקנה מהירה

# פרוס את היישום הראשון שלך

```

**💡 תוצאת הפרק**: לפרוס בהצלחה יישום ווב פשוט ל-Azure באמצעות AZD

**✅ תנאי הצלחה:**

```bash
# לאחר השלמת פרק 1 תוכל לבצע את הפעולות הבאות:
```

**📊 משך זמן נדרש:** 30-45 דקות  
**📈 רמת כישורים לאחר:** יכול/ה לפרוס יישומים בסיסיים באופן עצמאי

### 🤖 פרק 2: פיתוח הממוקד ב-AI (מומלץ למפתחי AI)
**דרישות מוקדמות**: השלמת פרק 1  
**משך**: 1-2 שעות  
**מורכבות**: ⭐⭐

#### מה תלמדו
- אינטגרציה של Microsoft Foundry עם AZD
- פריסת יישומים המונעים על-ידי AI
- הבנת קונפיגורציות של שירותי AI

#### משאבי למידה
- **🎯 התחל כאן**: [אינטגרציה עם Microsoft Foundry](docs/microsoft-foundry/microsoft-foundry-integration.md)
- **📖 דוגמאות/תבניות**: [פריסת מודלים של AI](docs/microsoft-foundry/ai-model-deployment.md) - פרוס ונהל מודלי AI
- **🛠️ סדנה**: [מעבדת סדנת AI](docs/microsoft-foundry/ai-workshop-lab.md) - הכנת פתרונות ה-AI שלך לשימוש עם AZD
- **🎥 מדריך אינטראקטיבי**: [חומרי הסדנה](workshop/README.md) - למידה בדפדפן עם MkDocs * סביבה DevContainer
- **📋 תבניות**: [תבניות Microsoft Foundry](../..)
- **📝 דוגמאות**: [דוגמאות פריסה ב-AZD](examples/README.md)

#### תרגילים מעשיים
```bash
# פרוס את אפליקציית הבינה המלאכותית הראשונה שלך
azd init --template azure-search-openai-demo
azd up

# נסה תבניות נוספות של בינה מלאכותית
azd init --template openai-chat-app-quickstart
azd init --template agent-openai-python-prompty
```

**💡 תוצאת הפרק**: לפרוס ולהגדיר אפליקציית צ'אט המונעת ב-AI עם יכולות RAG

**✅ תנאי הצלחה:**
```bash
# לאחר פרק 2, תדעו לבצע את הפעולות הבאות:
azd init --template azure-search-openai-demo
azd up
# לבדוק את ממשק הצ'אט של הבינה המלאכותית
# לשאול שאלות ולקבל תשובות המופקות על ידי בינה מלאכותית עם מקורות
# לאמת שאינטגרציית החיפוש פועלת
azd monitor  # לבדוק ש-Application Insights מציג טלמטריה
azd down --force --purge
```

**📊 משך זמן נדרש:** 1-2 שעות  
**📈 רמת כישורים לאחר:** יכול/ה לפרוס ולהגדיר יישומי AI מוכנים לייצור  
**💰 מודעות עלויות:** הבן/י עלויות פיתוח מוערכות של $80-150/חודש, ועלויות ייצור של $300-3,500/חודש

#### 💰 שיקולים של עלויות לפריסות AI

**סביבת פיתוח (מוערכת $80-150/חודש):**
- Azure OpenAI (תשלום לפי שימוש - Pay-as-you-go): $0-50/חודש (בהתבסס על שימוש בטוקנים)
- AI Search (רמת Basic): $75/חודש
- Container Apps (Consumption): $0-20/חודש
- Storage (Standard): $1-5/חודש

**סביבת ייצור (מוערכת $300-3,500+/חודש):**
- Azure OpenAI (PTU עבור ביצועים יציבים): $3,000+/חודש OR Pay-as-go with high volume
- AI Search (רמת Standard): $250/חודש
- Container Apps (Dedicated): $50-100/חודש
- Application Insights: $5-50/חודש
- Storage (Premium): $10-50/חודש

**💡 טיפים לאופטימיזציית עלויות:**
- השתמש ב-**Free Tier** של Azure OpenAI ללמידה (כולל 50,000 טוקנים/חודש)
- הרץ `azd down` כדי לשחרר משאבים כאשר לא מפתחים באופן פעיל
- התחל בחיוב מבוסס צריכה, שדרג ל-PTU רק בסביבת ייצור
- השתמש ב-`azd provision --preview` להערכת עלויות לפני פריסה
- אפשר קנה מידה אוטומטי: שלם רק עבור השימוש האמיתי

**ניטור עלויות:**
```bash
# בדוק עלויות חודשיות משוערות
azd provision --preview

# נטר את העלויות בפועל בפורטל Azure
az consumption budget list --resource-group <your-rg>
```

## 📖 מה זה Azure Developer CLI?

Azure Developer CLI (azd) היא ממשק שורת פקודה המיועד למפתחים שמאיץ את תהליך הבנייה והפריסה של יישומים ב-Azure. הוא מספק:

- **פריסות מבוססות תבניות** - השתמש בתבניות מוכנות עבור דפוסי יישומים נפוצים
- **תשתית כקוד** - נהל משאבי Azure באמצעות Bicep או Terraform  
- **זרימות עבודה משולבות** - ספק, פרוס ונטר יישומים בצורה חלקה
- **נוח למפתחים** - מותאם לפרודוקטיביות וחוויית המפתח

### **AZD + Microsoft Foundry: מושלם לפריסות AI**

**מדוע AZD לפתרונות AI?** AZD מתמודד עם האתגרים המרכזיים שמפתחים בתחום ה-AI נתקלים בהם:

- **תבניות מוכנות ל-AI** - תבניות מוגדרות מראש עבור Azure OpenAI, Cognitive Services ועומסי עבודה של ML
- **פריסות AI מאובטחות** - דפוסי אבטחה מובנים לשירותי AI, מפתחות API ונקודות קצה של מודלים  
- **תבניות AI לפרודקשן** - שיטות עבודה מומלצות לפריסות AI סקיילביליות וחסכוניות
- **זרימות עבודה AI מקצה לקצה** - מפיתוח מודלים ועד פריסה לפרודקשן עם ניטור נאות
- **אופטימיזציית עלות** - הקצאת משאבים חכמה ואסטרטגיות סקיילינג לעומסי עבודה של AI
- **אינטגרציה עם Microsoft Foundry** - חיבור חלק לקטלוג המודלים ונקודות הקצה של Microsoft Foundry

## 🔧 מדריך פתרון בעיות מהיר

**בעיות נפוצות למתחילים ופתרונות מיידיים:**

### ❌ "azd: command not found"

```bash
# התקן את AZD תחילה
# Windows (PowerShell):
winget install microsoft.azd

# macOS:
brew tap azure/azd && brew install azd

# Linux:
curl -fsSL https://aka.ms/install-azd.sh | bash

# אמת את ההתקנה
azd version
```

### ❌ "No subscription found" or "Subscription not set"

```bash
# רשימת המנויים הזמינים
az account list --output table

# הגדר את המנוי כברירת מחדל
az account set --subscription "<subscription-id-or-name>"

# הגדר עבור סביבת AZD
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# אמת
az account show
```

### ❌ "InsufficientQuota" or "Quota exceeded"

```bash
# נסה אזור אחר ב-Azure
azd env set AZURE_LOCATION "westus2"
azd up

# או השתמש ב-SKUs קטנים יותר בסביבת הפיתוח
# ערוך infra/main.parameters.json:
{
  "sku": "B1"  // Instead of "P1V2"
}
```

### ❌ "azd up" fails halfway through

```bash
# אפשרות 1: נקה ונסה שוב
azd down --force --purge
azd up

# אפשרות 2: פשוט תקן את התשתית
azd provision

# אפשרות 3: בדוק לוגים מפורטים
azd show
azd logs
```

### ❌ "Authentication failed" or "Token expired"

```bash
# לאמת מחדש
az logout
az login

azd auth logout
azd auth login

# לאמת את ההזדהות
az account show
```

### ❌ "Resource already exists" or naming conflicts

```bash
# AZD יוצר שמות ייחודיים, אבל אם יש התנגשות:
azd down --force --purge

# לאחר מכן נסה שוב עם סביבה חדשה
azd env new dev-v2
azd up
```

### ❌ Template deployment taking too long

**זמני המתנה רגילים:**
- אפליקציית ווב פשוטה: 5-10 דקות
- אפליקציה עם בסיס נתונים: 10-15 דקות
- יישומי AI: 15-25 דקות (הספקת OpenAI איטית)

```bash
# בדוק את ההתקדמות
azd show

# אם תקוע יותר מ-30 דקות, בדוק את פורטל Azure:
azd monitor
# חפש פריסות שנכשלו
```

### ❌ "Permission denied" or "Forbidden"

```bash
# בדוק את התפקיד שלך ב-Azure
```

## 🔧 מדריך פתרון בעיות מהיר

**בעיות נפוצות למתחילים ופתרונות מיידיים:**

<details>
<summary><strong>❌ "azd: command not found"</strong></summary>

```bash
# התקן את AZD תחילה
# Windows (PowerShell):
winget install microsoft.azd

# macOS:
brew tap azure/azd && brew install azd

# לינוקס:
curl -fsSL https://aka.ms/install-azd.sh | bash

# וודא את ההתקנה
azd version
```
</details>

<details>
<summary><strong>❌ "No subscription found" or "Subscription not set"</strong></summary>

```bash
# רשימת המנויים הזמינים
az account list --output table

# הגדר את המנוי כברירת מחדל
az account set --subscription "<subscription-id-or-name>"

# הגדר לסביבת AZD
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# אמת
az account show
```
</details>

<details>
<summary><strong>❌ "InsufficientQuota" or "Quota exceeded"</strong></summary>

```bash
# נסה אזור אחר ב-Azure
azd env set AZURE_LOCATION "westus2"
azd up

# או השתמש ב-SKU-ים קטנים יותר בסביבת הפיתוח
# ערוך את הקובץ infra/main.parameters.json:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>

<details>
<summary><strong>❌ "azd up" fails halfway through</strong></summary>

```bash
# אפשרות 1: נקה ונסה שוב
azd down --force --purge
azd up

# אפשרות 2: תקן רק את התשתית
azd provision

# אפשרות 3: בדוק יומנים מפורטים
azd show
azd logs
```
</details>

<details>
<summary><strong>❌ "Authentication failed" or "Token expired"</strong></summary>

```bash
# אמת מחדש
az logout
az login

azd auth logout
azd auth login

# אמת את ההזדהות
az account show
```
</details>

<details>
<summary><strong>❌ "Resource already exists" or naming conflicts</strong></summary>

```bash
# AZD מייצר שמות ייחודיים, אבל אם יש התנגשות:
azd down --force --purge

# אז יש לנסות שוב עם סביבה חדשה
azd env new dev-v2
azd up
```
</details>
