# תרומה ל-AZD למתחילים

תודה על העניין שלכם בתרומה לקורס AZD למתחילים! מסמך זה מספק קווים מנחים ומידע לתורמים.

## כיצד לתרום

### דיווח על בעיות

אם אתם מוצאים באגים, שגיאות, או יש לכם הצעות לשיפור:

1. **בדקו בעיות קיימות**: חפשו [בעיות קיימות](https://github.com/microsoft/azd-for-beginners/issues) כדי להימנע מכפילויות
2. **צור בעיה חדשה**: השתמשו בתבנית הבעיה המתאימה
3. **ספק פרטים**: כללו שלבים לשחזור, התנהגות צפויה, וצילומי מסך אם רלוונטי

### הצעת שינויים

1. **בצעו Fork של המאגר**: צרו Fork משלכם של הפרויקט
2. **צרו סניף (Branch)**: צרו סניף תכונה עבור השינויים שלכם
3. **בצעו שינויים**: עקבו אחרי תקני הקידוד והתיעוד שלנו
4. **בדקו את השינויים שלכם**: וודאו שכל הקישורים פועלים והתוכן מוצג כראוי
5. **שלחו Pull Request**: ספקו תיאור ברור של השינויים שלכם

### סוגי תרומות

נשמח לקבל את סוגי התרומות הבאים:

#### שיפורים בתוכן
- תקנו שגיאות הקלדה, שגיאות דקדוק ובעיות עיצוב
- שפרו את בהירות התיעוד הקיים
- עדכנו מידע מיושן
- הוסיפו הסברים או הקשרים חסרים

#### דוגמאות חדשות
- הוספת תרחישי פריסה מהעולם האמיתי
- צרו תבניות AZD חדשות למקרים נפוצים
- ספקו דוגמאות קוד ותירגולים נוספים

#### תמיכה בתרגום
- עזרו לשמור על תרגומים קיימים
- שפרו את איכות התרגום
- הוסיפו תרגומים לתוכן חדש

#### דוחות באגים
- דווחו על קישורים שבורים או דוגמאות קוד שגויות
- זהו אי-התאמות בתיעוד
- דגישו בעיות נגישות

## הנחיות לתרומה

### תקני תיעוד

- השתמשו בשפה ברורה וידידותית למתחילים
- כללו מטרות למידה ותוצאות צפויות
- ספקו הוראות שלב-אחר-שלב עם פלטים צפויים
- הוסיפו קישורי ניווט לתוכן קשור
- השתמשו בעיצוב וסגנון Markdown עקבי

### דוגמאות קוד

- וודאו שכל דוגמאות הקוד נבדקו ועובדות
- כללו הערות שמסבירות מושגים מרכזיים
- עקבו אחר שיטות העבודה המומלצות של Azure ו-AZD
- ספקו את הפלט הצפוי כאשר הדבר רלוונטי

### הודעות Commit

השתמשו בהודעות Commit ברורות ותיאוריות:
- `docs: fix typo in installation guide`
- `feat: add new container app example`
- `fix: correct broken link in chapter 2`
