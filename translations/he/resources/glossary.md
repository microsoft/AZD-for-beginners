## הקדמה

מילון מונחים מקיף זה מספק הגדרות למונחים, מושגים וקיצורים המשמשים ב-Azure Developer CLI ובפיתוח ענן של Azure. הפניה חיונית להבנת תיעוד טכני, פתרון תקלות ותקשורת אפקטיבית לגבי פרויקטים של azd ושירותי Azure.

## מטרות הלמידה

בעזרת מילון מונחים זה תוכלו:  
- להבין את טרמינולוגיית ומושגי Azure Developer CLI החיוניים  
- לשלוט באוצר מילים טכני של פיתוח בענן Azure  
- להתייחס ביעילות לטרמינולוגיית תשתית כקוד ופריסה  
- להבין שמות שירותי Azure, קיצורים ומטרותיהם  
- לקבל הגדרות לפתרון תקלות וטרמינולוגיית ניפוי שגיאות  
- ללמוד מושגים מתקדמים בארכיטקטורת ופיתוח Azure  

## תוצאות הלמידה

בהתייחסות קבועה למילון זה תוכלו:  
- לתקשר בצורה יעילה באמצעות טרמינולוגיית Azure Developer CLI נכונה  
- להבין טוב יותר תיעוד טכני והודעות שגיאה  
- לנווט בביטחון בשירותי Azure ובמושגים  
- לפתור בעיות באמצעות אוצר מילים טכני מתאים  
- לתרום לדיוני צוות בשפה טכנית מדויקת  
- להרחיב בצורה שיטתית את הידע שלכם בפיתוח ענן Azure  

## א

**ARM Template**  
תבנית Azure Resource Manager. פורמט תשתית כקוד מבוסס JSON המשמש להגדרה ופריסה דקלרטיבית של משאבי Azure.

**App Service**  
פלטפורמת PaaS של Azure לאירוח אפליקציות ווב, APIs של REST ו-backends למובייל ללא ניהול תשתית.

**Application Insights**  
שירות ניטור ביצועי אפליקציות (APM) של Azure המספק תובנות עמוקות על ביצועי האפליקציה, זמינות ושימוש.

**Azure CLI**  
ממשק שורת הפקודה לניהול משאבי Azure. משמש את azd לאימות וחלק מהפעולות.

**Azure Developer CLI (azd)**  
כלי שורת פקודה הממוקד במפתח המאיץ את תהליך הבנייה והפריסה של אפליקציות ל-Azure באמצעות תבניות ותשתית כקוד.

**azure.yaml**  
קובץ ההגדרות הראשי לפרויקט azd המגדיר שירותים, תשתית, וחיבורי פריסה.

**Azure Resource Manager (ARM)**  
שירות פריסה וניהול של Azure המספק שכבת ניהול ליצירה, עדכון ומחיקה של משאבים.

## ב

**Bicep**  
שפת DSL ייעודית שפותחה על ידי Microsoft לפריסת משאבי Azure. מספקת תחביר פשוט יותר מתבניות ARM תוך הידור ל-ARM.

**Build**  
תהליך הידור קוד מקור, התקנת תלותים והכנת אפליקציות לפריסה.

**Blue-Green Deployment**  
אסטרטגיית פריסה המשתמשת בשתי סביבות ייצור זהות (כחול וירוק) כדי למזער זמן השבתה וסיכון.

## ג

**Container Apps**  
שירות מכולות ללא שרת של Azure המאפשר הרצת אפליקציות מכולות מבלי לנהל תשתית מורכבת.

**CI/CD**  
אינטגרציה רציפה/פריסה רציפה. שיטות אוטומטיות לאינטגרציה של שינויים בקוד ופריסת אפליקציות.

**Cosmos DB**  
שירות מסדי נתונים רב-מודלים ומופץ עולמית של Azure עם SLA מקיפים לעומס, זמן תגובה, זמינות ועקביות.

**Configuration**  
הגדרות ופרמטרים השולטים בתפקוד האפליקציה ואפשרויות הפריסה.

## ד

**Deployment**  
תהליך התקנת וקביעת תצורה של אפליקציות ותלותיהן על תשתית יעד.

**Docker**  
פלטפורמה לפיתוח, משלוח והרצה של אפליקציות באמצעות טכנולוגיית מכולות.

**Dockerfile**  
קובץ טקסט המכיל הוראות לבניית תמונת מכולה ב-Docker.

## ה

**Environment**  
יעד פריסה המייצג מופע ספציפי של האפליקציה שלכם (כגון פיתוח, בדיקה, ייצור).

**Environment Variables**  
ערכי הגדרה המאוחסנים כזוגי מפתח-ערך לגישה בזמן ריצה.

**Endpoint**  
כתובת URL או כתובת רשת שאליה ניתן לגשת לאפליקציה או שירות.

## ו

**Function App**  
שירות חישוב ללא שרת של Azure המאפשר הרצת קוד מופעל על ידי אירועים ללא ניהול תשתית.

## ז

**GitHub Actions**  
פלטפורמת CI/CD המשולבת עם מאגרי GitHub לאוטומציה של זרימות עבודה.

**Git**  
מערכת ניהול גרסאות מבוזרת למעקב אחרי שינויים בקוד מקור.

## ח

**Hooks**  
סקריפטים או פקודות מותאמות אישית שרצות בנקודות מסוימות במחזור חיי הפריסה (לפני פרוביזיה, אחרי פרוביזיה, לפני פריסה, אחרי פריסה).

**Host**  
סוג שירות Azure שבו תתפרס האפליקציה (למשל appservice, containerapp, function).

## ט

**Infrastructure as Code (IaC)**  
פרקטיקה של הגדרה וניהול תשתית באמצעות קוד במקום תהליכים ידניים.

**Init**  
תהליך אתחול פרויקט azd חדש, בדרך כלל מתבנית קיימת.

## י

**JSON**  
JavaScript Object Notation. פורמט המרה נפוץ לקבצי הגדרות ותשובות API.

**JWT**  
JSON Web Token. תקן להעברת מידע מאובטח בין צדדים כאובייקט JSON.

## כ

**Key Vault**  
שירות Azure לאחסון מאובטח וניהול סודות, מפתחות ותעודות.

**Kusto Query Language (KQL)**  
שפת שאילתות לניתוח נתונים ב-Azure Monitor, Application Insights ושירותי Azure נוספים.

## ל

**Load Balancer**  
שירות שמפזר תעבורת רשת נכנסת על גבי מספר שרתים או מופעים.

**Log Analytics**  
שירות Azure לאיסוף, ניתוח ופירוש נתוני טלמטיה מסביבות ענן ומקומיות.

## מ

**Managed Identity**  
תכונה ב-Azure המספקת שירותי Azure עם זהות מנוהלת אוטומטית לאימות מול שירותים אחרים.

**Microservices**  
גישה ארכיטקטונית שבונה אפליקציות כאוסף של שירותים קטנים ועצמאיים.

**Monitor**  
פתרון ניטור מאוחד של Azure המאפשר תצפית מלאה על אפליקציות ותשתיות.

## נ

**Node.js**  
סביבת ריצה ב-JavaScript המבוססת על מנוע V8 של Chrome לפיתוח אפליקציות צד שרת.

**npm**  
מנהל חבילות עבור Node.js לניהול תלותים וחבילות.

## ס

**Output**  
ערכים שמוחזרים מפריסת תשתית שניתן להשתמש בהם באפליקציות או משאבים אחרים.

## ע

**Package**  
תהליך הכנת קוד אפליקציה ותלותיה לפריסה.

**Parameters**  
ערכי קלט המועברים לתבניות תשתית להתאמת הפריסות.

**PostgreSQL**  
מערכת מסדי נתונים יחסיים קוד פתוח הנתמכת כשירות מנוהל ב-Azure.

**Provisioning**  
תהליך יצירה והגדרת משאבי Azure שהוגדרו בתבניות תשתית.

## ק

**Quota**  
מגבלות על כמות המשאבים שניתן ליצור במנוי או באזור Azure.

## ר

**Resource Group**  
מיכל לוגי של משאבי Azure שמשותפים למחזור חיים, הרשאות ומדיניות.

**Resource Token**  
מחרוזת ייחודית שנוצרת על ידי azd להבטיח שמות משאבים ייחודיים בפריסות.

**REST API**  
סגנון ארכיטקטוני לעיצוב אפליקציות רשת באמצעות שיטות HTTP.

**Rollback**  
תהליך חזרה לגרסה קודמת של אפליקציה או תצורת תשתית.

## ש

**Service**  
רכיב של האפליקציה שלכם שמוגדר ב-azure.yaml (למשל, ממשק ווב, backend API, מסד נתונים).

**SKU**  
יחידת מלאי. מייצג שכבות שירות שונות או רמות ביצועים למשאבי Azure.

**SQL Database**  
שירות מסד נתונים יחסיים מנוהל של Azure מבוסס Microsoft SQL Server.

**Static Web Apps**  
שירות Azure לבניית ופריסת אפליקציות ווב מלאות מקוד מקור במאגרי קוד.

**Storage Account**  
שירות Azure המספק אחסון ענני לאובייקטי נתונים כולל blob, קבצים, תורים וטבלאות.

**Subscription**  
מיכל חשבון Azure שמכיל קבוצות משאבים ומשאבים, יחד עם ניהול חיוב וגישה.

## ת

**Template**  
מבנה פרויקט מראש המכיל קוד אפליקציה, הגדרות תשתית וקונפיגורציה לתרחישים נפוצים.

**Terraform**  
כלי קוד פתוח לתשתית כקוד התומך בספקי ענן שונים כולל Azure.

**Traffic Manager**  
מאזן עומס תעבורה מבוסס DNS של Azure לפיזור תעבורה על פני אזורי Azure עולמיים.

## ו׳ (U)

**URI**  
זיהוי אחיד למשאב. מחרוזת המזהה משאב מסוים.

**URL**  
מיקום אחיד למשאב. סוג URI המפרט היכן נמצא המשאב ואיך לגשת אליו.

## ו

**Virtual Network (VNet)**  
בלוק בניה בסיסי לרשתות פרטיות ב-Azure, המספק בידוד וסגמנטציה.

**VS Code**  
Visual Studio Code. עורך קוד פופולרי עם אינטגרציה מצוינת ל-Azure ו-azd.

## ש

**Webhook**  
פונקציית קריאה חוזרת HTTP המופעלת על ידי אירועים ספציפיים, נפוצה בצנרות CI/CD.

**What-if**  
תכונת Azure המציגה אילו שינויים יבוצעו בפריסה ללא ביצוע בפועל.

## י

**YAML**  
YAML אינה שפת סימון. תקן סריאליזציה לקריאת אדם המשמש לקבצי הגדרות כמו azure.yaml.

## ז

**Zone**  
מיקומים נפרדים פיזית באזור Azure המספקים רדונדנציה וזמינות גבוהה.

## מונחים בהקשר ספציפי

### מונחי פיתוח  
- **Hot Reload**: עדכון אוטומטי של אפליקציות במהלך פיתוח ללא הפעלה מחדש  
- **Build Pipeline**: תהליך אוטומטי של בנייה ובדיקה של קוד  
- **Deployment Slot**: סביבה זמנית בתוך App Service  
- **Environment Parity**: שמירת דמיון בין סביבות פיתוח, בדיקה וייצור  

### מונחי אבטחה  
- **Managed Identity**: תכונת Azure המציעה ניהול אוטומטי של אישורים  
- **Key Vault**: אחסון מאובטח לסודות, מפתחות ותעודות  
- **RBAC**: בקרת גישה מבוססת תפקידים למשאבי Azure  
- **Network Security Group**: חומת אש וירטואלית לשליטה על תעבורת רשת  

### מונחי ניטור  
- **Telemetry**: איסוף אוטומטי של מדידות ונתונים  
- **Application Performance Monitoring (APM)**: ניטור ביצועי תוכנה  
- **Log Analytics**: שירות לאיסוף וניתוח נתוני יומנים  
- **Alert Rules**: התראות אוטומטיות מבוססות מדדים או תנאים  

### מונחי פריסה  
- **Blue-Green Deployment**: אסטרטגיית פריסה ללא השבתה  
- **Canary Deployment**: פריסה הדרגתית לתת-קבוצה של משתמשים  
- **Rolling Update**: החלפה סדרתית של מופעי אפליקציה  
- **Rollback**: חזרה לגרסת אפליקציה קודמת
