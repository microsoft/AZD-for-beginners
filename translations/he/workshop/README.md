# סדנת AZD למפתחים של AI

ברוכים הבאים לסדנה מעשית ללימוד Azure Developer CLI (AZD) עם דגש על פריסת יישומי AI. סדנה זו עוזרת לכם לקבל הבנה מעשית של תבניות AZD ב-3 שלבים:

1. **גילוי** - למצוא את התבנית שמתאימה לכם.
1. **פריסה** - לפרוס ולאמת שהיא פועלת.
1. **התאמה אישית** - לשנות ולחזור על התהליך כדי להפוך אותה לשלכם!

במהלך הסדנה תכירו גם כלים וזרמי עבודה מרכזיים למפתחים, שיעזרו לכם לייעל את מסע הפיתוח מקצה לקצה.

<br/>

## מדריך מבוסס דפדפן

שיעורי הסדנה הם ב-Markdown. ניתן לנווט בהם ישירות ב-GitHub - או להפעיל תצוגה מקדימה בדפדפן כפי שמוצג בתמונה למטה.

כדי להשתמש באפשרות זו - בצעו fork של המאגר לפרופיל שלכם, והפעילו את GitHub Codespaces. ברגע שהטרמינל של VS Code פעיל, הקלידו את הפקודה הזו:

תצוגת הדפדפן הזו פועלת ב-GitHub Codespaces, מיכלי פיתוח, או עותק מקומי עם Python ו-MkDocs מותקנים.

```bash title="" linenums="0"
mkdocs serve > /dev/null 2>&1 &
```

כעבור מספר שניות, תופיע תיבת דו-שיח קופצת. בחרו באפשרות `Open in browser`. המדריך מבוסס האינטרנט ייפתח עכשיו בכרטיסיית דפדפן חדשה. כמה יתרונות של תצוגה מקדימה זו:

1. **חיפוש מובנה** - מצאו מילות מפתח או שיעורים במהירות.
1. **אייקון העתקה** - הצביעו מעל קטעי קוד כדי לראות אפשרות זו.
1. **החלפת ערכת נושא** - החליפו בין ערכות נושא כהות ובהירות.
1. **קבלו עזרה** - לחצו על אייקון Discord בכותרת התחתונה כדי להצטרף!

## מה תלמדו

חשבו על תבנית AZD כסביבת למידה לחקור יכולות וכלים שונים לפיתוח מקצה לקצה ב-Microsoft Foundry. בסיום הסדנה, תהיה לכם הבנה אינטואיטיבית של כלים ורעיונות שונים בהקשר זה.

| רעיון  | מטרה |
|:---|:---|
| **Azure Developer CLI** | להבין פקודות וכל זרמי עבודה של הכלי |
| **תבניות AZD**| להבין מבנה וקונפיגורציה של הפרויקט |
| **Azure AI Agent**| לבצע פריסה וניהול פרויקט Microsoft Foundry |
| **Azure AI Search**| לאפשר הנדסת הקשר עם סוכנים |
| **תצפית**| לחקור מעקב, ניטור והערכות |
| **Red Teaming**| לחקור בדיקות עוינות ומניעתן |

## מבנה הסדנה

הסדנה בנויה לקחת אתכם למסע מגילוי תבניות, לפריסה, פירוק, והתאמה אישית - תוך שימוש בתבנית הפתיחה הרשמית [Getting Started with AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) כבסיס.

### [מודול 1: בחירת תבנית AI](docs/instructions/1-Select-AI-Template.md) (30 דקות)

- מהן תבניות AI?
- איפה אפשר למצוא תבניות AI?
- איך ניתן להתחיל לבנות סוכני AI?
- **מעבדה**: התחלה מהירה ב-Codespaces או מיכל פיתוח

### [מודול 2: אימות תבנית AI](docs/instructions/2-Validate-AI-Template.md) (30 דקות)

- מהי אדריכלות תבנית ה-AI?
- מהו זרם העבודה של פיתוח AZD?
- איך ניתן לקבל עזרה בפיתוח AZD?
- **מעבדה**: לפרוס ולאמת תבנית סוכני AI

### [מודול 3: פירוק תבנית AI](docs/instructions/3-Deconstruct-AI-Template.md) (30 דקות)

- לחקור את הסביבה שלך בתיקייה `.azure/` 
- לחקור את הגדרת המשאבים בתיקייה `infra/` 
- לחקור את קונפיגורציית AZD ב-`azure.yaml`s
- **מעבדה**: לשנות משתני סביבה ולפרוס מחדש

### [מודול 4: הגדרת תבנית AI](docs/instructions/4-Configure-AI-Template.md) (30 דקות)
- לחקור: יצירת תוכן משולבת (Retrieval Augmented Generation)
- לחקור: הערכת סוכנים ו-Red Teaming
- לחקור: מעקב וניטור
- **מעבדה**: לחקור סוכן AI + תצפית

### [מודול 5: התאמת תבנית AI](docs/instructions/5-Customize-AI-Template.md) (30 דקות)
- להגדיר: PRD עם דרישות תרחיש
- להגדיר: משתני סביבה ל-AZD
- ליישם: לולאות חיבור למשימות נוספות
- **מעבדה**: להתאים את התבנית לתרחיש שלי

### [מודול 6: פירוק התשתית](docs/instructions/6-Teardown-Infrastructure.md) (30 דקות)
- סיכום: מהן תבניות AZD?
- סיכום: למה להשתמש ב-Azure Developer CLI?
- שלבים הבאים: לנסות תבנית אחרת!
- **מעבדה**: לפרק תשתיות ולנקות
