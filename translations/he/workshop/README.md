# סדנת AZD למפתחי AI

ברוכים הבאים לסדנה המעשי ללמידת Azure Developer CLI (AZD) עם דגש על פריסת יישומי AI. סדנה זו תעזור לכם להבין במעשה את תבניות AZD ב-3 שלבים:

1. **גילוי** - מצאו את התבנית שמתאימה לכם.
1. **פריסה** - פרסו ואמתו שהיא עובדת
1. **התאמה אישית** - שנו וחזרו על התהליך כדי להפוך אותה לשלכם!

במהלך הסדנה גם תחשפו לכלי הפיתוח המרכזיים ולזרימות העבודה, שיסייעו לכם לייעל את מסע הפיתוח מקצה אל קצה.

<br/>

## מדריך מבוסס דפדפן

שיעורי הסדנה נמצאים ב-Markdown. תוכלו לגלוש בהם ישירות בגיטהאב - או להפעיל תצוגה מקדימה מבוססת דפדפן כפי שמוצג בצילום המסך למטה.

![Workshop](../../../translated_images/he/workshop.75906f133e6f8ba0.webp)

כדי להשתמש באפשרות זו - צרו פורק של המאגר לפרופיל שלכם, והפעילו GitHub Codespaces. לאחר שהטרמינל של VS Code פעיל, הקלידו את הפקודה הבאה:

```bash title="" linenums="0"
mkdocs serve > /dev/null 2>&1 &
```

כעבור כמה שניות, תראו תיבת דו-שיח קופצת. בחרו באפשרות `Open in browser`. המדריך מבוסס האינטרנט ייפתח עכשיו בכרטיסייה חדשה בדפדפן. כמה יתרונות בתצוגה מקדימה זו:

1. **חיפוש מובנה** - מצאו מילות מפתח או שיעורים במהירות.
1. **אייקון העתקה** - לחצו על קטעי קוד כדי לראות אפשרות זו
1. **החלפת נושא** - החליפו בין ערכות נושא בהירות וכהות
1. **קבלת עזרה** - לחצו על אייקון Discord בתחתית כדי להצטרף!

## סקירת הסדנה

**רמה:** מתחילים עד בינוניים

## מה תלמדו

חשבו על תבנית AZD כסביבת למידה לחקור יכולות וכלים שונים לפיתוח מקצה לקצה על Microsoft Foundry. בסיום הסדנה, אמורה להיות לכם הבנה אינטואיטיבית של מגוון כלים ומושגים בהקשר זה.

| מושג  | מטרה |
|:---|:---|
| **Azure Developer CLI** | להבין פקודות וזרימות עבודה |
| **תבניות AZD**| להבין מבנה פרויקט והגדרות |
| **סוכן AI של Azure**| להקצות ולפרוס פרויקט Microsoft Foundry |
| **Azure AI Search**| לאפשר הנדסת הקשר עם סוכנים |
| **תצפיתיות**| לבדוק מעקבים, ניטור והערכות |
| **Red Teaming**| לבדוק ניסויים עוינים ופיצויים |

## מבנה הסדנה

הסדנה בנויה לקחת אתכם במסע מגילוי תבנית, דרך פריסה, פירוק והתאמה אישית - על בסיס תבנית ההתחלה הרשמית [Getting Started with AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents).

### [מודול 1: בחירת תבנית AI](docs/instructions/1-Select-AI-Template.md) (30 דקות)

- מה הן תבניות AI?
- איפה אפשר למצוא תבניות AI?
- איך להתחיל לבנות סוכני AI?
- **מעבדה**: התחלה מהירה עם GitHub Codespaces

### [מודול 2: אימות תבנית AI](docs/instructions/2-Validate-AI-Template.md) (30 דקות)

- מהי ארכיטקטורת תבנית AI?
- מהי זרימת הפיתוח של AZD?
- איך לקבל עזרה בפיתוח ב-AZD?
- **מעבדה**: פרוסו ואמתו תבנית סוכני AI

### [מודול 3: פירוק תבנית AI](docs/instructions/3-Deconstruct-AI-Template.md) (30 דקות)

- סיירו בסביבה שלכם ב-`.azure/`
- סיירו בהגדרת המשאבים ב-`infra/`
- סיירו בקונפיגורציית AZD בקבצי `azure.yaml`
- **מעבדה**: שנו משתני סביבה ופרסו מחדש

### [מודול 4: קביעת תצורת תבנית AI](docs/instructions/4-Configure-AI-Template.md) (30 דקות)
- סיירו: הפקה מועשרת באמצעות אחזור מידע
- סיירו: הערכת סוכן ו-Red Teaming
- סיירו: מעקב וניטור
- **מעבדה**: חקר סוכן AI + תצפיתיות

### [מודול 5: התאמת תבנית AI](docs/instructions/5-Customize-AI-Template.md) (30 דקות)
- הגדר: דרישות מוצר (PRD) עם תרחישי שימוש
- קבע: משתני סביבה ל-AZD
- יישם: חבילות lifecycle למשימות נוספות
- **מעבדה**: התאם את התבנית לתרחיש שלי

### [מודול 6: פירוק תשתית](docs/instructions/6-Teardown-Infrastructure.md) (30 דקות)
- סיכום: מהן תבניות AZD?
- סיכום: למה להשתמש ב-Azure Developer CLI?
- צעדים הבאים: נסה תבנית שונה!
- **מעבדה**: סילוק תשתית וניקוי
