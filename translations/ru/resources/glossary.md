Этот подробный глоссарий содержит определения терминов, понятий и аббревиатур, используемых в Azure Developer CLI и разработке в облаке Azure. Важный справочник для понимания технической документации, решения проблем и эффективного общения о проектах azd и сервисах Azure.

## A

**ARM Template**  
Шаблон Azure Resource Manager. Формат Infrastructure as Code на базе JSON для декларативного определения и развертывания ресурсов Azure.

**App Service**  
Предложение Azure как платформы как услуги (PaaS) для хостинга веб-приложений, REST API и мобильных бекендов без управления инфраструктурой.

**Application Insights**  
Сервис мониторинга производительности приложений Azure (APM), предоставляющий глубокое понимание производительности, доступности и использования приложений.

**Azure CLI**  
Интерфейс командной строки для управления ресурсами Azure. Используется azd для аутентификации и некоторых операций.

**Azure Developer CLI (azd)**  
Ориентированный на разработчиков командный инструмент, ускоряющий процесс создания и развертывания приложений в Azure с использованием шаблонов и Infrastructure as Code.

**azure.yaml**  
Основной конфигурационный файл проекта azd, в котором определяются сервисы, инфраструктура и хуки развертывания.

**Azure Resource Manager (ARM)**  
Сервис развертывания и управления Azure, обеспечивающий слой управления для создания, обновления и удаления ресурсов.

## B

**Bicep**  
Предметно-ориентированный язык (DSL), разработанный Microsoft для развертывания ресурсов Azure. Обеспечивает более простой синтаксис по сравнению с ARM-шаблонами, при этом компилируется в ARM.

**Build**  
Процесс компиляции исходного кода, установки зависимостей и подготовки приложений к развертыванию.

**Blue-Green Deployment**  
Стратегия развертывания, использующая две идентичные производственные среды (синюю и зелёную) для минимизации простоев и рисков.

## C

**Container Apps**  
Сервис безсерверных контейнеров Azure, позволяющий запускать контейнеризированные приложения без управления сложной инфраструктурой.

**CI/CD**  
Непрерывная интеграция/непрерывное развертывание. Автоматизированные практики интеграции изменений кода и развертывания приложений.

**Cosmos DB**  
Глобально распределённый многомодельный сервис базы данных Azure, предоставляющий всеобъемлющие соглашения об уровне обслуживания по пропускной способности, задержкам, доступности и согласованности.

**Configuration**  
Настройки и параметры, управляющие поведением приложения и опциями развертывания.

## D

**Deployment**  
Процесс установки и настройки приложений и их зависимостей на целевой инфраструктуре.

**Docker**  
Платформа для разработки, доставки и запуска приложений с использованием контейнеризации.

**Dockerfile**  
Текстовый файл с инструкциями для сборки образа Docker-контейнера.

## E

**Environment**  
Целевая среда развертывания, представляющая собой конкретный экземпляр вашего приложения (например, разработка, тестирование, продакшен).

**Environment Variables**  
Конфигурационные значения, хранящиеся в виде пар ключ-значение, к которым приложения могут обращаться во время выполнения.

**Endpoint**  
URL или сетевой адрес, по которому можно получить доступ к приложению или сервису.

## F

**Function App**  
Сервис безсерверных вычислений Azure, который позволяет запускать код, реагирующий на события, без управления инфраструктурой.

## G

**GitHub Actions**  
Платформа CI/CD, интегрированная с репозиториями GitHub для автоматизации рабочих процессов.

**Git**  
Распределённая система контроля версий, используемая для отслеживания изменений в исходном коде.

## H

**Hooks**  
Пользовательские скрипты или команды, которые выполняются в определённые моменты жизненного цикла развертывания (preprovision, postprovision, predeploy, postdeploy).

**Host**  
Тип сервиса Azure, куда будет развернуто приложение (например, appservice, containerapp, function).

## I

**Infrastructure as Code (IaC)**  
Практика определения и управления инфраструктурой с помощью кода вместо ручных процессов.

**Init**  
Процесс инициализации нового проекта azd, обычно из шаблона.

## J

**JSON**  
JavaScript Object Notation. Формат обмена данными, широко используемый для конфигурационных файлов и API-ответов.

**JWT**  
JSON Web Token. Стандарт для безопасной передачи информации между сторонами в виде JSON-объекта.

## K

**Key Vault**  
Сервис Azure для безопасного хранения и управления секретами, ключами и сертификатами.

**Kusto Query Language (KQL)**  
Язык запросов, используемый для анализа данных в Azure Monitor, Application Insights и других сервисах Azure.

## L

**Load Balancer**  
Сервис, распределяющий входящий сетевой трафик между несколькими серверами или экземплярами.

**Log Analytics**  
Сервис Azure для сбора, анализа и реагирования на телеметрические данные из облачных и локальных сред.

## M

**Managed Identity**  
Функция Azure, предоставляющая службам Azure автоматически управляемую идентификацию для аутентификации в других службах Azure.

**Microservices**  
Архитектурный подход, при котором приложения строятся как коллекция небольших, независимых сервисов.

**Monitor**  
Унифицированное решение Azure для мониторинга, обеспечивающее полный обзор приложений и инфраструктуры.

## N

**Node.js**  
Среда выполнения JavaScript, построенная на движке V8 Chrome, для создания серверных приложений.

**npm**  
Менеджер пакетов для Node.js, управляющий зависимостями и пакетами.

## O

**Output**  
Значения, возвращаемые после развертывания инфраструктуры, которые могут использоваться приложениями или другими ресурсами.

## P

**Package**  
Процесс подготовки кода приложения и зависимостей к развертыванию.

**Parameters**  
Входные значения, передаваемые в инфраструктурные шаблоны для настройки развертываний.

**PostgreSQL**  
Система управления реляционными базами данных с открытым исходным кодом, поддерживаемая в Azure как управляемый сервис.

**Provisioning**  
Процесс создания и настройки ресурсов Azure, определённых в инфраструктурных шаблонах.

## Q

**Quota**  
Ограничения на количество ресурсов, которые можно создать в подписке или регионе Azure.

## R

**Resource Group**  
Логический контейнер для ресурсов Azure, которые имеют одинаковый жизненный цикл, разрешения и политики.

**Resource Token**  
Уникальная строка, генерируемая azd, для обеспечения уникальности имён ресурсов при развертываниях.

**REST API**  
Архитектурный стиль для проектирования сетевых приложений с использованием методов HTTP.

**Rollback**  
Процесс возврата к предыдущей версии приложения или конфигурации инфраструктуры.

## S

**Service**  
Компонент вашего приложения, определённый в azure.yaml (например, веб-фронтенд, API-бэкенд, база данных).

**SKU**  
Единица хранения запасов. Представляет разные уровни сервиса или производительности для ресурсов Azure.

**SQL Database**  
Управляемый реляционный сервис базы данных Azure на базе Microsoft SQL Server.

**Static Web Apps**  
Сервис Azure для создания и развертывания полностековых веб-приложений из репозиториев исходного кода.

**Storage Account**  
Сервис Azure, обеспечивающий облачное хранение для объектов данных, включая блобы, файлы, очереди и таблицы.

**Subscription**  
Контейнер учетной записи Azure, содержащий группы ресурсов и ресурсы с соответствующим управлением биллингом и доступом.

## T

**Template**  
Готовая структура проекта, содержащая код приложения, определения инфраструктуры и конфигурации для типовых сценариев.

**Terraform**  
Инструмент Infrastructure as Code с открытым исходным кодом, поддерживающий множество облачных провайдеров, включая Azure.

**Traffic Manager**  
DNS-базированный балансировщик нагрузки Azure для распределения трафика по глобальным регионам Azure.

## U

**URI**  
Унифицированный идентификатор ресурса. Строка, которая идентифицирует конкретный ресурс.

**URL**  
Унифицированный локатор ресурса. Тип URI, который указывает, где расположен ресурс и как к нему получить доступ.

## V

**Virtual Network (VNet)**  
Фундаментальный строительный блок приватных сетей Azure, обеспечивающий изоляцию и сегментацию.

**VS Code**  
Visual Studio Code. Популярный редактор кода с отличной интеграцией Azure и azd.

## W

**Webhook**  
HTTP-обратный вызов, вызываемый определёнными событиями, часто используемый в CI/CD пайплайнах.

**What-if**  
Функция Azure, которая показывает, какие изменения будут сделаны при развертывании без фактического выполнения.

## Y

**YAML**  
YAML — это не язык разметки. Читаемый человеком стандарт сериализации данных, используемый для конфигурационных файлов, таких как azure.yaml.

## Z

**Zone**  
Физически отдельные локации внутри региона Azure, обеспечивающие избыточность и высокую доступность.

## Термины по контекстам

### Термины разработки
- **Hot Reload**: Автоматическое обновление приложений во время разработки без перезапуска
- **Build Pipeline**: Автоматизированный процесс сборки и тестирования кода
- **Deployment Slot**: Стейджинг-среда внутри App Service
- **Environment Parity**: Поддержание сходства сред разработки, тестирования и продакшена

### Термины безопасности
- **Managed Identity**: Функция Azure, обеспечивающая автоматическое управление учетными данными
- **Key Vault**: Безопасное хранилище для секретов, ключей и сертификатов
- **RBAC**: Управление доступом к ресурсам Azure на основе ролей
- **Network Security Group**: Виртуальный файервол для контроля сетевого трафика

### Термины мониторинга
- **Telemetry**: Автоматизированный сбор измерений и данных
- **Application Performance Monitoring (APM)**: Мониторинг производительности программного обеспечения
- **Log Analytics**: Сервис для сбора и анализа логов
- **Alert Rules**: Автоматические уведомления на основе метрик или условий

### Термины развертывания
- **Blue-Green Deployment**: Стратегия развертывания без простоев
- **Canary Deployment**: Постепенное внедрение для части пользователей
- **Rolling Update**: Поочерёдная замена экземпляров приложения
- **Rollback**: Откат к предыдущей версии приложения
