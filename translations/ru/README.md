## 🆕 Что нового в azd сегодня

> 📌 Этот курс проверен на версии **`azd 1.25.6`** (июнь 2026). Выполните `azd version` для проверки текущей версии и `azd upgrade` для обновления.

Azure Developer CLI вышел за рамки традиционных веб-приложений и API. Сегодня azd — это единый инструмент для развертывания **любых** приложений в Azure — включая приложения с ИИ и интеллектуальных агентов.

Что это значит для вас:

- **ИИ-агенты теперь являются полноценными нагрузками azd.** Вы можете инициализировать, развертывать и управлять проектами ИИ-агентов с помощью привычного рабочего процесса `azd init` → `azd up`.
- **Полный жизненный цикл агента из CLI.** Расширение `azure.ai.agents` теперь охватывает весь путь: `azd ai agent init` для создания каркаса, `azd ai agent invoke` для тестирования (с выводом времени ответа), `azd ai agent eval generate` и `azd ai agent optimize` для измерения и улучшения качества, и `azd ai agent delete` для очистки.
- **Больше строительных блоков ИИ.** Новые предварительные расширения — `azure.ai.skills` и `azure.ai.connections` — позволяют управлять повторно используемыми навыками агентов и подключениями Foundry прямо через azd.
- **Интеграция Microsoft Foundry** обеспечивает развертывание моделей, хостинг агентов и настройку ИИ-сервисов непосредственно в экосистеме шаблонов azd.
- **Ещё более плавная повседневная работа.** Последние версии сделали `azd init` идемпотентным (безопасно запускать несколько раз), `azd auth login` теперь автоматически очищает устаревшие токены, а также добавлена удобная подсказка при первом запуске `azd tool`.
- **Основной рабочий процесс не изменился.** Независимо от того, разворачиваете ли вы приложение для списка дел, микросервис или мультиагентное ИИ-решение, команды остаются одинаковыми.

> **Примечание для пользователей Aspire:** Microsoft теперь называет продукт просто **Aspire** (ранее ".NET Aspire"). Поддержка azd для Aspire не изменилась — только название обновлено.

Если вы уже пользовались azd, поддержка ИИ — это естественное расширение, а не отдельный инструмент или продвинутый трек. Если вы начинаете с нуля, вы освоите один рабочий процесс, который подходит для всего.

## 🚀 Что такое Azure Developer CLI (azd)?

**Azure Developer CLI (azd)** — это дружелюбный к разработчику инструмент командной строки, который упрощает развертывание приложений в Azure. Вместо ручного создания и подключения десятков ресурсов Azure вы можете развернуть целое приложение одной командой.

### Магия команды `azd up`

```bash
# Эта единая команда делает всё:
# ✅ Создаёт все ресурсы Azure
# ✅ Настраивает сеть и безопасность
# ✅ Собирает код вашего приложения
# ✅ Разворачивает в Azure
# ✅ Даёт вам рабочий URL
azd up
```

**Вот и всё!** Никаких кликов в Azure Portal, никаких сложных ARM-шаблонов, которые нужно сначала изучить, никакой ручной настройки — только рабочие приложения в Azure.

## ❓ Azure Developer CLI против Azure CLI: в чём разница?

Это самый частый вопрос у начинающих. Вот простой ответ:

| Возможность | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|-------------|---------------------|--------------------------------|
| **Назначение** | Управление отдельными ресурсами Azure | Развертывание готовых приложений |
| **Подход** | Ориентирован на инфраструктуру | Ориентирован на приложения |
| **Пример** | `az webapp create --name myapp...` | `azd up` |
| **Кривая обучения** | Нужно знать сервисы Azure | Нужно знать своё приложение |
| **Лучше для** | DevOps, инфраструктура | Разработчики, создание прототипов |

### Простая аналогия

- **Azure CLI** — это как иметь все инструменты для строительства дома — молотки, пилы, гвозди. Вы можете построить всё, но нужно знать строительство.
- **Azure Developer CLI** — это как нанять подрядчика — вы описываете, что хотите, а он занимается строительством.

### Когда использовать каждый

| Сценарий | Используйте это |
|----------|----------------|
| "Хочу быстро развернуть веб-приложение" | `azd up` |
| "Мне нужно создать только учётную запись хранилища" | `az storage account create` |
| "Я создаю полноценное ИИ-приложение" | `azd init --template azure-search-openai-demo` |
| "Мне нужно отладить конкретный ресурс Azure" | `az resource show` |
| "Хочу развертывание в продакшене за минуты" | `azd up --environment production` |

### Они работают вместе!

AZD использует Azure CLI под капотом. Вы можете использовать оба:
```bash
# Разверните приложение с помощью AZD
azd up

# Затем точно настройте конкретные ресурсы с помощью Azure CLI
az webapp config set --name myapp --always-on true
```

## 🎯 Начало работы в 3 шага

Перед началом убедитесь, что ваша машина готова для выбранного шаблона:

**Windows:**
```powershell
.\validate-setup.ps1
```

**macOS / Linux:**
```bash
bash ./validate-setup.sh
```

Если какой-то проверочный шаг не пройден, сначала исправьте это, затем продолжайте быстрый старт.

### Шаг 1: Установите AZD (2 минуты)

**Windows:**
```powershell
winget install microsoft.azd
```

**macOS:**
```bash
brew tap azure/azd && brew install azd
```

**Linux:**
```bash
curl -fsSL https://aka.ms/install-azd.sh | bash
```

### Шаг 2: Аутентификация для AZD

```bash
# Необязательно, если вы планируете непосредственно использовать команды Azure CLI в этом курсе
az login

# Требуется для рабочих процессов AZD
azd auth login
```

Если не уверены, что выбрать, следуйте полному процессу настройки в [Установка и настройка](docs/chapter-01-foundation/installation.md#authentication-setup).

### Шаг 3: Разверните свое первое приложение

```bash
# Инициализировать из шаблона
azd init --template todo-nodejs-mongo

# Развернуть в Azure (создаёт всё!)
azd up
```

**🎉 Готово!** Ваше приложение теперь работает в Azure.

### Очистка (не забудьте!)

```bash
# Remove all resources when done experimenting
azd down --force --purge
```

## Обзор курса

Освойте Azure Developer CLI (azd) через структурированные главы, разработанные для пошагового обучения. **Особое внимание уделено развертыванию AI-приложений с интеграцией Microsoft Foundry.**
### Почему этот курс необходим современным разработчикам

Основываясь на мнениях сообщества Microsoft Foundry в Discord, **45% разработчиков хотят использовать AZD для рабочих нагрузок AI**, но сталкиваются с трудностями:
- Сложные многосервисные AI-архитектуры
- Лучшие практики развертывания AI в производственной среде  
- Интеграция и настройка сервисов Azure AI
- Оптимизация затрат на AI-нагрузки
- Поиск и устранение проблем, специфичных для развертывания AI

### Цели обучения

Пройдя этот структурированный курс, вы сможете:
- **Освоить основы AZD**: ключевые концепции, установка и настройка

- **Реализовать инфраструктуру как код**: управлять ресурсами Azure с помощью шаблонов Bicep
- **Устранять неполадки развертываний**: решать распространённые проблемы и отлаживать ошибки
- **Оптимизировать для продакшена**: безопасность, масштабирование, мониторинг и управление затратами

## Перед началом: учетные записи, доступ и предпосылки

Перед началом главы 1 убедитесь, что у вас есть следующее. Позднее в этом руководстве установка предполагает, что эти базовые моменты уже выполнены.

- **Подписка Azure**: можно использовать существующую подписку с работы или личный аккаунт, либо создать [бесплатную пробную подписку](https://aka.ms/azurefreetrial) для начала.
- **Разрешение на создание ресурсов Azure**: для большинства упражнений вам потребуется как минимум доступ **Contributor** к целевой подписке или группе ресурсов. В некоторых главах предполагается возможность создавать группы ресурсов, управляемые идентичности и назначения RBAC.
- [**Аккаунт GitHub**](https://github.com): полезен для форка репозитория, отслеживания собственных изменений и использования GitHub Codespaces на семинаре.
- **Необходимые локальные инструменты для шаблонов**: некоторые шаблоны требуют локальных инструментов, таких как Node.js, Python, Java или Docker. Запустите валидатор установки перед началом, чтобы заранее выявить недостающие инструменты.
- **Базовое владение терминалом**: не нужно быть экспертом, но следует уверенно выполнять команды типа `git clone`, `azd auth login` и `azd up`.

> **Работаете в корпоративной подписке?**  
> Если ваша среда Azure управляется администратором, заранее убедитесь, что у вас есть право развертывать ресурсы в выбранной подписке или группе ресурсов. Если нет, попросите песочницу или доступ Contributor перед началом.

> **Новичок в Azure?**  
> Начните с бесплатной пробной подписки или тарифа pay-as-you-go на https://aka.ms/azurefreetrial, чтобы выполнить упражнения полностью без задержек на уровне соглашений арендатора.

- AI Search (базовый уровень): $75/мес  
- Container Apps (оплата по потреблению): $0-20/мес  
- Хранение (стандарт): $1-5/мес  

**Продакшн-среда (примерно $300-$3500+/мес):**  
- Модели Microsoft Foundry (PTU для стабильной производительности): $3000+/мес ИЛИ оплата по потреблению при большом объеме  
- AI Search (стандарт): $250/мес  
- Container Apps (выделенный): $50-100/мес  
- Application Insights: $5-50/мес  
- Хранение (премиум): $10-50/мес  

**💡 Советы по оптимизации затрат:**  
- Используйте **бесплатные уровни** моделей Microsoft Foundry для обучения (включено 50,000 токенов Azure OpenAI)  
- Выполняйте `azd down`, чтобы освобождать ресурсы, когда не разрабатываете  
- Начинайте с оплаты по потреблению, PTU используйте только для продакшена  
- Используйте `azd provision --preview` для оценки затрат перед развертыванием  
- Включайте автоскейлинг: платите только за фактическое использование  

**Мониторинг затрат:**  
```bash
# Проверьте предполагаемые ежемесячные затраты
azd provision --preview

# Отслеживайте фактические затраты в портале Azure
az consumption budget list --resource-group <your-rg>
```

## 📖 Глубокое погружение: Возможности AZD

Помимо базового функционала, AZD предлагает мощные возможности для продакшен-развертываний:

- **Развертывания по шаблонам** - Использование готовых шаблонов для распространённых приложений  
- **Инфраструктура как код** - Управление Azure-ресурсами с помощью Bicep или Terraform  
- **Интегрированные рабочие процессы** - Бесшовное создание, развертывание и мониторинг приложений  
- **Дружественность для разработчиков** - Оптимизировано для продуктивности и удобства разработчиков

### **AZD + Microsoft Foundry: Идеально для AI-развертываний**

**Почему AZD для AI-решений?** AZD решает основные сложности разработчиков AI:

- **Готовые AI-шаблоны** - Преднастроенные шаблоны для Microsoft Foundry Models, Azure AI Services и ML нагрузок  
- **Безопасность AI-развертываний** - Встроенные шаблоны безопасности для AI-сервисов, ключей API и моделей  
- **Продакшен-паттерны AI** - Лучшие практики для масштабируемых, экономичных AI-приложений  
- **Конечные AI рабочие процессы** - От разработки моделей до развертывания с мониторингом  
- **Оптимизация затрат** - Умное выделение ресурсов и масштабирование AI-нагрузок  
- **Интеграция с Microsoft Foundry** - Бесшовный доступ к каталогу моделей и конечным точкам Foundry

## 🔧 Быстрое руководство по устранению неполадок

**Распространённые проблемы для новичков и их быстрые решения:**

<details>
<summary><strong>❌ "azd: команда не найдена"</strong></summary>

```bash
# Сначала установите AZD
# Windows (PowerShell):
winget install microsoft.azd

# macOS:
brew tap azure/azd && brew install azd

# Linux:
curl -fsSL https://aka.ms/install-azd.sh | bash

# Проверьте установку
azd version
```
</details>

<details>
<summary><strong>❌ "Подписка не найдена" или "Подписка не установлена"</strong></summary>

```bash
# Список доступных подписок
az account list --output table

# Установить подписку по умолчанию
az account set --subscription "<subscription-id-or-name>"

# Установить для среды AZD
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# Проверить
az account show
```
</details>

<details>
<summary><strong>❌ "InsufficientQuota" или "Превышена квота"</strong></summary>

```bash
# Попробуйте другой регион Azure
azd env set AZURE_LOCATION "westus2"
azd up

# Или используйте меньшие SKU в разработке
# Отредактируйте infra/main.parameters.json:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>

<details>
<summary><strong>❌ "azd up" прерывается посередине</strong></summary>

```bash
# Вариант 1: Очистить и повторить попытку
azd down --force --purge
azd up

# Вариант 2: Просто исправить инфраструктуру
azd provision

# Вариант 3: Проверить подробный статус
azd show

# Вариант 4: Проверить логи в Azure Monitor
azd monitor --logs
```
</details>

<details>
<summary><strong>❌ "Ошибка аутентификации" или "Срок действия токена истек"</strong></summary>

```bash
# Повторная аутентификация для AZD
azd auth logout
azd auth login

# По желанию: обновите Azure CLI, если вы используете команды az
az logout
az login

# Проверить аутентификацию
az account show
```
</details>

<details>
<summary><strong>❌ "Ресурс уже существует" или конфликты имен</strong></summary>

```bash
# AZD генерирует уникальные имена, но в случае конфликта:
azd down --force --purge

# Затем повторите попытку с новой средой
azd env new dev-v2
azd up
```
</details>

<details>
<summary><strong>❌ Развертывание шаблона занимает слишком много времени</strong></summary>

**Нормальное время ожидания:**
- Простое веб-приложение: 5-10 минут
- Приложение с базой данных: 10-15 минут
- AI-приложения: 15-25 минут (провизирование OpenAI происходит медленно)

```bash
# Проверить прогресс
azd show

# Если застряли более 30 минут, проверьте портал Azure:
azd monitor --overview
# Ищите неудачные развертывания
```
</details>

<details>
<summary><strong>❌ "Доступ запрещён" или "Запрещено"</strong></summary>

```bash
# Проверьте свою роль в Azure
az role assignment list --assignee $(az account show --query user.name -o tsv)

# Вам нужна роль не ниже "Участник"
# Попросите администратора Azure предоставить:
# - Участник (для ресурсов)
# - Администратор доступа пользователей (для назначений ролей)
```
</details>

<details>
<summary><strong>❌ Не удаётся найти URL развернутого приложения</strong></summary>

```bash
# Показать все конечные точки сервиса
azd show

# Или открыть портал Azure
azd monitor

# Проверить конкретный сервис
azd env get-values
# Найти переменные *_URL
```
</details>
