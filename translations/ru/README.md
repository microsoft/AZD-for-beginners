## 🆕 Что нового в azd сегодня

Azure Developer CLI вышел за рамки традиционных веб-приложений и API. Сейчас azd — это универсальный инструмент для развертывания **любых** приложений в Azure — включая приложения с ИИ и интеллектуальных агентов.

Что это значит для вас:

- **ИИ-агенты теперь полноценные рабочие нагрузки azd.** Вы можете инициализировать, развертывать и управлять проектами ИИ-агентов с помощью уже знакомого вам конвейера `azd init` → `azd up`.
- **Интеграция Microsoft Foundry** добавляет развертывание моделей, хостинг агентов и конфигурацию ИИ-сервисов напрямую в экосистему шаблонов azd.
- **Основной рабочий процесс не изменился.** Независимо от того, разворачиваете ли вы todo-приложение, микросервис или мультиагентное ИИ-решение, команды остаются одинаковыми.

Если вы уже использовали azd, поддержка ИИ — это естественное расширение, а не отдельный инструмент или продвинутый курс. Если вы только начинаете, вы выучите один рабочий процесс для всего.

## 🚀 Что такое Azure Developer CLI (azd)?

**Azure Developer CLI (azd)** — удобный для разработчиков интерфейс командной строки, который упрощает развертывание приложений в Azure. Вместо ручного создания и связывания десятков ресурсов Azure вы можете развернуть целое приложение одной командой.

### Волшебство `azd up`

```bash
# Одна команда делает всё:
# ✅ Создаёт все ресурсы Azure
# ✅ Настраивает сеть и безопасность
# ✅ Собирает код вашего приложения
# ✅ Развёртывает в Azure
# ✅ Предоставляет рабочий URL
azd up
```

**Вот и всё!** Без кликов в Azure Portal, без сложных шаблонов ARM, без ручной настройки — просто рабочие приложения в Azure.

## ❓ Azure Developer CLI vs Azure CLI: в чём разница?

Это самый частый вопрос от новичков. Вот простой ответ:

| Черта | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **Назначение** | Управление отдельными ресурсами Azure | Развёртывание полноценных приложений |
| **Подход** | Ориентирован на инфраструктуру | Ориентирован на приложения |
| **Пример** | `az webapp create --name myapp...` | `azd up` |
| **Крутизна обучения** | Нужно знать сервисы Azure | Нужно знать своё приложение |
| **Лучше для** | DevOps, инфраструктура | Разработчики, прототипирование |

### Простая аналогия

- **Azure CLI** — это как иметь все инструменты для строительства дома — молотки, пилы, гвозди. Вы можете построить всё, но должны знать строительство.
- **Azure Developer CLI** — это как нанять строительную бригаду — вы описываете, что хотите, а они выполняют строительство.

### Когда использовать каждый

| Сценарий | Используйте |
|----------|----------|
| "Хочу быстро развернуть веб-приложение" | `azd up` |
| "Мне нужно создать только аккаунт хранения" | `az storage account create` |
| "Я строю полноценное ИИ-приложение" | `azd init --template azure-search-openai-demo` |
| "Мне нужно отладить конкретный ресурс Azure" | `az resource show` |
| "Хочу развертывание, готовое к продакшену за минуты" | `azd up --environment production` |

### Они работают вместе!

AZD использует Azure CLI под капотом. Вы можете использовать оба:
```bash
# Разверните ваше приложение с помощью AZD
azd up

# Затем тонко настройте конкретные ресурсы с помощью Azure CLI
az webapp config set --name myapp --always-on true
```

## 🎯 Начало работы за 3 шага

### Шаг 1: Установите AZD (2 минуты)

**Windows:**
```powershell
winget install microsoft.azd
```

**macOS:**
```bash
brew tap azure/azd && brew install azd
```

**Linux:**
```bash
curl -fsSL https://aka.ms/install-azd.sh | bash
```

### Шаг 2: Войдите в Azure

```bash
azd auth login
```

### Шаг 3: Разверните своё первое приложение

```bash
# Инициализировать из шаблона
azd init --template todo-nodejs-mongo

# Развернуть в Azure (создаёт всё!)
azd up
```

**🎉 Всё!** Ваше приложение теперь работает в Azure.

### Очистка (не забудьте!)

```bash
# Remove all resources when done experimenting
azd down --force --purge
```

## 🔧 Быстрое руководство по устранению неполадок

**Распространенные проблемы у новичков и немедленные решения:**

<details>
<summary><strong>❌ "azd: команда не найдена"</strong></summary>

```bash
# Сначала установите AZD
# Windows (PowerShell):
winget install microsoft.azd

# macOS:
brew tap azure/azd && brew install azd

# Linux:
curl -fsSL https://aka.ms/install-azd.sh | bash

# Проверьте установку
azd version
```
</details>

<details>
<summary><strong>❌ "Подписка не найдена" или "Подписка не установлена"</strong></summary>

```bash
# Список доступных подписок
az account list --output table

# Установить подписку по умолчанию
az account set --subscription "<subscription-id-or-name>"

# Установить для среды AZD
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# Проверить
az account show
```
</details>

<details>
<summary><strong>❌ "InsufficientQuota" или "Квота превышена"</strong></summary>

```bash
# Попробуйте другой регион Azure
azd env set AZURE_LOCATION "westus2"
azd up

# Или используйте меньшие SKU в разработке
# Отредактируйте infra/main.parameters.json:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>

<details>
<summary><strong>❌ "azd up" прерывается на середине выполнения</strong></summary>

```bash
# Опция 1: Очистить и повторить попытку
azd down --force --purge
azd up

# Опция 2: Просто исправить инфраструктуру
azd provision

# Опция 3: Проверить подробный статус
azd show

# Опция 4: Проверить логи в Azure Monitor
azd monitor --logs
```
</details>

<details>
<summary><strong>❌ "Неудачная аутентификация" или "Срок действия токена истек"</strong></summary>

```bash
# Повторная аутентификация
az logout
az login

azd auth logout
azd auth login

# Проверить аутентификацию
az account show
```
</details>

<details>
<summary><strong>❌ "Ресурс уже существует" или конфликты имен</strong></summary>

```bash
# AZD генерирует уникальные имена, но при конфликте:
azd down --force --purge

# Затем повторите попытку с новым окружением
azd env new dev-v2
azd up
```
</details>

<details>
<summary><strong>❌ Длительное развертывание шаблона</strong></summary>

**Обычное время ожидания:**
- Простое веб-приложение: 5-10 минут
- Приложение с базой данных: 10-15 минут
- AI-приложения: 15-25 минут (провиженинг OpenAI идет медленно)

```bash
# Проверить прогресс
azd show

# Если застряли более 30 минут, проверьте портал Azure:
azd monitor
# Ищите неудачные развертывания
```
</details>

<details>
<summary><strong>❌ "Доступ запрещен" или "Запрещено"</strong></summary>

```bash
# Проверьте вашу роль в Azure
az role assignment list --assignee $(az account show --query user.name -o tsv)

# Вам нужна роль не ниже "Contributor"
# Попросите администратора Azure выдать:
# - Contributor (для ресурсов)
# - User Access Administrator (для назначений ролей)
```
</details>

<details>
<summary><strong>❌ Не удается найти URL развернутого приложения</strong></summary>

```bash
# Показать все конечные точки сервиса
azd show

# Или открыть портал Azure
azd monitor

# Проверить конкретный сервис
azd env get-values
# Искать переменные *_URL
```
</details>
