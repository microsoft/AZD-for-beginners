## 🆕 Что нового в azd сегодня

> 📌 Этот курс проверен на версии **`azd 1.27.1`** (июль 2026). Выполните `azd version`, чтобы проверить вашу сборку, и `azd upgrade`, чтобы получить последнюю.

Azure Developer CLI вышел за рамки обычных веб-приложений и API. Сегодня azd — это единый инструмент для развертывания **любого** приложения в Azure — включая приложения с искусственным интеллектом и интеллектуальных агентов.

Вот что это значит для вас:

- **Агенты искусственного интеллекта теперь являются полноценными нагрузками azd.** Вы можете инициализировать, развертывать и управлять проектами AI-агентов с помощью привычного рабочего процесса `azd init` → `azd up`.
- **Полный жизненный цикл агента из CLI.** Расширение `azure.ai.agents` теперь охватывает весь процесс — `azd ai agent init` для создания шаблона, `azd ai agent invoke` для тестирования (с выводом времени ответа), `azd ai agent eval generate` и `azd ai agent optimize` для измерения и улучшения качества, и `azd ai agent delete` для очистки.
- **Больше строительных блоков AI.** Новые расширения в превью — `azure.ai.skills` и `azure.ai.connections` — позволяют управлять повторно используемыми навыками агентов и подключениями Foundry напрямую через azd.
- **Интеграция с Microsoft Foundry** предоставляет развертывание моделей, хостинг агентов и настройку AI-сервисов непосредственно в экосистеме шаблонов azd.
- **Более плавные повседневные задачи.** В последних выпусках `azd init` стал идемпотентным (безопасным для повторного запуска), `azd auth login` автоматически очищает устаревшие токены, добавлена дружелюбная подсказка для первого запуска `azd tool`.
- **Основной рабочий процесс не изменился.** Независимо от того, развертываете ли вы приложение todo, микросервис или многоагентное AI-решение, команды остаются теми же.

> **Примечание для пользователей Aspire:** Microsoft теперь просто называет продукт **Aspire** (ранее ".NET Aspire"). Поддержка Aspire в azd не изменилась — изменилось только имя.

Если вы уже использовали azd, поддержка AI — это естественное расширение, а не отдельный инструмент или продвинутый курс. Если вы начинаете с нуля, вы выучите один рабочий процесс, который подходит для всего.

## 🚀 Что такое Azure Developer CLI (azd)?

**Azure Developer CLI (azd)** — это удобный для разработчиков инструмент командной строки, который упрощает развертывание приложений в Azure. Вместо того, чтобы вручную создавать и подключать десятки ресурсов Azure, вы можете развернуть целые приложения одной командой.

### Волшебство команды `azd up`

```bash
# Эта одна команда делает всё:
# ✅ Создаёт все ресурсы Azure
# ✅ Настраивает сеть и безопасность
# ✅ Собирает код вашего приложения
# ✅ Развёртывает в Azure
# ✅ Предоставляет рабочий URL
azd up
```

**Вот и всё!** Без кликов в Azure Portal, без изучения сложных ARM-шаблонов, без ручной настройки — только работающие приложения в Azure.

## ❓ Azure Developer CLI vs Azure CLI: в чём разница?

Это самый частый вопрос новичков. Вот простой ответ:

| Особенность | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **Цель** | Управление отдельными ресурсами Azure | Развёртывание целых приложений |
| **Подход** | Ориентирован на инфраструктуру | Ориентирован на приложения |
| **Пример** | `az webapp create --name myapp...` | `azd up` |
| **Кривая обучения** | Нужно знать сервисы Azure | Достаточно знать своё приложение |
| **Лучше подходит для** | DevOps, инфраструктура | Разработчики, прототипирование |

### Простая аналогия

- **Azure CLI** — это как иметь полный набор инструментов для строительства дома — молотки, пилы, гвозди. Можно построить что угодно, но нужно разбираться в строительстве.
- **Azure Developer CLI** — это как нанять подрядчика — вы описываете, что хотите, а он строит.

### Когда использовать каждый инструмент

| Сценарий | Используйте |
|----------|----------|
| "Хочу быстро развернуть веб-приложение" | `azd up` |
| "Нужно создать только аккаунт хранилища" | `az storage account create` |
| "Строю полное AI-приложение" | `azd init --template azure-search-openai-demo` |
| "Нужно отладить конкретный ресурс Azure" | `az resource show` |
| "Хочу производственное развертывание за минуты" | `azd up --environment production` |

### Они работают вместе!

AZD использует Azure CLI под капотом. Можно использовать оба:
```bash
# Разверните ваше приложение с помощью AZD
azd up

# Затем точно настройте конкретные ресурсы с помощью Azure CLI
az webapp config set --name myapp --always-on true
```

## 🎯 Начало работы в 3 шага

Перед началом убедитесь, что ваша машина готова к шаблону, который вы хотите развернуть:

**Windows:**
```powershell
.\validate-setup.ps1
```

**macOS / Linux:**
```bash
bash ./validate-setup.sh
```

Если какая-либо проверка не пройдена, исправьте сначала это, затем продолжайте быстрое начало.

### Шаг 1: Установите AZD (2 минуты)

**Windows:**
```powershell
winget install microsoft.azd
```

**macOS:**
```bash
brew tap azure/azd && brew install azd
```

**Linux:**
```bash
curl -fsSL https://aka.ms/install-azd.sh | bash
```

### Шаг 2: Аутентификация для AZD

```bash
# Необязательно, если вы планируете использовать команды Azure CLI напрямую в этом курсе
az login

# Обязательно для рабочих процессов AZD
azd auth login
```

Если не уверены, какой именно вам нужен, пройдите полный процесс установки в [Установка и настройка](docs/chapter-01-foundation/installation.md#authentication-setup).

### Шаг 3: Разверните ваше первое приложение

```bash
# Инициализация из шаблона
azd init --template todo-nodejs-mongo

# Развернуть в Azure (создает все!)
azd up
```

**🎉 Готово!** Ваше приложение теперь работает в Azure.

### Очистка (не забудьте!)

```bash
# Удалите все ресурсы после завершения экспериментов
azd down --force --purge
```

## Обзор курса

Освойте Azure Developer CLI (azd) через структурированные главы, созданные для постепенного обучения. **Особое внимание уделено развертыванию AI-приложений с интеграцией Microsoft Foundry.**


### Почему этот курс важен для современных разработчиков

Исходя из данных сообщества Microsoft Foundry в Discord, **45% разработчиков хотят использовать AZD для AI-нагрузок**, но сталкиваются с проблемами:
- Сложные много-сервисные AI-архитектуры
- Лучшие практики для запуска AI в продакшене  
- Интеграция и настройка Azure AI сервисов
- Оптимизация затрат на AI-нагрузки
- Устранение неполадок специфичных для AI-деплоймента

### Цели обучения

Пройдя этот структурированный курс, вы:
- **Овладеете основами AZD**: ключевые концепции, установка и настройка
- **Запустите AI-приложения**: Использование AZD с сервисами Microsoft Foundry
- **Реализуете инфраструктуру как код**: Управление ресурсами Azure с помощью шаблонов Bicep
- **Устраните проблемы с деплойментом**: Решение типовых ошибок и отладка проблем
- **Оптимизируете продакшн**: Безопасность, масштабирование, мониторинг и контроль затрат
- **Создадите мультиагентные решения**: Запуск сложных AI-архитектур

## Перед началом: аккаунты, доступы и предположения

Перед тем, как начать Главу 1, убедитесь, что у вас есть все необходимое. Последующие шаги установки ориентированы на то, что эти базовые вещи уже выполнены.

- **Подписка Azure**: Можно использовать существующую рабочую или личную подписку, либо создать [бесплатный пробный период](https://aka.ms/azurefreetrial) для ознакомления.
- **Права на создание ресурсов Azure**: Для большинства упражнений необходим минимум уровень доступа **Contributor** на целевой подписке или группе ресурсов. В некоторых главах потребуется возможность создавать группы ресурсов, управляемые идентичности и назначать RBAC.
- [**Аккаунт GitHub**](https://github.com): Полезен для форка репозитория, отслеживания изменений и использования GitHub Codespaces в воркшопе.
- **Предварительные требования для шаблонов**: Некоторые шаблоны требуют локальных инструментов, таких как Node.js, Python, Java или Docker. Запустите валидатор установки заранее, чтобы выявить отсутствующие инструменты.
- **Базовые навыки работы с терминалом**: Не обязательно быть экспертом, но нужно уметь выполнять команды вроде `git clone`, `azd auth login` и `azd up`.

> **Работаете в корпоративной подписке?**
> Если ваша среда Azure управляется администратором, заранее уточните, можете ли вы деплоить ресурсы в нужной подписке или группе ресурсов. Если нет – запросите sandbox-подписку или права Contributor до начала.

> **Новичок в Azure?**
> Начните с бесплатного пробного или pay-as-you-go аккаунта на https://aka.ms/azurefreetrial, чтобы пройти упражнения полностью без ожидания одобрения на уровне арендатора.

## 📖 Глубокое погружение: Возможности AZD

Помимо базовых функций, AZD предоставляет мощные возможности для производственных развертываний:

- **Развертывания на основе шаблонов** - Используйте готовые шаблоны для часто встречающихся паттернов приложений

- **Ориентированность на разработчиков** - Оптимизация производительности и удобства разработчиков

### **AZD + Microsoft Foundry: Идеально для AI развертываний**

**Почему AZD для AI решений?** AZD решает основные задачи, с которыми сталкиваются разработчики AI:

- **Готовые AI шаблоны** - Преднастроенные шаблоны для Microsoft Foundry Models, Azure AI Services и ML нагрузок
- **Безопасные AI развертывания** - Встроенные паттерны безопасности для AI сервисов, API ключей и конечных точек моделей  
- **Паттерны производственного AI** - Лучшие практики масштабируемых и экономичных AI приложений
- **Енд-ту-енд AI 워rkflows** - От разработки моделей до производства с полноценным мониторингом
- **Оптимизация затрат** - Интеллектуальное распределение ресурсов и стратегии масштабирования AI нагрузок
- **Интеграция Microsoft Foundry** - Бесшовное подключение к каталогу моделей Microsoft Foundry и конечным точкам

## 🔧 Быстрое руководство по устранению неполадок

**Распространённые проблемы начинающих и немедленные решения:**

<details>
<summary><strong>❌ "azd: команда не найдена"</strong></summary>

```bash
# Сначала установите AZD
# Windows (PowerShell):
winget install microsoft.azd

# macOS:
brew tap azure/azd && brew install azd

# Linux:
curl -fsSL https://aka.ms/install-azd.sh | bash

# Проверьте установку
azd version
```
</details>

<details>
<summary><strong>❌ "Подписка не найдена" или "Подписка не установлена"</strong></summary>

```bash
# Список доступных подписок
az account list --output table

# Установить подписку по умолчанию
az account set --subscription "<subscription-id-or-name>"

# Установить для окружения AZD
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# Проверить
az account show
```
</details>

<details>
<summary><strong>❌ "Недостаточно квоты" или "Превышена квота"</strong></summary>

```bash
# Попробуйте другой регион Azure
azd env set AZURE_LOCATION "westus2"
azd up

# Или используйте меньшие SKU в разработке
# Отредактируйте infra/main.parameters.json:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>

<details>
<summary><strong>❌ Команда "azd up" завершается с ошибкой на середине</strong></summary>

```bash
# Вариант 1: Очистить и повторить попытку
azd down --force --purge
azd up

# Вариант 2: Просто исправить инфраструктуру
azd provision

# Вариант 3: Проверить подробный статус
azd show

# Вариант 4: Проверить журналы в Azure Monitor
azd monitor --logs
```
</details>

<details>
<summary><strong>❌ "Ошибка аутентификации" или "Срок действия токена истёк"</strong></summary>

```bash
# Повторно аутентифицируйтесь для AZD
azd auth logout
azd auth login

# Необязательно: обновите также Azure CLI, если вы выполняете команды az
az logout
az login

# Проверьте аутентификацию
az account show
```
</details>

<details>
<summary><strong>❌ "Ресурс уже существует" или конфликт имён</strong></summary>

```bash
# AZD генерирует уникальные имена, но в случае конфликта:
azd down --force --purge

# Затем повторите попытку с новым окружением
azd env new dev-v2
azd up
```
</details>

<details>
<summary><strong>❌ Развёртывание шаблона занимает слишком много времени</strong></summary>

**Обычное время ожидания:**
- Простое веб-приложение: 5-10 минут
- Приложение с базой данных: 10-15 минут
- ИИ-приложения: 15-25 минут (provisioning OpenAI идет медленно)

```bash
# Проверить прогресс
azd show

# Если задержка более 30 минут, проверьте портал Azure:
azd monitor --overview
# Ищите неудачные развертывания
```
</details>

<details>
<summary><strong>❌ "Доступ запрещён" или "Запрещено"</strong></summary>

```bash
# Проверьте свою роль в Azure
az role assignment list --assignee $(az account show --query user.name -o tsv)

# Вам нужна роль не ниже "Contributor"
# Попросите вашего администратора Azure предоставить:
# - Contributor (для ресурсов)
# - User Access Administrator (для назначений ролей)
```
</details>

<details>
<summary><strong>❌ Не удаётся найти URL развернутого приложения</strong></summary>

```bash
# Показать все конечные точки сервиса
azd show

# Или открыть портал Azure
azd monitor

# Проверить конкретный сервис
azd env get-values
# Искать переменные *_URL
```
</details>
