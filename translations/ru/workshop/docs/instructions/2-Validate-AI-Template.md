## 1. Введение

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) или `azd` — это инструмент командной строки с открытым исходным кодом, который упрощает рабочий процесс разработчика при создании и развертывании приложений в Azure. 

[AZD шаблоны](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) — это стандартизированные репозитории, которые включают пример кода приложения, ресурсы _инфраструктуры как кода_ и файлы конфигурации `azd` для единой архитектуры решения. Создание инфраструктуры становится простым — достаточно команды `azd provision`, в то время как команда `azd up` позволяет за один раз создать инфраструктуру **и** развернуть приложение!

В результате, начать разработку приложения может быть так же просто, как найти подходящий _стартовый шаблон AZD_, который максимально соответствует вашим потребностям приложения и инфраструктуры — а затем настроить репозиторий в соответствии с вашими требованиями.

Прежде чем начать, убедимся, что у вас установлен Azure Developer CLI.

1. Откройте терминал VS Code и введите эту команду:

      ```bash title="" linenums="0"
      azd version
      ```

1. Вы должны увидеть что-то вроде этого!

      ```bash title="" linenums="0"
      azd version 1.19.0 (commit b3d68cea969b2bfbaa7b7fa289424428edb93e97)
      ```

**Теперь вы готовы выбрать и развернуть шаблон с помощью azd**

## 3. Активация шаблона

Давайте попробуем развернуть этот шаблон и убедиться, что он валиден. Мы будем следовать инструкциям из раздела [Начало работы](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started).

1. Нажмите [эту ссылку](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) — подтвердите действие по умолчанию `Create codespace`
1. Откроется новая вкладка браузера — дождитесь загрузки сессии GitHub Codespaces
1. Откройте терминал VS Code в Codespaces — введите следующую команду:

   ```bash title="" linenums="0"
   azd up
   ```

Выполните шаги рабочего процесса, которые будут запущены:

1. Вас попросят войти в Azure — следуйте инструкциям для аутентификации
1. Введите уникальное имя окружения для себя — например, я использовал `nitya-mshack-azd`
1. Это создаст папку `.azure/` — вы увидите подпапку с именем окружения
1. Вам предложат выбрать подписку — выберите по умолчанию
1. Вам предложат выбрать расположение — используйте `East US 2`

Теперь дождитесь завершения создания инфраструктуры. **Это занимает 10-15 минут**

1. По завершении в вашей консоли появится сообщение УСПЕХА, похожее на это:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. В вашем Azure Portal теперь будет создана группа ресурсов с именем окружения:

      ![Infra](../../../../../translated_images/ru/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **Теперь вы готовы проверить развернутую инфраструктуру и приложение**.

Azure Container App разворачивает конечную точку, которая подключается к AI агенту, созданному в проекте Microsoft Foundry для этого шаблона. Давайте посмотрим, что это значит.

1. Вернитесь на страницу _Обзор_ вашей группы ресурсов в Azure Portal

1. Нажмите на ресурс `Microsoft Foundry` в этом списке

1. Вы должны увидеть это. Нажмите кнопку `Go to Microsoft Foundry Portal`. 
   ![Foundry](../../../../../translated_images/ru/04-view-foundry-project.fb94ca41803f28f3.webp)

1. Вы увидите страницу проекта Foundry для вашего AI приложения
   ![Project](../../../../../translated_images/ru/05-visit-foundry-portal.d734e98135892d7e.webp)

1. Нажмите на `Agents` — вы увидите стандартного агента, созданного в вашем проекте
   ![Agents](../../../../../translated_images/ru/06-visit-agents.bccb263f77b00a09.webp)

1. Выберите его — вы увидите детали агента. Обратите внимание на следующее:

      - Агент по умолчанию использует Поиск по файлам (всегда)
      - В свойстве `Knowledge` указано, что загружено 32 файла (для поиска по файлам)

      - Они соответствуют 12 файлам клиентов и 20 файлам продуктов в папке `src/files`

**Вы проверили работу агента!** 

1. Ответы агента основаны на знаниях в этих файлах. 
1. Теперь вы можете задавать вопросы, связанные с этими данными, и получать обоснованные ответы.
1. Пример: файл `customer_info_10.json` описывает 3 покупки, сделанные "Amanda Perez"

## 7. Встроенная наблюдаемость

Наблюдаемость означает инструментирование вашего приложения для генерации данных, которые могут использоваться для понимания, отладки и оптимизации его работы. Чтобы познакомиться с этим:

1. Нажмите кнопку `View Run Info` — вы увидите этот вид. Это пример работы [трассировки агента](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground). _Также можно получить этот вид, нажав Thread Logs в меню верхнего уровня_.

   - Познакомьтесь с шагами запуска и инструментами, вовлеченными агентом
   - Поймите общий счётчик токенов (в сравнении с использованием выходных токенов) для ответа
   - Поймите задержки и где тратится время на выполнение

      ![Agent](../../../../../translated_images/ru/10-view-run-info.b20ebd75fef6a1cc.webp)

1. Нажмите вкладку `Metadata`, чтобы увидеть дополнительные атрибуты запуска, которые могут быть полезны для контекста при отладке проблем позже.   

      ![Agent](../../../../../translated_images/ru/11-view-run-info-metadata.7966986122c7c2df.webp)


1. Нажмите вкладку `Evaluations`, чтобы увидеть автооценки, сделанные по ответу агента. Они включают оценки безопасности (например, самоповреждение) и специализированные оценки агента (например, разрешение намерений, выполнение задач).

      ![Agent](../../../../../translated_images/ru/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. И наконец, нажмите вкладку `Monitoring` в боковом меню.

      - Выберите вкладку `Resource usage` на отображаемой странице и посмотрите метрики.
      - Отслеживайте использование приложения с точки зрения стоимости (токены) и нагрузки (запросы).
      - Отслеживайте задержку приложения до первого байта (обработка ввода) и последнего байта (вывода).

      ![Agent](../../../../../translated_images/ru/13-monitoring-resources.5148015f7311807f.webp)

## 8. Переменные окружения

До этого мы проходили развертывание в браузере — проверили, что инфраструктура создана, а приложение работает. Но чтобы работать с приложением _через код_, нам нужно настроить локальную среду разработки с соответствующими переменными, необходимыми для работы с этими ресурсами. Использование `azd` облегчает это.

1. Azure Developer CLI [использует переменные окружения](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) для хранения и управления настройками конфигурации для развертываний приложения.

1. Переменные окружения хранятся в `.azure/<env-name>/.env` — это ограничивает их область видимости средой `env-name`, используемой во время развертывания, и помогает изолировать среды между разными целями развертывания в одном репозитории.

1. Переменные окружения автоматически загружаются командой `azd` при выполнении конкретной команды (например, `azd up`). Обратите внимание, что `azd` не читает автоматически переменные окружения _уровня ОС_ (например, установленные в shell) — вместо этого используйте `azd set env` и `azd get env` для передачи информации в скриптах.


Давайте попробуем несколько команд:

1. Получить все заданные переменные окружения для `azd` в этой среде:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      Вы увидите что-то вроде:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. Получить конкретное значение — например, я хочу узнать, задано ли значение `AZURE_AI_AGENT_MODEL_NAME`

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      Вы увидите что-то подобное — по умолчанию оно не установлено!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. Установить новую переменную окружения для `azd`. Здесь мы обновляем имя модели агента. _Примечание: любые изменения будут сразу отражены в файле `.azure/<env-name>/.env`._

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      Теперь мы должны увидеть, что значение установлено:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. Обратите внимание, что некоторые ресурсы являются постоянными (например, развертывания модели) и потребуют не только команды `azd up` для повторного развертывания. Давайте попробуем удалить исходное развертывание и заново развернуть с изменёнными переменными окружения.

1. **Обновление** Если вы ранее развертывали инфраструктуру с помощью шаблона azd — вы можете _обновить_ состояние локальных переменных окружения в зависимости от текущего состояния вашего развертывания Azure, используя эту команду:

      ```bash title="" linenums="0"
      azd env refresh
      ```

      Это мощный способ _синхронизировать_ переменные окружения между двумя и более локальными средами разработки (например, команда из нескольких разработчиков) — позволяя развернутой инфраструктуре служить источником правды для состояния переменных окружения. Члены команды просто _обновляют_ переменные, чтобы снова синхронизироваться.
