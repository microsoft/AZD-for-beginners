## 1. Введение

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) или `azd` — это инструмент командной строки с открытым исходным кодом, упрощающий рабочий процесс разработчика при создании и развертывании приложений в Azure.

[AZD Templates](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) — это стандартизированные репозитории, включающие пример кода приложения, активы _инфраструктуры как кода_ и конфигурационные файлы `azd` для комплексной архитектуры решения. Развертывание инфраструктуры становится таким же простым, как выполнение команды `azd provision` — а команда `azd up` позволяет одновременно развернуть инфраструктуру **и** приложение!

Таким образом, начать процесс разработки приложения может быть так же просто, как найти подходящий _AZD Starter шаблон_, который максимально соответствует требованиям вашего приложения и инфраструктуры — а затем настроить репозиторий под свои сценарии.

Прежде чем начать, давайте убедимся, что у вас установлен Azure Developer CLI.

1. Откройте терминал в VS Code и выполните эту команду:

      ```bash title="" linenums="0"
      azd version
      ```

1. Вы должны увидеть примерно следующее!

      ```bash title="" linenums="0"
      azd version 1.27.1 (commit <current-build>)
      ```

**Теперь вы готовы выбрать и развернуть шаблон с помощью azd**

## 3. Активация шаблона

Давайте попробуем развернуть этот шаблон и убедимся, что он корректен. Мы будем следовать указаниям из раздела [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started).

1. Выберите рабочую среду для репозитория шаблона:

      - **GitHub Codespaces**: Нажмите [эту ссылку](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) и подтвердите создание `Create codespace`
      - **Локальный клон или dev container**: Клонируйте `Azure-Samples/get-started-with-ai-agents` и откройте его в VS Code

1. Дождитесь готовности терминала VS Code, затем введите следующую команду:

   ```bash title="" linenums="0"
   azd up
   ```

Выполните шаги рабочего процесса, которые будут запущены:

1. Вас попросят войти в Azure — следуйте инструкциям для аутентификации
1. Введите уникальное имя среды — например, я использовал `nitya-mshack-azd`
1. Будет создана папка `.azure/` — вы увидите в ней подпапку с именем среды
1. Вас попросят выбрать подписку — выберите по умолчанию
1. Вас попросят указать местоположение — используйте `East US 2`

Теперь дождитесь завершения развертывания. **Это занимает 10-15 минут**

1. По завершении консоль отобразит сообщение SUCCESS, похожее на это:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. В вашем портале Azure теперь будет группа ресурсов с именем этой среды:

      ![Infra](../../../../../translated_images/ru/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **Теперь вы готовы проверить развернутую инфраструктуру и приложение**.

## 5. Проверка агента

Azure Container App разворачивает конечную точку, которая подключается к AI агенту, размещённому в проекте Microsoft Foundry для этого шаблона. Давайте посмотрим, что это значит.

1. Вернитесь на страницу _Обзор_ группы ресурсов в портале Azure

1. Нажмите на ресурс `Microsoft Foundry` в списке

1. Вы увидите это. Нажмите кнопку `Перейти в Microsoft Foundry Portal`. 
   ![Foundry](../../../../../translated_images/ru/04-view-foundry-project.fb94ca41803f28f3.webp)

1. Вы откроете страницу проекта Foundry для вашего AI приложения
   ![Project](../../../../../translated_images/ru/05-visit-foundry-portal.d734e98135892d7e.webp)

1. Нажмите на `Agents` — вы увидите стандартного агента, развернутого в вашем проекте
   ![Agents](../../../../../translated_images/ru/06-visit-agents.bccb263f77b00a09.webp)

1. Выберите его — вы увидите детали агента. Обратите внимание на следующее:

      - Агент по умолчанию использует File Search (всегда)
      - В разделе `Knowledge` указано, что загружено 32 файла (для поиска по файлам)
      ![Agents](../../../../../translated_images/ru/07-view-agent-details.0e049f37f61eae62.webp)

1. В меню слева найдите опцию `Data+indexes` и кликните для просмотра деталей.

      - Вы увидите 32 загруженных файла данных для знания.
      - Эти файлы соответствуют 12 файлам клиентов и 20 файлам продуктов в папке `src/files`
      ![Data](../../../../../translated_images/ru/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**Вы подтвердили работу агента!** 

1. Ответы агента основываются на знаниях из этих файлов.
1. Теперь вы можете задавать вопросы, связанные с этими данными, и получать обоснованные ответы.
1. Например: `customer_info_10.json` описывает 3 покупки, сделанные "Амандой Перес"

Вернитесь на вкладку браузера с конечной точкой Container App и спросите: `Какие продукты принадлежат Аманде Перес?`. Вы должны увидеть нечто подобное:

## 7. Встроенная наблюдаемость

Наблюдаемость предполагает инструментирование вашего приложения для генерации данных, которые могут быть использованы для понимания, отладки и оптимизации его работы. Чтобы получить представление об этом:

1. Нажмите кнопку `Просмотреть информацию о запуске` — вы увидите этот экран. Это пример [трассировки агента](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) в действии. _Вы также можете открыть это окно, выбрав Thread Logs в верхнем меню_.

   - Ознакомьтесь со шагами выполнения и инструментами, использованными агентом
   - Понимайте общее количество токенов (в сравнении с токенами вывода) для ответа
   - Понимайте задержку и места, где тратится время выполнения

      ![Agent](../../../../../translated_images/ru/10-view-run-info.b20ebd75fef6a1cc.webp)

1. Нажмите вкладку `Метаданные`, чтобы увидеть дополнительные атрибуты запуска, которые могут помочь при отладке.

      ![Agent](../../../../../translated_images/ru/11-view-run-info-metadata.7966986122c7c2df.webp)


1. Нажмите вкладку `Оценки`, чтобы увидеть автооценки, сделанные по ответу агента. Включает оценки безопасности (например, Самоповреждение) и специфические оценки агента (например, разрешение намерений, соблюдение задачи).

      ![Agent](../../../../../translated_images/ru/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. И последнее, но не менее важное - нажмите вкладку `Мониторинг` в боковом меню.

      - Выберите вкладку `Использование ресурсов` на отображаемой странице — и посмотрите метрики.
      - Отслеживайте использование приложения в терминах затрат (токенов) и нагрузки (запросов).
      - Отслеживайте задержки приложения до первого байта (обработка ввода) и до последнего байта (вывод).

## 8. Переменные среды

Пока что мы проходили процесс развертывания в браузере и проверяли, что инфраструктура развернута, а приложение работает. Но чтобы работать с приложением _через код_, нам нужно настроить локальную среду разработки с соответствующими переменными, необходимыми для работы с этими ресурсами. Использование `azd` это упрощает.

1. Azure Developer CLI [использует переменные среды](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) для хранения и управления настройками конфигурации для развертываний приложений.

1. Переменные среды хранятся в `.azure/<env-name>/.env` — это ограничивает их область действия средой `env-name`, используемой при развертывании, и помогает изолировать среды между различными целями развертывания в одном репозитории.

1. Переменные среды автоматически загружаются командой `azd` при выполнении конкретной команды (например, `azd up`). Обратите внимание, что `azd` не читает автоматически переменные среды уровня ОС (например, установленные в оболочке) — вместо этого используйте `azd set env` и `azd get env` для передачи информации внутри скриптов.


Давайте попробуем несколько команд:

1. Получите все переменные среды, установленные для `azd` в этой среде:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      Вы увидите что-то вроде:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. Получите конкретное значение — например, я хочу узнать, установлено ли значение `AZURE_AI_AGENT_MODEL_NAME`

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      Вы увидите примерно следующее — по умолчанию оно не установлено!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. Установите новую переменную среды для `azd`. Здесь мы обновляем имя модели агента. _Обратите внимание: любые изменения сразу отражаются в файле `.azure/<env-name>/.env`.

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      Теперь значение должно быть установлено:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. Обратите внимание, что некоторые ресурсы постоянны (например, развертывания моделей) и требуют больше, чем просто `azd up` для серверного повторного развертывания. Давайте попробуем удалить оригинальное развертывание и развернуть заново с измененными переменными среды.

1. **Обновить** Если вы ранее развертывали инфраструктуру с помощью шаблона azd — вы можете _обновить_ состояние локальных переменных среды на основе текущего состояния вашего развертывания в Azure командой:

      ```bash title="" linenums="0"
      azd env refresh
      ```


      Это мощный способ _синхронизировать_ переменные окружения между двумя или более локальными средами разработки (например, команда с несколькими разработчиками), позволяющий использованной инфраструктуре служить эталоном состояния переменных окружения. Члены команды просто _обновляют_ переменные, чтобы снова синхронизироваться.
