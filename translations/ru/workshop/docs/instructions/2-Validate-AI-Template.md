## 1. Введение

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) или `azd` — это инструмент командной строки с открытым исходным кодом, который упрощает рабочий процесс разработчика при создании и развертывании приложений в Azure. 

[AZD Templates](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) — это стандартизированные репозитории, включающие пример кода приложения, ресурсы _инфраструктуры как код_, а также файлы конфигурации `azd` для единой архитектуры решения. Создание инфраструктуры становится таким же простым, как выполнение команды `azd provision` — в то время как использование `azd up` позволяет одновременно создать инфраструктуру **и** развернуть приложение!

Таким образом, начать разработку вашего приложения может быть так же просто, как найти подходящий _AZD Starter template_, который максимально соответствует вашим требованиям к приложению и инфраструктуре — а затем настроить репозиторий под ваши сценарии.

Прежде чем начать, убедитесь, что у вас установлен Azure Developer CLI.

1. Откройте терминал в VS Code и введите следующую команду:

      ```bash title="" linenums="0"
      azd version
      ```

1. Вы должны увидеть что-то подобное!

      ```bash title="" linenums="0"
      azd version 1.23.12 (commit <current-build>)
      ```

**Теперь вы готовы выбрать и развернуть шаблон с помощью azd**

## 3. Активация шаблона

Давайте попробуем развернуть этот шаблон и убедимся, что он действителен. Мы будем следовать инструкциям в разделе [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started).

1. Выберите рабочее окружение для репозитория шаблона:

      - **GitHub Codespaces**: Нажмите [по этой ссылке](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) и подтвердите создание codespace
      - **Локальный клон или dev контейнер**: Клонируйте `Azure-Samples/get-started-with-ai-agents` и откройте его в VS Code

1. Дождитесь готовности терминала VS Code, затем наберите следующую команду:

   ```bash title="" linenums="0"
   azd up
   ```

Выполните шаги рабочего процесса, которые это вызовет:

1. Вас попросят войти в Azure — следуйте инструкциям для аутентификации
1. Введите уникальное имя окружения — например, я использовал `nitya-mshack-azd`
1. Будет создана папка `.azure/` — в ней будет подпапка с именем окружения
1. Вам предложат выбрать имя подписки — выберите по умолчанию
1. Вас попросят указать локацию — используйте `East US 2`

Теперь ждите завершения создания инфраструктуры. **Это займет 10-15 минут**

1. По завершении в консоли появится сообщение об УСПЕХЕ, подобное этому:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. В вашем Azure портале теперь будет создана группа ресурсов с именем этого окружения:

      ![Infra](../../../../../translated_images/ru/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **Теперь вы готовы проверить развернутую инфраструктуру и приложение**.

1. Нажмите на ресурс `Microsoft Foundry` в этом списке

   ![Foundry](../../../../../translated_images/ru/04-view-foundry-project.fb94ca41803f28f3.webp)

Наблюдаемость — это инструментирование вашего приложения для генерации данных, которые можно использовать для понимания, отладки и оптимизации его работы. Чтобы ознакомиться с этим:

1. Нажмите кнопку `View Run Info` — вы увидите это окно. Это пример работы [трассировки агента](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground). _Вы также можете попасть сюда, нажав Thread Logs в главном меню_.

   - Ознакомьтесь с шагами выполнения и инструментами, задействованными агентом
   - Поймите общее количество токенов (и использование токенов вывода) для ответа
   - Оцените задержку и где тратится время во время выполнения

      ![Agent](../../../../../translated_images/ru/10-view-run-info.b20ebd75fef6a1cc.webp)

1. Нажмите вкладку `Metadata`, чтобы увидеть дополнительные атрибуты для выполнения, которые могут помочь при отладке проблем позже.   

      ![Agent](../../../../../translated_images/ru/11-view-run-info-metadata.7966986122c7c2df.webp)


1. Нажмите вкладку `Evaluations`, чтобы увидеть автоматические оценки ответа агента. Они включают оценки безопасности (например, саморазрушение) и специфичные для агента оценки (например, разрешение намерений, выполнение задач).

      ![Agent](../../../../../translated_images/ru/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. И наконец, нажмите вкладку `Monitoring` в боковом меню.

      - Выберите вкладку `Resource usage` на странице и просмотрите метрики.
      - Отслеживайте использование приложения по затратам (токены) и нагрузке (запросы).
      - Отслеживайте задержку приложения до первого байта (обработка ввода) и последнего байта (вывода).

      ![Agent](../../../../../translated_images/ru/13-monitoring-resources.5148015f7311807f.webp)

## 8. Переменные окружения

До сих пор мы работали через браузер — и проверили, что инфраструктура создана и приложение работает. Но чтобы работать с приложением _сначала на уровне кода_, необходимо настроить локальное окружение разработки с соответствующими переменными, необходимыми для работы с этими ресурсами. Использование `azd` упрощает это.

1. Azure Developer CLI [использует переменные окружения](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) для хранения и управления настройками конфигурации развертывания приложений.

1. Переменные окружения хранятся в `.azure/<env-name>/.env` — это ограничивает их область действия конкретным окружением `env-name`, использованным при развертывании, и помогает изолировать окружения для разных целей развертывания в одном репозитории.

1. Переменные окружения автоматически загружаются командой `azd` всякий раз, когда выполняется конкретная команда (например, `azd up`). Обратите внимание, что `azd` не читает автоматически _системные_ переменные окружения ОС (например, установленные в оболочке) — вместо этого используйте команды `azd set env` и `azd get env` для передачи информации в скриптах.


Давайте попробуем несколько команд:

1. Получите все переменные окружения, установленные для `azd` в этом окружении:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      Вы увидите примерно следующее:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. Получите конкретное значение — например, я хочу узнать, установлено ли значение `AZURE_AI_AGENT_MODEL_NAME`

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      Вы увидите примерно следующее — оно не было установлено по умолчанию!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. Установите новую переменную окружения для `azd`. Здесь мы обновляем имя модели агента. _Примечание: любые изменения немедленно отражаются в файле `.azure/<env-name>/.env`._

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      Теперь мы должны увидеть, что значение установлено:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. Обратите внимание, что некоторые ресурсы являются персистентными (например, развертывания моделей) и потребуют больше, чем просто `azd up`, чтобы принудительно перенастроить развертывание. Давайте попробуем удалить исходное развертывание и развернуть заново с изменёнными переменными окружения.

1. **Обновление** Если вы ранее развертывали инфраструктуру с помощью шаблона azd — вы можете _обновить_ состояние ваших локальных переменных окружения на основе текущего состояния вашего развертывания Azure с помощью этой команды:

      ```bash title="" linenums="0"
      azd env refresh
      ```

      Это мощный способ _синхронизировать_ переменные окружения между двумя или более локальными средами разработки (например, команда с несколькими разработчиками) — позволяя развернутой инфраструктуре служить источником достоверной информации о состоянии переменных окружения. Члены команды просто _обновляют_ переменные, чтобы снова синхронизироваться.
