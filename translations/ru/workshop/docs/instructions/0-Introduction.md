# AZD для разработчиков ИИ

!!! info "ЦЕЛЕВАЯ АУДИТОРИЯ ЭТОГО МАСТЕР-КЛАССА"
   
    Этот мастер-класс предназначен для всех, кто хочет создавать агентные ИИ-приложения на Microsoft Foundry, используя лучшие инструменты и рабочие процессы. Мастер-класс подходит как для новичков, так и для опытных разработчиков, имеющих определённые знания об Azure, концепциях ИИ и Python. Пожалуйста, проверьте предварительные требования перед началом.

    **Длительность:** 3-4 часа  
    **Уровень:** от начального до среднего  

---


## Предварительные требования

- Личный аккаунт GitHub → [Создать бесплатно](https://github.com/signup)
- Активный аккаунт Azure → [Создать бесплатно](https://aka.ms/free)
- Знание основных концепций ИИ, разработка на Python, умение работать с командной строкой
- Опыт работы с VS Code и инструментами и рабочими процессами GitHub.

---

## Цели обучения

К концу этого мастер-класса вы будете уметь:

- [X] **Работать с Azure Developer CLI** → для создания, развертывания и управления ресурсами Azure.
- [X] **Использовать шаблоны AI-приложений** → для быстрого старта проектов с ИИ-агентами на Microsoft Foundry.
- [X] **Настраивать шаблоны** → для таких функций, как Red Teaming, оценки, трассировка и мониторинг.
- [X] **Анализировать и исследовать** → шаблоны AZD и рабочие процессы с помощью GitHub Copilot.
- [X] **Кастомизировать и повторно развёртывать** → стартовый шаблон ИИ-агента с учётом ваших требований.

---

## Структура мастер-класса

Мастер-класс построен из модулей, каждый из которых включает практическое занятие:

1. **Введение** — Понимание того, что такое шаблоны AZD и как они работают.
1. **Выбор** — Поиск подходящего «стартерного» шаблона для вашего следующего ИИ-проекта.
1. **Проверка** — Развёртывание и изучение шаблона для подтверждения его функциональности.
1. **Деконструкция** — Разбор шаблона для понимания его компонентов и настроек.
1. **Конфигурация** — Изучение активации и использования различных функций Microsoft Foundry.
1. **Кастомизация** — Перестройка приложения с учётом ваших данных и требований.
1. **Удаление** — Изучение удаления инфраструктуры и восстановления ресурсов после мягкого удаления.
