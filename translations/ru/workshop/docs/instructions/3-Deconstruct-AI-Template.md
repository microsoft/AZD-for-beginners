## 1. Активируйте GitHub Copilot

### 1.1 Установите GitHub Copilot Chat

Пора познакомиться с [GitHub Copilot в режиме агента](https://code.visualstudio.com/docs/copilot/chat/chat-agent-mode). Теперь мы можем использовать естественный язык, чтобы описать нашу задачу на высоком уровне и получить помощь в её выполнении. Для этой лабораторной работы мы будем использовать [бесплатный план Copilot](https://github.com/github-copilot/signup), который имеет ежемесячные ограничения на количество выполнения и взаимодействие в чате.

Расширение можно установить из маркетплейса, но оно уже должно быть доступно в вашей среде Codespaces. _Нажмите `Open Chat` в выпадающем меню значка Copilot — и введите команду, например, `What can you do?`_ — вас могут попросить войти в систему. **GitHub Copilot Chat готов к работе**.

### 1.2 Установите MCP сервера

Для эффективной работы режима агента ему нужен доступ к правильным инструментам, которые помогут получать знания или выполнять действия. Здесь помогают MCP серверы. Мы настроим следующие серверы:

1. [Azure MCP Server](../../../../../workshop/docs/instructions)
1. [Microsoft Docs MCP Server](../../../../../workshop/docs/instructions)

Чтобы их активировать:

1. Создайте файл под названием `.vscode/mcp.json`, если он не существует
1. Скопируйте в него следующий содержимое — и запустите серверы!
   ```json title=".vscode/mcp.json"
   {
      "servers": {
         "Azure MCP Server": {
            "command": "npx",
            "args": [
            "-y",
            "@azure/mcp@latest",
            "server",
            "start"
            ]
         },
         "microsoft.docs.mcp": {
            "type": "http",
            "url": "https://learn.microsoft.com/api/mcp"
         }
      }
   }
   ```

??? warning "Возможно, вы получите ошибку, что `npx` не установлен (нажмите, чтобы развернуть решение)"

      Чтобы исправить это, откройте файл `.devcontainer/devcontainer.json` и добавьте следующую строку в раздел features. Затем перестройте контейнер. Теперь у вас должен быть установлен `npx`.

      ```title="" linenums="0"
         "features": {
            "ghcr.io/devcontainers/features/node:1": {},
            ...
         },
      ```

### 1.3 Проверьте GitHub Copilot Chat

**Сначала выполните `az login` для аутентификации в Azure из командной строки VS Code.**

Теперь вы сможете запрашивать статус своей подписки Azure и задавать вопросы о развернутых ресурсах или конфигурации. Попробуйте следующие команды:

1. `List my Azure resource groups`
1. `#foundry list my current deployments`

Вы также можете задавать вопросы о документации Azure и получать ответы, основанные на сервере Microsoft Docs MCP. Попробуйте:

1. `#microsoft_docs_search What is Azure Developer CLI?`
1. `#microsoft_docs_search Show me a Python tutorial to chat with deployed model`

Или попросить примеры кода для выполнения задачи. Попробуйте:

1. `Give me a Python code example that uses AAD for an interactive chat client`

В режиме `Ask` это предоставит код, который вы можете скопировать и опробовать. В режиме `Agent` это может пойти дальше и создать необходимые ресурсы для вас — включая скрипты настройки и документацию — чтобы помочь выполнить задачу.

**Теперь вы готовы начать изучение репозитория с шаблоном**

## 2. Разбор архитектуры

??? prompt "СПРОСИТЕ: Объясните архитектуру приложения на docs/images/architecture.png одним абзацем"

      Это приложение — чат с поддержкой ИИ, построенный на Azure, демонстрирующий современную архитектуру на основе агентов. Решение основано на Azure Container App, в котором размещён основной код приложения, обрабатывающий пользовательский ввод и генерирующий интеллектуальные ответы через ИИ-агента.
      
      Архитектура использует проект Microsoft Foundry в качестве основы для ИИ-возможностей, подключаясь к сервисам Azure AI, которые предоставляют модели языка (например, gpt-4.1-mini) и функционал агента.
      
      Взаимодействие с пользователем происходит через фронтенд на React, который связывается с бэкендом FastAPI, общающимся с сервисом ИИ-агента для генерации контекстных ответов.
      
      В системе реализованы возможности поиска знаний через поиск по файлам или сервис Azure AI Search, позволяющие агенту получать и цитировать информацию из загруженных документов.
      
      Для обеспечения операционной эффективности архитектура включает комплексный мониторинг через Application Insights и Log Analytics Workspace для трассировки, логирования и оптимизации производительности.
      
      Azure Storage служит blob-хранилищем для данных приложения и загружаемых файлов, а Managed Identity обеспечивает безопасный доступ между ресурсами Azure без хранения учётных данных. Вся система спроектирована для масштабируемости и удобства сопровождения, контейнеризированное приложение автоматически масштабируется по требованию, при этом предоставляя встроенные функции безопасности, мониторинга и CI/CD через управляемую экосистему сервисов Azure.

### 3.1 Основная архитектура приложения

Этот шаблон следует паттерну **веб-приложения полного стека** с:

- **Бэкенд**: Python FastAPI с интеграцией Azure AI
- **Фронтенд**: TypeScript/React с системой сборки Vite
- **Инфраструктура**: Шаблоны Azure Bicep для облачных ресурсов
- **Контейнеризация**: Docker для консистентного развертывания

### 3.2 Инфраструктура как код (bicep)

Слой инфраструктуры использует **шаблоны Azure Bicep**, организованные модульно:

   - **`main.bicep`**: Оркестрация всех ресурсов Azure
   - **Модули `core/`**: Повторно используемые компоненты для различных сервисов
      - AI сервисы (Microsoft Foundry Models, AI Search)
      - Хостинг контейнеров (Azure Container Apps)
      - Мониторинг (Application Insights, Log Analytics)
      - Безопасность (Key Vault, Managed Identity)

### 3.3 Исходники приложения (`src/`)

**Бэкенд API (`src/api/`)**:

- REST API на FastAPI
- Интеграция агенты Foundry
- Управление поисковым индексом для поиска знаний
- Возможности загрузки и обработки файлов

**Фронтенд (`src/frontend/`)**:

- Современное SPA на React/TypeScript
- Vite для быстрой разработки и оптимизированных сборок
- Интерфейс чата для взаимодействия с агентом

**База знаний (`src/files/`)**:

- Примеры данных клиентов и продуктов
- Демонстрирует поиск по файлам
- Форматы JSON и Markdown


### 3.4 DevOps и автоматизация

**Скрипты (`scripts/`)**:

- Кросс-платформенные PowerShell и Bash скрипты
- Проверка и настройка окружения
- Конфигурация после развертывания
- Управление квотами моделей

**Интеграция Azure Developer CLI**:

- Конфигурация `azure.yaml` для рабочих процессов `azd`
- Автоматизированное provision и развертывание
- Управление переменными окружения

### 3.5 Тестирование и обеспечение качества

**Фреймворк оценки (`evals/`)**:

- Оценка производительности агентов
- Тестирование качества ответов на запросы
- Автоматизированный pipeline оценок

**Безопасность ИИ (`airedteaming/`)**:

- Тесты «красной команды» для безопасности ИИ
- Сканирование уязвимостей безопасности
- Практики ответственного ИИ
