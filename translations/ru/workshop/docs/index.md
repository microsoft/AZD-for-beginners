# AZD для разработчиков ИИ

> Проверено на `azd 1.23.12` в марте 2026 г.

> [!IMPORTANT]  
> **Этот воркшоп оснащён руководством по воркшопу, которое вы можете просмотреть в своём локальном браузере. Чтобы начать, откройте репозиторий в GitHub Codespaces, контейнере для разработки или локальном клоне с установленным MkDocs — затем дождитесь появления активного терминала VS Code и введите:**  
> `mkdocs serve > /dev/null 2>&1 &`  
> **Вы должны увидеть всплывающее окно с предложением открыть страницу превью в браузере.**

Добро пожаловать на практический воркшоп по изучению Azure Developer CLI (AZD) с акцентом на развертывание приложений ИИ. Этот воркшоп поможет вам получить практическое понимание шаблонов AZD в 3 этапа:

1. **Открытие** — найдите шаблон, подходящий именно вам.  
1. **Развертывание** — разверните и проверьте, что всё работает.  
1. **Настройка** — модифицируйте и улучшайте, чтобы сделать его своим!

В ходе воркшопа вы также познакомитесь с основными инструментами и рабочими процессами для разработчиков, которые помогут оптимизировать ваше сквозное разработческое путешествие.

## Обзор воркшопа

**Длительность:** 3-4 часа  
**Уровень:** от начального до среднего  
**Требования:** Знакомство с Azure, концепциями ИИ, VS Code и инструментами командной строки.

## Чему вы научитесь

Думайте о шаблоне AZD как о учебной песочнице для изучения различных возможностей и инструментов сквозной разработки в Microsoft Foundry. К концу этого воркшопа у вас будет интуитивное понимание различных инструментов и концепций в этом контексте.

| Концепция  | Цель |
|:---|:---|
| **Azure Developer CLI** | Понять команды инструмента и рабочие процессы|
| **Шаблоны AZD**| Понять структуру проекта и конфигурацию|
| **Azure AI Agent**| Создание и развертывание проекта Microsoft Foundry |
| **Azure AI Search**| Включить контекстную инженерию с помощью агентов |
| **Наблюдаемость**| Изучить трассировку, мониторинг и оценку |
| **Red Teaming**| Изучить противодействие тестированию и мер защиты |
| | |
