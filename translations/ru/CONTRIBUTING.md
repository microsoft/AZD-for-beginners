# Как внести вклад в AZD For Beginners

Спасибо за ваш интерес к участию в курсе AZD For Beginners! Этот документ содержит рекомендации и информацию для участников.

## Как внести вклад

### Сообщение об ошибках

Если вы обнаружили баги, ошибки или у вас есть предложения по улучшению:

1. **Проверьте существующие проблемы**: Поиск по [existing issues](https://github.com/microsoft/azd-for-beginners/issues), чтобы избежать дублирования
2. **Создайте новый issue**: Используйте соответствующий шаблон для issue
3. **Предоставьте детали**: Укажите шаги для воспроизведения, ожидаемое поведение и скриншоты, если это применимо

### Предложение изменений

1. **Сделайте форк репозитория**: Создайте собственный форк проекта
2. **Создайте ветку**: Создайте feature-ветку для ваших изменений
3. **Внесите изменения**: Следуйте нашим стандартам кодирования и документации
4. **Проверьте свои изменения**: Убедитесь, что все ссылки работают и контент отображается корректно
5. **Отправьте Pull Request**: Предоставьте четкое описание ваших изменений

### Типы вкладов

Мы приветствуем следующие виды вкладов:

#### Улучшения содержимого
- Исправление опечаток, грамматики и проблем форматирования
- Повышение ясности существующей документации
- Обновление устаревшей информации
- Добавление недостающих объяснений или контекста

#### Новые примеры
- Добавление реальных сценариев деплоя
- Создание новых шаблонов AZD для типичных случаев использования
- Предоставление дополнительных примеров кода и упражнений

#### Помощь с переводом
- Помощь в поддержке существующих переводов
- Улучшение качества перевода
- Добавление переводов для нового контента

#### Отчёты об ошибках
- Сообщение о сломанных ссылках или неверных примерах кода
- Выявление несоответствий в документации
- Пометка проблем с доступностью

## Руководство по внесению вклада

### Стандарты документации

- Используйте понятный, дружественный к новичкам язык
- Включайте учебные цели и ожидаемые результаты
- Предоставляйте пошаговые инструкции с ожидаемыми результатами
- Добавляйте навигационные ссылки на связанные материалы
- Используйте единообразное форматирование и стиль Markdown

### Примеры кода

- Убедитесь, что все примеры кода протестированы и работают
- Включайте комментарии, объясняющие ключевые концепции
- Следуйте лучшим практикам Azure и AZD
- Указывайте ожидаемый вывод, где это применимо

### Сообщения коммита

Используйте понятные, описательные сообщения коммита:
- `docs: fix typo in installation guide`
- `feat: add new container app example`
- `fix: correct broken link in chapter 2`

## Кодекс поведения

Пожалуйста, прочитайте и соблюдайте наш [Code of Conduct](CODE_OF_CONDUCT.md), чтобы поддерживать гостеприимное и инклюзивное сообщество.

## Получение помощи

- **Вопросы**: Откройте [GitHub Discussion](https://github.com/microsoft/azd-for-beginners/discussions)
- **Сообщество**: Присоединяйтесь к [Azure Discord](https://discord.gg/microsoft-azure)
- **Документация**: Ознакомьтесь с [документацией Azure Developer CLI](https://learn.microsoft.com/azure/developer/azure-developer-cli/)
