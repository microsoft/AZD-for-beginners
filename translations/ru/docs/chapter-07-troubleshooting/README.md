# Глава 7: Устранение неполадок и отладка

## Обзор

Эта глава поможет вам диагностировать и решать распространённые проблемы при работе с Azure Developer CLI. От сбоев развертывания до специфических проблем с ИИ.

> Проверено на `azd 1.25.6` в июне 2026 года.

## Цели обучения

После изучения этой главы вы сможете:
- Диагностировать распространённые сбои развертывания AZD
- Отлаживать проблемы с аутентификацией и разрешениями
- Решать проблемы с подключением к ИИ-сервисам
- Использовать Azure Portal и CLI для устранения неполадок

## 🚨 Быстрые решения

### Проблемы с аутентификацией
```bash
# Требуется для рабочих процессов AZD
azd auth login

# Необязательно, если вы также используете команды Azure CLI напрямую
az login

azd auth status
```

### Сбои при подготовке
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### Конфликты ресурсов
```bash
azd down --force --purge
azd env new different-name
azd up
```

### Превышение квоты
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 Справочник кодов ошибок

| Ошибка | Причина | Решение |
|-------|-------|----------|
| `AuthenticationError` | Не выполнен вход | `azd auth login` |
| `ResourceNotFound` | Отсутствует ресурс | Проверьте имена ресурсов |
| `QuotaExceeded` | Ограничения подписки | Запросите увеличение квоты |
| `InvalidTemplate` | Ошибка синтаксиса Bicep | `az bicep build` |
| `Conflict` | Ресурс уже существует | Используйте новое имя или удалите |
| `Forbidden` | Недостаточно прав | Проверьте роли RBAC |

## 🔄 Сброс и восстановление

```bash
# Мягкая перезагрузка (сохранить ресурсы, развернуть код заново)
azd deploy --force

# Жесткая перезагрузка (удалить всё, начать заново)
azd down --force --purge
azd up
```
