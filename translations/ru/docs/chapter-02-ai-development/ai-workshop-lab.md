**Продолжительность:** 2-3 часа  
**Уровень:** Средний  
**Предварительные знания:** Базовые знания Azure, знакомство с концепциями ИИ/МО

## Модуль 2: Развертывание вашего первого приложения ИИ

### Шаг 2.1: Инициализация среды

1. **Создайте новую среду AZD:**
```bash
azd env new myai-workshop
```

2. **Установите необходимые параметры:**
```bash
# Установите предпочтительный регион Azure
azd env set AZURE_LOCATION eastus

# Необязательно: установите конкретную модель OpenAI
azd env set AZURE_OPENAI_MODEL gpt-35-turbo
```

### Шаг 2.2: Развертывание инфраструктуры и приложения

1. **Разверните с помощью AZD:**
```bash
azd up
```

**Что происходит при `azd up`:**
- ✅ Создается сервис Microsoft Foundry Models
- ✅ Создается сервис Cognitive Search
- ✅ Настраивается App Service для веб-приложения
- ✅ Конфигурируются сеть и безопасность
- ✅ Развертывается код приложения
- ✅ Настраивается мониторинг и логирование

2. **Отслеживайте прогресс развертывания** и обратите внимание на создаваемые ресурсы.

### Шаг 2.3: Проверка вашего развертывания

1. **Проверьте развернутые ресурсы:**
```bash

```

2. **Откройте развернутое приложение:**
```bash
azd show --output json | grep "webAppUrl"
```

3. **Протестируйте функциональность ИИ:**
   - Перейдите в веб-приложение

## Модуль 4: Безопасность и лучшие практики продакшена

### Шаг 4.1: Безопасное управление учетными данными

**Актуальная проблема**: Многие приложения ИИ хранят ключи API "вшитыми" или в небезопасном виде.

**Решение AZD**: Управляемая идентичность + интеграция с Key Vault.

1. **Просмотрите конфигурацию безопасности в вашем шаблоне:**
```bash
# Ищите конфигурацию Key Vault и Managed Identity
grep -r "keyVault\|managedIdentity" infra/
```

2. **Проверьте работу управляемой идентичности:**
```bash
# Проверьте, правильно ли настроена идентификация веб-приложения
az webapp identity show --name YOUR_APP_NAME --resource-group YOUR_RG
```

### Шаг 4.2: Сетевая безопасность

1. **Включите приватные конечные точки** (если еще не настроены):

Добавьте в ваш bicep-шаблон:
```bicep
// Private endpoint for OpenAI
resource openAIPrivateEndpoint 'Microsoft.Network/privateEndpoints@2023-04-01' = {
  name: 'pe-openai-${uniqueString(resourceGroup().id)}'
  location: location
  properties: {
    subnet: {
      id: vnet.properties.subnets[0].id
    }
    privateLinkServiceConnections: [
      {
        name: 'openai-connection'
        properties: {
          privateLinkServiceId: openAIAccount.id
          groupIds: ['account']
        }
      }
    ]
  }
}
```

### Шаг 4.3: Мониторинг и наблюдаемость

1. **Настройте Application Insights:**
```bash
# Application Insights должен быть настроен автоматически
# Проверьте конфигурацию:
az monitor app-insights component show --app YOUR_APP_NAME --resource-group YOUR_RG
```

2. **Настройте мониторинг, специфичный для ИИ:**

Добавьте кастомные метрики для операций ИИ:
```bicep
// In your web app configuration
resource webApp 'Microsoft.Web/sites@2023-01-01' = {
  properties: {
    siteConfig: {
      appSettings: [
        {
          name: 'APPLICATIONINSIGHTS_CONNECTION_STRING'
          value: applicationInsights.properties.ConnectionString
        }
        {
          name: 'OPENAI_MONITOR_ENABLED'
          value: 'true'
        }
      ]
    }
  }
}
```

**Следуйте этому шаблону преобразования приложения:**

1. **Создайте базовую структуру:**
```bash
mkdir my-ai-app-azd
cd my-ai-app-azd

# Инициализация шаблона AZD
azd init --template minimal
```

2. **Создайте azure.yaml:**
```yaml
# Metadata
name: my-ai-app
metadata:
  template: my-ai-app-template@0.0.1-beta

# Services definition
services:
  api:
    project: ./api
    host: containerapp
  web:
    project: ./web
    host: staticwebapp
    
# Hooks for custom deployment logic  
hooks:
  predeploy:
    shell: sh
    run: echo "Preparing AI models..."
```

3. **Создайте шаблоны инфраструктуры:**

**infra/main.bicep** - основной шаблон:
```bicep
@description('Primary location for all resources')
param location string = resourceGroup().location

@description('Name of the OpenAI service')
param openAIServiceName string = 'openai-${uniqueString(resourceGroup().id)}'

// Your AI services here
module openAI 'modules/openai.bicep' = {
  name: 'openai'
  params: {
    name: openAIServiceName
    location: location
  }
}
```

**infra/modules/openai.bicep** - модуль OpenAI:
```bicep
@description('Name of the OpenAI service')
param name string

@description('Location for the OpenAI service')
param location string

resource openAIAccount 'Microsoft.CognitiveServices/accounts@2023-05-01' = {
  name: name
  location: location
  kind: 'OpenAI'
  sku: {
    name: 'S0'
  }
  properties: {
    customSubDomainName: name
  }
}

output endpoint string = openAIAccount.properties.endpoint
output name string = openAIAccount.name
```

## Модуль 6: Устранение распространенных проблем

### Распространенные проблемы при развертывании

#### Проблема 1: Превышена квота сервиса OpenAI  
**Симптомы:** Ошибка квоты во время развертывания  
**Решения:**  
```bash
# Проверьте текущие квоты
az cognitiveservices usage list --location eastus

# Запросите увеличение квоты или попробуйте другой регион
azd env set AZURE_LOCATION westus2
azd up
```
  
#### Проблема 2: Модель недоступна в регионе  
**Симптомы:** Ошибки развертывания модели или отсутствие ответов ИИ  
**Решения:**  
```bash
# Проверить доступность модели по региону
az cognitiveservices model list --location eastus

# Обновить до доступной модели
azd env set AZURE_OPENAI_MODEL gpt-35-turbo-16k
azd deploy
```
  
#### Проблема 3: Проблемы с правами доступа  
**Симптомы:** Ошибки 403 Forbidden при вызове сервисов ИИ  
**Решения:**  
```bash
# Проверить назначения ролей
az role assignment list --scope /subscriptions/YOUR_SUB/resourceGroups/YOUR_RG

# Добавить отсутствующие роли
az role assignment create \
  --assignee YOUR_PRINCIPAL_ID \
  --role "Cognitive Services OpenAI User" \
  --scope /subscriptions/YOUR_SUB/resourceGroups/YOUR_RG
```
