## Обзор

В этой главе рассматриваются настройка окружения, паттерны аутентификации и лучшие практики безопасности для развертываний с помощью Azure Developer CLI.

> Проверено на `azd 1.23.12` в марте 2026 года.

## Цели обучения

Выполнив эту главу, вы научитесь:
- Осваивать иерархию конфигурации AZD
- Управлять несколькими окружениями (dev, staging, prod)
- Реализовывать безопасную аутентификацию с управляемыми идентичностями
- Настраивать параметры, специфичные для окружения

## 🚀 Быстрый старт

```bash
# Создать несколько окружений
azd env new dev
azd env new staging
azd env new prod

# Переключить окружения
azd env select prod

# Установить переменные окружения
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# Просмотреть конфигурацию
azd env get-values
```

---

## 🔧 Иерархия конфигурации

AZD применяет настройки в следующем порядке (последние переопределяют предыдущие):

1. **Значения по умолчанию** (встроенные в шаблоны)
2. **azure.yaml** (конфигурация проекта)
3. **Переменные окружения** (`azd env set`)
4. **Флаги командной строки** (`--location eastus`)

---

## 🔐 Лучшие практики безопасности

```bash
# Используйте управляемую идентичность (рекомендуется)
azd env set AZURE_USE_MANAGED_IDENTITY true

# Проверьте статус аутентификации AZD
azd auth status

# По желанию: проверьте контекст Azure CLI, если планируете выполнять команды az
az account show

# Выполните повторную аутентификацию при необходимости
azd auth login

# По желанию: обновите аутентификацию Azure CLI для команд az
az login
```
