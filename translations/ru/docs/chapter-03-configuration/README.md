## Обзор

В этой главе рассматривается настройка окружения, шаблоны аутентификации и лучшие практики безопасности для развертываний с помощью Azure Developer CLI.

> Проверено на `azd 1.27.1` в июле 2026 года.

## Цели обучения

По завершении этой главы вы сможете:
- Освоить иерархию конфигурации AZD
- Управлять несколькими окружениями (dev, staging, prod)
- Реализовать безопасную аутентификацию с управляемыми идентичностями
- Настраивать параметры, специфичные для окружения

## 🚀 Быстрый старт

```bash
# Создать несколько сред
azd env new dev
azd env new staging
azd env new prod

# Переключить среды
azd env select prod

# Установить переменные среды
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# Просмотреть конфигурацию
azd env get-values
```

---

## 🔧 Иерархия конфигурации

AZD применяет настройки в следующем порядке (последующие переопределяют предыдущие):

1. **Значения по умолчанию** (встроенные в шаблоны)
2. **azure.yaml** (конфигурация проекта)
3. **Переменные окружения** (`azd env set`)
4. **Параметры командной строки** (`--location eastus`)

---

## 🔐 Лучшие практики безопасности

```bash
# Используйте управляемую идентичность (рекомендуется)
azd env set AZURE_USE_MANAGED_IDENTITY true

# Проверьте статус аутентификации AZD
azd auth status

# Опционально: проверьте контекст Azure CLI, если планируете запускать команды az
az account show

# Повторно выполните аутентификацию при необходимости
azd auth login

# Опционально: обновите аутентификацию Azure CLI для команд az
az login
```
