# அத்தியாயம் 6: வெளியீட்டிற்கு முன் திட்டமிடல் மற்றும் சரிபார்த்தல்

**📚 பாடநெறி**: [ஆரம்பகாரர்களுக்கான AZD](../../README.md) | **⏱️ கால அளவு**: 1 மணி நேரம் | **⭐ சிக்கல்தன்மை**: நடுத்தர

---

## கண்ணோட்டம்

இந்த அத்தியாயம் உங்கள் பயன்பாட்டை வெளியிடுவதற்கு முன் அவசியமான திட்டமிடல் மற்றும் சரிபார்ப்பு படிகளைcovers செய்கிறது. பொருத்தமான திறன் திட்டமிடல், SKU தேர்வு மற்றும் முன்னேற்பாடு சோதனைகள் மூலம் படப்பிடிப்பான தவறுகளைத் தவிர்க்க கற்றுக் கொள்ளுங்கள்.

## கற்றல் நோக்கங்கள்

இந்த அத்தியாயத்தை முடித்தவுடன், நீங்கள்:
- வெளியீட்டு முன் முன்னேற்பாடு சோதனைகளை இயக்கலாம்
- திறன் திட்டமிடல் மற்றும் வள தேவைகளை மதிப்பிடலாம்
- செலவு சீரமைப்பிற்காக பொருத்தமான SKUs தேர்ந்தெடுக்கலாம்
- கண்காணிப்பிற்காக Application Insights அமைக்கலாம்
- குழு ஒருங்கிணைப்பு முறைமைகள் பற்றி புரிந்துகொள்ளலாம்

## 🚀 விரைவு துவக்கம்

```bash
# சப்ஸ்கிரிப்ஷன் குவோட்டாக்களை சரிபார்க்க
az vm list-usage --location eastus --output table

# நிறுவலின் முன்னோட்டம் (எந்த வளமும் உருவாக்கப்பட மாட்டாது)
azd provision --preview

# Bicep இலக்கணத்தை சரிபார்க்க
az bicep build --file infra/main.bicep

# சூழல் கட்டமைப்பை சரிபார்க்க
azd env get-values
```

## ☑️ வெளியீட்டிற்கு முன் சரிபார்ப்பு

### Before `azd provision`

- [ ] பிராந்தியத்திற்கு குவோட்டா உறுதிசெய்யப்பட்டது
- [ ] SKUs பொருத்தமாக தேர்ந்தெடுக்கப்பட்டுள்ளன
- [ ] செலவு கணிப்பு பரிசீலிக்கப்பட்டது
- [ ] பெயரிடும் நடைமுறை ஒரே மாதிரியாக உள்ளது
- [ ] பாதுகாப்பு/RBAC அமைக்கப்பட்டது

### Before `azd deploy`

- [ ] சுற்றுச்சூழல் மாறிலிகள் அமைக்கப்பட்டுள்ளன
- [ ] Key Vault இல் ரகசியங்கள் உள்ளன
- [ ] இணைப்பு ஸ்ட்ரிங்குகள் சரிபார்க்கப்பட்டுள்ளன
- [ ] ஆரோக்கியச் சோதனைகள் அமைக்கப்பட்டுள்ளன
