# அத்தியாயம் 6: முன்-பதிவேற்றத் திட்டமிடல் மற்றும் சரிபார்ப்பு

## கண்ணோட்டம்

இந்த அத்தியாயம் உங்கள் செயலியை பதிவேற்றுவதற்கு முன் அவசியமான திட்டமிடல் மற்றும் சரிபார்ப்பு படிகளை காண்கிறது. சரியான திறன் திட்டமிடல், SKU தேர்வு, மற்றும் முன்-பறப்புச் சரிபார்ப்புகள் மூலம் செலவின பிழைகளைத் தவிர்க்க கற்கவும்.

> ஜூன் 2026-இல் `azd 1.25.6` உடன் சரிபார்க்கப்பட்டது.

## கற்றல் குறிக்கோள்கள்

By completing this chapter, you will:
- பணியிட முன் சரிபார்ப்புகளை இயக்குவது
- திறன் பற்றிய திட்டமிடல் மற்றும் வள தேவைகளை மதிப்பிடுவது
- செலவின முன்னோக்கி சரியான SKU-களை தேர்வு செய்வது
- கண்காணிப்புக்காக Application Insights ஐ அமைப்பது
- குழு ஒத்துழைப்பு முறைமைகளைய் 이해ுவது

## 🚀 விரைவு தொடக்கம்

```bash
# சப்ஸ்கிரிப்ஷன் குவோட்டாக்களை சரிபார்க்கவும்
az vm list-usage --location eastus --output table

# டிப்ளாய்மென்டின் முன்னோட்டம் (எந்த வளங்களும் உருவாக்கப்பட மாட்டாது)
azd provision --preview

# Bicep சின்டாக்ஸை சரிபார்க்கவும்
az bicep build --file infra/main.bicep

# சூழல் கட்டமைப்பை சரிபார்க்கவும்
azd env get-values
```

## ☑️ முன்-பதிவேற்ற சரிபார்ப்பு பட்டியல்

### `azd provision` முன்னர்

- [ ] மண்டலத்திற்கான குவோட்டா சரிபார்க்கப்பட்டது
- [ ] SKU-கள் சரியான முறையில் தேர்வுசெய்து உள்ளன
- [ ] கட்டண மதிப்பீடு பரிசீலிக்கப்பட்டது
- [ ] பெயரிடுதல் நடைமுறை இணக்கமானது
- [ ] பாதுகாப்பு/RBAC அமைக்கப்பட்டது

### `azd deploy` முன்னர்

- [ ] சூழல்மாறிகள் அமைக்கப்பட்டுள்ளன
- [ ] Key Vault-இல் ரகசியங்கள் உள்ளன
- [ ] இணைப்பு ஸ்ட்ரிங்குகள் சரிபார்க்கப்பட்டுள்ளன
- [ ] ஆரோக்கியச் சரிபார்ப்புகள் அமைக்கப்பட்டுள்ளன
