# அத்தியாயம் 6: முன்-படைப்பு திட்டமிடல் மற்றும் சரிபார்ப்பு

## மேலோட்டம்

இந்த அத்தியாயம் உங்கள் பயன்பாட்டை பதிப்பிக்க முன்பான அவசியமான திட்டமிடல் மற்றும் சரிபார்ப்பு படிகளை உள்ளடக்குகிறது. சரியான திறன் திட்டமிடல், SKU தேர்வு மற்றும் முன்-பரிசோதனைகள் மூலம் செலவான தவறுகளைத் தவிர்க்க கற்றுக்கொள்ளுங்கள்.

> `azd 1.23.12`-க்கு எதிராக சரிபார்க்கப்பட்டது  மார்ச் 2026.

## கற்றல் இலக்குகள்

இந்த அத்தியாயத்தை முடித்தவுடன், நீங்கள்:
- பதிப்பிக்கும் முன் முன்-பரிசோதனைகளை இயக்க முடியும்
- திறன் திட்டமிடல் மற்றும் வளத் தேவைகளை மதிப்பிட முடியும்
- செலவின நிவாரணத்திற்கு பொருத்தமான SKU-களைத் தேர்ந்தெடுக்க முடியும்
- கண்காணிப்பிற்காக Application Insights-ஐ கட்டமைக்க முடியும்
- அணிப் பரிசர்ப்புக் கொள்கைகளைப் புரிந்துகொள்வீர்கள்

## 🚀 விரைவு துவக்கம்

```bash
# சப்ஸ்கிரிப்ஷன் வரம்புகளை சரிபார்
az vm list-usage --location eastus --output table

# வினியோகத்தின் முன்னோட்டம் (எந்த வளமும் உருவாக்கப்படாது)
azd provision --preview

# Bicep வாக்கிய அமைப்பை சரிபார்
az bicep build --file infra/main.bicep

# சூழல் கட்டமைப்பை சரிபார்
azd env get-values
```

## ☑️ முன்-படைப்பு சரிபார்ப்பு பட்டியல்

### முன் `azd provision`

- [ ] மண்டலத்திற்கு கோட்டா சரிபார்க்கப்பட்டுள்ளது
- [ ] SKU-கள் பொருத்தமாக தேர்வு செய்யப்பட்டுள்ளது
- [ ] செலவு மதிப்பீடு பரிசீலிக்கப்பட்டது
- [ ] பெயரிடும் ஒழுங்கு ஒத்திருக்கிறது
- [ ] பாதுகாப்பு/RBAC அமைக்கப்பட்டுள்ளது

### முன் `azd deploy`

- [ ] சூழல் மாறிகள் அமைக்கப்பட்டுள்ளன
- [ ] Key Vault-இல் ரகசியங்கள் உள்ளன
- [ ] இணைப்பு வரிகள் சரிபார்க்கப்பட்டுள்ளன
- [ ] ஹெல்த் சோதனைகள் அமைக்கப்பட்டுள்ளன
