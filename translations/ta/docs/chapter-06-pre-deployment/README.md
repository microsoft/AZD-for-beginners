# அத்தியாயம் 6: முன்-பாதுகாப்பு திட்டமிடல் மற்றும் சரிபார்ப்பு

**📚 பாடநெறி**: [AZD For Beginners](../../README.md) | **⏱️ கால அளவு**: 1 மணி நேரம் | **⭐ சிக்கல்தன்மை**: நடுத்தர

## கண்ணோட்டம்

உங்கள் செயலியை மாற்றுவதற்கு முன் அவசியமான திட்டமிடல் மற்றும் சரிபார்ப்பு படிகளை இந்த அத்தியாயம் மற்றும் விவரிக்கிறது. சரியான திறன் திட்டமிடல், SKU தேர்வு மற்றும் முன் பயிற்சி சோதனைகளால் செலவுடைய தவறுகளைத் தவிர எப்படி கற்றுக்கொள்ளலாம் என்பதை அறியவும்.

> `azd 1.27.1` இல் 2026 ஜூலை மாதம் சரிபார்க்கப்பட்டது.

## படிப்புக் குறிக்கோள்கள்

இந்த அத்தியாயத்தை முடித்தவுடன், நீங்கள்:
- மாற்றத்திற்கு முன் முன் பயிற்சி சோதனைகள் நடத்துவீர்கள்
- திறன் திட்டமிடல் மற்றும் வள தேவைகளை மதிப்பிடுவீர்கள்
- செலவு மேம்பாட்டிற்கான சரியான SKUகளைத் தேர்வு செய்வீர்கள்
- கண்காணிப்பிற்கு செயலி அறிவும் (Application Insights) அமைப்பீர்கள்
- குழு ஒருங்கிணைப்பு முறைகளைப் புரிந்துகொள்வீர்கள்

## 🚀 விரைவு துவக்கம்

```bash
# சந்தா அளவுகோல்களை சரிபார்க்கவும்
az vm list-usage --location eastus --output table

# பிரதி மேம்பாட்டை முன்னோட்டப்படுத்து (ஒரு வளமும் உருவாகவில்லை)
azd provision --preview

# பைசெப் வாக்கியவியலினை உறுதிப்படுத்தவும்
az bicep build --file infra/main.bicep

# சுற்றுச்சூழல் அமைவுகளை சரிபார்க்கவும்
azd env get-values
```

## ☑️ முன்-மாற்றச் சரிபார்ப்பு பட்டியல்

### `azd provision` முன்

- [ ] பிராந்தியத்திற்கு கொள்ளளவு சரிபார்க்கப்பட்டது
- [ ] SKUகள் பொருத்தமாக தேர்ந்தெடுக்கப்பட்டன
- [ ] செலவுத் மதிப்பீடு ஆய்வு செய்யப்பட்டு உள்ளது
- [ ] பெயரிடும் மாநாட்டில் ஒற்றுமை உள்ளது
- [ ] பாதுகாப்பு/RBAC அமைக்கப்பட்டது

### `azd deploy` முன்

- [ ] சுற்று மாறிகள் அமைக்கப்பட்டுள்ளன
- [ ] Key Vault இல் ரகசியங்கள் உள்ளன
- [ ] இணைப்பு தொடுக்கைகள் சரிபார்க்கப்பட்டுள்ளன
- [ ] ஆரோக்கிய சோதனைகள் அமைக்கப்பட்டுள்ளன
