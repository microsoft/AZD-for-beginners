## மேலோட்டம்

இந்த அத்தியாயம் உங்கள் பயன்பாட்டை வினியோகத்திற்கு முன் அவசியமான திட்டமிடல் மற்றும் சரிபார்ப்பு படிகளை எடுத்துரைக்கிறது. தகுதி வாய்ந்த திறன் திட்டமிடல், SKU தேர்வு மற்றும் முன்னணிசோதனைகள் மூலம் செலவான தவறுகளைத் தவிர்ப்பதை கற்றுக்கொள்ளுங்கள்.

## கற்றல் இலக்குகள்

By completing this chapter, you will:
- வினியோகத்திற்கு முன் முன்னணிசோதனைகளை இயக்குதல்
- திறன் திட்டமிடல் மற்றும் வள தேவைகளை மதிப்பீடு செய்தல்
- செலவுக் குறைப்புக்காக உகந்த SKU-களைத் தேர்வு செய்தல்
- கண்காணிப்பிற்காக Application Insights ஐ கட்டமைத்தல்
- அணி ஒத்துழைப்பு முறைமைகளைப் புரிந்துகொள்ளுதல்

## 🚀 விரைவு தொடக்கம்

```bash
# சப்ஸ்கிரிப்ஷன் வரம்புகளை சரிபார்க்கவும்
az vm list-usage --location eastus --output table

# வினியோகம் முன்னோட்டம் (வளங்கள் உருவாக்கப்பட மாட்டாது)
azd provision --preview

# Bicep வாக்கிய அமைப்பை சரிபார்க்கவும்
az bicep build --file infra/main.bicep

# சுற்றுச்சூழல் கட்டமைப்பை சரிபார்க்கவும்
azd env get-values
```

## ☑️ வினியோகத்திற்கு முன் சரிபார்க்கும் பட்டியல்

### முன் `azd provision`

- [ ] மண்டலத்திற்கு ஒதுக்கீடு சரிபார்க்கப்பட்டது
- [ ] SKU-கள் சரியாக தேர்ந்தெடுக்கப்பட்டன
- [ ] செலவு மதிப்பீடு பரிசீலிக்கப்பட்டது
- [ ] பெயரிடும் நடைமுறை ஒரேநிலையாக உள்ளது
- [ ] பாதுகாப்பு/RBAC கட்டமைக்கப்பட்டது

### முன் `azd deploy`

- [ ] சுற்றுச்சூழல் மாறில்கள் அமைக்கப்பட்டுள்ளன
- [ ] Key Vault இல் ரகசியங்கள் உள்ளன
- [ ] இணைப்பு ஸ்டிரிங்குகள் சரிபார்க்கப்பட்டன
- [ ] ஆரோக்கியச் சோதனைகள் அமைக்கப்பட்டன
