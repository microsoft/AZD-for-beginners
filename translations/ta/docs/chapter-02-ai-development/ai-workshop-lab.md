## பணிமனை கண்ணோட்டம்

இந்த கைபழக்கம் ஆய்வகம் டெவலப்பர்களை ஏற்கனவே உள்ள AI மாதிரியை எடுத்துக் கொண்டு Azure Developer CLI (AZD) பயன்படுத்தி அதை பணியிடும் செயல்முறையில் வழிநடத்துகிறது. நீங்கள் Microsoft Foundry சேவைகளைப் பயன்படுத்தி தயாரிப்பு AI பணியிடலுக்கான அவசியமான நடைமுறைகளை கற்றுக்கொள்ளப்போகிறீர்கள்.

> **சரிபார்ப்பு குறிப்பு (2026-07-13):** இந்த பணிமனை `azd` `1.27.1`-க்கு எதிராக மதிப்பாய்வு செய்யப்பட்டது. உங்கள் உள்ளூர் நிறுவல் பழம்பெருந்ததுவானால், AZD-வை புதுப்பித்து துவங்குங்கள், அப்போத்தான் அங்கீகாரம், மாதிரி, பணியிடல் பணிஆடி கீழ்காணும் கட்டளைகளுடன் பொருந்தும்.

## பகுதி 1: AI பயன்பாடுகளுக்கான AZD கட்டமைப்பை புரிதல்

### AI AZD மாதிரியின் கட்டமைப்பு

AI-தயாரிக்கப்பட்ட AZD மாதிரியில் முக்கிய கோப்புகளை ஆய்வு செய்யவும்:

```
azure-search-openai-demo/
├── azure.yaml              # AZD configuration
├── infra/                   # Infrastructure as Code
│   ├── main.bicep          # Main infrastructure template
│   ├── main.parameters.json # Environment parameters
│   └── modules/            # Reusable Bicep modules
│       ├── openai.bicep    # Microsoft Foundry Models configuration
│       ├── search.bicep    # Azure AI Search setup
│       └── webapp.bicep    # Web app configuration
├── app/                    # Application code
├── scripts/               # Deployment scripts
└── .azure/               # AZD environment files
```

### **ஆய்வகம் பயிற்சி 1.1: பொருத்தலை ஆராய்க**

1. **azure.yaml கோப்பை பரிசீலிக்கவும்:**
```bash
cat azure.yaml
```

**என்ன பார்க்க வேண்டும்:**
- AI கூறுகளுக்கான சேவை வரையறைகள்
- சூழல் மாறிலி ஒதுக்கீடுகள்
- ஹோஸ்ட் கட்டமைப்புகள்

2. **main.bicep கட்டமைப்பை பரிசீலிக்கவும்:**
```bash
cat infra/main.bicep
```

**அதிக முக்கிய AI முறைகள் கண்டுபிடிக்க:**
- Microsoft Foundry Models சேவை வழங்கல்
- Azure AI தேடல் ஒருங்கிசைவு
- பாதுகாப்பான விசை மேலாண்மை
- வலையமைப்பு பாதுகாப்பு கட்டமைப்புகள்

### **ைகைச்சுருக்கம்:** இவை AI க்கான எதற்கு முக்கியம்

- **சேவை சார்புகள்**: AI பயன்பாடுகள் பல நிலையான சேவைகளைக் கோருகின்றன
- **பாதுகாப்பு**: API விசைகள் மற்றும் முனையங்கள் பாதுகாப்பான மேலாண்மையை தேவைபடுகின்றன
- **தரவு திறனுக்கேற்ற அளவுகோல்**: AI பணி சுமைகளுக்கு தனித்துவமான அளவுகோல் தேவை
- **செலவு மேலாண்மை**: AI சேவைகள் சரியான அமைப்பில்லாமல் அதிக செலவு ஆகலாம்

## பகுதி 2: உங்கள் முதல் AI பயன்பாட்டை பணியிடல்

### கட்டளை 2.1: சூழலை துவக்கி அமைக்க

1. **புதிய AZD சூழலை உருவாக்கவும்:**
```bash
azd env new myai-workshop
```

2. **தேவையான அளவுருக்களை அமைக்கவும்:**
```bash
# உங்கள் விருப்பமான Azure பிராந்தியத்தை அமைக்கவும்
azd env set AZURE_LOCATION eastus

# விருப்பத்தின் படி: குறிப்பிட்ட OpenAI மாதிரியை அமைக்கவும்
azd env set AZURE_OPENAI_MODEL gpt-4.1-mini
```

### கட்டளை 2.2: கட்டமைப்பு மற்றும் பயன்பாட்டை பணியிடுங்கள்

1. **AZD மூலம் Deploy செய்யவும்:**
```bash
azd up
```

**`azd up` இடையே நிகழும் செயல்கள்:**
- ✅ Microsoft Foundry Models சேவையை வழங்குதல்
- ✅ Azure AI தேடல் சேவையை உருவாக்குதல்
- ✅ வலை பயன்பட்ட சேவையை உருவாக்கல்
- ✅ வலையமைப்பு மற்றும் பாதுகாப்பு அமைப்பு
- ✅ பயன்பாட்டு குறியீட்டை பணியிடல்
- ✅ கண்காணிப்பு மற்றும் பதிவேடு அமைப்பு

2. **Deploy முன்னேற்றத்தை கண்காணி**வும் உருவாக்கப்பட்ட வளங்களை கவனியுங்கள்.

### கட்டளை 2.3: உங்கள் Deploy பாத்தறிக்கை

1. **Deploy செய்யப்பட்ட வளங்களை சரிபார்க்கவும்:**
```bash
azd show
```

2. **Deploy செய்யப்பட்ட பயன்பாட்டை திறக்கவும்:**
```bash
azd show
```

`azd show` வெளியீட்டில் காணப்படும் வலை முனையை திறக்கவும்.

3. **AI செயல்திறனை சோதிக்கவும்:**
   - வலை பயன்பாட்டை அணுகவும்
   - மாதிரி கேள்விகளை முயற்சிக்கவும்
   - AI பதில்கள் செயல்பாட்டை உறுதிசெய்யவும்

### **ஆய்வகம் பயிற்சி 2.1: பிரச்சனை தீர்க்கும் பயிற்சி**

**நிலைமை**: உங்கள் Deploy வெற்றி பெற்றது ஆனால் AI பதில் தரவில்லை.

**பொதுவான பிரச்சனைகள் சரிபார்க்க:**
1. **OpenAI API விசைகள்**: அவை சரியாக அமைக்கப்பட்டுள்ளனவா?
2. **மாதிரி கிடைப்பாடு**: உங்கள் பிராந்தியத்தில் மாதிரி ஆதரவு உள்ளதா எனை காண்க
3. **நெட்வொர்க் இணைப்பு**: சேவைகள் தொடர்பு கொள்ளுமா என்பதை உறுதிப்படுத்துக
4. **RBAC அனுமதிகள்**: பயன்பாடு OpenAI அணுகுமுறை பெற்றுள்ளதா என்பதை சரிபார்க்கவும்

**பிழை கண்டறிதல் கட்டளைகள்:**
```bash
# சுற்றுச்சூழல் மாறிகள் சோதிக்கவும்
azd env get-values

# 배포 பதிவுகளை காண்க
az webapp log tail --name YOUR_APP_NAME --resource-group YOUR_RG

# OpenAI 배포 நிலையை சரிபார்க்கவும்
az cognitiveservices account deployment list --name YOUR_OPENAI_NAME --resource-group YOUR_RG
```

## பகுதி 3: உங்கள் தேவைகளுக்கு AI பயன்பாடுகளை தனிப்பயனாக்கல்

### கட்டளை 3.1: AI பொருத்தலை மாற்றுதல்

1. **OpenAI மாதிரியை புதுப்பிக்கவும்:**
```bash
# வேறொரு மாதிரிக்கு மாற்றவும் (உங்கள் பகுதியிலும் அனுமதிக்கப்பட்டிருந்தால்)
azd env set AZURE_OPENAI_MODEL gpt-4.1

# புதிய கட்டமைப்புடன் மீண்டும் நிலைநிறுத்தவும்
azd deploy
```

2. **மேலும் AI சேவைகளைச் சேர்க்கவும்:**

Document Intelligence சேர்க்க infra/main.bicep திருத்து:

```bicep
// Add to main.bicep
resource documentIntelligence 'Microsoft.CognitiveServices/accounts@2023-05-01' = {
  name: 'doc-intel-${uniqueString(resourceGroup().id)}'
  location: location
  kind: 'FormRecognizer'
  sku: {
    name: 'F0'  // Free tier for workshop
  }
  properties: {
    customSubDomainName: 'doc-intel-${uniqueString(resourceGroup().id)}'
  }
}
```

### கட்டளை 3.2: சூழல்-சிறப்பான பொருத்தல்கள்

**சிறந்த நடைமுறை**: வளர்ச்சி மற்றும் தயாரிப்பிற்கான வித்தியாசமான பொருத்தல்கள்.

1. **தயாரிப்பு சூழலை உருவாக்கவும்:**
```bash
azd env new myai-production
```

2. **தயாரிப்பு-சூழல் சிறப்பான அளவுருக்களை அமைக்கவும்:**
```bash
# உற்பத்தி பொதுவாக உயர் SKU-களை பயன்படுத்துகிறது
azd env set AZURE_OPENAI_SKU S0
azd env set AZURE_SEARCH_SKU standard

# கூடுதல் பாதுகாப்பு அம்சங்களை இயக்கு
azd env set ENABLE_PRIVATE_ENDPOINTS true
```

### **ஆய்வகம் பயிற்சி 3.1: செலவுத்தன்மை மேம்பாடு**

**சவால்**: வளர்ச்சிக்கான செலவுத்தன்மையான மாதிரியை அமைப்பது.

**பணிகள்:**
1. இலவச/அடிப்படை நிலைகளுக்கு எந்த SKU-களை மாற்றலாம் என கண்டறிதல்
2. குறைந்த செலவுக்கான சூழல் மாறிலி அமைப்புகள்
3. தயாரிப்பு வடிவத்துடன் செலவுகளை ஒப்பிடு Deploy செய்

**தீர்வு குறிப்புகள்:**
- Azure AI சேவைகளுக்கு வாய்ப்பு இருந்தால் F0 (இலவச) நிலை பயன்படுத்தவும்
- வளர்ச்சிக்கான தேடல் சேவைக்கு அடிப்படை நிலை பயன்படுத்தவும்
- செயலிகளுக்கு Consumption திட்டத்தை பயன்படுத்த பரிசீலனை செய்யவும்

## பகுதி 4: பாதுகாப்பு மற்றும் தயாரிப்பு சிறந்த நடைமுறைகள்

### கட்டளை 4.1: பாதுகாப்பான அங்கீகார மேலாண்மை

**தற்போது சவால்**: பல AI பயன்பாடுகள் API விசைகளை கோடு அல்லது பாதுகாப்பற்ற சேமிப்பில் கடைப்பிடிக்கின்றன.

**AZD தீர்வு**: மேலாண்மை அடையாளம் + விசை குவியலுடன் ஒருங்கிசைவு.

```bash

grep -r "keyVault\|managedIdentity" infra/
```

2. **மேலாண்மை அடையாளம் இயங்குகிறதா என்பதை உறுதிசெய்யவும்:**

```bash
# வலைப் பயன்பாட்டிற்கு சரியான அடையாளக் கட்டமைப்பு உள்ளதா என்பதை சரிபார்க்கவும்
```

### கட்டளை 5.1: மதிப்பீட்டுப் படிவம்

**உங்கள் பயன்பாட்டை மாற்றுவதற்கு முன்னர்**, இந்த கேள்விகளுக்கு பதில் அளியுங்கள்:

   - எந்த கணினி வளங்கள் தேவைப்படுகின்றன?
   - தரவுத்தளம் தேவைபடுகிறதா?
   - சேவைகளுக்கு இடையேயான சார்புகளேன?

   - உங்கள் பின்புலம் எந்த ஒழுங்குமுறை தேவைகள் உடையது?

3. **அளவுகோல் தேவைகள்:**
   - எவ்வளவு சுமையை எதிர்பார்க்கிறீர்கள்?
   - தானாக அளவு மாற்றம் தேவையா?
   - பிராந்திய தேவைகள் உள்ளதா?

2. **செலவு கட்டுப்பாடுகளை அமல்படுத்தவும்:**
- பட்ஜெட் எச்சரிக்கைகள் அமைக்கவும்
- தானியங்கி அளவுக் கொள்கைகள் பயன்படுத்தவும்
- கோரிக்கை கேஷிங் ஆதிக்கம் செய்க
- OpenAI குறியீட்டை கண்காணிக்கவும்

### **ஆய்வகம் பயிற்சி 7.1: செயல்திறன் மேம்பாடு**

**பணி**: உங்கள் AI பயன்பாட்டை செயல்திறன் மற்றும் செலவு இரண்டாகவும் மேம்படுத்தவும்.

**மேம்படுத்த வேண்டிய அளவுகோல்கள்:**
- சராசரி பதில் நேரத்தை 20% குறைக்கவும்
- மாதாந்திர செலவை 15% குறைக்கவும்
- 99.9% நேர மேலாண்மையை பராமரிக்கவும்

**செயல்திட்டங்கள்:**
- பதில் கேச்சிங் அமல்படுத்து
- குறியீட்டு திறனை மேம்படுத்த பதில்களை கட்டமை
- பொருத்தமான கணினி SKU-களை பயன்படுத்தவும்
- சரியான தானியங்கி அளவுக் கொள்கைகளை அமைக்கவும்

நீங்கள் தயாரிப்பு நிலையான AI இயக்கு வாடிக்கையாளர் சேவை டொயை உருவாக்கும் பொறுப்புடன் இருக்கிறீர்கள், கீழ்காணும் தேவைகளுடன்:

**செயல்திறன் தேவைகள்:**
- வாடிக்கையாளர் தொடர்புகளுக்கு வலை இடைமுகம்
- பதில்களுக்கான Microsoft Foundry Models உடன் ஒருங்கிணைவு
- Azure AI தேடலை பயன்படுத்தி ஆவண தேடல் திறன்
- ஏற்கனவே உள்ள வாடிக்கையாளர் தரவுத்தளத்துடன் ஒருங்கிணைவு
- பன்மொழி ஆதரவு

**இல்லாத-செயல்திறன் தேவைகள்:**
- 1000 ஒரே சமயத்தில் பயனாளரை கையாளுதல்
- 99.9% நேர இயக்க நம்பகத்தன்மை
- SOC 2 ஒத்துழைப்பு
- $500/மாதத்திற்கு கீழ் செலவு
- பல சூழல்களுக்கு (வளர்ச்சி, பரிசோதனை, தயாரிப்பு) Deploy செய்தல்
