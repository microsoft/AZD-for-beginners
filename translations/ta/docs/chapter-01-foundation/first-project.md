## அறிமுகம்

உங்கள் முதல் Azure Developer CLI திட்டத்திற்கு வரவேற்கிறோம்! azd பயன்படுத்தி Azure-ல் ஒரு முழு-ஸ்டாக் பயன்பாட்டை உருவாக்குதல், வெளியிடுதல் மற்றும் நிர்வகிப்பதற்கான முழுமையான நடைமுறை பயிற்சி இது. இது React முன்னிணை, Node.js API பின்னிணை மற்றும் MongoDB தரவுத்தளத்தைக் கொண்ட ஒரு உண்மையான todo பயன்பாட்டுடன் நீங்கள் பணியாற்றுவீர்கள்.

## கற்றல் குறிக்கோள்கள்

இந்த பயிற்சியை முடித்தவுடன், நீங்கள்:
- மாதிரிகளை (templates) பயன்படுத்தி azd திட்ட ஆரம்பத்திடை செயல்முறையில் தேர்ச்சி பெறுவீர்கள்
- Azure Developer CLI திட்ட அமைப்பு மற்றும் கட்டமைப்பு கோப்புகளை புரிந்துகொள்ளுவீர்கள்
- இன்பிராஸ்ட்ரக்சர் ஒதுக்கீட்டுடன் Azure-க்கு முழு பயன்பாட்டை வெளியிடுவீர்கள்
- பயன்பாட்டு புதுப்பிப்புகள் மற்றும் மீண்டும் வெளியீடு (redeployment) மூலோபாயங்களை நடைமுறைப்படுத்துவீர்கள்
- வளர்ச்சி மற்றும் பரிசோதனை (staging) போன்ற பல சூழல்களை நிர்வகிப்பீர்கள்
- வளங்களின் சுத்திகரிப்பு மற்றும் செலவு மேலாண்மை நடைமுறைகளை பொருந்தச் செய்பவராக இருப்பீர்கள்

## கற்றல் முடிவுகள்

இதை முடித்தவுடன், நீங்கள் சுவரொன்று செய்ய முடியும்:
- மாதிரிகளைக் கொண்டு azd திட்டங்களை தனக்கே துவக்கி கட்டமைக்க அறிவாளியாக இருப்பீர்கள்
- azd திட்டங்களுக்கு உரிய கட்டமைப்புகளை திறந்தறிந்து திருத்தவல்லவர் ஆயிருப்பீர்கள்
- ஒரு கட்டளை மூலம் முழு-ஸ்டாக் பயன்பாட்டை Azure-க்கு வெளியிடவல்லவர் ஆகுவீர்கள்
- பொதுவான வெளியீடு சிக்கல்களை மற்றும் அங்கீகாரம் தொடர்பான பிரச்சனைகளை தீர்க்கறிந்திருப்பீர்கள்
- வெவ்வேறு வெளியீட்டு நிலைகளுக்கான பல Azure சூழல்களை நிர்வகிக்கிறீர்கள்
- பயன்பாட்டு புதுப்பிப்புகளுக்கான தொடர்ச்சியான வெளியீட்டு பணிமுறைகளை (CI/CD) நடைமுறைப்படுத்துவீர்கள்

## படி 1: ஒரு மாதிரியை தேர்வு செய்து துவக்குங்கள்

React முன்னிணையும் Node.js API பின்னிணையும் கொண்ட பிரபலமான todo பயன்பாட்டு மாதிரியைத் துவக்குவோம்.

```bash
# கிடைக்கும் வார்ப்புருக்களை உலாவவும்
azd template list

# todo செயலி வார்ப்புருவை தொடக்கவும்
mkdir my-first-azd-app
cd my-first-azd-app
azd init --template todo-nodejs-mongo

# வினாக்களைப் பின்பற்றவும்:
# - ஒரு சூழலின் பெயரை உள்ளிடவும்: "dev"
# - ஒரு சந்தாவை தேர்ந்தெடுக்கவும் (உங்களுக்கு பல இருந்தால்)
# - ஒரு பிராந்தியத்தை தேர்ந்தெடுக்கவும்: "East US 2" (அல்லது உங்கள் விருப்பமான பிராந்தியம்)
```

### இப்போது என்ன நடந்தது?
- உங்கள் உள்ளூரில் மாதிரி குறியீட்டை பதிவிறக்கப்பட்டது
- சேவைகள் வரையறைகளை கொண்ட `azure.yaml` கோப்பு உருவாக்கப்பட்டது
- `infra/` அடைவில் இன்ப்ராஸ்ட்ரக்சர் குறியீடு அமைக்கப்பட்டது
- ஒரு சூழல் கட்டமைப்பு இட்டை உருவாக்கப்பட்டது

## படி 2: திட்ட அமைப்பை ஆராயுங்கள்

azd எங்களுக்கு உருவாக்கியவற்றை பார்ப்போம்:

```bash
# திட்டத்தின் அமைப்பை பார்க்கவும்
tree /f   # விண்டோஸ்
# அல்லது
find . -type f | head -20   # macOS/லினக்ஸ்
```

நீங்கள் இதைப் பார்க்க வேண்டும்:
```
my-first-azd-app/
├── .azd/
│   └── config.json              # Project configuration
├── .azure/
│   └── dev/                     # Environment-specific files
├── .devcontainer/               # Development container config
├── .github/workflows/           # GitHub Actions CI/CD
├── .vscode/                     # VS Code settings
├── infra/                       # Infrastructure as code (Bicep)
│   ├── main.bicep              # Main infrastructure template
│   ├── main.parameters.json     # Parameters for deployment
│   └── modules/                # Reusable infrastructure modules
├── src/
│   ├── api/                    # Node.js backend API
│   │   ├── src/               # API source code
│   │   ├── package.json       # Node.js dependencies
│   │   └── Dockerfile         # Container configuration
│   └── web/                   # React frontend
│       ├── src/               # React source code
│       ├── package.json       # React dependencies
│       └── Dockerfile         # Container configuration
├── azure.yaml                  # azd project configuration
└── README.md                   # Project documentation
```

### புரிந்துகொள்ள வேண்டிய முக்கிய கோப்புகள்

**azure.yaml** - உங்கள் azd திட்டத்தின் இதயம்:
```bash
# திட்டத்தின் கட்டமைப்பைப் பார்க்கவும்
cat azure.yaml
```

**infra/main.bicep** - இன்ப்ராஸ்ட்ரக்சர் வரையறை:
```bash
# உள்கட்டமைப்பு குறியீட்டை காண்க
head -30 infra/main.bicep
```

## படி 3: உங்கள் திட்டத்தை தனிப்பயனாக்குங்கள் (விருப்பத்தேர்வு)

வெளியிடுவதற்கு முன், பயன்பாட்டை தனிப்பயனாக்கலாம்:

### முன்னிணையை மாற்றவும்
```bash
# React பயன்பாட்டு கூறைத் திறக்கவும்
code src/web/src/App.tsx
```

ஒரு எளிய மாற்றத்தை செய்யவும்:
```typescript
// தலைப்பை கண்டுபிடித்து அதை மாற்றுங்கள்
```

### சுற்றுச்சூழல் மாறிலிகளை கட்டமைக்கவும்

```bash
# அனைத்து சுற்றுச்சூழல் மாறிகளையும் காண்க
```

## படி 4: Azure-க்கு வெளியிடுக

இப்போது சிறந்த பகுதி - அனைத்தையும் Azure-க்கு வெளியிடுங்கள்!

```bash
# அடித்தள கட்டமைப்பையும் பயன்பாட்டையும் வெளியிடுக
azd up

# இந்த கட்டளை இதை செய்யும்:
# 1. Azure வளங்களை ஏற்பாடு செய்வது (App Service, Cosmos DB மற்றும் மற்றவை)
# 2. உங்கள் பயன்பாட்டை உருவாக்கும்
# 3. நிறுவப்பட்ட வளங்களுக்கு உங்கள் பயன்பாட்டை வெளியிடும்
# 4. பயன்பாட்டின் URL-ஐ காண்பிக்கும்
```

### வெளியீடு நடக்கும் போது என்ன நடக்கிறது?

`azd up` கட்டளை இவைகளை செய்கிறது:
1. **ஒதுக்கீடு** (`azd provision`) - Azure வளங்களை உருவாக்குகிறது
2. **இடைப்பெட்டி** - உங்கள் பயன்பாட்டு குறியீட்டை கட்டமைக்கிறது
3. **வெளியீடு** (`azd deploy`) - குறியீட்டை Azure வளங்களுக்கு திறக்கிறது

### எதிர்பார்க்கப்படும் வெளியீடு
```

https://portal.azure.com/#@/resource/subscriptions/{subscription-id}/resourceGroups/rg-my-first-azd-app-dev

Navigate to the Todo app at:

```

## படி 5: உங்கள் பயன்பாட்டை சோதிக்கவும்

### உங்கள் பயன்பாட்டிற்கு அணுகுதல்
வெளியீட்டு வெளியீட்டில் கொடுக்கப்பட்ட URL ஐ கிளிக் செய்யவும், அல்லது எப்போதும் பெறவும்:
```bash
# விண்ணப்பத்தின் எண்ட்பாயின்டுகளைப் பெறவும்
azd show

# உங்கள் உலாவியில் விண்ணப்பத்தை திறக்கவும்
azd show --output json | jq -r '.services.web.endpoint'
```

### Todo பயன்பாட்டை சோதிக்கவும்
1. **ஒரு todo உருப்படியை சேர்க்கவும்** - "Add Todo" என்பதைக் கிளிக் செய்து ஒரு பணியை பதிவு செய்யவும்
2. **முழுமையாக்கமாக ნიშნிக்கவும்** - முடிந்த உருப்படிகளை சரிபார்க்கவும்
3. **உருப்படிகளை நீக்கவும்** - தேவையில்லாத todo-வை நீக்கவும்

### உங்கள் பயன்பாட்டை கண்காணிக்கவும்
```bash
# உங்கள் வளங்களுக்கான Azure போர்டலை திறக்கவும்
azd monitor

# பயன்பாட்டு பதிவுகளைப் பார்க்கவும்
azd monitor --logs

# நேரடி அளவீடுகளைப் பார்க்கவும்
azd monitor --live
```

### ✅ உங்கள் வெளியீட்டை உறுதிசெய்யவும்

தொடர்வதற்கு முன், அனைத்தும் நன்கு செயல்பட்டு இருப்பதை உறுதிசெய்ய இந்த சுருக்கப்பட்ட சரிபார்ப்பு பட்டியலை இயக்கவும் — "deploy succeeded" என்பது "app works" என்று கருத வேண்டாம்:

```bash
# 1. எண்ட்பாயிண்ட் உள்ளது மற்றும் அணுகக்கூடியது என்பதை உறுதிசெய்
azd show

# 2. எண்ட்பாயிண்டைப் ஸ்மோக்-டெஸ்ட் செய் (HTTP 200 எதிர்பார்க்கப்படுகிறது)
curl -I "$(azd show --output json | jq -r '.services.web.endpoint')"

# 3. உங்கள் செயலி ஹெல்த் எண்ட்பாயிண்ட் ஒன்றை வெளிப்படுத்தினால் அதை சரிபார்க்கவும்
curl "$(azd show --output json | jq -r '.services.web.endpoint')/health"
```

**வெளியீடு சரிபார்க்கப்படும் போது:**
- ✅ `azd show` ஒரு அணுகக்கூடிய endpoint URL ஐ பட்டியலிடுகிறது
- ✅ அந்த URL உங்கள் உலாவியில் பிழையில்லாமல் திறக்கிறது
- ✅ அடிப்படை அம்சங்கள் வேலை செய்கின்றன (சேர்க்க/முழுமையாக்க/நீக்க)
- ✅ `azd monitor --logs` கோரிக்கைகள் வந்ததை எதிர்பாராத பிழைகளின்றி காட்டுகிறது

ஏதேனுமொரு சோதனை தோல்வியடைந்தால், [அத்தியாயம் 7: சிக்கல் தீர்வு](../chapter-07-troubleshooting/README.md) இல் குதிக்கவும்.

## படி 6: மாற்றங்கள் செய்து மறுதொகை (Redeploy) செய்யவும்

ஒரு மாற்றம் செய்து அதனை எவ்வளவு எளிதில் புதுப்பிக்க முடிகிறது என்று பார்க்கலாம்:

### API ஐ மாற்றவும்
```bash
# API குறியீட்டை திருத்தவும்
code src/api/src/routes/lists.js
```

ஒரு தனிப்பயன் பதில் தலைப்பை சேர்க்கவும்:
```javascript
// ஒரு ரூட் ஹேண்ட்லரை கண்டுபிடித்து சேர்க்கவும்:
res.header('X-Powered-By', 'Azure Developer CLI');
```

### குறியீட்டு மாற்றங்களை மட்டும் வெளியிடவும்
```bash
# விண்ணப்பக் குறியீட்டையே மட்டும் வினியோகிக்கவும் (அடிப்படை கட்டமைப்பைத் தவிர்க்கவும்)
azd deploy

# அடிப்படை கட்டமைப்பு ஏற்கனவே இருப்பதால் இது 'azd up'-ஐவிட மிகவும் வேகமாகும்
```

## படி 7: பல சுற்றுச்சூழல்களை நிர்வகிக்கவும்

வேலைக்கு முன் மாற்றங்களை சோதிக்க ஸ்டேஜிங் சுற்றுச்சூழலை உருவாக்குங்கள்:

```bash
# புதிய ஸ்டேஜிங் சுற்றுச்சூழலை உருவாக்கவும்
azd env new staging

# ஸ்டேஜிங் சுற்றுச்சூழலுக்கு வெளியிடவும்
azd up

# மீண்டும் dev சுற்றுச்சூழலுக்கு மாறவும்
azd env select dev

# அனைத்து சுற்றுச்சூழல்களையும் பட்டியலிடவும்
azd env list
```

### சுற்றுச்சூழை ஒப்பீடு
```bash
# டெவ் சூழலை பார்க்கவும்
azd env select dev
azd show

# ஸ்டேஜிங் சூழலை பார்க்கவும்
azd env select staging
azd show
```

## படி 8: வளங்களை சுத்தம் செய்யவும்

சோதனை செய்த பிறகு, தொடர்ச்சியான கட்டணங்களைத் தவிர்க்க வளங்களை சுத்தம் செய்யவும்:

```bash
# தற்போதைய சூழலுக்கான அனைத்து Azure வளங்களையும் நீக்கவும்
azd down

# உறுதிப்படுத்தலைக் கேட்காமல் வலியுறுத்தி நீக்கவும் மற்றும் மென்மையாக நீக்கப்பட்ட வளங்களை நிர்மூலமாக அகற்றவும்
azd down --force --purge

# குறிப்பிட்ட சூழலை நீக்கவும்
azd env select staging
azd down --force --purge
```

## பாரம்பரிய பயன்பாடு vs. AI-சக்தி வாய்ந்த பயன்பாடு: செயன்முறை ஒரே மாதிரி

நீங்கள் appena ஒரு பாரம்பரிய வலைப் பயன்பாட்டை வெளியிட்டது. ஆனால் AI-சக்தி வாய்ந்த பயன்பாட்டை (உதாரணத்திற்கு Microsoft Foundry Models-ஐ பயன்படுத்திய அரட்டை செயலி) வெளியிட விரும்பினால் என்ன?

மகிழ்ச்சியான செய்தி: **செயல்முறை அதேதே.**

| படி | பாரம்பரிய Todo பயன்பாடு | AI அரட்டை பயன்பாடு |
|------|-------------------------|--------------------|
| துவக்கம் | `azd init --template todo-nodejs-mongo` | `azd init --template azure-search-openai-demo` |
| அங்கீகாரம் | `azd auth login` | `azd auth login` |
| வெளியீடு | `azd up` | `azd up` |
| கண்காணி | `azd monitor` | `azd monitor` |
| சுத்தம் செய் | `azd down --force --purge` | `azd down --force --purge` |

தேர்வான வேறுபாடு நீங்கள் தொடங்கும் **மாதிரி** தான். ஒரு AI மாதிரி கூடுதல் இன்ப்ராஸ்ட்ரக்சரை (உதாரணமாக Microsoft Foundry Models வளம் அல்லது AI Search ஒழுங்கு) கொண்டிருக்கலாம், ஆனால் azd அவற்றை அனைத்தையும் கையாளும். புதிய கட்டளைகளை கற்றுக்கொள்ள அல்லது வேறு கருவியை ஏற்க அல்லது வெளியீட்டு முறையை மாற்ற தேவையில்லை.

இது azd-வின் மூலக் கோட்பாடு: **ஒரே பணிநடை, எந்தப் பணியும்.** இந்த பயிற்சியில் நீங்கள் பயிற்சி பெற்றிருந்த திறன்கள் — துவக்குதல், வெளியீடு, கண்காணித்தல், மறுபதிவு மற்றும் சுத்திகரிப்பு — AI பயன்பாடுகளுக்கும் முகவர்கள் கூடிப்போன்றவையாகும்.

```bash
cd my-first-azd-app

# தனிப்பயன் சூழலை உருவாக்கு

```

### பயிற்சி 3: பல-சுற்றுச்சூழை பணிநடை (25 நிமிடங்கள்)
**நோக்கு**: சுற்றுச்சூழை நிர்வாகம் மற்றும் வெளியீட்டு மூலோபாயங்களில் தேர்ச்சி பெறுதல்

```bash

azd env new dev-$(whoami)

azd env set LOG_LEVEL debug
azd up

# வளர்ச்சி URL-ஐ குறித்துக் கொள்ளவும்
DEV_URL=$(azd show --output json | jq -r '.services.web.endpoint')
echo "Dev: $DEV_URL"

# ஸ்டேஜிங் சூழலை உருவாக்கவும்
azd env new staging-$(whoami)
azd env set ENVIRONMENT_TYPE staging
azd env set LOG_LEVEL info
azd up

# ஸ்டேஜிங் URL-ஐ குறித்துக் கொள்ளவும்
STAGING_URL=$(azd show --output json | jq -r '.services.web.endpoint')
echo "Staging: $STAGING_URL"

# சூழல்களை ஒப்பிடவும்
azd env list

# இரு சூழல்களையும் பரிசோதிக்கவும்
curl "$DEV_URL/health"
curl "$STAGING_URL/health"

# இரு சூழல்களையும் சுத்தம் செய்யவும்
azd env select dev-$(whoami) && azd down --force --purge
azd env select staging-$(whoami) && azd down --force --purge
```

**வெற்றி критерிய:**
- [ ] இரு சுற்றுச்சூழல்களும் வெவ்வேறு கட்டமைப்புகளுடன் உருவாக்கப்பட்டன
- [ ] இரு சூழல்களும் வெற்றிகரமாக வெளியிடப்பட்டன
- [ ] `azd env select` மூலம் சூழல்களை இடம் மாற்ற முடியிறது
- [ ] சுற்றுச்சூழை மாறிலிகள் இடையே வேறுபட்டு இருக்கின்றன
- [ ] இரு சூழல்களும் வெற்றிகரமாக சுத்தம் செய்யப்பட்டன

