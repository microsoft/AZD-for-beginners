## அறிமுகம்

[Your First Project](first-project.md) இல் நீங்கள் டெம்ப்ளேட்டிலிருந்து தொடங்கி ஒரு செயலியை desple வேண்டும். ஆனால் பெரும்பாலான நேரங்களில் உங்கள் கணினியில் ஒரு செயலி—Node.js API, Python Flask சேவை, .NET வலை செயலி—ஏற்கனவே ஒரு கோப்புறையில் இருக்கிறது. இந்த பாடம் அந்த உள்ளமைவான கோடை இல் azd ஐச் சேர்ப்பது எப்படி என்று காட்டுகிறது, இதனால் `azd up` மூலம் நீங்கள் அதை desple செய்யலாம், எந்த டெம்ப்ளேட்டும் தேவையில்லை.

## கற்றல் இலக்குகள்

இந்த பாடத்துக்குப் பிறகு, நீங்கள்:
- azd திட்டத்தை தொடங்க மூன்று வழிகளை புரிந்துகொள்வீர்கள்
- உள்ளமைவான கொடை அடைவைக்குள் `azd init` இயக்கு முறையை இயக்குவீர்கள்
- `azure.yaml` மற்றும் `infra/` கோப்புறை உங்கள் செயலிக்கு என்ன செய்கின்றன என்பதை புரிந்துகொள்வீர்கள்
- azd உடன் அதிகாரத்தை உருவாக்க வேண்டும் அல்லது உங்கள் உருவாக்கத்தை எழுத வேண்டிய நேரத்தை அறிந்திருப்பீர்கள்
- உங்கள் உள்ளமைவான செயலியை `azd up` மூலம் Azure க்கு desple செய்ய முடியும்

## கற்றல் முடிவுகள்

இந்த பாடத்தை முடித்தவுடன், நீங்கள்:
- ஏற்கனவே உங்களிடம் இருக்கும் ஒரு திட்டத்தில் azd ஐ துவக்க முடியும்
- அடிப்படை `azure.yaml` கோப்பை படித்து தொகுக்க முடியும்
- `azd infra generate` மூலம் தொடக்க கட்டமைப்பை உருவாக்க முடியும்
- உங்கள் செயலிக்கு பொருத்தமான Azure ஹோஸ்டை தேர்ந்தெடுக்க முடியும்
- உங்கள் சொந்த பயன்பாட்டை desple செய்து சுத்திகரிக்க முடியும்

## azd திட்டத்தை துவக்க மூன்று வழிகள்

| தொடக்கக்கூடு | கட்டளை | எப்போது பயன்படுத்துவது |
|----------------|---------|-------------|
| **ஒரு டெம்ப்ளேட்டில் இருந்து** | `azd init --template <name>` | கற்றல், அல்லது நிரூபிக்கப்பட்ட மாதிரியில் இருந்து புதிய செயலியைத் தொடங்கும்போது |
| **உங்கள் உள்ளமைவான கோடிலிருந்து** | `azd init` (உங்கள் திட்ட கோப்புறையில்) | உங்கள் செயலி ஏற்கனவே உண்டு மற்றும் அதை desple செய்ய விரும்பும் போது |
| **Git ரெப்போவிலிருந்து** | `azd init --from-code` (கிளோன் செய்யப்பட்ட repo இல்) | ஏற்கனவே உள்ள ரெப்போவில் azd ஐ ஏற்றுவதற்கு |

நீங்கள் ஏற்கனவே முதல் விருப்பத்தைப் பயிற்சி செய்தீர்கள். இந்த பாடம் இரண்டாவது — மிகச் சாதாரணமான புராதானமான சூழ்நிலையை — கற்டர்கிறது.

## படி 1: உங்கள் திட்டத்தில் `azd init` இயக்கு

உங்கள் உள்ளமைவான திட்ட கோப்புறையின் உள்ளே ஒரு டெர்மினல் திறக்கவும் மற்றும் இயக்கு:

```bash
cd my-existing-app
azd init
```

azd எப்படி துவக்கவேண்டும் என கேட்கிறார். தேர்ந்தெடுக்கவும்:

```
? How do you want to initialize your app?
> Use code in the current directory
  Select a template
```

**"Use code in the current directory."** என்பதை தேர்ந்தெடுக்கவும். பின்னர் azd உங்கள் கோப்புறையை ஸ்கேன் செய்து, உங்கள் மொழி மற்றும் கட்டமைப்பை கண்டறிந்து, ஒரு ஹோஸ்டை பரிந்துரைக்கிறது.

### azd ஏதை கண்டறிகிறது

azd `package.json`, `requirements.txt`, `pom.xml`, `*.csproj`, அல்லது `Dockerfile` போன்ற சிக்னல்களைத் தேடி பொருந்தும் Azure ஹோஸ்டை பரிந்துரைக்கும்:

| உங்கள் செயலி | சாத்தியமான கண்டறியப்பட்ட ஹோஸ்ட் |
|----------|----------------------|
| Node.js / Python / .NET web app | Azure App Service அல்லது Azure Container Apps |
| பிணைக்கப்பட்ட செயலி (`Dockerfile`) | Azure Container Apps |
| Function செயலி | Azure Functions |
| Static site (React/Vue build output) | Azure Static Web Apps |

கண்டறைந்த சேவையை(யை) உறுதிசெய்து, azd உங்களுக்கு தேவையான கோப்புகளை ஸ்காஃபோல்ட் செய்கிறது.

## படி 2: azd என்ன உருவாக்கியது என்பதை புரிந்துகொள்ளுங்கள்

init பிறகு, உங்கள் திட்டத்தில் இரண்டு புதிய பொருட்கள் இருக்கலாம்:

```
my-existing-app/
├── azure.yaml          # ← NEW: tells azd what to deploy and where
├── infra/              # ← NEW: Infrastructure as Code (Bicep by default)
│   ├── main.bicep
│   └── main.parameters.json
├── src/ (your code)    # unchanged
└── ...                 # your existing files, untouched
```

### `azure.yaml` — திட்டக் குருப்பு

இது azd திட்டத்தின் முதใจ தாங்கும் பகுதி. ஒரு மிகக் குறைந்த வடிவம் இவ்வாறு இருக்கும்:

```yaml
# azure.yaml
name: my-existing-app
services:
  web:
    project: ./src           # path to your app code
    language: js             # js | python | dotnet | java
    host: appservice         # appservice | containerapp | function | staticwebapp
```

`services` பிரிவு முக்கியமான பகுதி: ஒவ்வொரு நுழைவேலை உங்கள் கோடை கோப்புறையை ஒரு Azure ஹோஸ்டுடன் இணைக்கிறது. உங்கள் செயலிக்கு முன் முடக்கமும் API யும் இருந்தால், நீங்கள் இரண்டு சேவைகளை கொண்டிருப்பீர்கள்.

### `infra/` — உங்கள் Azure வளங்கள் கோடாக

`infra/` கோப்புறை உங்கள் செயலிக்கு தேவையான Azure வளங்களை வரையறுக்கும் Bicep கோப்புகளை வைத்திருக்கிறது (App Service, தரவுத்தளம், மற்றும் பிற). நீங்கள் இவை எல்லாம் கைமுறையாக எழுதத் தேவையில்லை—azd ஒரு வேலைப்பாட்டுத் தொடக்கத்தை உருவாக்குகிறது. பின்னர் நீங்கள் அவற்றை தொகுத்து வளங்களைச் சேர்க்க அல்லது பாதுகாப்பை உறுதிப்படுத்தும் வகையில் திருத்தலாம் (பரிசீலனை [அத்தியாயம் 4](../chapter-04-infrastructure/README.md) இல்).

> **Tip:** desple செய்யும் முன் உருவாக்கப்பட்ட கட்டமைப்பைப் பார்க்க அல்லது தனிப்பயன் செய்ய விரும்பினீர்களா? `azd infra generate` (அல்லது `azd infra synth`) இயக்கி IaC ஐ டிஸ்க்கில் எழுதுங்கள், அப்படியே நீங்கள் அதனை சரிபார்க்கவும் பதிப்புப்பதிவு செய்யவும் முடியும்.

## படி 3: தேவையான கட்டமைப்பை அமைக்கவும்

உங்கள் செயலிக்கு அமைப்புகள் அல்லது ரகசியங்கள் (ஒரு இணைப்புச் சரம், API விசையம்) தேவைப்பட்டால், அவற்றை கடினமாக குறியீட்டில் வைப்பதே தவறு. சுற்றுச்சூழல் மதிப்புகளைப் பயன்படுத்தவும்:

```bash
# ஒரு சூழலை உருவாக்கவும்
azd env new dev

# ஒரு ரகசியமற்ற மதிப்பை அமைக்கவும்
azd env set API_VERSION 1.0.0
```

எழிவுக்கு சொந்தமான ரகசியங்களுக்காக, அவைகளை Key Vault இலில் சேமித்து உங்கள் கட்டமைப்பில் அவற்றை சார-reference செய்யுங்கள் — காண்க [அத்தியாயம் 3: கட்டமைப்பு & அங்கீகாரம்](../chapter-03-configuration/authsecurity.md).

## படி 4: desple

இப்போ தரையிலேயே நீங்கள் அறிந்த workflow ஐப் பயன்படுத்துங்கள்:

```bash
# அங்கீகரிக்கவும் (azdக்காக அவசியம்)
azd auth login

# உருவாக்கப்படவிருக்கும் வளங்களின் முன்னோட்டத்தைப் பார்க்கவும்
azd provision --preview

# இன்ஃப்ராஸ்ட்ரக்சரை ஒதுக்கி உங்கள் குறியீட்டை அமைக்கவும்
azd up
```

இது முடிந்தவுடன், azd உங்கள் செயலியின் URL ஐ அச்சிடும். எவ்வாறெனில் எந்த azd செயலியையும் சரிபார்க்கும் முறைபோல் அதைச் சரிபார்க்கவும்:

```bash
azd show           # எண்ட்பாயின்ட்களை காட்டு
azd monitor --logs # தேவையானால் பதிவுகளைச் சரிபார்
```

## பொதுவான முதன்முறை பிரச்சினைகள்

| நற்சீர் | சாத்தியமான காரணம் | தீர்வு |
|---------|--------------|-----|
| azd என் செயலியை கண்டறியவில்லை | manifest இல்லாதது (உதாரணம், `package.json`) | manifest ஐச் சேர்க்கவும், அல்லது `azd init` சிக்தியில் ஹோஸ்டை கைமுறையாகத் தேர்ந்தெடுக்கவும் |
| `azd up` போது கட்டமைப்பு தோல்வி | செயலிக்கு ஒரு build படி தேவை | `azure.yaml` இல் சேவையின் கீழ் `buildCommand`/`outputPath` ஐச் சேர்க்கவும் |
| செயலி துவங்கி ஆனால் பிழைகளைத் திரும்பத் தருகிறது | கட்டமைப்பு/ரகசியம் காணவில்லை | `azd env set` உடன் மதிப்புகளை அமைக்கவும் அல்லது Key Vault ஐ இணைத்துக்கொள்ளவும் |
| தவறான ஹோஸ்ட் தேர்ந்தெடுக்கப்பட்டது | தானாக கண்டறிதல் தவறாக ஊகித்தது | `azure.yaml` இல் `host:` ஐ திருத்தி மீண்டும் `azd up` இயக்கவும் |

மேலும் விவரங்களுக்கு, காண்க [அத்தியாயம் 7: பிரச்சினைத் தீர்ப்பு](../chapter-07-troubleshooting/README.md).

## சுருக்கம்

- `azd init` → **"Use code in the current directory"** உங்கள் அருகிலுள்ள செயலிக்கு azd ஐச் சேர்க்கும்.
- `azure.yaml` உங்கள் கோடை கோப்புறைகளை Azure ஹோஸ்டுகளுடன் வரைபடம் செய்கிறது; `infra/` வளங்களை Bicep ஆக வரையறுக்கிறது.
- `azd infra generate` உருவாக்கப்பட்ட கட்டமைப்பை நீங்கள் பரிசீலிக்க அல்லது தனிப்பயனாக்க அனுமதிக்கிறது.
- ஒருமுறை துவங்கியவுடன், உங்கள் உள்ளமைவான செயலி டெம்ப்ளேட்-அடிப்படையிலான செயலி போன்று அதே `azd up` / `azd down` வேலைप्रவை பயன்படுத்தும்.
