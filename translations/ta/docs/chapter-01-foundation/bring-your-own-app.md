- **📖 தற்போதைய அத்தியாயம்**: அத்தியாயம் 1 - அடித்தளம் மற்றும் விரைவு தொடக்கம்

## அறிமுகம்

[உங்கள் முதல் திட்டம்](first-project.md) இல், நீங்கள் ஒரு டெம்ப்ளேட்டிலிருந்து துவங்கி ஒரு செயலியை தாக்கல் செய்தீர்கள். ஆனால் பெரும்பாலான நேரங்களில், உங்களிடம் ஏற்கனவே ஒரு செயலி உள்ளது - Node.js API, Python Flask சேவை, .NET வலை செயலி - உங்கள் கணினியில் ஒரு கோப்புறையில் இருந்துதான் உள்ளது. இந்த பாடம் அந்த கிடைத்த குறியீட்டுக்குள் azd ஐச் சேர்க்க எப்படி என்பதை காட்டுகிறது, அதனால் நீங்கள் `azd up` மூலம் அதனை வெளியிடலாம், எந்த டெம்ப்ளேட் வேண்டாமென.

## கற்றல் இலக்குகள்

இந்த பாடம் முடிந்தவுடன், நீங்கள்:
- azd திட்டத்தைத் தொடங்கும் மூன்று வழிகளையும் புரிந்துகொள்ளலாம்
- ஏற்கனவே உள்ள குறியீட்டு அடிப்படையில் `azd init` இயக்கலாம்
- `azure.yaml` மற்றும் `infra/` கோப்புறை உங்கள் செயலிக்காய் என்ன செய்கின்றன என்பதைப் புரிந்துகொள்ளலாம்
- azd உருமாற்ற கட்டமைப்பை உருவாக்கும் போது அல்லது உங்கள் சொந்தத்தை எழுதும் போது எப்போது வெளியிடுவது என்பதை அறியலாம்
- உங்கள் அக்டிவ செயலை Azureக்கு `azd up` மூலம் வெளியிடலாம்

## கற்றல் முடிவுகள்

இந்த பாடத்தை முடித்தபின், நீங்கள்:
- உங்களுக்குத் தொடங்கியுள்ள திட்டத்தில் azd ஐ ஆரம்பிக்கலாம்
- அடிப்படையான `azure.yaml` கோப்பைக் படித்து திருத்தலாம்
- `azd infra generate` மூலம் ஆரம்ப கட்டமைப்பை உருவாக்கலாம்
- உங்கள் செயலிக்குப் பொருத்தமான Azure ஹோஸ்ட்டை தேர்வுசெய்யலாம்
- உங்கள் சொந்த பயன்பாட்டை வெளியிட்டு சுத்தம் செய்யலாம்

## azd திட்டத்தைத் தொடங்கும் மூன்று வழிகள்

| துவக்க புள்ளி | கட்டளை | எப்போது பயன்படுத்துவது |
|----------------|---------|-------------|
| **டெம்ப்ளேட்டிலிருந்து** | `azd init --template <name>` | கற்றலுக்காக, அல்லது நிரூபிக்கப்பட்ட மாதிரியில் இருந்து புதிய செயலியைத் தொடங்கும்போது |
| **உங்களது உள்ள குறியீட்டிலிருந்து** | `azd init` (உங்கள் திட்ட கோப்புறையில்) | உங்களிடம் ஏற்கனவே செயலி உள்ளது, அதை வெளியிட விரும்பும்போது |
| **Git ரெப்போவில் இருந்து** | `azd init --from-code` (கிளோன் செய்த ரெப்போவில்) | ஏற்கனவே உள்ள ரெப்போசிடரிக்கான azd ஐ ஏற்றுக்கொள்ளும்போது |

நீங்கள் முதலாம் விருப்பத்தை ஏற்கனவே பயிற்சி செய்துள்ளீர்கள். இந்த பாடம் இரண்டாவது வழியை கையாள்கிறது — மிகவும் பொதுவான நிஜ உலக சம்பவம்.

## படி 1: உங்கள் திட்டத்தில் `azd init` ஐ இயக்கவும்

உங்கள் உள்ள திட்டக் கோப்புறையிலேயே ஒரு டெர்மினல் திறக்கவும் மற்றும் இயக்கவும்:

```bash
cd my-existing-app
azd init
```

azd எப்படி தொடங்கவேண்டும் என்று கேட்கும். தேர்ந்தெடுக்கவும்:

```
? How do you want to initialize your app?
> Use code in the current directory
  Select a template
```

**"உள்ளக கோப்புறையில் உள்ள குறியீட்டை பயன்படுத்தவும்."** என்பதைத் தேர்ந்தெடுக்கவும். azd உங்கள் கோப்புறையை ஸ்கேன் செய்து, உங்கள் மொழி மற்றும் கட்டமைப்பை கண்டறிந்து, ஹோஸ்ட்டை பரிந்துரைக்கிறது.

### azd கண்டறியும் போது

azd, `package.json`, `requirements.txt`, `pom.xml`, `*.csproj`, அல்லது `Dockerfile` போன்ற அறிகுறிகளைத் தேடி, பொருத்தமான Azure ஹோஸ்ட் ஒன்றை பரிந்துரைக்கிறது:

| உங்கள் செயலி | சாத்தியமான கண்டறியப்பட்ட ஹோஸ்ட் |
|----------|----------------------|
| Node.js / Python / .NET வலை செயலி | Azure App Service அல்லது Container Apps |
| கன்டெய்னரைச் சேர்ந்த செயலி (`Dockerfile`) | Azure Container Apps |
| செயல்பாட்டு செயலி | Azure Functions |
| நிலையான தளம் (React/Vue கட்டமைப்பு விளைவு) | Azure Static Web Apps |

கண்டறியப்பட்ட சேவையை உறுதிப்படுத்தவும், மேலும் azd தேவையான கோப்புகளை உருவாக்கும்.

## படி 2: azd உருவாக்கியதைப் புரிந்துகொள்ளுங்கள்

init முடிந்தவுடன், உங்கள் திட்டத்தில் இரண்டு புதியவைகள் இருக்கும்:

```
my-existing-app/
├── azure.yaml          # ← NEW: tells azd what to deploy and where
├── infra/              # ← NEW: Infrastructure as Code (Bicep by default)
│   ├── main.bicep
│   └── main.parameters.json
├── src/ (your code)    # unchanged
└── ...                 # your existing files, untouched
```

### `azure.yaml` — திட்ட வரையறை

இது azd திட்டத்தின் இதயம். ஒரு குறைந்தபட்ச மாதிரி இப்படிப்பட்டது:

```yaml
# azure.yaml
name: my-existing-app
services:
  web:
    project: ./src           # path to your app code
    language: js             # js | python | dotnet | java
    host: appservice         # appservice | containerapp | function | staticwebapp
```

`services` பகுதி முக்கியமானது: ஒவ்வொரு உள்ளீட்டும் உங்கள் குறியீட்டின் ஒரு கோப்புறையை Azure ஹோஸ்ட்டுடன் இணைக்கிறது. உங்கள் செயலிக்குக் கையொன்றும் API ஒன்று இருந்தால், இரண்டு சேவைகள் இருக்கும்.

### `infra/` — உங்கள் Azure வளங்கள் குறியீடாக

`infra/` கோப்புறை உங்கள் செயலிக்குத் தேவையான Azure வளங்களை (App Service, தரவுத்தளம், மற்றும் பிற) வரையறுக்கும் Bicep கோப்புகளை கொண்டிருக்கிறது. நீங்கள் இவற்றை கையால் எழுத தேவையில்லை — azd ஒரு வேலை செய்யும் ஆரம்பப்புள்ளியை உருவாக்குகிறது. பிறகு நீங்கள் அவற்றை திருத்தி வளங்கள் சேர்க்கவோ அல்லது பாதுகாப்பை காக்கவோ (அத்தியாயம் 4ல் விவரிக்கப்பட்டுள்ள) முடியும்.

> **சுயவிவரம்:** துவக்க கட்டமைப்பை வெளியிடுவதற்கு முன் நீங்கள் பார்க்கவோ அல்லது தனிப்பயனாக்கவோ விரும்பினால்? `azd infra generate` (அல்லது `azd infra synth`) இயக்கி IaC ஐ டிஸ்குகூட எழுதவும், அதைப் பார்வையிடவும் பதிப்பு கட்டுப்பாட்டைப் பயன்படுத்தவும்.

## படி 3: தேவையான அமைப்பை அமைக்கவும்

உங்கள் செயலிக்கு அமைப்புகள் அல்லது ரகசியங்கள் (இணைப்பு சருடன், API சாவி) தேவைப்பட்டால், அதை குறியீட்டில் நேரடியாக கொடுக்க வேண்டாம். சூழல் மதிப்புகளைப் பயன்படுத்தவும்:

```bash
# ஒரு சூழலை உருவாக்கவும்
azd env new dev

# ரகசியமற்ற மதிப்பைக் அமைக்கவும்
azd env set API_VERSION 1.0.0
```

உண்மையான ரகசியங்களுக்காக, அவற்றை Key Vault இல் சேமித்து, உங்கள் கட்டமைப்பிலிருந்து அவற்றை குறிப்பிடவும் — [அத்தியாயம் 3: கட்டமைப்பு மற்றும்認டித்தல்](../chapter-03-configuration/authsecurity.md) பார்.

## படி 4: வெளியிடவும்

இப்போது நீங்கள் ஏற்கனவே அறிந்த வேலைப்பாட்டை பயன்படுத்துங்கள்:

```bash
# அங்கீகாரம் (azd க்கு தேவையானது)
azd auth login

# உருவாக்கப்படவுள்ள வளங்களை முன்னோட்டம் செய்யவும்
azd provision --preview

# கட்டமைப்பை ஏற்படுத்தி உங்கள் குறியீட்டை பாரம்பரியப்படுத்தவும்
azd up
```

முடிந்தபோது, azd உங்கள் செயலியின் URL ஐ அச்சிடும். எந்த azd செயலிக்கும் போன்று அதனை சரிபார்க்கவும்:

```bash
azd show           # அணுக்குகளைக் காட்டு
azd monitor --logs # தேவையானால் பதிவுகளை பரிசீலிக்கவும்
```

## பொதுவான ஆரம்ப ஊடான சிக்கல்கள்

| அறிகுறி | சாத்தியமான காரணம் | சரி செய்ய வேண்டியது |
|---------|--------------|-----|
| azd எனது செயலியை கண்டறியவில்லை | அறிக்கை இல்லாது (உதா: `package.json`) | அறிக்கையைச் சேர்க்கவும் அல்லது `azd init` இல் ஹோஸ்ட்டை கைவிட்டு தேர்ந்தெடுக்கவும் |
| `azd up` காலத்தில் கட்டமைப்பு தோல்வி | செயலிக்கு கட்டமைப்பு படி வேண்டும் | `azure.yaml` இல் சேவையின் கீழ் `buildCommand`/`outputPath` ஐச் சேர்க்கவும் |
| செயலி துவங்கி பிழைகள் காட்டுகிறது | கோப்பு/ரகசியம் காணாது | `azd env set` கொண்டு மதிப்புகளை அமைக்கவும் அல்லது Key Vault ஐ இணைக்கவும் |
| தவறான ஹோஸ்ட் தேர்ந்தெடுக்கப்பட்டது | தானாக கண்டறிதல் தவறானது | `azure.yaml` இல் `host:` ஐ மாற்றி மீண்டும் `azd up` இயக்கவும் |

மேலும் விவரங்களுக்கு [அத்தியாயம் 7: பிரச்சினைகள் தீர்ப்பு](../chapter-07-troubleshooting/README.md) பார்வையிடவும்.

## சுருக்கம்

- `azd init` → **"உள்ளக கோப்புறையில் உள்ள குறியீட்டை பயன்படுத்தவும்"** என்பது உங்களுக்கு ஏற்கனவே இருக்கும் செயலிக்குச் azd ஐச் சேர்க்கிறது.
- `azure.yaml` உங்கள் குறியீட்டு கோப்புறைகளை Azure ஹோஸ்ட்களுக்கு மையப்படுத்துகிறது; `infra/` வளங்களை Bicep ஆக வரையறுக்கிறது.
- `azd infra generate` உருவாக்கப்பட்ட கட்டமைப்பை பார்வையிட அல்லது தனிப்பயனாக்க அனுமதிக்கிறது.
- ஒருமுறை ஆரம்பித்தவுடன், உங்கள் ஏற்கனவே உள்ள செயலி டெம்ப்ளேட் அடிப்படையிலான செயலி போன்று அதே `azd up` / `azd down` வேலைப்பாட்டைக் கையாள்கிறது.
