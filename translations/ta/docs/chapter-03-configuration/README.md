# அத்தியாயம் 3: கட்டமைப்பு மற்றும் அங்கீகாரம்

**📚 பாடநெறி**: [ஆரம்பக்காரர்களுக்கான AZD](../../README.md) | **⏱️ காலம்**: 45-60 நிமிடங்கள் | **⭐ சிக்கல்தன்மை**: இடைநிலை

---

## கண்ணோட்டம்

இந்த அத்தியாயம் Azure Developer CLI மூலம் செய்யப்படும் பதிப்பீடுகளுக்கான சூழல் கட்டமைப்பு, அங்கீகாரம் மாதிரிகள் மற்றும் பாதுகாப்பு சிறந்த நடைமுறைகள் பற்றியது.

> `azd 1.23.12` உடன் 2026 மார்ச் மாதத்தில் சரிபார்க்கப்பட்டது.

## கற்கும் நோக்கங்கள்

இந்த அத்தியாயத்தை முடித்தவுடன், நீங்கள்:
- AZD கட்டமைப்பு வரிசையை கையாளும் திறன் பெறுவீர்கள்
- பல சூழல்களை (dev, staging, prod) நிர்வகிப்பது
- மேலாண்மை அடையாளங்கள் மூலம் பாதுகாப்பான அங்கீகாரத்தை நடைமுறைப்படுத்துவது
- சூழலுக்கு சிறப்பான அமைப்புகளை கட்டமைப்பது

## 🚀 துவக்க வழிமுறை

```bash
# பல சூழல்களை உருவாக்கவும்
azd env new dev
azd env new staging
azd env new prod

# சூழல்களை மாற்றவும்
azd env select prod

# சூழல் மாறிலிகளை அமைக்கவும்
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# கட்டமைப்பை பார்க்கவும்
azd env get-values
```

---

## 🔧 கட்டமைப்பு வரிசை

AZD அமைப்புகளை இந்த வரிசையில் பயன்படுத்துகிறது (பின்னர் வரும் அமைப்புகள் முந்தையவற்றை மீறும்):

1. **இயல்புநிலை மதிப்புகள்** (டெம்ப்ளேடுகளில் உட்பொருத்தப்பட்டவை)
2. **azure.yaml** (திட்ட அமைப்பு)
3. **சூழல் மாறிலிகள்** (`azd env set`)
4. **கமாண்ட்-லைன் கொடிகள்** (`--location eastus`)

---

## 🔐 பாதுகாப்பு சிறந்த நடைமுறைகள்

```bash
# மேலாண்மை அடையாளத்தை பயன்படுத்தவும் (பரிந்துரைக்கப்படுகிறது)
azd env set AZURE_USE_MANAGED_IDENTITY true

# AZD அங்கீகார நிலையை சரிபார்க்கவும்
azd auth status

# விருப்பமானது: நீங்கள் az கட்டளைகளை இயக்க திட்டமிட்டால் Azure CLI சூழலை சரிபார்க்கவும்
az account show

# தேவையானால் மீண்டும் அங்கீகரிக்கவும்
azd auth login

# விருப்பமானது: az கட்டளைகளுக்காக Azure CLI அங்கீகாரத்தை புதுப்பிக்கவும்
az login
```
