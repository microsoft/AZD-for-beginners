## 1. அறிமுகம்

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) அல்லது `azd` என்பது செயலி மேம்படுத்தும் மற்றும் Azureக்கு விண்ணப்பங்களை நன்கொடுக்கும் போது மேம்படுத்துநர் வேலை흐름த்தை எளிதாக்கும் ஒரு திறந்த மூல கட்டளை வரி கருவி ஆகும்.

[AZD டெம்ப்ளேட்டுகள்](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) என்பது ஒழுங்குமுறை நடுவண்-கோப்பு மாதிரிகள் கொண்ட, பாலைவனக் கட்டமைப்பு-நடுவண் மற்றும் `azd` கட்டமைப்பு கோப்புக்களை உள்ளடக்கிய ஒரு ஒருமுறை தீர்வு கட்டமைப்புக்கான குவித்துக் கொள்ளியை உள்ளடக்கியது. கட்டமைப்பை உருவாக்குவது `azd provision` கட்டளைப் போல் எளிது - மேலும் `azd up` கட்டளை கட்டமைப்பையும் விண்ணப்பத்தையும் ஒரே நேரத்தில் 배포 செய்ய உதவுகிறது!

இதனால், உங்கள் விண்ணப்ப மேம்பாட்டு செயலியை துவக்குவது, உங்கள் விண்ணப்ப மற்றும் கட்டமைப்பு தேவைகளுக்கு அருகிலுள்ள சரியான _AZD Starter template_ ஐ கண்டறிந்து, உங்கள் கட்டமைவு தேவைகளுக்கு ஏற்ப குவித்துக் கொள்ளியை தனிப்பயனாக்குவது போல் எளிதாக இருக்க முடியும்.

முன்னர், Azure Developer CLI நிறுவப்பட்டிருக்கின்றது என்பதை உறுதிப்படுத்திக் கொள்வோம்.

1. ஒருகுடன் VS Code டெர்மினலை திறந்து கீழ்காணும் கட்டளையை இயக்கவும்:

      ```bash title="" linenums="0"
      azd version
      ```

1. நீங்கள் இதுபோல் எதையாவது காண்பீர்கள்!

      ```bash title="" linenums="0"
      azd version 1.27.1 (commit <current-build>)
      ```

**இப்போது azd உடன் டெம்ப்ளேட்டை தேர்ந்தெடுத்து 배포 செய்ய தயாராக உள்ளீர்கள்**

## 2. டெம்ப்ளேட் தேர்வு

Microsoft Foundry தளம் பொதுவாக பிரபல தீர்வு நிலைகளைக் குறிக்கும் [AZD டெம்ப்ளேட்டுகளின் தொகுப்புகளை](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) கொண்டுள்ளது, உதாரணமாக _மல்டி-ஏஜென்ட் வேலை흐름 தானியம்_ மற்றும் _மல்டி-மோடல் உள்ளடக்க செயலாக்கம்_. இந்த டெம்ப்ளேட்டுகளை Microsoft Foundry இணையதளத்திலிருந்து கூட கண்டுபிடிக்க முடியும்.

1. [https://ai.azure.com/templates](https://ai.azure.com/templates) சென்று
1. கேட்டதும் Microsoft Foundry போர்டலை இலாகின் செய்யுங்கள் - நீங்கள் இதுபோல் காண்பீர்கள்.

![Pick](../../../../../translated_images/ta/01-pick-template.60d2d5fff5ebc374.webp)


**அடிப்படை** விருப்பங்கள் உங்கள் ஆரம்ப டெம்ப்ளேட்டுகள் ஆகும்:

1. [ ] [AI Chat உடன் துவங்கவும்](https://github.com/Azure-Samples/get-started-with-ai-chat): உங்கள் தரவுடன் Azure Container Appsக்கான அடிப்படைக் அரட்டை செயலியை 배포 செய்கிறது. அடிப்படைக் AI chatbot நிலையை ஆராய இந்த டெம்ப்ளேட்டை பயன்படுத்தவும்.
1. [X] [AI முகவர்களுடன் துவங்கவும்](https://github.com/Azure-Samples/get-started-with-ai-agents): Foundry முகவர்களுடன் ஒரு தரமான AI முகவரையும் 배포 செய்கிறது. கருவிகள் மற்றும் மாதிரிகள் உடைய முகவரியியல் AI தீர்வுகளைக் கண்டறிய இதைப் பயன்படுத்துங்கள்.

இரண்டாவது இணைப்பை புதிய உலாவிய முகவரியில் திறக்கவும் (அல்லது தொடர்புடைய அட்டைக்கான `Open in GitHub` கிளிக் செய்யவும்). நீங்கள் AZD டெம்ப்ளேட்டுக்கான குவித்துக் கொள்ளியை காண்பீர்கள். README ஐ ஆராய இரு நிமிடமாயிற்று. விண்ணப்ப கட்டமைப்பு இதுபோல் காண்பிக்கப்படுகிறது:

## 3. டெம்ப்ளேட் செயல்படுத்தல்

இந்த டெம்ப்ளேட்டை 배포 செய்து சரிவரை சரிபார்ப்போம். [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) பிரிவின் வழிகாட்டுதல்களை பின்பற்றுவோம்.

1. டெம்ப்ளேட் குவித்துக் கொள்ளைக்கான வேலை பரப்பை தேர்ந்தெடுக்கவும்:

      - **GitHub Codespaces**: [இந்த இணைப்பை](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) கிளிக் செய்து `Create codespace` என்பதை உறுதிப்படுத்தவும்
      - **உள்ளூர்க்ளோன் அல்லது dev container**: `Azure-Samples/get-started-with-ai-agents` க்ளோன் செய்து VS Codeஇல் திறக்கவும்

1. VS Code டெர்மினல் தயார் ஆகும் வரை காத்திருந்து பின்வரும் கட்டளையை கொண்டு செயல்படுத்தவும்:

   ```bash title="" linenums="0"
   azd up
   ```

இதுவால் தூண்டப்படும் வேலை흐름 படிகளைக் கடைப்பிடிக்கவும்:

1. Azure இல் உள்நுழையும்படி கேட்கப்படுகிறது - அங்கீகார அறிவுறைகளை பின் தொடரவும்
1. தனித்துவமான வேலைப்பరిచயப் பெயரை உள்ளிடவும் - உதாரணம்: நான் `nitya-mshack-azd` பயன்படுத்தியுள்ளேன்
1. இதனால் `.azure/` கோப்புறை உருவாகும் - அடுக்குக்கோப்புறை (subfolder) வேலைப்பரிசயப் பெயருடன் தோன்றும்
1. சந்தா பெயரை தேர்ந்தெடுக்க கேட்கப்படும் - இயல்புநிலை தேர்வை எடுக்கவும்
1. இடம் தேர்வுக்காகக் கேட்கப்படும் - `East US 2` பயன்படுத்தவும்

இப்பொழுது, கட்டமைப்பு உருவாக்கத் தேவைப்படும் நேரத்தை காத்திருங்கள். **இது 10-15 நிமிடங்கள் ஆகும்**

1. முடிந்ததும், உங்கள் கன்சோலில் SUCCESS செய்தி இவ்வாறு காட்டப்படும்:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. உங்கள் Azure போர்டல் இப்போது அந்த வேலைப்பரிசயப் பெயருடன் ஒரு provisioned resource group ஐ காண்பிக்கும்:

      ![Infra](../../../../../translated_images/ta/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **நீங்கள் இப்போது 배포 செய்யப்பட்ட கட்டமைப்பையும் விண்ணப்பத்தையும் சரிபார்க்க தயாராக உள்ளீர்கள்**.

## 4. டெம்ப்ளேட் சரிபார்ப்பு

1. Azure போர்டல் [Resource Groups](https://portal.azure.com/#browse/resourcegroups) பக்கத்திற்கு செல்லவும் - கேட்டதும் உள்நுழையவும்
1. உங்கள் வேலைப்பரிசயப் பெயரின் RGஐ கிளிக் செய்யவும் - மேலேயுள்ள பக்கத்தை நீங்கள் காண்பீர்கள்

      - Azure Container Apps வளத்தின் மேல் கிளிக் செய்யவும்
      - _Essentials_ பகுதியிலுள்ள விண்ணப்ப URLஐ (மேல் வலம்) கிளிக் செய்யவும்

1. இப்படியான ஹோஸ்ட் செய்யப்பட்ட விண்ணப்ப முன்னணி UIக காண்பீர்கள்:

   ![App](../../../../../translated_images/ta/03-test-application.471910da12c3038e.webp)

1. சில [மாதிரி கேள்விகளை](https://github.com/Azure-Samples/get-started-with-ai-agents/blob/main/docs/sample_questions.md) கேட்டு பாருங்கள்

      1. கேளுங்கள்: ```பிரான்ஸின் தலைநகரம் என்ன?``` 
      1. கேளுங்கள்: ```$200க்கு உட்பட்ட இரண்டு நபர்களுக்கு சிறந்த காலணையை மற்றும் அதன் அம்சங்களை என்ன?```

1. கீழ்க்காணும் பதில்களைப் போலவே பதில்கள் பெறுவீர்கள். _இது எப்படிப்பட்டதாக வேலை செய்கிறது?_

      ![App](../../../../../translated_images/ta/03-test-question.521c1e863cbaddb6.webp)

## 5. முகவர் சரிபார்ப்பு

Azure Container App இந்த டெம்ப்ளேட்டுக்கான Microsoft Foundry திட்டத்தில் provision செய்யப்பட்ட AI முகவருடன் இணைக்கப்படும் ஒரு endpointஐ 배포 செய்கிறது. அதற்க்முன்னிலையை பார்ப்போம்.

1. Azure போர்டல் _Overview_ பக்கத்திற்கு உங்கள் resource groupக்குள் திரும்பவும்

1. அந்த பட்டியலில் உள்ள `Microsoft Foundry` வளத்தை கிளிக் செய்யவும்

1. இதைப் போல காண்பீர்கள். `Go to Microsoft Foundry Portal` பொத்தானை கிளிக் செய்யவும். 
   ![Foundry](../../../../../translated_images/ta/04-view-foundry-project.fb94ca41803f28f3.webp)

1. உங்கள் AI விண்ணப்பத்திற்கு Foundry திட்டப் பக்கத்தை காண்பீர்கள்
   ![Project](../../../../../translated_images/ta/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` என்பதை கிளிக் செய்யுங்கள் - உங்கள் திட்டத்தில் provision செய்யப்பட்ட இயல்புநிலை முகவரைக் காண்பீர்கள்
   ![Agents](../../../../../translated_images/ta/06-visit-agents.bccb263f77b00a09.webp)

1. அதை தேர்வு செய்யுங்கள் - நீங்கள் முகவர் விபரங்களை காண்பீர்கள். கீழ்காணும் விஷயங்களை கவனியுங்கள்:

      - முகவர் இயல்பாக (எப்பொழுதும்) கோப்பு தேடும் நுட்பத்தைப் பயன்படுத்துகிறது
      - முகவரின் `Knowledge` பகுதியில் 32 கோப்புகள் பதிவேற்றம் செய்யப்பட்டுள்ளன (கோப்பு தேடல் க்கானது)
      ![Agents](../../../../../translated_images/ta/07-view-agent-details.0e049f37f61eae62.webp)

1. இடது பட்டியலில் உள்ள `Data+indexes` விருப்பத்தைத் தேடிக் கிளிக் செய்து கூடிய நுட்பங்களை பாருங்கள். 

      - அறிவுக்கு (knowledge) பதிவேற்றம் செய்யப்பட்ட 32 தரவு கோப்புகளைப் பார்க்க முடியும்.
      - இவை `src/files` ஆஃப்- கிளையினாக இருக்கும் 12 வாடிக்கையாளர் கோப்புகள் மற்றும் 20 தயாரிப்பு கோப்புகளுக்குச் சேர்ந்தவை.
      ![Data](../../../../../translated_images/ta/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**நீங்கள் முகவர் இயங்குதளத்தை சரிபார்த்துவிட்டீர்கள்!**

1. முகவரின் பதில்கள் அந்த கோப்புகளில் உள்ள அறிவுக்கு அடிப்படையாக இருக்கும்.
1. நீங்கள் இப்போது அந்த தரவுக்கு தொடர்பான கேள்விகளை கேட்கலாம் மற்றும் அடிப்படையாக பதில்களைப் பெறலாம்.

## 6. முகவர் விளையாட்டு அரங்கம்

1. Microsoft Foundryவில் உள்ள `Agents` பக்கத்துக்கு திரும்பி இயல்புநிலை முகவரைக் தேர்ந்தெடுக்கவும்

## 7. உள்ளமைவாக உள்ள கவனிப்புத் திறன்

கவனிப்புத் திறன் என்பது உங்கள் விண்ணப்பத்தை இயங்கச் செய்து, பிழைகளை எளிதாக்கவும், அதனுடன் செயல்படவும் பயன்படும் தரவை உருவாக்குவதற்கு நடவடிக்கைகள் எடுக்கப்படுவதை குறிக்கிறது. இதற்கான உணர்வை பெற:

1. `View Run Info` பொத்தானை கிளிக் செய்யவும் - இவ்வாறு ஒரு பார்வையை காண்பீர்கள். இது [Agent மடி கண்காணிப்பு](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) இன் உதாரணம். _மேல்தர பொருளாக்கையில் Thread Logs ஐ கிளிக் செய்தும் இதே பார்வையை பெறலாம்_.

   - முகவரின் செயல்முறை படிகளும் பயனுள்ள கருவிகளும் புரிந்து கொள்ளவும்
   - பதிலுக்கான மொத்த Token எண்ணிக்கை மற்றும் வெளியீட்டு tokens பயன்பாட்டைப் பாருங்கள்
   - ஆய்வில் எவ்வளவு நேரம் செலவிடப்படுகிறது என்பதையும் கண்டறியவும்

1. `Metadata` பிரிவை கிளிக் செய்தால், பின்னர் பிழை திருத்தத்தில் உதவும் கூடுதல் தகவல்களைக் காணலாம்.

1. `Evaluations` பிரிவை கிளிக் செய்யவும் - முகவர் பதிலின் சுய-அடிப்படையிலான மதிப்பீடுகளைக் காணலாம். இதில் பாதுகாப்பு மதிப்பீடுகள் (எ.கா., சுய-கொலை ஆபத்து) மற்றும் முகவரிடம் தனிப்பட்ட மதிப்பீடுகள் (எ.கா., நோக்கு தீர்மானம், பணி ஒத்துழைப்பு) அடங்கும்.

## 8. சூழல் மாறிலிகள்

இப்பொழுது வரை, உலாவியில் 배포 செய்து கட்டமைப்பைப் சரிபார்த்தோம். ஆனால் விண்ணப்பத்தை _கோடு முதலில்_ பயன்படுத்த, உள்ளூர் மேம்பாட்டு சூழலை அவசியமான மாறிலிகளால் அமைக்க வேண்டும். `azd` இதை எளிதாக்குகிறது.

1. Azure Developer CLI [சூழல் மாறிலிகளை பயன்படுத்துகிறது](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) விண்ணப்ப 배포களுக்கு கட்டமைப்பு அமைப்புகளை நிர்வகிக்க.

1. சூழல் மாறிலிகள் `.azure/<env-name>/.env` இல் சேமிக்கப்படுகின்றன - இது அ Deploy நிர்வகிக்கப்பட்ட `எந்தியல் பெயர்` சூழலுக்குப் பரிமாணிக்கவும் பிற deployment இலக்குகளுக்கு இடையேயான சூழலை தனித்துவமாக்க உதவும்.

1. சூழல் மாறிலிகள் எந்தவொரு `azd` கட்டளையும் செயல்படும் பொழுதும் தானாக ஏற்றப்படுகின்றன (எ.கா., `azd up`). ஆனால் `azd` இயந்திர நிலை (OS-level) சூழல் மாறிலிகளை தானாக வாசிப்பதில்லை (உதாரணமாக, ஷெல்லில் அமைக்கப்பட்டவை) - பதிலாக `azd set env` மற்றும் `azd get env` மூலம் ஸ்கிரிப்ட்களில் தகவலை மாற்றவும்.


சில கட்டளைகளை முயற்சிப்போம்:

1. இந்த சூழலில் `azd` க்கான அனைத்து சூழல் மாறிலிகளையும் பெறவும்:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      இதுபோல் காண்பீர்கள்:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. ஒரு குறிப்பிட்ட மதிப்பை பெறவும் - உதாரணம்: `AZURE_AI_AGENT_MODEL_NAME` மதிப்பை அமைத்துள்ளோமா என பார்ப்பது

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      இதுபோல் காண்பீர்கள் - இயல்புநிலைப் படி அமைக்கப்படவில்லை!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd` க்கான புதிய சூழல் மாறியை அமைக்கவும். இங்கே, முகவர் மாதிரி பெயரை புதுப்பிக்கிறோம். _குறிப்பு: எந்தப் பண்பாடுகளும் `.azure/<env-name>/.env` கோப்பில் உடனடியாக பிரதிபலிக்கும்.

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      இப்போது, மதிப்பு அமைக்கப்பட்டுள்ளதாக நாம் காண்போம்:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. சில வளங்கள் நிலையானவை (எ.கா., மாதிரி 배포கள்) மற்றும் மாற்றப்பட்ட சூழல் மாறிலிகளுடன் மீண்டும் 배포 செய்ய `azd up` மட்டும் போதாது. அசல் 배포அடை நீக்கி மாற்றப்பட்ட சூழல் மாற்றங்களுடன் மீண்டும் 배포 முயற்சி செய்யலாம்.

1. **மீளமைக்கவும்** முந்தைய டெம்ப்ளேட் மூலம் கட்டமைப்பை 배포 செய்திருந்தால் - இக்கட்டளை மூலம் தற்போதைய Azure 배포 நிலைக்கு ஏற்ப உள்ளூர் சூழல் மாறிலிகளின் நிலையை மீளமைக்கலாம்:

      ```bash title="" linenums="0"
      azd env refresh
      ```


      இது இரண்டு அல்லது அதற்கும் மேற்பட்ட உள்ளூர் மேம்பாட்டு சூழல்களில் (எ.கா., பல மேம்பாட்டாளர்கள் கொண்ட குழு) சுற்றுச்சூழல் மாறிலிகளை _sync_ செய்ய ஒரு சக்திவாய்ந்த வழி - விதவிதமான சூழல் மாறிலி நிலையை நிலையான உண்மையாக வழங்குவதற்கு பயன்படுத்தப்படும் தளம் ஆகும். குழு உறுப்பினர்கள் மாறிலிகளை _refresh_ செய்யும் மூலம் மீண்டும் ஒத்திசைக்க முடியும்.
