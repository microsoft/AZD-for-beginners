## 1. அறிமுகம்

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) அல்லது `azd` என்பது Azure க்கு பயன்பாடுகளை கட்டமைக்கும் மற்றும் ஒதுக்கும்போது டெவலப்பர் பணியை எளிதாக்கும் ஒரு open-source கட்டளை வரி கருவி.

[AZD Templates](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) என்பது மாதிரி பயன்பாட்டு குறியீடு, _infrastructure-as-code_ சொத்துகள் மற்றும் ஒருங்கிணைந்த தீர்வு கட்டமைப்புக்கான `azd` கட்டமைப்பு கோப்புகளை கொண்ட ஸ்திரமான ரெப்போசிடரிகள் ஆகும். மூலதனத்தை உருவாக்குவது ஒரு `azd provision` கட்டளையைப் போல எளிதாகிறது — மேலும் `azd up` பயன்படுத்தினால், ஒரே சமயத்தில் மூலதனத்தை உருவாக்கியும் உங்கள் பயன்பாட்டை பதிக்கவும் முடியும்!

முயற்சிக்கத் தேவையான AZD Starter வார்ப்புருவை கண்டுபிடித்து, அதனை உங்கள் நிலைப்பகுதிக்கே ஏற்ப திருத்தி கொண்டு வருவது மூலம் பயன்பாட்டு மேம்பாட்டை வேகமாக ஆரம்பிக்கலாம்.

தொடங்குவதற்கு முன், உங்கள் கணினியில் Azure Developer CLI நிறுவப்பட்டுள்ளதா என்று சரிபார்ப்போம்.

1. VS Code டெர்மினலைத் திறக்கவும் மற்றும் இந்த கட்டளையை உள்ளிடவும்:

      ```bash title="" linenums="0"
      azd version
      ```

1. நீங்கள் இதுபோன்றதைப் பார்க்க வேண்டும்!

      ```bash title="" linenums="0"
      azd version 1.25.6 (commit <current-build>)
      ```

**இப்போது நீங்கள் azd கொண்டு ஒரு வார்ப்புருவை தேர்வு செய்து நிறுவ தயாராக உள்ளீர்கள்**

## 2. வார்ப்புரு தேர்வு

Microsoft Foundry தளத்தில் [பரிந்துரைக்கப்பட்ட AZD வார்ப்புருக்கள் தொகுப்பு](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) உள்ளது, அவை _மல்டி-ஏஜென்ட் வேலையமைவு தானியக்கம்_ மற்றும் _பல முகாமுறை உள்ளடக்க செயலாக்கம்_ போன்ற பிரபல தீர்வு senarios-ஐ கவர் செய்கின்றன. நீங்கள் Microsoft Foundry போர்டலைப் செல்லவும் அங்கு இந்த வார்ப்புருக்களை கண்டறியலாம்.

1. பார்வையிடவும் [https://ai.azure.com/templates]
1. கேட்டது போல் Microsoft Foundry போர்டலில் உள்நுழைக - நீங்கள் இதுபோன்று ஒரு பார்வையை காண்பீர்கள்.

![தேர்வு](../../../../../translated_images/ta/01-pick-template.60d2d5fff5ebc374.webp)


The **Basic** options are your starter templates:

1. [ ] [Get Started with AI Chat](https://github.com/Azure-Samples/get-started-with-ai-chat) இது உங்கள் தரவுடன் ஒரு அடிப்படை உரையாடல் செயலியை Azure Container Apps-ல் பிரயோகப்படுத்துகிறது. இதை ஒரு அடிப்படை AI சொடிக்கையாளர் காட்சியை ஆராய பயன்படுத்துங்கள்.
1. [X] [Get Started with AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) இது Foundry Agents உடன் ஒரு தரநிலைக் AI முகவரையினை (Agent) பிரயோகப்படுத்துகிறது. கருவிகள் மற்றும் மாதிரிகள் கொண்ட முகவரிப் பொருட்களின் AI தீர்வுகளுக்கு இதன் மூலம் பழகலாம்.

இரண்டாவது இணைப்பை புதிய உலாவி தாவலில் திறக்கவும் (அல்லது சம்பந்தப்பட்ட கார்டைத் திறக்க `Open in GitHub` ஐ கிளிக் செய்க). நீங்கள் இந்த AZD வார்ப்புருவுக்கான ரெப்போசிடரியை காணலாம். README ஐ ஒரு நிமிடம் ஆய்வு செய்யுங்கள். பயன்பாட்டு கட்டமைப்பு இதுபோல காணப்படும்:

![கட்டமைப்பு](../../../../../translated_images/ta/architecture.8cec470ec15c65c7.webp)

## 3. வார்ப்புரு செயல்படுத்தல்

இவ்வார்ப்புருவை பிரயோகப்படுத்தி அது செல்லுபடியாகுகிறதா என்று சரிபார்ப்போம். [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) செயற்கை வழிகாட்டுதல்களை பின்பற்றுவோம்.

1. வார்ப்புரு ரெப்போசிடரிக்கான வேலை சூழலை தேர்ந்தெடு:

      - **GitHub Codespaces**: இந்த இணைப்பை கிளிக் செய்க [https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents] மற்றும் `Create codespace` ஐ உறுதிசெய்க
      - **Local clone or dev container**: `Azure-Samples/get-started-with-ai-agents` ஐ கிளோன் செய்து அதை VS Code இல் திறக்கவும்

1. VS Code டெர்மினல் தயார் ஆகும் வரை காத்திருந்து, பின்னர் கீழ்க்கண்ட கட்டளையை உள்ளிடவும்:

   ```bash title="" linenums="0"
   azd up
   ```

Complete the workflow steps that this will trigger:

1. உங்களுக்கு Azure இல் உள்நுழைய கேட்டுக் கொள்வார்கள் - அங்குள்ள அங்கீகார οδηγநீடுகளை பின்பற்றவும்
1. உங்கள் மாற்றமான சூழல் பெயரை உள்ளிடவும் - உதாரணமாக, நான் `nitya-mshack-azd` ஐ பயன்படுத்தினேன்
1. இது  `.azure/` கோப்புறையை உருவாக்கும் - நீங்கள் env பெயருடன் ஒரு துணைக் கோப்புறையை காண்பீர்கள்
1. ஒரு சந்தா பெயரை தேர்வு செய்ய உங்களுக்கு கேட்டவிடம் வரும் - இயல்புநிலை (default) ஐத் தேர்வு செய்க
1. இடத்தை கேட்டவிடம் வரும் - `East US 2` ஐ பயன்படுத்தவும்

இப்போது, provision முடிந்துவிடும்வரை காத்திருங்கள். **இது 10-15 நிமிடங்கள் ஆகும்**

1. முடிந்ததும், உங்கள் கன்சோலில் இந்த மாதிரியான SUCCESS செய்தியை காண்பீர்கள்:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. உங்கள் Azure போர்டலில் இப்போது அந்த env பெயருடன் ஒரு provision செய்யப்பட்ட resource group இருக்கும்:

      ![Infra](../../../../../translated_images/ta/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **இப்போது நீங்கள் provision செய்யப்பட்ட மூலதனத்தையும் பயன்பாட்டையும் சரிபார்க்கத் தயார்**.

## 4. வார்ப்புரு சரிபார்ப்பு

1. Azure போர்டல் [Resource Groups](https://portal.azure.com/#browse/resourcegroups) பக்கத்தை பார்வையிடவும் - கேட்டால் உள்நுழையவும்
1. உங்கள் சூழல் பெயருக்கான RG ஐ கிளிக் செய்க - மேலே காட்டிய பக்கத்தை நீங்கள் காண்பீர்கள்

      - Azure Container Apps ஆதாரத்தை கிளிக் செய்க
      - _Essentials_ பகுதியில் உள்ள Application Url ஐ கிளிக் செய்க (மேல்பக்க வலது)

1. நீங்கள் இதுபோன்ற ஹோஸ்ட் செய்யப்பட்ட பயன்பாட்டு முன் முகப்பு UI ஒன்றைப் பார்க்க வேண்டும்:

   ![பயன்பாடு](../../../../../translated_images/ta/03-test-application.471910da12c3038e.webp)

1. சில [மாதிரி கேள்விகளை](https://github.com/Azure-Samples/get-started-with-ai-agents/blob/main/docs/sample_questions.md) கேட்டுப் பாருங்கள்

      1. கேளுங்கள்: ```What is the capital of France?``` 
      1. கேளுங்கள்: ```What's the best tent under $200 for two people, and what features does it include?```

1. கீழே காட்டப்பட்டவைகளுக்கு ஒத்த பதில்களை நீங்கள் பெறுவீர்கள். _ஆனால் இது எப்படிச் செயல்படுகிறது?_ 

      ![பயன்பாடு](../../../../../translated_images/ta/03-test-question.521c1e863cbaddb6.webp)

## 5.  முகவர் சரிபார்ப்பு

Azure Container App ஒரு endpoint ஐ பிரயோகப்படுத்துகிறது, இது இந்த வார்ப்புருவுக்காக Microsoft Foundry திட்டத்தில் provision செய்யப்பட்ட AI Agent-ஐ இணைக்கிறது. இதன் பொருள் என்ன என்பதைப் பார்க்கலாம்.

1. உங்கள் resource group-ஐ Azure போர்டல் _Overview_ பக்கத்தில் மீண்டும் திறக்கவும்

1. அந்த பட்டியலில் இருந்து `Microsoft Foundry` ஆதாரத்தை கிளிக் செய்க

1. நீங்கள் இதைப் பார்க்கலாம். `Go to Microsoft Foundry Portal` பட்டனை கிளிக் செய்க.
   ![Foundry](../../../../../translated_images/ta/04-view-foundry-project.fb94ca41803f28f3.webp)

1. உங்கள் AI பயன்பாட்டிற்கான Foundry Project பக்கத்தை நீங்கள் காண்பீர்கள்
   ![Project](../../../../../translated_images/ta/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` ஐ கிளிக் செய்க - உங்கள் திட்டத்தில் provision செய்யப்பட்ட இயல்புநிலை Agent-ஐ நீங்கள் காண்பீர்கள்
   ![Agents](../../../../../translated_images/ta/06-visit-agents.bccb263f77b00a09.webp)

1. அதை தேர்ந்தெடுங்கள் - Agent விவரவங்களை நீங்கள் பார்க்கலாம். கீழ்காணும் விஷயங்களை கவனிக்க வேண்டும்:

      - முகவர் இயல்பாக File Search ஐ பயன்படுத்துகிறது (எப்போதும்)
      - முகவரின் `Knowledge` காட்சி, அதற்கு 32 கோப்புகள் பதிவேற்றப்பட்டுள்ளன (file search க்காக)
      ![Agents](../../../../../translated_images/ta/07-view-agent-details.0e049f37f61eae62.webp)

1. இடது மெனுவில் `Data+indexes` விருப்பத்தைத் தேடி, விவரங்களுக்கு கிளிக் செய்க. 

      - நீங்கள் அறிவுக்கு பதிவேற்றப்பட்ட 32 தரவு கோப்புகளை காண்பீர்கள்.
      - இவை `src/files` கீழேயுள்ள 12 customer கோப்புகள் மற்றும் 20 product கோப்புகளை உடையவை ஆகும்
      ![Data](../../../../../translated_images/ta/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**நீங்கள் முகவர் செயல்பாட்டை சரிபார்த்துள்ளீர்கள்!**

1. முகவர் பதில்கள் அந்த கோப்புகளில் உள்ள அறிவில் அடிப்படையாக அமைந்துள்ளன.
1. இப்போது நீங்கள் அந்த தரவுடன் தொடர்புடைய கேள்விகளை கேட்டு, நிலைநிறைவு பெற்ற பதில்களைப் பெறலாம்.
1. உதாரணம்: `customer_info_10.json` "Amanda Perez" என்பவரின் 3 வாங்குதல்களை விவரிக்கிறது

Container App endpoint உடைய உலாவிப் தாவலை மீண்டும் திறந்து கேளுங்கள்: `What products does Amanda Perez own?`. நீங்கள் இதுபோன்றதை காணலாம்:

Microsoft Foundry இன் திறன்களை மேலும் புரிந்துகொள்ள, Agents Playground இல் அந்த முகவரியை சோதிக்கலாம்.

1. Microsoft Foundry-இல் `Agents` பக்கத்திற்கு திரும்புங்கள் - இயல்புநிலை முகவரியை தேர்ந்தெடுங்கள்
1. `Try in Playground` விருப்பத்தை கிளிக் செய்க - நீங்கள் இதுபோன்ற Playground UI ஒன்றைப் பெறுவீர்கள்
1. அதே கேள்வியை கேளுங்கள்: `What products does Amanda Perez own?`

    ![தரவு](../../../../../translated_images/ta/09-ask-in-playground.a1b93794f78fa676.webp)

அதே (அல்லது இதற்கு ஒத்த) பதிலை நீங்கள் பெறுவீர்கள் - மேலும் முகவரியின் செயல்திறன், செலவு, மற்றும் தரத்தைப் புரிந்துகொள்ள உபயோகப்படுத்தக்கூடிய கூடுதல் தகவல்களையும் காணலாம். உதாரணமாக:

1. பதில் எந்த தரவு கோப்புகளை "நிலைப்படுத்த" பயன்படுத்தியது என்று மேற்கோள் கொடுக்கப்பட்டுள்ளது
1. எந்த கோப்பு லேபிள์ மீது மௌஸ் வைக்கினால் - அந்த தரவு உங்கள் விசாரணைக்கு மற்றும் காட்டப்பட்ட பதிலுக்கு பொருந்துகிறதா என்பது பார்க்கவும்

பதில் கீழ் ஒரு _புள்ளிவிவர_ வரிசையைக் காணலாம்.

1. எந்த அளவுகோலையும் மவுஸ் வைத்து பாருங்கள் - உதாரணமாக, Safety. நீங்கள் இதுபோன்றதை காணலாம்
1. மதிப்பீட்டான மதிப்பெண் உங்கள் உணர்விற்கேற்ப பதிலின் பாதுகாப்பு நிலைக்கு பொருந்துகிறதா?

## 7. உள்ளமைக்கப்பட்ட கண்காட்சிச் சீர்திருத்தம் (Built-in Observability)

Observability என்பது உங்கள் பயன்பாட்டை புரிந்து கொள்ள, பிழைகளை கண்டறிய மற்றும் சிறப்பாக செயல்படுத்த தேவையான தரவுகளை உருவாக்குவதற்கான கருவிகள் மற்றும் அளவைகள். இதைப் பற்றிய உணர்வைப் பெற:

1. `View Run Info` பொத்தானை கிளிக் செய்க - நீங்கள் இந்த காட்சி ஒன்றைப் பார்ப்பீர்கள். இது [Agent tracing](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) உடை ஒரு செயல்பாட்டைக் காட்டுகிறது. _மேல்நிலைக் மெனு-இல் Thread Logs ஐ கிளிக் செய்வதும் இதே காட்சியை கொடுக்கும்_.

   - இயக்கப் படிகளையும் முகவர் பயன்படுத்திய கருவுகளையும் உணர்வு பெறுங்கள்
   - பதிலிற்கான மொத்த Token எண்ணிக்கையை (வெளியீட்டு token பயன்பாட்டுடன் ஒப்பீடு) புரிந்துகொள்ளுங்கள்
   - தாமதம் மற்றும் செயலாக்கத்தில் நேரம் எங்கு செலவாகிறது என்பதைப் புரிந்துகொள்ளுங்கள்

      ![Agent](../../../../../translated_images/ta/10-view-run-info.b20ebd75fef6a1cc.webp)

1. `Metadata` தாவலை கிளிக் செய்து, ஓட்டத்தின் மேலதிக பண்புகளைப் பார்க்கவும், அவை பிற நேரங்களில் பிழைபடுத்தல்களுக்கு உதவக்கூடும்.   

      ![Agent](../../../../../translated_images/ta/11-view-run-info-metadata.7966986122c7c2df.webp)


1. `Evaluations` தாவலை கிளிக் செய்து, முகவர் பதிலின் மீது தானாகச் செய்த மதிப்பீடுகளைப் பார்க்கவும். இவை பாதுகாப்பு மதிப்பீடுகள் (உதாரணமாக, Self-harm) மற்றும் முகவரிக்கு தொடர்புடைய மதிப்பீடுகள் (உதாரணமாக, நோக்கம் தீர்மானம், பணி ஒழுங்கு) ஆகியவற்றை உள்ளடக்கியதாக இருக்கும்.

      ![Agent](../../../../../translated_images/ta/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. கடைசியில், சைட்பார் மெனுவில் `Monitoring` தாவலை கிளிக் செய்க.

      - காட்டப்பட்ட பக்கத்தில் `Resource usage` தாவலை தேர்வு செய்து, மெட்ரிக்களைப் பார்க்கவும்.
      - tokens மூலம் செலவுகளையும் (costs) மற்றும் லோடு (requests) அடிப்படையில் பயன்பாடு கண்காணிக்கவும்.
      - முதல் பைட் (முழுவதான உள்ளீடு செயலாக்கம்) மற்றும் கடைசி பைட் (வெளியீடு) வரை பயன்பாட்டின் தாமதத்தை கண்காணிக்கவும்.

      ![Agent](../../../../../translated_images/ta/13-monitoring-resources.5148015f7311807f.webp)

## 8. சுற்றுச்சூழல் மாறிலிகள்

இந்நேரம் வரை, நாம் உலாவியில் பிரயோகப்படுத்துதல் மற்றும் அது வெற்றியடையும்போது மூலதனத்தையும் பயன்பாட்டையும் சரிபார்த்தோம். ஆனால் குறியீடு மையமாகப் பணியாற்ற, இந்த ஆதாரங்களுடன் பணியாற்ற தேவையான சார்ந்த மாறிலிகளைக் உங்கள் உள்ளூர் மேம்பாட்டு சூழலில் அமைக்க வேண்டும். `azd` இதை எளிதாக்குகிறது.

1. Azure Developer CLI [சுற்றுச்சூழல் மாறிலிகளை](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) பயன்பாட்டுக்கான ஒதுக்கீட்டு அமைப்புகளை சேமிக்கவும் மற்றும் நிர்வகிக்கவும் பயன்படுத்துகிறது.

1. சுற்றுச்சூழல் மாறிலிகள் `.azure/<env-name>/.env` இல் சேமிக்கப்படுகின்றன - இது அவற்றை deployment செய்த env-name சூழலுக்கே வரையறுக்கிறது மற்றும் அதே ரெப்போசிடரியில் வெவ்வேறு deployment இலக்குகளுக்கு இடையே சூழல்களை தனித்துவமாக வைத்துக்கொள்ள உதவுகிறது.

1. குறிப்பிட்ட கட்டளை செயல்படும் போது (உதாரணம், `azd up`) `azd` சுற்றுச்சூழல் மாறிலிகளை தானாக ஏற்றும். `azd` இயல்பாக OS நிலைமைகள் (shell இல் அமைக்கப்பட்டவை) மாதிரியான அமைப்புகளை தானாக வாசிக்காது - இதில் பதிலாக `azd set env` மற்றும் `azd get env` ஐ ஸ்கிரிப்ட்களில் தகவலை பரிமாற பயன்படுத்துங்கள்.


சில கட்டளைகளை முயற்சிக்கலாம்:

1. இந்த சூழலுக்காக azd உள்ள அனைத்து சுற்றுச்சூழல் மாறிலிகளையும் பெறவும்:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      நீங்கள் இதுபோன்றதை காண்பீர்கள்:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. ஒரு குறிப்பிட்ட மதிப்பை பெறுங்கள் - உதாரணமாக, `AZURE_AI_AGENT_MODEL_NAME` மதிப்பு அமைக்கப்பட்டாரானா என்பதை அறிய நான் விரும்புகிறேன்

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      நீங்கள் இதுபோன்றதை காண்பீர்கள் - இது இயல்பாக அமைக்கப்படவில்லை!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd` க்காக புதிய சுற்றுச்சூழல் மாறிலியை அமைக்கவும். இங்கே, நாம் Agent மாதிரி பெயரை புதுப்பிக்கிறோம். _குறிப்பு: எந்தவொரு மாற்றமும் உடனுக்குடன் `.azure/<env-name>/.env` கோப்பில் பிரதிபலிக்கப்படும்._

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      இப்போது, மதிப்பு அமைக்கப்பட்டிருப்பதை நாம் காணலாம்:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. சில ஆதாரங்கள் நிரந்தரமானவையாக இருக்கும் (உதாரணமாக, மாதிரி deploy-களும்) மற்றும் அவற்றை மீண்டும் deploy செய்ய `azd up` மட்டும் போதாது. அசல் deployment-ஐ நீக்கி, மாற்றிய env மாறிலிகளுடன் மீண்டும் deploy செய்ய முயற்சிப்போம்.

1. **Refresh** நீங்கள் முன்பு ஒரு azd வார்ப்புரு பயன்படுத்தி மூலதனத்தை deploy செய்திருந்தால் - தற்போதைய Azure deployment-ன் நிலைக்கு அடிப்படையாக உங்கள் உள்ளூர் சுற்றுச்சூழல் மாறிலிகளின் நிலையை _refresh_ செய்ய இந்த கட்டளையைப் பயன்படுத்தலாம்:

      ```bash title="" linenums="0"
      azd env refresh
      ```

      இது இரண்டு அல்லது அதற்கு மேற்பட்ட உள்ளூர் வளர்ச்சி சூழல்களில் (உதாரணம்: பல டெவலப்பர்கள் கொண்ட குழு) இடையே சூழ்நிலை மாறிகளை _ஒத்திசைவு_ செய்வதற்கான சக்திவாய்ந்த வழி ஆகும் - deployed செய்யப்பட்ட உள்கட்டமைப்பு env மாறி நிலைக்கு உண்மையான ஆதாரமாக செயல்பட உதவுகிறது. குழு உறுப்பினர்கள் மாறிகளை எளிதாக _மீண்டும் புதுப்பித்து_ மீண்டும் ஒத்திசைக்கிறார்கள்.
