## Getting Started

### Q: Azure Developer CLI (azd) என்றால் என்ன?
**A**: Azure Developer CLI (azd) என்பது உங்கள் பயன்பாட்டை உள்ளூரில் இருந்து Azureக்குப் பிரவேசிப்பதை விரைவுபடுத்தும் டெவலபர்-முன்னிலைக் கமாண்ட்-லைன் கருவி. இது டெம்ப்ளேடுகள் மூலம் சிறந்த நடைமுறைகளை வழங்குகிறது மற்றும் முழு விநியோக வாழ்நாள் சுழற்சியிலும் உதவுகிறது.

### Q: azd மற்றும் Azure CLI எப்படி வித்தியாசமாக இருக்கிறது?
**A**: 
- **Azure CLI**: Azure வளங்களை முகாமைய செய்வதற்கான பொதுக் கருவி
- **azd**: பயன்பாட்டு விநியோக வேலைவாய்ப்புகளுக்கு டெவலபர்-முகம்மை கொண்ட கருவி
- azd உள் செயல்படுவதற்கு Azure CLI-வை பயன்படுத்துகிறது ஆனாலும் பொதுவான டெவலப்மெண்ட் பின்னணிகளுக்கான உயர் நிலை சுருக்கங்களை வழங்குகிறது
- azd டெம்ப்ளேடுகள், சூழல் மேலாண்மை மற்றும் விநியோக தானியங்கி செயல்பாடுகளை உடையது

### Q: azd பயன்படுத்த Azure CLI நிறுவ வேண்டுமா?
**A**: ஆம், azd க்கு Azure CLI ஆங்கீகாரம் மற்றும் சில செயல்களுக்கு தேவைப்படும். முதலில் Azure CLI-யை நிறுவி, பிறகு azd-வை நிறுவவும்.

### Q: azd எந்த progrmming மொழிகளை ஆதரிக்கிறது?
**A**: azd மொழியிலிருந்து சுதந்திரமானது. இது பணியாற்றுகிறது:
- Node.js/JavaScript/TypeScript
- Python
- .NET/C#
- Java
- Go
- PHP
- நிலையான வலைத்தளங்கள்
- கன்டெய்னரைச்செய்யப்பட்ட பயன்பாடுகள்

### Q: azd-வை ஏற்கனவே உள்ள திட்டங்களுடன் பயன்படுத்தலாமா?
**A**: ஆம்! நீங்கள் அல்லது:
1. `azd init` பயன்படுத்தி ezd (azd) கட்டமைப்பை ஏற்கனவே உள்ள திட்டங்களுக்கு சேர்க்கலாம்
2. உங்கள் உள்ளமைவு azd டெம்ப்ளேட் அமைப்புக்கு ஒத்துப்போக끔 செய்து மாற்றலாம்
3. உங்கள் உள்ள கட்டமைப்பின் அடிப்படையில் தனிப்பயன் டெம்ப்ளேடுகளை உருவாக்கலாம்

## Authentication & Access

### Q: azd மூலம் Azure-ல் எவ்வாறு அங்கீகரிப்பது?
**A**: உபயோகியுங்கள் `azd auth login` இது Azure அங்கீகாரம் కోసం உலாவியில் ஒரு சாளரம் திறக்கும். CI/CD நிலைகளுக்காக, சர்வீஸ் பிரின்சிபல்களினும் அல்லது மேலாண்மை அடையாளங்களினும் பயன்படுத்தவும்.

### Q: meerdere Azure subscription-களுடனும் azd பயன்படுத்தலாமா?
**A**: ஆம். ஒவ்வொரு சூழலுக்கும் எந்த subscription பயன்படுத்தவேண்டுமென்று குறிப்பிட `azd env set AZURE_SUBSCRIPTION_ID <subscription-id>` பயன்படுத்தவும்.

### Q: azd-யுடன் deployment செய்ய என்ன அனுமதிகள் வேண்டும்?
**A**: பொதுவாக உங்களுக்கு தேவைப்படும்:
- ரிசோர்ஸ் குழாம் அல்லது subscription-ல் **Contributor** பங்கு
- ரோல் ஒதுக்கீடுகளைச் செய்தல் தேவைக்கு **User Access Administrator**
- குறிப்பிட்ட அனுமதிகள் டெம்ப்ளேட் மற்றும் விநியோகப்படும் வளங்களின் அடிப்படையில் மாறுபடும்

### Q: azd-ஐ CI/CD குழப்புகளில் பயன்படுத்தலாமா?
**A**: முழுமையாக! azd CI/CD ஒருங்கிணைப்பிற்கு வடிவமைக்கப்பட்டுள்ளது. அங்கீகாரத்திற்கு சர்வீஸ் பிரின்சிபல்களை பயன்படுத்தவும் மற்றும் கட்டமைப்புக்கான சூழல் மாறிகளை அமைக்கவும்.

### Q: GitHub Actions-ல் அங்கீகாரத்தை எப்படி கையாள்வது?
**A**: சர்வீஸ் பிரின்சிபல் கடவுச்சொற்களுடன் Azure Login action-ஐ பயன்படுத்துங்கள்:
```yaml
- uses: azure/login@v1
  with:
    creds: ${{ secrets.AZURE_CREDENTIALS }}
- run: azd deploy --no-prompt
```

## Templates & Projects

### Q: azd டெம்ப்ளேடுகளை எங்கே காணலாம்?
**A**: 
- அதிகாரப்பூர்வ டெம்ப்ளேடுகள்: [Azure-Samples/awesome-azd](https://github.com/Azure-Samples/awesome-azd)
- சமூகம் உருவாக்கிய டெம்ப்ளேடுகள்: GitHub-ல் "azd-template" என தேடுங்கள்
- கிடைக்கும் டெம்ப்ளேடுகளை உலாவ `azd template list` பயன்படுத்தவும்

### Q: தனிப்பயன் டெம்ப்ளேட் எப்படி உருவாக்குவது?
**A**: 
1. ஏற்கனவே உள்ள டெம்ப்ளேட் அமைப்புடன் தொடங்குங்கள்
2. `azure.yaml`, மையமைப்பு கோப்புகள், மற்றும் பயன்பாட்டு குறியீட்டை மாற்றுங்கள்
3. `azd up` மூலம் முறையாக சோதியுங்கள்
4. பொருத்தமான குறிச்சொற்களுடன் GitHub-க்கு வெளியிடுங்கள்

### Q: டெம்ப்ளேட் இல்லாமல் azd பயன்படுத்தலாமா?
**A**: ஆம், ஏற்கனவே உள்ள திட்டத்தில் `azd init` பயன்படுத்தி தேவையான கட்டமைப்பு கோப்புகளை உருவாக்கவும். `azure.yaml` மற்றும் மையமைப்பு கோப்புகளை கைமுறையாக கட்டமைக்க வேண்டும்.

### Q: அதிகாரப்பூர்வ மற்றும் சமூகவாழ்வு டெம்ப்ளேடுகளில் என்ன வேறுபாடு?
**A**: 
- **அதிகாரப்பூர்வ டெம்ப்ளேடுகள்**: Microsoft மூலம் பராமரிக்கப்படுகின்றன, சீராக புதுப்பிக்கப்படுகின்றன, பரிபூரண ஆவணப்படுத்தல்
- **சமூக டெம்ப்ளேடுகள்**: டெவலப்பர்கள் உருவாக்கியது, சிறுபாலான பயன்பாட்டு வழக்கங்கள் இருக்கலாம், பராமரிப்பும் தரமும் மாறுபடும்

### Q: என் திட்டத்தில் ஒரு டெம்ப்ளேட்டை எப்படி புதுப்பிப்பது?
**A**: டெம்ப்ளேடுகள் தானாக புதுப்பிக்கப்பட மாட்டாது. நீங்கள்:
1. மூலம் டெம்ப்ளேட்டிலிருந்து மாற்றங்களை கையேடாக ஒப்பிட்டு நீக்கி இணைக்கலாம்
2. புதுப்பிக்கப்பட்ட டெம்ப்ளேட்டுடன் `azd init` கொண்டு புதியதாக தொடங்கலாம்
3. புதுப்பிக்கப்பட்ட டெம்ப்ளேட்டிலிருந்து குறிப்பிட்ட முன்னேற்றங்களை cherry-pick செய்யலாம்

- தரவுத்தளங்கள் (SQL, PostgreSQL, Cosmos DB)
- ஸ்டோரேஜ், Key Vault, Application Insights
- நெட்வொர்க்கிங், பாதுகாப்பு மற்றும் கண்காணிப்பு வளங்கள்

### Q: பல பிராந்தியங்களுக்கு விநியோகிக்கலாமா?
**A**: ஆம், உங்கள் Bicep டெம்ப்ளேட்டுகளில் பல பிராந்தியங்களை ஒருங்கிணைக்கவும் மற்றும் ஒவ்வொரு சூழலுக்கும் ஏற்ற இடம் பண்பாராம்பரை (location parameter) சரியாக அமைக்கவும்.

### Q: தரவுத்தள ஸ்கீமா மைக்ரேஷன்களை எவ்வாறு கையாள்வது?
**A**: `azure.yaml`-இல் deployment hooks ஐப் பயன்படுத்துங்கள்:
```yaml
hooks:
  postdeploy:
    posix:
      run: ./scripts/migrate-database.sh
    windows:
      run: ./scripts/migrate-database.ps1
```

### Q: நான் பயன்பாடுகள் இல்லாமல் வெறும் கட்டமைப்புகளை மட்டுமே விநியோகிக்கலாமா?
**A**: ஆம், உங்கள் டெம்ப்ளேட்டுகளில் வரையறுக்கப்பட்ட கட்டமைப்பு கூறுகளை மட்டுமே விநியோகிக்க `azd provision` பயன்படுத்துங்கள்.

### Q: ஏற்கனவே உள்ள Azure வளங்களுக்கு எப்படி deployment செய்ய வேண்டும்?
**A**: இது சிக்கலானது மற்றும் நேரடியாக ஆதரிக்கப்பட மாட்டாது. நீங்கள்:
1. Bicep டெம்ப்ளேட்டுகளுக்கு ஏற்கனவே உள்ள வளங்களை இறக்குமதி செய்யலாம்
2. டெம்ப்ளேட்டுகளில் உள்ள ஏற்கனவே உள்ள வள குறிப்புகளைக் குறிப்பிடுங்கள்
3. வளங்களை உருவாக்கவோ அல்லது குறிப்பிடவோ உத்தேசப்படும்படி டெம்ப்ளேட்டுகளை மாற்றுங்கள்

### Q: Bicep மாற்றாக Terraform பயன்படுத்தலாமா?
**A**: தற்போது, azd முதன்மையாக Bicep/ARM டெம்ப்ளேடுகளை ஆதரிக்கிறது. Terraform ஆதரவு அதிகாரபூர்வமாக கிடைக்காது, இருப்பினும் சமூகவழி தீர்வுகள் இருக்கக்கூடும்.

## Configuration & Environments

### Q: வித்தியாசமான சூழல்களை (dev, staging, prod) எப்படி நிர்வகிப்பது?
**A**: `azd env new <environment-name>` கொண்டு தனித்தல அல்லாத சூழல்களை உருவாக்கி ஒவ்வொன்றிற்கும் வித்தியாசமான அமைப்புகளை நிர்வகிக்கவும்:
```bash
azd env new development
azd env new staging  
azd env new production
```

### Q: சூழல் கட்டமைப்புகள் எங்கே சேமிக்கப்படுகின்றன?
**A**: உங்கள் திட்ட அடைவைந்துள்ள `.azure` கோப்புறைஉள்ளே. ஒவ்வொரு சூழலுக்கும் தனித்துப் கோப்புறை மற்றும் கட்டமைப்பு கோப்புகள் இருக்கும்.

### Q: சூழல்- குறிப்பிட்ட கட்டமைப்புகளை எப்படி அமைக்குவது?
**A**: சூழல் மாறிகளை அமைக்க `azd env set` பயன்படுத்தவும்:
```bash
azd env set AZURE_LOCATION eastus
azd env set DATABASE_TIER Basic
```

### Q: குழு உறுப்பினர்களுக்குப் பங்கு சூழல் கட்டமைப்புகளைப் பகிரமுடியுமா?
**A**: `.azure` கோப்புறையில் நுட்பமான தகவல்கள் உள்ளன மற்றும் அதை version control-க்கு commit செய்யக் கூடாது. பதிலாக:
1. தேவையான சூழல் மாறிகளை ஆவணப்படுத்துங்கள்
2. சூழல்களை அமைக்க deployment ஸ்கிரிப்டுகளை பயன்படுத்து
3. நுட்பமான கட்டமைப்புக்காக Azure Key Vault-ஐப் பயன்படுத்துங்கள்

### Q: டெம்ப்ளேட் பழுதுகளை எப்படி மேற்பார்வை செய்வது?
**A**: டெம்ப்ளேட் அளவுருக்களுக்கு ஒத்துச் செல்லும் சூழல் மாறிகளை அமைக்கவும்:
```bash
azd env set LOCATION "West US 2"
azd env set SKU_NAME "B1"
```

## Troubleshooting

### Q: `azd up` ஏன் தோல்வி அடைவது?
**A**: பொதுவான காரணங்கள்:
1. **அங்கீகாரம் சிக்கல்கள்**: `azd auth login` இயக்கவும்
2. **புரபித்த அனுமதிகள்**: உங்கள் Azure பங்கு ஒதுக்கீடுகளை சரிபார்க்கவும்
3. **வள பெயரிடல் மோதல்கள்**: AZURE_ENV_NAME மாற்றவும்
4. **கோட்டா/திறன் சிக்கல்கள்**: பிராந்திய கிடைக்கும் தன்மையை சரிபார்க்கவும்
5. **டெம்ப்ளேட் பிழைகள்**: Bicep டெம்ப்ளேடுகளை சரிபார்க்கவும்

### Q: deployment தோல்விகளை எப்படி டீபெக் செய்வது?
**A**: 
1. விரிவான output க்காக `azd deploy --debug` பயன்படுத்தவும்
2. Azure போர்டலில் deployment வரலாற்றை சரிபார்க்கவும்
3. Azure போர்டலில் Activity Log ஐப் பரிசீலிக்கவும்
4. தற்போதைய சூழல் நிலையை காண `azd show` பயன்படுத்தவும்

### Q: என் சூழல் மாறிகள் ஏன் வேலை செய்யவில்லை?
**A**: சரிபார்க்கவும்:
1. மாறிகள் பெயர்கள் டெம்ப்ளேட் அளவுருக்களுக்கு சரியாக பொருந்துகின்றன
2. இடைவெளிகள் உள்ளுள்ள மதிப்புகள் சரியாக கொட்டிங் செய்யப்பட்டுள்ளன
3. சூழல் தேர்ந்தெடுக்கப்பட்டது: `azd env select <environment>`
4. மாறிகள் சரியான சூழலில் அமைக்கப்பட்டுள்ளன

### Q: தோல்வியான deployment-ஐ எப்படி சுத்தம் செய்யலாம்?
**A**: 
```bash
azd down --force --purge
```
இது அனைத்து வளங்களையும் மற்றும் சூழல் கட்டமைப்பையும் அகற்கும்.

### Q: deploymentக்கு பிறகு என் பயன்பாடு அணுகமுடியலாமா?
**A**: சரிபார்க்கவும்:
1. deployment வெற்றிகரமாக முடிந்துள்ளதா
2. பயன்பாடு இயங்குகிறதா (Azure போர்டலில் லாக்ஸ் சரிபார்க்கவும்)
3. நெட்வொர்க் பாதுகாப்பு குழுக்கள் போக்குவரத்தை அனுமதிக்கிறதா
4. DNS/கஸ்டம் டொமைன்கள் சரியாக கட்டமைக்கப்பட்டுள்ளதா

## Cost & Billing

### Q: azd deployments-க்கு எவ்வளவு செலவு வரும்?
**A**: செலவுகள் இது போன்றவற்றின் அடிப்படையில் இருக்கும்:
- deployment செய்யப்பட்ட Azure சேவைகள்
- தேர்ந்தெடுக்கப்பட்ட சேவை நிலைகள்/SKUs
- பிராந்திய விலை வேறுபாடுகள்
- பயன்பாட்டு பாடுகள்

அனுமானங்களுக்காக [Azure Pricing Calculator](https://azure.microsoft.com/pricing/calculator/) பயன்படுத்தவும்.

### Q: azd deployment-களில் செலவுகளை எப்படி கட்டுப்படுத்துவது?
**A**: 
1. வளர்ச்சி சூழல்களுக்கு குறைந்த நிலை SKUs பயன்படுத்தவும்
2. Azure பட்ஜெட்கள் மற்றும் அலெர்ட்களை அமைக்கவும்
3. அவசியமில்லாத நேரங்களில் வளங்களை அகற்ற `azd down` பயன்படுத்தவும்
4. பொருத்தமான பிராந்தியங்களை தேர்ந்தெடுங்கள் (இடத்தின் அடிப்படையில் செலவுகள் மாறுபடுகின்றன)
5. Azure Cost Management கருவிகளைப் பயன்படுத்தவும்

### Q: azd டெம்ப்ளேடுகளுக்கான இலவச நிலை விருப்பங்கள் உள்ளதா?
**A**: பல Azure சேவைகள் இலவச நிலைகளை வழங்குகின்றன:
- App Service: இலவச நிலை கிடைக்கும்
- Azure Functions: மாதத்திற்கு 1M இலவச செயற்பாடுகள்
- Cosmos DB: 400 RU/s உடன் இலவச நிலை
- Application Insights: மாதத்திற்கு முதல் 5GB இலவசம்

இலவச நிலைகள் கிடைக்கும் இடங்களில் டெம்ப்ளேட்டுகளை பிரயோகிக்கவும்.

### Q: deployment முன் செலவுகளை எவ்வாறு கணக்கிடுவது?
**A**: 
1. எந்த வளங்கள் உருவாக்கப்படுகின்றன என்பதை பார்க்க `main.bicep` டெம்ப்ளேட் செக் செய்யவும்
2. குறிப்பிட்ட SKUs உடன் Azure Pricing Calculator ஐப் பயன்படுத்தவும்
3. முதலில் ஒரு development சூழலுக்கு deployment செய்து உண்மையான செலவுகளை கண்காணிக்கவும்
4. விரிவான செலவுக் கணக்காய்விற்கு Azure Cost Management ஐப் பயன்படுத்தவும்

## Best Practices

### Q: azd திட்ட அமைப்புக்கான சிறந்த நடைமுறைகள் என்ன?
**A**: 
1. பயன்பாட்டு குறியீட்டை கட்டமைப்பிலிருந்து பிரித்து வைக்கவும்
2. `azure.yaml` இல் பொருத்தமான சேவை பெயர்களை பயன்படுத்தவும்
3. கட்டும் ஸ்கிரிப்டுகளில் சரியான பிழை கையாளுதலை செயல்படுத்தவும்
4. சூழல்-விசேஷக் கட்டமைப்புகளை பயன்படுத்தவும்
5. விரிவான ஆவணப்படுத்தலைச் சேர்க்கவும்

### Q: பல சேவைகளை azd-ல் எப்படி ஒழுங்குபடுத்த வேண்டும்?
**A**: பரிந்துரைக்கப்பட்ட கட்டமைப்பைப் பயன்படுத்தவும்:
```
├── src/
│   ├── web/          # Frontend service
│   ├── api/          # Backend service  
│   └── worker/       # Background service
├── infra/            # Infrastructure templates
├── azure.yaml        # azd configuration
└── README.md         # Project documentation
```

### Q: `.azure` கோப்புறையை version control-க்கு commit செய்யலாமா?
**A**: **A**: **இல்லை!** `.azure` கோப்புறை நுட்பமான தகவல்களை கொண்டுள்ளது. அதை `.gitignore` இல் சேர்க்கவும்:
```gitignore
.azure/
```

### Q: ரகசியங்கள் மற்றும் நுணுக்கமான கட்டமைப்புகளை எப்படி கையாள வேண்டியது?
**A**: 
1. ரகசியங்களுக்கு Azure Key Vault-ஐ பயன்படுத்துங்கள்
2. பயன்பாட்டு கட்டமைப்பில் Key Vault ரகசியங்களை குறிப்பிடுங்கள்
3. ரகசியங்களை version control-க்குக் commit செய்யக் கூடாது
4. சேவை-இலிருந்து-சேவைக்கு அங்கீகாரத்திற்கு மேலாண்மை அடையாளங்களை (managed identities) பயன்படுத்துங்கள்

### Q: azd உடன் CI/CD க்கு பரிந்துரைக்கப்பெற்ற அணுகுமுறை என்ன?
**A**: 
1. ஒவ்வொரு கட்டத்திற்கு தனித்தனி சூழல்களைப் பயன்படுத்துங்கள் (dev/staging/prod)
2. deploymentக்கு முன்னர் தானியங்கி சோதனைகளை செயல்படுத்துங்கள்
3. அங்கீகாரத்திற்கு சர்வீஸ் பிரின்சிபல்களை பயன்படுத்துங்கள்
4. நுட்பமான கட்டமைப்புகளை pipeline ரகசியங்கள்/மாறிகளில் சேமிக்கவும்
5. உற்பத்தி deployment க்களுக்கு ஒப்புதல் வாயில்களை (approval gates) செயல்படுத்துங்கள்

## Advanced Topics

### Q: azd-யை தனிப்பயன் செயல்பாடுகளுடன் விரிவாக்கலாமா?
**A**: ஆம், `azure.yaml`-இல் deployment hooks மூலம்:
```yaml
hooks:
  predeploy:
    run: ./scripts/custom-setup.sh
  postdeploy:
    run: ./scripts/custom-config.sh
```

### Q: azd-ஐ ஏற்கனவே உள்ள DevOps செயல்முறைகளுடன் எப்படி ஒருங்கிணைப்பது?
**A**: 
1. ஏற்கனவே உள்ள pipeline ஸ்கிரிப்ட்களில் azd கமாண்டுகளைப் பயன்படுத்துங்கள்
2. குழுக்களுக்காக azd டெம்ப்ளேடுகளில் ஸ்டாண்டர்டைச் செயல்படுத்துங்கள்
3. தற்போதைய கண்காணிப்பு மற்றும் அலெர்டிங்குடன் ஒருங்கிணையுங்கள்
4. pipeline ஒருங்கிணைப்புக்காக azd-இன் JSON output-ஐ பயன்படுத்துங்கள்

### Q: azd-ஐ Azure DevOps-உடன் பயன்படுத்தலாமா?
**A**: ஆம், azd எந்த CI/CD அமைப்புடனும் வேலை செய்கிறது. azd கமாண்டுகளைப் பயன்படுத்தும் Azure DevOps pipelines உருவாக்குங்கள்.

### Q: azd-க்கு எப்படி பங்களிக்கலாம் அல்லது சமுதாய டெம்ப்ளேடுகளை உருவாக்கலாம்?
**A**:
1. **azd கருவி**: [Azure/azure-dev](https://github.com/Azure/azure-dev)க்கு பங்களிக்கவும்
2. **வார்ப்புருக்கள்**: [வார்ப்புரு வழிமுறைகள்](https://github.com/Azure-Samples/awesome-azd)ஐப் பின்பற்றி வார்ப்புருக்கள் உருவாக்கவும்
3. **ஆவணங்கள்**: [MicrosoftDocs/azure-dev-docs](https://github.com/MicrosoftDocs/azure-dev-docs)இல் உள்ள ஆவணங்களுக்கு பங்களிக்கவும்

### Q: azd க்கான திட்டம் என்ன?
**A**: திட்டமிடப்பட்ட அம்சங்கள் மற்றும் மேம்பாடுகளுக்கான திட்டத்தைக் காண [அதிகாரபூர்வ வரைபடம்](https://github.com/Azure/azure-dev/projects) ஐப் பார்க்கவும்.

### Q: மற்ற பதிவேற்ற கருவிகளிலிருந்து azd க்கு எப்படி இடமாற்றம் செய்வது?
**A**: 
1. தற்போதைய பதிவேற்ற கட்டமைப்பை பகுப்பாய்வு செய்யவும்
2. சமமான Bicep வார்ப்புருக்களை உருவாக்கவும்
3. தற்போதைய சேவைகளுடன் பொருந்துமாறு `azure.yaml` ஐ உள்ளமைக்கவும்
4. வளர்ச்சி சுற்றுச்சூழலில் முழுமையாக சோதிக்கவும்
5. படிப்படியாக சுற்றுச்சூழல்களை இடமாற்றம் செய்யவும்
