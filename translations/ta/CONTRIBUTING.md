# AZD தொடங்குவோருக்கான பங்களிப்பு

AZD தொடங்குவோருக்கான பாடநெறிக்கு பங்களிக்க உங்கள் ஆர்வத்திற்கு நன்றி! இந்த ஆவணம் பங்களிப்பாளர்களுக்கான வழிகாட்டுதல்கள் மற்றும் தகவல்களை வழங்குகிறது.

## எப்படி பங்களிக்கலாம்

### பிரச்சனைகள் தெரிவிக்க

நீங்கள் பிழைகள், தவறுகள் அல்லது மேம்படுத்தலுக்கான பரிந்துரைகள் கண்டுபிடித்தால்:

1. **உள்ள பிரச்சனைகளைச் சரிபார்க்கவும்**: நகல்களைத் தவிர்ப்பதற்காக [உள்ள பிரச்சனைகள்](https://github.com/microsoft/azd-for-beginners/issues) ஐத் தேடுங்கள்
2. **புதிய பிரச்சனையை உருவாக்கவும்**: பொருத்தமான பிரச்சனை மாதிரியைப் பயன்படுத்துங்கள்
3. **விவரங்களை வழங்கவும்**: மறுமொழி பெறுவதற்கான படிகள், எதிர்பார்த்த நடத்தை மற்றும் தேவையானால் ஸ்கிரீன்ஷாட்கள் சேர்க்கவும்

### மாற்றங்களை பரிந்துரைப்பது

1. **ரெப்போசிட்டரியை Fork செய்யவும்**: திட்டத்தின் உங்கள் சொந்த fork ஐ உருவாக்கவும்
2. **ஒரு கிளையை உருவாக்கவும்**: உங்கள் மாற்றங்களுக்கு ஒரு feature கிளை உருவாக்குங்கள்
3. **மாற்றங்கள் செய்யவும்**: எங்கள் குறியீட்டு மற்றும் ஆவணத் தரநிலைகளைக் கவனமாக பின்பற்றுங்கள்
4. **மாற்றங்களை சோதிக்கவும்**: அனைத்து இணைப்புகளும் வேலை செய்கிறதா மற்றும் உள்ளடக்கம் சரியாக காணப்படுகிறதா என்பதை உறுதிசெய்யுங்கள்
5. **Pull Request சமர்ப்பிக்கவும்**: உங்கள் மாற்றங்களின் தெளிவான விளக்கத்தை வழங்குங்கள்

### பங்களிப்புகளின் வகைகள்

நாம் பின்வரும் வகையான பங்களிப்புகளை வரவேற்கின்றோம்:

#### உள்ளடக்க மேம்படுத்தல்கள்
- தட்டச்சு பிழைகள், இலக்கண பிழைகள் மற்றும் வடிவமைப்பு சிக்கல்களை சரிசெய்தல்
- உள்ள ஆவணங்களின் தெளிவை மேம்படுத்தல்
- பழமையான தகவல்களை புதுப்பித்தல்
- காணாமல் இருக்கிற விளக்கங்கள் அல்லது சூழ்நிலைகளைச் சேர்த்தல்

#### புதிய உதாரணங்கள்
- உண்மையான உலகில் பயன்படும் இணைப்புகளுக்கான deployment senarios சேர்க்குதல்
- பொதுவான பயன்பாடுகளுக்கான புதிய AZD قالبங்கள் (templates) உருவாக்குதல்
- கூடுதல் குறியீட்டு எடுத்துக்காட்டுகள் மற்றும் பயிற்சிகள் வழங்குதல்

#### மொழிபெயர்ப்பு ஆதரவு
- உள்ளம்காணும் மொழிபெயர்ப்புகளை பராமரிக்க உதவுதல்
- மொழிபெயர்ப்பு தரத்தை மேம்படுத்துதல்
- புதிய உள்ளடக்கங்களுக்கு மொழிபெயர்ப்புகளைச் சேர்த்தல்

#### பிழை அறிக்கைகள்
- உடைந்த இணைப்புகள் அல்லது தவறான குறியீட்டு எடுத்துக்காட்டுகளைப் புகARTறித்தல்
- ஆவண இடையே பொருத்தமின்மைகளை கண்டறிதல்
- அணுகல் சாதகமாக இல்லாத பிரச்சனைகளை சுட்டிக்காட்டு

## பங்களிப்பு வழிகாட்டுதல்கள்

### ஆவணத் தரநிலைகள்

- தெளிவான, தொடக்கநிலையினருக்கான மொழியைப் பயன்படுத்துங்கள்
- கற்கும் இலக்குகள் மற்றும் பெறும் முடிவுகளை சேர்க்கவும்
- எதிர்பார்க்கப்படும் வெளியீடுகளுடன் படி படியாக வழிமுறைகளை வழங்கவும்
- தொடர்புடைய உள்ளடக்கத்திற்கு வழிசெலுத்தும் இணைப்புகளைச் சேர்க்கவும்
- ஒற்றுமையான வடிவமைப்பு மற்றும் மார்க்டவுன் стиль ஐ (style) பயன்படுத்தவும்

### குறியீட்டு எடுத்துக்காட்டுகள்

- அனைத்து குறியீட்டு எடுத்துக்காட்டுகளும் சோதிக்கப்பட்டு செயல்படுவதை உறுதிசெய்யுங்கள்
- முக்கிய கருத்துக்களை விளக்கும் கருத்துரைகளை (comments) சேர்க்கவும்
- Azure மற்றும் AZD சிறந்த நடைமுறைகளை பின்பற்றவும்
- தேவையான இடங்களில் எதிர்பார்க்கப்படும் வெளியீடுகளை வழங்கவும்

### கமிட் செய்திகளை எழுதுவது

தெளிவான, விவரமான கமிட் செய்திகளைப் பயன்படுத்துங்கள்:

- `fix: correct broken link in chapter 2`
