## 🆕 azd இல் இன்று என்ன புதியது

Azure Developer CLI சாதாரண வெப் செயலிகள் மற்றும் API களைவிட அதிகம் விரிவடைந்துள்ளது. இப்போது, azd என்பது Azure இலுக்குச் சர்வதேசமாக எந்தவொரு பயன்பாட்டையும் வழங்குவதற்கான ஒரே கருவி — AI இயக்கப்பட்ட பயன்பாடுகள் மற்றும் புத்திசாலித்தன்மை முகவர்கள் உட்பட.

இதன் பொருள் உங்களுக்கு:

- **AI முகவர்கள் இப்போது azd இல் முதல்தரமான வேலைப்பகுதிகள் ஆக உள்ளன.** நீங்கள் அதே `azd init` → `azd up` வேலைநிரலை பயன்படுத்தி AI முகவர் திட்டங்களை ஆரம்பித்து, இடுக்கி, மற்றும் நிர்வகிக்க முடியும்.
- **Microsoft Foundry ஒருங்கிணைவு** மாதிரியை இடுகை செய்தல், முகவர் ஹோஸ்டிங் மற்றும் AI சேவை கட்டமைப்புகளை நேரடியாக azd மாதிரிக் கண்கோரியில்அள்ளடக்கியது.
- **முக்கிய வேலைநிலை மாறவில்லை.** நீங்கள் todo செயலி, மைக்ரோசர்விஸ் அல்லது பல-முகவர் AI தீர்வை வெளியிடுகிறீர்களோ, கட்டளைகள் அதேவையாக இருக்கும்.

நீங்கள் முன்பு azd ஐப் பயன்படுத்தியிருந்தால், AI ஆதரவு இயல்பான நீட்சியாகும் — தனித்த கருவி அல்லது ஒரு மேம்பட்ட பாடப் பாதை அல்ல. நீங்கள் புதிதாக துவங்கினால், எல்லாவற்றுக்கும் வேலை செய்யும் ஒரு வேலைநிரலை கற்றுக் கொள்வீர்கள்.

## 🚀 Azure Developer CLI (azd) என்றால் என்ன?

**Azure Developer CLI (azd)** என்பது Azure இல் செயலிகளை அளவிட எளிமையாக்கும், டெவலப்பர் நட்பான கட்டளை வரிசை கருவி. பல Azure வளங்களை கைமுறையாக உருவாக்கி இணைக்கும் பதிலாக, ஒரு einzigen கட்டளையொன்றால் முழு பயன்பாடுகளை இடுகை செய்ய முடியும்.

### `azd up` இன் மாயாஜாலம்

```bash
# இந்த ஒற்றை கட்டளை எல்லாவற்றையும் செய்கிறது:
# ✅ அனைத்து Azure வளங்களையும் உருவாக்குகிறது
# ✅ நெட்வொர்க்கிங் மற்றும் பாதுகாப்பை அமைக்கிறது
# ✅ உங்கள் பயன்பாட்டு கோ드를 கட்டமைக்கிறது
# ✅ Azure-க்கு நியமிக்கிறது
# ✅ உங்களுக்கு ஒரு வேலை செய்யும் URL கொடுக்கிறது
azd up
```

**அதே தான்!** Azure போர்ட்டலுக்கு கிளிக் செய்ய வேண்டியதில்லை, சிக்கலான ARM மாதிரிகள் முதலில் கற்பதற்கு தேவையில்லை, கைமுறை அமைப்புகள் வேண்டாம் - வெறும் செயல்படும் செயலிகள் Azure இல்.

## ❓ Azure Developer CLI vs Azure CLI: வேறுபாடு என்ன?

இதுதான் புதியவர்களின் மிகவும் பொதுவான கேள்வி. இங்கே எளிய பதில்:

| அம்சம் | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **நோக்கம்** | தனி Azure வளங்களை நிர்வகிக்க | முழு பயன்பாடுகளை வழங்க |
| **மனப்பான்மை** | உள்கட்டமைப்புக்கு கவனம் | பயன்பாட்டுக்கு கவனம் |
| **உதாரணம்** | `az webapp create --name myapp...` | `azd up` |
| **கற்றல் வளைவு** | Azure சேவைகளை தெரிந்திருக்க வேண்டும் | உங்கள் செயலியை தெரிந்து கொள்ளுங்கள் |
| **சிறந்தது** | DevOps, உள்கட்டமைப்பு | டெவலப்பர்கள், வடிவமைப்பு மற்றும் முன்னோட்டம் |

### எளிய ஒப்புமை

- **Azure CLI** என்பது ஒரு வீடு கட்டுவதற்கான எல்லா கருவிகளையும் வைத்திருப்பதைப் போன்றது — பெருகிகள், நறுக்கும் கருவிகள், நகல்கள். நீங்கள் எதையும் கட்டிக்கொள்ள முடியும், ஆனால் கட்டுமானத்தைப் பற்றி தெரிந்திருக்க வேண்டும்.
- **Azure Developer CLI** என்பது ஒரு கட்டமைப்பாளரை வேலைக்கு எடுத்துக் கொள்ளுவது போன்றது — நீங்கள் எதைப் படைக்க விரும்புகிறீர்கள் என்பதை விவரிக்கின்றீர்கள், அவர்கள் கட்டுமானத்தை கவனித்து செய்கிறார்கள்.

### எப்போது எதை பயன்படுத்தலாம்

| காட்சி | இதைப் பயன்படுத்தவும் |
|----------|----------|
| "என் வெப் செயலியை விரைவாக வெளியிட விரும்புகிறேன்" | `azd up` |
| "எனக்கு ஒரு storage account மட்டும் உருவாக்க வேண்டும்" | `az storage account create` |
| "நான் ஒரு முழு AI பயன்பாட்டை கட்டுகிறேன்" | `azd init --template azure-search-openai-demo` |
| "எனக்கு ஒரு குறிப்பிட்ட Azure வளத்தை தவிர்க்க சிக்கலை டீபக் செய்ய வேண்டும்" | `az resource show` |
| "நான் நிமிடங்களில் தயாராகும் production-level இடுகையை விரும்புகிறேன்" | `azd up --environment production` |

### அவை ஒன்றாக வேலை செய்யும்!

AZD கீழ் Azure CLI ஐ பயன்படுத்துகிறது. நீங்கள் இரண்டையும் பயன்படுத்தலாம்:
```bash
# உங்கள் செயலியை AZD கொண்டு வெளியிடவும்
azd up

# பின்னர் Azure CLI கொண்டு குறிப்பிட்ட வளங்களை நுணுக்கமாக உள்ளமைக்கவும்
az webapp config set --name myapp --always-on true
```

## 🎯 3 படிகளில் தொடக்கம்

**Windows:**
```powershell
.\validate-setup.ps1
```

**macOS / Linux:**
```bash
bash ./validate-setup.sh
```

If any required check fails, fix that first and then continue with the quick start.

### படி 1: AZD ஐ நிறுவவும் (2 நிமிடம்)

**Windows:**
```powershell
winget install microsoft.azd
```

**macOS:**
```bash
brew tap azure/azd && brew install azd
```

**Linux:**
```bash
curl -fsSL https://aka.ms/install-azd.sh | bash
```

### படி 2: AZD க்காக அங்கீகாரம் பெறுங்கள்

```bash
# இந்த பாடத்தில் நேரடியாக Azure CLI கட்டளைகள் பயன்படுத்த திட்டமிட்டால் விருப்பமானது
az login

# AZD வேலைப்பாடுகளுக்கு அவசியமானது
azd auth login
```

If you are not sure which one you need, follow the full setup flow in [Installation & Setup](docs/chapter-01-foundation/installation.md#authentication-setup).

### படி 3: உங்கள் முதல் செயலியை வெளியிடுங்கள்

```bash
# வார்ப்புருவில் இருந்து துவங்கவும்
azd init --template todo-nodejs-mongo

# Azure-க்கு அமைக்கவும் (எல்லாவற்றையும் உருவாக்குகிறது!)
azd up
```

**🎉 முடிந்துவிட்டது!** உங்கள் செயலி இப்போது Azure இல் உயிருடன் உள்ளது.

### சுத்தப்படுத்தல் (மறக்காதீர்கள்!)

```bash
# Remove all resources when done experimenting
azd down --force --purge
```

இந்த கட்டமைக்கப்பட்ட பாடநெறியை முடித்தால், நீங்கள்:
- **AZD அடிப்படைத் திறன்களில் தேர்ச்சி பெறுங்கள்**: மூலக் கருத்துகள், நிறுவல் மற்றும் உள்ளமைவு
- **AI செயலிகளை வெளியிடுங்கள்**: Microsoft Foundry சேவைகளுடன் AZD ஐப் பயன்படுத்துதல்
- **Infrastructure as Code ஐ செயல்படுத்துங்கள்**: Bicep மாதிரிகளுடன் Azure வளங்களை நிர்வகித்தல்

## நீங்கள் தொடங்குவதற்கு முன்: கணக்குகள், அணுகல் மற்றும் கருதிகள்

அத்தியாயம் 1 ஐத் தொடங்குவதற்கு முன், கீழ்க்காணும் அம்சங்களை ஏற்கனவே வைத்திருக்கவும். இந்த கையேட்டின் பின்னர் உள்ள நிறுவல் படிகள் இவையே முன்னதாக கையாளப்பட்டிருப்பதாக கணக்கிடும்.
- **ஒரு Azure சந்தா**: நீங்கள் பணிக்கான ஏற்கனவே உள்ள சந்தாவைக் பயன்படுத்து அல்லது உங்கள் சொந்த கணக்கில் காணலாம், அல்லது தொடங்க இது போன்ற [free trial](https://aka.ms/azurefreetrial) ஒன்றை உருவாக்குங்கள்.
- **Azure வளங்களை உருவாக்க அனுமதி**: பெரும்பாலான பயிற்சிகளுக்காக, இலக்குக் சந்தா அல்லது resource group-இல் குறைந்தது **Contributor** அணுகல் இருக்க வேண்டும். சில அத்தியாயங்களில் resource groups, managed identities, மற்றும் RBAC ஒதுக்கீடுகளை உருவாக்க முடியும் என கருதப்படலாம்.
- [**ஒரு GitHub கணக்கு**](https://github.com): களஞ்சியத்தை fork செய்வது, உங்கள் சொந்த மாற்றங்களை கண்காணிப்பது, மற்றும் workshop க்காக GitHub Codespaces ஐப் பயன்படுத்த இது பயனுள்ளது.
- **டெம்ப்ளேட் இயக்கநேர முன்னோக்கியங்கள்**: சில டெம்ப்ளேட்களுக்கு Node.js, Python, Java, அல்லது Docker போன்ற உள்ளூர் கருவிகள் அவசியம். தொடக்கத்திற்கு முன் setup validator-ஐ இயக்கி காணாமல் போன கருவிகளை அவசரமாக கண்டறியுங்கள்.
- **அடிப்படை டெர்மினல் பழக்கம்**: நிபுணர் இருக்க தேவையில்லை, ஆனால் `git clone`, `azd auth login`, மற்றும் `azd up` போன்ற கட்டளைகளை இயக்குவதில் ஆறுதல் வேண்டும்.

> **ஒரு நிறுவன சந்தாவில் பணியாற்றுகிறீர்களா?**
> உங்கள் Azure சூழல் ஒரு நிர்வாகியால் நிர்வகிக்கப்படுவதாக இருந்தால், நீங்கள் பயன்படுத்த திட்டமிட்ட சந்தா அல்லது resource group இல் வளங்களை அமைக்க முடிவதாக முன்கூட்டியே உறுதிசெய்க. இல்லையெனில், தொடங்கும் முன் ஒரு sandbox சந்தா அல்லது Contributor அணுகலை கேட்கவும்.

> **Azure நுட்பத்தில் புதியவரா?**
> உங்கள் சொந்த Azure trial அல்லது pay-as-you-go சந்தாவுடன் https://aka.ms/azurefreetrial தொடங்கி, tenant-அடிப்படை அங்கீகாரங்களை காத்திருக்காமல் பயிற்சிகளை முடிக்க முடியும்.

- **பாதுகாப்பான AI டெப்ளாய்மென்டுகள்** - AI சேவைகள், API விசைகள் மற்றும் மாதிரி எண்ட்பாயின்டுகளுக்கான உட்பொதிவான பாதுகாப்பு மாதிரிகள்  
- **உற்பத்தி AI மாதிரிகள்** - அதிகளவுருவான, செலவினமிக்க இல்லாத AI பயன்பாட்டு டெப்ளாய்மென்ட் சிறந்த நடைமுறைகள்
- **முழு-முனை AI வேலைபாடுகள்** - மாதிரி மேம்பாட்டிலிருந்து உற்பத்தி டெப்ளாய்மென்ட் வரை சரியான கண்காணிப்புடன்
- **செலவு 최적화** - AI பணிகளுக்கான புத்திசாலி வள ஒதுக்கீடு மற்றும் அளவுஎழுச்சிยุதிகள்
- **Microsoft Foundry இணைப்பு** - Microsoft Foundry மாதிரி каталோ மற்றும் எண்ட்பாயின்டுகளுடன் இணைப்பு

```bash
# AZD இற்காக மீண்டும் அங்கீகாரம் செய்யவும்
azd auth logout
azd auth login

# தேர்ந்தெடுக்கக்கூடியது: நீங்கள் az கட்டளைகளை இயக்கினால் Azure CLI-ஐவும் புதுப்பிக்கவும்
az logout
az login

# அங்கீகாரத்தை சரிபார்க்கவும்
az account show
```
</details>

<details>
<summary><strong>❌ "Resource already exists" or naming conflicts</strong></summary>

```bash
# AZD தனித்துவமான பெயர்களை உருவாக்குகிறது, ஆனால் மோதல் ஏற்பட்டால்:
azd down --force --purge

# அப்போது புதிய சூழலுடன் மீண்டும் முயற்சிக்கவும்
azd env new dev-v2
azd up
```
</details>

<details>
<summary><strong>❌ Template deployment taking too long</strong></summary>

**சாதாரண காத்திருக்கும் நேரங்கள்:**
- எளிய வலை பயன்பாடு: 5-10 நிமிடங்கள்
- தரவுத்தளத்துடன் பயன்பாடு: 10-15 நிமிடங்கள்
- AI பயன்பாடுகள்: 15-25 நிமிடங்கள் (OpenAI வழங்கல் மெதுவாக இருக்கும்)

```bash
# முன்னேற்றத்தை சரிபார்க்கவும்
azd show

# 30 நிமிடத்திற்கும் அதிகமாக நிலைத்து இருந்தால், Azure போர்டலைச் சரிபார்க்கவும்:
azd monitor --overview
# தோல்வியுற்ற நிறுவல்களைத் தேடவும்
```
</details>

<details>
<summary><strong>❌ "Permission denied" or "Forbidden"</strong></summary>

```bash
# உங்கள் Azure பங்கை சரிபார்க்கவும்
az role assignment list --assignee $(az account show --query user.name -o tsv)

# உங்களுக்கு குறைந்தபட்சமாக "Contributor" பங்கு தேவை
# உங்கள் Azure நிர்வாகியை கீழ்காணும் பங்குகளை வழங்குமாறு கேளுங்கள்:
# - Contributor (வளங்களுக்காக)

```
