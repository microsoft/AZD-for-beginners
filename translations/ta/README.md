## 🆕 azd இல் இன்று என்ன புதிது

> 📌 இந்த பாடநெறி **`azd 1.25.6`** (ஜூன் 2026) மீது சரிபார்க்கப்பட்டிருக்கிறது. உங்கள் கட்டமைப்பை சரிபார்க்க `azd version` ஓட்டவும், மற்றும் சமீபத்தியதைக் பெற `azd upgrade` செய்க.

Azure Developer CLI கம்ப்யூட்டர் வழக்கமான வெப் அப்ளிகேஷன்களையும் API-களையும் கடந்தும் வளர்ந்துள்ளது. இன்று, azd என்பது Azure-க்கு எந்தவொரு பயன்பாட்டையும் (AI சக்தியுள்ள பயன்பாடுகள் மற்றும் அறிந்து செயல்படும் முகவரிகள் உட்பட) deploy செய்ய ஒரே கருவியாக மாறியுள்ளது.

உங்களுக்கு இதன் பொருள் என்னவென்றால்:

- **AI முகவரிகள் இப்போது azd வேலைப்படுத்தல்களில் முதன்மை இடம் பெற்றுள்ளன.** நீங்கள் அதே `azd init` → `azd up` வேலைப்பழக்கத்தை பயன்படுத்தி AI முகவர் திட்டங்களை initialize, deploy மற்றும் நிர்வகிக்கலாம்.
- **CLI மூலம் முழு முகவர் வாழ்நாள் சுழற்சி.** `azure.ai.agents` விரிவாக்கம் இப்போது முழு பயணத்தையும் கையாளுகிறது—scaffold செய்ய `azd ai agent init`, சோதிக்க `azd ai agent invoke` (பதில்-நேர அளவீட்டு வெளிப்பாட்டுடன்), தரத்தை அளக்கவும் மேம்படுத்தவும் `azd ai agent eval generate` மற்றும் `azd ai agent optimize`, மற்றும் சுத்தம் செய்ய `azd ai agent delete`.
- **மேலும் AI கட்டுமான தொகுதிகள்.** புதிய முன்னோட்ட விரிவாக்கங்கள்—`azure.ai.skills` மற்றும் `azure.ai.connections`—azd மூலம் நேரடியாக மறுபயன்பாடுகூடிய முகவர் திறன்கள் மற்றும் Foundry இணைப்புகளை நிர்வகிக்க мүмкіндік கொடுக்கின்றன.
- **Microsoft Foundry ஒருங்கிணைப்பு** மாதிரி deployசெய்தல், முகவர் ஹோஸ்டிங், மற்றும் AI சேவை கட்டமைப்பை நேரடியாக azd டெம்ப்ளேட் சூழலில் கொண்டு வருகிறது.
- **அன்றாட அடிப்படை செயல்களில் மென்மையாக்கம்.** சமீபத்திய வெளியீடுகள் `azd init` ஐ idempotent (மீண்டும் ஓட்டுவதற்கு பாதுகாப்பான) ஆக்கியுள்ளன, `azd auth login` பழைய டோக்கன்களை தானாக அகற்றும் வகையில் மாற்றியுள்ளன, மற்றும் நண்பனான `azd tool` முதன்மை ஓட்ட அமைப்புக் கேள்வியைக் கூட சேர்த்துள்ளன.
- **மூல வேலைப்பழக்கம் மாறவில்லை.** நீங்கள் todo செயலியை, மைக்ரோசர்வீஸை, அல்லது பல முகவர் AI தீர்வுகளை deploy செய்தாலும், கட்டளைகள் ஒரே சமமாக இருக்கும்.

> **Aspire பயனர்களுக்கான குறிப்பு:** Microsoft இப்போது இந்த தயாரிப்பை எளிய பெயராக **Aspire** என்று குறிப்பிடுகிறது (முந்தைய பெயர் ".NET Aspire"). azd இன் Aspire ஆதரவு மாற்றமின்றி தொடர்கிறது—மாற்றம் வந்தது சுலபமாக பெயர்தான்.

முன்னதாக azd ஐப் பயன்படுத்தியிருந்தால், AI ஆதரவு ஒரு இயல்பான விரிவாகும்—விதிவிரோத கருவியோ அல்லது மேல்நிலை பாதையோ அல்ல. புதிதாகத் தொடங்கினால், அனைத்திற்கும் பொருந்தும் ஒரே வேலைப்பழக்கத்தை நீங்கள் கற்றுக்கொள்வீர்கள்.

## 🚀 Azure Developer CLI (azd) என்றால் என்ன?

**Azure Developer CLI (azd)** என்பது டெவலப்பர்களுக்கான நட்பு கொண்ட கட்டளை வரிசை கருவி, Azureக்கு பயன்பாடுகளை deploy செய்வதை எளிதாக்குகிறது. நூற்றடிகளாக Azure வளங்களை கைமுறையாக உருவாக்கி இணைக்கும்போது நேரத்தை கழிக்குவதற்குப் பதிலாக, ஒரு கட்டளையால் முழு பயன்பாடுகளையும் deploy செய்யலாம்.

### `azd up` என்ன கற்பனை செய்கிறது

```bash
# இந்த ஒரே கட்டளை எல்லாவற்றையும் செய்கிறது:
# ✅ அனைத்து அசூர் வளங்களையும் உருவாக்குகிறது
# ✅ நெட்வொர்க்கிங் மற்றும் பாதுகாப்பை அமைக்கிறது
# ✅ உங்கள் பயன்பாட்டு கோடியை கட்டுகிறது
# ✅ அசூருக்கு பிரதி இடுகிறது
# ✅ வேலைபோன்ற URL ஐ வழங்குகிறது
azd up
```

**அதேது!** Azure போர்டலை கிளிக் செய்ய தேவையில்லை, ARM டெம்ப்ளேட்களை முதலில் கற்றுக் கொள்ள தேவையில்லை, கைமுறை கட்டமைப்புகள் தேவையில்லை - பயன்பாடுகள் நேரடியாக Azureல் இயங்கும்.

## ❓ Azure Developer CLI vs Azure CLI: வேறுபாடு என்ன?

இதுதான் துவக்கக்காரர்கள் அடிக்கடி கேட்கும் கேள்வி. இங்கே எளிய பதில்:

| அம்சம் | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **நோக்கம்** | தனிப்பட்ட Azure வளங்களை நிர்வகிக்க | முழுமையான பயன்பாடுகளை deploy செய்ய |
| **மனப்பான்மை** | அடித்தள-கவனம் | பயன்பாடு-கவனம் |
| **உதாரணம்** | `az webapp create --name myapp...` | `azd up` |
| **கற்றல் வளைவு** | Azure சேவைகளை அறிந்திருக்க வேண்டும் | உங்கள் பயன்பாட்டை மட்டும் தெரிந்திருக்கவும் போதுமானது |
| **பொருத்தமானது** | DevOps, அடித்தளம் | மேம்படுத்துனர்கள், முன்மாதிரி உருவாக்குதல் |

### எளிய ஒப்புமை

- **Azure CLI** என்பது வீடு கட்டுவதற்கான அனைத்து கருவிகளையும் வைத்திருப்பதுபோல்—ஹாம்மர்கள், சாய், நெயில். நீங்கள் எதையும் கட்டக்கூடும், ஆனால் கட்டுமானத்தைப் பற்றி அறிவிருக்க வேண்டும்.
- **Azure Developer CLI** என்பது ஒரு ஒப்பந்தக்காரரை வேலைக்காக அழைத்து வைத்திருப்பதுபோல்—நீங்கள் என்னவேண்டும் என்று விவரிக்கிறீர்கள், அவர்கள் கட்டிடத்தை நிர்வகிக்கின்றார்கள்.

### எப்போது எதைப் பயன்படுத்துவது

| சூழ்வு | இதைப் பயன்படுத்தவும் |
|----------|----------|
| "நான் என் வெப் செயலியை விரைவாக deploy செய்ய விரும்புகிறேன்" | `azd up` |
| "நான் வெறும் ஒரு storage account உருவாக்க வேண்டும்" | `az storage account create` |
| "நான் முழுமையான AI பயன்பாட்டை உருவாக்குகிறேன்" | `azd init --template azure-search-openai-demo` |
| "நான் ஒரு குறிப்பிட்ட Azure வளத்தை டீபக் செய்ய வேண்டும்" | `az resource show` |
| "நான் நிமிடங்களில் production-தயார் deployment விரும்புகிறேன்" | `azd up --environment production` |

### அவை ஒன்றாக வேலை செய்கின்றன!

AZD மேற்கணையின் கீழ் Azure CLI-ஐ பயன்படுத்துகிறது. நீங்கள் இரண்டையும் பயன்படுத்தலாம்:
```bash
# உங்கள் செயலியை AZD உடன் வழங்கவும்
azd up

# பின்னர் Azure CLI கொண்டு குறிப்பிட்ட வளங்களை நுணுக்கமாகச் சரிசெய்க
az webapp config set --name myapp --always-on true
```

## 🎯 3 படிகளில் துவக்கம்

தொடங்குவதற்கு முன்னர், நீங்கள் deploy செய்யவிருக்கும் டெம்ப்ளேட்டிற்காக உங்கள் இயந்திரம் தயார் என்று உறுதிசெய்யுங்கள்:

**Windows:**
```powershell
.\validate-setup.ps1
```

**macOS / Linux:**
```bash
bash ./validate-setup.sh
```

எந்தவொரு தேவையான சோதனை தோல்வியடைந்தால், முதலில் அதை சரிசெய்து விரைவு தொடக்கத்துடன் தொடருங்கள்.

### படி 1: AZD ஐ நிறுவுதல் (2 நிமிடம்)

**Windows:**
```powershell
winget install microsoft.azd
```

**macOS:**
```bash
brew tap azure/azd && brew install azd
```

**Linux:**
```bash
curl -fsSL https://aka.ms/install-azd.sh | bash
```

### படி 2: AZD க்கான அங்கீகாரம்

```bash
# இந்த பாடத்தில் நேரடியாக Azure CLI கட்டளைகளை பயன்படுத்த திட்டமிட்டிருந்தால் விருப்பமானது
az login

# AZD பணிச்சூழல்களுக்கு அவசியம்
azd auth login
```

நீங்கள் எந்த ஒன்றை வேண்டும் என்பது நிச்சயமாக இல்லை என்றால், [Installation & Setup](docs/chapter-01-foundation/installation.md#authentication-setup) இல் முழு அமைப்பு பாயை பின்பற்றவும்.

### படி 3: உங்கள் முதல் செயலியை Deploy செய்யுங்கள்

```bash
# ஒரு மாதிரியில் இருந்து துவக்கம் செய்க
azd init --template todo-nodejs-mongo

# Azureக்குள் செயல்படுத்துக (எல்லாம் உருவாக்குகிறது!)
azd up
```

**🎉 அது முடிந்தது!** உங்கள் செயலி இப்போது Azure இல் செயல்படுகிறது.

### சுத்தைப்படுத்துதல் (மறக்காதீர்கள்!)

```bash
# Remove all resources when done experimenting
azd down --force --purge
```

## பாடநெறி கண்ணோட்டம்

கட்டமைக்கப்பட்ட அத்தியாயங்கள் மூலம் Azure Developer CLI (azd) ஐ அடிக்கடி கற்றுக்கொள்வதற்கான திறன்களை வல்லுநராக்குங்கள். **Microsoft Foundry ஒருங்கிணைப்புடன் AI பயன்பாட்டை deploy செய்வதில் சிறப்பு கவனம்.**

Microsoft Foundry Discord சமுதாயக் கருத்துகளின் அடிப்படையில், **45% டெவலப்பர்கள் AI பணி ஏற்றுமதிகளுக்கு AZD பயன்படுத்த விரும்புகிறார்கள்** ஆனால் பின்வரும் சவால்களை எதிர்கொள்கின்றனர்:

- AI பணி ஏற்றுமதிகளுக்கான செலவு 최적화
- AI-சார்ந்த 배포 சிக்கல்களை தீர்க்குதல்

### கற்றல் இலக்குகள்

இந்த கட்டமைக்கப்பட்ட பாடநெறியை முடித்தவுடன், நீங்கள்:

- **AI பயன்பாடுகளை 배포 செய்யலாம்**: Microsoft Foundry சேவைகளுடன் AZD பயன்படுத்துதல்
- **Infrastructure as Code செயல்படுத்தலாம்**: Bicep மாதிரிகளுடன் Azure வளங்களை நிர்வகிக்கும்
- **배포 குறையினைகளை தீர்க்கலாம்**: பொதுவான சிக்கல்களை தீர்த்து பிழைப் பார்

## நீங்கள் தொடங்குவதற்கு முன்: கணக்குகள், அணுகல் மற்றும்ophagus கருதுகோள்கள்

அத்தியாயம் 1 ஐத் தொடங்குவதற்கு முன், கீழ்க்காணும்வை ஐயமுள்ளதா என்பதை உறுதிசெய்யவும். இந்த வழிகாட்டியின் பின் உள்ள நிறுவல் படிகள் இவை அடிப்படையாக ஏற்கப்படும்.

- **An Azure subscription**: நீங்கள் வேலைவாய்ப்பு அல்லது உங்கள் சொந்த கணக்கிலிருந்து உள்ள саб்ஸ்கிரிப்ஷன் பயன்படுத்தலாம், அல்லது தொடங்க [free trial](https://aka.ms/azurefreetrial) ஒன்றை உருவாக்கலாம்.
- **Permission to create Azure resources**: பல பயிற்சிகளுக்கு, நோக்குவதாக உள்ள sabscription அல்லது resource group இல் குறைந்தது **Contributor** அணுகல் இருக்க வேண்டும். சில அத்தியாயங்கள் resource groups, managed identities, மற்றும் RBAC ஒதுக்கீடுகள் உருவாக்குவதாகவும் கருதலாம்.
- [**A GitHub account**](https://github.com): அதிர்ச்சிகரமான ரெப்பொசிடொரியை fork செய்வதற்கும், உங்கள் மாற்றங்களை கண்காணிப்பதற்கும், மற்றும் கையேடு பணிக்காக GitHub Codespaces பயன்படுத்துவதற்கும் இது பயன்படும்.
- **Template runtime prerequisites**: சில சாஞ்சிகைகள் உள்ளூர் கருவிகள் போன்றவை தேவைப்படலாம்: Node.js, Python, Java, அல்லது Docker ஆகியவை. தொடங்குவதற்கு முன் setup validator ஐ இயக்கி நீங்க தக்க கருவிகள் இல்லாமையை முன்னர் பிடியுங்கள்.
- **Basic terminal familiarity**: நீங்கள் நிபுணரா இருக்க வேண்டும் என்பது அவசியமில்லை, ஆனால் `git clone`, `azd auth login`, மற்றும் `azd up` போன்ற கட்டளைகளை இயக்குவதில் வசதியாக இருக்க வேண்டும்.

> **An enterprise subscription இல் பணி செய்யவா?**
> உங்கள் Azure சுற்றுப்பார்வை நிர்வாகியால் நிர்வகிக்கப்படுகிறதெனில், நீங்கள் பயன்படுத்தத் திட்டமிட்ட subscription அல்லது resource group இல் வளங்களை 배포 செய்ய முடியும் என்று முன்கூட்டியே உறுதிசெய்யவும். இல்லையெனில், தொடங்குவதற்கு முன் sandbox subscription அல்லது Contributor அணுகலை கேளுங்கள்.

> **Azure இல் புதியவரா?**
> உங்கள் சொந்த Azure trial அல்லது pay-as-you-go subscription இல் தொடங்குங்கள்: https://aka.ms/azurefreetrial இதனால் tenant-தர நிலை ஒப்புதல்களை காத்திருக்காமல் பயிற்சிகளை முடிக்க முடியும்.

azd version              # நிறுவப்பட்ட பதிப்பை காட்டுகிறது

azd up                  # Azure-க்கு வெளியிடுகிறது
azd show                # ஓடியிருக்கிற செயலியின் URL-ஐ காட்டுகிறது
# விண்ணப்பம் உலாவியில் திறக்கப்பட்டு சரியாக செயல்படுகிறது
azd down --force --purge  # வளங்களை சுத்தம் செய்கிறது

### 🤖 அத்தியாயம் 2: AI-முதன்மை மேம்பாடு (AI டெவலப்பர்களுக்கு பரிந்துரைக்கப்படுகிறது)

**சிக்கல்**: ⭐⭐

#### நடைமுறை பயிற்சிகள்
```bash
# உங்கள் முதல் செயற்கை நுண்ணறிவு பயன்பாட்டை வெளியிடவும்
azd init --template azure-search-openai-demo
azd up

# மேலும் செயற்கை நுண்ணறிவு வார்ப்புருக்களை முயற்சிக்கவும்
azd init --template openai-chat-app-quickstart
azd init --template agent-openai-python-prompty
```

**💡 அத்தியாய முடிவு**: RAG திறன்களுடன் AI சக்தி கொண்ட chatte பயன்பாட்டை 배포 செய்து கட்டமைக்கவும்

**✅ வெற்றி சரிபார்ப்பு:**
```bash
# அத்தியாயம் 2 முடிந்தவுடன், நீங்கள் இதைச் செய்யக்கூடியவராக இருப்பீர்கள்:
azd init --template azure-search-openai-demo
azd up
# AI உரையாடல் இடைமுகத்தை சோதிக்கவும்
# கேள்விகள் கேட்டு ஆதாரங்களுடன் AI‑சக்தி கொண்ட பதில்களைப் பெறவும்
# தேடல் ஒருங்கிணைப்பு வேலை செய்கிறதா என்பதை உறுதிப்படுத்தவும்
azd monitor  # Application Insights டெலிமெட்ரி காட்டப்படுகிறதா என்பதைச் சரிபார்க்கவும்
azd down --force --purge
```

**📊 நேர முதலீடு:** 1-2 மணி  
**📈 பயிற்சி முடிவில் திறன் நிலை:** உற்பத்தி-தயாரான AI பயன்பாடுகளை 배포 செய்து கட்டமைக்க முடியும்  
**💰 செலவு விழிப்புணர்வு:** dev செலவுகள் $80-150/மாதம், உற்பத்தி செலவுகள் $300-3500/மாதம் என புரிந்துகொள்ளவும்

#### 💰 AI 배포க்கான செலவு கவணிப்புகள்

**வளர்ச்சி சூழல் (கணிக்கப்பட்டது $80-150/மாதம்):**
- Microsoft Foundry Models (Pay-as-you-go): $0-50/மாதம் (token பயன்பாட்டின் அடிப்படையில்)
- AI Search (Basic tier): $75/மாதம்
- Container Apps (Consumption): $0-20/மாதம்
- Storage (Standard): $1-5/மாதம்

**உற்பத்தி சூழல் (கணிக்கப்பட்டது $300-3,500+/மாதம்):**
- Microsoft Foundry Models (PTU for consistent performance): $3,000+/மாதம் அல்லது அதிக அளவுக்கு Pay-as-go
- AI Search (Standard tier): $250/மாதம்
- Container Apps (Dedicated): $50-100/மாதம்
- Application Insights: $5-50/மாதம்
- Storage (Premium): $10-50/மாதம்

**💡 செலவு 최적화 குறிப்புகள்:**
- கற்க அதற்காக **Free Tier** Microsoft Foundry Models பயன்படுத்தவும் (Azure OpenAI 50,000 tokens/மாதம் உட்பட)
- செயலில் இல்லாதபோது வளங்களை வகைப்படுத்த `azd down` ஓட்டவும்
- தொடக்கத்தில் consumption-based பில்லிங்கைப் பயன்படுத்தி, உற்பத்திக்கு PTU மட்டும் மேம்படுத்தவும்
- 배포க்கு முன் செலவுகளை கணக்கிட `azd provision --preview` பயன்படுத்தவும்
- auto-scaling ஐ இயக்கு: உண்மையான பயன்பாட்டிற்கு மட்டும் பணம் கொடுங்கள்

**செலவு கண்காணிப்பு:**
```bash
# மாதாந்திர மதிப்பிடப்பட்ட செலவுகளை சரிபார்க்கவும்
azd provision --preview

# Azure போர்டலில் உண்மையான செலவுகளை கண்காணிக்கவும்
az consumption budget list --resource-group <your-rg>
```

- **AI-பூர்வமான வார்ப்புருக்கள்** - Microsoft Foundry Models, Azure AI சேவைகள் மற்றும் ML பணிச்சுமைகளைத் தானாக அமைக்கும் முன்-கட்டமைக்கப்பட்ட வார்ப்புருக்கள்

- **முழு-சுற்றான AI வேலைபாடுகள்** - மாடல் அபிவிருத்தியிலிருந்து உற்பத்தி அமர்த்தலுக்கு வரை சரியான கண்காணிப்புடன்
- **செலவு மேம்பாடு** - AI பணிச்சுமைகளுக்கான புத்திசாலி வள ஒதுக்கீடு மற்றும் ஸ்கேலிங் தந்திரங்கள்
- **Microsoft Foundry ஒருங்கிணைப்பு** - Microsoft Foundry மாடல் பட்டியலும் இடைமுகங்களுடனும் எளிதான இணைப்பு

## 🔧 Quick Troubleshooting Guide

**தொடக்கக்காரர்கள் எதிர்கொள்ளும் பொதுவான பிழைகள் மற்றும் உடனடி தீர்வுகள்:**

<details>
<summary><strong>❌ "azd: கட்டளை காணப்படவில்லை"</strong></summary>

```bash
# முதலில் AZD ஐ நிறுவவும்
# விண்டோஸ் (பவர்‌ஷெல்):
winget install microsoft.azd

# macOS:
brew tap azure/azd && brew install azd

# லினக்ஸ்:
curl -fsSL https://aka.ms/install-azd.sh | bash

# நிறுவலை சரிபார்க்கவும்
azd version
```
</details>

<details>
<summary><strong>❌ "சப்ஸ்கிரிப்ஷன் கிடைக்கவில்லை" அல்லது "சப்ஸ்கிரிப்ஷன் அமைக்கப்படவில்லை"</strong></summary>

```bash
# கிடைக்கக்கூடிய சந்தாக்களை பட்டியலிடவும்
az account list --output table

# இயல்புநிலை சந்தாவை அமைக்கவும்
az account set --subscription "<subscription-id-or-name>"

# AZD சூழலுக்காக அமைக்கவும்
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# சரிபார்க்கவும்
az account show
```
</details>

<details>
<summary><strong>❌ "InsufficientQuota" அல்லது "கொட்டா மீறப்பட்டது"</strong></summary>

```bash
# வேறு Azure பிராந்தியத்தை முயற்சி செய்யவும்
azd env set AZURE_LOCATION "westus2"
azd up

# அல்லது அபிவிருத்தியில் சிறிய SKUகளைப் பயன்படுத்தவும்
# infra/main.parameters.json ஐ திருத்தவும்:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>

<details>
<summary><strong>❌ "azd up" நடுவே தோல்வியடைகிறது</strong></summary>

```bash
# விருப்பம் 1: சுத்தம் செய்து மீண்டும் முயற்சிக்கவும்
azd down --force --purge
azd up

# விருப்பம் 2: கட்டமைப்பையே சரிசெய்யவும்
azd provision

# விருப்பம் 3: விரிவான நிலையைச் சரிபார்க்கவும்
azd show

# விருப்பம் 4: Azure Monitor இல் பதிவுகளைப் பார்க்கவும்
azd monitor --logs
```
</details>

<details>
<summary><strong>❌ "அங்கீகாரம் தோல்வியடைந்தது" அல்லது "டோக்கன் காலாவதி"</strong></summary>

```bash
# AZD க்காக மீண்டும் அங்கீகாரம் செய்யவும்
azd auth logout
azd auth login

# விருப்பமானது: நீங்கள் az கட்டளைகளை இயக்கினால் Azure CLI-யையும் புதுப்பிக்கவும்
az logout
az login

# அங்கீகாரத்தை சரிபார்க்கவும்
az account show
```
</details>

<details>
<summary><strong>❌ "வளம் ஏற்கனவே உள்ளது" அல்லது பெயர் முரண்பாடுகள்</strong></summary>

```bash
# AZD தனித்துவமான பெயர்களை உருவாக்குகிறது, ஆனால் மோதல் ஏற்பட்டால்:
azd down --force --purge

# அப்போது புதிய சூழ்நிலையுடன் மீண்டும் முயற்சிக்கவும்
azd env new dev-v2
azd up
```
</details>

<details>
<summary><strong>❌ டெம்ப்ளேட் டெப்பிளாய்மென்ட் நீண்ட நேரம் எடுக்கும்</strong></summary>

**சாதாரண காத்திருப்புத் நேரங்கள்:**
- எளிய வலை பயன்பாடு: 5-10 நிமிடங்கள்
- தரவுத்தளத்துடன் பயன்பாடு: 10-15 நிமிடங்கள்
- AI செயலிகள்: 15-25 நிமிடங்கள் (OpenAI வழங்குதல் மந்தமாக இருக்கும்)

```bash
# முன்னேற்றத்தை சரிபார்க்கவும்
azd show

# 30 நிமிடங்களுக்கு மேல் சிக்கியிருந்தால், Azure போர்டலைச் சரிபார்க்கவும்:
azd monitor --overview
# தோல்வியடைந்த டெப்ளாய்மெண்ட்களைப் பார்க்கவும்
```
</details>

<details>
<summary><strong>❌ "அனுமதி மறுக்கப்பட்டது" அல்லது "தடை செய்யப்பட்டது"</strong></summary>

```bash
# உங்கள் Azure பங்கினை சரிபார்க்கவும்
az role assignment list --assignee $(az account show --query user.name -o tsv)

# உங்களுக்கு குறைந்தபட்சம் "Contributor" பங்கு வேண்டும்
# உங்கள் Azure நிர்வாகியிடம் வழங்குமாறு கேளுங்கள்:
# - Contributor (வளங்களுக்கு)
# - User Access Administrator (பங்கு ஒதுக்கீடுகளுக்கு)
```
</details>

<details>
<summary><strong>❌ வெளியிடப்பட்ட பயன்பாடு URL ஐ கண்டறிய முடியவில்லை</strong></summary>

```bash
# அனைத்து சேவை எண்ட்பாயின்ட்களைக் காட்டு
azd show

# அல்லது Azure போர்டலை திற
azd monitor

# குறிப்பிட்ட சேவையை சரிபார்க்கவும்
azd env get-values
# *_URL மாறிலிகளைத் தேடு
```
</details>

- டெவலப்பர்களின் **45%** AZD-ஐ AI வேலைப்பாடுகளுக்காக பயன்படுத்த விரும்புகிறார்கள்
- **முக்கிய சவால்கள்**: பன்மை சேவை டெப்பிளாய்மென்ட், அங்கீகார மேலாண்மை, உற்பத்தி தயார் நிலை  
- **அவசியமாக கோரப்பட்டவை**: AI-சார்ந்த டெம்ப்ளேட்கள், பிழைதிருத்த கையேடுகள், நல்ல நடைமுறைகள்

**எங்கள் சமூகத்தில் இணையுங்கள், இதனால்:**
- உங்கள் AZD + AI அனுபவங்களை பகிர்ந்து உதவி பெறுங்கள்
- புதிய AI டெம்ப்ளேட்களின் ஆரம்ப முன்னோட்டங்களை அணுகுங்கள்
- AI டெப்பிளாய்மென்ட் சிறந்த நடைமுறைகளுக்கு பங்களிக்கவும்
- எதிர்கால AI + AZD அம்ச மேம்பாட்டை பாதிக்கவும்
