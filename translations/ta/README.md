## 🆕 azd இல் இன்று என்ன புதிது

Azure Developer CLI பாரம்பரிய வலை பயன்பாடுகள் மற்றும் API களைத் தாண்டி வளர்ந்துள்ளது. இப்போது, azd என்பது Azure க்கு **எந்த** பயன்பாட்டையும் பரப்புவதற்கான ஒரே கருவியாக உள்ளது — AI-சக்தியுடைய பயன்பாடுகள் மற்றும் நுண்ணறிவு முகவர்கள் உட்பட.

இதன் உங்கள் பயன்பாட்டிற்கு என்ன அர்த்தம் என்று கீழே உள்ளது:

- **AI முகவர்கள் இப்போது முதன்மை azd பணியாகின்றன.** நீங்கள் আগে அறிந்திருந்ததைப் போலவே `azd init` → `azd up` பணிவழியைப் பயன்படுத்தி AI முகவர் திட்டங்களை துவக்கவும், தளமிறக்கவும் மற்றும் நிர்வகிக்கவும் முடியும்.
- **Microsoft Foundry ஒருங்கிணைப்பு** மாதிரி டிப்ளாய்மென்ட், முகவர் ஹோஸ்டிங், மற்றும் AI சேவை கட்டமைப்பை நேரடியாக azd டெம்ப்ளேட் экோசிஸ்டத்தில் கொண்டு வருகிறது.
- **முக்கிய பணிவழி மாற்றமல்.** நீங்கள் todo செயலி, ஒரு மைக்ரோசர்விஸ் அல்லது ஒரு பன்முகவர் AI தீர்வைபோல் எந்தவொரு செயலியையும் பரப்பினாலும், கட்டளைகள் ஒரே மாதிரியே இருக்கும்.

நீங்கள் முந்தையதாக azd பயன்படுத்தியிருந்தால், AI ஆதரவு ஒரு இயல்பு விரிவாகும் — தனி கருவோ அல்லது உயர் நிலை பாடமோ அல்ல. புதிதாக துவங்கினால், அனைத்திற்கும் வேலை செய்யும் ஒரே பணிவழியை நீங்கள் கற்றுக்கொள்வீர்கள்.

## 🚀 Azure Developer CLI (azd) என்றால் என்ன?

**Azure Developer CLI (azd)** என்பது Azure க்கு பயன்பாடுகளை எளிதாக பரப்புவதற்கு உதவும் டெவலப்பர் நட்பு கட்டளை வரி கருவி. பல Azure வளங்களை கையால் உருவாக்கி இணைக்கும் பதிலாக, நீங்கள் ஒரே கட்டளையால் முழு பயன்பாடுகளையும் பரப்ப முடியும்.

### `azd up` இன் மந்திரம்

```bash
# இந்த ஒரே கட்டளை அனைத்தையும் செய்கிறது:
# ✅ அனைத்து அஸ்யூர் வளங்களையும் உருவாக்குகிறது
# ✅ நெட்வொர்க்கிங் மற்றும் பாதுகாப்பை நிலைநிறுத்துகிறது
# ✅ உங்கள் பயன்பாட்டு குறியீட்டைக் கட்டமைக்கிறது
# ✅ அஸ்யூருக்கு பின்வரிசைப்படுத்துகிறது
# ✅ உங்களுக்கு செயல்படும் URL ஐ வழங்குகிறது
azd up
```

**அதே போல்!** Azure போர்டல் திறந்து ஒரு சில கிளிக்குகள், ARM டெம்ப்ளேட்களை முன் முதலாவது கற்றுக் கொள்ள வேண்டியவையாக்கவும் அல்லது கையேடு அமைப்புகள் தேவையில்லை — வென் எளிய செயலிகள் Azure இல் வேலை செய்கின்றன.

## ❓ Azure Developer CLI மற்றும் Azure CLI: என்ன வேறுபாடு?

இதுவே புதியவர்களால் மிகவும் அடிக்கடி கேட்கப்படும் கேள்வி. சுலபமான பதில் இதோ:

| அம்சம் | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **நோக்கம்** | தனியார் Azure வளங்களை நிர்வகிக்க | முழு பயன்பாடுகளை பரப்ப  |
| **மனப்பான்மை** | அமைப்பு-மையமானது | பயன்பாடு-மையமானது |
| **உதாரணம்** | `az webapp create --name myapp...` | `azd up` |
| **கற்றல் வளைவு** | Azure சேவைகளை அறிவதத் தேவையாய் இருக்கும் | உங்கள் செயலியை அறிவதுதான் போதும் |
| **எதற்கு சிறந்தது** | DevOps, அமைப்பு | உருவாக்குனர்கள், மாதிரி உருவாக்கல் |

### எளிய ஒப்புமை

- **Azure CLI** என்பது ஒரு வீட்டை கட்டுவதற்கான அனைத்து கருவிகளையும் வைத்திருப்பது போல — ஹாம்மர்கள், அல்வைகள், நயில்கள். நீங்கள் எந்த இதயத்தையும் கட்டலாம், ஆனால் கட்டுமானத்தை அரசியலாக அறிந்திருக்க வேண்டும்.
- **Azure Developer CLI** என்பது ஒரு கட்டுமானக் கான்ட்ராக்டரை நியமித்து கொள்ளுவது போல — நீங்கள் என்ன வேண்டும் என்பதை விளக்கினால், அவர்கள் கட்டுப்பாட்டை கையாளுவர்.

### எப்போது எதை பயன்படுத்துவது

| சூழ்நிலை | இதை பயன்படுத்தவும் |
|----------|----------|
| "நான் என் வலை பயன்பாட்டை விரைவாக பரப்ப விரும்புகிறேன்" | `azd up` |
| "நான் ஒரு சேமிப்பு கணக்கை மட்டும் உருவாக்க வேண்டும்" | `az storage account create` |
| "நான் முழு AI செயலியை உருவாக்குகிறேன்" | `azd init --template azure-search-openai-demo` |
| "நான் ஒரு குறிப்பிட்ட Azure வளத்தை பிழையுணர வேண்டும்" | `az resource show` |
| "நான் நிமிடங்களில் உற்பத்திக்குத் தயாரான டெப்ளாய்மெண்ட் விரும்புகிறேன்" | `azd up --environment production` |

### அவை ஒன்றாக வேலை செய்கின்றன!

AZD உள்ளமைவில் Azure CLI ஐ பயன்படுத்துகிறது. நீங்கள் இரண்டையும் பயன்படுத்தலாம்:
```bash
# உங்கள் செயலியை AZD உடன்.deploy(நிறுவவும்) செய்யவும்
azd up

# அதன்பிறகு Azure CLI உடன் குறிப்பிட்ட வளங்களை நுட்பமாக சரிசெய்க
az webapp config set --name myapp --always-on true
```

### படி 3: உங்கள் முதல் பயன்பாட்டை பரப்பவும்

```bash
# ஒரு வார்ப்புருவிலிருந்து துவக்கம் செய்யவும்
azd init --template todo-nodejs-mongo

# Azure இல் நிலைநிறுத்தவும் (எல்லாவற்றையும் உருவாக்குகிறது!)
azd up
```

**🎉 அதுதான்!** உங்கள் செயலி இப்போது Azure இல் உயிருடன் உள்ளது.

### சுத்தம் செய்தல் (மறக்காதீர்கள்!)

```bash
# Remove all resources when done experimenting
azd down --force --purge
```

#### நடைமுறை பயிற்சிகள்
```bash
# உங்கள் முதல் செயற்கை நுண்ணறிவு பயன்பாட்டை வெளியிடுங்கள்

# மேலும் செயற்கை நுண்ணறிவு வார்ப்புருக்களை முயற்சி செய்யுங்கள்
azd init --template openai-chat-app-quickstart

```

**💡 அத்தியாய முடிவு**: RAG திறன்களுடன் ஒரு AI-சக்தியுடைய உரையாடல் பயன்பாட்டை வெளியிட்டு கட்டமைக்கலாம்

**✅ வெற்றியின் சரிபார்த்தல்:**
```bash
# அத்தியாயம் 2 முடிந்த பிறகு, நீங்கள் இதைச் செய்யக்கூடியதாக இருப்பீர்கள்:
azd init --template azure-search-openai-demo
azd up
# AI அரட்டை இடைமுகத்தை சோதிக்கவும்
# கேள்விகள் கேட்டு ஆதாரங்களுடன் கூடிய AI சார்ந்த பதில들을ப் பெறவும்
# தேடல் ஒருங்கிணைப்பு சரியாக செயல்படுகிறதா என்பதைச் சரிபார்க்கவும்
azd monitor  # Application Insights டெலிமெட்ரியை காட்டுகிறதா என்பதைச் சரிபார்க்கவும்
azd down --force --purge
```

**📊 கால முதலீடு:** 1-2 மணி  
**📈 பயிற்சி முடிந்த பிறகு திறன்:** தயாரிப்பு-தயாரான AI பயன்பாடுகளை வெளியிட்டு கட்டமைக்க முடியும்  
**💰 செலவு அறிதல்:** $80-150/மாதம் வளர்ச்சி செலவுகள், $300-3500/மாதம் தயாரிப்பு செலவுகள் என்பவற்றைப் புரிந்துகொள்ளுங்கள்

#### 💰 AI வெளியீடுகளுக்கான செலவு கருத்துகள்

**வளர்ச்சி சூழல் (கணிக்கப்பட்ட $80-150/மாதம்):**
- Microsoft Foundry Models (Pay-as-you-go): $0-50/மாதம் (டோக்கன் பயன்பாட்டின் அடிப்படையில்)
- AI Search (அடிப்படை அடுக்கு): $75/மாதம்
- Container Apps (நுகர்வு): $0-20/மாதம்
- Storage (Standard): $1-5/மாதம்

**தயாரிப்பு சூழல் (கணிக்கப்பட்ட $300-3,500+/மாதம்):**
- Microsoft Foundry Models (இடமாற்றத் திறன் தொடர்பான PTU): $3,000+/மாதம் அல்லது பெரிய பருமனுடன் Pay-as-go
- AI Search (Standard அடுக்கு): $250/மாதம்
- Container Apps (ஒதுக்கீடு): $50-100/மாதம்
- Application Insights: $5-50/மாதம்
- Storage (Premium): $10-50/மாதம்

**💡 செலவு குறைப்புத் குறிப்புகள்:**
- கற்றலுக்காக Microsoft Foundry Models இன் **Free Tier** ஐ பயன்படுதுங்கள் (Azure OpenAI 50,000 tokens/மாதம் இணைக்கப்பட்டுள்ளது)
- செயல்பாடு இல்லாத போது வளங்களை நீக்க `azd down` கமாண்டை இயக்கவும்
- பயன்படுத்தல் அடிப்படையிலான பில்லிங்கில் இருந்து துவங்கி, தயாரிப்பிற்காக மட்டும் PTU க்கு மேம்படுத்தவும்
- வெளியீடுக்கு முன் செலவை மதிப்பிட `azd provision --preview` ஐப் பயன்படுத்தவும்
- தானாக அளவை நீட்டிக்கும் (auto-scaling) அமுப்பை செயல்படுத்து: உண்மையான பயன்பாட்டுக்கே மட்டுமே செலவ் செலுத்துங்கள்

**செலவு கண்காணிப்பு:**
```bash
# மதிப்பிடப்பட்ட மாதச் செலவுகளைச் சரிபார்க்கவும்
azd provision --preview

# Azure போர்டலில் உண்மையான செலவுகளை கண்காணிக்கவும்
az consumption budget list --resource-group <your-rg>
```

- **ஒன்றிணைந்த பணிநிரல்கள்** - பயன்பாடுகளை இடமாற்றம் செய்ய, பதிப்பேற்று மற்றும் கண்காணிக்க தாமாகProvision செய்யுமாறு ஒருங்கிணைக்கின்றன
- **வिकசிப்பாளர்-நட்பானது** - விகசிப்பாளர் உற்பத்தித்திறன் மற்றும் அனுபவத்திற்காக 최적화 செய்யப்பட்டுள்ளது

### **AZD + Microsoft Foundry: AI பதிப்புகளுக்கு சிறந்தது**

**ஏன் AZD ஆஐ தீர்வுகளுக்காக?** AZD AI டெவலபர்களால் எதிர்கொள்ளப்படும் முக்கிய சவால்களை தீர்க்கிறது:

- **AI-தயார் வார்ப்புருக்கள்** - Microsoft Foundry மாடல்கள், Cognitive Services மற்றும் ML பணிகளுக்கான முன்னிறுத்தப்பட்ட வார்ப்புருக்கள்
- **பாதுகாப்பான AI பதிப்புகள்** - AI சேவைகள், API விசைகள் மற்றும் மாடல் இடைமுகங்களுக்கு உள்ளமைக்கப்பட்ட பாதுகாப்பு நடைமுறைகள்  
- **உற்பத்தி AI மாதிரிகள்** - அளவீடு செய்யக்கூடிய, செலவு-திறன் மிக்க AI பயன்பாடுகளுக்கான சிறந்த நடைமுறைகள்
- **முதல் முதல் இறுதி வரை AI பணிநிரல்கள்** - மாடல் மேம்பாட்டிலிருந்து உற்பத்தி பதிப்புவரை சரியான கண்காணிப்போடும்
- **செலவு ஒளிவுத்தல்** - AI பணிகளுக்கான ஸ்மார্ট வள ஒதுக்கீடு மற்றும் அளவீட்டு 전략ங்கள்

```bash
# வேறு Azure பிராந்தியத்தை முயற்சி செய்யவும்

azd up

# அல்லது மேம்பாட்டில் சிறிய SKUகளை பயன்படுத்தவும்
```

</details>

<details>
<summary><strong>❌ "azd up" நடுவில் தோல்வி அடைகிறது</strong></summary>

```bash
# விருப்பம் 1: சுத்தம் செய்து மீண்டும் முயற்சி செய்
azd down --force --purge
azd up

# விருப்பம் 2: கட்டமைப்பை மட்டும் சரிசெய்
azd provision

# விருப்பம் 3: விரிவான நிலையை சரிபார்க்கவும்
azd show

# விருப்பம் 4: Azure Monitor இல் உள்ள பதிவுகளைப் சரிபார்க்கவும்
azd monitor --logs
```
</details>

<details>
<summary><strong>❌ "அங்கீகரிப்பு தோல்வியடைந்தது" அல்லது "டோக்கன் காலாவதியாகியுள்ளது"</strong></summary>

<details>
<summary><strong>❌ "வளம் ஏற்கனவே உள்ளது" அல்லது பெயர் மோதல்கள்</strong></summary>

```bash
# AZD தனித்துவமான பெயர்களை உருவாக்குகிறது, ஆனால் மோதல் ஏற்பட்டால்:
azd down --force --purge

# அப்போது புதிய சூழ்நிலையுடன் மீண்டும் முயற்சி செய்யவும்
azd env new dev-v2
azd up
```
</details>

<details>
<summary><strong>❌ டெம்ப்ளேட் வழங்கல் அதிக நேரம் எடுக்கும்</strong></summary>

**சாதாரண காத்திருப்பு நேரங்கள்:**
- எளிய இணைய பயன்பாடு: 5-10 நிமிடம்
- தரவுத்தளத்துடன் பயன்பாடு: 10-15 நிமிடம்
- AI பயன்பாடுகள்: 15-25 நிமிடம் (OpenAI வழங்கல் மெதுவாக இருக்கலாம்)

```bash
# முன்னேற்றத்தை சரிபார்க்கவும்
azd show

# 30 நிமிடத்துக்கு மேல் இடம்பிடித்தால், Azure Portal-ஐ சரிபார்க்கவும்:
azd monitor

```
