## 🆕 azd இல் இன்று என்ன புதிது

> 📌 இந்த படிப்பு **`azd 1.27.1`** (ஜூலை 2026) விருப்பப்படுத்தப்பட்டுள்ளது. உங்கள் கட்டுமானத்தை பார்க்க `azd version` ஓப்பனை இயக்கவும், புதியதுபற்ற `azd upgrade` ஐ இயக்கவும்.

Azure Developer CLI பாரம்பரிய வலை பயன்பாடுகள் மற்றும் API களை கடந்த வளர்ச்சி பெற்றுள்ளது. இன்று, azd என்பது Azureக்கு **எந்தவொரு** பயன்பாட்டையும் பயணப்படுத்தும் ஒரே கருவி ஆகி விட்டது - அதில் AI இயக்க பயன்பாடுகள் மற்றும் அறிவுசார் முகவர்கள் அடங்கும்.

இதோ உங்களுக்கு இதனால் என்ன உள்ளது:

- **AI முகவர்கள் இNOW முதன்மை azd வேலைகள் ஆகிவிட்டன.** நீங்கள் `azd init` → `azd up` என்ற ஒரே வேலைப்பாட்டு முறையை பயன்படுத்தி AI முகவர் திட்டங்களை தொடங்கி, நேர்த்தும், மேலாண்மை செய்யலாம்.
- **CLI மூலம் ஒரு முழு முகவர் வாழ்க்கைசுழற்சி.** `azure.ai.agents` விரிவாக்கம் முழு பயணத்தை உள்ளடக்கியது—`azd ai agent init` மூலம் கோவையை உருவாக்கவும், `azd ai agent invoke` மூலம் சோதனை செய்யவும் (பதில்-நேரம் வெளியீடு உடன்), `azd ai agent eval generate` மற்றும் `azd ai agent optimize` மூலம் தரத்தை அளவிடவும் மேம்படுத்தவும், மேலும் `azd ai agent delete` மூலம் சுத்தம் செய்யவும்.
- **மேலும் AI கட்டுமான துணுக்குகள்.** புதிய முன்னோட்ட விரிவாக்கங்கள்—`azure.ai.skills` மற்றும் `azure.ai.connections`—azd மூலம் மறுசுழற்சி சாத்தியமான முகவர் திறன்கள் மற்றும் Foundry இணைப்புகளை மேலாண்மை செய்ய அனுமதிக்கின்றன.
- **Microsoft Foundry ஒருங்கிணைப்பு** மாதிரி பயணப்படுத்தல், முகவர் விருந்தாளி மற்றும் AI சேவை அமைப்பு அனைத்தும் நேரடியாக azd மாதிரிச் சுற்றுப்புறத்தில் இணைக்கப்பட்டுள்ளது.
- **நாளாந்திர அடிப்படைகள் மென்மையாக்கம்.** சமீபத்திய வெளியீடுகள் `azd init` ஐ மீண்டும் இயக்க பாதுகாப்பானவாறு செய்தது, `azd auth login` பழைய டோக்கன்களை தானாக தெளிவாக்குகிறது, மேலும் நட்பு `azd tool` முதன்முதலாவது அமைப்பு தூண்டலை அனுபவமாக்கியது.
- **மூல வேலைப்பாட்டு மாற்றமில்லை.** நீங்கள் ஒரு டுடூ பயன்பாடை, ஒரு மைக்ரோசேவை, அல்லது பல முகவர் AI தீர்வை வெளியிடினாலும் கட்டளைகள் ஒரே மாதிரியே இருக்கும்.

> **Aspire பயனர்களுக்கு குறிப்புகள்:** Microsoft இப்போது பொருளை **Aspire** (முன்னர் ".NET Aspire") என அழைக்கின்றது. azd இன் Aspire ஆதரவு மாற்றமில்லை—பெயர் மட்டுமே புதுப்பிக்கப்பட்டது.

நீங்கள் முன்பு azd ஐ பயன்படுத்தினால், AI ஆதரவு ஒரு இயல்பான நீட்சியாகும்—ஒரு தனித்த கருவி அல்லது முன்னேறிய பாதை அல்ல. நீங்கள் புதிதாக ஆரம்பிப்பவராவீர்கள் என்றால், அனைத்துக்கும் ஒரே வேலைப்பாட்டு முறையை கற்றுக்கொள்ளுவீர்கள்.

## 🚀 Azure Developer CLI (azd) என்றால் என்ன?

**Azure Developer CLI (azd)** என்பது Azureக்கு பயன்பாடுகளை வெளியிட எளிதாக்கும் ஒரு டெவலப்பர் நேசமான கட்டளை வரி கருவி ஆகும். பல Azure வளங்களை கைமுறையாக உருவாக்கி இணைப்பதற்கு பதிலாக, ஒருமே கட்டளையுடன் முழு பயன்பாடுகளை வெளியிட முடியும்.

### `azd up` இன் மகிமை

```bash
# இந்த ஒரே கட்டளை எல்லாவற்றையும் செய்கிறது:
# ✅ அனைத்து Azure வளங்களையும் உருவாக்குகிறது
# ✅ நெட்வொர்கிங் மற்றும் பாதுகாப்பை உள்ளமைக்கிறது
# ✅ உங்கள் செயலி குறியீட்டை கட்டமைக்கிறது
# ✅ Azure இற்கு பதிக்கிறது
# ✅ உங்களுக்கு ஒரு செயலில் இருக்கும் URL ஐ வழங்குகிறது
azd up
```

**இதுதான்!** Azure போர்டலைச் சுட்டி கிளிக் செய்ய வேண்டியது இல்லை, சிக்கலான ARM படிவங்களை முதலில் கற்க வேண்டியதுமில்லை, கைமுறையான கட்டமைப்பும் தேவையில்லை - Azureல் வேலை செய்யும் செயலிகள் மட்டுமே.

## ❓ Azure Developer CLI மற்றும் Azure CLI: வேறுபாடு என்ன?

இது தொடக்கத்திற்க்கான பொதுவான கேள்வி. சுலபமான விடை:

| அம்சம் | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **பொருள்** | தனி Azure வளங்களை நிர்வகிக்க | முழு பயன்பாடுகளை வெளியிட |
| **மனப்பான்மை** | கட்டமைப்பு மையமாக | பயன்பாடு மையமாக |
| **உதாரணம்** | `az webapp create --name myapp...` | `azd up` |
| **கற்றல் வளைவு** | Azure சேவைகளை தெரிந்துகொள்ள வேண்டும் | உங்கள் பயன்பாட்டை தெரிந்துகொள் |
| **சிறந்தது** | DevOps, கட்டமைப்பு | டெவலப்பர்கள், மாதிரி வடிவமைப்பு |

### எளிய ஒப்புமை

- **Azure CLI** என்பது வீடு கட்டுவதற்கான அனைத்து கருவிகளும் இருக்கின்றன - கடைகள், பஞ்சு, புத்தகங்கள். நீங்கள் எதையும் கட்டலாம், ஆனால் கட்டுமானத்தை தெரிந்திருக்க வேண்டும்.
- **Azure Developer CLI** என்பது ஒரு ஒப்பந்த தீர்க்காட்சியாளர் பணியமர்த்துவது போல - நீங்கள் வேண்டுமென்றதை விவரிக்கிறீர்கள், அவர்கள் கட்டுமானத்தை கவனிக்கின்றனர்.

### எப்பொழுது எதை பயன்படுத்துவது

| நிலைமை | இதை பயன்படுத்தவும் |
|----------|----------|
| "என் வலை பயன்பாட்டை விரைவில் வெளியிட விரும்புகிறேன்" | `azd up` |
| "எனக்கு ஒரு சேமிப்பு கணக்கு மட்டும் உருவாக்கவேண்டும்" | `az storage account create` |
| "நான் முழு AI பயன்பாட்டை கட்டுகிறேன்" | `azd init --template azure-search-openai-demo` |
| "நான் ஒரு குறிப்பிட்ட Azure வளத்தை தப்புதலை செய்யவேண்டும்" | `az resource show` |
| "நான் நிமிடங்களில் தயாரிப்பு வாசக வெளியீடு விரும்புகிறேன்" | `azd up --environment production` |

### அவை ஒன்றாக வேலை செய்கின்றன!

AZD Azure CLI ஐ அடிப்படையில் பயன்படுத்துகிறது. நீங்கள் இரண்டையும் பயன்படுத்தலாம்:
```bash
# உங்கள் பயன்பாட்டை AZD உடன் வெளியிடவும்
azd up

# பின்னர் Azure CLI உடன் குறிப்பிட்ட வளங்களை சரிசெய்யவும்
az webapp config set --name myapp --always-on true
```

## 🎯 3 படிகளின் தொடக்கம்

துவங்குவதற்கு முன், நீங்கள் வெளியிட விரும்பும் மாதிரிக்கான உங்கள் கணினி தயார் என்பதை உறுதிப்படுத்துக:

**Windows:**
```powershell
.\validate-setup.ps1
```

**macOS / Linux:**
```bash
bash ./validate-setup.sh
```

அவசியமான சோதனை தோல்வி அடைந்தால், முதலில் அதை சரி செய்த பிறகு வேகமான துவக்கத்துடன் தொடரவும்.

### படி 1: AZD ஐ நிறுவுக (2 நிமிடங்கள்)

**Windows:**
```powershell
winget install microsoft.azd
```

**macOS:**
```bash
brew tap azure/azd && brew install azd
```

**Linux:**
```bash
curl -fsSL https://aka.ms/install-azd.sh | bash
```

### படி 2: AZD க்கான அங்கீகாரம்

```bash
# இந்த பாடத்தில் நேரடியாக Azure CLI கட்டளைகளைப் பயன்படுத்த திட்டமிட்டால் விருப்பமானவை
az login

# AZD பணிநெறிகளுக்கு தேவையானது
azd auth login
```

நீங்கள் எந்ததை தேவை என்பதை உறுதியாக அறியாவிட்டால், [Installation & Setup](docs/chapter-01-foundation/installation.md#authentication-setup) இல் முழு அமைப்பு ஓட்டத்தை பின்பற்றவும்.

### படி 3: உங்கள் முதல் செயலியை வெளியிடுதல்

```bash
# ஒரு மாதிரியில் இருந்து துவக்கம் செய்
azd init --template todo-nodejs-mongo

# அசுருக்கு பிரதி நிலை செய் (எல்லாவற்றையும் உருவாக்கும்!)
azd up
```

**🎉 இதுதான்!** உங்கள் செயலி இப்போது Azure இல் செயல்படுகிறது.

### சுத்தம் செய்தல் (மறக்காதீர்கள்!)

```bash
# பரிசோதனை முடிந்ததும் 모든 வளங்களை அகற்றவும்
azd down --force --purge
```

## பாட பாட நோக்கு

கட்டமைக்கப்பட்ட அத்தியாயங்களின் மூலம் Azure Developer CLI (azd) ஐ இயக்குச்செயல்படுத்தவும். **Microsoft Foundry ஒருங்கிணைப்புடன் AI பயன்பாட்டு வெளியீட்டில் சிறப்பு கவனம்.**


### இக்கோர்ஸ் நவீன மேம்பாட்டாளர்களுக்கு ஏன் அவசியம்

Microsoft Foundry Discord சமூக அறிவுரைகளின் அடிப்படையில், **45% மேம்பாட்டாளர்கள் AI பணிகளுக்காக AZD பயன்படுத்த விரும்புகிறார்கள்** ஆனால் பின்வரும் சவால்களை சந்திக்கிறார்கள்:
- சிக்கலான பன்முக சேவை AI கட்டமைப்புகள்
- தயாரிப்பு AI வழங்கல் சிறந்த நடைமுறைகள்  
- Azure AI சேவை ஒருங்கிணைப்பு மற்றும் கட்டமைப்பு
- AI பணிகளுக்கான செலவு மேம்படுத்தல்
- AI-சிறப்பான வழங்கல் பிரச்சினைகளை தீர்க்கும் நடைமுறைகள்

### கற்றல் நோக்கங்கள்

இந்த கட்டமைக்கப்பட்ட கோர்ஸை முடிப்பதன் மூலம், நீங்கள்:
- **AZD அடிப்படைகளை நன்கு கற்றுக்கொள்க**: முக்கியக் கருத்துகள், நிறுவல் மற்றும் கட்டமைப்பு
- **AI பயன்பாடுகளை வழங்குக**: Microsoft Foundry சேவைகளுடன் AZD பயன்படுத்துக
- **கட்டமைப்பை குறியீடு போல அமல்படுத்துக**: Bicep வார்ப்புருக்கள் மூலம் Azure வளங்களை நிர்வகி
- **வழங்கல்களை பிழைத்திருத்துக**: பொதுவான பிரச்சினைகளை தீர்த்து பிழைகளை கண்டறிதல்
- **தயாரிப்புக்காக மேம்படுத்துக**: பாதுகாப்பு, பரிமாணம், கண்காணிப்பு, மற்றும் செலவு மேலாண்மை
- **பன்முக முகவர் தீர்வுகளை உருவாக்குக**: சிக்கலான AI கட்டமைப்புகளை வழங்குக

## துவங்குவதற்கு முன்: கணக்குகள், அணுகல் மற்றும் கருத்துக்கள்

அத்தியாயம் 1 தொடங்குவதற்கு முன்னதாக, பின்வரும் விஷயங்கள் தயார் நிலையில் இருக்க வேண்டும். இந்த வழிகாட்டியிலுள்ள நிறுவல் படிநிலைகள் இத்தன்மை நிரூபிக்கப்பட்டுள்ளது.

- **ஒரு Azure சந்தா**: வேலைக்கு அல்லது உங்கள் சொந்தக் கணக்கிலிருந்து முந்தைய சந்தாவை பயன்படுத்தலாம் அல்லது [இலவச சோதனை](https://aka.ms/azurefreetrial) ஒன்றை உருவாக்கி துவங்கலாம்.
- **Azure வளங்களை உருவாக்க அனுமதி**: பல பயிற்சிகளுக்கான குறைந்தது **கொடுத்து-ஆக்குனர்** அணுகல் இலக்கு சந்தா அல்லது வள குழுவில் வேண்டும். சில அத்தியாயங்கள் வள குழுக்களை, முகாமைப் பெற்ற அடையாளங்களை, மற்றும் RBAC நியமனங்களை உருவாக்க முடியும் என்று கருதலாம்.
- [**ஒரு GitHub கணக்கு**](https://github.com): இந்த இடுகையை பிரித்தல், உங்கள் சொந்த மாற்றங்களை கண்காணித்தல் மற்றும் GitHub Codespaces பயன்படுத்த லேப்டாப் பயன்பாடு பயனளிக்கும்.
- **வார்ப்புரு இயங்குதள முன்நிலைத் தேவைகள்**: சில வார்ப்புருக்கள் Node.js, Python, Java அல்லது Docker போன்ற உள்ளூர் கருவிகள் தேவைப்படலாம். துவங்குவதற்கு முன் அமைப்பு பரிசோதகரை இயக்கி இழந்த கருவிகளை முன்னரே கண்டறியவும்.
- **அடிப்படை டெர்மினல் பரிச்சயம்**: நீங்கள் வல்லுநராய் இருக்க தேவையில்லை, ஆனால் `git clone`, `azd auth login`, மற்றும் `azd up` போன்ற கட்டளைகளை இயக்குவதில் சௌகரியமாக இருக்க வேண்டும்.

> **ஒரு நிறுவன சந்தாவில் பணியாற்றுகிறீர்களா?**
> உங்கள் Azure சுற்றுச்சூழல் நிர்வாகி மூலம் கையாளப்பட்டால், நீங்கள் பயன்படுத்தும் சந்தா அல்லது வள குழுவில் வளங்களை வழங்க முடியும் என்பதை முன்கூட்டியே உறுதிசெய்யுங்கள். இல்லையெனில் துவங்குவதற்கு முன் sandbox சந்தா அல்லது கொடுத்து-ஆக்குனர் அணுகலைக் கேளுங்கள்.

> **Azure இல் புதியவரா?**
> முதலில் உங்கள் சொந்த Azure சோதனை அல்லது கட்டணம் அடிப்படையிலான சந்தாவுடன் https://aka.ms/azurefreetrial துவங்கி பயிற்சிகளை முற்றிலும் செய்யுங்கள், இடைக்கால உள்நாட்டு ஒப்புதல்களை காத்திருக்க வேண்டாம்.

#### 💰 AI வழங்கலுக்கான செலவு கணக்கீடுகள்

**மேம்பாட்டு சுற்றுச்சூழல் (அடிப்படையில் $80-150/மாதம்):**

- கன்டெயினர் பயன்பாடுகள் (நுகர்வு): $0-20/மாதம்
- சேமிப்பு (நிலையான): $1-5/மாதம்

- கன்டெயினர் பயன்பாடுகள் (தனித்த சேவை): $50-100/மாதம்
- அப்ளிகேஷன் இன்சைட்ஸ்: $5-50/மாதம்
- சேமிப்பு (பிரீமியம்): $10-50/மாதம்

**💡 செலவு மேம்படுத்தல் குறிப்புகள்:**
- கற்றலுக்காக **இலவச நிலை** Microsoft Foundry மாதிரிகளை பயன்படுத்தவும் (Azure OpenAI 50,000 டோக்கன்கள்/மாதம் அடங்கும்)
- செயல்களில் இல்லாத போது வளங்களை விட `azd down` இயக்கவும்

  - ARM வடிவத்திலிருந்து ஒரே கிளிக் நிறுவல்

## 🔧 விரைவு பிரச்சினை தீர்க்கும் வழிகாட்டி

**ஆரம்பகாலர்கள் சந்திக்கும் பொதுவான பிரச்சினைகள் மற்றும் உடனடி தீர்வுகள்:**

<details>
<summary><strong>❌ "azd: கட்டளை காணப்படவில்லை"</strong></summary>

```bash
# முதலில் AZD ஐ நிறுவவும்
# Windows (PowerShell):
winget install microsoft.azd

# macOS:
brew tap azure/azd && brew install azd

# Linux:
curl -fsSL https://aka.ms/install-azd.sh | bash

# நிறுவலை சரிபார்க்கவும்
azd version
```
</details>

<details>
<summary><strong>❌ "சந்தா காணப்படவில்லை" அல்லது "சந்தா அமைக்கப்படவில்லை"</strong></summary>

```bash
# கிடைக்கக்கூடிய சந்தாக்களை பட்டியலிடுக
az account list --output table

# அசாதாரண சந்தாவை அமைக்கவும்
az account set --subscription "<subscription-id-or-name>"

# AZD சூழலுக்கான அமைப்பு
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# சரிபார்க்கவும்
az account show
```
</details>

<details>
<summary><strong>❌ "InsufficientQuota" அல்லது "Quota மூடியது"</strong></summary>

```bash
# வேறுபட்ட அசூர் பிராந்தியத்தை முயற்சி செய்க
azd env set AZURE_LOCATION "westus2"
azd up

# அல்லது அபிவிருத்தியில் சிறிய SKUs பயன்படுத்தவும்
# infra/main.parameters.json ஐ திருத்துக:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>

<details>
<summary><strong>❌ "azd up" பாதி வழியிலேயே தோல்வியடைந்தது</strong></summary>

```bash
# விருப்பம் 1: சுத்தம் செய்து மீண்டும் முயற்சி செய்க
azd down --force --purge
azd up

# விருப்பம் 2: கட்டமைப்பை மட்டும் சரிசெய்யவும்
azd provision

# விருப்பம் 3: விரிவான நிலையை சரிபார்க்கவும்
azd show

# விருப்பம் 4: அசூர் மானிட்டரில் பதிவு பதிவுகளை பரிசோதிக்கவும்
azd monitor --logs
```
</details>

<details>
<summary><strong>❌ "அங்கீகாரம் தோல்வியடைந்தது" அல்லது "டோக்கன் காலாவதி"</strong></summary>

```bash
# AZDக்கு மீண்டும் அங்கீகாரம் செய்யவும்
azd auth logout
azd auth login

# விருப்பமானது: நீங்கள் az கட்டளைகளை இயக்குகிறீர்கள் என்றால் Azure CLIயையும் புதுப்பிக்கவும்
az logout
az login

# அங்கீகாரம் சரிபார்க்கவும்
az account show
```
</details>

<details>
<summary><strong>❌ "வளமை இனிப்பு ஏற்கனவே உண்டு" அல்லது பெயரிடல் மோதல்கள்</strong></summary>

```bash
# AZD தனித்துவமான பெயர்களை உருவாக்குகிறது, ஆனால் மோதல் ஏற்பட்டால்:
azd down --force --purge

# பின்னர் புதிய சுற்றுச்சூழலுடன் மீண்டும் முயற்சி செய்யவும்
azd env new dev-v2
azd up
```
</details>

<details>
<summary><strong>❌ மாதிரி நிறுவல் மிக நேரம் எடுத்துக்கொள்கிறது</strong></summary>

**பொதுவான காத்திருப்பு நேரங்கள்:**
- எளிய வலை செயலி: 5-10 நிமிடங்கள்
- தரவுத்தளத்தை உடைய செயலி: 10-15 நிமிடங்கள்
- AI பயன்பாடுகள்: 15-25 நிமிடங்கள் (OpenAI வழங்கல் மெதுவாக உள்ளது)

```bash
# முன்னேற்றத்தை சரிபார்
azd show

# 30 நிமிடங்களுக்கு மேல் தடுமாறினால், Azure போர்டலை சரிபார்:
azd monitor --overview
# தோல்வியடைந்த 배치களைப் பார்க்கவும்
```
</details>

<details>
<summary><strong>❌ "அனுமதி மறுக்கப்பட்டது" அல்லது "நினைவில் இருந்து தடைதீரப்பட்டது"</strong></summary>

```bash
# உங்கள் Azureப் பணியாளர் நிலையை சரிபார்க்கவும்
az role assignment list --assignee $(az account show --query user.name -o tsv)

# குறைந்தது "பங்களிப்பாளர்" பணியாளர் நிலை தேவை
# உங்கள் Azure நிர்வாகியை கேட்டுக்கொள்ளவும்:
# - பங்களிப்பாளர் (வளங்களுக்கான)
# - பயனர் அணுகல் நிர்வாகி (பணியாளர் பணிகள் வழங்க)
```
</details>

<details>
<summary><strong>❌ நிறுவப்பட்ட செயலியின் URL காணவில்லை</strong></summary>

```bash
# அனைத்து சேவை முனைமுகங்களையும் காட்டு
azd show

# அல்லது அசுரே போர்டலை திறக்கவும்
azd monitor

# குறிப்பிட்ட சேவையை சரி பார்க்கவும்

# *_URL மாறிலிகளை தேடவும்
```
</details>

2. **அறிவுச் சரிபார்ப்பு**: உங்கள் அத்தியாயத்தின் FAQ பிரிவை மறுபரிசீலனை செய்யவும்

### பாடநெறி முடிவின் நன்மைகள்

- **தயாரிப்பு அனுபவம்**: Azureக்கு உண்மையான AI செயலிகளை நிறுவியுள்ளீர்கள்
- **தொழில்முறை திறன்கள்**: தொழிற்சாலை-தயாரிப்பு நிறுவன அமைப்பு திறன்  
- **சமுதாய அங்கீகாரம்**: Azure டெவலப்பர் சமுதாயத்தின் செயலில் உறுப்பினர்
- **தொழில் முன்னேற்றம்**: தேவையுள்ள AZD மற்றும் AI நிறுவல் நுட்பங்கள்
