# AZD For Beginners में योगदान

Thank you for your interest in contributing to the AZD For Beginners course! This document provides guidelines and information for contributors.

## योगदान कैसे करें

### समस्याओं की रिपोर्टिंग

If you find bugs, errors, or have suggestions for improvements:

1. **मौजूदा इश्यूज़ जांचें**: Search [मौजूदा इश्यूज़](https://github.com/microsoft/azd-for-beginners/issues) to avoid duplicates
2. **नया इश्यू बनाएं**: Use the appropriate issue template
3. **विवरण प्रदान करें**: Include steps to reproduce, expected behavior, and screenshots if applicable

### परिवर्तन प्रस्तावित करना

1. **रिपॉज़िटरी फोर्क करें**: Create your own fork of the project
2. **एक ब्रांच बनाएं**: Create a feature branch for your changes
3. **परिवर्तन करें**: Follow our coding and documentation standards
4. **अपने परिवर्तन परीक्षण करें**: Verify all links work and content renders correctly
5. **पुल अनुरोध प्रस्तुत करें**: Provide a clear description of your changes

### योगदान के प्रकार

We welcome the following types of contributions:

#### सामग्री में सुधार
- टाइपो, व्याकरण, और फ़ॉर्मेटिंग समस्याओं को ठीक करें
- मौजूदा दस्तावेज़ीकरण की स्पष्टता सुधारें
- पुरानी जानकारी अपडेट करें
- गायब स्पष्टीकरण या संदर्भ जोड़ें

#### नए उदाहरण
- वास्तविक दुनिया परिनियोजन परिदृश्यों को जोड़ें
- सामान्य उपयोग मामलों के लिए नए AZD टेम्पलेट बनाएं
- अतिरिक्त कोड नमूने और अभ्यास प्रदान करें

#### अनुवाद सहायता
- मौजूदा अनुवादों को बनाए रखने में मदद करें
- अनुवाद गुणवत्ता सुधारें
- नई सामग्री के लिए अनुवाद जोड़ें

#### बग रिपोर्ट
- टूटी हुई लिंक्स या गलत कोड उदाहरण रिपोर्ट करें
- दस्तावेज़ीकरण में असंगतियों की पहचान करें
- पहुँच (accessibility) समस्याओं को फ्लैग करें

## योगदान दिशानिर्देश

### दस्तावेज़ीकरण मानक

- स्पष्ट, शुरुआती के अनुकूल भाषा का उपयोग करें
- सीखने के उद्देश्य और परिणाम शामिल करें
- अपेक्षित आउटपुट के साथ चरण-दर-चरण निर्देश प्रदान करें
- संबंधित सामग्री के लिए नेविगेशनल लिंक जोड़ें
- सुसंगत फ़ॉर्मेटिंग और मार्कडाउन शैली का उपयोग करें

### कोड उदाहरण

- सुनिश्चित करें कि सभी कोड उदाहरण परीक्षण किए गए और कार्य कर रहे हैं
- प्रमुख अवधारणाओं की व्याख्या करने वाले टिप्पणियाँ शामिल करें
- Azure और AZD सर्वोत्तम प्रथाओं का पालन करें
- जहाँ लागू हो अपेक्षित आउटपुट प्रदान करें

### कमिट संदेश

Use clear, descriptive commit messages:
- `docs: fix typo in installation guide`
- `feat: add new container app example`
- `fix: correct broken link in chapter 2`

## आचार संहिता

कृपया हमारा [Code of Conduct](CODE_OF_CONDUCT.md) पढ़ें और पालन करें ताकि एक स्वागतयोग्य और समावेशी समुदाय बना रहे।
