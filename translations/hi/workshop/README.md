# AI डेवलपर्स के लिए AZD वर्कशॉप

Azure Developer CLI (AZD) को सीखने के लिए और AI एप्लिकेशन तैनाती पर ध्यान केंद्रित करने वाली इस हैंड-ऑन वर्कशॉप में आपका स्वागत है। यह वर्कशॉप आपको 3 चरणों में AZD टेम्पलेट्स की व्यावहारिक समझ प्राप्त करने में मदद करती है:

1. **डिस्कवरी** - आपके लिए सही टेम्पलेट खोजें।
1. **डिप्लॉयमेंट** - तैनात करें और सत्यापित करें कि यह काम करता है
1. **कस्टमाइजेशन** - इसे अपने अनुसार संशोधित और दोहराएँ!

इस वर्कशॉप के दौरान, आपको कोर डेवलपर टूल्स और वर्कफ़्लो से भी परिचित कराया जाएगा, ताकि आपका एंड-टू-एंड डेवलपमेंट सफर अधिक सुचारु हो सके।

<br/>

## ब्राउज़र-आधारित गाइड

वर्कशॉप के पाठ Markdown में हैं। आप इन्हें सीधे GitHub पर नेविगेट कर सकते हैं - या नीचे दिखाए गए स्क्रीनशॉट की तरह ब्राउज़र-आधारित प्रिव्यू लॉन्च कर सकते हैं।

![वर्कशॉप](../../../translated_images/hi/workshop.75906f133e6f8ba0.webp)

इस विकल्प का उपयोग करने के लिए - रिपॉजिटरी को अपने प्रोफ़ाइल पर fork करें, और GitHub Codespaces लॉन्च करें। एक बार VS Code टर्मिनल सक्रिय हो जाने पर, इस कमांड को टाइप करें:

```bash title="" linenums="0"
mkdocs serve > /dev/null 2>&1 &
```

कुछ सेकंड में, आपको एक पॉप-अप डायलॉग दिखाई देगा। `Open in browser` विकल्प चुनें। वेब-आधारित गाइड अब एक नए ब्राउज़र टैब में खुल जाएगा। इस प्रिव्यू के कुछ लाभ:

1. **बिल्ट-इन सर्च** - कीवर्ड या पाठ जल्दी खोजें।
1. **कॉपी आइकन** - कोडब्लॉक्स पर होवर करके इस विकल्प को देखें
1. **थीम टॉगल** - डार्क और लाइट थीम के बीच स्विच करें
1. **मदद प्राप्त करें** - फ़ूटर में Discord आइकन पर क्लिक करके शामिल हों!

## वर्कशॉप अवलोकन

**अवधि:** 3-4 घंटे  
**स्तर:** शुरुआती से मध्यवर्ती  
**पूर्व-आवश्यकताएँ:** Azure, AI अवधारणाओं, VS Code और कमांड-लाइन टूल्स की परिचितता।

यह एक हैंड-ऑन वर्कशॉप है जहाँ आप अभ्यास करके सीखते हैं। एक बार जब आप एक्सरसाइज पूरी कर लें, तो हम आपकी सीख जारी रखने के लिए AZD For Beginners पाठ्यक्रम की समीक्षा करने की सलाह देते हैं, ताकि आप सुरक्षा और उत्पादकता सर्वोत्तम प्रथाओं में आगे बढ़ सकें।

| Time| Module  | Objective |
|:---|:---|:---|
| 15 mins | [Introduction](docs/instructions/0-Introduction.md) | चरण सेट करें, उद्देश्यों को समझें |
| 30 mins | [Select AI Template](docs/instructions/1-Select-AI-Template.md) | विकल्पों का अन्वेषण करें और स्टार्टिंग चुनें | 
| 30 mins | [Validate AI Template](docs/instructions/2-Validate-AI-Template.md) | डिफ़ॉल्ट समाधान को Azure पर तैनात करें |
| 30 mins | [Deconstruct AI Template](docs/instructions/3-Deconstruct-AI-Template.md) | संरचना और कॉन्फ़िगरेशन का अन्वेषण करें |
| 30 mins | [Configure AI Template](docs/instructions/4-Configure-AI-Template.md) | उपलब्ध सुविधाओं को सक्रिय करें और आज़माएँ |
| 30 mins | [Customize AI Template](docs/instructions/5-Customize-AI-Template.md) | टेम्पलेट को अपनी आवश्यकताओं के अनुसार अनुकूलित करें |
| 30 mins | [Teardown Infrastructure](docs/instructions/6-Teardown-Infrastructure.md) | क्लीनअप और संसाधनों को रिलीज़ करें |
| 15 mins | [Wrap-Up & Next Steps](docs/instructions/7-Wrap-up.md) | सीखने के संसाधन, वर्कशॉप चुनौती |

## आप क्या सीखेंगे

AZD टेम्पलेट को एक लर्निंग सैंडबॉक्स के रूप में सोचें ताकि आप Microsoft Foundry पर एंड-टू-एंड डेवलपमेंट के विभिन्न क्षमताओं और टूल्स का अन्वेषण कर सकें। इस वर्कशॉप के अंत तक, आपको इस संदर्भ में विभिन्न टूल्स और अवधारणाओं की एक सहज समझ हासिल होनी चाहिए।

| Concept  | Objective |
|:---|:---|
| **Azure Developer CLI** | उपकरण के कमांड और वर्कफ़्लो को समझें|
| **AZD Templates**| प्रोजेक्ट संरचना और कॉन्फ़िग का समझें|
| **Azure AI Agent**| Microsoft Foundry प्रोजेक्ट प्रोविजन और डिप्लॉय करें  |
| **Azure AI Search**| एजेंट्स के साथ संदर्भ इंजीनियरिंग सक्षम करें |
| **Observability**| ट्रेसिंग, मॉनिटरिंग और मूल्यांकनों का अन्वेषण करें |
| **Red Teaming**| विरोधी परीक्षण और निवारक उपायों का अन्वेषण करें |

<br/>

## वर्कशॉप संरचना

वर्कशॉप को इस तरह संरचित किया गया है कि यह आपको टेम्पलेट डिस्कवरी से लेकर डिप्लॉयमेंट, डी-कंस्ट्रक्शन और कस्टमाइज़ेशन तक का सफर कराए - आधिकारिक [Getting Started with AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) स्टार्टर्स टेम्पलेट को आधार के रूप में उपयोग करते हुए।

### [मॉड्यूल 1: AI टेम्पलेट चुनें](docs/instructions/1-Select-AI-Template.md) (30 मिनट)

- AI टेम्पलेट क्या हैं?
- AI टेम्पलेट्स मुझे कहाँ मिल सकते हैं?
- मैं AI एजेंट्स बनाना कैसे शुरू कर सकता/सकती हूँ?
- **प्रयोगशाला**: GitHub Codespaces के साथ क्विकस्टार्ट

### [मॉड्यूल 2: AI टेम्पलेट सत्यापित करें](docs/instructions/2-Validate-AI-Template.md) (30 मिनट)

- AI टेम्पलेट आर्किटेक्चर क्या है?
- AZD डेवलपमेंट वर्कफ़्लो क्या है?
- AZD डेवलपमेंट में मुझे मदद कैसे मिल सकती है?
- **प्रयोगशाला**: AI एजेंट्स टेम्पलेट को डिप्लॉय और सत्यापित करें

### [मॉड्यूल 3: AI टेम्पलेट डी-कंस्ट्रक्ट करें](docs/instructions/3-Deconstruct-AI-Template.md) (30 मिनट)

- अपने वातावरण का अन्वेषण करें `.azure/` 
- अपने रिसोर्स सेटअप का अन्वेषण करें `infra/` 
- अपने AZD कॉन्फ़िगरेशन का अन्वेषण करें `azure.yaml`s
- **प्रयोगशाला**: एनवायरनमेंट वेरिएबल्स संशोधित करें और पुनः तैनात करें

### [मॉड्यूल 4: AI टेम्पलेट कॉन्फ़िगर करें](docs/instructions/4-Configure-AI-Template.md) (30 मिनट)
- अन्वेषण: Retrieval Augmented Generation
- अन्वेषण: Agent Evaluation & Red Teaming
- अन्वेषण: Tracing & Monitoring
- **प्रयोगशाला**: AI एजेंट + अवेवेबलिटी का अन्वेषण

### [मॉड्यूल 5: AI टेम्पलेट कस्टमाइज़ करें](docs/instructions/5-Customize-AI-Template.md) (30 मिनट)
- परिभाषित करें: परिदृश्य आवश्यकताओं के साथ PRD
- कॉन्फ़िगर करें: AZD के लिए एनवायरनमेंट वेरिएबल्स
- लागू करें: जोड़े गए कार्यों के लिए लाइफसाइकल हुक्स
- **प्रयोगशाला**: अपने परिदृश्य के लिए टेम्पलेट कस्टमाइज़ करें

### [मॉड्यूल 6: इन्फ्रास्ट्रक्चर टियरडाउन](docs/instructions/6-Teardown-Infrastructure.md) (30 मिनट)
- संक्षेप: AZD टेम्पलेट क्या हैं?
- संक्षेप: Azure Developer CLI का उपयोग क्यों करें?
- अगले कदम: किसी अलग टेम्पलेट को आज़माएँ!
- **प्रयोगशाला**: इन्फ्रास्ट्रक्चर डीप्रोविजन और क्लीनअप
