## 1. परिचय

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) या `azd` एक ओपन-सोर्स कमांडलाइन टूल है जो Azure पर एप्लिकेशन बनाते और डिप्लॉय करते समय डेवलपर कार्यप्रवाह को सरल बनाता है।

[AZD टेम्प्लेट्स](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) मानकीकृत रिपॉजिटरीज़ हैं जिनमें नमूना एप्लिकेशन कोड, _इन्फ्रास्ट्रक्चर-ऐज-कोड_ संपत्तियाँ, और `azd` कॉन्फ़िगरेशन फाइलें शामिल होती हैं ताकि एक सुसंगत समाधान वास्तुकला बन सके। इन्फ्रास्ट्रक्चर की प्रोविजनिंग `azd provision` कमांड जितनी सरल है - जबकि `azd up` कमांड एक साथ इन्फ्रास्ट्रक्चर प्रोविजन और आपकी एप्लिकेशन डिप्लॉय करता है!

परिणामस्वरूप, आपकी एप्लिकेशन विकास प्रक्रिया की शुरुआत उतनी आसान हो सकती है जितना कि सही _AZD स्टार्टर टेम्प्लेट_ ढूँढना जो आपकी एप्लिकेशन और इन्फ्रास्ट्रक्चर की जरूरतों के सबसे करीब हो - फिर रिपॉजिटरी को अपनी परिदृश्य आवश्यकताओं के अनुसार अनुकूलित करना।

शुरू करने से पहले, सुनिश्चित करें कि आपके पास Azure Developer CLI इंस्टॉल है।

1. VS कोड टर्मिनल खोलें और यह कमांड टाइप करें:

      ```bash title="" linenums="0"
      azd version
      ```

1. आपको कुछ ऐसा दिखाई देना चाहिए!

      ```bash title="" linenums="0"
      azd version 1.27.1 (commit <current-build>)
      ```

**आप अब azd के साथ टेम्प्लेट चुनने और डिप्लॉय करने के लिए तैयार हैं**

## 2. टेम्प्लेट चयन

Microsoft Foundry प्लेटफ़ॉर्म के साथ एक [सिस्टम अनुशंसित AZD टेम्प्लेट्स](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) आते हैं जो लोकप्रिय समाधान परिदृश्यों को कवर करते हैं जैसे कि _मल्टि-एजेंट वर्कफ़्लो ऑटोमेशन_ और _मल्टि-मोडल कंटेंट प्रोसेसिंग_। आप ये टेम्प्लेट्स Microsoft Foundry पोर्टल पर जाकर भी खोज सकते हैं।

1. [https://ai.azure.com/templates](https://ai.azure.com/templates) पर जाएं
1. जब प्रम्प्ट करें तो Microsoft Foundry पोर्टल में लॉग इन करें - आपको ऐसा कुछ दिखाई देगा।

![Pick](../../../../../translated_images/hi/01-pick-template.60d2d5fff5ebc374.webp)


**Basic** विकल्प आपके प्रारंभिक टेम्प्लेट्स हैं:

1. [ ] [AI Chat के साथ शुरू करें](https://github.com/Azure-Samples/get-started-with-ai-chat) जो कि एक बुनियादी चैट एप्लिकेशन को आपके डेटा के साथ Azure कंटेनर एप्स पर डिप्लॉय करता है। इसे एक बुनियादी AI चैटबॉट परिदृश्य एक्सप्लोर करने के लिए उपयोग करें।
1. [X] [AI एजेंट्स के साथ शुरू करें](https://github.com/Azure-Samples/get-started-with-ai-agents) जो एक मानक AI एजेंट (Foundry एजेंट्स के साथ) भी डिप्लॉय करता है। इसे एजेंटिक AI समाधानों से परिचित होने के लिए उपयोग करें जिनमें टूल्स और मॉडल शामिल हैं।

दूसरे लिंक को नए ब्राउज़र टैब में खोलें (या संबंधित कार्ड के लिए `Open in GitHub` क्लिक करें)। आपको इस AZD टेम्प्लेट के लिए रिपॉजिटरी दिखाई देगी। README का अवलोकन करने के लिए एक मिनट लें। एप्लिकेशन वास्तुकला इस प्रकार दिखती है:

![Arch](../../../../../translated_images/hi/architecture.8cec470ec15c65c7.webp)

## 3. टेम्प्लेट सक्रियण

इस टेम्प्लेट को डिप्लॉय करके देखें और सुनिश्चित करें कि यह वैध है। हम [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) अनुभाग के दिशानिर्देशों का पालन करेंगे।

1. टेम्प्लेट रिपॉजिटरी के लिए एक कार्यशील पर्यावरण चुनें:

      - **GitHub Codespaces**: [इस लिंक](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) पर क्लिक करें और `Create codespace` की पुष्टि करें
      - **स्थानीय क्लोन या डेवलपमेंट कंटेनर**: `Azure-Samples/get-started-with-ai-agents` को क्लोन करें और इसे VS कोड में खोलें

1. जब तक VS कोड टर्मिनल तैयार न हो जाए, प्रतीक्षा करें, फिर निम्न कमांड टाइप करें:

   ```bash title="" linenums="0"
   azd up
   ```

इस कमांड से शुरू होने वाले वर्कफ़्लो चरण पूरे करें:

1. आपको Azure में लॉग इन करने के लिए कहा जाएगा - प्रमाणीकरण के निर्देशों का पालन करें
1. अपने लिए एक अद्वितीय वातावरण नाम दर्ज करें - उदाहरण के लिए, मैंने `nitya-mshack-azd` इस्तेमाल किया
1. यह `.azure/` फोल्डर बनाएगा - आप उस पर्यावरण नाम का एक उपफोल्डर देखेंगे
1. आपको एक सदस्यता नाम चुनने के लिए कहा जाएगा - डिफ़ॉल्ट चुनें
1. आपको लोकेशन के लिए पूछा जाएगा - `East US 2` का उपयोग करें

अब, प्रोविजनिंग पूरा होने की प्रतीक्षा करें। **इसमें 10-15 मिनट लगते हैं**

1. जब पूरा हो जाए, आपका कंसोल इस प्रकार का SUCCESS संदेश दिखाएगा:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. अब आपके Azure पोर्टल में उस पर्यावरण नाम के साथ एक प्रोविज़न की गई संसाधन समूह होगा:

      ![Infra](../../../../../translated_images/hi/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **अब आप डिप्लॉय किए गए इन्फ्रास्ट्रक्चर और एप्लिकेशन को सत्यापित करने के लिए तैयार हैं**।

      - Azure Container Apps संसाधन पर क्लिक करें
      - _Essentials_ सेक्शन (ऊपर दाएं) में Application Url पर क्लिक करें

1. आपको इस तरह का होस्टेड एप्लिकेशन फ्रंट-एंड UI दिखाई देना चाहिए:

## 5. एजेंट सत्यापन

Azure Container App एक एंडपॉइंट डिप्लॉय करता है जो इस टेम्प्लेट के लिए Microsoft Foundry प्रोजेक्ट में प्रोविज़न किए गए AI एजेंट से जुड़ता है। आइए देखें इसका क्या मतलब है।

1. अपने संसाधन समूह के लिए Azure पोर्टल _Overview_ पेज पर वापस जाएं

1. उस सूची में `Microsoft Foundry` संसाधन पर क्लिक करें

1. आपको यह दिखाई देगा। `Go to Microsoft Foundry Portal` बटन पर क्लिक करें। 
   ![Foundry](../../../../../translated_images/hi/04-view-foundry-project.fb94ca41803f28f3.webp)

1. आपको आपके AI एप्लिकेशन के लिए Foundry प्रोजेक्ट पेज दिखाई देगा
   ![Project](../../../../../translated_images/hi/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` पर क्लिक करें - आप अपने प्रोजेक्ट में डिफ़ॉल्ट एजेंट को देखेंगे
   ![Agents](../../../../../translated_images/hi/06-visit-agents.bccb263f77b00a09.webp)

1. इसे चुनें - और आप एजेंट विवरण देखेंगे। निम्नलिखित बातों को नोट करें:

      - एजेंट डिफ़ॉल्ट रूप से फ़ाइल सर्च का उपयोग करता है (हमेशा)
      - एजेंट के `Knowledge` में दिखाया गया है कि इसमें 32 फाइलें अपलोड की गई हैं (फ़ाइल सर्च के लिए)
      ![Agents](../../../../../translated_images/hi/07-view-agent-details.0e049f37f61eae62.webp)

1. बाएं मेनू में `Data+indexes` विकल्प देखें और विवरण के लिए क्लिक करें। 

      - आपको ज्ञान के लिए अपलोड की गई 32 डेटा फाइलें दिखेंगी।
      - ये `src/files` के तहत 12 ग्राहक फाइलों और 20 उत्पाद फाइलों के अनुरूप होंगी
      ![Data](../../../../../translated_images/hi/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**आपने एजेंट संचालन को सत्यापित किया!** 

1. एजेंट प्रतिक्रियाएँ उन फाइलों में मौजूद ज्ञान पर आधारित होती हैं। 
1. अब आप उस डेटा से संबंधित प्रश्न पूछ सकते हैं, और स्थापित प्रतिक्रियाएँ प्राप्त कर सकते हैं।
1. उदाहरण: `customer_info_10.json` में "Amanda Perez" द्वारा की गई 3 खरीदारी का वर्णन है

कंटेनर ऐप एंडपॉइंट वाली ब्राउज़र टैब पर वापस जाएं और पूछें: `Amanda Perez के पास कौन से उत्पाद हैं?`। आपको कुछ ऐसा दिखाई देना चाहिए:

![Data](../../../../../translated_images/hi/09-ask-in-aca.4102297fc465a4d5.webp)

## 6. एजेंट प्लेग्राउंड

Microsoft Foundry की क्षमताओं के लिए थोड़ी और समझ बनाने के लिए, एजेंट को Agents Playground में चलाते हैं। 

1. Microsoft Foundry में वापस `Agents` पेज पर जाएं - डिफ़ॉल्ट एजेंट चुनें
1. `Try in Playground` विकल्प पर क्लिक करें - आपको इस प्रकार का प्लेग्राउंड UI मिलेगा
1. वही प्रश्न पूछें: `Amanda Perez के पास कौन से उत्पाद हैं?`

    ![Data](../../../../../translated_images/hi/09-ask-in-playground.a1b93794f78fa676.webp)

आपको वही (या समान) प्रतिक्रिया मिलेगी - लेकिन आपको अतिरिक्त जानकारी भी मिलेगी जिसका उपयोग आप अपने एजेंटिक एप की गुणवत्ता, लागत, और प्रदर्शन को समझने के लिए कर सकते हैं। उदाहरण के लिए:

1. ध्यान दें कि प्रतिक्रिया में उन डेटा फाइलों का उल्लेख है जिनका उपयोग जवाब को "स्थापित" करने के लिए किया गया है
1. किसी भी फाइल लेबल पर होवर करें - क्या डेटा आपकी क्वेरी और प्रदर्शित प्रतिक्रिया से मेल खाता है?

आप प्रतिक्रिया के नीचे एक _stats_ पंक्ति भी देखेंगे। 

1. किसी भी मीट्रिक पर होवर करें - जैसे Safety. आपको कुछ ऐसा दिखाई देगा
1. क्या आंका गया रेटिंग आपकी प्रतिभूति के अनुसार जवाब की सुरक्षा स्तर से मेल खाती है?

## 7. इनबिल्ट ऑब्ज़र्वेबिलिटी

ऑब्ज़र्वेबिलिटी आपके एप्लिकेशन को डिबग, समझने, और ऑप्टिमाइज़ेशन के लिए जिन डेटा को उत्पन्न करने के लिए इंस्ट्रूमेंट करना है। इसके लिए एक समझ पाने के लिए:

1. `View Run Info` बटन पर क्लिक करें - आपको यह दृश्य दिखाई देगा। यह [एजेंट ट्रेसिंग](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) का एक उदाहरण है। _आप इस दृश्य को टॉप-लेवल मेनू में Thread Logs पर क्लिक करके भी प्राप्त कर सकते हैं_।

   - एजेंट द्वारा उपयोग किए गए रन स्टेप्स और टूल्स का अंदाज़ा लगाएं
   - प्रतिक्रिया के लिए कुल टोकन गिनती (आउटपुट टोकन्स उपयोग के विरुद्ध) समझें
   - विलंबता और निष्पादन में समय कहां बिताया जा रहा है, इसे समझें

      ![Agent](../../../../../translated_images/hi/10-view-run-info.b20ebd75fef6a1cc.webp)

1. समस्याओं को डिबग करने के लिए सहायक संदर्भ प्रदान कर सकने वाले रन के लिए अतिरिक्त गुण देखने के लिए `Metadata` टैब पर क्लिक करें।   

      ![Agent](../../../../../translated_images/hi/11-view-run-info-metadata.7966986122c7c2df.webp)


1. एजेंट प्रतिक्रिया पर स्वचालित आकलन देखने के लिए `Evaluations` टैब पर क्लिक करें। इनमें सुरक्षा मूल्यांकन (जैसे, Self-harm) और एजेंट-विशिष्ट मूल्यांकन (जैसे, Intent resolution, Task adherence) शामिल हैं।

      ![Agent](../../../../../translated_images/hi/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. अंत में, साइडबार मेनू में `Monitoring` टैब पर क्लिक करें।

      - प्रदर्शित पेज में `Resource usage` टैब चुनें - और मीट्रिक्स देखें।
      - लागत (टोकन) और लोड (रिक्वेस्ट) के संदर्भ में एप्लिकेशन उपयोग ट्रैक करें।
      - पहले बाइट (इनपुट प्रोसेसिंग) और अंतिम बाइट (आउटपुट) के लिए एप्लिकेशन विलंबता ट्रैक करें।

## 8. पर्यावरण चर

अब तक, हमने ब्राउज़र में डिप्लॉयमेंट का संचालन किया है - और सत्यापित किया है कि हमारा इन्फ्रास्ट्रक्चर प्रोविज़न है और एप्लिकेशन चल रहा है। लेकिन _कोड-प्रथम_ काम करने के लिए, हमें स्थानीय विकास पर्यावरण को उन संबंधित चर के साथ कॉन्फ़िगर करना होगा जो इन संसाधनों के साथ काम करने के लिए आवश्यक हैं। `azd` का उपयोग इसे आसान बनाता है।

1. Azure Developer CLI [पर्यावरण चर](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) का उपयोग एप्लिकेशन डिप्लॉयमेंट्स के कॉन्फ़िगरेशन सेटिंग्स को स्टोर और प्रबंधित करने के लिए करता है।

1. पर्यावरण चर `.azure/<env-name>/.env` में संग्रहीत होते हैं - यह उन्हें उस `env-name` पर्यावरण में सीमित करता है जिसका उपयोग डिप्लॉयमेंट के दौरान किया गया, और एक ही रिपॉजिटरी में विभिन्न डिप्लॉयमेंट लक्ष्यों के बीच पर्यावरणों को अलग करने में मदद करता है।

1. पर्यावरण चर स्वचालित रूप से `azd` कमांड द्वारा लोड किए जाते हैं जब भी वह किसी विशेष कमांड (जैसे, `azd up`) को चलाता है। ध्यान दें कि `azd` स्वतः OS-स्तरीय पर्यावरण चर (जैसे, शेल में सेट किए गए) को नहीं पढ़ता - इसके बजाय स्क्रिप्ट के भीतर जानकारी स्थानांतरित करने के लिए `azd set env` और `azd get env` का उपयोग करें।


कुछ कमांड्स आज़माएं:

1. इस पर्यावरण में `azd` के लिए सेट सभी पर्यावरण चर प्राप्त करें:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      आपको कुछ ऐसा दिखाई देगा:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. एक विशिष्ट मान प्राप्त करें - उदाहरण के लिए, मैं जानना चाहता हूं कि क्या हमने `AZURE_AI_AGENT_MODEL_NAME` मान सेट किया है

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      आपको कुछ ऐसा दिखाई देगा - यह डिफ़ॉल्ट रूप से सेट नहीं था!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd` के लिए एक नया पर्यावरण चर सेट करें। यहाँ, हम एजेंट मॉडल नाम अपडेट कर रहे हैं। _ध्यान दें: किए गए किसी भी परिवर्तन को तुरंत `.azure/<env-name>/.env` फाइल में प्रतिबिंबित किया जाएगा।_

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      अब, हमें मान सेट पाया जाना चाहिए:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. ध्यान दें कि कुछ संसाधन स्थायी हैं (जैसे, मॉडल डिप्लॉयमेंट) और उनके लिए केवल `azd up` से अधिक की आवश्यकता होगी। आइए मूल डिप्लॉयमेंट को हटाएं और परिवर्तित env vars के साथ पुनःडिप्लॉय करें।

1. **रिफ्रेश** यदि आपने पहले किसी azd टेम्प्लेट का उपयोग करके इन्फ्रास्ट्रक्चर डिप्लॉय किया है - तो आप इस कमांड का उपयोग करके अपनी स्थानीय पर्यावरण चर की स्थिति को आपकी वर्तमान Azure डिप्लॉयमेंट की स्थिति के आधार पर _रिफ्रेश_ कर सकते हैं:

      ```bash title="" linenums="0"
      azd env refresh
      ```


      यह दो या अधिक स्थानीय विकास परिवेशों (जैसे, कई डेवलपर्स वाली टीम) में पर्यावरण चर को _sync_ करने का एक मजबूत तरीका है - जिससे तैनात आधारभूत संरचना env चर की स्थिति के लिए वास्तविक सच्चाई के रूप में कार्य करती है। टीम के सदस्य बस चर को _refresh_ करके फिर से सिंक में आ जाते हैं।
