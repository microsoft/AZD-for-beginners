## 1. परिचय

[Azure डेवलपर CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) या `azd` एक ओपन-सोर्स कमांडलाइन टूल है जो Azure पर एप्लिकेशन बनाने और तैनात करने के दौरान डेवलपर वर्कफ़्लो को सरल बनाता है। 

[AZD टेम्पलेट्स](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) मानकीकृत रिपोजिटरी हैं जिनमें सैंपल एप्लिकेशन कोड, _इन्फ्रास्ट्रक्चर-एज़-कोड_ संपत्तियाँ, और एक समग्र समाधान आर्किटेक्चर के लिए `azd` कॉन्फ़िगरेशन फाइलें शामिल हैं। इंफ्रास्ट्रक्चर प्रोविजनिंग उतना ही सरल बन जाता है जितना एक `azd provision` कमांड चलाना - जबकि `azd up` का उपयोग करने से आप एक ही बार में इंफ्रास्ट्रक्चर प्रोविजन और अपना एप्लिकेशन तैनात कर सकते हैं!

इसके परिणामस्वरूप, अपने एप्लिकेशन विकास प्रक्रिया की शुरुआत करना उतना ही आसान हो सकता है जितना सही _AZD Starter टेम्पलेट_ ढूँढना जो आपके एप्लिकेशन और इंफ्रास्ट्रक्चर आवश्यकताओं के सबसे करीब हो - फिर रिपोजिटरी को अपने परिदृश्य आवश्यकताओं के अनुसार अनुकूलित करना।

शुरू करने से पहले, चलिए सुनिश्चित करते हैं कि आपके पास Azure डेवलपर CLI इंस्टॉल है।

1. Open a VS Code terminal and type this command:

      ```bash title="" linenums="0"
      azd version
      ```

1. आपको कुछ ऐसा दिखाई देना चाहिए!

      ```bash title="" linenums="0"
      azd version 1.19.0 (commit b3d68cea969b2bfbaa7b7fa289424428edb93e97)
      ```

**अब आप azd के साथ एक टेम्पलेट चुनने और तैनात करने के लिए तैयार हैं**

## 2. टेम्पलेट चयन

Microsoft Foundry प्लेटफ़ॉर्म के साथ एक [अनुशंसित AZD टेम्पलेट्स का सेट](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) आता है जो लोकप्रिय समाधान परिदृश्यों को कवर करता है जैसे _मल्टी-एजेंट वर्कफ़्लो ऑटोमेशन_ और _मल्टी-मोडल कंटेंट प्रोसेसिंग_. आप Microsoft Foundry पोर्टल पर जाकर भी इन टेम्पलेट्स को खोज सकते हैं।

1. Visit [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. जब संकेत दिया जाए तो Microsoft Foundry पोर्टल में लॉग इन करें - आपको कुछ ऐसा दिखाई देगा।

![चुनें](../../../../../translated_images/hi/01-pick-template.60d2d5fff5ebc374.webp)


**Basic** विकल्प आपके स्टेटर टेम्पलेट्स हैं:

1. [ ] [AI चैट के साथ शुरू करें](https://github.com/Azure-Samples/get-started-with-ai-chat) जो Azure Container Apps पर आपका डेटा लेकर एक बेसिक चैट एप्लिकेशन तैनात करता है। इसका उपयोग एक बुनियादी AI चैटबॉट परिदृश्य का अन्वेषण करने के लिए करें।
1. [X] [AI एजेंट्स के साथ शुरू करें](https://github.com/Azure-Samples/get-started-with-ai-agents) जो Foundry Agents के साथ एक मानक AI एजेंट भी तैनात करता है। उपकरणों और मॉडलों को शामिल करने वाले एजेंटिक AI समाधानों से परिचित होने के लिए इसका उपयोग करें।

दूसरे लिंक को एक नए ब्राउज़र टैब में खोलें (या संबंधित कार्ड के लिए `Open in GitHub` पर क्लिक करें)। आपको इस AZD टेम्पलेट के लिए रिपोजिटरी दिखाई देगी। README का एक मिनट के लिए अन्वेषण करें। एप्लिकेशन आर्किटेक्चर इस प्रकार दिखता है:

![आर्किटेक्चर](../../../../../translated_images/hi/architecture.8cec470ec15c65c7.webp)

## 3. टेम्पलेट सक्रियकरण

आइए इस टेम्पलेट को तैनात करने की कोशिश करें और सुनिश्चित करें कि यह वैध है। हम [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) सेक्शन में दिए दिशानिर्देशों का पालन करेंगे।

1. [यह लिंक](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) पर क्लिक करें - डिफ़ॉल्ट कार्रवाई `Create codespace` की पुष्टि करें
1. यह एक नया ब्राउज़र टैब खोलता है - GitHub Codespaces सत्र के लोड होने का इंतज़ार करें
1. Codespaces में VS Code टर्मिनल खोलें - निम्नलिखित कमांड टाइप करें:

   ```bash title="" linenums="0"
   azd up
   ```

Complete the workflow steps that this will trigger:

1. आपसे Azure में लॉग इन करने के लिए कहा जाएगा - प्रमाणीकरण के निर्देशों का पालन करें
1. अपने लिए एक अनूठा environment नाम दर्ज करें - उदाहरण के लिए, मैंने `nitya-mshack-azd` किया
1. यह एक `.azure/` फ़ोल्डर बनाएगा - आप उस env नाम के साथ एक सबफ़ोल्डर देखेंगे
1. आपसे एक सब्सक्रिप्शन नाम चुनने के लिए कहा जाएगा - डिफ़ॉल्ट चुनें
1. आपसे एक स्थान के लिए कहा जाएगा - `East US 2` का उपयोग करें

अब, आप प्रोविजनिंग के पूरे होने का इंतज़ार करते हैं। **यह 10-15 मिनट लेता है**

1. पूरा होने पर, आपका कंसोल इस तरह का SUCCESS संदेश दिखाएगा:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. अब आपका Azure पोर्टल उस env नाम के साथ एक प्रोविजन की गई रिसोर्स ग्रुप दिखाएगा:

      ![इन्फ्रास्ट्रक्चर](../../../../../translated_images/hi/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **अब आप तैनात किए गए इंफ्रास्ट्रक्चर और एप्लिकेशन को सत्यापित करने के लिए तैयार हैं**।

## 4. टेम्पलेट सत्यापन

1. Azure Portal [Resource Groups](https://portal.azure.com/#browse/resourcegroups) पेज पर जाएँ - जब संकेत दिया जाए लॉग इन करें
1. अपने environment नाम के लिए RG पर क्लिक करें - आप ऊपर दिया गया पृष्ठ देखेंगे

      - Azure Container Apps रिसोर्स पर क्लिक करें
      - _Essentials_ सेक्शन (ऊपर दायां) में Application Url पर क्लिक करें

1. आपको इस तरह का होस्ट किया गया एप्लिकेशन फ्रंट-एंड UI दिखाई देना चाहिए:

   ![एप्लिकेशन](../../../../../translated_images/hi/03-test-application.471910da12c3038e.webp)

1. कुछ [नमूना प्रश्न](https://github.com/Azure-Samples/get-started-with-ai-agents/blob/main/docs/sample_questions.md) पूछकर देखें

      1. पूछें: ```फ्रांस की राजधानी क्या है?``` 
      1. पूछें: ```दो लोगों के लिए $200 से कम में सबसे अच्छी टेंट कौन-सी है, और इसमें कौन-कौन सी सुविधाएँ हैं?```

1. आपको नीचे दिखाए गए समान उत्तर मिलने चाहिए। _लेकिन यह कैसे काम करता है?_ 

      ![एप्लिकेशन](../../../../../translated_images/hi/03-test-question.521c1e863cbaddb6.webp)

## 5.  एजेंट सत्यापन

Azure Container App एक एन्डपॉइंट तैनात करता है जो इस टेम्पलेट के लिए Microsoft Foundry प्रोजेक्ट में प्रोविजन किए गए AI एजेंट से जुड़ता है। आइए देखें इसका क्या मतलब है।

1. अपने रिसोर्स ग्रुप के Azure Portal _Overview_ पेज पर वापस जाएँ

1. सूची में `Microsoft Foundry` रिसोर्स पर क्लिक करें

1. आपको यह दिखाई देना चाहिए। `Go to Microsoft Foundry Portal` बटन पर क्लिक करें। 
   ![Foundry](../../../../../translated_images/hi/04-view-foundry-project.fb94ca41803f28f3.webp)

1. आपको अपने AI एप्लिकेशन के लिए Foundry प्रोजेक्ट पेज दिखाई देगा
   ![परियोजना](../../../../../translated_images/hi/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` पर क्लिक करें - आप अपने प्रोजेक्ट में प्रोविजन किए गए डिफ़ॉल्ट एजेंट को देखेंगे
   ![एजेंट्स](../../../../../translated_images/hi/06-visit-agents.bccb263f77b00a09.webp)

1. इसे चुनें - और आप एजेंट विवरण देखेंगे। निम्नलिखित बातों पर ध्यान दें:

      - एजेंट डिफ़ॉल्ट रूप से File Search का उपयोग करता है (हमेशा)
      - एजेंट की `Knowledge` बताती है कि इसमें 32 फाइलें अपलोड की गई हैं (फाइल सर्च के लिए)
      ![एजेंट्स](../../../../../translated_images/hi/07-view-agent-details.0e049f37f61eae62.webp)

1. बाएँ मेनू में `Data+indexes` विकल्प देखें और विवरण के लिए क्लिक करें। 

      - आपको नॉलेज के लिए अपलोड की गई 32 डेटा फाइलें दिखाई देंगी।
      - ये `src/files` के अंतर्गत 12 कस्टमर फाइलों और 20 प्रोडक्ट फाइलों के अनुरूप होंगी 
      ![डेटा](../../../../../translated_images/hi/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**आपने एजेंट ऑपरेशन का सत्यापन कर लिया है!** 

1. एजेंट के उत्तर उन फाइलों में निहित ज्ञान पर आधारित हैं। 
1. अब आप उस डेटा से संबंधित प्रश्न पूछ सकते हैं, और ग्राउंडेड उत्तर प्राप्त कर सकते हैं।
1. उदाहरण: `customer_info_10.json` "Amanda Perez" द्वारा की गई 3 खरीद का वर्णन करता है

ब्राउज़र टैब पर वापिस जाएँ जिसमें Container App एन्डपॉइंट खुला है और पूछें: `Amanda Perez के कौन से उत्पाद हैं?`. आपको कुछ इसी तरह दिखना चाहिए:

![डेटा](../../../../../translated_images/hi/09-ask-in-aca.4102297fc465a4d5.webp)

## 6. एजेंट प्लेग्राउंड

Microsoft Foundry की क्षमताओं के लिए थोड़ा और सहज ज्ञान विकसित करने के लिए, एजेंट को Agents Playground में चलाकर देखें। 

1. Microsoft Foundry में `Agents` पेज पर लौटें - डिफ़ॉल्ट एजेंट चुनें
1. `Try in Playground` विकल्प पर क्लिक करें - आपको एक Playground UI ऐसा दिखना चाहिए
1. वही प्रश्न पूछें: `Amanda Perez के कौन से उत्पाद हैं?`

    ![डेटा](../../../../../translated_images/hi/09-ask-in-playground.a1b93794f78fa676.webp)

आपको वही (या समान) उत्तर मिलता है - लेकिन आपको अतिरिक्त जानकारी भी मिलती है जिससे आप अपने एजेंटिक ऐप की गुणवत्ता, लागत, और प्रदर्शन को समझ सकते हैं। उदाहरण के लिए:

1. ध्यान दें कि प्रतिक्रिया में उन डेटा फाइलों का हवाला दिया गया है जिनका उपयोग प्रतिक्रिया को "ग्राउंड" करने के लिए किया गया
1. किसी भी फाइल लेबल पर होवर करें - क्या डेटा आपके प्रश्न और प्रदर्शित उत्तर से मेल खाता है?

आपको प्रतिक्रिया के नीचे एक _stats_ पंक्ति भी दिखेगी। 

1. किसी भी मीट्रिक पर होवर करें - उदाहरण के लिए, Safety। आपको कुछ ऐसा दिखाई देगा
1. क्या आंकी गई रेटिंग आपके सहज ज्ञान के अनुसार प्रतिक्रिया की सुरक्षा स्तर से मेल खाती है?

      ![डेटा](../../../../../translated_images/hi/10-view-run-info-meter.6cdb89a0eea5531f.webp)

## 7. बिल्ट-इन ऑब्ज़र्वेबिलिटी

ऑब्ज़र्वेबिलिटी का मतलब है अपने एप्लिकेशन को इस प्रकार इंस्ट्रूमेंट करना ताकि वह डेटा उत्पन्न करे जिसे संचालन को समझने, डीबग करने, और अनुकूलित करने के लिए उपयोग किया जा सके। इसे समझने के लिए:

1. `View Run Info` बटन पर क्लिक करें - आपको यह दृश्य दिखाई देगा। यह [Agent tracing](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) का एक उदाहरण है। _आप यह दृश्य Thread Logs पर शीर्ष-स्तरीय मेनू में क्लिक करके भी प्राप्त कर सकते हैं_।

   - रन स्टेप्स और एजेंट द्वारा उपयोग किए गए टूल्स का अनुभव करें
   - प्रतिक्रिया के लिए कुल Token गणना (और आउटपुट टोकन उपयोग के विरुद्ध) समझें
   - निष्पादन में विलंब और समय कहाँ व्यतीत हो रहा है यह समझें

      ![एजेंट](../../../../../translated_images/hi/10-view-run-info.b20ebd75fef6a1cc.webp)

1. रन के अतिरिक्त गुणों को देखने के लिए `Metadata` टैब पर क्लिक करें, जो बाद में डिबगिंग मुद्दों के लिए उपयोगी संदर्भ प्रदान कर सकते हैं।   

      ![एजेंट](../../../../../translated_images/hi/11-view-run-info-metadata.7966986122c7c2df.webp)


1. `Evaluations` टैब पर क्लिक करें ताकि एजेंट प्रतिक्रिया पर स्वचालित-अंकन देखें। इनमें सुरक्षा मूल्यांकन (जैसे, Self-harm) और एजेंट-विशिष्ट मूल्यांकन (जैसे, Intent resolution, Task adherence) शामिल हैं।

      ![एजेंट](../../../../../translated_images/hi/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. अंत में, साइडबार मेनू में `Monitoring` टैब पर क्लिक करें।

      - प्रदर्शित पृष्ठ में `Resource usage` टैब चुनें - और मीट्रिक्स देखें।
      - लागत (टोकन) और लोड (अनुरोधों) के संदर्भ में एप्लिकेशन उपयोग को ट्रैक करें।
      - इनपुट प्रोसेसिंग (पहला बाइट) और आउटपुट (अंतिम बाइट) के लिए एप्लिकेशन विलंबता को ट्रैक करें।

## 8. परिवेश चर

अब तक, हमने ब्राउज़र में तैनाती के माध्यम से चलकर यह सत्यापित किया है कि हमारी इंफ्रास्ट्रक्चर प्रोविजन हुई है और एप्लिकेशन परिचालन में है। लेकिन एप्लिकेशन को _कोड-प्रथम_ तरीके से काम करने के लिए, हमें अपने स्थानीय विकास वातावरण को उन संबंधित वेरिएबल्स के साथ कॉन्फ़िगर करना होगा जिनकी इन रिसोर्सेज के साथ काम करने के लिए आवश्यकता होती है। `azd` का उपयोग करना इसे आसान बनाता है।

1. Azure Developer CLI [पर्यावरण चर का उपयोग करता है](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) एप्लिकेशन तैनातियों के लिए कॉन्फ़िगरेशन सेटिंग्स को स्टोर और प्रबंधित करने के लिए।

1. पर्यावरण चर `.azure/<env-name>/.env` में संग्रहीत होते हैं - यह उन्हें तैनाती के दौरान उपयोग किए गए `env-name` वातावरण तक सीमित करता है और आपको एक ही रिपो में विभिन्न तैनाती लक्ष्यों के बीच वातावरण अलग करने में मदद करता है।

1. पर्यावरण चर `azd` कमांड द्वारा स्वचालित रूप से लोड किए जाते हैं जब भी यह एक विशिष्ट कमांड निष्पादित करता है (उदा., `azd up`)। ध्यान दें कि `azd` OS-स्तर के पर्यावरण चर (उदा., शेल में सेट किए गए) को स्वचालित रूप से नहीं पढ़ता - इसके बजाय स्क्रिप्ट के भीतर जानकारी स्थानांतरित करने के लिए `azd set env` और `azd get env` का उपयोग करें।


आइए कुछ कमांड आज़माएँ:

1. इस वातावरण में `azd` के लिए सेट किए गए सभी पर्यावरण चर प्राप्त करें:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      आपको कुछ ऐसा दिखाई देगा:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. एक विशिष्ट मान प्राप्त करें - उदाहरण के लिए, मैं जानना चाहता हूँ कि क्या हमने `AZURE_AI_AGENT_MODEL_NAME` मान सेट किया है

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      आपको कुछ ऐसा दिखाई देगा - यह डिफ़ॉल्ट रूप से सेट नहीं था!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd` के लिए एक नया पर्यावरण चर सेट करें। यहाँ, हम एजेंट मॉडल नाम अपडेट करते हैं। _नोट: कोई भी बदलाव तुरंत `.azure/<env-name>/.env` फ़ाइल में परिलक्षित होगा।_

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      अब, हमें मिलना चाहिए कि मान सेट है:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. ध्यान दें कि कुछ रिसोर्सेज स्थायी होते हैं (उदा., मॉडल डिप्लॉयमेंट्स) और उन्हें सिर्फ `azd up` से पुनःतैनाती के लिए अधिक की आवश्यकता होगी। चलिए मूल तैनाती को नष्ट करने और बदले हुए env vars के साथ पुनःतैनात करने की कोशिश करते हैं।

1. **रिफ्रेश** यदि आपने पहले किसी azd टेम्पलेट का उपयोग करके इंफ्रास्ट्रक्चर तैनात किया था - तो आप इस कमांड का उपयोग करके अपने स्थानीय पर्यावरण चर की स्थिति को वर्तमान Azure तैनाती की स्थिति के आधार पर _रिफ्रेश_ कर सकते हैं:

      ```bash title="" linenums="0"
      azd env refresh
      ```

      यह दो या अधिक स्थानीय विकास परिवेशों में पर्यावरण चर को _सिंक_ करने का एक शक्तिशाली तरीका है (उदा., कई डेवलपर्स वाली टीम) - जिससे परिनियोजित अवसंरचना पर्यावरण चर की स्थिति के लिए सत्य स्रोत के रूप में काम करे। टीम सदस्य बस वेरिएबल्स को _रिफ्रेश_ करके फिर से सिंक में आ जाते हैं।
