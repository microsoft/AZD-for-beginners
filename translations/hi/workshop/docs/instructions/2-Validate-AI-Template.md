## 1. परिचय

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) या `azd` एक ओपन-सोर्स कमांडलाइन टूल है जो Azure पर अनुप्रयोगों को बनाने और डिप्लॉय करने के दौरान डेवलपर वर्कफ़्लो को सरल बनाता है।

[AZD टेम्पलेट](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) मानकीकृत रिपॉजिटरीज़ हैं जिनमें नमूना एप्लिकेशन कोड, _इन्फ्रास्ट्रक्चर-एज़-कोड_ संसाधन, और `azd` कॉन्फ़िगरेशन फाइलें शामिल होती हैं ताकि एक सामंजस्यपूर्ण सॉल्यूशन आर्किटेक्चर बनाया जा सके। इन्फ्रास्ट्रक्चर प्रोविज़निंग बस एक `azd provision` कमांड जितनी सरल है - जबकि `azd up` का उपयोग करके आप इन्फ्रास्ट्रक्चर **और** अपनी एप्लिकेशन दोनों एक साथ प्रोवाइड कर सकते हैं!

इसलिए, अपने एप्लिकेशन विकास प्रक्रिया को शुरू करना उतना ही आसान हो सकता है जितना कि सही _AZD स्टार्टर टेम्पलेट_ ढूँढना जो आपके एप्लिकेशन और इन्फ्रास्ट्रक्चर आवश्यकताओं के सबसे करीब हो - फिर रिपॉजिटरी को अपने परिदृश्य आवश्यकताओं के अनुसार अनुकूलित करना।

शुरू करने से पहले, सुनिश्चित करें कि आपके पास Azure Developer CLI इंस्टॉल है।

1. एक VS Code टर्मिनल खोलें और यह कमांड टाइप करें:

      ```bash title="" linenums="0"
      azd version
      ```

1. आपको कुछ ऐसा दिखाई देगा!

      ```bash title="" linenums="0"
      azd version 1.23.12 (commit <current-build>)
      ```

**अब आप azd के साथ एक टेम्पलेट चुनने और डिप्लॉय करने के लिए तैयार हैं**

## 2. टेम्पलेट चयन

Microsoft Foundry प्लेटफ़ॉर्म में [एक सेट ऑफ़ सिफारिश किए गए AZD टेम्पलेट्स](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) शामिल हैं, जो लोकप्रिय समाधान परिदृश्यों को कवर करते हैं जैसे _मल्टी-एजेंट वर्कफ़्लो ऑटोमेशन_ और _मल्टी-मोडल कंटेंट प्रोसेसिंग_। आप Microsoft Foundry पोर्टल पर जाकर भी इन टेम्पलेट्स को खोज सकते हैं।

1. जाएं [https://ai.azure.com/templates](https://ai.azure.com/templates)
1. जब संकेत दिया जाए तो Microsoft Foundry पोर्टल में लॉग इन करें - आपको कुछ ऐसा दिखेगा।

![Pick](../../../../../translated_images/hi/01-pick-template.60d2d5fff5ebc374.webp)


**Basic** विकल्प आपके स्टार्टर टेम्पलेट्स हैं:

1. [ ] [AI Chat के साथ शुरुआत करें](https://github.com/Azure-Samples/get-started-with-ai-chat) जो एक बेसिक चैट एप्लिकेशन को _आपके डेटा_ के साथ Azure Container Apps पर डिप्लॉय करता है। इसे एक बेसिक AI चैटबॉट परिदृश्य का पता लगाने के लिए उपयोग करें।
1. [X] [AI Agents के साथ शुरुआत करें](https://github.com/Azure-Samples/get-started-with-ai-agents) जो Foundry Agents के साथ एक स्टैंडर्ड AI एजेंट को भी डिप्लॉय करता है। इसे एजेंटिक AI समाधानों को समझने के लिए उपयोग करें जिसमें टूल्स और मॉडल शामिल हों।

दूसरे लिंक को एक नए ब्राउज़र टैब में खोलें (या संबंधित कार्ड के लिए `Open in GitHub` पर क्लिक करें)। आपको इस AZD टेम्पलेट का रिपॉजिटरी दिखाई देगा। README को एक्सप्लोर करने के लिए एक मिनट लें। एप्लिकेशन आर्किटेक्चर कुछ ऐसा दिखता है:

![Arch](../../../../../translated_images/hi/architecture.8cec470ec15c65c7.webp)

## 3. टेम्पलेट सक्रियण

आइए इस टेम्पलेट को डिप्लॉय करें और सुनिश्चित करें कि यह मान्य है। हम [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) सेक्शन में दिए निर्देशों का पालन करेंगे।

1. टेम्पलेट रिपॉजिटरी के लिए एक वर्किंग एनवायरनमेंट चुनें:

      - **GitHub Codespaces**: इस [लिंक](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) पर क्लिक करें और `Create codespace` की पुष्टि करें
      - **लोकल क्लोन या डेव कंटेनर**: `Azure-Samples/get-started-with-ai-agents` क्लोन करें और इसे VS Code में खोलें

1. जब तक VS Code टर्मिनल तैयार न हो जाए, तब तक प्रतीक्षा करें, फिर निम्नलिखित कमांड टाइप करें:

   ```bash title="" linenums="0"
   azd up
   ```

इससे ट्रिगर होने वाले वर्कफ़्लो स्टेप्स को पूरा करें:

1. आपको Azure में लॉग इन करने के लिए कहा जाएगा - प्रामाणिकरण के निर्देशों का पालन करें
1. अपने लिए एक अद्वितीय एनवायरनमेंट नाम दर्ज करें - उदाहरण के लिए, मैंने `nitya-mshack-azd` का उपयोग किया
1. यह एक `.azure/` फ़ोल्डर बनाएगा - आप एनवायरनमेंट नाम के साथ एक सबफ़ोल्डर देखेंगे
1. आपको एक सब्सक्रिप्शन नाम चुनने के लिए कहा जाएगा - डिफ़ॉल्ट चुनें
1. आपको एक लोकेशन चुनने के लिए कहा जाएगा - `East US 2` चुनें

अब, आप प्रोविज़निंग के पूरा होने का इंतजार करें। **यह 10-15 मिनट तक ले सकता है**

1. जब पूरा हो जाए, तो आपका कंसोल SUCCESS संदेश दिखाएगा जैसा कि नीचे है:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. आपका Azure पोर्टल अब उस एनवायरनमेंट नाम के साथ एक प्रोविजनड रिसोर्स ग्रुप दिखाएगा:

      ![Infra](../../../../../translated_images/hi/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **अब आप डिप्लॉय की गई इन्फ्रास्ट्रक्चर और एप्लिकेशन को मान्य करने के लिए तैयार हैं**।

1. Azure पोर्टल पर [Resource Groups](https://portal.azure.com/#browse/resourcegroups) पेज पर जाएं - प्रॉम्प्ट आने पर लॉग इन करें
1. अपने एनवायरनमेंट नाम के RG पर क्लिक करें - आप ऊपर दी गई पेज देखें

      - Azure Container Apps रिसोर्स पर क्लिक करें
      - _Essentials_ सेक्शन (ऊपर दाएं) में Application Url पर क्लिक करें

1. आपको एक होस्टेड एप्लिकेशन फ्रंट-एंड UI ऐसा दिखेगा:

      1. पूछें: ```फ्रांस की राजधानी क्या है?``` 
      1. पूछें: ```दो लोगों के लिए $200 से कम में सबसे अच्छा तंबू कौन सा है, और इसमें कौन-कौन सी विशेषताएं हैं?```

1. आपको नीचे दिखाए गए समान उत्तर मिलेंगे। _लेकिन यह कैसे काम करता है?_

## 5. एजेंट की मान्यता

Azure Container App एक एन्डपॉइंट डिप्लॉय करता है जो उस AI एजेंट से जुड़ता है जो इस टेम्पलेट के लिए Microsoft Foundry प्रोजेक्ट में प्रोविजन किया गया है। चलिए देखते हैं इसका क्या मतलब है।

1. अपने रिसोर्स ग्रुप के लिए Azure पोर्टल _Overview_ पेज पर वापस जाएं

1. उस सूची में `Microsoft Foundry` संसाधन पर क्लिक करें

1. आपको यह दिखाई देगा। `Go to Microsoft Foundry Portal` बटन पर क्लिक करें।
   ![Foundry](../../../../../translated_images/hi/04-view-foundry-project.fb94ca41803f28f3.webp)

1. आपको अपने AI एप्लिकेशन का Foundry प्रोजेक्ट पेज दिखेगा
   ![Project](../../../../../translated_images/hi/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` पर क्लिक करें - आपको आपके प्रोजेक्ट में डिफ़ॉल्ट एजेंट दिखेगा
   ![Agents](../../../../../translated_images/hi/06-visit-agents.bccb263f77b00a09.webp)

1. इसे चुनें - और एजेंट के विवरण देखें। निम्नलिखित ध्यान दें:

      - एजेंट डिफ़ॉल्ट रूप से फ़ाइल खोज (File Search) का उपयोग करता है (हमेशा)
      - एजेंट `Knowledge` में दिखाता है कि 32 फाइलें अपलोड की गई हैं (फाइल सर्च के लिए)
      ![Agents](../../../../../translated_images/hi/07-view-agent-details.0e049f37f61eae62.webp)

1. बाएं मेनू में `Data+indexes` विकल्प खोजें और विवरण के लिए क्लिक करें।

      - आपको नॉलेज के लिए 32 डेटा फाइलें अपलोड दिखेंगी।
      - ये `src/files` के तहत 12 ग्राहक फाइलों और 20 उत्पाद फाइलों के अनुरूप होंगी 
      ![Data](../../../../../translated_images/hi/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**आपने एजेंट ऑपरेशन को मान्य किया!**

1. एजेंट की प्रतिक्रियाएँ उन फाइलों में नॉलेज पर आधारित होती हैं।
1. अब आप उन डेटा से संबंधित प्रश्न पूछ सकते हैं और आधारभूत प्रतिक्रियाएं प्राप्त कर सकते हैं।
1. उदाहरण: `customer_info_10.json` "Amanda Perez" द्वारा की गई 3 खरीदों का वर्णन करता है।

ब्राउज़र टैब में Container App एन्डपॉइंट पर वापस जाएं और पूछें: `Amanda Perez के पास कौन से उत्पाद हैं?`। आपको कुछ ऐसा मिलेगा:

![Data](../../../../../translated_images/hi/09-ask-in-aca.4102297fc465a4d5.webp)

## 6. एजेंट प्लेग्राउंड

आइए Microsoft Foundry की क्षमताओं के लिए थोड़ा और अंतर्ज्ञान बनाएं, एजेंट को Agents Playground में परख कर।

1. Microsoft Foundry में `Agents` पेज पर वापस जाएं - डिफ़ॉल्ट एजेंट चुनें
1. `Try in Playground` विकल्प पर क्लिक करें - आपको ऐसा प्लेग्राउंड UI मिलेगा
1. वही प्रश्न पूछें: `Amanda Perez के पास कौन से उत्पाद हैं?`

    ![Data](../../../../../translated_images/hi/09-ask-in-playground.a1b93794f78fa676.webp)

आपको वही (या समान) प्रतिक्रिया मिलती है - लेकिन आपको अतिरिक्त जानकारी भी मिलेगी जिसका उपयोग आप अपने एजेंटिक ऐप की गुणवत्ता, लागत, और प्रदर्शन को समझने के लिए कर सकते हैं। उदाहरण के लिए:

1. ध्यान दें कि प्रतिक्रिया में "ग्राउंड" करने के लिए उपयोग की गई डेटा फाइलों का उल्लेख है
1. इन फ़ाइलों के लेबल पर होवर करें - क्या डेटा आपके प्रश्न और प्रदर्शित प्रतिक्रिया से मेल खाता है?

प्रतिक्रिया के नीचे एक _stats_ पंक्ति भी दिखाई देती है।

1. किसी भी मीट्रिक पर होवर करें - उदाहरण के लिए, Safety। आपको कुछ ऐसा दिखेगा
1. क्या आंका गया रेटिंग प्रतिक्रिया की सुरक्षा स्तर के लिए आपके अंतर्ज्ञान से मेल खाता है?

## 7. अंतर्निहित पर्यवेक्षणीयता

Observability का मतलब है अपने एप्लिकेशन को इस प्रकार उपकरणबद्ध करना कि वह डेटा उत्पन्न करे जिसका उपयोग इसके संचालन को समझने, डिबग और अनुकूलित करने के लिए किया जा सके। इसे समझने के लिए:

1. `View Run Info` बटन पर क्लिक करें - आपको यह दृश्य दिखेगा। यह [Agent tracing](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) का एक उदाहरण है। _आप इसे टॉप-लेवल मेनू में Thread Logs पर क्लिक करके भी प्राप्त कर सकते हैं_।

   - एजेंट द्वारा उपयोग किए गए रन स्टेप्स और टूल्स की जानकारी लें
   - प्रतिक्रिया के लिए कुल टोकन गणना (आउटपुट टोकन की तुलना में) समझें
   - लेटेंसी और निष्पादन में समय कहाँ खर्च हो रहा है यह समझें

      ![Agent](../../../../../translated_images/hi/10-view-run-info.b20ebd75fef6a1cc.webp)

1. डिबगिंग के लिए उपयोगी संदर्भ प्रदान कर सकने वाले रन के अतिरिक्त गुण देखने के लिए `Metadata` टैब पर क्लिक करें।

      ![Agent](../../../../../translated_images/hi/11-view-run-info-metadata.7966986122c7c2df.webp)


1. एजेंट प्रतिक्रिया पर किए गए स्वचालित आकलन देखने के लिए `Evaluations` टैब पर क्लिक करें। इनमें सुरक्षा मूल्यांकन (जैसे, Self-harm) और एजेंट-विशिष्ट मूल्यांकन (जैसे, Intent resolution, Task adherence) शामिल हैं।

      ![Agent](../../../../../translated_images/hi/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. अंत में, साइडबार मेनू में `Monitoring` टैब पर क्लिक करें।

      - प्रदर्शित पेज में `Resource usage` टैब चुनें - और मीट्रिक्स देखें।
      - लागत (टोकन) और लोड (रिक्वेस्ट) के रूप में एप्लिकेशन उपयोग को ट्रैक करें।
      - पहले बाइट (इनपुट प्रोसेसिंग) और अंतिम बाइट (आउटपुट) के लिए एप्लिकेशन लेटेंसी को ट्रैक करें।

## 8. पर्यावरण चरों (Environment Variables)

अब तक, हमने ब्राउज़र में डिप्लॉयमेंट देखी है - और वैलिडेट किया है कि हमारा इन्फ्रास्ट्रक्चर प्रोविजन किया गया है और एप्लिकेशन कार्यरत है। लेकिन कोड-पहले (_code-first_) काम करने के लिए, हमें अपनी स्थानीय विकास पर्यावरण को उन उपयुक्त चरों से कॉन्फ़िगर करना होगा जो इन संसाधनों के साथ काम करने के लिए आवश्यक हैं। `azd` का उपयोग यह आसान बनाता है।

1. Azure Developer CLI [पर्यावरण चरों](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) का उपयोग एप्लिकेशन डिप्लॉयमेंट के लिए कॉन्फ़िगरेशन सेटिंग्स संग्रहित और प्रबंधित करने के लिए करता है।

1. पर्यावरण चर `.azure/<env-name>/.env` में संग्रहित होते हैं - यह इन्हें उस `env-name` एनवायरनमेंट तक सीमित करता है जो डिप्लॉयमेंट के दौरान उपयोग किया गया था और आपको एक ही रिपॉजिटरी में विभिन्न डिप्लॉयमेंट लक्ष्यों के बीच एनवायरनमेंट को पृथक करने में मदद करता है।

1. `azd` कमांड जब भी एक विशिष्ट कमांड (जैसे, `azd up`) निष्पादित करता है, पर्यावरण चर स्वचालित रूप से लोड होते हैं। ध्यान दें कि `azd` स्वचालित रूप से _OS-स्तरीय_ पर्यावरण चर (जैसे, शेल में सेट किए गए) को नहीं पढ़ता - इसके बजाय स्क्रिप्ट्स के अंदर जानकारी स्थानांतरित करने के लिए `azd set env` और `azd get env` का उपयोग करें।


कुछ कमांड्स आज़माएं:

1. इस एनवायरनमेंट में `azd` के लिए सेट सभी पर्यावरण चर प्राप्त करें:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      आपको कुछ ऐसा दिखाई देगा:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. एक विशिष्ट मान प्राप्त करें - उदाहरण के लिए, मैं जानना चाहता हूँ कि क्या हमने `AZURE_AI_AGENT_MODEL_NAME` मान सेट किया है

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      आपको कुछ ऐसा दिखाई देगा - यह डिफ़ॉल्ट रूप से सेट नहीं था!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd` के लिए नया पर्यावरण चर सेट करें। यहाँ, हम एजेंट मॉडल नाम अपडेट कर रहे हैं। _ध्यान दें: कोई भी परिवर्तन तुरंत `.azure/<env-name>/.env` फाइल में परिलक्षित होगा।_

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      अब, हमें मान सेट दिखना चाहिए:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. ध्यान दें कि कुछ संसाधन स्थायी होते हैं (जैसे, मॉडल डिप्लॉयमेंट) और उन्हें पुनः डिप्लॉय करने के लिए केवल `azd up` से अधिक प्रयास की आवश्यकता हो सकती है। चलिए मूल डिप्लॉयमेंट को हटाकर परिवर्तित एनवायरनमेंट वेरिएबल्स के साथ पुनः डिप्लॉय करने की कोशिश करते हैं।

1. **रिफ्रेश** यदि आपने पहले कभी azd टेम्पलेट का उपयोग करके इन्फ्रास्ट्रक्चर डिप्लॉय किया है - तो आप इस कमांड के माध्यम से अपने स्थानीय एनवायरनमेंट वेरिएबल्स की वर्तमान Azure डिप्लॉयमेंट स्थिति के आधार पर स्थिति _रिफ्रेश_ कर सकते हैं:

      ```bash title="" linenums="0"
      azd env refresh
      ```

यह एक शक्तिशाली तरीका है दो या अधिक स्थानीय विकास पर्यावरणों (जैसे, कई डेवलपर्स वाली टीम) के बीच पर्यावरण चर को _sync_ करने का - जिससे तैनात अवसंरचना को env चर की स्थिति के लिए वास्तविक आधार के रूप में सेवा देने की अनुमति मिलती है। टीम के सदस्य बस चर _refresh_ करते हैं ताकि वे फिर से सिंक में आ सकें।
