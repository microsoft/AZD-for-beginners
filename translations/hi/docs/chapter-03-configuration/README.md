# अध्याय 3: कॉन्फ़िगरेशन और प्रमाणीकरण

**📚 पाठ्यक्रम**: [AZD For Beginners](../../README.md) | **⏱️ अवधि**: 45-60 मिनट | **⭐ जटिलता**: मध्यम

---

## अवलोकन

यह अध्याय Azure Developer CLI परिनियोजन के लिए वातावरण कॉन्फ़िगरेशन, प्रमाणीकरण पैटर्न, और सुरक्षा सर्वोत्तम प्रथाओं को कवर करता है।

## सीखने के उद्देश्य

By completing this chapter, you will:
- AZD कॉन्फ़िगरेशन पदानुक्रम में महारत हासिल करें
- एकाधिक वातावरणों (dev, staging, prod) का प्रबंधन करें
- प्रबंधित आइडेंटिटीज़ के साथ सुरक्षित प्रमाणीकरण लागू करें
- वातावरण-विशिष्ट सेटिंग्स कॉन्फ़िगर करें

## 🚀 त्वरित आरंभ

```bash
# कई पर्यावरण बनाएँ
azd env new dev
azd env new staging
azd env new prod

# पर्यावरण बदलें
azd env select prod

# पर्यावरण चर सेट करें
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# विन्यास देखें
azd env get-values
```

---

## 🔧 कॉन्फ़िगरेशन पदानुक्रम

AZD निम्न क्रम में सेटिंग्स लागू करता है (बाद वाली पहले वाली पर ओवरराइड करती है):

1. **डिफ़ॉल्ट मान** (टेम्पलेट में निर्मित)
2. **azure.yaml** (प्रोजेक्ट कॉन्फ़िगरेशन)
3. **पर्यावरण चर** (`azd env set`)
4. **कमांड-लाइन फ़्लैग्स** (`--location eastus`)

---

## 🔐 सुरक्षा सर्वोत्तम प्रथाएँ

```bash
# प्रबंधित पहचान का उपयोग करें (अनुशंसित)
azd env set AZURE_USE_MANAGED_IDENTITY true

# प्रमाणीकरण स्थिति की जाँच करें
azd auth whoami
az account show

# यदि आवश्यक हो तो पुनः प्रमाणीकरण करें
azd auth login
az login
```
