# Chapter 3: कॉन्फ़िगरेशन और प्रमाणीकरण

**📚 कोर्स**: [AZD फॉर बिगिनर्स](../../README.md) | **⏱️ अवधि**: 45-60 मिनट | **⭐ जटिलता**: मध्यवर्ती

---

## अवलोकन

यह अध्याय Azure Developer CLI तैनाती के लिए पर्यावरण कॉन्फ़िगरेशन, प्रमाणीकरण पैटर्न और सुरक्षा सर्वोत्तम अभ्यास को कवर करता है।

> मार्च 2026 में `azd 1.23.12` के खिलाफ सत्यापित।

## अध्ययन उद्देश्य

इस अध्याय को पूरा करने पर, आप:
- AZD कॉन्फ़िगरेशन पदानुक्रम में महारत हासिल करेंगे
- कई पर्यावरणों (डेव, स्टेजिंग, प्रोड) का प्रबंधन करेंगे
- प्रबंधित पहचान के साथ सुरक्षित प्रमाणीकरण लागू करेंगे
- पर्यावरण-विशिष्ट सेटिंग्स को कॉन्फ़िगर करेंगे

## 🚀 त्वरित प्रारंभ

```bash
# कई वातावरण बनाएं
azd env new dev
azd env new staging
azd env new prod

# वातावरण बदलें
azd env select prod

# वातावरण चर सेट करें
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# कॉन्फ़िगरेशन देखें
azd env get-values
```

---

## 🔧 कॉन्फ़िगरेशन पदानुक्रम

AZD इस क्रम में सेटिंग्स लागू करता है (बाद वाला पहले वाले को ओवरराइड करता है):

1. **डिफ़ॉल्ट मान** (टेम्पलेट में बिल्ट-इन)
2. **azure.yaml** (प्रोजेक्ट कॉन्फ़िगरेशन)
3. **पर्यावरण चर** (`azd env set`)
4. **कमान्ड-लाइन फ्लैग** (`--location eastus`)

## 🔐 सुरक्षा सर्वोत्तम अभ्यास

```bash
# प्रबंधित पहचान का उपयोग करें (सिफारिश की गई)
azd env set AZURE_USE_MANAGED_IDENTITY true

# AZD प्रमाणीकरण स्थिति जांचें
azd auth status

# वैकल्पिक: यदि आप az कमांड चलाने की योजना बनाते हैं तो Azure CLI संदर्भ सत्यापित करें
az account show

# आवश्यकता हो तो पुनः प्रमाणीकरण करें
azd auth login

# वैकल्पिक: az कमांड के लिए Azure CLI प्रमाणीकरण को ताज़ा करें
az login
```
