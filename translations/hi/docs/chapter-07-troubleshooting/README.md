# अध्याय 7: समस्या निवारण और डिबगिंग

**📚 कोर्स**: [AZD फॉर बिगिनर्स](../../README.md) | **⏱️ अवधि**: 1-1.5 घंटे | **⭐ जटिलता**: मध्यवर्ती

---

## अवलोकन

यह अध्याय आपको Azure Developer CLI के साथ काम करते समय सामान्य समस्याओं का निदान और समाधान करने में मदद करता है। तैनाती विफलताओं से लेकर AI-विशिष्ट समस्याओं तक।

> मार्च 2026 में `azd 1.23.12` के खिलाफ मान्य किया गया।

## अध्ययन उद्देश्य

इस अध्याय को पूरा करने के बाद, आप:
- सामान्य AZD तैनाती विफलताओं का निदान कर पाएंगे
- प्रमाणीकरण और अनुमति समस्याओं का डिबग कर पाएंगे
- AI सेवा कनेक्टिविटी समस्याओं को हल कर पाएंगे
- समस्या निवारण के लिए Azure पोर्टल और CLI का उपयोग कर सकेंगे

## 🚨 त्वरित समाधान

### प्रमाणीकरण समस्याएं
```bash
# AZD वर्कफ़्लोज़ के लिए आवश्यक
azd auth login

# वैकल्पिक यदि आप सीधे Azure CLI कमांड भी उपयोग कर रहे हैं
az login

azd auth status
```

### प्रावधान विफलताएँ
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### संसाधन संघर्ष
```bash
azd down --force --purge
azd env new different-name
azd up
```

### कोटा पार
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 त्रुटि कोड संदर्भ

| त्रुटि | कारण | समाधान |
|-------|-------|----------|
| `AuthenticationError` | लॉग इन नहीं हुआ | `azd auth login` |
| `ResourceNotFound` | संसाधन नहीं मिला | संसाधन नाम जांचें |
| `QuotaExceeded` | सदस्यता सीमाएं | कोटा बढ़ाने का अनुरोध करें |
| `InvalidTemplate` | बाइसप सिंटेक्स त्रुटि | `az bicep build` |
| `Conflict` | संसाधन पहले से मौजूद है | नया नाम उपयोग करें या हटाएँ |
| `Forbidden` | अपर्याप्त अनुमतियाँ | RBAC भूमिकाएँ जांचें |

## 🔄 रीसेट और पुनर्प्राप्ति

```bash
# सॉफ्ट रीसेट (संसाधनों को रखें, कोड को पुनः तैनात करें)
azd deploy --force

# हार्ड रीसेट (सब कुछ मिटाएं, नया शुरुआत करें)
azd down --force --purge
azd up
```
