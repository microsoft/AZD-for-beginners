# अध्याय 7: समस्या निवारण और डिबगिंग

**📚 पाठ्यक्रम**: [AZD शुरुआती लोगों के लिए](../../README.md) | **⏱️ अवधि**: 1-1.5 घंटे | **⭐ जटिलता**: मध्यम

---

## अवलोकन

यह अध्याय Azure Developer CLI के साथ काम करते समय अक्सर आने वाली समस्याओं का निदान और समाधान करने में आपकी मदद करता है। तैनाती विफलताओं से लेकर AI-विशिष्ट समस्याओं तक।

## सीखने के उद्देश्य

इस अध्याय को पूरा करने पर, आप:
- सामान्य AZD डिप्लॉयमेंट विफलताओं का निदान करना
- प्रमाणीकरण और अनुमति समस्याओं को डिबग करना
- AI सेवा कनेक्टिविटी समस्याओं को हल करना
- समस्या निवारण के लिए Azure पोर्टल और CLI का उपयोग करना

## 🚨 त्वरित समाधान

### प्रमाणीकरण समस्याएँ
```bash
azd auth login
az login
azd auth whoami
```

### प्रोविज़निंग विफलताएँ
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### संसाधन संघर्ष
```bash
azd down --force --purge
azd env new different-name
azd up
```

### कोटा पार
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 त्रुटि कोड संदर्भ

| त्रुटि | कारण | समाधान |
|-------|-------|----------|
| `AuthenticationError` | लॉग इन नहीं किया गया | `azd auth login` |
| `ResourceNotFound` | संसाधन मौजूद नहीं है | संसाधन के नाम जांचें |
| `QuotaExceeded` | सब्सक्रिप्शन सीमाएँ | कोटा वृद्धि का अनुरोध करें |
| `InvalidTemplate` | Bicep सिन्टैक्स त्रुटि | `az bicep build` |
| `Conflict` | संसाधन पहले से मौजूद है | नया नाम उपयोग करें या हटाएँ |
| `Forbidden` | पर्याप्त अनुमतियाँ नहीं हैं | RBAC भूमिका जांचें |

## 🔄 रीसेट और पुनर्प्राप्ति

```bash
# सॉफ्ट रीसेट (संसाधनों को बनाए रखें, कोड को पुनः परिनियोजित करें)
azd deploy --force

# हार्ड रीसेट (सब कुछ हटाएँ, नई शुरुआत करें)
azd down --force --purge
azd up
```
