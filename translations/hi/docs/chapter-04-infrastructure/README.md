# अध्याय 4: इन्फ्रास्ट्रक्चर एज़ कोड और डिप्लॉयमेंट

**📚 कोर्स**: [आरंभ करने वालों के लिए AZD](../../README.md) | **⏱️ अवधि**: 1-1.5 घंटे | **⭐ जटिलता**: मध्यम

---

## अवलोकन

यह अध्याय Bicep टेम्पलेट्स के साथ इन्फ्रास्ट्रक्चर एज़ कोड (IaC) पैटर्न, संसाधन प्रोविजनिंग, और Azure डेवलपर CLI का उपयोग करके डिप्लॉयमेंट रणनीतियों को कवर करता है।

> जुलाई 2026 में `azd 1.27.1` के खिलाफ सत्यापित।

## सीखने के उद्देश्य

इस अध्याय को पूरा करके, आप:
- Bicep टेम्पलेट संरचना और वाक्य-विन्यास को समझेंगे
- `azd provision` के साथ Azure संसाधनों का Provision करेंगे
- `azd deploy` के साथ एप्लिकेशन डिप्लॉय करेंगे
- ब्लू-ग्रीन और रोलिंग डिप्लॉयमेंट रणनीतियों को लागू करेंगे

## 🚀 त्वरित प्रारंभ

```bash
# टेम्पलेट से प्रारंभ करें
azd init --template azure-functions-python-v2-http

# जो बनाया जाएगा उसका पूर्वावलोकन करें
azd provision --preview

# केवल अवसंरचना का प्रावधान करें
azd provision

# केवल कोड तैनात करें
azd deploy

# या दोनों को एक साथ करें
azd up
```

## 📁 AZD परियोजना संरचना

```
my-project/
├── azure.yaml            # ← Project definition
├── infra/                # ← Infrastructure as Code
│   ├── main.bicep        # Entry point
│   ├── main.parameters.json
│   └── modules/          # Reusable components
│       ├── web.bicep
│       └── database.bicep
└── src/                  # Application code
    └── api/
```

## 🔧 आवश्यक कमांड

| कमांड | विवरण |
|---------|-------------|
| `azd init` | परियोजना प्रारंभ करें |
| `azd provision` | Azure संसाधन बनाएं |
| `azd deploy` | एप्लिकेशन कोड डिप्लॉय करें |
| `azd up` | प्रोविजन + डिप्लॉय |
| `azd down` | सभी संसाधन हटाएं |
