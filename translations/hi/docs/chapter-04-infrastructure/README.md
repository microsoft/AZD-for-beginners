# अध्याय 4: कोड के रूप में इन्फ्रास्ट्रक्चर और तैनाती

**📚 पाठ्यक्रम**: [शुरुआती के लिए AZD](../../README.md) | **⏱️ अवधि**: 1-1.5 घंटे | **⭐ जटिलता**: मध्यवर्ती

---

## अवलोकन

यह अध्याय Bicep टेम्पलेट्स के साथ कोड के रूप में इन्फ्रास्ट्रक्चर (IaC) पैटर्न, संसाधन प्रावधान, और Azure Developer CLI का उपयोग करके तैनाती रणनीतियों को कवर करता है।

> मार्च 2026 में `azd 1.23.12` के खिलाफ सत्यापित।

## अध्ययन के उद्देश्य

इस अध्याय को पूरा करके, आप:
- Bicep टेम्पलेट संरचना और सिंटैक्स को समझेंगे
- `azd provision` के साथ Azure संसाधनों का प्रावधान करेंगे
- `azd deploy` के साथ एप्लिकेशन तैनात करेंगे
- ब्लू-ग्रीन और रोलिंग तैनाती रणनीतियों को लागू करेंगे

## 🚀 त्वरित शुरुआत

```bash
# टेम्पलेट से प्रारंभ करें
azd init --template azure-functions-python-v2-http

# क्या बनाया जाएगा उसका पूर्वावलोकन करें
azd provision --preview

# केवल बुनियादी ढांचे की व्यवस्था करें
azd provision

# केवल कोड तैनात करें
azd deploy

# या दोनों एक साथ
azd up
```

## 📁 AZD परियोजना संरचना

```
my-project/
├── azure.yaml            # ← Project definition
├── infra/                # ← Infrastructure as Code
│   ├── main.bicep        # Entry point
│   ├── main.parameters.json
│   └── modules/          # Reusable components
│       ├── web.bicep
│       └── database.bicep
└── src/                  # Application code
    └── api/
```

## 🔧 आवश्यक कमांड

| कमांड | विवरण |
|---------|-------------|
| `azd init` | परियोजना प्रारंभ करें |
| `azd provision` | Azure संसाधन बनाएं |
| `azd deploy` | एप्लिकेशन कोड तैनात करें |
| `azd up` | प्रावधान + तैनाती |
| `azd down` | सभी संसाधन हटाएं |
