# अध्याय 6: प्री-डिप्लॉयमेंट योजना और सत्यापन

**📚 कोर्स**: [AZD For Beginners](../../README.md) | **⏱️ अवधि**: 1 घंटा | **⭐ जटिलता**: मध्यवर्ती

---

## अवलोकन

यह अध्याय आपके एप्लिकेशन को डिप्लॉय करने से पहले आवश्यक योजना और सत्यापन चरणों को कवर करता है। उचित क्षमता योजना, SKU चयन, और प्रीफ्लाइट चेक के साथ महंगे गलतियों से बचना सीखें।

> `azd 1.27.1` के खिलाफ जुलाई 2026 में सत्यापित।

## सीखने के उद्देश्य

इस अध्याय को पूरा करके, आप:
- डिप्लॉयमेंट से पहले प्रीफ्लाइट चेक चलाएंगे
- क्षमता की योजना बनाएंगे और संसाधन आवश्यकताओं का अनुमान लगाएंगे
- लागत अनुकूलन के लिए उपयुक्त SKUs का चयन करेंगे
- निगरानी के लिए एप्लिकेशन इंसाइट्स कॉन्फ़िगर करेंगे
- टीम समन्वय पैटर्न को समझेंगे

## 🚀 त्वरित प्रारंभ

```bash
# सब्सक्रिप्शन कोटा जांचें
az vm list-usage --location eastus --output table

# तैनाती पूर्वावलोकन (कोई संसाधन नहीं बनाए गए)
azd provision --preview

# बाइसेप सिंटैक्स मान्य करें
az bicep build --file infra/main.bicep

# पर्यावरण विन्यास जांचें
azd env get-values
```

## ☑️ प्री-डिप्लॉयमेंट चेकलिस्ट

### `azd provision` से पहले

- [ ] क्षेत्र के लिए कोटा सत्यापित करें
- [ ] SKUs उपयुक्त रूप से चुने गए हैं
- [ ] लागत अनुमान की समीक्षा की गई है
- [ ] नामकरण प्रथा सुसंगत है
- [ ] सुरक्षा/RBAC कॉन्फ़िगर किया गया है

### `azd deploy` से पहले

- [ ] परिवेश चर सेट किए गए हैं
- [ ] की वॉल्ट में गुप्त जानकारी है
- [ ] कनेक्शन स्ट्रिंग्स सत्यापित हैं
- [ ] स्वास्थ्य जांच कॉन्फ़िगर की गई है
