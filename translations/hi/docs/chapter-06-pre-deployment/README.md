## अवलोकन

यह अध्याय आपके एप्लिकेशन को परिनियोजित करने से पहले आवश्यक योजना और सत्यापन चरणों को कवर करता है। उचित क्षमता योजना, SKU चयन, और पूर्व-परिनियोजन जाँच के साथ महंगी गलतियों से बचना सीखें।

> जून 2026 में `azd 1.25.6` के खिलाफ सत्यापित।

## सीखने के उद्देश्य

इस अध्याय को पूरा करने पर, आप:
- परिनियोजन से पहले पूर्व-परिनियोजन जाँच चलाएँ
- क्षमता की योजना बनाएं और संसाधन आवश्यकताओं का अनुमान लगाएँ
- लागत अनुकूलन के लिए उपयुक्त SKU चुनें
- मॉनिटरिंग के लिए Application Insights कॉन्फ़िगर करें
- टीम समन्वय पैटर्न समझें

## 🚀 त्वरित प्रारम्भ

```bash
# सदस्यता कोटाएँ जांचें
az vm list-usage --location eastus --output table

# डिप्लॉयमेंट का पूर्वावलोकन (कोई संसाधन बनाए नहीं जाएंगे)
azd provision --preview

# Bicep वाक्यविन्यास सत्यापित करें
az bicep build --file infra/main.bicep

# पर्यावरण विन्यास जांचें
azd env get-values
```

## ☑️ परिनियोजन पूर्व चेकलिस्ट

### `azd provision` से पहले

- [ ] क्षेत्र के लिए कोटा सत्यापित किया गया
- [ ] SKUs उपयुक्त रूप से चुने गए
- [ ] लागत अनुमान की समीक्षा की गई
- [ ] नामकरण कन्वेंशन सुसंगत है
- [ ] सुरक्षा/RBAC कॉन्फ़िगर किया गया

### `azd deploy` से पहले

- [ ] पर्यावरण वेरिएबल्स सेट किए गए
- [ ] Key Vault में Secrets
- [ ] कनेक्शन स्ट्रिंग्स सत्यापित की गईं
- [ ] हेल्थ चेक्स कॉन्फ़िगर किए गए
