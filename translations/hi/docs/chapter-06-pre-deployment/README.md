# अध्याय 6: प्री-डिप्लॉयमेंट योजना और सत्यापन

**📚 पाठ्यक्रम**: [AZD फ्रॉर बिगिनर्स](../../README.md) | **⏱️ अवधि**: 1 घंटा | **⭐ कठिनाई**: मध्यवर्ती

---

## अवलोकन

यह अध्याय आपके एप्लिकेशन को डिप्लॉय करने से पहले आवश्यक योजना और सत्यापन चरणों को कवर करता है। उचित क्षमता योजना, SKU चयन, और प्रीफ्लाइट जांच के साथ महंगे गलतियों से बचना सीखें।

> मार्च 2026 में `azd 1.23.12` के विरुद्ध सत्यापित।

## सीखने के उद्देश्य

इस अध्याय को पूरा करके, आप:
- डिप्लॉयमेंट से पहले प्रीफ्लाइट जांच चलाएंगे
- क्षमता की योजना बनाएंगे और संसाधन आवश्यकताओं का अनुमान लगाएंगे
- लागत अनुकूलन के लिए उपयुक्त SKU चुनेंगे
- निगरानी के लिए एप्लिकेशन इंसाइट्स कॉन्फ़िगर करेंगे
- टीम समन्वय पैटर्न को समझेंगे

## 🚀 त्वरित प्रारंभ

```bash
# सदस्यता कोटा जांचें
az vm list-usage --location eastus --output table

# तैनाती पूर्वावलोकन (कोई संसाधन नहीं बनाए गए)
azd provision --preview

# बाइसैप सिंटैक्स मान्य करें
az bicep build --file infra/main.bicep

# पर्यावरण विन्यास जांचें
azd env get-values
```

## ☑️ प्री-डिप्लॉयमेंट चेकलिस्ट

### `azd provision` से पहले

- [ ] क्षेत्र के लिए कोटा सत्यापित
- [ ] SKU उचित रूप से चयनित
- [ ] लागत अनुमान की समीक्षा की गई
- [ ] नामकरण कन्वेंशन सुसंगत
- [ ] सुरक्षा/RBAC कॉन्फ़िगर किया गया

### `azd deploy` से पहले

- [ ] पर्यावरणीय चर सेट किए गए
- [ ] Key Vault में गुप्त जानकारी
- [ ] कनेक्शन स्ट्रिंग्स की पुष्टि की गई
- [ ] हेल्थ चेक्स कॉन्फ़िगर किए गए
