# शब्दावली - Azure और AZD शब्दावली

**सभी अध्यायों के लिए संदर्भ**
- **📚 कोर्स होम**: [AZD For Beginners](../README.md)
- **📖 आधार सीखें**: [Chapter 1: AZD Basics](../docs/getting-started/azd-basics.md)
- **🤖 एआई शब्दावली**: [Chapter 2: AI-First Development](../docs/microsoft-foundry/microsoft-foundry-integration.md)

## परिचय

यह विस्तृत शब्दावली Azure Developer CLI और Azure क्लाउड डेवलपमेंट में उपयोग किए जाने वाले शब्दों, अवधारणाओं और संक्षेपाक्षरों की परिभाषाएँ प्रदान करती है। तकनीकी दस्तावेज़ों को समझने, समस्याओं का निवारण करने, और azd प्रोजेक्ट्स और Azure सेवाओं के बारे में प्रभावी ढंग से संवाद करने के लिए आवश्यक संदर्भ।

## सीखने के लक्ष्य

इस शब्दावली का उपयोग करके, आप:
- Azure Developer CLI की आवश्यक शब्दावली और अवधारणाओं को समझेंगे
- Azure क्लाउड विकास शब्दावली और तकनीकी शब्दों में निपुणता हासिल करेंगे
- Infrastructure as Code और डिप्लॉयमेंट शब्दावली का कुशलतापूर्वक संदर्भ लेंगे
- Azure सेवा नामों, संक्षेपाक्षरों, और उनके उद्देश्य को समझेंगे
- ट्रबलशूटिंग और डिबगिंग शब्दावली के लिए परिभाषाएँ प्राप्त करेंगे
- उन्नत Azure आर्किटेक्चर और विकास अवधारणाओं को सीखेंगे

## सीखने के परिणाम

नियमित संदर्भ के साथ इस शब्दावली का उपयोग करके, आप सक्षम होंगे:
- उचित Azure Developer CLI शब्दावली का उपयोग कर प्रभावी संवाद करना
- तकनीकी दस्तावेज़ों और त्रुटि संदेशों को अधिक स्पष्ट रूप से समझना
- आत्मविश्वास के साथ Azure सेवाओं और अवधारणाओं के माध्यम से नेविगेट करना
- उपयुक्त तकनीकी शब्दावली का उपयोग करते हुए समस्याओं का निवारण करना
- सटीक तकनीकी भाषा के साथ टीम चर्चाओं में योगदान देना
- व्यवस्थित रूप से अपना Azure क्लाउड विकास ज्ञान बढ़ाना

## A

**ARM Template**  
Azure Resource Manager टेम्पलेट। JSON-आधारित Infrastructure as Code फ़ॉर्मेट जो Azure संसाधनों को डिक्लेरेटिव तरीके से परिभाषित और डिप्लॉय करने के लिए उपयोग होता है।

**App Service**  
वेब अनुप्रयोगों, REST API, और मोबाइल बैकएंड्स को बिना इंफ्रास्ट्रक्चर प्रबंधित किए होस्ट करने के लिए Azure का platform-as-a-service (PaaS) प्रस्ताव।

**Application Insights**  
Azure की एप्लिकेशन प्रदर्शन निगरानी (APM) सेवा जो एप्लिकेशन के प्रदर्शन, उपलब्धता, और उपयोग पर गहरी जानकारी प्रदान करती है।

**Azure CLI**  
Azure संसाधनों के प्रबंधन के लिए कमांड-लाइन इंटरफ़ेस। azd द्वारा प्रमाणीकरण और कुछ ऑपरेशनों के लिए उपयोग किया जाता है।

**Azure Developer CLI (azd)**  
डेवलपर-केंद्रित कमांड-लाइन टूल जो टेम्पलेट्स और Infrastructure as Code का उपयोग करके Azure पर अनुप्रयोगों को बनाने और डिप्लॉय करने की प्रक्रिया को तेज़ करता है।

**azure.yaml**  
azd प्रोजेक्ट के लिए मुख्य कॉन्फ़िगरेशन फ़ाइल जो सेवाओं, इंफ्रास्ट्रक्चर, और डिप्लॉयमेंट हुक्स को परिभाषित करती है।

**Azure Resource Manager (ARM)**  
Azure की डिप्लॉयमेंट और प्रबंधन सेवा जो संसाधनों को बनाने, अपडेट करने, और हटाने के लिए एक प्रबंधन परत प्रदान करती है।

## B

**Bicep**  
Microsoft द्वारा विकसित डोमेन-विशेष भाषा (DSL) जो Azure संसाधनों को डिप्लॉय करने के लिए है। यह ARM टेम्पलेट्स की तुलना में सरल सिंटैक्स प्रदान करती है और ARM में कम्पाइल होती है।

**Build**  
सोर्स कोड को संकलित करने, निर्भरताएँ स्थापित करने, और अनुप्रयोगों को डिप्लॉयमेंट के लिए तैयार करने की प्रक्रिया।

**Blue-Green Deployment**  
डिप्लॉयमेंट रणनीति जो डाउनटाइम और जोखिम को कम करने के लिए दो समान प्रोडक्शन वातावरणों (blue और green) का उपयोग करती है।

## C

**Container Apps**  
Azure की सर्वरलेस कंटेनर सेवा जो जटिल इंफ्रास्ट्रक्चर प्रबंधित किए बिना कंटेनरयुक्त अनुप्रयोगों को चलाने में सक्षम बनाती है।

**CI/CD**  
Continuous Integration/Continuous Deployment। कोड परिवर्तनों को एकीकृत करने और अनुप्रयोगों को डिप्लॉय करने के लिये स्वचालित प्रथाएँ।

**Cosmos DB**  
Azure की वैश्विक रूप से वितरित, मल्टी-मॉडल डेटाबेस सेवा जो थ्रूपुट, लेटेंसी, उपलब्धता, और कंसिस्टेंसी के लिए व्यापक SLA प्रदान करती है।

**Configuration**  
सेटिंग्स और पैरामीटर जो अनुप्रयोग व्यवहार और डिप्लॉयमेंट विकल्पों को नियंत्रित करते हैं।

## D

**Deployment**  
लक्षित इंफ्रास्ट्रक्चर पर अनुप्रयोगों और उनकी निर्भरताओं को इंस्टॉल और कॉन्फ़िगर करने की प्रक्रिया।

**Docker**  
कंटेनराइज़ेशन तकनीक का उपयोग करके अनुप्रयोगों को विकसित, शिप, और चलाने के लिए प्लेटफ़ॉर्म।

**Dockerfile**  
डॉकर कंटेनर इमेज बनाने के निर्देशों वाली टेक्स्ट फ़ाइल।

## E

**Environment**  
एक डिप्लॉयमेंट लक्ष्य जो आपके अनुप्रयोग के एक विशिष्ट इंस्टेंस का प्रतिनिधित्व करता है (उदा., development, staging, production)।

**Environment Variables**  
कन्फ़िगरेशन मान जो key-value जोड़े के रूप में संग्रहीत होते हैं और आवेदन रनटाइम पर इन्हें एक्सेस कर सकता है।

**Endpoint**  
URL या नेटवर्क पता जहाँ कोई अनुप्रयोग या सेवा एक्सेस की जा सकती है।

## F

**Function App**  
Azure की सर्वरलेस कम्प्यूट सेवा जो इंफेंट-ड्रिवन कोड चलाने में सक्षम बनाती है बिना इंफ्रास्ट्रक्चर प्रबंधित किए।

## G

**GitHub Actions**  
GitHub रिपॉज़िटरीज़ के साथ एकीकृत CI/CD प्लेटफ़ॉर्म जो वर्कफ़्लो को स्वचालित करता है।

**Git**  
डिस्ट्रिब्यूटेड वर्शन कंट्रोल सिस्टम जो सोर्स कोड में परिवर्तनों को ट्रैक करने के लिए उपयोग होता है।

## H

**Hooks**  
कस्टम स्क्रिप्ट या कमांड जो डिप्लॉयमेंट लाइफसाइकल के विशिष्ट बिंदुओं पर चलते हैं (preprovision, postprovision, predeploy, postdeploy)।

**Host**  
Azure सेवा प्रकार जहाँ एक एप्लिकेशन डिप्लॉय किया जाएगा (उदा., appservice, containerapp, function)।

## I

**Infrastructure as Code (IaC)**  
इन्फ्रास्ट्रक्चर को मैन्युअल प्रक्रियाओं के बजाय कोड के माध्यम से परिभाषित और प्रबंधित करने का अभ्यास।

**Init**  
एक नए azd प्रोजेक्ट को इनिशियलाइज़ करने की प्रक्रिया, आमतौर पर किसी टेम्पलेट से।

## J

**JSON**  
JavaScript Object Notation। डेटा इंटरचेंज फ़ॉर्मेट जो कॉन्फ़िगरेशन फ़ाइलों और API रिस्पॉन्स के लिए सामान्यतः उपयोग होता है।

**JWT**  
JSON Web Token। पार्टियों के बीच सुरक्षित रूप से जानकारी ट्रान्समिट करने के लिए JSON ऑब्जेक्ट के रूप में मानक।

## K

**Key Vault**  
Azure की सेवा जो सीक्रेट्स, कीज़, और सर्टिफिकेट्स को सुरक्षित रूप से संग्रहीत और प्रबंधित करती है।

**Kusto Query Language (KQL)**  
Azure Monitor, Application Insights, और अन्य Azure सेवाओं में डेटा का विश्लेषण करने के लिए उपयोग की जाने वाली क्वेरी भाषा।

## L

**Load Balancer**  
सेवा जो इनकमिंग नेटवर्क ट्रैफ़िक को कई सर्वरों या इंस्टेंसेज़ में वितरित करती है।

**Log Analytics**  
क्लाउड और ऑन-प्रिमाइसेस परिवेशों से टेलीमेट्री डेटा एकत्र करने, विश्लेषण करने, और उस पर कार्रवाई करने की Azure सेवा।

## M

**Managed Identity**  
Azure सुविधा जो अन्य Azure सेवाओं के लिए प्रमाणिकरण हेतु स्वचालित रूप से प्रबंधित पहचान प्रदान करती है।

**Microservices**  
वह आर्किटेक्चरल दृष्टिकोण जहाँ अनुप्रयोगों को छोटे, स्वतंत्र सेवाओं के संग्रह के रूप में बनाया जाता है।

**Monitor**  
Azure का एकीकृत मॉनिटरिंग समाधान जो अनुप्रयोगों और इंफ्रास्ट्रक्चर पर फुल-स्टैक ऑब्ज़रवेबिलिटी प्रदान करता है।

## N

**Node.js**  
Chrome के V8 JavaScript इंजन पर निर्मित JavaScript रनटाइम जो सर्वर-साइड अनुप्रयोगों के लिए उपयोग होता है।

**npm**  
Node.js के लिए पैकेज मैनेजर जो निर्भरताओं और पैकेजों का प्रबंधन करता है।

## O

**Output**  
इन्फ्रास्ट्रक्चर डिप्लॉयमेंट से लौटाए गए मान जिन्हें अनुप्रयोगों या अन्य संसाधनों द्वारा उपयोग किया जा सकता है।

## P

**Package**  
अनुप्रयोग कोड और निर्भरताओं को डिप्लॉयमेंट के लिए तैयार करने की प्रक्रिया।

**Parameters**  
इन्फ्रास्ट्रक्चर टेम्पलेट्स में अनुकूलन हेतु पास किए गए इनपुट मान।

**PostgreSQL**  
ओपन-सोर्स रिलेशनल डेटाबेस सिस्टम जिसे Azure में एक मैनेज्ड सेवा के रूप में सपोर्ट किया जाता है।

**Provisioning**  
इन्फ्रास्ट्रक्चर टेम्पलेट्स में परिभाषित Azure संसाधनों को बनाने और कॉन्फ़िगर करने की प्रक्रिया।

## Q

**Quota**  
किसी Azure सब्सक्रिप्शन या क्षेत्र में बनाई जा सकने वाली संसाधनों की सीमा।

## R

**Resource Group**  
Azure संसाधनों का तार्किक कंटेनर जिनका लाइफसाइकल, अनुमतियाँ, और नीतियाँ साझा होती हैं।

**Resource Token**  
azd द्वारा उत्पन्न अद्वितीय स्ट्रिंग जो यह सुनिश्चित करती है कि डिप्लॉयमेंट्स के बीच संसाधन नाम यूनिक हों।

**REST API**  
HTTP विधियों का उपयोग करके नेटवर्केड एप्लिकेशन डिजाइन करने की आर्किटेक्चरल शैली।

**Rollback**  
एक एप्लिकेशन या इन्फ्रास्ट्रक्चर कॉन्फ़िगरेशन के पिछले संस्करण पर वापस जाने की प्रक्रिया।

## S

**Service**  
आपके अनुप्रयोग का एक घटक जो azure.yaml में परिभाषित होता है (उदा., web frontend, API backend, database)।

**SKU**  
Stock Keeping Unit। Azure संसाधनों के विभिन्न सर्विस टियर्स या प्रदर्शन स्तरों का प्रतिनिधित्व करता है।

**SQL Database**  
Microsoft SQL Server पर आधारित Azure की मैनेज्ड रिलेशनल डेटाबेस सेवा।

**Static Web Apps**  
सोर्स कोड रिपॉज़िटरीज़ से पूरा-स्टैक वेब अनुप्रयोग बनाने और डिप्लॉय करने के लिए Azure सेवा।

**Storage Account**  
Azure सेवा जो ब्लॉब्स, फ़ाइलें, क्यूज़, और तालिकाओं सहित डेटा ऑब्जेक्ट्स के लिए क्लाउड स्टोरेज प्रदान करती है।

**Subscription**  
Azure अकाउंट कंटेनर जो रिसोर्स ग्रुप्स और रिसोर्सेज़ को रखता है, साथ ही बिलिंग और एक्सेस प्रबंधन से जुड़ा होता है।

## T

**Template**  
पूर्व-निर्मित प्रोजेक्ट संरचना जिसमें एप्लिकेशन कोड, इन्फ्रास्ट्रक्चर परिभाषाएँ, और सामान्य परिदृश्यों के लिए कॉन्फ़िगरेशन शामिल होता है।

**Terraform**  
ओपन-सोर्स Infrastructure as Code टूल जो Azure सहित कई क्लाउड प्रदाताओं का समर्थन करता है।

**Traffic Manager**  
Azure का DNS-आधारित ट्रैफ़िक लोड बैलेंसर जो वैश्विक Azure क्षेत्रों में ट्रैफ़िक वितरित करता है।

## U

**URI**  
Uniform Resource Identifier। एक स्ट्रिंग जो किसी विशेष संसाधन की पहचान करती है।

**URL**  
Uniform Resource Locator। URI का एक प्रकार जो बताता है कि किसी संसाधन का स्थान क्या है और उसे कैसे प्राप्त किया जाए।

## V

**Virtual Network (VNet)**  
Azure के लिए प्राइवेट नेटवर्क का मौलिक निर्माण खंड, जो अलगाव और सेगमेंटेशन प्रदान करता है।

**VS Code**  
Visual Studio Code। एक लोकप्रिय कोड एडिटर जिसमें Azure और azd के साथ उत्कृष्ट एकीकरण है।

## W

**Webhook**  
विशिष्ट ईवेंट द्वारा ट्रिगर किया गया HTTP कॉलबैक, जो सामान्यतः CI/CD पाइपलाइन्स में उपयोग होता है।

**What-if**  
Azure की वह सुविधा जो डिप्लॉयमेंट द्वारा किए जाने वाले परिवर्तनों को बिना वास्तव में लागू किए दिखाती है।

## Y

**YAML**  
YAML Ain't Markup Language। मानव-पठनीय डेटा सीरियलाइज़ेशन मानक जो azure.yaml जैसी कॉन्फ़िगरेशन फ़ाइलों के लिए उपयोग होता है।

## Z

**Zone**  
Azure क्षेत्र के अंदर भौतिक रूप से अलग स्थान जो रेडंडेंसी और उच्च उपलब्धता प्रदान करते हैं।

## संदर्भ-विशिष्ट शब्द

### विकास संबंधी शब्द
- **Hot Reload**: विकास के दौरान बिना रीस्टार्ट के अनुप्रयोगों का स्वचालित अपडेट
- **Build Pipeline**: कोड बिल्ड और टेस्ट करने की स्वचालित प्रक्रिया
- **Deployment Slot**: App Service के भीतर एक स्टेजिंग पर्यावरण
- **Environment Parity**: विकास, स्टेजिंग, और प्रोडक्शन पर्यावरणों को समान रखना

### सुरक्षा संबंधी शब्द
- **Managed Identity**: प्रमाण-पत्र प्रबंधन स्वचालित रूप से प्रदान करने वाली Azure सुविधा
- **Key Vault**: सीक्रेट्स, कीज़, और सर्टिफिकेट्स के लिए सुरक्षित भंडारण
- **RBAC**: Azure संसाधनों के लिए रोल-आधारित एक्सेस कंट्रोल
- **Network Security Group**: नेटवर्क ट्रैफ़िक को नियंत्रित करने के लिए वर्चुअल फ़ायरवॉल

### निगरानी संबंधी शब्द
- **Telemetry**: मापन और डेटा का स्वचालित संग्रह
- **Application Performance Monitoring (APM)**: सॉफ़्टवेयर प्रदर्शन की निगरानी
- **Log Analytics**: लॉग डेटा एकत्र करने और विश्लेषण करने की सेवा
- **Alert Rules**: मेट्रिक्स या स्थितियों के आधार पर स्वचालित सूचनाएँ

### डिप्लॉयमेंट शब्द
- **Blue-Green Deployment**: शून्य-डाउनटाइम डिप्लॉयमेंट रणनीति
- **Canary Deployment**: उपयोगकर्ताओं के एक उपसमुच्चय के लिए क्रमिक रोलआउट
- **Rolling Update**: एप्लिकेशन इंस्टेंसेज़ के क्रमिक प्रतिस्थापन
- **Rollback**: पिछले एप्लिकेशन संस्करण पर वापस जाना
