# शब्दावली - Azure Developer CLI शब्दावली

## परिचय

यह व्यापक शब्दावली Azure Developer CLI और Azure क्लाउड विकास में उपयोग किए जाने वाले शब्दों, अवधारणाओं और संक्षेपाक्षरों की परिभाषाएँ प्रदान करती है। तकनीकी दस्तावेज़ों को समझने, समस्याओं का समाधान करने और azd प्रोजेक्ट्स और Azure सेवाओं के बारे में प्रभावी ढंग से संवाद करने के लिए आवश्यक संदर्भ।

## सीखने के लक्ष्य

इस शब्दावली का उपयोग करके, आप:
- Azure Developer CLI के आवश्यक शब्दावली और अवधारणाओं को समझेंगे
- Azure क्लाउड विकास की शब्दावली और तकनीकी शब्दों में महारत हासिल करेंगे
- Infrastructure as Code और परिनियोजन शब्दावली को कुशलतापूर्वक संदर्भित करेंगे
- Azure सेवा नाम, संक्षेपाक्षर और उनके उद्देश्यों को समझेंगे
- समस्या निवारण और डिबगिंग शब्दावली की परिभाषाओं तक पहुंच प्राप्त करेंगे
- उन्नत Azure आर्किटेक्चर और विकास अवधारणाओं को सीखेंगे

## सीखने के परिणाम

इस शब्दावली का नियमित संदर्भ लेने से, आप सक्षम होंगे:
- सही Azure Developer CLI शब्दावली का उपयोग करके प्रभावी ढंग से संवाद करें
- तकनीकी दस्तावेज़ों और त्रुटि संदेशों को अधिक स्पष्ट रूप से समझें
- आत्मविश्वास के साथ Azure सेवाओं और अवधारणाओं को नेविगेट करें
- उपयुक्त तकनीकी शब्दावली का उपयोग करके समस्याओं का समाधान करें
- टीम चर्चाओं में सटीक तकनीकी भाषा के साथ योगदान दें
- व्यवस्थित रूप से Azure क्लाउड विकास ज्ञान का विस्तार करें

## A

**ARM Template**  
Azure Resource Manager टेम्पलेट। JSON-आधारित Infrastructure as Code प्रारूप जो Azure संसाधनों को डिक्लेरेटिव रूप से परिभाषित और परिनियोजित करता है।

**App Service**  
Azure का प्लेटफ़ॉर्म-एज़-ए-सर्विस (PaaS) ऑफ़रिंग जो वेब एप्लिकेशन, REST APIs, और मोबाइल बैकएंड को होस्ट करता है बिना इंफ्रास्ट्रक्चर प्रबंधन के।

**Application Insights**  
Azure की एप्लिकेशन प्रदर्शन निगरानी (APM) सेवा जो एप्लिकेशन प्रदर्शन, उपलब्धता, और उपयोग के बारे में गहन अंतर्दृष्टि प्रदान करती है।

**Azure CLI**  
Azure संसाधनों को प्रबंधित करने के लिए कमांड-लाइन इंटरफ़ेस। azd द्वारा प्रमाणीकरण और कुछ संचालन के लिए उपयोग किया जाता है।

**Azure Developer CLI (azd)**  
डेवलपर-केंद्रित कमांड-लाइन टूल जो टेम्पलेट्स और Infrastructure as Code का उपयोग करके Azure पर एप्लिकेशन बनाने और परिनियोजित करने की प्रक्रिया को तेज करता है।

**azure.yaml**  
azd प्रोजेक्ट के लिए मुख्य कॉन्फ़िगरेशन फ़ाइल जो सेवाओं, इंफ्रास्ट्रक्चर, और परिनियोजन हुक को परिभाषित करती है।

**Azure Resource Manager (ARM)**  
Azure की परिनियोजन और प्रबंधन सेवा जो संसाधनों को बनाने, अपडेट करने, और हटाने के लिए प्रबंधन परत प्रदान करती है।

## B

**Bicep**  
Microsoft द्वारा विकसित डोमेन-विशिष्ट भाषा (DSL) जो Azure संसाधनों को परिनियोजित करती है। ARM टेम्पलेट्स की तुलना में सरल सिंटैक्स प्रदान करती है और ARM में संकलित होती है।

**Build**  
स्रोत कोड को संकलित करने, निर्भरताओं को स्थापित करने, और एप्लिकेशन को परिनियोजन के लिए तैयार करने की प्रक्रिया।

**Blue-Green Deployment**  
परिनियोजन रणनीति जो डाउनटाइम और जोखिम को कम करने के लिए दो समान उत्पादन वातावरण (ब्लू और ग्रीन) का उपयोग करती है।

## C

**Container Apps**  
Azure की सर्वरलेस कंटेनर सेवा जो कंटेनरीकृत एप्लिकेशन को जटिल इंफ्रास्ट्रक्चर प्रबंधन के बिना चलाने में सक्षम बनाती है।

**CI/CD**  
कंटीन्यूअस इंटीग्रेशन/कंटीन्यूअस डिप्लॉयमेंट। कोड परिवर्तनों को एकीकृत करने और एप्लिकेशन को परिनियोजित करने के लिए स्वचालित प्रथाएँ।

**Cosmos DB**  
Azure की वैश्विक रूप से वितरित, मल्टी-मॉडल डेटाबेस सेवा जो थ्रूपुट, विलंबता, उपलब्धता, और स्थिरता के लिए व्यापक SLA प्रदान करती है।

**Configuration**  
सेटिंग्स और पैरामीटर जो एप्लिकेशन व्यवहार और परिनियोजन विकल्पों को नियंत्रित करते हैं।

## D

**Deployment**  
लक्षित इंफ्रास्ट्रक्चर पर एप्लिकेशन और उनकी निर्भरताओं को स्थापित और कॉन्फ़िगर करने की प्रक्रिया।

**Docker**  
कंटेनरीकरण तकनीक का उपयोग करके एप्लिकेशन विकसित करने, शिप करने, और चलाने के लिए प्लेटफ़ॉर्म।

**Dockerfile**  
डॉकर कंटेनर इमेज बनाने के लिए निर्देशों वाली टेक्स्ट फ़ाइल।

## E

**Environment**  
एक परिनियोजन लक्ष्य जो आपके एप्लिकेशन की एक विशिष्ट इंस्टेंस का प्रतिनिधित्व करता है (जैसे, विकास, स्टेजिंग, उत्पादन)।

**Environment Variables**  
कॉन्फ़िगरेशन मान जो कुंजी-मूल्य जोड़े के रूप में संग्रहीत होते हैं और एप्लिकेशन रनटाइम पर एक्सेस कर सकते हैं।

**Endpoint**  
URL या नेटवर्क पता जहां एप्लिकेशन या सेवा एक्सेस की जा सकती है।

## F

**Function App**  
Azure की सर्वरलेस कंप्यूट सेवा जो इवेंट-ड्रिवन कोड को इंफ्रास्ट्रक्चर प्रबंधन के बिना चलाने में सक्षम बनाती है।

## G

**GitHub Actions**  
GitHub रिपॉजिटरी के साथ एकीकृत CI/CD प्लेटफ़ॉर्म जो वर्कफ़्लो को स्वचालित करता है।

**Git**  
स्रोत कोड में परिवर्तनों को ट्रैक करने के लिए वितरित संस्करण नियंत्रण प्रणाली।

## H

**Hooks**  
कस्टम स्क्रिप्ट या कमांड जो परिनियोजन जीवनचक्र के विशिष्ट बिंदुओं पर चलते हैं (जैसे, preprovision, postprovision, predeploy, postdeploy)।

**Host**  
Azure सेवा प्रकार जहां एप्लिकेशन परिनियोजित किया जाएगा (जैसे, appservice, containerapp, function)।

## I

**Infrastructure as Code (IaC)**  
कोड के माध्यम से इंफ्रास्ट्रक्चर को परिभाषित और प्रबंधित करने की प्रथा, मैनुअल प्रक्रियाओं के बजाय।

**Init**  
आमतौर पर टेम्पलेट से एक नया azd प्रोजेक्ट प्रारंभ करने की प्रक्रिया।

## J

**JSON**  
जावास्क्रिप्ट ऑब्जेक्ट नोटेशन। कॉन्फ़िगरेशन फ़ाइलों और API प्रतिक्रियाओं के लिए सामान्य डेटा इंटरचेंज प्रारूप।

**JWT**  
जावास्क्रिप्ट वेब टोकन। JSON ऑब्जेक्ट के रूप में पार्टियों के बीच जानकारी को सुरक्षित रूप से प्रसारित करने का मानक।

## K

**Key Vault**  
Azure की सेवा जो रहस्यों, कुंजियों, और प्रमाणपत्रों को सुरक्षित रूप से संग्रहीत और प्रबंधित करती है।

**Kusto Query Language (KQL)**  
Azure Monitor, Application Insights, और अन्य Azure सेवाओं में डेटा का विश्लेषण करने के लिए उपयोग की जाने वाली क्वेरी भाषा।

## L

**Load Balancer**  
सेवा जो आने वाले नेटवर्क ट्रैफ़िक को कई सर्वरों या इंस्टेंसों में वितरित करती है।

**Log Analytics**  
क्लाउड और ऑन-प्रिमाइसेस वातावरण से टेलीमेट्री डेटा एकत्र करने, विश्लेषण करने, और कार्रवाई करने के लिए Azure सेवा।

## M

**Managed Identity**  
Azure की विशेषता जो Azure सेवाओं को अन्य Azure सेवाओं के लिए स्वचालित रूप से प्रबंधित पहचान प्रदान करती है।

**Microservices**  
आर्किटेक्चरल दृष्टिकोण जहां एप्लिकेशन को छोटे, स्वतंत्र सेवाओं के संग्रह के रूप में बनाया जाता है।

**Monitor**  
Azure का एकीकृत निगरानी समाधान जो एप्लिकेशन और इंफ्रास्ट्रक्चर में पूर्ण-स्टैक अवलोकन प्रदान करता है।

## N

**Node.js**  
Chrome के V8 जावास्क्रिप्ट इंजन पर आधारित जावास्क्रिप्ट रनटाइम जो सर्वर-साइड एप्लिकेशन बनाने के लिए उपयोग किया जाता है।

**npm**  
Node.js के लिए पैकेज मैनेजर जो निर्भरताओं और पैकेजों को प्रबंधित करता है।

## O

**Output**  
इंफ्रास्ट्रक्चर परिनियोजन से लौटाए गए मान जो एप्लिकेशन या अन्य संसाधनों द्वारा उपयोग किए जा सकते हैं।

## P

**Package**  
एप्लिकेशन कोड और निर्भरताओं को परिनियोजन के लिए तैयार करने की प्रक्रिया।

**Parameters**  
इंफ्रास्ट्रक्चर टेम्पलेट्स को पास किए गए इनपुट मान जो परिनियोजन को अनुकूलित करते हैं।

**PostgreSQL**  
Azure में एक प्रबंधित सेवा के रूप में समर्थित ओपन-सोर्स रिलेशनल डेटाबेस सिस्टम।

**Provisioning**  
इंफ्रास्ट्रक्चर टेम्पलेट्स में परिभाषित Azure संसाधनों को बनाने और कॉन्फ़िगर करने की प्रक्रिया।

## Q

**Quota**  
Azure सब्सक्रिप्शन या क्षेत्र में बनाए जा सकने वाले संसाधनों की सीमा।

## R

**Resource Group**  
Azure संसाधनों के लिए तार्किक कंटेनर जो समान जीवनचक्र, अनुमतियों, और नीतियों को साझा करते हैं।

**Resource Token**  
azd द्वारा उत्पन्न अद्वितीय स्ट्रिंग जो सुनिश्चित करती है कि संसाधन नाम परिनियोजन में अद्वितीय हों।

**REST API**  
HTTP विधियों का उपयोग करके नेटवर्क एप्लिकेशन डिज़ाइन करने के लिए आर्किटेक्चरल शैली।

**Rollback**  
एप्लिकेशन या इंफ्रास्ट्रक्चर कॉन्फ़िगरेशन के पिछले संस्करण पर वापस जाने की प्रक्रिया।

## S

**Service**  
आपके एप्लिकेशन का एक घटक जो azure.yaml में परिभाषित है (जैसे, वेब फ्रंटएंड, API बैकएंड, डेटाबेस)।

**SKU**  
स्टॉक कीपिंग यूनिट। Azure संसाधनों के लिए विभिन्न सेवा स्तरों या प्रदर्शन स्तरों का प्रतिनिधित्व करता है।

**SQL Database**  
Microsoft SQL Server पर आधारित Azure की प्रबंधित रिलेशनल डेटाबेस सेवा।

**Static Web Apps**  
स्रोत कोड रिपॉजिटरी से पूर्ण-स्टैक वेब एप्लिकेशन बनाने और परिनियोजित करने के लिए Azure सेवा।

**Storage Account**  
Azure सेवा जो डेटा ऑब्जेक्ट्स के लिए क्लाउड स्टोरेज प्रदान करती है, जिसमें ब्लॉब्स, फाइल्स, क्वीज़, और टेबल्स शामिल हैं।

**Subscription**  
Azure खाता कंटेनर जो संसाधन समूहों और संसाधनों को रखता है, साथ ही संबंधित बिलिंग और एक्सेस प्रबंधन।

## T

**Template**  
पूर्व-निर्मित प्रोजेक्ट संरचना जिसमें एप्लिकेशन कोड, इंफ्रास्ट्रक्चर परिभाषाएँ, और सामान्य परिदृश्यों के लिए कॉन्फ़िगरेशन शामिल है।

**Terraform**  
ओपन-सोर्स Infrastructure as Code टूल जो Azure सहित कई क्लाउड प्रदाताओं का समर्थन करता है।

**Traffic Manager**  
Azure का DNS-आधारित ट्रैफ़िक लोड बैलेंसर जो वैश्विक Azure क्षेत्रों में ट्रैफ़िक वितरित करता है।

## U

**URI**  
यूनिफॉर्म रिसोर्स आइडेंटिफायर। स्ट्रिंग जो किसी विशेष संसाधन की पहचान करती है।

**URL**  
यूनिफॉर्म रिसोर्स लोकेटर। URI का प्रकार जो संसाधन का स्थान और उसे कैसे प्राप्त करें, निर्दिष्ट करता है।

## V

**Virtual Network (VNet)**  
Azure में निजी नेटवर्क के लिए मौलिक निर्माण ब्लॉक, जो अलगाव और विभाजन प्रदान करता है।

**VS Code**  
विजुअल स्टूडियो कोड। लोकप्रिय कोड संपादक जिसमें Azure और azd के साथ उत्कृष्ट एकीकरण है।

## W

**Webhook**  
विशिष्ट घटनाओं द्वारा ट्रिगर किया गया HTTP कॉलबैक, आमतौर पर CI/CD पाइपलाइनों में उपयोग किया जाता है।

**What-if**  
Azure की विशेषता जो दिखाती है कि परिनियोजन द्वारा क्या परिवर्तन किए जाएंगे, बिना वास्तव में इसे निष्पादित किए।

## Y

**YAML**

## संदर्भ-विशिष्ट शब्द

### विकास शब्द
- **Hot Reload**: विकास के दौरान एप्लिकेशन को बिना पुनः आरंभ किए स्वचालित रूप से अपडेट करना
- **Build Pipeline**: कोड को बनाने और परीक्षण करने की स्वचालित प्रक्रिया
- **Deployment Slot**: App Service के भीतर स्टेजिंग वातावरण
- **Environment Parity**: विकास, स्टेजिंग, और उत्पादन वातावरण को समान रखना

### सुरक्षा शब्द
- **Managed Identity**: स्वचालित क्रेडेंशियल प्रबंधन प्रदान करने वाली Azure विशेषता
- **Key Vault**: रहस्यों, कुंजियों, और प्रमाणपत्रों के लिए सुरक्षित संग्रहण
- **RBAC**: Azure संसाधनों के लिए भूमिका-आधारित एक्सेस नियंत्रण
- **Network Security Group**: नेटवर्क ट्रैफ़िक को नियंत्रित करने के लिए वर्चुअल फ़ायरवॉल

### निगरानी शब्द
- **Telemetry**: माप और डेटा का स्वचालित संग्रह
- **Application Performance Monitoring (APM)**: सॉफ़्टवेयर प्रदर्शन की निगरानी
- **Log Analytics**: लॉग डेटा एकत्र करने और विश्लेषण करने के लिए सेवा
- **Alert Rules**: मेट्रिक्स या शर्तों के आधार पर स्वचालित सूचनाएँ

### परिनियोजन शब्द
- **Blue-Green Deployment**: शून्य-डाउनटाइम परिनियोजन रणनीति
- **Canary Deployment**: उपयोगकर्ताओं के एक उपसमूह को धीरे-धीरे रोलआउट करना
- **Rolling Update**: एप्लिकेशन इंस्टेंस का क्रमिक प्रतिस्थापन
- **Rollback**: एप्लिकेशन के पिछले संस्करण पर वापस जाना
