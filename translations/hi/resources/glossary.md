# शब्दावली - Azure और AZD शब्दावली

**सभी अध्यायों के लिए संदर्भ**
- **📚 कोर्स होम**: [AZD For Beginners](../README.md)
- **📖 मूल बातें सीखें**: [अध्याय 1: AZD मूल बातें](../docs/getting-started/azd-basics.md)
- **🤖 एआई शर्तें**: [अध्याय 2: एआई-प्रथम विकास](../docs/microsoft-foundry/microsoft-foundry-integration.md)

## परिचय

यह व्यापक शब्दावली Azure Developer CLI और Azure क्लाउड विकास में प्रयुक्त शब्दों, अवधारणाओं, और संक्षेपाक्षरों की परिभाषाएँ प्रस्तुत करती है। तकनीकी दस्तावेज़ों को समझने, समस्याओं का निवारण करने, और azd परियोजनाओं तथा Azure सेवाओं के बारे में प्रभावी रूप से संवाद करने के लिए यह एक आवश्यक संदर्भ है।

## सीखने के लक्ष्य

इस शब्दावली का उपयोग करके, आप:
- Azure Developer CLI की आवश्यक शब्दावली और अवधारणाओं को समझेंगे
- Azure क्लाउड विकास की शब्दावली और तकनीकी शर्तों में दक्षता हासिल करेंगे
- Infrastructure as Code और डिप्लॉयमेंट शब्दावली का प्रभावी संदर्भ ले सकेंगे
- Azure सेवा नामों, संक्षेपाक्षरों, और उनके प्रयोजनों को समझेंगे
- समस्याओं के निवारण और डिबगिंग शब्दावली के लिए परिभाषाएँ देख सकेंगे
- उन्नत Azure आर्किटेक्चर और विकास अवधारणाएँ सीखेंगे

## सीखने के परिणाम

इस शब्दावली का नियमित संदर्भ लेकर, आप सक्षम होंगे:
- उचित Azure Developer CLI शब्दावली का उपयोग करके प्रभावी रूप से संवाद करना
- तकनीकी दस्तावेज़ों और त्रुटि संदेशों को अधिक स्पष्टता से समझना
- आत्मविश्वास के साथ Azure सेवाओं और अवधारणाओं में नेविगेट करना
- उपयुक्त तकनीकी शब्दावली का उपयोग करके समस्याओं का निवारण करना
- सटीक तकनीकी भाषा के साथ टीम चर्चाओं में योगदान देना
- व्यवस्थित रूप से अपने Azure क्लाउड विकास ज्ञान का विस्तार करना

## A

**ARM Template**  
Azure Resource Manager टेम्पलेट। JSON-आधारित Infrastructure as Code प्रारूप जिसका उपयोग Azure संसाधनों को घोषणात्मक रूप से परिभाषित और तैनात करने के लिए किया जाता है।

**App Service**  
Azure का प्लेटफ़ॉर्म-एज़-ए-सर्विस (PaaS) प्रस्ताव जो वेब एप्लिकेशन, REST API, और मोबाइल बैकएंड को बिना इन्फ्रास्ट्रक्चर मैनेज किए होस्ट करने के लिए है।

**Application Insights**  
Azure की एप्लिकेशन परफॉर्मेंस मॉनिटरिंग (APM) सेवा जो एप्लिकेशन के प्रदर्शन, उपलब्धता, और उपयोग के बारे में गहरी जानकारी प्रदान करती है।

**Azure CLI**  
Azure संसाधनों का प्रबंधन करने के लिए कमांड-लाइन इंटरफेस। प्रमाणीकरण और कुछ ऑपरेशनों के लिए azd द्वारा उपयोग किया जाता है।

**Azure Developer CLI (azd)**  
डेवलपर-केंद्रित कमांड-लाइन टूल जो टेम्पलेट और Infrastructure as Code का उपयोग करके Azure पर एप्लिकेशन बनाने और तैनात करने की प्रक्रिया को तेज करता है।

**azure.yaml**  
azd परियोजना के लिए मुख्य कॉन्फ़िगरेशन फ़ाइल जो सेवाओं, इन्फ्रास्ट्रक्चर, और डिप्लॉयमेंट हुक्स को परिभाषित करती है।

**Azure Resource Manager (ARM)**  
Azure की तैनाती और प्रबंधक सेवा जो संसाधनों के निर्माण, अद्यतन, और हटाने के लिए एक प्रबंधन परत प्रदान करती है।

## B

**Bicep**  
Microsoft द्वारा विकसित डोमेन-विशिष्ट भाषा (DSL) जो Azure संसाधनों को तैनात करने के लिए है। यह ARM टेम्पलेट्स की तुलना में सरल सिंटैक्स प्रदान करता है और ARM में संकलित होता है।

**Build**  
स्रोत कोड को संकलित करने, निर्भरताएँ स्थापित करने, और एप्लिकेशन को तैनाती के लिए तैयार करने की प्रक्रिया।

**Blue-Green Deployment**  
डिप्लॉयमेंट रणनीति जो डाउनटाइम और जोखिम को कम करने के लिए दो समान प्रोडक्शन वातावरण (नीला और हरा) का उपयोग करती है।

## C

**Container Apps**  
Azure की सर्वरलेस कंटेनर सेवा जो जटिल इन्फ्रास्ट्रक्चर को प्रबंधित किए बिना कंटेनरयुक्त एप्लिकेशन चलाने में सक्षम बनाती है।

**CI/CD**  
Continuous Integration/Continuous Deployment। कोड परिवर्तनों को एकीकृत करने और एप्लिकेशन तैनात करने के लिए स्वचालित प्रथाएँ।

**Cosmos DB**  
Azure की ग्लोबली वितरित, मल्टी-मॉडल डेटाबेस सेवा जो थ्रूपुट, विलंबता, उपलब्धता, और संगति के लिए व्यापक SLA प्रदान करती है।

**Configuration**  
ऐसी सेटिंग्स और पैरामीटर जो एप्लिकेशन के व्यवहार और तैनाती विकल्पों को नियंत्रित करते हैं।

## D

**Deployment**  
लक्षित इन्फ्रास्ट्रक्चर पर एप्लिकेशन और उनकी निर्भरताओं को स्थापित और कॉन्फ़िगर करने की प्रक्रिया।

**Docker**  
कंटेनरीकरण प्रौद्योगिकी का उपयोग करके एप्लिकेशन विकसित करने, भेजने, और चलाने का प्लेटफ़ॉर्म।

**Dockerfile**  
Docker कंटेनर इमेज बनाने के निर्देशों वाला टेक्स्ट फ़ाइल।

## E

**Environment**  
एक तैनाती लक्ष्य जो आपके एप्लिकेशन का एक विशिष्ट उदाहरण दर्शाता है (उदा., development, staging, production)।

**Environment Variables**  
कनफ़िगरेशन मान जो कुंजी-मूल्य युग्म के रूप में संग्रहीत होते हैं जिन्हें एप्लिकेशन रनटाइम पर एक्सेस कर सकते हैं।

**Endpoint**  
URL या नेटवर्क पता जहाँ किसी एप्लिकेशन या सेवा तक पहुँचा जा सकता है।

## F

**Function App**  
Azure की सर्वरलेस कंप्यूट सेवा जो इवेंट-ड्रिवन कोड चलाने में सक्षम बनाती है बिना इन्फ्रास्ट्रक्चर प्रबंधित किए।

## G

**GitHub Actions**  
GitHub रिपॉज़िटरीज़ के साथ एकीकृत CI/CD प्लेटफ़ॉर्म जो वर्कफ़्लो को स्वचालित करता है।

**Git**  
वितरित वर्शन नियंत्रण प्रणाली जो स्रोत कोड में परिवर्तनों को ट्रैक करने के लिए उपयोग होती है।

## H

**Hooks**  
कस्टम स्क्रिप्ट या कमांड जो तैनाती जीवनचक्र के विशिष्ट बिंदुओं पर चलते हैं (preprovision, postprovision, predeploy, postdeploy)।

**Host**  
Azure सेवा प्रकार जहाँ एक एप्लिकेशन तैनात होगा (उदा., appservice, containerapp, function)।

## I

**Infrastructure as Code (IaC)**  
इन्फ्रास्ट्रक्चर को मैनुअल प्रक्रियाओं के बजाय कोड के माध्यम से परिभाषित और प्रबंधित करने का अभ्यास।

**Init**  
सामान्यतः एक टेम्पलेट से नए azd प्रोजेक्ट को प्रारम्भ करने की प्रक्रिया।

## J

**JSON**  
JavaScript Object Notation। डेटा इंटरचेंज फ़ॉर्मेट जो कॉन्फ़िगरेशन फ़ाइलों और API प्रतिक्रियाओं के लिए सामान्यतः उपयोग किया जाता है।

**JWT**  
JSON Web Token। पार्टियों के बीच सुरक्षित रूप से जानकारी भेजने के लिए JSON ऑब्जेक्ट के रूप में एक मानक।

## K

**Key Vault**  
गुप्त, कुंजी, और प्रमाणपत्रों को सुरक्षित रूप से संग्रहित और प्रबंधित करने के लिए Azure की सेवा।

**Kusto Query Language (KQL)**  
Azure Monitor, Application Insights, और अन्य Azure सेवाओं में डेटा का विश्लेषण करने के लिए प्रयुक्त क्वेरी भाषा।

## L

**Load Balancer**  
सेवा जो आने वाले नेटवर्क ट्रैफ़िक को कई सर्वरों या इंस्टेंसों में वितरित करती है।

**Log Analytics**  
क्लाउड और ऑन-प्रिमाइसेस वातावरण से टेलीमेट्री डेटा एकत्र, विश्लेषण, और कार्रवाई करने के लिए Azure सेवा।

## M

**Managed Identity**  
Azure सुविधा जो अन्य Azure सेवाओं के लिए प्रमाणीकृत करने हेतु स्वतः प्रबंधित पहचान प्रदान करती है।

**Microservices**  
ऐसी आर्किटेक्चरल पद्धति जहाँ एप्लिकेशन छोटे, स्वतंत्र सेवाओं के संग्रह के रूप में बनाए जाते हैं।

**Monitor**  
Azure का एकीकृत मॉनिटरिंग समाधान जो एप्लिकेशन और इन्फ्रास्ट्रक्चर के पार फुल-स्टैक अवलोकनीयता प्रदान करता है।

## N

**Node.js**  
Chrome के V8 JavaScript इंजन पर निर्मित जावास्क्रिप्ट रनटाइम जो सर्वर-साइड एप्लिकेशन बनाने के लिए उपयोग होता है।

**npm**  
Node.js के लिए पैकेज मैनेजर जो निर्भरताओं और पैकेजों का प्रबंधन करता है।

## O

**Output**  
इन्फ्रास्ट्रक्चर तैनाती से लौटाए गए मान जिन्हें एप्लिकेशन या अन्य संसाधनों द्वारा उपयोग किया जा सकता है।

## P

**Package**  
एप्लिकेशन कोड और निर्भरताओं को तैनाती के लिए तैयार करने की प्रक्रिया।

**Parameters**  
इन्फ्रास्ट्रक्चर टेम्पलेट्स को कस्टमाइज़ करने के लिए पास किए जाने वाले इनपुट मान।

**PostgreSQL**  
ओपन-सोर्स रिलेशनल डेटाबेस सिस्टम जिसे Azure में मैनेज्ड सेवा के रूप में समर्थन मिलता है।

**Provisioning**  
इन्फ्रास्ट्रक्चर टेम्पलेट्स में परिभाषित Azure संसाधनों को बनाना और कॉन्फ़िगर करने की प्रक्रिया।

## Q

**Quota**  
Azure सदस्यता या रीजन में बनाई जा सकने वाली संसाधनों की मात्रा पर सीमाएँ।

## R

**Resource Group**  
Azure संसाधनों के लिए तार्किक कंटेनर जो समान लाइफसाइकल, अनुमतियाँ, और नीतियाँ साझा करते हैं।

**Resource Token**  
azd द्वारा जेनरेट किया गया अद्वितीय स्ट्रिंग जो तैनातियों के दौरान संसाधन नामों की अद्वितीयता सुनिश्चित करता है।

**REST API**  
नेटवर्केड एप्लिकेशन डिज़ाइन करने के लिए HTTP विधियों का उपयोग करने वाली आर्किटेक्चरल शैली।

**Rollback**  
एप्लिकेशन या इन्फ्रास्ट्रक्चर कॉन्फ़िगरेशन के पिछले संस्करण पर लौटने की प्रक्रिया।

## S

**Service**  
आपके azure.yaml में परिभाषित आपके एप्लिकेशन का एक घटक (उदा., वेब फ्रंटएंड, API बैकएंड, डेटाबेस)।

**SKU**  
Stock Keeping Unit। Azure संसाधनों के लिए विभिन्न सेवा स्तरों या प्रदर्शन श्रेणियों का प्रतिनिधित्व करता है।

**SQL Database**  
Microsoft SQL Server पर आधारित Azure की मैनेज्ड रिलेशनल डेटाबेस सेवा।

**Static Web Apps**  
सोर्स कोड रिपॉज़िटरीज़ से फुल-स्टैक वेब एप्लिकेशन बनाने और तैनात करने के लिए Azure सेवा।

**Storage Account**  
ब्लॉब्स, फाइल्स, क्यूज़, और टेबल सहित डेटा ऑब्जेक्ट्स के लिए क्लाउड स्टोरेज प्रदान करने वाली Azure सेवा।

**Subscription**  
Azure अकाउंट कंटेनर जो रिसोर्स ग्रुप्स और संसाधनों को रखता है, साथ ही बिलिंग और एक्सेस प्रबंधन से जुड़ा होता है।

## T

**Template**  
प्री-निर्मित प्रोजेक्ट संरचना जिसमें एप्लिकेशन कोड, इन्फ्रास्ट्रक्चर परिभाषाएँ, और सामान्य परिदृश्यों के लिए कॉन्फ़िगरेशन शामिल होता है।

**Terraform**  
ओपन-सोर्स Infrastructure as Code टूल जो Azure सहित कई क्लाउड प्रोवाइडरों का समर्थन करता है।

**Traffic Manager**  
ग्लोबल Azure क्षेत्रों में ट्रैफ़िक वितरित करने के लिए Azure का DNS-आधारित ट्रैफ़िक लोड बैलेंसर।

## U

**URI**  
Uniform Resource Identifier। किसी विशेष संसाधन की पहचान करने वाली स्ट्रिंग।

**URL**  
Uniform Resource Locator। URI का प्रकार जो बताता है कि किसी संसाधन को कहाँ खोजा जाए और कैसे पुनःप्राप्त किया जाए।

## V

**Virtual Network (VNet)**  
Azure में प्राइवेट नेटवर्क के लिए मूलभूत निर्माण ब्लॉक, जो अलगाव और सेगमेंटेशन प्रदान करता है।

**VS Code**  
Visual Studio Code। लोकप्रिय कोड संपादक जिसमें उत्कृष्ट Azure और azd एकीकरण है।

## W

**Webhook**  
विशिष्ट घटनाओं द्वारा ट्रिगर किया गया HTTP कॉलबैक, जो सामान्यतः CI/CD पाइपलाइनों में उपयोग होता है।

**What-if**  
Azure सुविधा जो यह दिखाती है कि किसी तैनाती द्वारा कौन से परिवर्तन किए जाएंगे बिना उन्हें वास्तव में निष्पादित किए।

## Y

**YAML**

## संदर्भ-विशिष्ट शर्तें

### विकास से संबंधित शर्तें
- **Hot Reload**: विकास के दौरान एप्लिकेशन को बिना पुनरारंभ किए स्वचालित रूप से अपडेट करना
- **Build Pipeline**: कोड बिल्ड और परीक्षण के लिए स्वचालित प्रक्रिया
- **Deployment Slot**: App Service के भीतर स्टेजिंग वातावरण
- **Environment Parity**: विकास, स्टेजिंग, और प्रोडक्शन वातावरणों को समान बनाए रखना

### सुरक्षा से संबंधित शर्तें
- **Managed Identity**: स्वचालित क्रेडेंशियल प्रबंधन प्रदान करने वाली Azure सुविधा
- **Key Vault**: सीक्रेट्स, कुंजियाँ, और प्रमाणपत्रों के लिए सुरक्षित भंडारण
- **RBAC**: Azure संसाधनों के लिए भूमिका-आधारित एक्सेस नियंत्रण
- **Network Security Group**: नेटवर्क ट्रैफ़िक को नियंत्रित करने के लिए वर्चुअल फ़ायरवॉल

### मॉनिटरिंग से संबंधित शर्तें
- **Telemetry**: माप और डेटा का स्वचालित संग्रह
- **Application Performance Monitoring (APM)**: सॉफ़्टवेयर प्रदर्शन की मॉनिटरिंग
- **Log Analytics**: लॉग डेटा एकत्र करने और विश्लेषण करने की सेवा
- **Alert Rules**: मीट्रिक्स या शर्तों के आधार पर स्वचालित सूचनाएँ

### डिप्लॉयमेंट से संबंधित शर्तें
- **Blue-Green Deployment**: शून्य-डाउनटाइम डिप्लॉयमेंट रणनीति
- **Canary Deployment**: उपयोगकर्ताओं के एक उपसमूह के लिए धीरे-धीरे रोलआउट
- **Rolling Update**: एप्लिकेशन इंस्टेंसेस के क्रमिक प्रतिस्थापन
- **Rollback**: एप्लिकेशन के पिछले संस्करण पर लौटना
