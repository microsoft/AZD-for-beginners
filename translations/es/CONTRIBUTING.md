# Contribuir a AZD For Beginners

¡Gracias por tu interés en contribuir al curso AZD For Beginners! Este documento proporciona pautas e información para los colaboradores.

## Cómo Contribuir

### Reportar problemas

If you find bugs, errors, or have suggestions for improvements:

1. **Revisa los problemas existentes**: Busca [problemas existentes](https://github.com/microsoft/azd-for-beginners/issues) para evitar duplicados
2. **Crea un nuevo issue**: Usa la plantilla de issue correspondiente
3. **Proporciona detalles**: Incluye pasos para reproducir, comportamiento esperado y capturas de pantalla si aplica

### Proponer cambios

1. **Haz un fork del repositorio**: Crea tu propio fork del proyecto
2. **Crea una rama**: Crea una rama feature para tus cambios
3. **Realiza los cambios**: Sigue nuestros estándares de código y documentación
4. **Prueba tus cambios**: Verifica que todos los enlaces funcionen y el contenido se renderice correctamente
5. **Envía una Pull Request**: Proporciona una descripción clara de tus cambios

### Tipos de contribuciones

Agradecemos los siguientes tipos de contribuciones:

#### Mejoras de contenido
- Corregir errores tipográficos, gramática y problemas de formato
- Mejorar la claridad de la documentación existente
- Actualizar información desactualizada
- Añadir explicaciones o contexto faltante

#### Nuevos ejemplos
- Añadir escenarios de despliegue del mundo real
- Crear nuevas plantillas AZD para casos de uso comunes
- Proporcionar muestras de código adicionales y ejercicios

#### Soporte de traducción
- Ayudar a mantener las traducciones existentes
- Mejorar la calidad de las traducciones
- Añadir traducciones para contenido nuevo

#### Informes de errores
- Reportar enlaces rotos o ejemplos de código incorrectos
- Identificar inconsistencias en la documentación
- Señalar problemas de accesibilidad

## Pautas de contribución

### Estándares de documentación

- Usa un lenguaje claro y accesible para principiantes
- Incluye objetivos y resultados de aprendizaje
- Proporciona instrucciones paso a paso con salidas esperadas
- Añade enlaces de navegación al contenido relacionado
- Usa un formato y estilo de Markdown consistentes

### Ejemplos de código

- Asegúrate de que todos los ejemplos de código estén probados y funcionen
- Incluye comentarios que expliquen los conceptos clave
- Sigue las mejores prácticas de Azure y AZD
- Proporciona la salida esperada cuando sea aplicable

### Mensajes de commit

Usa mensajes de commit claros y descriptivos:
- `docs: fix typo in installation guide`
- `feat: add new container app example`
- `fix: correct broken link in chapter 2`

## Código de conducta

Por favor lee y sigue nuestro [Código de Conducta](CODE_OF_CONDUCT.md) para mantener una comunidad acogedora e inclusiva.
