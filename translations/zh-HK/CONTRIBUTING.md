# 為 AZD For Beginners 做出貢獻

感謝你有興趣為 AZD For Beginners 課程做出貢獻！本文件提供給貢獻者的指引與資訊。

## 如何貢獻

### 回報問題

如果你發現錯誤、瑕疵或對改善有建議：

1. **檢查現有議題**：搜尋 [現有議題](https://github.com/microsoft/azd-for-beginners/issues) 以避免重複
2. **建立新議題**：使用適當的議題範本
3. **提供詳情**：包含重現步驟、預期行為，以及如適用的截圖

### 建議變更

1. **Fork 儲存庫**：建立你自己的專案 fork
2. **建立分支**：為你的變更建立一個功能分支
3. **進行修改**：遵守我們的程式碼與文件標準
4. **測試你的修改**：確認所有連結可用且內容正確呈現
5. **提交 Pull Request**：清楚說明你的變更

### 貢獻類型

我們歡迎以下類型的貢獻：

#### 內容改進
- 修正文法錯誤、拼字與格式問題
- 提升現有文件的清晰度
- 更新過時資訊
- 補充缺少的說明或背景

#### 新範例
- 新增實務部署範例
- 為常見使用情境建立新的 AZD 範本
- 提供額外的程式碼範例與練習題

#### 翻譯支援
- 協助維護現有翻譯
- 改善翻譯品質
- 為新內容新增翻譯

#### 錯誤回報
- 回報失效連結或錯誤的程式碼範例
- 指出文件不一致之處
- 標記可及性（accessibility）問題

## 貢獻指南

### 文件標準

- 使用清楚且適合新手的語言
- 包含學習目標與成果
- 提供步驟指引與預期輸出
- 新增導覽連結至相關內容
- 使用一致的格式與 Markdown 風格

### 程式碼範例

- 確保所有程式碼範例已測試且可運作
- 加入註解說明關鍵概念
- 遵循 Azure 與 AZD 的最佳實務
- 在適用時提供預期輸出

### Commit 訊息

使用清楚且描述性的提交訊息：
- `docs: fix typo in installation guide`
- `feat: add new container app example`
- `fix: correct broken link in chapter 2`

## 行為準則

請閱讀並遵守我們的 [行為準則](CODE_OF_CONDUCT.md)，以維持一個友善且包容的社群。

## 尋求協助

- **問題**：開一個 [GitHub Discussion](https://github.com/microsoft/azd-for-beginners/discussions)
- **社群**：加入 [Azure Discord](https://discord.gg/microsoft-azure)
- **文件**：檢閱 [Azure Developer CLI 文件](https://learn.microsoft.com/azure/developer/azure-developer-cli/)
