# Changelog - AZD Для Початківців

## Вступ

Цей журнал змін документує всі помітні зміни, оновлення та покращення в репозиторії AZD Для Початківців. Ми дотримуємося принципів семантичного версіонування і підтримуємо цей журнал, щоб допомогти користувачам зрозуміти, що змінилося між версіями.

## Навчальні Цілі

Переглянувши цей журнал змін, ви зможете:
- Бути в курсі нових функцій та доданого контенту
- Розуміти вдосконалення існуючої документації
- Відстежувати виправлення помилок та корекції для забезпечення точності
- Слідкувати за еволюцією навчальних матеріалів з часом

## Результати Навчання

Після ознайомлення з записами журналу змін ви зможете:
- Виявляти новий контент і ресурси, доступні для навчання
- Розуміти, які розділи оновлено чи покращено
- Планувати свій навчальний шлях на основі найактуальніших матеріалів
- Вносити відгуки та пропозиції для майбутніх покращень

### [v3.19.1] - 2026-03-27

#### Уточнення вступу для початківців, перевірка налаштувань та очищення фінальних команд AZD
**Ця версія підкріплює перевірку AZD 1.23 проходженням документів, орієнтованих на початківців: вона уточнює інструкції щодо автентифікації за допомогою AZD, додає скрипти перевірки локальних налаштувань, перевіряє ключові команди у живому CLI AZD та видаляє останні застарілі посилання на команди англійського оригіналу поза журналом змін.**

#### Додано
- **🧪 Скрипти перевірки налаштувань для початківців** з `validate-setup.ps1` та `validate-setup.sh`, щоб учні могли підтвердити потрібні інструменти перед початком Розділу 1
- **✅ Попередні кроки перевірки налаштувань** у головному README й README Розділу 1, щоб відсутні передумови виявлялися до виконання `azd up`

#### Змінено
- **🔐 Інструкції з автентифікації для початківців** тепер послідовно трактують `azd auth login` як основний шлях для робочих процесів AZD, при цьому `az login` згадується як необов’язковий, якщо безпосередньо не використовуються команди Azure CLI
- **📚 Потік введення в Розділі 1** тепер спрямовує учнів перевірити локальне налаштування перед інсталяцією, автентифікацією та першою розгортанням
- **🛠️ Повідомлення валідатора** чітко розділяють блокуючі вимоги й необов’язкові попередження Azure CLI для шляху лише AZD для початківців
- **📖 Документація з конфігурування, усунення несправностей і прикладів** тепер відрізняє обов’язкову автентифікацію AZD від необов’язкового входу Azure CLI, де раніше обидва способи подавалися без контексту

#### Виправлено
- **📋 Залишкові посилання на команди з англійського джерела** оновлені до поточних форм AZD, включно з `azd config show` у шпаргалці та `azd monitor --overview`, коли мала на увазі інструкція з огляду Azure Portal
- **🧭 Заявлення для початківців у Розділі 1** пом’якшене, щоб уникнути надмірних обіцянок гарантованої відсутності помилок або відкатів у всіх шаблонах і ресурсах Azure
- **🔎 Верифікація команд у живому CLI** підтвердила підтримку: `azd env get-values`, `azd template list`, `azd extension list --installed`, `azd copilot consent list`, `azd mcp start`, `azd provision --preview`, `azd monitor --logs` і `azd down --force --purge`

### [v3.19.0] - 2026-03-26

#### Верифікація AZD 1.23.12, розширення середовища майстерні та оновлення AI моделей
**Ця версія проводить перевірку документації проти `azd` `1.23.12`, оновлює застарілі приклади команд AZD, оновлює рекомендації щодо моделей AI до поточних стандартів і розширює інструкції майстерні поза межі GitHub Codespaces, додаючи підтримку контейнерів розробника і локальних клонів.**

#### Додано
- **✅ Примітки про верифікацію у ключових розділах та документах майстерні**, щоб зробити тестовану базу AZD очевидною для учнів із новішими чи старішими збірками CLI
- **⏱️ Інструкції щодо таймаутів розгортання** для довготривалих розгортань AI додатків з використанням `azd deploy --timeout 1800`
- **🔎 Кроки інспекції розширень** з командою `azd extension show azure.ai.agents` в документації AI робочого процесу
- **🌐 Ширші інструкції по середовищах майстерні**, що охоплюють GitHub Codespaces, контейнери розробника та локальні копії з MkDocs

#### Змінено
- **📚 README вступи розділів** тепер послідовно вказують на верифікацію за `azd 1.23.12` у розділах засад, конфігурації, інфраструктури, мультиагентів, передрозгортання, усунення несправностей та продакшену
- **🛠️ Посилання на команди AZD** оновлені до актуальних форм у документах:
  - `azd config list` → `azd config show`
  - `azd env show` → `azd env list` або `azd env get-value(s)` залежно від контексту
  - `azd auth whoami` → `azd auth status`
  - `azd monitor` → `azd monitor --overview` там, де передбачено огляд Application Insights
- **🧪 Приклади provision preview** спрощені до поточних підтримуваних варіантів, як-от `azd provision --preview` та `azd provision --preview -e production`
- **🧭 Потік майстерні** оновлений, щоб учні могли виконувати лабораторії в Codespaces, контейнері розробника або локальному клоні, а не лише в Codespaces
- **🔐 Інструкції автентифікації** віддають перевагу `azd auth login` для робочих процесів AZD, тоді як `az login` позиціонується як необов’язковий при прямому використанні команд Azure CLI

#### Виправлено
- **🪟 Команди інсталяції для Windows** нормалізовані до поточного регістру пакетів `winget` у посібнику інсталяції
- **🐧 Інструкції інсталяції для Linux** виправлені, щоб уникнути непідтримуваних специфічних для дистрибутивів інструкцій менеджера пакетів `azd` і натомість вказувати на релізні артефакти за потреби
- **📦 Приклади моделей AI** оновлені з застарілих `gpt-35-turbo` і `text-embedding-ada-002` на поточні версії `gpt-4.1-mini`, `gpt-4.1` і `text-embedding-3-large`
- **📋 Фрагменти для розгортання та діагностики** виправлені з використанням актуальних команд для середовища та статусу в логах, скриптах і кроках усунення несправностей
- **⚙️ Інструкції GitHub Actions** оновлені з `Azure/setup-azd@v1.0.0` до `Azure/setup-azd@v2`
- **🤖 Інструкції MCP/Copilot consent** оновлені з `azd mcp consent` до `azd copilot consent list`

#### Покращено
- **🧠 Інструкції в розділі AI** тепер краще пояснюють поведінку `azd ai`, чутливу до перегляду, логін для певного орендаря, поточне використання розширень і оновлені рекомендації з розгортання моделей
- **🧪 Інструкції майстерні** тепер використовують реалістичніші приклади версій і зрозумілішу мову налаштування середовища для практичних лабораторій
- **📈 Документи з продакшену та усунення несправностей** тепер краще відповідають актуальним прикладам моніторингу, запасної моделі та рівня вартості

## Внесок у журнал змін

### Повідомлення про зміни
Під час внеску у цей репозиторій, будь ласка, забезпечте, щоб записи в журналі змін містили:

1. **Номер версії**: Згідно із семантичним версіонуванням (major.minor.patch)
2. **Дата**: Дата релізу чи оновлення у форматі YYYY-MM-DD
3. **Категорія**: Added, Changed, Deprecated, Removed, Fixed, Security
4. **Чіткий опис**: Конкретний опис змін
5. **Оцінка впливу**: Як зміни впливають на існуючих користувачів

### Категорії змін

#### Додано
- Нові функції, розділи документації чи можливості

- Додаткові інструменти, скрипти чи утиліти

#### Змінено
- Зміни в існуючій функціональності чи документації
- Оновлення для покращення зрозумілості або точності
- Рефакторинг контенту або організації

#### Застаріле
- Функції або підходи, що припиняються
- Розділи документації, заплановані до видалення
- Методи, що мають кращі альтернативи

#### Видалено
- Відсутні або неактуальні функції, документація чи приклади
- Застаріла інформація або припинені підходи
