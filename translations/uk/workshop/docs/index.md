# AZD для розробників ШІ Workshop

> Перевірено з `azd 1.25.6` у червні 2026 року.

> [!IMPORTANT]  
> **Цей воркшоп оснащений посібником, який ви можете переглянути у вашому локальному браузері. Щоб розпочати, відкрийте репозиторій у GitHub Codespaces, контейнері для розробки або локальному клоні з встановленим MkDocs — потім зачекайте, поки з’явиться активний термінал VS Code і введіть:**  
> `mkdocs serve > /dev/null 2>&1 &`  
> **Ви повинні побачити спливаюче вікно з пропозицією відкрити сторінку попереднього перегляду в браузері.**

Ласкаво просимо на практичний воркшоп для вивчення Azure Developer CLI (AZD) з акцентом на розгортання AI-застосунків. Цей воркшоп допоможе вам отримати практичне розуміння шаблонів AZD у 3 кроки:

1. **Відкриття** – знайти шаблон, який підходить саме вам.  
1. **Розгортання** – розгорнути та перевірити, що він працює.  
1. **Налаштування** – змінити та адаптувати під свої потреби!

Протягом цього воркшопу ви також ознайомитеся з основними інструментами розробника та робочими процесами, що допоможуть вам оптимізувати повний цикл розробки.

## Огляд воркшопу

**Тривалість:** 3-4 години  
**Рівень:** Початковий - середній  
**Вимоги:** Знайомство з Azure, концепціями ШІ, VS Code і інструментами командного рядка.

Це практичний воркшоп, де ви навчаєтесь на практиці. Після завершення вправ рекомендуємо ознайомитися з навчальним курсом AZD For Beginners, щоб продовжити свій шлях навчання з безпеки та кращих практик продуктивності.

## Чого ви навчитеся

Думайте про шаблон AZD як про навчальний майданчик для дослідження різних можливостей і інструментів для повного циклу розробки у Microsoft Foundry. До кінця цього воркшопу ви матимете інтуїтивне розуміння різних інструментів і концепцій у цьому контексті.

| Концепція  | Мета |
|:---|:---|
| **Azure Developer CLI** | Розуміння команд інструменту та робочих процесів |
| **AZD-шаблони** | Розуміння структури проекту та налаштувань |
| **Azure AI Agent** | Надання ресурсів і розгортання проекту Microsoft Foundry |
| **Azure AI Search** | Забезпечення контекстного інжінірингу за допомогою агентів |
| **Observability** | Дослідження трасування, моніторингу і оцінок |
| **Red Teaming** | Огляд атакувального тестування та методів запобігання |
| | |
