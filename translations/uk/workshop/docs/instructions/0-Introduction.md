# AZD Для розробників штучного інтелекту

!!! info "АУДИТОРІЯ ЦЬОГО СЕМІНАРУ"
   
    Цей семінар призначений для всіх, хто прагне створювати агентські AI-додатки на Microsoft Foundry, використовуючи найкращі інструменти та робочі процеси. Семінар підходить як для початківців, так і для досвідчених розробників, які мають деяке знайомство з Azure, концепціями штучного інтелекту та кодом на Python. Будь ласка, перевірте попередні вимоги перед початком.

    **Тривалість:** 3-4 години  
    **Рівень:** Початковий до середнього  

---


## Попередні вимоги

- Особистий обліковий запис GitHub → [Створити безкоштовно](https://github.com/signup)
- Активний обліковий запис Azure → [Створити безкоштовно](https://aka.ms/free)
- Знайомство з концепціями штучного інтелекту, розробкою на Python, інструментами командного рядка
- Досвід роботи з VS Code та інструментами й робочими процесами GitHub.

---

## Цілі навчання

До кінця цього семінару ви зможете:

- [X] **Працювати з Azure Developer CLI** → для розгортання, деплою та керування ресурсами Azure.
- [X] **Використовувати шаблони AI-додатків** → для швидкого старту AI-агентських проектів на Microsoft Foundry.
- [X] **Налаштовувати шаблони** → для функцій, таких як Red Teaming, Оцінювання, Відстеження та Моніторинг.
- [X] **Розбирати та досліджувати** → шаблони AZD та робочі процеси за допомогою GitHub Copilot.
- [X] **Налаштовувати та повторно розгортати** → стартовий шаблон AI-агента з вашими власними вимогами.

---

## План семінару

Семінар структурований у модулі, кожен з яких базується на практичному лабораторному завданні:

1. **Вступ** – Отримайте уявлення про те, що таке шаблони AZD і як вони працюють.
1. **Вибір** – Знайдіть правильний "стартовий" шаблон для вашого наступного AI-проекту.
1. **Перевірка** – Розгорніть та дослідіть шаблон, щоб переконатися, що він працює, як заявлено.
1. **Розбір** – Розберіть шаблон, щоб зрозуміти його компоненти та конфігурацію.
1. **Налаштування** – Навчіться активувати та використовувати різні функції Microsoft Foundry.
1. **Кастомізація** – Реконструюйте додаток з урахуванням ваших даних та вимог.
1. **Зняття** – Навчіться деактивувати інфраструктуру та відновлювати м’яко видалені ресурси.
