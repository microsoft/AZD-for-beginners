## 1. Активуйте GitHub Copilot

### 1.1 Встановіть GitHub Copilot Chat

Час ознайомитися з [GitHub Copilot у режимі агента](https://code.visualstudio.com/docs/copilot/chat/chat-agent-mode). Тепер ми можемо використовувати природну мову, щоб описати завдання на високому рівні й отримати допомогу у виконанні. Для цієї лабораторної роботи ми використаємо [безкоштовний план Copilot](https://github.com/github-copilot/signup), який має щомісячні обмеження на кількість виконань та чат-взаємодій.

Розширення можна встановити через маркетплейс, і воно часто вже доступне у Codespaces або середовищах dev container. _Натисніть `Open Chat` у випадаючому меню іконки Copilot — і введіть запит, наприклад `What can you do?`_ — можливо, доведеться увійти в систему. **GitHub Copilot Chat готовий**.

### 1.2. Встановіть MCP сервери

Щоб режим агента був ефективний, йому потрібен доступ до потрібних інструментів для отримання знань або виконання дій. Тут допоможуть MCP сервери. Налаштуємо такі сервери:

1. [Azure MCP Server](../../../../../workshop/docs/instructions)
1. [Microsoft Docs MCP Server](../../../../../workshop/docs/instructions)

Для активації цих серверів:

1. Створіть файл `.vscode/mcp.json`, якщо його немає
1. Скопіюйте у цей файл наступне — і запустіть сервери!
   ```json title=".vscode/mcp.json"
   {
      "servers": {
         "Azure MCP Server": {
            "command": "npx",
            "args": [
            "-y",
            "@azure/mcp@latest",
            "server",
            "start"
            ]
         },
         "microsoft.docs.mcp": {
            "type": "http",
            "url": "https://learn.microsoft.com/api/mcp"
         }
      }
   }
   ```

??? warning "Можлива помилка, що `npx` не встановлений (клацніть, щоб побачити виправлення)"

      Щоб виправити це, відкрийте файл `.devcontainer/devcontainer.json` і додайте цей рядок у секцію features. Потім перебудуйте контейнер. Тепер `npx` має бути встановлений.

      ```title="" linenums="0"
         "features": {
            "ghcr.io/devcontainers/features/node:1": {},
            ...
         },
      ```

### 1.3. Перевірте GitHub Copilot Chat

**Спершу виконайте `azd auth login` для автентифікації в Azure через термінал VS Code. Використовуйте `az login` тільки якщо плануєте запускати Azure CLI команди безпосередньо.**

Тепер ви маєте змогу запитувати статус вашої Azure підписки та задавати питання про розгорнуті ресурси чи конфігурацію. Спробуйте такі запити:

1. `List my Azure resource groups`
1. `#foundry list my current deployments`

Ви також можете ставити питання про документацію Azure й отримувати відповіді, базовані на Microsoft Docs MCP сервері. Спробуйте ці запити:

1. `#microsoft_docs_search What is Azure Developer CLI?`
1. `#microsoft_docs_search Show me a Python tutorial to chat with deployed model`

Або попросіть код для виконання завдання. Спробуйте цей запит.

1. `Give me a Python code example that uses AAD for an interactive chat client`

У режимі `Ask` це надасть код, який можна копіювати і спробувати. У режимі `Agent` це може піти далі — і створити відповідні ресурси для вас, включно з установчими скриптами та документацією, щоб допомогти виконати завдання.

**Ви тепер готові почати дослідження репозиторію шаблону**

## 2. Розбір архітектури

??? prompt "ASK: Поясніть архітектуру додатку на docs/images/architecture.png у 1 абзаці"

      Цей додаток — це чат із штучним інтелектом, побудований на Azure, що демонструє сучасну архітектуру на базі агентів. Рішення зосереджено навколо Azure Container App, який хостить основний код додатку, що обробляє ввід користувача та генерує інтелектуальні відповіді через AI агента.
      
      Архітектура використовує Microsoft Foundry Project як основу AI можливостей, підключаючись до Azure AI Services, які надають базові мовні моделі (наприклад, gpt-4.1-mini) та функціональність агента. Взаємодія користувача проходить через React-фронтенд до FastAPI бекенду, який комунікує з AI агентом для генерації контекстних відповідей.
      
      Система включає можливості отримання знань через пошук по файлах або через Azure AI Search сервіс, що дозволяє агенту отримувати доступ і цитувати інформацію з завантажених документів.
      
      Для забезпечення робочої надійності архітектура включає комплексний моніторинг за допомогою Application Insights та Log Analytics Workspace для трасування, логування та оптимізації продуктивності.
      
      Azure Storage надає сховище blob для даних додатку та завантажень файлів, а Managed Identity забезпечує безпечний доступ між Azure ресурсами без зберігання облікових даних. Вся система розрахована на масштабованість та підтримку, з автоматичним масштабуванням контейнерного додатку залежно від навантаження, а також вбудованими функціями безпеки, моніторингу та CI/CD через екосистему керованих послуг Azure.

### 3.1. Основна архітектура додатку

Цей шаблон відповідає патерну **full-stack веб-додатка** з:

- **Бекенд**: Python FastAPI з інтеграцією Azure AI
- **Фронтенд**: TypeScript/React з системою збірки Vite
- **Інфраструктура**: Azure Bicep шаблони для хмарних ресурсів
- **Контейнеризація**: Docker для уніфікованого розгортання

### 3.2 Infra як код (bicep)

Шар інфраструктури використовує модульні шаблони **Azure Bicep**:

   - **`main.bicep`**: Оркеструє всі Azure ресурси
   - **Модулі `core/`**: Повторно використовувані компоненти для різних сервісів
      - AI сервіси (Microsoft Foundry Models, AI Search)
      - Хостинг контейнерів (Azure Container Apps)
      - Моніторинг (Application Insights, Log Analytics)
      - Безпека (Key Vault, Managed Identity)

### 3.3 Код додатку (`src/`)

**Бекенд API (`src/api/`)**:

- FastAPI REST API
- Інтеграція Foundry Agents
- Управління пошуковим індексом для отримання знань
- Можливості завантаження та обробки файлів

**Фронтенд (`src/frontend/`)**:

- Сучасний React/TypeScript SPA
- Vite для швидкої розробки та оптимізованих збірок
- Чат-інтерфейс для взаємодії з агентом

**База знань (`src/files/`)**:

- Приклади даних клієнтів та продуктів
- Демонструє доступ до знань через файли
- Приклади у форматах JSON та Markdown


### 3.4 DevOps та автоматизація

**Скрипти (`scripts/`)**:

- Кросплатформенні PowerShell та Bash скрипти
- Перевірка та налаштування середовища
- Налаштування після розгортання
- Управління квотою моделей

**Інтеграція Azure Developer CLI**:

- Конфігурація `azure.yaml` для робочих процесів `azd`
- Автоматичне створення та розгортання ресурсів
- Управління змінними середовища

### 3.5 Тестування та забезпечення якості

**Фреймворк оцінювання (`evals/`)**:

- Оцінка продуктивності агента
- Тестування якості відповідей на запити
- Автоматизована лінійка оцінок

**Безпека AI (`airedteaming/`)**:

- Тестування «червоною командою» для безпеки AI
- Перевірка на вразливості безпеки
- Практики відповідального AI
