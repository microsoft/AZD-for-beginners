## 1. Активуйте GitHub Copilot

### 1.1 Встановіть GitHub Copilot Chat

Час дослідити [GitHub Copilot з Режимом Агента](https://code.visualstudio.com/docs/copilot/chat/chat-agent-mode). Тепер ми можемо описувати наше завдання природною мовою на високому рівні і отримувати допомогу в його виконанні. Для цієї лабораторії ми будемо використовувати [безкоштовний план Copilot](https://github.com/github-copilot/signup), який має місячний ліміт на завершення та чат-взаємодії.

Розширення можна встановити з маркетплейсу, але воно вже має бути доступне у вашому середовищі Codespaces. _Натисніть `Open Chat` у випадаючому меню іконки Copilot - і введіть запит, наприклад, `What can you do?`_ - можливо, вас попросять увійти в систему. **GitHub Copilot Chat готовий до роботи**.

### 1.2. Встановіть MCP Сервери

Для ефективної роботи режиму Агента йому потрібен доступ до правильних інструментів, які допоможуть йому отримувати знання або виконувати дії. Тут на допомогу приходять MCP сервери. Ми налаштуємо наступні сервери:

1. [Azure MCP Server](../../../../../workshop/docs/instructions)
1. [Microsoft Docs MCP Server](../../../../../workshop/docs/instructions)

Щоб активувати їх:

1. Створіть файл `.vscode/mcp.json`, якщо він не існує
1. Скопіюйте вміст нижче у цей файл - і запустіть сервери!
   ```json title=".vscode/mcp.json"
   {
      "servers": {
         "Azure MCP Server": {
            "command": "npx",
            "args": [
            "-y",
            "@azure/mcp@latest",
            "server",
            "start"
            ]
         },
         "microsoft.docs.mcp": {
            "type": "http",
            "url": "https://learn.microsoft.com/api/mcp"
         }
      }
   }
   ```

??? warning "Ви можете отримати помилку, що `npx` не встановлений (натисніть для розкриття способу виправлення)"

      Щоб виправити це, відкрийте файл `.devcontainer/devcontainer.json` і додайте цей рядок в розділ features. Потім перебудуйте контейнер. Тепер у вас має бути встановлений `npx`.

      ```title="" linenums="0"
         "features": {
            "ghcr.io/devcontainers/features/node:1": {},
            ...
         },
      ```

### 1.3. Перевірка GitHub Copilot Chat

**Спершу використайте `az login` для автентифікації з Azure через командний рядок VS Code.**

Тепер ви зможете запитувати статус вашої підписки Azure та ставити питання щодо розгорнутих ресурсів або конфігурації. Спробуйте ці запити:

1. `List my Azure resource groups`
1. `#foundry list my current deployments`

Ви також можете задавати питання про документацію Azure та отримувати відповіді, засновані на Microsoft Docs MCP сервері. Спробуйте ці запити:

1. `#microsoft_docs_search What is Azure Developer CLI?`
1. `#microsoft_docs_search Show me a Python tutorial to chat with deployed model`

Або можете попросити кодові приклади для виконання завдання. Спробуйте цей запит.

1. `Give me a Python code example that uses AAD for an interactive chat client`

У режимі `Ask` це надасть код, який можна скопіювати та випробувати. У режимі `Agent` це може піти далі і створити відповідні ресурси для вас - включаючи скрипти налаштування та документацію - для допомоги у виконанні завдання.

**Тепер ви готові почати дослідження репозиторію шаблону**

## 2. Розбір архітектури

??? prompt "ASK: Поясніть архітектуру застосунку в docs/images/architecture.png одним абзацом"

      Цей застосунок є чат-додатком з підтримкою ШІ, побудованим на Azure, що демонструє сучасну архітектуру на основі агентів. Рішення зосереджене навколо Azure Container App, що розміщує основний код застосунку, який обробляє введення користувача та генерує інтелектуальні відповіді через ШІ-агента. 
      
      Архітектура використовує Microsoft Foundry Project як основу AI можливостей, підключаючись до Azure AI Services, які надають підлягаючі мовні моделі (наприклад, GPT-4o-mini) та функціональність агента. Взаємодія користувача проходить через фронтенд на React до бекенду на FastAPI, який спілкується зі службою AI агента для генерації контекстних відповідей. 
      
      Система включає можливості пошуку знань через пошук по файлах або сервіс Azure AI Search, що дозволяє агенту отримувати доступ та посилатися на інформацію з завантажених документів. Для забезпечення операційної досконалості архітектура містить комплексний моніторинг через Application Insights і Log Analytics Workspace для трасування, логування та оптимізації продуктивності. 
      
      Azure Storage надає зберігання обʼєктів для даних застосунку і завантажень файлів, а Managed Identity забезпечує безпечний доступ між ресурсами Azure без зберігання облікових даних. Весь проєкт спроектований для масштабованості та підтримуваності, з контейнеризованим застосунком, який автоматично масштабується в залежності від навантаження і надає вбудовані засоби безпеки, моніторингу та CI/CD через екосистему керованих сервісів Azure.

### 3.1. Основна архітектура застосунку

Цей шаблон слідує патерну **повноцінного веб-застосунку** з:

- **Бекенд**: Python FastAPI з інтеграцією Azure AI
- **Фронтенд**: TypeScript/React з системою збірки Vite
- **Інфраструктура**: шаблони Azure Bicep для хмарних ресурсів
- **Контейнеризація**: Docker для послідовного розгортання

### 3.2 Інфраструктура як код (bicep)

Шар інфраструктури використовує модульні шаблони **Azure Bicep**:

   - **`main.bicep`**: Оркестрація усіх ресурсів Azure
   - **модулі `core/`**: Повторно використовувані компоненти для різних служб
      - AI сервіси (Azure OpenAI, AI Search)
      - Хостинг контейнерів (Azure Container Apps)
      - Моніторинг (Application Insights, Log Analytics)
      - Безпека (Key Vault, Managed Identity)

### 3.3 Джерельний код застосунку (`src/`)

**Backend API (`src/api/`)**:

- REST API на базі FastAPI
- Інтеграція з Foundry Agents
- Керування індексом пошуку для отримання знань
- Можливості завантаження і обробки файлів

**Фронтенд (`src/frontend/`)**:

- Сучасне SPA на React/TypeScript
- Vite для швидкої розробки і оптимізованих збірок
- Інтерфейс чату для взаємодії з агентом

**База знань (`src/files/`)**:

- Приклади даних клієнтів і продуктів
- Демонструє отримання знань на основі файлів
- Приклади у форматах JSON та Markdown


### 3.4 DevOps та Автоматизація

**Скрипти (`scripts/`)**:

- Кросплатформені PowerShell та Bash скрипти
- Перевірка та налаштування середовища
- Конфігурація після розгортання
- Управління квотою моделей

**Інтеграція Azure Developer CLI**:

- Конфігурація `azure.yaml` для робочих процесів `azd`
- Автоматизоване провізіонування та розгортання
- Керування змінними середовища

### 3.5 Тестування та контроль якості

**Фреймворк оцінки (`evals/`)**:

- Оцінка продуктивності агента
- Тестування якості відповіді на запити
- Автоматизований процес оцінювання

**Безпека ШІ (`airedteaming/`)**:

- Тестування червоною командою з питань безпеки ШІ
- Сканування вразливостей безпеки
- Відповідальна практика ШІ
