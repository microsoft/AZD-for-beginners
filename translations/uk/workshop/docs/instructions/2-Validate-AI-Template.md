## 1. Вступ

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) або `azd` — це інструмент командного рядка з відкритим кодом, який спрощує робочий процес розробника при створенні та розгортанні додатків у Azure.

[AZD Шаблони](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) — це стандартизовані репозиторії, що містять приклади коду додатків, об’єкти _інфраструктура як код_, і файли конфігурації `azd` для узгодженої архітектури рішення. Забезпечення інфраструктури стає настільки простим, як команда `azd provision` — а використання `azd up` дозволяє одночасно створити інфраструктуру **і** розгорнути додаток!

Таким чином, початок розробки додатку може бути таким же простим, як пошук правильного _AZD Starter шаблону_, що найбільше відповідає вашим потребам у додатку та інфраструктурі — а потім налаштування репозиторію відповідно до ваших вимог.

Перш ніж почати, переконайтеся, що у вас встановлений Azure Developer CLI.

1. Відкрийте термінал VS Code та введіть цю команду:

      ```bash title="" linenums="0"
      azd version
      ```

1. Ви повинні побачити щось подібне!

      ```bash title="" linenums="0"
      azd version 1.19.0 (commit b3d68cea969b2bfbaa7b7fa289424428edb93e97)
      ```

**Тепер ви готові обрати та розгорнути шаблон за допомогою azd**

## 3. Активація шаблону

Спробуємо розгорнути цей шаблон і переконатися, що він справний. Слідкуємо за інструкціями в розділі [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started).

1. Натисніть [це посилання](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) — підтвердіть стандартну дію, щоб `Create codespace`
1. Відкриється нова вкладка браузера — зачекайте, поки сесія GitHub Codespaces завантажиться
1. Відкрийте термінал VS Code у Codespaces — введіть таку команду:

   ```bash title="" linenums="0"
   azd up
   ```

Виконайте кроки робочого процесу, які це запустить:

1. Вас попросять увійти в Azure — виконайте інструкції для автентифікації
1. Введіть унікальне ім’я середовища для себе — наприклад, я використав `nitya-mshack-azd`
1. Це створить папку `.azure/` — ви побачите підпапку з ім’ям середовища
1. Вас попросять обрати підписку — оберіть за замовчуванням
1. Вас попросять вказати розташування — використайте `East US 2`

Тепер чекайте завершення забезпечення. **Процес займе 10-15 хвилин**

1. Коли завершиться, у консолі з'явиться повідомлення ПРО УСПІХ, схоже на це:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. Тепер у вашому Azure порталі буде створена ресурсна група з ім’ям середовища:

      ![Infra](../../../../../translated_images/uk/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **Тепер ви готові перевірити розгорнуту інфраструктуру та додаток**.

## 5. Валідація агента

Azure Container App розгортає кінцеву точку, що підключається до AI Агента, створеного у проекті Microsoft Foundry для цього шаблону. Давайте подивимося, що це означає.

1. Поверніться до сторінки огляду Azure Portal для вашої групи ресурсів

1. Клацніть по ресурсу `Microsoft Foundry` у списку

1. Ви побачите це. Натисніть кнопку `Go to Microsoft Foundry Portal`.
   ![Foundry](../../../../../translated_images/uk/04-view-foundry-project.fb94ca41803f28f3.webp)

1. Ви побачите сторінку проекту Foundry для вашого AI додатку
   ![Project](../../../../../translated_images/uk/05-visit-foundry-portal.d734e98135892d7e.webp)

1. Натисніть на `Agents` — побачите стандартного агента, створеного у вашому проекті
   ![Agents](../../../../../translated_images/uk/06-visit-agents.bccb263f77b00a09.webp)

1. Оберіть його — і побачите деталі агента. Зверніть увагу на наступне:

      - Агент використовує Пошук по файлах за замовчуванням (завжди)
      - В поле `Knowledge` агента вказано, що завантажено 32 файли (для пошуку по файлах)
      ![Agents](../../../../../translated_images/uk/07-view-agent-details.0e049f37f61eae62.webp)

1. Знайдіть опцію `Data+indexes` у лівому меню та натисніть для перегляду деталей.

      - Ви побачите 32 файли даних, завантажені в базу знань.
      - Вони відповідають 12 файлам клієнтів та 20 файлам продуктів у папці `src/files`
      ![Data](../../../../../translated_images/uk/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**Ви перевірили роботу агента!**

1. Відповіді агента базуються на знаннях, містяться у цих файлах.
1. Тепер ви можете ставити запитання, пов’язані з цими даними, і отримувати обґрунтовані відповіді.
1. Приклад: `customer_info_10.json` описує 3 покупки, зроблені "Amanda Perez".

Поверніться до вкладки браузера з кінцевою точкою Container App і запитайте: `What products does Amanda Perez own?`. Ви повинні побачити щось подібне:

![Data](../../../../../translated_images/uk/09-ask-in-aca.4102297fc465a4d5.webp)

## 7. Вбудована спостережливість

Спостережливість означає інструментування вашого додатка для генерації даних, які можна використати для розуміння, налагодження та оптимізації його роботи. Щоб ознайомитися з цим:

1. Натисніть кнопку `View Run Info` — ви побачите цей вигляд. Це приклад [трасування агента](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) в дії. _Цей вигляд також можна отримати, натиснувши Thread Logs у верхньому меню_.

   - Оцініть кроки виконання та інструменти, задіяні агентом
   - Зрозумійте загальну кількість токенів (у порівнянні з використанням токенів для виводу) для відповіді
   - Зрозумійте затримки та де витрачається час під час виконання

      ![Agent](../../../../../translated_images/uk/10-view-run-info.b20ebd75fef6a1cc.webp)

1. Натисніть вкладку `Metadata`, щоб побачити додаткові атрибути виконання, які можуть бути корисними для налагодження проблем згодом.

      ![Agent](../../../../../translated_images/uk/11-view-run-info-metadata.7966986122c7c2df.webp)


1. Натисніть вкладку `Evaluations`, щоб побачити автоматичні оцінки відповіді агента. Вони включають оцінки безпеки (наприклад, самопошкодження) і спеціфічні для агента оцінки (наприклад, розпізнавання намірів, дотримання завдань).

      ![Agent](../../../../../translated_images/uk/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. І нарешті, натисніть вкладку `Monitoring` у бічному меню.

      - Виберіть вкладку `Resource usage` на сторінці, що відкриється — і перегляньте метрики.
      - Відстежуйте використання додатку з точки зору витрат (токенів) та навантаження (запитів).
      - Відстежуйте затримку додатка до першого байта (обробка введення) і останнього байта (вивід).

      ![Agent](../../../../../translated_images/uk/13-monitoring-resources.5148015f7311807f.webp)

## 8. Змінні оточення

Поки що ми розглядали розгортання через браузер — і перевірили, що наша інфраструктура створена, а додаток працює. Але для роботи з кодом додатку _перш за все_ потрібна конфігурація локального середовища розробки відповідними змінними, необхідними для роботи з цими ресурсами. Використання `azd` спрощує це.

1. Azure Developer CLI [використовує змінні оточення](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) для збереження і керування налаштуваннями конфігурації для розгортання додатків.

1. Змінні оточення зберігаються у файлі `.azure/<env-name>/.env` — це обмежує їх дію межами середовища з ім’ям `env-name`, використаного під час розгортання, і допомагає вам ізолювати середовища між різними цілями розгортання в одному репозиторії.

1. Змінні оточення автоматично завантажуються командою `azd` щоразу при виконанні конкретної команди (наприклад, `azd up`). Зверніть увагу, що `azd` не читає автоматично змінні оточення _на рівні ОС_ (наприклад, встановлені в оболонці) — натомість використовуйте `azd set env` та `azd get env` для передачі інформації в скриптах.


Спробуємо кілька команд:

1. Отримайте всі змінні оточення, встановлені для `azd` у цьому середовищі:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      Ви побачите щось подібне:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. Отримайте конкретне значення — наприклад, хочу дізнатись, чи встановлена змінна `AZURE_AI_AGENT_MODEL_NAME`

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      Ви побачите щось подібне — вона не встановлена за замовчуванням!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. Встановіть нову змінну оточення для `azd`. Тут ми оновлюємо назву моделі агента. _Примітка: будь-які зміни негайно відображаються у файлі `.azure/<env-name>/.env`._

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      Тепер ми повинні побачити встановлене значення:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. Зверніть увагу, що деякі ресурси зберігаються постійно (наприклад, розгортання моделей) і потребують не лише `azd up` для примусового оновлення. Спробуйте видалити спочатку початкове розгортання, а потім повторно розгорнути з оновленими змінними середовища.

1. **Оновлення** Якщо ви раніше вже розгортали інфраструктуру за допомогою azd шаблону — можна _оновити_ стан локальних змінних оточення на основі поточного стану розгортання в Azure за цією командою:

      ```bash title="" linenums="0"
      azd env refresh
      ```

      Це потужний спосіб _синхронізувати_ змінні середовища між двома або більше локальними середовищами розробки (наприклад, команда з кількох розробників) — дозволяючи розгорнутій інфраструктурі служити за істину стану змінних середовища. Учасники команди просто _оновлюють_ змінні, щоб знову синхронізуватися.
