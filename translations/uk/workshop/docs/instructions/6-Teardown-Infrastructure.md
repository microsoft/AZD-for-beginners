## Бонусні вправи

Перед тим, як видалити проект, знайдіть кілька хвилин для вільного дослідження.

!!! info "Спробуйте ці запити для досліджень"

    **Експериментуйте з GitHub Copilot:**
    
    1. Запитайте: `Які інші шаблони AZD я міг би спробувати для сценаріїв з багатьма агентами?`
    2. Запитайте: `Як я можу налаштувати інструкції агента для медичного випадку використання?`
    3. Запитайте: `Які змінні середовища контролюють оптимізацію витрат?`
    
    **Досліджуйте портал Azure:**
    
    1. Перегляньте метрики Application Insights для вашого розгортання
    2. Перевірте аналіз витрат для забезпечених ресурсів
    3. Знову дослідіть майданчик агентів порталу Microsoft Foundry

## Видалення інфраструктури

1. Зняття інфраструктури таке ж просте, як:
      
      ```bash title="" linenums="0"
      azd down --purge
      ```
1. Прапорець `--purge` гарантує, що також будуть видалені ресурси Cognitive Service, які були пом’якшено видалені, звільняючи тим самим квоти, що їх утримували ці ресурси. Після завершення ви побачите щось на кшталт цього:
      
      ```bash title="" linenums="0"
      ? Total resources to delete: 11, are you sure you want to continue? Yes
      Deleting your resources can take some time.
      (✓) Done: Deleted resource group rg-nitya-mshack-azd
      (✓) Done: Purging Cognitive Account: aoai-3cz3zkynhvpbc

      SUCCESS: Your application was removed from Azure in 11 minutes 4 seconds.
      ```

1. (За бажанням) Якщо зараз знову виконати `azd up`, ви помітите, що модель gpt-4.1 буде розгорнута, оскільки змінна середовища була змінена (і збережена) у локальній папці `.azure`.
