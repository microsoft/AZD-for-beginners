## 1. Аналогія з будівельником

Побудова сучасного підприємницького AI-застосунку _з нуля_ може бути лякаючою. Це трохи схоже на те, як самостійно зводити новий будинок цеглина за цеглиною. Так, це можливо! Але це не найефективніший спосіб досягти бажаного результату!

Замість цього ми часто починаємо з існуючого _проектного плану_ і працюємо з архітектором, щоб адаптувати його до наших особистих вимог. І саме такий підхід варто застосовувати при створенні інтелектуальних застосунків. Спочатку знайдіть гарну архітектуру, що відповідає вашій проблемній області. Потім співпрацюйте з архітектором рішень, щоб налаштувати та розробити рішення під ваш конкретний сценарій.

А де ж знайти ці проєктні плани? І як знайти архітектора, який готовий навчити нас налаштовувати та розгортати ці плани самостійно? У цьому воркшопі ми відповімо на ці питання, познайомивши вас із трьома технологіями:

1. [Azure Developer CLI](https://aka.ms/azd) – відкритий інструмент, який прискорює шлях розробника від локальної розробки (build) до розгортання в хмарі (ship).
1. [Microsoft Foundry Templates](https://ai.azure.com/templates) – стандартизовані репозиторії з відкритим кодом, що містять прикладний код, інфраструктуру та файли конфігурації для розгортання архітектури AI-рішень.
1. [GitHub Copilot Agent Mode](https://code.visualstudio.com/docs/copilot/chat/chat-agent-mode) – агент кодування на основі знань Azure, який допомагає орієнтуватися в кодовій базі та вносити зміни, використовуючи природну мову.

Зібравши ці інструменти, ми тепер можемо _знаходити_ потрібний шаблон, _розгортати_ його для перевірки працездатності та _налаштовувати_ під конкретні сценарії. Давайте розберемося, як це працює.

## 2. Azure Developer CLI

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) (або `azd`) — це інструмент з відкритим кодом для командного рядка, що прискорює ваш шлях від коду до хмари завдяки комплексу зрозумілих команд, які послідовно працюють як у середовищі розробки (IDE), так і у CI/CD (DevOps).

За допомогою `azd` ваш шлях розгортання може бути таким простим:

- `azd init` – Ініціалізує новий AI-проєкт на основі існуючого шаблону AZD.
- `azd up` – Одним кроком створює інфраструктуру та розгортає ваш застосунок.
- `azd monitor` – Отримуйте моніторинг і діагностику в режимі реального часу для розгорнутого застосунку.
- `azd pipeline config` – Налагоджуйте CI/CD пайплайни для автоматизованого розгортання в Azure.

**🎯 | ВПРАВА**: <br/> Вивчіть інструмент командного рядка `azd` у вашому поточному воркшоп-середовищі. Це може бути GitHub Codespaces, контейнер розробника або локальний клон із встановленими попередніми вимогами. Почніть з введення цієї команди, щоб дізнатися, що може інструмент:

```bash title="" linenums="0"
azd help
```

## 3. Шаблон AZD

Для роботи `azd` йому потрібно знати, яку інфраструктуру створювати, які налаштування конфігурації застосовувати і який застосунок розгортати. Ось для цього існують [шаблони AZD](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/azd-templates?tabs=csharp).

Шаблони AZD — це репозиторії з відкритим кодом, які поєднують прикладний код із файлами інфраструктури та конфігурації, необхідними для розгортання архітектури рішення.
Використовуючи підхід _Infrastructure-as-Code_ (IaC), вони дозволяють зберігати визначення ресурсів шаблону та налаштування конфігурації під системою контролю версій (так само як і код застосунку) — створюючи багаторазові та послідовні робочі процеси для користувачів проєкту.

При створенні або повторному використанні шаблону AZD для _вашого_ сценарію врахуйте такі питання:

1. Що ви будуєте? → Чи існує шаблон із початковим кодом для цього сценарію?
1. Як побудовано ваше рішення? → Чи є шаблон з необхідними ресурсами?
1. Як розгортається ваше рішення? → Подумайте про `azd deploy` з pre/post-процесами!
1. Як можна його ще оптимізувати? → Подумайте про вбудований моніторинг та автоматизацію пайплайнів!

**🎯 | ВПРАВА**: <br/> Відвідайте [Awesome AZD](https://azure.github.io/awesome-azd/) галерею та використайте фільтри, щоб дослідити понад 250 існуючих шаблонів. Спробуйте знайти той, що відповідає _вашим_ вимогам сценарію.

## 4. Шаблони AI-застосунків

Для застосунків з штучним інтелектом Microsoft надає спеціалізовані шаблони з **Microsoft Foundry** та **Foundry Agents**. Ці шаблони пришвидшують шлях створення інтелектуальних, готових до продуктивного використання застосунків.

### Шаблони Microsoft Foundry та Foundry Agents

Обрати шаблон нижче для розгортання. Кожен шаблон доступний на [Awesome AZD](https://azure.github.io/awesome-azd/) і може бути ініціалізований однією командою.

| Шаблон | Опис | Команда розгортання |
|--------|-------|---------------------|
| **[AI Chat with RAG](https://azure.github.io/awesome-azd/?tags=ai&tags=rag)** | Чат-застосунок з Retrieval Augmented Generation за допомогою Microsoft Foundry | `azd init -t azure-samples/azure-search-openai-demo` |
| **[Foundry Agent Service Starter](https://azure.github.io/awesome-azd/?tags=ai&tags=agents)** | Створення агентів ШІ з Foundry Agents для автономного виконання завдань | `azd init -t azure-samples/foundry-agent-service-starter` |
| **[Multi-Agent Orchestration](https://azure.github.io/awesome-azd/?tags=ai&tags=agents)** | Координація кількох Foundry Agents для складних робочих процесів | `azd init -t azure-samples/multi-agent-orchestration` |
| **[AI Document Intelligence](https://azure.github.io/awesome-azd/?tags=ai&tags=document)** | Витяг і аналіз документів за допомогою моделей Microsoft Foundry | `azd init -t azure-samples/ai-document-processing` |
| **[Conversational AI Bot](https://azure.github.io/awesome-azd/?tags=ai&tags=bot)** | Створення інтелектуальних чатботів з інтеграцією Microsoft Foundry | `azd init -t azure-samples/ai-chat-protocol` |
| **[AI Image Generation](https://azure.github.io/awesome-azd/?tags=ai&tags=dalle)** | Генерація зображень за допомогою DALL-E через Microsoft Foundry | `azd init -t azure-samples/ai-image-generation` |
| **[Semantic Kernel Agent](https://azure.github.io/awesome-azd/?tags=ai&tags=semantic-kernel)** | Агент ШІ, що використовує Semantic Kernel із Foundry Agents | `azd init -t azure-samples/semantic-kernel-agent` |
| **[AutoGen Multi-Agent](https://azure.github.io/awesome-azd/?tags=ai&tags=autogen)** | Багатоагентні системи на основі фреймворку AutoGen | `azd init -t azure-samples/autogen-multi-agent` |

### Швидкий початок

1. **Перегляньте шаблони**: Відвідайте [https://azure.github.io/awesome-azd/](https://azure.github.io/awesome-azd/) та відфільтруйте за `AI`, `Agents` або `Microsoft Foundry`
2. **Обрати шаблон**: Виберіть той, що відповідає вашому випадку використання
3. **Ініціалізувати**: Запустіть команду `azd init` для обраного шаблону
4. **Розгорнути**: Виконайте `azd up` для створення інфраструктури та розгортання

**🎯 | ВПРАВА**: <br/>
Обрати один із шаблонів вище, що відповідає вашому сценарію:

- **Створюєте чатбота?** → Почніть з **AI Chat with RAG** або **Conversational AI Bot**
- **Потрібні автономні агенти?** → Спробуйте **Foundry Agent Service Starter** або **Multi-Agent Orchestration**
- **Обробляєте документи?** → Використайте **AI Document Intelligence**
- **Хочете допомогу у кодуванні з ШІ?** → Дослідіть **Semantic Kernel Agent** або **AutoGen Multi-Agent**

```bash title="Example: Deploy the AI Chat with RAG template" linenums="0"
azd init -t azure-samples/azure-search-openai-demo
azd up
```
