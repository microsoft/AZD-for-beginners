# AZD для розробників ШІ - майстерня

Ласкаво просимо до практичної майстерні з вивчення Azure Developer CLI (AZD) з акцентом на розгортання AI-додатків. Ця майстерня допоможе вам здобути практичне розуміння шаблонів AZD у 3 кроки:

1. **Відкриття** – знайдіть шаблон, який підходить саме вам.
1. **Розгортання** – розгорніть і перевірте, що він працює.
1. **Налаштування** – змінюйте та адаптуйте під себе!

Під час цієї майстерні вам також буде представлено основні інструменти розробника та робочі процеси, які допоможуть оптимізувати ваш шлях розробки від початку до кінця.

<br/>

## Посібник у браузері

Уроки майстерні написані у форматі Markdown. Ви можете переглядати їх безпосередньо на GitHub або запустити браузерний перегляд, як показано на скріншоті нижче.

![Workshop](../../../translated_images/uk/workshop.75906f133e6f8ba0.webp)

Щоб скористатися цією опцією, форкніть репозиторій у свій профіль і запустіть GitHub Codespaces. Коли термінал VS Code стане активним, введіть цю команду:

```bash title="" linenums="0"
mkdocs serve > /dev/null 2>&1 &
```

Через кілька секунд зʼявиться діалогове вікно. Оберіть опцію `Open in browser`. Тепер веб-версія посібника відкриється у новій вкладці браузера. Переваги такого перегляду:

1. **Вбудований пошук** – швидко знаходьте ключові слова або уроки.
1. **Іконка копіювання** – наведіть курсор на блоки коду, щоб побачити цю опцію.
1. **Перемикач теми** – змінюйте темну і світлу теми.
1. **Отримайте підтримку** – клацніть іконку Discord у футері, щоб приєднатися!

## Чому ви навчитесь

Думайте про шаблон AZD як про навчальний майданчик для вивчення різних можливостей і інструментів кінцевої розробки на Microsoft Foundry. Наприкінці цієї майстерні ви матимете інтуїтивне розуміння різних інструментів і концепцій у цьому контексті.

| Концепція  | Мета |
|:---|:---|
| **Azure Developer CLI** | Розуміння команд інструменту та робочих процесів |
| **Шаблони AZD**| Розуміння структури проекту та конфігурації |
| **Azure AI Agent**| Розгортання проекту Microsoft Foundry |
| **Azure AI Search**| Забезпечення інженерії контексту з агентами |
| **Спостережуваність**| Вивчення трасування, моніторингу і оцінок |
| **Red Teaming**| Вивчення протидії атакам і їх пом’якшення |

## Структура майстерні

Майстерня побудована так, аби провести вас шляхом від відкриття шаблону, до розгортання, розбору і налаштування, використовуючи офіційний стартовий шаблон [Getting Started with AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents).

### [Модуль 1: Вибір шаблону ШІ](docs/instructions/1-Select-AI-Template.md) (30 хв)

- Що таке шаблони ШІ?
- Де можна знайти шаблони ШІ?
- Як почати створювати AI-агентів?
- **Лабораторна робота**: Швидкий старт з GitHub Codespaces

### [Модуль 2: Перевірка шаблону ШІ](docs/instructions/2-Validate-AI-Template.md) (30 хв)

- Що таке архітектура шаблону ШІ?
- Що таке робочий процес розробки AZD?
- Як отримати допомогу з розробки AZD?
- **Лабораторна робота**: Розгортання та перевірка шаблону AI Agents

### [Модуль 3: Розбір шаблону ШІ](docs/instructions/3-Deconstruct-AI-Template.md) (30 хв)

- Ознайомлення з середовищем у `.azure/`
- Ознайомлення з налаштуванням ресурсів у `infra/`
- Ознайомлення з конфігурацією AZD у `azure.yaml` файлах
- **Лабораторна робота**: Зміна змінних середовища та повторне розгортання

### [Модуль 4: Налаштування шаблону ШІ](docs/instructions/4-Configure-AI-Template.md) (30 хв)
- Вивчення: Retrieval Augmented Generation
- Вивчення: Оцінка агента та Red Teaming
- Вивчення: Трасування та моніторинг
- **Лабораторна робота**: Дослідження AI-Agent+спостережуваності

### [Модуль 5: Індивідуалізація шаблону ШІ](docs/instructions/5-Customize-AI-Template.md) (30 хв)
- Визначення: PRD з вимогами сценарію
- Налаштування: змінні середовища для AZD
- Реалізація: hooks життєвого циклу для додаткових завдань
- **Лабораторна робота**: Індивідуалізація шаблону для вашого сценарію

### [Модуль 6: Демонтаж інфраструктури](docs/instructions/6-Teardown-Infrastructure.md) (30 хв)
- Підсумок: Що таке шаблони AZD?
- Підсумок: Чому варто використовувати Azure Developer CLI?
- Наступні кроки: Спробуйте інший шаблон!
- **Лабораторна робота**: Демонтаж інфраструктури та очищення

## Виклик майстерні

Хочете кинути собі виклик і зробити більше? Ось кілька пропозицій проектів — або поділіться своїми ідеями з нами!

| Проект | Опис |
|:---|:---|
|1. **Розбір складного шаблону ШІ** | Використайте робочі процеси і інструменти, які ми описали, і спробуйте розгорнути, перевірити й кастомізувати інший шаблон рішення ШІ. _Що ви дізналися?_ |
|2. **Налаштування під свій сценарій**  | Спробуйте написати PRD (Документ вимог до продукту) для іншого сценарію. Потім використайте GitHub Copilot у вашому репозиторії шаблону в Agent Model - і попросіть створити робочий процес налаштування для вас. _Що ви дізналися? Як можна покращити ці рекомендації?_ |
