# Майстер-клас AZD для розробників ШІ

Ласкаво просимо на практичний майстер-клас з вивчення Azure Developer CLI (AZD) з акцентом на розгортання AI-застосунків. Цей майстер-клас допоможе вам отримати прикладні знання про шаблони AZD у 3 кроки:

1. **Відкриття** — знайдіть шаблон, який підходить саме вам.
1. **Розгортання** — розгорніть і перевірте, що все працює.
1. **Налаштування** — змінюйте і вдосконалюйте, щоб зробити його своїм!

Протягом майстер-класу ви також ознайомитесь з основними інструментами розробника та робочими процесами, щоб оптимізувати повний цикл розробки.

<br/>

## Посібник у браузері

Уроки майстер-класу написані у форматі Markdown. Ви можете переглядати їх безпосередньо на GitHub або запустити попередній перегляд у браузері, як показано на скріншоті нижче.

![Workshop](../../../translated_images/uk/workshop.75906f133e6f8ba0.webp)

Щоб скористатися цим варіантом, зробіть форк репозиторію у свій профіль і запустіть GitHub Codespaces. Після активації терміналу VS Code введіть цю команду:

Цей перегляд у браузері працює в GitHub Codespaces, dev-контейнерах або локальному клоні з встановленими Python і MkDocs.

```bash title="" linenums="0"
mkdocs serve > /dev/null 2>&1 &
```

Через кілька секунд з’явиться спливаюче діалогове вікно. Виберіть опцію `Open in browser`. Веб-гіт з гайдом тепер відкриється в новій вкладці браузера. Деякі переваги цього перегляду:

1. **Вбудований пошук** — швидко знаходьте ключові слова або уроки.
1. **Іконка копіювання** — наведіть курсор на блоки коду, щоб побачити цю опцію.
1. **Перемикач теми** — переключайтесь між темною та світлою темами.
1. **Отримати допомогу** — натисніть іконку Discord у футері, щоб приєднатися!

## Чого ви навчитесь

Думайте про шаблон AZD як про пісочницю для вивчення різних можливостей та інструментів повного циклу розробки на Microsoft Foundry. Наприкінці майстер-класу ви маєте інтуїтивно розуміти основні інструменти та концепції в цьому контексті.

| Концепція  | Мета |
|:---|:---|
| **Azure Developer CLI** | Зрозуміти команди та робочі процеси інструменту |
| **Шаблони AZD**| Зрозуміти структуру проекту та конфігурації |
| **Azure AI Agent**| Поставити та розгорнути проект Microsoft Foundry |
| **Azure AI Search**| Забезпечити контекстну інженерію з агентами |
| **Обсервабельність**| Розглянути трасування, моніторинг та оцінки |
| **Red Teaming**| Вивчити супротивне тестування і методи протидії |

## Структура майстер-класу

Майстер-клас побудовано так, щоб провести вас від відкриття шаблону до розгортання, аналізу та кастомізації — використовуючи офіційний стартовий шаблон [Getting Started with AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) як основу.

### [Модуль 1: Вибір шаблону AI](docs/instructions/1-Select-AI-Template.md) (30 хв)

- Що таке шаблони AI?
- Де можна знайти шаблони AI?
- Як почати створювати AI-агентів?
- **Лабораторна робота**: швидкий старт у Codespaces або dev контейнері

### [Модуль 2: Перевірка шаблону AI](docs/instructions/2-Validate-AI-Template.md) (30 хв)

- Що таке архітектура шаблону AI?
- Що таке робочий процес розробки AZD?
- Як отримати допомогу з AZD Development?
- **Лабораторна робота**: розгортання та перевірка шаблону AI агентів

### [Модуль 3: Аналіз шаблону AI](docs/instructions/3-Deconstruct-AI-Template.md) (30 хв)

- Дослідити середовище в `.azure/`
- Вивчити налаштування ресурсів в `infra/`
- Вивчити конфігурацію AZD у `azure.yaml`
- **Лабораторна робота**: змінити змінні середовища та перепросунути

### [Модуль 4: Налаштування шаблону AI](docs/instructions/4-Configure-AI-Template.md) (30 хв)
- Ознайомлення: Retrieval Augmented Generation
- Ознайомлення: оцінка агента та Red Teaming
- Ознайомлення: трасування та моніторинг
- **Лабораторна робота**: дослідження AI агента + обсервабельності

### [Модуль 5: Кастомізація шаблону AI](docs/instructions/5-Customize-AI-Template.md) (30 хв)
- Визначення: PRD із вимогами сценарію
- Налаштування: змінні середовища для AZD
- Реалізація: Lifecycle Hooks для додаткових завдань
- **Лабораторна робота**: кастомізація шаблону під мій сценарій

### [Модуль 6: Згорнення інфраструктури](docs/instructions/6-Teardown-Infrastructure.md) (30 хв)
- Підсумок: що таке AZD шаблони?
- Підсумок: навіщо використовувати Azure Developer CLI?
- Наступні кроки: спробуйте інший шаблон!
- **Лабораторна робота**: видалити інфраструктуру та очистити

Бажаєте взяти для себе складніше завдання? Ось кілька ідей для проектів – або поділіться своїми ідеями з нами!

| Проект | Опис |
|:---|:---|
|1. **Аналіз складного AI шаблону** | Використайте описані робочі процеси та інструменти, щоб розгорнути, перевірити та налаштувати інший AI шаблон рішення. _Що ви дізналися?_ |
|2. **Кастомізація під свій сценарій**  | Спробуйте написати PRD (документ вимог до продукту) для іншого сценарію. Потім використайте GitHub Copilot у вашому репозиторії шаблону в Agent Model — і попросіть створити для вас робочий процес кастомізації. _Що ви дізналися? Як можна покращити ці пропозиції?_ |
