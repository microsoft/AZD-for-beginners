## Вступ

Цей всебічний глосарій містить визначення термінів, понять та акронімів, що використовуються в Azure Developer CLI та розробці в хмарі Azure. Необхідний для розуміння технічної документації, виправлення помилок та ефективного спілкування про проекти azd та сервіси Azure.

## Цілі навчання

Використовуючи цей глосарій, ви зможете:
- Зрозуміти основні терміни та концепції Azure Developer CLI
- Оволодіти словником та технічними термінами розробки в хмарі Azure
- Ефективно користуватися термінологією Infrastructure as Code та деплою
- Усвідомлювати назви сервісів Azure, акроніми та їх призначення
- Отримувати визначення термінів для усунення несправностей та налагодження
- Вивчати просунуті концепції архітектури та розробки Azure

## Очікувані результати

Регулярно звертаючись до цього глосарію, ви зможете:
- Ефективно спілкуватися, використовуючи правильну термінологію Azure Developer CLI
- Кращe розуміти технічну документацію та повідомлення про помилки
- Впевнено орієнтуватися в сервісах та поняттях Azure
- Вирішувати проблеми, використовуючи відповідний технічний словник
- Вносити вклад у командні дискусії, застосовуючи точну технічну мову
- Систематично поглиблювати знання про розробку в хмарі Azure

## A

**ARM Template**  
Шаблон Azure Resource Manager. Формат Infrastructure as Code на основі JSON для декларативного визначення і розгортання ресурсів Azure.

**App Service**  
Пропозиція платформи як послуги (PaaS) Azure для хостингу веб-застосунків, REST API та мобільних бекендів без необхідності керування інфраструктурою.

**Application Insights**  
Сервіс моніторингу продуктивності застосунків Azure, що надає глибоке розуміння продуктивності, доступності та використання застосунків.

**Azure CLI**  
Інтерфейс командного рядка для керування ресурсами Azure. Використовується azd для автентифікації та деяких операцій.

**Azure Developer CLI (azd)**  
Командний інструмент, орієнтований на розробників, що прискорює процес створення та розгортання застосунків в Azure з використанням шаблонів та Infrastructure as Code.

**azure.yaml**  
Основний конфігураційний файл проекту azd, який визначає сервіси, інфраструктуру та хуки розгортання.

**Azure Resource Manager (ARM)**  
Сервіс розгортання та керування Azure, що забезпечує рівень менеджменту для створення, оновлення та видалення ресурсів.

## B

**Bicep**  
Мова специфічного домену (DSL), розроблена Microsoft для розгортання ресурсів Azure. Забезпечує простішу синтаксис порівняно з ARM шаблонами і компілюється в ARM.

**Build**  
Процес компіляції вихідного коду, встановлення залежностей та підготовки застосунків для розгортання.

**Blue-Green Deployment**  
Стратегія розгортання, що використовує два ідентичні продуктивні середовища (синє та зелене) для мінімізації простоїв і ризиків.

## C

**Container Apps**  
Безсерверний сервіс контейнерів Azure, що дозволяє запускати контейнеризовані застосунки без керування складною інфраструктурою.

**CI/CD**  
Безперервна інтеграція/безперервне розгортання. Автоматизовані практики інтеграції змін коду та розгортання застосунків.

**Cosmos DB**  
Глобально розподілена багатомодельна база даних Azure, що забезпечує комплексні SLA за пропускною здатністю, затримками, доступністю та консистентністю.

**Configuration**  
Налаштування та параметри, які контролюють поведінку застосунку та опції розгортання.

## D

**Deployment**  
Процес встановлення та налаштування застосунків і їхніх залежностей на цільовій інфраструктурі.

**Docker**  
Платформа для розробки, доставки та запуску застосунків за допомогою технології контейнеризації.

**Dockerfile**  
Текстовий файл, що містить інструкції для побудови образу Docker контейнера.

## E

**Environment**  
Цільове середовище розгортання, яке представляє конкретний екземпляр вашого застосунку (наприклад, розробка, проміжне середовище, продуктивне середовище).

**Environment Variables**  
Конфігураційні значення, збережені у вигляді ключ-значення, до яких застосунки можуть звертатися під час виконання.

**Endpoint**  
URL або мережева адреса, за якою можна отримати доступ до застосунку або сервісу.

## F

**Function App**  
Безсерверний обчислювальний сервіс Azure, що дозволяє запускати подіє-орієнтований код без керування інфраструктурою.

## G

**GitHub Actions**  
Платформа CI/CD, інтегрована з репозиторіями GitHub для автоматизації робочих процесів.

**Git**  
Розподілена система контролю версій, що використовується для відстеження змін в коді.

## H

**Hooks**  
Користувацькі скрипти або команди, що виконуються у визначені моменти життєвого циклу розгортання (передпровізія, післяпровізія, перед розгортанням, після розгортання).

**Host**  
Тип сервісу Azure, на якому буде розгорнутий застосунок (наприклад, appservice, containerapp, function).

## I

**Infrastructure as Code (IaC)**  
Практика визначення та керування інфраструктурою за допомогою коду, а не вручну.

**Init**  
Процес ініціалізації нового проекту azd, зазвичай з шаблону.

## J

**JSON**  
JavaScript Object Notation. Формат обміну даними, широко використовується у конфігураційних файлах та відповідях API.

**JWT**  
JSON Web Token. Стандарт для безпечної передачі інформації між сторонами у вигляді об'єкта JSON.

## K

**Key Vault**  
Сервіс Azure для безпечного зберігання та керування секретами, ключами та сертифікатами.

**Kusto Query Language (KQL)**  
Мова запитів, що використовується для аналізу даних у Azure Monitor, Application Insights та інших сервісах Azure.

## L

**Load Balancer**  
Сервіс, який розподіляє вхідний мережевий трафік між кількома серверами або екземплярами.

**Log Analytics**  
Сервіс Azure для збору, аналізу та реагування на телеметричні дані з хмарних та локальних середовищ.

## M

**Managed Identity**  
Функція Azure, яка надає службам Azure автоматично керовану ідентичність для аутентифікації до інших сервісів Azure.

**Microservices**  
Архітектурний підхід, при якому застосунки будуються як набір малих, незалежних сервісів.

**Monitor**  
Єдине рішення Azure для моніторингу, яке забезпечує комплексне спостереження за застосунками та інфраструктурою.

## N

**Node.js**  
Середовище виконання JavaScript, побудоване на движку V8 від Chrome, для створення серверних застосунків.

**npm**  
Менеджер пакетів для Node.js, що керує залежностями і пакетами.

## O

**Output**  
Значення, що повертаються після розгортання інфраструктури і можуть використовуватися застосунками або іншими ресурсами.

## P

**Package**  
Процес підготовки коду застосунку та залежностей до розгортання.

**Parameters**  
Вхідні значення, передані до шаблонів інфраструктури для налаштування розгортань.

**PostgreSQL**  
Відкрита реляційна система керування базами даних, підтримувана як керований сервіс в Azure.

**Provisioning**  
Процес створення та налаштування ресурсів Azure, визначених у шаблонах інфраструктури.

## Q

**Quota**  
Обмеження на кількість ресурсів, які можна створити в підписці або регіоні Azure.

## R

**Resource Group**  
Логічний контейнер для ресурсів Azure, які мають спільний життєвий цикл, дозволи та політики.

**Resource Token**  
Унікальний рядок, згенерований azd для забезпечення унікальності імен ресурсів між розгортаннями.

**REST API**  
Архітектурний стиль для створення мережевих застосунків за допомогою HTTP методів.

**Rollback**  
Процес відкату до попередньої версії застосунку або конфігурації інфраструктури.

## S

**Service**  
Компонент вашого застосунку, визначений у azure.yaml (наприклад, веб-фронтенд, API-бекенд, база даних).

**SKU**  
Одиниця зберігання запасів. Представляє різні рівні сервісів або продуктивності ресурсів Azure.

**SQL Database**  
Керована реляційна база даних Azure на основі Microsoft SQL Server.

**Static Web Apps**  
Сервіс Azure для створення та розгортання повноцінних веб-застосунків із репозиторіїв вихідного коду.

**Storage Account**  
Сервіс Azure, що надає хмарне сховище для об'єктів даних, включаючи блоби, файли, черги та таблиці.

**Subscription**  
Контейнер облікового запису Azure, який містить групи ресурсів і ресурси з відповідним управлінням доступом та обліком.

## T

**Template**  
Попередньо побудована структура проекту, що містить код застосунку, визначення інфраструктури та конфігурації для поширених сценаріїв.

**Terraform**  
Відкритий інструмент Infrastructure as Code, який підтримує кілька хмарних провайдерів, у тому числі Azure.

**Traffic Manager**  
DNS-базований балансувальник навантаження Azure для розподілу трафіку між глобальними регіонами Azure.

## U

**URI**  
Уніфікований ідентифікатор ресурсу. Рядок, який ідентифікує певний ресурс.

**URL**  
Уніфікований локатор ресурсу. Тип URI, який визначає місцезнаходження ресурсу та спосіб його отримання.

## V

**Virtual Network (VNet)**  
Основний будівельний блок приватних мереж в Azure, що забезпечує ізоляцію та сегментацію.

**VS Code**  
Visual Studio Code. Популярний редактор коду з чудовою інтеграцією Azure та azd.

## W

**Webhook**  
HTTP зворотний виклик, що запускається певними подіями, часто використовується в CI/CD конвеєрах.

**What-if**  
Функціонал Azure, який показує, які зміни будуть внесені при розгортанні, без фактичного виконання.

## Y

**YAML**  
YAML Ain't Markup Language. Стандарт читабельної серіалізації даних, який використовується у конфігураційних файлах, таких як azure.yaml.

## Z

**Zone**  
Фізично окремі розташування в межах регіону Azure, що забезпечують резервування та високу доступність.

## Контекстно-залежні терміни

### Терміни розробки
- **Hot Reload**: Автоматичне оновлення застосунків під час розробки без перезапуску
- **Build Pipeline**: Автоматизований процес збірки та тестування коду
- **Deployment Slot**: Середовище проміжного розгортання всередині App Service
- **Environment Parity**: Збереження схожості між середовищами розробки, проміжним та продуктивним

### Терміни безпеки
- **Managed Identity**: Функція Azure для автоматичного керування обліковими даними
- **Key Vault**: Безпечне сховище для секретів, ключів та сертифікатів
- **RBAC**: Контроль доступу на основі ролей для ресурсів Azure
- **Network Security Group**: Віртуальний міжмережевий екран для контролю трафіку

### Терміни моніторингу
- **Telemetry**: Автоматизований збір метрик та даних
- **Application Performance Monitoring (APM)**: Моніторинг продуктивності програмного забезпечення
- **Log Analytics**: Сервіс для збору та аналізу логів
- **Alert Rules**: Автоматичні сповіщення на основі метрик або умов

### Терміни розгортання
- **Blue-Green Deployment**: Стратегія розгортання без простою
- **Canary Deployment**: Поступове впровадження для частини користувачів
- **Rolling Update**: Послідовна заміна екземплярів застосунку
- **Rollback**: Відкат до попередньої версії застосунку
