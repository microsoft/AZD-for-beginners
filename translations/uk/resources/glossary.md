## Вступ

Цей детальний глосарій надає визначення термінів, концепцій та абревіатур, що використовуються у Azure Developer CLI та розробці в хмарі Azure. Необхідне посилання для розуміння технічної документації, усунення проблем і ефективної комунікації щодо проектів azd та сервісів Azure.

## Цілі навчання

Використовуючи цей глосарій, ви:  
- Розумітимете основну термінологію та концепції Azure Developer CLI  
- Опануєте словник Azure cloud розвитку та технічні терміни  
- Ефективно посилатиметесь на терміни Infrastructure as Code і розгортання  
- Усвідомите назви Azure сервісів, абревіатури та їх призначення  
- Отримаєте визначення для термінів усунення неполадок та налагодження  
- Вивчите просунуті концепції архітектури та розвитку Azure  

## Результати навчання

Регулярно звертаючись до цього глосарію, ви зможете:  
- Ефективно спілкуватися, використовуючи правильну термінологію Azure Developer CLI  
- Краще розуміти технічну документацію та повідомлення про помилки  
- Впевнено орієнтуватися у сервісах і концепціях Azure  
- Вирішувати проблеми, застосовуючи відповідну технічну лексику  
- Брати участь у командних обговореннях з точною технічною мовою  
- Систематично розширювати свої знання з розробки у хмарі Azure  

## A

**ARM Template**  
Шаблон Azure Resource Manager. Формат Infrastructure as Code на базі JSON для декларативного опису та розгортання ресурсів Azure.

**App Service**  
Платформа як послуга (PaaS) Azure для розміщення веб-додатків, REST API та мобільних бекендів без управління інфраструктурою.

**Application Insights**  
Служба моніторингу продуктивності додатків Azure, що надає глибокі дані про продуктивність, доступність та використання додатків.

**Azure CLI**  
Інтерфейс командного рядка для керування ресурсами Azure. Використовується azd для автентифікації та деяких операцій.

**Azure Developer CLI (azd)**  
Інструмент командного рядка, орієнтований на розробників, який прискорює процес створення та розгортання додатків в Azure із використанням шаблонів та Infrastructure as Code.

**azure.yaml**  
Основний конфігураційний файл проекту azd, який визначає сервіси, інфраструктуру та хуки розгортання.

**Azure Resource Manager (ARM)**  
Служба Azure для розгортання і керування ресурсами, що забезпечує шар управління для створення, оновлення та видалення ресурсів.

## B

**Bicep**  
Мова доменно-специфічна (DSL), розроблена Microsoft для розгортання ресурсів Azure. Має простішу синтаксис порівняно з ARM шаблонами, компілюється в ARM.

**Build**  
Процес компіляції вихідного коду, встановлення залежностей та підготовки додатків до розгортання.

**Blue-Green Deployment**  
Стратегія розгортання, що використовує два ідентичні виробничі середовища (синє та зелене) для мінімізації часу простою та ризиків.

## C

**Container Apps**  
Безсерверний сервіс контейнерів Azure, що дозволяє запускати контейнеризовані додатки без управління складною інфраструктурою.

**CI/CD**  
Безперервна інтеграція/безперервне розгортання. Автоматизовані практики інтеграції змін коду та розгортання додатків.

**Cosmos DB**  
Глобально розподілена мультимодельна база даних Azure, що надає комплексні SLA для пропускної здатності, затримки, доступності та узгодженості.

**Configuration**  
Налаштування та параметри, що керують поведінкою додатка та варіантами розгортання.

## D

**Deployment**  
Процес встановлення та налаштування додатків та їх залежностей на цільовій інфраструктурі.

**Docker**  
Платформа для розробки, доставки та запуску додатків із використанням технології контейнеризації.

**Dockerfile**  
Текстовий файл із інструкціями для побудови образу Docker контейнера.

## E

**Environment**  
Цільове середовище розгортання, що представляє конкретний екземпляр вашого додатку (наприклад, розробка, тестування, продакшн).

**Environment Variables**  
Значення конфігурації, збережені у вигляді пар ключ-значення, до яких додатки можуть звертатись під час виконання.

**Endpoint**  
URL або мережна адреса, за якою можна отримати доступ до додатку або сервісу.

## F

**Function App**  
Безсерверний обчислювальний сервіс Azure, який дозволяє запускати код на основі подій без управління інфраструктурою.

## G

**GitHub Actions**  
Платформа CI/CD, інтегрована з репозиторіями GitHub для автоматизації робочих процесів.

**Git**  
Розподілена система контролю версій для відстеження змін у вихідному коді.

## H

**Hooks**  
Користувацькі скрипти або команди, що виконуються у визначені моменти життєвого циклу розгортання (preprovision, postprovision, predeploy, postdeploy).

**Host**  
Тип сервісу Azure, на якому буде розгорнуто додаток (наприклад, appservice, containerapp, function).

## I

**Infrastructure as Code (IaC)**  
Практика визначення та керування інфраструктурою через код замість ручних процесів.

**Init**  
Процес ініціалізації нового проекту azd, зазвичай на основі шаблону.

## J

**JSON**  
JavaScript Object Notation. Формат обміну даними, що часто використовується для конфігураційних файлів та API-відповідей.

**JWT**  
JSON Web Token. Стандарт для безпечної передачі інформації між сторонами у вигляді JSON-об’єкта.

## K

**Key Vault**  
Сервіс Azure для безпечного збереження та керування секретами, ключами та сертифікатами.

**Kusto Query Language (KQL)**  
Мова запитів, що використовується для аналізу даних в Azure Monitor, Application Insights та інших сервісах Azure.

## L

**Load Balancer**  
Сервіс, що розподіляє вхідний мережевий трафік між декількома серверами або екземплярами.

**Log Analytics**  
Сервіс Azure для збору, аналізу та реагування на телеметричні дані з хмарних та локальних середовищ.

## M

**Managed Identity**  
Функція Azure, яка надає службам Azure автоматично керовану ідентичність для автентифікації до інших сервісів Azure.

**Microservices**  
Архітектурний підхід, при якому додатки створюються як набір невеликих незалежних сервісів.

**Monitor**  
Єдине рішення моніторингу Azure, що забезпечує повний стек спостереження за додатками та інфраструктурою.

## N

**Node.js**  
Середовище виконання JavaScript, побудоване на V8 двигуні Chrome, для створення серверних додатків.

**npm**  
Менеджер пакетів для Node.js, що керує залежностями і пакетами.

## O

**Output**  
Значення, які повертаються після розгортання інфраструктури і можуть використовуватись додатками або іншими ресурсами.

## P

**Package**  
Процес підготовки коду додатку та залежностей до розгортання.

**Parameters**  
Вхідні значення, що передаються в шаблони інфраструктури для налаштування розгортання.

**PostgreSQL**  
Відкрита реляційна база даних, підтримувана як керована служба в Azure.

**Provisioning**  
Процес створення та налаштування ресурсів Azure, визначених у шаблонах інфраструктури.

## Q

**Quota**  
Обмеження на кількість ресурсів, які можна створити в підписці Azure або регіоні.

## R

**Resource Group**  
Логічний контейнер для ресурсів Azure, що мають спільний життєвий цикл, дозволи та політики.

**Resource Token**  
Унікальний рядок, згенерований azd для забезпечення унікальності імен ресурсів серед розгортань.

**REST API**  
Архітектурний стиль для створення мережевих додатків за допомогою HTTP методів.

**Rollback**  
Процес відкату до попередньої версії додатку або конфігурації інфраструктури.

## S

**Service**  
Компонент вашого додатку, визначений в azure.yaml (наприклад, веб-фронтенд, API бекенд, база даних).

**SKU**  
Артикул товару. Представляє різні рівні сервісу або продуктивності для ресурсів Azure.

**SQL Database**  
Керована реляційна база даних Azure, базована на Microsoft SQL Server.

**Static Web Apps**  
Сервіс Azure для створення і розгортання повноцінних веб-додатків із репозиторіїв вихідного коду.

**Storage Account**  
Сервіс Azure для хмарного зберігання даних, включаючи блоби, файли, черги та таблиці.

**Subscription**  
Контейнер акаунта Azure, що містить групи ресурсів і ресурси, з оплатою та управлінням доступом.

## T

**Template**  
Попередньо побудована структура проекту з кодом додатка, визначенням інфраструктури та конфігурацією для поширених сценаріїв.

**Terraform**  
Відкрите програмне забезпечення Infrastructure as Code, що підтримує кілька постачальників хмари, включно з Azure.

**Traffic Manager**  
DNS-базований балансувальник навантаження Azure для розподілу трафіку між глобальними регіонами Azure.

## U

**URI**  
Універсальний ідентифікатор ресурсу. Рядок, що ідентифікує конкретний ресурс.

**URL**  
Універсальний локатор ресурсу. Тип URI, що вказує місцезнаходження ресурсу та спосіб його отримання.

## V

**Virtual Network (VNet)**  
Базовий будівельний блок приватних мереж в Azure, що забезпечує ізоляцію та сегментацію.

**VS Code**  
Visual Studio Code. Популярний редактор коду з відмінною інтеграцією Azure та azd.

## W

**Webhook**  
HTTP-зворотний виклик, що запускається певними подіями, часто використовується в CI/CD пайплайнах.

**What-if**  
Функція Azure, що показує, які зміни будуть внесені під час розгортання, не виконуючи його фактично.

## Y

**YAML**  
YAML — не розміткова мова. Людинозрозумілий стандарт серіалізації даних, що використовується для конфігураційних файлів, як azure.yaml.

## Z

**Zone**  
Фізично окремі локації в межах регіону Azure, які забезпечують резервування та високу доступність.

## Контекстно специфічні терміни

### Терміни розвитку
- **Hot Reload**: Автоматичне оновлення додатків під час розробки без перезавантаження  
- **Build Pipeline**: Автоматизований процес створення та тестування коду  
- **Deployment Slot**: Середовище для тестування в межах App Service  
- **Environment Parity**: Підтримка схожості середовищ розробки, тестування і продакшн  

### Терміни безпеки
- **Managed Identity**: Функція Azure для автоматичного керування обліковими даними  
- **Key Vault**: Безпечне сховище для секретів, ключів та сертифікатів  
- **RBAC**: Контроль доступу на основі ролей для ресурсів Azure  
- **Network Security Group**: Віртуальний фаєрвол для контролю мережевого трафіку  

### Терміни моніторингу
- **Telemetry**: Автоматичний збір замірів і даних  
- **Application Performance Monitoring (APM)**: Моніторинг продуктивності програмного забезпечення  
- **Log Analytics**: Сервіс збору та аналізу даних журналів  
- **Alert Rules**: Автоматизовані сповіщення на основі метрик або умов  

### Терміни розгортання
- **Blue-Green Deployment**: Стратегія розгортання без простою  
- **Canary Deployment**: Поступове розгортання для частини користувачів  
- **Rolling Update**: Послідовна заміна екземплярів додатку  
- **Rollback**: Відкат до попередньої версії додатку
