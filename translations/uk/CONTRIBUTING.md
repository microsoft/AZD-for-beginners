# Внесок до AZD для початківців

Дякуємо за ваш інтерес до внеску в курс AZD для початківців! Цей документ містить вказівки та інформацію для охочих долучитися.

## Як внести свій вклад

### Повідомлення про проблеми

Якщо ви знайшли помилки, недоліки або маєте пропозиції щодо покращення:

1. **Перевірте наявні проблеми**: Пошукайте [існуючі проблеми](https://github.com/microsoft/azd-for-beginners/issues), щоб уникнути дублювання
2. **Створіть нову проблему**: Використайте відповідний шаблон для issue
3. **Надайте деталі**: Включіть кроки для відтворення, очікувану поведінку та знімки екрану, якщо це доречно

### Пропозиції змін

1. **Відфоркати репозиторій**: Створіть власний форк проєкту
2. **Створити гілку**: Створіть гілку для ваших змін
3. **Внести зміни**: Дотримуйтеся наших стандартів кодування та документації
4. **Протестувати зміни**: Переконайтеся, що всі посилання працюють і вміст відображається правильно
5. **Надіслати Pull Request**: Надайте чіткий опис ваших змін

### Типи внесків

Ми вітаємо такі типи внесків:

#### Покращення контенту
- Виправлення описок, граматики та форматування
- Підвищення зрозумілості існуючої документації
- Оновлення застарілої інформації
- Додавання відсутніх пояснень або контексту

#### Нові приклади
- Додавання реальних сценаріїв розгортання
- Створення нових шаблонів AZD для поширених випадків використання
- Надання додаткових прикладів коду та вправ

#### Підтримка перекладів
- Допомога у підтримці існуючих перекладів
- Покращення якості перекладів
- Додавання перекладів для нового вмісту

#### Звіти про помилки
- Повідомлення про биті посилання або некоректні приклади коду
- Виявлення невідповідностей у документації
- Позначення проблем з доступністю

## Керівництво щодо внеску

### Стандарти документації

- Використовуйте зрозумілу, дружню для початківців мову
- Включайте навчальні цілі та очікувані результати
- Надавайте покрокові інструкції з очікуваними результатами
- Додавайте навігаційні посилання до пов’язаного вмісту
- Використовуйте послідовне форматування та стиль Markdown

### Приклади коду

- Переконайтеся, що всі приклади коду протестовані та працюють
- Додавайте коментарі, які пояснюють ключові концепції
- Дотримуйтеся найкращих практик Azure і AZD
- За можливості надавайте очікуваний результат

### Повідомлення комітів

Використовуйте чіткі, описові повідомлення комітів:
- `docs: fix typo in installation guide`
- `feat: add new container app example`
- `fix: correct broken link in chapter 2`
