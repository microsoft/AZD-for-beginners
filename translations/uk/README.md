## 🆕 Що нового в azd сьогодні

> 📌 Цей курс перевірено на версії **`azd 1.25.6`** (червень 2026). Запустіть `azd version` для перевірки вашої збірки та `azd upgrade` для оновлення до останньої.

Azure Developer CLI вийшов за межі традиційних веб-додатків та API. Сьогодні azd — це єдиний інструмент для розгортання **будь-яких** додатків в Azure — включно з додатками на основі штучного інтелекту та інтелектуальними агентами.

Ось що це означає для вас:

- **AI агенти тепер є повноцінними робочими навантаженнями azd.** Ви можете ініціалізувати, розгортати та керувати проєктами AI агентів за допомогою знайомого вам робочого процесу `azd init` → `azd up`.
- **Повний життєвий цикл агента з CLI.** Розширення `azure.ai.agents` тепер охоплює весь процес — `azd ai agent init` для створення шаблону, `azd ai agent invoke` для тестування (з показом часу відповіді), `azd ai agent eval generate` і `azd ai agent optimize` для вимірювання та підвищення якості, а також `azd ai agent delete` для очищення.
- **Більше будівельних блоків AI.** Нові попередні розширення — `azure.ai.skills` та `azure.ai.connections` — дозволяють керувати повторно використовуваними навичками агентів і підключеннями Foundry безпосередньо через azd.
- **Інтеграція з Microsoft Foundry** забезпечує розгортання моделей, хостинг агентів і налаштування AI сервісів безпосередньо в екосистемі шаблонів azd.
- **Плавніший щоденний базовий досвід.** У останніх релізах `azd init` став ідемпотентним (безпечним для повторного запуску), `azd auth login` автоматично очищує застарілі токени, а для першого запуску додано дружнє налаштування `azd tool`.
- **Основний робочий процес не змінився.** Незалежно від того, чи розгортаєте ви todo-додаток, мікросервіс або мультіагентне AI-рішення, команди залишаються тими ж.

> **Примітка для користувачів Aspire:** Microsoft тепер називає продукт просто **Aspire** (раніше ".NET Aspire"). Підтримка Aspire в azd не змінилась — змінилася лише назва.

Якщо ви вже використовували azd, підтримка AI — це природне розширення, а не окремий інструмент чи просунутий трек. Якщо ви починаєте з нуля, ви вивчите один робочий процес, який підходить для всього.

## 🚀 Що таке Azure Developer CLI (azd)?

**Azure Developer CLI (azd)** — це інструмент командного рядка, орієнтований на розробника, який спрощує розгортання додатків в Azure. Замість того, щоб вручну створювати та підключати десятки ресурсів Azure, ви можете розгорнути цілий додаток однією командою.

### Магія `azd up`

```bash
# Одна команда робить усе:
# ✅ Створює всі ресурси Azure
# ✅ Налаштовує мережу та безпеку
# ✅ Будує код вашого додатка
# ✅ Розгортає в Azure
# ✅ Надає вам робоче посилання (URL)
azd up
```

**Ось і все!** Ніяких кліків у порталі Azure, жодних складних ARM-шаблонів для вивчення, жодної ручної конфігурації — тільки робочі додатки в Azure.

## ❓ Azure Developer CLI проти Azure CLI: у чому різниця?

Це найпоширеніше питання серед початківців. Ось проста відповідь:

| Функція | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **Призначення** | Керування окремими ресурсами Azure | Розгортання повних додатків |
| **Підхід** | Орієнтований на інфраструктуру | Орієнтований на додаток |
| **Приклад** | `az webapp create --name myapp...` | `azd up` |
| **Крива навчання** | Треба знати Azure сервіси | Просто знай свій додаток |
| **Для кого** | DevOps, інфраструктура | Розробники, прототипування |

### Проста аналогія

- **Azure CLI** — це як мати всі інструменти для будівництва будинку — молотки, пилки, цвяхи. Ти можеш побудувати що завгодно, але потрібно знати будівництво.
- **Azure Developer CLI** — це як найняти підрядника — ви описуєте, що хочете, а він займається будівництвом.

### Коли що використовувати

| Сценарій | Використовуйте це |
|----------|------------------|
| "Хочу швидко розгорнути веб-додаток" | `azd up` |
| "Мені потрібно просто створити обліковий запис сховища" | `az storage account create` |
| "Я будую повний AI-додаток" | `azd init --template azure-search-openai-demo` |
| "Мені потрібно відлагодити конкретний ресурс Azure" | `az resource show` |
| "Хочу розгортання, готове для виробництва, за лічені хвилини" | `azd up --environment production` |

### Вони працюють разом!

AZD використовує Azure CLI під капотом. Ви можете користуватися обома:
```bash
# Розгорніть свій додаток за допомогою AZD
azd up

# Потім налаштуйте конкретні ресурси за допомогою Azure CLI
az webapp config set --name myapp --always-on true
```

## 🎯 Початок роботи у 3 кроки

Перш ніж почати, переконайтеся, що ваш комп’ютер готовий для шаблону, який ви хочете розгорнути:

**Windows:**
```powershell
.\validate-setup.ps1
```

**macOS / Linux:**
```bash
bash ./validate-setup.sh
```

Якщо якась перевірка не пройшла, спочатку виправте це, а потім продовжуйте швидкий старт.

### Крок 1: Встановіть AZD (2 хвилини)

**Windows:**
```powershell
winget install microsoft.azd
```

**macOS:**
```bash
brew tap azure/azd && brew install azd
```

**Linux:**
```bash
curl -fsSL https://aka.ms/install-azd.sh | bash
```

### Крок 2: Авторизуйтеся для AZD

```bash
# Необов'язково, якщо ви плануєте використовувати команди Azure CLI безпосередньо в цьому курсі
az login

# Необхідно для робочих процесів AZD
azd auth login
```

Якщо не впевнені, що саме вам потрібно, дотримуйтесь повного процесу налаштування в розділі [Installation & Setup](docs/chapter-01-foundation/installation.md#authentication-setup).

### Крок 3: Розгорніть свій перший додаток

```bash
# Ініціалізувати з шаблону
azd init --template todo-nodejs-mongo

# Розгорнути в Azure (створює все!)
azd up
```

**🎉 Готово!** Ваш додаток тепер живий в Azure.

### Очистка (не забудьте!)

```bash
# Remove all resources when done experimenting
azd down --force --purge
```

## Огляд курсу

Опануйте Azure Developer CLI (azd) через структуровані розділи, розроблені для послідовного навчання. **Особлива увага приділяється розгортанню AI-додатків з інтеграцією Microsoft Foundry.**
### Чому цей курс є необхідним для сучасних розробників

На основі інформації з спільноти Microsoft Foundry Discord, **45% розробників хочуть використовувати AZD для AI-навантажень**, але стикаються з проблемами:
- Складні багатосервісні AI-архітектури
- Кращі практики розгортання AI у виробництво  
- Інтеграція та конфігурація служби Azure AI
- Оптимізація вартості AI-навантажень
- Усунення несправностей специфічних для розгортання AI

### Навчальні цілі

Виконавши цей структурований курс, ви зможете:
- **Опанувати основи AZD**: ключові концепції, встановлення та конфігурація
- **Розгортати AI-застосунки**: використовувати AZD з Microsoft Foundry сервісами
- **Реалізувати інфраструктуру як код**: керувати ресурсами Azure за допомогою шаблонів Bicep
- **Вирішувати проблеми розгортання**: усувати типові помилки та відлагоджувати
- **Оптимізувати для виробництва**: безпека, масштабування, моніторинг та управління витратами
- **Створювати багаток агентні рішення**: розгортати складні AI-архітектури

## Перед початком: акаунти, доступ і припущення

Перш ніж почати Розділ 1, переконайтеся, що у вас є наступне. Кроки встановлення в цьому посібнику передбачають, що базові вимоги вже виконані.

- **Підписка Azure**: можете використати існуючу підписку з роботи або власну, або створити [безкоштовну пробну версію](https://aka.ms/azurefreetrial) для початку.
- **Дозвіл на створення ресурсів у Azure**: для більшості вправ потрібен принаймні доступ **Contributor** до цільової підписки або групи ресурсів. Деякі розділи можуть також вимагати можливість створювати групи ресурсів, керовані ідентичності та RBAC-призначення.
- [**Акаунт GitHub**](https://github.com): корисний для форків репозиторію, відстеження власних змін і використання GitHub Codespaces для воркшопу.
- **Необхідні інструменти для запуску шаблонів**: деякі шаблони потребують локальних інструментів, таких як Node.js, Python, Java або Docker. Запустіть перевірку налаштувань перед початком, щоб виявити відсутні інструменти.
- **Базове знайомство з терміналом**: не потрібно бути експертом, але ви маєте комфортно виконувати команди, наприклад, `git clone`, `azd auth login` та `azd up`.

> **Працюєте у корпоративній підписці?**  
> Якщо ваша Azure-середовище управляється адміністратором, заздалегідь підтвердіть, що зможете розгортати ресурси у підписці або групі ресурсів, яку плануєте використовувати. Якщо ні, попросіть про пісочницю або доступ Contributor перед початком.

> **Новачок у Azure?**

### 🚀 Розділ 1: Основи та швидкий старт  
**Вимоги**: підписка Azure, базове знання командного рядка  
**Тривалість**: 30-45 хвилин  
**Складність**: ⭐

#### Чому ви навчитеся
- Розуміння основ Azure Developer CLI
- Встановлення AZD на вашу платформу
- Ваше перше успішне розгортання

#### Навчальні матеріали
- **🎯 Почніть тут**: [Що таке Azure Developer CLI?](#what-is-azure-developer-cli)
- **📖 Теорія**: [Основи AZD](docs/chapter-01-foundation/azd-basics.md) – ключові поняття та термінологія
- **⚙️ Налаштування**: [Встановлення та налаштування](docs/chapter-01-foundation/installation.md) – керівництва для платформи
- **🛠️ Практика**: [Ваш перший проєкт](docs/chapter-01-foundation/first-project.md) – покроковий підручник
- **📋 Короткий довідник**: [Шпаргалка команд](resources/cheat-sheet.md)

#### Практичні вправи
```bash
# Швидка перевірка встановлення
azd version

# Розгортання вашого першого додатку
azd init --template todo-nodejs-mongo
azd up
```

**💡 Результат розділу**: Успішно розгорнути простий вебдодаток в Azure з AZD

**✅ Критерій успіху:**  
```bash
# Після завершення Розділу 1 ви зможете:
azd version              # Показує встановлену версію
azd init --template todo-nodejs-mongo  # Ініціалізує проєкт
azd up                  # Розгортає в Azure
azd show                # Відображає URL запущеного додатка
# Додаток відкривається в браузері і працює
azd down --force --purge  # Очищає ресурси
```
  
**📊 Час на виконання:** 30-45 хв  
**📈 Рівень навичок після:** Здатність самостійно розгортати базові застосунки

- **📖 Патерни**: [Розгортання AI моделей](docs/chapter-02-ai-development/ai-model-deployment.md) – розгортання та управління AI моделями
- **🛠️ Воркшоп**: [Лабораторія AI Workshop](docs/chapter-02-ai-development/ai-workshop-lab.md) – зробіть ваші AI рішення готовими для AZD
- **🎥 Інтерактивне керівництво**: [Матеріали воркшопу](workshop/README.md) – навчання в браузері з середовищем MkDocs * DevContainer
- **📋 Шаблони**: [Microsoft Foundry Шаблони](#ресурси-воркшопу)

#### Практичні вправи
```bash
# Розгорніть свій перший AI-додаток
azd init --template azure-search-openai-demo
azd up

# Спробуйте додаткові шаблони AI
azd init --template openai-chat-app-quickstart
azd init --template agent-openai-python-prompty
```

**💡 Результат розділу**: Розгорнути та налаштувати AI-чат додаток з можливістю RAG

**✅ Критерій успіху:**  
```bash
# Після Розділу 2 ви повинні вміти:
azd init --template azure-search-openai-demo
azd up
# Тестувати інтерфейс чат-бота з ШІ
# Задавати питання та отримувати відповіді зі ШІ з посиланнями на джерела
# Перевірити, що інтеграція пошуку працює
azd monitor  # Переконатися, що Application Insights показує телеметрію
azd down --force --purge
```
  
**📊 Час на виконання:** 1-2 години  
**📈 Рівень навичок після:** Здатність розгортати і налаштовувати AI-додатки, готові до виробництва  
**💰 Усвідомлення витрат:** Розуміння вартості девелопменту $80-150/місяць, виробництва $300-3500/місяць

#### 💰 Питання вартості для AI-розгортань

**Середовище розробки (орієнтовно $80-150/місяць):**  
- Моделі Microsoft Foundry (плата за використання): $0-50/місяць (залежно від токенів)  
- AI Search (базовий рівень): $75/місяць  
- Container Apps (споживання): $0-20/місяць  
- Сховище (стандарт): $1-5/місяць  

**Середовище виробництва (орієнтовно $300-3500+/місяць):**  
- Моделі Microsoft Foundry (PTU для стабільної продуктивності): $3000+/місяць АБО плата за використання при великому обсязі  
- AI Search (стандартний рівень): $250/місяць  
- Container Apps (виділене): $50-100/місяць  
- Application Insights: $5-50/місяць  
- Сховище (преміум): $10-50/місяць  

**💡 Поради з оптимізації витрат:**  
- Використовуйте **безкоштовний рівень** Microsoft Foundry Models для навчання (уключено 50,000 токенів Azure OpenAI)  
- Виконуйте `azd down` для відключення ресурсів, коли не працюєте активно  
- Починайте з оплати за споживання, переходьте на PTU тільки для виробництва  
- Використовуйте `azd provision --preview` для оцінки витрат перед розгортанням  
- Ввімкніть автоскейлінг: сплачуйте лише за фактичне використання  

**Моніторинг витрат:**  
```bash
# Перевірте приблизні місячні витрати
azd provision --preview

# Відстежуйте фактичні витрати в порталі Azure
az consumption budget list --resource-group <your-rg>
```

## 📖 Поглиблено: Можливості AZD

Поза базовими функціями, AZD надає потужні засоби для виробничих впроваджень:

- **Впровадження за шаблонами** - Використання готових шаблонів для типових патернів застосунків  
- **Інфраструктура як код** - Керування ресурсами Azure через Bicep або Terraform  
- **Інтегровані робочі процеси** - Плавне створення, впровадження та моніторинг застосунків  
- **Зручність для розробників** - Оптимізовано для продуктивності та зручності розробки  

### **AZD + Microsoft Foundry: ідеально для AI-впроваджень**

**Чому AZD для AI-рішень?** AZD вирішує головні виклики, з якими стикаються AI-розробники:

- **Готові AI-шаблони** - Передналаштовані шаблони для Microsoft Foundry Models, Azure AI Services та ML-робочих навантажень  
- **Безпечні AI-впровадження** - Вбудовані шаблони безпеки для AI-сервісів, API ключів і кінцевих точок моделей  
- **Виробничі AI-патерни** - Найкращі практики для масштабованих, економних при використанні AI-застосунків  
- **Повні AI-робочі процеси** - Від розробки моделей до виробничого впровадження з належним моніторингом  
- **Оптимізація витрат** - Розумний розподіл ресурсів і масштабу для AI-навантажень  
- **Інтеграція Microsoft Foundry** - Безшовне підключення до каталогу моделей і кінцевих точок Microsoft Foundry

## 🔧 Швидкий посібник з усунення несправностей

**Поширені проблеми, з якими стикаються початківці, та швидкі рішення:**

<details>
<summary><strong>❌ "azd: команда не знайдена"</strong></summary>

```bash
# Спочатку встановіть AZD
# Windows (PowerShell):
winget install microsoft.azd

# macOS:
brew tap azure/azd && brew install azd

# Linux:
curl -fsSL https://aka.ms/install-azd.sh | bash

# Перевірте встановлення
azd version
```
</details>

<details>
<summary><strong>❌ "Підписка не знайдена" або "Підписка не встановлена"</strong></summary>

```bash
# Перелік доступних підписок
az account list --output table

# Встановити підписку за замовчуванням
az account set --subscription "<subscription-id-or-name>"

# Встановити для середовища AZD
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# Перевірити
az account show
```
</details>

<details>
<summary><strong>❌ "InsufficientQuota" або "Перевищено квоту"</strong></summary>

```bash
# Спробуйте інший регіон Azure
azd env set AZURE_LOCATION "westus2"
azd up

# Або використовуйте менші SKUs у розробці
# Відредагуйте infra/main.parameters.json:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>

<details>
<summary><strong>❌ "azd up" завершується помилкою посередині</strong></summary>

```bash
# Варіант 1: Очистити та повторити спробу
azd down --force --purge
azd up

# Варіант 2: Просто виправити інфраструктуру
azd provision

# Варіант 3: Перевірити детальний статус
azd show

# Варіант 4: Перевірити журнали в Azure Monitor
azd monitor --logs
```
</details>

<details>
<summary><strong>❌ "Помилка автентифікації" або "Термін дії токена минув"</strong></summary>

```bash
# Повторно автентифікуватися для AZD
azd auth logout
azd auth login

# За потреби: оновіть також Azure CLI, якщо ви виконуєте команди az
az logout
az login

# Перевірте автентифікацію
az account show
```
</details>

<details>
<summary><strong>❌ "Ресурс вже існує" або конфлікти імен</strong></summary>

```bash
# AZD генерує унікальні імена, але якщо виникає конфлікт:
azd down --force --purge

# Тоді спробуйте ще раз з новим середовищем
azd env new dev-v2
azd up
```
</details>

<details>
<summary><strong>❌ Розгортання шаблону триває надто довго</strong></summary>

**Звичайний час очікування:**
- Простий веб-додаток: 5-10 хвилин
- Додаток з базою даних: 10-15 хвилин
- AI-додатки: 15-25 хвилин (проміжок на provisioning OpenAI довший)

```bash
# Перевірте прогрес
azd show

# Якщо затримка триває понад 30 хвилин, перевірте портал Azure:
azd monitor --overview
# Перевірте на наявність невдалих розгортань
```
</details>

<details>
<summary><strong>❌ "Доступ заборонено" або "Заборонено"</strong></summary>

```bash
# Перевірте вашу роль в Azure
az role assignment list --assignee $(az account show --query user.name -o tsv)

# Вам потрібна мінімальна роль "Співавтор"
# Попросіть адміністратора Azure надати:
# - Співавтор (для ресурсів)
# - Адміністратор доступу користувачів (для призначення ролей)
```
</details>

<details>
<summary><strong>❌ Не вдається знайти URL розгорнутого додатку</strong></summary>

```bash
# Показати всі кінцеві точки сервісу
azd show

# Або відкрити портал Azure
azd monitor

# Перевірити конкретний сервіс
azd env get-values
# Шукати змінні *_URL
```
</details>
