# Розділ 6: Планування та перевірка перед розгортанням

**📚 Курс**: [AZD для початківців](../../README.md) | **⏱️ Тривалість**: 1 година | **⭐ Складність**: Середній рівень

---

## Огляд

У цьому розділі розглядаються основні етапи планування та перевірки перед розгортанням вашого додатку. Навчіться уникати дорогих помилок за допомогою належного планування потужностей, вибору SKU та передпольотних перевірок.

## Навчальні цілі

Після проходження цього розділу ви зможете:
- Проводити передпольотні перевірки перед розгортанням
- Планувати потужності та оцінювати потреби у ресурсах
- Вибирати відповідні SKU для оптимізації витрат
- Налаштовувати Application Insights для моніторингу
- Розуміти схеми координації командної роботи

## 🚀 Швидкий старт

```bash
# Перевірте квоти підписки
az vm list-usage --location eastus --output table

# Попередній перегляд розгортання (ресурси не створюються)
azd provision --preview

# Перевірка синтаксису Bicep
az bicep build --file infra/main.bicep

# Перевірка конфігурації середовища
azd env get-values
```

## ☑️ Контрольний список перед розгортанням

### Перед `azd provision`

- [ ] Перевірена квота для регіону
- [ ] Правильно обрані SKU
- [ ] Переглянута оцінка вартості
- [ ] Узгоджена конвенція іменування
- [ ] Налаштовано безпеку / RBAC

### Перед `azd deploy`

- [ ] Встановлені змінні оточення
- [ ] Секрети в Key Vault
- [ ] Перевірені рядки підключення
- [ ] Налаштовані перевірки стану
