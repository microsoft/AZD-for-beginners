# Розділ 6: Планування та перевірка перед розгортанням

**📚 Курс**: [AZD Для початківців](../../README.md) | **⏱️ Тривалість**: 1 година | **⭐ Складність**: Середній рівень

---

## Огляд

У цьому розділі розглядаються важливі кроки планування та перевірки перед розгортанням вашого застосунку. Навчіться уникати дорогих помилок за допомогою правильного планування потужностей, вибору SKU та попередніх перевірок.

> Перевірено за `azd 1.25.6` у червні 2026 року.

## Цілі навчання

Виконавши цей розділ, ви зможете:
- Запускати попередні перевірки перед розгортанням
- Планувати потужності й оцінювати потреби у ресурсах
- Вибирати відповідні SKU для оптимізації витрат
- Налаштовувати Application Insights для моніторингу
- Розуміти патерни координації команди

## 🚀 Швидкий старт

```bash
# Перевірити квоти підписки
az vm list-usage --location eastus --output table

# Попередній перегляд розгортання (ресурси не створюються)
azd provision --preview

# Перевірити синтаксис Bicep
az bicep build --file infra/main.bicep

# Перевірити конфігурацію середовища
azd env get-values
```

## ☑️ Перелік перевірок перед розгортанням

### Перед `azd provision`

- [ ] Перевірено квоти для регіону
- [ ] Відповідно вибрані SKU
- [ ] Переглянута оцінка вартості
- [ ] Конвенції найменування послідовні
- [ ] Налаштовано безпеку/RBAC

### Перед `azd deploy`

- [ ] Встановлені змінні середовища
- [ ] Секрети у Key Vault
- [ ] Перевірені рядки підключення
- [ ] Налаштовано перевірки стану
