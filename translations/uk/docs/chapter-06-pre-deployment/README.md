# Розділ 6: Планування та валідація перед розгортанням

**📚 Курс**: [AZD для початківців](../../README.md) | **⏱️ Тривалість**: 1 година | **⭐ Складність**: Середній рівень

---

## Огляд

Цей розділ охоплює основні кроки планування та валідації перед розгортанням вашого застосунку. Навчіться уникати дорогих помилок за допомогою правильного планування потужностей, вибору SKU та перевірок перед польотом.

> Перевірено на `azd 1.27.1` у липні 2026 року.

## Цілі навчання

Завершивши цей розділ, ви зможете:
- Виконувати перевірки перед розгортанням
- Планувати потужності та оцінювати вимоги до ресурсів
- Вибирати відповідні SKU для оптимізації витрат
- Налагоджувати Application Insights для моніторингу
- Розуміти патерни координації команди

## 🚀 Швидкий старт

```bash
# Перевірте квоти підписки
az vm list-usage --location eastus --output table

# Попередній перегляд розгортання (ресурси не створюються)
azd provision --preview

# Перевірте синтаксис Bicep
az bicep build --file infra/main.bicep

# Перевірте конфігурацію середовища
azd env get-values
```

## ☑️ Контрольний список перед розгортанням

### Перед `azd provision`

- [ ] Перевірена квота для регіону
- [ ] Вибрані відповідні SKU
- [ ] Переглянута оцінка вартості
- [ ] Послідовність у схемі найменувань
- [ ] Налаштована безпека/RBAC

### Перед `azd deploy`

- [ ] Встановлені змінні середовища
- [ ] Секрети у Key Vault
- [ ] Перевірені рядки підключення
- [ ] Налаштовані перевірки стану
