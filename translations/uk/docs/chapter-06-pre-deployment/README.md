# Розділ 6: Планування та перевірка перед розгортанням

**📚 Course**: [AZD для початківців](../../README.md) | **⏱️ Duration**: 1 година | **⭐ Complexity**: Середній

---

## Огляд

Цей розділ охоплює основні кроки планування та перевірки перед розгортанням вашого застосунку. Навчіться уникати дорогих помилок за допомогою належного планування потужностей, вибору SKU та попередніх перевірок.

## Цілі навчання

Виконавши цей розділ, ви зможете:
- Запустити попередні перевірки перед розгортанням
- Запланувати потужності та оцінити потреби в ресурсах
- Вибрати відповідні SKU для оптимізації витрат
- Налаштувати Application Insights для моніторингу
- Зрозуміти шаблони координації команди

## 🚀 Швидкий старт

```bash
# Перевірити квоти підписки
az vm list-usage --location eastus --output table

# Попередній перегляд розгортання (без створення ресурсів)
azd provision --preview

# Перевірити синтаксис Bicep
az bicep build --file infra/main.bicep

# Перевірити конфігурацію середовища
azd env get-values
```

## ☑️ Контрольний список перед розгортанням

### Перед `azd provision`

- [ ] Квоту перевірено для регіону
- [ ] SKU обрані належним чином
- [ ] Переглянуто оцінку витрат
- [ ] Конвенція найменувань узгоджена
- [ ] Налаштовано безпеку/RBAC

### Перед `azd deploy`

- [ ] Встановлено змінні середовища
- [ ] Секрети розміщено в Key Vault
- [ ] Перевірено рядки підключення
- [ ] Налаштовано перевірки працездатності
