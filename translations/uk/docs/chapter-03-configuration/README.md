# Розділ 3: Конфігурація та автентифікація

**📚 Курс**: [AZD для початківців](../../README.md) | **⏱️ Тривалість**: 45-60 хвилин | **⭐ Складність**: Середній рівень

---

## Огляд

У цьому розділі розглядається конфігурація середовища, шаблони автентифікації та найкращі практики безпеки для розгортань із використанням Azure Developer CLI.

> Перевірено на `azd 1.25.6` у червні 2026 року.

## Цілі навчання

Після вивчення цього розділу ви зможете:
- Опановувати ієрархію конфігурації AZD
- Керувати кількома середовищами (dev, staging, prod)
- Впроваджувати безпечну автентифікацію з керованими ідентичностями
- Налаштовувати параметри, специфічні для середовища

## 🚀 Швидкий старт

```bash
# Створити кілька середовищ
azd env new dev
azd env new staging
azd env new prod

# Перемикати середовища
azd env select prod

# Встановити змінні середовища
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# Переглянути конфігурацію
azd env get-values
```

---

## 🔧 Ієрархія конфігурації

AZD застосовує налаштування в такому порядку (пізніше перекриває раніше):

1. **Значення за замовчуванням** (вбудовані в шаблони)
2. **azure.yaml** (конфігурація проекту)
3. **Змінні середовища** (`azd env set`)
4. **Параметри командного рядка** (`--location eastus`)

---

## 🔐 Найкращі практики безпеки

```bash
# Використовуйте керовану ідентичність (рекомендується)
azd env set AZURE_USE_MANAGED_IDENTITY true

# Перевірте статус автентифікації AZD
azd auth status

# За бажанням: перевірте контекст Azure CLI, якщо плануєте запускати команди az
az account show

# Повторно автентифікуйтесь за потреби
azd auth login

# За бажанням: оновіть автентифікацію Azure CLI для команд az
az login
```
