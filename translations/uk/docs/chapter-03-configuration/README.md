# Розділ 3: Налаштування та автентифікація

**📚 Курс**: [AZD для початківців](../../README.md) | **⏱️ Тривалість**: 45-60 хвилин | **⭐ Складність**: Середній рівень

---

## Огляд

У цьому розділі описано налаштування середовища, шаблони автентифікації та найкращі практики безпеки для розгортання Azure Developer CLI.

> Перевірено на `azd 1.27.1` у липні 2026 року.

## Цілі навчання

Завершивши цей розділ, ви:
- Опануєте ієрархію налаштувань AZD
- Керуватимете кількома середовищами (розробка, тестування, продакшн)
- Впровадите безпечну автентифікацію за допомогою керованих ідентичностей
- Налаштуєте параметри, специфічні для середовищ

## 🚀 Швидкий старт

```bash
# Створити кілька середовищ
azd env new dev
azd env new staging
azd env new prod

# Перемкнути середовища
azd env select prod

# Встановити змінні середовища
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# Переглянути конфігурацію
azd env get-values
```

---

## 🔧 Ієрархія налаштувань

AZD застосовує налаштування у такому порядку (пізніші переважають ранні):

1. **Значення за замовчуванням** (вбудовані у шаблони)
2. **azure.yaml** (конфігурація проєкту)
3. **Змінні середовища** (`azd env set`)
4. **Параметри командного рядка** (`--location eastus`)

---

## 🔐 Найкращі практики безпеки

```bash
# Використовуйте керовану ідентичність (рекомендується)
azd env set AZURE_USE_MANAGED_IDENTITY true

# Перевірте стан автентифікації AZD
azd auth status

# Необов’язково: перевірте контекст Azure CLI, якщо плануєте виконувати команди az
az account show

# Повторно автентифікуйтеся за потреби
azd auth login

# Необов’язково: оновіть автентифікацію Azure CLI для команд az
az login
```
