# Розділ 3: Конфігурація та автентифікація

**📚 Курс**: [AZD Для початківців](../../README.md) | **⏱️ Тривалість**: 45-60 хвилин | **⭐ Складність**: Середній рівень

---

## Огляд

У цьому розділі розглядаються конфігурація середовища, патерни автентифікації та кращі практики безпеки для розгортання Azure Developer CLI.

> Перевірено на `azd 1.23.12` у березні 2026 року.

## Цілі навчання

Після проходження цього розділу ви зможете:
- Опановувати ієрархію конфігурації AZD
- Керувати кількома середовищами (dev, staging, prod)
- Реалізовувати безпечну автентифікацію з керованими ідентичностями
- Налаштовувати параметри, специфічні для середовища

## 🚀 Швидкий старт

```bash
# Створити кілька середовищ
azd env new dev
azd env new staging
azd env new prod

# Перемикати середовища
azd env select prod

# Встановити змінні середовища
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# Переглянути конфігурацію
azd env get-values
```

---

## 🔧 Ієрархія конфігурації

AZD застосовує налаштування в такому порядку (пізніші перекривають попередні):

1. **Значення за замовчуванням** (вбудовані в шаблони)
2. **azure.yaml** (конфігурація проєкту)
3. **Змінні середовища** (`azd env set`)
4. **Параметри командного рядка** (`--location eastus`)

---

## 🔐 Кращі практики безпеки

```bash
# Використовуйте керовану ідентичність (рекомендується)
azd env set AZURE_USE_MANAGED_IDENTITY true

# Перевірте стан автентифікації AZD
azd auth status

# Необов’язково: перевірте контекст Azure CLI, якщо плануєте виконувати команди az
az account show

# Переавторизуйтесь, якщо потрібно
azd auth login

# Необов’язково: оновіть автентифікацію Azure CLI для команд az
az login
```
