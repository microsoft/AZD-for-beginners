# Розділ 7: Вирішення проблем та відлагодження

**📚 Курс**: [AZD для початківців](../../README.md) | **⏱️ Тривалість**: 1-1.5 години | **⭐ Складність**: Середній рівень

---

## Огляд

Цей розділ допоможе вам діагностувати та усувати поширені проблеми при роботі з Azure Developer CLI. Від збоїв розгортання до специфічних проблем з AI.

> Перевірено для `azd 1.23.12` у березні 2026 року.

## Навчальні цілі

Після проходження цього розділу ви зможете:
- Діагностувати поширені помилки розгортання AZD
- Відлагоджувати проблеми з автентифікацією та дозволами
- Усувати проблеми з підключенням до AI сервісів
- Використовувати Azure Portal та CLI для усунення неполадок

## 🚨 Швидкі виправлення

### Проблеми автентифікації
```bash
# Необхідно для робочих процесів AZD
azd auth login

# Необов’язково, якщо ви також безпосередньо використовуєте команди Azure CLI
az login

azd auth status
```

### Збої при підготовці середовища
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### Конфлікти ресурсів
```bash
azd down --force --purge
azd env new different-name
azd up
```

### Перевищення квоти
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 Довідник кодів помилок

| Помилка | Причина | Рішення |
|-------|-------|----------|
| `AuthenticationError` | Не виконано вхід | `azd auth login` |
| `ResourceNotFound` | Відсутній ресурс | Перевірте імена ресурсів |
| `QuotaExceeded` | Обмеження підписки | Запросіть збільшення квоти |
| `InvalidTemplate` | Помилка синтаксису Bicep | `az bicep build` |
| `Conflict` | Ресурс існує | Використайте нове ім'я або видаліть |
| `Forbidden` | Недостатньо дозволів | Перевірте ролі RBAC |

## 🔄 Скидання та відновлення

```bash
# М’яке скидання (зберегти ресурси, повторно розгорнути код)
azd deploy --force

# Жорстке скидання (видалити все, почати спочатку)
azd down --force --purge
azd up
```
