## Gambaran Kursus

Kuasi Azure Developer CLI (azd) melalui bab-bab berstruktur yang direka untuk pembelajaran berperingkat. **Tumpuan khusus pada penyebaran aplikasi AI dengan integrasi Microsoft Foundry.**

### Mengapa Kursus Ini Penting untuk Pembangun Moden

Berdasarkan pandangan komuniti Microsoft Foundry Discord, **45% pembangun ingin menggunakan AZD untuk beban kerja AI** tetapi menghadapi cabaran dengan:
- Seni bina AI pelbagai perkhidmatan yang kompleks
- Amalan terbaik penyebaran AI ke produksi  
- Integrasi dan konfigurasi perkhidmatan Azure AI
- Pengoptimuman kos untuk beban kerja AI
- Menyelesaikan isu khusus penyebaran AI

### Objektif Pembelajaran

Dengan menyelesaikan kursus berstruktur ini, anda akan:
- **Menguasai Asas AZD**: Konsep teras, pemasangan, dan konfigurasi
- **Menyebarkan Aplikasi AI**: Menggunakan AZD dengan perkhidmatan Microsoft Foundry
- **Melaksanakan Infrastruktur sebagai Kod**: Mengurus sumber Azure dengan templat Bicep
- **Menyelesaikan Masalah Penyebaran**: Menyelesaikan isu biasa dan debug masalah
- **Mengoptimumkan untuk Produksi**: Keselamatan, penskalaan, pemantauan, dan pengurusan kos
- **Membina Penyelesaian Multi-Ejen**: Menyebarkan seni bina AI yang kompleks

## ğŸ“– Apakah Azure Developer CLI?

Azure Developer CLI (azd) ialah antara muka baris perintah berfokus pembangun yang mempercepatkan proses membina dan menyebarkan aplikasi ke Azure. Ia menyediakan:

- **Penyebaran berasaskan templat** - Gunakan templat siap untuk corak aplikasi biasa
- **Infrastruktur sebagai Kod** - Urus sumber Azure menggunakan Bicep atau Terraform  
- **Aliran kerja bersepadu** - Menyediakan, menyebarkan, dan memantau aplikasi secara lancar
- **Mesra Pembangun** - Dioptimumkan untuk produktiviti dan pengalaman pembangun

### **AZD + Microsoft Foundry: Sempurna untuk Penyebaran AI**

**Mengapa AZD untuk Penyelesaian AI?** AZD menangani cabaran utama yang dihadapi pembangun AI:

- **Templat Sedia AI** - Templat konfigurasi awal untuk Azure OpenAI, Cognitive Services, dan beban kerja ML
- **Penyebaran AI Selamat** - Corak keselamatan terbina untuk perkhidmatan AI, kunci API, dan titik akhir model  
- **Corak AI Produksi** - Amalan terbaik untuk penyebaran aplikasi AI yang boleh diskalakan dan kos-efektif
- **Aliran Kerja AI Dari Hujung ke Hujung** - Dari pembangunan model ke penyebaran produksi dengan pemantauan yang sesuai
- **Pengoptimuman Kos** - Peruntukan sumber pintar dan strategi penskalaan untuk beban kerja AI
- **Integrasi Microsoft Foundry** - Sambungan lancar ke katalog model dan titik akhir Microsoft Foundry

## ğŸ”§ Panduan Penyelesaian Masalah Ringkas

**Masalah biasa yang dihadapi pemula dan penyelesaian segera:**

### âŒ "azd: command not found"

```bash
# Pasang AZD terlebih dahulu
# Windows (PowerShell):
winget install microsoft.azd

# macOS:
brew tap azure/azd && brew install azd

# Linux:
curl -fsSL https://aka.ms/install-azd.sh | bash

# Sahkan pemasangan
azd version
```

### âŒ "No subscription found" or "Subscription not set"

```bash
# Senaraikan langganan yang tersedia
az account list --output table

# Tetapkan langganan lalai
az account set --subscription "<subscription-id-or-name>"

# Tetapkan untuk persekitaran AZD
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# Sahkan
az account show
```

### âŒ "InsufficientQuota" or "Quota exceeded"

```bash
# Cuba rantau Azure yang berbeza
azd env set AZURE_LOCATION "westus2"
azd up

# Atau gunakan SKU yang lebih kecil semasa pembangunan
# Sunting infra/main.parameters.json:
{
  "sku": "B1"  // Instead of "P1V2"
}
```

### âŒ "azd up" gagal separuh jalan

```bash
# Pilihan 1: Bersihkan dan cuba semula
azd down --force --purge
azd up

# Pilihan 2: Hanya baiki infrastruktur
azd provision

# Pilihan 3: Semak log terperinci
azd show
azd logs
```

### âŒ "Authentication failed" or "Token expired"

```bash
# Sahkan semula
az logout
az login

azd auth logout
azd auth login

# Sahkan pengesahan
az account show
```

### âŒ "Resource already exists" atau konflik penamaan

```bash
# AZD menghasilkan nama unik, tetapi jika berlaku konflik:
azd down --force --purge

# Kemudian cuba semula dengan persekitaran baharu
azd env new dev-v2
azd up
```

### âŒ Penyebaran templat mengambil masa terlalu lama

**Masa menunggu biasa:**
- Aplikasi web ringkas: 5-10 minit
- Aplikasi dengan pangkalan data: 10-15 minit
- Aplikasi AI: 15-25 minit (penyediaan OpenAI perlahan)

```bash
# Semak kemajuan
azd show

# Jika tersekat >30 minit, semak Portal Azure:
azd monitor
# Cari penyebaran yang gagal
```

### âŒ "Permission denied" or "Forbidden"

```bash
# Periksa peranan Azure anda
az role assignment list --assignee $(az account show --query user.name -o tsv)

# Anda memerlukan sekurang-kurangnya peranan "Contributor"
# Minta pentadbir Azure anda untuk memberi:
# - Contributor (untuk sumber)
# - User Access Administrator (untuk penugasan peranan)
```

### âŒ Tidak dapat mencari URL aplikasi yang telah dideploy

```bash
# Tunjukkan semua titik akhir perkhidmatan
azd show

# Atau buka Portal Azure
azd monitor

# Semak perkhidmatan tertentu
azd env get-values
# Cari pemboleh ubah *_URL
```
