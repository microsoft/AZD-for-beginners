# Bengkel AZD untuk Pembangun AI

Selamat datang ke bengkel praktikal untuk belajar Azure Developer CLI (AZD) dengan fokus pada penyebaran aplikasi AI. Bengkel ini membantu anda memperoleh pemahaman aplikasi tentang templat AZD dalam 3 langkah:

1. **Penemuan** - cari templat yang sesuai untuk anda.
1. **Penyebaran** - sebarkan dan sahkan bahawa ia berfungsi
1. **Penyesuaian** - ubah suai dan ulang untuk menjadikannya milik anda!

Sepanjang bengkel ini, anda juga akan diperkenalkan dengan alat dan aliran kerja utama pembangun, untuk membantu anda memperkemas perjalanan pembangunan anda dari awal hingga akhir.

<br/>

## Panduan Berasaskan Penyemak Imbas

Pelajaran bengkel dalam Markdown. Anda boleh menavigasinya terus di GitHub - atau lancarkan pratonton berasaskan penyemak imbas seperti yang ditunjukkan dalam tangkapan skrin di bawah.

![Workshop](../../../translated_images/ms/workshop.75906f133e6f8ba0.webp)

Untuk menggunakan pilihan ini - garpu repositori ke profil anda, dan lancarkan GitHub Codespaces. Setelah terminal VS Code aktif, taip arahan ini:

```bash title="" linenums="0"
mkdocs serve > /dev/null 2>&1 &
```

Dalam beberapa saat, anda akan melihat dialog pop-up. Pilih pilihan untuk `Open in browser`. Panduan berasaskan web kini akan dibuka dalam tab penyemak imbas baru. Beberapa manfaat pratonton ini:

1. **Carian terbina dalam** - cari kata kunci atau pelajaran dengan cepat.
1. **Ikon salin** - arahkan kursor ke blok kod untuk melihat pilihan ini
1. **Tukar tema** - beralih antara tema gelap dan cerah
1. **Dapatkan bantuan** - klik ikon Discord di kaki halaman untuk sertai!

## Apa Yang Akan Anda Pelajari

Fikirkan Templat AZD sebagai kotak pasir pembelajaran untuk meneroka pelbagai kemahiran dan alat bagi pembangunan end-to-end di Microsoft Foundry. Menjelang akhir bengkel ini, anda sepatutnya mempunyai deria intuitif untuk pelbagai alat dan konsep dalam konteks ini.

| Konsep  | Objektif |
|:---|:---|
| **Azure Developer CLI** | Fahami arahan alat dan aliran kerja|
| **Templat AZD**| Fahami struktur projek dan konfigurasi|
| **Ejen AI Azure**| Sediakan & sebarkan projek Microsoft Foundry  |
| **Carian AI Azure**| Aktifkan kejuruteraan konteks dengan ejen |
| **Ketelitian Pengawasan**| Terokai penjejakan, pemantauan dan penilaian |
| **Red Teaming**| Terokai ujian adversarial dan mitigasi |

## Struktur Bengkel

Bengkel ini disusun untuk membawa anda melalui perjalanan dari penemuan templat, ke penyebaran, pembongkaran, dan penyesuaian - menggunakan templat permulaan rasmi [Getting Started with AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) sebagai asas.

### [Modul 1: Pilih Templat AI](docs/instructions/1-Select-AI-Template.md) (30 minit)

- Apakah Templat AI?
- Di mana boleh saya temui Templat AI?
- Bagaimana saya boleh mula membina Ejen AI?
- **Makmal**: Permulaan Pantas dengan GitHub Codespaces

### [Modul 2: Sahkan Templat AI](docs/instructions/2-Validate-AI-Template.md) (30 minit)

- Apakah Seni Bina Templat AI?
- Apakah Aliran Kerja Pembangunan AZD?
- Bagaimana saya boleh mendapatkan bantuan dengan Pembangunan AZD?
- **Makmal**: Sebarkan & Sahkan templat Ejen AI

### [Modul 3: Bongkar Templat AI](docs/instructions/3-Deconstruct-AI-Template.md) (30 minit)

- Terokai persekitaran anda dalam `.azure/` 
- Terokai penyediaan sumber anda dalam `infra/` 
- Terokai konfigurasi AZD anda dalam `azure.yaml`s
- **Makmal**: Ubah Suai Pembolehubah Persekitaran & Sebarkan Semula

### [Modul 4: Konfigurasi Templat AI](docs/instructions/4-Configure-AI-Template.md) (30 minit)
- Terokai: Penjanaan Terimbuh Pulih
- Terokai: Penilaian Ejen & Red Teaming
- Terokai: Penjejakan & Pemantauan
- **Makmal**: Terokai Ejen AI + Ketelitian Pengawasan 

### [Modul 5: Sesuaikan Templat AI](docs/instructions/5-Customize-AI-Template.md) (30 minit)
- Tentukan: PRD dengan Keperluan Senario
- Konfigurasi: Pembolehubah Persekitaran untuk AZD
- Laksanakan: Hook Kitaran Hidup untuk tugas tambahan
- **Makmal**: Sesuaikan templat untuk senario saya

### [Modul 6: Bongkar Infrastruktur](docs/instructions/6-Teardown-Infrastructure.md) (30 minit)
- Ulas semula: Apakah Templat AZD?
- Ulas semula: Kenapa guna Azure Developer CLI?
- Langkah seterusnya: Cuba templat yang berbeza!
- **Makmal**: Nyahperuntukan infrastruktur & bersihkan
