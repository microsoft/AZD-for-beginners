# AZD за разработчици на AI

!!! info "ЦЕЛЕВА АУДИТОРИЯ НА ТАЗИ РАБОТИЛНИЦА"
   
    Тази работилница е за всеки, който иска да създава агентни AI приложения в Microsoft Foundry, използвайки водещи инструменти и работни потоци. Работилницата е подходяща както за начинаещи, така и за опитни разработчици, които имат някаква запознатост с Azure, концепции за AI и Python код. Моля, проверете предварителните изисквания преди да започнете.

    **Продължителност:** 3-4 часа  
    **Ниво:** От начинаещ до междинно  

---


## Предварителни изисквания

- Личен GitHub акаунт → [Създайте един безплатно](https://github.com/signup)
- Активен Azure акаунт → [Създайте един безплатно](https://aka.ms/free)
- Запознатост с концепции за AI, разработка с Python, инструменти на командния ред
- Опит с VS Code и инструментите и работните потоци на GitHub.

---

## Цели за обучение

Към края на тази работилница ще знаете как да:

- [X] **Да работите с Azure Developer CLI** → за създаване, внедряване и управление на вашите ресурси в Azure.
- [X] **Да използвате шаблони за AI приложения** → за бърз старт на AI Agent проекти в Microsoft Foundry.
- [X] **Конфигуриране на шаблони** → за функционалности като Red Teaming, Evaluations, Tracing & Monitoring
- [X] **Декомпозиране и изследване** → AZD шаблони и работни потоци, използвайки GitHub Copilot
- [X] **Персонализиране и повторно внедряване** → стартовия шаблон за AI Agent с вашите персонални изисквания

---

## Структура на работилницата

Работилницата е организирана в модули, всеки от които включва практическо лабораторно упражнение:

1. **Introduction** - Get an understanding of what AZD templates are, and how they work
1. **Selection** - Discover the right "starter" template for your next AI project
1. **Validation** - Deploy and explore the template to verify it works as advertised
1. **Deconstruction** - Take the template apart to understand its components & configuration
1. **Configuration** - Learn to activate and use different Microsoft Foundry features
1. **Customization** - Reconstruct the application with your data and requirements in mind
1. **Teardown** - Learn to deprovision infrastructure and recover soft-deleted resources
