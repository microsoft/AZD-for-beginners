## 1. Активиране на GitHub Copilot

### 1.1 Инсталиране на GitHub Copilot Chat

Време е да изследваме [GitHub Copilot с Agent Mode](https://code.visualstudio.com/docs/copilot/chat/chat-agent-mode). Сега можем да използваме естествен език, за да опишем задачата си на високо ниво, и да получим помощ при изпълнението ѝ. За тази лаборатория ще използваме [Copilot Free план](https://github.com/github-copilot/signup), който има месечен лимит за завършвания и чат взаимодействия.

Разширението може да бъде инсталирано от marketplace и често вече е налично в Codespaces или среди с dev контейнер. _Кликнете `Open Chat` от падащото меню на иконата Copilot - и въведете команда като `What can you do?`_ - може да бъдете подканени да влезете в системата. **GitHub Copilot Chat е готов.**

### 1.2. Инсталиране на MCP сървъри

За да бъде ефективен Agent Mode, той трябва да има достъп до правилните инструменти, които да му помагат да извлича знания или да предприема действия. Тук MCP сървърите могат да помогнат. Ще конфигурираме следните сървъри:

1. [Azure MCP Server](../../../../../workshop/docs/instructions)
1. [Microsoft Docs MCP Server](../../../../../workshop/docs/instructions)

За да ги активирате:

1. Създайте файл с име `.vscode/mcp.json`, ако не съществува
1. Копирайте следното в този файл - и стартирайте сървърите!
   ```json title=".vscode/mcp.json"
   {
      "servers": {
         "Azure MCP Server": {
            "command": "npx",
            "args": [
            "-y",
            "@azure/mcp@latest",
            "server",
            "start"
            ]
         },
         "microsoft.docs.mcp": {
            "type": "http",
            "url": "https://learn.microsoft.com/api/mcp"
         }
      }
   }
   ```

??? warning "Може да получите грешка, че `npx` не е инсталиран (кликнете, за да разширите за решение)"

      За да го оправите, отворете файла `.devcontainer/devcontainer.json` и добавете този ред в секцията features. После изградете наново контейнера. Трябва вече да имате инсталиран `npx`.

      ```title="" linenums="0"
         "features": {
            "ghcr.io/devcontainers/features/node:1": {},
            ...
         },
      ```

### 1.3. Тествайте GitHub Copilot Chat

**Първо използвайте `azd auth login`, за да се удостоверите с Azure от командния ред във VS Code. Използвайте `az login` също, само ако планирате директно да изпълнявате Azure CLI команди.**

Сега трябва да можете да направите заявка за състоянието на вашия Azure абонамент и да задавате въпроси за разположени ресурси или конфигурация. Опитайте тези команди:

1. `List my Azure resource groups`
1. `#foundry list my current deployments`

Можете също да зададете въпроси за документацията на Azure и да получите отговори, базирани на Microsoft Docs MCP сървъра. Опитайте тези команди:

1. `#microsoft_docs_search What is Azure Developer CLI?`
1. `#microsoft_docs_search Show me a Python tutorial to chat with deployed model`

Или можете да поискате фрагменти с код за изпълнение на задача. Опитайте тази команда.

1. `Give me a Python code example that uses AAD for an interactive chat client`

В режим `Ask` това ще предостави код, който можете да копирате и изпробвате. В режим `Agent`, това може да отиде една стъпка по-напред и да създаде съответните ресурси за вас – включително скриптове за настройка и документация – за да ви помогне да изпълните тази задача.

**Сега сте подготвени да започнете изследване на шаблонното хранилище**

## 2. Разглобете архитектурата

??? prompt "ПОПИТАЙТЕ: Обяснете архитектурата на приложението в docs/images/architecture.png с 1 параграф"

      Това приложение е чат приложение с изкуствен интелект, създадено върху Azure, което демонстрира съвременна архитектура базирана на агенти. Решението се върти около Azure Container App, който хоства основния код на приложението, който обработва входа от потребителя и генерира интелигентни отговори чрез AI агент. 
      
      Архитектурата използва Microsoft Foundry Project като основа за AI възможностите, свързвайки се с Azure AI Services, които предоставят подлежащите езикови модели (като gpt-4.1-mini) и функционалността на агента. Потребителските взаимодействия се насочват през React-базирания frontend към FastAPI backend, който комуникира с AI агент услугата за генериране на контекстуални отговори. 
      
      Системата включва възможности за извличане на знания чрез търсене във файлове или Azure AI Search услуга, позволявайки на агента да получава достъп и да цитира информация от качени документи. За оперативно съвършенство архитектурата включва пълно наблюдение чрез Application Insights и Log Analytics Workspace за проследяване, записване и оптимизация на производителността. 
      
      Azure Storage осигурява blob сегмент за данни на приложението и качване на файлове, докато Managed Identity осигурява сигурен достъп между Azure ресурсите без съхранение на идентификационни данни. Цялото решение е проектирано за мащабируемост и поддръжка, като контейнеризираното приложение се мащабира автоматично според търсенето, като осигурява вградена сигурност, наблюдение и CI/CD възможности чрез екосистемата от управлявани услуги на Azure.

### 3.1. Основна архитектура на приложението

Този шаблон следва модел на **full-stack уеб приложение** с:

- **Backend**: Python FastAPI с интеграция на Azure AI
- **Frontend**: TypeScript/React с Vite билд система
- **Инфраструктура**: Azure Bicep шаблони за облачни ресурси
- **Контейнеризация**: Docker за консистентно разгръщане

### 3.2 Infra като код (bicep)

Инфраструктурният слой използва **Azure Bicep** шаблони, организирани модулно:

   - **`main.bicep`**: Оркестрира всички Azure ресурси
   - **`core/` модули**: Прениспользваеми компоненти за различни услуги
      - AI услуги (Microsoft Foundry модели, AI Search)
      - Хостинг на контейнери (Azure Container Apps)
      - Мониторинг (Application Insights, Log Analytics)
      - Сигурност (Key Vault, Managed Identity)

### 3.3 Изходен код на приложението (`src/`)

**Backend API (`src/api/`)**:

- REST API базиран на FastAPI
- Интеграция на Foundry агенти
- Управление на търсещ индекс за извличане на знания
- Качване и обработка на файлове

**Frontend (`src/frontend/`)**:

- Модерен React/TypeScript SPA
- Vite за бърза разработка и оптимизирани билдове
- Чат интерфейс за взаимодействия с агенти

**База знания (`src/files/`)**:

- Примери с клиентски и продуктовите данни
- Демонстрира извличане на знания чрез файлове
- Примери във формат JSON и Markdown


### 3.4 DevOps и автоматизация

**Скриптове (`scripts/`)**:

- Крос-платформени PowerShell и Bash скриптове
- Проверка и настройка на средата
- Конфигурация след разгръщане
- Управление на квоти за модели

**Интеграция с Azure Developer CLI**:

- Конфигурация `azure.yaml` за `azd` работни процеси
- Автоматизирано предоставяне и разгръщане
- Управление на променливи на средата

### 3.5 Тестване и осигуряване на качество

**Рамка за оценка (`evals/`)**:

- Оценка на представянето на агенти
- Тестване качеството на заявки и отговори
- Автоматизирана оценъчна линия

**AI безопасност (`airedteaming/`)**:

- Тестове на "червения екип" за AI безопасност
- Сканиране за уязвимости в сигурността
- Отговорни AI практики
