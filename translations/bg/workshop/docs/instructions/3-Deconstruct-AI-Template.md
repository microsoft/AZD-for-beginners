## 1. Активиране на GitHub Copilot

### 1.1 Инсталиране на GitHub Copilot Chat

Време е да разгледаме [GitHub Copilot в Agent режим](https://code.visualstudio.com/docs/copilot/chat/chat-agent-mode). Сега можем да използваме естествен език, за да опишем задачата си на високо ниво и да получим помощ при изпълнението. За този лабораторен упражнение ще използваме [безплатния план на Copilot](https://github.com/github-copilot/signup), който има месечен лимит за завършвания и чат взаимодействия.

Разширението може да бъде инсталирано от marketplace-а, но вече трябва да е налично във вашата Codespaces среда. _Кликнете `Open Chat` от падащото меню на иконата Copilot - и въведете подсказка като `What can you do?`_ - може да бъдете подканени да влезете. **GitHub Copilot Chat е готов.**

### 1.2. Инсталиране на MCP сървъри

За да бъде режимът Agent ефективен, той се нуждае от достъп до правилните инструменти, които да му помогнат да извлича знания или да предприема действия. Тук MCP сървърите могат да помогнат. Ще конфигурираме следните сървъри:

1. [Azure MCP Server](../../../../../workshop/docs/instructions)
1. [Microsoft Docs MCP Server](../../../../../workshop/docs/instructions)

За да ги активирате:

1. Създайте файл с име `.vscode/mcp.json`, ако не съществува
1. Копирайте следното в този файл - и стартирайте сървърите!
   ```json title=".vscode/mcp.json"
   {
      "servers": {
         "Azure MCP Server": {
            "command": "npx",
            "args": [
            "-y",
            "@azure/mcp@latest",
            "server",
            "start"
            ]
         },
         "microsoft.docs.mcp": {
            "type": "http",
            "url": "https://learn.microsoft.com/api/mcp"
         }
      }
   }
   ```

??? warning "Може да получите грешка, че `npx` не е инсталиран (кликнете, за да разширите поправката)"

      За да поправите това, отворете файла `.devcontainer/devcontainer.json` и добавете този ред в секцията features. След това пресъздайте контейнера. Сега трябва да имате инсталиран `npx`.

      ```title="" linenums="0"
         "features": {
            "ghcr.io/devcontainers/features/node:1": {},
            ...
         },
      ```

### 1.3. Тестване на GitHub Copilot Chat

**Първо използвайте `az login`, за да се автентикирате в Azure от командния ред на VS Code.**

Сега трябва да можете да запитате статуса на вашия Azure абонамент и да задавате въпроси за разположените ресурси или конфигурацията. Опитайте тези подсказки:

1. `List my Azure resource groups`
1. `#foundry list my current deployments`

Можете също да питате за документацията на Azure и да получавате отговори, базирани на Microsoft Docs MCP сървъра. Опитайте тези подсказки:

1. `#microsoft_docs_search What is Azure Developer CLI?`
1. `#microsoft_docs_search Show me a Python tutorial to chat with deployed model`

Или можете да поискате фрагменти от код, за да изпълните задача. Опитайте тази подсказка.

1. `Give me a Python code example that uses AAD for an interactive chat client`

В режим `Ask` това ще предостави код, който можете да копирате и пробвате. В режим `Agent` това може да отиде една стъпка по-нататък и да създаде съответните ресурси за вас - включително скриптове за настройка и документация - за да ви помогне да изпълните задачата.

**Вече сте готови да започнете да разглеждате шаблонното репозиторио**

## 2. Деконструиране на архитектурата

??? prompt "ПОИСК: Обяснете архитектурата на приложението в docs/images/architecture.png в 1 абзац"

      Това приложение е AI-задвижано чат приложение, изградена на Azure, което демонстрира модерна архитектура, базирана на агенти. Решението се съсредоточава около Azure Container App, който хоства основния код на приложението, обработва входа от потребителя и генерира интелигентни отговори чрез AI агент.
      
      Архитектурата използва Microsoft Foundry Project като основа за AI възможности, свързвайки се с Azure AI Services, които предоставят основните езикови модели (като gpt-4.1-mini) и агентна функционалност. Потребителските взаимодействия преминават през фронтенд, базиран на React, към бекенд FastAPI, който комуникира със сервиса на AI агента за генериране на контекстуални отговори.
      
      Системата включва възможности за извличане на знания чрез търсене във файлове или услугата Azure AI Search, позволявайки на агента да достъпва и цитира информация от качени документи. За оперативно съвършенство архитектурата включва комплексен мониторинг чрез Application Insights и Log Analytics Workspace за трасироване, логване и оптимизация на производителността.
      
      Azure Storage предоставя blob сторидж за данни на приложението и качвания на файлове, докато Managed Identity осигурява защитен достъп между ресурсите на Azure без съхраняване на креденшъли. Цялото решение е проектирано за мащабируемост и поддръжка, като контейнеризираното приложение се мащабира автоматично според търсенето, предоставяйки вградено обезопасяване, наблюдение и CI/CD възможности чрез управляваните услуги на Azure.

### 3.1. Основна архитектура на приложението

Този шаблон следва модел на **пълно-стек уеб приложение** с:

- **Бекенд**: Python FastAPI с интеграция в Azure AI
- **Фронтенд**: TypeScript/React с Vite build система
- **Инфраструктура**: Azure Bicep шаблони за облачни ресурси
- **Контейнеризация**: Docker за последователно разгръщане

### 3.2 Инфраструктура като код (bicep)

Слойът за инфраструктура използва **Azure Bicep** шаблони, организирани модулно:

   - **`main.bicep`**: Оркестра всички Azure ресурси
   - **`core/` модули**: Преизползваеми компоненти за различни услуги
      - AI услуги (Microsoft Foundry Models, AI Search)
      - Хостинг на контейнери (Azure Container Apps)
      - Мониторинг (Application Insights, Log Analytics)
      - Сигурност (Key Vault, Managed Identity)

### 3.3 Изходен код на приложението (`src/`)

**Бекенд API (`src/api/`)**:

- REST API, базиран на FastAPI
- Интеграция с Foundry Agents
- Управление на search индекс за извличане на знания
- Качване и обработка на файлове

**Фронтенд (`src/frontend/`)**:

- Модерен React/TypeScript SPA
- Vite за бърза разработка и оптимизирани билдове
- Чат интерфейс за взаимодействие с агента

**База знания (`src/files/`)**:

- Примерни данни за клиенти и продукти
- Демонстрира извличане на знания от файлове
- Примери в JSON и Markdown формати


### 3.4 DevOps и автоматизация

**Скриптове (`scripts/`)**:

- Крос-платформени PowerShell и Bash скриптове
- Валидация и настройка на средата
- Конфигурация след деплой
- Управление на квоти за модели

**Интеграция с Azure Developer CLI**:

- Конфигурация `azure.yaml` за `azd` workflows
- Автоматизирано провизиране и деплой
- Управление на променливи на средата

### 3.5 Тестване и осигуряване на качеството

**Фреймуърк за оценка (`evals/`)**:

- Оценка на представянето на агентите
- Тестване на качеството на отговорите на заявки
- Автоматизиран pipeline за оценка

**AI безопасност (`airedteaming/`)**:

- Red team тестване за безопасност на AI
- Сканиране за уязвимости в сигурността
- Практики за отговорен AI
