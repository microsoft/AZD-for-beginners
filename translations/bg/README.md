## 🆕 Какво е новото в azd днес

Azure Developer CLI се разшири отвъд традиционните уеб приложения и API. Днес azd е единственият инструмент за разгръщане на **всякакви** приложения в Azure — включително приложения с изкуствен интелект и интелигентни агенти.

Ето какво означава това за вас:

- **AI агентите вече са първокласни azd натоварвания.** Можете да инициализирате, разгръщате и управлявате AI проектите с агенти, използвайки същия работен процес `azd init` → `azd up`, който вече познавате.
- **Интеграцията с Microsoft Foundry** въвежда разгръщане на модели, хостинг на агенти и конфигурация на AI услуги директно в екосистемата на azd шаблоните.
- **Ядреният работен процес не се е променил.** Независимо дали разгръщате приложение за списък със задачи, микросървис или мултиагентно AI решение, командите са същите.

Ако сте използвали azd преди, поддръжката на AI е естествено продължение — не отделен инструмент или напреднал път. Ако започвате от нулата, ще научите един работен процес, който работи за всичко.

## 🚀 Какво е Azure Developer CLI (azd)?

**Azure Developer CLI (azd)** е команден инструмент, ориентиран към разработчици, който улеснява разгръщането на приложения в Azure. Вместо ръчно да създавате и свързвате десетки Azure ресурси, можете да разположите цялото приложение с една команда.

### Магията на `azd up`

```bash
# Тази единствена команда прави всичко:
# ✅ Създава всички ресурси в Azure
# ✅ Конфигурира мрежата и сигурността
# ✅ Изгражда кода на вашето приложение
# ✅ Разгръща в Azure
# ✅ Предоставя ви работещ URL
azd up
```

**Това е всичко!** Няма кликане в Azure портал, няма сложни ARM шаблони за учене, няма ръчна конфигурация — само работещи приложения в Azure.

## ❓ Azure Developer CLI срещу Azure CLI: Каква е разликата?

Това е най-често задаваният въпрос от начинаещите. Ето простия отговор:

| Функция | **Azure CLI (`az`)** | **Azure Developer CLI (`azd`)** |
|---------|---------------------|--------------------------------|
| **Цел** | Управление на отделни Azure ресурси | Разгръщане на пълни приложения |
| **Подход** | Ориентиран към инфраструктурата | Ориентиран към приложението |
| **Пример** | `az webapp create --name myapp...` | `azd up` |
| **Ниво на знание** | Трябва да познавате Azure услугите | Просто познавайте приложението си |
| **Подходящ за** | DevOps, инфраструктура | Разработчици, прототипиране |

### Просто сравнение

- **Azure CLI** е като да имате всички инструменти за построяване на къща — чукове, триони, пирони. Можете да построите всичко, но трябва да знаете строителство.
- **Azure Developer CLI** е като да наемете изпълнител — вие описвате какво искате и той се занимава с построяването.

### Кога да използвате кой

| Сценарий | Използвайте това |
|----------|------------------|
| "Искам бързо да разгръдна уеб приложението си" | `azd up` |
| "Трябва ми само сторидж акаунт" | `az storage account create` |
| "Строя пълно AI приложение" | `azd init --template azure-search-openai-demo` |
| "Искам да дебъгна конкретен Azure ресурс" | `az resource show` |
| "Искам готово за продукция разгръщане за минути" | `azd up --environment production` |

### Работят заедно!

AZD използва Azure CLI "под капака". Можете да използвате и двата:
```bash
# Разположете приложението си с AZD
azd up

# След това фино настройте конкретни ресурси с Azure CLI
az webapp config set --name myapp --always-on true
```

## 🎯 Започнете в 3 стъпки

### Стъпка 1: Инсталирайте AZD (2 минути)

**Windows:**
```powershell
winget install microsoft.azd
```

**macOS:**
```bash
brew tap azure/azd && brew install azd
```

**Linux:**
```bash
curl -fsSL https://aka.ms/install-azd.sh | bash
```

### Стъпка 2: Влезте в Azure

```bash
azd auth login
```

### Стъпка 3: Разположете първото си приложение

```bash
# Инициализирай от шаблон
azd init --template todo-nodejs-mongo

# Разгръщане в Azure (създава всичко!)
azd up
```

**🎉 Готово!** Вашето приложение вече е в Azure.

### Почистете след себе си (Не забравяйте!)

```bash
# Remove all resources when done experimenting
azd down --force --purge
```

## 🔧 Бързо ръководство за отстраняване на проблеми

**Чести проблеми, с които се сблъскват начинаещите, и незабавни решения:**

<details>
<summary><strong>❌ "azd: команда не е намерена"</strong></summary>

```bash
# Инсталирайте първо AZD
# Windows (PowerShell):
winget install microsoft.azd

# macOS:
brew tap azure/azd && brew install azd

# Linux:
curl -fsSL https://aka.ms/install-azd.sh | bash

# Проверете инсталацията
azd version
```
</details>

<details>
<summary><strong>❌ "Не е намерен абонамент" или "Абонаментът не е зададен"</strong></summary>

```bash
# Изброи наличните абонаменти
az account list --output table

# Задай подразбиращ се абонамент
az account set --subscription "<subscription-id-or-name>"

# Настрой за AZD среда
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# Провери
az account show
```
</details>

<details>
<summary><strong>❌ "InsufficientQuota" или "Квотата е надвишена"</strong></summary>

```bash
# Опитайте различен регион на Azure
azd env set AZURE_LOCATION "westus2"
azd up

# Или използвайте по-малки SKU за разработка
# Редактирайте infra/main.parameters.json:
{
  "sku": "B1"  // Instead of "P1V2"
}
```
</details>

<details>
<summary><strong>❌ "azd up" спира по средата на процеса</strong></summary>

```bash
# Вариант 1: Изчисти и опитай отново
azd down --force --purge
azd up

# Вариант 2: Поправи само инфраструктурата
azd provision

# Вариант 3: Провери подробния статус
azd show

# Вариант 4: Провери логовете в Azure Monitor
azd monitor --logs
```
</details>

<details>
<summary><strong>❌ "Неуспешна автентикация" или "Токенът е изтекъл"</strong></summary>

```bash
# Пре-аутентикация
az logout
az login

azd auth logout
azd auth login

# Потвърдете автентикацията
az account show
```
</details>

<details>
<summary><strong>❌ "Ресурсът вече съществува" или конфликти с имената</strong></summary>

```bash
# AZD генерира уникални имена, но ако има конфликт:
azd down --force --purge

# След това опитайте отново със свежа среда
azd env new dev-v2
azd up
```
</details>

<details>
<summary><strong>❌ Внедряването на шаблона отнема твърде много време</strong></summary>

**Нормални времена за чакане:**
- Прост уебсайт: 5-10 минути
- Приложение с база данни: 10-15 минути
- AI приложения: 15-25 минути (Осигуряването на OpenAI е бавно)

```bash
# Проверете напредъка
azd show

# Ако сте заседнали >30 минути, проверете Azure портала:
azd monitor
# Потърсете неуспешни разгръщания
```
</details>

<details>
<summary><strong>❌ "Отказан достъп" или "Забранено"</strong></summary>

```bash
# Проверете вашата Azure роля
az role assignment list --assignee $(az account show --query user.name -o tsv)

# Трябва ви поне роля "Contributor"
# Помолете вашия Azure администратор да ви даде:
# - Contributor (за ресурси)
# - User Access Administrator (за разпределяне на роли)
```
</details>

<details>
<summary><strong>❌ Не може да се намери URL на внедреното приложение</strong></summary>

```bash
# Покажи всички крайни точки на услугата
azd show

# Или отвори Azure портала
azd monitor

# Провери конкретна услуга
azd env get-values
# Потърси променливи *_URL
```
</details>
