## Преглед

Тази глава обхваща конфигурирането на средите, модели за удостоверяване и най-добри практики за сигурност при разгръщания с Azure Developer CLI.

> Проверено спрямо `azd 1.27.1` през юли 2026 г.

## Учебни цели

След завършването на тази глава ще:
- Управлявате йерархията на конфигурацията на AZD
- Администрирате множество среди (dev, staging, prod)
- Прилагате сигурно удостоверяване с управлявани идентичности
- Конфигурирате настройки, специфични за средата

## 🚀 Бърз старт

```bash
# Създаване на множество среди
azd env new dev
azd env new staging
azd env new prod

# Смяна на среди
azd env select prod

# Задаване на променливи на средата
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# Преглед на конфигурацията
azd env get-values
```

---

## 🔧 Йерархия на конфигурацията

AZD прилага настройките в следния ред (по-късните заместват по-ранните):

1. **По подразбиране стойности** (вградени в шаблоните)
2. **azure.yaml** (конфигурация на проекта)
3. **Променливи на средата** (`azd env set`)
4. **Параметри от командния ред** (`--location eastus`)

---

## 🔐 Най-добри практики за сигурност

```bash
# Използвайте управлявана идентичност (препоръчително)
azd env set AZURE_USE_MANAGED_IDENTITY true

# Проверете статуса на автентикация на AZD
azd auth status

# По желание: проверете контекста на Azure CLI, ако планирате да изпълнявате az команди
az account show

# Повторно автентикирайте, ако е необходимо
azd auth login

# По желание: обновете автентикацията на Azure CLI за az команди
az login
```
