## Преглед

В тази глава се разглеждат конфигурирането на средата, модели за удостоверяване и най-добрите практики за сигурност при разгръщания с Azure Developer CLI.

## Учебни цели

След като завършите тази глава, ще:
- Овладяване на йерархията на конфигурацията на AZD
- Управление на множество среди (dev, staging, prod)
- Прилагане на сигурно удостоверяване с управлявани идентичности
- Конфигуриране на настройки, специфични за средата

## 🚀 Бърз старт

```bash
# Създаване на множество среди
azd env new dev
azd env new staging
azd env new prod

# Превключване между среди
azd env select prod

# Задаване на променливи на средата
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# Преглед на конфигурацията
azd env get-values
```

---

## 🔧 Иерархия на конфигурацията

AZD прилага настройки в следния ред (по-късните отменят по-ранните):

1. **Стойности по подразбиране** (вградени в шаблоните)
2. **azure.yaml** (конфигурация на проекта)
3. **Променливи на средата** (`azd env set`)
4. **Флагове на командния ред** (`--location eastus`)

---

## 🔐 Най-добри практики за сигурност

```bash
# Използвайте управлявана идентичност (препоръчително)
azd env set AZURE_USE_MANAGED_IDENTITY true

# Проверете състоянието на удостоверяването
azd auth whoami
az account show

# Удостоверете се отново при необходимост
azd auth login
az login
```
