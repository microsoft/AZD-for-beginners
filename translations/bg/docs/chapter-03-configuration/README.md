# Глава 3: Конфигурация и удостоверяване

**📚 Курс**: [AZD за начинаещи](../../README.md) | **⏱️ Продължителност**: 45-60 минути | **⭐ Сложност**: Средно ниво

## Преглед

Тази глава разглежда конфигурацията на средата, модели на удостоверяване и най-добри практики за сигурност при внедряване с Azure Developer CLI.

> Валидирано с `azd 1.23.12` през март 2026 г.

## Учебни цели

С изпълнението на тази глава ще:
- Овладеете йерархията на конфигурация на AZD
- Управлявате множество среди (dev, staging, prod)
- Прилагате сигурно удостоверяване с управлявани идентичности
- Конфигурирате настройки специфични за средата

## 🚀 Бърз старт

```bash
# Създайте няколко среди
azd env new dev
azd env new staging
azd env new prod

# Превключване между среди
azd env select prod

# Задайте променливи на средата
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# Преглед на конфигурацията
azd env get-values
```

---

## 🔧 Йерархия на конфигурацията

AZD прилага настройките в този ред (по-късните отменят по-ранните):

1. **По подразбиране** (вградени в шаблоните)
2. **azure.yaml** (конфигурация на проекта)
3. **Променливи на средата** (`azd env set`)
4. **Флагове от командния ред** (`--location eastus`)

---

## 🔐 Най-добри практики за сигурност

```bash
# Използвайте управлявана идентичност (препоръчително)
azd env set AZURE_USE_MANAGED_IDENTITY true

# Проверете състоянието на удостоверяване на AZD
azd auth status

# По избор: проверете контекста на Azure CLI, ако планирате да изпълнявате az команди
az account show

# Удостоверете се отново, ако е необходимо
azd auth login

# По избор: обновете удостоверяването на Azure CLI за az команди
az login
```
