## Преглед

Тази глава обхваща конфигурацията на средата, модели на удостоверяване и най-добри практики за сигурност за разгръщания с Azure Developer CLI.

След като завършите тази глава, ще:

- Внедрите сигурно удостоверяване с управлявани идентичности
- Конфигурирате настройки, специфични за средата

## 🚀 Quick Start

```bash
# Създаване на няколко среди
azd env new dev
azd env new staging
azd env new prod

# Превключване между среди
azd env select prod

# Задаване на променливи на средата
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# Преглед на конфигурацията
azd env get-values
```

---

## 🔧 Йерархия на конфигурацията

AZD прилага настройките в следния ред (по-късните отменят по-ранните):

1. **Стойности по подразбиране** (вградени в шаблоните)
2. **azure.yaml** (конфигурация на проекта)
3. **Променливи на средата** (`azd env set`)
4. **Флагове на командния ред** (`--location eastus`)

## 🔐 Най-добри практики за сигурност

```bash
# Използвайте управлявана идентичност (препоръчително)
azd env set AZURE_USE_MANAGED_IDENTITY true

# Проверете статуса на удостоверяването на AZD
azd auth status

# Незадължително: проверете контекста на Azure CLI, ако планирате да изпълнявате команди az
az account show

# Удостоверете се отново, ако е необходимо
azd auth login

# Незадължително: обновете удостоверяването на Azure CLI за командите az
az login
```
