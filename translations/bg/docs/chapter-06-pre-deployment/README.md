# Глава 6: Планиране и валидация преди разгръщане

**📚 Курс**: [AZD за начинаещи](../../README.md) | **⏱️ Продължителност**: 1 час | **⭐ Сложност**: Средно ниво

---

## Преглед

Тази глава обхваща основните стъпки за планиране и валидация преди разгръщането на вашето приложение. Научете как да избегнете скъпи грешки с правилно планиране на капацитета, избор на SKU и предварителни проверки.

> Потвърдено с `azd 1.25.6` през юни 2026 г.

## Учебни цели

Като завършите тази глава, вие ще:
- Провеждате предварителни проверки преди разгръщане
- Планирате капацитета и оценявате изискванията за ресурси
- Избирате подходящи SKU за оптимизация на разходите
- Конфигурирате Application Insights за мониторинг
- Разбирате модели за координация в екипа

## 🚀 Бърз старт

```bash
# Проверете квотите на абонамента
az vm list-usage --location eastus --output table

# Прегледайте разполагането (без създаване на ресурси)
azd provision --preview

# Проверете синтаксиса на Bicep
az bicep build --file infra/main.bicep

# Проверете конфигурацията на средата
azd env get-values
```

## ☑️ Контролен списък преди разгръщане

### Преди `azd provision`

- [ ] Квотата потвърдена за региона
- [ ] SKU-та избрани подходящо
- [ ] Прегледана оценка на разходите
- [ ] Конвенцията за именуване е последователна
- [ ] Сигурност/RBAC конфигурирани

### Преди `azd deploy`

- [ ] Зададени променливи на средата
- [ ] Тайни в Key Vault
- [ ] Проверени низове за връзка
- [ ] Конфигурирани проверки на здравето
