**📚 Курс**: [AZD за начинаещи](../../README.md) | **⏱️ Продължителност**: 1 час | **⭐ Сложност**: Средно ниво

## Преглед

Тази глава обхваща основните стъпки за планиране и валидация преди разгръщането на вашето приложение. Научете как да избегнете скъпи грешки с правилно планиране на капацитет, избор на SKU и предварителни проверки.

> Потвърдено спрямо `azd 1.27.1` през юли 2026 г.

## Учебни цели

След завършване на тази глава ще можете:
- Да изпълнявате предварителни проверки преди разгръщане
- Да планирате капацитет и да оценявате изискванията за ресурси
- Да избирате подходящи SKU за оптимизация на разходите
- Да конфигурирате Application Insights за мониторинг
- Да разбирате модели на координация в екипа

## 🚀 Бърз старт

```bash
# Проверка на квотите за абонамент
az vm list-usage --location eastus --output table

# Преглед на внедряването (без създаване на ресурси)
azd provision --preview

# Проверка на синтаксиса на Bicep
az bicep build --file infra/main.bicep

# Проверка на конфигурацията на средата
azd env get-values
```

## ☑️ Контролен списък преди разгръщане

### Преди `azd provision`

- [ ] Квотата за региона е проверена
- [ ] SKU са избрани правилно
- [ ] Прегледана е оценката на разходите
- [ ] Конвенцията за именуване е последователна
- [ ] Конфигурирана е сигурността/RBAC

### Преди `azd deploy`

- [ ] Зададени са променливи на средата
- [ ] Тайните са в Key Vault
- [ ] Проверени са низовете за връзка
- [ ] Конфигурирани са проверки на здравето
