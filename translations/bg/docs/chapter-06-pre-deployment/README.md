# Глава 6: Планиране и валидация преди внедряване

**📚 Курс**: [AZD за начинаещи](../../README.md) | **⏱️ Продължителност**: 1 час | **⭐ Сложност**: Средно

---

## Преглед

Тази глава обхваща съществени стъпки за планиране и валидация преди внедряване на вашето приложение. Научете как да избегнете скъпи грешки с правилно планиране на капацитета, избор на SKU и предварителни проверки.

## Учебни цели

Като завършите тази глава, вие ще:
- Извършвате предварителни проверки преди внедряване
- Планирате капацитета и оценявате изискванията към ресурсите
- Избирате подходящи SKU-та за оптимизация на разходите
- Конфигурирате Application Insights за наблюдение
- Разбирате модели за координация на екипа

## 🚀 Бърз старт

```bash
# Проверете квотите за абонамента
az vm list-usage --location eastus --output table

# Преглед на разгръщането (няма да бъдат създадени ресурси)
azd provision --preview

# Проверете синтаксиса на Bicep
az bicep build --file infra/main.bicep

# Проверете конфигурацията на средата
azd env get-values
```

## ☑️ Контролен списък преди внедряване

### Преди `azd provision`

- [ ] Квотата е проверена за региона
- [ ] Избрани са подходящи SKU-та
- [ ] Прегледана е оценката на разходите
- [ ] Номенклатурата за именуване е последователна
- [ ] Конфигурирана е сигурност/RBAC

### Преди `azd deploy`

- [ ] Зададени са променливи на средата
- [ ] Тайни в Key Vault
- [ ] Проверени са низовете за връзка
- [ ] Конфигурирани са проверки на здравето
