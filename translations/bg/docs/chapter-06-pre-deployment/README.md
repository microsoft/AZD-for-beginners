# Глава 6: Планиране и валидиране преди разгръщане

**📚 Курсове**: [AZD За начинаещи](../../README.md) | **⏱️ Продължителност**: 1 час | **⭐ Сложност**: Средно ниво

---

## Обзор

Тази глава обхваща основните стъпки за планиране и валидиране преди разгръщането на приложението ви. Научете как да избегнете скъпи грешки с правилно планиране на капацитета, избор на SKU и проверки преди полет.

> Валидирано с `azd 1.23.12` през март 2026 г.

## Учебни цели

Чрез завършване на тази глава, вие ще:
- Изпълнявате проверки преди разгръщане
- Планирате капацитета и преценявате изискванията за ресурси
- Избирате подходящи SKU за оптимизация на разходите
- Конфигурирате Application Insights за мониторинг
- Разбирате модели на координация в екип

## 🚀 Бърз старт

```bash
# Проверка на квотите на абонамента
az vm list-usage --location eastus --output table

# Преглед на разгръщането (без създаване на ресурси)
azd provision --preview

# Валидация на синтаксиса на Bicep
az bicep build --file infra/main.bicep

# Проверка на конфигурацията на средата
azd env get-values
```

## ☑️ Списък за проверка преди разгръщане

### Преди `azd provision`

- [ ] Квотата е проверена за региона
- [ ] SKU са избрани правилно
- [ ] Прегледана е оценка на разходите
- [ ] Правилата за наименования са спазени
- [ ] Конфигурирана е сигурност/RBAC

### Преди `azd deploy`

- [ ] Настроени са променливи на средата
- [ ] Секретите са в Key Vault
- [ ] Проверени са низове за връзка
- [ ] Конфигурирани са проверки на здравето
