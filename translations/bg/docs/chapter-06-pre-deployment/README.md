# Глава 6: Планиране и валидация преди внедряване

**📚 Курс**: [AZD за начинаещи](../../README.md) | **⏱️ Продължителност**: 1 час | **⭐ Сложност**: Средно

---

## Преглед

Тази глава обхваща съществени стъпки за планиране и валидация преди внедряване на вашето приложение. Научете как да избегнете скъпи грешки с правилно планиране на капацитета, избор на SKU и предварителни проверки.

## Учебни цели

След като завършите тази глава, ще:
- Извършите предварителни проверки преди внедряване
- Планирате капацитета и оцените необходимите ресурси
- Изберете подходящи SKU-та за оптимизиране на разходите
- Конфигурирате Application Insights за наблюдение
- Разберете модели за координация на екипа

## 🚀 Бърз старт

```bash
# Проверете квотите на абонамента
az vm list-usage --location eastus --output table

# Преглед на разгръщането (без създаване на ресурси)
azd provision --preview

# Проверете синтаксиса на Bicep
az bicep build --file infra/main.bicep

# Проверете конфигурацията на средата
azd env get-values
```

## ☑️ Контролен списък преди внедряване

### Преди `azd provision`

- [ ] Квотата е проверена за региона
- [ ] Подходящо избрани SKU-та
- [ ] Прегледана оценка на разходите
- [ ] Конвенцията за именуване е последователна
- [ ] Конфигурирана сигурност/RBAC

### Преди `azd deploy`

- [ ] Зададени променливи на средата
- [ ] Тайни в Key Vault
- [ ] Проверени низове за връзка
- [ ] Конфигурирани проверки за здраве
