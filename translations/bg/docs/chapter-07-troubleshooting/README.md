Тази глава ви помага да диагностицирате и разрешите често срещани проблеми при работа с Azure Developer CLI. От неуспехи при внедряване до специфични проблеми с AI.

> Проверено с `azd 1.23.12` през март 2026 г.

След завършване на тази глава ще можете да:

- Решавате проблеми с връзката към AI услуги
- Използвате Azure Portal и CLI за отстраняване на проблеми

## 🚨 Бързи решения

### Проблеми с удостоверяването
```bash
# Изисква се за AZD работни потоци
azd auth login

# По избор, ако също използвате директно Azure CLI команди
az login

azd auth status
```

### Неуспехи при осигуряване
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### Конфликти на ресурсите
```bash
azd down --force --purge
azd env new different-name
azd up
```

### Превишаване на квотата
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 Референция на кодове за грешки

| Грешка | Причина | Решение |
|-------|-------|----------|
| `AuthenticationError` | Не сте влезли в системата | `azd auth login` |
| `ResourceNotFound` | Липсващ ресурс | Проверете имената на ресурсите |
| `QuotaExceeded` | Лимити по абонамента | Заявете увеличаване на квотата |
| `InvalidTemplate` | Грешка в синтаксиса на Bicep | `az bicep build` |
| `Conflict` | Ресурсът съществува | Използвайте ново име или изтрийте |
| `Forbidden` | Недостатъчни разрешения | Проверете ролите в RBAC |

## 🔄 Нулиране и възстановяване

```bash
# Меко нулиране (запазване на ресурсите, презареждане на кода)
azd deploy --force

# Твърдо нулиране (изтриване на всичко, започване отначало)
azd down --force --purge
azd up
```
