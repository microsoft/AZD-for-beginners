# Глава 7: Отстраняване на проблеми и отстраняване на грешки

**📚 Курс**: [AZD за начинаещи](../../README.md) | **⏱️ Продължителност**: 1-1.5 часа | **⭐ Сложност**: Средно

---

## Преглед

Тази глава ще ви помогне да диагностицирате и разрешите често срещани проблеми при работа с Azure Developer CLI. От неуспехи при внедряване до специфични за AI проблеми.

## Учебни цели

Като завършите тази глава, ще:
- Диагностирате чести неуспехи при внедряване с AZD
- Отстраните грешки при удостоверяване и разрешения
- Разрешите проблеми със свързаността на AI услуги
- Използвате Azure Portal и CLI за отстраняване на проблеми

## 🚨 Бързи решения

### Проблеми с удостоверяването
```bash
azd auth login
az login
azd auth whoami
```

### Грешки при провизиране
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### Конфликти на ресурси
```bash
azd down --force --purge
azd env new different-name
azd up
```

### Превишен лимит
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

---

## 📋 Справочник на кодовете за грешки

| Грешка | Причина | Решение |
|-------|-------|----------|
| `AuthenticationError` | Не сте влезли | `azd auth login` |
| `ResourceNotFound` | Липсва ресурс | Проверете имената на ресурсите |
| `QuotaExceeded` | Ограничения на абонамента | Поискайте увеличение на квотата |
| `InvalidTemplate` | Грешка в синтаксиса на Bicep | `az bicep build` |
| `Conflict` | Ресурсът съществува | Използвайте друго име или изтрийте |
| `Forbidden` | Недостатъчни разрешения | Проверете RBAC ролите |

## 🔄 Нулиране и възстановяване

```bash
# Меко нулиране (запазване на ресурсите, повторно разгръщане на кода)
azd deploy --force

# Пълно нулиране (изтриване на всичко, започване отначало)
azd down --force --purge
azd up
```
