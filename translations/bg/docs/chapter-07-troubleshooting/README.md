## 🚨 Бързи решения

### Проблеми с удостоверяването
```bash
# Задължително за AZD работни потоци
azd auth login

# По избор, ако директно използвате команди на Azure CLI
az login

azd auth status
```

### Грешки при провизиране
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### Конфликти на ресурси
```bash
azd down --force --purge
azd env new different-name
azd up
```

### Превишена квота
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

---

## 📋 Справочник с кодове за грешки

| Грешка | Причина | Решение |
|-------|-------|----------|
| `AuthenticationError` | Не сте влезли в системата | `azd auth login` |
| `ResourceNotFound` | Липсващ ресурс | Проверете имената на ресурсите |
| `QuotaExceeded` | Ограничения на абонамента | Поискайте увеличение на квотата |
| `InvalidTemplate` | Грешка в синтаксиса на Bicep | `az bicep build` |
| `Conflict` | Ресурсът съществува | Използвайте ново име или изтрийте |
| `Forbidden` | Недостатъчни разрешения | Проверете RBAC ролите |

## 🔄 Нулиране и възстановяване

```bash
# Меко нулиране (запазване на ресурсите, повторно разгръщане на кода)
azd deploy --force

# Пълно нулиране (изтриване на всичко, започване отначало)
azd down --force --purge
azd up
```
