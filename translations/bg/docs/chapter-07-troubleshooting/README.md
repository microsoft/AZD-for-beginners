**📚 Курс**: [AZD за начинаещи](../../README.md) | **⏱️ Продължителност**: 1-1.5 часа | **⭐ Сложност**: Средно ниво

## Преглед

Тази глава ви помага да диагностицирате и разрешавате чести проблеми при работа с Azure Developer CLI. От неуспехи при внедряване до проблеми, специфични за AI.

> Проверено с `azd 1.27.1` през юли 2026 г.

## Учебни цели

След завършване на тази глава ще:
- Диагностицирате чести неуспехи при внедряване с AZD
- Отстранявате проблеми с удостоверяване и разрешения
- Решавате проблеми с връзката към AI услуги
- Използвате Azure Portal и CLI за отстраняване на проблеми

## 🚨 Бързи поправки

### Проблеми с удостоверяване
```bash
# Задължително за работни процеси на AZD
azd auth login

# Опционално, ако също използвате директно команди на Azure CLI
az login

azd auth status
```

### Неуспехи при предоставяне
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### Конфликти с ресурси
```bash
azd down --force --purge
azd env new different-name
azd up
```

### Превишена квота
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 Справочник за кодове на грешки

| Грешка | Причина | Решение |
|-------|---------|---------|
| `AuthenticationError` | Не сте влезли | `azd auth login` |
| `ResourceNotFound` | Липсващ ресурс | Проверете имената на ресурсите |
| `QuotaExceeded` | Ограничения на абонамента | Заявете увеличаване на квотата |
| `InvalidTemplate` | Грешка в синтаксиса на Bicep | `az bicep build` |
| `Conflict` | Ресурсът съществува | Използвайте ново име или изтрийте |
| `Forbidden` | Недостатъчни разрешения | Проверете RBAC ролите |

## 🔄 Нулиране и възстановяване

```bash
# Мек рестарт (запазване на ресурсите, повторно внедряване на кода)
azd deploy --force

# Твърд рестарт (изтриване на всичко, започване от нулата)
azd down --force --purge
azd up
```
