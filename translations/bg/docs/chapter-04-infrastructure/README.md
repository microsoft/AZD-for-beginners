# Глава 4: Инфраструктура като код и внедряване

**📚 Курс**: [AZD за начинаещи](../../README.md) | **⏱️ Продължителност**: 1-1.5 часа | **⭐ Сложност**: Средно ниво

---

## Преглед

Тази глава разглежда модели за инфраструктура като код (IaC) с Bicep шаблони, подготовка на ресурси и стратегии за внедряване с помощта на Azure Developer CLI.

> Верифицирано с `azd 1.23.12` през март 2026 г.

## Учебни цели

След като завършите тази глава, ще можете:
- Да разбирате структурата и синтаксиса на Bicep шаблони
- Да подготвяте Azure ресурси с `azd provision`
- Да внедрявате приложения с `azd deploy`
- Да прилагате стратегии за внедряване blue-green и rolling

## 🚀 Бърз старт

```bash
# Инициализиране от шаблон
azd init --template azure-functions-python-v2-http

# Преглед на това, което ще бъде създадено
azd provision --preview

# Осигуряване само на инфраструктура
azd provision

# Разгръщане само на код
azd deploy

# Или и двете заедно
azd up
```

## 📁 Структура на AZD проекта

```
my-project/
├── azure.yaml            # ← Project definition
├── infra/                # ← Infrastructure as Code
│   ├── main.bicep        # Entry point
│   ├── main.parameters.json
│   └── modules/          # Reusable components
│       ├── web.bicep
│       └── database.bicep
└── src/                  # Application code
    └── api/
```

---

## 🔧 Основни команди

| Команда | Описание |
|---------|-------------|
| `azd init` | Инициализиране на проект |
| `azd provision` | Създаване на Azure ресурси |
| `azd deploy` | Внедряване на приложен код |
| `azd up` | подготовка + внедряване |
| `azd down` | Изтриване на всички ресурси |
