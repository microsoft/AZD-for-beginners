# Глава 4: Инфраструктура като код и внедряване

**📚 Курс**: [AZD за начинаещи](../../README.md) | **⏱️ Продължителност**: 1-1.5 часа | **⭐ Сложност**: Средно

---

## Преглед

Тази глава обхваща модели за Инфраструктура като код (IaC) с Bicep шаблони, предоставяне на ресурси и стратегии за внедряване с помощта на Azure Developer CLI.

> Валидирано с `azd 1.25.6` през юни 2026 г.

## Учебни цели

След като завършите тази глава, ще:
- Разберете структурата и синтаксиса на Bicep шаблоните
- Осигурите Azure ресурси с `azd provision`
- Разположите приложения с `azd deploy`
- Прилагате стратегии за внедряване blue-green и rolling

## 🚀 Бърз старт

```bash
# Инициализиране от шаблон
azd init --template azure-functions-python-v2-http

# Преглед на това, което ще бъде създадено
azd provision --preview

# Осигуряване само на инфраструктурата
azd provision

# Само разгръщане на кода
azd deploy

# Или и двете заедно
azd up
```

## 📁 Структура на проекта AZD

```
my-project/
├── azure.yaml            # ← Project definition
├── infra/                # ← Infrastructure as Code
│   ├── main.bicep        # Entry point
│   ├── main.parameters.json
│   └── modules/          # Reusable components
│       ├── web.bicep
│       └── database.bicep
└── src/                  # Application code
    └── api/
```

---

## 🔧 Основни команди

| Command | Description |
|---------|-------------|
| `azd init` | Инициализира проект |
| `azd provision` | Създава Azure ресурси |
| `azd deploy` | Внедрява кода на приложението |
| `azd up` | осигуряване + внедряване |
| `azd down` | Изтрива всички ресурси |
