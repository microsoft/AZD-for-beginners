# AZD 面向 AI 开发者的研讨会

> 已在 2026 年 3 月使用 `azd 1.23.12` 验证。

> [!IMPORTANT]  
> **本研讨会配有一个可以在本地浏览器中预览的研讨会指南。要开始，请在 GitHub Codespaces、开发容器（dev container）或已安装 MkDocs 的本地克隆中打开仓库——然后等待直到看到一个活动的 VS Code 终端并输入：**  
> `mkdocs serve > /dev/null 2>&1 &`

欢迎参加以 AI 应用部署为重点的 Azure Developer CLI (AZD) 实操研讨会。本研讨会通过 3 个步骤帮助你获得对 AZD 模板的实用理解：

1. **Discovery** - 找到适合你的模板。  
1. **Deployment** - 部署并验证其可用性  
1. **Customization** - 修改并迭代以使其符合你的需求！

在本研讨会过程中，你还将接触到核心开发者工具和工作流，帮助你简化端到端的开发流程。

## 研讨会概览

**时长：** 3-4 小时  
**级别：** 初级到中级  
**先决条件：** 熟悉 Azure、AI 概念、VS Code 和命令行工具。

这是一个通过实践学习的实操研讨会。完成练习后，我们建议你回顾 AZD 入门课程，以继续学习安全性和生产力最佳实践。

| Time| Module  | Objective |
|:---|:---|:---|
| 15 mins | Introduction | Set the stage, understand the goals |
| 30 mins | Select AI Template | Explore options and pick starter | 
| 30 mins | Validate AI Template | Deploy default solution to Azure |
| 30 mins | Deconstruct AI Template | Explore structure and configuration |
| 30 mins | Configure AI Template | Activate and try available features |
| 30 mins | Customize AI Template | Adapt the template to your needs |
| 30 mins | Teardown Infrastructure | Cleanup and release resources |
| 15 mins | Wrap-Up & Next Steps | Learning resources, Workshop challenge |
| | |

## 你将学到什么

将 AZD 模板视为一个学习沙盒，用于探索在 Microsoft Foundry 上进行端到端开发的各种功能和工具。完成本研讨会后，你应该对相关工具和概念有直观的了解。

| Concept  | Objective |
|:---|:---|
| **Azure Developer CLI** | 了解工具命令和工作流|
| **AZD 模板**| 了解项目结构和配置|
| **Azure AI Agent**| 预配并部署 Microsoft Foundry 项目  |
| **Azure AI Search**| 通过代理实现上下文工程 |
| **Observability**| 探索追踪、监控和评估 |
| **Red Teaming**| 探索对抗性测试和缓解措施 |
