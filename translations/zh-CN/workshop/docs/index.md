# AZD 面向 AI 开发者的研讨会

> [!IMPORTANT]  
> **本研讨会配备了一个可在本地浏览器预览的研讨会指南。要开始，请在仓库上启动 GitHub Codespaces——然后等待直到您看到一个活动的 VS Code 终端并输入：**  
> `mkdocs serve > /dev/null 2>&1 &`

欢迎来到以 AI 应用部署为重点的 Azure Developer CLI (AZD) 实操研讨会。本研讨会通过 3 个步骤，帮助你获得对 AZD 模板的实用理解：

1. **发现** - 找到适合你的模板。
1. **部署** - 部署并验证其可用性
1. **自定义** - 修改并迭代，使其成为你的！

在本研讨会过程中，你还将接触到核心开发者工具和工作流，帮助你简化端到端开发流程。

## 研讨会概览

**时长：** 3-4 小时  
**难度等级：** 初级到中级  
**先决条件：** 熟悉 Azure、AI 概念、VS Code 和命令行工具。

这是一个通过实践学习的研讨会。完成练习后，我们建议回顾 AZD 入门课程，以继续学习安全性和生产力最佳实践。

| 时间| 模块  | 目标 |
|:---|:---|:---|
| 15 mins | 介绍 | 定下基调，理解目标 |
| 30 mins | 选择 AI 模板 | 探索选项并选择入门模板 | 
| 30 mins | 验证 AI 模板 | 将默认解决方案部署到 Azure |
| 30 mins | 拆解 AI 模板 | 探索结构和配置 |
| 30 mins | 配置 AI 模板 | 激活并尝试可用功能 |
| 30 mins | 自定义 AI 模板 | 使模板适应你的需求 |
| 30 mins | 拆除基础设施 | 清理并释放资源 |
| 15 mins | 总结与后续步骤 | 学习资源，研讨会挑战 |
| | |

## 你将学到什么

将 AZD 模板视为一个学习沙箱，用来探索 Microsoft Foundry 上的端到端开发各种功能和工具。完成本研讨会后，你应该对该环境中的各种工具和概念有直观的认识。

| 概念  | 目标 |
|:---|:---|
| **Azure Developer CLI** | 了解工具命令和工作流|
| **AZD 模板**| 理解项目结构和配置|
| **Azure AI Agent**| 预配并部署 Microsoft Foundry 项目  |
| **Azure AI Search**| 通过 agent 启用上下文工程 |
| **可观测性**| 探索追踪、监控和评估 |
| **红队测试**| 探索对抗性测试和缓解措施 |
