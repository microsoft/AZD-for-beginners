# 为 AZD For Beginners 做贡献

感谢您有兴趣为 AZD For Beginners 课程做出贡献！本文档为贡献者提供指南和信息。

## 如何贡献

### 报告问题

如果您发现错误、问题，或有改进建议：

1. **检查现有问题**：搜索 [existing issues](https://github.com/microsoft/azd-for-beginners/issues) 以避免重复
2. **创建新问题**：使用适当的问题模板
3. **提供详细信息**：包括重现步骤、预期行为，以及适用时的屏幕截图

### 提议更改

1. **Fork 仓库**：创建项目的个人 fork
2. **创建分支**：为您的更改创建一个功能分支
3. **进行更改**：遵循我们的编码和文档标准
4. **测试您的更改**：验证所有链接可用且内容正确呈现
5. **提交拉取请求**：提供对您更改的清晰描述

### 贡献类型

我们欢迎以下类型的贡献：

#### 内容改进
- 修正拼写、语法和格式问题
- 提高清晰度的现有文档
- 更新过时信息
- 添加缺失的解释或上下文

#### 新示例
- 添加真实世界的部署场景
- 为常见用例创建新的 AZD 模板
- 提供额外的代码示例和练习

#### 翻译支持
- 帮助维护现有翻译
- 提高翻译质量
- 为新内容添加翻译

#### 错误报告
- 报告损坏的链接或不正确的代码示例
- 识别文档不一致之处
- 标记可访问性问题

## 贡献指南

### 文档标准

- 使用清晰、适合初学者的语言
- 包含学习目标和成果
- 提供带有预期输出的分步说明
- 添加指向相关内容的导航链接
- 使用一致的格式和 Markdown 风格

### 代码示例

- 确保所有代码示例经过测试并可运行
- 包含解释关键概念的注释
- 遵循 Azure 和 AZD 的最佳实践
- 在适用处提供预期输出

### 提交信息

使用清晰、描述性的提交信息：
- `docs: fix typo in installation guide`
- `feat: add new container app example`
- `fix: correct broken link in chapter 2`

## 行为准则

请阅读并遵守我们的 [Code of Conduct](CODE_OF_CONDUCT.md)，以维护一个热情且包容的社区。

- **社区**：加入 [Azure Discord](https://discord.gg/microsoft-azure)
- **文档**：查看 [Azure Developer CLI docs](https://learn.microsoft.com/azure/developer/azure-developer-cli/)
