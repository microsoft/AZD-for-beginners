# अध्याय ३: कन्फिगरेसन र प्रमाणीकरण

**📚 कोर्ष**: [AZD प्रथम चरण](../../README.md) | **⏱️ अवधि**: ४५-६० मिनेट | **⭐ जटिलता**: मध्यम

---

## अवलोकन

यस अध्यायले Azure Developer CLI परिनियोजनहरूको वातावरण कन्फिगरेसन, प्रमाणीकरण ढाँचा, र सुरक्षा उत्तम अभ्यासहरू समेट्छ।

> जुलाई २०२६ मा `azd 1.27.1` सँग प्रमाणित।

## सिकाइ लक्ष्यहरू

यो अध्याय पूरा गरेर, तपाईं:
- AZD कन्फिगरेसन पदक्रममा निपूर्ण हुनुहुनेछ
- बहु वातावरणहरू (डेभ, स्टेजिङ, उत्पादन) व्यवस्थापन गर्नुहुनेछ
- प्रबन्धित पहिचानहरूसँग सुरक्षित प्रमाणीकरण लागू गर्नुहुनेछ
- वातावरण-विशिष्ट सेटिङहरू कन्फिगर गर्नुहुनेछ

## 🚀 छिटो शुरुवात

```bash
# धेरै वातावरणहरू सिर्जना गर्नुहोस्
azd env new dev
azd env new staging
azd env new prod

# वातावरणहरू परिवर्तन गर्नुहोस्
azd env select prod

# वातावरण भेरिएबलहरू सेट गर्नुहोस्
azd env set AZURE_LOCATION eastus
azd env set SKU_NAME P1v3

# कन्फिगरेसन हेर्नुहोस्
azd env get-values
```

---

## 🔧 कन्फिगरेसन पदक्रम

AZD ले यी आदेशहरूमा सेटिङहरू लागू गर्दछ (पछिका पहिलालाई ओभरराईड गर्छ):

१. **पूर्वनिर्धारित मानहरू** (टेम्प्लेटमा निर्मित)
२. **azure.yaml** (परियोजना कन्फिगरेसन)
३. **वातावरण चरहरू** (`azd env set`)
४. **कमाण्ड-लाइन झण्डाहरू** (`--location eastus`)

---

## 🔐 सुरक्षा उत्तम अभ्यासहरू

```bash
# प्रबन्धित परिचय प्रयोग गर्नुहोस् (सुझाव गरिएको)
azd env set AZURE_USE_MANAGED_IDENTITY true

# AZD प्रमाणीकरण स्थिति जाँच गर्नुहोस्
azd auth status

# वैकल्पिक: यदि तपाईंले az कमाण्डहरू चलाउने योजना बनाउनु भयो भने Azure CLI सन्दर्भ प्रमाणित गर्नुहोस्
az account show

# आवश्यक परे पुनः प्रमाणीकरण गर्नुहोस्
azd auth login

# वैकल्पिक: az कमाण्डहरूको लागि Azure CLI प्रमाणीकरण ताजा गर्नुहोस्
az login
```
