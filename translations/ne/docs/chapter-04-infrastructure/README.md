# अध्याय 4: पूर्वाधारलाई कोडको रूपमा र परिनियोजन

**📚 पाठ्यक्रम**: [AZD प्रारम्भकर्ताहरूका लागि](../../README.md) | **⏱️ अवधि**: 1-1.5 घण्टा | **⭐ जटिलता**: मध्यम

---

## अवलोकन

यस अध्यायले Bicep टेम्पलेटहरूसँग पूर्वाधारलाई कोडको रूपमा (IaC) का ढाँचा, Azure स्रोतहरूको प्राविजनिङ, र Azure Developer CLI प्रयोग गरेर परिनियोजन रणनीतिहरू समेट्छ।

## सिकाइ लक्ष्यहरू

यस अध्याय पूरा गरेपछि तपाइँले:
- Bicep टेम्पलेटको संरचना र वाक्य-विन्यास बुझ्नुहुनेछ
- `azd provision` द्वारा Azure स्रोतहरू प्राविजन गर्नुहोस्
- `azd deploy` द्वारा आवेदनहरू परिनियोजन गर्नुहोस्
- ब्लु-ग्रीन र रोलिङ परिनियोजन रणनीतिहरू कार्यान्वयन गर्नुहोस्

## 🚀 द्रुत सुरुवात

```bash
# टेम्पलेटबाट आरम्भ गर्नुहोस्
azd init --template azure-functions-python-v2-http

# के सिर्जना हुने हो भन्ने पूर्वावलोकन गर्नुहोस्
azd provision --preview

# केवल पूर्वाधार प्रदान गर्नुहोस्
azd provision

# केवल कोड परिनियोजन गर्नुहोस्
azd deploy

# वा दुबै एकैसाथ
azd up
```

## 📁 AZD परियोजना संरचना

```
my-project/
├── azure.yaml            # ← Project definition
├── infra/                # ← Infrastructure as Code
│   ├── main.bicep        # Entry point
│   ├── main.parameters.json
│   └── modules/          # Reusable components
│       ├── web.bicep
│       └── database.bicep
└── src/                  # Application code
    └── api/
```

## 🔧 आवश्यक कमाण्डहरू

| कमाण्ड | विवरण |
|---------|-------------|
| `azd init` | परियोजना आरम्भ गर्नुहोस् |
| `azd provision` | Azure स्रोतहरू सिर्जना गर्नुहोस् |
| `azd deploy` | आवेदनको कोड परिनियोजन गर्नुहोस् |
| `azd up` | प्राविजन + परिनियोजन |
| `azd down` | सबै स्रोतहरू मेटाउनुहोस् |
