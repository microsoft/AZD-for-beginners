# अध्याय 4: पूर्वाधारलाई कोडको रूपमा र परिनियोजन

**📚 पाठ्यक्रम**: [AZD सुरुकर्ताहरूका लागि](../../README.md) | **⏱️ अवधि**: 1-1.5 घण्टा | **⭐ जटिलता**: मध्यम

---

## अवलोकन

यस अध्यायले Bicep टेम्पलेटहरूसँग Infrastructure as Code (IaC) ढाँचा, स्रोत प्राविजनिङ्, र Azure Developer CLI प्रयोग गरी परिनियोजन रणनीतिहरू समेट्छ।

> जाँच गरिएको: `azd 1.25.6` जुन 2026 मा।

## सिकाइ उद्देश्यहरू

यस अध्याय पूरा गरेर, तपाईंले:
- Bicep टेम्पलेटको संरचना र सिन्ट्याक्स बुझ्नुहुनेछ
- Azure स्रोतहरू `azd provision` प्रयोग गरी प्राविजन गर्नुहोस्
- एप्लिकेसनहरू `azd deploy` प्रयोग गरी परिनियोजन गर्नुहोस्
- ब्लु-ग्रीन र रोलिङ परिनियोजन रणनीतिहरू लागू गर्नुहोस्

## 🚀 छिटो सुरुवात

```bash
# टेम्पलेटबाट आरम्भ गर्नुहोस्
azd init --template azure-functions-python-v2-http

# के सिर्जना गरिनेछ भनेर पूर्वावलोकन गर्नुहोस्
azd provision --preview

# केवल पूर्वाधार प्रावधान गर्नुहोस्
azd provision

# केवल कोड तैनाथ गर्नुहोस्
azd deploy

# वा दुवै एकसाथ
azd up
```

## 📁 AZD परियोजना संरचना

```
my-project/
├── azure.yaml            # ← Project definition
├── infra/                # ← Infrastructure as Code
│   ├── main.bicep        # Entry point
│   ├── main.parameters.json
│   └── modules/          # Reusable components
│       ├── web.bicep
│       └── database.bicep
└── src/                  # Application code
    └── api/
```

---

## 🔧 आवश्यक आदेशहरू

| आदेश | विवरण |
|---------|-------------|
| `azd init` | परियोजना आरम्भ गर्नुहोस् |
| `azd provision` | Azure स्रोतहरू सिर्जना गर्नुहोस् |
| `azd deploy` | एप्लिकेसन कोड परिनियोजन गर्नुहोस् |
| `azd up` | प्राविजन + परिनियोजन |
| `azd down` | सबै स्रोतहरू मेटाउनुहोस् |
