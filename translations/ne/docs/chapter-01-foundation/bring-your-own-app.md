# आफ्नो एप ल्याउनुहोस् - अवस्थित प्रोजेक्टमा azd थप्नुहोस्

**अध्याय नेभिगेसन:**
- **📚 कोर्स गृह**: [सुरुवातका लागि AZD](../../README.md)
- **📖 वर्तमान अध्याय**: अध्याय १ - आधार र छिटो सुरु
- **⬅️ अघिल्लो**: [तपाईंको पहिलो प्रोजेक्ट](first-project.md)
- **➡️ अर्को**: [डेभ कन्टेनरहरू र कोडस्पेसेस](dev-containers.md)

> `azd 1.27.1` अनुसार प्रमाणित, जुलाई २०२६ मा।

## परिचय

[तपाईंको पहिलो प्रोजेक्ट](first-project.md)मा तपाईंले टेम्प्लेटबाट सुरु गरेर एप सन्चालन गर्नुभयो। तर प्रायः तपाईंलाई पहिले नै आफ्नो मेसिनमा एउटा एप हुन्छ — Node.js API, Python Flask सेवा, .NET वेब एप — एउटा फोल्डरमा। यस पाठले देखाउँछ कसरी त्यहि अवस्थित कोडमा azd थप्ने ताकि तपाईं `azd up` प्रयोग गरेर त्यो स्थापना गर्न सक्नुहुनेछ, कुनै टेम्प्लेट आवश्यक छैन।

## सिकाइ लक्ष्यहरू

यस पाठको अन्त्यमा, तपाईं:
- azd प्रोजेक्ट सुरु गर्ने तीन तरिका बुझ्ने
- अवस्थित कोडबेसमा `azd init` चलाउने
- तपाईंको एपको लागि `azure.yaml` र `infra/` फोल्डर के गर्छ बुझ्ने
- azd ले इन्फ्रास्ट्रक्चर कहिले सिर्जना गर्ने र कहिले आफैं लेख्ने थाहा पाउने
- `azd up` द्वारा अवस्थित एपलाई Azure मा तैनाथ गर्ने

## सिकाइ नतिजाहरू

यस पाठ पुरा गरेपछि तपाईं सक्षम हुनुहुनेछ:
- पहिलेदेखि भएको प्रोजेक्टमा azd आरम्भ गर्ने
- आधारभूत `azure.yaml` फाइल पढ्ने र सम्पादन गर्ने
- `azd infra generate` बाट प्रारम्भिक इन्फ्रास्ट्रक्चर सिर्जना गर्ने
- तपाईंको एपको लागि उपयुक्त Azure होस्ट छनोट गर्ने
- आफैँको अनुप्रयोग तैनाथ गर्ने र सफा गर्ने

---

## azd प्रोजेक्ट सुरु गर्ने तीन तरिका

| सुरु गर्ने स्थान | आदेश | कहिले प्रयोग गर्ने |
|----------------|---------|-------------|
| **टेम्प्लेटबाट** | `azd init --template <name>` | सिकाइका लागि वा प्रमाणित नमुनाबाट नयाँ एप सुरु गर्दा |
| **अवस्थित कोडबाट** | `azd init` (तपाईंको प्रोजेक्ट फोल्डरमा) | तपाईंको पहिलादेखि भएको एप छ र तैनाथ गर्न चाहनुहुन्छ |
| **Git रिपोबाट** | `azd init --from-code` (क्लोन गरिएको रिपोमा) | अवस्थित रिपोजिटरीमा azd अपनाउदै |

तपाईंले पहिलो विकल्प पहिले अभ्यास गर्नुभयो। यस पाठले दोस्रो छाप्छ — सबैभन्दा सामान्य वास्तविक प्रयोगको परिदृश्य।

---

## चरण १: तपाईंको प्रोजेक्टमा `azd init` चलाउनुहोस्

टर्मिनल खोल्नुहोस् **अवस्थित प्रोजेक्ट फोल्डरभित्र** र चलाउनुहोस्:

```bash
cd my-existing-app
azd init
```

azd तपाईंलाई कसरी आरम्भ गर्ने सोध्छ। चयन गर्नुहोस्:

```
? How do you want to initialize your app?
> Use code in the current directory
  Select a template
```

**"हालको निर्देशिका मा रहेको कोड प्रयोग गर्नुहोस्।"** छनोट गर्नुहोस्। azd ले तपाईंको फोल्डर स्क्यान गर्छ, तपाईंको भाषा र फ्रेमवर्क पत्ता लगाउँछ, र होस्ट प्रस्ताव गर्छ।

### azd के पत्ता लगाउँछ

azd ले `package.json`, `requirements.txt`, `pom.xml`, `*.csproj`, वा `Dockerfile` जस्ता संकेतहरू खोज्छ र उपयुक्त Azure होस्ट सिफारिस गर्छ:

| तपाईंको एप | सम्भावित पत्ता लागेको होस्ट |
|----------|----------------------|
| Node.js / Python / .NET वेब एप | Azure App Service वा Container Apps |
| कन्टेनरयुक्त एप (`Dockerfile`) | Azure Container Apps |
| फङ्सन एप | Azure Functions |
| स्थिर साइट (React/Vue बिल्ड आउटपुट) | Azure Static Web Apps |

पत्ता लागेको सेवा पुष्टि गर्नुहोस्, र azd ले तपाईंलाई आवश्यक फाइलहरू निर्माण गर्छ।

---

## चरण २: azd ले के सिर्जना गर्‍यो बुझ्नुहोस्

init पछि, तपाईंको प्रोजेक्टमा दुई नयाँ कुरा हुनेछ:

```
my-existing-app/
├── azure.yaml          # ← NEW: tells azd what to deploy and where
├── infra/              # ← NEW: Infrastructure as Code (Bicep by default)
│   ├── main.bicep
│   └── main.parameters.json
├── src/ (your code)    # unchanged
└── ...                 # your existing files, untouched
```

### `azure.yaml` — प्रोजेक्ट परिभाषा

यो azd प्रोजेक्टको मुटु हो। एउटा न्यूनतम यस्तै देखिन्छ:

```yaml
# azure.yaml
name: my-existing-app
services:
  web:
    project: ./src           # path to your app code
    language: js             # js | python | dotnet | java
    host: appservice         # appservice | containerapp | function | staticwebapp
```

`services` ब्लक मुख्य भाग हो: प्रत्येक इन्ट्रीले तपाईंको कोडको फोल्डरलाई Azure होस्टसँग जोड़्छ। यदि तपाईंको एपसँग फ्रन्टएन्ड र API दुबै छन् भने, दुई सेवा हुनेछ।

### `infra/` — तपाईंका Azure स्रोतहरू कोडका रूपमा

`infra/` फोल्डरमा Bicep फाइलहरू राखिन्छन् जसले तपाईंको एपलाई आवश्यक Azure स्रोतहरू परिभाषित गर्छन् (App Service, डाटाबेस आदि)। तपाईंले यी आफैंले लेख्न जरुरी छैन — azd ले कार्यरत आरम्भिक बिन्दु सिर्जना गर्छ। तपाईंले पछि स्रोतहरू थप्न वा सुरक्षा कडाइ गर्न सम्पादन गर्न सक्नुहुन्छ ([अध्याय ४](../chapter-04-infrastructure/README.md) मा देखाइन्छ)।

> **सुझाव:** तैनाथ गर्नु अघि सिर्जना गरिएको इन्फ्रास्ट्रक्चर हेर्न वा अनुकूल बनाउन चाहनुहुन्छ? `azd infra generate` चलाउनुहोस् (यो `azd infra synth` पनि हो) जसले IaC लाई डिस्कमा लेख्छ र तपाईंले समीक्षा गरी संस्करण नियन्त्रण गर्न सक्नुहुन्छ।

---

## चरण ३: आवश्यक कन्फिगरेसन सेट गर्नुहोस्

यदि तपाईंको एपलाई सेटिङ्स वा गोप्य जानकारीहरू (जस्तै कनक्शन स्ट्रिङ, API कुञ्जी) आवश्यक छ भने, तिनीहरूलाई हार्डकोड नगर्नुहोस्। वातावरण मानहरू प्रयोग गर्नुहोस्:

```bash
# एक वातावरण सिर्जना गर्नुहोस्
azd env new dev

# गैर-गोप्य मान सेट गर्नुहोस्
azd env set API_VERSION 1.0.0
```

वास्तविक गोप्य जानकारीहरूको लागि, तिनीहरूलाई Key Vault मा राख्नुहोस् र तपाईंको इन्फ्रास्ट्रक्चरबाट तिनीहरूलाई संदर्भित गर्नुहोस् — हेर्नुहोस् [अध्याय ३: कन्फिगरेसन र प्रमाणीकरण](../chapter-03-configuration/authsecurity.md)।

---

## चरण ४: तैनाथ गर्नुहोस्

अब त्यहि कार्यप्रवाह प्रयोग गर्नुहोस् जुन तपाईंलाई थाहा छ:

```bash
# प्रमाणीकरण गर्नुहोस् (azd को लागि आवश्यक)
azd auth login

# सिर्जना गरिने स्रोतहरूको पूर्वावलोकन गर्नुहोस्
azd provision --preview

# पूर्वाधार प्रावधान गर्नुहोस् र तपाईंको कोड परिनियोजन गर्नुहोस्
azd up
```

सकिएपछि, azd ले तपाईंको एपको URL देखाउँछ। जाँच गर्नुहोस् जसरी कुनै azd एप जाँचिन्छ:

```bash
azd show           # अन्त्यबिन्दुहरू देखाउनुहोस्
azd monitor --logs # आवश्यक परेमा लगहरू जाँच गर्नुहोस्
```

## सामान्य पहिलो पल्टका समस्याहरू

| लक्षण | सम्भावित कारण | समाधान |
|---------|--------------|-----|
| azd ले मेरो एप पत्ता लगाएन | manifest हराएको (जस्तै `package.json`) | manifest थप्नुहोस्, वा `azd init` संचालन गर्दा होस्ट म्यानुअली छान्नुहोस् |
| `azd up` चलाउँदा निर्माण असफल भयो | एपलाई निर्माण चरण आवश्यक छ | `azure.yaml` मा सेवा अन्तर्गत `buildCommand`/`outputPath` थप्नुहोस् |
| एप सुरु भए तापनि त्रुटिहरू फर्काउँछ | कन्फिग/सिकरेट हराएको | `azd env set` बाट मानहरू सेट गर्नुहोस् वा Key Vault संलग्न गर्नुहोस् |
| गलत होस्ट चयन भयो | स्वत: पत्ता लगाउने अनुमान लगायो | `azure.yaml` मा `host:` सम्पादन गर्नुहोस् र फेरि `azd up` चलाउनुहोस् |

थपका लागि, [अध्याय ७: समस्याको समाधान](../chapter-07-troubleshooting/README.md) हेर्नुहोस्।

## सफा गर्नुहोस्

```bash
azd down --force --purge
```

## सारांश

- `azd init` → **"हालको निर्देशिका मा रहेको कोड प्रयोग गर्नुहोस्"** विकल्पले तपाईंको पहिलेबाट रहेको एपमा azd थप्छ।
- `azure.yaml` ले तपाईंको कोड फोल्डरहरूलाई Azure होस्ट्ससँग जोड्छ; `infra/` स्रोतहरू Bicep मा परिभाषित गर्छ।
- `azd infra generate` ले सिर्जना गरिएको इन्फ्रास्ट्रक्चर समीक्षा वा अनुकूल बनाउने सुविधा दिन्छ।
- एकपटक आरम्भ भएपछि, तपाईंको अवस्थित एपले टेम्प्लेट-आधारित एपसँग उस्तै `azd up` / `azd down` कार्यप्रवाह प्रयोग गर्छ।
