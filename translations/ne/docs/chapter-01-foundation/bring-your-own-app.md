# आफ्नो एप ल्याउनुहोस् - विद्यमान प्रोजेक्टमा azd थप्ने

**अध्याय नेभिगेसन:**
- **📚 कोर्स होम**: [AZD आरम्भिकहरूको लागि](../../README.md)
- **📖 वर्तमान अध्याय**: अध्याय 1 - आधार र शीघ्र सुरु
- **⬅️ अघिल्लो**: [तपाईंको पहिलो प्रोजेक्ट](first-project.md)
- **➡️ अर्को**: [डेभ कन्टेनरहरू र Codespaces](dev-containers.md)

> जुन 2026 मा `azd 1.25.6` सँग प्रमाणित गरिएको।

## परिचय

[तपाईंको पहिलो प्रोजेक्ट](first-project.md) मा तपाइँले टेम्प्लेटबाट सुरु गरेर एप डिप्लोय गर्नुभयो। तर धेरैजसो अवस्थामा तपाइँसँग पहिले नै एउटा एप हुन्छ — Node.js API, Python Flask सेवा, .NET वेब एप — जुन तपाइँको मेसिनको फोल्डरमा राखिएको हुन्छ। यो पाठले कसरी त्यस्तो विद्यमान कोडमा azd थप्ने र `azd up` द्वारा डिप्लोय गर्ने देखाउँछ, कुनै टेम्प्लेट आवश्यक पर्दैन।

## सिकाइ लक्ष्यहरू

यस पाठको अन्त्यसम्म, तपाइँले:
- azd प्रोजेक्ट सुरु गर्ने तीन तरिकाहरू बुझ्ने
- विद्यमान कोडबेस भित्र `azd init` चलाउने
- `azure.yaml` र `infra/` फोल्डरले तपाइँको एपका लागि के गर्छ भनी बुझ्ने
- कहिले azd लाई इन्फ्रास्ट्रक्चर जेनेरेट गर्न दिने र कहिले आफैँ लेख्ने भनी थाहा पाउने
- `azd up` का साथ तपाइँको विद्यमान एप Azure मा डिप्लोय गर्ने

## सिकाइ परिणामहरू

यस पाठ सम्पन्न गरेपछि, तपाइँ सक्नुहुनेछ:
- पहिले नै रहेको प्रोजेक्टमा azd इनिसियलाइज गर्ने
- एउटा आधारभूत `azure.yaml` फाइल पढ्ने र सम्पादन गर्ने
- `azd infra generate` द्वारा स्टार्टर इन्फ्रास्ट्रक्चर जेनेरेट गर्ने
- तपाइँको एपका लागि उपयुक्त Azure होस्ट चयन गर्ने
- आफ्नै एप्लिकेशन डिप्लोय र क्लीनअप गर्ने

## azd प्रोजेक्ट सुरु गर्ने तीन तरिकाहरू

| सुरु गर्ने बिन्दु | कमाण्ड | कहिले प्रयोग गर्ने |
|----------------|---------|-------------|
| **टेम्प्लेटबाट** | `azd init --template <name>` | सिकाइको लागि, वा प्रमाणित नमूनाबाट नयाँ एप सुरु गर्दा |
| **आफ्नो विद्यमान कोडबाट** | `azd init` (तपाइँको प्रोजेक्ट फोल्डरमा) | तपाइँसँग पहिले नै एप छ र यसलाई डिप्लोय गर्न चाहनुहुन्छ |
| **Git रेपोबाट** | `azd init --from-code` (क्लोन गरिएको रेपोमा) | विद्यमान रिपोजिटरीमा azd अपनाउँदा |

तपाइँले पहिलेको विकल्प अभ्यास गर्नुभएको थियो। यो पाठ दोस्रो विकल्प कभर गर्छ — सबैभन्दा सामान्य वास्तविक-विश्व परिदृश्य।

## चरण 1: तपाइँको प्रोजेक्टमा `azd init` चलाउनुहोस्

तपाइँको विद्यमान प्रोजेक्ट फोल्डर भित्र टर्मिनल खोल्नुहोस् र चलाउनुहोस्:

```bash
cd my-existing-app
azd init
```

azd ले तपाइँलाई इनिसियलाइज कसरी गर्ने सोध्छ। छान्नुहोस्:

```
? How do you want to initialize your app?
> Use code in the current directory
  Select a template
```

छान्नुहोस् **"हालको निर्देशिकामा रहेको कोड प्रयोग गर्नुहोस्."** azd त्यसपछि तपाइँको फोल्डर स्क्यान गर्छ, तपाइँको भाषा र ढाँचा पहिचान गर्छ, र एउटा होस्ट प्रस्ताव गर्छ।

### azd के पत्ता लगाउँछ

azd ले `package.json`, `requirements.txt`, `pom.xml`, `*.csproj`, वा `Dockerfile` जस्ता संकेतहरू खोज्छ र मेल खाने Azure होस्ट सुझाउँछ:

| तपाइँको एप | सम्भावित पहिचान गरिएको होस्ट |
|----------|----------------------|
| Node.js / Python / .NET वेब एप | Azure App Service वा Container Apps |
| कन्टेनराइज्ड एप (`Dockerfile`) | Azure Container Apps |
| Function एप | Azure Functions |
| स्ट्याटिक साइट (React/Vue निर्माण निस्कर्ष) | Azure Static Web Apps |

पत्ता लागेको सेवा(हरू) पुष्टि गर्नुहोस्, र azd ले तपाइँलाई आवश्यक फाइलहरू स्क्याफोल्ड गर्छ।

---

## चरण 2: azd ले के सिर्जना गर्‍यो बुझ्नुहोस्

init पछि, तपाइँको प्रोजेक्टमा दुई नयाँ चीजहरू हुनेछ:

```
my-existing-app/
├── azure.yaml          # ← NEW: tells azd what to deploy and where
├── infra/              # ← NEW: Infrastructure as Code (Bicep by default)
│   ├── main.bicep
│   └── main.parameters.json
├── src/ (your code)    # unchanged
└── ...                 # your existing files, untouched
```

### `azure.yaml` — प्रोजेक्ट परिभाषा

यो azd प्रोजेक्टको हृदय हो। एउटा न्यूनतम उदाहरण यसरी देखिन्छ:

```yaml
# azure.yaml
name: my-existing-app
services:
  web:
    project: ./src           # path to your app code
    language: js             # js | python | dotnet | java
    host: appservice         # appservice | containerapp | function | staticwebapp
```

`services` ब्लक मुख्य भाग हो: हरेक एंट्रीले तपाइँको कोडको एउटा फोल्डरलाई Azure होस्टसँग म्याप गर्छ। यदि तपाइँको एपसँग फ्रन्टएण्ड र API दुवै छन् भने, तपाइँसँग दुई सेवा हुनेछ।

### `infra/` — कोडको रूपमा तपाइँका Azure स्रोतहरू

`infra/` फोल्डरमा Bicep फाइलहरू हुन्छन् जसले तपाइँको एपलाई आवश्यक Azure स्रोतहरू परिभाषित गर्छन् (App Service, डेटाबेस, आदि)। यी हातले लेख्न आवश्यक छैन — azd ले काम गर्ने शुरुवाती बिन्दु जेनेरेट गर्छ। पछि तपाइँ तिनीहरूलाई सम्पादन गरेर स्रोतहरू थप्न वा सुरक्षा कडा बनाउन सक्नुहुन्छ (कभर गरिएको छ [अध्याय 4](../chapter-04-infrastructure/README.md) मा)।

> **सुझाव:** जेनेरेट गरिएको इन्फ्रास्ट्रक्चर डिप्लोय गर्नु अघि हेर्न वा अनुकूलन गर्न चाहनुहुन्छ? `azd infra generate` (जसलाई `azd infra synth` पनि भनिन्छ) चलाएर IaC डिस्कमा लेख्नुहोस् ताकि तपाइँले समीक्षा र भर्सन-कन्ट्रोल गर्न सक्नुहुनेछ।

---

## चरण 3: आवश्यक कन्फिगरेसन सेट गर्नुहोस्

यदि तपाइँको एपलाई सेटिङ्स वा सीक्रेटहरू चाहिन्छ (जस्तै कनेक्सन स्ट्रिङ, API कुञ्जी), तिनीहरूलाई हार्डकोड नगर्नुहोस्। वातावरण मानहरू प्रयोग गर्नुहोस्:

```bash
# एक वातावरण सिर्जना गर्नुहोस्
azd env new dev

# गोप्य नभएको मान सेट गर्नुहोस्
azd env set API_VERSION 1.0.0
```

वास्तविक गोप्यताहरूका लागि, तिनीहरूलाई Key Vault मा स्टोर गर्नुहोस् र तपाइँको इन्फ्रास्ट्रक्चरबाट सन्दर्भ गर्नुहोस् — हेर्नुहोस् [अध्याय 3: कन्फिगरेसन र प्रमाणीकरण](../chapter-03-configuration/authsecurity.md)।

## चरण 4: डिप्लोय

अब त्यही workflow प्रयोग गर्नुहोस् जुन तपाइँले पहिले नै जान्नु भएको छ:

```bash
# प्रमाणीकरण गर्नुहोस् (azd का लागि आवश्यक)
azd auth login

# सिर्जना गरिने संसाधनहरूको पूर्वावलोकन गर्नुहोस्
azd provision --preview

# पूर्वाधार तयार गर्नुहोस् र आफ्नो कोड तैनाथ गर्नुहोस्
azd up
```

पूरा हुँदा, azd ले तपाइँको एपको URL प्रिन्ट गर्छ। अरू कुनै azd एप जस्तै यसलाई जाँच गर्नुहोस्:

```bash
azd show           # एन्डपोइन्टहरू देखाउनुहोस्
azd monitor --logs # आवश्यक परेमा लगहरू जाँच गर्नुहोस्
```

## सामान्य पहिलो-पटक समस्याहरू

| लक्षण | सम्भावित कारण | समाधान |
|---------|--------------|-----|
| azd ले मेरो एप पत्ता लगाउन सकेन | म्यानिफेस्ट हराइरहेको छ (उदाहरणका लागि, `package.json`) | म्यानिफेस्ट थप्नुहोस्, वा `azd init` समयमा होस्ट म्यानुअली छान्नुहोस् |
| `azd up` क्रममा बिल्ड असफल भयो | एपलाई बिल्ड चरण आवश्यक छ | `azure.yaml` मा सेवाको अन्तर्गत `buildCommand`/`outputPath` थप्नुहोस् |
| एप सुरु हुन्छ तर त्रुटिहरू फर्काउँछ | कन्फिग/सीक्रेट हराइरहेको छ | `azd env set` प्रयोग गरेर मानहरू सेट गर्नुहोस् वा Key Vault जोड्नुहोस् |
| गलत होस्ट छानियो | अटो-डिटेक्सनले अनुमान गर्‍यो | `azure.yaml` मा `host:` सम्पादन गर्नुहोस् र `azd up` पुन: चलाउनुहोस् |

थपका लागि, हेर्नुहोस् [अध्याय 7: समस्या समाधान](../chapter-07-troubleshooting/README.md)।

## सारांश

- `azd init` → **"हालको निर्देशिकामा रहेको कोड प्रयोग गर्नुहोस्"** ले पहिले नै भएको एपमा azd थप्छ।
- `azure.yaml` ले तपाइँका कोड फोल्डरहरूलाई Azure होस्टसंग म्याप गर्छ; `infra/` ले स्रोतहरू Bicep का रूपमा परिभाषित गर्छ।
- `azd infra generate` ले जेनेरेट गरिएको इन्फ्रास्ट्रक्चर समीक्षा वा अनुकूलन गर्न दिन्छ।
- एक पटक इनिसियलाइज भएपछि, तपाइँको विद्यमान एपले टेम्प्लेट-आधारित एप जस्तै उस्तै `azd up` / `azd down` workflow प्रयोग गर्छ।
