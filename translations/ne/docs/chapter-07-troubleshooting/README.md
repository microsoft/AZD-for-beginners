**📚 कोर्स**: [AZD शुरुवातीका लागि](../../README.md) | **⏱️ अवधि**: १-१.५ घण्टा | **⭐ जटिलता**: मध्यम

---

## अवलोकन

यस अध्यायले Azure Developer CLI सँग काम गर्दा सामान्य समस्याहरू पहिचान गर्न र समाधान गर्न मद्दत गर्दछ। परिनियोजन असफलताबाट AI-विशिष्ट समस्याहरू सम्म।

> जुलाई २०२६ मा `azd 1.27.1` विरुद्ध प्रमाणित।

## सिकाइ उद्देश्यहरू

यो अध्याय पूरा गर्दा, तपाईंले:
- सामान्य AZD परिनियोजन असफलताहरू निदान गर्ने
- प्रमाणीकरण र अनुमतिका समस्याहरू डिबग गर्ने
- AI सेवा कनेक्टिविटी समस्याहरू समाधान गर्ने
- समस्या निवारणका लागि Azure पोर्टल र CLI प्रयोग गर्ने

## 🚨 छिटो समाधानहरू

### प्रमाणीकरण समस्याहरू
```bash
# AZD कार्यप्रवाहहरूको लागि आवश्यक
azd auth login

# यदि तपाईं Azure CLI कमाण्डहरू सिधै प्रयोग गर्दै हुनुहुन्छ भने वैकल्पिक
az login

azd auth status
```

### प्रावधान असफलताहरू
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### स्रोत द्वन्द्वहरू
```bash
azd down --force --purge
azd env new different-name
azd up
```

### कोटा अतिक्रमण
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 त्रुटि कोड सन्दर्भ

| त्रुटि | कारण | समाधान |
|-------|-------|----------|
| `AuthenticationError` | लगइन गरिएको छैन | `azd auth login` |
| `ResourceNotFound` | स्रोत हराएको छ | स्रोत नामहरू जाँच गर्नुहोस् |
| `QuotaExceeded` | सदस्यता सीमा | कोटा वृद्धि अनुरोध गर्नुहोस् |
| `InvalidTemplate` | बाइसप सिन्ट्याक्स त्रुटि | `az bicep build` |
| `Conflict` | स्रोत पहिले नै छ | नयाँ नाम प्रयोग गर्न वा मेटाउनुस् |
| `Forbidden` | अपर्याप्त अनुमतिहरू | RBAC भूमिका जाँच गर्नुहोस् |

## 🔄 रिसेट र पुन: प्राप्ति

```bash
# सफ्ट रिसेट (स्रोतहरू राख्नुहोस्, कोड पुनःतैनाथ गर्नुहोस्)
azd deploy --force

# हार्ड रिसेट (सबै मेटाउनुहोस्, नयाँबाट सुरु गर्नुहोस्)
azd down --force --purge
azd up
```
