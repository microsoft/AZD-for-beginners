# अध्याय 7: समस्या निवारण & डिबगिङ

**📚 पाठ्यक्रम**: [AZD शुरुआतीहरूका लागि](../../README.md) | **⏱️ अवधि**: 1-1.5 घण्टा | **⭐ जटिलता**: मध्यम

---

## अवलोकन

यस अध्यायले Azure Developer CLI सँग काम गर्दा देखा पर्ने सामान्य समस्याहरू निदान र समाधान गर्न मद्दत गर्छ। परिनियोजन विफलताबाट लिएर AI-विशिष्ट समस्यासम्म।

> सत्यापित गरिएको `azd 1.25.6` मा जून 2026।

## सिकाइ उद्देश्यहरू

यस अध्याय पूरा गरेर, तपाईंले:
- सामान्य AZD परिनियोजन असफलताहरू निदान गर्नुहोस्
- प्रमाणीकरण र अनुमति सम्बन्धी समस्याहरू डिबग गर्नुहोस्
- AI सेवाको कनेक्टिभिटी समस्याहरू समाधान गर्नुहोस्
- समस्या समाधानका लागि Azure Portal र CLI प्रयोग गर्नुहोस्

## 🚨 छिटो समाधानहरू

### प्रमाणीकरण समस्याहरू
```bash
# AZD कार्यप्रवाहहरूको लागि आवश्यक
azd auth login

# यदि तपाईं Azure CLI कमाण्डहरू प्रत्यक्ष रूपमा पनि प्रयोग गर्दै हुनुहुन्छ भने वैकल्पिक
az login

azd auth status
```

### प्रोभिजनिङ असफलताहरू
```bash
azd show
azd monitor --logs
az deployment sub list --query "[?properties.provisioningState!='Succeeded']"
```

### स्रोत द्वन्द्वहरू
```bash
azd down --force --purge
azd env new different-name
azd up
```

### कोटा नाघियो
```bash
az vm list-usage --location eastus --output table
azd env set AZURE_LOCATION westus2
azd up
```

## 📋 त्रुटि कोड सन्दर्भ

| त्रुटि | कारण | समाधान |
|-------|-------|----------|
| `AuthenticationError` | लगइन गरिएको छैन | `azd auth login` |
| `ResourceNotFound` | स्रोत हराइरहेको छ | स्रोत नामहरू जाँच गर्नुहोस् |
| `QuotaExceeded` | सब्सक्रिप्शन सीमाहरू | कोटा वृद्धि अनुरोध गर्नुहोस् |
| `InvalidTemplate` | Bicep सिन्ट्याक्स त्रुटि | `az bicep build` |
| `Conflict` | स्रोत अवस्थित छ | नयाँ नाम प्रयोग गर्नुहोस् वा मेटाउनुहोस् |
| `Forbidden` | पर्याप्त अनुमतिहरू छैनन् | RBAC भूमिका जाँच गर्नुहोस् |

## 🔄 रिसेट र रिकभरी

```bash
# सफ्ट रिसेट (संसाधनहरू कायम राख्नुहोस्, कोड पुनः तैनाथ गर्नुहोस्)
azd deploy --force

# हार्ड रिसेट (सबै कुरा मेटाउनुहोस्, नयाँबाट सुरु गर्नुहोस्)
azd down --force --purge
azd up
```
