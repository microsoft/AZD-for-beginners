# अध्याय ६: पूर्व-रुपांतर योजना र प्रमाणीकरण

**📚 कोर्स**: [AZD For Beginners](../../README.md) | **⏱️ अवधि**: १ घण्टा | **⭐ जटिलता**: मध्यम

---

## अवलोकन

यो अध्याय तपाईंको अनुप्रयोग तैनाथ गर्नु अघि आवश्यक योजना र प्रमाणीकरण चरणहरू समावेश गर्दछ। सही क्षमता योजना, SKU चयन, र पूर्व-उडान जाँचहरूसँग महँगो गल्तीहरूबाट बच्न सिक्नुहोस्।

> `azd 1.23.12` मार्च २०२६ मा प्रमाणीकरण गरियो।

## सिकाइ उद्देश्यहरू

यस अध्याय पूरा गरेर, तपाईं:
- तैनाथ गर्नु अघि पूर्व-उडान जाँचहरू चलाउनुहोस्
- क्षमता योजना बनाउनुहोस् र स्रोत आवश्यकताहरू अनुमान गर्नुहोस्
- लागत अनुकूलनका लागि उपयुक्त SKU चयन गर्नुहोस्
- अनुगमनका लागि Application Insights कन्फिगर गर्नुहोस्
- टोली समन्वय ढाँचाहरू बुझ्नुहोस्

## 🚀 छिटो सुरूवात

```bash
# सदस्यता कोटा जाँच गर्नुहोस्
az vm list-usage --location eastus --output table

# प्रक्षेपण पूर्वावलोकन (कुनै स्रोतहरू सिर्जना भएका छैनन्)
azd provision --preview

# बाइसप व्याकरण मान्य गर्नुहोस्
az bicep build --file infra/main.bicep

# वातावरण कन्फिगरेसन जाँच गर्नुहोस्
azd env get-values
```

## ☑️ पूर्व-तैनाथ चेकलिष्ट

### `azd provision` अघि

- [ ] क्षेत्रको कोटा प्रमाणित गरियो
- [ ] SKU हरु उपयुक्त रूपमा चयन गरियो
- [ ] लागत अनुमान समीक्षा गरियो
- [ ] नामकरण नियम संगत छ
- [ ] सुरक्षा/RBAC कन्फिगर गरियो

### `azd deploy` अघि

- [ ] वातावरण चर सेट गरियो
- [ ] Key Vault मा गोप्य जानकारी राखियो
- [ ] जडान स्ट्रिङहरू प्रमाणित गरियो
- [ ] स्वास्थ्य जाँचहरू कन्फिगर गरियो
