## अवलोकन

यस अध्यायले तपाईंको एप्लिकेसन डिप्लोय गर्नु अघि आवश्यक योजना र मान्यकरण कदमहरू समेट्छ। उचित क्षमता योजना, SKU चयन, र प्रीफ्लाइट जाँचहरूसँग महँगो त्रुटिहरूबाट जोगिन सिक्नुहोस्।

## सिकाइ लक्ष्यहरू

यस अध्याय पूरा गरेपछि, तपाईं:
- डिप्लोयमेन्ट अघि प्रीफ्लाइट जाँचहरू चलाउन सक्नुहुनेछ
- क्षमता योजना बनाउने र स्रोत आवश्यकताहरू अनुमान गर्ने
- लागत अनुकूलनका लागि उपयुक्त SKU हरू चयन गर्ने
- निगरानीको लागि Application Insights कन्फिगर गर्ने
- टोली समन्वय ढाँचाहरू बुझ्ने

## 🚀 छिटो सुरु

```bash
# सब्सक्रिप्सन कोटा जाँच गर्नुहोस्
az vm list-usage --location eastus --output table

# परिनियोजनको पूर्वावलोकन (कुनै संसाधनहरू सिर्जना हुँदैनन्)
azd provision --preview

# Bicep वाक्य-विन्यास मान्य गर्नुहोस्
az bicep build --file infra/main.bicep

# पर्यावरण कन्फिगरेसन जाँच गर्नुहोस्
azd env get-values
```

## ☑️ पूर्व-डिप्लोयमेन्ट जाँच-सूची

### पहिले `azd provision`

- [ ] क्षेत्रको कोटा जाँचिएको
- [ ] SKU हरू उपयुक्त रूपमा चयन गरिएको
- [ ] लागत अनुमान समीक्षा गरिएको
- [ ] नामकरण परम्परा सुसंगत छ
- [ ] सुरक्षा/RBAC कन्फिगर गरिएको

### पहिले `azd deploy`

- [ ] Environment भेरिएबलहरू सेट गरिएको
- [ ] Key Vault मा Secrets राखिएका छन्
- [ ] कनेक्शन स्ट्रिङहरू जाँचिएका
- [ ] हेल्थ चेकहरू कन्फिगर गरिएको
