## 1. परिचय

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) वा `azd` एक खुला-स्रोत कमाण्डलाइन उपकरण हो जसले Azure मा एप्लिकेशन निर्माण र तैनाती गर्दा विकासकर्ता कार्यप्रवाहलाई सरल बनाउँछ। 

[AZD Templates](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) मानकीकृत रिपोजिटरीहरू हुन् जसमा नमूना एप्लिकेशन कोड, इन्फ्रास्ट्रक्चर-एज़-कोड सम्पत्ति, र `azd` कन्फिगरेसन फाइलहरू समावेश हुन्छन् ताकि समेटिएको समाधान आर्किटेक्चर प्राप्त होस्। पूर्वाधार प्रावधान गर्नु `azd provision` कमाण्ड जस्तै सरल हुन्छ - जबकि `azd up` प्रयोग गर्दा तपाईंले पूर्वाधार प्रावधानीकरण **र** एकै पटक आफ्नो एप्लिकेशन तैनाथ गर्न सक्नुहुन्छ!

नतिजा स्वरूप, तपाईंको एप विकास प्रक्रियामा छिटो शुरु गर्न उपयुक्त _AZD Starter template_ खोज्नु र रिपोजिटरीलाई तपाईंको आवश्यकताअनुसार अनुकूलन गर्नु पर्याप्त हुन्छ।

सुरू गर्नु भन्दा पहिले, आउनुहोस् सुनिश्चित गरौं कि तपाईंसँग Azure Developer CLI इन्स्टल गरिएको छ।

1. VS Code को टर्मिनल खोल्नुहोस् र यो कमाण्ड टाइप गर्नुहोस्:

      ```bash title="" linenums="0"
      azd version
      ```

1. तपाईंले यस प्रकार केही देख्नुपर्नेछ!

      ```bash title="" linenums="0"
      azd version 1.25.6 (commit <current-build>)
      ```

**अब तपाईं azd प्रयोग गरेर टेम्पलेट चयन र तैनाथ गर्न तयार हुनुहुन्छ**

## 2. टेम्पलेट चयन

Microsoft Foundry प्लेटफर्मसँग लोकप्रिय समाधान परिदृश्यहरू कभर गर्ने [सिफारिस गरिएका AZD टेम्पलेटहरूको सेट](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) आउँछ जस्तै _मल्टि-एजेण्ट वर्कफ्लो अटोमेसन_ र _मल्टि-मोडल सामग्री प्रशोधन_। तपाईं यी टेम्पलेटहरू Microsoft Foundry पोर्टलमा गएर पनि पत्ता लगाउन सक्नुहुन्छ।

1. [https://ai.azure.com/templates](https://ai.azure.com/templates) मा जानुहोस्
1. अनुरोध गर्दा Microsoft Foundry पोर्टलमा लगइन गर्नुहोस् - तपाईंले यस प्रकार देख्नुहुनेछ।

![छान्नुहोस्](../../../../../translated_images/ne/01-pick-template.60d2d5fff5ebc374.webp)


**Basic** विकल्पहरू तपाईंका स्टार्टर टेम्पलेटहरू हुन्:

1. [ ] [Get Started with AI Chat](https://github.com/Azure-Samples/get-started-with-ai-chat) जसले Azure Container Apps मा तपाईंको डाटासँग सँगै एक आधारभूत च्याट एप्लिकेशन तैनाथ गर्छ। यसलाई प्रयोग गरेर आधारभूत AI च्याटबट परिदृश्य अन्वेषण गर्नुहोस्।
1. [X] [Get Started with AI Agents](https://github.com/Azure-Samples/get-started-with-ai-agents) जसले Foundry Agents सहित एक मानक AI एजेण्ट पनि तैनाथ गर्छ। उपकरण र मोडेलहरू समावेश गर्ने एजेन्टिक AI समाधानहरूसँग परिचित हुन यसलाई प्रयोग गर्नुहोस्।

दोस्रो लिङ्क नयाँ ब्राउजर ट्याबमा खोल्नुहोस् (वा सम्बन्धित कार्डको लागि `Open in GitHub` क्लिक गर्नुहोस्)। तपाईंले यस AZD टेम्पलेटको रिपोजिटरी देख्नुहुनेछ। README अन्वेषण गर्न केही समय निकाल्नुहोस्। एप्लिकेशन आर्किटेक्चर यसरी देखिन्छ:

![आर्किटेक्चर](../../../../../translated_images/ne/architecture.8cec470ec15c65c7.webp)

## 3. टेम्पलेट सक्रियण

यस टेम्पलेटलाई तैनाथ गरेर मान्य गरौं। हामी [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) सेक्सनमा दिइएका दिशानिर्देशनहरू अनुसरण गर्नेछौं।

1. टेम्पलेट रिपोजिटरीको लागि कार्य गर्ने वातावरण रोज्नुहोस्:

      - **GitHub Codespaces**: यस लिंकमा क्लिक गर्नुहोस् [this link](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) र `Create codespace` पुष्टि गर्नुहोस्
      - **Local clone or dev container**: `Azure-Samples/get-started-with-ai-agents` क्लोन गर्नुहोस् र VS Code मा खोल्नुहोस्

1. VS Code टर्मिनल तयार नहुँदासम्म पर्खनुहोस्, त्यसपछि निम्न कमाण्ड टाइप गर्नुहोस्:

   ```bash title="" linenums="0"
   azd up
   ```

योले ट्रिगर गर्ने वर्कफ्लो चरणहरू पूरा गर्नुहोस्:

1. तपाईंलाई Azure मा लगइन गर्न अनुरोध गरिनेछ - प्रमाणिकरणका लागि निर्देशहरू पालना गर्नुहोस्
1. तपाईंको लागि एक अद्वितीय वातावरण नाम प्रविष्ट गर्नुहोस् - जस्तै, मैले `nitya-mshack-azd` प्रयोग गरेँ
1. यसले `.azure/` फोल्डर सिर्जना गर्नेछ - तपाईंले वातावरण नाम सहित एउटा उपफोल्डर देख्नुहुनेछ
1. तपाईंलाई सदस्यता नाम चयन गर्न अनुरोध गरिनेछ - डिफ़ॉल्ट छनौट गर्नुहोस्
1. तपाईंलाई स्थान (location) का लागि अनुरोध गरिनेछ - `East US 2` प्रयोग गर्नुहोस्

अब, प्रावधान पूरा हुन पर्खनुहोस्। **यसलाई 10-15 मिनेट लाग्न सक्छ**

1. जब सकिन्छ, तपाईंको कन्सोलले तल जस्तो SUCCESS सन्देश देखाउनेछ:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
1. तपाईंको Azure पोर्टल अब उक्त वातावरण नामसहितको प्रावधान गरिएको रिसोर्स समूह देखाउनेछ:

      ![पूर्वाधार](../../../../../translated_images/ne/02-provisioned-infra.46c706b14f56e0bf.webp)

1. **अब तपाईं तैनाथ गरिएको पूर्वाधार र एप्लिकेशनलाई मान्य गर्न तयार हुनुहुन्छ**।

## 5. एजेण्ट मान्यकरण

Azure Container App ले एक एन्डपोइन्ट तैनाथ गर्छ जुन यस टेम्पलेटका लागि Microsoft Foundry प्रोजेक्टमा प्रावधान गरिएको AI Agent सँग जडान हुन्छ। यसको अर्थ के हो भनेर हेर्नुहोस्।

1. Azure Portal मा फर्केर आफ्नो रिसोर्स समूहको _Overview_ पृष्ठ खोल्नुहोस्

1. सूचीमा `Microsoft Foundry` रिसोर्समा क्लिक गर्नुहोस्

1. तपाईंले यस देख्नुहुनेछ। `Go to Microsoft Foundry Portal` बटनमा क्लिक गर्नुहोस्। 
   ![Foundry](../../../../../translated_images/ne/04-view-foundry-project.fb94ca41803f28f3.webp)

1. तपाईंले आफ्नो AI एप्लिकेशनको लागि Foundry Project पृष्ठ देख्नुहुनेछ
   ![प्रोजेक्ट](../../../../../translated_images/ne/05-visit-foundry-portal.d734e98135892d7e.webp)

1. `Agents` मा क्लिक गर्नुहोस् - तपाईंले प्रोजेक्टमा प्रावधान गरिएको डिफ़ॉल्ट एजेण्ट देख्नुहुनेछ
   ![एजेण्टहरू](../../../../../translated_images/ne/06-visit-agents.bccb263f77b00a09.webp)

1. यसलाई चयन गर्नुहोस् - र तपाईंले एजेण्ट विवरणहरू देख्नुहुनेछ। तलका कुराहरू ध्यान दिनुहोस्:

      - एजेण्टले डिफ़ॉल्ट रूपमा File Search प्रयोग गर्छ (सधैं)
      - एजेण्टको `Knowledge` ले जनाउँछ कि त्यसमा 32 फाइलहरू अपलोड गरिएको छ (फाइल सर्चका लागि)
      ![एजेण्टहरू](../../../../../translated_images/ne/07-view-agent-details.0e049f37f61eae62.webp)

1. बायाँ मेनूमा `Data+indexes` विकल्प खोज्नुहोस् र विवरणका लागि क्लिक गर्नुहोस्। 

      - तपाईंले ज्ञानका लागि अपलोड गरिएका 32 डेटा फाइलहरू देख्नुहुनेछ।
      - यीले `src/files` अन्तर्गतका 12 ग्राहक फाइलहरू र 20 उत्पादन फाइलहरूलाई जनाउँछन् 
      ![डाटा](../../../../../translated_images/ne/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**तपाईंले एजेण्ट सञ्चालनलाई मान्य गर्नुभयो!** 

1. एजेण्टका उत्तरहरू ती फाइलहरूमा रहेको ज्ञानमा आधारित छन्। 
1. अब तपाईं सोधपुछ गर्न सक्नुहुन्छ र आधारभूत उत्तरहरू प्राप्त गर्न सक्नुहुन्छ।
1. उदाहरण: `customer_info_10.json` ले "Amanda Perez" द्वारा गरिएका 3 खरिदहरू वर्णन गर्छ

Container App एन्डपोइन्ट भएको ब्राउजर ट्याबमा फर्केर सोध्नुहोस्: `What products does Amanda Perez own?`. तपाईंले यस प्रकार देख्नुहुनेछ:

## 6. एजेण्ट प्लेग्राउन्ड

1. `Try in Playground` विकल्पमा क्लिक गर्नुहोस् - तपाईंलाई यो प्रकारको Playground UI देखिनेछ
1. सोध्नुहोस् सोही प्रश्न: `What products does Amanda Perez own?`

तपाईंले उस्तै (वा समान) उत्तर पाउनुहुनेछ - तर तपाईंलाई थप जानकारी पनि मिल्छ जुनले तपाईंलाई एजेण्टिक एपको गुणस्तर, लागत, र प्रदर्शन बुझ्न मद्दत गर्छ। उदाहरणका लागि:

1. यी फाइल लेबलहरूमाथि हभर गर्नुहोस् - के डेटा तपाईंको प्रश्न र प्रदर्शन गरिएको जवाफसँग मेल खान्छ?

तल जवाफको मुनि एउटा _stats_ पंक्ति पनि देख्नुहुन्छ। 

1. कुनै मेट्रिक जस्तै Safety मा हभर गर्नुहोस्। तपाईं यस प्रकार देख्नुहुनेछ
1. मूल्याङ्कन गरिएको रेटिंगले जवाफको सुरक्षा स्तरमा तपाईंको अन्तर्ज्ञानसँग मेल खान्छ?

## 7. पूर्वनिर्मित अवलोकनक्षमता

Observability भनेको तपाईंको एप्लिकेशनले डाटा उत्पन्न गर्ने गरी उपकरणीकरण गर्नु हो जसले यसको सञ्चालन बुझ्न, डिबग गर्न र अनुकूलन गर्न सहयोग गर्छ। यसको अनुभव पाउनका लागि:

1. `View Run Info` बटनमा क्लिक गर्नुहोस् - तपाईंले यो दृश्य देख्नुहुनेछ। यो [Agent tracing](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) को एक उदाहरण हो। _तपाईंले यो दृश्य थ्रेड लगहरूमा क्लिक गरेर पनि प्राप्त गर्न सक्नुहुन्छ_।

   - रन चरणहरू र एजेण्टले प्रयोग गरेको उपकरणहरूबारे बुझ्नुहोस्
   - जवाफका लागि कुल टोकन गणना (र आउटपुट टोकन प्रयोग) बुझ्नुहोस्
   - लेटेन्सी र कार्यान्वयनमा समय कहाँ खर्च भयो थाहा पाउनुहोस्

      ![एजेण्ट](../../../../../translated_images/ne/10-view-run-info.b20ebd75fef6a1cc.webp)

1. रनको लागि थप सन्दर्भ दिन सक्ने एट्रिब्युटहरू हेर्न `Metadata` ट्याबमा क्लिक गर्नुहोस्।   

      ![एजेण्ट](../../../../../translated_images/ne/11-view-run-info-metadata.7966986122c7c2df.webp)


1. `Evaluations` ट्याबमा क्लिक गरी एजेण्टको जवाफमा गरिएका स्वचालित मूल्याङ्कनहरू हेर्नुहोस्। तिनीहरूले सुरक्षा मूल्याङ्कनहरू (जस्तै, Self-harm) र एजेण्ट-विशेष मूल्याङ्कनहरू (जस्तै, उद्देश्य समाधान, कार्य पालना) समावेश गर्छन्।

      ![एजेण्ट](../../../../../translated_images/ne/12-view-run-info-evaluations.ef25e4577d70efeb.webp)

1. अन्तमा, साइडबार मेनूमा `Monitoring` ट्याबमा क्लिक गर्नुहोस्।

## 8. वातावरण चरहरू

अहिलेसम्म, हामीले ब्राउजरमा तैनाथिको माध्यमबाट हेर्यौं - र हाम्रो पूर्वाधार प्रावधान गरिएको र एप्लिकेशन सञ्चालनमा रहेको मान्य गर्यौं। तर कोड-प्रथम तरिकाले काम गर्नका लागि, हामीले स्थानीय विकास वातावरणलाई ती सम्बन्धित भेरिएबलहरूसँग कन्फिगर गर्नु आवश्यक छ जुन ती रिसोर्सहरूसँग काम गर्न चाहिन्छ। `azd` प्रयोग गर्दा यो सजिलो हुन्छ।

1. Azure Developer CLI ले एप्लिकेशन तैनातीहरूको लागि कन्फिगरेसन सेटिङहरू भण्डारण र व्यवस्थापन गर्न [environment variables](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) प्रयोग गर्छ।

1. वातावरण चरहरू `.azure/<env-name>/.env` मा भण्डार गरिन्छ - यसले तिनीहरूलाई तैनातीको समयमा प्रयोग गरिएको `env-name` वातावरणमा सीमित गर्छ र समान रिपोमा विभिन्न तैनाती लक्ष्यहरू बीच वातावरण अलग राख्न मद्दत गर्छ।

1. जबसुकै `azd` विशिष्ट कमाण्ड (जस्तै, `azd up`) चलाउँछ, वातावरण चरहरू स्वतः लोड हुन्छन्। ध्यान दिनुहोस् कि `azd` ले स्वतः OS स्तरका वातावरण चरहरू (जस्तै, शेलमा सेट गरिएका) पढ्दैन - यसको सट्टा स्क्रिप्टहरू भित्र जानकारी सर्न `azd set env` र `azd get env` प्रयोग गर्नुहोस्।


केही कमाण्डहरू चलाएर प्रयास गरौं:

1. यस वातावरणमा `azd` का लागि सेट गरिएका सबै वातावरण चरहरू प्राप्त गर्नुहोस्:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      तपाईंले यस प्रकार देख्नुहुनेछ:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

1. विशेष मान प्राप्त गर्नुहोस् - जस्तै, मैले हेर्न चाहन्छु कि `AZURE_AI_AGENT_MODEL_NAME` मान सेट गरिएको छ कि छैन

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      तपाईंले यस प्रकार देख्नुहुनेछ - यो डिफ़ॉल्टले सेट गरिएको थिएन!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

1. `azd` का लागि नयाँ वातावरण चर सेट गर्नुहोस्। यहाँ, हामी एजेण्ट मोडेल नाम अपडेट गर्दैछौं। _नोट: गरिएका कुनै पनि परिवर्तनहरू तुरुन्तै `.azure/<env-name>/.env` फाइलमा प्रतिबिम्बित हुनेछन्।_

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      अब, हामीले पाउनु पर्छ कि मान सेट गरिएको छ:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

1. केही रिसोर्सहरू स्थायी हुन्छन् (जस्तै, मोडेल तैनाथिहरू) र तिनीहरूलाई बलपूर्वक पुन:तैनाथ गर्न केवल `azd up` भन्दा बढी आवश्यक पर्न सक्छ। आउनुहोस् सुरुको तैनाती हटाएर परिवर्तित env vars सँग पुन:तैनाथ गर्ने प्रयास गरौं।

1. **Refresh** यदि तपाईंले पहिले azd टेम्पलेट प्रयोग गरेर पूर्वाधार तैनाथ गर्नुभएको थियो - तपाईं आफ्नो स्थानीय वातावरण चरहरूको राज्यलाई Azure तैनातीको वर्तमान अवस्थामा आधारित गरी _refresh_ गर्न सक्नुहुन्छ यस कमाण्ड प्रयोग गरेर:

      ```bash title="" linenums="0"
      azd env refresh
      ```

      यो दुई वा बढी स्थानीय विकास वातावरणहरू (जस्तै, धेरै विकासकर्ताहरू भएको टिम) बीच वातावरण भेरिएबलहरू _समक्रमण_ गर्ने एक शक्तिशाली तरिका हो - जसले तैनात गरिएको पूर्वाधारलाई वातावरण भेरिएबल स्थिति को वास्तविक प्रमाणको रूपमा काम गर्न अनुमति दिन्छ। टिमका सदस्यहरूले केवल चरहरूलाई _रिफ्रेश_ गरी फेरि समक्रमणमा आउँछन्।
