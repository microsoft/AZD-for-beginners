## १. परिचय

[Azure Developer CLI](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/) वा `azd` एक खुला स्रोत कमाण्डलाइन उपकरण हो जुन विकासकर्ताको वर्कफ्लो सरल बनाउँछ जब Azure मा अनुप्रयोगहरू निर्माण र तैनाथ गरिन्छ।

[AZD टेम्प्लेटहरू](https://learn.microsoft.com/azure/developer/azure-developer-cli/azd-templates) मानकीकृत भण्डारहरू हुन् जुन नमुना अनुप्रयोग कोड, _पूर्वाधार-अपनी-कोड_ सम्पत्ति, र `azd` कन्फिगरेसन फाइलहरू समावेश गर्छन् एक समेकित समाधान वास्तुकलाको लागि। पूर्वाधार प्रावधान गर्नको लागि `azd provision` आदेश जति सजिलो हुन्छ - र `azd up` प्रयोग गर्दा तपाईं एकै पटक पूर्वाधार प्रावधान **र** अनुप्रयोग तैनाथ गर्न सक्नुहुन्छ!

फलस्वरूप, तपाईंको अनुप्रयोग विकास प्रक्रिया सुरु गर्नको लागि उपयुक्त _AZD Starter टेम्प्लेट_ खोज्नु त्यति सजिलो हुन सक्छ जुन तपाईंको अनुप्रयोग र पूर्वाधार आवश्यकतासँग सबैभन्दा मिल्दोजुल्दो हो - र त्यसपछि भण्डारलाई तपाईंको परिदृश्य आवश्यकताहरूअनुसार अनुकूलित गर्नुहोस्।

सुरु गर्नु अघि, हामी पक्का गरौं कि तपाईंले Azure Developer CLI स्थापना गर्नुभएको छ।

१. VS Code टर्मिनल खोल्नुहोस् र यो आदेश टाइप गर्नुहोस्:

      ```bash title="" linenums="0"
      azd version
      ```

१. तपाईंले यसै जस्तो केही देख्नुहुनेछ!

      ```bash title="" linenums="0"
      azd version 1.27.1 (commit <current-build>)
      ```

**अब तपाईंले azd सँग एक टेम्प्लेट चयन गरी तैनाथ गर्न तयार हुनुहुन्छ**

## २. टेम्प्लेट चयन

Microsoft Foundry प्लेटफर्मसँग [सिफारिस गरिएका AZD टेम्प्लेटहरूको सेट](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/ai-template-get-started) आउँछ जसले लोकप्रिय समाधान परिदृश्यहरूको कभर गर्छ जस्तै _मल्टि-एजेन्ट वर्कफ्लो स्वचालन_ र _मल्टि-मोडल सामग्री प्रशोधन_। तपाईं यी टेम्प्लेटहरू Microsoft Foundry पोर्टलमा गएर पनि पाउन सक्नुहुन्छ।

१. [https://ai.azure.com/templates](https://ai.azure.com/templates) मा जानुहोस्
१. जब अनुरोध हुन्छ, Microsoft Foundry पोर्टलमा लगइन गर्नुहोस् - तपाईं यसै जस्तो केही देख्नुहुनेछ।

![Pick](../../../../../translated_images/ne/01-pick-template.60d2d5fff5ebc374.webp)


**Basic** विकल्पहरू तपाईंका स्टार्टर टेम्प्लेटहरू हुन्:

१. [ ] [AI Chat सँग सुरूवात गर्नुहोस्](https://github.com/Azure-Samples/get-started-with-ai-chat) जुन एक आधारभूत च्याट अनुप्रयोग Azure Container Apps मा _तपाईंको डाटा सहित_ तैनाथ गर्दछ। यसलाई प्रयोग गरेर एक आधारभूत AI च्याटबोट परिदृश्य अन्वेषण गर्नुहोस्।
१. [X] [AI Agents सँग सुरूवात गर्नुहोस्](https://github.com/Azure-Samples/get-started-with-ai-agents) जसले पनि Foundry Agents सहित एक मानक AI एजेन्ट तैनाथ गर्दछ। यसलाई एजेन्टिक AI समाधानहरू जस्तै उपकरणहरू र मोडेलहरू समावेश गरेर परिचित हुन प्रयोग गर्नुहोस्।

दोस्रो लिंक नयाँ ब्राउजर ट्याबमा खोल्नुहोस् (वा सम्बन्धित कार्डको लागि `Open in GitHub` क्लिक गर्नुहोस्)। तपाईंले यस AZD टेम्प्लेटको भण्डार देख्नुहुनेछ। README खोज्न केही समय लिनुहोस्। अनुप्रयोग वास्तुकला यसप्रकार छ:

## ३. टेम्प्लेट सक्रियकरण

अब हामी यस टेम्प्लेटलाई तैनाथ गर्ने प्रयास गर्नेछौं र यसको वैधता सुनिश्चित गर्नेछौं। हामी [Getting Started](https://github.com/Azure-Samples/get-started-with-ai-agents?tab=readme-ov-file#getting-started) खण्डका दिशानिर्देशनहरू फलो गर्नेछौं।

१. टेम्प्लेट भण्डारको लागि कार्य गर्ने वातावरण चयन गर्नुहोस्:

      - **GitHub Codespaces**: [यो लिंक](https://github.com/codespaces/new/Azure-Samples/get-started-with-ai-agents) मा क्लिक गर्नुहोस् र `Create codespace` कन्फर्म गर्नुहोस्
      - **स्थानीय क्लोन वा डिभ कन्टेनर**: `Azure-Samples/get-started-with-ai-agents` क्लोन गर्नुहोस् र VS Code मा खोल्नुहोस्

१. VS Code टर्मिनल तयार हुने सम्म पर्खनुहोस्, त्यसपछि निम्न आदेश टाइप गर्नुहोस्:

   ```bash title="" linenums="0"
   azd up
   ```

यसले ट्रिगर गर्ने वर्कफ्लो चरणहरू पूरा गर्नुहोस्:

१. Azure मा लगइन गर्न अनुरोध गरिनेछ - प्रमाणित गर्न निर्देशनहरू पालन गर्नुहोस्
१. तपाईंको लागि एक अद्वितीय वातावरण नाम प्रविष्ट गर्नुहोस् - मलाई `nitya-mshack-azd` प्रयोग गरियो
१. यसले `.azure/` फोल्डर सिर्जना गर्नेछ - तपाईंले वातावरण नामसहितको उपफोल्डर देख्नुहुनेछ
१. सदस्यता नाम चयन गर्न अनुरोध गरिनेछ - पूर्वनिर्धारित चयन गर्नुहोस्
१. स्थानको लागि अनुरोध गरिनेछ - `East US 2` प्रयोग गर्नुहोस्

अब, प्रावधान गर्ने काम पूरा हुन पर्खनुहोस्। **यसमा १०-१५ मिनेट लाग्छ**

१. जब सम्पन्न हुन्छ, तपाईंको कन्सोलले यसरी SUCCESS सन्देश देखाउनेछ:
      ```bash title="" linenums="0"
      SUCCESS: Your up workflow to provision and deploy to Azure completed in 10 minutes 17 seconds.
      ```
१. अब तपाईंको Azure पोर्टलमा उक्त वातावरण नामसहित प्रावधान गरिएको स्रोत समूह हुनेछ:

      ![Infra](../../../../../translated_images/ne/02-provisioned-infra.46c706b14f56e0bf.webp)

१. **अब तपाईंले तैनाथ गरिएको पूर्वाधार र अनुप्रयोगलाई मान्य गर्न तयार हुनुहुन्छ**।

## ५. एजेन्ट मान्यता

Azure Container App ले एउटा अन्तबिन्दु तैनाथ गर्छ जुन यस टेम्प्लेटका लागि Microsoft Foundry प्रोजेक्टमा प्रावधान गरिएको AI एजेन्टसँग जडित छ। यसले के अर्थ राख्छ हेर्नुहोस्।

१. तपाईंको स्रोत समूहको Azure पोर्टल _Overview_ पृष्ठमा फर्कनुहोस्

१. त्यस सूचीमा `Microsoft Foundry` स्रोतमा क्लिक गर्नुहोस्

१. तपाईंले यो देख्नुहुनेछ। `Go to Microsoft Foundry Portal` बटनमा क्लिक गर्नुहोस्। 
   ![Foundry](../../../../../translated_images/ne/04-view-foundry-project.fb94ca41803f28f3.webp)

१. तपाईंले तपाईंको AI अनुप्रयोगको Foundry प्रोजेक्ट पृष्ठ देख्नुहुनेछ
   ![Project](../../../../../translated_images/ne/05-visit-foundry-portal.d734e98135892d7e.webp)

१. `Agents` मा क्लिक गर्नुहोस् - तपाईंले तपाईंको प्रोजेक्टमा पूर्वनिर्धारित एजेन्ट देख्नुहुनेछ
   ![Agents](../../../../../translated_images/ne/06-visit-agents.bccb263f77b00a09.webp)

१. यसलाई छनौट गर्नुहोस् - र तपाईं एजेन्ट विवरणहरू देख्नुहुनेछ। निम्न कुरामा ध्यान दिनुहोस्:

      - एजेन्टले डिफल्ट रूपमा फाइल खोज प्रयोग गर्छ (सधैं)
      - एजेन्टको `Knowledge` मा देखिन्छ कि यसमा ३२ फाइलहरू अपलोड गरिएका छन् (फाइल खोजको लागि)
      ![Agents](../../../../../translated_images/ne/07-view-agent-details.0e049f37f61eae62.webp)

१. बाँया मेनुमा `Data+indexes` विकल्प खोज्नुहोस् र विवरणहरूको लागि क्लिक गर्नुहोस्। 

      - तपाईंले ज्ञानका लागि अपलोड गरिएका ३२ डेटा फाइलहरू देख्नुहुनेछ।
      - ती `src/files` अन्तर्गत १२ ग्राहक फाइलहरू र २० उत्पादन फाइलहरू मेल खान्छन्
      ![Data](../../../../../translated_images/ne/08-visit-data-indexes.5a4cc1686fa0d19a.webp)

**तपाईंले एजेन्ट सञ्चालनको मान्यता गर्नुभयो!** 

१. एजेन्टका प्रतिक्रिया ती फाइलहरूमा भएको ज्ञानमा आधारित छन्। 
१. अब तपाईं त्यो डाटासँग सम्बन्धित प्रश्नहरू सोध्न सक्नुहुन्छ र आधारित प्रतिक्रिया पाउनुहुन्छ।
१. उदाहरण: `customer_info_10.json` ले "Amanda Perez" द्वारा गरिएको ३ खरिदहरू वर्णन गर्छ

ब्राउजर ट्याबमा Container App अन्तबिन्दुमा फर्केर सोध्नुहोस्: `Amanda Perez कुन-कुन उत्पादनहरू मालिक छ?`। तपाईंले यसरी केही देख्नुहुनेछ:

## ६. एजेन्ट खेलमैदान

१. Microsoft Foundry मा `Agents` पृष्ठमा फर्कनुहोस् - डिफल्ट एजेन्ट चयन गर्नुहोस्
१. `Try in Playground` विकल्पमा क्लिक गर्नुहोस् - तपाईंले यस प्रकारको Playground UI पाउनुहुनेछ
१. समान प्रश्न सोध्नुहोस्: `Amanda Perez कुन-कुन उत्पादनहरू मालिक छ?`

    ![Data](../../../../../translated_images/ne/09-ask-in-playground.a1b93794f78fa676.webp)

तपाईंले उस्तै (वा समान) प्रतिक्रिया पाउनुहुनेछ - तर तपाईंले थप जानकारी पनि पाउनुहुनेछ जसले तपाईंको एजेन्टिक अनुप्रयोगको गुणस्तर, लागत, र प्रदर्शन बुझ्न मद्दत गर्छ। उदाहरणका लागि:

१. प्रतिक्रिया "ग्राउन्ड" गर्न प्रयोग गरिएका डाटा फाइलहरूको उल्लेख गर्छ
१. यी कुनै पनि फाइल लेबलहरूमा होभर गर्नुहोस् - के डाटा तपाईंको प्रश्न र देखाइएको प्रतिक्रियासँग मेल खान्छ?

तपाईं प्रतिक्रियाको तल एउटा _stats_ पङ्क्ति पनि देख्नुहुनेछ। 

१. कुनै पनि मेट्रिकमा होभर गर्नुहोस् - जस्तै Safety। तपाईंले यस प्रकार देख्नुहुनेछ
१. मूल्याङ्कन गरिएको रेटिंग तपाईंको प्रतिक्रिया सुरक्षा स्तरको अन्तर्दृष्टिसँग मेल खान्छ?

## ७. निर्मित अवलोकनयोग्यता

अवलोकनयोग्यता भन्नाले तपाईंको अनुप्रयोगलाई डेटा उत्पादन गर्न सुसज्जित गर्न हो जुन यसको कार्य सञ्चालनलाई बुझ्न, डिबग गर्न, र अप्टिमाइज गर्न प्रयोग गर्न सकिन्छ। यसको भावना पाउन:

१. `View Run Info` बटनमा क्लिक गर्नुहोस् - तपाईंले यो दृश्य देख्नुहुनेछ। यो [Agent tracing](https://learn.microsoft.com/en-us/azure/ai-foundry/how-to/develop/trace-agents-sdk#view-trace-results-in-the-azure-ai-foundry-agents-playground) को उदाहरण हो। _तपाईंले माथिल्लो स्तरको मेनुमा Thread Logs मा क्लिक गरेर पनि यो दृश्य प्राप्त गर्न सक्नुहुन्छ_।

   - एजेन्टले प्रयोग गरेका रन चरणहरू र उपकरणहरूको भावना लिनुहोस्
   - प्रतिक्रियाको लागि कुल Token संख्या (प्रयोग भएका आउटपुट टोकनहरू सापेक्ष) बुझ्नुहोस्

## ८. वातावरण चरहरू

अहिलेसम्म हामीले ब्राउजरमा तैनाथीकरण हेर्यौं - र पक्का गर्यौं कि हाम्रो पूर्वाधार प्रावधान गरिएको छ र अनुप्रयोग सञ्चालनमा छ। तर अनुप्रयोगलाई _पहिले कोडको साथ_ काम गर्न, हामीले यी स्रोतहरूसँग काम गर्न आवश्यक सम्बन्धित चरहरू सेट गरेर हाम्रो स्थानीय विकास वातावरण कन्फिगर गर्नुपर्छ। `azd` प्रयोग गर्दा यो सजिलो हुन्छ।

१. Azure Developer CLI [पर्यावरण चरहरू](https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/manage-environment-variables?tabs=bash) प्रयोग गर्दछ जुन अनुप्रयोग तैनाथीकरणका लागि कन्फिगरेसन सेटिङहरू भण्डारण र व्यवस्थापन गर्न सकिन्छ।

१. वातावरण चरहरू `.azure/<env-name>/.env` मा भण्डारण हुन्छन् - यसले तिनीहरूलाई प्रयोग गरिएको `env-name` वातावरणमा सीमित गर्दछ र उत्तिकै भण्डारमा फरक तैनाथीकरण लक्ष्यों बीच वातावरणहरू अलग राख्न मद्दत गर्दछ।

१. वातावरण चरहरू स्वतः `azd` आदेशले विशिष्ट आदेश चलाउँदा (जस्तै, `azd up`) लोड गर्छ। ध्यान दिनुहोस् `azd` स्वचालित रूपमा _OS-स्तर_ वातावरण चरहरू (जस्तै, शेलमा सेट गरिएका) पढ्दैन - यसको सट्टा स्क्रिप्टहरूमा जानकारी ट्रान्सफर गर्न `azd set env` र `azd get env` प्रयोग गर्नुहोस्।


केही आदेशहरू प्रयास गरौं:

१. यस वातावरणमा `azd` का लागि सेट गरिएका सबै वातावरण चरहरू पाउन:

      ```bash title="" linenums="0"
      azd env get-values
      ```
      
      तपाईं यसै जस्तो केही देख्नुहुनेछ:

      ```bash title="" linenums="0"
      AZURE_AI_AGENT_DEPLOYMENT_NAME="gpt-4.1-mini"
      AZURE_AI_AGENT_NAME="agent-template-assistant"
      AZURE_AI_EMBED_DEPLOYMENT_NAME="text-embedding-3-small"
      AZURE_AI_EMBED_DIMENSIONS=100
      ...
      ```

१. एउटा विशेष मान पाउन - जस्तै, मैले जान्न चाहन्छु कि `AZURE_AI_AGENT_MODEL_NAME` मान सेट गरिएको छ कि छैन

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```
      
      तपाईं यसरी देख्नुहुनेछ - यो पूर्वनिर्धारित रूपमा सेट गरिएको थिएन!

      ```bash title="" linenums="0"
      ERROR: key 'AZURE_AI_AGENT_MODEL_NAME' not found in the environment values
      ```

१. `azd` का लागि नयाँ वातावरण चर सेट गर्नुहोस्। यहाँ, हामीले एजेन्ट मोडल नाम अपडेट गर्यौं। _टिप: गरिएका परिवर्तनहरू तुरुन्त `.azure/<env-name>/.env` फाइलमा प्रतिबिम्बित हुन्छन्।

      ```bash title="" linenums="0"
      azd env set AZURE_AI_AGENT_MODEL_NAME gpt-4.1
      azd env set AZURE_AI_AGENT_MODEL_VERSION 2025-04-14
      azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 150
      ```

      अब, हामीले मान सेट भएको पाउनु पर्छ:

      ```bash title="" linenums="0"
      azd env get-value AZURE_AI_AGENT_MODEL_NAME 
      ```

१. ध्यान दिनुहोस् कि केही स्रोतहरू स्थायी हुन्छन् (जस्तै, मोडल तैनाथीकरणहरू) र परिवर्तित वातावरण चरहरूसँग पुनः तैनाथीकरण गर्न `azd up` भन्दा बढी आवश्यक पर्छ। मूल तैनाथीकरणलाई हटाएर फेरि परिवर्तित env vars सँग पुनः तैनाथ गर्नुहोस् प्रयास गरौं।

१. **रिफ्रेश गर्नुहोस्** यदि तपाईंले पहिले कुनै azd टेम्प्लेट प्रयोग गरेर पूर्वाधार तैनाथ गर्नुभएको छ भने - तपाईंले व्यवस्थापन गर्न सक्नुहुन्छ स्थानीय वातावरण चरहरूको अवस्था आधारित वर्तमान Azure तैनाथीकरण अवस्थितिमा यो आदेश प्रयोग गरेर:

      ```bash title="" linenums="0"
      azd env refresh
      ```


      यो एक शक्तिशाली तरिका हो दुई वा सो भन्दा बढी स्थानीय विकास वातावरणहरूमा पर्यावरण चरहरू _समकालिक_ गर्न (उदाहरणका लागि, धेरै विकासकर्ताहरू भएको टोली) - तैनाथ गरिएको पूर्वाधारलाई पर्यावरण चर स्थितिको आधार सत्‍ता बन्न दिन्छ। टोलीका सदस्यहरू केवल चरहरू _रिफ्रेश_ गरेर पुनः समकालिक हुन्छन्।
