## परिचय

यो व्यापक FAQ ले Azure Developer CLI (azd) र Azure डिप्लोयमेन्टसम्बन्धी सबैभन्दा सामान्य प्रश्नहरूको उत्तरहरू प्रदान गर्दछ। सामान्य समस्याहरूका त्वरित समाधानहरू पत्ता लगाउनुहोस्, उत्तम अभ्यासहरू बुझ्नुहोस्, र azd अवधारणा र कार्यप्रवाहहरूमा स्पष्टता प्राप्त गर्नुहोस्।

## सिकाइ लक्ष्यहरू

यस FAQ लाई समीक्षा गरेर, तपाईंले:
- सामान्य Azure Developer CLI प्रश्नहरू र समस्याहरूका त्वरित उत्तरहरू पत्ता लगाउनुहुनेछ
- व्यावहारिक Q&A ढाँचामा प्रमुख अवधारणाहरू र शब्दावली बुझ्नुहुनेछ
- बारम्बार समस्या र त्रुटि परिदृश्यहरूको लागि समस्या समाधान समाधानहरू पहुँच गर्नुहुनेछ
- अनुकूलनसम्बन्धी सामान्य प्रश्नहरूमार्फत उत्तम अभ्यासहरू सिक्नुहुनेछ
- विज्ञस्तरका प्रश्नहरूमार्फत उन्नत सुविधाहरू र क्षमताहरू पत्ता लगाउनुहुनेछ
- लागत, सुरक्षा, र डिप्लोयमेन्ट रणनीति सम्बन्धी सल्लाह छिटो सन्दर्भ गर्न सक्षम हुनुहुनेछ

## Getting Started

### Q: Azure Developer CLI (azd) भनेको के हो?
**A**: Azure Developer CLI (azd) एउटा विकासक-केंद्रित कमाण्ड-लाइन उपकरण हो जसले तपाईंको अनुप्रयोगलाई स्थानीय विकास वातावरणबाट Azure मा पठाउन लाग्ने समय छिटो बनाउँछ। यसले टेम्पलेटहरू मार्फत उत्तम अभ्यासहरू प्रदान गर्दछ र सम्पूर्ण डिप्लोयमेन्ट जीवनचक्रमा मद्दत गर्छ।

### Q: azd र Azure CLI बीच के फरक छ?
**A**: 
- **Azure CLI**: Azure स्रोतहरू व्यवस्थापन गर्नको लागि सामान्य-उद्देश्यको उपकरण
- **azd**: अनुप्रयोग डिप्लोयमेन्ट कार्यप्रवाहहरूका लागि विकासक-केंद्रित उपकरण
- azd ले आन्तरिक रूपमा Azure CLI प्रयोग गर्छ तर सामान्य विकास परिदृश्यहरूको लागि उच्च-स्तरीय अमूर्तताहरू प्रदान गर्छ
- azd मा टेम्पलेटहरू, वातावरण व्यवस्थापन, र डिप्लोयमेन्ट स्वचालन समावेश छन्

### Q: azd प्रयोग गर्न Azure CLI स्थापना गर्नै पर्ने हो?
**A**: **A**: हो, azd ले प्रमाणीकरण र केही अपरेसनहरूको लागि Azure CLI आवश्यक गर्दछ। पहिले Azure CLI स्थापना गर्नुस्, त्यसपछि azd स्थापना गर्नुहोस्।

### Q: azd कुन प्रोग्रामिङ भाषाहरू समर्थन गर्छ?
**A**: azd भाषा-निरपेक्ष छ। यसले निम्नसँग काम गर्छ:
- Node.js/JavaScript/TypeScript
- Python
- .NET/C#
- Java
- Go
- PHP
- Static websites
- Containerized applications

### Q: के म azd लाई विद्यमान परियोजनासँग प्रयोग गर्न सक्छु?
**A**: हो! तपाईंले वा त:
1. `azd init` प्रयोग गरेर विद्यमान परियोजनाहरूमा azd कन्फिगरेसन थप्न सक्नुहुन्छ
2. विद्यमान परियोजनाहरूलाई azd टेम्पलेट संरचनासँग मेल खाने गरी अनुकूल पार्न सक्नुहुन्छ
3. आफ्नो विद्यमान आर्किटेक्चरमा आधारित अनुकूल टेम्पलेटहरू सिर्जना गर्न सक्नुहुन्छ

## Authentication & Access

### Q: azd प्रयोग गरेर कसरी Azure सँग प्रमाणीकरण गर्ने?
**A**: `azd auth login` प्रयोग गर्नुहोस् जसले Azure प्रमाणीकरणका लागि ब्राउजर विन्डो खोल्नेछ। CI/CD परिदृश्यहरूका लागि service principals वा managed identities प्रयोग गर्नुहोस्।

### Q: के म azd लाई बहु Azure सदस्यतासँग प्रयोग गर्न सक्छु?
**A**: हो। प्रत्येक वातावरणका लागि कुन सदस्यता प्रयोग गर्ने भनेर निर्दिष्ट गर्न `azd env set AZURE_SUBSCRIPTION_ID <subscription-id>` प्रयोग गर्नुहोस्।

### Q: azd मार्फत डिप्लोय गर्नका लागि मलाई कुन अनुमति(permissions) चाहिन्छ?
**A**: साधारणतया तपाईंलाई चाहिन्छ:
- **Contributor** भूमिका संसाधन समूह वा सदस्यताको लागि
- **User Access Administrator** यदि त्यस्ता स्रोतहरू डिप्लोय गर्दा भूमिका असाइनमेन्टहरू आवश्यक पर्छ भने
- विशिष्ट अनुमति प्रयोग गरिने टेम्पलेट र स्रोतहरूमा निर्भर गर्दछ

### Q: के म azd लाई CI/CD पाइपलाइनहरूमा प्रयोग गर्न सक्छु?
**A**: अवश्य! azd CI/CD एकीकरणको लागि डिजाइन गरिएको छ। प्रमाणीकरणका लागि service principals प्रयोग गर्नुहोस् र कन्फिगरेसनका लागि वातावरण चरहरू सेट गर्नुहोस्।

### Q: GitHub Actions मा प्रमाणीकरण कसरी ह्यान्डल गर्ने?
**A**: service principal प्रमाणहरू सहित Azure Login action प्रयोग गर्नुहोस्:
```yaml
- uses: azure/login@v1
  with:
    creds: ${{ secrets.AZURE_CREDENTIALS }}
- run: azd deploy --no-prompt
```

## Templates & Projects

### Q: azd टेम्पलेटहरू कहाँ फेला पार्न सकिन्छ?
**A**: 
- Official templates: [Azure-Samples/awesome-azd](https://github.com/Azure-Samples/awesome-azd)
- Community templates: GitHub मा "azd-template" खोज्नुहोस्
- उपलब्ध टेम्पलेटहरू ब्राउज गर्न `azd template list` प्रयोग गर्नुहोस्

### Q: म कसरी कस्टम टेम्पलेट बनाउने?
**A**: 
1. विद्यमान टेम्पलेट संरचनाबाट सुरु गर्नुहोस्
2. `azure.yaml`, पूर्वाधार फाइलहरू, र अनुप्रयोग कोड परिमार्जन गर्नुहोस्
3. `azd up` सँग राम्रोसँग परीक्षण गर्नुहोस्
4. उपयुक्त ट्यागहरूसहित GitHub मा प्रकाशित गर्नुहोस्

### Q: के म टेम्पलेटबिना azd प्रयोग गर्न सक्छु?
**A**: हो, विद्यमान परियोजनामा आवश्यक कन्फिगरेसन फाइलहरू सिर्जना गर्न `azd init` प्रयोग गर्नुहोस्। तपाईंले `azure.yaml` र पूर्वाधार फाइलहरू म्यानुअली कन्फिगर गर्नुपर्नेछ।

### Q: आधिकारिक र समुदाय टेम्पलेटहरूबीच के फरक छ?
**A**: 
- **Official templates**: Microsoft द्वारा मर्मत गरिएको, नियमित रूपमा अपडेट हुने, व्यापक डकुमेन्टेसन सहित
- **Community templates**: विकासकर्ताहरूले सिर्जना गरेका, विशेष प्रयोग केसहरूका लागि हुनसक्छन्, गुणस्तर र मर्मतमा भिन्नता हुनसक्छ

### Q: मेरो प्रोजेक्टमा टेम्पलेट कसरी अपडेट गर्ने?
**A**: टेम्पलेटहरू स्वचालित रूपमा अपडेट हुँदैनन्। तपाईंले गर्न सक्नेहरू:
1. स्रोत टेम्पलेटबाट परिवर्तनहरू म्यानुअली तुलना र मर्ज गर्नुहोस्
2. अपडेट गरिएको टेम्पलेट प्रयोग गरेर `azd init` सँग नयाँ सुरु गर्नुहोस्
3. अपडेटेड टेम्पलेटबाट विशेष सुधारहरू चुरी-पिक गर्नुहोस्

## Deployment & Infrastructure

### Q: azd ले कुन Azure सेवाहरू डिप्लोय गर्न सक्छ?
**A**: azd ले Bicep/ARM टेम्पलेटहरू मार्फत कुनै पनि Azure सेवा डिप्लोय गर्न सक्छ, जस्तै:
- App Services, Container Apps, Functions
- Databases (SQL, PostgreSQL, Cosmos DB)
- Storage, Key Vault, Application Insights
- Networking, security, र monitoring स्रोतहरू

### Q: के म बहु क्षेत्रमा डिप्लोय गर्न सक्छु?
**A**: हो, आफ्नो Bicep टेम्पलेटहरूमा बहु क्षेत्रहरू कन्फिगर गर्नुहोस् र प्रत्येक वातावरणका लागि location प्यारामिटर उपयुक्त रूपमा सेट गर्नुहोस्।

### Q: म डाटाबेस स्किमा माइग्रेशनहरू कसरी ह्यान्डल गर्ने?
**A**: `azure.yaml` मा deployment hooks प्रयोग गर्नुहोस्:
```yaml
hooks:
  postdeploy:
    posix:
      run: ./scripts/migrate-database.sh
    windows:
      run: ./scripts/migrate-database.ps1
```

### Q: के म केवल पूर्वाधार मात्र एप्लिकेसन बिना डिप्लोय गर्न सक्छु?
**A**: हो, टेम्पलेटहरूमा परिभाषित पूर्वाधार कम्पोनन्टहरू मात्र डिप्लोय गर्न `azd provision` प्रयोग गर्नुहोस्।

### Q: म विद्यमान Azure स्रोतहरूमा कसरी डिप्लोय गर्ने?
**A**: यो जटिल छ र सिधै समर्थन गरिएको छैन। तपाईंले गर्न सक्नुहुन्छ:
1. आफ्नो Bicep टेम्पलेटहरूमा विद्यमान स्रोतहरू आयात गर्नुहोस्
2. टेम्पलेटहरूमा विद्यमान स्रोत सन्दर्भहरू प्रयोग गर्नुहोस्
3. स्रोतहरू सशर्त रूपमा सिर्जना वा सन्दर्भ गर्ने गरी टेम्पलेटहरू परिमार्जन गर्नुहोस्

### Q: के म Terraform प्रयोग गर्न सक्छु Bicep को सट्टा?
**A**: हाल, azd प्रायः Bicep/ARM टेम्पलेटहरूलाई समर्थन गर्छ। Terraform को आधिकारिक समर्थन उपलब्ध छैन, यद्यपि समुदाय-आधारित समाधानहरू हुनसक्छन्।

## Configuration & Environments

### Q: म फरक वातावरणहरू (dev, staging, prod) कसरी व्यवस्थापन गर्ने?
**A**: `azd env new <environment-name>` प्रयोग गरेर छुट्टा वातावरणहरू सिर्जना गर्नुहोस् र प्रत्येकको लागि फरक सेटिङहरू कन्फिगर गर्नुहोस्:
```bash
azd env new development
azd env new staging  
azd env new production
```

### Q: वातावरण कन्फिगरेसनहरू कहाँ सुरक्षित हुन्छन्?
**A**: प्रोजेक्ट डाइरेक्टरीभित्र `.azure` फोल्डरमा। प्रत्येक वातावरणसँग आफ्नै फोल्डर र कन्फिग फाइलहरू हुन्छन्।

### Q: म वातावरण-विशिष्ट कन्फिगरेसन कसरी सेट गर्ने?
**A**: वातावरण चरहरू कन्फिगर गर्न `azd env set` प्रयोग गर्नुहोस्:
```bash
azd env set AZURE_LOCATION eastus
azd env set DATABASE_TIER Basic
```

### Q: के म टोली सदस्यहरू माझ वातावरण कन्फिगरेसनहरू साझा गर्न सक्छु?
**A**: `.azure` फोल्डरमा संवेदनशील जानकारी समावेश हुन्छ र यसलाई भर्सन कन्ट्रोलमा कमिट गर्नु हुँदैन। बरु:
1. आवश्यक वातावरण चरहरू डकुमेन्ट गर्नुहोस्
2. वातावरण सेटअप गर्न डिप्लोयमेन्ट स्क्रिप्टहरू प्रयोग गर्नुहोस्
3. संवेदनशील कन्फिगरेसनका लागि Azure Key Vault प्रयोग गर्नुहोस्

### Q: म टेम्पलेट प्रारम्भिक मानहरू कसरी ओभरराइड गर्ने?
**A**: टेम्पलेट प्यारामिटरहरूसँग मेल खाने वातावरण चरहरू सेट गर्नुहोस्:
```bash
azd env set LOCATION "West US 2"
azd env set SKU_NAME "B1"
```

## Troubleshooting

### Q: किन `azd up` असफल भइरहेको छ?
**A**: सामान्य कारणहरू:
1. **प्रमाणीकरण समस्याहरू**: `azd auth login` चलाउनुहोस्
2. **अपर्याप्त अनुमति**: आफ्नो Azure भूमिका आवंटनहरू जाँच गर्नुहोस्
3. **स्रोत नामकरण द्वन्द्वहरू**: AZURE_ENV_NAME परिवर्तन गर्नुहोस्
4. **कोटा/क्षमता समस्याहरू**: क्षेत्रीय उपलब्धता जाँच गर्नुहोस्
5. **टेम्पलेट त्रुटिहरू**: Bicep टेम्पलेटहरू मान्य गर्नुहोस्

### Q: म डिप्लोयमेन्ट असफलताहरू कसरी डिबग गर्ने?
**A**: 
1. विस्तृत आउटपुटका लागि `azd deploy --debug` प्रयोग गर्नुहोस्
2. Azure पोर्टलमा डिप्लोयमेन्ट इतिहास जाँच गर्नुहोस्
3. Azure पोर्टलमा Activity Log समीक्षा गर्नुहोस्
4. वर्तमान वातावरण स्थिति देखाउन `azd show` प्रयोग गर्नुहोस्

### Q: मेरो वातावरण चरहरू किन काम गरिरहेका छैनन्?
**A**: जाँच गर्नुहोस्:
1. चर नामहरू टेम्पलेट प्यारामिटरहरूसँग ठीक मिल्छन्
2. मानहरूमा स्पेसहरू भएमा उचित रूपमा उद्धरण गरिएको छ
3. वातावरण चयन गरिएको छ: `azd env select <environment>`
4. चरहरू सही वातावरणमा सेट गरिएको छ

### Q: मैलाले भरिएको डिप्लोयमेन्टहरू कसरी क्लिनअप गर्ने?
**A**: 
```bash
azd down --force --purge
```
यसले सबै स्रोतहरू र वातावरण कन्फिगरेसनहरू हटाउँछ।

### Q: डिप्लोयमेन्टपछि मेरो अनुप्रयोग पहुँचयोग्य किन छैन?
**A**: जाँच गर्नुहोस्:
1. डिप्लोयमेन्ट सफलतापूर्वक पूरा भयो कि भएन
2. अनुप्रयोग चलिरहेको छ (Azure पोर्टलमा लगहरू जाँच गर्नुहोस्)
3. नेटवर्क सुरक्षा समूहहरूले ट्राफिक अनुमति दिइरहेका छन्
4. DNS/कस्टम डोमेनहरू ठीकसँग कन्फिगर गरिएको छ

## Cost & Billing

### Q: azd डिप्लोयमेन्टहरूले कति खर्च लाग्नेछन्?
**A**: लागतहरूमा निर्भर गर्दछ:
- डिप्लोय गरिएका Azure सेवाहरू
- चयन गरिएका सेवा टियरहरू/SKUs
- क्षेत्रीय मूल्य विभेदहरू
- प्रयोग ढाँचाहरू

अनुमानहरूको लागि [Azure मूल्य निर्धारण क्यालकुलेटर](https://azure.microsoft.com/pricing/calculator/) प्रयोग गर्नुहोस्।

### Q: azd डिप्लोयमेन्टहरूमा लागत कसरी नियन्त्रण गर्ने?
**A**: 
1. विकास वातावरणका लागि कम-टियर SKUs प्रयोग गर्नुहोस्
2. Azure बजेट र अलर्ट सेटअप गर्नुहोस्
3. आवश्यक नभएमा स्रोतहरू हटाउन `azd down` प्रयोग गर्नुहोस्
4. उपयुक्त क्षेत्रहरू भ्रमण गर्नुहोस् (लागत स्थानअनुसार फरक पर्छ)
5. Azure Cost Management उपकरणहरू प्रयोग गर्नुहोस्

### Q: के azd टेम्पलेटहरूका लागि निःशुल्क टियर विकल्पहरू छन्?
**A**: धेरै Azure सेवाहरूले निःशुल्क टियरहरू प्रस्ताव गर्छन्:
- App Service: निःशुल्क टियर उपलब्ध
- Azure Functions: 1M निःशुल्क executions/महिना
- Cosmos DB: 400 RU/s सहित निःशुल्क टियर
- Application Insights: पहिलो 5GB/महिना निःशुल्क

उपलब्ध अवस्थामा निःशुल्क टियरहरू प्रयोग गर्न टेम्पलेटहरू कन्फिगर गर्नुहोस्।

### Q: डिप्लोयमेन्ट अघि लागतहरू कसरी अनुमान गर्ने?
**A**: 
1. कुन स्रोतहरू सिर्जना हुन्छन् हेर्न टेम्पलेटको `main.bicep` समीक्षा गर्नुहोस्
2. विशिष्ट SKUs सहित Azure मूल्य निर्धारण क्यालकुलेटर प्रयोग गर्नुहोस्
3. वास्तविक लागतहरू अनुगमन गर्न पहिले विकास वातावरणमा डिप्लोय गर्नुहोस्
4. विस्तृत लागत विश्लेषणका लागि Azure Cost Management प्रयोग गर्नुहोस्

## Best Practices

### Q: azd प्रोजेक्ट संरचनाका लागि उत्तम अभ्यासहरू के हुन्?
**A**: 
1. पूर्वाधार र अनुप्रयोग कोड अलग राख्नुहोस्
2. `azure.yaml` मा अर्थपूर्ण सेवा नामहरू प्रयोग गर्नुहोस्
3. बिल्ड स्क्रिप्टहरूमा उचित त्रुटि ह्यान्डलिङ लागू गर्नुहोस्
4. वातावरण-विशिष्ट कन्फिगरेसन प्रयोग गर्नुहोस्
5. व्यापक डकुमेन्टेसन समावेश गर्नुहोस्

### Q: azd मा बहु सेवाहरूलाई कसरी संगठन गर्ने?
**A**: सिफारिस गरिएको संरचना प्रयोग गर्नुहोस्:
```
├── src/
│   ├── web/          # Frontend service
│   ├── api/          # Backend service  
│   └── worker/       # Background service
├── infra/            # Infrastructure templates
├── azure.yaml        # azd configuration
└── README.md         # Project documentation
```

### Q: के मैले `.azure` फोल्डरलाई भर्सन कन्ट्रोलमा कमिट गर्नुपर्छ?
**A**: **होइन!** `.azure` फोल्डरमा संवेदनशील जानकारी हुन्छ। यसलाई `.gitignore` मा थप्नुहोस्:
```gitignore
.azure/
```

### Q: गोप्य र संवेदनशील कन्फिगरेसन कसरी ह्यान्डल गर्ने?
**A**: 
1. गोप्यहरूको लागि Azure Key Vault प्रयोग गर्नुहोस्
2. अनुप्रयोग कन्फिगरेसनमा Key Vault गोप्यहरूको सन्दर्भ राख्नुहोस्
3. कहिल्यै गोप्यहरू भर्सन कन्ट्रोलमा कमिट नगर्नुहोस्
4. सेवा-देखि-सेवा प्रमाणीकरणका लागि managed identities प्रयोग गर्नुहोस्

### Q: azd सँग CI/CD को लागि सिफारिस गरिने दृष्टिकोण के हो?
**A**: 
1. प्रत्येक चरण (dev/staging/prod) का लागि अलग वातावरणहरू प्रयोग गर्नुहोस्
2. डिप्लोयमेन्ट अघि स्वचालित परीक्षणहरू लागू गर्नुहोस्
3. प्रमाणीकरणका लागि service principals प्रयोग गर्नुहोस्
4. संवेदनशील कन्फिगरेसन पाइपलाइन गोप्य/चरहरूमा राख्नुहोस्
5. उत्पादन डिप्लोयमेन्टहरूको लागि अनुमोदन गेटहरू लागू गर्नुहोस्

## Advanced Topics

### Q: के म azd लाई कस्टम कार्यक्षमतासहित विस्तार गर्न सक्छु?
**A**: हो, `azure.yaml` मा deployment hooks मार्फत:
```yaml
hooks:
  predeploy:
    run: ./scripts/custom-setup.sh
  postdeploy:
    run: ./scripts/custom-config.sh
```

### Q: म azd लाई विद्यमान DevOps प्रक्रियाहरूसँग कसरी एकीकृत गर्ने?
**A**: 
1. विद्यमान पाइपलाइन स्क्रिप्टहरूमा azd कमाण्डहरू प्रयोग गर्नुहोस्
2. टोलीहरूबीच azd टेम्पलेटहरू मानकीकृत गर्नुहोस्
3. विद्यमान मोनिटरिङ र अलर्टिङसँग एकीकृत गर्नुहोस्
4. पाइपलाइन एकीकरणका लागि azd को JSON आउटपुट प्रयोग गर्नुहोस्

### Q: के म azd लाई Azure DevOps सँग प्रयोग गर्न सक्छु?
**A**: हो, azd कुनै पनि CI/CD प्रणालीसँग काम गर्छ। azd कमाण्डहरू प्रयोग गर्ने Azure DevOps पाइपलाइनहरू सिर्जना गर्नुहोस्।

### Q: म azd मा कसरी योगदान गर्न सक्छु वा सामुदायिक टेम्पलेटहरू कसरी सिर्जना गर्ने?
**A**: तपाईंले गर्न सक्नुहुन्छ:
1. **azd उपकरण**: [Azure/azure-dev](https://github.com/Azure/azure-dev) मा योगदान गर्नुहोस्
2. **Templates**: [टेम्पलेट दिशानिर्देशहरू](https://github.com/Azure-Samples/awesome-azd) अनुसार टेम्पलेटहरू सिर्जना गर्नुहोस्
3. **प्रलेखन**: [MicrosoftDocs/azure-dev-docs](https://github.com/MicrosoftDocs/azure-dev-docs) मा कागजातहरूमा योगदान गर्नुहोस्

### Q: azd को रोडम्याप के हो?
**A**: योजनाबद्ध सुविधाहरू र सुधारहरूको लागि [आधिकारिक रोडम्याप](https://github.com/Azure/azure-dev/projects) जाँच गर्नुहोस्।

### Q: म अन्य परिनियोजन उपकरणहरूबाट azd मा कसरी स्थानान्तरण गर्छु?
**A**: 
1. वर्तमान परिनियोजन वास्तुकला विश्लेषण गर्नुहोस्
2. समतुल्य Bicep टेम्पलेटहरू सिर्जना गर्नुहोस्
3. वर्तमान सेवाहरू सँग मेल खाने गरी `azure.yaml` कन्फिगर गर्नुहोस्
4. विकास वातावरणमा पूर्ण रूपमा परीक्षण गर्नुहोस्
5. बिस्तारै वातावरणहरू माईग्रेट गर्नुहोस्
