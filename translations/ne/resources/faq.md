# बारम्बार सोधिने प्रश्नहरू (FAQ)

## परिचय

यो विस्तृत FAQ ले Azure Developer CLI (azd) र Azure तैनातीहरू सम्बन्धी सामान्य प्रश्नहरूको उत्तर प्रदान गर्दछ। सामान्य समस्याहरूको छिटो समाधान खोज्नुहोस्, उत्तम अभ्यासहरू बुझ्नुहोस्, र azd का अवधारणाहरू र कार्यप्रवाहहरूको स्पष्टता प्राप्त गर्नुहोस्।

## सिकाइका लक्ष्यहरू

यस FAQ लाई समीक्षा गरेर, तपाईं:
- Azure Developer CLI सम्बन्धी सामान्य प्रश्नहरू र समस्याहरूको छिटो उत्तर पाउन सक्नुहुन्छ
- व्यावहारिक प्रश्नोत्तर ढाँचाबाट प्रमुख अवधारणाहरू र शब्दावली बुझ्न सक्नुहुन्छ
- बारम्बार हुने समस्याहरू र त्रुटि परिदृश्यहरूको समाधान पहुँच गर्न सक्नुहुन्छ
- अनुकूलन सम्बन्धी सामान्य प्रश्नहरू मार्फत उत्तम अभ्यासहरू सिक्न सक्नुहुन्छ
- विशेषज्ञ स्तरका प्रश्नहरू मार्फत उन्नत सुविधाहरू र क्षमताहरू पत्ता लगाउन सक्नुहुन्छ
- लागत, सुरक्षा, र तैनाती रणनीति मार्गदर्शनलाई प्रभावकारी रूपमा सन्दर्भ गर्न सक्नुहुन्छ

## सिकाइका परिणामहरू

यस FAQ लाई नियमित रूपमा सन्दर्भ गरेर, तपाईं:
- प्रदान गरिएको समाधानहरू प्रयोग गरेर स्वतन्त्र रूपमा सामान्य Azure Developer CLI समस्याहरू समाधान गर्न सक्नुहुन्छ
- तैनाती रणनीतिहरू र कन्फिगरेसनहरू बारे सूचित निर्णय लिन सक्नुहुन्छ
- azd र अन्य Azure उपकरणहरू र सेवाहरू बीचको सम्बन्ध बुझ्न सक्नुहुन्छ
- समुदायको अनुभव र विशेषज्ञ सिफारिसहरूमा आधारित उत्तम अभ्यासहरू लागू गर्न सक्नुहुन्छ
- प्रमाणीकरण, तैनाती, र कन्फिगरेसन समस्याहरू प्रभावकारी रूपमा समाधान गर्न सक्नुहुन्छ
- FAQ अन्तर्दृष्टि र सिफारिसहरू प्रयोग गरेर लागत र प्रदर्शन अनुकूलन गर्न सक्नुहुन्छ

## सुरु गर्दै

### प्रश्न: Azure Developer CLI (azd) के हो?
**उत्तर**: Azure Developer CLI (azd) एक डेवलपर-केंद्रित कमाण्ड-लाइन उपकरण हो जसले तपाईंको एप्लिकेसनलाई स्थानीय विकास वातावरणबाट Azure सम्म पुर्‍याउन लाग्ने समयलाई तीव्र बनाउँछ। यसले टेम्प्लेटहरू मार्फत उत्तम अभ्यासहरू प्रदान गर्दछ र सम्पूर्ण तैनाती जीवनचक्रमा मद्दत गर्दछ।

### प्रश्न: azd Azure CLI भन्दा कसरी फरक छ?
**उत्तर**: 
- **Azure CLI**: Azure स्रोतहरू व्यवस्थापन गर्न सामान्य उद्देश्यको उपकरण
- **azd**: एप्लिकेसन तैनाती कार्यप्रवाहहरूको लागि डेवलपर-केंद्रित उपकरण
- azd ले Azure CLI आन्तरिक रूपमा प्रयोग गर्दछ तर सामान्य विकास परिदृश्यहरूको लागि उच्च-स्तरीय अमूर्तता प्रदान गर्दछ
- azd ले टेम्प्लेटहरू, वातावरण व्यवस्थापन, र तैनाती स्वचालन समावेश गर्दछ

### प्रश्न: azd प्रयोग गर्न मलाई Azure CLI स्थापना गर्न आवश्यक छ?
**उत्तर**: हो, azd ले प्रमाणीकरण र केही अपरेसनहरूको लागि Azure CLI आवश्यक छ। पहिले Azure CLI स्थापना गर्नुहोस्, त्यसपछि azd स्थापना गर्नुहोस्।

### प्रश्न: azd ले कुन-कुन प्रोग्रामिङ भाषाहरूलाई समर्थन गर्दछ?
**उत्तर**: azd भाषा-निर्भर छैन। यसले निम्नसँग काम गर्दछ:
- Node.js/JavaScript/TypeScript
- Python
- .NET/C#
- Java
- Go
- PHP
- स्थिर वेबसाइटहरू
- कन्टेनराइज्ड एप्लिकेसनहरू

### प्रश्न: के म azd लाई विद्यमान परियोजनाहरूमा प्रयोग गर्न सक्छु?
**उत्तर**: हो! तपाईं निम्न तरिकाहरू प्रयोग गर्न सक्नुहुन्छ:
1. `azd init` प्रयोग गरेर विद्यमान परियोजनाहरूमा azd कन्फिगरेसन थप्नुहोस्
2. विद्यमान परियोजनाहरूलाई azd टेम्प्लेट संरचनासँग मिलाउन अनुकूल बनाउनुहोस्
3. तपाईंको विद्यमान आर्किटेक्चरमा आधारित कस्टम टेम्प्लेटहरू सिर्जना गर्नुहोस्

## प्रमाणीकरण र पहुँच

### प्रश्न: azd प्रयोग गरेर Azure मा कसरी प्रमाणीकरण गर्ने?
**उत्तर**: `azd auth login` प्रयोग गर्नुहोस् जसले Azure प्रमाणीकरणको लागि ब्राउजर विन्डो खोल्छ। CI/CD परिदृश्यहरूको लागि, सेवा प्रिन्सिपलहरू वा व्यवस्थापित पहिचानहरू प्रयोग गर्नुहोस्।

### प्रश्न: के म azd लाई धेरै Azure सदस्यताहरूमा प्रयोग गर्न सक्छु?
**उत्तर**: हो। प्रत्येक वातावरणको लागि कुन सदस्यता प्रयोग गर्ने निर्दिष्ट गर्न `azd env set AZURE_SUBSCRIPTION_ID <subscription-id>` प्रयोग गर्नुहोस्।

### प्रश्न: azd प्रयोग गरेर तैनाती गर्न मलाई कुन-कुन अनुमति आवश्यक छ?
**उत्तर**: सामान्यतया तपाईंलाई निम्न आवश्यक छ:
- स्रोत समूह वा सदस्यतामा **Contributor** भूमिका
- भूमिका असाइनमेन्टहरू आवश्यक पर्ने स्रोतहरू तैनाती गर्दा **User Access Administrator**
- टेम्प्लेट र तैनाती गरिएका स्रोतहरू अनुसार विशिष्ट अनुमति फरक हुन सक्छ

### प्रश्न: के म azd लाई CI/CD पाइपलाइनहरूमा प्रयोग गर्न सक्छु?
**उत्तर**: अवश्य! azd CI/CD एकीकरणको लागि डिजाइन गरिएको हो। प्रमाणीकरणको लागि सेवा प्रिन्सिपलहरू प्रयोग गर्नुहोस् र कन्फिगरेसनको लागि वातावरण चरहरू सेट गर्नुहोस्।

### प्रश्न: GitHub Actions मा प्रमाणीकरण कसरी व्यवस्थापन गर्ने?
**उत्तर**: सेवा प्रिन्सिपल प्रमाणहरू प्रयोग गरेर Azure Login action प्रयोग गर्नुहोस्:
```yaml
- uses: azure/login@v1
  with:
    creds: ${{ secrets.AZURE_CREDENTIALS }}
- run: azd deploy --no-prompt
```

## टेम्प्लेटहरू र परियोजनाहरू

### प्रश्न: म azd टेम्प्लेटहरू कहाँ फेला पार्न सक्छु?
**उत्तर**: 
- आधिकारिक टेम्प्लेटहरू: [Azure-Samples/awesome-azd](https://github.com/Azure-Samples/awesome-azd)
- समुदाय टेम्प्लेटहरू: "azd-template" को लागि GitHub खोजी गर्नुहोस्
- उपलब्ध टेम्प्लेटहरू ब्राउज गर्न `azd template list` प्रयोग गर्नुहोस्

### प्रश्न: म कसरी कस्टम टेम्प्लेट सिर्जना गर्न सक्छु?
**उत्तर**: 
1. विद्यमान टेम्प्लेट संरचनाबाट सुरु गर्नुहोस्
2. `azure.yaml`, पूर्वाधार फाइलहरू, र एप्लिकेसन कोड परिमार्जन गर्नुहोस्
3. `azd up` प्रयोग गरेर राम्रोसँग परीक्षण गर्नुहोस्
4. उपयुक्त ट्यागहरू सहित GitHub मा प्रकाशित गर्नुहोस्

### प्रश्न: के म टेम्प्लेट बिना azd प्रयोग गर्न सक्छु?
**उत्तर**: हो, विद्यमान परियोजनामा `azd init` प्रयोग गरेर आवश्यक कन्फिगरेसन फाइलहरू सिर्जना गर्नुहोस्। तपाईंले `azure.yaml` र पूर्वाधार फाइलहरू म्यानुअली कन्फिगर गर्न आवश्यक छ।

### प्रश्न: आधिकारिक र समुदाय टेम्प्लेटहरू बीच के फरक छ?
**उत्तर**: 
- **आधिकारिक टेम्प्लेटहरू**: Microsoft द्वारा व्यवस्थापन गरिएको, नियमित रूपमा अपडेट गरिएको, व्यापक दस्तावेजीकरण
- **समुदाय टेम्प्लेटहरू**: डेवलपरहरूद्वारा सिर्जना गरिएको, विशेष प्रयोग केसहरू हुन सक्छ, गुणस्तर र मर्मत फरक हुन सक्छ

### प्रश्न: मेरो परियोजनामा टेम्प्लेट कसरी अपडेट गर्ने?
**उत्तर**: टेम्प्लेटहरू स्वतः अपडेट हुँदैनन्। तपाईं:
1. स्रोत टेम्प्लेटबाट परिवर्तनहरू म्यानुअली तुलना र मर्ज गर्न सक्नुहुन्छ
2. अपडेट गरिएको टेम्प्लेट प्रयोग गरेर `azd init` बाट नयाँ सुरु गर्नुहोस्
3. अपडेट गरिएको टेम्प्लेटबाट विशिष्ट सुधारहरू चयन गर्नुहोस्

## तैनाती र पूर्वाधार

### प्रश्न: azd ले कुन-कुन Azure सेवाहरू तैनाती गर्न सक्छ?
**उत्तर**: azd ले Bicep/ARM टेम्प्लेटहरू मार्फत कुनै पनि Azure सेवा तैनाती गर्न सक्छ, जस्तै:
- App Services, Container Apps, Functions
- Databases (SQL, PostgreSQL, Cosmos DB)
- Storage, Key Vault, Application Insights
- Networking, सुरक्षा, र निगरानी स्रोतहरू

### प्रश्न: के म धेरै क्षेत्रहरूमा तैनाती गर्न सक्छु?
**उत्तर**: तपाईंको Bicep टेम्प्लेटहरूमा धेरै क्षेत्रहरू कन्फिगर गर्नुहोस् र प्रत्येक वातावरणको लागि स्थान प्यारामिटर उपयुक्त रूपमा सेट गर्नुहोस्।

### प्रश्न: म डेटाबेस स्कीमा माइग्रेशन कसरी व्यवस्थापन गर्ने?
**उत्तर**: `azure.yaml` मा तैनाती हुकहरू प्रयोग गर्नुहोस्:
```yaml
hooks:
  postdeploy:
    posix:
      run: ./scripts/migrate-database.sh
    windows:
      run: ./scripts/migrate-database.ps1
```

### प्रश्न: के म एप्लिकेसनहरू बिना मात्र पूर्वाधार तैनाती गर्न सक्छु?
**उत्तर**: हो, `azd provision` प्रयोग गरेर टेम्प्लेटहरूमा परिभाषित पूर्वाधार घटकहरू मात्र तैनाती गर्नुहोस्।

### प्रश्न: म विद्यमान Azure स्रोतहरूमा कसरी तैनाती गर्ने?
**उत्तर**: यो जटिल छ र प्रत्यक्ष रूपमा समर्थित छैन। तपाईं:
1. विद्यमान स्रोतहरूलाई तपाईंको Bicep टेम्प्लेटहरूमा आयात गर्न सक्नुहुन्छ
2. टेम्प्लेटहरूमा विद्यमान स्रोत सन्दर्भहरू प्रयोग गर्नुहोस्
3. स्रोतहरू सर्तपूर्वक सिर्जना वा सन्दर्भ गर्न टेम्प्लेटहरू परिमार्जन गर्नुहोस्

### प्रश्न: के म Bicep को सट्टा Terraform प्रयोग गर्न सक्छु?
**उत्तर**: हाल, azd मुख्य रूपमा Bicep/ARM टेम्प्लेटहरूलाई समर्थन गर्दछ। Terraform समर्थन आधिकारिक रूपमा उपलब्ध छैन, यद्यपि समुदाय समाधानहरू हुन सक्छ।

## कन्फिगरेसन र वातावरणहरू

### प्रश्न: म विभिन्न वातावरणहरू (dev, staging, prod) कसरी व्यवस्थापन गर्ने?
**उत्तर**: `azd env new <environment-name>` प्रयोग गरेर अलग-अलग वातावरणहरू सिर्जना गर्नुहोस् र प्रत्येकको लागि फरक सेटिङहरू कन्फिगर गर्नुहोस्:
```bash
azd env new development
azd env new staging  
azd env new production
```

### प्रश्न: वातावरण कन्फिगरेसनहरू कहाँ भण्डारण गरिन्छ?
**उत्तर**: तपाईंको परियोजना निर्देशिकाको `.azure` फोल्डरमा। प्रत्येक वातावरणको आफ्नै फोल्डर र कन्फिगरेसन फाइलहरू हुन्छन्।

### प्रश्न: म वातावरण-विशिष्ट कन्फिगरेसन कसरी सेट गर्ने?
**उत्तर**: `azd env set` प्रयोग गरेर वातावरण चरहरू कन्फिगर गर्नुहोस्:
```bash
azd env set AZURE_LOCATION eastus
azd env set DATABASE_TIER Basic
```

### प्रश्न: के म वातावरण कन्फिगरेसनहरू टीम सदस्यहरू बीच साझा गर्न सक्छु?
**उत्तर**: `.azure` फोल्डरमा संवेदनशील जानकारी समावेश हुन्छ र यसलाई संस्करण नियन्त्रणमा प्रतिबद्ध गर्नु हुँदैन। यसको सट्टा:
1. आवश्यक वातावरण चरहरू दस्तावेजीकरण गर्नुहोस्
2. वातावरण सेटअप गर्न तैनाती स्क्रिप्टहरू प्रयोग गर्नुहोस्
3. संवेदनशील कन्फिगरेसनको लागि Azure Key Vault प्रयोग गर्नुहोस्

### प्रश्न: म टेम्प्लेट डिफल्टहरू कसरी ओभरराइड गर्ने?
**उत्तर**: टेम्प्लेट प्यारामिटरहरूसँग मेल खाने वातावरण चरहरू सेट गर्नुहोस्:
```bash
azd env set LOCATION "West US 2"
azd env set SKU_NAME "B1"
```

## समस्या समाधान

### प्रश्न: किन `azd up` असफल भइरहेको छ?
**उत्तर**: सामान्य कारणहरू:
1. **प्रमाणीकरण समस्या**: `azd auth login` चलाउनुहोस्
2. **अपर्याप्त अनुमति**: तपाईंको Azure भूमिका असाइनमेन्टहरू जाँच गर्नुहोस्
3. **स्रोत नामकरण द्वन्द्वहरू**: AZURE_ENV_NAME परिवर्तन गर्नुहोस्
4. **कोटा/क्षमता समस्या**: क्षेत्रीय उपलब्धता जाँच गर्नुहोस्
5. **टेम्प्लेट त्रुटिहरू**: Bicep टेम्प्लेटहरू मान्य गर्नुहोस्

### प्रश्न: म तैनाती असफलताहरू कसरी डिबग गर्ने?
**उत्तर**: 
1. विस्तृत आउटपुटको लागि `azd deploy --debug` प्रयोग गर्नुहोस्
2. Azure पोर्टल तैनाती इतिहास जाँच गर्नुहोस्
3. Azure पोर्टलमा गतिविधि लग समीक्षा गर्नुहोस्
4. वर्तमान वातावरण स्थिति प्रदर्शन गर्न `azd show` प्रयोग गर्नुहोस्

### प्रश्न: किन मेरा वातावरण चरहरू काम गरिरहेका छैनन्?
**उत्तर**: जाँच गर्नुहोस्:
1. चर नामहरू टेम्प्लेट प्यारामिटरहरूसँग ठ्याक्कै मेल खान्छन्
2. मानहरूमा स्पेसहरू भएमा सही रूपमा उद्धृत गरिएको छ
3. वातावरण चयन गरिएको छ: `azd env select <environment>`
4. चरहरू सही वातावरणमा सेट गरिएको छ

### प्रश्न: म असफल तैनातीहरू कसरी सफा गर्ने?
**उत्तर**: 
```bash
azd down --force --purge
```
यसले सबै स्रोतहरू र वातावरण कन्फिगरेसन हटाउँछ।

### प्रश्न: किन मेरो एप्लिकेसन तैनाती पछि पहुँचयोग्य छैन?
**उत्तर**: जाँच गर्नुहोस्:
1. तैनाती सफलतापूर्वक सम्पन्न भयो
2. एप्लिकेसन चलिरहेको छ (Azure पोर्टलमा लगहरू जाँच गर्नुहोस्)
3. नेटवर्क सुरक्षा समूहहरूले ट्राफिकलाई अनुमति दिन्छ
4. DNS/कस्टम डोमेनहरू सही रूपमा कन्फिगर गरिएको छ

## लागत र बिलिङ

### प्रश्न: azd तैनातीहरूको लागत कति हुनेछ?
**उत्तर**: लागतहरू निर्भर गर्दछ:
- तैनाती गरिएका Azure सेवाहरू
- सेवा स्तर/SKU चयन गरिएको
- क्षेत्रीय मूल्य फरक
- प्रयोगका ढाँचाहरू

अनुमानहरूको लागि [Azure Pricing Calculator](https://azure.microsoft.com/pricing/calculator/) प्रयोग गर्नुहोस्।

### प्रश्न: म azd तैनातीहरूमा लागत कसरी नियन्त्रण गर्ने?
**उत्तर**: 
1. विकास वातावरणहरूको लागि तल्लो-स्तरका SKU प्रयोग गर्नुहोस्
2. Azure बजेटहरू र अलर्टहरू सेट गर्नुहोस्
3. आवश्यक नभएको बेला स्रोतहरू हटाउन `azd down` प्रयोग गर्नुहोस्
4. उपयुक्त क्षेत्रहरू चयन गर्नुहोस् (लागत स्थान अनुसार फरक हुन्छ)
5. Azure Cost Management उपकरणहरू प्रयोग गर्नुहोस्

### प्रश्न: के azd टेम्प्लेटहरूको लागि निःशुल्क स्तर विकल्पहरू छन्?
**उत्तर**: धेरै Azure सेवाहरूले निःशुल्क स्तरहरू प्रदान गर्छन्:
- App Service: निःशुल्क स्तर उपलब्ध
- Azure Functions: प्रति महिना 1M निःशुल्क कार्यान्वयनहरू
- Cosmos DB: 400 RU/s सहित निःशुल्क स्तर
- Application Insights: प्रति महिना पहिलो 5GB निःशुल्क

जहाँ उपलब्ध छ, टेम्प्लेटहरूलाई निःशुल्क स्तरहरू प्रयोग गर्न कन्फिगर गर्नुहोस्।

### प्रश्न: तैनाती अघि लागत कसरी अनुमान गर्ने?
**उत्तर**: 
1. टेम्प्लेटको `main.bicep` समीक्षा गरेर कुन स्रोतहरू सिर्जना गरिन्छ हेर्नुहोस्
2. विशिष्ट SKU हरूसँग Azure Pricing Calculator प्रयोग गर्नुहोस्
3. वास्तविक लागत अनुगमन गर्न पहिले विकास वातावरणमा तैनाती गर्नुहोस्
4. विस्तृत लागत विश्लेषणको लागि Azure Cost Management प्रयोग गर्नुहोस्

## उत्तम अभ्यासहरू

### प्रश्न: azd परियोजना संरचनाको लागि उत्तम अभ्यासहरू के हुन्?
**उत्तर**: 
1. एप्लिकेसन कोडलाई पूर्वाधारबाट अलग राख्नुहोस्
2. `azure.yaml` मा अर्थपूर्ण सेवा नामहरू प्रयोग गर्नुहोस्
3. निर्माण स्क्रिप्टहरूमा उचित त्रुटि ह्यान्डलिंग कार्यान्वयन गर्नुहोस्
4. वातावरण-विशिष्ट कन्फिगरेसन लागू गर्नुहोस्
5. व्यापक दस्तावेजीकरण समावेश गर्नुहोस्

### प्रश्न: म azd मा धेरै सेवाहरू कसरी व्यवस्थित गर्ने?
**उत्तर**: सिफारिस गरिएको संरचना प्रयोग गर्नुहोस्:
```
├── src/
│   ├── web/          # Frontend service
│   ├── api/          # Backend service  
│   └── worker/       # Background service
├── infra/            # Infrastructure templates
├── azure.yaml        # azd configuration
└── README.md         # Project documentation
```

### प्रश्न: के म `.azure` फोल्डरलाई संस्करण नियन्त्रणमा प्रतिबद्ध गर्नुपर्छ?
**उत्तर**: **होइन!** `.azure` फोल्डरमा संवेदनशील जानकारी समावेश हुन्छ। यसलाई `.gitignore` मा थप्नुहोस्:
```gitignore
.azure/
```

### प्रश्न: म गोप्य र संवेदनशील कन्फिगरेसन कसरी व्यवस्थापन गर्ने?
**उत्तर**: 
1. गोप्य जानकारीको लागि Azure Key Vault प्रयोग गर्नुहोस्
2. एप्लिकेसन कन्फिगरेसनमा Key Vault गोप्य जानकारीहरू सन्दर्भ गर्नुहोस्
3. गोप्य जानकारीलाई संस्करण नियन्त्रणमा कहिल्यै प्रतिबद्ध नगर्नुहोस्
4. सेवा-देखि-सेवा प्रमाणीकरणको लागि व्यवस्थापित पहिचानहरू प्रयोग गर्नुहोस्

### प्रश्न: azd को साथ CI/CD को लागि सिफारिस गरिएको दृष्टिकोण के हो?
**उत्तर**: 
1. प्रत्येक चरण (dev/staging/prod) को लागि अलग-अलग वातावरणहरू प्रयोग गर्नुहोस्
2. तैनाती अघि स्वचालित परीक्षण कार्यान्वयन गर्नुहोस्
3. प्रमाणीकरणको लागि सेवा प्रिन्सिपलहरू प्रयोग गर्नुहोस्
4. पाइपलाइन गोप्य जानकारी/चरहरूमा संवेदनशील कन्फिगरेसन भण्डारण गर्नुहोस्
5. उत्पादन तैनातीहरूको लागि अनुमोदन गेटहरू कार्यान्वयन गर्नुहोस्

## उन्नत विषयहरू

### प्रश्न: के म azd लाई कस्टम कार्यक्षमता संग विस्तार गर्न सक्छु?
**उत्तर**: हो, `azure.yaml` मा तैनाती हुकहरू मार्फत:
```yaml
hooks:
  predeploy:
    run: ./scripts/custom-setup.sh
  postdeploy:
    run: ./scripts/custom-config.sh
```

### प्रश्न: म विद्यम
1. हालको डिप्लोयमेन्ट आर्किटेक्चरको विश्लेषण गर्नुहोस्  
2. समान Bicep टेम्प्लेटहरू तयार गर्नुहोस्  
3. `azure.yaml` लाई हालका सेवाहरूको अनुरूप कन्फिगर गर्नुहोस्  
4. विकास वातावरणमा राम्रोसँग परीक्षण गर्नुहोस्  
5. वातावरणहरू क्रमिक रूपमा माइग्रेट गर्नुहोस्
