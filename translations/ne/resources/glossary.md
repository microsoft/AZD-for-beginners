# शब्दावली - Azure र AZD पदावली

**सबै अध्यायहरूको सन्दर्भ**

- **📖 आधारहरू सिक्नुहोस्**: [Chapter 1: AZD Basics](../docs/getting-started/azd-basics.md)
- **🤖 AI सम्बन्धी सर्तहरू**: [Chapter 2: AI-First Development](../docs/microsoft-foundry/microsoft-foundry-integration.md)

## परिचय

यो विस्तृत शब्दावलीले Azure Developer CLI र Azure क्लाउड विकासमा प्रयोग गरिने शब्दहरू, अवधारणाहरू, र सङ्क्षेपहरूको परिभाषा प्रदान गर्दछ। प्राविधिक दस्तावेज पढ्न, समस्याहरू समाधान गर्न, र azd परियोजनाहरू तथा Azure सेवाहरूको बारेमा प्रभावकारी रूपमा संवाद गर्न आवश्यक सन्दर्भ हो।

## सिक्ने लक्ष्यहरू

यो शब्दावली प्रयोग गरेर, तपाईं:
- Azure Developer CLI का आवश्यक पदावली र अवधारणाहरू बुझ्न सक्नुहुनेछ
- Azure क्लाउड विकासको शब्दावली र प्राविधिक शब्दहरूमा दक्ष हुन सक्नुहुनेछ
- Infrastructure as Code र डिप्लोयमेन्ट सम्बन्धी शब्दहरू कुशलतापूर्वक सन्दर्भ गर्न सक्नुहुनेछ
- Azure सेवा नामहरू, संक्षेपहरू, र तिनीहरूको उद्देश्यहरू बुझ्न सक्नुहुनेछ
- समस्याहरू समाधान र डिबगिङ सम्बन्धी शब्दहरूको परिभाषा पहुँच गर्न सक्नुहुनेछ
- उन्नत Azure आर्किटेक्चर र विकास अवधारणाहरू सिक्न सक्नुहुनेछ

## सिक्नुका नतिजाहरू

नियमित रूपमा यस शब्दावलीलाई सन्दर्भ गर्दा, तपाईं सक्षम हुनुहुनेछ:
- उचित Azure Developer CLI पदावली प्रयोग गरी प्रभावकारी रूपमा संवाद गर्न
- प्राविधिक दस्तावेज र त्रुटि सन्देशहरूलाई स्पष्ट रूपमा बुझ्न
- आत्मविश्वासका साथ Azure सेवाहरू र अवधारणाहरू नेभिगेट गर्न
- उपयुक्त प्राविधिक शब्दावली प्रयोग गरी समस्याहरू समाधान गर्न
- सहकर्मी छलफलहरूमा सही प्राविधिक भाषा प्रदान गर्न
- व्यवस्थित रूपमा आफ्नो Azure क्लाउड विकास ज्ञान विस्तार गर्न

## A

**ARM Template**  
Azure Resource Manager टेम्पलेट। Azure स्रोतहरू घोषणा गर्ने र तैनाथ गर्ने JSON-आधारित Infrastructure as Code ढाँचा।

**App Service**  
वेब अनुप्रयोग, REST API हरू, र मोबाइल ब्याकएन्ड होस्ट गर्नका लागि Azure को platform-as-a-service (PaaS) अफर, जसले पूर्वाधार व्यवस्थापन नगरी सेवा उपलब्ध गराउँछ।

**Application Insights**  
एप्लिकेशन प्रदर्शन, उपलब्धता, र उपयोगको गहिरो जानकारी प्रदान गर्ने Azure को application performance monitoring (APM) सेवा।

**Azure CLI**  
Azure स्रोतहरू व्यवस्थापन गर्नको लागि कमाण्ड-लाइन इन्टरफेस। azd द्वारा प्रमाणीकरण र केही अपरेसनहरूका लागि प्रयोग गरिन्छ।

**Azure Developer CLI (azd)**  
डेभलपर-केंद्रित कमाण्ड-लाइन उपकरण जसले टेम्पलेट र Infrastructure as Code प्रयोग गरी Azure मा अनुप्रयोग निर्माण र तैनाथ गर्ने प्रक्रियालाई तीव्र पार्दछ।

**azure.yaml**  
azd परियोजनाको मुख्य कन्फिगरेसन फाइल जसले सेवाहरू, पूर्वाधार, र डिप्लोयमेन्ट हुकहरू परिभाषित गर्छ।

**Azure Resource Manager (ARM)**  
Azure को डिप्लोयमेन्ट र व्यवस्थापन सेवा जसले स्रोतहरू सिर्जना, अपडेट, र मेटाउनको लागि प्रबन्धन तह प्रदान गर्छ।

## B

**Bicep**  
Microsoft द्वारा विकास गरिएको domain-specific language (DSL) जसले Azure स्रोतहरू तैनाथ गर्न प्रयोग हुन्छ। ARM टेम्पलेटभन्दा सरल सिन्ट्याक्स प्रदान गर्छ र ARM मा कम्पाइल हुन्छ।

**Build**  
स्रोत कोड कम्पाइल गर्ने, निर्भरताहरू इन्स्टल गर्ने, र अनुप्रयोगहरूलाई तैनाथिको लागि तयार गर्ने प्रक्रिया।

**Blue-Green Deployment**  
दुई समान उत्पादन वातावरणहरू (ब्लु र ग्रीन) प्रयोग गरी डाउनटाइम र जोखिम न्यून गर्ने डिप्लोयमेन्ट रणनीति।

## C

**Container Apps**  
जटिल पूर्वाधार व्यवस्थापन नगरी कन्टेनराइज्ड अनुप्रयोगहरू चलाउन सक्षम बनाउने Azure को serverless container सेवा।

**CI/CD**  
Continuous Integration/Continuous Deployment। कोड परिवर्तनहरू समेकन र अनुप्रयोगहरू तैनाथ गर्नका लागि स्वचालित अभ्यासहरू।

**Cosmos DB**  
दुनिया भर वितरित, बहु-मोडल डेटाबेस सेवा जसले थ्रूपुट, लेटेन्सी, उपलब्धता, र स्थिरताको लागि समग्र SLA हरू प्रदान गर्छ।

**Configuration**  
अनुप्रयोग व्यवहार र डिप्लोयमेन्ट विकल्पहरू नियन्त्रण गर्ने सेटिङ र प्यारामिटरहरू।

## D

**Deployment**  
लक्षित पूर्वाधारमा अनुप्रयोगहरू र तिनीहरूको निर्भरता स्थापना र कन्फिगर गर्ने प्रक्रिया।

**Docker**  
कन्टेनराइजेशन प्रविधि प्रयोग गरी अनुप्रयोगहरू विकास, पठाउने, र चलाउने प्लेटफर्म।

**Dockerfile**  
Docker कन्टेनर इमेज बनाउने निर्देशनहरूसँग भरिएको टेक्स्ट फाइल।

## E

**Environment**  
तपाईंको अनुप्रयोगको निश्चित उदाहरण प्रतिनिधित्व गर्ने डिप्लोयमेन्ट लक्ष्य (जस्तै: विकास, स्टेजिङ, उत्पादन)।

**Environment Variables**  
चलाइरहँदा अनुप्रयोगहरूले पहुँच गर्न सक्ने कुञ्जी-मूल्य जोडीहरूमा भण्डार गरिएका कन्फिगरेसन मानहरू।

**Endpoint**  
कुनै अनुप्रयोग वा सेवामा पहुँच गर्ने URL वा नेटवर्क ठेगाना।

## F

**Function App**  
इभेन्ट-ड्राइभन कोड चलाउन सक्षम गर्ने Azure को serverless compute सेवा, जसले पूर्वाधार व्यवस्थापन आवश्यक नपार्दछ।

## G

**GitHub Actions**  
GitHub रिपोजिटरीहरूसँग एकीकृत CI/CD प्लेटफर्म जुन वर्कफ्लोहरू स्वचालित पार्न प्रयोग गरिन्छ।

**Git**  
स्रोत कोडमा भएका परिवर्तनहरू ट्र्याक गर्न प्रयोग गरिने वितरित भेरियन नियन्त्रण प्रणाली।

## H

**Hooks**  
डिप्लोयमेन्ट जीवनचक्रका विशेष बिन्दुहरूमा चल्ने कस्टम स्क्रिप्ट वा कमाण्डहरू (preprovision, postprovision, predeploy, postdeploy)।

**Host**  
एउटा Azure सेवा प्रकार जहाँ अनुप्रयोग तैनाथ गरिनेछ (उदाहरण: appservice, containerapp, function)।

## I

**Infrastructure as Code (IaC)**  
पूर्वाधारलाई म्यानुअल प्रक्रियाको सट्टा कोड मार्फत परिभाषित र व्यवस्थापन गर्ने अभ्यास।

**Init**  
साधारणतया टेम्पलेटबाट नयाँ azd परियोजना प्रारम्भ गर्ने प्रक्रिया।

## J

**JSON**  
JavaScript Object Notation। कन्फिगरेसन फाइलहरू र API प्रतिक्रियाहरूका लागि सामान्यतया प्रयोग हुने डाटा विनिमय ढाँचा।

**JWT**  
JSON Web Token। पक्षहरूबीच सुरक्षीत रूपमा जानकारी प्रसारण गर्न प्रयोग हुने JSON वस्तु ढाँचा।

## K

**Key Vault**  
सिक्रेटहरू, कुञ्जीहरू, र सर्टिफिकेटहरू सुरक्षित रूपमा भण्डारण र व्यवस्थापन गर्न Azure को सेवा।

**Kusto Query Language (KQL)**  
Azure Monitor, Application Insights, र अन्य Azure सेवाहरूमा डाटा विश्लेषण गर्न प्रयोग हुने क्वेरी भाषा।

## L

**Load Balancer**  
आउँदो नेटवर्क ट्राफिकलाई धेरै सर्भर वा इन्स्ट्यान्सहरूमा वितरण गर्ने सेवा।

**Log Analytics**  
क्लाउड र अन-प्रिमाइसेस वातावरणबाट टेलीमेट्री डाटा संकलन, विश्लेषण, र कार्य गर्नका लागि Azure सेवा।

## M

**Managed Identity**  
अन्य Azure सेवाहरूमा प्रमाणिकरण गर्न स्वचालित रूपमा व्यवस्थापित आइडेन्टिटी प्रदान गर्ने Azure सुविधा।

**Microservices**  
अनुप्रयोगहरू साना, स्वतन्त्र सेवाहरूको संग्रहको रूपमा निर्माण गर्ने वास्तुकला दृष्टिकोण।

**Monitor**  
अनुप्रयोग र पूर्वाधारमा पूर्ण-स्ट्याक अवलोकनयोग्यताको लागि Azure को एकीकृत मनिटरिङ समाधान।

## N

**Node.js**  
Chrome को V8 JavaScript इन्जिनमा आधारित सर्वर-साइड अनुप्रयोग निर्माणका लागि JavaScript रनटाइम।

**npm**  
Node.js का निर्भरताहरू र प्याकेजहरू व्यवस्थापन गर्न प्रयोग हुने प्याकेज म्यानेजर।

## O

**Output**  
पूर्वाधार डिप्लोयमेन्टबाट फिर्ता गरिएका मानहरू जुन अनुप्रयोग वा अन्य स्रोतहरूले प्रयोग गर्न सक्छन्।

## P

**Package**  
अनुप्रयोग कोड र निर्भरताहरूलाई डिप्लोयमेन्टका लागि तयार पार्ने प्रक्रिया।

**Parameters**  
टेम्पलेटहरूमा पास गरिएका इनपुट मानहरू जसले डिप्लोयमेन्टहरूलाई अनुकूलन गर्छ।

**PostgreSQL**  
खुला स्रोत रिलेशनल डेटाबेस प्रणाली जसलाई Azure मा मेनेज्ड सेवा रूपमा समर्थन गरिन्छ।

**Provisioning**  
पूर्वाधार टेम्पलेटहरूमा परिभाषित Azure स्रोतहरू सिर्जना र कन्फिगर गर्ने प्रक्रिया।

## Q

**Quota**  
Azure सदस्यता वा क्षेत्रमा सिर्जना गर्न सकिने स्रोतहरूको मात्रा सम्बन्धी सीमा।

## R

**Resource Group**  
उही जीवनचक्र, अनुमति, र नीतिहरू साझा गर्ने Azure स्रोतहरूको तार्किक कन्टेनर।

**Resource Token**  
Deployment हरूमा स्रोत नामहरू अद्वितीय हुन सुनिश्चित गर्न azd द्वारा उत्पन्न गरिएको अद्वितीय स्ट्रिङ।

**REST API**  
HTTP विधिहरू प्रयोग गरी नेटवर्क सम्पर्कमा आधारित अनुप्रयोगहरू डिजाइन गर्ने आर्किटेक्चरल शैली।

**Rollback**  
अनुप्रयोग वा पूर्वाधार कन्फिगरेसनको पहिलेको संस्करणमा फर्किने प्रक्रिया।

## S

**Service**  
azure.yaml मा परिभाषित तपाईंको अनुप्रयोगको एउटा कम्पोनेन्ट (उदाहरण: वेब फ्रन्टएन्ड, API ब्याकएन्ड, डेटाबेस)।

**SKU**  
Stock Keeping Unit। Azure स्रोतहरूको फरक सेवा तहहरू वा प्रदर्शन स्तरहरू प्रतिनिधित्व गर्छ।

**SQL Database**  
Microsoft SQL Server मा आधारित Azure को मेनेज्ड रिलेशनल डेटाबेस सेवा।

**Static Web Apps**  
स्रोत कोड रिपोजिटरीहरूबाट फुल-स्ट्याक वेब अनुप्रयोगहरू निर्माण र तैनाथ गर्न Azure सेवा।

**Storage Account**  
ब्लब्स, फाइलहरू, क्यूहरू, र तालिकाहरू सहित डाटा वस्तुहरूको लागि क्लाउड भण्डारण प्रदान गर्ने Azure सेवा।

**Subscription**  
Azure खाता कन्टेनर जसले रिसोर्स समूहहरू र स्रोतहरू समावेश गर्दछ, र बिलिङ तथा पहुँच व्यवस्थापनसँग सम्बन्धित हुन्छ।

## T

**Template**  
पूर्व-निर्मित परियोजना संरचना जसले साझा परिदृश्यहरूको लागि अनुप्रयोग कोड, पूर्वाधार परिभाषाहरू, र कन्फिगरेसन समावेश गर्दछ।

**Terraform**  
ओपन सोर्स Infrastructure as Code उपकरण जसले Azure सहित बहु क्लाउड प्रदायकहरू समर्थन गर्छ।

**Traffic Manager**  
वैश्विक Azure क्षेत्रहरूमा ट्राफिक वितरण गर्नको लागि Azure को DNS-आधारित ट्राफिक लोड बेलान्सर।

## U

**URI**  
Uniform Resource Identifier। कुनै विशेष स्रोतलाई चिन्हित गर्ने स्ट्रिङ।

**URL**  
Uniform Resource Locator। एउटा URI प्रकार जुन स्रोत कहाँ अवस्थित छ र कसरी प्राप्त गर्ने भन्ने specifies गर्छ।

## V

**Virtual Network (VNet)**  
Azure मा निजी नेटवर्कहरूको मौलिक निर्माण ब्लक, जसले अलगाव र सेग्मेन्टेसन प्रदान गर्छ।

**VS Code**  
Visual Studio Code। उत्कृष्ट Azure र azd एकीकरण सहितको लोकप्रिय कोड सम्पादक।

## W

**Webhook**  
विशिष्ट घटनाहरूले ट्रिगर गर्ने HTTP कलब्याक, प्रायः CI/CD पाइपलाइनहरूमा प्रयोग गरिन्छ।

**What-if**  
डिप्लोयमेन्टले के-कस्तो परिवर्तनहरू गर्नेछ भनेर वास्तविक रूपमा कार्यान्वयन नगरी देखाउने Azure सुविधा।

## Y

**YAML**  
YAML Ain't Markup Language। azure.yaml जस्ता कन्फिगरेसन फाइलहरूका लागि प्रयोग हुने मानव-पठनीय डाटा सिरियलाइजेशन मापदण्ड।

## Z

**Zone**  
Azure क्षेत्रभित्रका भौतिक रूपमा पृथक स्थानहरू जसले redundancy र उच्च उपलब्धता प्रदान गर्छन्।

## सन्दर्भ-विशेष शब्दहरू

### विकास सम्बन्धी शब्दहरू
- **Hot Reload**: विकासको समयमा अनुप्रयोगहरूलाई पुनस्टार्ट नगरी स्वचालित रूपमा अपडेट गर्ने सुविधा
- **Build Pipeline**: कोड निर्माण र परीक्षणका लागि स्वचालित प्रक्रिया
- **Deployment Slot**: App Service भित्रको स्टेजिङ वातावरण
- **Environment Parity**: विकास, स्टेजिङ, र उत्पादन वातावरणहरूलाई समान राख्ने अभ्यास

### सुरक्षा सम्बन्धी शब्दहरू
- **Managed Identity**: स्वचालित प्रमाणीकरण क्रेडेन्शियल व्यवस्थापन प्रदान गर्ने Azure सुविधा
- **Key Vault**: सिक्रेटहरू, कुञ्जीहरू, र सर्टिफिकेटहरूको सुरक्षित भण्डारण
- **RBAC**: Azure स्रोतहरूको लागि भूमिकामा आधारित पहुँच नियन्त्रण
- **Network Security Group**: नेटवर्क ट्राफिक नियन्त्रण गर्नको लागि भर्चुअल फायरवाल

### मनिटरिङ सम्बन्धी शब्दहरू
- **Telemetry**: मापनहरू र डाटा स्वचालित रूपमा संकलन गर्ने प्रक्रिया
- **Application Performance Monitoring (APM)**: सफ्टवेयर प्रदर्शनको मनिटरिङ
- **Log Analytics**: लग डाटा संकलन र विश्लेषण गर्ने सेवा
- **Alert Rules**: मेट्रिक्स वा सर्तहरूका आधारमा स्वचालित सूचनाहरू

### डिप्लोयमेन्ट सम्बन्धी शब्दहरू
- **Blue-Green Deployment**: शून्य-डाउनटाइम डिप्लोयमेन्ट रणनीति
- **Canary Deployment**: प्रयोगकर्ताहरूको उपसमूहमा क्रमिक रोलआउट
- **Rolling Update**: अनुप्रयोग इन्स्ट्यान्सहरूको क्रमिक रूपमा प्रतिस्थापन
- **Rollback**: पहिलेको अनुप्रयोग संस्करणमा फर्किने प्रक्रिया
