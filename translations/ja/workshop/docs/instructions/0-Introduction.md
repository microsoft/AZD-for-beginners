# AZD for AI 開発者

!!! info "ワークショップの対象者"
   
    このワークショップは、最高水準のツールとワークフローを活用して Microsoft Foundry 上でエージェント型 AI アプリケーションを構築したいと考えている方を対象としています。ワークショップは、Azure、AI の概念、Python コードにある程度馴染みのある初心者および経験のある開発者の両方に適しています。開始する前に前提条件を確認してください。

    **Duration:** 3-4 hours  
    **Level:** Beginner to Intermediate  

---


## 前提条件

- 個人の GitHub アカウント → [無料で作成](https://github.com/signup)
- 有効な Azure アカウント → [無料で作成](https://aka.ms/free)
- AI の概念、Python 開発、コマンドラインツールに関する基本的な知識
- VS Code と GitHub のツールおよびワークフローの使用経験

---

## 学習目標

本ワークショップの終了時には、以下のことができるようになります:

- [X] **Work with Azure Developer CLI** → Azure リソースをプロビジョニング、デプロイ、および管理する方法
- [X] **Use AI App Templates** → Microsoft Foundry 上の AI Agent プロジェクトを迅速に立ち上げる方法
- [X] **Configure Templates** → Red Teaming、評価、トレーシング & 監視などの機能を設定する方法
- [X] **Deconstruct and Explore** → GitHub Copilot を使用して AZD テンプレートとワークフローを分解・探索する方法
- [X] **Customize and Redeploy** → AI Agent スターターテンプレートをカスタム要件に合わせてカスタマイズし再デプロイする方法

---

## ワークショップの概要

ワークショップはモジュール構成で、各モジュールはハンズオンのラボ演習を軸に進行します:

1. **Introduction** - AZD テンプレートとは何か、その仕組みを理解する
1. **Selection** - 次の AI プロジェクトに適した "starter" テンプレートを見つける
1. **Validation** - テンプレートをデプロイして検証し、期待どおりに動作することを確認する
1. **Deconstruction** - テンプレートを分解して、そのコンポーネントと設定を理解する
1. **Configuration** - Microsoft Foundry のさまざまな機能を有効化して使用する方法を学ぶ
1. **Customization** - あなたのデータと要件を考慮してアプリケーションを再構築する
1. **Teardown** - インフラをデプロビジョニングし、ソフト削除されたリソースを復元する方法を学ぶ
